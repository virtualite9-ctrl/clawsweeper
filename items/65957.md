## Summary

Keep open. Current main still lacks the PR's central direct-webchat prompt-selection fix: heartbeat `target: "none"` resolves to no delivery, and `runHeartbeatOnce` still converts that into `canRelayToUser: false` for exec/cron prompts. The route-poisoning half is partly superseded by #66073, but the direct webchat relay prompt behavior remains useful and should be reviewed as a narrowed PR.

## Best Possible Solution

Keep this PR open as an active implementation candidate. Rebase it onto current main, keep it narrowed to the direct internal webchat `target: "none"` relay-prompt exception, rely on #66073 for system-event route preservation, and run the targeted heartbeat/session tests plus the normal changed gate before merge.
