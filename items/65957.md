## Summary

Keep this PR open. Current main has absorbed the route-preservation half of the work, but it still does not implement the PR's central direct-webchat behavior: when heartbeat target resolves to `none`, exec/cron prompts are still built as internal-only even if the persisted route is a direct WebChat operator route.

## Best Possible Solution

Keep this PR open and ask for a current-main rebase that preserves the direct-WebChat `target: "none"` relay-prompt behavior with targeted tests, removes route-preservation code already covered on main, and drops unrelated wizard/setup changes unless they are still independently needed.
