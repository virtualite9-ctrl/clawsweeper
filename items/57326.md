Keep this open. Current main still has the reported helper-path CLI dispatch gaps, and open same-author PR #72499 remains an implementation candidate that explicitly targets these paths, so the issue should be resolved by reviewing/landing that PR or an equivalent patch rather than closing it now.

## Best Possible Solution

Keep this issue open and review/land #72499, or an equivalent scoped patch, that applies the existing resolveCliRuntimeExecutionProvider(...) ?? provider plus isCliProvider(...) dispatch pattern to queued followups, models status --probe, LLM slug generation, and cron. The fix should preserve runtime config, auth profile, session binding, routing/cancellation context, and add focused regression coverage for those paths.
