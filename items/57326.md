## Summary

Keep this issue open. Current main still has several reported helper paths that bypass the CLI runtime-resolution and `runCliAgent()` dispatch pattern, and open same-author PR #72499 remains an unmerged implementation candidate with closing syntax for this issue.

## Best Possible Solution

Keep this issue open and review or rebase PR #72499, or land an equivalent scoped patch. The fix should apply `resolveCliRuntimeExecutionProvider(...) ?? provider` before `isCliProvider(...)` in queued followups, auth probes, slug generation, and cron, route CLI-backed runs through `runCliAgent(...)`, preserve runtime config/auth/session/cancellation context, and add focused regression coverage for each helper path.
