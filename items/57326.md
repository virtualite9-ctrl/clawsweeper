## Summary

Keep this issue open. Current main still has the reported helper paths bypassing the CLI dispatch pattern, and same-author PR #72499 is still an open implementation candidate linked to this issue.

## Best Possible Solution

Keep this issue open and review PR #72499, or an equivalent scoped patch, that applies the existing resolveCliRuntimeExecutionProvider(...) ?? provider plus isCliProvider(...) dispatch pattern to queued followups, models status probes, LLM slug generation, and cron. The accepted fix should preserve runtime config, auth profile selection, session binding, routing/cancellation context, and add focused regression coverage for each helper path before closing this issue.
