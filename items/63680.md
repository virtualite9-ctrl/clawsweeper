## Summary

Keep open. Current main still records the requesting connection for exec approvals but does not enforce it in `exec.approval.resolve`; the same approval-capable connection can request and resolve an exec approval. The PR is a focused core gateway security fix, not ClawHub/external work, and the related earlier PRs are closed unmerged rather than canonical shipped fixes.

## Best Possible Solution

Keep #63680 open and review it as the active focused fix: add the same-connection guard in `exec.approval.resolve`, keep self-deny and null-connId compatibility if maintainers agree, update existing same-connection tests to use a separate resolver, and ensure the new regression tests clean up pending timers.
