## Summary

Current main still does not implement the requester/resolver separation that this PR proposes. The gateway records `requestedByConnId` on exec approval requests, but `exec.approval.resolve` only validates the decision against the request policy before the shared resolver calls `manager.resolve`; neither the shared resolver nor `ExecApprovalManager.resolve` blocks same-connection approval. This PR remains an active, focused implementation candidate rather than obsolete or superseded.

## Best Possible Solution

Keep this PR open. The best path is to review and land a focused gateway fix that rejects non-`deny` resolutions when the resolver connection matches `snapshot.requestedByConnId`, preserves self-deny and different-connection approval behavior, updates the current bypass test to use a separate resolver connection, keeps the focused regression tests, and runs the changed gateway checks before merge.
