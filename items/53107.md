## Summary

Current main still does not implement cron job tags/categories. The persisted cron job type, Gateway cron RPC contract, server-side list/filter options, CLI add/edit/list/show surfaces, Control UI filters/form/job cards, macOS cron model/editor, docs, and agent-facing cron tool all lack a supported `tags` field or tag-based filtering/bulk operations.

## Best Possible Solution

Keep this issue open. The best path is an additive core cron contract change: add optional normalized `tags?: string[]` to the cron job model, Gateway add/update/list schemas, persistence/normalization, agent cron tool schema, CLI add/edit/list/show and list filtering, Control UI create/edit/display/filter controls, macOS cron model/editor if that surface remains supported, and docs/tests. Tag-based bulk enable/disable can either land with the same list/update contract or be split into a narrower follow-up after the core tags field ships.
