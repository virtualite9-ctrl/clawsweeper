## Summary

Current main still does not implement cron job tags/categories. The prior ClawSweeper keep-open comment remains accurate against 52249927ac2dc579a2f8e72aa99a30e0ab4f9a11: tags are absent from the persisted cron model, Gateway contract/list filtering, CLI, Control UI form/list filters, agent cron tool schema, and cron docs.

## Best Possible Solution

Keep this issue open. The best implementation is an additive core cron contract change: add normalized optional tags?: string[] to the cron job model, persistence/normalization, Gateway add/update/list schemas and tag filtering, agent cron tool schema/recovery, CLI add/edit/list/show and tag filter support, Control UI create/edit/display/filter controls, docs, and regression tests. Tag-based bulk enable/disable should either ship on top of the same list/update contract or be split into a narrower follow-up after tags are persisted and filterable. This belongs in core, not ClawHub, because the requested behavior is organization and filtering for OpenClaw's built-in scheduler state across core APIs and first-party UI/CLI surfaces.
