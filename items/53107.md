## Summary

Keep this issue open. Current main still lacks a structured cron tags/categories contract across persistence, Gateway validation, server filtering, CLI, agent tool, Control UI, and docs. The prior ClawSweeper keep-open comment remains accurate against current main SHA 0835f9409aac0f6048c7d03ec5a2148b7e5001c5; the related reports cover adjacent cron history/UI/hooks/catch-up work, not this tag schema and filtering request.

## Best Possible Solution

Keep this issue open. The best path is an additive core cron contract change: add normalized optional tags?: string[] to the cron job model and persistence, accept and validate it in Gateway add/update/list schemas, implement tag filtering in cron.list/listPage, expose create/edit/list/show and tag filter controls in the CLI and Control UI, update the agent cron tool schema/description, document the feature on the public cron docs, and add focused regression tests. Tag-based bulk enable/disable can follow after the foundational tags contract ships or be tracked as a narrower follow-up.
