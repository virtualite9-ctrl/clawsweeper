## Summary

Keep PR #69567 open. Current main still has custom checkout blocks that derive a base64 Basic-auth header from `github.token` and pass it to `git` via `http.extraheader` without registering the derived value with `::add-mask::`. The PR is a focused CI/security hardening change against a real remaining workflow surface, and the provided discussion supports the proposed one-line-per-block fix rather than showing it is obsolete or superseded.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land a minimal workflow hardening change that registers each derived `auth_header` with `::add-mask::` before any `git` invocation, or to fold the duplicated checkout logic into a shared helper later if maintainers want to reduce repeated CI script blocks.
