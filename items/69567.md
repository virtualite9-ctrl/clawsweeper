## Summary

Keep PR #69567 open. Current main still constructs base64 `x-access-token` checkout headers in `.github/workflows/ci.yml` and uses them as `AUTHORIZATION: basic` extraheaders without any `::add-mask::` registration. The PR is a focused security hardening with positive review/context, not obsolete or implemented on main.

## Best Possible Solution

Keep the PR open and review it as a plausible security hardening. The best maintainer path is to add `echo "::add-mask::${auth_header}"` immediately after every derived checkout header is constructed, including deciding whether `.github/workflows/ci-check-testbox.yml` should be covered, then rely on the existing workflow audit lane and normal PR CI before merge.
