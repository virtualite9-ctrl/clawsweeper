## Summary

Keep #71689 open. The issue is recent, has a concrete malformed `tasks/runs.sqlite` failure mode, and current `main` still only logs restore failures rather than snapshotting, recovering, rebuilding, or surfacing an operator event for task-registry SQLite corruption.

## Best Possible Solution

Keep this open and implement the recovery in core task registry storage, likely around `src/tasks/task-registry.store.sqlite.ts` plus the restore/operator event path in `src/tasks/task-registry.ts`: detect SQLite corruption errors, close the cached DB, preserve the bad DB/WAL/SHM files, attempt a conservative recover or rebuild-empty fallback with clear operator diagnostics, and add a focused regression test using a deliberately malformed `runs.sqlite`.
