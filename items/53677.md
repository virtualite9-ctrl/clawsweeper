## Summary

Current `main` does not implement this optimization for regular memory-file reads. The active host-backed `readMemoryFile()` path still reads the entire file with `fs.readFile(...)` before slicing, so there is not strong evidence to close this PR as already implemented or obsolete.

## Risks / Open Questions

- The PR was authored against an older path (`src/memory/read-file.ts` in the GitHub context), while current `main` has the relevant logic in `src/memory-host-sdk/host/read-file.ts`, so landing it now likely requires a rebase/adaptation rather than a simple merge.
- Because the optimization is only present in the QMD backend today, closing this PR would risk dropping a still-unimplemented performance improvement for the standard memory-file backend.
