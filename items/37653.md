## Summary

Current `main` does not already contain this docs update, and the proposed PR is not accurate enough to close as obsolete: `docs/tools/web.md` still documents the older parameter set, while runtime code still supports `language`/`date_after`/`date_before` in provider-specific ways and only Brave accepts date-range `freshness` syntax. This PR still needs maintainer follow-up rather than conservative closure.
