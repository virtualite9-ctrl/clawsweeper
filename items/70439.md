## Summary

Keep open. Current main partially addresses Signal audio metadata: Signal now fetches attachments, forwards MediaPath/MediaType/MediaPaths/MediaTypes, and documents filename MIME fallback for voice notes. The requested Signal audio preflight hook is still missing: Signal mention gating runs before attachment fetch/transcription, Signal does not call transcribeFirstAudio, and there is no Signal config surface for channels.signal.audio/preflight_cmd. The maintainer comment on the issue also explicitly kept it open for this same gap.

## Best Possible Solution

Keep the issue open. The best product/code path is a shared channel audio-preflight helper or seam wired into Signal after access checks but before mention/command gating, using the existing tools.media.audio/transcribeFirstAudio pipeline. Signal should use the transcript for agent/command text when appropriate, set Transcript and MediaTranscribedIndexes, preserve the current metadata path, and document template variables only if maintainers intentionally add an external command-hook surface.
