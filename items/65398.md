## Summary

Keep open. Current main still lacks both Feishu fixes described by this PR, and the earlier related PR #56520 is closed/unmerged rather than a canonical shipped fix. The PR is also not ready to close as implemented because its module-level registration guard would conflict with plugin reload semantics that clear registered state.

## Best Possible Solution

Keep this PR open but require revision before merge: preserve Feishu top-level shared policy fields for named-account configs through the setup-promotion contract, add regression coverage for the legacy migration case, and replace the process-wide Feishu registration guard with registry-load-scoped idempotency or duplicate-safe registration that still re-registers after activating reloads. If a newer PR takes over both fixes, close this PR as superseded by that canonical item.
