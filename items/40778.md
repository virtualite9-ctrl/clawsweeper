## Summary

Current main treats new channel integrations as plugin work, not core, and now documents an existing WeCom community plugin (`@wecom/wecom-openclaw-plugin`) that covers this request.

## Risks / Open Questions

- The repo evidence shows scope and documentation, not a direct runtime verification of the external `@wecom/wecom-openclaw-plugin` package itself.
- The WeCom listing appears to be newer than the latest tagged release, so the close rationale is scope/community-plugin based rather than a shipped core implementation.

## Close Comment

Closing as `clawhub` after a Codex review. Current main is explicit that optional integrations like new channels should usually live as plugins rather than be merged into OpenClaw core (`VISION.md:54-61`), and the plugin SDK/docs already treat messaging channels as first-class external plugin work (`docs/plugins/sdk-channel-plugins.md:11-35`, `docs/tools/plugin.md:11-16`). On current `main`, the community plugin catalog now includes a dedicated WeCom channel plugin from the Tencent WeCom team with install instructions for `@wecom/wecom-openclaw-plugin` (`docs/plugins/community.md:115-127`, added on main in `84fc00af`). Given that this request is now covered by the plugin path the project wants for optional channel integrations, this belongs in ClawHub/community-plugin space rather than OpenClaw core.
