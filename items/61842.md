## Summary

Close as obsolete. Current `main` no longer routes exec preflight through the approval-unavailable reply path this PR patches: YOLO host exec skips that preflight layer entirely, remaining preflight failures throw direct `exec preflight:` errors, and the current approval-unavailable unions still only model real approval-routing failures.

## Evidence

- **YOLO host exec now skips exec preflight:** `shouldSkipExecScriptPreflight` returns true for gateway `security=full` plus `ask=off`, and the main exec path checks that guard before any preflight validation. A current test asserts that the same YOLO mode does not surface an `exec preflight:` message and instead returns a normal completed/failed result.
