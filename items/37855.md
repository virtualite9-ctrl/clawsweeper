## Summary

Keep open. Current `main` at `55389f4be29359e2dd57ebaf548ba4a70331a445` does not implement cron lifecycle events in the internal hook system: the allowed internal hook event types still exclude `cron`, the user docs still list no `cron:*` internal hook events, and `src/gateway/server-cron.ts` still only broadcasts cron events and handles finished-job webhooks instead of dispatching internal hooks.

## Evidence

- **Internal hook types exclude cron:** `src/hooks/internal-hook-types.ts` defines `InternalHookEventType` as only `command | session | agent | gateway | message`; there is no `cron` event type.
  - file: [src/hooks/internal-hook-types.ts:1](https://github.com/openclaw/openclaw/blob/55389f4be29359e2dd57ebaf548ba4a70331a445/src/hooks/internal-hook-types.ts#L1)
  - sha: [55389f4be293](https://github.com/openclaw/openclaw/commit/55389f4be29359e2dd57ebaf548ba4a70331a445)
- **Docs still omit cron events:** `docs/automation/hooks.md` lists the supported internal hook events (`command:*`, `session:*`, `agent:bootstrap`, `gateway:startup`, `message:*`) and does not list any `cron:*` events.
  - file: [docs/automation/hooks.md:34](https://github.com/openclaw/openclaw/blob/55389f4be29359e2dd57ebaf548ba4a70331a445/docs/automation/hooks.md#L34)
  - sha: [55389f4be293](https://github.com/openclaw/openclaw/commit/55389f4be29359e2dd57ebaf548ba4a70331a445)
- **Cron gateway path does not trigger internal hooks:** `src/gateway/server-cron.ts` `onEvent` still calls `params.broadcast("cron", evt, ...)` and then handles only `finished` webhook/failure-delivery behavior; there is no `createInternalHookEvent(...)`, `triggerInternalHook(...)`, or `fireAndForgetHook(...)` call in this path.
  - file: [src/gateway/server-cron.ts:417](https://github.com/openclaw/openclaw/blob/55389f4be29359e2dd57ebaf548ba4a70331a445/src/gateway/server-cron.ts#L417)
  - sha: [55389f4be293](https://github.com/openclaw/openclaw/commit/55389f4be29359e2dd57ebaf548ba4a70331a445)
- **Current internal-hook call sites are elsewhere, not cron:** A repo search for `triggerInternalHook|createInternalHookEvent|fireAndForgetHook` shows internal hook dispatch in gateway startup and session/command flows, but not in `src/gateway/server-cron.ts`. Example existing call sites: `src/gateway/server-startup-post-attach.ts:274` and `src/gateway/session-reset-service.ts:488`.
  - command: `rg -n 'triggerInternalHook|fireAndForgetHook|createInternalHookEvent' src/gateway/server-cron.ts src/hooks src/gateway`
  - sha: [55389f4be293](https://github.com/openclaw/openclaw/commit/55389f4be29359e2dd57ebaf548ba4a70331a445)
