## Summary

Keep #37855 open. Current main still has no first-class `cron` internal hook event type, and the gateway cron `onEvent` path broadcasts cron events plus finished-job webhook/run-log handling without bridging `CronEvent` into `createInternalHookEvent` or `triggerInternalHook`. The earlier same-scope implementation PRs #37881 and #37885 were closed unmerged, and the later same-scope #47903 is closed, so this issue remains the best canonical tracker.

## Best Possible Solution

Keep #37855 open as the canonical cron internal-hooks lifecycle request. The concrete path is an additive core change: add `cron` to `InternalHookEventType`, define a typed `CronEvent`-based hook context, dispatch gateway cron `onEvent` data through `createInternalHookEvent` plus non-blocking hook delivery, document `cron`, `cron:added`, `cron:updated`, `cron:removed`, `cron:started`, and `cron:finished` on https://docs.openclaw.ai/automation/hooks, and add focused hook/gateway tests for mutation and execution events.
