## Summary

Keep #37855 open. Current main still does not expose cron lifecycle events through the Internal Hooks contract: `cron` is absent from `InternalHookEventType`, the cron gateway `onEvent` bridge only broadcasts WebSocket events and finished-job webhook/failure handling, and the hooks docs/tests do not document or cover `cron:*` events. The related implementation PRs called out in the discussion, #37881 and #37885, were closed unmerged, and #47903 is also closed, so #37855 remains the best canonical tracker.

## Best Possible Solution

Keep #37855 open as the canonical cron Internal Hooks lifecycle request. The concrete implementation should add `cron` to `InternalHookEventType`, define a typed `CronEvent`-based hook context, dispatch Gateway cron `onEvent` data through `createInternalHookEvent` plus non-blocking hook delivery, document `cron`, `cron:added`, `cron:updated`, `cron:removed`, `cron:started`, and `cron:finished` on https://docs.openclaw.ai/automation/hooks, and add focused hook/gateway tests for mutation and execution events.
