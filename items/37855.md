## Summary

Keep #37855 open. Current main still emits cron lifecycle events inside CronService and broadcasts them through the gateway WebSocket/webhook path, but the internal hook contract still has no cron event type and the gateway does not bridge CronEvent into triggerInternalHook. The same-scope implementation PRs #37881 and #37885 are closed unmerged, and the later same-scope #47903 is closed, so this remains the best canonical tracker.

## Best Possible Solution

Keep #37855 open as the canonical internal-hooks cron lifecycle request. The concrete path is an additive core change: add a typed cron internal hook event based on CronEvent, dispatch server-cron onEvent data through createInternalHookEvent plus non-blocking fireAndForgetHook/triggerInternalHook handling, document cron, cron:added, cron:updated, cron:removed, cron:started, and cron:finished in docs/automation/hooks.md, and add focused hook/gateway tests covering mutation and execution events.
