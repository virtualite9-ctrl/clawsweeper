Current main still has the legacy `warnWhenAllowlistIsOpen` behavior: it returns immediately for any non-empty `plugins.allow`, so a populated-but-mismatched allowlist still gets no specific mismatch warning. The PR remains a focused implementation candidate for the linked warning-DX bug, with the main landing concerns being review/test freshness, changelog attribution, and whether it should avoid closing the broader linked issue outright.
