## Summary

Keep #16387 open. Current main still treats configured provider headers as static strings, the Fireworks plugin does not add `x-session-affinity`, and the newer provider-owned turn-state hooks do not implement generic `models.providers.*.headers` or `models.providers.*.request.headers` templating for custom config providers. The prior issue discussion and closed unmerged PR attempts are useful evidence, but no current-main implementation or canonical superseding item closes the remaining request.

## Best Possible Solution

Keep the issue open. The best maintainer path is a narrow, documented resolver for dynamic values in `models.providers.*.headers` and likely `models.providers.*.request.headers` at request dispatch, or an explicit product decision to handle this provider-by-provider through `resolveTransportTurnState`. If config templating is chosen, cover at least session identity, `agent.id`, and run/turn identity, with tests for Fireworks `x-session-affinity`, LiteLLM/Langfuse-style headers, main turns, compaction, and OpenAI-compatible transports.
