## Summary

Keep #16387 open. Current main still treats `models.providers.*.headers` and `models.providers.*.request.headers` as static strings, the bundled Fireworks plugin does not set `x-session-affinity`, and the provider-owned turn-state hooks are not a generic config templating solution for custom providers. The issue discussion adds concrete Fireworks and LiteLLM/Langfuse use cases, and the related implementation PRs #16448 and #18213 were closed unmerged rather than shipped.

## Best Possible Solution

Keep the issue open. The best path is a narrow, documented dynamic-value resolver for model provider config headers, likely covering both `models.providers.*.headers` and `models.providers.*.request.headers` at request dispatch. It should define supported placeholders for session identity, agent identity, and per-turn/run identity, then add tests for Fireworks `x-session-affinity`, LiteLLM/Langfuse-style headers, main turns, compaction, and OpenAI-compatible transports. If maintainers prefer provider-owned behavior only, that should be an explicit product decision and should document how custom providers can achieve the same result.
