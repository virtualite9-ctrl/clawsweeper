## Summary

Keep #16387 open. Current main still treats `models.providers.*.headers` and `models.providers.*.request.headers` as static/literal configuration, and the bundled Fireworks plugin does not add `x-session-affinity` or a generic runtime template resolver for `{{session.key}}`, `{{agent.id}}`, or `{{run.id}}`.

## Best Possible Solution

Keep #16387 open. The best maintainer path is a narrow, documented runtime header-template resolver for `models.providers.*.headers` and `models.providers.*.request.headers`, evaluated at request dispatch with explicit placeholders for session key, agent identity, and per-turn/run identity. Add regression coverage for Fireworks `x-session-affinity`, LiteLLM/Langfuse-style headers, main turns, compaction, retries, and OpenAI-compatible transports. If maintainers prefer provider-owned behavior only, document that decision and the plugin/API path that replaces custom-provider config templating.
