## Summary

Keep PR #71843 open. Current main still lets the Slack agent `react` tool await `reactSlackMessage()` without catching Slack `already_reacted`, while the PR is a narrow fix with targeted regression coverage for linked issue #69005.

## Best Possible Solution

Keep #71843 open for maintainer review. Land this PR or an equivalent Slack plugin patch that treats Slack `already_reacted` as idempotent success for the agent `react` tool, returns the normal `{ ok: true, added: emoji }` payload, preserves propagation for unrelated Slack errors, and keeps focused regression tests. If maintainers prefer a broader helper-level solution, explicitly supersede it with #58242/#50881 or a new canonical PR rather than closing this useful fix as stale.
