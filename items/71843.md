## Summary

Keep PR #71843 open. Current main still propagates Slack `already_reacted` from the agent-facing `react` action, while the PR is a focused, still-open fix with regression coverage for linked issue #69005. The status-reaction path already treats duplicate reaction adds as a no-op, so this is a real Slack behavior gap rather than stale cleanup.

## Best Possible Solution

Keep #71843 open for maintainer review. Land this PR or an equivalent Slack plugin patch that treats Slack `already_reacted` as idempotent success for the agent `react` tool, returns the normal `{ ok: true, added: emoji }` payload, preserves propagation for unrelated Slack errors, and keeps focused regression tests. If maintainers prefer a broader helper-level fix, explicitly supersede it with #58242/#50881 or a new canonical PR rather than closing this useful fix as stale.
