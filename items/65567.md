## Summary

Keep this PR open. Current main still has the unguarded `hashText(event.text)` and downstream `emitSystemStatus(event.text)` paths, and the PR discussion shows the author updated the branch to cover both crash sites after review. The linked issue was closed as superseded by this PR, not because main implemented the fix.

## Best Possible Solution

Keep this PR open and review it as a real fix candidate. The best path is to land this branch or a superseding patch that guards both the repeat-suppression hash and the status emission path, with a focused regression test for a `usage_update` status event missing `text`. A slightly stronger version would also align the internal ACP runtime event contract or normalize textless ACP status updates at the runtime boundary.
