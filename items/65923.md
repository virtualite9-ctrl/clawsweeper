## Summary

Current main still matches the report: omitted `session.reset` resolves to `daily` at 4:00 AM, the inbound session path applies that policy to roll sessions, and tests explicitly preserve the default. The newer commits since the prior Codex review only touched unrelated compaction docs/help, so the issue remains a concrete product/UX decision rather than an implemented, duplicate, or external-only cleanup candidate.

## Best Possible Solution

Keep this open as a focused product/UX bug. Maintainers should decide the intended contract for omitted `session.reset`: either add a true no-automatic-reset/manual-only mode and update schema, exported types, runtime, tests, config help, and docs, or deliberately retain the implicit daily reset and make it highly visible in FAQ/config/status/setup/startup surfaces. Coordinate with broader daily-reset UX work such as #61238, but do not close this narrower unset-default report until that contract is implemented or explicitly superseded.
