## Summary

Keep open. Current `main` partially addresses the usability side with server-side allowlist filtering for `models.list`, but it does not implement the requested auth-gated visibility or the proposed configurable filter modes.

## Evidence

- **Current server behavior is allowlist-based, not auth-based:** `models.list` now loads the catalog, runs `buildAllowedModelSet`, and returns the filtered result as the runtime-allowed catalog.
  - file: [src/gateway/server-methods/models.ts:13](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/src/gateway/server-methods/models.ts#L13)

  - sha: [ec3dbd22a4de](https://github.com/openclaw/openclaw/commit/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46)
- **Empty allowlist still exposes the full catalog:** `buildAllowedModelSetWithFallbacks` derives visibility from `cfg.agents?.defaults?.models`; when that map is empty it sets `allowAny = true` and returns the full catalog. That is materially different from the issue's requested `authenticated` / `configured` modes.
