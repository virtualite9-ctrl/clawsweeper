## Summary

Keep open. Current `main` does not expose an `entities` parameter on the shared `message` tool, the Telegram plugin does not add one, and the Telegram action/send path only forwards text plus a small fixed set of options. There is lower-level HTML-mode support in the Telegram sender, but that mode is not exposed through the `message` tool, so this issue's requested capability is not implemented on `main`.

## Evidence
