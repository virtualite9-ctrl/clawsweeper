## Summary

Current main already solves the Slack multi-agent thread wakeup problem with the shipped `channels.slack.thread.requireExplicitMention` setting. When enabled, Slack message handling suppresses implicit thread mention kinds, docs describe the behavior, tests cover drop/allow/default cases, and the changelog records the feature under `2026.4.7`.

## Best Possible Solution

Close as implemented and keep the shipped `channels.slack.thread.requireExplicitMention` control. If maintainers later want `requireMention: true` itself to imply this behavior by default, that should be a separate compatibility/product decision against the current Slack config surface.
