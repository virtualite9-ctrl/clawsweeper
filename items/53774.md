## Summary

Keep open. Current main still has a fixed TUI footer and no footer template, custom footer fields, cost/balance/cooldown footer display, or plugin status-provider seam wired into TUI rendering. Existing usage and quota surfaces are adjacent but not the requested persistent customizable footer.

## Best Possible Solution

Keep this issue open. The maintainer path is to either implement a real TUI footer configuration surface across `src/tui/tui.ts`, `TuiOptions`, CLI/config/docs, and focused tests, or split it into a canonical footer-template issue plus a reviewed plugin status-provider API that the TUI render path actually consumes. Existing `/status`, `/usage`, and provider quota code should be treated as data sources, not as an implementation of the persistent customizable footer.
