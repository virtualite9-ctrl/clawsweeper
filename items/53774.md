## Summary

Keep open. Current `main` has related cost/quota surfaces, but it still does not implement a configurable TUI footer or any footer hook/template for cost, balance, or cooldown fields.

## Evidence

- **TUI footer is hard-coded:** `updateFooter()` builds a fixed footer from agent, session, model, think/fast/verbose/reasoning, and token counts, then joins those parts directly. There is no config/template or plugin hook in this path.
  - file: [src/tui/tui.ts:816](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/tui/tui.ts#L816)

  - file: [docs/web/tui.md:51](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/docs/web/tui.md#L51)

## Risks / Open Questions

- The issue body's 'current behavior' is stale: current main already shows a richer footer than the report describes, so any future implementation should likely re-scope the request around customization rather than basic visibility.
- Balance and cooldown data are provider-specific and unevenly available, so a real implementation needs a product decision on which providers/fields are mandatory versus best-effort.
