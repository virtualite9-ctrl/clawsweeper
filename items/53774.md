## Summary

Current main has adjacent usage, cost, quota, and rate-limit reporting, but it still does not implement configurable TUI footer fields or a plugin/template hook for cost, balance, or cooldown status. This remains a concrete core TUI and plugin-API feature request, not a conservative cleanup close.

## Best Possible Solution

Keep the issue open or manually consolidate it with a chosen canonical TUI-footer item. The likely product path is a core TUI footer configuration surface plus, if plugins should supply live cost/balance/cooldown data, a documented footer/status contribution provider wired into src/tui/tui.ts and the public plugin SDK docs.
