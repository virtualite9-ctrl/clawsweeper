## Summary

Keep open. Current main still hard-codes the TUI footer and has no footer template, footer fields, or plugin status-provider seam wired into the TUI render path. Existing usage/cost support is separate command and per-response footer behavior, not the persistent customizable footer requested here. The open README-only PR #72483 also uses closing syntax, so this issue should remain open until a real implementation is reviewed and merged.

## Best Possible Solution

Keep this issue open. The best maintainer path is to close or rewrite #72483, then either implement a real TUI footer configuration surface across `src/tui/tui.ts`, `TuiOptions`, CLI/docs, and tests, or split the work into a canonical footer-template issue plus a reviewed plugin status-provider API such as #64086/#72557. Existing `/status`, `/usage full`, `/usage cost`, and provider usage snapshot plumbing can supply some data, but the persistent TUI footer customization still needs a core render-path and configuration/API decision.
