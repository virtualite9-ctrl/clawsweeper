## Summary

Keep open. The issue asks for `openclaw doctor` to detect stale persisted `skillsSnapshot` metadata whose cached skill paths point at an old runtime root, deleted temp-backed runtime, or bundled-skill path that disagrees with the current runtime. Current `main` still has adjacent session/skills maintenance, but I found no doctor contribution or test that scans persisted session `skillsSnapshot.prompt` or `resolvedSkills` for this runtime-root drift, and the report’s verified manual remediation remains a real product gap rather than something already implemented or superseded.

## Best Possible Solution

Keep the issue open and implement a focused doctor health contribution, likely near the existing state/session integrity checks. It should scan all agent session stores for `skillsSnapshot.prompt` and `skillsSnapshot.resolvedSkills` paths, resolve the active OpenClaw runtime/bundled-skills root, flag missing or inactive roots and temp-backed runtime paths, clearly report live runtime healthy versus cached session metadata stale, and in repair mode back up the store and remove only the affected `skillsSnapshot` fields so they rebuild on the next turn. Add regression tests with stale bundled paths and temp-backed cached paths.
