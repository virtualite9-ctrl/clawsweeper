## Summary

Keep #39605 open. Current main at 406ae72fd2789c78053f6b94006ec8f488bc1c00 still routes Discord native commands and Slack slash commands through startup-captured config, while resolveAgentRoute derives DM session keys from the supplied cfg.session.dmScope. Telegram has a fresh-runtime-config pattern, but the Discord and Slack surfaces named by the issue remain unresolved; the supplied GitHub context also shows PR #39617 is still the open fix attempt for this exact bug.

## Best Possible Solution

Keep #39605 open and land PR #39617 or an equivalent maintainer-reviewed fix that supplies fresh runtime config before Discord native command route resolution and Slack slash command route resolution. Add focused Discord and Slack regression tests proving DM slash/native commands honor updated session.dmScope without recreating the provider. Telegram can remain on its existing fresh-runtime-config pattern unless maintainers want the issue narrowed after the Discord/Slack fix lands.
