## Summary

Keep #39605 open. Current main still routes Discord native commands and Slack slash commands using the monitor startup config object, while route/session-key resolution still derives DM session keys from the supplied `cfg.session.dmScope`. Telegram has a fresh-runtime-config fix and regression coverage, but the issue’s Discord and Slack surfaces remain unresolved; the linked PR #39617 is still the canonical open fix attempt in the provided context.

## Best Possible Solution

Keep #39605 open and land #39617 or an equivalent maintainer-reviewed fix that reloads or otherwise supplies current runtime config before Discord native command route resolution and Slack slash command route resolution. Add Discord and Slack regression tests matching the Telegram coverage, proving native/slash command routing honors changed `session.dmScope` or route bindings without recreating the provider.
