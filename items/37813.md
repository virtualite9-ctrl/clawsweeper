## Summary

Keep #37813 open. Current main has addressed meaningful parts of the report, including configured fallback-chain handling and model-not-found visibility, but the security-relevant tool-policy request remains unresolved: tool policy is still resolved from the active fallback candidate provider/model rather than from the originally requested model whose tools.byProvider deny list the reporter configured.

## Best Possible Solution

Keep #37813 open until maintainers either implement requested-model tool-policy preservation across fallback attempts or explicitly document that fallback attempts use the active fallback candidate's policy. A code fix would carry the originally requested provider/model, or a pre-resolved requested-model tool policy, through runWithModelFallback into runAgentAttempt/runEmbeddedPiAgent and both createOpenClawCodingTools and applyFinalEffectiveToolPolicy, with regression coverage proving tools.byProvider[requestedProvider/requestedModel].deny remains enforced when the run falls back to another provider/model.
