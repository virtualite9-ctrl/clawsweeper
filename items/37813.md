## Summary

Keep #37813 open. Current main has addressed meaningful parts of the report, including configured fallback-chain handling and newer Gemini/xAI compatibility, but the remaining security-relevant request is still not implemented: tool policy is resolved from the active fallback model, not from the originally requested model whose `tools.byProvider` deny list the reporter expects to preserve.

## Best Possible Solution

Keep #37813 open until maintainers either implement requested-model tool-policy preservation across fallback attempts or explicitly document that fallback attempts use the active fallback candidate's policy. A code fix would carry the originally requested provider/model, or a pre-resolved requested-model tool policy, through `runWithModelFallback` into embedded tool creation and bundled-tool filtering, then add regression coverage proving a `tools.byProvider[requestedProvider/requestedModel].deny` entry remains enforced when the run falls back to another provider/model.
