## Summary

Keep #37813 open. Current main has partially addressed the report by honoring explicit fallback overrides and improving model-not-found visibility, and the NVIDIA comment provides a useful provider-specific workaround, but the security-relevant tool-policy part remains unresolved: fallback attempts still resolve `tools.byProvider` from the active fallback provider/model, not from the originally requested model whose deny list the reporter configured.

## Best Possible Solution

Keep #37813 open until maintainers either preserve requested-model tool policy across model fallback or explicitly document that fallback attempts use the active fallback candidate's policy. A code fix should carry the originally requested provider/model, or a pre-resolved requested-model tool policy, through `runWithModelFallback` into `runAgentAttempt`, `runEmbeddedPiAgent`, `createOpenClawCodingTools`, and `applyFinalEffectiveToolPolicy`, with regression coverage proving `tools.byProvider[requestedProvider/requestedModel].deny` remains enforced after fallback to another provider/model.
