## Summary

Keep #47849 open. Current main still treats Control UI gateway auth as shared-secret or external identity-aware proxy auth, and the session APIs/listing are not tenant- or user-scoped. The request is a real product/security feature request for built-in multi-user login plus session isolation, not something already implemented or safely closeable as stale, duplicate, external-only, or ClawHub/plugin work.

## Best Possible Solution

Keep the issue open for maintainer product/security judgment. A good implementation path would add a first-class multi-user auth/identity model, persist session ownership metadata, enforce owner/admin authorization across sessions.list/get/create/patch/delete/reset/compact/preview/subscribe and chat flows, update Control UI login/session UX, and document how this differs from trusted-proxy/Tailscale access and the current personal-assistant trust model.
