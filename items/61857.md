## Summary

Current `main` already routes `openclaw update` through the npm binary that owns the installed global OpenClaw prefix, and the changelog shows this shipped in `v2026.4.5`. The report matches older behavior on `2026.4.2`, but it is no longer an open maintainer item on current OpenClaw.

## Evidence

- **Shipped changelog entry:** `CHANGELOG.md` records this exact updater fix under release `2026.4.5`: prefer the npm binary that owns the installed global OpenClaw prefix for mixed Homebrew/nvm-style setups.
  - file: [CHANGELOG.md:1315](https://github.com/openclaw/openclaw/blob/8b76392e3e7934b1cdd6d461148263fd56141ec8/CHANGELOG.md#L1315)

  - sha: [8b76392e3e79](https://github.com/openclaw/openclaw/commit/8b76392e3e7934b1cdd6d461148263fd56141ec8)
- **npm command is derived from installed package prefix:** `inferNpmPrefixFromPackageRoot` and `resolvePreferredNpmCommand` derive the owning npm path from the installed package root and choose `/prefix/bin/npm` (or `npm.cmd`) instead of a bare `npm` lookup.
  - file: [src/infra/update-global.ts:371](https://github.com/openclaw/openclaw/blob/8b76392e3e7934b1cdd6d461148263fd56141ec8/src/infra/update-global.ts#L371)

  - sha: [8b76392e3e79](https://github.com/openclaw/openclaw/commit/8b76392e3e7934b1cdd6d461148263fd56141ec8)
- **Resolved command is threaded through global-root and install argv resolution:** `resolveGlobalRoot`, `resolveGlobalInstallTarget`, `globalInstallArgs`, and `globalInstallFallbackArgs` all use the resolved command object, so subsequent `root -g` checks and installs stay pinned to the owning npm binary.
  - file: [src/infra/update-global.ts:433](https://github.com/openclaw/openclaw/blob/8b76392e3e7934b1cdd6d461148263fd56141ec8/src/infra/update-global.ts#L433)

  - sha: [8b76392e3e79](https://github.com/openclaw/openclaw/commit/8b76392e3e7934b1cdd6d461148263fd56141ec8)
- **Updater passes package root into install-target resolution:** The package-update path calls `detectGlobalInstallManagerForRoot` and `resolveGlobalInstallTarget` with `pkgRoot`, then builds the update command from that resolved install target.
  - file: [src/infra/update-runner.ts:1285](https://github.com/openclaw/openclaw/blob/8b76392e3e7934b1cdd6d461148263fd56141ec8/src/infra/update-runner.ts#L1285)

  - file: [src/infra/update-global.test.ts:210](https://github.com/openclaw/openclaw/blob/8b76392e3e7934b1cdd6d461148263fd56141ec8/src/infra/update-global.test.ts#L210)
