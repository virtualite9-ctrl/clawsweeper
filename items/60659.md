## Summary

Keep open. Current `main` still implements session reset only as global `session.*` config with per-chat-type and per-channel overrides. There is no per-agent `agents.list.*.session` surface and no per-binding reset policy in the binding schema or session-init path, so the feature request remains unimplemented and coherent.

## Evidence

- **Per-agent config has no session override surface:** `AgentConfig` enumerates per-agent overrides such as model, thinking, heartbeat, sandbox, tools, and runtime, but no `session` field. The binding types likewise expose only routing match metadata plus optional ACP runtime fields.
