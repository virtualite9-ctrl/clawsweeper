## Summary

Current main still does not auto-render a plain chat text image URL as an inline image. The implemented paths are structured image blocks, explicit MEDIA/markdown-image extraction for assistant output, and sanitized markdown links; remote image rendering is intentionally constrained by markdown image handling and Control UI CSP, so this needs explicit product/security design rather than cleanup closure.

## Best Possible Solution

Keep this feature request open for maintainer product/security design. A good implementation would add an explicit safe image-preview path for chat messages, likely using same-origin/proxied or blob-backed rendering with scheme/host policy and tests in ui/src/ui/markdown.test.ts, ui/src/ui/chat/grouped-render.test.ts, src/media/parse.test.ts, and src/gateway/control-ui-csp.test.ts, rather than converting arbitrary plain text URLs directly to img tags.

## Risks / Open Questions

- A simple regex-to-img implementation would bypass the current deliberate distinction between plain text, markdown, structured media, and MEDIA directives.
- Remote image previews have privacy/security implications because they can expose operator browser network fetches unless routed through an explicit same-origin proxy, allowlist, or blob-based fetch path.
- Changing this behavior needs tests across markdown sanitization, grouped chat rendering, media parsing, and Control UI CSP so existing hardening is not accidentally weakened.
