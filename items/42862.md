## Summary

Keep this feature request open. Current main renders structured image blocks, transcript media, and assistant media directives, but it does not automatically promote a plain text image URL in a chat message into an inline image. The existing markdown renderer also intentionally avoids remote <img> output for ordinary markdown-rendered text, so this needs explicit product/security design rather than stale cleanup.

## Best Possible Solution

Keep open for maintainer product/security design. A good implementation would add a deliberate chat-rendering path for safe image previews, probably with explicit allowlist/CSP behavior and regression tests in ui/src/ui/markdown.test.ts and ui/src/ui/chat/grouped-render.test.ts, rather than converting arbitrary text URLs directly to <img>.

## Risks / Open Questions

- Implementing the request as a simple regex-to-<img> conversion would conflict with the current Control UI CSP and the markdown renderer's existing remote-image hardening.
- The desired behavior needs a maintainer decision on privacy, SSRF/browser-fetch exposure, CSP policy, allowed schemes/hosts, and whether this applies to user messages, assistant messages, or both.
