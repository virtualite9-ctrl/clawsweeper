## Summary

Keep open. Current main does not expose `gateway.systemEvents.recoveryNotice` or an equivalent recovery-notice policy, and the recovery paths still queue prompt-facing `System:` notices. Related items do not supersede this: #71289 narrowed whole-file recovery triggers, #49579 is an open WhatsApp-specific banner filter, and #39307 asks for more visible context-reset signaling rather than customer-facing suppression.

## Best Possible Solution

Keep the issue open for maintainer product review. If accepted, implement this as a core gateway config surface: add a typed and documented `gateway.systemEvents.recoveryNotice` mode, gate the config-recovery and restart-sentinel prompt/outbound notice paths while preserving restart continuations, and add focused tests for the default, log-only, and off modes plus docs/changelog updates.
