## Summary

Keep open. Current main still hardcodes config recovery notices into prompt-facing system events and, for restart-sentinel recovery, outbound channel delivery. The gateway config type/schema has no `gateway.systemEvents` or `recoveryNotice` field, and related items do not supersede the requested general operator-controlled knob.

## Best Possible Solution

Keep the issue open and implement this in core gateway config, not as a plugin. Add a typed/schema-backed `gateway.systemEvents.recoveryNotice` policy or equivalent, apply it to both prompt-facing `enqueueSystemEvent` and restart-sentinel outbound notice delivery, preserve restart continuations, and cover `user`, `log`, and `off` behavior with focused tests plus docs/changelog.

- The implementation needs to preserve the current default behavior for operator safety while allowing customer-facing deployments to suppress prompt/outbound notice text.
