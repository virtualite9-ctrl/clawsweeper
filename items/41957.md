## Summary

Keep open. Current main still does not implement this PR's central auto-reply behavior: typing TTL is not budgeted from model timeout/fallback attempts, timeout failures still fall through to generic failure copy, and the proposed regression coverage is absent. There is an older overlapping open PR #40760 by the same author with the same underlying patch, but without a maintainer-chosen canonical thread this should remain open for review or consolidation rather than auto-closed.

## Best Possible Solution

Keep this PR open for maintainer review or consolidation with #40760. The best implementation is a current-main rebase that computes a bounded typing TTL from the same effective fallback candidate budget used by `runWithModelFallback`, adds narrow timeout-specific final messaging if still desired, and includes targeted tests for TTL bounds, configured-primary append behavior, and timeout copy.
