## Summary

Keep open. Current `main` at `89755d1c79678610720cdb9ce504cfd62c261f64` still does not map Discord `ChannelNameChange`, `ChannelIconChange`, or `ChannelFollowAdd` into `resolveDiscordSystemEvent`, and the Discord preflight path only suppresses messages when that resolver returns a system-event string. The PR remains a plausible, narrowly scoped bug fix rather than something already implemented or obsolete.
