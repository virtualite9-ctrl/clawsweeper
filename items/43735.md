## Summary

Keep open. Current main does load workspace skills and documents prompt filtering/budgets, but the reported stale `skillsSnapshot` path is still plausible: CLI and Gateway session reuse both depend on process-local version counters, so a persisted snapshot with version 0 can still survive a restart and look fresh when workspace skills changed while no watcher was active. The related implementation PR #48089 is closed unmerged, and #44898 is closed rather than serving as an open canonical fix.

## Best Possible Solution

Keep this open until maintainers either add restart-safe invalidation for persisted CLI/Gateway `skillsSnapshot` data or explicitly narrow the documented contract. A robust fix would persist and compare a deterministic inventory fingerprint covering visible skill roots, loaded skill metadata, relevant config/filter/eligibility inputs, and prompt limits, with regression tests for an existing session whose version-0 snapshot survives a process restart after a workspace skill is added while no watcher is active.
