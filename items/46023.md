## Summary

Keep #46023 open. Current main still computes Telegram `mediaMaxBytes`, but the final-reply media delivery path does not thread that value into `deliverReplies`, `deliverMediaReply`, or `loadWebMedia`. The related fix PR #46425 was closed unmerged, and its intended change is not present in current main.

## Best Possible Solution

Keep the issue open and implement the narrow #46425-style fix on current main: add optional `mediaMaxBytes` to `deliverReplies` and `deliverMediaReply`, pass it through the normal buffered final-reply path and native-command reply paths, include `maxBytes: params.mediaMaxBytes` in `buildOutboundMediaLoadOptions`, and add regression tests for both configured and omitted caps.
