## Summary

Keep open. Current main still lacks the PR's `createHostWorkspaceReadTool`/host read operations implementation and its privacy-isolation regression test; the read tool is still guarded only when `tools.fs.workspaceOnly` resolves true. The PR remains an open implementation candidate for the linked security issue, but it is XL, bundles unrelated changes, and has unresolved review/security-policy questions, so it needs maintainer/security review rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer/security review. The best path is to split unrelated changes, decide the intended agent filesystem privacy policy and migration story, then land a narrow read-tool isolation fix in `src/agents/pi-tools.read.ts` and `src/agents/pi-tools.ts` with focused tests for cross-agent memory reads, path traversal, parent-directory access, and explicit `workspaceOnly` behavior.
