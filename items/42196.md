## Summary

Keep open. Current main still allows `openclaw cron add` without `--agent`, sends `agentId: undefined` for omitted agent selection, and runtime execution falls back to the configured default agent. Provided GitHub context also shows open PR #42245 explicitly linked as the implementation candidate, so this issue should remain open until that PR is reviewed, merged, or closed.

## Best Possible Solution

Keep this issue open and review PR #42245. The likely maintainer path is to land that PR or an equivalent focused change that warns when an agent-turn cron job is created without `--agent`, with a cron CLI regression test. If maintainers prefer stricter behavior, require `--agent` for agent-turn cron creation and update the cron CLI and scheduled-task docs.
