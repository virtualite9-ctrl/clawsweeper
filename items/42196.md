## Summary

Keep open. Current main still accepts `openclaw cron add` without `--agent`, forwards an omitted agent as `undefined`, and the cron runtime still resolves missing agent ids to the default agent. The issue also has an open implementation PR with closing syntax, so it should remain open until that PR or an equivalent fix is reviewed and merged or deliberately closed.

## Best Possible Solution

Keep this issue open and review PR #42245. The likely maintainer path is to land that PR or an equivalent focused change that warns when an agent-turn cron job is created without `--agent`, with a cron CLI regression test. If maintainers prefer stricter behavior, require `--agent` for agent-turn cron creation and update the cron CLI and scheduled-task docs.
