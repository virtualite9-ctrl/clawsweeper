## Summary

Keep open. Current main still contains the reported bypass: global `tools.exec` causes `exec`/`process` to be added back into `profileAlsoAllow`, even when the selected profile is `messaging` and explicit `alsoAllow` omits `exec`. The related fix PR #47991 was closed unmerged, and current tests still assert the problematic behavior.

## Best Possible Solution

Keep the issue open. The likely fix belongs in `src/agents/pi-tools.policy.ts`: global `tools.exec`/`tools.fs` safety configuration should not implicitly grant runtime/filesystem tool access when an explicit profile/alsoAllow excludes those tools. Preserve any intentionally supported agent-level tool-section behavior, then add regression coverage in `src/agents/pi-tools.policy.test.ts` and a model-visible tool filtering path such as `src/agents/pi-tools.ts` or `src/gateway/tool-resolution.ts`. Closed PR #47991 is useful prior art but needs maintainer review, rebase, and validation against current main.
