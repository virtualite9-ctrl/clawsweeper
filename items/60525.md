## Summary

Current `main` already handles this iMessage monitor case: RPC payloads are normalized before dispatch, normal `is_from_me: true` DM rows are dropped before entering the inbound queue, and self-chat/echo reflections are explicitly suppressed. The same monitor files are unchanged between `main` (`926068b14fc661e915a0247e9ecb5949fb95b4f7`) and release `v2026.4.23` (`a9797214338ba31c52c796adbb75afb16e0684a9`).

## Evidence

- **RPC text normalization is implemented in the current parser:** `parseIMessageNotification` validates iMessage RPC payloads and normalizes both `text` and `reply_to_text` through `stripImessageLengthPrefixedUtf8Text(...)` before enqueueing.
  - file: [extensions/imessage/src/monitor/parse-notification.ts:82](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/extensions/imessage/src/monitor/parse-notification.ts#L82)
  - sha: [926068b14fc6](https://github.com/openclaw/openclaw/commit/926068b14fc661e915a0247e9ecb5949fb95b4f7)
- **The normalizer removes the length-delimited UTF-8 wrapper from imsg text payloads:** `stripImessageLengthPrefixedUtf8Text` detects the leading protobuf-style `0x0a <len> <utf8>` wrapper and returns the inner text payload.
  - file: [extensions/imessage/src/monitor/strip-imsg-length-prefixed-text.ts:26](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/extensions/imessage/src/monitor/strip-imsg-length-prefixed-text.ts#L26)
  - sha: [926068b14fc6](https://github.com/openclaw/openclaw/commit/926068b14fc661e915a0247e9ecb5949fb95b4f7)
- **Normal outbound DM rows are dropped before they can enter the inbound queue:** `resolveIMessageInboundDecision` only treats a message as self-chat when `sender`, `chat_identifier`, and explicit `destination_caller_id` all line up; otherwise `is_from_me` returns `{ kind: "drop", reason: "from me" }`. It also separately drops self-chat reflections and echo-cache matches.
  - file: [extensions/imessage/src/monitor/inbound-processing.ts:210](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/extensions/imessage/src/monitor/inbound-processing.ts#L210)
  - sha: [926068b14fc6](https://github.com/openclaw/openclaw/commit/926068b14fc661e915a0247e9ecb5949fb95b4f7)
- **Regression tests cover wrapper stripping and the from-me/self-chat/echo paths:** The monitor tests assert wrapper stripping for `text`/`reply_to_text`, dispatch of real self-chat, dropping normal `is_from_me` outbound DMs, and dropping self-chat agent echoes/reflections.
  - file: [extensions/imessage/src/monitor/self-chat-dedupe.test.ts:341](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/extensions/imessage/src/monitor/self-chat-dedupe.test.ts#L341)
  - sha: [926068b14fc6](https://github.com/openclaw/openclaw/commit/926068b14fc661e915a0247e9ecb5949fb95b4f7)
- **The shipped release already contains the same relevant implementation:** No relevant monitor-file changes exist between release `v2026.4.23` and current `main` for `parse-notification.ts`, `strip-imsg-length-prefixed-text.ts`, `inbound-processing.ts`, `parse-notification.test.ts`, and `self-chat-dedupe.test.ts`.
  - command: `git diff --name-only a9797214338ba31c52c796adbb75afb16e0684a9..926068b14fc661e915a0247e9ecb5949fb95b4f7 -- extensions/imessage/src/monitor/parse-notification.ts extensions/imessage/src/monitor/strip-imsg-length-prefixed-text.ts extensions/imessage/src/monitor/inbound-processing.ts extensions/imessage/src/monitor/parse-notification.test.ts extensions/imessage/src/monitor/self-chat-dedupe.test.ts`
