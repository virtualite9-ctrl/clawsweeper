## Summary

Keep #13304 open. Current main still does not support content-based agent routing in bindings: binding types/schema/docs list only static channel/account/peer/guild/team/role fields, route resolution receives no message text to match or strip, and existing plugin hooks can claim or short-circuit messages but cannot return a normal target agent plus transformed BodyForAgent. The prior issue comment already reached the same keep-open conclusion, and the linked #10969/#11220 are model-routing requests rather than canonical trackers for agent routing.

## Best Possible Solution

Keep the issue open. Implement either bindings[].match.content with deterministic prefix/regex matching and stripMatch behavior, or add a documented pre-routing plugin hook that can return agentId, session behavior, and transformed BodyForAgent before normal dispatch. Update config types, strict zod schema, generated config help/docs, resolver precedence, representative channel call sites, and route/channel regression tests.
