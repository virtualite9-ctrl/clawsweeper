## Summary

Keep #13304 open. Current main still routes bindings by static channel/account/peer/guild/team/role metadata only; it has no `bindings[].match.content`, regex/prefix matcher, or `stripMatch` behavior, and the available plugin hooks cannot select a different configured agent with a transformed `BodyForAgent` for normal dispatch. The prior issue discussion also explicitly kept this open, and the linked #10969/#11220 are model-routing items, not canonical trackers for content-based agent routing.

## Best Possible Solution

Keep the issue open. Implement either `bindings[].match.content` with deterministic prefix/regex semantics and `stripMatch`, or a documented pre-routing plugin hook that can return target `agentId`, session behavior, and transformed `BodyForAgent`. Update config types, strict zod schema, generated config help/docs, resolver precedence, representative channel call sites, and route/channel regression tests.
