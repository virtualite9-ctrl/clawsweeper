## Summary

Keep open. Current main still does not implement Android/Termux gateway service installation or a daemon mode for unsupported platforms. The CLI service layer is limited to darwin/linux/win32 adapters and still throws `Gateway service install not supported on ${process.platform}` for anything else, which includes Node on Android/Termux. Existing docs mention foreground `openclaw gateway run` and Android as a companion node, but they do not document Termux gateway hosting or a Termux service workaround for this report.

## Best Possible Solution

Keep this open for maintainer/product handling. The narrow code path would be a Termux-aware service adapter in the daemon service layer, probably wiring `termux-services` or another Android-safe supervisor into `src/daemon/service.ts` plus tests and CLI/docs updates. If Android/Termux gateway hosting is intentionally unsupported, the best lower-scope fix is to document that explicitly in the public install/gateway/Android docs and give a supported foreground/manual-supervisor workaround instead of leaving users to infer it from the unsupported-platform error.
