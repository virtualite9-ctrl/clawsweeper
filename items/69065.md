## Summary

Keep open. Current main still has no Android/Termux gateway service adapter: the managed service registry is limited to macOS launchd, Linux systemd, and Windows Scheduled Tasks, and unsupported platforms still throw the exact reported error. The docs say Android is a companion node and the Gateway should run on macOS/Linux/Windows, but they do not resolve the Termux CLI hosting request with either Termux service support or explicit foreground-mode guidance for Android/Termux.

## Best Possible Solution

Keep #69065 open until maintainers choose and implement the Android/Termux boundary. The likely path is either a focused Termux service adapter in the gateway service layer, or explicit docs and CLI messaging that Android/Termux must run `openclaw gateway run` under a user-managed foreground/supervisor setup. Add regression coverage around Android/unsupported `process.platform` service commands so `gateway start/install/status` fail with actionable guidance instead of the current bare unsupported-platform error.
