## Summary

Keep open. Current main still does not support Android/Termux as a managed Gateway service platform: the service registry is limited to macOS, Linux, and Windows, and unsupported platforms still throw the exact `Gateway service install not supported on <platform>` error. The docs now frame Android as a companion node and list service targets for macOS/Linux/WSL2/Windows, but they do not provide Termux service support or a Termux-specific supported foreground/supervisor runbook for the reported workflow.

## Best Possible Solution

Keep #69065 open. Maintainers should either add an Android/Termux service adapter or fallback path, such as termux-services support, or explicitly document that Android/Termux Gateway hosting is unsupported and recommend a supported foreground/supervisor workflow (`openclaw gateway run`, tmux, or external Termux supervision) with CLI errors/hints that point users there.
