## Summary

Current `main` already supersedes this PR: the bundled ACPX plugin now delegates runtime behavior to `acpx/runtime` 0.5.3, and the shipped runtime no longer contains the old in-repo `--format json --json-strict` arg-building path this PR was patching.

## Evidence

- **ACPX runtime is now delegated upstream:** Current `main` imports `AcpxRuntime` from the published `acpx/runtime` package instead of maintaining the old in-repo verb/prompt arg builder that this PR edited.
  - file: [extensions/acpx/src/runtime.ts:1](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/extensions/acpx/src/runtime.ts#L1)

  - file: [extensions/acpx/src/service.ts:43](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/extensions/acpx/src/service.ts#L43)
