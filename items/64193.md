## Summary

Keep open. Current main still lacks the PR's StepFun follow-up changes: cross-surface onboarding refresh, stale managed-model cleanup, clearer StepFun auth copy, auth.order/profile-store-backed region inference, and the added regression coverage. The related #60032 is the merged baseline that added the bundled provider, not a duplicate or superseding fix for these follow-up changes.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to review the latest PR head, verify the StepFun model/default product decision, run the targeted StepFun/provider-auth tests plus the changed gate in a normal dev environment, then either land the focused provider-plugin follow-up or request narrow changes.
