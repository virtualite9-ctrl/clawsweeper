## Summary

Keep #11460 open. Current main still supports WhatsApp outbound `react` only; it does not advertise or handle the generic `message reactions` query action, does not store/query historical WhatsApp reactions, and does not surface inbound WhatsApp reaction events. The related #13641 was closed unmerged and #48570 remains open/unmerged, so they are useful implementation input but not shipped fixes.

## Best Possible Solution

Keep #11460 open until the bundled WhatsApp plugin implements a complete reaction read path: advertise and handle the generic `reactions` message action, retrieve or persist historical reactions where feasible, surface inbound add/remove reaction events through the normal routing/access-control pipeline, and add focused WhatsApp tests plus docs. Treat #48570 as useful prior implementation input, not as a shipped superseding fix.
