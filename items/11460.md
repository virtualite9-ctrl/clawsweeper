## Summary

Keep #11460 open. Current main still only supports outbound WhatsApp reaction writes through `react`; it does not advertise or handle the generic `reactions` query action, does not expose WhatsApp inbound reaction events, and has no documented or tested WhatsApp reaction history/read path. The linked PR #13641 was closed unmerged and only attempted inbound event surfacing, so it does not satisfy the query/history request.

## Best Possible Solution

Keep the issue open until the WhatsApp plugin has a complete reaction read path: advertise and handle the generic `reactions` message action, retrieve or persist WhatsApp reaction history, surface inbound add/remove events with correct routing and access checks, and add focused tests plus docs for CLI/tool behavior. The broader open PR #48570 can inform implementation, but it is not a shipped superseding fix for this issue yet.
