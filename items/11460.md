## Summary

Keep #11460 open. Current main still only supports outbound WhatsApp `react`; it does not advertise or dispatch the generic `reactions` query action, does not implement a WhatsApp reaction history/query runtime path, and does not wire inbound WhatsApp reaction events into the current inbound monitor. The linked PR #13641 was closed unmerged, and PR #48570 is still open/unmerged and only a potential implementation reference, not shipped behavior.

## Best Possible Solution

Keep #11460 open until the bundled WhatsApp plugin implements a complete reaction read path: advertise and handle the generic `reactions` message action, retrieve or persist reaction history where feasible through Baileys/wacli/OpenClaw storage, surface inbound add/remove reaction events through normal routing and access checks, and add focused WhatsApp tests plus docs. PR #48570 can be used as implementation input, but it should not be treated as a shipped superseding fix.
