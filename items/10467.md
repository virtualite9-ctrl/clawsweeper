## Summary

Keep #10467 open. Current main still does not implement the requested multi-lane `sessions_spawn` support: the tool schema has no `lane` parameter, native and ACP child dispatches hardcode the `subagent` lane, and subagent concurrency config remains a single scalar rather than per-lane limits. The issue discussion also includes a recent maintainer-posted Codex review reaching the same keep-open conclusion, plus a user comment confirming the single-lane bottleneck is a real pain point.

## Best Possible Solution

Keep #10467 open until core adds an optional validated `sessions_spawn.lane` field, forwards it through both native and ACP spawn paths to the Gateway `agent` call, supports backward-compatible per-lane subagent concurrency configuration with default fallback to `subagent`, and updates docs plus regression tests for default lane behavior, custom lanes, invalid lane names, and config reload.
