## Summary

Keep #10467 open. Current main still does not implement multi-lane `sessions_spawn`: the tool schema has no `lane` parameter, native and ACP spawn dispatch hard-code the `subagent` lane, configuration remains a single scalar for the global subagent lane, and docs/tests still describe/assert one global lane. The issue also has a recent maintainer-posted Codex review and a user +1 supporting the same keep-open conclusion.

## Best Possible Solution

Keep #10467 open until core adds an optional validated `sessions_spawn.lane` field, forwards it through native and ACP spawn paths to the gateway agent call, supports backward-compatible per-lane subagent concurrency configuration with default fallback to `subagent`, and updates docs plus regression tests for omitted lane, custom lanes, invalid lane names, and config reload behavior.
