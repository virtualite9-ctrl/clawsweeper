## Summary

Keep #10467 open. Current main still lacks a public `sessions_spawn.lane` parameter, both native and ACP spawn paths still dispatch child runs through the fixed `subagent` lane, and config/docs/tests still describe one global subagent concurrency cap. The same-day maintainer-posted Codex review in the issue also explicitly kept it open, and the relevant files have not changed since that review SHA.

## Best Possible Solution

Keep #10467 open until core adds an optional validated `sessions_spawn.lane` field, forwards it through both native and ACP spawn paths, defaults omitted lane to `subagent`, supports backward-compatible per-lane subagent concurrency configuration with hot reload, and updates docs plus tests for default lane, custom lanes, invalid names, and config fallback behavior.
