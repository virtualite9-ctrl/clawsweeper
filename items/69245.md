## Summary

Keep open. Current main still does not implement the PR's requested `openai-completions` cache-ttl eligibility, and the PR is not obsolete or superseded. The remaining maintainer question is whether eligibility should be broad for every `openai-completions` provider or narrowed to verified provider/model prompt-cache support.

## Best Possible Solution

Keep the PR open for maintainer/provider review. The best implementation is a focused eligibility path for OpenAI-compatible providers only when the provider/model actually supports prompt caching, likely through `compat.supportsPromptCacheKey`, provider-owned `isCacheTtlEligible`, or a similarly explicit capability, with regression coverage for the helper, extension factory inclusion, and cache-ttl timestamp append behavior.
