## Summary

Keep open. Current main still has the reported `sessions_spawn(runtime="subagent", streamTo="parent")` failure path: the shared schema exposes ACP-only `streamTo`, the executor rejects it before dispatching to `spawnSubagentDirect`, and the unit test still locks in that rejection. PR #72331 is the active candidate fix, but it is not merged into current main.

## Best Possible Solution

Keep this issue open until PR #72331 or an equivalent narrow fix lands. The fix should live in `src/agents/tools/sessions-spawn-tool.ts`: treat `streamTo` as ACP-only request data, ignore it for `runtime="subagent"`, preserve ACP pass-through to `spawnAcpDirect`, add a schema description for ACP-only use, and update `src/agents/tools/sessions-spawn-tool.test.ts` so the strict-provider shape dispatches to `spawnSubagentDirect` instead of erroring.
