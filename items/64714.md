## Summary

Keep open. Current main still has the reported `sessions_spawn(runtime="subagent", streamTo="parent")` failure path: the shared schema exposes ACP-only `streamTo`, the executor rejects it before reaching `spawnSubagentDirect`, and the unit test still asserts that rejection. The provided related-item context shows #72331 is the active maintainer-labeled candidate fix, but it is still open and not merged into current main.

## Best Possible Solution

Keep this issue open until #72331 or an equivalent narrow fix lands. The implementation should live in `src/agents/tools/sessions-spawn-tool.ts`: treat `streamTo` as ACP-only request data, ignore it for `runtime="subagent"`, preserve ACP pass-through to `spawnAcpDirect`, add a schema description for ACP-only use, and update `src/agents/tools/sessions-spawn-tool.test.ts` so the strict-provider call shape dispatches to `spawnSubagentDirect` instead of erroring.
