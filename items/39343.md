## Summary

Keep open. Current `main` only addresses part of the request: Telegram albums with `media_group_id` are buffered into one processing turn, but LINE still handles each inbound media event independently, and the shared inbound debounce path explicitly excludes media. The requested cross-channel/configurable gateway-layer media batching is not implemented on current `main`.

## Evidence

  - file: [src/channels/inbound-debounce-policy.ts:11](https://github.com/openclaw/openclaw/blob/cb2c36b04975e34ab7014a11575d6589aa96ae32/src/channels/inbound-debounce-policy.ts#L11)

## Risks / Open Questions

- Telegram-only buffering may make the issue look partially solved, but the issue's requested gateway-layer/configurable batching still does not exist for LINE or the shared inbound path.
- Closing as implemented would hide a real remaining gap for rapid multi-image sends on channels without native grouping metadata.
