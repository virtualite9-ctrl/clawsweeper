## Summary

Keep #39343 open. Current `main` still only implements the Telegram album side of the request: Telegram `media_group_id` updates are buffered into one agent turn, but LINE still handles each media webhook event independently and the documented shared inbound debounce remains text-only with media flushing immediately.

## Best Possible Solution

Keep #39343 open and implement the missing bundled-channel behavior: preserve the existing Telegram `media_group_id` album buffer, add LINE same-sender/chat/time-window media batching with clear timer and text-flush semantics, decide whether this extends `messages.inbound` or uses a media-specific channel option, document the behavior on the public docs, and add LINE plus Telegram regression coverage for batching keys, timer flush, text flush, replay-cache timing, and reply-token behavior.
