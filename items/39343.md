## Summary

Keep #39343 open. Current main partially satisfies the request for Telegram albums by buffering `media_group_id`, but LINE still processes each media event as its own agent turn, and there is no media-specific batching config or docs. The prior ClawSweeper comment reached the same conclusion; current main at `4a3030df9efa72c44bab567a4390ac0c4876a73f` still supports it.

## Best Possible Solution

Keep the issue open and finish the bundled-channel behavior: preserve Telegram `media_group_id` album buffering, add LINE same-sender/chat/time-window media batching with timer and text-flush semantics, expose and document a media-specific delay or a clear extension of `messages.inbound`, and add LINE plus Telegram regression coverage for batching, independent keys, and edge cases.
