## Summary

Keep open. Current `main` and the latest release still hard-stop Voice Tab when `SpeechRecognizer.isRecognitionAvailable(context)` is false, so the reported Chinese OEM behavior is not implemented. The PR discussion also surfaced a credible unresolved regression risk: if this patch lands as-is, `ERROR_CLIENT` would still fall into the restart path and could loop on devices with no usable recognizer.
