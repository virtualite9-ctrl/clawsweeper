## Summary

Keep #38691 open. Current main still hard-disables the Android Voice Tab manual mic path when `SpeechRecognizer.isRecognitionAvailable(context)` is false, and there is still no configured OEM `voice_recognition_service` fallback. The PR is stale and needs the retry-loop review concern addressed, but the requested bug fix remains useful and unimplemented on main.

## Best Possible Solution

Keep the PR open for revision. The best path is to rebase the fix onto `apps/android/app/src/main/java/ai/openclaw/app/voice/MicCaptureManager.kt`, add a generic configured OEM recognizer fallback using the secure `voice_recognition_service` component and `SpeechRecognizer.createSpeechRecognizer(context, component)`, and preserve a clean disable path when no recognizer backend is usable. Retain the Xiaomi manual-device validation as useful evidence and add focused Android/Robolectric coverage if the recognizer seam can be tested without brittle platform mocking.
