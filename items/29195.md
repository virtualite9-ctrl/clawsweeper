## Summary

Keep #29195 open. Current main has meaningful ACP task/status and timeout improvements, and #31044 documented ACPX permissions, but the central ACPX/Codex non-interactive reliability problem is still present: ACPX still defaults to approve-reads/fail, that default is passed directly into the embedded runtime, parent agents still cannot inspect cross-agent ACP children under default tree visibility, and docs still describe stale ACP process cleanup as manual. Related #72064 also reinforces that current v2026.4.24 users still hit the same default-permission failure mode and use explicit acpx permission config as the workaround.

## Best Possible Solution

Keep #29195 open as the canonical ACPX/Codex non-interactive reliability umbrella, or split and link narrower follow-ups before closing it. The concrete remaining path is a safe permission preflight or default/profile decision for ACPX-backed Codex sessions, requester-facing remediation for permission/config failures, same-parent visibility for cross-agent ACP children without requiring global all visibility, and automatic stale harness cleanup/reporting that does not rely on manual ps/kill steps.
