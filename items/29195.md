## Summary

Keep #29195 open. Current main has documentation and partial ACP task/status improvements, but the reported ACPX/Codex failure family is not fully resolved: the default non-interactive permission configuration still fails writes/exec, cross-agent ACP child visibility is still not solved by default tree visibility, and docs still describe manual stale-process cleanup as an active troubleshooting path.

## Best Possible Solution

Keep #29195 open for the remaining ACPX/Core work: decide a safe non-interactive permission default or preflight policy, surface permission/config failures with actionable messages to the requester, allow a spawning agent to inspect its own ACP child without global visibility, and replace manual stale-process cleanup with automatic timeout/reporting semantics. Preserve the shipped docs, /acp status, background task tracking, and runTimeoutSeconds plumbing as partial progress.
