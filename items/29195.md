## Summary

Keep #29195 open. Current main has useful improvements and documentation for ACPX/Codex, including the merged permission docs from #31044 and task/status/timeout plumbing, but the central non-interactive reliability problems are not fully resolved: ACPX still defaults to approve-reads/fail, default session-tool visibility still blocks a parent agent from inspecting a cross-agent Codex ACP child, and the ACP docs still document manual stale-process cleanup.

## Best Possible Solution

Keep #29195 open as the canonical ACPX/Codex non-interactive reliability umbrella, or split it into narrower tracked issues only after creating canonical follow-ups. The product path should decide a safe default or preflight for ACPX permissions, surface permission/config failures with requester-facing remediation, allow spawning agents to inspect their own cross-agent ACP children without global visibility, and replace manual stale-process cleanup with automatic timeout/reporting semantics while preserving the shipped docs and status/task improvements.
