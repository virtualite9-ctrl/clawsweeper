## Summary

Keep #30381 open. Current main still resolves `x-openclaw-agent-id` as the selected agent, but then independently validates the request body `model` and rejects non-OpenClaw model ids before dispatching the chat completion. The requested OpenAI SDK compatibility behavior is not implemented, and the issue discussion includes recent production-impact comments.

## Best Possible Solution

Keep the issue open and implement a header-aware bypass around `src/gateway/http-utils.ts` and `src/gateway/openai-http.ts`: when `x-openclaw-agent-id` resolves to a valid agent for chat completions, ignore the OpenAI body `model` for agent selection/validation while still using the agent's configured model and still validating explicit `x-openclaw-model`. Add a regression test for `model: "gpt-4"` plus `x-openclaw-agent-id`, and update the OpenAI HTTP docs if that becomes the supported compatibility contract.
