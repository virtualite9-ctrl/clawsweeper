## Summary

Keep #30381 open. Current main still rejects a generic request-body `model` before `/v1/chat/completions` dispatch, even when `x-openclaw-agent-id` has selected the target agent. The active issue comments describe a concrete production need, and current tests/docs still encode the stricter agent-target model contract.

## Best Possible Solution

Implement a header-aware bypass in the OpenAI-compatible model validation path: when `x-openclaw-agent-id` resolves to a valid agent, ignore the request body `model` for agent/model validation and selection, while still validating explicit `x-openclaw-model` backend overrides. Add regression coverage for `model: "gpt-4"` plus `x-openclaw-agent-id` on `/v1/chat/completions`, consider the same shared-helper behavior for `/v1/responses`, and update https://docs.openclaw.ai/gateway/openai-http-api if this becomes the supported compatibility contract.
