## Summary

Keep #30381 open. Current main still resolves `x-openclaw-agent-id` as the target agent, but `/v1/chat/completions` still validates the request body `model` independently and returns HTTP 400 for non-OpenClaw model ids before dispatching the agent turn. The requested OpenAI SDK compatibility behavior is not implemented.

## Best Possible Solution

Implement a header-aware bypass around the chat completions model validation path: when `x-openclaw-agent-id` resolves to a valid agent, ignore the OpenAI request body `model` for validation and agent/model selection, while still honoring and validating explicit `x-openclaw-model`. Add a regression test for `model: "gpt-4"` plus `x-openclaw-agent-id`, and update https://docs.openclaw.ai/gateway/openai-http-api if this becomes the supported compatibility contract.
