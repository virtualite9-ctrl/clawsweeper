## Summary

Keep open. Current main still interpolates dynamic memory schema table identifiers directly into SQLite DDL in an exported memory SDK helper, so this PR addresses a real current code path. It should remain open for maintainer review and iteration because the patch has no regression tests and an existing review note identifies a possible regression in the new table-name quoting contract.

## Best Possible Solution

Keep this PR open for maintainer review and iteration. The best path is a revised memory-schema fix that defines the SQLite identifier contract, quotes or validates dynamic identifiers consistently in both `src/` and `packages/`, preserves or explicitly rejects schema-qualified and pre-quoted identifier forms, adds regression tests for malicious and edge-case identifiers, and then lands if the security model holds.

## Risks / Open Questions

- Landing the PR unchanged could regress valid pre-quoted or dotted SQLite identifiers if `quoteTable` blindly splits on every dot.
- The PR currently lacks focused regression tests for malicious identifier input, schema-qualified table names, and the chosen contract for pre-quoted identifiers.
- Maintainers still need to decide whether this exported helper should accept arbitrary table identifiers, validate a narrow contract, or own full SQLite identifier quoting.
