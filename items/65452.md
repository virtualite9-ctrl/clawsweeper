## Summary

Keep this PR open. Current main still interpolates the memory schema table identifiers directly into SQLite DDL on an exported plugin SDK surface, so the reported class is not obsolete. The PR is focused and security-relevant, but it still needs maintainer review and tests because its `quoteTable` contract may regress valid pre-quoted or dotted SQLite identifiers.

## Best Possible Solution

Keep this PR open for maintainer review and iteration. The best path is a revised memory-schema fix that defines the accepted SQLite identifier contract, quotes or validates dynamic table identifiers consistently in both `src/` and `packages/`, preserves or explicitly rejects schema-qualified and pre-quoted forms, adds regression tests for malicious and edge-case identifiers, and then lands if the security model holds.

## Risks / Open Questions

- Landing the PR unchanged could regress callers that intentionally pass pre-quoted or dotted SQLite identifiers if `quoteTable` blindly splits on every dot.
- The PR currently lacks focused regression tests for malicious identifier input, schema-qualified names, and the chosen behavior for pre-quoted identifiers.
- Current main still has the unquoted exported helper path, so closing this PR without a replacement would leave the reported issue unaddressed.
