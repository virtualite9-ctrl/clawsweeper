## Summary

Keep open. Current `main` does not include the proposed `inboundRootId` fallback, and the PR as written conflicts with the shipped/tested `replyToMode: "off"` contract by potentially forcing threaded replies when no explicit reply target was requested.

## Evidence

- **Current main still resolves Mattermost reply roots as thread root first, then raw replyToId:** `resolveMattermostReplyRootId` on `main` only returns `threadRootId` or `replyToId`; there is no `inboundRootId` safeguard in the shipped code.
  - file: [extensions/mattermost/src/mattermost/monitor.ts:233](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/extensions/mattermost/src/mattermost/monitor.ts#L233)
  - command: `nl -ba extensions/mattermost/src/mattermost/monitor.ts | sed -n '228,242p'`
  - sha: [c070509b7f8f](https://github.com/openclaw/openclaw/commit/c070509b7f8fdbe479e5d829643782c272e30e83)
- **Current main explicitly suppresses existing thread roots when replyToMode is off:** The Mattermost threading logic returns `undefined` for existing thread roots under `replyToMode: "off"`, so off-mode is a real behavioral contract in current tests and code.
  - file: [extensions/mattermost/src/mattermost/monitor.ts:342](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/extensions/mattermost/src/mattermost/monitor.ts#L342)
  - command: `nl -ba extensions/mattermost/src/mattermost/monitor.ts | sed -n '342,364p'`
  - sha: [c070509b7f8f](https://github.com/openclaw/openclaw/commit/c070509b7f8fdbe479e5d829643782c272e30e83)
- **Main test coverage asserts the off-mode contract:** Regression tests on `main` expect threaded messages to stay top-level when `replyToMode` is `off`, which is the same behavior the PR review comment says would regress if `inboundRootId` were returned unconditionally.
  - file: [extensions/mattermost/src/mattermost/monitor.test.ts:447](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/extensions/mattermost/src/mattermost/monitor.test.ts#L447)

- **Shared outbound delivery policy preserves explicit payload replies but suppresses fallback replyToId when off:** Core outbound tests verify that `replyToMode: "off"` removes fallback reply targets while still allowing explicit payload reply IDs.
