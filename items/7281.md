## Summary

Current main still lacks configurable fallbackChannel/fallbackChannels support for hook mapping delivery. The current hook config, strict schema, mapping pipeline, dispatch payload, cron delivery model, docs, and outbound delivery path still carry a single channel/to target; existing fallbackChannel identifiers are unrelated pre-delivery/session fallback helpers. The existing issue discussion already kept #7281 open, and current main adds no contradictory implementation.

## Best Possible Solution

Keep #7281 open for maintainer/product handling. The best implementation is typed fallbackChannel or ordered fallbackChannels support on hook mappings, carried through schema/help/docs, transforms, dispatch payloads, CronDelivery planning, and outbound delivery, with alternates attempted only after primary delivery failure and tests for primary-fails/fallback-succeeds plus all-targets-failed reporting. Keep this distinct from channel="last", bestEffort delivery, and failureDestination.
