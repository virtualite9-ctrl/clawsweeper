## Summary

Keep open. Current `main` still has a direct code path where cron task records are created with `createdAt = Date.now()` while `startedAt` is supplied from an earlier scheduler timestamp, so `startedAt < createdAt` remains possible and matches the report.

## Evidence

- **Task registry still timestamps creation independently:** `createTaskRecord` computes `const now = Date.now()` and writes `createdAt: now` even when `startedAt` was already provided by the caller, with `lastEventAt` falling back to that earlier `startedAt` value.
  - file: [src/tasks/task-registry.ts:1440](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/src/tasks/task-registry.ts#L1440)
  - command: `nl -ba src/tasks/task-registry.ts | sed -n '1420,1495p'`
  - sha: [6bc0dc8fb671](https://github.com/openclaw/openclaw/commit/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440)
- **Main cron execution path passes earlier startedAt into task creation:** The timer-backed cron path calls `createRunningTaskRun` with `startedAt: params.startedAt` and `lastEventAt: params.startedAt`, so if task creation happens even slightly later the stored record can invert the timestamps.
  - file: [src/cron/service/timer.ts:140](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/src/cron/service/timer.ts#L140)
  - command: `nl -ba src/cron/service/timer.ts | sed -n '140,156p'`
  - sha: [6bc0dc8fb671](https://github.com/openclaw/openclaw/commit/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440)
- **Manual cron execution path has the same behavior:** Manual cron runs reserve `preflight.now` first, then pass that earlier value into `tryCreateManualTaskRun`, which creates the running task with `startedAt: params.startedAt`; the task registry still assigns `createdAt` at insert time.
  - file: [src/cron/service/ops.ts:577](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/src/cron/service/ops.ts#L577)
  - command: `nl -ba src/cron/service/ops.ts | sed -n '566,586p'; nl -ba src/cron/service/ops.ts | sed -n '425,455p'`
  - sha: [6bc0dc8fb671](https://github.com/openclaw/openclaw/commit/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440)
- **OpenClaw still treats this as an audit inconsistency:** The audit code explicitly flags `startedAt < createdAt` as `inconsistent_timestamps`, and the public tasks docs still describe that warning while also stating that every cron execution creates a task record.
  - file: [src/tasks/task-registry.audit.ts:49](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/src/tasks/task-registry.audit.ts#L49)
