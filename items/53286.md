## Summary

Keep this issue open. Current main still suppresses estimated dollars for explicitly priced non-API-key providers in the two reported surfaces: `/status` and `/usage full`. The same-author PR #53303 was closed unmerged, so there is no open paired implementation candidate to wait on and no shipped fix on main.

## Best Possible Solution

Keep this issue open. Implement the display-layer fix by resolving explicit model cost metadata independently of auth mode, then show estimated cost in both `/status` and `/usage full` for `aws-sdk`, OAuth, token, mixed, and API-key providers when pricing is explicitly configured. Add regression coverage for a Bedrock-style `aws-sdk` provider with nonzero model cost and update the public API usage/token-use docs.
