## Summary

Keep this issue open. Current main still hides estimated cost in the two reported display paths when the active provider is Bedrock-style `aws-sdk` auth, even though explicit model cost config can already be resolved independently of auth mode. The same-author fix PR #53303 was closed unmerged, and adjacent #29100 covers auto-discovered Bedrock cost metadata rather than this explicit-cost display bug.

## Best Possible Solution

Keep this issue open. Implement a narrow display-layer fix for `/status` and the response usage footer: resolve model cost config independently of auth, show cost when API-key/mixed auth is used or when explicit usable nonzero pricing exists for `aws-sdk`, OAuth, or token providers, and add regression tests for an `amazon-bedrock` provider with `auth: "aws-sdk"` and explicit cost config. Update the public token-use and usage-tracking docs to describe the new explicit-pricing behavior.
