## Summary

Keep this issue open. Current main still gates estimated cost display on auth mode in both reported surfaces, so an Amazon Bedrock provider resolved as `aws-sdk` remains token-only even when explicit nonzero model pricing is configured. The related same-author PR #53303 was closed unmerged, so there is no implementation candidate currently shipped on main.

## Best Possible Solution

Keep this issue open. Implement a focused display-layer fix in `src/status/status-message.ts` and `src/auto-reply/reply/agent-runner.ts` so explicit usable model pricing is resolved independently of auth mode, then show estimated cost for `aws-sdk`, OAuth, token, mixed, and API-key providers when nonzero pricing is configured. Add regression tests for a Bedrock-style `auth: "aws-sdk"` provider with explicit cost config in both `/status` and `/usage full`, and update the public token-use / usage-tracking docs.
