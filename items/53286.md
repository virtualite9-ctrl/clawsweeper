## Summary

Keep this issue open. Current main still gates both `/status` cost display and `/usage full` reply-footers on auth mode, so `aws-sdk` providers with explicit nonzero model cost config will still show tokens without dollar estimates. The same-author PR that claimed to fix it was closed unmerged, and the relevant docs still describe estimated costs as API-key-only.

## Best Possible Solution

Keep this issue open and implement the requested decoupling in core usage display: resolve model cost metadata independently of auth mode, show estimated cost for `/status` and `/usage full` when the active model has explicit nonzero cost metadata even under `aws-sdk`/OAuth/token auth, and add regression tests for an `aws-sdk` Bedrock-style provider plus matching docs updates.
