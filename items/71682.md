## Summary

Current main already has a supported LLM idle-timeout config path and CLI write coverage, so part of #71682 is addressed. I would still keep it open because the report also says the LLM cannot change the timeout and current agent-facing `gateway config.patch/apply` allowlist does not permit `agents.defaults.llm.idleTimeoutSeconds`; timeout-profile/fallback behavior is also still tracked separately by open #51057.

## Best Possible Solution

Keep open for maintainer triage. The practical path is to document the exact CLI fix (`openclaw config set agents.defaults.llm.idleTimeoutSeconds <seconds>`) and decide whether the agent-facing gateway config tool should also allow bounded edits to `agents.defaults.llm.idleTimeoutSeconds`; leave the timeout auth-profile cooldown question to the existing #51057 thread unless this issue adds a distinct repro.
