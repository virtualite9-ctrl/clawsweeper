## Summary

Keep open. Current main partially solves the user/operator side with `agents.defaults.llm.idleTimeoutSeconds`, CLI write coverage, docs, schema, and a runtime timeout hint. It is not safely closeable because the report also says the LLM/agent cannot change the timeout and the setting is lost after gateway restart; current agent-facing `gateway config.patch/apply` still fail-closed to a narrow allowlist that excludes `agents.defaults.llm.idleTimeoutSeconds`. Related timeout cooldown/faster fallback behavior remains open in #51057, and current tests still assert timeout profile rotation does not persist cooldown.

## Best Possible Solution

Keep open for maintainer triage. The practical path is to document the operator command `openclaw config set agents.defaults.llm.idleTimeoutSeconds <seconds>` for slow local models, then decide whether the owner-only agent-facing `gateway config.patch/apply` allowlist should permit bounded edits to `agents.defaults.llm.idleTimeoutSeconds`. Keep timeout auth-profile cooldown and faster fallback policy aligned with #51057 unless this issue adds a distinct reproduction that needs separate handling.
