## Summary

Keep #71682 open. Current main has a supported user-facing LLM idle-timeout config path, CLI write coverage, docs, and timeout error guidance, so the direct "set a larger timeout" part is partly addressed. The report is still not safely closeable because the agent-facing gateway config path that agents are told to use cannot currently edit `agents.defaults.llm.idleTimeoutSeconds`, and timeout/fallback profile behavior remains a live related issue in #51057 rather than fully resolved here.

## Best Possible Solution

Keep open for maintainer triage. The practical fix is to document the operator command `openclaw config set agents.defaults.llm.idleTimeoutSeconds <seconds>` for slow local models, then decide whether the owner-only agent-facing `gateway config.patch/apply` allowlist should permit bounded edits to `agents.defaults.llm.idleTimeoutSeconds`. Leave timeout-auth-profile cooldown and faster fallback policy to the already-open #51057 unless #71682 adds a distinct repro that needs separate tracking.
