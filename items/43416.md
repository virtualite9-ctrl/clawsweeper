## Summary

Keep this PR open. Current main still lacks a shared native Swift assistant-message copy affordance, so the PR is not obsolete, but the proposed implementation should not merge as-is because it copies raw `primaryText` instead of the visible assistant text produced by `AssistantTextParser`. The prior review discussion usefully resolved timer, hit-testing, tool-result role, and platform pasteboard concerns, but the remaining visible-text clipboard issue is still substantive.

## Best Possible Solution

Keep this PR open and request revisions: derive the clipboard payload from the same `AssistantTextParser.segments(from:includeThinking:)` path used by `ChatAssistantTextBody`, keep the assistant-role guard and platform pasteboard handling, add a focused parser/helper test for visible copied text, and verify the shared native chat UI on macOS and iOS.
