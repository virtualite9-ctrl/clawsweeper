## Summary

Keep open. Current main does not implement the native Swift assistant-message copy button, and this PR remains a plausible focused UI improvement, but the current diff is not ready to close or land because it copies raw assistant text rather than the same visible text that the renderer shows.

## Best Possible Solution

Keep this PR open and request revisions: compute the clipboard payload from the same visible assistant-text segments used by `ChatAssistantTextBody`, keep the assistant-role guard, retain the cancellable copied-state task, and verify the native shared chat UI on both macOS and iOS. A focused test around the visible-text transformation would make the remaining risk much easier to review.

## Risks / Open Questions

- Merging the PR as-is could copy hidden `<think>` content and tags to the clipboard when traces are disabled, which is a privacy/security-sensitive mismatch between visible UI and copied data.
- The PR currently has no source-level test or documented verification that the copied text matches the rendered assistant text transformation.
- Because the touched package is shared across iOS and macOS, maintainers should require platform build or manual verification before merge.
