## Summary

Keep #37131 open. Current main has partial no-LLM paths, especially `POST /tools/invoke`, `/bash`, plugin commands, and skill slash commands that can dispatch to named tools. The umbrella request is still not implemented across the requested coding-tool, cron, CLI, and generic slash-command surfaces: Gateway HTTP resolves only OpenClaw/plugin tools, cron still supports only `systemEvent` and `agentTurn`, `openclaw agent` is still message-turn based, and there is no generic `/tool` slash executor.

## Best Possible Solution

Keep #37131 open as the umbrella product/API request until maintainers either ship a security-reviewed direct invocation contract across HTTP, cron, CLI, and slash-command surfaces, or explicitly split the remaining work into canonical narrower issues and close this as superseded by those trackers. A good path is to review the HTTP coding-tool slice separately, design cron `toolCall`/`exec` delivery semantics, and add documented CLI/slash grammar that reuses the same auth, owner-only, deny-list, and tool-policy pipeline.
