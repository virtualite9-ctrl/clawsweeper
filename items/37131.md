## Summary

Keep #37131 open. Current main has partial direct invocation via `POST /tools/invoke`, `/bash`, skill command tool dispatch, and MCP bridges, but the requested generic no-LLM tool path is still not implemented across the headline surfaces: `exec`/`read`/`write` are not wired into the Gateway HTTP resolver, cron still only supports `systemEvent` and `agentTurn`, the CLI `agent` command has no `--tool` mode, and there is no generic `/tool` slash command. Related #63919 and #18160 are useful open slices, not shipped/superseding coverage for the umbrella request.

## Best Possible Solution

Keep #37131 open as the umbrella product/API request until maintainers either ship a security-reviewed direct tool invocation contract across HTTP, cron, slash command, and CLI surfaces, or explicitly split the remaining scope into canonical narrower issues and close this as superseded by those trackers.
