## Summary

Keep #37131 open. Current main has partial no-LLM paths (`POST /tools/invoke`, `/bash`, and skill slash commands that dispatch to a named tool), but the umbrella request is not implemented across the requested surfaces. The Gateway HTTP resolver still does not compose the coding tool factory that owns `read`, `write`, and `exec`; cron still only accepts `systemEvent` and `agentTurn`; `openclaw agent` has no `--tool` mode; and there is no generic `/tool` slash command in the command registry. Related #63919 and #18160 are useful open slices, not shipped or canonical replacements for the whole request.

## Best Possible Solution

Keep #37131 open as the umbrella product/API request until maintainers either ship a security-reviewed direct invocation contract across HTTP, cron, CLI, and slash-command surfaces, or explicitly split the remaining work into canonical narrower issues and close this as superseded by those trackers. A likely path is to land/review the HTTP coding-tool slice separately, design a cron `toolCall`/`exec` payload with delivery semantics, and add a documented CLI/slash grammar that reuses the same auth, owner-only, deny-list, and tool-policy pipeline.
