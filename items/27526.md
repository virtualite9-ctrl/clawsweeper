## Summary

Keep #27526 open. Current main has partial session management, chat session switching, labels/search, runtime overrides, checkpoint branching, and sub-agent/background-run support, but it does not implement the requested WebChat conversation directory with folders/tags/archive/pinning/recommendations or the multi-AI comparison/roles/consensus/committee UX. The provided discussion also has explicit maintainer signal that this issue is the canonical tracker after duplicate #27603 was consolidated here.

## Best Possible Solution

Keep #27526 as the canonical product tracker. The best path is incremental: define session metadata/protocol support for folders, tags, archive state, session pinning, richer search/recommendations, and directory navigation in WebChat/Control UI; then design an explicit multi-model/role collaboration UX and protocol for side-by-side comparison, consensus, agenda/minutes, and project-task extraction. Narrow PRs such as REST/session listing or checkpoint work can stay separate but should not close this umbrella until the central user-facing workflow ships.
