## Summary

Keep open. Current main has partial install hardening, but not the requested built-in scan for ClawHub skill install/update/load/publish flows. The ClawHub skill path still resolves a version, downloads an archive, extracts it, and installs the directory without calling the built-in skill install scanner. The AgentShield and ClawGuard links in the discussion are useful external implementation references, but optional tools do not satisfy the requested on-by-default OpenClaw behavior.

## Best Possible Solution

Keep the issue open, or have a maintainer manually consolidate it with the broader protected security tracking items. The product path should add built-in policy-backed scanning to native ClawHub skill install/update paths, decide how publish-side scanning belongs between OpenClaw and ClawHub, and use AgentShield/ClawGuard as possible references rather than relying on users to install optional scanner skills.
