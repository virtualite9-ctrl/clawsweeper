## Summary

Keep PR #39888 open. Current main still does not emit a `model.usage` diagnostic event from the `openclaw agent` command path, while reply-runner does and diagnostics plugins consume that event for OTEL/Prometheus token accounting. The submitted patch is stale because `src/commands/agent.ts` is now only a re-export, but the underlying behavior gap remains valid.

## Best Possible Solution

Refresh this PR or replace it with a small main-targeted change that emits a trusted `model.usage` diagnostic after `updateSessionStoreAfterAgentRun` in the current CLI/local agent path. The implementation should include a stable CLI channel label, agent/session/provider/model fields, usage, last-call usage, context limit/used, cost, and duration; preserve reply-runner behavior; and add a focused regression test for CLI diagnostic usage emission.
