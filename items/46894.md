## Summary

Keep open. Current main still has the WebUI-specific bug path: the Control UI intercepts `/model ...` as a local command, and `executeModel` only treats an empty argument as informational; any non-empty argument, including `status` or `list`, is sent to `sessions.patch`. Docs and the server-side directive path already recognize `/model list` and `/model status`, and the linked PRs #46895 and #47069 are open/unmerged, so the issue is not implemented on main and should remain as the bug tracker until a fix lands.

## Best Possible Solution

Land a focused Control UI fix in `ui/src/ui/chat/slash-command-executor.ts` that recognizes reserved `/model` info subcommands before the patch path. At minimum, `/model status` and `/model list` should not call `sessions.patch`; ideally the WebUI behavior should align with the documented server directive semantics, with regression tests in `ui/src/ui/chat/slash-command-executor.node.test.ts` for `/model status`, `/model list`, and a normal `/model <provider/model>` patch.
