## Summary

Keep this PR open. Current main still has the exact non-adaptive Anthropic thinking fallback this PR targets, and the zero-budget path is reachable from the current token adjustment logic. The PR is not obsolete or implemented on main, but it should add focused regression coverage before landing.

## Best Possible Solution

Keep this PR open as a plausible implementation candidate, but require targeted tests before merge: one asserting `thinking` is omitted when the resolved legacy Anthropic budget is `0`, and one preserving the intended positive/default budget behavior. Maintainers should also decide whether the broader undefined-config default from related issue #63016 is intentional or needs a separate follow-up.
