## Summary

Current `main` does not implement the requested startup warning. `startGatewayServer` immediately loads the gateway startup config via `loadGatewayStartupConfigSnapshot(...)` with no preflight scan for `CLAWDBOT_*` env keys, and there is no warning helper or test coverage for such a warning in `src/gateway`. The report also still matches current behavior: `src/config/paths.test.ts:70` explicitly asserts that legacy `CLAWDBOT_GATEWAY_PORT` is ignored, so users can still hit the silent-misconfiguration case this PR is trying to address.
