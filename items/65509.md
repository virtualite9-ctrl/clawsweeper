## Summary

Keep this PR open. Current main still ignores OPENCLAW_STATE_DIR for the default/main agent workspace path, and the PR addresses a valid isolation gap, but the submitted patch needs revision because it uses the env value raw and now targets an older file location.

## Best Possible Solution

Keep the PR open and request a focused revision against src/agents/workspace-default.ts. The implementation should use a normalized OPENCLAW_STATE_DIR value when provided, preserve existing unset-env and OPENCLAW_HOME fallback behavior, append the OPENCLAW_PROFILE suffix as before, and add tests for absolute, tilde/relative, default-profile, non-default-profile, and unset-env cases.
