## Summary

Keep open. Current main partially timestamps Slack thread-history text via the generic inbound envelope, but it does not implement the requested image-specific timestamp, sender, and thread-position annotation before image/media payloads. The linked implementation PR #48725 was closed unmerged, and the requested cross-channel behavior remains absent.

## Best Possible Solution

Keep the issue open. A good implementation should add a channel-safe metadata model for media attachments in thread history, start with Slack where the report was observed, include timestamp/sender/thread-position metadata adjacent to the image/media prompt payload, and add config/schema/docs/tests without relying on core deep imports from bundled plugins.
