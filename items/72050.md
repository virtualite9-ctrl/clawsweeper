## Summary

Keep PR #72050 open for maintainer review. Current main does not ship a `subclaw` command or a directory-argument TUI launcher, and the PR targets a plausible CLI/session workflow. It is not merge-ready as submitted because the diff shown in the provided GitHub context omits the sub-CLI descriptor catalog update and bundles unrelated gateway/scripts/auth/runtime changes with the feature.

## Best Possible Solution

Keep the PR open for maintainer review or contributor revision. The best path is to split unrelated gateway/scripts/auth/runtime changes out, then decide whether `subclaw` belongs in core or should follow the ClawHub/plugin path already discussed for #71832. If accepted in core, land a narrow PR that updates the descriptor catalog, wires `cwd` through TUI safely, derives stable per-directory session keys, handles directory errors consistently, and adds targeted CLI/TUI docs and tests.
