## Summary

Keep open. Current main still rejects `streamTo: "parent"` when `sessions_spawn` resolves to `runtime="subagent"`, and the subagent spawn path has no stream/progress parameter or parent relay. The shipped ACP path has a dedicated parent stream relay, but that implementation is not wired into native subagents. Related reports/PRs cover the hard-error symptom or broader progress callback discussion, but the requested real-time native subagent progress support is not implemented on current main.

## Best Possible Solution

Keep the issue open as a core sessions_spawn/subagent runtime feature. The best implementation would add an explicit native-subagent progress contract, likely `streamTo: "parent"` on `SpawnSubagentParams`, remove or narrow the ACP-only guard, start a parent relay for native subagent runs using gateway/agent lifecycle events or task-run progress, and add tests for run mode, session/thread mode, cleanup, timeout, no requester session, and non-streaming back-compat. Coordinate separately with the narrower hard-error fix so dropping ACP-only fields does not close this real streaming request.
