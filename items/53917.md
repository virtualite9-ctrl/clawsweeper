## Summary

Keep open. Current main still has the Feishu agent reply media path described in the report: `sendMediaFeishu` requires `mediaLocalRoots` to allow host-local paths, but `createFeishuReplyDispatcher` neither accepts nor forwards roots when delivering reply attachments. Direct Feishu action/outbound paths have partial plumbing, which makes this a narrow remaining reply-dispatcher bug rather than cleanup material.

## Best Possible Solution

Keep this issue open and resolve it with a focused Feishu-owned change. Extend the Feishu reply dispatcher contract to carry the appropriate outbound media roots or media access, wire both bot dispatcher call sites, pass the roots through `sendMediaReplies` into `sendMediaFeishu`, and add a reply-dispatcher regression test for local media. Maintainers should also decide whether Feishu should expose documented channel/account `mediaLocalRoots` config, likely coordinating with the existing related PRs #49082, #51375, and #39523.
