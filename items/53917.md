## Summary

Keep open. Current `main` still drops `mediaLocalRoots` on the Feishu reply-dispatcher path, so local-path media sent from agent replies can still be blocked by the post-CVE local-root check.

## Evidence

- **Dispatcher params have no media root field:** `CreateFeishuReplyDispatcherParams` does not define `mediaLocalRoots`, so the reply dispatcher has no way to receive the configured allowlist.
  - file: [extensions/feishu/src/reply-dispatcher.ts:75](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/extensions/feishu/src/reply-dispatcher.ts#L75)
  - command: `nl -ba extensions/feishu/src/reply-dispatcher.ts | sed -n '68,110p'`
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Reply-dispatcher media send omits mediaLocalRoots:** `sendMediaReplies` calls `sendMediaFeishu` with `cfg`, `to`, `mediaUrl`, reply metadata, and `accountId`, but does not pass `mediaLocalRoots`.
  - file: [extensions/feishu/src/reply-dispatcher.ts:346](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/extensions/feishu/src/reply-dispatcher.ts#L346)
  - command: `nl -ba extensions/feishu/src/reply-dispatcher.ts | sed -n '342,360p'`
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Feishu media sender still requires the roots:** `sendMediaFeishu` documents that local-path sends require `mediaLocalRoots` after CVE-2026-26321 and forwards them to `loadWebMedia(..., { localRoots })`. Without that field, local paths stay blocked.
  - file: [extensions/feishu/src/media.ts:573](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/extensions/feishu/src/media.ts#L573)
  - command: `nl -ba extensions/feishu/src/media.ts | sed -n '573,618p'`
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Both bot callsites cannot supply the roots:** The active-agent and single-agent `createFeishuReplyDispatcher(...)` callsites pass account/reply metadata but no `mediaLocalRoots`, matching the missing field in the dispatcher type.
  - file: [extensions/feishu/src/bot.ts:1088](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/extensions/feishu/src/bot.ts#L1088)
  - command: `nl -ba extensions/feishu/src/bot.ts | sed -n '1082,1104p;1191,1212p'`
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Other Feishu outbound paths do pass the roots:** The direct outbound adapter forwards `mediaLocalRoots` into `sendMediaFeishu` for both local-image-text shim and explicit media sends, so the missing pass-through is specific to the reply-dispatcher path.
  - file: [extensions/feishu/src/outbound.ts:166](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/extensions/feishu/src/outbound.ts#L166)
  - command: `nl -ba extensions/feishu/src/outbound.ts | sed -n '156,184p;258,275p'`
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **No reply-dispatcher regression test covers this field:** `rg -n "mediaLocalRoots" extensions/feishu/src/reply-dispatcher.test.ts` returns no matches, while `extensions/feishu/src/outbound.test.ts` and `extensions/feishu/src/media.test.ts` do cover `mediaLocalRoots` forwarding in other paths.
  - command: `rg -n "mediaLocalRoots" extensions/feishu/src/reply-dispatcher.test.ts`
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)

## Risks / Open Questions

- Agent replies on Feishu that resolve to local image/file paths can still fail silently or degrade to text/card-only delivery because the local-root allowlist never reaches `sendMediaFeishu`.
- The bug is easy to miss because other Feishu outbound paths already pass `mediaLocalRoots`, and there is no targeted reply-dispatcher regression test for this case.
