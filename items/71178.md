## Summary

Keep #71178 open. Current main still lacks the `update.run` restart-continuation producer path that the maintainer explicitly called out, and the existing tests still assert that `update.run` writes no continuation. The linked PR #71573 remains the canonical fix attempt in the provided context, but this checkout has not incorporated that behavior.

## Best Possible Solution

Keep #71178 open until PR #71573 or an equivalent main-branch fix adds `continuationMessage` to the `update.run` protocol, forwards it from `src/agents/tools/gateway-tool.ts`, builds a restart success continuation in `src/gateway/server-methods/update.ts`, and updates the tests that currently assert `continuation` is undefined. After that lands, maintainers should either close this as fixed for the narrowed `update.run` continuation bug or split the remaining graceful-drain and post-reset bridge requests into explicit follow-up issues.
