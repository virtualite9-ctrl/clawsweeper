## Summary

Keep #71178 open. Current main still lacks the update.run restart-continuation producer path, while the existing restart-continuation consumer exists and the update tests still assert no continuation is written. The maintainer discussion explicitly says the specific update.run report should stay open, and the related PR #71573 is presented as an open, unmerged fix for only a subset of the issue.

## Best Possible Solution

Keep #71178 open until the update.run continuation fix lands on main, likely by adding continuationMessage to the update.run protocol schema, parsing/forwarding it through restart request and the agent gateway tool, and building a restart success continuation in src/gateway/server-methods/update.ts when a successful update restart is scheduled. After that lands, maintainers should decide whether graceful drain/fallback and system-prompt-reset bridge behavior remain in this issue or move to narrower follow-ups.
