## Summary

Keep open. Current main still omits the requester agent's configured identity name from `sessions_send` agent-to-agent prompt contexts, and the prior fix PR #38579 is closed unmerged. The implementation still passes requester session/channel metadata only into the initial message, reply, and announce prompt builders.

## Best Possible Solution

Implement the narrow core fix in `sessions_send`: derive the requester agent id from the effective requester session key, resolve the configured `agents.list[].identity.name`, pass an optional requester name through the initial message, reply, and announce A2A prompt builders, and add focused regression tests for all three contexts while preserving current output when the name is absent. PR #38579 is a useful prior reference but should be reapplied/reviewed against current main.
