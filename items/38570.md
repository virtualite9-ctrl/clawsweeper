## Summary

Keep open. Current main still omits the requester agent's configured identity name from the sessions_send agent-to-agent prompt contexts. The prior fix PR #38579 directly targeted this bug, but the provided GitHub context shows it was closed unmerged, and current main does not contain the requester-name plumbing or regression tests.

## Best Possible Solution

Implement the narrow core fix in sessions_send: derive the requester agent id from the effective requester session key, resolve the requester `agents.list[].identity.name` from the current config, and include `Agent 1 (requester) name: <name>.` in the initial message, reply, and announce A2A prompt contexts when the configured name is non-empty. Add focused regression tests for all three prompt contexts and preserve existing output when no requester identity name is configured. PR #38579 is a useful prior implementation reference, but it should be reapplied/reviewed against current main rather than treating the issue as fixed.
