## Summary

Keep open. Current main still omits the requester agent's configured identity name from `sessions_send` agent-to-agent prompt contexts. The related fix PR #38579 targeted this exact bug but is closed unmerged, and the current source still only passes requester session/channel metadata into the message, reply, and announce prompt builders.

## Best Possible Solution

Implement the narrow core fix in `sessions_send`: derive the requester agent id from the effective requester session key, resolve the requester `agents.list[].identity.name` from current config, and include `Agent 1 (requester) name: <name>.` in the initial message, reply, and announce A2A prompt contexts when the configured name is non-empty. Add focused regression tests for all three contexts and preserve current output when no requester identity name is configured. PR #38579 is a useful prior implementation reference but should be reapplied/reviewed against current main.
