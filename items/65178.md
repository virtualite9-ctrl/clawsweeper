## Summary

Keep this PR open. Current main still has the gateway image-support resolver using exact case-sensitive provider/model matching, so the linked configured-model image-drop bug remains plausible. The PR targets the right area, but the submitted fallback is too broad: it can make an explicit provider borrow image capability from a same-id model under another provider, which conflicts with the existing fail-closed contract and the review comments already left on the PR.

## Best Possible Solution

Keep this PR open for contributor or maintainer revision. The best patch is a narrow gateway fix: use `findModelInCatalog` for provider-scoped lookups, only run model-only fallback when no provider was supplied, preserve explicit-provider fail-closed behavior, and add regression tests for mixed-case provider/model IDs, provider aliases, providerless fallback, and provider-mismatch cases. If the configured-provider catalog gap from #65211 is still part of the same user-visible bug, fold that into the corrected gateway/catalog fix or track it with a canonical open follow-up.
