## Summary

Keep this PR open. Current main still uses exact case-sensitive provider/model matching in the gateway image-support resolver, so the reported mixed-case configured-model path remains plausible. The PR targets a real bug linked to #65165, but it should be revised before merge because its model-only fallback can bypass the existing explicit-provider fail-closed behavior.

## Best Possible Solution

Keep this PR open for maintainer or contributor follow-up. The best patch is a narrower gateway fix: use `findModelInCatalog` for provider-scoped lookups, only use model-only fallback when no provider was supplied, preserve explicit-provider fail-closed behavior, and add regression tests for mixed-case provider/model IDs, provider aliases, providerless fallback, and provider-mismatch cases. If the catalog gap from #65211 is still part of the same user-visible bug, fold that configured-provider model metadata path into the same corrected fix or track it explicitly.
