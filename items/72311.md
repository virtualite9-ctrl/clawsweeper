## Summary

Current main addresses the reported v2026.4.24 bundled-plugin dependency reinstall loop. The runtime-deps installer now anchors npm in an OpenClaw-owned package root, strips global/prefix npm settings, verifies package sentinels after npm exits, and tests cover the non-isolated external stage/global-prefix failure class. I found no tagged release containing this mainline fix yet.

## Best Possible Solution

Close as implemented on main. Keep the staged runtime-deps install root, npm env sanitization, package-manifest anchoring, and post-install sentinel verification; ship them in the next release. If this reproduces on a build containing b40b85c21ad61960e0db5b541b6f8ff10b7cff59 or later, the next issue should include npm config, resolved plugin stage dir, doctor output, and install logs.

## Evidence
