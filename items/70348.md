## Summary

Keep open. Current `main` still synthesizes compaction entries into `chat.history`, the web UI still renders those markers as chat dividers, and the `chat.history` sanitizer only filters assistant commentary/skip tokens today. The PR's behavior is therefore not already implemented on `main`. At the same time, the current patch direction has an unresolved correctness concern from review: its raw text-prefix filter could drop legitimate user or assistant messages that begin with the same phrases, so this needs maintainer product/contract judgment rather than conservative cleanup closure.
