## Summary

Keep open. Current main still does not implement this PR's requested chat.history filtering for compaction markers or pre/post-compaction prompts, and the PR itself needs revision because its text-prefix filtering can drop legitimate visible messages. The synthetic compaction marker also feeds an existing Web UI divider, so that part needs maintainer/product judgment rather than cleanup closure.

## Best Possible Solution

Keep the PR open and ask for a revision against current main's src/gateway/chat-display-projection.ts path. The safer implementation is provenance/metadata-backed filtering for internal memory-flush and post-compaction artifacts, with regression coverage for chat.history and session-history snapshots, while preserving or separately deciding the current compaction-divider UI behavior.
