## Summary

Keep #48488 open. Current main still has the core failure mode: `pump()` awaits each queued task directly, and lane completion plus the next `pump()` call only happen after that promise settles. The exact fix PR #48690 remains open/unmerged, and current task-maintenance/restart recovery work does not add an automatic task-level lane timeout.

## Best Possible Solution

Keep the issue open and use #48690 or an equivalent maintainer-owned patch as the implementation path: add a task-level timeout/abort policy to `src/process/command-queue.ts`, extend the enqueue type/options intentionally, preserve `resetAllLanes` generation safety, decide defaults/opt-outs for long compaction or cron work, and add focused command-queue tests proving a never-settling task releases the lane and pumps queued work.

## Evidence
