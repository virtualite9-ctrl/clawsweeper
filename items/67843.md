## Summary

Keep open. Current main still receives `agentDir` and `workspaceDir` in `applyExtraParamsToAgent` but does not forward them into the provider stream-wrapper context. The PR is a narrow implementation candidate with a direct regression test, and the broader Codex native web-search behavior remains separately tracked on #65909.

## Best Possible Solution

Keep this PR open for maintainer review and landing, or cherry-pick the same `agentDir`/`workspaceDir` forwarding into `wrapProviderStreamFn` plus the regression test. Keep broader native Codex web-search behavior questions on #65909.
