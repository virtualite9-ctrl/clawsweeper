## Summary

Keep open. Current main still omits `agentDir` and `workspaceDir` from the provider stream-wrapper context, while the existing provider hook contract and OpenAI/Codex stream wrapper already expect `agentDir` to reach Codex native web-search auth detection. The PR is a narrow implementation candidate with a regression test, and the related #65909 remains open for any broader end-to-end Codex native web-search behavior.

## Best Possible Solution

Keep this PR open for maintainer review and landing, or cherry-pick the same `agentDir`/`workspaceDir` forwarding into `wrapProviderStreamFn` plus the regression test. After landing, leave any broader Codex native web-search activation/tool-emission questions on #65909.
