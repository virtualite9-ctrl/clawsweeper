## Summary

Keep open. Current `main` has partial mitigation for Linux/systemd user-bus failures, but the gateway install path still hard-fails when both direct `systemctl --user` and the newer `--machine <user>@ --user` retry cannot reach the user bus. The CLI install flow also still surfaces the raw install error without the recovery hints that exist elsewhere.

## Evidence

- **Install still throws on unavailable user bus:** `installSystemdService()` calls `writeSystemdUnit()`, which calls `assertSystemdAvailable()`. That helper still throws `systemctl --user unavailable: ...` when the status probe cannot reach the user bus after retry handling.
  - file: [src/daemon/systemd.ts:414](https://github.com/openclaw/openclaw/blob/22f23fa5ab6f625835bfc3dbaba73c200775027c/src/daemon/systemd.ts#L414)
  - sha: [22f23fa5ab6f](https://github.com/openclaw/openclaw/commit/22f23fa5ab6f625835bfc3dbaba73c200775027c)
- **Machine-scope retry is only a best-effort fallback:** `execSystemctlUser()` first tries `systemctl --user ...`, then retries `systemctl --machine <user>@ --user ...` for bus-unavailable errors, but it just returns the failing result if that second call also cannot connect.
  - file: [src/daemon/systemd.ts:367](https://github.com/openclaw/openclaw/blob/22f23fa5ab6f625835bfc3dbaba73c200775027c/src/daemon/systemd.ts#L367)
  - sha: [22f23fa5ab6f](https://github.com/openclaw/openclaw/commit/22f23fa5ab6f625835bfc3dbaba73c200775027c)
- **Current shell-level repro still hits the same failure class:** On the review host, removing the session bus env reproduced the exact Ubuntu 24.04-style failure: both `env -u DBUS_SESSION_BUS_ADDRESS -u XDG_RUNTIME_DIR systemctl --user status` and `env -u DBUS_SESSION_BUS_ADDRESS -u XDG_RUNTIME_DIR systemctl --machine "$USER@" --user status` exited 1 with `Failed to connect to bus: No medium found`. That means the new retry path is not sufficient for all current-main environments.
  - command: `env -u DBUS_SESSION_BUS_ADDRESS -u XDG_RUNTIME_DIR systemctl --user status; env -u DBUS_SESSION_BUS_ADDRESS -u XDG_RUNTIME_DIR systemctl --machine "$USER@" --user status`
- **CLI install still emits only the raw failure:** `installDaemonServiceAndEmit()` catches install errors and calls `fail()` with `Gateway install failed: ...` only; it does not attach the Linux/systemd recovery hints that exist in `src/daemon/systemd-hints.ts`.
  - file: [src/cli/daemon-cli/response.ts:143](https://github.com/openclaw/openclaw/blob/22f23fa5ab6f625835bfc3dbaba73c200775027c/src/cli/daemon-cli/response.ts#L143)
  - sha: [22f23fa5ab6f](https://github.com/openclaw/openclaw/commit/22f23fa5ab6f625835bfc3dbaba73c200775027c)
- **The new `No medium found` coverage is probe-level, not install-success proof:** Current tests explicitly treat `systemctl --user unavailable: Failed to connect to bus: No medium found` as a non-fatal pre-install probe case in `maybeInstallDaemon()`, but that coverage does not prove the actual install succeeds once `service.install()` reaches `assertSystemdAvailable()`.
  - file: [src/commands/configure.daemon.test.ts:157](https://github.com/openclaw/openclaw/blob/3a5e535beddd67a2adb3dca02a83e27c239b0c7f/src/commands/configure.daemon.test.ts#L157)
  - sha: [3a5e535beddd](https://github.com/openclaw/openclaw/commit/3a5e535beddd67a2adb3dca02a83e27c239b0c7f)
