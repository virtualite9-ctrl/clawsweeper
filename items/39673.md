## Summary

Keep #39673 open. Current main has partial Linux/systemd hardening, but `openclaw gateway install` can still fail on the reported `systemctl --user unavailable: Failed to connect to bus` class during the real install path, and the install failure path still emits the raw failure without the systemd-unavailable remediation used by status output.

## Best Possible Solution

Keep #39673 open and fix the Linux gateway install path directly: either recover reliably for the reported user-bus/no-medium class or fail with explicit headless/Linux systemd remediation hints in `gateway install`. Add a regression test that exercises the real install path, not only the preliminary `isLoaded` probe, when `assertSystemdAvailable`, `daemon-reload`, `enable`, or `restart` sees `Failed to connect to bus` / no-medium output. Keep #63561 cross-linked as adjacent systemd-user detection work, but do not close this issue as its duplicate unless a maintainer chooses a single canonical tracker.
