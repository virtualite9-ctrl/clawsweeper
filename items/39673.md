## Summary

Keep #39673 open. Current main still has an unrecovered Linux user-systemd install failure path for `Failed to connect to bus` / no-medium-style errors, and `gateway install` still reports the raw install failure without the systemd-unavailable remediation hints that status output renders.

## Best Possible Solution

Keep #39673 open and fix the Linux gateway install path directly: either recover reliably for the reported user-bus/no-medium class or fail with explicit headless/Linux systemd remediation hints in `gateway install`. Add a regression test that exercises the real `service.install` / `writeSystemdUnit` / activation path when `assertSystemdAvailable`, `daemon-reload`, `enable`, or `restart` sees `Failed to connect to bus` / no-medium output. Keep #63561 cross-linked as adjacent systemd-user detection work, but do not close this issue as its duplicate unless a maintainer chooses one canonical tracker.
