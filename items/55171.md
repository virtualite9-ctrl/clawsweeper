## Summary

Keep open. Current `main` still treats runtime conversation bindings as authoritative cross-agent routes, and the PR's agent-mismatch cleanup would conflict with intentional `/focus` and ACP session binding behavior rather than proving the item is obsolete.

## Evidence

- **Runtime binding route on main preserves the bound session's agent:** `resolveRuntimeConversationBindingRoute()` resolves `boundAgentId` from the bound session key and rewrites the route to that agent/session without any mismatch-based unbind path. That means cross-agent bindings are current intended behavior on `main`, not stale by default.
