## Summary

Keep open. Current `main` still has the help-text gap this PR is trying to fix, but the proposed wording in #69975 is not accurate enough to land as-is because cron schedules without `--tz` resolve to the gateway host timezone, not UTC.

## Evidence

- **Current `cron add` help is still incomplete on `main`:** `cron add` currently says `--tz` is only for cron expressions, even though adjacent help already tells users to use `--tz` for offset-less `--at` datetimes. That means the underlying UX/docs problem is still present on `main`.
