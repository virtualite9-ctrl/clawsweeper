## Summary

Keep #69975 open. Current main still exposes cron-only `--tz` option help in both `cron add` and `cron edit`, while runtime code, tests, and docs confirm `--tz` also applies to offset-less ISO `--at` datetimes. The PR addresses a real remaining CLI help gap, but it should be amended before merge because the proposed `default: UTC` wording is wrong for `--cron`: cron schedules without `--tz` use the gateway host timezone, while offset-less `--at` defaults to UTC.

## Best Possible Solution

Keep the PR open and amend the help text: say `--tz` applies to `--cron` expressions and offset-less `--at` datetimes, align `cron edit --at` with `cron add`, and avoid a blanket `default: UTC` claim. If defaults are mentioned, split them explicitly: cron without `--tz` uses the gateway host timezone; offset-less `--at` without `--tz` is treated as UTC.
