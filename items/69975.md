## Summary

Keep #69975 open. Current main still has the misleading `--tz` help text in both `cron add` and `cron edit`, and the prior duplicate attempts are closed, so the PR tracks real remaining CLI help-text work. The PR should be amended before merge because its current `default: UTC` wording is wrong for `--cron`; cron schedules without `--tz` fall back to the gateway host timezone, while offset-less `--at` values default to UTC unless `--tz` is supplied.

## Best Possible Solution

Keep #69975 open and ask for a small amendment: retain the focused help-text clarification for both `cron add` and `cron edit`, but remove or qualify `default: UTC` so the help text does not imply cron expressions default to UTC. A suitable direction is wording that says `--tz` interprets `--cron` expressions and offset-less `--at` datetimes, with separate default behavior documented or kept out of the short option text.
