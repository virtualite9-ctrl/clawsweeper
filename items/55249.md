## Summary

Keep open. Current `main` has session-label storage and resolution in gateway/UI paths, but the user-facing `openclaw sessions` CLI still does not expose label management and does not show or return labels in its listing output, so the requested feature is not implemented end-to-end.

## Evidence

- **CLI surface lacks a label command:** The registered `openclaw sessions` command only supports listing flags (`--json`, `--verbose`, `--store`, `--agent`, `--all-agents`, `--active`) plus the `cleanup` subcommand. There is no `label` subcommand or other CLI mutation path for session labels.
  - file: [src/cli/program/register.status-health-sessions.ts:124](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/src/cli/program/register.status-health-sessions.ts#L124)
  - command: `nl -ba src/cli/program/register.status-health-sessions.ts | sed -n '120,220p'`
  - sha: [c070509b7f8f](https://github.com/openclaw/openclaw/commit/c070509b7f8fdbe479e5d829643782c272e30e83)
- **CLI docs do not document label support:** The public sessions CLI docs list only basic listing and cleanup usage. They do not describe label display, label editing, or label-based session management.
  - file: [docs/cli/sessions.md:12](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/docs/cli/sessions.md#L12)

- **Text-mode `openclaw sessions` does not print labels:** The CLI header and row formatter only emit `Agent`, `Kind`, `Key`, `Age`, `Model`, `Tokens (ctx %)`, and `Flags`. No label column is rendered.
