## Summary

Keep open. Current main has clearer ACP/acpx and Discord binding docs, including Gateway-host prerequisites and bind/thread guidance, but it still does not implement the reported topology where a Raspberry Pi Gateway selects a remote Mac mini node to run the ACP harness.

## Best Possible Solution

Keep this open for maintainer product judgment. The near-term docs fix should explicitly state that ACP/acpx harnesses execute on the Gateway host today and recommend running the Gateway/acpx on the Mac or using an operator-managed wrapper when that is the desired deployment. If OpenClaw should support this topology, design an additive ACP runtime node selector plus node-host process execution support rather than overloading cwd, thread bindings, or backend ids.
