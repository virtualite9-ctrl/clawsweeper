## Summary

Keep #38671 open. Current main still only supports cron `systemEvent` and `agentTurn` payloads, and detached cron execution still goes through the isolated agent/model path rather than a direct shell-command runner. Related PR #62789 is useful prior implementation work, but the provided context shows it is open, unmerged, backend/runtime/server only, and explicitly split from CLI support.

## Best Possible Solution

Keep #38671 open until OpenClaw either ships direct command cron support end to end or maintainers explicitly reject the product direction. The likely implementation is an additive cron payload kind for direct command execution, backend execution that bypasses the agent for pure command jobs, optional command-output-to-agent hybrid mode, CLI/UI/docs support, and regression coverage for validation, timeout/termination, output truncation, failures, delivery, run logs, and security policy. PR #62789 is useful prior art but should not supersede the issue unless it lands with the missing surfaces or a narrower canonical follow-up is opened.
