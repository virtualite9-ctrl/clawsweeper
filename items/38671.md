## Summary

Keep open. Current main still does not implement direct shell-command cron jobs or the proposed command+message hybrid path. Cron payloads are limited to systemEvent and agentTurn, the CLI requires exactly --system-event or --message, and execution still routes main jobs through heartbeat/system events and detached jobs through an isolated agent turn. The issue discussion adds concrete use cases and edge cases, and there is no provided related plugin/extension or canonical duplicate that would justify closing it as clawhub or duplicate.
