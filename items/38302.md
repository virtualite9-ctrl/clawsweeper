## Summary

Keep #38302 open. Current main still does not support per-account Slack native command prefixes or account-level native-name mappings for built-in slash commands. Slack account config exposes only the existing slashCommand fields, the strict schema/metadata do not allow nativePrefix or nativeNames, the command registry resolves native names only by provider, and Slack native registration/dispatch calls those provider-scoped helpers without passing account slash-command naming data. The issue also has a second user confirming the multi-app Slack collision, while the two linked implementation attempts (#38326 and #38391) are closed and unmerged in the provided GitHub context.

## Best Possible Solution

Keep the issue open and implement this in the bundled Slack plugin/core command contract. A maintainer-grade solution should add a validated account-level Slack native command naming contract, likely nativePrefix-derived names or an explicit nativeNames map; thread the resolved account config through native command spec generation, Slack registration, argument-menu values, and findCommandByNativeName dispatch; keep backward compatibility for current unprefixed commands and /agentstatus; update Slack setup/docs; and add focused multi-account tests proving prefixed native commands route to the intended built-in commands.
