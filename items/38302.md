## Summary

Keep #38302 open. Current main still lacks the requested per-account Slack native command prefix or native-name mapping. Slack account config and validation expose only the existing slashCommand fields, native command name resolution is provider-scoped, and Slack native registration/dispatch does not pass account naming data. The issue discussion includes a second user confirming the same multi-app Slack collision, and the related implementation attempts #38326 and #38391 are closed and unmerged in the provided context.

## Best Possible Solution

Keep the issue open and implement it in the bundled Slack plugin plus the native-command contract. A maintainer-grade fix should add a validated account-level Slack native naming option, either nativePrefix-derived names or an explicit nativeNames map; thread the resolved account config through native spec generation, Slack registration, argument-menu values, and findCommandByNativeName dispatch; preserve current unprefixed behavior and the /status to /agentstatus override; update Slack setup docs; and add focused multi-account tests proving prefixed native commands route to the intended built-in commands.
