## Summary

Keep #37661 open. Current main has tool-call loop detection and several exact duplicate-delivery suppressions, but I did not find an output-level guard that detects repeated assistant text generation, repeated sentence/ngram patterns, or a per-turn duplicate outbound-message circuit breaker. The April 11 reproduction comment and related #39536 support that this is a text/output flood problem, not just #34574's tool-call loop gap.

## Best Possible Solution

Keep this issue open and implement an output-level guard. The likely product path is a focused assistant-text repetition detector in the streaming/final reply pipeline, plus an outbound per-turn duplicate/message cap where channel sends are planned, with tests covering repeated deltas, repeated sentence fragments in one assistant message, and repeated channel sends. Keep #34574 separate for tool-call loops because this issue is about pure text generation/output flooding.
