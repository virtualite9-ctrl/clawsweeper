## Summary

Keep #37661 open. Current main still has tool-call loop detection and exact duplicate/event suppression, but I found no assistant-output repetition/ngram guard or shared outbound per-turn flood cap that would stop the reported pure text repetition and channel spam. The Apr 11 follow-up is important because it confirms the problem is not limited to subagent timeout and remains distinct from #34574's tool-call loop issue.

## Best Possible Solution

Keep this issue open. Implement a conservative output-level repetition guard in the assistant streaming/final reply pipeline, plus a shared per-turn outbound duplicate/message cap before channel delivery. Add focused tests for repeated deltas, repeated sentence/ngram fragments within one assistant message, block streaming/final replay, and repeated channel sends. Keep #34574 separate because it tracks tool-call loops, while #37661 tracks pure assistant text/output flooding.

## Risks / Open Questions

- The exact trigger is provider/model-specific and intermittent, so regression coverage should use synthetic repeated-delta, repeated final-text, and repeated outbound-send fixtures rather than relying on the reporter's custom endpoint.
- A text repetition guard can false-positive on legitimate repeated logs, code blocks, tables, or quoted text; thresholds should be conservative and preferably configurable.
