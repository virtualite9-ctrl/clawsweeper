## Summary

Keep #37661 open. Current main still has tool-call loop detection and exact duplicate suppression, but I did not find a guard that detects repeated assistant text fragments/ngrams or a per-turn outbound message cap that would stop the reported pure text repetition flood. The related #34574 remains a separate tool-call loop issue, while closed #39536 is related evidence for the same output-level circuit-breaker gap rather than a canonical open replacement.

## Best Possible Solution

Keep this issue open. Implement an output-level repetition guard in the assistant streaming/final reply pipeline, plus a per-turn outbound duplicate/message cap in dispatch/delivery planning. Add focused tests for repeated deltas, repeated sentence/ngram fragments in one assistant message, repeated block-stream payloads, and repeated channel sends. Keep #34574 separate because it tracks tool-call loops, not pure text output repetition.
