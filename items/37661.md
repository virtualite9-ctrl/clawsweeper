## Summary

Keep #37661 open. Current main still has tool-call loop detection and exact duplicate/event suppression, but I did not find an output-level guard for repeated assistant text fragments/ngrams or a per-turn outbound message cap that would stop the reported pure text/channel flood. The related #34574 is still a separate tool-call loop issue, and closed #39536 is supporting evidence for the same output circuit-breaker gap rather than a canonical replacement.

## Best Possible Solution

Keep this issue open. Implement an output-level repetition guard in the assistant streaming/final reply pipeline, plus a per-turn outbound duplicate/message cap in dispatch/delivery planning. Add focused tests for repeated deltas, repeated sentence/ngram fragments in one assistant message, repeated block-stream payloads, and repeated channel sends. Keep #34574 separate because it tracks tool-call loops, not pure text output repetition.
