## Summary

Current main still lacks a documented user-facing `sessions_spawn` policy to suppress or redirect native run-mode sub-agent completion announcements. ACP-only `streamTo: "parent"` and nested-subagent internal delivery do not cover the reported iMessage/DM native sub-agent use case.

## Best Possible Solution

Add a documented `sessions_spawn` completion delivery policy, preserving current channel delivery as the default. A small enum such as `completionDelivery: "channel" | "parent" | "none"` should thread through the tool schema, native spawn params, ACP tracking as applicable, registry records, announce dispatch, task delivery status, docs, and focused regression tests for top-level channel-bound sessions such as BlueBubbles/iMessage, WhatsApp, and Telegram. Reuse the existing nested-subagent internal injection path for parent-only delivery where possible.
