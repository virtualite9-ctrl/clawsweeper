## Summary

Keep open. Current main still lacks a documented, user-facing `sessions_spawn` policy to suppress or redirect native run-mode sub-agent completion announcements for top-level channel-bound sessions. Existing code still defaults normal native sub-agent completions to external channel delivery when a requester route exists; nested sub-agent internal delivery and ACP-only `streamTo: "parent"` do not cover the reported iMessage/DM native sub-agent use case.

## Best Possible Solution

Add an explicit, documented `sessions_spawn` completion delivery policy for native and ACP run-mode spawns, preserving current channel delivery as the default. A small enum such as `completionDelivery: "channel" | "parent" | "none"` should thread through the tool schema, spawn params, registry record, announce dispatch, task delivery status, docs, and regression tests for top-level channel-bound sessions like iMessage/BlueBubbles, WhatsApp, and Telegram. Reuse the existing nested-subagent internal injection path for parent-only delivery where possible.
