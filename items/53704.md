## Summary

Keep open. Current `main` still uses `agents.defaultId` for the Agents UI "Set Default" action, but the config schema only accepts `agents.defaults` and `agents.list`, and runtime default resolution reads `agents.list[].default`. The report remains plausible and is not implemented on `main`.

## Evidence

- **Current UI still writes `agents.defaultId`:** The Agents panel `onSetDefault` handler on current `main` still calls `updateConfigFormValue(state, ["agents", "defaultId"], agentId)`, so the unsupported key is still being written from the UI path discussed in the PR.
