## Summary

Keep PR #51964 open. Current main still returns `file-type`'s sniffed MIME verbatim, so `image/apng` can flow through the gateway attachment path as the final image MIME. The provided PR is a narrow fix for linked bug #51881 and is not implemented or superseded on main.

## Best Possible Solution

Keep the PR open as a focused bug fix. The best path is to normalize sniffed `image/apng` to `image/png` in the shared media MIME detection path, add a targeted regression test for `detectMime`/gateway attachment parsing, and update the changelog with proper attribution before landing.
