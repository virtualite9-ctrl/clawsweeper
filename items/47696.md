## Summary

Keep open. Current `main` does not implement the proposed `/trust` and `/untrust` session-scoped exec elevation flow, and the active exec override path still relies only on inline directives, persisted session exec settings, and agent defaults.

## Evidence

- **Slash-command docs on main:** Current docs list `/elevated` and `/exec`, but no `/trust` or `/untrust` commands.
  - file: [docs/tools/slash-commands.md:101](https://github.com/openclaw/openclaw/blob/d38ed0831d7372c8505dc944aea69b5b83da1435/docs/tools/slash-commands.md#L101)
- **Built-in command registry on main:** The built-in registry defines `elevated` and `exec` commands in this section; there is no `trust`/`untrust` command definition nearby or elsewhere in the registry search results.
  - file: [src/auto-reply/commands-registry.shared.ts:782](https://github.com/openclaw/openclaw/blob/d38ed0831d7372c8505dc944aea69b5b83da1435/src/auto-reply/commands-registry.shared.ts#L782)
- **Command handlers on main:** The loaded command-handler list has no `handleTrustCommand` or `handleUntrustCommand`, so there is no runtime entry point for the feature on current `main`.
  - file: [src/auto-reply/reply/commands-handlers.runtime.ts:37](https://github.com/openclaw/openclaw/blob/d38ed0831d7372c8505dc944aea69b5b83da1435/src/auto-reply/reply/commands-handlers.runtime.ts#L37)
- **Exec override resolution on main:** Effective exec overrides still come from inline directives, `sessionEntry`, and agent defaults only; there is no trust-window override in the current resolution path.
  - file: [src/auto-reply/reply/get-reply-exec-overrides.ts:7](https://github.com/openclaw/openclaw/blob/d38ed0831d7372c8505dc944aea69b5b83da1435/src/auto-reply/reply/get-reply-exec-overrides.ts#L7)
- **Reply directive wiring on main:** `resolveReplyDirectives` still calls `resolveReplyExecOverrides` with only `directives`, `sessionEntry`, and `agentExecDefaults`, which matches the absence of a temporary trust layer.
  - file: [src/auto-reply/reply/get-reply-directives.ts:587](https://github.com/openclaw/openclaw/blob/d38ed0831d7372c8505dc944aea69b5b83da1435/src/auto-reply/reply/get-reply-directives.ts#L587)
- **History search:** A `git log --oneline --grep='/trust' --grep='untrust' --grep='trust window' --all --decorate` search in the checkout returned no matching landed history on current `main`.
