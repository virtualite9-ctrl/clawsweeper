## Summary

Keep this issue open. Current main still reads workspace bootstrap files through the boundary-confined file opener, turns guard failures into `missing: true`, and has no shipped fallback for out-of-workspace symlinked bootstrap files. The linked implementation candidate, PR #40230, is still open and unmerged with explicit closing syntax, so the issue should remain open until that PR is reviewed/landed or rejected.

## Best Possible Solution

Keep the issue open and use PR #40230 as the concrete implementation candidate. Maintainers should review whether trusted workspace bootstrap symlinks may intentionally resolve outside the workspace; if yes, land or rework a targeted `src/agents/workspace.ts` fallback with regression coverage for out-of-workspace symlinked bootstrap files while preserving the existing security guards. Related open reports #38622 and #64472 can then be consolidated after the fix ships.
