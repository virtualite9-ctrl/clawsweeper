## Summary

Current main still treats out-of-workspace symlinked workspace bootstrap reads as guarded read failures and then renders them as missing. The linked implementation PR remains open and unmerged, so this issue should stay open for maintainer review.

## Best Possible Solution

Keep this issue open. Maintainers should review PR #40230 as the concrete implementation candidate, decide whether trusted workspace bootstrap symlinks may resolve outside the workspace, and if accepted land a targeted `src/agents/workspace.ts` fallback with regression coverage. After a fix ships, consolidate the overlapping open reports #38622 and #64472.
