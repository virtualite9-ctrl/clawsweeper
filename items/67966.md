## Summary

Current main still does not implement local-managed Playwright navigation interception. Local-managed profiles keep `usesPersistentPlaywright: false`, `openTab` only calls `createPageViaPlaywright` when that capability is true, and explicit strict SSRF mode still requires inspectable redirect-hop handling for HTTP(S) URLs before raw CDP tab creation can run. The existing Playwright guarded-navigation path exists for remote CDP profiles, but it is not selected for local-managed browser mode.

## Best Possible Solution

Keep this issue open and implement a focused `extensions/browser` change that routes local-managed OpenClaw profiles through the existing Playwright-backed `createPageViaPlaywright` and `gotoPageWithNavigationGuard` path when strict SSRF policy requires redirect-hop inspection, while preserving Chrome MCP existing-session behavior and safe raw-CDP fallbacks where Playwright is unavailable. Add local-managed strict-mode regression tests for IP-literal navigation and redirect blocking, plus docs if the browser capability matrix changes.
