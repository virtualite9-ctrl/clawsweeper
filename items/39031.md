## Summary

Keep #39031 open. Current main still does not implement legacy iMessage `imsg rpc` tapback forwarding to the agent: the monitor subscribes to ordinary messages only, the payload/parse/config surfaces do not model reaction metadata, and the inbound path dispatches text messages rather than reaction/system events. The related context confirms #39322 is still the open implementation PR, while #60677 is only a filtering fix and #60446 was closed in favor of this canonical issue.

## Best Possible Solution

Keep #39031 open as the canonical legacy iMessage `imsg rpc` tapback-to-agent feature request. The best path is to review, rebase, or supersede #39322 with a typed implementation that subscribes to reaction notifications, parses target/reaction metadata, routes tapbacks as reaction/system events rather than ordinary text, defines access-control and dedupe behavior, and updates config/docs/tests. If legacy `imsg` reaction forwarding is intentionally out of scope, document that explicitly and point users to BlueBubbles for richer iMessage reaction support.
