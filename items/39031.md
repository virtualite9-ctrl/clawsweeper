## Summary

Keep #39031 open. Current main still does not implement legacy `channels.imessage` / `imsg rpc` tapback forwarding to the agent: the monitor subscribes only for message notifications with attachment metadata, the parser/types do not model reaction-event fields for dispatch, inbound processing only emits ordinary text-message contexts, and the iMessage docs/config expose attachments but no reaction forwarding control. BlueBubbles does implement reaction system events, but that is the separate recommended iMessage path and does not satisfy this legacy imsg feature request. Related context supports #39031 as canonical: #60446 was closed back to this issue, while #39322 remains an open implementation candidate.

## Best Possible Solution

Keep #39031 open as the canonical legacy iMessage `imsg rpc` tapback-to-agent feature request. Maintainers should review, rebase, or supersede #39322 with a typed implementation that covers reaction subscription, payload parsing, access control, debounce/dedupe behavior, system-event vs text-message semantics, config/docs, and regression tests. If legacy imsg reaction support is intentionally out of scope, make that explicit in #39031 and document BlueBubbles-only reaction parity.
