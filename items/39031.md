## Summary

Current main does not implement the requested legacy iMessage `imsg rpc` tapback forwarding. The related discussion is useful: BlueBubbles already has tapback/reaction system events, and #60677/#60274 cover filtering tapbacks that are misclassified as regular messages, but that does not satisfy #39031's requested inbound reaction event surface for the iMessage plugin.

## Evidence

- **Issue asks for iMessage tapbacks to become agent events:** #39031 requests forwarding tapback metadata such as reacted message id, emoji, sender, and timestamp to the agent. The provided related #60274/#60677 context describes a narrower bug fix: filtering tapback text so it is not treated as a normal message.
