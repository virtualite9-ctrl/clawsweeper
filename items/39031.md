## Summary

Keep #39031 open. Current main still does not forward legacy iMessage `imsg rpc` tapback reactions to the agent: the iMessage monitor only subscribes to ordinary message notifications, the payload/parser/inbound context are text-message oriented, and the iMessage config/docs expose attachments but no reaction notification mode. BlueBubbles now has separate reaction system events, but that is the recommended alternate iMessage path and does not implement this legacy `channels.imessage` request. Related context also points to this issue as canonical: #60446 was closed back to #39031, while #39322 remains an open implementation PR and #60677 is an open filtering PR, not a shipped forwarding fix.

## Best Possible Solution

Keep #39031 open as the canonical legacy iMessage `imsg rpc` tapback-to-agent feature request. Maintainers should review, rebase, or supersede #39322 with a typed implementation that subscribes to reaction notifications, parses target/reaction metadata, routes them as reaction/system events instead of ordinary text, defines access-control and dedupe behavior, and updates config/docs/tests. If legacy imsg reaction forwarding is intentionally out of scope, document that explicitly and point users to BlueBubbles for reaction-event support.
