## Summary

Keep this issue open. Current main has a first-class bundled browser plugin with headless launch support, remote CDP, custom launch args, and optional sandbox/browser Docker paths, but it still does not satisfy the central request to bundle a Chromium runtime that works without external browser setup. Local managed launch still depends on a configured or system-installed Chromium-family executable, and Docker/browser binary installation remains opt-in/manual.

## Best Possible Solution

Keep this issue open and scope it to the remaining product decision: ship a pinned Chromium runtime by default, publish/advertise an official browser-enabled Docker variant, or add installer/doctor flows that automatically install and persist Playwright Chromium and wire it into executable discovery. Reuse the existing browser plugin, headless config, remote CDP support, extraArgs, and sandbox-browser container rather than creating a second browser-control stack.
