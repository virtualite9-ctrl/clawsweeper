## Summary

Keep open. Current `main` has already moved macOS gateway restart and auto-update restart flows to a `kickstart`-first pattern, which addresses part of the report. But the LaunchAgent plist still hardcodes `ThrottleInterval = 1` and still does not emit an `ExitTimeOut` key, so the issue's core launchd configuration concerns are not implemented on `main`.

## Evidence

- **LaunchAgent throttle is still hardcoded to 1:** `LAUNCH_AGENT_THROTTLE_INTERVAL_SECONDS` is still exported as `1` on current `main`.
  - file: [src/daemon/launchd-plist.ts:6](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/src/daemon/launchd-plist.ts#L6)
  - sha: [6bc0dc8fb671](https://github.com/openclaw/openclaw/commit/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440)
- **Generated plist still omits ExitTimeOut:** `buildLaunchAgentPlist()` renders `RunAtLoad`, `KeepAlive`, `ThrottleInterval`, `Umask`, `ProgramArguments`, and log paths, but no `ExitTimeOut` key.
  - file: [src/daemon/launchd-plist.ts:116](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/src/daemon/launchd-plist.ts#L116)
  - sha: [6bc0dc8fb671](https://github.com/openclaw/openclaw/commit/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440)
- **Normal gateway restart is now kickstart-first:** `restartLaunchAgent()` enables the service and calls `launchctl kickstart -k`, only falling back to `bootstrap` if the service is not loaded. The normal restart path does not `bootout` the service.
  - file: [src/daemon/launchd.ts:656](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/src/daemon/launchd.ts#L656)
  - sha: [6bc0dc8fb671](https://github.com/openclaw/openclaw/commit/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440)
- **Auto-update restart helper is also kickstart-first:** The detached macOS update restart script tries `launchctl kickstart -k` first, then `enable` + `bootstrap` + `kickstart` if needed.
  - file: [src/cli/update-cli/restart-helper.ts:119](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/src/cli/update-cli/restart-helper.ts#L119)
  - sha: [6bc0dc8fb671](https://github.com/openclaw/openclaw/commit/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440)
- **Tests explicitly lock in no-bootout restart behavior:** The launchd tests assert that restart uses `kickstart` and does not call `bootout`.
  - file: [src/daemon/launchd.test.ts:649](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/src/daemon/launchd.test.ts#L649)
  - sha: [6bc0dc8fb671](https://github.com/openclaw/openclaw/commit/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440)
