## Summary

Current main still lacks memory-lancedb per-agent scoping, and this draft PR is the only open implementation candidate in the provided context. It should stay open, but it needs a rebase plus correctness and security fixes before maintainer review can consider merging it.

## Best Possible Solution

Keep this PR open as a plausible bundled-plugin implementation candidate. The best path is to rebase onto current main, keep the change inside extensions/memory-lancedb, use trusted ctx.agentId consistently in tool factories and hook contexts, add a backward-compatible agentScoping/config/manifest surface, fix LanceDB filtering and migration observability, and add regression coverage for agent A/agent B isolation, legacy shared memories, duplicate detection, forget/delete scoping, and default unscoped compatibility.
