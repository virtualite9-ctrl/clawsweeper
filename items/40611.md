## Summary

Current main still preserves the reported heartbeat behavior: a requests-in-flight skip is retried through the shared 1s wake retry, and there is no heartbeat-specific retryDelayMs/busy-session backoff config. The linked fixing PR is closed unmerged, while the related open items cover broader lock-scope or activity-based heartbeat designs rather than this regression.

## Best Possible Solution

Keep this issue open and repair the heartbeat scheduling path. The best product direction is a refreshed version of #40629: add a separate requests-in-flight heartbeat retry delay, likely around 60s by default, decide whether it should be configurable under `agents.defaults.heartbeat` and per-agent heartbeat config, then update schema/types/generated config metadata/docs plus wake and scheduler regression tests. Leave #13744 for broader session-lock scope/configuration and #14051 for activity-based heartbeat semantics.
