## Summary

Current main still has the reported heartbeat retry/lock contention path: `requests-in-flight` skips are retried through the shared 1s wake retry, tests assert that 1s retry behavior, and the requested `heartbeat.retryDelayMs` or separate in-flight retry delay is not present in current schema/docs/types. The related fixing PR #40629 was closed unmerged, while #13744 and #14051 remain broader related work rather than canonical replacements for this regression.

## Best Possible Solution

Keep this issue open and fix it in core heartbeat scheduling. Add a separate requests-in-flight retry delay, likely defaulting around 60s, and decide whether to expose it as `agents.defaults.heartbeat.retryDelayMs` plus per-agent heartbeat config. Wire schema/types/generated config metadata/docs, update the wake/runner tests so busy-session skips no longer assert the shared 1s retry, and keep #13744 for broader session-lock scope/configuration and #14051 for activity-based heartbeat behavior.
