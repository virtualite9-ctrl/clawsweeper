## Summary

Current main still has the reported heartbeat busy-session retry behavior: requests-in-flight skips are requeued through the shared 1s wake retry, runner comments/tests explicitly preserve that behavior, and there is still no heartbeat retryDelayMs or equivalent config/docs surface. The related fixing PR #40629 is closed unmerged, while #13744 and #14051 are broader open follow-ups rather than replacements for this regression.

## Best Possible Solution

Keep this issue open and fix the core heartbeat scheduling path. A repaired version of #40629 is the closest product direction: add a separate busy-session/request-in-flight heartbeat retry delay, likely defaulting around 60s, decide whether to expose it as `agents.defaults.heartbeat.retryDelayMs` plus per-agent config, and update schema/types/config reload/generated metadata/docs plus heartbeat wake and scheduler tests so busy-session skips no longer assert the shared 1s retry. Leave #13744 for broader session-lock scope/configuration and #14051 for activity-based heartbeat semantics.
