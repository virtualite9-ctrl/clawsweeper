## Summary

Keep PR #47687 open. Current main still only supports generic Telegram emoji reaction notifications: no `reactionSemantics` config, no stable `reaction_key`, no custom-emoji reaction normalization, and no semantic heartbeat wake from the Telegram reaction path. The PR remains a focused implementation candidate for the still-open #47677 request, and the provided review thread shows the author addressed concrete review feedback.

## Best Possible Solution

Keep PR #47687 open as the active implementation candidate for #47677. Maintainers should review the Telegram-owned runtime/config design and the public SDK wake seam, then land a refined version with docs/tests or redirect the work into a narrower canonical issue if the API shape needs redesign.
