## Summary

Keep open. Current main at 631552c554095b0c520376db9131dac0f5427f94 still has only generic Telegram reaction notifications: standard emoji are enqueued as system events, but there is no `reactionSemantics` config, no stable `reaction_key`, no custom-emoji reaction input normalization, and no heartbeat wake request from the reaction path. PR #47687 is a focused implementation for the still-open #47677 request, and the provided review thread shows the author addressed several concrete review findings rather than abandoning them.

## Best Possible Solution

Keep PR #47687 open as the active implementation candidate for #47677. Maintainers should review the public SDK wake seam and the Telegram-owned runtime/config design, then either land a refined version with docs and tests or redirect the remaining work into a narrower canonical issue if the API shape needs redesign.
