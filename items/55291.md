## Summary

Keep open. Current `main` still has the orphaned-media cleanup gap described by PR #55291, and the existing test file does not cover the later failure paths the PR adds. I did not find evidence in the checked-out `main` that this fix already landed.

## Evidence

- **Current implementation only removes the saved file in one failure branch:** `ensureMediaHosted()` saves media first, then calls `getTailnetHostname()` and checks port availability. Cleanup via `fs.rm(saved.path)` happens only when the port is free and `startServer` is false, so later failures outside that branch still leave the saved file behind.
  - file: [src/media/host.ts:32](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/src/media/host.ts#L32)
  - command: `nl -ba src/media/host.ts | sed -n '1,120p'`
  - sha: [c070509b7f8f](https://github.com/openclaw/openclaw/commit/c070509b7f8fdbe479e5d829643782c272e30e83)
- **Hostname lookup can throw before any cleanup path runs:** `getTailnetHostname()` shells out to `tailscale status --json` and throws the last error if lookup fails, which matches one of the PR's reported leak paths.
  - file: [src/infra/tailscale.ts:101](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/src/infra/tailscale.ts#L101)
  - command: `sed -n '1,220p' src/infra/tailscale.ts`
  - sha: [c070509b7f8f](https://github.com/openclaw/openclaw/commit/c070509b7f8fdbe479e5d829643782c272e30e83)
- **Unexpected port-check errors are rethrown, not normalized into the existing cleanup branch:** `ensurePortAvailable()` rethrows non-`EADDRINUSE` errors, and `isPortFree()` rethrows anything except `PortInUseError`, so unexpected probe failures after save also escape without cleanup on current `main`.
  - file: [src/infra/ports.ts:29](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/src/infra/ports.ts#L29)

  - sha: [c070509b7f8f](https://github.com/openclaw/openclaw/commit/c070509b7f8fdbe479e5d829643782c272e30e83)
- **Current tests only cover the existing 'server not allowed to start' cleanup case:** `src/media/host.test.ts` has cases for cleanup when server start is disallowed, for successful server start, and for port-already-in-use reuse. It does not cover hostname lookup failure, unexpected port-check failure, or media-server startup failure.
  - file: [src/media/host.test.ts:112](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/src/media/host.test.ts#L112)
