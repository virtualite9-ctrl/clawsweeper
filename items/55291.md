## Summary

Keep #55291 open. Current main still has the post-save media cleanup gap the PR targets: `ensureMediaHosted()` persists media first, then can throw during hostname lookup, unexpected port probing, or media server startup without deleting the saved file. The PR is a focused active bug fix with regression coverage for those exact failure paths, and it has recent author follow-up rather than stale/low-signal characteristics.

## Best Possible Solution

Do not close this PR as implemented, stale, duplicate, or out of scope. The best path is maintainer review of the focused cleanup change, preserving the original thrown error while making cleanup best-effort for every failure after `saveMediaSource()` succeeds.
