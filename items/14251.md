## Summary

Keep #14251 open. Current main still manages macOS gateway lifecycle through per-user LaunchAgent helpers only; system-level /Library/LaunchDaemons detection is limited to diagnostic status scans and does not provide stop/uninstall scope handling or sudo guidance. The prior issue discussion also contains an April 26 maintainer/Codex comment explicitly keeping this open, and related #14315 is a closed narrower per-user orphan LaunchAgent cleanup issue, not a replacement for this LaunchDaemon lifecycle request.

## Best Possible Solution

Keep the issue open until macOS gateway lifecycle management becomes launchd-scope-aware. A good fix should model user vs system scope, detect /Library/LaunchDaemons/ai.openclaw.gateway.plist, provide exact sudo guidance or explicit --system stop/uninstall handling, ensure status does not imply the managed service is stopped while a KeepAlive system daemon remains active, and add focused tests/docs for both user LaunchAgents and system LaunchDaemons.
