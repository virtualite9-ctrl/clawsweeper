## Summary

Keep #14251 open. Current main still does not make macOS gateway lifecycle scope-aware for system LaunchDaemons: stop/uninstall remain wired to the per-user LaunchAgent service path, while system LaunchDaemon detection is limited to status --deep discovery and cleanup hints. The April 26 maintainer/Codex comment in the issue also explicitly kept this open and treated #14315 as narrower, not superseding this request.

## Best Possible Solution

Keep the issue open until macOS gateway lifecycle management becomes launchd-scope-aware: detect /Library/LaunchDaemons/ai.openclaw.gateway.plist during stop/uninstall, avoid success responses while a KeepAlive system daemon can remain active, provide exact sudo bootout/rm guidance or explicit --system handling, and add focused tests/docs for user LaunchAgents versus system LaunchDaemons.
