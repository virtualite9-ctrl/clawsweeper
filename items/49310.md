## Summary

Keep open. Current `main` still drops the TUI back to `idle` after final chat events and does not restore any follow-up busy state from session history, so the PR’s requested behavior is not implemented on `main`.

## Evidence

- **sessions_yield marker exists in current main:** The runner still persists a specific follow-up context marker for `sessions_yield`, so there is a concrete signal the TUI could use.
  - file: [src/agents/pi-embedded-runner/run/attempt.sessions-yield.ts:10](https://github.com/openclaw/openclaw/blob/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4/src/agents/pi-embedded-runner/run/attempt.sessions-yield.ts#L10)

  - file: [src/tui/tui-event-handlers.ts:197](https://github.com/openclaw/openclaw/blob/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4/src/tui/tui-event-handlers.ts#L197)
