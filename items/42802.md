## Summary

Keep open. Current main still supports documented `bindings[]` for Discord route/ACP bindings, but it does not implement the requested selected-agent Channel Bindings display in the Gateway Agents tab or the proposed read-only `routing.list` RPC. The same-author external implementation candidate linked in the discussion should be reviewed or ported, not treated as a reason to close this issue.

## Best Possible Solution

Keep this issue open. The best maintainer path is to review or port the linked same-author implementation candidate, then add a scoped read-only routing-bindings API or safe config-derived equivalent, expose typed UI state/controller loading, render selected-agent route and ACP bindings in Agents -> Channels, and add focused Gateway/UI coverage before closing as implemented.
