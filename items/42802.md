## Summary

Current main still lacks the requested selected-agent Discord/channel binding display and the proposed read-only `routing.list` Gateway RPC. The same-author external implementation candidate linked in the discussion is useful evidence of a possible patch path, but it has not landed in `openclaw/openclaw`, so this issue should stay open.

## Best Possible Solution

Keep this issue open. The maintainer path is to review or port the linked same-author implementation candidate, add a scoped read-only route-binding source such as `routing.list` or an equivalent safe config-derived API, authorize it under the correct read scope, wire typed UI state/loading/error handling, render selected-agent route and ACP bindings in Agents -> Channels, and add focused Gateway/UI coverage before closing as implemented.
