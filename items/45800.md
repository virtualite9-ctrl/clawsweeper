## Summary

Keep open. Current main still renders the WebChat plus/New session control next to the send controls and wires it directly to `handleSendChat("/new")` with no confirmation or relocation. The related PR #45863 appears to be the direct proposed fix in the provided GitHub context, but it is open and unmerged, so the issue is not implemented on main.

## Best Possible Solution

Keep #45800 open until WebChat either confirms destructive New session clicks or relocates/separates the plus control enough to avoid accidental resets. The likely implementation point is `ui/src/ui/chat/run-controls.ts` plus the `onNewSession` wiring in `ui/src/ui/app-render.ts`, with regression coverage that canceling the prompt does not send `/new` and preserves the draft/session.
