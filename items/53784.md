## Summary

Keep open. Current `main` does not implement the PR’s Feishu audio-format routing change: `extensions/feishu/src/media.ts` still recognizes only `.opus`/`.ogg` (and `audio/ogg`/`audio/opus`) as playable audio, and the Feishu test suite still asserts that an MP3 is sent as a generic file attachment.

## Evidence

- **Current Feishu routing still hardcodes only opus/ogg audio:** `detectFileType()` returns `"opus"` only for `.opus` and `.ogg`, and `resolveFeishuOutboundMediaKind()` only maps `.opus`/`.ogg` plus `audio/ogg`/`audio/opus` to `msgType: "audio"`. There is no `isAudioFileName()` or `mimeKind === "audio"` fallback in current `main`.
  - file: [extensions/feishu/src/media.ts:496](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/extensions/feishu/src/media.ts#L496)
  - command: `nl -ba extensions/feishu/src/media.ts | sed -n '496,568p'`
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Current Feishu test expects MP3 to remain a generic file:** The existing test case is named `falls back to generic file for unsupported audio formats` and asserts `song.mp3` uploads as `file_type: "stream"` with `msg_type: "file"`. That matches the reported bug, not the PR’s desired behavior.
  - file: [extensions/feishu/src/media.test.ts:239](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/extensions/feishu/src/media.test.ts#L239)
  - command: `nl -ba extensions/feishu/src/media.test.ts | sed -n '239,260p'`
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Shared audio helper exists but Feishu does not use it:** `src/media/mime.ts` already defines `AUDIO_FILE_EXTENSIONS` including `.aac`, `.caf`, `.flac`, `.m4a`, `.mp3`, `.oga`, `.ogg`, `.opus`, and `.wav`, plus `isAudioFileName()`. Current Feishu routing does not call this helper.
  - file: [src/media/mime.ts:54](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/media/mime.ts#L54)
