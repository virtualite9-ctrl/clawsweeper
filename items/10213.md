## Summary

Keep #10213 open. Current main at 7e13f3f5140168e31027b39a9855ade3fadf24d5 still has OpenClaw/local compaction and OpenAI Responses server-side compaction, but not Anthropic-native server-side compaction: no config/schema surface, no Anthropic request injection, no compaction block round-trip, and no `stop_reason: "compaction"` handling. The linked PR #10214 was closed unmerged, #13752 was a closed same-feature report rather than a current replacement, and #65287 is broader Anthropic compaction plus cache TTL work rather than a clear canonical superseder for this focused request.

## Best Possible Solution

Keep #10213 open as the focused tracker for Anthropic server-side compaction. A complete fix should add an explicit opt-in config or per-model/provider params, validate and document them, inject `compact-2026-01-12` and `context_management.edits` only for direct Anthropic-compatible support, preserve/replay Anthropic `compaction` blocks in streamed and stored assistant history, handle `stop_reason: "compaction"`, add logging/verbose behavior, and add focused Anthropic provider/transport tests. Treat #65287 as related broader cache/TTL design unless maintainers explicitly mark it canonical.
