## Summary

Keep #10213 open. Current main still does not implement Anthropic-native server-side compaction: there is no config/schema option for the requested opt-in, Anthropic request shaping only covers existing beta/service-tier/fast-mode behavior, and the native Anthropic transport does not add `context_management.edits` or preserve provider compaction blocks. The linked implementation PR #10214 is closed unmerged, and #65287 is related broader design work rather than a clear maintainer-designated replacement. The latest issue discussion also explicitly kept this tracker open.

## Best Possible Solution

Keep #10213 open as the focused Anthropic server-side compaction tracker. A complete maintainer-quality fix should add an explicit opt-in config or model/provider param, validate and document it, inject the Anthropic beta plus `context_management.edits` only for eligible direct Anthropic requests, preserve streamed/stored compaction blocks for replay, handle provider compaction stop/event behavior and logging, and add focused tests for both the Anthropic plugin wrapper and native transport paths. Treat #65287 as related broader design work unless maintainers explicitly mark it canonical.
