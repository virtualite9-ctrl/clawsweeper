## Summary

Keep #13634 open. Current main has generic workspace bootstrap-file injection, but the requested account/global Slack `systemPrompt` surface is still missing from Slack config types, strict schema, docs, and runtime plumbing. Existing Slack `systemPrompt` support is per-channel/wildcard only and is not applied to DMs or MPIM/group DMs.

## Best Possible Solution

Keep #13634 open. Implement an account-level `channels.slack.systemPrompt` config surface in Slack types, schema, generated config metadata, docs, and runtime. Define precedence for exact channel entries, wildcard channel entries, and account/global prompt; apply it consistently to Slack message and slash-command handling for channels, MPIM/group DMs, and DMs. Do not add Slack-specific workspace bootstrap injection; preserve the existing generic Project Context bootstrap pipeline.
