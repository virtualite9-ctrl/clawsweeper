## Summary

Keep #13634 open. Current main already has generic Project Context workspace bootstrap-file injection, but the requested account/global Slack `channels.slack.systemPrompt` surface is still missing. Slack prompt support remains scoped to nested channel entries and wildcard channel entries, and DMs/MPIMs still do not get that prompt path.

## Best Possible Solution

Keep #13634 open. Implement an account-level `channels.slack.systemPrompt` in Slack config types, strict schema, generated config metadata/UI hints, docs, and runtime context. Define precedence as exact channel entry, then wildcard channel entry, then account/global prompt, and apply it consistently for Slack message and slash-command handling across channels, private channels, DMs, and MPIM/group DMs. Preserve the existing generic Project Context bootstrap pipeline rather than adding Slack-specific workspace file injection.
