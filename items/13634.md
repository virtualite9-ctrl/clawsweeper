## Summary

Keep #13634 open. Current main already handles the workspace-file injection part through generic Project Context bootstrap files, but the remaining requested account/global Slack `channels.slack.systemPrompt` surface is still missing from Slack config types, strict schema/generated metadata, docs, and runtime prompt plumbing. Slack only applies `systemPrompt` from `channels.slack.channels.<id>` or `channels.slack.channels["*"]` for room channels, and the related PR #62792 is Slack media/file ingestion rather than prompt/config parity.

## Best Possible Solution

Keep #13634 open. Implement an account-level `channels.slack.systemPrompt` in Slack config types, strict schema, generated metadata/UI hints, docs, and runtime context. Define precedence as exact channel entry, then wildcard channel entry, then account/global prompt, and apply it consistently to Slack message and slash-command handling across channels, private channels, DMs, and MPIM/group DMs. Keep using the shared Project Context bootstrap pipeline for AGENTS.md/SOUL.md/USER.md/MEMORY.md instead of adding Slack-specific workspace-file injection.
