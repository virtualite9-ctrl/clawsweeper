## Summary

Keep this issue open. Current main supports the maintainer-suggested Matrix DM isolation path and has generic WebChat session-message refresh handling, but the issue’s observed failure is still not proven fixed: Matrix can record inbound metadata before any transcript-visible message exists, and the unresolved maintainer request for a fresh isolated-DM repro plus logs remains the right next step.

## Best Possible Solution

Keep this issue open. Maintainers should ask for, or wait for, the fresh reproduction using the suggested isolated Matrix DM config plus gateway logs and session JSONL. If it reproduces, the likely fix belongs around the Matrix monitor through reply-dispatch/transcript-event path, with a regression proving that a Matrix DM inbound event routes to the intended session and produces either a WebChat/session-list-visible transcript update or a clear logged drop/queue reason.
