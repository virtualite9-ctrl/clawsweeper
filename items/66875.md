## Summary

Current main implements the WebChat reconciliation fix well enough to close this issue: same-run final assistant payloads no longer force a redundant chat.history reload, active session.message reloads are deferred against the active run, final chat cleanup is unconditional, and stale/empty history reloads preserve optimistic local messages instead of replacing them wholesale. The related maintainer PR #72325 is merged and the Unreleased changelog explicitly records this fix, crediting the root-cause reports and follow-up PRs.

## Best Possible Solution

Close as implemented on main and keep the narrow #72325 reconciliation behavior: suppress redundant final reloads when the assistant payload is already locally rendered, defer active-run session.message reloads, and preserve optimistic local messages across any remaining legitimate history refreshes.
