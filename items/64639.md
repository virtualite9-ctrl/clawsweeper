## Summary

Keep open. Current main still hard-codes the personal `~/.agents/skills` source (`agents-skills-personal`) in skill discovery and has no config/schema/docs support for disabling that source. Existing agent skill allowlists and per-skill `enabled: false` filtering happen after discovery or at individual skill granularity, so they do not satisfy the requested source-level scan toggle. The live issue remains open with no protected labels and no same-author/open PR or exact `disableSources` PR found; related open PRs mentioning `agents-skills-personal` address symlink handling, not source disablement.

## Best Possible Solution

Keep this issue open and implement a small core skills-loader config surface, such as `skills.load.disableSources: ["agents-skills-personal"]` or an explicit `skills.scan.personal: false`. Apply the decision before each source is loaded, update config types, strict Zod schema, generated schema/help metadata, and public skills docs, then add regression tests proving the personal source is skipped while existing precedence, allowlists, prompt limits, and per-skill visibility behavior still work.
