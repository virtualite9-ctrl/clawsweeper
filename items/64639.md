## Summary

Keep open. Current main still loads and watches the personal `~/.agents/skills` source unconditionally, and the public config schema/docs do not expose a source-level disable option. Newer prompt-budget and per-agent allowlist controls can reduce visible prompt bloat, but they happen after discovery or only cap output; they do not satisfy the requested ability to skip scanning `agents-skills-personal`.

## Best Possible Solution

Keep this issue open. Implement a core skills-loader config surface that can disable `agents-skills-personal` before both loading and watcher registration, likely as a narrowly typed source list or explicit `personal` scan boolean. Update config types, strict Zod schema, generated schema/help metadata, public skills docs at https://docs.openclaw.ai/tools/skills-config, and regression tests proving the personal source is skipped while existing precedence, per-agent allowlists, prompt limits, and per-skill `enabled: false` behavior continue to work.
