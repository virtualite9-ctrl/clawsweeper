## Summary

Keep open. Current main still collapses the reported model fallback/runtime failure paths into `unknown`: the empty message/no status branch returns `unknown`, the explicit OpenAI no-details message is mapped to `unknown`, and the final catchall returns `unknown`. The public failover reason unions and cooldown policy also only know about `unknown`, so the requested `empty_response`, `no_error_details`, and `unclassified` labels are not implemented.

## Best Possible Solution

Keep the issue open and implement a narrow classifier/contract change: add `empty_response`, `no_error_details`, and `unclassified` reason support where the model fallback reason is surfaced; map the three current `unknown` paths distinctly; preserve transient cooldown behavior for these new labels; ensure redacted `errorPreview`/detail is populated for unclassified messages; and update focused tests in `src/agents/pi-embedded-helpers*.test.ts`, `src/agents/failover-policy.test.ts`, fallback notice/logging tests, and model failover docs.
