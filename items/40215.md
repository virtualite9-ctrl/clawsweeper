## Summary

Keep this issue open. Current main still does not implement cumulative context/window or model metadata in the `/usage full` footer, and it still lacks an inherited `agents.defaults.responseUsage` default. The provided GitHub context also shows open PR #69618 as an unmerged implementation candidate for this same request, so the issue should remain open while that PR is reviewed.

## Best Possible Solution

Keep this issue open until #69618, or a smaller replacement, is reviewed and either landed or closed. The implementation should add source-backed model/context/window/percent data to `/usage full`, add and document inherited `agents.defaults.responseUsage` semantics while preserving session overrides, and cover formatter, reply-runner, command, config schema, and docs behavior.
