## Summary

Keep #40215 open. Current main still formats `/usage` as a per-response token/cache/cost footer, with `full` adding only the session key. It does not include active model, cumulative context used/window/percent in the reply footer, or an inherited `agents.defaults.responseUsage` config default. The related implementation PRs in the provided context are open/unmerged, and #65487 was closed as a duplicate of this canonical issue.

## Best Possible Solution

Keep #40215 open as the canonical request. The best implementation path is to enrich the shared response usage footer with active model plus fresh cumulative context used/window/percent data, add an inherited `agents.defaults.responseUsage` config contract with schema/types/docs/tests, preserve session-level override semantics, and coordinate with #69618 or a narrower reviewed replacement before closing this issue.
