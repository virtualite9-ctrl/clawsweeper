## Summary

Keep this issue open. Current main still implements `/usage full` as a per-response token/cache/cost footer plus optional session key; it does not show cumulative context/window percentage, model metadata, or an inherited `agents.defaults.responseUsage` default. The provided GitHub context also shows #69618 as an open, unmerged implementation candidate with closing syntax for this issue, so the right maintainer action is to review that PR or a narrower replacement, not close the issue now.

## Best Possible Solution

Keep this issue open while #69618, or a smaller replacement, is reviewed. The implementation should enrich `/usage full` with model plus cumulative context/window/percentage data, add documented inherited `agents.defaults.responseUsage` semantics while preserving session overrides, and cover the formatter, reply runner, `/usage` command, config schema/docs, and relevant reply-composition tests.
