## Summary

Current `main` does not implement PR #54716. `loadCombinedSessionStoreForGateway` still treats every literal `session.store` as a single-store/default-agent fallback, so the literal `agents/<id>/sessions/sessions.json` case from #54435 is not covered on `main`. The live gateway surfaces still depend on that function, and current tests only cover the templated `{agentId}` discovery path, so this PR remains actionable rather than safe to auto-close.

## Evidence

- **Literal `session.store` still short-circuits to single-store mode on `main`:** `loadCombinedSessionStoreForGateway` returns early for any non-template `session.store`, resolves exactly one store file, and canonicalizes entries under `defaultAgentId`. It does not consult `resolveAgentsDirFromSessionStorePath` before taking that fallback.
