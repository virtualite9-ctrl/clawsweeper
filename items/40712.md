## Summary

Current main mitigates planned gateway restarts by draining active runs and replaying a one-shot post-restart continuation, but it does not implement the issue's requested durable partial-generation persistence or startup recovery from incomplete assistant messages. The report is therefore only partially addressed, not implemented well enough to close conservatively.

## Risks / Open Questions

- The user-facing restart problem for explicit gateway restarts may already be much improved on main because planned restarts drain active work and can inject a post-restart continuation.
- The still-open gap is crash-safe or forced-restart recovery of in-flight assistant output; this issue may need narrowing if maintainers only care about graceful restarts, not crash persistence.
