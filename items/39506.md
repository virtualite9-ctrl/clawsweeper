## Summary

Current `main` already carries outbound local-media roots through the message-send pipeline into the Feishu adapter, and the latest release `v2026.4.22` contains the same implementation. The reported `path-not-allowed` regression is obsolete against current code.

## Close Comment

Closing as implemented on `main` after Codex review. Current OpenClaw already routes local media allow-roots through the outbound send pipeline into Feishu: `src/infra/outbound/outbound-send-service.ts:79` resolves agent-scoped media access from the concrete media source, `src/media/local-roots.ts:117` widens the allowlist to include the parent directory of that local file when policy allows it, and Feishu forwards and uses those roots in `extensions/feishu/src/outbound.ts:264` and `extensions/feishu/src/media.ts:613`. The latest shipped release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`) already contains this behavior, and there is regression coverage in `src/infra/outbound/outbound-send-service.test.ts:492`, `extensions/feishu/src/outbound.test.ts:92`, and `extensions/feishu/src/media.test.ts:335`. The exact config key mentioned in the report is not how current code paths authorize the upload anymore, so this specific regression is no longer open on current main.
