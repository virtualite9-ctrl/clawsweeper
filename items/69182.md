## Summary

Keep #69182 open. Current main enforces a bounded timeout and fail-open result for active-memory, which likely addresses the more severe related #70481 timeout-enforcement report, but it still intentionally runs active-memory as an awaited `before_prompt_build` step before the main reply. The original complaint, chat replies can be delayed up to the configured active-memory timeout, remains true by design on current main.

## Best Possible Solution

Keep the issue open as the canonical product/performance discussion for active-memory reply-path latency. If maintainers want the proposed behavior, change `extensions/active-memory/index.ts` so recall cannot hold first visible output for the full configured timeout, or intentionally lower/autotune the default timeout and document the tradeoff; keep #70481 closed as the separate timeout-enforcement fix.
