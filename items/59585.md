## Summary

Current main already injects workspace identity/bootstrap files into every agent run, so switching channels creates separate session history but should not drop `SOUL.md`/`IDENTITY.md` persona context for the same agent/workspace.

## Evidence

- **System prompt injects identity/bootstrap every turn:** The system prompt docs explicitly list `AGENTS.md`, `SOUL.md`, `IDENTITY.md`, and `USER.md` under workspace bootstrap injection and state that these files are injected into the context window on every turn.
  - file: [docs/concepts/system-prompt.md:99](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/docs/concepts/system-prompt.md#L99)
  - command: `nl -ba docs/concepts/system-prompt.md | sed -n '99,132p'`
  - sha: [a9797214338b](https://github.com/openclaw/openclaw/commit/a9797214338ba31c52c796adbb75afb16e0684a9)
- **Workspace docs describe these files as standing session context:** The agent workspace docs say `AGENTS.md` is loaded at the start of every session, `SOUL.md` and `USER.md` are loaded every session, and `IDENTITY.md` is the agent's name/vibe file.
  - file: [docs/concepts/agent-workspace.md:62](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/docs/concepts/agent-workspace.md#L62)
  - command: `nl -ba docs/concepts/agent-workspace.md | sed -n '62,83p'`
  - sha: [a9797214338b](https://github.com/openclaw/openclaw/commit/a9797214338ba31c52c796adbb75afb16e0684a9)
- **Runtime loads bootstrap files for each run:** `resolveBootstrapContextForRun()` resolves workspace bootstrap files for the run and converts them into injected context files; this is the central path used before prompt construction.
  - file: [src/agents/bootstrap-files.ts:217](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/src/agents/bootstrap-files.ts#L217)
  - command: `nl -ba src/agents/bootstrap-files.ts | sed -n '217,275p'`
  - sha: [ec3dbd22a4de](https://github.com/openclaw/openclaw/commit/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46)
- **Normal sessions are not stripped of identity files:** `filterBootstrapFilesForSession()` only trims bootstrap files for subagent or cron session keys. Ordinary sessions keep the full bootstrap file set, so there is no channel-specific identity drop in this filter.
  - file: [src/agents/workspace.ts:544](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/src/agents/workspace.ts#L544)
  - command: `nl -ba src/agents/workspace.ts | sed -n '544,560p'`
  - sha: [ec3dbd22a4de](https://github.com/openclaw/openclaw/commit/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46)
- **Both runner paths inject the files into the prompt:** The CLI prepare path resolves bootstrap context before prompt build, the embedded runner does the same, and `buildAgentSystemPrompt()` appends those context files under `# Project Context`. There is also a test asserting `SOUL.md` and `IDENTITY.md` contents appear in the built prompt.
  - file: [src/agents/cli-runner/prepare.ts:128](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/src/agents/cli-runner/prepare.ts#L128)

## Risks / Open Questions

- I did not run a live multi-channel reproduction; this conclusion is based on current main code, docs, and release-tag inspection only.
- If the reporter was actually switching between different workspaces, profiles, or gateway instances, docs note that only one active workspace is used at a time and that extra workspace directories can cause confusing state drift.
