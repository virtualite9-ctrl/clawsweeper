## Summary

Keep PR #70877 open. The reported problem is still real on current main: generic tool-loop detection is disabled by default in the runtime resolver, `resolveToolLoopDetectionConfig` does not inject an enabled default, tests explicitly lock the default-disabled behavior, and docs/schema/type metadata still describe the default as false. The PR also contains useful reproduction context, but its final reviewed state is not cleanly merge-ready because the provided review comments identify unresolved default/schema and failing-test concerns. Related PRs overlap on the same default-enable work, but the provided context shows them as open/unshipped rather than canonical merged replacements.

## Best Possible Solution

Keep this PR open as a plausible focused fix, but require a clean revision before merge: enable loop detection at the runtime default used by both `detectToolCallLoop` and wrapped tool execution, preserve explicit `enabled: false` inheritance for global and per-agent config, update non-generated schema/help/type/docs sources plus regenerated baselines, and add regression coverage for omitted config, explicit false, and global-false plus agent-partial override cases.
