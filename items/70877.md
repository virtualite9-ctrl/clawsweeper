## Summary

Keep this PR open. Current main still leaves tool-loop detection disabled by default in the runtime path, passes absent/partial config through without synthesizing `enabled: true`, and has tests, docs, schema help, generated schema text, and exported config comments that still describe the default as false. The PR is an active, focused implementation candidate for a real core agent safety problem, while the overlapping related PRs are also open attempts rather than a merged canonical replacement.

## Best Possible Solution

Keep this PR open for maintainer review or consolidation with the overlapping loop-detection PRs. The best fix is a minimal core change that enables loop detection by default in the actual runtime/default resolver, preserves explicit and inherited disables, updates zod schema source plus generated config artifacts/docs/types/UI-visible defaults, and adds focused coverage for no-config default-on, explicit false, global false with partial agent overrides, and schema/help consistency.
