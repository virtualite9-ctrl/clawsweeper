## Summary

Keep #9835 open. Current main still has only partial Telegram reaction support: inbound reaction events, outbound `message action=react`, and a generic minimal/extensive prompt hook. The requested `reactionGuidance: "off" | "minimal" | "full"` config, strict schema/docs/generated metadata, gating by `reactionNotifications`, and incoming-reaction soft-CTA prompt guidance are still absent. The relevant files are unchanged since the provided April 26 Codex review comment that also kept this open.

## Best Possible Solution

Keep #9835 open. If maintainers want this product behavior, implement it in the bundled Telegram plugin and agent prompt composition path: add and document `reactionGuidance` with `off|minimal|full` semantics, define how it composes with `reactionNotifications` and existing `reactionLevel`, gate incoming soft-CTA prompt guidance on enabled reaction notifications, update strict schema/generated config metadata/docs, and add focused tests for off/minimal/full plus notification gating.
