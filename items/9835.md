## Summary

Keep #9835 open. Current main still implements Telegram inbound reaction events and outbound agent reaction guidance, but not the requested configurable two-way prompt guidance: there is no Telegram `reactionGuidance` config option, and the prompt does not interpret incoming user reactions as soft CTAs.

## Best Possible Solution

Keep the issue open. If maintainers want this feature, implement it in the Telegram plugin/config prompt path: add and document `reactionGuidance`, define how it composes with `reactionLevel` and `reactionNotifications`, inject minimal/full incoming reaction guidance only when enabled, update generated config metadata/docs, and add focused tests for off/minimal/full plus notification gating.
