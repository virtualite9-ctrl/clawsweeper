## Summary

Keep #9835 open. Current main still does not implement the requested configurable two-way Telegram reaction prompt guidance: Telegram already emits inbound reaction system events and has outbound reaction guidance, but prompt injection is driven by `reactionLevel`, there is no `reactionGuidance: "off" | "minimal" | "full"` Telegram config, and the injected `## Reactions` section does not interpret incoming user reactions as soft CTAs.

## Best Possible Solution

Keep #9835 open. If maintainers want this behavior, implement it in the bundled Telegram plugin and prompt composition path: add and document `reactionGuidance` with the requested off/minimal/full behavior, define how it composes with `reactionNotifications` and existing `reactionLevel`, inject incoming soft-CTA guidance only when enabled, update generated config metadata/docs, and add focused tests for off/minimal/full plus notification gating.
