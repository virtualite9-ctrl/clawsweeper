## Summary

Keep this PR open. Current main still opens the memory-core SQLite index with only `PRAGMA busy_timeout = 5000`, while the active sync and safe-reindex paths go through that helper. The PR is a plausible focused implementation candidate, not something already shipped or clearly superseded by the older overlapping WAL PRs.

## Best Possible Solution

Keep this PR open as an implementation candidate. The best path is to amend it with a `journal_mode` result check or explicit durability tradeoff comment, add focused memory-core tests for WAL/synchronous on fresh opens and safe reindex, then have maintainers choose this PR or #51986 as the canonical WAL change and close the other after that decision.
