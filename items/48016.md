## Summary

Current `main` already supports resetting one session through the gateway RPC surface, and the generic CLI RPC helper provides the requested CLI path: `openclaw gateway call sessions.reset --params '{"key":"..."}'`.

## Risks / Open Questions

- There is still no dedicated `openclaw sessions reset <key>` convenience subcommand; this close relies on the issue's requested alternative gateway RPC path, which is already present and documented.

## Close Comment

Closing from a Codex review because current `main` already supports the requested single-session reset flow through the gateway RPC surface. The method is implemented at `src/gateway/server-methods/sessions.ts:1349` as `sessions.reset`, the protocol docs list it at `docs/gateway/protocol.md:338`, and the CLI already exposes the generic RPC entrypoint at `docs/cli/gateway.md:279`, so the supported command is `openclaw gateway call sessions.reset --params '{"key":"agent:main:nextcloud-talk:group:xxx"}'`. This behavior is also covered by gateway tests in `src/gateway/server.sessions.gateway-server-sessions-a.test.ts:1369`. There is not a dedicated `openclaw sessions reset` subcommand, but the issue asked for either a CLI command or a gateway RPC method, and that RPC path is already available on current `main`.
