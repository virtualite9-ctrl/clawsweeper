## Summary

Keep open. Current main still has the single `getApiKeyAndHeaders` compaction auth preflight and no compat resolver, while the related #56099 is closed unmerged rather than a canonical shipped fix. The PR discussion also shows the initial review gaps were addressed in a follow-up PR commit, so this is still a plausible focused bug-fix artifact pending maintainer product judgment on custom/newer registry compatibility.

## Best Possible Solution

Do not auto-close. A maintainer should either rebase and land a narrow compaction auth resolver with regression tests for missing `getApiKeyAndHeaders`, provider fallback, and legacy fallback, or explicitly decide that OpenClaw only supports the pinned `@mariozechner/pi-coding-agent` `ModelRegistry` contract and close the PR with that rationale.
