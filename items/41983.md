## Summary

Current main still uses a shared control-plane write budget for clients with a known device ID or client IP, so the per-connection isolation proposed by this PR is not implemented or obsolete. The PR remains a focused gateway behavior change, but it needs maintainer review and rebase work before it could safely land.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to rebase it onto current main, preserve the bucket pruning and hard cap, rename/update the stale keying test, add or keep a regression test proving two concurrent sessions from the same device/IP have isolated write budgets, and update the gateway docs/changelog if maintainers choose per-connection control-plane write budgets as the intended security/product behavior.
