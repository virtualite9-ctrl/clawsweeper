## Summary

Keep this PR open. Current main still keys control-plane write budgets by `deviceId+clientIp` whenever either identity field is present, so concurrent gateway sessions from the same device/IP still share the same 3-per-60s write budget. The PR is a focused gateway behavior change with a regression test, but it needs rebase/rework before merge because the bot review correctly flagged a stale test name and the branch must preserve current main's bucket pruning and hard cap.

## Best Possible Solution

Keep the PR open or ask the contributor to rebase onto current main, preserve the existing stale-bucket pruning and hard cap, rename/update the stale keying test, and keep a focused regression test proving two concurrent gateway sessions from the same device/IP get independent control-plane write budgets. If maintainers intentionally want shared device/IP budgeting, close only with that explicit product rationale.
