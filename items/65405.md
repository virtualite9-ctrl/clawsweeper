## Summary

Keep this issue open. Current main still treats `agents.defaults.models` as the runtime allowlist for `models.list`; a config that only defines `models.providers.<provider>.models` is used for catalog metadata/provider definitions, but it does not filter the Control UI `/model` catalog to configured providers only.

## Best Possible Solution

Keep this issue open and make the desired semantics explicit. A likely implementation path is to update the gateway model-list behavior, probably around `src/gateway/server-methods/models.ts` and `buildAllowedModelSetWithFallbacks`, so Control UI model choices can optionally or by default derive a configured-only view from `models.providers` when no `agents.defaults.models` allowlist exists. Add regression coverage for the report's shape: `models.providers.minimax.models` and `models.providers.zhipu.models` with no `agents.defaults.models`, asserting unrelated built-in providers are excluded if that is the chosen product behavior.
