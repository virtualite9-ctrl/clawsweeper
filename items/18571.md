## Summary

Keep #18571 open. Current main has the pre-delivery `message_sending` hook and it can rewrite or cancel outbound content, but the requested deterministic-verification context is still missing: the shared outbound path does not pass `sessionKey` or `runId` into the sending hook, `OutboundSessionContext` cannot carry a run id, and the hook event has no bounded tool-call/result history or transcript handle. The existing issue comment that kept this open remains accurate on current main e29d3516bf05591e6007bfb8ccdaea6d76194d41.

## Best Possible Solution

Keep #18571 open. The best fix belongs in the core plugin hook contract and shared outbound delivery path: propagate `sessionKey` and a real `runId` into `message_sending`/`message_sent`, expose a bounded documented transcript or tool-call/result history view for deterministic pre-send verification, and add focused regression tests for correlation plus cancel/rewrite behavior across shared outbound delivery and direct message-sending paths.
