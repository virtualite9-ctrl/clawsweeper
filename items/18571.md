## Summary

Keep #18571 open. Current main has `message_sending` cancel/rewrite support, but it still does not expose the requested deterministic verification contract: the shared outbound pre-send hook is invoked without `sessionKey` or `runId`, the outbound session carrier has no `runId`, and the message hook event has no bounded tool-call/result history or transcript handle. The existing issue discussion that kept this open still matches current main.

## Best Possible Solution

Keep the issue open and implement an additive plugin hook contract: propagate `sessionKey` and a real `runId` into `message_sending` and `message_sent` across shared outbound delivery and direct reply paths, expose a bounded documented transcript or tool-call/result history view suitable for deterministic pre-send verification, and add regression tests for correlation plus cancel/rewrite behavior.
