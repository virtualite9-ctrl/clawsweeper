## Summary

Keep #18571 open. Current main has a `message_sending` hook that can rewrite or cancel outbound content, but the shared outbound delivery and direct reply call sites still do not provide the requested deterministic verification context: no propagated `sessionKey`, no `runId`, and no bounded transcript/tool-call history on the pre-send hook path.

## Best Possible Solution

Keep the issue open and implement an additive plugin hook contract: propagate `sessionKey` and a real `runId` into `message_sending` and `message_sent` across shared outbound delivery and direct reply paths, expose a bounded documented transcript or tool-call/result evidence view for deterministic verification, and add regression tests for correlation plus cancel/rewrite behavior.
