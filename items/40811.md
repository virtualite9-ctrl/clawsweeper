## Summary

Current main already fixes the reported `openclaw update --yes` LaunchAgent/plist drift by re-resolving the installed package root after update and reinstalling the gateway service from the updated canonical entrypoint, with tests covering `index.js` preference over legacy `entry.js`.

## Close Comment

Closing from a Codex review: current `main` already implements this update-path fix. The package-manager update flow now re-resolves the installed package root after the global update and carries that forward as `result.root` (`src/cli/update-cli/update-command.ts:389`), then the post-update refresh step runs `gateway install --force` from that updated install using the shared gateway entrypoint resolver (`src/cli/update-cli/update-command.ts:256`). That resolver now prefers `dist/index.js` before legacy `dist/entry.js` (`src/daemon/gateway-entrypoint.ts:4`), and service generation also normalizes a running `dist/entry.js` process back to `dist/index.js` when both exist (`src/daemon/program-args.ts:18`). The matching tests are in `src/cli/update-cli/update-command.test.ts:8` and `src/daemon/program-args.test.ts:44`. I also checked that these files are unchanged between `v2026.4.22` and current `main`, so this appears shipped in `v2026.4.22`.
