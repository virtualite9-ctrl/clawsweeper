## Summary

Keep #17424 open. Current main still does not implement the requested `agents.defaults.memorySearch.dimensions` / per-agent `memorySearch.dimensions` config key, and the active OpenAI-compatible memory embedding request builders still do not forward a configured `dimensions` value to inline or batch `/embeddings` calls. Main has adjacent `outputDimensionality` plumbing for some providers, but that does not satisfy the issue's OpenAI-compatible dimensions request.

## Best Possible Solution

Keep #17424 open. Maintainers should decide the public contract (`dimensions`, `outputDimensionality`, or an alias pair), then update schema/types/help/docs, default and per-agent merge logic, OpenAI-compatible inline and batch request builders, provider cache key data, and reindex behavior. Add focused schema, resolver, request-body, batch, cache-key, and reindex tests.

## Risks / Open Questions

- Some OpenAI-compatible providers may reject a `dimensions` field, so the implementation should be provider/model-capability aware rather than blindly adding it to every remote embedding request.
- Changing embedding dimensions must be reflected in provider cache key data and reindex identity; the current OpenAI runtime cache key only includes provider, baseUrl, model, and sanitized headers.
