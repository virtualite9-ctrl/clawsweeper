## Summary

Keep #17424 open. Current main still has only partial dimension plumbing: `outputDimensionality` exists in config and is passed into memory provider creation, but the OpenAI/OpenAI-compatible memory embedding path still does not send a `dimensions` field for inline `/embeddings` requests or OpenAI batch JSONL requests. The issue's requested `memorySearch.dimensions` config key is also not accepted by the strict schema.

## Best Possible Solution

Keep #17424 open. Decide whether to support `dimensions` as an alias or standardize docs/UI on `outputDimensionality`, then carry the resolved value into the OpenAI remote embedding client and runtime cache key, add `dimensions` to supported inline and batch `/embeddings` request bodies, and add tests for defaults, per-agent overrides, OpenAI inline requests, OpenAI batch JSONL, and reindex/cache identity.
