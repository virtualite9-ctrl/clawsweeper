## Summary

Keep #17424 open. Current main still only exposes the provider-specific `outputDimensionality` plumbing and does not accept the requested `memorySearch.dimensions` key or send OpenAI/OpenAI-compatible `dimensions` in inline or batch embedding requests. The issue discussion also has a recent maintainer/Codex comment explicitly identifying this same gap, and the current checkout still matches it.

## Best Possible Solution

Keep #17424 open. Decide whether the public config should accept `dimensions` as the requested key, make it an alias for `outputDimensionality`, or standardize documentation on one name; then thread the resolved value into OpenAI/OpenAI-compatible inline `/embeddings` bodies and OpenAI batch JSONL bodies, include it in OpenAI runtime cache/reindex identity, and add focused schema, resolver, inline request, batch request, and cache/reindex tests for defaults and per-agent overrides.
