## Summary

Current `main` already contains a targeted fix for transient subagent timeout/failover announces: lifecycle `error` events are deferred through a grace window, cancelled by later `start`/`end` events, and covered by a regression test that matches the reported duplicate-completion pattern.

## Evidence

- **Transient lifecycle errors are explicitly deferred:** Current `main` documents and implements a 15s grace window because embedded runs can emit transient lifecycle `error` events while provider/model retry is still in progress, avoiding premature failure cleanup/announce.
  - file: [src/agents/subagent-registry.ts:163](https://github.com/openclaw/openclaw/blob/51dd4f288f67ef4df4c7a121f078f508535df1fd/src/agents/subagent-registry.ts#L163)
  - command: `nl -ba src/agents/subagent-registry.ts | sed -n '160,168p'`
  - sha: [51dd4f288f67](https://github.com/openclaw/openclaw/commit/51dd4f288f67ef4df4c7a121f078f508535df1fd)
- **Retry start/end cancel the pending error path:** The lifecycle listener now routes `phase === "error"` through `schedulePendingLifecycleError(...)`, but clears that pending error on later `start` and `end` events before completion cleanup runs, which prevents the timed-out attempt from announcing when the retry succeeds.
  - file: [src/agents/subagent-registry.ts:670](https://github.com/openclaw/openclaw/blob/51dd4f288f67ef4df4c7a121f078f508535df1fd/src/agents/subagent-registry.ts#L670)
  - command: `nl -ba src/agents/subagent-registry.ts | sed -n '670,707p'`
  - sha: [51dd4f288f67](https://github.com/openclaw/openclaw/commit/51dd4f288f67ef4df4c7a121f078f508535df1fd)
- **Completion cleanup clears any pending failure timer:** `completeSubagentRun(...)` begins by clearing the pending lifecycle error for that run, so the earlier timeout-side error timer cannot later emit a second completion after the successful retry finishes.
  - file: [src/agents/subagent-registry-lifecycle.ts:628](https://github.com/openclaw/openclaw/blob/51dd4f288f67ef4df4c7a121f078f508535df1fd/src/agents/subagent-registry-lifecycle.ts#L628)

  - sha: [51dd4f288f67](https://github.com/openclaw/openclaw/commit/51dd4f288f67ef4df4c7a121f078f508535df1fd)
- **Regression test matches the reported retry-then-success sequence:** The e2e regression test emits an `error`, confirms no announce fires during the grace window, then emits `start` and `end` and verifies exactly one final `ok` announce.
  - file: [src/agents/subagent-registry.lifecycle-retry-grace.e2e.test.ts:334](https://github.com/openclaw/openclaw/blob/51dd4f288f67ef4df4c7a121f078f508535df1fd/src/agents/subagent-registry.lifecycle-retry-grace.e2e.test.ts#L334)
