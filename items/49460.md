## Summary

Keep open. Current `main` has the newer Telegram dispatcher-pool and HTTP/1.1 work, but it still does not set TCP `keepAlive` or `keepAliveInitialDelay` on the Telegram `connect` options, so the core behavior requested by PR #49460 is not implemented on `main`. The unrelated dangling-symlink script fix from the PR appears to already be covered on `main` by a different implementation.
