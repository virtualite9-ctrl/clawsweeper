## Summary

Keep open. I did not find strong evidence that current `main` fixes or obsoletes #59767. The relevant macOS voice wake matching code in the checked-out tree is materially the same as `v2026.4.1`, so this is not an `implemented_on_main` close. The issue is also still plausible because the tester UI says to "say a trigger word," but the tester’s detection path still prefers a non-empty post-trigger command, which can produce the reporter’s `No trigger heard` result for trigger-only tests.
