## Summary

Keep this issue open. Current main still has the live deadlock shape: `runReplyAgent` collects streamed tool-result delivery promises and awaits `Promise.allSettled(pendingToolTasks)` without a timeout before the active reply operation is completed. Existing transcript repair and session-manager guards cover persisted/replay hygiene, but they do not bound this active live drain.

## Best Possible Solution

Keep this issue open and resolve it by reviewing, landing, or superseding PR #64733 with an equivalent main-branch fix: add a bounded timeout or cancellation path around the `pendingToolTasks` drain, ensure `replyOperation.complete()`/failure cleanup can run, log or surface the timeout, and add focused regression coverage for a never-settling tool-result delivery task.
