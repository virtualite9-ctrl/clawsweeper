## Summary

Keep this issue open. Current `main` still has the live deadlock shape described in the issue: tool-result delivery promises are tracked in `pendingToolTasks`, and the reply runner waits on the whole set with an unbounded `Promise.allSettled()` before reaching the cleanup path that completes and clears the active reply operation. Existing transcript repair and missing-result guards cover persisted/replay cleanup, but they do not bound this active live drain. The related open PR #64733 remains an implementation candidate, not a shipped fix.

## Best Possible Solution

Keep this issue open and resolve it by reviewing, landing, or superseding PR #64733 with an equivalent main-branch fix: add a bounded timeout or cancellation path around the `pendingToolTasks` drain, log or surface the timeout, guarantee reply-operation cleanup runs so the session lane can recover, and add focused regression coverage for a never-settling tool-result delivery task.
