## Summary

Keep open. Current main has related ACP/task completion cleanup, but it still does not implement this PR's core changes: ACP turn capability-normalization guidance, task-level `spawnedBy` metadata, parent-spawned ACP silent-wake routing, or a shared single-owner completion delivery gate. The related #65242 is also open and unmerged, so it is not a shipped replacement.

## Best Possible Solution

Keep this PR open for maintainer review or author revision. The best path is to rebase against current main, preserve the already-shipped output-selection behavior, add any needed ACP normalization and parent-wake routing with durable task metadata, and resolve the untrusted completion-text and persistence review concerns. If #65242 becomes the canonical gate-based fix, consolidate the remaining work there instead of closing this PR as already implemented.
