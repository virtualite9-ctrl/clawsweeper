## Summary

Keep PR #56538 open. Current main still has the Windows CJK/mojibake failure mode in the process exec paths that this PR targets: `runExec` still requests UTF-8 strings from `execFile`, `runCommandWithTimeout` still uses default `Buffer.toString()`, and the supervisor child adapter does the same. The PR is not merge-ready as-is, but the underlying bug remains real and tracked by #50519, so this is not a conservative cleanup close.

## Best Possible Solution

Keep PR #56538 open for maintainer review or explicit consolidation with #50519 and the other Windows exec encoding PRs. The best fix is a shared Windows code-page-aware decoder used by `runExec`, `runCommandWithTimeout`, and supervisor child output, with raw-buffer capture or stateful streaming `TextDecoder`, regression tests for GBK/UTF-8 and split multibyte chunks, and no unrelated ACPX/CLAUDE.md changes.
