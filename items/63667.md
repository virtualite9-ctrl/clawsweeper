## Summary

Current main already fixes the reported MiniMax image-tool failure path: MiniMax chat models are treated as text-only, image understanding routes through the plugin-owned MiniMax-VL-01 media provider, and local image files are loaded as bytes/mime before being sent as base64 data-image payloads. The implementation is present in the shipped v2026.4.24 release.

## Best Possible Solution

Close this as implemented and keep the shipped MiniMax media-understanding path: local Telegram-saved image files should flow through the image tool as buffers, while MiniMax image analysis should use minimax/MiniMax-VL-01 or minimax-portal/MiniMax-VL-01 rather than the text-only M2.7 chat models. Users who explicitly set imageModel to MiniMax-M2.7-highspeed should remove that override or switch it to MiniMax-VL-01.
