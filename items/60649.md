## Summary

Keep open. Current `main` does not contain this PR’s `currentWindowTokens` change, and the existing gateway/UI path still has the stale-token behavior the PR is trying to address. At the same time, the PR discussion surfaced a real correctness problem in the submitted patch, so there is not strong evidence for a conservative close.
