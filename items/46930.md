## Summary

Keep open. Current main still routes `openclaw gateway status --deep` extra-service findings through the daemon status printer, and that printer emits the exact `Other gateway-like services detected (best effort):` advisory plus cleanup/recommendation lines via `defaultRuntime.error`, which maps to `console.error`. The docs describe this as a warning/advisory path and acknowledge intentional multi-gateway setups, so the requested downgrade to warn/info remains a valid product/CLI logging cleanup.

## Best Possible Solution

Keep #46930 open and implement a narrow CLI output change in `src/cli/daemon-cli/status.print.ts`: emit the extra gateway-like service heading, service list, cleanup hints, and recommendation/isolation text through a warning/info-style channel instead of `defaultRuntime.error`, while preserving existing error severity for true failure states. Add a focused `printDaemonStatus` test with non-empty `extraServices` that asserts the advisory is not sent to `runtime.error` and update docs only if the visible wording changes.

## Risks / Open Questions

- Changing this should preserve real error output for unhealthy service, failed probe, and port-listener mismatch paths; only the extra gateway-like advisory/recommendation block should move to warn/info/log semantics.
- A regression test should cover `extraServices.length > 0` so future changes do not move this advisory back onto `runtime.error`.
