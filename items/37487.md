## Summary

Current main still does not implement first-class per-profile browser proxy configuration or per-agent browser profile binding. OpenClaw has multi-profile browser support and a global `browser.extraArgs` proxy workaround, and the `m13v/browser-lock` comment plus closed PR #42806 are useful prior art, but neither is a shipped OpenClaw browser/agent config contract.

## Best Possible Solution

Keep #37487 open. The best path is a bundled browser-plugin/core-agent implementation: add optional secret-aware `browser.profiles.<name>.proxy`, apply it only to managed local profile launches, surface redacted `proxyConfigured`/diagnostics, add `agents.defaults.browser.profile` and `agents.list[].browser.profile` with explicit sub-agent inheritance semantics, and document/test concurrent isolated multi-profile use. Treat PR #42806 and `m13v/browser-lock` as prior art, not as a shipped fix.

## Risks / Open Questions

- The existing global `browser.extraArgs` proxy workaround can be mistaken for the requested feature, but it cannot provide per-profile or per-agent concurrent egress routing.
- The requested feature crosses bundled browser-plugin config, core agent config, secret handling, diagnostics, and sub-agent inheritance semantics, so it needs maintainer product/API design rather than stale cleanup.
- External prior art exists (`m13v/browser-lock`), but adopting it still requires OpenClaw-owned tests/docs and compatibility with current browser profile lifecycle.
