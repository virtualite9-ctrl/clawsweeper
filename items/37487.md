## Summary

Current main still lacks the requested first-class per-profile browser proxy config and per-agent browser profile binding. OpenClaw supports isolated named browser profiles and a global `browser.extraArgs` proxy workaround, and the linked `m13v/browser-lock` project plus closed PR #42806 are useful prior art, but neither is a shipped OpenClaw config/API contract for concurrent per-agent proxy routing.

## Best Possible Solution

Keep #37487 open. The best path is a bundled browser-plugin/core-agent implementation: add optional secret-aware `browser.profiles.<name>.proxy`, apply it only to managed local profile launches, surface redacted `proxyConfigured` and failure diagnostics, add `agents.defaults.browser.profile` and `agents.list[].browser.profile` with explicit sub-agent inheritance semantics, and document/test concurrent isolated multi-profile use. Treat PR #42806 and `m13v/browser-lock` as prior art, not as shipped fixes.

## Risks / Open Questions

- The existing global `browser.extraArgs` proxy workaround can be mistaken for this request, but it cannot provide per-profile or per-agent concurrent egress routing.
- The request crosses bundled browser plugin config, core agent config, secret handling, diagnostics, and sub-agent inheritance semantics, so it needs maintainer product/API design rather than stale cleanup.
- External prior art exists, but adopting it still requires OpenClaw-owned tests, docs, and compatibility with the current browser profile lifecycle.
