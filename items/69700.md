## Summary

Current main still treats every non-index Markdown file under `sources/` as a wiki source page and applies the required `id`, `pageType`, and `updatedAt` lint path. The docs still contain the cited “imported raw material” wording, while the importers and CLI docs imply a managed source-page wrapper with provenance frontmatter. Related #64696 is useful background for bridge/lint durability, but it is broader and open, not a canonical shipped fix for this narrower contract question.

## Best Possible Solution

Keep the issue open for maintainer product/docs judgment. The likely path is to make the memory-wiki contract explicit: either document that `sources/` contains managed source pages that wrap raw imports and therefore require frontmatter, or add a deliberate raw/preserve mode that lint can detect without weakening checks for bridge/imported source pages. Add focused regression coverage for manual raw `sources/*.md` behavior once the contract is chosen.
