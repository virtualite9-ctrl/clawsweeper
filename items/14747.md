## Summary

Current main still does not provide a user-configurable lane wait diagnostic threshold. The queue default remains hardcoded, the cron enqueue path still uses a fixed warning threshold, the strict diagnostics config schema/types/help metadata expose no lane-wait field, and the related implementation PR #16366 was closed unmerged in the provided GitHub context.

## Best Possible Solution

Keep #14747 open until a focused patch adds a documented configurable lane wait diagnostic threshold, likely `diagnostics.laneWaitWarnMs`, across exported config types, strict zod schema, labels/help/generated metadata, public docs, and regression tests. Wire `enqueueCommandInLane` so explicit per-call `warnAfterMs` still wins while the default comes from config, and make a maintainer decision on whether cron manual enqueue inherits that default or gets a documented cron-specific threshold.
