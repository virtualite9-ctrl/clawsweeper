## Summary

Keep #69678 open. Current main still hard-codes cron summary extraction to 2000 UTF-16 chars, writes only that `summary` into the per-job JSONL run log, and exposes only `cron.runLog.maxBytes` / `keepLines` in config, schema, docs, and tests. The related provided reports are adjacent cron-output/history work, but none is a canonical fix for this specific per-entry summary truncation.

## Best Possible Solution

Implement a small core cron/config fix: add `cron.runLog.maxSummaryChars` with a bounded default, validate and document it in the config schema/help/generated metadata, plumb it into isolated cron summary selection or run-log append, keep existing `maxBytes` / `keepLines` pruning as the disk bound, and add targeted tests for >2000-character summaries and config parsing.
