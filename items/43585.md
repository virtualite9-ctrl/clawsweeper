## Summary

Keep this PR open. Current main still lacks deterministic `$include` path validation for NUL bytes and raw/resolved path length, and the PR is a focused security-hardening candidate with unresolved review feedback rather than obsolete or implemented work.

## Best Possible Solution

Keep this PR open as the implementation candidate. Revise it to reject NUL bytes before path resolution, enforce raw and normalized/resolved include-path limits with `Buffer.byteLength(..., "utf8")`, make the exclusive fewer-than-4096-byte boundary explicit in errors and docs, add regression tests for NUL, raw limit, resolved limit, boundary acceptance, and multibyte byte-limit cases, then run the targeted config include test and changed gate before maintainer merge.
