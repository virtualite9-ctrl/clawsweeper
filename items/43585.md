## Summary

Keep open. Current main still does not implement the proposed config `$include` hardening: include paths are resolved and normalized before any NUL-byte or path-length validation, the tests only assert that a NUL path throws somehow, and the docs still only describe root-boundary, missing-file, parse, and circular-include errors. The PR is a focused security-hardening candidate, but it still needs review fixes before merge, especially the byte-length issue raised on the PR.

## Best Possible Solution

Keep this PR open as the implementation candidate. Revise it to enforce the raw and resolved include-path limit in bytes, make the exclusive 4096 boundary unambiguous in error messages/docs, add regression tests for NUL, raw limit, resolved limit, and multibyte byte-limit cases, then run the targeted config include test and changed gate before maintainer merge.
