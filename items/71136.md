## Summary

Keep this issue open. Current main has partial context-token safety work, but it does not implement the requested shared minimum-of-agent-config, model config, discovery, and defaults rule. Several paths still let configured agent/default values win before model metadata, per-agent `contextTokens` is accepted by config validation but not propagated through `ResolvedAgentConfig`, and `maxTokens` clamping remains provider-local and inconsistent.

## Best Possible Solution

Implement a shared effective token-limit resolver that computes the minimum of available agent default/per-agent config, provider model config (`contextTokens`/`contextWindow` and `maxTokens`), discovered runtime model metadata, and defaults. Wire it through run setup, auto-reply, session persistence/status, and transport stream params, then add focused tests for high `agents.defaults.contextTokens`, high `agents.list[].contextTokens`, and requested `maxTokens` above the selected model limit.
