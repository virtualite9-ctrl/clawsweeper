## Summary

Keep #71136 open. Current main has a partial runtime cap for `agents.defaults.contextTokens`, but it still uses precedence-based resolution in several relevant paths and does not apply the requested minimum-of-all-available-values rule to `maxTokens`.

## Best Possible Solution

Keep the issue open and implement a shared token-limit resolver that computes the conservative positive minimum across per-agent/default agent config, provider model config (`contextWindow`/`contextTokens`/`maxTokens`), runtime/discovered model metadata, and hard defaults. Apply it consistently to runtime context budgeting, session defaults/persistence, and outbound `maxTokens` stream options, with focused tests for oversized agent defaults, oversized provider model rows versus discovered metadata, per-agent overrides, and provider transports that currently forward requested `maxTokens` directly.

## Evidence

That handles one case but is not a general min across agent config, model config, discovery, and defaults, and it does not include per-agent `agents.list[].contextTokens`.
