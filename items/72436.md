## Summary

Keep this PR open. Current main has adjacent session persistence, compaction, transcripts, and outbound delivery queue recovery, but it still does not implement the central proposed behavior: an append-only `recovery-events.jsonl` sidecar next to the session store for sanitized inbound, routing, `/compact`, and outbound delivery breadcrumbs. The PR is an active draft implementation candidate, not cleanup-close material.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to evaluate or narrow the proposed core implementation as a best-effort append-only session recovery event log in `src/config/sessions`, with call sites for inbound metadata, route resolution, manual compaction, and outbound delivery, plus focused tests for redaction, file permissions, non-blocking append failures, and no message-body logging. Coordinate with #72209 if maintainers want one recovery/checkpoint ledger, but this PR should not be closed as already implemented.
