## Summary

Keep this PR open. Current main has adjacent session persistence, runtime trajectory capture, compaction checkpoints, restart/orphan recovery, outbound delivery diagnostics, and a write-ahead outbound delivery queue, but it still does not implement the central requested behavior: an append-only `recovery-events.jsonl` sidecar next to the session store with sanitized lifecycle breadcrumbs for inbound metadata, route resolution, manual `/compact`, and outbound delivery success/failure.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to evaluate or narrow the proposed core implementation as a best-effort, append-only, redacted session recovery event log in `src/config/sessions`, with call sites for inbound metadata, route resolution, manual compaction, and outbound delivery, plus focused tests for redaction, permissions, non-blocking failures, and no message-body logging. If maintainers prefer a unified recovery ledger with the adjacent passive-checkpoint work, this PR should be redirected or split rather than closed as already implemented.
