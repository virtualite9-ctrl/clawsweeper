## Summary

Keep #19929 open. Current main still has only DM identity/session sharing controls; non-direct channel/group peers are keyed by channel + peer id, threads remain separate, and there is no strict config field for selective channel/thread session groups. The related PR #20078 was closed unmerged, and the related open #7524 is narrower group-to-main work rather than a full replacement for selective cross-channel/thread groups.

## Best Possible Solution

Keep #19929 open as the concrete tracker for opt-in shared sessions across selected group/channel/thread chats. A likely implementation belongs in core routing/config: add a backward-compatible session grouping config, route it through buildAgentSessionKey/resolveAgentRoute and outbound/thread-aware paths, then update schema/help/docs and add regression tests for grouped routing, ungrouped compatibility, queue serialization, reply delivery, and thread inheritance.
