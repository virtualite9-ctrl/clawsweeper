## Summary

Keep #19929 open. Current main still has no selective shared-session routing for chosen group/channel/thread chats: non-direct peers are keyed by channel and peer id, the route path only threads DM controls through session-key construction, and strict config schemas expose no channelGroups/groupScope/includeThreads-style option. The related PR #20078 is closed and unmerged, while #7524 is an open but narrower group-to-main request, not a replacement for selective cross-channel/thread grouping.

## Best Possible Solution

Keep #19929 open as the concrete tracker for selective shared sessions across chosen group/channel/thread chats. A good implementation belongs in core routing/config: add a backward-compatible grouping config, thread it through buildAgentSessionKey/resolveAgentRoute and channel thread/outbound paths, update strict schemas/types/help/docs/generated metadata, and add regression tests for grouped routing, ungrouped compatibility, reply delivery, queue serialization, thread inheritance, and sandbox/trust-boundary behavior. Reuse useful design from closed PR #20078, but re-review it before merging.
