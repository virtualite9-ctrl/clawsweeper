## Summary

Keep #19929 open. Current main still isolates non-direct group/channel/thread sessions by channel, peer kind, and peer id, and the config/schema/docs expose DM controls only (`dmScope`, `identityLinks`) rather than a selective `channelGroups` or equivalent shared-session map. The closed PR #20078 is useful prior work, but it was not merged, and the maintainer comment already recorded on the issue explicitly kept this open.

## Best Possible Solution

Keep this issue open as the concrete tracker for opt-in shared sessions across selected channel/group/thread chats. The likely core path is to add a documented, backward-compatible session grouping config, propagate it through `resolveAgentRoute()`/`buildAgentSessionKey()`, decide thread inheritance semantics explicitly, update config schema/types/help/docs, and add routing/queue regression tests. PR #20078 can be treated as prior art, but the implementation should be reviewed against current routing and security boundaries.
