## Summary

Keep open. Current main appears to have fixed the token-drift loop and the `gateway.auth.token`-specific plaintext projection, but the broader macOS LaunchAgent install-path concern remains: `gateway install` still collects state-dir `.env` values and auth-profile env refs into the service environment, and the LaunchAgent plist renderer serializes those environment entries as plaintext `EnvironmentVariables`.

## Best Possible Solution

Keep this issue open and finish the macOS install-path fix. The likely maintainer path is to make the LaunchAgent plist contain only non-secret service metadata, then load `$OPENCLAW_STATE_DIR/.env` and resolve supported SecretRefs at gateway startup through an OpenClaw-owned runtime entrypoint or equivalent launchd-safe mechanism. The fix should cover `gateway install --force`, update/reinstall/doctor repair paths, stale plist cleanup, env SecretRefs, file SecretRefs, structured SecretRef objects, and regression tests proving provider/channel secrets and `gateway.auth.token` are not written as plaintext plist `EnvironmentVariables`.
