## Summary

Keep open. Current main fixes part of the gateway token SecretRef drift/persistence path, but the central macOS LaunchAgent install-path concern remains: durable .env/config/auth-profile values are still collected into the gateway install environment, and launchd still renders that environment inline into plist `EnvironmentVariables`.

## Best Possible Solution

Keep this issue open and finish the macOS install-path fix: stop serializing secret-bearing durable `.env`, config, auth-profile, provider, and channel values into LaunchAgent plist `EnvironmentVariables`; load the state-dir `.env` or supported SecretRefs at gateway startup through a launchd-safe runtime entrypoint or equivalent; scrub stale plist entries; and add regression coverage for `install --force`, update/reinstall, ENV/FILE/structured SecretRefs, auth profiles, provider/channel secrets, and `gateway.auth.token`.
