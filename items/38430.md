## Summary

Keep #38430 open. Current `main` still has no user-visible policy, formatter, or config switch that expands `~/...` in final agent replies to full absolute paths. The code continues to intentionally compact skill prompt paths to `~/...`, tests assert that prompt behavior, and host edit recovery can still echo the original tilde path in tool output. The related PR #38495 proposed `OPENCLAW_EXPAND_PATHS`, but the provided GitHub context shows it is closed and unmerged, so it does not supersede the issue.

## Best Possible Solution

Keep the issue open. The best maintainer path is a product-level fix that distinguishes internal prompt/tool path handling from final user-visible replies, such as explicit final-reply prompt guidance, a delivery-time formatter, or a documented config switch, with regression coverage for final replies while preserving intentional skill-prompt compaction and file-tool tilde input expansion.
