## Summary

Keep #38430 open. Current main still has no final-reply policy, formatter, or config switch that expands `~/...` in user-visible assistant replies. The code does expand tilde paths for host file-tool inputs, but it also intentionally compacts some prompt paths to `~/...` and preserves `~/...` in at least one tool-visible success message, so the reported final-answer usability issue is not implemented on main.

## Best Possible Solution

Keep the issue open. Maintainers should decide the product boundary between internal prompt/tool path handling and user-visible final replies, then implement either explicit final-reply prompt guidance, a delivery-time formatter, or a documented config switch with regression coverage for final replies while preserving intentional skill-prompt compaction and file-tool tilde input expansion.
