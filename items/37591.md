## Summary

Keep #37591 open. Current main still has the reported macOS companion-node contract mismatch: the agent `exec host=node` path invokes `system.run.prepare` before `system.run`, the gateway rejects node commands that were not advertised by the node, and the macOS Swift companion node still advertises and handles `system.run` without defining, advertising, or dispatching `system.run.prepare`. The issue discussion’s later repro and linked PR #43330 remain relevant; #43330 is closed/unmerged and #69725/#38781 are still implementation directions rather than shipped fixes on current main.

## Best Possible Solution

Keep #37591 open as the canonical tracker until maintainers ship a reviewed fix. A direct macOS fix should add shared Swift prepare command/types, advertise `system.run.prepare` from `MacNodeModeCoordinator`, dispatch it in `MacNodeRuntime`, return the canonical `systemRunPlan` shape expected by `executeNodeHostCommand`, and add focused Swift plus agent/gateway regression coverage. A compatibility fallback like #69725 may also be useful for already-shipped macOS nodes that will continue to omit prepare.
