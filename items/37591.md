## Summary

Keep #37591 open. Current main still requires the `system.run.prepare` handshake for `exec host=node`, while the macOS Swift companion node still advertises and handles `system.run` without defining, advertising, or dispatching `system.run.prepare`. The linked PR #43330 was relevant but closed/unmerged; #38781/#69725 remain unshipped implementation directions.

## Best Possible Solution

Keep #37591 open as the canonical tracker until maintainers ship a reviewed fix. The best implementation is to add shared Swift `system.run.prepare` command/types, advertise it from `MacNodeModeCoordinator`, dispatch it in `MacNodeRuntime`, return the gateway-compatible canonical plan shape, and add focused Swift plus agent/gateway regression coverage. A compatibility fallback for already-shipped macOS nodes that only advertise `system.run` may also be useful.
