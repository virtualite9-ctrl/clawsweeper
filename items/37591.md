## Summary

Keep #37591 open. Current main still unconditionally calls `system.run.prepare` for `exec host=node`, while the Swift macOS companion node still advertises and dispatches `system.run` without defining or handling `system.run.prepare`. The related #43330 fix was closed unmerged, and #38781/#69725 remain unshipped implementation directions rather than canonical replacements.

## Best Possible Solution

Keep #37591 open as the canonical tracker until maintainers ship a reviewed fix. The likely product path is to add shared Swift `system.run.prepare` command/types, advertise it from `MacNodeModeCoordinator`, dispatch it in `MacNodeRuntime`, return the gateway-compatible canonical plan shape, and cover it with focused Swift plus agent/gateway regression tests. A separately reviewed compatibility fallback for already-shipped macOS nodes that only advertise `system.run` may also be useful.
