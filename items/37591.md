## Summary

Keep open. Current `main` still hard-requires `system.run.prepare` for `exec host=node`, but the macOS companion-node code on `main` and the latest release `v2026.4.22` still do not define, advertise, or dispatch that command.

## Evidence

- **Gateway exec path requires prepare first:** `src/agents/bash-tools.exec-host-node.ts:100` calls `node.invoke` with `command: "system.run.prepare"` before any `system.run`, and `src/agents/bash-tools.exec-host-node.ts:117` throws if the prepare payload is missing/invalid. There is no fallback path for nodes that only expose `system.run`.
  - file: [src/agents/bash-tools.exec-host-node.ts:100](https://github.com/openclaw/openclaw/blob/55389f4be29359e2dd57ebaf548ba4a70331a445/src/agents/bash-tools.exec-host-node.ts#L100)

  - sha: [55389f4be293](https://github.com/openclaw/openclaw/commit/55389f4be29359e2dd57ebaf548ba4a70331a445)

  - file: [apps/shared/OpenClawKit/Sources/OpenClawKit/SystemCommands.swift:3](https://github.com/openclaw/openclaw/blob/55389f4be29359e2dd57ebaf548ba4a70331a445/apps/shared/OpenClawKit/Sources/OpenClawKit/SystemCommands.swift#L3)
