## Summary

Keep #37591 open. Current main still has the macOS companion-node contract mismatch: the agent `exec host=node` path always calls `system.run.prepare`, the gateway rejects undeclared node commands, and the Swift macOS node still advertises and dispatches `system.run` without defining or handling `system.run.prepare`. The related fix PRs in the provided context are open or closed-unmerged, not shipped replacements.

## Best Possible Solution

Keep #37591 open as the canonical bug tracker until maintainers ship a reviewed fix. The likely product path is to either add a gateway-compatible `system.run.prepare` implementation to the Swift macOS companion node, including advertised command support and regression coverage, or land a carefully reviewed compatibility fallback that preserves the approval-plan security contract for already-shipped nodes that only advertise `system.run`.
