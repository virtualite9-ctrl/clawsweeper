## Summary

Keep open. Current `main` does not provide strong evidence that #64004 is fixed: the Sessions tab itself only refreshes `sessions.list`, but the Control UI still does additional control-plane work on connect, Overview still fans out into several slow-capable RPCs including `sessions.usage` and `usage.cost`, and I did not find a shipped changelog/history entry that clearly resolves the reported latency pattern after the reporter's April 20, 2026 follow-up logs.

## Evidence

- **Sessions tab on current main is narrow, but only after active-tab refresh:** `refreshActiveTab()` routes the `sessions` tab to `loadSessions(app)` only; there is no obvious UI-side wait on `sessions.usage` or `usage.cost` inside the Sessions tab path itself.
  - file: [ui/src/ui/app-settings.ts:357](https://github.com/openclaw/openclaw/blob/5b8bd6371c666a5de43c82305e0315771e8e49d5/ui/src/ui/app-settings.ts#L357)
  - command: `nl -ba ui/src/ui/app-settings.ts | sed -n '333,365p'`
  - sha: [5b8bd6371c66](https://github.com/openclaw/openclaw/commit/5b8bd6371c666a5de43c82305e0315771e8e49d5)
- **Gateway connect still triggers extra control RPCs before/alongside active-tab refresh:** On every connect, current main still calls `subscribeSessions`, `loadAssistantIdentity`, `loadAgents`, `loadHealthState`, `loadNodes`, `loadDevices`, and then `refreshActiveTab`. That means opening/reconnecting the Control UI can still create multi-request pressure beyond `sessions.list` alone.
  - file: [ui/src/ui/app-gateway.ts:330](https://github.com/openclaw/openclaw/blob/5b8bd6371c666a5de43c82305e0315771e8e49d5/ui/src/ui/app-gateway.ts#L330)
