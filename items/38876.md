## Summary

Keep #38876 open. Current main still has string-only extraction for embedded `toolresult` / `tool_result` blocks, so array-shaped `text` or `content` still falls through to an empty tool-card output. The supplied PR/issue discussion supports this PR as the current focused fix path, not an obsolete or superseded one.

## Best Possible Solution

Keep #38876 open as the canonical fix path, or land an equivalent maintainer patch in `ui/src/ui/chat/tool-cards.ts` that extracts and joins text from array-shaped `text` and `content` fields for embedded `toolresult` / `tool_result` blocks. Add focused regression coverage in `ui/src/ui/chat/tool-cards.node.test.ts` for array `content`, array `text`, typed text blocks, ignored non-string fragments, and multi-fragment joining.
