## Summary

Close as `clawhub`. Current `main` already provides generic MCP client/runtime support plus plugin and bundle paths for MCP servers, and the project vision explicitly keeps optional third-party MCP integrations out of core when plugin/ClawHub paths already cover them. The issue itself points to an existing external `invinoveritas` implementation, which fits that distribution model better than a core OpenClaw integration.

## Evidence

- **Vision: optional capability should ship as plugins, including bundle-style MCP servers:** The project vision says core should stay lean, optional capability should usually ship as plugins, and bundle-style plugins are specifically for stable external surfaces such as skills and MCP servers; plugin promotion belongs in ClawHub.

  - file: [VISION.md:84](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/VISION.md#L84)

  - file: [docs/plugins/bundles.md:98](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/docs/plugins/bundles.md#L98)
