## Summary

Keep open. Current main still has the same unguarded Control UI render path for cron jobs, the UI loader trusts `cron.list` rows, and the linked focused fixes #54550 and #54552 were closed without merging. I did not find a shipped equivalent regression test or source change that makes malformed cron payloads safe in the UI.

## Best Possible Solution

Keep this issue open. The best maintainer path is to revive or reimplement the focused #54552 approach on current main: guard Control UI cron payload dereferences, preferably validate/filter malformed `cron.list` rows at the UI boundary as defense in depth, and add a jsdom regression test that proves malformed cron entries do not prevent switching back to chat. After that lands, close this issue as implemented.
