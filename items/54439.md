## Summary

Keep this issue open. Current main still has the reported Control UI crash path: sidebar navigation to Cron loads cron rows into UI state without runtime payload validation, and the shared render path still reads `job.payload.kind` before checking that `payload` exists. The two focused fixes linked in the discussion, #54550 and #54552, are closed and unmerged.

## Best Possible Solution

Keep this issue open. The best maintainer path is to revive or reimplement the focused #54552 approach on current main: validate or filter malformed `cron.list` rows at the UI and/or server boundary, guard shared Cron UI helpers before reading `payload.kind`, and add a jsdom/browser regression test proving malformed Cron entries do not prevent switching back to Chat.
