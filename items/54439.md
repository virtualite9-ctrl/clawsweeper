## Summary

Current main still leaves the reported Control UI crash path plausible. `cron.list` rows are accepted into UI state without payload-shape validation, `renderApp` still computes cron model suggestions before tab-specific rendering and dereferences `job.payload.kind`, and the focused PRs linked from the discussion were closed unmerged.

## Best Possible Solution

Keep this issue open. The best maintainer path is to revive or reimplement the focused #54552 approach on current main: validate or filter malformed `cron.list` rows at the UI boundary, guard shared Control UI cron render helpers before reading `payload.kind`, and add a jsdom/browser regression test proving malformed cron entries do not prevent switching back to Chat.
