## Summary

Current main already fixes this: the Control UI model picker stores provider-qualified option values from the catalog and sends the selected value directly to `sessions.patch`, with regression tests covering stale-provider and picker-patch cases.

## Evidence

- **Picker option values are provider-qualified:** `buildQualifiedChatModelValue` returns `provider/model` values and avoids re-prefixing already-qualified refs, which is the core fix for the reported bad `openai/claude-*` construction.
  - file: [ui/src/ui/chat-model-ref.ts:5](https://github.com/openclaw/openclaw/blob/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4/ui/src/ui/chat-model-ref.ts#L5)

- **Control UI builds options from qualified catalog values:** `resolveChatModelSelectState` builds picker options via `buildChatModelOptionFromLookup`, so the dropdown value comes from the catalog entry's fully qualified provider/model ref rather than being reconstructed from current UI state.
  - file: [ui/src/ui/chat-model-select-state.ts:71](https://github.com/openclaw/openclaw/blob/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4/ui/src/ui/chat-model-select-state.ts#L71)

  - file: [ui/src/ui/chat/session-controls.ts:100](https://github.com/openclaw/openclaw/blob/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4/ui/src/ui/chat/session-controls.ts#L100)
