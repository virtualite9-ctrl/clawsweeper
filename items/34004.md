## Summary

Keep #34004 open. Current main still exposes only the narrower Discord `set-presence` action behind `channels.discord.actions.presence`; it does not implement the requested `self-profile`/`selfProfile` gate, nickname/avatar mutation, self-only target guards, multi-account self-profile routing requirement, or `bestEffort` partial result handling. PR #34010 is closed unmerged, and #33270 was closed as superseded by this narrower issue rather than serving as a remaining canonical tracker.

## Best Possible Solution

Keep #34004 open. If maintainers want this in the bundled Discord plugin, implement a scoped Discord self-profile action behind `channels.discord.actions.selfProfile` with self-only target checks, explicit `accountId` handling for multi-account ambiguity, strict avatar validation, gateway preflight for presence fields, focused tests, and docs. If maintainers prefer ClawHub, document the intended plugin path for safely targeting the relevant Discord bot account/profile first.
