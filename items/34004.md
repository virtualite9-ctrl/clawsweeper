## Summary

Keep #34004 open. Current main still has only Discord `set-presence` behind the existing `channels.discord.actions.presence` gate; it does not implement the requested Discord `self-profile`/`selfProfile` action, nickname/avatar mutation, strict self-only target checks, multi-account self-profile routing, or `bestEffort` partial-result behavior. The related implementation PR #34010 is closed unmerged, and #33270 is closed rather than an open canonical tracker.

## Best Possible Solution

Keep #34004 open. If maintainers want this in the bundled Discord plugin, implement a scoped Discord self-profile action behind `channels.discord.actions.selfProfile` with nickname/avatar/status support, strict bot-self target checks, explicit `accountId` behavior for multi-account ambiguity, avatar MIME/base64/size validation, gateway preflight for presence fields, tests, and Discord docs. If maintainers prefer ClawHub, make that product decision explicit and document either a plugin path that can safely own the Discord bot account or the plugin API seam needed to extend bundled Discord actions.
