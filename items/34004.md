## Summary

Keep #34004 open. Current main (67b9167b808f2ff065c91665d6f36256a5c23ef2) still only implements the narrower Discord `set-presence` action behind `channels.discord.actions.presence`; it does not add the requested `self-profile`/`selfProfile` gate, Discord nickname/avatar mutation, self-only target guards, multi-account self-profile routing requirement, or `bestEffort` partial result handling. The direct PR #34010 is closed unmerged in the provided context, and the broader #33270 is closed rather than a canonical open tracker for this remaining work.

## Best Possible Solution

Keep #34004 open. If maintainers want this in the bundled Discord plugin, implement a scoped Discord self-profile action behind `channels.discord.actions.selfProfile` with self-only target checks, explicit `accountId` handling for multi-account ambiguity, strict avatar validation, gateway preflight for presence fields, focused tests, and docs. If maintainers prefer ClawHub, first document the intended plugin path for safely targeting the relevant Discord bot account/profile instead of silently closing the bundled-plugin feature request.
