## Summary

PR #69421 is superseded by the current main update contract. Main now explicitly keeps package-mode `openclaw update` as a refresh flow even when current and target versions match, because that refresh also keeps packaged sidecars, plugin records, completion, doctor, and restart work aligned. Applying this PR now would regress current source, tests, and docs rather than implement the maintained behavior.

## Best Possible Solution

Close this PR as superseded by the current main update contract. Keep the shipped refresh-on-same-version behavior unless maintainers decide on a new, narrower design, such as a separate lightweight `--check`/`--dry-run` path or a sidecar-aware skip that still preserves plugin/channel/config follow-up work.
