## Summary

Keep this issue open. The provided discussion includes a maintainer update explicitly keeping it in the active runtime-investigation lane, and current main only documents the QMD limitation/workaround; it has not added OpenClaw-side symlink pruning, visited-realpath cycle detection, or QMD exclusion controls for indexed roots.

## Best Possible Solution

Keep this issue open and drive it as a focused QMD traversal robustness fix. The next useful maintainer path is to add a controlled regression around a workspace-visible temp monorepo symlink loop, then either make QMD traversal cycle-safe/no-follow through upstream or expose/apply an OpenClaw-side QMD exclusion/pruning policy before qmd update can recurse. The existing docs workaround should remain, but it is not a product fix.
