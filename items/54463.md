## Summary

Keep this issue open. Current main still has the QMD traversal robustness gap described here: docs acknowledge workspace-visible temp repos with recursive package-manager symlinks can break QMD indexing, config exposes no QMD exclusion/no-follow/cycle policy, and the runtime still delegates configured collections to `qmd update` without OpenClaw-side symlink pruning or realpath-cycle handling. The issue also has explicit maintainer comments keeping the runtime problem open after the docs correction, plus a current-behavior repro from the reporter.

## Best Possible Solution

Keep this issue open as a focused QMD traversal robustness item. Add the controlled symlink-loop repro as regression coverage, then make traversal cycle-safe/no-follow upstream or add an OpenClaw-side pre-update pruning/exclusion policy before `qmd update` can recurse through workspace-visible temp monorepos. Keep the current docs workaround until a shipped runtime fix exists.

- Users who enable QMD with broad indexed workspace roots can still break indexing when visible temporary monorepos include recursive package-manager symlink structures.
