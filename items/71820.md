## Summary

Keep PR #71820 open. Current main still lacks the requested opt-in BlueBubbles HTTP API fallback for inbound reply-context cache misses, and the PR is a focused draft implementation with useful tests. It is not obsolete, duplicate, or already implemented; it should stay open for maintainer review and fixups.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to revise the fallback so it reuses or extends the existing guarded `BlueBubblesClient` request path for `GET /api/v1/message/{guid}`, keep the behavior opt-in unless maintainers choose otherwise, preserve channel-level config inheritance for multi-account deployments, and retain focused tests for cache misses, failures, dedupe, config inheritance, and SSRF policy.
