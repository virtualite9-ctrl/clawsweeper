## Summary

Current main still has the temp cleanup race that PR #72220 addresses. The staging script removes every `.openclaw-runtime-deps-*` directory before staging without recording ownership or checking whether another live build owns that temp directory, and current tests only cover stale temp cleanup retry behavior. The PR is a plausible focused fix, but the provided review comment identifies a remaining backup-dir ownership gap, so it should stay open for maintainer review and fixup rather than be closed as obsolete.

## Best Possible Solution

Keep PR #72220 open. The runtime-deps temp cleanup race remains present on current main, and the PR is a focused scripts/test fix. Maintainers should review the implementation, fix the backup-dir ownership gap from the review comment, then validate with the targeted script test and build gate before merge.
