## Summary

Keep this PR open. Current main still has the runtime-postbuild cleanup race the PR targets: staging temp directories are ownerless, cleanup removes every `.openclaw-runtime-deps-*` temp directory unconditionally before each plugin staging pass, and the existing tests cover stale cleanup retry rather than preserving live-owner temp directories.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land this PR or an equivalent change in `scripts/stage-bundled-plugin-runtime-deps.mjs` that writes owner metadata for active runtime-deps staging temp directories, skips cleanup for live owners, keeps the concurrent-cleanup regression test, and resolves or explicitly accepts the backup-directory owner-protection gap noted in review.
