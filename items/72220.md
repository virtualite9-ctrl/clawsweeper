## Summary

Keep this PR open. Current main still has the runtime-postbuild bundled runtime-deps temp cleanup behavior the PR targets: staging/temp directories are created without owner metadata, and cleanup deletes every `.openclaw-runtime-deps-*` entry before each staging pass. The adjacent runtime-deps install lock and run-node build lock do not fully cover plain build/runtime-postbuild overlap in this script.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land this PR or an equivalent change in `scripts/stage-bundled-plugin-runtime-deps.mjs` that marks active build/postbuild staging temp directories with owner metadata, skips cleanup for live owners, keeps the concurrent-cleanup regression test, and resolves or consciously accepts the backup-directory owner gap noted in review.
