## Summary

Keep open. Current main still has the reported MS Teams DM media gap: non-HTML DM inline images or file attachments can be reduced to unresolved placeholders, direct download skips non-HTTPS `msteams://media/...` references, and the Bot Framework/Graph fallbacks remain gated on HTML `<attachment>` ids. The related #40463 is still only an open implementation candidate in the provided context, not shipped behavior.

## Best Possible Solution

Keep this issue open until the msteams plugin changes inbound DM media fallback behavior. The likely fix belongs in `extensions/msteams/src/monitor-handler/inbound-media.ts` plus focused tests: after direct download fails, personal DM media should attempt the Bot Framework attachment path for real non-HTML media attachments or resolvable `msteams://media` placeholders, while preserving the #58617 guard against mention-only HTML noise. Review #40463 for reusable pieces, but close only after a merged fix includes regression coverage for the RSC-only personal DM inline image/file case.

A DM media attachment with no HTML `<attachment>` tag still has no fallback after direct download returns empty.
