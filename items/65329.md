## Summary

Current main still leaves the reported MS Teams personal DM non-HTML media path unresolved. Direct downloads do not resolve `msteams://media/...` placeholders, and the Bot Framework and Graph fallbacks are still gated on HTML `<attachment>` IDs, so inline DM images or attachments without that HTML wrapper can still be dropped.

## Best Possible Solution

Keep this issue open until the msteams plugin changes inbound DM media fallback behavior and adds focused regression coverage. The likely fix belongs in `extensions/msteams/src/monitor-handler/inbound-media.ts` plus tests: after direct download fails, personal DM media should attempt the Bot Framework attachment path for real non-HTML media attachments or resolvable `msteams://media` placeholders, while preserving the #58617 guard against mention-only HTML noise. Review #40463 for reusable implementation details, but close only after a merged fix covers the RSC-only personal DM inline image/file case.
