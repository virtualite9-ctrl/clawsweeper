## Summary

Keep PR #39323 open. Current main still does not implement owner notifications for new DM pairing requests, does not emit a `pairing:request` hook/event, and has no `pairing.notify` config surface. The related device-pair notification code is for node/device pairing, not DM sender pairing, and closed issue #38998 does not supersede the remaining work.

## Best Possible Solution

Keep the PR open as the active implementation candidate, or replace it with a maintainer-owned narrower design. The best path is to add a documented, versioned `pairing:request` event or plugin hook emitted only for freshly created DM pairing requests, then implement owner notification through a plugin or minimal core config. This should not close as ClawHub-only while the core/plugin surface still lacks an immediate DM pairing request event.
