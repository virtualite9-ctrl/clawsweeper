## Summary

Keep PR #51337 open. Current main still hard-codes gateway hook agent jobs to `sessionTarget: "isolated"`, and the cron runner treats that as `forceNew`, so the linked #51208 session-continuation bug is not implemented on main. The PR is a focused bug-fix attempt with useful discussion, but its head also has a later unresolved P1 review concern about `main`/global session canonicalization, so it needs author or maintainer follow-through rather than cleanup closure.

## Best Possible Solution

Keep the PR and linked bug active. The best path is to rebase or adapt the PR onto current main, add a first-class hook `sessionTarget` config/request policy if maintainers want that API, ensure gateway hook dispatch passes the intended target into cron without forcing isolated runs, persist the correct job session key for delivery routing, and add regression tests for mapped hooks, direct request gating, `current -> isolated`, `session:*`, configured `mainKey`, and `session.scope = "global"`.
