## Summary

Keep this issue open. Current main still installs Telegram sequentialize middleware before Telegram handlers, and ordinary callback queries still acknowledge inside the downstream `callback_query` handler. Because normal callback data shares the same chat/topic sequential key as regular messages, a button press can still wait behind an active same-topic agent turn before `answerCallbackQuery` runs. The contributor’s linked fixes #42181 and #42516 are closed unmerged in the provided GitHub context, so there is no canonical merged fix to close against.

## Best Possible Solution

Implement the Telegram plugin fix on current main: add a pre-sequentialize middleware in `extensions/telegram/src/bot-core.ts` that immediately answers callback queries with caught/logged failures, remove or avoid redundant handler-level acknowledgement in `extensions/telegram/src/bot-handlers.runtime.ts`, and add a regression test proving acknowledgement happens before the queued callback action is released when a same-topic turn is busy.
