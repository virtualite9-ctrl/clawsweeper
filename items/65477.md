## Summary

Keep open. Current main still builds the per-response `/usage` footer from token/cache/cost/session data only; it resolves the active provider/model elsewhere but does not pass that label into the usage-footer formatter. Existing response-prefix model interpolation and `/status` model display are separate surfaces, and the related usage-footer PRs in the provided context remain open/unmerged implementation candidates.

## Best Possible Solution

Keep this issue open unless maintainers explicitly consolidate it into a broader usage-footer tracker. The narrow implementation should pass the resolved `providerUsed` and `modelUsed` into `formatResponseUsageLine`, format them with the existing provider/model label helper, prepend or otherwise include that label in the `/usage` footer for the intended modes, preserve current cost/cache/session behavior, and add focused formatter plus `runReplyAgent` tests.
