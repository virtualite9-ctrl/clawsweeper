## Summary

Keep this issue open. Current main still does not include the active provider/model in the per-response `/usage` footer: the runner resolves the actual model for persistence, cost, events, and response-prefix interpolation, but the footer formatter only receives token/cache/cost inputs and `full` mode only adds the session key. Related PRs remain open implementation candidates rather than shipped behavior.

## Best Possible Solution

Keep this issue open and land a narrow usage-footer change, or explicitly consolidate it into a canonical model-attribution/footer tracker. The implementation should pass the resolved `providerUsed` and `modelUsed` into `formatResponseUsageLine`, format the label consistently with the existing response-prefix provider/model helpers, preserve current token/cache/cost/session behavior, and add focused formatter plus `runReplyAgent` tests for `tokens` and `full` modes.
