## Summary

Keep open. Current main still leaves the per-skill destination untouched after a failed `fsp.cp`, while the open PR adds the missing cleanup. The downstream Claude skill plugin still has the exact missing-file warning path this PR targets, and existing sync tests do not cover failed-copy cleanup yet.

## Best Possible Solution

Keep the PR open. The right product/code direction is to land a small failed-copy cleanup in `syncSkillsToWorkspace`, ideally with a regression test in the existing skills sync suite and either accepted or explicitly justified handling of cleanup `rm` failures. This is not obsolete on current main.
