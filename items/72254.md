## Summary

Keep this PR open. Current `main` still lacks the failed-copy cleanup this PR proposes, and the downstream Claude skills plugin still warns when snapshot-resolved skill files are missing. The PR is a focused implementation candidate, not obsolete or superseded; the remaining maintainer question is how much regression coverage and cleanup-failure logging to require before landing.

## Best Possible Solution

Keep this PR open. The best path is to land a small cleanup in `syncSkillsToWorkspace` after per-skill copy failure, add a focused regression test in the existing skills sync suite, and either log cleanup failures or explicitly preserve best-effort cleanup semantics for this error path.
