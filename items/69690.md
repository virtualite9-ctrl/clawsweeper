## Summary

Keep PR #69690 open. Current main has adjacent gateway-bindable registry reuse, but it still does not implement this PR's specific omitted-`coreGatewayHandlers` fix: gateway startup includes core handler method names in the plugin loader cache key, while helper/runtime loads can request gateway binding without re-supplying those handlers, and main has no active-state method-name record or synthetic-handler compatibility path. The PR discussion is concrete, includes repro logs and tests, and later commits addressed the bot-flagged default-mode inheritance risk, so it is not safe maintainer cleanup to close it as obsolete.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to rebase it or apply an equivalent patch on current main that persists the active gateway method-name set, uses it only for explicit gateway-bindable compatibility checks, and adds regression coverage for omitted handlers, mismatched method sets, and default-mode non-inheritance.
