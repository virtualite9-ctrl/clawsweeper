## Summary

Current `main` still has the cache-key mismatch described in PR #69690. Gateway startup loads a `gateway-bindable` registry with `coreGatewayHandlers`, the loader cache key incorporates those handler method names, and helper/runtime callers commonly omit `coreGatewayHandlers`, so current compatibility checks can miss the already-active registry and fall through to a fresh load. This PR addresses a live bug on `main`, so it should not be auto-closed; it still needs normal maintainer review on the proposed fix.
