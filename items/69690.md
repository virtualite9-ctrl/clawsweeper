## Summary

Keep open. Current main has adjacent gateway-bindable registry reuse logic, including the default-mode guard related to #61785, but it still does not implement this PR's exact fix: a gateway startup registry loaded with coreGatewayHandlers gets a cache key containing core gateway method names, while helper/runtime loads can request gateway binding without supplying those handlers, leaving the active registry cache-incompatible and allowing a fresh runtime load.

## Best Possible Solution

Keep the PR open for maintainer review or adaptation. The best path is to merge or reimplement the PR's active core-gateway-method recording plus synthetic-handler compatibility check, with tests proving that a gateway-bindable startup registry loaded with coreGatewayHandlers is reused when a helper load omits those handlers, while mismatched method sets and default-mode capability widening remain rejected.
