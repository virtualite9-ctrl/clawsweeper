## Summary

Keep open. Current main still has global cron storage and global cron CRUD/run/read paths without a trusted caller ownership scope. The issue also has concrete production reports of cross-user leakage/tampering and a maintainer comment from April 26, 2026 explicitly saying to keep #26370 open until list/update/remove/run/runs enforce cron ownership.

## Best Possible Solution

Keep #26370 open until core cron ownership is enforced. The likely product/code path is to pass a trusted caller scope from createCronTool into Gateway cron RPCs, enforce agentId/sessionKey ownership for list/update/remove/run/runs in Gateway/service code, preserve a separate admin/operator all-jobs path, migrate or backfill existing global jobs safely, and add regression tests plus docs for multi-agent and multi-user cron isolation.
