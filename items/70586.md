## Summary

Keep PR #70586 open. Current main still has a real Slack liveness coverage gap for message-subtype system events and DM-skipped app_mention events, but the submitted patch needs revision because normal message/app_mention flows already track liveness in createSlackMessageHandler, so the PR as written would double-update those events.

## Best Possible Solution

Keep the PR open and revise it rather than auto-close it. The best fix is to cover the registrar-only early-return paths, such as subtype system events and DM-skipped app_mention events, while preserving exactly one liveness update for normal message/app_mention events that already flow through createSlackMessageHandler. A focused regression test should assert both the newly covered early-return cases and no double trackEvent call on ordinary accepted messages.
