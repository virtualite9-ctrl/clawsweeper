## Summary

Current `main` already covers the underlying Discord cron/announce runtime problem through a different architecture: Discord announce delivery is exercised through the outbound adapter, Discord channel runtime access is fail-soft with a direct send fallback, and bundled facade activation checks now resolve identity from the actual loaded module path while preserving activation gating. The PR’s original `discord-runtime-surface` target no longer exists on `main`, so the branch is obsolete rather than mergeable.
