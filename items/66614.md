## Summary

Keep this issue open. Current main still only protects fenced code blocks in the streaming block chunker, has no markdown table-span safe-break guard, and the supplied GitHub context shows open PR #66344 with closing syntax as the active implementation candidate rather than merged shipped behavior.

## Best Possible Solution

Keep this issue open and review/land #66344, or land a smaller equivalent fix. The fix should add markdown table-span detection in the shared markdown/chunker layer, guard every `EmbeddedBlockChunker` break path with table-safe boundaries as well as fence-safe boundaries, and add focused regression tests for text + table + text, partial streamed tables, fenced tables, and oversized-table fallback. After the PR merges, GitHub can close this issue as implemented.
