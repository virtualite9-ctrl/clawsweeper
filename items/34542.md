## Summary

Keep open. Current main partially covers this PR via newer runtime diagnostics for `sessions_spawn(mode="session")`, but the PR's prompt/tool-copy/docs guidance is still not fully implemented on main.

## Best Possible Solution

Keep this PR open or replace it with a focused refresh on current main: preserve the shipped runtime diagnostics from #67790, add explicit `mode="run"` fallback guidance to `buildAgentSystemPrompt`, `buildSubagentSystemPrompt`, `describeSessionsSpawnTool`, and the public subagent/ACP docs, then add prompt/tool-description assertions so models stop requesting impossible persistent thread-bound sessions on unsupported requester channels.
