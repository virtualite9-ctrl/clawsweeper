## Summary

Keep this PR open. Current main already has stronger runtime diagnostics for `mode="session"` and unavailable thread binding, but the PR's prompt/tool-copy guidance remains partially missing: the main agent prompt, subagent prompt, and `sessions_spawn` tool description/schema still do not clearly tell agents to avoid persistent thread-bound sessions on unsupported channels and fall back to one-shot `mode="run"`.

## Best Possible Solution

Keep the PR open or ask the contributor to refresh it against current main: preserve the already-shipped runtime diagnostics, then add focused guidance to `buildAgentSystemPrompt`, `buildSubagentSystemPrompt`, `describeSessionsSpawnTool`/`createSessionsSpawnToolSchema`, and the public subagent/ACP docs so unsupported requester channels steer agents toward one-shot `mode="run"`. Add prompt/tool-copy assertions for the new wording.
