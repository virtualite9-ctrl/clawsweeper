## Summary

Keep this issue open. Current main still displays and resolves configured agent avatars, and Quick Settings has a browser-local assistant avatar override, but there is no first-party Agents-tab upload flow that writes image bytes under the agent workspace and updates IDENTITY.md.

## Best Possible Solution

Keep this issue open and implement a first-party Control UI agent avatar upload path in the gateway protocol/server methods plus the Agents tab UI. The likely solution is an admin-scoped upload method or endpoint that validates image bytes, writes a generated safe filename under the selected agent workspace avatars/ directory, updates IDENTITY.md through the existing identity helpers, refreshes agent identity/avatar metadata, and adds focused gateway/UI regression coverage.
