## Summary

Keep this issue open. Current main still supports displaying configured avatars and setting avatar strings through existing identity/config paths, but it does not implement the requested Agents-tab image upload workflow: there is no `agents.avatar.upload` RPC/schema/scope, no `workspace/avatars/` byte-storage handler, and no `/agents` avatar file picker. Related avatar-display fixes such as #17298 and #64998 do not supersede this upload/write request.

## Best Possible Solution

Keep this issue open and implement a first-party Control UI agent avatar upload flow. The likely path is a scoped admin RPC or upload endpoint that validates image type and size, safely writes bytes under `avatars/<agentId>.<ext>` inside the selected agent workspace, updates `IDENTITY.md` and the agent identity/config through existing helpers, then wires the Agents tab avatar click/picker to call it and refresh `agents.list`, `agent.identity.get`, and `/avatar/<agentId>`. Add gateway tests for validation, safe writes, and identity updates, plus UI tests for the Agents-tab picker.
