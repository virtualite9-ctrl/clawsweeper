## Summary

Keep this issue open. The linked patch describes a concrete first-party Agents-tab upload workflow, and current main still only supports configured avatar strings, avatar display routes, editable text workspace files, and a browser-local assistant avatar override. It does not implement a durable image upload path that writes agent avatar bytes under the selected workspace and updates `IDENTITY.md`.

## Best Possible Solution

Keep this issue open and implement a first-party Control UI agent avatar upload path. The likely fix belongs in gateway protocol/server methods plus Agents-tab UI: add an admin-scoped upload method or endpoint, validate image type and size, safely write bytes under the selected agent workspace `avatars/` directory, update `IDENTITY.md` through existing identity helpers, refresh agent identity/avatar state, and add gateway/UI regression tests.
