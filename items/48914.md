## Summary

Keep #48914 open. Current main still has the reported warning-classification gap: built-in group IDs such as `group:memory` are documented and expanded as core tool groups, but the warning path only recognizes known core tool IDs when choosing the gated-core warning wording. The PR addresses a real issue, though the provided diff still needs cleanup before merge because it removes part of the dedupe-cache regression test and should keep the own-property group lookup.

## Best Possible Solution

Keep the PR open and request a small update, or land an equivalent maintainer patch: add a helper that treats `CORE_TOOL_BY_ID` entries and own `CORE_TOOL_GROUPS` keys as known core policy entries, use it only in the warning-classification path, add the `group:memory` regression test, preserve the existing dedupe-cache replay test, and run the focused agent tool-policy test plus the changed gate.
