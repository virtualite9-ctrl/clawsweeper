## Summary

Keep this PR open. Current main still requests 200 messages during Control UI chat history hydration and still renders a 200-message chat history window, so the PR's central performance change is not implemented. The change remains plausible and narrowly scoped, but it needs a current-main rebase because the render-window logic now lives in `ui/src/ui/chat/build-chat-items.ts` rather than directly in the chat view.

## Best Possible Solution

Keep this PR open and ask for a current-main update. The best implementation is to introduce a shared initial chat history window constant in `ui/src/ui/chat/constants.ts`, use it from `ui/src/ui/controllers/chat.ts` for the `chat.history` request and from `ui/src/ui/chat/build-chat-items.ts` for the render slice/notice, then update `ui/src/ui/controllers/chat.test.ts` and add the render-window assertion in `ui/src/ui/chat/build-chat-items.test.ts`. No gateway protocol, server, dependency, or security-sensitive change is needed.
