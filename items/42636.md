## Summary

Keep this PR open. Current main still uses a 200-message Control UI chat history request and a 200-message render window, so the requested 80-message performance guard is not implemented. The PR is useful but stale against current main because render-window logic now lives in `ui/src/ui/chat/build-chat-items.ts`, while the PR diff still changes the old view-local path.

## Best Possible Solution

Keep this PR open and ask for a current-main update. The best implementation is to add a shared initial chat history window constant in `ui/src/ui/chat/constants.ts`, use it from `ui/src/ui/controllers/chat.ts` for the `chat.history` request and from `ui/src/ui/chat/build-chat-items.ts` for the render slice/notice, update `ui/src/ui/controllers/chat.test.ts`, and add a direct render-window assertion in `ui/src/ui/chat/build-chat-items.test.ts`. No gateway protocol, server, dependency, workflow, or security-sensitive change is needed.
