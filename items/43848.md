## Summary

Keep open. Current main partially addresses the Telegram forum-topic `threadId` loss, but the full reported bug is not implemented: `ANNOUNCE_SKIP` still suppresses visible announce delivery instead of falling back to the target reply, and the default A2A reply loop can still inject `inter_session` turns back into the requester session. The related fix PRs #43869 and #44064 are useful prior art, but both are closed unmerged in the provided GitHub context.

## Best Possible Solution

Keep this issue open and use the closed unmerged PRs #43869 and #44064 as prior-art references. The maintainer fix should stay in the `sessions_send` A2A path: preserve the existing Telegram `threadId` implementation, add regression coverage for Telegram topic handoffs where announce returns `ANNOUNCE_SKIP`, deliver the target reply when visible delivery would otherwise disappear, and suppress or redesign same-agent Telegram topic-to-topic ping-pong so target replies are not written back as fresh requester `inter_session` turns.
