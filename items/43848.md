## Summary

Current main only covers the Telegram forum-topic threadId routing portion. The ANNOUNCE_SKIP visible-delivery fallback and requester transcript pollution/ping-pong paths still remain in the sessions_send A2A flow, so this issue should stay open.

## Best Possible Solution

Keep this issue open. A maintainer fix should preserve the current threadId routing, add a visible-delivery fallback for Telegram topic handoffs where the announce step returns ANNOUNCE_SKIP after a real target reply, and suppress or redesign same-agent Telegram topic-to-topic ping-pong before it records fresh inter_session turns in the requester session. #43869 and #44064 are useful prior-art references but are closed and unmerged.
