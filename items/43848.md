## Summary

Current main only covers the Telegram forum-topic threadId portion of the report. The ANNOUNCE_SKIP visible-delivery fallback and requester transcript pollution paths still exist in the sessions_send A2A flow, so this issue should stay open.

## Best Possible Solution

Keep this issue open. A maintainer fix should preserve the current threadId routing, add regression coverage for Telegram topic handoffs where announce returns ANNOUNCE_SKIP, deliver the target reply when visible delivery would otherwise disappear, and suppress or redesign same-agent Telegram topic-to-topic ping-pong before it records fresh inter_session turns in the requester session. Use #43869 and #44064 as prior-art references, but do not close this issue until a maintained fix lands.
