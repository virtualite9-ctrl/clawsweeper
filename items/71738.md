## Summary

Keep #71738 open. Current main still has the Matrix thread-history and reply-placement behaviors the PR targets: room history is flat per room, Matrix reads do not have a thread-specific path, thread sends still add fallback reply metadata, and threaded edits still put reply context on the outer replace relation. The PR is recent, focused Matrix plugin work; the unresolved review comments are change requests to address before merge, not evidence that the PR is obsolete or out of scope.

## Best Possible Solution

Keep #71738 open as the active Matrix plugin fix. The best path is to have the author address the unresolved review findings, especially stale thread watermark invalidation after thread-queue eviction and pagination token correctness, keep the focused room-history/send/read regression tests, then rerun the Matrix extension test lane and extension typecheck before maintainer review.
