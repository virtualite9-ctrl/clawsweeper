## Summary

Keep open. Current main has OpenTelemetry model-call tracing and W3C `traceparent` propagation, but it does not implement the PR's configurable `diagnostics.sessionTracing` surface or the requested `x-session-id` / `x-session-name` style headers on outgoing LLM requests. The existing provider/plugin seams are provider-owned and do not expose both `runId` and `sessionKey` as a generic all-provider request-header injection API, so this is not strong enough for `implemented_on_main`, `clawhub`, or duplicate/superseded closure.

## Best Possible Solution

Keep the PR open for maintainer product/API review. The best path is either to port and harden the PR as a core `diagnostics.sessionTracing` feature with config schema, docs, privacy notes, and tests, or to add an explicit plugin API that lets observability plugins inject per-run request headers safely across providers. Current main's OTEL/traceparent support should be cited as related prior art, not treated as a complete replacement for the requested configurable session headers.
