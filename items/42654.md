## Summary

Keep open. Current main and the latest release still emit the combined `Current time: ... / YYYY-MM-DD HH:MM UTC` line, so this PR is not obsolete or implemented. The PR is a plausible targeted fix, but it is incomplete/stale because current main has several prompt consumers and tests that still expect the old single-line format, and the PR’s changed test path no longer exists on main.

## Best Possible Solution

Keep this PR open for author follow-up or supersede it with a revised PR that changes `resolveCronStyleNow` to emit separate `Current time:` and `Reference UTC:` lines, then updates all affected cron, heartbeat, session reset, post-compaction, memory-core, and prompt-composition expectations together. After that, maintainers can review the focused behavior change normally.
