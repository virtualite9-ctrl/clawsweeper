## Summary

Keep open. Current main and the latest release still emit the combined `Current time: ... / YYYY-MM-DD HH:MM UTC` line, so the requested two-line `Reference UTC:` behavior is not implemented or obsolete. The PR is a plausible focused fix, but it is stale/incomplete because current main has broader prompt consumers and tests still asserting the old format.

## Best Possible Solution

Keep this PR open for author follow-up or supersede it with a revised PR that changes `resolveCronStyleNow` to emit separate `Current time:` and `Reference UTC:` lines, then updates all affected cron, heartbeat, session reset, post-compaction, memory-core, and prompt-composition expectations together. Maintainers can then review the focused prompt behavior change normally.
