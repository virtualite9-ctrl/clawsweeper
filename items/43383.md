## Summary

Current main still appears to have the reported sandbox skill prompt mismatch. Sandbox setup now mirrors eligible skills into the non-rw sandbox workspace, but the embedded attempt and compaction paths still reuse the host-built skills snapshot; when that snapshot has resolved skills, live sandbox skill loading is skipped and the snapshot prompt wins. Related PR #38789 remains open as the focused implementation candidate, while #43526, #43789, and #51471 were closed unmerged or superseded.

## Best Possible Solution

Keep this issue open and review or land PR #38789, or implement the same fix directly on main. The fix should rebuild or remap the available-skills prompt from sandbox-readable skill entries for sandboxed non-rw attempt and compaction runs instead of returning the host-built snapshot prompt. Preserve skill filters, agent allowlists, environment override behavior, remote eligibility notes, and empty-sandbox behavior, then add regression coverage for snapshot-present attempt and compaction runs.
