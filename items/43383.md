## Summary

Current main still appears to have the reported sandbox skill-path mismatch. Skills are mirrored into non-rw sandbox workspaces, but the embedded attempt and compaction paths still carry the original host-built skills snapshot through to prompt resolution; when that snapshot has `resolvedSkills`, live sandbox skill entries are skipped and `resolveSkillsPromptForRun()` returns the snapshot prompt unchanged. The related PRs #43526 and #51471 were closed unmerged, and the remaining implementation path is the canonical sandbox skill prompt work in #38789 plus this issue.

## Best Possible Solution

Keep this issue open and finish the sandbox-specific skill prompt fix in the agents code path, likely together with #38789. For non-rw sandbox attempt and compaction runs, rebuild or remap the skills prompt from the sandbox-readable skill set instead of returning a host-built snapshot prompt, while preserving `skillsSnapshot.skillFilter`, remote eligibility notes, env override alignment, and empty-sandbox behavior. Add regression coverage for snapshot-present non-rw sandbox attempt and compaction runs so `~/.openclaw/skills/...` cannot reappear in sandbox prompts.
