## Summary

Current main still appears to have the reported sandbox skill prompt mismatch. Non-rw sandbox setup mirrors eligible skills into the sandbox workspace, but the embedded attempt and compaction paths still pass the host-built skills snapshot through; when that snapshot contains resolved skills and a prompt, live sandbox skill loading is skipped and the snapshot prompt wins. The prior ClawSweeper keep-open comment remains accurate on current main, and #38789 remains the focused open implementation candidate while #43526, #43789, and #51471 are closed unmerged or superseded.

## Best Possible Solution

Keep this issue open and review or land #38789, or implement the same fix directly on main. The fix should rebuild or remap the available-skills prompt from sandbox-readable skill entries for sandboxed non-rw attempt and compaction runs instead of returning the host-built snapshot prompt. It should preserve skill filters, agent allowlists, environment override behavior, remote eligibility notes, and empty-sandbox behavior, then add regression coverage for snapshot-present attempt and compaction runs.
