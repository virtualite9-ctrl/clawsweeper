## Summary

Keep #46585 open. Current main still has the macOS node choose an explicit camera device ID before applying the requested facing, so `--device-id <Continuity camera> --facing front` and `--facing back` can hit the same external/unspecified AVCapture device. The cross-referenced issues #58158/#58159 were about `camera.snap` allowlist availability, not Continuity Camera front/back selection, so they do not supersede this report.

## Best Possible Solution

Keep the issue open for a macOS node camera fix or explicit limitation. The likely paths are: detect when `deviceId` resolves to a single external/Continuity device with `position == .unspecified` and return a clear unsupported-facing error, or implement a verified AVFoundation selection path if macOS exposes distinct Continuity front/back devices. Update `docs/nodes/camera.md` and add focused macOS/CLI coverage for the chosen behavior.
