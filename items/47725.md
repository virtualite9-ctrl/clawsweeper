## Summary

Issue #47725 describes an optional multi-agent orchestration system. Current main scope explicitly routes this kind of capability to plugins/skills published on ClawHub rather than OpenClaw core, and the vision doc also excludes agent-hierarchy and heavy orchestration frameworks as default core architecture.

## Close Comment

Closing as a `clawhub` scope item after Codex review. The request is for an optional multi-agent orchestration/"agent society" system, and current main explicitly routes that kind of capability out of OpenClaw core: `VISION.md:54` says optional capability should usually ship as plugins, `VISION.md:69` says new skills should be published to ClawHub first, and `VISION.md:101-107` says OpenClaw is not merging agent-hierarchy frameworks or heavy orchestration layers as default core architecture right now. The plugin docs also place `agent harnesses` and `skills` in the plugin surface (`docs/tools/plugin.md:11-15`), and the skills CLI is built around installing skills from ClawHub (`docs/cli/skills.md:12-18`, `docs/cli/skills.md:40-45`). Given that scope, this is better pursued as a ClawHub skill/plugin rather than a core OpenClaw issue.
