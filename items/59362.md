## Summary

Keep open. Current main still routes host exec tool commands through the Windows PowerShell shell path, so it matches the reported root cause rather than proving the console-flash issue fixed or obsolete. Prior windowsHide work covers the spawned child options, but this report is specifically that windowsHide is insufficient when the spawned process is PowerShell.

## Best Possible Solution

Keep #59362 open. The likely fix belongs in the exec tool host execution path: add a conservative Windows direct-spawn mode only for safely parsed simple executable invocations, keep PowerShell for true shell syntax, preserve approval analysis, and add Windows-focused tests proving simple commands bypass PowerShell while shell commands still behave correctly. Manual Windows verification should confirm no visible console window appears for the direct path.
