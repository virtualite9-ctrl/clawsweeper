## Summary

Keep open. Current `main` still wires `openclaw voicecall` directly to the plugin runtime instead of the existing gateway RPC methods, so the CLI and a running gateway plugin would both try to own the webhook server. The code path still calls `ensureRuntime()` from the CLI actions, `createVoiceCallRuntime()` still starts `webhookServer.start()`, and the webhook server still binds `config.serve.port` with no cross-process coordination.

## Evidence
