## Summary

Keep this PR open. Current main has shipped overlapping strict-agentic/update_plan handling and a 120s LLM idle timeout, but it still does not implement the PR's dispatch-level plan-stall fail-close or memory session-delta retry/backoff behavior. The PR also has unresolved review findings that make it unsafe to close as implemented or land as-is.

## Best Possible Solution

Keep this PR open, request a rebase onto current main, and split it into smaller reviewed patches. The useful remaining slices are dispatch-level fail-closed handling for plan/control progress stalls, memory session-delta fetch-failure retry/backoff that preserves pending work, and memory search fallback that preserves existing hybrid/vector semantics. Treat shipped strict-agentic behavior from #64241 and later parity work as partial overlap, not a replacement for the remaining dispatch and memory fixes.
