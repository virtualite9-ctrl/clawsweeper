## Summary

Keep this PR open. Current main has later strict-agentic plan-only retry/blocking behavior, but it does not implement the PR’s two central changes: a dispatch-level fail-closed guard for a resolver that emits plan progress and then stalls, or retry/backoff preservation for memory session-delta sync after fetch failures. Several additional hunks from the PR are also absent or intentionally different on main, and the PR has unresolved review findings that should be addressed before any landing decision.

## Best Possible Solution

Keep this PR open, but ask for a rebase and split/narrowing. The highest-value path is to preserve the still-missing fixes as smaller reviewed patches: dispatch-level fail-closed handling for plan/control progress stalls, memory session-delta retry/backoff that preserves pending work, and any memory keyword fallback that maintains current hybrid/vector semantics. The strict-agentic plan-only work already on main should be treated as the baseline, and the unresolved review comments should be fixed before landing.
