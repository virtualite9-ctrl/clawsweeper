## Summary

Keep open. Current main still does not route ACP server diagnostics away from stdout before ACP startup, and the issue describes a real protocol correctness bug. The current `serveAcpGateway()` loads config, resolves the gateway bootstrap, starts the Gateway client, then binds the ACP NDJSON transport to `process.stdout` without importing or calling `routeLogsToStderr()`. Existing logging code still emits subsystem info/debug lines through stdout unless `loggingState.forceConsoleToStderr` is set, and the supplied GitHub context shows the direct fix PRs are still open rather than merged/shipped.

## Best Possible Solution

Keep #49060 open and land a small core fix, likely by importing `routeLogsToStderr` in `src/acp/server.ts`, calling it at the very start of `serveAcpGateway()` before config/bootstrap/gateway work, and adding focused startup coverage that proves ACP server mode enables stderr routing before any stdout-backed NDJSON transport can be corrupted. Coordinate with the existing open fix PRs (#49072/#49085) rather than closing the issue prematurely.
