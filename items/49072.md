## Summary

Current `main` still leaves `openclaw acp` without an ACP-specific `routeLogsToStderr()` call, while the ACP docs describe a stdio protocol surface and comparable stdout-producing commands already redirect diagnostics off stdout. The cleanup is therefore not implemented on `main`, and the PR remains a plausible fix worth maintainer review rather than auto-close.
