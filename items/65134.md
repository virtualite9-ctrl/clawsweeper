## Summary

Keep this PR open. Current main still runs session-memory slug generation through `runEmbeddedPiAgent`, creates a temp session file, resolves the default agent model, and queues through embedded session/global lanes. The PR’s core slug-generator change addresses a real remaining problem also tracked by open issue #33962 and matches the existing lightweight `completeSimple` pattern, but the submitted branch should be narrowed because it also changes timeout-failover logging and does not add regression coverage for the slug path.

## Best Possible Solution

Keep this PR open as a plausible implementation candidate. Ask the contributor to narrow it to the slug-generator change, drop or split the timeout-failover logging commit, and add focused regression coverage proving `generateSlugViaLLM` uses `completeSimple`, avoids temp session files and embedded-runner lanes, prefers `agents.defaults.heartbeat.model` when configured, and falls back to the primary model otherwise. Cross-link #33962 as the canonical user-facing issue for the remaining product problem.
