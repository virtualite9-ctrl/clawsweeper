## Summary

Current `main` still reproduces the reported trailing-whitespace bug in `chunkTextByBreakResolver`, and the existing test coverage does not cover the failing final-chunk cases from the issue. The issue should remain open.

## Evidence

- **Current implementation still pushes untrimmed final chunk:** `chunkTextByBreakResolver` trims loop-emitted chunks with `trimEnd()` but still does `chunks.push(remaining)` for the final chunk, so trailing whitespace is preserved there.

- **Reproduced on current main:** A read-only `node --input-type=module -e ...` reproduction against the current function logic returned `["! "]` for `"  ! "` with limit `2`, and `["a","b "]` for `"a b "` with limit `2`, matching the issue's reported bug.
