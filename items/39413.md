## Summary

Keep open. Current main still does not implement a configurable `tools.exec.shell` or per-agent shell override, and the PR is a focused attempt at a real core exec behavior gap. Related issue #49931 tracks the broader product request, but the provided context does not show a merged fix or explicit maintainer consolidation that supersedes this PR.

## Best Possible Solution

Keep the PR open for maintainer review or explicit consolidation into #49931 or a newer canonical PR. A complete implementation should define the supported shell override contract, add typed global and per-agent config plus schema/help/docs, route the selected shell through direct exec, PTY supervisor, gateway approval follow-up runs, and any node/sandbox policy decisions, then add tests for precedence, fallback, Windows shell arguments, validation, approval paths, and docs/schema drift.
