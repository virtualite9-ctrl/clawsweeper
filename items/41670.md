## Summary

Keep open. Current main still omits `encoding_format` on the shared remote embedding request path, while this PR is a narrow one-line fix against the current `src/memory-host-sdk` file. The change matches the existing float-vector parser and does not introduce security-sensitive surfaces. Related #43910 is broader configurability work and its live diff still targets older `src/memory/*` paths, so it is not a clean superseder for this focused compatibility fix.

## Best Possible Solution

Keep this PR open as a valid implementation candidate. The best path is to review and land the minimal `encoding_format: "float"` change after adding or confirming a focused regression test for the shared remote embedding provider, then handle configurable `encodingFormat` or base64 decoding separately through #43910 or a fresh current-main PR if maintainers want that broader feature.
