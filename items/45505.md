## Summary

Keep open. Current main includes the narrow late-success `compactionCount` reconciliation from #45493, but the broader request remains unresolved: token/context snapshots, session id/file rotation, UI/status payload state, and downstream lifecycle consumers still converge through separate run-finalization paths rather than one authoritative post-compaction completion boundary.

## Best Possible Solution

Keep this as a core session-state architecture follow-up. The best path is to promote the existing count-only late-success hook into a single idempotent compaction-completion reconciliation boundary that deliberately owns compaction-derived durable state, including count, token snapshot/freshness, session id/file rotation, and any downstream lifecycle consumers that are semantically tied to successful compaction completion.
