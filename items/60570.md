## Summary

Issue #60570 is implemented on current `main`: `tools.exec.host=auto` now allows an explicit `host=node` override when no sandbox runtime is active, and the behavior is covered by code, tests, docs, and changelog entries. The stricter block only remains for sandboxed sessions, which is now documented policy rather than the old strict-equality bug described in the report.

## Evidence

- **Current resolver allows `auto` -> `node` when no sandbox is active:** `isRequestedExecTargetAllowed` returns `true` for `configuredTarget === "auto"` unless a sandbox runtime is active and the request would bypass that sandbox; `resolveExecTarget` uses that result before throwing.
