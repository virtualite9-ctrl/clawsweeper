## Summary

Keep #6625 open. Current main still exposes and enforces sub-agent run timeouts, plus post-timeout partial-progress recovery, but there is still no configurable pre-timeout grace period, no sessions_spawn warning parameter, and no native or ACP path that injects a checkpoint warning before the timeout expires.

## Best Possible Solution

Keep #6625 open for core sub-agent lifecycle work. Add a documented sub-agent timeout-warning/grace setting, optionally a sessions_spawn override, schedule a warning before runTimeoutSeconds expires, deliver it through an appropriate child-session control/message path for native and ACP runs where feasible, and cover it with focused tests and docs while preserving the existing post-timeout partial-progress recovery.

## Risks / Open Questions

- Existing post-timeout partial-progress extraction can be mistaken for this feature, but it does not give the sub-agent a pre-timeout opportunity to write files, summarize state, or complete a checkpoint.
- A correct implementation needs careful lifecycle design across native and ACP runs so the warning does not race the hard timeout, cleanup, announce delivery, steering, or restart/orphan recovery.
