## Summary

Keep #6625 open. Current main at c7a0d9b1889ca09ab5bfd84cda39a8d50d3d5768 still implements hard sub-agent run timeouts plus post-timeout partial-progress/announce handling, but I found no configurable pre-timeout grace window or injected checkpoint warning for native or ACP sub-agents. The existing issue comment also explicitly keeps this open and distinguishes it from partial-result persistence work.

## Best Possible Solution

Keep #6625 open for core sub-agent lifecycle work. Add a documented agents.defaults.subagents warning/grace setting, schedule a warning before runTimeoutSeconds, deliver it through the existing child-session control/message path for native and ACP sub-agents where feasible, and add focused tests/docs while preserving current post-timeout partial-progress and fallback-delivery behavior.
