## Summary

Keep open. Current `main` does not already implement this fix: Slack error logging still routes through generic formatters that only surface the top-level message/cause, and there is no `formatSlackApiError`-style helper wired into Slack call sites. The PR as described would still be needed in some form, but the submitted patch itself appears insufficient because the helper is not present on `main` and current logging paths remain unchanged.

## Evidence

- **No Slack-specific formatter on main:** A repo search found no `formatSlackApiError` symbol or equivalent helper in the current tree.
  - command: `rg -n "formatSlackApiError" extensions/slack src`
