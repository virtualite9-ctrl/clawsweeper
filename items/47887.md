## Summary

Keep open. Current `main` still contains the reported LAN/setup-code behavior: iOS manual connects still force TLS for every non-loopback host, iOS setup-code parsing/tests still reject non-loopback `ws://` URLs, setup-code generation still emits bootstrap-only payloads, and both TypeScript and Swift clients still prefer `bootstrapToken` over `password` when both are present. There is partial mitigation for bootstrap-token reuse in `NodeAppModel`, but the issue is not implemented on `main` well enough to close.
