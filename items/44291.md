## Summary

Keep #44291 open. Current main has Windows CI coverage, but the Windows lane defaults to bash and only runs the focused `pnpm test:windows:ci` suite; the lone `shell: pwsh` step is a Defender setup helper. The contributor commands named in the issue, including docs checks and strict smoke, are still run from Ubuntu/bash-oriented lanes rather than native PowerShell.

## Best Possible Solution

Add a scoped native PowerShell CI smoke, either as a small new Windows job or an additional `checks-windows` matrix task, that uses `shell: pwsh` for selected contributor commands such as `pnpm check:docs` and a strict-smoke-safe subset. Update `docs/ci.md` so contributors and maintainers know which commands are guaranteed to work in native PowerShell.
