## Summary

Current main already registers Anthropic as an image-capable media-understanding provider, and the published v2026.4.24 npm tarball contains the same Anthropic provider metadata/runtime. Current main also auto-registers config-defined providers that declare image-capable models, so custom ids such as anthropic-new/anthropic-paygo do not need sed-injected provider stubs when their model config includes image input metadata.

## Best Possible Solution

Close this issue as already implemented. Keep the shipped Anthropic provider registration and config-provider auto-registration path. For custom Anthropic-compatible provider ids, the expected configuration is to declare the vision model with input ["text", "image"]; if a user can still reproduce on an unmodified published package, they should file a new issue with the exact models.providers config, imageModel/tool invocation, bundle filename/hash, and logs because the current source and v2026.4.24 tarball do not match the reported missing Anthropic registry state.
