## Summary

Keep open. Current main registers the bundled Anthropic media-understanding provider and has a generic configured-provider workaround, but it does not implement the requested Anthropic-family routing for provider ids like `anthropic-new`, `anthropic-paygo`, or `nexos` without explicit image-capable model metadata.

## Best Possible Solution

Keep this open and fix it in source, not by patching generated dist. The best path is to add a tested generic Anthropic-compatible media-understanding route, such as alias/family handling for configured Anthropic-flavor providers or safe image-capability inference for `models.providers.*` entries using `api: "anthropic-messages"` with Claude vision model ids. Add regression coverage for `anthropic-new`, `anthropic-paygo`, and a custom proxy-style id, and document the custom-provider contract or the required `models.providers.*.models[].input` metadata.
