## Best Possible Solution

Keep this PR open as a focused bug-fix candidate. Port the cache-ownership check to `src/agents/github-copilot-token.ts`, preserve the SDK/plugin re-export path, add regression tests for different GitHub tokens and legacy unbound cache entries, and review it together with #43658.

## Risks / Open Questions

- The PR cannot be merged as-is without porting because current main uses `src/agents/github-copilot-token.ts` plus `openclaw/plugin-sdk/github-copilot-token`, while the PR targets the old `src/providers/github-copilot-token.ts` path.
- The proposed fingerprint stores a stable credential-derived identifier on disk. It is not the raw PAT, but maintainers should decide whether that is acceptable or prefer a profile/account-owned cache key.
- Even with a fingerprint check, the single cache file would still evict valid tokens when alternating profiles; that is an efficiency follow-up, not a reason to close this correctness fix.
