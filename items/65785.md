## Summary

Keep open. Current main still does not allow MiniMax web_search to use MINIMAX_OAUTH_TOKEN: the runtime credential resolver only checks config, MINIMAX_CODE_PLAN_KEY, MINIMAX_CODING_API_KEY, and MINIMAX_API_KEY, while MiniMax OAuth is a real bundled provider path. The PR remains useful, though it needs rebasing onto the current lazy/runtime file split and should also address the review note that MINIMAX_API_KEY is missing from web_search envVars metadata.

## Best Possible Solution

Keep the PR open and ask the contributor or maintainer to rebase it onto current main by updating extensions/minimax/src/minimax-web-search-provider.runtime.ts for credential resolution and missing-key messaging, extensions/minimax/src/minimax-web-search-provider.ts for envVars metadata, and the MiniMax web_search tests/registration assertions. Include MINIMAX_API_KEY in envVars alongside MINIMAX_OAUTH_TOKEN to resolve the existing metadata mismatch, then run the targeted MiniMax web_search test lane.
