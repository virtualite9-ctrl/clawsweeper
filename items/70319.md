## Summary

Keep open. Current main still has abort and queue primitives, but it does not yet provide the unified run-generation invalidation, stale-output fencing, pre-tool gate, or new-message takeover guarantee requested here. The same-author PR #70363 is an open draft and explicitly partial, so it is an implementation candidate rather than a reason to close this issue.

## Best Possible Solution

Keep this issue open as the canonical core interruptibility tracker. Review or split #70363, then land focused changes for a session/run generation primitive, stale-output fences across reply/typing/followup/tool-progress emission sites, a pre-tool abort/generation gate in the Pi tool path, new-message takeover that invalidates and cancels the active run, and targeted tests proving stale runs cannot emit visible effects or launch further tools.
