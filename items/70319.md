## Summary

Keep open. Current main still has the scattered abort primitives described in #70319, but it does not yet implement the requested unified per-session generation invalidation or broad stale-output fencing. The linked #70363 is useful related work, but the provided PR context says it is draft and explicitly partial, and its head commit is not present in this checkout.

## Best Possible Solution

Keep #70319 open as the umbrella issue until a reviewed implementation lands on main. The best path is to use #70363 as a possible foundation, but require follow-up coverage for the remaining requested guarantees: per-session generation invalidation, pre-tool gating, stale-output fences across block/non-block replies, typing/progress/events/tool results/followups, subprocess/scope cancellation, and explicit new-message takeover semantics for the intended session APIs.
