## Summary

Keep #58730 open. Current main has meaningful partial coverage: optional sandbox backends, host exec approval/allowlist controls, context compaction/pruning, persisted session transcripts, and hook surfaces. But the central security/product request is not fully implemented: sandboxing is still optional/off by default, unsandboxed host exec still defaults to full/no-prompt behavior unless configured otherwise, there is no merged macOS `sandbox-exec` or Linux `bwrap` wrapper in the exec tool schema/runtime, and the linked #62937 PR is open/draft and only covers part of the umbrella request.

## Best Possible Solution

Keep the item open or split it into focused maintainer-owned follow-ups: one for default exec sandbox/isolation policy, one for OS-level backend support if desired, and one for permission-model changes that go beyond the existing `security`/`ask` approvals contract. Review #62937 as a partial macOS prototype, but do not close #58730 until the remaining core security behavior is either shipped or explicitly superseded by narrower canonical issues.
