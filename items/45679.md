## Summary

Keep #45679 open. Current main supports manual Telegram silent delivery and a separate `silentErrorReplies` option, but it does not expose or implement a `silentVerbose`/`verboseSilent` config that keeps verbose/streaming content visible while sending only those intermediate messages with `disable_notification: true`. The related #2249 and #9239 context remains distinct: manual `--silent` exists, and hiding tool progress exists, but neither solves auto-silent visible verbose previews.

## Best Possible Solution

Implement a Telegram-scoped, default-off `silentVerbose` config using the name agreed in the issue discussion. Add it to Telegram config types, zod schema, generated/config UI metadata, and docs; resolve it through account/channel config; apply it to visible intermediate verbose/streaming delivery paths, including draft preview initial sends/materialization and block/non-final verbose messages, while leaving normal final replies notifying. Add focused tests proving verbose/preview messages pass `disable_notification: true`, final replies do not, and `silentErrorReplies` still works.
