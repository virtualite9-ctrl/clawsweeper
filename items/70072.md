## Summary

Keep #70072 open. Current main has partial introspection through `openclaw memory promote-explain` and JSON promotion previews, and the related #68882 gate issue is closed, but the requested deep-run artifact remains missing: the scheduled deep report still only records ranked/promoted counts, verbose candidate details are logger-only, and there is no `openclaw memory dreaming doctor` or equivalent aggregate gate-reachability command.

## Best Possible Solution

Keep this open and implement a focused `memory-core` diagnostic path: factor ranking diagnostics so deep sweeps can record candidate score components, thresholds, promoted/rejected state, and gate-failure reasons in `memory/dreaming/deep/YYYY-MM-DD.md`; optionally emit JSONL when `verboseLogging` is enabled; and either add an aggregate doctor command under the existing memory/doctor surfaces or split that third request into a narrower follow-up.
