## Summary

Keep open. Current main has useful partial introspection through `openclaw memory promote-explain` and Dreams UI status data, and #68882’s zero-promotion bug was separately closed as implemented, but #70072’s central additive request remains missing: deep-pass reports still do not list each ranked/rejected candidate with gate reasons, verbose scoring is logged rather than written as structured workspace data, and there is no `openclaw memory dreaming doctor` diagnostic command.

## Best Possible Solution

Keep the issue open. The best implementation path is to add a first-class deep candidate evaluation result in `memory-core` that preserves both accepted and rejected candidates with gate reasons, use it to enrich `memory/dreaming/deep/YYYY-MM-DD.md`, optionally write structured verbose scoring under `memory/.dreams/`, and expose aggregate gate reachability through `openclaw memory dreaming doctor` or an equivalent expansion of `openclaw memory status --deep`.
