## Summary

Keep PR #39389 open. Current main still lacks the Feishu-channel documentation this PR targets, but the submitted wording needs revision because Feishu `defaultAccount` is separate from the routing bucket used when `match.accountId` is omitted.

## Best Possible Solution

Keep the PR open and revise the Feishu docs patch. The best version belongs in `docs/channels/feishu.md`: document that omitted `match.accountId` follows the routing contract for the literal `default` account key, explicit Feishu account ids such as `main` match named accounts, and `accountId: "*"` matches any configured Feishu account. Also clarify that `channels.feishu.defaultAccount` is outbound/default account selection and should not be conflated with inbound binding omission. Coordinate wording with #49769 if that broader PR proceeds, but keep the Feishu-specific guidance on the Feishu channel page.
