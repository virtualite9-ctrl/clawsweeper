## Summary

Keep open. Current `main` does not implement this Feishu doc change, and the proposed wording appears unsafe for Feishu multi-account routing because route matching keys an omitted `match.accountId` to the literal `default` account while Feishu inbound routing passes the concrete active account id (for example `main`).

## Evidence

- **Omitted binding accountId is keyed to literal `default`:** `resolveAccountPatternKey` maps an empty account pattern to `DEFAULT_ACCOUNT_ID`, and account-scoped matching then looks up only `byAccount.get(accountId)` plus `accountId:"*"` wildcard bindings.
  - file: [src/routing/resolve-route.ts:232](https://github.com/openclaw/openclaw/blob/cb2c36b04975e34ab7014a11575d6589aa96ae32/src/routing/resolve-route.ts#L232)
  - command: `nl -ba src/routing/resolve-route.ts | sed -n '228,250p'; nl -ba src/routing/resolve-route.ts | sed -n '428,450p'`
  - sha: [cb2c36b04975](https://github.com/openclaw/openclaw/commit/cb2c36b04975e34ab7014a11575d6589aa96ae32)
- **Feishu inbound routing uses the concrete active account id:** Feishu bot and comment flows call `resolveAgentRoute` with `account.accountId`, not with an omitted account id or `channels.feishu.defaultAccount`. A named default such as `main` stays `main` at route time.
  - file: [extensions/feishu/src/bot.ts:590](https://github.com/openclaw/openclaw/blob/cb2c36b04975e34ab7014a11575d6589aa96ae32/extensions/feishu/src/bot.ts#L590)
  - command: `nl -ba extensions/feishu/src/bot.ts | sed -n '584,606p'; nl -ba extensions/feishu/src/comment-handler.ts | sed -n '132,165p'`
  - sha: [cb2c36b04975](https://github.com/openclaw/openclaw/commit/cb2c36b04975e34ab7014a11575d6589aa96ae32)
- **Feishu docs currently scope `defaultAccount` to outbound selection:** The Feishu channel doc says `defaultAccount` controls which account is used when outbound APIs do not specify an `accountId`; the current Feishu multi-agent section does not document any inbound exception.
  - file: [docs/channels/feishu.md:229](https://github.com/openclaw/openclaw/blob/cb2c36b04975e34ab7014a11575d6589aa96ae32/docs/channels/feishu.md#L229)
