## Summary

Keep open. Current main still has the reported bug: non-Azure custom provider onboarding writes new models with `input: ["text"]`, and current tests assert that behavior. PR #72171 is a recent, focused fix attempt for open issue #51869, but the provided PR review context shows the current head is still incomplete for existing configs and has model-pattern coverage gaps, so it needs revision rather than cleanup closure.

## Best Possible Solution

Keep #72171 open for maintainer review and ask the contributor to revise it against current main: update `src/commands/onboard-custom-config.ts` so non-Azure custom onboarding can mark image-capable models as `input: ["text", "image"]` through a conservative, well-tested heuristic or explicit prompt/flag; preserve deliberate user customizations; handle existing text-only entries on re-onboard deliberately; and add regression tests for new models, existing models, negative text-only cases, and the reviewed model-family gaps. Coordinate with #51869 and the older competing PRs (#51893/#51965) so only one fix path remains canonical.
