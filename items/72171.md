## Summary

Keep open. Current main still hardcodes non-Azure custom-provider models to `input: ["text"]`, so the bug tracked by #51869 is not implemented on main. PR #72171 is a focused fix attempt, but the provided review comments show the current head remains incomplete for existing configs and has model-pattern coverage gaps, so it needs revision rather than cleanup closure.

## Best Possible Solution

Keep #72171 open for maintainer review and ask for a revision against current main. The core onboarding fix should deliberately mark image-capable non-Azure custom models as `input: ["text", "image"]`, preserve deliberate user customizations, define how re-onboarding existing text-only entries should behave, and add regression tests for new models, existing models, negative text-only cases, and the reviewed model-family gaps. Coordinate with #51869 and older competing PRs so only one fix remains canonical.
