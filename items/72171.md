## Summary

Keep open. Current main at df542f75 still hardcodes non-Azure custom-provider onboarding to text-only model input, and runtime image delivery is gated by that metadata. #72171 is a relevant, current-helper PR for the linked open bug #51869; review comments raise scope/heuristic refinements, not a reason to close it as obsolete.

## Best Possible Solution

Keep #72171, or a maintainer-selected canonical follow-up, open until the fix ships. The best implementation belongs in `applyCustomApiConfig()`: detect or ask for image-capable custom models, set `input: ["text", "image"]` for matching new non-Azure models, handle existing text-only entries on re-onboard without unintended metadata clobbering, and cover both paths in `src/commands/onboard-custom-config.test.ts`.
