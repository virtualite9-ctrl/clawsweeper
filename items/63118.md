## Summary

Current main still does not provide the requested public session lifecycle management. It has lower-level abort plumbing, Chat Stop, sub-agent kill paths, and session metadata/delete/compaction controls, but no `openclaw sessions stop/kill/pause/resume`, no Sessions-page Stop/Pause/Resume controls, and no enforceable session pause/resume or cost/quota lifecycle model.

## Best Possible Solution

Keep this open as the broad session lifecycle tracker. Implement a narrow `openclaw sessions stop <session-key>` first, backed by the existing `sessions.abort`/`chat.abort` plumbing, with CLI docs and tests. Add a Sessions-page Stop control only when it is wired to real Gateway abort behavior and visible active-run state. Treat #35735 as separate deletion cleanup, and defer pause/resume/resource quotas until the Gateway runtime, provider cancellation, and accounting model can enforce them.

## Risks / Open Questions

- Closing now would hide a real public CLI and Sessions UI lifecycle gap that current lower-level abort primitives only partially cover.
- Pause/resume and resource quotas need enforceable Gateway/runtime/provider semantics; metadata-only controls would mislead users about whether work and cost are actually stopped.
- A future `sessions stop` should have clear authorization, active-run status reporting, docs, and regression tests so it is not confused with delete/reset cleanup.
