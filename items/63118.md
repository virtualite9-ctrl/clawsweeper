## Summary

Keep open. Current main has useful lower-level session controls, but it still lacks the requested `openclaw sessions stop/kill/pause/resume` CLI lifecycle surface, per-session Stop/Pause/Resume controls on the Sessions UI page, and a real pause/resume/resource-limit model.

## Best Possible Solution

Keep this open as the broad session lifecycle tracker. The pragmatic path is to first add narrow CLI commands backed by existing Gateway behavior, such as `openclaw sessions stop <key>` using `sessions.abort` and a clearly named delete/archive command using `sessions.delete`; keep `rm/clear` discussion with #35735; then decide whether true pause/resume and resource limits need new protocol/runtime support or should be split into scoped follow-ups. The Sessions UI should expose only controls backed by real Gateway behavior.
