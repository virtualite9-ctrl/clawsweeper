## Summary

Current `main` already exposes pre-reset and pre-compaction lifecycle hooks that cover the requested `/new` and `/compact` interception window, via both internal hooks and plugin hooks.

## Risks / Open Questions

- The exact proposed `agents.defaults.sessionHooks.onBeforeNew/onBeforeCompact` config shape does not exist; the close relies on equivalent existing hook surfaces rather than the same config API.

## Close Comment

Closing from a Codex review because current `main` already has the lifecycle interception requested here, just under the existing hook systems rather than a new `sessionHooks` config block. `/new` and `/reset` already fire pre-reset hooks before the session is mutated (`src/gateway/session-reset-service.ts:488`, `src/auto-reply/reply/commands-reset-hooks.ts:95`), and compaction already fires pre-compaction hooks before summarization (`src/agents/pi-embedded-runner/compaction-hooks.ts:173`). The public docs also expose these surfaces directly as `command:new`, `command:reset`, and `session:compact:before` (`docs/automation/hooks.md:36`), while the plugin SDK exposes `before_reset` and `before_compaction` (`src/plugins/hook-types.ts:211`, `src/plugins/hook-types.ts:668`). There is test coverage for both paths in `src/gateway/server.sessions.gateway-server-sessions-a.test.ts:3008` and `src/agents/pi-embedded-runner/compact.hooks.test.ts:720`. This is also present in the latest shipped release, `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`).
