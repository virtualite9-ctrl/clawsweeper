## Summary

Keep open. Current `main` still treats manual scrolling as "near bottom" until the viewport moves at least 450px off the bottom, so auto-follow is not paused immediately on any manual scroll input. The linked open PR #69520 in the issue timeline also suggests this is recognized as unfinished work rather than already resolved.

## Evidence

- **Auto-follow still sticks while near bottom:** `scheduleChatScroll()` keeps following whenever `chatUserNearBottom` is true or the current distance from bottom is under the 450px threshold. That means a small manual scroll near the bottom does not disable follow mode.
  - file: [ui/src/ui/app-scroll.ts:57](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/ui/src/ui/app-scroll.ts#L57)

- **Manual scroll handler only updates threshold state:** `handleChatScroll()` derives `chatUserNearBottom` purely from `distanceFromBottom < NEAR_BOTTOM_THRESHOLD` and clears the new-messages indicator when back near bottom. There is no separate manual-interrupt flag for wheel/drag/touch input.
  - file: [ui/src/ui/app-scroll.ts:126](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/ui/src/ui/app-scroll.ts#L126)
