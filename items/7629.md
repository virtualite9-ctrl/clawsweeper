## Summary

Keep open. Current main still leaves the bundled `memory-lancedb` plugin on vector-only LanceDB retrieval, with no hybrid config, no FTS index setup, and no query-text path into LanceDB search. The BM25+vector implementation on main is for the separate `memory-core` builtin engine, and PR #7636 was closed unmerged.

## Best Possible Solution

Keep #7629 open as a focused bundled `memory-lancedb` parity enhancement. The best implementation belongs in `extensions/memory-lancedb`: add typed `hybrid` config plus manifest/docs entries, create or maintain an FTS index on `text`, pass original query text through recall/forget/CLI/hooks, use verified LanceDB hybrid search or a verified fallback/reranker when enabled, preserve vector-only fallback, and add targeted tests for config validation, FTS setup, exact-keyword recall, fallback behavior, and query text propagation.
