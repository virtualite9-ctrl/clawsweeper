## Summary

Keep #7629 open. Current main still has `memory-lancedb` on vector-only LanceDB retrieval with no FTS index, no hybrid/BM25 config, no query-text propagation into database search, and tests that assert `vectorSearch` behavior. The existing BM25+vector hybrid implementation is for the separate bundled `memory-core` engine, not LanceDB parity. PR #7636 attempted this exact feature but was closed unmerged, and the latest provided maintainer review comment also kept the issue open.

## Best Possible Solution

Keep #7629 open as a focused bundled `memory-lancedb` parity enhancement. The best path is in `extensions/memory-lancedb`: add typed `hybrid` config plus manifest/docs entries, create or maintain an FTS index on `text`, thread the original query text through recall/forget/CLI/hooks and duplicate checks, use verified LanceDB hybrid search or a tested fallback/reranker when enabled, preserve vector-only fallback, and add tests for config validation, FTS setup, exact-keyword recall, fallback behavior, and query-text propagation.
