## Summary

Keep open. Current `main` still does not wire coding tools into `resolveGatewayScopedTools`, so `/tools/invoke` does not implement the PR’s stated behavior despite docs describing HTTP deny-list gating for tools like `exec`. The PR also still needs follow-up on the current HTTP path because `handleToolsInvokeHttpRequest` already runs `runBeforeToolCallHook`, while `createOpenClawCodingTools` returns tools already wrapped with that hook and stores adjusted params that the HTTP path never consumes.

## Risks / Open Questions

- Closing this PR would drop a still-valid core gap: current docs describe deny-list-gated coding tools on `/tools/invoke`, but current `main` does not provide them.
