## Summary

Keep PR #69222 open. Current main only implements the newer `/api/web_search` endpoint and honors `https://ollama.com` as a configured base URL; it does not implement the PR’s compatibility chain, legacy `/api/experimental/web_search` fallback, local-404 cloud retry, or cloud/local bearer-token scoping. The linked bug #69132 remains the canonical open user problem, while duplicate PR #69178 is closed and unmerged.

## Best Possible Solution

Keep #69222 open as the active implementation path for #69132, but rebase it onto current main’s #71741 partial fix and adjust the auth resolver so provider-config local auth and env-sourced Ollama Cloud fallback can coexist. The merged version should retain regression tests for stable local, legacy experimental, cloud fallback, no-cloud-without-env-key, and token boundary behavior, plus matching docs/changelog updates.
