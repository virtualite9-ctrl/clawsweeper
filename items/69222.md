## Summary

Keep open. Current main partially overlaps the PR by using Ollama's `/api/web_search` path and honoring an explicit `https://ollama.com` base URL, but it still makes exactly one request to the configured host, has no local experimental fallback, has no 404-triggered Ollama Cloud fallback, and does not implement the PR's local/cloud auth scoping. The PR remains useful, though it should be retargeted against current main and address the existing mixed-auth review comment.

## Best Possible Solution

Keep #69222 open or have the author retarget it into a smaller follow-up on top of current main: preserve the shipped `/api/web_search` and explicit `https://ollama.com` behavior, add tested 404 fallback to the legacy local endpoint and env-keyed Ollama Cloud when appropriate, update docs/changelog, and fix the mixed provider-config plus `OLLAMA_API_KEY` auth-scoping issue called out in review. The open linked bug #69132 remains the right user-facing tracker for the remaining runtime failure.
