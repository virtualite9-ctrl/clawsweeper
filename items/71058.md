## Summary

Keep open. Current main still treats `channels.msteams` as a single Teams bot/app configuration, so the requested multi-bot/per-agent Teams identity support is not implemented. The existing routing/bindings system can select agents, but the Teams plugin only exposes the default account and starts one Bot Framework webhook/adapter from one credential set.

## Best Possible Solution

Implement this in the bundled Microsoft Teams plugin by adding a backward-compatible multi-account config shape, account resolution/listing, per-account credential and webhook dispatch support, account-scoped conversation/reference storage where needed, docs/tests, and guidance for binding Teams accounts or peers to agents. Keep the current single-object form as the default shorthand.
