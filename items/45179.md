## Summary

Current main has narrow temp/media cleanup and path-safety checks, but it does not implement the requested built-in temp artifact classification/provenance model, janitor classification, or broad runtime-critical /tmp guardrails. The issue is concrete, recent, and core-safety oriented rather than a closeable ClawHub/plugin idea.

## Best Possible Solution

Keep the issue open and treat it as a core safety/ops design item. A conservative path is to introduce a small typed temp artifact registry under the OpenClaw state dir, wire it into OpenClaw-owned temp writes and first-class tool surfaces, expose cleanup/reporting semantics in doctor/security or a janitor command, and add targeted doctor/service audits that warn or fail when active runtime/build/install paths resolve under /tmp, /private/tmp, or other non-durable temp roots. ClawHub can host optional SOPs, but the default provenance and runtime guardrails need core hooks or core implementation first.
