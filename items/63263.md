## Summary

Keep open. Current main still lacks the central infer CLI reliability fixes proposed by this PR: local model runs are not isolated with a per-run session key, image describe still does not accept or thread --prompt, structured web-search error payloads are returned instead of causing auto fallback, and openai-codex is still absent from image auto-priority. The PR remains a plausible implementation candidate, with review comments that should be addressed before landing.

## Best Possible Solution

Keep this PR open as an implementation candidate. The maintainer path is to revise the PR so web-search fallback is limited to auth/availability-style structured failures, make image --prompt reliably override request prompts through the media runner, keep the already-present agentDir behavior aligned, add openai-codex image autoPriority if that provider should participate in auto image fallback, and then run the targeted CLI, web-search, and media-understanding checks before review/landing.
