## Summary

Keep this PR open. Current main has superseded the original local `infer model run` session-key fix by moving local runs to the lean completion path, but the other user-visible reliability gaps are still present: `image describe` advertises `--prompt` without registering/threading it, structured web-search error payloads still return as successful provider results, and `openai-codex` still lacks image media-understanding auto fallback priority. The PR remains a plausible implementation candidate, but it needs rebase and review fixes before landing.

## Best Possible Solution

Keep this PR open as an implementation candidate. Rebase it onto current main, drop or adapt the obsolete local session-key change, register and thread `image describe --prompt`, make request-scoped image prompt overrides win over per-entry prompts, narrow structured web-search fallback to auth/availability failures, add `openai-codex` image `autoPriority` consistently in manifest and runtime metadata if it should participate in auto fallback, then rerun the targeted CLI, web-search, and media-understanding tests before maintainer review.

- The branch is stale against current main: local model runs already use the lean completion path, `agentDir` is already passed for image describe, the OpenAI Codex default model is now `gpt-5.5`, and manifest-backed media metadata now matters for fallback order.
