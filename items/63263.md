## Summary

Keep this PR open. Current main still lacks the central infer CLI reliability fixes: local model runs do not pass a per-run infer session key, image describe still has no registered or threaded `--prompt` option, structured web-search error payloads are returned as successful results instead of triggering fallback, and `openai-codex` still lacks image media-understanding `autoPriority`. The PR is still a plausible implementation candidate, but it needs rebase and review fixes before landing.

## Best Possible Solution

Keep this PR open as an implementation candidate. The maintainer path is to rebase it onto current main, preserve the existing `agentDir` behavior, add a per-run local infer `sessionKey`, register and thread `image describe --prompt`, make request-scoped prompt overrides win over per-entry prompts, narrow structured web-search fallback to availability/auth-style failures, add `openai-codex` image `autoPriority` consistently across manifest/runtime metadata if that provider should participate in auto fallback, and rerun the targeted CLI, web-search, and media-understanding tests before review or landing.
