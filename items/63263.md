## Summary

Keep this PR open. Current main still lacks the central infer CLI reliability fixes proposed here: local model runs are not isolated with a per-run infer session key, image describe still advertises but does not register/thread `--prompt`, structured web-search error payloads still stop fallback, and `openai-codex` is still excluded from image auto fallback. The PR is a plausible implementation candidate, but its existing review comments should be addressed before landing. The `agentDir` portion is already present on main, so that subpart is not a blocker either way.

## Best Possible Solution

Keep this PR open as an implementation candidate. The maintainer path is to revise it so web-search fallback is limited to availability/auth-style structured failures, make image `--prompt` reliably override request prompts through the media runner even when model entries define prompts, keep the already-present `agentDir` behavior aligned, add `openai-codex` image `autoPriority` if that provider should participate in auto image fallback, and then run the targeted CLI, web-search, and media-understanding checks before review/landing.

## Risks / Open Questions

- The proposed structured-error fallback is currently too broad: falling back on any top-level string `error` can hide validation failures and return results from a different provider that did not enforce the selected provider's constraints.
- The proposed image `--prompt` wiring needs to override per-model media entry prompts, not only the capability-level config prompt, or users with configured entry prompts will see the CLI flag ignored.
