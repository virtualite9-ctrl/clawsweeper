## Summary

Current `main` no longer misreports missing bundled entry files as plugin-root escapes. The bundled entry loader now includes the actual failure reason and underlying `ENOENT` detail, and there is a regression test for the missing-sidecar case.

## Evidence

- **Safe open path separates missing files from validation failures:** `safe-open-sync` classifies `ENOENT`, `ENOTDIR`, and `ELOOP` as `reason: "path"`, while boundary/shape violations stay under `validation`; missing files are no longer collapsed into the same bucket as path-escape validation failures.
  - file: [src/infra/safe-open-sync.ts:17](https://github.com/openclaw/openclaw/blob/8b76392e3e7934b1cdd6d461148263fd56141ec8/src/infra/safe-open-sync.ts#L17)

- **Bundled entry error message now reports reason and OS error detail:** The bundled channel entry loader formats failures as `bundled plugin entry "..." failed to open ... reason "...": <error>`, including importer path, resolved path, plugin root, and the underlying error message instead of always claiming an escape.
