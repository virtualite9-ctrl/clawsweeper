## Summary

Current main already implements the requested optional inbound debounce: `messages.inbound.debounceMs` and `messages.inbound.byChannel` configure global/per-channel windows, the shared debouncer batches same-key rapid messages with a sliding timer, and channel integrations key by account/conversation/sender before dispatching one combined turn. The docs and changelog cover the shipped behavior and its text-first media caveat.

## Best Possible Solution

Close this broad feature request as shipped. Keep the existing `messages.inbound` config and shared debouncer as the core solution; handle any remaining media/voice-note batching as narrower channel-specific follow-up work rather than reopening the generic debounce request.
