## Summary

Keep open. Current main still lacks Discord per-sender mention gating: config only exposes guild/channel-wide `requireMention` plus sender allowlists, and runtime resolves one mention-required boolean without sender-specific policy. The issue is concrete and a prior maintainer comment already kept it open after checking the same gap.

## Best Possible Solution

Keep tracking this issue. The likely implementation belongs in the bundled Discord plugin/config surface as an additive guild/channel option, with typed config, strict zod schema, generated config docs, Discord docs, and preflight tests. The resolver needs sender identity input so selected sender IDs require a mention while unlisted senders, including allowed bridge bots, can bypass mention gating when otherwise authorized.
