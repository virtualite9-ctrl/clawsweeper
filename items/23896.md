## Summary

Keep open. Current main still does not implement Discord per-sender mention gating; the config surface, strict schema, generated metadata, docs, and preflight runtime all expose only guild/channel-level `requireMention` plus sender allowlists.

## Best Possible Solution

Keep #23896 open. The best implementation is an additive Discord plugin/config change: add typed guild/channel `requireMentionFromIds`-style config, extend the strict zod schema and generated config metadata/docs, pass sender identity into the mention-requirement resolver, and add Discord preflight tests proving listed human senders must mention while unlisted allowed bridge bots can trigger without mentions.
