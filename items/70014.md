## Summary

Keep this issue open. Current main still prefers configured agent workspaces over the runtime/primary workspace, so a config with AGI sub-agent entries can process only those workspaces and skip the primary `/root/.openclaw/workspace` unless it is explicitly present in `agents.list`.

## Best Possible Solution

Keep the issue open and fix memory-core workspace enumeration so scheduled dreaming processes the runtime/primary workspace plus configured agent workspaces, deduped by normalized path. The likely path is to append `ctx.workspaceDir` or the memory manager status workspace at the call sites that have runtime context, then add regression coverage for a non-empty `agents.list` with subagents while the primary workspace is only available from the trigger/status context.
