## Summary

Keep #70014 open. Current main still has the reported enumeration gap: memory-core dreaming builds its sweep list from `agents.list`, only falls back to the default agent when that list is empty, and only uses the trigger context workspace when no configured workspace exists. That means a primary workspace passed by the cron/heartbeat context can still be skipped when configured AGI agent workspaces are present. The related #65653 session-cleanup issue is separate and does not supersede this workspace-enumeration bug.

## Best Possible Solution

Keep the issue open and fix memory-core's dreaming workspace enumeration so scheduled sweeps include the active primary/default workspace as well as configured agent workspaces, with deduping. Add regression coverage for a config with `agents.defaults.workspace` or cron `ctx.workspaceDir` plus AGI entries in `agents.list`, and clarify docs if maintainers intend different multi-agent semantics.
