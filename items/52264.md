## Summary

Keep open. Current main does not implement the PR's stated startup normalization for explicit state-dir overrides: `OPENCLAW_STATE_DIR` is still resolved from the current cwd each time it is read, and neither the gateway CLI path nor direct `startGatewayServer()` pins it at startup. The linked #52250 path appears partly addressed by gateway-bindable plugin registry reuse on main, but that does not supersede this narrower hardening PR.

## Best Possible Solution

Keep the PR open or ask the contributor to refresh it narrowly: add a shared startup env-normalization helper for explicit `OPENCLAW_STATE_DIR` and likely `OPENCLAW_CONFIG_PATH`, call it before any gateway CLI/direct-server config or plugin discovery, add focused path-pinning tests, and remove unrelated generated Plugin SDK baseline churn. If maintainers consider #52250 fully solved and do not want to support relative overrides, close manually with that product decision rather than automated cleanup.
