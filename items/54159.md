## Summary

Keep open. Current `main` still blocks in the shared overflow path by awaiting `reader.cancel()` in `src/media/read-response-with-limit.ts`, so the PR's core fix is not implemented on `main`. Discord now has an outer per-attachment timeout/fallback that prevents an indefinite wedge, but that is only a mitigation; it does not make oversized inbound media fail fast, and the focused regression test proposed in the PR is still absent.
