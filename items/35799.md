## Summary

Keep #35799 open. Current main partially mitigates heartbeat herding with an internal deterministic per-agent phase and cron-expression jobs already have deterministic top-of-hour staggering, but the requested public product surface is still missing: no prime cadence mode, no configurable/user-selectable heartbeat phase jitter, no resolved phase/status exposure, and cron `every` schedules remain fixed intervals without stagger/jitter support.

## Best Possible Solution

Keep the issue open and narrow it into a deliberate scheduler/API change: retain the existing internal heartbeat phase mitigation, add backward-compatible heartbeat controls for deterministic/user-selectable phase and optional prime cadence if maintainers want that behavior, extend cron `every` type/schema/normalizer/CLI/UI/status/docs/tests with explicit jitter/stagger semantics, and expose/log resolved schedule details for operators.
