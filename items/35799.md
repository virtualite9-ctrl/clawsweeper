## Summary

Keep #35799 open. Current main partially mitigates heartbeat herding with an internal deterministic per-agent phase and cron-expression jobs already support deterministic stagger, but the requested public scheduler surface is still missing: no prime cadence mode, no configurable/user-selectable heartbeat phase jitter, no resolved phase/status exposure, and cron `every` schedules remain fixed intervals that reject/prune stagger settings.

## Best Possible Solution

Keep the issue open and narrow it into a deliberate scheduler/API change: preserve the internal heartbeat stable phase, add backward-compatible heartbeat controls for deterministic/user-selectable phase and optional prime cadence if maintainers want that behavior, extend cron `every` type/schema/normalizer/CLI/UI/status/docs/tests with explicit jitter/stagger semantics, and expose/log resolved schedule details for operators.
