## Summary

Keep #35799 open. Current main has partial internal mitigation via stable heartbeat phasing and cron-expression top-of-hour staggering, but the requested public scheduler/API surface is still missing: no prime cadence mode, no heartbeat phase-jitter controls or resolved phase/status exposure, and cron `every` remains a fixed interval where stagger/exact options are rejected or pruned.

## Best Possible Solution

Keep the issue open and turn it into a deliberate scheduler/API change. Preserve the shipped internal stable heartbeat phase, then decide and implement backward-compatible public controls for heartbeat phase/jitter and optional prime cadence. Extend cron `every` types, gateway protocol, normalizer, CLI, UI/status output, docs, and tests with explicit jitter/stagger semantics, including resolved schedule observability for operators.
