## Summary

Keep this PR open. Current main still drops Discord upload MIME types for standard, component, and native interaction send paths, and WebP still falls through the shared optimizer to JPEG/PNG. The PR addresses a real user-visible bug, but it should remain under maintainer review because its WebP bypass changes shared media behavior, review has raised validation/privacy questions, and overlapping open PRs #41265 and #41714 should be consolidated rather than auto-closing useful work.

## Best Possible Solution

Keep this PR open for maintainer review or supersede it with a focused canonical fix for #41256. The best path is to preserve verified media.contentType in all relevant Discord Blob upload paths, decide the Discord/shared WebP and animated-image policy explicitly, add regression coverage for MIME preservation and frame preservation, address metadata and pixel-limit validation before bypassing optimization, keep WhatsApp docs/tests aligned with the chosen shared behavior, and consolidate overlapping PRs #41265 and #41714.
