## Summary

Keep open. Current main still has the reported Discord upload MIME bug and still routes WebP images through the shared optimizer, so PR #41419 is not obsolete or implemented. The PR contains useful work for open bug #41256, but its shared WebP bypass and MIME propagation have unresolved review/security concerns, so the best maintainer path is to revise or supersede it with a safer focused fix rather than auto-close it.

## Best Possible Solution

Keep the PR open for maintainer review or supersede it intentionally with a focused fix for #41256: preserve Discord upload MIME types only when the type is buffer-verified, preserve static/animated WebP without re-encoding only after magic-byte and decoded-dimension validation, update both standard and component Discord upload tests, and reconcile the shared-media impact on WhatsApp before merge.

## Risks / Open Questions

- Merging the PR as-is may broaden shared media behavior beyond Discord and bypass decoded pixel-limit validation for WebP unless a header-verified validation step is added.
- Forwarding `media.contentType` directly to Discord upload blobs should distinguish buffer-sniffed MIME from extension/header fallback to avoid declaring attacker-controlled content types.
- There are sibling open PR attempts in the provided context (#41265 and #41714), but none is shown as merged or canonical; auto-closing this one as a duplicate would risk discarding useful tests and discussion before a maintainer chooses the replacement.
