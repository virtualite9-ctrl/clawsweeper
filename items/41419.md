## Summary

Keep this PR open. Current main still drops Discord upload MIME types in both standard and component media sends, and WebP still flows through the shared optimizer to JPEG. The PR addresses a real open bug, but it should not be auto-landed or auto-closed because its WebP bypass changes shared media behavior and has unresolved validation/privacy questions raised in review.

## Best Possible Solution

Keep the PR open for maintainer review or supersede it with a focused canonical fix for #41256. The best path is to preserve Discord upload MIME types from buffer-verified content for both standard and component sends, decide the shared WebP/animated-image policy explicitly, add regression coverage for Discord MIME and WebP frame preservation, address metadata and pixel-limit validation before bypassing optimization, and consolidate the parallel open PRs.
