## Summary

Keep this PR open. Current main still does not export `stringEnum` or `optionalStringEnum` from the root `openclaw/plugin-sdk` entry, related reports show a real external-plugin compatibility problem, and the PR is a plausible implementation candidate. It is not ready to close as obsolete or implemented, but it likely needs contract/API-baseline updates and an explicit maintainer decision to promote these helpers onto the deprecated root SDK surface.

## Best Possible Solution

Keep this PR open as the implementation candidate. If maintainers want root-entry backward compatibility, update the PR to add the re-export plus the root SDK contract expectation, API baseline hash, and any small docs/changelog note needed for the promoted compatibility surface. If maintainers reject widening the root SDK, close it only after documenting the migration path and keeping #68279 as the canonical tracking item.

## Risks / Open Questions

- As submitted, the PR appears incomplete for the current contract suite because `src/plugins/contracts/plugin-sdk-index.test.ts` enforces an exact root runtime export list.
- Public SDK surface changes should also keep the generated Plugin SDK API baseline aligned; the provided review comment flags `docs/.generated/plugin-sdk-api-baseline.sha256` as missing from the PR.
- There is a product-boundary decision here: current docs and SDK boundary rules prefer focused subpaths, so maintainers should explicitly choose backward compatibility on the root entry versus directing affected external plugins to `openclaw/plugin-sdk/core` or another focused subpath.
