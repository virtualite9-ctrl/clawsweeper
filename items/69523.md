## Summary

Keep issue #69523 open. Current main has partial internal plumbing for auto-expanding tool cards, but the shipped Control UI still passes `autoExpandToolCalls: false`, and neither browser-local UI settings nor `openclaw.json` config expose the requested option. The existing tool-calls control only toggles visibility of tool calls/results, not expansion by default.

## Best Possible Solution

Keep the issue open and wire the existing `autoExpandToolCalls` path to a user-facing preference. If this becomes an `openclaw.json` option, add the config type, zod schema, labels/help/docs, generated schema checks, and Control UI consumption. If it is browser-local, add a `UiSettings` field, a Settings/chat-control toggle, persistence tests, and render tests proving new tool cards and tool result messages open by default while still allowing manual collapse.
