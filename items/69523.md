## Summary

Keep open. Current main has internal rendering support for an `autoExpandToolCalls` boolean, but the app never exposes it to users: the Control UI passes `autoExpandToolCalls: false`, browser-local settings only include hide/show tool calls, and the strict `ui` config schema does not accept the proposed `ui.chat.expandToolsByDefault` key.

## Best Possible Solution

Keep the issue open and implement a user-facing preference that drives the existing `autoExpandToolCalls` render path. The narrowest fix is to add a persisted browser-local Control UI setting and a chat/settings toggle; if maintainers want the exact requested `openclaw.json` behavior, add `ui.chat.expandToolsByDefault` to config types, Zod schema, generated docs/schema metadata, and bridge it into Control UI bootstrap defaults without replacing the existing manual collapse behavior.
