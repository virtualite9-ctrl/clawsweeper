## Summary

Keep #69523 open. Current main has partial internal support for auto-expanding tool disclosures, but the Control UI never exposes or enables it: `renderApp` passes `autoExpandToolCalls: false` unconditionally, browser-local settings only persist show/hide tool calls, and the shared OpenClaw config schema/type has no `ui.chat.expandToolsByDefault` or equivalent field.

## Best Possible Solution

Add a real user-facing preference and wire it into the existing auto-expand path. If maintainers want shared config as proposed, add `ui.chat.expandToolsByDefault` to `OpenClawConfig`, the Zod/schema help/docs surface, and have Control UI pass the resolved value into `renderChat`. If browser-local behavior is preferred, add a persisted `chatAutoExpandToolCalls` setting and Settings/toolbar control. Either path should add tests that verify tool-call cards and standalone tool-output rows open by default and remain manually collapsible.
