## Summary

Keep #71940 open. Current main still routes normal headed viewport screenshots through CDP `Page.captureScreenshot` with no Hyprland/grim fallback, no `browser.hyprlandCapture` config/schema support, and no Hyprland troubleshooting docs. The linked bug #64317 has concrete Omarchy/Hyprland reproduction and workaround evidence, and the PR discussion contains actionable review feedback rather than evidence that the work is obsolete.

## Best Possible Solution

Keep the PR open as the active implementation path for #64317. The best maintainer path is to rebase it onto current main, add `browser.hyprlandCapture` to the strict config schema/help/docs, address the Hyprland virtual-output cleanup and per-profile-headless review comments, keep the fallback opt-in and viewport-only, then validate with targeted route/config tests plus a real Hyprland grim capture smoke.
