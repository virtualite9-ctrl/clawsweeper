## Summary

Keep open. Current main reduces the hash-input side of the report and the checked-in bundle now looks deduped, but the generator itself is still install-layout-sensitive: the A2UI Rolldown config still prefers `ui/node_modules`, still lacks a `signal-polyfill` singleton alias, and there is no regression coverage proving bundle/hash stability with and without `ui/node_modules`. Open PR #68934 remains a relevant implementation candidate, not shipped proof.

## Best Possible Solution

Fix the A2UI generator path itself: make dependency resolution canonical across install layouts, or alias the full singleton dependency set including `signal-polyfill` and `signal-polyfill/`; then add a regression check that proves generated bundle/hash stability with and without `ui/node_modules`. Coordinate with open PR #68934 if maintainers want that patch as the vehicle.
