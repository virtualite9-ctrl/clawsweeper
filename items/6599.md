## Summary

Keep open. Current main still does not implement `/models test-fallback` or an equivalent `openclaw models test-fallback` command. Existing fallback runtime and `models status --probe` are related, but the probe validates configured auth/model reachability directly rather than intentionally skipping the primary model and exercising the configured fallback chain. The provided issue context also shows PR #43342 attempted this feature and was closed unmerged.

## Best Possible Solution

Continue tracking #6599 until maintainers implement or explicitly decline a first-class fallback-chain probe. A strong implementation would add `/models test-fallback` and likely `openclaw models test-fallback`, reuse the real fallback resolution/run path with an intentional primary skip, support first-fallback and all-fallback modes, add focused slash/CLI tests, and document how it differs from `models status --probe`.
