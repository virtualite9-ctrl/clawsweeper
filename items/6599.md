## Summary

Keep #6599 open. Current main still has model fallback runtime and live auth probing, but it does not implement `/models test-fallback` or an equivalent `openclaw models test-fallback` command that intentionally skips the primary model and exercises the configured fallback chain. The direct implementation attempt, PR #43342, is closed unmerged, and the latest issue discussion already records a maintainer/Codex keep-open review.

## Best Possible Solution

Keep #6599 open until maintainers implement or explicitly decline a first-class fallback-chain probe. A strong solution would add `/models test-fallback` and likely `openclaw models test-fallback`, reuse the real fallback resolution/run path with an intentional primary skip, support first-fallback and all-fallback modes, respect active agent/session auth context, add focused slash/CLI tests, and document how it differs from `models status --probe`.

## Risks / Open Questions

- A real fallback-chain probe would make live model calls, so UX should make token and rate-limit effects explicit.
- Implementation should avoid mutating session model overrides, auth-profile pinning, transcripts, or cooldown state while probing.
- `models status --probe` overlaps conceptually, so docs should clearly distinguish auth reachability from exercising the actual fallback chain.
