## Summary

Current `main` already downloads Telegram inbound media from `getFile().file_path` into inbound storage, surfaces download failures instead of failing silently, and includes focused regression coverage for the exact `file_path`/missing-file-path cases. The same implementation is present in the shipped `v2026.4.22` release commit.

## Evidence

- **Inbound media now saves downloaded Telegram files into inbound storage:** `resolveMedia()` requires `file.file_path` and passes it to `downloadAndSaveTelegramFile()`, which fetches Telegram `/file/bot.../<file_path>` content and persists it via `saveMediaBuffer(..., "inbound", ...)`.
