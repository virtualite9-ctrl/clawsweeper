## Summary

Current main has already changed the PDF path that matched this crash: local PDF extraction is plugin-owned, `@napi-rs/canvas`/Skia is no longer a required bundled runtime dependency, and missing canvas support now degrades the PDF image fallback instead of failing the turn. The change shipped by v2026.4.24.

## Best Possible Solution

Close this issue as implemented on main and keep the current plugin-owned, optional-canvas PDF extraction design. If a user can still reproduce on v2026.4.24 or newer with a fresh install, track it as a narrower follow-up around stale optional canvas runtime deps or isolating manually installed native PDF renderers outside the gateway process.
