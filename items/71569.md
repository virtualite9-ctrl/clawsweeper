## Summary

Keep open. Current main still has the Mattermost preview-streaming docs/config/runtime mismatch: docs advertise `channels.mattermost.streaming` with `off | partial | block | progress`, but Mattermost's strict schema, public type, generated metadata, doctor compatibility path, and normal inbound runtime still do not implement mode-aware preview streaming. The related #70253 overlaps on ignored streaming config but is broader and not a clear canonical replacement; #71565 was superseded by this focused report.

## Best Possible Solution

Implement the documented Mattermost preview streaming contract in the bundled Mattermost plugin: add `streaming` schema/types/generated metadata and doctor/compat handling for root and per-account configs; resolve `off | partial | block | progress` in the Mattermost runtime; skip draft preview creation for `off`; make `progress` deliver the final answer as a normal notifying post; and add focused config/runtime tests. Keep this issue as the focused tracker and reference #70253 only for the overlapping ignored-config symptom.
