## Summary

Keep #71569 open. Current main partially improves Mattermost preview behavior by editing/finalizing a single draft post when possible, but the central issue remains: docs advertise `channels.mattermost.streaming`, while the Mattermost config schema/types/generated metadata still omit that key, and the main inbound runtime starts draft preview streaming unconditionally instead of honoring `off | partial | block | progress`.

## Best Possible Solution

Keep the issue open and fix it in the Mattermost plugin: add `streaming` to Mattermost schema/types/generated metadata, resolve and honor `off`, `partial`, `block`, `progress`, and `streaming.preview.toolProgress` in the inbound monitor, preserve the current single-post in-place finalization where appropriate, and add regression tests proving the documented key is accepted and changes delivery behavior. Then decide explicitly how final-answer notifications should work for edited Mattermost preview posts.
