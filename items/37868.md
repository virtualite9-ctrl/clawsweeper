## Summary

Keep open. Current `main` does not implement the requested Feishu audio-message support for common audio formats like MP3/WAV/M4A; it still only treats Opus/Ogg as native Feishu `audio` messages and explicitly tests MP3 as a generic file attachment.

## Evidence

- **Current audio detection is still limited to Opus/Ogg:** `detectFileType()` only returns `"opus"` for `.opus` and `.ogg`; `.mp3`, `.wav`, `.m4a`, `.aac`, `.flac`, and `.wma` are not handled and therefore fall through to `"stream"`.
  - file: [extensions/feishu/src/media.ts:496](https://github.com/openclaw/openclaw/blob/55389f4be29359e2dd57ebaf548ba4a70331a445/extensions/feishu/src/media.ts#L496)
  - command: `nl -ba extensions/feishu/src/media.ts | sed -n '493,522p'`
  - sha: [55389f4be293](https://github.com/openclaw/openclaw/commit/55389f4be29359e2dd57ebaf548ba4a70331a445)
- **Outbound routing still maps only Opus/Ogg MIME/extensions to Feishu `audio`:** `resolveFeishuOutboundMediaKind()` sends `msgType: "audio"` only for `.opus`, `.ogg`, `audio/ogg`, or `audio/opus`. Other audio types are not recognized here.
  - file: [extensions/feishu/src/media.ts:524](https://github.com/openclaw/openclaw/blob/55389f4be29359e2dd57ebaf548ba4a70331a445/extensions/feishu/src/media.ts#L524)
  - command: `nl -ba extensions/feishu/src/media.ts | sed -n '524,568p'`
  - sha: [55389f4be293](https://github.com/openclaw/openclaw/commit/55389f4be29359e2dd57ebaf548ba4a70331a445)
- **Current test suite asserts MP3 is unsupported and sent as a file:** The test named `falls back to generic file for unsupported audio formats` uses `song.mp3` / `audio/mpeg` and expects upload `file_type: "stream"` plus outbound `msg_type: "file"`. That is the opposite of the issue request.
  - file: [extensions/feishu/src/media.test.ts:239](https://github.com/openclaw/openclaw/blob/55389f4be29359e2dd57ebaf548ba4a70331a445/extensions/feishu/src/media.test.ts#L239)
  - command: `nl -ba extensions/feishu/src/media.test.ts | sed -n '239,263p'`
  - sha: [55389f4be293](https://github.com/openclaw/openclaw/commit/55389f4be29359e2dd57ebaf548ba4a70331a445)
- **Docs still describe Feishu voice delivery in Opus terms:** The TTS docs say Feishu voice-message delivery is Opus-based and call out MP3 as a non-native format on some provider paths, which matches the current implementation focus rather than the requested broader audio support.
  - file: [docs/tools/tts.md:428](https://github.com/openclaw/openclaw/blob/55389f4be29359e2dd57ebaf548ba4a70331a445/docs/tools/tts.md#L428)
  - command: `nl -ba docs/tools/tts.md | sed -n '426,434p'`
