## Summary

Keep this issue open. Current main still supports inbound per-topic Telegram agent routing, explicit outbound topic targets, and current-context auto-threading, but it does not provide a config-level per-agent default outbound topic/thread binding for proactive sends.

## Best Possible Solution

Keep this issue open. If maintainers want the feature, implement it in the bundled Telegram plugin/config surface with a typed outbound-default topic binding, zod schema and generated config metadata/help/docs updates, and focused tests. Precedence should be explicit threadId or :topic: target first, current conversation/tool context next, then configured per-agent outbound default.
