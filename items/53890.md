## Summary

Keep open. Current `main` supports explicit Telegram topic-qualified outbound targets, but it does not implement the requested config-level default outbound topic/thread binding per agent. Topic config still only exposes inbound/session-routing fields such as `agentId`, and the outbound route resolver derives thread routing only from explicit target/thread inputs or current session state.

## Evidence

- **Topic config has no outbound flag; only `agentId` and related topic settings:** `TelegramTopicConfig` includes `agentId`, error policy, and topic behavior fields, but no outbound/default-thread flag. The only built-in default target nearby is channel-level `defaultTo`, documented as a CLI `--deliver` fallback, not a per-agent topic binding.
