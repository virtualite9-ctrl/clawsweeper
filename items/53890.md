## Summary

Keep open. Current main supports inbound Telegram per-topic agent routing and explicit/current-context outbound topic routing, but it does not implement a config-level default outbound topic/thread binding per agent.

## Best Possible Solution

Keep this open and implement a Telegram-owned outbound default topic feature if maintainers want it. The likely path is to add typed config/schema/docs/tests for per-agent outbound topic defaults, define precedence such as explicit `threadId` or topic-qualified target first, then current conversation context, then configured per-agent default, and wire that through message-tool/proactive send routing without changing generic core behavior beyond a reusable seam if needed.
