## Summary

Current main still does not implement a Telegram config-level default outbound topic/thread binding per agent. The existing Telegram topic config remains inbound-routing oriented, explicit outbound thread targets are supported, and current-context auto-threading is supported, but proactive sends cannot reverse-map an agent to a configured default topic without passing threadId or a :topic: target.

## Best Possible Solution

Keep this issue open. If maintainers want the feature, implement it in the bundled Telegram plugin/config surface with typed topic-level outbound-default metadata, zod schema and generated config metadata/help/docs updates, and focused tests. Runtime precedence should be explicit threadId or :topic: target first, current conversation/tool context next, then configured per-agent outbound default, with ambiguity handling for multiple defaults.
