## Summary

Current main still does not implement config-level default outbound Telegram topic/thread binding per agent. Telegram topic config remains inbound/routing-focused, an `outbound` topic field would be rejected by the strict schema, and outbound sends resolve topic/thread routing only from explicit params, topic-qualified targets, or current channel context. The prior ClawSweeper keep-open comment remains accurate against the current main SHA, and the related items do not supersede this Telegram-specific feature request.

## Best Possible Solution

Keep this issue open. If maintainers want the feature, implement it in the Telegram plugin/config surface with typed config, schema/help/generated docs, and focused tests. A conservative design should keep the logic Telegram-owned and define precedence as explicit threadId or `:topic:` target first, current conversation/tool context next, then configured per-agent outbound default.
