## Summary

Current `main` does not implement this docs clarification on the Control UI page. The newly added `docs/web/control-ui.md` still tells users to run only host-native `openclaw devices list/approve`, while other current docs already show the Docker Compose equivalents for the same pairing flow. This PR still addresses a real docs inconsistency and likely just needs a rebase onto the newer page added on `main`.

## Evidence

It does not include the Docker Compose equivalents proposed by the PR.
