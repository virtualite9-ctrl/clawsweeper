## Summary

Keep open. Current main still has no native warm-reuse path for `sessions_spawn`: native subagent spawning unconditionally creates a fresh UUID-backed child session key, the tool/config schemas expose no native reuse key or pool selector, and docs/tests continue to scope `resumeSessionId` to ACP rather than OpenClaw-native subagents. The related merged PRs #64468 and #72331 addressed adjacent cache/resume-schema behavior, not this issue’s requested native subagent reuse contract.

## Best Possible Solution

Keep this issue open. The maintainer path is to design an explicit native subagent reuse contract around `src/agents/subagent-spawn.ts`, `src/agents/tools/sessions-spawn-tool.ts`, `src/config/zod-schema.agent-defaults.ts`, and `docs/tools/subagents.md`, choosing opt-in reuse versus automatic pooling, documenting isolation/history tradeoffs, handling concurrency and archive cleanup, and adding focused native `runtime="subagent"` regression tests. Treat #64468 and #72331 as related cache/runtime cleanup, not as canonical resolution.

## Risks / Open Questions

- A native warm-reuse implementation could mix unrelated child history unless reuse is explicit, scoped, and documented.
- Automatic pooling would need allocation/locking semantics so concurrent spawns do not reuse an active child session unsafely.
- Archive cleanup, follow-up routing, thread/session mode, and context-engine preparation need a defined contract for reused native child sessions.
