## Summary

The issue should close as duplicate/superseded by the newer open #70991. Current main has partial abort/task-cancel plumbing, but the remaining actionable Control UI/Gateway reconnect-stop gap is still present and is already tracked there.

## Best Possible Solution

Close #46376 as a duplicate/superseded report and keep #70991 as the canonical open issue. The best fix belongs in core Control UI/Gateway behavior: recover or derive abortable active runs after reconnect, keep a session-scoped Stop control available, make /stop feedback visible, and add regression coverage for gateway restart/disconnect during a running task.
