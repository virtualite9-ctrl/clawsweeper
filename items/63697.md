## Summary

Keep this PR open. Current main still handles native `/new`, `/reset`, `/clear`, and `/compact` locally in the shared ChatViewModel before reaching the generic `chat.send` transport path, while this open PR is a plausible but broad implementation candidate with unresolved functional and approval-scope review concerns.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to split or re-scope it into a focused native Chat UI change that forwards reset/compact slash commands through `chat.send`, preserves reset/compact transcript semantics with Swift regression tests, and separately reviews the broader UI/model-picker/exec-approval changes. Coordinate APNs-related client expectations with #67493 before shipping the full offline reply experience.
