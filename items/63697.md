## Summary

Keep this PR open. Current main still locally intercepts native chat `/new`, `/reset`, `/clear`, and `/compact` before the iOS/shared Chat UI reaches the generic `chat.send` transport path, while this contributor PR remains an implementation candidate. It should not be closed conservatively because the central behavior gap still exists, the PR is paired with open same-author APNs companion work, and the PR also contains broad UI/approval-scope changes that need maintainer review rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to split or re-scope it into a focused shared native Chat UI change that forwards reset/compact slash commands through `chat.send`, preserves transcript semantics with Swift regression tests, and reviews the broader UI/model-picker/exec-approval changes separately. Coordinate offline APNs client expectations with #67493 before shipping that full experience.
