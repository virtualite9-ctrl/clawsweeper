## Summary

Current main still intercepts `/new`, `/reset`, `/clear`, and `/compact` in the shared native chat view model before `chat.send`, and the existing Swift tests assert that this local path calls `sessions.reset`/`sessions.compact` with no sent chat run. The Gateway side already has a `chat.send` slash-command path, so the PR’s original product problem remains real rather than obsolete. The related APNs PRs are companion/split server work, not a replacement for this iOS client routing change. Keep open, but the PR needs maintainer review and likely narrowing/rebase because it grew well beyond slash routing and has unresolved correctness review comments.

## Best Possible Solution

Keep the PR open. Ask for a focused rebase/split: first land the minimal iOS/shared chat change that routes standalone slash commands through Gateway `chat.send` while preserving correct reset/refresh semantics and adding Swift regression tests, then handle broader chat UI polish and companion APNs work separately. Address the outstanding P1/P2 review comments before merge; #67493/#63800 should remain the separate Gateway APNs track.
