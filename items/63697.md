## Summary

Current main still has the native Chat UI behavior this PR targets: exact `/new`, `/reset`, `/clear`, and `/compact` inputs are intercepted in `OpenClawChatViewModel` and handled through local `sessions.reset` / `sessions.compact` calls before the generic `chat.send` path runs. The PR remains a plausible implementation candidate, but it is broad, has unresolved review concerns, and has an open same-author companion PR for APNs work, so it should stay open for maintainer review rather than conservative cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to either re-scope it to a focused native Chat UI change that forwards slash commands through `chat.send` while preserving reset/compact transcript semantics with regression tests, or split the broader UI/model-picker/exec-approval work into smaller PRs coordinated with #67493. Before merge, resolve the outstanding review comments and explicitly verify the operator approval scope behavior does not regress.
