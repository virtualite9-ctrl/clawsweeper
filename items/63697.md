## Summary

Current main still has the native Chat UI behavior this PR is trying to change: `/new`, `/reset`, `/clear`, and `/compact` are intercepted in `OpenClawChatViewModel` and routed to local session reset/compact methods before the generic `chat.send` path can run. The PR remains a plausible implementation candidate, but it has expanded beyond slash-command routing and has unresolved review concerns plus an open same-author companion PR for offline APNs alerts.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to either land a focused native Chat UI change that forwards slash commands through `chat.send` while preserving reset/compact history semantics with regression tests, or split the broader UI/APNs/model-picker work into smaller PRs coordinated with #67493.
