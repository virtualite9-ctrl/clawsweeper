## Summary

Keep this PR open. Current main still lacks the proposed typing-TTL expiry feedback hook, so it is not implemented, and the related open typing/timeout items are adjacent rather than a full superseding replacement. The PR is also not landable as-is because its supplied diff mixes the intended auto-reply change with unrelated macOS remote-connection changes and a Tlon dependency-source change that need separate review.

## Best Possible Solution

Keep this PR open as a reference implementation, but do not land it as-is. The maintainable path is a fresh, narrow auto-reply PR against current main that implements the TTL-expiry feedback contract or reconciles it with #40760 and #72447, drops the unrelated macOS and Tlon dependency changes, handles both synchronous throws and async rejections from custom handlers, and adds focused tests for TTL expiry, suppressed typing, routed replies, and final-reply ordering.
