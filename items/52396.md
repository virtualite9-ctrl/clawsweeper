## Summary

Current `main` still leaves the reported gap in place. The shared Discord retry runner only retries `RateLimitError`/429, and that runner is what outbound Discord send paths use for DM creation and message/media/forum-thread sends. The broader transient-error retry logic mentioned in the issue exists only in other Discord-specific paths, not in the main outbound client runner.

## Evidence

- **Discord retry runner still only retries 429:** `createDiscordRetryRunner()` passes `shouldRetry: (err) => err instanceof RateLimitError` and no broader transient matcher.
  - file: [extensions/discord/src/retry.ts:24](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/extensions/discord/src/retry.ts#L24)
