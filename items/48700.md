## Summary

Keep #48700 open. Current main has WebChat image attachment support, but not the requested Excel/Word/PDF/PowerPoint upload-and-analysis workflow. The UI accepts and previews only image MIME types, the client serializes attachments as image payloads, and the gateway parser drops non-image payloads instead of routing office documents to an analysis path.

## Best Possible Solution

Keep the issue open as a core Control UI/Gateway feature request. The best implementation path is to design a document attachment contract for WebChat, extend the composer accept/filter/preview/error states, route PDF uploads through existing media refs and the PDF tool where appropriate, add a real extraction or tool-routing story for Word/Excel/PowerPoint, and cover the browser client plus gateway parser with focused tests.
