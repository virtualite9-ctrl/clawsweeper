## Summary

Current main does not implement the requested `memorySearch.local.maxThreads` control. The local embedding config, schema, docs, and resolver expose `modelPath`, `modelCacheDir`, and `contextSize`, but no thread-count setting, and the node-llama-cpp integration still calls `getLlama({ logLevel: LlamaLogLevel.error })` without forwarding a `maxThreads` value.

## Best Possible Solution

Keep this issue open. The best implementation is to add a validated positive integer `agents.defaults.memorySearch.local.maxThreads` and per-agent override, thread it through config types, Zod schema, generated schema/help, resolved memory-search config, memory embedding provider SDK types, and `createLocalEmbeddingProvider`, then pass it to `getLlama` and add focused tests/docs for default omission and configured forwarding.
