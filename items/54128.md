## Summary

Keep open. Current `main` does not implement a `memorySearch.local.maxThreads` setting or pass any thread-count option into `node-llama-cpp` for local embeddings, so the requested capability is still missing.

## Evidence

- **Local embedding runtime does not pass thread configuration:** `createLocalEmbeddingProvider()` reads `local.modelPath`, `local.modelCacheDir`, and `local.contextSize`, then calls `getLlama({ logLevel: LlamaLogLevel.error })` with no `maxThreads` or similar option.
  - file: [src/memory-host-sdk/host/embeddings.ts:26](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/memory-host-sdk/host/embeddings.ts#L26)

  - file: [src/config/types.tools.ts:390](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/config/types.tools.ts#L390)
