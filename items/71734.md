## Summary

Keep #71734 open. Current main has the adjacent normal `before_tool_call` plugin hook, basic plugin tool catalog fields, and static core/UI tool display maps, but it does not implement the requested trusted pre-plugin policy registration stage, external-plugin rejection for privileged policy, first-class tool display/safety metadata registry, or a core-tool decoration rule. The linked #71731 context is an open docs/RFC PR and explicitly does not implement the SDK/runtime surface.

## Best Possible Solution

Keep #71734 open and resolve it through a focused core SDK/protocol implementation after the RFC decision: add a bundled/trusted pre-plugin tool-policy registration stage, reject privileged registration from external plugins, add first-class tool display/category/safety metadata projected through catalog/effective inventory/UI lookup, define or explicitly reject core-tool decoration, and cover ordering, override, metadata projection, and disablement cleanup with fixture tests.
