## Summary

Keep #69754 open. Current main still does not give Feishu-specific fallback guidance for built-in approval UX when approval buttons are not rendered. Feishu registers only same-chat approval authorization, while the approval forwarder falls back to generic text plus generic interactive metadata that Feishu's normal outbound adapter does not render as approval buttons.

## Best Possible Solution

Keep #69754 open. The best fix is either to add a Feishu approval capability/render adapter that turns built-in approval pending/resolved payloads into Feishu cards when possible, or to add explicit Feishu-aware fallback copy when generic interactive metadata degrades to text. The docs should also explain Feishu approval behavior separately from generic message-presentation and ACP cards.
