## Summary

Keep #69754 open. Current main has clearer generic `/approve` fallback text and shared interactive approval metadata, but Feishu still does not have a native approval/render adapter for built-in approval buttons, its `capabilities` config is generic and `inlineButtons` is not consumed by the Feishu approval path, and the Feishu docs do not yet explain approval fallback behavior.

## Best Possible Solution

Keep the issue open and implement either Feishu-native approval rendering or explicit channel-aware fallback copy. A conservative fix would detect when the approval target is Feishu and no native approval renderer is available, then say that Feishu approval buttons are unavailable on this surface and that the user should reply in the same Feishu chat with the exact `/approve <id> ...` command; update `docs/channels/feishu.md` and the approvals docs to describe the text-command approval path.
