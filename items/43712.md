## Summary

Current main still does not clearly fix the reported in-place source update failure. The git updater still builds inside the existing checkout after a normal tracked-file clean check, while the implicated A2UI lit renderer output remains ignored and `scripts/bundle-a2ui.mjs` still invokes the lit TypeScript build without first cleaning or rejecting stale `vendor/a2ui/renderers/lit/dist` output. The earlier Telegram SDK-helper mismatch appears fixed, but the central stale/generated A2UI artifact collision remains plausible and unhandled.

## Best Possible Solution

Keep this issue open and fix the source-update/A2UI edge case directly. The best implementation path is to clean or explicitly reject stale `vendor/a2ui/renderers/lit/dist` before `canvas:a2ui:bundle` during in-place source updates, add regression coverage for ignored generated A2UI output causing TS5055-style collisions, and document a safe recovery command for dirty source checkouts.
