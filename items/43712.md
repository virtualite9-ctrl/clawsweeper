## Summary

Keep this issue open. Current main still has the same plausible in-place git update failure mode: a git checkout under a package root runs the git updater, the clean check ignores generated files, the stable git path checks out and builds in the same tree, and A2UI bundling still compiles `vendor/a2ui/renderers/lit` without cleaning the ignored `dist` output first. The secondary Telegram SDK-helper mismatch appears fixed, but the central stale A2UI artifact/update-path issue remains unhandled.

## Best Possible Solution

Keep this issue open and fix the source-update/A2UI edge case directly. The best implementation path is to clean or explicitly reject stale `vendor/a2ui/renderers/lit/dist` before `canvas:a2ui:bundle` during in-place git updates, add regression coverage for ignored generated A2UI output causing TS5055-style collisions, and add a safe recovery note to the public updating docs for source checkouts.
