## Summary

Keep this issue open. Current main still has an in-place git update path that can build inside the existing checkout after a tracked-file-only clean check, while the reported ignored A2UI renderer output remains ignored and the A2UI bundler still compiles without cleaning that output. The secondary Telegram SDK-helper mismatch appears fixed, but the central source-update/A2UI artifact collision is still plausible and unhandled.

## Best Possible Solution

Keep this issue open and fix the source-update/A2UI edge case directly. The best implementation path is to clean or explicitly reject stale `vendor/a2ui/renderers/lit/dist` before `canvas:a2ui:bundle` during in-place git updates, add regression coverage for ignored generated A2UI output causing TS5055-style collisions, and add a safe recovery note to the public updating docs for source checkouts.
