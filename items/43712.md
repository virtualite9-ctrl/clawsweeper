## Summary

Keep open. Current main has update hardening that likely fixes adjacent problems, and the Telegram runtime mismatch symptom is now covered by the plugin SDK export, but the central source-checkout/A2UI build failure is not clearly fixed. A git checkout living at `/usr/lib/node_modules/openclaw` is still treated as an in-place git update, ignored `vendor/a2ui` build artifacts can survive the clean-worktree check, and `canvas:a2ui:bundle` still runs `tsgo` without clearing the implicated A2UI `dist` directory first.

## Best Possible Solution

Keep this open and fix the source-update/A2UI edge case directly. The likely maintainer path is to clean or explicitly validate `vendor/a2ui/renderers/lit/dist` before `canvas:a2ui:bundle` during source updates, add a regression test that simulates ignored stale A2UI output causing TS5055-style collisions, and document the safe recovery command for source checkouts if the updater aborts.
