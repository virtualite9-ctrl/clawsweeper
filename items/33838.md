## Summary

Keep #33838 open. Current main still models node commands as string names plus unknown params/results, with no protocol, pairing, registry, discovery, or validation path for node-declared per-command input/output schemas. The provided issue discussion and prior ClawSweeper comment describe a real remaining gateway protocol/API gap, and the related items do not supersede it.

## Best Possible Solution

Keep the issue open for an additive gateway protocol/API design. A good implementation would retain existing commands: string[] compatibility while adding optional per-command metadata, carry it through connect, node pairing, persistence, node.list/node.describe discovery, and node-host/plugin registration surfaces, then validate node.invoke params and/or results according to an explicit maintainer decision.
