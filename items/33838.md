## Summary

Keep #33838 open. Current main still has no mechanism for nodes to declare per-command input/output schemas: the connect, pairing, storage, runtime registry, discovery, invoke, and plugin node-host surfaces all model commands as string names plus unknown params/results. The prior issue discussion and related ClawSweeper context are still accurate; adjacent node-command items do not supersede this protocol/API gap.

## Best Possible Solution

Keep the issue open for an additive gateway protocol/API design. The best implementation would preserve existing `commands: string[]` compatibility while adding optional per-command metadata such as input/output JSON Schema, carrying it through connect, pairing approval, storage, `node.list`/`node.describe` discovery, and plugin node-host registration, then validating `node.invoke` params/results where maintainers decide that is safe and compatible.
