## Summary

Keep this PR open. Current main still lacks `api.registerProviderRuntimeAuthOverride` or an equivalent external plugin seam for a separate trusted plugin to provide per-call auth, base URL, or headers for existing built-in providers without replacing the provider implementation. The PR is a plausible core/plugin-SDK extension rather than ClawHub-only work, but it needs maintainer API/security review before merge.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land a revised additive core/plugin-SDK seam, or an explicitly chosen alternate seam, that lets trusted plugins provide per-call credentials, base URL, and request headers for existing providers without owning those providers. Before merge, resolve the remaining review comments and keep focused coverage for ordering, throw logging, mode validation, reentrancy, alias normalization, modelId propagation, availability semantics, transport override sanitization, and side-path fallback behavior.
