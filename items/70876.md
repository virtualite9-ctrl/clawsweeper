## Summary

Current main still has the /tools inventory mismatch this PR is trying to fix: the command path only renders the core/plugin/channel effective inventory, while the session run path separately materializes configured MCP tools and appends them after final policy filtering. The PR is not obsolete, but it should not be closed as implemented or superseded from the available evidence; it needs a focused revision because the provided PR context shows unrelated installer changes and review-noted defects mixed into the branch.

## Best Possible Solution

Keep this PR open for maintainer or contributor follow-up, but require a focused revision. The best path is to drop the unrelated install-package-dir changes, add MCP tools to /tools and /tools verbose through the existing session MCP runtime path, represent them as an MCP inventory source/group, preserve reservedToolNames collision behavior, apply the same final effective tool policy used by the run path, and add compact plus verbose regression tests. If maintainers prefer cleaner history, supersede this with a fresh narrow PR linked to #70874's user-visible bug.
