## Summary

Keep open. Current `main` still implements the exact broad base64 heuristic reported in the issue, and the tests still encode that behavior as expected for a non-credential key. The false-positive warning remains reproducible from code inspection. One part of the report is overstated: strict mode blocks by key allowlist, not because of this value heuristic.
