## Summary

Current main still has the reported broad value-only sandbox env-var heuristic, so the issue remains valid. The implementation warns on any 80+ character base64-character-class value, including long alphanumeric, hex-like, and JWT-segment-like strings, and existing tests still encode that behavior for a non-credential key.

## Best Possible Solution

Keep this issue open. Tighten src/agents/sandbox/sanitize-env-vars.ts by adding structural base64 checks and/or passing the env-var key into validateEnvVarValue so value warnings are limited to credential-like names, then add focused regression tests for long alphanumeric, hex-like, and JWT-segment-like legitimate values while preserving existing secret-blocking behavior.
