## Summary

Current main still does not expose an agent-facing `feishu_bitable_delete_record` tool. The bundled Feishu Bitable surface registers metadata, list/get/create/update record, create app, and create field tools, while record deletion is only used internally when cleaning up newly created Bitable placeholders. The related open PRs #40697 and #41368 are implementation candidates, not shipped fixes, and closed duplicate #61330 points back to this issue as the canonical tracker.

## Best Possible Solution

Keep this issue open as the canonical tracker. The likely fix belongs in `extensions/feishu/src/bitable.ts`: add a typed `deleteRecord` helper around `client.bitable.appTableRecord.delete`, add a schema for `app_token`, `table_id`, `record_id`, and optional `accountId`, register `feishu_bitable_delete_record` through the existing Bitable tool helper so contextual `agentAccountId` routing is preserved, return a structured success result, and add focused Feishu plugin tests for registration, routing, SDK call parameters, and error handling. Review or consolidate #40697 and #41368 rather than closing this issue before a fix merges.
