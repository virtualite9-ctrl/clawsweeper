## Summary

Keep #31583 open. Current main still blocks the reported `GOG_KEYRING_PASSWORD` case: `skills.entries.<skill>.env` keys are collected as pending overrides, but sensitive-looking keys are only re-allowed when declared by skill metadata, and the bundled `gog` skill still does not declare `GOG_KEYRING_PASSWORD`. Host exec inherits `process.env` after skill env application, so the remaining blocker is still the skill env sanitizer, not the exec spawn path.

## Best Possible Solution

Keep #31583 open until #50960 or an equivalent maintainer-reviewed fix lands. The likely fix belongs in `src/agents/skills/env-overrides.ts`: explicitly allow user-configured `skills.entries.<skill>.env` keys that match sensitive patterns, such as `GOG_KEYRING_PASSWORD`, while preserving the hard block for dangerous host/runtime override variables. Add regression coverage for undeclared configured `_PASSWORD`/`_TOKEN` keys and verify the host `exec` child-process path inherits the applied env.
