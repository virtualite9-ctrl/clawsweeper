## Summary

Keep open. Current `main` still has the exact bug this PR fixes: `allowBundled: []` is normalized away and then treated as unrestricted, while the docs describe `allowBundled` as an allowlist that restricts bundled skills when set. The PR is a focused bug fix with a directly relevant regression test, and the related ClawSweeper title matches are unrelated false positives.

## Best Possible Solution

Keep the PR open and either land or reimplement the same narrow fix: preserve explicit empty `skills.allowBundled` arrays, make `undefined` mean unrestricted and `[]` mean block all bundled skills, keep workspace/managed skills unaffected, and add the regression test from the PR before running the changed skills/agent validation lane.
