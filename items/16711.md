## Summary

Keep #16711 open. Current main still has only the capped inline Control UI composer and does not implement the requested Expand button, large draft modal, Apply/Cancel/Send flow, or modal keyboard behavior. The provided issue discussion already contains a recent Codex/maintainer review keeping this open, and the related items are adjacent composer/modal work rather than a canonical replacement.

## Best Possible Solution

Keep this open for first-class Control UI work. A good implementation should add an Expand icon near the composer actions, a large editor modal with independent draft state, Close/Cancel, Apply, and Send semantics, Enter-as-newline plus Ctrl/Cmd+Enter-to-send behavior, and focused UI tests covering draft ownership, queue/send behavior, attachments, and existing dialog patterns.
