## Summary

Keep #16711 open. Current main still has only the capped inline Control UI chat textarea plus the existing New session, Export, Send/Queue, and Stop controls; there is no Expand button, no large composer modal, and no modal keyboard path where Enter inserts a newline while Ctrl/Cmd+Enter sends. The related items in the provided context are adjacent composer/session/modal work, not a canonical tracker for this long-prompt editor request, and the existing maintainer-side Codex review comment explicitly kept this issue open.

## Best Possible Solution

Keep this open for first-class Control UI work. The likely implementation belongs around `ui/src/ui/views/chat.ts`, `ui/src/ui/chat/run-controls.ts`, `ui/src/ui/app-render.ts`, and `ui/src/styles/chat/layout.css`: add an Expand control near the composer actions, modal draft state with Close/Cancel, Apply, and Send semantics, modal keyboard behavior where Enter inserts a newline and Ctrl/Cmd+Enter sends, plus focused UI tests. Coordinate with adjacent composer/session modal issues, but do not close #16711 unless a canonical issue specifically tracks the expanded long-prompt composer modal.
