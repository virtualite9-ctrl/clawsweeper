## Summary

Keep this PR open. Current main still scopes Control UI session tokens by the exact parsed gateway host, so `localhost`, `127.0.0.1`, and `[::1]` remain fragmented. The PR addresses a plausible current UX/auth gap reflected in the related localhost token-missing reports, but the proposed implementation needs revision before merge because its `host.startsWith("127.")` check can also match DNS hostnames such as `127.example.com` or `127.0.0.1.nip.io`, causing non-loopback gateway token scopes to collide with local loopback scopes.

## Best Possible Solution

Keep the PR open, but request a small revision before merge: normalize only exact `localhost` and numeric loopback IP literals, preferably via the existing shared IP helper rather than a string prefix; preserve port and pathname isolation; add negative tests for `127.example.com` and `127.0.0.1.nip.io`; keep the added IPv4/IPv6 loopback alias tests; then run the appropriate UI changed gate before landing.
