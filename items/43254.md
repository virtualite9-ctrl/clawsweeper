## Summary

Keep this PR open. Current main still scopes Control UI session tokens by the exact normalized gateway URL, so `localhost`, `127.0.0.1`, and `[::1]` aliases do not share a token scope. The PR targets the correct Control UI storage surface and the discussion shows the author addressed the first bot review, but the current PR diff still has a security/correctness issue: `host.startsWith("127.")` treats ordinary DNS names such as `127.example.com` and `127.0.0.1.nip.io` as loopback token scopes. The branch also appears stale against the current `UiSettings` shape.

## Best Possible Solution

Keep this PR open and request a focused revision. Normalize only `localhost` and actual numeric loopback IP literals, preserve protocol/port/pathname isolation, add negative tests for `127.example.com` and `127.0.0.1.nip.io`, update the new tests for the current `UiSettings` shape, then run the appropriate UI changed gate before landing.
