## Summary

Keep this PR open. Current main still scopes Control UI session tokens by the exact normalized gateway URL, so localhost, 127.0.0.1, and [::1] aliases do not share the same token scope. The PR targets the right storage surface and adds useful regression tests, but it should not land as-is because its `host.startsWith("127.")` check also treats ordinary DNS names like `127.example.com` or `127.0.0.1.nip.io` as loopback token scopes.

## Best Possible Solution

Keep this PR open and request a focused revision: normalize only `localhost` and actual numeric loopback IP literals, preferably using the shared canonical IP helper if suitable for the UI bundle or an equivalent UI-local parser; preserve protocol, port, and pathname isolation; add negative tests for `127.example.com` and `127.0.0.1.nip.io`; update the added tests for the current `UiSettings` shape; then run the appropriate UI changed gate before landing.
