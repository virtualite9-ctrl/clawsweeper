## Summary

Keep this issue open. Current main still does not implement the requested Mattermost threading reliability work: DMs remain explicitly non-threaded, direct-message history is disabled, the WebSocket client lacks ping/pong keepalive configuration, thread history is only in memory with no `/posts/{threadRootId}/thread` backfill path, and mention-only messages are still dropped after normalization. Related #51104 and #53186 track parts of the problem but do not supersede the broader request.

## Best Possible Solution

Keep this open and implement it as Mattermost-plugin-owned reliability work under `extensions/mattermost`. Coordinate #51104 and #53186, preserve the current flat-DM default, add an explicit DM/per-chat-type threading option, add configurable WebSocket ping/pong keepalive, backfill empty thread-scoped history from Mattermost safely, handle mention-only invocations, and cover the behavior with Mattermost plugin tests plus config/docs updates.

## Risks / Open Questions

- Changing DM threading semantics can regress users relying on the documented flat-DM behavior; preserve the current default and add an explicit DM/per-chat-type opt-in if implemented.
- Thread-history backfill should respect Mattermost sender authorization and context-visibility policy before injecting fetched thread content into model context.
- WebSocket ping/pong support needs cleanup on close/error/abort and compatibility handling for test fakes or ws-compatible clients that may not expose `ping`/`pong`.
