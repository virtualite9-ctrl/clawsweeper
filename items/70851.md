## Summary

Keep #70851 open. Current main still lacks an explicit internal-session visibility contract, and the chat picker only filters global/unknown and cron rows before adding the remaining sessions. The PR is aimed at a real remaining UI problem, but its current diff is an interim heuristic with review-noted false-positive and coverage concerns, so it should not be closed as implemented or superseded.

## Best Possible Solution

Keep the PR open and convert it into the cleaner product fix: add an explicit gateway/session-row internal visibility classification for subagent, ACP, and node-backed execution sessions; add a `sessions.list` option so normal pickers hide internals by default while the Sessions/admin view can request them; update UI types/event merging; then filter the chat picker using that contract with tests for hidden internals, the active-session exception, ACP/node-backed rows, and any `spawnDepth` fallback.
