## Summary

Keep open. Current main still exposes internal subagent/ACP-style sessions through the chat session picker, and PR #70851 is not obsolete or implemented elsewhere. The remaining best fix is the one described in the PR body: classify internal/runtime sessions at the gateway/session-source contract and have the normal chat picker hide that explicit classification while debug/admin session views can still request internals.

## Best Possible Solution

Keep this PR open or replace it with a narrower canonical fix that adds an explicit internal/runtime visibility classification to gateway session rows, updates the chat picker to hide that classification by default while preserving the active internal session, keeps Sessions/admin/debug views able to show internals explicitly, and covers subagent, ACP, node-backed, active-session, and false-positive key cases in focused tests.
