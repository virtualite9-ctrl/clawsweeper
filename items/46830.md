## Summary

Keep open. Current main partially implements the requested behavior for Microsoft Teams, but the shared reply dispatcher and other affected channel auto-reply paths still only log delivery failures instead of injecting a delivery-failure event into the agent context.

## Best Possible Solution

Keep the issue open and implement a shared delivery-failure contract for auto-reply delivery: when final/block/tool reply delivery fails, enqueue a compact system event containing channel, target/thread, error class, retry hint, and message preview, then adapt Feishu, Telegram, Discord, MSTeams, and other bundled channels to use the shared helper with focused regression tests. MSTeams can remain the reference implementation, but the broad feature should not close until the generic path or the main bundled channels are covered.
