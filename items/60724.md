## Summary

Current `main` already threads the actually selected model through fallback-aware agent runtime paths, persists that runtime model on the session, and exposes selected-vs-active model state in `/status` and related agent-facing surfaces.

## Evidence

- **Fallback selection callback uses actual dispatched model:** `runWithModelFallback` invokes `onModelSelected` only after the active provider/model candidate is chosen, with an explicit comment that this includes fallback and exposes the actual model.
