## Summary

Keep this PR open. Current main has picked up part of the thread/auth reply behavior, but the central group-scope Telegram command registration/cleanup is still absent, and the plugin-command "Command not found." fallback still does not preserve thread params. The PR branch is stale against the current Telegram plugin layout, so the best path is a focused rebase or successor patch rather than closing as obsolete.

## Best Possible Solution

Keep this PR open until the remaining Telegram plugin behavior is implemented. The focused maintainer path is to rebase or replace it with a patch in extensions/telegram that registers and clears both default and all_group_chats command scopes in the current menu sync/native-disabled paths, preserves thread params for the plugin-command not-found fallback via resolveTelegramThreadSpec/buildTelegramThreadParams, and adds regression tests for dual registration, dual cleanup, retry/cache failure behavior, and threaded fallback replies.
