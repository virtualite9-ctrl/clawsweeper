## Summary

Keep PR #6457 open. Current main only covers part of the thread/auth-reply behavior; the group-scope command registration/cleanup is still absent, and the plugin-command "Command not found." fallback is still sent without topic/thread params.

## Best Possible Solution

Keep #6457 open or replace it with a focused Telegram plugin patch. The best fix belongs in the current Telegram plugin code: register and clear both default and `all_group_chats` command scopes in the menu sync/native-disable paths, preserve topic params for the plugin-command not-found fallback using the existing Telegram thread helper, and add regression coverage for dual registration, dual cleanup, retry/cache behavior, and threaded fallback replies.
