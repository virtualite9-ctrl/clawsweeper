## Summary

Keep this PR open. Current main has absorbed part of the Telegram thread/auth reply behavior, largely via the thread-spec work from #6833, but the central group-scope command registration and cleanup are still absent, and the plugin-command "Command not found." fallback still sends without thread params. The branch is stale against the current `extensions/telegram` layout, so the useful path is a focused rebase or replacement patch rather than closing it.

## Best Possible Solution

Keep this PR open until the remaining Telegram plugin behavior is implemented. The maintainer path is to rebase or replace it with a focused patch under `extensions/telegram` that registers and clears both default and `all_group_chats` command scopes, writes the command hash cache only after all required scope syncs succeed, preserves thread params for the plugin-command not-found fallback via `resolveTelegramThreadSpec` and `buildTelegramThreadParams`, and adds regression tests for dual registration, dual cleanup, retry/cache failure behavior, and threaded fallback replies including General-topic id 1 behavior.
