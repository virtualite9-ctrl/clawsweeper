## Summary

Keep PR #6457 open. Current main has part of the Telegram thread-routing foundation, including thread-aware native auth replies, but it still lacks the PR's central group-scope command registration/cleanup and still has an unthreaded plugin-command "Command not found." fallback.

## Best Possible Solution

Keep #6457 open, or replace it with a focused Telegram plugin patch, until OpenClaw registers Telegram native command menus for both default and `all_group_chats` scopes, clears both scopes when native commands are disabled, applies the existing thread helpers to the plugin-command not-found fallback, and adds tests for dual registration, dual cleanup, cache/retry semantics, and threaded fallback replies.
