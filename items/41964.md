## Summary

Keep open. Current main still has the opposite behavior from this PR: non-Control-UI-visible external-channel runs are gated out of chat and node-session fanout, and tests assert that suppression. The underlying TUI live-display gap remains plausible, but the proposed one-line guard removal needs maintainer/security design because it widens hidden external-channel run delivery to session subscribers.

## Best Possible Solution

Keep the PR open for maintainer review, but request a revised design before merge. A safer path is either an authorized operator/TUI live-view channel or TUI support for the existing `session.message`/session-event transcript surface, while preserving `isControlUiVisible` for general Control UI and node-session fanout. Coordinate that with #38829/#45417 so inbound user transcript updates and assistant streaming are handled consistently.
