## Summary

Keep this PR open. Current main still preserves the hidden-run `isControlUiVisible` gate that the PR wants to remove, so the proposed behavior is not implemented. The underlying TUI live-display gap remains plausible, but the one-line guard removal broadens a security-sensitive visibility boundary for external-channel runs and should be revised or explicitly approved before merge.

## Best Possible Solution

Keep this PR open, but request a revised design before merge. Preserve `isControlUiVisible` for general Control UI and node-session fanout, wire TUI session mode to the existing `sessions.messages.subscribe` / `session.message` transcript stream, and add a narrowly authorized assistant-streaming path if transcript events are not enough. Coordinate with #38829 and #45417, then add regression tests for external-channel sessions plus explicit review of the visibility/fanout boundary.
