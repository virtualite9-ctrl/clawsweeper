## Summary

Keep this PR open. Current main has not implemented the proposed behavior, and the underlying TUI live-display gap remains plausible. However, the PR's one-line guard removal broadens a visibility boundary for non-control-UI-visible external-channel runs, so it should be revised or explicitly approved by maintainers rather than closed as stale or merged as-is.

## Best Possible Solution

Keep this PR open, but request a revised design before merge. Preserve `isControlUiVisible` for general Control UI and node-session fanout, then add an authorized TUI/operator live-view path, likely by subscribing the TUI to `sessions.messages.subscribe` / `session.message` and adding a narrowly scoped assistant-streaming mechanism if transcript events are not enough. Coordinate with #38829 and #45417 so inbound user transcript updates and assistant live streaming are solved consistently, with regression tests for external-channel sessions and a security review of the fanout boundary.
