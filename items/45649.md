## Summary

Keep open. Current main still leaves the reported v2 tool-bubble problem actionable: tool rows default to manual collapse, the main app explicitly disables auto-expansion, collapsed summaries still use small muted/ellipsized text, and the web UI still prefixes tool details with "with".

## Best Possible Solution

Keep the issue open and fix it in the v2 web chat UI: improve `ui/src/styles/chat/tool-cards.css` contrast/type scale, stop premature summary truncation or show enough command text within the message width, add a real auto-expand/default-expanded path or preference instead of forcing per-tool clicks, and remove the blanket `with` prefix for already-readable tool details. Add focused jsdom tests for default expansion/wording plus a visual regression check for the tool bubble summary.
