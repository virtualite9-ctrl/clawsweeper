## Summary

Keep #11747 open. Current main still treats Discord, Telegram, and Slack reactions as passive system events queued for the next prompt, with no reactionAutoRespond/reactionTriggersTurn-style config, no reaction path that dispatches an immediate agent turn, and no public reaction hook that would let a plugin implement the requested workflow cleanly. Related #38714 remains the narrower open Discord hook/API tracker; closed #60805 and merged #34045 do not supersede this cross-channel reaction-wakes-agent request.

## Best Possible Solution

Keep #11747 open for maintainer product/API design. The best path is a documented cross-channel reaction wake option, such as reactionAutoRespond or reactionTriggersTurn, that preserves current passive system-event behavior while allowing authorized reactions to trigger an immediate routed agent turn, optionally scoped by emoji/policy. If maintainers prefer plugin-first automation, core still needs a reaction event hook and/or safe wake/dispatch API first; #38714 can continue tracking the narrower Discord hook surface.
