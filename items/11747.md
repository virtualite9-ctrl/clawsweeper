## Summary

Keep #11747 open. Current main still implements Discord, Telegram, and Slack reactions as routed system events queued for the next prompt, not as inputs that wake an immediate agent turn. The prior maintainer/Codex comment explicitly kept this open, and the related closed plugin-forwarding reports do not ship or supersede the requested cross-channel reaction wake behavior.

## Best Possible Solution

Keep #11747 open for maintainer product/API design. The best path is a documented cross-channel reaction wake contract, such as reactionAutoRespond or reactionTriggersTurn, that preserves existing passive system-event behavior while allowing authorized reactions to trigger an immediate routed agent turn. If maintainers prefer plugin-first automation, core still needs reaction event forwarding plus a wake/turn API before this can move cleanly to ClawHub/plugin implementations; #38714 can cover the narrower Discord plugin-hook piece.
