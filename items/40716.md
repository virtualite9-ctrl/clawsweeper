## Summary

Keep this PR open. The discussion includes a maintainer comment explicitly keeping it as the canonical Track B delivery-mirror consumer-path candidate under #69208, and current main only partially mitigates the underlying problem: replay/context now drops transcript-only OpenClaw assistant artifacts, but `chat.history`, `sessions.get`, and session-history surfaces still read/project raw transcript messages without a shared provider/model filter for delivery-mirror artifacts.

## Best Possible Solution

Keep this PR or an equivalent narrow replacement alive under #69208 Track B. The best implementation is to centralize a transcript-only OpenClaw assistant artifact predicate, preserve the existing model-replay handling, and apply the predicate consistently to `chat.history`, `sessions.get`, and any intended session-history HTTP/SSE consumer path before pagination and byte budgeting. Add focused regression coverage for the visible/API paths while leaving transcript persistence intact as an audit trail.
