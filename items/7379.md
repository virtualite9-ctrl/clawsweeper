## Summary

Keep #7379 open. Current main still has GitHub Copilot Premium/Chat quota data available through provider-usage and `/status`, but the per-response `/usage` footer path only formats token/cache/cost/session data. The related PR #43098 directly targeted this request but is closed unmerged, so it remains useful prior work rather than a shipped fix.

## Best Possible Solution

Keep #7379 open until the reply footer path intentionally reuses or loads cached provider usage for the active GitHub Copilot provider, appends Premium/Chat quota without breaking `/usage tokens` versus `/usage full` semantics, and adds a focused regression test. PR #43098 is the best prior implementation to review, but it is closed unmerged and not shipped on current main.
