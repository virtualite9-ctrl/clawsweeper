## Summary

Keep #7379 open. Current main still separates provider quota display from the per-response `/usage` footer: `/status` loads and formats provider usage windows, and GitHub Copilot supplies Premium/Chat windows, but the reply footer still only formats turn token/cache/cost data plus the full-mode session key. The related PR #43098 is closed unmerged, and the existing review comment also kept the issue open.

## Best Possible Solution

Keep #7379 open. The best path is a focused reply-footer change that reuses the active provider's cached or timeout-bounded provider usage summary for GitHub Copilot OAuth sessions, appends formatted Premium/Chat quota when available, degrades silently on auth/fetch errors, and adds a regression test in `src/auto-reply/reply/agent-runner.misc.runreplyagent.test.ts`.

## Risks / Open Questions

- A future implementation should avoid adding an unbounded provider-usage network call to every reply; use cached data or a short timeout so final response delivery is not delayed.
- The implementation should deliberately preserve `/usage tokens` versus `/usage full` semantics and OAuth/API-key cost hiding instead of silently expanding all usage modes.
- The prior PR #43098 is useful implementation/test prior art, but it is closed unmerged and would need current-main review before reuse.
