## Summary

Keep #7379 open. Current main still exposes Copilot Premium/Chat quota through the provider-usage/status path, but the per-response `/usage` footer only formats turn token/cache/cost data plus the full-mode session key. The directly related PR #43098 was closed unmerged, so the requested footer behavior is not shipped or superseded.

## Best Possible Solution

Keep #7379 open. The best path is to extend the reply footer path to reuse the active provider's cached or timeout-bounded usage summary for GitHub Copilot OAuth sessions, append Premium/Chat quota without changing `/usage tokens` versus `/usage full` semantics unexpectedly, and add a focused regression test. PR #43098 is the best prior implementation to review, but it is closed unmerged and not shipped on current main.
