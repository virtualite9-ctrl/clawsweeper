## Summary

Keep #7379 open. Current main still implements GitHub Copilot Premium/Chat quota loading for `/status` and provider usage surfaces, but the per-response `/usage` footer path still formats only current-turn token/cache/cost data plus the full-mode session key. Related PR #43098 directly targeted this request but is closed and unmerged, so it remains useful prior work rather than a shipped fix.

## Best Possible Solution

Keep #7379 open until the reply footer path intentionally reuses or loads cached provider usage for the active GitHub Copilot provider, appends Premium/Chat quota without changing `/usage tokens` versus `/usage full` semantics, and adds a focused regression test. PR #43098 is the best prior implementation to review, but it was closed unmerged and is not shipped on current main.
