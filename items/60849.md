## Summary

Keep #60849 open. Current main still only accepts `logging.redactSensitive` values `"off"` and `"tools"` in the Zod schema, exported config type, generated JSON schema/help, docs, and help-quality expectations. The PR is not implemented or superseded on main, and the linked crash-loop issue #60828 being closed does not leave a canonical open replacement for the remaining work. The PR does need follow-up before merge because the review discussion correctly notes a missing quality-test expectation and an unfinished `"all"` vs `"tools"` runtime distinction.

## Best Possible Solution

Keep the PR open and ask for a focused update: decide the intended product contract for `"all"`, then update the Zod schema, exported type, generated schema/help, docs, security audit remediation, `schema.help.quality.test.ts`, and targeted config/redaction tests. If `"all"` is meant to protect session transcripts and file output beyond `"tools"`, add the required call-site behavior before merge; otherwise document it as an accepted mode with current semantics or choose the docs-only rejection path explicitly.
