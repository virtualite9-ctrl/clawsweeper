## Summary

Keep open. Current main contains the #61776 per-provider baseUrl fix inside the Ollama stream factory, but I do not see a full-path fix for the remaining native multi-Ollama routing report: the custom API registry is still keyed by `api`, and runtime provider lookup does not appear to map arbitrary configured provider ids like `ollama2` back to the bundled Ollama provider plugin.

## Best Possible Solution

Keep the issue open and fix the runtime ownership/registration path, not just the stream factory. The likely maintainer path is to make configured provider ids whose provider config uses `api: "ollama"` resolve to the bundled Ollama runtime plugin, or to register provider-specific custom API aliases and rewrite the runtime model to those aliases. Add a regression test that configures `ollama` and `ollama2` with different baseUrls but the same native `api: "ollama"`, then verifies normal turns and compaction use the selected provider's baseUrl instead of the first registered API handler.
