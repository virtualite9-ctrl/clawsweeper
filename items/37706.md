## Summary

Keep open. Current `main` still hard-codes Feishu `@_all` as a bot mention, so the reported false positive remains in the shipped code path.

## Evidence

- **Current main still treats `@_all` as a bot mention:** `checkBotMentioned()` returns `true` whenever `event.message.content` contains `"@_all"`, before checking actual mention metadata.
  - file: [extensions/feishu/src/bot-content.ts:229](https://github.com/openclaw/openclaw/blob/55389f4be29359e2dd57ebaf548ba4a70331a445/extensions/feishu/src/bot-content.ts#L229)

  - sha: [55389f4be293](https://github.com/openclaw/openclaw/commit/55389f4be29359e2dd57ebaf548ba4a70331a445)
- **Mention gating still depends on that flag:** `parseFeishuMessageEvent()` sets `ctx.mentionedBot` from `checkBotMentioned()`, and group handling only suppresses processing when `requireMention && !ctx.mentionedBot`. An `@_all` hit therefore still satisfies mention gating.
  - file: [extensions/feishu/src/bot.ts:100](https://github.com/openclaw/openclaw/blob/55389f4be29359e2dd57ebaf548ba4a70331a445/extensions/feishu/src/bot.ts#L100)
