## Summary

Keep #37706 open. Current main still treats raw Feishu/Lark `@_all` content as `mentionedBot=true`, so a group-wide broadcast can still pass mention-gated group handling. The related PRs in the provided context are useful fix attempts, but the source and test coverage show the bug has not shipped fixed on main.

## Best Possible Solution

Keep #37706 open as the canonical Feishu/Lark bug. Land a focused Feishu plugin fix, likely via #44256, #49767, #49770, or a refreshed equivalent, that removes the raw `@_all` shortcut, relies on actual bot mention metadata and rich-text post `at` targets, and adds regression coverage for broadcast-only `@all`/`@_all` plus broadcast-with-real-bot-mention cases.
