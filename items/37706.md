## Summary

Keep #37706 open. Current main still treats raw Feishu/Lark `@_all` content as a bot mention, and that `mentionedBot=true` still bypasses the group `requireMention` gate. The related PR attempts in the provided context are either closed unmerged or still open, so the requested behavior has not shipped on main.

## Best Possible Solution

Keep #37706 open as the canonical Feishu/Lark `@all` false-positive bug. The best fix is a focused Feishu plugin change that removes the raw `@_all` shortcut, relies on actual bot mention metadata and rich-text post `at` targets, and adds regression tests for broadcast-only `@all`/`@_all` plus broadcast-with-real-bot-mention cases.
