## Summary

Keep open. Current main still treats Feishu/Lark `@_all` broadcast content as a bot mention: `checkBotMentioned()` returns `true` before checking actual bot mention metadata, `parseFeishuMessageEvent()` exposes that as `mentionedBot`, and group handling uses `mentionedBot` to bypass mention-gated suppression. The related PRs/comments confirm this is actionable, but none of that work is present on current main.

## Best Possible Solution

Keep #37706 open as the canonical Feishu/Lark bug until a focused fix removes the raw `@_all` bot-mention shortcut, preserves real bot mentions via metadata/post `at` tags, and adds regression coverage for `@_all`-only broadcasts plus `@_all` combined with a real bot mention. The existing attempted PRs can be used as implementation references, but the issue should not close before the behavior lands on main.
