## Summary

Keep #37706 open. Current main still contains the Feishu/Lark `@_all` raw-content shortcut, so a group-wide broadcast is still converted into `mentionedBot=true` and then passes the group `requireMention` gate. Related PRs in the provided discussion corroborate the fix direction, but the open/closed PR attempts have not shipped on main.

## Best Possible Solution

Keep #37706 open as the canonical Feishu/Lark bug. Land a focused Feishu plugin fix, likely via #44256, #50394, or a refreshed equivalent, that removes the raw `@_all` shortcut from `checkBotMentioned()`, relies on actual bot mention metadata and post `at` tags, and adds regression coverage for broadcast-only `@all`/`@_all` plus broadcast-with-real-bot-mention cases.
