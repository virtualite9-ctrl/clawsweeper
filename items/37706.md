## Summary

Keep #37706 open. Current main still treats raw Feishu/Lark `@_all` content as a bot mention, and that `mentionedBot=true` value still bypasses the group `requireMention` gate. The linked PRs are relevant fix attempts, but the provided context shows the focused fixes are either closed unmerged or still open, and the current checkout has not shipped the requested behavior.

## Best Possible Solution

Keep #37706 open as the canonical Feishu/Lark bug. Land a focused Feishu plugin fix, likely via #44256, #50394, or a refreshed equivalent, that removes the raw `@_all` shortcut, relies on actual bot mention metadata and rich-text post `at` targets, and adds regression coverage for broadcast-only `@all`/`@_all` plus broadcast-with-real-bot-mention cases.
