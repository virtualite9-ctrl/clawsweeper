## Summary

Keep open. Current main has the WebChat image-attachment plumbing, but the same hidden file input plus attach-button click path already existed in the reported v2026.3.13-1 source. I did not find a later source change, test, or live verification that specifically addresses the reported no-op click.

## Best Possible Solution

Keep the issue open until maintainers either reproduce and fix the runtime no-op or add a focused regression/manual verification path. A conservative fix would use a scoped Lit ref or label/input association instead of a global `document.querySelector`, set the attach button to `type="button"`, add a view test that the button triggers the colocated file input and that selection updates `chatAttachments`, and clarify whether WebChat upload is intentionally image-only.
