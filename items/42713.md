## Summary

Keep open. Current main still preserves provider/model details for individual fallback attempts, but it does not carry the originating session_id/sessionKey/lane through the exhausted-fallback wrapper or the terminal "Embedded agent failed before reply" log. The prior ClawSweeper comment identified the same remaining gap, and current main still has that behavior.

## Best Possible Solution

Keep this issue open and implement a narrow attribution fix: pass sessionId/sessionKey or lane into runWithModelFallback from agent-command and auto-reply paths, carry it on FallbackSummaryError or the terminal wrapper log context, emit attempted providers/models plus normalized session_id/sessionKey in structured JSONL fields, and add focused tests for model_fallback_decision plus the terminal `Embedded agent failed before reply` log path.
