## Summary

Keep open. Current main has partial provider/model attribution for fallback attempts, but the terminal exhausted-fallback wrapper still does not carry structured session_id/sessionKey/lane metadata. The open issue's central logging request remains valid on current main.

## Best Possible Solution

Add session attribution to the model fallback surface: pass sessionId/sessionKey or lane into runWithModelFallback from agent-command and auto-reply paths, include attempted providers/models plus normalized session_id/sessionKey in the exhausted-fallback error/log context, and add focused tests for model_fallback_decision or wrapper-error logs that assert provider chain and session attribution.
