## Summary

Keep open. Current main still reproduces the reported mismatch: doctor detects missing recent session transcripts, but its preview hint still prints `openclaw sessions cleanup --store "..." --dry-run` without `--fix-missing`, while the cleanup implementation only counts missing transcripts when `fixMissing` is true. The two related PRs in the provided timeline targeted the right one-line fix but are closed and unmerged, so they do not resolve the issue on main.

## Best Possible Solution

Keep this issue open and land the narrow fix in `src/commands/doctor-state-integrity.ts`: add `--fix-missing` to the doctor preview cleanup command for missing transcripts, then tighten `src/commands/doctor-state-integrity.test.ts` so it explicitly expects the preview hint to include `--dry-run --fix-missing`. The existing cleanup command behavior can stay unchanged because it already supports the correct dry-run preview when that flag is present.
