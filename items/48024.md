## Summary

Current main already contains a stronger version of this Telegram offset fix, including regression coverage, and the same implementation is present in the shipped `v2026.4.22` release tree.

## Evidence

- **Current main blocks failed updates from advancing the watermark:** `extensions/telegram/src/bot-core.ts` tracks `failedUpdateIds`, clamps `maybePersistSafeWatermark()` against both pending and failed update IDs, and only advances `highestCompletedUpdateId` when middleware completed successfully.

- **Current main records failed updates instead of forgetting them:** In the update wrapper, failed updates are removed from `pendingUpdateIds` and added to `failedUpdateIds`; successful updates alone call `maybePersistSafeWatermark()`. This is stronger than the PR's original `completedSuccessfully` guard because later successful updates cannot jump past an earlier failed one.
