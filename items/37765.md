## Summary

Current `main` already documents and implements OpenResponses session continuity for both `user` and `previous_response_id`, so the reported silent-ignore behavior is obsolete.

## Evidence

- **Docs now describe `user` session routing and `previous_response_id` support:** The OpenResponses API doc says requests are stateless by default, a stable `user` value derives a reusable session key, and `previous_response_id` reuses the earlier response session within the same scope.
