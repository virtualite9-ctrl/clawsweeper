## Summary

Keep open. The issue is recent, has concrete Mindie logs/config, and current main still does not show a complete fix for strict OpenAI-compatible endpoints that reject request fields. The related #38902 context is relevant, but it is closed and does not provide a shipped Mindie fix or canonical active follow-up.

## Best Possible Solution

Keep this issue open and fix the OpenAI-compatible custom-provider path or add a documented Mindie/provider-specific compat surface. The likely maintainer path is to support strict backends with request-shaping controls for fields such as `stream_options` and default `tool_choice`, validate `requiresStringContent`/tool disabling against the reported failure, and add request-builder regression tests plus docs for the supported config keys.
