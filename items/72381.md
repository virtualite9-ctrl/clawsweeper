## Summary

Keep #72381 open. Current main still lacks the requested Google Meet `realtime.agentId` config/schema surface and does not pass an agent id into the shared realtime consult runtime, so Google Meet consults still fall back to `main`. The shared runtime already supports optional `agentId`, and Voice Call already uses that path, so the PR is a focused, non-obsolete plugin fix. Related PR #72372 covers Google Meet chrome-node bridge cleanup, not consult agent routing.

## Best Possible Solution

Keep the PR open and review it as a plausible narrow fix: add `realtime.agentId` to the Google Meet plugin config type, resolver, manifest schema/help, pass it to `consultRealtimeVoiceAgent`, and keep the resolver plus embedded-agent invocation tests. Consider adding one Google Meet docs line if docs are required for this advanced config surface.
