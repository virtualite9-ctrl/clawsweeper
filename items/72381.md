## Summary

Keep this PR open. Current main still lacks the Google Meet `realtime.agentId` config/schema surface and does not pass an agent id into Google Meet realtime consults, while the shared realtime consult runtime and Voice Call plugin already support optional agent routing. Related #72372 is complementary chrome-node bridge cleanup, not a superseding implementation of this routing fix, and the provided review discussion is positive.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land or refine the narrow plugin fix: add `realtime.agentId` to Google Meet config resolution and metadata/schema, normalize it with the existing routing helper, pass it into `consultRealtimeVoiceAgent`, scope consult sessions by agent id, and keep the resolver plus embedded-agent invocation tests.
