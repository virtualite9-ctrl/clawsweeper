## Summary

Keep open. Current main still has an observable path where subagent completion proceeds with no usable child result: the announce flow retries output capture, then falls through to `(no output)` and delivers an announce. Existing tests even assert that completion-mode announces can be sent with `(no output)`. The related #68464 PR appears relevant and may become the canonical fix, but it is open/unmerged in the provided context and current main lacks its payload/claim changes.

## Best Possible Solution

Keep the issue open until main has a fail-closed completion-delivery contract: for completion-mode subagent announces, persist or derive a usable user-facing payload before delivery, otherwise defer/retry without surfacing `(no output)` or meta fallback text. #68464 is the likely place to finish this if maintainers agree it covers the missing-payload race; otherwise add a narrower fix and regression test around `runSubagentAnnounceFlow` and lifecycle cleanup.
