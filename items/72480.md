## Summary

Keep this PR open. Current main still sends model provider requests through strict `fetchWithSsrFGuard` without the env-proxy auto-upgrade, while the PR is a narrow implementation candidate with focused tests and no unrelated security-sensitive file changes.

## Best Possible Solution

Keep this PR open for maintainer review and CI. The best fix is to land this PR or an equivalent patch in `buildGuardedModelFetch` that applies trusted env-proxy mode only when there is no explicit provider dispatcher policy and `shouldUseEnvHttpProxyForUrl(url)` is true, while preserving strict behavior for explicit direct/TLS/env-proxy/explicit-proxy transport policies and retaining the two regression tests.
