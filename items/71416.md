## Summary

Keep #71416 open. Current main still implements the reported behavior: `openclaw agent` attempts the Gateway path, catches any Gateway failure, emits the fallback diagnostic on the runtime error path, and then runs the embedded agent. Tests and docs still codify the automatic stderr-only JSON-mode fallback, and no opt-in flag or stable JSON transport/fallback marker exists. The related merged PR #71478 explicitly left this issue in scope.

## Best Possible Solution

Keep the issue open for a focused CLI behavior change. The best fix is to make fallback explicit and script-detectable, likely by failing closed unless an `--allow-embedded-fallback` option is passed or by adding a stable JSON field such as `meta.transport`/`meta.fallback` plus clear non-JSON diagnostics, then updating `src/commands/agent-via-gateway.ts`, CLI option registration, tests, and `docs/cli/agent.md` together.
