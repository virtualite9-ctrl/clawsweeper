## Summary

Keep #71416 open. Current main still has the reported product behavior: `openclaw agent` attempts Gateway execution by default, catches any Gateway failure, emits only `Gateway agent failed; falling back to embedded: ...`, and then runs the embedded agent path. The CLI has `--local` but no opt-in fallback flag, and the docs/tests still codify stderr diagnostics rather than a nonzero exit, stdout banner, or stable JSON transport marker. Related PR #71478 explicitly left this issue out of scope.

## Best Possible Solution

Keep the issue open for a focused CLI behavior change. The best fix is to make embedded fallback explicit and script-detectable: either fail closed unless an `--allow-embedded-fallback` flag is passed, or add a stable JSON marker such as `meta.transport`/`meta.fallback` plus a clear non-JSON warning path. Update `src/commands/agent-via-gateway.ts`, CLI option registration, JSON output handling, tests, and `docs/cli/agent.md` together.
