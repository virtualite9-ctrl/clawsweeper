## Summary

Keep #14207 open. Current main still hard-codes inline/non-batch memory indexing concurrency at 4; the only exposed concurrency knob is `remote.batch.concurrency`, which applies when provider Batch API mode is active and does not cover the custom OpenAI-compatible inline embedding proxy path described in the issue. The existing maintainer-side Codex comment and related #68812/#68822 context remain accurate: adjacent retry/concurrency work exists, but it is not shipped or a clear canonical replacement for this specific performance request.

## Best Possible Solution

Keep #14207 open until OpenClaw adds an agent-scoped non-batch memory indexing concurrency setting, wires it through `MemorySearchConfig`, `ResolvedMemorySearchConfig`, strict Zod schema, schema labels/help/generated metadata, docs, and focused tests, then uses it in `getIndexConcurrency()` for inline/non-batch indexing without changing the existing provider Batch API meaning of `remote.batch.concurrency`. Coordinate with #68812/#68822 so retry/backoff and concurrency config land coherently.
