## Summary

Keep #14207 open. Current main still hard-codes non-Batch memory indexing fanout at 4 and does not expose an agent-scoped inline indexing concurrency setting. The existing `remote.batch.concurrency` knob is only for provider Batch API jobs and does not cover the reported custom OpenAI-compatible/Ollama inline embedding path.

## Best Possible Solution

Keep #14207 open until OpenClaw adds an agent-scoped inline/non-Batch memory indexing concurrency setting, wires it through `MemorySearchConfig`, `ResolvedMemorySearchConfig`, the strict Zod schema, generated config help/labels, public memory config docs, and focused tests, then uses it in `getIndexConcurrency()` for non-Batch indexing while preserving `remote.batch.concurrency` for provider Batch API jobs.
