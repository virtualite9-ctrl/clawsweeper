## Summary

Current `main` does not implement Telegram Bot command-menu localization. Built-in native command descriptions are still hard-coded in English, Telegram menu sync forwards those descriptions directly to `setMyCommands(...)`, and the documented/configured Telegram surface only exposes custom command entries and native-command toggles, not any locale or per-language description mapping. This issue remains a valid feature gap on the bundled Telegram plugin.

## Evidence

- **Issue request is specific and still current:** Issue #55239 asks for Chinese Telegram Bot command descriptions such as `/tts → 文字转语音` and `/help → 帮助` for the built-in Telegram command menu. There are no comments or timeline items narrowing or superseding that request.
  - command: `provided GitHub context`
- **Built-in native command descriptions remain hard-coded English:** Current `main` defines built-in native commands like `help`, `commands`, `tools`, `skill`, `status`, and `tasks` with English-only `description` strings such as `Show available commands.` and `Show current status.` in the shared command registry.

## Risks / Open Questions

- Closing this would drop a still-real product gap: current Telegram built-in menu descriptions remain English-only for Chinese users on `main`.
- `customCommands` is only a partial workaround because docs and tests show it adds extra menu entries and cannot override native commands, so it does not satisfy the reported request for localized built-ins.
