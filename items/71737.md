## Summary

Keep #71737 open. Current main has adjacent runtime and hook surfaces, but it does not implement the RFC's requested lifecycle-owned SDK contracts: typed sanitized agent-event subscriptions, plugin-namespaced run-scoped storage, plugin/session-owned scheduler cleanup, or heartbeat-specific prompt contributions. The linked #71731 is an open RFC/docs PR rather than a merged implementation, and the related Plan Mode implementation PRs referenced in the issue were closed unmerged or superseded.

## Best Possible Solution

Keep #71737 open as the focused SDK/RFC tracker unless maintainers intentionally consolidate it into #71731. The best implementation path is additive core SDK work: typed sanitized agent lifecycle subscriptions, plugin-namespaced run storage cleaned on terminal events, plugin/session-owned scheduler helpers with reset/disable cleanup, and deterministic heartbeat prompt contribution handling with fixture plugin tests.
