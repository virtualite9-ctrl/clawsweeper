## Summary

Current main already handles the reported native Windows QMD shim path: QMD subprocess startup now resolves PATH/PATHEXT `.cmd` shims, unwraps npm-style wrappers to a direct Node entrypoint, avoids shell execution, and keeps graceful builtin fallback. The docs also explicitly state WSL2 is the better-supported Windows path for QMD/full Windows use.

## Best Possible Solution

Close this issue and keep the shipped Windows wrapper-resolution path plus WSL2 guidance. Future reports should include the actual generated wrapper content if a standard npm-installed `@tobilu/qmd` shim still fails.

## Evidence
