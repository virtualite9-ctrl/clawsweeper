## Summary

Keep this PR open. Current main still lacks the proposed IPv4-only BYOH clarification in the gateway CLI docs, gateway config type docs, and no-GUI onboarding hint, while the current gateway bind/link code supports the PR premise that `lan`, `tailnet`, and `custom` BYOH-facing paths are IPv4-oriented today.

## Best Possible Solution

Keep this PR open for maintainer review or merge after normal validation. The best current product path is to document the current IPv4-only BYOH behavior in the CLI/config/onboarding surfaces now, while treating native IPv6 BYOH support as a separate behavior change if maintainers want it.
