## Summary

Keep open. Current main still lacks the PR's IPv4-only BYOH documentation and onboarding hint, while the current gateway implementation does resolve the relevant BYOH bind paths through IPv4-only helpers. The PR remains focused docs/test work rather than an obsolete or already-implemented item.

## Best Possible Solution

Keep PR #71594 open for maintainer review or merge after normal docs/test validation. The best product path is to document the current IPv4-only BYOH bind behavior in the gateway CLI/config/onboarding surfaces, while leaving broader native IPv6 BYOH support for a separate behavior change if maintainers want it.
