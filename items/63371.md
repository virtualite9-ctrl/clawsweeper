## Summary

Keep open. #63371 is recent and broad, but it names real OpenClaw browser/search surfaces that are not fully implemented or superseded on current main. Current main has web_search, web_fetch, SearXNG, and browser-plugin infrastructure, but I found no Lightpanda or PinchTab implementation and no comparative tool matrix matching the issue’s requested verification. The related ClawSweeper context also says some browser hardening work remains open or unimplemented, so there is not a strong close reason.

## Best Possible Solution

Keep #63371 open unless a maintainer intentionally converts it into narrower tracking. Best path is to split it into focused work: concrete SearXNG/search-path flake repros and fixes, browser hardening follow-ups such as the existing #42606/#49773 areas, an optional Lightpanda plugin/spike if the current browser plugin API is sufficient, and a documented QA matrix with reproducible commands.
