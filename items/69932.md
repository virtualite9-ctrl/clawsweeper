## Summary

Keep PR #69932 open. Current main still has unguarded BlueBubbles typing start, restart, and stop calls in `monitor-processing.ts`; the shared chat helper only no-ops when cached Private API status is explicitly `false`, so the PR addresses a real remaining bundled BlueBubbles channel bug rather than duplicate/superseded work from #65447.

## Best Possible Solution

Keep the PR open for maintainer review or landing. The best fix is either this narrow guard plus disabled-path regression test, or an equivalent implementation that preserves the intended behavior for explicit `private_api: false` while maintainer-deciding whether unknown/null status should skip, attempt, or refresh before typing calls.
