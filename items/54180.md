## Summary

Current `main` still appears to have the reported nested-lane bottleneck. Cron-triggered isolated agent runs are routed onto `CommandLane.Nested`, but gateway startup and hot reload only set concurrency for `Cron`, `Main`, and `Subagent`, while the command queue defaults unconfigured lanes to `maxConcurrent: 1`. That matches the issue's root-cause description, so there is not strong evidence to close this item.
