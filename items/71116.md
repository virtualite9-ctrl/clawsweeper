## Summary

Keep this issue open. Current main has SecretRef runtime documentation, protocol notes, SDK helpers for atomic JSON writes and file locks, and tests around the secrets RPC handlers, but it still does not implement the issue's requested hotspot guidance for `src/gateway/server-methods/secrets.ts` or a central plugin credential-persistence contract. The same author also has the paired PR #71156 open and unmerged, so the right maintainer path is to review, revise, or land that PR rather than auto-close this issue.

## Best Possible Solution

Keep this issue open until maintainers either merge #71156 or land another focused documentation change that adds secrets-surface `AGENTS.md` guidance and a central plugin credential-persistence contract. The likely best path is to review #71156, then add or cross-link public plugin docs covering file locks, atomic writes, lifecycle/TTL semantics, failure behavior, and when plugin credential state should use SecretRef-backed config versus local plugin storage.
