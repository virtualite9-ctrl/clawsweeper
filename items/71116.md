## Summary

Keep #71116 open. Current main has SecretRef runtime docs, tests, and SDK storage primitives, but it does not yet contain the requested secrets-specific maintainer guidance or a centralized extension credential-persistence contract. The related PR #71156 is still open and only partially covers the issue, so it is not a shipped implementation or a clear superseding canonical item.

## Best Possible Solution

Keep the issue open as the umbrella architecture/docs item. Maintainers should decide whether #71156 is the desired first slice, then land or revise docs that cover the secrets RPC invariants plus extension credential persistence contract; split the CI token audit and optional shared helper into explicit follow-up issues if they should not remain in this umbrella.
