## Summary

Keep this issue open. Current main still records OpenRouter-compatible streamed usage from token counts/local pricing and has no post-stream OpenRouter generation metadata reconciliation. The open closing PR #72598 is README-only, so it is not a valid implementation candidate.

## Best Possible Solution

Keep this issue open. Close or reject README-only PR #72598 separately, then implement the fix in the bundled OpenRouter provider/transport path: after a verified OpenRouter stream completes and a response id is known, fetch `/generation` with the resolved OpenRouter credential and persist the authoritative billed cost. Add regression coverage for tier-priced streamed usage, successful generation reconciliation, and lookup-failure fallback.
