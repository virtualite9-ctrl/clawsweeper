## Summary

Current main still has the Telegram `/models` dotted-provider callback bug. `buildProviderKeyboard()` emits raw provider IDs like `mdl_list_hf.co_1`, but `parseModelCallbackData()` still only accepts letters, digits, `_`, and `-` for `mdl_list_*` callbacks. The Telegram callback handler depends on that parser before it reaches the model-list branch. Related PRs #40251 and #46035 targeted this fix but are closed unmerged in the provided GitHub context.

## Best Possible Solution

Keep #38745 open and land a narrow Telegram plugin fix in `extensions/telegram/src/model-buttons.ts`: parse `mdl_list_` callbacks in a way that supports dotted provider IDs, preferably by splitting after the `mdl_list_` prefix at the final page separator or by expanding the provider regex to include `.`. Add regression coverage for `parseModelCallbackData('mdl_list_hf.co_1')`, provider keyboard generation with `hf.co`, and a Telegram callback/list test proving dotted providers render model-list buttons.
