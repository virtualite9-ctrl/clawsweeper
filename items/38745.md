## Summary

Current main still appears to have the reported Telegram model-list bug: provider-list and pagination callbacks preserve the raw provider id, but parseModelCallbackData() only accepts letters, numbers, underscores, and hyphens for mdl_list callbacks, so mdl_list_hf.co_1 still fails to parse. Related PRs #40251 and #46035 positively identified the same fix, but both are closed unmerged in the provided GitHub context.
