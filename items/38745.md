## Summary

Keep #38745 open. Current main still builds Telegram provider-list callback data with raw provider IDs, but the parser still only accepts letters, digits, underscores, and hyphens for `mdl_list_*` provider IDs. A dotted provider such as `hf.co` still produces `mdl_list_hf.co_1`, which cannot parse into the model-list branch. The related fix PRs #40251 and #46035 are closed and unmerged in the provided context, and their referenced commits are not present in this checkout.

## Best Possible Solution

Keep #38745 open and land a narrow Telegram plugin fix in `extensions/telegram/src/model-buttons.ts`: parse `mdl_list_` callbacks in a way that supports dotted provider IDs, preferably by splitting after the `mdl_list_` prefix at the final page separator or by expanding the provider character set. Add regression coverage for `parseModelCallbackData('mdl_list_hf.co_1')`, provider keyboard generation for `hf.co`, and paginated model-list callbacks for dotted providers.
