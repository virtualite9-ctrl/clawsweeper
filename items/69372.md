## Summary

Current main does not implement the PR's requested Control UI canvas hardening or refresh behavior, and the PR has a substantive unresolved review concern about forcing websocket reconnects during active runs. This is a real product/code path, not a cleanup-close candidate.

## Best Possible Solution

Keep the PR open for maintainer review or request changes. The likely best implementation is to keep the fail-closed iframe URL handling, add coverage for canvas and a2ui positive/negative paths, and refresh the Control UI canvas capability without websocket teardown, probably through a dedicated operator-safe capability refresh path rather than the node-only `node.canvas.capability.refresh` method.
