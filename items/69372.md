## Summary

Keep PR #69372 open. Current main still has the Control UI canvas behavior the PR is trying to address, so it is not obsolete or implemented-on-main. However, the proposed timer-driven WebSocket reconnect should not be auto-merged as-is because the existing close/re-hello paths reject pending RPCs and clear active chat run state, matching the P1 review concern in the PR discussion.

## Best Possible Solution

Keep the PR open, but require a safer core fix before merge: keep the fail-closed iframe URL behavior, add positive coverage for a2ui scoped rewriting, and replace timer-driven transport teardown with a non-disruptive capability refresh path for operator Control UI sessions, or gate refresh until the UI is idle. An additive gateway method or hello field carrying capability expiry would be cleaner than hardcoding a reconnect against the WebSocket TTL.
