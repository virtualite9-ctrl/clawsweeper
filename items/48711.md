## Summary

Keep #48711 open. Current main has meaningful memory work (memory-core, pre-compaction flush, optional Active Memory, dreaming/backfill, and session-id validation), but I did not find a shipped default behavior or regression gate that solves the issue's central bug: reliable capture and recall of project/task state across fresh sessions, mutations, and broad natural-language recall. The issue is also recent, unprotected, and has related open memory/session reliability reports in the discussion.

## Best Possible Solution

Keep the issue open and convert it into a concrete reliability track: add or import a sanitized live/QA memory-task gate, decide whether PROJECTS.md/TASKS.md-style workspace state should be a documented default or an opt-in pack, harden default durable capture and fresh-session recall ordering, and split the already-improved long-session-id validation into a separate resolved subitem if needed.
