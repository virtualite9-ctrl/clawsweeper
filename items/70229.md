## Summary

Keep #70229 open. Current main still unconditionally adds Docker `--security-opt no-new-privileges`, has no `sandbox.docker.noNewPrivileges` config key in the resolver/types/Zod schema, and does not document an opt-out. The linked bug #43996 has a concrete reproduction isolating the Docker flag, and the PR is a focused fix with tests/docs/dangerous-config reporting rather than obsolete or duplicative work.

## Best Possible Solution

Keep the PR open for maintainer review and rebase as needed. The best path is to land an equivalent secure-default `sandbox.docker.noNewPrivileges` boolean, including resolver precedence, strict config schema/types, generated config metadata, docs, Docker args tests, and dangerous-config reporting, so #43996 can be resolved without users patching installed `dist` output.
