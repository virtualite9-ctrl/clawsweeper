## Summary

Keep #29552 open. Current main has adjacent split-brain/version-drift hardening, but the reported gateway pairing failure is still plausible and not fully fixed: local gateway calls can still prefer a stale OPENCLAW_GATEWAY_TOKEN over gateway.auth.token and do not retry with config auth after AUTH_TOKEN_MISMATCH. The directly related fallback PR #29676 was closed unmerged, and users reported the bug persisted through April 22, 2026.

## Best Possible Solution

Keep this open. The best maintainer path is to add a guarded local-only retry in src/gateway/call.ts: after AUTH_TOKEN_MISMATCH, only when there is no remote mode/URL override and no explicit token/password, retry once using config-resolved gateway.auth.token/gateway.auth.password. Add regression coverage for stale OPENCLAW_GATEWAY_TOKEN. If env-first local call precedence is intentional product behavior, document that explicitly and split the update/version-drift symptoms into a narrower canonical issue.
