## Summary

Keep #29552 open. Current main has adjacent version-drift hardening and documentation, but the remaining gateway auth failure is still not fixed: local CLI gateway calls can resolve a stale `OPENCLAW_GATEWAY_TOKEN` ahead of `gateway.auth.token` and do not retry with config auth after `AUTH_TOKEN_MISMATCH`. The directly related fallback PR #29676 was closed unmerged, and reporters confirmed the bug still persisted as recently as April 22, 2026.

## Best Possible Solution

Keep this open. The best maintainer path is to add a guarded local-only retry in `src/gateway/call.ts`: after `AUTH_TOKEN_MISMATCH`, when there is no remote URL/URL override and no explicit token/password, retry once using resolved `gateway.auth.token`/`gateway.auth.password` from config. Add regression coverage for stale `OPENCLAW_GATEWAY_TOKEN`. If env-first local call precedence is intentional, document that product decision and split the package/update version-drift symptom into a narrower canonical issue.
