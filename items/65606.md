## Summary

Current main still has the QQBot 4009 bug this PR is trying to fix. `ReconnectState.handleClose` still groups `SESSION_TIMEOUT` with invalid-session close codes and returns `clearSession: true`, while `GatewayConnection.handleHello` already has a resume path when `sessionId` and `lastSeq` are preserved. The PR is a focused implementation candidate, but its patch targets the old `extensions/qqbot/src/gateway.ts` layout, so it needs to be ported to the current `extensions/qqbot/src/engine/gateway/` files before it can land.

## Best Possible Solution

Keep this PR open and request a port to current main: update `extensions/qqbot/src/engine/gateway/reconnect.ts` so `GatewayCloseCode.SESSION_TIMEOUT` preserves session state and describes a resume reconnect, update the `SESSION_TIMEOUT` comment in `constants.ts`, add a focused regression test for 4006, 4007, 4009, and unrelated close codes, then run the QQBot plugin test lane plus changed checks.
