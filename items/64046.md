## Summary

Keep this issue open. Current main has substantial redaction work for config API/UI/log sinks, but the broader request is not fully implemented: config persistence still writes JSON plaintext and plaintext `.bak` copies, and the issue discussion explicitly frames it as an active umbrella with remaining diagnostics/direct-sink audit work.

## Best Possible Solution

Keep this as the umbrella tracker, mark completed subareas in the checklist, and split/land focused follow-ups for config at-rest secret strategy, plaintext backup behavior, diagnostics/direct-sink redaction policy, and regression coverage. Current UI/config API/file-log masking should be treated as partial progress, not closure.
