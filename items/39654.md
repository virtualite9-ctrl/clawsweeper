## Summary

Current `main` does not implement this PR’s schema coercion. `commands.allowFrom` still points at the shared record-only `ElevatedAllowFromSchema`, and the current tests/docs/types still model `commands.allowFrom` as a per-provider record with `"*"` fallback. This is not strong close evidence; keep the PR open for maintainer judgment on whether to accept the alternate flat-array config shape and update the public contract accordingly.

## Evidence
