## Summary

Keep this PR open. The checked-out main still lacks the turn-start stale `pluginDebugEntries` cleanup, preserve-activity store helper path, and verbose-mode suppression after failed cleanup that this PR proposes. The PR is a focused bug fix with targeted tests and an addressed bot-review note, so it remains useful work for maintainer review rather than an auto-close candidate.

## Best Possible Solution

Keep this PR open and review it as a focused runtime/session-store bug fix. The best path is to land or revise the proposed generic turn-start cleanup: clear `pluginDebugEntries` at reply-turn start with preserve-activity semantics, make cleanup best-effort, suppress disk rehydration for the rest of the turn when clearing fails, and keep Active Memory's writer behavior extension-owned. The proposed tests in `src/config/sessions.test.ts` and `src/auto-reply/reply/agent-runner.misc.runreplyagent.test.ts` are the right guardrail surface.
