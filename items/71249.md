## Summary

Keep PR #71249 open. Current main still lacks the PR's core runtime behavior: there is no generic turn-start clear for turn-scoped plugin debug entries, verbose refresh reloads persisted entries verbatim, and existing reply-runner coverage only locks in happy-path inline debug output. The provided related items are stale plugin config/API cleanup threads, not a canonical duplicate for this verbose debug-banner bug, and the PR discussion shows the bot review concern was addressed in a follow-up commit.

## Best Possible Solution

Keep the PR open for maintainer review or rebase. The right product/code direction is a scoped runtime fix: clear turn-scoped pluginDebugEntries at reply-turn start, preserve updatedAt for metadata-only cleanup, make cleanup failures best-effort, suppress stale plugin debug rehydration for that turn, and keep the targeted store/reply-runner regression tests.
