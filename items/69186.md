## Summary

Keep #69186 open. Current main has the server-side `agent_end` plugin hook that closed the related #59199 hook request, and Control UI has browser push notification plumbing, but it does not implement the user-visible feature requested here: an optional completion/success sound for Control UI chat turns, with an on/off setting or custom sound URL. The existing terminal chat event path only updates messages, clears run state, reloads history when needed, and flushes the queue.

## Best Possible Solution

Keep the issue open for a small Control UI feature: add a browser-local preference for completion sounds, play a short bundled/Web Audio cue on successful terminal `chat` events for the active run, avoid sound on abort/error unless explicitly configured, and optionally add a custom sound URL only if the URL/security model is clear. The existing `agent_end` hook can remain the plugin/server-side automation path, but it does not replace the requested first-party Control UI UX.
