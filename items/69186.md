## Summary

Keep #69186 open. Current main has the terminal chat event needed to know when a Control UI/WebChat turn finishes, and it has generic Web Push subscription/test plumbing, but it does not implement an audible completion cue, a completion-notification trigger, a browser-local sound setting, or a custom sound URL. The related closed #59199 covers a server-side `agent_end` plugin hook, not this user-facing Control UI audio/notification feature.

## Best Possible Solution

Keep the issue open. A narrow implementation should add an opt-in Control UI/WebChat completion cue tied to terminal `chat` events for the active run, persist the preference in browser-local `UiSettings` or a documented `gateway.webchat` option, account for browser audio permission/autoplay constraints, and add focused tests around `handleChatGatewayEvent`/settings persistence plus docs under Control UI/WebChat. The existing `agent_end` hook can remain the plugin automation seam, but it is not a substitute for the requested in-browser user-facing sound.
