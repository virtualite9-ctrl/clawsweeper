## Summary

Keep #69186 open. Current main has the backend signal needed to know when an agent turn ends, including the related `agent_end` plugin hook noted by prior ClawSweeper context, but it does not implement the requested user-facing Control UI completion sound, on/off setting, or custom sound URL. The existing Control UI notification surface is Web Push subscription/test plumbing, and the chat final-event path only updates transcript/stream state and clears the active run.

## Best Possible Solution

Keep the issue open as a focused Control UI enhancement. The likely implementation belongs in `ui/src/ui`: add a persisted opt-in UI setting, optionally a custom sound URL/test button, and a small sound helper triggered from the `state: "final"` chat event path for the active run, with unit coverage around final events and settings persistence. If maintainers prefer a ClawHub/plugin route, first define and document a browser-side plugin/UI extension API rather than treating the existing server-side `agent_end` hook as sufficient.
