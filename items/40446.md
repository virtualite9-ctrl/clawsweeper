## Summary

Keep open. Current main has adjacent protections, but it does not implement this PR's passive after-turn execution-health monitor, `agents.defaults.executionHealth` config, or file-burst/no-effect/error-cascade signals. The PR is concrete and not superseded; the best maintainer path is review/rebase/fix rather than automated close.

## Best Possible Solution

Keep this PR open for maintainer review. If passive execution-health monitoring belongs in core, rebase the branch, split unrelated parser/payload changes where possible, preserve payload callback chaining, make `tool_use` parsing consumer-specific or otherwise prove it safe, and add/refresh tests for cross-turn file bursts, no-effect loops, error cascades, bounded arg hashing, and config/docs. If maintainers decide the existing tool-loop and strict-agentic direction is the product answer, create or point to a canonical superseding issue before closing this PR.
