## Summary

Current `main` does not implement per-agent TTS configuration. TTS remains a global `messages.tts` surface, the per-agent config type has no `tts` field, and the docs still describe only global or plugin-scoped TTS overrides. The issue comment's `[[tts]]` tagged-mode workaround is consistent with current behavior, so there is still a real feature gap to track.
