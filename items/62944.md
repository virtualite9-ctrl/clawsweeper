## Summary

Keep open. Current main still hardcodes `timeoutMs: 30_000` in the agent `image` tool path, while the documented/configured media-understanding timeout is only wired through the separate inbound media runner. The GitHub context also shows two open, unmerged PRs with closing syntax for this issue, so the right maintainer action is to review/land or close those implementation candidates rather than close this issue now.

## Best Possible Solution

Keep this issue open and review the linked open PRs. The fix should update `src/agents/tools/image-tool.ts` to resolve a timeout from the canonical media image config, pass that value to all `describeImage`/`describeImages` calls, and add regression coverage for configured timeout forwarding on single and multiple image inputs. After a PR merges, GitHub can close this issue as implemented.
