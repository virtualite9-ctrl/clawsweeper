## Summary

Keep #12219 open. Current main still does not ship a `skill.yaml`/`skill.yml` permission manifest standard, install-time skill permission consent, `openclaw skills audit`, signing/attestation, or trust-grade support; it also now has explicit maintainer discussion saying the item should stay open for product/security design.

## Best Possible Solution

Keep #12219 open for explicit maintainer product/security design. The likely path is a versioned skill permission manifest with validation during ClawHub and direct skill install/update flows, visible install-time permission review for `openclaw skills install`, an audit/trust report or equivalent, and integration with existing `before_install` and `before_tool_call` enforcement hooks. Use `@ellistevo/openclaw-secure`, ClawHub `shell-security-ultimate`, and `globalcaos/tinkerclaw` as references while coordinating with broader skill supply-chain tracking such as #10827 and #50090.
