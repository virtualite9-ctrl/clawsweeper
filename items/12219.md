## Summary

Keep #12219 open. Current main has generic before_tool_call and before_install hook surfaces plus some skill code scanning, but it still does not ship the requested skill.yaml/skill.yml permission manifest standard, install-time permission consent for openclaw skills install, openclaw skills audit command, or signing/attestation/trust-grade path. The latest maintainer discussion also explicitly kept this issue open and treated @ellistevo/openclaw-secure, ClawHub shell-security-ultimate, and globalcaos/tinkerclaw as useful references rather than shipped replacements.

## Best Possible Solution

Keep #12219 open for maintainer product/security design. Define a versioned skill permission manifest, validate it during ClawHub/direct skill install and updates, show declared permissions before openclaw skills install, add an audit/trust report or equivalent, and connect declarations to before_install and before_tool_call enforcement where appropriate. Treat @ellistevo/openclaw-secure, ClawHub shell-security-ultimate, and globalcaos/tinkerclaw as useful references, not shipped replacements.
