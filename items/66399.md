## Summary

Current main still does not implement the requested supervisor-level SIGTERM grace period before SIGKILL for exec/CLI supervised runs. There is adjacent process-tree graceful termination and a SIGKILL wait fallback, but the supervisor still requests SIGKILL directly and the child adapter still sends direct SIGKILL immediately after invoking the tree helper; tests still assert SIGKILL on timeout/cancel paths.

## Best Possible Solution

Keep this issue open and implement a focused process-supervisor fix: add a documented SIGTERM grace window before SIGKILL for supervised exec/CLI cancellation and timeouts, preserve a bounded hard-kill/wait fallback across child and PTY adapters, handle stdin/stdout/stderr draining or destruction only where safe, and update supervisor, adapter, exec, and CLI-runner tests to assert SIGTERM-first behavior plus SIGKILL escalation. Coordinate with #71710 for agent SIGTERM handling, but do not treat it as superseding this supervisor-side work.
