## Summary

Current main still does not satisfy #14601. The bundled OpenRouter catalog, onboarding default, tests, and public OpenRouter docs expose `openrouter/auto` and Kimi K2.6, but not `openrouter/moonshotai/kimi-k2.5`. OpenRouter does support manual/dynamic model refs, so users may work around the missing row, but that is not the requested onboard/model-selector entry.

## Best Possible Solution

Keep #14601 open. Either add `moonshotai/kimi-k2.5` to `extensions/openrouter/provider-catalog.ts` with matching capability/cost metadata, update `docs/providers/openrouter.md`, and add focused OpenRouter catalog/onboarding coverage, or have a maintainer explicitly document/comment that K2.6 supersedes K2.5 for OpenRouter onboarding/model selection.
