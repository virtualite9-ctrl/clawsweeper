## Summary

Current main still does not add `openrouter/moonshotai/kimi-k2.5` / `moonshotai/kimi-k2.5` to the bundled OpenRouter onboarding/model-picker catalog. OpenRouter supports dynamic/manual refs and the bundled catalog now has Kimi K2.6, but that does not satisfy this issue's specific selector-row request.

## Best Possible Solution

Keep #14601 open. Either add `moonshotai/kimi-k2.5` to `extensions/openrouter/provider-catalog.ts` with matching docs and catalog tests, or have maintainers explicitly document/comment that K2.6 supersedes K2.5 for OpenRouter onboarding/model selection.
