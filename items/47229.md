## Summary

Keep open. Current main no longer appears to use the Anthropic SDK `MessageStream` parser for this route, but the replacement Anthropic Messages transport still parses streamed `tool_use.input` JSON with a dependency helper that calls `JSON.parse()` without the unsafe-integer guard. The related Ollama fix exists, but it is not shared into the Anthropic `input_json_delta` path.

## Best Possible Solution

Keep this open and fix it in core transport code: move the unsafe-integer-preserving JSON literal quoting into a shared parser, use it for Anthropic `input_json_delta` parsing before `block.arguments` is populated, and add a regression test in `src/agents/anthropic-transport-stream.test.ts` that streams a `tool_use` argument like `{"to":1481220477346119781}` and asserts the resulting tool argument is the exact string.
