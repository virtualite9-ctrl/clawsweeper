## Summary

Keep open. Current `main` does not contain this fix, and the PR discussion surfaces an unresolved correctness risk in the proposed approach. The logging code on `main` still uses the dynamic `resolveNodeRequireFromMeta(...)("../config/config.js")` path the PR is trying to replace, while user-facing docs still promise that `logging.file` controls the log path. At the same time, the review thread identified plausible recursive `loadConfig()` regressions if this patch lands as written, and current tests explicitly guard against re-entrant config loads during console capture.
