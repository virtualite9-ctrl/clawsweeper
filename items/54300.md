## Summary

Keep open. Current main still leaves the two narrow hardening gaps called out in the prior review: Docker label names are interpolated into a Go template without validation, and the thread-ownership plugin still builds the ownership URL with raw path segments. The TLS/private-WS and Control UI dangerous flags remain documented, tested break-glass behavior, so that portion needs maintainer security/product judgment rather than automated closure.

## Best Possible Solution

Keep this issue open for maintainer security triage. Land a narrow hardening change that validates Docker inspect label names in readDockerContainerLabel and encodes or strictly validates thread-ownership channel/thread path segments with regression tests. Separately decide whether the private-WS and Control UI break-glass policy or error wording should change, since that behavior is currently documented, tested, and audited as intentional.
