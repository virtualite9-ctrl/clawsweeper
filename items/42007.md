## Summary

Keep open. Current main still lacks Feishu text keyword triggers under requireMention, so this PR is not obsolete. The PR is a plausible bundled Feishu plugin improvement, but the current patch needs revision before merge because the reviewed diff does not update broadcast active-agent selection/WasMentioned, accepts empty trigger strings, and omits tests/docs/generated config metadata.

## Best Possible Solution

Keep the PR open and ask for a revision in the bundled Feishu plugin. The implementation should validate non-empty trigger strings, resolve triggers consistently across top-level, account, wildcard group, and exact group config, treat a keyword match as an effective mention for both single-agent and broadcast dispatch, update activeAgentId and WasMentioned semantics, and add focused tests plus Feishu docs/generated config metadata updates.

## Risks / Open Questions

- Merging the PR as-is can silently fail in Feishu broadcast groups: a keyword-only message can pass the inbound gate while activeAgentId remains null, so no active Feishu reply dispatcher is selected.
- The proposed z.array(z.string()) plus ctx.content.includes(t) accepts empty strings; includes("") is always true and would effectively bypass requireMention for every group message if configured.
- The proposed public config field lacks focused tests, Feishu docs, and generated config metadata updates, so setup/config tooling and user-facing docs would drift.
- Security review: the provided diff only touches Feishu source/config schema files and does not change CI, dependencies, lockfiles, scripts, package publishing, or secret handling; the observable security-sensitive concern is the empty-trigger mention-gate bypass above.
