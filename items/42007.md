## Summary

Keep this PR open. Current main still does not support Feishu text keyword triggers under `requireMention`; the Feishu gate only checks native Feishu mentions. The PR is a plausible implementation candidate, but the existing review comments are substantiated: the patch needs broadcast-path handling, effective `WasMentioned` semantics, non-empty trigger validation, tests, and docs before merge.

## Best Possible Solution

Keep the PR open and ask for a revision in the Feishu bundled plugin. The best implementation should add a validated non-empty `mentionTriggers` config surface, resolve it consistently with account and per-group Feishu config, treat a keyword match as an effective mention for both single-agent and broadcast dispatch, update `activeAgentId` and `WasMentioned`, and add focused tests for single-agent, broadcast, per-group overrides, and empty/whitespace triggers, plus Feishu docs/config metadata updates.

## Risks / Open Questions

- Merging the PR as currently described would likely make keyword-triggered messages fail silently in Feishu broadcast groups because no active agent would be selected.
- The proposed `z.array(z.string())` shape would allow empty strings; with `includes("")`, an empty trigger would match every group message and bypass `requireMention`.
- Adding a new public Feishu config field should update tests, docs, and generated config metadata so setup/config tooling stays aligned.
