## Summary

Current main still matches the reported bug: the OpenAI Realtime browser-session and voice-bridge paths fall back to `process.env.OPENAI_API_KEY` and use that value directly. The linked PR #72126 is a plausible fix and directly supersedes the implementation work once merged, but it is still open and unmerged, so the issue should remain open until the fix lands on main.

## Best Possible Solution

Keep #72120 open until a fix lands on main. The likely path is to finish/review #72126 or an equivalent patch that centralizes OpenAI Realtime API-key resolution, resolves supported Keychain-style refs or fails locally with a clear error, caches successful resolved values, and adds targeted tests for both `createBrowserSession` and `createBridge`.
