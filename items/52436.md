## Summary

Keep open. Current `main` still appears to have the reported attribution bug: `sessions.usage` groups by `storeEntry.channel ?? storeEntry.origin.provider`, but session metadata stores `channel` as the delivery/group channel while `origin.provider` records the originating channel. For cross-channel sessions, that still prefers the wrong field. The same code is also present in release `v2026.4.22`, so there is no strong evidence the issue is already fixed on shipped code.
