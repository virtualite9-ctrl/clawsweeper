## Summary

Current `main` does not show a landed fix for this issue. The embedded start diagnostic still does not recognize `filePath`/`file`, and the current read runtime now requires canonical `path` anyway, so the requested alias-alignment behavior is not implemented on `main`. The original 'warning despite successful read' scenario may be partly obsolete, but the code remains inconsistent enough that there is not strong evidence to close conservatively.
