## Summary

Keep open. Current `main` does not implement `payload.model: "default"` for isolated cron runs, and the stale PR approach is not sufficient against the current model-selection flow.

## Evidence

- **Current cron model selection still treats any non-empty payload.model as a concrete override:** `resolveCronModelSelection()` applies `resolveAllowedModelRef()` whenever `payload.model` is a non-empty string; there is no special-case for `"default"` before override resolution.
  - file: [src/cron/isolated-agent/model-selection.ts:104](https://github.com/openclaw/openclaw/blob/55389f4be29359e2dd57ebaf548ba4a70331a445/src/cron/isolated-agent/model-selection.ts#L104)

  - sha: [55389f4be293](https://github.com/openclaw/openclaw/commit/55389f4be29359e2dd57ebaf548ba4a70331a445)
- **Bare model strings are parsed as literal model IDs under the default provider:** `parseModelRef()` maps a slash-less value like `"default"` to `normalizeModelRef(defaultProvider, "default")`, so current code would interpret it as a literal model name, not as a sentinel for runtime default-model resolution.
  - file: [src/agents/model-selection-normalize.ts:75](https://github.com/openclaw/openclaw/blob/55389f4be29359e2dd57ebaf548ba4a70331a445/src/agents/model-selection-normalize.ts#L75)
