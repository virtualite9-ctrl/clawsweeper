## Summary

Current main still matches the reported leak: `OpenClawChannelBridge` keeps `pendingClaudePermissions` and `pendingApprovals` in unbounded maps, only deletes them on matching replies/resolved events, and `close()` only clears `pendingWaiters`. The linked fix PR #71648 is open and directly targets this issue, so this should stay open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep #71646 open until #71648 or an equivalent maintainer-approved fix lands. The fix should clear both pending maps on bridge close, expire stale Claude permission and approval entries using bounded timers/sweeps, preserve the MCP protocol surface, and add focused regression tests around TTL eviction and close cleanup; separately decide whether a hard cap/FIFO guard is needed for burst resistance.
