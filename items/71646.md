## Summary

Current `main` still matches #71646: `OpenClawChannelBridge` keeps `pendingClaudePermissions` and `pendingApprovals` as unbounded instance maps, does not clear them on bridge close, and has no TTL/sweeper/cap path for stale entries. The linked fix PR #71648 is still the active unmerged path, so the issue should remain open.

## Best Possible Solution

Keep #71646 open until #71648 or an equivalent maintainer-approved fix lands. The fix should clear both pending maps on bridge close, expire stale Claude permission and approval entries, preserve the existing MCP protocol/tool surface, and add focused regression coverage; maintainers can decide whether hard cap/FIFO burst protection belongs in the same patch or a follow-up.
