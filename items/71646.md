## Summary

Current main still matches the reported leak: `pendingClaudePermissions` and `pendingApprovals` are plain instance maps with no cap, no TTL/sweeper, and no close-time clear. The same-author implementation PR #71648 is still open and explicitly paired with this issue, so this issue should stay open until that PR, or an equivalent maintainer-approved fix, lands.

## Best Possible Solution

Keep this issue open and use #71648, or an equivalent maintainer-approved patch, as the implementation path. The fix should clear both pending maps on bridge close, expire stale Claude permission and approval entries using existing approval expiry metadata where available, avoid post-close writes, preserve MCP notification/tool protocol behavior, and add focused regression tests. A hard cap/FIFO guard can land with the fix if maintainers want burst resistance now, or be tracked as a narrow follow-up.
