## Summary

Keep open. Current main has partial overlap via experimental broadcast groups and thread-bound agent/session bindings, but it does not implement the requested first-class team-channel model where multiple agents share one channel as peers with shared ongoing context and agent-to-agent tagging/moderation semantics.

## Evidence

- **Broadcast groups are limited scope:** The only documented multi-agent shared-channel feature is experimental Broadcast groups, and it is explicitly limited to WhatsApp.
  - file: [docs/channels/broadcast-groups.md:15](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/docs/channels/broadcast-groups.md#L15)
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **Broadcast groups do not share agent conversation state:** Broadcast groups keep separate session keys and separate conversation history per agent; the docs explicitly say an agent does not see other agents' messages. That does not satisfy the request for all team agents to know what is happening in one shared channel context.
  - file: [docs/channels/broadcast-groups.md:166](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/docs/channels/broadcast-groups.md#L166)
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **Current /agents surface is only thread binding visibility:** The existing `/agents` command only lists thread-bound agents for the current session. It is not a team-channel abstraction.
  - file: [docs/tools/slash-commands.md:121](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/docs/tools/slash-commands.md#L121)
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **Core multi-agent docs still route shared groups to one agent or broadcast:** The current multi-agent guidance says shared groups should be bound to one agent or use Broadcast groups, which confirms there is no broader first-class team-channel model on main.
  - file: [docs/concepts/multi-agent.md:224](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/docs/concepts/multi-agent.md#L224)
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)

## Risks / Open Questions

- There is partial overlap with existing multi-agent features, so a future maintainer may want to consolidate this with related shared-channel requests rather than implement a separate concept.
