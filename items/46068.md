## Summary

Keep #46068 open. Current main has related Codex/OpenAI auth and model-route improvements, but I could not prove that they fix the reported `provider=gmn/gpt-5.3-codex` 401 regression. The issue is recent, has a second user confirming it, and still needs a concrete reproduction/config trace before conservative closure.

## Best Possible Solution

Keep the issue open and ask for a fresh reproduction on current release/main: exact OpenClaw version, install method, `agents.defaults.model` and provider config with secrets redacted, auth profile status for the affected agent, the output of `openclaw models list --provider openai-codex` or the actual provider id behind `gmn`, and the raw provider error around the 401. If the user intended Codex subscription auth, the likely product path is either a doctor/config migration from the old `gmn/gpt-5.3-codex` route to `openai-codex/gpt-5.5`, or a clearer auth hint when no usable OpenAI Codex OAuth profile is selected.
