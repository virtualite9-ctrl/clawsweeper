## Summary

Keep open. Current `main` has related safeguards for version-managed Node runtimes and gateway service drift, but it does not implement the issue’s requested behavior: no installer or doctor path scans for multiple global OpenClaw installs across different Node environments, warns about duplicate `node_modules/openclaw` roots, or recommends cleanup of stale installs.

## Evidence

- **Installer only reports the active shell Node/npm, not competing global installs:** `show_install_plan()` prints OS/install method/version only, and `print_active_node_paths()` prints the current `node`/`npm` binaries for the active shell. There is no enumeration of other Node managers or other global `openclaw` package roots.
  - file: [scripts/install.sh:351](https://github.com/openclaw/openclaw/blob/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503/scripts/install.sh#L351)

- **Installer install path is anchored to the current shell's npm prefix:** `npm_global_bin_dir()` and the npm install flow use `npm prefix -g` / `npm root -g` from the current shell only. `check_existing_openclaw()` only checks whether `openclaw` is on PATH, so a stale install in another Node environment would not be detected.
