## Summary

Keep #14874 open. Current main still has no optional SIE warn/strict verification at the skill loader boundary. Existing skill loading, config, plugin hooks, ClawHub install metadata, and install-time scan hooks are adjacent protections, but they do not warn or reject before skills become prompt/runtime-visible. The directly related implementation PR #14893 was closed unmerged, and the existing issue discussion already records the same keep-open conclusion.

## Best Possible Solution

Keep #14874 open. The best path is a disabled-by-default core loader verification policy, or a documented loader-time verification hook/API, that can warn or reject before skills are exposed to prompts, slash-command discovery, snapshots, sandbox sync, or runtime use. Existing ClawHub install metadata and plugin before_install hooks should remain adjacent protections, not treated as this feature.
