## Summary

Keep #14874 open. Current main still does not implement optional SIE verification at the skill-loader boundary, and the related PR #14893 was closed unmerged. Existing install-time scanning, ClawHub archive integrity for plugins, and `before_install` hooks are adjacent hardening, not loader-time warn/strict signed-instruction enforcement before skills become prompt/snapshot/runtime-visible.

## Best Possible Solution

Keep #14874 open. The best path is a disabled-by-default loader-time verification policy, or a documented core loader verification hook/API, that can warn or reject before skills are exposed to prompts, slash-command discovery, snapshots, sandbox sync, or runtime use. Existing ClawHub install flows and `before_install` hooks should remain adjacent hardening, not treated as this feature.
