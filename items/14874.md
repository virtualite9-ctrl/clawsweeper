## Summary

Keep #14874 open. Current main still lacks the requested optional SIE warn/strict verification gate in the TypeScript skill loader/config surface, and the directly related implementation PR #14893 was closed unmerged. Existing safeguards cover path containment, size limits, allowlists, dangerous installer metadata scans, and ClawHub plugin archive integrity, but they do not classify loaded skills as unsigned/valid-signed/invalid-signed or enforce warn/strict behavior before skills reach prompts, slash commands, snapshots, sandbox sync, or runtime use.

## Best Possible Solution

Keep #14874 open. The best path is a disabled-by-default core skill-loader verification policy, or a first-class loader-time verification hook/API, that makes deterministic warn/strict decisions for unsigned, valid-signed, and invalid-signed skills before they are exposed to prompts, slash commands, skill snapshots, sandbox sync, or runtime use. The closed PR #14893 is useful prior art, but any implementation should be aligned with the current TypeScript loader, config schema, docs, and maintainer security design.
