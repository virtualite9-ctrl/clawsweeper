## Summary

Keep this PR open. Current main still hardwires Feishu mention forwarding: non-bot mention targets are injected into the agent prompt and passed to the outbound reply dispatcher, and there is no `mentionForwardMode` config or policy seam. The provided PR is a focused implementation candidate for the closed related bug, with docs/tests and no obvious security-sensitive diff surface.

## Best Possible Solution

Review and land a rebased version of this Feishu plugin fix after addressing or accepting the log-ordering nit and running the targeted Feishu config, policy, and bot dispatch tests. The right product shape is the PR's additive `mentionForwardMode: "auto" | "none"` seam, preserving the current `auto` default while applying one resolved policy to both prompt injection and outbound reply mentions.
