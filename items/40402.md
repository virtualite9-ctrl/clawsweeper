## Summary

Keep open. Current main still does not implement `--profile-id` for `openclaw models auth login`; only `paste-token` accepts that flag. The Codex OAuth identity work now derives automatic email/stable IDs, which fixes the narrower overwrite bug for many OpenAI Codex accounts, but it does not provide operator-chosen OAuth profile aliases. The related open PR #49315 remains the right implementation candidate to review rather than a reason to close this issue now.

## Best Possible Solution

Keep this issue open as the tracker for explicit OAuth profile aliases. Review and land #49315 or equivalent work that adds `--profile-id <id>` to `models auth login`, validates that the requested ID belongs to the selected provider, carries it through `LoginOptions` and auth persistence, remaps exactly one returned profile or rejects ambiguous multi-profile auth results, and updates CLI docs plus parser/persistence tests.
