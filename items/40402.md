## Summary

Keep open. Current main still does not implement `--profile-id` for `openclaw models auth login`; only `paste-token` accepts that flag. The OpenAI Codex identity work now fixes much of the automatic multi-account overwrite problem, but it still does not provide operator-chosen OAuth profile aliases. The related open PR #49315 remains an implementation candidate to review rather than a reason to close this issue before merge.

## Best Possible Solution

Keep this issue open as the tracker for explicit OAuth profile aliases. Review and land #49315 or equivalent work that adds `--profile-id <id>` to `models auth login`, carries it through `LoginOptions` and auth persistence, validates that the requested id belongs to the selected provider, remaps exactly one returned profile or rejects ambiguous multi-profile results, and updates CLI docs plus parser/persistence tests.
