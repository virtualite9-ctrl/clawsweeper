## Summary

Current `main` still does not implement the requested `--profile-id` flag for `openclaw models auth login`. The CLI only exposes `--profile-id` for `paste-token`, `LoginOptions` has no profile override, and auth persistence writes the profile IDs returned by the provider auth method. The Codex OAuth identity work now improves automatic profile naming from email or stable token identity, but it does not provide an operator-chosen alias such as `openai-codex:account2`. The directly related implementation PR #49315 is still an open candidate in the provided GitHub context, so the issue should remain open until that or equivalent work lands.

## Best Possible Solution

Keep this issue open and use it as the tracker for explicit OAuth profile aliases. The likely implementation path is to add `--profile-id <id>` to `models auth login`, validate that the profile ID belongs to the selected provider and does not collide unexpectedly, plumb the value through `LoginOptions` and auth persistence, reject ambiguous multi-profile auth results, then update CLI/docs and add parser plus persistence tests. Review/land #49315 or an equivalent PR before closing.
