## Summary

Keep open. Current main still puts the Slack manifest JSON inside the setup intro note, so the linked copy/paste bug is not implemented on main. This PR is a plausible implementation candidate, but its raw `console.log` approach needs revision because it bypasses wizard session delivery and would run after the env-var shortcut path unless guarded.

## Best Possible Solution

Keep the PR open but request a revised patch. The best maintainer path is to make the Slack manifest copyable outside the framed clack note while preserving wizard/session delivery, suppressing manifest output on the env-var shortcut path, and adding focused regression coverage for CLI output and session-based setup delivery.
