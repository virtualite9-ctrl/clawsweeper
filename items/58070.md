## Summary

Keep open. Current main still does not generate a macOS LaunchAgent `ProcessType=Interactive`, so the central plist behavior requested by this PR remains unimplemented. Main now covers part of the `WorkingDirectory` concern differently by defaulting gateway install plans to the OpenClaw state directory, which means the PR likely needs a small rebase/alignment rather than closure as obsolete. The PR is also paired with an open same-author issue it closes, and the later merged launchd supervision work explicitly kept this ProcessType/WorkingDirectory change separate.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to rebase it on current main, preserve or deliberately rework the `ProcessType=Interactive` plist change with focused `src/daemon/launchd.test.ts` coverage, and align the `WorkingDirectory` behavior with the already-landed state-dir default rather than unconditionally falling back to `/`. The linked same-author issue should stay open until this PR is merged or explicitly declined by a maintainer.
