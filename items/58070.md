## Summary

Keep open. Current main still does not generate `ProcessType=Interactive` in macOS LaunchAgent plists, so the central behavior remains unimplemented. Main partially covers the `WorkingDirectory` concern by defaulting gateway install plans to the OpenClaw state directory, which means this PR likely needs rebase/alignment rather than closure. The PR is also paired with open same-author issue #58061, and related merged launchd work explicitly left this ProcessType/WorkingDirectory change separate.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to rebase it on current main, preserve or deliberately reject the `ProcessType=Interactive` plist change with focused `src/daemon/launchd.test.ts` coverage, and reconcile the `WorkingDirectory` behavior with the already-landed state-dir default instead of unconditionally defaulting to `/`. Keep #58061 open until this PR is merged or explicitly declined.
