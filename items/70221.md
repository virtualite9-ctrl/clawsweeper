## Summary

Keep #70221 open. Current main still treats `[embed]`/canvas previews as height-only: the parser normalizes `height` to `preferredHeight`, the chat and sidebar iframe renderers only apply inline height, and the shared preview CSS forces iframe width to `100%`. This is a real Control UI rich-output feature request, not a duplicate or ClawHub/plugin cleanup candidate.

## Best Possible Solution

Implement this in the Control UI rich-output/canvas contract rather than closing it: add a sanitized `width` or `preferredWidth` field to the `[embed]` parser and structured canvas preview type, decide accepted units and responsive clamping, apply it consistently in chat-message and sidebar iframe renderers, and update the rich-output docs plus normalization/rendering tests.
