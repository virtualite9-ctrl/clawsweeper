## Summary

Keep open. Current main still defaults allowed elevated sessions to `on` when no directive/session/agent default is configured, and the exec tool still treats active `on`/`ask` default levels as elevated for calls that omit `elevated: true`. The related PR #46908 was closed unmerged in the provided GitHub context and the requested sandbox-default plus opt-in elevated mode is not implemented on main.

## Best Possible Solution

Keep the issue open and implement the remaining product fix in core: make enabled elevated access default to `off` unless explicitly configured, then add or redefine a documented opt-in mode where ordinary exec stays on the sandbox and only `exec(elevated: true)` escapes with approval. Update `get-reply-directives`, directive persistence/status text, `createExecTool` tests, elevated/exec docs, and regression coverage for sandboxed sessions with `tools.elevated.enabled: true`.
