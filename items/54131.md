## Summary

Current main already implements the reported WhatsApp `MEDIA:` auto-reply path: reply delivery parses inline `MEDIA:` directives into structured media fields, normalizes relative/workspace media paths into outbound attachments, and the WhatsApp web auto-reply sender loads and sends those media payloads. The shipped `v2026.4.22` tag also contains the directive-parsing and workspace-staging coverage, so this appears fixed in released code as well.
