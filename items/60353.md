## Summary

Keep PR #60353 open. Current main still hydrates Slack thread starter files and falls back to those files as inbound media for text-only thread replies, which matches the linked bug #60335. The PR is a focused fix with relevant manual dogfooding notes and targeted tests that are not present on main.

## Best Possible Solution

Keep the PR open and land this fix or an equivalent Slack plugin change: stop hydrating thread starter media for thread replies, keep starter text/history context, preserve direct media attached to the reply, and add the PR's regression tests for first-turn, existing-session, and reply-owned-media cases.
