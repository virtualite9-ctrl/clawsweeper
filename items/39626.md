## Summary

Keep open. Current main has a narrow mitigation for proxy-like OpenAI-compatible providers by defaulting `tool_choice` to `"auto"`, but the shared visible-text sanitizer and stream/final delivery paths still do not strip the reported bare `exec{...}` pseudo-call form. The prior ClawSweeper keep-open assessment still matches current main.

## Best Possible Solution

Keep this issue open as a behavior bug. Preserve the shipped proxy `tool_choice: "auto"` mitigation, then add either provider-specific root-cause handling for the affected model/config or a conservative user-visible sanitizer for actual tool-name-plus-JSON pseudo calls such as `exec{...}`. Add focused regression coverage for `sanitizeAssistantVisibleText`/`extractAssistantVisibleText`, text-end/message-end block streaming, and the shell/Telegram delivery path.
