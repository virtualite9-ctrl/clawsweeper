## Summary

Current `main` has several defenses against leaked tool-call scaffolding, but the evidence does not show that the exact reported bare-text form `exec{"command": ...}` is fixed. I found sanitization and tests for `[Tool Call: ...]`, `[Tool Result ...]`, `[Historical context ...]`, and XML-style tool payload leakage, plus streaming-path stripping and prompt guidance not to narrate routine tool calls. I did not find code or tests that specifically handle the issue’s exact unbracketed `toolName{json}` output, and I could not run a reliable reproduction in this checkout.
