## Summary

Keep open. Current main still lacks the Discord bot-to-bot allowlist clarification this docs PR adds, and the runtime code supports the final wording: guild user and role allowlists are evaluated together before mention gating, while allowBots separately controls whether bot-authored messages are admitted.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to merge or lightly revise the Discord docs change so the Bot to bot loops section explains users/roles member access, allowBots="mentions", and, if closing #58010 is desired, the allowFrom bot: prefix behavior in the inbound allowlist context.
