## Summary

Keep PR #60383 open. Current main still has the Feishu interactive-card fallback parsing gap: `parseInteractiveCardContent()` only handles flat card-template `markdown` and `div` elements, while the PR adds useful unshipped handling for Feishu post-format fallback cards with nested arrays, text/link elements, and titles.

## Best Possible Solution

Keep #60383 open or fold its focused fallback parser/tests into the canonical Feishu interactive-card parsing effort. The best implementation is a shared Feishu card-content parser used by `getMessageFeishu()`, thread/quote fetching, and eventually inbound event parsing, carrying over this PR's nested fallback title/text/link coverage and then validating with targeted Feishu extension tests plus the changed gate.
