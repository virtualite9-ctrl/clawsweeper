## Summary

Keep this PR open. Current main still has the public plugin SDK gap described by the PR: channel agent-tool factories receive only config, while the adjacent core tool path already receives trusted sender context. The PR is a narrow additive core SDK candidate for third-party channel plugins, not a ClawHub-only feature and not implemented on main.

## Best Possible Solution

Review and land this PR, or an equivalent core SDK change, with a narrow regression test. The right implementation is to extend `ChannelAgentToolFactory` and `listChannelAgentTools` params with optional `requesterSenderId` and `senderIsOwner`, thread `options.senderId` and `options.senderIsOwner` from `createOpenClawCodingTools`, and document the fields as trusted server-injected context. This belongs in core because third-party channel plugins need the generic sender-context seam to implement sender-aware access control.
