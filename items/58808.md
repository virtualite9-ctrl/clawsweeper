## Summary

Keep this PR open. Current main still has the SDK gap described by the PR: channel agent-tool factories receive only config, while the adjacent core tool path receives trusted sender context. The PR is a narrow additive public plugin SDK change, not obsolete and not a ClawHub-only feature.

## Best Possible Solution

Review and land this PR, or an equivalent core SDK change, with a narrow regression test. The right fix is to extend `ChannelAgentToolFactory` and `listChannelAgentTools` params with optional `requesterSenderId` and `senderIsOwner`, thread `options.senderId` and `options.senderIsOwner` from `createOpenClawCodingTools`, and document that these fields are trusted server-injected context. This belongs in core because external channel plugins need the generic sender-context seam to implement sender-aware access control.
