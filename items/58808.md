## Summary

Keep this PR open. Current main still has the reported SDK seam gap: channel agent-tool factories are typed and called with config only, while adjacent core/message-tool paths already receive trusted requester sender context. The PR remains a narrow additive implementation candidate for a concrete downstream DMWork use case, not obsolete or ClawHub-only.

## Best Possible Solution

Review and land this PR, or an equivalent core SDK change: extend ChannelAgentToolFactory/listChannelAgentTools params with optional requesterSenderId and senderIsOwner, thread options.senderId/options.senderIsOwner from createOpenClawCodingTools, and add a narrow regression test around a fake channel plugin agentTools factory.
