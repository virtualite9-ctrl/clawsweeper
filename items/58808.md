## Summary

Keep this PR open. Current main still has the exact SDK seam gap described by the PR: channel agent-tool factories receive only config, while adjacent core/message-tool paths already receive trusted requester sender context. This is a real public plugin SDK contract issue, not a ClawHub-only plugin implementation item, and the PR remains a plausible focused fix.

## Best Possible Solution

Keep this PR open or land an equivalent core SDK change. The best implementation is to extend ChannelAgentToolFactory params with optional trusted requesterSenderId and senderIsOwner, thread options.senderId/options.senderIsOwner from createOpenClawCodingTools into listChannelAgentTools, and add a narrow regression test around a fake channel plugin agentTools factory.
