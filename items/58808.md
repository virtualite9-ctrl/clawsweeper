## Summary

Keep PR #58808 open. Current main still lacks the requested trusted sender context on the public ChannelAgentToolFactory path, while adjacent message-tool/plugin discovery paths already receive requesterSenderId and senderIsOwner. This is a real core plugin SDK seam gap, not a ClawHub-only plugin implementation item.

## Best Possible Solution

Keep the PR open or land an equivalent core SDK change: extend ChannelAgentToolFactory params with optional trusted requesterSenderId and senderIsOwner, thread options.senderId/options.senderIsOwner from createOpenClawCodingTools into listChannelAgentTools, and add a narrow regression test with a fake channel plugin agentTools factory.
