## Summary

Keep PR #58808 open. Current main still lacks the requested sender context on the public `ChannelAgentToolFactory` path, while the adjacent core `message` tool and channel message-action discovery paths already receive `requesterSenderId` and `senderIsOwner`. This is a real core/plugin SDK seam gap, not a ClawHub-only plugin implementation item.

## Best Possible Solution

Keep the PR open or land an equivalent core SDK change: extend `ChannelAgentToolFactory` params with optional trusted `requesterSenderId` and `senderIsOwner`, thread those fields from `createOpenClawCodingTools` into `listChannelAgentTools`, and add a focused regression test. Because this is a public plugin contract gap, closing as ClawHub would leave third-party channel plugins without the needed core-provided sender context.
