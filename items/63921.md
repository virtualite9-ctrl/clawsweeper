## Summary

Keep open. Current `main` contains partial mitigations for this report, but not a strong end-to-end fix: OpenClaw now strips inbound user metadata envelopes and skips cron/heartbeat chatter in Dreaming ingestion, yet assistant reply directives are still written into session transcripts and the session-corpus export path does not strip them before REM ingestion. Given the later confirming comment and still-open linked follow-up work, there is not enough evidence to close conservatively.

## Evidence

- **Assistant reply directives are still persisted into transcripts:** `buildTranscriptReplyText` still writes `[[reply_to:<id>]]`, `[[reply_to_current]]`, and `[[audio_as_voice]]` into assistant transcript text on current `main`, so the raw session source can still contain the exact scaffolding tokens called out in the issue.
