## Summary

Keep open. Current `main` still records raw `process.argv`/`process.execArgv` into `config-audit.jsonl` for both config write and config observe paths, and the append path writes those records with `JSON.stringify()` without any redaction step.

## Evidence

## Risks / Open Questions

- Security bug remains on current `main`: CLI-passed secrets can still be persisted at rest in `~/.openclaw/logs/config-audit.jsonl`.
- Both write-time and observe-time audit entries are affected, so closing as implemented or cannot-reproduce would be inaccurate.
- There is no current regression test preventing future leakage or proving a fix landed on `main`.
