## Summary

This issue should remain open. Current main has real partial cleanup for heartbeat prompts, heartbeat acknowledgements, and queued system-event transcript prompts, but the pre-compaction memory-flush prompt still uses a normal visible prompt path, so the broader Control UI request is not fully implemented.

## Best Possible Solution

Keep this issue open. The best maintainer path is to reuse the existing hidden transcript contract for memory-flush runs: pass transcriptPrompt: "" or stable internal provenance for pre-compaction memory-flush turns so chat.history and Control UI suppress them while raw audit/replay data remains available. Add focused chat.history coverage for heartbeat, queued restart/system events, and memory-flush prompt suppression.
