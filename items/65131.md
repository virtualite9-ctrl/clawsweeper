## Summary

Keep open. Current main has strong partial cleanup for heartbeat/NO_REPLY and display projection, but the broader Control UI request is not fully implemented because pre-compaction memory flush still uses a normal visible prompt path and the chat-history projection has no structural filter for memory-flush/internal system turns.

## Best Possible Solution

Keep the issue open. The best maintainer path is to make internal/runtime turns share one display contract: heartbeat prompts and OK rows stay hidden, queued restart/system events stay out of visible transcript prompts, and memory-flush runs should pass transcriptPrompt: "" or stable internal provenance so chat.history and Control UI can suppress them while raw audit/replay data remains available. Add focused tests for chat.history covering heartbeat, queued restart/system events, and pre-compaction memory flush prompts.
