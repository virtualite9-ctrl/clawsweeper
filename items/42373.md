## Summary

Current main still does not implement `costCurrency` or an equivalent currency metadata/display surface. The model provider schema and config types only accept numeric pricing, the documented contract is USD per million tokens, runtime/session fields and diagnostics name cost as USD, and CLI/chat/Control UI formatters still emit dollar-prefixed labels. The prior ClawSweeper comment in the provided discussion correctly kept this issue open, and the issue has no protected label or maintainer author association requiring auto-close avoidance beyond the normal keep-open decision.

## Best Possible Solution

Keep this issue open for a real core pricing-currency contract. Maintainers should first choose the boundary for currency metadata, then update config schema/types, generated/public docs, plugin manifest/model catalog semantics, cost lookup, session/transcript/diagnostic naming, CLI/chat/status output, and Control UI usage views. Mixed-currency aggregates should be labeled, split, converted by an explicit rate source, or suppressed rather than silently summed under one symbol.
