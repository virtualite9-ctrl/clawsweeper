## Summary

Keep this issue open. Current main still has no `costCurrency` or equivalent config/type/schema surface, model pricing is documented and typed as USD, and user-visible cost formatting still emits dollar-prefixed labels across CLI, status, chat usage, telemetry, and Control UI surfaces.

## Best Possible Solution

Keep this issue open for a real core config and display enhancement. Maintainers should choose the contract boundary first: global, per-provider, per-model, or a deliberately narrower display-only default. Then add a validated schema/type field with USD as the default, update generated config docs/help and public docs, carry currency metadata through cost lookup and session/usage summaries, and update all cost renderers. Mixed-currency totals should be labeled, split, or suppressed rather than silently summed under one symbol.
