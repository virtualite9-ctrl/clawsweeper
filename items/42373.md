## Summary

Keep open. The provided discussion is a concrete feature request for CNY/EUR/etc. display, not a stale low-info bug report, and the only comment is the stale bot notice. Current main still has no `costCurrency` config surface, the provider/model schema is strict and only accepts numeric `cost`, and cost display/usage helpers still format values as USD with `$`. The title-search related items are generic config-option issues and do not supersede this request.

## Best Possible Solution

Keep this issue open for a real config and display enhancement. A conservative path is to add explicit currency metadata with USD as the default, validate it in the config/schema contract, update docs, and thread it through `/status`, `/usage full`, `/usage cost`, CLI summaries, Control UI usage/overview cards, and chat metadata. Maintainers should decide whether the option belongs per provider, per model, or as a global default before implementation.

## Risks / Open Questions

- A display-only currency symbol can mislead when sessions aggregate multiple providers with different billing currencies; the implementation should decide whether currency is global, per provider, per model, or whether mixed-currency totals need labeling/suppression.
- Renaming cost semantics away from USD would affect plugin manifest/docs and diagnostics that currently describe estimated model cost as USD.
