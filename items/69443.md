## Summary

Keep open. Current main still does not handle the reported raw `gateway call agent` shape where `params.to` is an `agent:*` session key. The Gateway only treats `params.sessionKey` as the session selector; `params.to` is forwarded as a delivery target and then the agent command resolves it like a direct recipient, which can collapse to `agent:main:main`. The built-in subagent announce path uses `sessionKey` correctly, but that does not fix this reported misuse/silent misroute path.

## Best Possible Solution

Keep the issue open and fix the Gateway `agent` boundary conservatively: if `params.to` is shaped like an `agent:*` session key and `params.sessionKey` is absent, either reject with a clear `INVALID_REQUEST` telling callers to pass `sessionKey`, or intentionally canonicalize it into `sessionKey` before delivery planning. Add a regression test for the raw RPC shape using a third-party channel-style key such as `agent:main:openclaw-weixin:direct:...`.
