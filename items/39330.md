## Summary

Keep #39330 open. Current main still appends the internal unscheduled-reminder note into reply payloads and returns those guarded payloads through final delivery, and current tests still assert that the note is user-visible in affected no-cron cases. The PR remains useful, but its remaining review concern is still real: stripping should be limited to text OpenClaw injected, not any legitimate model output ending with the same sentence.

## Best Possible Solution

Keep #39330 open for maintainer review. The best fix belongs in src/auto-reply/reply/agent-runner.ts and src/auto-reply/reply/agent-runner-reminder-guard.ts: prevent internally injected reminder-guard notes from reaching final delivery while preserving legitimate model output containing the same sentence. Gate sanitization to payloads modified by the reminder guard, or restructure the guard so it records the condition without appending user-visible text, then update the existing reminder-guard tests for cron.add success, existing active cron coverage, unrelated cron jobs, disabled cron jobs, missing sessionKey, and cron-store read failure.
