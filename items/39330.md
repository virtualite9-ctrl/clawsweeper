## Summary

Keep #39330 open. Current main still appends the internal unscheduled-reminder note to reply payloads and returns those guarded payloads through the final delivery path, while tests still assert the note remains user-visible in the affected scenarios. The PR is still useful, but should address the remaining review concern by stripping only text that OpenClaw itself injected.

## Best Possible Solution

Keep #39330 open for maintainer review. The best fix is a narrow change in src/auto-reply/reply/agent-runner.ts and src/auto-reply/reply/agent-runner-reminder-guard.ts that prevents internally injected reminder-guard notes from reaching channel-visible payloads while preserving legitimate model output containing the same sentence. Update the existing reminder-guard tests to assert clean delivery for the current cron.add, existing-cron, unrelated-cron, disabled-cron, missing-sessionKey, and cron-store-failure cases.
