## Summary

Keep #7661 open. Current main has a shared `channel-create` action name and Discord-owned channel management, but the Slack-centered request is still missing: Slack discovery does not advertise `channel-create`, Slack dispatch/runtime have no creation branch, Slack docs/scopes do not include channel creation, and the CLI still exposes only `message channel info` and `message channel list`.

## Best Possible Solution

Keep #7661 open. Implement Slack-owned `channel-create` in `extensions/slack` with a documented `channels:manage` setup path, an action gate, a `conversations.create` runtime wrapper, support for the requested name/topic/private parameters where product wants them, discovery/dispatch coverage, tests, and an `openclaw message channel create` CLI mapping through the existing message-action runner. Preserve Discord's existing channel management implementation and document provider-specific flags.
