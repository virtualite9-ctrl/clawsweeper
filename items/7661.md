## Summary

Keep #7661 open. Current main still does not implement the requested `openclaw message channel create` CLI path or Slack `channel-create` support. The shared action vocabulary includes `channel-create`, and Discord has provider-specific runtime plumbing, but Slack discovery/dispatch/runtime/docs still expose only send/read/edit/delete/pins/member/emoji/file actions and the Slack setup scopes still omit `channels:manage`. A prior 2026-04-26 maintainer/Codex review comment also kept this issue open for the same missing Slack-centered behavior, and current main still matches that assessment.

## Best Possible Solution

Keep #7661 open. Implement Slack-owned `channel-create` support in `extensions/slack` with a documented `channels:manage` setup path, discovery/action gating, a `conversations.create` runtime wrapper, dispatch/runtime tests, and `docs/channels/slack.md` updates. Add an `openclaw message channel create` CLI mapping through the existing message-action runner with provider-appropriate flags for Slack and Discord, rather than treating Discord's existing runtime plumbing as sufficient.
