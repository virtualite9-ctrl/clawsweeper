## Summary

Keep open. Current `main` still has the false-positive `No reply from agent.` logging path, while the existing tests and runner logic continue to treat exact `NO_REPLY` as an intentional silent completion. The PR’s core behavior is not already implemented on `main`, and the only recorded review concern in the provided discussion is a small helper-duplication refactor, not a correctness blocker.
