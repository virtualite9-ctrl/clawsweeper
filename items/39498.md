## Summary

Current `main` does not contain this PR's change, and there is not strong evidence to close it under any allowed reason. `normalizeSecretInputString` still only trims strings in `src/config/types.secrets.ts:116`, and the PR's added direct coverage file `src/config/types.secrets.test.ts` is not present. Main does have newer targeted ByteString-hardening utilities in `src/utils/normalize-secret-input.ts:16` with tests in `src/utils/normalize-secret-input.test.ts:16`, and some header-bound/runtime paths already use that stronger normalization (`src/web/provider-runtime-shared.ts:80`, `src/secrets/runtime-web-tools.ts:203`). But that is not the same as this PR's global config-boundary rewrite. The review concerns on the PR also still map to current code structure: non-header secrets such as Matrix passwords are still read through `normalizeResolvedSecretInputString` in `extensions/matrix/src/matrix/client/config.ts:363`, and credential presence checks still rely on `normalizeSecretInputString` semantics in `src/agents/auth-profiles/credential-state.ts:68`. So this is neither clearly implemented on `main` nor safely obsolete.
