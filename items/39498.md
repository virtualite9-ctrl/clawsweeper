## Summary

Keep #39498 open. The ByteString crash class is still real on current main, but the PR's global change to `normalizeSecretInputString` is too broad for generic SecretInput semantics and would corrupt valid non-header secrets such as Matrix passwords. The right fix is scoped header-bound normalization, not changing the shared config normalizer.

## Best Possible Solution

Keep the PR open, but request a narrower patch. Preserve trim-only semantics for `normalizeSecretInputString`, `hasConfiguredSecretInput`, and non-header credentials. Apply `normalizeSecretInput`/`normalizeOptionalSecretInput` only where secrets become HTTP header values, starting with Comfy Cloud `X-API-Key` and any shared provider request auth/header sanitizer. Add regression coverage for the Comfy pasted-U+2502 path and a guard proving Matrix-style password/body credentials keep non-Latin1 characters.
