## Summary

Keep #70882 open. Current main still forwards bundle MCP tool arguments through unchanged, and the linked Synology/Pydantic failure in #70872 is concrete and not otherwise fixed or superseded. The PR is a plausible focused fix, but its proposed implementation should be hardened before merge because review comments called out schema-array handling plus potential unsafe/unbounded JSON.parse behavior.

## Best Possible Solution

Keep the PR open and ask for a revised implementation in the bundle MCP materialize/runtime boundary: coerce only schema-declared top-level object/array properties, handle JSON Schema string and array-form types, avoid prototype-sensitive assignment, cap JSON parsing size or otherwise bound work, and add regression tests for #70872 plus the hardening edge cases. If maintainers prefer a different central tool-argument normalization seam, use this PR and #70872 as the canonical reproduction.
