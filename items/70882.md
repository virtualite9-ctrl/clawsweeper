## Summary

Keep #70882 open. Current main still forwards bundle MCP tool input unchanged into the MCP SDK, so a record like `{ params: "{}" }` would remain a string-valued nested property. The PR is a focused, unmerged fix for open #70872 and still looks relevant, though it likely needs maintainer review of the bot-reported edge cases before merge.

## Best Possible Solution

Keep the PR open and have the author or maintainer update it against current main with a narrowly scoped bundle-MCP coercion helper, regression tests for stringified object and array properties, and explicit handling for dangerous keys/size limits plus JSON Schema `type` array-form if maintainers accept that scope.
