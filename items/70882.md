## Summary

Keep #70882 open. Current main still forwards bundled MCP tool arguments unchanged, and the linked #70872 report gives a concrete failing Synology/Pydantic case where a stringified `params` object reaches the MCP server. The PR is a plausible focused fix with regression-test coverage, but it should be reviewed/hardened rather than auto-closed.

## Best Possible Solution

Keep the PR open and ask for a maintainer-reviewed, hardened version of the same product fix in `src/agents/pi-bundle-mcp-materialize.ts`, with focused tests in `src/agents/pi-bundle-mcp-tools.materialize.test.ts` proving object and array schema properties are coerced from valid JSON strings before MCP `callTool` while already-correct values pass through unchanged.
