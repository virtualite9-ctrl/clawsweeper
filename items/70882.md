## Summary

Keep #70882 open. Current main still exposes bundle MCP tools and forwards tool-call input to the MCP runtime unchanged, so a schema-declared object/array property arriving as a JSON string would still be passed through as a string. The PR is a focused fix for open bug #70872 and is not implemented, duplicated, or superseded on main.

## Best Possible Solution

Keep the PR open and route it through normal maintainer review. The best path is a hardened bundle-MCP boundary fix in src/agents/pi-bundle-mcp-materialize.ts or the adjacent runtime call boundary, with regression tests covering schema-declared object and array params received as JSON strings, plus safeguards for unsafe keys, parse size, and JSON Schema array-form type values.
