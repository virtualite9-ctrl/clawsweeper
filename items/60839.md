## Summary

Keep open. Current `main` still intentionally rejects loopback-source `trusted-proxy` auth, the docs and tests codify that behavior, and this PR's implementation is not merge-ready because it removes that guard in a security-sensitive path without corresponding test/doc changes and leaves the reviewed dead-code concern unresolved in the submitted diff.
