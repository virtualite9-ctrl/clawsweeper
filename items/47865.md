## Summary

Keep open. Current `main` still routes `sessionTarget: "main"` cron system events through heartbeat delivery to the last active chat route, and the heartbeat prompt still asks the model to relay cron/exec events to the user when chat delivery is available. I did not find a main-branch fix for the reported behavior.

## Evidence

That function never consults `job.delivery` before choosing the user-facing route.
