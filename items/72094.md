## Summary

Keep #72094 open. Current main has a substantial bundled Codex app-server harness and `/codex` chat binding, but I did not find first-class Codex Desktop Computer Use routing: no `desktopControl`/`computerUse` config, no exposed probe for `computer_use`, and no desktop-control provider selection. PeekabooBridge is implemented, but it is a separate macOS automation broker. The related discussion explicitly positions this as the narrower follow-up to #67776 rather than a duplicate of the PeekabooBridge close.

## Best Possible Solution

Keep #72094 open as the canonical narrow request for Codex Desktop Computer Use routing. The best path is a scoped design/implementation in or around the bundled Codex plugin: probe Codex app-server/Desktop feature availability, expose an explicit desktop-control policy/provider route, define channel allowlist and approval behavior for Discord/Slack-style origins, and either route through `/codex`/`agentRuntime.id: codex` or document the missing upstream app-server API if Codex cannot expose background computer-use sessions yet.
