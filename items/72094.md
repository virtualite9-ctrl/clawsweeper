## Summary

Keep #72094 open. Current main has a Codex app-server harness and the PeekabooBridge path, but it does not implement the requested Codex Desktop Computer Use routing, `codex-desktop` desktop-control provider, `computerUse` ACP/session hint, or channel-safe desktop-control policy. The issue is recent, concrete, and intentionally separated from the prior PeekabooBridge closure, so it needs maintainer product/security judgment rather than cleanup closure.

## Best Possible Solution

Keep this as a maintainer design/implementation tracker. The likely path is in the bundled `codex` plugin or an explicitly documented plugin seam: add Codex Desktop/computer-use availability probing, a selectable desktop-control route or session hint, channel-safe permission policy, result/artifact delivery back to OpenClaw conversations, and tests/docs. If maintainers decide this should live outside core, first confirm the current Plugin SDK/app-server surfaces are sufficient, then move the implementation to a ClawHub plugin with a small documented API follow-up only if needed.
