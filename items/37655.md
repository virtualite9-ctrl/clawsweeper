## Summary

Current `main` already uses a responsive Cron Jobs layout that prevents the New Job panel from overlapping the jobs area, and the shipped `v2026.4.22` release contains that implementation.

## Evidence

- **Cron page no longer renders the jobs area as the overlapping table described in the issue:** The Cron view renders the jobs pane as a card with filters and a `.list` of job entries, while the New Job form is a separate sibling section in the workspace layout.
  - file: [ui/src/ui/views/cron.ts:437](https://github.com/openclaw/openclaw/blob/55389f4be29359e2dd57ebaf548ba4a70331a445/ui/src/ui/views/cron.ts#L437)

- **Current layout gives the jobs pane and form dedicated grid columns:** `.cron-workspace` is a CSS grid with `grid-template-columns: minmax(0, 1.2fr) minmax(340px, 0.8fr)`, and the main pane explicitly has `min-width: 0`, which avoids the right-side form covering the left pane.
  - file: [ui/src/styles/components.css:913](https://github.com/openclaw/openclaw/blob/55389f4be29359e2dd57ebaf548ba4a70331a445/ui/src/styles/components.css#L913)
