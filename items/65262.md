## Summary

Keep this PR open. Current main still does not implement the central CLI diagnostic fix for `openclaw commands` with `plugins.allow`, and this PR remains a plausible implementation candidate for the linked bug #65083. It should not be closed as obsolete, but it should be narrowed before merge because the PR context still shows a brittle string-based control-flow concern and an unrelated ACP Matrix test change.

## Best Possible Solution

Keep this PR open and ask for a focused update: implement structured built-in runtime slash-command classification for `openclaw commands` using the built-in chat command registry or a typed diagnostic result, preserve plugin allow/disable diagnostics for real plugin command aliases, add `run-main` and exit-path regression coverage for `commands` with and without `plugins.allow`, and drop or split the unrelated ACP Matrix assertion changes before maintainer review and merge.
