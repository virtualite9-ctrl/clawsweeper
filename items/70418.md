## Summary

Keep open. Current main has cacheRetention, cache-TTL pruning, and documented heartbeat keep-warm guidance, but it does not implement the proposed dedicated Cache TTL Warmer config/subsystem. The related #70417/#70602 heartbeat work is explicitly orthogonal to #70418 and does not cover a non-persistent, no-delivery, exact-prefix refresh task.

## Best Possible Solution

Keep the issue open for maintainer product/API direction. If accepted, the likely path is either a core scheduler/agent-runner feature or a new narrow plugin API that can build the same normal-turn provider request through the cache boundary, issue a minimal refresh, discard output, skip recent activity, and avoid transcript/delivery/session mutations. It should include config/docs plus unit and live/contract cache tests; #70417-style heartbeat alignment should remain a separate concern.
