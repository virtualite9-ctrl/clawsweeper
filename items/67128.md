## Summary

Keep open. Current `main` does have a Telegram `/usage` command, but it only toggles the response-usage footer or returns a coarse cost summary for the current session/today/last 30 days. It does not provide the requested per-session or per-sub-agent breakdown, nor per-agent API-call counts. The backend already has richer session usage data, so this reads as an unimplemented feature request rather than something to close.

## Evidence
