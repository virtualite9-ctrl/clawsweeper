## Summary

Keep PR #70301 open. Current main already has the central `requireAuth: true` change for `/bot-approve`, but the PR is not fully obsolete because current main lacks the follow-up QQ-only framework-command guard and the stronger regression coverage described in the PR discussion.

## Best Possible Solution

Keep the PR open or ask the author to rebase it onto current main, preserving the remaining useful pieces: the QQ-only `ctx.channel` guard in `extensions/qqbot/src/bridge/commands/framework-registration.ts`, `/bot-approve ?` help preservation, and regression tests that cover both framework registration and non-execution through the pre-dispatch path.
