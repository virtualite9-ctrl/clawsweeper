## Summary

Keep #9987 open. Current main already exposes and tests backend pending exec approval listing via `exec.approval.list`, but the requested user-facing `openclaw approvals pending` command is still absent from the approvals CLI, CLI tests, and CLI docs. The related PR attempts #10013 and #10341 were closed unmerged, so there is no shipped CLI surface to close against.

## Best Possible Solution

Keep #9987 open until OpenClaw adds `openclaw approvals pending` in `src/cli/exec-approvals-cli.ts`, backed by the existing `exec.approval.list` gateway method. The first pass should include human-readable table output with command details, `--json` output for monitoring, focused tests in `src/cli/exec-approvals-cli.test.ts`, and docs updates in `docs/cli/approvals.md` plus `docs/cli/index.md`. Treat `--watch` as a follow-up unless maintainers explicitly want it in the first CLI surface.
