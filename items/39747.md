## Summary

Keep #39747 open. Current main still resolves explicit heartbeat targets through `normalizeDeliverableOutboundChannel(rawTarget)` before any plugin bootstrap can make plugin-only channel ids visible. Telegram is now covered as a bundled channel, but the user-facing contract still allows non-bundled/plugin channel ids, and the latest PR head contains relevant unmerged work in the right bootstrap-before-recognition area.

## Best Possible Solution

Keep #39747, or a focused successor, open until heartbeat target resolution bootstraps channel plugin metadata before deliverable-channel recognition, retries normalization after registry initialization, and adds a regression test for a plugin-only `agents.defaults.heartbeat.target` with an initially empty active channel registry. Maintainers should also decide whether `heartbeat.target` should count as gateway startup channel intent.
