## Summary

Keep open. Current `main` clearly handles the bundled `telegram` heartbeat target, but the broader plugin-channel case described by the PR is not strongly proven implemented: `resolveHeartbeatDeliveryTarget` still normalizes `heartbeat.target` before any plugin bootstrap, and `resolveOutboundChannelPlugin` still refuses unknown channels before its bootstrap path runs.

## Evidence

- **Heartbeat target still normalizes before bootstrap:** `resolveHeartbeatDeliveryTarget` sets `target` by calling `normalizeDeliverableOutboundChannel(rawTarget)` directly. If that returns `undefined`, the code falls back to `target = "none"`; there is no retry or bootstrap step in this path.
  - file: [src/infra/outbound/targets.ts:97](https://github.com/openclaw/openclaw/blob/cb2c36b04975e34ab7014a11575d6589aa96ae32/src/infra/outbound/targets.ts#L97)
  - command: `nl -ba src/infra/outbound/targets.ts | sed -n '84,110p'`
  - sha: [cb2c36b04975](https://github.com/openclaw/openclaw/commit/cb2c36b04975e34ab7014a11575d6589aa96ae32)
- **Bootstrap only happens after successful normalization:** `normalizeDeliverableOutboundChannel` returns `undefined` unless `isDeliverableMessageChannel(normalized)` is already true. `resolveOutboundChannelPlugin` only calls `maybeBootstrapChannelPlugin` after that normalization succeeds, so arbitrary not-yet-registered plugin channel ids are still not obviously covered.
  - file: [src/infra/outbound/channel-resolution.ts:19](https://github.com/openclaw/openclaw/blob/cb2c36b04975e34ab7014a11575d6589aa96ae32/src/infra/outbound/channel-resolution.ts#L19)
  - command: `nl -ba src/infra/outbound/channel-resolution.ts | sed -n '18,80p'`
  - sha: [cb2c36b04975](https://github.com/openclaw/openclaw/commit/cb2c36b04975e34ab7014a11575d6589aa96ae32)
- **Telegram now resolves as a bundled channel independent of plugin registry bootstrap:** Bundled channel ids are loaded from the bundled channel catalog into `CHANNEL_IDS`, and deliverable-channel normalization accepts those ids. That means the PR's motivating `telegram` example is now handled by the bundled-channel path rather than by plugin bootstrap.
