## Summary

Keep this PR open. Current main still generates Windows gateway task scripts without a UTF-8 code-page bootstrap, and `readScheduledTaskCommand` would currently parse a future `chcp 65001>nul` line as the command. The PR remains a focused implementation candidate for the linked Windows Chinese-username gateway failure, with low supply-chain risk based on the provided diff.

## Best Possible Solution

Keep this PR open for maintainer review, or replace it with an equivalent main-branch fix in `src/daemon/schtasks.ts`: emit a deterministic UTF-8 code-page bootstrap in generated Windows gateway task scripts, teach `readScheduledTaskCommand` to ignore that exact bootstrap during read-back, and keep the Chinese-path regression coverage while preserving current managed-env audit behavior.
