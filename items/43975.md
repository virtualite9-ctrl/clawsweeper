## Summary

Keep this PR open. Current main still does not bootstrap UTF-8 in generated Windows gateway task scripts, and the current parser would treat a future `chcp 65001>nul` line as the command unless the parser change lands too. The PR remains a focused implementation candidate for the linked Windows Chinese-username gateway failure.

## Best Possible Solution

Keep this PR open for maintainer review, or replace it with an equivalent main-branch fix. The fix belongs in `src/daemon/schtasks.ts`: emit a deterministic UTF-8 code-page bootstrap in generated Windows gateway task scripts, teach `readScheduledTaskCommand` to ignore that exact bootstrap during read-back, and keep the Chinese-path regression coverage while preserving the newer environmentValueSources audit behavior on main.
