## Summary

Keep #28025 open. Current main still does not implement a Slack `thread.replyOnToolUse` option or an equivalent per-turn tool-use-gated routing policy; Slack reply targets are still selected from configured reply mode and existing thread context before tool callbacks run.

## Best Possible Solution

Keep #28025 open. If accepted, add a default-false typed Slack plugin config option, carry a reliable per-turn tool-use signal into Slack reply planning before streaming/final delivery chooses a target, preserve explicit thread replies plus existing `replyToMode` and `thread.inheritParent` behavior, and add Slack tests for DMs, channels, existing threads, reply-mode interactions, streaming, and tool/progress callbacks.
