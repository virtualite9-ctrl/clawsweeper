## Summary

Current `main` already contains targeted WhatsApp inbound handling for linked-device/self-chat `fromMe` events, plus regression tests that match the reported Mac-client symptom pattern. The code now drops only genuine gateway echoes and still processes owner/self inbound events.

## Evidence

- **Current inbound filter only skips tracked gateway echoes:** `monitorWebInbox` now ignores `fromMe` messages only when their message ID matches a recent outbound message sent by OpenClaw itself, instead of blanket-dropping all `fromMe` inbound events.
