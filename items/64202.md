## Summary

Current main no longer has the old Scene-tab trace-card layout described in #64202. Dreaming is now split into compact `Scene` plus separate scrollable `Diary` and `Advanced` tabs, and the same implementation is already present in release `v2026.4.23`, so the reported Scene clipping/overlap path is obsolete on shipped code.

## Evidence

- **Dreaming page split into three tabs:** `renderDreaming` now routes `/control/dreaming` between `Scene`, `Diary`, and `Advanced` tabs instead of keeping all Dreaming surfaces in one Scene layout.
  - file: [ui/src/ui/views/dreaming.ts:290](https://github.com/openclaw/openclaw/blob/5b8bd6371c666a5de43c82305e0315771e8e49d5/ui/src/ui/views/dreaming.ts#L290)
  - sha: [5b8bd6371c66](https://github.com/openclaw/openclaw/commit/5b8bd6371c666a5de43c82305e0315771e8e49d5)
- **Current Scene tab is compact only:** `renderScene` renders the decorative scene, status, and phase chrome only; it does not render the old tall trace-card/grid content that the issue discussion described as clipping/overlapping.
  - file: [ui/src/ui/views/dreaming.ts:370](https://github.com/openclaw/openclaw/blob/5b8bd6371c666a5de43c82305e0315771e8e49d5/ui/src/ui/views/dreaming.ts#L370)

  - file: [ui/src/styles/dreams.css:457](https://github.com/openclaw/openclaw/blob/5b8bd6371c666a5de43c82305e0315771e8e49d5/ui/src/styles/dreams.css#L457)
  - sha: [5b8bd6371c66](https://github.com/openclaw/openclaw/commit/5b8bd6371c666a5de43c82305e0315771e8e49d5)
- **Diary surface is also scrollable:** The Diary tab uses its own `.dreams-diary` container with `flex: 1` and `overflow: auto`, reinforcing that long Dreaming content is no longer forced into the compact Scene hero container.
  - file: [ui/src/styles/dreams.css:675](https://github.com/openclaw/openclaw/blob/5b8bd6371c666a5de43c82305e0315771e8e49d5/ui/src/styles/dreams.css#L675)
