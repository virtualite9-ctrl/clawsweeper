## Summary

Current main already implements the requested behavior by storing static API-key auth profiles as env-backed refs (`keyRef`) instead of stale plaintext keys, stripping plaintext on save when a ref exists, and resolving the current ref-backed secret at runtime. The relevant code paths are already present in release `v2026.4.22` and unchanged from that tag to current main.

## Evidence

- **API-key onboarding stores env refs:** `buildApiKeyCredential()` converts `--secret-input-mode ref` or `${ENV}` input into `keyRef` instead of persisting a raw key.
  - file: [src/plugins/provider-auth-helpers.ts:61](https://github.com/openclaw/openclaw/blob/d38ed0831d7372c8505dc944aea69b5b83da1435/src/plugins/provider-auth-helpers.ts#L61)
- **Onboarding test expects OPENAI_API_KEY ref, not plaintext:** The auth onboarding test asserts `openai:default` is saved with `keyRef: { source: "env", provider: "default", id: "OPENAI_API_KEY" }` and no `key` field.
  - file: [src/commands/onboard-auth.test.ts:296](https://github.com/openclaw/openclaw/blob/d38ed0831d7372c8505dc944aea69b5b83da1435/src/commands/onboard-auth.test.ts#L296)
- **Persisted auth store strips plaintext when a ref exists:** `saveAuthProfileStore()` writes the persisted auth store payload, and the dedicated save test verifies `openai:default.key` is removed while `keyRef` remains.
  - file: [src/agents/auth-profiles/store.ts:375](https://github.com/openclaw/openclaw/blob/d38ed0831d7372c8505dc944aea69b5b83da1435/src/agents/auth-profiles/store.ts#L375)
- **Save-path regression test covers the stale-key case:** The test builds an auth store with both `key` and `keyRef`, saves it, and asserts the persisted `auth-profiles.json` keeps only `keyRef` for `OPENAI_API_KEY`.
  - file: [src/agents/auth-profiles.store.save.test.ts:20](https://github.com/openclaw/openclaw/blob/d38ed0831d7372c8505dc944aea69b5b83da1435/src/agents/auth-profiles.store.save.test.ts#L20)
- **Runtime resolves current ref-backed secret:** `resolveApiKeyForProfile()` resolves `cred.keyRef`/`cred.tokenRef` through the current config/secret resolver at runtime instead of trusting a cached plaintext value.
  - file: [src/agents/auth-profiles/oauth.ts:291](https://github.com/openclaw/openclaw/blob/d38ed0831d7372c8505dc944aea69b5b83da1435/src/agents/auth-profiles/oauth.ts#L291)
- **Runtime snapshot coverage rehydrates env-backed auth:** The secrets runtime integration test loads `auth-profiles.json` containing only `keyRef`, injects current env values, refreshes the snapshot, and expects the resolved runtime profile to contain the current key while preserving the ref.
  - file: [src/secrets/runtime.auth.integration.test.ts:166](https://github.com/openclaw/openclaw/blob/d38ed0831d7372c8505dc944aea69b5b83da1435/src/secrets/runtime.auth.integration.test.ts#L166)
- **E2E onboarding guard rejects plaintext persistence:** The npm onboarding smoke checks that `auth-profiles.json` contains `OPENAI_API_KEY` and explicitly fails if the raw OpenAI test key was persisted.
  - file: [scripts/e2e/npm-onboard-channel-agent-docker.sh:351](https://github.com/openclaw/openclaw/blob/d38ed0831d7372c8505dc944aea69b5b83da1435/scripts/e2e/npm-onboard-channel-agent-docker.sh#L351)

## Close Comment

Closing from a Codex review because this is already implemented on current `main`, and the same auth-profile code is already present in `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`). Static API-key onboarding now writes env-backed `keyRef` entries instead of caching plaintext keys (`src/plugins/provider-auth-helpers.ts:61`, `src/commands/onboard-auth.test.ts:296`), persisted auth-store saves strip plaintext when a `keyRef`/`tokenRef` exists (`src/agents/auth-profiles/store.ts:375`, `src/agents/auth-profiles.store.save.test.ts:20`), and runtime resolves the current ref-backed secret when loading auth profiles (`src/agents/auth-profiles/oauth.ts:291`; see snapshot coverage in `src/secrets/runtime.auth.integration.test.ts:166`). I also compared those code paths against the `v2026.4.22` tag and found no differences there, so this behavior is already shipped.
