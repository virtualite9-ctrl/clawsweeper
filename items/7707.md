## Summary

Keep #7707 open. Current main has adjacent provenance, source labels, session visibility filtering, temporal decay, Active Memory, and Memory Wiki concepts, but it still lacks the requested persistent memory trust tiers, trust-policy config, review/expiration enforcement by trust level, and trust-weighted retrieval/downranking.

## Best Possible Solution

Keep #7707 open for maintainer/product/security design. A complete solution should add structured origin trust metadata at memory ingestion, persist it through memory schemas/backends and SDK contracts, document trust-policy config, enforce retrieval-time filtering/downweighting/review gates, support expiration by trust tier, preserve provenance through compaction and Memory Wiki flows, expose trust state where useful in UI/docs, and add regression coverage across builtin, QMD, Active Memory, session-memory, and compaction paths. HMAKT99's AKF comment is useful prior art, not shipped behavior.
