## Summary

Current main still does not canonicalize an automatic session auth-profile override from a legacy OAuth `provider:default` profile to a preferred OAuth alias. This PR remains a focused, plausible auth-routing bug fix, but it needs a rebase onto the current split auth-profile modules and maintainer review of the identity-matching semantics before landing.

## Best Possible Solution

Keep this PR open. Rebase it onto current main, port the implementation to the current split `auth-profiles/*` imports and mocked `session-override.test.ts` harness, preserve the identity-mismatch and rotation-progress regression cases, then run the targeted auth-profile test lane and changed gate before maintainer review or landing.
