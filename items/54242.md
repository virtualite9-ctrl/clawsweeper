## Summary

Keep open. Current `main` still does not rewrite or validate hardlink link targets in backup tarballs: backup creation remaps only `entry.path`, verification inspects only archive entry paths, and the backup create/verify code is unchanged from the latest release while the `tar` dependency remains `7.5.13`. That means the reported class of archive can still plausibly verify successfully while failing broad extraction on macOS `bsdtar`.

## Evidence

- **Backup creation only rewrites archive entry paths:** `createBackupArchive()` calls `tar.c()` with `portable: true` and `preservePaths: true`, and the only mutation in `onWriteEntry` is `entry.path = remapArchiveEntryPath(...)`. There is no handling for hardlink metadata such as a link target/path.

- `gh` was not authenticated in this environment (`GH_TOKEN` missing), so I could not inspect related GitHub PRs/issues beyond the provided context.
