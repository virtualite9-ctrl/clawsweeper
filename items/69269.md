## Summary

Keep #69269 open. Current main still has both reported invariant gaps: reset hook contexts do not include `messageProvider`, compaction hooks use the raw provider/channel value instead of a shared normalized resolver, and the common compaction accounting helper still ignores zero-token compaction results. The linked PR #69270 appears to be the right canonical fix path, but it is still an open PR and the fix is not on current main.

## Best Possible Solution

Keep #69269 open until a coordinated fix like #69270 lands. The fix should add a shared hook message-provider resolver, use it in direct compaction, queued/context-engine compaction, command reset, and gateway reset paths, normalize explicit providers, avoid inventing providers for non-channel session keys, and update cached `totalTokens`/freshness/cost state for finite `tokensAfter` values including `0`, with regression tests for mixed-case providers, channel-scoped session keys, non-channel keys, and zero-token compactions.
