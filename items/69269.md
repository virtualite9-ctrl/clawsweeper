## Summary

Keep #69269 open. Current main still does not preserve the reported reset hook `messageProvider` invariant, and the shared compaction-count update path still ignores `tokensAfter === 0`, which is the token-accounting failure described in the issue. The provided GitHub context also shows PR #69270 is open and explicitly targets this coordinated fix, so the issue should remain open until that or an equivalent fix lands.

## Best Possible Solution

Keep the issue open and land or adapt the coordinated fix from #69270: introduce one shared hook-message-provider resolver, apply it to direct compaction, engine-owned compaction, command reset, and gateway reset, avoid manufacturing providers for non-channel keys, and update compaction accounting so a defined `tokensAfter: 0` clears stale token/cost state. Add focused regression tests for mixed-case provider normalization, non-channel session keys, reset/gateway hook contexts, and zero-token compaction outcomes.
