## Summary

Keep #69269 open. Current main only partially addresses the token-accounting side and still leaves the reported hook-context invariant gaps: auto compaction, command reset, and gateway reset do not provide `messageProvider`, engine-owned compaction uses a raw channel/provider value rather than a shared normalized resolver, and the common session compaction counter path still ignores `tokensAfter === 0`. The linked coordinated fix PR #69270 is still open/not merged in the provided GitHub context, so the issue should remain open until that or an equivalent fix lands.

## Best Possible Solution

Keep #69269 open and land/review #69270 or an equivalent coordinated fix. The best implementation is a shared hook message-provider resolver used by auto compaction, engine-owned compaction, `/new`/`/reset`, and gateway `sessions.reset`; it should normalize explicit providers, infer only from deliverable channel-scoped session keys, leave non-channel keys unresolved, and update accounting so `tokensAfter: 0` resets cached token/cost fields with regression coverage for mixed-case providers and zero-token compactions.
