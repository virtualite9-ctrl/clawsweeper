## Summary

Keep this issue open. Current main still matches the reported gap: the voice-call Twilio streaming path sends the raw spoken text into telephony TTS, while the telephony TTS runtime/config boundary has no directive parsing or per-utterance override parameter. The existing directive parser and message-channel TTS path support `[[tts:voiceId=...]]`, so the remaining work is wiring that behavior into the voice-call telephony path rather than closing this as implemented.

## Best Possible Solution

Implement this in the voice-call owner path with a small additive telephony TTS contract change: parse and strip `[[tts:...]]` directives before Twilio media-stream synthesis using the existing `parseTtsDirectives` policy, pass provider override data or an equivalent per-utterance merged config into `textToSpeechTelephony`, forward providerOverrides through telephony preparation/synthesis, and add focused tests covering directive stripping plus ElevenLabs `voiceId`/`modelId` override propagation for telephony.
