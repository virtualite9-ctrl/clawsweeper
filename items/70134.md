## Summary

Keep open. Current `main` still keys the models ready-cache off `auth-profiles.json` mtime instead of semantic auth state, so the PR's requested behavior is not implemented on `main`. The shipped auth resolution is also sensitive to profile order, legacy `apiKey` aliases, and OAuth access-token content, so the remaining PR discussion is still material rather than obsolete.
