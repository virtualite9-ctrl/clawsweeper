## Summary

Keep open. Current main still fingerprints the models-config ready cache with `auth-profiles.json` mtime, and I did not find an equivalent semantic auth-profile fingerprint helper or regression tests on main. The PR addresses a real cache invalidation surface, but it still needs normal maintainer review because the provided review discussion leaves an OAuth token-content fingerprint tradeoff unresolved.

## Best Possible Solution

Keep PR #70134 open for normal maintainer/code review. The likely best implementation is a dedicated semantic auth-profile fingerprint used by `buildModelsJsonFingerprint`, with regression coverage for no-op rewrites, meaningful key/ref/OAuth/order changes, `models.json` mtime invalidation, and parse/read error behavior, plus an explicit resolution of whether OAuth access/refresh token content must be hashed for cache correctness.
