## Summary

Current main fixes the reported Bonjour crash path. The v2026.4.24 plugin entry did not pass the process rejection handler into the Bonjour advertiser, but main now wires both unhandled-rejection and uncaught-exception handlers, classifies the exact `CIAO PROBING CANCELLED` message, shares the handler registry across plugin-staged module copies, and has tests plus a changelog entry for the crash-loop fix.

## Best Possible Solution

Close this issue as fixed on main and keep the current Bonjour plugin/runtime handler implementation. The reporter’s v2026.4.24 crash should be addressed by the next release that includes commit 5baf90ffeffbaf5725e60bfb8f280306bb7b6946.

## Evidence

- **Exact rejection is handled:** The advertiser registers the supplied unhandled-rejection handler and returns true for classified ciao process errors. Cancellations are logged at debug level instead of falling through to the global fatal unhandled-rejection path.
