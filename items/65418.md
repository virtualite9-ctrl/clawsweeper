## Summary

Keep open. Current main still applies the surrogate-stripping sanitizer to signed Anthropic thinking blocks on replay, and the pinned pi-ai dependency used by the Anthropic Vertex path has the same serializer behavior. Existing tests prove signed thinking is preserved earlier in history sanitization, but they do not cover the reported unpaired-surrogate mutation at payload serialization.

## Best Possible Solution

Patch signed Anthropic `thinking` and `redacted_thinking` replay so opaque signed content is forwarded unchanged, while keeping surrogate sanitization on unsigned text fallback and new outbound text. Add a regression test that builds an Anthropic payload containing a signed thinking block with an unpaired surrogate and asserts the payload `thinking` value is byte-for-byte unchanged; for Anthropic Vertex, update or patch the pinned `@mariozechner/pi-ai` serializer rather than only fixing OpenClaw's local transport copy.
