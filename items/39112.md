## Summary

Keep PR #39112 open. Current main still zero-fills omitted native Ollama usage counters, the dashboard/session usage path consumes persisted transcript usage rather than estimating later, and the PR’s underlying fix remains useful even though its patch targets the old `src/agents/ollama-stream.*` layout and needs porting to the bundled Ollama plugin.

## Best Possible Solution

Keep this PR open or replace it with an equivalent current-main fix in `extensions/ollama/src/stream.ts`: estimate native Ollama input/output usage only when counters are absent, preserve exact nonnegative counters including explicit zero, estimate from converted message/image/tool/tool-schema payloads and accumulated streamed output, and add focused regression coverage in the current Ollama plugin tests.
