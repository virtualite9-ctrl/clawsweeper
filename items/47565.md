## Summary

Keep open. Current main only partially addresses the related microphone-only case by allowing same-origin microphone access, but #47565 asks for configurable Control UI Permissions-Policy and CSP overrides, especially adding local inference servers to CSP connect-src. The relevant headers remain hardcoded and the strict config schema has no accepted permissionsPolicy, cspOverrides, or equivalent field.

## Best Possible Solution

Keep the issue open for a core gateway/config change. A good implementation would add a narrow, validated Control UI header configuration surface, wire it through GatewayControlUiConfig, the strict zod schema, generated config docs/help, and the Control UI header builders, with tests proving defaults remain locked down while explicit operators can add microphone permissions and local connect-src endpoints for custom Control UI roots.
