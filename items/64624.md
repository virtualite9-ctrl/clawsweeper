## Summary

Keep open. Current main still sends WhatsApp channel connected/disconnected lifecycle notices straight into the resolved session system-event queue, and I did not find a threshold, verbosity setting, reconnect coalescing, or surface-aware filter. The adjacent related items are about transient tool-error warnings and Feishu health reporting; neither supersedes this requested channel lifecycle notification policy.

## Best Possible Solution

Keep this issue open and implement a tested channel lifecycle notification policy. A practical path is a narrow WhatsApp/account-level setting such as `statusEvents` and/or `notifyAfterMs`, coalescing reconnects that recover before the threshold while keeping serious stop/auth notices visible. Add regression coverage around the WhatsApp monitor plus `system-events` draining behavior; if multiple channels need it, expose an additive plugin SDK helper.
