## Best Possible Solution

Close this PR without merging. Keep the current main resolver behavior that honors provider-prefixed configured rows for bare routed requests. If exact-first semantics for mixed bare/prefixed configured rows are desired, open or land a small follow-up on current main with a focused regression test.
