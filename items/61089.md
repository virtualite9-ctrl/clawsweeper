## Summary

Keep PR #61089 open. Current main still has the web UI slash-command path importing shared command registry code that pulls in `thinking.js` and provider-thinking runtime, so the central browser-bundle leak is still live. The PR contains a useful direction for the thinking-level dependency injection, but it should be narrowed because its session chat-type rewrite is now obsolete on main.

## Best Possible Solution

Keep the item open and ask for a narrowed rebase: retain only the `commands-registry.shared.ts` / `commands-registry.data.ts` dependency-injection change, make browser/shared code default to `thinking.shared.js`, inject `thinking.js` only from the Node registry path, drop the stale session chat-type/provider wiring, and add a small esbuild/Vite-style regression proving `ui/src/ui/chat/slash-commands.ts` does not bundle `src/auto-reply/thinking.ts` or provider runtime.
