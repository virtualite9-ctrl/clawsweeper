## Summary

Keep open. Current main still renders the Control UI chat session dropdown as a plain native select with grouped options and no clear-completed, per-entry dismiss, or completed-subagent separator. The backend and UI types now expose subagent lifecycle state, and the separate Sessions tab can delete selected sessions through `sessions.delete`, but that is only a partial workaround and does not implement the dropdown-specific request.

## Best Possible Solution

Keep the issue open. The best implementation is a status-aware chat session selector cleanup path: classify completed subagent rows using `isSubagentSessionKey()` plus `status`/`endedAt`, expose either an adjacent `Clear completed` action or a custom dropdown/menu that can host per-row dismiss controls and separators, call the existing `sessions.delete` RPC for eligible rows, refresh `sessions.list`, and cover the selector behavior with focused UI tests.
