## Summary

Keep #39138 open. Current main still replays outbound delivery-queue JSON entries on gateway startup without a stale-age cutoff or replay-count cap, so an old retryCount=0 entry with no lastAttemptAt can still be resent after restart.

## Best Possible Solution

Implement a conservative outbound recovery policy in src/infra/outbound: expire or move stale pending deliveries to failed before replay, preserve crash-window recovery for recent sends, and add focused tests for old retryCount=0 entries, maxRecoveryMs interaction, and restart recovery. If maintainers want operator control, add documented config/schema for max age and possibly max replay count.
