## Summary

Keep #39138 open. Current main still starts outbound delivery recovery on gateway startup and can replay every pending `delivery-queue/*.json` entry without a stale-age cutoff or replay-count cap for old first-replay entries.

## Best Possible Solution

Implement a conservative outbound recovery policy in `src/infra/outbound`: expire or move stale pending deliveries to `failed/` before replay, preserve crash-window recovery for recent sends, and add focused tests for old `retryCount=0` entries with no `lastAttemptAt`, `maxRecoveryMs` interaction, and restart recovery. If maintainers want operator control, add documented config/schema for max age and possibly a max replay count.
