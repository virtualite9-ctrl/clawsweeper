## Summary

Keep #17810 open. Current main still loads SOUL.md, IDENTITY.md, USER.md, and MEMORY.md as bootstrap/context files, sorts known basenames, and adds fixed SOUL guidance, but it does not implement configurable context-file elevation, section labels, or exclusion from the flat Project Context block. The provided issue context also shows the matching PR #18110 was closed unmerged and the latest maintainer/Codex discussion explicitly kept the issue open.

## Best Possible Solution

Continue tracking #17810 for maintainer product direction. If accepted, implement the feature in the agent prompt/bootstrap path with documented config/schema support, deterministic matching and ordering, elevated sections before Project Context, exclusion from the flat context unless explicitly retained, and focused tests for system-prompt rendering plus config validation/docs. If maintainers prefer externalization, first add or document a public plugin/API surface that can safely restructure bootstrap prompt sections without replacing the whole system prompt.
