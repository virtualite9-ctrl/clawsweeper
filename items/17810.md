## Summary

Keep #17810 open. Current main still gives SOUL.md, IDENTITY.md, USER.md, and MEMORY.md deterministic ordering inside the flat Project Context block rather than configurable elevated prompt sections, and there is no shipped config/schema/docs/test surface for `context.elevation` or an equivalent. The related implementation PR #18110 remains closed unmerged, and the latest issue discussion already recorded the same unresolved product state.

## Best Possible Solution

Continue tracking #17810 for explicit maintainer product direction. If accepted, implement a documented config/schema surface for deterministic context-file elevation before `# Project Context`, preserve existing behavior when unset, exclude elevated files from the flat block unless explicitly retained, and add focused prompt-rendering plus config/docs coverage. If maintainers prefer externalization, first define or document a public plugin/API surface that lets plugins safely restructure bootstrap prompt sections without replacing the whole system prompt.
