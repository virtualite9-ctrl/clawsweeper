## Summary

Keep #17810 open. Current main still gives SOUL.md, IDENTITY.md, USER.md, and MEMORY.md sort priority but renders them as ordinary files under `# Project Context`; there is no `context.elevation`/section config surface, tests still assert flat rendering, docs still describe Project Context injection, and the related implementation PR #18110 is closed unmerged.

## Best Possible Solution

Continue tracking #17810 for maintainer product direction. If accepted in core, add a documented config/schema surface for deterministic context-file elevation before Project Context, preserve current behavior when unset, exclude elevated files from the flat block unless explicitly retained, and cover prompt rendering plus config/docs tests. If maintainers prefer plugin externalization, document the intended context-engine or bootstrap-hook path clearly enough that contributors can implement it without replacing the whole system prompt.
