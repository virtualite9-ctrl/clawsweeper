## Summary

Keep open. Current main partially addresses plugin installation/update, but it does not fully solve the reported discovery and inspection problem: the documented ClawHub CLI surface still presents search/install as skill-oriented and only documents plugin publishing, native `openclaw skills search` is explicitly skills-only, and the live package search endpoint still errors for the reporter's example query. The related install/update issues are narrower and now appear fixed separately, so they do not supersede this remaining ClawHub plugin discovery/docs/server surface issue.

## Best Possible Solution

Keep this issue open and narrow it to the remaining actionable work: document the plugin/package discovery commands in `docs/tools/clawhub.md`, ensure users are directed from plugin docs to the correct package inspect/explore surface, and fix or track the ClawHub package query search path that currently returns server errors for valid plugin names. The already-shipped `openclaw plugins install` and update fixes should be left as separate completed work.
