## Summary

Current `main` still separates ClawHub skill discovery from ClawHub plugin installation. The repo documents `clawhub` as a skills/auth/publish tool, supports plugin install/update through native `openclaw plugins ...`, and contains package-search APIs that are not wired into any user-facing plugin search/inspect CLI. That means the issue’s core gap is still present, so there is not strong evidence for closure.

## Evidence

- **ClawHub docs scope `clawhub` away from plugin install/search flows:** The current ClawHub docs say to use native `openclaw` commands to search/install/update skills and install plugins from ClawHub, while the separate `clawhub` CLI is for auth/publish/delete/sync workflows.
  - file: [docs/tools/clawhub.md:10](https://github.com/openclaw/openclaw/blob/5b8bd6371c666a5de43c82305e0315771e8e49d5/docs/tools/clawhub.md#L10)
