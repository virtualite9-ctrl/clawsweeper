## Summary

Keep open. Current main still matches the reported core scheduling gap: bootstrap maintenance runs through the foreground/default path, deferred debt execution is only enabled for background maintenance, and the only automatic background scheduling path is turn maintenance. The hot-cache policy appears to be owned by the external Lossless Claw context-engine plugin, but OpenClaw still lacks the host scheduling/API contract needed to make bootstrap/reconcile-created debt reliably consumable.

## Best Possible Solution

Keep this issue open. The best path is a narrow core fix that lets bootstrap/reconcile-created deferred compaction debt schedule or run a valid background maintenance pass, or an additive context-engine maintenance contract that lets engines explicitly request a debt-consuming follow-up or report that policy intentionally superseded debt. Add regression coverage for the bootstrap path, then coordinate the hot-cache policy semantics with Lossless Claw so operator-visible debt is either paid down or clearly non-sticky by design.
