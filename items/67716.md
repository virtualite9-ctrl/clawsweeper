## Summary

Keep open. Current main still matches the reported core scheduling gap: bootstrap maintenance runs through the default foreground path, deferred debt execution is only enabled for background maintenance, and the only automatic background scheduling branch is turn maintenance. The hot-cache policy appears plugin-owned, but the host still lacks a clear scheduling/API path for bootstrap/reconcile-created debt.

## Best Possible Solution

Keep this issue open. The best path is a narrow OpenClaw host fix that lets bootstrap/reconcile-created deferred compaction debt schedule or run an allowed background maintenance pass, or an additive context-engine maintenance contract so engines can explicitly request a debt-consuming follow-up or report that policy intentionally superseded debt. Add a bootstrap-path regression test and coordinate hot-cache semantics with the Lossless Claw plugin.
