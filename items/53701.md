## Summary

Current `main` does not implement human-readable Telegram forum topic names in the Control UI session selector. Telegram topic names are cached into inbound context metadata, but session labels/display names are still derived from chat title, raw topic id, or slugified group metadata, and the Telegram topic config has no `name` field fallback.

## Evidence

- **Telegram caches topic names but only as inbound metadata:** Telegram forum topic names are collected from `forum_topic_*` service messages and stored as `TopicName` on the inbound context, which shows the plumbing exists but stops at message metadata.
  - file: [extensions/telegram/src/bot-message-context.ts:165](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/extensions/telegram/src/bot-message-context.ts#L165)
  - command: `nl -ba extensions/telegram/src/bot-message-context.ts | sed -n '157,204p'`
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Conversation label still uses raw topic id:** Group conversation labels come from `buildGroupLabel`, which formats `"<chat title> id:<chatId> topic:<messageThreadId>"`; it does not use the cached topic name.
  - file: [extensions/telegram/src/bot/helpers.ts:358](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/extensions/telegram/src/bot/helpers.ts#L358)
  - command: `nl -ba extensions/telegram/src/bot/helpers.ts | sed -n '352,372p'`
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Session metadata persists slugified display names, not topic names:** When session metadata is recorded, `displayName` is set from `buildGroupDisplayName(...)`, and that helper slugifies subject/channel/id into values like `telegram:g-...` rather than a human-readable topic title.
  - file: [src/config/sessions/metadata.ts:166](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/config/sessions/metadata.ts#L166)
  - command: `nl -ba src/config/sessions/metadata.ts | sed -n '160,178p'`
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Gateway session rows feed the UI from displayName/label only:** The gateway session row exposes `displayName` from persisted session metadata or `buildGroupDisplayName(...)`, and the Control UI resolves selector labels from `label` or `displayName`. There is no path here that reads Telegram `TopicName`.
  - file: [src/gateway/session-utils.ts:1256](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/gateway/session-utils.ts#L1256)

  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **No config fallback field for per-topic names:** `TelegramTopicConfig` includes routing and policy fields, but no `name` property, so the issue's proposed config fallback is also not on `main`.
  - file: [src/config/types.telegram.ts:232](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/config/types.telegram.ts#L232)
