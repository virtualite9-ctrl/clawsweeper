## Summary

Keep #7679 open. Current main still defaults Telegram DMs to pairing, still documents/tests pairing as the default path, and Telegram setup only supports manual numeric allowFrom entry rather than token-verified owner ID capture during onboarding. The provided maintainer-context comment by steipete already kept this issue open, and the current relevant source has not changed since that review.

## Best Possible Solution

Keep #7679 open. The maintainer path should implement Telegram onboarding that generates a short verification token, asks the intended owner to send that exact token to the bot, captures the matching sender's numeric Telegram user ID, writes dmPolicy="allowlist" plus allowFrom for that account, and updates schema/runtime/docs/tests with compatibility for existing pairing-mode configurations.
