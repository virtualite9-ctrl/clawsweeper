## Summary

Current main already satisfies the requested safe path: Kimi Coding still defaults to the Anthropic-compatible coding endpoint, but explicit `models.providers.kimi-coding.api: "openai-completions"` is preserved by the provider merge and used by runtime model resolution, so `/coding/v1` can be routed through the OpenAI-compatible transport.

## Best Possible Solution

Close as implemented. Keep Kimi Coding's default Anthropic-compatible endpoint for the built-in provider, and use the existing explicit `models.providers.<kimi>.api` override for Kimi's OpenAI-compatible `/coding/v1` endpoint; a future Kimi-specific regression test or auto-detection nicety can be handled separately if maintainers want it.
