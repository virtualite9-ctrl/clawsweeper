## Summary

Keep open. Current main ships the SecretRef foundation for env/file/exec and documents 1Password through an exec-provider recipe, but it still does not implement native keyring/Keychain or native 1Password provider sources. The issue also has a same-day maintainer Codex review by steipete explicitly keeping it open for that remaining scope, and the provided context does not identify a narrower canonical follow-up that fully replaces it.

## Best Possible Solution

Keep #17311 open, but narrow it to the remaining native keyring/Keychain and native 1Password provider design. A complete implementation would add those sources across the SecretRef source contract, config schemas, resolver runtime, Gateway protocol schema, CLI/setup flows, plugin/setup helpers, docs, and tests. If env/file/exec plus documented exec recipes are intended to be final, maintainers should record that product decision and open focused follow-ups for any remaining macOS operator pain before closing this broad issue.
