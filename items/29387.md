## Summary

Keep #29387 open. Current main still resolves bootstrap injection from the workspace directory only and has no `agentDir` merge/override or ignored-file warning. Docs now describe per-agent workspaces and `agentDir` as state, but that is not the requested runtime diagnostic or agentDir bootstrap behavior. The related attempted fixes #29450 and #29545 are closed/unmerged in the provided GitHub context, so they do not supersede the issue.

## Best Possible Solution

Keep #29387 open as the canonical bug. The best fix is either a narrow diagnostic that threads explicit agentDir/agentId context through the resolver callsites and warns when recognized bootstrap filenames exist in an explicitly configured `agentDir` but are not injected, or a deliberate agentDir merge/override contract with allowlist behavior, hook precedence, caller coverage, regression tests across embedded/CLI/compaction paths, and docs explaining workspace versus agentDir semantics.
