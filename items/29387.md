## Summary

Keep #29387 open. Current main still resolves prompt bootstrap files exclusively from the effective workspace directory. `agentDir` remains a separate per-agent state/auth/session directory, and the bootstrap resolver/callsites still do not inspect it, merge from it, or warn when bootstrap-named files are present there. The linked attempted fixes (#29450 and #29545) are closed unmerged in the provided GitHub context and their head commits are not present in this checkout.

## Best Possible Solution

Keep #29387 open as the canonical bug. The best maintainer path is either a narrow diagnostic that threads explicit `agentDir` context into bootstrap callsites and warns on recognized bootstrap filenames in `agentDir`, or a deliberate per-agent bootstrap merge/override contract for explicitly configured `agentDir` files with allowlist filtering, hook precedence, caller coverage, regression tests, and docs that clearly distinguish workspace bootstrap files from agentDir state.
