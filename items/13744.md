## Summary

Keep #13744 open. Current main partially fixes the error-classification part by using `SessionWriteLockTimeoutError` and excluding it from model-timeout failover, but the main requests remain unresolved: there is still no public `openclaw.json` or `OPENCLAW_*` surface for session write-lock `timeoutMs`/`staleMs`, and the embedded run still holds the session write lock across session preparation, prompt/model execution, and teardown.

## Best Possible Solution

Keep #13744 open until OpenClaw adds a supported configuration surface for session write-lock `timeoutMs` and `staleMs`, wires it into embedded run and transcript mutation paths, and either narrows the lock to actual transcript critical sections or documents the broad per-session lifecycle lock as intentional. Preserve the existing `SessionWriteLockTimeoutError` failover suppression and add focused contention/config coverage for concurrent same-session callers.
