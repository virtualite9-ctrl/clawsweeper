## Summary

Keep #13744 open. Current main has typed session-write-lock timeout errors and failover suppression, but the main requested behavior is still missing: embedded attempts acquire the session file lock near the start of the run without timeout/stale config, release it only in final cleanup, and expose no supported openclaw.json/env surface for sessionWriteLock timeoutMs/staleMs.

## Best Possible Solution

Keep #13744 open until OpenClaw adds a supported openclaw.json and/or OPENCLAW_* surface for session write lock timeoutMs and staleMs, wires it into embedded run/session mutation paths, and either narrows the file-lock critical section to transcript mutation phases or explicitly documents broad per-session lifecycle locking as intentional. Preserve the existing SessionWriteLockTimeoutError failover suppression and add focused contention tests covering concurrent same-session callers.
