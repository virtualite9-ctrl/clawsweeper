## Summary

Keep #13744 open. Current main has some adjacent fixes for typed session write-lock timeout errors and failover suppression, but the main request remains unresolved: no supported openclaw.json/env surface for session write-lock timeoutMs/staleMs, and the embedded attempt still holds the session file lock across the broader run lifecycle instead of only transcript mutation sections. The existing maintainer/Codex comment in the issue is still accurate against current main 7e13f3f5140168e31027b39a9855ade3fadf24d5.

## Best Possible Solution

Keep #13744 open until OpenClaw adds a supported openclaw.json and/or OPENCLAW_* surface for session write-lock timeoutMs and staleMs, wires those values into embedded run and transcript mutation paths, and either narrows the file-lock critical section to actual transcript read/repair/prewarm/append/rewrite work or explicitly documents the broad per-session lifecycle lock as intentional. Preserve the existing SessionWriteLockTimeoutError failover suppression and add focused contention/config plumbing tests for concurrent same-session callers.
