## Summary

Keep PR #60860 open. Current main has partial Google Vertex awareness, but it does not implement the PR's central native Vertex behavior: a distinct `google-vertex` provider registration, ADC token exchange, or Vertex `/projects/{project}/locations/{location}/publishers/google/models/...` request routing. The linked issues #58775 and #60736 still describe a real remaining product gap rather than cleanup noise.

## Best Possible Solution

Keep the PR open or explicitly supersede it with a maintainer-owned Google Vertex implementation. The best code path is to make `google-vertex` a real Google plugin provider with manifest metadata, ADC/service-account token exchange, region/project resolution, Vertex request URL construction, and regression tests covering the linked auth/routing failures; then close #58775 and #60736 through that canonical fix.
