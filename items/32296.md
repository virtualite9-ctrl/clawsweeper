## Summary

Keep #32296 open. Current main contains several adjacent hardenings for WebChat routing, queued active turns, orphaned user-turn repair, and prompt current-message selection, but I did not find high-confidence evidence that the broad cross-channel “agent replies to the previous message” symptom is fixed. The issue discussion also includes a recent report that it still reproduced on origin/main after the narrower fixes shipped.

## Best Possible Solution

Keep #32296 as the canonical tracker for remaining current-message/session-context confusion. Ask affected users for current-main session JSONL, gateway logs, channel/provider, queue mode, and transcript shape; then close or cross-link narrower fixed variants such as #31573 and #65388 while adding regression coverage around shared queued reply/session-runner paths for concurrent, steered, compacted, and fallback turns.
