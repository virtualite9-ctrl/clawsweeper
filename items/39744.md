## Summary

Current main already implements real-time Control UI/WebChat chat updates, and the same behavior is present in the shipped release v2026.4.22.

## Close Comment

Closing from a Codex review because current `main` already has the real-time WebChat/Control UI update path this report asks for. User messages are appended immediately in `ui/src/ui/controllers/chat.ts:325`, assistant websocket `delta`/`final` events update the visible chat in `ui/src/ui/controllers/chat.ts:451` and `ui/src/ui/controllers/chat.ts:456`, and the gateway event layer auto-reloads transcript state on terminal chat events and `session.message` in `ui/src/ui/app-gateway.ts:472` and `ui/src/ui/app-gateway.ts:481`. There is regression coverage in `ui/src/ui/app-gateway.node.test.ts:622` and `ui/src/ui/app-gateway.node.test.ts:860`, and this behavior is already present in the shipped release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`). If you still hit refresh-only behavior on a newer build, please open a fresh issue with the exact browser/version, websocket logs, and whether it only happens after a disconnect/reconnect.
