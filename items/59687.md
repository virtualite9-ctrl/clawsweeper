## Summary

Keep open. Current `main` appears to have improved the duplicate-suppression side for Slack DMs via provider-normalized target matching, but the Slack native-streaming path still forwards any text payload without filtering `isReasoning`, so the reasoning-leak half of this issue is not implemented on `main`.

## Evidence

- **Generic suppression now normalizes providers and targets:** `shouldSuppressMessagingToolReplies()` normalizes the provider, origin target, and tool target, then compares them via `targetsMatchForSuppression()`. This is broader than the issue's claim that only Telegram got thread-aware comparison, and it is the current main implementation I reviewed.
