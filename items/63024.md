## Summary

Keep this issue open. Current main has a related Gateway/plugin-runtime-deps fix for staged `tslog`, but the reported failure is the pnpm-global `openclaw onboard --install-daemon` startup path. That path still imports the logging barrel before command dispatch, the logger still bare-imports `tslog`, and the supported docs still include Node 24 + pnpm global onboarding. I found no exact packaged pnpm-global Node 24 `--install-daemon` proof on current main.

## Best Possible Solution

Keep this issue open until maintainers either add a startup/package fix that makes classic CLI startup independent of bare `tslog` resolution, or produce/retain exact proof that a current packaged pnpm-global install on Node 24 can run `openclaw onboard --install-daemon` without the reported `tslog` import failure. The related #72228 staged-runtime-deps fix should stay, but it is not sufficient closure evidence for this issue.
