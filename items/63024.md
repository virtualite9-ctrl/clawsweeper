## Summary

Keep this issue open. The report is a concrete, recent Linux/pnpm global startup regression on a documented Node 24 path, and current main still routes classic CLI startup through the logging barrel and a bare `tslog` runtime dependency before `onboard --install-daemon` can run. The latest published tarball still has the same bare `tslog` import shape, so there is not enough evidence to close as implemented or obsolete.

## Best Possible Solution

Keep this issue open and verify with a packaged pnpm global install on Node 24. If it reproduces, fix the CLI startup/logging packaging path so classic onboarding does not fail on fragile `tslog` resolution before command dispatch, either by bundling or importing the logger dependency in a resolver-safe way, and add packaged install/startup smoke coverage for this path.
