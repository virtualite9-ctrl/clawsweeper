## Summary

Keep this issue open. It is a recent, concrete packaged-startup regression, and current main still routes classic `openclaw onboard --install-daemon` startup through the logging barrel and a bare `tslog` runtime dependency before command dispatch. A related open PR (#72493) is working on core logger runtime-dependency staging for #72228, but it is not merged and does not yet prove the pnpm-global onboarding path in this issue is fixed.

## Best Possible Solution

Keep this issue open. The best maintainer path is to run a packaged pnpm-global install smoke on Node 24 for `openclaw onboard --install-daemon`, then fix the CLI startup/logging packaging path so the classic onboarding command cannot fail on fragile `tslog` resolution before command dispatch. If #72493 is reviewed, merged, and proven to cover this exact path, close this afterward as implemented; otherwise add a focused packaging/import fix and packaged startup regression coverage.
