## Summary

Keep open. Current `main` still does not implement `env:DISCORD_BACKUPS_CHANNEL` delivery targets for cron. Docs only document `${VAR_NAME}` config substitution and explicit Discord delivery target formats, while cron preserves raw `delivery.to` values and its message-tool verification path only normalizes channel-specific target syntax, not `env:` indirections. That means the reported `ok` content generation plus failed cron delivery accounting remains plausible on `main`.

## Evidence

  - file: [docs/help/environment.md:77](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/docs/help/environment.md#L77)

This points to `${DISCORD_BACKUPS_CHANNEL}` as the supported pattern, not `env:DISCORD_BACKUPS_CHANNEL`.
