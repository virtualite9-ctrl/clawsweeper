## Summary

Keep this PR open. Current main still lacks the proposed macOS LaunchAgent reinstall behavior: gateway service proxy variables are copied from the current install environment, and the launchd install path writes the new plist without merging proxy keys from the existing plist. The PR is a focused daemon/docs/test change and not obsolete, but the posted lowercase-proxy precedence review is real and should be fixed before merge.

## Best Possible Solution

Keep the PR open and ask for a focused revision: merge existing LaunchAgent proxy vars only after normalizing proxy-key families, ensure explicit current overrides clear conflicting stale variants, add regression tests for lowercase-vs-uppercase precedence and clearing behavior, and keep the troubleshooting doc note if the implementation lands.
