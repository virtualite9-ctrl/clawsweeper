## Summary

Keep open. Current main still contains the reported Gemini CLI OAuth client-config extraction path and still uses it as the fallback when explicit OAuth client environment variables are absent. The related open PR improves diagnostics only; it does not remove or gate the extraction behavior.

## Best Possible Solution

Keep this issue open for maintainer security/product review. The likely fix belongs in the bundled Google plugin: remove automatic Gemini CLI source/bundle credential extraction by default, or require an explicit extraction-specific opt-in before filesystem discovery. Update extensions/google/oauth.credentials.ts, extensions/google/gemini-cli-provider.ts, Gemini OAuth tests, and public Google/Gemini provider docs so the behavior and consent model are explicit; review #41991 against that direction before merge.
