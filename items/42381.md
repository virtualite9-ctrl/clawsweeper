## Summary

Keep open. Current main still lacks a remote node-host guard for `openclaw models auth login`, while provider auth results are persisted to the local resolved agent auth store. This PR remains a plausible implementation candidate for the linked open bug, but needs rebase/maintainer review before landing.

## Best Possible Solution

Keep this PR open, rebase it onto current main, and review/land a focused remote node-host guard in `src/commands/models/auth.ts` with regression coverage in `src/commands/models/auth.test.ts`. Prefer reusing the shared `src/gateway/net.ts:isLoopbackHost` semantics and getting maintainer sign-off on whether the guard applies to all auth methods or only OAuth-like flows. After merge, #42291 can close as implemented.
