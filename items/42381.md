## Summary

Keep this PR open. Current main still lets `openclaw models auth login` run provider auth and persist returned OAuth profiles to the resolved local agent auth store without checking whether the CLI is running on a node host pointed at a remote gateway. The linked bug remains open, and this PR is still a plausible focused implementation candidate, but it needs maintainer review before landing.

## Best Possible Solution

Keep this PR open, rebase it onto current main, and review/land a focused remote node-host guard in `src/commands/models/auth.ts` with regression coverage in `src/commands/models/auth.test.ts`. Prefer shared `src/gateway/net.ts:isLoopbackHost` semantics and maintainer confirmation of the auth-method scope. After it lands, #42291 can close as implemented.
