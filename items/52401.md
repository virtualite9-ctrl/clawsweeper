## Summary

Keep open. Current `main` does not implement the requested `agent_error` hook, so there is nothing to close as already shipped. The PR is still substantive work, and its current design also touches a timing-sensitive terminal lifecycle path by awaiting plugin code inside `handleAgentEnd` before compaction-resolution and terminal lifecycle emission.

## Risks / Open Questions

- Closing this PR as implemented-on-main would be incorrect because `main` still has no `agent_error` hook contract or runner.
- The PR's current await inside `handleAgentEnd` can delay terminal lifecycle error emission and compaction resolution, which is risky given the fallback logic in `src/agents/agent-command.ts` and the compaction-wait invariants already covered in lifecycle tests.
