## Summary

Keep #7524 open. Current main still has only DM-scoped consolidation (`dmScope`) and continues to route group/channel peers to isolated peer session keys. The requested per-group route-to-main behavior is not present in the core routing contract, strict config schemas, route-binding overrides, or user-facing docs. The broader #19929 remains open but is not clearly canonical, and the latest maintainer-facing discussion explicitly kept #7524 open.

## Best Possible Solution

Keep #7524 open as the focused tracker unless a maintainer explicitly supersedes it with #19929. The best implementation is an additive core routing/config contract for selective group/channel/thread-to-main or named shared sessions, wired through `buildAgentPeerSessionKey()`, `buildAgentSessionKey()`, `resolveAgentRoute()`, route binding overrides, strict schemas/types/help/generated metadata, public docs, and regression coverage for BlueBubbles, Signal, Telegram, and broader group/channel routing.
