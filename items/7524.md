## Summary

Keep #7524 open. Current main still only supports `dmScope` for direct messages; group/channel peers always resolve to isolated non-main session keys, strict config schemas and route-binding overrides have no `groupScope` or equivalent, and public docs still describe groups/channels as isolated. Related #19929 is broader and open, but the provided discussion does not explicitly supersede this focused issue.

## Best Possible Solution

Keep #7524 open as the focused tracker unless a maintainer explicitly designates #19929 as canonical. The best implementation is an additive core routing/config contract for selective group/channel/thread-to-main or named shared sessions, wired through `buildAgentPeerSessionKey()`, `buildAgentSessionKey()`, `resolveAgentRoute()`, route binding overrides, strict schemas/types/help/generated metadata, public docs, and regression coverage for BlueBubbles, Signal, Telegram, Matrix, and generic group/channel routing.
