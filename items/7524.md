## Summary

Keep #7524 open. Current main still only supports `dmScope` for direct peers; group/channel peers fall through to isolated `agent:<agentId>:<channel>:<kind>:<peerId>` keys. The strict global/session binding schemas and public docs still expose no `groupScope`, group-to-main, or shared-session mapping. The related #19929 is broader and open, but the provided discussion does not establish it as the canonical replacement for this focused BlueBubbles/Signal group-to-main request.

## Best Possible Solution

Keep #7524 open as the focused tracker unless maintainers explicitly mark #19929 canonical. The best implementation is an additive routing/config contract for selective group/channel/thread-to-main or named shared sessions, wired through `buildAgentPeerSessionKey()`, `buildAgentSessionKey()`, `resolveAgentRoute()`, strict config schemas/types/help/generated metadata, relevant channel group configs, public docs, and regression coverage for BlueBubbles, Signal, Telegram, Matrix, Discord/Slack channel/thread routing, and generic non-direct session keys.
