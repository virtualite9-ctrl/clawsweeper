## Summary

Keep this PR open. Current main does not implement `gateway.bind: "netbird"`, does not have NetBird interface detection, and still documents Tailscale/explicit TLS workarounds for secure remote Control UI access. The PR is a plausible focused implementation for the linked NetBird request, with active user interest and no protected-label or maintainer-authored auto-close blocker; it needs maintainer review and a refresh against current main rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review and author refresh. If OpenClaw wants first-class NetBird support, implement it in core gateway/config/docs/tests because current plugin surfaces do not expose gateway bind-mode extension: update the bind enum/schema/help, NetBird interface detection, bind resolver, connection details, Control UI links, TLS runtime/fingerprint/status probes, onboarding/config flows, docs, and regression tests. If maintainers prefer not to add service-specific bind modes, open or redirect to a deliberate gateway bind extension/API design instead of closing this focused implementation as obsolete.
