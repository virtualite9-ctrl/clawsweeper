## Summary

Current main implements the startup-timeout path reported here. The same reporter's broader #68781 thread covered the identical `history failed: Error: gateway request timeout for chat.history` symptom and was closed after verifying the startup fixes. Source now has a non-plugin secrets preflight fast path, startup-gated `chat.history` returns retryable `UNAVAILABLE`, sidecars re-enable the method after startup, and the TUI preserves/retries that retry metadata.

## Best Possible Solution

Close this issue as implemented. Keep the current startup fast path, startup-gated RPC behavior, and TUI retry handling; any remaining post-v2026.4.24 reproduction should be filed as a fresh, narrower issue with startup trace, `gateway status --deep`, exact versions, and whether the gateway process was restarted after upgrade.
