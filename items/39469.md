## Summary

Current `main` prevents the duplicate same-turn `delivery-mirror` append that caused Control UI assistant replies to be recorded twice in session JSONL. The fix is generic in transcript append logic, trims visible text before comparison, has regression coverage, and is documented in the `2026.4.15` changelog.

## Evidence

- **Transcript append now dedupes redundant delivery-mirror writes:** Before appending a `provider:"openclaw", model:"delivery-mirror"` assistant message, the transcript writer checks the latest assistant entry and reuses it when the visible text matches. The extracted text is trimmed, so whitespace-only differences like the report’s leading `\n\n` do not create a second row.
  - file: [src/config/sessions/transcript.ts:204](https://github.com/openclaw/openclaw/blob/cb2c36b04975e34ab7014a11575d6589aa96ae32/src/config/sessions/transcript.ts#L204)
  - command: `nl -ba src/config/sessions/transcript.ts | sed -n '170,320p'`
- **Regression test proves no second JSONL assistant row is written:** The test appends a normal assistant message, then calls `appendAssistantMessageToSessionTranscript()` with the same text and asserts the returned message id is the original one and the transcript still has only two lines total (header + one assistant message), not a new `delivery-mirror` row.
  - file: [src/config/sessions/transcript.test.ts:149](https://github.com/openclaw/openclaw/blob/cb2c36b04975e34ab7014a11575d6589aa96ae32/src/config/sessions/transcript.test.ts#L149)
  - command: `nl -ba src/config/sessions/transcript.test.ts | sed -n '145,230p'`
- **Shipped in changelog release entry:** The changelog for `2026.4.15` states: `Sessions/Codex: skip redundant delivery-mirror transcript appends only when the latest assistant message has the same visible text, preventing duplicate visible replies on Codex-backed turns without suppressing repeated answers across turns.`
  - file: [CHANGELOG.md:579](https://github.com/openclaw/openclaw/blob/cb2c36b04975e34ab7014a11575d6589aa96ae32/CHANGELOG.md#L579)
  - command: `nl -ba CHANGELOG.md | sed -n '570,585p'`

## Risks / Open Questions

- Historical session JSONL files created before the fix can still contain old duplicate `delivery-mirror` rows; the change prevents new duplicates but does not rewrite existing transcripts.
- Broader transcript/history leakage work may still exist in separate issues, but the specific duplicate same-turn session-JSONL recording described here is implemented on current `main`.

## Close Comment

Closing after Codex review: current `main` already implements the fix for this specific bug. `src/config/sessions/transcript.ts:204` now suppresses a redundant `delivery-mirror` append when the latest assistant message already has the same visible text, and the comparison trims text first (`src/config/sessions/transcript.ts:267`), so the whitespace-only variant shown in this report is covered. The regression test at `src/config/sessions/transcript.test.ts:149` verifies that the mirror write reuses the original assistant message id and does not add a second assistant row to the JSONL. This was shipped in `v2026.4.15` (`CHANGELOG.md:579`). Historical transcripts created before that release can still contain old duplicate rows, but current `main` no longer double-records new turns in the way reported here.
