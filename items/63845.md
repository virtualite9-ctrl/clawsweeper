## Summary

Keep this issue open. Current main has relevant mechanisms for inspecting context cost, pruning old tool results, compaction, skill prompt budgets, tool allow/profiles, local-model lean mode, and workflow/plugin aggregation, but it still does not answer the maintainer guidance being requested: tested skill/tool-count ceilings, SKILL.md trigger collision practices for 65+ domain skills, recommended compaction/pruning settings for high-count tool sessions, low-end model reference configurations, or whether hybrid smart aggregation tools are a recommended pattern.

## Best Possible Solution

Keep this open for maintainer guidance or convert it into a focused docs task. The best path is a practical guide under Context, Tools/plugins, Skills, or Local models that states whether any tested tool/skill-count ceiling exists; teaches `/context detail`, `tools.profile`, `tools.allow`/`deny`, `skills.limits`, `disable-model-invocation`, pruning, compaction, and local-model lean mode for tool-heavy sessions; gives small/medium local-model guidance; and explicitly describes when a hybrid domain aggregation plugin/workflow tool is acceptable, including traceability, output caps, and follow-up composability.
