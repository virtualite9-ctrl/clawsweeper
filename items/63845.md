## Summary

Keep #63845 open. Current main has several relevant mechanisms and docs for high-skill/high-tool setups, including skill allowlists, bounded skill prompt catalogs, optional plugin tools, workflow-style single-call patterns, tool-result truncation, pruning, compaction, and general local-model cautions. But it does not yet provide the specific maintainer guidance the issue asks for: a tested tool/skill-count ceiling, high-count SKILL.md trigger/routing best practices, recommended compaction/pruning values for 10+ tool-call turns, or low-end model reference configurations. The issue is recent, specific, user-authored, unlabeled, and not superseded by the provided related items, so none of the conservative close reasons apply.

## Best Possible Solution

Keep the issue open for a maintainer-authored best-practices answer or docs update. The most useful path is a short guide covering high-count skill/tool design: per-agent skill segmentation, SKILL.md description hygiene, when to make tools optional or aggregate them behind one plugin tool/workflow, recommended pruning/compaction/tool-result caps for large tool outputs, and low-end local-model constraints or reference configs. The guide should point to the existing plugin, Lobster/llm-task, skills limits, session pruning, compaction, and loop-detection docs rather than adding new core behavior unless a missing plugin API is identified.
