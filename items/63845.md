## Summary

Keep this issue open. Current main has relevant building blocks and partial documentation for context inspection, pruning, compaction, skill prompt budgets, local-model lean mode, plugin tools, and workflow-style aggregation, but it still does not provide the requested maintainer guidance on tested tool/skill-count ceilings, SKILL.md trigger best practices, recommended settings for high-count tool sessions, low-end model reference configurations, or whether hybrid smart aggregation tools are a recommended pattern.

## Best Possible Solution

Keep this open for maintainer guidance or convert it into a focused docs task. The strongest path is a new or expanded docs section under Context, Tools/plugins, Skills, or Local models that states there is no fixed tested tool-count ceiling unless maintainers can provide one; documents how to inspect prompt cost with `/context detail`; explains `tools.profile`, `tools.allow`/`deny`, `skills.limits`, `disable-model-invocation`, context pruning, and compaction knobs for tool-heavy sessions; gives local-model guidance for small/medium models; and explicitly describes when a hybrid domain aggregation tool or workflow-backed plugin is acceptable and how to preserve traceability and bounded output.
