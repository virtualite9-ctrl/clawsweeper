## Summary

Keep this issue open. Current main has useful context, skill, tool-policy, compaction, pruning, local-model, plugin, and workflow controls, but it still does not provide the maintainer guidance requested: tested high-count tool/skill ceilings, SKILL.md trigger-collision practices, workload-specific compaction/pruning defaults, low-end local-model reference profiles, or a clear recommendation on hybrid server-side aggregation tools.

## Best Possible Solution

Keep open for maintainer guidance or convert it into a focused docs task. The best outcome is a practical guide under Context, Skills, Tools/plugins, or Local models that states whether tested tool/skill-count ceilings exist; gives SKILL.md trigger-collision practices; recommends /context detail, skills.limits, agents.defaults.skills, tools.profile, tools.allow/deny, pruning, compaction, tool-result caps, and localModelLean patterns for high-count tool sessions; and explicitly describes when a hybrid domain aggregation plugin/workflow tool is acceptable, including traceability, output caps, latency, and composability tradeoffs.
