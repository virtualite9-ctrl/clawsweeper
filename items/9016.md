## Summary

Keep #9016 open. Current main has estimated cost reporting and transcript aggregation, but it still does not expose OpenRouter's provider-reported per-response cost to agent runtime/session metadata. The OpenRouter provider uses the shared OpenAI-compatible transport, whose usage parser reads token counts, sets cacheWrite to 0, computes local model-pricing cost, and ignores OpenRouter numeric usage.cost, cost_details.upstream_inference_cost, and cache_write_tokens. The linked ClawRouter workaround is useful context, and #9696 covered estimates, but neither closes this exact runtime metadata gap.

## Best Possible Solution

Keep #9016 open. The best implementation is to capture OpenRouter raw usage at the shared OpenAI-compatible response/stream boundary, preserve provider-reported cost separately from OpenClaw's local pricing estimate with source/units, include cost_details.upstream_inference_cost and prompt_tokens_details.cache_write_tokens, carry those fields through transcript/session usage and EmbeddedPiAgentMeta, and expose per-turn plus cumulative values through session_status or a dedicated usage/cost tool. Telegram and other reply footers can render exact OpenRouter cost when available and fall back to existing estimates otherwise.
