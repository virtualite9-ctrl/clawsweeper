## Summary

Keep open. Current main does not yet provide the requested user-facing `plugins.entries.memory-core.config.dreaming.model` override. The memory-core plugin schema still keeps `dreaming` closed without a `model` key, docs still describe Dream Diary generation as using the default runtime model, and the dreaming narrative runtime calls do not pass a model even though the generic plugin subagent API supports one. The related closed #64360 appears to be the duplicate, while #63372 remains the canonical tracker with concrete user cost/model-separation use cases in the comments.

## Best Possible Solution

Keep #63372 open and implement one public, backward-compatible memory-core dreaming model override path. The likely product path is to accept the requested `plugins.entries.memory-core.config.dreaming.model` as the sweep-wide default, update `extensions/memory-core/openclaw.plugin.json`, config docs/help, and resolver tests, then thread the resolved model into all dreaming narrative subagent runs while preserving fallback to the agent primary when unset. If maintainers prefer the newer `dreaming.execution.defaults.model` shape, make that explicit in schema/docs and tests, and ensure memory-core actually forwards it through the trusted plugin subagent model-override path.
