## Summary

Current `main` no longer routes Telegram forum topics through a `:thread:` child ACP session key. Telegram group topics now resolve to canonical group-topic session keys, and a regression test covers that exact route, so the reported `ACP_SESSION_INIT_FAILED` path is obsolete on `main` even though `readAcpSessionEntry()` itself still does exact-key lookup.

## Evidence

- **Telegram group topics now use canonical group-topic session keys:** `resolveTelegramOutboundSessionRoute()` builds the peer id as `<chatId>:topic:<topicId>` for group chats with a topic and returns the base route directly for groups, avoiding a `:thread:` child alias for Telegram forum topics.
  - file: [extensions/telegram/src/channel.ts:463](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/extensions/telegram/src/channel.ts#L463)
  - command: `nl -ba extensions/telegram/src/channel.ts | sed -n '448,505p'`
  - sha: [6bc0dc8fb671](https://github.com/openclaw/openclaw/commit/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440)
- **Regression test matches the reported Telegram-topic shape:** The current test explicitly asserts that target `-100:topic:99` resolves to session key `agent:main:telegram:group:-100:topic:99` with the same value as `baseSessionKey`, i.e. no ACP thread child key is involved.
  - file: [extensions/telegram/src/session-route.test.ts:49](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/extensions/telegram/src/session-route.test.ts#L49)

- No released tag could be tied to this change from the available evidence; the latest provided release (`v2026.4.23`) predates the proving commit.
