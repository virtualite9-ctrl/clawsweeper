## Summary

Keep this issue open. Current main still converts blank iMessage media-only sends into visible <media:image> / <media:${kind}> text and sends that string as the outbound RPC text field. The related fix PRs are closed unmerged, and the latest release tag still contains the same behavior.

## Best Possible Solution

Keep this issue open. The best fix belongs in the iMessage plugin send path: stop synthesizing visible <media:...> fallback text for outbound media-only sends, send an empty string plus file to the iMessage RPC when media exists, preserve captions and reply_to metadata for text+media sends, add or update direct send tests for blank text plus media, and update deliver/echo/self-chat tests so dedupe no longer depends on user-visible placeholder text.
