## Summary

Keep #40252 open. At current `origin/main` SHA `09a635a28b2b0029ef01cb7224eb16e75c3a9a1e`, the legacy iMessage plugin still synthesizes `<media:image>` / `<media:${kind}>` for blank media-only sends and sends that value as the JSON-RPC `text` field alongside `file`. The two related fix PRs in the supplied issue context (#40341 and #46343) are closed unmerged, and current tests still encode placeholder-based echo/cache behavior.

## Best Possible Solution

Keep the issue open and fix the legacy iMessage plugin send path in `extensions/imessage/src/send.ts` so media-only sends pass empty text plus `file`, while preserving text+media captions, reply IDs, directive stripping for real text, and the empty-payload guard. Add focused regression coverage for `sendMessageIMessage()` and media-only reply delivery to assert image/audio/video media-only sends do not use `<media:image>` or similar placeholders as visible text.
