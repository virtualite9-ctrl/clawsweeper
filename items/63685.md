## Summary

Keep open. Current main supports the corrected Google Gemma 4 model id `gemma-4-26b-a4b-it`, but the reported failing id `gemma-4-26b` is still accepted as a Gemma model and sent to Google unchanged. The related PR is open and adjacent, not a shipped fix for this setup/model-id failure.

## Best Possible Solution

Keep this issue open and fix the setup/model-id path directly: confirm whether terminal onboarding or model selection can offer `gemma-4-26b`, remove or correct that choice to `gemma-4-26b-a4b-it`, or add provider-side validation with a friendly error before dispatching to Google. Add regression coverage for the reported `gemma-4-26b` path, separate from the already-covered corrected Gemma 4 id. Review #68996 separately for the native-route concern, but do not treat it as a shipped fix for this issue until merged and verified.
