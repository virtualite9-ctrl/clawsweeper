## Summary

Keep open. Current main still has the runtime mutual-exclusion behavior, but it does not include the model-facing schema descriptions or session-tool docs guidance proposed by PR #63339. The related PR #63556 is complementary high-level tool-description work, not a canonical replacement for this schema/docs change.

## Best Possible Solution

Keep PR #63339 open or fold it into a maintainer-owned combined change that adds schema field descriptions for `sessionKey`, `label`, and `agentId`, updates `docs/concepts/session-tool.md`, and optionally coordinates with #63556 for the higher-level tool description. The current main behavior supports the guidance, but the guidance itself is still missing.
