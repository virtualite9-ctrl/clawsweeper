## Summary

Current `main` already implements the local backend shared-auth pairing bypass that this report says is missing, and the `sessions_spawn`/`sessions_send` tool paths route through that implementation.

## Evidence

- **Gateway calls from agent-side tools identify as backend gateway clients by default:** `callGateway()` defaults non-CLI callers to backend mode with client name `gateway-client`, which is the identity the gateway bypass checks for.
  - file: [src/gateway/call.ts:621](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/gateway/call.ts#L621)

- **Gateway handshake explicitly skips pairing for direct-local backend shared-auth connects:** `shouldSkipLocalBackendSelfPairing()` returns true for backend `gateway-client` connections that are `direct_local`, have no browser Origin header, and authenticated with token/password or device token.
  - file: [src/gateway/server/ws-connection/handshake-auth-helpers.ts:249](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/gateway/server/ws-connection/handshake-auth-helpers.ts#L249)
