## Summary

Current `main` still does not implement alerting or persistent-state tracking for repeated `skipped` cron runs. In `applyJobResult`, only `result.status === "error"` increments the alert counter and evaluates `resolveFailureAlert`; all non-error outcomes, including `"skipped"`, fall into the reset branch that clears `consecutiveErrors` and `lastFailureAlertAtMs`. The cron job state/schema on `main` also has no separate skip counter/timestamp fields, and a repo-wide search finds no `skipAlert`, `includeSkipped`, `consecutiveSkips`, or `lastSkipAlertAtMs` support. This issue is recent and concrete, so it should stay open rather than be closed as stale or already fixed.

## Evidence

- **Failure alerts only run on error status:** `applyJobResult` increments `consecutiveErrors`, resolves `failureAlert`, and emits alerts only inside the `result.status === "error"` branch. The following `else` branch resets the alert state for all non-error outcomes, so `"skipped"` does not alert and also clears prior failure-alert state.
