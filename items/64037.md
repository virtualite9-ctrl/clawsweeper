## Summary

Keep this PR open. Current main still does not expose inbound media on PluginHookAgentContext, and before_agent_reply still returns before the normal inbound media staging path when a plugin claims the turn. The PR addresses a real remaining plugin API gap, though it needs rebase onto the current hook type split plus tests/docs/API review before it can land.

## Best Possible Solution

Keep the PR open and ask for a rebase onto current main. The best path is to add mediaPaths, mediaUrls, and mediaTypes to PluginHookAgentContext in src/plugins/hook-types.ts, populate defensive copies in the before_agent_reply context in src/auto-reply/reply/get-reply.ts, preserve URL-only attachments, document the fields on the public plugin hooks page, and add focused tests for text-only, local media, URL-only media, and mutation safety before maintainer API review.
