## Summary

Current main already addresses the reported `.jsonl.lock` leak after LLM idle timeout: embedded attempts release the session lock from an unconditional teardown `finally`, the lock release removes the sidecar file, and the session-lock watchdog can recover long-held in-process locks without a gateway restart. The related stuck-running timeout symptom remains better tracked by the already-linked open issue #63819.

## Best Possible Solution

Close this lock-file leak as fixed on main and in the current release. Keep the shipped unconditional cleanup and watchdog recovery; track any remaining stuck-running timeout lifecycle behavior in the already-linked #63819.
