## Summary

Keep #61100 open. Current main still lacks Discord `upload-file` discovery and dispatch/runtime handling, while the PR is a focused fix for the linked #60652 bug. The existing Discord send path can already attach media through `sendMessageDiscord(..., { mediaUrl })`, so the PR’s approach remains plausible and useful rather than obsolete or out of scope.

## Best Possible Solution

Keep the PR open for maintainer review or have a maintainer recreate the same focused fix on a fresh branch: add Discord `upload-file` discovery under the existing messages gate, map the public action to an internal runtime action, route it through the existing `sendMessageDiscord` media upload path, and add focused Discord discovery/dispatch/runtime contract tests. This is bundled Discord plugin behavior, not ClawHub-only work.
