## Summary

Keep open. Current `main` still has the Discord preview tool-progress path that can accumulate duplicate entries: it only dedupes the newest exact line against the immediately previous line, while tool and item events can report the same tool under different labels such as `tool: session_status` and `session_status`. The linked PR #72213 is directly relevant, but the provided context says it is still open and unmerged, so the issue remains a valid tracker until a fix lands.

## Best Possible Solution

Keep #72205 open and land PR #72213 or an equivalent maintainer-approved fix that normalizes tool-progress labels and dedupes against existing preview entries in `extensions/discord/src/monitor/message-handler.process.ts`, with a focused regression test in `extensions/discord/src/monitor/message-handler.process.test.ts` covering `onToolStart("tool: name")` plus `onItemEvent("name")` in Discord block streaming mode.

## Evidence

- **Current dedupe only checks adjacent exact text:** `pushPreviewToolProgress` normalizes whitespace, compares only `previewToolProgressLines.at(-1)` to the new line, then appends the new line. It does not compare against all existing lines and does not normalize away the `tool: ` prefix described in the issue/PR.
