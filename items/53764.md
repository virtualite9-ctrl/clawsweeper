## Summary

Current main already exposes Tavily as real plugin tools, and the later shipped Pi tool-allowlist fix matches the reported GPT-5 embedded-runtime symptom where bundled/client tools stopped executing. The Tavily plugin registers `tavily_search` and `tavily_extract`, the agent runtime appends plugin tools into the available tool set, and the bundled contract/docs both enumerate those tool names.

## Close Comment

Closing from a Codex review because current `main` already has the Tavily tools wired into the agent runtime, and the remaining GPT-5 embedded-runtime symptom called out here matches a later shipped fix. Concretely: `extensions/tavily/index.ts:10` registers `tavily_search` and `tavily_extract`, `src/agents/openclaw-tools.ts:325` appends plugin tools into the runtime tool set, and `src/plugins/tools.ts:75` resolves registered plugin tools from the runtime registry. The bundled contract also still requires those exact tool names in `test/helpers/plugins/plugin-registration-contract-cases.ts:125`. In addition, `CHANGELOG.md:304` shows the shipped `v2026.4.21` Pi fix that restored bundled/client tool execution for embedded OpenAI/OpenAI Codex GPT-5 runs. If this still reproduces on `v2026.4.21+`, please open a fresh issue with the exact model/runtime path, current `tools.allow`/`tools.deny` config, and a fresh runtime tool inventory capture.
