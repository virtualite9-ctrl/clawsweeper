## Summary

Keep #48108 open. Current main still has the reported stale-dropdown path: the Control UI session selector renders all `sessions.list` rows and then unconditionally adds the active `state.sessionKey`, so a deleted active Discord/channel session that is absent from the backend list remains visible as a fallback option. The related fix PR #48190 was closed unmerged, and I did not find an equivalent shipped fix or regression test on main.

## Best Possible Solution

Keep the issue open and revive a focused version of #48190: after a fresh `sessions.list` response, stop adding missing normal direct/channel active sessions to the chat session dropdown and normalize the active selection to an existing/default session, while preserving intentional fallback visibility for ephemeral subagent/cron sessions. Add regression coverage in `ui/src/ui/chat/session-controls.ts` or `ui/src/ui/app-render.helpers.node.test.ts` for a deleted Discord/channel session key absent from `sessions.list`.
