## Summary

Keep #9656 open. Current main still flattens Signal `editMessage` envelopes into normal inbound messages, does not model edit `targetSentTimestamp`, uses the new envelope timestamp as the inbound `MessageSid`, and has no edit-aware reply suppression or focused edit-envelope tests. Related PR #9701 attempted this work but is closed unmerged, so it should be treated as useful history, not shipped behavior.

## Best Possible Solution

Implement this in the bundled Signal plugin under `extensions/signal`. Add a typed `editMessage` envelope with validated `targetSentTimestamp`, preserve edit metadata or an `[edited]` marker in agent/session context, use the target timestamp for dedupe or transcript replacement, suppress automatic replies for edits unless the edited text explicitly mentions or commands the agent, and add focused tests for edit envelopes, `MessageSid`/dedupe behavior, history/marker behavior, and mention-gated group edits. PR #9701 can inform the implementation, but it was closed unmerged and should not be treated as shipped.
