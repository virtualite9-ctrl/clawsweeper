## Summary

Keep open. Current main still preserves the documented Mattermost contract that DMs are non-threaded, but the outbound root resolver remains chat-kind unaware and can still map a surviving payload replyToId into Mattermost root_id. The open maintainer PR #72305 is the right implementation candidate, not a reason to close before merge.

## Best Possible Solution

Keep this issue open and review/land PR #72305 or an equivalent Mattermost-plugin change. The fix should make `resolveMattermostReplyRootId` chat-kind aware, pass `kind` from inbound/button/model-picker/draft-preview delivery paths, prevent Mattermost `root_id` for direct-message auto-replies under the current docs contract, and add focused regression tests for direct chats with explicit payload `replyToId`. Treat configurable DM threading as a separate product/API follow-up.
