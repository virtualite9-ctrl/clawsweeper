## Summary

Keep #44810 open. Current main still surfaces the ACP child no-output path with the same generic “may be waiting for interactive input” stall text, and the parent relay still only treats assistant/lifecycle agent events as progress. The related PR #44949 directly targeted this issue but was closed unmerged, so it does not supersede the open issue.

## Best Possible Solution

Keep the issue open and revive or rework the #44949 direction: preserve ACP prompt/status/runtime activity events from ACPX, forward them through gateway and agent event bridges with the correct runId, update the parent relay to distinguish prompt-after-silence from true interactive wait, and add guarded proxy-env diagnostics plus regression tests for the Codex ACP no-output stall path.
