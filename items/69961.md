## Summary

Keep open. Current main still writes session-memory transcript text verbatim, so the core bug targeted by #69961 is not implemented. The linked #69943 discussion independently confirms the same handler/transcript root cause, and the existing shared assistant-visible sanitizer is only wired to user-visible/history surfaces, not this hook. There is overlapping open work in #71257, but it does not make this PR safely obsolete by itself.

## Best Possible Solution

Keep #69961 open as a plausible focused fix. The best path is for maintainers to choose the canonical session-memory sanitizer implementation, address the current review edge cases, avoid generated artifact churn, and land regression coverage for both transcript extraction and handler persistence so poisoned memory cannot be re-injected across `/new` or `/reset`.
