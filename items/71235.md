## Summary

Keep PR #71235 open. Current main still lacks the proposed unversioned `python` shim in the Dockerfiles, while the sandbox images and sandbox-common defaults do install `python3`. The PR is narrow, recent, and the author already addressed the bot review comments by switching to `$(command -v python3)` and `! command -v python` in PR head commit `6b44ae4`, so it is not obsolete, duplicate, or maintainer-owned auto-close material.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land or adapt the conditional shim across Dockerfiles that can contain `python3`, preserving any existing `python` on PATH and linking to `$(command -v python3)`.
