## Summary

Keep #69727 open. Current main has assistant-message copy and canvas-open actions in the Control UI, but no per-message print button, no `window.print()` path, and no print-only chat styling. This is a core Control UI/web frontend ergonomics request, not an obvious ClawHub/plugin cleanup close.

## Best Possible Solution

Add a focused Control UI message action beside copy/open-in-canvas for assistant text messages. It should render an isolated print document or print-only DOM using the existing sanitized markdown pipeline, include print-friendly table/code styling, hide chat chrome and other messages, and add targeted UI tests for action rendering plus the print helper behavior. Docs changes are optional unless the final UX introduces a user-facing setting or new documented workflow.
