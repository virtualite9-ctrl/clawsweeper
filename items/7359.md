## Summary

Keep #7359 open. Current main still separates Slack DM/main sessions from Slack channel sessions, mirrors outbound `message` sends only into the destination session, and default session-tool visibility blocks a DM/main session from reading an unrelated same-agent channel transcript. The 2026-04-26 maintainer/Codex issue comment also explicitly kept this open, and I found no later self-activity ledger or scoped Slack bridge in current main.

## Best Possible Solution

Keep #7359 open and implement a scoped Slack self-activity recall path. The maintainer-safe shape is an outbound activity ledger or session-history bridge keyed by agent id, Slack account/team, destination channel/thread, and sent Slack message id/timestamp, exposed from DM/main context only for the agent's own outbound Slack activity. Add regression coverage for: Slack DM/main session sends a `message` tool post to a Slack channel, then the DM asks whether the agent sent it and can recall the sent message without broad cross-session visibility.
