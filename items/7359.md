## Summary

Keep #7359 open. Current main still separates Slack DM/main sessions from Slack channel sessions, mirrors outbound `message` sends into the destination Slack session only, and the default session-tool visibility does not let a DM/main session read unrelated same-agent channel history. The provided issue discussion also includes a 2026-04-26 steipete/Codex comment explicitly keeping this exact gap open, so conservative cleanup should not close it.

## Best Possible Solution

Keep #7359 open and implement a scoped Slack self-activity recall path. A good product shape is an outbound activity ledger or session-history bridge keyed by agent id, Slack team/account, destination channel/thread, and Slack message id/timestamp, exposed from DM/main context only for the agent's own outbound Slack sends. Add regression coverage for a Slack DM/main turn that posts to a Slack channel via the `message` tool, then later recalls that sent message without enabling broad cross-session transcript visibility.
