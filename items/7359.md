## Summary

Keep #7359 open. Current main still separates Slack DM/main sessions from Slack channel sessions, and outbound message-tool sends are mirrored into the resolved destination session rather than a DM-visible self-activity record. The existing sessions tools do not solve this by default because session-tool visibility defaults to the current session tree, not arbitrary same-agent channel sessions.

## Best Possible Solution

Keep #7359 open and implement a scoped Slack self-activity recall path. The likely product shape is an outbound activity ledger or session-history bridge keyed by agent id, Slack team/account, destination channel/thread, and Slack message id/timestamp, exposed from DM/main context only for the agent's own outbound Slack sends. Add regression coverage for a Slack DM/main turn that posts to a Slack channel via the message tool and later recalls that sent message without granting broad cross-session transcript visibility.
