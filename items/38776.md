## Summary

Keep open. Current `main` still only extracts text from flat Feishu interactive-card `elements` or `body.elements`, so the quoted-card bug described by this PR is not implemented away. The PR also has unresolved review concerns about empty-array fallback behavior, so this needs maintainer review rather than conservative cleanup.

## Evidence

That does not cover header titles, `i18n_elements`, template variables, or other card variants described in the PR.
