## Summary

Keep #38776 open. Current main still only partially parses Feishu quoted interactive-card content: it handles top-level `elements` and `body.elements` with `div`/`markdown`, but not header titles, `i18n_elements`, template variables, recursive column/nested elements, or card-reference/raw-card variants. Related broader attempts are not merged, and the PR still contains useful focused work, though it should address existing review feedback before landing.

## Best Possible Solution

Keep #38776 open unless maintainers explicitly choose a canonical successor. The best path is a focused Feishu plugin parser update in `extensions/feishu/src/send.ts` with regression tests for header titles, non-empty fallback across `elements`/`body.elements`/`i18n_elements`, primitive template variables, nested card elements, and raw/post/card-reference shapes that Feishu returns through the current message read APIs. If a broader PR is selected instead, close #38776 only after linking that canonical PR or issue as the replacement.
