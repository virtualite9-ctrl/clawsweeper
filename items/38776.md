## Summary

Keep #38776 open. Current main still only partially handles quoted Feishu interactive cards: it reads top-level `elements` and `body.elements` for `markdown`/`div` text, but the PR's header, `i18n_elements`, `template_variable`, and nested `column_set`/generic `elements` extraction remains unimplemented and unsuperseded by a merged replacement.

## Best Possible Solution

Keep #38776 open unless maintainers explicitly select a broader canonical successor. The best path is a focused Feishu plugin parser update in `extensions/feishu/src/send.ts` with regression tests for header titles, non-empty fallback across `elements`/`body.elements`/`i18n_elements`, primitive template variables, nested card elements, and raw/post/card-reference shapes returned by Feishu message read APIs.
