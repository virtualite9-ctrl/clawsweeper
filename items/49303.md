## Summary

Current `main` does not implement this PR's auto-retry behavior. The role-ordering paths still reset the session and immediately return a user-facing "please try again" message, and the added one-shot retry guard/test coverage described in the PR are absent. That is not strong enough evidence for `implemented_on_main` or `cannot_reproduce`, so this PR should stay open for maintainer judgment.
