## Summary

Issue #59587 asks for a prebuilt browser-automation skill template. Current main and project vision place new skills and optional capability in ClawHub/plugin space rather than OpenClaw core, and OpenClaw already ships the underlying browser automation surface plus docs for creating and publishing skills/plugins.

## Evidence

- **Vision scope: new skills belong in ClawHub, not core:** Project vision says optional capability should usually ship as plugins, plugin promotion belongs in ClawHub, new skills should be published through ClawHub first, and 'New core skills when they can live on ClawHub' is explicitly listed under 'What We Will Not Merge (For Now)'.
  - file: [VISION.md:54](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/VISION.md#L54)

  - file: [docs/tools/browser.md:10](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/docs/tools/browser.md#L10)
