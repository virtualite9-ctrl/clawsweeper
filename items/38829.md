## Summary

Keep #38829 open. Current main still lacks the TUI transcript subscription and event handling needed for cross-surface real-time message updates; the gateway exposes the required session.message surface and Control UI handles it, but src/tui only routes chat/chat.side_result/agent events.

## Best Possible Solution

Keep #38829 open. The best fix is a focused TUI implementation: add TuiBackend/GatewayChatClient subscribe and unsubscribe methods for the active session transcript stream, subscribe on connect and session changes, handle session.message for the current session by appending or reloading history without disrupting active local streaming, ignore other-session events, and add TUI tests for cross-surface delivery, active-run deferral, unsubscribe/session-switch behavior, and session filtering. Coordinate with #41964 so chat-event fanout fixes complement the transcript subscription path.
