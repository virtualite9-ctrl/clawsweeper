## Summary

Keep #38829 open. Current main still does not implement the TUI side of real-time cross-surface transcript updates: the gateway exposes `sessions.messages.subscribe` and emits `session.message`, and Control UI consumes those events, but `src/tui` neither subscribes to session transcript events nor routes `session.message` into history reload/rendering.

## Best Possible Solution

Keep #38829 open. The best fix is a focused TUI change: add `TuiBackend`/`GatewayChatClient` subscribe and unsubscribe methods for `sessions.messages.subscribe` on the active session, subscribe after connect and on session switches, unsubscribe stale sessions on switch/stop, handle `session.message` for the current session by reloading or appending history without disrupting an active local stream, ignore other-session events, and add TUI regression tests for cross-surface delivery, active-run deferral, session switching/unsubscribe, and session filtering. Coordinate separately with #41964 because chat-event fanout is related but does not replace transcript subscription.
