## Summary

Keep #38829 open. Current main still lacks the TUI subscription and event-handling path needed for real-time cross-surface transcript updates. The gateway already exposes and tests `sessions.messages.subscribe` / `session.message`, and Control UI consumes `session.message`, but `src/tui` neither subscribes to those events nor routes them into history reload/rendering. The related PR #41964 is still only a narrower chat-event fanout fix in the provided context and does not replace the missing transcript subscription path.

## Best Possible Solution

Keep #38829 open. The best fix is a focused TUI implementation: add `TuiBackend` / `GatewayChatClient` wrappers for `sessions.subscribe` or `sessions.messages.subscribe` plus unsubscribe, subscribe after gateway connect and when the active session changes, handle `session.message` for the current session by reloading or appending history without disrupting an active local stream, ignore other-session updates, clean up stale subscriptions on session switch/stop, and add TUI regression tests for cross-surface delivery, active-run deferral, session switching/unsubscribe, and session filtering. Treat #41964 as related but narrower chat-event fanout work, not as the canonical fix for this transcript subscription gap.
