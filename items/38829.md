## Summary

Keep open. Current main has live assistant `chat` websocket handling in the TUI, but the reported reproduction is cross-surface: a message sent from web UI or another channel should appear in the terminal in real time. Gateway has a separate `session.message` subscription/broadcast path for transcript messages, and the web UI handles it, but `src/tui` neither subscribes to `sessions.messages.subscribe` nor handles `session.message` events. That leaves a meaningful part of the requested behavior missing.
