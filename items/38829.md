## Summary

Keep #38829 open. Current main still lacks a TUI subscription or handler for gateway transcript events, so messages appended from another surface can be present in `chat.history` while the running TUI has no path to reload or append them live. The related #41964 is still an open, narrower gateway chat-event PR and would not add the missing TUI transcript subscription path.

## Best Possible Solution

Keep #38829 open. The best fix is a focused TUI change: add subscribe/unsubscribe methods to `TuiBackend` and `GatewayChatClient`, subscribe to `sessions.messages.subscribe` for the active session after connect and after session switches, handle `session.message` for the active session by reloading or appending history without disrupting an active local stream, ignore other sessions, clean up stale subscriptions on switch/stop, and add TUI regression tests for cross-surface delivery, active-run deferral, session switching/unsubscribe, and session filtering. Treat #41964 as related external-channel chat fanout work, not the canonical fix for this issue.
