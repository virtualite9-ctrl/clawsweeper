## Summary

Keep #17876 open. Current main has model set/status/scan/fallback helpers, but it still lacks the requested staged model migration workflow: arbitrary candidate testing, promote-with-old-primary-as-fallback, and one-command rollback. The latest maintainer comment in the issue also explicitly says to keep it open.

## Best Possible Solution

Keep #17876 open for maintainer/product follow-through. The likely core path is an `openclaw models test/migrate/promote/rollback` workflow built on the existing status, fallback, auth probe, and scan primitives: add a candidate as fallback, run provider/tool/subagent probes for an arbitrary model ref, promote only after verification while moving the old primary into fallbacks, and persist enough prior-state metadata for rollback. Add docs and focused tests for command behavior and config transitions.
