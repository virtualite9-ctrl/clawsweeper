## Summary

Keep #17876 open. Current main still has immediate primary model mutation, manual fallback commands, and OpenRouter-focused scan/probe helpers, but no staged migration workflow for testing an arbitrary candidate model, promoting it while moving the old primary to fallback, or rolling back in one command. The issue discussion also includes a maintainer-authored Codex review explicitly saying to keep it open.

## Best Possible Solution

Keep #17876 open for a core models CLI workflow. Build on the existing status/probe, fallback, auth, and scan primitives: add a candidate as fallback, test it with an arbitrary provider/model ref, promote only after verification while moving the old primary into fallbacks, persist enough prior state for `openclaw models rollback`, then add focused config-transition tests and CLI docs.
