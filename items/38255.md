## Summary

Keep PR #38255 open. Current main at 631552c554095b0c520376db9131dac0f5427f94 still does not expose `channels.telegram.streamThrottleMs` or `channels.telegram.minInitialChars`, still hardcodes Telegram draft preview `minInitialChars` in dispatch, and still leaves Telegram draft stream cadence at the internal 1000ms default unless a caller passes `throttleMs`. The linked issue #38066 remains the same open user problem, and this PR is still the active implementation path rather than obsolete cleanup.

## Best Possible Solution

Keep PR #38255 open as the active implementation path for #38066. The best path is to add `channels.telegram.streamThrottleMs` and `channels.telegram.minInitialChars` to the Telegram schema, exported config types, generated config metadata, and user docs; thread both values through `dispatchTelegramMessage` into `createTelegramDraftStream`; preserve separate draft-vs-message fallback cadence semantics; add a timer-safe upper bound for `streamThrottleMs`; and keep focused config, Telegram dispatch, draft-stream, and shared-loop tests around validation, fallback, final flush, and configured cadence behavior.
