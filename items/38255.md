## Summary

Keep PR #38255 open. Current main still does not expose `channels.telegram.streamThrottleMs` or `channels.telegram.minInitialChars`, still hardcodes Telegram draft initial text threshold in dispatch, and still uses the internal 1000ms draft stream default unless a caller passes `throttleMs`. The linked issue #38066 remains the same open user problem, so this PR is not obsolete cleanup.

## Best Possible Solution

Keep PR #38255 open as the active implementation path for #38066. The best path is to add `channels.telegram.streamThrottleMs` and `channels.telegram.minInitialChars` to the Telegram schema, exported config types, generated metadata, UI/help/docs; thread both values through `dispatchTelegramMessage` into `createTelegramDraftStream`; preserve separate draft-vs-message fallback cadence semantics; add a timer-safe upper bound for `streamThrottleMs`; and keep focused config, Telegram dispatch, draft-stream, and shared-loop tests around validation, fallback, final flush, and configured cadence behavior.
