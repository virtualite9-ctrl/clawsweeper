## Summary

Keep #13239 open. Current main has provider docs, auth/probe status, model catalog metadata, and plugin compatibility/deprecation registries, but it still does not ship the requested public provider x model-family x capability support-lifecycle matrix with stable/experimental/deprecated/broken states, deadlines, tracked issue links, generated docs, and doctor output. The Matrix plugin incident linked in the discussion is useful motivation, but it is a specific plugin API-version regression and does not supersede this broader provider policy request.

## Best Possible Solution

Keep the issue open until maintainers approve and ship a provider compatibility policy plus a machine-readable registry keyed by provider, model family, capability, lifecycle status, deadline, replacement, and canonical issue link. Generate the public docs page from that registry and surface a concise doctor or models-status view while keeping live auth/probe health separate from support lifecycle state.
