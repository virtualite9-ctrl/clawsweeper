## Summary

Current main still does not publish the requested provider x model-family x capability compatibility matrix, lifecycle/deprecation policy, generated docs surface, or doctor/status output. Existing docs and CLI surfaces cover provider setup, model listing, auth/probe health, and partial model-catalog status metadata, but not the broader support-lifecycle policy requested in #13239. The related Matrix plugin issue #52899 is useful motivation, but it was a closed plugin-specific regression and does not supersede this broader provider policy proposal.

## Best Possible Solution

Keep #13239 open until maintainers approve and ship a provider compatibility policy plus a machine-readable registry keyed by provider, model family, capability, lifecycle status, deadline, replacement, and canonical issue link. Generate a public docs page from that registry and surface concise lifecycle state in CLI doctor or models status output, while keeping live auth/probe health separate from support lifecycle state.
