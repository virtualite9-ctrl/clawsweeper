## Summary

Keep open. Current main still has provider lists, per-provider docs, model auth/probe status, media capability tables, and plugin compatibility/deprecation tracking, but not the requested public provider x model-family x capability lifecycle matrix with stable/experimental/deprecated/broken states, deadlines, issue links, and doctor output.

## Best Possible Solution

Keep the issue open until maintainers approve and ship a provider compatibility policy plus a machine-readable provider compatibility registry. A good path is to add lifecycle metadata by provider, model family, capability, status, deprecation deadline, replacement, and canonical issue link, then generate public docs and surface a concise doctor or models-status view while keeping live auth probes separate from support lifecycle state.
