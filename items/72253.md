## Summary

Keep PR #72253 open. Current main still treats structured exec completions, including `code 0` and `signal SIGTERM`, as relayable exec events when a heartbeat delivery target exists. The PR is a focused, current bug-fix attempt for #72252 and is not implemented on main, not cleanly superseded by the broader #69492 audience-classification issue, and not made obsolete by adjacent PRs such as #72218.

## Best Possible Solution

Keep #72253 open for maintainer review. The best path is to refine or land a narrow heartbeat fix that keeps structured `code 0` and restart-cleanup `signal SIGTERM` exec completions internal, preserves relay for real failures such as `code 1`, and resolves the text-classification/trust-boundary concerns either in this PR or by aligning it with the broader #69492 event-audience primitive.

## Risks / Open Questions

- The PR still needs maintainer review of the Aisle bot concerns: the patch classifies internal-only completions by event text shape, and the bot flagged untrusted text spoofing and prompt-injection risks in the surrounding exec relay path.
- There is active overlap with #69492, #67273, and #71860. Maintainers should decide which PR is canonical, but that overlap is not enough to close #72253 because it covers the concrete SIGTERM cleanup case that remains missing on main.
- The product contract should be explicit: structured `signal SIGTERM` cleanup completions are benign in the reported restart path, but a meaningful SIGTERM failure could be hidden if the classifier is too broad.
