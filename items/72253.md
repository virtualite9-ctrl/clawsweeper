## Summary

Keep this PR open. Current main at c45a7d7a7aec62acfe6f11ca2f1bf42b90b951a2 still treats structured `Exec completed (..., code 0)` and `Exec failed (..., signal SIGTERM)` as relayable exec-completion events, and still has the exec-completion bypass that can revive `HEARTBEAT_OK`-stripped text. The PR is a recent focused fix for the linked restart-cleanup chat-noise bug, not something already implemented or cleanly superseded by the broader #69492 design discussion.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is a narrow core heartbeat fix that marks trusted/local structured `code 0` and cleanup `signal SIGTERM` exec completions internal, preserves user relay for actionable failures such as `code 1`, and either lands this focused patch or deliberately folds it into the broader #69492/#67273 event-audience work.
