## Summary

Close #61435 as superseded by the canonical open PR #67055. Current main has the Google Chat `replyToMode` config surface, but the monitor still has the exact remaining thread propagation points described here; PR #67055 directly tracks and patches those points for `replyToMode: "off"`. The broader streamed/follow-up thread metadata leakage remains separately tracked by open issue #69422.

## Best Possible Solution

Close this issue as superseded by open PR #67055 for the narrow `replyToMode: "off"` Google Chat DM/top-level reply behavior. Keep #69422 open for the broader architectural thread metadata work across streaming and follow-up queues.
