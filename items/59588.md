## Summary

Keep open. Current `main` still makes Feishu native voice-message delivery depend on Opus/Ogg input, while the Microsoft TTS provider still defaults to MP3 and does not switch formats for voice-note targets. The reported `messages.tts.provider: microsoft` + Feishu path therefore still plausibly sends a file attachment instead of a native voice message.

## Evidence

- **Feishu only promotes Opus/Ogg to native audio messages:** `resolveFeishuOutboundMediaKind` returns `msgType: "audio"` only for `.opus`, `.ogg`, `audio/ogg`, or `audio/opus`; everything else falls through to generic file routing, so MP3 stays a file attachment.
  - file: [extensions/feishu/src/media.ts:539](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/extensions/feishu/src/media.ts#L539)
  - command: `nl -ba extensions/feishu/src/media.ts | sed -n '494,575p'`
  - sha: [ec3dbd22a4de](https://github.com/openclaw/openclaw/commit/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46)
- **Feishu tests cover Opus-as-audio, not MP3-as-voice:** Current tests assert `.opus` is sent with `msg_type: "audio"` and documents are sent as `msg_type: "file"`; there is no Microsoft/MP3 voice-note coverage showing the reported path is fixed.
  - file: [extensions/feishu/src/media.test.ts:171](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/extensions/feishu/src/media.test.ts#L171)

In the current implementation there is no `req.target === "voice-note"` override to force Opus/Ogg for Feishu.

That means the exact reported config is still not guaranteed to produce a native Feishu voice bubble on `main`.

Reviewing current `main` still shows Feishu audio promotion gated on Opus/Ogg plus Microsoft defaulting to MP3, so there is not strong evidence that the reported regression is actually resolved.

## Risks / Open Questions

- I did not run a live Feishu reproduction because this review is read-only and the workspace lacks the credentials/runtime setup needed for an end-to-end channel check.
- If a user explicitly configures Microsoft to an Ogg/Opus output format, Feishu may behave correctly; the open issue is specifically about the reported default Microsoft TTS path.
- `docs/tools/tts.md` also contains a broad statement that Feishu audio is delivered as a voice message, which appears looser than the actual provider-specific behavior and may itself need follow-up.
