## Summary

Current main still only provides SQLite-backed memory indexing/search for Markdown memory files plus adjacent wiki/LanceDB plugin layers. It does not define the requested durable typed SQLite memory record schema, canonical current record, revision trail, explicit lifecycle statuses, validity windows, or normalized entity/subject fields.

## Best Possible Solution

Keep this issue open. The best implementation path is a versioned durable-memory SQLite model in the memory owner surface, likely `memory-core` plus the memory-host SDK storage contract: canonical `memories` rows, `memory_revisions`, typed record/status/confidence/timestamp/validity/entity fields, migrations and contract tests, public docs, and integration with write, provenance, review, forget, and conflict flows. Coordinate with #42647, #42648, and #42650; #42649 only covers retrieval.
