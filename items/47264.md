## Summary

Keep open. Current main still appears to reproduce the reported LINE multi-account failure: `accounts.default` is listed as an account but is not used when resolving the default account credentials, and named LINE accounts default to disabled unless `enabled: true` is set on the account. That means the default account is treated as unconfigured and the named account is skipped before webhook startup can register routes.

## Best Possible Solution

Implement the narrowly scoped behavior from the closed PR #47528 on current main: make `resolveLineAccount` read `channels.line.accounts.default` for the default account, treat configured named LINE accounts as enabled unless explicitly disabled, and add regression tests covering the reported `accounts.default` plus named-account webhook-path configuration. A focused startup/route-registration test would further prove the 404 symptom is fixed.
