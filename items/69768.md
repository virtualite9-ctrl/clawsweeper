## Summary

Keep #69768 open. The provided issue discussion identifies a concrete Gemini tool-schema cleaner gap, and current main still does not strip JSON Schema `if`/`then`/`else` conditionals on the Gemini path. The related PR #69964 proposed exactly that fix but was closed unmerged, so there is no shipped implementation to close against.

## Best Possible Solution

Land a narrow core fix in `src/agents/schema/clean-for-gemini.ts` adding `if`, `then`, and `else` to the Gemini unsupported-keyword set, with regression coverage in `src/agents/schema/clean-for-gemini.test.ts` proving nested conditionals are stripped while ordinary properties remain. After that, consider a small docs follow-up for production cron failure visibility via `cron.failureDestination` if maintainers want the operational guidance from the report captured publicly.
