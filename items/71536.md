## Summary

Keep #71536 open. Current main still has multiple intentional gates that can produce the reported Telegram “You are not authorized to use this command.” reply for `/restart`, and I did not find source, history, docs, or tests proving the reported 2026.4.23 Telegram restart authorization failure is fixed on main. The report is recent, labeled bug/regression, and is salvageable with exact sanitized config/log evidence.

## Best Possible Solution

Keep the issue open and ask for sanitized `channels.telegram`, `channels.telegram.accounts.<id>`, `commands.allowFrom`, `commands.ownerAllowFrom`, DM/group/topic context, Telegram `from.id`/`chat.id`, and gateway logs around the rejected `/restart`. If reproduced, add a focused regression test that drives Telegram native `/restart` through `extensions/telegram/src/bot-native-commands.ts` into `src/auto-reply/reply/commands-session.ts`, then fix whichever auth layer incorrectly drops the owner identity.
