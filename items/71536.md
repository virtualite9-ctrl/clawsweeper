## Summary

Keep open. This is a recent non-maintainer bug/regression report for a bundled Telegram native `/restart` path, and current `main` still has distinct Telegram native pre-dispatch authorization plus generic owner-only restart authorization that can both produce the same “You are not authorized to use this command.” response. I did not find source, test, docs, release, or related-item evidence that this specific regression is fixed or superseded.

## Best Possible Solution

Keep this issue open. The maintainer path is to ask for or reproduce the exact Telegram config, then add a focused regression test in `extensions/telegram/src` that drives native `/restart` through Telegram authorization into the generic restart handler with `channels.telegram.allowFrom`, `channels.telegram.groupAllowFrom`, `commands.allowFrom.telegram`, and `commands.ownerAllowFrom`. If reproduced, fix the Telegram native preauth/owner handoff or fresh config resolution so a properly owner-allowlisted Telegram sender can restart without weakening non-owner command gating.
