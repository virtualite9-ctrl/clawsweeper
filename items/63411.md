## Summary

Keep #63411 open. Current main still uses an activity-based LLM idle watchdog: each streamed chunk resets the timeout, so a model that keeps emitting repetitive text can run until other limits. Existing loop detection is scoped to tool calls, not assistant text/thinking streams, and there is no shipped n-gram/semantic stream breaker. There is some replay hygiene for gateway-injected abort partials, but that is not a loop-specific breaker or rollback for repeated model output. Related PR #39961 is still open work, not a shipped fix, and #64500 tracks a separate tool ping-pong breaker gap.

## Best Possible Solution

Keep this open and treat it as a core runner safety issue. The best fix is a bounded streaming-output repetition guard in the model stream/message path, with config/schema/docs/tests, clear abort/liveness metadata, and replay hygiene that prevents loop-aborted provider output from poisoning the next turn. Coordinate with #39961 for implementation ideas and keep #64500 separate for tool ping-pong breaker behavior.
