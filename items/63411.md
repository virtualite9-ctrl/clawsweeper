## Summary

Current main still does not implement the requested semantic streaming repetition breaker or loop-aware context sanitization. The idle timeout remains chunk-activity based, existing loop detection is scoped to tool calls, and abort partial retention is explicitly documented. The related #39961 remains an open implementation candidate, not shipped behavior.

## Best Possible Solution

Keep this issue open. Review or adapt #39961, or implement an equivalent core stream guard near the assistant stream subscription path: detect repeated text/thinking while the stream is active, abort with explicit loop metadata, prevent buffered repetitive content from being drained as a final reply, and quarantine loop-aborted partial assistant output from future model replay while preserving transcript visibility.
