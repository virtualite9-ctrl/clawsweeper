## Summary

Current main still does not implement the reported semantic streaming repetition breaker. The LLM idle watchdog remains chunk-activity based, streamed assistant text/thinking is appended and emitted without n-gram or structural loop checks, and the shipped loop detector/config surface is scoped to tool-call loops. There is some failed/aborted-turn replay hygiene, but no loop-specific streaming guard or quarantine path comparable to the open implementation candidate in #39961.

## Best Possible Solution

Keep this issue open. The best maintainer path is to review or adapt #39961, or implement an equivalent core agent-stream guard near the stream subscription/wrapper layer: detect repeated text/thinking patterns while the stream is active, abort with explicit loop metadata, and exclude or quarantine loop-aborted partial assistant output from future model replay while preserving transcript/audit visibility.
