## Summary

Keep open. Current main still has only activity-based LLM idle timeout and tool-call loop detection. It has some failed/aborted-turn replay hygiene, but no active text/thinking n-gram or semantic repetition breaker and no loop-specific quarantine/rollback path for a detected streaming repetition loop. The prior automated review's linked PR #39961 remains the closest implementation candidate in the provided context, while #64500 is a separate tool ping-pong breaker issue.

## Best Possible Solution

Keep this issue open. The best maintainer path is to review or adapt #39961, or implement an equivalent core agent-stream guard near the stream subscription/wrapper layer: detect repeated text/thinking n-grams while the stream is active, abort with explicit loop metadata, and quarantine or omit loop-aborted partial assistant output from future model replay while preserving user-visible audit metadata where appropriate. Track #64500 separately because it is about tool-call ping-pong breaker behavior.
