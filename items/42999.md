## Summary

Keep this issue open. Current main still uses fixed 16k/32k context-window guard thresholds, and the tests explicitly lock that behavior. The existing `agents.defaults.contextTokens` support caps the effective model context budget, but it does not implement the requested percentage-based warning/block thresholds. The related RFC/code PRs in the provided context are closed and not merged, so they do not supersede the remaining work.

## Best Possible Solution

Implement a small threshold resolver in `src/agents/context-window-guard.ts` that derives warning and hard-min thresholds from the resolved effective context window, preserves safe floors and explicit overrides, then update the guard tests, Pi runtime call site, custom onboarding assumptions, and config/docs text so small and large model behavior is intentional.
