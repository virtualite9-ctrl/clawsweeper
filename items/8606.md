## Summary

Keep open. Current main still prunes context by returning rewritten messages only, with no `session:prune` or `context:prune` hook in the pruning path, internal hook docs, or plugin hook catalog. The directly related implementation PR #12581 remains open in the provided GitHub context, while #9761 was closed unmerged.

## Best Possible Solution

Keep #8606 open until PR #12581 or a replacement maintainer-approved implementation lands. The likely implementation belongs in `src/agents/pi-hooks/context-pruning/*`: collect safe pruning metadata, emit an internal `session:prune` or equivalent lifecycle event after pruning, and add tests plus public hook documentation for the chosen event and payload semantics.
