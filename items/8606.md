## Summary

Keep #8606 open. Current main still does not expose a `session:prune` or `context:prune` hook, and the current pruning implementation still returns only rewritten messages without pruning metadata for hook consumers. The related implementation PR #12581 is still separate from `main`, while #9761 was closed unmerged in the provided GitHub context.

## Best Possible Solution

Keep #8606 open until #12581 or a maintainer-approved replacement lands. The implementation likely belongs in `src/agents/pi-hooks/context-pruning/*`: collect safe pruning metadata, emit a `session:prune` or equivalent lifecycle event after successful pruning, add focused tests, and document the new event in `docs/automation/hooks.md`, `docs/plugins/hooks.md` if it becomes a plugin hook, and `docs/concepts/session-pruning.md`.
