## Summary

Current main still does not implement per-agent memory-wiki vault configuration. The code continues to resolve one `plugins.entries["memory-wiki"].config` at plugin registration, agent config has no `plugins` override slot, and wiki reads/writes still use the captured `config.vault.path`. The related open PR #67584 is an implementation candidate, but it is unmerged, so this issue should remain open.

## Best Possible Solution

Keep this issue open as the canonical narrow request for memory-wiki vault isolation. The best path is to review and land PR #67584 or an equivalent implementation that resolves the memory-wiki vault per active agent/context, with coverage for `wiki_search`, `wiki_get`, `wiki_apply`, gateway/wiki surfaces where applicable, docs, and tests proving two agents on one gateway cannot read or write each other's wiki vaults while preserving the current single-vault default.
