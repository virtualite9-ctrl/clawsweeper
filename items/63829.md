## Summary

Keep this issue open. Current main still does not implement per-agent memory-wiki vault isolation: the plugin resolves one global config at registration, the config schema has no per-agent vault mode or vault.path templating, wiki read/write/gateway paths use the captured vault path, and the strict agent config schema still has no agents.list[].plugins override. The related open PR #67584 is an implementation candidate, but it is not merged into main.

## Best Possible Solution

Keep this issue open as the memory-wiki vault isolation tracker. Review and land #67584 or an equivalent memory-wiki-owned change that resolves the vault root per invocation context or supports an explicit per-agent vault mapping, preserves the current single-vault default, updates schema/docs, and adds regression tests proving two agents on one gateway cannot read or write each other's wiki vault through wiki_search, wiki_get, and wiki_apply.
