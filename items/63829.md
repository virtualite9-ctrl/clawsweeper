## Summary

Current main still lacks per-agent memory-wiki vault isolation. The issue discussion describes a real multi-agent memory-bleed problem, and open PR #67584 is a plausible implementation candidate, but it is not merged and main still resolves one memory-wiki vault config per plugin registration.

## Best Possible Solution

Keep this issue open as the canonical narrow request. Review and land PR #67584 or an equivalent memory-wiki-owned implementation that resolves the vault path per active tool/gateway context, covers wiki_search, wiki_get, wiki_apply, status/lint/CLI or clearly documents their scope, and adds tests proving two agents on one gateway cannot read or write each other's wiki vaults while preserving the current single-vault default.
