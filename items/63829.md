## Summary

Keep this issue open. Current main still resolves memory-wiki vault configuration once from the single global plugin entry, the wiki read/write paths continue to use that captured vault path, and the strict agent config shape still has no per-agent plugins override. The related open implementation candidate #67584 is the right path to review or supersede, but it has not landed on main.

## Best Possible Solution

Keep this issue open as the memory-wiki vault isolation tracker. Review and land #67584 or an equivalent memory-wiki-owned implementation that resolves the vault root per invocation context or accepts an explicit per-agent vault mapping, preserves the current single-vault default, updates docs/config schema, and adds tests proving two agents on one gateway cannot read or write each other's wiki vaults through wiki_search, wiki_get, and wiki_apply.
