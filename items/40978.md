## Summary

Current `main` already forwards inbound image attachments into ACP-bound turns, preserves those attachments on the gateway path for ACP-bound sessions, and includes both regression and live coverage; the changelog shows this shipped in `v2026.3.11`.

## Evidence

- **ACP dispatch now forwards inbound media attachments into runtime turns:** `tryDispatchAcpReply` resolves inbound media attachments from the message context and passes them as `attachments` to `acpManager.runTurn(...)` for ACP-bound sessions.
  - file: [src/auto-reply/reply/dispatch-acp.ts:409](https://github.com/openclaw/openclaw/blob/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503/src/auto-reply/reply/dispatch-acp.ts#L409)
- **Inbound image attachments are materialized for ACP:** `resolveAcpAttachments(...)` normalizes inbound attachments, reads allowed local image files, base64-encodes them, and returns ACP turn attachments.
  - file: [src/auto-reply/reply/dispatch-acp-attachments.ts:27](https://github.com/openclaw/openclaw/blob/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503/src/auto-reply/reply/dispatch-acp-attachments.ts#L27)
- **Gateway keeps image attachments when the session is ACP-bound:** The gateway explicitly preserves image attachments for ACP-bound sessions even when the parent OpenClaw session model is text-only.
  - file: [src/gateway/server-methods/chat.ts:2204](https://github.com/openclaw/openclaw/blob/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503/src/gateway/server-methods/chat.ts#L2204)
- **Regression test covers ACP-bound attachment preservation:** There is a unit test asserting that text-only sessions bound to ACP keep image attachments instead of stripping/offloading them away from the ACP path.
  - file: [src/gateway/server-methods/chat.directive-tags.test.ts:2119](https://github.com/openclaw/openclaw/blob/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503/src/gateway/server-methods/chat.directive-tags.test.ts#L2119)
- **Live ACP bind smoke covers image understanding through a bound session:** The live bind test sends an attached image through a bound ACP session and asserts that the bound agent classifies the image.
  - file: [src/gateway/gateway-acp-bind.live.test.ts:783](https://github.com/openclaw/openclaw/blob/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503/src/gateway/gateway-acp-bind.live.test.ts#L783)
- **Changelog records the shipped fix:** The `v2026.3.11` changelog explicitly says ACP runtime attachments now forward normalized inbound image attachments into ACP runtime turns.
  - file: [CHANGELOG.md:2768](https://github.com/openclaw/openclaw/blob/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503/CHANGELOG.md#L2768)
