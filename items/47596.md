## Summary

Keep open. Current main still implements ACP `streamTo="parent"` as a parent-session progress relay for assistant text and lifecycle only. Tool-call data exists on the agent event surface and is already projected for the separate OpenClaw ACP bridge, but `startAcpSpawnParentStreamRelay()` ignores those streams, and the docs still describe `streamTo` as progress summaries rather than tool/thinking visibility. Related #67590 is broader channel/plugin streaming and was closed as ClawHub; #47597 covers native subagent `streamTo` and does not supersede this ACP-runtime relay gap.

## Best Possible Solution

Keep the issue open and implement it in the ACP parent relay. The likely maintainer path is to extend `src/agents/acp-spawn-parent-stream.ts` to consume existing `tool`, `item`, `command_output`, `patch`, `approval`, and optionally `thinking` streams for the child run, format concise parent-safe progress lines, truncate/redact args and results, and honor a clear verbosity policy. Add regression coverage in `src/agents/acp-spawn-parent-stream.test.ts` for tool start/update/result and docs under `docs/tools/acp-agents.md` describing exactly what `streamTo="parent"` surfaces.

## Risks / Open Questions
