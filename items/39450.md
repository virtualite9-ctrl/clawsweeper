## Summary

Keep open. I did not find strong evidence that current `main` fixes this. The current Control UI still persists only the newly selected session, and the chat dropdown only force-adds the current session if it is missing from `sessions.list`, not previously visited sessions like `main:local`.

## Evidence

- **Session switch overwrites persisted selection:** Switching sessions updates both `settings.sessionKey` and `settings.lastActiveSessionKey` to the newly selected session, so moving from `main:local` to `main:main` drops the prior session from persisted UI state.
  - file: [ui/src/ui/app-render.ts:1482](https://github.com/openclaw/openclaw/blob/cb2c36b04975e34ab7014a11575d6589aa96ae32/ui/src/ui/app-render.ts#L1482)
  - sha: [cb2c36b04975](https://github.com/openclaw/openclaw/commit/cb2c36b04975e34ab7014a11575d6589aa96ae32)
- **Storage keeps no recent-session history:** Per-gateway persistence stores only one `sessionKey` and one `lastActiveSessionKey`; there is no MRU/recent list that would preserve a previously visited non-current session across refresh.
  - file: [ui/src/ui/storage.ts:330](https://github.com/openclaw/openclaw/blob/cb2c36b04975e34ab7014a11575d6589aa96ae32/ui/src/ui/storage.ts#L330)
  - sha: [cb2c36b04975](https://github.com/openclaw/openclaw/commit/cb2c36b04975e34ab7014a11575d6589aa96ae32)
- **Dropdown only backfills the active session:** `resolveSessionOptionGroups` builds options from `sessions.list` rows, then calls `addOption(sessionKey)` once. That protects the current session if absent, but not a prior session like `main:local` after switching away.
  - file: [ui/src/ui/chat/session-controls.ts:498](https://github.com/openclaw/openclaw/blob/cb2c36b04975e34ab7014a11575d6589aa96ae32/ui/src/ui/chat/session-controls.ts#L498)
  - sha: [cb2c36b04975](https://github.com/openclaw/openclaw/commit/cb2c36b04975e34ab7014a11575d6589aa96ae32)
- **No relevant fix between latest release and current main:** Diffing `v2026.4.22..cb2c36b04975e34ab7014a11575d6589aa96ae32` for the relevant UI files showed only unrelated `app-gateway` pending-abort work and a test env header change, not a session-dropdown persistence fix.
  - command: `git diff --stat v2026.4.22..cb2c36b04975e34ab7014a11575d6589aa96ae32 -- ui/src/ui/storage.ts ui/src/ui/storage.node.test.ts ui/src/ui/app-gateway.ts ui/src/ui/app-gateway.node.test.ts ui/src/ui/chat/session-controls.ts ui/src/ui/chat/session-controls.test.ts ui/src/ui/controllers/sessions.ts ui/src/ui/app-chat.ts`
