## Summary

Keep open. Current main still does not implement the requested `MEDIA:/path#doc` or equivalent document-delivery hint for WhatsApp/generic media tokens. Existing `forceDocument`/`asDocument` plumbing is documented and consumed as Telegram-only, while WhatsApp still preserves detected image media as image payloads.

## Best Possible Solution

Keep this issue open and implement a supported outbound document-delivery hint. The likely fix belongs in the shared media token/loading contract plus WhatsApp delivery: parse a reserved `#doc` suffix or structured `asDocument` hint, strip only the reserved hint before local path resolution, preserve content type and filename, force downstream delivery as `document`, wire it through WhatsApp send and auto-reply paths, and add focused parser/loader plus WhatsApp image-as-document tests and docs.
