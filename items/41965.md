## Summary

Keep open. Current main still lacks a supported `MEDIA:/path#doc` or equivalent document-delivery hint for WhatsApp media. The existing `forceDocument`/`asDocument` path remains Telegram-only, while WhatsApp continues to classify image MIME/kind as image and send it through the image path.

## Best Possible Solution

Keep this issue open and implement a supported outbound document-delivery hint. The likely path is a shared media token/loading contract that parses a reserved `#doc` or structured `asDocument` hint, strips only that reserved hint before path resolution, preserves content type and filename, forces downstream delivery as `document`, wires it through WhatsApp normal send and auto-reply paths, and adds focused parser/loader plus WhatsApp image-as-document tests and docs. The closed unmerged #42044 can be used as prior art, but current main still needs a landed implementation.

## Risks / Open Questions

- Existing generic-looking `forceDocument`/`asDocument` names can be mistaken for a WhatsApp fix, but the current WhatsApp adapter does not consume them.
- A future `#doc` design needs a precise reserved-suffix contract so legitimate URL fragments, local filenames containing `#`, filename derivation, MIME preservation, and local-read safety checks remain well-defined.
