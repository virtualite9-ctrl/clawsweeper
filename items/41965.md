## Summary

Keep open. Current main has related Telegram-only `forceDocument` support, but it does not implement the requested `MEDIA:/path#doc` hint and does not force WhatsApp image media to document delivery. The prior implementation PR was closed unmerged, and the current media loader/WhatsApp send path still derives delivery kind from MIME type.

## Best Possible Solution

Keep this issue open and implement the capability deliberately: add a structured outbound media document hint or support the proposed `#doc` suffix by stripping it before media resolution, preserving the original filename/content type, forcing the loaded media kind or downstream delivery hint to `document`, and wiring that through the WhatsApp send path. Add focused tests in the shared media loader/parser and WhatsApp delivery path, then update CLI/channel docs if the behavior becomes supported beyond Telegram.
