## Summary

Keep open. Current main still does not implement the requested `MEDIA:/path#doc` document-delivery hint for WhatsApp or generic media tokens. The only related implementation PR in the provided context, #42044, was closed unmerged, and current code/docs still treat document forcing as Telegram-only user-facing behavior while WhatsApp continues to send detected image MIME media through image paths.

## Best Possible Solution

Keep this issue open and implement a supported outbound media document hint. The likely fix belongs in the shared media token/loading contract plus WhatsApp delivery: parse an explicit `#doc` suffix or structured `asDocument` hint, strip it before local path resolution when it is the reserved hint, preserve content type and filename, force downstream delivery as `document`, wire it through WhatsApp send and auto-reply paths, and add focused parser/loader plus WhatsApp image-as-document tests and user-facing docs.

## Risks / Open Questions

- The generic `forceDocument` field in outbound plumbing can be mistaken for a full fix, but WhatsApp does not currently consume it and agent-emitted `MEDIA:` directives still cannot carry a document hint.
- A future `#doc` implementation needs a deliberate contract so real remote URL fragments, local filenames containing `#`, filename derivation, MIME preservation, and host-read safety checks remain well-defined.
