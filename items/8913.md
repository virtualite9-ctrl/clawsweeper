## Summary

Keep #8913 open. Current main still lacks configurable reasoning/thinking display formatting: the shared formatter hardcodes italic Markdown, runner/subscriber/Telegram paths do not accept or thread a display-format option, config/docs only expose reasoning visibility and provider transport thinking format, and the related implementation PR #25336 is closed unmerged. The existing April 26, 2026 maintainer/Codex review comment reached the same keep-open conclusion, and I found no newer current-main implementation.

## Best Possible Solution

Keep #8913 open. A good fix should define a documented reasoning display-format contract, validate it in the chosen config scope, thread it through embedded runner payloads, subscribe handlers, reasoning streaming, and Telegram split/dispatch, preserve the current italic default for compatibility, and add regression coverage for default italic plus configured code/spoiler/blockquote behavior.
