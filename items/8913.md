## Summary

Current main still does not implement configurable thinking/reasoning block display formatting per channel or per agent. The shared formatter remains italic-only, core and Telegram call sites do not accept a display-format option, schemas/docs only expose reasoning visibility or provider transport thinking format, and related PR #25336 is closed unmerged. The issue should stay open.

## Best Possible Solution

Keep #8913 open. A good fix should define a documented reasoning/thinking display-format contract, validate it in the chosen config scope, thread it through embedded runner payloads, subscribe handlers, reasoning streaming, and Telegram split/dispatch, preserve the stable `Reasoning:` prefix unless explicitly migrated, and add regression tests for default italic plus configured code/spoiler/blockquote behavior.
