## Summary

Keep #8913 open. Current main still hardcodes reasoning display as `Reasoning:\n_..._` and does not expose a per-channel or per-agent reasoning display-format setting. The related implementation PR #25336 was closed unmerged after review found incomplete threading, so it does not supersede the issue.

## Best Possible Solution

Keep #8913 open. A complete fix should define and document a reasoning display-format contract, preserve italic as the compatibility default, validate the config shape, decide override precedence, thread the selected format through embedded run payloads, subscribe/message-end/streaming callbacks, and the Telegram reasoning lane, then add regression tests for default italic plus configured code/spoiler/blockquote behavior.
