## Summary

Keep PR #72174 open. Current main still routes both macOS attach-only entry points through `GatewayLaunchAgentManager.set(enabled: false, ...)`, and that method invokes `openclaw gateway uninstall`. This contradicts the documented attach-only contract that the app should not manage launchd and should only attach to an existing Gateway. The PR is a narrow, relevant fix with focused regression coverage, and no canonical merged/open item currently supersedes this exact attach-only/UI path.

## Best Possible Solution

Keep #72174 open for normal maintainer review. The best fix is the PR's narrow direction: make attach-only and the Debug Settings toggle persist only the disable-launchagent marker, avoid daemon uninstall/disable commands from those paths, and keep the focused regression test proving no daemon command is issued. Before merge, run the macOS Swift test filters from the PR and the repo's appropriate changed gate when available.
