## Summary

PR #72174 should remain open. Current main still has the reported macOS attach-only bug: both the `--attach-only`/`--no-launchd` launch path and the Debug Settings toggle write the disable marker, then call the launchd disable/uninstall path. The related launchd lifecycle work does not supersede this app-side marker-only fix.

## Best Possible Solution

Keep PR #72174 open for maintainer review or landing. The best fix is the PR's marker-only attach-only path for both the launch flags and Debug Settings toggle, plus the focused regression test proving no daemon commands are issued.
