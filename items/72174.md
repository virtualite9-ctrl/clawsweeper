## Summary

Keep open. Current main still contains the exact attach-only daemon-management behavior this PR removes: both the launch flag path and Debug Settings toggle write the disable marker and then call `GatewayLaunchAgentManager.set(enabled: false)`, which maps to `gateway uninstall`. That conflicts with the current docs promise that attach-only never installs or manages launchd.

## Best Possible Solution

Keep the PR open and review it as a plausible focused bug fix. The best path is to land or reimplement the marker-only attach-only behavior on main, with the regression test proving no daemon command is issued when attach-only is applied.
