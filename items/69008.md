## Summary

Keep #69008 open. Current main still treats gateway agent delivery as explicit-only, so omitted `deliver` on Telegram group-bound agent sessions remains session-only even when the session key encodes a Telegram group/topic destination.

## Best Possible Solution

Implement the gateway-side default proposed by the issue: detect `agent:*:telegram:group:*` with optional `:topic:*`, compute default visible delivery only when `request.deliver !== false`, feed that into `wantsDelivery`, and keep final external send suppressed when the resolved channel is the internal message channel. Add focused tests in the gateway agent delivery path for Telegram group, Telegram topic, explicit `deliver:false`, and the listed nonmatching session keys.
