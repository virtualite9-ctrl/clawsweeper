## Summary

Keep open. Current main still gates both delivery intent and the final outbound `deliver` flag on `request.deliver === true`, so Telegram group-bound agent sessions with omitted `deliver` remain session-only. The issue also has a linked external fix attempt from the reporter (`glfruit/openclaw#1`), which supports that this is an actionable implementation gap rather than stale discussion.

## Best Possible Solution

Keep the issue open and implement the narrow gateway default requested: add a helper that recognizes `agent:*:telegram:group:*` and `agent:*:telegram:group:*:topic:*`, use it to treat omitted `deliver` as delivery intent only for those session keys, preserve explicit `deliver: false`, and add focused regression tests for group, group topic, false override, Telegram DM, main, webchat, and non-Telegram group sessions.

## Risks / Open Questions

- Changing the default affects externally visible Telegram group behavior, so the fix should preserve `deliver: false` as an explicit session-only override and avoid auto-upgrading non-Telegram or DM sessions.
- The implementation should keep the existing internal-channel suppression path so resolved `webchat` or other internal-only targets remain session-only.
