## Summary

Keep this PR open. Current main still lacks gateway-side provider/auth narrowing for `models.list`, and the related open PRs are not merged canonical replacements. The PR direction is useful, but the submitted helper is incomplete against current auth semantics and needs fixes before it can safely land.

## Best Possible Solution

Keep this PR open for maintainer review or fold it into an explicit canonical successor. The best implementation is a shared provider-availability path used by gateway/model-picker surfaces, preserving the full-catalog fallback for unconfigured installs while matching current auth semantics for profiles, env keys, custom config keys, Bedrock AWS SDK/default-chain auth, and synthetic/local provider auth. Add focused regression coverage for gateway `models.list`, Bedrock auth, synthetic local/LAN providers, and UI/TUI consumers.

## Risks / Open Questions

- Landing the submitted helper as-is could silently hide usable Amazon Bedrock models for users authenticated through AWS SDK credentials or the default AWS credential chain once filtering activates.
- Landing the submitted helper as-is could hide usable local/custom providers that rely on synthetic local auth rather than an explicit API key.
- A separate gateway-only helper risks diverging from runtime auth resolution, CLI/model-picker hints, and provider availability behavior.
- Related #59693 and #64953 are still open and unmerged in the provided context; #64953 addresses `models.mode=replace`, not the full auth-gated provider filtering requested here.
