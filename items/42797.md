## Summary

Keep this PR open. Current main still does not auth-filter gateway `models.list` when no explicit allowlist is configured, so the PR addresses a real remaining gap. The submitted helper is security-sensitive and incomplete against current auth semantics, especially Bedrock AWS SDK/default-chain auth and synthetic local provider auth, so it should be revised rather than closed or landed as-is.

## Best Possible Solution

Keep this PR open for maintainer review, but require changes before merge. The best path is to replace the PR-local auth heuristic with `hasAvailableAuthForProvider` or an equivalent shared gateway-safe provider-availability helper, preserve the full-catalog fallback for truly unconfigured installs, and add focused gateway/UI/TUI regression coverage for env keys, auth profiles, config keys, Bedrock AWS SDK/default-chain auth, and synthetic local/LAN provider auth. If #59693 becomes the explicit canonical implementation and lands, this PR can then be closed as superseded.

## Risks / Open Questions

- Landing the submitted helper as-is could silently hide usable Amazon Bedrock models for users authenticated via AWS SDK credentials, AWS profile, or the default credential chain.
- Landing the submitted helper as-is could hide usable Ollama/local/LAN custom providers that rely on synthetic local auth rather than an explicit API key.
- A gateway-local auth heuristic could drift from runtime auth resolution, CLI model listing, and provider-owned synthetic auth behavior.
- Related #59693 and #64953 are open implementation candidates, but neither is merged on current main; #64953 covers `models.mode=replace`, not the full auth-gated provider-filtering behavior.
