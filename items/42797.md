## Summary

Keep this PR open. Current main still does not implement gateway-side provider-auth narrowing: `models.list` returns the configured allowlist or the full catalog, and UI/TUI consumers still use that RPC directly. The related #59693 work is broader, open, and not a clear complete replacement for this narrower PR; the review comments identify real fix-before-merge issues, not an auto-close reason.

## Best Possible Solution

Keep the PR open for maintainer review or fold it into an explicit canonical successor. The best implementation is a shared gateway/model-picker filtering path that preserves the full-catalog fallback for unconfigured installs, matches `resolveApiKeyForProvider` or `hasAvailableAuthForProvider` semantics, covers Bedrock AWS SDK and synthetic/local provider auth, and adds focused regression tests for gateway `models.list` plus UI/TUI picker consumers.
