## Summary

Keep this PR open. Current main still lacks `api.runtime.usage` or an equivalent supported plugin SDK path for model-pricing resolution, while the internal pricing helpers already exist and are tested. The PR is a focused additive plugin runtime API proposal with recent author follow-up, not stale cleanup or a ClawHub-only plugin request.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land a narrow additive `api.runtime.usage` contract if maintainers accept these helper signatures, with a rebase, plugin runtime docs, SDK API baseline alignment, a small runtime-shape test, and changed-gate proof before merge.
