## Summary

Keep open. Current `main` still contains a concrete lifecycle bug in the Nostr channel: `startNostrGatewayAccount(...)` returns immediately with a `{ stop() }` handle instead of keeping the account task pending until abort. The gateway manager treats any resolved `startAccount` task as a channel exit and schedules auto-restarts, which matches the reported loop and means the item is not implemented/fixed on `main`.

## Evidence

- **Nostr gateway start resolves immediately:** `startNostrGatewayAccount` awaits `startNostrBus(...)`, stores the bus, logs startup, and then returns a `{ stop() }` object at once. There is no `waitUntilAbort(...)`/`runPassiveAccountLifecycle(...)` call to keep the channel task alive.
  - file: [extensions/nostr/src/gateway.ts:79](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/extensions/nostr/src/gateway.ts#L79)
  - command: `nl -ba extensions/nostr/src/gateway.ts | sed -n '70,255p'`
  - sha: [c070509b7f8f](https://github.com/openclaw/openclaw/commit/c070509b7f8fdbe479e5d829643782c272e30e83)
- **Gateway manager restarts any resolved startAccount task:** `server-channels.ts` wraps `startAccount(...)` in `trackedPromise`; once that promise settles, it marks the account stopped and schedules an `auto-restart attempt ...`. A channel whose `startAccount` resolves immediately will be treated as a crash/exit loop.
  - file: [src/gateway/server-channels.ts:418](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/src/gateway/server-channels.ts#L418)
  - command: `nl -ba src/gateway/server-channels.ts | sed -n '400,520p'`
  - sha: [c070509b7f8f](https://github.com/openclaw/openclaw/commit/c070509b7f8fdbe479e5d829643782c272e30e83)
- **Repo tests document the restart-loop contract:** `server-channels.test.ts` explicitly asserts that a trivial `startAccount = async () => {}` causes repeated restarts until the cap is reached. That is the same shape as the current Nostr gateway adapter.
  - file: [src/gateway/server-channels.test.ts:157](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/src/gateway/server-channels.test.ts#L157)
  - command: `nl -ba src/gateway/server-channels.test.ts | sed -n '150,205p'`
  - sha: [c070509b7f8f](https://github.com/openclaw/openclaw/commit/c070509b7f8fdbe479e5d829643782c272e30e83)
- **Expected passive-channel pattern stays pending until abort:** `waitUntilAbort(...)` and `runPassiveAccountLifecycle(...)` exist specifically so passive channel/provider tasks remain pending while running and only resolve on abort. The Nostr gateway does not use that pattern.
