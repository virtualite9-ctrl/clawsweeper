## Summary

Keep #8355 open. Current main still batches the classic voice-call STT/agent/TTS path: it awaits one completed embedded-agent text response, then awaits one completed telephony TTS audio buffer, then chunks that finished buffer to Twilio. The newer realtime voice mode is a separate mutually exclusive architecture, and the related streaming TTS PR #42187 was closed unmerged.

## Best Possible Solution

Keep #8355 open as the canonical provider-generic voice-call feature request. The best implementation path is to add a sentence-level pipeline in the voice-call plugin: expose embedded-agent partial replies or sentence chunks, add a streaming telephony TTS capability with buffered fallback, stream chunks through Twilio and Telnyx media handlers, and preserve queue clearing, abort, and barge-in behavior for in-flight synthesis and playback.
