## Summary

Keep this PR open. Current main still lacks a general embedded-runner payload sanitizer for empty `tools` arrays and orphaned `tool_choice`, while the related `toolsAllow` forwarding bug has already landed separately. The PR remains a plausible focused fix, but should get maintainer review and direct regression coverage before merge.

## Best Possible Solution

Keep this PR open for maintainer review or a successor patch. The best path is a narrow embedded-runner egress sanitizer that strips empty `tools` arrays and orphaned `tool_choice`, covers session HTTP fallback, boundary-aware, and custom stream paths, preserves non-empty tools, composes with existing payload wrappers, and avoids redoing the already-landed `toolsAllow` forwarding work.
