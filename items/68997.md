## Summary

Keep #68997 open. Current main still has the reported lazy loader and the source runtime barrel, but the build entry list that creates stable root dist sidecars does not include `task-registry-control.runtime`, so there is not enough source-side evidence that the published npm tarball now ships the file required by `openclaw tasks cancel` for ACP/subagent task cancellation.

## Best Possible Solution

Add `task-registry-control.runtime` to the stable core dist entries, or replace the loader with an import path that tsdown actually emits. Then add release/package validation that fails when `dist/task-registry-control.runtime.js` is missing and, ideally, a packaged CLI smoke covering ACP/subagent `openclaw tasks cancel` or `tasks flow cancel`.
