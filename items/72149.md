## Summary

Keep PR #72149 open. Current main still forwards `options.temperature` into Anthropic Messages payloads when thinking is disabled, including for `claude-opus-4-7`; the PR is a focused code/test fix for a real request-shaping bug reported in #72123. The review comment about `top_p`/`top_k` is worth checking, but current main does not appear to forward those through this Anthropic transport path.

## Best Possible Solution

Keep the PR open and route it through normal maintainer review. The best code path is to land a narrow model-aware sampling-parameter guard in `src/agents/anthropic-transport-stream.ts` with regression coverage for Opus 4.7 omitting `temperature`, while preserving temperature behavior for other Anthropic models. A follow-up can decide whether the same policy should be shared with Anthropic Vertex or centralized.
