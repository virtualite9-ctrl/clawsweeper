## Summary

Keep #72149 open. Current main does not implement the PR's central fix: Anthropic transport still forwards an explicit temperature on non-thinking requests regardless of Claude Opus 4.7 model ID. The linked #72123 reproduction is concrete, and the PR is a narrow two-file fix with a targeted regression test in the provided diff.

## Best Possible Solution

Land #72149 or an equivalent Anthropic transport patch that normalizes Opus 4.7 model matching, omits unsupported sampling parameters such as `temperature` for Claude Opus 4.7 request payloads, preserves existing behavior for other Anthropic models, and keeps the focused regression test. Track the no-Opus-access fallback question separately if maintainers want that product behavior clarified.
