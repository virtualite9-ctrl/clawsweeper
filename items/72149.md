## Summary

Keep PR #72149 open. Current main still forwards `options.temperature` into native Anthropic Messages payloads whenever thinking is not enabled, including `claude-opus-4-7`, and the existing test file only covers Opus 4.7 adaptive thinking, not omission of `temperature`. The supplied #72123 discussion gives concrete v2026.4.24 live logs showing first-attempt Anthropic 400s from `temperature=0.2`, and Anthropic’s migration guide says Opus 4.7 requests should omit `temperature`, `top_p`, and `top_k`. The PR’s proposed code and regression test target the right current code path, though maintainer review should consider the QC note about sampling-parameter coverage and whether the shared/Vertex Anthropic path needs the same policy.

## Best Possible Solution

Keep the PR open and use it as the active fix path for #72123’s observable regression. The best maintainer direction is to land a narrow Opus 4.7 request-shaping fix with regression coverage, preferably factoring the rule into a shared Anthropic payload/request policy if Vertex or other Anthropic-compatible paths also need the same omission behavior. Add at least the PR’s native regression test, plus case-normalization coverage if accepting the normalized helper change.
