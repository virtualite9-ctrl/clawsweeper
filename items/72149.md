## Summary

Keep this PR open. Current main still forwards `temperature` in the native Anthropic Messages transport for non-thinking requests, and the PR is a focused implementation candidate for the live Opus 4.7 failure reported in #72123.

## Best Possible Solution

Keep the PR open and review or land this focused native Anthropic Messages transport fix, or apply an equivalent patch on main that omits unsupported Opus 4.7 sampling params with a regression test. Before merge, confirm the relevant Anthropic transport path has no remaining `top_p`/`top_k` pass-through and add a small uppercase/model-alias normalization test if maintainers want the extra coverage.
