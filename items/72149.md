## Summary

PR #72149 remains a live implementation candidate. Current main still forwards `temperature` in the native Anthropic Messages transport for non-thinking requests, while the PR narrowly adds an Opus 4.7 guard and regression test for the production failure reported in #72123.

## Best Possible Solution

Keep this PR open and review or land this focused native Anthropic Messages transport fix, or apply an equivalent patch on main that omits unsupported Opus 4.7 sampling parameters with regression coverage. Before merge, confirm whether `top_p`/`top_k` can reach this transport path and add a small normalization test if maintainers want coverage for the helper change.
