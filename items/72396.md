## Summary

Keep open. Current main supports a single writable plugin runtime dependency stage root, but it does not implement layered read-only baseline directories plus a final writable install directory. The issue is a concrete core runtime-deps feature, and the follow-up discussion adds useful operator requirements around strict read/write split, deterministic cache keys, and health reporting.

## Best Possible Solution

Keep this issue open and design it as a core plugin runtime-dependency staging enhancement: parse a documented stage-root list, resolve dependency sentinels left-to-right across read-only baseline roots plus the writable final root, install/repair only into the final root, register loader aliases for the selected satisfying root, make cache keys deterministic across OpenClaw version and runtime-deps manifests, and add focused tests/docs for Docker/systemd/offline deployments. Coordinate but do not collapse it into #71730, which is a related EACCES/runtime-deps bug report rather than a complete layered-stage feature spec.

## Risks / Open Questions

- Implementing this needs careful path-list semantics across POSIX and Windows because `path.delimiter` differs and colons can currently be part of the unparsed env value on POSIX.
- The loader alias, NODE_PATH registration, retained manifest, install lock, and doctor/health paths currently assume one install root; layered lookup should update those contracts together.
