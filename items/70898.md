## Summary

Keep #70898 open. Current main still silently rotates the in-reply session id/file without seeding recent transcript context, and Telegram DM handling does not have a separate DM history fallback. The PR is not obsolete or implemented on main, but it is not cleanly closeable as merged/superseded because late review concerns remain after the latest PR commit.

## Best Possible Solution

Keep the PR open and request a small rebase/fixup rather than closing it. The right product direction is still a core auto-reply/session fix: preserve a bounded, provider-safe recent user/assistant tail across silent compaction/role-ordering rotations, while keeping explicit `/new` and `/reset` fresh. Before merge, address the remaining review threads by normalizing or validating the replay tail so it cannot recreate strict role-ordering failures, and avoid full transcript reads on the recovery path.
