## Summary

Keep this PR open. Current main still forwards wizard select and multiselect option labels directly to Clack, and the existing wizard tests only cover tokenized filtering. The related onboarding crash fixes handle channel metadata/catalog fallback, docs-link fallback, or text-prompt trim paths; they do not implement this prompt-adapter label fallback. The PR diff is narrow and does not introduce security-sensitive infrastructure or supply-chain changes.

## Best Possible Solution

Keep this PR open for normal maintainer review. The best path is to land a narrow label-normalization helper in src/wizard/clack-prompter.ts with unit coverage for select, multiselect, and ideally searchable paths, while either adding a precise crash reproduction or reframing the PR as defensive fallback/display hardening if the crash claim cannot be reproduced.
