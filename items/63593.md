## Summary

Keep this PR open. Current main still treats an unresolved explicit `--tag latest` lookup as downgrade risk, so the false downgrade prompt this PR targets remains present. The PR discussion shows the review concern about unresolved non-latest tags was addressed by preserving that conservative guard while exempting unresolved `latest`.

## Best Possible Solution

Keep this PR open, rebase it onto current main, and review/land the narrow update CLI change that exempts unresolved explicit `latest` lookups from downgrade confirmation while preserving the unresolved non-latest guard. The implementation belongs in `src/cli/update-cli/update-command.ts`, with regression coverage in `src/cli/update-cli.test.ts`.
