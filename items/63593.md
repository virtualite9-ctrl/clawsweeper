## Summary

Keep open. Current main still treats any registry-resolvable target with an unresolved target version as downgrade risk, so `openclaw update --tag latest` can still hit the non-interactive downgrade stop when the latest lookup returns null. The PR remains a focused fix with a follow-up commit that addressed the review concern by preserving the conservative guard for unresolved non-latest tags.

## Best Possible Solution

Keep the PR open and carry its narrow behavior into current main: only suppress the downgrade prompt for unresolved `latest` lookups, preserve the existing conservative stop for unresolved non-latest registry tags, and land the paired regression tests in `src/cli/update-cli.test.ts`.
