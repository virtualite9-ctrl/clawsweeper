## Summary

Keep this PR open. Current main still archives the resolved state asset without a tar filter or backup-plan exclusion for `extensions/*/node_modules`, and current tests/docs do not show the proposed default exclusion. The PR is still open/unmerged, is the exact implementation thread that superseded #64144, and its later commits address the review-noted Windows separator and canonical state-path issues.

## Best Possible Solution

Keep this PR open for maintainer review. Either land or recreate the narrow default exclusion with the PR's Windows path normalization, canonical state-path handling, and regression tests, or explicitly include the same `extensions/*/node_modules` behavior in the broader backup exclude-pattern work before closing this PR.
