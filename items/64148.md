## Summary

Keep this PR open. Current main still archives the canonical state directory without a tar filter or extension-local node_modules exclusion, and the broader open backup exclude-pattern work is not merged and does not clearly supersede this default behavior. The PR is a narrow implementation candidate, and the provided/live discussion shows the author addressed the Windows separator and canonical state-path review feedback.

## Best Possible Solution

Keep this PR open for maintainer review. Either land or recreate the narrow default exclusion with canonical state-path matching, Windows path normalization, and regression coverage plus backup/restore docs; or explicitly fold the same extensions/*/node_modules default into #44288 before closing this PR.
