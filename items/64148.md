## Summary

Keep this PR open. Current main still backs up the resolved state directory as a whole and passes it directly to `tar.c()` without a filter or backup-plan pruning for `extensions/*/node_modules`. The PR is a focused implementation candidate for the closed paired feature issue, and its later commits address the review-noted Windows path separator and canonical state-path concerns. Security review found no new dependency, workflow, package-publishing, or external code execution changes in the provided PR diff; the remaining concern is product/restore semantics, not supply-chain expansion.

## Best Possible Solution

Keep this PR open for maintainer review. Either land or recreate the narrow default exclusion with Windows path normalization, canonical state-path handling, restore-facing docs/tests as needed, and regression coverage, or explicitly fold the same `extensions/*/node_modules` behavior into the broader backup exclude-pattern work before closing this PR.
