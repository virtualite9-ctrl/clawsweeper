## Summary

Atomic Chat is a useful optional local OpenAI-compatible model-provider integration, but current main already has the provider plugin, ClawHub/npm install, and self-hosted OpenAI-compatible helper surfaces needed to ship it outside core. The PR is not implemented on main and should be moved to a standalone community/provider plugin rather than added as another bundled provider.

## Best Possible Solution

Close the core PR as a ClawHub/community-plugin scope item. The best path is to extract the PR’s Atomic Chat provider into a standalone package, use the existing provider plugin SDK plus shared self-hosted OpenAI-compatible helpers, publish it through ClawHub or npm, and only reopen core work if a concrete missing plugin API is identified.
