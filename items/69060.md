## Summary

Keep PR #69060 open. Current main does not implement `x-openclaw-session`: the gateway still prioritizes only `x-openclaw-session-key`, then the OpenAI `user` field, then a random UUID fallback. The related #68920 is a broader latency/light-context issue and does not supersede this focused PR, which still appears to address one concrete remaining session-stability gap. Before merge, the best path is to add tests/docs and resolve the validation hardening question from the existing bot review.

## Best Possible Solution

Keep #69060 open as a plausible focused fix. The best maintainer path is to either update this PR or apply an equivalent main-branch change that adds `x-openclaw-session` as an agent-scoped fallback after `user`, documents it in the OpenAI HTTP API page, and adds targeted tests for priority order, stable reuse, different-agent isolation, and the random UUID fallback. Treat #68920 as the broader follow-up for light-context/voice-mode latency work, not as a replacement for this PR.
