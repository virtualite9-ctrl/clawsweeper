## Summary

Keep PR #69060 open. Current main still lacks the proposed `x-openclaw-session` fallback: `/v1/chat/completions` resolves only `x-openclaw-session-key`, then the OpenAI `user` field, then a random UUID. The linked #68920 remains a broader latency/light-context issue, so it does not supersede this focused session-stability PR. The PR should be updated with tests/docs and the existing validation hardening question before merge rather than closed as cleanup.

## Best Possible Solution

Keep #69060 open as a focused, still-relevant fix. The best maintainer path is to update this PR or land an equivalent change that adds `x-openclaw-session` after `user` and before random UUID fallback, keeps `x-openclaw-session-key` as the full override, documents the header in the OpenAI HTTP API docs, and adds targeted tests for priority order, stable reuse, different-agent isolation, and unchanged random fallback.
