## Summary

Keep #69060 open. Current main still does not implement the proposed bare `x-openclaw-session` header: the OpenAI-compatible HTTP path only honors `x-openclaw-session-key`, the request `user`, or a random UUID. The PR is a focused candidate for the stable-session subproblem from #68920, not obsolete cleanup.

## Best Possible Solution

Keep the PR open as a useful candidate. The best path is to update or reimplement it on current main with focused tests for `x-openclaw-session-key` precedence, `user` precedence, agent-scoped `x-openclaw-session` reuse, random fallback, docs for the public header, and a small validation policy for the session hint. Leave #68920 open for the separate lightContext/prompt assembly work.
