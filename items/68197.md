## Summary

Keep this PR open. Current main still has the LINE behavior described by the PR: non-text group messages bypass mention gating because LINE sets `canDetectMention` only for text messages, and the proposed `requireMentionForNonText` config key is not implemented in the LINE type or strict schema. The PR is a focused LINE plugin implementation candidate with tests in the provided diff, and the security review did not find unrelated CI, dependency, lockfile, install, release, or package-publishing changes.

## Best Possible Solution

Keep this PR open as an active implementation candidate. Maintainers should review the LINE-only opt-in, confirm the default-preserving behavior is desired, and then either land it with the included regression tests or request a docs/config-reference follow-up for the new LINE group setting.
