## Summary

Keep #39900 open. Current main still accepts provider-owned TTS config structurally, but the bundled OpenAI TTS provider does not preserve `extraBody`/`extra_body` or merge arbitrary fields into the `/audio/speech` request body. The closed duplicate #39909 should not supersede this because #39900 remains the canonical open item.

## Best Possible Solution

Keep the issue open and implement this in the bundled OpenAI speech provider: accept `extraBody` and `extra_body` as plain JSON objects, preserve them through config normalization, pass them into `openaiTTS`, merge them into the `/audio/speech` payload with documented precedence, cover normal and telephony synthesis, add focused `extensions/openai` tests, and document the field on the public OpenAI/TTS docs.
