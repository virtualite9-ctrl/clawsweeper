## Summary

Keep #39900 open. Current main still lacks `extraBody`/`extra_body` support in the bundled OpenAI-compatible TTS path: arbitrary config can now pass the generic TTS schema, but the OpenAI speech provider normalizes only known fields and sends only the fixed OpenAI `/audio/speech` payload.

## Best Possible Solution

Keep the issue open and implement a narrow provider-owned fix in the bundled OpenAI speech provider: accept `extraBody` and `extra_body` as JSON records, preserve them through OpenAI TTS config normalization, merge them into both normal and telephony `/audio/speech` request bodies with documented precedence, add focused OpenAI speech-provider tests, and document the field on https://docs.openclaw.ai/providers/openai.
