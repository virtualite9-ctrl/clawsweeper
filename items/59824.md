## Summary

Current `main` already carries a MiniMax-specific streaming compat fix: OpenClaw now injects `thinking: { type: "disabled" }` for MiniMax Anthropic streaming, applies it in the shared request path, tests it, and documents the behavior. That matches the issue’s reported `reasoning_content`/thinking leakage class closely enough to close as implemented on `main`.

## Evidence

- **MiniMax streaming compat wrapper:** `createMinimaxThinkingDisabledWrapper` exists specifically to stop MiniMax Anthropic streaming from leaking `reasoning_content` into visible output by injecting `thinking: { type: "disabled" }` when the caller did not set thinking explicitly.
  - file: [src/agents/pi-embedded-runner/minimax-stream-wrappers.ts:45](https://github.com/openclaw/openclaw/blob/16c7de085c99884367d34eaa9e52806d62ebf871/src/agents/pi-embedded-runner/minimax-stream-wrappers.ts#L45)

  - sha: [16c7de085c99](https://github.com/openclaw/openclaw/commit/16c7de085c99884367d34eaa9e52806d62ebf871)
- **Wrapper is applied on the main request path:** The shared extra-params pipeline applies the MiniMax thinking-disabling wrapper for outgoing streams, so MiniMax Anthropic requests get the compat behavior by default.
  - file: [src/agents/pi-embedded-runner/extra-params.ts:453](https://github.com/openclaw/openclaw/blob/16c7de085c99884367d34eaa9e52806d62ebf871/src/agents/pi-embedded-runner/extra-params.ts#L453)

## Risks / Open Questions

- The proving fix commit is on current `main`; it does not appear to be in the latest tagged release `v2026.4.23`, so released builds may still reproduce until the next ship.
- The wrapper intentionally preserves an explicit MiniMax `thinking` override, so a user who forces thinking back on could still re-enable the upstream problematic stream shape.
