## Summary

Current main still has the reported fast-return path for main-session `systemEvent` cron jobs, and I did not find the requested ghost-run warning, configurable threshold, or run-log annotation. The related implementation PR #63111 was closed unmerged, so it does not supersede or ship the fix.

## Best Possible Solution

Keep this issue open. The best maintainer path is to implement a scoped ghost-run signal in `src/gateway/server-cron.ts` for finished main-session `systemEvent` next-heartbeat runs, or change the cron-service outcome so queued heartbeat handoff is distinct from confirmed agent processing. Add a configurable threshold, regression coverage around fast `status: "ok"` main-session runs, and docs for the new config surface.
