## Summary

Keep #47606 open. Current main has adjacent pieces: durable background task records, stale task audit findings, an opt-in tool-call loop detector, CLI no-output watchdogs, and prompt-level execution guidance. Those do not implement the issue’s requested runtime guardrails for artifact-gated status messages, persisted anti-drift run state with blocker/escalation fields, escalation timers, or no-idle-after-green enforcement. Related ClawSweeper reports #71014 and #12565 are useful context but cover UI status visibility and tool-governance/security controls, not this reliability control.

## Best Possible Solution

Keep open. The best path is a focused core/runtime design: add a feature-flagged execution-guard state machine tied to agent lifecycle and chat/status emission, persist guard state across restart, define artifact types, enforce non-terminal status gating and repeated-status circuit breaking, add configurable escalation/no-artifact/no-phase-change/post-green timers with cooldown, emit structured guard events, and cover normal runs plus detached tasks with regression tests.
