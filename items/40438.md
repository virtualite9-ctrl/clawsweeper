## Summary

Keep this PR open. Current main still lacks a configurable directory exclusion contract for memory indexing, so the central feature remains useful. It should not be merged as-is because the memory implementation moved since the PR was written and the PR includes an unrelated process-wide network preloader.

## Best Possible Solution

Keep this PR open and salvage the focused feature: rebase onto current main, remove ipv4-fix.cjs, add an additive memorySearch.excludeDirs or broader exclude-pattern contract across schema/types/help/generated docs, thread it through resolveMemorySearchConfig, packages/memory-host-sdk/src/host/internal.ts, and extensions/memory-core sync/status/watch paths, skip excluded roots as well as descendants, and either implement equivalent QMD exclusion semantics or document QMD scope explicitly with tests.
