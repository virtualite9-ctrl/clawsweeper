## Summary

Keep this PR open. Current main still does not provide the requested `final_reply_payloads` hook or an equivalent final-delivery plugin seam that can see resolved provider/model/usage context and modify finalized payloads before the usage footer is appended. Existing response-prefix model templating helps one user-facing model-tag use case, but it is not a plugin hook and does not replace the PR's API request.

## Best Possible Solution

Keep the PR open for maintainer/API review. The best path is to rebase and finish a supported core plugin API change that gives plugins stable final-delivery access to resolved provider/model and usage context, caches the global hook runner at the call site, updates hook types/runner exports/docs, and adds focused tests for final reply assembly, usage-footer ordering, and fallback-selected model versus actual runtime model. Coordinate the final shape with #69110 rather than closing this implementation candidate before a replacement lands.
