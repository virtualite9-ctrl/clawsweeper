## Summary

Keep this PR open. Current main still does not provide the requested `final_reply_payloads` hook or an equivalent final-delivery plugin seam that can see resolved provider/model/usage context and modify finalized payloads before the usage footer is appended. The PR is not safe to merge as-is without API review, docs, and tests, but it remains a real implementation candidate rather than obsolete cleanup.

## Best Possible Solution

Keep this PR open for maintainer/API review. The best path is to rebase it on current main, coordinate the final model-identity/delivery boundary shape with #69110, cache the global hook runner at the call site, update hook types/runner exports/public docs, and add focused tests proving final payload modification occurs before usage-footer append and receives the resolved provider/model/usage context.
