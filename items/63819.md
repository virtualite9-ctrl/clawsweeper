## Summary

Keep open. Current main still has the timer-only pending terminal lifecycle-error path described by the reporter's workaround, and a later non-error event can still clear the deferred terminal cleanup before session lifecycle state is finalized. The issue is recent, has a second-user confirmation, and has no current source/test evidence proving this v2026.4.9 stuck-running timeout regression is fixed.

## Best Possible Solution

Add a focused regression test in src/gateway/server-chat.agent-events.test.ts for lifecycle start/error scheduling pending terminal cleanup, then a later non-error event arriving without lifecycle end, and require terminal lifecycle persistence/cleanup rather than leaving the session running. Then update src/gateway/server-chat.ts so pendingTerminalLifecycleErrors retains the deferred event/options and finalizes it when clearing is not part of an intentional retry/fallback replacement, preserving the existing fallback-success behavior.
