## Summary

Keep #12208 open. Current main still does not implement the requested behavior: native chat bubbles render their body content in plain SwiftUI `VStack`s and Android Compose `Column`s, while scrolling exists only at the transcript/list level. The latest provided issue discussion already reached the same keep-open conclusion, and there are no affected-file changes between that reviewed SHA and current `main`.

## Best Possible Solution

Keep the issue open for a focused native chat UI fix. Define the current equivalent of `.large` or tall bubbles, wrap the message body area in a bounded vertical scroll container in `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatMessageViews.swift`, add Android parity in `apps/android/app/src/main/java/ai/openclaw/app/ui/chat/ChatMessageViews.kt`, preserve transcript-level scrolling, and add focused render coverage or documented manual mobile verification for tall-but-not-truncated content.
