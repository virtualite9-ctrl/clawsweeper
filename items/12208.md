## Summary

Keep #12208 open. Current main still does not implement a bounded inner scroll body for every large native chat bubble; the shared SwiftUI and Android chat paths render bubble bodies as direct stacks inside only the outer transcript/list scroll. The existing steipete/Codex issue comment also kept this open, and the current checkout has not superseded that finding.

## Best Possible Solution

Keep #12208 open for a focused native chat UI/product decision. The likely implementation belongs in `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatMessageViews.swift`, with Android parity in `apps/android/app/src/main/java/ai/openclaw/app/ui/chat/ChatMessageViews.kt`: define the current equivalent of a large bubble, wrap only the bounded body area in a vertical scroll container, preserve outer transcript scrolling, and add focused render/smoke coverage or documented manual mobile verification for tall-but-not-truncated content.
