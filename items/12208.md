## Summary

Keep #12208 open. Current main still renders native chat bubble bodies as unbounded plain SwiftUI/Compose stacks inside only the outer transcript scroll, so the requested inner scroll behavior for all large bubbles is not implemented.

## Best Possible Solution

Keep #12208 open for a focused native chat UI fix. Define the current equivalent of a large bubble, wrap only the bounded message body area in an inner vertical scroll container in `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatMessageViews.swift`, add Android parity in `apps/android/app/src/main/java/ai/openclaw/app/ui/chat/ChatMessageViews.kt`, preserve the existing outer transcript scroll behavior, and add focused render/smoke coverage or documented manual mobile verification for tall-but-not-truncated content.
