## Summary

Keep #9820 open. Current main still does not add native JSONL support for `memorySearch.extraPaths`: discovery, `memory_get`, watcher globs, tests, and docs remain Markdown plus enabled multimodal media only. Existing session-transcript JSONL indexing is a separate `sessions` source and does not satisfy arbitrary raw JSONL logs from message hooks.

## Best Possible Solution

Keep #9820 open until maintainers either add native JSONL support to builtin `memorySearch.extraPaths` or document a narrower product direction. A complete fix should update file admission, watcher globs, chunking/line mapping, `memory_get` rendering, tests, config help/types, and memory docs; if arbitrary JSONL is out of scope, document the intended session-memory, QMD/custom collection, or plugin-owned converter path.
