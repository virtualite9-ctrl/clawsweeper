## Summary

Keep #9820 open. Current main still does not provide native JSONL support for builtin memorySearch.extraPaths: extra-path discovery admits .md files plus enabled multimodal media only, memory_get rejects non-.md reads, watcher configuration is Markdown/media-oriented, and docs/types still describe extra paths as Markdown files or directories. Existing session-transcript JSONL indexing is a separate sessions source and does not satisfy arbitrary raw JSONL logs configured through extraPaths.

## Best Possible Solution

Keep tracking #9820 until maintainers either add native JSONL support to builtin memorySearch.extraPaths or explicitly document a narrower product direction. A complete fix should update extra-path admission, watcher globs, indexing/chunking with JSONL line mappings, memory_get/read behavior, focused tests, config help/types, and the public memory docs. If arbitrary JSONL should stay outside builtin memory, document the intended alternative through session transcript memory, QMD/custom collections, or a plugin-owned converter path.
