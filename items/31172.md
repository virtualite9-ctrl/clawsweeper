## Summary

Keep PR #31172 open. Current main still does not implement a configurable Feishu streaming card throttle: Feishu uses a fixed 160 ms runtime throttle, the dispatcher does not pass a throttle value from account config, the strict validated config schemas omit `streamingThrottleMs`, and the manifest/docs/tests do not expose or cover the setting. The linked issue #31136 was stale-closed, not shipped, and the PR author recently reiterated the Feishu stability need.

## Best Possible Solution

Keep the PR open for maintainer/plugin-owner follow-up. A mergeable Feishu plugin patch should add `streamingThrottleMs` to the shared Feishu config schema and manifest metadata, preserve the account/top-level merge semantics, pass the resolved value into `FeishuStreamingSession`, update the public Feishu docs, and add tests for default behavior plus configured throttle plumbing. The default should be decided deliberately against current main's 160 ms behavior and the PR's 100 ms compatibility argument.
