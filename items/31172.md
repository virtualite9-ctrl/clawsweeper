## Summary

Keep PR #31172 open. Current main still does not implement a Feishu `streamingThrottleMs` config path: the streaming session uses a hardcoded 160ms throttle, the reply dispatcher does not pass a config-derived throttle, and the Feishu schema/manifest/docs/tests do not expose or verify the requested setting. The linked issue #31136 was stale-closed rather than implemented, and the author recently reaffirmed the need for Feishu channel stability.

## Best Possible Solution

Keep the PR open for maintainer review or a rebased patch. A complete solution should add `streamingThrottleMs` to the Feishu config schema and plugin metadata, pass the resolved account/top-level value into `FeishuStreamingSession`, document it on the public Feishu channel docs page, and add tests for the default throttle plus configured throttle path. Maintainers should decide whether the default should preserve current main's 160ms behavior or use the PR's proposed 100ms.
