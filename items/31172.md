## Summary

Keep PR #31172 open. Current main still does not implement a Feishu `streamingThrottleMs` configuration path: the runtime uses a hardcoded 160ms throttle, the dispatcher constructs `FeishuStreamingSession` without a config-derived value, and schema/manifest/docs/tests do not expose or verify the requested setting. The linked issue #31136 was stale-closed rather than implemented, and the author recently reaffirmed the need for Feishu channel stability.

## Best Possible Solution

Keep the PR open for maintainer review or a rebased patch. A complete solution should add `streamingThrottleMs` to the Feishu config schema and plugin metadata, pass the resolved account/top-level value into `FeishuStreamingSession`, document it on `https://docs.openclaw.ai/channels/feishu`, and add tests for the default throttle and configured throttle path. Maintainers should decide whether the default should preserve current main's 160ms behavior or change to the PR's proposed 100ms.
