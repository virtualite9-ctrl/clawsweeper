## Summary

Keep PR #31172 open. Current main still does not expose or plumb a Feishu `streamingThrottleMs` config value: the streaming card session uses a fixed 160ms throttle, the dispatcher constructs the session without a throttle argument, and Feishu schema/manifest/docs/tests do not cover the requested setting. The linked issue #31136 was stale-closed rather than shipped, and the PR discussion includes recent author follow-up that the Feishu stability need remains.

## Best Possible Solution

Keep the PR open for maintainer review or a rebased patch. The best path is to add `streamingThrottleMs` to the Feishu config schema and plugin manifest, pass the resolved account/top-level value into `FeishuStreamingSession`, document it on the Feishu channel docs page, and add tests for both the default throttle and configured throttle plumbing. Maintainers should decide whether the default should preserve current main's 160ms behavior or use the PR's proposed 100ms.
