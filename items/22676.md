## Summary

Current main still exposes the Signal restart race described in #22676. The Signal daemon stop contract remains synchronous, monitor teardown does not await signal-cli exit, and gateway hot reload can start the channel again immediately after stopChannel returns. The issue should remain open while PR #71863 or an equivalent fix is reviewed.

## Best Possible Solution

Keep #22676 open until Signal daemon shutdown is made asynchronous and awaited through monitor/channel reload teardown: send SIGTERM, wait for process exit, escalate to SIGKILL on timeout, and only then allow restart. PR #71863 appears to be the active candidate; maintainers should review it or land an equivalent scoped fix with regression coverage for daemon stop semantics and aborted/reload shutdown.
