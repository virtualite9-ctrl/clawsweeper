## Summary

Keep #22676 open. Current main still has the reported Signal daemon restart race: Signal daemon stop is synchronous fire-and-forget, monitor teardown does not await signal-cli exit, and channel hot reload stops then immediately starts Signal. The focused fix is still represented by open PR #71863 rather than landed on main.

## Best Possible Solution

Keep #22676 open until PR #71863 or an equivalent focused fix lands on main. The fix should make Signal daemon stop asynchronous, send SIGTERM, wait for signal-cli exit, escalate to SIGKILL on timeout, and await that shutdown through monitor/channel reload teardown before starting a replacement Signal channel, with regression coverage for stop wait/escalation and abort or reload-driven monitor shutdown.
