## Summary

Keep this PR open. Current main still rejects workspace bootstrap symlinks that resolve outside the workspace boundary and then renders those files as missing, while the linked reports and discussion show a concrete user-facing bug. The PR remains a plausible implementation candidate, subject to maintainer/security review because it creates a narrow exception to the workspace boundary reader.

## Best Possible Solution

Keep this PR open for explicit maintainer review or replacement. The best path is a narrow change in `src/agents/workspace.ts` that intentionally supports symlinked fixed top-level bootstrap files while preserving max-size, regular-file, hardlink, and fd/same-file race protections, plus regression tests for external absolute and relative symlinks, dangling symlinks, oversized targets, hardlinked targets, non-regular targets, and extra-bootstrap traversal boundaries. If maintainers choose to support this behavior, update the workspace/context docs and consolidate the overlapping #38622, #38650, #40210, and #64472 threads around the chosen fix.
