## Summary

Keep this PR open. Current main still reports externally symlinked workspace bootstrap files as missing because the fixed bootstrap loader goes through the boundary-file reader with no symlink fallback, while this PR is a focused implementation candidate for the open user-facing bug linked by #40210, #38622, and #64472. It is security-sensitive, so the right maintainer action is review or replacement, not cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer/security review or replace it with a maintainer-authored patch. The best fix belongs in `src/agents/workspace.ts`: support symlinked fixed top-level bootstrap files only, preserve boundary-reader safety properties as closely as possible, keep extra/glob bootstrap paths constrained to the workspace, and include regression tests for absolute and relative external symlinks, symlink-to-hardlink targets, dangling symlinks, non-regular targets, oversized targets, and extra-bootstrap rejection. After one patch lands, close the overlapping implementation/report items against the chosen canonical fix.
