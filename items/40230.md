## Summary

Keep this PR open. Current main still lacks a symlink fallback for workspace bootstrap files, and the PR is a plausible focused implementation for open user-facing bug reports. Because it deliberately changes boundary-file behavior around workspace-root validation, it needs maintainer/security review rather than conservative cleanup closure.

## Best Possible Solution

Keep this PR open for explicit maintainer/security review or replacement. The best implementation is a narrow change in `src/agents/workspace.ts` that supports symlinked fixed top-level bootstrap files only, preserves the existing boundary-reader safety properties, includes regression tests for external absolute and relative symlinks plus hardlink, dangling symlink, non-regular target, max-size, and extra-bootstrap rejection cases, and then consolidates the related open issues/PRs around the chosen patch.
