## Summary

Keep this PR open. Current main still lacks the central update invariant: skipped update results are still serialized as successful RPC payloads, the Control UI still only reacts to explicit `ok: false`, update-triggered restarts still use the generic restart path without update-specific health/version verification, and there is no gateway `update.status` surface on main. The PR is an active implementation candidate with recent author follow-up and review fixes in the provided discussion.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to review and either land or narrow a core fix that makes skipped/error `update.run` outcomes non-ok, surfaces stable reason codes in the Control UI, blocks unsafe global package self-updates before replacing files, and performs update-specific process replacement with health/version verification. Before merge, maintainers should verify CI, the process-respawn security posture, the admin-only `update.status` exposure, and whether the unrelated UI defensive change should be split.
