## Summary

Current `main` does not already implement this PR’s requested behavior. The cleanup-hint path still generates commands for the active gateway from the current profile instead of the detected extra services, and both doctor/status still call that helper with no service list. The doctor-specific SecretRef leniency described in the PR is also not present on `main`.
