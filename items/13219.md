## Summary

Keep #13219 open. Current main still has transcript/session-log based usage summaries, Control UI aggregation, provider quota snapshots, and optional OTel diagnostics, but it does not implement the requested native append-only per-call usage ledger with durable logs/usage.jsonl-style storage, event ids, source/job/run attribution, or ledger-first readers. The issue discussion also explicitly distinguishes adjacent work such as #49064, #70205, burn0, and provider quota tracking from this ledger request.

## Best Possible Solution

Keep #13219 open until OpenClaw adds a typed append-only UsageLedgerEntry written from the normalized model-call usage/cost seam. The ledger should store provider/model, agentId, sessionKey, sessionId, runId/callId, source/job attribution, token/cache/cost fields, status, cost status/breakdown, latency, and idempotency metadata without prompt/completion content. Then usage.cost, session usage readers, CLI/UI summaries, and a backfill command should prefer the ledger while preserving transcript scanning as compatibility fallback; provider quota snapshots and optional real-time external tools should remain separate adjacent surfaces.
