## Summary

Keep #13219 open. Current main has useful adjacent usage surfaces: transcript-backed cost summaries, per-session/model dashboard data, provider quota snapshots, plugin telemetry hooks, and a gateway `usage-cost` command. It still lacks the requested native append-only per-call usage ledger with durable event IDs, source/job/run attribution, idempotency/backfill semantics, and ledger-first readers. The latest issue discussion explicitly says #49064 and #70205 are adjacent transcript-backed work rather than closure for this ledger request.

## Best Possible Solution

Keep #13219 open until core adds a typed append-only `UsageLedgerEntry` written at the normalized model-call usage/cost seam. The ledger should store provider/model, agent/session/run/source/job attribution, token/cache/cost fields, cost status/breakdown, status, latency, and durable idempotency metadata without prompt or completion content. Then `usage.cost`, CLI/UI summaries, and an explicit backfill command should prefer the ledger while preserving transcript scanning as a compatibility fallback; provider quota snapshots and optional real-time plugin/tool integrations should remain adjacent surfaces.
