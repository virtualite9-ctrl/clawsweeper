## Summary

Keep #13219 open. Current main has transcript-backed cost summaries, UI/dashboard views, provider quota snapshots, diagnostics telemetry, and cron-specific run-log usage telemetry, but it still lacks the requested native append-only per-call usage ledger / usage.jsonl-style log with durable event ids, source/job/run attribution, idempotency, and ledger-first readers.

## Best Possible Solution

Keep #13219 open until OpenClaw adds a typed append-only UsageLedgerEntry written at the normalized model-call usage/cost seam. The ledger should store provider/model, agentId, sessionKey/sessionId, run/call attribution, source/job metadata, token/cache/cost fields, status, cost status/breakdown, latency, and idempotency metadata without prompt/completion content. Then usage.cost, CLI/UI summaries, and a backfill command should prefer the ledger while preserving transcript scanning as a compatibility fallback; provider quota snapshots and optional real-time/plugin tools should remain adjacent surfaces.
