## Summary

Keep this PR open. Current main still awaits link-understanding on the inbound reply path without a local non-fatal guard, and dispatch errors still bubble up to channel handlers where Feishu logs the same failed-dispatch symptom from the linked report. The PR is narrow and relevant, but it should get maintainer review and ideally a regression test before landing.

## Best Possible Solution

Keep the PR open for maintainer review or replacement. The best fix is to make link-understanding enrichment non-fatal on the shared reply path, add a focused regression test for rejected link-understanding runtime/import behavior, and consider a follow-up or expanded patch for the symmetric media-understanding lazy import boundary.
