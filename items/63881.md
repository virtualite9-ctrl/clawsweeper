## Summary

Keep open. Current `main` does not implement the PR’s MCP deny-list hardening: the shared dangerous-tool list and `gateway.tools.allow`/`deny` are still scoped to HTTP, while the plugin-tools MCP bridge only strips `ownerOnly` tools and wraps `before_tool_call` hooks. The docs also describe the plugin-tools bridge as an explicit opt-in trust-boundary expansion, so whether to add HTTP-style deny defaults on MCP is still a maintainer security/product decision, not a conservative auto-close.
