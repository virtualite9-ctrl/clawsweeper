## Summary

Keep #47452 open. Current main still has no Qiniu provider/replay policy and the generic `anthropic-messages` fallback only drops replayed thinking blocks for Claude model IDs. A custom provider such as `qiniu/moonshotai/kimi-k2.5` therefore resolves to `dropThinkingBlocks: false`, despite the reporter configuring `reasoning: false` and `compat.supportsReasoningEffort: false`. The related PR #47459 was closed unmerged, and its proposed `dropThinkingBlockModelHints` mechanism is not present in current main.

## Best Possible Solution

Keep the issue open and fix the custom Anthropic-compatible provider path. A conservative fix would add a documented replay-policy/config contract for custom providers or honor explicit model compatibility such as `compat.supportsReasoningEffort: false` / supported content types when deciding whether to strip or convert `thinking` and `redacted_thinking` blocks. Add a focused regression test for `provider: "qiniu"`, `api: "anthropic-messages"`, `modelId: "moonshotai/kimi-k2.5"` proving thinking blocks are removed before replay/send.
