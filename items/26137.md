## Summary

Keep #26137 open. Current main has meaningful partial supply-chain/security hardening, including SECURITY.md, Dependabot, CodeQL, CI security audit jobs, and npm provenance, but the central OpenSSF Best Practices/Scorecard/tooling umbrella is still not implemented, and the existing issue discussion already reached the same keep-open conclusion.

## Best Possible Solution

Keep #26137 open as the umbrella supply-chain hardening request, or split it into maintainer-owned tasks for OpenSSF Scorecard, SECURITY-INSIGHTS.yml, Dependency Review Action, OSV Scanner, Best Practices Badge/application, remaining GitHub Actions SHA pinning, Allstar/org policy, branch protection and signed-commit decisions, and SLSA/release attestations. Preserve SECURITY.md, Dependabot, CodeQL, zizmor, dependency audit, and npm provenance as partial progress rather than treating them as full implementation.
