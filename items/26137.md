## Summary

Keep #26137 open. Current main has useful partial supply-chain hardening, but the central OpenSSF umbrella request is still not implemented: no Scorecard workflow/badge, Best Practices badge reference, SECURITY-INSIGHTS file, Dependency Review Action, OSV Scanner, Allstar policy, SLSA/GitHub Release attestation workflow, or full GitHub Actions SHA pinning. The existing 2026-04-26 Codex/maintainer-facing comment also explicitly kept this issue open for the same reason, and related #57265 described #26137 as proposed but not implemented rather than superseding it.

## Best Possible Solution

Keep #26137 open as the umbrella supply-chain hardening tracker, or split it into maintainer-owned follow-ups for OpenSSF Scorecard, SECURITY-INSIGHTS.yml, Dependency Review Action, OSV Scanner, Best Practices Badge/application and README badge, remaining Actions SHA pinning, Allstar/org policy, branch protection/signed-commit decisions, and SLSA/GitHub Release attestations. Preserve SECURITY.md, Dependabot, CodeQL/zizmor/private-key checks, dependency audit, and npm --provenance as partial progress.
