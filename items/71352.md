## Summary

Keep open. Current `main` still drops `error.message` and `error.stack` from diagnostic stability bundles, the existing tests explicitly lock in that omission, and the bundle writer is already wired into real startup/fatal-error paths. This is a real current behavior gap, not something already implemented or safe to auto-close.

## Evidence
