## Summary

Keep this PR open. Current main already superseded the setup `allowFrom` username-resolution direction by making Telegram setup numeric-ID-only and moving legacy username repair to doctor, but the PR's target-resolution hardening is still not implemented on main. Because the remaining piece touches the channel resolver contract and an access-control-adjacent Telegram setup surface, this is a maintainer review/rebase decision rather than a conservative cleanup close.

## Best Possible Solution

Keep the PR open for contributor or maintainer follow-up. Rebase it onto current main, drop the obsolete setup username-resolution pieces, preserve the numeric-only Telegram allowFrom contract, and decide whether to add focused defensive target-resolution coercion or keep the public string-only resolver contract unchanged.
