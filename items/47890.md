## Summary

Current `main` does not implement the requested Feishu outbound `sendPayload` path for interactive cards. The Feishu outbound adapter still only exposes `sendText` and `sendMedia`, while the generic outbound pipeline only routes `channelData`/interactive payloads through `sendPayload`. `sendCardFeishu()` exists on `main`, but it is not wired into `extensions/feishu/src/outbound.ts`, so the issue remains actionable.

## Evidence

- **Feishu outbound adapter has no sendPayload:** `feishuOutbound` defines `deliveryMode`, chunking, then spreads `createAttachedChannelResultAdapter(...)` with only `sendText` and `sendMedia`; there is no `sendPayload` implementation in this adapter.
  - file: [extensions/feishu/src/outbound.ts:152](https://github.com/openclaw/openclaw/blob/d38ed0831d7372c8505dc944aea69b5b83da1435/extensions/feishu/src/outbound.ts#L152)
  - sha: [d38ed0831d73](https://github.com/openclaw/openclaw/commit/d38ed0831d7372c8505dc944aea69b5b83da1435)
- **Interactive card sender exists but is not connected to outbound payload delivery:** `sendCardFeishu()` sends Feishu messages with `msgType: "interactive"`, and `sendStructuredCardFeishu()` / `sendMarkdownCardFeishu()` both reuse it. This proves card transport exists separately from the outbound adapter.
  - file: [extensions/feishu/src/send.ts:495](https://github.com/openclaw/openclaw/blob/d38ed0831d7372c8505dc944aea69b5b83da1435/extensions/feishu/src/send.ts#L495)
  - sha: [d38ed0831d73](https://github.com/openclaw/openclaw/commit/d38ed0831d7372c8505dc944aea69b5b83da1435)
- **Channel contract still expects sendPayload for rich payload delivery:** `ChannelOutboundAdapter` includes an optional `sendPayload(ctx)` hook for outbound payload handling.
  - file: [src/channels/plugins/outbound.types.ts:131](https://github.com/openclaw/openclaw/blob/d38ed0831d7372c8505dc944aea69b5b83da1435/src/channels/plugins/outbound.types.ts#L131)
  - sha: [d38ed0831d73](https://github.com/openclaw/openclaw/commit/d38ed0831d7372c8505dc944aea69b5b83da1435)
- **Outbound pipeline only uses rich payload path when adapter implements sendPayload:** `deliverOutboundPayloads()` calls `handler.sendPayload(...)` only when the adapter provides it and the payload has presentation/interactive/channelData content.
  - file: [src/infra/outbound/deliver.ts:876](https://github.com/openclaw/openclaw/blob/d38ed0831d7372c8505dc944aea69b5b83da1435/src/infra/outbound/deliver.ts#L876)
