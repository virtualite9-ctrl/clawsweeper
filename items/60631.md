## Summary

Keep open. Current `main` still parses Feishu card-action callbacks using only `context.open_id`, `context.user_id`, and `context.chat_id`, and drops payloads that do not satisfy that shape as malformed. I did not find a main-branch fix or test coverage for the reported callback field mismatch.

## Evidence

- **Current parser still hard-requires the old context fields:** `parseFeishuCardActionEventPayload` reads `context.open_id`, `context.user_id`, and `context.chat_id`, then returns `null` unless all three are present.
  - file: [extensions/feishu/src/monitor.account.ts:184](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/extensions/feishu/src/monitor.account.ts#L184)

  - sha: [6bc0dc8fb671](https://github.com/openclaw/openclaw/commit/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440)
- **Malformed callbacks are silently dropped at the event handler boundary:** The `card.action.trigger` handler calls the parser and logs `ignoring malformed card action payload` before returning when parsing fails.
  - file: [extensions/feishu/src/monitor.account.ts:370](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/extensions/feishu/src/monitor.account.ts#L370)
