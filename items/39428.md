## Summary

Keep open. Current `main` at `cb2c36b04975e34ab7014a11575d6589aa96ae32` still routes Discord inbound traffic only through explicit binding tiers and otherwise falls back to the default agent; there is no account-name-to-agent auto-match on `main`. Docs and diagnostics also describe explicit account-scoped bindings as the supported configuration.

## Evidence

- **Routing code still falls back to default agent:** `resolveAgentRoute` checks peer/guild/team/account/channel binding tiers, then returns `choose(resolveDefaultAgentId(input.cfg), "default")`. There is no branch that maps `accountId` to a same-named `agentId`.
