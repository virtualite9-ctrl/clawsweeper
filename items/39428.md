## Summary

Keep #39428 open. Current main still routes Discord inbound messages through the generic binding resolver, and that resolver falls through to the configured default agent when no explicit binding matches. The pending PR #39485 directly addresses the requested Discord accountId-to-agentId fallback, but it is not on current main.

## Best Possible Solution

Keep #39428 open until PR #39485 or an equivalent maintainer-approved routing change lands on main. The likely fix belongs after explicit binding tiers and before default-agent fallback, should avoid surprising overrides for the literal default account, and should add regression coverage for Discord accountId matching plus default-account behavior.
