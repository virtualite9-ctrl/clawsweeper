## Summary

Current `main` still backs up the entire state tree, and installed plugins live under `~/.openclaw/extensions`, so `extensions/*/node_modules` remains part of backup archives today. The request is coherent, matches current code/docs, and there is related unlanded work (#64148), so this is not a conservative close.

## Evidence

- **Backup plan includes the whole state directory:** `resolveBackupPlanFromDisk()` resolves `stateDir` and `resolveBackupPlanFromPaths()` adds that directory as the `state` backup asset before any archiving. There is no exclusion for `extensions/*/node_modules` under the state tree.
  - file: [src/commands/backup-shared.ts:146](https://github.com/openclaw/openclaw/blob/5b8bd6371c666a5de43c82305e0315771e8e49d5/src/commands/backup-shared.ts#L146)

- **Archive creation has no tar filter:** `createBackupArchive()` passes each asset root directly to `tar.c()` and only remaps archive paths via `onWriteEntry`. The `tar.c()` options do not include a filter callback or any logic to skip nested `node_modules` directories.
  - file: [src/infra/backup-create.ts:353](https://github.com/openclaw/openclaw/blob/5b8bd6371c666a5de43c82305e0315771e8e49d5/src/infra/backup-create.ts#L353)
