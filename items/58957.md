## Summary

Keep open. Current main has partial context-overflow improvements, but it does not fully solve the issue discussion’s remaining failure modes: proactive preflight compaction still skips heartbeat turns, skips sessions with fresh persisted token totals, and computes preflight/memory-flush thresholds from `reserveTokensFloor` rather than the configured effective `reserveTokens`. The issue is recent, has concrete follow-up analysis, and overlaps an open canonical sub-bug/PR rather than being fully implemented or obsolete.

## Best Possible Solution

Keep #58957 open until the remaining compaction/model-switch UX is fixed or split into canonical follow-ups. The best implementation path is to make proactive compaction evaluate fresh persisted token counts directly, decide whether heartbeat turns should compact or use isolated sessions by policy, compute thresholds from the effective reserve `max(reserveTokens, reserveTokensFloor)`, and add regression tests covering long active sessions, heartbeat-only sessions, model fallback on context overflow, and model switch recovery messaging. #66830/#69379 can cover the reserve-token asymmetry, but they do not fully supersede the broader issue.
