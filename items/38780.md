## Summary

Keep PR #38780 open. Current main still lacks the PR's central self-elected continuation surface: `continue_work`, `continue_delegate`, `request_compaction`, fallback continuation tokens, continuation config, delegate store/runtime, and gateway continuation trigger/drain fields. The restart-continuation work on main is related substrate, not a replacement for the PR.

## Best Possible Solution

Keep PR #38780 open for explicit maintainer review. If maintainers want this capability in core, the best path is a smaller opt-in continuation runtime that reuses the shipped restart/session-delivery, heartbeat, subagent, compaction, and TaskFlow substrate; updates config, protocol, clients, docs, and tests coherently; and resolves the branch review comments before merge. If maintainers decide the product direction is wrong or belongs outside core, close it deliberately as a maintainer/product decision rather than conservative cleanup.
