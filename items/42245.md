## Summary

Keep this PR open. Current main still lacks the missing-`--agent` warning for `openclaw cron add --message`, and the runtime still falls back to the configured default agent when no job agent is supplied. The related maintainer PR #72326 is adjacent session-agent inference work, not a duplicate or shipped replacement for this CLI warning.

## Best Possible Solution

Keep this PR open as a candidate fix for #42196. The best path is to refresh it on current main, keep the warning scoped to `cron add` agent-turn jobs, emit it on stderr so JSON stdout remains parseable, include regression coverage for missing/blank/explicit agent and system-event cases, and make the warning wording consistent with the configured-default-agent behavior.
