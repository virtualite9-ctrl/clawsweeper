## Summary

Keep this PR open. Current main still accepts `openclaw cron add --message ...` without `--agent`, forwards `agentId: undefined`, and has no missing-agent warning path or regression coverage. The PR remains a focused candidate for the linked cron UX issue, and the adjacent cron-tool agent-inference work is a different surface.

## Best Possible Solution

Keep this PR open as an implementation candidate.
