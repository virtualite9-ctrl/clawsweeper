## Summary

Keep open. Current main still enforces workspace-root-only boundary checks, so the linked workspace-bootstrap symlink bug is not implemented. This PR is paired with the still-open linked issue and is a plausible implementation candidate, but the submitted patch needs revision because review comments identify real path-boundary widening and missing coverage.

## Best Possible Solution

Keep this PR and #64472 open, but require a safer revision before merge: scope any exception to the intended workspace-bootstrap path, canonicalize the OpenClaw state root before comparison, avoid allowing direct absolute paths to sibling `~/.openclaw` directories, gate the exception to appropriate workspace roots, and add positive and negative regression tests.
