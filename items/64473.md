## Summary

Keep open. Current `main` still enforces workspace-root-only boundary checks, so the linked workspace-bootstrap symlink bug is not implemented. This PR is paired with the still-open linked issue #64472 and remains a plausible implementation candidate, but the submitted patch needs a safer revision because it relaxes a shared security boundary too broadly.

## Best Possible Solution

Keep this PR and #64472 open, but require a revised implementation before merge. The safer path is a narrow, tested exception for the intended workspace-bootstrap symlink traversal only: canonicalize the OpenClaw state root, avoid allowing direct absolute paths to sibling `~/.openclaw` directories, gate the exception to appropriate workspace roots or a dedicated boundary mode, preserve rejection for credentials/session paths and non-workspace boundaries, and add positive and negative regression tests.
