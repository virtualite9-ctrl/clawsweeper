## Summary

Current `main` already suppresses JSON `{"action":"NO_REPLY"}` control envelopes through the shared silent-payload path, preserves media while blanking silent envelope text, and avoids suppressing JSON objects that include extra fields. The shipped fix is documented in the `2026.3.28` changelog entry for `#56612`, so this PR's JSON-leak fix is already covered on `main`; I did not find a remaining live `channelData: { action: "NO_REPLY" }` runtime path in the current tree.

## Evidence

- **Shared JSON silent-envelope detector on main:** `isSilentReplyEnvelopeText` treats only strict single-key JSON `{ "action": "NO_REPLY" }` envelopes as silent, and `isSilentReplyPayloadText` routes both exact-token and JSON-envelope cases through the same helper.
  - file: [src/auto-reply/tokens.ts:46](https://github.com/openclaw/openclaw/blob/55389f4be29359e2dd57ebaf548ba4a70331a445/src/auto-reply/tokens.ts#L46)
- **Reply normalization clears silent JSON text while preserving media:** `normalizeReplyPayload` resolves the effective `silentToken` first and then clears any `isSilentReplyPayloadText(...)` payload to empty text unless the whole payload is silent, which covers the media-present case without leaking the JSON text.
  - file: [src/auto-reply/reply/normalize-reply.ts:55](https://github.com/openclaw/openclaw/blob/55389f4be29359e2dd57ebaf548ba4a70331a445/src/auto-reply/reply/normalize-reply.ts#L55)
