## Summary

Current `main` already sources the dashboard's Daily Token Usage from OpenClaw session/transcript data, not from provider-wide API-key usage. The provider usage endpoint still exists, but it is a separate code path and is not what the usage UI requests.

## Close Comment

Closing from a Codex review. Current `main` already routes the dashboard usage view through `sessions.usage` and `usage.cost` (`ui/src/ui/controllers/usage.ts:185`), not through the provider-wide `usage.status` path. In the gateway, `usage.status` is explicitly the provider usage/quota endpoint (`src/gateway/server-methods/usage.ts:393`, `docs/gateway/protocol.md:283`), while the dashboard usage data is aggregated from OpenClaw session/transcript files (`src/infra/session-cost-usage.ts:388`, `src/infra/session-cost-usage.ts:697`). The same request path is already present in the shipped `v2026.4.22` release (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`). If you still see a mismatch on `v2026.4.22` or newer, please open a new issue with a fresh repro, screenshots, and the exact dashboard payload/version so maintainers can compare the live data path.
