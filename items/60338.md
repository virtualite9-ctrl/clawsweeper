## Summary

Current `main` and the shipped `v2026.4.23` release no longer install deprecated `node-domexception@1.0.0` from a root `gaxios` chain: `node-domexception` is overridden to `@nolyfill/domexception`, and `gaxios@7.1.4` is scoped to the Google Chat plugin rather than the repo root.

## Evidence

- **Root manifest overrides deprecated package name:** The root manifest now aliases `node-domexception` to `npm:@nolyfill/domexception@1.0.28` in both the top-level `overrides` block and the `pnpm.overrides` block, which directly addresses the deprecated transitive package named in the report.
  - file: [package.json:1660](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/package.json#L1660)
  - command: `nl -ba package.json | sed -n '1657,1685p'`
  - sha: [926068b14fc6](https://github.com/openclaw/openclaw/commit/926068b14fc661e915a0247e9ecb5949fb95b4f7)
- **Lockfile resolves fetch-blob through the non-deprecated alias:** `fetch-blob@3.2.0` now resolves `node-domexception` to `@nolyfill/domexception@1.0.28` in the lockfile, not to deprecated `node-domexception@1.0.0`.
  - file: [pnpm-lock.yaml:12457](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/pnpm-lock.yaml#L12457)
