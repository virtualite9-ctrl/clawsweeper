## Summary

Keep #22438 open. Current main still has fixed bootstrap filtering plus adjacent `contextInjection` and `lightweight` controls, but it does not implement the requested schema-backed `agents.defaults.bootstrapTier` option or a `minimal | standard | full` tier resolver. The linked implementation PR #22439 remains open and explicitly tracks this issue, and the latest discussion reframes the still-missing configurable policy surface clearly.

## Best Possible Solution

Keep #22438 open until #22439 or a successor lands, or until maintainers explicitly choose fixed allowlists plus per-run `lightContext` as the intended product design. A complete solution should wire `agents.defaults.bootstrapTier` through schema, generated config metadata, docs, bootstrap resolution, hook behavior, subagent/cron defaults, and focused tests for `minimal | standard | full`.
