## Summary

Keep #22438 open. Current main partially reduces the original pain with a broader fixed subagent/cron bootstrap allowlist and per-run lightweight context, but it still does not implement the requested config-driven `agents.defaults.bootstrapTier` surface or `minimal | standard | full` tier policy. The linked implementation PR #22439 remains the relevant successor path in the provided GitHub context, and the latest maintainer/Codex discussion already explicitly kept this issue open with the author confirming the remaining gap.

## Best Possible Solution

Keep #22438 open until maintainers either merge a schema-backed `agents.defaults.bootstrapTier` design, likely through #22439 or a successor, or explicitly decide that the current fixed allowlist plus per-run `lightContext` is the intended product direction. A complete implementation should thread the tier through `src/agents/workspace.ts`, `src/agents/bootstrap-files.ts`, config schema/types/help/labels/generated schema, public config docs, bootstrap hook behavior, and focused bootstrap tests.
