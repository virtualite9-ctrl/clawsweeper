## Summary

Keep open. Current main still stores config backups next to the config file as `openclaw.json.bak`, `openclaw.json.bak.1`, etc. It does not use a dedicated `config-backup/` directory or datetime-based names. The linked PR #20877 directly addressed this request, but the provided GitHub context shows it was closed unmerged, and its referenced commits are not present in local main history.

## Best Possible Solution

Continue tracking this as an open core config enhancement. The likely implementation belongs in `src/config/backup-rotation.ts` plus the config write and recovery paths, with tests updated to assert `config-backup/openclaw.json.bak.<datetime>` behavior and docs updated from the current adjacent `.bak` wording. PR #20877 is useful prior art but was closed unmerged, so it should not be treated as a shipped or canonical superseding fix.
