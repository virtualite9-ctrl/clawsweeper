## Summary

Keep open. Current main still stores OpenClaw config backups beside the config as `openclaw.json.bak` plus numbered `.bak.1` through `.bak.4`; it does not implement the requested `config-backup/` directory or datetime-based naming. The linked PR #20877 is useful prior art, but the provided GitHub context shows it was closed unmerged.

## Best Possible Solution

Keep this open as a core config enhancement. Implement the backup directory and datetime naming in `src/config/backup-rotation.ts`, update `writeConfigFile`, config mutation/include writes, recovery/audit/log paths, tests, and user-facing docs, and use PR #20877 as prior art while fixing its retention-count issue and preserving compatibility with existing adjacent backups.
