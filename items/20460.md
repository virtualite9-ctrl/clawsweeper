## Summary

Keep open. Current main still stores OpenClaw config backups beside the config file as `${configPath}.bak` plus numbered `.bak.1` through `.bak.4`; it does not create a `config-backup/` directory or use datetime-suffixed names. The linked PR #20877 is useful prior art, but the provided GitHub context shows it was closed unmerged, and current code/tests/docs still codify adjacent numbered backups.

## Best Possible Solution

Keep this open as a core config enhancement. Implement the backup directory and datetime naming in `src/config/backup-rotation.ts`, then align `writeConfigFile`, config mutation/include writes, recovery/audit/log paths, tests, and user-facing docs. PR #20877 can be used as prior art, but a final fix should correct its retention-count issue and preserve compatibility with existing adjacent backups.

## Risks / Open Questions

- A future implementation must deliberately migrate or continue recognizing legacy adjacent `.bak` files so last-known-good recovery, doctor repair, and audit paths do not lose usable backups.
