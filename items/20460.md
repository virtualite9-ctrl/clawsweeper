## Summary

Keep #20460 open. Current main still stores automatic config-write backups next to the config file as a numbered `.bak` ring, and the requested dedicated `config-backup/` directory with datetime-based names is not shipped. The directly related PR #20877 was closed unmerged, while #39982 and #40392 remain open implementation candidates rather than current behavior.

## Best Possible Solution

Keep #20460 open as a core config-backup ergonomics request unless maintainers explicitly choose a narrower canonical issue. A complete fix should update `src/config/backup-rotation.ts`, config write/recovery/doctor paths, docs, and tests to define the dedicated backup directory, datetime naming, retention cap, collision handling, and compatibility or migration behavior for existing adjacent numbered backups, coordinating with #20877, #39923, #39982, and #40392.
