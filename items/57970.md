## Summary

Keep open. Current main has not implemented the PR's central fix: missing Mattermost channel type still maps to `channel`, the core `ChatType` union has no `unknown`, and route/dispatch paths still use that value without an unknown fail-closed guard. The PR remains a plausible bug fix, but its latest review context also identifies a remaining blank `channel_type` fallback issue that should be addressed before merge.

## Best Possible Solution

Keep the PR open. The best path is to finish the `unknown` ChatType approach on the Mattermost routing paths, preserve API fallback behavior for missing or blank event `channel_type`, add focused tests for null/undefined/blank plus fallback recovery, and then rebase/merge if checks pass.
