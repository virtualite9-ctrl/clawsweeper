## Summary

Keep open. Current main still has the core behavior this PR is trying to change: `dreaming-narrative-*` session keys include `nowMs`, the subagent idempotency key is the same session key, and plugin `deleteSession` still routes through admin-scoped `sessions.delete`. The PR also should not merge as-is because the provided synthetic-scope approach is bypassed when a request-scoped client already exists, matching the P1 review comments in the PR context.

## Best Possible Solution

Keep the PR open but request changes or supersede it with a narrower maintainer-owned fix: stabilize `memory-core` narrative session keys per workspace and phase, keep `nowMs` only in the idempotency key, and add a narrowly authorized internal cleanup path for `memory-core` `dreaming-narrative-*` sessions that works for both fallback and request-scoped plugin runtime clients. Regression tests should cover the allowed memory-core cleanup path, unrelated session denial, non-memory-core denial, and request-scoped client behavior.
