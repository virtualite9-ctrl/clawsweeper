## Summary

Keep open. Current `main` does not already implement this cleanup: `memory-core` still bakes `nowMs` into the dreaming narrative session key, and gateway subagent session deletion still requires admin unless the caller already has it. The PR also appears to have an unresolved correctness problem for the normal request-scoped plugin path: `dispatchGatewayMethod()` prefers the existing request-scoped `client` over any synthetic scopes, while gateway handlers always install a request scope with `client`. That matches the Codex review comment on the PR and means the proposed `memory-core` cleanup scope would likely not take effect in real gateway-driven runs.
