## Summary

Keep PR #70464 open. Current main still uses timestamped `dreaming-narrative-*` session keys, still uses the same value for `sessionKey` and `idempotencyKey`, and still routes plugin `deleteSession` through admin-scoped `sessions.delete` without a memory-core-specific cleanup allowance. The PR targets real remaining cleanup/leak behavior, but the provided review comments show it likely needs revision rather than automated closure.

## Best Possible Solution

Keep #70464 open or replace it with a cleaner maintainer patch. The best path is to stabilize narrative session keys per workspace and phase, keep `nowMs` only in the idempotency key, add pre-run cleanup, and implement a narrow memory-core-only allowance for deleting its own `dreaming-narrative-*` sessions that works for both fallback and request-scoped gateway clients. The final branch should drop unrelated dependency metadata changes and add regression tests for stable keys, idempotency divergence, memory-core allowed cleanup, wrong-key denial, wrong-plugin denial, and request-scoped client behavior.
