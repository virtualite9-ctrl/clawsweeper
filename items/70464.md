## Summary

Keep open. Current main at 7e13f3f5140168e31027b39a9855ade3fadf24d5 still uses timestamped `dreaming-narrative-*` session keys, still uses the same value for `sessionKey` and `idempotencyKey`, and still routes plugin `deleteSession` through admin-scoped `sessions.delete` without a memory-core-specific cleanup allowance. The PR is a plausible focused fix for the linked cleanup/leak reports, but it needs maintainer review and likely revisions rather than automated closure.

## Best Possible Solution

Keep the PR open or convert it into a cleaner maintainer patch: make narrative session keys stable per workspace and phase, keep `nowMs` only in the subagent idempotency key, add pre-run cleanup, and implement a narrow `memory-core` allowance for deleting only its own `dreaming-narrative-*` sessions that works for both fallback and request-scoped gateway clients. The final PR should drop unrelated dependency metadata changes and add regression tests for stable keys, idempotency divergence, allowed memory-core cleanup, wrong-key denial, wrong-plugin denial, and request-scoped client behavior.
