## Summary

Keep open. Current main still uses timestamped `dreaming-narrative-*` session keys, still uses that same key as the subagent idempotency key, and still routes plugin `deleteSession()` through `sessions.delete` without a memory-core cleanup exception. The PR is not implemented on main, and the related merged PR #67023 only normalized sweep timestamps; the broader cleanup/admin-scope problem remains tracked by the open related issue #68252. The PR itself likely needs revision because review comments identified that its synthetic scope approach would not work when a request-scoped client is already present.

## Best Possible Solution

Keep the PR open for maintainer/contributor follow-up. The best path is a narrow fix that keeps narrative session keys stable per workspace and phase, keeps `nowMs` in the idempotency key, and grants or routes cleanup in a way that works for both fallback and request-scoped plugin clients without broadening unrelated plugin privileges. Add gateway tests for memory-core allowed cleanup, wrong key/plugin blocked, and request-scoped client behavior; remove unrelated runtime dependency edits. Keep #68252 as the canonical remaining bug tracker unless a newer maintainer-owned fix supersedes it.
