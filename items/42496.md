## Summary

Keep this PR open. Current main still lacks the TUI runtime-port discovery path: `resolveGatewayConnection()` builds the Gateway URL from config/env/defaults, while the gateway lock payload does not persist the active listener port. The PR remains a focused implementation candidate for the linked open bug, with useful review follow-up already incorporated, but it still needs maintainer review on the remaining non-Linux stale-lock/PID-reuse concern before merge.

## Best Possible Solution

Review and finish this PR rather than close it. The likely path is to keep the lock-file runtime-port approach, add or justify a cross-platform owner-identity guard for stale locks, keep the regression tests focused on TUI resolving the active custom port, and then land it to close the linked open bug.
