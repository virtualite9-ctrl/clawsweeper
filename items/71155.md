## Summary

Keep #71155 open. Current main still has the reported stability-bundle bug: `DiagnosticStabilityBundle.error` only supports `name` and `code`, and `readSafeErrorMetadata()` still serializes only those fields. The PR remains the canonical active fix for #71071, especially after #71352 was closed as the harder-to-review duplicate, but it should not be merged until the maintainer-requested NUL-regex and missing-test blockers are fixed.

## Best Possible Solution

Keep #71155 open as the canonical fix for #71071. Ask the author to replace the literal NUL regex with an escaped text form such as `/\x00/g`, add focused regression tests for NUL stripping and stack truncation, then run the targeted logging test and changed gate before merge.
