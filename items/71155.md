## Summary

Keep #71155 open. Current main still has the reported stability-bundle bug: error metadata is limited to `name` and `code`, and the startup/restart failure paths pass their thrown errors through that serializer. The PR is the canonical active fix for #71071, but it still has the maintainer-requested blocker: the serializer file is binary in the diff due to a literal NUL regex, and tests still do not cover NUL stripping or stack truncation.

## Best Possible Solution

Keep #71155 open as the canonical fix for #71071. Ask the author to replace the literal NUL regex with an escaped text form such as `/\x00/g`, add regression tests for NUL stripping and stack truncation, then run the targeted logging test and changed gate before merge.
