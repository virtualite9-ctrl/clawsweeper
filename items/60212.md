## Summary

Keep open. Current main does not implement the PR’s empty-sanitized-final repair: it only retries likely interim acknowledgements, empty payloads are still treated as heartbeat/no-delivery, and CLI bundled MCP is still enabled unconditionally for bundleMcp backends. The PR addresses a real remaining gap, but it should not be closed or applied verbatim because the existing PR review raised a concrete CLI session-reuse concern.

## Best Possible Solution

Keep PR #60212 open and update it for current main’s `run-executor` shape. The best fix is a narrowly gated empty-output repair after `resolveCronPayloadOutcome`, rerunning the same cron session once with tools disabled, preserving CLI session reuse or using `extraSystemPromptStatic` so the repair retains original context, disabling bundled MCP for CLI-backed no-tools mode, and adding regression tests for embedded repair success, repair failure as explicit error, and CLI bundled-MCP/session-reuse behavior.
