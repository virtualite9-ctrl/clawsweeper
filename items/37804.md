## Summary

Keep #37804 open. Current main has meaningful partial support for config-injected and forward-compatible models, especially in runtime resolution and `models list`, but it still does not implement the RFC's central invariant: one shared resolved model descriptor with identity, transport, capabilities, provenance, and availability consumed consistently by status, list, runtime, capability gates, and docs.

## Best Possible Solution

Keep #37804 open as the umbrella design tracker until maintainers either land a shared resolved-model descriptor or split/link canonical follow-ups. The likely product/code path is a typed resolver that combines registry, `models.providers`, provider dynamic/forward-compat hooks, transport, capabilities, provenance, and availability, then migrates `models status`, `models list`, runtime execution, picker/auto-reply consumers, thinking gates, and docs to consume that one result.
