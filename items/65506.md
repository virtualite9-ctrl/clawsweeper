## Summary

Current main still only partially hardens the QA harness. Normal-path gateway cleanup exists, and qa-channel readiness probing has been added, but the host suite path still lacks explicit interruption ownership for detached gateway children, fixed sleeps remain in restart/readiness paths, and qa-channel config is still emitted before the QA transport is confirmed healthy. The related implementation PR #65511 was closed unmerged, so there is no shipped fix to close against.

## Best Possible Solution

Keep this issue open and land a focused QA harness fix in qa-lab: add explicit SIGINT/SIGTERM ownership for host suite gateway children and temp roots, add a safe stale QA-suite-root reap strategy where feasible, replace restart sleeps with concrete gateway/transport readiness handshakes, and gate qa-channel config emission or activation on a healthy QA bus. Add regression coverage for interrupted suite cleanup, temp-root removal, restart readiness, and config generation when the QA transport is unavailable.
