## Summary

Keep this issue open. Current main still only partially addresses the report: ordinary finally-path cleanup exists, but the host QA suite command does not own SIGINT/SIGTERM cleanup for detached gateway children, fixed restart sleeps remain, and qa-channel config is generated before the QA transport readiness check. The linked implementation PR #65511 was closed unmerged, so there is no shipped fix to close against.

## Best Possible Solution

Keep this issue open and land a focused qa-lab fix: add signal-aware host suite ownership for gateway children and temp roots, add safe stale QA-suite-root reaping where ownership can be proven, replace restart sleeps with concrete gateway/config/transport readiness handshakes, gate qa-channel activation or config emission on a healthy QA bus, and add regression coverage for interrupted cleanup, temp-root removal, restart readiness, and unavailable QA transport config generation.
