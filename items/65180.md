## Summary

Keep this PR open. Current main still has the two central gaps the PR targets: local `infer model run` does not pass an explicit per-run session id/key, so `agentCommand` resolves the agent's main session path, and transcript fallback resolution still builds the fallback from the default agent transcript root instead of the `sessionsDir` derived from the active store path. The PR is a focused draft implementation candidate, not obsolete cleanup.

## Best Possible Solution

Keep this PR open as a focused draft implementation candidate. The best path is to review and refine the code so local one-shot `infer model run` uses an explicit ephemeral session id/key or equivalent stateless mode, and transcript fallback resolves fresh files under the profile-local `sessionsDir` derived from `storePath`; keep focused regression tests for both behaviors and decide separately whether the canary baseline docs should be removed or rewritten.
