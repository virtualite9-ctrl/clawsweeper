## Summary

Keep this PR open. Current main has not absorbed the two central behavior changes: local `infer model run` still calls `agentCommand` without an explicit per-run `sessionId`/`sessionKey`, and transcript fallback still builds the fallback path from the agent default transcript root rather than the profile-local sessions directory derived from `storePath`. The PR remains a focused draft implementation candidate, not obsolete cleanup.

## Best Possible Solution

Keep this PR open as a focused draft implementation candidate. The best path is to review or refine the code so local one-shot `model run` uses an explicit per-run session id/key or equivalent stateless session mode, and transcript fallback prefers the profile-local `sessionsDir` derived from `storePath`; keep focused regression tests for both behaviors and decide separately whether the canary baseline docs belong in the repository.
