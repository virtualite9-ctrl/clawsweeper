## Summary

Keep this issue open. The provided GitHub context shows open PR #43984 still explicitly links itself as the fix for this issue and remains unmerged, so the cleanup rule requires leaving the issue open until that PR is reviewed, merged, closed, or superseded. Current main also still has the reported failure path: the chat dropdown is built from `sessions.list` rows plus only the current session key, while `sessions.list` can filter a stale main session out when `activeMinutes` is set and heartbeat remains active.

## Best Possible Solution

Review and land PR #43984 or close/supersede it with an equivalent narrow fix. The shipped behavior should guarantee the resolved main session remains selectable when `sessions.list` with `activeMinutes` would otherwise return only heartbeat, with regression coverage for stale main plus active heartbeat.
