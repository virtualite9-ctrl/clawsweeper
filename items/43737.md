## Summary

Keep this issue open. Current main has partial mitigations, but the dropdown builder still renders only `sessions.list` rows plus the current session, and the gateway still applies `activeMinutes` as a blanket freshness filter. PR #43984 is the open implementation candidate and should be reviewed, landed, or explicitly superseded before closing this issue.

## Best Possible Solution

Keep this issue open until maintainers land or explicitly supersede PR #43984, or ship an equivalent narrow fix. The safest product path is to ensure the resolved main session key is always selectable in the chat dropdown and add regression coverage for a `sessions.list` result containing only heartbeat while the canonical main session exists.
