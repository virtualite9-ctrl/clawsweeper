## Summary

Current main still does not implement this PR's central behavior. Gateway startup loads internal hooks once from the default workspace, while current docs still describe workspace hooks as per-agent. The PR is a plausible fix candidate, but it is stale against the current startup layout and needs a careful rebase because hook policy/security rules have tightened since its branch point.

## Best Possible Solution

Keep this PR open, but require a rebase or replacement patch against current main. The best implementation belongs in the current gateway startup path, likely `src/gateway/server-startup-post-attach.ts`, using `listAgentWorkspaceDirs` for startup-time workspace enumeration while preserving `hasConfiguredInternalHooks`, configured-name filtering, explicit workspace-hook opt-in, boundary checks, plugin/managed/extra-dir precedence, and the current no-override policy for workspace hooks. Add focused regression tests for an enabled hook in a non-default agent workspace, event/session workspace isolation, startup behavior, failure isolation, and precedence/security policy before review and landing.
