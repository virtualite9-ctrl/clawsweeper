## Summary

Keep this PR open. Current main still loads internal hooks once from the default workspace during Gateway startup, while docs still describe workspace hooks as per-agent. The PR is stale and needs rework before merge, but it targets a real remaining gap rather than an already-implemented or clearly superseded change.

## Best Possible Solution

Keep this PR open, but require a rebase and scoped rewrite against current main. The best path is to add a startup-specific multi-workspace loading path in the current `src/gateway/server-startup-post-attach.ts` flow, using `listAgentWorkspaceDirs`, while preserving current hook policy: workspace hooks remain explicit opt-in, cannot override shared/plugin/bundled hooks, and must keep existing boundary-file protections and trusted-code warnings. Add focused regression coverage for loading an enabled hook from a non-default agent workspace, isolation between workspaces, failure isolation, and current precedence behavior. Then review/land the refreshed PR or replace it with a smaller maintainer-owned patch.
