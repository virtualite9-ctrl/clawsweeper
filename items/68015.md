## Summary

Keep this issue open. Current main partially covers the report with documented exec host values, task/status summaries, and MCP/process cleanup paths, but it does not fully implement the central requests: `tools.exec.host="host"` still has no nearest-value hint, runtime fanout observability is split across scoped surfaces rather than unified, and the model-free local maintenance-runner/systemd-timer pattern is not documented.

## Best Possible Solution

Keep this issue open for maintainer triage and split it into focused work: add a small invalid-enum suggestion for `tools.exec.host` if maintainers want that UX; design a unified operator status surface that joins process sessions, MCP runtime state, tasks/subagents/cron records, and runtime load; document the deterministic local maintenance-runner pattern using external scripts or systemd user timers for work that does not need model turns; treat inline dashboard log review as a separate product decision.
