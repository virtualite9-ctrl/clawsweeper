## Summary

Keep this issue open. Current main partially addresses the field report with clearer exec-host docs, task/status surfaces, and MCP cleanup, but the main requests are not fully implemented: `tools.exec.host` still has no nearest-value hint for `host`, runtime fanout observability remains fragmented, and the model-free local maintenance-runner pattern is not documented.

## Best Possible Solution

Keep this issue open for maintainer triage and split it into focused work: add a small invalid-enum hint for `tools.exec.host` if maintainers want that UX; design a unified operator status surface that joins process sessions, MCP runtime state, task/subagent/cron records, and runtime load; document the deterministic local maintenance-runner pattern using external scripts/systemd user timers where work does not need model turns; handle inline dashboard log review separately if maintainers want that product surface.
