## Summary

PR #72167 is a duplicate fix attempt for the same remaining `sessions cleanup --fix-missing` stale-`sessionFile` bug already tracked by earlier open PR #69954 and issues #50248/#63897. Current `main` is still unfixed, so the underlying bug should stay open, but this newer PR should be closed in favor of converging the fix on the canonical earlier PR or an equivalent maintainer branch.

## Best Possible Solution

Close PR #72167 as a duplicate/superseded fix attempt and consolidate the work on earlier open PR #69954 or a maintainer-owned replacement. The final fix should make cleanup use safe transcript-candidate semantics, preserve valid sessions when `<sessionId>.jsonl` exists, handle malformed legacy session IDs without aborting cleanup, and keep focused regression tests for stale stored `sessionFile`, both-missing, and valid stored-path cases.
