## Summary

Current main does not fully solve the reported Dockge/Docker Compose first-run problem. The image default includes `--allow-unconfigured`, but the checked-in Compose service overrides the image command and still starts `gateway --bind ... --port ...` without `--allow-unconfigured`, so a pasted Compose YAML plus `.env` can still hit the missing `gateway.mode=local` guard unless pre-start setup/config commands have already run.

## Best Possible Solution

Keep this open and design a safe first-run Docker Compose/Dockge path. A good fix would provide a pasteable Compose/profile or adjust the Compose command/setup contract so a fresh `.env` with explicit auth can start without manual `openclaw setup`/`config set`, while retaining `gateway.mode` protection for existing damaged configs and auth requirements for `lan`/`auto` binds.
