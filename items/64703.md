## Summary

Current main still parses unquoted scientific notation through the fallback string path, so this PR is not obsolete and does not fit any conservative close reason. The PR remains a focused implementation candidate: it changes the numeric regex in `parseConfigValue` and adds direct unit coverage for exponent notation. Earlier bot review comments about a malformed heartbeat test appear stale because the final PR diff no longer includes that file.

## Best Possible Solution

Keep this PR open for normal maintainer review. The likely best path is to land a focused `parseConfigValue` change with colocated unit coverage for exponent notation and existing booleans/null/plain numbers/JSON/quoted strings, or explicitly document/reject exponent coercion if maintainers want unquoted `1e5` preserved as a string.
