## Summary

Current `main` already satisfies this request through the generic agent `message` tool: the tool schema exposes a `media` parameter, the Feishu plugin advertises `send` on the message tool and routes `media` sends through `feishuOutbound.sendMedia`, and the shipped changelog for `v2026.4.23` explicitly says Feishu `message(..., media=...)` was fixed to use the outbound media path.

## Evidence

- **Release note: shipped in v2026.4.23:** `CHANGELOG.md` records, under the `2026.4.23` release section, that the message tool now routes Feishu `message(..., media=...)` sends through the outbound media path so Feishu file/image attachments actually send.

## Risks / Open Questions

- I did not audit every non-Feishu channel for equivalent media parity; this close is based on the Feishu request and the generic cross-channel message-tool path already present on `main`.
- The implementation uses the existing `message` tool with `action: "send"` plus `media`, not a separately named `send_media` tool, so callers expecting that exact tool name would still need to use the generic message-tool interface.
