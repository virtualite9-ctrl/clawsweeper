## Summary

Keep #69102 open. Current main does not expose Gemini Flex service-tier configuration for the bundled Google provider. The direct Gemini transport builds request bodies from cachedContent, generationConfig, systemInstruction, tools, and toolConfig only, while existing service_tier support is limited to OpenAI/Codex and Anthropic paths.

## Best Possible Solution

Keep the issue open and implement a narrow Google-provider change in `extensions/google`: accept `agents.defaults.models["google/<model>"].params.serviceTier` and `service_tier` for direct `api: "google-generative-ai"` models, emit top-level `service_tier: "flex"`, preserve explicit caller payload values, and add focused transport tests plus Google provider docs. Timeout guidance may also be needed because Flex can queue longer than standard requests.
