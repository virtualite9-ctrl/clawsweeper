## Summary

Keep #69102 open. Current main does not expose or forward Gemini Flex `service_tier` for the bundled Google/Gemini provider. The direct Gemini request builder only serializes the existing Gemini fields, and the generic extra-params wrapper only maps a small allowlist such as temperature, maxTokens, transport, openaiWsWarmup, and cachedContent. Existing service-tier support is provider-specific for OpenAI and Anthropic, not Google.

## Best Possible Solution

Implement this in the bundled Google provider rather than closing: accept `serviceTier` and `service_tier` from model params for direct `google-generative-ai` models, validate at least `flex`, inject REST `service_tier: "flex"` at the Gemini request root, preserve caller/root payload values if already present, and add focused coverage in `extensions/google/transport-stream.test.ts` plus Google provider docs under advanced configuration. Consider documenting timeout expectations separately because Flex can wait much longer than standard Gemini traffic.
