## Summary

Keep #69102 open. Current main has a native Google/Gemini transport, but it does not forward `serviceTier` or `service_tier` from model params into Gemini requests. The existing service-tier support is documented and implemented for OpenAI/Anthropic/Codex paths, while Google’s direct Gemini advanced config currently covers `cachedContent` and thinking behavior only.

## Best Possible Solution

Implement this in the bundled Google provider: accept `agents.defaults.models["google/<model>"].params.serviceTier` and/or `service_tier`, normalize to the Gemini transport’s required request field for direct `api: "google-generative-ai"` calls, add transport tests proving the payload contains the Flex tier, and document the Google provider config under `docs/providers/google.md`.
