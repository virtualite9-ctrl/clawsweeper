## Summary

Current main already handles inbound Signal reactions as dedicated reaction notifications instead of falling through unsupported-media handling. The Signal monitor recognizes both `envelope.reactionMessage` and `dataMessage.reaction`, skips the attachment fallback for reaction-only payloads, and enqueues a reaction system event. The same logic, docs, and regression tests are present in release `v2026.4.22`.
