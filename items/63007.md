## Summary

Keep this PR open. Current main still lacks the targeted gateway/message_sending behavior: outbound gateway send builds session identity, but message_sending does not receive agentId/session identity; hook cancellation is still represented as no delivery result; and gateway send still reports the generic No delivery result. The PR is also paired with the same-author open umbrella #63011, so it remains an implementation candidate rather than cleanup-close material.

## Best Possible Solution

Keep this PR open for maintainer review, or replace it with an equivalent focused patch reviewed alongside #63011. The best implementation should add optional agentId to the public message hook context, pass outbound agentId/sessionKey into message_sending, preserve cancellation reason through a narrow callback or blocked-result shape, surface explicit guarded gateway-send denial, update docs/types if the public contract changes, and cover the gateway/outbound behavior with tests.
