## Summary

Keep this PR open. Current main still has the targeted gateway/message_sending gap: outbound gateway send builds session identity, but message_sending does not receive agentId/session identity at the hook call site, hook cancellation is represented as an empty delivery result, and gateway send still reports that as the generic No delivery result. The same-author umbrella #63011 remains open, so this draft PR should be reviewed, rebased, or replaced rather than closed.

## Best Possible Solution

Keep this PR open for maintainer review, or replace it with an equivalent focused patch reviewed alongside #63011. The fix should add optional agentId to the public message hook context, pass outbound agentId/sessionKey into message_sending, preserve cancellation reason through a narrow callback or blocked-result shape, surface an explicit guarded gateway-send denial, update hook docs/types, and cover the gateway/outbound behavior with tests.
