## Summary

Keep this PR open. Current main has gateway-send session derivation and mirroring, but the two central fixes are still missing: `message_sending` does not receive outbound agent/session identity, and guarded cancellation still collapses to the generic `No delivery result` failure. The PR discussion includes useful Telegram live verification, and related #32621 is closed as stale rather than serving as a shipped substitute for this narrower gateway-send behavior.

## Best Possible Solution

Keep the PR open or replace it with an equivalent focused maintainer patch: add `agentId` to the public message hook context, pass outbound `session.agentId` and `session.key` into `message_sending`, expose cancellation reason through a narrow delivery callback/result, surface an explicit guarded gateway-send denial, and cover it with gateway/outbound regression tests plus a small plugin-hook docs update.
