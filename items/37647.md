## Summary

Keep open. Current main still has explicit logic and regression tests that suppress commentary-only assistant text in shared extraction and `chat.history`, and the TUI history/render path depends on those helpers. The original empty-session-key theory is weaker on current main because agent events now backfill blank `sessionKey` values from run context, but there is no strong evidence that the user-visible TUI-missing-reply bug is fully fixed on `main`.
