## Summary

Keep #72139 open. A prior token-mismatch restart loop was partially addressed by the bounded device-token retry path, but current main still intentionally reconnects on `AUTH_TOKEN_MISMATCH` when no cached device-token retry was attempted, and `chat.history` still performs synchronous full transcript file reads and JSON parsing on the gateway event loop before applying response-size caps. The issue's Slack Socket Mode disconnect report is therefore still plausible and actionable rather than implemented or purely duplicate.

## Best Possible Solution

Keep the issue open and fix both halves narrowly: make Control UI pause reconnects and surface an auth prompt on terminal shared-token auth failures when no trusted device-token retry can succeed, while preserving the one trusted device-token retry path; then make `chat.history` non-starving by avoiding full synchronous transcript reads/projection on the gateway event loop, for example with bounded tail reads, async/yielding parsing, caching, or a worker-backed projection path. Add regression coverage for no-cached-token auth mismatch reconnect behavior and for large transcript `chat.history` not starving another socket/heartbeat-style task.
