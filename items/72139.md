## Summary

Keep #72139 open. Current main has bounded `chat.history` payloads and auth error UI hints, but it still intentionally reconnects on `AUTH_TOKEN_MISMATCH` when no cached device-token retry occurred, and `chat.history` still performs synchronous full transcript read/parse work before limiting the response.

## Best Possible Solution

Keep the issue open. The best fix is in core Control UI/Gateway behavior: make `AUTH_TOKEN_MISMATCH` terminal after the single trusted device-token retry, and terminal immediately when that retry cannot be attempted; surface a clear token re-entry action. Separately, make `chat.history` non-starving by avoiding full synchronous transcript reads/parses on the event loop, using bounded tail reads or async/yielded projection, and add regression coverage for token-mismatch no-retry behavior plus large-history responsiveness while Socket Mode timers remain healthy.
