## Summary

Current main still has the dispatcher-forwarding Slack media behavior that this PR proposes to change, and the active linked Slack attachment bug remains open. The PR is a focused implementation candidate, but it needs maintainer review because the proposed dispatcher stripping trades against the currently documented/released intent to preserve the SSRF pinned dispatcher path.

## Best Possible Solution

Keep this PR open for maintainer review. The concrete path is to rebase it onto current main, decide whether Slack media should strip the upstream dispatcher or preserve the pinned dispatcher path, update the regression test to lock that contract, and then either land the fix for #63905 or close it with a source-backed maintainer decision.
