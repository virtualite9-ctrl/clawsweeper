## Summary

Current main has not implemented this PR's dispatcher-stripping behavior. Slack media still forwards the SSRF guard dispatcher into the runtime fetch path, and the current regression test explicitly asserts that forwarding. The related open Slack attachment bug and overlapping closed PRs make this an active maintainer-review candidate, not conservative cleanup.

## Best Possible Solution

Keep this PR open for maintainer review. The concrete path is to rebase it against current main, validate the Slack media SSRF/runtime-fetch contract against #63905, decide whether dispatcher stripping or the current runtime dispatcher path is the intended boundary, and land or close it with that source-backed decision.
