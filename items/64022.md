## Summary

Keep open. Current `main` still forwards the SSRF guard's `dispatcher` through Slack media fetch, and the colocated test still expects that behavior. This PR is therefore not implemented on `main`, and the provided discussion says #64766 is complementary rather than a replacement, so there is not strong evidence for a cleanup close.

## Evidence

- **Slack media fetch still forwards dispatcher-backed init:** `createSlackMediaFetch()` still switches to `fetchWithRuntimeDispatcher` when `init.dispatcher` is present, then forwards `...init` unchanged into the downstream fetch call. That is the exact behavior this PR proposes to change.
