## Summary

Keep this PR open. Current main still advertises `hello-ok.auth` as optional in the TypeBox schema and generated Swift models, while the runtime handler now always emits `auth`; this PR remains a focused implementation candidate for same-author open bug #68160. The related PR #68227 is closed unmerged and does not supersede this work.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land one complete implementation that makes `hello-ok.auth` required in the TypeBox schema, regenerates both Swift protocol models, updates Swift consumers/tests, confirms docs and focused protocol validation are aligned, and then closes #68160 after merge; maintainers can borrow any useful docs or test coverage from the closed unmerged #68227 if desired.
