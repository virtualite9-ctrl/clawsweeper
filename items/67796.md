## Summary

Keep this PR open. Current main still has the Mattermost message-tool send path resolving attachments only from `params.media`, while the shared message tool schema and other channel handlers support `filePath` and `path`. The PR is a focused implementation candidate, but its current fallback short-circuits on blank `media` and lacks regression tests for the new aliases.

## Best Possible Solution

Keep this PR open as a plausible Mattermost message-tool bug fix. The best path is to revise the fallback to normalize each candidate independently (`media`, then `filePath`, then `path` or the maintainer-chosen precedence), add focused Mattermost handleAction regression tests for both aliases and blank-media fallthrough, and coordinate the remaining #67784 work with #52120 and #58439 rather than closing this implementation candidate prematurely.
