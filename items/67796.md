## Summary

Keep this PR open. Current main still leaves Mattermost's channel-owned send handler resolving attachments only from `params.media`, while the public message tool schema exposes `path` and `filePath` and comparable channel handlers already consume those aliases. The PR is a focused implementation candidate, but as written it still has a whitespace `media` short-circuit and no Mattermost handleAction regression tests for the aliases.

## Best Possible Solution

Keep this PR open as a plausible Mattermost message-tool bug fix. The best maintainer path is to revise the fallback to normalize each candidate independently, add Mattermost handleAction tests for `filePath`, `path`, and blank `media` fallthrough, and either coordinate the remaining #67784 work with #52120/#58439 or replace these small PRs with one complete Mattermost send-path fix.

- This PR addresses only the alias portion of #67784; thread-context and mediaLocalRoots behavior remains separate and should be coordinated with #52120 and #58439 or an integrated replacement.
