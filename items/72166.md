## Summary

Keep #72166 open. Current main at df542f75a9c186001a3fbfbac9310791314893c1 still does not include `cron.enabled` or `agents.defaults.compaction.mode` in the agent-facing gateway config mutation allowlist. Both are real documented config fields, and related PR #72215 is a focused open implementation candidate rather than a shipped fix.

## Best Possible Solution

Keep #72166 open as the canonical request until #72215 or an equivalent scoped change lands. If maintainers accept the trust-boundary expansion, add exactly `cron.enabled` and `agents.defaults.compaction.mode` to `ALLOWED_GATEWAY_CONFIG_PATHS` with positive guard tests. If rejected, document that direct operator-admin `config.patch` or CLI config writes are the supported programmatic path while the agent-facing gateway tool stays narrow.
