## Summary

Keep #72166 open. Current main still does not add `cron.enabled` or `agents.defaults.compaction.mode` to the agent-facing `ALLOWED_GATEWAY_CONFIG_PATHS` guard. The paths are valid documented config fields, and direct operator-admin `config.patch` is a separate API surface, but the specific requested allowlist change remains unimplemented and needs maintainer/security judgment rather than auto-close.

## Best Possible Solution

Keep the issue open and clarify the intended surface. If maintainers want these fields to be agent-tunable, add exactly `cron.enabled` and `agents.defaults.compaction.mode` to `ALLOWED_GATEWAY_CONFIG_PATHS`, add focused allow/block regression tests, and update the security/config docs if the trust boundary changes. If the intended use is external automation rather than model-driven edits, document that direct admin `config.patch` or `openclaw config set` is the supported programmatic path while the agent-facing gateway tool stays narrow.
