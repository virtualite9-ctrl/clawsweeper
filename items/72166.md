## Summary

Keep #72166 open. Current main still lacks `cron.enabled` and `agents.defaults.compaction.mode` in the agent-facing gateway config mutation allowlist; #72215 is an open implementation candidate, but the issue should remain the canonical request until a maintainer accepts or rejects that trust-boundary change.

## Best Possible Solution

Keep the issue open as the canonical request. If maintainers want these fields to be agent-tunable, merge #72215 or an equivalent scoped patch that adds exactly `cron.enabled` and `agents.defaults.compaction.mode` to `ALLOWED_GATEWAY_CONFIG_PATHS` with positive guard coverage. If maintainers consider either path too sensitive for model-driven config edits, document that direct operator-admin `config.patch` or `openclaw config set` is the supported programmatic path while the agent-facing gateway tool stays narrow.
