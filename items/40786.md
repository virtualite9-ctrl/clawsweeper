## Summary

Keep open. Current `main` does not implement `.gitignore`-style backup excludes or exclude-file support for `openclaw backup create`; the only scope controls present are `--no-include-workspace` and `--only-config`.

## Evidence

- **CLI surface lacks exclude options:** `backup create` currently registers `--output`, `--json`, `--dry-run`, `--verify`, `--only-config`, and `--no-include-workspace`. There is no `--exclude`, `--exclude-file`, `--smart-exclude`, or similar flag.
  - file: [src/cli/program/register.backup.ts:21](https://github.com/openclaw/openclaw/blob/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503/src/cli/program/register.backup.ts#L21)
  - command: `nl -ba src/cli/program/register.backup.ts | sed -n '1,120p'`
  - sha: [b0d2003fe65b](https://github.com/openclaw/openclaw/commit/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503)
- **Backup plan includes full source roots, not filtered subpaths:** The planner builds candidates from the state dir, config path, external credentials dir, and workspace dirs, then only deduplicates or marks covered paths. There is no exclude-pattern evaluation step.
  - file: [src/commands/backup-shared.ts:146](https://github.com/openclaw/openclaw/blob/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503/src/commands/backup-shared.ts#L146)

- **Archive writer tars every planned asset directly:** Archive creation passes `[manifestPath, ...result.assets.map((asset) => asset.sourcePath)]` straight into `tar.c(...)`, with no file-level filtering hook for ignore patterns.
  - file: [src/infra/backup-create.ts:353](https://github.com/openclaw/openclaw/blob/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503/src/infra/backup-create.ts#L353)
