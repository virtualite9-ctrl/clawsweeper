## Summary

Keep open. Current `main` has parent-config fallback for Discord threads, but not for ordinary guild text channels under a Discord category, so the requested category-level allowlist/config behavior is not implemented end-to-end.

## Evidence

- **Inbound message preflight only resolves a parent for thread channels:** `message-handler.preflight.ts` computes `earlyThreadParentId` only when `resolveDiscordThreadChannel(...)` succeeds, i.e. for threads, then passes only that thread parent into channel-config resolution.
  - file: [extensions/discord/src/monitor/message-handler.preflight.ts:591](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/extensions/discord/src/monitor/message-handler.preflight.ts#L591)
  - command: `nl -ba extensions/discord/src/monitor/message-handler.preflight.ts | sed -n '591,622p' && nl -ba extensions/discord/src/monitor/message-handler.preflight.ts | sed -n '770,785p'`
  - sha: [6bc0dc8fb671](https://github.com/openclaw/openclaw/commit/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440)
- **Thread-parent fallback is what channel matching currently tests:** Discord monitor tests cover inheriting parent config for thread channels, not category-parent fallback for normal channels.
  - file: [extensions/discord/src/monitor.test.ts:367](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/extensions/discord/src/monitor.test.ts#L367)

  - sha: [6bc0dc8fb671](https://github.com/openclaw/openclaw/commit/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440)
- **Docs describe thread inheritance, not category-level guild channel config:** Discord docs state that threads inherit parent channel config unless overridden, while the allowlist docs still say that when a guild has a `channels` map, non-listed channels are denied.
  - file: [docs/channels/discord.md:413](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/docs/channels/discord.md#L413)
  - command: `nl -ba docs/channels/discord.md | sed -n '413,421p' && nl -ba docs/channels/discord.md | sed -n '576,585p'`

That inconsistency is evidence the requested behavior is not implemented for the main inbound message path.

## Risks / Open Questions

- Closing as implemented would be inaccurate because the requested behavior affects ordinary Discord guild channels, and current main only has documented/tested parent inheritance for threads.
- There is no evidence in current docs, tests, or the main inbound message path that `channels.discord.guilds.<id>.channels` supports category-level entries that automatically apply to newly created child channels.
