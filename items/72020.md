## Summary

Keep #72020 open. Current main still lacks Signal inbound status-reaction lifecycle wiring, while the linked fix PR #72025 is open and unmerged. The report is concrete, recent, and reproducible by code inspection, so it is not cleanup-ready as implemented, duplicate, stale, or external-scope work.

## Best Possible Solution

Keep #72020 open until #72025 or an equivalent Signal-owned fix lands on main. The fix should create a Signal status reaction controller for eligible direct and group inbound messages, route dispatch lifecycle callbacks through dispatchInboundMessage, use sendReactionSignal/removeReactionSignal as the adapter surface, and add focused Signal monitor regression coverage for direct and group routing.
