## Summary

Keep #69010 open. Current main has adjacent turn-source routing guards, but it still does not implement this PR's gateway-agent deliveryContext precedence change: the agent handler keeps stored session deliveryContext as primary over the current request hint, so stale persisted routes can remain authoritative in the session state.

## Best Possible Solution

Keep the PR open and ask maintainers to rebase or port the targeted gateway-agent merge logic onto current main. The desired path is a narrow fix in src/gateway/server-methods/agent.ts with regression coverage for #68378: a concrete current request route should win over conflicting stale persisted deliveryContext, channel-only and same-channel partial updates should preserve the intended route semantics, and canonical same-channel targets should not drop thread context.
