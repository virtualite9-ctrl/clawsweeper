## Summary

Keep this PR open. Current main still lacks the retained run-trace timeline store and `agent.timeline` gateway RPC that form the central feature. Some sub-fixes are already present on main, but the PR remains useful and is also too broad and security-sensitive to land as-is. The open #68204 issue is the right canonical design target, not enough by itself to auto-close this PR.

## Best Possible Solution

Keep this PR open for explicit maintainer handling, but do not merge it as-is. Split out or drop the already-shipped sub-fixes, then replace the remaining trace work with a focused design aligned with #68204: a canonical run trace schema, safe gateway/API exposure, ownership checks, redaction, bounded retention/span caps, and targeted tests. Unrelated sandbox, cron, plan-search, simulation, roadmap, generated-client, and timing changes should be separate PRs or removed.

## Risks / Open Questions

- Do not land this branch as-is: it mixes the trace feature with cron critic-loop work, procedural memory, sandbox execution/template code, simulation lanes, plan search, generated Swift protocol files, docs, and unrelated test timing changes.
- The proposed `agent.timeline` surface needs explicit object-level authorization, redaction rules, bounded retention, and span caps before exposing run traces through the gateway.
- The sandbox execution-policy additions have unresolved review concerns around dynamic Python import bypasses and parser edge cases in a code-execution boundary.
- The critic-loop additions have an unresolved review concern where `needs_replan` can hide real run errors as successful outcomes.
