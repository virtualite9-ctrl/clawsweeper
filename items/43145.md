## Summary

Keep this PR open. Current main has the block-reply and fenced-chunking sub-fixes, and it has trajectory export diagnostics, but it does not implement the PR's central `agent.timeline` RPC or retained `agent-run-trace` timeline store. The PR is not obsolete enough for an auto-close: the remaining run-timeline idea is still real, but the branch is too broad and has unresolved security-sensitive review findings that require maintainer review or a focused replacement.

## Best Possible Solution

Keep the PR open for maintainer handling, but do not land it as-is. The useful path is to split the already-shipped sub-fixes away, then either replace the remaining trace work with a focused PR aligned to #68204 or redesign `agent.timeline` with ownership checks, redaction, bounded retention, span caps, and targeted tests. Unrelated sandbox, plan-search, cron, and roadmap work should be separate PRs or dropped from this branch.
