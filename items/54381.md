## Summary

Current main already fixes the reported midnight rollover behavior. Subsystem logging no longer keeps a forever-cached file child logger, and the root file transport re-resolves the active dated rolling log path on each write. Regression coverage now verifies that a long-lived subsystem logger writes post-midnight entries to the new day's file.

## Best Possible Solution

Close this issue as implemented on main. Keep the current design: subsystem loggers stay lightweight and the shared file transport follows the active rolling date file at write time, with the existing regression tests guarding long-lived gateway processes across midnight.
