## Close Comment

Thanks for the proposal. This was a Codex review against current `main`, and the feature looks better scoped as a ClawHub skill/plugin or other external integration rather than an OpenClaw core issue.

The current project direction is explicit here: `VISION.md:54` says optional capability should usually ship as plugins, `VISION.md:68` says new skills should go to ClawHub first, and `VISION.md:101` lists new core skills that can live on ClawHub under 'What We Will Not Merge (For Now)'. The existing docs also already expose the extension/building-block path: `docs/tools/plugin.md:11` documents plugins/skills as the way to add capabilities, `docs/cli/infer.md:13` explicitly encourages building skills on top of `openclaw infer`, and `docs/gateway/index.md:88` documents `/v1/embeddings` for RAG-style pipelines.

Given that scope, I’m closing this here as a ClawHub-scope request rather than a core maintainer task. If someone wants to pursue it, a ClawHub plugin/skill or docs-side integration would be the right path.
