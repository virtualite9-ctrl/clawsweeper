## Summary

Keep open. Current main does route `driver: "existing-session"` through Chrome DevTools MCP `--autoConnect`, but the current codebase still explicitly expects and surfaces `DevToolsActivePort` attach failures for this path. I did not find strong evidence on `main` that the reported Chrome 145+/146 behavior regression has been fixed well enough to close conservatively.
