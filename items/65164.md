## Summary

Keep open. The issue is recent, concrete, and still describes a plausible unresolved current-main bug: native Windows Gateway remains supported, the gateway-wide network bootstrap still only handles env-proxy dispatcher setup, the requested model-prewarm skip switch is absent, and current mitigations are narrower than a process-wide Windows HTTP failure affecting Telegram REST, Discord REST, model/catalog HTTP, and local approval handshakes.

## Best Possible Solution

Keep this open for a targeted maintainer fix. Reproduce on native Windows 10 with Node 22 and multiple Discord plus Telegram providers, instrument gateway-process fetch/undici dispatcher state and libuv handles, then add a bounded prewarm workaround such as OPENCLAW_SKIP_MODEL_PREWARM or a prewarm timeout while separately fixing the gateway-wide HTTP failure. Regression coverage should prove channel startup is not blocked indefinitely and that Telegram REST, Discord REST, model/catalog HTTP, and local approval handshakes remain healthy in the same gateway process.
