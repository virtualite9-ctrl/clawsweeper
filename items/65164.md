## Summary

Keep open. Current main still does not show a source-backed fix for the reported native Windows gateway-process HTTP failure or the requested model-prewarm escape hatch: Windows Gateway remains in scope, gateway network bootstrap is still only env-proxy setup, configured primary-model prewarm still runs before channels start, and no OPENCLAW_SKIP_MODEL_PREWARM-style switch exists.

## Best Possible Solution

Keep this issue open for a targeted maintainer fix. Reproduce on native Windows 10 with Node 22 and multiple Discord plus Telegram providers, instrument gateway-process fetch/undici dispatcher state and libuv handles, add a bounded workaround such as OPENCLAW_SKIP_MODEL_PREWARM or a prewarm timeout, and separately fix the process-wide HTTP failure with regression coverage for Telegram REST, Discord REST, model/catalog HTTP, and local approval handshakes in the same gateway process.
