## Summary

Current main still treats billing failures as profile-wide disabled state, so sibling models behind the same LiteLLM auth profile remain blocked. The related billing message fix is present, but it does not change cooldown scope or allow disabling billing backoff.

## Best Possible Solution

Keep this open and implement a deliberate model-scoped or proxy-aware billing cooldown policy for OpenAI-compatible aggregators such as LiteLLM, or add a documented supported way to disable billing backoff for user-managed proxy failover. The likely code path is auth-profile usage state plus model fallback, with coverage for one LiteLLM profile serving two sibling models where billing on one model does not block the other, and config/docs updates if the public contract changes.
