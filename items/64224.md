## Summary

Keep open. Current `main` still implements the opposite of the request: billing failures are stored as profile-wide disables, model-scoped bypass only exists for `rate_limit`, and config still requires `billingBackoffHours` to be a positive number so a true `0` disable is not supported.

## Evidence

- **Billing failures still enter the disabled lane:** `computeNextProfileUsageStats()` treats `billing` as a `disabledFailureReason` and writes `disabledUntil` / `disabledReason`, using the 5h->24h billing backoff config.
  - file: [src/agents/auth-profiles/usage.ts:559](https://github.com/openclaw/openclaw/blob/5b8bd6371c666a5de43c82305e0315771e8e49d5/src/agents/auth-profiles/usage.ts#L559)
  - command: `nl -ba src/agents/auth-profiles/usage.ts | sed -n '360,620p'`
