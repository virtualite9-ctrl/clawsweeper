## Summary

Current `main` already initializes the global Undici env-proxy dispatcher for the CLI before any routed `openclaw message ...` command reaches `runMessageAction()`, so this PR's proposed extra bootstrap is redundant for the bug it describes.

## Evidence

- **CLI startup already bootstraps proxy handling:** `runCli()` normalizes env, starts debug proxy capture, then calls `ensureGlobalUndiciEnvProxyDispatcher()` before routing commands.

## Risks / Open Questions

- There is still no targeted regression test covering `HTTP_PROXY`/`HTTPS_PROXY` for `openclaw message ...` commands, so future CLI startup refactors could regress this silently.
- This review only closes the PR as redundant for the reported CLI path; it does not prove every non-CLI caller of `runMessageAction()` is independently proxy-bootstrapped.
