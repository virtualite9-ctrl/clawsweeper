## Summary

Keep #39688 open. Current main still emits `message:received` and `message:sent` internal hooks without retaining the hook event or draining `event.messages` back through reply delivery, while the public hook type/docs still describe `messages` as user-sendable. The provided discussion also points to open PR #41375 as the active maintainer fix path rather than evidence that the bug is already shipped.

## Best Possible Solution

Keep #39688 open until PR #41375 or an equivalent maintainer-approved fix lands. The fix should drain internal hook replies only where a safe reply target exists, add regression coverage for hook-produced reply delivery, and update the hooks docs so `event.messages` auto-delivery matches the actual reply-capable surfaces.
