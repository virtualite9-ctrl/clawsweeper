## Summary

Current main still does not deliver messages added by `message:received` or `message:sent` internal hook handlers. The issue remains a current behavior bug, and the active maintainer path appears to be open PR #41375 or an equivalent fix rather than cleanup closure.

## Best Possible Solution

Keep #39688 open until PR #41375 or an equivalent maintainer-approved fix lands. The fix should drain internal hook replies only on reply-capable surfaces, add regression coverage for hook-produced reply delivery, and update the hooks docs so `event.messages` auto-delivery is limited to the events/surfaces that actually route replies.
