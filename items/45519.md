## Summary

Keep open. Current main still does not implement the requested Control UI feature for collapsible assistant messages. Existing collapse behavior is limited to tool output/tool cards and JSON blocks, while normal assistant markdown is rendered inline with no length threshold, default-collapsed state, per-message Show more/Show less toggle, or collapse-all/expand-all chat control.

## Best Possible Solution

Implement this in the Control UI chat renderer: add per-session/per-message collapse state for assistant bubbles, derive collapse eligibility from rendered line count or measured height, keep the streaming/latest assistant reply expanded, provide accessible Show more/Show less controls, and consider chat-level Collapse all / Expand all controls. Preserve the existing tool-card and JSON collapse behavior, and add focused UI tests for long prose, code blocks, tables, mobile width, and streaming transitions.
