## Summary

Current `main` already contains ACP session-recovery logic for the reported failure chain: stale persisted ACP resume IDs are stripped and retried fresh, and early `acpx exited with code 1` / missing resumed-session failures are retried instead of leaving the session broken. The accompanying tests cover the same `Resource not found` and `acpx exited with code 1` signatures from this report.

## Evidence

- **Fresh retry after stale persisted resume ID fails:** `AcpSessionManager.ensureRuntimeHandle` now catches `ACP_SESSION_INIT_FAILED` on persisted resume, logs the failure, removes stale `acpxSessionId`/`agentSessionId`, and retries `ensureSession()` without `resumeSessionId`. This directly matches the issue/comment pattern where dead-session repair hit `Resource not found` during resume.
