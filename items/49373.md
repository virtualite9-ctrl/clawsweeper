## Summary

Keep open. Current `main` still resolves Discord thread bindings through the current bot account only, so a follow-up message handled by another Discord account would miss the existing binding and fall back to normal routing instead of honoring exclusive thread ownership.

## Evidence

- **Preflight lookup uses the current Discord account:** Discord preflight resolves runtime conversation bindings with `accountId: params.accountId`, so lookup is tied to the receiving bot account.
  - file: [extensions/discord/src/monitor/message-handler.preflight.ts:646](https://github.com/openclaw/openclaw/blob/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4/extensions/discord/src/monitor/message-handler.preflight.ts#L646)
  - command: `nl -ba extensions/discord/src/monitor/message-handler.preflight.ts | sed -n '632,692p'`
  - sha: [f3c37a946cc4](https://github.com/openclaw/openclaw/commit/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4)
- **Session binding service resolves by channel+account only:** `resolveByConversation` selects one adapter for the normalized conversation's channel/account pair and calls only that adapter; it does not scan other Discord accounts for an existing binding.
  - file: [src/infra/outbound/session-binding-service.ts:353](https://github.com/openclaw/openclaw/blob/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4/src/infra/outbound/session-binding-service.ts#L353)
  - command: `nl -ba src/infra/outbound/session-binding-service.ts | sed -n '172,190p;338,365p'`
  - sha: [f3c37a946cc4](https://github.com/openclaw/openclaw/commit/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4)
- **Discord thread binding manager lookup is account-scoped:** `getByThreadId` builds the binding key from `accountId + threadId` and rejects entries whose `entry.accountId !== accountId`, so another bot account cannot see the binding.
  - file: [extensions/discord/src/monitor/thread-bindings.manager.ts:333](https://github.com/openclaw/openclaw/blob/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4/extensions/discord/src/monitor/thread-bindings.manager.ts#L333)
  - command: `nl -ba extensions/discord/src/monitor/thread-bindings.manager.ts | sed -n '330,360p'`
  - sha: [f3c37a946cc4](https://github.com/openclaw/openclaw/commit/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4)
- **Monitor creates one thread-binding manager per Discord account:** Discord provider startup instantiates the thread-binding manager with the specific `account.accountId`, reinforcing per-account ownership rather than cross-account exclusivity.
  - file: [extensions/discord/src/monitor/provider.ts:774](https://github.com/openclaw/openclaw/blob/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4/extensions/discord/src/monitor/provider.ts#L774)
