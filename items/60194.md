## Summary

Current `main` uses plugin-owned Brave web-search config (`plugins.entries.brave.config.webSearch.apiKey`), auto-detects configured API-backed providers before DuckDuckGo, and auto-enables provider plugins from plugin-owned `webSearch` config. The report relied on `skills.entries.brave-search`, which is not the current web-search config contract.

## Evidence

- **Brave docs now point to the plugin-owned config path:** The Brave setup docs show `plugins.entries.brave.config.webSearch.apiKey` as the canonical config location and note that only legacy `tools.web.search.apiKey` is compatibility-loaded.
