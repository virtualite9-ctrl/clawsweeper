## Summary

Keep PR #72001 open. Current main still has the reported Gmail wizard/config mismatch: the wizard writes the Gmail preset but does not write `hooks.allowedSessionKeyPrefixes`, while the built-in Gmail preset uses a templated `sessionKey` that the gateway config resolver rejects without prefixes. The PR is relevant but should not be cleanup-closed; it likely needs maintainer revision because current docs/runtime also require `hooks.allowRequestSessionKey=true` for templated mapping session keys, and the existing review comment flags a real defaultSessionKey/prefix regression risk in the PR as written.

## Best Possible Solution

Keep the PR open for maintainer review or revision. The best fix is a targeted Gmail setup change that emits a complete hook session policy for the Gmail preset, preserves existing prefixes/defaultSessionKey safely, handles static-shadow configurations without forcing an incompatible prefix, and adds regression coverage for both config-load and actual mapped Gmail dispatch behavior.
