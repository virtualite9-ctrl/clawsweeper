## Summary

Keep open. Current `main` still appears to have the gap described in #54192: CLI fallback/resume paths reuse `cliSessionId` and resubmit the prompt, while the embedded runner has explicit orphaned-user-message repair and the CLI runner does not.

## Evidence

- **CLI fallback path reuses existing CLI session binding:** `runWithModelFallback(...)` calls `runCliAgent(...)` with `cliSessionId: cliSessionBinding?.sessionId`, so fallback attempts continue against the existing CLI session when one is bound.
  - file: [src/auto-reply/reply/agent-runner-execution.ts:944](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/auto-reply/reply/agent-runner-execution.ts#L944)
  - command: `nl -ba src/auto-reply/reply/agent-runner-execution.ts | sed -n '880,1005p'`
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **CLI runner has no orphaned-user-message repair:** `runPreparedCliAgent()` always appends the current prompt as a new user turn for hooks, then calls `executeCliAttempt(...)`. Its only retry special-case is `session_expired`, which retries without the session id; there is no logic analogous to `mergeOrphanedTrailingUserPrompt(...)` for resumed-session fallback retries.
  - file: [src/agents/cli-runner.ts:147](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/agents/cli-runner.ts#L147)
  - command: `nl -ba src/agents/cli-runner.ts | sed -n '100,340p'`
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Resumed CLI execution still rebuilds and sends the prompt:** `executePreparedCliRun()` sets `useResume` from `cliSessionIdToUse`, but still rebuilds `prompt`, resolves prompt input, and passes `promptArg`/`stdin` into CLI args for the resumed run. There is no dedupe or trailing-user-turn inspection before sending.
  - file: [src/agents/cli-runner/execute.ts:179](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/agents/cli-runner/execute.ts#L179)
  - command: `nl -ba src/agents/cli-runner/execute.ts | sed -n '175,330p'`
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Embedded runner already has the requested repair pattern:** The embedded attempt path explicitly checks for a trailing user leaf and calls `mergeOrphanedTrailingUserPrompt(...)` to merge/remove orphaned user messages before continuing, which matches the issue's proposed fix direction and highlights the missing parity in the CLI path.
  - file: [src/agents/pi-embedded-runner/run/attempt.ts:2132](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/agents/pi-embedded-runner/run/attempt.ts#L2132)
