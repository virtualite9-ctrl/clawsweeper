## Summary

Current `main` does not contain this PR's token-resync change, and static inspection does not give high-confidence grounds to auto-close it. The current web UI still resolves media auth from `hello.auth.deviceToken` or `settings.token`, while the gateway test suite explicitly shows `hello.auth.deviceToken` is absent on shared-token control-UI connections. That makes the reported gap plausible enough that conservative cleanup should leave the PR open.
