## Summary

Keep this PR open. Current main still lacks the proposed structured `status --all` degraded-state recovery guidance: it prints config issues, the last gateway error line, skipped channel status, and raw health errors, but does not emit the PR's targeted `Cause:` / `Fix:` hints for invalid config, bind failures, gateway unreachable/query-failed paths, or OAuth re-auth. The PR is focused candidate work, but it needs a rebase and review fixes before merge.

## Best Possible Solution

Keep the PR open as candidate work. Rebase it onto current main, implement the recovery classifiers in `src/commands/status-all/diagnosis.ts`, reuse the existing OAuth refresh failure helper from `src/agents/auth-profiles/oauth-refresh-failure.ts`, address the duplicate-output and bounded-status matching review comments, then run the changed command/test lane before maintainers review for merge.
