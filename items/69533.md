## Summary

Keep #69533 open. Current main still hardcodes the Dream Diary narrative system prompt and prompt builder inside `memory-core`, the memory-core dreaming config/schema/docs expose scheduling and phase/storage tuning but no narrative prompt/template field, and the Dreams UI only toggles enabled state plus diary actions. The provided related items are broader system-prompt/cache reports and do not supersede this dream-specific customization request.

## Best Possible Solution

Implement this as a memory-core plugin-owned feature, not a generic core system-prompt override: add a typed, schema-validated config path under `plugins.entries.memory-core.config.dreaming` for the Dream Diary narrative prompt/template; define supported placeholders and defaults; thread it through `buildNarrativePrompt`/`generateAndAppendDreamNarrative`; add focused tests for default compatibility, placeholder rendering, and invalid config; then expose it in the Dreaming settings UI and update the public Dreaming/memory config docs.
