## Summary

Keep #25493 open. Current main still hardcodes the requested Safety and OpenClaw CLI Quick Reference prompt sections, and the supported alternatives remain whole-prompt replacement, legacy prompt hooks, or narrow provider/runtime section overrides that do not cover Safety or CLI reference. The related PR #25938 was closed unmerged in the provided GitHub context, and current source inspection matches the earlier maintainer-side Codex review comment that kept the issue open.

## Best Possible Solution

Keep #25493 open. The best maintainer path is to either add documented and tested section/template overrides for selected OpenClaw-owned prompt sections such as Safety, tool-call/system tips, and CLI reference, or make an explicit product decision that those sections are intentionally not section-editable and document `systemPromptOverride` plus prompt hooks as the supported escape hatches.
