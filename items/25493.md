## Summary

Keep #25493 open. Current main still hardcodes the requested Safety and OpenClaw CLI Quick Reference prompt sections, does not load proposed workspace override files like SAFETY.md, and only exposes whole-prompt replacement or narrow provider section overrides that do not cover these core sections. The related PR #25938 was closed unmerged, and the latest maintainer Codex comment also kept this issue open.

## Best Possible Solution

Keep #25493 open. The best maintainer path is to either add documented, tested section/template overrides for selected OpenClaw-owned prompt sections with explicit safety and cache-stability constraints, or make a deliberate product decision that Safety/core sections are intentionally not section-editable and document systemPromptOverride as the supported full-replacement escape hatch.
