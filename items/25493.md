## Summary

Keep #25493 open. Current main still hardcodes the Safety and OpenClaw CLI Quick Reference sections, does not load proposed workspace override files such as SAFETY.md, and only offers whole-prompt replacement or narrow provider-owned section overrides that do not cover the requested core sections. The related implementation PR #25938 was closed unmerged, and the same touched files are unchanged from the prior maintainer Codex review SHA to current main.

## Best Possible Solution

Keep #25493 open. A complete maintainer path is to add documented and tested section/template overrides for selected OpenClaw-owned prompt sections, or explicitly document that Safety/core sections are intentionally not section-editable and systemPromptOverride is the supported full-replacement escape hatch.
