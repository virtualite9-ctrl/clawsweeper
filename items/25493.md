## Summary

Keep #25493 open. Current main still has prompt customization surfaces, but not the requested editable system prompt template or section-level workspace override flow such as `SAFETY.md`. The Safety and CLI quick-reference sections remain hardcoded in `buildAgentSystemPrompt`, provider overrides are limited to three named sections, and workspace bootstrap loading does not recognize the proposed override filenames. The related PR #25938 is useful prior work but was closed unmerged, so it does not implement or supersede the issue.

## Best Possible Solution

Keep tracking this as a core prompt-builder/product decision. A complete fix would add a documented, tested section/template override path for selected OpenClaw-owned prompt sections, likely wired through `src/agents/system-prompt.ts` and workspace/config loading, with explicit safety and cache-boundary behavior. Alternatively, a maintainer can explicitly reject editable Safety/core sections and document the supported customization boundary.
