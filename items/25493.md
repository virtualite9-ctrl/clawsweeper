## Summary

Keep #25493 open. Current main still hardcodes the requested Safety and OpenClaw CLI Quick Reference sections, does not load proposed section override files such as SAFETY.md, and only supports whole-prompt replacement or narrow provider/runtime section overrides that do not cover these core sections. The related PR #25938 was closed unmerged, and the latest provided issue discussion also records a Codex review keeping the issue open.

## Best Possible Solution

Keep #25493 open. The best maintainer path is to either add documented, tested section or template overrides for selected OpenClaw-owned prompt sections, with explicit safety and prompt-cache constraints, or make a deliberate product decision that Safety and core prompt sections are intentionally not section-editable and document systemPromptOverride as the supported full-replacement escape hatch.
