## Summary

Keep #25493 open. Current main still does not support editable system-prompt templates or section-level workspace overrides such as SAFETY.md; Safety and the CLI quick-reference remain hardcoded, provider section overrides cover only three unrelated section ids, and the documented config escape hatch replaces the entire assembled prompt.

## Best Possible Solution

Keep #25493 open as a core prompt-builder/product decision. A complete fix should add documented, tested section/template overrides for selected OpenClaw-owned prompt sections, or explicitly document that Safety/core sections are intentionally not section-editable and that systemPromptOverride is the supported full-replacement escape hatch.

## Risks / Open Questions

- The existing whole-prompt override is only a workaround and requires users to copy future OpenClaw-owned prompt text manually.
- Editable Safety and core prompt sections need explicit maintainer product judgment around safety semantics and prompt-cache stability.
- A ClawHub plugin is not a complete substitute today because plugin hooks can append/prepend or replace the whole prompt, but do not receive a stable section-level core prompt override API for Safety or CLI reference.
