## Summary

Current main still uses exact provider/model allowlist keys and does not implement hyphen-stripped variants, but this PR should not be closed as obsolete or merged as-is. The proposed unconditional hyphen stripping broadens the model authorization gate for distinct provider ids, while current code and docs use explicit provider aliases/canonicalization rather than a global hyphen-insensitive provider-id contract.

## Best Possible Solution

Keep this PR open for maintainer or contributor rework. The best path is to first provide a concrete reproduction showing which runtime path rewrites a hyphenated provider id, then fix that specific path or model provider equivalence through explicit provider/plugin alias metadata. Regression tests should prove known aliases resolve while distinct providers such as `my-provider` and `myprovider` remain separately allowlisted unless explicitly linked.
