## Summary

Current main only partially fixes #48771. The Windows `schtasks` `Last Result` parser issue is already handled and covered by tests, but the restart health probe still does not treat `1008` closes with `pairing required`, `device identity required`, `device`, `pair`, or `connect failed` as reachable. That leaves the central false-timeout scenario open. PR #48801 is an open related fix attempt, but it has not landed, so the issue should remain open.

## Best Possible Solution

Keep #48771 open until PR #48801 or an equivalent maintainer-owned patch lands. The source fix should broaden restart reachability so local 1008 policy closes that prove a live gateway, especially `pairing required` and `device identity required`, count as healthy-enough during restart verification, with focused coverage in `src/cli/daemon-cli/restart-health.test.ts`. The already-shipped schtasks `Last Result` parser fix should remain as-is.
