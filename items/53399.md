## Summary

Keep open. Current main has partial Chrome MCP mitigations, but the reported default-path hang and orphan cleanup concern are still not fully closed: existing-session still defaults to spawning `npx chrome-devtools-mcp@latest`, `/tabs` does not pass its 300ms reachability budget into the Chrome MCP list path, and the browser plugin still uses the SDK stdio transport directly instead of OpenClaw's process-tree cleanup transport.

## Best Possible Solution

Keep this open and implement a focused browser-plugin fix: propagate timeout and AbortSignal through Chrome MCP availability/list calls, use or adapt the process-tree-aware OpenClaw stdio transport for Chrome MCP subprocesses, close or kill failed pending sessions reliably, and add a regression that simulates a hung default `npx`/`client.connect`/`list_pages` path through `/tabs` proving bounded response time and subprocess cleanup. Keep the `mcpCommand`/`mcpArgs` escape hatch as a workaround.
