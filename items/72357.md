## Summary

Keep #72357 open. Current main still starts bundled LSP stdio servers without a detached process group, disposes them with direct child-process kill only, and Gateway shutdown still sweeps bundled MCP runtimes but not bundled LSP runtimes. The related PR #72367 is the right kind of fix path in the provided context, but it is not merged into this checkout.

## Best Possible Solution

Keep #72357 open and land a focused lifecycle fix, likely via #72367 or equivalent: spawn bundled POSIX LSP stdio servers into disposable process groups, perform graceful LSP shutdown/exit with a bounded wait, call killProcessTree for still-running servers so descendants such as tsserver exit, add a Gateway shutdown sweep for active bundle-LSP runtimes, and cover the behavior with focused bundled-LSP runtime and Gateway close tests.
