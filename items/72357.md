## Summary

Keep open. Current main at ff6044f4411d3e720940d60990e624bd7be29d46 still starts bundled LSP stdio servers without a detached process group and disposes them with direct parent-process kill only, so nested children such as tsserver are not conservatively covered. The linked PR #72367 appears to target the right fix but is open and not merged in the provided GitHub context.

## Best Possible Solution

Keep #72357 open and land a focused lifecycle fix, likely the approach in #72367: spawn bundled POSIX LSP stdio servers into disposable process groups, perform graceful LSP shutdown/exit with a bounded wait, call killProcessTree for still-running servers so descendants such as tsserver exit, add a gateway shutdown sweep for active bundle-LSP runtimes, and cover the behavior with focused runtime and server-close tests.
