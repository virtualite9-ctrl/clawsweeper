## Summary

Keep open. Current main still has plugin slash commands, gatewayClientScopes pass-through, and scoped plugin gateway RPC methods, but it does not implement host-enforced command requiredScopes, trusted reserved-root ownership, or a continueAgent command result. The related RFC PR #71731 is still an open docs/RFC artifact in the provided context, and #71676 closed unmerged, so no merged implementation supersedes this issue.

## Best Possible Solution

Keep the issue open as a real SDK/API design gap. A focused implementation should add an additive plugin command contract for requiredScopes, define trusted reserved-root ownership for bundled/trusted plugins, enforce scopes in host dispatch before handlers run, map continueAgent results into agent continuation for text and native command paths, and add fixture tests for insufficient-scope denial, read-only commands, default stop behavior, continuation, reserved-root rejection, and native/text parity.
