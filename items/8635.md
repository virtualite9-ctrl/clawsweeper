## Summary

Keep #8635 open. Current main still does not expose raw structured token/cost usage through the `session_status` tool. The tool returns a status text card plus minimal details, while structured per-session usage exists only on separate Gateway `sessions.usage` methods and is not available through the default/minimal agent-facing status tool surface.

## Best Possible Solution

Keep the issue open until OpenClaw adds a documented structured agent-facing usage API, either by extending `session_status` with stable `usage.session`, `usage.lastTurn`, cost, and provider-window availability fields, or by adding an equivalent supported tool that reuses the existing `sessions.usage`/transcript usage parsers while preserving session visibility checks. Add focused tests for the tool contract and docs for agent-facing token/cost budgeting behavior.
