## Summary

Keep #8635 open. Current main still does not expose raw structured token/cost usage through the agent-facing `session_status` tool. The tool returns a rendered `/status`-style card plus minimal details, while reusable structured usage remains on separate gateway RPC methods and is not available through an equivalent session tool contract. The related #39400 remains adjacent, not a canonical replacement for this broader raw session/last-turn usage API request.

## Best Possible Solution

Keep the issue open until OpenClaw adds a documented structured agent-facing usage surface. The likely path is to extend `session_status.details` with stable `usage.session`, `usage.lastTurn`, cost, budget/window availability, source/unavailable metadata, and tests, or add an equivalent supported tool that reuses `sessions.usage` and transcript usage parsers while preserving current visibility checks.
