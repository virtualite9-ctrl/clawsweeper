## Summary

Keep #8635 open. Current main still does not expose raw, structured token usage through the agent-facing `session_status` tool; it returns a `/status`-style text card plus `details.statusText`. The existing structured usage aggregation lives under separate Gateway `sessions.usage` methods, and related fixes only improve display/accounting fallback behavior rather than adding the requested API surface.

## Best Possible Solution

Keep the issue open until maintainers add a structured usage payload to `session_status` or define and document an equivalent supported agent-facing usage API. The implementation should reuse existing transcript/session usage parsers and Gateway usage result types, preserve current visibility checks, define stable fields for session totals, last turn, cost, and provider budget/window data, then add focused session tool tests and docs updates.
