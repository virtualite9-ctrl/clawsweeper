## Summary

Current main still does not provide a typed session-lane busy or wait-timeout result to `openclaw agent`. Same-session runs are serialized, the queue only emits diagnostic wait information, Gateway dispatch failures are generic, and the CLI has no queue-policy option or lane-busy error decode.

## Best Possible Solution

Keep this issue open and fix the Gateway/CLI contract. Add a typed session-lane busy or lane-wait-timeout result from the session queue/agent dispatch path, propagate `lane`, `queueAhead`, and `waitedMs` through the Gateway final/error response, make `openclaw agent` print an actionable message, and add focused regression coverage for blocked same-session CLI dispatch. If queueing remains the default, add and document an explicit queue policy or force-queue option.
