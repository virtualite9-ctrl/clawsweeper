## Summary

Keep open. Current main still serializes same-session agent turns through the session lane, lane wait details remain diagnostic-only, the gateway accepts the request before dispatch completion, and the CLI has no typed session-lane busy or wait-timeout path to surface the requested actionable error.

## Best Possible Solution

Keep this issue open and fix the gateway/CLI contract. Add a typed session-lane wait/busy result or error from the queue/agent dispatch path, propagate `lane`, `queueAhead`, and `waitedMs` through the gateway final/error frame, make `openclaw agent` print an actionable message, and add focused regression coverage for a blocked same-session CLI dispatch. If queueing remains default, add and document an explicit queue policy option rather than silently timing out or falling back.
