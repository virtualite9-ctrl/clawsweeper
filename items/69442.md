## Summary

Keep open. Current main has partial docs-discovery support through `docs:list`, `/start/docs-directory`, and `/start/hubs`, but it does not implement the requested generated `docs_map.md` sibling with per-page H2/H3/H4 heading hierarchy. The issue is recent, concrete, and docs-owned, with no protected maintainer signal that would justify auto-closing it.

## Best Possible Solution

Keep the issue open for a docs/tooling implementation: add a generated public docs map that parses `docs/**/*.md` H2/H3/H4 headings, exposes it as a stable docs-site artifact or page near the existing LLM docs entry point, and wires the generator/check into the docs publish path for `docs/**` changes.
