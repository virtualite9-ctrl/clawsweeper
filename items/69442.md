## Summary

Keep open. Current main has useful docs discovery surfaces (`docs:start/hubs`, `openclaw docs`, and `docs:list`), but none implements the requested public single-file docs map with per-page H2/H3/H4 heading hierarchy. The issue is recent, contributor-authored, has no protected-maintainer auto-close constraint, and no canonical duplicate in the supplied related context.

## Best Possible Solution

Keep the issue open and implement it as docs infrastructure: extend or add a docs-generation script that walks published docs pages, extracts stable H2/H3/H4 headings, emits a public `docs_map.md` or equivalent sibling to `llms.txt`, and wires it into the docs publish/check path without replacing the existing docs hubs or live search.
