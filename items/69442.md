## Summary

Keep open. The request is concrete and still actionable: current main has page-level docs discovery (`docs/start/hubs.md`, `docs/start/docs-directory.md`, `pnpm docs:list`, and `openclaw docs` search), but I found no generated `docs_map.md`/`docs-map.md`/`llms` sibling that lists every page's H2/H3/H4 heading hierarchy. The provided related items (#50584 and #63784) are title-search overlaps, not canonical duplicates for this docs-map feature.

## Best Possible Solution

Add a generated docs-map artifact that parses docs markdown headings (H2/H3/H4) into a deterministic `docs_map.md` or equivalent public route, then wire it into docs checks/publish sync. Keep `docs/start/hubs.md` for human page discovery and use the new generated map for LLM navigation.
