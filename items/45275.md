## Summary

Current main already fixes the Windows `pnpm ui:build` path-with-spaces failure reported in #45275. The root script still routes `ui:build` through `scripts/ui.js`, and that launcher now detects Windows `.cmd/.bat/.com` runners, quotes the resolved `pnpm.cmd` path before spawning, and has regression coverage for `C:\Program Files\nodejs\pnpm.cmd`. The changelog also records this exact UI/Windows fix for #45275. The related discussion and #48289 correctly identified the same runner quoting problem, while #55101 later duplicated the same failure mode and is already closed.

## Best Possible Solution

Close #45275 as implemented on current main. Keep the shipped `scripts/ui.js` launcher behavior and regression test; any remaining work should be limited to separately retiring or superseding the older unmerged #48289 PR against the already-fixed main branch.
