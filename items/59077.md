## Summary

Keep open. Current main still ships the unclear default lifecycle reaction emojis reported in #59077, including `stallSoft: "🥱"` and `stallHard: "😨"`, and the Discord/Slack/Telegram status reaction paths still consume those shared defaults unless users configure overrides. The issue is recent, specific, and has a confirming user comment, so it should remain available for a deliberate UX/defaults change.

## Best Possible Solution

Keep the issue open and make a focused core UX change in the shared status-reaction defaults. At minimum, replace the confusing stall defaults with clearer status indicators, then align inline comments, Telegram variants/fallbacks, Slack mappings as needed, exact-default tests, config docs, and changelog attribution for the reporter/commenter.
