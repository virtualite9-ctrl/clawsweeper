## Summary

Keep open. Current main still uses the reported unclear default status reaction emojis: `stallSoft` is `🥱`, `stallHard` is `😨`, and other defaults match the issue table. The existing config override surface is real, but it does not resolve the request to improve defaults for all users. Related reports in the provided context concern other status/reaction bugs and do not supersede this default-emoji UX request.

## Best Possible Solution

Keep the issue open and make a focused core/status-reaction change: update `DEFAULT_EMOJIS` in `src/channels/status-reactions.ts` to clearer defaults, align Telegram variants and Slack emoji-name mapping as needed, update docs/config comments/generated help, add literal-value tests for the intended defaults, and include a user-facing changelog entry.
