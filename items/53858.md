## Summary

Keep this issue open. Current main still has the Nostr DM subscription filter-shape bug and the channel lifecycle path that makes the gateway treat a started Nostr account as exited, triggering auto-restart attempts. The related open PR #42832 is an implementation candidate, but it has not merged and does not directly close this issue.

## Best Possible Solution

Keep this issue open as the canonical Nostr restart-loop/subscription tracker. Review and land a rebased #42832, or prepare a fresh focused patch against current main: pass a plain `Filter` object to `SimplePool.subscribeMany`, remove the unsafe cast, keep `startNostrGatewayAccount` pending until `ctx.abortSignal` aborts, close the subscription and underlying SimplePool relay connections during cleanup, and add regression coverage for the serialized REQ filter shape plus the channel-manager lifecycle so Nostr does not auto-restart after a successful start.
