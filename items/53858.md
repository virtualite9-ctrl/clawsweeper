## Summary

Current main still matches the reported Nostr restart-loop and malformed subscription behavior. The issue should stay open while the linked Nostr fix PR remains unmerged or until an equivalent patch lands on main.

## Best Possible Solution

Keep this issue open as the current Nostr restart-loop/subscription tracker. Review and land PR #42832 if it still applies, or prepare a fresh patch on current main that passes a plain `Filter` object to `SimplePool.subscribeMany`, removes the unsafe cast, keeps `startNostrGatewayAccount` pending until abort via the existing channel-lifecycle helper, closes the subscription and pool during cleanup, and adds regression coverage for the serialized REQ filter shape plus gateway lifecycle so successful Nostr startup is not treated as task exit.

## Risks / Open Questions

- Closing this now would hide a current, concrete bundled Nostr channel bug before a fix has landed on main.
- A narrow filter-shape fix should also handle lifecycle cleanup, otherwise restart attempts can continue to leave relay connections behind.
- The same `subscribeMany` array-cast pattern appears in `extensions/nostr/src/nostr-profile-import.ts`, so maintainers should decide whether to fix all Nostr filter-shape misuse together.
