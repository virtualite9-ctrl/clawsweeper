## Summary

Current main still matches the reported Nostr restart-loop failure. The Nostr DM bus still passes an array-wrapped filter to `SimplePool.subscribeMany`, the gateway supervisor still restarts settled channel start tasks, and the Nostr gateway start path still resolves immediately after creating the bus. Related PR #42832 is open and appears to target the right areas, but it has not landed, so this issue should remain open as the active bug tracker.

## Best Possible Solution

Keep this issue open as the canonical current Nostr restart-loop/subscription tracker. Review and rebase PR #42832 or prepare a fresh patch on current main that passes a plain `Filter` object to `SimplePool.subscribeMany`, removes the unsafe cast, keeps `startNostrGatewayAccount` pending until `ctx.abortSignal` aborts, closes both the subscription and SimplePool relay connections during cleanup, and adds regression coverage for the serialized REQ filter shape plus gateway lifecycle so a successful Nostr start is not treated as an exit.
