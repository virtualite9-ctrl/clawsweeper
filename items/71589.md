## Summary

Keep #71589 open. Current main still only suppresses assistant output when the assistant phase resolves to `commentary`; it does not suppress unphased assistant messages that end with `stopReason: "toolUse"` or include tool-call content. The PR is a recent, active fix for linked issue #71588, and the discussion shows the author followed up on the bot review gaps for `text_end` coverage and snake_case tool-call variants.

## Best Possible Solution

Keep #71589 open for maintainer review. The best path is to land or refine a narrow embedded-subscriber fix that suppresses unphased tool-use assistant continuations from user-visible replies, preserves explicit `final_answer` output and opted-in streaming/progress surfaces, clears already-buffered suppressed text, and keeps regression coverage for both `message_end` and `text_end` block-reply modes. Split unrelated QA/status/lockfile fixes if maintainers want a cleaner landing branch.
