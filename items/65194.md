## Summary

Keep this PR open. Current main still enumerates raw iMessage account IDs for gateway startup, so the reported duplicate default watcher path is not fixed on main. The PR is paired with the same-author open bug #65141 and remains an implementation candidate, but it needs revision before landing.

## Best Possible Solution

Keep this PR open and request a focused iMessage-owned revision. The fix should affect the account list actually used by gateway monitor startup, skip `default` only when it is fallback-only and an enabled named account covers the same local backend, preserve `default` when it has distinct backend identity such as `cliPath` or `dbPath`, handle normalized default keys, and add regression tests for the reported config, disabled-placeholder configs, and distinct-runtime defaults. A maintainer can then choose whether this PR or #65383 should be the canonical implementation.
