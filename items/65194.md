## Summary

Keep this PR open. Current main still starts iMessage accounts from the raw plugin account list, so a config containing both `default` and a named account can still reach two independent watcher startups. The PR is paired with open bug #65141 and remains an implementation candidate, but the submitted patch still needs revision because its new filtered helper is not wired into the gateway startup enumeration path.

## Best Possible Solution

Keep the PR open and request a narrower iMessage-owned revision. The fix should affect the account list actually used by gateway monitor startup, skip `default` only when it is fallback-only and a startup-eligible named account covers the same local backend, preserve `default` when it has distinct backend identity such as `cliPath` or `dbPath`, handle normalized default keys, and add focused regression tests for the reported config plus disabled-placeholder and distinct-runtime cases. A maintainer should also decide whether to continue this PR or the related open alternative #65383, but this PR should not be auto-closed while paired bug #65141 remains open.
