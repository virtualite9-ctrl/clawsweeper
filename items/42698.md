## Summary

Keep open. Current main has moved Bonjour handling into the bundled plugin and already suppresses several Ciao cancellation/assertion paths, but it still misses the exact upstream `IPv4 address changed from undefined to defined` assertion that this PR reports. The PR cannot merge as-is because it targets obsolete `src/infra` files, but the underlying fix is still useful and should be ported to the current plugin-local classifier. Security review found no supply-chain or permissions concern in the provided diff.

## Best Possible Solution

Keep this PR open, but ask for a rebase or maintainer port: update `extensions/bonjour/src/ciao.ts` to match both upstream IPv4 assertion wordings, add focused coverage in `extensions/bonjour/src/ciao.test.ts` and the advertiser handler test if needed, then run the Bonjour/plugin test lane. The old `src/infra` patch should be treated as useful evidence rather than a directly mergeable diff.
