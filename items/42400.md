## Summary

Keep #42400 open. Current main does not implement `neverReply`, does not expose `channels.defaults.neverReply`, and the existing alternatives only cover narrower cases: mention-skipped Telegram/Signal ingest, pending history for require-mention skips, and `sendPolicy: deny` outbound suppression that still runs the agent. The plugin-first direction is plausible and was raised constructively via the `openclaw-listen-only` comment, but current main still does not broadcast `inbound_claim` to unbound plugins, so the PR should not be closed as ClawHub/superseded without a maintainer decision or a merged plugin API path.

## Best Possible Solution

Keep the PR open for maintainer product/API judgment. The likely best path is either to finish a narrowly reviewed core `neverReply` design with per-channel tests and docs, or to land the missing plugin API needed for a ClawHub `openclaw-listen-only` style plugin and then supersede this core PR with that canonical plugin/API track.
