## Summary

Keep open: current `main` still has the reported `chat.history` fallback gap. The handler returns a model-centric `thinkingLevel` when the session has no persisted value, without consulting `cfg.agents.list[].thinkingDefault`, and the UI stores that returned value. The PR’s proposed fix/test are not present on `main`.

## Evidence

- **Current `chat.history` path still skips per-agent fallback:** On `main`, `chat.history` sets `thinkingLevel = entry?.thinkingLevel` and otherwise calls `resolveThinkingDefault(...)` directly. There is no `resolveAgentConfig(cfg, sessionAgentId)?.thinkingDefault` check in this handler.
  - file: [src/gateway/server-methods/chat.ts:1936](https://github.com/openclaw/openclaw/blob/2cd2732ab6695a4b3220ce41166b74aca7097c5f/src/gateway/server-methods/chat.ts#L1936)
  - command: `nl -ba src/gateway/server-methods/chat.ts | sed -n '1874,1958p'`
  - sha: [2cd2732ab669](https://github.com/openclaw/openclaw/commit/2cd2732ab6695a4b3220ce41166b74aca7097c5f)
- **Shared resolver only reads global defaults, not per-agent config:** `resolveThinkingDefault` reads per-model config and `cfg.agents?.defaults?.thinkingDefault`, then falls through to model heuristics/defaults. It does not accept an `agentId` or read `cfg.agents.list[].thinkingDefault`.
  - file: [src/agents/model-thinking-default.ts:26](https://github.com/openclaw/openclaw/blob/2cd2732ab6695a4b3220ce41166b74aca7097c5f/src/agents/model-thinking-default.ts#L26)
