## Summary

Keep open. Current main still has the exact chat.history fallback gap: it uses entry.thinkingLevel when present, otherwise calls resolveThinkingDefault, and that resolver does not read cfg.agents.list[].thinkingDefault. The documented thinking resolution order says per-agent defaults should win before global/model fallback, so PR #70259 remains a plausible focused bug fix rather than cleanup. Related PR #70302 is an adjacent SessionRow path, not a replacement for this chat.history endpoint fix.

## Best Possible Solution

Keep #70259 open or port its small chat.history change onto current main with the focused regression test it proposed. The ideal maintained fix is: entry.thinkingLevel remains highest priority, chat.history then uses resolveAgentConfig(cfg, sessionAgentId)?.thinkingDefault, and only then falls back to resolveThinkingDefault with the catalog. Coordinate separately with #70302 for the sessions.list/sessions.changed SessionRow surface.
