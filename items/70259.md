## Summary

Keep open. Current main still has the exact chat.history fallback bug described by PR #70259: when a session entry lacks a persisted thinkingLevel, the handler calls resolveThinkingDefault with only cfg/provider/model and never checks the per-agent cfg.agents.list[].thinkingDefault. The PR is a focused fix with a sensible test plan and a positive bot review; the related #70302 is adjacent session-row work, not a superseding fix for chat.history.

## Best Possible Solution

Keep #70259 open and either merge or port its narrow chat.history fallback change with a regression test: preserve entry.thinkingLevel as highest priority, then resolveAgentConfig(cfg, sessionAgentId)?.thinkingDefault, then the existing resolveThinkingDefault model/global fallback. Coordinate with #70302 so chat.history and sessions.list expose consistent thinking levels.
