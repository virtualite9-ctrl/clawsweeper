## Summary

Keep this PR open. Current main still suppresses text-only tool summaries and tool-start statuses for normal non-forum group sessions without considering `CommandAuthorized`, while this open PR is a focused implementation candidate for the missing authorized-command exception.

## Best Possible Solution

Keep this PR open and review it as an implementation candidate. The best path is to rebase/adapt it to current main, preserve the provider-agnostic condition, allow tool summaries and tool-start statuses only for non-forum group turns where `CommandAuthorized === true` and the current body is a real control command, retain normal group chatter suppression, and keep regression coverage for authorized, unauthorized, mention-prefixed, and path-like slash inputs.
