## Summary

Keep this PR open. Current main already handles Feishu auto-mode card streaming and preserves boolean/default streaming behavior, but it still does not implement the PR's central object-form `streaming` config, configurable `throttleMs`, or config-driven title. The PR head is also not safe to merge as-is because it is stale against newer Feishu streaming behavior and would regress omitted/default `streaming` configs.

## Best Possible Solution

Keep this PR open for focused rework or maintainer review. The best path is to rebase onto current main, implement a typed Feishu streaming config resolver that accepts boolean and object forms while preserving `undefined` as default-enabled and `false`/`{ enabled: false }` as disabled, wire bounded `throttleMs` and title/header behavior deliberately, update `extensions/feishu/openclaw.plugin.json`, public Feishu docs, and generated config metadata if applicable, then add regression tests for omitted/default streaming, `streaming: false`, `{ enabled: false }`, custom `throttleMs`, custom title/header behavior, and auto-mode card/plain-text routing.
