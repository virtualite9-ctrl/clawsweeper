## Summary

Keep open. Current main already supports Feishu Card Kit streaming for auto-detected card/markdown replies and preserves `streaming: false`, but it still only accepts `channels.feishu.streaming` as a boolean. The PR's remaining object-config contract (`enabled`, `throttleMs`, `title`) is not implemented in the current schema, plugin manifest, docs, runtime wiring, or tests, so it is not safely closeable as implemented or superseded.

## Best Possible Solution

Keep this PR open for maintainer or contributor rework. The best path is a narrowed rebase against current main that implements only the missing Feishu object-form streaming config across `extensions/feishu/src/config-schema.ts`, `extensions/feishu/openclaw.plugin.json`, generated config metadata, docs, and runtime wiring. It should preserve current main's default-enabled boolean behavior and auto-mode routing, add bounded `throttleMs` and admin-configured title support if still desired, and include regression tests for omitted/default streaming, `streaming: false`, `{ enabled: false }`, custom throttle, and custom title behavior.
