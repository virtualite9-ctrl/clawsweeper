## Summary

Keep this PR open. Current main has Feishu auto-mode streaming and default boolean behavior, but it still does not accept the PR's object-form `streaming` config or wire `throttleMs`/config-driven title. The PR is useful but stale and not safe to merge as-is.

## Best Possible Solution

Keep this PR open for focused rework or maintainer review. The best path is to rebase onto current main, add a typed Feishu streaming config resolver that accepts boolean and object forms while preserving `undefined` as default-enabled and `false`/`{ enabled: false }` as disabled, wire bounded `throttleMs` and any title/header override without regressing identity-derived headers/notes, update plugin manifest/docs/generated metadata, and add regression tests for omitted/default streaming, disabled forms, object config, custom throttle/title, and auto-mode card/plain-text routing.
