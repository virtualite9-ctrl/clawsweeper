## Summary

This PR adds a new bundled skill under the core `skills/` tree, but current OpenClaw policy routes new skills to ClawHub/workspace distribution instead of merging them into core unless there is a clear product, security, or maintainer-ownership reason. The branch discussion shows useful work on a self-contained skill bundle, which strengthens the case for publishing it via ClawHub rather than keeping this core PR open.
