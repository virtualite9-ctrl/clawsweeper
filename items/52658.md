## Summary

Keep open. Current `main` still handles Discord reactions as routed system events, not as plugin `message_received` hook events, so the requested behavior is not implemented.

## Evidence

- **Discord docs describe reaction notifications as system events:** The Discord channel docs say reaction events are turned into system events and attached to the routed Discord session; they do not describe plugin hook delivery.
  - file: [docs/channels/discord.md:709](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/docs/channels/discord.md#L709)

- **Discord registers separate reaction listeners from message listeners:** `registerDiscordMonitorListeners` wires `DiscordMessageListener` for message creates, then separate `DiscordReactionListener`/`DiscordReactionRemoveListener` instances for reactions. The reaction path is not routed through the message handler.
  - file: [extensions/discord/src/monitor/provider.startup.ts:253](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/extensions/discord/src/monitor/provider.startup.ts#L253)
