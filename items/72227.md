## Summary

Keep open. Current main has related optimistic-history reconciliation, but I did not find strong enough source/test/history evidence to close this specific fresh regression about a local `You` message coexisting with a gateway/self echo labeled `openclaw-control-ui`. The exact sender-label echo case should remain actionable until reproduced/fixed or covered by a targeted regression test.

## Best Possible Solution

Keep the issue open. Add a focused Control UI controller/render regression test that simulates a local optimistic user message plus a gateway/history/session echo for the same send carrying `senderLabel: "openclaw-control-ui"`; then reconcile by replacing/suppressing the self echo using stable run/idempotency/client identity while preserving real multi-sender user grouping.
