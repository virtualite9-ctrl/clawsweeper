## Summary

Keep #69413 open. Current main still gates Signal group messages only through configured text mention regexes after placeholder hydration; it does not compare `dataMessage.mentions[]` UUID/phone metadata against the bot account. The PR is a focused Signal plugin bug fix with relevant tests and discussion, and the provided follow-up commit addressed the review concern about validating mention spans.

## Best Possible Solution

Keep the PR open and review it as a scoped Signal plugin bug fix. The best path is to merge or revise the native metadata detection in `extensions/signal/src/monitor/mentions.ts` and `event-handler.ts`, keep the span-validation guard from the follow-up commit, add/keep the UUID and phone mention-gating tests, and decide whether the `canDetectMention` no-pattern case should be included now or tracked separately.

## Evidence
