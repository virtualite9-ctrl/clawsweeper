## Summary

Keep PR #69413 open. Current main still hydrates Signal mention placeholders into @uuid/@phone text, but mention gating only checks configured text regexes, so a native mention of the bot's own Signal UUID/phone is not recognized unless the user has configured a matching UUID/phone pattern. The PR is a focused bundled Signal plugin bug fix, and the provided review context shows the malformed-span review comment was addressed in the PR follow-up commit.

## Best Possible Solution

Keep this PR open as a plausible Signal plugin bug fix. The best path is to review or merge the native uuid/phone metadata detection with regression tests, and decide whether to also update canDetectMention so native metadata alone can enforce requireMention when no text mentionPattern is configured.
