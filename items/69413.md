## Summary

Keep PR #69413 open. Current main still only gates Signal group mentions through text regexes after hydrating native Signal mention placeholders to @uuid/@phone, so native metadata for the bot's own uuid/phone is not recognized unless users configure an exact uuid/phone mention pattern. The PR remains a focused Signal plugin bug fix, and the provided review thread shows the span-validation concern was addressed in the PR follow-up commit.

## Best Possible Solution

Keep the PR open and either merge it after maintainer review or reimplement the same behavior on main: detect native Signal mentions by matching dataMessage.mentions[].uuid/number against the configured bot account identity, keep the span/placeholder validation from the PR follow-up, preserve existing text-pattern matching, and add regression coverage for bot uuid, bot phone, and non-bot mention cases.
