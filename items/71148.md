## Summary

Keep open. The report is recent, concrete, and independently confirmed. Current main still ships the Lobster plugin pinned to `@clawdbot/lobster` 2026.4.6, and that package source still has the reported `sharedAjv.compile(...)` and `ajv.compile(userOutputSchema)` hot paths without content-hash memoization or `removeSchema()` cleanup. OpenClaw’s current Lobster plugin runs that package in-process inside the gateway, so this remains an actionable OpenClaw packaging/runtime issue rather than something to auto-close.

## Best Possible Solution

Keep #71148 open. Fix `@clawdbot/lobster` by adding a shared schema-content validator cache or equivalent compile-once strategy for the reported `sharedAjv` and `llm_invoke` schema paths, add a regression/stress test that repeated identical schemas do not create unbounded compiled validators, publish a fixed Lobster package, then bump `extensions/lobster/package.json` and `pnpm-lock.yaml` in OpenClaw. Consider a bounded/LRU cache or explicit `removeSchema()` only as a secondary guard once the common repeated-schema path is memoized.
