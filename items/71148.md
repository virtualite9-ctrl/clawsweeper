## Summary

Current main implements the reported fix: the Lobster plugin installs an Ajv compile cache before loading the embedded Lobster runtime, deduplicates content-identical schemas by stable SHA-256 key, bounds the per-instance cache to 512 entries, and clears/evicts through Ajv removeSchema. Regression tests cover both repeated local Ajv compilation and the installed Lobster sharedAjv runtime path. The fix is on main but not contained in any local release tag I found.

## Best Possible Solution

Close as implemented on main. Keep the shipped Lobster Ajv compile-cache integration and runtime dependency declaration, then let the next release carry the Unreleased changelog fix. Any future heap growth after that release should be handled as a new report with fresh snapshots against the cached runtime.
