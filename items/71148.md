## Summary

Keep open. The reported leak is still plausibly present in current main: OpenClaw still bundles and runs the Lobster embedded runtime in-process, still pins @clawdbot/lobster 2026.4.6 with ajv 8.18.0, and there is no current OpenClaw wrapper-side mitigation or dependency bump that would address the reported Ajv compile-per-call retention. The issue also has a second independent production confirmation in the discussion, so this is not stale or low-signal.

## Best Possible Solution

Keep #71148 open as a real bundled Lobster runtime memory bug. The likely maintainer path is to fix @clawdbot/lobster upstream by memoizing or otherwise bounding Ajv schema compilation for repeated schemas, add regression coverage around repeated workflow/command validation, then bump extensions/lobster/package.json and pnpm-lock.yaml in OpenClaw and document the fix in the changelog. If OpenClaw needs an interim mitigation, it should be a deliberate dependency patch or runtime wrapper approved by maintainers, not an auto-close.
