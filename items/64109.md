## Summary

Current `main` already covers this bug well enough: the qa-lab scenario pack is now optional in npm distributions, `discovery-eval` still has default refs, plugin CLI registration no longer aborts startup on registration errors, and completion-cache coverage exists for the same missing-pack failure mode. This PR is redundant on top of current `main`.

## Evidence

- **Missing qa scenario pack is explicitly non-fatal on main:** `readTextFile()` now returns an empty string when `qa/scenarios/index.md` is absent, and `readQaScenarioPack()` returns an empty pack with an inline comment saying this avoids completion-cache and similar crashes when npm packages do not ship `qa/scenarios/`.
