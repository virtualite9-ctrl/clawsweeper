## Summary

Keep open. Current main still emits `gateway.trusted_proxies_missing` as `warn` for `gateway.bind="loopback"`, and the current test/docs still encode that behavior. The PR is a meaningful fix attempt, but the latest PR discussion flags security-relevant false-negative risks in the proposed effective-bind classifier for `auto` and `custom` fallback cases, so it should be revised rather than closed or merged as-is.

## Best Possible Solution

Keep the PR open, but require a revised implementation before merge. The fix should derive audit severity from fail-closed effective-bind classification aligned with `resolveGatewayBindHost`, avoid environment-only assumptions for `auto`, account for `canBindToHost` fallback behavior, and add regression tests for loopback, non-loopback, custom loopback, custom fallback, and auto container/non-container paths.
