## Summary

Keep open. Current main still has the old behavior: `gateway.trusted_proxies_missing` is only emitted for literal `gateway.bind="loopback"` and is hard-coded as `warn`. The PR is a real implementation candidate, but it is not obsolete or implemented on main, and its latest revision still needs maintainer review around effective-bind classification for `auto` and custom loopback binds.

## Best Possible Solution

Keep this PR open and request a corrected, maintainer-reviewed implementation. The safest path is to downgrade only definitely local-only exposure, fail closed for ambiguous `auto` or fallback-prone binds, keep sibling checks (`bind_no_auth`, `loopback_no_auth`, and Control UI origin checks) semantically coherent, add regression coverage for loopback, lan, custom loopback, custom exposed, and auto/container or fallback behavior, and update the audit-check docs if the severity taxonomy changes.
