## Summary

Keep open. Current `main` still lacks an `openclaw plugins clean` subcommand, while stale plugin cleanup remains routed through `openclaw doctor --fix`. The PR is coherent, narrowly scoped, and addresses an acknowledged user-facing gap rather than something already implemented on `main`.
