## Summary

Keep this issue open. Current main includes an adjacent maintainer fix for compacted transcript rotation, but I did not find source or test evidence that the reported post-compaction retry/session-lock freeze is fully fixed. The active embedded run still returns diagnostics to `idle` only when the retry attempt reaches terminal cleanup, and stuck-session diagnostics remain warning/telemetry only rather than a recovery path.

## Best Possible Solution

Keep this issue open as the canonical tracker for the compaction retry/session-lock freeze. Add a focused regression that exercises overflow compaction followed by retry success, retry error, and a hung or lost retry terminalization path; assert that active embedded run state clears, diagnostics return to idle or a bounded abort state, `queueDepth` drains, and follow-up messages can proceed. If the transcript-rotation fix is intended to resolve the report, add an integration-level proof tying that fix to the `processing` lock and stuck-session diagnostics before closing.
