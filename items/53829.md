## Close Comment

Closing from a Codex review: current `main` already ships Control UI localization for Chinese (`zh-CN`). The implementation is in `ui/src/i18n/lib/registry.ts` and `ui/src/i18n/locales/zh-CN.ts`, the language picker is in `ui/src/ui/views/overview.ts`, startup locale detection/persistence is in `ui/src/i18n/lib/translate.ts` and `ui/src/ui/storage.ts`, and the behavior is documented in `docs/web/control-ui.md`. This does not use an `openclaw.json ui.locale` key; instead it auto-detects from the browser and persists the user’s chosen locale per browser session/settings. Given that the requested user-facing zh-CN localization and locale switching are already present on current `main` (`8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf`), this issue no longer needs to stay open.
