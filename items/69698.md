## Summary

Current main still has the reported transport/schema mismatch. Persistent exec approval allowlist entries can include internal `source` metadata, `exec.approvals.get` only redacts socket data before returning the snapshot, and the Control UI clones that snapshot and sends it back to `exec.approvals.set`, whose schema rejects unknown allowlist properties. PR #69732 is the directly related proposed fix, but it is still open/unmerged in the provided GitHub context, so the issue should remain open.

## Best Possible Solution

Keep #69698 open until #69732 or an equivalent patch lands. The fix should redact internal allowlist metadata such as `source` from gateway and node exec approval snapshot transport payloads while preserving it in the persisted store, and should include regression coverage proving `exec.approvals.get` returns a UI-saveable file that `exec.approvals.set` accepts.
