## Summary

Current main still has the Control UI round-trip bug: persisted exec approval allowlist entries can contain internal `source` metadata, gateway/node snapshots still return that metadata, the UI saves the returned file back unchanged, and the protocol validator rejects `source` on `exec.approvals.set`. The linked PR #69732 is an active unmerged fix attempt, not shipped behavior.

## Best Possible Solution

Keep #69698 open until #69732 or an equivalent fix lands on main. The best fix is to strip transport-internal allowlist metadata such as `source` from gateway and node-host exec approval snapshots, or otherwise sanitize the Control UI save payload, and add regression coverage proving an `exec.approvals.get` snapshot can be submitted back through `exec.approvals.set` without validation errors.
