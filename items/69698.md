## Summary

Current `main` still has the reported Control UI round-trip bug. Persisted exec allowlist entries can include internal `source` metadata, `exec.approvals.get` returns the normalized file after only socket redaction, the Control UI clones that snapshot and sends it back unchanged on save, and the `exec.approvals.set` schema still rejects allowlist entry properties outside the public transport shape. The linked PR #69732 appears to target the right fix, but it is open/not merged in the provided context, so the issue should remain open.

## Best Possible Solution

Keep #69698 open and land a transport redaction fix, ideally the linked PR #69732 or equivalent: preserve `source` internally for durable allow-always semantics, but strip internal allowlist metadata from gateway and node approval snapshot payloads before they reach Control UI/CLI transport, with a regression test for get-then-set round trips containing `source`.
