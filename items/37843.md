## Summary

Keep #37843 open. Current main still treats `payload.model: "default"` as a non-empty cron model override, and the related fix PRs were closed unmerged rather than shipped.

## Best Possible Solution

Keep the issue open. Implement `default` in the cron runtime contract by treating trimmed `payload.model === "default"` as unset before `resolveAllowedModelRef`, update `resolveCronFallbacksOverride` so the sentinel preserves omitted-model fallback behavior, and add focused tests for omitted, empty, whitespace-trimmed `default`, explicit provider/model, invalid/disallowed models, and fallback handling. Update cron CLI/docs if `--model default` becomes supported user-facing behavior.
