## Summary

Keep #37843 open. Current main still treats cron payload.model: "default" as a non-empty model override, routes it through normal model resolution, and also treats it as an override for fallback-policy purposes. The related fix PRs in the provided context are closed unmerged, so the requested sentinel behavior is not shipped.

## Best Possible Solution

Keep the issue open. Implement a shared cron helper that trims payload.model and treats exact "default" as unset before resolveAllowedModelRef, use that same helper in resolveCronFallbacksOverride so fallback behavior matches omitted model, add focused coverage for undefined, empty, whitespace-only, whitespace-trimmed default, explicit provider/model, invalid/disallowed models, and fallback-chain behavior, then update CLI/docs if --model default becomes supported user-facing behavior.
