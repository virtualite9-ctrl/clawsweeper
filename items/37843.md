## Summary

Current `main` does not implement `payload.model: "default"` as a runtime-default sentinel for cron jobs. The isolated cron selector still treats any non-empty `payload.model` as a literal override, and bare `default` would be parsed as `<defaultProvider>/default` and then validated like any other model id. Tests and docs cover empty/whitespace-as-unset and explicit per-job model overrides, but not the requested `"default"` behavior. The relevant selector file is also unchanged between `v2026.4.22` and `HEAD`.

## Evidence
