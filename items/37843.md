## Summary

Keep #37843 open. Current main still trims cron `payload.model` and resolves every non-empty value as an explicit model override, so `model: "default"` is not treated like an omitted model at runtime. The related fix PRs in the supplied GitHub context are closed unmerged, and current docs/tests do not show this sentinel as shipped.

## Best Possible Solution

Keep the issue open. Add a small shared cron helper that trims `payload.model` and treats exact `default` as unset, then use it in both `resolveCronModelSelection` and `resolveCronFallbacksOverride` so model selection and fallback policy match omitted/empty model behavior. Add focused cron tests for undefined, empty, whitespace-only, `default`, whitespace-trimmed `default`, explicit provider/model, invalid/disallowed models, and fallback-chain behavior. Update the CLI/help/docs if `--model default` is intended as a supported user-facing value.
