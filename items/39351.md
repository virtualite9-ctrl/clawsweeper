## Summary

Keep open. Current `main` partially hardens Telegram reply IDs, but it does not implement the PR's finite-positive validation. The current normalization still accepts `0` and negative integers, and the existing tests only cover junk strings/`NaN`, not non-positive numeric IDs.

## Evidence
