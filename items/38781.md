## Summary

Keep #38781 open. Current main still has the macOS Swift node contract gap: the TypeScript node command contract includes `system.run.prepare`, and the agent `exec host=node` path still invokes it before `system.run`, but the macOS app does not define, advertise, or dispatch that command. #69725 is an open alternate fallback PR, not a shipped superseding fix.

## Best Possible Solution

Keep #38781 open until maintainers either merge a direct macOS Swift `system.run.prepare` implementation or explicitly choose and merge an alternate fallback such as #69725. The direct fix should define the Swift command/types, advertise the command from the macOS node, dispatch it in `MacNodeRuntime`, return the plan shape expected by `executeNodeHostCommand`, and add focused tests for advertisement, dispatch, shell-wrapper/rawCommand, env, and path-canonicalization behavior.
