## Summary

Keep #38781 open. Current main still has the macOS node `system.run.prepare` contract gap: the Swift macOS node declares and handles `system.run`, `system.which`, `system.notify`, and exec approval commands, but not `system.run.prepare`, while the agent node exec path still calls `system.run.prepare` before `system.run`. The related alternate fallback PR #69725 is open and unmerged, so this PR is not obsolete or implemented on main.

## Best Possible Solution

Keep #38781 open until maintainers either merge a direct Swift macOS `system.run.prepare` implementation or explicitly choose and merge an alternate fallback such as #69725. The direct path should add the prepare command/types, advertise it from macOS node mode, dispatch it in `MacNodeRuntime`, return the plan shape expected by `executeNodeHostCommand`, and cover command advertisement, dispatch, shell-wrapper/rawCommand, env, and path-canonicalization behavior with focused tests.
