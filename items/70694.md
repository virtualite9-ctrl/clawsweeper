## Summary

Keep open. Current main clearly forwards `sessions_spawn.task` into the native subagent child prompt path, but that only rules out a simple dropped-argument bug. The issue is fresh, reports repeated live failures with implausibly low child input-token accounting, and there is no shipped diagnostic guard, regression test, or canonical duplicate for native subagent prompt loss/truncation.

## Best Possible Solution

Keep #70694 open. The best maintainer path is a narrow diagnostic/regression pass around native `sessions_spawn`: carry a task length/hash through tool args, gateway `agent` params, child prompt submission, and completion stats; add a long-task regression for the subagent runtime; and surface a warning or error when child input accounting is implausibly low or the initial task is absent from the submitted prompt.
