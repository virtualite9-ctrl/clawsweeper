## Summary

Keep open. Current main appears to forward native `sessions_spawn` tasks through the expected source path, but that does not resolve #70694: the report is recent, detailed, and specifically about intermittent live child runs where long task text appears absent from the model/provider payload or accounting. I found no shipped fix, no canonical duplicate for native `runtime:"subagent"`, and no regression proof that a 2k-3.5k task survives the full gateway/runner/provider boundary with meaningful diagnostics.

## Best Possible Solution

Keep the issue open and add a focused investigation path: capture the native subagent spawn payload at the gateway `agent` call, the compiled embedded prompt (`context.assembled.promptChars` and trace prompt), and provider usage/cache counters for a long-task run. Then add a regression test that spawns a 2k-3.5k task and asserts the full task appears in the child `params.message`/compiled prompt or surfaces a warning if observed input tokens are implausibly low relative to non-cached prompt text.
