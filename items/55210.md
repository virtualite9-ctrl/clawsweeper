## Summary

Keep open. Current `main` already emits a separate terminal stop chunk on normal stream finalization, but it still sends `"Error: internal error"` and `finish_reason: "stop"` in the same SSE chunk on the error path, and the current test suite still expects that behavior. This PR is therefore not redundant on `main` even though part of it overlaps with newer gateway finalization code.

## Evidence
