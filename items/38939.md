## Summary

Keep #38939 open. Current main still lacks Feishu dispatch-time arbitration across sibling bot accounts, and the related #39853/#39001 PRs are open, narrower, and not shipped replacements for this PR's central duplicate-reply problem.

## Best Possible Solution

Keep the PR open or replace it with a corrected Feishu plugin implementation. The maintainer path should arbitrate before dispatch using structured mentioned bot open_ids, derive the coordinator from Feishu default-account selection rather than a literal main id, handle specialist-only, coordinator-plus-specialist, and multi-specialist group mentions, avoid substring-only implicit mention matching, and add regression tests for each case.
