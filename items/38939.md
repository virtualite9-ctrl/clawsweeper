## Summary

Current `main` still lacks Feishu inbound mention-arbitration for multi-bot group delivery, and this PR has unresolved correctness concerns from review. The item is still actionable, so it should stay open rather than be auto-closed.

## Evidence

- **Current main only checks whether the current bot was mentioned:** `checkBotMentioned()` returns true only for `@all`, direct mention matches, or post-content mention matches. There is no logic to arbitrate between a primary account and sibling specialist accounts before dispatch.

There is no `shouldSkipDispatchForMentionPolicy`-style gate on `main` vs specialist accounts in current `main`.
