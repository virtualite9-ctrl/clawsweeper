## Summary

Keep PR #38939 open. Current main at 406ae72 still lacks Feishu dispatch-time arbitration across sibling bot accounts, so the duplicate-reply behavior described by the PR remains plausible. Related PRs #39853 and #39001 do not supersede this because they address inbound metadata exposure and merge-forward formatting, not pre-dispatch sibling bot arbitration. The PR should remain open or be replaced by a corrected implementation because review comments also identify issues in the PR as written.

## Best Possible Solution

Keep the PR open or replace it with a corrected Feishu plugin implementation. The maintainer path should arbitrate before dispatch using structured mentioned bot open_ids, derive the coordinator from Feishu default-account selection rather than a literal main id, handle specialist-only, coordinator-plus-specialist, and multi-specialist group mentions, avoid substring-only implicit mention matching, and add focused regression tests for those cases.
