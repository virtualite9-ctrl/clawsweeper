## Summary

Keep this PR open. Current main still does not refresh QMD session exports from transcript-update events, and the closed predecessor #65017 was not merged. The PR is a focused implementation candidate, but the current branch needs revision because its non-forced session-delta path can drop transcript changes during an in-flight QMD update, and current main already exposes a memory-core foundation/runtime event boundary that should be considered before adding another agent-harness export.

## Best Possible Solution

Keep this PR open and request revision or maintainer review. The best implementation is to subscribe QmdMemoryManager to transcript-update events only when QMD session export is enabled, filter events to the manager's agent/session files, debounce updates, clean up timers/subscriptions on close, and use the forced/queued QMD update path so transcript updates during a pending update schedule a follow-up export/index pass. Prefer the existing memory-core host foundation or documented runtime.events boundary unless maintainers intentionally want agent-harness expanded, and add focused tests for debounce, agent filtering, cleanup, and the in-flight update race.
