## Summary

Keep this PR open. Current main still lacks QMD session-export refresh from transcript-update events, and this PR is a focused replacement for the closed #65017 work, but it needs revision before merge because the proposed session-delta path can drop transcript changes during an in-flight QMD update and the SDK boundary has shifted on current main.

## Best Possible Solution

Keep this PR open and request revision or maintainer review. The best implementation is to wire QmdMemoryManager to transcript-update events only when QMD session export is enabled, filter updates to the manager's agent, debounce them, clean up timers/subscriptions on close, and use the forced/queued QMD update path so transcript updates during pendingUpdate schedule a follow-up export/index pass. Prefer the existing memory-core host foundation or documented runtime event boundary unless maintainers explicitly want agent-harness expanded, and add focused tests for debounce, agent filtering, cleanup, and the in-flight update race.
