## Summary

Keep open. Current main has improved some subpoints, especially the plugin allowlist warning and quiet Claude CLI diagnostics when no Claude CLI signal exists, but the central diagnostics inconsistency remains. `status --deep --json` still embeds the gateway health path, health still builds Discord summaries from a minimal snapshot that omits runtime/token metadata, and doctor still warns on a SecretRef-managed gateway token before using the async SecretRef resolver that other paths already have.

## Best Possible Solution

Keep this issue open. Fix `health` and `status --deep --json` to reuse the same channel account projection contract as `channels.status`, including runtime `running`/`connected`, credential `tokenSource`, probe, and audit fields. Then update doctor gateway auth to resolve applicable `gateway.auth.token` SecretRefs through the async gateway SecretRef credential path, or consume the same resolvability result as secrets audit, before warning. Add focused regressions for a running configured Discord account and a resolvable gateway auth SecretRef.
