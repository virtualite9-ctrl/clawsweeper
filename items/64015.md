## Summary

Keep open. Current `main` appears to have fixed several parts of #64015 (plugin allowlist warning, Claude CLI drift warning, and the status/channels runtime split), but the doctor gateway-token SecretRef check still looks config-shape-based rather than resolvability-based, so the composite issue is not fully implemented on `main`.

## Evidence

- **Gateway token SecretRef warning still keys off the unresolved config shape:** `runGatewayAuthHealth()` resolves `gateway.auth.token` from `ctx.cfg` and emits the "Gateway token is managed via SecretRef and is currently unavailable" note whenever `resolveSecretInputRef(...).ref` is present. The doctor flow seeds `ctx.cfg` from `configResult.cfg`, and `loadAndMaybeMigrateDoctorConfig()` starts from `snapshot.sourceConfig`, which preserves SecretRef objects rather than proving they are resolvable in-command. I do not see a resolvability check in this path.
