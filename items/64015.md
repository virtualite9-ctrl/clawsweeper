## Summary

Keep open. Current main still has the central diagnostics consistency problem: status --deep includes the gateway health snapshot, but health builds Discord channel summaries from a minimal snapshot that omits runtime and credential-source fields, so it can still report running false/tokenSource none while channels.status uses the richer runtime/account path. Doctor also still emits the gateway SecretRef unavailable note from the synchronous auth resolver path instead of sharing the SecretRef resolvability path used by secrets audit. Some subpoints appear improved, but enough of the reported false positives remain.

## Best Possible Solution

Keep this issue open. Fix health and status --deep to reuse the same runtime/account projection contract as channels.status, preserving running, connected, tokenSource, probe, and audit fields. Then update doctor gateway auth to resolve applicable SecretRefs through the async SecretRef credential path or consume the same resolvability result as secrets audit before warning. Add focused regressions for a running configured Discord account and a resolvable gateway.auth.token SecretRef.
