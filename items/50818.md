## Summary

Keep open. Current main has some session-scoped wake plumbing and the #70258 reply-route fallback for async exec-event replies, but it does not implement this PR's remaining cron/session-key targeting work. In particular, current main has no `resolveEventSessionKey`, does not remap cron-run event queues with custom `session.mainKey`, and still treats explicit `heartbeat.target: "none"` exec completions as internal-only. The linked remaining bug #52305 is still open in the provided context.

## Best Possible Solution

Keep #50818 open. Rebase it onto current main, preserve the already-shipped #70258 reply-route fallback, and revalidate only the remaining #52305 session-targeting scenarios: cron-run enqueue/wake key alignment, custom `session.mainKey`, ACP/CLI watchdog events, node exec events, and the explicit `target: "none"` exec-completion delivery decision.
