## Summary

Current `main` does not contain the Windows PowerShell cmdlet allowlist fallback, and the PR’s latest reviewed design still has a plausible Windows execution regression: once a bare cmdlet is treated as allowlisted, node-host rewrites the run to `segments[0].argv`, which would drop the PowerShell wrapper and likely spawn the cmdlet token directly. This is not strong enough evidence to close conservatively.

## Evidence

- **Current main lacks the proposed cmdlet fallback:** `resolveSegmentAllowlistMatch` on `main` only checks direct executable matches, shell positional argv matches, and shell script matches. There is no Windows bare-cmdlet fallback against the PowerShell executable, so the requested behavior is not already implemented on `main`.
  - file: [src/infra/exec-approvals-allowlist.ts:410](https://github.com/openclaw/openclaw/blob/2cd2732ab6695a4b3220ce41166b74aca7097c5f/src/infra/exec-approvals-allowlist.ts#L410)
  - command: `nl -ba src/infra/exec-approvals-allowlist.ts | sed -n '410,490p'`
  - sha: [2cd2732ab669](https://github.com/openclaw/openclaw/commit/2cd2732ab6695a4b3220ce41166b74aca7097c5f)
- **Windows allowlist execution still strips shell payloads to segment argv:** `resolveSystemRunExecArgv` rewrites allowlisted Windows shell payloads to `segments[0].argv`. If a bare PowerShell cmdlet were newly marked allowlisted, this path would drop the PowerShell host wrapper and execute the inner argv directly.
  - file: [src/node-host/invoke-system-run-allowlist.ts:119](https://github.com/openclaw/openclaw/blob/2cd2732ab6695a4b3220ce41166b74aca7097c5f/src/node-host/invoke-system-run-allowlist.ts#L119)

  - sha: [2cd2732ab669](https://github.com/openclaw/openclaw/commit/2cd2732ab6695a4b3220ce41166b74aca7097c5f)
- **Existing tests show wrapper-preserving behavior matters:** There is already a POSIX regression test that expects allowlisted inner scripts to continue running through their transport shell wrapper. That reinforces the concern that Windows cmdlet allowlisting needs matching execution-path handling, not just allowlist matching.
  - file: [src/node-host/invoke-system-run.test.ts:1413](https://github.com/openclaw/openclaw/blob/2cd2732ab6695a4b3220ce41166b74aca7097c5f/src/node-host/invoke-system-run.test.ts#L1413)
