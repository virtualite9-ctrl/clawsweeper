## Summary

Keep #70341 open. Current main still lacks the PowerShell cmdlet allowlist fallback, so unresolved bare cmdlets cannot match a PowerShell-path allowlist entry. The PR is not obsolete, and the PR discussion shows the author addressed Greptile's P2 follow-ups, but the remaining Codex P1 review is real: once the PR marks a bare cmdlet as allowlist-satisfied, current Windows node-host execution would rewrite the shell payload to the bare segment argv and try to spawn `Get-ChildItem` directly.

## Best Possible Solution

Keep #70341 open as the focused fix vehicle for #52952, but require an update that preserves or reconstructs the PowerShell host when the allowlist match is synthesized for a bare cmdlet. The implementation should keep the resolution layer honest, add node-host execution-plan coverage for the Windows shell-command path, and retain the already-addressed memoization and argPattern test improvements.
