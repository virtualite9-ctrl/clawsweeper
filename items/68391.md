## Summary

Current main still has the reported behavior: `models` declares a parent `--agent` option, but the `set` and `set-image` subcommands do not read or reject it, and their command handlers still update global default model fields. The related fix PR #68399 is closed unmerged and its proposed guard is not present in the checked-out code.

## Best Possible Solution

Land a narrow CLI fix that prevents silent global mutation. The safest maintainer path is to revive the #68399 approach: resolve inherited `--agent` in `models set` and `models set-image`, throw a clear error before calling the write command, and add regression tests proving the write handlers are not invoked. If maintainers want the richer behavior, wire these commands to update `agents.list[].model` using the existing per-agent config surface and update `docs.openclaw.ai/cli/models` plus `docs.openclaw.ai/gateway/config-agents`.
