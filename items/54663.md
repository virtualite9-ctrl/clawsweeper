## Summary

Keep this issue open. Current main still has no schema-backed config option to allow verbose/tool-result delivery in ordinary group chats; the dispatch path still suppresses text-only tool summaries for non-forum groups by default, and related #44027 is only an open implementation candidate for `/verbose` behavior rather than the requested config policy.

## Best Possible Solution

Keep this issue open. The best path is a narrow typed config policy, defaulting to false, with channel defaults plus provider/group/topic overrides as appropriate. Wire it into `dispatch-from-config` and ACP projection, add regression tests proving ordinary groups stay silent by default but deliver text summaries when explicitly enabled, preserve media and exec-approval behavior, and update public docs with a trusted-group warning while coordinating with #44027.

## Risks / Open Questions

- Verbose tool-result output in shared chats can expose commands, paths, URLs, plugin diagnostics, and tool-returned data; any implementation should default to false and document trusted-group use only.
- The config policy should be reconciled with #44027 so OpenClaw does not grow inconsistent mechanisms for group tool-result delivery.
