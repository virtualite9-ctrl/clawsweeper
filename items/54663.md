## Summary

Keep open. Current main still drops text-only tool-result summaries in ordinary non-forum group chats, and I did not find any channel/default config key that lets users opt back in. The existing implementation and tests preserve only forum-topic summaries, media-only tool payloads, and exec-approval payloads for groups.

## Best Possible Solution

Keep this issue open. The best path is a narrow, typed, schema-backed opt-in such as `channels.defaults.allowVerboseToolResults` plus provider group/topic overrides, defaulting to false. Feed that policy into `dispatch-from-config`'s `shouldSendToolSummaries` and verbose-progress gates, add tests proving ordinary groups stay silent by default but deliver text summaries when explicitly enabled, and update public docs with a trusted-group warning.
