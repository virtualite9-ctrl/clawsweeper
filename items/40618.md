## Summary

Keep this PR open. Current main still hardcodes bundled session-memory snapshots under `<workspace>/memory/`, and the PR addresses a real memory indexing/noise problem called out in discussion. The proposed shape fits the existing per-hook config model, but the PR should not merge as-is because its pre-write `fs.mkdir(outputRoot)` can create directories through an in-workspace symlink before the alias-safe writer rejects the final write, and the localized docs edits conflict with current docs policy.

## Best Possible Solution

Keep this PR open and request a small revision: preserve the default `memory` output, add optional workspace-relative `hooks.internal.entries.session-memory.outputDir`, reject empty/absolute/traversal values, write through `writeFileWithinRoot` rooted at the workspace with relative `outputDir/filename`, remove the separate pre-write mkdir or replace it with an alias-safe helper, add regression coverage for custom output dirs, invalid fallback, and symlink alias behavior, and drop localized docs edits. Coordinate with #54524 for broader duplicate-indexing guidance or dedupe policy.
