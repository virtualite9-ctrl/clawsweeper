## Summary

Keep this PR open. Current main still hardcodes bundled `session-memory` output under `<workspace>/memory/`, while the PR addresses a concrete memory indexing/noise conflict described in the discussion and related #54524. The work is not implemented or superseded, but the PR should be revised before merge for symlink-safe directory creation and repo docs policy.

## Best Possible Solution

Keep this PR open and request a small revision: preserve the default `memory` output, add optional workspace-relative `hooks.internal.entries.session-memory.outputDir`, reject empty/absolute/traversal values, write through `writeFileWithinRoot` rooted at the workspace with a relative `outputDir/filename`, avoid any pre-write mkdir that can follow symlinks outside the workspace, add regression tests for custom output dirs, invalid fallback, and symlink alias behavior, and remove localized docs edits. Coordinate with #54524 for any broader duplicate-indexing docs or dedupe policy.
