## Summary

Keep #72179 open. Current main still has internal cron started/finished events, completion webhook/run-log handling, and general plugin hooks, but it does not expose a public typed `cron_lifecycle` plugin hook or hook-runner method. The related issue #8209 was closed as superseded by this PR, and older PR #8855 is closed/unmerged, so #72179 is the active canonical implementation candidate rather than a duplicate or ClawHub-only item.

## Best Possible Solution

Keep #72179 open for maintainer review as the canonical implementation candidate for the remaining cron lifecycle plugin API gap. The best path is to iterate or merge an additive typed `cron_lifecycle` hook that bridges gateway cron started/finished events to plugins with docs and regression tests, while moving external audit/telemetry integrations to plugins built on that core API.
