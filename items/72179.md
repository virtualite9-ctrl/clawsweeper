## Summary

Keep #72179 open. Current main has internal cron `started`/`finished` events and gateway broadcast/webhook handling, but no public `cron_lifecycle` plugin hook, no hook-runner method, and no plugin docs for that surface. The PR is a focused additive core plugin API candidate for open issue #8209, superseding closed unmerged PR #8855, not something that can be closed as already implemented, duplicate, or ClawHub-only.

## Best Possible Solution

Keep #72179 open for maintainer review as the current implementation candidate for #8209. The best path is to iterate or merge an additive core `cron_lifecycle` plugin hook with typed event fields, gateway emission for started/finished cron jobs, docs, and regression tests; external audit/telemetry integrations can then live as plugins on top of that API. Do not close this as ClawHub-only because the missing piece is the core plugin extension point, and do not close it as duplicate because #8855 is closed/unmerged and #8209 is the feature request this PR implements.
