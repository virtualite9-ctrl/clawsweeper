## Summary

Keep PR #32558 open. Current main still lacks a Microsoft Graph upload-session path for Microsoft Teams OneDrive/SharePoint file uploads; the active code and tests only cover single-request `/content` uploads. The PR is a focused channel reliability fix split from broader closed work, and the related predecessor PRs were closed unmerged rather than superseding this implementation.

## Best Possible Solution

Keep this PR open for Microsoft Teams maintainer review. The best path is to land a focused `extensions/msteams/src/graph-upload.ts` change that preserves simple uploads where appropriate, adds Graph upload-session support for OneDrive and SharePoint, handles `202 nextExpectedRanges` and final `200/201`, uses Graph-valid chunk sizing and retry/stall limits, and includes focused regression tests for both upload targets. The cutoff should be reviewed against current Graph guidance before merge. ([learn.microsoft.com](https://learn.microsoft.com/en-us/graph/api/driveitem-createuploadsession?view=graph-rest-1.0))
