## Summary

Keep PR #32558 open. Current main still lacks a Graph upload-session path in the active Microsoft Teams OneDrive/SharePoint upload helpers, and the PR remains a focused Teams reliability improvement rather than conservative cleanup. Current Microsoft Graph docs weaken the PR's old 4 MB threshold because direct `driveItem` content upload is documented up to 250 MB, but upload sessions are still the documented resumable path and Microsoft recommends them for larger/resumable transfers. ([learn.microsoft.com](https://learn.microsoft.com/en-us/graph/api/driveitem-put-content?view=graph-rest-1.0))

## Best Possible Solution

Keep this PR open for Microsoft Teams maintainer review. The best path is a focused `extensions/msteams/src/graph-upload.ts` implementation that uses current Graph guidance for when to switch to upload sessions, sends sequential chunks sized as valid 320 KiB multiples, omits `Authorization` on upload-session PUTs, handles `202 nextExpectedRanges` and final `200/201`, retries/resumes transient failures, and covers OneDrive plus SharePoint in regression tests. If maintainers decide upload sessions are unnecessary under the current 100 MB OpenClaw cap, close it as an explicit product decision and update the stale 4 MB comment/docs accordingly.
