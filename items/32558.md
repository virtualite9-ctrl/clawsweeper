## Summary

Keep this PR open. Current main still only does single-request Microsoft Graph `/content` uploads for Microsoft Teams OneDrive and SharePoint paths, while this PR is a focused attempt to add upload-session fallback tests and implementation. The PR needs maintainer review and rebasing, not closure: the patch is scoped to the Teams upload helper and tests, but Greptile already found a stall-count off-by-one and the current main now has User-Agent handling that the stale patch must preserve.

## Best Possible Solution

Keep this PR open for Microsoft Teams maintainer review. Rebase it onto current main, preserve existing User-Agent handling, fix the stall-count guard, review the fallback threshold against current Graph guidance and the observed Teams failure mode, then land a focused `extensions/msteams/src/graph-upload.ts` implementation with regression tests for OneDrive and SharePoint upload-session behavior.
