## Summary

Keep PR #32558 open. Current main still lacks a Graph upload-session fallback in the active Microsoft Teams OneDrive and SharePoint upload helpers, and the focused PR is not superseded by the closed related PRs. The PR likely needs maintainer revisions before merge, especially the stale 4 MB threshold and reviewed stall-count off-by-one, but the central capability remains missing.

## Best Possible Solution

Keep this PR open for Microsoft Teams maintainer review. The best implementation is a focused upload-session path inside `extensions/msteams/src/graph-upload.ts` for OneDrive and SharePoint large/resumable uploads, with the threshold updated to current Graph guidance, sequential 5-10 MiB chunks that are multiples of 320 KiB, no Authorization header on upload-session PUTs, handling for `202 nextExpectedRanges` and final `200/201`, and regression tests covering OneDrive, SharePoint, retry/resume, and stall detection.
