## Summary

Keep this issue open. Current main still does not satisfy the reported Feishu text-command fast path: Feishu message events enter the same-key inbound debouncer before dispatch, the generic debouncer intentionally keeps non-debounced same-key items behind an active flush, and Feishu sequential-key handling only special-cases abort text and /btw while /status remains on the base chat key. Open PR #43512 is also still an unmerged implementation candidate with closing syntax for this issue, so this should remain open for maintainer review.

## Best Possible Solution

Keep this issue open and have maintainers review, land, or replace #43512 with a current-main Feishu fix that detects command-only Feishu messages before the same-key inbound debouncer/serialization path, preserves existing authorization and mention-gating behavior, and adds regression coverage proving /stop, /new, and /status execute during an active same-chat Feishu run.

## Evidence
