## Summary

Keep #39497 open. Current main still has the reported /agent read-guard gap in the workspace-only sandbox path: the guard maps only containerWorkdir, the sandboxed read-tool wiring passes only that option, and current regression coverage still expects /agent/secret.txt to fail with Path escapes sandbox root when the workspace-root guard is enabled. The /agent read-only mount is still a documented and implemented workspaceAccess: "ro" surface, and the related fix PRs in the provided context were closed unmerged.

## Best Possible Solution

Keep the issue open. The likely fix belongs in the sandboxed file-tool guard/wiring: make wrapToolWorkspaceRootGuardWithOptions validate with the sandbox mount table, or pass an explicit /agent mapping only for the workspaceAccess: "ro" agent workspace mount. Add regression coverage proving /agent/... reads resolve in the intended read-only path while write/edit/apply_patch remain blocked. Treat /tmp reads as a separate policy decision.
