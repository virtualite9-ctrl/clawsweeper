## Summary

Keep open. Current main has partial related fixes for cached, snapshot, and failed plugin-register memory capability restoration, but it does not implement PR #69458's central active-scope behavior. An activating scoped plugin load still clears the memory capability and then skips plugins outside `onlyPluginIds`, so a later active scope that excludes the selected memory plugin can still leave `listActiveMemoryPublicArtifacts` empty. Related PRs #67063 and #68041 overlap, but they are open/unmerged and do not prove this exact OC-40 active scope behavior is shipped on main.

## Best Possible Solution

Keep the PR open for maintainer consolidation. The best product/code path is a minimal mainline fix that preserves the selected memory capability across activating plugin-scope swaps that exclude the memory plugin, while still clearing stale capability on full loader resets or memory-slot disablement, plus regression coverage for the exact OC-40 memory-wiki artifact path. Coordinate with #67063/#68041 rather than closing this until one canonical fix ships.
