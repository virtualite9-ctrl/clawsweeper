## Summary

Keep open. Current `main` still stores the active memory capability inside the scope-swapped memory state and clears it on activating reloads, so the OC-40 scope-switch case described by this PR is not implemented on `main`. The PR discussion also shows follow-up fixes were applied on-branch, so this is not a stale or clearly superseded cleanup candidate.

## Evidence

- **Capability still lives in scope-swapped state on main:** `registerMemoryCapability` still writes to `memoryPluginState.capability`, and `clearMemoryPluginState()` still resets that field. That means the active capability remains part of the scope-swapped memory state on current `main`, not a workspace-scoped value that survives scope changes.
