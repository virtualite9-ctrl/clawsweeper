## Summary

Keep open. Current main has partial memory-capability fixes for cache hits and activate:false snapshot loads, but it still stores the active capability inside the scope-swapped memoryPluginState and clears it during activating plugin-scope reloads. The PR’s central OC-40 scenario, where a later plugin scope excludes the memory plugin but memory-wiki still needs the active memory capability, is not implemented on main. Related PRs are open or closed-unmerged, so there is no shipped canonical replacement to close this against.

## Best Possible Solution

Keep #69458 open or fold it into a maintainer-selected canonical PR that solves OC-40 end to end: preserve the active memory capability across plugin scopes that exclude the memory plugin, define reset semantics for full loader clears and slots.memory="none", keep failed/snapshot/cached loads isolated, and add regression coverage for the exact memory-wiki public-artifact path. Close duplicate attempts only after that behavior is present on main or one canonical open PR clearly owns the remaining work.
