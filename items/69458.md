## Summary

Keep #69458 open. Current main has fixes for cached loads, activate:false snapshot loads, and failed registration rollback preserving `memoryPluginState.capability`, but it does not implement the PR's central active scope-swap behavior: an active scoped load that excludes the memory plugin still calls `clearMemoryPluginState()`, and that function still clears the capability. The related open PRs cover adjacent wiki/memory CLI and snapshot-loss work, but they do not make this PR obsolete on current main.

## Best Possible Solution

Keep the PR open or consolidate it with a canonical memory-scope fix. The best product path is to add a focused regression for active scope A registering the memory capability, then active scope B excluding the memory plugin, and decide whether to preserve the active capability across scoped loads or always include the selected memory slot in scopes that need memory-wiki public artifacts. Reconcile that with #67063/#68041 and resolve the existing review comments before landing.
