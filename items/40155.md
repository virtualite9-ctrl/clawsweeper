## Summary

Keep this issue open. Current main has shared reply dispatch for WebUI and Telegram, and vLLM is not globally tool-disabled, but Telegram still passes channel/topic skill filters and Telegram-specific context into the shared run. There is no source, test, changelog, release, or history evidence that the reported Telegram + vLLM + himalaya mismatch is fixed.

Capture the resolved skill snapshot, effective tool inventory, model compat, messageProvider/channel metadata, and replyOptions for both paths. If Telegram loses himalaya or exec availability, fix the Telegram context/skillFilter/tool-policy path and add a regression test around Telegram dispatch; if it no longer reproduces with that evidence, close later as cannot_reproduce or implemented_on_main.
