## Summary

Keep #32515 open. Current main still does not implement one-click Anthropic browser OAuth onboarding. The bundled Anthropic provider exposes Claude CLI reuse, manual setup-token, and API-key setup; existing OAuth/token handling is runtime compatibility after credentials already exist, not a browser sign-in/account-detection flow.

## Best Possible Solution

Keep this open for maintainer/product handling. If accepted, implement an Anthropic provider-owned browser OAuth auth method in `extensions/anthropic`, expose it through `providerAuthChoices`, use the existing `ProviderAuthContext.openUrl`/OAuth helper seams, persist an auth profile, choose safe subscription-vs-API defaults, update onboarding/configure/docs, and add focused regression coverage. If Anthropic cannot support a sanctioned browser OAuth flow, maintainers should explicitly retitle or close around the supported Claude CLI/setup-token/API-key UX. Keep #48420 separate because it tracks runtime handling for existing Anthropic OAuth/token credentials, not this one-click onboarding flow.
