## Summary

Keep #32515 open. Current main still does not implement one-click Anthropic browser OAuth onboarding: the bundled Anthropic provider exposes Claude CLI reuse, manual setup-token paste, and API-key auth, while docs present those as the Anthropic setup paths. Existing sk-ant-oat runtime handling and ANTHROPIC_OAUTH_TOKEN compatibility reduce friction for already-obtained tokens, but they do not provide the requested openclaw onboard browser sign-in flow.

## Best Possible Solution

Keep the issue open for maintainer/product handling. If accepted, implement this in the bundled Anthropic provider as a provider-owned OAuth auth method using ProviderAuthContext.openUrl and the existing OAuth callback helpers, persist the resulting auth profile, choose safe API-key-versus-subscription defaults, update onboarding/configure/docs, and add regression coverage. If Anthropic does not support a sanctioned browser OAuth flow for this use, maintainers should explicitly retitle or close around the supported Claude CLI/setup-token/API-key UX.
