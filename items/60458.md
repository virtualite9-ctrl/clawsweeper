## Summary

Keep open: current `main` still hardcodes `http://`/`ws://` in the shared Control UI link helper, and non-interactive onboarding plus status/dashboard surfaces still consume that helper. The reported TLS/LAN mismatch is not implemented away on `main` or in the latest release snapshot.

## Evidence
