## Summary

Keep open. Current `main` does not already implement this diagnostic improvement, and the PR as proposed still has unresolved logic gaps in the zero-output image path.

## Evidence

- **Current main only warns on `failed` decisions:** `runCapability` logs a warning only when `decision.outcome === "failed"`; otherwise it falls back to verbose logging. That means skipped zero-output image paths do not get the requested warning.
  - file: [src/media-understanding/runner.ts:919](https://github.com/openclaw/openclaw/blob/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503/src/media-understanding/runner.ts#L919)
  - command: `nl -ba src/media-understanding/runner.ts | sed -n '900,980p'`
  - sha: [b0d2003fe65b](https://github.com/openclaw/openclaw/commit/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503)
- **All-skipped attempts remain non-warning paths:** Empty output and `MediaUnderstandingSkipError` are both recorded as `outcome: "skipped"`. `hasFailedMediaAttempt` only checks for `"failed"`, so a zero-output run where every attempt is skipped will end as `decision.outcome === "skipped"` and will not warn on current `main`.
  - file: [src/media-understanding/runner.ts:730](https://github.com/openclaw/openclaw/blob/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503/src/media-understanding/runner.ts#L730)
  - command: `nl -ba src/media-understanding/runner.ts | sed -n '700,747p'; nl -ba src/media-understanding/runner.ts | sed -n '765,768p'`
  - sha: [b0d2003fe65b](https://github.com/openclaw/openclaw/commit/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503)
- **No-provider path still returns silently as skipped:** When no media-understanding entries resolve, `runCapability` returns `outputs: []` with `outcome: "skipped"` and empty attempts. There is no warning at that early return, so the PR's intended no-provider diagnostic is not implemented on `main`.
