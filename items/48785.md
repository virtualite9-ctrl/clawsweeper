## Summary

Keep open. Current main still models `tools.sessions.visibility` as one coarse enum and routes `sessions_send`, `sessions_history`, `sessions_list`, and session metadata/status through the same action-aware guard. `tools.agentToAgent` adds a cross-agent allow gate, but it does not provide the requested independent send/history/list/metadata knobs.

## Best Possible Solution

Keep this issue open as core config/security work. A good implementation would add a backward-compatible structured visibility model, map the existing string shorthands into that model, make `createSessionVisibilityChecker` enforce per-capability scopes for send/history/list/metadata/status, keep `tools.agentToAgent` as a separate cross-agent allow gate, and update generated config docs plus focused tests for mixed policies such as cross-agent send allowed while history is denied.
