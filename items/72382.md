## Summary

Keep #72382 open. It is a fresh, concrete installer report against a repo-owned surface, and current main still has the same class of behavior: the default installer can warn that the npm global bin dir is missing from PATH while still printing a successful install message, with post-install verification disabled unless the user passes `--verify`.

## Best Possible Solution

Keep the issue open and fix the installer path. The likely maintainer direction is to make a default npm install either leave an immediately usable `openclaw` command or avoid declaring success when the command will not be available after the script exits; add focused coverage for npm global prefix/PATH behavior and update install troubleshooting docs if the intended fix requires opening a new shell.
