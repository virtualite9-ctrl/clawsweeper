## Summary

Current main still does not implement the Feishu UI quote path described in #29487. The plugin supports fetched reply context when `message.parent_id` is present, but there is no modeled, parsed, fetched, or tested `quote.message_id`/quote ID path. Related PR #29501 was closed unmerged and was only a partial parent-message parsing fix; #60891 is related to parse failure handling, not the missing quote identifier.

## Best Possible Solution

Keep #29487 open. Implement the Feishu plugin behavior by verifying the UI quote payload, extending the event/context types to carry the quoted message ID, parsing `quote.message_id` or the verified equivalent, preferring it for explicit quote context while preserving `parent_id` fallback, and adding regression tests for quote-only UI replies plus existing parent_id replies under current group context visibility rules.
