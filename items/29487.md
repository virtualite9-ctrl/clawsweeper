## Summary

Keep #29487 open. Current main still fetches Feishu quoted/replied context only from `message.parent_id`; it does not model, parse, or prefer the UI quote identifier `quote.message_id` called out in the issue discussion, so quote-only Feishu events can still reach the agent without the quoted message content.

## Best Possible Solution

Keep #29487 open. Implement the missing behavior in the bundled Feishu plugin by verifying the UI quote payload shape, extending the event/context types with a separate quoted-message id, parsing `quote.message_id` or the verified equivalent, preferring that id when fetching quoted content while retaining `parent_id` fallback, and adding regression tests for quote-only UI replies plus existing parent-id replies under current group context visibility rules.
