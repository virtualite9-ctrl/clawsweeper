## Summary

Keep open. Current main still only fetches Feishu quoted/replied context through `message.parent_id`; it does not model, parse, fetch, or test the Feishu UI quote identifier called out in the issue discussion (`quote.message_id`). The earlier broad parent-message work from #29501 improved fetched parent-message parsing but did not add the missing quote-ID event path, and #60891 was a separate closed parse-failure report.

## Best Possible Solution

Keep #29487 open. Implement this in the bundled Feishu plugin by verifying the UI quote payload shape, extending the Feishu event/context types to carry the quoted message ID, parsing `quote.message_id` or the verified equivalent, preferring that explicit quote ID when fetching quoted content while preserving `parent_id` fallback, and adding regression tests for quote-only UI replies plus existing parent-id replies under group context visibility rules.
