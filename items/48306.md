## Summary

Keep #48306 open. Current main has repaired the latest published appcast state for v2026.4.24, and the latest release now has the expected macOS zip/dmg/dSYM assets, but the repo still does not show a public guard that prevents the reported class of failure. The public macOS release workflow remains validation-only, appcast validation only checks Sparkle version/build metadata, and the Sparkle client path still relies on Sparkle’s generic download failure handling rather than a repository-specific missing-asset fallback or clearer recovery path.

## Best Possible Solution

Keep the issue open until the macOS release/update path has an auditable guard: build/sign/notarize/upload zip+dmg+dSYM, verify the exact appcast enclosure URL resolves successfully, and only then publish or commit appcast.xml. Add a public release-check or documented artifact proof for this invariant, and consider a macOS app-side clearer missing-update recovery message plus visible app/gateway version-skew status so operators are not stranded if release state is invalid again.
