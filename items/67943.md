## Summary

This issue should stay open. The live GitHub state still has same-author PR #67946 open and unmerged with explicit closing syntax, so it is the active implementation candidate. Current main also still contains the stale-lineage paths described: ordinary session state can preserve stored subagent lineage, the agent run can fall back to `sessionEntry.spawnedBy`, and `sessions_spawn` depth checks use stored `spawnDepth`/`spawnedBy` metadata.

## Best Possible Solution

Keep this issue open while #67946 remains open. Maintainers should review and either land a narrowed version of that PR or replace it with an equivalent fix that clears persisted subagent lineage on ordinary top-level session turns, preserves lineage only for real spawned/subagent/ACP sessions, avoids stale `sessionEntry.spawnedBy` as normal-run spawn context, and adds focused regression coverage.
