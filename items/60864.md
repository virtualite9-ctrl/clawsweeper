## Summary

Current `main` has a restart-sentinel continuation path for explicit/session-scoped restarts, but it does not fully implement this RFC's broader checkpoint + auto-resume behavior. The landed work is strong evidence of progress, not closure: config/apply and config/patch restarts still persist only a note, not a continuation payload; update restarts also omit continuation; and the requested run lifecycle states/checkpoint semantics (`paused_for_restart`, `resuming`, `resumed`, `failed_resume`) are not present on `main`.

## Evidence
