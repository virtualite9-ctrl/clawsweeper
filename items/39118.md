Keep PR #39118 open. Current main still lacks the PR’s core doctor safety behavior for dependency-only companion units, active-state filtering of extra gateway-like services, and running-gateway protection before ExecStart rewrites. The best path is to filter dependency-only companion references without suppressing real custom gateway units, make active-service filtering scope-aware for user versus system services, apply the running-gateway ExecStart guard after all ExecStart-level audit issues have been computed, and add focused regression tests for companion dependencies, custom-named gateways, system-scope deep scans, inactive extras, and active custom launchers.
