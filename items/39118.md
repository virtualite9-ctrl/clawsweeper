## Summary

Keep PR #39118 open. Current main still lacks the PR’s central doctor safety behavior: dependency-only companion systemd units can still be detected as gateway-like, extra gateway-like services are not active-state filtered, and service repair can still rewrite ExecStart without checking whether the gateway unit is already running.

## Best Possible Solution

Keep the PR open and request a revised implementation. The best path is to filter dependency-only companion references without hiding real custom gateway units, make active-service filtering scope-aware for user versus system services, apply the running-gateway ExecStart guard after all ExecStart-level audit issues are known, and add regression tests for companion dependencies, custom-named gateways, system-scope deep scans, inactive extras, and active custom launchers.
