## Summary

Keep #7456 open. Current main at c7a0d9b1889ca09ab5bfd84cda39a8d50d3d5768 still has a linear `openclaw onboard` flow, a prompt contract that only returns prompt values or cancellation, and Gateway wizard RPC methods for start/next/cancel/status with no previous-step/back operation. The directly related back-navigation PR #7762 and dialog-foundation PR #18286 are both closed unmerged in the provided GitHub context, and #5880 is interrupted-session resume, not in-session navigation.

## Best Possible Solution

Keep #7456 open. The best implementation path is a core wizard navigation contract: add a typed Back/previous-step result to `WizardPrompter` and Gateway wizard schemas/session handling, refactor `runSetupWizard` into a typed state machine that preserves answers across backward and forward movement, align local submenu Back behavior with that model, and add CLI/RPC regression coverage. PR #7762 is useful prior art but should not be treated as shipped because it was closed unmerged.
