## Summary

Keep #7456 open. Current main still exposes a forward-only onboarding wizard: the prompt contract has no typed Back/previous result, the Clack adapter only handles prompt cancellation, `runSetupWizard` is still a sequential flow, and Gateway wizard RPC only supports start/next/cancel/status. The related back-navigation PR #7762 was closed unmerged, and #5880 covered interrupted-session resume rather than in-session navigation.

## Best Possible Solution

Keep #7456 open. The best implementation is a typed wizard navigation contract: add a Back/previous-step result to `WizardPrompter`, carry it through Clack and Gateway wizard schemas/session handling, refactor `runSetupWizard` into a typed state machine that preserves answers across backward and forward movement, and add CLI/RPC regression coverage. PR #7762 is useful prior art but should not be treated as shipped.
