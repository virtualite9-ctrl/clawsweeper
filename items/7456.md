## Summary

Keep #7456 open. Current main still has no shared onboarding Back/previous navigation contract: the CLI prompt API, Clack adapter, wizard session, Gateway schema/handlers, generated app protocol model, macOS wizard client, and docs remain forward/cancel only. The related Back PR #7762 was closed unmerged, #5880 is only interrupted-session resume, and #18286 was a separate closed dialog-tool proposal rather than a shipped onboarding wizard fix.

## Best Possible Solution

Keep #7456 open. The best product/code path is an additive typed wizard navigation contract: add explicit Back/previous semantics through WizardPrompter, the Clack adapter, WizardSession state/history, Gateway wizard schemas and handlers, generated clients, and macOS/Control UI rendering; refactor onboarding into a state-machine-like flow that preserves answers while moving backward and forward; and add CLI/RPC regression coverage. PR #7762 is useful prior art but should not be treated as shipped.
