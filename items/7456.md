## Summary

Keep #7456 open. Current main still lacks wizard-wide Back navigation: the prompt contract returns prompt values or cancellation only, the Clack adapter does not inject a Back action, `runSetupWizard` remains a sequential flow, and Gateway wizard RPC exposes only start/next/cancel/status. The related PR #7762 was closed unmerged, and #5880 tracks interrupted-session resume rather than in-session previous-step navigation.

## Best Possible Solution

Keep #7456 open. The best implementation is a typed wizard navigation contract: add an explicit Back/previous-step result through `WizardPrompter`, Clack, and Gateway wizard schemas/session handling; refactor `runSetupWizard` into a typed state machine that preserves answers across backward and forward movement; and add CLI/RPC regression coverage. PR #7762 is useful prior art but should not be treated as shipped.
