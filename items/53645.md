## Summary

Keep open. Current `main` does not show a strong fix for the reported Feishu reply/reference instability. The secondary `card`-parameter complaint appears addressed, but group-message handling still drops non-mention replies, and Feishu reply-target failures for thread replies still hard-fail rather than degrading gracefully.

## Evidence

- **Group replies still require an explicit bot mention:** `handleMessage` resolves `requireMention` and returns early for group messages when `!ctx.mentionedBot`, which would produce the reported `replies=0` behavior on reply-based follow-ups that do not explicitly @mention the bot.
  - file: [extensions/feishu/src/bot.ts:466](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/extensions/feishu/src/bot.ts#L466)
  - command: `nl -ba extensions/feishu/src/bot.ts | sed -n '410,490p'`
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Mention detection does not treat reply-to-bot as a mention:** `checkBotMentioned` only recognizes `@_all`, explicit mention objects, or post-message mentions. It does not inspect `parent_id`/reply context to infer an implicit mention when a user replies to the bot in a group.
  - file: [extensions/feishu/src/bot-content.ts:229](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/extensions/feishu/src/bot-content.ts#L229)
