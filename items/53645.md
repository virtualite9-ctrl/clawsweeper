## Summary

Keep open. Current main has solid lower-level Feishu reply delivery plumbing, but the message-tool `send` path that matches the report still appears to drop explicit `replyTo` for Feishu plugin sends. The report also has a confirming comment that private chats were more stable, so this is not just a one-off stale report.

## Best Possible Solution

Keep this issue open and fix it in the Feishu plugin owner module. In `extensions/feishu/src/channel.ts`, make `action: "send"` honor explicit `replyTo`/`reply_to` as a normal `replyToMessageId` while keeping `thread-reply` as the explicit `reply_in_thread` path. Add focused tests for Feishu message-tool sends with `replyTo` across text, presentation/card, and media paths, plus a test that the already-fixed optional `card` schema does not regress.
