## Summary

Keep open. Current main still drops Feishu message-tool `replyTo` on ordinary `send`: the generic outbound path resolves `replyTo`, and Feishu lower-level send helpers can reply, but the Feishu plugin action handler only maps a reply target for `thread-reply`, not for normal `send`.

## Best Possible Solution

Keep this issue open and fix it in the Feishu plugin owner module. `extensions/feishu/src/channel.ts` should resolve `replyTo`/`reply_to` for ordinary `action: "send"`, pass it as `replyToMessageId` or media `replyToId` for text, presentation/card, and media sends, and keep `thread-reply` as the explicit `reply_in_thread` path requiring `messageId`. Add focused Feishu channel action tests for `send` with explicit `replyTo` across text, card/presentation, and media sends, plus a guard that media-only sends do not require a card payload.
