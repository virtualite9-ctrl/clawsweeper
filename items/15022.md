## Summary

Keep #15022 open. Current main still preserves/splits block reply boundaries around tool execution and assistant-message indexes, and it does not ship the hold/resume coalescer or final/tool-interleaved text coalescing config requested by the issue. The linked PR #16275 described a plausible fix, but the provided GitHub context shows it closed unmerged and its APIs are absent from current main.

## Best Possible Solution

Keep #15022 open as the focused core reply-pipeline issue. The likely implementation should add a core hold/defer mechanism across tool execution, handle sequential and parallel tool calls safely, preserve media/reasoning/duplicate-suppression behavior, and add regression tests proving text/tool/text turns can deliver as one outbound message when the new behavior is enabled.
