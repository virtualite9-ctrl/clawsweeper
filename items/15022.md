## Summary

Keep #15022 open. Current main still lacks the requested opt-in coalescing/final-only delivery for assistant text blocks interleaved with tool calls, and the related hold/resume fix in #16275 was closed unmerged rather than shipped.

## Best Possible Solution

Keep #15022 open as the focused core reply-pipeline feature. The best path is an opt-in agent/channel delivery mode that buffers or coalesces assistant text across text -> tool -> text -> tool -> text turns, likely using hold/defer semantics similar to #16275, with config schema/docs and regression tests for both final payload assembly and block-streamed sequential/parallel tool execution.
