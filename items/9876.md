## Summary

Keep #9876 open. Current main still carries `sessionKey` into the macOS native approval model, but `ExecApprovalsPromptPresenter.buildAccessoryView` does not render it. The richer requester/channel context is also not decoded by the Swift popup model, even though gateway exec approval payloads and tests preserve turn-source metadata on the TypeScript side. The prior Codex review comment from steipete also kept this open, and the current checked-out main still has the same native UI gap.

## Best Possible Solution

Keep #9876 open. The narrow fix should add a trimmed, non-empty `Session` detail row in `apps/macos/Sources/OpenClaw/ExecApprovalsSocket.swift` inside `ExecApprovalsPromptPresenter.buildAccessoryView`. The fuller fix should extend the Swift prompt request model used by native macOS prompts to decode safe gateway turn-source/requester metadata, map it to readable labels, and add focused macOS coverage for socket, gateway, and node-host prompt paths.
