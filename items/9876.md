## Summary

Keep #9876 open. Current main still does not render `sessionKey` in the macOS exec approval popup and does not decode/display channel or sender/requester context in the native prompt model. The Control UI still shows a Session row, so the parity gap remains. The prior issue comment also explicitly kept this open, and related #52368 appears to have addressed session propagation rather than the macOS display surface.

## Best Possible Solution

Keep #9876 open. The narrow fix belongs in apps/macos/Sources/OpenClaw/ExecApprovalsSocket.swift: trim `request.sessionKey` and render a non-empty `Session` detail row in `ExecApprovalsPromptPresenter.buildAccessoryView`. The fuller fix should extend the native prompt request model to decode safe turn-source/requester metadata already carried by gateway approval requests, display readable Channel/Sender/Thread context where available, and add focused macOS coverage for gateway and node prompt paths.
