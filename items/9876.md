## Summary

Keep #9876 open. Current main still carries `sessionKey` in the macOS exec approval prompt request but `ExecApprovalsPromptPresenter.buildAccessoryView` does not render it. The richer requester/channel context exists in gateway exec approval payloads and generated protocol types, but the Swift popup request model used by the macOS presenter still does not decode or display those fields. The latest provided Codex review comment also explicitly kept the issue open, and current main has not changed the affected behavior.

## Best Possible Solution

Keep #9876 open. The narrow fix belongs in `apps/macos/Sources/OpenClaw/ExecApprovalsSocket.swift`: trim `request.sessionKey` and add a non-empty `Session` detail row in `ExecApprovalsPromptPresenter.buildAccessoryView`. The fuller fix should extend the Swift prompt request model used by native macOS prompts to decode safe gateway turn-source/requester metadata, map it to user-readable labels, and add focused macOS prompt/model coverage for socket, node-host, and gateway prompt paths.
