## Summary

Keep open. Current `main` does not implement Lemonade provider support, and the PR’s feature remains coherent and potentially useful. The work is stale against current architecture, though: self-hosted provider builders now live in extension-local files and current SDK contract tests explicitly omit provider-specific exports from `src/plugin-sdk/*`, so the PR would need a rebase/update rather than closure.

## Evidence

- **No Lemonade artifacts on current main:** A repo-wide file search for Lemonade-specific artifacts returned no matches on `main` (`extensions/lemonade`, `docs/providers/lemonade.md`, `LEMONADE_API_KEY`, `buildLemonadeProvider`, `lemonade-defaults`).

  - file: [docs/providers/index.md:48](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/docs/providers/index.md#L48)
