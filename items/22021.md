## Summary

Keep #22021 open. Current main still does not implement `X-Actual-Model` or an equivalent OpenAI-compatible HTTP response contract for the resolved runtime provider/model. The maintainer/Codex comment on the issue remains accurate against `e29d3516`: the HTTP handlers use the request-side agent target model in response bodies and SSE events, while actual model selection data exists deeper in the reply pipeline and is not wired into these transport handlers.

## Best Possible Solution

Keep the issue open. The best implementation path is to thread resolved provider/model metadata from `onModelSelected`, `agentMeta`, or a new run-result metadata seam into `src/gateway/openai-http.ts` and `src/gateway/openresponses-http.ts`; set and document the response contract for non-streaming responses; define workable SSE semantics; and add focused tests in the two gateway HTTP test files plus public Gateway docs.
