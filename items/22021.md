## Summary

Current main still does not implement the requested `X-Actual-Model` response header or an equivalent transport-layer exposure of the resolved runtime model. The OpenAI-compatible Gateway handlers continue to return the request/agent target model in response bodies and SSE events, while the actual selected-model callback exists only in the reply pipeline and is not wired into these HTTP responses. The provided maintainer/Codex comment is consistent with current source and docs, and the related closed items do not supersede this specific HTTP header request.

## Best Possible Solution

Keep this issue open. The best path is to wire the resolved model/provider from the existing selected-model/fallback pipeline into the OpenAI-compatible Gateway HTTP surfaces, set a documented response header for non-streaming and streaming responses where feasible, and add focused tests in `src/gateway/openai-http.test.ts` and `src/gateway/openresponses-http.test.ts`. Public docs should be updated at https://docs.openclaw.ai/gateway/openai-http-api when the contract is decided.
