## Summary

Keep #22021 open. Current main still has no `X-Actual-Model` response header or equivalent transport-level actual-runtime-model field for the OpenAI-compatible HTTP endpoints. The latest maintainer/Codex comment remains accurate against `7e13f3f5140168e31027b39a9855ade3fadf24d5`: Gateway handlers still emit the request-side agent target model, while actual provider/model metadata exists deeper in the agent runtime but is not threaded back into HTTP responses.

## Best Possible Solution

Keep the issue open. The best implementation path is to define a Gateway HTTP response contract for actual provider/model metadata, thread resolved model data from `onModelSelected`, `agentMeta`, or a new run-result metadata seam into `src/gateway/openai-http.ts` and `src/gateway/openresponses-http.ts`, decide non-streaming vs SSE behavior, document it on `https://docs.openclaw.ai/gateway/openai-http-api` and the OpenResponses page, and add focused tests in both Gateway HTTP test files.
