## Summary

Keep this issue open. Current main still has a global inbound media store: `saveMediaBuffer` defaults to `inbound`, bundled channel paths still pass the literal `inbound`, managed inbound references reject nested/scoped IDs, and sandbox staging still flattens media into `media/inbound/<filename>`. The related implementation PR #46162 is closed and unmerged, while duplicate reports #25712 and #67194 point back here as the canonical tracker.

## Best Possible Solution

Keep this issue open as core media-pipeline work. A maintainer-quality fix should define a backwards-compatible scoped inbound media contract, thread account or resolved agent context through bundled and plugin media saves, support scoped `media://inbound/<scope>/<id>` and absolute-path resolution, preserve existing flat `media/inbound` files, update sandbox staging and cleanup, and add regression coverage plus docs. PR #46162 is useful prior art but should be reopened, reviewed, or replaced before this issue closes.
