## Summary

Keep open. Current main still writes inbound media into the shared `media/inbound` pool, bundled channel save paths still pass the literal `"inbound"`, and the managed-media read side/test contract still rejects nested scoped inbound references. The related implementation PR #46162 is closed and unmerged, while duplicate reports #25712 and #67194 point back here as the canonical tracker.

## Best Possible Solution

Keep this issue open as core media-pipeline work. A maintainer-quality fix should define a backwards-compatible scoped inbound media contract, thread account or resolved agent context through bundled and plugin media saves, support scoped `media://inbound/<scope>/<id>` and absolute-path resolution, update sandbox staging and cleanup, preserve access to existing flat `media/inbound` files, and add regression coverage plus docs. PR #46162 can be used as prior art but should be reviewed, reopened, or replaced.
