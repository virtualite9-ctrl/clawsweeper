## Summary

Keep this issue open. Current main still saves inbound media into a shared flat `media/inbound` store, bundled Discord/Telegram/Signal media paths still pass the literal `"inbound"` subdir, and the read-side media reference tests still reject nested inbound paths that an account- or agent-scoped implementation would need. The prior implementation PR #46162 is closed and unmerged in the provided context, while #67194 was closed as a duplicate of this issue, so this remains the canonical tracker.

## Best Possible Solution

Keep this issue open as core media-pipeline work. A maintainer-quality fix should define a backwards-compatible scoped inbound media contract, thread account or resolved agent context through bundled channel/plugin media saves, support scoped `media://inbound` and absolute-path resolution, update sandbox staging and cleanup behavior, preserve access to existing flat `media/inbound` files, and add regression coverage plus docs. PR #46162 is useful prior art but should be reviewed or replaced because it is closed and unmerged.
