## Summary

Keep open. Current main still lacks the requested first-class `author` field on `chat.send`, does not persist author metadata in chat-send transcript rows, and only derives displayed user attribution from an existing `senderLabel` or inbound metadata envelope. The issue is recent, has no protected-label or maintainer-author auto-close blocker, and the provided related title-search items do not supersede this specific gateway protocol request.

## Best Possible Solution

Keep this issue open for a gateway protocol/storage implementation: add a validated optional `author` object to `ChatSendParamsSchema`, thread it through `chat.send` persistence, surface `senderLabel` plus `authorId`/`authorName` in `chat.history` and `session.message`, and add focused gateway/protocol/UI tests. Treat the sanitizer timestamp hardening as a separable follow-up unless maintainers want it bundled.
