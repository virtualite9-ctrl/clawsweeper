## Summary

Keep open. Current main has related generic outbound diagnostics and WhatsApp send/reaction logging, but it does not fully implement the issue's requested WhatsApp delivery-state reconciliation. The direct WhatsApp auto-reply path still delivers through its own `deliverWebReply` path, only marks `didSendReply` after the send returns, and lacks a structured per-attempt snapshot or operator-visible warning that distinguishes assistant transcript text from actual provider delivery for group text/reply sends.

## Best Possible Solution

Keep the issue open and implement a unified WhatsApp delivery-state snapshot for direct auto-replies and shared outbound sends. It should record text, quoted reply, media, and reaction attempts after provider invocation, include returned provider message ids or failure reasons, surface a clear operator/debug warning when assistant text exists without successful provider delivery, and add regression coverage for reaction-success/text-failure and reply-failure split paths.
