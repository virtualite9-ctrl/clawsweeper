## Summary

Keep open. Current `main` still has separate WhatsApp reaction and message/reply send paths, and I did not find a maintainer-grade fix that reconciles session-visible assistant text with actual WhatsApp transport success for this path. The generic reply flow still resolves the model reply before outbound delivery, while the WhatsApp monitor path only remembers sent text after `deliverReply` returns; there is no corresponding delivery-state warning or transcript/transport reconciliation on this route.
