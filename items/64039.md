## Summary

Keep open. Current `main` still has a real asymmetry between explicit Telegram sends and the normal reply pipeline: explicit sends record the sent-message cache, but the default reply/draft-finalization path does not. That matches a core part of the report, and the preview cleanup code means a disappearing-final-reply failure mode is still plausible. I do not have strong evidence for `implemented_on_main` or `cannot_reproduce`.
