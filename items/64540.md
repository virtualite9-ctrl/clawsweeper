## Summary

Keep this PR open. Current main still has the reported empty-heartbeat fallback behavior, and the PR is a plausible implementation candidate, but its one-file change would leave other heartbeat policy paths using the existing `every: "0m"` disable contract and could make summary/status behavior diverge from scheduling, system-prompt, bootstrap, and ACP spawn behavior.

## Best Possible Solution

Keep this PR open for maintainer review or contributor rework. If OpenClaw wants `heartbeat: {}` to mean disabled, add one shared empty-heartbeat predicate and apply it consistently across `src/infra/heartbeat-summary.ts`, `src/infra/heartbeat-runner.ts`, `src/agents/heartbeat-system-prompt.ts`, `src/agents/bootstrap-files.ts` through the shared prompt gate, and `src/agents/acp-spawn.ts`; then add regression tests for defaults and per-agent empty heartbeat configs, including inherited defaults, and update the public heartbeat/config docs. If maintainers want `every: "0m"` to remain the only disable contract, document that on #64293 and close or revise this PR instead of landing a partial behavior change.
