## Summary

Keep open. Current main still treats empty heartbeat objects as default-enabled, while this PR only changes the summary helper at its head and leaves scheduler and system-prompt policy paths aligned with the old behavior.

## Best Possible Solution

Keep this PR open for maintainer review or contributor rework. If OpenClaw should accept `heartbeat: {}` as an explicit disable signal for #64293, implement one shared empty-heartbeat predicate and apply it consistently across `src/infra/heartbeat-summary.ts`, `src/infra/heartbeat-runner.ts`, and `src/agents/heartbeat-system-prompt.ts`, then add regression tests for defaults and per-agent empty heartbeat configs with inherited defaults. If maintainers want `every: "0m"` to remain the only disable contract, document that decision and close the linked bug instead of merging this partial change.
