## Summary

Keep open. The PR is a focused implementation candidate for #64293, but current main still treats empty heartbeat objects as default-enabled, and the PR head only changes the summary helper surface. Scheduler selection, system-prompt gating, and related heartbeat policy paths need to be aligned before this can safely land.

## Best Possible Solution

Keep this PR open for maintainer review or contributor rework. If OpenClaw wants heartbeat: {} to mean disabled, add one shared empty-heartbeat predicate and apply it consistently across src/infra/heartbeat-summary.ts, src/infra/heartbeat-runner.ts, src/agents/heartbeat-system-prompt.ts, and src/agents/acp-spawn.ts, then add regression tests for defaults and per-agent empty heartbeat configs with and without inherited defaults, plus docs/help updates. If maintainers want every: "0m" to remain the only disable contract, document that explicitly on #64293 and close or revise this PR instead of landing a partial behavior change.
