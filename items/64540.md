## Summary

Keep this PR open. Current `main` still has the reported empty-heartbeat fallback behavior, and the branch is a plausible implementation candidate for #64293, but its one-file change would not align all heartbeat policy paths. Landing it as-is could make summary/status behavior diverge from scheduling, system-prompt guidance, bootstrap `HEARTBEAT.md` filtering, and ACP spawn behavior.

## Best Possible Solution

Keep this PR open for maintainer review or contributor rework. If OpenClaw wants `heartbeat: {}` to mean disabled, introduce one shared empty-heartbeat predicate and apply it consistently across heartbeat summary, runner selection/scheduling, system-prompt guidance, bootstrap prompt gating, and ACP spawn policy, then add regression tests and update the public heartbeat docs. If maintainers want `every: "0m"` to remain the only disable contract, close or revise the branch and document that outcome on #64293.
