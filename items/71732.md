## Summary

Keep #71732 open. Current main still lacks the requested generic plugin session-extension and host-routed patch-action SDK contract. The related #72082 PR is an open implementation candidate, but it has not landed on main, and the current source still only has adjacent core `sessions.patch`, plugin RPC, runtime helper, and post-write hook surfaces.

## Best Possible Solution

Keep #71732 open as a real core/plugin-SDK API gap until #72082 or a successor lands on main. The best implementation path is a generic host-owned session-extension contract on `OpenClawPluginApi`, typed gateway protocol support for plugin patch actions, schema validation before writes, explicit public projection for session rows/details, reset/delete/compaction/plugin-disable cleanup, stable error codes, and fixture-plugin contract tests. After that PR lands, close this as implemented with the merged SHA and release evidence.
