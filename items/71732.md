## Summary

Keep open. Current `main` does not implement the requested namespaced plugin session extension and patch-action API. The observable session patch path is still a fixed core protocol and handler, with an adjacent post-write `session:patch` hook and plugin-owned RPC registration, but no SDK contract for typed plugin session state that participates in native patch validation, public projection, broadcast, reset/delete/compaction cleanup, or disabled-plugin behavior.

## Best Possible Solution

Keep #71732 open for maintainer design and implementation. The best path is a focused OpenClaw core/plugin-SDK PR that adds a namespaced session-extension registry, gateway patch-action contract, schema validation, host-enforced scopes, public projection, broadcast semantics, reset/delete/compaction/disablement lifecycle handling, and fixture-plugin coverage. This belongs in OpenClaw core/plugin SDK because external plugins cannot add missing gateway protocol, persistence, projection, and lifecycle hooks themselves.
