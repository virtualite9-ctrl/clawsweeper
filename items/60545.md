## Summary

Keep open. Current main only partially supersedes this PR: fallback retries now preserve the original body, but the explicit session-id rollover cleanup, `openclaw agent --json` final-envelope writer path, and one-shot sandbox/browser-bridge teardown are not implemented on main. The PR discussion is also important evidence: bot reviews supported the runtime intent but flagged compile and concurrency problems that mean the submitted diff is not merge-ready as-is.

## Best Possible Solution

Keep the item open or replace it with a rebased maintainer patch. The best path is to retain main's newer fallback retry behavior, add explicit session-id rollover cleanup with tests that clear all session-scoped continuity fields, route the final `agent --json` envelope through `writeRuntimeJson`, and implement sandbox/browser-bridge teardown with ownership/refcounting or private-scope gating instead of unconditional scope-key disposal.
