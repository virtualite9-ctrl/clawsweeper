## Summary

Keep this issue open. Current main has the runtime guard that makes `channels.<id>.enabled=false` suppress bundled channel plugin auto-enable, and generic plugin/gateway docs cover the pieces separately. The specific BlueBubbles documentation requested here is still missing: the BlueBubbles page does not compare channel disablement with `plugins.entries.bluebubbles.enabled=false`, does not include the composed loopback/token example with both BlueBubbles layers disabled, and does not explain startup mutation/repair behavior in that BlueBubbles context.

## Best Possible Solution

Keep this open and add a focused section to `docs/channels/bluebubbles.md` explaining `channels.bluebubbles.enabled=false` versus `plugins.entries.bluebubbles.enabled=false`, with a JSON5 safe local example using `gateway.mode: "local"`, `gateway.bind: "loopback"`, `gateway.auth.mode: "token"`, `channels.bluebubbles.enabled: false`, and `plugins.entries.bluebubbles.enabled: false`. That section should briefly state what startup may persist, repair, or mutate, and link to the public plugin and gateway configuration docs.
