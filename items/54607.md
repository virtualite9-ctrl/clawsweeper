## Summary

Keep open. Current main has partial runtime and generic plugin guidance, but the requested BlueBubbles-specific docs gap remains: the BlueBubbles guide does not explain the difference between disabling the channel and disabling the plugin, does not include the safe loopback/token example, and does not answer the startup mutation/auto-enable questions in one place.

## Best Possible Solution

Add a BlueBubbles docs section, likely in `docs/channels/bluebubbles.md`, titled around disabling or safe local operation. It should compare `channels.bluebubbles.enabled=false` with `plugins.entries.bluebubbles.enabled=false`, include the requested loopback/token example with both BlueBubbles flags disabled, and state which startup/config writes can occur, including metadata and auto-enable behavior. Cross-link the public plugin and gateway configuration pages rather than relying on readers to assemble the answer from separate generic docs.
