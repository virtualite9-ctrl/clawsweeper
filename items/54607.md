## Summary

Keep this issue open. Current main has the runtime guard that makes `channels.<id>.enabled=false` block bundled channel plugin auto-enable, plus generic plugin and gateway docs. The requested BlueBubbles-specific documentation is still missing: the BlueBubbles page does not explain `channels.bluebubbles.enabled=false` versus `plugins.entries.bluebubbles.enabled=false`, does not include the composed safe loopback/token example with both BlueBubbles layers disabled, and does not describe startup mutation/repair behavior in that BlueBubbles context.

## Best Possible Solution

Keep this open and add a focused section to `docs/channels/bluebubbles.md` that explains `channels.bluebubbles.enabled=false` versus `plugins.entries.bluebubbles.enabled=false`, includes a JSON5 safe local example with `gateway.mode="local"`, `gateway.bind="loopback"`, `gateway.auth.mode="token"`, and both BlueBubbles disablement layers set to false, and briefly states what startup may persist, repair, or mutate with links to the public plugin and gateway configuration docs.
