## Summary

Current `main` already implements realtime Control UI transcript updates over the gateway WebSocket, including live `session.message` broadcasts and Control UI reload handling for both new messages and final chat events. The same path also carries live usage/token metadata, so the reported need to refresh the page to see replies or token counts is addressed on `main`.
