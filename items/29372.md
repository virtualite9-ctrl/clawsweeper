## Summary

Keep #29372 open. Current main still does not implement `agents.list[].subagents.workspace`; the per-agent config type, strict runtime schema, generated schema, docs, and current spawn path preserve the requester workspace for default/same-agent subagents. The related cross-agent workspace fix is shipped, but it covers explicit cross-agent `agentId` spawns and does not solve the requested default subagent workspace override.

## Best Possible Solution

Keep #29372 open for a core config/runtime implementation if maintainers want this isolation behavior. A complete fix should add and document `agents.list[].subagents.workspace`, update TypeScript types, zod/runtime/generated schemas, path normalization/help docs, and spawn workspace resolution, with focused tests for omitted `agentId`, explicit same-agent spawns, `requireAgentId`, cross-agent preservation, and bootstrap isolation.
