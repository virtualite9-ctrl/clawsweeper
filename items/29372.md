## Summary

Keep #29372 open. Current main still does not implement `agents.list[].subagents.workspace`: the per-agent config type and runtime/generated schemas omit the field, docs only describe model/thinking/allowlist/requireAgentId subagent defaults, and omitted-`agentId` / same-agent spawns still preserve the caller workspace. The related #40171/#40176 work fixed cross-agent target workspace inheritance, but it does not supersede this requested child-workspace override.

## Best Possible Solution

Keep #29372 open for a core config/runtime implementation if maintainers want this behavior. A complete solution would add a documented `agents.list[].subagents.workspace` field, decide whether `agents.defaults.subagents.workspace` is also needed, update TS types, zod/runtime/generated schemas, help/docs, path normalization, and native spawn workspace resolution for omitted `agentId` and explicit same-agent spawns, with tests covering `requireAgentId`, cross-agent preservation, schema acceptance/rejection, and bootstrap workspace isolation.
