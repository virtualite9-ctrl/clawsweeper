## Summary

Keep #29372 open. Current main still lacks an `agents.list[].subagents.workspace` config surface, and same-agent/default subagent spawns still preserve the caller's inherited workspace. The related #40171 cross-agent workspace bug is addressed, but that fix does not implement the requested default child-workspace override.

## Best Possible Solution

Keep #29372 open for maintainer/product judgment. A complete fix would add a documented `agents.list[].subagents.workspace` config field, decide whether `agents.defaults.subagents.workspace` is also needed, update TS types, zod/runtime/generated schemas, config help/docs, path normalization, and same-agent/default spawn workspace resolution without regressing the already-fixed cross-agent target workspace behavior. Add regression tests for omitted `agentId`, explicit same-agent `agentId`, strict schema acceptance/rejection, `requireAgentId`, and cross-agent workspaces.
