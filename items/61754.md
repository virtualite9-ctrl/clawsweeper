## Summary

Current `main` no longer falls back to `"AWS_PROFILE"` in either Bedrock config path, regression tests cover the instance-role case, and the latest shipped tag I could inspect (`v2026.4.23`) contains the corrected implementation, so the 2026.4.5 packaging miss is no longer an open `main` issue.

## Evidence

- **Current Bedrock discovery path no longer injects AWS_PROFILE:** `resolveBedrockConfigApiKey` returns `resolveAwsSdkEnvVarName(env)` directly, with an inline note that Bedrock should fall back to the AWS SDK default credential chain instead of persisting a fake marker.
