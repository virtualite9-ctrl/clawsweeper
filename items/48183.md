## Summary

Keep open. Current main still has the Feishu HTTP server cleanup pattern reported in #48183: `server.close()` is called without awaiting the close callback, `closeAllConnections()` is not used, and `httpServers` entries are deleted/cleared immediately. The linked PR #48588 targets this exact bug but is still open and not present on main; broader ResourceManager follow-ups (#48793, #49804) do not yet resolve the Feishu-specific leak.

## Best Possible Solution

Keep #48183 open until the Feishu cleanup fix lands on main. The narrow product/code path is to make Feishu HTTP server shutdown awaitable in both `stopFeishuMonitorState()` and `monitorWebhook()` abort cleanup, call `closeAllConnections()` where appropriate, clear `httpServers` only after close completion or a bounded timeout, update `stopFeishuMonitor()` callers/tests to await cleanup, and add regression coverage for targeted and global HTTP server cleanup. PR #48588 is the closest existing implementation candidate; broader ResourceManager work should remain a follow-up unless it lands first and covers this path.
