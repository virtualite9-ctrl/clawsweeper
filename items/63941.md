## Summary

Current `main` does not implement this PR's self-hosted Firecrawl support. The Firecrawl client on `main` still hard-requires `https:` and still allowlists only `api.firecrawl.dev`, and the tests on `main` still assert that `127.0.0.1` is rejected. The PR discussion also identified a second SSRF-guard blocker that the PR author addressed on the PR branch, which further supports keeping this open rather than closing it as already fixed or obsolete.

## Evidence

- **Current main still hardcodes the official host and requires HTTPS:** `resolveEndpoint()` on current `main` throws unless the base URL uses `https:` and the hostname is in `ALLOWED_FIRECRAWL_HOSTS`, which is still just `api.firecrawl.dev`. That means self-hosted LAN/private Firecrawl endpoints are still blocked on `main`.
