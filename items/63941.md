## Summary

Keep open. Current main still rejects the self-hosted Firecrawl workflow, and this unmerged PR is a focused implementation candidate for the linked bug. It should be reviewed against the overlapping #59666 work and the SSRF-sensitive endpoint behavior rather than closed by cleanup automation.

## Best Possible Solution

Keep this PR open for maintainer review. Land this PR or a maintainer-owned equivalent that permits admin-configured self-hosted Firecrawl endpoints, keeps public hosts on HTTPS and the strict guard, allows cleartext HTTP only for private/loopback targets, routes trusted private endpoints through the existing trusted web-tools guard, preserves the restored SecretRef coverage, adds endpoint/SSRF regression tests, updates docs/tools/firecrawl.md, then close linked bug #63877 as implemented and resolve or supersede #59666.
