## Summary

Keep #7424 open. Current main still lacks an ordinary per-agent model allowlist: it supports per-agent primary/fallback defaults and a global `agents.defaults.models` allowlist/catalog, plus separate plugin subagent `allowedModels`, but there is no `agents.list[]` model allowlist schema or consistent agent-scoped enforcement for `/models`, `/model`, UI model pickers, `sessions.patch`, or OpenAI-compatible model overrides. Related #9010 is closed as a duplicate of #7424, so #7424 remains the canonical tracker.

## Best Possible Solution

Keep #7424 open as the canonical feature request. Add an agent-scoped model allowlist, likely under `agents.list[].model.allowlist` or `agents.list[].models`, then enforce it anywhere models are listed or selected while preserving existing global `agents.defaults.models` catalog semantics and documenting the inheritance/intersection rule.
