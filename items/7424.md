## Summary

Keep #7424 open. Current main still has a global `agents.defaults.models` model catalog/allowlist and per-agent `model` primary/fallback overrides, but no ordinary per-agent model allowlist field and no fully agent-scoped model listing or selection enforcement. Related #9010 is already closed as a duplicate, so #7424 remains the canonical tracker.

## Best Possible Solution

Keep #7424 open as the canonical feature request. Add an agent-scoped model allowlist, likely under `agents.list[].model.allowlist` or `agents.list[].models`, define how it combines with `agents.defaults.models`, and enforce it wherever models are listed or selected: gateway `models.list`, `/model` and `sessions.patch`, OpenAI-compatible override paths, Control UI model pickers, config schema/types/generated docs, and regression tests for both visibility and rejection of disallowed models.
