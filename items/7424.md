## Summary

Current main still does not implement the requested per-agent model allowlist. It has global `agents.defaults.models` allowlist/catalog behavior, per-agent primary/fallback defaults, and plugin subagent `allowedModels`, but no ordinary `agents.list[]` model allowlist schema or agent-scoped model picker/list enforcement. The related #9010 is closed as a duplicate of #7424, so this remains the canonical tracker.

## Best Possible Solution

Keep #7424 open as the canonical feature request. Add an agent-scoped model allowlist, likely under `agents.list[].model.allowlist` or `agents.list[].models`, then enforce it wherever users can list or switch models while preserving the existing global `agents.defaults.models` catalog semantics.
