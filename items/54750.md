## Summary

Current main already treats memory as a plugin-scoped surface with an exclusive memory slot, documented memory-plugin APIs, and published external memory-plugin examples. This PR adds a large optional episodic-memory feature to bundled `memory-core`, but repo vision and plugin docs point that kind of optional capability toward external plugins/ClawHub rather than new bundled core/plugin surface.

## Risks / Open Questions

- The PR-linked `openclaw-episodic-memory` repo appears to be an external implementation/library, not a finished installable OpenClaw plugin yet, so packaging work would still be needed.
- If maintainers later decide episodic memory should become the recommended bundled/default memory path, closing this PR now may discard a useful prototype rather than a permanently out-of-scope idea.
