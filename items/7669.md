## Summary

Keep #7669 open. Current main still has no supported npm `dev` dist-tag publish path: the public npm release workflow only offers `beta`/`latest`, real publishes require release tags, the publish helper normalizes non-`latest` requests to `beta`, release-plan types only model stable/beta, and the live npm registry currently has no `dev` dist-tag. Docs and updater behavior still make dev git-first, with npm `dev` only optional when published.

## Best Possible Solution

Keep #7669 open until maintainers either implement a first-class main-snapshot npm `dev` publish path, including workflow trigger/input support, release-plan validation, publish-helper support, post-publish verification, and docs; or explicitly retire npm dev snapshots and update public release-channel, update, installer, and release docs so dev is documented as git-only.
