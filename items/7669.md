## Summary

Keep #7669 open. Current main still does not implement a supported npm `dev` dist-tag publish path: the public release workflow only accepts `beta`/`latest`, the publish helper and release-plan types only preserve `beta`/`latest`, release docs describe real publish as tag-based, and the live npm registry currently has no `dev` dist-tag.

## Best Possible Solution

Keep #7669 open until maintainers either implement a first-class main-snapshot npm `dev` publish path, including workflow trigger/input support, release-plan validation, publish-helper preservation of `dev`, post-publish verification, and docs; or explicitly retire npm dev snapshots and update public release-channel/update/release docs to make `dev` git-only.
