## Summary

Keep #7669 open. Current main still does not implement the requested npm `dev` dist-tag publish path: the release workflow exposes only `beta`/`latest`, real publishes require release tags rather than moving `main`, the publish helper and release-plan contract only model stable/beta, tests cover stable/beta only, and the docs still describe npm `dev` as optional/conditional while the actual `dev` update channel uses a git checkout.

## Best Possible Solution

Keep #7669 open until maintainers either implement a first-class npm `dev` snapshot channel for main builds, including workflow trigger/input support, release-plan validation, publish-helper support, tests, post-publish verification, and docs; or explicitly retire npm dev snapshots and update release-channel, updater, installer, and release docs so `dev` is documented as git-only or otherwise unsupported for npm.
