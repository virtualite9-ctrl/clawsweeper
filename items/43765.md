## Summary

Keep this PR open. Current main partially covers the heartbeat event replay problem with a safer consume-on-success path, but it does not implement the Feishu websocket health snapshot, the exec non-zero foreground failure semantics, or the first-visible diagnostics/design work proposed here. The PR is not merge-ready as-is because the provided review context includes blocking P0/P1 review findings and the branch mixes several unrelated runtime, docs, ACPX, and diagnostics changes.

## Best Possible Solution

Keep this PR open only as a source of candidate fixes, not as something to land wholesale. The best path is to ask for a current-main rebase or replacement split into focused PRs: one for Feishu websocket runtime snapshot/lifecycle health with tests, one for exec/process terminal status and non-zero exit semantics, and one separate diagnostics/design PR if first-visible observability is still desired. The heartbeat replay piece should be compared against the consume-on-success implementation already on main before carrying any drain/requeue code forward.
