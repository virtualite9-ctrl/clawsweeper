## Summary

Keep this PR open as candidate work, not as something ready to close or merge wholesale. Current main only covers the heartbeat replay slice; Feishu websocket health snapshots, the proposed exec failure semantics, and first-visible diagnostics are still not implemented, while the PR branch is stale, XL, and has unresolved module-load and supply-chain-sensitive changes.

## Best Possible Solution

Keep this PR open only as a source of candidate fixes. Ask for a current-main rebase or replacement split: drop or compare the heartbeat slice against main's consume-on-success implementation, land Feishu websocket health as a focused bundled plugin PR, handle exec terminal-status semantics as a separate product-reviewed PR, design first-visible diagnostics independently, and put ACPX dependency/command-launch changes through explicit maintainer and security review.

## Risks / Open Questions

- The PR is XL and mixes heartbeat, Feishu channel health, exec/process semantics, ACPX dependency and command launch behavior, ACP spawn metadata, diagnostics, and docs, which makes conservative review and rollback hard.
- The PR branch has concrete module-load blockers reported in review comments: duplicate Feishu logger declarations and a missing session-generation module.
- The ACPX portion adds executable third-party packages and changes process launch behavior; this is a supply-chain and command-execution surface that should be isolated and explicitly reviewed.
- Changing all non-zero foreground exec exits from completed to failed is a user-visible agent/tool semantics change and may affect workflows that intentionally use non-zero exit codes.
