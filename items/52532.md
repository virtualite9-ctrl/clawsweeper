## Close Comment

Closing as `clawhub` after a Codex review. On current `main`, OpenClaw’s stated scope is to keep optional capability out of core and ship it as plugins instead (`VISION.md:54-70`), with memory explicitly modeled as an exclusive plugin slot (`VISION.md:63-64`, `docs/tools/plugin.md:109-112`, `docs/plugins/manifest.md:707`). The public SDK and runtime already expose first-class seams for third-party memory plugins (`docs/plugins/sdk-overview.md:211-223`, `src/plugins/types.ts:2130-2160`, `src/plugins/memory-state.ts:143-151`). In this thread, NEXO was also reported as already available through external integrations, including a ClawHub skill and an npm memory plugin. Given that, this belongs in the external plugin/ClawHub ecosystem rather than OpenClaw core. If there is a concrete gap in the memory-plugin contract that blocks third-party memory plugins, please open a focused issue for that seam.
