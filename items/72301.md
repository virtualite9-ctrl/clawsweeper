## Summary

Keep open. Current main at 42db865673a05bd5fcf0a276496c29e266183df0 still forwards `temperature` through the Anthropic Messages transport for non-thinking requests, including `claude-opus-4-7`. The PR targets a real remaining path from #72123, but the submitted patch still needs cleanup and regression coverage before it is a safe maintainer close/merge candidate.

## Best Possible Solution

Keep the PR open or land an equivalent small fix directly on main: strip `temperature` for `claude-opus-4-7` and `claude-opus-4.7` in the Anthropic Messages transport option/payload path, preserve temperature forwarding for supported Anthropic models, and add focused regression tests proving both Opus 4.7 spellings omit temperature while a supported model still forwards it. Clean up the submitted formatting before merge.
