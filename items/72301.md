## Summary

Keep open. Current main at ff6044f4411d3e720940d60990e624bd7be29d46 still forwards `options.temperature` through the Anthropic Messages transport for non-thinking requests, including `claude-opus-4-7`. The PR targets a real remaining path from #72123, but should be cleaned up and covered with a regression test before landing.

## Best Possible Solution

Keep the PR or land an equivalent small fix directly on main: strip `temperature` for `claude-opus-4-7` and `claude-opus-4.7` in `resolveAnthropicTransportOptions()` and/or `buildAnthropicParams()`, preserve temperature forwarding for unaffected Anthropic models, and add Anthropic transport regression tests proving both Opus 4.7 spellings omit temperature while a supported model still forwards it.
