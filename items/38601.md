## Summary

Keep #38601 open. Current main still suppresses explicit `pty: true` whenever exec resolves to a sandbox runtime, and I found no warning/error branch, user-facing sandbox caveat, or regression test covering sandboxed `pty:true`; the related fix PR remains unmerged in the provided context.

## Best Possible Solution

Keep the issue open and land a narrow exec-tool fix: when `params.pty === true && sandbox`, emit a clear warning or intentionally reject before `usePty` is suppressed. Add a focused regression test for sandboxed `pty:true`, and update `docs/tools/exec.md`, the tool schema, and the exec tool description to state the sandbox PTY limitation.
