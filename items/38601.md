## Summary

Keep #38601 open. Current main still suppresses an explicit `pty:true` request whenever exec resolves to a sandbox runtime, and I found no warning/error path, user-facing sandbox caveat, or regression test covering that feedback. The directly related fix PR #39349 remains open/unmerged in the provided context, while #46392 was closed unmerged, so the issue is neither implemented nor superseded.

## Best Possible Solution

Keep #38601 open and land a narrow exec-tool fix. Maintainers should choose the intended contract: either pass `pty:true` through to sandbox backends that support it, or intentionally suppress it with a clear warning or rejection before `usePty` is set false. Add a focused regression test for sandboxed `pty:true`, and align `docs/tools/exec.md`, `src/agents/bash-tools.schemas.ts`, and `src/agents/bash-tools.descriptions.ts` so users and agents see the same behavior.
