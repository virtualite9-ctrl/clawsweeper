## Summary

Keep open. Current main still suppresses PTY when a sandbox runtime is selected, but I found no warning or error for an explicit pty:true request. The related fix PR #39349 is still open and #46392 was closed unmerged, so the remaining work is not implemented on main.

## Evidence

- **Current exec implementation still disables PTY under sandbox:** The exec tool passes pty: params.pty === true && !sandbox into the gateway approval path, and later computes const usePty = params.pty === true && !sandbox before calling runExecProcess. There is no adjacent warning branch for params.pty === true && sandbox.
  - file: [src/agents/bash-tools.exec.ts:1646](https://github.com/openclaw/openclaw/blob/81a41fe5bee350c6a035bfcc458eb8be380afe2c/src/agents/bash-tools.exec.ts#L1646)

  - sha: [81a41fe5bee3](https://github.com/openclaw/openclaw/commit/81a41fe5bee350c6a035bfcc458eb8be380afe2c)
- **Runtime only receives the already-suppressed PTY flag:** The sandbox runtime builds a child spawn spec and forwards opts.usePty to the backend/docker args. Since the caller computes usePty as false when sandbox is present, sandboxed pty:true does not reach this runtime as an enabled PTY request.
  - file: [src/agents/bash-tools.exec-runtime.ts:682](https://github.com/openclaw/openclaw/blob/81a41fe5bee350c6a035bfcc458eb8be380afe2c/src/agents/bash-tools.exec-runtime.ts#L682)
