## Summary

Keep #38601 open. Current main still disables explicit `pty:true` when exec resolves to a sandbox runtime, and I found no warning/error path, user-facing sandbox caveat, or regression test proving the requested feedback is implemented. The related fix PR #39349 is still open and unmerged, while #46392 was closed unmerged, so the issue is not shipped or superseded.

## Best Possible Solution

Keep #38601 open and land a narrow exec-tool fix. At minimum, when `params.pty === true` and the effective host is sandbox, emit a clear warning or reject intentionally before `usePty` is suppressed; if sandbox PTY is now supported by the backend contract, consider passing the flag through instead. Add a focused regression test for sandboxed `pty:true`, and update `docs/tools/exec.md`, `src/agents/bash-tools.schemas.ts`, and `src/agents/bash-tools.descriptions.ts` so agents and users see the same contract.
