## Summary

Keep #38601 open. Current main still suppresses an explicit `pty: true` request whenever exec resolves to a sandbox runtime, and I found no current warning/error path, schema caveat, user-facing docs caveat, or regression test for sandboxed `pty:true`. The related fix PR #39349 is still open/unmerged in the provided context, while #46392 was closed unmerged, so the issue is not implemented or superseded.

## Best Possible Solution

Keep the issue open and land a narrow exec-tool fix: when `params.pty === true && sandbox`, emit a clear warning or intentionally reject before `usePty` is suppressed. Add a focused regression test for sandboxed `pty:true`, and update the exec docs plus tool schema/help text to state the sandbox PTY limitation.
