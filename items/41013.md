## Summary

Keep open. I could not prove the PR is already implemented or obsolete on current `main`. Current user-message display still depends on `stripInboundMetadata` / `stripLeadingInboundMetadata`, which do not handle `<session-recap>` blocks, but the repo also no longer contains any `session-recap` emitters I could trace on `main`. The PR therefore might still address a real gap, yet it is not safe to close because the evidence for obsolescence is indirect and the PR has unresolved correctness concerns in review.
