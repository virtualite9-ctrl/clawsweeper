## Summary

Keep #38520 open. Current main has adjacent compaction safeguards, including silent memory flush, start/end user notices, post-compaction context reinjection, and observe-only hooks, but it still does not implement the requested compaction lifecycle: configurable pre-compaction warning thresholds, a visible structured handoff window/turn with timeout policy, or a bounded/audited deferral mechanism. The related issue context is partial or closed and does not provide a canonical open replacement for the remaining work.

## Best Possible Solution

Keep this open for maintainer product/API design. A complete implementation likely belongs in core compaction configuration and runtime event handling: validated warning/critical threshold config, defined channel/UI delivery for pre-compaction warnings, a structured pre-compaction handoff turn with timeout and prompt policy, and a bounded/audited deferral contract if maintainers want agents or plugins to pause imminent compaction. It should update config schema/types/help/docs, compaction runner/event paths, any SDK or hook contracts needed, and focused tests for warnings, handoff timeout behavior, and deferral limits.
