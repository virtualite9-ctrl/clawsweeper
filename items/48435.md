## Summary

Keep #48435 open. Current main does not expose Slack `unfurlLinks`/`unfurlMedia` config and does not pass Slack `unfurl_links`/`unfurl_media` through `chat.postMessage`; the existing Slack docs/config metadata list delivery knobs like chunking, media limits, and streaming, but not unfurl controls. The issue comment correctly frames this as a config expressiveness gap, and I did not find a shipped implementation or a canonical duplicate that covers this exact Slack outbound behavior.

## Best Possible Solution

Implement this in the bundled Slack plugin as a typed per-account/base config option, likely `channels.slack.unfurlLinks` and `channels.slack.unfurlMedia` with account inheritance, schema/UI hints/docs, and focused send tests proving `chat.postMessage` receives `unfurl_links`/`unfurl_media` for normal text, block fallback, and chunked follow-up posts. Decide separately whether file-upload `initial_comment` needs equivalent behavior, since Slack file upload APIs do not share the same `chat.postMessage` payload shape.
