## Summary

Keep open. Current main still exposes `openclaw infer model run` as a text-only command: the registered flags omit `--file`, `runModelRun` only accepts prompt/model/transport, and neither the local agent path nor gateway path passes images or attachments. Existing `infer image describe`, `audio transcribe`, and `video describe` are separate media-understanding workflows, not repeatable file input plus arbitrary prompt through `model.run`. The related #69030 raw-mode issue is open and related, but it does not supersede this multimodal `--file` request.

## Best Possible Solution

Keep #63700 open and implement the feature in the core infer/capability CLI: add repeatable `--file` to `model.run` metadata and commander parsing, resolve MIME and base64 content safely, pass image files through the existing local `agentCommand` image path and gateway `attachments` path, define the supported audio behavior explicitly, and add focused CLI tests plus docs for local/gateway, multi-image, unsupported MIME, and interaction with any future raw-mode work.
