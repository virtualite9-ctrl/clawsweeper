## Summary

Keep open. Current main still does not implement repeatable file attachments for `openclaw infer model run`; the command remains text-only, and both local and gateway execution pass only the prompt. Dedicated image/audio/video infer commands partially overlap, but they do not satisfy the requested multimodal model-run workflow.

## Best Possible Solution

Keep this issue open and implement the missing core CLI capability. Add repeatable `--file` handling to `src/cli/capability-cli.ts`, MIME-detect and validate supported media, pass image files through local `agentCommand` `images`/`imageOrder` and gateway `attachments`, and document the final audio semantics. Add focused CLI tests for single image, multiple images, unsupported media, and gateway/local parameter forwarding. Coordinate expectations with #69030 because raw mode and multimodal attachments both affect `infer model run`, but keep this as the canonical tracker for file attachments.
