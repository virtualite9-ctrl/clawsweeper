## Summary

Keep this issue open. Current main still does not provide repeatable `--file` input on `openclaw infer model run`: metadata and command registration are prompt-only, local execution sends only a text message through the lean completion path, and gateway execution calls `agent` without `attachments`. Existing `image describe` and `audio transcribe` commands are separate workflows and do not satisfy the requested custom prompt plus model-run file attachments. The related #69030 raw-mode concern is adjacent; current main has moved local model run to a lean completion path, but that does not implement multimodal file input.

## Best Possible Solution

Keep the issue open and implement the core CLI capability. Add repeatable `--file` to `model.run` metadata and command registration, read and MIME-validate supported files, forward image files through the lean local model-run path and gateway `attachments`, add focused CLI tests and docs updates, and separately define the audio representation before claiming `wav` or `mp3` support.
