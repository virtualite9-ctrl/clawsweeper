## Summary

Keep this issue open. Current main still has `openclaw infer model run` as a prompt-only model command: the metadata and command registration do not expose repeatable `--file`, and execution forwards only the prompt string to local and gateway agent paths. Separate `image describe` and `audio transcribe` commands exist, but they do not provide the requested custom prompt plus file attachments on `model run`. #69030 is adjacent raw-mode work, not a duplicate of the missing multimodal file-input surface.

## Best Possible Solution

Keep this issue open and implement the core CLI capability. Add repeatable `--file` handling to `src/cli/capability-cli.ts` metadata and `infer model run` registration, read and MIME-validate supported files, forward image files to local `agentCommand` through `images`/`imageOrder`, forward gateway runs through `attachments`, define the audio representation before claiming audio support, and update `docs/cli/infer.md` plus focused CLI tests. Coordinate with #69030 because raw mode and multimodal attachments both affect `infer model run`, but do not treat it as a duplicate.
