## Summary

Keep this issue open. Current main still rewrites Discord outbound `@Name` text only when the account-scoped in-memory directory cache already contains that handle; unknown handles are intentionally preserved as plaintext. I did not find a deterministic config-backed mention alias map, inbound-message cache seeding, or a merged/superseding related item that solves the reported multi-agent Discord handoff problem.

## Best Possible Solution

Keep this issue open and implement the fix in the Discord plugin: add an account-scoped deterministic mention alias source such as `channels.discord.mentionAliases` or `channels.discord.accounts.<id>.mentionAliases`, and/or seed the existing directory cache from trusted inbound Discord author/member metadata. Update config schema/docs and add regression tests for configured aliases, account isolation, unknown-name preservation, reserved mentions, code spans, and normal, webhook, and structured outbound send paths.

## Risks / Open Questions

- A fix must remain account-scoped, and possibly guild-aware, because Discord display names and usernames can collide across accounts and guilds.
- Outbound rewriting creates real Discord notifications, so configured aliases or trusted metadata should continue preserving `@everyone`, `@here`, unknown names, and markdown code spans unless explicitly resolved.
- Adjacent Discord mention work, such as Unicode handle parsing, does not replace the deterministic alias/cache-seeding request here.
