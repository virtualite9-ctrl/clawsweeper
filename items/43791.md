## Summary

Keep this PR open. Current main still lacks the PR's embedded-run buffer for preserving full tool-result ReplyPayload objects in final returned payloads. Main has a narrower orphaned tool-media fallback, but that only carries media/audio fields and does not cover text, channelData, delivery, presentation, interactive metadata, spokenText, or other ReplyPayload fields. The PR discussion also shows useful follow-up commits addressing earlier review gaps, so the branch remains a plausible implementation candidate rather than obsolete cleanup.

## Best Possible Solution

Keep this PR open for contributor or maintainer follow-up. The best path is to rebase it onto current main, preserve full ReplyPayload fields only for non-streaming callers, keep the compaction retry reset, settle timeout behavior with regression coverage, reconcile the existing orphaned media fallback, widen or document the embedded run result payload contract, and fix changelog attribution before merge.
