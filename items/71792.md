## Summary

Keep this PR open. Current main still rolls stale daily/idle sessions into a new session and emits plugin lifecycle hooks, but it does not emit the internal command:reset event that bundled session-memory listens for. The PR remains an active, focused fix with daily, idle, and fresh-session coverage, and I did not find a security or supply-chain reason to close it.

## Best Possible Solution

Keep this PR open and review or land it as the focused fix for automatic daily/idle rollover memory persistence. The best implementation path is to emit the existing internal command:reset event for non-command stale rollovers with commandSource values auto:daily and auto:idle, while preserving the current session-memory command-event contract unless maintainers choose a broader lifecycle hook redesign.
