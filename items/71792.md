## Summary

Keep PR #71792 open. Current main still performs daily/idle stale rollover and emits plugin lifecycle hooks, but it does not emit the internal `command:reset` hook that the bundled `session-memory` handler listens for. The PR is active, focused, and not superseded by a shipped main implementation.

## Best Possible Solution

Keep #71792 open and review it as the focused fix for automatic daily/idle rollover memory persistence. The likely best path is to emit the existing internal `command:reset` event for non-command stale rollovers with `commandSource` values such as `auto:daily` and `auto:idle`, while preserving the current explicit `/new` and `/reset` hook path and adding/keeping regression tests for daily, idle, and fresh-session no-op behavior.
