## Summary

Keep #71792 open. Current main still creates new sessions for stale daily/idle rollovers and emits plugin session lifecycle hooks, but it does not emit the internal `command:reset` event that the bundled `session-memory` hook listens to. The PR is focused, active, and addresses an open bug path also described by related #11960 and broader #15828; the older related #43533 was closed unmerged, so there is no strong close reason.

## Best Possible Solution

Keep #71792 open and review it as the focused fix for automatic daily/idle rollover memory persistence. The best path is likely to emit the existing internal `command:reset` event for stale non-command rollovers with explicit `commandSource` values, preserving the current `session-memory` command-event contract unless maintainers choose a broader lifecycle-hook redesign.
