## Summary

Keep open. Current main still maps `minimal` and `low` thinking to `output_config.effort: "low"` for adaptive Claude Opus 4.7 models, and the existing tests only cover `xhigh`/Opus 4.6 behavior. The PR is a focused fix for a concrete Copilot `400 invalid_reasoning_effort` report in #70322, with targeted regression coverage described in the PR context.

## Best Possible Solution

Keep the PR open for maintainer review and either merge this narrow clamp with its regression tests or implement the same compatibility rule through a provider/model reasoning-effort map for the Anthropic Messages path. #70322 should remain the canonical bug report until a fix lands on main.
