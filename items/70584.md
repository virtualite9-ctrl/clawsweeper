## Summary

Keep open. Current main still lacks the requested low/minimal-to-medium clamp in the adaptive Anthropic Messages transport, and this PR is a focused implementation candidate for same-author open bug #70322. Copilot's built-in model defaults now set reasoning:false, which partially reduces default reachability, but the shared adaptive path and Copilot wrapper still do not enforce the reported compatibility when reasoning is enabled.

## Best Possible Solution

Keep this PR open for maintainer review. The likely path is to land or revise a focused compatibility fix so GitHub Copilot Claude Opus 4.7 cannot send unsupported low/minimal adaptive effort, with regression tests for low, minimal, and an unaffected adaptive Claude model. If maintainers do not want a global Opus 4.7 rule in shared Anthropic transport, revise toward a provider-aware Copilot/Anthropic compatibility seam instead.
