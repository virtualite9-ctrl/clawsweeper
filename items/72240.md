## Summary

Keep #72240 open. Current main has some internal supervisor metadata and diagnostic events, but the user-facing exec result still drops key SIGKILL diagnostics such as exitSignal, failureKind/termination reason, run/log pointer, timeout budget, and supervisor-vs-external-kill cause. The linked issues are adjacent and still open, but none clearly supersedes this macOS user-facing diagnostic gap.

## Best Possible Solution

Keep the issue open and implement structured exec failure diagnostics in the core exec/supervisor path. The best fix would surface exitSignal, failureKind/terminationReason, timeout limit, duration, session/run id, output tail/truncation state, and whether OpenClaw initiated cancellation when knowable; add tests for timeout SIGKILL, external SIGKILL, foreground failures, and background process poll/log; and consider docs or guardrails for recursive scans over large OpenClaw state/log directories. Coordinate with #66399 and #69242 without treating them as full duplicates.
