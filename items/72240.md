## Summary

Keep #72240 open. Current main has some exec diagnostics and timeout classification, but the user-facing exec failure path still lacks the requested SIGKILL cause metadata, run/log pointer, RSS/process-tree details, and structured exit-signal fields. The related reports are adjacent and still open, not clear superseders for this macOS diagnostic gap.

## Best Possible Solution

Keep this open as a core exec/supervisor diagnostics bug. The best fix is to extend the supervisor/exec outcome contract and foreground/background tool results with structured termination metadata: known initiator when OpenClaw issued the kill, timeout limit and elapsed runtime, exitSignal/failureKind/termination reason, retained stdout/stderr tail, run/session id or diagnostics pointer, and resource/process-tree facts when available. Add focused regression tests for timeout SIGKILL, manual/tool-call cancellation, and external signal exits, and consider guardrails for broad scans of large OpenClaw state/log directories.
