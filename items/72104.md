## Summary

Current main already implements the requested safe sequencing for OpenClaw-managed updates: `update.run` awaits the package update result, the global package install helper awaits the package-manager process and verifies the installed dist inventory, and SIGUSR1 is scheduled only after a successful update. The CLI package-update path also verifies that a restarted managed gateway reports the expected updated version, covering the mixed-version restart failure shape reported for 2026.4.23 -> 2026.4.24.

## Best Possible Solution

Close #72104 as implemented on current main. Keep the shipped/sequenced updater behavior and post-restart version verification; use #44881 or a new narrowly-scoped issue for externally supervised raw package-manager upgrade flows that bypass OpenClaw's updater.
