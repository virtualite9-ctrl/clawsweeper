## Summary

Keep open. Current main still drops Discord upload MIME types and still sends WebP/non-GIF images through the shared optimizer, so the central bug is not implemented. This PR is also paired with the same author's still-open bug #41256 and has useful but unresolved review feedback, so it needs maintainer review or a deliberate superseding fix rather than cleanup closure.

## Best Possible Solution

Keep this PR open while maintainers decide the canonical fix for #41256. The best path is to propagate verified media.contentType into all Discord upload Blob paths, add focused regression tests for MIME preservation, and implement WebP/animated preservation with a narrowly scoped media contract that does not regress shared WhatsApp/media compression behavior.
