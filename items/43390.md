## Summary

Keep this PR open. Current main still has the reported module-level `storeLocks` Map growth path: `locked()` stores one entry per `state.deps.storePath` and never deletes it after the queued operation resolves. The PR is a focused cron-service fix with targeted tests, and the related #43386 refactor was closed unmerged rather than shipping an equivalent cleanup.

## Best Possible Solution

Keep this PR open for maintainer review, rebase if needed, and land a focused cleanup that deletes the `storeLocks` entry when the completing promise is still the current per-store lock. Preserve or adjust the targeted regression coverage so it proves both post-completion pruning and queued-operation retention.
