## Summary

Keep this issue open. Current main still time-bounds only initial MCP connection setup, while MCP catalog listing, MCP tool execution, and several plugin hook phases can still await indefinitely or fail closed on the reply path. The existing related Chrome MCP diagnostic issue was narrower and does not supersede this broader resilience request.

## Best Possible Solution

Keep this issue open and implement a core resilience layer for external MCP/plugin work: short configurable catalog and tool-call budgets, repeated-failure cooldown/circuit state, degraded empty-tool materialization for optional unhealthy MCP capability, status/log diagnostics that distinguish channel health from external capability health, and regression tests for slow listTools, slow callTool, repeated failure cooldown, and blocking plugin hook phases.
