## Summary

Keep this PR open. Current main still does not implement `session.reset.mode = "off"`; the runtime reset policy, exported config type, zod validation, generated schema/help, and existing tests all remain daily/idle only. The PR is a plausible implementation candidate but still needs normal maintainer review and cleanup before merge.

## Best Possible Solution

Keep this PR open and review it as the implementation candidate. The best path is a focused patch that adds `session.reset.mode: "off"` consistently across exported types, zod validation, generated schema/help/docs, and `evaluateSessionFreshness`; adds a unit test proving stale daily/idle timestamps still return fresh when mode is off; regenerates `docs/.generated/config-baseline.sha256`; and removes or splits unrelated runtime-logger cleanup.
