## Summary

Keep this PR open. Current main still does not support `session.reset.mode = "off"`, and the PR is a plausible implementation candidate for a core session lifecycle knob, but it needs focused cleanup before maintainer merge.

## Best Possible Solution

Keep this PR open as the implementation candidate. The best maintainer path is a focused update against current main that adds `session.reset.mode: "off"` consistently across exported types, Zod validation, generated schema/help/docs, and `evaluateSessionFreshness`; adds a direct unit test proving stale daily/idle timestamps remain fresh when mode is off; regenerates `docs/.generated/config-baseline.sha256`; updates public session/config docs; and removes or splits the unrelated SDK runtime-logger cleanup.
