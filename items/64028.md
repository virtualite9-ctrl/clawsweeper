## Summary

Keep this PR open. Current main still does not support `session.reset.mode = "off"`, and the PR is a plausible core implementation candidate, but it needs cleanup before maintainer merge: a direct regression test, config baseline proof, public docs, and removal or splitting of the unrelated SDK return-type cleanup.

## Best Possible Solution

Keep this PR open as the implementation candidate. The best maintainer path is a focused update against current main that adds `session.reset.mode: "off"` consistently across exported types, Zod validation, generated schema/help/docs, and `evaluateSessionFreshness`; adds a direct unit test proving stale daily/idle timestamps stay fresh when mode is off; regenerates and commits `docs/.generated/config-baseline.sha256`; updates the public session/config docs; and removes or splits the unrelated SDK runtime-logger cleanup.
