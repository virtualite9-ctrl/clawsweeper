## Summary

Keep #19859 open. Current main still hardcodes WebChat/Control UI plain Enter to send and Shift+Enter for newline, and neither the browser UI, native Swift composer, gateway config schema/types, nor WebChat docs expose the requested send-on-enter preference or Ctrl/Cmd+Enter send mode.

## Best Possible Solution

Keep the issue open for a focused WebChat/Control UI enhancement. A good implementation should decide the product surface, then plumb a send-on-enter preference through the browser composer and native Swift composer, support Ctrl/Cmd+Enter to send when plain Enter inserts a newline, and update config schema/types, docs, UI copy, and focused composer tests.
