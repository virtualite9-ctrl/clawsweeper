## Summary

Keep #19859 open. Current main still hard-codes Enter/Return-to-send behavior in both browser WebChat and native macOS WebChat, and there is no persisted UI setting, gateway config schema field, or public WebChat docs entry for send-shortcut behavior. #8147 is an overlapping open issue, but the latest issue discussion explicitly kept #19859 open unless maintainers choose to consolidate it.

## Best Possible Solution

Keep #19859 open unless maintainers explicitly consolidate it into #8147. A complete fix should add a typed WebChat send-shortcut preference, wire it through the browser Control UI/WebChat composer and native Swift WebChat composer, support Ctrl/Cmd+Enter to send when plain Enter inserts a newline, and update settings UI, config schema/types/help, public WebChat docs, and focused composer tests.
