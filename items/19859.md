## Summary

Current main still does not implement #19859. The browser Control UI/WebChat composer sends on unmodified Enter, the persisted UI settings and gateway webchat config/schema expose no send-on-enter or send-shortcut preference, and the native macOS WebChat composer still documents and implements fixed Return-to-send behavior. Related #8147 overlaps but remains open; the latest #19859 discussion explicitly kept this broader request open pending maintainer consolidation.

## Best Possible Solution

Keep #19859 open unless maintainers explicitly consolidate it into #8147. A complete fix should choose the product surface, add a typed chat send-shortcut/send-on-enter preference, wire it through the browser Control UI/WebChat composer and native Swift WebChat composer, support Ctrl/Cmd+Enter to send when plain Enter inserts a newline, and update config schema/types/help, settings UI, public docs, and focused composer tests. The fuquyama comment's Ctrl/Cmd+Enter suggestion should be considered because Shift+Enter is already the newline shortcut.
