## Summary

Keep open. Current `main` still has a plausible bug path matching the report: `getLogger()` can rebuild the base logger when `logging.file` changes, but subsystem loggers cache a child logger once and keep writing through that stale child. The gateway and many long-lived services use top-level subsystem loggers, and there is no shipped fix on `main` relative to the latest release.

## Evidence

- **Base logger can change, but only at the root logger layer:** `getLogger()` recalculates settings and rebuilds the cached root logger when the resolved file path changes.
  - file: [src/logging/logger.ts:237](https://github.com/openclaw/openclaw/blob/8b76392e3e7934b1cdd6d461148263fd56141ec8/src/logging/logger.ts#L237)
  - command: `nl -ba src/logging/logger.ts | sed -n '235,245p'`
  - sha: [8b76392e3e79](https://github.com/openclaw/openclaw/commit/8b76392e3e7934b1cdd6d461148263fd56141ec8)
- **Subsystem loggers cache a child logger permanently:** `createSubsystemLogger()` stores `fileLogger` in a closure and only calls `getChildLogger(...)` the first time. After that, file writes keep using the original child logger even if the root logger was rebuilt for a new `logging.file` path.
  - file: [src/logging/subsystem.ts:325](https://github.com/openclaw/openclaw/blob/8b76392e3e7934b1cdd6d461148263fd56141ec8/src/logging/subsystem.ts#L325)
  - command: `nl -ba src/logging/subsystem.ts | sed -n '325,353p'`
  - sha: [8b76392e3e79](https://github.com/openclaw/openclaw/commit/8b76392e3e7934b1cdd6d461148263fd56141ec8)
- **File writes go through the cached child logger:** `logToFile()` invokes methods directly on the cached `fileLogger`, so once that child is bound to the old transport/path there is no refresh path inside `createSubsystemLogger()`.
  - file: [src/logging/subsystem.ts:302](https://github.com/openclaw/openclaw/blob/8b76392e3e7934b1cdd6d461148263fd56141ec8/src/logging/subsystem.ts#L302)

- **Gateway uses long-lived subsystem loggers:** The gateway creates a module-level subsystem logger (`const log = createSubsystemLogger("gateway")`), which is exactly the kind of long-lived logger that would retain a stale child logger across config timing differences.
  - file: [src/gateway/server.impl.ts:113](https://github.com/openclaw/openclaw/blob/8b76392e3e7934b1cdd6d461148263fd56141ec8/src/gateway/server.impl.ts#L113)
