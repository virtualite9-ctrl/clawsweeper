## Summary

Keep #69544 open. Current main still derives mention patterns from both `identity.name` and `identity.emoji`, and Matrix feeds those regexes into room mention gating. The existing per-agent `groupChat.mentionPatterns` override is real and documented, but there is no Matrix-level or per-agent boolean that disables only emoji-derived mentions while preserving name-derived defaults.

## Best Possible Solution

Add a backward-compatible config contract for this behavior, preferably a Matrix channel/account default plus optional per-agent override if maintainers want both scopes. The implementation should keep the current default true, update config types/schema/help/generated docs, and add focused tests proving Matrix room messages containing only the agent emoji do not trigger when the new option is disabled while name/native Matrix mentions still work.
