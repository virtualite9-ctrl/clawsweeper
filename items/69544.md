## Summary

Keep #69544 open. Current main still derives the agent identity emoji into generic mention patterns and Matrix consumes those regexes for room mention detection. I found no Matrix channel, global groupChat, or per-agent groupChat config switch that disables only emoji-derived mentions; the documented workaround remains overriding mentionPatterns yourself.

## Best Possible Solution

Add a backwards-compatible config path that preserves current behavior by default but lets Matrix users suppress identity.emoji-derived mention regexes, ideally with a Matrix channel default plus a per-agent override. The implementation should update shared mention-pattern derivation or let Matrix request filtered derived patterns, then align Matrix schema/types, generated config metadata/help, docs, and focused Matrix mention tests covering emoji present in normal room text.
