## Summary

Keep #69544 open. Current main still auto-derives mention regexes from both `identity.name` and `identity.emoji`, Matrix uses those regexes for room mention gating, and the Matrix/global/per-agent schemas do not expose an emoji-mention toggle. The existing explicit `mentionPatterns` override can work around the false positive, but the requested Matrix or per-agent switch is not implemented.

## Best Possible Solution

Keep the issue open and implement a narrow, backwards-compatible mention policy: add a documented Matrix channel default and/or per-agent `groupChat` option that controls whether derived `identity.emoji` participates in fallback mention patterns, with tests covering Matrix shared-room false positives and the existing explicit `mentionPatterns` override behavior.
