## Risks / Open Questions

- The PR changes model-facing system prompt content; omitted skill names may disclose local or project skill metadata to remote model providers unless maintainers explicitly accept or gate that behavior.
- PR head escapes XML markup but still renders skill names without control-character normalization; because names come from frontmatter or directory basename without a slug whitelist, newline/control-character prompt injection remains a concrete review concern.
- Escaped omitted names containing characters such as `&` or `<` may not be directly usable with the advertised exact-name `openclaw skills info` lookup, which currently matches raw name or skillKey.
