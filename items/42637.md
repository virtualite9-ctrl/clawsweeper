## Summary

Current main still has the omitted-skill discoverability gap: truncation emits only the generic count warning and no omitted skill names or `openclaw skills info` recovery guidance. This open contributor PR is still the implementation candidate, not cleanup to close. It should stay open for maintainer review, especially because it changes model-facing prompt content and the discussion includes prompt-budget and safe-name-rendering concerns.

## Best Possible Solution

Keep this PR open. The best path is to review the current contributor patch, require tests for omitted-name discoverability under both count and char truncation, verify `maxSkillsPromptChars` is enforced on the final emitted prompt, and ensure omitted names are rendered safely while still remaining usable with `openclaw skills info`. If those checks pass, land the PR rather than closing it as stale or already implemented.
