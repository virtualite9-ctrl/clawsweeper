## Summary

Keep #20321 open. Current main still does not implement Telegram `groupPolicy: "members"`: the shared type/schema, Telegram config docs, runtime policy evaluation, and tests only cover `open`, `disabled`, and `allowlist`. The issue also has a fresh maintainer comment explicitly saying to keep it open, and the linked chat-only-trigger plugin is useful prior art/workaround rather than the requested all-members Telegram allowlist policy.

## Best Possible Solution

Keep #20321 open for explicit maintainer/product handling. The likely implementation belongs in the bundled Telegram plugin plus the shared config contract: add `groupPolicy: "members"`, define all-member verification semantics, wire Telegram Bot API membership/count checks with caching and invalidation, update schema/types/docs/security guidance, and add focused Telegram runtime tests. Treat the closed PRs and `openclaw-chat-only-trigger` as useful prior art, not as a close basis.
