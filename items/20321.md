## Summary

Keep #20321 open. Current main at 406ae72fd2789c78053f6b94006ec8f488bc1c00 still does not implement Telegram `groupPolicy: "members"`: the public config type, strict schema, generated metadata/help, Telegram runtime, docs, and tests still model only `open`, `disabled`, and `allowlist`. The provided discussion and related PR context are useful prior art, including the chat-only-trigger plugin workaround, but the closed unmerged PRs and plugin workaround do not satisfy the requested bundled all-members Telegram policy.

## Best Possible Solution

Keep #20321 open for explicit bundled Telegram plugin and shared config-contract work. A complete fix should add `members` to exported `GroupPolicy`, strict schema, generated metadata/help, docs, and examples; implement fail-closed Telegram all-members verification in the inbound and native-command authorization paths with bounded `getChatMemberCount`/`getChatMember` use, caching, and join/leave invalidation; update security audit behavior; and add focused config, runtime, docs, and Telegram tests. Treat #12045, #28351, #28392, #33551, and `openclaw-chat-only-trigger` as prior art rather than close evidence.
