## Summary

Keep #20321 open. Current main at 7e13f3f5140168e31027b39a9855ade3fadf24d5 still does not implement Telegram `groupPolicy: "members"`: the shared config type/schema, Telegram runtime authorization, tests, and docs only cover `open`, `disabled`, and `allowlist`. The closed PR attempts are useful prior art, and constansino's `openclaw-chat-only-trigger` plugin is a related workaround, but it does not satisfy the requested all-current-members Telegram allowlist policy.

## Best Possible Solution

Keep #20321 open for explicit bundled Telegram plugin and shared config-contract work. The likely path is to add `groupPolicy: "members"` to the shared type/schema and Telegram docs, define fail-closed all-member verification semantics, implement Telegram Bot API membership/count checks with caching and join/leave invalidation in the bundled Telegram plugin, update security/doctor guidance, and add focused runtime tests. Treat the closed PRs and `openclaw-chat-only-trigger` as useful prior art, not as a close basis.
