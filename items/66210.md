## Summary

Keep open. The M365/Outlook part of #66210 overlaps the ClawHub/plugin-scope direction and the related #66208 cleanup, but this PR also exposes gaps in current main for Teams-specific thread action/approval behavior. Current main has generic plugin tools, hooks, HTTP routes, Task Flow, and plugin approvals, yet Microsoft Teams does not currently expose the PR's Teams interactive dispatch, thread action helper, or thread-list/send-thread surfaces through a documented plugin API. Because I cannot confirm the requested Teams thread copilot approval flow is fully buildable outside core with today’s public APIs, this should not be auto-closed as `clawhub`. It also should not be treated as implemented on main: current main has no `extensions/m365`, no `action-approval-runtime` subpath, no Teams `thread-list` action, and no Teams thread approval implementation.

## Best Possible Solution

Keep #66210 open for maintainer judgment, but ask for a split. The M365/Outlook service integration should move to a ClawHub/npm plugin or separate product discussion. Any core-worthy work should be narrowed to additive, documented Teams/plugin SDK seams such as Teams plugin interactive dispatch for Adaptive Cards, public thread list/send-thread capability, and focused tests/docs. If maintainers decide those seams are not desired, close after a maintainer-owned ClawHub/plugin-path comment rather than automated cleanup.
