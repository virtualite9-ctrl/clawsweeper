## Summary

Keep open. Current main still uses chokidar glob targets for `SKILL.md` files and only handles `add`, `change`, and `unlink`, so the PR’s reported macOS `unlinkDir` deletion gap is not implemented on main. The PR is still useful, but it should be narrowed or revised before merge because the provided review context flags widened watch scope and the branch includes unrelated QA/script/plugin-index changes.

## Best Possible Solution

Keep the item open and ask for a focused skills-watcher fix: preserve the current intent of watching only skill manifest paths where possible, add explicit handling for `unlinkDir` cases that macOS chokidar 5 emits when a skill folder or watched root is removed, and include regression coverage for child skill directory deletion and watched-root deletion. Unrelated QA tooling, timeout, uninstall import, and plugin-index changes should be split out or dropped from this PR.
