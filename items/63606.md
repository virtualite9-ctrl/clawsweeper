## Summary

Keep open. Current main has a different Bedrock application-inference-profile prompt-cache path that prevents no-cache wrapping and injects cache points after a cached GetInferenceProfile lookup, so part of the PR’s original user problem is already addressed. But the PR also carries still-relevant model-name/config plumbing for cache-retention and cache-ttl eligibility paths; current main still does not pass or accept model names there, and cache-ttl eligibility for opaque Bedrock application profile IDs still falls back to `isAnthropicBedrockModel(modelId)` only. This is not obsolete enough for conservative cleanup.

## Best Possible Solution

Keep the PR open for maintainer review or request a narrower update: rebase onto current main, drop duplicate no-cache/cache-point injection work that main already solved, and preserve only the still-useful model-name/config fallback or provider-owned cache-ttl eligibility path with focused tests for opaque Bedrock application inference profile IDs.
