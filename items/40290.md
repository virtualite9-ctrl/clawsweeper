## Summary

Keep this PR open. Current main still strips credential-shaped keys from configured sandbox Docker env, so the central sandbox env problem is not implemented or obsolete. The PR has a plausible core fix, but it should not land as-is because the bundled skill-filter change bypasses existing skill eligibility safeguards.

## Best Possible Solution

Keep the PR open for author or maintainer follow-up. Narrow it to the sandbox Docker env behavior: add `forceAllowKeys` for keys explicitly present in configured `sandbox.docker.env`, add sanitizer/create-args regression tests for allowed configured credentials and still-blocked unconfigured credentials, and remove or redesign the skill-filter eligibility bypass so `enabled: false`, bundled allowlist policy, and runtime eligibility continue to win.
