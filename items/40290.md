## Summary

Keep open. Current main still blocks credential-shaped keys supplied through configured sandbox Docker env, so the bug this PR targets remains real. The env sanitizer direction is plausible, but the latest PR head still mixes in a skill eligibility bypass and lacks regression tests, so it should be revised rather than closed or merged as-is.

## Best Possible Solution

Keep this PR open for a focused revision. Narrow it to the Docker sandbox env behavior: add a `forceAllowKeys`-style sanitizer option, have `buildSandboxCreateArgs` force-allow only keys explicitly present in configured `sandbox.docker.env`, preserve value validation, add sanitizer and create-args regression tests, and drop or redesign the skill-filter bypass so `shouldIncludeSkill` remains authoritative.
