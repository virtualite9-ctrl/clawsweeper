## Summary

Current main still mutates user-configured heartbeat prompts that mention HEARTBEAT.md: it resolves the custom prompt, then unconditionally passes the resulting text through `appendHeartbeatWorkspacePathHint`. The docs still promise custom heartbeat prompt bodies are sent verbatim, and the provided GitHub context shows PR #40756 is an open closing-syntax implementation candidate rather than a merged fix.

## Best Possible Solution

Keep this issue open and resolve it through PR #40756 or an equivalent maintainer patch. The fix should distinguish user-authored heartbeat prompts from built-in/internal heartbeat prompts, skip `appendHeartbeatWorkspacePathHint` for configured custom prompt text, preserve the workspace-path hint for default/internal prompts, and add a regression test for a custom prompt mentioning `HEARTBEAT.md`.
