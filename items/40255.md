## Summary

Current main still appears to mutate user-configured heartbeat prompts that mention HEARTBEAT.md, and an open unmerged PR already targets this regression with closing syntax. This issue should remain open until that PR or an equivalent patch lands.

## Best Possible Solution

Keep this issue open and resolve it by reviewing and landing PR #40756 or an equivalent maintainer patch. The fix should skip `appendHeartbeatWorkspacePathHint` for user-authored configured heartbeat prompts, preserve the hint for built-in/default and internally generated exec or cron prompts, and add a focused regression test for a custom prompt that mentions `HEARTBEAT.md`.
