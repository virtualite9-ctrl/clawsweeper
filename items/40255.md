## Summary

Keep #40255 open. The checked-out main branch still resolves user-configured heartbeat prompts and then passes the resulting prompt through the workspace HEARTBEAT.md hint appender, so custom prompt text that mentions HEARTBEAT.md can still be modified before it is sent. The provided GitHub context also shows PR #40756 as the active open fix attempt, while earlier related PRs were closed unmerged duplicates.

## Best Possible Solution

Keep #40255 open and resolve it through PR #40756 or an equivalent maintainer patch. The fix should distinguish user-configured heartbeat prompts from default/internal prompts, skip appendHeartbeatWorkspacePathHint for user-authored prompt text, preserve workspace-path guidance for the built-in/default heartbeat prompt, and add a regression test for a custom prompt that mentions HEARTBEAT.md.
