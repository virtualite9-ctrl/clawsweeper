## Summary

Current `main` already routes explicit `exec host=node` through the node-execution path, includes regression coverage for the elevated/auto-host case discussed in the issue, and the same implementation is present in released `v2026.4.23`.

## Evidence

- **Node exec path is explicit on main:** `createExecTool` switches `host=node` requests onto `executeNodeHostCommand`, instead of local gateway execution, and preserves only an explicit node cwd to avoid the cross-platform approval failure called out in code comments.
  - file: [src/agents/bash-tools.exec.ts:1519](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/src/agents/bash-tools.exec.ts#L1519)
