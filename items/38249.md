## Summary

Keep #38249 open. Current main still has current-schema validation, legacy doctor migrations, `meta.lastTouchedVersion` stamping, and generic `doctor --fix` unknown-key pruning, but it does not implement `openclaw doctor --config-diff`, version-to-version schema comparison, categorized upgrade reporting, or migration guidance. Related #39780 is a separate Control UI apply/diff workflow, and closed #40015 does not supersede this CLI upgrade-diff request.

## Best Possible Solution

Keep #38249 open until maintainers either implement or explicitly reject a real CLI upgrade-diff workflow. A complete solution should add `openclaw doctor --config-diff` CLI plumbing, docs, and tests; compare the active config against the current schema and a trustworthy previous-version baseline derived from `meta.lastTouchedVersion`; report unrecognized keys, invalid enum values, new/removed/renamed keys, and migration suggestions; and reserve `--fix` for explicit safe migrations with backups. Keep #39780 separate for the Control UI apply/diff workflow.
