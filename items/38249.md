## Summary

Keep #38249 open. Current main still does not implement `openclaw doctor --config-diff`, version-to-version config schema comparison, categorized upgrade reporting, or migration guidance. Existing code covers current-schema validation, unknown-key pruning, and generic `doctor --fix` repair paths, but that is not the requested upgrade-safe diff workflow. Related #39780 is a Control UI before/after apply flow, and closed #40015 does not supersede this CLI migration-diff request.

## Best Possible Solution

Keep #38249 open until maintainers either implement or explicitly reject a CLI upgrade-diff workflow. A complete implementation should add `openclaw doctor --config-diff` CLI plumbing, docs, and tests; compare the active config against the current schema and a trustworthy previous-version baseline derived from `meta.lastTouchedVersion`; report unknown, invalid enum, removed, renamed, and newly available keys with migration guidance; and reserve `--fix` for explicit safe migrations with backups. Keep #39780 separate for Control UI before/after apply flows.
