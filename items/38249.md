## Summary

Keep #38249 open. Current main still has current-schema validation, `meta.lastTouchedVersion` stamping, legacy doctor migrations, and generic `doctor --fix` unknown-key pruning, but it does not implement the requested `openclaw doctor --config-diff` flag, version-to-version schema comparison, categorized added/removed/renamed key reporting, or upgrade migration guidance. The prior ClawSweeper comment reached the same result; the small changes since then add doctor compatibility inventory metadata, not the user-facing CLI workflow.

## Best Possible Solution

Keep #38249 open until maintainers either implement or explicitly reject a real CLI upgrade-diff workflow. A complete solution should add `openclaw doctor --config-diff` plumbing, docs, and tests; compare the active config against the current schema and a trustworthy previous-version baseline from `meta.lastTouchedVersion`; report unrecognized keys, invalid enum values, new/removed/renamed keys, and migration suggestions; and reserve `--fix` for safe migrations with backups. Keep #39780 as the separate Control UI apply/diff workflow.
