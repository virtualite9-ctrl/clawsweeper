## Summary

Keep open. Current main still lacks the requested `agents.setDefault` Gateway RPC: only the existing agent CRUD and file helper methods are registered, scoped, handled, and documented. `agents.update` still cannot accept a `default` mutation, the config contract remains `agents.list[].default`, and the Control UI still stages the invalid `agents.defaultId` key connected to open related bug #59070.

## Best Possible Solution

Implement an additive core Gateway method: add typed `agents.setDefault` params/result schemas, validators, method registry entry, ADMIN scope entry, and a server handler that rejects unknown configured agent ids, sets only the selected `agents.list[]` entry to `default: true`, clears the flag from all others, writes config, and returns `{ ok: true, defaultId }`. Update protocol docs and switch the Control UI Set Default path away from `agents.defaultId`; add focused handler/protocol tests plus UI/controller coverage. This should also address the related #59070 invalid-config symptom.
