## Summary

Current main does not implement the requested `agents.setDefault` Gateway RPC. The existing Gateway surface still exposes `agents.list`, `agents.create`, `agents.update`, and `agents.delete`; `agents.update` cannot mutate the `default` flag, and the Control UI Set Default path still writes the invalid `agents.defaultId` config key that the related #59070 report describes.

## Best Possible Solution

Keep this issue open and implement an additive core Gateway method. The likely fix belongs in the Gateway protocol and server-method surface: add typed `agents.setDefault` params/result schemas, register it in the method list and ADMIN scope, update `agents.list[]` so only the requested configured agent has `default: true`, reject unknown agent ids, return `{ ok: true, defaultId }`, update protocol docs, and switch the Control UI Set Default action away from writing `agents.defaultId`. Add focused server-method tests plus a UI/controller test covering the button path; this would also address the related #59070 invalid-config symptom.
