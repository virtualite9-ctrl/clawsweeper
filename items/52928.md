## Summary

Current main still does not implement the requested `agents.setDefault` Gateway RPC. The missing method is visible across the server handlers, advertised method list, ADMIN scope map, protocol schemas/types, protocol docs, and the Control UI path that still stages `agents.defaultId`, which also lines up with related open bug #59070.

## Best Possible Solution

Keep this issue open. Implement an additive core Gateway method `agents.setDefault` with params/result schemas, protocol exports, advertised method entry, ADMIN scope entry, docs, and focused handler tests. The handler should reject unknown configured agent ids, update `agents.list[]` so only the selected entry has `default: true`, clear/omit `default` on other entries, write config, and return `{ ok: true, defaultId }`. Then update the Control UI Set Default action to call this RPC instead of staging `agents.defaultId`, which should also address related #59070.
