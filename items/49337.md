## Summary

Keep open. Current `main` still truncates or misroutes literal unclosed `<think>` mentions in Telegram final text, so this is not implemented on `main`; and the submitted PR still has an unresolved safety concern that its `finalText` heuristic could leak malformed reasoning in final Telegram replies.

## Evidence

- **Shared strict stripping still drops text after an unclosed think tag:** `stripReasoningTagsFromText()` enters `inThinking` on an opening tag and, in strict mode, does not append trailing text when no closing tag appears. For input like `use the <think> tag for reasoning`, that leaves only the prefix before the tag.
