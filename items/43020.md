## Summary

Keep this issue open. The current Mattermost path still disables core block streaming while using a single draft preview post, and the assistant-message boundary hook only resets local partial text; it does not finalize or rotate the Mattermost draft post per turn. The two related fix PRs in the provided context were closed unmerged, and current docs describe the shipped behavior as one draft preview post rather than independent streamed posts per assistant segment.

## Best Possible Solution

Keep this issue open and fix it in the Mattermost plugin path. The likely maintainer direction is to rotate or finalize the draft preview at logical assistant-message boundaries when partial draft streaming has disabled core block streaming, then suppress duplicate final payloads. Add a focused Mattermost regression test that simulates assistant text, a tool boundary, a second assistant text segment, and final delivery, asserting that earlier streamed text is preserved instead of overwritten or batched at the end.
