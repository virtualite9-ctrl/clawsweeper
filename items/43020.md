## Summary

Keep this open. Current main still routes the Mattermost preview path through one draft post for the whole run, disables core block-streaming delivery, and only clears local partial text at assistant-message boundaries instead of preserving or rotating the completed draft segment. The related fix PRs #43041 and #46330 are closed unmerged in the provided GitHub context.

## Best Possible Solution

Keep the issue open and fix it in the Mattermost plugin path. At assistant-message or tool boundaries, the preview streaming code should flush/finalize or rotate the current draft post before the next segment updates the preview, while still suppressing duplicate final payloads. Add a focused Mattermost regression test that simulates text, a tool or assistant boundary, more text, and final delivery.
