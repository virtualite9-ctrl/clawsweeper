## Summary

Keep this issue open: current main still has the Mattermost edit-in-place path wired to one draft preview for the whole run, while core text block delivery is disabled and the assistant-boundary hook only resets local partial text instead of finalizing or rotating the visible post.

## Best Possible Solution

Keep the issue open and fix it in the Mattermost plugin path. On assistant-message/tool boundaries, finalize or rotate the current draft preview before the next segment updates the post, while suppressing duplicate final delivery for segments already shown. Add a focused Mattermost regression test that simulates partial text, a tool or assistant boundary, another partial text segment, and final delivery.
