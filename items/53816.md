## Summary

## Close Comment

Closing from a Codex review because current `main` already avoids the Windows CI failure described here. The reported leak source is still present in `src/cli/daemon-cli/lifecycle-core.test.ts:104`, but the test topology has changed: `test/vitest/vitest.unit-fast-paths.mjs:37` routes `src/plugin-sdk/**/*.test.ts` into the `unit-fast` lane, `test/vitest/vitest.scoped-config.ts:37` excludes those unit-fast files from scoped projects, and `scripts/lib/ci-node-test-plan.mjs:89` splits CI into separate `core-unit-fast-support` and `agentic-commands` shards. `.github/workflows/ci.yml:136` now builds the CI matrix from that split plan, so `src/plugin-sdk/channel-policy.test.ts` and `src/cli/daemon-cli/lifecycle-core.test.ts` no longer run in the same Windows CI shard/process on `main`. I do not see a released tag for this change from the current checkout, so I’m marking it fixed on `main` only.
