## Summary

Keep #69426 open. Current main still gates `params.context1m: true` behind `provider === "anthropic"` or `provider === "claude-cli"`, so the PR’s central GitHub Copilot/proxy bug is not implemented on main. The PR discussion and linked #69353 describe a real provider-proxy behavior gap, and current tests only cover direct Anthropic/Claude CLI positive paths, not `github-copilot` or other proxied Anthropic-capable providers.

## Best Possible Solution

Keep the PR or a maintainer-owned replacement open and land a small core fix in `src/agents/context.ts` that treats `context1m: true` as a model-capability opt-in rather than an Anthropic-provider-only path, with regression tests for proxied Anthropic-capable models and the non-Opus/Sonnet guard.
