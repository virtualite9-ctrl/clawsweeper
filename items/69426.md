## Summary

Keep #69426 open. Current main has not implemented the PR’s central behavior: `resolveContextTokensForModel` still requires the provider to be `anthropic` or `claude-cli` before `params.context1m: true` can return the 1M context window, so a proxied provider such as `github-copilot` still falls through to configured/discovered/fallback context limits. The supplied PR discussion and linked #69353 describe a concrete bug with reproduction context, and the PR’s proposed regression tests are not present on main.

## Best Possible Solution

Keep #69426 open for maintainer review or adaptation. The likely fix belongs in `src/agents/context.ts`: make the `context1m` planning path capability/model-prefix based rather than hard-gated to `anthropic` and `claude-cli`, while preserving the explicit opt-in and non-Opus/Sonnet guard. Add focused regression tests for proxied Claude models, at minimum the PR’s `github-copilot/claude-opus-4-6` positive case and `github-copilot/claude-haiku-3-5` negative case; consider the Bedrock/proxy scenario raised in review before landing.
