## Summary

Keep #69426 open. Current main still does not implement the PR's central behavior: `params.context1m: true` is still gated to `anthropic` and `claude-cli`, so proxied Anthropic-capable providers such as `github-copilot` remain blocked on this path.

## Best Possible Solution

Keep the PR open. Rebase it onto current main, preserve the explicit `context1m` opt-in plus Opus/Sonnet prefix gate, add regression coverage for `github-copilot` and any verified Anthropic-family proxy provider, and align request-path/header behavior before merge.
