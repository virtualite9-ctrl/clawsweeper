## Summary

Current `main` preserves explicit empty plugin scopes during provider/runtime loading, which matches the reported root cause for the false `memory-lancedb-pro` schema warning. The code and regression tests now enforce that an empty requested-plugin scope stays scoped-empty instead of widening into unrelated plugin loads.

## Evidence
