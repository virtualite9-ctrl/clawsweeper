## Summary

Keep #49062 open. Current main still does not define or document the requested Telegram `channel_post` authorization policy for `groups[chatId].allowFrom`: native command handling is still `ctx.message`-only, the existing auth path requires a sender id for per-group sender overrides, and the docs/tests only cover normal group sender allowlists. The related PR #49063 is useful context, but its own description explicitly leaves sender-scoped `groups[chatId].allowFrom` semantics for `channel_post` out of scope, so it does not supersede this issue.

## Best Possible Solution

Keep the issue open until the Telegram plugin defines one explicit policy for `channel_post` with `groups[chatId].allowFrom`: either ignore sender-scoped per-chat allowlists after explicit chat allow for channel posts, or reject that configuration with a clear user-facing message. The implementation should update `extensions/telegram/src/bot-native-commands.ts`, add regression coverage for allowed/denied `channel_post` native commands with per-group `allowFrom`, and document the chosen rule in `docs/channels/telegram.md` and slash-command docs if needed.
