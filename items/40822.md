## Summary

Current `main` already restores Perplexity OpenRouter/Sonar compatibility. The Perplexity web-search runtime now honors configured `baseUrl`, switches to chat-completions when `baseUrl`/`model` overrides are present or the route is non-direct, and the docs/changelog show this shipped in `v2026.3.8`.

## Evidence

- **Configured `baseUrl` is used for chat-completions requests:** `resolvePerplexityBaseUrl()` returns the explicit configured `perplexity.baseUrl` when present, and `runPerplexitySearch()` builds the request URL from that base URL as `${baseUrl}/chat/completions`.

  - command: `nl -ba extensions/perplexity/src/perplexity-web-search-provider.runtime.ts | sed -n '90,340p'`
- **Runtime switches off native `/search` when OpenRouter or overrides are in play:** `resolvePerplexityTransport()` selects `chat_completions` whenever a legacy override exists or the resolved base URL is not the direct Perplexity host, which covers OpenRouter proxy setups and explicit `baseUrl` overrides.
  - file: [extensions/perplexity/src/perplexity-web-search-provider.runtime.ts:142](https://github.com/openclaw/openclaw/blob/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503/extensions/perplexity/src/perplexity-web-search-provider.runtime.ts#L142)
