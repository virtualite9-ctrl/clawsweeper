## Summary

Keep #48641 open. The report is recent, concrete, and the provided context shows the targeted fix PR (#49365) was closed unmerged. Current main has Discord DM plumbing, but it still preserves account-scoped `allowFrom` override semantics and I did not find a shipped fix or regression test for the reported multi-account DM allowlist drop.

## Best Possible Solution

Keep the issue open for Discord channel owner review. Reproduce on current main with a multi-account Discord config and decide whether global `channels.discord.allowFrom` should merge into account allowlists or remain an overridable default. If merging is the intended behavior, implement it in `extensions/discord/src/accounts.ts` or the shared account helper with focused regression tests for default and named accounts. If override semantics are intentional, update the issue with the supported configuration shape and add clearer startup/drop diagnostics for blocked DMs.
