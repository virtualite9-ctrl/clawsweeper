## Summary

Keep open. Current `main` still has only the minimal `build` Makefile target, so this is not implemented already, and the repo’s documented contributor workflow is still the `pnpm` command surface. The PR may still be useful, but it adds a second maintenance surface and at least one proposed wrapper (`make format`) would not match the current check-vs-fix semantics, so this needs maintainer judgment rather than conservative auto-close.
