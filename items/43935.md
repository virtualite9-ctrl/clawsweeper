## Summary

Keep open. Current main still collapses `channels.<channel>.accounts.<accountId>.*` changes into channel-wide hot reloads, while the lower-level channel manager already supports account-scoped start/stop. The paired implementation PR #43938 remains open and contains closing syntax for this issue, so the right maintainer path is to review, update, and land or close that PR rather than close this issue now.

## Best Possible Solution

Keep this issue open while #43938 remains open. The best path is to update and review that PR, or an equivalent core gateway change, so account-only paths under `channels.<channel>.accounts.<accountId>.*` plan and execute account-scoped restarts while global or mixed channel changes still use full-channel restarts. Add focused planner and handler tests on current main before landing.
