## Summary

Keep #12590 open. Current main still has the reported counter synchronization path: a memory-flush-triggered compaction can write the post-increment `compactionCount` back into `memoryFlushCompactionCount`, while the flush gate skips when those values are equal.

## Best Possible Solution

Keep #12590 open. Fix `src/auto-reply/reply/agent-runner-memory.ts` so a flush-triggered compaction does not persist the post-increment `compactionCount` as `memoryFlushCompactionCount`; preserve the pre-increment count or replace the equality marker with clearer state. Then update the synchronized-counter test and add regression coverage proving consecutive eligible auto-compaction cycles run `memoryFlush` each time. PR #51421 is the current open implementation reference.
