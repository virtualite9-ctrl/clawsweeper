## Summary

Keep this PR open. Current main 0835f9409aac0f6048c7d03ec5a2148b7e5001c5 still lacks the shared normalization for leading `functions ` tool-call IDs and prefixed tool names that this PR targets. The predecessor #39091 is closed unmerged, and this PR remains open, mergeable, and actively maintained with focused tests. Security review found no workflow, dependency, lockfile, publishing, or downloaded-artifact changes, but the node-host exec workdir changes touch command-execution approval behavior and should be explicitly reviewed or split before merge.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land a narrow agent-side fix that normalizes only the leading `functions ` versus `functions.` ID equivalence where matching needs it, strips `functions.` / `functions ` prefixes before transcript-repair allowed-name checks and `sessions_spawn` redaction, preserves raw IDs for pending-state and replay-safe signed-thinking behavior, and keeps the regression tests. Review or split the node-host exec workdir/approval change before merge.
