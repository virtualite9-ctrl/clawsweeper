## Summary

Keep this PR open. Current main still lacks the PR's central `functions.exec:0` / `functions exec:0` tool-call ID pairing behavior in transcript repair and ID sanitization, and `sanitizeToolCallInputs` still trims tool names without stripping `functions.` / `functions ` prefixes before allowlist checks or `sessions_spawn` redaction. The related predecessor #39091 is closed unmerged, not a shipped canonical fix.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is a narrow agent-side fix that normalizes only the intended leading `functions ` versus `functions.` provider mangling where pairing requires it, strips `functions.` / `functions ` prefixes before allowed-name checks and `sessions_spawn` redaction, preserves raw IDs where pending-state and replay-safe signed thinking require byte stability, and includes regression tests for mixed dot/space IDs plus prefixed tool names. Split the node workdir changes unless they are directly required for this bug fix.
