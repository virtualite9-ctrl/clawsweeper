## Summary

Keep PR #39245 open. Current main preserves canonical Kimi IDs and normalizes provider-prefixed tool names for live dispatch, but it still does not implement the PR’s central mixed-ID behavior: pairing an assistant call such as `functions.exec:0` with a tool result such as `functions exec:0`. The related PR #39091 is closed unmerged, and #39245 still needs maintainer review because the active discussion includes collision-safety concerns for normalized ID matching.

## Best Possible Solution

Keep #39245 open for maintainer review. The best path is a narrow agent-side fix that preserves current canonical Kimi ID behavior and replay-safe raw IDs, normalizes only the leading `functions ` versus `functions.` provider mangling where interoperability requires it, binds normalized pairing to additional context, adds regression coverage for the mixed dot/space ID case, and keeps prefixed `sessions_spawn` redaction covered.
