## Summary

Keep this PR open. Current main fee16865b229cf3d652041eadb12a1b25dba07c0 still lacks shared equivalence for `functions exec:0` versus `functions.exec:0` in tool-call ID sanitization/pairing, and still does not strip `functions.` / `functions ` prefixes in `session-transcript-repair` before allowed-name checks or `sessions_spawn` redaction. The PR is open, active, and no merged canonical replacement exists. The security pass did not find workflow, dependency, lockfile, package publishing, or downloaded-artifact changes, but the node-host exec workdir/approval behavior is command-execution-sensitive and should be explicitly reviewed or split.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land a narrow agent-side fix that normalizes only leading `functions ` versus `functions.` where ID matching requires it, strips `functions.` / `functions ` prefixes before allowed-name checks and `sessions_spawn` redaction, preserves raw IDs for pending-state and replay-safe signed-thinking behavior, and keeps the regression tests shown here. The node-host exec workdir/approval change should be reviewed as command-execution behavior or split into a separate PR.
