## Summary

Current `main` does not implement the requested QMD no-rerank exposure. The QMD config surface only exposes `searchMode`/`searchTool`, the direct CLI path always builds plain `qmd query ... --json -n ...` args, and the mcporter/MCP path sends `searches` + `limit` without any `rerank: false` field.

## Evidence

- **Config type has no rerank toggle:** `MemoryQmdConfig` exposes `command`, `mcporter`, `searchMode`, `searchTool`, `includeDefaultMemory`, `paths`, `sessions`, `update`, `limits`, and `scope`; there is no `noRerank` or similar field.
  - file: [src/config/types.memory.ts:13](https://github.com/openclaw/openclaw/blob/8b76392e3e7934b1cdd6d461148263fd56141ec8/src/config/types.memory.ts#L13)
  - sha: [8b76392e3e79](https://github.com/openclaw/openclaw/commit/8b76392e3e7934b1cdd6d461148263fd56141ec8)
- **User docs list no rerank option:** The QMD backend config table documents `command`, `searchMode`, `includeDefaultMemory`, `paths`, and session settings only. No config key is documented for disabling reranking.
  - file: [docs/reference/memory-config.md:392](https://github.com/openclaw/openclaw/blob/8b76392e3e7934b1cdd6d461148263fd56141ec8/docs/reference/memory-config.md#L392)
  - sha: [8b76392e3e79](https://github.com/openclaw/openclaw/commit/8b76392e3e7934b1cdd6d461148263fd56141ec8)
- **Direct CLI search path never appends `--no-rerank`:** Search execution builds args via `buildSearchArgs(...)`, then runs `qmd`. For `query` mode, `buildSearchArgs` returns only `["query", query, "--json", "-n", limit]` before collection filters are added.
  - file: [extensions/memory-core/src/memory/qmd-manager.ts:1150](https://github.com/openclaw/openclaw/blob/8b76392e3e7934b1cdd6d461148263fd56141ec8/extensions/memory-core/src/memory/qmd-manager.ts#L1150)
  - sha: [8b76392e3e79](https://github.com/openclaw/openclaw/commit/8b76392e3e7934b1cdd6d461148263fd56141ec8)
- **MCP/mcporter query path sends no `rerank: false` flag:** For the unified QMD `query` MCP tool, `callArgs` contains `searches` and `limit` (plus optional collections). There is no request-level rerank toggle in the payload.
  - file: [extensions/memory-core/src/memory/qmd-manager.ts:1948](https://github.com/openclaw/openclaw/blob/8b76392e3e7934b1cdd6d461148263fd56141ec8/extensions/memory-core/src/memory/qmd-manager.ts#L1948)
  - sha: [8b76392e3e79](https://github.com/openclaw/openclaw/commit/8b76392e3e7934b1cdd6d461148263fd56141ec8)
- **Existing tests only cover current knobs and current query args:** Backend-config tests verify `searchMode` and `searchTool` only, and qmd-manager tests expect `query` invocations without `--no-rerank`, which matches the current implementation rather than the requested feature.
  - file: [src/memory-host-sdk/host/backend-config.test.ts:323](https://github.com/openclaw/openclaw/blob/8b76392e3e7934b1cdd6d461148263fd56141ec8/src/memory-host-sdk/host/backend-config.test.ts#L323)
  - sha: [8b76392e3e79](https://github.com/openclaw/openclaw/commit/8b76392e3e7934b1cdd6d461148263fd56141ec8)
- **Repo search found no existing no-rerank wiring:** A code search for `noRerank`, `--no-rerank`, and `rerank: false` under `packages/memory-host-sdk`, `src`, `docs`, and `extensions` produced no relevant implementation hits.
  - command: `rg -n -e "noRerank" -e "rerank: false" -e "--no-rerank" packages/memory-host-sdk src docs extensions`
  - sha: [8b76392e3e79](https://github.com/openclaw/openclaw/commit/8b76392e3e7934b1cdd6d461148263fd56141ec8)

## Risks / Open Questions

- A request-level `memory_search.rerank` flag would extend a public tool contract, so future work needs schema/docs/test coordination, not just QMD plumbing.
- The QMD backend/config surface is mirrored across `src/` and `packages/`, so any implementation will need to keep both copies aligned.
