## Summary

Close as ClawHub. Current main does not ship a bundled LocalAI provider, but this PR is a self-contained provider plugin with no demonstrated core or SDK gap. The current project direction, plugin docs, SDK exports, and existing self-hosted OpenAI-compatible provider helpers all support moving this LocalAI implementation to an external ClawHub/npm plugin instead of adding another bundled core plugin.

## Best Possible Solution

Move the LocalAI provider draft into an external LocalAI provider plugin published through ClawHub/npm, reusing `openclaw/plugin-sdk/provider-setup` and the manifest/onboarding structure already sketched in the PR. Keep OpenClaw core focused on generic provider SDK seams; open a focused SDK issue or PR only if LocalAI needs a missing generic hook.
