## Summary

Keep #12855 open. Current main ships a useful default-off package auto-updater under `update.auto`, but it does not implement the issue's requested first-class workflow: configurable wall-clock/cron scheduling, confirmation and notify-only modes, configured channel delivery, or post-update version/changelog/health notifications.

## Best Possible Solution

Keep #12855 open as the canonical product/implementation tracker for the remaining first-class auto-update workflow. Extend the existing `update.auto` surface rather than adding a parallel updater: add schema/types/help/docs for mode, schedule, timezone, notification settings; implement HH:MM or cron timezone scheduling; add `confirm`, `silent`, and `notify-only` delivery through configured channels; persist post-restart update state; and report old/new version, changelog link or summary, and health after restart. Keep #65011 focused on UI/discoverability and treat #12896 as a closed unmerged attempt.
