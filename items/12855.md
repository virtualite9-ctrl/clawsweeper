## Summary

Keep #12855 open. Current main has a real default-off package auto-updater under `update.auto`, and Bortlesboat’s linked guide kit shows operators can use those primitives today, but the issue’s core request is still not implemented: confirmation/notify-only/silent modes, HH:MM or cron timezone scheduling, configured channel notification, and post-update version/changelog/health reporting remain absent. The closed PR #12896 did target this request but was not merged, and #65011 tracks UI/discoverability rather than replacing this workflow request.

## Best Possible Solution

Keep #12855 open as the canonical product/implementation tracker for the remaining first-class auto-update workflow. Extend the existing `update.auto` path through config schema/types/help/docs, runtime HH:MM or cron timezone scheduling, `confirm`/`silent`/`notify-only` modes, configured channel delivery, post-restart version/changelog/health notifications, and focused tests. Keep #65011 scoped to UI/discoverability for the updater that already exists, and treat #12896 as a closed unmerged attempt rather than a shipped fix.
