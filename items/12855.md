## Summary

Keep #12855 open. Current main has a useful default-off package auto-updater under update.auto, but it does not implement the requested first-class workflow: configurable wall-clock/cron scheduling, timezone, confirm or notify-only modes, channel-targeted notifications, or post-restart version/changelog/health reporting. The latest provided maintainer/Codex comment also explicitly kept this issue open and separated the UI/discoverability follow-up into #65011.

## Best Possible Solution

Keep #12855 open as the canonical engine/workflow issue. Extend the existing core update.auto package updater with schema, types, help, docs, and tests for mode, schedule, timezone, notifyAfterUpdate, and notifyChannel; add HH:MM or cron timezone scheduling; implement notify-only, confirm, and silent flows through configured channel targets; persist update state across restart; and report old/new version, changelog link or summary, and health after restart. Leave #65011 as the separate Control UI/onboarding/discoverability follow-up.
