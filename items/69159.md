## Summary

Keep open. Current main partially supersedes the PR’s error-handling direction with SessionWriteLockTimeoutError and failover suppression, but the embedded attempt path still does not pass the run timeout into acquireSessionWriteLock, so the central lock-wait fix remains missing.

## Best Possible Solution

Keep the PR open and ask for an update against current main: add timeoutMs: params.timeoutMs to the acquireSessionWriteLock call in src/agents/pi-embedded-runner/run/attempt.ts, reuse or deliberately extend the existing SessionWriteLockTimeoutError/code API rather than adding a parallel error class, export the chosen error/helper from the plugin SDK harness surface if SDK consumers are expected to branch on it, and add a regression test proving the embedded attempt forwards the run timeout. Leave the broader lock-scope/configurability work to #13744.
