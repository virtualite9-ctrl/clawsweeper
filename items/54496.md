## Summary

Keep open. Current main has reasoning visibility controls, but they only affect whether reasoning is shown/streamed. The requested core behavior, producing thinking while preventing thinking blocks from being persisted in the session transcript, is still not implemented.

## Best Possible Solution

Implement a core transcript-persistence policy that is independent of thinking effort and reasoning visibility. The likely path is to keep `thinkingLevel` for model reasoning production, keep `reasoningLevel` for user/channel visibility, and add a documented persistence mode that strips `type: "thinking"`/`redacted_thinking` before session transcript writes or stores only replay-safe opaque state outside user/history/memory transcript APIs. Cover `sessions.get`, ACP loadSession, compaction/replay, built-in session memory, and provider replay edge cases with tests.

## Risks / Open Questions

- A real fix must preserve provider replay compatibility: current code keeps the latest signed thinking block because some providers need signed reasoning context for continuation.
- Changing stored transcript shape can affect ACP session replay, `sessions.get`, session history tools, compaction, and memory indexing, so this needs a narrow migration/compatibility design and regression tests.
