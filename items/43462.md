## Summary

Keep this PR open. Current main still has case-sensitive `memory/` directory-prefix checks in the current memory host SDK mirrors and temporal-decay path parsing, so the requested behavior is not implemented. The PR is useful, but it is stale against current main because the original `src/memory/*` implementation moved and the fix needs to be ported.

## Best Possible Solution

Keep this PR open as a useful but stale implementation candidate. Port the directory-prefix behavior to current main by lowercasing only normalized `memory/` directory checks in `src/memory-host-sdk/host/internal.ts`, `packages/memory-host-sdk/src/host/internal.ts`, and `extensions/memory-core/src/memory/temporal-decay.ts`; use the same lowercased normalized value for the dated-memory regex; consider the equivalent QMD default-memory path gate in `extensions/memory-core/src/memory/qmd-manager.ts`; add focused regression tests for `MEMORY/notes.md`, `Memory/daily.md`, and `MEMORY/YYYY-MM-DD.md`; and preserve canonical root `MEMORY.md` behavior rather than restoring legacy lowercase root indexing.
