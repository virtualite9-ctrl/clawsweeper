## Summary

Keep this PR open. Current main still has case-sensitive `memory/` directory-prefix checks in the moved memory host SDK, packaged SDK mirror, QMD read gate, and temporal-decay path parsing. The PR is stale against current main because the original `src/memory/*` files no longer exist, but it remains a useful implementation candidate after porting.

## Best Possible Solution

Keep this PR open as a useful but stale implementation candidate. Port the directory-prefix behavior to current main by lowercasing only normalized `memory/` directory checks in `src/memory-host-sdk/host/internal.ts`, `packages/memory-host-sdk/src/host/internal.ts`, and `extensions/memory-core/src/memory/temporal-decay.ts`; use the lowercased value for the dated-memory regex; consider the equivalent QMD default-memory path gate in `extensions/memory-core/src/memory/qmd-manager.ts`; add focused regression tests for `MEMORY/notes.md`, `Memory/daily.md`, and `MEMORY/YYYY-MM-DD.md`; and preserve current canonical root `MEMORY.md` behavior rather than restoring legacy lowercase root indexing.
