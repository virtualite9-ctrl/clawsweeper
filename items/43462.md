## Summary

Keep this PR open. Current main has moved the relevant implementation out of the original PR paths, but the requested case-insensitive directory-prefix behavior is still not implemented in the current memory host SDK and memory-core temporal-decay code. The latest PR commit also addressed the reviewer-identified dated-path regression, so the useful path is to port the fix to the current files rather than close it as obsolete.

## Best Possible Solution

Port the PR’s directory-prefix behavior to current main by lowercasing only the normalized `memory/` directory-prefix checks in `src/memory-host-sdk/host/internal.ts`, `packages/memory-host-sdk/src/host/internal.ts`, and `extensions/memory-core/src/memory/temporal-decay.ts`; apply the same lowercased value to the dated-memory regex; and add focused regression tests for `MEMORY/notes.md`, `Memory/daily.md`, and `MEMORY/YYYY-MM-DD.md`. Keep current canonical root `MEMORY.md` semantics intact.
