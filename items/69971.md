## Summary

Keep #69971 open. Current main still lacks both requested behaviors: plugin install still reaches installer fallbacks without an explicit profile-aware extensionsDir, and plugin discovery still skips symlinked directory entries. The PR is a relevant fix candidate, but its current approach also has an unresolved review concern because discovery’s global root is based on resolveConfigDir while the PR computes install root from resolveStateDir, which can diverge in OPENCLAW_CONFIG_PATH-only or legacy-state-dir layouts.

## Best Possible Solution

Keep the PR open and ask for a revision that makes plugin install and discovery share the same managed extensions-root contract after profile/env resolution. The fix should pass that root through npm/path/link install paths, consider the onboarding/channel setup install path, and keep/add regression coverage for --profile, OPENCLAW_CONFIG_PATH without OPENCLAW_STATE_DIR, legacy-state-dir fallback, and symlink plus broken-symlink discovery.
