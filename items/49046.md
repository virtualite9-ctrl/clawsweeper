## Summary

Keep #49046 open. Current main has a concrete `model.usage` diagnostic payload shape in TypeScript and tests, but the user-facing logging/OpenTelemetry docs still describe the event only in category terms and do not publish the exact JSON keys or a sample entry for external parsers. The issue is recent, author is not protected, and the remaining work is an actionable docs/contract clarification rather than a closeable duplicate or already-implemented documentation request.

## Best Possible Solution

Keep the issue open and satisfy it with a docs/contract update: add a `model.usage` payload table and JSON example to the public logging/OpenTelemetry docs, clarify camelCase field names versus provider-native snake_case token counters, document `usage` versus `context` semantics, and explicitly state which surface external readers should parse if the rolling `/tmp/openclaw/openclaw-YYYY-MM-DD.log` file is not the supported source for these diagnostic events.
