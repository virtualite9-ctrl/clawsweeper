## Summary

Current main still appears to have the reported legacy SQLite migration bug, and the issue is paired with an open same-author PR that should be reviewed rather than preemptively closing the report.

## Best Possible Solution

Keep this issue open and review #65767, or an equivalent maintainer patch, to migrate/rebuild legacy `flow_runs` tables so `owner_session_key` no longer blocks inserts, preserve existing ownership in `owner_key`, and add a focused regression test for `create_flow` or `createManagedTaskFlow` against the legacy SQLite schema.
