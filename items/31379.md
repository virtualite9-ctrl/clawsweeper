## Summary

Current main still has the reported Control UI behavior: `resolveChannelOrder` uses `channelMeta` or `channelOrder` exclusively when those snapshot arrays are present, while `channels.status` builds those arrays from loaded channel plugins only. A custom-only channel snapshot can still hide the built-in channel cards. The focused fix PR #44165 is open and unmerged, and the earlier PR #31350 was closed unmerged, so this is not implemented or safely superseded yet.

## Best Possible Solution

Keep #31379 open until #44165 or an equivalent focused Control UI fix lands. The best fix is in `ui/src/ui/views/channels.ts`: merge snapshot-provided custom channel ids with the built-in channel id list, preserve snapshot/custom order first, de-dupe ids, keep the existing enabled-channel promotion sort, and add regression coverage for custom-only `channelMeta`, custom-only `channelOrder`, and no-snapshot fallback.
