## Summary

Current main still appears to have the reported Control UI behavior: when `channels.status` includes a custom-only `channelMeta` or `channelOrder`, the Channels view renders only those ids and does not append the built-in channel cards. The focused fix PR #44165 is still open/unmerged in the provided context, and the earlier PR #31350 was closed unmerged, so this issue should remain open until a fix lands.

## Best Possible Solution

Keep #31379 open until #44165 or an equivalent focused Control UI fix lands. The best implementation path is to update `ui/src/ui/views/channels.ts` so `resolveChannelOrder()` merges snapshot-provided custom ids with the built-in channel id list, preserves snapshot/custom order first, de-dupes ids, keeps the existing enabled-channel promotion sort, and adds regression tests in `ui/src/ui/views/channels.test.ts` for custom-only `channelMeta`, custom-only `channelOrder`, and the no-snapshot fallback.
