## Summary

Current main still has the reported Channels settings behavior. The Control UI renders exactly the ids returned by `channelMeta` or `channelOrder`, while `channels.status` builds those fields from loaded channel plugins only. A custom-only loaded channel snapshot can still omit the bundled channel cards. The focused fix PR #44165 is still open in the provided GitHub context, and the earlier PR #31350 was closed unmerged, so this issue remains the right tracker until a fix lands.

## Best Possible Solution

Keep #31379 open until PR #44165 or an equivalent focused Control UI fix lands. The likely implementation belongs in `ui/src/ui/views/channels.ts`: merge snapshot-provided custom ids with the built-in channel id list, de-dupe while preserving snapshot/custom order first, retain the enabled-channel promotion sort, and add regression coverage in `ui/src/ui/views/channels.test.ts` for custom-only `channelMeta`, custom-only `channelOrder`, and the no-snapshot fallback.
