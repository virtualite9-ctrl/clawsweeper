## Summary

Keep #10029 open. Current main still renders WebChat fenced and indented code blocks as escaped monospace text with language labels, copy buttons, and JSON collapse chrome, but it does not tokenize code or emit syntax-highlighting markup/styles. The prior maintainer/Codex discussion also correctly distinguished the shipped copy-button work from this still-missing syntax-highlighting request.

## Best Possible Solution

Keep #10029 open until Control UI/WebChat integrates a sanitization-compatible syntax highlighter in `ui/src/ui/markdown.ts` or an adjacent renderer layer, adds theme CSS for highlighted tokens, and adds focused tests proving highlighted fenced code while preserving copy, language label, JSON collapse, cache, and fallback behavior.
