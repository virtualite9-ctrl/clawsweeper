## Summary

Keep #10029 open. Current main still renders WebChat code blocks as escaped text with language labels, copy button chrome, and JSON collapse behavior, but it does not tokenize code or apply syntax-highlighting markup/theme. The existing April 26 maintainer/Codex comment remains accurate against current main; the related #30390 copy-button work is separate and does not satisfy this request.

## Best Possible Solution

Keep #10029 open until Control UI/WebChat integrates a sanitization-compatible syntax highlighter in `ui/src/ui/markdown.ts` or an adjacent renderer layer, adds matching theme CSS, and adds focused tests proving highlighted fenced code while preserving copy, language label, JSON collapse, cache, and sanitization behavior.
