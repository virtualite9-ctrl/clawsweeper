## Summary

Keep #10029 open. Current main still renders WebChat code fences as escaped monospace code with a language class, language label, copy button, and JSON collapse behavior, but it does not apply syntax highlighting in the WebChat markdown renderer. The prior maintainer/Codex comment remains accurate against current main e29d3516bf05591e6007bfb8ccdaea6d76194d41.

## Best Possible Solution

Keep #10029 open until WebChat implements real syntax highlighting in ui/src/ui/markdown.ts with an approved highlighter or markdown-it highlight integration, adds theme CSS for Control UI, and includes focused tests that prove highlighted fenced code while preserving the existing copy button, language label, JSON collapse, and sanitization behavior.
