## Summary

Keep open. Current main still documents a plugin publish surface that same-day maintainer verification says is not present in the shipped `clawhub@latest` client, and this checkout does not contain a core fix for the reported 120s package-publish timeout. The repo’s own ClawHub release workflow appears to use a pinned checkout of `openclaw/clawhub` rather than the npm-published CLI, which supports the reporter’s docs/client surface mismatch rather than resolving it.

## Best Possible Solution

Keep the issue open and fix the mismatch at the source of truth: either publish a new `clawhub` CLI that supports the documented `package publish` source forms, `--dry-run`, and `--json`, and investigate/diagnose the 120s publish timeout with client/server-side timeout ownership; or update OpenClaw docs and release helpers to match the currently shipped `clawhub@latest` surface. The workflow's pinned external ClawHub source should be reconciled with the npm-published CLI so external plugin authors use the same path as maintainers.
