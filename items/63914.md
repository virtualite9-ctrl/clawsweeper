## Summary

Keep open. Current `main` still exposes `sessions_spawn.runtime` to the model and still routes `runtime:"acp"` directly into the ACP path without a guard that rejects configured in-config agent ids. The prompt/docs separation between `agents_list` and ACP harness ids exists, but the implementation does not close the reported footgun, and the latest release does not appear to contain a code fix.

## Evidence

There is no pre-check here for "configured OpenClaw agent id used with ACP runtime".
