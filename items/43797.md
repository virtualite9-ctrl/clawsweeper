## Summary

Keep open. Current main still prunes sandbox runtimes and registry entries without removing the non-shared workspace directory, and the provided GitHub context shows an open implementation PR (#43835) with closing syntax for this issue. Per maintainer cleanup rules, the issue should stay open while that PR is reviewed, landed, or explicitly closed.

## Best Possible Solution

Keep this issue open and review #43835 as the implementation candidate. The likely product fix belongs in `src/agents/sandbox/prune.ts`: after successful runtime removal for non-shared scope entries, resolve the workspace directory from the configured sandbox `workspaceRoot` and registry scope key, remove it recursively with safe error handling, and add focused tests that prove prune deletes session/agent workspaces while preserving shared scope. If #43835 is not acceptable, close or supersede that PR and land an equivalent tested fix before closing this issue as implemented.
