## Summary

Current `main` already implements reactive Control UI chat updates: user messages are appended immediately, assistant delta/final gateway events update the in-memory transcript, and gateway `session.message`/terminal chat events trigger `chat.history` reloads so persisted messages appear without a manual page refresh.

## Evidence

- **User messages are appended optimistically before the send request completes:** `sendChatMessage` pushes a new `role: "user"` entry into `state.chatMessages` before awaiting `chat.send`, so the sender does not need a full page refresh to see their own message.
  - file: [ui/src/ui/controllers/chat.ts:294](https://github.com/openclaw/openclaw/blob/cb2c36b04975e34ab7014a11575d6589aa96ae32/ui/src/ui/controllers/chat.ts#L294)

  - sha: [cb2c36b04975](https://github.com/openclaw/openclaw/commit/cb2c36b04975e34ab7014a11575d6589aa96ae32)
- **Assistant responses are applied from live gateway chat events:** `handleChatEvent` updates `chatStream` on `delta` events and appends the assistant message on `final`/`aborted`, clearing run state afterward. That is the real-time path for streamed replies.
  - file: [ui/src/ui/controllers/chat.ts:429](https://github.com/openclaw/openclaw/blob/cb2c36b04975e34ab7014a11575d6589aa96ae32/ui/src/ui/controllers/chat.ts#L429)

## Close Comment

Closing from a Codex review because current `main` already has the real-time update path this report asks for. `ui/src/ui/controllers/chat.ts:294` appends the user message immediately before `chat.send` finishes, `ui/src/ui/controllers/chat.ts:429` applies live `chat` delta/final events directly to `chatStream` and `chatMessages`, and `ui/src/ui/app-gateway.ts:472` plus `ui/src/ui/app-gateway.ts:481` reload `chat.history` on terminal `chat` / `session.message` events so persisted transcript changes show up without a manual page refresh. The behavior is also covered by tests in `ui/src/ui/controllers/chat.test.ts:112` and `ui/src/ui/app-gateway.node.test.ts:731`. If you can still reproduce this on a current build, please open a new issue with updated repro steps and logs so it can be scoped to the remaining variant.
