## Summary

Keep #13670 open. Current main still does not implement the requested always-visible Control UI topbar display for active model plus token/context usage. The existing surfaces are adjacent but scoped elsewhere: topbar status only renders chat mobile controls and the theme toggle, Debug data remains Debug-tab scoped, chat model selection lives in chat controls, and usage/context indicators appear in slash-command output, message metadata, the Usage page, or high-context composer notices rather than the global topbar.

## Best Possible Solution

Keep #13670 open until maintainers either implement a focused Control UI topbar status component or explicitly decline the placement. A good implementation should live in the topbar render path, derive active model from the current session/default model state, derive token/context usage from fresh session usage or session rows rather than promoting Debug-tab polling into global UI state, and add focused UI tests for visibility, freshness/stale handling, and responsive layout.
