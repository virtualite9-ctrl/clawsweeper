## Summary

Current main still does not implement the requested always-visible Control UI topbar display for active model plus token/context usage. The repo has adjacent model and usage surfaces in the chat header, composer context notice, per-message metadata, usage tab, and debug tab, but those do not satisfy the specific topbar status request. The provided GitHub context also includes a maintainer-authored Codex review comment from 2026-04-26 already keeping this issue open, and current source inspection at e29d3516bf05591e6007bfb8ccdaea6d76194d41 still matches that conclusion.

## Best Possible Solution

Keep #13670 open until maintainers either implement a focused Control UI topbar status component or explicitly decline that placement. A good implementation should live in the topbar render path, derive the active model from current session/default model state, derive token/context usage from fresh session usage data where possible, avoid using debug-tab-only polling as normal UI state, and add focused UI tests plus docs if the surface ships.
