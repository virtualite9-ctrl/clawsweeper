## Summary

Keep #13670 open. Current main still does not implement the requested always-visible Control UI topbar display for active model plus token/context usage. The data exists in adjacent Debug, Usage, Overview, and chat surfaces, but the topbar still only renders navigation/search plus chat mobile toggle and theme controls. A maintainer-owned review comment also explicitly kept this open on 2026-04-26, and current main has not changed that surface.

## Best Possible Solution

Keep #13670 open until maintainers either implement a focused Control UI topbar status component or explicitly decline that placement. A good implementation should live in the topbar render path, derive the active model from current session/default model state, derive fresh token/context usage from session usage data, handle stale/unavailable states, and add responsive UI coverage.
