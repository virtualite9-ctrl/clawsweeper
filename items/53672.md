## Summary

The reported 2026.3.22 npm-package regression is already addressed on current `main`, and the shipped `v2026.4.22` release contains hard prepack and release guards that block publishing an npm package without the Control UI HTML plus bundled asset payload.

## Close Comment

Closing from a Codex review: this npm-packaging regression is already addressed on current `main`, and the shipped `v2026.4.22` release contains the guardrails for it. In `scripts/openclaw-prepack.ts`, packing now fails unless both `dist/control-ui/index.html` and a non-empty `dist/control-ui/assets/` payload exist, and `scripts/openclaw-npm-release-check.ts` rejects any `npm pack` result missing those paths. The release workflow also builds the Control UI and runs those checks before packing in `.github/workflows/openclaw-npm-release.yml`, with regression coverage in `test/openclaw-prepack.test.ts` and `test/openclaw-npm-release-check.test.ts`. That means the 2026.3.22 missing-Control-UI-assets failure path is no longer shippable.
