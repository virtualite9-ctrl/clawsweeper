## Summary

Keep #71185 open. Current main still contains the unsafe support-capture path the PR targets: `buildSkillsCapture` maps every `resolvedSkills` entry, passes `skill.filePath`/`skill.baseDir` directly into `redactPathForSupport`, and sorts via `left.name.localeCompare(...)`; `redactPathForSupport` still calls `file.startsWith(...)` with no null guard. The PR is a focused defensive fix with targeted regression tests and positive bot/Codex review context, so it remains useful rather than obsolete or closeable.

## Best Possible Solution

Keep the PR open for maintainer review/merge or request any desired producer-side follow-up separately. The best code path is to land a narrowly scoped consumer hardening change with the regression tests from the PR, while preserving the symlink-escape security behavior in `src/agents/skills/workspace.ts`.
