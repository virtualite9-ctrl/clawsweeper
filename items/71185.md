## Summary

Current main still has the pre-PR null-unsafe trajectory support-capture behavior. `buildSkillsCapture` trusts every `resolvedSkills` entry, redacts `filePath`/`baseDir`, and sorts with `left.name.localeCompare(...)`; `redactPathForSupport` still accepts only `string` and immediately calls `startsWith`. The PR is a recent, focused defensive fix with targeted regression coverage, and the provided related items do not supersede this support-capture issue.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land or refine the focused consumer-side hardening in `src/trajectory/metadata.ts` and `src/logging/diagnostic-support-redaction.ts` with the proposed regression tests in `src/trajectory/metadata.test.ts`; any producer-side cleanup in `src/agents/skills/workspace.ts` can remain a separate follow-up if maintainers want to eliminate partial entries at the source.
