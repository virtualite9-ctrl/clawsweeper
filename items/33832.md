## Summary

Keep #33832 open. Current main has sanitized model-call lifecycle diagnostics and duration metrics, but it still does not implement the requested Gateway logging for request body size, response body size, or time-to-first-byte. The linked implementation PR #35513 was closed unmerged, so the feature remains unshipped.

## Best Possible Solution

Keep #33832 open until OpenClaw core adds privacy-safe model API request/response byte counts plus first-byte and total latency logging or telemetry, likely near the existing model-call diagnostic wrapper or provider transport boundary. Add focused tests proving size/timing fields are emitted without raw model content or secrets. If maintainers decide existing OTEL/Prometheus duration telemetry is the intended replacement, document that product decision before closing.
