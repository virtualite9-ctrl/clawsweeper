## Summary

Keep #33832 open. Current main wraps model stream calls with sanitized diagnostic lifecycle events and total duration telemetry, but it still does not log request body size, response body size, or time-to-first-byte for model API calls. The linked implementation PR #35513 was closed unmerged, so the requested Gateway logging remains unshipped.

## Best Possible Solution

Keep #33832 open until OpenClaw core adds privacy-safe model API request/response byte counts plus first-byte and total latency logging or telemetry, likely near the existing model-call diagnostic wrapper or provider transport boundary. Add focused tests proving the size/timing fields are emitted without raw model content or secrets. If maintainers choose OTEL/Prometheus duration telemetry as the supported replacement instead, document that product decision before closing.
