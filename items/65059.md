## Summary

Keep open. Current main still hardcodes MiniMax usage polling to the CN host and does not pass configured endpoint data into `fetchMinimaxUsage`, so the user-visible bug linked from this PR is not implemented on main. This PR remains a plausible implementation candidate, but the current diff needs maintainer review or revision because the proposed host lookup can diverge from the credential source and lacks URL-routing regression coverage.

## Best Possible Solution

Keep this PR open for maintainer review or author revision. The best fix is to consolidate the overlapping MiniMax usage endpoint work with #63073 and #65054, derive the usage host from the same credential/config source that supplied the token, preserve intentional CN/global fallback behavior, and add focused tests that capture the requested URL for global, CN, missing/invalid baseUrl, and both-provider-config cases.
