## Summary

Keep #38439 open. Current main has the `/avatar/<agentId>` route, IDENTITY.md fallback, avatar metadata, and `too_large` reporting, but the Windows namespace path-containment bug described in the discussion is still present in source and the related non-default-agent/backtick regression work remains unmerged in the provided GitHub context.

## Best Possible Solution

Keep #38439 open until #38923 or an equivalent maintainer-reviewed fix lands. The fix should normalize Windows namespaced and drive-letter paths before avatar containment checks, preserve the existing workspace-boundary protections and `too_large` metadata, strip single-backtick wrappers from IDENTITY.md avatar values if accepted as part of the related cluster, and add regression coverage for a non-default agent such as `gaia` resolving a workspace-relative IDENTITY.md avatar on Windows-style paths.
