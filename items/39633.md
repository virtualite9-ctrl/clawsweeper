## Summary

Current `main` already implements provider-level cooldown/circuit-breaker behavior for model fallback: provider/profile failure state is tracked, providers with all profiles unavailable are skipped or probed again near expiry, cooldown state resets after expiry/success, and the behavior is configurable and documented.

## Close Comment

Closing from a Codex review because current `main` already implements this behavior. Provider/model fallback now tracks provider auth-profile cooldown state, skips providers whose profiles are all unavailable, and re-probes them near cooldown expiry instead of continuing to hammer the same failing provider (`src/agents/model-fallback.ts:523`, `src/agents/model-fallback.ts:700`). The recovery path is also in place: expired cooldowns are cleared and success resets the failure state (`src/agents/auth-profiles/usage.ts:410`, `src/agents/auth-profiles/usage.ts:487`). The config and behavior are documented in `docs/gateway/configuration-reference.md:714` and `docs/concepts/model-failover.md:213`. This landed on `main` in `7d471837369885c12d540d6bafa319ff41129b89`, after `v2026.4.22`, so there is not a released tag for it yet. If there is still a missing gap beyond this shipped provider cooldown/probe/fallback behavior, please open a focused follow-up with that specific scenario.
