## Summary

Keep open. Current main still matches the reported risk: host exec defaults to `security="full"` and `ask="off"` on gateway/node, approval is not required in that mode, and I found no hard runtime guard for package-manager self-removal commands such as `npm uninstall -g openclaw` or near-name confusion with `opencli`. The related open #54416 is adjacent exec-safety work, but it does not fully supersede this self-uninstall report.

## Best Possible Solution

Keep this issue open and make an explicit maintainer decision in the core exec safety boundary. The likely fix belongs in the exec approval/analysis path: require confirmation or deny high-risk host commands, especially package-manager removal of OpenClaw itself and confusing near-name targets, with regression coverage for gateway/node `full/off`, allowlist/on-miss, sandbox behavior, and the adjacent #54416 self-management/config-command class. If maintainers intentionally accept this in YOLO mode, document that risk plainly instead of closing as fixed.
