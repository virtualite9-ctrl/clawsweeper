## Summary

Keep open. Current main still has the same core behavior described in the issue: gateway/node host exec defaults to `security="full"` and `ask="off"`, approval is only required for `ask="always"` or allowlist misses, and there is no hard guard for package-manager self-uninstall commands such as `npm uninstall -g openclaw`. The related open #54416 is adjacent exec-safety work, but it does not fully supersede this self-removal report.

## Best Possible Solution

Keep this issue open and make an explicit maintainer decision in the exec safety boundary. The likely product/code path is an execution-time confirmation or denial rule in the core exec approval/analysis path for high-risk host commands, especially package-manager removal of OpenClaw itself and confusing near-name package targets, with regression coverage for gateway/node `full/off`, `allowlist/on-miss`, sandbox behavior, and any plugin hook interaction. If maintainers intentionally accept this in YOLO mode, document that exact risk plainly instead of closing as fixed.
