## Summary

Keep open. Current main still lacks per-dialed-number voice/persona routing in the bundled voice-call plugin: config, manifest, runtime response generation, TTS selection, tests, and docs all remain global even though provider events preserve the dialed `to` number.

## Best Possible Solution

Keep this issue open and implement the feature inside the bundled `voice-call` plugin. Add a typed, documented per-number route table to the config schema and manifest; normalize and match inbound `event.to` at call creation; store a stable per-call effective config or route id; and make inbound greeting, agent/model/system prompt, realtime paths, and TTS voice selection use that effective config with global fallback. Add focused tests for Twilio/Telnyx/Plivo `to` routing, fallback behavior, and per-number overrides.
