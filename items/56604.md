## Summary

Keep open. Current main still captures provider-supplied dialed numbers, but the bundled Voice Call plugin has no typed per-number route table and still resolves inbound greetings, response agent/model/prompt, realtime instructions, and TTS voice from the global plugin config. The recent maintainer comment also explicitly kept this open, and related #59245 is broader outbound task-call behavior rather than a duplicate for inbound per-number routing.

## Best Possible Solution

Keep this issue open and implement it in the bundled voice-call plugin: add a typed and documented per-number route table to the config schema and manifest, normalize event.to at call creation for supported providers, store a stable per-call route id or effective config, and make inbound greeting, response agent/model/system prompt, realtime instructions/consult behavior, and TTS voice selection use that effective config with global fallback. Add focused tests for per-number matches, fallback behavior, provider To-number normalization, and stable per-call routing.
