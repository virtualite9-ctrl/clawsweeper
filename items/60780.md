## Summary

Keep open. Current `main` still has the two-path delivery shape from the report: thread-bound ACP session spawns inline-deliver into the bound thread, and the same ACP spawn is also tracked as a completion-announcing subagent run. The completion delivery path still resolves bound-session completions back to the active thread binding, so the issue's duplicate-thread-response mechanism is still present in code.
