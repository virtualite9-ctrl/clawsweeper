## Summary

Keep open. Current main still resolves `openclaw skills install` and `update` through `resolveDefaultAgentId(config)` instead of inferring the agent from `process.cwd()`, while #72726 is an open, unmerged implementation candidate that uses closing syntax for this issue. Per review policy, this should stay open until that PR or a replacement fix lands.

## Best Possible Solution

Keep this issue open and review/land or supersede #72726. The best fix is a shared skills workspace resolver that prefers an explicit agent override if accepted, then `resolveAgentIdByWorkspacePath(config, process.cwd())`, then the default agent fallback, with regression tests for install/update and likely the read-only skills commands for consistency.
