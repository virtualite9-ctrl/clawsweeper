## Summary

Keep this PR open. Current main still preserves `sourceConfig` at the status overview layer but drops it before the status memory probe boots the memory runtime, so the PR remains a plausible fix for the false-positive `plugins.allow is empty` warning reported in #62049.

## Best Possible Solution

Keep this PR open and review a scoped rebase. The best implementation is to thread `overview.sourceConfig` as an internal `activationSourceConfig` through `executeStatusScanFromOverview`, `resolveStatusMemoryStatusSnapshot`, `resolveSharedMemoryStatusSnapshot`, `getActiveMemorySearchManager`, and the memory runtime bootstrap, while keeping plugin hook arguments limited to the declared `{ cfg, agentId, purpose }` contract and preserving omission of optional `purpose`. Split unrelated extension/type cleanup out before landing, then run the focused status and memory-runtime regression tests plus the changed gate.
