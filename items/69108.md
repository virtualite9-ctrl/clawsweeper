## Summary

Keep open. Current main already contains the `--agent` + `--session-id` precedence fix and regression coverage, so that part of the draft PR is obsolete. The PR's other user-visible scope, `openclaw agent --new-session`, is still absent from current main CLI registration, command option types, docs, changelog, and tests, while the linked feature request #54864 remains open. This is a focused recent PR with useful remaining implementation work rather than a closeable duplicate or fully implemented item.

## Best Possible Solution

Keep the PR open for maintainer review, but scope it to the remaining `--new-session` feature: generate a UUID once in the top-level `agentCliCommand` path, reject simultaneous `--new-session` and `--session-id`, pass the effective session id through both gateway and embedded fallback paths, return it in JSON in a script-friendly way, document it in `docs/cli/agent.md`, and add focused CLI/command tests. The already-shipped `--session-id` precedence fix on main should remain as-is.
