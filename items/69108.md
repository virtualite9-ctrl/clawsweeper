## Summary

Keep open. Current main already contains the `--agent` + `--session-id` isolation fix and regression coverage, so that part of #69108 is obsolete. The PR's other user-visible scope, `openclaw agent --new-session`, is still absent from the CLI registration, shared command option type/path, docs, changelog, and tests. The linked feature request #54864 remains open, so this recent draft PR still contains potentially useful remaining work rather than being fully implemented or safely superseded.

## Best Possible Solution

Keep #69108 open for maintainer review, but narrow it to the remaining `--new-session` feature: generate one UUID in the shared CLI path before gateway/local dispatch, reject simultaneous `--new-session` and `--session-id`, pass the effective session id through both gateway and embedded fallback paths, return the generated id in JSON in a script-friendly shape, document it in `docs/cli/agent.md`, and add focused CLI/command tests. Leave the already-shipped session-id precedence fix on main as-is.
