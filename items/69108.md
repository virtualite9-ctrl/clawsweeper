## Summary

Keep open. Current main already fixes the `--agent` + `--session-id` collapse by resolving caller-supplied session ids before the agent main-session fallback and by scoping agent-specific lookup to the requested agent store. However, the PR's second user-visible feature, `openclaw agent --new-session`, is still not implemented or documented on main, and the related #54864 remains the canonical open feature request. The useful path is not to auto-close this PR outright, but to have a maintainer decide whether to extract/rebase the `--new-session` portion against the newer session resolver from #70985.

## Best Possible Solution

Keep the PR open for maintainer judgment or ask the contributor to rebase and narrow it to the still-missing `--new-session` behavior. The session-id isolation bug should stay with the current main implementation from #70985; the remaining product/API work belongs either in this PR after rebase or in the canonical open feature request #54864.
