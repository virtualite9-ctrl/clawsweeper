## Summary

Current main does not implement the requested context-saving behavior. `config.patch` still returns the redacted full config on both no-op and write success paths, `config.get` still returns the full snapshot, and the owner-only agent `gateway` tool serializes the full gateway result back into the agent tool output. The related comment's note that the gateway tool has the same bloat path is accurate.

## Best Possible Solution

Keep the issue open. The best fix is a protocol-aware change that makes `config.patch` return a compact success payload such as `ok`, `changedPaths`, `restart`, and sentinel/restart metadata without echoing the full config, while leaving full snapshots available through `config.get`. Add regression coverage for the direct gateway RPC and the agent-facing `gateway` tool output, and consider a separate additive `config.get` path filter if maintainers want the commenter's scoped-read extension.
