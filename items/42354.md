## Summary

Keep this issue open. Current main still supports Feishu WebSocket mode as the default path, but it does not include an OpenClaw-owned heartbeat/reconnect repair for the reported startup/long-connection failure, and the related maintainer implementation candidate #72411 remains open rather than shipped.

## Best Possible Solution

Keep this issue open and treat #72411 or a successor maintainer-approved PR as the active fix path. The fix should validate current @larksuiteoapi/node-sdk behavior, wire wsClient.start settlement into an abort-aware reconnect/backoff supervisor in the Feishu plugin, add focused regression tests for startup failure/retry behavior, and clarify Feishu versus Lark domain/webhook requirements in the Feishu docs if that remains a user-facing workaround.
