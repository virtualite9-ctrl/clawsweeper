## Summary

Keep open. Current main still does not implement the Feishu WebSocket heartbeat/reconnect fix reported here: the Feishu client factory still constructs the Lark WSClient without local heartbeat config or reconnect policy, the monitor still starts one client once without awaiting/retrying async start failure, and the active maintainer PR #72411 remains open rather than shipped.

## Best Possible Solution

Keep this issue open and use #72411 or a successor maintainer-approved PR as the active fix path. The fix should verify the current @larksuiteoapi/node-sdk behavior, implement an abort-aware Feishu WebSocket supervisor/retry path in extensions/feishu/src/monitor.transport.ts if needed, add focused regression coverage for createFeishuWSClient and monitorWebSocket start/reconnect failures, and clarify Feishu versus Lark webhook/domain requirements in the Feishu docs if that remains a user-facing workaround.
