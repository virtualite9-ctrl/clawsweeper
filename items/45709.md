## Summary

Keep #45709 open. Current main still has the duplicate Usage header composition: the shell renders the shared tab title/subtitle for every non-config tab, including Usage, and the Usage view also renders its own Usage title/subtitle block. The earlier linked fix PR #45715 was closed unmerged, and the newer exact fix PR #67290 is still open and not shipped.

## Best Possible Solution

Keep #45709 open until a focused UI fix lands on main. The fix should deduplicate the Usage tab header by either suppressing the shared shell header for Usage or removing/adapting the local Usage header, while preserving mobile context and keeping i18n source/generated files aligned. PR #67290 is the current candidate to resolve this, but it is not merged.
