## Summary

Keep #71889 open. Current main still has the reported behavior: setup/onboarding can persist the main workspace as `agents.defaults.workspace`, the interactive `agents add` wizard uses `resolveAgentWorkspaceDir(cfg, agentId)` as the "Workspace directory" prompt default, and the resolver joins non-default agent ids under `agents.defaults.workspace`, producing `~/.openclaw/workspace/<id>` instead of the documented `~/.openclaw/workspace-<agentId>`. The related fix PRs #71899 and #71941 are open and unmerged, so they are useful implementation paths rather than shipped/superseding evidence for closing the issue.

## Best Possible Solution

Keep #71889 open as the canonical bug until a fix lands. The best path is a narrow `agents add` prefill helper or equivalent that suggests `~/.openclaw/workspace-<agentId>` only when `agents.defaults.workspace` is the normal main workspace, preserves explicit per-agent workspaces and intentionally custom shared bases, and adds regression coverage in both resolver/helper tests and `agents.add` wizard tests.
