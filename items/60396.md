## Summary

Current main appears to fix the reporter's DashScope/OpenAI-compatible root cause by always requesting streaming usage for native DashScope endpoints, but the broader feature request here, generic fallback token estimation when providers return no usable usage data, is still not implemented in the session/TUI path.

## Evidence

- **Issue discussion split the problem in two:** The maintainer comments on #60396 say the reporter's Bailian/DashScope case likely comes from missing `stream_options.include_usage` (#59328) or DashScope usage parsing (#52346), while the broader estimation idea overlaps #48365.
- **OpenAI-completions transport now requests usage:** `buildOpenAICompletionsParams` on current `main` always sends `stream_options: { include_usage: true }`, which addresses the DashScope/OpenAI-compatible streaming-usage gap described in the issue discussion.
