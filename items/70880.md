## Summary

The request is coherent, but it targets email-specific MCP tool UX that OpenClaw currently scopes to plugins or bundles rather than core. Current `main` only ships generic MCP conversation bridge tools plus plugin-owned MCP tool exposure, so this belongs in a ClawHub plugin/bundle or as a plugin-API proposal if existing seams prove insufficient.

## Evidence

- **Vision scopes optional MCP capability to plugins/bundles, not core:** `VISION.md` says core should stay lean, optional capability should usually ship as plugins, bundle-style plugins package MCP servers, and OpenClaw should avoid duplicating existing plugin or ClawHub paths.
  - file: [VISION.md:54](https://github.com/openclaw/openclaw/blob/89755d1c79678610720cdb9ce504cfd62c261f64/VISION.md#L54)
  - sha: [89755d1c7967](https://github.com/openclaw/openclaw/commit/89755d1c79678610720cdb9ce504cfd62c261f64)
- **Plugin docs explicitly route new tools to plugins published on ClawHub or npm:** `docs/plugins/building-plugins.md` states plugins can add tools and that authors do not need to add them to the OpenClaw repository; they should publish to ClawHub or npm. The quick-start shows `api.registerTool(...)` as the supported path.
  - file: [docs/plugins/building-plugins.md:11](https://github.com/openclaw/openclaw/blob/89755d1c79678610720cdb9ce504cfd62c261f64/docs/plugins/building-plugins.md#L11)
  - sha: [89755d1c7967](https://github.com/openclaw/openclaw/commit/89755d1c79678610720cdb9ce504cfd62c261f64)
- **Bundle docs already define MCP-server-as-plugin distribution:** `docs/plugins/bundles.md` documents that enabled bundles can contribute MCP server config and OpenClaw exposes supported bundle MCP tools during agent turns, which matches the requested email-specific MCP behavior living outside core.
  - file: [docs/plugins/bundles.md:73](https://github.com/openclaw/openclaw/blob/89755d1c79678610720cdb9ce504cfd62c261f64/docs/plugins/bundles.md#L73)
  - sha: [89755d1c7967](https://github.com/openclaw/openclaw/commit/89755d1c79678610720cdb9ce504cfd62c261f64)
- **Current core MCP server is a generic conversation bridge, not an email MCP server:** `docs/cli/mcp.md` and `src/mcp/channel-tools.ts` show the current `openclaw mcp serve` toolset: `conversations_list`, `conversation_get`, `messages_read`, `attachments_fetch`, `events_poll`, `events_wait`, `messages_send`, `permissions_list_open`, and `permissions_respond`. None are email-specific search/content tools.
  - file: [src/mcp/channel-tools.ts:23](https://github.com/openclaw/openclaw/blob/89755d1c79678610720cdb9ce504cfd62c261f64/src/mcp/channel-tools.ts#L23)
  - sha: [89755d1c7967](https://github.com/openclaw/openclaw/commit/89755d1c79678610720cdb9ce504cfd62c261f64)
- **Built-in MCP tool server is intentionally tiny:** `src/mcp/openclaw-tools-serve.ts` currently exposes only the built-in `cron` tool through the core tools MCP server, reinforcing that OpenClaw core is not shipping an email-tool surface here.
  - file: [src/mcp/openclaw-tools-serve.ts:14](https://github.com/openclaw/openclaw/blob/89755d1c79678610720cdb9ce504cfd62c261f64/src/mcp/openclaw-tools-serve.ts#L14)
  - sha: [89755d1c7967](https://github.com/openclaw/openclaw/commit/89755d1c79678610720cdb9ce504cfd62c261f64)
- **Plugin-owned MCP metadata already controls tool names and descriptions:** `src/mcp/plugin-tools-serve.ts` loads plugin-registered tools, and `src/mcp/plugin-tools-handlers.ts` forwards each tool's own `name`, `description`, and `inputSchema` to MCP clients. That is the right layer for improving email-tool discoverability, filters, and summarized outputs.
  - file: [src/mcp/plugin-tools-handlers.ts:21](https://github.com/openclaw/openclaw/blob/89755d1c79678610720cdb9ce504cfd62c261f64/src/mcp/plugin-tools-handlers.ts#L21)
  - sha: [89755d1c7967](https://github.com/openclaw/openclaw/commit/89755d1c79678610720cdb9ce504cfd62c261f64)
- **Requested email tool names do not exist on current main:** A source search over `src`, `docs`, `extensions`, `VISION.md`, and `CHANGELOG.md` found no `search_emails`, `search_all_emails`, or `get_email_content` identifiers in the repository at `89755d1c`.
  - command: `rg -n "search_emails|search_all_emails|get_email_content" -S src docs extensions VISION.md CHANGELOG.md`
  - sha: [89755d1c7967](https://github.com/openclaw/openclaw/commit/89755d1c79678610720cdb9ce504cfd62c261f64)

## Risks / Open Questions

- If the reporter actually meant a separate external MCP email server, the right follow-up is in that plugin/server repository rather than OpenClaw core.
