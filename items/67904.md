## Summary

Keep open. Current main appears to solve the immediate `speech-core/runtime-api.js` parity failure through a narrow hardcoded runtime-core allowlist, but it does not implement the PR’s final metadata-backed manifestless-package facade handling. The PR still contains potentially useful work: deriving manifestless runtime facade identity from `@openclaw/<dir>` package metadata, avoiding fallback for invalid manifests, and replacing the hardcoded core list with explicit package metadata.

## Best Possible Solution

Keep the PR open for maintainer judgment or a follow-up patch that replaces the current hardcoded runtime-core facade allowlist with explicit manifestless package metadata, validates `package.json.name === @openclaw/<dir>`, rejects invalid manifest-backed packages instead of falling back, and preserves the existing regression coverage for the parity-gate `speech-core/runtime-api.js` failure.
