## Summary

Keep open. Current main still exposes the reported bug path: the session-memory slug generator runs `runEmbeddedPiAgent` on `temp:slug-generator` without `authProfileFailurePolicy: "local"`, while the embedded runner supports that helper-local policy and otherwise persists classified billing/auth/rate-limit failures into shared auth-profile state. The classifier still intentionally treats Anthropic extra-usage messages as billing, so an internal slug-generation failure can still become a profile-wide billing cooldown.

## Best Possible Solution

Keep the issue open and fix `generateSlugViaLLM` by passing `authProfileFailurePolicy: "local"` or an equivalent helper-local failure scope to `runEmbeddedPiAgent`. Add a focused regression test in `src/hooks/llm-slug-generator.test.ts` that asserts the slug generator uses the local policy, and ideally a seam test proving billing/auth-classified slug failures do not call `markAuthProfileFailure` for the shared auth profile. Leave the broader billing classifier behavior unchanged unless maintainers make a separate product decision.
