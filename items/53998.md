## Summary

Keep open. Current main still matches the reported secrets-audit false positives for bare `$VAR` references: the parser only recognizes `${VAR}`, config audit reports non-ref non-empty secret strings as `PLAINTEXT_FOUND`, and the auth-profile audit path still falls through to plaintext detection after recording a ref. The configure/apply mixed-profile type mismatch also remains fatal. The linked #54141 is still an open implementation candidate with closing intent, so this issue should stay open until that PR is landed or closed.

## Best Possible Solution

Keep this issue open while #54141 is reviewed and either landed or closed. The best maintainer path is a focused fix that recognizes bare `$VAR` and `${VAR}` env refs consistently in config and auth-profile audit paths, prevents auth-profile ref findings from falling through to `PLAINTEXT_FOUND`, and then splits any remaining `secrets configure` mixed-profile handling, scrub behavior, and rotation guidance into narrower follow-up work if maintainers still want those changes.
