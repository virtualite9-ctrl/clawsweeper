## Summary

Keep open. Current main still does not recognize bare `$VAR` as a SecretRef, and auth-profile audit still records a detected ref then falls through to `PLAINTEXT_FOUND`. There is also an open unmerged PR with closing syntax for this issue, so the issue should remain open until that PR is reviewed/landed or explicitly rejected.

## Best Possible Solution

Keep the issue open and use #54141 as the implementation candidate for the core `$VAR` false-positive path. Maintainers should review or land that PR, then either verify the remaining configure/OAuth and `.env` audit surfaces are covered with tests or split those leftovers into narrower follow-up issues before closing this one.
