## Summary

Keep #71593 open. Current main has useful SecretRef primitives, including env/file/exec providers and manifest-declared plugin config SecretInput paths, but it does not implement plugin-defined SecretRef providers or universal core schema coverage. The requested gap remains visible in closed SecretRef source schemas, resolver dispatch, protocol schemas, and unsupported/uncovered surfaces such as `skills.entries.*.env`. The linked AWS Secrets Manager PR #43643 was closed unmerged, so it does not supersede this issue.

## Best Possible Solution

Keep this open as a core/plugin API design issue. Maintainers should decide whether `exec` providers are the intended extensibility answer and document that explicitly, or add a versioned plugin SecretRef-provider contract. If adding the contract, update SecretRef types, config schemas, Gateway protocol schemas, resolver/plan validation, docs, tests, and the remaining requested SecretInput surfaces such as `skills.entries.*.env`; also document how plugin manifests declare config SecretInput paths.
