## Summary

Keep #48279 open. Current main still exposes `unstable_listSessions` on the ACP agent instead of `listSessions`, while OpenClaw is pinned to ACP SDK 0.20.0 and that SDK dispatches `session/list` by checking `agent.listSessions`. The related fix PR #48292 was closed unmerged, and I did not find a current source/test implementation that resolves the reported regression.

## Best Possible Solution

Implement the ACP bridge fix in `src/acp/translator.ts` by exposing `listSessions` for `session/list` under the current ACP SDK, optionally retaining `unstable_listSessions` as a compatibility alias only if OpenClaw still needs to support older SDKs. Add a regression test that exercises SDK-style `session/list` dispatch or directly verifies the public handler name, then align the ACP docs claim with the tested behavior.
