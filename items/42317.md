## Summary

Keep this PR open. Current main still does not honor configured session.mainKey in the core runtime routing path, and the PR remains the focused implementation candidate for the linked feature request. It is not mergeable as-is without rebase/path updates and the unresolved custom-main-key delivery guard, but it is still useful work rather than obsolete or closeable.

## Best Possible Solution

Keep this PR open as the canonical implementation candidate. Maintainers should rebase it onto current main, port the Telegram and WhatsApp hunks to the plugin paths, preserve configured session.mainKey through runtime defaults and every route/session-key builder, update custom-main-key recognition in session-delivery guards, align docs/schema help, and then run the changed config/routing/plugin tests before merge or requested revisions.
