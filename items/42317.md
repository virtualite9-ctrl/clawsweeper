## Summary

Keep this PR open. Current main still does not honor `session.mainKey` for the core routing path: runtime config materialization overwrites any configured value to `"main"`, and `resolveAgentRoute()` still builds both peer and main session keys with `DEFAULT_MAIN_KEY`. The linked feature issue was closed as superseded by this PR, not because the behavior shipped. The PR is a plausible implementation candidate, but it needs maintainer review/rebase against current main and should address the remaining main-session guard concern before landing.

## Best Possible Solution

Keep the PR open as the canonical implementation candidate for custom `session.mainKey` routing. The maintainer path is to rebase it onto current main, move the Telegram/broadcast changes to the current files or remove obsolete hunks, add/adjust coverage for main-session guard behavior in `session-delivery.ts`, align docs/config help, then run the changed routing/config/plugin tests before merging or requesting revisions.
