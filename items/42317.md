## Summary

Current main still does not implement the PR's central behavior. Runtime config still rewrites custom session.mainKey to "main", core routing and several channel/outbound route builders still use DEFAULT_MAIN_KEY, and the PR remains a focused implementation candidate. It needs a rebase/path update plus the unresolved custom-main-key delivery guard before maintainers can merge it safely.

## Best Possible Solution

Keep this PR open as the canonical implementation candidate. Maintainers should rebase it onto current main, port channel hunks to the current plugin paths, preserve configured session.mainKey through runtime defaults and every route/session-key builder, update delivery guards to recognize custom main keys, align docs/schema help, and then run the changed config/routing/plugin tests before merge or requested revisions.
