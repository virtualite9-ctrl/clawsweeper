## Summary

Keep this PR open. Current main still writes initial input to a child process stdin after only checking that the stream exists, and the current test suite does not include the non-writable stdin/EPIPE regression case the PR targets. The PR is a focused two-file bug-fix candidate for a real CLI backend path.

## Best Possible Solution

Keep this PR open for maintainer review. The likely best path is to land a scoped fix in `src/process/supervisor/adapters/child.ts` that prevents initial stdin write/end from crashing failed CLI spawns, with regression coverage in `src/process/supervisor/adapters/child.test.ts` for non-writable or throwing stdin behavior.

## Risks / Open Questions

- The proposed test models non-writable stdin by overriding a PassThrough property, so maintainers may still want a stronger regression that better represents a closed or EPIPE stream.
- The one-line guard only handles streams that are already non-writable; a stream that reports writable and then throws during the initial write/end path may still need broader try/catch handling.
