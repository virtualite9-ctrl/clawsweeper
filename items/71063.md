## Summary

Keep #71063 open. Current main and latest release still perform the initial child stdin write/end when `child.stdin` merely exists, with no `writable` guard or regression test for this failure mode. The PR is a focused bug-fix with test coverage and addressed bot feedback, though the exact fix still deserves maintainer validation against async `EPIPE` from the Linux shell wrapper path.

## Best Possible Solution

Keep the PR open for maintainer review. The right fix belongs in the process supervisor child adapter or adjacent CLI-runner stdin handling: prevent unsafe initial stdin write/end on failed child processes, handle stdin stream errors where necessary, and land regression coverage for both non-writable stdin and the Linux OOM-shell missing-binary async `EPIPE` path.
