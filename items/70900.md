## Summary

Keep this PR open. Current main still has the unguarded assistant `surface_error` throw path, while the surrounding code can still produce `failoverFailure=false` with a classified `failoverReason` from stale assistant error text. The PR remains a focused implementation candidate rather than obsolete or superseded.

## Best Possible Solution

Keep this PR open and either merge it or land an equivalent minimal fix in `src/agents/pi-embedded-runner/run/assistant-failover.ts`: require `params.failoverFailure` before throwing `FailoverError` from the assistant `surface_error` branch, while preserving #70848/#70124 behavior for genuine provider failures with `stopReason: "error"`. Add the proposed focused regression test for `failoverFailure=false` plus classified `failoverReason`, then run the changed runner tests before landing.
