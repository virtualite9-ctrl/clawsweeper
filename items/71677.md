## Summary

Keep open. Current main still routes Feishu card.action.trigger callbacks through a Promise<void> handler that awaits handleFeishuCardAction and returns no callback response. The handler deduplicates repeated tokens, but there is no ocf1 disable-state field, no transitional disabled-card return, and no test/docs evidence that instant card updates are implemented.

## Best Possible Solution

Keep the issue open and implement this in the bundled Feishu plugin: extend the ocf1 interaction contract with an opt-in disable-state field or equivalent typed metadata, make card.action.trigger return an immediate Feishu card callback response while dispatching the synthetic command asynchronously, preserve duplicate-token protection, and add focused tests for the returned WS/webhook callback payload plus no-regression tests for existing quick-action and approval cards.
