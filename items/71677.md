## Summary

Keep open. Current main still handles Feishu card.action.trigger callbacks as a void handler: it deduplicates callback tokens and dispatches synthetic commands, but it does not return an immediate Feishu callback payload, does not dispatch asynchronously for instant UI feedback, and has no ocf1 disable-state field or disabled-card helper.

## Best Possible Solution

Keep the issue open and implement this in the bundled Feishu plugin: extend the ocf1 interaction contract with opt-in disable-state metadata, make card.action.trigger return an immediate Feishu callback response while dispatching the synthetic command asynchronously, preserve duplicate-token protection, and add focused tests for WS/webhook returned callback payloads plus quick-action and approval-card regressions.
