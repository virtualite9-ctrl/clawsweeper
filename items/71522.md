## Summary

Keep open. The report is new, detailed, and still plausible on current `main`. I found no main-branch fix for OpenAI-compatible remote memory indexing against OpenRouter; the nearby shipped fix in #70836 was explicitly limited to the built-in `local` embedding provider. One reported symptom is misinterpreted, though: with `memorySearch.remote.batch.enabled = false`, current code will intentionally show `Batch: disabled (failures 0/2)`, so that part does not indicate a sticky persisted failure state.
