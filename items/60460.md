## Summary

Current `main` already enforces CDP SSRF policy during browser profile creation, rejects strict-mode private-network `cdpUrl` values before persisting config, and preserves the fast `driver=existing-session` validation path. This PR's core behavior is present on `main` in commit `51c11cfd903033947f01f96a7169ebf92d194064`, so the PR can be closed as already implemented.

## Evidence
