## Summary

Keep this PR open. Current main still indexes memory through the existing fixed-size `chunkMarkdown(content, { tokens, overlap })` path, and the multi-strategy config, strategy resolver, strategy implementations, and per-strategy reindex metadata proposed here are not shipped or clearly superseded. The related memory chunking items overlap, but they remain open and none is a merged canonical replacement for this PR.

## Best Possible Solution

Keep this PR open as an implementation candidate. Maintainers should decide the product/API direction for bundled memory-core chunking, coordinate it with #44395, #48795, #48808, and #53650, require the bot review conversations to be resolved or re-reviewed, and validate the final shape with focused tests for default fixed-size compatibility, strategy-specific chunking, schema/help drift, LLM-assisted failure behavior, and full-reindex metadata changes.
