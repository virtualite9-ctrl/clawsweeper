## Summary

Current main still uses the fixed `chunkMarkdown(content, { tokens, overlap })` memory indexing path and exposes only token/overlap chunking config. The multi-strategy framework in this PR is not shipped or clearly superseded, and the related chunking items remain open rather than merged canonical replacements.

## Best Possible Solution

Keep this PR open as an implementation candidate. Maintainers should choose the memory-core chunking direction across #44395, #48795, #48808, and #53650, decide whether this should land as one broad feature or be split into smaller staged changes, require the review conversations to be resolved on the latest head, and validate default fixed-size compatibility, strategy-specific config/schema/docs, reindex metadata migration, and LLM-assisted error/data-egress behavior before merge.
