## Close Comment

Closing from a Codex review: current `main` already contains the packaged-install fix for this failure class. The reported error text comes from `src/gateway/server-methods/web.ts:13`, where the gateway says `web login provider is not available` only when no loaded channel plugin exposes the QR login methods. The bundled WhatsApp plugin is that provider in `extensions/whatsapp/src/shared.ts:213`, and packaged startup now repairs bundled plugin runtime dependencies before import in `src/plugins/loader.ts:2202`, with regression coverage in `src/plugins/loader.test.ts:1241`. This was shipped in `v2026.4.20` and called out in `CHANGELOG.md:410`, including WhatsApp specifically. If this is still reproducible on `v2026.4.20` or newer, please open a fresh issue with the exact Umbrel app version and gateway startup/plugin-loading logs.
