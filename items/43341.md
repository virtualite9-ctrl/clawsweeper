## Summary

Current main still does not implement the reported bundled TUI live transcript sync. The gateway has session transcript broadcast primitives, and Control UI consumes them, but the TUI neither subscribes to nor handles `session.message` / `sessions.changed`. #43918 fixed a related outbound route-inheritance bug, and #51691 overlaps as a broader open tracker, but neither proves this Telegram-to-TUI live-update regression is resolved.

## Best Possible Solution

Keep this issue open and implement bundled TUI live transcript sync: subscribe to the active canonical session with `sessions.messages.subscribe` on connect and session changes, handle `session.message` with active-run deferral semantics similar to Control UI, update session metadata from `sessions.changed`, and add focused TUI regression tests. Coordinate with #51691 as the broader tracker if maintainers want to consolidate, but choose the canonical issue explicitly before closing either one.
