## Summary

Keep this issue open. Current main still exposes the gateway transcript event stream, and Control UI consumes it, but the bundled TUI still does not subscribe to `sessions.subscribe` / `sessions.messages.subscribe` or handle `session.message` / `sessions.changed`. The standalone TUI linked in the issue discussion is a useful external workaround, not evidence that the bundled OpenClaw TUI regression is fixed.

## Best Possible Solution

Keep this issue open. Implement bundled TUI live transcript sync by adding TUI backend subscription support, subscribing to the active canonical session on connect and session changes, handling `session.message` with active-run deferral/history reload semantics, applying `sessions.changed` metadata, and adding focused TUI regression tests. Treat the external `oclaw` project as useful prior art/workaround, not as a reason to close the core bundled TUI regression. Coordinate with #51691 only if maintainers explicitly choose that broader issue as the canonical tracker.
