## Summary

Current main still lacks an operator-configurable browser URL/domain/path denylist or warnlist. The browser navigation policy remains SSRF/private-network focused, and the existing URL-pattern helper is wired to response matching, not navigation refusal or warnings.

## Best Possible Solution

Keep #39097 open. The best implementation is a maintainer-approved browser URL policy surface, likely `browser.denylist` or `browser.blockedUrls` plus optional `warnlist`, wired through typed config, strict zod/generated schema/help, browser config resolution, shared navigation policy options, managed Playwright and existing-session Chrome MCP navigation guards, clear user-facing errors/warnings, docs, and focused tests for exact host, wildcard domain, and path-pattern behavior.
