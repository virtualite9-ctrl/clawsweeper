## Summary

Keep #16670 open. Current main has memory-search config types, docs, auto-detection, status/doctor diagnostics, and SecretRef plumbing, but the onboarding/setup and configure wizards still do not include the requested mandatory memory/embedding setup step for provider, model, API key or SecretRef, plus endpoint verification. The latest issue discussion also contains a maintainer/Codex review explicitly keeping the issue open for this same remaining gap.

## Best Possible Solution

Keep #16670 open until OpenClaw either adds a dedicated memory/embedding setup path or a maintainer explicitly decides against making it mandatory. The likely implementation belongs in onboarding/configure: use the existing memory embedding provider registry and config schema, write `agents.defaults.memorySearch` provider/model/API-key or SecretRef settings, verify readiness with a small embedding probe or `openclaw memory status --deep`, and update public docs to clarify API-key requirements, Codex OAuth limitations for embeddings, and daemon/LaunchAgent environment visibility.
