## Summary

Keep this PR open. Current main still has the Feishu mention-cascade behavior: inbound mentionTargets are forwarded into reply dispatch and then converted into outbound Feishu @mention markup. The PR is a recent, focused implementation candidate for linked bug #70065, and the latest discussion shows active author follow-up plus a maintainer ping rather than abandonment.

## Best Possible Solution

Keep this PR open for Feishu plugin maintainer review. The best path is to land or revise a focused Feishu plugin fix that stops unconditional automatic mention propagation for fallback, error, ack, ordinary plain-text, card, and streaming-card replies; keeps mention context sanitized and clearly context-only for the agent; and makes an intentional call on explicit mention conversion in card/streaming replies with regression coverage.
