## Summary

Keep #71396 open. Current main still forwards inbound Feishu mention targets into outbound reply dispatch and still tells the agent replies will automatically @mention targets, so the PR is not obsolete or implemented on main. The PR also has unresolved review/security discussion around preserving explicit card mention conversion and sanitizing mention names before this can be safely merged.

## Best Possible Solution

Keep #71396 open and have the Feishu plugin maintainer finish the bundled-plugin fix: stop unconditional mention propagation for fallback/error/ack/ordinary replies, preserve deliberate explicit mention behavior with stable IDs and safe card token conversion, sanitize or structure mention context before it reaches the model prompt, and keep regression coverage for plain text, card, streaming-card, and fallback paths.
