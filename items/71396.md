## Summary

Keep this PR open. Current main at 66f4b52db31ba2b0d61c74b184a23f26d31edce3 still contains the Feishu mention-cascade behavior: inbound mentionTargets are extracted, passed into the reply dispatcher, and then used to prepend outbound Feishu mention markup on first text/card replies and streaming-card close. The PR is recent, unmerged, Feishu-scoped, and remains the active implementation candidate for linked bug #70065 rather than obsolete cleanup.

## Best Possible Solution

Keep this PR open for Feishu plugin maintainer review. The best path is to revise or land the bundled Feishu plugin fix so fallback, error, ack, plain-text, card, and streaming-card replies no longer cascade inbound mentionTargets automatically, while prompt mention context is size-bounded and any intentional explicit mention behavior remains covered by regression tests.
