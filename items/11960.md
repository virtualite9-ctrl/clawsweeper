## Summary

Keep #11960 open. Current main still does not persist bundled session-memory on automatic daily/idle rollovers: the bundled hook only listens to command:new/command:reset, and the stale rollover path archives the old transcript plus fires plugin session_end/session_start hooks without emitting the internal reset hook path. The secondary hook-config schema mismatch is fixed on main.

## Best Possible Solution

Keep #11960 open until main ships the auto daily/idle rollover memory persistence path. The narrow best fix is to emit the existing reset-hook contract from the stale rollover path, with previousSessionEntry available to session-memory and before_reset hooks, and cover daily/idle rollovers with regression tests. Treat the hook-config schema portion as already fixed.
