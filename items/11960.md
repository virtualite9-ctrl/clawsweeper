## Summary

Keep #11960 open. Current main still does not persist bundled session-memory on automatic daily/idle rollovers: the bundled hook is registered and filtered only for command:new/command:reset, while the stale rollover path archives the previous transcript and fires plugin session_end/session_start but not the internal reset hook or plugin before_reset path. The secondary hook-config schema mismatch is fixed on main. The issue discussion correctly points to open implementation PRs #61675 and #71792, but neither is merged in the provided GitHub context, so the original issue remains the active user-facing tracker.

## Best Possible Solution

Keep #11960 open until OpenClaw ships the daily/idle auto-rollover persistence path on main, likely by landing #71792 or an equivalent fix that dispatches the existing reset hook contract with previousSessionEntry for stale rollovers and covers it with regression tests. Treat the hook config schema portion as already fixed, but do not close the issue while bundled session-memory still misses automatic reset persistence.
