## Summary

Keep #11960 open. Current main at 406ae72fd2789c78053f6b94006ec8f488bc1c00 still solves only the secondary hook-config schema mismatch; the primary requested behavior remains missing because automatic daily/idle stale rollovers do not emit the internal command reset hook that bundled session-memory listens to. The related implementation PRs #71792 and #61675 are still open in the provided GitHub context, so this issue remains the canonical tracker until a fix lands or is replaced.

## Best Possible Solution

Keep #11960 open until PR #71792, PR #61675, or a replacement lands a narrow fix that fires the existing reset hook behavior, or an equivalent documented session reset surface, during automatic daily and idle stale rollover. The fix should pass previousSessionEntry to session-memory, preserve typed session_end/session_start behavior, run before_reset if that remains the intended plugin contract, and add regression coverage for both daily and idle rollover.
