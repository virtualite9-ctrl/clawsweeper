## Summary

Keep #47499 open. Current main has browser automation, sandboxed-browser noVNC observer links, and node screen capabilities, but it does not ship an embedded Control UI VNC/desktop viewer or an agent-to-dashboard desktop takeover permission flow. The linked PR #47497 was closed unmerged, and its proposed VNC UI/proxy files are absent from main. This is also not a clean ClawHub-only close yet because the current plugin API exposes tools/routes/methods, not a Control UI panel/tab extension seam.

## Best Possible Solution

Keep this open for maintainer product/security design, preferably split into narrower work: decide whether Control UI should gain a first-class or plugin-extensible visual computer panel; keep semantic browser/accessibility/node observation as the control foundation; use VNC only as an explicit visual preview; and route any desktop takeover through owner-only permissions, short-lived grants, and clear denial/fallback behavior. If maintainers want this outside core, first add the missing Control UI extension seam, then publish the VNC integration as a ClawHub plugin.
