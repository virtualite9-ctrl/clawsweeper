## Summary

Keep #69634 open. Current main still renders assistant bubble actions when `canCopyMarkdown || canExpand` is true, but the `.chat-bubble` class list does not include the existing `has-copy` spacing class, so the PR's targeted fix is not implemented on main and the linked bug #69605 remains plausibly actionable.

## Best Possible Solution

Keep the PR or a canonical replacement open, rebase it onto current main, add a focused `grouped-render` test asserting assistant bubbles with copy/expand actions get `has-copy` while bubbles without actions do not, then land the small renderer change if UI review agrees with the existing 70px padding contract.
