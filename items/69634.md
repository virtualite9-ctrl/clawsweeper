## Summary

Keep #69634 open. Current main still has the exact layout gap this PR targets: the assistant bubble renders floating message-level actions, and CSS already defines the `has-copy` padding rule, but `renderGroupedMessage` never applies `has-copy` to the bubble. The related PRtags cluster is useful context, but the related PRs in the provided context are closed/unmerged and do not supersede this open fix.

## Best Possible Solution

Rebase or land this narrow UI fix, or an equivalent change, on current main. Add a focused `grouped-render.test.ts` regression asserting assistant bubbles with copy/expand actions receive `has-copy` and bubbles without those actions do not, so the CSS spacing contract stays locked.
