## Summary

Keep open. Current main still has the overlap-prone structure: assistant bubbles render the floating message-level action bar, and CSS has a `.chat-bubble.has-copy` spacing rule, but the renderer never applies `has-copy` to normal assistant bubbles. This PR is a focused implementation candidate for #69605 rather than obsolete cleanup.

## Best Possible Solution

Keep this PR open as a valid implementation candidate for #69605. The best path is to land this exact `has-copy` class fix or an equivalent maintainer patch, ideally with a small regression test locking the relationship between `.chat-bubble-actions` and `.chat-bubble.has-copy`. No core security or supply-chain concern is apparent from the scoped diff.
