## Summary

Keep #69634 open. Current main still defines `.chat-bubble.has-copy` padding but never applies `has-copy` to the rendered chat bubble when message-level copy/expand actions are shown, so the reported overlap remains plausible and the PR is a focused fix for open issue #69605 rather than obsolete cleanup.

## Best Possible Solution

Keep the PR or land an equivalent fix after rebase: apply `has-copy` to assistant bubbles whenever message-level copy or expand actions render, and add a focused regression test in `ui/src/ui/chat/grouped-render.test.ts` asserting that assistant bubbles with actions get the class while bubbles without actions do not.
