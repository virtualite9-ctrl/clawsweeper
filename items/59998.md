## Summary

Current main does not implement `tools.exec.shell`. The config type and strict zod schema have no shell field, local exec shell selection is still hard-coded through `getShellConfig()`, and node-host Windows shell wrapping remains hard-coded to `cmd.exe`. This is a core exec/runtime configuration request, not a clean ClawHub close, because it asks to change the built-in exec tool's default shell behavior rather than add an optional external tool/plugin.
