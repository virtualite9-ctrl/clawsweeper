## Summary

Keep open. Current main has slash-command autocomplete and an `agents.list` RPC, but it does not implement Slack-style `@agent` autocomplete, mention chips, mention validation in the composer, or mention-chip rendering in chat history.

## Best Possible Solution

Keep the issue open. Implement this in the Control UI chat surface, likely reusing or extending `agents.list` for a stable agent directory, then add `@` trigger/filtering, keyboard-accessible dropdown behavior, safe plain-text serialization to `chat.send`, and validated mention-chip rendering in history. Add focused UI tests around trigger parsing, selection, escaping/deletion, disconnected/empty-agent states, and preservation of existing slash-command behavior.

## Risks / Open Questions

- The already-shipped slash-command autocomplete is adjacent but materially different, so closing this as implemented would lose the requested @mention UX.
- A rich contenteditable composer would touch send serialization, keyboard handling, paste behavior, accessibility, and history rendering; this needs focused UI tests rather than a small cosmetic change.
- Current plugin surfaces appear aimed at channels/providers/tools/commands/config metadata, not replacing the core Control UI chat composer, so this should stay a core Control UI/gateway feature unless maintainers first add an explicit UI extension API.
