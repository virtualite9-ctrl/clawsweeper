## Summary

Current `main` already handles attach-only browser profiles on the non-destructive path the issue asks for: attach-only/remote CDP availability now uses remote-class probe timeouts, returns attach-specific unreachability errors, and explicitly skips the local restart/ownership path; proxy timeout surfaces also now return status diagnostics instead of restart guidance. The changelog shows this shipped in `v2026.4.14`.
