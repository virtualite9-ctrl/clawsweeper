## Summary

Current main still contains the reported TUI fallback path: when the TUI knows the context window but has no positive total token count, the footer formatter renders `tokens ?/<context>` and omits the percentage. The closed related PR proposed the narrow fix but was not merged, so this issue still tracks real remaining work.

## Best Possible Solution

Keep this issue open. Reapply the narrow #43044 behavior in `src/tui/tui-formatters.ts`: when context is known and total is null or undefined, render `0/<ctx> (0%)` instead of `?/<ctx>`. Add a focused regression test in `src/tui/tui-formatters.test.ts`, and consider a session-refresh regression covering defaults or sparse session rows that provide context without total.
