## Summary

Keep this issue open. Current main still has the reported TUI display fallback: a missing `totalTokens` plus known `contextTokens` renders as `tokens ?/<ctx>`, and the session-list path can still produce that state.

## Best Possible Solution

Keep this issue open. Reimplement the narrow #43044 behavior in `src/tui/tui-formatters.ts`: when context is known and total is null or undefined, render `0/<ctx> (0%)` instead of `?/<ctx>`, then add a focused regression test in `src/tui/tui-formatters.test.ts` and, if needed, a session refresh test for defaults plus missing totals.
