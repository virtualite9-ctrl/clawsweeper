## Summary

Do not close this yet. Current main has provider usage fixes that may solve some related OpenAI-compatible token reports, but the TUI display-layer behavior described here is still present when the footer has a known context window and no usable total token snapshot: it renders `tokens ?/200k`. The linked fixing PR was closed unmerged, and current main/release still have the old formatter behavior.

## Best Possible Solution

Keep this open and either revive the small TUI formatter fix from #43044 or implement the equivalent deliberately: align the TUI footer with `/status`/`session_status` by showing `0/<ctx> (0%)` when context is known but the current total is absent or zero, then add a focused regression test in `src/tui/tui-formatters.test.ts` and, if needed, a session-action test proving context defaults plus missing totals render predictably.
