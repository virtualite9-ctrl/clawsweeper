## Summary

Current main still contains the reported TUI display path: when the TUI knows the context window but receives no positive total token count, the footer renders `tokens ?/<context>` and omits the percentage. The related narrow fix in #43044 was closed unmerged, so this issue still tracks real remaining work.

## Best Possible Solution

Keep this issue open. Reapply the narrow #43044 behavior in `src/tui/tui-formatters.ts`: when context is known and total is null or undefined, render `0/<ctx> (0%)` instead of `?/<ctx>`. Add a focused regression test in `src/tui/tui-formatters.test.ts`, and consider a session-refresh regression covering list rows/defaults that provide context without a positive total.
