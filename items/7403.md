## Summary

Keep #7403 open. Current main still lacks an enforceable per-session Private/Secret/Incognito mode: there is no Gateway/session flag, no session-store enforcement bit, session creation still creates a JSONL transcript, normal assistant transcript paths still append to disk, and the only related shipped knob (`agents.defaults.contextInjection: "never"`) suppresses bootstrap injection globally rather than providing private session persistence guarantees.

## Best Possible Solution

Keep #7403 open as the canonical product request. A complete implementation should add an explicit private/ephemeral session mode through Gateway protocol and session state, enforce no transcript/media/session-history writes, suppress workspace bootstrap and memory reads/writes, expose Mission Control and per-channel controls, and add regression tests proving private sessions neither read workspace/memory context nor write persistent history.

## Risks / Open Questions

- A partial implementation could create a false privacy guarantee unless it covers protocol/session state, transcript/media writes, lifecycle events, memory/context reads and writes, channel commands, and Mission Control UI together.
- Existing mitigations such as transcript redaction, session maintenance/pruning, and contextInjection are useful but do not prevent private data from being written to disk in the first place.
