## Summary

Keep open. Current main still does not implement the requested enforceable Private/Secret/Incognito mode: there is no per-session or per-channel private flag in the Gateway protocol, no session config or stored session state for no-persist behavior, and the normal session creation/append paths still create and write transcript files. The related context makes #7403 the canonical tracker because #12242 was closed as a duplicate and #52478 was closed unmerged.

## Best Possible Solution

Keep #7403 open as the canonical product request. A complete fix should add an explicit private/ephemeral session mode through the Gateway protocol, session config/store state, transcript creation and append paths, workspace bootstrap and memory read/write paths, Mission Control and channel controls, with regression tests proving private sessions neither read workspace/memory context nor write session history or memory files.
