## Summary

Keep open. Current main still lacks an enforceable Private/Secret/Incognito session mode: no Gateway protocol field, no session config or SessionEntry state, no transcript no-persist branch, and workspace bootstrap files are still loaded for ordinary runs. The provided related context makes #7403 the canonical tracker: #12242 was closed as a duplicate, and #52478 was closed unmerged after deferring runtime filesystem/session-history enforcement.

## Best Possible Solution

Keep #7403 open as the canonical product request. A complete fix should add an explicit private/ephemeral session mode through the Gateway protocol, persisted session/config state, transcript creation and append no-write paths, workspace bootstrap and memory read/write suppression, Mission Control UI indicators/toggles, and per-channel controls, with regression tests proving private sessions do not read workspace/memory context and do not write session history or memory files.
