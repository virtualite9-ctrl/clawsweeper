## Summary

Keep #72097 open. Current main still treats plugin tool availability as a tool-list construction/session-creation concern: plugin install refreshes persisted metadata but still instructs users to restart the gateway, and embedded agent runs pass a fixed customTools/tool allowlist into createAgentSession. I found runtime inventory APIs, but no registry-change broadcast, toolset delta, or active-session refresh path that would solve the reported install-then-use flow.

## Best Possible Solution

Keep the issue open for a core runtime fix. After successful plugin install/registry refresh, OpenClaw should either notify active agent sessions and refresh their customTools/session allowlist before the next LLM turn, or make the embedded runner re-resolve plugin tools at each turn boundary. Preserve existing tool policy checks, update the active plugin registry consistently, add regression coverage for install-then-use without gateway restart, and update plugin docs once restart is no longer required for this flow.
