## Summary

Keep #72097 open. Current main still builds plugin tools into an agent session only when the run/compaction session is created, and plugin install still tells users to restart the gateway. I did not find a registry-change broadcast, toolset-delta, or per-turn re-resolution path that would update active sessions after a tool-providing plugin install.

## Best Possible Solution

Implement this in core plugin-runtime/session lifecycle code. On successful plugin install or registry refresh, notify active agent sessions or re-resolve plugin tools at the next turn boundary, then update `customTools`, the session tool allowlist, and any prompt/tool inventory state before the next model call. Add regression coverage for install -> immediate use in an existing session, plus coverage that active registry reuse still avoids duplicate plugin runtime instances.
