## Summary

Keep #72097 open. Current main has the plugin tool registration surface and gateway-bindable active registry reuse, but it still treats plugin install/load changes as a restart-boundary operation and freezes the agent tool allowlist/customTools when a run session is created. I did not find a current registry-change broadcast, toolset-delta, or per-turn active-session tool refresh path that would make newly installed plugin tools available to already-active sessions without a restart/new session path.

## Best Possible Solution

Implement this as a core plugin-runtime/session lifecycle improvement, not as an external ClawHub plugin. The likely path is to make plugin install/registry refresh notify active agent sessions, or re-resolve plugin tools at the next turn boundary, then update `customTools`, the active tool allowlist, and any prompt/tool inventory state before the next model call. Add regression coverage around a tool-providing plugin install followed by immediate use in an existing session, plus coverage that existing gateway-bindable runtime reuse still avoids duplicate plugin runtime instances.
