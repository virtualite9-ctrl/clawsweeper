## Summary

Keep open. Current main has partial improvements for Claude CLI defaults, generic model-fallback skip logging, and transient auto-fallback session overrides, but it still does not document the reported root/systemd Claude CLI constraint or emit the requested CLI-backend-specific skip diagnostic. The reporter's follow-up says the local workaround succeeded while explicitly keeping the maintainer requests open.

## Best Possible Solution

Keep this open as a focused Claude CLI operations issue. The best maintainer path is to add an explicit pre-dispatch CLI-backend skip diagnostic with provider and reason, document the root/systemd Claude CLI constraint and supported workaround or non-root recommendation on the public CLI Backends/Anthropic docs, and decide whether root detection should fail loudly or support an intentional opt-in. Retain the current Claude CLI defaults and transient auto-fallback session behavior.
