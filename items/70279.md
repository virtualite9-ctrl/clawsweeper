## Summary

Keep open. Current main has partial fixes and better guidance around Claude CLI backends, but the issueŌĆÖs remaining maintainer requests are still concrete: root/systemd behavior is not documented, there is no root-specific Claude CLI handling, and pre-dispatch skips are logged only as generic model-fallback decisions rather than the requested CLI-backend diagnostic.

## Best Possible Solution

Keep this issue open as a focused Claude CLI operations bug. The best maintainer path is to add an explicit pre-dispatch CLI-backend skip diagnostic, document the root/systemd Claude CLI constraint and supported workaround or non-root recommendation on the public CLI Backends/Anthropic docs, and decide whether root detection should fail loudly or support an intentional opt-in. Existing session-runtime separation and Claude CLI env/default fixes should be retained.
