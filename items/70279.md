## Summary

Keep open. Current main has adjacent improvements for Claude CLI defaults, fallback logging, and transient auto-fallback overrides, but the remaining issue requests are still actionable: no CLI-specific pre-dispatch skip diagnostic, no root/systemd or IS_SANDBOX documentation, and fallback candidate selection is still persisted before the fallback attempt rather than gated on the primary having run.

## Best Possible Solution

Keep the issue open and narrow the maintainer task: add a CLI-aware pre-dispatch skip diagnostic when the skipped candidate resolves to a CLI backend, document Claude CLI root/systemd behavior and the supported non-root or IS_SANDBOX path on the public CLI backend docs, and decide whether fallback selection should avoid writing session overrides when the primary was skipped pre-dispatch or whether the current auto-clearing behavior should be made explicit and tested for this scenario.
