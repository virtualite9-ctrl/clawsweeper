## Summary

Keep open. Current main still does not implement the cron humanization this PR proposes: cron jobs still render raw `Cron <expr>` labels, and Dreaming phase rows still display only the next-run time while ignoring the already-normalized phase cron expression. The PR is recent, focused, includes tests in the provided diff, and the provided review context shows the concrete minute-offset and missing-cron fallback concerns were addressed by follow-up commits.

## Best Possible Solution

Keep this PR open for maintainer review, rebase, CI, and possible landing. The best path is to preserve the focused cron-description utility and regression tests, keep the follow-up fixes for non-zero minute offsets and missing-cron fallback, and decide during review whether the Dreaming phase rows should surface configured phase cron expressions or the current unified managed dreaming sweep.
