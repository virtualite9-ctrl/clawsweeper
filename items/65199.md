## Summary

Keep this PR open. Current main still does not implement the cron humanization proposed here: cron jobs still render raw `Cron <expr>` labels, and Dreaming phase rows still display only the next-run time. The PR is focused and the provided discussion shows the concrete review findings were addressed by follow-up commits, though the Dreaming copy should be reconciled with current main's unified managed dreaming sweep.

## Best Possible Solution

Keep this PR open for maintainer review, rebase, CI, and possible landing. The best path is to preserve a focused cron-description utility with regression tests, wire it into `formatCronSchedule`, and decide during review whether Dreaming phase rows should show the configured sweep cadence, phase cron text for legacy/status payloads, or a different label that matches the current unified managed dreaming sweep.
