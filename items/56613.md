## Summary

Keep open. Current main has implemented the per-agent TTS half through `agents.list[].tts`, but the Android Voice/Talk path still has no agent/session picker and still routes manual mic and Talk Mode through the resolved main/default session key. The issue discussion also has a maintainer comment explicitly keeping this open for Talk/Voice UI session selection.

## Best Possible Solution

Keep this issue open for the remaining Talk/Voice session-selection work, especially Android. A good fix would add selected Voice/Talk agent or session state, expose a picker or reuse the existing Chat selector pattern, route `MicCaptureManager` and `TalkModeManager` through that selected key instead of only `resolveMainSessionKey()`, add focused routing/UI tests, and keep `agents.list[].tts` as the canonical per-agent voice config instead of adding the proposed `talk.voiceId` shape.
