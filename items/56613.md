## Summary

Keep open. Current main already implements the per-agent TTS half through `agents.list[].tts`, and iOS now has a settings-level Bot picker that updates Talk routing, but the Android Voice tab still has no agent/session picker and Android Talk Mode still sends through the resolved main session key. The central remaining request is therefore still actionable in this repo.

## Best Possible Solution

Keep this issue open for the remaining Talk/Voice session-selection work. The likely implementation path is to add an Android Voice/Talk selector or selected-agent state, wire that selected session key into `TalkModeManager` instead of only the resolved main/default session, add focused Android UI/routing tests, and align iOS/macOS behavior without replacing the already-shipped `agents.list[].tts` per-agent voice configuration.
