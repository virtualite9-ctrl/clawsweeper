## Summary

Keep open. The per-agent TTS half is implemented on current main via `agents.list[].tts`, but the remaining Talk/Voice agent/session switching request is still actionable: Android Voice/Talk still has no session picker and still routes voice sends through the resolved main session key rather than a selected agent/session.

## Best Possible Solution

Keep this issue open for the remaining Talk/Voice session-selection work. A good fix would add an Android Voice/Talk selected-agent or session picker, wire that selected key into `TalkModeManager` and manual voice sends instead of only the resolved main/default key, add focused Android routing/UI tests, and preserve the already-shipped `agents.list[].tts` per-agent voice configuration.
