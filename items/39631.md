## Summary

Keep #39631 open. Current main still intentionally turns failed mutating tool calls such as `edit`/`write` into user-facing `⚠️ ... failed` error payloads, and Telegram still delivers `isError` final payloads as chat messages rather than suppressing or normalizing them. The linked fix PRs #39636 and #39717 were closed unmerged, and the broader #39406 remains open rather than superseding this specific Telegram leak report.

## Best Possible Solution

Keep the issue open until OpenClaw deliberately changes the agent reply payload policy for mutating tool failures. The likely fix belongs around `resolveToolErrorWarningPolicy` and `buildEmbeddedRunPayloads`: suppress or normalize failed mutating-tool warning payloads when a user-facing assistant reply already exists, preserve a fallback warning when no assistant reply exists, keep full detail in logs/debug surfaces, and add regression coverage in both payload tests and Telegram dispatch tests for failed `edit()` not leaking a raw tool warning into chat.
