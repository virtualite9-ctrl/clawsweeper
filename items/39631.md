## Summary

Keep open. Current `main` still intentionally emits mutating tool failure warnings into user-facing payloads, and the `edit` tool summary still includes the file path and content length, so the Telegram behavior described in #39631 is not implemented away on `main`. There are partial controls for heartbeat runs and Telegram delivery/provider errors, but not a general fix for user-triggered mutating tool failures in Telegram chats.

## Evidence

- **Mutating tool failures are still configured to warn:** `resolveToolErrorWarningPolicy` still returns `showWarning: true` for mutating tool errors unless `suppressToolErrorWarnings` is explicitly set.
  - file: [src/agents/pi-embedded-runner/run/payloads.ts:95](https://github.com/openclaw/openclaw/blob/cb2c36b04975e34ab7014a11575d6589aa96ae32/src/agents/pi-embedded-runner/run/payloads.ts#L95)
- **Warning text is still emitted to user-facing payloads:** `buildEmbeddedRunPayloads` still appends `⚠️ ${toolSummary} failed` to reply payloads when `warningPolicy.showWarning` is true.
  - file: [src/agents/pi-embedded-runner/run/payloads.ts:350](https://github.com/openclaw/openclaw/blob/cb2c36b04975e34ab7014a11575d6589aa96ae32/src/agents/pi-embedded-runner/run/payloads.ts#L350)
- **Edit tool summary still contains path and char count:** `resolveWriteDetail` formats `edit` as `in <path> (<chars> chars)`, which matches the shape of the leaked Telegram message reported in the issue.
  - file: [src/agents/tool-display-common.ts:243](https://github.com/openclaw/openclaw/blob/cb2c36b04975e34ab7014a11575d6589aa96ae32/src/agents/tool-display-common.ts#L243)
- **Tests still expect mutating tool warnings to remain visible:** Current tests only suppress mutating tool warnings when `suppressToolErrorWarnings` is set, and explicitly verify that mutating tool errors are still shown even when assistant output exists.
  - file: [src/agents/pi-embedded-runner/run/payloads.errors.test.ts:293](https://github.com/openclaw/openclaw/blob/cb2c36b04975e34ab7014a11575d6589aa96ae32/src/agents/pi-embedded-runner/run/payloads.errors.test.ts#L293)
- **Telegram dispatch tests still send tool warnings into chat replies:** Telegram message-dispatch tests assert that a tool warning reply is delivered to chat while the primary answer stays in the preview/final message path.
  - file: [extensions/telegram/src/bot-message-dispatch.test.ts:663](https://github.com/openclaw/openclaw/blob/cb2c36b04975e34ab7014a11575d6589aa96ae32/extensions/telegram/src/bot-message-dispatch.test.ts#L663)
- **Available suppression knobs do not cover this default user-turn case:** Docs scope `suppressToolErrorWarnings` to heartbeat runs, while Telegram `errorPolicy` is documented for delivery/provider errors rather than mutating tool warnings during normal chat turns.
  - file: [docs/gateway/heartbeat.md:243](https://github.com/openclaw/openclaw/blob/cb2c36b04975e34ab7014a11575d6589aa96ae32/docs/gateway/heartbeat.md#L243)
