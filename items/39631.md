## Summary

Keep #39631 open. Current main still turns failed mutating tool calls such as `edit` into user-facing `ã ÿ¡ ... failed` payloads, and Telegram dispatch still delivers final `isError` payloads to chat instead of suppressing or normalizing them. The related fix PRs #39636 and #39717 were closed unmerged, and the broader #39406 remains open rather than superseding this Telegram-specific leak report.

## Best Possible Solution

Keep the issue open until OpenClaw deliberately changes the user-facing policy for mutating tool failures. The likely fix belongs around `resolveToolErrorWarningPolicy` and `buildEmbeddedRunPayloads`: suppress or normalize failed mutating-tool warning payloads when a user-facing assistant reply already exists, preserve a clean fallback when no assistant reply exists, keep full detail in logs/debug surfaces, and add regression coverage in both payload tests and Telegram dispatch tests for failed `edit()` not leaking a raw tool warning into chat.
