## Summary

Keep #72081 open. Current main intentionally does not print `#token=...` in `openclaw dashboard` output, while still sending a tokenized URL to clipboard/browser for non-SecretRef gateway tokens. The report is still actionable because WSL/headless or clipboard-unavailable workflows can leave the user with only the non-tokenized printed URL, and current UI copy still promises `openclaw dashboard --no-open` gives a tokenized URL.

## Best Possible Solution

Keep the default no-token-in-terminal behavior, but add a safe explicit path for headless or clipboard-unavailable users to authenticate the dashboard, such as clearer guidance to retrieve/paste `gateway.auth.token`, a deliberately named opt-in command/flag that avoids accidental log exposure, or UI/docs copy that no longer promises `openclaw dashboard --no-open` prints a tokenized URL. The implementation likely belongs in `src/commands/dashboard.ts`, its tests, and the dashboard/Control UI guidance surfaces.
