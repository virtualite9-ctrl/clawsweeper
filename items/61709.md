## Summary

Keep open. Current `main` still documents model auth as per-agent, the exposed CLI/configuration surfaces operate on one agent store at a time, and there is no evidence of a host-wide atomic auth-profile migration or `set-active` command across all agent stores. Existing cross-agent behavior is limited to fallback/copy/audit paths, not coordinated writes.
