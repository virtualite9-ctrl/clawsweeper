## Summary

Keep this PR open. Current main still has the finite WhatsApp reconnect cap and terminal max-attempt stop path this PR is trying to remove, so the work is still product-relevant even though the patch is stale against the current WhatsApp plugin layout.

## Best Possible Solution

Keep this PR open, but require a refresh for the current WhatsApp plugin architecture. The likely path is to update extensions/whatsapp/src/reconnect.ts, extensions/whatsapp/src/connection-controller.ts, and extensions/whatsapp/src/auto-reply/monitor.ts so retryable disconnects never permanently kill the monitor, while logged-out and session-conflict paths still stop. Add/adjust tests for default unlimited retries, bounded degraded-mode retry, and non-retryable closes; reconcile web.reconnect.maxAttempts docs/schema wording before merge.
