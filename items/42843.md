## Summary

Keep this PR open. Current main still has the finite WhatsApp reconnect cap and terminal stop path the PR is trying to remove, so the underlying product problem is still present, but the patch needs to be refreshed for the current WhatsApp plugin layout.

## Best Possible Solution

Keep this PR open, but require a refresh against the current WhatsApp plugin architecture. The likely path is to update extensions/whatsapp/src/reconnect.ts, extensions/whatsapp/src/connection-controller.ts, and extensions/whatsapp/src/auto-reply/monitor.ts so retryable disconnects never permanently kill the monitor, while logged-out and session-conflict closes remain terminal. Add or adjust tests for default unlimited retries, bounded degraded-mode retry, non-retryable closes, and reconcile web.reconnect.maxAttempts schema/help/docs/generated metadata before merge.
