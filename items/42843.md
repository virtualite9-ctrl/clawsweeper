## Summary

Keep this PR open. Current main has moved the WhatsApp code into `extensions/whatsapp`, and it has several later reconnect/active-listener fixes, but the specific behavior proposed here is still not implemented: the default reconnect cap is still finite and the monitor still shuts down after that cap is reached.

## Best Possible Solution

Keep the PR open, but refresh it for the current WhatsApp plugin architecture. The likely implementation belongs in `extensions/whatsapp/src/reconnect.ts`, `extensions/whatsapp/src/connection-controller.ts`, and `extensions/whatsapp/src/auto-reply/monitor.ts`, with tests updated to prove that retryable disconnects do not permanently kill the monitor while non-retryable logged-out/session-conflict paths still stop. The refresh should also reconcile `web.reconnect.maxAttempts` docs/schema wording and fix the degraded-mode log noted in review.
