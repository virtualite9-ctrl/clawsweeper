## Summary

Current `main` has a dedicated `/btw` snapshot path that replays sanitized session context through a separate tool-less request, rather than chaining upstream Responses item IDs. Gateway and UI handling plus regression tests on `main` match the issue’s failing shape closely enough to treat this as implemented there.

## Evidence

- **BTW now replays sanitized snapshot messages, not provider item ids:** `toSimpleContextMessages` keeps only sanitized user/assistant content and strips tool-result details before `/btw` submits a separate side-question request. The request payload is built from plain `messages` plus a wrapped `/btw` prompt, so the current path does not depend on persisted upstream `rs_resp_*` items.
