## Summary

Keep open. Current main still lacks create/update-time Feishu/Lark announce target rejection, #39599 remains open, and #39683 is a focused but not merge-ready candidate because its raw `delivery.to` requirement can reject resolvable jobs and can block legacy-job remediation on update.

## Best Possible Solution

Keep the PR open but request a rework. The correct fix should validate unresolved Feishu/Lark announce targets through the existing delivery-resolution seam, allow explicit `delivery.to` and deterministic session/plugin resolution, avoid blocking unrelated updates on legacy jobs, keep provider-specific target hints in the Feishu plugin surface, and add focused create/update regression coverage.
