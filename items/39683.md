## Summary

Current `main` still lacks create/update validation for Feishu/Lark announce jobs without `delivery.to`, so this PR is not implemented on `main` and should stay open. The bug remains real because `cron add` still permits missing `--to` for announce delivery, and the failure is deferred to delivery-target resolution at runtime. The PR likely needs rebasing and a small follow-up for legacy invalid-job toggle behavior and docs.

## Risks / Open Questions

- If this PR is closed now, current `main` still accepts Feishu/Lark announce jobs without a delivery target and only surfaces the error later during delivery resolution.
- As proposed, enforcing the new check inside `applyJobPatch` would also affect toggle-only updates. `cron disable` sends only `{ enabled: false }` from `src/cli/cron-cli/register.cron-simple.ts:54`, while persisted invalid jobs can still load from `src/cron/service/store.ts:38`, so legacy invalid Feishu/Lark jobs could become harder to disable without an explicit remediation path.
- The docs portion of the PR is out of date with the current docs tree and docs i18n policy, so the patch likely needs rebasing even if the code/test change is still desired.
