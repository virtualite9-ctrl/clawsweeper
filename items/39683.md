## Summary

Keep #39683 open. Current main still does not reject Feishu/Lark announce cron jobs without a delivery target at create/update time, and the linked canonical bug #39599 remains open. The PR is a focused candidate but still needs rework because the submitted raw `delivery.to` check can reject jobs whose target is resolvable from session/plugin routing and can block simple updates to legacy jobs.

## Best Possible Solution

Keep the PR open but request rework. The fix should validate unresolved Feishu/Lark announce targets through the existing delivery-resolution seam, allow explicit `delivery.to` and deterministic session/plugin resolution, avoid blocking unrelated updates to legacy jobs, keep provider-specific target hints in the Feishu plugin surface, and add focused create/update regression coverage.
