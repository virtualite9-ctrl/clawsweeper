## Summary

Keep #11623 open. Current main still has adjacent macOS status, HoverHUD, WebChat, Canvas, and Talk overlay surfaces, but it does not implement multiple configurable floating per-agent bubbles with persisted positions, mouse-follow/anchor behavior, or per-agent chat launch/routing. The prior issue discussion already reached the same keep-open conclusion, and current main still matches it.

## Best Possible Solution

Keep #11623 open. The best path is either a native macOS implementation under `apps/macos/Sources/OpenClaw` that derives bubble state from agent/session activity, persists positions, supports drag/anchor or mouse-follow behavior, and routes chat to the selected agent, or a maintainer-designed native macOS UI/window plugin API that would make an external ClawHub implementation feasible.

## Risks / Open Questions

- Closing as implemented would lose a still-missing macOS product feature: multiple configurable always-on-top per-agent bubbles with persisted positions and per-agent chat routing.
- Closing as clawhub is premature unless maintainers first add and document a native macOS SwiftUI/AppKit window extension point or explicitly decide this UI belongs outside core.
- The feature touches always-on-top windows, click-through/input behavior, persistence, session routing, and macOS UX, so it needs maintainer product judgment.
