## Summary

Keep #11623 open. Current main still has only adjacent macOS surfaces: a menu-bar anchored hover HUD, a WebChat window/panel, a debug agent-events window, and a single Talk Mode orb. It does not implement multiple configurable per-agent floating Clawi bubbles, mouse-follow/anchor behavior, persisted bubble positions, or bubble-launched per-agent chat routing. A ClawHub close is also premature because the documented/native plugin API does not expose a macOS SwiftUI/AppKit window extension point.

## Best Possible Solution

Keep #11623 open. The best path is either a native macOS implementation under apps/macos/Sources/OpenClaw/ that connects bubble state to agent/session activity, persisted positions, and chat routing, or a maintainer-led design for a native macOS UI/window extension API before moving this to ClawHub/plugins.
