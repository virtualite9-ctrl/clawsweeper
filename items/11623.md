## Summary

Keep #11623 open. Current main still has adjacent macOS UI surfaces, but not the requested multiple configurable per-agent floating Clawi bubbles with persisted positions, drag/anchor or mouse-follow behavior, and bubble-launched per-agent chat routing. The prior issue discussion already reached the same conclusion, and the current plugin API still does not expose a native macOS SwiftUI/AppKit window extension point that would make this a straightforward ClawHub close.

## Best Possible Solution

Keep #11623 open. The best path is either a native macOS implementation under apps/macos/Sources/OpenClaw/ that connects bubble state to agent/session activity, persists positions, and routes chat to the selected agent, or a maintainer-led native macOS UI/window plugin API design before this can move to ClawHub/plugins.
