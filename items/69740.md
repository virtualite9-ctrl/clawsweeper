## Summary

Keep open. Current main has not implemented the PR's central production guard for debug environment variables: there is no `src/agents/debug-env.ts`, and the relevant `OPENCLAW_DEBUG_*` flags are still read directly in agent, auto-reply, health, Telegram, and memory paths. The PR also needs changes before it could land because the discussion already identified incomplete migration and unrelated UI/install-smoke churn.

## Best Possible Solution

Keep the PR open for maintainer review, but request a scoped rewrite: decide the intended production policy first, then implement it through an existing shared env/runtime seam such as `src/infra/env.ts` plus the plugin SDK runtime export if plugins need it, migrate all relevant debug flag call sites, add focused tests for production and non-production behavior, and remove unrelated UI/install-smoke changes from this PR.
