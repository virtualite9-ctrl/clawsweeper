## Summary

Keep open. Current main does not implement the PR's core session-store recovery behavior, and the PR is a plausible gateway bug-fix direction, but it is not merge-ready because the provided review context identifies unresolved recovery-path bugs and the PR template lacks repro/verification evidence.

## Best Possible Solution

Keep the PR open for maintainer/contributor follow-up. The best path is to require a concrete repro or failing test for corrupt session-store behavior, fix the reviewed recovery-path issues (`<` threshold and catching failures before target lookup reads the store), add focused tests for corrupt JSON, `.prev` restore, transcript fallback, and steady-state one-entry stores, then reassess whether this belongs as core gateway/session-store behavior.
