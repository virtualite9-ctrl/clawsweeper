## Summary

Keep #38762 open. Current main at 631552c still has the reported mixed-shape bug: `browser action="act"` accepts fully nested `request.ref` and fully flattened legacy params, but when `request` is an object it returns that object before merging top-level `ref`, `targetId`, or `timeoutMs`. The related PR #38790 targeted the right compatibility fix, but the provided GitHub context shows it was closed unmerged, and the same gap is still present in current source and tests. The misleading reachability wording has been partly improved, but the requested top-level ref compatibility remains missing.

## Best Possible Solution

Keep the issue open and land a narrow compatibility normalizer in `extensions/browser/src/browser-tool.ts`: when `action: "act"` receives an object `request`, clone it and fill only missing `request.ref`, `request.targetId`, and finite `request.timeoutMs` from top-level params. Add focused tests in `extensions/browser/src/browser-tool.test.ts` for the reported mixed shape, nested-value precedence, and unchanged fully flattened legacy behavior.
