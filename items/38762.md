## Summary

Keep #38762 open. Current main has improved the old browser-service error wrapping for service-side 400s, but it still does not support the reported mixed parameter shape: `browser action="act" ref="e146" request={"kind":"click"}` drops the top-level `ref` before sending `/act`, so the click body still lacks `ref` or `selector`. The related PR #38790 directly targeted this but was closed unmerged, so it does not supersede the issue.

## Best Possible Solution

Keep the issue open and land a narrow compatibility normalizer in `extensions/browser/src/browser-tool.ts`: when `action: "act"` receives an object `request`, clone it and fill only missing `request.ref`, `request.targetId`, and finite `request.timeoutMs` from top-level params. Preserve nested request precedence, keep fully flattened legacy behavior unchanged, and add focused tests in `extensions/browser/src/browser-tool.test.ts` for the mixed shape, nested-value precedence, and timeout/targetId fill behavior.
