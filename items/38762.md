## Summary

Keep open. Current main partially addresses the misleading reachability wording for browser-service application errors, but it still does not accept the issue's reported mixed shape `action=act ref=... request={kind: ...}`. The act request parser returns `params.request` unchanged whenever it is an object, so top-level `ref` is ignored in that case; existing tests only cover fully flattened act params and explicitly preserve nested-request precedence. The linked PR #38790 proposed the missing compatibility normalizer but was closed unmerged, so it does not supersede the issue.
