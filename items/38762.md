## Summary

Keep #38762 open. Current main at 4a3030df9efa72c44bab567a4390ac0c4876a73f still ignores top-level `ref` when `browser action="act"` also receives an object `request`; the object is returned before legacy flattened fields are merged. The misleading reachability wrapper appears partly improved, but the requested mixed-shape compatibility remains missing. Related PR #38790 targeted this exact fix but the provided GitHub context shows it was closed unmerged.

## Best Possible Solution

Keep the issue open and land a narrow compatibility normalizer in `extensions/browser/src/browser-tool.ts`: when `action: "act"` receives an object `request`, clone it and fill only missing `request.ref`, `request.targetId`, and finite `request.timeoutMs` from top-level params. Preserve nested request precedence and add focused tests in `extensions/browser/src/browser-tool.test.ts` for the reported mixed shape, nested-value precedence, and unchanged fully flattened legacy behavior.
