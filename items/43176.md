## Summary

Keep this PR open. Current main still lacks the requested broad `$HOME` or `/` shell `find` timeout cap and the matching system-prompt guidance, while this PR remains a plausible focused fix for a real exec-budget failure mode. It should not be closed as stale or obsolete; it needs maintainer review and parser hardening before merge.

## Best Possible Solution

Keep this PR open and request changes before merge. The best path is to rebase onto current main, add a narrow shared broad-find timeout helper used by every foreground exec path, parse all `find` start roots, resolve relative roots against command `workdir`, support `~`, `$HOME`, and `${HOME}` forms, preserve explicit timeout and real `-prune`/`-xdev`/`-mount` operator intent, avoid escape hatches from later pipeline or `-exec` arguments, and keep focused regression tests for root, home, relative workdir, multiple roots, escape hatches, node/gateway routing where applicable, and prompt guidance.
