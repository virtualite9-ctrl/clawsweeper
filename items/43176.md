## Summary

Keep this PR open. Current main still does not implement the broad $HOME or / shell find timeout cap or the matching system-prompt guidance, and this PR remains a plausible focused fix for a real exec-budget failure mode. It still needs maintainer review and parser fixes before merge.

## Best Possible Solution

Keep this PR open and request changes before merge. The best path is to rebase onto current main, add a narrow shared broad-find timeout helper used by every exec path that can run foreground commands, parse all find start paths, resolve relative roots against the command workdir, support ~, $HOME, and ${HOME} forms, preserve explicit timeout and real -prune/-xdev/-mount intent, avoid escape hatches from later pipeline or -exec arguments, and keep focused regression tests for root, home, relative workdir, multiple roots, escape hatches, and prompt guidance.
