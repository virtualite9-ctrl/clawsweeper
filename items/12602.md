## Summary

Keep #12602 open. Current main already implements the direct Slack Block Kit payload path for agent replies, shared presentation rendering, and interactive buttons/selects, but the remaining native Slack markdown-table/rich Markdown delivery path is still not implemented: `markdown.tables: "block"` remains schema-valid but is coerced to `code` before Slack outbound rendering. The issue discussion also includes a recent Codex/maintainer review explicitly keeping this umbrella open until that table/rich output work is resolved or superseded.

## Best Possible Solution

Keep #12602 open unless maintainers explicitly designate #57708 or another narrower issue as canonical. The remaining implementation belongs in the bundled Slack plugin: carry `markdownToIRWithMeta` table metadata through outbound Slack delivery when `markdown.tables: "block"` is configured, render supported Markdown tables/rich Markdown as Slack Block Kit table or rich blocks, add focused Slack outbound tests and public docs, and avoid duplicating the already-shipped raw `channelData.slack.blocks`, shared `presentation`, and interactive directive support.
