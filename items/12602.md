## Summary

Keep #12602 open. Current main implements major Slack Block Kit slices for agent messages: authored `channelData.slack.blocks`, shared `presentation` rendering, interactive buttons/selects, validation, fallback text, and documented interactive reply directives. The remaining Slack native table/rich-rendering path is still not implemented: `markdown.tables: "block"` remains schema/IR-valid but is coerced to `code` before Slack runtime delivery, so markdown tables are still sent through text/code rendering rather than Slack Block Kit table blocks.

## Best Possible Solution

Keep #12602 open unless a maintainer explicitly supersedes it with a narrower canonical Slack rich markdown/table issue. The best implementation path is in the bundled Slack plugin: carry `markdownToIRWithMeta` table metadata through outbound delivery when `markdown.tables: "block"` is configured, render supported markdown tables as Slack Block Kit table or richer blocks, add focused outbound tests and public Slack docs, and avoid duplicating the already-shipped raw `channelData.slack.blocks`, shared `presentation`, and interactive directive support.
