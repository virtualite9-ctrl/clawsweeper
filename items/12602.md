## Summary

Keep #12602 open. Current main already implements major Slack Block Kit slices: raw `channelData.slack.blocks` delivery, Slack message-tool `blocks`, shared `presentation` rendering to Block Kit, and opt-in interactive reply directives. The remaining richer/table-rendering gap is still present on current main: `markdown.tables: "block"` is accepted by schema/IR seams but coerced back to `code` for runtime delivery, and tests still assert that fallback. The latest provided issue discussion also explicitly kept this umbrella open until that remaining Slack rich/table path is implemented or superseded.

## Best Possible Solution

Keep #12602 open as the umbrella for the remaining Slack rich-output work unless a maintainer explicitly supersedes it with a narrower canonical issue such as the open Slack rich markdown/table follow-up. The likely implementation belongs in the bundled Slack plugin: carry markdown table/rich markdown metadata through to native Slack Block Kit table or rich rendering end to end, add focused outbound tests, and update the public Slack or message-presentation docs. Treat raw Block Kit payloads, shared presentation, and interactive reply directives as already shipped.
