## Summary

Keep PR #60184 open. Current main still forwards captured console output to the file logger without checking `isFileLogLevelEnabled`, while docs say file logs are controlled by `logging.level`. The PR remains a focused fix for an actionable logging/privacy bug, but it should be revised with a regression test and with the unrelated named-agent test mock change dropped or justified.

## Best Possible Solution

Keep the PR open and ask for a focused revision: rebase onto current main, guard the console-capture file write with `isFileLogLevelEnabled(level)` before `getLoggerLazy()`, add a regression test in `src/logging/console-capture.test.ts` for `logging.level: warn` suppressing `console.log` from file logging while preserving console passthrough, and drop or separately justify the unrelated named-agent test mock change.
