## Summary

Current `main` already moved `openclaw gateway status` onto a read-only status path that skips plugin loading, and the plugin SDK/docs now provide manifest/setup-entry and deferred-startup seams specifically to avoid full runtime registration on cold status/startup paths.

## Risks / Open Questions

- I did not replay the reporter's exact qclaw-plugin environment in this read-only review; this close is based on current code, tests, and changelog evidence.
- The evidence is strongest for the `gateway status` cold path explicitly called out in the changelog. External plugins that still do heavy synchronous work in their full `register(api)` path may still need plugin-side cleanup to benefit fully.
