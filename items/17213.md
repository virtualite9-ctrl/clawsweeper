## Summary

Keep #17213 open. Current main has narrower reliability work for lifecycle end/error fallback, incomplete terminal turns, and strict-agentic plan-only blocked states, but it still does not implement the requested atomic run.begin -> run.exec -> mandatory run.commit/run.abort contract with structured abort blockers across the shared event and gateway surfaces. The existing maintainer Codex review on the issue reached the same conclusion, #40710 is closed unmerged, and #55721 was a narrower shipped dispatch/incomplete-turn fix rather than a canonical replacement.

## Best Possible Solution

Keep #17213 open as a core reliability/product-contract item. The best path is a maintainer-approved, backward-compatible lifecycle design that defines explicit begin intent, execution evidence, mandatory commit or abort terminal state, and structured abort/blocker details across AgentEvent types, Gateway protocol/schema, agent emitters, plugin or harness-facing surfaces, clients, docs, and regression tests. Treat #40710 and #55721 as useful prior art and narrower fixes, not as superseding this issue.
