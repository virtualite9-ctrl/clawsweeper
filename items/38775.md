## Summary

Keep #38775 open. Current main still has explicit-order/lastUsed/cooldown auth-profile selection and reporting-only provider usage snapshots; it does not implement pre-request usage-aware routing, auth.balancing config, proactive utilization circuit breakers, polling/cache/shadow accounting, or an async plugin API that could externalize this cleanly.

## Best Possible Solution

Keep #38775 open for maintainer/product design. A solid implementation should add first-class usage-aware auth-profile routing across Gateway request paths: provider-owned Anthropic usage parsing mapped to per-profile headroom, cached/polled utilization state, proactive 5h/7d thresholds or circuit breakers, shadow accounting for concurrent starts, fallback to current reactive cooldown rotation for profiles without usage data, auth.balancing schema/help/docs, and regression tests for utilization-based profile choice plus cooldown interaction. If maintainers prefer plugin ownership, first add a documented async pre-request profile-selection API with usage-cache integration.
