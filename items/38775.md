## Summary

Keep open. Current main still routes auth profiles by explicit order, profile type, last-used round robin, and reactive cooldown/disabled state. The Anthropic OAuth usage endpoint is implemented for usage/status reporting, but there is no usage-aware auth-profile selection strategy or `auth.balancing` configuration that would proactively route to the profile with more headroom.
