## Summary

Keep #38775 open. Current main still implements ordered/last-used auth-profile rotation plus reactive cooldown/failover, while Anthropic usage snapshots are wired to `/usage` and status/reporting surfaces rather than pre-request profile selection. There is no `auth.balancing` config, usage-aware selector, proactive utilization circuit breaker, or async usage cache/polling path for Gateway routing.

## Best Possible Solution

Keep #38775 open for maintainer/product decision. A solid implementation would add first-class usage-aware auth-profile routing across Gateway request paths: provider-owned usage parsing for Anthropic, cached per-profile headroom state, proactive utilization thresholds/circuit breakers, shadow accounting for concurrent requests, fallback to current reactive rotation for profiles without usage data, `auth.balancing` schema/help/docs, and focused regression tests for utilization-based profile choice plus cooldown interaction. If maintainers prefer externalization, first add a documented async plugin API for pre-request profile selection and usage-cache integration.
