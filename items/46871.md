## Summary

Current main still has the reported split-brain behavior in the CLI: `openclaw nodes status` renders from live `node.list`, while unfiltered `openclaw nodes list` renders and counts only `node.pair.list`. If `node.list` contains a paired/connected node but `node.pair.list` is empty or stale, `nodes status` can still show `Paired: 1 · Connected: 1` while `nodes list` shows `Paired: 0`. The related fix PR #50871 described the right direction but is closed unmerged, and the current checkout does not contain that merge/live-state logic.

## Best Possible Solution

Keep the issue open and fix the CLI list path. The likely implementation belongs in `src/cli/nodes-cli/register.status.ts`: fetch `node.list` for `nodes list`, merge paired or connected live nodes that are absent from `node.pair.list`, compute summary counts from the rendered merged set, and keep `--connected`, `--last-connected`, and `--json` behavior stable. Add a regression test where `node.pair.list` returns `paired: []` and `node.list` returns a paired, connected node, asserting `nodes list` includes it and reports `Paired: 1`.
