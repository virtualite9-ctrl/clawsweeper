## Summary

Keep #38982 open. Current main has startup light/dark TUI palette support, but it still does not implement this PR's central runtime behavior: no `/theme` command, no `/theme list`, no searchable picker with live preview, and no cancel/revert or commit path. The overlapping #44382 is still open/unmerged in the provided context and is not established as the maintainer-designated canonical replacement.

## Best Possible Solution

Keep #38982 open unless a maintainer explicitly designates #44382 or another PR as canonical. The best product path is to coordinate the overlapping TUI theming PRs and land one focused implementation that adds a runtime theme API, registers `/theme`, supports `/theme list`, opens a searchable picker with live preview, commits on select, reverts on cancel, repaints all TUI surfaces consistently, updates public TUI docs, and adds targeted command/theme tests.
