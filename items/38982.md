## Summary

Keep #38982 open. Current main still does not ship the PR's central behavior: no `/theme` command, no `/theme list`, no runtime theme-switching API, and no searchable picker with live preview/cancel/commit. The related #44382 overlaps but is also open and not shown as a merged or maintainer-designated replacement.

## Best Possible Solution

Keep #38982 open unless a maintainer explicitly chooses #44382 as the canonical TUI theming path. The best product path is to coordinate the overlapping PRs and land one focused implementation that adds a runtime theme API, registers `/theme`, supports `/theme list`, opens a searchable picker with live preview, commits on select, reverts on cancel, repaints all TUI surfaces consistently, updates the public TUI docs, and adds targeted command/theme tests.
