## Summary

Current `main` still has the reported gateway session-reset bypass, and this PR is not safe to close as already implemented. The gateway handler still pre-resolves and persists an existing `sessionId` before dispatch, while `resolveSession()` still skips freshness rollover whenever `opts.sessionId` is provided. The PR’s proposed fix also appears incomplete versus the existing rollover helpers because the gateway path still carries rollover-sensitive state like `systemSent` and does not perform the bootstrap/session-file cleanup used elsewhere.

## Evidence
