## Summary

Current main already addresses the reported permanent Anthropic billing-disable state: billing failures are persisted as bounded `disabledUntil` windows, expired windows are cleared, single-provider billing cooldowns can be probed instead of skipped forever, and `models auth login` now clears provider lockouts during deliberate re-authentication. The implementation is also present in the available `v2026.4.24` release tag.

## Best Possible Solution

Close this issue as implemented. Keep the shipped bounded billing-disable state, probe behavior, and provider re-auth reset path; if maintainers want `openclaw onboard` itself to share the same stale-lockout clearing hook, track that as a narrow follow-up rather than keeping this broader permanent-stuck report open.
