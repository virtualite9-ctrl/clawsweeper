## Risks / Open Questions

- A naive automatic injection could leak sensitive or user-chosen agent display names into AWS CloudTrail or invocation logs; the eventual patch should use conservative values and document the exposure.
- AWS Bedrock request metadata has key/value/count and reserved-prefix constraints noted in the related discussion; the implementation should validate before forwarding.
- If implemented through provider hooks, the hook context likely needs a generic documented agent identity field rather than hardcoding Bedrock-specific behavior in unrelated core paths.
