## Summary

Current main still does not inject Bedrock `requestMetadata`, and #60698 is an open, unmerged implementation candidate with explicit closing linkage. The maintainer-safe outcome is to keep this issue open for PR review or a successor patch.

## Best Possible Solution

Keep the issue open and have maintainers review #60698 or a successor patch. The best implementation should add per-agent Bedrock `requestMetadata` on the active Bedrock stream path, derive agent id/name from verified runtime context, preserve non-Bedrock behavior, add focused option-injection coverage, and coordinate with #57360 for configurable metadata.
