## Summary

Current main still does not implement per-agent Bedrock `requestMetadata`, and an open, unmerged PR explicitly links itself as the implementation candidate for this issue. The issue should stay open for maintainer review of that PR or a successor patch.

## Best Possible Solution

Keep the issue open and have maintainers review #60698 or a successor patch. The best implementation should add per-agent Bedrock `requestMetadata` on the active pi-ai Bedrock stream path, derive agent id/name from verified runtime context, preserve non-Bedrock behavior, merge safely with any future configured metadata, and add focused tests for option injection and zero-extra-params behavior.
