## Summary

Keep open. Current main still has the supervised gateway respawn path exit with code 0, so PR #70466 is not implemented or obsolete. The canonical systemd unit now uses Restart=always, which mitigates standard installs, but the PR's narrower systemd Restart=on-failure/custom-unit behavior remains a real code-path gap and needs maintainer review plus a non-launchd systemd regression test.

## Best Possible Solution

Keep PR #70466 open for maintainer review. The best path is to update the PR against current main, split spawned and supervised restart handling as proposed, add a Linux/systemd supervised restart test that asserts immediate `exitProcess(1)` without the launchd delay, and explicitly note that canonical `Restart=always` mitigates standard installs but does not cover custom `Restart=on-failure` units.
