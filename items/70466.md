## Summary

Keep open. Current main still exits 0 for supervised gateway restarts, so the PR's stated behavior is not implemented on main. The current canonical systemd unit uses Restart=always, which mitigates fresh installs, but the linked report and PR target older/custom Restart=on-failure units where the exit code still matters.

## Best Possible Solution

Keep #70466 open. The best path is to land an equivalent focused fix that splits spawned and supervised restart handling, keeps spawned restarts at exit 0, preserves the launchd delay, exits non-zero for supervised restarts, and adds a non-launchd/systemd supervised regression test. If maintainers instead want `Restart=always` to be the only supported recovery contract, close the linked issue explicitly with docs/doctor migration guidance rather than auto-closing this PR as already implemented.
