## Summary

Keep open. Current main still exits 0 for the supervised full-process restart path, so the PR's requested non-zero systemd recovery behavior is not implemented on main. The current canonical systemd unit uses Restart=always, which mitigates standard new installs, but that does not make this PR obsolete for existing/custom Restart=on-failure services and does not satisfy the PR's code-path change. The linked issue #70354 provides concrete reproduction and logs, and the PR is a focused two-file fix, though it should add a non-launchd/systemd supervised restart regression test before landing.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to update the PR with a Linux/non-launchd supervised restart test asserting immediate exitProcess(1), preserve the spawned restart exit 0 behavior, and then land it if maintainers want OpenClaw to recover under existing or custom systemd Restart=on-failure units as well as the current canonical Restart=always unit.
