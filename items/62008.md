## Summary

Current `main` already covers the PR’s user-visible fix in a safer form: the OpenAI-compatible audio transcription caller now explicitly opts out of pinned DNS only for that multipart request, and the shared upload filename helper already rewrites `.aac` audio uploads to `.m4a`. The remaining PR delta is obsolete and conflicted with review feedback about broadening the shared `pinDns` default for all SDK consumers.
