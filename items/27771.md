## Summary

Keep PR #27771 open. Current main at e29d3516bf05591e6007bfb8ccdaea6d76194d41 still does not accept `channels.signal.configPath` or pass `--config` to the Signal daemon. The PR itself is stale against the current pluginized Signal layout, but the requested behavior remains valid, the contributor explicitly invited takeover, and an existing maintainer Codex review already recorded this as takeover/adaptation work rather than cleanup closure.

## Best Possible Solution

Keep the PR open for maintainer takeover or a replacement PR. The best implementation is to add optional `channels.signal.configPath` to the current Signal config type and strict schema, generated metadata/UI hints, and docs; trim and omit blank values; propagate it through `monitorSignalProvider` into `spawnSignalDaemon`; place `--config <path>` before daemon startup arguments; and add focused tests for set, blank, unset, and schema acceptance cases in the current `extensions/signal` layout.
