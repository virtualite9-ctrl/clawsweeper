## Summary

Keep this PR open. Current main still does not implement `channels.signal.configPath`, while the request remains a concrete bundled Signal plugin feature. The branch is stale against the current `extensions/signal` layout, but the useful path is maintainer takeover or a replacement patch, not conservative cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer takeover or a replacement patch. Port the feature to the current bundled Signal plugin by adding optional `channels.signal.configPath` to the Signal config type, strict schema, generated channel metadata/UI hints, docs, configured-account detection, and autostart tests; propagate it through `monitorSignalProvider` into `spawnSignalDaemon`; and emit `--config <path>` before the `daemon` subcommand while trimming and omitting blank values.
