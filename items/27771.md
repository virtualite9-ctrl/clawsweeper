## Summary

Keep PR #27771 open. Current main still does not support `channels.signal.configPath`, and the requested behavior remains concrete for the bundled Signal plugin. The existing patch is stale because Signal now lives under `extensions/signal`, but the feature itself has not been superseded or implemented.

## Best Possible Solution

Keep the PR open for maintainer takeover or a replacement patch. Port the feature to the current bundled Signal plugin by adding optional `channels.signal.configPath` to the Signal config type, strict schema, generated channel metadata/UI hints, docs, configured-account detection, and autostart tests; trim and omit blank values; propagate the value through `monitorSignalProvider` into `spawnSignalDaemon`; and emit `--config <path>` before the `daemon` subcommand.
