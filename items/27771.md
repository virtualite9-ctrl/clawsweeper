## Summary

Keep this PR open. Current main still does not implement `channels.signal.configPath`, and the request remains a concrete bundled Signal plugin feature rather than conservative cleanup. The branch is stale against the current `extensions/signal` layout, but the author explicitly invited maintainers to take over, so the best path is a maintainer port or replacement patch.

## Best Possible Solution

Keep this PR open for maintainer takeover or a replacement patch. Port the feature to the current bundled Signal plugin by adding optional `channels.signal.configPath` to the Signal config type/schema, generated metadata/UI hints, docs, configured-account detection, and autostart tests; propagate it through `monitorSignalProvider` into `spawnSignalDaemon`; and emit `--config <path>` before `daemon` while trimming and omitting blank values.
