## Summary

Keep #71837 open. Current main has partial memory-core dreaming denoise and phase-signal support, but the PR's central behavior is still not implemented: automatic REM candidate truths are not written back as grounded short-term candidates, deep gating/diversity still ignores phase reinforcement, and the package session extractor lacks the broader scaffold stripping proposed by the PR.

## Best Possible Solution

Keep #71837 open. The best path is to rebase it onto current main, preserve existing denoise safeguards, address the review comments around REM weighting/query identity, and validate the memory-core session extraction, REM preview, and short-term promotion tests before maintainer merge consideration.
