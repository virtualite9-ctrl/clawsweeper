## Summary

Keep open. Current `main` still uses `Dockerfile.sandbox-common` for the documented `scripts/sandbox-common-setup.sh` flow, and that Dockerfile does not implement the requested `apt-get` retry or `--fix-missing` handling.

## Evidence

- **Requested hardening is absent in current sandbox-common Dockerfile:** `Dockerfile.sandbox-common:24-28` runs `apt-get update`, `apt-get upgrade`, and `apt-get install` directly. There is no `Acquire::Retries` option and no `--fix-missing` flag.
  - file: [Dockerfile.sandbox-common:24](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/Dockerfile.sandbox-common#L24)

  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **The reported path is active and documented on main:** `scripts/sandbox-common-setup.sh:36-47` still builds `Dockerfile.sandbox-common`, and `docs/gateway/sandboxing.md:357-365` still tells users to run that script to build `openclaw-sandbox-common:bookworm-slim`.
  - file: [scripts/sandbox-common-setup.sh:36](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/scripts/sandbox-common-setup.sh#L36)

  - file: [scripts/docker/install-sh-nonroot/Dockerfile:7](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/scripts/docker/install-sh-nonroot/Dockerfile#L7)
