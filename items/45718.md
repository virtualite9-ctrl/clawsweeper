## Summary

Keep open. Current main partially narrows the reported behavior, but it does not fully fix the session-store bloat: `skillsSnapshot` and `systemPromptReport` are still serialized into `sessions.json`, and the linked fix PR #48425 is still open/unmerged.

## Best Possible Solution

Keep #45718 open until a core session-store serialization fix lands. The best path is to review, land, or supersede #48425 with a main-branch change that omits truly transient `skillsSnapshot` data from `sessions.json`, aligns disk-budget measurements with the saved representation, preserves only the required warning-dedupe state from `systemPromptReport`, and adds regression coverage for save/load behavior.
