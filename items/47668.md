## Summary

Keep #47668 open. Current main has several partial mitigations for context growth, including default auto-compaction, overflow retry compaction, live tool-result truncation, and optional session pruning. However, the specific discussion-linked strikethrough context-file issue is still not implemented on main, pruning remains off by default for non-Anthropic/Ollama setups, and the related fix PR #70605 is still open rather than merged. The issue is recent, actionable, and not safe to close as implemented or superseded yet.

## Best Possible Solution

Keep the issue open until maintainers either merge a tested sanitizer fix for struck-through context-file content and decide the remaining Ollama/non-Anthropic context-growth behavior, or explicitly mark a canonical follow-up such as the broader memory/context umbrella. A good implementation path is a narrow strikethrough sanitizer with regression tests in src/agents/system-prompt.test.ts, followed by product judgment on whether non-Anthropic session pruning or earlier compaction thresholds should be defaulted or documented for local-model users.
