## Summary

Keep open. Current main does not implement `channels.msteams.webhook.host`: the Microsoft Teams config type/schema/docs still expose only `webhook.port` and `webhook.path`, and the runtime still calls `expressApp.listen(port)` without a host. The PR is a focused implementation for still-open #62766, though its existing review comments should be addressed before merge.

## Best Possible Solution

Keep #63347 open and ask for a current-main refresh that preserves the focused implementation: add `channels.msteams.webhook.host` to types/schema/generated metadata, pass a trimmed nonblank host to `expressApp.listen(port, host)`, preserve existing default bind behavior unless a maintainer explicitly changes the product default, log either the actual bound address from the server or a clearly unspecified bind marker, document loopback use without breaking the baseline direct-ingress setup, and keep lifecycle coverage for default, configured-host, and blank-host behavior.
