## Summary

Keep open. Current main still has unguarded gateway-critical `os.networkInterfaces()` paths: `pickPrimaryLanIPv4()` and `listTailnetAddresses()` call `readNetworkInterfaces()`, which directly invokes `os.networkInterfaces()` and propagates sandbox `uv_interface_addresses` failures. The tests still assert throwing behavior for the reported failure. The linked PR #71950 is relevant and targets the right behavior, but the provided context says it is open and unmerged, so the issue is not implemented on main.

## Best Possible Solution

Keep #71721 open and land PR #71950, or an equivalent maintainer-reviewed patch, that switches the gateway LAN and tailnet discovery callers to `safeNetworkInterfaces()` and updates the regression tests to assert graceful fallback instead of thrown errors.
