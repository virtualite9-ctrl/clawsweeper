## Summary

Keep open. Current main still propagates `os.networkInterfaces()` failures through the gateway LAN and tailnet discovery paths, and the existing regression tests explicitly expect those helpers to throw on a `uv_interface_addresses` failure. The linked PR #71950 is relevant and appears to target the right files, but it is open and unmerged, so it does not supersede or implement the issue on main yet.

## Best Possible Solution

Keep #71721 open and use PR #71950, or an equivalent maintainer-reviewed patch, to switch the gateway LAN and tailnet callers from `readNetworkInterfaces()` to the safe helper and update the tests to assert graceful fallback instead of thrown errors.
