## Summary

Keep open. Current `main` supports interactive Codex OAuth via browser login and device pairing, but it does not implement the requested non-interactive setup path for `openai-codex` via env/callback input.

## Evidence

- **`models auth login` is still TTY-only:** `modelsAuthLoginCommand` immediately throws when stdin is not a TTY, so the Codex auth entrypoint is not usable as a non-interactive command.
  - file: [src/commands/models/auth.ts:584](https://github.com/openclaw/openclaw/blob/d38ed0831d7372c8505dc944aea69b5b83da1435/src/commands/models/auth.ts#L584)
  - command: `nl -ba src/commands/models/auth.ts | sed -n '580,660p'`
  - sha: [d38ed0831d73](https://github.com/openclaw/openclaw/commit/d38ed0831d7372c8505dc944aea69b5b83da1435)
- **Non-interactive onboarding requires provider support that Codex auth does not provide:** The non-interactive onboarding path exits with an error when a provider auth method lacks `runNonInteractive`.
  - file: [src/commands/onboard-non-interactive/local/auth-choice.plugin-providers.ts:130](https://github.com/openclaw/openclaw/blob/d38ed0831d7372c8505dc944aea69b5b83da1435/src/commands/onboard-non-interactive/local/auth-choice.plugin-providers.ts#L130)
  - command: `nl -ba src/commands/onboard-non-interactive/local/auth-choice.plugin-providers.ts | sed -n '90,140p'`
  - sha: [d38ed0831d73](https://github.com/openclaw/openclaw/commit/d38ed0831d7372c8505dc944aea69b5b83da1435)
- **OpenAI Codex auth methods are interactive-only:** The bundled `openai-codex` provider defines `oauth` and `device-code` auth methods with `run(...)` handlers only; there is no `runNonInteractive` implementation for either method.
  - file: [extensions/openai/openai-codex-provider.ts:418](https://github.com/openclaw/openclaw/blob/d38ed0831d7372c8505dc944aea69b5b83da1435/extensions/openai/openai-codex-provider.ts#L418)
  - command: `nl -ba extensions/openai/openai-codex-provider.ts | sed -n '414,452p'`
  - sha: [d38ed0831d73](https://github.com/openclaw/openclaw/commit/d38ed0831d7372c8505dc944aea69b5b83da1435)
- **No env-backed Codex OAuth auth surface is declared:** The plugin manifest declares `providerAuthEnvVars` only for `openai` via `OPENAI_API_KEY`; there is no corresponding env variable surface for `openai-codex`.
  - file: [extensions/openai/openclaw.plugin.json:9](https://github.com/openclaw/openclaw/blob/d38ed0831d7372c8505dc944aea69b5b83da1435/extensions/openai/openclaw.plugin.json#L9)
  - command: `nl -ba extensions/openai/openclaw.plugin.json | sed -n '1,14p'`
  - sha: [d38ed0831d73](https://github.com/openclaw/openclaw/commit/d38ed0831d7372c8505dc944aea69b5b83da1435)
- **Docs describe headless interaction, not unattended setup:** The OpenAI provider docs still instruct users to run interactive login and, for headless setups, use `--device-code`; they do not document any env/callback-url based non-interactive Codex OAuth setup.
  - file: [docs/providers/openai.md:112](https://github.com/openclaw/openclaw/blob/d38ed0831d7372c8505dc944aea69b5b83da1435/docs/providers/openai.md#L112)
