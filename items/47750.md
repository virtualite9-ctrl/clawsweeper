## Summary

Keep open. Current main does not provide a non-interactive Codex OAuth setup path: `models auth login` is still TTY-gated, the CLI exposes no callback/env option, the bundled OpenAI Codex provider only registers browser OAuth and device-code auth methods, and docs still describe browser/device-code/manual redirect paste flows rather than an env-supplied callback URL.

## Best Possible Solution

Keep the issue open as a valid bundled OpenAI provider/auth enhancement. The likely product path is to add a narrowly scoped non-interactive Codex OAuth completion surface, probably on `models auth login --provider openai-codex` and/or non-interactive onboarding, with explicit callback-url input, state validation, auth-profile persistence, tests for non-TTY behavior, and docs under the OpenAI provider and CLI auth pages.
