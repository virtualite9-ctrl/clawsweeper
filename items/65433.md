## Summary

Keep this PR open. Current main still does not contain the PR's Gemini CLI 3.1 template preference or Sonnet 4.6 default-1M status behavior, and the NUL hardening is only partly overlapping: field-level inbound metadata strips NULs, but the generic untrusted JSON sanitizer still does not strip them from arbitrary structured payload strings.

## Best Possible Solution

Keep this PR open for maintainer review, but ask for a rebase or split. The NUL portion should be reduced to the remaining generic `sanitizeUntrustedJsonValue` gap if still needed; the Gemini template change should be validated against the current Gemini CLI/catalog behavior; and any Sonnet 4.6 1M fallback should follow the explicit-provider guard pattern used for Opus 4.7 while preserving `models.providers` overrides.
