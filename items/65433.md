## Summary

Keep this PR open. Current main still lacks the PR's Gemini 3.1-first CLI template behavior and Sonnet 4.6 default-1M context fallback, and the NUL handling only partly overlaps because arbitrary structured payload strings still pass through the generic JSON sanitizer without NUL stripping. The PR is not ready to land as-is because the Sonnet fallback needs the existing explicit-provider guard and the Gemini Flash template change needs validation against current model-ID normalization.

## Best Possible Solution

Keep this PR open for maintainer review, ideally after a rebase or split. The NUL hunk should be reduced to the remaining generic `sanitizeUntrustedJsonValue` gap if still desired; the Gemini template hunk should be validated against current Gemini CLI/catalog behavior and existing Flash alias normalization; and the Sonnet 4.6 1M fallback should follow the explicit-provider guard pattern while preserving `models.providers` overrides and adding focused status/context regression tests.
