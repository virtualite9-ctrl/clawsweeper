## Summary

Keep #59818 open. Current main still has ASCII-only Discord mention-candidate parsing, account-scoped directory lookup, no preflight/startup cache seeding for bot-to-bot handles, and no CJK regression coverage. The PR is still relevant, but the provided review thread raises unresolved correctness risks in the proposed implementation, so it should be revised rather than closed as implemented or obsolete.

## Best Possible Solution

Keep the PR open or consolidate it with the related Unicode mention work only after a maintainer chooses a canonical branch. The safest implementation path is a Discord-plugin fix that keeps mention boundaries narrow, supports Unicode/CJK handles, rewrites before delivery/chunk boundaries where needed, populates handle cache only from policy-allowed/scoped contexts, scopes guild nicknames by guild or excludes them from an account-global map, and adds regression tests for CJK handles, already-formatted mentions, URLs, duplicate handles, and multi-bot startup/preflight seeding.
