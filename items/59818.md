## Summary

Keep open. Current `main` still lacks the requested CJK and cross-account Discord mention rewrite behavior, and the PR’s proposed implementation still has unresolved correctness concerns from review that need maintainer follow-up rather than cleanup closure.

## Evidence

- **Current matcher is still ASCII-only:** `MENTION_CANDIDATE_PATTERN` on `main` only matches `[a-z0-9_.-]`, so CJK handles such as `@架构师` are not recognized for rewriting.
