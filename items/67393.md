## Summary

Keep open. The report is recent, specific, and still not proven fixed on current main: the current tree has useful Gateway startup import/RSS reductions, but no source, docs, tests, or release evidence that addresses the reported Linux VmSize/VSZ 22GB anonymous-mapping symptom with normal RSS.

## Best Possible Solution

Keep this issue open and request a focused current-main or latest-release reproduction: exact OpenClaw and Node versions, launch method, ps RSS/VSZ over time, /proc/<pid>/smaps or pmap -x summaries, mapping counts, enabled memory backend/plugins/providers, safe-mode or minimal-config results, and whether current unreleased Gateway startup deferrals change VmSize. If this is benign Linux/Node virtual address reservation, document it in Linux/Gateway troubleshooting; if RSS, commit, or OOM pressure grows, fix the concrete Gateway startup path or dependency that owns the mappings.
