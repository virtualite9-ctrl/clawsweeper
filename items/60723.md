## Summary

Keep open. Current `main` does not implement either half of this PR: the Android `play` flavor still disables SMS/Call Log, and `querySmsMmsMessagesByPhone` still does per-row MMS hydration on the mixed by-phone path. The PR discussion also contains an unresolved P1 regression note that the proposed 50-row MMS hydration cap can drop valid results before filtering/pagination, so there is not strong evidence to close this conservatively.
