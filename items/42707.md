## Summary

Keep this PR open. Current main still has the Telegram thread-binding manager and generic session-binding support, but it still does not register Telegram subagent lifecycle hooks and the agent resume path still does not hydrate run context from a resolved session entry. The PR is not merge-ready as written because it was authored against older module paths and imports core internals from Telegram production code, but the central bug is still real and the patch remains a useful implementation candidate.

## Best Possible Solution

Keep this PR open as a stale but relevant implementation candidate. Rebase it onto current main; add Telegram registerFull/subagent-hooks-api wiring like Discord/Matrix/Feishu; keep handler code inside the Telegram plugin boundary using current Telegram target helpers and plugin SDK facades; preserve the already-shipped shared session-binding and Telegram thread-binding state; hydrate resolveAgentRunContext from the resolved SessionEntry on --session-id resumes; and add focused tests for Telegram sessions_spawn(thread=true, mode="session"), resume routing hydration, delivery-target resolution, unbind cleanup, and legacy telegram:group:/tg: target forms. The related open feature issue #42986 can remain the tracking issue, but this PR should not be auto-closed while it still contains relevant implementation work.

## Risks / Open Questions

- The PR is stale against current main: Telegram now uses defineBundledChannelEntry/registerFull-style bundled entries, while the patch modifies an older register(api) shape.
- The PR imports core internals from Telegram production code via ../../../src paths, violating the extension boundary; a rebased version should use plugin SDK facades or local Telegram barrels.
- The review comment about target-prefix normalization is material: the PR's resolver strips telegram: but not telegram:group: or tg: forms, while current Telegram target helpers explicitly handle those prefixes.
- The PR changes session binding and outbound completion routing, so maintainer review should require account scoping, target normalization, unbind cleanup, and regression tests for topic-qualified Telegram targets before merge.
