## Summary

Keep this PR open. Current main has some Telegram thread-binding infrastructure and shared session-binding adapter state, but it still lacks Telegram subagent lifecycle hook registration and still does not hydrate resumed agent run context from persisted session delivery metadata. The PR is stale and not merge-ready as written, but the central Telegram `sessions_spawn(thread=true, mode="session")` bug remains real and the patch remains a useful implementation candidate alongside the related open feature issue #42986.

## Best Possible Solution

Keep this PR open as a stale but relevant implementation candidate. Rebase it onto current main; add Telegram `registerFull` lifecycle hook wiring like Discord, Matrix, and Feishu; keep Telegram hook code inside the plugin boundary through SDK or local API facades; reuse the shipped Telegram thread-binding manager and session-binding adapter; hydrate `resolveAgentRunContext` from the resolved `SessionEntry` on `--session-id` resumes; and add focused tests for Telegram `sessions_spawn(thread=true, mode="session")`, resume routing hydration, delivery-target resolution, unbind cleanup, and legacy Telegram target prefixes. The related open issue #42986 can continue tracking the feature until a corrected implementation lands.
