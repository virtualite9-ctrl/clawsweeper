## Summary

Keep this PR open. Current main has Telegram thread-binding storage/adapters and shared adapter state, but it still does not register Telegram subagent lifecycle hooks and the agent CLI resume path still does not hydrate requester routing from the persisted session entry. The PR is not ready to land as-is because current main has moved to plugin SDK facades and the review comment about Telegram target-prefix normalization is valid, but the underlying product bug is not fully implemented on main.

## Best Possible Solution

Keep the PR open as a stale but still relevant implementation candidate. The best path is to rebase it onto current main, implement Telegram subagent hook wiring through a Telegram `registerFull`/lazy `subagent-hooks-api.ts` pattern like Discord and Matrix, use plugin SDK runtime facades instead of core-relative imports, reuse Telegram target parsing helpers, preserve the already-shipped global adapter map behavior, add session-entry hydration to the current `src/agents/command/run-context.ts` path, and add focused tests for Telegram topic-bound `sessions_spawn(thread=true, mode="session")`, `agent --session-id` resume context, cleanup, and legacy target prefixes.
