## Summary

Keep #48117 open. Current main still routes the default Windows PowerShell installer through a single `npm install -g` attempt and returns failure on any nonzero npm exit; there is no TLS certificate-error detection, CA export, or scoped retry for `UNABLE_TO_VERIFY_LEAF_SIGNATURE`. The related Windows fix PR #48416 was closed unmerged, while the merged TLS CA work in #51146 is Linux/nvm-specific and does not cover this Windows installer report.

## Best Possible Solution

Keep the issue open and implement a Windows installer fix in `scripts/install.ps1`: classify npm TLS certificate failures from captured stdout/stderr, then either establish the correct CA trust for the npm subprocess or perform a narrowly scoped retry with a clear warning and remediation guidance. Add targeted installer tests for the known npm error strings and update the Windows installer docs if the user-facing behavior changes.

## Risks / Open Questions

- A naive `strict-ssl=false` fix would weaken TLS verification if applied globally or silently; any retry should be scoped to this installer invocation and clearly explained to the user.
