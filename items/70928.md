## Summary

Current main still does not route the WebChat/Gateway TUI outbound reply path through the hook-aware delivery seams. `chat.send` creates a raw reply dispatcher and calls the lower-level inbound dispatcher, so `message_sending` is not installed for this path and `message_sent` is not emitted by the reply dispatcher. The transcript append path also bypasses the existing `before_message_write` helpers. The related broad tracker #50126 remains the right umbrella, and #12584 was closed unmerged, so this issue still represents real remaining work.

## Best Possible Solution

Keep this issue open and fix the WebChat/Gateway TUI path directly or under #50126. The best implementation is to move `chat.send` reply delivery onto a shared hook-aware dispatcher/observer seam, or explicitly add equivalent `message_sending`, `message_sent`, and `before_message_write` calls around the WebChat/TUI reply and transcript append paths. Add regression coverage for `chat.send` and Gateway-backed TUI so a registered plugin hook observes assistant replies on these default local surfaces before closing.
