## Summary

Keep #70928 open. Current main still has a real WebChat/TUI outbound lifecycle gap for `message_sending`/`message_sent`: WebChat `chat.send` uses an internal UI reply path instead of the outbound delivery hook seam, and local TUI emits final chat events from agent lifecycle events without running message delivery hooks. Some related hooks are partially implemented now, notably `agent_end` for normal agent runs and `before_message_write` for guarded SessionManager writes, so the issue should probably be narrowed rather than closed. The related PR #12584 is closed unmerged, and the broader #50126 umbrella remains open, but #70928 adds specific WebChat/TUI coverage that is not proven fixed on main.

## Best Possible Solution

Keep the issue open and either implement a shared UI/internal outbound lifecycle observer for WebChat and TUI or explicitly fold this WebChat/TUI scope into #50126 with maintainer confirmation. Add focused regression tests for `chat.send` and local/gateway TUI proving the expected `message_sending`/`message_sent` behavior, and document any channel matrix or intentional exclusions for `before_message_write`.
