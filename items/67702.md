## Summary

Keep this PR open. Current main still has no pre-git-update guard or recovery path for a same-install gateway that is running while git-mode update mutates the checkout in place. The PR is a relevant implementation candidate, but its current diff still needs revision for wrapper-backed service commands and for restoring a gateway if the update is skipped or fails after a pre-stop. Security review found no supply-chain-sensitive files in the PR diff, but the availability risk from stopping a managed gateway without guaranteed restoration is real.

## Best Possible Solution

Keep this PR open and request a revised implementation in src/cli/update-cli/update-command.ts: detect a same-install running gateway before in-place git checkout/build work, handle direct dist entrypoints and wrapper-backed service commands, and only stop the service when restart/restoration is guaranteed across success, skipped, and failed update paths. Add focused src/cli/update-cli.test.ts coverage for absolute, relative, wrapper, different-install, --no-restart, dirty/skipped, and failed-update scenarios.
