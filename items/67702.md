## Summary

Keep this PR open. Current main still lacks a pre-git-update guard or restoration path for a same-install gateway that is running while git-mode update mutates dist in place. The PR is a relevant implementation candidate, but the current patch still needs revision for wrapper-backed service commands and for restoring a gateway that was stopped before a skipped or failed update.

## Best Possible Solution

Keep this PR open and request a revised implementation in `src/cli/update-cli/update-command.ts`: detect a same-install running gateway before in-place git checkout/build work, handle direct dist entrypoints and wrapper-backed service commands, and only stop when restart/restoration is guaranteed across success, skipped, and failed update paths. Add focused `src/cli/update-cli.test.ts` coverage for absolute, relative, wrapper, different-install, `--no-restart`, dirty/skipped, and failed-update scenarios.
