## Summary

Keep this PR open. Current main still lacks the LINE schema guard the PR proposes: `LineConfigSchema` and account schemas accept `dmPolicy: "open"` without requiring `allowFrom: ["*"]`, while runtime handling treats `open` as allowing all DM senders. The PR is a focused LINE-owned hardening change with favorable review context, not a cleanup-close candidate.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land a LINE-owned schema fix that reuses `requireOpenAllowFrom`, includes focused tests for top-level and account-level `dmPolicy: "open"` cases, and resolves the remaining account-inheritance/migration-note questions before merge.
