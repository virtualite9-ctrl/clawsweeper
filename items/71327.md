## Summary

Keep PR #71327 open. Current main does not implement the LINE schema guard: `dmPolicy: "open"` can still be parsed with missing or non-wildcard `allowFrom`, while the LINE runtime treats `open` as accepting all DM senders. The PR is a focused, plausible extension-owned hardening change with tests in the provided diff and is not obsolete or superseded by current main.

## Best Possible Solution

Keep the PR open for maintainer review and normal validation. The best path is to merge a focused LINE-owned schema hardening change with regression tests, plus any maintainer-requested migration/docs note, because current main still has the mismatch and no narrower canonical item supersedes it.
