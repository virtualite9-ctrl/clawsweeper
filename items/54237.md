## Summary

Current `main` does not implement a Feishu card-header disable/customize option. Card replies still always derive the header title from agent identity when cards are used, so the issue's requested behavior is not implemented on `main` beyond the existing global `renderMode: "raw"` workaround.

## Evidence

- **Feishu config has render mode but no card-header setting:** `extensions/feishu/src/config-schema.ts` defines `renderMode`, `streaming`, and related Feishu options, but there is no `cardHeader`, `cardHeaderTitle`, or equivalent header customization flag.
  - file: [extensions/feishu/src/config-schema.ts:48](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/extensions/feishu/src/config-schema.ts#L48)
  - command: `nl -ba extensions/feishu/src/config-schema.ts | sed -n '1,320p'`
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Card header is built unconditionally from identity:** `resolveCardHeader` always returns a title from `identity.name`/`identity.emoji` or `agentId`, with no disable/empty-title branch.
  - file: [extensions/feishu/src/reply-dispatcher.ts:45](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/extensions/feishu/src/reply-dispatcher.ts#L45)
