## Summary

Keep #63688 open. Current main has overlapping empty-response retry machinery, but it is scoped differently from this PR: clean empty responses are retried only through the strict-agentic/Gemini incomplete-turn guard, while MiniMax is not in that supported provider/model set. The PR discussion specifically cited MiniMax Anthropic-compatible SSE returning a clean stop with no content, and current main still has live-test handling that treats MiniMax empty responses as a skip condition. The PR also has an unresolved P1 review concern about yield/client-tool turns, so the work should be rebased and narrowed rather than auto-closed as implemented.

## Best Possible Solution

Keep the PR open or ask the contributor to rebase against the current incomplete-turn helpers. The best path is a narrow provider-safe implementation for MiniMax or Anthropic-compatible clean-stop empty responses, with tests proving no retry after clientToolCall, yieldDetected, deterministic approval prompts, messaging delivery, mutating tool side effects, visible text, or stopReason=error paths. The test comments should also be translated to English before review can proceed.
