## Summary

Current main implements the intended fix in a different, stronger shape: memory-search config resolution no longer cold-loads plugin capability providers from the runtime resolver, so the old missing-cfg warning path is gone. The related discussion and #66856 tracing were valid, but this PR’s exact cfg-threading diff is now obsolete.

## Best Possible Solution

Close #63848 as implemented on main. Keep the shipped boundary where memory-search config resolution uses only already-registered adapters, and handle any still-reproducing status/doctor/gateway warning reports in narrower follow-up issues rather than merging this stale one-file diff.
