## Summary

Current `main` does not implement the requested Cron calendar/timeline UI. The shipped Control UI cron surface is still a filterable flat jobs list plus run history and edit form, so the issue remains a coherent open feature request.

## Evidence

- **Docs still describe list-based cron support:** The Control UI docs enumerate cron support as `list/add/edit/run/enable/disable + run history` and the Cron panel notes cover form validation and edit controls, but do not mention a calendar, timeline, week view, drag-and-drop rescheduling, or heatmap stats.
