## Summary

Keep this issue open. Current main still shows the Control UI cron surface as a summary strip, filterable flat jobs list, run-history list, and edit form; it does not implement the requested timeline/calendar/day/week views, heatmap, drag-to-reschedule flow, or real cron-expression validation. The provided GitHub context also shows #41892 as an open, unmerged implementation candidate with closing syntax, so the issue should remain open until that PR lands or maintainers choose a replacement plan.

## Best Possible Solution

Keep this issue open as the canonical Control UI cron calendar/timeline request. Review, land, shrink, or replace #41892 as the implementation path; close only after a merged mainline change ships a maintainer-approved cron timeline/calendar surface, or after maintainers create a narrower canonical follow-up for any remaining scope.
