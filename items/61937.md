## Summary

Current `main` contains the multipart FormData normalization, auth-header preservation, and multipart-specific `pinDns: false` handling that address this transcription failure; the changelog says the multipart field-drop fix shipped in `v2026.4.12`, and a follow-up commenter confirmed the issue was no longer reproducing on `2026.4.14`.

## Evidence

- **Runtime fetch now normalizes cross-realm FormData before dispatcher-backed undici fetch:** `normalizeRuntimeFormData(...)` rebuilds incoming `FormData` with the runtime `undici.FormData` implementation, preserves filenames, and clears stale `content-length`/`content-type` before `fetchWithRuntimeDispatcher(...)` calls runtime fetch.
