## Summary

Current main still matches the reported behavior: `/status` is advertised as a slash command, but it is not marked `executeLocal`, and the web UI slash menu only sends immediately on Enter for commands that are `executeLocal` and have no args. Selecting `/status` therefore fills the draft and requires a second Enter. The linked PR #45602 is still open/not merged, and the simple `executeLocal: true` approach would need a matching local status executor or a different menu-send path.

## Best Possible Solution

Keep the issue open and fix the web UI command-selection path deliberately. Either make selecting an exact no-arg Gateway command like `/status` submit the draft once, or implement a real local `/status` executor before setting `executeLocal: true`; add focused regression coverage for selecting `/status` from the slash menu with one Enter.
