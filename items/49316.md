## Summary

Keep open. Current main still ships `@mariozechner/pi-ai` `0.70.0`, and the published `openai-completions` parser in that version still reads only `prompt_tokens` / `completion_tokens`. OpenClaw has downstream alias normalization, but it runs after `pi-ai` has already produced `message.usage`, and there is no repo-side patch overriding that parser. That means the mlx-vlm `input_tokens` / `output_tokens` mismatch described in #49316 is still plausible on current main.
