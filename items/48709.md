## Summary

Keep #48709 open. Current main has fixes around Gemini thinking budgets, visible `<think>` tag stripping, and mixed text/tool streaming, but the central high-severity `textSignature` bloat path is still present: Gemini `thoughtSignature` on normal text is persisted as `textSignature` and replayed to the same Google model as `thoughtSignature`. The Google replay sanitizer strips `thoughtSignature` fields, not `textSignature`, so the reported session-history bloat and re-submission pressure are not fully resolved.

## Best Possible Solution

Keep the issue open and fix it in the Google provider/replay path: decide whether same-model Gemini text signatures should be dropped, capped, externalized, or retained only for the most recent replay-required turn; add focused tests that large `textSignature` values do not accumulate or get resent unbounded, while preserving valid Gemini tool/thinking signature replay where required. The existing think-tag and mixed text/tool mitigations should be kept, but they are not sufficient to close the central bloat/session-failure report.
