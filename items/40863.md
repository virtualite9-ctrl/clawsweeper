## Summary

Keep open: current `main` still contains the core behavior the issue calls out. `sessions_send`'s A2A follow-up wait is still hard-capped at 60s, and the regular `sessions_send` wait path still defaults to 30s and returns `status: "timeout"` on that path independently of `agents.defaults.subagents.announceTimeoutMs`.

## Evidence

- **A2A wait path still capped at 60s:** `runSessionsSendA2AFlow` still calls `waitForAgentRun` with `Math.min(params.announceTimeoutMs, 60_000)`, so configured values above 60s are not fully honored on this path.
  - file: [src/agents/tools/sessions-send-tool.a2a.ts:50](https://github.com/openclaw/openclaw/blob/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503/src/agents/tools/sessions-send-tool.a2a.ts#L50)
