## Summary

Keep this issue open. Current main at 7d9dc8cf24fc143bec95318491e0247c37863e63 still does not expose `channels.whatsapp.silentErrorReplies`; Telegram has the config and applies it to error sends, while WhatsApp schema/UI/runtime/docs lack the equivalent. The open #68465 PR is a closing-syntax implementation candidate, so the issue should stay open pending review/merge.

## Best Possible Solution

Keep the issue open and review #68465. A complete fix should add `channels.whatsapp.silentErrorReplies` to the WhatsApp schema/types/generated metadata/UI hints/docs, suppress only gateway-synthesized `isError` replies when enabled, preserve ordinary agent text responses, and include enabled/disabled WhatsApp inbound-dispatch regression tests.
