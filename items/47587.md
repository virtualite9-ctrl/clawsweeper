## Summary

Keep open. Current main appears to fix the media-header half of #47587, but the central multi-image understanding gap remains: media understanding still selects only the first image by default, and docs confirm `tools.media.*.attachments` defaults to `mode: "first"` and `maxAttachments: 1`. Related #54484 covers path/header exposure, not automatic transcription of every Telegram album image.

## Best Possible Solution

Keep #47587 open and narrow it to the remaining product/code decision: Telegram media albums should either opt into all-image understanding by default, or the media-understanding default/policy should change for multi-image inbound messages. Add a regression test with a 2-3 image Telegram media group proving the agent prompt includes every media path and either `[Image 1/N]...` descriptions for all images or clearly documented opt-in behavior if maintainers choose configuration over default processing.
