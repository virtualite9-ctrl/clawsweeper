## Summary

Keep this issue open. Current main still has the LiteLLM/OpenAI tool-schema compatibility gap for bundle MCP tools, and #66415 is only an open implementation candidate rather than a shipped fix.

## Best Possible Solution

Keep this issue open. The maintainer path is to review #66415 or a superseding PR, require regression coverage that feeds bundle MCP-style schemas through LiteLLM/OpenAI completions serialization, and add post-materialization OpenAI-compatible schema normalization or a provider-family hook that handles nested array-valued `type`, missing object metadata, and optional fields without changing non-OpenAI behavior. Diagnostics should also expose the rejected tool name and schema path.
