## Summary

Keep open. Current main still uses the sandbox browser socat relay and CDP port-offset design this issue asks maintainers to evaluate. The related #62873 work shipped readiness/fail-fast hardening, but it explicitly left the proxy architecture in place, and current source, docs, and tests still treat the relay and cdpSourceRange behavior as supported sandbox-browser contract.

## Best Possible Solution

Keep this issue open as the sandbox browser CDP architecture tracking item unless a maintainer explicitly decides to retain the relay. A complete solution should remove the socat dependency and port-offset logic from `Dockerfile.sandbox-browser` and `scripts/sandbox-browser-entrypoint.sh`, update `src/agents/sandbox/browser.ts` orchestration/config hashing and source-range enforcement, preserve equivalent readiness and ingress/access-control guarantees, and update sandbox docs plus tests around `cdpSourceRange` or its replacement.
