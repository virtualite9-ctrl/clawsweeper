## Summary

Keep open. Current main still uses the sandbox browser socat relay, CDP port offset, and cdpSourceRange ingress model this issue asks maintainers to evaluate. The related merged hardening PR improved readiness/fail-fast behavior but explicitly deferred full socat removal as follow-up architecture work.

## Best Possible Solution

Keep this issue open as the core sandbox browser CDP architecture tracker. A complete implementation should remove socat and CDP port-offset logic from Dockerfile.sandbox-browser and scripts/sandbox-browser-entrypoint.sh only if maintainers accept direct CDP, update src/agents/sandbox/browser.ts orchestration and config hashing, preserve equivalent readiness and access-control guarantees, and update sandbox docs plus focused tests. If maintainers decide the relay remains necessary, document that decision and close this issue then.
