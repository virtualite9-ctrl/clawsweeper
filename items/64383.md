## Summary

Keep open. Current main still uses the sandbox browser socat relay and CDP port-offset design this feature asks maintainers to evaluate. The related #62873 work shipped readiness/fail-fast hardening, but it explicitly left the proxy architecture in place, and current code, docs, and tests still treat the relay and cdpSourceRange as part of the supported sandbox browser contract.

## Best Possible Solution

Keep this issue open as the architectural tracking item unless a maintainer explicitly decides to retain the relay. A direct-CDP implementation should remove the socat dependency and port-offset logic from Dockerfile.sandbox-browser and scripts/sandbox-browser-entrypoint.sh, update src/agents/sandbox/browser.ts orchestration and config hashing, preserve equivalent readiness and ingress/access-control guarantees, and update the sandbox docs and tests around cdpSourceRange or its replacement.
