## Summary

Current main already implements the reported Feishu pre-tool delivery behavior. The related PR #46019 was closed unmerged, but its core race is covered on main: block reply callbacks are invoked synchronously, tool-start handling flushes buffered block replies before continuing, Feishu streams visible partial text into its card UI, and regression tests cover the pre-tool flush ordering.

## Best Possible Solution

Close #46002 as implemented on main and keep the shipped synchronous block-reply/tool-start flush plus Feishu streaming-card partial delivery path. If users still reproduce this on v2026.4.24 or newer, track it as a fresh regression with exact Feishu streaming/renderMode config, logs, version, and a short transcript.
