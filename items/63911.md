## Summary

Current `main` already contains a Discord preflight fix that hydrates missing mention metadata from REST when a message includes raw mention syntax, plus a regression test covering the same `no-mention` failure shape described in the issue.

## Evidence

- **Current main hydrates missing mention metadata:** `preflightDiscordMessage` now calls `hydrateDiscordMessageIfNeeded`, and that helper specifically re-fetches a Discord message when the payload text contains raw mention syntax (`<@...>`, role mentions, `@everyone`, `@here`) but the inbound event has no mention metadata.
