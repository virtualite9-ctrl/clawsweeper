## Summary

Keep this PR open. Current main still does not implement or expose the requested owner-only `task` and `team` agent tools, and the PR remains a plausible implementation candidate. It should not be closed as implemented or obsolete, but it needs maintainer review because it is stacked on #67331 and has unresolved review feedback around schema shape, runtime input errors, owner-only authorization, direct `/tools/invoke` behavior, and filesystem permission hardening.

## Best Possible Solution

Keep this PR open for maintainer review. Resolve or land #67331 first, then rebase or split this PR so task/team tool wiring is isolated; address the schema and input-error review comments; decide whether task/team belong in tool catalog/docs and HTTP deny/approval-class policy; and validate owner-only behavior through focused agent-tool and `/tools/invoke` tests before merge.
