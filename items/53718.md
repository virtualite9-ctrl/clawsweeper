## Summary

Keep open. Current `main` still does not expand `$VAR` or `${VAR}` placeholders in `resolveUserPath`/`resolveHomeRelativePath`, so the PR addresses behavior that is not implemented on `main` and still appears relevant.

## Evidence

- **Current path resolver only handles leading tilde:** `resolveHomeRelativePath` trims input, expands only when the path starts with `~`, and otherwise returns `path.resolve(trimmed)`. There is no env-placeholder expansion step in current `main`.
  - file: [src/infra/home-dir.ts:109](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/infra/home-dir.ts#L109)
  - command: `nl -ba src/infra/home-dir.ts | sed -n '102,143p'`
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **User-facing helper still delegates to that resolver:** `resolveUserPath` still forwards directly to `resolveHomeRelativePath`, so configured paths using `${XDG_CONFIG_HOME}` or `$XDG_CONFIG_HOME` inherit the same missing behavior.
  - file: [src/utils.ts:125](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/utils.ts#L125)

  - file: [src/infra/home-dir.test.ts:162](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/infra/home-dir.test.ts#L162)
