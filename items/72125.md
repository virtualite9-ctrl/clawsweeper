## Summary

Keep #72125 open. Current main still routes normal Feishu conversation replies through the Feishu-specific `createFeishuReplyDispatcher`, whose delivery branches call Feishu send helpers directly and do not emit `message_sent` plugin hooks or internal `message:sent` hooks. The linked PR #72138 appears to be the active fix path, but it is open/unmerged and its head commit is not present in this checkout.

## Best Possible Solution

Keep #72125 open and use PR #72138, or an equivalent maintainer-reviewed patch, as the fix path. The fix should add Feishu normal-reply sent-hook emission for text, structured/card, streaming-card, media, and failure branches, pass the routed session key/group metadata from `bot.ts`, and add Feishu-specific regression tests for plugin `message_sent` and internal `message:sent` behavior.
