## Summary

Current main already implements the timestamp-ordering fix for the task registry. PR #71871 merged a shared timestamp normalizer that is applied on task create, task update, and registry restore, covering both the original #60632 cron/startedAt skew and the lifecycle-event bypass noted in #60718 review comments. Regression tests now cover create, update, restore, and audit-visible inconsistent_timestamps behavior.

## Best Possible Solution

Close #60718 and keep the merged #71871 implementation on main. It solves the observable #60632 task timestamp invariant with a central normalizer rather than the narrower startedAt-only clamp proposed here, and it covers the lifecycle-event path raised in review.
