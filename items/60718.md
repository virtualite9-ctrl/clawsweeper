## Summary

Keep open. Current `main` still permits `startedAt < createdAt` in the task registry, the premature caller timestamps described in the PR are still present, and the PR discussion shows unresolved review gaps rather than a landed or obsolete fix.

## Evidence

- **Current create path still stores unclamped `startedAt`:** On current `main`, `createTaskRecord` still sets `const now = Date.now()`, then writes `createdAt: now`, `startedAt: params.startedAt`, and `lastEventAt: params.lastEventAt ?? params.startedAt ?? now`. That preserves the reported inversion whenever callers pass an earlier `startedAt`.
  - file: [src/tasks/task-registry.ts:1440](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/src/tasks/task-registry.ts#L1440)
  - sha: [6bc0dc8fb671](https://github.com/openclaw/openclaw/commit/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440)
- **Other registry write paths are also still unclamped:** `setTaskTimingById` still assigns `patch.startedAt = params.startedAt`, and the lifecycle listener still assigns `patch.startedAt = startedAt` from event data without comparing against `createdAt`. The invariant is therefore still breakable on current `main` even beyond initial creation.
  - file: [src/tasks/task-registry.ts:1227](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/src/tasks/task-registry.ts#L1227)
  - sha: [6bc0dc8fb671](https://github.com/openclaw/openclaw/commit/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440)
- **Premature caller timestamps remain in current main:** Current callers still inject pre-insert timestamps into `createRunningTaskRun`, including ACP spawn (`startedAt: Date.now()`), subagent registration (`startedAt: now`), and ACP session background tasks (`startedAt` from captured `turnStartedAt`). That matches the PR's root-cause description.
  - file: [src/agents/acp-spawn.ts:1315](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/src/agents/acp-spawn.ts#L1315)

  - sha: [6bc0dc8fb671](https://github.com/openclaw/openclaw/commit/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440)
- **The fix is not present in main test coverage:** Current `main` has `src/tasks/task-registry.test.ts` but no `src/tasks/cron-timestamp.test.ts`, which is the regression test added by this PR. That is further evidence the proposed fix has not already landed on `main`.
