## Summary

Keep #39661 open. Current main still exposes Discord `requireMention` only as a guild/channel boolean, while `users` and `roles` remain sender allowlists. The runtime resolver returns one mention requirement for the guild/channel context and does not accept sender identity, so two users in the same Discord channel cannot currently have different mention-trigger policies.

## Best Possible Solution

Keep the issue open for bundled Discord plugin work. A good fix would add a typed sender-specific mention policy to the Discord guild/channel config contract, validate it in zod and generated config metadata, define precedence between guild defaults, guild sender overrides, channel defaults, and channel sender overrides, pass sender identity into the Discord mention resolver, and add docs plus regression tests showing two users in the same channel with different trigger behavior.
