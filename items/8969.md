## Summary

Keep #8969 open. Current main implements the first quick win as `openclaw skills check`, but the requested status manifest-registry diagnostics and operator-visible non-tool skill prompt-rewrite warning are still missing. The latest issue discussion also explicitly kept this scope open for those two remaining gaps.

## Best Possible Solution

Keep #8969 open, or split it into two focused follow-ups. Preserve `openclaw skills check` as the shipped implementation for the first quick win. Add manifest-registry diagnostics to `openclaw status`, `openclaw status --json`, and `openclaw status --all`, and emit an operator-visible diagnostic when a non-tool skill command rewrites the request for model-directed execution. Cover both with focused status and skill-command tests plus public docs updates.
