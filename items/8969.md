## Summary

Keep #8969 open. Current main has shipped the first quick win as `openclaw skills check`, but the requested plugin registry diagnostics are still not exposed through `openclaw status`/`status --json`/`status --all`, and the non-tool skill command path still rewrites the request body without an operator-visible warning.

## Best Possible Solution

Keep #8969 open or split it into two focused follow-ups. Preserve the shipped `openclaw skills check` behavior, then add manifest-registry diagnostics to `openclaw status`, `openclaw status --json`, and `openclaw status --all`; add an operator-visible warning/log when a non-tool skill command rewrites the request for model-directed execution. Cover the status payload/text surfaces and the skill-command rewrite branch with focused tests and update the public status/skills docs.
