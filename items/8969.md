## Summary

Keep #8969 open. Current main implements the first requested quick win as `openclaw skills check`, but the requested manifest-registry diagnostics in `openclaw status` and the operator-visible warning for prompt-rewrite skill invocations are still not implemented.

## Best Possible Solution

Keep #8969 open, or split the remaining work into two focused follow-ups. Preserve `openclaw skills check` as the shipped implementation for the first quick win. Add manifest-registry diagnostics to `openclaw status`, `openclaw status --json`, and `openclaw status --all`, then emit an operator-visible diagnostic when a non-tool skill command rewrites the request for model-directed execution. Cover both with focused status and skill-command tests plus public docs updates.
