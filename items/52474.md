## Summary

Current main already preserves provider-qualified model refs in the desktop/macOS chat picker path, forwards them unchanged to `sessions.patch`, and includes regression coverage for slash-containing provider/model selections. The changelog shows this shipped in version 2026.4.10 as the session/model-selection fix that avoids bogus provider prefixes.
