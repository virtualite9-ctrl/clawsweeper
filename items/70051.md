## Summary

Keep PR #70051 open. Current main still unconditionally handles node `exec.started` / `exec.finished` / `exec.denied` by canonicalizing the session key, enqueueing a system event, and requesting an `exec-event` heartbeat wake. The current heartbeat-isolated session marker exists and is written by the heartbeat runner, but `server-node-events` does not inspect it. Adjacent heartbeat fixes on main stabilize isolated session keys and inline exec event prompt content, but they do not suppress this specific node-event feedback path.

## Best Possible Solution

Keep the PR open or land an equivalent narrow fix in `src/gateway/server-node-events.ts`: load the session entry alongside the canonical key, return early for node `exec.*` events when `entry.heartbeatIsolatedBaseSessionKey` is present, and add the proposed regression test. Also retain existing tests for normal exec notifications, `notifyOnExit`, canonicalization, and heartbeat-runner handling of deliberately queued isolated-session exec events.
