## Summary

Keep PR #70051 open. Current main still relays every `exec.started` / `exec.finished` / `exec.denied` node event through `server-node-events` after canonicalizing only the session key, then enqueues a system event and requests an `exec-event` heartbeat wake. Synthetic heartbeat-isolated sessions are already marked with `heartbeatIsolatedBaseSessionKey`, but the gateway exec-event path does not inspect that marker, so the PR's narrow guard still addresses a real remaining feedback path.

## Best Possible Solution

Keep the PR open and have maintainers review or land the narrow `server-node-events` guard: load the resolved session entry, return early for node `exec.*` events only when the entry is a marked heartbeat-isolated synthetic session, and add the focused regression test proving no system event or heartbeat wake is emitted for that case while ordinary exec notifications still pass.
