## Summary

Keep open. Current `main` still replays node `exec.*` events into system events for any resolved session and immediately requests an `exec-event` heartbeat wake, with no guard for synthetic heartbeat-isolated sessions. The repository already marks those synthetic sessions with `heartbeatIsolatedBaseSessionKey`, so the PR is addressing a real unmerged gap rather than obsolete behavior.

## Evidence

- **Current main still enqueues and wakes on all node exec events:** `handleNodeEvent` resolves the canonical session key, then always calls `enqueueSystemEvent(...)` and `requestHeartbeatNow(...)` for queued `exec.started` / `exec.finished` / `exec.denied` events. There is no check for `heartbeatIsolatedBaseSessionKey` in this path on current `main`.
