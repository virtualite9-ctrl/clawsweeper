## Summary

Keep PR #70051 open. Current main still turns node-level `exec.started` / `exec.finished` / `exec.denied` events into system events and immediate `exec-event` heartbeat wakes without checking whether the resolved session is a synthetic heartbeat-isolated session. Related isolated-heartbeat work on main covers suffix stability and inline exec prompts, but not this source event/wake suppression path.

## Best Possible Solution

Keep #70051 open. The best maintainer path is to land a narrow `server-node-events` guard, backed by the proposed regression test, that suppresses node-level exec lifecycle replay for session entries marked with `heartbeatIsolatedBaseSessionKey` while preserving normal exec notifications and existing heartbeat-runner handling of already-queued events.
