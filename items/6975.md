## Summary

Current main implements the user-visible fix for #6975: Telegram threaded replies now attach native quote excerpts automatically when original text is available, with tests, docs, and a changelog entry that explicitly marks #6975 fixed. The exact `replyContextLength` knob was not added; the maintainer fix chose automatic Telegram-safe quote excerpts instead.

## Best Possible Solution

Close this issue and keep the shipped automatic native quote excerpt implementation. If users later need configurable shorter quote snippets, track that as a new focused enhancement because current main already solves the observable lost-context behavior without adding the originally proposed `replyContextLength` field.
