## Summary

Current main no longer permits the reported contradiction where a node pairing is logged as auto-approved but is not persisted. The gateway logs auto-approval only after approveDevicePairing returns, and approveDevicePairing writes the paired device store before returning. The hello response includes the issued device token and the client stores it through the device-auth store, covering the reported device-auth.json path. The linked PR #44640 was useful context for the original persistence diagnosis, but the behavior is already present on main and in the latest checked release tag.

## Best Possible Solution

Close this issue as implemented on main. Keep the current behavior: gateway auto-approval is tied to durable device-pairing persistence, hello-ok returns a reusable device token, and clients persist that token to device-auth storage. Any new report should include current-version logs if a filesystem permission failure still blocks writes.
