## Summary

Current main still has a plausible code path for this report: the pre-compaction memory-flush run submits its own internal prompt without a transcript-safe replacement, the embedded runner will merge/remove an orphaned trailing user leaf into that next prompt, and the web UI/history filters intentionally hide only assistant-side silent/control messages, not user-role messages. That means a queued user turn can still be rewritten into the memory-flush prompt and remain visible as the wrong user text in Control UI.

## Evidence

- **Memory-flush run submits an internal prompt without a transcript-safe replacement:** `runMemoryFlushIfNeeded()` starts a separate `trigger: "memory"` embedded run with `prompt: activeMemoryFlushPlan.prompt` and `silentExpected: true`, but does not pass any `transcriptPrompt` override. That leaves the submitted prompt as the transcript-visible user message unless some other layer rewrites it.

  - command: `nl -ba src/auto-reply/reply/agent-runner-memory.ts | sed -n '730,790p'`
  - sha: [5b8bd6371c66](https://github.com/openclaw/openclaw/commit/5b8bd6371c666a5de43c82305e0315771e8e49d5)
- **Transcript rewrite is optional and skipped when no transcriptPrompt is supplied:** `rewriteSubmittedPromptTranscript()` returns immediately when `transcriptPrompt` is `undefined` or identical to the submitted prompt, so internal prompts remain verbatim in history when callers omit a safe replacement.
  - file: [src/agents/pi-embedded-runner/run/transcript-prompt-rewrite.ts:56](https://github.com/openclaw/openclaw/blob/5b8bd6371c666a5de43c82305e0315771e8e49d5/src/agents/pi-embedded-runner/run/transcript-prompt-rewrite.ts#L56)

- **Embedded runner still merges/removes a trailing user leaf into the next prompt:** Before prompt submission, `attempt.ts` checks for a trailing user leaf and calls `mergeOrphanedTrailingUserPrompt(...)`; if `removeLeaf` is true it branches/reset-leaf and replaces the active session context. This is the exact mechanism that can make the queued user turn disappear and be absorbed into the next internal prompt.
  - file: [src/agents/pi-embedded-runner/run/attempt.ts:2162](https://github.com/openclaw/openclaw/blob/5b8bd6371c666a5de43c82305e0315771e8e49d5/src/agents/pi-embedded-runner/run/attempt.ts#L2162)

- **Merge helper always removes the orphaned user leaf and prepends it to the new prompt:** `mergeOrphanedTrailingUserPrompt()` always returns `removeLeaf: true`; when text exists it prepends `[Queued user message ...]` plus the orphaned user text ahead of the new prompt. There is no trigger-specific exemption for `memory` here.
