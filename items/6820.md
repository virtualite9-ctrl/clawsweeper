## Summary

Current main still does not implement the requested `xhigh` support for `openai-codex/gpt-5.2`. The OpenAI Codex provider recognizes base `gpt-5.2` as a modern model, but its xhigh allowlist only includes `gpt-5.2-codex` and other newer/codex-suffixed ids. The previously linked fix PRs are closed and unmerged, and the current focused tests/harness still omit the requested provider/model pair.

## Best Possible Solution

Keep #6820 open until the bundled OpenAI Codex provider either adds base `gpt-5.2` to `OPENAI_CODEX_XHIGH_MODEL_IDS` with focused regression coverage for `resolveThinkingProfile` or `listThinkingLevels("openai-codex", "gpt-5.2")`, or maintainers explicitly document that base OpenAI Codex GPT-5.2 remains high-only while `gpt-5.2-codex` is xhigh-capable. The broader drift problem can continue under RFC #37804, but it does not supersede this concrete missing provider/model case.
