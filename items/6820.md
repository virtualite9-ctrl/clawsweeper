## Summary

Keep #6820 open. Current main still does not expose `xhigh` for `openai-codex/gpt-5.2`: the OpenAI Codex provider recognizes base `gpt-5.2` as a modern model, but its xhigh allowlist only includes newer base 5.4/5.5 ids and codex-suffixed 5.3/5.2/5.1 ids. The exact/prefix helper does not make `gpt-5.2` match `gpt-5.2-codex`, current tests omit the requested provider/model pair, and the related fix PRs were closed unmerged. The broader RFC #37804 tracks model capability drift but does not supersede this concrete missing allowlist case.

## Best Possible Solution

Keep #6820 open until the OpenAI Codex provider either adds base `gpt-5.2` to `OPENAI_CODEX_XHIGH_MODEL_IDS` with focused regression coverage for `listThinkingLevels("openai-codex", "gpt-5.2")` or `resolveThinkingProfile`, or maintainers explicitly document that base OpenAI Codex GPT-5.2 is high-only while `gpt-5.2-codex` is xhigh-capable. Let #37804 continue tracking the broader forward-compat/capability drift design.
