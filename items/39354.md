## Summary

Close. Current main already avoids the reported Telegram duplicate-reasoning path by only creating a reasoning preview lane when the session is explicitly `reasoning:stream`, and the shipped tests already assert that `onReasoningStream` is absent otherwise. This behavior is also called out in the `v2026.4.22` changelog, so the PR is redundant against current main.

## Close Comment

Closing from a Codex review. Current `main` already covers this behavior: `extensions/telegram/src/bot-message-dispatch.ts:326-355` only creates the Telegram reasoning preview lane when the session is explicitly `reasoning:stream`, so `onReasoningStream`/`onReasoningEnd` are not exposed for normal or `reasoningLevel: "off"` sessions. The existing regression test at `extensions/telegram/src/bot-message-dispatch.test.ts:644` already asserts that `onReasoningStream` is `undefined` unless reasoning is set to stream. This was also shipped in `v2026.4.22` and documented in `CHANGELOG.md:1129`, so there is no remaining change for this PR to land.
