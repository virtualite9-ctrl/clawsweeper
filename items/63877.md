## Summary

Keep open. Current `main` still hard-rejects non-HTTPS Firecrawl base URLs and any host other than `api.firecrawl.dev`, and the docs/tests explicitly preserve that behavior, so the reported self-hosted `http://` and private-host cases are still unfixed on `main` despite the discussion linking follow-up PRs.

## Evidence

- **Current Firecrawl client still enforces https and official host only:** `resolveEndpoint()` throws if the protocol is not `https:` and if the hostname is not in `ALLOWED_FIRECRAWL_HOSTS`, which is currently only `api.firecrawl.dev`. That directly matches the reported failures for `http://host.openshell.internal:3002/v1` and `https://host.openshell.internal:444/v1`.
