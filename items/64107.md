## Summary

Current `main` already supports disabling WhatsApp group mention gating per account via `channels.whatsapp.accounts.<id>.groups["*"]` or a specific group entry with `requireMention: false`. The schema, runtime resolution, docs, and regression tests for that behavior are present on `main` in commit `51dd4f288f67ef4df4c7a121f078f508535df1fd` (2026-04-24), but this does not appear to be in release `v2026.4.23` yet.

## Evidence

- **Schema accepts per-group requireMention in WhatsApp config, including under account-scoped groups:** `WhatsAppGroupEntrySchema` includes optional `requireMention`, `buildWhatsAppCommonShape()` includes `groups`, and `WhatsAppAccountSchema` reuses that common shape for `accounts.<id>` entries.
  - file: [src/config/zod-schema.providers-whatsapp.ts:21](https://github.com/openclaw/openclaw/blob/51dd4f288f67ef4df4c7a121f078f508535df1fd/src/config/zod-schema.providers-whatsapp.ts#L21)
  - sha: [51dd4f288f67](https://github.com/openclaw/openclaw/commit/51dd4f288f67ef4df4c7a121f078f508535df1fd)
- **Runtime resolves mention gating from the resolved account groups map:** `resolveWhatsAppInboundPolicy()` builds a WhatsApp-only config from `account.groupPolicy` plus `account.groups`, then calls `resolveChannelGroupRequireMention()` against that account-scoped config.
  - file: [extensions/whatsapp/src/inbound-policy.ts:106](https://github.com/openclaw/openclaw/blob/51dd4f288f67ef4df4c7a121f078f508535df1fd/extensions/whatsapp/src/inbound-policy.ts#L106)
  - sha: [51dd4f288f67](https://github.com/openclaw/openclaw/commit/51dd4f288f67ef4df4c7a121f078f508535df1fd)
- **Regression test covers named-account/default-account inheritance with requireMention disabled:** The test configures `channels.whatsapp.accounts.default.groups["*"].requireMention = false` and expects a plain group message for account `work` to process successfully (`result.shouldProcess === true`).
  - file: [extensions/whatsapp/src/auto-reply/web-auto-reply-monitor.test.ts:273](https://github.com/openclaw/openclaw/blob/51dd4f288f67ef4df4c7a121f078f508535df1fd/extensions/whatsapp/src/auto-reply/web-auto-reply-monitor.test.ts#L273)
  - sha: [51dd4f288f67](https://github.com/openclaw/openclaw/commit/51dd4f288f67ef4df4c7a121f078f508535df1fd)
- **Docs now show the supported account-level config shape:** The WhatsApp docs include an example under `accounts.work.groups` with a group entry using `requireMention: false`, and explain that account-level `groups` replaces the root groups map.
  - file: [docs/channels/whatsapp.md:579](https://github.com/openclaw/openclaw/blob/51dd4f288f67ef4df4c7a121f078f508535df1fd/docs/channels/whatsapp.md#L579)
  - sha: [51dd4f288f67](https://github.com/openclaw/openclaw/commit/51dd4f288f67ef4df4c7a121f078f508535df1fd)
- **Fix landed after the latest tagged release:** `git rev-parse v2026.4.23` resolves to `899e927dab9ef2d6164004d34a30410b3506b6f5`, while the implementing commit on current `main` is `51dd4f288f67ef4df4c7a121f078f508535df1fd`; the fix therefore appears newer than the latest release listed in the task context.
  - command: `git rev-parse v2026.4.23`

## Risks / Open Questions

- The supported shape on current `main` is `channels.whatsapp.accounts.<id>.groups["*"]` or a specific group entry with `requireMention: false`; a top-level `channels.whatsapp.accounts.<id>.requireMention` field is still not part of the schema evidence reviewed here.
- Because the implementing commit is newer than `v2026.4.23`, users on the latest tagged release may still see the older validation/runtime behavior until they upgrade to a build that includes `51dd4f288f67ef4df4c7a121f078f508535df1fd` or later.
