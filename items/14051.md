## Summary

Keep #14051 open. Current main still does not implement activity-based heartbeat idle pausing: the heartbeat config/type/schema expose interval, active-hours, routing, prompt, timeout, context, and reasoning controls, but no `idleTimeoutMinutes` or equivalent recent-activity window. The runner and scheduler remain interval/active-hours/queue/preflight gated, and existing `lastInteractionAt` tracking is documented and tested for session reset freshness rather than heartbeat scheduling. The provided issue discussion already kept this open, and the linked #40611 and #46844 are adjacent heartbeat/talk-mode idle issues, not canonical replacements for this heartbeat feature request.

## Best Possible Solution

Keep #14051 open. A proper fix should add a product-approved heartbeat idle-timeout config key to exported types, strict runtime schema, schema help/generated docs, config-honor inventory, user docs, and focused tests. Runtime should track or reuse last allowed inbound user activity at the correct agent/session/channel boundary and gate heartbeat runs or scheduling on that activity window while preserving existing interval, active-hours, retry, targeted wake, task wake, light-context, and isolated-session behavior.
