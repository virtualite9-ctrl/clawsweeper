## Summary

Keep #14051 open. Current main still does not implement activity-based heartbeat idle pausing: heartbeat config, schema, docs, runner, scheduler, and tests still describe interval/active-hours/busy-lane behavior, not a recent allowed-inbound-activity gate or idle-timeout window. Related #59567 was closed as a duplicate in favor of this canonical tracker, so it does not supersede #14051.

## Best Possible Solution

Keep #14051 open as the canonical product/API tracker. A proper fix should add a maintainer-approved heartbeat idle-timeout or idle-only config key to exported types, strict runtime schema, generated config docs/help and config-honor inventory, plus docs and focused tests. Runtime should gate heartbeat scheduling or execution on recent allowed inbound user/channel activity at the correct agent/session/channel boundary while preserving existing interval, active-hours, targeted wake, task wake, requests-in-flight retry, light-context, and isolated-session behavior.
