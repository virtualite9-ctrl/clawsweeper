## Summary

Keep open. Current main already exposes cron job identifiers and running markers through the model, Gateway protocol, runtime state, tests, and CLI, but the Control UI still does not visibly render each job's `id` or convert `state.runningAtMs` into an active running state on cron job cards. The agent cron panel shares the same presenter gap.

## Best Possible Solution

Keep this open and scope it as a Control UI cron visibility enhancement: render each cron job's `id`, derive an active running state from `job.state?.runningAtMs` before falling back to last-run status, update the shared agent cron presenter if that panel should match, add English i18n labels, and cover running, idle, disabled, and identifier display in focused UI tests.
