## Summary

Keep open. Current main has the underlying cron identifier and running-state data in the model, Gateway protocol, runtime state, and CLI, but the Control UI cron job cards still do not visibly render each job ID or treat `state.runningAtMs` as a running status. The agent cron panel has the same presenter gap.

## Best Possible Solution

Scope this as a Control UI cron visibility enhancement: render each cron job's stable `id`, derive an active running state from `job.state?.runningAtMs` before falling back to last-run status, align the agent cron panel if it should mirror the main cron view, add English i18n labels, and cover running, idle, disabled, and identifier display in focused UI tests.
