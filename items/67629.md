## Summary

Keep open. Current main already has stable cron job IDs and running-state data in the cron model, Gateway protocol, runtime state, tests, and CLI output, but the Control UI cron job cards still do not visibly render each job ID or derive status from `state.runningAtMs`. The prior ClawSweeper comment identified the same remaining UI gap, and the current checkout still matches that conclusion.

## Best Possible Solution

Implement this as a focused Control UI cron visibility enhancement: render each job's stable ID on the job card, derive an active running status from `job.state?.runningAtMs` before falling back to last-run status, add English i18n labels, and cover running, idle, disabled, and identifier display in focused UI tests. If there is a separate agent cron panel that should mirror the main cron view, align that presenter too.
