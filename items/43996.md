## Summary

Keep open. Current main still always adds Docker `--security-opt no-new-privileges` for sandbox containers, exposes no `sandbox.docker.noNewPrivileges` opt-out, and the supplied GitHub context shows open PR #70229 explicitly targets this issue with closing syntax. That PR should be reviewed, fixed, landed, or closed before this issue is closed.

## Best Possible Solution

Keep this issue open while PR #70229 remains open. Review that PR as the implementation candidate: it should add a default-on `sandbox.docker.noNewPrivileges` or equivalent explicit override, update schema/types/docs/security audit output, and add regression coverage for both default hardening and the disabled override. After a vetted fix lands on main and ships, this issue can close as implemented.

## Risks / Open Questions

- A configurable opt-out weakens sandbox hardening when disabled, so the best implementation should keep the secure default, make the override explicit, and surface it in security/dangerous-config reporting.
