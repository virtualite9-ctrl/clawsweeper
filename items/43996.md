## Summary

Keep #43996 open. Current main still unconditionally adds Docker `--security-opt no-new-privileges` for sandbox containers, has no documented/configured opt-out, and has no fallback retry path for the reported EPERM startup failure. The linked fix PR #70229 is still open/unmerged in the supplied GitHub context, so the issue is not implemented or superseded on main yet.

## Best Possible Solution

Keep the issue open until #70229 or an equivalent maintainer-approved fix lands. The best path is a secure-default `sandbox.docker.noNewPrivileges` opt-out with schema/types/docs coverage, dangerous-config/status reporting for `false`, and tests proving both the default hardening and the explicit break-glass behavior.
