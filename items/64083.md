## Summary

Current `main` now treats fallback-persisted session overrides as transient: it clears direct `modelOverrideSource: "auto"` overrides at the start of the next turn and retries the configured primary model, with regression coverage for the reported stuck-on-fallback behavior. The fix landed after the latest tagged release, so it is implemented on `main` but not yet in a released tag.

## Evidence

- **Primary retry/self-heal logic on current main:** `createModelSelectionState` now explicitly clears direct auto-failover session overrides and resets the run back to the configured primary: comment says the session should "self-heal when the primary recovers," then it clears the stored override and sets `provider = defaultProvider` / `model = defaultModel`.
