## Summary

Current main still only partially addresses the report. The docs explain that `openclaw mcp list/show/set/unset` manage OpenClaw-owned `mcp.servers`, but the CLI help and human `mcp list` output remain generic and still do not warn that mcporter `config/mcporter.json` servers are excluded. The focused related PR was closed unmerged, so the remaining CLI/docs/test work is still actionable.

## Best Possible Solution

Keep this issue open and land a focused CLI/docs/test change like the closed related PR attempted: label non-JSON `openclaw mcp list` output as OpenClaw-managed, add a short note that mcporter `config/mcporter.json` servers are not included, tighten `mcp list` help text, and add a regression test in `src/cli/mcp-cli.test.ts` while leaving JSON output unchanged.
