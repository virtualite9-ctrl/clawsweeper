## Summary

Current main still only partially addresses the report. The docs explain that `openclaw mcp list/show/set/unset` manage OpenClaw-owned `mcp.servers`, but the CLI help and human output remain generic and do not warn that mcporter `config/mcporter.json` servers are excluded. The focused PR #69726 targeted this issue, but the provided GitHub context shows it is closed and unmerged, so the remaining work is still actionable.

## Best Possible Solution

Keep this issue open and land a focused CLI/docs/test change like #69726 attempted: label non-JSON `openclaw mcp list` output as OpenClaw-managed, add a short note that mcporter `config/mcporter.json` servers are not included, tighten `mcp list` help text, and add a regression test in `src/cli/mcp-cli.test.ts` while leaving JSON output unchanged.
