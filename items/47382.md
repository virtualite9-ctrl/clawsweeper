## Summary

Keep #47382 open. Current main has adjacent liveness features, including typing indicators, preview/tool-progress streaming, status reactions, transport stall watchdogs, and LLM idle aborts, but I did not find the requested gateway-level, per-channel silence watchdog that sends configurable operator-visible beacon messages based on last delivered visible output. This belongs in core channel/gateway behavior, not ClawHub, because it needs active-run state plus cross-channel delivery/visibility tracking.

## Best Possible Solution

Keep #47382 open as the focused core tracking item for a channel/gateway silence watchdog. A good implementation would add a typed per-channel config surface, track active work plus last operator-visible delivery time across final replies, block/preview updates, tool-visible output, and beacon sends, then send bounded configurable beacon messages until visible output resumes or the run ends. It should integrate with existing typing/status reactions and channel delivery visibility hooks, with tests for at least Telegram, Discord, and Slack behavior.
