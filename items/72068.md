## Summary

PR #72068 should close as a ClawHub/plugin-boundary item, not because the idea is invalid. It proposes a new optional Fleet Orchestrator/Mission Control integration with its own bundled plugin, CLI verbs, bearer-token HTTP mutate API, task store, and rollout flow. Current main already exposes the plugin seams needed for that work outside core, while the project vision keeps core lean and explicitly sets a high bar for optional bundled plugins and heavy orchestration layers.

## Best Possible Solution

Close this core PR and move the Fleet Orchestrator work to an externally maintained ClawHub/plugin package using the existing registerHttpRoute, registerCli, and registerService seams. If implementation work reveals a real missing OpenClaw plugin API, open a narrow API-design issue or PR for that seam; if maintainers later want first-party bundled promotion, bring back a smaller reviewed PR with the P1 security/behavior findings fixed.
