## Summary

Current main implements the Control UI drag-and-drop/file-picker attachment workflow for images and non-video files. Dropped files are turned into chat attachments, sent through `chat.send`, and non-image files are persisted as managed media refs such as `media://inbound/<id>`, which satisfies the requested usable file reference workflow even though it is not literal local-path insertion.

## Best Possible Solution

Keep the current managed attachment/media-ref implementation on main. If maintainers want literal local-path insertion for a native desktop-only surface, track that separately as a product decision rather than keeping this Control UI regression open.
