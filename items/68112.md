## Summary

Keep this PR open. Current main still awaits startup catch-up before the first cron timer arm, and the catch-up path can still propagate persistence failures before `armTimer(state)` is reached. The PR remains a focused implementation candidate, though it likely needs maintainer review and rebase against newer cron changes on main.

## Best Possible Solution

Keep this PR open for maintainer review. Rebase it onto current main and either land it after targeted validation of startup catch-up failure resilience, one-shot interruption semantics, timeout-disabled jobs, and persistence-failure paths, or replace it with an equivalent main-branch fix that guarantees `start()` arms the cron timer even when startup catch-up or follow-up persistence fails.
