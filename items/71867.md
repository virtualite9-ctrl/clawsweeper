## Summary

Keep #71867 open. Current main still reports doctor model-auth health from one implicit auth store and labels the note generically as `Model auth`; it does not enumerate agent stores, include an agent id, or detect divergent per-agent auth state. The related #71923 PR is the active focused fix, but it is not on current main.

## Best Possible Solution

Keep #71867 open until #71923 or an equivalent fix lands. The best implementation is to make doctor auth health label the checked agent store, enumerate relevant configured agent auth stores when multiple stores exist, call out divergent auth state, and add focused regression coverage for multi-agent doctor output.
