## Summary

Keep #71867 open. Current main still reports doctor auth health from one implicit auth store with generic `Model auth`, `Auth profile cooldowns`, and `OAuth refresh errors` note titles. The related PR #71923 is an open candidate fix, not behavior that has shipped on main.

## Best Possible Solution

Keep the issue open and implement doctor-side agent attribution. A conservative fix would pass an explicit agent scope into doctor auth health, label `Model auth`, cooldown, and OAuth refresh notes with the agent id and preferably the auth store path, add an optional `--agent <id>` or deterministic configured-agent scan if desired, and cover multi-agent divergence/read-only behavior in focused tests and doctor docs.

## Risks / Open Questions

- A fix that enumerates agent stores should avoid non-read-only explicit subagent auth loads, because the current store loader can persist inherited main auth into a subagent path.
- Maintainers need to choose the product shape: label only the implicit/default store, add `doctor --agent <id>`, or scan configured stores and warn on divergence.
