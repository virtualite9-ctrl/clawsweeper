## Summary

Keep open. Current main has not implemented the PR’s central prompt-metadata behavior: `chat.send` still has no `authoredContent` parameter, the reply pipeline has no prepared-user-prompt callback, and chat transcript updates/history have no authored/runtime prompt metadata path. The provided PR discussion also flags that the proposed implementation still needs true on-disk transcript persistence for this metadata, so it is not cleanly obsolete or safe to close as implemented.

## Best Possible Solution

Keep the PR open, but ask for a focused update: rebase onto current main, drop or split the unrelated Poros/commands.list changes that are already covered elsewhere, add the authoredContent schema/API and prepared model-input capture, and complete persistence by rewriting or otherwise recording __openclaw authoredContent/modelInput into the on-disk transcript so chat.history works after reconnect. Add a regression test that sends through chat.send and then reloads history from disk, not only a test seeded with prewritten __openclaw metadata.
