## Summary

Current main already hard-caps overflow-driven compaction retries in embedded agent runs, then halts with a user-facing context-overflow error instead of looping indefinitely.

## Evidence

- **Runtime hard cap:** `MAX_OVERFLOW_COMPACTION_ATTEMPTS = 3` is defined in the embedded runner, limiting overflow compaction retries per run.
  - file: [src/agents/pi-embedded-runner/run.ts:498](https://github.com/openclaw/openclaw/blob/cb2c36b04975e34ab7014a11575d6589aa96ae32/src/agents/pi-embedded-runner/run.ts#L498)
  - sha: [cb2c36b04975](https://github.com/openclaw/openclaw/commit/cb2c36b04975e34ab7014a11575d6589aa96ae32)
- **Retry then stop behavior:** On repeated overflow, the runner increments `overflowCompactionAttempts`, retries only while below the cap, and then returns a blocking `context_overflow` error with `/reset` guidance.
  - file: [src/agents/pi-embedded-runner/run.ts:1043](https://github.com/openclaw/openclaw/blob/cb2c36b04975e34ab7014a11575d6589aa96ae32/src/agents/pi-embedded-runner/run.ts#L1043)
  - sha: [cb2c36b04975](https://github.com/openclaw/openclaw/commit/cb2c36b04975e34ab7014a11575d6589aa96ae32)
- **Regression coverage:** Test explicitly verifies compaction retries stop after 3 attempts and the 4th overflow returns an error.
  - file: [src/agents/pi-embedded-runner/run.overflow-compaction.loop.test.ts:326](https://github.com/openclaw/openclaw/blob/cb2c36b04975e34ab7014a11575d6589aa96ae32/src/agents/pi-embedded-runner/run.overflow-compaction.loop.test.ts#L326)
  - sha: [cb2c36b04975](https://github.com/openclaw/openclaw/commit/cb2c36b04975e34ab7014a11575d6589aa96ae32)
- **Shipped in latest release:** The `v2026.4.22` release commit contains the same overflow-compaction cap and stop-on-limit logic.
  - file: [src/agents/pi-embedded-runner/run.ts:468](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/src/agents/pi-embedded-runner/run.ts#L468)
  - command: `git show 00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5:src/agents/pi-embedded-runner/run.ts | rg -n "MAX_OVERFLOW_COMPACTION_ATTEMPTS|Context overflow: prompt too large|context overflow detected \(attempt"`

## Close Comment

Closing from a Codex review: current main already ships the safeguard this request was asking for. Embedded agent runs now hard-cap overflow compaction retries at 3 in `src/agents/pi-embedded-runner/run.ts:498`, retry only up to that limit in `src/agents/pi-embedded-runner/run.ts:1043`, and then stop with a user-facing context-overflow error plus reset guidance in `src/agents/pi-embedded-runner/run.ts:1231`. The regression test `src/agents/pi-embedded-runner/run.overflow-compaction.loop.test.ts:326` covers the exact "retry 3 times, then give up" behavior. This is already present in release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`). The specific `compaction.maxPerSession` config key was not added, but the token-burn loop prevention requested here is already implemented on main.
