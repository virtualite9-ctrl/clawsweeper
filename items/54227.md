## Summary

Keep this issue open. Current main still implements Discord component consumption per component id, while the docs and message-tool schema present non-reusable Discord components as single-use by default. The related #54233 is closed and its review explicitly left the per-panel consumption bug to this issue, so it does not supersede the remaining work.

## Best Possible Solution

Keep this issue open as the focused Discord plugin bug. Implement panel-wide consumption for non-reusable component messages, likely by assigning a shared consumption group to entries created from the same component message/panel and invalidating sibling non-reusable entries after the first authorized click. Add regression coverage for two sibling buttons where the second click returns the expired response and does not dispatch, while preserving reusable:true repeated use, unauthorized-click behavior that leaves entries active, and modal trigger/form lifecycle.
