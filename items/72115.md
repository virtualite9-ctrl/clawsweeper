## Summary

Keep open. Current main does not contain the PR's Skill Workshop hardening: writes still rely on string-prefix path containment, apply does a single prepare immediately before write with no stale-workspace comparison, Skill Workshop paths do not enforce core skills prompt-budget limits, and the Plugin SDK does not expose the proposed security/skills runtime seams. The PR is recent, coherent, and its discussion shows earlier review findings were addressed in later PR commits, so it remains useful maintainer-review work rather than cleanup-close material.

## Best Possible Solution

Keep #72115 open for maintainer review. The best path is to review or land the focused hardening if the SDK seam additions and qa-lab routing changes are acceptable, or ask the author to split the qa-lab parity fix from the Skill Workshop security/prompt-budget work if maintainers want narrower review boundaries.
