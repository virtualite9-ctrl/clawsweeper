## Summary

Keep open. Current main does not contain the PR's central Skill Workshop hardening: path checks are still string-prefix based, apply writes after a single prepare without a stale-workspace recheck, and the plugin does not enforce the workspace skills prompt-budget path before queuing/applying proposals. The PR is recent, active, and addresses real bundled plugin/SDK behavior rather than ClawHub-only work or a duplicate of the closed core `skills_manage` experiment.

## Best Possible Solution

Keep #72115 open. The best path is maintainer review of the PR as a focused bundled Skill Workshop hardening change: preserve the plugin-local write path, add the narrow SDK exports needed for shared path and prompt-budget policy, keep TOCTOU/quarantine/prompt-budget regression tests, and split the qa-lab mock routing fix only if the dirty-candidate scope blocks landing.
