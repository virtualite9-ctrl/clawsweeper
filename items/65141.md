Keep this issue open. Current main still has a viable duplicate iMessage watcher path for `accounts.default` plus a named account, and live GitHub data shows same-author PR #65194 is open and unmerged with #65383 as an open alternate fix path.

## Best Possible Solution

Keep this issue open and review/land or close the linked implementation PRs. The best code path is an iMessage-owned startup/account-enumeration fix that treats `channels.imessage.accounts.default` as fallback-only when a named account uses the same effective watcher backend, while preserving distinct `default` runtimes when `cliPath` or `dbPath` genuinely differ, plus regression coverage for the reported config shape.
