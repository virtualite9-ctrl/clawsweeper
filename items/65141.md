## Summary

Keep this issue open. Current main still enumerates both `accounts.default` and a named iMessage account and starts one gateway task per returned account id, and the provided GitHub context shows same-author PR #65194 is still open, unmerged, and explicitly intended to close this issue. PR #65383 is also an open alternate fix path.

## Best Possible Solution

Keep this issue open and review/land or close PR #65194, reconciling it with the alternate open PR #65383. The likely fix belongs in the iMessage account enumeration path: treat `channels.imessage.accounts.default` as fallback-only when named iMessage accounts exist unless it declares a distinct runtime/backend such as a different `cliPath` or `dbPath`, and add regression coverage for the reported default-plus-named config.
