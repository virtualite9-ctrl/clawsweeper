## Summary

Current `main` already has the WhatsApp setup-entry and read-only channel-resolution paths that keep WhatsApp discoverable after startup/restart, and the CLI login path now resolves missing channel plugins through the channel catalog/setup registry instead of immediately failing as unsupported.

## Close Comment

Closing from a Codex review because current `main` already covers this behavior. WhatsApp now ships a dedicated bundled `setupEntry` (`extensions/whatsapp/package.json:24`, `extensions/whatsapp/setup-entry.ts:1`), and the read-only channel discovery path explicitly backfills configured bundled channels from that setup surface (`src/channels/plugins/read-only.ts:372`). That is the path used by `openclaw channels list` (`src/commands/channels/list.ts:105`), so a configured WhatsApp channel no longer depends on the full runtime plugin already being loaded just to appear in CLI/status surfaces. The login path also resolves installable/setup-backed channels before it would throw `Unsupported channel` (`src/cli/channel-auth.ts:89`), with WhatsApp regression coverage in `src/cli/channel-auth.test.ts:314`. I could not prove the exact first shipped fix release from this shallow checkout, so I am only closing as implemented on current `main`.
