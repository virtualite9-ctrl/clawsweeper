## Summary

Keep open. Current main still only suppresses exact NO_REPLY or the JSON action envelope, does not collapse the reported NO_REPLY.NO_REPLY form, and the reasoning tag stripper is XML-tag based. The linked PR #69472 is the active fix path, but it is open and not shipped on main.

## Best Possible Solution

Keep #69470 open until PR #69472 or an equivalent narrowly scoped patch lands. The fix should stay in src/auto-reply/tokens.ts, add focused coverage in src/auto-reply/tokens.test.ts, suppress reasoning-prefaced silent replies and doubled trailing tokens, and preserve #19537's protection for substantive replies ending with NO_REPLY.
