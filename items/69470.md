## Summary

Issue #69470 should close as superseded by the directly linked open PR #69472, which is the active code/test vehicle for the same remaining bug. Current main does not yet implement the requested tolerant reasoning-prefaced NO_REPLY suppression, so this is not an implemented-on-main close.

## Best Possible Solution

Use PR #69472 as the canonical active tracker and review or land that shared `src/auto-reply/tokens.ts` fix, preserving #19537 exact-token semantics while adding narrowly tested reasoning-prefaced/doubled-token suppression.

## Evidence

- **Current silent payload gate is still exact-token-or-JSON only:** `isSilentReplyPayloadText` returns only `isSilentReplyText(text, token) || isSilentReplyEnvelopeText(text, token)`, and `isSilentReplyText` intentionally matches only the exact silent token with surrounding whitespace to preserve #19537 behavior.
  - file: [src/auto-reply/tokens.ts:32](https://github.com/openclaw/openclaw/blob/4a3030df9efa72c44bab567a4390ac0c4876a73f/src/auto-reply/tokens.ts#L32)

  - file: [src/auto-reply/tokens.ts:21](https://github.com/openclaw/openclaw/blob/4a3030df9efa72c44bab567a4390ac0c4876a73f/src/auto-reply/tokens.ts#L21)

- **Shared reply directive gate uses the same payload classifier:** `parseReplyDirectives` computes `isSilent` through `isSilentReplyPayloadText`, so the proposed fix belongs in the shared auto-reply token layer rather than a Discord-only path.
