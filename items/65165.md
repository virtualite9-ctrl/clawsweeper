## Summary

Keep open. Current main still gates image attachment retention through an exact provider/model catalog lookup and has no generic fallback to configured `models.providers.*.models[].input` metadata. The provided GitHub context also shows open PR #65178 uses closing syntax for this issue and remains unmerged, so the issue should stay open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open. The best maintainer path is to review and either land #65178 or replace it with an equivalent Gateway fix that uses normalized provider/model matching and falls back to configured provider model metadata when the Gateway catalog misses. Add focused regression coverage for case-insensitive slash-containing model ids, provider aliases, catalog misses, and explicit `input: ["text", "image"]` custom provider rows.
