## Summary

Keep open. Current main still uses exact provider/model matching in the Gateway image-support resolver and has no generic fallback to configured `models.providers.*.models[].input` metadata. The provided context also shows PR #65178 is open, unmerged, and explicitly attempts to fix this issue, so this issue should remain open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open. The best maintainer path is to review and either land #65178 or replace it with an equivalent Gateway fix that normalizes provider/model matching and falls back to configured provider model metadata when the Gateway catalog misses. Add focused regression coverage for case-insensitive slash-containing model ids, provider aliases, catalog misses, and explicit `input: ["text", "image"]` custom provider rows.
