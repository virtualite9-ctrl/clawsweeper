## Summary

Keep this PR open, but do not merge it as written. Current main already has the safer Google Chat auth path and channel runtime snapshots for `channels.status`, while `openclaw health` still lacks gateway runtime state. The useful remaining work is a narrow health/runtime snapshot change; the PR's manual Google service-account token exchange is stale and security-sensitive against current main.

## Best Possible Solution

Keep the PR open as a useful but stale implementation candidate. Replace it with a narrow gateway/health patch that passes the current `ChannelRuntimeSnapshot` into `getHealthSnapshot`, uses the existing read-only channel plugin path, and preserves `probe`, `lastProbeAt`, `configured`, `enabled`, and plugin summary behavior for all channels. Leave Google Chat auth on the current `google-auth.runtime` path unless a separately evidenced auth bug is fixed with the same SSRF and credential validation guarantees.

- The PR is stale against current main's read-only health implementation and could reintroduce heavier plugin loading or behavior drift in the cold health path.
