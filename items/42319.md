## Summary

Keep this PR open, but it should not merge as written. Current main still lacks the useful health/runtime snapshot behavior the PR is trying to add, while the PR's Google Chat auth rewrite is stale and security-sensitive against the current SSRF-guarded google-auth path.

## Best Possible Solution

Keep this PR open as a useful but stale implementation candidate. The best path is a narrow gateway/health patch that passes the current `ChannelRuntimeSnapshot` into `getHealthSnapshot`, uses the existing read-only channel plugin path, preserves `probe`, `lastProbeAt`, `configured`, `enabled`, and plugin summary behavior for all channels, and leaves Google Chat auth on the current SSRF-guarded google-auth runtime unless a separately evidenced auth bug is fixed with equivalent endpoint and credential safeguards.

## Risks / Open Questions

- The PR is stale against current main's read-only health/plugin loading path; a direct merge could reintroduce behavior drift outside the narrow health/runtime fix.
