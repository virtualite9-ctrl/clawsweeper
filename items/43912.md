## Summary

Keep open. Current main still skips bundled pre-install resolution for scoped specs: `resolveBundledInstallPlanBeforeNpm` returns `null` for any non-bare spec, and the colocated test explicitly asserts that `@openclaw/voice-call` is skipped. The install command then proceeds to ClawHub/npm and persists a successful npm install, so the duplicate global install path the PR targets is still present. The PR is focused on the implicated helper and test, and the provided bot review/comment context supports the same narrow root cause.

## Best Possible Solution

Keep this PR open for maintainer review. The likely product/code path is to land or adapt the PR’s narrow helper change: trim the raw spec, keep plugin-id lookup for bare specs, use exact `npmSpec` lookup for non-bare scoped specs, and retain tests for matching and non-matching scoped specs. Before merge, update the CLI plugin docs if maintainers want the scoped bundled-package behavior documented explicitly.
