## Summary

Keep this PR open. Current main still skips bundled pre-install resolution for scoped npm specs, while the bundled-source layer already has exact npmSpec lookup support and real bundled plugin metadata such as @openclaw/feishu can satisfy that path. The PR remains a focused implementation candidate, with a maintainer product/docs decision needed because current CLI docs describe scoped specs as the way to force npm for bare-name collisions.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land or adapt the narrow helper change if maintainers want exact bundled npmSpec matches to win before ClawHub/npm: trim the raw spec, keep pluginId lookup for bare specs, use exact npmSpec lookup for scoped/non-bare specs, let non-matches continue to ClawHub/npm, update the helper tests, and adjust docs plus PR wording to describe the exact official scoped-spec exception.
