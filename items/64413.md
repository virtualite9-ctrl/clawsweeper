## Summary

Current main still does not implement normalized delivery failure reasons across cron and messaging runs. The earlier ClawSweeper keep-open review remains consistent with the current main SHA: cron, gateway protocol, Control UI, message CLI, and task delivery still expose coarse status enums and raw error strings rather than shared reason codes such as not_configured, channel_unavailable, attempted_failed, or rate_limited.

## Best Possible Solution

Keep this issue open. The best path is an additive normalized delivery outcome model shared by cron run logs, message action results, background task delivery, Gateway protocol schemas, Control UI filters/cards, CLI JSON/text output, and docs. Preserve existing fields for compatibility, add stable reason codes such as not_configured, channel_unavailable, skipped, attempted_failed, and rate_limited, and cover cron target-resolution failures, outbound adapter configuration failures, provider/rate-limit errors, best-effort skips, task terminal delivery fallback, and UI/API filtering in tests.
