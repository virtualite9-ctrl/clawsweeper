## Summary

Current main still does not normalize delivery failure reasons across cron and messaging runs. The existing ClawSweeper keep-open comment remains consistent with HEAD 52249927ac2dc579a2f8e72aa99a30e0ab4f9a11: cron, Gateway protocol, Control UI, task delivery, and message CLI/tool results still expose coarse status values and raw error strings rather than shared reason codes such as not_configured, channel_unavailable, attempted_failed, or rate_limited.

## Best Possible Solution

Keep this issue open. The best product/code path is an additive normalized delivery outcome model shared by cron run logs, message action results, background task delivery, Gateway protocol schemas, Control UI filters/cards, CLI JSON/text output, and docs. Preserve existing compatibility fields, add stable reason codes such as not_configured, channel_unavailable, skipped, attempted_failed, and rate_limited, and cover cron target-resolution failures, outbound adapter configuration failures, provider/rate-limit errors, best-effort skips, task terminal delivery fallback, and UI/API filtering in tests.
