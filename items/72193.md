## Summary

Close #72193 as a duplicate of the already-open canonical tracker #67031. Current main still has the reported 5 MB sanitizer default, 1200px dimension default, and 6 MB shared image byte cap, and #67031 is explicitly tracking the broader hardcoded image byte/dimension configurability gap across sanitize layers.

## Best Possible Solution

Keep #67031 open as the canonical implementation/design issue. The best fix is a conservative config contract for shared image sanitizer byte and dimension limits, preserving existing agents.defaults.imageMaxDimensionPx compatibility while threading the configured limits through prompt image loading, transcript/tool image sanitization, docs, schema/help, and tests.
