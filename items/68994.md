## Summary

Keep #68994 open. Current main still preserves canonical cross-root session transcript paths unchanged, the existing test suite asserts that stale-root behavior, and Docker setup does not normalize persisted config/session JSON paths before onboarding. The PR is a plausible focused fix, though its submitted shell fallback needs review before merge.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to update the patch so session path resolution rebases stale canonical `agents/<id>/sessions/<file>` paths to the current state root, add/adjust regression tests for the Docker `/home/node` to current-home scenario, and fix the Docker setup normalization fallback before merge. Do not close it as duplicate of the broader Docker EACCES volume-permission items unless a new canonical PR explicitly covers stale persisted session paths too.
