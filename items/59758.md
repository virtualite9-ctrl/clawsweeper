## Summary

Keep open. Current main has part of the requested helper ordering, but the normal Mattermost inbound path still forces direct-message reply mode to `off`, so an inbound DM thread with `post.root_id` does not preserve that thread root for outbound replies.

## Best Possible Solution

Keep the issue open and apply a narrow Mattermost plugin fix: direct top-level DMs should continue to produce no `replyToId`, but direct messages that arrive with `post.root_id` should preserve that root id for session context and outbound `root_id` regardless of the direct-message auto-threading default. Add regression coverage for `kind: "direct"` with `threadRootId` plus the existing top-level DM case.
