## Summary

Keep open. Current `main` still has a plausible DM-threading regression path for Mattermost: DMs force `replyToMode` off, but outbound replies can still inherit an implicit `replyToId` from the current post and send it as Mattermost `root_id`, which matches the report rather than disproving it.

## Evidence

- **DMs are explicitly forced to `replyToMode: "off"`:** `resolveMattermostReplyToMode` hard-codes direct chats to `"off"`, and the regression test expects that behavior.
  - file: [extensions/mattermost/src/mattermost/accounts.ts:133](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/extensions/mattermost/src/mattermost/accounts.ts#L133)
  - command: `nl -ba extensions/mattermost/src/mattermost/accounts.ts | sed -n '129,145p'; nl -ba extensions/mattermost/src/mattermost/accounts.test.ts | sed -n '96,108p'`
  - sha: [ec3dbd22a4de](https://github.com/openclaw/openclaw/commit/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46)
- **Existing thread roots are only preserved when `replyToMode !== "off"`:** `resolveMattermostEffectiveReplyToId` returns `threadRootId` only if `replyToMode` is not `off`; otherwise direct chats fall through to `undefined`. That means a DM thread root is not preserved once direct-chat mode forces `off`.
  - file: [extensions/mattermost/src/mattermost/monitor.ts:342](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/extensions/mattermost/src/mattermost/monitor.ts#L342)

If `effectiveReplyToId` is `undefined`, the code falls back to `payload.replyToId`, and `reply-delivery.ts` forwards that through to the Mattermost sender.
  - file: [extensions/mattermost/src/mattermost/monitor.ts:1705](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/extensions/mattermost/src/mattermost/monitor.ts#L1705)

- **Generic reply plumbing implicitly sets `replyToId` to the current message id:** Core reply plumbing adds `replyToId = currentMessageId` even without an explicit tag, and the test suite asserts that behavior. That gives top-level DMs a reply target unless the channel strips it later.
