## Summary

Keep open. Current main does not implement `host="cloud"`, a `CloudSandboxProvider`, or cloud browser target routing, and the PR is proposing a new core exec/plugin seam rather than only a third-party plugin. It is also not safe to close as implemented, duplicate, or ClawHub-only because the existing plugin service registry can host background services but current exec/browser routing has no cloud provider contract to call. The PR is not merge-ready as-is: the provided review comments identify unresolved security and lifecycle issues that need author/maintainer resolution.

## Best Possible Solution

Keep this PR open as a core exec/plugin API proposal. The best path is maintainer review plus a reworked implementation that defines a documented cloud sandbox provider contract, routes exec/browser through it with parity to gateway/node approval and environment rules, handles process polling/kill/output limits safely, and updates tests/docs. If maintainers decide cloud sandbox routing should not be in core, close it explicitly as a product/API scope decision rather than as implemented, duplicate, stale, or ClawHub-only.
