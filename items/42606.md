## Summary

Keep open. The requested hardening is still valid on current main, but the PR branch patches stale `src/browser` files while the active noVNC bootstrap route now lives in the browser plugin under `extensions/browser`. Current main still lacks the CSP nonce, `nosniff`, and frame-deny headers for that route.

## Best Possible Solution

Port the narrow hardening to `extensions/browser/src/browser/bridge-server.ts` and `extensions/browser/src/browser/bridge-server.auth.test.ts`: generate a per-response script nonce, return CSP plus HTML together so they cannot drift, set `Content-Security-Policy`, `X-Content-Type-Options: nosniff`, and `X-Frame-Options: DENY`, and preserve the existing observer-token redirect behavior.

## Risks / Open Questions

- Merging the branch as written would not harden the live browser plugin route because current main no longer has the patched `src/browser` files.
- Until ported, the token-gated noVNC bootstrap still serves an inline redirect script without a document CSP, nonce, `X-Content-Type-Options: nosniff`, or `X-Frame-Options: DENY`.
- The port must preserve the one-time observer token flow, avoid a `Location` header, and keep the noVNC password in the URL fragment only.
