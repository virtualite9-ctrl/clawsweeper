## Summary

Keep this PR open. The security hardening is still valid on current main, but the PR patches stale `src/browser` paths while the live noVNC bootstrap route now lives under `extensions/browser` and still lacks the requested CSP, nonce, `nosniff`, and frame-deny headers.

## Best Possible Solution

Port the narrow hardening to `extensions/browser/src/browser/bridge-server.ts` and `extensions/browser/src/browser/bridge-server.auth.test.ts`: generate a per-response script nonce, return CSP plus HTML together so they cannot drift, set `Content-Security-Policy`, `X-Content-Type-Options: nosniff`, and `X-Frame-Options: DENY`, and preserve the existing one-time token flow with the password only in the fragment. Do not merge the current branch until it is rebased or rewritten onto the live browser plugin files.

## Risks / Open Questions

- Merging the branch as written would not harden the live browser plugin route because it patches stale `src/browser` paths.
- Until ported, the token-gated noVNC bootstrap page still serves an inline redirect script without a document CSP, nonce, `X-Content-Type-Options: nosniff`, or `X-Frame-Options: DENY`.
- The port needs to preserve the existing one-time observer token behavior, absence of a `Location` header, and password placement in the URL fragment only.
