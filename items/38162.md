## Summary

Keep PR #38162 open. Current main still lacks the requested `sessionSaveRedirectPath` behavior, the supporting `postHookActions` coordination mechanism, and the related session-memory policy fields. The PR remains concrete, security-sensitive core hook work rather than obsolete cleanup or a ClawHub-only item.

## Best Possible Solution

Keep #38162 open. Resolve or supersede #38161 first, then review #38162 as the redirect-specific follow-up with maintainer-approved workspace-containment semantics, symlink/retraction behavior, late block/unblock semantics, docs for the new hook context fields, and redirect-focused tests. If maintainers decide not to add this core policy surface, close it only after that explicit product/security decision and point plugin authors to an approved external-hook replacement path.
