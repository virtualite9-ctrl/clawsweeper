## Summary

Keep this issue open. Current main already ships the Korean Control UI locale, registration, docs, tests, and changelog evidence, but the issue also requests default AI-agent language persistence and automatic Korean search localization. Those agent/search behaviors are still not implemented as a platform default, and the paired same-author PR #53346 is closed unmerged.

## Best Possible Solution

Keep this issue open, but narrow the remaining maintainer scope to agent/search behavior. Preserve the shipped Korean Control UI locale. Design and test a default primary-language response policy in the agent prompt or conversation layer with override and literal-text preservation rules, and add Korean search localization only through provider-aware web-search parameters/configuration rather than assuming a universal `search_lang` field.
