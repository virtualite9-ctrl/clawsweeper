## Summary

Keep this issue open. Current main already has Korean Control UI locale support, and that UI portion is also present in latest release v2026.4.24, but the requested AI-agent language-persistence behavior is not implemented in the default agent system prompt. The same-author PR #53346 was closed unmerged, so there is no merged PR that resolves the remaining agent-side request.

## Best Possible Solution

Keep the issue open but narrow the remaining work: preserve the shipped Korean Control UI locale, then implement and test the agent-side language-persistence behavior in `src/agents/system-prompt.ts` and related prompt tests. If maintainers want localized search defaults, add explicit tested guidance or config-aware behavior for Korean queries instead of relying only on provider-level `search_lang` support.
