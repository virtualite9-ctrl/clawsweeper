## Summary

Keep #8495 open. Current main still blocks or skips unauthorized WhatsApp group traffic only; it does not auto-leave unauthorized groups, emit a WhatsApp group-join hook/event, or expose a WhatsApp leave-group action. The provided issue discussion also includes a maintainer-side Codex review explicitly keeping this open, and related #11489 remains broader/open rather than a shipped replacement.

## Best Possible Solution

Keep #8495 open. The best path is a deliberate bundled WhatsApp plugin change: add opt-in config/schema/docs for auto-leaving unauthorized groups, implement a Baileys-backed leave path with account and target authorization, cover unauthorized group admission/join behavior with tests, and decide whether join notification belongs in a WhatsApp-specific event, a generic group-join plugin hook, or the broader open #11489 group-pairing design. A manual leave action should only ship if the channel action/runtime surface can enforce safe target authorization.
