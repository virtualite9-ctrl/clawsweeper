## Summary

Keep #8495 open. Current main still implements WhatsApp group allowlisting as message admission/skip behavior only; it does not auto-leave unauthorized groups, emit a WhatsApp group-join hook/event, or expose a WhatsApp leave-group message action. The provided discussion also includes an April 26, 2026 Codex review comment explicitly keeping this issue open, and related #11489 is broader/open rather than a shipped replacement.

## Best Possible Solution

Keep #8495 open. The best path is an explicit bundled WhatsApp plugin design: opt-in config/schema/docs for auto-leave, a Baileys-backed leave path that enforces account and group authorization, tests around unauthorized group admission and join/reconciliation behavior, and a deliberate product decision on whether the join notification belongs in a WhatsApp-specific event, a generic group-join plugin hook, or the broader #11489 group-pairing flow. Manual leaveGroup support should only be added if the WhatsApp channel action/runtime surface can enforce target authorization safely.
