## Summary

Keep #8495 open. Current main still blocks unauthorized WhatsApp group traffic, but it does not auto-leave from unauthorized groups, expose a WhatsApp group-join hook/event, or support a WhatsApp leave-group message action. Related #11489 is broader and still open, so it does not supersede this WhatsApp-specific request.

## Best Possible Solution

Keep #8495 open for a deliberate bundled WhatsApp plugin feature. Add opt-in config/schema/docs for unauthorized-group auto-leave, implement a verified Baileys/runtime group leave operation with target/account authorization, add a group participant/join update listener or equivalent join detection, cover denied group and allowlist behavior with tests, and coordinate the owner notification/group-join event shape with the broader open #11489 design.
