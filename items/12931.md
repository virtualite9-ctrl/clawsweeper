## Summary

Current main still implements bare `/new` and `/reset` as a model-backed Session Startup greeting. The existing model override path only applies when reset commands have trailing text, and the public session config/schema/docs do not expose `resetGreeting`, `resetGreetingModel`, a silent reset mode, or a gateway-generated acknowledgement option. The prior issue comment is still accurate against current main e29d3516.

## Best Possible Solution

Keep this open for a core session/reset behavior change. A complete fix should add typed and schema-validated session config for bare reset greeting behavior, document it, and cover both requested modes: a configured cheaper greeting model for the bare reset turn and a no-model path that either sends no message or a gateway-generated acknowledgement. The likely implementation spans bare reset prompt construction, `runPreparedReply`, the gateway `agent` bare reset handler, reset model override handling, generated config metadata/help, and tests for bare `/new`, bare `/reset`, `/new <model>`, startup-context/bootstrap behavior, and persistence of existing model overrides.
