## Summary

Keep #12931 open. Current main still routes bare `/new` and `/reset` into a model-backed Session Startup greeting prompt, the model override path only runs when reset has trailing text, and the typed/schema/docs config surface still lacks `resetGreeting`, `resetGreetingModel`, silent reset, or static acknowledgement behavior. The prior Codex issue comment remains accurate against current main SHA 7e13f3f5140168e31027b39a9855ade3fadf24d5.

## Best Possible Solution

Keep this open for a core session/reset behavior change. The best implementation would add typed and schema-validated config for bare reset greeting behavior, support both a configured cheaper greeting model and a no-model silent/static-ack path, wire it through both `src/auto-reply/reply/get-reply-run.ts` and `src/gateway/server-methods/agent.ts`, document the options in public session/slash-command/config docs, and add regression tests for bare `/new`, bare `/reset`, `/new <model>`, startup-context/bootstrap interactions, and existing model override persistence.
