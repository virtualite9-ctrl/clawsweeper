## Summary

Keep #12931 open. Current main still sends bare `/new` and `/reset` through a model-facing Session Startup greeting prompt in both auto-reply and gateway agent paths, and the typed/session schema surface still lacks `resetGreeting`, `resetGreetingModel`, silent reset, or static acknowledgement controls. The existing April 26 maintainer-side Codex review comment also kept this open, and the current implementation evidence still supports that decision.

## Best Possible Solution

Keep the issue open for a core session/reset behavior change. The best implementation would add typed and schema-validated session config for reset greeting behavior, support both a configured cheaper greeting model and a no-model silent/static acknowledgement path, wire it through both auto-reply and gateway bare reset paths, preserve existing `/new <model>` semantics deliberately, and update session/slash-command/config docs plus regression tests for bare `/new`, bare `/reset`, `/new <model>`, bootstrap/startupContext interactions, and model override persistence.
