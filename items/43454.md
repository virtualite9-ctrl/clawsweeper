## Summary

Current main still does not implement the requested workspace/internal `HOOK.md` lifecycle events. The adjacent Plugin SDK hooks (`subagent_ended`, `agent_end`, `before_tool_call`, `after_tool_call`) are real and documented, but they are a different extension surface than the internal workspace hook API requested here. The report is concrete, recent, and not superseded by the provided related context.

## Best Possible Solution

Keep this issue open for maintainer API judgment. The concrete path is either to add a narrow internal-hook bridge for `subagent:ended` and define documented `agent:turnComplete` / tool-count events with tests, or to make an explicit product decision that these workflows should use existing Plugin SDK hooks and update docs so workspace hook authors are not left guessing.
