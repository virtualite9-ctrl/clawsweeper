## Summary

Keep open. Current main has strong adjacent Plugin SDK coverage (`agent_end`, `before_tool_call`, `after_tool_call`, `subagent_ended`) and a bundled Skill Workshop plugin that uses those hooks for threshold-based skill capture, but it still does not implement the requested internal/workspace `HOOK.md` events (`subagent:ended`, `agent:turnComplete`, or a tool-call threshold event). The issue is recent, concrete, has only the `stale` label, and the related open issue #49019 is a different `/hooks/agent` result-return API request rather than a canonical duplicate.

## Best Possible Solution

Keep this issue open for maintainer API judgment. The concrete implementation path is either to add a narrow internal-hook bridge for `subagent:ended` and define documented `agent:turnComplete` / tool-count events with tests, or to make an explicit docs/product decision that lifecycle automation belongs in Plugin SDK hooks and existing plugins such as Skill Workshop rather than workspace `HOOK.md` hooks.
