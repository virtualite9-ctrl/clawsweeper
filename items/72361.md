## Summary

Keep this PR open. Current main still has the reported `/allowlist --store` policy gap: store-only edits skip the `resolveConfigWriteDeniedText` guard and call `updatePairingStoreAllowlist` directly. The PR is paired with open issue #72360 via closing syntax, and its two-file patch adds the missing guard plus focused tests, so it should be reviewed/merged or superseded by an equivalent fix rather than closed during cleanup.

## Best Possible Solution

Keep this PR open and review/merge it, or manually port the same narrow fix: apply `resolveConfigWriteDeniedText` to store-targeted `/allowlist add|remove --store` mutations before `updatePairingStoreAllowlist`, using the explicit channel/account config-write target, and keep the channel-level false, account-level false, and allowed regression tests. Because this PR is paired with open issue #72360 via closing syntax and by the same author, close both only after the implementation lands or an equivalent maintainer-approved fix supersedes it.
