## Summary

Keep open. Current `main` still rejects `gateway.bind="lan"|"custom"` with `gateway.tailscale.mode="serve"|"funnel"`, and the runtime legacy-migration path does not include the proposed loopback normalization. I did not find evidence that PR #41009 is already implemented on `main`.

## Evidence

- **Gateway runtime migrations do not implement the proposed tailscale bind rewrite:** `LEGACY_CONFIG_MIGRATIONS_RUNTIME_GATEWAY` only contains `gateway.controlUi.allowedOrigins-seed-for-non-loopback` and `gateway.bind.host-alias->bind-mode`. There is no migration that rewrites explicit non-loopback bind modes to `loopback` when `gateway.tailscale.mode` is `serve`/`funnel`.
  - file: [src/commands/doctor/shared/legacy-config-migrations.runtime.gateway.ts:54](https://github.com/openclaw/openclaw/blob/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503/src/commands/doctor/shared/legacy-config-migrations.runtime.gateway.ts#L54)
  - command: `nl -ba src/commands/doctor/shared/legacy-config-migrations.runtime.gateway.ts | sed -n '1,140p'`
  - sha: [b0d2003fe65b](https://github.com/openclaw/openclaw/commit/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503)
- **Config load path applies only the existing runtime compat migrations before validation:** `resolveLegacyConfigForRead(...)` calls `applyRuntimeLegacyConfigMigrations(resolvedConfigRaw)` and then validates the result. Since the runtime migration set above lacks the tailscale bind normalization, a legacy `bind=lan`/`tailnet`/non-loopback `custom` config will reach validation unchanged.
  - file: [src/config/io.ts:1072](https://github.com/openclaw/openclaw/blob/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503/src/config/io.ts#L1072)

- **Validation on main still hard-rejects non-loopback serve/funnel binds:** `validateGatewayTailscaleBind` returns an error unless the bind resolves to loopback or `custom`+`127.0.0.1`. An explicit legacy `bind: "lan"` with `tailscale.mode: "serve"` still fails with `gateway.bind must resolve to loopback...`.
  - file: [src/config/validation.ts:543](https://github.com/openclaw/openclaw/blob/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503/src/config/validation.ts#L543)
