## Summary

Keep open. Current main still renders `openclaw logs` timestamps as UTC by default: it parses the top-level `time` field, but then formats it with `Date#toISOString()` unless `--local-time` or a valid `TZ` env var enables local rendering. The existing tests and docs still describe/assert UTC-by-default behavior with local time as an opt-in flag. Related PRs #46782 and #47365 attempt the requested default-local behavior but are still open, while #47240 was closed unmerged.

## Best Possible Solution

Keep #46748 open as the canonical issue. The likely fix belongs in `src/cli/logs-cli.ts`: make human-readable `openclaw logs` output default to host-local timestamp rendering with an explicit UTC opt-out, update the CLI docs, and add regression coverage showing a `+08:00` log `time` field stays local by default. The console-log timezone concern from dmak's comment should either be covered by the same regression set or split into a narrower follow-up if Docker process timezone detection is distinct.
