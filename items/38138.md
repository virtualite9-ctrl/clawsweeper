## Summary

Keep #38138 open. Current main still routes Discord thread messages from static channel/account/peer/parentPeer inputs before mention detection, and the later mention logic only gates the already selected effective route. The prior ClawSweeper keep-open comment remains consistent with current main at 4a3030df9efa72c44bab567a4390ac0c4876a73f.

## Best Possible Solution

Keep the issue open. Implement a Discord mention-based routing override before mention gating, either as a highest-priority route tier or a documented Discord preflight override that maps native bot identity and per-agent mentionPatterns to an agent. Add regression coverage for a Discord thread inheriting parentPeer while explicitly mentioning another configured agent, then update the public Discord and multi-agent routing docs.
