## Summary

Keep #38138 open. Current main still resolves Discord agent routing from static channel/account/guild/role/peer/parentPeer inputs before Discord mention detection, and mentionPatterns/native mentions only gate the already selected effective route. The provided prior ClawSweeper comment reached the same conclusion on 2026-04-26, and the relevant routing/preflight files have not changed since that review.

## Best Possible Solution

Keep the issue open. Implement a Discord mention-based routing override before mention gating, either as a highest-priority route tier or a documented Discord preflight override that maps native bot mentions and per-agent mentionPatterns to an agent. Add regression coverage for a Discord thread inheriting a parentPeer binding while explicitly mentioning another configured agent, then update Discord and multi-agent docs with the final precedence rules.
