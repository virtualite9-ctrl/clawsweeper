## Summary

Current main still lacks a top-level `auth` or `auth.profiles` reload rule, so `auth.profiles.*` remains unmatched and falls through to the restart path. The issue is also paired with open PR #68257 by the same author, so the maintainer path is to review and land, revise, or close that implementation candidate rather than close this issue early.

## Best Possible Solution

Keep this issue open and route maintainer review to PR #68257 or an equivalent gateway reload planner fix. The fix should classify OAuth/profile-refresh diffs under `auth.profiles.*` as no-op or dynamically read changes, keep `gateway.auth.*` as restart-triggering gateway configuration, and add regression tests for both behaviors.
