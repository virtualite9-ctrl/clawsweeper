## Summary

Keep this issue open. Current main still renders the Control UI Sessions Updated column with a relative timestamp, not the requested full `YYYY-MM-DD HH:mm` date-time, and the linked implementation PR #53445 is still open/unmerged with closing syntax for this issue.

## Best Possible Solution

Keep this issue open while #53445 remains open. Maintainers should review and land that PR if acceptable, or implement the same narrow Control UI change on main: add a tested full local date-time formatter, render the Sessions Updated column as `YYYY-MM-DD HH:mm`, and optionally preserve relative time as hover context.
