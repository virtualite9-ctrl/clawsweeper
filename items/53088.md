## Summary

Current main still does not implement the requested Control UI Sessions list full date/time display. The Sessions table computes the Updated cell with the relative formatter and renders that string directly, while the row already has an updatedAt timestamp available. The provided GitHub context also shows open, unmerged PR #53445 with closing syntax for this issue, so this issue should remain open until that PR lands or an equivalent main-branch fix replaces it.

## Best Possible Solution

Keep this issue open while #53445 remains open. Maintainers should review and land that PR if acceptable, or replace it with the same narrow Control UI change on main: add a tested full date-time formatter for YYYY-MM-DD HH:mm, render the Sessions Updated column with it, and optionally keep the relative timestamp as hover/title context without changing the sessions API or sort behavior.
