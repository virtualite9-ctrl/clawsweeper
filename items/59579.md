## Summary

The original ~60s scheduled-cron timeout regression is already fixed on `main` and was shipped in `v2026.4.9`. Current code explicitly marks scheduled runs as `trigger: "cron"`, and the idle-timeout resolver now disables the default LLM idle watchdog for cron runs unless the operator configured a specific timeout. The issue discussion itself also shows the symptom changed in `v2026.4.9` from the reported ~60s profile timeout to a separate ~300s job timeout, which is now tracked separately in #63805.

## Evidence
