## Summary

Keep this PR open. Current main has related update/config safeguards, but it still does not implement the PR’s focused behavior: bounded doctor summaries after failed `update.run`, sentinel formatting for those summaries, or doctor-backed diagnostics when config-reload restart checks fail.

## Best Possible Solution

Keep the PR open for maintainer review or author refresh. The best path is to rebase it onto current main, keep the bounded doctor-summary behavior for failed `update.run` if maintainers want that UX, add sentinel schema/formatter support, and redesign the config-reload failure diagnostic so it reports without stale-sentinel risk. If maintainers prefer a broader native self-healing design, fold the useful failure-summary pieces into #55347 rather than closing this contribution as already implemented.
