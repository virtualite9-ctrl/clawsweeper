## Summary

Current main still has the reported preflight compaction short-circuit for fresh persisted token totals. The recent main changes add a separate active-transcript-byte trigger and session-rotation refresh handling, but they do not make `totalTokensFresh: true` token counts reach the normal threshold check. The issue should stay open until a focused fix lands or maintainers explicitly mark it duplicate of a canonical related issue such as #63892.

## Best Possible Solution

Keep this issue open and land one focused implementation that evaluates the compaction threshold using fresh persisted `totalTokens` instead of returning early. Add a regression test in `src/auto-reply/reply/agent-runner-memory.test.ts` for `totalTokensFresh: true` with cached tokens above the threshold triggering `compactEmbeddedPiSession()`. If maintainers prefer #63892 as the canonical tracker, explicitly mark this issue duplicate after choosing that path; otherwise close it only after the fix merges.
