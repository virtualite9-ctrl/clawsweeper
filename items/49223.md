## Summary

Keep open. Current `main` still routes `sessions_send` through the generic `inter_session` A2A path, explicitly teaches the target to use `REPLY_SKIP` / `ANNOUNCE_SKIP`, has no structured delivery-intent field, and still suppresses announce delivery on skip tokens. I did not find code evidence that the reported WhatsApp inter-session delivery suppression has been fixed.

## Evidence

- **No structured delivery intent in sessions_send:** `sessions_send` still only accepts `sessionKey`, `label`, `agentId`, `message`, and `timeoutSeconds`; there is no typed delivery-mode or explicit outbound-delivery flag on the tool input.
  - file: [src/agents/tools/sessions-send-tool.ts:37](https://github.com/openclaw/openclaw/blob/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4/src/agents/tools/sessions-send-tool.ts#L37)
  - sha: [f3c37a946cc4](https://github.com/openclaw/openclaw/commit/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4)
- **Cross-session sends are still injected as generic inter_session messages:** The tool sends the target run with `deliver: false`, the internal message channel, and `inputProvenance.kind = "inter_session"`, so the target still receives an undifferentiated inter-session turn rather than a dedicated delivery request.
  - file: [src/agents/tools/sessions-send-tool.ts:275](https://github.com/openclaw/openclaw/blob/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4/src/agents/tools/sessions-send-tool.ts#L275)
  - sha: [f3c37a946cc4](https://github.com/openclaw/openclaw/commit/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4)
- **Prompt guidance still prioritizes skip tokens:** The helper text for the reply/announce stages still says `If you want to stop the ping-pong, reply exactly "REPLY_SKIP".` and `If you want to remain silent, reply exactly "ANNOUNCE_SKIP". Any other reply will be posted to the target channel.` There is no carve-out for explicit delivery requests.
  - file: [src/agents/tools/sessions-send-helpers.ts:94](https://github.com/openclaw/openclaw/blob/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4/src/agents/tools/sessions-send-helpers.ts#L94)
  - sha: [f3c37a946cc4](https://github.com/openclaw/openclaw/commit/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4)
- **Announce delivery is still suppressed by skip-token handling:** `runSessionsSendA2AFlow` only posts when the announce reply is non-empty and not `ANNOUNCE_SKIP`; current tests still model the announce step returning `ANNOUNCE_SKIP` and expect no `send` call, which matches the reported suppression behavior instead of fixing it.
  - file: [src/agents/tools/sessions-send-tool.a2a.ts:131](https://github.com/openclaw/openclaw/blob/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4/src/agents/tools/sessions-send-tool.a2a.ts#L131)
