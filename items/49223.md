## Summary

Keep open. Current main still routes `sessions_send` target turns as generic `inter_session` messages, still advertises `REPLY_SKIP`/`ANNOUNCE_SKIP` in the A2A prompts, and still suppresses visible delivery when the announce step returns `ANNOUNCE_SKIP`. I did not find a typed delivery-intent mode, a prompt carve-out for explicit outbound delivery requests, or a WhatsApp regression test. Related #43848 overlaps the sessions_send skip behavior, but it is not a clear canonical replacement for this WhatsApp explicit-delivery issue.

## Best Possible Solution

Keep the issue open and fix it in the core `sessions_send`/A2A path. The best product direction is a backward-compatible delivery-intent signal or explicit prompt/runtime carve-out so requests to post a normal channel message cannot be suppressed by generic `REPLY_SKIP`, `ANNOUNCE_SKIP`, or `NO_REPLY` guidance. Add focused tests for a WhatsApp group target with authorized/always activation, explicit delivery wording, and forbidden skip-token outcomes, while preserving skip tokens for genuine coordination chatter.
