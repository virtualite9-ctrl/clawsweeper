## Summary

Keep open. Current main still implements stuck-session diagnostics as a simple processing-age warning with no progress marker, no repeated-warning backoff, and no ACP/reply/tool/block progress refresh. The PR is recent, focused, includes regression-test and docs updates, and replaces #71969, which was closed unmerged by the too-many-PRs guard rather than superseded by main.

## Best Possible Solution

Keep #72010 open. The best path is maintainer review of the focused diagnostic behavior, then rebase if needed and merge after the stated changed/test lanes remain green. The implementation belongs in core diagnostics and dispatch/ACP paths, not ClawHub, because it changes built-in stuck-session warning semantics.
