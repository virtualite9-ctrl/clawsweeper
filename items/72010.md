## Summary

Keep open. Current `main` has not implemented the PR’s central diagnostics change: stuck-session warnings still use raw elapsed time since the last queued/state event, there is no diagnostic progress marker, ACP runtime events are not wired as progress, repeated stuck warnings do not back off, and the public config docs still describe an age threshold rather than no-progress semantics. The PR is also recent, contributor-authored, and the provided review context shows the earlier diagnostics-enabled concern was addressed in the PR rather than superseded on `main`.

## Best Possible Solution

Keep PR #72010 open for maintainer review or landing. The best path is to review the PR as the active implementation of no-progress stuck-session diagnostics, including the diagnostics.enabled gating fix noted by the Codex review comment, the ACP/runtime progress wiring, the repeated-warning backoff, and the associated tests/docs/generated config updates. If maintainers do not want this behavior, close with product rationale rather than as obsolete.
