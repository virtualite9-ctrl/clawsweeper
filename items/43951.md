## Summary

Keep open. Current main still does not preserve `accountId` on direct `POST /hooks/agent`, while cron delivery itself already honors `delivery.accountId`; this PR is a plausible paired fix for open #43866, but it needs maintainer review and a rebase because the cron message-tool policy area has changed on main.

## Best Possible Solution

Keep the PR open. Best path is to rebase it onto current main, preserve the focused hook-side fix (`accountId` normalization, job delivery propagation, replay-key scoping, observable normalization failure, and regression tests), adapt or remove stale cron tool-policy hunks against the current `deliveryMode`/`forceMessageTool` implementation, add docs for `accountId`, and fix the changelog attribution before maintainer review.
