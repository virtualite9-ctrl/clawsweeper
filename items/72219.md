## Summary

Keep #72219 open. Current main still has the reported cron trace behavior: `delivery.mode: "none"` produces a non-requested delivery state, but `buildCronDeliveryTrace` still serializes a `delivery.resolved` failure object. The PR is a focused implementation for linked issue #72210 and is not obsolete, protected, or superseded by another canonical PR in the provided context.

## Best Possible Solution

Keep the PR open as the active fix for #72210. The best maintainer path is to review and land a narrow change in `buildCronDeliveryTrace` that omits `delivery.resolved` only for `deliveryPlan.mode === "none"`, keeps `deliveryStatus: "not-requested"`, preserves existing announce/webhook traces, and includes the proposed regression test in `src/cron/isolated-agent/run.message-tool-policy.test.ts`.
