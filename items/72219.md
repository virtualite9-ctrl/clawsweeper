## Summary

Keep this PR open. Current main still serializes a failure-shaped `delivery.resolved` object for intentionally disabled cron delivery, while linked issue #72210 remains open and this PR is a focused implementation candidate with a regression test.

## Best Possible Solution

Keep this PR open for maintainer review. Land this patch or an equivalent targeted change that omits the misleading `delivery.resolved` failure for intentionally disabled fallback delivery, preserves message-tool and explicit-target semantics, and keeps the added regression test for the linked cron run-record bug.
