## Summary

Current main still rejects whitespace in `OPENCLAW_CONFIG_DIR` and `OPENCLAW_WORKSPACE_DIR` during Docker setup, matching the reported failure. Related PRs identify the same root cause, but the visible fix attempts are closed unmerged or still open, so the original issue should remain open until a fix lands.

## Best Possible Solution

Keep #44599 open and land a narrow Docker setup fix: remove the blanket whitespace rejection for `OPENCLAW_CONFIG_DIR` and `OPENCLAW_WORKSPACE_DIR`, continue rejecting newline/CR/tab control characters, quote any generated YAML mount strings that need it, and add an e2e regression test using config and workspace directories with spaces. Once that lands, close this issue from the fixing PR and close or supersede duplicate fix PRs.
