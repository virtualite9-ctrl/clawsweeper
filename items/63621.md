## Summary

PR #63621 adds a zh-TW glossary under `docs/.i18n`, but the stated goal is Control UI terminology. Current `main` shows the Control UI i18n generator reads `ui/src/i18n/.i18n/glossary.zh-TW.json`, not `docs/.i18n`, and the docs translation/sync surfaces do not currently include zh-TW as an active generated docs locale. The remaining UI terminology/parity work is already tracked by PR #59706, with #52357 as the original related PR.

## Best Possible Solution

Close this PR as superseded for core cleanup: keep #59706 as the canonical Control UI zh-TW terminology/parity thread, and ask for any durable glossary work to come back as a smaller targeted PR against `ui/src/i18n/.i18n/glossary.zh-TW.json` with demonstrated UI pipeline misses.
