## Summary

Keep open. I found no strong evidence that current `main` implements Claude Code ACP slash-command parity for `/add-dir`. The ACP surface on `main` still documents and advertises only OpenClaw `/acp` controls, and the ACP dispatch code only special-cases `/new`/`/reset` plus `!` commands. There is no repo implementation, test, or doc mention of `add-dir` support.

## Evidence

- **No `add-dir` implementation found:** `rg -n "add-dir" src docs extensions test --glob '!extensions/diffs/**'` returned no matches, so there is no current first-class implementation, test, or docs for `/add-dir` support in OpenClaw ACP paths.
  - command: `rg -n "add-dir" src docs extensions test --glob '!extensions/diffs/**'`
  - sha: [b0d2003fe65b](https://github.com/openclaw/openclaw/commit/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503)
- **ACP advertised commands are OpenClaw-only:** The ACP available-command list contains OpenClaw commands such as `help`, `status`, `reset`, `new`, `bash`, and `compact`; it does not expose harness-native commands like `add-dir`.
  - file: [src/acp/commands.ts:4](https://github.com/openclaw/openclaw/blob/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503/src/acp/commands.ts#L4)
  - sha: [b0d2003fe65b](https://github.com/openclaw/openclaw/commit/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503)
- **Docs only describe `/acp` controls:** The ACP agents docs list only `/acp spawn|cancel|steer|close|status|set-mode|set|cwd|permissions|timeout|model|reset-options|sessions|doctor|install` as the supported command surface and map runtime options through `/acp ...`; there is no documented Claude Code slash-command passthrough such as `/add-dir`.
  - file: [docs/tools/acp-agents.md:487](https://github.com/openclaw/openclaw/blob/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503/docs/tools/acp-agents.md#L487)
  - sha: [b0d2003fe65b](https://github.com/openclaw/openclaw/commit/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503)
- **ACP dispatch has no dedicated Claude slash-command handling:** `shouldBypassAcpDispatchForCommand` only treats `/new` and `/reset` as special slash commands, otherwise only handling `!` commands. There is no dedicated handling for harness-native slash commands like `/add-dir`, which matches the reported feature gap.
  - file: [src/auto-reply/reply/dispatch-acp-command-bypass.ts:26](https://github.com/openclaw/openclaw/blob/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503/src/auto-reply/reply/dispatch-acp-command-bypass.ts#L26)
  - sha: [b0d2003fe65b](https://github.com/openclaw/openclaw/commit/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503)
