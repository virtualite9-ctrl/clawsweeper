## Summary

Keep #38604 open. Current main still resolves Docker sandbox `pidsLimit` only from explicit agent/global config, and `docker create` only emits `--pids-limit` when that resolved value is already a positive number. The related open PR #44098 is a plausible implementation path but is not merged or shipped; the later closed PR attempts were also unmerged. The WASM runtime idea raised in the discussion is broader security work, but it does not replace the missing Docker sandbox PID default reported here.

## Best Possible Solution

Keep #38604 open until a focused core fix lands. The likely path is to add a maintainer-approved default PID limit, apply it in `resolveSandboxDockerConfig` while preserving explicit agent/global overrides, add regression coverage for default and override behavior, confirm `buildSandboxCreateArgs` emits the resolved default, and update the public sandbox/config docs. PR #44098 can remain the canonical implementation path if maintainers want to continue it.
