## Summary

Keep open. Current main still does not provide a default Docker sandbox pidsLimit: config resolution preserves only explicit agent/global values, Docker create args emit --pids-limit only when a positive value is already present, and the config schema/docs/tests cover explicit configuration rather than a runtime default. The related PR #44098 remains the best visible implementation path in the provided GitHub context, but it is still open and unmerged.

## Best Possible Solution

Keep #38604 open until a focused core fix lands. The best path is to add a maintainer-approved DEFAULT_SANDBOX_PIDS_LIMIT or equivalent, apply it in resolveSandboxDockerConfig while preserving explicit agent/global overrides, add regression coverage for default and override behavior, ensure buildSandboxCreateArgs emits the resolved default, and update the public sandbox/config docs. PR #44098 can be the canonical implementation path if maintainers want to continue it; otherwise land an equivalent focused patch.

## Risks / Open Questions

- Closing now would leave default Docker sandbox containers without a core PID-limit fallback unless users explicitly configure pidsLimit.
- The docs example can imply a PID limit is present even though current runtime behavior only applies it when configured.
- Maintainers still need to choose the default value and confirm compatibility for workloads that rely on Docker's default PID behavior.
