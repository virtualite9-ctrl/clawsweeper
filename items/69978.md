## Summary

Keep #69978 open. Current main still has only a context-level TUI streaming watchdog timeout, no user-facing `/watchdog` command or persistence, and no per-run warning dedupe. Related #68596 and #69081 are useful context and should be coordinated with, but they do not fully supersede the duplicate-warning plus TUI command request.

## Best Possible Solution

Keep the issue open and either implement it directly in the TUI or split it into coordinated work with #68596/#69081. The likely core change belongs in `src/tui`: add a mutable/persisted watchdog setting, expose `/watchdog` in `commands.ts` and `tui-command-handlers.ts`, thread the value into `createEventHandlers`, and add per-run warning dedupe cleared on final/abort/session change with regression tests.
