## Summary

Keep open. Current main still renders the Control UI chat transcript as a hardcoded polite live region while streamed assistant text is inserted into that transcript. There is also an open implementation PR, #65575, that uses closing syntax for this issue and directly changes this aria-live behavior with focused tests, so the issue should remain open until that PR is reviewed and merged or rejected.

## Best Possible Solution

Keep this issue open and treat #65575 as the current implementation candidate. The likely fix belongs in ui/src/ui/views/chat.ts, with focused tests in ui/src/ui/views/chat.test.ts proving the transcript remains polite when idle but suppresses announcements while sending or streaming. If maintainers reject #65575, an equivalent main-branch fix should still address the same live-region behavior before closing this issue.
