## Summary

Keep #17683 open. Current main has partial controls for isolated cron runs, lightweight context, tool allow-lists, timeouts, webhook-triggered agent runs, and post-run usage/cost telemetry, but it still lacks the requested first-class scoped/script-limited automation mode with native per-job token caps, daily spend ceilings, budget aborts, dry-run/cost previews, and admin-defined script-only execution.

## Best Possible Solution

Keep #17683 open as the product/API tracker. The best path is to add a native scoped automation mode across cron, hooks, gateway protocol, CLI, and docs: per-job token caps, daily spend ceilings, graceful budget aborts based on runtime usage accounting, admin-defined script-only execution, dry-run/plan cost preview, and structured planned-vs-actual run logs. Smaller follow-ups can split budget enforcement, script-only execution, and preview/logging.
