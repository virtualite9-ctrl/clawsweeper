## Summary

Keep #17683 open. Current main still provides partial scoped automation controls, but not the requested first-class scoped/script-limited mode with per-job or daily budget enforcement, budget aborts, dry-run cost previews, or planned-vs-actual run logs. The existing April 26 Codex/maintainer comment already kept this open, and the relevant current-main cron/hook/runner surfaces have not changed since that review.

## Best Possible Solution

Keep #17683 open as the broader product/API tracker. The best implementation is a native scoped automation mode spanning cron, /hooks/agent, gateway protocol/schema, CLI, embedded runner accounting, docs, and tests: per-job token caps, daily spend ceilings, graceful budget aborts, admin-defined script-only execution, dry-run cost previews, and structured planned-vs-actual run logs, with #38248 treated as a narrower cost-guard component.
