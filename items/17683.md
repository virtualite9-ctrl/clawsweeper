## Summary

Keep #17683 open. Current main has partial controls for cron and hook-triggered isolated agent turns, including lightweight context, timeouts, tool allow-lists, isolated sessions, and post-run usage/cost telemetry, but it still lacks the requested first-class scoped/script-limited mode with native token/spend ceilings, budget aborts, script-only execution, dry-run cost preview, and planned-vs-actual run logs.

## Best Possible Solution

Keep #17683 open as the product/API tracker. The best path is a native scoped automation mode spanning cron, hooks, gateway protocol, CLI, embedded runner accounting, docs, and tests: per-job token caps, daily spend ceilings, graceful budget aborts using runtime usage accounting, admin-defined script-only execution, dry-run/plan cost preview, and structured planned-vs-actual run logs.
