## Summary

Keep this issue open. Current main still does not expose a CDP/IME-style text insertion action in the browser act schema, request union, normalizer, CLI/docs, managed Playwright dispatch, or existing-session Chrome MCP path. The existing `type` action still resolves to Playwright `fill`/`type` for managed/raw CDP profiles and Chrome MCP `fill` for existing-session profiles, which is the behavior the issue says is insufficient for contenteditable IME cases like the X composer.

## Best Possible Solution

Keep this issue open and implement the capability in the bundled browser plugin as an additive browser act kind. The likely path is to extend `BrowserActRequest`, `BROWSER_ACT_KINDS`, `ACT_KINDS`, normalization, CLI/tool docs, timeout handling, and `/act` dispatch; focus the target element/tab; call CDP `Input.insertText` for managed/raw CDP profiles; define either supported behavior or a clear unsupported error for Chrome MCP existing-session profiles; and add focused tests for schema acceptance, normalization, dispatch, and docs.
