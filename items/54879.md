## Summary

Keep this issue open. Current main still does not expose a browser `act` kind for CDP/IME-native text insertion, and the implemented text paths still route through Playwright `type`/`fill` or Chrome MCP `fill`, not CDP `Input.insertText`. The issue is concrete, recent, unprotected, and the prior Codex review comment reached the same keep-open conclusion against a nearby main revision.

## Best Possible Solution

Implement an additive browser plugin `act` kind, likely `insertTextIme` or `cdpInsertText`, rather than closing this issue. The change should update the browser tool schema, `BrowserActRequest`, `ACT_KINDS`, normalization, CLI/docs, timeout handling, `/act` dispatch, and tests. For managed/raw CDP profiles, focus the requested target and send CDP `Input.insertText`; for existing-session Chrome MCP profiles, either provide an equivalent low-level path or document and return a specific unsupported error.
