## Summary

Keep this issue open. Current main still lacks an additive browser act kind for CDP/IME-native text insertion, and both existing text paths remain implemented as Playwright type/fill or Chrome MCP fill semantics rather than CDP Input.insertText.

## Best Possible Solution

Implement an additive browser plugin act kind, likely insertTextIme or cdpInsertText, in the bundled browser plugin. It should update the tool schema, BrowserActRequest, ACT_KINDS, normalization, CLI/docs, timeout handling, /act dispatch, managed/raw CDP execution using Input.insertText after focusing the target, and tests for schema/normalization/dispatch/docs. Existing-session Chrome MCP behavior should be defined explicitly rather than silently falling back to fill semantics.
