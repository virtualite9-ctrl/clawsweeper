## Summary

Keep #34400 open. Current main has partial recursive support: builtin file discovery recursively walks `memory/`, and the builtin watcher was recently repaired to watch directory paths instead of glob paths. Meaningful requested behavior is still incomplete because nested dated memory files are still classified as evergreen for temporal decay, dirty `memory_search` still starts sync in the background, QMD watch targets still join collection paths with glob patterns, and user-facing tool/config/docs copy still advertises flat `memory/*.md`.

## Best Possible Solution

Keep #34400 open and land a focused memory-core follow-up. The likely maintainer path is: support nested dated paths in temporal decay, decide whether dirty `memory_search` should await sync for first-call freshness, audit/fix QMD collection watch targets, update tool/prompt/CLI/config/docs copy from `memory/*.md` to `memory/**/*.md`, and add regression tests for nested dated files plus watcher target construction. Reuse the concrete analysis from closed PRs #34585 and #35691, but account for the builtin watcher fix already present in e53c068d.
