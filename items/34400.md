## Summary

Keep #34400 open. Current main partially supports recursive default memory discovery, but it still does not fully solve the issue discussion: nested dated memory paths are not parsed for temporal decay, dirty searches still trigger background sync instead of awaiting freshness, watcher targets still include chokidar glob paths for extra/QMD directories, and user-facing copy still advertises flat `memory/*.md` files. The related fix PRs #34585 and #35691 were closed unmerged, so there is no canonical landed replacement to close against.

## Best Possible Solution

Keep #34400 open and land a focused memory-core fix or maintainer decision. The likely path is to support nested dated memory paths in temporal decay, replace chokidar glob watch targets with directory targets plus filtering for builtin extraPaths and QMD collections where needed, decide whether dirty `memory_search` should await sync for first-call freshness, update tool/prompt/config/docs copy to `memory/**/*.md`, and add regression tests for nested dated files and watcher target construction. Reuse useful analysis from the closed unmerged PRs #34585 and #35691 rather than closing this issue as complete.
