## Summary

Keep open. Current main partially supports recursive collection of default memory files, but the issue's follow-up runtime and user-facing gaps are still present: dated files under memory subdirectories are still treated as evergreen, built-in dirty search sync is still fire-and-forget, and multiple user-facing descriptions still say memory/*.md. The related fixing PRs in the provided GitHub context are closed unmerged, so there is no canonical merged fix to close against.

## Best Possible Solution

Keep #34400 open and land a focused memory-core fix or maintainer decision. The likely implementation path is to update dated-memory path recognition for nested memory files, decide whether first dirty search must await sync for freshness, repair any chokidar v5 glob watcher paths still in use, and align tool/prompt/config/docs strings with the shipped recursive behavior. Tests should cover nested dated files and first-search freshness for the selected backend behavior.
