## Summary

Keep #34400 open. Current main recursively discovers and indexes default `memory/` Markdown files, but the issue discussion identified broader runtime and user-facing gaps that remain on current main: nested dated memory files still bypass temporal-decay date parsing, dirty `memory_search` still starts sync in the background instead of awaiting freshness, extraPath directory watching still passes glob paths to chokidar 5, and several user-facing strings still say `memory/*.md`. The related PRs #34585 and #35691 were closed unmerged, so there is no complete shipped fix to close against.

## Best Possible Solution

Keep #34400 open and land a focused memory-core fix or maintainer decision. The likely implementation is to allow nested dated memory paths in temporal decay, decide whether dirty builtin `memory_search` should await sync for first-call freshness, watch extraPath directories using chokidar 5-compatible directory paths plus filtering, and align tool descriptions, prompt guidance, config help, CLI/docs copy, and regression tests with recursive `memory/**/*.md` behavior.
