## Summary

Keep #34400 open. Current main recursively discovers default memory Markdown files, but the broader issue discussion is still valid: nested dated memory files do not receive path-date temporal decay, dirty `memory_search` still starts sync asynchronously, extraPath directory watching still uses glob targets despite chokidar 5, and user-facing copy still says `memory/*.md`. The related fix PRs were closed unmerged.

## Best Possible Solution

Keep #34400 open and land a focused memory-core fix or maintainer decision. The likely path is to support nested dated memory paths in temporal decay, replace extraPath watcher globs with chokidar-5-compatible directory targets plus filtering, decide whether dirty `memory_search` should await sync for first-call freshness, update tool/prompt/config/docs copy to `memory/**/*.md`, and add regression tests for nested dated files and watcher targets.
