## Summary

Keep open. Current `main` still matches the report: Feishu media upload supports a `duration` field, but `sendMediaFeishu` does not pass it for audio uploads, and the latest release snapshot shows the same omission.

## Evidence

- **Upload helper already supports duration:** `uploadFileFeishu` declares `duration?: number` and forwards it into `client.im.file.create`, so the Feishu upload path is capable of sending audio/video duration metadata.
  - file: [extensions/feishu/src/media.ts:367](https://github.com/openclaw/openclaw/blob/8b76392e3e7934b1cdd6d461148263fd56141ec8/extensions/feishu/src/media.ts#L367)
  - command: `nl -ba extensions/feishu/src/media.ts | sed -n '367,392p'`
  - sha: [8b76392e3e79](https://github.com/openclaw/openclaw/commit/8b76392e3e7934b1cdd6d461148263fd56141ec8)
- **Current caller still omits duration:** `sendMediaFeishu` routes non-image media through `uploadFileFeishu` but passes only `cfg`, `file`, `fileName`, `fileType`, and `accountId`; no `duration` is computed or forwarded for `audio`/`media` messages.
  - file: [extensions/feishu/src/media.ts:625](https://github.com/openclaw/openclaw/blob/8b76392e3e7934b1cdd6d461148263fd56141ec8/extensions/feishu/src/media.ts#L625)
  - command: `nl -ba extensions/feishu/src/media.ts | sed -n '625,637p'`
  - sha: [8b76392e3e79](https://github.com/openclaw/openclaw/commit/8b76392e3e7934b1cdd6d461148263fd56141ec8)
- **Existing audio test does not cover duration:** The Feishu media test for `.opus` verifies `file_type: "opus"` and `msg_type: "audio"`, but it does not assert that upload payloads include `duration`, so this regression is currently unguarded by tests.
  - file: [extensions/feishu/src/media.test.ts:171](https://github.com/openclaw/openclaw/blob/8b76392e3e7934b1cdd6d461148263fd56141ec8/extensions/feishu/src/media.test.ts#L171)
  - command: `nl -ba extensions/feishu/src/media.test.ts | sed -n '171,189p'`
  - sha: [8b76392e3e79](https://github.com/openclaw/openclaw/commit/8b76392e3e7934b1cdd6d461148263fd56141ec8)
- **Latest release still has the omission:** The same `uploadFileFeishu` call shape appears in release `v2026.4.23`, so there is no evidence this was fixed and shipped after the reporter's `2026.4.5` version.
  - file: [extensions/feishu/src/media.ts:631](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/extensions/feishu/src/media.ts#L631)
  - command: `git show a9797214338ba31c52c796adbb75afb16e0684a9:extensions/feishu/src/media.ts | nl -ba | sed -n '631,637p'`
  - sha: [a9797214338b](https://github.com/openclaw/openclaw/commit/a9797214338ba31c52c796adbb75afb16e0684a9)

## Risks / Open Questions

- Feishu audio messages likely continue to display a 0-second duration because OpenClaw uploads opus audio without the metadata field Feishu expects.
- A future fix needs both duration extraction for supported audio/video formats and a focused regression test, otherwise the bug can recur silently.
