## Risks / Open Questions

- The PR would delete existing legacy numeric .bak.N rollback snapshots on the first upgraded config write before the datetime backup set is populated.
- The PR documents but does not prevent same-millisecond timestamp collisions, so one backup can be overwritten or dropped during burst writes.
- There are two open implementation candidates for #39923, so maintainers still need to choose a canonical direction or request a combined revision.
