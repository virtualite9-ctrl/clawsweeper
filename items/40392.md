Keep this PR open. Current main still implements numbered config backup rotation, and this PR remains a coherent implementation candidate for the open same-author feature issue #39923. The overlapping PR #39982 tracks similar work, but there is no maintainer signal choosing it as canonical or superseding this PR.

## Best Possible Solution

Keep this PR open for maintainer review or requested changes. The best path is to choose one canonical datetime-backup implementation, preserve or migrate existing .bak.N rollback files during rollout, keep bounded retention, handle timestamp collisions without clobbering snapshots, update the live config-write tests, and close #39923 only after the selected implementation merges.
