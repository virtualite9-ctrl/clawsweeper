## Summary

Current `main` still matches the reported false-positive path: `gateway probe` can probe both an SSH tunnel URL and the configured remote URL, de-duplicates only exact URL matches, and emits `multiple_gateways` solely from the count of reachable targets. The per-target gateway `self` presence is already extracted but is not used to distinguish one gateway over two transports from two distinct gateways.

## Best Possible Solution

Keep the issue open. The likely fix belongs in `src/commands/gateway-status/output.ts`: group reachable probes by a stable gateway identity derived from `self` presence when available, suppress or downgrade the warning when all reachable transports identify the same gateway, and keep `multiple_gateways` for distinct identities or ambiguous cases. Add focused tests for `sshTunnel` plus `configRemote` sharing one identity and for distinct identities still warning, then update the gateway CLI docs if warning copy changes.

## Evidence
