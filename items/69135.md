## Summary

Current main still matches the reported false-positive path: `gateway probe` adds an SSH tunnel target plus configured remote targets when the URLs differ, then emits `multiple_gateways` for any two reachable probes without comparing gateway identity or presence. The issue is recent, unaffiliated, has no protected labels, and is a concrete CLI/gateway bug report with enough reproduction detail to keep open.

## Best Possible Solution

Keep the issue open and fix the warning policy in `src/commands/gateway-status/output.ts`, with focused tests in `src/commands/gateway-status/output.test.ts` or the command suite. The likely maintainer-grade fix is to group reachable targets by a stable gateway identity when available, preferably `gateway.identity.get` if the probe surface can safely fetch it, or a conservative `self` presence key as a fallback. Only emit `multiple_gateways` for two or more distinct identities, and use clearer informational wording for one gateway reachable through multiple transports. Update the CLI/gateway docs if warning semantics change.
