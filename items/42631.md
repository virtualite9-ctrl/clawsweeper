## Summary

Keep open. Current main still does not provide a first-class model/provider override for cron jobs that use `sessionTarget: "main"` with `payload.kind: "systemEvent"`; model selection remains agent-turn-only, and the related implementation PR #42708 was closed unmerged.

## Best Possible Solution

Keep this issue open. The best implementation is an additive, validated cron contract for model/provider overrides on `main + systemEvent` jobs, likely via documented `payload.model` support or a new `sessionOverride`/top-level override field. It should update protocol schema, runtime types, normalization, CLI/tool/docs surfaces, gateway/main-session execution, and tests proving the override reaches only the cron-triggered run and does not persist into unrelated interactive use.
