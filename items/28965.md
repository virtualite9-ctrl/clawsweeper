## Summary

Keep #28965 open. Current main still has targeted model status/list/set/scan/auth commands and per-session model override clearing, but it does not implement the requested first-class `openclaw models reconcile` command or scoped bulk `openclaw sessions model-reset` workflow. The related open issues #39811 and #41736 overlap model/config DX but do not supersede this specific reconcile plus session-reset proposal.

## Best Possible Solution

Keep #28965 open until OpenClaw implements an explicit model configuration reconciliation command plus scoped bulk session model override reset, or maintainers intentionally consolidate it into a canonical model/config SSOT design issue. The likely implementation belongs in the existing `models` and `sessions` CLI surfaces with `--dry-run`, `--json`, `--agent`, `--all-agents`, `--store`, optional match filters, regression tests for override clearing, and docs for model migration/troubleshooting.
