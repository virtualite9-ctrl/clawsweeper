## Summary

Keep #28965 open. Current main still has only targeted model normalization and per-session reset paths; it does not implement the requested `openclaw models reconcile` command or bulk `openclaw sessions model-reset` workflow. The prior ClawSweeper comment remains accurate after re-checking current main at 7e13f3f5140168e31027b39a9855ade3fadf24d5.

## Best Possible Solution

Keep this issue open until OpenClaw either implements first-class core CLI support for model configuration reconciliation plus bulk session model override reset, or a maintainer explicitly marks a canonical issue for the same concrete work. The implementation likely belongs in the core CLI, config update helpers, and session-store maintenance layer, with docs updates for `docs/cli/models.md`, `docs/cli/sessions.md`, and the model selection/failover concepts.
