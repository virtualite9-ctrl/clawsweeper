## Summary

Keep #28965 open. Current main still has targeted model normalization and per-session reset paths, but it does not implement the requested first-class `openclaw models reconcile` command or bulk `openclaw sessions model-reset` workflow.

## Best Possible Solution

Keep this issue open until OpenClaw implements deliberate core CLI support for model configuration reconciliation plus scoped bulk session model override reset, or maintainers explicitly designate a canonical issue for that exact remaining work. The likely implementation belongs in `openclaw models`, `openclaw sessions`, config update helpers, session-store maintenance code, regression tests for dry-run/scoped reset behavior, and user-facing model migration/troubleshooting docs.
