## Summary

Keep #72004 open. Current main still has the ordering the PR is trying to fix: the Control UI chat controller appends final/aborted assistant messages to `chatMessages` before clearing `chatStream`, while the renderer can build the view from both committed history and live stream state. The linked bug #71992 remains open, and the earlier similar PR #34164 was closed unmerged rather than superseding this work.

## Best Possible Solution

Keep #72004 open and review it as the likely focused fix for #71992. The best path is to land the stream-state-clearing reorder with the regression test after normal CI or equivalent maintainer validation, rather than closing it as implemented, duplicate, stale, or ClawHub work.
