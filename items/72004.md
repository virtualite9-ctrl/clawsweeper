## Summary

Keep #72004 open. Current main still appends assistant final/aborted messages to `chatMessages` before clearing `chatStream`, while the render path builds chat items from both committed messages and the live stream. The provided PR context also shows the linked bug #71992 is still open and the earlier similar PR #34164 was closed unmerged, so this focused PR is not obsolete or superseded.

## Best Possible Solution

Keep #72004 open for maintainer review or landing. The best fix is the PR's narrow approach: snapshot the streamed fallback, clear `chatStream`/`chatRunId`/`chatStreamStartedAt` before any `chatMessages` mutation in both `final` and `aborted`, and keep the regression test that observes stream state at the exact mutation point. Before merge, run the relevant UI changed gate when the environment supports it.
