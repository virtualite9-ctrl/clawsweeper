## Summary

Current `main` already covers this UI-refresh need with gateway-driven transcript updates instead of client polling. The web UI now reloads chat history on `session.message`, defers that reload safely while a run is active, and already renders live `chat` delta/final updates, so this PR's polling approach is obsolete on `main`.
