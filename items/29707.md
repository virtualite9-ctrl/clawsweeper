## Summary

Keep #29707 open. Current main has newer Bedrock application-inference-profile cache handling for Claude profiles, but direct Amazon Nova Bedrock model IDs are still excluded from cache-retention eligibility and still wrapped with `cacheRetention: "none"`. AWS's current Bedrock docs also show Amazon Nova prompt-caching and `cachePoint` usage, so this is still a real bundled Bedrock plugin/runtime gap rather than an obsolete request. ([docs.aws.amazon.com](https://docs.aws.amazon.com/bedrock/latest/userguide/prompt-caching.html))

## Best Possible Solution

Keep #29707 open for the bundled `amazon-bedrock` plugin. The best fix is to add explicit Nova-family cache eligibility for supported direct Bedrock model IDs, pass `cacheRetention` through instead of forcing `none`, inject or preserve Bedrock Nova `cachePoint` metadata according to the current AWS contract, keep unsupported non-Anthropic Bedrock models protected, and update the Bedrock regression tests plus https://docs.openclaw.ai/reference/prompt-caching.
