## Summary

Keep #29707 open. Current main still does not implement native Amazon Nova prompt caching in the bundled Amazon Bedrock provider: regular Nova model IDs are excluded from cache-retention eligibility, the Bedrock wrapper forces non-Anthropic calls to `cacheRetention: "none"`, and tests/docs still codify the older #20815/#20866 safety fix rather than a Nova-specific caching path.

## Best Possible Solution

Keep the issue open for the bundled Amazon Bedrock provider owner path. A proper fix should verify AWS Nova prompt-caching behavior, add Nova-family cache eligibility and runtime pass-through, make cachePoint insertion conditional on the current Nova Converse contract rather than the Anthropic matcher, preserve forced-off behavior for unsupported non-Anthropic Bedrock models, and update regression tests plus the public prompt-caching docs at https://docs.openclaw.ai/reference/prompt-caching.
