## Summary

Keep #29707 open. Current main still does not provide native Amazon Nova prompt caching on Bedrock: regular Nova model IDs are excluded from cache-retention eligibility, the Amazon Bedrock plugin wraps non-Anthropic Bedrock streams with `cacheRetention: "none"`, and docs/tests still describe Nova/Mistral cache settings as forced off. Recent related work improves Anthropic Bedrock/application-inference-profile cache handling, but it does not add a Nova-native cache path or remove the no-cache wrapper for Nova.

## Best Possible Solution

Keep the issue open for the bundled Amazon Bedrock provider owner path. A proper fix should verify AWS Nova prompt-caching semantics, add Nova model-family eligibility in cache-retention resolution, pass or translate `cacheRetention` for Nova without injecting Anthropic `cachePoint` metadata, preserve the #20815 crash fix with regression tests, and update the public prompt-caching docs once the supported behavior is proven.
