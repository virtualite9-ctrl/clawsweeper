## Summary

Keep open: current `main` does not implement this PR’s alias-resolution fix. `resolveSubagentSpawnModelSelection()` still returns bare alias strings unchanged, and the subagent spawn path can persist those raw values into child-session runtime model fields.

## Evidence

- **Spawn resolver still returns raw bare strings:** On `ec3dbd22`, `resolveSubagentSpawnModelSelection()` only normalizes/trims the override or configured value and falls back to defaults; it never consults `buildModelAliasIndex`, so aliases such as `opus`/`gpt` remain unresolved here.
  - file: [src/agents/model-selection.ts:251](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/src/agents/model-selection.ts#L251)
  - command: `nl -ba src/agents/model-selection.ts | sed -n '239,268p'`
  - sha: [ec3dbd22a4de](https://github.com/openclaw/openclaw/commit/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46)
- **Spawn path persists the raw resolvedModel into runtime fields:** `persistInitialChildSessionRuntimeModel()` splits `resolvedModel` on `/` and writes `model` plus optional `modelProvider` directly to the child session entry. A bare alias therefore gets persisted as a raw model string with no provider.
  - file: [src/agents/subagent-spawn.ts:204](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/src/agents/subagent-spawn.ts#L204)
  - command: `nl -ba src/agents/subagent-spawn.ts | sed -n '204,228p'`

- **Default selections can clear the resolved override first:** When the patched model matches the agent default, `applyModelOverrideToSessionEntry()` deletes `providerOverride` and `modelOverride`. That leaves the later runtime write as the surviving model identity for the child session.
  - file: [src/sessions/model-overrides.ts:24](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/src/sessions/model-overrides.ts#L24)
