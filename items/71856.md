## Summary

Keep open. Current main does not implement the requested TUI startup recent-session summary, and the PR is a focused core TUI change linked to #71719 rather than cleanup material. The branch still needs maintainer review because the provided PR diff fetches sessions after every `onConnected` callback, so the Greptile reconnection-duplication concern remains valid even after the helper function follow-up commit.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to revise the TUI startup implementation so it runs only on the first non-local connection after history load, uses the existing bounded `sessions.list` fields, handles failures best-effort, and adds targeted tests for formatting, current-session exclusion, and reconnect deduplication before merge.
