## Summary

Keep open. Current main still does not implement a TUI startup recent-session summary: the connection flow only refreshes agents, updates the header, loads history, and sets connected status, while loadHistory rebuilds the transcript view without a distinct recap block. The PR is the canonical implementation path for closed issue #71719, and current session-list APIs already expose derived titles and last-message previews, so the work is plausible core TUI work rather than cleanup material. It still needs implementation review because the branch adds the summary path on every connection/reconnection and lacks a clear regression test for the startup summary behavior.

## Best Possible Solution

Keep PR #71856 open as the canonical implementation discussion for #71719. The best path is to revise the TUI startup-summary implementation in `src/tui/tui.ts`, gate it to initial connection or otherwise avoid duplicate reconnect messages, clarify whether the summary should describe the current session or the most recent other session, and add focused tests around the formatting and connection-flow behavior using the existing TUI backend/listSessions seam.
