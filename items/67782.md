## Summary

Keep this PR open. Current main still performs delete-then-set for changed non-empty Telegram command menus, while this PR is a narrow follow-up that moves deletion to the explicit empty-menu path and adds targeted regression coverage.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land the narrow Telegram plugin change after rebase and targeted validation: call `setMyCommands` directly for changed non-empty menus, preserve `deleteMyCommands` for explicit empty-menu clearing, keep the new regression tests, and optionally add focused failure fallback if maintainers want to mitigate stale-menu behavior on `setMyCommands` failure.
