## Summary

Keep this PR open. Current main still performs delete-then-set for changed non-empty Telegram command menus, while this PR is a narrow implementation candidate that moves deletion to the explicit empty-menu path and adds targeted regression coverage.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to rebase and validate the narrow Telegram plugin change, land the direct setMyCommands path for changed non-empty menus, preserve deleteMyCommands for explicit empty-menu clearing inside syncTelegramMenuCommands, keep the regression tests, and decide whether a fallback is needed for setMyCommands failure after skipping delete.
