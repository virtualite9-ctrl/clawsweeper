## Summary

Keep #25883 open. Current main still does not provide the requested Redis/BullMQ-style distributed job orchestration or horizontal worker coordination; cron, command queues, and task tracking remain Gateway-local, in-process, local-file, or local-SQLite backed.

## Best Possible Solution

Keep #25883 open for maintainer architecture review. The best path is to decide whether OpenClaw should add a distributed scheduler/job backend abstraction, possibly plugin-capable, with worker claiming, shared queue/state, retry/backoff, dead-letter or failure state, cron support, and observability; otherwise document Redis/BullMQ-style orchestration as an external deployment pattern. Preserve the linked ScalyClaw prototype as prior art.
