## Summary

Keep PR #71857 open. Current main still rejects owner-prefixed ClawHub skill slugs before registry lookup, and the PR addresses a concrete user-facing CLI/gateway mismatch from #71767 rather than an obsolete, duplicate, or out-of-repo request.

## Best Possible Solution

Keep this PR open for maintainer review. The best code path is the narrow parser change proposed by #71857: accept optional `<owner>/<slug>` only for requested ClawHub skill install/update, validate both segments under the existing ASCII slug guard, strip the owner before registry/install-dir/lockfile use, preserve tracked legacy behavior, and keep focused regression tests plus the changelog entry.
