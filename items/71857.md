## Summary

Keep PR #71857 open. Current main still rejects owner-prefixed ClawHub skill slugs before any registry call, and I did not find an implemented or superseding fix on main. The PR addresses a concrete user-facing mismatch from #71767 rather than optional ClawHub/plugin work outside core.

## Best Possible Solution

Keep the PR open for maintainer review. The best product/code path is a narrow parser change in `src/agents/skills-clawhub.ts` that accepts optional `<owner>/<slug>` for requested ClawHub skill install/update, validates both segments under the existing ASCII slug guard, strips the owner before registry/install-dir/lockfile use, and keeps focused regression tests plus the changelog entry.
