## Summary

Keep this PR open. Current main still rejects `owner/slug` ClawHub skill inputs before registry lookup, while the live ClawHub API resolves the reported skill by bare slug and returns 404 for the owner-prefixed API path. The PR is a focused implementation candidate for the linked bug and is not implemented, duplicated, or superseded on main.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land a narrow shared ClawHub skill slug parser in `src/agents/skills-clawhub.ts` that accepts optional `<owner>/<slug>` for requested install/update inputs, validates both segments under the existing requested-slug rules, canonicalizes to the bare slug before registry/download/install-dir/lockfile use, preserves legacy tracked-slug update compatibility, and ships with focused regression tests and a changelog entry.
