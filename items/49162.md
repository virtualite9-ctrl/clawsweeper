## Summary

Keep #49162 open. Current main has adjacent reliability behavior, including hard-coded embedding query timeouts, hard-coded embedding retry loops, QMD search timeout config, active-memory cache TTL, and an embedding cache, but it does not implement the requested schema-backed `agents.defaults.memorySearch.query.timeoutMs`, `query.cacheTtlMs`, or `query.retry` controls. Ryce's related durability comment is useful context, but #49056 and #49233 are complementary and do not supersede this core config/runtime request.

## Best Possible Solution

Keep the issue open and implement it as a narrow core memory-search config enhancement: add schema/types/help/docs for `agents.defaults.memorySearch.query.timeoutMs`, `query.cacheTtlMs`, and `query.retry`; resolve them per agent; wire the memory_search path and builtin/QMD managers to honor timeout, query-result cache TTL, and policy-driven retry; and add tests for config validation, resolution, runtime timeout/degradation, cache reuse, and retry behavior.
