## Summary

Keep #69110 open. Current main still has the reported provenance gap: `message_sending` sees outbound content and delivery metadata but no actual runtime provider/model, while `llm_output` has provider/model on a separate fire-and-forget path. The related open PR #65149 is directly relevant and may be a good implementation direction, but it is unmerged and does not make the issue obsolete on current main.

## Best Possible Solution

Keep the issue open and coordinate it with #65149 or a narrower maintainer-owned follow-up. The durable fix should thread resolved runtime model/provider provenance from the agent run into the final delivery/hook layer, document the public hook contract, and add regression coverage showing fallback model identity is available before outbound delivery and can be used to enforce or annotate truthful model tags.
