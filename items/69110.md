## Summary

Keep #69110 open. Current main still does not pass resolved runtime provider/model identity into `message_sending` delivery hooks or enforce/verify model self-tags at the gateway boundary. The resolved model is known in the agent run path and is exposed to `llm_output`, diagnostics, session usage, and response-prefix machinery, but the outbound delivery hook event/context remains delivery-only. Related open PRs such as #65149 and #39761 may be useful coordination points, but no merged main implementation currently solves the reported trust-boundary problem.

## Best Possible Solution

Keep the issue open and use it as the canonical bug/trust-boundary report unless maintainers choose a narrower canonical PR. The likely core path is to thread resolved provider/model identity from the agent result through final reply assembly and outbound delivery, expose it in a documented plugin hook contract, and add regression tests proving a hook or gateway policy can detect, rewrite, block, or audit mismatched model tags across fallback scenarios.
