## Summary

Current main still implements an after-the-fact reminder transparency note, not the requested pre-send gate that forces either a revised answer or a concrete scheduled action before delivery. The newer plugin hook surface helps with policy hooks generally, but the auto-reply path still appends the note and the documented finalize hook does not expose the per-turn scheduling evidence needed to close this as already handled or ClawHub-only.

## Best Possible Solution

Keep this issue open. The best implementation is a focused pre-send gate in the agent reply/finalization path that reuses the existing commitment detector, successfulCronAdds, and session-cron coverage check, then allows one retry/revision before dispatch: either remove unsupported commitment language or create a concrete cron/task/heartbeat-backed action. If maintainers prefer a plugin-first policy, first expose the needed per-turn scheduling evidence and session-cron coverage through a documented reply/finalization hook.
