## Summary

Current main still implements the existing post-generation transparency note, not the requested pre-send gate. The detector, reply pipeline, and tests all preserve the hollow commitment text and append the disclaimer when no cron.add succeeded. The existing finalize hook can request one more model pass, but the documented/type-level event does not expose the per-turn scheduling evidence needed to implement this policy cleanly as an external plugin today.

## Best Possible Solution

Keep this issue open. The best implementation is a focused gate in the agent finalization or reply pipeline that reuses the existing reminder-commitment detector, `successfulCronAdds`, and session-cron checks, then requests one revision before dispatch: either remove the unsupported commitment language or create a concrete scheduled action. If maintainers prefer plugin-first policy, first expose documented per-turn scheduling/task/heartbeat evidence through `before_agent_finalize` or another narrow hook contract.
