## Summary

Current main still implements the existing post-generation transparency note, not the requested pre-send gate that forces the agent to either revise the commitment away or create a scheduled action before delivery.

## Best Possible Solution

Keep this issue open. The best path is a focused implementation in the agent finalization or reply pipeline that reuses the existing reminder-commitment detector plus `successfulCronAdds` and existing session-cron checks, then requests one revision before dispatch: remove the unsupported commitment language or create a concrete scheduled action. If maintainers prefer this as plugin policy, first expose the missing per-turn scheduling/task/heartbeat evidence through a documented hook contract.
