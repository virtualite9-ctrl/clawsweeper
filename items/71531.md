## Summary

Keep open. Current `main` still does not expose a Slack-specific Bolt `logLevel`/`logger` config surface, and the bundled Slack plugin still constructs Bolt `App` instances without forwarding either option. This is a new Slack plugin config feature request, not an already-implemented fix, and it is neither stale nor out of scope.

## Evidence

- **Slack config type has no log-level field:** `SlackAccountConfig` lists Slack channel/account options such as `mode`, `signingSecret`, `commands`, `streaming`, `healthMonitor`, and `typingReaction`, but no `logLevel` or `logger` property exists in the public config contract.
