## Summary

Current main still has the reported sandbox write/edit mismatch. The sandboxed write and edit tools route through the sandbox filesystem bridge, the bridge still runs the pinned mutation helper, and that helper still exits with `sandbox pinned mutation helper requires python3 or python` when Python is absent. Default Docker sandbox provisioning can still create `openclaw-sandbox:bookworm-slim` by pulling and tagging plain `debian:bookworm-slim`, while the repo Dockerfile/script path that installs Python is separate. SimSef's comment positively corroborates the root cause and workaround: a custom sandbox image with `python3` makes `functions.write` and `functions.edit` work.

## Best Possible Solution

Keep this issue open and fix the runtime contract. The narrow product fix is to ensure the default Docker sandbox image creation and doctor repair paths always provide the helper dependency, including stale or auto-tagged default images that lack Python, with regression coverage for the missing-image provisioning path. A broader fix could remove the Python dependency from the pinned mutation helper or improve the error to name the helper runtime/image and PATH so users can diagnose the mismatch directly.
