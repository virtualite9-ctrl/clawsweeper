## Summary

Keep this issue open. Current main still has the reported product mismatch: sandboxed write/edit tool mutation paths require Python in the helper runtime, while default Docker runtime provisioning can still auto-tag bare debian:bookworm-slim as openclaw-sandbox:bookworm-slim. Shell writes can therefore succeed in a writable workspace while functions.write fails through the Python-dependent pinned helper.

## Best Possible Solution

Keep this issue open and fix the runtime contract. The narrow fix is to stop auto-tagging bare debian:bookworm-slim as the default sandbox image, ensure default image creation and doctor repair produce an image with python3, detect or repair stale default images that lack the helper dependency, and add regression coverage for missing/default-image provisioning. A broader fix could remove the Python dependency from the pinned mutation helper or make the failure message explicitly identify the helper runtime and image.
