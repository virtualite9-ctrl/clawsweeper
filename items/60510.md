## Summary

Keep open. Current `main` still emits `warn` for pre-handshake WebSocket closes unless the user agent matches `swiftpm-testing-helper`, and the probe path still uses `GATEWAY_CLIENT_MODES.PROBE` plus `client.stop()` during settlement, so the reported internal-probe false positive is not implemented away on `main`.

## Evidence

- **Pre-connect close logging still only downgrades SwiftPM helper traffic:** `attachGatewayWsConnectionHandler` defines `isNoisySwiftPmHelperClose()` and uses it as the only branch that changes the log level for `closed before connect`; otherwise the message is still logged with `logWsControl.warn`. There is no internal-probe helper or loopback+origin+1000 filter in current `main`.
  - file: [src/gateway/server/ws-connection.ts:294](https://github.com/openclaw/openclaw/blob/3ffd944e6ba6dfe4c7c4c275848f1c0a1f246e28/src/gateway/server/ws-connection.ts#L294)

  - file: [src/gateway/probe.ts:170](https://github.com/openclaw/openclaw/blob/3ffd944e6ba6dfe4c7c4c275848f1c0a1f246e28/src/gateway/probe.ts#L170)
