## Summary

Keep #13613 open. Current main still has gateway-local cron scheduling with local config/schema/status, a process-local lock chain, and no distributed leader/lock-provider seam. The existing steipete Codex review comment already kept this open and correctly treats related #13582 as separate local scheduler reliability work rather than a canonical HA coordination tracker.

## Best Possible Solution

Keep #13613 open until OpenClaw implements explicit HA cron coordination in the cron scheduler or adds a documented scheduler/lock-provider plugin API. The fix should cover config/schema/docs, due-job ownership semantics, leader/lock health visibility in status or CLI, and multi-process or multi-node regression coverage. Treat #13582 as separate shipped/local scheduler reliability work, not as the canonical tracker for this HA request.
