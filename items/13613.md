## Summary

Keep #13613 open. Current main still has gateway-local cron execution with local/shared-file runtime state and an in-process lock chain, but no distributed coordination config, Redis/etcd/DynamoDB backend, leader election, leader/status visibility, or public scheduler lock-provider plugin seam. The existing maintainer/Codex issue comment also kept this open, and the related #13582 is a separate closed local scheduler reliability bug rather than a canonical HA coordination tracker.

## Best Possible Solution

Keep #13613 open until maintainers either implement explicit HA cron coordination in core, or first add a documented scheduler/lock-provider plugin API that lets ClawHub backends provide Redis/etcd/DynamoDB-style coordination safely. The implementation should include config/schema/docs, health or CLI visibility for leadership/lock state, and multi-process or multi-node regression coverage. Treat #13582 as a separate closed local scheduler reliability bug, not the canonical tracker for this HA request.
