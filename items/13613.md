## Summary

Keep #13613 open. Current main still treats cron as a Gateway-local scheduler: there is no cron coordination config/schema, no distributed lock or leader-election backend, no leader/lock health or CLI surface, and the plugin hook surface only supports cron inspection and mutation rather than due-job ownership. The existing April 26 Codex review already kept this open, and the relevant cron/config/gateway/docs/plugin paths have no diff between that review SHA and current main.

## Best Possible Solution

Keep #13613 open until OpenClaw either implements distributed cron ownership in the core scheduler or adds a documented scheduler/lock-provider extension API that can support it. A complete fix should define config/schema/docs, due-job ownership semantics across Gateways, leader or lock health in status/CLI, and multi-process or multi-node regression coverage. Treat #13582 as shipped local scheduler reliability work, not the canonical tracker for HA coordination.
