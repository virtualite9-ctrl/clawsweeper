## Summary

PR #63668 adds an optional third-party MrScraper web-fetch/scraping provider as a bundled plugin, but current OpenClaw direction and docs put this kind of commercial service integration on the external plugin/ClawHub path. Current main already exposes the needed plugin seams for web-fetch providers, agent tools, skills, config schema, env metadata, and local/npm/ClawHub installation, so there is no missing core API that requires bundling this in `extensions/`.

## Best Possible Solution

Close this core PR and have MrScraper publish and maintain the plugin through ClawHub or npm using the existing OpenClaw plugin SDK. Keep any truly missing generic plugin API or setup gap as a separate focused OpenClaw issue/PR, independent of bundling MrScraper.
