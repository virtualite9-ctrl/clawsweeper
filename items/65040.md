## Summary

Keep this PR open. Current main still has exclusive `plugins.allow` behavior, and the four user-facing advisory/error surfaces targeted by the PR still do not consistently warn that a non-empty allowlist disables unlisted plugins. The PR is stale and needs a rebase because the security audit implementation moved, but the text fix is still useful and not superseded by current main.

## Best Possible Solution

Ask the contributor to rebase onto current main, move the audit remediation edit to `src/security/audit-plugins-trust.ts`, keep the safer no-generated-allowlist approach from the discussion, and add targeted tests for the loader warning, security audit remediation, CLI missing-plugin messages, and config validation warning.
