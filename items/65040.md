## Summary

Keep this PR open. Current main still has the exclusive `plugins.allow` behavior and some adjacent mitigations/docs, but the live advisory/error surfaces named by the PR still do not consistently warn users that setting `plugins.allow` disables unlisted plugins. The submitted diff is stale where security audit code moved from `audit-extra.async.ts` to `audit-plugins-trust.ts`, but the work is still a useful implementation candidate rather than obsolete.

## Best Possible Solution

Keep this PR open as a useful focused text fix. Ask for a rebase onto current main, move the audit remediation change to `src/security/audit-plugins-trust.ts`, keep the safer explanatory wording that avoids generated allowlist suggestions, and add targeted assertions for the loader warning, security audit remediation, CLI missing-plugin messages, and config validation warning.
