## Summary

Current main implements the requested non-silent fallback for subagent completion announces. Completion-mode direct announce failures now fall back to direct channel `sendMessage` delivery using the captured child result, and Telegram outbound delivery is chunked under the platform message limit. The fix is on main in commit a1b656705992e31df99147c786217fcc517c68b3; I found no release tag containing that commit, so fixedRelease is null.

## Best Possible Solution

Keep the shipped mainline fallback: failed subagent completion announce-agent delivery should fall back to direct outbound channel delivery, relying on each channel adapter's chunking or formatting limits. No separate OpenClaw core issue is needed unless a future report shows a channel adapter that cannot chunk or otherwise deliver the fallback text.
