## Summary

The report matches an older 2026.3.x regression, but current `main` already has explicit Feishu channel-plugin outbound/message-tool support, associated tests, and the same relevant code is present in shipped release `v2026.4.23` with no diff versus `HEAD` on those files.

## Evidence

- **Feishu message-tool discovery is implemented:** `describeFeishuMessageTool` advertises Feishu outbound actions including `send` and `thread-reply` for configured accounts.
  - file: [extensions/feishu/src/channel.ts:182](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/extensions/feishu/src/channel.ts#L182)
  - sha: [ec3dbd22a4de](https://github.com/openclaw/openclaw/commit/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46)
- **Feishu action handler performs outbound sends:** The Feishu channel plugin registers `describeMessageTool` and `handleAction`; the `send`/`thread-reply` branch calls Feishu runtime send helpers instead of falling through as an unknown channel.
  - file: [extensions/feishu/src/channel.ts:671](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/extensions/feishu/src/channel.ts#L671)
  - sha: [ec3dbd22a4de](https://github.com/openclaw/openclaw/commit/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46)
- **Unknown-channel check now goes through channel plugin resolution:** Agent delivery resolves `deliveryChannel` through `getChannelPlugin(normalizeChannelId(...))` before the `Unknown channel` error path, so bundled channels like Feishu are recognized when their plugin is present.
  - file: [src/agents/command/delivery.ts:223](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/src/agents/command/delivery.ts#L223)
  - sha: [ec3dbd22a4de](https://github.com/openclaw/openclaw/commit/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46)
- **Tests cover Feishu send surface:** The Feishu channel tests assert that the advertised action surface includes `send` and also verify that a `send` action calls `sendMessageFeishu(...)` successfully.
  - file: [extensions/feishu/src/channel.test.ts:196](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/extensions/feishu/src/channel.test.ts#L196)
  - sha: [ec3dbd22a4de](https://github.com/openclaw/openclaw/commit/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46)
- **Shipped release matches current main on relevant files:** Comparing `v2026.4.23` to `HEAD` shows no diff in the Feishu channel plugin and delivery-path files relevant to this report, so the implemented behavior is already shipped.
  - command: `git diff --stat v2026.4.23..HEAD -- extensions/feishu/index.ts extensions/feishu/channel-plugin-api.ts extensions/feishu/src/channel.ts src/agents/command/delivery.ts src/channels/plugins/message-tool-api.ts`
  - sha: [a9797214338b](https://github.com/openclaw/openclaw/commit/a9797214338ba31c52c796adbb75afb16e0684a9)
- **Issue discussion points at older affected builds:** The issue body reports `2026.3.23-2`, and the later follow-up comment reports the same behavior on `2026.3.24`, which is consistent with an older regression rather than the current `main` codepath.

## Risks / Open Questions

- `docs/cli/message.md` still omits Feishu from the generic channel list, which could confuse users even though the current Feishu channel implementation exists on `main` and in `v2026.4.23`.
