## Summary

Keep open. Current main still does not implement `sessions_delegate` or `sessions_delegate_batch`, and the existing `sessions_spawn` and `sessions_send` tools are not equivalent to a fresh synchronous spawn-and-wait delegation primitive. This PR remains a plausible implementation candidate, but it needs maintainer product/API judgment and cleanup for sanitizer coverage plus output-capture/cleanup ordering before merge.

## Best Possible Solution

Keep this PR open as the implementation candidate for synchronous subagent delegation. Maintainers should decide the product/API direction, rebase it onto current main, add sanitizer coverage for delegate attachments, fix output-capture versus cleanup ordering including finally-release behavior, and land focused tests plus docs/display/policy updates if the new core tool surface is accepted.
