## Summary

Keep this PR open. Current main still does not implement the central requested tools, `sessions_delegate` and `sessions_delegate_batch`; the existing session tools remain `sessions_spawn` for non-blocking fresh child work and `sessions_send` for waiting on an existing visible session. The PR is still a plausible implementation candidate, but the provided review discussion shows unresolved correctness/security risks that should be fixed before merge.

## Best Possible Solution

Keep this PR open as the implementation candidate for synchronous subagent delegation. Maintainers should first decide whether `sessions_delegate` and `sessions_delegate_batch` belong in the core agent tool surface, then require fixes for the remaining redaction and output-capture/cleanup review findings, update focused tests, and align the session-tool docs plus tool policy/display surfaces before merge.
