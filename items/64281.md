## Summary

Keep this PR open. Current main still lacks `sessions_delegate` and `sessions_delegate_batch`, and the existing `sessions_spawn` plus `sessions_send` tools do not provide the same fresh synchronous spawn-and-wait primitive. The PR remains a plausible implementation candidate, but it needs maintainer product/API review and security cleanup before merge.

## Best Possible Solution

Keep this PR open as the implementation candidate for synchronous subagent delegation. Maintainers should decide the product/API direction, rebase it onto current main, keep the diff focused on the missing delegate tools, add sanitizer coverage for delegate attachments, harden output-capture versus cleanup ordering, and land focused tests plus docs/display/policy updates if the new core tool surface is accepted.
