## Summary

Keep this PR open. Current main still lacks the central requested `sessions_delegate` and `sessions_delegate_batch` tools, and existing `sessions_spawn`/`sessions_send` behavior is not equivalent. The PR remains a plausible implementation candidate, but it needs maintainer product judgment plus fixes for the remaining security and output-capture/cleanup review risks before merge.

## Best Possible Solution

Keep this PR open as the implementation candidate for synchronous subagent delegation. Maintainers should decide whether these tools belong in the core agent surface, then require fixes for delegate attachment redaction and output-capture/cleanup ordering, rebase around the already-shipped `prepareSubagentSpawn` hook, and update focused tests plus session-tool docs/display/policy surfaces before merge.
