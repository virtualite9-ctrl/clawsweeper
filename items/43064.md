## Summary

Keep this issue open. Current main still appears to use the raw heartbeat `responsePrefix` without the model-selection callback/context needed for `{model}` interpolation, and there is an open same-author PR with closing syntax that is the proper implementation candidate.

## Best Possible Solution

Keep this issue open and review the linked implementation PRs, especially #43065 as the same-author candidate. The desired fix belongs in `src/infra/heartbeat-runner.ts`: reuse the shared reply-prefix context path, pass `onModelSelected` into `getReplyFromConfig()`, resolve the prefix after model selection, and add focused heartbeat regression coverage for templated prefixes and `HEARTBEAT_OK` stripping.
