## Summary

Keep #38683 open. Current main still enforces Telegram `dmPolicy="open"` as an explicit config contract requiring `allowFrom` to include `"*"`; the shipped repair paths are `openclaw doctor --fix` and setup/config-write helpers, not startup/config-load auto-heal. The PR remains a focused, security-sensitive product decision rather than obsolete work.

## Best Possible Solution

Keep the PR open for explicit maintainer product judgment. Maintainers should decide whether to preserve the current explicit `dmPolicy="open"` plus `allowFrom: ["*"]` contract and rely on `openclaw doctor --fix`/setup helpers, or accept a narrow startup/config-load normalization that patches only the intended Telegram account and includes the missing tests, docs, and changelog.

## Risks / Open Questions

- If maintainers accept startup auto-heal, the PR still needs maintainer product approval, safe account-scoped normalization, warning/test isolation, missing/empty/existing-without-wildcard regression coverage, docs, and changelog.
