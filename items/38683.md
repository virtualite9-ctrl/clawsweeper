## Summary

Keep #38683 open. Current main still enforces Telegram `dmPolicy="open"` as an explicit config contract requiring `allowFrom` to include `"*"`; the shipped repair path is `openclaw doctor --fix` and setup/config-write guidance, not startup/config-load auto-heal. The PR is a focused, actionable product decision about whether to relax that cold-start failure mode, and its discussion includes a security-sensitive account-inheritance bug that the author addressed in follow-up commit `792311545edfe18520724a6a448f248f3c0155ee`.

## Best Possible Solution

Keep the PR open for explicit maintainer product judgment. Decide whether Telegram should preserve the current explicit `dmPolicy="open"` plus `allowFrom: ["*"]` validation contract and rely on `openclaw doctor --fix`/setup paths, or accept a narrowly scoped startup/config-load normalization that patches only the intended Telegram account, avoids shared warning state, includes missing/empty/existing-without-wildcard tests, and updates docs/changelog.
