## Summary

Current `main` still rejects Telegram `dmPolicy: "open"` unless the effective `allowFrom` contains `"*"`, and the docs still describe that as required explicit opt-in. `main` does now have a `doctor --fix` repair path that adds the wildcard, so the PR is partially superseded, but it does not remove the cold-start validation failure this PR was opened to prevent. That is not strong enough for an auto-close as implemented.
