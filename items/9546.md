## Summary

Current main still does not implement the requested WebChat setting. The New messages indicator is derived directly from scroll state, rendered whenever that state is true, and there is no persisted UI or gateway.webchat option to disable, dismiss, customize, or replace it with forced autoscroll.

## Best Possible Solution

Keep #9546 open. A narrow implementation should add a persisted WebChat new-message indicator mode, feed it into scheduleChatScroll and renderChat, and update focused UI tests plus public WebChat/config docs such as https://docs.openclaw.ai/web/webchat.
