## Summary

Keep #9546 open. Current main still has no WebChat setting to disable, customize, or auto-scroll instead of showing the New messages indicator. The prior issue comment keeping this open remains accurate against current HEAD: the indicator is still driven directly by scroll state, rendered whenever that state is true, and neither persisted UI settings nor `gateway.webchat` config/docs expose a notification mode.

## Best Possible Solution

Keep #9546 open. A narrow fix should add a persisted WebChat/Control UI preference or documented `gateway.webchat` option for the new-message indicator mode, gate both `scheduleChatScroll` behavior and `renderChat` display through that setting, and update focused UI tests plus public WebChat/config docs.
