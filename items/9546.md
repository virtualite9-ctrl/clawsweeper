## Summary

Current main still lacks the requested WebChat preference. The UI still derives the visible `New messages` control from scroll state, renders it directly when `showNewMessages` is true, and exposes no persisted UI setting or `gateway.webchat` config option to disable, customize, or replace it with forced autoscroll.

## Best Possible Solution

Keep #9546 open. A narrow fix should add a persisted WebChat/Control UI preference or documented `gateway.webchat` option for the new-message indicator mode, gate both scroll-state behavior and chat rendering through that setting, and update focused UI tests plus the public WebChat/config docs.
