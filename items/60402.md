## Summary

Keep open. Current `main` still has the heartbeat-disable behavior this PR is trying to fix: session-targeted ACP/event wakes short-circuit when global heartbeats or the interval schedule are disabled, and the scheduler still re-arms after disabled skips instead of fully pausing.

## Evidence

- **`runHeartbeatOnce` still hard-blocks disabled heartbeat schedules:** On `main`, `runHeartbeatOnce` returns `skipped/disabled` immediately when `areHeartbeatsEnabled()` is false, and again when `resolveHeartbeatIntervalMs(...)` is falsy. There is no `allowEventWakeWithoutHeartbeat` or similar event-scoped bypass, so targeted `exec-event`/`wake`/`cron`/`hook` runs still short-circuit under disabled heartbeat scheduling.
  - file: [src/infra/heartbeat-runner.ts:728](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/src/infra/heartbeat-runner.ts#L728)
  - command: `nl -ba src/infra/heartbeat-runner.ts | sed -n '728,752p'`
  - sha: [926068b14fc6](https://github.com/openclaw/openclaw/commit/926068b14fc661e915a0247e9ecb5949fb95b4f7)
- **Runner still rejects targeted wakes before targeted dispatch:** `startHeartbeatRunner` returns `disabled` before it reaches the targeted `requestedSessionKey || requestedAgentId` branch whenever global heartbeats are off or no heartbeat-enabled agents are scheduled. That means targeted ACP parent wakes still never reach `runOnce` in the disabled state.
  - file: [src/infra/heartbeat-runner.ts:1456](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/src/infra/heartbeat-runner.ts#L1456)
  - command: `nl -ba src/infra/heartbeat-runner.ts | sed -n '1450,1510p'`
  - sha: [926068b14fc6](https://github.com/openclaw/openclaw/commit/926068b14fc661e915a0247e9ecb5949fb95b4f7)
- **Disabled state still re-arms interval timers:** The runner's `finally` block still calls `scheduleNext()` for every non-`requests-in-flight` exit, including disabled skips. That matches the P2 review concern in the PR discussion and conflicts with the docs' stated `disable: pause heartbeats` behavior.
  - file: [src/infra/heartbeat-runner.ts:1553](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/src/infra/heartbeat-runner.ts#L1553)

- **Docs still define disable as a pause:** The public CLI docs say `system heartbeat disable` should `pause heartbeats`, so the current re-arm-on-disabled behavior is not just an internal refactor gap; it is user-visible contract drift.
  - file: [docs/cli/system.md:45](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/docs/cli/system.md#L45)
