## Summary

Keep PR #60402 open. Current main still blocks heartbeat runs before targeted ACP/session wake handling when heartbeats are globally disabled, while ACP parent streaming still relies on queued system events plus `requestHeartbeatNow`. The PR is a focused, still-relevant fix for #52249 and its review feedback appears to have been addressed in the PR discussion rather than superseded on main.

## Best Possible Solution

Keep #60402 open as a plausible targeted fix. The best path is to rebase or refresh it against current main, preserve the disabled-agent guard and disabled-timer regressions from the PR discussion, and land it only after current-main tests cover targeted ACP/session wakes when periodic heartbeat scheduling is disabled. Close it only if an equivalent merged change fixes #52249 and includes comparable regression coverage.
