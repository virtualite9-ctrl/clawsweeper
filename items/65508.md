## Summary

Current main already implements the requested user-facing tool XML stripping through the shared assistant-visible sanitizer used by Pi embedded reply extraction and history/session surfaces. The open PR is obsolete for the core sanitizer request and has grown unrelated package, QA, workflow, and extension-test churn while chasing CI/rebase fallout.

## Best Possible Solution

Close this PR as obsolete and keep the shipped shared assistant-visible sanitizer as the source of truth. Any remaining disagreement about preserving literal `<tool_call>` examples should be split into a focused sanitizer follow-up with targeted tests and no unrelated CI/release/QA churn.
