## Summary

Keep #19482 open. Current main still recreates `HEARTBEAT.md` whenever it is missing, including existing workspaces where the user deliberately deleted it. The PR addresses a real remaining behavior gap, though its negative-path test should be tightened to model a fully initialized workspace with only `HEARTBEAT.md` absent.

## Best Possible Solution

Keep the PR open and either land it after tightening the negative regression test, or implement the same behavior on main: only seed `HEARTBEAT.md` for brand-new workspaces, while preserving missing-file heartbeat semantics for existing workspaces. The best test should preseed the other canonical workspace files and leave only `HEARTBEAT.md` absent.

Because `writeFileIfMissing` uses `flag: "wx"`, any existing workspace missing `HEARTBEAT.md` will get the template recreated on startup.
