## Summary

Keep #19482 open. Current main still unconditionally recreates HEARTBEAT.md during workspace bootstrap, while heartbeat runtime/docs/tests preserve a meaningful difference between a missing file and an empty template file. The PR is still a focused fix for a live behavior gap, though its negative regression test should be tightened to model a fully initialized workspace with only HEARTBEAT.md removed.

## Best Possible Solution

Keep the PR open and land it after tightening the negative regression fixture, or apply the same code change directly on main: only seed HEARTBEAT.md for brand-new workspaces while preserving missing-file heartbeat semantics for existing workspaces.
