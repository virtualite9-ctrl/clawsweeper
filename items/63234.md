## Summary

Keep open. Current main does not contain the loader-level invalid-config log dedupe that would resolve #63234. The issue was referenced by a dedicated fix commit, but that commit is not reachable from the checked-out main and the current loader still logs each plugin config validation failure directly in both runtime and CLI-metadata loader paths. Existing related config-reload dedupe work does not cover this plugin-loader log path.

## Best Possible Solution

Keep #63234 open until a loader-level fix lands on main. The best implementation is the narrow approach already prototyped by commit `4cbb5897de2c7a7a755351ced554caddfcc9b4c9`: dedupe identical plugin invalid-config log lines per process/error fingerprint while preserving plugin diagnostics and registry error state, and add the regression test that repeated loader calls produce only one visible invalid-config log. If maintainers prefer broader graceful degradation, link this issue to the plugin-scoped config degradation work, but #61683 does not currently replace this specific log-spam fix.
