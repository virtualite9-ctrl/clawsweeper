## Summary

Keep this PR open. Current main still lacks an interactive metadata step for custom provider setup: the wizard collects endpoint/auth/model identity data, then writes hardcoded model metadata defaults. The PR addresses a real remaining UX gap and is related to open #48338, so it should be refreshed and reviewed rather than closed as stale or obsolete.

## Best Possible Solution

Keep the PR open as a plausible implementation candidate. Best path is to rebase it onto current main, re-run the command/configure tests, resolve any remaining review comments, and decide whether the metadata prompt should fully cover the current model metadata surface or remain a focused context-window/max-token setup aid. If landed, link it as a partial or direct fix for the custom-provider UX tracked in #48338.
