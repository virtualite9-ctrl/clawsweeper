## Summary

Keep this PR open. Current main still lacks the interactive custom-provider metadata step proposed here: `openclaw configure --section model` reaches the custom provider setup flow, but that flow only collects endpoint/auth/model identity and writes default metadata. The related custom-provider context-window UX issue #48338 is still open, so this PR remains a plausible implementation candidate rather than conservative cleanup.

## Best Possible Solution

Keep this PR open as a plausible implementation path for custom-provider metadata setup UX. The best path is to rebase it onto current main, align the prompt with provider-level metadata, `contextTokens`, Azure defaults, and the current input-modality surface, then either land it with targeted configure/model-metadata tests or supersede it with an equivalent maintainer-owned fix for #48338.
