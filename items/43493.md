## Summary

Keep this PR open. Current main still has the custom provider configure flow, default metadata writes, and config/schema support for contextWindow/contextTokens/maxTokens, but it does not provide the interactive custom-provider metadata step proposed here. The related custom-provider context-window UX issue #48338 remains open, so this PR is still a plausible implementation candidate rather than conservative cleanup.

## Best Possible Solution

Keep this PR open as a plausible implementation path. The best maintainer path is to rebase it onto current main, align the wizard with current custom-provider defaults and metadata fields (`contextWindow`, `contextTokens`, `maxTokens`, reasoning, input modalities, cost, provider-level defaults), keep the prompt limited to custom provider setup, and land it with focused configure/model-metadata tests. If maintainers prefer a different UX, supersede it with an equivalent fix for #48338 rather than closing the remaining user problem.
