## Summary

Keep this PR open. Current main still has the steer-mode gap: steer injection is gated on streaming, non-streaming active runs reject queued steer input, and the fallback is still a queued followup. The PR is a focused implementation candidate for open #10960, but it needs rebase/conflict resolution before maintainer review.

## Best Possible Solution

Keep this PR open as the implementation candidate for #10960. Rebase it onto current main, preserve the non-heartbeat abort guard, adapt imports/tests to the current pi-embedded-runner paths, update queue docs if the abort-to-unblock behavior is accepted, and rerun the targeted auto-reply tests before maintainer review.
