## Summary

Keep this PR open. Current main still gates steer injection on `isStreaming`, active non-streaming runs still fall back to followup instead of being aborted/unblocked, and the linked soft-steer request #10960 remains open. The PR is a plausible implementation candidate, but its current shape has a real heartbeat regression risk because the abort fallback runs before active-run queue policy can drop heartbeat turns.

## Best Possible Solution

Keep this PR open as an implementation candidate. Revise it so active non-streaming `steer` user messages abort/unblock the current run only when queue policy would actually enqueue the followup, preserve the streaming injection fast path, keep heartbeat turns from aborting active runs, clean up the misleading steer-backlog test name and unused `isStreaming` contract noted in review, then rebase and run the targeted changed tests.
