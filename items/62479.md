## Summary

Close PR #62479 as ClawHub/plugin-scope cleanup. The PR adds a standalone Python RBAC proxy package under `packages/openclaw-guard/`, while current OpenClaw scope and docs direct optional integrations to external plugins/ClawHub. Current main already has trusted-proxy identity, operator scopes, plugin HTTP routes, CLI hooks, and service registration surfaces for this class of guard/proxy work without adding a one-off core package.

## Best Possible Solution

Do not merge the standalone Python proxy into OpenClaw core. Publish or iterate it as an external ClawHub/npm plugin or sidecar in a separate repository, using the existing trusted-proxy/operator-scope and plugin route/service/CLI surfaces; keep any remaining native multi-user/RBAC requirements tracked in #8081 for explicit maintainer design.
