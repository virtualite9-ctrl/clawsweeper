## Summary

Keep this PR open. Current main still lacks the requested Slack `file_shared` Socket Mode fallback, setup manifest entry, docs entry, and tests. The PR is stale against the removed `src/slack` layout, but the central Slack plugin behavior remains a valid implementation candidate rather than something implemented, superseded, or safely closeable.

## Best Possible Solution

Keep this PR open as a useful implementation candidate, but ask for a rebase/port into `extensions/slack`. The best path is a Slack plugin `file_shared` handler under `extensions/slack/src/monitor/events`, registration from `extensions/slack/src/monitor/events.ts`, setup/docs updates for the Slack bot event subscription, and focused tests for large-file fallback, multi-file behavior, normal-message dedupe, app_mention interaction, busy-channel lookup, and authorization/security gates. Reuse the existing attachment/media pipeline instead of carrying the old override-note approach.

## Risks / Open Questions

- The PR cannot merge as-is because it targets the old `src/slack` layout instead of `extensions/slack`.
- The proposed fallback still needs a more robust, anchored history lookup than a broad recent-message window, plus tests for busy-channel misses and duplicate delivery.
- A correct implementation must dedupe against both normal `message`/`file_share` delivery and duplicate `file_shared` retries without dropping legitimate multi-file uploads.
- The port needs setup manifest, public Slack docs, and focused tests; the current PR diff does not include those surfaces.
- Security-sensitive review item: the fallback must keep current Slack account/team mismatch checks, DM/group/channel authorization, media download SSRF policy, and avoid logging full Slack event bodies or tokens.
