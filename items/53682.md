## Summary

Keep open. Current `main` still sends approval-gated gateway heredoc commands through `enforcedCommand`, so the reported behavior is not implemented on `main`, and the proposed one-line PR change would also affect the `askFallback === "allowlist"` timeout path that currently auto-approves allowlist-satisfied commands after timeout.

## Evidence

- **Current main still requires heredoc approval but executes the rebuilt command:** `processGatewayAllowlist()` marks allowlist-satisfied heredoc commands as requiring explicit approval, then later executes `runExecProcess({ ..., execCommand: enforcedCommand, ... })` in the approval follow-up path instead of the original command text.
  - file: [src/agents/bash-tools.exec-host-gateway.ts:164](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/agents/bash-tools.exec-host-gateway.ts#L164)
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Approved follow-up still uses enforcedCommand on main:** The background approval execution path on current `main` still passes `execCommand: enforcedCommand` at the point where the PR proposed swapping in the original heredoc command.
  - file: [src/agents/bash-tools.exec-host-gateway.ts:375](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/agents/bash-tools.exec-host-gateway.ts#L375)
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Timeout fallback reaches the same execution branch:** If approval times out and `askFallback === "allowlist"`, current code sets `approvedByAsk = true` for allowlist-satisfied commands before flowing into the same `runExecProcess(...)` call. A blanket switch to `params.command` for every heredoc approval would therefore also change timeout-fallback behavior.
  - file: [src/agents/bash-tools.exec-host-gateway.ts:316](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/agents/bash-tools.exec-host-gateway.ts#L316)
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Base approval state treats missing decisions as timed-out fallback:** `resolveBaseExecApprovalDecision()` returns `timedOut: true` when no approval arrives; combined with the gateway logic above, that is the mechanism behind the `askFallback === "allowlist"` auto-approval path called out in review.
  - file: [src/agents/bash-tools.exec-host-shared.ts:180](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/agents/bash-tools.exec-host-shared.ts#L180)
