## Summary

Keep this PR open. Current main still does not include the PR's gateway model-catalog context-window cache population, and the synchronous session-listing path still avoids async discovery, so the proposed fix remains a plausible implementation candidate. The PR should be revised before merge because review discussion and current resolver safeguards show correctness risks around slash-containing model IDs, global cache state, and unconditional 200k fallback behavior.

## Best Possible Solution

Keep this PR open and request a targeted revision: resolve session-row context windows from precise catalog or persisted runtime data rather than an unconditional `DEFAULT_CONTEXT_TOKENS` fallback, avoid mixing raw slash model IDs with synthetic provider-qualified cache keys, reset any new context-cache side effects through test helpers, and keep regression coverage for Anthropic plus alias and slash-provider cases.
