## Summary

Keep #69030 open. Current main partially mitigates the reported context leak by running `infer model run` with `modelRun: true`, `promptMode: "none"`, and no tools, so the original AGENTS/SOUL/skills/tool-routing repro is likely improved. It still does not implement the requested raw mode: there is no `--raw` flag or `model.raw` capability, both local and gateway paths still route through the agent command/runtime, the run still resolves the default agent/main session and transcript file, and the model still receives a non-empty OpenClaw identity system prompt.

## Best Possible Solution

Keep the issue open and implement a real raw/stateless model path for `openclaw infer model run`, either as `--raw` or a new `model.raw` capability. The implementation should bypass `agentCommand`/gateway `agent`, avoid default-agent session/transcript writes, send an empty or explicitly documented system prompt, use the existing simple-completion provider/auth helpers where practical, return provider/model attribution, and add local/gateway CLI tests plus docs explaining how raw mode differs from the current agent-runtime model-run mode.
