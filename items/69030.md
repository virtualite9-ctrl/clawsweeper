## Summary

Keep #69030 open. Current main has a partial mitigation for `infer model run` by forcing `promptMode: "none"` and disabling tools, but it still has no `--raw` flag and still routes both local and gateway runs through the default agent runtime/session path. That leaves the requested raw/stateless benchmarking mode unresolved.

## Best Possible Solution

Keep the issue open and implement a true raw/stateless inference path: add an explicit `--raw` or `model.raw` surface, route it through the simple-completion/provider runtime instead of `agentCommand`/gateway `agent`, keep tools and agent prompt context empty, avoid the default agent session transcript/lock path, and add focused tests proving raw mode does not call the agent runtime while the existing agent-wrapped mode remains backward compatible.

## Risks / Open Questions

- A raw implementation needs to preserve provider/model auth resolution without inheriting default-agent prompt, tools, session history, or session locks.
- Maintainers should decide whether this is best exposed as `--raw` on `model.run` for compatibility or as a separate `model.raw` capability; either path should update local and gateway behavior, docs, and tests.
