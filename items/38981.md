## Summary

Keep PR #38981 open. Current main still does not implement `agents.defaults.workspaceConfig.allowedExternalPaths` or per-agent `workspaceConfig.allowedExternalPaths`, and the agent bootstrap plus gateway file APIs remain rooted to the workspace safety helpers. The PR is a concrete core agent/gateway trust-boundary change with useful review discussion and author follow-up, not a ClawHub-only item or obsolete duplicate.

## Best Possible Solution

Keep PR #38981 open for maintainer review. The best path is either to land a carefully reviewed core implementation covering config types, schemas/help/docs, workspace bootstrap reads, extra bootstrap files, gateway `agents.files.get/set`, and symlink/write regression tests, or to make an explicit maintainer product/security decision that external symlinked workspace context remains unsupported and document that boundary. This should stay in OpenClaw core because the missing behavior sits in the agent/gateway file safety layer, not in an external plugin surface.
