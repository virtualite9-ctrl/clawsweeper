## Summary

Keep open. Current main still has the runtime mutual-exclusion guard for `sessions_send`, but it does not include PR #63377's schema field descriptions or session-tool docs guidance for choosing exactly one of `sessionKey` and `label`. The PR remains a plausible, narrowly scoped docs/schema cleanup for issue #63308 rather than something implemented or safely auto-closeable.

## Best Possible Solution

Keep the PR open or consolidate it with the canonical #63308 follow-up. The best implementation is a tiny docs/schema update in `src/agents/tools/sessions-send-tool.ts` and `docs/concepts/session-tool.md` that explicitly tells models/users to provide exactly one of `sessionKey` or `label`, prefer `sessionKey` when known, and use `agentId` only as a label-resolution hint. A focused assertion for schema descriptions would make the behavior harder to regress, but the core runtime path already has coverage for target validation.
