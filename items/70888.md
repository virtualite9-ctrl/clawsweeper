## Summary

Current main still contains the no-cfg early return that this PR targets, and the announce path still classifies requester sessions through that no-cfg depth lookup. The PR is a concrete implementation candidate with relevant regression coverage, so it should not be closed as obsolete.

## Best Possible Solution

Keep this PR open as an active fix candidate. Best path is maintainer review of the targeted subagent-depth change and regression tests, with the unrelated Bedrock, QA-helper, and WhatsApp test changes either split out or explicitly accepted as CI unblockers before merge.
