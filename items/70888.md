## Summary

Keep open. Current main still skips persisted session-store lookup when `getSubagentDepthFromSessionStore()` is called without `cfg`, and announce classification still uses that no-`cfg` path. The PR is a plausible focused bug fix that is not implemented on main or superseded by local history.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land a narrow version of the `subagent-depth.ts` fix with regression tests for persisted explicit requester keys on the no-`cfg` path, while pruning or separately justifying unrelated drift-fix commits.
