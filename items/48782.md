## Summary

Current main has agent/workspace-scoped memory, multi-agent isolation, bootstrap context files, and an optional session-memory hook. Those mitigate multi-project workflows when each project is modeled as a separate agent/workspace, but they do not implement the issue's requested same-session project declaration, project switching confirmation, or project-organized MEMORY.md behavior.

## Best Possible Solution

Keep open for maintainer product/API triage. The practical path is either to document one agent/workspace per project as the supported workaround, or design a project-context feature/plugin that records the active project, injects project-specific context, and scopes memory search/write behavior with tests for switching projects in the same session.
