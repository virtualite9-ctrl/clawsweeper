## Summary

Current `main` does not implement this seam: `ProviderPrepareExtraParamsContext` still has no `sessionKey`/`sessionId`, the embedded-runner extra-params path does not thread those values into provider hooks, and the PR's LiteLLM example is incomplete because arbitrary `extraParams` keys are not automatically injected into the request body.

## Evidence

- **Provider hook context still lacks session identity:** `ProviderPrepareExtraParamsContext` on current `main` only exposes `config`, `agentDir`, `workspaceDir`, `provider`, `modelId`, `extraParams`, and `thinkingLevel`; there is no `sessionKey` or `sessionId` field.

  - sha: [c070509b7f8f](https://github.com/openclaw/openclaw/commit/c070509b7f8fdbe479e5d829643782c272e30e83)

  - file: [src/agents/pi-embedded-runner/extra-params.ts:175](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/src/agents/pi-embedded-runner/extra-params.ts#L175)
