## Summary

Keep #69054 open. Current main supports storing and routing `delivery.accountId`, but it does not implement `openclaw cron lint`, does not warn at cron scheduler startup for announce jobs missing `delivery.accountId`, and does not reject announce-mode cron add/update requests that target a multi-account channel without an explicit account id. The related #69055 is a companion logging request, not a duplicate of this lint/enforcement work.

## Best Possible Solution

Keep the issue open and implement a core cron lint/validation path: add a reusable cron delivery lint helper that can determine per-channel multi-account state through the channel/plugin account APIs, expose `openclaw cron lint --warn-only`, call it from gateway startup, and enforce it in cron add/update paths after a documented warning period. The companion #69055 should remain separate because it tracks delivery attribution logging, not config linting.
