## Summary

Keep open. Current `main` still lacks the Feishu reply-path wiring this PR is trying to add: local-path media sends require `mediaLocalRoots`, but the reply dispatcher does not accept/forward that field and the Feishu bot does not derive/pass agent-scoped roots into `createFeishuReplyDispatcher`.

## Evidence

- **`sendMediaFeishu` still requires local roots for local-path uploads:** The current Feishu media sender documents that post-CVE local file paths require `mediaLocalRoots`, and it only passes `localRoots` to `loadWebMedia` when that field is provided.
  - file: [extensions/feishu/src/media.ts:678](https://github.com/openclaw/openclaw/blob/89755d1c79678610720cdb9ce504cfd62c261f64/extensions/feishu/src/media.ts#L678)
  - command: `nl -ba extensions/feishu/src/media.ts | sed -n '678,750p'`
  - sha: [89755d1c7967](https://github.com/openclaw/openclaw/commit/89755d1c79678610720cdb9ce504cfd62c261f64)
- **Reply dispatcher on `main` does not accept or forward `mediaLocalRoots`:** `CreateFeishuReplyDispatcherParams` has no `mediaLocalRoots` field, and the `sendMediaFeishu` call in `sendMediaReplies()` omits it. That means reply-path media sends cannot satisfy the allowlist requirement above.
  - file: [extensions/feishu/src/reply-dispatcher.ts:99](https://github.com/openclaw/openclaw/blob/89755d1c79678610720cdb9ce504cfd62c261f64/extensions/feishu/src/reply-dispatcher.ts#L99)
  - command: `nl -ba extensions/feishu/src/reply-dispatcher.ts | sed -n '80,430p'`
  - sha: [89755d1c7967](https://github.com/openclaw/openclaw/commit/89755d1c79678610720cdb9ce504cfd62c261f64)
- **Feishu bot call sites on `main` do not derive/pass agent-scoped media roots:** Both `createFeishuReplyDispatcher(...)` call sites in `handleFeishuMessage` construct the dispatcher without `mediaLocalRoots`, and the file does not import `getAgentScopedMediaLocalRoots`. So even if the dispatcher accepted the field, current callers would still leave it undefined.
