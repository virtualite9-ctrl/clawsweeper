## Summary

Keep this PR open. Current main still lacks a Feishu `replyToMode` config surface and still sends normal group replies with a Feishu reply target, so the requested behavior control is not implemented on main. The PR is useful but should not be treated as merge-ready without addressing the remaining streaming/media `first`-mode review concern and adding tests/docs/metadata.

## Best Possible Solution

Keep the PR open and ask for a Feishu-owned implementation pass: add `replyToMode` to the Feishu schema, plugin manifest/config metadata, and docs; resolve precedence across group, account, and top-level config; apply the mode consistently to streaming cards, static cards, text chunks, and media; and add focused tests for `off`, `first`, and `all`, including a streaming-plus-media case. The existing shared reply-to policy helpers in `src/infra/outbound/reply-policy.ts` are a good reference for single-use behavior.
