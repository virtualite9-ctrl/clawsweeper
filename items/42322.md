## Summary

Keep this PR open. Current main still lacks a Feishu `replyToMode` config surface and still routes Feishu replies through a fixed reply target, so the requested behavior control is not obsolete. The PR is useful but not merge-ready: it needs a rebase onto the current Feishu reply-target/threading code, metadata/docs/test coverage, and a complete `first` policy for streaming plus media.

## Best Possible Solution

Keep this PR open and ask for a Feishu-owned implementation pass: rebase onto current main, add `replyToMode` to the strict schema plus manifest/generated metadata/docs, define precedence across group/account/top-level config, apply one reply policy consistently to streaming cards, static cards, text chunks, and media, and add focused tests for `off`, `first`, and `all` including streaming-plus-media. The shared `src/infra/outbound/reply-policy.ts` helpers are the right implementation reference.
