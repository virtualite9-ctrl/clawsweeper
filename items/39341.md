## Summary

Keep #39341 open. Current main still emits the Discord native slash-command no-allowlists warning from `dm.allowFrom` plus the pairing/store allowlist only, while runtime authorization, config validation, and docs all treat top-level `channels.discord.allowFrom` as valid. The exact-scope PR #39387 is closed unmerged, and #67222 covered related remediation wording rather than a shipped fix.

## Best Possible Solution

Keep #39341 as the canonical tracker. Fix `extensions/discord/src/security-audit.ts` by including normalized `discordCfg.allowFrom` in `ownerAllowFromConfigured` alongside legacy `dm.allowFrom` and the pairing/store allowlist, then add a focused regression in `extensions/discord/src/security-audit.test.ts` proving top-level `channels.discord.allowFrom` suppresses `channels.discord.commands.native.no_allowlists`.
