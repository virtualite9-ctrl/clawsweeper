## Summary

Keep #39341 open. Current main still omits top-level `channels.discord.allowFrom` from the Discord native slash-command owner allowlist audit, while runtime authorization, schema validation, docs, and remediation text all treat the top-level path as valid.

## Best Possible Solution

Keep #39341 as the canonical tracker. Update `extensions/discord/src/security-audit.ts` so `ownerAllowFromConfigured` includes normalized top-level `discordCfg.allowFrom` alongside legacy `dm.allowFrom` and the pairing/store allowlist, then add a regression in `extensions/discord/src/security-audit.test.ts` proving `channels.discord.allowFrom` suppresses `channels.discord.commands.native.no_allowlists`. The previously linked PR #39387 was exact-scope but closed unmerged, so it can be used as context rather than treated as shipped.
