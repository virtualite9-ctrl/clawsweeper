## Summary

Keep open. Current main still exposes the requester session to the target during `sessions_send`, still runs the peer A2A ping-pong path for normal sessions, and only skips that follow-up for parent-owned one-shot ACP children. That is a narrower ACP-specific mitigation, not a general fix for the duplicate-message report in #39476.
