## Summary

Keep #62110 open. Current main does not implement `agents.defaults.skipOptionalBootstrapFiles`: the config schema/types expose only the coarse `skipBootstrap` flag, and `ensureAgentWorkspace()` still writes `SOUL.md`, `IDENTITY.md`, `USER.md`, and `HEARTBEAT.md` whenever bootstrap creation is enabled. The PR is not obsolete or superseded by the earlier attempts; #62058 and #62104 are closed unmerged, while #62110 remains the active implementation attempt. However, the current PR needs maintainer/code-review follow-up because review comments identify incomplete propagation to other bootstrap callers and validation gaps.

## Best Possible Solution

Keep the PR open and ask for a focused revision: add a validated `agents.defaults.skipOptionalBootstrapFiles` contract, thread it through every `ensureAgentWorkspace()` caller that resolves agent defaults, preserve legacy setup-completion detection when skipped USER/IDENTITY files already exist and are customized, and add workspace/config tests plus docs for the new option.
