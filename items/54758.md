## Summary

Keep this PR open. Current main still captures structured Control UI connect-handshake failures for `onClose`, but pending RPC requests are still flushed with a generic `gateway closed (...)` error, so the PR addresses a real remaining gap. The provided diff is narrow and security-benign: it touches only the UI gateway client and its unit test.

## Best Possible Solution

Keep the PR open and have maintainers rebase or recreate the same narrow fix on current main: when `pendingConnectError` is present in `GatewayBrowserClient`'s close handler, flush pending requests with `new GatewayRequestError(pendingConnectError)` while preserving the generic close error for ordinary disconnects. Keep the targeted `ui/src/ui/gateway.node.test.ts` regression test for pending request rejection with `details.code`.
