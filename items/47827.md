## Summary

Current `main` already implements the requested cold-start behavior closely enough: `status --json` is routed to a lean fast path that skips plugin/runtime-heavy work, and `gateway call` goes straight through the gateway RPC client without any command policy that would preload plugins.

## Close Comment

Closing from a Codex review: current `main` already has the requested cold-start behavior. `status --json` is routed through `src/cli/program/routed-command-definitions.ts:62` into the lean JSON path, and `src/commands/status.scan.fast-json.ts:20` now skips plugin compatibility loading, channel runtime data, and memory inspection unless `--all` is requested. The regression tests in `src/commands/status.scan.fast-json.test.ts:53` and `src/commands/status.scan.test.ts:98` explicitly assert that JSON status does not preload the plugin registry and does not run gateway/update probes on cold start. For `openclaw gateway call`, the current command path goes directly from `src/cli/gateway-cli/register.ts:423` to `src/cli/gateway-cli/call.ts:26` and then `src/gateway/call.ts`, while CLI bootstrap only loads plugins when `loadPlugins` is enabled (`src/cli/command-path-policy.ts:5`, `src/cli/command-bootstrap.ts:14`). I also verified the `status --json` fast-path landed in commit `ed6b487e2033d87a85b5713a964cc677c70353f8` on `main`; I did not find a release tag containing that commit yet, so I’m marking this implemented on `main` without a shipped release tag.
