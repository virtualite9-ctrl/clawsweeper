## Summary

Keep open. Current main still does not preserve the successful-compaction plus retried-timeout cause chain in the external chat failure path. Mixed fallback exhaustion remains generic when verbose failure details are off, and the existing tests explicitly lock that behavior for mixed fallback causes.

## Best Possible Solution

Keep this open. The best fix is to carry structured compaction-attempt state from the embedded runner into the failure/fallback path, then update `src/auto-reply/reply/agent-runner-execution.ts` to format mixed fallback exhaustion after successful compaction as a specific external-chat recovery message: compaction succeeded, the retried model call hit the LLM idle timeout, and fallback candidates were unavailable or exhausted. Add a regression covering successful timeout/overflow compaction followed by timeout plus unavailable fallback candidates on an external chat context.
