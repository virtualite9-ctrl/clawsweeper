## Summary

Keep #48082 open. The report is recent, specific, and describes a native SIGSEGV on a Feiniu NAS custom Linux kernel where both `pty=true` and `pty=false` exec paths crash while native spawn and a manual node-pty probe work. Current main has useful related improvements, including PTY spawn fallback, sanitized diagnostics/stability export, and a Linux OOM-score child wrapper with an opt-out, but I found no source/changelog/history evidence that the Feiniu SIGSEGV itself has been fixed or that current main can no longer reproduce it.

## Best Possible Solution

Keep the issue open as a platform-specific exec crash. Ask the reporter to retest on the latest release/current main, attach `openclaw gateway diagnostics export` and `openclaw gateway stability --bundle latest` output, and run an A/B check with `OPENCLAW_CHILD_OOM_SCORE_ADJ=0` because current Linux exec/PTY children use the OOM-score wrapper. If it still crashes, add payload-free pre-spawn diagnostics around exec adapter selection, prepared argv shape, PTY vs child mode, and documented core-dump/backtrace collection for native crashes on Linux/NAS kernels.
