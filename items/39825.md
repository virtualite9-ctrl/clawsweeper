## Summary

Keep #39825 open. Current main still has the same Discord voice join failure shape: `/vc join` waits for `VoiceConnectionStatus.Ready` with a fixed 15s timeout, destroys the connection on readiness failure, and records the session only after the wait succeeds, so `/vc status` then reports no active sessions. The issue also has recent cross-host field reports and maintainer signal to keep it open.

## Best Possible Solution

Keep #39825 open as the Discord native voice readiness-abort tracker unless maintainers intentionally choose #63098 as the canonical replacement. A real fix likely belongs in `extensions/discord/src/voice/manager.ts`: add regression coverage for `entersState` aborts/timeouts before session activation, improve diagnostics, and either add bounded retry/backoff or a documented configurable connect timeout if that is the intended product behavior.
