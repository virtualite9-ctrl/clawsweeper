## Summary

Keep open. Current main still does not expose or honor a spawn-time `tools` allowlist for `sessions_spawn`; the only subagent tool filtering path remains config/session-derived policy, while native and ACP spawn parameters omit any tool-policy field.

## Best Possible Solution

Keep tracking this issue until maintainers either add a designed spawn-time tool policy through the `sessions_spawn` schema, native/ACP spawn param types, child-session metadata or run payload, and tool-policy pipeline with tests, or explicitly document that dynamic per-spawn tool allowlists are unsupported in favor of agent profiles and `tools.subagents.tools` config.
