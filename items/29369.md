## Summary

Keep open. Current main still does not expose or honor a spawn-time `tools` allowlist on `sessions_spawn`; the observable request remains a real core tool/API and policy-pipeline feature, not an already-shipped behavior or duplicate cleanup.

## Best Possible Solution

Keep #29369 open until maintainers design and implement a spawn-time tool policy through the `sessions_spawn` schema, native and ACP spawn parameter types, child-session/run metadata, and effective tool-policy resolution pipeline. The implementation should document precedence against existing `tools.allow`, `agents.list[].tools`, provider/group/sandbox policy, and `tools.subagents.tools`, with deny-wins tests for both native and ACP-backed spawns.
