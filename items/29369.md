## Summary

Keep #29369 open. Current main still does not expose or honor a spawn-time `tools` allowlist on `sessions_spawn`; the implementation and docs continue to route tool policy through configured global, provider, agent, group, sandbox, and subagent policy layers instead of per-call parameters.

## Best Possible Solution

Keep #29369 open until maintainers design and implement spawn-time tool policy for `sessions_spawn`. The likely path is to add schema/type support, parse and validate a narrow `tools` policy object, propagate it through both native and ACP spawn paths, merge it into the effective policy pipeline as a final narrowing step, document precedence against existing `tools.allow`, `agents.list[].tools`, group, sandbox, and `tools.subagents.tools` policy, and add tests for deny-wins behavior.
