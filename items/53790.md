## Summary

The reported webchat reply-delivery regression is already addressed on current `main`, and the same webchat handling is present in the latest shipped release `v2026.4.22`. Compared with `v2026.3.22`, current code now keeps tool-stream state alive until history reload finishes, handles detached `chat.side_result` replies explicitly, and defers `session.message` reloads until the active run clears.

## Close Comment

Closing from a Codex review because current `main` already contains the webchat reply-delivery changes that address this regression, and the same path is present in shipped release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`). In `ui/src/ui/app-gateway.ts`, webchat now keeps tool-stream state alive until `loadChatHistory()` completes, handles detached `chat.side_result` replies explicitly, and defers `session.message` reloads until the active run clears; coverage is in `ui/src/ui/app-gateway.node.test.ts`. I also compared against `v2026.3.22`, where those handlers were absent and tool-stream state was reset immediately, which matches the regression window described here. If this is still reproducible on `v2026.4.22` or newer, please open a fresh issue with current logs and a minimal repro.
