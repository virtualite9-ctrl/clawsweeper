## Summary

Keep open. Current `main` still matches the PR's reported bug: gateway lane setup configures `cron`, `main`, and `subagent`, but not `nested`, while cron-triggered inner agent work is explicitly remapped onto the `nested` lane. That means cron jobs can still serialize behind the default `nested` concurrency of `1` even when `cron.maxConcurrentRuns > 1`.
