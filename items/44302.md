## Summary

Current main does not implement optional webchat-to-external-channel mirroring. The current gateway/chat path intentionally keeps WebChat clients on the internal webchat route, and the reply dispatch policy suppresses routing from internal WebChat unless an explicit delivery route is set. No session config key or shipped implementation for `mirrorWebchatToLastExternalChannel` exists in the checkout.

## Best Possible Solution

Keep the issue open. The best implementation path is a core, opt-in `session.mirrorWebchatToLastExternalChannel` setting wired through the config schema/docs and the WebChat reply delivery path so a WebChat turn still replies in Control UI and also sends a nonblocking duplicate to the preserved last external route. Reuse the #52068 design intent, but rebase it on the current routing safeguards that prevent WebChat clients from inheriting external routes by default.
