## Summary

Keep open. Current main still has the reported product gap: `exec host=node` unconditionally sends `system.run.prepare`, the macOS companion node still does not advertise that command, and the gateway rejects undeclared node commands before forwarding. Related PR #38781 is an app-side alternative, not a clear superseder for this agent-side fallback, especially for already-deployed macOS companion nodes.

## Best Possible Solution

Keep PR #69725 open and review it as a plausible focused bug fix. The best path is to route `executeNodeHostCommand` based on the selected node's declared commands: keep the remote `system.run.prepare` round trip when advertised, synthesize a stable local `SystemRunApprovalPlan` when only `system.run` is advertised, and keep a regression test for macOS companion nodes that omit prepare. Coordinate with #37591/#38781, but do not close this PR as obsolete unless a canonical main-branch fix covers legacy no-prepare nodes too.
