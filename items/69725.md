## Summary

Keep open. Current main still unconditionally invokes `system.run.prepare` for `exec host=node` after only verifying `system.run`, while the macOS companion node still advertises and handles `system.run`/`system.which`/`system.notify` without `system.run.prepare`. The related open issue/PR track the same product bug, but no merged/current-main implementation supersedes this PR, and #69725 remains a distinct compatibility fix for nodes that omit prepare.

## Best Possible Solution

Keep #69725 open for maintainer review. The best path is likely to preserve compatibility by adding a well-tested local `systemRunPlan` fallback for nodes that declare `system.run` but not `system.run.prepare`, while separately considering #38781 or a follow-up to add native prepare support to the macOS app for newer nodes. Before landing, fix the changelog attribution and ensure the security tradeoff around skipped node-side filesystem hardening is explicitly accepted.
