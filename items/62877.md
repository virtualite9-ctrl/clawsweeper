## Summary

Current main still returns the raw `Usage: /btw <side question>` text for an empty `/btw` request, and the existing unit test asserts that unescaped output. The requested producer-side markdown-safe usage string has not landed, so this issue should remain open.

## Best Possible Solution

Keep this issue open and fix the producer-side usage text in `src/auto-reply/reply/commands-btw.ts`, preferably by rendering the whole command as code such as `Usage: `/btw <side question>`` or by escaping the placeholder. Update `commands-btw.test.ts`, then audit nearby auto-reply command usage strings with angle-bracket placeholders for the same markdown-channel behavior.
