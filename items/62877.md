## Summary

Current main still returns and tests the raw `Usage: /btw <side question>` producer text, so the markdown-rendering bug remains actionable in OpenClaw core auto-reply code. The issue is recent, has a concrete reproduction, and has no protected labels or maintainer-author constraint in the provided GitHub context.

## Best Possible Solution

Keep this issue open. Update `src/auto-reply/reply/commands-btw.ts` so the no-question usage text renders as inline code or escapes the placeholder, update `src/auto-reply/reply/commands-btw.test.ts`, and audit nearby auto-reply usage strings with raw angle-bracket placeholders for the same markdown/HTML rendering problem.
