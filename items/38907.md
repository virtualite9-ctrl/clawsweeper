## Summary

Keep open. The current main code still has a credible path matching the report: `openclaw acp` creates fallback `acp:<uuid>` bridge sessions, Gateway canonicalizes those to `agent:<default>:acp:<uuid>`, and the ACPX reply_dispatch hook can classify ACP-looking session keys without stored ACP metadata as `ACP_SESSION_INIT_FAILED`. I did not find a main-branch change that proves this specific bridge/session-metadata failure is fixed.
