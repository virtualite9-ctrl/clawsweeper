## Summary

Keep this PR open. Current main still lacks the memory text-search fallback requested by the PR: no-provider memory search returns an empty array when FTS is unavailable, and the keyword helper is still FTS-table-only. The PR remains a useful implementation candidate, but it should be rebased and narrowed before merge because the provided review context identifies provider-backed vector/hybrid behavior risks and the PR diff includes unrelated stale-base changes outside memory-core.

## Best Possible Solution

Keep this PR open as a useful but not-ready implementation candidate. The best maintainer path is to rebase it onto current main, narrow the diff to the memory-core fallback and directly related tests, add regression coverage for populated `chunks` with unavailable FTS, and explicitly preserve provider-backed vector-only and hybrid-disabled behavior before merge.
