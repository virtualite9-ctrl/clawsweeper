## Summary

Keep this issue open. Current main still contains the macOS local config save fallback and a separate AppState direct config writer that bypass the gateway baseHash conflict contract, so the reported stale config.set plus local disk clobber path is not fully fixed. Gateway Talk RPCs are read/broadcast/synthesis paths, which narrows the trigger, but it does not remove the unsafe macOS config mutation surfaces described in the report.

## Best Possible Solution

Keep this issue open and fix the macOS-owned config mutation path. The best path is to make ConfigStore.save conflict-aware end to end: do not fall back to OpenClawConfigFile.saveDict after a stale gateway config.set; refresh from config.get and preserve gateway.auth and all talk.providers before retrying; route AppState gateway sync through the same fresh snapshot/baseHash path or restrict it to app-local fields; and add macOS regression coverage for stale config.set rejection, token-auth local gateway config, multi-provider Talk config, and no gateway auth mutation or restart when enabling Talk Mode. #43244 can still address gateway RPC races, but it should not be treated as the full fix for this macOS path.
