## Summary

Keep this issue open. Current main still contains the macOS config mutation surfaces called out in the report: ConfigStore can fall back from a rejected gateway config.set to a direct local file write, and AppState still has a direct saveDict path outside the gateway baseHash contract. The related open PR #43244 is scoped to gateway RPC write serialization and explicitly does not cover direct non-gateway writers, so it is not a complete fix for this macOS Talk Mode/local gateway failure path.

## Best Possible Solution

Keep this issue open and fix the macOS-owned config mutation path. Make ConfigStore.save conflict-aware end to end: do not fall back to OpenClawConfigFile.saveDict after a stale gateway config.set; refresh with config.get and preserve gateway.auth plus all talk.providers before retrying; route AppState gateway sync through the same fresh snapshot/baseHash path or restrict it to app-local fields only; and add macOS regression coverage for stale config.set rejection, token-auth local gateway config, multi-provider Talk config, and no gateway auth mutation or restart when enabling Talk Mode.
