## Summary

Keep open. The underlying Telegram bare `group:<numeric>` target bug is still present on current main, while this PR is stale and not merge-ready because it edits obsolete `src/telegram/*` paths and misses the active shared parser.

## Best Possible Solution

Keep this PR open as a stale but relevant implementation candidate until a current-layout fix lands. The best maintainer path is a narrow patch in `extensions/telegram/src/targets.ts` that normalizes bare `group:<numeric>` before topic/colon parsing while preserving explicit `telegram:group:` behavior, plus regression coverage for `stripTelegramInternalPrefixes`, chat-id and lookup normalization, `parseTelegramTarget`, `resolveTelegramTargetChatType`, send delivery, actual Telegram plugin target normalization/session announce routing, and exec approval button routing. After that lands, close or supersede this obsolete PR rather than merging it as-is.
