## Summary

Keep open. Current main still preserves bare Telegram `group:<numeric>` targets instead of normalizing them, and the submitted patch is incomplete because it does not update `parseTelegramTarget`. The work is stale against the current `extensions/telegram` layout, but the underlying behavior is not implemented on main.

## Best Possible Solution

Keep this open or replace it with a current narrow Telegram fix in `extensions/telegram/src/targets.ts`: normalize bare `group:<numeric>` before topic/colon parsing, preserve explicit `telegram:group:` handling, and add regression coverage for `normalizeTelegramChatId`, `normalizeTelegramLookupTarget`, `parseTelegramTarget`, `resolveTelegramTargetChatType`, and the send path.
