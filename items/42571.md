## Summary

Keep open. Current main still preserves bare Telegram `group:<numeric>` targets in the active Telegram parser, so the requested delivery-only normalization is not implemented. The submitted PR is also not merge-ready because it targets obsolete `src/telegram/*` paths and misses the shared parser entry point, but the underlying bug remains actionable in `extensions/telegram/src/*`.

## Best Possible Solution

Keep this PR open as a stale but relevant implementation candidate. The best fix is a narrow current-layout patch in `extensions/telegram/src/targets.ts` that normalizes bare `group:<numeric>` before topic/colon parsing while preserving explicit `telegram:group:` behavior, plus regression coverage for `stripTelegramInternalPrefixes`, `normalizeTelegramChatId`, `normalizeTelegramLookupTarget`, `parseTelegramTarget`, `resolveTelegramTargetChatType`, send delivery, and approval-button routing.
