## Summary

Keep PR #69084 open. Current main still keys live WebChat heartbeat suppression to `showOk`, so the requested `showAlerts=false` behavior is not implemented, but the submitted one-line swap is not safe enough to close or merge as-is because `showOk` and `showAlerts` have distinct documented meanings.

## Best Possible Solution

Keep the PR open and revise `src/gateway/server-chat.ts` to classify heartbeat output before deciding visibility: suppress OK acknowledgments when `showOk=false`, suppress non-OK alert content when `showAlerts=false`, preserve OK-only acknowledgments when `showOk=true`, and add focused tests for `showOk=false/showAlerts=true`, `showOk=true/showAlerts=false`, and both-false WebChat heartbeat outputs.
