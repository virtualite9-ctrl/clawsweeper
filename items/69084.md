## Summary

Keep PR #69084 open. Current main still keys live WebChat heartbeat chat suppression to `showOk`, so the reported `showAlerts=false` alert-suppression gap is not fully implemented. The PR also should not be closed as safe or obsolete because the one-line swap to `showAlerts` conflicts with the documented two-flag contract and review comments flagged an OK-only visibility regression.

## Best Possible Solution

Keep the PR open and revise the Gateway live chat filter to classify heartbeat output before applying visibility: suppress OK-only acknowledgments when `showOk=false`, suppress non-OK alert content when `showAlerts=false`, preserve OK-only acknowledgments when `showOk=true`, and add focused tests for `showOk=false/showAlerts=true`, `showOk=true/showAlerts=false`, and both-false WebChat heartbeat outputs.
