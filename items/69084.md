## Summary

Keep #69084 open. Current main still keys WebChat heartbeat chat suppression to `showOk`, so the reported `showAlerts=false` WebChat leak is not implemented on main. The PR also should not be auto-closed as obsolete because the provided review discussion identifies a real follow-up concern: a plain `showOk` to `showAlerts` swap loses OK-only acknowledgement behavior when `showOk=true` and `showAlerts=false`.

## Best Possible Solution

Keep the PR open and request a targeted revision: classify heartbeat WebChat output as OK/ack versus alert, apply `showOk` only to OK acknowledgements and `showAlerts` only to alert content, and add focused Gateway tests for default behavior, `showAlerts=false`, and `showOk=true/showAlerts=false` OK-only acknowledgements. Coordinate with the broader consumer-path leakage issues only if the final fix needs a shared event-audience primitive.
