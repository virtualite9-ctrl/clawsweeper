## Summary

Current main still has the reported split-state behavior: exec approvals resolve through home-based `~/.openclaw` defaults and do not consult `OPENCLAW_STATE_DIR`, while the shared state-dir resolver and doctor warning path do. PR #62938 is also still an open, unmerged implementation candidate with closing syntax for this issue, so this issue should remain open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review, land, or close PR #62938. The best fix belongs in `src/infra/exec-approvals.ts`: resolve the exec approvals file and socket under `OPENCLAW_STATE_DIR` / `resolveStateDir()` before the current home default, add regression coverage in `src/infra/exec-approvals-store.test.ts`, preserve symlink protections, and update user-facing exec approvals docs if the storage path wording changes.
