## Summary

Keep this issue open. Current main still resolves exec approvals JSON/socket through hard-coded `~/.openclaw` defaults and does not consult `OPENCLAW_STATE_DIR`, while the shared state-dir resolver and doctor warning path do. PR #62938 remains open and unmerged with closing syntax for this issue. The later merged PR #72377 is useful adjacent work for trusted home symlinks, but it did not resolve the state-dir split.

## Best Possible Solution

Keep this issue open and review, land, or close PR #62938. The best fix belongs in `src/infra/exec-approvals.ts`: resolve the approvals file and socket under `OPENCLAW_STATE_DIR` / `resolveStateDir()` before the current home default, add regression coverage in `src/infra/exec-approvals-store.test.ts`, preserve symlink-hardening behavior, and update exec approvals docs if storage wording changes.
