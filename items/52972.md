## Summary

Keep open. Current main still only suppresses the reminder note when `successfulCronAdds` is nonzero or an enabled same-session cron job already exists, and the Pi tool-end handler still increments that counter only for the native `cron` tool add path. The discussion identified the right gap: PR #53024 tried to add exec/bash `openclaw cron add` tracking, but it is closed unmerged in the provided context, and a later comment reports the bug still reproducing on 2026.4.8.

## Best Possible Solution

Implement PR #53024 or an equivalent current-main fix: record turn-local success when `openclaw cron add` succeeds through exec/bash, or route assistant-created reminders through the native `cron.add` tool so `successfulCronAdds` is set consistently. Add regression coverage for successful exec-based cron add, failed exec cron add, and final auto-reply or Telegram text that confirms a reminder without appending the false note.
