## Summary

Keep open. Current `main` still implements cron persistence as a single store file plus companion state file, documents `~/.openclaw/cron/jobs.json` as the supported definition store, and exposes no `cron/jobs/` directory support or `openclaw cron migrate-to-dir` command.

## Evidence

- **Cron store still defaults to one JSON file:** `resolveDefaultCronStorePath()` returns `cron/jobs.json`, and `loadCronStore()` / `saveCronStore()` read and write that one store path plus the derived `-state.json` file. There is no directory scan or per-job file handling in this module.
  - file: [src/cron/store.ts:31](https://github.com/openclaw/openclaw/blob/55389f4be29359e2dd57ebaf548ba4a70331a445/src/cron/store.ts#L31)
  - sha: [55389f4be293](https://github.com/openclaw/openclaw/commit/55389f4be29359e2dd57ebaf548ba4a70331a445)
- **CLI has no migrate-to-dir command:** The cron CLI registers only `status`, `list`, `add`, simple commands (`rm/remove/delete`, `enable`, `disable`, `show`, `runs`, `run`), and `edit`. There is no `migrate-to-dir` or other directory-store migration surface.
  - file: [src/cli/cron-cli/register.ts:14](https://github.com/openclaw/openclaw/blob/55389f4be29359e2dd57ebaf548ba4a70331a445/src/cli/cron-cli/register.ts#L14)
  - sha: [55389f4be293](https://github.com/openclaw/openclaw/commit/55389f4be29359e2dd57ebaf548ba4a70331a445)
- **Remove/show/run command set is single-store oriented:** The simple cron command registration covers remove/show/runs/run and toggles only; nothing here suggests per-file job operations or migration commands.
  - file: [src/cli/cron-cli/register.cron-simple.ts:71](https://github.com/openclaw/openclaw/blob/55389f4be29359e2dd57ebaf548ba4a70331a445/src/cli/cron-cli/register.cron-simple.ts#L71)
