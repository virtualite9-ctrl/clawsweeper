## Summary

Keep PR #70605 open. Current main still forwards context-file content through the system prompt sanitizer without removing Markdown or HTML strikethrough text, and the existing tests cover context-file inclusion/omission but not this behavior. The PR is a focused, plausible fix with relevant tests, and no protected-label or maintainer-author rule requires automatic closure. It still needs normal maintainer review because the PR discussion raised sanitizer edge cases around regex overmatching and tilde fences.

## Best Possible Solution

Keep the PR open for normal review. The best path is to either land a carefully scoped sanitizer in src/agents/system-prompt.ts with regression tests for Markdown strikethrough, HTML tag variants, attributes, multiline content, and tilde-fenced code preservation, or close it later only if a separate merged change implements the same context-file behavior.
