## Summary

Keep open. Current main only partially mitigates the broader naming mismatch: the related #46687 startup-context path now loads slugged session-memory artifacts, but the specific #69536 dreaming/backfill path still filters for exact `YYYY-MM-DD.md` daily files. The bundled hook still emits `memory/YYYY-MM-DD-slug.md`, while CLI rem-backfill, Control UI dream-diary backfill, live daily ingestion, and short-term promotion eligibility still reject or skip those slugged files.

## Best Possible Solution

Keep #69536 open and fix the remaining dreaming/backfill path. Add a shared daily-memory artifact parser that accepts both canonical `YYYY-MM-DD.md` and date-prefixed slugged `YYYY-MM-DD-*.md` files while excluding generated dreaming artifacts. Use it in `extensions/memory-core/src/cli.runtime.ts`, `extensions/memory-core/src/dreaming-phases.ts`, and `src/gateway/server-methods/doctor.ts`; update `extensions/memory-core/src/short-term-promotion.ts` so slugged source paths remain eligible and rehydratable instead of being rewritten to `memory/YYYY-MM-DD.md`. Add focused tests for a slugged session-memory file through CLI rem-backfill, Control UI backfill, live light ingestion, and promotion rehydration, then update memory/dreaming docs to describe the supported file patterns.
