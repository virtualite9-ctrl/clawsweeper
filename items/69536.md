## Summary

Keep open. Current main still has the reported naming mismatch for the dreaming/backfill path: the bundled session-memory hook writes `memory/YYYY-MM-DD-slug.md`, while memory-core daily ingestion and `memory rem-backfill` only accept bare `YYYY-MM-DD.md` daily files. General `memory_search` can index `memory/*.md`, and dreaming can ingest redacted session transcripts when available, but those do not make the hook-generated slugged files participate as daily notes in grounded backfill or live daily ingestion. The related #46687 cleanup covered startup context visibility, not this dreaming/backfill promotion gap.

## Best Possible Solution

Keep #69536 open and resolve it in the memory-core/session-memory boundary. The most conservative product fix is either to make the bundled session-memory hook append structured session sections to the canonical `memory/YYYY-MM-DD.md` daily note, or to broaden memory-core daily/backfill discovery to include date-prefixed slugged files while preserving original source paths, deterministic ordering, dedupe/checkpoint behavior, and exclusions for generated dreaming artifacts. The change should add regression coverage for `memory rem-backfill --path ./memory --stage-short-term` and live light-phase ingestion with a `YYYY-MM-DD-slug.md` input, then update the hook and memory/dreaming docs so the file contract is explicit.
