## Summary

Keep #69536 open. Current main has a partial related fix for startup context loading slugged session-memory files, but the reported dreaming/backfill path still only discovers date-only `memory/YYYY-MM-DD.md` files while the bundled session-memory hook still writes `memory/YYYY-MM-DD-slug.md`.

## Best Possible Solution

Keep this open and fix the product boundary deliberately. The likely maintainer path is either to change `session-memory` to append into canonical `memory/YYYY-MM-DD.md`, or to broaden memory-core's daily/backfill scanners to accept date-prefixed slug files. If broadening scanners, preserve real source paths for refs and dedupe, handle multiple files for the same day without collapsing them, and add regression tests for live light daily ingestion, `memory rem-harness/rem-backfill --path`, `--stage-short-term`, and `doctor.memory.backfillDreamDiary`; then update the memory and hook docs so users see one coherent contract.
