## Summary

Keep #69536 open. The related #46687 cleanup covered startup-context loading of slugged session-memory files, but current main still preserves the reported dreaming/backfill mismatch: session-memory writes `memory/YYYY-MM-DD-slug.md`, while dreaming daily ingestion, `memory rem-backfill`, gateway Dream Diary backfill, and short-term promotion path checks still accept date-only `YYYY-MM-DD.md` daily files or the separate `.dreams/session-corpus` path.

## Best Possible Solution

Keep the issue open and fix the product boundary directly. Either change `session-memory` to append to canonical `memory/YYYY-MM-DD.md`, or introduce a shared memory-file classifier/date extractor used by dreaming daily ingestion, CLI `rem-backfill`, gateway Dream Diary backfill, and short-term promotion so slugged session-memory files are ingested consistently. Add regression tests for `memory/YYYY-MM-DD-slug.md` across the CLI/gateway/dreaming paths and update hook, memory, and dreaming docs to describe one coherent convention.
