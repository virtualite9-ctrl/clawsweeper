## Summary

Keep this issue open. Current main still forces Telegram DM partial preview lanes through message/editMessageText transport, while the lower-level draft stream still supports DM-preferring auto/sendMessageDraft behavior. The reported regression is still plausible and not implemented by the latest release evidence provided.

## Best Possible Solution

Keep this issue open. The best fix is a deliberate Telegram dispatcher change that restores `auto`/sendMessageDraft transport for DM answer partial previews when safe, preserves duplicate-final suppression and materialization behavior, decides whether reasoning previews should use the same transport, updates the Telegram docs, and adds regression coverage proving smooth DM partial previews without duplicate final messages.

## Risks / Open Questions

- A naive switch back to draft transport for every DM lane could reintroduce the duplicate-final/materialization behavior that the current forced message transport comment is guarding against.
- Reasoning and answer preview lanes may need separate transport decisions, because current code and tests handle them as distinct lanes.
- The Telegram docs currently document direct-chat previews as editMessageText, so any behavior restoration should update docs and tests together.
