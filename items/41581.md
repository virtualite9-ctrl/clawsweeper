## Summary

Keep this issue open. Current main still forces Telegram DM partial preview streaming through the message/editMessageText transport, while the lower-level draft stream still supports auto DM draft transport. Current tests and docs lock in the edit-message behavior, and the related fix attempts #62207 and #62210 were closed unmerged, so there is no implemented or superseding fix on main.

## Best Possible Solution

Keep the issue open. The best fix is a deliberate Telegram dispatcher change that restores auto/sendMessageDraft transport for DM answer partial previews when safe, preserves duplicate-final suppression and reasoning-lane behavior, updates the docs to match the chosen product behavior, and adds regression coverage proving DM answer previews are smooth without reintroducing duplicate final sends.
