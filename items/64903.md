## Summary

Keep open. Current main still starts NodeForegroundService as a dataSync foreground service on targetSdk 36, with no guard or fallback around startForeground quota exhaustion. The source, manifest, tests, docs, and Android foreground-service policy all still match the reported crash path.

## Best Possible Solution

Keep this issue open and fix it in the Android app. The likely product path is to stop treating the always-on gateway connection as a long-running dataSync foreground service on targetSdk 36, add quota/start-failure handling around NodeForegroundService, implement any needed onTimeout demotion/stop behavior, and cover the new behavior with focused Android tests plus device or emulator proof.
