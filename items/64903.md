## Summary

Keep open. Current main still routes Android launch/reconnect through NodeForegroundService, starts it as a foreground service with dataSync as the base type on targetSdk 36, and has no guard around startForeground quota exhaustion. The recent Talk Mode work adds microphone escalation, but it does not resolve the reported dataSync ForegroundServiceStartNotAllowedException path.

## Best Possible Solution

Keep this issue open and fix it in the Android app. The likely maintainer path is to stop treating the always-on gateway connection as a long-running dataSync foreground service on targetSdk 36, add explicit handling around startForeground failures and dataSync quota exhaustion, implement any required onTimeout demotion/stop behavior, and cover it with focused Android tests plus real-device or emulator proof.
