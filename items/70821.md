## Summary

Keep #70821 open. Current `main` still does not expose `xhigh` from the `amazon-bedrock` plugin's `resolveThinkingProfile` for Claude Opus 4.7, while the shared Anthropic transport already maps Opus 4.7 `xhigh` to `output_config.effort: "xhigh"`. The PR is a focused fix for a real provider-profile gap, not obsolete cleanup.

## Best Possible Solution

Keep the PR open and review it as a focused Amazon Bedrock plugin bug fix. The best implementation is the PR's narrow provider-profile change plus regression test: advertise `xhigh` only for Claude Opus 4.7 Bedrock model ids, preserve Claude 4.6 `adaptive` behavior, and rely on the already-shipped Anthropic transport mapping for the actual `output_config.effort: "xhigh"` payload.
