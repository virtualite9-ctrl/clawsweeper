## Summary

Keep #38844 open. Current main has strong runtime mitigations: browser upload can pass `ref`/`inputRef`/`element`, `/hooks/file-chooser` can arm upload and click a `ref` in one handler, and dispatcher timeouts keep action context instead of always reporting browser-control outage. The remaining gap is still user-facing guidance: public browser-control docs still describe upload as an arming call before the triggering click and still say action CSS selectors are unsupported, which can steer callers back into the split stale-ref workflow reported here.

## Best Possible Solution

Keep the issue open until maintainers either explicitly accept the runtime mitigations as sufficient or land the remaining guidance and coverage. The concrete path is to update `docs/tools/browser-control.md`, `docs/cli/browser.md` if needed, and `extensions/browser/skills/browser-automation/SKILL.md` to prefer single-call `upload` with `ref` or direct `inputRef` on dynamic pages, clarify selector support versus existing-session limits, and add a focused contract assertion that upload-with-ref arms and clicks in one intended workflow.
