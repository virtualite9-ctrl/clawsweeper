## Summary

Keep this PR open. Current main still has a dispatch-level empty-output path: if model dispatch returns no reply payload and no tool/block/final callback queued anything, `dispatchReplyFromConfig` records completion and returns with no user-visible reply. The PR is a focused implementation candidate, and the provided review discussion shows it was narrowed to direct/private contexts after CI feedback.

## Best Possible Solution

Review and update this PR against current main rather than auto-close it. The likely maintainer path is a narrow fallback after dispatch counts are known, only for direct-style user delivery with no routed origin, no suppressed delivery, no queued tool/block/final output, no block-stream output, and no intentional handled/no-text branch, plus regression tests for both the direct empty-output case and the channel/group no-op case.
