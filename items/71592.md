## Summary

Keep this issue open. Current main still advertises shared TUI slash commands such as `/status` and `/compact`, but the TUI command handler has no local embedded handling for them; the fallback still adds the raw slash text as a user message and sends it to the embedded agent run path. The prior ClawSweeper comment correctly identified this gap, and it remains present at current main `110fa97f2aa171d601d1736a5f4855d299f6fb88`.

## Best Possible Solution

Keep the issue open and fix the TUI local embedded command boundary. Known shared commands advertised in TUI autocomplete, especially `/status` and `/compact`, should either execute through a shared local command adapter or return an explicit local unsupported/system response in `opts.local` mode, but they should not fall through to `sendMessage(raw)`. Add focused local-mode regression tests for both commands and keep gateway-connected forwarding behavior intact.
