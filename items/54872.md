## Summary

Keep #54872 open. Current main does not implement the PR's requested behavior: `tools.media` model schema still accepts generic provider/model entries with no LiteLLM routing-alias rejection, and the media runtime still turns `agents.defaults.imageModel` refs such as `litellm/vision` into provider entries without a guard. The linked issue #54865 describes a concrete user-facing failure, and the PR discussion/review comments show active refinement rather than an obsolete or duplicated change.

## Best Possible Solution

Keep #54872 open for maintainer review or replacement. The best path is a narrow config-validation change that rejects LiteLLM routing aliases in `tools.media` provider entries and in `agents.defaults.imageModel` only when that value can feed media understanding, with regression tests for shared/image/audio/video model lists, CLI exemptions, fallback behavior, provider capability inference, docs/help text, generated schema metadata, and changelog placement.
