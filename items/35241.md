## Summary

Keep open. Current main still does not expose `--session-key` on `openclaw agent`, while the resolver, embedded command type, gateway protocol, and gateway handler already support explicit session keys. The PR addresses a real core CLI parity gap with downstream demand, so no conservative auto-close reason applies.

## Best Possible Solution

Keep the PR open. The best path is a scoped core CLI change that adds `--session-key` to `openclaw agent`, threads it through `AgentCliOpts` and `resolveSessionKeyForRequest()` for both gateway and embedded fallback paths, preserves current gateway/embedded validation behavior, and updates the CLI docs/help/tests. If maintainers decide not to expose raw session keys, that should be an explicit product decision rather than an automated stale close.
