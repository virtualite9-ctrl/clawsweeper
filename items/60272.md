## Summary

Current main already provides pluginized context-engine and hook extension points, and the richer claw-code-style behavior requested here fits the repo’s documented ClawHub/plugin scope rather than a new OpenClaw core architecture layer.

## Risks / Open Questions

- OpenClaw core still does not offer a standardized shell-command hook runner with the proposed exit-code contract; operators wanting that exact UX need a plugin or hook-pack wrapper.
- If maintainers later decide a shared shell-hook protocol is important across multiple plugins, this scope call could need revisiting.
