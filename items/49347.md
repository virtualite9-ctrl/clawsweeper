## Summary

Keep open. Current main partially improved the published/root install path, but the reported full source workspace install path still includes extension workspaces and still resolves the WhatsApp Baileys dependency to the same GitHub codeload libsignal tarball that triggered the ECONNRESET report. A maintainer comment explicitly says to leave this open for source/workspace install reliability after the root npm install fixes.

## Best Possible Solution

Keep this as a source/workspace install reliability issue. The best fix is either to make normal source installs avoid optional extension-owned GitHub codeload tarballs until the relevant plugin is being developed or activated, or to add a documented resilient fallback/cache/mirror strategy for the WhatsApp/libsignal tarball. At minimum, add source-install troubleshooting docs for GitHub codeload ECONNRESET while preserving the plugin-local dependency boundary established by the recent root install fixes.
