## Summary

Keep open. Current main still preserves inline/bare TTS directive behavior, and this PR changes a public TTS directive contract while also carrying unrelated Slack/config-validation changes. A maintainer comment already says to rework rather than merge as-is.

## Best Possible Solution

Keep this PR open for a narrower revision: split or drop the unrelated Slack/config-validation changes, have maintainers explicitly decide whether inline/bare TTS directives remain supported or migrate to standalone-only syntax, then update `src/tts/directives.ts`, `src/tts/directives.test.ts`, `test/helpers/plugins/tts-contract-suites.ts`, `docs/tools/tts.md`, runtime prompt hints if needed, and changelog coverage in the same focused change.
