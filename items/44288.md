## Summary

Keep #44288 open. Current main does not implement backup exclude patterns, `--smart-exclude`, `.backupignore`, protected-path guards, or manifest `excludedStats`; the backup CLI still only exposes output/json/dry-run/verify/only-config/workspace toggles and the archive writer still tars every planned asset without an exclude filter. The PR is also not superseded: the provided context shows earlier competing PRs #40804 and #40810 are closed, while the core feature request #40786 and large-backup issues #42273/#42282 remain open and this PR had recent author follow-up on April 26, 2026.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to finish the backup-exclude implementation against current `main`, resolve or file a canonical follow-up for non-state asset filtering, trim unrelated dependency changes if present, and then validate the CLI forwarding, archive creation, verify compatibility, protected-path guard behavior, and docs before merging.
