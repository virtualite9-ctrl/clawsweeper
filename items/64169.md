## Summary

Current `main` already has session-scoped bundled MCP teardown for isolated runs and gateway session cleanup, plus stdio transport process-tree reaping. This matches the reported leak path of MCP stdio children accumulating under the gateway, although I did not identify a released tag newer than `v2026.4.23` that contains it.

## Evidence

- **Session MCP runtimes are explicitly disposable:** The bundled MCP runtime manager caches runtimes by session and `disposeSession()` removes the runtime, clears session-key mappings, and calls `runtime.dispose()` so a finished or reset session does not retain its MCP pool.
  - file: [src/agents/pi-bundle-mcp-runtime.ts:376](https://github.com/openclaw/openclaw/blob/5b8bd6371c666a5de43c82305e0315771e8e49d5/src/agents/pi-bundle-mcp-runtime.ts#L376)
  - command: `nl -ba src/agents/pi-bundle-mcp-runtime.ts | sed -n '295,485p'`
  - sha: [5b8bd6371c66](https://github.com/openclaw/openclaw/commit/5b8bd6371c666a5de43c82305e0315771e8e49d5)
- **Isolated cron runs opt into end-of-run MCP cleanup:** Isolated cron agent turns pass `cleanupBundleMcpOnRunEnd: params.job.sessionTarget === "isolated"` into the embedded runner, directly targeting the fresh-session path described in the issue.
  - file: [src/cron/isolated-agent/run-executor.ts:165](https://github.com/openclaw/openclaw/blob/5b8bd6371c666a5de43c82305e0315771e8e49d5/src/cron/isolated-agent/run-executor.ts#L165)
  - command: `nl -ba src/cron/isolated-agent/run-executor.ts | sed -n '158,170p'`
  - sha: [5b8bd6371c66](https://github.com/openclaw/openclaw/commit/5b8bd6371c666a5de43c82305e0315771e8e49d5)
- **Embedded runner retires session MCP runtime after the run:** When `cleanupBundleMcpOnRunEnd` is enabled, the embedded runner retires the MCP runtime by session key or session id in its `finally` block, covering isolated-session runs after completion.
  - file: [src/agents/pi-embedded-runner/run.ts:2297](https://github.com/openclaw/openclaw/blob/5b8bd6371c666a5de43c82305e0315771e8e49d5/src/agents/pi-embedded-runner/run.ts#L2297)
  - command: `nl -ba src/agents/pi-embedded-runner/run.ts | sed -n '2294,2315p'`
  - sha: [5b8bd6371c66](https://github.com/openclaw/openclaw/commit/5b8bd6371c666a5de43c82305e0315771e8e49d5)
- **Gateway session reset/delete also tears down MCP runtime:** Gateway session cleanup now calls `retireSessionMcpRuntime(...)` after aborting and waiting for the embedded run to end, so manual reset/delete paths reap the session-owned runtime too.
  - file: [src/gateway/session-reset-service.ts:229](https://github.com/openclaw/openclaw/blob/5b8bd6371c666a5de43c82305e0315771e8e49d5/src/gateway/session-reset-service.ts#L229)
  - command: `nl -ba src/gateway/session-reset-service.ts | sed -n '225,240p'`
  - sha: [5b8bd6371c66](https://github.com/openclaw/openclaw/commit/5b8bd6371c666a5de43c82305e0315771e8e49d5)
- **Stdio transport reaps the child process tree:** The stdio MCP transport ends stdin, waits briefly, then calls `killProcessTree(processToClose.pid)` if the process did not exit cleanly, which addresses leftover child processes under the gateway.
  - file: [src/agents/mcp-stdio-transport.ts:112](https://github.com/openclaw/openclaw/blob/5b8bd6371c666a5de43c82305e0315771e8e49d5/src/agents/mcp-stdio-transport.ts#L112)
  - command: `nl -ba src/agents/mcp-stdio-transport.ts | sed -n '112,136p'`
  - sha: [5b8bd6371c66](https://github.com/openclaw/openclaw/commit/5b8bd6371c666a5de43c82305e0315771e8e49d5)
- **Regression coverage exists for both run-end and reset cleanup:** Tests assert one-shot embedded runs dispose the session MCP runtime, and gateway `sessions.reset` disposes the prior session runtime as part of cleanup.
  - file: [src/agents/pi-embedded-runner.e2e.test.ts:460](https://github.com/openclaw/openclaw/blob/5b8bd6371c666a5de43c82305e0315771e8e49d5/src/agents/pi-embedded-runner.e2e.test.ts#L460)
  - command: `nl -ba src/agents/pi-embedded-runner.e2e.test.ts | sed -n '458,479p'`
