## Summary

Keep #28913 open. Current main still does not implement source- or wrapper-aware autoCapture filtering for memory-lancedb. The requested `captureFilter.excludeWrapped` config is absent from the parser and manifest, `shouldCapture()` remains trigger-based and wrapper-unaware, and the `agent_end` hook still sends user-message text directly through `shouldCapture()` before storing memories.

## Best Possible Solution

Keep the issue open and implement a backward-compatible memory-lancedb option such as `captureFilter.excludeWrapped`. The change should add typed config parsing, manifest schema/UI hints, and an `agent_end` pre-filter that skips configured `EXTERNAL_UNTRUSTED_CONTENT` blocks before `shouldCapture()`, with tests for wrapped external text containing trigger words like `always` while preserving direct user preference capture.
