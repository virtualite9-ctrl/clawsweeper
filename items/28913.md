## Summary

Keep #28913 open. Current main still has no source/wrapper-aware autoCapture filter for memory-lancedb: the plugin config and manifest do not expose `captureFilter`/`excludeWrapped`, `shouldCapture()` is still trigger-based and unaware of `EXTERNAL_UNTRUSTED_CONTENT`, and the `agent_end` auto-capture path stores matching user-message text without a wrapper/source exclusion step. The existing ClawSweeper comment is consistent with current main at 67b9167b808f2ff065c91665d6f36256a5c23ef2.

## Best Possible Solution

Keep the issue open and implement a backward-compatible `memory-lancedb` option such as `captureFilter.excludeWrapped`. The fix should add typed config parsing, manifest schema/UI hints, and an `agent_end` pre-filter that skips configured `EXTERNAL_UNTRUSTED_CONTENT` blocks before `shouldCapture()`. Add tests for wrapped external text containing trigger words like `always` while preserving direct user preference auto-capture; update user-facing plugin docs if the option becomes public.
