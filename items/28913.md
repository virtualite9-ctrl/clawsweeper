## Summary

Keep #28913 open. Current main still has memory-lancedb autoCapture driven by shouldCapture trigger heuristics, and the memory plugin has no captureFilter/excludeWrapped config, manifest schema entry, or wrapper/source-aware pre-filter before storing auto-captured memories. The existing ClawSweeper comment’s keep-open conclusion still matches current main at 406ae72fd2789c78053f6b94006ec8f488bc1c00.

## Best Possible Solution

Keep the issue open. Implement a backward-compatible memory-lancedb option such as plugins.entries.memory-lancedb.config.captureFilter.excludeWrapped, expose it in the typed parser and openclaw.plugin.json schema/UI hints, apply a robust source/wrapper-aware pre-filter before shouldCapture in the agent_end auto-capture path, and add regression tests proving wrapped EXTERNAL_UNTRUSTED_CONTENT with trigger words like always is skipped while normal direct preference capture still works.
