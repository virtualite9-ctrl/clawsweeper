## Summary

Keep #28913 open. Current main still lacks a source/wrapper-aware autoCapture filter for memory-lancedb: the config parser and manifest do not expose captureFilter/excludeWrapped, shouldCapture remains trigger-based and EXTERNAL_UNTRUSTED_CONTENT-unaware, and the agent_end auto-capture path stores matching user-message text without any wrapper/source exclusion step.

## Best Possible Solution

Keep the issue open and implement a backward-compatible memory-lancedb option such as `captureFilter.excludeWrapped`. Add typed config parsing, manifest schema/UI hints, and an agent_end pre-filter that skips configured EXTERNAL_UNTRUSTED_CONTENT blocks before shouldCapture. Cover wrapped external text containing trigger words like `always` in tests while preserving direct user preference auto-capture; update user-facing plugin docs if the option becomes public.
