## Summary

Keep #28913 open. Current main still has no source-based autoCapture filter for memory-lancedb: config parsing and the plugin manifest do not expose captureFilter/excludeWrapped, shouldCapture remains trigger-based and wrapper-unaware, and the agent_end capture path stores matching user-message text without checking EXTERNAL_UNTRUSTED_CONTENT boundaries.

## Best Possible Solution

Keep the issue open and implement a backward-compatible memory-lancedb option such as captureFilter.excludeWrapped. Add typed config parsing, manifest schema/UI hints, and an agent_end pre-filter that skips configured external-content boundary blocks before shouldCapture. Cover EXTERNAL_UNTRUSTED_CONTENT with trigger words like always in tests while preserving direct user preference auto-capture; update user-facing plugin docs if the option becomes public.

- The eventual fix should be narrow and configurable so ordinary direct user preference capture, including existing always/prefer trigger behavior, is not broadly disabled.
