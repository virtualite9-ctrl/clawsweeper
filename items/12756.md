## Summary

Keep #12756 open. Current main still does not implement Telegram `message read`: the shared action exists, but the bundled Telegram plugin does not advertise `read`, does not map it to a Telegram runtime action, has no `readMessages`/inbound-store runtime handler, and the docs/tests still describe Telegram without read support. The related PR #14367 is useful prior art for an inbound-store design, but the provided GitHub context shows it was closed unmerged, and current main does not contain that implementation. The recent maintainer/Codex comment by steipete also explicitly kept this issue open, which is consistent with the current code evidence.

## Best Possible Solution

Keep #12756 open. Implement Telegram `message read` end-to-end in the bundled Telegram plugin: advertise `read` in discovery, map the shared action to a Telegram runtime handler, maintain a bounded Telegram-safe inbound message store or equivalent source, align CLI/tool params (`target`/`to`, `limit`, `before`, `after`, and topic/thread handling), document platform limitations, and add focused adapter/runtime/contract tests. Treat closed PR #14367 as useful prior art rather than shipped proof.
