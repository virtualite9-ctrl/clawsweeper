## Summary

Keep #12756 open. Current main still has the shared `read` action, but the bundled Telegram plugin does not advertise, gate, map, document, or implement Telegram message reads. The prior implementation PR #14367 was closed unmerged, and the issue discussion already treats #12756 as the canonical remaining tracker.

## Best Possible Solution

Keep #12756 open and implement the feature in the bundled Telegram plugin: add `read` discovery, action mapping, runtime handling backed by a bounded inbound message store, config/schema/help, docs, tests, and CLI `--target` compatibility. PR #14367 is useful prior art but does not supersede the issue because it was closed unmerged.
