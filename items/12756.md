## Summary

Keep #12756 open. Current main at 7e13f3f5140168e31027b39a9855ade3fadf24d5 still has the shared message `read` action, but the bundled Telegram plugin does not advertise, map, dispatch, document, or implement Telegram message reads. The related PR #14367 remains closed unmerged in the provided GitHub context, so it is useful prior art rather than a shipped fix.

## Best Possible Solution

Keep #12756 open and implement Telegram `message read` in the bundled Telegram plugin end-to-end: discovery/action mapping, config gating, runtime handling backed by an explicit bounded inbound message store, `openclaw message read --channel telegram --target ...` parameter compatibility, docs, and focused contract/runtime tests. Use PR #14367 as prior art, but do not treat it as shipped.
