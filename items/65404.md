## Summary

Keep open. Current main still matches the reported codex-cli observability bug: the CLI runner prepares bundle MCP config, but builds the system prompt report with `tools: []`, so `tools.entries` and `schemaChars` cannot reflect injected MCP tools. The same-author PR #65409 is still the implementation candidate and explicitly declares it closes this issue.

## Best Possible Solution

Keep the issue open and review PR #65409 as the current implementation candidate. The durable fix should materialize the effective codex-cli bundle MCP tool catalog before `buildSystemPromptReport`, pass those tools instead of `[]`, and add regression coverage asserting `systemPromptReport.tools.entries` and `schemaChars` reflect injected MCP tools.
