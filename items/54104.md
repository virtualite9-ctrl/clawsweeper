## Summary

Keep open. Current `main` still does not recognize `google-vertex` as a real plugin/provider id for auth or plugin config. It is only a hook-only alias under the bundled `google` plugin, while stale `plugins.entries.google-vertex` config is only handled as a generic stale entry with a `doctor --fix` cleanup message, not a rename/migration.

There is no manifest provider/auth alias for `google-vertex`.
