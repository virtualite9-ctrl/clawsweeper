## Summary

Keep PR #39372 open. Current main still rejects the exact requested headless mode="session" without thread=true behavior in both native subagent and ACP sessions_spawn paths, and docs/tests still define session mode as thread-bound. The PR is still relevant, but the ACP inline-delivery concern from the review discussion remains a real merge blocker for the submitted shape.

## Best Possible Solution

Keep #39372 open as the canonical maintainer decision point for decoupling session persistence from channel thread binding. If OpenClaw should support headless persistent sessions, update both native and ACP spawn guards, make ACP inline delivery require an actual bound conversation/thread, keep failed-start cleanup correct, use distinct headless accepted notes, and update docs/tests. If maintainers prefer the current mode="run" plus sessions_send workflow, close the PR with that explicit product rationale.
