## Summary

Keep PR #39372 open. Current main still rejects the exact headless `mode="session"` without `thread=true` behavior in both native subagent and ACP spawn paths, and the docs/tests still encode the thread-bound session contract. The PR also has a still-relevant ACP inline-delivery concern that needs maintainer or author follow-up before it can be considered merge-ready.

## Best Possible Solution

Keep #39372 open as the canonical maintainer decision point. If OpenClaw should support headless persistent sessions, update both native and ACP spawn guards, make ACP inline delivery require an actual thread/conversation binding, keep failed-start cleanup correct, add distinct headless accepted notes, and update docs/tests. If maintainers prefer the current `mode="run"` plus `sessions_send`/completion-event workflow, close the PR with that explicit product rationale.
