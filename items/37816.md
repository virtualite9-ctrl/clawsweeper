## Summary

Keep open. Current `main` has Telegram preview streaming, `/verbose` tool bubbles, and separate reasoning visibility, but it does not implement the requested Web UI-style full activity visibility with persistent per-step Telegram messages for thinking, tool activity, memory operations, and final output.

## Evidence

- **Telegram docs only expose preview/block modes, not a full activity mode:** Current Telegram docs list `channels.telegram.streaming` as only `off | partial | block | progress`; they describe edited preview messages for text replies and Telegram-only reasoning streamed into the preview, not a persistent per-step activity feed.
  - file: [docs/channels/telegram.md:274](https://github.com/openclaw/openclaw/blob/55389f4be29359e2dd57ebaf548ba4a70331a445/docs/channels/telegram.md#L274)
  - command: `nl -ba docs/channels/telegram.md | sed -n '268,296p'`
  - sha: [55389f4be293](https://github.com/openclaw/openclaw/commit/55389f4be29359e2dd57ebaf548ba4a70331a445)
- **Verbose/reasoning docs are narrower than the request:** `/verbose` is documented as tool summaries and optional tool outputs only, while `/reasoning` shows a separate `Reasoning:` message or Telegram preview stream. The docs do not describe full visibility for intermediate agent actions like memory operations or a durable step-by-step activity log.
  - file: [docs/tools/thinking.md:73](https://github.com/openclaw/openclaw/blob/55389f4be29359e2dd57ebaf548ba4a70331a445/docs/tools/thinking.md#L73)
  - command: `nl -ba docs/tools/thinking.md | sed -n '72,96p'`
  - sha: [55389f4be293](https://github.com/openclaw/openclaw/commit/55389f4be29359e2dd57ebaf548ba4a70331a445)
- **Telegram dispatch only maintains answer and reasoning lanes:** The Telegram dispatcher creates exactly two draft lanes, `answer` and `reasoning`. There is no third lane or general event channel for persistent activity messages covering broader agent activity.
  - file: [extensions/telegram/src/bot-message-dispatch.ts:339](https://github.com/openclaw/openclaw/blob/55389f4be29359e2dd57ebaf548ba4a70331a445/extensions/telegram/src/bot-message-dispatch.ts#L339)
  - command: `nl -ba extensions/telegram/src/bot-message-dispatch.ts | sed -n '330,410p'`
  - sha: [55389f4be293](https://github.com/openclaw/openclaw/commit/55389f4be29359e2dd57ebaf548ba4a70331a445)
- **Tool progress is folded into preview behavior, not durable parity-style activity messages:** When preview tool progress is enabled, the dispatcher suppresses default tool-progress messages and pushes tool/item progress into the answer preview stream. That is materially different from the issue's requested persistent separate Telegram messages for each activity step.
  - file: [extensions/telegram/src/bot-message-dispatch.ts:390](https://github.com/openclaw/openclaw/blob/55389f4be29359e2dd57ebaf548ba4a70331a445/extensions/telegram/src/bot-message-dispatch.ts#L390)

- **Reasoning support only splits replies into reasoning vs answer:** The reasoning coordinator only extracts `reasoningText` and `answerText`, confirming the current Telegram-specific visibility model is limited to those two surfaces rather than full agent activity parity.
