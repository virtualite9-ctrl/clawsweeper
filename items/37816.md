## Summary

Keep #37816 open. Current main still lacks an opt-in Telegram mode that emits every relevant agent activity step as durable, separate Telegram messages. The current surface covers preview edits, limited tool-progress summaries, `/verbose` tool bubbles, and `/reasoning` controls, but those are not Web UI parity for persistent per-activity visibility. The related Telegram reports are narrower closed bugs and do not supersede this feature request.

## Best Possible Solution

Keep this open as bundled Telegram plugin work. Add an explicit opt-in mode or config flag in `extensions/telegram` with schema, docs, and tests that emits durable separate messages for relevant agent activity events while preserving current preview-edit behavior by default. If existing replyOptions hooks do not cover memory or other activity sources, add a narrow additive channel/plugin event contract instead of hard-coding around it.

## Risks / Open Questions

- A durable per-activity Telegram mode could be noisy and may need rate-limit/backpressure handling, especially in groups.
- Reasoning, memory, command output, and tool result summaries can expose sensitive data, so the mode should be explicit opt-in with filtering.
- NO_REPLY and internal housekeeping turns need deliberate suppression so background memory or maintenance activity does not leak into user-facing chats.
