## Summary

Keep open. Current `main` does not implement this timeout-validation change: the gateway-facing CLI paths still register plain `--timeout` string options and still coerce with `Number(...)`, so invalid values can still flow past the CLI boundary.

## Evidence

- **Gateway call path still lacks CLI validation:** `src/cli/gateway-cli/call.ts` still defines `.option("--timeout <ms>", "Timeout in ms", "10000")` and forwards `timeoutMs: Number(opts.timeout ?? 10_000)`, with no strict parser or early rejection.
