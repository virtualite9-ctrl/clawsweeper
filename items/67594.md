## Summary

Keep this issue open. Current main still only recognizes top-level `memory/YYYY-MM-DD.md` files for temporal decay, so dated files under `memory/` subdirectories are still treated as evergreen. The linked PR #68155 is open and unmerged, so it is an implementation candidate rather than a reason to close this issue now.

## Best Possible Solution

Review and either land or close PR #68155. If it does not land, update `extensions/memory-core/src/memory/temporal-decay.ts` so dated markdown files under `memory/` subdirectories receive temporal decay, and add focused tests in `extensions/memory-core/src/memory/temporal-decay.test.ts` for nested dated paths plus non-dated evergreen paths.
