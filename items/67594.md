## Summary

Keep this issue open. Current main still only recognizes top-level `memory/YYYY-MM-DD.md` paths for temporal decay, so dated memory files in subdirectories are still treated as evergreen. The provided context also shows open, unmerged PR #68155 with closing syntax for this issue, so the issue should remain open until that PR is reviewed and merged or closed.

## Best Possible Solution

Review and either land or close PR #68155. If it lands, close this issue as implemented by the merged PR; otherwise, update `extensions/memory-core/src/memory/temporal-decay.ts` so nested dated files under `memory/` receive decay, and add focused coverage in `extensions/memory-core/src/memory/temporal-decay.test.ts` for nested dated paths plus non-dated evergreen paths.
