## Summary

Keep open. Current main still sends Feishu streaming replies as CardKit card_id references and the inbound quoted/thread message reader does not request or parse raw CardKit content, so the reported `[Interactive Card]` symptom remains actionable. The related fix PR was closed unmerged, and its `raw_card_content`/`json_card` approach is not present on main.

## Best Possible Solution

Keep this issue open and review or rework the closed Feishu fix against current main: add `card_msg_content_type: "raw_card_content"` to both `im.message.get` and thread `im.message.list`, parse `json_card`/CardKit summary/body text before falling back to legacy element parsing, and add regression tests for quoted-message and thread-history reads of streaming CardKit cards.
