## Summary

Keep #69445 open. Current main still always renders provider usage/quota when a usage snapshot is available for chat `/status` and the `session_status` tool, and the agent config/schema has no per-agent `status.hideUsage`, section allowlist, or equivalent visibility control. Related #10005 overlaps, but it is broader status-card suppression; #69445 adds a distinct per-agent usage/billing privacy requirement that is still actionable.

## Best Possible Solution

Keep #69445 open as a concrete feature gap. The likely fix belongs in the shared `/status` and `session_status` path: add a typed, schema-validated per-agent status visibility setting such as `agents.list[].status.hideUsage` or a section allowlist, thread it through `buildStatusReply`, `buildStatusText`, and `buildStatusMessage`, preserve current behavior by default, and add focused tests plus docs for chat `/status` and `session_status`. Coordinate wording and scope with #10005 if maintainers want one canonical status-card visibility issue.
