## Summary

Current main already starts supported plugin services from the runtime plugin registry after deferred gateway plugin loading and before the gateway sidecar phase is marked ready. The remaining "late register / hot reload" variant is outside the documented native plugin lifecycle: plugin `register(api)` is synchronous and native runtime lifecycle hot reload is explicitly unsupported.

## Best Possible Solution

Close this as implemented for the supported plugin lifecycle. Keep the shipped startup ordering and synchronous registration contract; if a plugin author needs true dynamic post-start service registration, that should be opened as a separate plugin API design request rather than tracked as this 2026.4.8 startup bug.

## Evidence

  - file: [src/gateway/server-startup-post-attach.ts:293](https://github.com/openclaw/openclaw/blob/921ffad7c7a6c6f96acecc32414f7edef6bfad23/src/gateway/server-startup-post-attach.ts#L293)
