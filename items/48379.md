## Summary

Keep #48379 open. Current main still sends fixed OpenRouter attribution headers with `X-OpenRouter-Title: OpenClaw`, has no agent-aware input or per-agent OpenRouter title config, and protects those attribution header keys from caller override. The related #50206 work is only an open, generic `X-OpenClaw-Agent` proxy-header PR and does not supersede this OpenRouter app-title request.

## Best Possible Solution

Keep the issue open. A good fix would be provider-owned in the bundled OpenRouter path: use the current documented `X-OpenRouter-Title` header, derive a privacy-safe agent-aware default such as `OpenClaw/<agentId>` or add an explicit config field, wire agent context into the attribution policy/wrapper, and cover it with schema/docs/tests. If the goal is durable cost reconciliation, also consider persisting agent/provider usage metadata in session state as the issue comment suggests.
