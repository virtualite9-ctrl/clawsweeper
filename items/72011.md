## Summary

PR #72011 should stay open. Current main at 7e13f3f5140168e31027b39a9855ade3fadf24d5 does not contain the PR's streaming-preview guard: the `handleMessageUpdate` partial reply path still sends `parsedFull.text` through `onPartialReply` without a streaming-aware bare `tool_calls` JSON sanitizer. The related bug #70797 is still open in the provided context, and prior PR #71954 was closed unmerged rather than superseding this work.

## Best Possible Solution

Keep #72011 open for maintainer review. The best path is to merge a corrected streaming-preview sanitizer into `src/agents/pi-embedded-subscribe.handlers.messages.ts` and shared text sanitizer tests, or fold the same behavior into a maintainer-owned fix for #70797 while addressing the open review concerns.
