## Summary

Keep #47584 open. Current main has iOS voice wake, foreground Talk Mode, URL-scheme deep links, and an AppIntents framework link, but it does not implement a native Siri/App Shortcuts AppIntent that accepts a spoken question and returns a Siri dialog/spoken reply. The related #40874 PR is open/unmerged and only describes an App Intent flow into Talk Mode, which is not enough to supersede this more specific request.

## Best Possible Solution

Keep the issue open as the canonical request for native iOS Siri/AppIntent support. The best implementation would add an iOS `AskOpenClawIntent`/App Shortcuts surface that reuses the existing paired gateway/operator chat path, returns a spoken dialog value, and explicitly handles security, unattended execution, background limits, and tests/docs. If #40874 later lands and covers this exact parameterized Siri dialog flow, close #47584 then with that merged SHA.
