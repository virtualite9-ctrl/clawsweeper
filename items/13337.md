## Summary

Keep #13337 open. Current main at 406ae72 does not implement a Vapi telephony provider in the bundled Voice Call plugin, and the recent issue discussion already records maintainer/Codex review signals to keep it open. The adjacent Clawdia/Vapi bridge is useful external work, but it is an HTTP bridge, not `provider: "vapi"` support in `@openclaw/voice-call`, and current public plugin docs do not show a telephony-provider registration seam that would make this exact request external-only today.

## Best Possible Solution

Keep #13337 open for maintainer/product judgment or an implementation PR. A complete bundled fix would add a Vapi `VoiceCallProvider`, Vapi config/env validation, runtime provider selection, manifest UI/env metadata, focused provider/runtime tests, and docs at https://docs.openclaw.ai/plugins/voice-call. If maintainers prefer external ownership, first document the Clawdia/Vapi bridge path or add a public Voice Call telephony-provider registration seam, then track Vapi as a ClawHub/plugin item.
