## Summary

Keep #13337 open. Current main still does not contain a Vapi phone provider for the Voice Call plugin, and the public plugin surfaces do not expose a telephony-provider registration seam that would let an external plugin satisfy the requested `voicecall` config/CLI/RPC behavior. The related Clawdia Vapi bridge and closed #6838 are useful adjacent work, but they do not implement `provider: "vapi"` inside `@openclaw/voice-call`.

## Best Possible Solution

Keep #13337 open for maintainer/product judgment or an implementation PR. A bundled fix would add a Vapi `VoiceCallProvider`, config/schema and manifest metadata, env resolution and validation for Vapi keys/assistant/phone/webhook settings, runtime provider selection, focused provider/runtime tests, and Voice Call docs. If maintainers prefer external ownership, first add/document a public phone-provider capability seam or explicitly endorse the existing Clawdia Vapi bridge as the supported path.
