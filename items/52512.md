## Summary

Current `main` no longer uses IP-based local-direct fallback for Canvas/A2UI auth. Bare `/__openclaw__/canvas/` access returning `401` is now the intended behavior; Canvas is authorized via bearer auth or short-lived node-scoped capability URLs instead, so this PR no longer matches the current contract.
