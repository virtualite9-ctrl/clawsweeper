## Summary

Keep open. Current main still only partially mitigates the report: zero-valued poll defaults are allowed and Discord-native `components` are no longer advertised in the generic message-tool schema, but shared poll creation fields still live on the same tool schema as ordinary sends, populated poll-looking params still reject `action="send"`, and an open implementation PR (#46455) still references this issue with GitHub closing syntax.

## Best Possible Solution

Keep this issue open. Maintainers should review, update, land, or explicitly supersede PR #46455; the likely code path is to make poll creation fields action-aware or strip model-populated poll noise on `action="send"`, preserve deliberate poll validation on the poll path, and add regression tests for Telegram/Feishu/Discord send-with-file plus Discord empty-modal-skeleton behavior.
