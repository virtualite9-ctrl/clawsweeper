## Summary

Keep open. Current main still has the core false-positive path for non-empty or true poll-looking fields on ordinary sends, and the provided GitHub context shows open PR #46455 still references this issue with closing syntax. Main has partial mitigation for zero-valued poll defaults and no longer advertises Discord-native components in the generic message schema, but the broader schema/runtime split requested here is not implemented.

## Best Possible Solution

Keep this issue open. Maintainers should review, update, land, or explicitly supersede PR #46455 or an equivalent patch that makes poll creation fields action-aware, strips model-populated poll/component noise on action="send", preserves strict validation on the real poll/component paths, and adds focused regression coverage for Telegram/Feishu/Discord send-with-file plus Discord empty-modal skeleton behavior.
