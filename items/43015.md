## Summary

Keep open. Current main only partially mitigates the report: zero-valued poll defaults no longer trip `message.send`, and Discord-native schema exposure appears reduced, but the shared message tool still exposes poll creation fields and the send path still rejects populated poll params. Also, open PR #46455 explicitly uses closing syntax for this issue, so this issue should stay open until that implementation candidate is merged, superseded, or closed by maintainer review.

## Best Possible Solution

Keep this issue open. Maintainers should review, update, land, or explicitly close/supersede PR #46455; the likely code path is a shared message-tool/action-layer fix that scopes poll creation fields to poll actions or strips model-populated poll noise on `action="send"`, while preserving deliberate poll misuse errors and adding focused regression tests for Telegram/Feishu/Discord send-with-file cases.
