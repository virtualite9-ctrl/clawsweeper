## Summary

Close #71663 as a duplicate of open #71575. Both issues report Telegram on 2026.4.23 exposing Working/tool/raw output during tool execution; current main still defaults Telegram preview tool progress on and tests assert tool identifiers/command text in the preview. The remaining product fix should be tracked once in #71575.

## Best Possible Solution

Keep one canonical Telegram bug open in #71575. The likely fix is to make Telegram partial preview hide or strictly sanitize tool metadata by default, possibly making `streaming.preview.toolProgress` opt-in or generic-only, then update Telegram dispatch tests and docs to match the chosen product behavior.
