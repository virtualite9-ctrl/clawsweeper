## Summary

Keep the PR open. Current main still reads the OpenAI Realtime API key from process.env.OPENAI_API_KEY and sends it as the bearer token for both browser-session and bridge paths, so an unresolved keychain SecretRef would still reach OpenAI. The PR is a focused fix for the linked reproduction, with review comments that should be handled before merge rather than a reason to close it.

## Best Possible Solution

Keep #72126 open for maintainer review. The best path is to land a focused OpenAI plugin fix that resolves keychain:<service>:<account> SecretRefs before both Realtime browser-session and voice-bridge auth, adds regression coverage for those paths, and avoids permanently memoizing transient Keychain failures.
