## Summary

Keep this PR open. Current main still lacks the proposed `tools.sessions.ownedAcp.enabled` config, docs, schema/help entries, and access-control implementation. The existing session visibility guard rejects cross-agent targets unless `tools.sessions.visibility="all"`, and `sessions_send(label + agentId)` still rejects sandboxed cross-agent lookups before any creator-owned ACP exception could apply. The PR is security-sensitive session-tool access work and has unresolved review concerns around bounded `sessions_list` behavior, so the best path is maintainer review/update rather than cleanup closure.

## Best Possible Solution

Keep the PR open and review it as the active implementation candidate for creator-owned ACP session-tool visibility. The likely path is to rebase onto current main, resolve the `sessions_list` boundedness/redundant-query review concerns, then run the changed session-tool/config/docs gates before merge. If maintainers decide against this product behavior, close with that explicit product decision rather than stale cleanup.
