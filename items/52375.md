## Summary

Keep open. Current `main` does not contain the `openclaw-weixin` runtime, so the reported Weixin/Wenmobai media failure cannot be verified or declared fixed from this repo alone. The issue’s claimed core gaps are partly contradicted by current code: OpenClaw persists and reuses `accountId` for reply routing and queued recovery, and plugin-owned outbound metadata can already ride inside `ReplyPayload.channelData`. That weakens the proposed core root cause, but it is not strong enough evidence to close because the actual failing implementation lives in the external plugin.
