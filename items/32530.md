## Summary

Keep #32530 open. Current main still has explicit agent registration as the supported contract and does not implement `agents.autoDiscover`, workspace-root scanning for `agent.json` or `openclaw-agent.json`, or a plugin API that can contribute discovered agent definitions. The #27706 discussion is useful related zero-restart context, but it does not supersede this separate external workspace discovery request.

## Best Possible Solution

Keep #32530 open and narrow it around external workspace agent discovery. The best maintainer path is either a first-class `agents.autoDiscover` config contract or a documented plugin-backed agent-definition discovery API that runs before normal agent enumeration, with trusted roots, accepted filenames, schema validation, deterministic ordering, explicit precedence against `agents.list`, watch/hot-reload behavior, docs, and regression tests. Link #27706 only as related zero-restart context.

## Risks / Open Questions

- A local script or trusted plugin could approximate the workflow by rewriting `agents.list`, but that is an out-of-band workaround rather than supported dynamic discovery.
- Any implementation needs explicit trusted roots, schema validation, deterministic scan order, precedence against `agents.list`, and watch or reload behavior to avoid config injection and nondeterministic routing.
