## Summary

Keep #32530 open. Current main hot-reloads explicit `agents.list` changes, but it still lacks the requested external workspace auto-discovery surface and a stable plugin API for contributing discovered agent definitions before normal config use.

## Best Possible Solution

Keep #32530 open and narrow it around external workspace agent discovery. A strong implementation path is either a core `agents.autoDiscover` schema or a documented plugin-backed agent-discovery API with trusted roots, accepted filenames, validation, deterministic ordering, explicit precedence, hot-reload/watch behavior, docs, and tests. Link #27706 only as related zero-restart context, not as the canonical tracker for this request.

## Risks / Open Questions

- External workspace discovery needs explicit trust roots, accepted filenames, schema validation, deterministic scan order, and precedence against explicit `agents.list` to avoid unsafe config injection.
- A plugin or script can approximate this by writing config, but without a stable agent-discovery API it would not provide the requested first-class dynamic discovery contract.
