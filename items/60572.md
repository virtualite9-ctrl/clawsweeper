## Summary

Current `main` still models memory as one exclusive plugin slot, not the requested multi-slot memory architecture. `main` does now special-case one coexistence scenario by loading `memory-core` alongside another selected memory plugin when dreaming is enabled, which addresses part of the discussion, but it does not implement purpose-specific memory sub-slots or per-agent memory-slot composition.

## Evidence

- **Issue scope and discussion:** The issue asks for multiple purpose-specific memory slots (`memory.recall`, `memory.compaction`, `memory.capture`, `memory.userModel`) plus per-agent slot overrides. Follow-up discussion specifically calls out `memory-core` dreaming needing to coexist with another memory plugin such as Honcho/Mem0.
