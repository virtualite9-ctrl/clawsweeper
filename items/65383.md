## Summary

Keep this PR open. Current main still lacks iMessage default/named watcher dedupe, the linked duplicate-reply bug remains open at https://github.com/openclaw/openclaw/issues/65141, and this PR is an implementation candidate with unresolved review risks rather than work already shipped or cleanly superseded.

## Best Possible Solution

Keep this PR open for maintainer review, but ask for a narrower revision: remove the unrelated outbound target changes, implement iMessage-owned watcher dedupe on the gateway startup path or an explicit startup eligibility seam instead of plugin-wide `isEnabled`, and keep focused tests for #65141 covering duplicate default/named backends, configured versus placeholder named accounts, disabled named accounts, and distinct `cliPath`/`dbPath` backends.
