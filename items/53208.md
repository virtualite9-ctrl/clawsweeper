## Summary

Keep this issue open. Current main has the generic message tool and WhatsApp outbound delivery path for explicit targets, and it allows same-provider cross-context sends by default, but I did not find a WhatsApp-specific implementation or regression test that makes a trusted DM sender's natural-language request like “tell Jack X” reliably resolve to the main user's WhatsApp target. Adjacent closed reports cover lower-level target normalization/outbound delivery, not this conversational relay behavior.

## Best Possible Solution

Keep the issue open and add a focused WhatsApp regression around an authorized direct sender asking for a relay to another approved direct target. The fix likely belongs in the WhatsApp-owned inbound/message-tool guidance or target-alias/session-route layer, with core changes only if a generic plugin API is missing. Existing outbound target normalization and same-provider cross-context delivery should be reused rather than replaced.
