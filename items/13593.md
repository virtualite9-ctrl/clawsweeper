## Summary

Keep #13593 open. Current main has useful partial cron observability, including stored last status/error/duration, consecutive-error state, durable JSONL run history, failure alerts, and generic `/health`/`/readyz` probes. It still does not satisfy the issue's central requests: structured logging for every cron execution attempt, warnings for skipped/non-throwing failures, a cron-specific health surface, or `openclaw cron list` annotations for overdue/repeatedly failing jobs. The related PR #46501 is relevant prior art, but the provided GitHub context shows it was closed unmerged.

## Best Possible Solution

Keep #13593 open. The maintainer path is to add structured per-attempt cron execution logging for scheduled timer runs, startup catch-up/missed runs, and manual runs; warn on skipped and non-throwing error outcomes; expose cron health either at `/health/cron` or as an explicit component in the existing readiness/health model; and add `openclaw cron list` annotations for overdue and repeatedly failing jobs. PR #46501 can be mined as prior art but should not be treated as shipped.
