## Summary

Keep #13593 open. Current main has cron run state, run history, failure alerts/backoff, and generic gateway probes, but it still does not implement the issue's central observability request: structured per-attempt cron execution logging, skipped-run warnings, cron-specific health reporting, or `openclaw cron list` warning annotations. The existing issue comment already reached the same keep-open conclusion, and the related PR #46501 is closed unmerged in the provided context.

## Best Possible Solution

Keep #13593 open. The best product path is to add structured per-attempt cron execution logging across scheduled timer runs, startup catch-up, and manual runs; warn on skipped outcomes; add cron health semantics either through `/health/cron` or an explicit cron component in the existing readiness/health model; and teach `openclaw cron list` to annotate overdue and repeatedly failing jobs with focused service, CLI, probe, docs, and regression-test coverage. PR #46501 remains useful prior art, but it was closed unmerged and is not present on current main.
