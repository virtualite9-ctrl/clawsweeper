## Summary

Keep PR #33914 open. Current main still does not implement the PR's central additive plugin-hook API: `PluginHookAgentContext` lacks `sourceProvider`, `senderId`, `senderName`, `senderIsOwner`, `groupId`, and `spawnedBy`, and the normal agent plus compaction hook contexts still omit those fields. The message-hook `sessionKey` slice has landed separately, so the PR is partly stale, but the trust-aware agent-hook context gap remains real.

## Best Possible Solution

Keep this PR open for maintainer review or a narrowed rebase. The best implementation is an additive plugin SDK/context update that exposes `sourceProvider`, `senderId`, `senderName`, `senderIsOwner`, `groupId`, and `spawnedBy` on `PluginHookAgentContext`; derives normalized origin identity consistently; preserves null-vs-undefined semantics; threads the fields through normal runs, queued runs, memory flush, compaction/reset, and relevant hook emission paths; and treats the already-landed message-hook `sessionKey` work as baseline rather than reintroducing stale changes.
