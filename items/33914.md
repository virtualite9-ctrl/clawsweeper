## Summary

Keep PR #33914 open. Current main still lacks the central additive plugin-hook API requested by the PR: `PluginHookAgentContext` does not expose `sourceProvider`, `senderId`, `senderName`, `senderIsOwner`, `groupId`, or `spawnedBy`, and the main agent plus compaction hook contexts do not pass those fields. The message-hook `sessionKey` part has landed separately, so the branch needs a narrowed rebase rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review or a narrowed rebase. The best path is an additive plugin SDK/context update exposing `sourceProvider`, `senderId`, `senderName`, `senderIsOwner`, `groupId`, and `spawnedBy` on `PluginHookAgentContext`; deriving normalized origin identity consistently; preserving null semantics; threading the fields through early run, attempt, queued/memory, preflight compaction, direct compaction, and reset hook contexts; and treating the already-landed message-hook `sessionKey` work as baseline instead of reintroducing stale changes.
