## Summary

Keep PR #33914 open. Current main still does not implement the PR's central public plugin hook context expansion: `PluginHookAgentContext` lacks `sourceProvider`, `senderId`, `senderName`, `senderIsOwner`, `groupId`, and `spawnedBy`, and the normal agent and compaction hook emission paths still build contexts without those identity fields. The related message-hook `sessionKey` work has landed separately, but that only covers one stale slice of the PR, not the trust-aware agent-hook API gap.

## Best Possible Solution

Keep this PR open for maintainer review or a narrowed rebase. The best implementation is an additive plugin SDK/context update that exposes `sourceProvider`, `senderId`, `senderName`, `senderIsOwner`, `groupId`, and `spawnedBy` on `PluginHookAgentContext`; derives normalized origin identity consistently; preserves null-vs-undefined semantics; threads the fields through normal runs, queued runs, memory flush, compaction/reset, and relevant agent hook emission paths; and treats the already-landed message-hook `sessionKey` work as baseline rather than reintroducing stale changes.
