## Summary

Current `main` already resets the WhatsApp watchdog window on reconnect and includes a regression test for the exact reconnect-loop scenario from #60337. The fix is present on `main` but not in the latest tagged release `v2026.4.23`.

## Evidence

- **Fresh connection state on reconnect:** Each new WhatsApp live connection is created with a fresh `startedAt` timestamp and `lastInboundAt: null`, so a successful reconnect does not inherit the previous idle timestamp.
  - file: [extensions/whatsapp/src/connection-controller.ts:103](https://github.com/openclaw/openclaw/blob/51c11cfd903033947f01f96a7169ebf92d194064/extensions/whatsapp/src/connection-controller.ts#L103)
  - command: `nl -ba extensions/whatsapp/src/connection-controller.ts | sed -n '86,116p'`
  - sha: [51c11cfd9030](https://github.com/openclaw/openclaw/commit/51c11cfd903033947f01f96a7169ebf92d194064)
- **Watchdog baseline resets from connection start:** The watchdog now computes inactivity from `connection.lastInboundAt ?? connection.startedAt` before forcing a 499 close. That means a reconnected socket gets a new timeout window even if no new inbound message has arrived yet.
  - file: [extensions/whatsapp/src/connection-controller.ts:563](https://github.com/openclaw/openclaw/blob/51c11cfd903033947f01f96a7169ebf92d194064/extensions/whatsapp/src/connection-controller.ts#L563)
  - command: `nl -ba extensions/whatsapp/src/connection-controller.ts | sed -n '544,579p'`
  - sha: [51c11cfd9030](https://github.com/openclaw/openclaw/commit/51c11cfd903033947f01f96a7169ebf92d194064)
- **User-visible timeout logging follows the same reset baseline:** The monitor’s timeout log also uses `snapshot.lastInboundAt ?? snapshot.startedAt`, matching the fixed watchdog behavior instead of reporting minutes from the pre-reconnect idle period.
  - file: [extensions/whatsapp/src/auto-reply/monitor.ts:300](https://github.com/openclaw/openclaw/blob/51c11cfd903033947f01f96a7169ebf92d194064/extensions/whatsapp/src/auto-reply/monitor.ts#L300)

  - sha: [51c11cfd9030](https://github.com/openclaw/openclaw/commit/51c11cfd903033947f01f96a7169ebf92d194064)
- **Targeted regression test covers the reported loop:** `main` includes an end-to-end regression test named `gives a reconnected listener a fresh watchdog window`, which exercises a 499 close, reconnect, and verifies the watchdog does not immediately re-fire on the replacement listener.
