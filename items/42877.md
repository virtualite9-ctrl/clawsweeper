## Summary

Keep open. Current main still lacks the requested built-in bounded memory CRUD/write tool with hard per-store character budgets, rejected over-limit writes, substring replace/remove, duplicate handling, and credential screening. Existing memory work covers search/read, runtime excerpt limits, startup/bootstrap injection caps, append-only flush, and promotion workflows, but not a hard-enforced persisted memory-store budget.

## Best Possible Solution

Keep this open for a focused memory-core or active-memory design. Implement a structured `memory`/`memory_write` tool with config-backed per-store character budgets, atomic add/replace/remove, duplicate detection, credential-pattern screening, and over-limit responses that include usage/current entries; update docs, config schema/help, and tests. If enforcement must cover generic `write`/`edit` too, add a generic memory-store guard seam instead of hardcoding one path in unrelated tools.
