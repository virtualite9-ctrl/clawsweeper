## Summary

Keep this issue open. Current main still has memory read/search tools, runtime excerpt/injection limits, append-only flush/promotion paths, and active-memory recall orchestration, but it does not provide the requested built-in bounded memory CRUD/write tool with hard per-store character budgets, rejected over-limit writes, substring replace/remove, duplicate handling, or credential screening.

## Best Possible Solution

Keep this open for a focused memory-core or active-memory design. Add a structured `memory` or `memory_write` tool with config-backed per-store character budgets, atomic add/replace/remove semantics, duplicate detection, credential-pattern screening, and over-limit responses that include usage/current entries. Update config schema/help, memory docs, and tests; if enforcement must cover generic file writes too, add a generic memory-store guard seam instead of only hardcoding one tool path.
