## Summary

Keep open. Current `main` still has the exact behavior the PR is trying to fix: `scripts/setup-auth-system.sh` edits the tracked `scripts/systemd/openclaw-auth-monitor.service` template in place before copying it into `~/.config/systemd/user`, so the cleanup in PR #53920 is not implemented on `main` yet.
