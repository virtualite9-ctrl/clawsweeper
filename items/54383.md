## Summary

Keep this issue open. Current main still forwards the configured compaction reserve token budget into pi-coding-agent summarization without clamping it to the selected model output limit, and the provided GitHub context shows same-author PR #54392 is still open with closing syntax for this issue.

## Best Possible Solution

Review and either land PR #54392 or replace it with an equivalent maintainer-approved fix that clamps compaction summarization output budgeting against the selected model's output limit before calling pi-coding-agent generateSummary. Add regression coverage for direct Anthropic Claude 4.6 1M-context compaction, ideally proving reserveTokensFloor=300000 cannot produce max_tokens above the model maxTokens ceiling.
