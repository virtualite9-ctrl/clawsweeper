## Summary

Keep this issue open. Current main still does not expose Kimi/Moonshot usage or balance: the provider usage ID/list is closed over other providers, default status and gateway usage surfaces fan out through that list, and the bundled Kimi/Moonshot plugins register auth/catalog/runtime behavior without provider-usage hooks. The related #20343 work is useful prior art but was closed unmerged, so it is not a shipped fix.

## Best Possible Solution

Keep this issue open and implement usage support in the bundled provider boundary: add Kimi/Moonshot usage auth and snapshot fetching through `resolveUsageAuth` and `fetchUsageSnapshot`, extend or generalize usage-provider discovery so `kimi`, `kimi-coding`, and `moonshot` are queryable by `/status`, Control UI/model-auth status, `openclaw status --usage`, gateway `usage.status`, and channel status/list surfaces, then add regression tests and update https://docs.openclaw.ai/concepts/usage-tracking. Treat a standalone `/quota` command as a separate product decision unless maintainers want it in addition to the existing `/status` and `openclaw status --usage` surfaces.
