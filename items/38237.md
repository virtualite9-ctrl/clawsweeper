## Summary

Keep PR #38237 open. Current main still has the Slack draft-preview identity gap: normal Slack delivery resolves and forwards the agent identity, but `createSlackDraftStream` has no identity parameter and its initial `sendMessageSlack` call omits identity. The PR targets a real remaining bug, though it is not merge-ready as-is because its branch targets the old `src/slack` layout and includes unrelated daemon cleanup plus a separate draft-stream duplicate-race change.

## Best Possible Solution

Keep PR #38237 and linked issue #38235 open until the identity-only fix is ported cleanly to current `extensions/slack`: add optional `SlackSendIdentity` support to `createSlackDraftStream`, pass `slackIdentity` from `dispatchPreparedSlackMessage`, forward identity only on the initial draft send, and add a focused regression test in `extensions/slack/src/draft-stream.test.ts`. Split out the unrelated daemon cleanup and any already-superseded duplicate-race work before merge.
