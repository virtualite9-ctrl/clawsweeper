## Summary

Keep open. Current main still resolves and forwards Slack agent identity for normal delivery, and `sendMessageSlack` supports `SlackSendIdentity`, but the draft/preview streaming path still does not accept or pass that identity into the initial `chat.postMessage` send. The linked issue #38235 remains a valid canonical bug report, and this PR’s identity-focused change is still useful, though the branch needs rebasing/scope cleanup because it also includes unrelated or superseded changes.

## Best Possible Solution

Keep this PR or its linked issue #38235 open until the identity fix is ported cleanly to current `extensions/slack`: add an optional `SlackSendIdentity` to `createSlackDraftStream`, pass `slackIdentity` from `dispatchPreparedSlackMessage`, forward it only to the initial `sendMessageSlack` call, and add a focused regression test in `extensions/slack/src/draft-stream.test.ts`. Drop the unrelated daemon cleanup and any already-shipped duplicate-race changes from the branch before merge.
