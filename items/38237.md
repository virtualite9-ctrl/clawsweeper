## Summary

Keep PR #38237 open. Current main still resolves Slack agent identity and forwards it through normal Slack delivery, but the draft/partial preview stream constructor has no identity parameter and its initial sendMessageSlack call omits identity, so the PR targets a real remaining bug. The PR should be rebased to current extensions/slack paths and split/narrowed because its provided diff also includes unrelated daemon cleanup and a separate duplicate-race change.

## Best Possible Solution

Keep PR #38237 and linked issue #38235 open until the identity-only fix is ported cleanly to current extensions/slack paths: add optional SlackSendIdentity support to createSlackDraftStream, pass slackIdentity from dispatchPreparedSlackMessage, forward identity only on the initial sendMessageSlack call, and add a focused regression test in extensions/slack/src/draft-stream.test.ts. Rebase and split out the unrelated daemon cleanup and already-superseded duplicate-race work before merge.
