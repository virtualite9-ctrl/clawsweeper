## Summary

Current main still implements the reported behavior: fenced `MEDIA:` lines are skipped by `splitMediaFromOutput()` and preserved as text. The issue is concrete, recent enough, and the related implementation PR was closed unmerged, so it should stay open for a real parser/docs fix.

## Best Possible Solution

Keep this issue open. The likely fix belongs in `src/media/parse.ts`: either extract own-line valid `MEDIA:` directives from fenced blocks or visibly warn/strip them, while preserving existing `isValidMedia` safety checks. Add focused tests in `src/media/parse.test.ts` for backtick fences, tilde fences, invalid/prose examples, and update assistant/docs guidance to say `MEDIA:` directives must not be wrapped in code fences if the product keeps the current skip behavior.

## Risks / Open Questions

- A parser change must keep the existing media validation and path/URL safety checks, otherwise extracting from fenced examples could accidentally promote unsafe or illustrative strings into attachments.
- Changing fenced behavior should be constrained to own-line valid `MEDIA:` directives and covered for both backtick and tilde fences so ordinary code examples still render as text.
