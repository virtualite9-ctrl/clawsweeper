## Summary

Keep this issue open. Current main still explicitly skips `MEDIA:` extraction inside markdown fences, in-tree tests still assert that fenced `MEDIA:` lines remain visible text, and the related implementation PR #41980 is closed unmerged rather than shipped.

## Best Possible Solution

Keep this issue open. The best fix belongs in `src/media/parse.ts`: either extract own-line valid `MEDIA:` directives from fenced blocks or deliberately surface a warning while preserving security validation. Add focused tests in `src/media/parse.test.ts` for backtick and tilde fences, invalid/prose examples, and valid own-line directives; update assistant/docs guidance if fenced directives remain unsupported.
