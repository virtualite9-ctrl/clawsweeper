## Summary

Keep open. Current main still has no config-level default task notify policy. ACP and subagent task creation do not pass a config-derived notifyPolicy, registry/task-flow defaults still fall back to done_only unless delivery is not_applicable/silent, and docs still describe ACP/subagent defaults plus only the per-task notify command.

## Best Possible Solution

Keep this issue open and implement a typed, schema-backed task notify default under agents.defaults.tasks. Resolve the effective policy before ACP, subagent, and Task Flow records are created, preserve current defaults when unset, keep cron/CLI/media silent semantics intact, and update Background Tasks plus CLI/config docs with focused tests for ACP, subagent, cron, and the existing per-task override.
