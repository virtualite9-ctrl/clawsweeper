## Summary

Keep open. Current main still does not provide a config-level default task notify policy. ACP and subagent task creation still omit notifyPolicy, the registry/task-flow fallback remains done_only unless callers explicitly pass silent or delivery is not_applicable, and docs still only describe current defaults plus the per-task notify command.

## Best Possible Solution

Keep this issue open and implement a typed, schema-backed task notify default under agents.defaults.tasks. Resolve the effective policy before ACP, subagent, and Task Flow records are created; preserve current behavior when unset; keep cron, CLI, and media silent defaults intact; and update Background Tasks plus config/CLI docs with focused tests for ACP, subagent, cron, media, and the existing per-task override.
