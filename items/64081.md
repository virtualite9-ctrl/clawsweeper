## Summary

Keep this issue open. Current main still lacks a typed, schema-backed, or documented config-level default task notification policy, and the main ACP, subagent, and Task Flow creation paths still fall back to `done_only` unless an individual task/flow passes `notifyPolicy`. The adjacent related reports do not supersede it: #71726 covers cron silent behavior, #59375 covers cross-session notification routing, #46926 covers background exec completion, and #42438 concerns blocked DM policy seams.

## Best Possible Solution

Implement a core config/schema/docs change, likely under `agents.defaults.tasks`, that resolves an effective default notify policy before ACP, subagent, and Task Flow records are created. Preserve current behavior when unset; keep cron, CLI, and media silent defaults intact; allow per-task overrides to win; and add focused regression coverage for ACP, subagent, Task Flow, cron, media, and explicit `tasks notify` overrides.
