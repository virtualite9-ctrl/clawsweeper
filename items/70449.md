## Summary

Keep open. Current `main` still emits Gemini tool-call thought signatures only as a sibling `thoughtSignature` field in the direct Google transport, while this PR adds the nested `functionCall.thought_signature` shape it claims is required. I did not find an equivalent implementation on `main`, and the related discussion points to another still-open PR on the same problem rather than a superseding merged fix.
