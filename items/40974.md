## Summary

Keep open. Current `main` does not implement this PR's UTC normalization for inbound metadata timestamps. `buildInboundUserContextPrefix()` now delegates timestamp formatting to the shared envelope formatter, and that formatter still defaults to host-local time unless `agents.defaults.envelopeTimezone` is set to `"utc"` or another override. The current tests and docs also codify local-by-default behavior, so this PR is neither implemented-on-main nor obsolete.
