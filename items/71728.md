## Summary

Keep open. The RFC is fresh and clear, and current main only has adjacent truth mechanisms: prompt guidance to verify, memory citations, input provenance, and memory-wiki claim metadata. Those do not implement claim-level confidence tiers/source attribution for every agent statement or autonomous completion reporting for unverified claims. This is not a safe ClawHub-only close because the request cuts across core reply payloads and completion-event contracts.

## Best Possible Solution

Keep the RFC open for maintainer product/design judgment. A credible path would define a core or SDK-visible trust signal for agent output, including claim-level source/evidence metadata where available, known-vs-inferred wording guidance, and autonomous completion-report fields for unverified claims. Memory citations and memory-wiki claims can be reused as inputs, but they are not sufficient as the whole solution.
