## Summary

Keep open. Current main has adjacent truthfulness mechanisms, but not the RFC's requested claim-level confidence tiers, source attribution for every factual claim, known-vs-inferred signaling, or autonomous completion reporting for unverified claims.

## Best Possible Solution

Keep the RFC open for maintainer product/design judgment. A credible implementation would define an additive core or SDK-visible trust signal for assistant output, likely covering reply payload metadata, autonomous task-completion fields for unverified claims, channel degradation behavior, and guidance for reusing memory/wiki/tool evidence where available.
