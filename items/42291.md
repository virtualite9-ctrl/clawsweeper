## Summary

Keep open. Current main still writes `models auth login` results to the resolved local agent auth store, and the supplied GitHub context shows an open closing PR (#42381) that specifically proposes blocking OAuth login on remote node hosts. Per the review rule, this issue should stay open until that implementation candidate is reviewed and either merged or closed.

## Best Possible Solution

Keep this issue open and review the linked open PR #42381. If maintainers accept the safer behavior, land a focused guard in `src/commands/models/auth.ts` with regression coverage in `src/commands/models/auth.test.ts`; otherwise design an explicit, authenticated gateway-side auth-profile mutation API before attempting credential sync.
