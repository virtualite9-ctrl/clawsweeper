## Summary

Keep open. Current main still routes `openclaw models auth login` through the local resolved agent auth store, with no remote node-host guard or gateway credential-sync path. The provided GitHub context also shows open PR #42381 with closing syntax for this issue, so the issue should remain open until that PR is reviewed and merged or closed.

## Best Possible Solution

Keep this issue open and review PR #42381. If maintainers accept the conservative behavior, land a focused remote node-host guard in `src/commands/models/auth.ts` with regression coverage in `src/commands/models/auth.test.ts`; if they want node-host OAuth to work directly, first design a secure gateway-side auth-profile mutation API.
