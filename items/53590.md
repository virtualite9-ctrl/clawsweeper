## Summary

Current `main` partially addresses the report, but not enough to close conservatively. The exact startup symptom is improved: gateway startup now prewarms an explicit configured primary model and logs a warning if resolution fails, and unknown-model resolution now returns actionable `Unknown model: <provider>/<id>` errors instead of relying on a later opaque failure. But there is still no general `doctor` validation for configured model IDs after updates: the only doctor-side catalog check in current `main` is for `hooks.gmail.model`, and `models status --check` still exits only on auth-health conditions, not invalid configured model refs. That leaves the broader cleanup requested by #53590 still incomplete.
