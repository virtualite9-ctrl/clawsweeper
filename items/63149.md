## Summary

Keep this issue open. Current main contains generic mitigations for load, including global lane caps, per-session serialization, heartbeat wake coalescing, subagent lane caps, and lower default concurrency than the reporter cited. However, the core reported failure mode remains plausible: heartbeat turns still run as full agent turns, can still spawn subagents into the shared in-process subagent lane, and current heartbeat preflight does not account for subagent-lane backlog, CPU isolation, or heartbeat-origin admission control. The related inference-guard PR #63518 was closed unmerged and is not present on current main.

## Best Possible Solution

Keep this issue open as a core gateway/agent scheduling performance bug. The best path is to add or validate heartbeat-origin resource policy: cap or serialize subagent spawns initiated by heartbeat turns, expose measurable lane/subagent backlog and drain signals, and add a focused regression or stress test for multiple heartbeat-enabled agents spawning background subagents while a channel remains responsive.
