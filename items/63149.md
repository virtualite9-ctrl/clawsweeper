## Summary

Keep this issue open. Current main has meaningful generic mitigations for gateway work scheduling, including heartbeat coalescing, phased heartbeat timing, per-session serialization, and global main/subagent lane caps. It still does not implement the core missing behavior from the report: heartbeat-origin resource policy that accounts for shared subagent backlog or serializes/caps heartbeat-triggered subagent fanout before it can contend with the local gateway process.

## Best Possible Solution

Keep this issue open as a core gateway and agent-scheduling performance bug. The best maintainer path is to add or validate heartbeat-origin resource policy: cap or serialize subagent spawns initiated by heartbeat turns, account for shared subagent-lane backlog before starting heartbeat runs, expose lane/backlog/drain/event-loop signals for diagnosis, and add a focused stress/regression test with multiple heartbeat-enabled agents spawning background subagents while channel responsiveness remains bounded.
