## Summary

Keep this issue open. Current main has generic load mitigations, including lower default agent concurrency, a shared subagent lane cap, wake coalescing, and main/session-lane heartbeat skips. Those do not prove the reported gateway degradation is fixed: heartbeat runs are still full agent turns, heartbeat admission does not consider shared subagent-lane backlog, and subagents still share the same in-process gateway resources. The related inference-guard plugin PR #63518 was closed unmerged and no equivalent implementation is present on current main.

## Best Possible Solution

Keep this open as a core gateway and agent-scheduling performance bug. The best maintainer path is to add or validate heartbeat-origin resource policy: cap or serialize subagent spawns initiated by heartbeat turns, account for shared subagent-lane backlog before starting heartbeat runs, expose measurable lane/backlog/drain signals, and add a focused regression or stress test with multiple heartbeat-enabled agents spawning background subagents while channel responsiveness remains bounded.
