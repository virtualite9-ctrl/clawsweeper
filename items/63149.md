## Summary

Keep this issue open. Current main has generic queue and heartbeat mitigations, but it still does not add heartbeat-origin admission control for subagent fanout or account for shared subagent backlog before starting heartbeat turns. The prior ClawSweeper comment is consistent with the current checkout evidence.

## Best Possible Solution

Keep this issue open as a core gateway and agent-scheduling performance bug. The best maintainer path is to add or validate heartbeat-origin resource policy: cap or serialize subagent spawns initiated by heartbeat turns, check shared subagent-lane backlog before starting heartbeat runs, expose lane/backlog/drain/event-loop diagnostics, and add a focused stress/regression test for multiple heartbeat-enabled agents spawning background subagents while channel responsiveness stays bounded.
