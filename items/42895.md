## Summary

Keep this PR open. Current main still resolves the internal home directory as OPENCLAW_HOME, HOME, USERPROFILE, then os.homedir(), with no Android/Termux PREFIX fallback and no tests for the reported Termux case. The PR is a focused implementation candidate for the already reported crash in #42828, and the review discussion shows the author addressed the initial tilde-expansion and generic-chroot concerns.

## Best Possible Solution

Keep this PR open, refresh it against current main, and review or land the focused Termux home-resolution fix with the included regression tests. If maintainers want a stricter guard, tighten resolveTermuxHome to recognize the canonical Termux prefix shape while preserving HOME, USERPROFILE, OPENCLAW_HOME, OPENCLAW_STATE_DIR, and OPENCLAW_CONFIG_PATH precedence.
