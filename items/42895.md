## Summary

Keep this PR open. Current main still resolves OpenClaw home paths through OPENCLAW_HOME, HOME, USERPROFILE, and os.homedir() only, so the reported Android/Termux crash path remains plausible. The PR is a focused implementation candidate for the linked crash report and includes targeted regression coverage for the missing fallback and tilde-expansion edge case.

## Best Possible Solution

Keep this PR open, refresh it against current main if needed, and review or land the focused Termux home-resolution fix with its regression tests. Preserve existing OPENCLAW_HOME, HOME, USERPROFILE, OPENCLAW_STATE_DIR, and OPENCLAW_CONFIG_PATH precedence, and consider tightening the Termux PREFIX guard while keeping the tilde-expansion and generic-chroot tests.
