## Summary

Keep #46398 open. Current main still does not implement a persistent Control UI/WebChat token usage indicator; it only has a high-threshold context warning near the composer and a draft-length token estimate. The linked implementation PR #48706 was closed unmerged, and the duplicate issues #45048 and #50071 were closed specifically to centralize the remaining work on #46398.

## Best Possible Solution

Keep #46398 as the canonical tracker and implement a focused WebChat/Control UI indicator that reads the active session's fresh `totalTokens` and `contextTokens`, displays exact/compact usage persistently after token data exists, handles stale or missing context limits gracefully, and adds focused UI tests proving low-usage sessions are visible while stale/empty data is hidden or degraded. The already-closed #45048 and #50071 should remain duplicates of this issue.
