## Summary

Current `main` now resolves provider auth from stored profiles when no explicit auth order exists, and the runtime uses that resolver for model execution. That matches the issue's proposed Option B, so the `openai-codex` OAuth login path no longer depends on a manual `models auth order set` step on `main`.

## Evidence

- **Resolver now falls back to stored profiles without explicit order:** `resolveAuthProfileOrder()` builds `baseOrder` from explicit order if present, otherwise from configured auth profiles, otherwise from `listProfilesForProvider(store, provider)`. That directly covers the reported `order: {}` case.
