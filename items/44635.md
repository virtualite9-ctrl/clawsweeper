## Summary

Keep #44635 open. Current main has useful named browser-profile support, including per-profile CDP URLs and existing-session userDataDir, but it does not implement per-agent browser profile isolation/defaults/access control. The original agents.list[].browser shape remains invalid, local managed userDataDir remains derived from profile name rather than configurable, and commenters' cdpUrl/programmatic-profile concerns are still not fully addressed.

## Best Possible Solution

Keep the issue open as the canonical per-agent browser isolation request. A good implementation would keep browser.profiles as the transport/profile map, add an explicit per-agent browser binding such as agents.list[].browser.defaultProfile plus an allowProfiles policy, cover cdpUrl and existing-session profiles, decide whether local managed userDataDir should remain derived or become configurable, and add schema, browser-tool, docs, and regression tests for unrecognized-key and cross-agent access cases.
