## Risks / Open Questions

- Security review: this PR adds Docker build-time execution of Homebrew's remote install script from HEAD without pinning or integrity verification; that may be acceptable only with explicit maintainer approval and clear documentation because it changes the supply-chain behavior of local Docker builds.
- Security/reliability review: unconditional HOMEBREW_* ENV values and PATH changes would affect images built without brew and could mislead runtime detection or operators.
- Functional review: the Homebrew install command quoting is likely to break because the fetched script is expanded into a single-quoted bash -c string.
- Functional review: the PR appears based on an older Dockerfile shape and reintroduces runtime-stage source COPY/build steps that conflict with current main's multi-stage Dockerfile design.
- Functional review: the docker-setup.sh diff uses raw docker build instead of current run_docker_build/docker_build_exec wrapper, which would drop the BuildKit/buildx/cache behavior currently centralized in scripts/lib/docker-build.sh.
- Functional review: OPENCLAW_SKIP_BUILD failure text references a nonexistent openclaw build command.
