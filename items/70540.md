## Summary

Keep #70540 open. Current main still exposes only memory_search and memory_get from memory-core; memory_add is not registered, documented, listed in the tool catalog, or handled by the gateway memory-tool path. The PR is a focused draft implementation for the related memory_add proposal rather than obsolete cleanup.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is a reviewed memory-core implementation that adds memory_add as a bundled plugin tool, updates the tool schema/catalog/group policy/docs/display surfaces, handles gateway invocation correctly for a plugin-backed catalog tool, and includes block-aware daily memory write/indexing tests.
