## Summary

Keep this PR open. Current main still selects a persisted stale past-due `nextRunAtMs` as runnable before checking whether `lastRunAtMs` already covers that slot, and the PR is a narrow implementation candidate with a matching regression test and no apparent security-sensitive diff scope.

## Best Possible Solution

Keep this PR open and review or land it, or port the same stale-slot guard into current main's `isRunnableJob`/startup catch-up runnable path with the PR's regression test. The linked bug is closed, but current main still appears to need this focused scheduler fix.
