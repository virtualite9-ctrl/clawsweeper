## Summary

Keep this PR open. Current main still selects an already-executed stale past-due cron slot as runnable during gateway startup catch-up, and the PR is a narrow implementation candidate with a matching regression test and no apparent security-sensitive diff scope.

## Best Possible Solution

Keep this PR open and review or land it, or port the same stale-slot guard into current main's `isRunnableJob`/startup catch-up runnable path with the PR's regression test. The linked bug #42640 is closed, but current main still appears to need this focused scheduler fix.
