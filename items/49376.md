## Summary

Keep open. Current `main` has a narrower overloaded-error mitigation path: fixed backoff before auth-profile rotation and then model fallback. It does not implement the requested agent/global configurable LLM retry policy with per-error selection, attempt arrays, or total-time caps, and the only array-based transient retry config documented today is `cron.retry` for one-shot cron jobs.

## Evidence

- **Only auth cooldown knobs exist for main LLM transient handling:** The exposed config for provider transient handling on the agent run path is `auth.cooldowns.overloadedProfileRotations`, `auth.cooldowns.overloadedBackoffMs`, and `auth.cooldowns.rateLimitedProfileRotations`; there is no `llmRetry` or equivalent agent/global retry object in the typed auth config.
  - file: [src/config/types.auth.ts:39](https://github.com/openclaw/openclaw/blob/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4/src/config/types.auth.ts#L39)

This is not the proposed configurable retry-with-backoff loop for the same request path.
