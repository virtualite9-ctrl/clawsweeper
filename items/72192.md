## Summary

#72192 requests the same remaining typing TTL work already tracked by open PR #72009: make the reply typing TTL configurable and raise the default from 2 minutes to 5 minutes. Current main still hardcodes the auto-reply typing controller TTL and only exposes typing mode/interval config, so this is not implemented-on-main; the right cleanup is to close this newer issue as a duplicate of the canonical open implementation thread.

## Best Possible Solution

Close #72192 as a duplicate of open PR #72009 and keep design, review, schema, docs, and tests for configurable typing TTL centralized there. The eventual product fix should expose a validated typingTtlSeconds-style config in agent defaults/session config, wire it into the reply typing controller, document it on the public typing indicators page, and cover the default/override behavior in tests.
