## Summary

Keep #45134 open. Current main has commentary/final_answer suppression, but it does not implement the issue's stopReason-gated buffering for ordinary unphased pre-tool text. The current code and tests still intentionally flush unphased block-reply text before tool execution, and final payload assembly still consumes accumulated assistantTexts.

## Best Possible Solution

Keep this open unless maintainers deliberately dedupe it into the broader open #25592 tracker. The product fix should add channel-safe pre-tool suppression for block replies and final payload assembly: buffer onBlockReply text until message_end stopReason is known, discard toolUse-turn buffers, remove or slice intermediate assistantTexts, and add regression coverage for Mattermost/text_end plus embedded CLI/final-payload behavior. Preserve the existing commentary-phase suppression as a subset, but do not treat it as the full fix.
