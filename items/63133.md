## Summary

Keep open. Current main still has the reported behavior: the lean `status --json` path skips memory collection by returning `memory: null`, but there is no collection-state marker, and the shared status formatter still renders every enabled/null memory snapshot as `unavailable`. The PR’s final diff is a focused implementation candidate that adds explicit checked/skipped state and regression coverage, and the earlier review concern appears addressed by reserving `not checked` for the explicit skipped path.

## Best Possible Solution

Keep this PR open for maintainer review/rebase and normal CI. The best code path is to land this focused `memoryCollection` plumbing, or reimplement the same idea on main: lean `status --json` without `--all` should mark memory collection as skipped/not checked, while attempted probe failures should continue to render as unavailable.
