## Summary

Keep open. Current main still has single provider/model/store memory search plus active-model filtering and fallback-by-reindex; it does not implement first-class multi-index embedding memory, per-index ingestion state, runtime read failover across prebuilt vector spaces, or per-index status/metrics. The related #55590 is closed but does not supersede this more precise remaining design request.

## Best Possible Solution

Keep the issue open and design this as memory-core or memory plugin-contract work: add backward-compatible multi-index config, a canonical chunk source with per-index ingestion/backfill state, an active/fallback index router that never mixes vector spaces in one query, per-index status/metrics for coverage and lag, and tests/docs that preserve current single-index configs plus existing `memorySearch.fallback` full-reindex semantics.
