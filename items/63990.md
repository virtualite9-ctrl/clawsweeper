## Summary

Keep open. Current `main` still models memory search as one embedding provider/model with a single optional provider fallback and one configured store path, so the requested multi-index/model-aware failover is not implemented. The issue is coherent, recent, and supported by concrete production context in the discussion, even if the eventual implementation may belong in the active memory plugin layer rather than a repo-wide core change.

## Evidence

- **Memory config docs still expose a single provider/model plus one fallback:** The public memory config reference documents `agents.defaults.memorySearch.provider`, `model`, and `fallback` only; there is no first-class `indexes[]`, `activeIndex`, or per-index store contract.
