## Summary

Keep this issue open. Current main still has a single resolved memory embedding provider/model/store per agent and a fallback path that switches provider then performs a full reindex. It does not provide first-class `indexes[]`/`activeIndex`, independent per-index ingestion state, read failover between ready indexes, or the requested per-index observability.

## Best Possible Solution

Keep the issue open for maintainer design. The likely path is additive memory-core plus public memory/plugin contract work: add backward-compatible `indexes[]`/`activeIndex` config, preserve a canonical chunk source, build per-index ingestion/backfill state, route reads to exactly one selected index with controlled fallback, expose per-index status/metrics, and keep existing single-index configs and full-reindex fallback behavior working during migration.
