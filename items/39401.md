## Summary

Keep #39401 open. Current main still renders the Agents overview directly without the proposed `keyed(selectedAgent.id, ...)` remount boundary or another selected-agent reset for the overview model controls, and the linked bug #39392 remains a concrete open repro. Related PR #39495 is closed unmerged, so it is not a shipped superseder.

## Best Possible Solution

Keep #39401 open or nominate a fresh canonical PR for #39392. The best implementation path is to add a per-selected-agent remount/reset boundary around the Agents overview model controls, then add focused regression coverage that switches between agents with different model settings and verifies the displayed primary/fallback controls reflect the selected agent.
