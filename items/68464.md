## Summary

Keep this PR open. Current main does not implement the central changes from the PR: completion-mode dispatch is still direct-first, subagent run records do not persist a delivery claim or stored user-safe delivery payload, and iMessage outbound sanitization still lacks the proposed OpenClaw internal-context stripping and code-fence normalization. The PR also has unresolved functional review concerns, so the right maintainer action is review/rework rather than cleanup close.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to rebase it onto current main, address the unresolved review findings, and decide whether to merge it as one hardening change or split it into focused patches: persisted delivery claim/idempotency, user-safe payload extraction, queue-first active-parent completion delivery, and iMessage outbound normalization. Coordinate with #69767 and #65037 so the durable completion-delivery work is not duplicated.
