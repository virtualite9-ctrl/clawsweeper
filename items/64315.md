## Summary

Current main still does not implement optional marquee or overflow-scrolling behavior for long status information. The request remains a valid product/UI enhancement for compact Control UI and TUI status surfaces, not an already-shipped fix, duplicate, or ClawHub-only item.

## Best Possible Solution

Keep this issue open as a product/UI enhancement. The best path is a reusable accessible overflow-status component/helper, followed by explicit opt-in adoption in selected Control UI and TUI status/footer surfaces. Treat #13670 and similar model/token visibility work as related status-content work, not a replacement for overflow behavior.
