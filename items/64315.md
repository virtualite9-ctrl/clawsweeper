## Summary

Keep this issue open. Current main still does not provide an optional marquee/overflow-scrolling status component or behavior for the Control UI, TUI header/status/footer, or plugin status surfaces. Existing code either renders static text, hides compact labels, uses ellipsis, or truncates long status content.

## Best Possible Solution

Keep this issue open as a product/UI enhancement. The best path is a reusable accessible overflow-status component/helper, then explicit opt-in use in selected Control UI and TUI status/footer surfaces. It should render statically when content fits, scroll only on real overflow, respect reduced-motion preferences, pause or allow user control on hover/focus, and coordinate with #13670 and #42788 for model/token/session label visibility work.
