## Summary

Current main still does not implement the requested optional marquee/overflow-scrolling behavior for long status information. The relevant Control UI and TUI surfaces render static labels, select text, wrapped notices, ellipsized text, hidden compact labels, or plain TUI Text nodes rather than overflow detection with delayed horizontal scrolling, hover/focus pause, reduced-motion handling, or manual horizontal fallback.

## Best Possible Solution

Keep this open as a product/UI enhancement. The best path is a reusable accessible overflow-status component/helper, then explicit opt-in use in selected Control UI and TUI status/footer surfaces. It should render statically when content fits, scroll only on real overflow, respect reduced-motion preferences, pause or allow user control on hover/focus, and treat #13670 as related but narrower model/token visibility work rather than a replacement for this request.
