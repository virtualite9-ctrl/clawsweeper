## Summary

Keep this PR open. The linked bug is still real on current main, but the submitted patch is not safe as-is: it uses `plugins.entries.<id>` presence to distinguish plugins from built-in chat commands, which would suppress the existing `plugins.allow` diagnostic for real bundled plugin commands that do not have explicit config entries.

## Best Possible Solution

Keep this PR open for revision or replace it with a safer superseding PR. The fix should use manifest/registry knowledge to distinguish real plugin ids from core chat commands, preserve the existing blocked-plugin diagnostic for `browser`, and add focused tests for both `commands` under restrictive `plugins.allow` and a real plugin command excluded by the allowlist.
