## Summary

Keep this PR open. Current main still has the misleading built-in chat-command `plugins.allow` diagnostic, but the submitted patch is not safe as-is because it treats absence from `plugins.entries` as evidence that a command is not a real plugin command.

## Best Possible Solution

Revise this PR, or replace it with a superseding focused PR. The fix should distinguish real plugin ids and manifest command aliases using registry/manifest knowledge instead of `plugins.entries` presence, preserve the existing `browser` allowlist diagnostic and `openclaw <command>` wording, and add tests for both `openclaw commands` under restrictive `plugins.allow` and a real plugin command excluded by the allowlist.
