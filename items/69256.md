## Summary

Keep open. Current main still has the direct-delivery cleanup path deleting `deleteAfterRun` isolated cron sessions without checking active descendant subagent runs. The PR addresses a plausible real bug, but review comments correctly identify that the patch also needs regression coverage and a second-stage cleanup path so `deleteAfterRun` is not skipped forever after descendants drain.

## Best Possible Solution

Keep the PR open and ask for an updated fix: direct structured/threaded cleanup should preserve the parent isolated cron session while descendant runs are active, then reliably retry/delete once the descendant work has drained. Add focused coverage in `src/cron/isolated-agent/delivery-dispatch.double-announce.test.ts` for active-descendant structured/threaded direct delivery plus the eventual cleanup behavior.
