## Summary

Keep #69256 open. Current main still has the direct-delivery cleanup bug the PR targets: structured/threaded direct delivery routes through `deliverViaDirectAndCleanup()`, whose `finally` calls `cleanupDirectCronSessionIfNeeded()`, and that helper currently deletes `deleteAfterRun` sessions without checking active descendant runs. However, the PR as written is not ready to close as implemented or superseded because review comments correctly identify missing regression coverage and a likely second-stage cleanup gap after descendants drain.

## Best Possible Solution

Keep the PR open but require an amended fix: add a descendant-aware guard for direct cleanup, add regression coverage for structured/threaded direct delivery with active descendants, and implement or reuse a second-stage cleanup path so `deleteAfterRun` sessions are deleted after descendant runs settle rather than being preserved indefinitely.
