## Summary

Keep #69256 open. Current main still has the direct-delivery cleanup path the PR targets, and it still deletes `deleteAfterRun` sessions without checking active descendants. The PR is plausible but needs revision: existing review comments correctly call out missing regression coverage and the need for a second-stage cleanup path after descendants drain.

## Best Possible Solution

Keep the PR open for revision. The robust fix should prevent direct cleanup while descendants are active, add a targeted regression for structured/threaded direct delivery with active descendants, and add or reuse a second-stage cleanup path after descendant runs drain so `deleteAfterRun` still eventually deletes the cron-owned session.
