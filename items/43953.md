## Summary

Keep this PR open. Current main still does not pass Feishu card-action `input_value`, `form_value`, `name`, `option`, or `options` through to the synthetic message path, and the paired same-author issue #43202 remains open. The PR is a plausible implementation candidate, but it is incomplete against current main because the production webhook parser still reconstructs `action` with only `value` and `tag`, so live callbacks would drop the new fields before the handler sees them.

## Best Possible Solution

Keep the PR open and ask for a refresh against current main: extend the production parser and event type to preserve Feishu `input_value`, `form_value`, `name`, `option`, and `options`; add parser-to-handler regression tests for those fields; and preserve existing structured-envelope plus legacy command/text extraction behavior for button and select callbacks.
