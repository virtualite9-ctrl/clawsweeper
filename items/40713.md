## Summary

Keep open. Current `main` still parses forwarded Discord text, but the inbound Discord path intentionally builds `baseText` with `includeForwarded: false` and then uses that `baseText` for `BodyForAgent`, `RawBody`, and `CommandBody`. For a forwarded-only message, `baseText` is `""`, so the agent-facing prompt body stays empty even though `messageText` includes the forwarded content.

## Evidence

- **Inbound preflight still excludes forwarded content from base text:** `preflightDiscordMessage` computes `baseText` with `resolveDiscordMessageText(message, { includeForwarded: false })`. That is the clean inbound text later threaded into agent-facing context.
  - file: [extensions/discord/src/monitor/message-handler.preflight.ts:574](https://github.com/openclaw/openclaw/blob/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503/extensions/discord/src/monitor/message-handler.preflight.ts#L574)
  - command: `nl -ba extensions/discord/src/monitor/message-handler.preflight.ts | sed -n '570,578p'`
  - sha: [b0d2003fe65b](https://github.com/openclaw/openclaw/commit/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503)
- **Forwarded resolver exists and is enabled for messageText:** The same preflight path separately computes `messageText` with `includeForwarded: true`, and `resolveDiscordMessageText()` appends forwarded snapshot/reference text only when that option is true.
  - file: [extensions/discord/src/monitor/message-handler.preflight.ts:449](https://github.com/openclaw/openclaw/blob/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503/extensions/discord/src/monitor/message-handler.preflight.ts#L449)
  - command: `nl -ba extensions/discord/src/monitor/message-handler.preflight.ts | sed -n '446,452p'`
  - sha: [b0d2003fe65b](https://github.com/openclaw/openclaw/commit/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503)
- **Agent-facing payload still prefers baseText over forwarded-inclusive text:** `finalizeInboundContext` is called with `BodyForAgent: baseText ?? text`, `RawBody: baseText`, and `CommandBody: baseText`. Because `??` does not fall back on empty string, a forwarded-only message keeps `BodyForAgent` as `""` instead of using forwarded-inclusive `text`.
  - file: [extensions/discord/src/monitor/message-handler.process.ts:488](https://github.com/openclaw/openclaw/blob/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503/extensions/discord/src/monitor/message-handler.process.ts#L488)
  - command: `nl -ba extensions/discord/src/monitor/message-handler.process.ts | sed -n '486,494p'`
  - sha: [b0d2003fe65b](https://github.com/openclaw/openclaw/commit/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503)
- **Prompt/session path prefers BodyForAgent over wrapped Body:** `finalizeInboundContext()` keeps explicit `BodyForAgent`, and session initialization sets `BodyStripped` from `BodyForAgent` before `Body`. That means the main agent prompt path uses the empty clean body instead of the envelope-wrapped `Body` that still contains forwarded text.
