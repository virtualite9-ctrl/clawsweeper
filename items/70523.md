## Summary

Keep PR #70523 open. Current main does not implement the PR’s fallback-aware daily-memory suppression: light/REM phase output is still written inline to `memory/YYYY-MM-DD.md` before narrative fallback runs, the narrative helper returns `void`/`null` rather than a caller-visible fallback result, and existing tests still assert raw `Candidate:` inline output. The PR discussion shows active follow-up on review findings and a concrete linked bug (#70509), while the related PRtags group does not identify a merged or clearly canonical replacement for this narrow daily-memory fallback fix.

## Best Possible Solution

Keep #70523 open for maintainer review or consolidation. The best path is to decide the canonical dreaming-diary cleanup branch, remove unrelated docs churn from this PR, preserve the primary phase-write ordering guarantees, and land a targeted fallback-suppression fix with regression coverage for light and REM daily files; if #70332 or another branch becomes canonical, close #70523 only then as superseded with that link.
