## Summary

Keep #38076 open. Current main still parses `init_skill --resources` case-sensitively: valid mixed-case values are rejected before deduplication, and the CLI passes the raw `--resources` string into that parser. The related fix PRs in the provided context are closed and unmerged, and current tests do not cover this behavior.

## Best Possible Solution

Implement the narrow parser fix in `skills/skill-creator/scripts/init_skill.py`: normalize each nonempty `--resources` token to lowercase before validation and deduplication, keep the existing lowercase output directory names and unknown-resource error behavior, and add focused `skills/skill-creator/scripts/test_init_skill.py` coverage for mixed-case acceptance, lowercase deduplication, and invalid-resource rejection.
