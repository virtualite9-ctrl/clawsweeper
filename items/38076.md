## Summary

Keep #38076 open. Current main still treats `init_skill --resources` as case-sensitive: `ALLOWED_RESOURCES` contains lowercase tokens, `parse_resources()` validates stripped raw input before any normalization, and `main()` passes `args.resources` directly into that parser. A read-only parser check against current main accepts lowercase values but rejects `Scripts,References,ASSETS`, and no focused `test_init_skill.py` regression test is present. The related PRs in the provided context are closed and unmerged, so there is no shipped or canonical fix to close against.

## Best Possible Solution

Land the narrow skill-creator fix on main: normalize each nonempty `--resources` token to lowercase before validation and order-preserving deduplication, keep lowercase resource directory names and the existing unknown-resource error behavior, and add focused `skills/skill-creator/scripts/test_init_skill.py` coverage for mixed-case acceptance, lowercase deduplication, and invalid-resource rejection.
