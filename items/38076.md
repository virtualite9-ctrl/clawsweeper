## Summary

Keep #38076 open. Current main still parses `init_skill --resources` case-sensitively: the allowed resource set is lowercase-only, `parse_resources()` validates stripped tokens before any lowercasing, and `main()` passes the raw CLI string directly into that parser. A bytecode-disabled CLI check on current main still rejects `Scripts,References,ASSETS`, and the linked fix PRs in the provided context are closed and unmerged.

## Best Possible Solution

Land the narrow parser fix in `skills/skill-creator/scripts/init_skill.py`: normalize each nonempty `--resources` token to lowercase before validation and order-preserving deduplication, preserve lowercase output directory names and the existing unknown-resource error path, and add focused `test_init_skill.py` coverage for mixed-case acceptance, lowercase deduplication, and invalid-resource rejection.
