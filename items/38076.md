## Summary

Keep #38076 open. Current main still rejects mixed-case `init_skill --resources` values because `parse_resources` strips tokens but does not lowercase them before validating against the lowercase-only allowlist. The CLI passes the raw `--resources` string into that parser, there is no focused regression coverage, and the related PRs in the provided GitHub context are closed and unmerged.

## Best Possible Solution

Implement the narrow parser fix in `skills/skill-creator/scripts/init_skill.py`: lowercase each nonempty `--resources` token before validation and deduplication, preserve the existing unknown-resource error behavior, and add focused `skills/skill-creator/scripts/test_init_skill.py` coverage for mixed-case accepted values, lowercase deduped output, and unknown-resource rejection.
