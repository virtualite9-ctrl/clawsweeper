## Summary

Current main still has partial transcript hygiene, not the requested first-party reopen/startup compensation seam. Missing tool results are repaired in replay/live-write paths, and orphan user tails are merged into the next prompt or resumed, but the persisted session file is not repaired append-only with tail-only synthetic toolResult rows or an explicit assistant recovery marker.

## Best Possible Solution

Keep this issue open. The best fix is a narrow first-party reopen/startup compensation seam near session-file open/load that inspects only the active transcript tail, appends synthetic error toolResult rows for unresolved eligible tail tool calls, appends one explicit assistant recovery marker for a persisted orphan-user tail, never rewrites or deletes existing rows, is idempotent across repeated reopen, and adds focused tests for missing tail result, partial tail results, orphan-user tail, healthy no-op, and repeated reopen no-op.
