## Summary

Keep #8147 open. Current main still sends Control UI/WebChat messages on plain Enter, with only Shift+Enter and IME composition exemptions. There is still no Cmd/Ctrl+Enter send mode, no chat send-shortcut prop, no persisted UI preference, and no `gateway.webchat` config/docs option. The related context reinforces keeping it open: the maintainer/Codex comment already marked #8147 as the canonical tracker, #39821 was closed as a duplicate, #38363 was closed unmerged, and #19859 is an overlapping open follow-up rather than a shipped fix.

## Best Possible Solution

Keep #8147 open as the canonical tracker. A complete fix should add a typed Control UI/WebChat send-shortcut preference or make an explicit product decision to change the default, persist it in the UI/settings surface, update composer placeholder and send-button shortcut hints, document any new config/user setting, and add focused tests for plain Enter, Shift+Enter, IME composition, and Cmd/Ctrl+Enter behavior.
