## Summary

Keep #8147 open. Current main still sends WebChat/Control UI messages on plain Enter, only exempts Shift+Enter and IME composition, and has no persisted UI or gateway WebChat setting for Cmd/Ctrl+Enter sending. The related context does not supersede it: PR #38363 was closed unmerged, #39821 was closed as a duplicate of this issue, and #19859 is an overlapping open follow-up rather than proof this item is fixed.

## Best Possible Solution

Keep #8147 open as the canonical tracker. A complete fix should add a typed Control UI/WebChat send-shortcut preference or intentionally change the default to Cmd/Ctrl+Enter, persist the setting, update composer placeholder and send-button shortcut hints, document the behavior if it becomes configurable, and add focused tests for plain Enter, Shift+Enter, IME composition, and Cmd/Ctrl+Enter.
