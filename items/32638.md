## Summary

Keep #32638 open. Current main still does not provide a generic provider-native reasoning value contract: the OpenAI-compatible resolver filters reasoning values to OpenAI-style efforts, the Groq special case is limited to one exact model and cannot preserve Groq's `default` value through the resolver, and LM Studio discovery still drops `allowed_options` such as `on`/`off`.

## Best Possible Solution

Keep the issue open and fix reasoning compatibility at the provider/model contract level. The implementation should allow provider-native request values outside the OpenAI effort enum, preserve discovered LM Studio `allowed_options` such as `on`/`off`, apply Groq `none`/`default` handling across affected Groq reasoning models, and add focused request-building regression tests for Groq and LM Studio OpenAI-compatible completions.
