## Summary

Keep open. Current `main` still does not accept Telegram `@bot` suffixes for inline directives, so the reported bug remains real, and the proposed PR likely needs revision rather than closure because inline parsing still has no bot-username validation context.

## Evidence

- **`/model` inline parsing still rejects Telegram suffixes:** `extractModelDirective()` still matches only bare `/model` and bare alias tokens; there is no optional `@bot` suffix in either regex, so `/model@BotName ...` will not be recognized on current `main`.

## Risks / Open Questions

- The underlying Telegram group bug from #40637 still appears unresolved on current `main` for inline directives like `/model@BotName`, `/think@BotName`, `/exec@BotName`, and `/queue@BotName`.
- The PR as proposed likely needs a follow-up change to validate `@bot` suffixes against the running bot username; otherwise multi-bot group chats could apply directives aimed at another bot.
