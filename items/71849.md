## Summary

Keep #71849 open. Current main still exposes only the shared `openclaw_agent_consult` path for Voice Call realtime; it does not implement a `realtime.fastContext` config surface, bounded direct memory/session lookup, partial-transcript accumulation for context detection, or explicit fast-context miss result. The maintainer comment already says to keep this open, and related PR #72237 is open/unmerged and addresses consult thinking/fast-mode overrides, not the requested fast memory context hot path.

## Best Possible Solution

Keep the issue open and implement the fast-context path inside the Voice Call plugin: add a default-off `realtime.fastContext` schema/manifest/docs surface, accumulate and normalize partial user transcript chunks, run a hard-deadline direct memory/session-context lookup using existing memory host/search primitives, inject either top snippets or an explicit miss into the realtime session, and preserve `openclaw_agent_consult` as the slower fallback. Add focused config, runtime, realtime-handler, and docs coverage.
