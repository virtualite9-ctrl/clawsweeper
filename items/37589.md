## Summary

Keep #37589 open. Current main still recreates array-backed display message objects and text parts even when no inline directive tags are stripped, and the related PRs called out in the issue context are closed and unmerged rather than a canonical shipped fix.

## Best Possible Solution

Implement the narrow optimization in `src/utils/directive-tags.ts`: reuse unchanged text parts, track whether any text changed, return the original message object when nothing was stripped, and add focused `toBe` regression tests for unchanged array messages and mixed changed/unchanged parts.

There is no early return when no text changed.
