## Summary

Keep #37589 open. Current main still recreates array-backed message objects and text parts in `stripInlineDirectiveTagsFromMessageForDisplay` even when `stripInlineDirectiveTagsForDisplay` reports no change, and the linked fix PRs are closed unmerged.

## Best Possible Solution

Implement the narrow identity-preservation fix in `src/utils/directive-tags.ts`: reuse unchanged text-part objects, track whether any text block changed, and return the original message object when nothing was stripped. Add focused Vitest coverage with `toBe` assertions for unchanged array-backed messages and mixed changed/unchanged text parts; alternatively, deliberately de-export/remove the helper if maintainers decide it should not remain plugin-facing.
