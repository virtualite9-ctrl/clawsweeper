## Summary

Keep #37589 open. Current main still recreates array-backed message objects and every text part in `stripInlineDirectiveTagsFromMessageForDisplay` even when no inline directive tags are stripped, while the linked fix PRs are closed unmerged.

## Best Possible Solution

Implement the narrow identity-preservation fix in `src/utils/directive-tags.ts`: preserve unchanged text-part object references, track whether any text block changed, and return the original message object when no directives were stripped. Add focused Vitest coverage with `toBe` assertions for unchanged array-backed messages and for mixed changed/unchanged text parts; if maintainers decide this helper should not stay plugin-facing, de-export or remove it deliberately instead.
