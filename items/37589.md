## Summary

Keep #37589 open. Current main still rebuilds array-backed messages and every text part in `stripInlineDirectiveTagsFromMessageForDisplay` even when no inline directive tag is stripped, while the referenced implementation PRs are closed and unmerged.

## Best Possible Solution

Implement the narrow identity-preservation fix in `src/utils/directive-tags.ts`: reuse unchanged text-part objects, track whether any text block changed, and return the original message object when nothing was stripped. Add focused Vitest coverage with `toBe` assertions for unchanged array-backed messages and mixed changed/unchanged text parts. If maintainers decide the exported helper is obsolete, remove or de-export it deliberately from the plugin text runtime and update API/test expectations.
