## Summary

Keep this PR open. Current main still does not implement configurable `memorySearch.remote.encodingFormat` for memory embedding requests, and the PR is a relevant implementation candidate, but it is not ready to close or merge as-is: current main has moved the active implementation into `src/memory-host-sdk` and bundled provider plugins, and the PR diff still misses the current config merge path that feeds resolved memory settings.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to rebase the configurable `encodingFormat` design onto current `src/memory-host-sdk` and provider-plugin architecture, add the field to schema/types/labels/docs and the resolved memory config merge path, propagate it through inline and batch embedding requests, decode base64 responses safely, and add targeted tests. Maintainers can then decide whether this broader PR supersedes the narrower #41670 or should be split into a minimal float-only compatibility fix plus a separate base64 feature.
