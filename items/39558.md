## Summary

Keep #39558 open. Current main still generates Feishu synthetic reaction message IDs with the `:reaction:<emoji>:<uuid>` suffix and still passes raw message IDs into Feishu API `message_id` path parameters. The PR remains a plausible fix for the open canonical bug #34528, with useful prior Greptile review support, but it needs a current-main refresh because newer Feishu surfaces such as `streaming-card.ts` are not covered and `send-message.ts` no longer exists on main.

## Best Possible Solution

Refresh #39558 or replace it with an equivalent Feishu plugin patch that centralizes stripping of OpenClaw's synthetic `:reaction:<emoji>:<uuid>` suffix at every Feishu API-facing `message_id` boundary while preserving suffixed IDs for internal dedup/session identity. Cover send/reply/get/edit/update, reactions, typing indicators, media resource and media replies, action/thread paths, and streaming-card replies, then add regression tests for the helper plus at least one API boundary receiving a suffixed reaction message ID.
