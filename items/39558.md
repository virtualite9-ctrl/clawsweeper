## Summary

Keep open. Current `main` still generates Feishu synthetic reaction message IDs with `:reaction:<emoji>:<uuid>` suffixes and still passes raw `messageId`/`replyToMessageId` values into Feishu message APIs across send/get/reaction/typing/media paths, so this is neither implemented on `main` nor obsolete. The checkout also has no `extensions/feishu/src/message-id.ts` utility from the PR. If anything, current `main` has an extra raw reply path in `extensions/feishu/src/streaming-card.ts` that would need the same normalization.
