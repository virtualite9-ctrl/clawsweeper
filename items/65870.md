## Summary

Current main still has the reported Brave Search docs URL bug: both Brave provider surfaces point at the legacy docs path, the docs config redirects that path to the canonical tools page, and the legacy docs page still exists alongside the canonical page. The issue also has two open contributor PRs with closing syntax, so it should stay open until maintainers review or replace those patches.

## Best Possible Solution

Keep this issue open. The best current path is to review and land or supersede one of the linked contributor PRs, updating both Brave `docsUrl` fields to `https://docs.openclaw.ai/tools/brave-search` and deciding whether `docs/brave-search.md` should become a minimal legacy stub or be removed if the Mintlify redirect is sufficient.
