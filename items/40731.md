## Summary

Keep open. Current `main` still builds Telegram native `/think` menus from the configured default provider/model, not the active session override, so the reported behavior is not implemented. The proposed patch is also stale against current `main` because thinking levels now treat `max` as a first-class canonical level and the command catalog moved from `commands-registry.data.ts` to `commands-registry.shared.ts`.

## Evidence

- **Telegram menu still uses default command context:** The Telegram native command handler calls `resolveCommandArgMenu({ command, args, cfg: runtimeCfg })` with no provider/model override, so menu choices are resolved from config defaults. Session-key-specific logic only starts later at `baseSessionKey`, after the menu path already returned.
