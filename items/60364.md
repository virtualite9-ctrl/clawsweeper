## Summary

Current main already reconciles stale task rows out of `running`, marks orphaned subagent/CLI tasks `lost` based on runtime-aware backing-state checks, and allows CLI task rows to be operator-cancelled. The relevant behavior is also documented and was shipped in release v2026.4.23.

## Evidence
