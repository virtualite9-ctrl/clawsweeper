## Summary

Keep #15828 open. Current main still exposes bundled session-memory as a manual `/new` and `/reset` internal hook only. Idle/daily rollover lifecycle hooks and `gateway_stop` plugin hooks exist, but the bundled session-memory handler is not wired to them, has no periodic/message-count checkpoint trigger, and has no LLM summary-output option beyond raw transcript extraction with LLM slug generation.

## Best Possible Solution

Keep #15828 open unless maintainers explicitly split or redirect it. A complete implementation should either extend the bundled `session-memory` hook with configurable idle/shutdown/message-count triggers, dedupe, summary-generation policy, schema/docs updates, and focused tests, or create a canonical plugin/API design tracker if maintainers decide this behavior should move out of the bundled hook.
