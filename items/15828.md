## Summary

Keep #15828 open. Current main still leaves bundled `session-memory` as a `/new` and `/reset` command hook only. Session idle/daily rollover and plugin lifecycle hooks exist, but they are not wired to this bundled hook, and shutdown, periodic/message-count checkpoints, and LLM summary output remain unimplemented.

## Best Possible Solution

Keep #15828 open unless maintainers intentionally split it. The product path should either extend bundled `session-memory` with configurable idle/shutdown/message-count triggers, dedupe, summary-generation policy, config/docs updates, and focused tests, or explicitly split daily/idle rollover work into #61675 while leaving shutdown, periodic checkpoints, and summary output tracked here or in linked follow-ups.
