## Summary

Keep this PR open. Current main still appears to have the Telegram native-command reply-threading behavior the PR is trying to fix: direct-chat `/new` and `/reset` replies still inherit the configured `replyToMode`, and generated native command replies still default to replying to the slash-command message. The PR is focused and security-neutral from the provided diff, but it needs to be ported from the old `src/telegram` paths to the current `extensions/telegram` plugin layout and should add `/reset` regression coverage.

## Best Possible Solution

Keep this PR open and port the narrow fix to `extensions/telegram/src/bot-native-commands.ts`: allow native command delivery to override `replyToMode`, set it to `"off"` only for direct-chat native `new` and `reset`, preserve group/topic and other-command threading behavior, and add regression tests for both `/new` and `/reset` with global Telegram `replyToMode: "first"`.
