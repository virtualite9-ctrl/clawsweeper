## Summary

Keep this PR open. Current main still lacks the Telegram native-command reply-mode override the PR attempted to add, and direct-chat native `/new` or `/reset` replies can still inherit threaded reply behavior. The PR needs to be ported from removed `src/telegram/*` paths to the current `extensions/telegram` plugin layout, with `/reset` coverage added. Security review found no supply-chain-sensitive changes in the provided PR diff.

## Best Possible Solution

Keep this PR open and port the narrow fix to `extensions/telegram/src/bot-native-commands.ts`: add a delivery-base override for `replyToMode`, set it to `"off"` only for direct-chat native `new` and `reset`, preserve group/topic and other-command threading behavior, and add regression coverage for both `/new` and `/reset` when global Telegram `replyToMode` is `"first"`.
