## Summary

Keep this PR open. Current main still resolves gateway image support from the model catalog plus narrow provider shims and has no fallback to explicit `models.providers.*.models[].input` metadata on catalog miss. The PR is a focused implementation candidate for the linked WebChat/custom-provider bug, though maintainers should reconcile it with the overlapping broader work in #70824 and verify the Qwen Coding Plan config-normalization caveat before landing.

## Best Possible Solution

Keep this PR open as a focused implementation candidate for the selected-provider explicit config fallback. The maintainer path is to rebase it on current main, keep catalog-hit precedence and alias-aware provider matching, add a Qwen/source-config regression if that behavior is intended, and reconcile it with #70824 and #64213 before landing or closing one as superseded.
