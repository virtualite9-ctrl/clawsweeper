## Summary

Keep this PR open. Current main still displays `entry.totalTokens` in the status Context line without respecting `totalTokensFresh: false`, and it has no fallback to the latest compaction checkpoint `tokensAfter`. The PR is an active, focused implementation candidate for the post-compaction stale context issue, with a regression test in the proposed diff and only a minor review cleanup noted in the discussion.

## Best Possible Solution

Keep the PR open and review it as the active implementation candidate. The best path is to adapt/merge the status-message change on current main, use `resolveFreshSessionTotalTokens(entry)` before trusting persisted totals, fall back to the newest valid `compactionCheckpoints[].tokensAfter` only when totals are stale or unavailable, address the unreachable-branch review note, and keep the focused regression test in `src/auto-reply/status.test.ts`.
