## Summary

Keep PR #71950 open. Current main still has the unsafe gateway/tailnet interface-discovery path the PR targets: `pickPrimaryLanIPv4()` and `listTailnetAddresses()` call `readNetworkInterfaces()` directly, and current tests still assert that `uv_interface_addresses` failures throw. The linked bug #71721 remains open, and the provided PR discussion includes a maintainer comment saying this is the right fix direction with a follow-up commit to preserve diagnostics.

## Best Possible Solution

Keep #71950 open for maintainer review/landing or minor rework. The best path is to adopt the PR's safe/inspection-based interface discovery for gateway LAN and tailnet helpers, preserve user-facing diagnostics, add the fallback regression tests, and then close #71721 when the fix is merged.
