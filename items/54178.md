## Summary

Keep open. Current main still renders the Agents page selector as a native `select` with only an `change` handler, and the latest release tag has the same selector path. I found adjacent UX tracking in #57067, but no source, test, docs, or canonical duplicate evidence that the automation-specific click behavior is fixed or intentionally unsupported.

## Best Possible Solution

Keep this issue open until current main is verified with the supported browser automation path. If pointer-based automation still cannot operate the Agents selector, replace the native select with an accessible automation-friendly combobox/listbox or visible agent list and add focused UI/browser coverage. If maintainers decide native-select popup opening by simulated clicks is not supported, document the supported automation path and coordinate with the broader #57067 Agents-page dropdown UX tracker.
