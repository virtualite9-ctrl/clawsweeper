## Summary

Keep this issue open. Current main still renders the Agents page selector as a native `<select>` with an `@change` handler, and there is no current source, test, docs, or related-item evidence that the simulated-click automation behavior is fixed or intentionally unsupported. The adjacent #57067 tracker may be the eventual UX path, but it is broader and does not explicitly cover this automation-specific report.

## Best Possible Solution

Keep this issue open until current main is verified with the supported browser automation path. If pointer-based automation still cannot operate the Agents selector, replace the native select with an accessible automation-friendly listbox/sidebar/list view and add focused UI or browser coverage. If maintainers decide native-select popup opening by simulated clicks is unsupported, document the supported automation path and coordinate with the broader #57067 Agents-page dropdown UX tracker.
