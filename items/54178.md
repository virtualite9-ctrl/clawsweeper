## Summary

Keep open. Current main and the latest release still render the Agents-page picker as a native change-driven select, with no source, docs, tests, changelog, or related-item evidence that the reported browser-automation click behavior has been fixed or superseded.

## Best Possible Solution

Keep this issue open until current main is verified with a supported browser automation path. If pointer-based automation still cannot operate the Agents selector, replace the native select with an accessible automation-friendly combobox/listbox or visible agent list and add focused browser/UI coverage. If maintainers decide untrusted `element.click()` cannot be supported for native selects, document the supported automation path explicitly and close with maintainer judgment.
