## Summary

Keep open. Current main still routes the bundled Slack message action path through a custom invoke callback that receives only the raw toolContext, while agent-scoped mediaLocalRoots/mediaReadFile are carried separately on ChannelMessageActionContext. The closed, unmerged #64632 directly targeted this issue, but the same missing context bridge is still visible in current main.

## Best Possible Solution

Keep this issue open. Recreate or revive the narrow #64632-style fix in extensions/slack/src/channel-actions.ts by building one augmented Slack action context from toolContext plus ctx.mediaLocalRoots/mediaReadFile and passing it to both options.invoke and the fallback runtime path. Add a regression test that exercises createSlackActions with options.invoke for upload-file and verifies agent-scoped media roots/readFile reach the invoke context.
