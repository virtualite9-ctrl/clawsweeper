## Summary

Keep this issue open. Current main still returns `status: "timeout"` immediately when the first nonzero-timeout `sessions_send` wait expires, with no post-timeout reply read or accepted-delivery compensation. The same author has open implementation candidate #68196 with closing syntax for this issue, so the maintainer path is to review, revise, merge, or close that PR before resolving this issue.

## Best Possible Solution

Keep this issue open and use #68196 as the active implementation path. The best fix belongs in the `sessions_send` and `run-wait` surface: after the agent RPC has accepted the run, handle wait-window expiry by checking for a new post-baseline assistant reply, returning `ok` for a late reply, returning `accepted` only when acceptance is provable, and preserving hard `timeout` when delivery cannot be established. Close this issue after the accepted fix lands on main, or if maintainers choose a different canonical implementation path.
