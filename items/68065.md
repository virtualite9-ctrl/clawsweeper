## Summary

Keep this issue open. Current main still maps the first nonzero-timeout sessions_send agent.wait timeout directly to status "timeout" without post-timeout reply or accepted-delivery compensation. The same reporter also has open implementation candidate #68196, which is explicitly paired with this issue, so the right maintainer path is to review, revise, merge, or close that PR before resolving this issue.

## Best Possible Solution

Keep this issue open and use #68196 as the active implementation path. The best fix belongs in the sessions_send/run-wait surface: after a successful agent RPC acceptance, handle wait-window expiry by checking for a new post-baseline assistant reply, returning ok for a late reply, returning accepted only when acceptance is provable, and preserving hard timeout when acceptance or delivery cannot be established. Close this issue after the accepted fix lands on main or the implementation path is superseded.
