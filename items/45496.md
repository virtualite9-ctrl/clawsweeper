## Summary

Close #45496 as duplicate/superseded by open #11676. Current main still does not run OpenClaw core or Gateway logic in-browser/WASM: it is a Node-hosted CLI/Gateway, while the existing browser surfaces are clients of that Gateway. The same remaining no-Node/WASM/runtime-portability work is already tracked by #11676, which remains open.

## Best Possible Solution

Close this duplicate tracking issue and consolidate the browser/WASM proposal into #11676. The right product path is a maintainer-approved runtime-portability design: stable schemas for tool calls, sessions, streaming/events, auth/model selection, browser storage adapters, and a clear security model before any in-browser Gateway implementation.
