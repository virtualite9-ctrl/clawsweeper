## Summary

Keep open. Current `main` still lacks a Talk-mode idle timeout after Voice Wake activation: Voice Wake enables Talk Mode and pauses the wake listener, Talk Mode resumes listening after each turn, and the only configured Talk timing is intra-turn `silenceTimeoutMs`, not between-turn auto-deactivation.

## Best Possible Solution

Add a first-class Talk idle timeout rather than closing this issue: define a validated `talk.idleTimeoutSeconds` or similar config field, parse it through shared Talk config, implement a between-turn idle timer in macOS/iOS/Android Talk runtimes that resets on user speech and assistant playback completion, disables Talk Mode on expiry, resumes Voice Wake where appropriate, and document the behavior on https://docs.openclaw.ai/nodes/talk.
