## Summary

Keep open. Current main has related provider-env scrubbing for the default `openclaw acp` client path, but the requested ACPX/Gemini harness hardening is not implemented: no auth-contract metadata, no fail-closed preflight hook, and no ACPX launch env-scrub policy for Gemini or other provider-specific agents.

## Best Possible Solution

Keep #69260 open for maintainer/API design. The best path is a generic ACP auth-contract surface: add typed runtime/session metadata for expected auth mode and fallback policy, pass a sanitized launch environment through the ACP manager/runtime contract, let the ACPX plugin declare per-agent scrub/preflight behavior, add Gemini-specific coverage for `GEMINI_API_KEY`, `GOOGLE_API_KEY`, and `GOOGLE_GENAI_*` env removal when OAuth is pinned, and document mixed-credential host risks on the public ACP/Gemini docs.
