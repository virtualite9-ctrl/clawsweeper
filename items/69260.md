## Summary

Keep #69260 open. Current main has related hardening for the built-in `openclaw acp` client path, but the issue’s central request remains unresolved for Gemini/acpx ACP harnesses: there is no declared ACP auth contract, no Gemini-specific ambient auth env scrub/preflight hook, and no mixed-credential warning in the ACP/Gemini docs.

## Best Possible Solution

Keep this open for a focused ACP/acpx hardening design. The best implementation path is to add a generic, backwards-compatible ACP agent auth contract in the plugin/runtime surface, let acpx agent definitions declare expected auth mode, fallback policy, scrubbed env keys, and optional fail-closed preflight, then implement Gemini defaults that pin the intended OAuth lane and strip conflicting Gemini/GenAI env vars. Add regression tests around `runtime.ensureSession` env/auth policy propagation and update the public ACP/Gemini docs with mixed-credential host guidance.
