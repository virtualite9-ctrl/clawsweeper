## Summary

Keep this PR open. Current main still sends Feishu card content as markdown-only card elements, so the native table rendering problem is not implemented on main. The PR is a relevant focused fix candidate, but it needs rework for current main and remaining review issues before it is merge-ready.

## Best Possible Solution

Keep this PR open as a useful Feishu card-table implementation candidate, but request a current-main rework: share parsing with the existing markdown IR where practical, convert card tables for both markdown and structured cards, preserve fenced code including tilde fences, extend quoted-card extraction for table elements, coordinate with the table-limit fallback work in #43690/#50429/#70651, and add focused regression tests for card rendering and reply-context extraction.
