## Summary

Keep this PR open. Current main still routes Feishu card-mode Markdown tables into markdown-only Card JSON 2.0 elements, so the requested native table rendering is not implemented or superseded. The PR is relevant, but it needs a current-main rework before merge because the active outbound/reply paths commonly use buildStructuredCard/sendStructuredCardFeishu, and the PR only hooks buildMarkdownCard in its submitted diff.

## Best Possible Solution

Keep this PR open as a useful Feishu native-table rendering candidate, but require a current-main rework before merge. The best path is to build a shared Card JSON 2.0 table conversion helper that preserves width_mode="fill", integrate it into both buildMarkdownCard and buildStructuredCard/sendStructuredCardFeishu delivery paths, protect both backtick and tilde fenced code blocks, preserve quoted-card text extraction for native table elements, and coordinate table-count fallback behavior with the open Feishu table-limit work tracked in #43690, #50429, and #70651.
