## Summary

Keep this PR open. Current main still sends Feishu card-mode table-shaped Markdown through markdown card elements instead of native Feishu table components, so the core behavior is neither implemented nor superseded. The PR is a relevant implementation candidate, but it needs a current-main rework before merge.

## Best Possible Solution

Keep this PR open as a useful Feishu native-table rendering candidate. Rebase and narrow it for current main by converting both `buildMarkdownCard` and `buildStructuredCard`/`sendStructuredCardFeishu` paths, preserving current schema 2.0 `width_mode` behavior, protecting both backtick and tilde fenced code regions, adding native-table extraction or markdown fallback for quoted card context, and coordinating table-count fallback with #43690/#50429/#70651. Add regression tests for outbound card routing, reply-dispatcher card replies, quoted-card extraction, and platform-limit fallback behavior.

## Risks / Open Questions

- The PR is stale against current main: its `buildMarkdownCard` tests expect `wide_screen_mode`, while current main intentionally tests `width_mode: "fill"` for schema 2.0 cards.
- The PR's current approach misses current-main structured cards and streaming cards, so merging it as-is would leave the main table-card behavior unfixed.
- Native table extraction for quoted/replied interactive cards is still missing; introducing table elements without extraction would lose table-only reply context and fall back to `[Interactive Card]`.
- The PR protects backtick fenced code blocks but not tilde fences, so table-shaped content inside valid `~~~` code fences can still be converted incorrectly.
- Feishu table components have platform limits, including page size and table-count limits, so the native conversion should coordinate with the fallback work in #43690, #50429, and #70651.
