## Summary

Keep #13597 open. Current `main` still does not have a dedicated AWS deployment guide for EC2/Lightsail, ECS/Fargate, or Lambda applicability. The existing docs provide generic Docker/Kubernetes/VPS guidance and Amazon Bedrock provider setup, but they do not cover the AWS production deployment runbook requested in the issue. The discussion remains actionable, including Docker Compose as the likely primary path, persistence/EFS, gateway exposure, AWS Secrets Manager, multi-agent topology, and the linked `aws-samples/sample-OpenClaw-on-AWS-with-Bedrock` repository.

## Best Possible Solution

Keep the issue open until OpenClaw ships an AWS install page, likely under `docs/install/aws.md`, linked from `docs/install/index.md`, `docs/vps.md`, and `docs/docs.json`. The guide should make Docker Compose the primary EC2/Lightsail path, cover durable state/EFS or equivalent storage, gateway bind/reverse proxy/ALB/TLS/auth/origin settings, security groups, IAM roles, Secrets Manager, logging/monitoring, backups, troubleshooting, performance and hardening, ECS/Fargate task/service/networking, Lambda applicability or non-support, multi-agent topology, and should evaluate or link the AWS sample repo from the issue discussion.
