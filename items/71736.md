## Summary

Keep this issue open. Current main now has an important slice from #72287: a generic `api.registerControlUiDescriptor(...)` SDK method and `plugins.uiDescriptors` Gateway projection. That is still narrower than the issue's requested Control UI contribution slots for chat modes, approval renderers/cards, event classifiers, input guards, status surfaces, and host-rendered UI behavior. The provided context also shows #71731 is still an open same-author RFC PR paired with this issue, so this should not be auto-closed out from under the paired design thread.

## Best Possible Solution

Keep this issue open as the Control UI contribution-slot design thread. The next maintainer path is to decide whether the shipped `registerControlUiDescriptor`/`plugins.uiDescriptors` foundation is the intended generic abstraction, then either update this issue/RFC to narrow the remaining work or implement the missing Control UI client consumption and descriptor kinds for chat modes, approvals, event classification, input guards, status surfaces, plugin actions, disablement, and sanitization fixture coverage.
