## Summary

Keep open. Current main has adjacent surfaces, but not the requested Control UI plugin contribution registry. Plugin commands can be discovered into the chat slash menu and plugin approval requests can be rendered through the existing generic approval queue, but there is no SDK/API surface for plugin-projected chat modes, input guards, event classifiers, status surfaces, or host-owned UI descriptor schemas. The linked RFC PR #71731 is still open and explicitly docs/RFC-only, so it does not supersede this issue.

## Best Possible Solution

Keep #71736 open as a focused SDK/UI RFC thread. The best path is a maintainer-reviewed core/plugin-SDK design for safe Control UI contribution descriptors, gateway projection from enabled plugins, UI sanitization, disablement cleanup, and fixture tests proving plugin enable/disable, generic UI actions, input guards, and approval/status rendering. Treat #71731 as related RFC context, not as an implementation that closes this issue.
