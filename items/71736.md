## Summary

Keep #71736 open. Current main still has adjacent plugin commands and generic plugin approval UI, but not the requested Control UI contribution descriptor registry for plugin-projected chat modes, input guards, event classifiers, status surfaces, or broader UI actions. The related implementation PR #72082 is still separate from current main, so the core SDK/UI design work remains live rather than cleanup-closeable.

## Best Possible Solution

Keep #71736 open as the focused RFC/design thread until a maintainer accepts and lands the generic host-hook implementation. The best path is an additive, safe Control UI descriptor contract covering plugin-projected chat modes, input guards, event classifiers, status surfaces, generic UI actions, gateway projection/listing, enable-disable cleanup semantics, sanitization, and fixture coverage. PR #72082 is the current implementation candidate to review against that bar.
