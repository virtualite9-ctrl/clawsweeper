## Summary

Keep this issue open. Current main still does not implement MathJax/KaTeX/LaTeX rendering in the Control UI chat renderer, and the open related PR #67103 is only an implementation candidate, not shipped behavior. The earlier PR #61510 that used closing syntax for this issue was closed unmerged, so it does not make the issue implemented.

## Best Possible Solution

Keep this issue open as the canonical user request, then review PR #67103 or a similar focused Control UI implementation. The likely product path is to add KaTeX or MathJax support inside `ui/src/ui/markdown.ts`, keep sanitization strict, add regression coverage in `ui/src/ui/markdown.test.ts`, and only close this issue after the implementation is merged to main.

## Risks / Open Questions

- Adding math rendering in this surface will need careful sanitizer review because rendered formula HTML would flow through `unsafeHTML(...)`; the open PR's claimed KaTeX/DOMPurify behavior should be reviewed before landing.
- Dollar-delimiter parsing can conflict with normal currency text and code spans, so the implementation should include focused renderer tests for inline math, block math, invalid LaTeX fallback, code spans, and ordinary dollar amounts.
