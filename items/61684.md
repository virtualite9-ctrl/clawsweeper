## Summary

The bundled Feishu channel already routes outbound images to Feishu native `image` messages on current `main`, and that behavior is already present in the shipped `v2026.4.23` release. The report appears to match older `2026.3.13` behavior or a different Feishu plugin/integration path.

## Evidence

- **Feishu image sends use native `image` messages:** `sendImageFeishu` sends `msg_type: "image"` with an `image_key`, and `sendMediaFeishu` routes image-like media to that path instead of the generic file path.
