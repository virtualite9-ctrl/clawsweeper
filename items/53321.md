## Summary

Keep open. Current main has substantial long-session mitigations, including silent memory flush, preflight compaction, queued-followup refresh, and opt-in successor transcript rotation, but it still does not implement the requested proactive auto session split policy with configurable token/context thresholds, invisible rollover semantics, active tool-call boundaries, or explicit continuity payloads for tasks and pending confirmations.

## Best Possible Solution

Keep this issue open as the canonical core product/design tracker. Build on the existing memory-flush, preflight compaction, queued-followup refresh, and truncateAfterCompaction machinery, then add a supported auto-split policy with threshold config, active tool-call boundaries, session-key routing updates, continuity payload handling for summaries/tasks/pending confirmations, docs, and focused regression tests.
