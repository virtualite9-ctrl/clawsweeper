## Summary

Keep this issue open. Current main still has only `systemPromptOverride`, which is documented and implemented as a full replacement for the assembled prompt. The requested additive `agents.defaults.systemPrompt`, `agents.defaults.rules`, and per-agent `systemPrompt`/`rules` config fields are not present in the schema, generated schema, docs, or shared prompt-building path. The related implementation PR is closed and unmerged, and the related issue is closed rather than an active canonical tracker.

## Best Possible Solution

Keep this issue open and implement a distinct additive prompt contribution surface. Add typed and schema-backed `agents.defaults.systemPrompt` and/or `agents.defaults.rules`, plus per-agent fields, with documented merge semantics where defaults and per-agent content append rather than replace. Resolve the contribution once at session creation and feed it through the shared prompt-building path for main, cron, subagent, CLI, and channel-triggered sessions. Preserve `systemPromptOverride` as the full replacement mechanism, and add schema, runtime, docs, and generated-schema coverage before closing.
