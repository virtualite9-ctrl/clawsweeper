## Summary

Keep this issue open. Current main still only supports `agents.defaults.systemPromptOverride` and `agents.list[].systemPromptOverride`, which replace the assembled system prompt instead of appending config-sourced rules. The linked implementation PR is closed and unmerged, and the related duplicate issue is also closed, so there is no active canonical tracker that preserves the remaining additive prompt feature.

## Best Possible Solution

Implement a distinct additive, schema-backed prompt contribution surface such as `agents.defaults.systemPrompt` and/or `agents.defaults.rules`, plus per-agent fields with documented merge order. Thread the resolved contribution through the shared prompt-building paths after bootstrap/context injection, keep `systemPromptOverride` as full replacement, update generated schema/help/docs and Control UI/config metadata, and add focused coverage for defaults, per-agent merge behavior, blank values, includes, and main/channel/cron/subagent sessions.

## Risks / Open Questions

- The requested `$include` behavior still needs product/security design for path resolution and whether to rely on existing config include handling or introduce field-specific include semantics.
- Implementation needs coverage across main, channel-triggered, cron, CLI-backed, and subagent session creation so additive rules apply consistently.
- `systemPromptOverride` should remain a separate full-replacement escape hatch; changing its semantics would be a compatibility risk for existing configs.
