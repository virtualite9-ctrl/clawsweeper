## Summary

Keep this issue open. Current main still exposes only `agents.defaults.systemPromptOverride` / `agents.list[].systemPromptOverride`, which replace the assembled prompt instead of adding config-sourced rules. The related implementation PR is closed and unmerged, and the related duplicate issue is closed, so there is no active canonical tracker preserving the remaining feature request.

## Best Possible Solution

Implement a distinct, schema-backed additive prompt contribution surface: `agents.defaults.systemPrompt` and/or `agents.defaults.rules`, plus per-agent fields with documented merge order. Thread the resolved contribution through the shared prompt-building path, update generated schema/help/docs and Control UI/config lookup metadata, preserve `systemPromptOverride` as full replacement, and add focused tests for defaults, per-agent merge behavior, blank values, includes, and main/channel/cron/subagent session creation.

## Risks / Open Questions

- The requested `$include` behavior needs an explicit decision on whether to rely on the existing config include processor or add field-specific string/array include semantics, with path-safety behavior documented.
- Implementation needs coverage across the shared prompt-building paths called out in discussion: main sessions, channel-triggered sessions, cron sessions, CLI-backed sessions, and subagent sessions.
- `systemPromptOverride` should remain a separate full-replacement escape hatch; overloading it with additive semantics would be a behavior change for existing configs.
