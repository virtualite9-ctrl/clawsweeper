## Summary

Keep this issue open. Current main still does not implement additive `agents.defaults.systemPrompt`, `agents.defaults.rules`, or per-agent `systemPrompt`/`rules`; the existing `systemPromptOverride` surface is a full replacement path, not the requested config-injected rules appended after bootstrap files. The related PR #52038 attempted the feature but is closed and unmerged, and the related #51598 discussion is closed rather than an active canonical tracker.

## Best Possible Solution

Keep the issue open and implement a distinct additive prompt contribution surface rather than reusing `systemPromptOverride`. The likely path is to add typed/schema-backed `agents.defaults.systemPrompt` and/or `agents.defaults.rules` plus per-agent config, resolve defaults and per-agent values with documented merge semantics, inject them through the shared prompt-building path across main, cron, subagent, CLI, and embedded sessions, preserve `systemPromptOverride` as a full replacement escape hatch, and add schema/runtime/docs tests. Nested `$include` support should be designed deliberately rather than inferred from the current override path.
