## Summary

Keep #14804 open. Current main still has per-scope sandbox workspaces plus manual/idle prune/recreate, but no first-class ephemeral or tmpfs-backed workspace lifecycle for isolated sessions. The existing issue discussion also already kept it open for the same core sandbox gap, and current main has not materially changed the sandbox config/schema/workspace lifecycle since that review.

## Best Possible Solution

Keep the issue open for a core sandbox lifecycle/config feature. The likely path is to add documented AgentSandboxConfig/schema fields for disposable or ephemeral workspaces, create a fresh workspace per isolated run where supported, clean it on completion, define Docker/SSH/OpenShell behavior including whether tmpfs is required or best-effort, add lifecycle tests, and update the public sandbox/config docs.

## Risks / Open Questions

- A future fix needs explicit cross-backend semantics: Docker can use disposable bind roots or tmpfs, while SSH/OpenShell need remote/local workspace creation and teardown rules.
- Docs should distinguish session/container isolation from disposable filesystem state so users do not assume current `scope: "session"` gives a clean filesystem each run.
