## Summary

Keep #14804 open. Current main still has sandbox scope isolation, workspace access controls, Docker tmpfs mounts for temp paths, SSH/OpenShell remote workspace flows, and stale runtime pruning, but it does not provide the requested first-class ephemeral or tmpfs-backed workspace mode for isolated sessions/runs. The existing 2026-04-26 review comment keeping this open remains accurate against current main e29d3516bf05591e6007bfb8ccdaea6d76194d41.

## Best Possible Solution

Keep #14804 open until core sandbox lifecycle/config adds a documented disposable workspace mode. A good fix should add an explicit sandbox setting to AgentSandboxConfig/AgentSandboxSchema, create a fresh per-run workspace where supported, clean it on completion, define Docker/SSH/OpenShell behavior including whether tmpfs is required or best-effort, and update sandbox lifecycle tests plus public docs at https://docs.openclaw.ai/gateway/sandboxing and https://docs.openclaw.ai/gateway/config-agents.
