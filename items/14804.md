## Summary

Keep #14804 open. Current main still lacks a first-class ephemeral or tmpfs-backed workspace mode for isolated sessions. The code supports fresh transcript/session ids for isolated cron runs, scoped sandbox workspaces, Docker tmpfs mounts for temporary container paths, manual sandbox recreate, and age/idle pruning, but none of those provide the requested clean per-run filesystem lifecycle.

## Best Possible Solution

Keep #14804 open until core sandbox lifecycle/config adds a documented disposable workspace mode. A good fix should add an explicit setting to AgentSandboxConfig and AgentSandboxSchema, create a fresh per-run workspace where supported, clean it on completion, define Docker/SSH/OpenShell behavior including whether tmpfs is required or best-effort, update sandbox lifecycle tests, and update public sandbox/config docs.
