## Summary

Keep this issue open. Current main still routes the default Docker-gateway sandbox workspace through the gateway container namespace (`/home/node/.openclaw/sandboxes/...`) and passes that raw path to `docker create -v`; the setup script enables sandboxing without configuring a host-visible `workspaceRoot` or an equivalent host/container parity mount for the sandbox root.

## Best Possible Solution

Keep the issue open and fix the Docker gateway sandbox path contract. The maintainer fix should either translate container-visible sandbox/workspace roots back to host-visible `OPENCLAW_CONFIG_DIR`/`OPENCLAW_WORKSPACE_DIR` paths before Docker creation, or have Docker setup configure a host-visible `agents.defaults.sandbox.workspaceRoot` with an identical gateway-visible parity mount. Add regression coverage proving Docker gateway sandbox creation does not emit `/home/node/.openclaw/sandboxes/...` as a Docker daemon bind source for the default Docker Desktop/macOS setup.
