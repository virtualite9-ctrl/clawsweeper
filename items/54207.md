## Summary

Keep this issue open. Current main still lets the Docker gateway default sandbox root resolve under the gateway container namespace and passes that raw path to Docker bind creation; the Docker setup path enables sandboxing without setting a host-visible sandbox workspace root or parity mount.

## Best Possible Solution

Keep this issue open and fix the Docker gateway sandbox path contract. The maintainer fix should either translate gateway-container sandbox roots back to host OPENCLAW_CONFIG_DIR/OPENCLAW_WORKSPACE_DIR paths before Docker creation, or have Docker setup configure agents.defaults.sandbox.workspaceRoot to a host-visible path with an identical gateway-visible parity mount. Add regression coverage proving the default Docker gateway sandbox path does not emit /home/node/.openclaw/sandboxes/... as a Docker daemon bind source on the Docker Desktop/macOS setup path.
