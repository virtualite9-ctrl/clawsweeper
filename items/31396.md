## Summary

Keep #31396 open. The unresolved “connect to a new model” wording may still map to provider connection latency; current main does not expose a model-provider connect timeout field, while the supplied related PR #41936 remains open.

## Best Possible Solution

Keep the issue open and ask for a concrete reproduction: exact command or prompt, channel/UI, provider and model, current/default model, whether a run was active, OpenClaw version/config, and logs around the freeze. If it reproduces in `/model`, `session_status(model=...)`, or `sessions.patch`, add a targeted live-switch regression test. If it is provider connection latency, resolve it through the provider timeout design tracked by PR #41936 or a narrower successor.
