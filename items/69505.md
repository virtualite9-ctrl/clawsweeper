## Summary

Keep issue #69505 open. Current main does not implement an `assistantPrefix`/assistant-prefill field on cron `agentTurn`, hook agent payloads, agent defaults, or CLI surfaces, and the existing `messages.responsePrefix` is a post-generation outbound prefix rather than a provider-native start-of-assistant-turn guarantee.

## Best Possible Solution

Add this as an explicit core/gateway capability rather than closing: define and validate `assistantPrefix` on cron `agentTurn` payloads, hook agent actions, agent defaults, and CLI/UI/docs surfaces; plumb it through cron execution into the embedded/CLI provider request layer; implement provider-native prefill for at least Anthropic and one OpenAI-compatible path with clear unsupported-provider no-op/debug semantics; add regression tests proving delivered cron output starts with the prefix across supported providers.
