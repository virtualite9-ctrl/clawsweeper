## Summary

Keep #69505 open. Current main does not implement an `assistantPrefix`/provider-prefill field for cron `agentTurn`, agent defaults, `openclaw agent`, or cron CLI surfaces. The existing `responsePrefix` feature is only an outbound channel prefix added after model output, so it does not solve the requested protocol-level guarantee that the model continuation starts with fixed text.

## Best Possible Solution

Keep the issue open for maintainer product judgment. If accepted, implement an additive core/runtime feature: add `assistantPrefix` to cron payload types, TypeBox/Zod schemas, normalization, cron tool, `cron add/edit`, `openclaw agent`/gateway agent params, agent defaults schema/help/docs, and runtime params; then wire provider-specific handling for at least Anthropic plus one OpenAI-compatible adapter with tests proving delivered cron output begins with the configured prefix. Clarify that `messages.responsePrefix` remains channel-side formatting, not a model prefill guarantee.
