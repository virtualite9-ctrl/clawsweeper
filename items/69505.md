## Summary

Keep #69505 open. Current main does not implement an `assistantPrefix`/equivalent cron payload field, agent default, CLI flag, provider prefill path, or documented config surface. The request is a concrete core/gateway/provider-contract feature, not a ClawHub-only plugin idea, because current plugin hooks can rewrite/cancel delivered messages but do not provide a supported per-turn provider-native assistant prefill contract.

## Best Possible Solution

Keep the issue open for maintainer product/API design. A good implementation would add an additive `assistantPrefix`-style field to cron payload/schema/normalization, agent defaults and per-agent config if maintainers want ad-hoc coverage, CLI/UI/docs surfaces, and a provider-facing runtime seam that supported adapters can honor before generation while unsupported providers no-op with debug logging. It should coordinate with the broader open final-output/preamble work (#62121/#50989) so OpenClaw does not grow conflicting output-control knobs.

## Risks / Open Questions

- Provider-native continuation/prefill semantics differ across Anthropic, OpenAI-compatible, DeepSeek-style, and Ollama transports, so the implementation needs a deliberate provider contract rather than a generic string append.
- Existing message-sending plugins can mitigate some channel leaks after generation, but they do not satisfy the requested guarantee that generation starts from a fixed assistant prefix.
