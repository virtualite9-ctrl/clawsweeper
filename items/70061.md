## Summary

Keep #70061 open. Current main still has the reported gap: Telegram DM topic tests still produce `agent:main:main:thread:<chatId>:<topicId>` for default-account private chats, while Active Memory's chat-type resolver does not classify that key as `direct`, so the `allowedChatTypes` guard can still skip the hook before the memory sub-agent runs.

## Best Possible Solution

Keep the issue open and patch Active Memory's chat-type inference with regression coverage. A targeted fix should classify `agent:<agent>:<mainKey|main>:thread:<chatId>:<topicId>` as `direct` when the hook context is a non-webchat channel such as Telegram, while preserving explicit `:group:` and `:channel:` behavior. Add an Active Memory test for the exact Telegram default-account DM topic key and consider a debug/status note for unclassified eligible sessions so this failure mode is not silent.

## Evidence

- **Active Memory resolver still misses default Telegram DM thread keys:** `resolveChatType()` recognizes `:group:`, `:channel:`, `:direct:`, `:dm:`, and only 3-part `agent:<agent>:<mainKey>` sessions. A key like `agent:main:main:thread:1234:42` has no direct/dm marker and is not 3 parts, so it returns `undefined`; `isAllowedChatType()` then returns false.
