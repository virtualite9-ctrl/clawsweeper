## Summary

Keep #70061 open. Current main still does not classify the reported Telegram DM-topic session-key shape as a direct chat for Active Memory. Telegram tests still document inbound DM topics as `agent:main:main:thread:<chatId>:<topicId>`, while Active Memory only recognizes `:group:`, `:channel:`, `:direct:`, `:dm:`, three-part main sessions, or webchat. The existing Active Memory regression coverage only covers `telegram:direct:...:thread:...`, not the shared `main:thread` key reported here.

## Best Possible Solution

Patch `extensions/active-memory/index.ts` so Telegram DM-topic thread sessions using the shared main-session key, for example `agent:main:main:thread:<chatId>:<topicId>`, classify as `direct` when the hook context identifies an interactive Telegram/non-webchat channel session. Add a focused Active Memory regression test for that exact key shape with `allowedChatTypes: ["direct"]`, and consider logging or trace-status coverage for unclassified chat-type skips so future session-key formats do not fail silently.
