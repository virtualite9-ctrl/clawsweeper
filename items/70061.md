## Summary

Keep #70061 open. Current main still produces Telegram DM topic session keys in the reported `agent:main:main:thread:<chatId>:<topicId>` shape, and Active Memory's `resolveChatType()` still does not classify that shape as `direct`, so the `allowedChatTypes` guard can skip the hook before memory lookup. Existing Active Memory coverage only exercises the `telegram:direct:...:thread` variant, not the default shared-DM thread key.

## Best Possible Solution

Fix Active Memory in core by classifying thread-suffixed direct/main session keys as `direct`, with a focused regression test for `agent:main:main:thread:<chatId>:<topicId>` and preferably a debug/status signal when a user-triggered interactive session is skipped because its chat type is unrecognized.
