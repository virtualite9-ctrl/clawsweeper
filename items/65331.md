## Summary

Keep this PR open. Current main still routes plugin text transforms through the indirect runtime-registry helper, while the related bug #64921 remains open; the PR is a plausible fix direction but is stale against current main and needs a regression test.

## Best Possible Solution

Keep this PR open for maintainer review or a revised patch. The best path is to update the current active-runtime-registry/text-transforms.runtime design so runtime text transforms read the active main-process registry through a direct or proven generic seam, remove or document any fallback, and add a regression test that plugin-registered textTransforms affect the CLI/provider prompt path. Use #64921 as the bug-tracking anchor until the behavior is fixed and tested.
