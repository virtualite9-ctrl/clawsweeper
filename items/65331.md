## Summary

Keep this PR open. Current main still resolves plugin text transforms through the indirect require-probing runtime helper, and the user-visible CLI/provider paths still consume that resolver. The PR targets a real remaining gap, but it needs revision for current main and a regression test before maintainers should land or close it.

## Best Possible Solution

Keep this PR open for maintainer review or a revised patch. The best fix is to update the current `active-runtime-registry`/`text-transforms.runtime` path so main-process runtime text transforms read the active registry directly or through a proven generic seam, remove or document any fallback, and add a regression test proving plugin-registered `textTransforms` are applied through the CLI/provider prompt path. Related #64921 should remain the bug-tracking anchor until that behavior is fixed and tested.
