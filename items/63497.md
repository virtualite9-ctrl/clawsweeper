## Summary

Keep this PR open. Current main has shared media-reference hydration and basic image-tool coverage, but the normal agent-tool path still passes `workspaceDir`; in that branch the image tool still rewrites `media://inbound/<id>` as a workspace-relative path before `loadWebMedia` can resolve the managed media URI. The PR remains a relevant fix candidate, though it should be adapted to the newer shared media-reference helpers.

## Best Possible Solution

Keep this PR or a replacement fix open. Preserve or resolve `media://inbound/<id>` before image-tool workspace-relative path resolution, preferably using the current shared media-reference helpers or by excluding `refInfo.isMediaStoreUrl` from the workspaceDir branch. Add regression coverage for `createImageTool` or `createOpenClawTools` with `workspaceDir` plus `fsPolicy: { workspaceOnly: true }`, covering both bare `media://inbound/<id>` and `MEDIA:media://inbound/<id>`.
