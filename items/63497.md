## Summary

Keep PR #63497 open. Current main has centralized media-reference handling and a basic `media://inbound` image-tool test, but the production image-tool path still appears to rewrite media-store URIs as workspace-relative paths when `workspaceDir` is present. That is the normal agent tool construction path, so the PR's core bug is not confidently obsolete.

## Best Possible Solution

Keep the PR or a replacement fix open. The best fix is to preserve `media://inbound/<id>` through image-tool resolution when `workspaceDir` is set, either by excluding `refInfo.isMediaStoreUrl` from workspace-relative resolution or resolving media-store refs before that branch. Add a regression test for `createImageTool({ workspaceDir, fsPolicy: { workspaceOnly: true } })` with both `media://inbound/<id>` and `MEDIA:media://inbound/<id>`.
