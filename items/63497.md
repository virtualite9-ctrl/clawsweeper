## Summary

Keep this PR open. Current main has centralized media-reference helpers and a direct image-tool test for `media://inbound`, but the normal agent tool path still passes `workspaceDir` and the image tool still resolves a preserved `media://inbound/<id>` URI as a workspace-relative filesystem path before `loadWebMedia` can hydrate it. The PR remains relevant, but should be rebased onto the shared helpers rather than merged mechanically.

## Best Possible Solution

Keep this PR or a replacement fix open. Adapt `src/agents/tools/image-tool.ts` so media-store URLs are preserved or resolved before workspace-relative path handling, preferably by using `resolveMediaReferenceLocalPath` or by excluding `refInfo.isMediaStoreUrl` from the `workspaceDir` branch. Add regression coverage through `createImageTool` or `createOpenClawTools` with `workspaceDir` and `fsPolicy: { workspaceOnly: true }`, covering both `media://inbound/<id>` and `MEDIA:media://inbound/<id>` while retaining the shared inbound-media safety guards.
