## Summary

Current `main` does not appear to implement the PR's cross-account Discord thread-owner suppression. The preflight path still resolves only the current account's thread binding and then falls through to generic/configured routing when that account has no local binding, which matches the PR's reported bug rather than proving it obsolete.

## Evidence

- **Preflight only checks current-account runtime/configured bindings:** `preflightDiscordMessage` resolves a runtime binding for `params.accountId`, then optionally a configured binding, and proceeds with routing. There is no foreign-account thread-owner lookup or early drop path for a thread already bound by another Discord account.
  - file: [extensions/discord/src/monitor/message-handler.preflight.ts:645](https://github.com/openclaw/openclaw/blob/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4/extensions/discord/src/monitor/message-handler.preflight.ts#L645)
  - command: `nl -ba extensions/discord/src/monitor/message-handler.preflight.ts | sed -n '590,720p'`
  - sha: [f3c37a946cc4](https://github.com/openclaw/openclaw/commit/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4)
- **Discord session-binding adapter is account-scoped:** The Discord thread-binding adapter resolves a conversation by calling `manager.getByThreadId(ref.conversationId)`, where `manager` is the per-account manager created for one account. That means another bot account will not see a binding owned by a different account through normal runtime binding resolution.
  - file: [extensions/discord/src/monitor/thread-bindings.manager.ts:691](https://github.com/openclaw/openclaw/blob/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4/extensions/discord/src/monitor/thread-bindings.manager.ts#L691)

- **Repo tests document same thread id can exist under different Discord accounts:** The lifecycle test binds the same `thread-1` under account `a` and account `b` and confirms each manager only sees its own record. That reinforces that current main has per-account isolation in storage, but no preflight guard for foreign ownership when only one account owns the thread.
