## Summary

Keep open. Current `main` still exposes skills to agents as a prompt-time `<available_skills>` list with `name`/`description`/`location`, and the built-in search surface is an operator/CLI ClawHub lookup rather than an agent runtime `skill_search` flow. Part of the issue body is somewhat outdated because skill snapshots can refresh without a Gateway restart, but the requested intent-driven runtime discovery is not implemented on `main`.

## Risks / Open Questions

- The issue body overstates the current restart limitation; if maintainers pick this up, the problem statement should be narrowed to agent-side discovery and matching rather than gateway/session reload behavior.
- ClawHub already owns registry search and discovery metadata, so future work should probably build on that surface instead of adding a separate parallel search/index path in core.
