## Summary

Keep open. Current main still deliberately blocks ACP spawns from sandboxed requester sessions and rejects required sandboxing for ACP, while the public docs still describe this as a current limitation. The related #32935 is closed and #45841 is explicitly a continuation, so there is no open canonical item in the provided context that supersedes this request.

## Best Possible Solution

Keep the issue open for a focused core design. The likely path is an explicit, audited sandbox-to-host ACP control surface or nested ACP sandbox mode, with config schema, availability/policy changes in `sessions_spawn` and `spawnAcpDirect`, backend/API support for spawn/steer/cancel across the boundary, and docs/tests updating the current sandbox compatibility limitation.
