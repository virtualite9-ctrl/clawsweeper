## Summary

Keep this PR open. Current main still has the explicit-empty-tool prompt/report gap the PR targets: an empty `toolNames` set falls into the legacy fallback tool catalog and tool-only guidance, skills are appended independently of tool availability, and prompt reports still count raw `skillsPrompt` rather than rendered prompt content. The related open payload-level PR does not fully supersede this system-prompt/report work.

## Best Possible Solution

Keep this PR open and review/rebase it as the prompt-layer fix, or split it into smaller pieces: explicit empty-tool prompt semantics in `src/agents/system-prompt.ts`, rendered-prompt-based report accounting in `src/agents/system-prompt-report.ts`, final effective tool/client-tool propagation through CLI and embedded runners, provider overlay alignment, and focused regression tests. Coordinate but do not replace it with #70790 unless that work also covers the prompt/report behavior.
