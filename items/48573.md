## Summary

Current main has substantial native subagent liveness and orphan-recovery coverage, but the ACP child-session gap raised in the issue discussion still appears present. ACP spawns create `agent:<id>:acp:<uuid>` session entries and `runtime: "acp"` task records outside the subagent run registry, while task maintenance treats an existing ACP session entry as a backing session rather than reconciling stale ACP `running` state. The related #68718 minions PR appears to target this class of durability/cascade-cancel work, but it is open and not shipped on main.

## Best Possible Solution

Keep #48573 open as an ACP/subagent durability bug. Current native subagent recovery should be preserved, but maintainers should either land/scope the #68718 durability work or add a narrower ACP cleanup path: register ACP child runs in an owned lifecycle registry, or add ACP startup/maintenance reconciliation that detects stale `acp.state: "running"` sessions with no live run context/process, terminalizes the task/session metadata, and covers parent termination plus gateway crash/restart with tests.
