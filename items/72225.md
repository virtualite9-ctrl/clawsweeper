## Summary

Keep #72225 open. Current main still has the three `activeMinutes: 0` Control UI refresh callsites the PR targets, and the session loader drops non-positive values before calling `sessions.list`, so the PR is not implemented on main. However, related session-dropdown history shows this active-window behavior has product tradeoffs around stale/current session visibility, so the right path is maintainer review, not cleanup closure.

## Best Possible Solution

Keep the PR open for maintainer review. The narrow implementation should either land with the added UI tests and explicit acceptance of the active-window product behavior, or be adjusted to preserve the selected/main session while still filtering old completed sessions. Review it alongside #43984 and the prior dropdown visibility/clutter reports before merging.
