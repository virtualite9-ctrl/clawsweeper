## Summary

Keep open. Current `main` still exposes only the binary `tools.fs.workspaceOnly` guard; it does not implement the requested filesystem ACL model with configurable `allowedPaths` / `denyPaths`, multi-root allowlists, or deny-glob semantics.

## Evidence

- **Config type surface only has workspaceOnly:** `FsToolsConfig` defines a single optional boolean, `workspaceOnly`, with no `allowedPaths` or `denyPaths` fields.
  - file: [src/config/types.tools.ts:296](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/config/types.tools.ts#L296)
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **Runtime schema only accepts workspaceOnly:** `ToolFsSchema` accepts only `{ workspaceOnly?: boolean }`, so the requested config keys are not valid on current `main`.
  - file: [src/config/zod-schema.agent-runtime.ts:484](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/config/zod-schema.agent-runtime.ts#L484)
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **Filesystem policy model is still a single boolean:** `ToolFsPolicy` and `resolveToolFsConfig()` carry only `workspaceOnly`; there is no path allow/deny list or glob-based ACL layer in the agent FS policy path.
  - file: [src/agents/tool-fs-policy.ts:7](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/agents/tool-fs-policy.ts#L7)
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **Host FS tools remain unrestricted when workspaceOnly is off:** `createHostWriteOperations()` and `createHostEditOperations()` explicitly allow writes/edits anywhere on the host when `workspaceOnly` is false, rather than enforcing a narrower allowlist/denylist.
  - file: [src/agents/pi-tools.read.ts:757](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/agents/pi-tools.read.ts#L757)
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **Tests still codify outside-workspace access when workspaceOnly is false or unset:** Current tests expect read/write/edit outside the workspace to succeed when `workspaceOnly=false` or unset, and only fail when `workspaceOnly=true`.
  - file: [src/agents/pi-tools.workspace-only-false.test.ts:82](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/agents/pi-tools.workspace-only-false.test.ts#L82)
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **Docs describe only the workspace-only guard:** The user-facing security docs and config help document `tools.fs.workspaceOnly` as the available filesystem hardening control; there is no documented `allowedPaths` / `denyPaths` feature.
  - file: [docs/gateway/security/index.md:1029](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/docs/gateway/security/index.md#L1029)
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
