## Summary

Keep #48510 open. Current main still has no structured capability descriptor on agent tool definitions, plugin tool registration, hook payloads, or tool catalog output. This is an actionable core/plugin-SDK contract request, not a ClawHub-only plugin idea, because external policy plugins cannot add or reliably receive this metadata without a shared OpenClaw API. Related #48503 is complementary and remains open, not a merged superseding item.

## Best Possible Solution

Keep the issue open and handle it as a core/plugin-SDK design item. A good implementation would add a small optional tool capability descriptor type to the shared AnyAgentTool/plugin registration surface, preserve it through wrappers/adapters/catalog APIs, declare built-in tool descriptors, and expose the resolved metadata to before_tool_call in coordination with #48503’s hook-enrichment discussion.
