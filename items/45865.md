## Summary

Keep open. Current main does not expose the requested injected `api.runtime.llm()` / `api.runtime.llm.complete()` Plugin SDK inference method. Existing alternatives include `api.runtime.agent.runEmbeddedAgent(...)`, `api.runtime.subagent`, `api.runtime.modelAuth`, and exported simple-completion helper subpaths, but those do not provide the requested provider-agnostic runtime LLM facade. The issue comments are substantive and supportive, and related PR #64294 appears to be an open implementation direction rather than shipped or canonical completed work.

## Best Possible Solution

Keep #45865 open and treat PR #64294 as a possible implementation path, not a completed replacement. The best product/code direction is a narrow injected `api.runtime.llm.complete()` or equivalent `api.runtime.llm()` facade that routes through OpenClaw's normal model/auth/provider runtime path, documents default model/session semantics, and adds PluginRuntime type/runtime/docs/API-baseline tests before shipping.
