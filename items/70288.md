## Summary

Keep open. Current `main` still returns `queuedFinal: false` for plugin-bound `inbound_claim` handlers that report `handled`, the latest release tag still has that behavior, and the proposed PR change appears incomplete because the buffered-dispatch path later recomputes `queuedFinal` from `counts.final`, which remains `0` for async plugin claims.
