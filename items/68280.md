## Summary

Keep open. Current main still lacks the PR’s fail-fast local probe-auth behavior, and this PR remains a useful implementation path for the gateway/CLI log-spam bug tracked from #61179. The latest PR head addresses prior override and unset-auth-mode review feedback, but it is not obsolete or already implemented on main; it still needs maintainer/security review around user-facing SecretRef failure details and complete output sanitization before landing.

## Best Possible Solution

Keep this PR open as the canonical implementation candidate. Before landing, revise it so local `gateway status`, summary `status`, and daemon status fail fast only for local loopback probes when auth is explicitly required and unresolved; preserve explicit URL/RPC override behavior and unset/none/trusted-proxy auth modes; make user-facing failure reasons generic or redacted while keeping detailed diagnostics out of normal output; sanitize every terminal/status surface that renders probe errors; then rebase on current main and run the targeted gateway/CLI regression tests.
