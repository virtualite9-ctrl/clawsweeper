## Summary

Keep #13479 open. Current main still does not provide the requested click-to-reveal cron job details/summary surface: cron job row clicks are wired to load run history, the job card renders prompt/system payload inline, and the stored `description` field is only exposed through the form/data contract, not the dashboard job detail surface.

## Best Possible Solution

Keep the issue open. The best implementation path is to add explicit selected-job detail or expansion state in `ui/src/ui/views/cron.ts`, render `job.description` as the human-readable summary alongside the existing prompt/system payload, preserve the existing run-history selection controls, add English i18n strings, and cover the behavior with focused `ui/src/ui/views/cron.test.ts` tests.
