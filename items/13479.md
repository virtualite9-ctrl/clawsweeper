## Summary

Keep #13479 open. Current main still only partially satisfies the request: the cron Web UI renders the job prompt/system payload inline, and the cron model persists an optional description, but clicking a cron job still selects run history rather than revealing a job detail/summary surface, and the dashboard job card does not render `job.description`.

## Best Possible Solution

Keep the issue open. Implement this in the Control UI cron surface: add selected-job detail or expansion behavior in `ui/src/ui/views/cron.ts` and related state as needed, render `job.description` as the human-readable summary alongside the existing prompt/system payload, preserve run-history controls, add English i18n copy, and cover it with focused `ui/src/ui/views/cron.test.ts` assertions.
