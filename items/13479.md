## Summary

Keep #13479 open. Current main still does not implement the requested click-to-reveal cron job detail/summary behavior: the cron dashboard shows prompt/system payload text inline, but job row clicks are wired to run-history selection and the dashboard job card does not render `job.description`. A maintainer comment on the issue already recorded the same remaining gap on April 26, 2026, and the current checkout still matches that assessment.

## Best Possible Solution

Keep #13479 open. The best implementation path is to add selected-job detail or expansion state in `ui/src/ui/views/cron.ts`, render `job.description` as the human-readable summary alongside the existing prompt/system payload display, preserve the existing run-history controls, add English i18n strings, and cover the behavior with focused `ui/src/ui/views/cron.test.ts` tests.
