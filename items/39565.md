## Summary

Keep #39565 open. Current main still has the pre-PR TUI sanitizer behavior: it strips only a limited punctuation set, runs URL/path/file-like checks against the raw token, and only preserves file-like tokens when the raw token contains `_`. That still allows dotted IDs without underscores and punctuation/markdown-wrapped entity IDs to be chunked with inserted spaces, so the linked bug #39505 remains actionable.

## Best Possible Solution

Keep this PR or the linked #39505 work open. The best path is a narrow TUI formatter fix in `src/tui/tui-formatters.ts` that classifies punctuation-stripped candidates, strips markdown asterisks at token edges, preserves dotted identifiers and digit-bearing hyphenated IDs, and keeps punctuation-only or natural-language hyphenated long tokens chunkable, with regression coverage in `src/tui/tui-formatters.test.ts`.
