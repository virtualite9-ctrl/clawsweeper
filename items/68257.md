## Summary

Keep this PR open. Current main still lacks a top-level `auth` reload rule, so `auth.profiles.*` changes still fall through to the unknown-path gateway restart behavior. The PR is a focused implementation candidate paired with open bug #68258, and the provided diff is limited to the gateway reload planner plus regression tests.

## Best Possible Solution

Review and land this PR or an equivalent patch on main: classify top-level `auth.profiles.*` OAuth/token-refresh writes as no-op for gateway restart planning, keep existing `gateway.auth.*` restart behavior, and add regression tests for `auth.profiles.*` plus mixed `gateway.auth`/`auth.profiles` batches. After merge, close #68258 as implemented.
