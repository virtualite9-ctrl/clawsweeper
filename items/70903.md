## Summary

Keep open. Current main has partial recovery work, but it still does not cover the issue's same-provider fallback topology or the added `openclaw configure --section model` re-auth stale-cooldown path.

## Best Possible Solution

Keep this issue open and narrow the implementation to the remaining gaps: treat same-provider-only fallback chains sharing disabled auth profiles like single-provider billing recovery so they receive a throttled recovery probe, and move stale lockout clearing into the shared provider auth-choice/configure path so successful re-auth clears the affected provider's `auth-state.json` usage entry. Add regression coverage for Anthropic same-profile primary plus same-provider fallback billing recovery and GitHub Copilot configure re-auth clearing a `format` cooldown.
