## Summary

Keep open. Current main still persists CLI ClawHub installs as resolved versioned specs, and current tests still assert that pinned behavior. The PR remains a focused core CLI bug fix, not ClawHub/plugin-discovery work: `plugins update` reuses the recorded spec, so persisting `clawhub:demo@1.2.3` from an originally unversioned install can keep later updates pinned.

## Best Possible Solution

Keep PR #58426 open and either merge it or adapt the same fix on main: persist `raw` for direct `clawhub:` installs and `preferredClawHubSpec` for bare-name ClawHub fallback, keep explicit `clawhub:pkg@version` installs pinned, remove the now-stale CLI test mock entry, and run the targeted CLI/update tests plus build in a writable maintainer validation pass.
