## Summary

Keep PR #58426 open. Current main still persists direct and bare-name ClawHub CLI installs as resolved versioned specs, while `plugins update` reuses the stored spec. That matches the bug this PR fixes, and the PR remains a focused core CLI metadata fix rather than ClawHub/plugin-discovery work.

## Best Possible Solution

Keep PR #58426 open and merge or adapt the same fix on main: persist `raw` for direct `clawhub:` CLI installs, persist `preferredClawHubSpec` for bare-name ClawHub fallback, keep explicit `clawhub:pkg@version` installs pinned, remove the stale CLI test mock entry, and run the targeted CLI install/update tests plus build in a writable maintainer pass.
