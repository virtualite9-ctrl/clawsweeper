## Summary

Keep PR #71828 open. Current main still starts the run-side typing TTL when the typing loop starts, but keepalive ticks only call the channel typing callback and do not refresh the controller TTL. That means the controller can still clean up during a long active run, which is exactly the Discord long-run behavior this PR targets. The PR’s one-line runtime change and regression test remain relevant rather than obsolete or superseded on main.

## Best Possible Solution

Keep PR #71828 open for maintainer review and normal CI. The best path is to land this narrow controller TTL refresh plus the regression test, or fold the same behavior into the broader open #65919 active-run presence work if maintainers want one consolidated implementation. The fix should continue to preserve cleanup after run completion and dispatch idle, and must not allow sealed controllers to restart typing.
