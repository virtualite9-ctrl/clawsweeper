## Summary

Keep PR #71828 open. Current main still has the exact shared typing-controller gap this PR targets: keepalive ticks send Discord typing, but they do not refresh the controller TTL, so cleanup can still stop typing during an active long run. The PR is recent, narrow, and has a regression test for the missing beyond-TTL active-run case.

## Best Possible Solution

Keep PR #71828 open for maintainer review. The best path is to land this targeted TTL refresh and regression test, or fold the same one-line controller fix into the broader #65919 presence lease work if maintainers want one consolidated implementation. Preserve cleanup after run completion plus dispatch idle, and keep the sealed-controller guard so late events cannot restart typing.
