## Summary

Keep this PR open. Current main still has the controller-level typing TTL expiry the PR targets: keepalive ticks invoke the channel typing callback, but they do not refresh `createTypingController`'s own TTL, so the shared controller can clean up while a long run is still active. The PR is a narrow implementation candidate with a direct regression test, and the related open/closed items do not supersede it.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land this focused controller TTL refresh and regression test if CI remains green, or explicitly fold the same behavior into a broader #65919 active-run presence lease rather than closing it as obsolete.
