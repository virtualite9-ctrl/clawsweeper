## Summary

Current `main` already provides the requested per-tool-call interception through plugin hooks: `before_tool_call` exposes tool name/params/toolCallId, can rewrite params, block execution, or require user approval before the tool runs. The gap in the issue is real for internal `HOOK.md` hooks, but the capability itself is already implemented on the supported plugin surface.

## Evidence
