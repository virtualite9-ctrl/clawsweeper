## Summary

Keep this PR open. Current main still has the MIME mapping for HTML/XML/CSS, but the host-read media guard only allows validated CSV/Markdown text fallbacks, so host-local HTML/XML/CSS documents still fall through to rejection. The latest PR head remains a focused implementation candidate and addresses the earlier security concern by requiring sniff compatibility plus validated text before allowing the new document MIME types.

## Best Possible Solution

Keep this PR open as the implementation candidate. The best path is to review the narrowed `src/media/web-media.ts` policy change, keep the HTML/XML/CSS acceptance plus renamed-binary rejection tests, resolve or explain the current failing CI checks, and then land if maintainers accept validated text documents beyond CSV/Markdown.
