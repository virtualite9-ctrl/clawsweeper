## Summary

Current main keeps the chosen media-understanding provider/model internally and exposes it in /status, but the prompt-visible inbound media body still omits the actual analysis model. The central request remains open.

## Best Possible Solution

Keep this issue open and implement a narrow upstream change in the media-understanding formatter: include a prompt-visible current-turn analysis provider/model line for each successful media output when known, with focused tests in src/media-understanding/format.test.ts and an apply-path regression proving BodyForAgent receives it. If provider/model is unavailable, the prompt should omit the line or explicitly represent it as unavailable so the assistant does not guess.
