## Close Comment

Closing from a Codex review because this appears implemented on current `main` and already shipped. The Feishu monitor now registers inbound handlers in fire-and-forget mode (`extensions/feishu/src/monitor.account.ts:657-663`), and the message handler returns immediately instead of awaiting dispatch inline (`extensions/feishu/src/monitor.message-handler.ts:320-329`). The changelog also records this exact fix as avoiding websocket event blocking in monitor dispatch (`CHANGELOG.md`). I also verified the same behavior is present in the `v2026.4.22` release commit `00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`, so this no longer looks like an open maintainer cleanup item on supported builds. If the same one-message-only websocket failure still reproduces on `v2026.4.22` or newer, please open a fresh issue with updated logs and config details.
