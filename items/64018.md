## Summary

Current main already resolved the npm-package QA-scenario failure by making the QA scenario pack optional in npm distributions and by explicitly forbidding `qa/` content from shipped tarballs, so this PR's `package.json` change is now obsolete and conflicts with the shipped packaging policy.
