## Summary

Keep #12554 open. Current main still has scoped background process machinery and foreground chat abort, but it does not expose a first-class Gateway/CLI operator action to list and kill all running exec/process sessions for a target OpenClaw session key. The existing `/sessions/:key/kill` HTTP route is subagent-run control, and `sessions.abort` only delegates to `chat.abort`; background execs intentionally survive tool-call abort and can only be managed one process-session id at a time through the agent `process` tool.

## Best Possible Solution

Keep #12554 open. The best path is a first-class Gateway/protocol and CLI operator surface such as `sessions.exec-list` plus `sessions.kill --kill-exec`, or an additive `sessions.abort` option, backed by the existing process registry/supervisor metadata. It should list active background exec/process records for a target session key, terminate only that session's records as a unit, enforce operator/requester authorization, document the contract, and add regression tests proving unrelated sessions are not affected.
