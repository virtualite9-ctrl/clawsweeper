## Summary

Keep #12554 open. Current main has useful lower-level machinery: exec process sessions are scope-keyed, the process tool can list/kill one process session, subagent/task cancellation can stop subagent runs, and a newer HTTP /sessions/:key/kill endpoint exists. But current main still does not expose the requested operator surface to list all exec processes for an OpenClaw session and terminate them as a session-wide unit; background exec sessions are still intentionally preserved across ordinary tool-call aborts.

## Best Possible Solution

Keep the issue open and implement an additive operator surface: a Gateway/protocol method such as sessions.exec-list plus sessions.kill with a killExec/terminateChildren option, or an explicit sessions.abort option with CLI wiring and docs. It should resolve the canonical session key, enumerate process registry/supervisor records for that session scopeKey, terminate all matching exec sessions through supervisor cancellation plus process-tree fallback, return per-process status, enforce operator/requester authorization, and add regression coverage for same-session kill, unrelated-session isolation, background exec behavior, HTTP/RPC/CLI validation, and subagent/task cancellation interactions.

## Risks / Open Questions

- Do not close this as implemented based only on POST /sessions/:sessionKey/kill; that endpoint stops subagent runs but does not expose session-scoped exec process enumeration or kill-all semantics.
- A correct fix needs explicit authorization and isolation so one session/operator cannot inspect or terminate unrelated sessions' exec processes.
- The fix should preserve current behavior where ordinary tool-call abort does not silently kill intentionally backgrounded exec sessions unless the caller requests stronger kill behavior.
