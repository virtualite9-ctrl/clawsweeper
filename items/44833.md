## Summary

Keep #44833 open. Current main still builds a captionless Telegram photo body as `<media:image>`, but the group mention gate checks only Telegram text/caption plus optional audio preflight transcript. For a photo-only group message with `requireMention: true` and `messages.groupChat.mentionPatterns: [".*"]`, the text checked for mentions is empty, so the wildcard pattern does not trigger and the message is skipped before the media-bearing context reaches dispatch. The related #44908 work shipped a WhatsApp/audio preflight fix, but it does not cover Telegram photo-only group activation.

## Best Possible Solution

Keep the issue open and fix it in the Telegram plugin. Add a regression test for a captionless group photo with `groups: { "*": { requireMention: true } }` and `messages.groupChat.mentionPatterns: [".*"]`, then adjust `resolveTelegramInboundBody`/mention gating so media-only Telegram group messages can pass when the configured group activation policy would otherwise respond, while preserving strict mention gating for ordinary groups.

## Risks / Open Questions

- A naive fix that treats every media-only group message as mentioned would weaken `requireMention` in busy groups. The fix should preserve normal mention policy and only allow image-only activation when the group is explicitly configured to respond, such as `requireMention: false`, `/activation always`, or a deliberately all-message pattern policy.
