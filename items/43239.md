## Summary

Keep this PR open for maintainer review or requested changes. The current checkout still does not thread a caller-provided filename through the generic message send pipeline into Feishu media sends, so the underlying bug is real. The PR is not safe to close as implemented or merge as-is because it remains too broad and includes security-sensitive unrelated changes plus review-flagged Feishu media regressions.

## Best Possible Solution

Keep this PR open only as an implementation candidate and request a narrow revision or replacement: thread filename from message tool params through MessageSendParams, payload normalization, ChannelOutboundContext, and Feishu sendMediaFeishu; preserve loadWebMedia/mediaLocalRoots and sanitizeFileNameForUpload; add focused tests for explicit filename forwarding, local basename fallback, remote non-image URLs, local-root rejection, and upload sanitization; drop all unrelated workflow, CODEOWNERS, local-script, local-doc, skill, uv.lock, and OpenAI WebSocket changes.
