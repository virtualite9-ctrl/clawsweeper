## Summary

Keep #70634 open. Current main still uses FIFO command/session lanes and FIFO busy-session followup queues, while `sessions_send` marks inter-session provenance but does not use that metadata to deprioritize agent-originated work behind human-originated messages. The KinthAI comment is useful supporting evidence: it reports the same starvation pattern and describes a two-tier human-before-agent lane dispatcher, but that design is not present in OpenClaw core.

## Best Possible Solution

Keep the issue open and implement a core queue priority contract. Classify work at the channel/gateway boundary, likely using existing `InputProvenance` plus an explicit queue priority, drain external human/user input ahead of `inter_session` agent work within the same session lane, preserve FIFO order inside each tier, and add regression tests covering a `sessions_send` loop with a later Discord/webchat human message.
