## Summary

Current `main` already preserves Feishu inbound mentions in message content instead of dropping them, and the same behavior is present in the latest shipped release `v2026.4.22`.

## Evidence

- **Inbound Feishu message parsing preserves non-bot mentions:** `parseFeishuMessageEvent` parses the raw message text and then calls `normalizeMentions(rawContent, event.message.mentions, botOpenId)`, so non-bot mentions are kept in `ctx.content` rather than discarded.
  - file: [extensions/feishu/src/bot.ts:105](https://github.com/openclaw/openclaw/blob/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4/extensions/feishu/src/bot.ts#L105)

- **Mention normalization renders explicit mention markers:** `normalizeMentions` replaces Feishu placeholder keys with `<at user_id="...">name</at>` for mentions that have an `open_id`, or `@name` as a fallback, which is the behavior the report asked for.
  - file: [extensions/feishu/src/bot-content.ts:246](https://github.com/openclaw/openclaw/blob/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4/extensions/feishu/src/bot-content.ts#L246)
