## Summary

This issue duplicates the same unresolved hardcoded 5 MiB media-store and sandbox-staging configurability gap already tracked by the earlier open #40880. Current main still lacks a schema-backed `media.maxBytes` setting and still uses `MEDIA_MAX_BYTES` in media store, media server, and sandbox staging paths; #40950 is the open, unmerged implementation candidate rather than a shipped fix.

## Best Possible Solution

Close this duplicate and keep #40880 as the canonical tracker. The best implementation path is to repair/land #40950 or submit a fresh focused patch that adds a bounded `media.maxBytes` config contract, preserves the 5 MB default, threads the resolved limit through media store/server and sandbox staging, preserves explicit per-call/channel overrides, and updates config docs plus regression tests.
