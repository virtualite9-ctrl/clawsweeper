## Summary

Keep open. Current main documents per-profile homes as a supported model and adds some mismatch diagnostics, but it does not implement the issue’s core requests: custom profile names still preserve exact casing/end up in distinct state dirs and service labels, so `Eagle` and `eagle` remain different targets, and status output is only a partial mitigation rather than authoritative multi-profile resolution.
