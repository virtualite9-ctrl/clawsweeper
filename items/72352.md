## Summary

Keep PR #72352 open. Current main still does not scope ordinary non-live implicit provider discovery from configured model/provider hints: the filter returns broad discovery for all non-live runs, even though the runtime already supports plugin-id scoping and model/provider ownership helpers exist. The PR is a focused agents/plugin startup optimization, not obsolete or out of scope.

## Best Possible Solution

Keep the PR open. Ask the contributor to address the review comment by making the fallback test exercise a configured model ref with no resolved owner, then validate the changed agents/plugin discovery surface with the repo's changed gate or targeted build/test lane. The implementation belongs in core agents/provider discovery because it optimizes an existing non-live startup path using existing plugin ownership and onlyPluginIds APIs.
