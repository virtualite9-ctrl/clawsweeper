## Summary

Keep open. Current main still returns broad provider discovery for all non-live runs, while this PR is a focused core agents performance fix that scopes discovery through existing provider/plugin ownership and runtime allowlist seams. The provided review context found only a test-quality gap, not a production correctness blocker.

## Best Possible Solution

Keep the PR open, address the review comment by making the fallback regression test exercise a configured model ref with no resolved owner, then run the changed agents/plugin checks or build gate expected for provider discovery runtime changes. The implementation belongs in core because it optimizes the existing agents model-provider discovery path and uses existing plugin ownership/filter APIs.
