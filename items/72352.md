## Summary

Current main still does not implement this PR's central optimization: ordinary non-live implicit provider discovery remains unscoped and returns `undefined` before consulting configured model/provider hints. The PR is a focused agents startup optimization that uses already-present provider ownership and `onlyPluginIds` discovery seams, with one review-noted test coverage issue to fix before merge.

## Best Possible Solution

Keep this PR open. Ask the contributor to address the review-noted fallback test gap, then validate the agents/provider discovery surface with the changed gate or targeted build/test lane before merging. The implementation belongs in core agents startup discovery because current main already has the provider ownership helpers and scoped `onlyPluginIds` runtime seam, but does not apply them to ordinary non-live configured model/provider discovery.
