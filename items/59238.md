## Summary

Current main, and the shipped v2026.4.24 tag, now carry Slack per-channel `systemPrompt` beyond assembly into the active agent prompt. Slack resolves the channel config into `GroupSystemPrompt`, the reply runner includes that prompt in both dynamic and static `extraSystemPrompt` parts, embedded Pi sessions apply the generated prompt to the live session plus Pi rebuild hooks, and CLI-backed sessions receive the same prompt with stable reuse hashing. The Discord voice follow-up remains separate in #47095.

## Best Possible Solution

Close this Slack text/channel issue as implemented and keep the current shared prompt-delivery path. Track remaining platform-specific gaps separately, especially the still-open Discord voice transcript path in #47095.
