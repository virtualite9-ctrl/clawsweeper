## Summary

Current `main` does not make this issue safely closable. The Feishu integration is still documented and packaged as a bot/channel plugin with doc/wiki/drive-style tools, not a full Feishu Open Platform agent-capability publishing surface; separately, the `nodes` tool is intentionally owner-only and `calendar.add` is still treated as a dangerous node command that requires explicit allowlisting. There is now a supported `commands.ownerAllowFrom` config for expanding owner-only access, but that is a manual config path, not evidence that the reported behavior was removed or that the requested Feishu meeting/calendar capability is fully implemented.

## Risks / Open Questions

- The report mixes at least two different surfaces: Feishu Open Platform agent permissions and OpenClaw's own Feishu bot plus node-command security model. That makes the root cause ambiguous, but not incoherent enough to close.
- If maintainers think `commands.ownerAllowFrom` plus `gateway.nodes.allowCommands` is the intended answer, this is better handled as a support/docs clarification; current main still does not show that the requested broader Feishu meeting/calendar capability was implemented.
