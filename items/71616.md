## Summary

Keep open. Current `main` has logic intended to append final assistant replies and replay deferred `session.message` reloads, but I did not find high-confidence evidence that the reported Control UI regression is already fixed. The existing tests cover adjacent terminal-event cases, not the exact successful-reply path described in #71616.

## Evidence

- **Current final-reply handling is only an intended code path, not proof the bug is fixed:** `handleChatEvent` appends the final assistant message or falls back to `chatStream`, then clears `chatRunId`. That is the expected rendering path for immediate assistant replies, but it does not by itself prove the reported UI behavior is resolved.
