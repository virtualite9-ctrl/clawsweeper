## Summary

Current main already fixes the Control UI race behind this report. Final chat events append the assistant reply immediately, then the final history reload preserves local user/assistant messages when `chat.history` returns either an older snapshot or an empty first-turn snapshot. This overlaps with the previously closed #70702 report, and current main extends that fix for the empty-history case described by first-message webchat flows.

## Best Possible Solution

Keep the current main implementation and close this issue as already fixed. No new core API or product work is needed; if a reporter can still reproduce on a build after the 2026.4.25 line ships, they should open a fresh issue with gateway logs, browser console output, and the observed `chat`/`chat.history` event sequence.
