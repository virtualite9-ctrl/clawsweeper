## Summary

Current main at 7e13f3f5140168e31027b39a9855ade3fadf24d5 still does not implement the requested trusted-proxy shared-secret fallback. The dispatcher returns trusted-proxy failures before token/password/Tailscale auth can run, startup validation rejects trusted-proxy plus token config, tests assert valid local direct tokens are rejected, and docs describe the current fail-closed contract. The linked direct fix PR #17746 is closed unmerged and the later fallback PR #64122 is still only related open work in the provided context, so this issue should remain open.

## Best Possible Solution

Keep #17761 open until maintainers either land an intentional trusted-proxy shared-secret fallback or explicitly decide the current fail-closed trusted-proxy contract is final. A complete fallback fix would need coordinated changes in `src/gateway/auth.ts`, `src/gateway/auth-resolve.ts`, startup validation, regression/e2e tests, and `docs/gateway/trusted-proxy-auth.md`, likely by continuing or superseding #64122.
