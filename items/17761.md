## Summary

Current main still does not implement the requested trusted-proxy shared-secret fallback. The auth dispatcher returns trusted-proxy failures before token/password/Tailscale checks, validation rejects trusted-proxy plus shared token config, tests assert valid local tokens are rejected, and public docs describe fail-closed loopback/mixed-token behavior.

## Best Possible Solution

Keep #17761 open until maintainers either land an intentional trusted-proxy shared-secret fallback or explicitly decide the fail-closed trusted-proxy contract is final. A complete fix would need coordinated changes in `src/gateway/auth.ts`, `src/gateway/auth-resolve.ts`, startup validation, regression tests, and public docs.
