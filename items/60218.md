## Summary

Current `main` already addresses this as an LLM idle-timeout and edit-diagnostics problem rather than a hard edit-tool write-size limit: the idle watchdog default is now 120s and configurable, idle-timeout failures point at the correct config key, and the host edit path has post-write recovery plus better mismatch diagnostics.

## Evidence
