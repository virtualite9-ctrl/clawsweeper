## Summary

Keep open. Current main still contains the Control UI churn and stale-index behavior this PR targets: unbounded chat/session refreshes, eager node/device loading on gateway hello, unconditional node polling, immediate sessions.changed reloads, and no-cache rather than no-store for index.html. The PR is a focused product fix with tests in its diff, but it needs the existing review comment addressed before merge.

## Best Possible Solution

Keep PR #69454 open. The best path is to rebase it onto current main, address the review by re-checking the tab inside the sessions.changed timeout or clearing the timer on tab changes, preserve the targeted regression tests for bounded session loads, tab-scoped node polling/preloads, debounced session reloads, and index no-store headers, then run the relevant Control UI and gateway checks.
