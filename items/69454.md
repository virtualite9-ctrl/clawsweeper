## Summary

Keep #69454 open. The PR targets a real, still-present Control UI churn/caching problem: current main still has unbounded chat session refreshes, eager node/device loads on gateway hello, global node polling, immediate `sessions.changed` reloads, and `index.html` served with `Cache-Control: no-cache`. The provided discussion is also actionable: Greptile found the direction safe and tested, while the Codex review comment identifies a concrete tab-switch guard that should be fixed before landing.

## Best Possible Solution

Keep the PR open and ask for an update against current main. The best path is to land the same narrow Control UI performance hardening, with the deferred reload tab-switch bug fixed and focused tests covering bounded session refreshes, Nodes-tab-scoped polling/preloads, debounced/suppressed `sessions.changed` reloads, and `index.html` `no-store` cache headers.
