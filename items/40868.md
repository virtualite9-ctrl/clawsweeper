## Summary

The reported isolated-cron timeout regression is already fixed on current `main`, with the fix documented as shipped in `v2026.3.13` and reflected in the current cron embedded-run lane handling.

## Close Comment

Closing as implemented on `main` after Codex review. The shipped fix is called out in `CHANGELOG.md:2555` for `v2026.3.13` (`Cron/isolated sessions: route nested cron-triggered embedded runner work onto the nested lane so isolated cron jobs no longer deadlock...`). Current `main` still contains that behavior in `src/cron/isolated-agent/run-executor.ts:179`, which routes cron embedded runs through `src/agents/lanes.ts:8` so nested work does not stay on the `cron` lane, and `src/agents/lanes.test.ts:14` covers that regression path. There was also a related isolated-cron queue-path fix in `CHANGELOG.md:2600` for `v2026.3.12`, and the reporter later confirmed the original timeout pattern was resolved in `v2026.3.13`. If someone still hits isolated cron timeouts on a newer build, please open a fresh issue with the current version, cron job config, and logs.
