## Summary

Keep open. Current `main` does not implement the PR’s fail-fast startup auth guard, and the proposed implementation is stale against today’s code plus too weak for current auth resolution semantics.

## Risks / Open Questions

- Closing this PR would drop a still-unimplemented bugfix request; current `main` still lacks the fail-fast startup auth guard described in the PR.
- Reviving or landing the PR as-is would risk false startup failures for providers authenticated via explicit config API keys, synthetic local auth, or AWS-SDK/default-chain auth because it relies on the narrower `hasAuthForProvider` helper.
- The PR also targets a pre-split gateway startup file, so any salvage requires a rebase/rework onto `src/gateway/server-startup-post-attach.ts` and current auth helpers.
