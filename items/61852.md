## Summary

Current `main` does not show a repo-side fix for the reported max-tokens cap problem. The generic Anthropic transport still accepts an explicit `maxTokens` override without clamping it to `model.maxTokens`, while Bedrock discovery does carry provider-specific `maxTokens` metadata and the Anthropic Vertex path already shows the desired clamp. Current overflow recovery does catch compaction failures and return a blocked error instead of crashing outright, so the crash symptom may be mitigated, but the requested cap behavior is not strongly evidenced as implemented on `main`.
