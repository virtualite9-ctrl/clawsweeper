## Summary

Keep open. Current `main` still hard-codes the TUI footer, the public plugin SDK/docs expose no footer-status contribution hook such as `registerStatusProvider`, and `VISION.md` explicitly treats missing plugin seams as valid core plugin-API extension work rather than something to close as ClawHub-only.

## Evidence

- **TUI footer is still closed over core session state only:** `updateFooter()` builds the footer from fixed core fields (`agent`, `session`, `model`, thinking/fast/verbose/reasoning, tokens) and writes them directly to the footer text. There is no plugin lookup, registry, or contribution path in the footer render path.
  - file: [src/tui/tui.ts:816](https://github.com/openclaw/openclaw/blob/5b8bd6371c666a5de43c82305e0315771e8e49d5/src/tui/tui.ts#L816)

  - sha: [5b8bd6371c66](https://github.com/openclaw/openclaw/commit/5b8bd6371c666a5de43c82305e0315771e8e49d5)
- **Public plugin API has no status-provider registration:** `OpenClawPluginApi` on current `main` exposes many registration points (`registerTool`, `registerHook`, `registerCommand`, `registerContextEngine`, memory/runtime/provider hooks, etc.), but no `registerStatusProvider` or comparable TUI footer contribution API.
  - file: [src/plugins/types.ts:2023](https://github.com/openclaw/openclaw/blob/5b8bd6371c666a5de43c82305e0315771e8e49d5/src/plugins/types.ts#L2023)

The documented tables include providers, tools, commands, hooks, services, and memory supplements, but nothing for TUI footer/status contributions.
