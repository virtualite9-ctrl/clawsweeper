## Summary

Current main already ships the behavior this PR proposes: isolated cron runs keep the `message` tool available for non-webhook delivery modes, direct message-tool sends suppress runner fallback when they hit the resolved target, and the user-facing cron docs describe the same shared-delivery model. The feature is present in the released `v2026.4.23` build, so this PR is redundant.
