## Summary

Keep open. Current `main` and the latest shipped tag still contain the reported document-click listener leak in `ui/src/ui/chat/grouped-render.ts`, so this is not implemented already. The PR is directionally useful, but conservative cleanup should not close it: the current code also leaves the listener behind when an already-open popover is dismissed by clicking the delete button again, and there is no regression test covering listener cleanup.

## Evidence

- **Current main still removes the popover without unregistering the document listener on Cancel/Delete:** On `main` (`2cd2732ab6695a4b3220ce41166b74aca7097c5f`), `cancel` calls `popover.remove()` directly and the `yes` handler also removes the popover directly, while the `document.removeEventListener(...)` call only exists inside `closeOnOutside`.
  - file: [ui/src/ui/chat/grouped-render.ts:562](https://github.com/openclaw/openclaw/blob/2cd2732ab6695a4b3220ce41166b74aca7097c5f/ui/src/ui/chat/grouped-render.ts#L562)

  - sha: [2cd2732ab669](https://github.com/openclaw/openclaw/commit/2cd2732ab6695a4b3220ce41166b74aca7097c5f)
