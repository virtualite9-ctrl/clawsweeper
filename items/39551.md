## Summary

Current `main` still rejects `sessions_send` calls when both `sessionKey` and `label` are present, so this PR is not implemented on `main` and should remain open for maintainer review.

## Evidence

- **Current guard still rejects mixed target fields:** `src/agents/tools/sessions-send-tool.ts:105-113` reads `sessionKey` and `label` and immediately returns `Provide either sessionKey or label (not both).` when both are present. That is the exact behavior this PR proposes to remove.

  - sha: [cb2c36b04975](https://github.com/openclaw/openclaw/commit/cb2c36b04975e34ab7014a11575d6589aa96ae32)
- **Schema still allows redundant fields together:** `src/agents/tools/sessions-send-tool.ts:37-43` defines `sessionKey`, `label`, and `agentId` as independent optional parameters, so mixed-input tool calls are structurally allowed by the current schema.
  - file: [src/agents/tools/sessions-send-tool.ts:37](https://github.com/openclaw/openclaw/blob/cb2c36b04975e34ab7014a11575d6589aa96ae32/src/agents/tools/sessions-send-tool.ts#L37)
