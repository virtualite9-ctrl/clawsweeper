## Summary

Keep open. Current main still contains the exact `sessions_send` rejection when both `sessionKey` and `label` are present, and the PR is a focused fix with regression coverage for that live mixed-selector path. Related PRs overlap, but the provided discussion does not identify a merged fix or maintainer-selected canonical replacement.

## Best Possible Solution

Keep #39551 open until maintainers choose and land one canonical fix for the live `sessions_send` mixed-selector bug. The best implementation is the narrow executor-side precedence rule proposed here: normalize a non-empty `sessionKey` as authoritative, ignore redundant `label` and `agentId` for target lookup, preserve label-only behavior and downstream visibility/A2A checks, update the model-facing description, and keep a regression test proving mixed inputs send to the `sessionKey` target without calling `sessions.resolve`.
