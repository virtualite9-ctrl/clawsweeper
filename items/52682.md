## Summary

Current `main` still routes cron isolated agent runs through `CommandLane.Nested`, and `Nested` still inherits the command queue default `maxConcurrent=1` because gateway startup/reload only configure `Cron`, `Main`, and `Subagent`. I did not find a main-branch fix or config surface for nested-lane concurrency, so this should stay open.
