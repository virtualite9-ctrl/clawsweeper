## Summary

Current main still does not implement a general `web_fetch` private-network opt-in. The shipped config/runtime surface only supports `tools.web.fetch.ssrfPolicy.allowRfc2544BenchmarkRange`, which helps fake-IP proxy ranges but does not allow localhost, LAN, or other private/internal addresses requested by #39604. Related PR #39630 remains open/unmerged, and #43700 was closed unmerged, so the issue is not obsolete or shipped.

## Best Possible Solution

Keep #39604 open until maintainers either land an explicit web_fetch private-network opt-in or decide not to support it. The likely implementation path is to update or replace PR #39630 against current main, wire a deliberately named default-off config under `tools.web.fetch.ssrfPolicy` (for example `dangerouslyAllowPrivateNetwork`) through schema/types/generated help, runtime fetch policy, cache key isolation, docs, and SSRF tests, while preserving the existing RFC2544-only option for fake-IP proxy users.
