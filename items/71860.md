## Summary

Keep PR #71860 open. Current main and latest release still relay successful structured exec completions by default, and current tests still expect `Exec completed (..., code 0)` to produce a Telegram message. Related #72253 overlaps and may be the broader consolidation path, but it is also open and there is no maintainer/canonical signal strong enough to close this focused PR as superseded. The PR discussion also shows the Greptile mixed-batch concern was addressed in the PR head.

## Best Possible Solution

Keep #71860 open for maintainer review. The best path is to decide the heartbeat exec-completion contract, then either land/refine this narrow successful-code-0 suppression with its regression tests or explicitly consolidate it into #72253 if maintainers choose that broader PR as canonical.
