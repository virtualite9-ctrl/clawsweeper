## Summary

Keep open. Current main has partial gateway cleanup for startup failures, but the Discord bot-identity failure path behind this PR is still not resolved: fetchUser('@me') failures return an undefined botUserId, startup proceeds, and require-mention handling still depends on botId being present. The PR also has unresolved review concerns, so it needs rework or maintainer judgment rather than cleanup closure.

## Best Possible Solution

Keep this PR open or supersede it with a focused Discord bot-identity startup fix. The fix should choose the intended policy (fail closed on unresolved bot identity, or use a validated applicationId fallback), preserve pre-lifecycle gateway cleanup with reconnect suppression if needed, remove unsafe botId-dependent mention short-circuits where appropriate, and add deterministic regression coverage for fetchUser('@me') rejection and missing-id responses.
