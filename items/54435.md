## Summary

Current main still contains the reported literal `session.store` failure mode, and the linked implementation candidate #54716 is open and unmerged. This issue should stay open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open while #54716 is reviewed and either landed or replaced with an equivalent fix. The fix should update `src/config/sessions/combined-store-gateway.ts` so literal per-agent paths under `agents/<id>/sessions/sessions.json` use all-agent discovery while preserving true shared literal-store behavior, and it should add regression coverage for that literal path.
