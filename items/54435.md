## Summary

Keep open. Current main still short-circuits literal `session.store` paths in `loadCombinedSessionStoreForGateway`, so a literal `.../agents/main/sessions/sessions.json` reads only that one file and attributes entries to the default agent. The broader discovery helper can scan sibling agent stores, but this path is bypassed for non-templated stores. PR #54716 is open, unmerged, and uses closing syntax for this issue, so the right maintainer action is to review, adapt, and land or close that PR before closing this issue.

## Best Possible Solution

Keep this issue open while PR #54716 is reviewed. The best fix is to adapt the PR's literal per-agent store discovery behavior to current `src/config/sessions/combined-store-gateway.ts`, keep the single shared-store fallback for non-agent-layout literal paths, add the literal-path regression on current main, and close this issue only after that or an equivalent fix lands.
