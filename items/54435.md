## Summary

Current main still has the reported literal `session.store` failure mode. The gateway `sessions.list` handler depends on `loadCombinedSessionStoreForGateway`, which still returns after reading one non-templated store path and canonicalizing entries to the default agent before all-agent discovery can run. The issue also has an open linked implementation candidate, PR #54716, with closing syntax, so it should remain open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open while PR #54716 is reviewed, rebased if needed, and either landed or replaced with an equivalent fix. The fix should update the current `src/config/sessions/combined-store-gateway.ts` literal-path handling so literal per-agent store paths under `agents/<id>/sessions/sessions.json` use all-agent discovery, preserve single-store behavior for true shared literal paths, and add a regression test for the reported literal path.
