## Summary

Keep open. Current main still includes the whole state directory as the first backup asset, and `--no-include-workspace` only prevents adding separate workspace candidates; it does not filter nested workspace contents out of `~/.openclaw`. The existing docs and tests describe/lock in the same behavior the reporter called misleading, and the broader exclude-pattern work is still in an open PR rather than shipped on main.

## Best Possible Solution

Keep this issue open until native backup either makes `--no-include-workspace` materially exclude nested workspace directories from the state archive, or intentionally clarifies the flag/docs/CLI copy and routes real nested exclusions through a shipped `--exclude` / `--exclude-file` mechanism. The likely implementation point is `src/commands/backup-shared.ts` plus `src/infra/backup-create.ts`, with regression coverage for a `~/.openclaw/workspace` file not appearing in the archive when the exclusion path is requested.
