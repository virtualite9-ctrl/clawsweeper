## Summary

Keep open. Current `main` still implements `--no-include-workspace` as "skip workspace discovery" rather than excluding workspace contents nested under the state asset, and both CLI help and docs still describe it as excluding workspace files / making backups smaller.

## Evidence

- **Planner always includes state; flag only removes discovered workspace roots:** `resolveBackupPlanFromPaths()` always seeds the candidate list with the state directory, while `includeWorkspace` only controls whether `workspaceDirs` are added as separate candidates.
  - file: [src/commands/backup-shared.ts:97](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/commands/backup-shared.ts#L97)
  - command: `nl -ba src/commands/backup-shared.ts | sed -n '70,220p'`
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Nested workspaces are treated as covered by state, not excluded from archive contents:** The planner marks any candidate whose canonical path is inside an already-included asset as `reason: "covered"`; with state included first, a workspace under `~/.openclaw` is skipped only as a duplicate top-level entry.
  - file: [src/commands/backup-shared.ts:195](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/commands/backup-shared.ts#L195)

- **Tests codify the reported behavior on current main:** `backup.test.ts` explicitly expects the default workspace under `~/.openclaw` to collapse into a single `state` backup root and records the workspace as `covered` rather than excluded.
  - file: [src/commands/backup.test.ts:87](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/commands/backup.test.ts#L87)

  - file: [src/cli/program/register.backup.ts:28](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/cli/program/register.backup.ts#L28)
