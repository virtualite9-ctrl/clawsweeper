## Summary

Current main still does not make `backup create --no-include-workspace` materially exclude default workspaces nested inside the state directory. The flag skips separate workspace discovery, but the backup still includes the state tree as an archive source and the writer has no nested workspace exclusion filter.

## Best Possible Solution

Keep this issue open until the backup planner/archive writer either excludes nested workspace directories when workspace inclusion is disabled, or the CLI/help/docs contract is explicitly narrowed so users understand it only skips separate workspace discovery. The strongest fix belongs in `src/commands/backup-shared.ts` and `src/infra/backup-create.ts`, with an archive-level regression test proving a file under the default nested workspace is absent. Maintainers can also use #40786/#44288 as the broader exclude-pattern path, but this issue should remain open until that work lands and covers this flag’s specific semantics.
