## Summary

Keep open. Current main still does not implement the PR’s central plugin-config graceful-degradation request, and the PR is a salvageable but unsafe fix attempt rather than obsolete cleanup: it changes runtime loader fail-closed behavior, leaves config-file validation fail-closed, and bundles unrelated fixes.

## Best Possible Solution

Keep the PR open for maintainer review, but do not merge it as-is. The best fix should target #70371/#61683 directly: make plugin-scoped config validation degrade to a per-plugin disabled/error state with clear diagnostics and doctor output, preserve explicit fail-closed semantics for required or security-critical runtime loads, avoid caching transient failed registries, and split unrelated qqbot/media/Vertex changes into separate PRs or drop the parts already handled on main.
