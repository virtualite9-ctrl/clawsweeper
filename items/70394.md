## Summary

Keep open. Current `main` does not implement the requested plugin-loader change, and the PR's main premise is not accurate for `ensurePluginRegistryLoaded`: that path is still used for full runtime activation, not just validate/reporting flows. The branch also mixes unrelated changes, while current `main` has already diverged in at least one of those areas, so this is not a safe cleanup close.
