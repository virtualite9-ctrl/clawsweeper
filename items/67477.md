## Summary

Keep this PR open. Current main shipped the shared inbound `message_sending` cancel/rewrite path via #70118, but WhatsApp auto-reply delivery still does not emit plugin `message_sent` or internal `message:sent` hooks, which is part of this PR's unique stated scope.

## Best Possible Solution

Keep this PR open, but rebase and narrow it around the still-missing WhatsApp auto-reply sent-hook parity. The already-shipped `message_sending` part should rely on the shared dispatcher behavior from #70118; the remaining patch should emit plugin `message_sent` and internal `message:sent` only from accurate delivered outcomes, with focused tests for cancellation, rewrites, blanked content, media fallback, chunking, and failure paths.
