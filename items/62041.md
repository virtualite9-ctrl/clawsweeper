## Summary

Current `main` already makes the media-delivery distinction explicit again: the system prompt documents `MEDIA:` as assistant delivery metadata, async media completions explicitly prefer the `message` tool to the current/original chat with `MEDIA:` only as fallback, and tests cover those paths. The same implementation is present in release `v2026.4.23`.

## Evidence

- **System prompt explicitly separates attachment delivery from other render paths:** `buildAssistantOutputDirectivesSection` documents `MEDIA:<path-or-url>` as attachment-delivery metadata, and `buildWebchatCanvasSection` separately says `[embed ...]` is distinct from `MEDIA:`.
