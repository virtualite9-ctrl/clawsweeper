## Summary

Keep this PR open. The linked Telegram TTS/voice-note bug has coverage in the direct bot reply path on current main, but the outbound plugin sendPayload path this PR targets is still not fully fixed: current Telegram sendPayload code still does not translate payload.audioAsVoice into the sendMessageTelegram asVoice option, while the Telegram sender only switches to sendVoice when asVoice is true. The PR needs a rebase to the current extension file layout and should address the existing review concern about text-only audioAsVoice payloads bypassing normal text chunking.

## Best Possible Solution

Keep the PR open for maintainer/contributor follow-up. The best path is to rebase it onto current main, move the Telegram changes into extensions/telegram/src/outbound-adapter.ts or the shared sendTelegramPayloadMessages path, translate payload.audioAsVoice or ctx.audioAsVoice into asVoice only for media sends, preserve normal text chunking for text-only payloads, and add focused regression tests in extensions/telegram/src/outbound-adapter.test.ts plus the existing outbound delivery test surface.
