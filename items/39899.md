## Summary

Keep PR #39899 open. Current main partially mitigates the reported loop by stripping unpaired surrogate code units from bundled text transport payloads and dropping failed assistant turns before replay, but it still does not implement the PR's surrogate-specific detector, friendly formatting, or session-reset recovery. Source inspection also shows replayed tool-call arguments still bypass the text sanitizer, so the PR is not safely obsolete from current-main evidence alone.

## Best Possible Solution

Keep the PR open for maintainer review. Rebase it onto current main and add a focused regression test that injects an unpaired surrogate through the remaining plausible paths: provider error formatting, OpenAI Responses tool-call arguments, and Anthropic tool_use inputs. If the test still reaches runAgentTurnWithFallback with a surrogate JSON rejection, implement the smallest detector, friendly message, and session reset branch consistent with the existing role-ordering and Gemini-corruption recovery. If the tests prove every supported path is already sanitized, close with that proof or replace the PR with a narrower sanitizer-hardening patch.
