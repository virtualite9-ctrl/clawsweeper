## Summary

Keep open. Current main still implements the reported behavior: AGENTS.md is normally already injected into the system prompt, but compaction also reinjects selected AGENTS.md sections into both the post-compaction follow-up context and the persisted compaction summary. The added `postCompactionSections: []` opt-out is useful, but it is not the default and it does not cover the safeguard summary suffix path, so the issue's default bloat/magic-heading concern remains actionable.

## Best Possible Solution

Keep #45488 open and fix the default compaction contract. The conservative path is to skip AGENTS.md reinjection when normal bootstrap/system-prompt context is retained, remove the independent `workspace-critical-rules` summary suffix or gate it behind the same explicit opt-in config, and add regression coverage proving repeated compactions do not duplicate AGENTS.md sections into the transcript/system-event path by default. If maintainers still want recovery prompts, make them an explicit config/template surface instead of hardcoded parsing of user-owned AGENTS.md headings.
