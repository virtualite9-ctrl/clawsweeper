## Summary

Current `main` already addresses the reported silent bootstrap truncation: bootstrap size limits are configurable, truncation produces an agent-visible warning block by default, and docs describe `/context` visibility for raw vs injected sizes. This behavior is also present in the latest release `v2026.4.22`.

## Close Comment

Closing from a Codex review because current OpenClaw already addresses the reported silent truncation behavior. On `main`, bootstrap truncation is configurable through `agents.defaults.bootstrapMaxChars` and `agents.defaults.bootstrapTotalMaxChars` (`src/agents/pi-embedded-helpers/bootstrap.ts`), and truncation now produces an agent-visible `[Bootstrap truncation warning]` block with affected-file details (`src/agents/bootstrap-budget.ts`). The current docs also state that `/context` shows raw vs injected sizes and whether truncation happened (`docs/concepts/context.md`). This behavior is already present in the latest shipped release, `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`). The specific idea of dropping the oldest content first is not how truncation currently works, but the reported bug of silent truncation without user-visible warning is already fixed.
