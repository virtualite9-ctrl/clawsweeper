## Summary

Keep PR #70799 open. Current main does not include the proposed outer guards for BlueBubbles typing/read receipt calls, the touched production path still invokes those helpers for inbound webhooks, and no provided related item supersedes this specific fix.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is a BlueBubbles-owned fix that prevents Private-API-only typing/read-receipt calls from producing user-visible failures when Private API is disabled, with a focused regression test covering the production mismatch. If the current outer-guard approach is accepted, preserve enabled/null probe semantics and keep the behavior scoped to the BlueBubbles plugin.
