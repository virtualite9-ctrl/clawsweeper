## Summary

Current main already solves the PR's observable GitHub Copilot IDE-header inference failure. The shipped Copilot provider stream wrapper injects dynamic IDE headers for both Anthropic Messages and OpenAI Responses, compaction summarization also adds them, and docs/tests cover the intended behavior. The PR's prepareRuntimeAuth change no longer has a remaining product delta to merge.

## Best Possible Solution

Close this PR as already implemented. Keep the shipped provider stream-wrapper approach, which covers Anthropic, OpenAI Responses, and compaction paths without adding a public SDK export or changing GitHub Copilot prepareRuntimeAuth solely for duplicate header injection.
