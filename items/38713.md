## Summary

Keep PR #38713 open. Current main still resolves `skills info <name>` with exact-only `name`/`skillKey` matching, the CLI passes the raw argument into that formatter, and current tests do not cover case-insensitive or separator-normalized lookups. The linked predecessor PR #38701 was closed unmerged, and the PR still addresses a real mismatch class documented by `skillKey` overrides.

## Best Possible Solution

Keep this PR open. Refresh it against current main and land a focused `skills info` resolver in `src/cli/skills-cli.format.ts` that trims the requested name, preserves exact matches first, allows only unique case-insensitive and separator-normalized matches, sanitizes not-found output, and adds regression tests in `src/cli/skills-cli.test.ts`; track any separate agent-facing activation issue independently.
