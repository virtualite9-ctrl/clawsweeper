## Summary

Keep PR #38713 open. Current main still resolves `openclaw skills info <name>` with exact `name`/`skillKey` equality only, forwards the raw CLI argument into that formatter, and lacks the PR's case-insensitive or separator-normalized regression coverage. The predecessor PR #38701 is closed unmerged, and the linked bug #38546 was closed without this fix landing on main.

## Best Possible Solution

Keep this PR open. The best path is to refresh it against current main and land a focused `skills info` resolver in `src/cli/skills-cli.format.ts` that trims the requested name, preserves exact matches first, allows only unique case-insensitive and separator-normalized matches, sanitizes not-found output, and adds regression tests in `src/cli/skills-cli.test.ts`; track any separate agent-facing activation issue independently.
