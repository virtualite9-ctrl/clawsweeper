## Summary

Keep open. Current `main` does not ship gateway A2A support, and the PR branch still shows unresolved correctness gaps in the JSON-RPC handler, so there is not strong evidence for any conservative close reason.

## Evidence

- **Current main has no A2A gateway routes:** `createGatewayHttpServer` wires hooks, models, embeddings, tools, session history/kill, OpenResponses, and OpenAI handlers, but no `/.well-known/agent.json` or `/a2a` stage exists on `main`. That means the feature is not already implemented on current `main`.
