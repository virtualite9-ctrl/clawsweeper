## Summary

Keep #70451 open. Current main addresses the broader #64306-style bootstrap preload for status/health/gateway status, but the hook-specific commands in this report still build a hook report through full plugin diagnostics/module loading during `hooks list` and `hooks enable`, so there is not strong enough evidence that the Windows hooks SIGKILL path is fixed.

## Best Possible Solution

Keep #70451 open and fix the hooks CLI path directly. The likely maintainer path is to make `hooks list`, `info`, `check`, and `enable` use cold metadata/config where possible, or add an explicit config-only path for enabling a known hook without full plugin runtime diagnostics. Add focused coverage for `openclaw hooks --help`, `hooks list`, and `hooks enable <bundled-hook>` on the no-plugin-runtime path, and document the supported config-only workaround/API status.
