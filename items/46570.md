## Summary

Keep open. Current main indexes memory files and has source-aware FTS/vector queries, but #46570's observable failure is not proven fixed for the builtin local hybrid path. The current hybrid selection still returns strict-scoring results immediately when any strict hit exists, so keyword-backed memory-file hits can remain below the default minScore/text-weight floor while session hits are returned. The closest builtin hybrid regression test for this edge is still skipped.

## Best Possible Solution

Keep #46570 open and add a focused builtin memory-core regression test: configure `sources: ["memory", "sessions"]`, enable session memory and hybrid defaults, create an exact memory-only keyword hit plus higher-scoring session/vector hits, then assert the memory hit is retained in `openclaw memory search` results. If it fails, adjust builtin hybrid final selection to preserve source diversity or relax keyword-backed memory hits before slicing when sessions dominate strict results.
