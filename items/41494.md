## Summary

Keep this issue open. Current main has several partial protections for structured Gemini/native reasoning and tagged reasoning, but the broader reported leak of Gemini reasoning as ordinary visible text, especially appended prose or multilingual preambles, is not implemented on main.

## Best Possible Solution

Keep this issue open as the umbrella for Gemini plain-text reasoning leakage. The best path is to review and land a narrow shared outbound sanitizer, likely through #60551 or a replacement, with provider/channel tests covering untagged planning prose, appended reasoning blocks, multilingual reasoning headers, tagged fragments, and normal prose pass-through.

## Evidence

- **Recent issue discussion reports unresolved user impact:** The provided GitHub context includes affected-user retests through 2026-04-19 on v2026.4.15, explicitly reporting multilingual reasoning headers such as Reasoning, Tankegang, Analyse, Begrunnelse, Analysis, and Thinking, and distinguishing this broader plain-text leak from narrower final-tag leakage in #48593. It also cross-references open candidate PR #60551 for stripping leaked reasoning preambles before outbound send.
- **Native Google transport only hides structured thoughts:** The Google transport maps Gemini parts with `part.thought === true` into `thinking` blocks and ordinary `part.text` into visible `text` blocks, so plain reasoning prose returned as normal text is still treated as deliverable answer content.

## Risks / Open Questions

- A broader preamble sanitizer could strip legitimate prose, quoted content, code, or multilingual answers if implemented too aggressively; the fix needs conservative matching and regression tests for pass-through cases.
- Some provider and channel paths already handle structured reasoning, so the remaining fix should target the actual outbound delivery boundary without regressing intentional `/reasoning on` or structured reasoning display.
