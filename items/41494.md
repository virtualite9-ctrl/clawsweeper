## Summary

Current main has partial Gemini reasoning protections, but it still does not cover the reported leak shape: Gemini reasoning emitted as ordinary visible text, including multilingual reasoning headers or untagged planning prose, before channel delivery.

## Best Possible Solution

Keep this issue open as the umbrella for Gemini plain-text reasoning leakage. The best path is to review and land #60551 or a replacement that adds a conservative outbound sanitizer at the delivery boundary, with provider/channel tests for untagged planning prose, appended reasoning blocks, multilingual reasoning headers, tagged fragments, and normal prose pass-through.

## Evidence

## Risks / Open Questions

- A broader preamble sanitizer can false-positive legitimate prose, quoted examples, code, or multilingual answers if implemented too aggressively.
- Some structured reasoning paths already behave correctly, so the fix should not regress intentional `/reasoning on` display or native `isReasoning` payload suppression.
- The open candidate PR #60551 should be reviewed or superseded rather than bypassed by closing the umbrella before a fix lands.
