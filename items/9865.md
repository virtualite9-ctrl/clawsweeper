## Summary

Keep #9865 open. Current main still lacks first-class provider Batch API routing for background LLM turns in cron, heartbeat, sessions_send, or sessions_spawn/subagent execution. Existing batch-related code is scoped to embeddings/media/non-turn mechanics, while provider execution remains immediate StreamFn-based. The issue also has an explicit same-day maintainer/Codex review comment keeping it open and treating related closed issues as duplicates/follow-up context, not superseding trackers.

## Best Possible Solution

Keep #9865 open as the canonical product/API tracker. The likely maintainer path is an additive provider/runtime batch-turn contract with submit, poll, cancel, timeout, failure, usage, and reconciliation semantics, followed by explicit opt-in surfaces for cron agentTurn jobs, heartbeat runs, sessions_send, and sessions_spawn/subagents, plus public plugin SDK docs and focused regression coverage. Treat #49326, #70606, and #56126 as closed duplicate/follow-up context rather than replacements.
