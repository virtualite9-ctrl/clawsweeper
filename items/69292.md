## Summary

Keep open. Current main has not implemented this PR's requested doctor auto-enable behavior: non-built-in channel plugins are still considered already enabled only when `plugins.entries.<id>.enabled === true`, then doctor materializes `plugins.entries` and allowlist state. The PR is also not safely obsolete or duplicated by the related closed PRs, but it likely needs revision before merge because workspace plugins are intentionally disabled by default unless explicitly trusted.

## Best Possible Solution

Keep the PR open but request changes: scope the skip to globally installed plugins that are already effectively enabled by discovery, preserve workspace trust gates by still enabling or allowlisting workspace plugins when needed, handle restrictive `plugins.allow`/`allowMissing`, and add regression tests for global-installed, workspace-untrusted, workspace-trusted, bundled, and restrictive-allowlist cases.
