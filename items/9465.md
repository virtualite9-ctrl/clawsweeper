## Summary

Keep #9465 open. Current main still lacks a persisted per-job cron hooks lifecycle (`hooks.pre`, `hooks.post`, `hooks.onError`) in the cron model, Gateway protocol, scheduler/manual execution paths, CLI, and Control UI. The existing plugin `before_agent_reply` seam can short-circuit some cron-driven agent turns with `ctx.jobId`, but it is a global plugin workaround rather than the requested job-defined scheduler contract.

## Best Possible Solution

Keep #9465 open until maintainers either land an additive first-class per-job cron hook lifecycle across persisted cron types, Gateway schemas/RPC, scheduler due/startup/manual paths, CLI, Control UI, docs, and tests, or explicitly supersede it with a canonical narrower tracker such as #45129 for shell pre-hook gating. Document the existing plugin hook path as a workaround, not as the replacement for this cron contract.
