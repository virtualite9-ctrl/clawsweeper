## Summary

Keep #9465 open. Current main still lacks persisted per-job cron hooks, including the narrower shell pre-hook gate requested in comments. Existing plugin hooks can short-circuit some cron-triggered agent runs, but they are global plugin lifecycle hooks, not job-defined pre/post/onError hooks in the cron schema, Gateway protocol, scheduler lifecycle, CLI, or UI.

## Best Possible Solution

Keep #9465 open until maintainers either land an additive first-class cron hook lifecycle across persisted cron types, Gateway schemas/RPC, scheduler startup/timer/manual execution paths, CLI/UI, docs, and tests, or explicitly supersede it with a canonical narrower tracker such as #45129 for shell pre-hook gating. Existing plugin hooks can remain a workaround for some deployments but should not be treated as closing the per-job cron hooks request.
