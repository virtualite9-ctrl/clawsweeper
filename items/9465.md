## Summary

Keep #9465 open. Current main at e29d3516bf05591e6007bfb8ccdaea6d76194d41 still does not implement the requested per-job cron hooks lifecycle or the narrower shell pre-hook gate discussed in the issue. The related PR #45129 is an open implementation path, not shipped behavior, and existing plugin/internal hooks are not equivalent to persisted cron job hooks that can skip a scheduled run before the model call.

## Best Possible Solution

Keep #9465 open until maintainers either land an additive per-job cron hook lifecycle across cron types, Gateway schemas/RPC, automatic startup/manual execution paths, CLI/UI surfaces, docs, and tests, or explicitly supersede it with a narrower canonical tracker such as PR #45129 for shell pre-hook gating.
