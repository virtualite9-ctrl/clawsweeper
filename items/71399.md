## Summary

Keep PR #71399 open. Current main still lacks an `agent:turn:end` internal hook, and this PR remains the focused implementation attempt for that missing ACP turn-completion seam rather than obsolete cleanup. The existing PR discussion also identifies concrete follow-up needed before merge: include the advertised failure `errorCode` in the payload and use bounded hook dispatch or justify not doing so.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to finish this PR or a successor by emitting `agent:turn:end` from `AcpSessionManager.recordTurnCompletion()` on success and failure, include `sessionKey`, `success`, `durationMs`, and optional `errorCode`, use bounded non-blocking hook dispatch or document why this hook is intentionally different, and keep the type guard plus success/failure manager tests aligned with the final contract.
