## Summary

Keep PR #71399 open. Current main still does not implement an `agent:turn:end` internal hook: ACP turn completion only updates latency/error counters, and the internal hook contract/docs have no `AgentTurnEnd` event or `agent:turn:end` catalog entry. The PR is a focused implementation attempt for the same missing surface, but the discussion already identifies merge-blocking contract details: the advertised failure `errorCode` is not in the proposed payload, and the PR uses unbounded fire-and-forget dispatch despite the existing bounded helper. A maintainer comment in the provided thread also explicitly kept this item open for review.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to finish this PR or a successor by adding a typed `agent:turn:end` internal hook, emitting it from `AcpSessionManager.recordTurnCompletion()` on success and failure with `sessionKey`, `success`, `durationMs`, and optional `errorCode`, using bounded non-blocking dispatch or documenting why this hook is intentionally different, and covering the event guard plus success/failure manager emission in tests. Update public internal-hook docs if the event becomes an operator-facing hook surface.
