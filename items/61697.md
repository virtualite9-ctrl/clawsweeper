## Summary

Keep open. Current main shows that `/hooks/gmail` routing exists and is tested, but the issue evidence also points to an underconfigured or mismatched external `gog gmail watch serve` setup. I do not have strong enough proof to close this as fixed or non-reproducible, and current main still has a watcher startup/logging path that can mislead external-webhook setups.

## Evidence

- **Gateway watcher auto-start still triggers from account-only config:** Gateway startup still calls Gmail watcher startup whenever `hooks.enabled` and `hooks.gmail.account` are set, which matches the issue's startup log context even if the user intended to run `gog` separately.
