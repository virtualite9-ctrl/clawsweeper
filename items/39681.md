## Summary

Keep #39681 open. Current main still does not add the requested Advanced Features / Next Steps bridge to `docs/tools/creating-skills.md`; the guide only has the starter workflow, a small metadata table, best practices, locations, and related links. The full Skills reference and implementation/tests show the omitted fields are real supported features, and the exact docs fix is only tracked by open, unmerged PR #59650 in the provided GitHub context.

## Best Possible Solution

Keep #39681 open until PR #59650 or an equivalent docs-only change lands. The best fix is a concise Advanced Features / Next Steps section in `docs/tools/creating-skills.md` linking to the public Skills reference for conditional activation, env/apiKey injection, command dispatch, template variables, invocation controls, testing guidance, and the full frontmatter/config reference.
