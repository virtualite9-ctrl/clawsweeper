## Summary

Keep #69943 open. Current main still has the reported bundled session-memory write/read gap: assistant/user text is extracted verbatim from the session transcript, written verbatim under `## Conversation Summary`, and startup context later loads memory file content without stripping model chat-template tokens, raw tool-call XML, role markers, or `NO_REPLY`. The related PR #69961 directly targets this issue, but it is still separate from `main` and unmerged.

## Best Possible Solution

Keep the issue open until the bundled `session-memory` path is fixed on main. The likely fix belongs in `src/hooks/bundled/session-memory/`: sanitize per-turn transcript text before it is summarized, sanitize again before writing memory files, and consider read-time sanitization in startup context so existing poisoned files cannot be re-injected. PR #69961 is the active implementation candidate and should either land with regression tests or be superseded by an equivalent maintainer-approved fix.
