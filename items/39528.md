## Summary

Keep #39528 open. Current main has meaningful hardening since the original 2026.3.2 report, but it still does not enforce a hard, recent explicit-user-consent gate for the sensitive model-callable paths the issue calls out: gateway restart, allowed config.apply/config.patch writes, and update.run restarts. The existing guardrails are owner-only/admin-scope, allowlist, dangerous-flag, and rate-limit controls, plus prompt text telling the model to ask first; they do not fully solve the reported failure mode where a model loses or ignores prompt-level red-line instructions in a long conversation.

## Best Possible Solution

Keep this open as a product/security-behavior bug. The best fix is an enforceable, test-backed recent explicit-consent mechanism for model-initiated gateway restart, config.apply, config.patch, and update.run, while preserving the current owner-only/admin-scope, allowlist, dangerous-flag, protected-path, and rate-limit guards. Add regression coverage for long sessions and compaction where SOUL.md, USER.md, MEMORY.md, and user red-line instructions remain available or are re-read before sensitive control-plane actions.
