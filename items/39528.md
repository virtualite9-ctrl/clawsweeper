## Summary

Keep #39528 open. Current main has useful hardening since the original 2026.3.2 report, including bootstrap/compaction safeguards, owner-only gateway tooling, admin-scoped control-plane RPCs, config-write allowlists, dangerous-flag guards, and regression tests. The central reported risk is not fully fixed: the model-facing owner-only gateway tool can still schedule a gateway restart, forward allowed config.apply/config.patch writes, and run update.run without a separate recent explicit user-consent token. The relevant files are unchanged from the prior ClawSweeper keep-open review commit to current HEAD, so the earlier assessment still holds.

## Best Possible Solution

Keep this open as a product/security-behavior bug. The best fix is a hard, test-backed recent explicit-consent mechanism for agent-initiated gateway restart, config.apply, config.patch, and update.run, while preserving the current protected-path, dangerous-flag, admin-scope, and rate-limit guards. Add regression coverage for long sessions and compaction where SOUL.md, USER.md, MEMORY.md, and user red-line instructions remain available or are re-read before sensitive control-plane actions.
