## Summary

Keep open. Current `main` contains partial mitigations for this class of problem, but not a strong enough end-to-end fix to close the report: the agent-facing `gateway` tool still allows autonomous `restart` and limited config writes at runtime, while the "ask first" rule is still prompt guidance rather than hard enforcement. The context-loss side also has mitigations, but I did not find issue-specific proof that it is fully resolved on current `main`.

## Evidence

## Risks / Open Questions

- The issue body does not provide a deterministic repro, so I did not prove the exact conversation-level context-loss behavior on current `main`.
- Current main has compaction/memory-flush safeguards, but those are mitigations for context retention, not clear proof that the reported multi-turn loss is fully gone.
- Closing as implemented would overstate the current security posture: restart and some config mutations are still possible through the owner-only gateway tool without a hard explicit-user-consent check.
