## Summary

Keep #70002 open. Current main still lacks the proposed repository guard on both docs workflows, and both jobs still consume the upstream-only `OPENCLAW_DOCS_SYNC_TOKEN` to operate on `openclaw/docs`. The PR is a focused two-line workflow fix, matches existing repository-guard patterns in other workflows, and the discussion includes positive review context rather than evidence that the work is obsolete.

## Best Possible Solution

Keep the PR open for maintainer review and likely merge the narrow job-level `github.repository == 'openclaw/openclaw'` guards, or land an equivalent change on main. The request is actionable in this repository and is not implemented on current main.
