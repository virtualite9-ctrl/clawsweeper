## Summary

Keep this PR open for maintainer review. Current main still has no devcontainer, benchmark workflow, or release SBOM workflow, so it is not implemented. The latest PR commit did address the initial Greptile functional review items, but the remaining diff is still security-sensitive CI/release work and should be revised rather than auto-closed.

## Best Possible Solution

Keep the PR open, but require a focused maintainer revision before merge. Split devcontainer, benchmark, and SBOM if needed; reuse `.github/actions/setup-node-env/action.yml`; call `test:startup:bench` or `test:startup:bench:check`; avoid or checksum-verify privileged tool installs; and generate any release SBOM from the shipped npm tarball or production dependency set. Coordinate the devcontainer part with #62503.

## Risks / Open Questions

- The devcontainer portion overlaps open devcontainer work in #62503 and should be coordinated or split.
- The benchmark workflow overlaps the existing startup benchmark harness and docs unless revised to reuse them.
