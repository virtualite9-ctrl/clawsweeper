## Summary

Keep this issue open. Current main has partial mitigations for bare `/new`/`/reset` memory startup and newer-config guards, but it does not fully resolve the reported 2026.4.10 memory/session continuity regression or the stale pinned CLI path/version-skew scenario. The same-author related issue #65358 is also still open, so closing only this side would split an unresolved paired report.

## Best Possible Solution

Keep this open or split it deliberately. Use #65358 as the canonical tracker for the 2026.4.10 memory/session continuity regression, then track the stale CLI-path work as an update/doctor/install improvement: enumerate PATH-resolved and known-location `openclaw` binaries, compare versions, inspect LaunchAgent/cron command paths that point at older binaries, and safely warn or guide repair/removal of stale wrappers when switching install methods.
