## Summary

Keep PR #69199 open. Current main still routes missing node:sqlite failures through the generic embedding/provider unavailable payload, and the existing tests only cover quota and generic failures. The PR remains the active narrow fix for the UX bug; the related fallback-search work is separate and still tracked elsewhere.

## Best Possible Solution

Keep #69199 open for maintainer review and likely merge after addressing, or explicitly accepting, the small normalization style suggestion. The best product path is to land the targeted SQLite-unavailable warning/action plus regression test in memory-core, while keeping file-based fallback search tracked separately in #59451/#62328.
