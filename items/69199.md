## Summary

Keep #69199 open. Current main still routes memory manager/search failures through buildMemorySearchUnavailableResult, which only distinguishes quota errors from generic embedding/provider errors. The targeted node:sqlite-unavailable branch and regression test proposed by the PR are not present on main, and the related fallback issue #59451 covers a different longer-term capability preservation path rather than this immediate UX fix.

## Best Possible Solution

Keep the PR open for maintainer review or land an equivalent small main-branch change in memory-core: add sqlite-missing detection in buildMemorySearchUnavailableResult, keep quota/generic behavior intact, and add the focused unavailable-payload regression test. Leave the broader fallback search work on #59451.
