## Summary

Keep open. Current main still does not honor `OPENCLAW_WRAPPER` or any `--wrapper` equivalent when generating daemon `ProgramArguments`. The PR tracks a real gap from #69400, but the supplied review discussion also shows it likely needs revision before merge: wrapper resolution should use the merged install/service environment, and executable validation would avoid writing a non-runnable wrapper into launchd/systemd config.

## Best Possible Solution

Keep the PR open and ask for a small revision: make wrapper resolution consume the merged install/service environment used by `buildGatewayInstallPlan` and the node install plan, preserve or intentionally manage `OPENCLAW_WRAPPER` in the generated service environment, validate that the wrapper is executable, and add regression tests for reinstall/update persistence plus invalid wrapper fallback. If maintainers want the broader #69400 shape, add a documented `--wrapper` install option separately rather than closing this focused PR.
