## Summary

Keep open. Current `main` does not implement the PR’s intended Google Vertex ADC fix, and the PR’s `<authenticated>` sentinel logic is stale against current `main`, which now uses `gcp-vertex-credentials` for ADC. The underlying bug path still appears plausible on `main`: ADC is resolved as a non-secret marker, classified as `api-key`, and then forwarded into the Google transport as `x-goog-api-key` instead of OAuth/Bearer auth.
