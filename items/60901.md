## Summary

The item proposes an external ACP↔Claude Code adapter and explicitly says OpenClaw core changes are unnecessary. Current main already has the relevant acpx command-override seams, project scope pushes this class of integration to external plugins/packages, and the issue discussion already points to a working external bridge (`proxy-acpx-x`).

## Evidence

## Risks / Open Questions

- `proxy-acpx-x` is external and not maintainer-reviewed here; permission-handling or auth edge cases may still exist in that package, but that does not justify a core OpenClaw tracking issue.
- The linked bridge is an npm package rather than a published ClawHub plugin today, so discoverability may still be weaker than the project-preferred distribution path.
