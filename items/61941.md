## Summary

Current main still separates plugin services from typed lifecycle hooks: Gateway startup does start registered services, but conversation hooks are registered through `api.on(...)` and non-bundled plugins must explicitly opt into `llm_input`/`llm_output`/`agent_end` access. I did not find evidence on `main` that services themselves receive LLM hook dispatch, so there is not strong support for closing this as fixed or obsolete.
