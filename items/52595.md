## Summary

Current `main` already treats missing session usage as unknown instead of zero for `/status`, and the same implementation is present in release `v2026.4.22`. The session usage writer marks missing context totals as stale/unknown, the status formatter renders unknown usage as `unknown/<ctx> (?%)` rather than `0/<ctx>`, and regression tests cover the missing-usage case.

## Evidence

## Close Comment

Closing as implemented. Codex review of current `main` found that missing session usage is already preserved as unknown rather than normalized into a misleading zero: `src/auto-reply/reply/session-usage.ts:157` marks missing context totals as stale/unknown, `src/commands/status.format.ts:27` renders unknown usage as `unknown/<ctx> (?%)` instead of `0/<ctx>`, and `src/commands/status.test.ts:1012` has regression coverage asserting `totalTokens`, `percentUsed`, and `remainingTokens` stay null when usage is missing.
