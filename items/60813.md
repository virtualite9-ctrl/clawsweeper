## Summary

Current `main` still has the LINE monitor's hardcoded generic dispatch-failure reply, and the requested typed failover routing is not wired into the bundled LINE plugin.

## Evidence

- **Hardcoded LINE fallback still present:** The LINE webhook monitor still catches auto-reply failures and sends the literal generic text `Sorry, I encountered an error processing your message.` instead of routing through typed failover classification.
  - file: [extensions/line/src/monitor.ts:285](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/extensions/line/src/monitor.ts#L285)
  - command: `nl -ba extensions/line/src/monitor.ts | sed -n '285,294p'`
  - sha: [6bc0dc8fb671](https://github.com/openclaw/openclaw/commit/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440)
- **LINE monitor does not import failover classification:** At the top of the LINE monitor, there is no import for `resolveFailoverReasonFromError`, `EmptyResponseError`, or any assistant error-formatting helper; the file only wires generic reply/runtime helpers before falling back to the hardcoded catch block.
  - file: [extensions/line/src/monitor.ts:1](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/extensions/line/src/monitor.ts#L1)
  - command: `nl -ba extensions/line/src/monitor.ts | sed -n '1,40p'`
  - sha: [6bc0dc8fb671](https://github.com/openclaw/openclaw/commit/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440)
- **Telegram still uses its own fallback path:** Current Telegram dispatch code still has separate fallback strings in its delivery path, so there is no shared typed fallback helper already adopted by LINE on `main` that would make this issue obsolete.
  - file: [extensions/telegram/src/bot-message-dispatch.ts:1145](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/extensions/telegram/src/bot-message-dispatch.ts#L1145)
  - command: `nl -ba extensions/telegram/src/bot-message-dispatch.ts | sed -n '1145,1156p'`
  - sha: [6bc0dc8fb671](https://github.com/openclaw/openclaw/commit/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440)
- **LINE remains supported product surface:** The docs still describe LINE as a bundled plugin in current OpenClaw releases, so this is not dead surface that can be dismissed as obsolete.
  - file: [docs/channels/line.md:14](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/docs/channels/line.md#L14)
