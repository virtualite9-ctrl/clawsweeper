## Summary

Keep open. Current main still builds the gateway health channel snapshot without the channel manager runtime state, while `channels.status` uses that runtime snapshot. The provided GitHub context also shows open PR #42586 explicitly closes this issue and remains unmerged, so the issue should not be closed independently before that implementation path is resolved.

## Best Possible Solution

Keep this issue open until #42586 is reviewed and either landed or closed. The likely fix belongs in the gateway health path: thread the channel manager runtime snapshot into `refreshGatewayHealthSnapshot()` / `getHealthSnapshot()`, merge per-channel/per-account runtime fields before `buildChannelSummary`, and add a regression test proving health and `channels.status` agree on `running` and `connected`.
