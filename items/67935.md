## Summary

Current main still does not provide an upgrade-safe Control UI chat message max-width setting. The chat group width remains hard-coded in bundled UI CSS, the renderer emits fixed `chat-group-messages` containers with no style/layout option, the gateway config type and strict schema do not accept a chat width or custom CSS field, and the Control UI bootstrap contract/state only carries existing identity/media/embed fields. This is a real bundled Control UI ergonomics request, not a ClawHub/plugin item.

## Best Possible Solution

Keep this issue open. Implement a narrow bundled Control UI layout setting, likely `gateway.controlUi.chatMessageMaxWidth` or a browser-local Appearance control, validate allowed CSS length/expression values, pass the resolved value through the Control UI bootstrap or UI settings state, set a root/shell CSS custom property, update `ui/src/styles/chat/grouped.css` to use `var(..., min(900px, 68%))`, and cover schema/bootstrap/rendering/docs behavior with focused tests.
