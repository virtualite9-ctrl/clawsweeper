## Summary

Current main still does not provide an upgrade-safe Control UI chat message max-width setting. The width is hard-coded in Control UI CSS, the renderer has no layout/style hook, the typed and validated gateway config lacks a chat width/custom CSS field, the bootstrap contract does not carry layout data, and the public docs only describe existing Control UI hosting/security/embed settings plus WebChat history sizing.

## Best Possible Solution

Keep this issue open. Implement a narrow Control UI chat layout setting, likely `gateway.controlUi.chatMessageMaxWidth` or a browser-local Appearance setting, validate the allowed CSS length/expression, pass it through the bootstrap or UI settings state, set a root/shell CSS custom property, update `ui/src/styles/chat/grouped.css` to use `var(..., min(900px, 68%))`, and cover schema/bootstrap/CSS/docs behavior with focused tests.
