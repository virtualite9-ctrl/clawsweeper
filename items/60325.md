## Summary

Keep open. Current `main` still requires the canonical `path` field for the `read` tool at runtime, so this PR's diagnostic-only alias expansion does not match actual behavior on `main` and would only suppress warnings for payloads that still are not valid end-to-end. The PR review note about alias inconsistency is also credible: path handling is already split across multiple helpers, and this change updates only one of them.
