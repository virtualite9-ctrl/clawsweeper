## Summary

Current main no longer matches PR #60325's premise. The PR assumes `file` and `filePath` are valid read-tool aliases that later normalize successfully, but current OpenClaw read execution requires canonical `path` before the underlying read tool runs, and tests explicitly reject legacy alias parameters. Adding the PR as-is would suppress diagnostics for invalid read calls rather than fix a false-positive warning.

## Best Possible Solution

Close this stale PR. Keep current main's canonical `path` read-tool contract; if legacy alias compatibility is still desired, track it as a fresh behavior change that updates runtime validation and schemas as well as diagnostics.
