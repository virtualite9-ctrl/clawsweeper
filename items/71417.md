## Summary

Current main still resumes the agent main session for `openclaw agent --agent <id>` when `--session-id` is omitted or blank, and omitted delivery channel still normalizes to `last`; there is no fresh-session flag, default-to-new behavior, or resume banner.

## Best Possible Solution

Keep this issue open and implement an explicit fresh-session path for `openclaw agent`: either make `--agent` without `--session-id` start a fresh session by default, or add a documented `--new-session`/`--fresh` flag plus a clear stderr resume banner when an existing session is reused. Cover both Gateway and `--local` paths, update `docs/cli/agent.md` and `docs/tools/agent-send.md`, and add focused tests for `--agent` with omitted, blank, explicit, and fresh-session controls.
