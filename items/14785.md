## Summary

Keep #14785 open. Current main has partial mitigations and policy workarounds, but the central request is still unresolved: OpenClaw still builds and dispatches broad tool schemas for the effective tool set, the `message` schema still exposes cross-channel parameters on scoped sessions, and the system prompt still includes a human-readable tool list. The issue discussion also explicitly treats #30359 as partial and includes a maintainer Codex review keeping #14785 as the canonical parent tracker.

## Best Possible Solution

Keep #14785 open as the canonical tool-schema overhead tracker. The best maintainer path is incremental core work: remove or make optional duplicated tool-list text, avoid schema payloads when no effective tools remain, prune `message` parameters by active channel/capability without breaking cross-channel jobs, keep long cron/tool guidance out of always-sent schemas, and design a compatible lazy or slim schema mode for plugin and declarative tools. Link narrower items such as #14619 as follow-ups instead of treating them as superseding this broader issue.
