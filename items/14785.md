## Summary

Keep #14785 open. Current main still forwards full effective tool parameter schemas into embedded sessions, the message tool still exposes broad cross-channel parameter groups even when action scoping is partially channel-aware, and the cron tool still carries tutorial-style guidance in its always-sent description. The issue discussion also treats prior work as partial and includes fresh maintainer/commenter evidence that the remaining overhead is still measurable.

## Best Possible Solution

Keep #14785 open as the canonical parent performance tracker. The best path is incremental core work: avoid sending schema payloads when no effective tools remain, prune message parameters by active channel/capability while preserving cross-channel jobs, move the cron tutorial out of the always-sent tool description, and design a compatible lazy or slim schema mode for plugin and declarative tool growth.
