## Summary

Keep #14785 open. Current main still forwards full parameter schemas for the effective tool set, the shared message tool still carries broad cross-channel fields, and the cron tool still embeds a long tutorial-style description. Existing policy controls such as `tools.allow`/`tools.deny` are useful workarounds, but they do not implement the issue’s requested channel-aware pruning, lazy/stub schemas, or mega-tool splitting.

## Best Possible Solution

Keep #14785 open as the parent performance tracker. The best path is incremental measured work: skip schemas when no effective tools exist, prune message parameters by active channel/capability, move the cron tutorial out of the always-sent tool description, and design a compatible lazy or slim schema mode for large plugin/declarative-tool installations.
