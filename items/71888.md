## Summary

Current main and the latest release already provide the requested OpenAI/Codex Pro model route. The supported route is `openai-codex/gpt-5.5-pro` for Codex OAuth, with provider runtime resolution, model-catalog augmentation, model-picker coverage, and regression tests. This does not prove a separate ChatGPT UI Deep Research product surface exists in OpenClaw, but it does satisfy the issue's requested supported model ID/provider route and clear catalog naming path.

## Best Possible Solution

Keep the shipped `openai-codex/gpt-5.5-pro` provider route and catalog behavior. If OpenAI later exposes a distinct Deep Research API/workflow separate from this model route, track that as a new focused provider-catalog/docs issue; packaged research-agent skills should remain ClawHub/plugin work unless a missing core API is identified.
