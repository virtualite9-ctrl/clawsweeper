## Summary

PR #68849 is a focused allowlist-only duplicate, but the linked bug discussion and current code show that adding `claude` to the Windows `.cmd` allowlist is only the first part of the fix. Current `main` still lacks the change, and the complete remaining work is tracked by #68819, which covers the deeper Windows `.cmd` supervisor spawn behavior and includes tests. #68792 is also a duplicate of the allowlist/test slice.

## Best Possible Solution

Close #68849 as superseded and keep the remaining work on the canonical Windows Claude CLI supervisor-spawn fix, preferably #68819 because it addresses both the missing `claude` shim and the deeper `.cmd` direct-spawn failure with tests. If maintainers choose a smaller path, they should still add regression coverage and account for Node's `.cmd/.bat` Windows behavior rather than merging the allowlist-only patch as-is.
