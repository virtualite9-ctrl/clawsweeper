## Summary

Current main already tracks chat.send runs for active-session interruption and abandoned-run cleanup via `chatAbortControllers`; the PR's proposed `addChatRun` hook in `chat.send` would be redundant because that path already uses the client run id as the agent run id.

## Close Comment

Closing after Codex review. Current `main` already tracks `chat.send` runs in `context.chatAbortControllers` (`src/gateway/server-methods/chat.ts:2242`), session interruption checks that existing map directly (`src/gateway/server-methods/sessions.ts:355`), and abandoned-run cleanup sweeps the same state (`src/gateway/server-maintenance.ts:108`). `chat.send` also passes `clientRunId` through as the agent `runId` (`src/gateway/server-methods/chat.ts:2503`, `src/auto-reply/reply/agent-runner-execution.ts:616`), so adding `addChatRun(runId, { sessionKey, clientRunId })` here would only create a redundant self-map. The `chatRunState` registry is already used for translated run ids like node/voice ingress (`src/gateway/server-node-events.ts:349`). Because the reported gap does not match current `main`, this PR is being closed as `cannot_reproduce`.
