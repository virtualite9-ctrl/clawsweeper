## Summary

Keep open. Current main still has adjacent bootstrap diagnostics in `/context list` and stored prompt-report data, but it does not implement this PR's central behavior: `/status` has no bootstrap file health section, edit mismatch errors do not mention bootstrap truncation, and the injected truncation warning lacks edit-failure guidance. The supplied PR is also a draft that needs rebase and tests, not an obsolete or superseded change.

## Best Possible Solution

Keep this draft PR open as an implementation candidate. Rebase it onto current main, move the status rendering change to `src/status/status-message.ts`, derive health thresholds from `SessionSystemPromptReport` and current config limits, avoid hardcoded 20,000-character messaging, and add focused tests for the bootstrap warning text, edit mismatch diagnostics, and `/status` output. Maintainers should then decide whether `/status` should duplicate per-file `/context list` details or show a concise health warning that points users to `/context list`.
