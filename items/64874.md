## Summary

Keep open. Current main has adjacent bootstrap diagnostics in `/context` and `doctor`, but it still does not implement this PR's central behavior: `/status` has no per-file bootstrap health, edit mismatch errors do not mention bootstrap truncation, and the injected bootstrap warning lacks the proposed edit-failure guidance. The related feature issue remains open and the complementary warning PR was closed unmerged, so this draft PR is not obsolete or superseded.

## Best Possible Solution

Keep this draft PR open as an implementation candidate. Rebase it onto current main, move the `/status` rendering change to `src/status/status-message.ts`, derive health thresholds from `SessionSystemPromptReport` and current config limits, avoid hardcoded 20,000-character messaging, and add focused tests for bootstrap warning text, edit mismatch diagnostics, and `/status` rendering. Maintainers should then decide whether `/status` should duplicate per-file `/context list` details or show a concise health warning that points users to `/context list`.
