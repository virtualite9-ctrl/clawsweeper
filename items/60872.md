## Summary

Keep open. Current `main` still encodes the reported behavior: ACP background turns accumulate assistant text only as transient `progressSummary`, terminal delivery ignores that field for successful runs, and the existing regression test explicitly expects an external channel to receive only `Background task done: ACP background task ...` after assistant output was produced.

## Evidence

- **ACP background success drops normal reply text at completion:** `runTurn` appends non-thought `text_delta` chunks into `taskProgressSummary`, but on success `markBackgroundTaskTerminal` only receives `terminalSummary` from `resolveBackgroundTaskTerminalResult()`, which returns a summary only for blocked/permission-style patterns. Ordinary assistant replies therefore end with `terminalSummary: null`.
