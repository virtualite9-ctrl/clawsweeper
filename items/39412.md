## Summary

Keep open. Current `main` already supersedes the daemon-token-persistence half of this PR, but it does not implement the provider `*_FILE` env-var loading half, so the request is not fully implemented on `main`.

## Risks / Open Questions

- Closing as implemented would drop the still-missing provider `*_FILE` behavior from env auth resolution and live-key collection on current `main`.
- If this PR is revived, it needs a rebase onto the newer gateway install/token flow already on `main`, not a straight merge of the stale branch.
- The PR diff shown in review also introduced at least one unresolved correctness concern for `_API_KEYS_FILE` multiline values, because `normalizeSecretInput()` strips newlines before list parsing.
