## Summary

Keep open. Current main still sends WhatsApp ack reactions as immediate fire-and-forget receipt feedback, but it does not track the ack reaction for post-reply cleanup, does not call the shared removal helper in the WhatsApp reply path, and the WhatsApp ackReaction config schema still rejects removal lifecycle keys.

## Best Possible Solution

Keep the issue open and implement the bundled WhatsApp ack lifecycle. The likely fix is to make WhatsApp ack sending return a trackable success promise, honor a documented cleanup setting such as messages.removeAckAfterReply or a WhatsApp-specific equivalent, call sendReactionWhatsApp with an empty emoji after a successful reply, and update schema/types/docs plus WhatsApp monitor tests.
