## Summary

Current main already implements the Control UI browser-safe session-key boundary that PR #60668 proposed. The maintainer concern about remaining leaks in `app.ts` and `app-render.ts` has been addressed: both now import from `ui/src/ui/session-key.ts`, and a search of `ui/src/ui` finds no remaining imports from `src/routing/session-key` or `src/sessions/session-key-utils`. Release history records the user-visible browser-bundle fix in `2026.4.5`, and the latest `v2026.4.24` tag still contains the same local import shape.

## Best Possible Solution

Close PR #60668 as implemented on current main. Keep the shipped `ui/src/ui/session-key.ts` browser-local helper and local imports in Control UI; if maintainers want the PR's extra import-guard regression coverage, track that as a fresh narrow test-only follow-up rather than keeping this obsolete PR open.
