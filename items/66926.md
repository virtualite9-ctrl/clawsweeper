## Summary

Current main still routes Google Gemini through tagged reasoning and injects the same <think>/<final> prompt without a native-tool-call exception, while the Google transport already sends and parses native function-calling blocks. The reported prompt/tool-call interference is therefore not implemented or superseded on main.

## Best Possible Solution

Keep this open and fix the Google provider reasoning-output path. The likely maintainer direction is to make direct Google Gemini use native reasoning output when provider tools/function calling are active, or amend the tagged reasoning prompt so tool calls are explicitly outside the <think>/<final> text contract and must be emitted through Gemini function calling. Add focused regression coverage around Google resolveReasoningOutputMode/system-prompt composition with tools while preserving transport tests that prove functionDeclarations are sent and functionCall parts are parsed.
