## Summary

Current `main` already filters legacy iMessage `is_from_me` rows before inbound dispatch, with regression coverage and a shipped changelog entry for the self-chat/echo fix.

## Evidence

- **Inbound filter on `is_from_me`:** `resolveIMessageInboundDecision` checks `params.message.is_from_me` and, for normal conversations, returns `{ kind: "drop", reason: "from me" }` before any inbound routing continues.

  - sha: [926068b14fc6](https://github.com/openclaw/openclaw/commit/926068b14fc661e915a0247e9ecb5949fb95b4f7)
- **Dropped `from me` events never dispatch:** The monitor treats `from me` as a loop-related drop reason and returns immediately on `decision.kind === "drop"`, so those rows do not reach reply dispatch.
