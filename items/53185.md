## Summary

Keep this issue open. Current main has meaningful built-in web_fetch sanitization and trusted bundled tool-result middleware, but it still does not implement the requested content-sanitizer.py pipeline, config-driven script preprocessing, external/plugin interception path, web_search sanitizer decision point, threat_level metadata, or HIGH/CRITICAL block behavior.

## Best Possible Solution

Keep this issue open but narrow it. Treat web_fetch hidden-content stripping as substantially implemented, then decide whether OpenClaw should add a trusted pre-model tool-result preprocessing API for web_fetch/web_search, extend bundled-only middleware with a documented external trust policy, or split only the broader taint/risk-class work into #54508 while tracking the custom script/preprocessing API question here.

## Risks / Open Questions

- Closing as implemented would conflate the shipped web_fetch hidden-content cleanup with the still-missing web_search/custom preprocessing and structured threat decision path.
- A script-based pre-model preprocessing hook is security-sensitive because it would execute configured code and rewrite tool output before model ingestion; it needs explicit trust boundaries, failure semantics, and runtime coverage before exposure.
