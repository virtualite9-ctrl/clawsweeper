## Summary

Keep this issue open. Current main has already shipped substantial `web_fetch` hardening: HTML extraction strips hidden elements, non-content tags, and invisible Unicode before wrapping fetched content as untrusted. But the issue also asks for a broader `web_fetch`/`web_search` preprocessing pipeline with structured threat decisions and custom script-style interception; that is not fully implemented for external plugins or for `web_search` results. The cross-referenced #54508 is related broader security work, but it does not fully supersede the narrower preprocessing-hook request.

## Best Possible Solution

Keep the issue open, but narrow it. The shipped web_fetch sanitizer should be treated as done; the remaining maintainer decision is whether OpenClaw should add a trusted core/plugin API for model-visible tool-result preprocessing, add built-in web_search result sanitization/threat metadata, or explicitly fold the taint/risk-class parts into #54508 while closing only after the preprocessing-hook question is resolved.
