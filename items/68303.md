## Summary

Current main addresses the reported Dashboard UX failure: chat sends render an optimistic user message before the gateway RPC returns, busy follow-ups are kept in a visible queued list with controls, disconnected sockets mark the UI offline/reconnecting path instead of accepting normal input, and docs/changelog describe the shipped queued-message behavior. The remaining OpenShell unknown-channel warning is a lower-level transport condition, but the observable Control UI “messages vanish then flush later” problem is already handled well enough on main and shipped by v2026.4.24.

## Best Possible Solution

Keep the shipped Control UI optimistic-send and visible queued-follow-up behavior. If maintainers still want OpenShell-specific stale-channel detection or forced WebSocket reconnects, track that as a narrower backend transport-resiliency issue rather than keeping this Dashboard disappearance report open.
