## Summary

Keep #70055 open. Current main appears to have addressed the worst reported symptom by replacing the old info-level external credential sync with runtime-only overlays and debug-level messages, but it still does not provide the requested config/env opt-out: external CLI auth profile resolution is still called unconditionally from the auth overlay path.

## Best Possible Solution

Keep the issue open for a narrow auth configuration change: add a user-facing `auth.externalCliSync` policy or equivalent env var, wire it into `overlayExternalAuthProfiles` and `resolveExternalCliAuthProfiles`, document the default-on behavior, and add tests proving opted-out auth loads do not read or overlay Codex/Claude/MiniMax CLI credentials.
