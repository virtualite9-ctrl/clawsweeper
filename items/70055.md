## Summary

Keep #70055 open. Current main has partially addressed the old noisy-sync behavior by moving external CLI auth handling into runtime-only overlays with debug-level bootstrap logging, but it still does not provide the requested user-facing config/env/CLI opt-out. The auth config type and generated schema have no `externalCliSync`-style field, while the runtime overlay path still unconditionally asks the hardcoded external CLI providers for credentials when auth profiles are overlaid.

## Best Possible Solution

Add a default-on external CLI auth policy under `auth` or an equivalent documented environment variable, thread it into `overlayExternalAuthProfiles`/`resolveExternalCliAuthProfiles`, and cover full disable plus provider include/exclude behavior in config schema, auth overlay tests, and user docs. The implementation should preserve current runtime-only behavior when enabled and avoid reading external CLI credential stores when disabled.
