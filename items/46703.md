## Summary

Keep #46703 open. Current main still builds session-memory filenames from `Date#toISOString()`, so the date prefix is UTC, not the configured or host-local timezone. The latest release tag has the same behavior, and the directly related fixes are still open/unmerged in the provided GitHub context.

## Best Possible Solution

Keep #46703 open as the canonical bug until a fix lands. The best implementation path is to update `src/hooks/bundled/session-memory/handler.ts` to resolve `agents.defaults.userTimezone` with host fallback, format the filename date/time via `Intl.DateTimeFormat.formatToParts` or an existing shared helper, update the Markdown header away from hard-coded UTC, and add regression coverage for negative and positive UTC-offset boundary times. PR #46721 is the focused starting point if maintainers want to revive or rebase it.
