## Summary

Keep open. Current main still ships the bundled canvas skill without YAML frontmatter, and the active skill loader requires a parsed description before a skill is loaded. The provided GitHub context also shows open PR #54361 with closing syntax for this issue, so the issue should stay open until that implementation candidate is reviewed and merged or closed.

## Best Possible Solution

Review and land PR #54361, or apply the equivalent fix on main: add valid `name` and trigger-rich `description` YAML frontmatter to `skills/canvas/SKILL.md`, then expand bundled frontmatter coverage to include canvas or all bundled skills. After that merge reaches main, this issue can close as implemented.
