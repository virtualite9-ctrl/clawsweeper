## Summary

Current `main` has partial startup-scope mitigations, but the provider-runtime path implicated by #62051 still lacks the active-registry compatibility fast path. Worker runs still pass a per-run `workspaceDir` into runtime plugin loading, the plugin-registry cache key still includes workspace roots, and `resolvePluginProviders()` still falls straight into `resolveRuntimePluginRegistry(...)`, so the redundant reload path described in PR #65497 remains plausible on current `main`.

## Evidence

- **Worker runs still trigger runtime plugin loading with per-run workspaces:** `runEmbeddedPiAgent` resolves a run-specific workspace and immediately calls `ensureRuntimePluginsLoaded({ config, workspaceDir: resolvedWorkspace, ... })`. `ensureRuntimePluginsLoaded` then calls `resolveRuntimePluginRegistry(loadOptions)` with that workspace.
  - file: [src/agents/pi-embedded-runner/run.ts:309](https://github.com/openclaw/openclaw/blob/8b76392e3e7934b1cdd6d461148263fd56141ec8/src/agents/pi-embedded-runner/run.ts#L309)
  - command: `nl -ba src/agents/pi-embedded-runner/run.ts | sed -n '290,314p' ; nl -ba src/agents/runtime-plugins.ts | sed -n '1,30p'`
  - sha: [8b76392e3e79](https://github.com/openclaw/openclaw/commit/8b76392e3e7934b1cdd6d461148263fd56141ec8)
- **Registry compatibility still keys on workspace roots:** The plugin loader cache key incorporates `roots.workspace` via `resolvePluginCacheInputs({ workspaceDir, ... })`, so different worker/session workspaces still produce different registry cache keys and miss the active registry.
  - file: [src/plugins/loader.ts:797](https://github.com/openclaw/openclaw/blob/8b76392e3e7934b1cdd6d461148263fd56141ec8/src/plugins/loader.ts#L797)

- **Provider runtime path still lacks the compatibility-only fast path:** `resolvePluginProviders()` computes runtime load options and then directly calls `resolveRuntimePluginRegistry(loadState.loadOptions)`. On current `main` it does not first ask `resolveCompatibleRuntimePluginRegistry(...)` for a no-reload hit.
  - file: [src/plugins/providers.runtime.ts:282](https://github.com/openclaw/openclaw/blob/8b76392e3e7934b1cdd6d461148263fd56141ec8/src/plugins/providers.runtime.ts#L282)

That same optimization is absent from `providers.runtime.ts`.
