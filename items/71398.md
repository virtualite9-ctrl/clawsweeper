## Summary

Keep open. Current main at df542f75 still treats `secretref:/...` as an ordinary plaintext string on the QQ Bot `clientSecret` path, so the reported upgrade regression is not implemented, superseded, or safe to auto-close. The supported SecretRef contract is structured object form, while QQ Bot docs say `clientSecret` accepts SecretRef input without covering the reported legacy shorthand.

## Best Possible Solution

Keep #71398 open. The best fix is to align QQ Bot with the SecretRef contract: add QQ Bot secret target/runtime coverage for `channels.qqbot.clientSecret` and `channels.qqbot.accounts.*.clientSecret`, add regression tests for structured file/env SecretRefs and the reported `secretref:/...` legacy shorthand, and either migrate/support that shorthand or reject it with clear doctor/config validation and docs pointing to the structured SecretRef form.
