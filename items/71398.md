## Summary

Keep open. Current `main` still treats the reported `secretref:/...` QQ Bot `clientSecret` value as a plaintext string rather than resolving or rejecting it, so the reported upgrade regression is still actionable.

## Best Possible Solution

Keep the issue open. Maintainers should decide whether `secretref:/...` is a supported legacy file-provider shorthand that needs migration/runtime compatibility, or invalid syntax that should fail validation/doctor with a clear remediation. The likely fix belongs in shared SecretInput migration/validation plus QQ Bot regression coverage for both `channels.qqbot.clientSecret` and `channels.qqbot.accounts.*.clientSecret`, with docs updated to show the supported structured SecretRef form.
