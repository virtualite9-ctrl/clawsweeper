## Summary

Current `main` now covers both plausible causes raised in the report thread: bundled Discord runtime deps are repaired before import in packaged installs, and Discord startup no longer waits on the health monitor when gateway readiness stalls because the lifecycle forces a short readiness timeout and reconnect path.

## Evidence

- **Bundled plugin deps are now repaired before loading enabled bundled plugins:** The plugin loader now calls `ensureBundledPluginRuntimeDeps(...)` for enabled bundled plugins before import and logs installed specs, matching the comment thread’s missing-postinstall diagnosis.
  - file: [src/plugins/loader.ts:2204](https://github.com/openclaw/openclaw/blob/16c7de085c99884367d34eaa9e52806d62ebf871/src/plugins/loader.ts#L2204)

  - file: [src/channels/plugins/bundled.ts:198](https://github.com/openclaw/openclaw/blob/16c7de085c99884367d34eaa9e52806d62ebf871/src/channels/plugins/bundled.ts#L198)
