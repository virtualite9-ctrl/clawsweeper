## Summary

Current main already covers the corrected behavior from PR #71807: OpenRouter catalog IDs are still normalized symmetrically through provider model-id hooks, and provider hook provider resolution is cached before plugin discovery/loading, so repeated pricing catalog normalization does not repeatedly discover provider plugins.

## Best Possible Solution

Close this PR as already covered by current main. Keep the shipped symmetric OpenRouter catalog normalization and cached provider hook resolution; if pricing refresh still profiles hot after current main, open a new targeted issue/PR with a fresh profile and a regression test for the specific remaining call volume.
