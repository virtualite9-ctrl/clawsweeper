## Summary

Keep open. Current `main` at `5b8bd6371c666a5de43c82305e0315771e8e49d5` still applies config-driven `mode=efficient` to explicit `format=ai` snapshot requests and still only parses `e`-prefixed AI snapshot refs, so the PR addresses behavior that is not yet implemented on `main`. The prior PR discussion is directionally useful: the bot review describes the change as narrow and regression-tested, which matches the gaps still visible in the current code.

## Evidence

That matches the bug the PR is trying to fix.
