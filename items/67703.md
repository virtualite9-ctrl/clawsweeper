## Summary

Keep this issue open. Current main still allows `gateway.auth.mode="none"` only at the generic auth decision layer; the unauthenticated loopback probe still omits device identity, `method: "none"` does not set `sharedAuthOk`, and the WebSocket missing-device policy still closes non-Control-UI clients with `1008: device identity required`. The related #69066 RFC is relevant, but it is open design work rather than a shipped fix for this concrete probe/followup failure.

## Best Possible Solution

Keep this issue open. The best fix is a narrowly scoped internal-service identity path, or an interim loopback-only internal caller exception, for Gateway-owned probe/followup/backend RPC clients under `gateway.auth.mode="none"`. Add regression coverage for `probeGateway` and daemon/backend `callGateway` paths without token/password/device identity, while preserving fail-closed behavior for remote, browser-origin, node-role, and non-internal clients. Align the durable implementation with #69066.
