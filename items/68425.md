## Summary

Keep open. Current main still redacts session-history text and exposes only the existing `contentRedacted` flag, without a stable machine-readable `[OPENCLAW-REDACTED]` marker or `notice` field. The linked PR #68483 is an open, unmerged implementation candidate with closing syntax, so this issue should remain open until that PR is reviewed, landed, or explicitly closed.

## Best Possible Solution

Keep this issue open and review PR #68483, or an equivalent narrow patch, for landing. The fix should add a stable machine-readable redaction marker/notice on agent-visible redacted tool output, keep existing redaction semantics intact, and include regression tests for both the shared redaction helper and `sessions_history` behavior.
