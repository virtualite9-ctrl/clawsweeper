## Summary

Keep #72214 open. Current main has partial Telegram network hardening, but it does not fully address the reported current-release failure mode where Telegram/undici timeouts can leave the channel unhealthy and degrade gateway health/RPC. The issue is recent, detailed, unprotected by maintainer labels, and not covered by a clearly open canonical duplicate.

## Best Possible Solution

Keep the issue open and implement a focused Telegram lifecycle hardening fix: add bounded timeouts/cancellation to command-menu startup calls and any remaining Bot API setup paths, publish an explicit degraded channel status for Telegram transport failures, ensure gateway health/status RPC never waits on Telegram startup/shutdown, and add regression tests with a never-settling Telegram fetch/startAccount path plus a stop timeout path.
