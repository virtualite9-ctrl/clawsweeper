## Summary

The request is a useful optional workspace convention, but it is not a core OpenClaw docs/runtime gap. The issue and follow-up explicitly frame PENDING.md as a user-maintained, docs-only pattern with no API or runtime change, and current OpenClaw already has the needed extension surfaces: workspace skills, plugin-shipped skills, ClawHub distribution, and HEARTBEAT.md tasks that can read workspace files when instructed. The related docs PRs #69637 and #69658 were closed unmerged, and current main has no PENDING.md docs/template, so the best maintainer cleanup is to direct this to a ClawHub skill/template rather than keep a core docs issue open.

## Best Possible Solution

Move the PENDING.md convention into a ClawHub workspace skill or bundle-style template that includes the SKILL.md guidance, example PENDING.md content, and HEARTBEAT.md prompt snippet. Keep core docs focused on the existing workspace, heartbeat, skills, and ClawHub extension points unless maintainers explicitly decide this pattern should become first-party documentation.
