## Summary

Keep open. Current main still has the unsafe persisted override normalization path this PR is trying to fix, so this is not implemented or superseded. The PR should not merge as-is because it also bundles unrelated heartbeat behavior changes that conflict with current docs/code semantics for empty heartbeat blocks.

## Best Possible Solution

Keep the PR open but request changes: drop or split out the heartbeat commit, keep a focused defensive guard for persisted model override fields using the repo's existing string-normalization pattern, and add a regression test proving non-string providerOverride/modelOverride values from the session store do not throw while valid string overrides still normalize correctly. Treat #68300 as complementary logging work, not a replacement for this root-cause guard.
