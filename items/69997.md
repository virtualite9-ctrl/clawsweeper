## Summary

Keep #69997 open. The underlying model-override hardening is still relevant on current main, but the PR is not cleanup-closeable because it contains useful work plus unrelated heartbeat changes that reviewers correctly flagged as a behavior regression. The canonical bug #68124 also remains unsolved; related PR #68300 only improves hook error logging, not this normalization path.

## Best Possible Solution

Keep the PR open for maintainer review, but require changes before merge: split or drop the heartbeat edits, keep the defensive `normalizeStoredOverrideModel` guard, and add focused regression coverage for non-string `providerOverride`/`modelOverride` values loaded from session-store data. The logging/debugging side can proceed through #68300 or another focused PR, while #68124 remains the canonical bug until a concrete fix lands.
