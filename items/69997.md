## Summary

Keep open. Current main still has the unguarded `normalizeStoredOverrideModel` path and #68124 remains open, so this is not implemented or obsolete. The PR should not merge as-is because it also bundles unrelated heartbeat behavior changes for #64293, is currently merge-dirty, and conflicts with current per-agent heartbeat inheritance semantics documented and tested on main.

## Best Possible Solution

Keep the item open for maintainer review, but request changes before merge: split the model override guard from the heartbeat behavior, add regression tests for corrupted/non-string persisted `providerOverride` and `modelOverride` values on the session/live-model selection paths, and handle the heartbeat `{}` semantics in a separate #64293-focused PR with explicit docs/tests for defaults versus per-agent empty blocks. Coordinate with #68300 for stack logging rather than treating it as a replacement for the root fix.
