## Summary

Keep open. Current `main` still has the unsafe `?.trim()` path in `normalizeStoredOverrideModel`, and the session-store load path does not sanitize `providerOverride` / `modelOverride` before they reach it, so this is not `implemented_on_main` or `cannot_reproduce`. But PR #69997 is not merge-safe as submitted because it also adds unrelated heartbeat changes that conflict with documented and tested `{}` inheritance behavior.
