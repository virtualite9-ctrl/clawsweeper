## Summary

Keep #12601 open. Current `main` still does not implement Slack Canvas content reading: the bundled Slack plugin exposes no Canvas/read-canvas action, does not dispatch any Slack `canvases.*` API call, setup/docs still omit `canvases:read`, and the related implementation PR #18752 was closed unmerged rather than shipped.

## Best Possible Solution

Keep this open for the bundled Slack plugin. A clean implementation should live primarily under `extensions/slack`: add a Canvas read action or dedicated tool, add `canvases:read` to setup/docs/config, call the appropriate Slack Canvas read API through the existing token/account model, return bounded markdown/section data, update the shared message-action contract if the generic message tool surface is used, and add discovery, dispatch, unit/contract tests, docs, and live Slack validation.
