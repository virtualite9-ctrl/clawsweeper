## Summary

Keep this PR open. Current main still lacks the PR's central Feishu behavior: carrying an update-card payload in card action envelopes, parsing the clicked card's message id, and patching the original card on confirm/cancel/quick-action clicks. The contributor PR remains a plausible implementation candidate, though it likely needs narrowing because the provided diff also includes unrelated Feishu reply-dispatcher and monitor.comment.test churn.

## Best Possible Solution

Keep this PR open as a candidate implementation. The best maintainer path is to review or ask for a focused rebase that only wires Feishu card update payloads through card-interaction/card-ux-shared, parses card-action message_id in monitor.account, calls updateCardFeishu from card-action with failure isolation, and adds targeted tests for confirm, cancel, quick actions, malformed uc, and missing message_id behavior. If maintainers prefer not to use this branch, close it only after opening or landing a narrower replacement for the same Feishu card-update behavior.
