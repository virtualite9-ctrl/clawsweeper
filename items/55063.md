## Summary

Close as duplicate/superseded by open #51251. Current main has already split reset handling into auto-sourced fallback pins versus explicit user selections, but the remaining request here, clearing explicit session model overrides on /new or /reset and falling back to the configured default, is already tracked by #51251.

## Best Possible Solution

Use #51251 as the canonical maintainer tracker for the remaining session model override/default reconciliation work. Any eventual fix should decide the product semantics for explicit /model selections across /new, /reset, and gateway restart, then align reset/status copy and tests with that decision.
