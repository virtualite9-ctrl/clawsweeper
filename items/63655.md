## Summary

Keep #63655 open. Current main still routes memory-lancedb embeddings through the OpenAI SDK without an explicit compatibility fix, while the pinned SDK version defaults omitted embedding `encoding_format` to `base64` and decodes the response. The existing memory-lancedb tests also assert the old request shape, and related reports/PRs in the provided context show the same root issue remains unresolved rather than shipped.

## Best Possible Solution

Fix `extensions/memory-lancedb` itself. The safest path is to bypass or wrap the OpenAI SDK embedding decode behavior so OpenAI-compatible providers that return `number[]` are preserved, without sending provider-rejected `encoding_format` fields unconditionally. Add focused tests that simulate a non-OpenAI baseUrl returning float arrays and confirm query/store vectors keep the configured dimensions.
