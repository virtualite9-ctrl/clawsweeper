## Summary

Keep this issue open. Current main only implements the narrower Ollama/base64 fix by always sending `encoding_format: "float"`; the reporter's follow-up says ZhiPu rejects that parameter entirely, and memory-lancedb still has no config or request path that omits it.

## Best Possible Solution

Keep this issue open and implement an owner-local memory-lancedb compatibility path: add an explicit embedding encoding/provider mode, provide an omit mode that posts without `encoding_format`, normalize returned embeddings by accepting float arrays and decoding only string/base64 responses, and add regression tests for both forced-float providers and reject-unknown-parameter providers.
