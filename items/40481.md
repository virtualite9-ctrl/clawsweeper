## Summary

Current main still leaves the generic exec-approval forwarding fallback with the placeholder `/approve <id>` instruction, so this PR is not obsolete. It remains useful, but should be rebased and reworked before merge because current main now has allowed-decision filtering, shared action descriptors, and native channel renderers that the PR must preserve.

## Best Possible Solution

Keep this PR open and rework it against current main. The implementation should update the generic fallback in `src/infra/exec-approval-forwarder.ts` to render concrete per-decision `/approve <actual-id> <decision>` lines, preferably using `buildExecApprovalActionDescriptors` or equivalent shared helpers, while preserving allowed-decision filtering, markdown-capability handling, native Telegram/shared interactive rendering, and non-markdown channel behavior. Add focused tests in `src/infra/exec-approval-forwarder.test.ts` for markdown-capable fallback, non-markdown fallback, and `ask=always` without `allow-always`.
