## Summary

Keep this PR open. Current main still emits the old placeholder `/approve <id>` syntax in the generic exec-approval forwarding fallback, while only native Telegram/shared interactive paths already provide full approval commands. The PR remains useful, but should be rebased onto the current approval-renderer code and preserve allowed-decision filtering.

## Best Possible Solution

Keep the PR open and rework it against current main so `src/infra/exec-approval-forwarder.ts` renders concrete per-decision `/approve <actual-id> <decision>` lines in the generic plain-text fallback, ideally via existing approval action helpers. Preserve native Telegram rendering, shared interactive buttons, non-markdown channel behavior, and current allowed-decision filtering for `ask=always`; update the forwarder tests for markdown-capable, non-markdown, and allow-always-unavailable cases.
