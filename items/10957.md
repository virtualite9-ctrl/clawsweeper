## Summary

Keep #10957 open. Current main supports the Android companion-node topology, pairing, command/capability registry, Play vs third-party flavor split, and release bundle mechanics, but it still explicitly says the Android app is not publicly released and does not document beta/tester access for an existing-Gateway Android phone node. The prior issue comment also kept it open for this same remaining access/documentation gap.

## Best Possible Solution

Keep #10957 open until maintainers either publish and document an Android beta/public distribution path for existing-Gateway Android phone nodes, or explicitly route Android beta-access requests to a support/auto-response process with clear public instructions. Continue Android capability, flavor split, signed bundle, and release-hardening work in the Android app/release surfaces separately.
