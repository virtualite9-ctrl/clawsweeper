## Summary

Keep open. Current `main` persists only machine-oriented inter-session provenance (`sourceSessionKey`, `sourceChannel`, `sourceTool`) and the chat/session UI renders user attribution only from `senderLabel`/inbound envelope metadata, so the requested human-readable sender attribution for `sessions_send` traffic is still an unresolved feature gap.
