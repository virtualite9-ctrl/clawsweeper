## Summary

Keep #69374 open. Current main still extracts WhatsApp `contextInfo` for mentions and quoted-reply metadata, but does not read Baileys `forwardingScore` / `isForwarded`, does not store them on the WhatsApp inbound message shape, and does not map them into `inbound_claim` or `message_received` plugin metadata. This is a concrete bundled WhatsApp/plugin hook API gap, not a ClawHub-only request.

## Best Possible Solution

Keep the issue open and implement an additive WhatsApp/core hook metadata pass-through: read forwarding fields from Baileys `contextInfo`, store optional `isForwarded` / `forwardingScore` on the WhatsApp inbound message/context path, expose them through `CanonicalInboundMessageHookContext`, and map them into `inbound_claim` and `message_received` metadata with focused WhatsApp extraction and hook-mapper tests plus a small docs note.
