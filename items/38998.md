## Summary

Keep open. Current main still lacks a typed `pairing:request` or equivalent hook/event for DM pairing requests. The pairing path creates pending requests and replies to the unknown sender, but only exposes a local `onCreated` callback used by channel implementations for logging. The request is coherent, recent, not protected by maintainer ownership, and not safely closable as ClawHub-only because a plugin would need a missing core/plugin event surface.

## Evidence

A targeted search for `pairing:request`, `pairing_request`, and `PluginHookPairing` found no matches in current main.
