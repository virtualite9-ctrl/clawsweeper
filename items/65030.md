## Summary

Keep this PR open. Current main includes the merged known Bedrock context-window lookup from #65952, but it still lacks this PR's fallback provenance path: unknown/defaulted Bedrock models get a materialized contextWindow, and the runtime guard still classifies any positive model-supplied window as source "model".

## Best Possible Solution

Keep this PR open as a still-relevant implementation candidate. The best path is a narrowed rebase onto current main that preserves #65952's known-window lookup, adds provenance only for unknown/defaulted Bedrock fallback values, updates ModelDefinitionConfig and ProviderRuntimeModel if a field is kept, threads the flag through every resolveContextWindowInfo call site, includes explicit defaultContextWindow provenance in the discovery cache key, and adds focused tests for unknown foundation models, unknown inference profiles, explicit defaultContextWindow, and runtime guard source classification.
