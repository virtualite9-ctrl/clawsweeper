## Summary

Keep this PR open. Current main has the merged Bedrock known-context-window lookup from #65952, but it still does not implement this PR's provenance behavior for literal/defaulted Bedrock context windows. Unknown Bedrock models and unknown inference profiles still receive a materialized fallback contextWindow, and the runtime guard still treats any positive model-supplied window as source "model" because no provenance field is threaded through the model or runtime types.

## Best Possible Solution

Keep this PR open as a still-relevant implementation candidate, but require a rebase onto current main and a narrower patch: preserve the known-context-window lookup, add fallback provenance only for unknown/defaulted Bedrock values, update ModelDefinitionConfig and ProviderRuntimeModel if a field is kept, thread it through every resolveContextWindowInfo call site, include explicit defaultContextWindow provenance in the discovery cache key, and add focused tests for unknown foundation models, unknown inference profiles, explicit defaultContextWindow, and runtime guard classification.
