## Summary

Close #46466 as not_actionable_in_repo. The reported launchctl domain is `gui/0`, which current code derives from UID 0, so the failure points to running the per-user LaunchAgent install as root/sudo or outside the target user's logged-in GUI session. OpenClaw supports macOS Gateway supervision as a per-user LaunchAgent, already emits the exact GUI-session/wrong-user guidance for `Bootstrap failed: 125`, and has regression coverage for that diagnostic path. Without a reproduction from the target non-root desktop user, the remaining action is local macOS session/setup rather than a source change in this repository.

## Best Possible Solution

Keep OpenClaw's supported macOS Gateway install as a per-user LaunchAgent with the existing explicit diagnostic. Treat root/sudo/headless macOS service setup as local environment administration unless maintainers explicitly decide to design and ship a LaunchDaemon flow; ask for a fresh non-root GUI-session reproduction if this still fails in the supported path.
