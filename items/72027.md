## Summary

Keep this PR open. Current main has not implemented the requested timeout reduction: the Gateway model-pricing cache still uses a 60s fetch timeout for both OpenRouter and LiteLLM, and the regression test still expects 60s timeout-warning text. The PR is narrow and still relevant, but it would reverse a changelog-documented 60s behavior, so the right outcome is maintainer review rather than conservative cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review. A Gateway maintainer should decide whether optional pricing enrichment should retain the documented 60s slow-upstream tolerance or adopt the PR's 10s fail-fast behavior; if fail-fast is preferred, merge or adapt the two-file timeout/test change and handle any broader startup decoupling or cancellation work separately.
