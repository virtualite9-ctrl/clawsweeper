## Summary

Current `main` still does not expose the WhatsApp web inbound-message watchdog timeout as user config. The runtime keeps an internal `messageTimeoutMs`/`watchdogCheckMs` tuning path, but config types, config resolution, and user docs only expose `web.enabled`, `web.heartbeatSeconds`, and `web.reconnect.*`, so issue #63925 remains a valid open feature request despite the "Fixed, closing." comment.
