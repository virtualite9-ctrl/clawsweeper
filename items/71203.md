## Summary

Keep PR #71203 open. Current main still only refreshes models.json through the primary-model startup warmup, that warmup returns early without agents.defaults.model.primary, and when it runs it targets only resolveOpenClawAgentDir(). The PR remains the active focused implementation path for the stale non-default agent cache bug from #71192, though it still needs review for the synthetic main/override-dir concern raised in review comments.

## Best Possible Solution

Keep the PR open and finish it. The best path is to land a startup helper that refreshes models.json for the active default agent dir plus explicitly configured agent dirs, with regression coverage for configured non-default agents and the no-default-primary case. Before merge, adjust or explicitly justify the main fallback behavior so startup refresh does not unintentionally write outside the selected override agent dir.
