## Summary

Keep open. Current main still only refreshes `models.json` through the default-agent primary-model warmup path, and that path returns before cache generation when no explicit default primary model is configured. The PR is a recent, focused implementation candidate for the remaining non-default agent startup-refresh gap described by #71192.

## Best Possible Solution

Keep this PR open for review. The best code path is a startup helper that refreshes the unique configured agent dirs independently of primary-model prewarm, with regression coverage for non-default agents and the no-default-primary case, while preserving default/override agent-dir semantics.
