## Summary

Keep open. Current main does not implement the PR's startup refresh for configured non-default agent models.json caches: gateway startup only prewarms the default agent dir for an explicit PI primary model, and there is no current startup helper/test that iterates configured agent dirs. The PR is recent and actively followed up by the contributor, and the linked bug report has a concrete stale-cache reproduction.

## Best Possible Solution

Keep #71203 open for maintainer review. The likely best fix is a narrow gateway startup helper that refreshes `models.json` for the active default dir plus configured agent dirs via `listAgentIds`/`resolveAgentDir`, with regression coverage for non-default agents, no-default-primary config, and the synthetic `main`/agent-dir override edge called out in review comments.
