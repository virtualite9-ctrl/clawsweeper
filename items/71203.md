## Summary

Keep open. Current main still only refreshes `models.json` for the default agent dir during the configured-primary startup warmup, and that path is skipped entirely when no explicit primary model is configured. The PR is a focused, active fix for the linked stale non-default agent cache report and is not implemented on main.

## Best Possible Solution

Keep the PR open for maintainer review. The likely path is to re-review the latest split helper, ensure it avoids unintended default-path writes noted in bot review, rebase on current main if needed, and validate with the gateway changed lane before merge.
