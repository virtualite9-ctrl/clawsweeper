## Summary

Keep open. Current `main` still conflates delivery-target failures with execution errors for normal cron delivery paths: the cron status model only allows `ok | error | skipped`, unresolved announce targets return `status: "error"` unless `bestEffort` is enabled, and Control UI only exposes `OK/Error/Skipped` run states. The repo does persist separate delivery metadata, but the requested warning-level execution/delivery split is not implemented on `f3c37a946cc49d26e5fe65a1043f7286a86aa4a4`.
