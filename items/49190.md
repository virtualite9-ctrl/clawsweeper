## Summary

Keep open. Current main still has only ok/error/skipped cron run statuses, non-best-effort announce/delivery-target failures still return status=error, and Control UI has no warning run state. Existing deliveryStatus metadata and the best-effort delivery toggle are partial workarounds, but they do not satisfy the requested execution-success/delivery-failure warning split.

## Best Possible Solution

Keep the issue open. The likely product/code path is to add a first-class cron warning state or explicit executionResult/deliveryResult split across src/cron/types.ts, gateway cron schemas, run-log filters, service state application, isolated delivery dispatch/finalize logic, Control UI types/rendering/filters, and docs/tests. Delivery-only failures should preserve the successful execution summary, record deliveryStatus/deliveryError, avoid consecutiveErrors/backoff/failure-alert increments, and render as yellow warning in Control UI.
