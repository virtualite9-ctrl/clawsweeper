## Summary

Current main already implements per-model thinking-level fallback: unsupported stored/default levels are remapped to the nearest supported level, explicit unsupported directives are rejected with valid options, the gateway exposes model-specific thinking options/defaults, and regression tests cover `xhigh -> high` behavior.

## Evidence

- **Core fallback helper:** `resolveSupportedThinkingLevel()` ranks provider/model-supported thinking levels and falls back to the nearest supported non-`off` level, so an unsupported `xhigh` resolves down to `high` when that is the largest supported level.

  - file: [src/auto-reply/thinking.test.ts:172](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/src/auto-reply/thinking.test.ts#L172)
