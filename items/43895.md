## Summary

Keep this PR open. Current main still builds the status session-default summary from one global configured model, and neither the CLI overview, JSON status type, nor TUI formatter has a `variesByAgent`/`varies by agent` state even though per-agent `agents.list[].model` overrides are a supported config surface.

## Best Possible Solution

Keep this PR open or replace it with a narrow maintainer patch. The best fix is to compute `sessions.defaults` from each listed agent's effective default model using the current status-safe resolver path, keep a concrete model only when all agent defaults match, expose an optional `variesByAgent` state through `StatusSummary` and `GatewayStatusSummary`, render `varies by agent` in CLI and TUI status surfaces, and add divergent/uniform per-agent regression tests.
