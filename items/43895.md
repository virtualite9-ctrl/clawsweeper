## Summary

Keep this PR open. Current main still reports one aggregate default model for status output even though per-agent model overrides are supported, so the user-visible bug described by the PR remains. The PR is plausible and narrow, but it needs rebase/adaptation to current status internals before landing.

## Best Possible Solution

Keep this PR open or replace it with a narrow maintainer patch. Compute `sessions.defaults` from each configured agent's effective status-safe default model, keep a concrete model only when all agent defaults match, expose an optional variation marker through `StatusSummary` and `GatewayStatusSummary`, render `varies by agent` in CLI and TUI status surfaces, and add divergent/uniform per-agent regression tests while preserving the read-only status path.
