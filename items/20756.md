## Summary

Keep open. Current main still resolves message-tool accountId only from explicit params, tool defaultAccountId, or agent binding, then WhatsApp falls back to its default-account helper. It does not auto-select the sole enabled account when the configured default account is disabled. Related PR #20801 attempted this fix but was closed unmerged.

## Best Possible Solution

Keep #20756 open. Implement a shared outbound account-resolution step in the message action path: preserve explicit accountId, prefer an enabled configured default account, auto-select exactly one enabled non-default account when no usable default is available, and return clear errors for zero or multiple enabled accounts. Add focused regression tests for runMessageAction and the WhatsApp disabled-default/single-enabled-account case, then update public channel docs.
