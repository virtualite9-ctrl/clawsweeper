## Summary

Keep #20756 open. Current main still does not auto-select the only enabled account for `message` tool sends when `accountId` is omitted. The shared message path only uses an explicit account, tool `defaultAccountId`, or agent binding fallback; WhatsApp then falls back to the default-account helper, which ignores enabled state. Related PR #20801 attempted the requested behavior but was closed unmerged.

## Best Possible Solution

Keep #20756 open. Implement shared outbound account resolution after channel selection in the message action path: preserve explicit account IDs and existing fallbacks, prefer a usable configured default, auto-select exactly one enabled account when no usable default exists, and return clear errors for zero or multiple enabled accounts. Add focused regression tests for `runMessageAction` and the WhatsApp disabled-default/single-enabled-account case, then update the public channel docs.
