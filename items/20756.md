## Summary

Current main still does not implement account-level auto-selection for the message tool when only one account is enabled. The related PR #20801 was closed unmerged in the provided context, and current code still falls back through explicit accountId, tool/default account, agent binding, then channel/plugin default behavior rather than filtering enabled accounts for the selected channel.

## Best Possible Solution

Keep #20756 open until core outbound account resolution is implemented. The likely fix belongs in the message action account-resolution path using the selected channel plugin's listAccountIds, resolveAccount, and isEnabled hooks: preserve explicit accountId, prefer an enabled default account, auto-select exactly one enabled non-default account when default is disabled or absent, and throw clear errors for zero or multiple enabled accounts. Add focused regression coverage around runMessageAction and the WhatsApp disabled-default/one-enabled-account case, then update the channel docs.
