## Summary

Current `main` already handles installed/runtime bundled plugin discovery through the shared resolver, which prefers built plugin trees (`dist-runtime/extensions` or `dist/extensions`) outside source/Vitest/tsx runs. This PR's Dockerfile-only override is superseded and would now violate the maintained Dockerfile contract on `main`.

## Evidence

- **Docker runtime no longer hardcodes bundled plugin discovery:** Current runtime image copies both `dist/` and `extensions/`, but does not set `OPENCLAW_BUNDLED_PLUGINS_DIR` in the Dockerfile runtime section.
