## Summary

Current main already fixes the reported plugin-tool exposure problem. The reporter was on 2026.3.13; the changelog records a later plugin-runtime fix that keeps plugin tools from disappearing due to mismatched runtime loads, and current source/test coverage shows gateway/channel tool construction now reuses the compatible gateway-bindable plugin registry.

## Best Possible Solution

Close this issue as fixed on current main. Keep the shipped gateway-bindable plugin registry/tool-loading behavior; if a user still sees this on a current release, ask for a new issue with the plugin entry source, manifest, config, tool policy, Feishu channel details, and logs. If the setup only contains SKILL.md, the correct path is to add a plugin entry that calls api.registerTool(...), not a core code change.
