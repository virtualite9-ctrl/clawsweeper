## Summary

Current main still does not match the documented Telegram callback contract for this issue. The callback_query handler consumes and routes Telegram callbacks, but the generic fallback synthetic message still uses the raw callback payload rather than the documented `callback_data: <value>` text. An open, unmerged PR #54962 explicitly references this issue with closing syntax and proposes the one-line fix, so this issue should stay open until that PR is reviewed/landed or superseded by an equivalent fix.

## Best Possible Solution

Keep this issue open and review/land PR #54962 or an equivalent patch in `extensions/telegram/src/bot-handlers.runtime.ts` that forwards generic callback fallback text as `callback_data: ${data}` while preserving native command callbacks. Add/update a regression test to assert the documented prefix, then close this issue after the fix merges.
