## Summary

Current main still routes generic Telegram callback_query payloads to the agent as the raw callback data, while the public Telegram docs promise the `callback_data: <value>` text shape. The linked PR #54962 remains open and unmerged, so this issue should stay open until that implementation or an equivalent fix lands.

## Best Possible Solution

Keep this issue open. Review and land PR #54962, or apply an equivalent change in `extensions/telegram/src/bot-handlers.runtime.ts` that forwards generic Telegram callback fallback text as `callback_data: ${data}` while preserving native `tgcmd:` callbacks and the existing approval, plugin, pagination, and model callback paths. Add a regression test for the documented prefix, then close after the fix merges.
