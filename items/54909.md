## Summary

Current main still does not implement the documented Telegram generic inline-button callback text contract. The callback_query handler exists and routes fallback callbacks into the agent, but generic callback data is forwarded as the raw payload instead of `callback_data: <value>`. The linked open PR #54962 remains the implementation candidate, so this issue should stay open until that fix or an equivalent patch lands.

## Best Possible Solution

Keep this issue open and review or land PR #54962, or apply an equivalent patch in `extensions/telegram/src/bot-handlers.runtime.ts` that forwards generic Telegram callback fallback text as `callback_data: ${data}` while preserving native command callbacks. Add or update a regression test that asserts the documented prefix for generic callbacks, then close this issue after the fix merges.
