## Summary

Keep this issue open. Current main still has the reported Raw-mode failure path: safe-bin normalization writes undefined keys, raw round-trip validation compares restored raw against the materialized runtime config, and the Control UI disables Raw mode when redaction returns no raw text. PR #59336 remains open and explicitly targets this issue, so it is the active implementation candidate rather than a reason to close early.

## Best Possible Solution

Keep this issue open and use PR #59336 as the active implementation candidate. A durable fix should delete empty safe-bin keys instead of assigning undefined, compare raw redaction restoration against the authored/source snapshot or otherwise normalize materialization-only diffs safely, add regression coverage for both failure modes, and address or split out the global substring redaction corruption reported in the comments.
