## Summary

Keep this issue open. Current main still has the reported Raw-mode failure path: safe-bin normalization writes undefined-valued keys, raw round-trip validation compares restored raw against the materialized runtime config, and Control UI disables Raw mode when redaction returns no raw text. The linked #59336 PR remains open and explicitly targets this issue, so it is an implementation candidate rather than a reason to close early.

## Best Possible Solution

Keep this issue open and use #59336, or a maintainer-owned replacement PR, as the active implementation path. A durable fix should delete empty safe-bin keys instead of assigning undefined, compare raw restoration against the authored/source snapshot or otherwise normalize materialization-only diffs safely, and add regression coverage for the safe-bin, materialized-config, and substring-redaction failure modes.
