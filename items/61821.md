## Summary

Current `main` already covers this report. The iMessage inbound gate no longer treats `sender === chat_identifier` as self-chat by itself; it now requires `destination_caller_id` to match the sender, so the reported DM false-positive path drops as `from me` instead of falling through to inbound dispatch. The parser also strips wrapped iMessage text payloads before downstream matching, and the shipped regression tests include the exact corrupted-text plus mismatched-`destination_caller_id` shape described here. The fix is present in release `v2026.4.23`.

## Evidence

- **Self-chat classification now requires explicit destination match:** `resolveIMessageInboundDecision` only sets `isSelfChat` when normalized `destination_caller_id` equals the normalized sender. If `is_from_me` is true and that condition is not met, the message is dropped as `from me` instead of entering the self-chat echo path.
