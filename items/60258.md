## Summary

Current main already reclaims stale session lock files when the recorded PID is dead or recycled, includes regression coverage for those cases, and ships a doctor repair path for leftover `.jsonl.lock` files. The latest release v2026.4.23 contains the same implementation.

## Evidence

- **Dead-PID stale lock reclaim is implemented:** `inspectLockPayload` marks a lock stale when `pidAlive` is false (`dead-pid`), and `acquireSessionWriteLock` removes that contended lock and retries acquisition instead of timing out indefinitely.
  - file: [src/agents/session-write-lock.ts:331](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/src/agents/session-write-lock.ts#L331)

- **Cross-platform PID liveness check exists:** `isPidAlive` uses `process.kill(pid, 0)` to detect whether the recorded lock holder is still alive; Linux additionally exposes `getProcessStartTime` for PID-reuse detection, while non-Linux still gets dead-PID detection.
