## Summary

Keep open. Current main still matches the reported memory-dreaming failure modes: light sleep still sorts by `lastRecalledAt` before `recallCount`, daily/session dreaming ingestion records `signalType: "daily"` while refreshing `lastRecalledAt`, deep promotion still gates `minRecallCount` on total signals rather than actual recall count, and rehydration still caps relocation windows at `preferredSpan + 3` or `8`. The issue is recent, author is not protected, labels are not protected, and no related item in the provided context supersedes this specific bug chain.

## Best Possible Solution

Keep the issue open and fix it in the bundled `memory-core` plugin. The likely product-safe path is to add regression tests that create competing daily-only and real recall entries, then update `extensions/memory-core/src/dreaming-phases.ts` so light sleep preserves real recall evidence ahead of daily recency noise. In `extensions/memory-core/src/short-term-promotion.ts`, separate actual recall count from daily/grounded reinforcement for the `minRecallCount` gate and scoring denominator, while preserving intentional grounded backfill behavior with a named contract if needed. Also widen `relocateCandidateRange` with a regression test for shifted 20-30 line snippets.
