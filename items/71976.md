## Summary

Keep open. Current main still contains the reported light-phase recency-first ordering, the deep ranking signal gate still counts recall + daily + grounded signals together, and rehydration still uses the narrow `preferredSpan + 3` / `8` window. The issue is recent, concrete, unprotected, and describes behavior in the bundled `memory-core` plugin rather than ClawHub or external administration scope.

## Best Possible Solution

Keep #71976 open. The best path is a targeted `memory-core` fix with regression tests: make light-phase staging stop burying real recall-backed entries behind daily/session refreshes, decide whether deep `minRecallCount` should be recall-only or renamed/retuned as a generic signal gate, broaden or otherwise harden rehydration relocation for shifted multi-line snippets, and update docs/tests to match the chosen semantics.
