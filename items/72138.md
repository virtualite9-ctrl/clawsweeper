## Summary

Keep PR #72138 open. Current main still does not implement Feishu normal-reply `message_sent` or internal `message:sent` hook emission, and the PR remains the focused fix for that product gap.

## Best Possible Solution

Keep the PR open or land an equivalent Feishu-owned fix. The maintainable path is to emit canonical plugin `message_sent` and internal `message:sent` from `extensions/feishu/src/reply-dispatcher.ts`, pass routed session key and group mirror metadata from `extensions/feishu/src/bot.ts`, preserve streaming-card message identity where available, avoid false failure hooks for partially delivered streaming text, and add Feishu regression tests for text, card, media, streaming finalization, and failure cases.
