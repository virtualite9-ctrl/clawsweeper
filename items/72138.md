## Summary

Keep PR #72138 open. Current main at 631552c554095b0c520376db9131dac0f5427f94 still has the Feishu normal-reply dispatcher sending replies directly without canonical `message_sent` or internal `message:sent` hook emission, while the generic outbound path already has that hook machinery. The PR remains a focused, relevant fix for open bug #72125 rather than something implemented, duplicated, or outside repo scope.

## Best Possible Solution

Keep the PR open or land an equivalent Feishu-owned fix. The maintainable direction is to add Feishu normal-reply hook emission in `extensions/feishu/src/reply-dispatcher.ts`, pass routed session key and group mirror metadata from `extensions/feishu/src/bot.ts`, include streaming-card message identity where available, avoid false failure hooks for partially delivered streaming text, and cover plain text, card, media, streaming finalization, and failure cases in Feishu tests.
