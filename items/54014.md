## Summary

Keep open. Current main has partial OPENCLAW_HOME support for effective home, default state dir, and default config path, with tests, but the issue’s requested default git checkout behavior is still not implemented: update CLI and the installer still default to the OS/HOME-based ~/openclaw unless OPENCLAW_GIT_DIR is explicitly set. The related #65536 is open and only covers sandbox tilde expansion, so it does not supersede the broader path-contract report.

## Best Possible Solution

Keep the issue open and narrow it to the remaining path contract. A maintainer should decide whether OPENCLAW_HOME is intended to seed the default OPENCLAW_GIT_DIR and installer onboarding paths; if yes, update src/cli/update-cli/shared.ts, scripts/install.sh, docs, and tests. Separately review #65536 for sandbox tilde expansion, and classify shell-env, skills, and trash behavior as either explicit OS-home exceptions or bugs that should move to the shared home resolver.
