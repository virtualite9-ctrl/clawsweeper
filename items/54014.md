## Summary

Current `main` partially addresses the report: default state/config resolution now respects `OPENCLAW_HOME`, but the repo still does not use `OPENCLAW_HOME` for default git checkout/update paths, and the docs still scope `OPENCLAW_HOME` to internal state/config resolution only. That is not strong enough to close this as implemented.

## Evidence

- **State/config paths now honor OPENCLAW_HOME:** `resolveStateDir` and `resolveCanonicalConfigPath` derive defaults through `resolveRequiredHomeDir`, so `~/.openclaw` and `openclaw.json` move under `OPENCLAW_HOME` unless explicitly overridden.
