## Summary

Keep open. Current main partially honors OPENCLAW_HOME for state/config, agent workspace, and agentDir defaults, but the broader reported contract is still not implemented: git checkout defaults, installer/onboarding defaults, and docs still use or document OS-home paths unless OPENCLAW_GIT_DIR or OPENCLAW_CONFIG_PATH is explicitly set.

## Best Possible Solution

Keep this issue open and narrow it to the OPENCLAW_HOME path contract. If git checkout and installer/onboarding defaults are internal OpenClaw paths, update src/cli/update-cli/shared.ts, scripts/install.sh, docs/help/environment.md, docs/cli/update.md, docs/install/installer.md, and focused tests so OPENCLAW_HOME seeds those defaults unless OPENCLAW_GIT_DIR, OPENCLAW_STATE_DIR, or OPENCLAW_CONFIG_PATH is explicitly set. If maintainers want git checkout, personal ~/.agents skills, trash, shell startup probing, or host-file tilde expansion to remain OS-home based, document those as explicit exceptions.
