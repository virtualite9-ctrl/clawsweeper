## Summary

Keep PR #69312 open. Current main still appears to have the reported MEDIA false-positive path: `splitMediaFromOutput()` only skips fenced code spans, then strips leading indentation and accepts `MEDIA:` via a case-insensitive line guard, so 4-space/tab indented code-block lines remain eligible for extraction. The related fence-closing change and regression tests are also not present on main. The PR is recent, focused, linked to #69311, and the discussion shows the author iterated on concrete review findings, so this is useful unmerged work rather than conservative cleanup.

## Best Possible Solution

Keep the PR open and have a maintainer review or land a corrected version that adds indented-code protection in `src/media/parse.ts`, tightens `parseFenceSpans()` if desired, and includes regression tests for indented MEDIA lines, whitespace preservation, lowercase `media:` behavior, and streaming directive buffering semantics.
