## Summary

Keep PR #69312 open. Current main still has the reported MEDIA false-positive path: `splitMediaFromOutput()` skips fenced code only, then trims indentation and treats indented `MEDIA:` lines as directives. The PR is not implemented or superseded on main, and its later review fixes look responsive, but it still needs maintainer review because it also changes shared fence parsing and lowercase MEDIA semantics that current tests explicitly cover.

## Best Possible Solution

Keep the PR open and review it as a plausible core bug fix. The best path is to rebase onto current main, preserve the indented-code-block protection and regression tests, then explicitly decide lowercase MEDIA compatibility and run targeted media, streaming-directive, markdown fence, and shared fence-consumer tests before merging.
