## Summary

Keep PR #69312 open. Current main still has the reported MEDIA false-positive path: `splitMediaFromOutput()` only excludes fenced code spans, then trims leading whitespace before matching `MEDIA:`, so 4-space/tab indented code lines are still eligible for extraction. The PR is a focused bug-fix attempt with tests and active author follow-up, not obsolete cleanup.

## Best Possible Solution

Keep the PR open for maintainer review. The right path is to land a focused fix in `src/media/parse.ts` with regression coverage for 4-space/tab indented code blocks, preserve intended whitespace behavior, update `src/markdown/fences.ts` only if the stricter CommonMark close-line behavior is accepted for all consumers, and reconcile lowercase MEDIA directive semantics across parser, streaming buffer, and tests.
