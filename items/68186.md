## Summary

Keep this PR open. Current main still has the reported macOS screen.snapshot behavior: malformed params are swallowed into defaults, snapshot bytes are base64-encoded into payloadJSON without a transport-size guard, and capture errors can forward framework detail. The PR is a plausible focused hardening fix for the paired open bug report #68181, but it should not be auto-closed or auto-landed because the paired issue is still open and a later Codex review comment identifies a remaining frame-size projection concern around dynamic request/node IDs.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land a corrected version of this hardening in the macOS node runtime: reject malformed screen.snapshot params while preserving nil-param defaults, sanitize true capture/encode failures, classify caller validation errors as INVALID_REQUEST, and bound the actual serialized node.invoke.result frame including dynamic request and node IDs. Keep the paired bug report open until that fix is merged, then close it as implemented.
