## Summary

Keep this PR open. Current main still supports `streamable-http` MCP servers, but the runtime only uses the cached MCP client and forwards a single `client.callTool()` call with no lost-session detection, reconnect, metadata refresh, or retry. The PR branch is a focused implementation candidate, and the latest hardening commit addresses the earlier bot-review concerns with a shared client factory, an in-flight reconnect guard, narrow session-loss detection, retryable failed reconnects, and targeted regression tests. Related #68187 is SSE-specific and remains separate, so it is not a duplicate close path for this streamable-http PR.

## Best Possible Solution

Keep this PR open for review. The best path is to validate and, if CI passes, land the PR branch’s streamable-http reconnect implementation in `src/agents/pi-bundle-mcp-runtime.ts` with the added regression coverage in `src/agents/pi-bundle-mcp-runtime.test.ts`, preserving the custom schema validator, single in-flight reconnect, narrow lost-session detection, metadata refresh, one retry, and retryability after failed reconnect attempts.
