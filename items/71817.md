## Summary

Keep PR #71817 open. Current main still has the Telegram and Feishu channel preview paths resolving reasoning visibility from session state only, even though the core config contract and reply resolver already support per-agent `agents.list[].reasoningDefault`. The PR is a focused fix with tests and active review discussion, not obsolete cleanup.

## Best Possible Solution

Keep the PR open for maintainer and channel-owner review. The best path is a focused merge or update that makes Telegram and Feishu reasoning preview resolution fall back to `agents.list[].reasoningDefault` after explicit session overrides, preserves explicit session `off`, and keeps Feishu's preview limited to the `stream` variant. Broader global/per-model reasoning defaults should stay tracked separately.
