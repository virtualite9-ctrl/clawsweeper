## Summary

Keep open. Current main fixes the top-level `usage.cost` aggregate path for reset/deleted archives, but the per-session/dashboard path from the reporter's latest comment still resolves and scans only one transcript, preferring the active `.jsonl` file when it exists. That means `sessions.usage` can still undercount a session that has both `sess.jsonl` and `sess.jsonl.reset.<timestamp>` files.

## Best Possible Solution

Keep the issue open and land a focused core fix that resolves all usage-counted transcripts for a session id, then has `loadSessionCostSummary()`, `sessions.usage`, timeseries, and logs handle active plus reset/deleted archives without counting `.bak` or checkpoint artifacts. Add a regression test with both `sess.jsonl` and `sess.jsonl.reset.<timestamp>` present that expects combined session cost/tokens for `sessions.usage`.
