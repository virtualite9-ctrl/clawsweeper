## Summary

Keep this PR open. Current main still only discovers immediate child skill directories, while the PR addresses an open linked regression (#56915) for nested `SKILL.md` files under a configured skills root. The request is in core skill loading, not a ClawHub/plugin-only item, and the PR context shows the main security review concerns were followed up with additional commits.

## Best Possible Solution

Keep the PR open for maintainer review or replacement with a maintainer-authored patch. The best path is a bounded recursive skill discovery change in `src/agents/skills/local-loader.ts` and `src/agents/skills/workspace.ts`, regression tests for nested workspace/managed/extra-dir skills plus symlink loop and escape cases, and a deliberate docs/watcher update if nested skill roots are now part of the supported contract.
