## Summary

Current main still has the one-level skill discovery behavior that #69663 is trying to change. `loadSkillsFromDirSafe()` only enumerates immediate child directories, and `loadWorkspaceSkillEntries()` skips first-level skill-root children unless that child directly contains `SKILL.md`, so a layout like `~/.openclaw/skills/coze/koze-retrieval/SKILL.md` from #56915 remains a plausible miss on main. The PR is a focused bug-fix attempt rather than obsolete cleanup, and it is not maintainer-authored or protected-labeled.

## Best Possible Solution

Keep #69663 open for maintainer review. The best path is to land a contained recursive discovery fix with regression coverage for nested managed/workspace/extra skill roots, address the remaining cosmetic review comments, and decide whether the watcher glob should be expanded for nested `SKILL.md` refresh parity. If maintainers do not want recursive category folders, document the one-level layout explicitly and resolve #56915 through that product decision instead.
