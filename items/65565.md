## Summary

Current main still has the reported Dashboard Agents UI/backend mismatch: Set Default stages `agents.defaultId`, while the schema-backed persisted default is `agents.list[].default` and the resolver still falls back to the first list entry when no agent has `default: true`.

## Best Possible Solution

Fix the Agents UI Set Default action to update `agents.list` by marking the selected agent `default: true` and clearing any other `default` flags. Add focused UI/controller coverage proving the serialized `config.set` payload contains valid `agents.list[].default` data and does not contain `agents.defaultId`. Keep the current resolver contract unless maintainers intentionally add and document a new schema-backed top-level default field.
