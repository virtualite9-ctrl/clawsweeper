## Summary

Keep open. Current main still has the reported UI/backend mismatch: the Agents view stages `agents.defaultId`, while the schema only accepts `agents.defaults` and `agents.list`, and runtime/docs use `agents.list[].default` with first-list-entry fallback. Open PR #72585 directly targets this bug and uses closing syntax, so the issue should remain open until that PR is reviewed and merged or otherwise resolved.

## Best Possible Solution

Keep this issue open and review PR #72585 as the active implementation candidate. The likely product fix is to update the Agents UI Set Default action to mark exactly one `agents.list[].default: true`, clear other default markers, and add regression coverage proving the serialized `config.set` payload remains schema-valid and does not include `agents.defaultId`.
