## Summary

Current `main` still does not implement the requested manual token/request rate limiter. OpenClaw has reactive 429 handling, Retry-After caps, auth-profile cooldowns, key/profile rotation, and model fallback, but no documented per-provider/per-model tokens-per-interval or requests-per-interval config surface or preflight scheduler that enforces those budgets before model calls.

## Best Possible Solution

Keep #30828 open for a core model request-path design: add a documented per-provider/per-model config schema for request and token budgets with interval windows, enforce it before shared model calls, reconcile actual response usage, optionally normalize provider headers such as OpenAI `x-ratelimit-*`, and cover scheduler interactions with auth-profile rotation, fallback, retries, streaming, and docs.
