## Summary

Current main still lacks the requested generic/manual model rate limiter. OpenClaw has reactive 429 handling, Retry-After bypass, auth-profile cooldown/rotation, model fallback, context token budgeting, webhook/gateway ingress limiters, and Codex account rate-limit display, but no documented config or scheduler that enforces per-model tokens-per-interval and requests-per-interval before model calls.

## Best Possible Solution

Keep #30828 open for a core model request-path design: add a documented per-provider/per-model, and possibly per-auth-profile, config schema for request and token budgets with interval windows; enforce it before shared model calls; reconcile actual usage after responses/streams; optionally ingest provider headers such as OpenAI x-ratelimit-*; and cover interactions with auth-profile rotation, fallback, retries, streaming, docs, and tests.
