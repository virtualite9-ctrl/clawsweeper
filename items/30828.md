## Summary

Keep #30828 open. Current main still has reactive rate-limit handling through Retry-After caps, auth-profile cooldowns, profile rotation, and model fallback, but it does not expose or enforce the requested manual tokens-per-interval or requests-per-interval limiter for model calls. The OpenAI x-ratelimit header suggestion from the discussion is observed/logged in live probe coverage and docs, but not used for scheduling or budget enforcement.

## Best Possible Solution

Keep the issue open for a core model request-path design: add a documented per-provider/per-model config schema for request and token budgets with interval windows, enforce it before shared model calls, reconcile usage after responses, optionally normalize provider headers such as OpenAI x-ratelimit remaining/reset, and cover scheduler/failover interactions with tests and docs.
