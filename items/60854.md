## Summary

Keep open. Current main still uses the low-opacity `--accent-subtle` token for global text selection, so the linked accessibility bug is not implemented on main. The PR remains relevant, though its final foreground/token choice should address the review feedback before merge.

## Best Possible Solution

Keep #60854 open and finish it as the canonical fix for #60850. The best path is a scoped `ui/src/styles/base.css` change that replaces the global low-opacity `--accent-subtle` selection background with dedicated theme-aware selection tokens, verifies readable foreground contrast across default, light, openknot, openknot-light, dash, dash-light, and custom-theme behavior, and adds a small CSS/token regression check if the UI test setup can cover it.
