## Summary

Keep this PR open, but do not land it unchanged. Current main still classifies `gateway` `config.schema.lookup` as a mutating tool call, so the PR's proposed `mutatingAction === false` suppression would not run on the real reported path. The bug remains actionable in core agent/gateway notification handling, and the PR needs a small root-cause fix plus boundary tests. Security review found no CI, dependency, lockfile, publishing, secret, or external code-execution changes in the PR surface.

## Best Possible Solution

Update this PR rather than close it: classify `gateway` `config.get` and `config.schema.lookup` as read-only in `src/agents/tool-mutation.ts` or otherwise pass the gateway action into the warning policy, keep the targeted `config schema path not found` suppression in the payload layer, and add regression tests for the real runtime classification plus inverse cases where mutating gateway failures still warn.
