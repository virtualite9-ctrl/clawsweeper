## Summary

Keep open. Current main still does not implement the requested suppression, and this PR is relevant but incomplete as submitted because its strict `mutatingAction === false` guard does not match the current runtime classification for `gateway` `config.schema.lookup`. The related #42898 classifier PR addresses part of the root cause but does not replace the narrow payload-layer suppression this PR is trying to add.

## Best Possible Solution

Revise this PR or coordinate it with #42898 so the canonical fix both classifies compound read-only gateway actions correctly and suppresses only the exact benign `config.schema.lookup` path-not-found miss. Add tests for the real runtime path from `buildToolMutationState` into `lastToolError` and payload rendering, plus inverse tests proving mutating gateway failures and other gateway errors still surface.
