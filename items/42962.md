## Summary

Keep this PR open, but do not land it unchanged. Current main still classifies `gateway` `config.schema.lookup` as mutating, so this PR's `mutatingAction === false` suppression would not run on the reported runtime path. The related open PR #42898 targets the classifier root cause, but a complete fix likely needs both the classifier correction and a narrow payload suppression for this benign lookup miss.

## Best Possible Solution

Revise this PR or coordinate it with #42898 so one canonical fix classifies compound read-only gateway actions correctly, suppresses only the exact benign `config.schema.lookup` path-not-found miss when appropriate, and adds tests for the real classifier-to-payload path plus inverse mutating gateway failures.
