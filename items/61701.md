## Summary

Current main implements the fix for the sustained idle gateway CPU/plugin-discovery loop. A later, more diagnostic report (#73088) reproduced the same one-core `openclaw-gateway` behavior on 2026.4.25 and was closed by a maintainer after the config-schema/plugin-metadata fix landed; current source now reuses the Gateway's plugin metadata snapshot for `config.get`/`config.schema` instead of rebuilding plugin manifest/channel schema metadata on each request.

## Best Possible Solution

Close this older broad gateway CPU report as implemented on current main. Keep the metadata-snapshot schema path, and let the newer narrower startup/sidecar blockers (#72846 and #72960) track any remaining post-ready channel or plugin-service stalls. If sustained idle gateway CPU still reproduces on a build containing this fix, the right path is a fresh issue with current version, startup trace, CPU profile/strace, config schema request evidence, and affected plugins/channels.
