## Summary

Keep #11665 open. Current main still sends `/hooks/agent` and mapped hook agent actions through an isolated CronJob, and `runCronIsolatedAgentTurn` passes `forceNew: true` for isolated jobs. `resolveCronSession` can reuse an existing session only when `forceNew` is false, so a stable hook `sessionKey` still cannot preserve multi-turn transcript history. The previously proposed fixes were closed unmerged, and no `sessionMode`/`reuseSession` hook API is present on current main.

## Best Possible Solution

Keep the issue open and implement a backward-compatible persistent hook-session option in core, likely `sessionMode: "isolated" | "persistent"` or equivalent on `/hooks/agent` payloads, hook mappings, and transform results, defaulting to isolated. Thread it through hook normalization, config types/schema/help, mapping merge, dispatch, docs, and regression tests, then pass `forceNew=false` only for the persistent hook path.
