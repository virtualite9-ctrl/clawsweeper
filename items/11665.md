## Summary

Keep #11665 open. Current main still sends `/hooks/agent` and mapped agent hooks through an isolated cron-style turn, and that isolated path still passes `forceNew: true`, so a stable hook `sessionKey` does not reuse the existing transcript/session id. The hook payload, mapping, transform, config schema, and docs expose `sessionKey` but no `sessionMode`, `reuseSession`, or equivalent persistent/reuse control. The related fix PRs in the provided context (#15084, #27123, #27193) are closed unmerged, and the latest issue discussion already reached the same keep-open conclusion.

## Best Possible Solution

Keep the issue open and implement a backward-compatible persistent hook-session option in core, likely `sessionMode: "isolated" | "persistent"` or an equivalent reuse flag. Thread it through direct `/hooks/agent` payloads, mapped hook actions, transform results, config types/schema/help, dispatch, docs, and regression tests, then pass `forceNew: false` only for the persistent hook path while preserving the current isolated default.
