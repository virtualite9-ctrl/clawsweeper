## Summary

Keep open. Current main has generic tool-loop detection, but it does not implement the proposal's specific exec-runtime same-file reread detector. The shipped guard is opt-in, hashes full tool parameters, and only special-cases polling-style process/command_status loops; it does not normalize sed/nl/cat/rg readback commands to a file identity or return a visible synthetic warning result for varying line-window rereads.

## Best Possible Solution

Keep this issue open. The best implementation is a narrow exec readback detector wired into the existing before-tool/exec path: classify only obvious read-only commands, normalize cwd plus target file paths while ignoring line ranges, track same-file and same-file-pair counts per run/session turn, emit a visible warning-only tool result and diagnostic event at the proposed thresholds, and add focused tests plus docs/config notes for false-positive tuning.
