## Summary

Close as `clawhub`. The request is coherent and useful, but current main scope pushes reusable procedural workflows into skills/plugins, not a new core artifact. Existing main surfaces already cover the proposed area through workspace `SKILL.md` procedures, the Skill Workshop plugin, and the OpenProse multi-agent orchestration plugin.

## Risks / Open Questions

- If maintainers later want one standardized external workflow format shared across multiple plugins, closing this as out-of-scope could hide that broader product need.

## Close Comment

Closing after a Codex review of current `main`. This is a useful idea, but it fits the repo’s ClawHub/plugin lane rather than OpenClaw core. The scope guardrails in `VISION.md:68` and `VISION.md:101` explicitly route new reusable skill-like behavior to ClawHub and avoid heavier core orchestration layers. Current `main` already has the right extension points for this: reusable procedure artifacts live as `SKILL.md` skills (`docs/tools/skills.md:9`), Skill Workshop can turn repeated workflows into workspace skills (`docs/plugins/skill-workshop.md:16`), and the `open-prose` plugin already provides multi-agent orchestration with serial handoffs and user checkpoints (`extensions/open-prose/README.md:21`, `extensions/open-prose/skills/prose/examples/README.md:125`). On that evidence, this should be pursued as a ClawHub skill/plugin or OpenProse package, not as a new native `playbooks/PLAYBOOK.md` artifact in core.
