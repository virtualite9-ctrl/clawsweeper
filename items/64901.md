## Summary

Keep this PR open. Current main still lacks the proposed default-source warning/flag, and the Bedrock-motivated behavior remains unresolved because Bedrock discovery still stores fallback contextWindow values as model metadata, so runtime would see source=model rather than source=default.

## Best Possible Solution

Keep this PR open and ask for changes. The best path is a source-aware implementation that distinguishes true provider/model metadata from hardcoded fallback values, likely by carrying optional contextWindow or context-window source metadata through Bedrock discovery and model config, then warning in resolveEffectiveRuntimeModel with tests for both generic default resolution and Bedrock fallback discovery.
