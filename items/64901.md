## Summary

Keep this PR open. Current main still does not implement the proposed default-source warning, and the PR’s current approach remains incomplete for the Bedrock fallback case discussed in review, even though current main has since improved Bedrock discovery with known context-window metadata for many models.

## Best Possible Solution

Keep the PR open for maintainer review or author changes. If the warning is still desired, make it source-aware enough to distinguish true provider/model metadata from hardcoded fallback values, including Bedrock unknown-model/defaultContextWindow fallbacks, and add tests for both generic default resolution and Bedrock fallback discovery. If maintainers consider the shipped Bedrock known-context-window metadata sufficient, close this PR explicitly as superseded by that product direction rather than as already implemented.
