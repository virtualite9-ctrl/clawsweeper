## Summary

Keep open. Current main still routes Telegram group sessions as agent-scoped keys and applies the generic session visibility plus agent-to-agent guard to `sessions_list` and `sessions_send`; there is no Telegram same-group cross-agent symmetry rule or regression test for the reported two-agent group case. The related #59933 remains distinct because it covers same-agent group listing, not this cross-agent one-way failure.

## Best Possible Solution

Keep this issue open. Reproduce with two configured agents in the same Telegram group under explicit `tools.sessions.visibility="all"` and enabled, allowlisted `tools.agentToAgent`; if it still reproduces, add regression coverage that both agents list both `agent:<agent>:telegram:group:<id>` keys and can `sessions_send` both directions, then fix the gateway/session-store/visibility path. Track #59933 separately because it is same-agent group listing, not this cross-agent mismatch.
