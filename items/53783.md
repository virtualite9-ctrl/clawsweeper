## Summary

Keep this issue open. Current main still has agent-scoped Telegram group session keys plus generic cross-agent session visibility guards, and I found no same-Telegram-group symmetry rule or regression coverage that proves both agents can list and send to each other in the reported two-agent group case. The related #59933 remains distinct because it covers same-agent group listing, not cross-agent one-way visibility/send failure.

## Best Possible Solution

Keep this issue open. Reproduce with two configured agents in the same Telegram group under explicit tools.sessions.visibility="all" and enabled, allowlisted tools.agentToAgent. If it still reproduces, add regression coverage proving both agents list both agent:<agent>:telegram:group:<id> keys and can sessions_send both directions, then fix the gateway/session-store/visibility path. Track #59933 separately because it is same-agent group listing, not this cross-agent mismatch.
