## Summary

Keep open. Current main still records Telegram group conversations as agent-scoped session keys and routes sessions_list/sessions_send through the generic session visibility and agent-to-agent policy. I did not find a Telegram same-group bidirectional visibility guarantee, a regression test for two agents in the same group, or a superseding merged fix. Related #59933 is explicitly same-agent group visibility, so it does not cover this cross-agent mismatch.

## Best Possible Solution

Keep this issue open. Reproduce with two configured agents in the same Telegram group using explicit tools.sessions.visibility="all" and enabled, allowlisted tools.agentToAgent. If the mismatch reproduces, add regression coverage that each agent can see both agent:<agent>:telegram:group:<id> keys in sessions_list and resolve/send through sessions_send both directions, then fix the gateway/session-store/visibility path. If same-group cross-agent sends are intentionally opt-in, improve the sessions_send/sessions_list diagnostics and docs so users get a clear config explanation instead of one-way missing-session behavior. Track #59933 separately because it covers same-agent group listing, not this cross-agent mismatch.
