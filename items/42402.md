## Summary

Keep open. Current main still has the env.vars crash path this PR targets: env.vars values are collected without a runtime string guard, then passed to containsEnvVarReference, which calls .includes() on a string-typed value. The PR is stale against the current file layout but remains a valid focused bug-fix candidate.

## Best Possible Solution

Keep this PR open as a focused bug-fix candidate. Rebase it onto current main by applying the non-string guard in src/config/config-env-vars.ts, move the regression coverage into src/config/config.env-vars.test.ts, and decide whether malformed env.vars values should warn before running the relevant config env-var test lane or changed gate.
