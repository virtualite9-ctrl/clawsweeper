## Summary

Keep open. Current main still lacks the `env.vars` runtime type guard this PR proposes; a truthy non-string parsed from JSON can still be collected, passed to `containsEnvVarReference`, and crash on `.includes()` before schema validation reports the bad config.

## Best Possible Solution

Keep this PR open as a valid focused bug-fix candidate. Rebase it onto current main by applying the non-string guard in `src/config/config-env-vars.ts`, move the regression test into `src/config/config.env-vars.test.ts`, and decide whether skipped non-string `env.vars` should emit a diagnostic before running the relevant config env-var test lane or changed gate.
