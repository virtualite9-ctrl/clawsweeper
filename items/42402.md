## Summary

Keep this PR open. Current main still lacks the specific runtime guard for truthy non-string values inside `env.vars`: `collectConfigEnvVarsByTarget` only skips falsy values before those entries are later passed to `containsEnvVarReference`, which still calls `.includes()` on a string-typed parameter. The broader config traversal path now guards strings before calling `containsEnvVarReference`, so the original linked issue is partly addressed, but this PR’s narrower `env.vars` crash remains plausible and useful. The PR needs a rebase because the implementation has moved from `src/config/env-vars.ts` to `src/config/config-env-vars.ts`.

## Best Possible Solution

Keep the PR open as a valid, focused bug-fix candidate. Rebase it onto current main by applying the guard in `src/config/config-env-vars.ts`, moving the regression test into `src/config/config.env-vars.test.ts`, and considering a small diagnostic for non-string `env.vars` values. Then run the relevant config env-var tests or changed gate.
