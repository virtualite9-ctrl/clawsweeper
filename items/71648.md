## Summary

Keep this PR open. Current main still has the MCP channel bridge leak pattern the PR targets: raw pendingClaudePermissions and pendingApprovals Maps are populated without TTL/sweeper or close-clear, and cleanup depends only on matched Claude replies or approval resolved events. The PR is paired with still-open issue #71646 from the same author, includes focused regression tests, and addressed its bot review feedback in head commit 9f16dd4823547354c6d7f85b0dab7fe672008010.

## Best Possible Solution

Keep this PR open for maintainer review and normal validation. The best path is to land a focused MCP bridge fix that wraps pending map entries with stable timestamps, adds a lazy self-terminating TTL sweeper, clears both pending maps on close, guards post-close writes, and keeps the separate cap/FIFO concern as a narrower follow-up if maintainers want it.
