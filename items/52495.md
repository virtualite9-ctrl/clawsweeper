## Summary

Close PR #52495 as superseded by current main docs and the existing review feedback. The PR adds `memory_search` / `memory_get` to the table for tools that are "available without installing any plugins," but current main documents those tools as active-memory-plugin tools instead, and the code confirms they are registered by memory plugins, not unconditional core built-ins.

## Best Possible Solution

Close this PR and keep the current docs model: built-in-only tools remain in the built-in table, while memory-tool docs stay under Memory, CLI Memory, plugin, and tool-group documentation. If discoverability still needs improvement, the best follow-up is a new narrow docs PR that explicitly labels these as active memory plugin tools rather than unconditional built-ins.

## Evidence
