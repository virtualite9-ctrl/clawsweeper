## Summary

Do not close. Current main still depends on the active browser chat run reaching a matching terminal event before it stops treating the session as busy, and I did not find a Control UI-side watchdog or other clear evidence that the reported "second message stays queued until refresh" failure has been fixed.

## Evidence

- **Busy state still blocks direct sends:** The browser chat stays busy whenever `chatRunId` is set, so a stale active-run flag still pushes follow-up input into the queue instead of sending immediately.
  - file: [ui/src/ui/app-chat.ts:60](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/ui/src/ui/app-chat.ts#L60)
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **UI still exposes Queue during active runs:** The run controls still switch the primary action to `Queue` while a run is active, which matches the reporter's observed symptom once the UI thinks a run never finished.
  - file: [ui/src/ui/chat/run-controls.ts:43](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/ui/src/ui/chat/run-controls.ts#L43)
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **Run state clears only on terminal chat events:** `handleChatEvent` clears `chatRunId` only on `final`, `aborted`, or `error` for the tracked run. If the terminal event is missed or does not match the tracked run, the busy state remains.
  - file: [ui/src/ui/controllers/chat.ts:433](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/ui/src/ui/controllers/chat.ts#L433)
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **Current tests intentionally preserve busy state for other-run finals:** There is still explicit regression coverage that a final event from another run must not clear the current `chatRunId`, so the client remains sensitive to run-id mismatches.
  - file: [ui/src/ui/controllers/chat.test.ts:410](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/ui/src/ui/controllers/chat.test.ts#L410)
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **No shipped Control UI fix found in latest release snapshot:** Comparing `v2026.4.22` against current main did not reveal a release-note-backed Control UI watchdog or similar state-reset fix for this path; the release snapshot still uses the same terminal-event clearing logic in `ui/src/ui/controllers/chat.ts`.
