## Summary

Current `main` has partial mitigation for the report: plugin discovery now resolves duplicate IDs by source precedence, `doctor --fix` repairs legacy bundled plugin load paths, and doctor preflight can recover configs with a non-JSON prefix. But the duplicate-warning behavior is still intentional for genuine installed-vs-bundled conflicts, and I did not find strong evidence in `main` that the reported gateway restart timeout/health-check failure is fully resolved. Conservative maintainer call: keep open.
