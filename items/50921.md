## Summary

Current main does not ship Eternal AI, but this PR is an optional third-party text-inference provider and does not identify a missing core/plugin API. OpenClaw's current direction and docs make provider plugins installable outside the core repo through ClawHub/npm, and current main exposes the provider registration, API-key auth, catalog, onboarding, and runtime hook seams needed for an Eternal AI provider. The PR branch itself is a useful posted provider-plugin implementation, but it should be adapted to the current SDK and published externally rather than bundled into core by default.

## Best Possible Solution

Close this core PR and direct the work to ClawHub/npm as an external Eternal AI provider plugin. The best implementation path is to adapt the posted #50921 extension to current SDK subpaths, use the documented provider-entry/API-key/catalog/onboarding helpers, and add provider-local hooks such as prepareExtraParams or wrapStreamFn if Eternal-specific request metadata or headers are required.
