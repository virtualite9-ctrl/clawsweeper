## Summary

Keep this issue open. Current main has shallow HTTP liveness/readiness probes, CLI/RPC health snapshots, model status commands, and an OpenAI-compatible `/v1/models` surface, but none is equivalent to the proposed auth-gated `/health/deep` endpoint that returns the active backend provider/model/fallback contract and routing/error telemetry without inference. The issue is recent, concrete, contributor-authored, and asks for maintainer API/security direction before a PR.

## Best Possible Solution

Keep the issue open for maintainer product/API direction. The likely implementation is an additive Gateway probe, either `/health/deep` or an extension of authenticated `/ready` details, backed by existing model/fallback resolution plus new bounded routing/error telemetry hooks. It should include tests for local versus remote auth, unauthenticated leakage boundaries, no-inference behavior, redaction, and docs on the public Gateway health/model pages.
