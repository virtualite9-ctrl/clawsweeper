## Summary

Keep open. Current main does not implement the requested native `/health/deep` HTTP probe, and existing health/readiness surfaces do not expose the active model/provider/fallback or routing/error telemetry described in #70191. The issue is recent, coherent, contributor-authored, and asks for maintainer API/auth/schema direction before a PR.

## Best Possible Solution

Leave #70191 open for maintainer product/API judgment. The best path is to decide the additive surface first: either implement a core-owned authenticated readiness/diagnostic endpoint with server-http tests, health schema/docs, and routing/error telemetry hooks, or define a plugin observability API that lets an external plugin expose the same data without importing gateway internals.
