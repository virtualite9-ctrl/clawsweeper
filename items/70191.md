## Summary

Keep open. Current main does not implement a native `/health/deep` HTTP probe or an equivalent cheap authenticated HTTP surface that returns the active model/provider/fallback contract. Existing HTTP probes are limited to `/health`, `/healthz`, `/ready`, and `/readyz`; the WS `health` snapshot is channel/session health oriented; and `status`/`models.list` expose adjacent data but not the proposed model-contract probe with routing/error telemetry. The issue is recent, has concrete prior art via the linked Nix overlay, and asks maintainer design questions about a core gateway/observability surface, so it should not be auto-closed or moved to ClawHub without maintainer direction.

## Best Possible Solution

Keep #70191 open for maintainer design review. If accepted, add an additive authenticated core HTTP probe with docs and server tests, reuse or deliberately revise the `/ready` detail auth policy, derive the default model/fallback contract through existing model-selection helpers, and add explicit routing/error telemetry hooks before exposing `lastRoutingDecision` or recent error fields. If maintainers prefer this outside core, first define the missing plugin/observability API contract rather than closing the request as already satisfiable.
