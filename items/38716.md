## Summary

Keep #38716 open. Current main still lacks Slack RTS support via assistant action tokens: no assistant thread event subscriptions, no action_token capture/storage, no assistant.search.context call, and no Slack search action exposed through the message tool. The related search.messages PR context is a different user-token Web API path and does not implement the requested assistant action-token RTS flow.

## Best Possible Solution

Keep the issue open. The best path is a focused Slack plugin feature: subscribe to assistant_thread_started and assistant_thread_context_changed, capture action_token only for the relevant assistant/user interaction scope, add an assistant.search.context helper, expose it through a Slack search action or dedicated tool, and update Slack setup/docs/tests for Agents & AI Apps and required search scopes. If maintainers prefer externalization, first track the missing plugin/API seam for accessing the existing Slack app event stream and assistant action tokens without replacing the bundled Slack channel.
