## Summary

Keep #38716 open. Current main at 4a3030df9efa72c44bab567a4390ac0c4876a73f still does not implement Slack RTS via assistant action tokens: the Slack plugin does not subscribe to assistant thread events, does not capture or store action_token, does not call assistant.search.context, and does not expose Slack search through the shared message action surface. The related search.messages PR context is a different user-token Web API search path and does not supersede assistant.search.context/action_token support.

## Best Possible Solution

Keep the issue open. The best implementation path is in the bundled Slack plugin unless maintainers make a product call to split it out: subscribe to assistant_thread_started and assistant_thread_context_changed, capture and scope action_token to the assistant thread, add an assistant.search.context helper exposed as a Slack message action or dedicated tool, gate setup on the required Slack app settings/scopes, and add focused tests plus Slack setup/docs updates. If maintainers prefer ClawHub, first open a narrower design item for how an external plugin can extend the existing Slack app/event stream and message action surface without replacing the bundled Slack channel.
