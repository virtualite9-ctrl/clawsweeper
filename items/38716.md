## Summary

Keep #38716 open. Current main does not extract Slack assistant action tokens or call Slack RTS `assistant.search.context`; Slack message actions still expose read/send-style operations only. The related #58143 discussion is useful context, but that PR was closed unmerged and proposed `search.messages` with a user token, not the requested assistant action-token RTS flow.
