## Summary

Keep this PR open. Current main still does not implement Telegram waiting-tip messages or the proposed `messages.waitingTips` config, and the PR overlaps the still-open Telegram wait-feedback request in #6946. It is not merge-ready as-is because the review discussion identifies a realistic cleanup race and an advertised-but-unimplemented `customTipsPath`, but those are request-for-changes issues rather than a conservative close reason.

## Best Possible Solution

Keep the PR open for maintainer/author follow-up. Rework it against current `extensions/telegram/src/bot-message-dispatch.ts`, fix the send/cancel cleanup race, remove or fully implement `customTipsPath`, add focused Telegram dispatch tests for success, fast-stream cancellation, cleanup failure, forum topics, and superseded turns, and decide whether the shipped feature should be a narrow processing indicator aligned with #6946 or a separate product-approved bilingual tip feature.
