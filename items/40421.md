## Summary

Current main still does not implement Telegram waiting-tip messages or the proposed messages.waitingTips config, and this PR remains a plausible implementation candidate rather than obsolete. It is not merge-ready as submitted: the review discussion identifies a send/cancel cleanup race, an advertised but unimplemented customTipsPath option, and the patch targets the older src/telegram path rather than the current extensions/telegram plugin layout.

## Best Possible Solution

Keep this PR open for author or maintainer follow-up. Rebase it onto extensions/telegram/src/bot-message-dispatch.ts, decide whether the feature should be a narrow Telegram processing indicator aligned with #6946 or a product-approved bilingual waiting-tip feature, fix the send/cancel cleanup race, remove or fully implement customTipsPath, and add focused Telegram dispatch coverage for fast-stream cancellation, successful cleanup, cleanup failure, forum-topic thread params, and superseded turns.
