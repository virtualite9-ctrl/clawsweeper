## Summary

Keep this issue open. Current main still has the reported malformed-key path: MSTeams mutates the resolved route's session key in place, the MSTeams helper passes an already-qualified base through unchanged, the shared suffix helper blindly appends another `:thread:` segment, and the route cache can retain the mutated object from the first cache-miss return.

## Best Possible Solution

Fix MSTeams thread routing so cached base routes remain immutable. Compute a separate thread-qualified session key from a clean base route, pass that derived key through MSTeams context/session dispatch without mutating the cached route object, and add a regression test for two routed channel-thread messages in the same channel asserting the second key is `base:thread:new`, never `base:thread:old:thread:new`.
