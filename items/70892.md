## Summary

Keep open. Current main still lacks the requested user-facing prompt-section gates for the CLI quick reference, reply-tag/directive guidance, and messaging guidance. The `<available_skills>` part is partially covered by existing agent skill allowlists, but the broader core config/docs request remains unimplemented and is not a ClawHub or duplicate cleanup candidate.

## Best Possible Solution

Keep the issue open for focused core prompt/config work: add typed, schema-backed default-true gates for the current full-prompt sections if maintainers want this operator control; reuse the existing agent skill allowlist for include-style `<available_skills>` filtering and add an exclude form only if needed; document required vs optional systemPromptOverride building blocks; and add prompt-builder/schema/docs tests covering char-reduction behavior and cache-boundary safety.
