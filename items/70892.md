## Summary

Keep open. Current main partially addresses the `<available_skills>` concern through the existing `agents.defaults.skills` / `agents.list[].skills` allowlist, but it does not implement user-facing per-section gates for the CLI quick reference, reply-tag guidance, or messaging component/tool guidance, and the requested `systemPromptOverride` structural-vs-optional cookbook is not present.

## Best Possible Solution

Keep the issue open as an actionable core prompt/config enhancement. The likely maintainer path is to add typed default-true prompt section gates under a stable config surface, thread them through prompt assembly and runtime call sites, update generated schema/help/docs, add focused prompt tests for each omitted block, and document required versus optional `systemPromptOverride` building blocks. For `<available_skills>`, prefer documenting or extending the existing skill allowlist unless maintainers specifically want a subtractive exclude list.
