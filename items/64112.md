## Summary

Keep this PR open. Current main still emits the plugin allowlist fallback from the raw missing primary command without first proving that the command is a real bundled plugin CLI root, and current tests still lack the `openclaw run` restrictive-allowlist regression. The PR remains a focused implementation candidate, with review attention still needed on the detection-error resilience concern.

## Best Possible Solution

Keep this PR open and either land it after resolving the detection-error resilience concern, or apply an equivalent main-branch fix: only emit `plugins.allow` / `plugins.entries.<id>.enabled=false` diagnostics after resolving the missing primary command to a real bundled plugin CLI root or declared manifest-owned command alias, and add the restrictive-allowlist regression proving `openclaw run` uses normal unknown-command handling.
