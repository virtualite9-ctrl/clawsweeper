## Summary

Keep this PR open. Current main still has the broad CLI fallback that can report a plugin allowlist error for an arbitrary unknown primary command, and I did not find the PR's equivalent real-bundled-plugin-root gate or its `openclaw run` regression coverage on main.

## Best Possible Solution

Keep this PR open and review or land it after handling the remaining detection-error resilience concern, or apply an equivalent main-branch fix: only emit plugin allowlist/disablement diagnostics after resolving the missing primary command to a real bundled plugin CLI root or manifest-owned command alias, and add the restrictive-allowlist regression for `openclaw run` falling through to normal unknown-command handling.
