## Summary

Keep this PR open. Current main still lets arbitrary missing primary commands fall into the plugin allowlist diagnostic path, and this PR remains a useful implementation candidate, though it now needs rebase/adaptation to the current run-main-policy split.

## Best Possible Solution

Keep this PR open and finish maintainer review. The best path is to rebase or apply the same behavior on current main: gate plugins.allow and plugins.entries.<id>.enabled=false diagnostics on a real bundled plugin CLI root, map aliased roots such as voicecall back to their owning plugin id, keep unknown commands such as run on the normal unknown-command path, and preserve the denylist/probe-failure regression coverage.
