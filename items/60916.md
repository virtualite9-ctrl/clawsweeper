## Summary

Current main already includes heartbeat reliability fixes that address this 2026.4.2 report: busy-lane heartbeats are retried instead of being drained/lost, `heartbeat.target="last"` is preserved through wake retries/coalescing, and the heartbeat path does not depend on `sessions_history`.

## Evidence

- **Heartbeat skips busy lanes instead of draining work:** `runHeartbeatOnce` now returns `requests-in-flight` when either the main lane or the resolved session lane is busy, with an inline note that the wake layer will retry rather than interrupt or lose the pending heartbeat.
