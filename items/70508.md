## Summary

Keep #70508 open. Current main still matches the reported Mattermost draft-stream bug: partial text updates one preview post, assistant/reasoning boundaries only clear local text state, and tool starts write status text into that same draft stream without flushing or rotating the post. The PR is a focused fix attempt, not implemented or superseded on main, but its current head has unresolved review concerns around async callback ordering and missing regression coverage.

## Best Possible Solution

Keep the PR open or replace it with an equivalent narrow Mattermost fix. The best path is to track whether Mattermost has streamed visible draft content, flush before rotating, call forceNewMessage at assistant-message/reasoning/tool boundaries as appropriate, handle onItemEvent/onToolStart ordering deterministically if rich tool status remains in scope, and add targeted Mattermost regression tests for partial -> tool -> assistant boundary sequences.
