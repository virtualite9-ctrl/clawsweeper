## Summary

Keep #70578 open. Current main has important guardrails against the old silent clobber path: source keys missing from the runtime projection are now preserved, plugin install refuses to proceed on generally invalid config, and suspicious size-drop writes are rejected before replacement. But the issue is not fully implemented: the root `OpenClawSchema` is still `.strict()`, unknown top-level config remains invalid rather than pass-through/preserved as an accepted round-trip contract, and the bundled/plugin CLI command allowlist behavior the report calls out is still current and explicitly tested.

## Best Possible Solution

Keep the issue open but narrow the fix. Main already has useful anti-clobber guardrails, so the remaining maintainer path should decide the config contract explicitly: either preserve unknown top-level authored data with a safe raw-source/pass-through mechanism plus regression coverage for `plugins install`, or keep strict rejection and add doctor/docs guidance for legacy top-level keys. Separately, decide whether bundled CLI command surfaces should bypass `plugins.allow` or whether the current allowlist-gated behavior should stay documented.
