## Summary

Keep open. Current main still leaves key Control UI chat, form, and config text controls below the 16px iOS Safari focus threshold, and open PR #64673 remains an unmerged implementation candidate for this issue.

## Best Possible Solution

Keep this issue open. Review and land PR #64673 after rebasing and broadening it if needed, or replace it with a maintainer patch that sets focusable Control UI text inputs to at least 16px on iOS/touch-primary devices. The patch should cover the main chat composer, agent chat textarea, generic .field controls, config inputs/textareas/selects, and include a small CSS regression test for those selectors.
