## Summary

Keep open. Current main still leaves Control UI chat and form inputs below the 16px iOS Safari focus threshold, and open PR #64673 remains an implementation candidate that uses closing syntax for this issue.

## Best Possible Solution

Keep this issue open and review/land PR #64673 after updating it against current main, or replace it with a maintainer patch that sets all focusable Control UI text inputs to at least 16px on iOS/touch-primary devices. The patch should cover the main chat composer, agent chat textarea, generic `.field` controls, config textareas/selects, and include a small CSS regression test for those selectors.
