## Summary

Keep open. Current main still preserves Telegram default tool-progress delivery in non-streaming mode, and an open, unmerged closing PR (#72482) is the active implementation candidate. The later comment also broadens the reasoning leak from Telegram to WebSocket/chat API behavior, so closing this as already fixed would be premature.

## Best Possible Solution

Keep this issue open while maintainers review PR #72482 and decide whether to split the broader reasoning transport/API gap into #41494, #54496, or a new canonical issue. The concrete fix should suppress default tool progress for Telegram non-streaming turns, update the test currently asserting the opposite, and add provider/chat-path coverage proving structured reasoning is hidden or separated before final delivery.
