## Summary

Keep #11655 open. Current main still only enables the Slack App Home messages tab in the manifest; it does not subscribe to `app_home_opened`, register a Home-tab monitor handler, call Slack `views.publish`, expose a Home-tab action/config surface, or document Home-tab publishing setup. The related implementation PRs were closed unmerged, and duplicate #52020 was closed back to #11655 as the canonical tracker.

## Best Possible Solution

Keep #11655 open as the canonical bundled Slack App Home tracker. Implement it in `extensions/slack` by adding `app_home_opened` to setup/docs, registering a monitor handler under `extensions/slack/src/monitor/events`, publishing a safe default Home view with Slack `views.publish`, deciding the supported customization/config or tool action path, and adding focused tests for event filtering, payload shape, error handling, manifest/docs coverage, and action/config behavior.
