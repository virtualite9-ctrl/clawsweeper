## Summary

Keep open. Current main contains the #69278 heartbeat prompt-policy fix for #69079, but this issue is a detailed post-fix report that the fresh-session `[object Object]` hallucination still occurred after installing that fix branch and wiping sessions/task DB. Recent main also added adjacent QMD memory scoping/startup changes, but they are not tied to this report and do not prove the reporter’s provider/translator-boundary symptom is fixed.

## Best Possible Solution

Keep this open as a current-main regression investigation. Reproduce on current main or the latest release with the reporter’s provider boundary captured, add a redacted full composed prompt/request diagnostic for one run, and then fix the confirmed source. If task summaries are implicated, add a guarded write-boundary diagnostic/rejection in `src/tasks/task-registry.ts` with focused coverage; if the custom translator is implicated, document or harden the provider payload boundary.
