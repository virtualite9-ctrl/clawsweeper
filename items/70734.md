## Summary

Keep open. Current main has the #69278 heartbeat trigger-policy fix, but #70734 specifically reports that the fresh-session `[object Object]` hallucination persisted on commit f8b7862 after HEARTBEAT_OK leakage was already gone. I did not find a later source change that conclusively fixes the fresh-session injection or custom translator/provider-boundary path.

## Best Possible Solution

Keep #70734 open as the remaining fresh-session prompt/provider-boundary contamination investigation. Ask for or collect a current-main `/export-trajectory` bundle plus a translator before/after payload capture for the first `hello` turn; inspect bootstrap/context-engine/memory prompt additions and the translator conversion path. If OpenClaw sends a clean provider payload, redirect or close as external translator behavior; if not, fix the implicated prompt composition path and consider adding a diagnostic guard around `normalizeTaskSummary` for literal `[object Object]` contamination.
