## Summary

Issue #64048 was reported against OpenClaw 2026.4.5, but current main already includes shared Ollama native-chat fixes that match the reported Control UI freeze path: native `/api/chat` requests now receive top-level `think` control for Ollama/Qwen models, session-model switches remap unsupported stored thinking levels, and `chat.send` runs are registered so lifecycle errors surface back to Control UI instead of leaving chat stuck waiting.

## Evidence

- **Ollama native chat now forwards top-level think control:** Current main wraps native Ollama transport requests with top-level `think=false` when thinking is off and `think=true` for non-off thinking levels, matching the qwen/llama freeze-idle class described in the issue.
