## Summary

Keep open. The reported false-positive capability tags still reproduce on the live ClawHub skill page for `earlvanze/renatus-icm`, and current `main` does not show an in-repo fix for that ClawHub-side classification behavior.

## Evidence

- **Live ClawHub page still shows the exact false-positive tags:** Direct page fetch for `https://clawhub.ai/earlvanze/renatus-icm` still renders `Capability signals` with `Crypto`, `Can make purchases`, and `Requires OAuth token`, plus an OpenClaw `Suspicious` verdict, matching the issue report.
  - command: `curl -fsSL 'https://clawhub.ai/earlvanze/renatus-icm' | rg -n "Crypto|Can make purchases|Requires OAuth|Suspicious|signal|verdict" -n -C 2`
- **Live ClawHub API confirms this is an active moderated skill entry:** The live skill API returns `renatus-icm` with `latestVersion.version` `2.1.4` and `moderation.verdict` `suspicious`, so this is not an obsolete or deleted listing.
  - command: `curl -fsSL 'https://clawhub.ai/api/v1/skills/renatus-icm'`
- **Repo docs place capability/tag discovery on ClawHub:** The ClawHub docs describe ClawHub as the public registry and discovery surface for skills, tags, and usage signals, and say it uses metadata to safely expose skill capabilities.
  - file: [docs/tools/clawhub.md:55](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/docs/tools/clawhub.md#L55)
  - sha: [6bc0dc8fb671](https://github.com/openclaw/openclaw/commit/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440)
- **Current main ClawHub skill detail type does not include a fixed capability-signal model:** `ClawHubSkillDetail` in current `main` only models basic skill metadata, version, platform metadata, and owner fields; it does not show any landed fix for capability-tag heuristics or owner overrides.
  - file: [src/infra/clawhub.ts:123](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/src/infra/clawhub.ts#L123)
  - sha: [6bc0dc8fb671](https://github.com/openclaw/openclaw/commit/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440)
- **Current main UI does not implement any corrective handling for those tags:** The ClawHub detail dialog on current `main` renders summary, owner, version, changelog, and platforms only; there is no in-repo logic here that would correct or suppress false-positive capability tags from ClawHub moderation.
  - file: [ui/src/ui/views/skills.ts:332](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/ui/src/ui/views/skills.ts#L332)
