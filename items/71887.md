## Summary

Keep #71887 open. Current main still has the stale `status --deep` wording in `openclaw channels status` CLI/help output, while `channels status` only registers `--probe`, `--timeout`, and `--json`; the top-level `openclaw status --deep` is the command that owns `--deep`. The PR remains directionally useful, but likely needs an update for current main because the config-only formatter now lives in a separate file.

## Best Possible Solution

Keep the PR open and ask for a small rebase/update against current main: clarify `src/cli/channels-cli.ts`, both channel status output paths (`src/commands/channels/status.ts` and `src/commands/channels/status-config-format.ts`), and keep/add the focused regression assertion that `channels status` output does not suggest `channels status --deep`. The docs should continue pointing users to the public top-level `openclaw status --deep` guidance at `https://docs.openclaw.ai/cli/status`.
