## Summary

Keep PR #71887 open. Current main still has stale/ambiguous `status --deep` guidance in active `channels status` CLI help and output, while the subcommand still only defines `--probe`, `--timeout`, and `--json`. The PR addresses a real remaining bug, but it likely needs a rebase/update because current main also has the config-only formatter split into `src/commands/channels/status-config-format.ts` with the same stale tip.

## Best Possible Solution

Keep the PR open and ask for a current-main rebase/amend that updates all active channel status help/output paths: `src/cli/channels-cli.ts`, `src/commands/channels/status.ts`, `src/commands/channels/status-config-format.ts`, and the focused channel status regression tests/docs. The product direction is correct: `channels status` should document only its own flags, while broader probes should explicitly point to top-level `openclaw status --deep`.
