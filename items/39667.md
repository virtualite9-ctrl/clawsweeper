## Summary

Issue #39667 is already implemented on current main and in the latest shipped release. The current Control UI signs the device-auth payload with the same resolved auth token it sends to the gateway, keeps cached device tokens separate for retry/fallback paths, persists shared auth in session-scoped storage for refreshes, and has regression tests covering both the shared-token signing path and the bounded device-token retry path.
