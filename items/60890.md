## Summary

Keep PR #60890 open. Current main still does not implement the PR's central user-facing behavior: `openclaw cron add` and `openclaw cron edit` do not expose `--thread-id`, do not compose Telegram topic targets from a dedicated flag, and `cron edit` still resolves existing job state with a single unpaged `cron.list` call in the schedule patch path. The provided PR discussion and related issue #50982 describe a real missing CLI feature, not obsolete work; earlier PRs #51581 and #60373 were closed unmerged, while #60890 is the more complete follow-up with tests and validation refinements.

## Best Possible Solution

Keep the PR or a maintainer-owned successor open until `cron add` and `cron edit` expose a dedicated `--thread-id <id>` path that composes Telegram topic targets safely, validates invalid combinations, updates focused CLI docs, and includes regression tests for add/edit topic delivery plus paged `cron edit` lookup. If maintainers prefer not to take this exact PR, close it only after opening or landing a canonical replacement for #50982/#60890.
