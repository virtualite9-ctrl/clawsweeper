## Summary

Keep open. Current `main` does not implement `openclaw cron add/edit --thread-id`, and `cron edit` still does a single-page `cron.list` lookup instead of the paged/stable lookup described in the PR. The docs show a manual `:topic:` target workaround, and cron’s data model already supports `delivery.threadId`, so the PR is still addressing a real gap rather than obsolete behavior.
