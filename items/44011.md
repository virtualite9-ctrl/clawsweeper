## Summary

Keep this PR open. Current main partially exposes `sessionKey` on typed message hooks, but it still does not expose `agentId` on `PluginHookMessageContext`, does not derive/pass it for inbound `message_received`, and does not thread session or mirror identity into outbound public `message_sending` / `message_sent` contexts. The PR remains a focused implementation candidate, with rebase, docs, and the requested fallback test still needed.

## Best Possible Solution

Keep this PR open as a valid implementation candidate. Rebase it onto current main, adapt the type change to `src/plugins/hook-message.types.ts`, add optional `agentId` to the message hook context and canonical mapper types, derive inbound `agentId` from the resolved session key, resolve outbound hook context once with mirror identity preferred over direct session identity, pass that same context to both `message_sending` and `message_sent`, add no-session/session-only/mirror-wins tests for success and failure paths, and document `ctx.agentId` on https://docs.openclaw.ai/plugins/hooks.
