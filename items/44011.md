## Summary

Keep this PR open. Current main still does not expose `agentId` on typed message hook context, does not derive it for `message_received`, and does not thread session or mirror identity into outbound typed `message_sending` / `message_sent` contexts. The PR remains a valid implementation candidate, with rebase, docs, and one requested fallback test still needed.

## Best Possible Solution

Keep this PR open as a valid implementation candidate. Rebase it onto current main, add optional `agentId` to the current message hook context and canonical mapper types, derive inbound `agentId` from the resolved session key, resolve outbound hook context once with mirror identity preferred over direct session identity, thread `sessionKey` and `agentId` into both `message_sending` and `message_sent`, add session-only and mirror-wins tests, and document `ctx.agentId` in the public plugin hook docs.
