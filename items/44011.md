## Summary

Keep this PR open. Current main still only partially covers the requested typed message-hook session context: `ctx.sessionKey` exists in the public message hook context type, but `ctx.agentId` is still absent, inbound `message_received` does not pass the resolved agent identity into the typed hook context, and outbound `message_sending` / `message_sent` still do not expose session or mirror identity through the public typed context. The PR remains a focused implementation candidate, but needs a rebase/adaptation plus the existing fallback coverage gap addressed before landing.

## Best Possible Solution

Keep this PR open as a valid implementation candidate. Rebase it onto current main, add optional `agentId` to the typed message hook context and canonical mapper types, make inbound hooks expose the resolved session/agent identity consistently, resolve outbound hook context once with mirror identity preferred over direct session identity, pass that same context to `message_sending` and `message_sent`, add no-session/session-only/mirror-wins tests for success and failure paths, and document `ctx.agentId` on https://docs.openclaw.ai/plugins/hooks.
