## Summary

Keep this PR open. Current main partially covers the area by exposing `sessionKey` on typed message hook contexts, but it does not expose `agentId`, and the outbound `message_sending` / `message_sent` paths still do not thread session identity into the typed plugin context. The related open PR #44348 is complementary for agent lifecycle hooks, not a replacement for message hooks.

## Best Possible Solution

Keep this PR open as a valid implementation candidate. The best path is to update it against current main by adding optional `agentId` to `PluginHookMessageContext` and the canonical message hook contexts, deriving/passing inbound `agentId` from the resolved session, resolving outbound mirror/session identity once, threading `sessionKey` and `agentId` into both `message_sending` and `message_sent`, adding session-only and mirror-wins typed hook tests, and updating `docs/plugins/hooks.md` to document the new field.
