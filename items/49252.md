## Summary

Current `main` already addresses this race in Control UI. The chat client now keeps the optimistic user message visible, defers `session.message` history reloads while a run is active to avoid wiping that message mid-run, and exposes queued follow-up prompts in the UI with steer/remove controls.

## Evidence

- **Deferred mid-run reload avoids the exact optimistic-message race:** `ui/src/ui/app-gateway.ts:490` documents and implements the fix: when `session.message` arrives during an active run, history reload is deferred because reloading mid-run races with the optimistic user-message update and delays the user message card.
  - file: [ui/src/ui/app-gateway.ts:490](https://github.com/openclaw/openclaw/blob/8ba22ca0dc889f5f1959f2f7533961f89524831a/ui/src/ui/app-gateway.ts#L490)

  - sha: [8ba22ca0dc88](https://github.com/openclaw/openclaw/commit/8ba22ca0dc889f5f1959f2f7533961f89524831a)
