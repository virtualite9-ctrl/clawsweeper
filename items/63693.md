## Summary

Current main fixes the reported false `plugins.allow` status warning by removing the cold plugin capability/provider load from memory-search config resolution. The linked cfg-threading PR #63848 was a useful diagnosis, but main now uses the registered-only memory embedding provider registry on this hot path instead.

## Best Possible Solution

Close #63693 as implemented on main. Keep the no-cold-load memory-search config implementation; treat #63848 as superseded by the current registered-only lookup rather than merging the older cfg-threading patch.
