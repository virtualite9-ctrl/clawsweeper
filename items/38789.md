## Summary

Keep PR #38789 open. Current main still does not rebuild sandbox skill prompts from live sandbox workspace entries: resolved snapshots skip live skill loading, and `resolveSkillsPromptForRun` still returns the persisted snapshot prompt before considering entries. The PR targets a real remaining agents bug, although the existing PR should not land unchanged because its review comments identify gaps around empty sandbox skill sets, preserving `skillFilter`, and keeping env overrides aligned with the rebuilt prompt.

## Best Possible Solution

Keep this PR open or replace it with a focused agents fix that rebuilds non-rw sandbox skill prompts from the same sandbox-aware, filter-preserving skill set used for runtime setup in both attempt and compaction paths. The final fix should avoid returning host snapshot prompts for empty sandbox skill sets, preserve remote eligibility and saved skill filters, keep env overrides aligned with the advertised skill set, and add regression tests for snapshot-present non-rw sandbox attempt and compaction behavior.
