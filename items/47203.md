## Summary

Current main still appears to match the reported failure mode. The browser navigate client and CLI keep a 20s request timeout, the /navigate route does not forward body timeout/loadState options into Playwright navigation, and Playwright still calls page.goto without waitUntil so it defaults to the load event. The related reports in the provided context are closed/stale or cover adjacent timeout/error-clarity paths, not a shipped fix for this Linux VNC navigate race.

## Best Possible Solution

Keep this issue open. A targeted fix should make browser navigate use the same configurable/slack timeout model as other browser actions, forward a bounded per-call timeout through browser.request and /navigate into navigateViaPlaywright, and either expose or default a safer waitUntil/loadState such as domcontentloaded or commit while preserving SSRF navigation guards. Add regression coverage at the client, route, and Playwright wrapper layers so the 20s transport timeout cannot race the navigation timeout again.
