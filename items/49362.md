## Summary

Current main already handles this report: `tools.profile: "coding"` still declares `apply_patch` and `image` as baseline profile entries, but runtime filtering now suppresses false "unknown entries" warnings for baseline gated core tools and keeps warnings only for real extra/typo allowlist entries.
