## Summary

Keep open. Current main still lacks both observable fixes from this PR: `models auth paste-token` has no `--token` option and still always prompts interactively, and invalid auth-profile load warnings do not include the accepted credential types. The linked issue is closed as superseded by this PR, so this PR remains the canonical implementation candidate rather than something to close now.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land this narrow change, or replace it with an equivalent maintainer patch, after adding focused tests for CLI forwarding, non-interactive token validation, empty-token failure, and invalid-type warning diagnostics, plus a small docs update for the new `paste-token --token` usage.
