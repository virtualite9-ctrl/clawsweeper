## Summary

Current main already covers this request: bare `/new` and `/reset` runs inject an explicit session-start instruction and optional startup-context prelude, workspace bootstrap files are auto-loaded every session, and gateway restarts can automatically run `BOOT.md` via the shipped `boot-md` startup hook. The same capability is also present in release `v2026.4.22`.

## Risks / Open Questions

- The request asked for a single config option, while the shipped solution is split across built-in session-start behavior, `agents.defaults.startupContext`, workspace bootstrap files, and hooks/`BOOT.md`.

## Close Comment

Closing as implemented after a Codex review. Current OpenClaw already has the requested startup/reboot bootstrap paths: bare `/new` and `/reset` runs inject an explicit Session Startup instruction (`src/auto-reply/reply/session-reset-prompt.ts:11`) and can prepend runtime-owned startup context for recent daily memory (`src/auto-reply/reply/get-reply-run.ts:376`, `docs/gateway/config-agents.md:142`). Workspace bootstrap files such as `AGENTS.md` are already loaded every session (`docs/concepts/agent-workspace.md:66`). For gateway restarts, the bundled `boot-md` hook listens to `gateway:startup` and runs `BOOT.md` through the agent runner (`src/hooks/bundled/boot-md/handler.ts:11`, `src/gateway/boot.ts:43`). OpenClaw also already exposes hook events for `command:new`, `command:reset`, and `gateway:startup` if you want custom automation beyond the built-ins (`docs/automation/hooks.md:34`). I also verified the same implementation is present in release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`).
