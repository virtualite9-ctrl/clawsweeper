## Summary

Keep PR #69696 open. Current main still performs case-sensitive OpenRouter base URL matching in the OpenRouter, Arcee, and embedded-runner normalization helpers, and the existing tests only cover lowercase stale `/v1` inputs. The PR is a small, directly relevant follow-up to the already-merged #68574 root fix rather than cleanup material.

## Best Possible Solution

Keep the PR open and have the author or a maintainer update it against current main. The implementation should preserve the proposed comparison-only lowercasing in the three existing helpers, keep custom proxy URLs untouched, and ideally add matching Arcee and embedded-runner regression coverage for mixed-case OpenRouter base URLs.
