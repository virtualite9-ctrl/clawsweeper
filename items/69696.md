## Summary

Keep open. Current `main` still handles OpenRouter base URL normalization case-sensitively across the OpenRouter, Arcee, and PI embedded runner paths, and current tests only cover lowercase legacy `/v1` inputs. The PR addresses a real gap that is not already implemented on `main`.

## Evidence

- **OpenRouter catalog helper is still case-sensitive on main:** `normalizeBaseUrl` only trims and removes trailing slashes, then `normalizeOpenRouterBaseUrl` compares against lowercase constants. An uppercase host like `https://OPENROUTER.AI/v1` would not match and would return `undefined`.
