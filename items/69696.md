## Summary

Keep PR #69696 open. Current main includes the earlier lowercase `/v1` to `/api/v1` OpenRouter/Arcee self-heal from #68574, but it does not implement this PR's case-insensitive URL matching or the added uppercase/custom-proxy regression coverage. The supplied Greptile review also supports the PR as a small, targeted follow-up rather than obsolete work.

## Best Possible Solution

Keep #69696 open and merge or port its narrow fix: lowercase only the comparison-normalized OpenRouter base URL in the OpenRouter, Arcee, and embedded-runner helpers, while preserving custom proxy passthrough and adding the proposed regression tests. #68574 solved the main stale lowercase `/v1` bug, but it does not supersede this case-insensitive edge-case hardening.
