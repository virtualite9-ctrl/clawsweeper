## Summary

Keep PR #69696 open. Current main still performs case-sensitive OpenRouter base URL matching in the OpenRouter, Arcee, and embedded-runner normalization helpers, while existing regression coverage only locks in lowercase stale `/v1` inputs. The PR is a focused follow-up to the already-merged stale-base-url fix rather than conservative cleanup material.

## Best Possible Solution

Keep the PR open and have the author or a maintainer update it against current main. The best implementation is the proposed comparison-only lowercasing in the three existing helpers, preserving custom proxy passthrough, with regression coverage for mixed-case OpenRouter base URLs across the touched provider and embedded-runner paths.
