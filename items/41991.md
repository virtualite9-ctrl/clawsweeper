## Summary

Keep open. Current main still collapses Gemini CLI OAuth credential extraction failures into a generic "Gemini CLI not found" error, while this PR proposes useful diagnostic detail. The PR does need to be rebased/ported because the old `extensions/google-gemini-cli-auth` surface has since been consolidated into the bundled `google` plugin.

## Best Possible Solution

Keep this PR open as a useful implementation candidate, but ask for a rebase/port onto `extensions/google/oauth.credentials.ts` plus focused regression tests that assert the thrown `resolveOAuthClientConfig()` message includes the specific extraction failure detail. If maintainers choose to remove or gate Gemini CLI credential extraction through #54289, fold this diagnostic work into that decision instead of merging it independently.
