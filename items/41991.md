## Summary

Keep this PR open. Current main still has the underlying diagnostic gap, but the PR targets the removed `extensions/google-gemini-cli-auth` path and should be ported to the current bundled Google plugin. The open security discussion in #54289 should also guide whether diagnostics are added or the automatic extraction path is removed/gated instead.

## Best Possible Solution

Keep this PR open as a focused implementation candidate, but ask the contributor to rebase and port the diagnostic state/error-message changes to `extensions/google/oauth.credentials.ts`. Add regression tests around `resolveOAuthClientConfig()` for missing binary, missing oauth2.js, parse failure, and extraction exceptions. Coordinate final behavior with #54289 so the result either adds safe diagnostics to the retained extraction path or is closed as superseded if automatic extraction is removed or gated.
