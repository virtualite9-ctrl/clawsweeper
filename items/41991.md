Keep this PR open. Current main still has the diagnostic gap the PR is trying to address, but the patch targets the removed `extensions/google-gemini-cli-auth` plugin path instead of the active bundled Google plugin. It should be ported and tested against `extensions/google/oauth.credentials.ts`, with final behavior coordinated with the related security discussion in #54289.

Ask the contributor to rebase and port the diagnostic state/error-message changes to `extensions/google/oauth.credentials.ts`, add focused regression tests around `resolveOAuthClientConfig()` for missing binary, missing `oauth2.js`, parse failure, and extraction exceptions, then resolve the final behavior in line with #54289.
