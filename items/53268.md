## Summary

Keep this issue open. Current main still does not provide a configurable URL or hostname deny list for `web_fetch`; the code only supports existing fetch limits plus one SSRF override, and the shared SSRF guard has hardcoded internal-host blocking plus allowlist support, not selective public-domain denial.

## Best Possible Solution

Keep this issue open and implement a core `web_fetch` hostname deny-list if maintainers want the behavior. A conservative path is to add a typed config key such as `tools.web.fetch.denyHosts`, validate it in strict runtime and generated schemas, document it on the public Web Fetch page, enforce exact plus `*.domain` matching before DNS and on redirects, and add focused config/runtime tests for accepted and rejected hosts plus concise blocked-fetch errors.
