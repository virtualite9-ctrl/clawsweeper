## Summary

Current main still does not implement a configurable URL or hostname deny list for web_fetch. The typed config, runtime schema, generated schema/help, runtime wiring, docs, and SSRF tests all show only existing private/internal host blocking plus allowlist-oriented SSRF controls. There is no exact or wildcard public-host deny policy such as tools.web.fetch.denyHosts, and the provided issue context shows no protected labels or linked implementation PR that would make this closeable.

## Best Possible Solution

Keep this issue open. If maintainers want the feature, add a typed and documented tools.web.fetch.denyHosts-style option, validate exact and *.domain patterns case-insensitively, enforce it before DNS and across redirects/provider fallback, return concise blocked-fetch errors, update generated schema/help/docs, and add focused config plus runtime tests.
