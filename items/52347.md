## Summary

Current `main` does not use the reported `models/embedding-001` path. The Gemini memory embedding adapter defaults to `gemini-embedding-001`, normalizes to `models/gemini-embedding-001`, and the official Gemini API docs still list `gemini-embedding-001` as the stable embeddings model while deprecating the older `embedding-001`. That makes the report's stated root cause obsolete against current `main`. ([ai.google.dev](https://ai.google.dev/gemini-api/docs/embeddings?utm_source=openai))
