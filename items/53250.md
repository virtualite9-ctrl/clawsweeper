## Summary

Current main still has the generic exec-approval unavailable fallback and does not include the requested Control UI bootstrap command, approvals inspection command, or concrete setup hints in the shared timeout/unavailable copy. The directly linked implementation PR #53347 was closed unmerged, so there is no shipped fix to close against.

## Best Possible Solution

Keep this open. Apply a small core UX change in `src/infra/exec-approval-reply.ts` that preserves channel-native setup guidance while adding concrete recovery steps for `openclaw dashboard --no-open`, `openclaw approvals get`, and the relevant `channels.<channel>.execApprovals.enabled` setup path. Add focused assertions in `src/infra/exec-approval-reply.test.ts`; keep token-bearing URLs out of chat output.
