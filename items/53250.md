## Summary

Current main still has the generic exec-approval unavailable fallback and does not include the requested Control UI bootstrap or CLI inspection hints. The linked implementation PR #53347 was closed unmerged, so the remaining UX work is still actionable in core.

## Best Possible Solution

Keep this open. Implement a small core UX change in `src/infra/exec-approval-reply.ts` that preserves channel-specific native approval setup text while adding concrete recovery steps for `openclaw dashboard --no-open`, `openclaw approvals get`, and the relevant `channels.<channel>.execApprovals.enabled` path. Add focused coverage in `src/infra/exec-approval-reply.test.ts`.
