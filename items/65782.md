## Summary

Keep this issue open. Current main still has the reported fixed-width coarse UTF-16 split in `chunkMarkdown`, the embedding path still passes raw text/structured inputs to providers without a lone-surrogate sanitizer, and the provided GitHub context shows open PR #65783 as the linked implementation candidate with closing syntax for this issue.

## Best Possible Solution

Keep this issue open and review PR #65783 as the implementation candidate. The likely fix remains the two-part memory change described by the reporter: adjust the outer `chunkMarkdown` coarse split so it cannot cut between UTF-16 surrogate halves, add a defensive embed-boundary sanitizer for lone surrogates in memory-core, and land focused regression tests for both the coarse emoji boundary and sanitizer behavior.

## Evidence
