## Summary

Keep #13175 open. Current main still does not implement generic Feishu inline/native card buttons for shared presentation payloads. Feishu can send fallback presentation cards and has useful structured card-action plumbing for approval/quick-action flows, but presentation button blocks still render as markdown labels rather than Feishu clickable action buttons, and the Feishu outbound adapter still lacks generic `renderPresentation`/`sendPayload` support. Related PR #47891 is narrower `channelData.feishu.card`/`sendPayload` work and does not supersede this broader inline-button request.

## Best Possible Solution

Keep #13175 open. Implement Feishu-owned native rendering for shared `MessagePresentation` buttons: add runtime outbound `presentationCapabilities` and `renderPresentation` or an equivalent delivery path, render Feishu Schema 2.0 `action`/`button` elements from button blocks including URL fallback semantics, wrap callback values in structured Feishu interaction envelopes, route clicks through `handleFeishuCardAction`, and add tests for clickable buttons, URL buttons, fallback behavior, and callback dispatch. Treat #47891 as related narrower `sendPayload`/native-card payload work unless maintainers explicitly broaden it to cover this issue.
