## Summary

Keep #13175 open. Current main still does not implement generic Feishu inline/native interactive card buttons for shared message presentations: Feishu presentation buttons are rendered as markdown fallback text, the outbound adapter exposes only text/media delivery, and the existing card-action path handles structured Feishu-specific envelopes rather than generating generic `[label](btn:action)`/presentation button callbacks. The prior issue comment's related PR #47891 remains narrower sendPayload/native-card payload work in the provided context, not a shipped fix for this feature.

## Best Possible Solution

Keep #13175 open. Implement Feishu-owned native rendering for shared presentation buttons: add Feishu outbound presentationCapabilities plus renderPresentation or equivalent sendPayload support, render Schema 2.0 action/button elements for value and URL buttons, wrap callback values in structured Feishu interaction envelopes, route clicks through handleFeishuCardAction, preserve fallback text for unsupported surfaces, and add tests for clickable buttons, URL fallback, unsupported fallback, and callback dispatch. Treat PR #47891 as related narrower sendPayload/native-card payload work unless maintainers explicitly broaden it.
