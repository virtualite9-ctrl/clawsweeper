## Summary

Keep #20230 open. Current main has shipped active assistant name/avatar display and distinct user-vs-assistant bubble styling, but the central remaining request is still missing: there is no per-agent chat bubble color contract, no assistant identity/style stored per message or group, and mixed-agent assistant messages still render through one active assistant identity.

## Best Possible Solution

Keep the issue open until maintainers either implement or explicitly reject per-agent chat styling and per-message assistant attribution. A complete fix likely belongs in the Control UI plus shared chat/session contracts: persist assistant identity on messages or message groups, add a validated optional color/styling field or render hook, render assistant-specific CSS variables/classes, and add focused tests for mixed-agent and historical-session attribution.
