## Summary

Keep #20230 open. Current main already uses a resolved active assistant name and distinct user-vs-assistant bubble styling, but it still does not provide per-agent configurable bubble colors or persisted per-message/per-agent assistant attribution for mixed or historical chat views. The only issue comment also kept this open for the same remaining scope.

## Best Possible Solution

Keep the issue open until maintainers either implement or explicitly reject a real per-agent chat styling and attribution surface. A complete implementation likely belongs in Control UI plus the shared chat/session metadata contract: persisted assistant agent identity per message/group, validated optional styling fields in config or protocol, rendering/CSS hooks for assistant-specific colors, and focused tests/docs for mixed-agent and historical-session attribution.
