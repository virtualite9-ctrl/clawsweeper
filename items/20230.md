## Summary

Keep #20230 open. Current main has partial coverage for the request: active assistant name/avatar is resolved and rendered, and user bubbles have distinct styling. The missing core behavior is still meaningful: there is no per-agent configurable bubble color contract, and assistant chat groups are rendered from one active assistant identity rather than persisted per-message/per-agent attribution.

## Best Possible Solution

Keep the issue open until maintainers implement or explicitly reject per-agent chat styling and assistant attribution. A complete solution likely belongs across Control UI and the shared chat/session metadata contract: persist assistant agent identity per message or group, add a validated optional color/styling field or render hook, render assistant-specific CSS variables/classes, and add focused tests/docs for mixed-agent and historical-session attribution.
