## Summary

Keep open. Current main partially satisfies the request by resolving and rendering the active assistant name and by styling user bubbles differently from assistant bubbles, but the broader requested behavior remains missing: per-agent configurable chat bubble colors and per-message/per-agent assistant attribution in mixed or historical chat views.

## Best Possible Solution

Keep #20230 open until maintainers either implement a real per-agent chat style/attribution surface or explicitly narrow the product direction. A complete fix likely needs message/group metadata for assistant agent identity, a validated config/protocol field for optional per-agent chat styling, WebUI rendering/CSS hooks, and docs/tests covering mixed-agent or historical session attribution.
