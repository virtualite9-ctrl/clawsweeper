## Summary

Keep open. Current `main` still lacks the PR's explicit error for unresolved workflow-like file paths, while the PR as submitted has review-identified regressions in inline pipeline handling. The `approvalId` plumbing from the first commit is already on `main`, so only the workflow-path portion remains useful and it needs revision rather than closure.

## Evidence

- **Current main still falls back to inline execution for unresolved workflow-like paths:** `detectWorkflowFile` returns `null` on any resolution failure, and `run()` immediately falls through to `runtime.runToolRequest({ pipeline, ctx })` when that happens. That means missing `*.lobster`/`*.yaml`/`*.json` paths still get treated as inline pipelines instead of producing the PR's proposed path-specific error.
