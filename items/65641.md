## Summary

Keep open. Current main has partial related gateway-auth cleanup and generic close hints, but it still does not implement the main RPC diagnostic behavior this PR targets: normal `callGateway` requests do not capture `GatewayClient.onConnectError` and include it in the final close error. The PR remains useful, but it needs revision because its broad loopback device-identity omission can regress paired local sessions that rely on cached device-token auth.

## Best Possible Solution

Keep this PR open for revision. The best fix is to preserve current main's narrow backend/shared-token loopback identity omission, decide and test the real `GatewayClient` explicit-null contract, capture `GatewayClient.onConnectError` inside `executeGatewayRequestWithScopes`, include that error in `formatGatewayCloseError`, and add focused regression tests for both real-constructor null handling and pre-handshake connect-error propagation through normal `callGateway` RPCs.
