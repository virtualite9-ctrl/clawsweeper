## Summary

Current `main` already implements a configurable rate-limit failover threshold. OpenClaw exposes it as `auth.cooldowns.rateLimitedProfileRotations`, enforces it in the embedded runner, documents the behavior, covers it with regression tests, and records it as shipped in the 2026.4.1 changelog.

## Close Comment

Closing as implemented. A Codex review of current `main` found that OpenClaw already supports this behavior via `auth.cooldowns.rateLimitedProfileRotations`, which controls how many same-provider rate-limit rotations happen before model fallback. The runtime enforcement is in `src/agents/pi-embedded-runner/run.ts:605`, the config surface is in `src/config/types.auth.ts:48`, the behavior is documented in `docs/concepts/model-failover.md:219` and `docs/gateway/configuration-reference.md:733`, and regression coverage exists in `src/agents/model-fallback.run-embedded.e2e.test.ts:801`. The changelog records this as shipped in `2026.4.1` at `CHANGELOG.md:1644`. If the intent was specifically to add a different config key name, that part was not adopted, but the requested capability is already present.
