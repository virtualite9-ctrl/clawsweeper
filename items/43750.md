## Summary

Keep this issue open. Current main has improved Discord native-command routing through configured bindings and CommandTargetSessionKey, but it does not implement the requested explicit `/new @AgentName` or `/reset @AgentName` targeting, and the related #58110 is a broader open UI/multi-agent proposal rather than a canonical fix for this command-routing bug.

## Best Possible Solution

Keep the issue open and implement a focused Discord/native-command enhancement: add a supported target-agent argument or mention parser for `/new` and `/reset`, resolve it through existing agent/binding authorization rules, reset only that target session, and add regression tests for shared guild channels, configured bindings, bound ACP sessions, and unauthorized target attempts. If maintainers decide smart defaults are desirable, track that separately from explicit targeting.
