## Summary

Keep this issue open. Current main routes Discord native commands through the effective channel/binding target and has native-command channel policy checks, but it still does not implement explicit `/new @AgentName` or `/reset @AgentName` targeting. The linked #58110 is broader multi-agent UI work and does not supersede this narrow Discord command behavior.

## Best Possible Solution

Keep this open and implement the fix in the bundled Discord native-command/session-reset path: add a supported optional target-agent option or mention/name parser for `/new` and `/reset`, resolve it through configured agents, Discord routing, bindings, thread bindings, and authorization/channel policy, set `CommandTargetSessionKey` only for the selected session, and add regression coverage for shared guild channels, bound sessions, blocked/disabled channels, unauthorized targets, and the no-target default behavior.
