## Summary

Keep this issue open. Current main still routes Discord native commands through the effective route, configured bindings, and CommandTargetSessionKey, but it does not implement explicit `/new @AgentName` or `/reset @AgentName` targeting for shared multi-agent Discord channels.

## Best Possible Solution

Keep the issue open and implement the fix in the bundled Discord plugin/native-command path: add a supported optional target-agent option or mention/name parser for `/new` and `/reset`, resolve it through existing Discord routing, configured ACP bindings, thread bindings, and authorization rules, set CommandTargetSessionKey only for the selected agent/session, and add regression coverage for shared guild channels, bound sessions, blocked/disabled channels, unauthorized target attempts, and the no-target default behavior.
