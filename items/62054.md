## Summary

Current `main` no longer treats completion announce retry exhaustion as terminal for a busy parent session: completion delivery now falls back from direct send to queued follow-up delivery, and failed cleanup announces remain persisted for retry on registry resume instead of being silently forgotten.

## Evidence

- **Completion announces now fall back to queue after direct failure:** `runSubagentAnnounceDispatch()` uses direct-first ordering for completion messages, but if the direct path fails it immediately runs `queue-fallback` and returns that result when queued delivery succeeds.
  - file: [src/agents/subagent-announce-dispatch.ts:88](https://github.com/openclaw/openclaw/blob/e6d04682d33c9576edc4ad77b031949f84407bec/src/agents/subagent-announce-dispatch.ts#L88)
  - sha: [e6d04682d33c](https://github.com/openclaw/openclaw/commit/e6d04682d33c9576edc4ad77b031949f84407bec)
- **Busy requester sessions are explicitly queued instead of relying only on direct delivery:** `maybeQueueSubagentAnnounce()` queues follow-up/collect/interrupt/steer-backlog announces when the requester session is active, which is the busy-parent case described in the report.
  - file: [src/agents/subagent-announce-delivery.ts:411](https://github.com/openclaw/openclaw/blob/e6d04682d33c9576edc4ad77b031949f84407bec/src/agents/subagent-announce-delivery.ts#L411)
  - sha: [e6d04682d33c](https://github.com/openclaw/openclaw/commit/e6d04682d33c9576edc4ad77b031949f84407bec)
- **Regression test covers active-parent completion fallback:** The e2e regression test simulates an active requester session, forces the first completion delivery attempt to fail, and verifies `runSubagentAnnounceFlow()` still reports success via queued follow-up delivery.
  - file: [src/agents/subagent-announce.format.e2e.test.ts:1716](https://github.com/openclaw/openclaw/blob/e6d04682d33c9576edc4ad77b031949f84407bec/src/agents/subagent-announce.format.e2e.test.ts#L1716)
  - sha: [e6d04682d33c](https://github.com/openclaw/openclaw/commit/e6d04682d33c9576edc4ad77b031949f84407bec)
- **Failed announce cleanup is persisted and retried after restart:** Persistence tests verify that when announce cleanup fails, the run stays retryable on disk (`cleanupHandled === false`) and is retried successfully after registry restart instead of being silently dropped.
  - file: [src/agents/subagent-registry.persistence.test.ts:396](https://github.com/openclaw/openclaw/blob/e6d04682d33c9576edc4ad77b031949f84407bec/src/agents/subagent-registry.persistence.test.ts#L396)
  - sha: [e6d04682d33c](https://github.com/openclaw/openclaw/commit/e6d04682d33c9576edc4ad77b031949f84407bec)
- **Current tests still exercise transient 120s timeout retries:** Timeout coverage verifies completion direct agent calls retry repeated `gateway timeout after 120000ms` failures before returning, which matches the original symptom path and shows the current code is testing that path explicitly.
  - file: [src/agents/subagent-announce.timeout.test.ts:316](https://github.com/openclaw/openclaw/blob/e6d04682d33c9576edc4ad77b031949f84407bec/src/agents/subagent-announce.timeout.test.ts#L316)
  - sha: [e6d04682d33c](https://github.com/openclaw/openclaw/commit/e6d04682d33c9576edc4ad77b031949f84407bec)
