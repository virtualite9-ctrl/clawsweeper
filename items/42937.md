## Summary

Keep this PR open. Current main still supports global `session.dmScope` and route-binding `bindings[].session.dmScope`, but it does not accept or apply the PR's requested `agents.list[].dmScope` per-agent default. The PR remains a plausible core routing/config change, but it needs a focused rebase against the newer binding-level override surface with an explicit precedence rule.

## Best Possible Solution

Keep this PR open and ask for a focused rebase/rework. The best implementation would add `AgentConfig.dmScope`, strict zod validation, help/label/generated schema/docs updates, and selected-agent lookup in route resolution while preserving binding-level overrides with explicit precedence. Add tests for per-agent override, global fallback, binding interaction, and route-cache behavior, plus documentation that calls out DM isolation implications.

## Risks / Open Questions

- The PR predates current `bindings[].session.dmScope`; a rebase must preserve binding-level overrides and define precedence, likely binding override > agent override > global default.
- Per-agent `dmScope: "main"` can reduce DM isolation for that agent even when the global default is stricter, so config help/docs and regression tests should make that explicit.
- The current zod literals for `dmScope` are still duplicated across session and binding schemas; rework should avoid introducing a third drift point or should centralize all dmScope schema uses.
