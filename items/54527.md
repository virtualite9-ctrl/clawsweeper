## Summary

Current `main` still has the stale subsystem file-logger cache that this PR fixes, and the PR discussion shows its main review concern was already addressed in follow-up commit `846f12d`. This remains a live bug fix, not a conservative close candidate.

## Evidence

- **Current main still caches the subsystem file logger forever:** `createSubsystemLogger()` initializes `fileLogger` once and reuses it on later file writes. There is no date-boundary or root-logger refresh path in either `emitLog()` or `raw()`, so a long-running process keeps the same child logger after midnight.
  - file: [src/logging/subsystem.ts:327](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/src/logging/subsystem.ts#L327)
  - command: `nl -ba src/logging/subsystem.ts | sed -n '325,360p' && nl -ba src/logging/subsystem.ts | sed -n '392,425p'`
  - sha: [c070509b7f8f](https://github.com/openclaw/openclaw/commit/c070509b7f8fdbe479e5d829643782c272e30e83)
- **Root logger already rotates by dated filename, but subsystem child loggers do not rebuild:** `resolveSettings()` derives the default file from `defaultRollingPathForToday()`, and `getLogger()` rebuilds when `settings.file` changes. `getChildLogger()` would pick up that new root, but `createSubsystemLogger()` only calls it on first use, which is the exact stale-child gap described in the PR.
  - file: [src/logging/logger.ts:376](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/src/logging/logger.ts#L376)
  - command: `nl -ba src/logging/logger.ts | sed -n '371,388p' && nl -ba src/logging/logger.ts | sed -n '482,505p' && nl -ba src/logging/logger.ts | sed -n '571,574p'`
  - sha: [c070509b7f8f](https://github.com/openclaw/openclaw/commit/c070509b7f8fdbe479e5d829643782c272e30e83)
- **Current main lacks rollover coverage in subsystem tests:** `src/logging/subsystem.test.ts` on `main` ends with console-suppression cases and does not include a midnight rollover test, so there is no local proof that the stale-child behavior has been covered or replaced elsewhere.
  - file: [src/logging/subsystem.test.ts:145](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/src/logging/subsystem.test.ts#L145)
  - command: `nl -ba src/logging/subsystem.test.ts | sed -n '145,210p'`

- **PR review feedback was already incorporated:** The PR’s first review concern was that rollover deadline computation mixed a captured `Date.now()` with a fresh `new Date()`. The follow-up PR commit `846f12d` explicitly changed `computeNextLocalMidnightMs(now)` to use the captured timestamp, and the author replied on the review thread confirming that fix.
