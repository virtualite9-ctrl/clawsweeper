## Close Comment

Closing from a Codex review because current main already implements the clarification and error handling this report asked for. OpenClaw now documents that the `user`/existing-session browser path is host-only and that Docker, headless-server, Browserless, and other remote deployments should use a CDP profile or a node host instead (`docs/cli/browser.md:222`, `docs/tools/browser.md:212`). The runtime also now surfaces explicit remote-browser failures such as `Remote CDP for profile "<name>" is not reachable ...` and `Remote CDP websocket for profile "<name>" is not reachable ...` (`extensions/browser/src/browser/server-context.availability.ts:240`). Separately, the raw Playwright locator timeout pattern from this issue is rewritten into actionable guidance to re-snapshot the page instead of surfacing the opaque `locator.click ... waiting for locator('aria-ref=...')` text (`extensions/browser/src/browser/pw-tools-core.shared.ts:53`, covered by `extensions/browser/src/browser/pw-tools-core.clamps-timeoutms-scrollintoview.test.ts:52`). These changes are already present in release `v2026.4.22`.
