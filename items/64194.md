## Summary

Current `main` already contains the config-compat guard that limits channel legacy-rule scans to configured/touched channels, plus regression tests for that path. The same implementation is present in the latest tagged release `v2026.4.23`, so this issue is already fixed well enough to close.

## Evidence

- **Legacy-rule collection is scoped to relevant configured/touched channels:** `collectRelevantChannelIdsForTouchedPaths` derives channel ids from the actual config and touched paths, and `collectChannelLegacyConfigRules` only loads legacy doctor rules for those ids instead of scanning all bootstrap channels.

  - file: [src/config/legacy.ts:17](https://github.com/openclaw/openclaw/blob/5b8bd6371c666a5de43c82305e0315771e8e49d5/src/config/legacy.ts#L17)
