## Summary

Keep open. Current main still lacks the PR's central restart-time behavior: inbound debounce buffers are closure-local with no global flush hook, followup queues have only per-key drain scheduling, and the SIGUSR1 loop drains active command/embedded work before closing the server but does not flush debouncers or wait on all followup queues. The merged #47719 work is related but narrower restart deferral/orphan recovery, not this buffer-drain fix.

## Best Possible Solution

Keep #46303 open. The best path is a smaller additive core fix: register active inbound debouncers through a lifecycle-safe, controlled API; perform a bounded flush before restart teardown; wait for all followup queues using the same pending predicate as the drain loop after active turns can complete; preserve existing plugin SDK return contracts; and add targeted tests for SIGUSR1 ordering, followup queue pending states, and channel teardown races.
