## Summary

Keep open. Current main still lacks the PR's central restart-time behavior: inbound debounced messages remain trapped in closure-local timers, there is no global inbound debounce flush API, followup queues have no all-queue drain waiter, and the SIGUSR1 restart path drains only active command/embedded work before server close. Related shipped restart/deferral work is narrower and does not supersede this PR.

## Best Possible Solution

Keep this PR open as an implementation candidate, but prefer a smaller maintainer-reviewed fix: add a lifecycle-safe, controlled registry or equivalent drain hook for active inbound debouncers; preserve the existing public plugin SDK contract; flush debouncers during SIGUSR1 restart before teardown can discard buffered items; wait for followup queues after active turns can actually complete; and add focused tests for restart ordering, all-queue pending states including dropped summaries, and channel teardown races.
