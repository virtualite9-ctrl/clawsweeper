## Summary

Keep open. Current main still does not implement the PR's central SIGUSR1 restart-time drain behavior: inbound debounce buffers remain closure-local, there is no all-followup-queue drain waiter, and the restart path only waits for active command/embedded/runtime-deps work before closing. The PR is a plausible implementation candidate, but the branch is broad and the discussion identifies real lifecycle, SDK compatibility, and security-sensitive risks that need maintainer review rather than auto-close.

## Best Possible Solution

Keep this PR open as an implementation candidate, but ask for a narrower maintainer-reviewed revision: add a lifecycle-safe, controlled drain surface for active inbound debouncers; keep the public plugin SDK backward-compatible; flush inbound debounce buffers at the correct restart phase; wait for active turns and all followup queues including dropped-summary state; and cover restart ordering plus channel teardown races with focused tests.
