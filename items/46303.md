## Summary

Keep open. Current main still lacks the PR's proposed global inbound-debouncer flush and all-followup-queue drain before SIGUSR1 restart. The shipped related restart work covers deferral/orphan recovery for active subagent work, but not the closure-local debounce timers or pending followup queues described by this PR. The PR remains a plausible core bug fix, though it still needs maintainer review for API compatibility, teardown ordering, and the remaining review concerns.

## Best Possible Solution

Keep the PR open. The best path is a smaller, additive core fix that registers active inbound debouncers, performs a bounded restart-time flush before teardown, waits for followup queues using the same pending predicate as the drain loop, preserves public SDK compatibility, and covers the restart ordering with targeted tests. Do not close as implemented or duplicate until current main has equivalent source and test coverage.
