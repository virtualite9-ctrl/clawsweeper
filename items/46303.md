## Summary

Keep open. Current main still lacks the PR's central restart-time drain behavior, and the PR remains an active implementation candidate rather than obsolete cleanup. The current PR branch is broad and still needs maintainer review for lifecycle ordering, plugin SDK compatibility, test isolation, and security-sensitive global-state design.

## Best Possible Solution

Keep this PR open as an implementation candidate, but ask for a narrower maintainer-reviewed revision. The best path is a lifecycle-safe drain surface for active inbound debouncers; backward-compatible SDK additions; restart ordering that flushes debounce work, waits for active turns, then drains followup queues including dropped-summary state; explicit handling for channel worker queues such as Discord; and focused tests for restart ordering, teardown races, SDK compatibility, and the security boundary.
