## Summary

Keep open. Current main still lacks the PR's central restart-time behavior: inbound debounce buffers remain closure-local, there is no global inbound debounce flush API, followup queues only have per-key drain scheduling, and the SIGUSR1 restart path drains active command/embedded work before closing the server without flushing debouncers or waiting for all followup queues. Related shipped restart work is narrower and does not supersede this PR.

## Best Possible Solution

Keep #46303 open. The best path is a smaller additive core fix: introduce a lifecycle-safe, controlled way to register and flush active inbound debouncers; flush them during SIGUSR1 restart before teardown can discard buffered items; wait for all followup queues after active turns can complete using the same pending predicate as the drain loop; preserve existing plugin SDK contracts; and add targeted tests for SIGUSR1 ordering, followup queue pending states, and channel teardown races.

## Risks / Open Questions

- The PR branch still needs maintainer review before merge, especially around public plugin SDK compatibility for createInboundDebouncer/flushKey and teardown ordering for globally tracked debouncers.
- A restart-time fix must avoid new races where ingress can enqueue after a one-time flush or after debouncer unregistration.
- Any all-followup drain helper should mirror the real drain predicate, including dropped-summary state, and run in an order that lets active turns finish before queued followups are judged drained.
