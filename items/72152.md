## Summary

Current main already fixes the Bonjour/CIAO gateway crash-loop reported in #72152. The #72089 cross-reference in the issue comments matches the verified root cause: v2026.4.24 did not wire the Bonjour plugin's CIAO handler into the host process path, while current main does, shares handler state across staged plugin runtime copies, classifies the CIAO cancellation/assertion failures, and caps stuck advertiser restarts so the gateway stays up.

## Best Possible Solution

Close #72152 as fixed on main. Keep the current Bonjour process-handler wiring, CIAO classifier, shared handler registry, and self-disable watchdog behavior; affected v2026.4.24 users should update to the fixed build and remove the manual Bonjour disable override if they need LAN discovery.
