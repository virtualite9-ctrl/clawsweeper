## Summary

Current main fixes the Bonjour/CIAO gateway crash reported against v2026.4.24: the bundled Bonjour plugin now wires process-level rejection/exception handlers into the advertiser, classifies CIAO cancellation/assertion failures as handled, and disables the advertiser after repeated stuck probing/announcing restarts instead of crash-looping the gateway. The related #72089 root-cause comment matches the v2026.4.24 code path, where the plugin entry only passed the logger and never registered the handler.

## Best Possible Solution

Close #72152 as implemented on main, keep the current Bonjour handler/recovery implementation and regression tests, and let the next release carry the fix. #72089 remains the useful related root-cause thread for the same CIAO cancellation crash family.
