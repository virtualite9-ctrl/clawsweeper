## Summary

Current main already fixes the v2026.4.24 Bonjour/mDNS startup crash class: the bundled Bonjour plugin now wires scoped CIAO process-error handlers into the shared process handler registry, classified CIAO cancellation/assertion errors return handled before fatal process exit, and stuck advertisers self-disable after bounded restarts. The v2026.4.24 tag was affected because it passed only the logger into the advertiser, but current main has the fix and regression coverage.

## Best Possible Solution

Close this duplicate release-regression report as implemented on current main. Keep the scoped Bonjour/CIAO process-handler implementation, bounded advertiser restart/self-disable behavior, and documented Bonjour-disable workaround; release/backport timing should be handled through normal release management rather than keeping another core issue open.
