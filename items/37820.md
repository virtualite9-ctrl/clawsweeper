## Summary

Keep open. Current `main` does not implement this PR’s per-agent workspace `.env` behavior. `main` only loads a single `.env` from the process CWD into `process.env` at startup, and `exec` still derives host env from `process.env` plus per-call overrides. There is no threaded `workspaceEnv`/`parseWorkspaceDotEnv` path in the current agent run or exec code.
