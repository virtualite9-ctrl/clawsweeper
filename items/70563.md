## Summary

Keep #70563 open. Current main has subagent thinking inheritance and a Kimi thinking wrapper, but I could not find Kimi-owned handling for the reported `reasoning_content` replay requirement on assistant tool-call turns. The related thinking-block replay fix on main is narrower, and other providers that need `reasoning_content` have explicit provider-specific payload repair and tests; Kimi does not.

## Best Possible Solution

Keep the issue open and fix it in the Kimi provider surface: confirm Kimi Coding's exact replay contract, then add Kimi-owned payload/replay handling so thinking-enabled assistant tool-call turns carry the provider-required `reasoning_content` or equivalent preserved reasoning metadata during `sessions_spawn` and follow-up tool continuations. Add focused Kimi stream/replay tests plus a `sessions_spawn` regression covering inherited thinking, then document Kimi Coding thinking/replay behavior in the provider docs.
