## Summary

Keep open. Current main still only auto-injects the Linux CA bundle for nvm-managed Node runtimes, so the PR's central fix remains real and is not implemented on main. The linked issue #59494 being closed does not match the current checkout behavior. The PR is also not protected by maintainer author association or protected labels, but it still needs maintainer review or a corrected follow-up because the review discussion identifies an NVS marker gap in the proposed implementation.

## Best Possible Solution

Keep #59500 open and land a corrected version: extend `src/bootstrap/node-extra-ca-certs.ts` beyond nvm, preferably by sharing or centralizing the version-manager marker contract with daemon runtime/audit code, add bootstrap tests for fnm, Volta, asdf, n, nodenv, nodebrew, and NVS default paths, and update nvm-specific docs/messages if the supported behavior becomes broader.
