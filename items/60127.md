## Summary

Keep open. Current main has adjacent isolation features (multi-agent routing, separate agent dirs/auth stores, per-agent sandbox/tool policy, and separate-gateway guidance), but it explicitly does not provide the requested shared-server hostile multi-tenancy with tenant identity, tenant-scoped RBAC, quotas, plugin enablement, and audit boundaries. The KinthAI comment is useful evidence of demand and names concrete missing core primitives rather than proving this can be closed as plugin-only work.

## Best Possible Solution

Keep this issue open for maintainer product and security triage. If in scope, the design should add tenant identity propagation through Gateway auth/request context, tenant-scoped state/config/plugins/skills/model auth/quotas/audit logs, and explicit cross-tenant denial tests and docs. If out of scope, maintainers should make that explicit in docs and point users toward separate gateways/hosts or managed external services such as KinthAI, with a separate plugin/API design discussion only for extension seams that external implementations truly need.
