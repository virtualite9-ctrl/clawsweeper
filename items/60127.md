## Summary

Keep open. Current main has adjacent per-agent isolation, gateway operator scopes, and shared-inbox hardening, but it still does not implement the requested shared-server tenant model with tenant identity, tenant-scoped RBAC, quotas, plugin enablement, and audit boundaries. The KinthAI discussion is useful evidence of demand and names missing core primitives rather than proving this is cleanly plugin-only work.

## Best Possible Solution

Keep this issue open for maintainer product and security triage. If in scope, design tenant identity propagation through Gateway auth/request context, tenant-scoped state/config/plugins/skills/model auth, per-tenant quotas, audit logs, and cross-tenant denial tests/docs. If out of scope, maintainers should document that shared hostile multi-tenancy is unsupported and point users to separate gateways/hosts or managed external services, while opening narrower API discussions only for specific extension seams external implementations need.
