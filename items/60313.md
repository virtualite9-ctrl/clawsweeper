## Summary

Current `main` already supports agent-scoped skill isolation through per-agent skill allowlists (`agents.defaults.skills` and `agents.list[].skills`), and that filter is applied across prompt construction, slash-command discovery, and skill snapshots. The same behavior is present in the `v2026.4.23` release tag.

## Evidence

- **Per-agent skills are part of the public config contract:** `AgentConfig` includes a `skills?: string[]` allowlist with explicit semantics: omitting it inherits defaults, while a provided list replaces defaults instead of merging.

## Risks / Open Questions
