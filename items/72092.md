## Summary

Keep PR #72092 open. Current main still hard-requires a non-empty API key in the media-understanding image, MiniMax fallback, and audio/video execution paths, while Bedrock docs and the chat path support `aws-sdk` credential-chain auth without a static key. The PR is a focused fix for open bug #72031, and its latest head addresses the reviewed audio/video empty-key rotation gap with `allowEmptyKey` plus regression tests.

## Best Possible Solution

Keep PR #72092 open for maintainer review and CI. The best path is to validate and iterate on this focused implementation so aws-sdk missing-key allowance flows through image, MiniMax fallback, and audio/video execution, including the empty-key rotation contract, then merge it if tests and review stay green.
