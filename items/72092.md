## Summary

Keep #72092 open. Current main still has the Bedrock aws-sdk media-auth gap in the image and audio/video media-understanding paths, while the PR is an active focused fix for #72031 and its latest commit addresses the review concern about empty keys being dropped by API-key rotation.

## Best Possible Solution

Keep the PR open and finish normal maintainer review/CI. The right product fix is to add the aws-sdk missing-key allowance to media-understanding image, MiniMax fallback, and audio/video provider execution paths, while preserving `requireApiKey` failures for every non-aws-sdk mode and covering image plus audio/video empty-key Bedrock regressions.
