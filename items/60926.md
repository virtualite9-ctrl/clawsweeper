## Summary

Current `main` explicitly prevents heartbeat runs from targeting subagent session keys and adds regression coverage for the reported failure mode, so this issue is already fixed on `main` even though the latest release tag does not appear to include that change yet.

## Evidence

- **Heartbeat runner rejects subagent targets:** `resolveHeartbeatSession()` now has an explicit guard that treats any forced `agent:*:subagent:*` session key as invalid for heartbeat delivery and falls back to the agent main session instead.
  - file: [src/infra/heartbeat-runner.ts:273](https://github.com/openclaw/openclaw/blob/afc0c32bd0eb9f984fcf5bcf26c9ea821834b74d/src/infra/heartbeat-runner.ts#L273)

  - sha: [afc0c32bd0eb](https://github.com/openclaw/openclaw/commit/afc0c32bd0eb9f984fcf5bcf26c9ea821834b74d)
- **Heartbeat skips when target session lane is busy:** `runHeartbeatOnce()` still checks `CommandLane.Main`, and now also checks the resolved session lane before running the heartbeat, returning `requests-in-flight` instead of interrupting an active turn.
  - file: [src/infra/heartbeat-runner.ts:759](https://github.com/openclaw/openclaw/blob/afc0c32bd0eb9f984fcf5bcf26c9ea821834b74d/src/infra/heartbeat-runner.ts#L759)
