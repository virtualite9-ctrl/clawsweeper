## Summary

Keep open. Current main has narrow guards for exact streaming/content duplication and WebChat history refresh duplication, but I did not find high-confidence evidence that it fixes the reported WebChat/model behavior: repeated assistant preambles around tool and approval steps with `omlx.qwen3.5.35b`. Related duplicate-delivery reports such as #58443 and #64606 are open or channel-specific, not clear canonical supersets for this issue.

## Best Possible Solution

Keep this issue open and ask for or capture a current-main reproduction with one raw session transcript plus `chat` and `agent` gateway events for the same run. If the duplicate is UI rendering/history refresh, add a regression around `ui/src/ui/controllers/chat.ts` or `ui/src/ui/app-gateway.ts`; if it is duplicate delivery from the embedded run, add per-run/idempotent suppression in the subscriber/delivery path; if it is model-generated verbosity around tools/approvals, tune the WebChat/system prompt or provider-specific behavior rather than closing as implemented.
