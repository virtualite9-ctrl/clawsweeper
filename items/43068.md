## Summary

Keep this PR open. Current main still lacks sandbox-only Python/Node research-script import validation, and the PR targets a real core exec hardening gap, but the proposed validator has unresolved bypasses and needs rebase work before it is mergeable.

## Best Possible Solution

Keep this PR open and request changes. The best path is to rebase onto current main, preserve the current validateScriptFileForShellBleed fail-closed behavior, harden the import validator for multiline Node import/re-export forms, whitespace in require/import calls, and Python bare-relative denied names, then add regression tests for those bypasses plus sandbox-vs-host behavior before maintainer review.

## Risks / Open Questions

- Merging the PR as-is could create false confidence in sandbox import enforcement because common JavaScript multiline import/re-export forms and require/import calls with whitespace can bypass the proposed regex scanner.

- Closing the PR would leave current main without sandbox-only import hardening for direct Python/Node research scripts.
