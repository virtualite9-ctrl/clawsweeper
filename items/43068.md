## Summary

Keep this PR open. Current main still lacks sandbox-only Python/Node research-script import validation, and the PR targets a real core exec hardening gap. The PR is not mergeable as-is because the proposed validator has unresolved bypasses and needs to be rebased onto the current exec preflight implementation.

## Best Possible Solution

Keep this PR open and request changes. The best path is to rebase onto current main, preserve the current validateScriptFileForShellBleed fail-closed behavior, harden import validation for multiline Node import/re-export syntax and whitespace-separated dynamic calls, handle Python bare-relative denied names, and add regression tests for those bypasses plus sandbox-vs-host behavior before maintainer review.

## Risks / Open Questions

- Closing this PR would leave current main without sandbox-only import hardening for direct Python/Node research scripts.
- Merging the PR as-is could create false confidence in sandbox import enforcement because common multiline Node import/re-export forms, whitespace-separated require/import calls, and Python bare-relative denied-name forms can bypass the proposed scanner.
- A rebase must preserve current main's expanded shell-bleed parsing, workdir-bounded file inspection, YOLO host skip behavior, and fail-closed ambiguous interpreter handling.
