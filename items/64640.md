## Summary

Keep open. Current main has superseded the webhooks half through merged synchronous registration and request-time SecretRef handling, but the memory-search cfg-forwarding problem still exists on main. This PR should be rebased and narrowed rather than closed.

## Best Possible Solution

Keep this PR open, but require a rebase and narrowing. Drop the obsolete webhooks changes covered by merged #67941, keep the memory-search work, switch `resolveMemorySearchConfig` to the cfg-aware memory embedding provider runtime resolver for primary, fallback, and multimodal validation lookup, add the focused runtime-config forwarding regression test, and then run the changed agent/plugin gates.
