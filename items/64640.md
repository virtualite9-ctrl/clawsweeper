## Summary

Keep open. Current main has already superseded the webhooks portion through merged synchronous registration/request-time secret handling, but the memory-search part is still useful: `resolveMemorySearchConfig` still uses registered-only memory embedding provider lookup and does not pass the active `OpenClawConfig` into provider, fallback, or multimodal provider resolution.

## Best Possible Solution

Keep this PR open, but require it to be rebased and narrowed. Drop the obsolete webhooks changes covered by merged #67941, change memory-search to use the cfg-aware memory embedding provider runtime resolver, pass the active `OpenClawConfig` through provider, fallback, and multimodal validation lookup, add the focused runtime-config forwarding regression test, then run the changed agent/plugin gates.
