## Summary

Keep #71921 open. Current main only preserves Bedrock image capability when Bedrock discovery returns a foundation model with `inputModalities: ["TEXT", "IMAGE"]`; inference profiles whose backing foundation model is absent from the discovered map still fall back to `input: ["text"]`. That matches the reported failure mode for Bedrock Claude profile IDs being treated as text-only, and there is an existing test that explicitly expects a Claude Opus 4.6-backed profile to remain text-only in that fallback path.

## Best Possible Solution

Add Bedrock plugin-owned known capability metadata for the affected Anthropic Claude 4.x Bedrock IDs and normalized regional/global profile variants, then use it when discovery cannot inherit a foundation model entry. Update `extensions/amazon-bedrock/discovery.test.ts` so Opus 4.7, Opus 4.6, and Sonnet 4.6 profile/foundation fallback entries resolve to `input: ["text", "image"]` while unknown models remain conservative text-only.
