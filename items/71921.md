## Summary

Current main still does not implement the requested Bedrock Claude vision-capability fix. The Amazon Bedrock plugin has known context-window metadata for the affected Claude Opus 4.7, Opus 4.6, and Sonnet 4.6 IDs, but input capability is still derived from AWS `inputModalities` or from an already discovered backing foundation model. If Bedrock discovery reports those rows as text-only, or if an inference profile is resolved without a discovered base model, OpenClaw still emits `input: ["text"]`, and the image paths then reject the model as non-vision. Related Claude 4.7 and context-window reports do not supersede this specific Bedrock vision metadata gap.

## Best Possible Solution

Keep #71921 open. The best fix is provider-owned Bedrock capability metadata in `extensions/amazon-bedrock/discovery.ts`: recognize the affected Claude Opus 4.7, Opus 4.6, and Sonnet 4.6 base IDs plus regional/global inference-profile forms as text+image capable, apply that override when mapping foundation models and inference profiles, and add focused discovery tests for the exact affected IDs when AWS returns stale text-only or missing modality data. Update Bedrock docs or changelog if the shipped behavior changes.
