## Summary

Keep #69605 open. Current main still has the reported layout hole: assistant message actions render as an absolutely positioned `.chat-bubble-actions` overlay, and the existing `.chat-bubble.has-copy` padding rule is not applied to assistant bubbles. The code-block click handler copies only `data-code` when the code-block button receives the click, but the message-level button still copies the whole assistant markdown when it is the hit target. The related fix PR #69634 is still open, while #69689 and #69635 are closed unmerged, so there is no shipped/current-main fix to close against.

## Best Possible Solution

Keep the issue open and land a focused UI fix, likely the #69634/#69689 approach: add `has-copy` or equivalent reserved spacing when assistant bubble actions render, then add a regression test in `ui/src/ui/chat/grouped-render.test.ts` that assistant bubbles with copy/expand actions receive the spacing class while user bubbles do not. A browser/manual verification of clicking the fenced-code `Copy` control near the top-right of an assistant bubble should confirm the clipboard receives only the code text.
