## Summary

Keep #69605 open. Current main still renders the assistant message-level copy action as an absolutely positioned overlay while the assistant bubble class list does not include the existing `has-copy` spacing class, so the reported code-block copy overlap/mis-click path is still present. Related PR #69634 tracks the same fix but remains open/unmerged; the closed related PRs did not land this behavior on main.

## Best Possible Solution

Keep the issue open until the Control UI chat renderer applies the existing `has-copy` layout contract, or an equivalent fix, whenever assistant bubble actions render. The likely maintainer path is to finish or supersede #69634, add a focused regression test in `ui/src/ui/chat/grouped-render.test.ts` asserting assistant bubbles with copy/expand actions get the spacing class, and preserve the existing code-block `data-code` copy behavior.
