## Summary

Keep #69605 open. Current main still has the exact layout gap described by the issue and related PRs: the CSS class that reserves space for message-level copy/expand actions exists, but the assistant bubble renderer does not apply it when those actions render. The code-block copy handler itself copies `data-code` when the code button receives the click, but the current layout can still leave the message-level copy button over that area.

## Best Possible Solution

Keep the issue open until a focused UI fix lands. The likely fix is to apply the existing `has-copy` class to assistant `.chat-bubble` elements whenever copy/expand actions render, preserving the current code-block copy handler, and add a regression test in `ui/src/ui/chat/grouped-render.test.ts` that locks the class/layout contract for assistant bubbles with markdown actions.
