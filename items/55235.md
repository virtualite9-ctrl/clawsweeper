## Summary

Keep open: current `main` still exposes config-schema generation only as a manual `openclaw config schema` command that prints JSON to stdout. There is no landed bootstrap/update sync path for a local schema artifact, no `maintain_config_json_schema` setting, and the issue timeline itself points to an implementation PR that is still open.

## Evidence

- **Schema generation is still a manual CLI subcommand:** `buildCliConfigSchema()` is only used by `runConfigSchema()`, and the command registration exposes it as `openclaw config schema`; the implementation writes JSON to stdout rather than syncing a sidecar file during bootstrap or update.
  - file: [src/cli/config-cli.ts:1431](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/src/cli/config-cli.ts#L1431)
  - sha: [c070509b7f8f](https://github.com/openclaw/openclaw/commit/c070509b7f8fdbe479e5d829643782c272e30e83)
- **Docs still instruct manual redirection to a file:** The user-facing CLI docs describe `config schema` as printing JSON to stdout and show `openclaw config schema > openclaw.schema.json` as the way to create a file, which matches a manual workflow rather than automatic maintenance beside `openclaw.json`.
  - file: [docs/cli/config.md:53](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/docs/cli/config.md#L53)
  - sha: [c070509b7f8f](https://github.com/openclaw/openclaw/commit/c070509b7f8fdbe479e5d829643782c272e30e83)
- **Tests cover stdout output, not startup/update sync:** The `config schema` tests assert that the command prints generated JSON as plain text and verify the `$schema` property in that output; there is no test coverage here for bootstrap/update-time artifact syncing or config mutation.
  - file: [src/cli/config-cli.test.ts:630](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/src/cli/config-cli.test.ts#L630)
