## Summary

Keep open. Current main still has the WhatsApp extension-local watchdog that defaults to 30 minutes, measures only inbound-message freshness, and force-closes the socket with status 499. The newer gateway health policy is transport-liveness based, but WhatsApp does not currently publish `lastTransportActivityAt`, and the issue is already the dedupe parent with an open fix PR (#71466) rather than an obsolete duplicate.

## Best Possible Solution

Keep #70678 open as the canonical bug until a fix lands. The best code path is to update the WhatsApp plugin so the watchdog uses real transport liveness, or otherwise disables/configures the inbound-message watchdog for quiet linked-device sessions, then add regression coverage proving a healthy quiet socket does not force-close with 499. PR #71466 or a successor should be reviewed against that behavior before closing the issue.
