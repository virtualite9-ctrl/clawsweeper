## Summary

Current main already surfaces ACP turn failures back to the parent, tracks ACP child runs in the background-task ledger with terminal notifications/visibility, and documents the exact non-interactive permission setting that the reporter later confirmed was the root cause. The released v2026.4.22 tree already contains the ACP exit-handling and parent-notification coverage.

## Close Comment

Closing as implemented on main after Codex review. The current ACP path no longer relies on silent waiting: `src/acp/control-plane/manager.core.ts` converts ACP runtime errors into terminal failures, `src/auto-reply/reply/dispatch-acp.ts` delivers final ACP error replies to the parent path, and ACP child runs are tracked in the background-task ledger with parent-facing failure notifications and status visibility (`src/tasks/task-registry.test.ts`, `src/agents/openclaw-tools.session-status.test.ts`). The released `v2026.4.22` tree already contains coverage for early `acpx exited with signal SIGTERM` handling and parent failure notification. Current docs also explicitly document the `nonInteractivePermissions` / `permissionMode` configuration that matched the reporter’s later confirmed root cause in `docs/tools/acp-agents-setup.md`. If someone can still reproduce this on `v2026.4.22+` with the documented ACP permission config, please open a fresh issue with the exact ACP config, agent/backend versions, and gateway logs.
