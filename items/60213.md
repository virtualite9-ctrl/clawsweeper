## Summary

Current main already persists post-compaction session rotation back into the session store and queued follow-up state, with tests covering rotated `sessionId`/`sessionFile` handling. The same implementation is present in release `v2026.4.23`, so the orphaned-session/new-empty-session symptom described in #60213 is already implemented on main.

## Evidence

- **Agent runner persists rotated session identity after compaction:** After auto-compaction, the reply runner forwards `runResult.meta?.agentMeta?.sessionId` into `incrementRunCompactionCount(...)` and refreshes queued follow-up state with the updated `sessionId` and `sessionFile`, so later turns stay attached to the rotated transcript instead of the old file.
  - file: [src/auto-reply/reply/agent-runner.ts:1551](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/src/auto-reply/reply/agent-runner.ts#L1551)
