## Summary

Keep #48304 open. Current main has useful adjacent pieces, including group/channel `toolsBySender`, trusted requester metadata passed through the message path, and plugin `before_tool_call` block/approval hooks, but it does not implement the requested gateway-level per-action/per-sender authorization for the `message` tool. The docs still describe the underlying multi-user risk: if several people can message one tool-enabled agent, they can steer the same permission set.

## Best Possible Solution

Keep the issue open for maintainer product/security judgment. A good fix would add a narrow, trusted authorization point for message actions, or first enrich the plugin/pre-tool policy API with trusted sender provenance and action classification, then document and test owner-only or owner-confirmed `message.send` behavior for shared workspaces.
