## Summary

Keep open. Current main still lets raw `toolResult` / `tool` content become subagent completion fallback output without checking `isError`, `is_error`, or status-based failed-tool markers. The direct fix in #39367 was closed unmerged, and the merged #59649 hardened adjacent task/status/followup leak surfaces but did not wire a failed-tool guard into this extractor path.

## Best Possible Solution

Keep the issue open and implement the narrow extractor fix in `src/agents/subagent-announce-output.ts`: detect failed tool results before populating `latestRawText`, reuse or mirror the existing `isToolResultError` semantics, and add focused regression tests proving failed tool results are ignored while successful tool-only completion fallback still works.
