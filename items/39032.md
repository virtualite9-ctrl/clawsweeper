## Summary

Keep open. Current main still lets subagent completion fallback use raw `toolResult`/`tool` text without checking `isError` or `is_error`, so the exact leak reported in #39032 is not resolved well enough to close. The related #39367 was described as the direct fix in the issue discussion, but the provided context says it was closed unmerged; the later #59649 was broader adjacent sanitization and does not prove this subagent completion path is fixed.
