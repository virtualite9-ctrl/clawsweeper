## Summary

Keep #44571 open. Current main has partial runtime fallback from the main auth store, so a non-main agent with no local auth store can see main-agent profiles, but the onboarding/provider-auth write path still persists token/API-key auth results to only one resolved agentDir. Anthropic setup-token still returns or writes a single `anthropic:default` profile for the current agentDir, local same-id profiles still override main profiles, and the related sync PR #44644 was closed unmerged. The requested all-configured-agent distribution or stale sibling repair is not fully shipped.

## Best Possible Solution

Keep the issue open and implement a generic provider-auth distribution policy: after successful onboarding/auth writes for a global/default setup, mirror secret-bearing token/API-key/OAuth profiles to configured `agents.list` agentDirs, or define a safe runtime preference for refreshed main credentials over stale same-id sibling credentials. Add regression tests for Anthropic setup-token with multiple configured agents, no local sibling auth file, and stale sibling `auth-profiles.json` shadowing a refreshed main profile.
