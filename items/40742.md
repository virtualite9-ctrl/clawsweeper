## Summary

Current main already contains the retry-state and replay-sanitization work that matches this report: fallback retries preserve the original task, embedded attempts sanitize/validate replay history before each provider call, OpenAI-to-Anthropic transcript handoff repairs or drops stale tool results, and the runner surfaces explicit timeout/incomplete-turn states instead of silently orphaning the run. The same relevant logic is present in release `v2026.4.22`.

Separately, timeout/incomplete embedded runs now surface explicit terminal error/liveness metadata instead of silently orphaning the turn (`src/agents/pi-embedded-runner/run.ts:1700`).
