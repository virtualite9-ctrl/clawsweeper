## Summary

Current main still routes pre-prompt compaction token accounting through the pinned pi-coding-agent estimator, and that estimator still has unchecked `.length` reads for malformed assistant, toolResult, bashExecution, and summary message shapes. The provided context also shows #63636 is an open, unmerged closing PR for this issue, so the right maintainer path is to review/land or supersede that PR rather than close this issue now.

## Best Possible Solution

Keep this issue open and review/land or close #63636. The durable fix should add guarded local token estimation or normalize malformed replay history before all compaction token-estimation call sites, covering missing/non-array assistant content, missing `text`/`thinking`/`name`, non-array toolResult/custom content, and missing bash/summary fields with regression tests in `src/agents/compaction.test.ts` and `src/agents/pi-embedded-runner/run/preemptive-compaction.test.ts`.
