## Summary

Current main still does not fully implement the reported recovery path. `sessions.send` still fails on a missing session entry, there is no registered `sessions.start`/`sessions_start` surface, and `sessions_send` timeout results still lack the delivery metadata discussed in the issue. The provided GitHub context identifies open PR #52921 as the implementation candidate and it uses closing syntax for this issue, so the issue should remain open until that PR is reviewed, landed, or closed.

## Best Possible Solution

Keep this issue open while maintainers review PR #52921. If the PR is correct and safe, land it and let this issue close through the linked PR. If not, keep this issue as the canonical report and implement the missing behavior in the gateway/session owner surfaces: an idempotent `sessions.start`/`sessions_start`, `sessions_send` auto-bootstrap for known agent configs, structured not-found errors, delivery metadata on accepted/timeout paths, focused regression tests, and docs updates.
