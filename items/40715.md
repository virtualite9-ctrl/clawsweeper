## Summary

Current main already supports multiple independently scheduled agent tasks with isolated/custom sessions, per-job agent targeting, task tracking, and configurable overlapping cron execution. The feature described in issue #40715 is already implemented and documented, and it was present in release v2026.4.22.

## Close Comment

Closing from a Codex review because this capability is already on `main` and was shipped in `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`). OpenClaw already has a built-in Gateway cron scheduler with independent `--at`, `--every`, and `--cron` jobs in `docs/automation/cron-jobs.md:12`, isolated/custom session execution in `docs/automation/cron-jobs.md:80`, per-job agent targeting in `docs/automation/cron-jobs.md:340`, and configurable overlapping execution via `cron.maxConcurrentRuns` in `docs/automation/cron-jobs.md:356` and `src/cron/service/timer.ts:107`. The scheduler actually fans out due jobs with worker slots in `src/cron/service/timer.ts:707`, and regression coverage verifies two jobs start concurrently when `maxConcurrentRuns` is set to 2 in `src/cron/service/timer.regression.test.ts:887`. If you still need a different orchestration model beyond the existing cron + isolated/custom-session flow, open a new issue with the exact missing workflow and constraints.
