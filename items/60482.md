## Summary

Keep open. Current `main` still has a real global-reset cleanup path for Tailscale Serve/Funnel: restart calls server shutdown, shutdown runs the registered Tailscale cleanup hook, and that hook uses `tailscale serve reset` / `tailscale funnel reset` rather than removing only an OpenClaw-owned binding. The report likely needs one extra detail about whether `gateway.tailscale.resetOnExit` (or `--tailscale-reset-on-exit`) was enabled, because docs/config still show that default as `false`, but the underlying ownership bug is not fixed on `main`.

## Evidence
