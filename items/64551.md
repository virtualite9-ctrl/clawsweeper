## Summary

Close #64551 as a duplicate of the older open tracker #37591. Current main still has the macOS companion `system.run.prepare` gap, but this issue reports the same failure mode already being tracked there; the useful restart/workaround failure and second 2026.4.15 reproduction should remain discoverable via the duplicate link.

## Best Possible Solution

Close #64551 as a duplicate of https://github.com/openclaw/openclaw/issues/37591 and keep the remaining work centralized there, with implementation/review continuing through the macOS `system.run.prepare` fix path such as PR #38781. The eventual fix should either add advertised and handled macOS companion support for `system.run.prepare`, or change the node exec contract so macOS companion nodes do not advertise an unusable `system.run` path.
