## Summary

The linked SecretRef health-check bug from #37718 is already handled on current `main` by guarded account-resolution logic in `src/commands/health.ts`. This PR targets a different code path, does not match the original reproducer, and also bundles unrelated Telegram changes with existing review findings.

## Evidence

## Risks / Open Questions

- I did not run a live reproduction in this environment because the requested review is read-only and `pnpm` is unavailable here; this decision is based on current main code, contracts, and the linked GitHub evidence.
- `src/commands/health.ts` still lacks a defensive non-string guard on `accountIdsToProbe`, so a future third-party plugin that violates the string-return contract could still hit that path even though #37718's reported failure mode is already covered.
