## Summary

Keep open. Current `main` still replays pending outbound queue entries on gateway startup without any stale-age cutoff, and transcript echo messages are still sent through that same queued path. The reported restart batch-flush behavior is therefore still plausible on current code, not implemented away.

## Evidence

- **Startup recovery still replays all pending outbound entries:** `recoverPendingDeliveries()` loads every pending queue entry, sorts by `enqueuedAt`, and attempts recovery for each one. The only built-in skips are max retries, retry backoff, or the overall recovery time budget; there is no age-based discard path.
