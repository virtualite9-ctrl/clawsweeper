## Summary

#8190 should remain open. Current main still has single-session reset plus per-session auth/cache invalidation behavior, but no gateway/API/command surface that globally refreshes auth for all active channel sessions while preserving conversation context. The existing issue discussion already includes a 2026-04-26 maintainer/Codex review comment keeping this as the canonical tracker, and I found no newer implementation on current main.

## Best Possible Solution

Keep #8190 open as the canonical tracker. The best implementation is an additive gateway/session capability, such as `sessions.refreshAuth` or an explicit all-sessions auth-refresh mode, that enumerates active sessions, invalidates auth-backed runtime and CLI caches, preserves transcripts and channel/session context where possible, and adds protocol, server, channel/session-store tests, plus user-facing docs.
