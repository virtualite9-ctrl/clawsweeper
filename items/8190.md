## Summary

Keep #8190 open. Current main still only provides single-session hard reset and current-session soft reset paths; it does not provide a global all-session auth refresh/reset that preserves conversation context across active channel sessions. The existing #7533 account/auth-order work is related but does not supersede this cross-session lifecycle capability, and the prior maintainer-facing comment already explicitly kept this issue open.

## Best Possible Solution

Keep #8190 as the canonical tracker. The likely best fix is an additive gateway/session capability, such as `sessions.refreshAuth` or an explicit all-sessions auth-refresh mode, that enumerates active session stores, invalidates auth-backed runtime and CLI caches, preserves transcripts/session context where possible, and adds protocol, gateway, channel/session-store tests plus user-facing docs. Treat #7533 as shipped account/auth-order switching, not a replacement for this global session refresh behavior.
