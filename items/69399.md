## Summary

Keep PR #69399 open. Current main still only prepends model fallback and fallback-cleared notices when verbose mode is enabled, and the proposed `agents.defaults.notifyOnModelFallback` config is absent from the runtime config type, Zod schema, generated/help metadata, and model-failover docs. The linked issue #67197 remains the same unimplemented user problem rather than a shipped or superseding fix.

## Best Possible Solution

Keep the PR or a maintainer-owned replacement open until OpenClaw has a reviewed config contract for normal-mode fallback notices, runtime wiring in `runReplyAgent`, generated config metadata, docs, and regression coverage for fallback transition, once-per-state suppression, fallback-cleared behavior, and default verbose-off silence.
