## Summary

Keep PR #69399 open. Current main still has the behavior the PR is meant to change: fallback and fallback-cleared reply notices are emitted only under verbose mode, `agents.defaults.notifyOnModelFallback` is absent from config types/schema/docs, and the existing tests explicitly assert that verbose-off replies do not include fallback notices while lifecycle events still fire. The linked feature issue #67197 remains a real, focused request rather than an implemented, duplicate, ClawHub-scoped, or out-of-repo item.

## Best Possible Solution

Keep the PR or a rebased successor open. The best product path is to add a narrow opt-in `agents.defaults.notifyOnModelFallback` on current main, wire both fallback and fallback-cleared reply notices through that flag without changing default behavior, update generated config/docs, and keep regression coverage for transition, once-per-state, cleared, and verbose-off lifecycle behavior.
