## Summary

Keep PR #71086 open. Current main has not implemented the requested suppression: mutating tool failures are still always surfaced, and the targeted test still asserts that behavior even after assistant output. The PR is a recent focused fix attempt, but it needs revision before maintainer cleanup because the review discussion identifies a real correctness risk: suppressing based only on any user-facing reply can hide unresolved mutating-tool failures after partial/error output.

## Best Possible Solution

Keep the PR open and ask for a narrow revision: suppress stale mutating-tool warnings only when the turn reached a successful final completion, preserve warnings for error/interrupted or unresolved mutating failures, update the stale block comment, and add regression coverage for both successful recovery and partial/error-output cases. Coordinate with open #39406 if maintainers want the broader suppression policy handled there.
