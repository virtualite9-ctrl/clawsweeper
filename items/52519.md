## Summary

The issue describes the legacy Chrome extension relay behavior from OpenClaw 2026.2.17. Current main no longer ships that host-local extension relay path: v2026.3.22 removed it, migrated users to Chrome MCP `existing-session`/`user`, and current tab list/open/focus flows for that mode go through Chrome MCP instead of the old relay/CDP split that produced enumerable-but-not-controllable tabs.
