## Summary

Keep #38246 open. Current main still routes WhatsApp visible auto-replies through the inbound message reply/sendMedia closures, which send to the inbound chat JID. Main-DM pinning only protects lastRoute metadata and does not block, reroute, or force explicit targets for plain replies in allowlisted third-party DMs. The WhatsApp config/schema also has no force-explicit-target or third-party plain-reply guard.

## Best Possible Solution

Keep the issue open and implement a WhatsApp plugin routing guard in the visible auto-reply delivery path before deliverWebReply sends payloads. Add typed config/schema/docs for the chosen policy and regression coverage for owner DMs, allowlisted non-owner DMs where plain replies are blocked or rerouted, and explicit message-tool sends that still work intentionally.
