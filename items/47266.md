## Summary

Current main implements the requested logfile rollover behavior. The file transport resolves the active rolling logfile at write time instead of keeping a stale path, and regression coverage verifies a long-lived subsystem logger crosses from one dated logfile to the next.

## Best Possible Solution

Close this issue as implemented on current main. Keep the write-time rolling path resolution and the long-lived subsystem logger regression test; handle any remaining source-location metadata concern as a separate focused logging issue if a maintainer wants that behavior changed.
