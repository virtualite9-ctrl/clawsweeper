## Summary

Keep open. Current main still does not serialize gateway `config.set`, `config.patch`, or `config.apply` mutations, and no equivalent race regression test appears to have landed. This PR is still a plausible implementation candidate for the live gateway RPC race, but it needs rebase/review before landing.

## Best Possible Solution

Keep this PR open as a still-relevant implementation candidate. The best maintainer path is to rebase or reimplement the serialization on current main, preserve the current config handler semantics, add race coverage for all three gateway write methods, resolve the queue-rejection review point, and then land it as the gateway-side fix for the lost-update race.
