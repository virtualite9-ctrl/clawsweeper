## Summary

Keep this PR open. Current main still has unthrottled OpenAI Responses, Azure OpenAI Responses, and OpenAI chat completions SDK calls, and no shipped `LLM_MAX_CONCURRENT` or provider-request limiter surface. The PR is a plausible implementation candidate, but the provided diff is incomplete and has merge-blocking correctness/format issues called out in review comments.

## Best Possible Solution

Keep this PR open for revision or maintainer replacement. A mergeable solution should add a tested provider-request limiter covering OpenAI Responses, Azure OpenAI Responses, and OpenAI chat completions; hold limiter slots until streamed responses finish or abort; reject queued work promptly on `AbortSignal`; parse invalid/empty concurrency configuration safely; and document the chosen config surface. If maintainers ship the same capability separately, this PR can then be closed as superseded.
