## Summary

Keep open. Current main still strips an own-line trailing `NO_REPLY` from auto-reply text and delivers the preceding narration, and the current tests explicitly preserve that behavior. This PR remains relevant, with related open follow-up #46626 showing the same author is still tracking remaining delivery paths rather than abandoning the fix.

## Best Possible Solution

Keep this PR open for maintainer review or author follow-up. The best path is a focused shared silent-token detector, updated `normalizeReplyPayload` regression tests that flip the current own-line trailing `NO_REPLY` expectation, and coordinated coverage with #46626 for directive, delivery, cron, gateway, and chat-history paths while preserving inline `😄 NO_REPLY` stripping.
