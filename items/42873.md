## Summary

Keep this PR open. Current main still does not implement the shared own-line `NO_REPLY` suppression this PR proposes for the normal auto-reply normalizer: it strips the trailing token and delivers the preceding text, and the colocated test still asserts that behavior. The PR is also paired with same-author open follow-up #46626, so it should not be auto-closed while that implementation path remains unresolved.

## Best Possible Solution

Keep this PR open for maintainer review or author follow-up. The best path is a rebased shared silent-token detector or equivalent trailing/own-line guard in `src/auto-reply/tokens.ts` and `src/auto-reply/reply/normalize-reply.ts`, with tests flipping the current own-line trailing `NO_REPLY` expectation while preserving inline `😄 NO_REPLY` stripping and current main's newer JSON, leading-token, case-insensitive, and cron behavior. Coordinate with #46626 and #69394 before closing duplicates.
