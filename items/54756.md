## Summary

Keep open. Current `main` has already absorbed the manual `openclaw update` service-refresh part of this PR, but it has not implemented the auto-update fallback change: `runAutoUpdateCommand()` still probes `dist/entry.js` before `dist/index.js`. Because the remaining behavior change is still live and the discussion confirms it addresses a real service-entrypoint problem, this is not strong enough for an `implemented_on_main` close.
