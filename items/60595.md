## Summary

Keep open. Current `main` still unconditionally returns `structuredClone(store)` from `loadSessionStore`, even when `skipCache: true`, and the locked session-store update paths still call that fresh-load path before mutating and saving. The PR's claimed optimization is therefore not implemented on `main`, and the provided discussion does not show a reason to close it conservatively.

## Evidence

- **Current main still clones on every uncached load:** `loadSessionStore` skips reading/writing the object cache when `opts.skipCache` is true, but after migrations/normalization it still ends with `return structuredClone(store);` unconditionally. The PR's proposed early return for `skipCache` is absent on current `main`.
  - file: [src/config/sessions/store-load.ts:151](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/src/config/sessions/store-load.ts#L151)
  - command: `nl -ba src/config/sessions/store-load.ts | sed -n '70,165p'`
  - sha: [6bc0dc8fb671](https://github.com/openclaw/openclaw/commit/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440)
- **Hot write paths still use `skipCache: true`:** Both `updateSessionStore` and `updateSessionStoreEntry` re-read from disk with `loadSessionStore(storePath, { skipCache: true })` inside the lock, then mutate the returned store and save it. That matches the PR description of a read-modify-write path where the extra clone is on the active path today.
  - file: [src/config/sessions/store.ts:428](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/src/config/sessions/store.ts#L428)

  - sha: [6bc0dc8fb671](https://github.com/openclaw/openclaw/commit/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440)
- **Cache protection exists on the normal cached path already:** The object cache clones on both cache read and cache write (`readSessionStoreCache` returns `structuredClone(cached.store)`, and `writeSessionStoreCache` stores `structuredClone(params.store)`). That supports the PR discussion's claim that alias protection is already handled for cached reads, while the `skipCache` path bypasses this cache entirely.
  - file: [src/config/sessions/store-cache.ts:55](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/src/config/sessions/store-cache.ts#L55)
