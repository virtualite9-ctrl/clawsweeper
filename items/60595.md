## Summary

Keep #60595 open. Current main still has the exact redundant clone the PR targets: `loadSessionStore(..., { skipCache: true })` bypasses cache reads and writes, but still returns `structuredClone(store)` unconditionally. The provided PR diff is a narrow two-file optimization with a regression test, and the related closed PRs are predecessor attempts rather than a merged or open canonical replacement.

## Best Possible Solution

Keep #60595 open for maintainer review, rebase/CI if needed, and land the narrow `opts.skipCache` early return plus the regression test unless a maintainer explicitly chooses a broader canonical session-store performance PR and links this one as superseded.
