## Summary

Keep #14526 open. Current main has manual backup creation/verification, update doctor checks, plugin sync failure handling, gateway restart/version/plugin-health diagnostics, and background auto-update timing. It still does not implement the requested opt-in safe-update workflow with automatic pre-update backup, retention policy, channel or real-agent smoke verification, or automatic rollback/restore. The linked openclaw-auto-update-kit is useful operator evidence/workaround, not a core implementation or superseding tracker.

## Best Possible Solution

Keep #14526 open until maintainers either implement an opt-in safe-update mode in the core update/auto-update flow or explicitly decide this operator workflow should remain external. A complete core path would likely extend `openclaw update` and `update.auto` with pre-update backup creation, retention limits, post-update gateway/status/channel/real-agent smoke checks, and controlled rollback/restore behavior built on the existing backup primitives, while coordinating with the narrower open backup and deep-health discussions.
