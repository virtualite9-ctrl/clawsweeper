## Summary

Current `main` already threads agent-scoped workspace media roots through the shared message-action path into Slack file uploads, matching the reported bug. The fix appears present on `main` in commit `51dd4f28`, with regression coverage in outbound dispatch, Slack send plumbing, and plugin SDK media loading tests.

## Evidence

- **Plugin-handled message actions now forward agent-scoped media roots:** `executeSendAction()` resolves agent-scoped outbound media access from the action params, then passes both `mediaLocalRoots` and `mediaReadFile` into channel dispatch. That directly covers `message` tool actions such as Slack `upload-file`/`filePath` instead of letting the channel fall back to default roots.
  - file: [src/infra/outbound/outbound-send-service.ts:79](https://github.com/openclaw/openclaw/blob/51dd4f28/src/infra/outbound/outbound-send-service.ts#L79)
  - command: `nl -ba src/infra/outbound/outbound-send-service.ts | sed -n '1,220p'; git blame -L 79,103 src/infra/outbound/outbound-send-service.ts`
  - sha: [51dd4f28](https://github.com/openclaw/openclaw/commit/51dd4f28)
- **Agent-scoped media access includes configured workspace roots:** `resolveAgentScopedOutboundMediaAccess()` derives `localRoots` from `getAgentScopedMediaLocalRootsForSources(...)` / `getAgentScopedMediaLocalRoots(...)`, resolves the agent workspace with `resolveAgentWorkspaceDir(...)`, and returns both `localRoots` and a scoped `readFile` capability.
  - file: [src/media/read-capability.ts:83](https://github.com/openclaw/openclaw/blob/51dd4f28/src/media/read-capability.ts#L83)
  - command: `nl -ba src/media/read-capability.ts | sed -n '1,180p'; git blame -L 83,131 src/media/read-capability.ts`
  - sha: [51dd4f28](https://github.com/openclaw/openclaw/commit/51dd4f28)
- **Slack upload path now passes explicit roots into outbound media loading:** `uploadSlackFile()` calls `loadOutboundMediaFromUrl()` with both `mediaAccess` and `mediaLocalRoots`, so Slack uploads no longer rely on `localRoots: undefined` in this path.
  - file: [extensions/slack/src/send.ts:255](https://github.com/openclaw/openclaw/blob/51dd4f28/extensions/slack/src/send.ts#L255)
  - command: `nl -ba extensions/slack/src/send.ts | sed -n '220,320p'; git blame -L 255,260 extensions/slack/src/send.ts`
  - sha: [51dd4f28](https://github.com/openclaw/openclaw/commit/51dd4f28)
- **Shared media loader preserves explicit local roots:** `buildOutboundMediaLoadOptions()` converts `mediaLocalRoots` / `mediaAccess.localRoots` into explicit `localRoots` for `loadWebMedia()`, which is the key behavior the issue said was missing in the Slack send path.
  - file: [src/media/load-options.ts:70](https://github.com/openclaw/openclaw/blob/51dd4f28/src/media/load-options.ts#L70)
  - command: `nl -ba src/media/load-options.ts | sed -n '1,180p'; git blame -L 70,115 src/media/load-options.ts`
  - sha: [51dd4f28](https://github.com/openclaw/openclaw/commit/51dd4f28)
- **Regression tests cover the fixed plumbing:** Tests now assert that plugin dispatch receives agent-scoped media roots, Slack `sendMedia` forwards `mediaLocalRoots`, and the plugin SDK forwards `mediaLocalRoots` into `loadWebMedia()` as explicit `localRoots`.
  - file: [src/infra/outbound/outbound-send-service.test.ts:464](https://github.com/openclaw/openclaw/blob/5b8bd6371c666a5de43c82305e0315771e8e49d5/src/infra/outbound/outbound-send-service.test.ts#L464)
