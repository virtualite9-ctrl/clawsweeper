## Summary

Keep #71784 open. Current main still routes `openclaw memory search` and in-conversation memory recall through a single live query embedding request without bounded retry for the reported transient transport failures. Related #44167 is useful but remains open and is batch/reindex-scoped, so it does not supersede this live-search bug.

## Best Possible Solution

Keep #71784 open. A conservative fix should add a narrow transient transport-error classifier and bounded retry/backoff to the live query embedding path, either in `embedQueryWithTimeout()` or the shared single-embed remote HTTP helper, while preserving immediate failures for auth/config errors. Add focused regression tests for query retries in `extensions/memory-core` and/or `src/memory-host-sdk`; keep #44167 as related batch/reindex work unless it is extended to cover live query embeddings too.
