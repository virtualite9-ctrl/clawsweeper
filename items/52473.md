## Summary

Current `main` does not auto-inject `SKILL.md` content for `/skill-name` invocations. Skill slash commands are still rewritten into a normal natural-language request, while the system prompt only instructs the model to read the selected `SKILL.md` on its own. The issue is recent, coherent, and still matches documented behavior, so there is not strong evidence to close it.

## Evidence

- **Slash skill commands still become plain model requests:** `src/auto-reply/reply/get-reply-inline-actions.ts:310-326` rewrites a skill command into text like `Use the "<skill>" skill for this request.` plus optional user input, then assigns that text to `ctx.Body`/`BodyForAgent`. There is no harness-side read of the skill file or injected skill content on this path.
  - file: [src/auto-reply/reply/get-reply-inline-actions.ts:310](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/auto-reply/reply/get-reply-inline-actions.ts#L310)
  - command: `nl -ba src/auto-reply/reply/get-reply-inline-actions.ts | sed -n '280,350p'`
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **System prompt still relies on model compliance:** `src/agents/system-prompt.ts:162-167` says the agent must scan `<available_skills>` and, if one clearly applies, `read its SKILL.md at <location>`. That is a soft instruction, not deterministic injection for slash-command use.
  - file: [src/agents/system-prompt.ts:162](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/agents/system-prompt.ts#L162)

- **Docs explicitly describe current behavior as forwarding to the model:** `docs/tools/slash-commands.md:203-206` states that skill commands are exposed as slash commands and that, by default, `skill commands are forwarded to the model as a normal request`; only `command-dispatch: tool` is deterministic.
  - file: [docs/tools/slash-commands.md:203](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/docs/tools/slash-commands.md#L203)
