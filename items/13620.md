## Summary

Keep #13620 open. Current main still lacks outbound Telegram album/media-group support: Telegram reply delivery fans out `mediaUrls` through single-media API calls, direct Telegram message actions accept only one `mediaUrl`/`media`, outbound payload delivery sequences each media URL separately, tests assert that sequential behavior, and the Telegram docs only document a single `mediaUrl`. The linked PRs #21309, #51449, and #51457 are useful prior art, but the provided GitHub context shows them closed unmerged and their referenced commits are not present in the current checkout.

## Best Possible Solution

Keep #13620 open and implement album support in the bundled Telegram plugin. The best path is a guarded `sendMediaGroup` flow for 2-10 groupable photos/videos across reply delivery, direct Telegram message actions, and outbound payload delivery, while preserving current single-item fallback for one item, GIF/audio/document handling, caption overflow, buttons, reply/thread/silent/pin behavior, and non-Telegram channels. Use #21309/#51449/#51457 as prior art, then add focused Telegram tests and user-facing docs for the resulting `mediaUrls` or album semantics.
