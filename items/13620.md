## Summary

Keep #13620 open. Current main still does not implement outbound Telegram album/media-group sending: reply delivery, direct Telegram sends, shared outbound helpers, tests, and docs continue to treat multiple media items as sequential one-by-one sends. Prior attempts #21309, #51449, and #51457 are useful implementation history but were closed unmerged, so they do not supersede the issue.

## Best Possible Solution

Keep #13620 open and implement outbound album support in the bundled Telegram plugin. The likely path is a sendMediaGroup flow for 2-10 groupable photos/videos across reply delivery, direct Telegram sends, and outbound payload delivery, with existing one-by-one behavior retained as fallback. Add focused Telegram tests and docs for the user-facing mediaUrls/mediaGroup/album parameter semantics.
