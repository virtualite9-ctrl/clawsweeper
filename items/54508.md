## Summary

Keep this issue open. Current main has useful external-content wrapping, input-provenance primitives, plugin tool hooks, and exec approval controls, but it does not implement the requested end-to-end taint persistence, content-derived execution prohibition, general exec risk classes, or provenance-aware secret/outbound enforcement. The issue comment usefully links narrower related threads, but there is no single merged canonical item that supersedes the umbrella request.

## Best Possible Solution

Keep this issue open as the architecture umbrella, or have maintainers explicitly split it into canonical implementation issues. The likely path is to carry provenance metadata through external content, tool results, summaries, persistence, and future reads; enrich `before_tool_call` with stable action/risk/provenance fields; and then enforce content-derived execution, secret access, and outbound action policy in the core tool/approval paths while linking narrower follow-ups such as #48503, #48519, #41140, #47492, and #54373.
