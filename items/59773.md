## Summary

Current `main` already implements Anthropic OAuth quota fetching and surfaces it through the existing status path used by `session_status`; the implementation is present in release `v2026.4.23`.

## Evidence

- **Anthropic OAuth quota fetch is implemented:** `fetchClaudeUsage()` calls Anthropic's OAuth usage endpoint, sends the `anthropic-beta: oauth-2025-04-20` header, maps `five_hour` / `seven_day` into usage windows, and keeps the bearer token internal to OpenClaw.
  - file: [src/infra/provider-usage.fetch.claude.ts:115](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/src/infra/provider-usage.fetch.claude.ts#L115)
  - sha: [ec3dbd22a4de](https://github.com/openclaw/openclaw/commit/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46)
- **Claude missing-scope fallback is implemented:** When the OAuth token lacks `user:profile`, OpenClaw falls back to a claude.ai session key/cookie and still returns usage windows instead of exposing credentials to the agent.
  - file: [src/infra/provider-usage.fetch.claude.ts:149](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/src/infra/provider-usage.fetch.claude.ts#L149)
  - sha: [ec3dbd22a4de](https://github.com/openclaw/openclaw/commit/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46)
- **`session_status` already surfaces provider usage:** Status rendering resolves the current usage provider, loads provider usage for it, and emits a `📊 Usage:` line when usage windows are available.
  - file: [src/status/status-text.ts:185](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/src/status/status-text.ts#L185)
  - sha: [ec3dbd22a4de](https://github.com/openclaw/openclaw/commit/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46)
- **Agent-facing docs match the requested behavior:** Docs say `session_status` is the `/status`-equivalent tool and that it reports usage; usage-tracking docs also state provider usage shows up in `/status` for the current provider.
  - file: [docs/concepts/session-tool.md:81](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/docs/concepts/session-tool.md#L81)
  - sha: [ec3dbd22a4de](https://github.com/openclaw/openclaw/commit/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46)
- **Tests cover Anthropic OAuth usage and web fallback:** Tests assert parsing of OAuth usage windows and the claude.ai fallback path when the OAuth token is missing `user:profile` scope.
  - file: [src/infra/provider-usage.fetch.claude.test.ts:56](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/src/infra/provider-usage.fetch.claude.test.ts#L56)
