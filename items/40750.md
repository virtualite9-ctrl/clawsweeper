## Summary

Current main already contains Telegram DM preview-routing and preview-finalization logic that addresses the reported stale-draft-after-final race, and the same implementation is present in release v2026.4.22.

## Evidence

- **DMs no longer use draft-preview transport for answer streaming:** The Telegram dispatcher explicitly switches DM answer previews to message-edit preview transport instead of draft transport, with an inline note that DM draft previews duplicate at materialize time. That removes the pinned-banner draft path described in the report.
  - file: [extensions/telegram/src/bot-message-dispatch.ts:334](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/extensions/telegram/src/bot-message-dispatch.ts#L334)
  - command: `nl -ba extensions/telegram/src/bot-message-dispatch.ts | sed -n '330,347p'`
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)
- **Late/superseded previews are archived and rotated instead of rebound:** When a new assistant message starts, the dispatcher materializes the prior preview, archives its message id/text snapshot, then forces a new preview generation. If a late preview id resolves after rotation, it is captured through onSupersededPreview rather than reused for the new answer.
  - file: [extensions/telegram/src/bot-message-dispatch.ts:350](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/extensions/telegram/src/bot-message-dispatch.ts#L350)

## Close Comment

Closing from a Codex review because this behavior is already implemented in current OpenClaw and is present in release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`). In the Telegram dispatcher, DM answer previews are routed through message-edit preview transport instead of DM draft transport (`extensions/telegram/src/bot-message-dispatch.ts:334`), and the answer-lane finalization path archives superseded previews and applies finals back to the matching preview message (`extensions/telegram/src/bot-message-dispatch.ts:350`, `extensions/telegram/src/bot-message-dispatch.ts:455`, `extensions/telegram/src/lane-delivery-text-deliverer.ts:419`). The regression case is also covered by tests that simulate a late preview-id resolution after a message boundary and assert that finals land on the correct preview ids without fallback duplicates (`extensions/telegram/src/bot-message-dispatch.test.ts:1492`).
