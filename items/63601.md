## Summary

Close as duplicate. Issue #61432 is still open and tracks the same remaining bug: `message:received` hooks can push to `event.messages`, but current `main` still fires that hook through `fireAndForgetHook(...)` without retaining or routing the event messages. The linked fix PRs #63795 and #63849 were closed unmerged, so they do not implement the fix on `main`.

## Best Possible Solution

Close #63601 as a duplicate of #61432. The eventual fix should be tracked there and should update `dispatchReplyFromConfig` to retain and await the `message:received` hook event, route any pushed `hookEvent.messages` to the originating channel, and add regression coverage for pushed replies.
