## Summary

Keep this PR open. Current main has shipped partial mitigations for sticky auto-failover and has a bounded overflow retry guard, but it still lacks the PR's dedicated failover state and the /new-vs-/reset LCM conversation identity split. The PR should be rebased and narrowed rather than closed as obsolete.

## Best Possible Solution

Keep this PR open, but require a rebase and probably split it into focused changes: preserve the auto-override cleanup already on main, decide whether a dedicated failoverState is still needed, implement the missing LCM conversation identity seam consistently across context-engine lifecycle calls, and keep only overflow diagnostics or circuit-breaker changes that add behavior beyond the existing bounded overflow guard.
