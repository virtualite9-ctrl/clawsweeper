## Summary

Keep open. Current main still does not strip a standalone trailing `NO_REPLY` in the gateway live-chat/TUI projection path. Exact control-token suppression, lead-fragment suppression, and glued leading-token stripping exist, and a shared helper already supports terminal-token stripping elsewhere, but that helper is not applied to gateway chat projection.

## Best Possible Solution

Keep this issue open and implement the missing gateway chat/display cleanup in `projectLiveAssistantBufferedText` or a shared display normalizer used by that path. Add regression tests for exact suppression, separated leading malformed output, glued-leading output, trailing suffix, newline suffix, mixed case, and non-trailing/embedded explanatory mentions; include the TUI embedded backend if it continues to share the projector.
