## Summary

Current `main` still does not implement the ACP-specific session model reporting this PR targets, and the PR discussion already identifies a correctness regression in the proposed patch. Conservative cleanup should keep this PR open for maintainer judgment rather than auto-close it.

## Evidence

- **ACP metadata does store a runtime model:** `SessionAcpMeta.runtimeOptions.model` is part of the persisted session contract, so ACP session rows have a dedicated source of truth available outside the generic `model` / `modelProvider` fields.
  - file: [src/config/sessions/types.ts:41](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/src/config/sessions/types.ts#L41)
  - command: `nl -ba src/config/sessions/types.ts | sed -n '1,75p'`
  - sha: [c070509b7f8f](https://github.com/openclaw/openclaw/commit/c070509b7f8fdbe479e5d829643782c272e30e83)
- **Current resolver ignores ACP runtime metadata:** `resolveSessionModelIdentityRef` only consults `entry.model`, `entry.modelProvider`, `fallbackModelRef`, then falls back to `resolveSessionModelRef`. It has no ACP branch and never reads `entry.acp?.runtimeOptions?.model`, so the specific behavior requested by the PR is not implemented on `main`.
  - file: [src/gateway/session-utils.ts:1169](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/src/gateway/session-utils.ts#L1169)

DEFAULT_MODEL` before returning the row. For ACP sessions without generic runtime model fields, that still reports the default model in the sessions list.
