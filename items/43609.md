## Summary

Keep this PR open. Current main still lacks the central Grok/xAI web_search fixes, and the PR contains useful implementation direction, but it needs to be rebased/ported to the current xAI plugin layout and split away from unrelated status-command changes before maintainer review.

## Best Possible Solution

Keep this PR open as useful contributor work, but ask for a rebase and narrower patch. The maintainer path is to port only the Grok/xAI changes into `extensions/xai/src/responses-tool-shared.ts`, `extensions/xai/src/web-search-shared.ts`, and `extensions/xai/src/web-search-provider.runtime.ts`; use `isAbortError()` with `{ cause }` when wrapping internal timeout aborts; add a Grok-specific 60s default while preserving explicit user overrides; and add focused xAI regression tests. The unrelated `src/commands/status*` changes should be dropped or split into a separate PR.
