## Summary

Keep this PR open. Current main still lacks the central Grok web_search fixes: xAI web search uses the shared 30s timeout, the shared Responses parser still dereferences output/content/annotation entries without null-object guards, and AbortError can still escape the provider path. The PR contains useful work, but it should be rebased and narrowed because the current branch also carries unrelated status-command changes with review-noted regressions.

## Best Possible Solution

Keep the PR open, but ask for a rebase and a narrower patch. The best path is to drop or split the unrelated status changes, port the Grok fix to current main by adding null/object guards in extensions/xai/src/responses-tool-shared.ts, wrap xAI web-search timeout AbortErrors into plain Errors with cause using the repo's abort detection pattern, make the Grok provider default to 60s while preserving user timeout overrides, and add focused regression tests under extensions/xai web-search/Responses helper tests.
