## Summary

Keep open. Current `main` has landed the generic agent-sent TTS/tool-media delivery fix that matches the comment about TTS being broken across channels, but the MiniMax -> Feishu path in `main` still does not implement Feishu-native voice-note delivery: MiniMax synthesis remains `mp3`/`voiceCompatible: false`, and Feishu only routes `.opus`/`.ogg` as `msg_type: "audio"`.

## Evidence

- **Issue discussion points to a generic cross-channel TTS delivery bug:** Commenter Conan-Scott said agent-sent TTS looked broken on all channels and referenced issue #63033. That matches the code now on `main`, which adds a generalized tool-result media delivery pipeline rather than a Feishu-only fix.
- **TTS tool now emits structured media for delivery:** `tts` now returns audio in `details.media.mediaUrl` and only marks `audioAsVoice` when the provider reports `voiceCompatible`, which is the new generic delivery mechanism on `main`.
  - file: [src/agents/tools/tts-tool.ts:73](https://github.com/openclaw/openclaw/blob/5b8bd6371c666a5de43c82305e0315771e8e49d5/src/agents/tools/tts-tool.ts#L73)
  - command: `nl -ba src/agents/tools/tts-tool.ts | sed -n '68,100p'`
  - sha: [5b8bd6371c66](https://github.com/openclaw/openclaw/commit/5b8bd6371c666a5de43c82305e0315771e8e49d5)
- **Main now has generic tool-result media extraction and queuing:** The embedded runner now reads structured `details.media`, queues pending tool media, and merges it into the next outbound reply or a media-only reply. That is strong evidence the original 'nothing is delivered' regression was addressed generically on `main`.
  - file: [src/agents/pi-embedded-subscribe.tools.ts:240](https://github.com/openclaw/openclaw/blob/5b8bd6371c666a5de43c82305e0315771e8e49d5/src/agents/pi-embedded-subscribe.tools.ts#L240)
