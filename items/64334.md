## Summary

Keep this issue open. Current main still does not add a distinct cumulative session-cost line to `/status` or `session_status`; it only shows an API-key-gated estimated cost based on displayed input/output tokens. The transcript-backed cumulative cost data exists and `/usage cost` can summarize it, but that separate command path does not satisfy the requested `/status` output. The issue is also linked by open, unmerged PR #72517, so it should stay open until that PR is reviewed, landed, or closed.

## Best Possible Solution

Keep this issue open. Review PR #72517 as the active closing candidate; if it remains docs-only, request changes or close it. The product fix should add a cumulative `Session cost` line in the shared `/status` and `session_status` path, reusing transcript/session cost data, preserving existing cost-visibility rules, and adding focused tests for multi-turn transcript totals plus OAuth/token hiding.
