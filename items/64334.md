## Summary

Keep this issue open. Current main still shows only a token-derived estimated `Cost:` line in the shared `/status`/`session_status` renderer, while transcript-backed cumulative cost is available elsewhere but is not carried into the status card. The provided GitHub context also shows open, unmerged PR #72517 uses closing syntax for this issue, so the issue should remain open until that PR is reviewed, landed with a real implementation, or closed.

## Best Possible Solution

Keep the issue open. Review PR #72517 as the active closing candidate; if it remains README-only, request code changes or close it. The product fix should add a cumulative transcript-backed session cost line to the shared `/status` and `session_status` renderer, reuse the existing transcript/session cost aggregation, preserve OAuth cost-hiding behavior, and add focused tests for multi-turn totals and auth visibility.
