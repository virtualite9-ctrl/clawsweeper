## Summary

Keep this issue open. Current main still does not add a cumulative transcript-backed session cost line to `/status` or `session_status`; the shared status renderer only shows an estimated current token cost as `💵 Cost:`. The existing transcript/session cost data is available elsewhere and `/usage cost` already exposes a separate cost summary, but that does not satisfy the requested `/status` output.

## Best Possible Solution

Keep this issue open. Review PR #72517 as the active closing candidate, but require a real code change before it can resolve the issue. The product fix should add a cumulative transcript-backed session cost line to the shared `/status` and `session_status` renderer, reuse existing transcript/session cost aggregation, preserve API-key versus OAuth visibility rules, and add focused tests for multi-turn totals, transcript fallback, and cost visibility.
