## Summary

Current `main` does not implement a configurable `tools.image.mediaLocalRoots`-style setting, and the image tool still rejects host-local image paths outside its built-in roots. The iMessage attachment flow still forwards original attachment paths into `MediaPath`/`MediaPaths`, so the reported workflow remains plausible on current `main`.

## Evidence

- **No image-tool local-root config in schema:** `ToolsMediaSchema` exposes media-understanding settings (`models`, `concurrency`, `asyncCompletion`, `image`, `audio`, `video`) but no `mediaLocalRoots` or similar image-tool allowlist field.
  - file: [src/config/zod-schema.core.ts:791](https://github.com/openclaw/openclaw/blob/d38ed0831d7372c8505dc944aea69b5b83da1435/src/config/zod-schema.core.ts#L791)

  - sha: [d38ed0831d73](https://github.com/openclaw/openclaw/commit/d38ed0831d7372c8505dc944aea69b5b83da1435)
