## Summary

Keep #47856 open. Current main still lacks a configurable image-tool local-roots surface, and the image tool still routes non-sandbox local image reads through a fixed default/workspace allowlist. iMessage attachment-root handling and media-understanding support are relevant partial paths, but they do not satisfy explicit image-tool access to host attachment paths such as Messages attachments.

## Best Possible Solution

Keep the issue open and implement a deliberate, documented image-tool media root policy. The likely product path is either a privileged config surface such as `tools.media.image.localRoots`/`tools.image.mediaLocalRoots`, or channel-aware staging that rewrites trusted inbound attachment paths into managed `media://inbound/*` or workspace media paths before the image tool sees them. The implementation should align config types, zod schema/help/docs, and tests for iMessage Messages attachments, `workspaceOnly`, managed inbound media, and rejected out-of-root paths.
