## Summary

Current main/release already hardens the health path and bundled Feishu against SecretRef-backed credentials, but the report also includes @yanhaidao/wecom runtime/account-routing failures outside this repo, so there is not strong enough evidence that the full issue is resolved end-to-end.

## Evidence

- **Health snapshot degrades on account-resolution failures:** `getHealthSnapshot()` now catches `plugin.config.resolveAccount()` and `plugin.config.isConfigured()` errors, records diagnostics, and falls back to read-only inspection instead of crashing the health refresh.
  - file: [src/commands/health.ts:168](https://github.com/openclaw/openclaw/blob/55389f4be29359e2dd57ebaf548ba4a70331a445/src/commands/health.ts#L168)
- **Bundled Feishu no longer calls string methods on raw SecretRef objects:** Feishu account resolution uses `normalizeString()` plus `coerceSecretRef()` in `resolveFeishuSecretLike()`, and `resolveFeishuBaseCredentials()` reads `appId`/`appSecret` through that helper rather than calling `.trim()` on raw config values.
  - file: [extensions/feishu/src/accounts.ts:54](https://github.com/openclaw/openclaw/blob/55389f4be29359e2dd57ebaf548ba4a70331a445/extensions/feishu/src/accounts.ts#L54)
- **Bundled Feishu has SecretRef regression coverage:** Tests assert that Feishu status treats SecretRef-backed credentials as configured, including env-backed `appId`/`appSecret` cases.
  - file: [extensions/feishu/src/setup-surface.test.ts:105](https://github.com/openclaw/openclaw/blob/55389f4be29359e2dd57ebaf548ba4a70331a445/extensions/feishu/src/setup-surface.test.ts#L105)
- **Health path still executes plugin summary code without a guard:** After the fallback resolution step, `getHealthSnapshot()` still awaits `plugin.status.buildChannelSummary()` directly. For external plugins, this repo review cannot prove that plugin-owned summary/runtime code is SecretRef-safe.
  - file: [src/commands/health.ts:336](https://github.com/openclaw/openclaw/blob/55389f4be29359e2dd57ebaf548ba4a70331a445/src/commands/health.ts#L336)

## Risks / Open Questions

- The issue body includes `@yanhaidao/wecom@2.2.7` outbound/account-resolution failures, and that plugin is not in this checkout, so its current behavior cannot be verified here.
- The bundled Feishu path looks fixed, but the report is broader than Feishu and includes runtime routing behavior that may still depend on external plugin code.
