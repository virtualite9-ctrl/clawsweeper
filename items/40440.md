## Summary

Keep open. Current main still records mention-gated Telegram group history as text-only entries and later builds `MediaPaths` only from the current message plus reply media, so media sent before an @mention remains unavailable to the agent. The two related fix PRs were closed unmerged, and no equivalent implementation or changelog entry is present.

## Best Possible Solution

Keep this issue open and implement the Telegram plugin fix: extend the shared reply-history and `MsgContext.InboundHistory` contracts with media refs, persist or refresh media captured from skipped group messages beyond the current media TTL, merge history media into the later `MediaPaths`/`MediaTypes` payload when the bot is mentioned, and add a focused regression test for image-without-mention followed by @mention.
