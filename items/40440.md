## Summary

Keep open. Current main still records skipped mention-gated Telegram group messages as text-only history entries, and later Telegram context construction only forwards current-turn media plus reply media. The related implementation attempts #40470 and #40490 are closed unmerged, so there is no shipped fix to close this issue against.

## Best Possible Solution

Keep this issue open and implement a focused Telegram plugin/shared-contract fix: extend `HistoryEntry` and `MsgContext.InboundHistory` with media references, persist or refresh downloaded media for skipped group messages beyond the short inbound TTL, merge history media into the later Telegram `MediaPaths`/`MediaTypes` payload when the bot is mentioned, update prompt metadata if needed, and add a regression test for image-without-mention followed by an @mention asking about that image.
