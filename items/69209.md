## Summary

Keep #69209 open. Current main still does not expose or honor the requested public `plugins.entries.memory-core.config.dreaming.model` override: the bundled memory-core schema is closed without a `model` key, Dream Diary docs still say it uses the default runtime model, and the narrative subagent run path does not pass any model override. The PR is therefore not obsolete or implemented on main, though it should still be reviewed/rebased carefully because the provided context shows mergeability and fallback-review concerns.

## Best Possible Solution

Keep the PR open and either rebase/adapt #69209 or land an equivalent maintainer-reviewed implementation that adds `plugins.entries.memory-core.config.dreaming.model`, documents it, propagates it through host dreaming config into light/REM/deep Dream Diary subagent runs, and preserves a narrow fallback to the session default model when the configured override cannot be used.
