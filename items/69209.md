## Summary

Keep #69209 open. Current main at 4a3030df9efa72c44bab567a4390ac0c4876a73f still does not expose or thread a top-level `plugins.entries.memory-core.config.dreaming.model` override, while this PR directly addresses the linked Dream Diary reliability request in #67409. The PRtags duplicate note for #66583 is not enough to close this PR because the provided context cannot verify #66583 and the feature is not present on main.

## Best Possible Solution

Keep the PR open for maintainer review or land a successor implementation that adds the `memory-core` manifest/schema/UI hint, resolves top-level `dreaming.model` into dreaming execution defaults, forwards the model through short-term/deep/light/REM Dream Diary narrative runs, handles model startup and post-start failure fallback to the session default, and covers the behavior with host-SDK plus memory-core tests. If maintainers prefer a different canonical tracker, link this PR from the remaining open dreaming.model issue rather than closing it as already implemented.
