## Summary

Keep open. Current main does not yet provide the requested user-facing `plugins.entries.memory-core.config.dreaming.model` override, and Dream Diary narrative runs still do not pass a model to the subagent runtime. The provided PR remains a plausible implementation path for the linked feature request; it should not be auto-closed as implemented or superseded from the available evidence.

## Best Possible Solution

Keep a canonical implementation path open for memory-core Dream Diary model overrides. The best product/code direction is to expose `dreaming.model` in the memory-core manifest and docs, resolve it into dreaming execution defaults, forward it to all Dream Diary narrative subagent runs, and retain a narrow fallback to the session default when the override cannot start or is not authorized. If #69209 is not the branch to land, supersede it only with a linked active PR that implements the same behavior and closes the same user-facing gap.
