## Summary

Keep open. Current main supports remote embedding endpoints and local/QMD ranking paths, but it does not expose a `memorySearch.remote.rerank` config shape, does not call a Cohere-style `/v1/rerank` endpoint, and does not provide a focused plugin/provider reranker adapter that would make this a clean ClawHub close. The request is concrete, recent, unprotected, and not superseded by the provided related title matches.

## Best Possible Solution

Keep this open for a focused memory-search design. The likely path is an additive reranker contract in `memorySearch` plus schema, docs, and tests, with builtin support for a Cohere-style remote reranker and a clear QMD integration story. If maintainers prefer plugin ownership, first add a documented reranker provider API so Qwen, Cohere, Jina, and similar rerankers can live in provider plugins without replacing the whole memory backend.
