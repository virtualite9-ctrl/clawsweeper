## Summary

Keep open. Current main supports remote embedding endpoints and local/MMR/QMD reranking paths, but not the requested remote reranker endpoint contract. The proposed config shape is not accepted by the strict schema, no resolved reranker settings reach runtime, builtin search does not call a Cohere-style `/v1/rerank`, and the plugin SDK surface is for embedding providers rather than reranker providers.

## Best Possible Solution

Keep this open for a focused memory-search design. The likely implementation is an additive remote reranker contract in `memorySearch`, schema/help/docs updates, resolver support, a Cohere-compatible HTTP reranker path for the builtin backend, explicit vector-only fallback semantics, batching/timeout tests, and a clear QMD integration story. If maintainers prefer provider ownership, first add a documented reranker provider API so Qwen, Cohere, Jina, and similar adapters can live in plugins without replacing the whole memory backend.
