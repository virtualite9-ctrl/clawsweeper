## Summary

Keep this PR open. Current main still injects the Control UI gateway client id into chat.send MsgContext sender fields, so the linked prompt-noise regression is still real. The PR is not obsolete, but its implementation is too broad because removing SenderId/SenderName/SenderUsername for every gateway chat.send caller also removes command-layer sender identity used by authorization, run attribution, and sender-bound lifecycle/focus ownership checks.

## Best Possible Solution

Keep this PR open, but request a narrower fix. Preserve trusted gateway caller identity for command authorization, run attribution, trace/ownership decisions, and lifecycle/focus binding checks, while suppressing only synthetic Control UI/internal gateway sender fields from prompt-visible untrusted metadata. Add paired regression coverage: Control UI initiated prompts should not include the synthetic sender block, and a different gateway sender must still be rejected when updating or rebinding a sender-owned focused conversation.
