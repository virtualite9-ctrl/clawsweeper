## Summary

Keep open. Current `main` does not implement the requested global `auth.externalCliSync` / `OPENCLAW_AUTH_EXTERNAL_CLI_SYNC` opt-out. The concrete Codex example in the issue looks partly obsolete on current `main` because Codex CLI is no longer in the core external-CLI sync provider list and tests assert it is not used as a runtime overlay source, but the generic feature request is still unimplemented.

## Risks / Open Questions

- The issue's Codex-specific motivation appears outdated on current `main`; if this stays open, the scope should probably be reframed around remaining external CLI/plugin-managed external-auth sources rather than Codex specifically.
- Existing `auth.order`, per-agent overrides, and explicit `@profileId` selection already cover many multi-account routing cases, so product value for a new global disable knob may need reconfirmation.
