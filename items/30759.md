## Summary

Current main still does not implement the requested WebChat document upload/extraction. The WebChat composer remains image-only, accepted attachments are serialized as images, and the Gateway chat.send attachment parser still drops non-image payloads. The cross-referenced #70438 is useful related work, but it is not a complete superseding tracker for PDF, Word/doc/docx, and TXT extraction as requested here.

## Best Possible Solution

Keep #30759 open unless maintainers intentionally create or designate a broader canonical WebChat document-upload issue. A complete fix likely belongs across `ui/src/ui/chat/attachment-support.ts`, `ui/src/ui/views/chat.ts`, `ui/src/ui/controllers/chat.ts`, and the Gateway `chat.send` attachment parser: broaden accepted file types, add document metadata and size/MIME validation with user-visible errors, serialize documents distinctly from images, wire safe TXT/PDF/Word extraction or offload behavior, and add UI plus Gateway tests for TXT, PDF, DOC, and DOCX.
