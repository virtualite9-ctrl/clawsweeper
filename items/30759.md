## Summary

Keep #30759 open. Current main has materially improved since the earlier review: #72299 landed generic non-video WebChat uploads, media persistence, PDF/text extraction through the media-understanding path, and UI/history file badges. But the original request explicitly includes Word/DOC/DOCX document text extraction, and current main still skips Office-style application/vnd binary attachments instead of extracting their contents.

## Best Possible Solution

Keep #30759 open, or manually consolidate it into a canonical WebChat document-upload tracker that explicitly covers the remaining Word/DOC/DOCX extraction and user-facing document policy. The best code path is to build on #72299's shipped transport/persistence work, add a safe Word/DOCX extractor or an explicit unsupported-file error for those accepted extensions, align size/MIME limits and docs, and cover PDF/TXT/Word behavior with focused Control UI, Gateway, media-understanding, and document-extractor tests.
