## Summary

Current main still does not implement WebChat document upload/extraction for PDF, Word/DOC/DOCX, or TXT. The WebChat composer accepts only images, the Control UI serializes attachments as images, and the Gateway `chat.send` parser drops non-image payloads. Related issue #70438 is useful but remains open and narrower than this issue.

## Best Possible Solution

Keep #30759 open unless maintainers explicitly designate a broader canonical WebChat document-upload tracker. A complete fix likely belongs across the WebChat attachment support, chat view/controller serialization, Gateway `chat.send` attachment parsing, and document extraction/offload path: broaden accepted file types, add document metadata and size/MIME validation with user-visible errors, serialize documents distinctly from images, wire safe TXT/PDF/Word extraction or offload behavior, and add UI plus Gateway tests for TXT, PDF, DOC, and DOCX.
