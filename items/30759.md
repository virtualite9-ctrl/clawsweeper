## Summary

Current main still does not implement WebChat document upload/extraction for PDF, Word/DOC/DOCX, or TXT. The WebChat composer only accepts image MIME types, serializes accepted attachments as `type: "image"`, and the Gateway `chat.send` attachment parser drops non-image payloads. Existing PDF/document extraction support is on the OpenResponses `input_file`/document-extract path, not the WebChat `chat.send` upload path, and it does not cover Word/DOC/DOCX. Related #70438 is useful but remains open and narrower than this issue.

## Best Possible Solution

Keep #30759 open unless maintainers explicitly designate a broader canonical WebChat document-upload tracker. A complete fix likely needs coordinated changes in the WebChat composer/preview state, Control UI `chat.send` serialization, Gateway non-image attachment parsing, MIME/size validation and user-visible errors, TXT/PDF/DOC/DOCX extraction or offload behavior, and focused UI/Gateway tests. Reuse the existing OpenResponses/document-extract pipeline where it fits, but add the missing WebChat contract and Word/DOCX support rather than treating the current image-only path as complete.
