## Summary

Keep #39847 open. Current main still lacks the requested shared outbound safety net: the outbound payload planner parses raw reply text, `normalizeReplyPayloadsForDelivery()` just projects that plan, and Discord delivery does not provide a metadata-stripping hook. Main has upstream mitigation for normal reply normalization, but direct/shared outbound planning remains uncovered and related fix PRs are still open.

## Best Possible Solution

Keep #39847 as the canonical bug. Land a focused source-level fix in `src/infra/outbound/payloads.ts` that strips inbound metadata before directive parsing/formatting in the shared outbound plan, and add regression tests in `src/infra/outbound/payloads.test.ts` covering Conversation/Sender metadata plus MEDIA/reply/audio/channelData behavior. Then close or supersede the related open fix PRs with that rationale.
