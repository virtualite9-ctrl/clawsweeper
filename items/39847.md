## Summary

Keep #39847 open. Current main still lacks the requested shared outbound safety net: `createOutboundPayloadPlanEntry()` parses raw `payload.text`, `normalizeReplyPayloadsForDelivery()` only projects that plan, and Discord delivery has no metadata-stripping hook. Main has narrower mitigations in normal reply sanitization and Pi replay history, but direct/shared outbound planning remains uncovered.

## Best Possible Solution

Keep #39847 as the canonical bug. Land a focused source-level fix in `src/infra/outbound/payloads.ts` that strips inbound metadata before directive parsing/formatting in the shared outbound plan, with regression tests preserving MEDIA/reply/audio/channelData behavior; then close or supersede the related open fix PRs with that rationale.
