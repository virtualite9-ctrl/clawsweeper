## Summary

Current `main` already implements session-scoped model switching for persisted sessions, including Feishu-routed turns. The session patch path stores the selected model and marks a pending live switch, Feishu inbound dispatch forwards the persisted `SessionKey`, and the embedded runner applies the pending selection on the next safe retry boundary. The changelog records this as shipped by `v2026.4.1` and later aligned session-model selection further.
