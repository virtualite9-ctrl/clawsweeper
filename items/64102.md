## Summary

Keep this PR open. Current main still only makes security-scan plugin install errors terminal before hook-pack fallback when `--dangerously-force-unsafe-install` is set; normal no-flag `security_scan_blocked` and `security_scan_failed` results can still continue into hook-pack fallback and produce the misleading follow-up this PR targets.

## Best Possible Solution

Keep this PR open for maintainer review or adaptation. The best implementation is to make `security_scan_blocked` and `security_scan_failed` terminal before local-path, linked-path, and npm hook-pack fallback regardless of `--dangerously-force-unsafe-install`, while preserving non-security hook-pack fallback and bundled-plugin npm recovery. Keep or adapt the focused no-flag npm regression test.
