## Summary

Keep this PR open. Current main still only suppresses hook-pack fallback for security-scan plugin install errors when `--dangerously-force-unsafe-install` is set, while this PR targets the normal no-flag blocked npm install path too.

## Best Possible Solution

Keep this PR open for maintainer review or adaptation. The best path is to make `security_scan_blocked` and `security_scan_failed` terminal before hook-pack fallback for both local-path and npm plugin install flows, regardless of `--dangerously-force-unsafe-install`, while retaining fallback for non-security plugin install errors and keeping the focused no-flag npm regression test.
