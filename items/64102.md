## Summary

Keep open. Current `main` only short-circuits hook-pack fallback for `security_scan_blocked`/`security_scan_failed` when `--dangerously-force-unsafe-install` is set, so the broader CLI behavior requested by PR #64102 is not implemented on `main` yet.

## Evidence

- **Fallback skip is gated by the force-unsafe flag:** `runPluginInstallCommand` derives `forceUnsafeInstall` only from `opts.dangerouslyForceUnsafeInstall === true`, and the early-exit helper returns true only when that flag is set alongside a security-scan code.
  - file: [src/cli/plugins-install-command.ts:365](https://github.com/openclaw/openclaw/blob/5b8bd6371c666a5de43c82305e0315771e8e49d5/src/cli/plugins-install-command.ts#L365)
  - command: `nl -ba src/cli/plugins-install-command.ts | sed -n '198,206p'; nl -ba src/cli/plugins-install-command.ts | sed -n '364,365p'`
  - sha: [5b8bd6371c66](https://github.com/openclaw/openclaw/commit/5b8bd6371c666a5de43c82305e0315771e8e49d5)
- **Normal npm security-scan failures still fall through to hook-pack fallback:** On npm install failure, if the force-unsafe-only guard does not trigger, current `main` still calls `tryInstallHookPackFromNpmSpec(...)` and formats the combined `Also not a valid hook pack` error.
  - file: [src/cli/plugins-install-command.ts:577](https://github.com/openclaw/openclaw/blob/5b8bd6371c666a5de43c82305e0315771e8e49d5/src/cli/plugins-install-command.ts#L577)
