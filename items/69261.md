## Summary

Keep PR #69261 open. Current main still has the unscoped Ollama embedding auth path: it resolves remote/provider/env keys without checking the resolved host and then attaches `Authorization` to whichever embedding base URL is used. The PR addresses a real remaining bug, though its review comments should be handled before merge.

## Best Possible Solution

Keep the PR open, rebase it onto current main, preserve the embedding-path host-scoped auth behavior, and address the provider opt-out review with focused regression coverage for remote/provider/env precedence and placeholder marker filtering.
