## Summary

Keep PR #69261 open. Current main still resolves Ollama embedding auth as remote apiKey, then provider apiKey, then env OLLAMA_API_KEY without checking the resolved request host, and it still unconditionally attaches any resolved key as Authorization. The PR is a focused fix for a real remaining bundled Ollama plugin bug, though its review comments should be addressed before merge.

## Best Possible Solution

Keep the PR open and have the contributor rebase onto current main, address the provider opt-out edge case with a regression test, and land the host-scoped auth selection plus marker filtering in `extensions/ollama/src/embedding-provider.ts`. The older `/api/embeddings` migration work is separate and should remain tracked by its own Ollama embedding endpoint issues/PRs.
