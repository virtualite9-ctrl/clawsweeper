## Summary

Keep PR #69261 open. Current main still has the reported Ollama embedding auth-scoping bug: the embedding provider resolves remote, provider, then env auth without comparing the selected key to the selected base URL, then unconditionally attaches `Authorization: Bearer ...` to the embedding request. Existing tests also still assert marker/env behavior that this PR is trying to change. The PR is not obsolete, though it should address the review comments before merge.

## Best Possible Solution

Keep the PR open and finish it or land an equivalent patch in `extensions/ollama/src/embedding-provider.ts`: resolve remote/provider/env auth independently, filter non-secret markers, attach env `OLLAMA_API_KEY` only to `https://ollama.com`, attach provider auth only to the provider host, attach remote auth only to the remote base URL, honor explicit opt-outs, and add focused regression tests for local/cloud, cross-host remote, loopback equivalence, placeholders, and SecretRef cases. Address the existing review comments before merge.
