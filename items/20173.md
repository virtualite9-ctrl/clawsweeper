## Summary

Keep #20173 open. Current main still does not implement Discord edited-message reprocessing: the bundled Discord monitor handles message-create events only, listener startup does not register an update listener, the config/type/schema/docs surface lacks the requested opt-in knobs, and same-message replay dedupe would block a naive edit path.

## Best Possible Solution

Keep #20173 open and implement it in the bundled Discord plugin. Add opt-in config/type/schema/docs and UI-hint coverage, register a Discord message-update listener, normalize edited payloads through the existing Discord preflight and inbound worker pipeline, preserve bot/self/access filters, make replay dedupe edit-aware, add debounce or rate limiting per message, define in-flight cancellation/reply semantics, and cover listener registration, config validation, and message-handler behavior with regression tests.
