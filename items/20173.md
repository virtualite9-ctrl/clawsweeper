## Summary

Keep #20173 open. Current main still only wires Discord inbound user turns through the MessageCreate listener path; there is no registered message-update listener, no `handleEdits`/`editDebounceMs` config surface, and no docs/tests for reprocessing edited Discord user messages. The existing maintainer-side Codex comment from April 26, 2026 also explicitly kept this item open, and the request is concrete bundled Discord channel behavior rather than a ClawHub/plugin cleanup close.

## Best Possible Solution

Implement this in the bundled Discord plugin: add opt-in config/type/schema/docs for edited-message handling, register the supported Carbon/Discord `MESSAGE_UPDATE` listener path, normalize edited payloads through the existing Discord preflight and inbound worker pipeline, preserve bot/self/access filters, avoid the current message-id replay guard suppressing legitimate edits, debounce or rate-limit per edited message, define in-flight cancellation/reply behavior, and add listener/startup/config/message-handler regression tests.
