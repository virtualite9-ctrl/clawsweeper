## Summary

Keep #20173 open. Current main still routes Discord inbound user turns through the bundled MessageCreate listener only; there is no registered message-update listener, no edited-message config/docs surface, and the current replay key is based on account/channel/message id, so an edited Discord message would need explicit edit-aware handling before it can trigger a second agent turn.

## Best Possible Solution

Keep #20173 open and implement it in the bundled Discord plugin. Add opt-in config/type/schema/generated metadata/docs coverage, register a Carbon MessageUpdateListener path, normalize update payloads through the existing Discord preflight and inbound worker pipeline, preserve current bot/self/access/mention filters, make replay dedupe edit-aware, define debounce/rate-limit and in-flight cancellation behavior, and add focused regression tests for listener registration, config validation, and edited-message reprocessing.
