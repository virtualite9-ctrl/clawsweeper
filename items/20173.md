## Summary

Keep #20173 open. Current main still handles Discord inbound agent turns through Carbon's MessageCreateListener only; it does not import or register Carbon's MessageUpdateListener, and the strict Discord config/types surface has no handleEdits or editDebounceMs option. The issue is concrete, unprotected, and belongs in the bundled Discord plugin path rather than a ClawHub close.

## Best Possible Solution

Implement this in the bundled Discord plugin: add an opt-in Discord config/type/schema/docs surface for edited-message handling, register a Carbon MessageUpdateListener, normalize updated message payloads through the existing preflight/inbound worker path, preserve bot/self/access filters, add debounce/replay protection, and cover it with listener/startup/config/message-handler tests.
