## Summary

Keep this PR open. Current main still ignores `channels.telegram.textChunkLimit` in Telegram send-time HTML/plain-text chunk planning, while the PR is a narrow patch for that exact remaining gap. The existing discussion is consistent with the code read: the change should be reviewed and landed, or replaced with an equivalent Telegram-owned fix, rather than closed as obsolete.

## Best Possible Solution

Review and land this PR, or an equivalent Telegram-owned patch, after rebasing onto current main. The fix should compute the send-time chunk limit with `resolveTextChunkLimit(cfg, "telegram", account.accountId, { fallbackLimit: 4000 })`, clamp it to Telegram's 4096 API maximum, use that value for every HTML/plain-text branch in `buildChunkedTextPlan`, and keep focused regression coverage for both the 4096 cap case and a lower configured limit.
