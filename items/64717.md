## Summary

Keep this PR open. Current main still hardcodes Telegram send-time HTML/plain-text chunk planning to 4000 characters, while the PR is a narrow Telegram-owned fix for the documented `channels.telegram.textChunkLimit` config path.

## Best Possible Solution

Review and land this PR, or replace it with an equivalent Telegram-owned patch. The implementation should resolve `channels.telegram.textChunkLimit` through `resolveTextChunkLimit(cfg, "telegram", account.accountId, { fallbackLimit: 4000 })`, clamp it to Telegram's 4096-character API maximum, apply that value consistently through `buildChunkedTextPlan`, and keep focused regression coverage for both the 4096 cap and a lower configured limit.
