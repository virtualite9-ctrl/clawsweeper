## Summary

Keep PR #69344 open. Current main still has the original TUI long-token sanitizer behavior: non-copy-sensitive tokens are split at 32 characters, there is no dot/paren/bracket code-fragment exemption, and tests still assert the 32-character narrow-terminal guard. The PR is not implemented or fully superseded, but the review comments identify real behavior tradeoffs that need maintainer review rather than cleanup closure.

## Best Possible Solution

Keep #69344 open for maintainer review. The best path is a focused TUI formatter policy in `src/tui/tui-formatters.ts` that preserves genuinely copy-sensitive or code-like fragments without broadly disabling narrow-terminal protection, with regression tests in `src/tui/tui-formatters.test.ts` covering both code snippets and generic long-token wrapping. Coordinate with #39565/#39505 rather than closing this as implemented or superseded.
