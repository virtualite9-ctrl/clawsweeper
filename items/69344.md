## Summary

Keep PR #69344 open. Current main at 7e13f3f5140168e31027b39a9855ade3fadf24d5 still has the original TUI long-token sanitizer behavior: generic non-whitespace tokens are chunked at 32 characters, and there is no dot/paren/bracket code-fragment exemption. The PR is not implemented or safely superseded, but the review discussion raises real tradeoffs around weakening narrow-terminal protection, so this needs maintainer review rather than conservative cleanup closure.

## Best Possible Solution

Keep #69344 open for maintainer review and coordinate it with #39565/#39505 and adjacent TUI copy-safety work. The best implementation path is a focused `src/tui/tui-formatters.ts` policy that preserves genuinely copy-sensitive or code-like fragments without broadly disabling the narrow-terminal safeguard, backed by regression tests in `src/tui/tui-formatters.test.ts` for both code snippets and generic long-token wrapping.
