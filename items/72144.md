## Summary

Close #72144 as superseded. Current main already addresses the OpenClaw-specific multi-agent storm and duplicate-model-cache parts with sequential startup, a global QMD embed queue/file lock, failure backoff, and shared model-cache symlinking. The remaining no-op model initialization cost is upstream QMD work already linked in the issue at tobi/qmd#604, and the remaining OpenClaw-side embed-disable behavior is already tracked by open #63626.

## Best Possible Solution

Close this broad report as superseded. Keep the shipped OpenClaw queue/lock/shared-cache behavior, use #63626 as the canonical OpenClaw issue for embed-disable or preflight semantics, and follow tobi/qmd#604 for the upstream no-op model-initialization fix.
