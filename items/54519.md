## Summary

Current main does not provide the requested per-account read-only channel mode. The shared channel/account config types and WhatsApp account config have no `readOnly` outbound policy, and the gateway/direct outbound paths still deliver text, media, and structured payloads to channel adapters without a config-level send block. Related context points to PR #71382 as a WhatsApp-specific implementation candidate, but that is not present in the checked-out main and does not close this broader channel-level request yet.

## Best Possible Solution

Keep this issue open. The best product/code path is a generic channel delivery policy, likely on shared channel/account config, enforced before enqueueing or adapter handoff in the shared gateway/direct outbound path. It should cover text, media, structured payloads, polls/actions where applicable, include schema/docs/config metadata, and add regression tests for direct and gateway delivery plus at least one bundled account-based channel such as WhatsApp. Coordinate with PR #71382 if it remains the WhatsApp-specific implementation candidate.
