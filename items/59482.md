## Summary

The reported local-Ollama/Qwen hang maps to a timeout-compaction path that current OpenClaw now turns into an explicit timeout error instead of silently leaving the turn hanging. That behavior is present on `main`, covered by tests, documented for slow local models, and already shipped in `v2026.4.23`.

## Evidence

- **Reporter log matches timeout-compaction path:** The issue body shows `[timeout-compaction]` warnings, `compaction did not reduce context`, then `embedded_run_failover_decision` with `decision:"surface_error"`, `failoverReason:"timeout"`, `timedOut:true`, `aborted:true`. That is the exact timeout/failover path current OpenClaw handles explicitly instead of leaving the turn orphaned.
- **Current main emits an explicit timeout payload:** `runEmbeddedPiAgent` now documents and implements the non-hanging behavior: when a run times out without assistant payloads, it emits a visible timeout error so callers do not lose the turn as an orphaned user message.
  - file: [src/agents/pi-embedded-runner/run.ts:1776](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/src/agents/pi-embedded-runner/run.ts#L1776)
  - sha: [ec3dbd22a4de](https://github.com/openclaw/openclaw/commit/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46)
- **Timeout + compaction path is covered by tests:** The timeout-triggered compaction tests assert that when compaction fails or does not reduce context, the result still returns an error payload containing `timed out`; a separate loop test asserts the same for runs that time out before any reply is produced.
  - file: [src/agents/pi-embedded-runner/run.timeout-triggered-compaction.test.ts:199](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/src/agents/pi-embedded-runner/run.timeout-triggered-compaction.test.ts#L199)

## Risks / Open Questions

- I did not run a live Ollama/Qwen reproduction in this read-only review, so this close is based on the reporter's logs plus current code/tests rather than fresh runtime verification.
- The local model may still be too slow for the default idle timeout; the fix here is that OpenClaw now surfaces a timeout/error state and documents the tuning knob, not that Qwen 27B necessarily finishes the workload under default settings.
