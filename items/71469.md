## Summary

Keep open. Current main still has the reported Google Gemini CLI OAuth behavior: the local flow attempts `ctx.openUrl(authUrl)` first and only prints the authorization URL in the `catch` block. The injected `openUrl` path can resolve without throwing because the core browser opener returns `false` on unsupported, invalid, test, or failed launch cases and the provider auth wrapper discards that boolean. The PR is a focused fix for open issue #70964, and the provided maintainer discussion explicitly says it is valid on current main and worth landing.

## Best Possible Solution

Keep #71469 open and land this PR, or an equivalent patch, by moving the Gemini CLI OAuth URL log before the best-effort browser launch and keeping the focused local-flow regression test. This is a real bundled Google plugin bug, not a cleanup close, duplicate, or ClawHub candidate.
