## Summary

Keep #71469 open. Current main still has the Google Gemini CLI OAuth deadlock path the PR fixes: the auth URL is only printed if `ctx.openUrl(authUrl)` throws, while the real browser opener returns `false` on many failures instead of throwing. The PR remains a focused, useful fix with a regression test, and a maintainer already reviewed it as valid and worth landing.

## Best Possible Solution

Land #71469 or an equivalent scoped patch: print the Gemini CLI OAuth authorization URL before the best-effort browser launch, keep the local callback flow unchanged, and include the focused regression test for log-before-open ordering in the Google plugin OAuth path.
