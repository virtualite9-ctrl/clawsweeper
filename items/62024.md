## Summary

Keep open. Current main still contains the reported Discord explicit-target bug: `parseDiscordExplicitTarget` returns `target.id`, which strips `user:` from explicit DM targets before cron/heartbeat delivery. Downstream Discord outbound normalization then treats the bare numeric value as a channel target, while the send layer only creates a DM when it receives a user target. The linked PR #62052 is closed and unmerged, so it did not resolve this issue on main.

## Best Possible Solution

Apply the one-field Discord plugin fix in `extensions/discord/src/channel.ts` so `parseDiscordExplicitTarget` returns `target.normalized`, then add focused regression coverage through `discordPlugin.messaging.parseExplicitTarget` plus at least one cron or heartbeat delivery-target test for `user:<id>`. After that, run the relevant Discord/cron/heartbeat tests and `pnpm check:changed` when the environment has repo tooling available.
