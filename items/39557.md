## Summary

Current main already contains session-routing logic and regression coverage that address the reported webchat/per-channel-peer misrouting. The code now clears stale main-session delivery state when per-channel DM keys are active, preserves established external routes for channel-peer sessions viewed from webchat, and keeps webchat-owned routing only when no external route exists.

## Evidence
