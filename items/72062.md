## Summary

Keep #72062 open. Current main still has no declarative `agents.list[].tools.deny_wrappers` or equivalent exec script-path deny policy. `tools.deny` filters tool names, while exec allowlist processing can still satisfy a shell-wrapper invocation through an allowlisted shell executable before considering the script path. Current main does have positive shell-script allowlist support, but that is not the requested per-agent deny-wins policy.

## Best Possible Solution

Keep open. The best product path is a core exec policy addition, likely under `agents.list[].tools.exec` or `agents.list[].tools`, that resolves supported wrapper script operands before approval prompting and enforces deny-wins over broad shell executable allowlists, durable approvals, and safe-bin/skill shortcuts. The implementation should update config types, strict schema, generated schema/help/docs, gateway and node exec dispatch, and tests for `sh|bash <script>`, dispatch wrappers, `-c`/`-s` exclusions, per-agent inheritance, and denial messages.
