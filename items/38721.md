## Summary

Keep #38721 open. Current main still has bounded shutdown handling for websocket, HTTP, MCP runtime disposal, active queue tasks, and embedded runs, but I did not find a gateway shutdown path that cancels, detaches, lists, or drains arbitrary ProcessSupervisor-managed child processes. The provided maintainer discussion also explicitly says #61565 fixed a separate websocket close hang and does not supersede this child-process/pipe-handle report.

## Best Possible Solution

Keep #38721 as the canonical tracker for gateway shutdown blocked by active child-process or pipe handles. The best fix is to add an explicit gateway shutdown policy for supervisor-managed exec and CLI child processes: stop accepting new process work, expose ProcessSupervisor active-run cancellation or bounded shutdown-drain APIs, cancel or detach active runs with a grace window, and add regression coverage proving active child handles cannot trip the shutdown watchdog. Treat #61565 as an adjacent websocket shutdown mitigation, not closure evidence for this issue.
