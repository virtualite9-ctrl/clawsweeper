## Summary

Keep #38721 open. Current main has adjacent shutdown hardening for websocket close, MCP runtime disposal, and systemd child cleanup, but I did not find a gateway shutdown path that enumerates, cancels, detaches, or bounded-drains active ProcessSupervisor exec/CLI child processes. The maintainer discussion also explicitly distinguishes this report from #61565’s websocket fix.

## Best Possible Solution

Keep #38721 as the canonical tracker for gateway shutdown blocked by active child-process or pipe handles. The best fix is to add an explicit shutdown contract for supervisor-managed exec and CLI child processes: stop accepting new process work, expose active-run enumeration plus cancel-all or bounded drain APIs on ProcessSupervisor, call that path from gateway shutdown before the watchdog expires, and add regression coverage proving active child handles cannot trip the shutdown watchdog. Treat #61565 and the newer MCP/systemd hardening as adjacent mitigations, not closure evidence for this report.
