## Summary

Keep #38721 open. Current main still has the gateway shutdown watchdog and adjacent bounded cleanup for websocket, HTTP, MCP, and active agent work, but I did not find a shutdown contract that enumerates, cancels, or bounded-drains ProcessSupervisor-managed child/PTY runs. The issue discussion also explicitly distinguishes #61565 as a websocket-close fix, not a fix for the reported referenced child-process and pipe handles.

## Best Possible Solution

Keep #38721 as the canonical tracker for gateway shutdown blocked by active child-process or pipe handles. The best fix is to add an explicit ProcessSupervisor shutdown contract: stop accepting new process work, expose active-run enumeration plus cancel-all or bounded-drain APIs, call that path from gateway shutdown before the watchdog expires, add diagnostics that identify the active child command/run, and add regression coverage proving active supervisor child handles cannot trip the shutdown watchdog. Treat #61565 and the current websocket/MCP/HTTP hardening as adjacent mitigations, not closure evidence.
