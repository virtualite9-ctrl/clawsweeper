## Summary

PR #71740 adds a service-specific AssemblyAI audio transcription provider. Current main already has the public media-understanding provider/plugin surface needed for this to live as an external plugin, and OpenClaw's stated direction is to keep optional provider integrations out of core unless a missing plugin API is identified. The PR and its superseded predecessor #41489 are useful implementation evidence, but the best maintainer outcome is to move this to ClawHub/npm plugin distribution rather than add another bundled core plugin.

## Best Possible Solution

Close the bundled-core PR and invite the author to publish AssemblyAI as an external ClawHub/npm provider plugin using the existing media-understanding plugin API. Keep OpenClaw core work open only if that external implementation identifies a concrete missing SDK/API seam.
