## Summary

Current `main` already implements the requested fix, and the shipped `v2026.4.23` release contains the same auth-profile filtering and regression coverage. Explicit `mode: "token"` no longer leaves OAuth profiles eligible for provider resolution, so Telegram follows the same corrected core auth path as other interfaces.

## Evidence

- **Auth-profile eligibility now rejects OAuth under token mode:** `resolveAuthProfileEligibility` rejects a profile when configured mode and stored credential type differ, except for the one-way `oauth` config -> `token` credential compatibility. That means `mode: "token"` no longer treats OAuth credentials as eligible.
