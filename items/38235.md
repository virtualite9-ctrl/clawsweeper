## Summary

Keep #38235 open. Current main still resolves Slack agent identity and passes it through normal/fallback Slack delivery, but the draft preview streaming constructor has no identity parameter and its initial send omits identity. Native Slack streaming also starts chatStream without username/icon fields. The focused fix remains tracked by open, unmerged PR #38237; related PR #46432 is closed and unmerged.

## Best Possible Solution

Keep the issue open and land or adapt PR #38237: add an optional SlackSendIdentity to createSlackDraftStream, pass slackIdentity from dispatch, and add a regression test proving the initial draft chat.postMessage send receives identity. Separately verify whether Slack native chatStream can support custom authorship; if not, document that native-streaming limitation while preserving identity for normal and draft-preview delivery.
