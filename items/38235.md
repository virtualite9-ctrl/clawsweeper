## Summary

Keep #38235 open. Current main still resolves and forwards Slack agent identity for normal/fallback delivery, but the draft preview streaming constructor has no identity parameter and its initial send omits identity. Native Slack streaming is also started through the current wrapper without username/icon fields. The focused fix is still represented by open, unmerged PR #38237; closed PR #46432 did not land.

## Best Possible Solution

Keep #38235 open and land or adapt PR #38237: add an optional SlackSendIdentity to createSlackDraftStream, pass slackIdentity from dispatch, forward it into the initial draft sendMessageSlack call, and add a regression test proving the first draft chat.postMessage receives identity. Separately verify Slack native streaming authorship support; if the API cannot customize username/icon, document that limitation while preserving identity for normal and draft-preview delivery.
