## Summary

Keep #38235 open. Current main still resolves Slack agent identity and passes it through normal/fallback Slack delivery, but the draft preview stream still has no identity parameter and its initial send omits `identity`. The native Slack streaming helper also starts `client.chatStream` without identity fields. The focused fix remains tracked by open PR #38237; closed PR #46432 did not land.

## Best Possible Solution

Keep the issue open and land or adapt PR #38237: add optional Slack send identity to `createSlackDraftStream`, pass `slackIdentity` from dispatch, and add a regression test that the initial draft `chat.postMessage` receives identity. Separately verify the Slack native streaming API; if it cannot support custom authorship, document that limitation while ensuring draft-preview and normal delivery preserve agent identity.
