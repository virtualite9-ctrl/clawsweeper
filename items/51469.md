## Summary

Current main has mitigations for earlier 3s handshake failures, but it does not implement this issue's remaining request: the pre-auth timeout is still 10s, there is no `gateway.handshakeTimeoutMs` config field, and the related PR that proposed the 15s/configurable fix is closed unmerged. Conservative cleanup should leave this open for maintainer product judgment or a scoped follow-up fix.

## Best Possible Solution

Keep this issue open. Maintainers should either land a scoped fix along the #51503 direction, with bounded validation and docs for `gateway.handshakeTimeoutMs`, or explicitly decide after reproducible current-main timing evidence that the existing 10s default plus `OPENCLAW_HANDSHAKE_TIMEOUT_MS` and startup-import slimming are sufficient.
