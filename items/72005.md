## Summary

Keep PR #72005 open. Current main still lacks server-rendered outbound status footers, fabricated-footer stripping, and context-threshold warning state. The PR is the canonical implementation thread for the already-superseded issue #71984, but the supplied review comments identify concrete bugs that should be fixed before merge rather than used as reasons to close it as obsolete.

## Best Possible Solution

Keep #72005 open and request a focused revision: use processed content on both direct and gateway send paths, skip stateful mutation for dry-run sends, persist all superseded thresholds atomically, add regression tests for gateway/direct/dry-run/threshold-jump behavior, and document the final config. If maintainers prefer plugin ownership, first add a documented outbound/session-telemetry plugin contract that makes the behavior implementable outside core.
