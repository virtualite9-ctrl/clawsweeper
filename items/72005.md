## Summary

Keep #72005 open. Current main does not implement server-rendered outbound status footers or context-threshold warnings, and the PR is not merged; the useful direction remains actionable, but the PR needs changes for the review-identified gateway, dry-run, and threshold-state correctness issues before it should land.

## Best Possible Solution

Keep the PR open but request changes. The best path is a core outbound-pipeline implementation that strips fabricated runtime footers before all real deliveries, renders optional server-authored footers from session telemetry, documents the config, and covers direct, gateway, mirror, dry-run, skipped-threshold, and concurrent-send cases with focused tests. If maintainers do not want this as core behavior, the linked product issue should be explicitly re-triaged rather than closing this PR as already handled.
