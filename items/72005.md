## Summary

Keep #72005 open. Current main still lacks server-rendered outbound status footers and context-threshold warnings, and the PR is the canonical implementation thread for the closed parent issue #71984. It should not be closed as obsolete, but it also should not land unchanged because the provided review discussion identifies concrete gateway, dry-run, threshold-recording, and concurrent-send issues.

## Best Possible Solution

Keep #72005 open and request changes. The best path is a core outbound-pipeline implementation that strips fabricated runtime footers before all real deliveries, renders optional server-authored footers from session telemetry, avoids dry-run state mutation, handles direct and gateway paths consistently, records all crossed thresholds atomically, and adds focused tests for direct, gateway, mirror, dry-run, skipped-threshold, and concurrent-send behavior.
