## Summary

Keep #46531 open. Current main still has the central setup/runtime mismatch: active model-provider env SecretRefs are required at gateway startup, unresolved env refs throw, and startup still converts that into a fatal `Startup failed: required secrets are unavailable` error. Dry-run and optional shell-env import are useful mitigations, but they do not make normal model-assisted/config writes daemon-environment-safe or prevent a restart crash loop.

## Best Possible Solution

Keep the issue open and fix the model/provider setup boundary. A robust fix should preflight new SecretRefs against the durable service environment used by daemon installs (`~/.openclaw/.env`, config `env`, and installed service env), offer to store the key in a daemon-visible location before saving, and avoid making unresolved non-selected/optional providers fatal at startup. Add regression coverage for a systemd-style env mismatch and for restart after adding a secondary Moonshot provider.
