## Summary

Keep #70529 open. Current main still has the built-in `user` existing-session browser profile and still launches Chrome DevTools MCP through bare `npx` by default; docs also still require the Gateway/node host to be able to spawn `npx chrome-devtools-mcp@latest`. The EasyClaw/Browser Relay extension path looks absent from current core, but the issue’s Desktop-specific `profile="user"` failure remains a real, separate bug. The provided maintainer comment explicitly says to keep this open as the packaged Desktop variant, overlapping but not superseded by #62288/#63926.

## Best Possible Solution

Keep the issue open and fix the Desktop/browser runtime path directly: existing-session attach should not depend on ambient `npx` in the packaged app. Either bundle/resolve a Chrome DevTools MCP driver explicitly for Desktop or produce a structured diagnostic that identifies missing `npx`, recommends the managed `openclaw` profile, and links the login handoff/browser management path. Keep #62288 for broader existing-session brittleness and #63926 for visible login handoff UX; #70529 should track the packaged Desktop failure mode.
