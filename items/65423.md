## Summary

Keep this PR open. Current main still passes embedded-run auth profile candidates in deterministic order, so the subagent thundering-herd behavior described by the PR is not implemented or superseded on main. The PR is a focused implementation candidate, but it needs revision before merge because the companion test duplicates the private shuffle helper and the patch needs to preserve current provider-preferred profile ordering semantics.

## Best Possible Solution

Keep the PR open and ask for a small rebase/revision: extract or add a production-owned, testable candidate-order helper, apply dispersion only to automatic spawned embedded runs, preserve locked-profile and provider-preferred profile semantics, and replace the copied helper test with regression coverage for the actual embedded-run candidate selection behavior.
