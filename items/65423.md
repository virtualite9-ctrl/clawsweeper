## Summary

Keep this PR open. Current main still builds embedded auth profile candidates deterministically, so the subagent thundering-herd behavior described by the PR is not implemented or superseded. The PR is a focused implementation candidate, but it needs revision because the added test duplicates the shuffle helper instead of exercising production behavior, and the patch must be rebased around current provider-preferred auth profile ordering.

## Best Possible Solution

Keep this PR open and ask for a small rebase/revision: add a production-owned, testable candidate-order helper or integration-level runner assertion; apply dispersion only to automatic spawned embedded runs; preserve locked-profile and provider-preferred profile semantics; and replace the copied shuffle-helper test with deterministic regression coverage for actual embedded-run candidate selection.
