## Summary

Keep open. Current `main` still hard-codes source-layout `../config/config.js` lookups across the logging stack, and the runtime postbuild step only writes stable aliases for `*.runtime`/`*.contract` chunks. The flattened-build gap described in PR #70335 does not appear implemented or obsolete on `main`.

## Evidence

- **`readLoggingConfig()` still assumes a source-layout config module:** `src/logging/config.ts` still calls `requireConfig?.("../config/config.js")` and returns `undefined` on failure. There is no shared logging config loader or hashed-sibling fallback in current `main`.
