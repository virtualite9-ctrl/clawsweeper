## Summary

Keep #39141 open. Current main still lacks the prerequisite session-scoped activity API/WS event requested in #39127 and does not implement an opt-in session activity watchdog or alert engine. Existing channel health, heartbeat, cron, CLI no-output, and subagent lifecycle mechanisms are adjacent but do not provide session activity/attention state, stale-run alerts, dedupe/cooldown policy, or operator/webhook delivery for sessions.

## Best Possible Solution

Keep #39141 open until #39127 or an equivalent session-scoped activity API lands. Then implement or expose a plugin-accessible opt-in watchdog that consumes that activity state, supports no-token structured alerts, configurable thresholds, dedupe/cooldowns, scoped operator/channel/webhook delivery, optional tokenized summaries, docs, and focused tests.
