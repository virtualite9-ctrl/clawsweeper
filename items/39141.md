## Summary

Keep #39141 open. Current main still lacks the prerequisite session-scoped activity/attention RPC and WebSocket state-change surface from #39127, and it does not implement the requested opt-in gateway watchdog, alert dedupe/cooldowns, operator delivery, or optional tokenized summaries. Existing channel health, diagnostics stuck-session warnings, cron failure alerts, task audit, and subagent orphan recovery are adjacent protections, not substitutes for the requested session activity watchdog.

## Best Possible Solution

Keep #39141 open until #39127, or an equivalent session-scoped activity/attention API with WS statechange events, lands. Then implement an opt-in gateway watchdog or plugin-accessible watchdog surface that consumes that reliable activity state, supports no-token structured alerts, configurable thresholds, dedupe/cooldowns, scoped operator/channel/webhook delivery, optional tokenized summaries, docs, and focused tests. Reuse existing diagnostics, cron alerts, channel health, task audit, and subagent recovery only where they fit; they do not currently satisfy the requested product behavior.
