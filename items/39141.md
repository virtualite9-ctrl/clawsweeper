## Summary

Keep #39141 open. Current main still lacks the prerequisite session-scoped activity API/WS event from #39127 and does not implement the opt-in gateway watchdog/alert engine requested here. Existing channel health checks and background task audit are adjacent, but they are channel/task scoped and do not provide per-session activity/attention state, waiting-state thresholds, child-tree rollups, dedupe/cooldown alerting, or operator/webhook alert delivery for sessions.

## Best Possible Solution

Keep #39141 open until #39127, or an equivalent per-session activity/attention API, lands. Then either implement the opt-in gateway watchdog in core or expose a plugin-accessible watchdog surface that consumes session activity state, supports no-token structured alerts, configurable thresholds, dedupe/cooldowns, scoped operator/channel/webhook delivery, optional tokenized summaries, docs, and focused tests. Reuse existing task audit/channel health concepts where they fit, but do not treat them as a substitute for session-scoped activity state.
