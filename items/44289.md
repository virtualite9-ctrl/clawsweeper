## Summary

Keep #44289 open. Current main has a partial registry-derived matrix generator, but it only writes the JSON artifact; the SecretRef credential-surface Markdown blocks are still hand-authored and only parity-tested after the fact, so the central request is not fully implemented.

## Best Possible Solution

Keep the issue open and extend the existing credential-matrix builder into a gen/check path that updates both `docs/reference/secretref-user-supplied-credentials-matrix.json` and the marked supported/unsupported blocks in `docs/reference/secretref-credential-surface.md` from registry metadata. Expose that through package scripts and keep the parity test as a generated-output check.
