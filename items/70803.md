## Summary

Keep open. Current main still rejects WhatsApp `@hosted.lid` targets in the shared target normalizer even though the runtime JID-to-E.164 path already accepts `@hosted.lid`; PR #70803 remains a narrow, relevant bug fix with matching regression coverage.

## Best Possible Solution

Keep PR #70803 open and land the narrow WhatsApp plugin fix after normal validation: extend `WHATSAPP_LID_RE` in `extensions/whatsapp/src/normalize-target.ts` to accept `hosted.lid`, and keep the added lower/upper-case regression tests in `extensions/whatsapp/src/resolve-target.test.ts`.

## Evidence
