## Summary

Keep open. Current main still enforces the 2 MB local avatar cap, but the CLI and gateway agent config docs still describe avatar sources without documenting the local file-size limit. The provided GitHub context also shows an open PR, #72172, with closing syntax for this issue, so the maintainer path is to review and land or close that PR rather than close the issue now.

## Best Possible Solution

Keep the issue open while maintainers review #72172. The narrow fix should document that local workspace avatar image files must be at most 2 MB in `docs/cli/agents.md` and `docs/gateway/config-agents.md`, matching `AVATAR_MAX_BYTES`; earlier closed unmerged attempts can remain closed after one docs implementation lands.
