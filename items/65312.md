## Summary

Keep this issue open. Current main still enforces the 2 MB local avatar cap, but the user-facing CLI and gateway agent docs still describe avatar sources without documenting that local workspace image files over 2 MB are rejected. The provided GitHub context also shows open PR #72172 with closing syntax for this issue, so the maintainer path is to review and land or close that PR rather than close the issue now.

## Best Possible Solution

Keep the issue open while maintainers review #72172. The narrow fix should document that local workspace avatar image files must be at most 2 MB in `docs/cli/agents.md` and `docs/gateway/config-agents.md`, matching `AVATAR_MAX_BYTES`; after that PR or an equivalent docs change lands, the issue can close as implemented.
