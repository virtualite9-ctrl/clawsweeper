## Summary

Keep #36617 open. Current main has Japanese support for the browser Control UI and generated docs pipeline, but the issue asks for broader core UX localization across onboarding, help, templates, common labels, and English fallback. Those core CLI/template surfaces are still English-first and there is no shared core i18n implementation outside the UI bundle.

## Best Possible Solution

Keep #36617 open until maintainers either narrow it to the already-shipped Control UI/docs work or implement a shared core/key-based i18n path for CLI onboarding, help output, common labels, and workspace templates. The concrete path is locale resolution outside the browser UI, English fallback, translated ja-JP resources for the agreed surfaces, and focused tests for fallback, wizard prompts, help rendering, and template generation.
