## Summary

Keep #36617 open. Current main has real ja-JP support for the browser Control UI and a separate docs translation/publish pipeline, but the issue asks for first-class Japanese localization across core UX, explicitly including onboarding, help, templates, and English fallback behavior. Those core CLI/onboarding/template surfaces still use English literals directly and there is no shared core i18n layer beyond the UI implementation.

## Best Possible Solution

Keep #36617 open until maintainers either narrow the localization scope or add a shared core/key-based i18n path beyond the existing Control UI implementation. A good implementation path is to preserve the current UI ja-JP and docs pipeline, then add locale selection and English fallback for CLI onboarding/wizard/help text and workspace templates, with tests for locale resolution, fallback, and generated bundle/schema drift.
