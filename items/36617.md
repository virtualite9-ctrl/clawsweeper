## Summary

Keep #36617 open. Current main has Japanese localization for the browser Control UI and docs publish pipeline, but the issue explicitly asks for key-based i18n across core UX such as onboarding, help, templates, and common labels. Those CLI/wizard/template surfaces remain English-first and there is no shared non-UI core i18n layer in `src/`.

## Best Possible Solution

Keep #36617 open until maintainers either narrow it to the already-shipped Control UI/docs support or implement a shared core i18n path for CLI help, onboarding/wizard prompts, common labels, and workspace templates. A concrete path is locale resolution outside the browser UI, English fallback, `ja-JP` resource files for agreed core surfaces, and focused tests for fallback, help rendering, wizard prompts, and template generation.
