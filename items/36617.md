## Summary

Keep #36617 open. Current main has ja-JP support for the browser Control UI and a separate docs translation pipeline, but the requested core UX localization remains broader: CLI onboarding, help, templates, and shared English fallback outside the UI are still English-first literals with no shared core i18n layer.

## Best Possible Solution

Keep #36617 open until maintainers either narrow the request to the shipped Control UI/docs work or add a shared core/key-based i18n path for CLI onboarding, help output, and workspace templates. A good implementation path is locale selection plus English fallback outside the UI, with focused tests for locale resolution, fallback behavior, wizard prompts, and template generation.
