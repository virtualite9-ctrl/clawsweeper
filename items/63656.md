## Summary

Keep open. Current main still has the ACP lifecycle routing gap this PR targets: `/acp cancel` and `/acp close` are documented and tested as local ACP controls, but the ACP dispatch bypass helper still returns `false` for `/acp ...` slash commands, so bound-session command candidates can continue into ACP turn dispatch instead of local handlers. The PR is not implemented on main and remains a plausible focused fix, though maintainers should reconcile it with the newer duplicate PRs noted in the related context.

## Best Possible Solution

Keep one focused ACP routing PR open and land the canonical version that routes `/acp ...` bound-session lifecycle commands to local handlers under the existing text-command/native-command policy, with regression coverage for both allowed and disallowed command surfaces. After landing, close the duplicate ACP-bypass PRs as superseded by the merged fix.
