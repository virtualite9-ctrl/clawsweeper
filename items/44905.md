## Summary

Keep open. Current main contains partial fixes for this family, especially exact NO_REPLY normalization with media and commentary-phase suppression, but it still does not implement the universal outbound/internal-trace sanitizer requested here. Discord’s send path still forwards text into Discord delivery without a send-level silent/control-token guard, Discord’s outbound adapter does not provide a sanitizer, and REPLY_SKIP is still only a sessions_send control token rather than a channel-delivery sentinel.

## Best Possible Solution

Keep this issue open unless maintainers intentionally consolidate it under #25592. The best fix is a channel-agnostic user-visible delivery sanitizer plus Discord send-level defense, with regression coverage for Discord text, media captions, webhook/thread sends, sessions_send announce, exact control tokens, commentary-phase text, to=functions.* envelopes, and short raw tool-call JSON.
