## Summary

Keep open. Current `main` distinguishes scope-limited probe failures as degraded/reachable for `openclaw gateway probe`, but the `openclaw status` pipeline still treats those same results as unreachable, so the issue is not fully implemented on `main`.

## Evidence

- **`gateway probe` now treats missing `operator.read` as degraded reachability:** Docs explicitly say `Read probe: limited - missing scope: operator.read` is degraded reachability, not full failure, and the command test covers that behavior.
  - file: [docs/cli/gateway.md:227](https://github.com/openclaw/openclaw/blob/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4/docs/cli/gateway.md#L227)
  - command: `nl -ba docs/cli/gateway.md | sed -n '222,230p'`
  - sha: [f3c37a946cc4](https://github.com/openclaw/openclaw/commit/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4)
- **`gateway probe` test asserts reachable+degraded for the reported symptom:** `runGatewayStatus` JSON test feeds `ok: false`, `connectLatencyMs: 51`, `error: "missing scope: operator.read"` and expects the probe command to treat it as degraded but reachable.
  - file: [src/commands/gateway-status.test.ts:346](https://github.com/openclaw/openclaw/blob/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4/src/commands/gateway-status.test.ts#L346)

- **`status` still defines reachability as only `probe.ok === true`:** The shared status snapshot builder sets `gatewayReachable` from `gatewayProbe?.ok === true`, with no equivalent of the gateway-probe command's `isScopeLimitedProbeFailure()` fallback.
  - file: [src/commands/status.scan.shared.ts:131](https://github.com/openclaw/openclaw/blob/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4/src/commands/status.scan.shared.ts#L131)

- **`status` formatting still renders the false-negative path as unreachable:** Overview formatting prints `unreachable (<probe error>)` whenever `gatewayReachable` is false, so a scope-limited read-probe failure still surfaces as unreachable in `openclaw status`.
  - file: [src/commands/status-all/format.ts:362](https://github.com/openclaw/openclaw/blob/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4/src/commands/status-all/format.ts#L362)
