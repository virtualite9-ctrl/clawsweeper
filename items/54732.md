## Summary

Keep open. Current main still trims the `message` action enum and plugin-owned schema contributions, but the shared core parameter groups are still added unconditionally, so the reported schema bloat remains present.

## Best Possible Solution

Implement action-aware shared schema trimming in `src/agents/tools/message-tool.ts`: pass the resolved action set into the shared property builder, include each shared schema group only when at least one active action can use it, preserve stable routing/send/reaction/read fields and any required cross-channel/gateway contract, and add focused tests proving basic send/react configurations omit unsupported poll/sticker/thread/event/moderation/gateway/channel-management/presence fields while advanced actions still expose their required parameters.
