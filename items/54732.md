## Summary

Keep open. Current main still scopes the message tool action enum and plugin-provided schema contributions, but the shared core property groups are still action-blind and unconditionally included in the tool schema.

## Best Possible Solution

Keep this issue open and implement the requested action-aware core schema trim in `src/agents/tools/message-tool.ts`: pass the resolved action set into `buildMessageToolSchemaProps()`/`buildMessageToolSchemaFromActions()`, include each shared core schema group only when the active actions need it, preserve generic routing/send fields, and add focused tests for minimal send/react-only or disabled-action configurations that assert unsupported core fields are absent while supported cross-channel fields remain.
