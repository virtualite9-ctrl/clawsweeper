## Summary

Keep open. Current main still trims the message action enum and plugin-owned schema contributions, but the shared core schema groups are still spread into every configured message tool schema without looking at the resolved action set.

## Best Possible Solution

Implement action-aware shared schema trimming in `src/agents/tools/message-tool.ts`: pass the resolved action set into the shared property builder, include each shared group only when at least one active action can use it, preserve stable send/routing/reaction/read fields and any gateway contract that must remain global, and add focused tests proving basic send/react configurations omit unsupported poll/sticker/thread/event/moderation/gateway/channel-management/presence fields while advanced actions still expose required parameters.

## Risks / Open Questions

- Gateway override fields (`gatewayUrl`, `gatewayToken`, `timeoutMs`) are shared execution-path parameters currently read by the message tool runtime, so trimming that group needs explicit contract judgment instead of blanket removal.
- A fix should use a conservative action-to-shared-schema map so cross-channel actions discovered from configured plugins keep every parameter they require.
