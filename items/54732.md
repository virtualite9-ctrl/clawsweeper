## Summary

Keep this issue open. Current main still scopes the `message` action enum and plugin-provided schema contributions, but `buildMessageToolSchemaProps()` still unconditionally adds the shared poll, sticker, thread, event, moderation, gateway, channel-management, and presence parameter groups to every generated message tool schema.

## Best Possible Solution

Keep this issue open and implement action-aware shared schema trimming in `src/agents/tools/message-tool.ts`: thread the resolved action set into `buildMessageToolSchemaProps()`, include shared core schema groups only when active actions can use them, preserve always-needed routing/send fields and cross-channel behavior, and add focused tests for minimal send/react-only and disabled-action configurations that assert unsupported core fields are absent while supported fields remain.
