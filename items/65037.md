## Summary

Keep this PR open. Current main still lacks the PR's central durable pending-final-delivery payload state for subagent announce retries: run records have retry counters and delivery-error fields, but no typed pending payload, and cleanup retries still rebuild announces directly from the live run record. The PR discussion contains useful review feedback and author follow-up, so the maintainer path is review/rebase/consolidation rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review, rebase, or consolidation into the active durable subagent completion-delivery work. The concrete path is to add typed durable pending-final-delivery payload state to SubagentRunRecord, snapshot it at the first failed final-delivery boundary, reuse it across restart/retry while allowing safe live outcome/frozen-text refreshes, clear it on success/give-up/steer replacement, and split or drop unrelated auto-reply abort-drain changes if current main no longer needs them.
