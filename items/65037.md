## Summary

Keep this PR open. Current main still lacks the PR's central durable pending-final-delivery payload state for subagent announce retries; retry cleanup rebuilds delivery attempts from the live run record, and the run record has no pending-final-delivery fields. The PR is a concrete implementation candidate, with review feedback and follow-up commits already represented in the discussion, so it should be reviewed, rebased, or consolidated with the broader open durable completion-delivery work rather than cleanup-closed.

## Best Possible Solution

Keep this PR open for maintainer review, rebase, or consolidation into the active durable subagent completion delivery work. The likely product path is to add typed durable pending-final-delivery payload state to SubagentRunRecord, snapshot it at the first failed final-delivery boundary, reuse it across restart/retry while allowing safe live outcome/frozen-text refreshes, and clear it on success, give-up, and steer replacement. Split or drop unrelated auto-reply abort queue changes if the rebased branch no longer needs them.
