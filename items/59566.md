## Summary

Current `main` already protects `/status` against this leak: the task line is built from sanitized/truncated helpers, regression tests cover leaked internal runtime context and long unlabelled task prompts, and the same implementation is present in the shipped `v2026.4.23` release.

## Evidence

- **`/status` task line uses sanitized helpers:** `formatSessionTaskLine()` now renders the task summary with `formatTaskStatusTitle(task)` and `formatTaskStatusDetail(task)` rather than printing raw `task` text directly.
  - file: [src/status/status-text.ts:85](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/src/status/status-text.ts#L85)
  - sha: [ec3dbd22a4de](https://github.com/openclaw/openclaw/commit/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46)
- **Task title/detail sanitization and truncation:** Task-status formatting caps titles at 80 chars and details at 120 chars, sanitizes free-form text, and strips embedded internal runtime-context markers before display.
  - file: [src/tasks/task-status.ts:11](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/src/tasks/task-status.ts#L11)
