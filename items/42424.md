## Summary

Keep this issue open. Current main still lets Discord bot identity lookup fail to undefined, and the self-message guards remain conditional on that resolved id. Replay/history/followup paths still do not carry trusted Discord bot identity metadata that would allow a second self-filter, although current main does clear session queues on reset/new rollover.

## Best Possible Solution

Keep this issue open and implement a Discord-owned fix: add a reliable bot identity fallback or fail-closed startup behavior, carry trusted Discord bot identity through finalized inbound context/history/followup metadata, re-check self-authored Discord turns before pending-history and followup replay, and add regression tests for failed fetchUser("@me"), allowBots self-message ingress, steer-backlog followups, and the existing /new queue cleanup path.
