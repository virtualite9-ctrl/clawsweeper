## Summary

Keep open. Current main has partial live-ingress Discord self-message protection when `botUserId` is known, but the reported failure mode is not fully fixed: bot identity lookup still fails open to `undefined`, the replay/followup path does not carry or re-apply a `BotUserId` self-filter, and `/new` refreshes queued followups onto the new session instead of clearing contaminated queued work. The linked fix PR #46338 was closed unmerged, so it is not a shipped implementation candidate on main.

## Best Possible Solution

Keep this issue open and implement a focused Discord-owned fix: add a reliable bot identity fallback or fail-closed behavior for own messages when identity lookup fails, carry trusted bot identity into the finalized Discord inbound context, re-check self-authored messages before pending-history/followup replay, and clear or reject contaminated queued followups during `/new` and `/reset`. Add regression tests for failed `fetchUser("@me")`, pending-history replay, `steer-backlog` followups, and `/new` queue cleanup.
