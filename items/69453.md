## Summary

Keep #69453 open. Current main has improved and documented OpenAI Codex model metadata, but the reported consistency problem is not fully closed: GPT-5.4 still has a 1,050,000 native context window with a 272,000 effective runtime cap, read-only status/default paths still use synchronous lookup behavior, and stale persisted context values can remain authoritative.

## Best Possible Solution

Keep this open and narrow the next maintainer decision to context-resolution consistency. Decide whether `openai-codex/gpt-5.4` should consistently report/use 272k effective context or default to the 1.05M native window, then align fresh session defaults, status/session listing, stale persisted session repair, and model catalog display around that provider-aware contract with regression coverage for cold sessions, `/new`, stale 200k entries, `openclaw status`, and model-list/catalog output.
