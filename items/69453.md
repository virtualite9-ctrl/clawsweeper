## Summary

Keep #69453 open. Current main documents and exposes the native/effective OpenAI Codex context split, but the exact reported consistency problem is not fully closed: GPT-5.4 still has a 1,050,000 native window with a 272,000 effective runtime cap, synchronous read-only paths still avoid discovered model loading, and stale persisted session values can still override provider metadata.

## Best Possible Solution

Keep the issue open and narrow it to a context-resolution consistency decision: either consistently present/use the 272k effective cap with native 1.05M metadata, or intentionally raise the default runtime cap. Then align fresh session defaults, `/new`, session persistence, status/session listing, stale 200k repair, and model catalog display with regression coverage for cold sessions and persisted stale entries.
