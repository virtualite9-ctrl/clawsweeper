## Summary

Keep PR #27061 open. Current main still accepts empty or whitespace-only skill `name` and `description` frontmatter because `quick_validate.py` strips each value and only runs validation inside truthy guards. The current test file still lacks the empty/whitespace regression cases that this PR adds. Related PRs #34742 and #35102 are closed and unmerged, so they do not supersede the remaining fix.

## Best Possible Solution

Keep PR #27061 open, or land an equivalent narrow fix, until `skills/skill-creator/scripts/quick_validate.py` explicitly rejects empty and whitespace-only `name` and `description` after stripping and `skills/skill-creator/scripts/test_quick_validate.py` includes focused regression tests. Before merge, update the PR rationale to focus on invalid skill metadata and skill discovery/triggering behavior rather than the outdated archive filename claim.
