## Summary

Keep PR #27061 open. Current main still accepts empty or whitespace-only `name` and `description` values in `SKILL.md` frontmatter, and main still lacks the regression tests this PR adds. The core validation fix remains useful, though the PR rationale should correct the archive filename claim because packaging names archives from the skill directory, not frontmatter `name`.

Keep PR #27061 open, or land an equivalent narrow fix, until `skills/skill-creator/scripts/quick_validate.py` explicitly rejects empty and whitespace-only `name` and `description` after stripping and `skills/skill-creator/scripts/test_quick_validate.py` includes focused regression tests.
