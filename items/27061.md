## Summary

Keep open. Current main still has the validation gap this PR targets: blank or whitespace-only `name` and `description` values are stripped and then skipped by truthy guards, so they can still pass validation. The related hardening PR #35102 is closed and unmerged per the provided context, and current main does not contain equivalent regression tests.

## Best Possible Solution

Keep this PR or an equivalent narrow fix open until `quick_validate.py` rejects empty and whitespace-only `name` and `description` values, with focused regression tests in `skills/skill-creator/scripts/test_quick_validate.py`. A maintainer should also normalize the user-facing error wording before merge because related PR #35102 used different messages.
