## Summary

Keep PR #27061 open. Current main still strips `name` and `description` in `quick_validate.py` but only validates them inside truthy guards, so empty or whitespace-only values can still pass. The related superseding PR #35102 was closed unmerged, and the latest maintainer/Codex comment also explicitly kept this PR open. The PR body should narrow its downstream packaging claim, because current packaging names archives from the skill directory basename rather than frontmatter `name`.

## Best Possible Solution

Keep PR #27061 open, or land an equivalent narrow fix, until `skills/skill-creator/scripts/quick_validate.py` rejects empty and whitespace-only `name` and `description` after stripping and `skills/skill-creator/scripts/test_quick_validate.py` includes focused regression tests for those cases. Before merge, update the PR rationale so it focuses on invalid skill metadata and discovery/triggering behavior rather than the now-outdated archive filename claim.
