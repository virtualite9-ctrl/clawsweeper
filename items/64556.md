## Summary

Current main still has the reported wake-hook routing mismatch: wake mappings retain agentId/sessionKey in config, but the wake action and dispatcher drop them and enqueue to the main session. PR #70268 is still open and explicitly targets this issue, so this should remain open until that PR is reviewed and merged or closed.

## Best Possible Solution

Keep this issue open and treat PR #70268 as the active implementation candidate. The fix should thread agentId/sessionKey through wake mapping resolution and dispatch, preserve the existing main-session fallback when routing is omitted, reuse allowedAgentIds/sessionKey-prefix/templated-session guardrails, and keep regression coverage for explicit sessionKey, agentId-only wake routing, templated wake session keys, direct /hooks/wake behavior, and global-session-scope edge cases.
