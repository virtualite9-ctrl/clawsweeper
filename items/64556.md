## Summary

Current main still ignores wake mapping routing fields. The wake action type carries only text/mode, mapped wake dispatch passes only those fields, and dispatchWakeHook always resolves the main session. The provided GitHub context also shows open PR #70268 with closing syntax for this issue, so this issue should remain open until that PR is reviewed and merged or closed.

## Best Possible Solution

Keep this issue open and review PR #70268 as the active implementation candidate. The likely fix belongs in the gateway hook mapping and dispatch path: carry agentId/sessionKey on wake HookAction, render/merge mapping session keys consistently, route mapped wake through the same agent/session policy helpers used by agent hooks, preserve legacy main-session fallback when routing is omitted, and add regression tests for explicit sessionKey, agentId fallback, templated-session guardrails, and unchanged direct-wake behavior unless maintainers accept the broader direct endpoint API.
