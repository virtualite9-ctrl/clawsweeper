## Summary

Current main still ignores wake mapping routing fields, and the provided GitHub context shows open PR #70268 explicitly references this issue with closing syntax. This issue should stay open until that PR is reviewed and either merged or closed.

## Best Possible Solution

Keep this issue open and treat PR #70268 as the active implementation candidate. The best fix is to thread mapped wake agentId and sessionKey through HookAction, server-http, and dispatchWakeHook, reuse the existing hook session-key and agent routing policy, and add regression coverage for explicit mapped sessionKey, mapped agentId fallback, templated-session guardrails, and the legacy main-session fallback when routing is omitted.
