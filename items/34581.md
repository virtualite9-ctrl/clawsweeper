## Summary

Keep this PR open. Current main still strict-parses authenticated Microsoft Teams Bot Framework webhook bodies with `express.json()` and has no raw-body repair path for invalid JSON escape sequences, so the reported Azure backoff failure remains plausible. The PR is focused and useful, but it needs maintainer review/rebase because it changes security-sensitive webhook parsing and 200-ack behavior after later auth hardening on main.

## Best Possible Solution

Keep this PR open for Microsoft Teams maintainer review. The best path is a focused rebase in `extensions/msteams/src/monitor.ts` that preserves the current bearer and Bot Framework JWT gates before body parsing, keeps the existing body-size limit, repairs only truly bare invalid JSON escapes, returns HTTP 200 only for authenticated unrecoverable Bot Framework JSON to avoid Azure backoff, and adds integrated middleware tests for repair success, unrecoverable acknowledgement, adapter call/no-call behavior, ordering, and payload limits.
