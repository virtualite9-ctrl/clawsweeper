## Summary

Current `main` now hardcodes the OpenClaw tool/runtime guidance the issue asked for: the shared agent system prompt explicitly enumerates built-in infrastructure such as `browser`, `sessions_list`, `sessions_history`, and `sessions_send`, and it instructs agents to use first-class tools directly instead of asking the user for equivalent commands. That behavior is covered by prompt tests, and GPT-5-family runs also receive an added tool-discipline contract.

## Close Comment

Closing from a Codex review because current `main` already implements the hardcoded OpenClaw-runtime guidance this report asked for. The shared agent prompt now explicitly lists built-in infrastructure such as `browser`, `sessions_list`, `sessions_history`, and `sessions_send` and tells agents to use first-class tools directly instead of asking the user for equivalent commands (`src/agents/system-prompt.ts:679`, `src/agents/system-prompt.ts:729`). That behavior is covered by tests (`src/agents/system-prompt.test.ts:276`, `src/agents/system-prompt.test.ts:291`). GPT-5-family runs also get an added tool-discipline contract that tells the model to prefer tool evidence over recall and keep using tools when that would improve correctness (`src/agents/gpt5-prompt-overlay.ts:58`; documented in `docs/providers/openai.md:290`). If you still hit a concrete failure on a current build, please open a new issue with the exact prompt, model/provider, and transcript/tool output for the failing turn.
