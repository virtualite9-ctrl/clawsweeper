## Summary

Keep #71714 open. Current main has a normal Enter-to-send path, but the report is recent, concrete, and Windows Chrome/Edge-specific. The composer still has an IME/composition guard that can bypass preventDefault and allow the textarea newline behavior, and there is no focused regression coverage or configurable send shortcut shipped.

## Best Possible Solution

Add focused Control UI composer tests for ordinary Enter sending, Shift+Enter newline behavior, and IME/composition Enter not sending. Then verify the Windows Chrome/Edge path from the report and either refine the composition/keyCode 229 guard or add a documented configurable send-shortcut setting in the Control UI.
