## Summary

Keep #48920 open. The original `isolatedSession` config-key failure is obsolete on current main and the latest release, but the broader reported bug remains: live docs are still synchronized from `main`, not from the latest release, and I did not find a version-gating/version-availability mechanism in the docs source.

## Best Possible Solution

Keep the issue open, but narrow it to a docs-release contract: either publish user-facing docs from the latest released tag, add visible feature availability/version markers for config keys, or add validation/help text that explains when a known docs key requires a newer OpenClaw version. The heartbeat implementation itself does not need more work for `isolatedSession`.
