## Summary

Current main still matches the reported failure mode: Control UI busy/Stop state is gated by `chatRunId`, normal cleanup still depends on terminal chat events or reconnect/reset paths, and the embedded timeout path aborts the active session without directly guaranteeing a terminal chat/lifecycle event when `agent_end` is missed. The merged #69747 fix helps lifecycle errors that do arrive, but it does not close this missing-terminal-event edge case.

## Best Possible Solution

Keep this issue open unless maintainers explicitly consolidate it into #42213/#42258. The best fix is to guarantee a terminal chat/lifecycle state for embedded timeout/abort even when `agent_end` is missed, and add a Control UI recovery path such as an `agent.wait`-backed stale-run watchdog so `chatRunId` cannot keep Stop visible or block queued sends indefinitely. Add regression coverage for both the backend timeout/abort terminal emission and the connected Control UI stale-run recovery path.
