## Summary

Current main still has the reported stuck-Control-UI failure mode. The UI remains busy while `chatRunId` is set, only clears it on terminal chat events or reconnect/reset paths, and does not use the available `agent.wait` recovery path. The embedded timeout path aborts the run but does not directly synthesize a terminal chat/lifecycle event for Control UI. The later #69747 fix covers lifecycle error events that do arrive, but it does not close this missing-terminal-event edge case.

## Best Possible Solution

Keep this issue open unless maintainers explicitly consolidate it into #42213/#42258. The robust fix should guarantee a terminal chat/lifecycle state for embedded timeout/abort even when `agent_end` is missed, and add a Control UI recovery path such as an `agent.wait`-backed stale-run watchdog so `chatRunId` cannot keep Stop and queued sends blocked indefinitely.
