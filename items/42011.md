## Summary

Keep open. Current main still has the reported failure shape if an embedded timeout/abort completes without a terminal lifecycle event: Control UI stays busy on a non-null chatRunId, chat.send only synthesizes a final event for non-agent-started runs, and the embedded timeout path still aborts the active session without directly emitting a terminal lifecycle event. Related #42213/#42258 track the same stale Control UI chat-run recovery gap, while closed #59536 covered timeout leakage rather than this stuck Stop state.

## Best Possible Solution

Keep this issue open or explicitly consolidate it into #42213/#42258. The robust fix should guarantee a terminal chat/lifecycle state for embedded timeout/abort even when agent_end is missed, and add a Control UI recovery path such as an agent.wait-backed stale-run watchdog so chatRunId cannot keep Stop and queued sends blocked indefinitely.
