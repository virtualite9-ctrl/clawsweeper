## Summary

Keep open. Current `main` still has an unresolved failure mode consistent with the report: `loadCronStore()` only accepts object-shaped stores with a `jobs` array, so a legacy/plain-array `jobs.json` is silently treated as zero jobs, and `openclaw doctor --fix` cannot repair it because doctor uses the same loader and returns early when `rawJobs.length === 0`. That is not strong evidence for closure.

## Evidence

- **Cron loader still drops non-object stores to zero jobs:** `loadCronStore()` converts any parsed top-level array into `{}` and then reads `parsedRecord.jobs`; that yields `[]` with no warning, matching the reported `jobs: 0` symptom.
  - file: [src/cron/store.ts:164](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/cron/store.ts#L164)
  - command: `nl -ba src/cron/store.ts | sed -n '150,210p'`
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **Runtime normalization only happens after successful load:** `ensureLoaded()` normalizes legacy job fields only after `loadCronStore()` returns jobs. If the loader returns zero jobs, none of the compatibility normalization runs.
  - file: [src/cron/service/store.ts:37](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/cron/service/store.ts#L37)
  - command: `nl -ba src/cron/service/store.ts | sed -n '30,90p'`
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **Doctor repair cannot recover zero-loaded stores:** `maybeRepairLegacyCronStore()` calls `loadCronStore()` and exits immediately when `rawJobs.length === 0`, so `openclaw doctor --fix` would not repair the reporter's lost-job case once the loader has already dropped everything.
  - file: [src/commands/doctor-cron.ts:138](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/commands/doctor-cron.ts#L138)

  - file: [docs/gateway/doctor.md:296](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/docs/gateway/doctor.md#L296)
