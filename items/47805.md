## Summary

Keep open. Current main still has no session-import or transcript auto-discovery path for the Control UI/session menu. The Gateway session menu is backed by `sessions.list`, which loads `sessions.json` through `loadCombinedSessionStoreForGateway()` and maps only store entries; manually copied `.jsonl` transcript files are not registered there. The CLI `openclaw sessions` path also reads only resolved session stores, and docs describe sessions as store-backed. There is transcript discovery for usage aggregation, but that is limited to usage/session cost reporting and is not wired into `sessions.list`, Control UI sessions, `sessions.preview`, or chat navigation.

## Best Possible Solution

Keep the issue open and implement an explicit core import path, preferably `openclaw sessions import <path>` plus a Gateway/API helper that validates an OpenClaw/PI session JSONL, derives or assigns a session key, copies or adopts the transcript under the target agent sessions directory, writes a corresponding `sessions.json` entry with label/timestamps/sessionFile, and makes the result visible through existing `sessions.list`, `sessions.preview`, and Control UI chat navigation. Add focused tests for imported transcript registration, preview/history access, duplicate key handling, malformed JSONL rejection, and docs under the sessions CLI/concepts pages.
