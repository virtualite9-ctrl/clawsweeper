## Summary

Keep open. Current `main` still builds the session menu from `sessions.json`-backed session stores only; manually copied `.jsonl` transcripts are treated as orphan transcript files rather than importable/discoverable sessions.

## Evidence

- **Docs scope session discovery to session stores:** `docs/cli/sessions.md` says Gateway/ACP discovery includes disk-only stores only when they resolve to regular `sessions.json` files under agent roots; it does not describe raw `.jsonl` transcript discovery or import.
  - file: [docs/cli/sessions.md:29](https://github.com/openclaw/openclaw/blob/d38ed0831d7372c8505dc944aea69b5b83da1435/docs/cli/sessions.md#L29)
  - sha: [d38ed0831d73](https://github.com/openclaw/openclaw/commit/d38ed0831d7372c8505dc944aea69b5b83da1435)
- **Store discovery code only validates sessions.json:** `resolveValidatedDiscoveredStorePathSync()` constructs `path.join(sessionsDir, "sessions.json")` and only accepts that file as a discovered store. Raw transcript files are never registered here.
  - file: [src/config/sessions/targets.ts:63](https://github.com/openclaw/openclaw/blob/d38ed0831d7372c8505dc944aea69b5b83da1435/src/config/sessions/targets.ts#L63)
  - sha: [d38ed0831d73](https://github.com/openclaw/openclaw/commit/d38ed0831d7372c8505dc944aea69b5b83da1435)
- **Gateway session listing iterates loaded store entries:** `loadCombinedSessionStoreForGateway()` loads session stores, and `listSessionsFromStore()` enumerates `Object.entries(store)` to produce the sessions list. There is no transcript scan/import path feeding the menu.
  - file: [src/gateway/session-utils.ts:1011](https://github.com/openclaw/openclaw/blob/d38ed0831d7372c8505dc944aea69b5b83da1435/src/gateway/session-utils.ts#L1011)
  - sha: [d38ed0831d73](https://github.com/openclaw/openclaw/commit/d38ed0831d7372c8505dc944aea69b5b83da1435)
- **UI session menu calls sessions.list:** The Control UI `loadSessions()` path requests `sessions.list`, so the menu depends on the Gateway store-backed listing rather than a direct filesystem transcript scan.
  - file: [ui/src/ui/controllers/sessions.ts:139](https://github.com/openclaw/openclaw/blob/d38ed0831d7372c8505dc944aea69b5b83da1435/ui/src/ui/controllers/sessions.ts#L139)
  - sha: [d38ed0831d73](https://github.com/openclaw/openclaw/commit/d38ed0831d7372c8505dc944aea69b5b83da1435)
- **Orphan .jsonl files are explicitly treated as not active session history:** Doctor state integrity warns that `.jsonl` files not referenced by `sessions.json` are orphan transcript files and 'are not part of any active session history', which is the opposite of the requested import behavior.
  - file: [src/commands/doctor-state-integrity.ts:897](https://github.com/openclaw/openclaw/blob/d38ed0831d7372c8505dc944aea69b5b83da1435/src/commands/doctor-state-integrity.ts#L897)
  - sha: [d38ed0831d73](https://github.com/openclaw/openclaw/commit/d38ed0831d7372c8505dc944aea69b5b83da1435)
- **CLI surface has no import subcommand:** The `sessions` CLI registers listing and `cleanup` only. There is no `sessions import` or similar command on current main.
  - file: [src/cli/program/register.status-health-sessions.ts:124](https://github.com/openclaw/openclaw/blob/d38ed0831d7372c8505dc944aea69b5b83da1435/src/cli/program/register.status-health-sessions.ts#L124)
  - sha: [d38ed0831d73](https://github.com/openclaw/openclaw/commit/d38ed0831d7372c8505dc944aea69b5b83da1435)
