## Summary

Keep #71697 open. The related #66505 did implement direct Ollama Cloud setup, but current main still does not normalize native `api: "ollama"` agent/Gateway replay into strict user/assistant alternation before posting to `/api/chat`; the existing replay-policy hook only covers OpenAI-compatible Ollama routes, and the current tests explicitly assert no policy for native Ollama.

## Best Possible Solution

Keep the issue open and fix the native Ollama provider path. The likely maintainer path is an Ollama-owned replay policy or converter-level normalization for `api: "ollama"` that merges/reshapes non-alternating user and assistant turns while preserving supported native Ollama tool-call behavior, plus regression tests that simulate the agent/Gateway replay history which currently reaches `/api/chat` as non-alternating roles.
