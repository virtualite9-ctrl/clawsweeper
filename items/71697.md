## Summary

Keep #71697 open. Current main still documents and supports Cloud-only native Ollama configuration, but the native `api: "ollama"` agent/Gateway path does not normalize replayed message history into strict user/assistant alternation before sending `/api/chat`. The related #66505 direct-cloud setup work is separate and does not cover this replay-policy gap.

## Best Possible Solution

Keep the issue open and fix the native Ollama provider path. The likely maintainer fix is an Ollama-owned replay policy or converter-level normalization for `api: "ollama"` that reshapes non-alternating Gateway/agent history into a sequence accepted by native `/api/chat`, while preserving native Ollama tool-call behavior. Add regression coverage for the documented Cloud-only config and a replay history with non-alternating roles.
