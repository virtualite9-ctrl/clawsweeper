## Summary

Keep #71575 open. Current main still intentionally exposes Telegram tool-progress preview lines by default, including `Working…`, tool names, and command/progress text. The related #71663 was closed as a duplicate of this issue, so #71575 remains the canonical tracker for the remaining Telegram visibility behavior.

## Best Possible Solution

Keep #71575 open as the canonical Telegram tool/progress visibility bug. The likely fix belongs in the Telegram preview dispatch path and shared preview-progress contract: either default Telegram `streaming.preview.toolProgress` to false for user chats, or strictly sanitize preview progress so it contains only safe high-level status and never raw command output, tool internals, or duplicates. Update `extensions/telegram/src/bot-message-dispatch.test.ts`, the relevant agent tool-event coverage, and Telegram/streaming docs to match the chosen maintainer product behavior.
