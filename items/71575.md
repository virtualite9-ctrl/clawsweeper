## Summary

Current main still renders visible Telegram tool-progress preview edits by default during partial preview streaming. The requested final-only Telegram behavior remains unimplemented, and the related #71663 context supports this issue as the canonical remaining item.

## Best Possible Solution

Keep this issue open for a Telegram-owned behavior change. The likely fix belongs in the Telegram preview streaming path: make Telegram hide tool-progress preview lines by default, add a stricter final-only mode for Telegram sessions, or sanitize/summarize progress so raw tool details never appear. Update `extensions/telegram/src/bot-message-dispatch.ts`, Telegram dispatch tests, and the Telegram/streaming docs together; keep #71663 closed as related duplicate context.
