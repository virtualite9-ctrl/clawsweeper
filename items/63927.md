## Summary

Current `main` already normalizes Discord `channel:<snowflake>` targets before ACP thread binding resolves the channel, and regression tests cover the exact prefixed-channel cases described in the issue. The latest shipped tag available in this checkout, `v2026.4.23`, also contains that implementation.

## Evidence

- **Current Discord binding code normalizes prefixed channel ids:** `resolveChannelIdForBinding` now normalizes both the optional explicit `channelId` and the `threadId` through `normalizeDiscordBindingChannelId(...)`, which delegates to `resolveDiscordChannelId(...)` and strips `channel:` prefixes before any Discord REST channel lookup.
