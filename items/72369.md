## Summary

Keep this issue open. Current main still routes bare `/new` and `/reset` through the startup prompt as the transcript-visible user body, while the Control UI only filters unrelated internal cases such as silent replies, heartbeat acks, synthetic repair tool results, empty user messages, and compaction markers. The provided related reports cover other WebChat history leaks/flicker cases, but they do not cover this bare reset startup prompt path.

## Best Possible Solution

Keep the issue open and fix the owner path: preserve the startup prompt for the agent runtime, but prevent it from surfacing as a normal user chat bubble. A conservative implementation would either persist/emit a transcript marker such as `__openclaw: { kind: "startup" }` and teach `chat.history`/WebChat to hide or render it as a neutral session divider, or split runtime-only startup instructions from the user-visible transcript prompt before the runner appends the turn. Add focused regression coverage around `get-reply-run`, `chat.history`, and `ui/src/ui/chat/build-chat-items.ts` or `ui/src/ui/controllers/chat.ts`.
