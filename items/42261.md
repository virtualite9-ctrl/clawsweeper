## Summary

Current main still has the reported callback coupling: live reasoning is enabled only when `onReasoningStream` is present, and the `stream: "thinking"` broadcast remains behind that same guard. The linked fix attempts (#15918 and #46121) are closed and unmerged in the provided GitHub context, so there is no canonical open implementation candidate and the issue should stay open.

## Best Possible Solution

Keep the issue open. The likely fix belongs in `src/agents/pi-embedded-subscribe.ts`: set `streamReasoning` from `reasoningMode === "stream"` plus the existing thinking-level visibility check, emit the `stream: "thinking"` agent event before any optional callback check, and add a focused regression test for callback-absent WS delivery. The closed unmerged PRs #15918 and #46121 are useful references but should not be treated as resolving the issue.
