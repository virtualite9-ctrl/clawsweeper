## Summary

Current main still matches the reported root cause: live `stream: "thinking"` agent events are only emitted when the optional `onReasoningStream` callback exists, so webchat/WS clients that rely on the gateway event bus remain blocked. The stale label is not enough to close because the issue is concrete, source-backed, and the related fix PRs are closed unmerged.

## Best Possible Solution

Keep this issue open. The likely fix belongs in `src/agents/pi-embedded-subscribe.ts`: initialize `streamReasoning` from stream mode plus reasoning visibility, remove `params.onReasoningStream` from the early-return condition, emit the `stream: "thinking"` agent event before optionally invoking the callback, and add a focused regression test for `reasoningMode: "stream"` without `onReasoningStream` still emitting a thinking agent event. Update `docs/tools/thinking.md` only if webchat/WS live reasoning is intended as a supported surface.
