## Summary

Current main still matches the reported root cause: `streamReasoning` is disabled unless `onReasoningStream` exists, and `emitReasoningStream()` returns before `emitAgentEvent({ stream: "thinking" })` for callback-less runs. Webchat dispatch does not provide `onReasoningStream`, while the gateway/event-bus path can carry generic `thinking` events if the subscribe layer emits them.

## Best Possible Solution

Keep this issue open. The likely fix belongs in `src/agents/pi-embedded-subscribe.ts`: initialize `streamReasoning` from stream mode plus reasoning visibility rather than callback presence, remove `params.onReasoningStream` from the early-return condition, emit `stream: "thinking"` before optionally invoking the surface callback, and add a focused callback-absent regression test proving the agent event bus receives live thinking. Update `docs/tools/thinking.md` only if webchat/WS live reasoning is intended as supported user-facing behavior.
