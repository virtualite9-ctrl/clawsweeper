## Summary

Current main already waits for the model fallback chain to finish before turning the failure into a user-facing reply in the auto-reply/iMessage path.

## Evidence

- **Fallback runner exhausts candidates before surfacing terminal failure:** `runWithModelFallback()` iterates candidates, records failed attempts, and only calls `throwFallbackFailureSummary()` after the loop finishes. That means a `FailoverError` from an intermediate candidate is not terminal by itself.
  - file: [src/agents/model-fallback.ts:746](https://github.com/openclaw/openclaw/blob/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4/src/agents/model-fallback.ts#L746)
  - sha: [f3c37a946cc4](https://github.com/openclaw/openclaw/commit/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4)
- **Per-attempt error hook is optional:** The only mid-loop callback is the optional `onError` hook that fires after a failed candidate, but it is not inherently user-facing.
  - file: [src/agents/model-fallback.ts:974](https://github.com/openclaw/openclaw/blob/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4/src/agents/model-fallback.ts#L974)
  - sha: [f3c37a946cc4](https://github.com/openclaw/openclaw/commit/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4)
- **Auto-reply/iMessage execution path does not pass `onError`:** `runAgentTurnWithFallback()` awaits `runWithModelFallback()` directly and passes config, runId, classification, and `run`, but no per-attempt `onError` handler. That removes the path that would surface an intermediate failure before fallback completion.
  - file: [src/auto-reply/reply/agent-runner-execution.ts:888](https://github.com/openclaw/openclaw/blob/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4/src/auto-reply/reply/agent-runner-execution.ts#L888)
  - sha: [f3c37a946cc4](https://github.com/openclaw/openclaw/commit/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4)
- **User-facing error text is only built after awaited fallback failure:** The auto-reply runner converts the failure into final user text only in the outer catch after the awaited fallback run throws, which happens after exhaustion rather than on the first fallback-eligible error.
  - file: [src/auto-reply/reply/agent-runner-execution.ts:1574](https://github.com/openclaw/openclaw/blob/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4/src/auto-reply/reply/agent-runner-execution.ts#L1574)
  - sha: [f3c37a946cc4](https://github.com/openclaw/openclaw/commit/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4)
- **Shipped release contains the same await-then-surface structure:** The tagged release `v2026.4.22` shows the same `runWithModelFallback()` loop and the same auto-reply call site that awaits it without an `onError` handler.
  - command: `git show v2026.4.22:src/agents/model-fallback.ts | sed -n '715,990p' && git show v2026.4.22:src/auto-reply/reply/agent-runner-execution.ts | sed -n '880,945p;1558,1615p'`
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)
