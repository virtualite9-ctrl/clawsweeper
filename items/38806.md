## Summary

Current main still appears to have the Feishu P2P direct-message thread reply bug. The inbound `root_id` and `thread_id` fields are parsed, but the reply-routing path remains group-gated: direct messages force non-thread reply mode, do not use the P2P thread root as the reply target, and strip reply metadata before dispatch. The provided related context identifies #38808 as the open targeted fix candidate; #38814 and #38996 were closed unmerged.

## Best Possible Solution

Keep #38806 open and resolve it through #38808 or an equivalent targeted Feishu patch. The fix should detect direct thread messages with `isDirect && Boolean(ctx.threadId)`, use the P2P thread root as the reply target, set `replyInThread` and `threadReply` for that case, preserve reply metadata for direct threads, keep root_id-only P2P quote replies on the triggering message, and add regression coverage for both P2P thread replies and P2P plain quote replies.
