## Summary

Current main still appears to have the Feishu P2P thread reply bug. Inbound `root_id` and `thread_id` are parsed, but reply routing remains gated on `isGroup`: direct messages force `replyInThread`/`threadReply` false, ignore `rootId` for the reply target, and strip reply metadata before normal sends. The provided related context still shows PR #38808 as the open targeted fix candidate, while #38814 and #38996 were closed unmerged.

## Best Possible Solution

Keep #38806 open and resolve it through PR #38808 or an equivalent targeted Feishu patch: detect direct thread messages with `isDirect && Boolean(ctx.threadId)`, use the P2P thread root as the reply target, set `replyInThread`/`threadReply` for that case, do not skip reply metadata for direct threads, preserve root_id-only P2P quote behavior, and add regression coverage for both P2P thread replies and P2P plain quote replies.
