## Summary

Keep #38806 open. Current main still preserves Feishu P2P thread fields on inbound events, but the reply-delivery path treats all direct messages as non-threaded: `replyInThread` and `threadReply` stay false, root-target selection is group-only, and `skipReplyToInMessages: !isGroup` strips the reply target before normal sends. The provided GitHub context also shows PR #38808 as the still-open targeted fix candidate, while #38814 and #38996 were closed unmerged.

## Best Possible Solution

Keep #38806 open and fix it through PR #38808 or an equivalent targeted Feishu patch: detect `isDirect && Boolean(ctx.threadId)`, use the P2P thread root as the reply target, set `replyInThread`/`threadReply` for that case, do not skip reply-to metadata for direct threads, preserve root_id-only P2P quote behavior, and add regression coverage for both P2P thread replies and P2P plain quote replies.
