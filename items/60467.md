## Summary

Current main already handles Anthropic OAuth/plugin tool dispatch by remapping tool names onto Claude Code wire names and back again, and the shipped regression test covers the exact `read` -> `Read` -> `read` flow described in the report. The same fix note is present in the `v2026.4.23` changelog, so this appears already implemented and released.

## Evidence

- **OAuth transport remaps outgoing tool names and maps streamed tool names back before dispatch:** `toClaudeCodeName()` rewrites known tool names for OAuth requests, `convertAnthropicTools()` applies that mapping to the request payload, and `fromClaudeCodeName()` maps streamed `tool_use` names back to the locally registered tool name.
