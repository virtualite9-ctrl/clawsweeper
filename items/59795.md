## Summary

Keep PR #59795 open. Current main still has the blanket same-target messaging-tool suppression that #59743 reports, so the PR is not obsolete or implemented on main. The PR is useful, but the latest review discussion identifies an unresolved correctness bug in the PR head: suppression must require the same origin-matching target to have sent text, not just any target in the turn.

## Best Possible Solution

Keep the PR open and revise it so suppression is decided per matching target: a same-origin target should suppress only when that same target has sentText true. Preserve same-origin media dedupe independently of text suppression, keep caption and whitespace handling aligned with the message action readers, and add regression coverage for media-only same-origin sends plus the mixed origin-media-only/cross-target-text case.
