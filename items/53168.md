## Summary

Current main implements the reported fix path. The report is against OpenClaw 2026.3.13 and shows the exact old 3001ms pre-auth WebSocket timeout followed by gateway closed (1000); PR #49262 raised that timeout to 10s, added the runtime override, and the current CLI/client path now waits for connect.challenge, signs the nonce, and yields before opening CLI gateway connections. The changelog records the fix in 2026.3.22.

## Best Possible Solution

Close this as shipped. Keep the current shared 10s handshake timeout, production env override, signed connect.challenge flow, and CLI event-loop yield. Treat any modern post-2026.3.22 no-reply or post-challenge hangs as separate focused reports, not as this old 3s timeout bug.
