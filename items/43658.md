## Summary

Keep open. Current main still uses one shared GitHub Copilot API-token cache that is not bound to the selected GitHub token or auth profile, so profile rotation can still receive another profile's cached Copilot API token. The issue also has open same-author PR #43659 with closing syntax as an implementation candidate, so this issue should remain open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and resolve it through #43659 or an equivalent maintainer-reviewed patch. The fix should bind cached Copilot API tokens to the selected GitHub token/profile, treat legacy unbound cache entries as misses or otherwise safe, and add regression tests proving a rotated Copilot auth profile performs a fresh token exchange instead of reusing another profile's cached Copilot API token.

## Risks / Open Questions

- Leaving current main unchanged means multiple GitHub Copilot auth profiles can continue sharing one cached Copilot API token, defeating quota rotation after one profile is rate-limited.
