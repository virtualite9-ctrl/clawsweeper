## Summary

Keep open. Current main still has RTL isolation only in the TUI formatter; the shared assistant-visible sanitizer, user-facing sanitizer, gateway live projection, and outbound delivery path still do not add bidi isolation before non-TUI channel sends.

## Best Possible Solution

Keep this issue open. Implement a shared RTL/bidi isolation helper under src/shared/text, reuse it from the TUI formatter, and apply it at the canonical user-visible outbound boundary before non-TUI channel chunking/send and any intended gateway live projection. Add focused tests for shared helper behavior, outbound/plain-text delivery, no-double-wrap for existing bidi controls, and representative Hebrew/Arabic punctuation cases.
