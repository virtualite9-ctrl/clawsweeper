## Summary

Keep open. Current main still implements RTL RLI/PDI isolation only in the TUI formatter; shared assistant-visible sanitization, gateway live projection, channel plain-text sanitization, and outbound delivery still forward Hebrew/Arabic text without a shared bidi-isolation stage. The earlier ClawSweeper comment and the related closed TUI/Telegram-preview reports do not supersede this outbound boundary bug.

## Best Possible Solution

Keep this issue open. Implement a shared RTL/bidi isolation helper under src/shared/text, reuse it from the TUI formatter, and apply it on the canonical user-visible outbound path before channel chunking/send. Add focused regression tests for the shared helper, outbound/plain-text delivery, gateway/live projection if that surface is intended to be covered, and no-double-wrap behavior for existing bidi controls.
