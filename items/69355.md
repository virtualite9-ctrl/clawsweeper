## Summary

Keep #69355 open. Current main still lacks a native `openclaw doctor` check for locally rebuilt npm installs versus pristine npm publish metadata, and the package update path still refreshes same-version npm installs. The PR targets a real core doctor/update diagnostic gap, but it needs revision before merge because the review comments identify concrete correctness and responsiveness issues.

## Best Possible Solution

Keep the PR open. Rebase it onto current main, address the review comments with package/build-info version parity, precise corrupt metadata reasons, fresh-cache/offline semantics, and a bounded npm registry fetch timeout, then land the warn-only doctor check with focused tests and docs if maintainers still want this install-posture signal in core.
