## Summary

Keep #69355 open. Current main has build metadata and package-update integrity checks, but it still does not implement the PR's requested `openclaw doctor` warning for locally rebuilt npm installs versus pristine registry publish times. The PR is a focused, plausible contribution with concrete operator evidence and tests in the provided PR context; it needs maintainer review and fixes for the existing review comments rather than cleanup closure.

## Best Possible Solution

Keep the PR open and ask the contributor or maintainer to refresh it against current `main`, preserve the proposed warn-only doctor surface, and address the review comments before considering merge. The feature belongs in core doctor/update diagnostics because it protects packaged OpenClaw installs from silent local rebuild drift and current plugin/ClawHub surfaces do not replace that package-integrity signal.

## Risks / Open Questions

- Merging the PR as-is would need the existing review comments addressed, especially a bounded registry fetch timeout for normal doctor responsiveness and cache TTL behavior to avoid stale false positives.
- The warning should stay non-blocking and silent when registry data is unavailable, matching the PR's stated safety contract.
- The implementation should validate `package.json.version` against `dist/build-info.json.version` so partial overlays are classified as corrupt instead of evaluating the wrong registry version.
