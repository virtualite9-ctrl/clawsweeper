## Summary

Keep #69355 open. Current main still lacks a native `openclaw doctor` check for locally rebuilt npm installs versus pristine npm publish metadata. The PR targets a real core doctor/update diagnostic gap, but it needs revision before merge because review comments identify concrete correctness and responsiveness issues.

## Best Possible Solution

Keep the PR open and revise it into a small core doctor diagnostic: read `package.json` plus `dist/build-info.json`, compare against npm publish metadata with bounded network I/O and TTL-respecting cache behavior, warn only for high-confidence local rebuild/unreleased/corrupt cases, stay silent when inconclusive, and cover the doctor emission matrix plus version-mismatch/cache/timeout cases in tests and docs.
