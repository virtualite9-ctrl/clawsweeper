## Summary

Keep #71084 open. Current main still implements the CLI-backed agent skip that the issue questions, and the docs/tests also describe and assert that behavior. Because the request is recent, concrete, core-runtime behavior, and tied to a closed-unmerged PR that was explicitly replaced by this discussion, it needs maintainer product/architecture judgment rather than conservative auto-close.

## Best Possible Solution

Keep the issue open for explicit maintainer direction. If CLI memory flush/preflight compaction is approved, update `src/auto-reply/reply/agent-runner-memory.ts`, replace the CLI-skip regression with CLI-allowed coverage, add preflight CLI coverage, and update `docs/reference/session-management-compaction.md` plus public config docs. If the embedded-only boundary is intentional, close manually with that design rationale and clarify `docs/gateway/config-agents.md`, which currently only mentions the read-only workspace skip.
