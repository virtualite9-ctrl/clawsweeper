## Summary

Keep open. Current main has the bundled `microsoft-foundry` provider but does not have a dedicated Microsoft Foundry provider docs page or provider navigation entry. PR #72162 adds that missing docs surface and its latest diff appears to have removed the Codex API-variant table that Greptile flagged on an earlier commit. Related PR #68502 overlaps on Azure AI Foundry documentation, but it documents a different `azure-ai-foundry` page and manual adapter-style provider setup rather than the bundled `microsoft-foundry` onboarding flow, so it is not a clean duplicate/superseder.

## Best Possible Solution

Keep PR #72162 open and treat it as useful docs work for the bundled `microsoft-foundry` provider. The best path is to revise it against current `extensions/microsoft-foundry/*` behavior, add any missing provider-directory/index navigation expected by docs policy, and reconcile or supersede the older Azure AI Foundry docs PR #68502 rather than closing this one as already handled.
