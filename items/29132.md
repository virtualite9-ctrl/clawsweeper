## Summary

Keep open. Current main has a trusted runtime tool-result middleware path that rewrites OpenClaw-owned tool output before Pi/Codex feed it back to the model, but that seam is intentionally bundled-only. The requested public plugin hook, `tool_result_before_model`, is still absent, and public plugin hooks still do not provide same-turn post-execution/pre-model tool-result mutation.

## Best Possible Solution

Keep the issue open until OpenClaw either adds a documented, trust-gated public API for post-execution/pre-model tool-result rewriting, or maintainers explicitly decide this capability must remain bundled-only. A complete implementation should specify ordering relative to `before_tool_call`, `after_tool_call`, and `tool_result_persist`, define failure behavior, and add runtime coverage proving the model receives only the sanitized result.
