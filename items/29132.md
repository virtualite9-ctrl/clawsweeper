## Summary

Keep open. Current main still has bundled-only pre-model tool-result middleware, but external plugin authors still cannot register the requested public synchronous `tool_result_before_model` hook or an equivalent post-execution/pre-model result mutation hook. The related closed issue #34144 and closed PR #14544 do not supersede this issue, and the overlapping open #62123 asks for an async transform before transcript write rather than this issue's synchronous same-turn pre-LLM policy boundary.

## Best Possible Solution

Keep this issue open until OpenClaw either lands a documented, trust-gated public post-execution/pre-model tool-result mutation hook, or maintainers explicitly decide this capability must remain bundled-only. A complete implementation should define ordering relative to `before_tool_call`, bundled `registerAgentToolResultMiddleware`, `after_tool_call`, `tool_result_persist`, and `before_message_write`; specify sync/async and failure semantics; update the public plugin SDK docs; and add runtime tests proving the model receives only the sanitized result.

## Risks / Open Questions

- Exposing model-visible tool-result rewriting to external plugins is a high-trust API/security decision and needs maintainer policy on ordering, sync versus async semantics, failure behavior, and which plugin origins may use it.
- There is meaningful overlap with #62123, so maintainers should coordinate any synchronous pre-model sanitizer with the proposed async transform hook rather than adding competing surfaces.
