## Summary

Keep open. Current main still does not expose a public same-turn pre-LLM tool-result sanitization hook. The code has trusted bundled-only tool-result middleware that can rewrite results before model consumption, but public plugin hooks still cover pre-tool params/blocking, after-tool observation, transcript persistence, and message dispatch, not post-execution/pre-model result mutation for external plugins.

## Best Possible Solution

Keep the issue open until OpenClaw either adds a documented, trust-gated public API for same-turn post-execution/pre-model tool-result rewriting, or maintainers explicitly decide that this capability must remain bundled-only. A complete implementation should define ordering relative to `before_tool_call`, `after_tool_call`, and `tool_result_persist`, specify failure behavior, and add runtime coverage proving the model only sees sanitized output.
