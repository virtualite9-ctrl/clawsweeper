## Summary

Keep open. Current main appears to have already addressed the thread-routing part of the report for normal Feishu groups, but the mention-detection part is still not implemented for text replies where Feishu leaves an `@_user_N` placeholder in content while omitting the bot from `message.mentions`. That means the reported no-response behavior is still plausible on main.

## Evidence

- **Mention detection still trusts `message.mentions` for text messages:** `checkBotMentioned()` returns `mentions.some(...)` as soon as any mentions are present, and only falls back to parsing content for `post` messages. There is no text-message fallback for orphaned `@_user_N` placeholders.
  - file: [extensions/feishu/src/bot-content.ts:229](https://github.com/openclaw/openclaw/blob/cb2c36b04975e34ab7014a11575d6589aa96ae32/extensions/feishu/src/bot-content.ts#L229)

  - sha: [cb2c36b04975](https://github.com/openclaw/openclaw/commit/cb2c36b04975e34ab7014a11575d6589aa96ae32)
- **Failed mention detection still suppresses group replies:** Group handling still exits early when `requireMention` is enabled and `ctx.mentionedBot` is false, so a dropped Feishu bot mention would still cause the bot not to respond.
  - file: [extensions/feishu/src/bot.ts:474](https://github.com/openclaw/openclaw/blob/cb2c36b04975e34ab7014a11575d6589aa96ae32/extensions/feishu/src/bot.ts#L474)
