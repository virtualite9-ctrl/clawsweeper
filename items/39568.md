## Summary

Keep #39568 open. Current main at 631552c554095b0c520376db9131dac0f5427f94 still has the two reported Feishu reply-mention failure paths: bot mention detection short-circuits on the structured mentions array, and reply root/thread metadata still forces thread reply mode. The linked fix PR #39569 is useful evidence, including production verification in its body, but the provided GitHub context says it is still open and unmerged.

## Best Possible Solution

Keep #39568 open until #39569 or an equivalent Feishu plugin fix lands. The maintainer path is to update checkBotMentioned() to detect orphaned Feishu reply mention placeholders, suppress rootId/threadReply routing for explicit bot @mentions in normal reply messages where the response should stay in the main group chat, and add targeted regression tests for both mention detection and non-thread reply dispatch.
