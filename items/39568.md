## Summary

Keep #39568 open. Current main still matches the reported Feishu reply-mention bug: text-message mention detection only trusts the structured mentions array, group require-mention handling drops the message when that check is false, and reply root_id still feeds thread-reply routing. The linked fix PR #39569 is still reported open/unmerged in the provided context.

## Best Possible Solution

Keep #39568 open until #39569 or an equivalent Feishu plugin fix lands. The maintainer path is to update checkBotMentioned() to detect orphaned Feishu reply mention placeholders, clear or suppress rootId/threadReply for explicit bot @mentions in reply messages where the bot should answer in the main group chat, and add targeted regression tests for both mention detection and non-thread reply behavior.
