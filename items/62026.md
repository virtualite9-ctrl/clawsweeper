## Summary

Current `main` already routes isolated cron/one-shot runs through `cleanupBundleMcpOnRunEnd`, and the embedded runner now retires bundled MCP runtimes at run end. That matches the reporter’s April 7 root-cause analysis, and the released `v2026.4.23` tree contains both the cleanup path and regression coverage for it.

## Evidence

- **Reporter follow-up matches the current code path:** The April 7 issue comment identified the missing `cleanupBundleMcpOnRunEnd` flag on isolated cron runs as the root cause, and the timeline cross-references follow-up PRs in the same area (#62925, #68450).
- **Released code now enables cleanup for isolated cron runs:** `runEmbeddedPiAgent` is called with `cleanupBundleMcpOnRunEnd: params.job.sessionTarget === "isolated"`, which is the exact missing flag the reporter called out.
  - file: [src/cron/isolated-agent/run-executor.ts:165](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/cron/isolated-agent/run-executor.ts#L165)
  - command: `git show v2026.4.23:src/cron/isolated-agent/run-executor.ts | nl -ba | sed -n '155,170p'`
  - sha: [a9797214338b](https://github.com/openclaw/openclaw/commit/a9797214338ba31c52c796adbb75afb16e0684a9)
- **Embedded runner disposes bundled MCP runtimes at run end:** When `cleanupBundleMcpOnRunEnd` is true, the runner retires the session MCP runtime by session key or session id with reason `embedded-run-end`.
  - file: [src/agents/pi-embedded-runner/run.ts:2138](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/agents/pi-embedded-runner/run.ts#L2138)

- **Regression test covers the isolated cron cleanup flag:** The cron fast-mode test asserts `cleanupBundleMcpOnRunEnd` defaults to `true` for the isolated run path, so this exact regression now has automated coverage.
