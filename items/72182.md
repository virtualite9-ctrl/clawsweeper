## Summary

Current main addresses the reported Onboard/model-switching failure path. The issue was filed against v2026.4.24, where model selection could load catalog/provider setup surfaces and trip over an optional bundled channel dependency even though Nostr was not enabled. Main now keeps those onboarding model prompts cold: skip-auth model selection disables catalog/provider setup loading, auth-driven model selection defers catalog loading until the user explicitly chooses Browse, and setup reads config with plugin validation skipped. Nostr still owns `nostr-tools` as a plugin-local runtime dependency, and docs intentionally say disabled bundled plugin deps are not repaired just because the plugin exists.

## Best Possible Solution

Keep the current main behavior: Onboard/model switching should stay cold unless the user explicitly browses the full model catalog or selects/configures a specific channel. Do not add `nostr-tools` to the root OpenClaw package just for disabled Nostr; ship the current cold-path fix in the next release and keep plugin-local dependency repair scoped to active/default-enabled bundled plugins.
