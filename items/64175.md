## Summary

Current main still appears to have the reported OpenRouter tool-capability bug. OpenRouter live metadata says `perplexity/sonar-deep-research` does not advertise `tools`, but OpenClaw's dynamic OpenRouter model path still records only reasoning/input/cost/token metadata and leaves `compat.supportsTools` unset, which makes the agent runtime treat the model as tool-capable by default.

## Best Possible Solution

Keep this issue open and fix it at the OpenRouter provider/runtime boundary: preserve whether OpenRouter `supported_parameters` includes `tools`, map that to `compat.supportsTools` for dynamic OpenRouter models, treat old cache entries conservatively, and add regression coverage for `openrouter/perplexity/sonar-deep-research` proving OpenClaw no longer sends tool schemas or raises a clear compatibility error before provider fallback.
