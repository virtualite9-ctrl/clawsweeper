## Summary

This PR is obsolete for the streaming/backpressure concern: current main and v2026.4.24 already use the pinned tar file-mode path, and upstream tar@7.5.13 implements that file-mode path by piping a Pack stream to a WriteStream with error propagation. The branch is also based on the old backup implementation location and includes stale OOM guidance for flags that are not registered on main.

## Best Possible Solution

Close this PR as already covered by the shipped tar file-mode streaming and atomic temp-file implementation. Keep any remaining large-installation UX work in the open exclude/progress tracking path, especially #42273 and #44288, rather than merging this stale explicit-pipe rewrite.
