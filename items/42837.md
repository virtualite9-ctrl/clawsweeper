## Summary

Keep open. Current main still creates Feishu dynamic-agent DM bindings without `match.accountId`, while Feishu inbound routing resolves and re-resolves using the runtime account id. For named Feishu accounts, the generated account-less binding remains default-account-only, so the reported fallback to `agent:main` is still plausible on current code.

## Best Possible Solution

Keep this issue open. Refresh the #42886 approach against current main: pass the normalized Feishu runtime `account.accountId` into `maybeCreateDynamicAgent`, include `match.accountId` when checking and creating dynamic bindings, update both message and comment dynamic-agent flows, preserve legacy account-less default-account behavior, and add focused regression tests for named-account creation, missing-binding repair, and legacy default-account routing.
