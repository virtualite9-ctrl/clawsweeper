## Summary

Keep open. Current main still has the Feishu dynamic-agent account-scoping bug described in the report: Feishu message/comment routing resolves with the runtime account id, but dynamic-agent creation checks and writes bindings without `match.accountId`, so a named-account DM can create an agent/binding and still re-resolve to `agent:main`. The related fix PR #42886 was closed unmerged, and I did not find an equivalent implementation on current main.

## Best Possible Solution

Keep this issue open. Refresh the #42886 approach against current main: pass the normalized Feishu runtime `account.accountId` into `maybeCreateDynamicAgent`, include `match.accountId` when checking and creating dynamic bindings for named accounts, update both message and comment dynamic-agent flows, preserve legacy account-less default-account behavior, and add focused regression tests for named-account creation, missing-binding repair, and legacy default-account routing.
