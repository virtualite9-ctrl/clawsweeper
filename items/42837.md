## Summary

Keep open. Current main still creates Feishu dynamic-agent DM bindings without `match.accountId`, while Feishu message handling resolves routes with the runtime `account.accountId` and the core router only treats account-less bindings as default-account bindings. The linked fix PR #42886 identified the same root cause and added the right account-aware direction, but it is closed and unmerged, so this is not implemented on main.

## Best Possible Solution

Keep this issue open and refresh the #42886 approach against current main: pass normalized `account.accountId` into `maybeCreateDynamicAgent`, include `match.accountId` on newly written and missing dynamic bindings, preserve legacy account-less bindings as default-account-only, update both Feishu message and comment flows, and add regression coverage for named accounts, default legacy bindings, existing-agent missing-binding repair, and full new-agent creation.
