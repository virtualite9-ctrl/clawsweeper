## Summary

Keep #72273 open. Current main still hardcodes the Discord gateway startup READY timeout at 15s and runtime READY timeout at 30s, while Discord multi-account startup is intentionally staggered by 10s per enabled account. The existing configurable Discord timeout knobs cover listener and inbound-worker budgets, not gateway READY lifecycle timeouts, so the reported request remains actionable in the bundled Discord plugin.

## Best Possible Solution

Keep the issue open and implement the fix in the bundled Discord plugin. Add a Discord-owned config surface, likely under `channels.discord` and per-account `channels.discord.accounts.<id>`, for startup/runtime gateway READY timeouts; wire those values into `runDiscordGatewayLifecycle`; choose a default that accounts for configured startup stagger in multi-account setups; and update Discord schema, UI hints, docs, and lifecycle tests.
