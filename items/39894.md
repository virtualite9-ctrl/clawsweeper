## Summary

Keep PR #39894 open. Current main still parses outbound reply text with `parseReplyDirectives(payload.text ?? "")` before delivery/JSON projection, and it does not strip echoed inbound metadata on that shared outbound payload path. The PR's core fix remains useful for the still-open canonical bug #39847, though its unrelated CI/macOS/test-runner changes should be split or removed before landing.

## Best Possible Solution

Keep PR #39894 or canonical issue #39847 open until main lands a narrow core fix: strip copied inbound metadata before `parseReplyDirectives()` in `src/infra/outbound/payloads.ts`, add a regression in `src/infra/outbound/payloads.test.ts` covering `Conversation info (untrusted metadata)` followed by visible text plus `MEDIA:`, and split unrelated CI/macOS/test-runner edits into separate work if still needed.
