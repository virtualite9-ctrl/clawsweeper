## Summary

Keep PR #39894 open. Current main still builds outbound payload plans by parsing raw payload text, so echoed inbound metadata can remain visible while MEDIA directives are extracted. The linked canonical bug #39847 remains open, and this PR contains a narrow useful fix plus regression coverage, though its unrelated CI/macOS/test-runner changes should be split before landing.

## Best Possible Solution

Keep PR #39894 or canonical issue #39847 open until main lands a narrow core fix: strip copied inbound metadata before parseReplyDirectives() in src/infra/outbound/payloads.ts, add a regression in src/infra/outbound/payloads.test.ts covering Conversation info/Sender metadata followed by visible text plus MEDIA, and split the unrelated CI/macOS/test-runner edits into separate work if still needed.
