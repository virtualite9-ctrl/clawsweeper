## Summary

Keep #38573 open. Current main still forces resolved thread/topic isolated cron deliveries through direct outbound delivery, and the supported cron delivery contract does not expose an opt-in route for text-only topic/forum completions to go through the main-session announce/agent-turn judgment path. The related #34160 subagent announce work is useful context, but it covers subagent completion handoff and does not add a cron completion-route selector.

## Best Possible Solution

Keep this open for maintainer product/API work. Add a backward-compatible cron completion-route option that defaults existing thread/topic targets to direct delivery, lets text-only isolated cron completions opt into main-session announce/agent-turn handling, keeps structured payloads on direct delivery, defines fail-closed versus direct-fallback behavior, and updates cron types/storage/normalization, gateway protocol, agent tool schema, CLI/UI, docs, and regression tests.
