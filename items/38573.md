## Summary

Keep #38573 open. Current main still forces cron deliveries with a resolved `threadId` through direct outbound delivery, and the cron config/tool/schema surfaces do not expose a per-job or global opt-in for topic/forum text completions to use the main-session announce/agent-turn path. Related #34160 is useful context, but it addressed subagent announce behavior for group/topic sessions; it does not supersede this cron-specific direct-delivery override. Current main has a partial awareness queue for later main-session context, but that happens after direct delivery and does not provide the requested pre-post main-agent rewrite/judgment flow.
