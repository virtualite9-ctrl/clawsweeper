## Summary

Keep #38573 open. Current main still forces cron deliveries with a resolved thread/topic id through direct outbound delivery, and the cron data model, gateway protocol, agent tool, CLI/UI surfaces, and docs still do not expose an opt-in route for text-only topic/forum completions to pass through a main-session announce/agent-turn flow before posting.

## Best Possible Solution

Keep this open for maintainer product/API work. Add a backward-compatible cron delivery route option, keep direct topic delivery as the default, allow text-only thread/topic completions to opt into main-session announce/agent-turn delivery, keep structured payloads on direct delivery, define fail-closed versus direct-fallback behavior, and update cron normalization/store/patch handling, gateway protocol, agent tool, CLI/UI, docs, and tests.
