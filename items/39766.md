## Summary

Keep open. Current main still requires `models.providers.<id>.models` in the source Zod schema and generated config schema, so the PR's central behavior is not implemented. The embedding runtime would use `models.providers.openai.baseUrl` if validation allowed the provider entry, which matches the linked bug path, and the related alternatives in the provided context are closed unmerged while #39589 remains open.

## Best Possible Solution

Keep this PR or canonical issue #39589 open until main accepts `models.providers.<id>` entries that omit `models`, preserves the parsed runtime contract as an array such as `[]`, updates the generated config schema, and adds focused regression coverage for schema parsing plus OpenAI embedding `baseUrl` fallback behavior with network preflight stubs.
