## Summary

Keep PR #39975 open. Current main still serializes `message_reference` into Discord Execute Webhook request bodies when `replyTo` is present, while the upstream Discord docs list `message_reference` only for Create Message. The submitted branch is stale because it edits the old `src/discord/send.outbound.ts` path, but the same focused fix is still needed in `extensions/discord/src/send.outbound.ts` with a webhook-body regression test.

## Best Possible Solution

Keep this PR open or replace it with an equivalent refreshed patch against `extensions/discord/src/send.outbound.ts`: remove `message_reference` only from `sendWebhookMessageDiscord`, preserve bot-token/channel reply behavior in `sendMessageDiscord` and related media/component paths, and add a focused webhook request-body regression test proving `replyTo` does not serialize into Execute Webhook payloads.
