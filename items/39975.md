## Summary

Keep PR #39975 open. Current `main` still serializes `message_reference` into Discord Execute Webhook JSON bodies when `replyTo` is present, so the PR’s central fix remains relevant. The submitted patch is stale because it targets the old `src/discord/send.outbound.ts` path, but the same focused change still belongs in `extensions/discord/src/send.outbound.ts` with a webhook-body regression test.

## Best Possible Solution

Keep this PR open or replace it with an equivalent refreshed patch against `extensions/discord/src/send.outbound.ts`: remove `message_reference` only from `sendWebhookMessageDiscord`, preserve `sendMessageDiscord` bot-token reply behavior, and add a webhook request-body regression test proving `replyTo` does not serialize into Execute Webhook payloads.
