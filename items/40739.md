## Summary

Current main already ships DuckDuckGo as a bundled `web_search` provider and the runtime falls back to a keyless provider instead of forcing Brave when no search API credentials are configured, so the Brave-missing-key behavior reported against 2026.3.2 is no longer the current implementation.

## Risks / Open Questions

- Custom third-party search skills still do not automatically rebind the built-in `web_search` tool; users still need to select a built-in provider explicitly or remove built-in `web_search` from the agent toolset if they only want a custom skill.

## Close Comment

Closing as implemented on current OpenClaw. Codex review against `main` found that OpenClaw now bundles DuckDuckGo as a native `web_search` provider (`extensions/duckduckgo/index.ts:4`, `extensions/duckduckgo/src/ddg-search-provider.shared.ts:5`) and the runtime falls back to a keyless provider when no search API credentials are configured (`src/web-search/runtime.ts:140`). The docs also now document `tools.web.search.provider: "duckduckgo"` with no API key required (`docs/tools/duckduckgo-search.md:21`, `docs/tools/web.md:161`). In other words, the Brave-missing-key behavior reported for 2026.3.2 is not the current behavior on `main` or recent releases.
