## Summary

Keep #8081 open. Current main has useful lower-level controls, including trusted-proxy identity, operator scopes, config redaction, per-agent auth stores, sandbox/tool policies, and sender/group tool policies, but it still documents a single trusted-operator gateway model and does not implement the requested shared-gateway admin/user/guest RBAC, per-user credential isolation, or audit semantics. The latest issue discussion also includes a maintainer-authored Codex review explicitly keeping this tracker open.

## Best Possible Solution

Keep #8081 open as the broad product/security tracker. A complete path should define gateway and channel caller identity, role- or scope-based authorization for sensitive config and credential operations, per-user or per-tenant secret isolation, audit logging, channel/group identity mapping, documented trust boundaries, and regression coverage across gateway auth, config read/write paths, HTTP APIs, and channel command surfaces. If maintainers choose an external-wrapper or ClawHub-first direction, document that explicitly and identify any missing plugin/API surfaces needed to support it outside core.
