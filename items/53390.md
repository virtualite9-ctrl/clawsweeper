## Summary

Keep open. Current main still has no viewport-aware text snapshot mode: the browser tool accepts snapshot format/mode/refs/selector/frame/labels/urls, but not a viewport or visible-only option, and the implementation still delegates text snapshots to page/root accessibility snapshot paths or full CDP accessibility trees. The only explicitly viewport-scoped snapshot behavior I found is labeled screenshots, not the text snapshot contract reported here.

## Best Possible Solution

Keep this issue open. Maintainers should decide the intended `browser snapshot` contract: either add a viewport-aware text snapshot mode across Playwright, raw CDP, and Chrome MCP paths, or explicitly document that text snapshots are page/root accessibility-tree snapshots and provide a separate visible-viewport text alternative. The implementation should include a long-page regression test that scrolls via evaluate or End key and then proves the follow-up snapshot represents post-scroll visible content when viewport mode is requested.
