## Summary

Keep this issue open. Current main still has no viewport-aware text snapshot mode for the browser tool; the snapshot contract remains page/root accessibility-tree based across Playwright, raw CDP, and Chrome MCP paths. The issue body's related #24951 is an act-timeout report, not a canonical duplicate, and the prior ClawSweeper comment also kept this open.

## Best Possible Solution

Keep this issue open. The best fix is to define an explicit viewport-aware text snapshot contract, implement it consistently for Playwright, raw CDP, and Chrome MCP snapshot paths, and add a long-page regression test that scrolls first and then verifies the next text snapshot reflects the current visible viewport.
