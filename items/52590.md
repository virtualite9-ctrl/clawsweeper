## Summary

Keep open. On current `main`, Feishu's manifest `skills` entry is still part of the documented skill-loading contract, and the current loader uses that field to surface bundled `SKILL.md` guidance to agents. The PR removes that contract surface but does not show a code path where manifest `skills` register Feishu tools, while Feishu tool registration still happens through `registerFull`. The PR is also stale against current `main` because one touched generated file path no longer exists.

## Evidence

- **Manifest contract:** The plugin manifest docs define `skills` as skill directories to load relative to the plugin root, not as tool-registration metadata.
  - file: [docs/plugins/manifest.md:161](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/docs/plugins/manifest.md#L161)
- **Feishu manifest still declares skills:** Current `main` keeps `"skills": ["./skills"]` in the Feishu manifest.
  - file: [extensions/feishu/openclaw.plugin.json:12](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/extensions/feishu/openclaw.plugin.json#L12)
- **Plugin skills are loaded into the skills prompt path:** `resolvePluginSkillDirs()` ignores plugins with empty `record.skills`, so removing Feishu's `skills` entry would stop this plugin from contributing its skill directories.
  - file: [src/agents/skills/plugin-skills.ts:72](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/agents/skills/plugin-skills.ts#L72)
- **Loaded plugin skill dirs are merged into workspace skill loading:** Workspace skill loading merges `resolvePluginSkillDirs(...)` into extra skill roots, which are then loaded as skills.
  - file: [src/agents/skills/workspace.ts:538](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/agents/skills/workspace.ts#L538)
- **Feishu tool registration is separate from manifest skills:** Feishu registers its runtime tools from `registerFull`, and `defineBundledChannelEntry().register()` calls that path only for full plugin loads; this code does not consult manifest `skills`.
  - file: [extensions/feishu/index.ts:73](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/extensions/feishu/index.ts#L73)
- **PR touches a stale file path:** Current `main` has `src/plugins/bundled-plugin-metadata.ts` and related tests, but not `src/plugins/bundled-plugin-metadata.generated.ts`, so the PR's second file change is against an older repo shape.
  - command: `find src -maxdepth 2 -type f | rg 'bundled-plugin-metadata'`

- No live reproduction was run in this read-only review, so the actual double-load path remains unidentified.
