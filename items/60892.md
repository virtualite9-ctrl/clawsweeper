## Summary

Current `main` already reconciles orphaned `running` task records into terminal `lost` records, uses that reconciled view for `tasks list/show/audit`, and starts the maintenance sweep automatically on gateway startup. The reported stale-ledger symptom is therefore implemented on `main` and present in release `v2026.4.23`.

## Evidence

- **Maintenance now downgrades orphaned active tasks:** `hasBackingSession()` checks cron ownership, CLI run contexts, ACP session entries, and subagent/CLI session-store presence; `runTaskRegistryMaintenance()` marks stale active tasks `lost` when that backing state is gone.
  - file: [src/tasks/task-registry.maintenance.ts:124](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/tasks/task-registry.maintenance.ts#L124)
