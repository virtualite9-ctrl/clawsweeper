## Summary

Keep PR #72033 open. Current main at 4a3030df9efa72c44bab567a4390ac0c4876a73f still exposes only `diagnostics.stability`; `diagnostics.pricing` is absent from the diagnostics handler map, READ_SCOPE authorization list, advertised gateway methods, CLI/docs, and tests. The existing pricing cache metadata helper and live startup refresh path are present, so the PR remains a focused gateway observability change rather than obsolete cleanup.

## Best Possible Solution

Keep the PR open for maintainer review. If maintainers want this operator diagnostic surface, the best path is a small core gateway change that adds `diagnostics.pricing` through the diagnostics handler, READ_SCOPE registration, advertised method list, focused empty/populated cache tests, and public protocol or operator docs if the RPC should be supported long term.
