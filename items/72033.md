## Summary

Keep PR #72033 open. Current main still does not expose `diagnostics.pricing`: the diagnostics handler, READ_SCOPE authorization table, advertised gateway method list, tests, and public protocol docs only cover `diagnostics.stability`. The PR is a focused core gateway observability addition that wires an existing pricing-cache metadata helper into the RPC surface, and the review comments were addressed on the PR head rather than making the work obsolete.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is a small core gateway change that adds `diagnostics.pricing` through the diagnostics handler, READ_SCOPE table, advertised method list, and focused empty/populated cache tests, with protocol or operator docs if maintainers want this to be a supported diagnostic RPC.
