## Summary

Keep PR #69691 open. Current main still hardcodes MiniMax VLM routing to `MiniMax-VL-01`, so the PR's requested `MiniMax-M2.7` allowlist behavior is not implemented. The PR is a focused bug-fix candidate tied to #69648, but it also conflicts with current docs/tests that intentionally route MiniMax image understanding through the plugin-owned `MiniMax-VL-01` media model, so this needs maintainer product/API judgment rather than automated cleanup.

## Best Possible Solution

Keep the PR open for maintainer review. If OpenClaw should support direct `minimax/MiniMax-M2.7` image requests through the VLM endpoint, land a version of this change with updated guard tests, corrected test wording, and docs reconciled with the current `MiniMax-VL-01` media-provider model. If maintainers prefer the current VL-01-only image route, close the PR with that explicit product rationale rather than automated stale cleanup.
