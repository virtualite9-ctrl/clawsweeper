## Summary

Keep #70607 open. Current main has not implemented the PR's requested cross-agent account preservation, but the PR is also not safe to close or merge as-is: the proposed one-character guard flip conflicts with the existing same-agent caller-account invariant and with multiple current tests that intentionally route cross-agent child/thread work through the target agent's bound account. This needs maintainer product judgment or a more targeted split between parent completion routing and child thread/binding routing.

## Best Possible Solution

Keep the PR open for maintainer/author follow-up. The best fix is to reproduce #70574 on current main, then either document the existing target-agent-bound-account contract or implement a targeted routing split: preserve the parent account for cross-agent completion announcements if that is the intended user-visible behavior, while keeping same-agent caller-account preservation and any target-agent account needed for child thread binding/ACP setup. Update `spawn-requester-origin` plus the native subagent, ACP, lifecycle, and thread-binding tests accordingly.
