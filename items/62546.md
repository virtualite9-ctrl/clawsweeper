## Summary

Close as `cannot_reproduce`: the reported 401 comes from opening the internal `/__openclaw__/canvas/` surface with a query token, while current `main` documents the browser dashboard at `/` and implements auth bootstrap there. The canvas route intentionally requires bearer auth or a canvas capability and is expected to return 401 in the reported scenario, so this does not show a Windows-specific dashboard regression.

## Evidence

- **Dashboard entrypoint is `/`, not `/__openclaw__/canvas/`:** The public dashboard docs say the Gateway dashboard/Control UI is served at `/` by default and the local quick-open URL is `http://127.0.0.1:18789/`.
  - file: [docs/web/dashboard.md:8](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/docs/web/dashboard.md#L8)

- **Control UI token bootstrap uses top-level URL token import, preferring `#token=`:** The UI bootstrap code prefers fragment tokens over query tokens, warns on `?token=`, imports them once for compatibility, and strips query tokens from the URL after bootstrap.
  - file: [ui/src/ui/app-settings.ts:192](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/ui/src/ui/app-settings.ts#L192)

- **Public Control UI docs match the bootstrap behavior:** The Control UI docs say `token` should be passed via the URL fragment (`#token=...`) and that legacy `?token=` query params are only a fallback imported once during bootstrap.
  - file: [docs/web/control-ui.md:426](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/docs/web/control-ui.md#L426)

  - sha: [c070509b7f8f](https://github.com/openclaw/openclaw/commit/c070509b7f8fdbe479e5d829643782c272e30e83)
- **Canvas auth path does not accept query-token login:** `authorizeCanvasRequest` only checks a bearer token from the request headers, then falls back to a canvas capability tied to an active WebSocket client. It does not read `?token=` for canvas routes.
  - file: [src/gateway/server/http-auth.ts:69](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/src/gateway/server/http-auth.ts#L69)
