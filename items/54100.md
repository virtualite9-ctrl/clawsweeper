## Summary

Current main already routes Slack Socket Mode channel/group traffic through the generic `message` event handler and has regression tests covering channel, private-group, and `app_mention` delivery, so the reported drop is implemented/fixed on main.

## Evidence

- **Current main uses unified Slack message handler:** `registerSlackMessageEvents` registers `ctx.app.event("message")` and documents that Slack subscriptions like `message.channels`/`message.groups` still arrive as `type: "message"`, with `channel_type` distinguishing channel vs private group. `app_mention` is handled separately and routed into the same message path for non-DM channels.
