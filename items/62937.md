## Summary

Keep open. This draft PR is a same-author implementation candidate for the still-open paired feature issue #58730, and current main still does not provide a macOS `sandbox-exec` backend or `sandboxProfile` exec parameter. The remaining decision is product/security design, not conservative cleanup.

## Best Possible Solution

Keep this PR open alongside #58730 for maintainer product/security review. If OpenClaw wants native macOS Seatbelt support, redesign it as a sandbox backend or backend capability under the existing `agents.defaults.sandbox` and `buildExecSpec` model, with config docs and real regression coverage. If maintainers decide macOS `sandbox-exec` support does not belong in core, close this PR and the paired issue together after that explicit decision.
