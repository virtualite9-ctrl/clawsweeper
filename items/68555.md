## Summary

Keep open. Current main still implements and documents the silent fallback behavior this PR targets, so the PR is not obsolete cleanup. The patch is focused, and the bot-reviewed `getLog` ReferenceError was addressed in the second PR commit, but the current revision still needs maintainer-quality follow-up before merge: avoid direct `console.error` in the pure resolver, add/update tests, and update docs if the behavior changes from warning fallback to hard failure.

## Best Possible Solution

Keep this PR open as a relevant focused bug-fix candidate. The best path is to revise it so explicit `payload.model` allowlist rejection returns `ok: false` with a clear error, let the existing caller surface/log that structured failure instead of adding `console.error` inside `resolveCronModelSelection`, update the tests that currently expect fallback, and update the cron docs if maintainers accept the behavior change.
