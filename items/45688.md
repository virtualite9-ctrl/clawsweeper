## Summary

Keep #45688 open. Current main has config UI improvements, but the main Config path still defaults to Quick Settings and its advanced mode still uses an accordion surface without the top search/tab bar. Large parts of config are also split into separate scoped tabs, so there is not strong evidence that the reported v2 global findability/filtering regression is solved.

## Best Possible Solution

Keep the issue open for a maintainer UI decision. A proper fix should either restore a single searchable/filterable full-config surface or add equivalent global search/filter/navigation across Quick Settings, advanced accordion, and the scoped settings tabs, with browser tests covering the actual main Config advanced path.
