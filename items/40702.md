## Summary

Current `main` already contains the Telegram approval-resolution path that addresses this report: Telegram callback handlers resolve approvals through the shared gateway resolver, non-`plugin:` IDs go to `exec.approval.resolve` first, and only approval-not-found responses can fall back to `plugin.approval.resolve`. The behavior is covered by tests and documented, and the same resolver/test files are already present in release `v2026.4.22`.

## Evidence

## Close Comment

Closing as implemented on `main` after a Codex review. The current Telegram callback path already resolves approval buttons through the shared gateway resolver (`extensions/telegram/src/bot-handlers.runtime.ts:1453`), and that resolver sends non-`plugin:` IDs to `exec.approval.resolve` first with a bounded fallback to `plugin.approval.resolve` only for approval-not-found responses (`src/infra/approval-gateway-resolver.ts:16`, `src/infra/approval-errors.ts:18`).
