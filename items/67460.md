## Summary

Keep this PR open. Current main still lacks the explicit-agent-mention precedence behavior requested by #67445, and this PR is a live implementation candidate, but it is not ready to close or land as-is because the patch bundles unrelated install/test changes and still carries functional and security-sensitive risks.

## Best Possible Solution

Keep this PR open for maintainer review, but narrow it before landing: split the postinstall and PDF-test changes, restore Discord `memberRoleIds`, decide whether Discord should opt into the new behavior, make the suppression signal mean another OpenClaw agent rather than any platform mention, and update SDK docs/tests for the new public mention policy.

## Risks / Open Questions

- Closing this PR now would discard a useful active implementation while current main still lacks the explicit-agent-mention precedence behavior requested by #67445.
- Landing the PR as-is would drop Discord sender role context by removing `memberRoleIds` from the preflight return payload.
- The suppression condition currently treats generic platform mentions as proof that another OpenClaw agent was mentioned; Discord role/everyone mentions, Slack user mentions, Teams mentions, and WhatsApp JIDs need a narrower other-agent signal before broad opt-in.
- The PR bundles unrelated postinstall and PDF-test changes with mention routing, increasing maintainer review scope and regression risk.
- The postinstall diff changes install-time recursive deletion behavior and needs a separate security review or a separate PR with boundary checks.
