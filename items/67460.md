## Summary

Keep this PR open. Current main still lacks the explicit-agent-mention precedence behavior requested by #67445, and the PR remains the live implementation thread. It should not be closed as obsolete or implemented, but it also should not land as-is because the current patch mixes unrelated install/test changes and still carries functional and security-sensitive review risks.

## Best Possible Solution

Keep this PR open for maintainer review, but narrow it before landing: split the postinstall and PDF-test changes, restore Discord `memberRoleIds`, make the suppression signal represent another OpenClaw agent rather than any platform mention, decide which channels should opt in, and update SDK docs/tests for the new public mention-policy contract.

## Risks / Open Questions

- Closing this PR would discard the canonical active implementation while current main still lacks the behavior requested by #67445.
- Landing the PR as-is would drop Discord `memberRoleIds` from the preflight payload, losing role context used by downstream Discord processing.
- The suppression signal in the PR treats generic platform mentions as proof that another OpenClaw agent was mentioned; that can suppress normal group chatter mentioning a person, role, or everyone depending on the channel.
- The PR bundles unrelated postinstall cleanup and a PDF test model constant change with mention routing, increasing review scope and regression risk.
- The postinstall diff changes recursive deletion behavior during install and needs either boundary checks in a separate PR or a dedicated security review before landing.
