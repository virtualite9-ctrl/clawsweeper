## Summary

Keep #70905 open. Current main still does not expose a stable agent id on `ChannelOutboundContext`; the id is carried in upstream session/mirror paths but is dropped before channel outbound adapter calls. This is a concrete additive plugin SDK/core seam, not a ClawHub-only integration request.

## Best Possible Solution

Implement the additive core SDK change: add optional `agentId?: string` to `ChannelOutboundContext`, thread the effective delivery agent id through `DeliverOutboundPayloadsParams`/`ChannelHandlerParams`/`createChannelOutboundContextBase`, and cover `sendText`, `sendMedia`, `sendPayload`, and formatted delivery contexts with regression tests plus channel plugin docs.

## Risks / Open Questions

- `ChannelOutboundContext` is a public plugin contract, so the implementation should remain additive, documented, and covered by focused SDK/outbound tests.
- The effective id precedence should be explicit, likely `params.session?.agentId ?? params.mirror?.agentId`, while preserving absence for human/system sends without an agent in scope.
