## Summary

Keep this issue open. Current main still does not expose a stable agent id on ChannelOutboundContext, and deliverOutboundPayloads still drops the existing session or mirror agent id when it constructs channel adapter send contexts. The request is an additive channel plugin SDK/core plumbing change, not a ClawHub-only plugin idea, because a plugin cannot infer this metadata for non-inbound sends without core passing it through.

## Best Possible Solution

Add optional agentId?: string to ChannelOutboundContext, thread a documented source such as params.session?.agentId ?? params.mirror?.agentId through ChannelHandlerParams and createChannelOutboundContextBase, and verify it reaches sendText, sendMedia, sendPayload, sendFormattedText, sendFormattedMedia, and renderPresentation contexts. Update channel plugin docs to distinguish stable agent attribution from display identity.
