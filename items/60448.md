## Summary

Keep open. Current `main` still injects sender/conversation metadata into the model-facing user prompt, including sender id/name/username-style fields; the codebase only strips those blocks back out at UI/history display boundaries.

## Evidence

- **Prompt assembly still prepends inbound metadata before the user body is sent to the agent:** `get-reply-run.ts` builds `inboundUserContext` with `buildInboundUserContextPrefix(...)` and joins it into `baseBodyForPrompt`, so the metadata block is part of the model input rather than just a display artifact.
  - file: [src/auto-reply/reply/get-reply-run.ts:436](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/src/auto-reply/reply/get-reply-run.ts#L436)
