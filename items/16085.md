## Summary

Keep #16085 open. Current main still has the bundled Signal plugin wired to native signal-cli JSON-RPC + SSE only, with no adapter/container client, no channels.signal.apiMode config, no bbernhard/signal-cli-rest-api docs, and no ws runtime dependency. The PR remains the active implementation candidate for container REST/WebSocket support after #10240 was closed.

## Best Possible Solution

Keep #16085 open as the active implementation candidate for bundled Signal container REST/WebSocket support. Review the adapter/container design in extensions/signal, require aligned config schema/help/docs/tests, verify the ws dependency handling, and prove native signal-cli JSON-RPC/SSE behavior remains unchanged before merge.

- The PR body says live containerized deployment was not verified end to end; maintainers should require live proof or an explicit manual verification note before landing.
