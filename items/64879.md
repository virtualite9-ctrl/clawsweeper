## Summary

Keep open. Current main still has providerVariant tagging and mock parity workflows, but the mock dispatcher does not use provider/model information to produce divergent OpenAI vs Anthropic plans, and the live workflows do not add the requested real-provider parity-report lane.

## Best Possible Solution

Keep this issue open. Implement Option A by deriving the provider variant inside extensions/qa-lab/src/providers/mock-openai/server.ts buildResponsesPayload() and adding provider-specific planned-tool/prose branches for at least three agentic parity scenarios, with tests asserting divergent OpenAI vs Anthropic plans. If maintainers prefer Option B, add a separate secret-gated CI lane that runs the same parity pack against real openai/gpt-5.x and anthropic/claude-opus-4-6 summaries and feeds them into openclaw qa parity-report.
