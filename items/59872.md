## Summary

Keep #59872 open. Current `main` still exposes `web_fetch` as a generic URL fetcher with no Twitter/X domain blocklist, rewrite route, or configurable tool-description override. The related FxTwitter fallback PR #66047 was closed unmerged, its proposed files are absent from `main`, and no equivalent implementation appears in the current web-fetch path or config schema.

## Best Possible Solution

Keep the issue open for maintainer/product design. The likely fix belongs in the core web-fetch/tool-definition layer: either a generic domain blocklist/routes mechanism with clear config/schema/docs/tests, or a deliberate Twitter/X-specific interception path with tests and fallback behavior. Any plugin/provider solution would need a new pre-fetch routing or tool-description extension seam before it can fully address this report.
