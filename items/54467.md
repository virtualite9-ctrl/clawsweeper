## Summary

Keep #54467 open. Current main still has no per-instance Bonjour service instance-name config: the bundled bonjour plugin always passes `formatBonjourInstanceName(ctx.machineDisplayName)` into the advertiser, while the advertiser only consults `OPENCLAW_MDNS_HOSTNAME` when no `opts.instanceName` was supplied. The recent #54910 close only covers the advertised hostname/TXT `lanHost` path, not this narrower same-host multi-gateway service instance-name collision.

## Best Possible Solution

Keep #54467 open and implement the fix in the bundled bonjour plugin: add a validated `instanceName` field to `extensions/bonjour/openclaw.plugin.json`, read `api.pluginConfig.instanceName` in `extensions/bonjour/index.ts`, pass the configured value through the existing `formatBonjourInstanceName` semantics into `startGatewayBonjourAdvertiser`, and add focused tests/docs for `plugins.entries.bonjour.config.instanceName`. Avoid adding a new core `gateway.bonjour.*` key because plugin-owned config already has a supported SDK path.
