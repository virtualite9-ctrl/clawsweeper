## Summary

Close as superseded by the active implementation PR #72055. Current main still lacks the per-instance Bonjour service-name config, but #72055 directly tracks and implements the remaining work with the plugin-scoped `plugins.entries.bonjour.config.instanceName` path, tests, docs, and changelog.

## Best Possible Solution

Close #54467 as superseded by #72055 and finish the fix there: add the bonjour plugin `instanceName` config schema, read `api.pluginConfig.instanceName` in `extensions/bonjour/index.ts`, keep the existing suffix semantics, and land the focused tests/docs/changelog from the PR. If #72055 is abandoned, reopen or recreate the same plugin-scoped implementation rather than adding a core `gateway.bonjour.*` key.

## Evidence
