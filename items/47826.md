## Summary

Current `main` does not implement the requested read-only gateway REST endpoints. The gateway HTTP server wires `/health|/healthz|/ready|/readyz`, OpenAI-compatible `/v1/*`, `/tools/invoke`, and a few specialized routes such as session kill/history, but there is no handler for `/api/sessions`, `/api/agents`, or `/api/health`. The underlying `agents.list`, `sessions.list`, and `sessions.get` capabilities still exist as gateway RPC methods rather than direct REST endpoints, so the issue is not implemented on `main`.
