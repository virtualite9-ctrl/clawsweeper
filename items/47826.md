## Summary

Keep open. Current main still does not expose the requested lightweight REST endpoints for `sessions.list`, `agents.list`, combined status, or `sessions.get`. The gateway HTTP router only wires existing built-in HTTP surfaces such as `/v1/models`, `/v1/embeddings`, `/v1/chat/completions`, `/v1/responses`, `/tools/invoke`, `/sessions/:key/kill`, `/sessions/:key/history`, plugin routes, Control UI, and health/readiness probes. The directly related PR #47863 is closed and unmerged, so it does not supersede the issue.

## Best Possible Solution

Keep #47826 open unless maintainers explicitly consolidate it into a canonical HTTP gateway-query tracker. A complete fix should add authenticated read-only Gateway HTTP endpoints for sessions, agents, status, and single-session reads, reuse existing `listSessionsFromStore`, `listAgentsForGateway`, and `sessions.get` semantics, and include route/auth/scope tests plus public gateway HTTP API docs.
