Keep this PR open. Current main still does not implement the proposed gateway/node daemon heap-size config or service `NODE_OPTIONS` injection, and the PR remains a plausible implementation candidate for related OOM work rather than obsolete cleanup.

## Best Possible Solution

## Risks / Open Questions

- The PR should not preserve or append arbitrary install-time `NODE_OPTIONS`; current main treats that as a blocked startup-control surface.
- `resolveNodeOptions` should detect both `--max-old-space-size` and Node's accepted `--max_old_space_size` spelling before adding a heap flag.
- The 85% of total system RAM default is a product/runtime policy choice that may be too aggressive on shared or memory-constrained hosts.
- The branch needs a rebase onto `src/wizard/setup.gateway-config.ts` and current generated config/schema documentation surfaces.
