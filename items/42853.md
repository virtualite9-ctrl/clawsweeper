## Summary

Keep open. Current main still lacks the daemon `NODE_OPTIONS` heap injection and `gateway.nodeMaxOldSpaceMb` config surface, and the related OOM context still points to this as useful unfinished work. The PR is not obsolete, but it needs a refresh and security-conscious adjustment before merge.

## Best Possible Solution

Keep this PR open for maintainer review, but require a refresh against current main. The best implementation should wire a documented `gateway.nodeMaxOldSpaceMb` through the gateway and node service install plans, update schema/help/generated docs as needed, construct a safe heap-only `NODE_OPTIONS` value without carrying arbitrary startup flags, recognize both Node heap flag spellings, and add focused daemon/config/wizard tests before landing.

## Risks / Open Questions

- The PR should not persist arbitrary install-time `NODE_OPTIONS`; that would weaken existing host-env hardening around Node startup flags.
- The heap-flag detector should handle both `--max-old-space-size` and `--max_old_space_size` to avoid silently appending a second heap cap.
- The 85% of total RAM default is a product/runtime policy choice that needs maintainer review, especially on shared or memory-constrained hosts.
- The PR predates current main's renamed wizard path (`src/wizard/setup.gateway-config.ts`), so it likely needs a rebase/refactor before it can be evaluated for merge.
