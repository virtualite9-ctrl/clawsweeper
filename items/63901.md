## Summary

Keep open. Current main still does not implement the requested session-aware provider request metadata: it supports static `extra_body`/`extraBody` passthrough and OpenRouter routing/app attribution, but not templated `trace`, automatic `user = sessionKey`, or a provider extra-param context carrying `sessionKey`. The related open PRs #54725 and #64813 remain implementation candidates, and #64813 is a same-author paired PR that should be reviewed or closed before this issue is closed.

## Best Possible Solution

Keep this issue open. The best path is to review/land or explicitly close the related implementation PRs, preferably by adding a session-aware provider hook or a narrowly scoped request-body attribution option with coverage for `sessionKey`, `sessionId`, agent id, static `extra_body` precedence, and OpenRouter/LiteLLM-compatible `user`/`trace` behavior.
