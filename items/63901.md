## Summary

Keep open. Current `main` does not implement OpenRouter `trace`/`user` request metadata or a session-key equivalent. The OpenRouter plugin currently forwards only provider routing metadata plus generic proxy wrappers, and the provider hook inputs still do not expose session/task identity needed for the proposed templating.

## Evidence

- **OpenRouter wrapper only consumes provider routing metadata:** `wrapOpenRouterProviderStream(...)` reads `ctx.extraParams.provider` and injects it into `model.compat.openRouterRouting`; it does not map any `user`, `trace`, session, graph, or task fields into the request payload.
