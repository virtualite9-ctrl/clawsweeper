## Summary

Keep open. Current main still does not implement dynamic provider request metadata for cost attribution. It has static `params.extra_body`/`extraBody` passthrough for OpenAI-compatible request bodies and native OpenAI transport turn metadata keyed by `sessionId`, but there is no templated `trace`, no automatic `user = sessionKey`, and no session-aware provider extra-param or stream-wrapper context for OpenRouter/LiteLLM-style request-body attribution. The related open PRs are implementation candidates, and the same-author paired PR #64813 is an additional reason to keep this issue open until reviewed or merged.

## Best Possible Solution

Keep this issue open and review the linked implementation path. The best fix is either to land a session-aware provider hook such as #54725 so plugins can safely inject `user`/`trace`, or to land a narrowly scoped request-body attribution option with tests for `sessionKey`, `sessionId`, agent id, and precedence with static `extra_body`; the same-author PR #64813 should be reviewed or closed before this issue is closed.
