## Summary

Keep open. Current main still uses the older direct-promotion dreaming path: scheduled deep dreaming ranks candidates and immediately calls `applyShortTermPromotions`, which appends to `MEMORY.md`. The PR's central stage-only maintenance loop with explicit `/dreaming apply`, `/dreaming rollback`, maintenance artifacts, `dailySignalFiles`, and managed-block-only writes is not implemented on main. The PR also has unresolved review concerns and unrelated Bedrock Mantle changes, so it should not be auto-closed as implemented or superseded.

## Best Possible Solution

Keep the PR open for maintainer/product review. The useful path is to make the memory-core change focused, resolve the apply-before-rollback-record P1, fix the configured daily-signal ingestion path, remove or split unrelated Bedrock Mantle edits, and then align docs, plugin schema, command help, and tests around the stage-only apply/rollback contract.
