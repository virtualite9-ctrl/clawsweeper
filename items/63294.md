## Summary

Current main already solves the reported custom-provider Responses reasoning path. The embedded runner now routes default OpenAI Responses traffic through OpenClaw’s boundary-aware transport, and that transport maps both pi-agent-core’s `options.reasoning` and the newer `options.reasoningEffort` into the Responses `reasoning` payload when the model declares `reasoning: true`. The same implementation is present in release v2026.4.24.

## Best Possible Solution

Close this issue as implemented and keep the current boundary-aware OpenAI Responses transport plus `options.reasoning` fallback. The remaining documented requirement is still correct: custom provider models must declare `reasoning: true` for OpenClaw to emit Responses reasoning parameters.
