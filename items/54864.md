## Summary

Keep open. Current main still does not expose `openclaw agent --new-session` or return a generated top-level `sessionId` in JSON output. Current main does have useful groundwork from the related session-id isolation work, and related PR #69108 proposed this exact feature, but that PR is closed and unmerged and the requested CLI flag remains absent.

## Best Possible Solution

Implement this as a narrow core CLI feature: add `--new-session` to `openclaw agent`, plumb `newSession` through `AgentCliOpts`, reject it with `--session-id`, generate one `crypto.randomUUID()` in `agentCliCommand`, pass that effective `sessionId` through gateway/local/fallback paths, include top-level `sessionId` in JSON output, and update parser, command, session-resolution, docs, and JSON-output tests.

- Future implementation should generate the session id exactly once before gateway/local/fallback selection so one invocation cannot split across sessions.
