## Summary

Keep open. Current main still lacks the requested `openclaw agent --new-session` flag and generated top-level `sessionId` JSON contract. The lower-level `--session-id` isolation path is present, and related PR #69108 proposed this feature, but that PR is closed and unmerged.

## Best Possible Solution

Implement this as a narrow core CLI feature: add `--new-session` to `openclaw agent`, plumb `newSession` through `AgentCliOpts`, reject it with `--session-id`, generate one `crypto.randomUUID()` in `agentCliCommand`, pass that id as the effective `sessionId` through gateway/local/fallback paths, add top-level `sessionId` to JSON output, and update parser, gateway/embedded, session-resolution, and docs coverage.
