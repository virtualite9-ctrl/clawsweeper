## Summary

Keep open. Current main has the lower-level `--session-id` isolation path, but it still does not expose the requested `openclaw agent --new-session` CLI flag, generate a fresh id for callers, or document/test top-level JSON reporting for that generated id. Related PR #69108 attempted this work, but the provided GitHub context shows it was closed unmerged, so it does not supersede the issue.

## Best Possible Solution

Keep this issue open and implement a narrow CLI feature: add `--new-session` to `openclaw agent`, make it mutually exclusive with `--session-id`, generate a fresh `crypto.randomUUID()` in the shared `agentCliCommand` path before gateway/local dispatch, pass it as the effective `sessionId`, surface the generated id in JSON output for reconnects, and update CLI docs plus parser/session-resolution tests.
