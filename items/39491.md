## Summary

Current `main` already implements the requested Mattermost fallback: the preview-final edit path first chunks the final text and only edits in place when it fits as a single chunk; otherwise it falls back to normal chunked delivery and best-effort preview cleanup.

## Close Comment

Closing from Codex review: current `main` already implements this fallback. In `extensions/mattermost/src/mattermost/monitor.ts:1635`, the preview-final path now runs `chunkMarkdownTextWithMode(...)` and only edits the streaming preview in place when the final text fits in exactly one chunk; otherwise it falls through to normal delivery. Normal delivery goes through `extensions/mattermost/src/mattermost/reply-delivery.ts:24`, which chunks long text before sending, and the generic preview finalizer in `src/channels/draft-preview-finalizer.ts:50` clears the orphaned preview only after successful fallback delivery, with best-effort delete behavior underneath. This behavior is also covered by the Mattermost and draft-finalizer tests on `main`. I did not identify a released tag containing this yet, so I’m closing as implemented on `main` rather than as shipped in a known release.
