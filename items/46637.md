## Summary

Keep open. Current main still captures OpenAI-compatible `reasoning_content` as a signed thinking block and the current replay policy for generic/strict `openai-completions` history does not strip or sanitize that block before the next request. The linked fix PRs for #46637 were closed unmerged, and later related ClawSweeper reports still found the same replay surface open for other OpenAI-compatible models.

## Best Possible Solution

Keep #46637 open. The best fix is a targeted replay-history policy for strict OpenAI-compatible backends such as oMLX/Qwen: strip or sanitize prior-turn `reasoning_content` thinking blocks before `convertMessages()` rehydrates them, while preserving documented provider-owned exceptions like DeepSeek and Z.AI. Add regression coverage around `sanitizeSessionHistory` plus `buildOpenAICompletionsParams()` with `thinkingSignature: "reasoning_content"` and unsafe control characters.
