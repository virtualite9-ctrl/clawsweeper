## Summary

Do not auto-close. #72016 asks for a persistent extension point that lets custom checks and fixes participate in `openclaw doctor`. Current `main` has plugin infrastructure and narrow channel/plugin doctor compatibility hooks, but no public API for arbitrary installed plugins to add custom doctor checks/fixes to the core doctor run.

## Best Possible Solution

Keep the issue open for maintainer product/API design. The likely path is an additive plugin-facing doctor contribution API, such as `registerDoctorCheck` or a manifest-plus-runtime doctor contribution contract, with read-only diagnostics, explicit opt-in repair mutations, stable ordering, non-interactive semantics, sanitized output, and tests/docs. Once that core API exists, environment-specific checks like this can live in ClawHub or local plugins without being overwritten by npm upgrades.
