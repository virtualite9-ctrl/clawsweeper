## Summary

Keep open. Current main has plugin-owned doctor compatibility hooks for legacy config migrations and channel-specific warnings/repairs, but it does not expose a general plugin API for adding arbitrary upgrade-stable `openclaw doctor` checks with `--fix` behavior. That makes this a plausible core/plugin-SDK design gap, not a ClawHub-only plugin request.

## Best Possible Solution

Keep the issue open as a core/plugin-SDK design item. The best path is an additive doctor contribution API, likely a typed `registerDoctorCheck` or doctor contribution surface with read-only diagnostics plus explicit repair hooks that respect `--fix`, `--non-interactive`, config persistence, plugin trust/loading policy, and docs/tests. Existing `doctor-contract-api` should stay scoped to compatibility migrations unless maintainers intentionally expand and document it.
