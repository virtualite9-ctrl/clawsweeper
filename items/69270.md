## Summary

Keep #69270 open. Current main does not contain the PR's coordinated compaction/reset invariant fixes: hook contexts still omit messageProvider in reset and subscribe-time compaction paths, and zero-token/manual compaction accounting can still leave stale token/cost/cache fields.

## Best Possible Solution

Keep the PR open and review it as the active implementation for the compaction/reset invariant cluster. The best path is to land a focused version that preserves hook messageProvider identity across subscribe compaction, engine-owned compaction, command reset, and gateway reset, while also resetting token/cost/cache bookkeeping for zero-token and manual compaction outcomes with the regression tests proposed in the PR.
