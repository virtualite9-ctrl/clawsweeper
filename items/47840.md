## Summary

Keep open. Current `main` still renders raw `modelProvider`/`providerOverride` values in the Control UI usage surfaces, and the usage RPC still forwards the stored raw provider string instead of a canonicalized value. The model picker has some newer normalization logic, and the Sessions table no longer shows a provider column, so parts of the original report have shifted, but the core inconsistency is still present on the usage page.
