## Summary

Keep open. Current main has canonical provider normalization for supported aliases such as `z-ai`/`z.ai` and `aws-bedrock`/`bedrock`, but Control UI usage display, filter, and export paths still consume raw provider strings from session/usage data. The related picker issues (#47620/#47621) address adjacent model-ref behavior, not this remaining provider display consistency gap. The exact `bailian` example is not present in current main, so that alias should be clarified separately.

## Best Possible Solution

Keep the issue open and fix the remaining display boundary. Add a shared provider-display normalizer aligned with `normalizeProviderId`, apply it to Control UI usage badges, filter suggestions, CSV export, and usage/session API provider fields as appropriate, and add regression coverage for supported aliases such as `z-ai -> zai` and `aws-bedrock -> amazon-bedrock`. If raw aliases are useful for debugging, show them as secondary tooltip text rather than the primary provider label.
