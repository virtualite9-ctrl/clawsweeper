## Summary

Keep this issue open. Current main still does not implement the requested first-class config/runtime surfaces: sandboxing exists under `agents.defaults.sandbox` / `agents.list[].sandbox`, but there is no root `sandbox` key, no `routing.rules` tag-to-provider/model router, no root `instances` declaration surface, and no `gateway.nodes.denyPaths` schema or path-enforcement chokepoint.

## Best Possible Solution

Keep this issue open and split the work deliberately: decide whether a root `sandbox` key should alias or replace the existing `agents.defaults.sandbox` semantics; design tag-aware provider/model routing with explicit metadata propagation and tests; decide whether multi-instance declarations belong in config or in the existing profile/gateway management model; and implement `gateway.nodes.denyPaths` only with schema/types/docs, doctor coverage, runtime path enforcement, and security tests.

## Risks / Open Questions

- The issue bundles four separable config/product surfaces; a single broad schema-only change would be hard to review safely.
- `gateway.nodes.denyPaths` is security-sensitive and would need runtime enforcement at file/tool/node chokepoints plus doctor and security tests, not only schema/types/docs.
- `routing.rules` needs an explicit metadata path from skill/agent calls into model selection; adding config before that would create a misleading policy surface.
- `instances.*` overlaps with the existing profile/config/state/port model for multiple gateways and needs maintainer product direction before implementation.
