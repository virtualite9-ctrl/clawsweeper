## Summary

Keep this issue open. Current main has adjacent sandbox, routing, multi-gateway, and node command-policy controls, but it still does not implement the requested config/runtime surface: no root `sandbox`, no `routing.rules`, no `instances`, and no `gateway.nodes.denyPaths`.

## Best Possible Solution

Keep this issue open and split the work deliberately: decide whether a root sandbox alias/policy should wrap `agents.defaults.sandbox`, design tag-based provider/model routing in the agent/model-selection layer with explicit metadata propagation, decide whether multi-instance declarations belong in config or profile/gateway management, and implement `gateway.nodes.denyPaths` only with schema/types/docs/doctor or security-audit coverage plus runtime path enforcement tests.

## Risks / Open Questions

- The issue bundles four separable product/API surfaces; implementing them as one schema change would raise compatibility and review risk.
- `gateway.nodes.denyPaths` would need a real enforcement design across gateway/node/filesystem/tool chokepoints, not just a schema field.
- `routing.rules` and `instances.*` need maintainer API direction before a conservative implementation can choose durable field shapes.
