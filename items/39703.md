## Summary

Keep #39703 open. Current main still implements FIFO behavior for the cron evaluator cache: cache hits return without promotion, and capacity eviction deletes the first Map key. The PR remains the focused canonical fix and its second commit addresses the earlier review feedback with key-level regression assertions.

## Best Possible Solution

Keep #39703 open as the canonical remaining cron cache LRU fix, or land an equivalent maintainer-owned patch that promotes cache hits with delete/re-set and includes key-level regression coverage proving the promoted key survives eviction while the next-oldest key is evicted.
