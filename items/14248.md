## Summary

Keep #14248 open. Current main still does not expose Slack Socket Mode ping/pong tuning in config, schema, runtime, tests, or docs. The issue also has recent production-impact comments, and the linked implementation PR #41473 is still open and unmerged, so it does not supersede the issue yet.

## Best Possible Solution

Keep #14248 open until PR #41473 or an equivalent patch lands on main. The fix should add a supported `channels.slack.socketMode` config object for `clientPingTimeout`, `serverPingTimeout`, and `pingPongLoggingEnabled`; validate and document it in Slack config types/schema/metadata/UI hints; preserve account inheritance; pass the values to `SocketModeReceiver` before start; add focused config/runtime tests; and update the public Slack/config docs.
