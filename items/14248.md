## Summary

Keep #14248 open. Current main at 7e13f3f5140168e31027b39a9855ade3fadf24d5 still lacks a supported Slack Socket Mode ping/pong tuning config surface, and PR #41473 is only an open implementation path in the provided GitHub context, not shipped behavior on main.

## Best Possible Solution

Keep #14248 open until PR #41473 or an equivalent patch lands on main. The best fix is to add a supported channels.slack.socketMode config surface for clientPingTimeout, serverPingTimeout, and pingPongLoggingEnabled, validate it in the Slack config schema/types, pass it into SocketModeReceiver construction, cover account inheritance and strict-schema behavior with focused tests, and update Slack docs/config metadata.
