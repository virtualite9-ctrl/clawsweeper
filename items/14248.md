## Summary

Keep #14248 open. Current main still does not expose Slack Socket Mode ping/pong tuning in the Slack config type, strict zod schema, UI/docs metadata, or runtime SocketModeReceiver construction. The related PR #41473 is an open, unmerged implementation path, not evidence that the issue is fixed on main.

## Best Possible Solution

Keep #14248 open until Slack adds a supported `channels.slack.socketMode` config surface, including type/schema/help/UI/docs coverage, per-account inheritance behavior, focused tests, and runtime wiring that passes `clientPingTimeout`, `serverPingTimeout`, and `pingPongLoggingEnabled` into the Slack Socket Mode client through the safest available dependency-backed path. PR #41473 is the related implementation starting point, but the issue should close only after that or an equivalent patch lands on main.
