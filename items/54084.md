## Summary

Current `main` no longer matches this PR’s premise. The CLI now registers only the invoked primary command lazily before parse, and `pairing` remains a lazy subcommand, so `openclaw pairing list` is not going through the old "register all commands and dependencies" path this PR was optimizing.
