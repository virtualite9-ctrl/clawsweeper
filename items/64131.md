## Summary

Keep open. The report has multiple independent confirmations, and current `main` does not provide strong enough evidence that the Dreams page header/layout overlap is fully resolved. The non-chat Dreams header still uses a shared single-row, overflow-hidden header without Dreams-specific responsive wrapping, so the reported zoom/overlap behavior remains plausible.

## Evidence

- **Multiple users corroborated the same UI regression:** The issue body reports a regression on OpenClaw 2026.4.9 with an exception/display abnormality on the Dreams page, and four follow-up comments independently confirm the same problem; one commenter specifically reports the top-right controls obscuring content and calls out an abnormally high stacking context affecting both Safari and Chrome.
  - command: `provided GitHub context for issue #64131 and comments`
- **Dreams controls still render inside the shared content header:** Current `main` renders the Dreams refresh/toggle controls inside the shared `.content-header` / `.page-meta` area rather than a Dreams-specific responsive container.
