## Summary

Current main still does not implement the central request. Memory indexing still uses fixed size chunking through `chunkMarkdown(content, { tokens, overlap })`; there is no heading/section strategy in the chunker, config schema, resolved config, reindex metadata, or user docs. The related discussion adds current production examples and points to useful implementation candidates, but those PRs are unmerged or closed unmerged, so this should stay open.

## Best Possible Solution

Keep this issue open. The best maintainer path is to implement heading-aware or section-aware chunking in the memory host SDK path used by memory-core, add a public config/schema/help/docs surface, persist the strategy in reindex metadata, and add focused tests proving adjacent `##`/`###` sections do not blend into one embedding/FTS chunk. Treat entity extraction and graph memory as a separate memory-core or memory-wiki design track unless maintainers intentionally keep this as the umbrella.
