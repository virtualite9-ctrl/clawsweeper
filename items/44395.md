## Summary

Keep #44395 open. Current main still indexes memory search content through the builtin `chunkMarkdown(content, { tokens, overlap })` path, which accumulates lines by estimated character budget and flushes only on size, not Markdown heading or section boundaries. Related work exists, including the comment linking #28071, #48795, #48808, #49108, and #51913, but the heading-aware memory-search chunker/config path is not shipped on current main.

## Best Possible Solution

Keep this issue open as the user-facing tracker unless maintainers choose a canonical replacement. The best implementation path is to add an opt-in heading/section-aware chunking mode in `packages/memory-host-sdk/src/host/internal.ts`, expose it through `agents.defaults.memorySearch.chunking`, persist it in memory index metadata so changing it triggers reindex, add memory-host-sdk and memory-core indexing tests for adjacent `##` sections, and document the config. Entity extraction/graph memory should remain a separate follow-up unless scoped into a memory plugin/backend API.
