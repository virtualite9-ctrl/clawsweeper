## Summary

Keep #44395 open. Current main still uses size-based active memory-search chunking with only token/overlap settings, and the requested heading-aware chunk boundaries are not implemented on the indexing path. Adjacent section-aware dreaming/REM code and memory-wiki source compilation do not solve the reported `memory_search` chunk blending, and automatic entity extraction also does not appear shipped.

## Best Possible Solution

Keep the issue open. The best path is to implement heading-aware or section-aware chunking in the active memory-search indexing path, with config/schema/help support, index metadata reindex detection, and focused tests showing separate `##`/`###` sections do not blend in one embedding chunk. Entity extraction/graph memory should be split into a narrower memory-wiki or memory-core design issue if maintainers want separate tracking, because current wiki source import/compile surfaces do not automatically derive entities, concepts, or syntheses from ingested memory.
