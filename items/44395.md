## Summary

Current main still does not implement the central request. Active memory indexing still chunks Markdown by estimated size with only token/overlap settings, so `##`/`###` sections can still be blended in one `memory_search` chunk. The repo has adjacent section-aware REM/dreaming code and memory-wiki source/claim pages, but those do not change the active memory-search chunking path or automatically extract entities/concepts/syntheses from imported sources. MMR reranking is now present as a partial stretch item, but it does not supersede the heading-aware chunking or entity extraction gaps.

## Best Possible Solution

Keep this issue open. The best path is to implement heading-aware or section-aware chunking in the active memory-search indexing path, including config/schema/help support, index metadata/reindex detection, and focused tests showing separate `##`/`###` sections do not blend into one embedding/FTS chunk. Treat #48795 and #48808 as possible implementation candidates to review. Split entity extraction/graph memory into a narrower memory-core or memory-wiki design issue if maintainers want that tracked separately, because current memory-wiki import/compile surfaces preserve and index source pages but do not automatically derive entities, concepts, or syntheses.
