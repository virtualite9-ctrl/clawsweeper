## Summary

Keep open. Current `main` mitigates the reported quota-exhaustion failure without changing the requested default: `commands.nativeSkills` still defaults to `"auto"`, but Telegram now trims/retries command registration and Discord drops per-skill native commands when the 100-command cap would be exceeded. Because the actual feature request to change the default is not implemented, and it remains a plausible product decision, there is not strong evidence to close conservatively.

## Risks / Open Questions

- Discord can still warn that deployment may fail if non-skill native commands alone exceed 100, so the broader command-cap story is not fully eliminated by the current mitigation.
- `gh pr view` could not be used in this environment because `GH_TOKEN` was unavailable, so the review relied on local code/docs/tests/history already present in the checkout.
