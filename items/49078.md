## Summary

Keep #49078 open. Current main still has cron delivery as a single `delivery` object with modes `none`, `announce`, and `webhook`; there is no first-class `output.targets` contract for file writes, multiple destinations, or deterministic session injection. Current docs, CLI, agent tool schema, runtime delivery plan, and Control UI all expose only chat/webhook/silent-style delivery, so the central feature request remains unimplemented rather than obsolete or safely closeable.

## Best Possible Solution

Keep this open as a core cron/runtime feature request. The best path is to design an additive `output` contract for cron jobs, likely in `src/cron/types.ts`, normalization/storage, CLI/Control UI, docs, and isolated-run finalization, with clear semantics for file path safety, append/write modes, channel delivery reuse, named/main session injection, multiple targets, and fallback/error handling. The existing `delivery` field can remain backward-compatible as the channel/webhook subset, but it does not solve deterministic file or session output routing today.
