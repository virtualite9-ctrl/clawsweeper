## Summary

Keep #24943 open. Current main still does not strip or convert general Markdown before iMessage delivery, and the provided issue discussion includes a maintainer-side review explicitly keeping this open until PR #56283 or an equivalent fix lands.

## Best Possible Solution

Keep #24943 open until PR #56283 or an equivalent main-branch implementation lands. The best path is a shared outbound Markdown-stripping decision for non-Markdown-capable/plain-text channels, applied after existing HTML/table/internal-directive handling, with focused regression tests for direct iMessage sends, monitor replies, media captions, config override behavior if added, and empty-after-strip edge cases.
