## Summary

Keep open. Current main still does not strip general Markdown before sending through the legacy iMessage plugin. The send path converts Markdown tables and strips internal directive tags, while the reply delivery path strips assistant-internal scaffolding, but neither applies the existing `stripMarkdown()` plain-text fallback used by BlueBubbles/TTS. The provided GitHub context also shows PR #25349 attempted this exact iMessage fix but was closed unmerged, and a maintainer Codex review on April 26, 2026 explicitly kept the issue open.

## Best Possible Solution

Implement the fallback in the legacy iMessage plugin, likely by applying the shared `stripMarkdown()` or the newer Markdown IR plain-text renderer in `extensions/imessage/src/send.ts` after table conversion and before directive stripping/RPC send, then add focused iMessage send/delivery tests for bold, italic, headers, blockquotes, inline code, and syntax-only edge cases. Keep the issue open until that behavior lands or a maintainer makes an explicit product decision to rely only on BlueBubbles for this capability.
