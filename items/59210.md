## Summary

Keep open. Current main still has the WhatsApp docs gap this PR targets, but the PR should be updated before merge because main now has QR image/data-url flows through the gateway, Control UI, and agent tool paths.

## Best Possible Solution

Keep #59210 open and ask for an update against current main: document that CLI WhatsApp login is still QR-based and terminal-oriented, recommend the current Control UI/gateway QR image flow for remote/headless setups, avoid claiming phone-number/pairing-code login is available, and include any required docs i18n glossary entry for new short labels.
