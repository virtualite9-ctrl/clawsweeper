## Summary

## Close Comment

Closing after Codex review. I could not verify the reported duplicate-registration path on current `main`.

Concrete evidence:
- `extensions/feishu/openclaw.plugin.json:12` still declares `skills`, but `docs/plugins/manifest.md:39-54` and `docs/plugins/manifest.md:161` define that field as manifest metadata read before plugin runtime loads, not a runtime registration entrypoint.
- `src/agents/skills/plugin-skills.ts:46-123` shows plugin `skills` are only resolved into workspace skill directories; that code never calls `registerTool(...)`.
- Feishu tool registration itself still happens in `extensions/feishu/index.ts:73-80` via `registerFull(...)`.
- The reported repro `openclaw status --verbose` no longer matches current behavior, because status now uses snapshot-only plugin inspection (`src/commands/status.scan.ts:65-67`, `src/plugins/status.ts:183-202`) instead of full plugin registration.

Given that the asserted root cause does not hold on current `main`, and the cited repro path is obsolete there, I’m closing this as not reproducible.
