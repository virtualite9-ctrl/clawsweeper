## Summary

Current `main` does not match the reported WhatsApp -> `sanitizeChatSendMessageInput()` object-flow. WhatsApp inbound handling extracts text/captions into a string body before auto-reply processing, and gateway `chat.send` validates `message` as a string before `sanitizeChatSendMessageInput()` runs. The specific `[object Object]` root-cause described in #52464 is therefore obsolete on current `main`.
