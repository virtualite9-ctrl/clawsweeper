## Summary

Keep open. Current main does not implement the PR's central ACP lifecycle fixes: `/acp spawn` still reports success immediately after `initializeSession` without a post-spawn runtime health probe, and the ACP manager only recovers stale `acpx exited` handles, not stale `Gateway disconnected`/`gateway closed` handles. The PR is focused and supported by useful discussion/tests, but it still has an unresolved review point about treating resolved `dead`/`no-session` status payloads as spawn-health failures.

## Best Possible Solution

Keep this PR open and finish it as a narrow ACP lifecycle fix: add a direct post-init `runtime.getStatus` spawn probe that cleans up and returns an ACP error on thrown failures or `dead`/`no-session` status content; add an explicit recoverable gateway-disconnect close-code allowlist for cancel, close, and early-turn retry; preserve non-stale closures such as 1008 as operator-visible failures; and keep the new regression tests in `src/acp/control-plane/manager.test.ts` and `src/auto-reply/reply/commands-acp.test.ts`.
