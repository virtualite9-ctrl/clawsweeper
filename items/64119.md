## Summary

Close as implemented on main. Current main already routes dispatcher-backed requests through `fetchWithRuntimeDispatcher`, and shipped code in `src/infra/net/runtime-fetch.ts` normalizes `FormData` into the undici runtime `FormData` implementation while removing stale `content-type`/`content-length` headers before dispatch. The released test coverage in `src/infra/net/runtime-fetch.test.ts` exercises that path. This makes the PR redundant even though it solved the same bug via a different patch.

## Evidence

- **Dispatcher path still funnels through runtime fetch:** `fetchWithSsrFGuard` uses `fetchWithRuntimeDispatcher(parsedUrl.toString(), init)` whenever a dispatcher is attached and the default fetch cannot honor per-request dispatchers, so the runtime-fetch normalization is on the exact path this PR targets.
  - file: [src/infra/net/fetch-guard.ts:404](https://github.com/openclaw/openclaw/blob/5b8bd6371c666a5de43c82305e0315771e8e49d5/src/infra/net/fetch-guard.ts#L404)
