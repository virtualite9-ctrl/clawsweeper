## Summary

Current main contains multiple Windows CLI startup/performance fixes that directly match this report: doctor plugin paths are lazy-loaded, bundled plugin startup was optimized, gateway/CLI hang behavior was fixed, and Windows `schtasks` handling is now bounded with Startup-folder fallback plus tests/docs covering the slow-path behavior. That is strong enough to treat the v2026.4.9 regression as implemented on main and shipped in a later release.
