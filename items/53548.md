## Summary

Current main still explicitly rejects `mode="session"` without `thread: true` in both native subagent and ACP spawn paths, and the docs/tests still encode the coupling. The related open PR #39372 is an implementation candidate, not shipped behavior.

## Best Possible Solution

Keep this issue open. The best path is to review and harden #39372 or a replacement that decouples persistence from thread binding in `src/agents/subagent-spawn.ts` and `src/agents/acp-spawn.ts`, creates persistent native and ACP child sessions without requiring channel thread binding, returns stable `childSessionKey`/label metadata for `sessions_send`, delivers initial/completion output through the requester or parent conversation when unbound, preserves current `thread=true` behavior, and updates tests plus `docs/tools/subagents.md` and `docs/tools/acp-agents.md`.

## Risks / Open Questions

- A safe implementation likely needs more than removing the two guards: unthreaded persistent ACP/subagent lifecycle, output delivery to the requester or parent, cleanup, and `sessions_send` continuation semantics need regression coverage.
- The existing subagent `mode="run"` plus kept cleanup may cover some native follow-up workflows, but it does not satisfy the explicit `mode="session"` contract and ACP `mode="run"` remains oneshot.
