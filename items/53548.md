## Summary

Current main still explicitly couples `mode="session"` to `thread: true` in both native subagent and ACP spawn paths. Tests and docs still encode that policy, and the related open PR #39372 is only an implementation candidate, not shipped behavior.

## Best Possible Solution

Keep this issue open. The best path is to review and harden #39372 or a replacement that decouples persistent session lifecycle from channel thread binding in `src/agents/subagent-spawn.ts` and `src/agents/acp-spawn.ts`, returns stable `childSessionKey`/label metadata for `sessions_send`, delivers unthreaded initial/completion output through the requester or parent conversation, preserves existing `thread=true` behavior, and updates tests plus `docs/tools/subagents.md` and `docs/tools/acp-agents.md`.
