## Summary

Keep open. Current main still explicitly rejects unthreaded `mode="session"` for both native subagents and ACP spawns, and the ACP fallback `mode="run"` is intentionally mapped to an `oneshot` runtime that is closed after the turn. The discussion describes a real remaining product gap rather than an already-shipped behavior.

## Best Possible Solution

Implement the requested decoupling in the owner code paths: allow unthreaded `mode="session"` for native subagents and ACP, keep thread binding optional, route initial/completion output through the parent/announce path when no thread binding exists, return stable `childSessionKey` and label metadata for `sessions_send`, and add regression coverage plus docs for unsupported-thread channels and Discord parent-orchestrator use.
