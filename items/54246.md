## Summary

Keep open. Current `main` still hardcodes the bare `/new`/`/reset` greeting prompt, including the "ask what they want to do" wording, and there is no config surface or documented `sessionResetPrompt` override in the agent defaults schema.

## Evidence

- **Hardcoded reset prompt remains on main:** `BARE_SESSION_RESET_PROMPT_BASE` is still a fixed string and still includes the action-oriented greeting instruction the issue reports.
  - file: [src/auto-reply/reply/session-reset-prompt.ts:11](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/auto-reply/reply/session-reset-prompt.ts#L11)
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Reset prompt builder has no configurable override:** `buildBareSessionResetPrompt()` only selects among three built-in prompt constants based on bootstrap mode, then appends the current time; it does not read any configurable prompt text from `OpenClawConfig`.
  - file: [src/auto-reply/reply/session-reset-prompt.ts:100](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/auto-reply/reply/session-reset-prompt.ts#L100)
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Agent defaults schema does not expose a session reset prompt field:** The current agent-defaults type includes `startupContext` for bare `/new` and `/reset`, but no `sessionResetPrompt` or equivalent override knob.
  - file: [src/config/types.agent-defaults.ts:248](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/config/types.agent-defaults.ts#L248)

  - file: [docs/gateway/config-agents.md:142](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/docs/gateway/config-agents.md#L142)
