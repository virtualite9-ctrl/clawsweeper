## Summary

Current main still does not implement the PR's central behavior: regular Telegram videos are sent without width/height in both direct-send and reply-delivery paths. The PR addresses a plausible, reproduced Telegram bug, but it should remain open because the fix is not on main and the submitted patch still needs a boundary-compliant revision and regression tests before it is mergeable.

## Best Possible Solution

Keep the PR open and request a focused revision: implement video-dimension probing through a Telegram-local helper or a narrow documented media-runtime/plugin-sdk seam, pass width and height only for regular Telegram sendVideo calls in extensions/telegram/src/send.ts and extensions/telegram/src/bot/delivery.replies.ts, keep GIF animations and video notes excluded, add regression tests for both send paths, and split or drop the unrelated MAX_VIDEO_BYTES change.
