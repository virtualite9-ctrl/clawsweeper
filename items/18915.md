## Summary

Keep open. Current main still does not implement the PR's central Telegram portrait-video fix: both direct Telegram sends and reply delivery call `sendVideo` without probing or forwarding `width`/`height`. The PR is still useful, but the submitted patch needs rework because it imports a new core `src/media` helper from Telegram plugin production code and includes an unrelated global video-size limit change.

## Best Possible Solution

Keep the PR open and ask for a focused revision. The fix should stay inside the Telegram plugin or use an approved `openclaw/plugin-sdk/*` media seam, probe regular video dimensions, forward `width`/`height` in both direct send and reply delivery paths, keep video notes and GIF/animation sends excluded, add regression tests for both send paths, and drop or split the unrelated global media-limit change.
