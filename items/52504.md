## Summary

Keep open. Current `main` has cross-channel DM session sharing via `session.identityLinks`, but it does not implement the requested automatic cross-surface catch-up/context-summary injection. The docs on `main` explicitly say multiple devices/channels can map to the same session while history is still not fully synced back to every client.

## Evidence

- **Docs explicitly say cross-device/channel history is not fully synced:** `docs/concepts/messages.md` states that multiple devices/channels can map to the same session, but history is 'not fully synced back to every client', and recommends one primary device for long conversations.
  - file: [docs/concepts/messages.md:73](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/docs/concepts/messages.md#L73)
