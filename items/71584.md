## Summary

Keep #71584 open. Current main still accepts `max` as a parsed thinking token, but Ollama does not advertise a provider thinking profile that includes `max`, so explicit `/think max` is rejected before the Ollama transport can run. Adjacent Ollama fixes only cover top-level `think: true/false`, not the requested `reasoning_effort: "max"` or a `max`-capable Ollama thinking profile.

## Best Possible Solution

Keep the issue open and fix it in the provider-thinking layer, not by bypassing validation. The likely maintainer path is to add an Ollama-owned `resolveThinkingProfile` for models that support `max` reasoning, or introduce a validated per-model supported-thinking-level contract that `/think` uses before runtime. Then add regression coverage for `ollama/deepseek-v4-flash:cloud` or an equivalent configured Ollama model showing `/think max` is accepted and the native request sends the correct provider option if Ollama Cloud requires `options.reasoning_effort: "max"` rather than just top-level `think: true`.
