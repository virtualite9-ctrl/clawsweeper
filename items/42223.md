## Summary

Keep this PR open. Current main still has the active-tab exception that leaves a collapsed sidebar group expanded, and the latest release still contains the same render logic. The PR is a plausible targeted fix, but it is stale against the current `nav-section` markup and should add a focused regression test before merge.

## Best Possible Solution

Keep this PR open as the canonical fix path. Update it against current main by changing the current `showItems` decision to remove the active-tab exception while preserving icon-only sidebar behavior when `navCollapsed` is true, then add a focused Control UI regression test for collapsing the active group.
