## Summary

Keep this PR open. Current main still contains the active-tab exception that this PR is trying to remove: a collapsed sidebar group is forced visible whenever it contains the active tab. The linked bug #42167 was closed as superseded by this PR, not because main already fixed it. The patch is stale against the current `nav-section` refactor, but the underlying fix remains useful.

## Best Possible Solution

Keep this PR open as the canonical fix path for #42167. Update it against current main by changing the current `showItems` logic so `isGroupCollapsed` hides child items even when the active tab is inside that group, then add a focused Control UI/browser regression test for collapsing the active group.
