## Summary

Keep this PR open. Current main still keeps a collapsed sidebar group visible when it contains the active tab, and the linked bug was closed as superseded by this PR rather than because the fix shipped.

## Best Possible Solution

Keep this PR open as the canonical fix path for the closed sidebar-collapse bug. Update it against current main by removing `hasActiveTab` from the `showItems` visibility decision while preserving the collapsed-icon sidebar behavior, then add a focused Control UI/browser regression test for collapsing the active group.
