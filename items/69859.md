## Summary

Keep open. Current main still does not accept or dispatch a top-level `threadId` from `POST /hooks/agent`, even though the cron delivery pipeline now supports `delivery.threadId`. The PR is not merge-ready as submitted, but the underlying feature remains a real OpenClaw gap and related issue #69856 was closed as superseded by this PR.

## Best Possible Solution

Keep the item open or replace it with a focused patch. The right fix is to rebase onto current main, add `threadId` parsing to `HookAgentPayload` and `normalizeAgentPayload`, include it in hook replay/dedupe scope, populate `CronJob.delivery.threadId` in `dispatchAgentHook` while preserving existing `deliver` semantics, and add targeted gateway/cron tests plus docs for `POST /hooks/agent`. Remove the backup file and split the unrelated model-selection change.
