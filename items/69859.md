## Summary

Current main still does not implement the central request: POST /hooks/agent does not parse or dispatch a request threadId. The downstream cron delivery stack can already carry an explicit delivery threadId, so the PR is pointed at a real gap, but its current form needs cleanup and tests rather than closure.

## Best Possible Solution

Keep the item open, but require a focused update: parse and normalize optional threadId in src/gateway/hooks.ts, include it in the hook replay key, pass it through dispatchAgentHook into job.delivery.threadId or the current canonical delivery field, add gateway and cron delivery tests proving threaded replies flow end to end, update /hooks/agent docs, and drop the unrelated model-selection change plus src/media/web-media.ts.backup.
