## Summary

Keep open. Current `main` still has a real docs gap around exact-command durable exec approvals: the code can persist `allow-always` as `=command:<hash>` entries when it cannot derive a stable executable/script pattern, and the current docs mostly describe path-based allowlists and interpreter/file binding instead. This PR is not obsolete, although its branch has accumulated unrelated changes that should be split out before merge consideration.
