## Summary

Current main still does not satisfy the OpenProse request. The bundled OpenProse grammar continues to constrain agent/session `model:` to `sonnet|opus|haiku`, while OpenClaw's underlying subagent model-selection path already accepts explicit model override strings and provider/model refs. This is a real bundled plugin behavior gap, not a duplicate of the related cron `payload.model` issues and not a ClawHub-only request.

## Best Possible Solution

Keep the issue open. The likely fix belongs in the bundled `open-prose` plugin skill/spec: broaden `model:` syntax in `extensions/open-prose/skills/prose/prose.md` and `compiler.md` to accept OpenClaw aliases and canonical `provider/model` refs, update `docs/prose.md`, and add focused validation/regression coverage or fixture examples showing a `.prose` agent/session forwards the model override into `sessions_spawn` rather than falling back silently.
