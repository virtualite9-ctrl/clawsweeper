## Summary

Keep #39400 open. Current main at 4a3030df9efa72c44bab567a4390ac0c4876a73f still exposes provider quota in `session_status` as an optional human-readable status line, not as stable structured usage fields with explicit availability, source, or reset metadata. The prior ClawSweeper keep-open comment remains accurate; the relevant files did not change since that review.

## Best Possible Solution

Keep the issue open. If cron/monitor use of `session_status` is supported, extend the agent-facing status contract with a structured `usage` object in `details` that includes provider/source, explicit availability or unavailable reason, normalized windows with percent/resetAt, and reset explanation where known. Preserve the current human-readable status line and add focused coverage for no auth, fetch error, empty window, timeout, and Codex 5h/Day/Week mapping behavior. If maintainers prefer not to expand `session_status`, track an equivalent dedicated usage tool and link #39400 to that path.
