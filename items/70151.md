## Summary

Keep PR #70151 open. Current main still appears to have the reported race: the embedded runner builds terminal payloads from the per-attempt assistant snapshot and can still surface the `payloads=0` incomplete-turn error without first polling the persisted session for a late assistant reply. The PR is focused and not superseded by the related ClawSweeper items provided.

## Best Possible Solution

Keep the PR open and have maintainers either land a refined version or implement the same behavior directly in `src/agents/pi-embedded-runner/run.ts`: before returning the terminal empty-turn error, reopen/read the persisted session after the attempt snapshot length, recover a current-attempt assistant message if one arrived late, build payloads from that refreshed assistant, and keep the focused regression test in `run.incomplete-turn.test.ts`.
