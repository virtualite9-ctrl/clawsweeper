## Summary

Keep PR #70151 open. Current main still has the terminal embedded-runner path building payloads from the attempt-local assistant and surfacing `payloads=0` incomplete-turn errors without refreshing the persisted session file. Later shipped empty-payload work covers an adjacent case where `lastAssistant` is already available but streamed text is blank; it does not cover the stale-snapshot / late persisted assistant race described by this PR.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to rebase it on current main, adapt the recovery to the existing final-answer fallback work from #71467, and add/keep a focused regression test that proves a late assistant message persisted to the session file prevents the false `payloads=0` incomplete-turn error.
