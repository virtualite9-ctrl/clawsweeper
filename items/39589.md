## Summary

Keep #39589 open. Current main still rejects the reporter's provider-only `models.providers.openai` entry because `models` remains required in the core provider schema and generated config schema. The OpenAI memory embedding path would use `models.providers.openai.baseUrl` if validation accepted the config, and docs show `agents.defaults.memorySearch.remote.baseUrl` as a separate workaround, but the reported config-contract bug is still actionable. Related PRs #46977 and #46980 proposed the same schema fix and are closed unmerged in the provided GitHub context.

## Best Possible Solution

Keep #39589 open until a focused config-contract fix lands. The likely fix is to make `ModelProviderSchema.models` optional with a default empty array, align `ModelProviderConfig` if needed, regenerate generated config schema artifacts, add a regression test proving `models.providers.openai` with only `baseUrl`/`apiKey`/`api` validates, and add or keep a memory embedding test proving that provider `baseUrl` is used when `memorySearch.remote.baseUrl` is absent.
