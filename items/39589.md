## Summary

Keep #39589 open. Current main still requires `models.providers.openai.models`, so the reporter's provider-only OpenAI configuration is rejected before the memory embedding path can use the custom `baseUrl`. The memory embedding client would honor `models.providers.<provider>.baseUrl` if validation accepted the entry, and docs also show `agents.defaults.memorySearch.remote.baseUrl` as a workaround, but the reported config-contract bug remains actionable. Related PRs #46977 and #46980 proposed the same fix and are closed unmerged in the provided GitHub context.

## Best Possible Solution

Keep #39589 open until a focused config-contract fix lands. The likely fix is to make `ModelProviderSchema.models` optional with a default empty array, regenerate generated config schema artifacts, add a regression test for a provider entry containing only `baseUrl`/`apiKey`/`api`, and add or keep a memory embedding test proving `models.providers.openai.baseUrl` is used when `memorySearch.remote.baseUrl` is absent.
