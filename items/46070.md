## Summary

Keep #46070 open. Current main still clears requested operator scopes for device-less shared-token/password WebSocket connects outside narrow Control UI and direct-local backend exceptions, and existing tests assert the resulting missing-scope behavior. The Paperclip adapter may also need changes, but the core compatibility question is still unresolved rather than implemented or clearly superseded.

## Best Possible Solution

Keep the issue open for maintainer/product judgment. The clean path is either to document that third-party gateway adapters must send device identity/device tokens and close only after an explicit maintainer decision, or to add a narrowly scoped, tested Gateway contract for trusted external adapters that preserves declared operator scopes without weakening the current device-identity boundary.
