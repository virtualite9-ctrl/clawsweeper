## Summary

Keep this issue open. Current main still does not implement the requested dedicated Slack user DM routing flow: Slack setup/docs expose bot-level events, the Socket Mode monitor is initialized around bot/app tokens, normal inbound replies are delivered with the bot token, and user-token writes remain a fallback path when no bot token is available. The prior ClawSweeper comment is still accurate against current main.

## Best Possible Solution

Implement this in the bundled Slack plugin: add explicit user_events setup/docs for Slack events on behalf of users, carry configured user-token identity and event origin through monitor context, distinguish eligible user-level message.im events from bot-level DMs, route those replies through the xoxp token when userTokenReadOnly is false, and add focused tests for inbound DM routing, token selection, setup manifest output, docs, and loop prevention.
