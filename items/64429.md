Keep this PR open. Current main exposes Slack file IDs in placeholder text, but it still does not implement the PR's structured `SlackMediaResult.fileId` and `MediaFileId`/`MediaFileIds` propagation, and the related Slack `download-file` schema work is open rather than a shipped replacement.

## Best Possible Solution

Keep this PR open for maintainer review.
