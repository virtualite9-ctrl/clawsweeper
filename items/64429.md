## Summary

Keep this PR open. Current main at fee16865b229cf3d652041eadb12a1b25dba07c0 still does not propagate Slack file IDs as structured `SlackMediaResult.fileId` or `MediaFileId`/`MediaFileIds` agent context fields. The PR is a plausible implementation candidate, but it needs a rebase, focused tests, and maintainer agreement on the agent-facing context contract. The adjacent #45626 message-tool schema work is related but does not replace this inbound-context change.

## Best Possible Solution

Keep this PR open for maintainer review. Rebase it onto current main, move the `SlackMediaResult` change into `extensions/slack/src/monitor/media-types.ts`, add focused Slack media and prepare tests for structured file ID propagation, and decide whether `MediaFileId`/`MediaFileIds` belong in the generic `MsgContext` contract or in a Slack-owned structured metadata shape. Coordinate with #45626 so agents can both see and pass Slack file IDs reliably.
