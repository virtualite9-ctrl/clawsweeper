## Summary

Keep this PR open. Current main still lacks the structured `SlackMediaResult.fileId` and `MediaFileId`/`MediaFileIds` context propagation proposed here, even though Slack file IDs are now visible in placeholder text and docs. The PR is still potentially useful but needs maintainer judgment on the shared context contract and coordination with the open message-tool schema gap.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to choose the intended agent-facing Slack file ID contract, then either update and land this PR with targeted Slack media/prepare tests and schema coordination, or replace it with a combined fix that covers both structured inbound context and the Slack `download-file` tool schema gap.
