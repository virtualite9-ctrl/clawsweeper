## Summary

Keep #69337 open. Current main still does not implement the PR's TUI/commentary fallback, and the newer live-chat code goes the opposite direction by suppressing commentary-phase assistant events. The PR should not merge as-is because it changes a shared helper and contradicts existing preview/WebChat/history contracts, but it is not safely closeable without a narrower TUI/provider fix or explicit maintainer rejection.

## Best Possible Solution

Keep the PR open for maintainer judgment, but do not merge it as-is. If the behavior is still needed, rework it at a TUI-specific rendering boundary or at the provider/finalization boundary, with regression tests that preserve shared suppression of commentary-only assistant text in previews, WebChat/history, and channel-visible surfaces.
