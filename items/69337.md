## Summary

Keep #69337 open. Current main has not implemented this PR's commentary fallback, but the proposed global change to `extractAssistantVisibleText` conflicts with current tests and non-TUI visibility contracts that deliberately suppress commentary-only assistant text. The PR may point at a real TUI/model-finalization gap, but it is not safely closeable as implemented, implemented on main, or superseded by a canonical landed fix.

## Best Possible Solution

Keep the PR open for maintainer judgment, but do not merge it as-is. If the behavior is still needed, implement it at a TUI-specific rendering boundary or provider/finalization boundary with regression tests that preserve shared history, preview, WebChat, and channel suppression of commentary-only assistant text.
