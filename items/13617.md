## Summary

Keep #13617 open. Current main still lacks the requested side-effect-free cron and skills testing framework: no `openclaw cron test`, `openclaw cron validate`, `openclaw skills test`, cron run dry-run protocol, CI-oriented validation output, or global tool dry-run interception. The latest issue discussion also explicitly keeps this open and treats closed #13605 as only partial, with open #41418 tracking related global tool dry-run work.

## Best Possible Solution

Keep #13617 open until OpenClaw ships a real cron/skills test surface or maintainers deliberately split it. A complete direction would add `openclaw cron test`, `openclaw cron validate`, structured `--json` CI output, `openclaw skills test`, cron protocol support for validation/dry-run, and integration with the global tool interception design tracked by #41418 rather than inventing an incompatible second dry-run mechanism.
