## Summary

Keep #13617 open. Current `main` still does not implement the requested side-effect-free cron and skills testing framework: cron has real `run` execution only, no `test` or `validate` command/protocol shape, skills has no test command or gateway method, and the global dry-run layer remains tracked separately by open #41418.

## Best Possible Solution

Keep #13617 open until maintainers either ship a real side-effect-free testing surface or intentionally split and supersede it with narrower canonical issues. The implementation should likely add cron `test`/`validate` CLI and protocol support, structured CI-friendly output, a skills test runner, and integrate with the global dry-run/tool-interception design tracked by #41418 rather than inventing a separate mock path.
