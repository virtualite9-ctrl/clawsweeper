## Summary

Keep #13617 open. Current main still lacks the requested side-effect-free testing framework for cron jobs and skills: no cron test/validate CLI, no skills test CLI, no cron dry-run protocol branch, and no global tool interception layer. The latest provided issue discussion also explicitly kept this open, with #13605 only partial and #41418 tracking related global dry-run work.

## Best Possible Solution

Keep #13617 open until OpenClaw ships or explicitly scopes a real cron/skills testing surface. A complete path would add `openclaw cron test`, `openclaw cron validate`, structured `--json` CI output, `openclaw skills test`, cron protocol support for validate/dry-run intent, and integration with the global tool dry-run design tracked by #41418 instead of a separate interception mechanism.
