## Summary

Current `main` has a generic `channels.modelByChannel` feature for channel/group/thread model overrides, but it does not fully implement the issue as requested. Slack and WhatsApp config schemas still do not expose `model` fields at `channels.slack.channels.*`, `channels.slack.dms.*`, `channels.whatsapp.groups.*`, or `channels.whatsapp.direct/dms.*`, and the override resolver is built around channel/group identifiers rather than documented per-DM config entries.

## Risks / Open Questions

- A direct-message override path may exist indirectly if some channel plugin populates `sessionEntry.groupId` for direct chats, but I did not find schema/docs evidence that would justify closing this as implemented.
- The existing top-level `channels.modelByChannel` feature may satisfy part of the request for groups/channels, so future triage should distinguish between extending that feature and adding the exact nested config fields proposed in the issue.
