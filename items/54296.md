## Summary

Keep open. Current main has fixed the exec-approvals atomic write half, but the transcript memory-pressure half remains in multiple core session/gateway paths that still materialize entire transcript files before scanning or trimming.

## Best Possible Solution

Keep this issue open. Treat exec-approvals persistence as already fixed, then land a focused core follow-up that introduces shared streaming or bounded transcript scan/tail helpers for src/gateway/server-methods/chat.ts, src/config/sessions/transcript.ts, src/gateway/server-methods/sessions.ts, and src/gateway/session-utils.fs.ts, with regression coverage for large transcripts and malformed-line cases.
