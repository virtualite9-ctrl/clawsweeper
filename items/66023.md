## Summary

Keep this PR open. Current main still lacks custom cron job IDs end-to-end, and this PR is a concrete implementation candidate for #65636 rather than obsolete cleanup. It should be deduplicated against #65669 and needs review cleanup before landing, especially case-insensitive custom-id collision handling and docs for the new CLI flag.

## Best Possible Solution

Keep this PR open as an implementation candidate for #65636. Before landing, deduplicate it against #65669, enforce lowercase custom ids or case-insensitive duplicate rejection, keep the CLI/gateway/service coverage, and update `docs/cli/cron.md` / https://docs.openclaw.ai/cli/cron for the new `--id` workflow.

## Risks / Open Questions

- Merging this PR as-is may allow case-variant custom ids that collide in cron run-log filenames on case-insensitive filesystems.
- There are two open implementation candidates for the same feature, including #65669, so maintainers should choose one canonical patch path before landing.
- Because this adds a user-visible `cron add --id` flag, the landing patch should update the cron CLI docs as well as source and tests.
