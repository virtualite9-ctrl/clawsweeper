## Summary

Current main has a partial outbound transcript mirror path, but it does not satisfy the issue’s central non-default routed-agent case. `openclaw message send` still seeds outbound routing with the default agent, then WhatsApp builds the outbound session key from that supplied agent id. The inbound binding resolver that would map `whatsapp` + `accountId=realtor-whatsapp` to `realtor` is not used in this CLI outbound-first path, so the opener can still be mirrored to the wrong agent/session instead of the routed agent’s transcript.

## Best Possible Solution

Keep the issue open and implement routed outbound session selection in the shared outbound send path. For sends without an explicit `agentId` or `sessionKey`, resolve the same channel/account/peer route used by inbound messages, then create the outbound session entry and mirror the opener into that routed agent’s transcript. Add focused tests for a WhatsApp non-default account binding, `session.dmScope` behavior such as `per-account-channel-peer`, and explicit override behavior so current agent-authored sends do not regress.
