## Summary

Keep open. Current `main` still preserves a mutating `lastToolError` after a successful retry with a different action fingerprint, and the payload formatter still emits a bare `failed` warning with no recovery context. The PR’s requested behavior is not implemented on `main`.

## Evidence

- **Recovered retry is not tracked:** `handleToolExecutionEnd` only clears `lastToolError` when `isSameToolMutationAction(...)` matches. If a mutating retry succeeds with different parameters, the error is left in place with no recovered marker.
  - file: [src/agents/pi-embedded-subscribe.handlers.tools.ts:818](https://github.com/openclaw/openclaw/blob/cb2c36b04975e34ab7014a11575d6589aa96ae32/src/agents/pi-embedded-subscribe.handlers.tools.ts#L818)
  - command: `nl -ba src/agents/pi-embedded-subscribe.handlers.tools.ts | sed -n '790,840p'`
  - sha: [cb2c36b04975](https://github.com/openclaw/openclaw/commit/cb2c36b04975e34ab7014a11575d6589aa96ae32)
- **Warnings still render as bare failures:** Mutating tool errors always return `{ showWarning: true, includeDetails }`, and the warning text is still formatted as `⚠️ ... failed` with no recovery suffix or alternate text.
  - file: [src/agents/pi-embedded-runner/run/payloads.ts:107](https://github.com/openclaw/openclaw/blob/cb2c36b04975e34ab7014a11575d6589aa96ae32/src/agents/pi-embedded-runner/run/payloads.ts#L107)
  - command: `nl -ba src/agents/pi-embedded-runner/run/payloads.ts | sed -n '96,120p;348,368p'`
  - sha: [cb2c36b04975](https://github.com/openclaw/openclaw/commit/cb2c36b04975e34ab7014a11575d6589aa96ae32)
- **No type support for recovery context:** `ToolErrorSummary` on `main` has `toolName`, `meta`, `error`, `timedOut`, `mutatingAction`, and `actionFingerprint`, but no `recovered` field to carry the context the PR adds.
  - file: [src/agents/tool-error-summary.ts:3](https://github.com/openclaw/openclaw/blob/cb2c36b04975e34ab7014a11575d6589aa96ae32/src/agents/tool-error-summary.ts#L3)
  - command: `nl -ba src/agents/tool-error-summary.ts | sed -n '1,80p'`
  - sha: [cb2c36b04975](https://github.com/openclaw/openclaw/commit/cb2c36b04975e34ab7014a11575d6589aa96ae32)
- **Current tests only prove exact-match clearing:** The existing handler test covers a failed `edit` followed by a successful matching `edit` that clears `lastToolError`, but there is no test on `main` for the different-fingerprint recovered case described by the PR.
  - file: [src/agents/pi-embedded-subscribe.handlers.tools.test.ts:223](https://github.com/openclaw/openclaw/blob/cb2c36b04975e34ab7014a11575d6589aa96ae32/src/agents/pi-embedded-subscribe.handlers.tools.test.ts#L223)
