## Summary

Keep open. Current main still lacks configurable bootstrap dynamic-file classification and configurable bootstrap file ordering. The prompt order and cache-boundary split remain hardcoded, the agent-defaults config schema/docs do not expose the requested settings, and #72520 is still open and unmerged with closing syntax for this issue, so this issue should remain open until that PR is reviewed/closed or a real implementation lands.

## Best Possible Solution

Keep this issue open. Maintainers should review or close #72520, then implement a real prompt/config change: add documented `agents.defaults` and likely per-agent settings for dynamic bootstrap files and optional ordering, thread them into bootstrap prompt assembly before sorting/splitting, preserve deterministic fallback order, update schema/help/generated docs, and add tests proving configured volatile files render below `OPENCLAW_CACHE_BOUNDARY` without destabilizing the stable prefix.
