## Summary

Current main still lacks the proposed Gateway directory RPC surface. The PR is a plausible in-repo gateway feature over the existing channel directory adapter contract, not a duplicate of the related WhatsApp behavior report and not something already implemented on main.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to complete it as a proper Gateway protocol addition: add typed params/result schemas and validators, focused tests for directory.self, directory.peers.list, directory.groups.list, method discovery, and read-scope authorization, update Gateway protocol docs/generated artifacts if this is a public RPC surface, then land this PR or an equivalent implementation. The related WhatsApp report #61744 is closed and concerns provider discovery/reconnect behavior, so it does not supersede this Gateway wrapper work.
