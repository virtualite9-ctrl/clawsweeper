## Summary

Keep open. Current main already has a bundled Nextcloud Talk plugin, but it does not implement a Nextcloud Talk typing indicator, expose a typing config option, or wire typing callbacks into the inbound reply lifecycle. This is an actionable plugin-owned enhancement in this repository, not a cleanup close.

## Best Possible Solution

Keep the issue open and implement it in `extensions/nextcloud-talk`: add a typed send/clear typing helper after verifying Nextcloud Talk auth semantics, wire it into the inbound reply lifecycle with `createChannelReplyPipeline`/typing callbacks, stop it on idle/error/cleanup, add focused tests for start/stop behavior, and document any new `channels.nextcloud-talk` config knob if one is introduced.
