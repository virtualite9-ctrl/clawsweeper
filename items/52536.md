## Summary

Current main already fixes the reported Slack first-reply misrouting by computing a dedicated streaming thread target from the reply planner and using that target before native streaming starts; regression tests now cover the first-reply thread case.

## Evidence

- **Streaming uses reply-planned thread hint:** `dispatchPreparedSlackMessage` now computes `streamThreadHint` with `resolveSlackStreamingThreadHint(...)` and feeds it into the native-streaming gate, so first replies can stream against the planned thread target instead of falling back to channel-level delivery.
