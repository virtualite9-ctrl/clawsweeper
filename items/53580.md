## Summary

Keep open. Current `main` does not contain the proposed `acp-provider` work, existing ACP support is centered on the bundled `acpx` runtime/session path rather than an ACP-backed model provider, and the repo guidance makes external plugin distribution a viable direction but not a strong enough basis to close this PR as out of scope. The PR also still has unresolved review findings, so closing it would be premature.
