## Summary

Keep #33329 open. Current main implements provider-level discovery toggles for Bedrock, GitHub Copilot, Hugging Face, and Ollama, and the plugin manifests expose schema/UI hints for the non-Bedrock toggles. However, the public docs still document only the Bedrock discovery config path, and the later issue discussion asks for a global disable-all model discovery control that current config/types/runtime do not provide.

## Best Possible Solution

Keep the issue open until the public docs document `plugins.entries.github-copilot.config.discovery.enabled`, `plugins.entries.huggingface.config.discovery.enabled`, and `plugins.entries.ollama.config.discovery.enabled` alongside Bedrock. Then either add a typed global model-discovery disable control before `resolveImplicitProviders` runs, or split that global switch into a focused follow-up that defines the config path and its interaction with `models.mode`.
