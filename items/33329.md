## Summary

Keep #33329 open. Current main has provider-level discovery toggles for GitHub Copilot, Hugging Face, Ollama, and Bedrock, but the user-facing docs still only document the Bedrock discovery config path, and the later issue discussion asks for a global disable-all model discovery control that current config/types/runtime do not provide.

## Best Possible Solution

Keep the issue open until public docs document the non-Bedrock provider toggles: plugins.entries.github-copilot.config.discovery.enabled, plugins.entries.huggingface.config.discovery.enabled, and plugins.entries.ollama.config.discovery.enabled. Then either add a typed global model-discovery disable control that is checked before resolveImplicitProviders runs, or split that global switch into a focused follow-up defining its config path and interaction with models.mode.
