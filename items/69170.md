## Summary

Keep open. Current main still exposes `sessions_spawn.streamTo` in the ACP-capable schema without the requested `runtime="acp"` field description, and the colocated tests still only assert the field exists plus that `runtime="subagent"` with `streamTo` is rejected. The PR directly adds the missing schema hint and regression assertion; the related duplicate #69203 is closed unmerged, and #69166 was closed in favor of #69170 rather than because main shipped the fix.

## Best Possible Solution

Keep #69170 open or explicitly consolidate it with #63121 via maintainer review. The implementation path should add the ACP-only `streamTo` description inside the current `acpAvailable` schema branch, add a regression assertion for that description with an ACP backend registered, and preserve or coordinate with any broader runtime behavior decision about silently ignoring ACP-only fields for native subagent spawns.
