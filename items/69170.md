## Summary

Keep open. Current main still exposes `sessions_spawn.streamTo` in the ACP-capable schema without the requested `runtime="acp"` field description, and current tests only assert that the field exists plus that `runtime="subagent"` with `streamTo` is rejected. The linked duplicate PR #69203 is closed unmerged, and #69166 was closed in favor of #69170, so this PR still tracks useful unshipped work.

## Best Possible Solution

Keep #69170 open or fold its exact schema-description assertion into the chosen canonical fix. A current-main version should add the ACP-only `streamTo` description inside the `acpAvailable` schema branch, update the regression test to cover the ACP-backend-present schema, and coordinate with #68397 only if maintainers choose the broader silent-drop behavior change.
