## Summary

Current `main` already makes the live tool-result hard cap configurable via `agents.defaults.contextLimits.toolResultMaxChars` / `agents.list[].contextLimits.toolResultMaxChars`, and the same knob is applied to write-time persistence plus overflow-recovery truncation. The latest tagged release present in this checkout (`v2026.4.23`) also contains the implementation, docs, and tests.

## Evidence
