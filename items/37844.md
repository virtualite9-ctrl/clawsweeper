## Summary

Current `main` already contains a Discord inbound replay guard keyed by `accountId:channelId:messageId`, wired before debounce/preflight, plus regression coverage that asserts a duplicated inbound Discord message is processed only once. Current `main` also includes the auto-reply media dedupe fix for the later #68056-style duplicate-send path, so the known duplicate-reply mechanisms discussed on this issue are implemented on `main`.
