## Summary

Current main still does not satisfy the reported WebChat-to-WeChat routing behavior. Browser WebChat sends `chat.send` with `deliver: false`, and the Gateway explicitly prevents WebChat clients from inheriting an existing external `deliveryContext`; tests assert that WebChat clients on channel-scoped sessions stay on the internal WebChat route.

## Best Possible Solution

Keep open for maintainer product judgment. A safe implementation would add an explicit, permission-gated WebChat/Control UI path for channel-connected sessions that can set an external originating route from `deliveryContext` when the user intentionally sends to that channel session, while preserving the current no-inherit safety for ordinary WebChat/main sessions. Add regression tests for a WebChat client, a plugin channel such as `openclaw-weixin`, `deliveryContext.channel/to/accountId/threadId`, and the existing no-inherit cases.
