## Summary

Keep #69326 open. Current main still treats browser Control UI/WebChat turns as internal WebChat replies: the Control UI connects in webchat mode, sends chat.send with deliver:false, and the gateway explicitly prevents WebChat clients from inheriting external delivery routes even when a session has channel deliveryContext. This is a core/UI routing policy question, not already implemented or safely closeable as an external WeChat-plugin issue.

## Best Possible Solution

Keep the issue open for maintainer-owned product/design work. A good path is to define an explicit cross-surface reply mode for Control UI/WebChat when the selected session has a routable external deliveryContext, then update ui/src/ui/controllers/chat.ts and src/gateway/server-methods/chat.ts with tests covering plugin channels such as openclaw-weixin while preserving the existing WebChat route-safety behavior for plain session browsing.
