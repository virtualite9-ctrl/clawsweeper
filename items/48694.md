## Summary

Current main already addresses the OpenClaw-side failure reported here: Slack Socket Mode `account_inactive` is classified as a non-recoverable auth error, the startup/reconnect paths preserve and inspect the Socket Mode error payload, and gateway supervision catches channel task failures instead of producing an unhandled promise rejection loop. This was shipped in v2026.3.2 via PR #32377.

## Best Possible Solution

Close as implemented and keep the shipped Slack non-recoverable auth handling. If a current release still receives `account_inactive` for tokens that pass direct Slack checks, that should be a fresh current-version Slack diagnostic issue with new logs showing the post-v2026.3.2 behavior.
