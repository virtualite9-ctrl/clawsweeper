## Summary

Keep open. Current main still does not implement the requested selected/default plus separate active routed model display for non-fallback runtime routing. Adjacent fixes shipped for runtime-only model selection and fallback reporting, but this issue asks for both identities to be visible at the same time.

## Best Possible Solution

Keep this open. Implement the split in the shared status rendering path: preserve the configured/session-selected model on `🧠 Model:`, and when the resolved active runtime model differs and `resolveActiveFallbackState` is not active, render `↪️ Active: provider/model` with auth details if they differ. Adjust `session_status` so runtime-only routed sessions do not collapse the runtime model into the selected primary when the configured default should remain visible. Add focused regression coverage in `src/auto-reply/status.test.ts` and `src/agents/openclaw-tools.session-status.test.ts`, including a non-fallback router case and preserving the existing fallback behavior.
