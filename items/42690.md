## Summary

Do not close this issue. Current main has adjacent fixes for persisted runtime model reporting and fallback status, but the exact requested behavior remains missing for a non-fallback routed active model when the selected/default model is intentionally preserved. The `session_status` card still renders the selected model as the main `Model:` line and only adds a separate runtime line for recognized fallback state, not for generic active runtime drift.

## Best Possible Solution

Keep the issue open. The best fix is in the shared status path: when `resolveSelectedAndActiveModel(...).activeDiffers` is true and `resolveActiveFallbackState` is not active, render a separate active runtime line for `/status` and `session_status` while preserving the selected/default `Model:` line. Add a regression test for an explicit selected/default model plus `sessionEntry.modelProvider/model` from a non-fallback router, and consider transcript model-snapshot or final assistant provider/model fallback only if the live entry is still sparse mid-turn.
