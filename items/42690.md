## Summary

Keep open. Current main still does not render the requested selected/default model plus separate non-fallback active routed model in `session_status`; it only renders a separate runtime model line for fallback state, and runtime-only sessions without explicit overrides are collapsed into the primary `🧠 Model:` line.

## Best Possible Solution

Keep this open. Implement the split in the shared status rendering path: keep the configured/session-selected model on `🧠 Model:`, add `↪️ Active: provider/model` when the resolved active runtime model differs and fallback state is not active, and add focused regression coverage in `src/auto-reply/status.test.ts` plus `src/agents/openclaw-tools.session-status.test.ts` for a non-fallback routed model case while preserving existing fallback behavior.
