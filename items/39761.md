## Summary

Keep PR #39761 open. Current main still does not expose agent reasoning metadata on the `message_sending` plugin hook, and the requested bridge from embedded-runner metadata into outbound delivery remains a real core plugin API gap rather than ClawHub-only work or a duplicate cleanup item.

## Best Possible Solution

Keep the PR open for maintainer review or adaptation. The best path is a backward-compatible `agentContext` or equivalent field on `PluginHookMessageSendingEvent`, threaded through `DeliverOutboundPayloadsCoreParams` and `applyMessageSendingHook()`, derived from current embedded-runner metadata in agent delivery paths, documented in plugin hooks docs, and covered by focused hook contract tests. Coordinate provider/model identity and retry semantics with #69110 instead of closing this PR as obsolete.
