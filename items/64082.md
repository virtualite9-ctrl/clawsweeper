## Summary

Current `main` already handles the exact `[media attached: ...]` prompt format described in the report. OpenClaw still emits `[media attached ...]` lines for inbound media, but the image loader now explicitly parses that format as well as `[Image: source: ...]`, with regression tests covering both paths and the prompt plumbing.

## Evidence
