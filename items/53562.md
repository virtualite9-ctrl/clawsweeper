## Summary

Keep open. Current `main` has Discord voice support, but the requested companion text-channel routing is not implemented: `voice.autoJoin` still only carries `guildId`/`channelId`, voice-session routing is still derived from the voice channel id, and spoken turns still run with `deliver: false`, so transcripts/replies are not posted into a paired text channel.

## Evidence

- **Auto-join config has no companion text channel field:** `autoJoin()` reads configured entries and forwards only `guildId` and `channelId` into `join()`. There is no `sessionChannelId` or equivalent config path in the runtime join flow.
  - file: [extensions/discord/src/voice/manager.ts:364](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/extensions/discord/src/voice/manager.ts#L364)
  - command: `nl -ba extensions/discord/src/voice/manager.ts | sed -n '356,388p'`
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Session route is still pinned to the voice channel id:** `join()` sets `sessionChannelId = channelInfo?.id ?? channelId` immediately after fetching the same voice channel, then resolves the agent route with that id. There is no branch that swaps in a separate text channel.
  - file: [extensions/discord/src/voice/manager.ts:474](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/extensions/discord/src/voice/manager.ts#L474)
  - command: `nl -ba extensions/discord/src/voice/manager.ts | sed -n '410,500p'`
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Voice ingress still suppresses channel delivery:** `processSegment()` calls `agentCommandFromIngress()` with `deliver: false`, so the transcribed turn and reply stay session-only and are not delivered into Discord text chat.
  - file: [extensions/discord/src/voice/manager.ts:781](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/extensions/discord/src/voice/manager.ts#L781)
  - command: `nl -ba extensions/discord/src/voice/manager.ts | sed -n '724,820p'`
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Docs and UI hints still describe auto-join as guild/channel only:** The Discord docs example and config UI hints both document `voice.autoJoin` as `guildId`/`channelId` entries only, with no `sessionChannelId` field or transcript-routing option.
  - file: [docs/channels/discord.md:969](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/docs/channels/discord.md#L969)
