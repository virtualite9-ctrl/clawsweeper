## Summary

Keep #38709 open. Current main still contains only the simpler view-local WebChat input history implementation, while this PR tracks additional state-layer hardening around loading windows, reload invalidation, bidirectional navigation mode, draft restoration, caret/textarea restoration, and programmatic draft updates. A maintainer also explicitly said this is not superseded by #71524 and asked for a rebase plus green lanes before standalone review.

## Best Possible Solution

Keep the PR open for maintainer review after the contributor rebases onto current main and restores green UI/check lanes. The likely product path is either to land a narrowed state-layer WebChat input-history hardening patch with focused tests, or have a maintainer explicitly decide that the remaining hardening is no longer desired and close it then.
