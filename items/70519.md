## Summary

Current main already fixes the Control UI/WebChat leak by keeping heartbeat/cron/exec wakeups out of visible transcript prompts and treating runtime-only wake context as transient system context. The PR's transcript-truncation approach is now obsolete and conflicts with the current append-only transcript policy. The fix is present on main at 58a31b12f7dfb41d5cfecd2b51a6adfcbfb2d03e, but I could not identify a release tag containing it, so fixedRelease is null.

## Best Possible Solution

Close this PR as implemented on main. Keep the shipped runtime/transcript separation and append-only transcript policy from 58a31b12 rather than merging the PR's transcript-truncation cleanup. Any remaining behavioral follow-up should build on the current runtime-only prompt path, not file truncation.
