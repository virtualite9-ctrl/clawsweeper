## Summary

Keep open. Current main still returns an empty GitHub Copilot provider catalog and falls back to conservative synthetic metadata for unknown Copilot models, so the PR’s central work is not implemented or superseded. The related closed issue #71936 points back to this PR rather than replacing it.

## Best Possible Solution

Keep #71924 open or replace it with an equivalent mainline change that fetches GitHub Copilot `/models` during provider catalog discovery, maps capability limits into model metadata, includes `gpt-5.5` in the correct thinking profile support, and adds deterministic tests for the catalog fetch and mapping behavior.
