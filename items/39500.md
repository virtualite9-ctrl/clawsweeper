## Summary

Current `main` no longer stays down on this class of invalid config edit. `commands.allowFrom` is still schema-validated as an object/record, but gateway startup and hot reload now auto-restore the last-known-good config, preserve the rejected payload as `.clobbered.*`, and emit a recovery warning for the main agent session.

## Risks / Open Questions

- `openclaw doctor --fix` still documents unknown-key stripping rather than coercing invalid typed values, so the specific malformed key is not normalized in-place; the issue is closed because the reported gateway-startup wedge is now handled on `main` by last-known-good recovery.
- This recovery depends on an existing last-known-good snapshot from a prior valid startup. If a user can still reproduce a hard startup block on current `main` without recovery, that would merit a fresh issue with exact logs and the preserved `.clobbered.*` file context.
