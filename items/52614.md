## Summary

Current `main` does not show this PR's fix on the OpenClaw side. `subscribeEmbeddedPiSession()` still subscribes immediately with no initial-replay guard, the embedded subscribe state has no `initialReplayInProgress`/preexisting-message tracking, and assistant `message_start`/`message_end` handlers still process any non-transcript-only assistant message. I did not find the proposed regression test on `main`, so there is not strong evidence to close this as already implemented or obsolete.
