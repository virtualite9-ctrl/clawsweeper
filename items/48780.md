## Summary

Keep open. Current main does not contain the linked PR’s arg_value suffix stripping, and the existing OpenAI-compatible tool-call repair is narrower than this report: it repairs balanced JSON with tiny trailing junk, not a `</arg_value>>` suffix embedded in exec/read string parameters. The linked PR #48792 is relevant and should be reviewed or refreshed, but it is still open/unmerged and not present on main.

## Best Possible Solution

Keep #48780 open. Review or refresh PR #48792, or implement the same idea with a focused sanitizer at the tool-argument boundary for exec/read/write/edit plus regression tests for Qwen/DashScope `</arg_value>>` values and Windows exec/read behavior.
