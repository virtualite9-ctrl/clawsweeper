## Summary

Keep open. Current `main` still has no dreaming diary language control: the narrative system prompt is hardcoded in English, the prompt builder has no language input, the `memory-core` manifest schema does not allow a `dreaming.language` key, and user-facing docs only document `enabled`/`frequency` (with `timezone` examples). The issue is also corroborated by a second user comment, so this is not strong close evidence.

## Evidence
