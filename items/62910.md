## Summary

Keep this PR open. Current main still has the exact `scripts/ui.js` pattern the PR targets: Windows `.cmd`/`.bat`/`.com` runners enable `shell: true`, while `run()` and `runSync()` still pass a separate args array to `spawn()`/`spawnSync()`. The linked gateway issue #62881 was closed based on a different Codex app-server startup path, not because this UI script surface was fixed. The PR is plausible, and the Greptile comment supports the intended direction, but it should be revised and tested rather than auto-closed.

## Best Possible Solution

Keep the PR open and ask for a current-main revision: route `scripts/ui.js` through the existing pnpm runner/windows command helper or add an equivalent tested resolver that materializes Windows `.cmd` calls without `shell: true` plus a separate args array. Add regression coverage for the final spawn spec, including `.cmd` runners, safe forwarded args, and paths or args containing spaces.
