## Summary

Keep #32438 open. Current main still writes only the configured JSONL gateway log file and formats log lines only in CLI/UI presentation layers. The older duplicate candidate #29378 is closed, and PR #32442 was closed unmerged, so no canonical open or shipped implementation currently covers the requested persistent sibling formatted log file.

## Best Possible Solution

Keep this issue open until maintainers either implement the core logger behavior or explicitly reject the product direction. A complete fix should derive a sibling path from `logging.file`, write one redacted human-readable line whenever the JSONL file is written, preserve existing JSONL consumers, define formatted-file rotation/retention behavior, add focused logger/log-tail tests, and update the public logging docs.
