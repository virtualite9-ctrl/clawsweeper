## Summary

Keep #32438 open. Current main still writes only the configured JSONL gateway log file; formatted output exists only in CLI/UI presentation paths and is not persisted as the requested sibling `-formatted` file. The prior duplicate candidate #29378 is closed and PR #32442 was closed unmerged, so there is no shipped or canonical open item that supersedes this request.

## Best Possible Solution

Keep this issue open until maintainers either implement the core logger behavior or explicitly reject the product direction. A complete implementation should derive the sibling path from `logging.file`, write a redacted human-readable line whenever the JSONL file is written, preserve existing JSONL consumers, define rotation/retention behavior for the sibling file, add focused logger/log-tail tests, and update the public logging docs.
