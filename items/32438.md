## Summary

Keep #32438 open. Current main still writes only the configured JSONL gateway log file; human-readable formatting exists for `openclaw logs` stdout, but there is no durable sibling `-formatted` log file. The earlier duplicate candidate #29378 is closed, and the attempted implementation PR #32442 was closed unmerged, so this remains the active tracker for the requested core logging behavior.

## Best Possible Solution

Keep this issue open until maintainers either implement or explicitly reject the durable formatted-file behavior. A complete fix should derive a sibling path from `logging.file`, write redacted human-readable lines whenever the JSONL file is written, preserve existing JSONL consumers, define rotation/retention behavior for the sibling file, add focused logger/log-tail tests, and update the public logging docs at https://docs.openclaw.ai/logging.
