## Summary

Keep this PR open. Current main still has the gateway stop-timeout bug the PR is trying to fix, but the submitted patch should not be merged as-is because the review comments identify a real stale-task race after the timeout cleanup.

## Best Possible Solution

Keep the PR open and revise src/gateway/server-channels.ts so stop timeout state becomes honest without allowing stale task completion to mutate runtime or restart state for a replacement lifecycle. Add a regression test where the timed-out task settles after a restart, then review and land the fixed PR or an equivalent maintainer patch to close #70024.
