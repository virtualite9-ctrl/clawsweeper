## Summary

Current main already supports stable, user-controlled session names through persisted session labels. Control UI exposes label editing, gateway persists labels through sessions.patch, WebChat display prefers labels over generated display names, and tests cover label precedence over noisy generated WebChat-style names.

## Best Possible Solution

Keep the current session-label implementation on main: persisted labels through sessions.patch, editable in the Control UI Sessions table, and preferred by WebChat display code. A separate follow-up could improve inline rename ergonomics or docs, but the core stable-name problem is already solved on main.
