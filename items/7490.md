## Summary

Keep #7490 open. Current main at c7a0d9b1889ca09ab5bfd84cda39a8d50d3d5768 still does not implement the requested optional per-agent `description` field: the TypeScript config type, strict runtime schema, generated schema, `agents_list` tool output, gateway agent summary surface, tests, and docs all expose agent identity/name/model/runtime metadata without descriptions. The provided GitHub context also shows the direct implementation PR #7530 was closed unmerged, so it does not supersede the issue.

## Best Possible Solution

Keep #7490 open until a maintainer either accepts and implements the additive core behavior or explicitly rejects the product surface. A complete fix should add `description?: string` to `AgentConfig`, accept and preserve a trimmed optional `description` in `AgentEntrySchema`, regenerate config schema/help metadata, document `agents.list[].description`, include it in `agents_list` output for configured agents, consider the gateway `agents.list` summary if UI discovery should match, and add focused tests for inclusion, omission, and strict rejection of unrelated unknown keys.
