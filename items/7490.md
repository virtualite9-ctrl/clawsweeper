## Summary

Keep #7490 open. Current main still does not implement the requested optional per-agent `description` field or include capability metadata in `agents_list`; the related implementation PR #7530 was closed unmerged, and the existing maintainer/Codex discussion already supports keeping this actionable core config/tool feature open.

## Best Possible Solution

Keep #7490 open until a maintainer accepts, implements, or explicitly rejects the additive core discovery surface. A complete fix should add `description?: string` to the agent config type and strict runtime schema, regenerate schema/help metadata, preserve the value through resolver/tool surfaces, include it in `agents_list` for configured agents, update public docs such as https://docs.openclaw.ai/tools/subagents, and add focused tests for inclusion, omission, and strict rejection of unrelated unknown keys.
