## Summary

Keep #7490 open. Current main at 406ae72fd2789c78053f6b94006ec8f488bc1c00 still does not accept an optional per-agent `description` config field or expose equivalent specialization metadata from `agents_list`. The related implementation PR #7530 was closed unmerged, and the existing issue discussion already recorded the same actionable gap.

## Best Possible Solution

Keep #7490 open until maintainers implement or explicitly reject a core discovery field. A complete fix should add optional `description` support to the agent config type, strict runtime schema, generated schema/help metadata, resolver path, and `agents_list` response, then update sub-agent/config docs and focused tests for inclusion, omission, and strict rejection of unrelated unknown keys.
