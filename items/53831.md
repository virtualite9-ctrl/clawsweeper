## Summary

Current `main` already contains shipped Windows bundled-plugin loader fixes that cover Discord startup/module-loading failures, and the Discord channel now emits explicit startup/status signals instead of disappearing silently.

## Evidence

- **Shipped Windows bundled-plugin loader fix:** `CHANGELOG.md` documents a Windows-specific bundled plugin/channel loader fix: bundled channel artifact loading and native-Jiti/plugin-loader paths were fixed on Windows so onboarding/doctor/bundled plugin installs work again.
  - file: [CHANGELOG.md:1141](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/CHANGELOG.md#L1141)
  - command: `nl -ba CHANGELOG.md | sed -n '1074,1144p'`
- **Windows native-Jiti disabled for bundled dist loads:** Current loader policy explicitly disables native Jiti on `win32`, which is the failure mode called out in the changelog for bundled plugin loading on Windows.
  - file: [src/plugins/sdk-alias.ts:698](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/plugins/sdk-alias.ts#L698)
  - command: `nl -ba src/plugins/sdk-alias.ts | sed -n '696,735p'`
- **Windows channel-loader regression test:** There is a dedicated test asserting channel plugin dist loads on Windows stay off Jiti native import.
  - file: [src/channels/plugins/module-loader.test.ts:95](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/channels/plugins/module-loader.test.ts#L95)
  - command: `git blame -L 84,120 -- src/channels/plugins/module-loader.test.ts`
- **Discord startup now logs and records bot/application status before monitor start:** Discord startup probes the bot, records status, warns about Message Content intent state, and always logs `starting provider` before starting the monitor.
  - file: [extensions/discord/src/channel.ts:741](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/extensions/discord/src/channel.ts#L741)

  - file: [extensions/discord/src/monitor/provider.ts:1111](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/extensions/discord/src/monitor/provider.ts#L1111)

## Close Comment

Closing as implemented on main after a Codex review. Current `main` already contains shipped Windows bundled-plugin loader fixes for bundled channel/plugin startup (`CHANGELOG.md:1140`-`1141`), and the current Discord channel code now records bot/application status and logs `[account] starting provider` before monitor startup (`extensions/discord/src/channel.ts:741`, `extensions/discord/src/channel.ts:761`). The monitor also emits explicit startup state (`extensions/discord/src/monitor/provider.ts:1111`, `extensions/discord/src/monitor/startup-status.ts:1`) instead of leaving Discord invisible. There are dedicated Windows regression tests for the bundled channel/public-surface loader path as well (`src/channels/plugins/module-loader.test.ts:95`, `src/plugins/public-surface-loader.test.ts:29`). Based on that evidence, this looks fixed in release `2026.4.7`; if you still hit a Discord startup failure on a current release, please open a fresh issue with exact version, install layout, and the gateway log lines around Discord startup.
