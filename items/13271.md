## Summary

Keep #13271 open. Current main has adjacent controls for per-channel allow/deny tool policy, repetitive tool-loop detection, before_tool_call plugin blocking, and manual sub-agent spawning, but it still does not implement a channel-scoped per-turn tool-call budget with warn/block/auto-delegate overflow behavior.

## Best Possible Solution

Keep #13271 open until OpenClaw has a documented channel-scoped or agent/channel-scoped tool-call budget contract. A complete path should add typed config/schema/help/docs, carry reliable channel/request context into tool-call enforcement, count tool calls per turn in the runtime, define warn/block/auto-delegate overflow behavior, wire auto-delegation through sessions_spawn/sub-agent completion, and add coverage for Discord plus at least one non-Discord channel. If maintainers prefer plugin ownership, first add the missing hook/runtime contract so a plugin can implement this cleanly.
