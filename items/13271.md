## Summary

Keep #13271 open. Current main still has adjacent mechanisms, including per-channel allow/deny tool policy, opt-in tool-loop detection, before_tool_call blocking, and explicit subagent/session-spawn APIs, but it does not implement the requested per-channel, per-turn tool-call budget with warn/block/auto-delegate overflow semantics. The existing issue discussion also includes a maintainer-side Codex review explicitly keeping this item open, and related #9912/#14373 do not supersede this channel-scoped request.

## Best Possible Solution

Keep #13271 open until OpenClaw adds a typed product/API path for per-channel, per-turn tool-call budgets. A complete fix should add documented channel or agent/channel-scoped config, pass reliable channel/request context into enforcement, define warn, block, and auto-delegate semantics, and cover Discord plus at least one non-Discord channel in tests. If maintainers prefer plugin ownership, add the missing hook/runtime contract first rather than closing this as ClawHub-only.
