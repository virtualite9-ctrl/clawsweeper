## Summary

Keep #72208 open. It is a fresh Windows/npm-global regression report against v2026.4.24, and current main does not show a clear shipped fix for the reported gateway startup hang, unreachable dashboard, and unresponsive Ctrl+C when OpenRouter and LiteLLM pricing fetches time out. The implicated pricing refresh path is still active, the release already had the async bootstrap deferral, and the post-release source diff only changes the timeout from 30s to 60s plus an import path.

## Best Possible Solution

Keep the issue open. Reproduce on Windows 11 with npm-global v2026.4.24 and current main while forcing OpenRouter and LiteLLM pricing fetches to time out, then verify HTTP readiness/dashboard reachability and SIGINT shutdown. If reproduced, decouple/cancel the pricing refresh so it cannot block startup or shutdown, and add a regression test that forces both pricing sources to hang or time out while asserting the gateway becomes reachable and closes promptly.
