Keep open. Current main still contains the implicated Gateway model-pricing refresh path without the unref/abort shutdown fix, and the linked fix PR #72306 is open rather than merged. The issue is a same-day regression with a second user confirmation, so it should remain open until the PR or an equivalent fix lands and is verified.

## Best Possible Solution

Keep #72208 open and use PR #72306 as the active canonical fix path. Land #72306 or an equivalent change that unrefs the pricing refresh timer, aborts in-flight LiteLLM/OpenRouter pricing fetches on gateway stop, and adds regression coverage; then verify Windows npm-global `openclaw gateway` with forced pricing timeouts for dashboard readiness and Ctrl+C shutdown before closing.
