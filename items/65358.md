## Summary

Keep this open. Current main has partial mitigations for session reset, backup creation, and oversized transcript compaction, but the central upgrade-safety request is still not implemented: updates do not automatically snapshot state/workspace/session data, rollback is still manual version pinning, CLI session repair is still list/cleanup only, legacy session overrides are intentionally preserved on reset, and status diagnostics still reduce bootstrap state to a BOOTSTRAP.md check rather than warning about missing MEMORY.md with daily memory present.

## Best Possible Solution

Keep this open and treat it as concrete upgrade-safety hardening. The best path is to wire an opt-out pre-update backup through the existing backup planner, add a supported restore/rollback flow for state/workspace/session data, expose CLI tooling to reset/delete one session key or clear model/provider/auth overrides, and add doctor/status diagnostics for stale session overrides plus missing `MEMORY.md` when daily memory files or workspace bindings suggest continuity should exist. Keep the existing reset and compaction mitigations as partial fixes, not as the complete resolution.
