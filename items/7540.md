## Summary

Keep #7540 open. Current main still does not subscribe the bundled WhatsApp/Baileys plugin to Baileys call events, has no WhatsApp call-tracking config or hook exposure, and the existing issue discussion says the request remains valid and should be handled in the WhatsApp plugin rather than the PSTN voice-call plugin.

## Best Possible Solution

Keep the issue open and implement it inside the bundled WhatsApp plugin: add an opt-in channel/account setting if maintainers want one, subscribe to Baileys call events, normalize and sanitize call payloads, apply the existing account, DM/group, pairing, and allowlist boundaries, then expose contact activity through a documented session-event, tool, or plugin-hook surface with focused config, docs, and tests.
