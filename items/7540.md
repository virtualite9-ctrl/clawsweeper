## Summary

Keep #7540 open. Current `main` still does not subscribe the bundled WhatsApp/Baileys plugin to Baileys `call` events, and the recent maintainer discussion explicitly says this remains valid and should stay WhatsApp-plugin-owned rather than being folded into the PSTN `voice-call` plugin.

## Best Possible Solution

Keep the issue open and implement the feature inside the bundled WhatsApp plugin: add an opt-in channel/account setting if maintainers want one, subscribe to Baileys `call`, normalize and sanitize call payloads, apply the existing account, DM/group, pairing, and allowlist boundaries, then expose contact activity through a documented session-event/tool/plugin-hook surface with focused config, docs, and tests.
