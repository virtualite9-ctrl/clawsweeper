## Summary

Keep #7540 open. Current main still does not subscribe the bundled WhatsApp plugin to Baileys `call` events, does not define WhatsApp call-tracking config, and does not expose sanitized WhatsApp call activity through the existing message/session/plugin-hook surfaces. The maintainer discussion in the issue also says the request remains valid and should stay WhatsApp-plugin-owned rather than being folded into the PSTN voice-call plugin.

## Best Possible Solution

Implement this in the bundled WhatsApp plugin under `extensions/whatsapp`: subscribe to Baileys `call` events, normalize and sanitize `WACallEvent` payloads, apply the same account/access-control boundaries as inbound messages, add an opt-in channel/account setting if maintainers want one, expose contact activity through a documented session-event, plugin-hook, or tool surface, and cover it with focused tests and docs. Do not close as ClawHub or voice-call-plugin work; this is WhatsApp Web channel behavior with maintainer validation.
