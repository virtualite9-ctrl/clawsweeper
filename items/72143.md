## Summary

Keep open. Current main still prepends the internal session-resume fallback text to direct user delivery after an exec approval follow-up resume failure, has no retry/backoff around the session follow-up call, and the linked PR #72148 is still open and explicitly handles only the prefix removal, not the retry portion of the issue.

## Best Possible Solution

Keep #72143 open until the exec approval follow-up path is fixed on main: retry the session follow-up with short bounded backoff, remove the direct-delivery internal prefix, log resume failure metadata internally when fallback is used, and update the existing follow-up tests to assert prefix-free fallback plus retry behavior. #72148 can be the starting point if maintainers accept a prefix-only first step, but it should not close the whole issue as-is.
