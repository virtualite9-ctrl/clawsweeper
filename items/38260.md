## Summary

Keep #38260 open. Current main still has the WebChat image attachment path that can reach in-process sharp/libvips image operations on Linux/Node, and I did not find source, docs, tests, changelog, or related-item evidence that the reported libvips SIGILL crash has been fixed or made obsolete.

## Best Possible Solution

Keep the issue open. A maintainer should reproduce or collect a core/backtrace on Linux npm-global Node 22, determine whether sharp's bundled libvips, system libvips selection, CPU feature dispatch, or OpenClaw's image optimization path is responsible, then ship a known-safe dependency/runtime fallback, documented Linux kill switch, or out-of-process image-processing isolation plus a WebChat image-flow regression smoke.
