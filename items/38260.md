## Summary

Keep #38260 open. Current main still sends Linux/Node WebChat image handling through in-process sharp/libvips via the media-understanding-core image-ops artifact, and I found no source, test, changelog, release, or related-item evidence proving the reported libvips-cpp SIGILL crash is fixed, superseded, or no longer reproducible.

## Best Possible Solution

Keep the issue open. A maintainer should reproduce or collect a core/backtrace on Linux npm-global Node 22, determine whether sharp's bundled libvips, system libvips selection, CPU feature dispatch, or OpenClaw's image optimization path is responsible, then ship a known-safe dependency/runtime fallback, documented kill switch, or out-of-process image-processing isolation plus a WebChat image-flow regression smoke.
