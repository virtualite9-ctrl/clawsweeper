## Summary

Keep open. Current main still defines `GroupChatSchema.historyLimit` with `.positive()`, while the documented behavior says `0` disables group history and related schemas use `.min(0)`. PR #65359 is the open same-author implementation candidate with closing syntax, so this issue should stay open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review/land PR #65359 or an equivalent patch that changes `GroupChatSchema.historyLimit` from `.positive()` to `.min(0)`, updates generated schema artifacts if required by the repo workflow, and adds focused regression coverage for zero, positive, and negative values. After the fix merges, this can close as implemented.
