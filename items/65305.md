## Summary

Current main still rejects `messages.groupChat.historyLimit: 0` because `GroupChatSchema.historyLimit` remains `z.number().int().positive().optional()`. The documented behavior says `0` disables group history, and the same-author PR #65359 is still the open implementation candidate that references this issue with closing syntax, so this issue should stay open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review/land PR #65359 or an equivalent patch that changes `GroupChatSchema.historyLimit` from `.positive()` to `.min(0)` and adds focused regression coverage for zero, positive, and negative values. After the fix merges, this issue can close as implemented.
