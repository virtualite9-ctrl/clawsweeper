## Summary

Current `main` already covers this request under existing knobs: `sessions_spawn` supports `cleanup: "delete"`, isolated cron runs support `deleteAfterRun`, and the control UI hides cron sessions from the session bar by default.

## Evidence

- **`sessions_spawn` has explicit auto-cleanup:** Spawned subagent runs accept `cleanup: "delete"` for run-mode sessions, and the announce path deletes the child session with `sessions.delete` when that cleanup mode is selected.
  - file: [src/agents/subagent-spawn.ts:584](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/agents/subagent-spawn.ts#L584)

- **Subagent docs already describe the cleanup flag:** The public subagent docs document `cleanup? (delete|keep)` and state that `cleanup: "delete"` archives the child session after announce.
  - file: [docs/tools/subagents.md:114](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/docs/tools/subagents.md#L114)
