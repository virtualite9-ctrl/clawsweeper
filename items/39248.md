## Summary

Keep #39248 open. The issue has concrete reproduction data, the current main path still makes native subagent children sandbox candidates under `agents.defaults.sandbox.mode: "non-main"`, and I found no current source/test/release evidence proving `sessions_spawn` child initialization now makes progress under that mode.

## Best Possible Solution

Keep the issue open until maintainers add a targeted regression test or live verification for native `sessions_spawn` under `agents.defaults.sandbox.mode: "non-main"`. The best fix path is to prove the child run starts, writes transcript/message state, sets `systemSent` and token state, and surfaces sandbox initialization or dispatch failures instead of silently timing out; if the repro still fails, patch the Docker-backed `resolveSandboxContext` or child dispatch path.
