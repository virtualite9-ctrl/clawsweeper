## Summary

Keep open. Current main has related update and gateway-service hardening, but it does not fully solve this Windows pnpm -g report: pnpm global shim refresh is still delegated to the package-manager flow, and local node-llama-cpp build reuse/migration across pnpm layout changes is not implemented. The linked PR #48927 was closed unmerged.

## Best Possible Solution

Keep the issue open but narrow it. Maintainers should either split it into two tracked fixes or update this issue with a current reproduction: one path for stale Windows pnpm global launchers/service metadata after package upgrades, and one path for explicit local embeddings that need a stable node-llama-cpp install/cache strategy or documented migration away from pnpm virtual-store localBuilds.
