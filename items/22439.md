## Summary

Keep open. Current main still does not implement the PR's proposed `agents.defaults.bootstrapTier` feature: bootstrap filtering remains binary by session type, config/docs do not expose `bootstrapTier`, and the extra bootstrap hook has no tier-aware path. The linked issue #22438 remains open, and a recent maintainer-side Codex review explicitly kept this PR open rather than treating it as implemented, obsolete, or superseded.

## Best Possible Solution

Keep #22439 open for maintainer product review alongside #22438. If accepted, the implementation needs a schema-backed `agents.defaults.bootstrapTier`, tier resolution in the bootstrap file path, tier-aware extra bootstrap handling, generated schema/docs updates, and focused tests for `minimal`, `standard`, `full`, and backward-compatible defaults. Close only if maintainers explicitly reject the product direction or a canonical follow-up fully covers configurable tiered bootstrap behavior.
