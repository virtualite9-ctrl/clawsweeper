## Summary

Keep PR #22439 open. Current main still has only the existing binary bootstrap filtering plus separate lightContext support; it does not implement schema-backed `agents.defaults.bootstrapTier`, `BootstrapTier`, `resolveBootstrapTier`, or `minimal`/`standard`/`full` filtering. The PR also has fresh author follow-up and a same-day maintainer-side Codex review comment explicitly keeping it open with canonical issue #22438.

## Best Possible Solution

Keep #22439 open for maintainer product review alongside #22438. If maintainers accept the direction, land a current-main version that adds schema-backed `agents.defaults.bootstrapTier`, a tier resolver, tier-aware bootstrap filtering, wiring through `resolveBootstrapFilesForRun` and the `bootstrap-extra-files` hook, generated config/help/docs updates, and focused tests for default behavior plus `minimal`, `standard`, and `full`. If maintainers prefer the existing `lightContext` or a different bootstrap-control design, close it manually after that explicit product decision.
