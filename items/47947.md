## Summary

Keep open. Current `main` still allows the reported malformed payloads: the WebSocket request builder unconditionally adds `tool_choice` while omitting `tools` when the array is empty, and the embedded-runner extra-params path still injects `parallel_tool_calls` without a corresponding non-empty `tools` array. The requested fix is not implemented on `main`.
