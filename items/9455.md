## Summary

Keep #9455 open. Current main has the Cron Jobs filter bar and partial filtering from merged #9510, but the remaining requested behavior is still missing: search does not include job IDs, and the Jobs last-run-status filter still cannot select jobs with absent/unknown status. The latest issue discussion also explicitly kept this open for those same gaps.

## Best Possible Solution

Keep #9455 open, but narrow it to the remaining delta on top of merged #9510: include job.id in the cron.list search haystack or client-side search, add unknown to the Cron Jobs last-status filter type and select options, map absent lastStatus/lastRunStatus into that unknown bucket, and add focused controller/service/view tests for job-id search and unknown-status filtering.
