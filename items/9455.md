## Summary

Keep #9455 open. Current main has the PR #9510 Cron Jobs filter bar and partial schedule/status filtering, but it still does not satisfy the requested job-id search or explicit unknown last-run-status filter.

## Best Possible Solution

Keep #9455 open but narrow it to the remaining delta on top of merged #9510: include `job.id` in the `cron.list` search haystack or otherwise make job-id search work from the Control UI, add `unknown` to the Cron Jobs last-run-status filter type and select options, map missing `lastStatus`/`lastRunStatus` to that filter, and add focused controller/service/view tests for both behaviors.
