## Summary

Keep #9455 open. Merged #9510 added the Cron Jobs filter bar, but current main still misses two explicit acceptance paths: search by jobId and filtering jobs whose last-run status is absent/unknown.

## Best Possible Solution

Keep #9455 open, but narrow the remaining work on top of #9510: include `job.id` in the `cron.list` search haystack, add an `unknown` Jobs last-status filter value and option, normalize `lastStatus ?? lastRunStatus` with missing status mapped to unknown, and add focused tests for job-id search plus unknown-status filtering.
