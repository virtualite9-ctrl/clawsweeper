## Summary

Keep this issue open. Current main still renders a single combined chat session picker grouped by agent, not separate Agent and Agent Session controls, and the related picker-noise work does not supersede the requested UI split.

## Best Possible Solution

Keep this issue open and implement the Control UI split in the owner UI surface: add an Agent selector sourced from `agentsList`, add an Agent Session selector filtered from `sessionsResult` by `parseAgentSessionKey(...).agentId`, switch agents to the most recent or main session via `buildAgentMainSessionKey`, and preserve the existing Model/Thinking `sessions.patch` behavior against the active `state.sessionKey`. Update both desktop `renderChatSessionSelect` and mobile `renderChatMobileToggle`, with focused tests for filtering, agent switching fallback, and session-bound Model/Thinking behavior.
