## Summary

Current main still implements the chat header and mobile chat controls as a single session picker grouped by agent, not separate Agent and Agent Session controls. The related picker-noise work around hiding internal/subagent sessions is complementary and does not supersede the requested UI split.

## Best Possible Solution

Keep this issue open. Implement the split in the Control UI chat surfaces: add an Agent selector sourced from agentsList, add an Agent Session selector filtered from sessionsResult by parseAgentSessionKey(...).agentId, switch agents to the most recent session for that agent or buildAgentMainSessionKey({ agentId }) as fallback, and preserve existing Model/Thinking sessions.patch behavior against the active state.sessionKey. Update both renderChatSessionSelect and renderChatMobileToggle with focused tests for filtering, agent switching fallback, and session-bound Model/Thinking behavior. Coordinate with #68959 for the shared agent-switcher gap, but do not close this issue as superseded unless a canonical item explicitly covers the full split-picker behavior.
