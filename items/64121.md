## Summary

Current main still has the combined chat session picker. Desktop and mobile render a single session select grouped by agent, with Model and Thinking controls still bound to the active session. The requested separate Agent and Agent Session controls are not implemented, and the related open agent-switcher issue is complementary rather than a complete superseding tracker.

## Best Possible Solution

Keep this issue open. Implement the split in the Control UI chat surfaces by adding an Agent selector backed by agentsList, adding an Agent Session selector filtered from sessionsResult by parseAgentSessionKey(...).agentId, switching an agent to its most recent session or buildAgentMainSessionKey({ agentId }) as fallback, and leaving Model/Thinking sessions.patch behavior targeted at state.sessionKey. Mirror the desktop header behavior in mobile controls and add focused tests for agent filtering, agent-switch fallback, and preserving session-bound Model/Thinking overrides. Coordinate with #68959, but do not close this issue unless a canonical item explicitly tracks the complete split-picker behavior.
