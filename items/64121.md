## Summary

Keep open. Current `main` still uses a single session picker in chat, with agent grouping only inside that picker; the requested split into separate Agent and Agent Session controls is not implemented on desktop or mobile.

## Evidence

- **Desktop chat header still uses one combined session picker:** `renderChatSessionSelect()` renders a single `<select>` bound to `state.sessionKey`, then separately renders Model and Thinking controls. There is no separate Agent control in this path.
