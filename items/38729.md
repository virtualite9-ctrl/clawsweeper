## Summary

Keep PR #38729 open. Current main still does not advertise `xhigh` or `max` for Anthropic Claude Opus 4.6, while the PR’s submitted `supportsXHighThinking` hook is stale against the active `resolveThinkingProfile` path. The transport can already map a forced Opus 4.6 `xhigh` run to Anthropic `output_config.effort: "max"`, so the underlying request remains plausible, but it needs a reworked provider-profile change and maintainer product judgment rather than closure.

## Best Possible Solution

Keep the PR open, but require a rework before merge. If OpenClaw should expose Claude Opus 4.6 max effort, implement it in `extensions/anthropic/register.runtime.ts` through `resolveThinkingProfile`, update Anthropic provider tests and `docs/tools/thinking.md`, and keep Sonnet 4.6 excluded unless runtime behavior supports a distinct max/xhigh level. If Opus 4.7-only advertisement is intentional product policy, a maintainer should close the PR with that rationale or link a canonical policy issue.
