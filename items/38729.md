## Summary

Keep PR #38729 open. Current main still does not advertise `xhigh`/`max` for Anthropic Claude Opus 4.6, so the requested capability is not implemented. The submitted one-hook patch is stale against the current provider-profile path, but the underlying request remains concrete and plausible because the Anthropic transport already maps a forced Opus 4.6 `xhigh` request to provider `effort: "max"`. No close reason applies.

## Best Possible Solution

Keep the PR open, but require a rework against the current provider-profile path. If OpenClaw should expose Claude Opus 4.6 max effort, implement it in `extensions/anthropic/register.runtime.ts` by adding the level through `resolveThinkingProfile`, update `extensions/anthropic/index.test.ts`, and update `docs/tools/thinking.md`. Keep Sonnet 4.6 excluded unless the transport/runtime can provide a distinct max/xhigh behavior. If the Opus 4.7-only boundary is intentional product policy, a maintainer should close the PR with that rationale or link a canonical policy issue.
