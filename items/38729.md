## Summary

Keep PR #38729 open. Current main still does not implement the requested Anthropic Claude Opus 4.6 `xhigh`/`max` advertisement: the Anthropic provider profile only adds `xhigh`/`max` for Opus 4.7, and tests explicitly assert Opus 4.6 excludes both. The PRâ€™s current `supportsXHighThinking` hook is stale against main because the shared thinking resolver returns a non-empty provider `resolveThinkingProfile` before consulting legacy hooks. The request is still plausible because the Anthropic transport can map an explicit Opus 4.6 `xhigh` request to provider effort `max`, but the remaining work needs a provider-profile implementation or a maintainer product decision to keep the Opus 4.7-only boundary.

## Best Possible Solution

Keep the PR open but request a rework against the current provider-profile path. If OpenClaw should expose Claude Opus 4.6 max effort, add the level through `extensions/anthropic/register.runtime.ts` `resolveThinkingProfile`, update `extensions/anthropic/index.test.ts`, and update `docs/tools/thinking.md` / public docs wording. Keep Sonnet 4.6 excluded unless the runtime can provide distinct max/xhigh behavior. If Opus 4.7-only advertisement is intentional policy, a maintainer should close with that rationale or link a canonical policy item.
