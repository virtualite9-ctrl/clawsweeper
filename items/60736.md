## Summary

Current `main` does not implement first-class Google Vertex support. `google-vertex` is still wired as an alias/manual-config path under the Google plugin, with no dedicated provider manifest entry, no onboarding/config surface for Vertex location, no `CLOUD_ML_REGION` fallback/default handling, and no built-in Vertex endpoint construction/routing.

## Evidence

- **`google-vertex` is still only a hook alias on the main Google provider:** The bundled Google provider registers `hookAliases: ["google-antigravity", "google-vertex"]` under provider id `google`, keeps only Gemini API-key auth env vars, and only installs a stream function for `google-generative-ai` models. There is no dedicated Vertex provider implementation here.
  - file: [extensions/google/provider-registration.ts:19](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/extensions/google/provider-registration.ts#L19)

  - sha: [6bc0dc8fb671](https://github.com/openclaw/openclaw/commit/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440)
- **Plugin manifest still lacks a first-class `google-vertex` provider/onboarding surface:** The Google plugin manifest exposes only `google` and `google-gemini-cli` as providers, only `google-gemini-cli` as an extra auth choice/backend, and its config schema contains only `webSearch` fields. There is no manifest/provider-auth/config entry for Vertex region/location.
