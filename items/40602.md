## Summary

Keep open. Current main still uses the zero-field-only Feishu Bitable placeholder cleanup predicate, while this PR narrowly targets the remaining case where Feishu returns placeholder rows with present but substantively empty default field values. The PR is not superseded by the related Feishu Bitable delete or permission work.

## Best Possible Solution

Keep this PR open as a plausible focused Feishu plugin bug fix. The best maintainer path is to add a narrow regression test around the create-app cleanup path or an extracted cleanup predicate, covering default-empty Feishu values plus non-empty counterexamples such as URL/link objects, files, users, locations, booleans, and numbers, then merge the predicate if the test proves it deletes only placeholder rows.
