## Summary

Keep #29725 open. Current main has static Discord presence, auto-presence, and a gated `setPresence`/`set-presence` action, but it still does not expose the requested typed code-plugin runtime helper `api.runtime.channel.discord.updatePresence()` or a documented equivalent plugin seam.

## Best Possible Solution

Keep the issue open until maintainers either add a narrow typed Discord presence runtime helper or document a supported code-plugin equivalent. A durable implementation likely belongs in the Discord plugin runtime surface, wired to the existing gateway registry/updatePresence path with account selection, action gating, connection checks, docs, and contract tests.
