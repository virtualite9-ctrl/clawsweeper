## Summary

Current main still does not expose the requested `api.runtime.channel.discord.updatePresence()` helper or a documented code-plugin equivalent. Discord presence can be set through config, auto-presence, and a gated internal/message-action path, but the specific supported plugin runtime seam remains missing.

## Best Possible Solution

Keep the issue open until maintainers either add a narrow typed Discord presence runtime helper or document a supported code-plugin equivalent. A durable implementation likely belongs in the Discord plugin runtime surface, backed by the existing gateway registry/updatePresence path with action gating, account selection, connection checks, public docs, and focused contract tests.
