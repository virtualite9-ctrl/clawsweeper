## Summary

Keep #29725 open. Current main still supports Discord presence through static config, auto-presence, and the gated `setPresence` message action, but it does not expose the requested supported code-plugin runtime API `api.runtime.channel.discord.updatePresence()` or a documented equivalent plugin seam.

## Best Possible Solution

Keep #29725 open until maintainers either add a typed, documented Discord presence runtime helper for code plugins or explicitly document a supported alternate code-plugin seam. A durable fix likely belongs in the Discord plugin runtime surface, wired to the existing gateway registry/updatePresence path, with docs and contract tests while preserving the core/plugin boundary.
