## Summary

Current main still lacks the requested `api.runtime.channel.discord.updatePresence()` helper. Discord presence is implemented for static config, auto-presence, and the gated `setPresence` message-action path, but that does not satisfy the issue’s requested typed code-plugin runtime API.

## Best Possible Solution

Keep #29725 open until maintainers either add a narrow Discord-owned `api.runtime.channel.discord.updatePresence()` helper, or document a supported code-plugin equivalent. The likely implementation belongs in the Discord plugin runtime surface, reusing the existing gateway registry/updatePresence path and adding focused tests/docs for gating, account selection, validation, and connection state.
