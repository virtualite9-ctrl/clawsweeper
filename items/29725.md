## Summary

Current main still does not expose the requested `api.runtime.channel.discord.updatePresence()` helper or a documented code-plugin equivalent. The Discord plugin has static presence config, auto-presence, and a gated `setPresence` message-action path backed by `gateway.updatePresence`, but the specific supported plugin runtime seam requested in #29725 remains missing.

## Best Possible Solution

Keep #29725 open until maintainers either add a narrow typed Discord presence runtime helper or document a supported code-plugin equivalent. A durable implementation likely belongs in the Discord plugin runtime surface, backed by the existing gateway registry/updatePresence path with action gating, account selection, connection checks, docs, and focused contract tests.
