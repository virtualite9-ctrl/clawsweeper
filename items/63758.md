## Summary

Keep open. Current main still creates and finalizes managed cron task records without setting or updating progressSummary, while /tasks, /status, and session_status only show live task detail for active tasks when progressSummary is present.

## Best Possible Solution

Keep this issue open and fix the cron producer paths, not the generic task registry. Add coarse progressSummary values when scheduled, startup catch-up, and manual cron task runs are created, update them at meaningful execution phases, and add regression coverage for cron task records plus /tasks or session_status so active cron work exposes live detail before terminalSummary is written.
