## Summary

Current main still matches the reported bug: cron task rows are created and finalized, but the cron producer paths do not seed or update `progressSummary`, so `/tasks`, `/status`, and `session_status` have no live detail beyond the task title while cron work is active. The prior automated review comment correctly identified the same gap on an earlier SHA, and the gap remains present at `fee16865b229cf3d652041eadb12a1b25dba07c0`.

## Best Possible Solution

Keep this issue open. The fix should be owned by the cron service: seed a coarse `progressSummary` when scheduled, startup catch-up, and manual cron task runs are created; update it through `recordTaskRunProgressByRunId` at meaningful execution phases; and add focused regression coverage proving active cron work has live detail in `/tasks`, `/status`, or `session_status` before `terminalSummary` is written.
