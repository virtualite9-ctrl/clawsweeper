## Summary

Current main still matches the reported bug. Cron task rows are created and finalized, but the cron scheduler, startup catch-up, and manual run paths still do not seed or update `progressSummary`; active `/tasks`, `/status`, and `session_status` detail therefore has no cron progress text to show before terminal completion.

## Best Possible Solution

Keep this issue open. The cron service should own the fix: seed a coarse `progressSummary` when scheduled, startup catch-up, and manual cron task rows are created; update it via `recordTaskRunProgressByRunId` at meaningful execution phases; and add focused regression coverage showing active cron work has live detail in `/tasks`, `/status`, or `session_status` before `terminalSummary` is written.
