## Summary

Keep open. Current main at 4878d3e059cee7fb0652cf973e82d76cc7371764 still creates and finalizes managed cron task records without setting `progressSummary`, while the `/tasks` and `session_status` status detail paths only include live detail when a queued/running task has that field populated.

## Best Possible Solution

Fix the cron producer paths, not the generic task registry. Add coarse `progressSummary` values when scheduled, startup catch-up, and manual cron task records are created, and update them at meaningful phases such as queued/running execution, delivery handoff, and cleanup. Add regression coverage for cron task records and the user-facing status formatter or `/tasks`/`session_status` path so active cron work exposes nonempty live detail before terminal summary is written.
