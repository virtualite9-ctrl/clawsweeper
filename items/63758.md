## Summary

Current main still appears to have the reported gap. The cron producer creates and finalizes task-ledger rows, but the automatic and manual cron paths do not set or update `progressSummary` while the run is active. The task registry and status surfaces already support progress summaries, so this remains an actionable cron producer bug rather than a storage/UI limitation.

## Best Possible Solution

Keep the issue open and patch the cron producer paths to set coarse `progressSummary` when cron runs start and, if feasible, update it at meaningful phases such as queued/started, isolated agent turn running, delivery pending, and cleanup/finalization. The likely implementation belongs in `src/cron/service/timer.ts` and `src/cron/service/ops.ts`, using the existing `recordTaskRunProgressByRunId`/`startTaskRunByRunId` task lifecycle APIs and adding focused cron task-ledger regression tests.
