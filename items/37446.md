## Summary

Keep #37446 open. Current main still lacks a stable `sessions_spawn` recovery/idempotency identity: the public tool schema exposes no caller-supplied key, native and ACP spawn paths still generate fresh child session keys and fresh gateway idempotency keys per call, and the existing gateway dedupe only helps when the same key is reused. The linked ClawHub `multi-agent-deployment-pro` skill is useful operational guidance, but it does not replace the missing core recovery contract.

## Best Possible Solution

Keep the issue open until core sub-agent orchestration has an additive idempotent recovery contract. The likely fix is a stable caller-supplied or deterministic `sessions_spawn` recovery/idempotency key, persisted key-to-child-session/run mapping, replay behavior that returns or reuses the existing child run after a lost ack or parent timeout, and regression coverage for both native and ACP spawn paths. The linked ClawHub skill should remain recommended operational guidance, not a substitute for the core API contract.
