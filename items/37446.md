## Summary

Keep #37446 open. Current main still lacks a stable caller-supplied or deterministic `sessions_spawn` recovery identity: the public tool schema has no idempotency/recovery parameter, native and ACP spawn paths still mint fresh child session keys and fresh gateway `agent.idempotencyKey` values per attempt, and gateway dedupe only works when the same key is reused. The linked ClawHub multi-agent deployment skill is useful operational guidance, but it does not replace the missing core retry/replay contract.

## Best Possible Solution

Keep the issue open until OpenClaw core has an additive idempotent recovery contract for sub-agent orchestration. The likely product/code path is a stable caller-supplied or deterministic `sessions_spawn` recovery key, persisted mapping from key to child session/run, replay behavior that returns or reuses the existing child run after parent timeout or lost acknowledgement, and regression coverage for both native and ACP spawn paths. The linked ClawHub skill should remain recommended operational guidance, not a substitute for the core API contract.
