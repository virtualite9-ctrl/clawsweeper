## Summary

Current `main` treats `unauthorized: gateway token missing` as the expected Control UI response when a remote/Docker browser has not supplied the configured gateway token/password yet, and the UI clears that error on successful handshake. The issue thread never established a reproducible bug beyond opening the UI shell without authenticated Gateway connection.

## Evidence

- **Gateway intentionally returns this exact auth-required message for Control UI clients:** `formatGatewayAuthFailureMessage` maps missing token auth for Control UI/WebChat clients to `unauthorized: gateway token missing (open the dashboard URL and paste the token in Control UI settings)`.
  - file: [src/gateway/server/ws-connection/auth-messages.ts:20](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/src/gateway/server/ws-connection/auth-messages.ts#L20)

- **Current Docker docs say remote/container users must paste the shared secret in Control UI settings:** The Docker guide's Control UI step says to open the UI and paste the configured token/password into Settings; this matches the reported warning rather than contradicting it.
  - file: [docs/install/docker.md:62](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/docs/install/docker.md#L62)

- **Current dashboard docs describe the same remote-auth flow:** The dashboard guide says remote/non-localhost access still requires the configured token/password unless using identity-bearing modes, and explicitly tells users to paste it into dashboard settings.
  - file: [docs/web/dashboard.md:38](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/docs/web/dashboard.md#L38)

  - sha: [ec3dbd22a4de](https://github.com/openclaw/openclaw/commit/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46)
- **Successful auth clears the red connection error in the UI:** On `onHello`, the Control UI sets `connected = true` and clears both `lastError` and `lastErrorCode`, so a lingering red auth banner would require the connection to still be unauthenticated.
  - file: [ui/src/ui/app-gateway.ts:287](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/ui/src/ui/app-gateway.ts#L287)
