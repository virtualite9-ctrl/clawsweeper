## Summary

Keep open. Current main still propagates the source session accountId into cross-provider message sends, and the linked fix PR is open/unmerged rather than shipped.

## Best Possible Solution

Keep #69525 open until core changes preserve explicit target accountId but suppress the source session accountId for allowed cross-provider sends. The fix should add a regression test where currentChannelProvider differs from channel, allowAcrossProviders is true, and the target plugin receives accountId undefined so its own account fallback can run. PR #69529 is the natural starting point, but it should be reviewed against the message-tool params.accountId path as well.
