## Summary

Keep this PR open. Current main still uses Feishu `replyToMessageId` as the typing-reaction target, while topic-mode and `replyInThread` group flows intentionally set `replyToMessageId` to the thread/topic root. The related closed bug report #70209 describes the same user-visible problem, but current main does not contain the proposed fix, so this PR remains a narrow implementation candidate rather than obsolete or superseded.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land or reimplement the narrow fix: add an optional inbound-message target to `createFeishuReplyDispatcher`, pass `ctx.messageId` from both Feishu inbound dispatch call sites, use `inboundMessageId ?? replyToMessageId` only for the typing reaction, preserve the existing reply target/thread routing, and keep focused tests for topic/replyInThread plus fallback behavior.
