## Summary

Keep this PR open. Current main still targets the Feishu typing reaction at `replyToMessageId`, while topic-mode and `replyInThread` group handling can intentionally set that value to the thread root. The related closed bug report #70209 supports the user-visible problem, but it does not supersede the PR because current main does not contain the fix.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land or reimplement the narrow fix: add an optional inbound-message target to `createFeishuReplyDispatcher`, pass `ctx.messageId` from both Feishu inbound dispatch call sites, use `inboundMessageId ?? replyToMessageId` only for the typing reaction, preserve existing reply target/thread routing, and keep focused tests for topic/replyInThread plus fallback behavior.
