## Summary

Keep this issue open. Current main still routes ZAI image understanding through the generic image helper without a ZAI Coding Plan endpoint override, and the supplied GitHub context shows open PR #70915 as the active implementation candidate for this issue.

## Best Possible Solution

Keep this issue open while maintainers review and either land or reject #70915. The best fix belongs in the ZAI plugin/onboarding boundary: route ZAI vision models (`glm-4.6v`, `glm-4.5v`, `glm-5v-turbo`) to the same-region non-coding ZAI endpoint when the configured provider endpoint is `zai-coding-global` or `zai-coding-cn`, and add regression tests proving coding text models stay on the Coding Plan URL while vision models use the standard multimodal URL.
