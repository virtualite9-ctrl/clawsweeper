## Summary

Keep open. Current main still routes ZAI vision models through the provider-wide Coding Plan base URL, and #70915 remains an open implementation candidate for this issue.

## Best Possible Solution

Keep this issue open while maintainers review #70915 or implement an equivalent bundled ZAI plugin fix. The best code direction is a stable ZAI media-understanding/model-resolution route correction that sends vision models such as `glm-5v-turbo`, `glm-4.6v`, and `glm-4.5v` from same-region Coding Plan URLs to the standard multimodal endpoint, with regression coverage proving text/coding models stay on Coding Plan while vision models use the standard endpoint.
