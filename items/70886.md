## Summary

Keep #70886 open. Current main still routes ZAI vision models through the configured provider-level ZAI endpoint, and the bundled ZAI media-understanding provider has no current override from Coding Plan endpoints to the non-coding vision endpoint. The related PR #70915 is an open, unmerged fix attempt, so the original bug should remain open until a fix lands or a maintainer chooses a canonical replacement.

## Best Possible Solution

Keep the issue open and use #70915, or a replacement maintainer-authored patch, to fix ZAI-owned routing. The fix should live in the ZAI plugin/media-understanding path or catalog/onboarding transport metadata, include regression coverage for `zai-coding-global` and `zai-coding-cn` with `glm-4.6v`/other vision models, and verify that text/coding GLM models continue to use the Coding Plan endpoint.
