## Summary

Keep open. Current main still bundles the Bonjour plugin with unpatched `@homebridge/ciao@1.3.6`; the root `patchedDependencies` and lockfile do not include the proposed `@homebridge/ciao` patch, and upstream ciao 1.3.6 still calls the Windows `arp -a | findstr /C:"---"` path via `childProcess.exec` without `windowsHide`. The linked report #71492 describes the same Windows console flicker and the documented workaround is disabling the bundled Bonjour plugin, so this PR is a focused current-main fix rather than obsolete maintainer cleanup.

## Best Possible Solution

Keep #71633 open for maintainer review. The best path is to land a narrow `@homebridge/ciao@1.3.6` pnpm patch, or an equivalent upstream/version bump if available, that adds `{ windowsHide: true }` to the Windows `arp` polling exec call; include lockfile consistency and either a focused regression check or explicit Windows manual verification.
