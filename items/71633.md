## Summary

Keep this PR open. Current main still ships the bundled Bonjour plugin with unpatched @homebridge/ciao@1.3.6, and upstream ciao still runs the Windows arp polling exec without an options object. The linked Windows flicker report #71492 maps to this runtime path, while the broader #72332 remains open rather than a merged canonical superseder.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land this narrow @homebridge/ciao@1.3.6 pnpm patch, or an equivalent upstream version bump, with lockfile consistency and focused Windows verification that ciao's arp polling exec receives windowsHide: true. If maintainers choose #72332 as the canonical broader fix, mark this PR superseded only after that path is explicitly reviewed or merged.
