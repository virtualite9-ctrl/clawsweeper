## Summary

Keep #71633 open. Current main still ships the default-enabled Bonjour plugin with unpatched @homebridge/ciao@1.3.6, and upstream ciao still executes the Windows ARP polling command without windowsHide. The PR is a narrow, current-main fix for the linked Windows console-flicker report rather than maintainer cleanup to close.

## Best Possible Solution

Keep #71633 open for maintainer review. The best path is to land this narrow @homebridge/ciao@1.3.6 pnpm patch, or an equivalent upstream version bump/broader PR if maintainers choose one, with lockfile consistency and Windows manual verification or a focused regression check that the ciao Windows ARP exec receives windowsHide: true.
