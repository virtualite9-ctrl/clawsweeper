## Summary

Keep this PR open for maintainer rework. Current main already fixes the generic `openclaw logs --follow`/`logs.tail` cross-date path, but the PR’s stated `openclaw channels logs` path still reads the configured file directly and has no cross-midnight regression coverage; the PR also bundles unrelated cron warning work that is already represented by open #42245 for #42196.

## Best Possible Solution

Keep this PR open, but require rework before merge: split the cron warning into the existing #42196/#42245 path, then either close the logging branch as superseded for generic `openclaw logs --follow` or replace it with a focused `openclaw channels logs` patch that uses a shared exact rolling-log resolver and adds regression tests for cross-midnight missing-current-day files, existing current-day files, custom `logging.file`, and channel filtering.
