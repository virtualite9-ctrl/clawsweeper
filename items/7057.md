## Summary

Keep #7057 open. Current main still runs the reported WSL-sensitive `pi-tools.workspace-paths` and `pi-tools.safe-bins` tests without WSL-specific guards or temp-dir cleanup hardening, and both related mitigation PRs were closed unmerged. The issue has concrete reproduction data and a recent maintainer-side Codex comment also kept it open.

## Best Possible Solution

Keep #7057 open until a focused test-only fix lands for `src/agents/pi-tools.workspace-paths.test.ts` and `src/agents/pi-tools.safe-bins.test.ts`. The maintainer path should use the existing WSL detector only where justified, harden temp-dir creation/cleanup around cwd-sensitive tests, and include WSL proof or diagnostics without bundling unrelated changes.
