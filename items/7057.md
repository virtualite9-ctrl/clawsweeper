## Summary

Keep #7057 open. Current main still runs the reported WSL-sensitive workspace-paths and safe-bins tests without WSL guards or temp-dir cleanup hardening, while the related mitigation PRs in the provided context were closed unmerged. The issue has concrete WSL2 reproduction details and WSL2 is documented as a supported/recommended Windows path.

## Best Possible Solution

Keep #7057 open until a focused test-only fix lands for `src/agents/pi-tools.workspace-paths.test.ts` and `src/agents/pi-tools.safe-bins.test.ts`. The maintainer path should either harden temp-dir creation/cleanup around cwd-sensitive tests or add a narrowly justified WSL skip using the existing WSL detector, with WSL proof or diagnostics and no unrelated changes.
