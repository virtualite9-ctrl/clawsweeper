## Summary

Current main already solves the PR's observable problem: `agents.defaults.compaction.model` is resolved before direct, queued/context-engine, CLI, and after-turn compaction paths consume the compaction runtime context. The exact `ctx.model ?? runtime?.model` operand swap from this PR is not on main, but the only production setter for safeguard runtime does not create a state where `runtime.model` is a different override while `ctx.model` is still the chat model.

## Best Possible Solution

Close this PR as already implemented/superseded by the shipped compaction-target resolution on main. Keep the current central `resolveEmbeddedCompactionTarget()` approach and only revisit the safeguard operand order if a real production path needs distinct initialized `ctx.model` and runtime override models.
