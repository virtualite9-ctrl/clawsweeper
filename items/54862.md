## Summary

Keep open. Current `main` does not implement OpenRouter `autoRouter.allowedModels`, and the PR still represents a real, user-facing provider feature request. The review history also surfaces unresolved integration concerns, so this is not conservative-cleanup close material.

## Evidence

- **Current main only forwards OpenRouter provider routing metadata:** `wrapOpenRouterProviderStream()` reads `extraParams.provider`, injects `compat.openRouterRouting`, and then applies the shared thinking wrapper. There is no handling for `extraParams.autoRouter.allowedModels`, no `auto-router` plugin injection, and no `allowed_models` payload shaping on current `main`. That means the PR’s requested behavior is not already implemented.
