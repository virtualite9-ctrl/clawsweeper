## Summary

Keep open. Current main has better WebChat reconnect handling and a server-side `chat.send` ACK/idempotency path, but the specific requested client-side outbound queue with reconnect replay is still missing. The Control UI drops out early when disconnected, flushes pending RPCs on socket close, and only restores/shows an error for failed sends rather than retaining an unACKed send queue and resending it after reconnect.

## Best Possible Solution

Keep the issue open and implement a WebChat/Control UI outbound send queue keyed by stable client-generated idempotency keys. The client should enqueue before clearing the composer, send only on an open authenticated socket, remove an item only after the `chat.send` ACK, replay unACKed items after reconnect, and show per-message failure state after bounded retries. Existing gateway `chat.send` ACK/idempotency behavior can be reused; the missing work is mainly in `ui/src/ui/gateway.ts`, `ui/src/ui/app-chat.ts`, and focused Control UI reconnect tests.
