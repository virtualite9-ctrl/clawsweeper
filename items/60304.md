## Summary

Keep #60304 open. Current main still registers Xiaomi MiMo reasoning models through the generic OpenAI-compatible path with no Xiaomi-specific stream normalization, and that generic parser still maps `reasoning_content` into `thinking` blocks rather than visible assistant `text`. The PR is a focused bug fix with relevant regression coverage in its patch and positive review feedback; no shipped or canonical merged replacement was found in the provided related context or current checkout.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to adapt or merge a Xiaomi-owned `wrapStreamFn` normalization that promotes reasoning-only final MiMo messages to visible text for `mimo-v2-pro` and `mimo-v2-omni`, preserve non-target Xiaomi behavior, and keep regression tests covering streamed events and `result()` output. If maintainers prefer a shared helper, extract it through the existing provider stream seam and migrate Xiaomi with tests in the same change.
