## Summary

Current `main` no longer eagerly loads bundled channel config/runtime surfaces on cold CLI startup, and the replacement path is guarded against Jiti/module-load failures. The new lazy metadata/runtime code shipped in `v2026.4.23`, which is sufficient evidence that the `v2026.4.5` CLI crash path has already been addressed on supported releases.

## Evidence

- **Lazy CLI config path on current main:** `ChannelsSchema` now reads bundled plugin metadata with `includeChannelConfigs: false` and only does targeted per-channel runtime lookup when a plugin-owned channel key is actually present, instead of broad eager bundled-channel config loading during config normalization.
