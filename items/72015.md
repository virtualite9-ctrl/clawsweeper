## Summary

Keep open. Current main partially addresses the related hard-timeout failure from #65517, but the central reliability/defaults request in #72015 remains: Active Memory is still a blocking pre-prompt sub-agent that defaults to the current/session/agent model when no plugin model is set, and QMD startup still arms every configured QMD-enabled agent with no global boot throttling or active-only allowlist.

## Best Possible Solution

Keep #72015 open or split it into focused follow-ups: make Active Memory opt into blocking behavior or add a short pre-prompt deadline/nonblocking mode, prefer or require a dedicated fast recall model instead of inheriting slow chat models by default, add per-agent/session recall concurrency limits, and add QMD boot throttling or active-agent scoping for multi-agent gateways. The already-shipped hard timeout from the related #65517 path can be noted as completed within the broader issue.
