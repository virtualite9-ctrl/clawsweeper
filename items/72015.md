## Summary

Keep open. Current main partially addresses one sub-claim from the report by enforcing a hard Active Memory timeout, matching the related #65517 resolution, but the central reliability/defaults request remains actionable: Active Memory is still a blocking `before_prompt_build` sub-agent that inherits the current/session model by default, and QMD gateway startup still arms QMD managers for every configured agent with memory search enabled while `onBoot` defaults true.

## Best Possible Solution

Keep #72015 open as the canonical broader reliability/defaults issue. The best fix path is to preserve the shipped hard-timeout behavior, then add explicit nonblocking/deadline or opt-in blocking semantics for Active Memory, safer model/default guidance or warnings, per-agent/session recall concurrency limits, QMD boot concurrency/lazy/active-only controls, and status/doctor visibility for high-cost configurations.
