## Summary

Current main implements non-fatal Bonjour/CIAO cancellation handling for the VPS crash reported against v2026.4.24. The bundled Bonjour plugin now classifies `CIAO ANNOUNCEMENT CANCELLED` and `CIAO PROBING CANCELLED`, registers scoped process handlers, and the global unhandled-rejection path returns before fatal exit when a registered handler handles the reason. Tests cover the cancellation classifier, handler wiring, and stuck-advertiser disable path.

## Best Possible Solution

Close this issue as implemented on main. Keep the shipped scoped Bonjour process-error handling, shared unhandled-rejection registry, and advertiser self-disable behavior; if users still reproduce a gateway exit on a tagged release after this fix ships, they should open a new issue with current version, full gateway logs, platform, and Bonjour/mDNS configuration.
