## Summary

Keep this issue open. Current main still renders `/config show` from the raw parsed config without applying config redaction, and the same contributor has an open, unmerged implementation candidate, #65637, that explicitly closes this issue. Per the maintainer workflow, the issue should stay open until that PR is reviewed and merged or intentionally replaced.

## Best Possible Solution

Keep this issue open and review/land #65637, or replace it with an equivalent maintainer-approved fix that applies schema-aware `redactConfigObject` or `redactConfigSnapshot` behavior before both full and path-specific `/config show` rendering. Add regression coverage for full-config and targeted secret paths, then close as implemented after the fix reaches main and a release can be identified.
