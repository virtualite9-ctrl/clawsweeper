## Summary

Keep #44309 open. Current main still lacks a first-class one-way dispatch/handoff mode for `sessions_send`; the existing behavior is still conversation-oriented, with only `timeoutSeconds` and a global ping-pong turn cap exposed.

## Best Possible Solution

Keep this open and implement an explicit `sessions_send` delivery intent, such as `mode: "conversation" | "dispatch"` with conversation as the default. Dispatch should enqueue/persist the target-session message and optionally return a delivery ack, while skipping requester reply-back turns, requester reinjection, and announce delivery unless explicitly requested.
