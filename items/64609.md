## Summary

Current main still matches the reported Telegram multi-account inconsistency: channel-level group/topic prompt and skill config is dropped from the merged account snapshot in multi-account mode, while the group mention policy still falls back to channel-level groups. The linked fix PR #64650 is closed and unmerged, so there is no implemented mainline fix to close against.

## Best Possible Solution

Keep this issue open. The best fix is a Telegram-owned resolver change, likely in `extensions/telegram/src/bot-core.ts`, that resolves prompt/skill/topic context from `channels.telegram.accounts.<accountId>.groups` first and falls back to `channels.telegram.groups` only for context construction, not account routing. Add a regression test in `extensions/telegram/src/bot.create-telegram-bot.test.ts` for two configured Telegram accounts with channel-level group/topic `systemPrompt` and `skills`. If maintainers decide channel-level group prompts should remain unsupported in multi-account mode, add a Telegram doctor warning and public docs clarification instead of leaving the partial behavior silent.

## Risks / Open Questions

- A naive fix in `mergeTelegramAccountConfig` would likely reintroduce #30673 by making routing/delivery inherit channel-level groups wholesale in multi-account setups.
- A prompt-only fallback must keep account-scoped `groups` authoritative; otherwise channel-level prompts or skill filters could apply to an account that intentionally has its own group map.
