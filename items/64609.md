## Summary

Current main still has the reported Telegram multi-account inconsistency: channel-level group config is suppressed by account config merging, prompt/topic resolution reads only the merged account groups, while mention policy still falls back to channel-level groups. Related PR #64650 is closed and unmerged, so there is no shipped fix to close this issue against.

## Best Possible Solution

Keep this issue open. The best fix is a Telegram-owned prompt/skills/topic resolution change, likely in `extensions/telegram/src/bot-core.ts`, that reads account-level `groups` first and falls back to `channels.telegram.groups` only for group-context prompt/skill/topic lookup. Add a regression test in `extensions/telegram/src/bot.create-telegram-bot.test.ts` for two configured accounts with channel-level group/topic `systemPrompt` and `skills`, while leaving account routing inheritance behavior from #30673 intact. If maintainers decide this fallback is not desired, add a `doctor` warning and docs clarification instead.

## Risks / Open Questions

- A fix must preserve the #30673 routing safeguard: do not make `resolveTelegramAccount()` wholesale-inherit channel-level groups for multi-account delivery.
- Docs currently describe Telegram root `groups` as intentionally suppressed for multi-account setups, so maintainers should choose either a prompt-only fallback with clear docs or a doctor warning if channel-level group prompts remain unsupported.
