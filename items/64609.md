## Summary

Current main still has the reported Telegram multi-account inconsistency: multi-account account merging suppresses channel-level `groups`, prompt/topic resolution reads only that merged account snapshot, while mention policy still falls back to channel-level groups. The related implementation PR is closed and unmerged in the provided GitHub context, so there is no shipped fix to close this against.

## Best Possible Solution

Keep this issue open. The best fix is a Telegram-owned prompt/skills/topic resolution change, likely in `extensions/telegram/src/bot-core.ts`, that reads account-level `groups` first and falls back to `channels.telegram.groups` only for group-context prompt/skill/topic lookup. Add a regression test in `extensions/telegram/src/bot.create-telegram-bot.test.ts` for two configured accounts with channel-level group/topic `systemPrompt` and `skills`, while leaving account routing inheritance behavior from #30673 intact. If maintainers reject that fallback, add a `doctor` warning and docs clarification for multi-account channel-level group prompt config.
