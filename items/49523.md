## Summary

Keep open. Current main still writes new session transcript headers from process-global `process.cwd()` in the reported mirrored transcript append path, and related open PRs remain unmerged rather than shipped.

## Best Possible Solution

Keep #49523 open until a focused fix lands. The best patch is to thread the target agent workspace or loaded config plus `agentId` into all transcript header writers, write `resolveAgentWorkspaceDir(cfg, agentId)` instead of `process.cwd()`, and add regression coverage proving header cwd does not follow process-global cwd during mirrored transcript creation.
