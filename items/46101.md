## Summary

Current main still has the heartbeat scoping behavior the issue describes, and I did not find a doctor warning for the shadowed-defaults case. Main now exposes the disabled state through health/status summaries and documents the rule, but the requested `openclaw doctor` warning or startup warning remains a concrete, actionable core improvement.

## Best Possible Solution

Keep this open and add a focused doctor check, likely near existing config/security doctor warnings, that detects `agents.defaults.heartbeat` plus at least one explicit `agents.list[].heartbeat` and warns for listed agents without explicit heartbeat blocks. Add a regression test using the `main` plus `ops/kazuo` shape, and keep the existing health/status disabled display as supporting visibility rather than the primary fix.
