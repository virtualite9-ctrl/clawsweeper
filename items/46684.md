## Summary

Close #46684 as duplicate/superseded by #13948. Current main still lacks action-level allow/deny for gateway actions, but #13948 is the broader open canonical tracker for action-level tool policy across multiplexed tools such as gateway and message. Related #54260 has already been folded into the same action-level policy cleanup path.

## Best Possible Solution

Close #46684 as a duplicate/superseded gateway-specific tracker and keep the remaining product/design work on #13948. The eventual fix should extend the typed tool policy contract, docs, and shared enforcement path to evaluate tool name plus action before execution for multiplexed tools such as `gateway` and `message`, rather than adding a one-off gateway-only workaround.
