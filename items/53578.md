## Summary

Keep open. I did not find strong evidence that current `main` fixes or obsoletes this regression. `google-gemini-cli` still runs the external `gemini` binary for turns (`extensions/google/cli-backend.ts:14`), so the reporter’s observed per-turn Gemini CLI OAuth startup/keychain delay is still compatible with current architecture, and I found no post-2026-03-24 maintainer fix in the inspected file history or changelog that specifically addresses Gemini CLI OAuth turn latency.

## Evidence

- **Gemini CLI backend is still the runtime path:** `google-gemini-cli` is configured as a CLI backend that spawns `gemini` for each run, with `--resume` for existing sessions but still using the external Gemini CLI process.
  - file: [extensions/google/cli-backend.ts:14](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/extensions/google/cli-backend.ts#L14)
- **Current provider code has no Google-specific refresh/latency fix hook:** The Google Gemini CLI provider formats OAuth auth for usage/reporting, but there is no provider-local `refreshOAuth` implementation here that would explain a fixed per-turn auth-delay fix on `main`.
  - file: [extensions/google/gemini-cli-provider.ts:120](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/extensions/google/gemini-cli-provider.ts#L120)
- **CLI auth epoch logic is stable-identity based, but that only preserves resume bindings:** Current `main` hashes auth epochs from stable identity fields, not rotating token material, which helps session reuse but does not remove the cost of invoking the external `gemini` CLI each turn.
  - file: [src/agents/cli-auth-epoch.ts:132](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/agents/cli-auth-epoch.ts#L132)
- **No clear post-issue Gemini latency fix in inspected history:** `git log --oneline --since='2026-03-24' -- extensions/google/cli-backend.ts src/agents/cli-runner/execute.ts src/agents/cli-auth-epoch.ts src/agents/cli-session.ts src/agents/command/session-store.ts src/agents/command/attempt-execution.ts` returned only unrelated commits (`40be5ad5 fix: harden GPT-5 runtime paths`, `0a3da5cd perf: slim auth profile test imports`).
  - command: `git log --oneline --since='2026-03-24' -- extensions/google/cli-backend.ts src/agents/cli-runner/execute.ts src/agents/cli-auth-epoch.ts src/agents/cli-session.ts src/agents/command/session-store.ts src/agents/command/attempt-execution.ts`
- **No changelog entry found for Gemini CLI OAuth turn-speed regression:** Inspected current changelog entries for Gemini CLI auth/session work; found login/discovery fixes and transcript/session fixes, but nothing clearly claiming a fix for consistent per-turn Gemini CLI OAuth slowness.
