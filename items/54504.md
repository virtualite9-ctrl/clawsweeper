## Summary

Keep this issue open. Current main still lacks lightweight label-scoped subagent instruction files, workspace/user precedence, YAML frontmatter handling, and an equivalent documented pre-spawn plugin/API contract for mutating the child prompt, model, thinking, and tool policy from a role file.

## Best Possible Solution

Keep this open for maintainer product/API design. The strongest path is either a narrow core resolver in the native sessions_spawn path for workspace/user-scoped role markdown files with documented precedence and frontmatter schema, or an explicit pre-spawn plugin hook that can mutate the pending child prompt, model/thinking, and tool policy before the child run starts. The implementation should define precedence against explicit spawn params, per-agent config, global defaults, and tools.subagents policy, then add docs and focused tests.
