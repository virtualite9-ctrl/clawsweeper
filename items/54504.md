## Summary

Keep open. Current main supports labeled subagent spawns plus per-spawn model/thinking overrides, full `agents.list` target agents, attachments, and generic prompt hooks, but it does not implement persistent label-scoped markdown role files or frontmatter-driven model/tool policy for `sessions_spawn`. The existing plugin hooks can approximate prompt/model mutation in some agent turns, but they are not a complete substitute for automatic `sessions_spawn(label=...)` profile resolution, especially tool policy.

## Best Possible Solution

Keep this open for maintainer product/API design. The best path is either a core `sessions_spawn` subagent-profile resolver that reads workspace/user-scoped role markdown, parses a narrow frontmatter schema, applies model/tool policy before launch, and documents precedence, or a new explicit plugin/API seam that lets a ClawHub plugin safely provide the same spawn-time profile resolution. Related #51598 and the closed prompt/persona/workspace items are adjacent but do not cover this label-scoped lightweight role-file workflow.
