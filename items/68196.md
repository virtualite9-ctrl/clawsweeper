## Summary

Keep this PR open. Current main still returns a hard `timeout` from `sessions_send` as soon as `agent.wait` times out, while the proposed helper and timeout-compensation path are absent. The PR is paired with open bug #68065 by the same author and uses closing syntax, so the maintainer path is to review/revise/land or explicitly reject the PR, not cleanup-close it.

## Best Possible Solution

Keep the PR open and review it as the implementation candidate for #68065. The best code direction is to make post-timeout handling explicit in `sessions_send`: after a successful `agent` RPC acceptance, re-read the latest assistant snapshot; return `ok` when a new post-baseline reply exists, return `accepted` when the accepted run is still running with no new reply, and preserve hard `timeout` when there is no acceptance proof. Keep the focused regression coverage, including the `timeoutSeconds: 0` guard, and address the accepted-state proof review comment before merge.
