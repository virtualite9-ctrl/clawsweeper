## Summary

Current `main` and the latest shipped release do not restart the whole gateway for `stale-socket`; the health monitor stops and starts only the affected channel/account, so the report no longer matches current behavior.

## Close Comment

Closing from a Codex review because this report does not match current OpenClaw behavior. On current `main`, the stale-socket health monitor logs `health-monitor: restarting` and then calls `channelManager.stopChannel(channelId, accountId)` and `channelManager.startChannel(channelId, accountId)` for that specific channel/account only, not a full gateway-process restart (`src/gateway/channel-health-monitor.ts:156`). That targeted scope is also covered by tests in `src/gateway/channel-health-monitor.test.ts:128` and `src/gateway/channel-health-monitor.test.ts:242`. The latest shipped release `v2026.4.22` contains the same per-channel restart path. Current Discord packaging also declares `@discordjs/voice` in `extensions/discord/package.json:6`, with contract coverage in `src/plugins/contracts/package-manifest.contract.test.ts:7`. If you still see whole-process restarts on a newer build, please open a fresh issue with the current version, supervisor/LaunchAgent logs, and the exact process-restart evidence, because that would be a different failure path from the gateway health monitor itself.
