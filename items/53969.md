## Summary

Keep open. Current main already covers part of the request: auth-store writes are atomic now, Codex OAuth refresh persists rotated credentials, and Codex CLI reuse is hardened to avoid blindly overwriting stored OpenClaw auth. But the issue is not fully implemented: refresh coordination is still keyed only by `(provider, profileId)`, not by logical OAuth enrollment, and some OAuth/profile mutation paths still bypass the locked store updater.
