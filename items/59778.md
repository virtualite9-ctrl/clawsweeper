## Summary

Current `main` already guards heartbeat runs from interrupting an active session, and the same guard is present in the shipped `v2026.4.23` release.

## Evidence

- **Busy session-lane guard on current main:** `runHeartbeatOnce` now checks the resolved session lane before starting a heartbeat and skips with `requests-in-flight` when that lane is busy; the inline comment says this is to avoid interrupting an active streaming turn.
  - file: [src/infra/heartbeat-runner.ts:782](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/src/infra/heartbeat-runner.ts#L782)

  - sha: [ec3dbd22a4de](https://github.com/openclaw/openclaw/commit/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46)
- **Retry path avoids racing the active run:** When a heartbeat is skipped for `requests-in-flight`, the runner does not advance/re-arm immediately and instead lets the wake-layer retry handle it, preventing the heartbeat scheduler from racing the current run.
  - file: [src/infra/heartbeat-runner.ts:1533](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/src/infra/heartbeat-runner.ts#L1533)
