## Summary

Keep open. The PR is not mergeable as written, but the underlying `status --json --all` problem is a real core/plugin-contract question: current main can return `null` before asking a registered memory runtime when the built-in sqlite store is absent. The linked plugin-side runtime PR is useful context, and the OpenClaw-side fix should be plugin-agnostic rather than hard-coding `memory-lancedb-pro`.

## Best Possible Solution

Keep the item open, but ask for a plugin-agnostic rewrite: status should either ask the selected registered memory runtime before built-in sqlite/qmd store preflights or use an explicit generic capability such as `providesOwnStatus`. The console-suppression part should silence or reroute raw console writes during JSON probing, not only subsystem logger output.
