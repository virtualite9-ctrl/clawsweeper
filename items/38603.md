## Summary

Keep open. Current main still lacks the requested Telegram send-adapter `isSilentReplyText` guard, even though Slack has the parity guard and the shared helper exists. Telegram has adjacent higher-level NO_REPLY normalization and there is an open fix attempt in PR #68304, but the requested send-level behavior is not implemented on main.

## Best Possible Solution

Keep #38603 open and review or replace PR #68304 with a focused change in `extensions/telegram/src/send.ts`: import/use `isSilentReplyText`, return a synthetic suppressed result before resolving/sending when `text` is exactly silent and there is no media or inline keyboard, and add `extensions/telegram/src/send.test.ts` coverage proving Telegram API calls are skipped. Track any larger pre-sentinel streaming-buffering behavior separately.
