## Summary

Keep open. Current main still does not implement the requested send-level Telegram NO_REPLY guard, even though nearby Slack code has that defensive check. There is related delivery-layer and agent-runner suppression, and the reporter noted an agent-side workaround, but the specific upstream guard remains missing and is still tracked by open PR #68304.
