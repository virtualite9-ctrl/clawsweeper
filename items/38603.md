## Summary

Keep #38603 open. Current main at 406ae72fd2789c78053f6b94006ec8f488bc1c00 still lacks the requested `sendMessageTelegram` exact silent-token guard, while Slack has the parity guard and Telegram only has higher-level delivery/dispatcher normalization for some reply paths. The open PR #68304 is an exact fix attempt, but it is not merged or shipped main behavior.

## Best Possible Solution

Keep #38603 open and use PR #68304 as the current candidate only after maintainer review, or replace it with a focused Telegram plugin change: import `isSilentReplyText` from `openclaw/plugin-sdk/reply-chunking`, return a synthetic suppressed `TelegramSendResult` before Telegram API work when `text` is exactly silent and there is no media or inline keyboard/buttons, and add `extensions/telegram/src/send.test.ts` regression coverage proving `api.sendMessage` is not called. Track broader pre-sentinel streaming-buffer behavior separately if maintainers want to solve that larger UX case.
