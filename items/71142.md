## Summary

Keep #71142 open. Current main supports Control UI/WebChat attachments, including non-video files, but the requested configurable upload limit is still missing: attachment parsing is hardcoded to 5,000,000 bytes at the chat/agent gateway call sites, and `gateway.controlUi` config has no upload-size field in the public type or zod schema. The related prior image-upload work (#36440) and recent non-image attachment work make the upload surface real, but they do not solve configurability.

## Best Possible Solution

Add an explicit Gateway/Control UI attachment upload limit config, validate it in `src/config/types.gateway.ts` and `src/config/zod-schema.ts`, document it in the public Control UI/config docs, and plumb it through `chat.send`, `agent`, and node-event attachment parsing instead of hardcoded `5_000_000`. The UI should receive/display the effective limit and reject oversize files client-side with a clear message. For values that approach or exceed the current WebSocket payload cap, either clamp with schema/docs guidance or introduce a chunked/HTTP managed-media upload path before allowing very large files.

## Risks / Open Questions

- A naive increase above the gateway WebSocket `MAX_PAYLOAD_BYTES` of 25 MiB could still fail because Control UI sends base64 attachments inside JSON frames; larger limits likely need validation, clamping, or a different upload path.
- Raising the limit should stay aligned with media-store, provider, and media-understanding caps so users do not upload files that are accepted by the UI but unusable downstream.
