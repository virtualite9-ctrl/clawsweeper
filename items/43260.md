## Summary

Current main still does not implement per-skill model routing. Skill metadata parsing, prompt catalog output, slash skill invocation, and plugin model hooks all operate without a skill-level model/preferredModel/modelProfile contract. The related open PR #58142 is an implementation candidate, not shipped behavior.

## Best Possible Solution

Keep this issue open as the canonical product request. The best path is to review and land, revise, or replace #58142 with a maintainer-approved implementation that adds a documented skill metadata contract, parses and persists preferred model/fallback fields, applies them at the intended skill invocation/read boundary with clear precedence, reuses the existing model allowlist/alias resolver, and adds focused tests for frontmatter parsing, slash skill invocation, prompt/read routing, fallbacks, and plugin-hook interaction.
