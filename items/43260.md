## Summary

Keep this issue open. Current main still has no `model`, `preferredModel`, fallback, or profile field in skill frontmatter/metadata, and the implemented adjacent routing paths are run-level (`before_model_resolve`), agent/subagent-level, or cron payload-level rather than skill-invocation-level. The discussion also adds concrete InvestorClaw production evidence, and related PR #58142 remains a candidate implementation rather than a reason to close the feature request before maintainer review or merge.

## Best Possible Solution

Keep the issue open as the canonical product request. The best path is to review/land or replace PR #58142 with a maintainer-approved implementation that adds a first-class skill metadata contract, parses and persists model/fallback fields, enforces them when a skill is invoked or read, documents precedence and failure behavior, and adds focused tests across frontmatter parsing, slash skill invocation, prompt/read routing, and existing model-routing hooks.
