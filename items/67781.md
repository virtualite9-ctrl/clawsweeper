## Summary

Keep this PR open, but it needs revision. Current main has some google-vertex 1M-context model matching coverage, but it still has the google-vertex ADC marker path and still silently returns no final reply for empty payload lists. The submitted patch is useful directionally, but as written it would override the existing Google thinking stream wrapper and would turn several intentionally empty successful reply paths into user-visible failures.

## Best Possible Solution

Revise this PR rather than close it. The google-vertex fix should compose a narrow ADC marker-stripping wrapper with the existing Google thinking wrapper, preferably through composeProviderStreamWrappers or an equivalent provider-owned helper, and add regression tests for ADC marker stripping, real GOOGLE_CLOUD_API_KEY passthrough, and non-Vertex Google thinking behavior. The empty-payload fix should only surface a friendly retry/context error when a non-silent run truly produced no delivered output and no renderable final payload, while preserving block streaming, direct block-key dedupe, same-target messaging-tool suppression, and silentExpected turns.
