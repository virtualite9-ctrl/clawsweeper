## Close Comment

Closing from a Codex review because current `main` already implements this behavior. The Control UI chat composer now handles pasted images, drag-and-drop files, and pre-send image previews/removal in `ui/src/ui/views/chat.ts:206`, `ui/src/ui/views/chat.ts:272`, and `ui/src/ui/views/chat.ts:302`, and those handlers are wired into the WebChat UI at `ui/src/ui/views/chat.ts:1068` and `ui/src/ui/views/chat.ts:1237`. The frontend also serializes the selected images into `chat.send` attachments in `ui/src/ui/controllers/chat.ts:213` and `ui/src/ui/controllers/chat.ts:236`. This was already shipped in `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`), and the release notes mention persisted Control UI user image uploads in `CHANGELOG.md:549`. If you still hit an image-upload regression on a newer build, please open a fresh issue with the exact OpenClaw version, browser, and a current repro.
