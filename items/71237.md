## Summary

Keep PR #71237 open. Current main does not contain `pruneOrphanedTranscripts` or an equivalent reusable orphan-transcript prune primitive. Existing main covers related but narrower paths: stale/capped store entries archive their known transcript files, disk-budget cleanup removes unreferenced primary `.jsonl` files only when a configured disk budget is exceeded, and doctor can archive detected orphans. That does not fully supersede this recent PR’s proposed opt-in direct unlink utility and tests.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to either land a carefully reviewed orphan-transcript primitive or fold it into `openclaw sessions cleanup` with explicit dry-run/enforce behavior, bounded concurrency, a union of preserved paths across stores, checkpoint snapshot preservation, custom `sessionFile` handling, and tests proving referenced transcripts are never deleted.
