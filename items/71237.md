## Summary

Keep this PR open. Current main has related cleanup for stale/capped entries, missing-entry pruning, doctor archival, and disk-budget pressure cleanup, but it still does not include the proposed `pruneOrphanedTranscripts` primitive or an equivalent general on-demand sweep that scans a sessions directory and unlinks already-unindexed `.jsonl` transcripts. The PR is recent, concrete, and has active review follow-up rather than being obsolete cleanup material.

## Best Possible Solution

Keep the PR open for maintainer review. The best product path is either to merge the well-tested primitive and follow it with explicit `openclaw sessions cleanup` wiring, or supersede it with a narrower PR that implements the full cleanup command path directly while collecting preserved paths from every relevant store, transcript candidate, topic transcript, and compaction checkpoint before unlinking anything.
