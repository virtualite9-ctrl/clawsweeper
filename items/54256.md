## Summary

Current `main` already implements a stable/volatile system-prompt split for prompt caching, and the system prompt runtime block no longer carries per-turn usage/cache/context stats. The shipped docs and changelog indicate this prompt-cache-boundary work was released in `v2026.4.5`.

## Evidence

- **System prompt now has an explicit cache boundary with dynamic content below it:** `src/agents/system-prompt.ts:913-971` sorts context files into stable vs dynamic groups, inserts `SYSTEM_PROMPT_CACHE_BOUNDARY`, and places dynamic project context, extra system prompt, heartbeat, and runtime after that seam.
  - file: [src/agents/system-prompt.ts:913](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/agents/system-prompt.ts#L913)
  - command: `nl -ba src/agents/system-prompt.ts | sed -n '820,980p'`
- **Runtime section is stable and does not include per-turn usage/cache/context lines:** `src/agents/system-prompt.ts:976-1017` shows `buildRuntimeLine(...)` only emits `agent`, `host`, `repo`, `os`, `arch`, `node`, `model`, `default_model`, `shell`, `channel`, `capabilities`, and `thinking`. There is no `usageLine`, `cacheLine`, or `contextLine` in the prompt builder.
  - file: [src/agents/system-prompt.ts:976](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/agents/system-prompt.ts#L976)
  - command: `nl -ba src/agents/system-prompt.ts | sed -n '976,1080p'`

  - file: [src/status/status-message.ts:874](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/status/status-message.ts#L874)

  - file: [src/agents/anthropic-payload-policy.test.ts:164](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/agents/anthropic-payload-policy.test.ts#L164)

## Close Comment

Closing as implemented on `main` after Codex review. Current prompt assembly already splits a cache-stable prefix from a dynamic suffix in `src/agents/system-prompt.ts:913-971`, and the runtime block at `src/agents/system-prompt.ts:976-1017` no longer includes per-turn `usageLine` / `cacheLine` / `contextLine` style stats. Anthropic boundary-aware shaping is also covered by `src/agents/anthropic-payload-policy.test.ts:164-180`, and the shipped prompt-caching notes in `CHANGELOG.md:1406-1408` place this work in `v2026.4.5`. If you still see cache busting on a current build, it looks like a different remaining invalidator than the one described here, so a fresh issue with a cache trace from a current version would be the right follow-up.
