## Summary

Issue #60280 is already implemented on current `main` and in release `v2026.4.23`: the bundled Ollama plugin now registers an image-capable media-understanding provider, the image path resolves providers from that registry instead of hardwiring Anthropic, tests cover `ollama/*` image routing, and the public Ollama docs now document `agents.defaults.imageModel.primary = "ollama/qwen2.5vl:7b"`.

## Evidence
