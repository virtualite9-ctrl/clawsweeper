## Summary

Current main still does not implement strict on-demand HEARTBEAT.md loading. The bootstrap loader still includes and reads HEARTBEAT.md in the standard workspace file set, and the resolver only drops it for normal runs when heartbeat guidance is disabled, not for every non-heartbeat run.

## Best Possible Solution

Keep #13968 open. Implement an isHeartbeat/run-kind-aware bootstrap loading path that omits HEARTBEAT.md for default non-heartbeat runs before disk reads, cache population/reuse, hook processing, and prompt injection; preserve HEARTBEAT.md for actual heartbeat runs and lightContext behavior; add focused tests for default enabled omission, heartbeat retention, disabled cadence/prompt-section behavior, cache behavior, and lightContext forwarding; then update heartbeat/system-prompt/token-use docs.
