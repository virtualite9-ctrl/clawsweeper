## Summary

Keep #13968 open. Current main still does not implement strict on-demand HEARTBEAT.md loading: normal default runs still read/load HEARTBEAT.md, and they still inject it when default-agent heartbeat guidance is enabled. The related closed #55311/lightContext path is adjacent but only covers lightweight heartbeat forwarding, not omission from non-heartbeat turns.

## Best Possible Solution

Implement an explicit run-kind/isHeartbeat-aware bootstrap loading path so default non-heartbeat runs omit HEARTBEAT.md before disk read, cache reuse, hook processing, and prompt injection, while actual heartbeat runs keep HEARTBEAT.md and lightweight heartbeat mode remains heartbeat-only. Add focused tests for default omission with enabled heartbeat guidance, heartbeat retention, disabled cadence/prompt-section behavior, and lightContext forwarding, then update token-use/system-prompt/heartbeat docs to describe the new behavior.
