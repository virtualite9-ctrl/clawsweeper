## Summary

Keep this PR open. Current main still resolves exec approvals file and socket paths from the home-relative `~/.openclaw` defaults, while `OPENCLAW_STATE_DIR` is the documented mutable state root override used elsewhere. The related open PR #62938 covers the same remaining bug, but it is not merged or clearly canonical yet, so this PR remains a valid implementation candidate until maintainers choose one path.

## Best Possible Solution

Keep this PR open while maintainers review the state-dir bug and choose a canonical implementation, likely coordinated with #62938. The final patch should make exec approvals file and socket resolution respect an explicit `OPENCLAW_STATE_DIR` through the existing state-dir resolver, preserve current default and legacy behavior when no override is set, update effective-policy reporting for the overridden path, and add focused regression tests for resolver and store writes.
