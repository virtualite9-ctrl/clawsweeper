## Summary

Keep PR #39269 open. Current main overlaps the provider/Codex JSON error sanitizer work, but several central fixes from the batch are still not fully implemented on main, especially shared outbound Discord Unknown Channel/10003 permanent-error handling and display-side filtering of delivery-mirror transcript rows from chat.history. The PR is still a rework/split candidate rather than an obsolete cleanup close.

## Best Possible Solution

Keep #39269 open, but do not merge the batch as-is. Split or reimplement the still-valid pieces: add shared outbound Discord Unknown Channel/code 10003 permanent-error classification with policy tests; hide provider=openclaw/model=delivery-mirror rows in the shared chat display projection before limit/budget semantics; add a real webchat-plus-stale-external-route regression around resolveAnnounceOrigin and completionDirectOrigin; and make duplicate plugin warning precedence an explicit maintainer policy decision. Leave the already-covered Codex/provider API error sanitizer on main.
