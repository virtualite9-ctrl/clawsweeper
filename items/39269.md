## Summary

Keep PR #39269 open. Current main overlaps part of the batch, especially the Codex-prefixed provider-error sanitizer, but several requested behaviors are still missing or intentionally different on main: shared outbound delivery still does not classify Discord Unknown Channel/10003 as permanent, chat.history still has no delivery-mirror display filter, webchat announce origin can still be replaced by a stale external session route, and duplicate plugin IDs from distinct origins still warn. The PR also has unresolved review concerns, so it should be split or reworked rather than merged as-is.

## Best Possible Solution

Keep #39269 open as useful follow-up signal, but do not merge it as-is. Split or reimplement the still-valid pieces: add shared outbound Unknown Channel/10003 permanent-error classification with policy tests; filter delivery-mirror messages in the shared chat display projection before history limiting; fix webchat announce origin handling without suppressing explicit completionDirectOrigin routes; and make the duplicate-plugin warning policy an explicit maintainer decision rather than silently changing diagnostics in a batch PR. Leave the already-covered Codex error sanitizer as-is on main.
