## Summary

Keep #7722 open. Current main only has the narrower `tools.fs.workspaceOnly` guard; it does not accept `tools.fileAccess`, `mode`, `allowedPaths`, or `denyPaths`, and `exec` still has no filesystem allow/deny enforcement. The issue discussion and same-day maintainer/Codex comment explicitly preserve this broader scope, with #5948 and PR #60981 only partial/contextual.

## Best Possible Solution

Keep #7722 open until OpenClaw either implements a versioned filesystem access policy surface, likely under `tools.fs` or a successor config contract, with deny-wins allow/deny path rules enforced across read/write/edit/apply_patch/local media and a defensible exec strategy, or a maintainer explicitly narrows/rejects that product direction. Treat `tools.fs.workspaceOnly`, sandbox backends, #5948, #8719, and PR #60981 as related partial/contextual work rather than completion of this request.
