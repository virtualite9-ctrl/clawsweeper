## Summary

Keep #23926 open. Current main has partial `before_install` coverage for gateway-managed skill dependency installs, but it still does not provide the requested skill install lifecycle: no `skill:pre-install` / `skill:post-install` events or equivalent post-install observation hook, and native ClawHub skill install/update paths used by CLI, Gateway, and UI still bypass the skill scan/hook helper.

## Best Possible Solution

Keep #23926 open as the concrete tracker for skill install lifecycle hook/API work. The best product path is to extend the current `before_install` contract or add an equivalent documented install lifecycle API so native OpenClaw ClawHub skill install/update/sync/UI/gateway paths run a blocking policy point before copying files, then add a documented post-install observational hook or record an explicit maintainer decision that post-install observation belongs elsewhere. Tests should cover gateway RPC, CLI ClawHub install/update, blocking behavior, findings/warnings, and post-install event semantics.
