## Summary

Keep #23926 open. Current main has a documented and tested `before_install` hook for gateway-managed skill dependency installers, but native ClawHub skill install/update and the web UI ClawHub install path still copy downloaded skill folders without that policy hook, and there is still no post-install observational lifecycle hook.

## Best Possible Solution

Keep #23926 open as the concrete tracker for skill install lifecycle hook/API work. Extend the documented `before_install` API or add equivalent skill lifecycle hooks so native ClawHub skill install/update/gateway/UI paths run a blocking policy point before copying files, then add a documented post-install observational hook carrying installedAt, targetDir, source, success/failure, and audit metadata. Separately clarify whether the standalone `clawhub` CLI can integrate with that API or needs ClawHub-owned policy support.
