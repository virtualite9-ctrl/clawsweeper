## Summary

Keep open. Current main still has only partial `before_install` coverage for the gateway-backed skill dependency installer, while the requested skill install lifecycle remains incomplete: no `skill:pre-install` / `skill:post-install` events, no post-install observational hook, and native OpenClaw ClawHub skill install/update paths still copy downloaded skill folders without the scan/hook helper. The existing maintainer comment on #23926 already reached the same conclusion, and current `main` still matches it.

## Best Possible Solution

Keep #23926 open as the concrete tracker for skill install lifecycle hook/API work. The best product path is to extend the current `before_install` contract or add an equivalent documented install lifecycle API so native OpenClaw ClawHub skill install/update/sync/UI/gateway paths run a blocking policy point before copying files, then add a documented post-install observational hook or record an explicit maintainer decision that post-install observation belongs elsewhere. Tests should cover gateway RPC, CLI ClawHub install/update, blocking behavior, findings/warnings, and any post-install event semantics.
