## Summary

Keep open. Current main still only has a partial plugin-level `before_install` hook for installer/security-scan paths. The requested `skill:pre-install` / `skill:post-install` lifecycle is not implemented, there is no post-install observational hook, and native ClawHub skill install/update still copies downloaded skill folders without routing through the `scanSkillInstallSource` / `runBeforeInstallHook` path. The prior Codex review comment remains accurate on current `main`; #50090 is broader ecosystem tracking, not a canonical replacement for this concrete API request.

## Best Possible Solution

Keep #23926 open as the concrete tracker for skill install lifecycle hook/API work. The best product path is to either extend the current `before_install` contract to cover native ClawHub skill install/update/sync and add an observational post-install hook, or record an explicit maintainer decision that marketplace scanning should stay in ClawHub and document the remaining OpenClaw-side boundary.
