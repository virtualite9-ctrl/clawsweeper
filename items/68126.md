## Summary

Keep open. Current main still has the reported Discord DM inbound context bug: the Discord reply plan creates a `channel:<dmChannelId>` reply target, DMs do not get an auto-thread context, and `processDiscordMessage` assigns `ctx.To` from that channel-shaped target before computing the `user:<authorId>` DM conversation target. The related fix PR was closed unmerged in the provided GitHub context, so this issue still tracks remaining work.

## Best Possible Solution

Land a narrow Discord plugin fix in `extensions/discord/src/monitor/message-handler.process.ts` that computes the DM conversation target before `effectiveTo` and uses it for direct-message `ctx.To`, while keeping `deliverTarget` channel-based for Discord API delivery. Add a regression test that exposes and asserts `ctx.To`, `ctx.From`, and `ctx.ChatType` for inbound Discord DMs, and keep component-interaction DM behavior as a separate decision if needed.
