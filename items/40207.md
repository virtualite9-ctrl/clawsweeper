## Summary

Current main still leaves the reported LaunchAgent failure path intact: CLI-managed macOS gateway plists derive StandardOutPath/StandardErrorPath from the OpenClaw state directory, defaulting under ~/.openclaw/logs, and the generated plist writes those paths verbatim. The issue has concrete macOS launchd exit-78 confirmations and remains actionable in core daemon/LaunchAgent generation.

## Best Possible Solution

Keep this issue open. The fix likely belongs in the CLI macOS LaunchAgent generation path: write StandardOutPath/StandardErrorPath to a launchd-safe non-symlinked user log location such as ~/Library/Logs/OpenClaw, or resolve the generated log directory through realpath before writing the plist. Add regression coverage for a symlinked state directory and plist regeneration, update runtime hints/status/docs to match the chosen log path, and consider adding an install-time health check that surfaces launchd EX_CONFIG failures clearly.
