## Summary

Keep this open. Current main still generates the CLI-managed macOS Gateway LaunchAgent stdout/stderr paths from the OpenClaw state directory under `.openclaw/logs`, writes those paths directly into the plist, and install success only rechecks launchd loaded state rather than RPC health. The issue has concrete macOS confirmations and remains an actionable core CLI/LaunchAgent bug.

## Best Possible Solution

Keep this issue open and fix the CLI-managed macOS LaunchAgent path in `src/daemon`: write `StandardOutPath` and `StandardErrorPath` to a launchd-safe non-symlinked user log location such as `~/Library/Logs/OpenClaw` or resolve the generated log directory with `realpath` before writing the plist. Add regression coverage for symlinked state directories and plist regeneration, update runtime hints/status/docs, and consider an install-time health check that reports launchd EX_CONFIG failures clearly.
