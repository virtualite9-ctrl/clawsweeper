## Summary

Keep #11301 open. Current main still does not add a runtime-container `/workspace` alias for Docker or Podman gateway/CLI containers. The only first-party `/workspace` behavior remains sandbox-specific, while Docker Compose, Docker setup helpers, Podman launch, and Quadlet all continue to mount the workspace at `/home/node/.openclaw/workspace`. The related PR #12764 was closed unmerged, so it is useful prior art rather than a shipped or superseding fix.

## Best Possible Solution

Keep #11301 open. Maintainers should either implement a safe runtime compatibility alias or second bind mount for `/workspace -> /home/node/.openclaw/workspace` across Dockerfile, Compose, Docker setup, Podman/Quadlet, and relevant smoke/docs coverage, or explicitly decide that `/workspace` is sandbox-only and update workspace/runtime docs to remove the ambiguity. Treat PR #12764 as useful prior art, not a canonical fix.
