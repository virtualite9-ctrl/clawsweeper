## Summary

Keep #11301 open. Current main e29d3516bf05591e6007bfb8ccdaea6d76194d41 still does not create a runtime-container `/workspace -> /home/node/.openclaw/workspace` symlink or bind mount for Docker/Podman users. The existing `/workspace` behavior is scoped to agent sandbox containers, and the directly related PR #12764 was closed unmerged.

## Best Possible Solution

Keep #11301 open until a maintainer either implements a safe Docker/Podman runtime compatibility alias or bind mount for `/workspace -> /home/node/.openclaw/workspace` and aligns docs/smokes, or explicitly decides `/workspace` is sandbox-only and updates Docker/runtime documentation plus issue expectations. Treat PR #12764 as useful prior art, not a superseding fix, because it is closed unmerged.
