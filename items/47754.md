## Summary

Keep open. Current `main` still builds keyword-search snippets by selecting full chunk text and truncating from the start, so the reported bug is not implemented on `main`. The PR's original file path is stale, but the underlying logic now lives in `extensions/memory-core/src/memory/manager-search.ts` and still needs the same fix.
