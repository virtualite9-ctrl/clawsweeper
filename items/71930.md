## Summary

Current main still drops Mattermost `post_edited` WebSocket events before they can reach Mattermost mention detection and agent wake routing. This is a bundled Mattermost plugin behavior bug, not a ClawHub cleanup item or a duplicate of the related WebSocket keepalive/rawBody/plugin reports.

## Best Possible Solution

Keep #71930 open and fix it in `extensions/mattermost`: accept Mattermost `post_edited` events carrying a post payload, route qualifying edits through the existing mention gate and enqueue path, make replay dedupe edit-aware using edit/update metadata or a dedicated edit key, and add focused `monitor-websocket` plus mention/replay regression tests.

## Risks / Open Questions

- A naive implementation that only lets `post_edited` enter the existing posted path may still drop mention edits because the current replay guard commits the original post id before mention gating completes.
- Maintainers should define duplicate-wake semantics for edits: wake when an edit newly adds a mention, but avoid re-waking on cosmetic edits to an already-mentioned post.
