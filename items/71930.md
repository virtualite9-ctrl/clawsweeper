## Summary

Keep open. Current main still drops Mattermost `post_edited` WebSocket events before they can reach the existing Mattermost mention gate and wake/session routing path. The request is for bundled Mattermost plugin behavior in this repo, not optional ClawHub work, and no related context shows a canonical duplicate or superseding fix.

## Best Possible Solution

Fix this in the bundled Mattermost plugin by accepting `post_edited` in the websocket parser/dispatcher, parsing the updated post body through the same post handling and mention-gating path where appropriate, and adding regression tests for edit-added mentions plus non-mention edits that must not spuriously wake the agent.
