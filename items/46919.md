## Summary

Keep #46919 open. Current main has partial adjacent behavior, but not the requested core capability: plaintext dashboard tokens are still embedded in browser/clipboard URL fragments, `gateway.auth.mode: "none"` is constrained to loopback/private-ingress patterns, Tailscale tokenless auth is limited to verified Serve Control UI/WebSocket auth, and HTTP APIs, node-role connections, direct tailnet binds, and device-less browsers still follow normal auth/device checks.

## Best Possible Solution

Keep the issue open for maintainer security/product judgment. If accepted, implement an explicit, clearly dangerous core gateway policy with narrow scope, defaults off, updated runtime auth checks, dashboard URL generation behavior, Control UI/node/HTTP tests, and public docs; otherwise close later with a maintainer-authored policy decision explaining why OpenClaw will keep tokens/device checks for direct tailnet and HTTP API access.
