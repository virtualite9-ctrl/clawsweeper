## Summary

Keep this issue open. Current main still derives the Mattermost interaction signing secret from the bot token using the public hardcoded HMAC key, and the same-author PR that references this issue with closing syntax is still open/unmerged in the provided GitHub context, so the issue should remain the tracker until a reviewed fix lands.

## Best Possible Solution

Keep this issue open as the Mattermost security tracker. Review and land PR #64546 or a successor that adds a persistent, SecretRef-aware per-account interaction signing secret separate from the bot token, updates schema/types/docs/tests, and closes this only after the fix is merged and shipped.
