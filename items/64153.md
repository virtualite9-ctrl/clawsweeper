## Summary

Current `main` no longer hardcodes Edge inside the agent `tts` tool. The tool now delegates to the shared speech-core TTS runtime, which resolves `messages.tts` provider config and passes `providers.microsoft.voice` (with legacy `edge` aliases normalized) into the Microsoft provider. A regression test on `main` asserts that a configured voice like `en-US-AvaNeural` is preserved. The best proving commit is `51dd4f288f67ef4df4c7a121f078f508535df1fd`; it does not appear to be in tagged release `v2026.4.23`, so the fix is on `main` but not mapped to a shipped release here.

## Evidence

- **Agent TTS tool uses shared config-aware runtime:** `createTtsTool()` loads config and calls `textToSpeech({ text, cfg, channel, timeoutMs })`, so the tool path now goes through the shared TTS runtime instead of a direct Edge-only call.
