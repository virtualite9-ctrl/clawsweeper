## Summary

Keep open. Current main still does not implement an agent-scoped `cron` field: agent config types and strict config schema omit it, Gateway cron still derives only top-level cron settings and the persisted jobs store, and docs continue to describe `~/.openclaw/cron/jobs.json` plus CLI/RPC management with optional `agentId` binding. The related ClawSweeper context is adjacent cron/agent work, not a canonical duplicate or shipped implementation.

## Best Possible Solution

Keep the issue open for maintainer product/API design. A core implementation would add a typed `agents.list[].cron` schema, generated config docs/help, and Gateway startup reconciliation that materializes or overlays agent-owned jobs with stable namespacing, clear precedence against `cron/jobs.json`, and tests for schema validation, scheduler merge behavior, duplicate IDs, disabled jobs, and agent model/workspace inheritance. If maintainers decide this should live outside core, the better path is an explicit plugin/API design for declarative cron reconciliation, because the requested `agents.list[]` field is currently not expressible by external plugins.
