## Summary

Keep this PR open. Current main still does not expose `ReplyToId` or `ReplyToBody` to plugin `inbound_claim` handlers, while those fields already exist upstream and are already used in the agent-facing inbound metadata path. The PR is a narrow additive implementation candidate, not obsolete, duplicated, or ClawHub-only.

## Best Possible Solution

Keep this PR open and review it as a small plugin hook API fix. The best path is to add optional `replyToId` and `replyToBody` to the canonical inbound hook context, map them from `FinalizedMsgContext`, expose them in `inbound_claim` metadata, and add a focused regression test in `src/hooks/message-hook-mappers.test.ts`.
