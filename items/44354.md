## Summary

Keep open. Current main still has the reported Discord health/status divergence at the source level: `openclaw health --json` builds a reduced channel snapshot without gateway runtime or credential-source fields, while `channels.status` builds account snapshots from the live gateway runtime before calling the same plugin summary hook. For Discord, the token summary defaults missing `running` to `false` and missing `tokenSource` to `none`, matching the issue symptom.

## Best Possible Solution

Keep this issue open and fix the health RPC path so channel summaries are built from the same gateway runtime-aware account snapshots used by `channels.status`, or otherwise pass the runtime snapshot into `getHealthSnapshot`. Add a regression test for Discord where an active runtime account with a config token reports matching `running` and `tokenSource` in `health --json` and `channels.status --probe --json`.
