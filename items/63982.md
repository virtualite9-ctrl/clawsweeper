## Summary

Current `main` resolves each agent's workspace from `agents.list[].workspace` before any default fallback, and the system prompt displays the resolved workspace for the active agent. The same implementation is present in release `v2026.4.23`.

## Evidence

- **Per-agent workspace resolution on main:** `resolveAgentWorkspaceDir()` first checks the target agent's own `workspace` field and returns it directly; only missing per-agent values fall back to `agents.defaults.workspace` or derived paths.
