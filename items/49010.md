## Summary

Keep #49010 open. Current main still sends and receives plain JSON-RPC over the OpenClaw stdio MCP transport, and the documented/configured MCP server surface has no opt-in MCPS/signing fields or transport-wrapper extension point. The reporter's linked razashariff/mcps-openclaw package is useful prior art, but this is security/core transport hardening or a plugin API design question, not safely closeable as implemented or ClawHub-only.

## Best Possible Solution

Keep the issue open for maintainer product/security judgment. The likely path is either a generic, opt-in signing/envelope layer in the MCP stdio transport/config with tests and docs, or a documented plugin/runtime hook that lets an external package such as razashariff/mcps-openclaw wrap OpenClaw-managed MCP transports without private core changes.
