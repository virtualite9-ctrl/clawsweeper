## Summary

Current main still contains the reported Discord configured ACP session-key collision path. A Discord thread that inherits a parent-channel binding is still materialized with the parent conversation id, and configured ACP persistent keys still hash only that materialized conversation id. The provided GitHub context also shows same-author PR #64322 is open and paired with this bug, so this issue should remain open until that implementation candidate is reviewed, merged, or replaced.

## Best Possible Solution

Keep this issue open and review, land, or replace PR #64322. The implementation should make inherited Discord parent-channel ACP bindings produce distinct persistent configured ACP session keys per thread, add regression coverage for two thread ids under one parent resolving to different target session keys, and preserve direct Discord thread bindings plus non-Discord canonical binding behavior.
