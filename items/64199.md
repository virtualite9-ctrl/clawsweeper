## Summary

Current main still has the reported configured ACP session-key collision path for Discord threads inheriting a parent-channel binding, and the same-author implementation PR #64322 remains open/unmerged. This issue should stay open until that PR is reviewed, landed, replaced, or closed.

## Best Possible Solution

Keep this issue open and review, rebase, land, or replace #64322. The fix should make inherited Discord parent-channel ACP bindings materialize distinct persistent configured ACP session keys per runtime thread, preserve direct parent-channel and direct thread bindings, avoid regressing non-Discord channel matching contracts, and add regression coverage for two Discord thread ids under one configured parent resolving to different target session keys.
