## Summary

Keep this PR open. Current main still appears to authorize `memory_get` reads with lexical path containment plus final-path `lstat`, which blocks direct symlink files but does not canonicalize the target and allowed root to catch an intermediate directory symlink escape. The PR is focused on that memory read boundary, and the provided discussion shows review feedback was addressed rather than making the work obsolete.

## Best Possible Solution

Keep this PR open and review or land it, or port the same canonical realpath containment approach into current main. The best implementation should cover builtin memory reads, extra memory paths, and QMD `qmd/<collection>/...` reads; preserve transient missing-file behavior; and keep symlink-permission-safe regression tests for direct and intermediate symlink cases.

## Evidence

- **Builtin memory_get uses lexical workspace containment:** `readMemoryFile` resolves the requested path under the workspace, derives a lexical relative path, and treats `memory/*` as allowed before any canonical containment check. It then reads the same `absPath`. With an intermediate symlink directory, `fs.lstat` on the final markdown file can see the outside target as a regular file.
  - file: [src/memory-host-sdk/host/read-file.ts:27](https://github.com/openclaw/openclaw/blob/7d9dc8cf24fc143bec95318491e0247c37863e63/src/memory-host-sdk/host/read-file.ts#L27)
