## Summary

Keep open. Current main still lacks abort-aware command-lane queue removal: `enqueueCommandInLane` only supports wait warnings, and `runEmbeddedPiAgent` checks abort only after queued work is dequeued. This PR is a focused implementation candidate for the still-open cron bug #43141, not stale cleanup material.

## Best Possible Solution

Keep this PR open and either review/land it after addressing the minor review comments and running the targeted command-queue/cron runner tests, or land an equivalent fix that adds queued-entry abort removal to `enqueueCommandInLane` and threads the cron abort signal through the default embedded runner enqueue wrappers.
