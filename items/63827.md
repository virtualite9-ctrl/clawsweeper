## Summary

Keep open. Current main still does not preselect the existing channel/account `dmPolicy` in the DM policy wizard prompt, so the narrower preservation behavior proposed by PR #63827 is not implemented. The linked issue #63809 was closed as not reproducible for the broader claim that `openclaw update` silently runs configure and rewrites access policy, but that does not supersede the remaining explicit configure-wizard default problem. The PR is not merge-ready as submitted because the provided review comments identify a compile-time scoping bug and the branch bundles many unrelated changes.

## Best Possible Solution

Keep the item open for maintainer action, but replace or heavily narrow the PR. The best fix is a small current-main patch in `src/flows/channel-setup.prompts.ts`: compute `current = policy.getCurrent(cfg, accountId)` before the `prompter.select` call, pass `initialValue: current`, keep the existing compare/write behavior, and add a focused regression test for preserving an existing `allowlist` or `disabled` policy when the wizard accepts the default. The broad, unrelated PR branch should be superseded by that focused change rather than landed directly.
