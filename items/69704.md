## Summary

Keep PR #69704 open. Current main already sends Brave API keys with `X-Subscription-Token` in both web and `llm-context` modes, and that behavior is present in `v2026.4.24`, but current main does not include the PR's requested regression tests. The PR is a focused test-only follow-up to closed bug #69673 and still adds useful missing coverage rather than being redundant.

## Best Possible Solution

Keep this PR open for maintainer review or merge after normal validation. The production Brave auth behavior is already fixed on main, but the best codebase outcome is to add focused regression coverage in `extensions/brave/src/brave-web-search-provider.test.ts` so future refactors cannot move the API key back into URL query parameters.
