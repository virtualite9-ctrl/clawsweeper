## Summary

Current `main` already treats Feishu `@all` as a valid mention for bot triggering, and multi-account Feishu monitors register one inbound handler per enabled account, so the practical behavior requested in #49203 is already implemented.

## Evidence

- **`@all` counts as a bot mention:** `checkBotMentioned()` returns `true` when message content contains `@_all`, before checking explicit bot IDs. That means Feishu `@all` satisfies mention detection for each bot account.
  - file: [extensions/feishu/src/bot-content.ts:229](https://github.com/openclaw/openclaw/blob/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4/extensions/feishu/src/bot-content.ts#L229)

  - sha: [f3c37a946cc4](https://github.com/openclaw/openclaw/commit/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4)
- **Mention gating honors that detection:** Group message handling resolves `requireMention`, then drops only when `requireMention && !ctx.mentionedBot`. Because `@all` sets `mentionedBot`, those messages pass the gate.
  - file: [extensions/feishu/src/bot.ts:466](https://github.com/openclaw/openclaw/blob/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4/extensions/feishu/src/bot.ts#L466)

  - file: [extensions/feishu/src/monitor.ts:56](https://github.com/openclaw/openclaw/blob/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4/extensions/feishu/src/monitor.ts#L56)

## Close Comment

Closing after Codex review: current `main` already implements the requested behavior. In `extensions/feishu/src/bot-content.ts:229`, Feishu `@all` (`@_all`) is treated as a valid mention, and `extensions/feishu/src/bot.ts:466` only suppresses group replies when `requireMention` is enabled and the bot was not mentioned. Multi-account Feishu setups are also handled per account in `extensions/feishu/src/monitor.ts:56` and `extensions/feishu/src/monitor.account.ts:328`, so enabled bots in the group each receive and evaluate the event. This behavior is also present in the shipped `v2026.4.22` release (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`).
