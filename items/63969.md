## Summary

Current `main` canonicalizes bundled plugin roots from `dist-runtime` symlinks to their real `dist` roots before skill containment checks run, so the reported false skill-path escape warning is no longer expected on `main` even though the latest tagged release predates that change.

## Evidence

- **Bundled plugin discovery now realpaths the plugin root before storing `candidate.rootDir`:** `addCandidate()` resolves `params.rootDir` through `safeRealpathSync(...)` and stores that canonical path on the discovered candidate. A plugin found under `dist-runtime/extensions/...` therefore carries the real `dist/extensions/...` root into later checks instead of the symlinked wrapper path.
  - file: [src/plugins/discovery.ts:481](https://github.com/openclaw/openclaw/blob/51dd4f288f67ef4df4c7a121f078f508535df1fd/src/plugins/discovery.ts#L481)
  - sha: [51dd4f288f67](https://github.com/openclaw/openclaw/commit/51dd4f288f67ef4df4c7a121f078f508535df1fd)
- **Plugin skill directories are resolved against the canonical manifest root:** `resolvePluginSkillDirs()` joins each declared skill path against `record.rootDir` from the manifest registry and only rejects paths whose realpath truly escapes that root. With the canonicalized root from discovery, staged `dist-runtime` skill symlinks no longer look like escapes.
  - file: [src/agents/skills/plugin-skills.ts:105](https://github.com/openclaw/openclaw/blob/51dd4f288f67ef4df4c7a121f078f508535df1fd/src/agents/skills/plugin-skills.ts#L105)
  - sha: [51dd4f288f67](https://github.com/openclaw/openclaw/commit/51dd4f288f67ef4df4c7a121f078f508535df1fd)
- **Skill loader still enforces containment, but now against the resolved root:** `loadWorkspaceSkillEntries()` realpaths the configured skills root and checks each candidate skill dir / `SKILL.md` with `resolveContainedSkillPath()`. This keeps the boundary guard in place while avoiding the false-positive caused by a symlinked runtime wrapper root.
  - file: [src/agents/skills/workspace.ts:380](https://github.com/openclaw/openclaw/blob/51dd4f288f67ef4df4c7a121f078f508535df1fd/src/agents/skills/workspace.ts#L380)
  - sha: [51dd4f288f67](https://github.com/openclaw/openclaw/commit/51dd4f288f67ef4df4c7a121f078f508535df1fd)
- **Regression test covers bundled discovery from `dist-runtime`:** `stage-bundled-plugin-runtime.test.ts` stages a runtime wrapper tree, points `OPENCLAW_BUNDLED_PLUGINS_DIR` at `dist-runtime/extensions`, runs discovery + manifest registry loading, and asserts the discovered runtime sources resolve back to the real built paths. This is the same symlinked runtime topology described in the issue.
  - file: [src/plugins/stage-bundled-plugin-runtime.test.ts:387](https://github.com/openclaw/openclaw/blob/51dd4f288f67ef4df4c7a121f078f508535df1fd/src/plugins/stage-bundled-plugin-runtime.test.ts#L387)
  - sha: [51dd4f288f67](https://github.com/openclaw/openclaw/commit/51dd4f288f67ef4df4c7a121f078f508535df1fd)
- **Blame shows the relevant implementation landed after the latest release:** `git blame -L 477,500 -- src/plugins/discovery.ts` attributes the root-canonicalization lines to commit `51dd4f288f67ef4df4c7a121f078f508535df1fd` on 2026-04-24, which is newer than the provided latest release `v2026.4.23`.
  - file: [src/plugins/discovery.ts:481](https://github.com/openclaw/openclaw/blob/51dd4f288f67ef4df4c7a121f078f508535df1fd/src/plugins/discovery.ts#L481)
  - command: `git blame -L 477,500 -- src/plugins/discovery.ts`
