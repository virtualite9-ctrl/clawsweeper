## Summary

Keep #6615 open. Current main at 631552c554095b0c520376db9131dac0f5427f94 still does not implement first-class exec-approval denylist mode, `ask: "on-match"`, or denylist rule storage. The issue discussion links useful plugin prior art, including SafeClaw and CRE/claude-rule-enforcer, and there were closed unmerged PR attempts (#43222 and #60684), but none of that ships the requested OpenClaw core exec-approvals behavior.

## Best Possible Solution

Keep #6615 open until OpenClaw either implements first-class exec-approval denylist support across config/types/schema, approvals file normalization, gateway/node runtime policy, protocol, CLI/Control UI, docs, and tests, or maintainers explicitly redirect the feature to ClawHub/plugin policy with any missing core authorization hooks tracked separately.
