## Summary

Keep PR #70918 open. Current main still has the zh-Hans navigation drift the PR targets: the curated zh-Hans nav only exposes the Memory overview page, while the English source navigation has a nested Memory group with seven pages. The docs sync pipeline uses `docs/.i18n/zh-Hans-navigation.json` as the zh-Hans overlay, so this is actionable in this repo rather than obsolete or external.

## Best Possible Solution

Keep this PR open for maintainer review. The best near-term path is to land or adapt the scoped `docs/.i18n/zh-Hans-navigation.json` update so zh-Hans exposes the same Memory pages as English; a follow-up can consider generating or checking zh-Hans nav parity to avoid future drift.
