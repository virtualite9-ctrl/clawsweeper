## Summary

Keep open. Current main forwards the run AbortSignal into tools and races the outer prompt call, but the specific tool execution interruption design remains unresolved: the tool wrapper still awaits each tool normally, process poll ignores its signal during the timeout loop, and sessions_send/gateway calls remain timeout-driven rather than caller-abortable.

## Best Possible Solution

Keep this issue open as a core agent-runtime design follow-up. Maintainers should define the contract first, then implement either a framework-level tool execution interruption safety net or a documented equivalent, add abort-aware process poll behavior that stops waiting without killing the background process, add cancellable gateway/sessions_send plumbing if desired, and cover transcript/partial-result behavior with focused tests.
