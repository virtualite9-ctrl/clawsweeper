## Summary

Keep open. Current main has prompt-level abort handling and forwards the run AbortSignal into tools, but the specific framework/tool execution design request remains unresolved: the process poll loop still ignores its signal, sessions_send does not accept or pass a signal, and the gateway request path remains timeout-only rather than caller-abortable.

## Best Possible Solution

Keep this issue open as a core agent-runtime design follow-up. The maintainer path should define the intended contract first, then add focused implementation and tests for a tool execution safety net or documented prompt-level equivalent, abort-aware process poll that stops waiting without killing the background process, optional cancellable gateway calls for sessions_send, and explicit transcript/partial-result behavior.
