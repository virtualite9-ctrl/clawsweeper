## Summary

Keep open. Current main still has the empty-heartbeat fallthrough path: an empty heartbeat object is schema-valid, the default agent is still considered heartbeat-enabled, and interval resolution falls through to the default cadence. Open PR #64540 explicitly targets this issue and remains open/unmerged, so the maintainer path is to review/land or reject that implementation candidate rather than cleanup-close this issue.

## Best Possible Solution

Keep this issue open and review #64540 or an equivalent maintainer-authored patch. The fix should explicitly define the intended contract for empty heartbeat objects, update both default and per-agent heartbeat resolution if `{}` is meant to disable, preserve the documented `every: "0m"` behavior, and add focused regression coverage for empty defaults and empty per-agent overrides.
