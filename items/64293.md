## Summary

Current main still does not implement the requested empty-object disable behavior. `agents.defaults.heartbeat: {}` remains schema-valid, the default agent remains heartbeat-enabled, the interval resolver falls through to the 30m default, and Anthropic config defaults can backfill `every` onto an empty heartbeat object. An open, unmerged PR (#64540) uses closing syntax for this exact issue, so this issue should stay open until that PR or an equivalent fix is reviewed and landed.

## Best Possible Solution

Keep this issue open and review PR #64540 or land an equivalent maintainer patch. The fix should define the empty-object heartbeat contract, apply it consistently to default and per-agent heartbeat resolution, preserve the documented `every: "0m"` disable behavior, handle Anthropic default backfill, and add focused regression tests plus docs/validation updates for the chosen contract.
