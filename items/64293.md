## Summary

Keep open. Current main still accepts `agents.defaults.heartbeat: {}` as schema-valid, treats the default agent as heartbeat-enabled, and falls through to a default interval instead of disabling. The provided GitHub context also shows open, unmerged PR #64540 uses closing syntax for this exact behavior, so this issue should stay open until that implementation candidate is reviewed, landed, or rejected.

## Best Possible Solution

Keep this issue open and review PR #64540 or an equivalent maintainer patch. The best fix should explicitly define the empty-object heartbeat contract, apply consistently to default and per-agent heartbeat resolution, preserve the documented `every: "0m"` disable behavior, cover Anthropic default injection, and add focused regression tests plus docs/validation updates for the chosen contract.

## Risks / Open Questions

- Maintainers still need to decide the intended contract: support `{}` as a disable signal, or keep `every: "0m"` as the only disable path and make validation/docs reject or discourage empty heartbeat objects.
