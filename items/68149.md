## Summary

Current main still does not implement the PowerShell ScheduledTasks behavior proposed here. It keeps the schtasks-first Windows gateway install/lifecycle/status path and the documented/tested Startup-folder fallback policy. This PR is recent, focused on core Windows daemon behavior, and still needs maintainer review rather than cleanup closure, especially because it changes OS command execution and Windows auto-start policy.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is a focused Windows daemon update in src/daemon/schtasks.ts that makes PowerShell ScheduledTasks the primary implementation, disables ExecutionTimeLimit, handles battery settings, preserves a valid schtasks fallback where appropriate, keeps uninstall cleanup for Scheduled Task plus Startup-folder/script artifacts, and updates docs/tests for the accepted admin/fallback policy.
