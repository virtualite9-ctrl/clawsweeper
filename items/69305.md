## Summary

Keep this PR open. Current main still emits zsh leaf command completions from option specs only, so the linked `wiki ingest <path>` bug is not implemented on main. The PR is a relevant fix attempt, and the duplicate PRs in the provided context are closed/unmerged, but this PR should address the remaining false-positive completion risk for non-filesystem `<path>` arguments before landing.

## Best Possible Solution

Keep #69305 open and request a small revision: add zsh positional specs for Commander registered arguments, keep optional/variadic semantics, and map filesystem actions using command-aware metadata or guarded heuristics so commands like `wiki ingest <path>` get `_files` while config dot/bracket paths and IDs stay neutral. Land with focused regression tests for wiki file completion, directory/optional/variadic behavior, and at least one non-filesystem `<path>` case.
