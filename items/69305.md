## Summary

Keep PR #69305 open. Current main still generates zsh leaf-command completions from option specs only, so the reported `openclaw wiki ingest <path>` positional completion bug is not implemented on main. The PR is a focused fix for open issue #69293, while the related duplicate PRs in the provided context are closed and unmerged. It still needs maintainer review or a revision for the unresolved false-positive risk around non-filesystem `<path>` arguments.

## Best Possible Solution

Keep the PR open and ask for a small revision or maintainer review: add zsh positional specs from Commander argument metadata, preserve optional positional semantics and escaping, and tighten filesystem action selection so real file/path commands like `wiki ingest` get `_files` while config-key paths and other domain identifiers do not. Keep or add focused tests in `src/cli/completion-cli.test.ts` for file path, directory, optional/variadic, and non-filesystem `<path>` cases.
