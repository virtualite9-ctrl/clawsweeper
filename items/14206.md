## Summary

Keep #14206 open. Current main still does not provide a Feishu outbound minimum-interval/rate-limit setting, and the Feishu reply, comment, outbound adapter, and message-tool send paths still call the send APIs directly without a shared per-account pacing hook. The related #13615 remains broader external API throttling work and does not supersede this Feishu-channel request.

## Best Possible Solution

Keep the issue open and implement the pacing in the Feishu plugin surface: add documented top-level and per-account outbound minimum-interval/rate-limit config, route reply chunks, comment replies, message-tool text/card/media sends, text-plus-media sends, and fallback sends through one per-account pacing helper, and add fake-timer tests plus config schema, manifest, generated metadata, and docs updates. Treat #13615 as broader external API rate-limiting work, not as the canonical replacement for this Feishu-specific channel behavior.
