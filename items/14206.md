## Summary

Keep #14206 open. Current main still lacks a Feishu outbound minimum-interval/rate-limit setting, and the key Feishu outbound paths still call the Feishu send/comment/card APIs directly rather than through a shared per-account pacing layer. The related #13615 tracks broader external API throttling and does not supersede this Feishu-specific channel request.

## Best Possible Solution

Keep the issue open. Implement Feishu outbound pacing in the bundled Feishu plugin: add documented top-level and per-account minimum-interval/rate-limit config, publish it through schema, manifest, generated metadata, and docs, route reply chunks, comment replies, message-tool text/card/media sends, pairing/bot notifications, fallbacks, and streaming card create/update/close paths through one per-account pacing helper, and cover the behavior with fake-timer tests. If #13615 later provides a suitable generic limiter contract, use it as the implementation mechanism but close #14206 only once Feishu exposes and applies the setting.
