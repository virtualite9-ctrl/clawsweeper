## Summary

Keep #14206 open. Current main still lacks a user-configurable Feishu outbound minimum-interval/rate-limit setting and still sends reply chunks, comment chunks, message-tool chunks, text-plus-media sequences, and fallback sends without a shared per-account pacing hook. The existing related mitigations reduce some Feishu API usage but do not solve the reported outbound message burst behavior.

## Best Possible Solution

Keep the issue open and implement this in the Feishu plugin surface: add documented top-level and per-account outbound minimum-interval/rate-limit config, route all Feishu outbound sends through a shared per-account pacing helper, and cover reply chunks, comment replies, message-tool text/card sends, text-plus-media sequences, and fallback sends with fake-timer tests plus config schema, manifest, generated metadata, and docs updates. Treat #13615 as broader external API rate-limiting work, not as a superseding canonical issue for this Feishu-specific channel behavior.
