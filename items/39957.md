## Summary

Keep PR #39957 open. Current main still lacks the PR's narrow `message_end && stopReason === "toolUse" && tool-call block` suppression, so unphased intermediate tool-use status text can still flow into Telegram-visible block replies. The newer commentary-phase suppression only covers messages explicitly marked as commentary and does not cover the legacy unphased case described by #24376/#39957.

## Best Possible Solution

Keep the PR open and land either this narrow `message_end && stopReason === "toolUse" && has structured tool-call block` suppression with regression coverage, or an equivalent fix in the assistant phase/display model. Preserve the PR's non-goals: normal `stop` turns, `toolUse` turns without tool-call blocks, and `text_end` block-reply flow should continue to emit.
