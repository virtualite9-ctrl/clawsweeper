## Summary

Keep PR #39957 open. Current main still lacks the PR's narrow `message_end && stopReason === "toolUse" && structured tool-call block` suppression, so unphased intermediate tool-use status text can still be extracted as visible assistant text and emitted through `onBlockReply` to Telegram-visible replies.

## Best Possible Solution

Keep the PR open and land either this narrow `message_end && stopReason === "toolUse" && has structured tool-call block` suppression with regression coverage, or an equivalent core display-phase fix. Preserve the PR's stated non-goals: normal `stop` turns, `toolUse` turns without tool-call blocks, and `text_end` block-reply delivery should continue to emit.
