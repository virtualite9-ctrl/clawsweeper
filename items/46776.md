## Summary

Keep open. Current main has adjacent Feishu work for @all detection and configured broadcast dispatch, but #46776 reports two concrete behaviors and the DM/group continuity part is still not implemented: Feishu DMs and group chats route to different session keys by design. There is not enough source/docs/history evidence to close this as implemented, stale, duplicate, or out of scope.

## Best Possible Solution

Keep this open and split the maintainer path into two explicit decisions: first, reproduce or add a targeted Feishu test for @all delivery with more than three agents/accounts and fix any queue/dedup/rate-limit loss; second, decide whether Feishu should gain an explicit opt-in DM/group session-linking feature, or document that cross-session continuity is intentionally handled through memory/session tools rather than shared transcripts.
