## Summary

Keep this issue open. Current main has adjacent pieces: embedded/runtime-plan harness selection, per-agent workspaces/auth/session stores, config hot reload, node/operator Gateway WebSocket roles, and ACP/subagent execution paths. It still does not implement the requested split into a lightweight gateway control plane plus separately registered per-agent compute runtimes, nor the requested local-split/distributed deployment modes or per-agent runtime lifecycle CLI.

## Best Possible Solution

Keep this issue open for maintainer architecture review. The best path is an additive design that preserves embedded Gateway mode, first defines a stable gateway-to-agent-runtime protocol and local process split with per-agent lifecycle commands, then layers container/Kubernetes backends and stricter per-agent secret/resource isolation on top. The design should reuse the existing runtime plan/harness, config reload, Gateway protocol, per-agent workspace/auth/session-store, sandbox, ACP, and node-host boundaries where they already solve part of the problem.
