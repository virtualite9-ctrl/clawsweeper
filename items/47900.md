## Close Comment

Closing from a Codex review because current `main` already implements this behavior. The Control UI chat send path now appends the user message immediately and includes uploaded images as inline `image` blocks before the backend round-trip (`ui/src/ui/controllers/chat.ts:315`). The renderer understands those inline image blocks and displays them in the chat bubble (`ui/src/ui/chat/grouped-render.ts:143`). Current `main` also explicitly defers `session.message` history reloads while a run is active so that the optimistic user message is not clobbered until the run reaches a terminal event (`ui/src/ui/app-gateway.ts:490`), with coverage in `ui/src/ui/app-gateway.node.test.ts:725`. Reviewed against `main` at `d38ed0831d7372c8505dc944aea69b5b83da1435`. If you still hit this on a newer build, please open a fresh issue with the exact version/build and a current repro.
