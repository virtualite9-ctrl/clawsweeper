## Summary

Keep #30389 open. Current main only partially satisfies the request: WebChat now has a visible attachment button and session model picker, but the upload path is still image-only and there is still no composer-visible skill/capability picker beyond slash-command and settings surfaces. The existing ClawSweeper comment and related open document-upload trackers match the current checkout behavior.

## Best Possible Solution

Keep #30389 open unless maintainers deliberately split it. Preserve the shipped attachment button and model picker, then complete WebChat document/file upload end to end through UI accept/validation/previews, chat.send serialization, Gateway parsing/offload/extraction, provider/tool handling, docs, and tests. Separately define a composer-visible skill/capability picker that maps to skills, tools, agents, task profiles, or model profiles instead of relying only on typed slash commands.
