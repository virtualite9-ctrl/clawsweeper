## Summary

Keep #30389 open. Current main now implements a visible chat attachment button, non-video file attachment handling, and a chat header model picker, but the requested composer-visible skill/capability menu remains only partially covered by slash-command completion and the global command palette rather than a dedicated capability picker.

## Best Possible Solution

Keep #30389 open unless maintainers deliberately split it. Preserve the shipped attachment button, non-video upload path, and model/thinking pickers; coordinate any remaining document-processing details with #30759/#70438; then define a dedicated composer-visible skill/capability picker backed by commands.list, skills.status, tools.effective, agents, task profiles, or model profiles, with clear behavior for domain capabilities such as finance, coding, writing, media generation, and reasoning.
