## Summary

Current main only partially satisfies #30389. WebChat has a discoverable attachment button and a session model selector, but the upload path is still image-only end to end and the requested visible skill/capability picker remains absent; the existing slash-command menu and /skill command do not cover the proposed finance/coding/writing/image/reasoning capability menu.

## Best Possible Solution

Keep #30389 open as an umbrella WebChat UX request or split it into narrower maintainer-owned issues. The shipped model picker and image attachment button should stay, while remaining work should add document/file upload end to end through the WebChat picker, validation/preview, chat.send serialization, Gateway parsing/extraction/offload, provider/tool handling, and docs/tests. Separately define a visible composer skill/capability picker that maps cleanly to skills, tools, agents, or task/model profiles rather than relying only on typed slash commands.
