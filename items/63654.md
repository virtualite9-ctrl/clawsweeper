## Summary

Keep open. Current main still intentionally blocks or omits qwen3.6-plus on Qwen Coding Plan endpoints, and the linked fixes remain open rather than shipped. The issue is recent, specific, not protected by maintainer ownership/labels, and remains actionable in the bundled qwen plugin rather than being stale, external, or already implemented.

## Best Possible Solution

Keep #63654 open as the canonical issue until maintainers decide the Qwen Coding Plan contract. The likely implementation belongs in the bundled qwen plugin: either remove the Coding Plan filter/suppression in `extensions/qwen/models.ts` and `extensions/qwen/index.ts`, or add a documented explicit opt-in path with regression tests and updated Qwen docs. After that lands, close or redirect the later duplicate #69027 and choose between PR #63987 and PR #66367.
