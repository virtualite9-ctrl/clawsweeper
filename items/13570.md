## Summary

Keep #13570 open. Current main still has no `contextPruning.forcePruneRatio` or equivalent high-usage TTL bypass: the public config type/schema/docs omit it, runtime settings do not parse it, and the context-pruning extension still returns before pruning while the cache TTL is fresh. The related implementation attempts, PR #13572 and its claimed consolidation PR #13512, are both closed unmerged, so they do not supersede the issue.

## Best Possible Solution

Keep #13570 open until a maintainer implements or explicitly declines the feature. A likely implementation should add `agents.defaults.contextPruning.forcePruneRatio` to the TypeScript config type, strict Zod schema, generated schema, docs, effective settings, and runtime; estimate context usage before the fresh-TTL early return; bypass TTL only when the configured threshold is met; preserve the existing TTL gate below threshold; and add regression tests for both above-threshold bypass and below-threshold preservation.
