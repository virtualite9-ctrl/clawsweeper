## Summary

Keep #13570 open. Current main still does not implement `contextPruning.forcePruneRatio` or an equivalent high-usage TTL bypass: the public config type, strict schema, generated schema, docs, effective runtime settings, and tests all omit the key, and the context-pruning extension still returns during a fresh TTL window before usage-ratio pruning can run. The directly related PR #13572 addressed this request but is closed unmerged, so it does not supersede or ship the feature.

## Best Possible Solution

Keep the issue open until a maintainer implements or explicitly declines the feature. A likely fix belongs in the context-pruning config contract and runtime: add `agents.defaults.contextPruning.forcePruneRatio` to the TypeScript config type, strict Zod schema, generated schema, docs, effective settings, and runtime; estimate context usage before the fresh-TTL early return; bypass TTL only when the configured ratio is met; and add regression tests for both above-threshold bypass and below-threshold TTL preservation.
