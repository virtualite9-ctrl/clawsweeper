## Summary

Keep open. Current main has timeout-recovery compaction for high-context LLM timeouts, but it does not implement the requested session-level circuit breaker: no configurable max consecutive timeout setting, no persisted per-session timeout/degraded breaker state, and no webhook/system-event path that stops future heartbeat/message retries after N failures.

## Best Possible Solution

Keep #45389 open or consolidate it into a canonical session-level breaker tracker with #42933/#52384/#59161. The best implementation belongs in the agent/session runtime: documented `agents.defaults` and per-agent config, persisted per-session consecutive timeout/model-error state, reset-on-success semantics, heartbeat/cron/auto-reply gating before new runs, and a structured system event/webhook when the breaker trips. The existing timeout-recovery compaction should remain as the first mitigation for high-context timeouts.
