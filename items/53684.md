## Summary

Keep open. Current main has adjacent gateway health/status probes, service lifecycle commands, LaunchAgent KeepAlive, channel health restarts, and new stability-bundle diagnostics for startup/restart failures, but it still does not provide the requested external watchdog, automatic gateway-down notification/recovery path, human-readable startup-error artifact, or first-class multi-instance health monitoring. The provided discussion also shows PR #53716 is open and unmerged with closing syntax for this issue, so the best maintainer path is to review or close that implementation candidate before resolving the issue.

## Best Possible Solution

Keep this issue open while PR #53716 remains open and unmerged. Maintainers should review that PR or a successor against the current gateway diagnostics, service lifecycle, LaunchAgent/systemd behavior, and docs, then either land a focused watchdog/startup-diagnostics/notification implementation with tests or close the PR and keep this issue as the canonical feature request.
