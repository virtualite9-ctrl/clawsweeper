## Summary

Current `main` has partial adjacent work for diagnostics and channel-level recovery, but it does not implement the requested gateway-level failure recovery and notification mechanism well enough to close this issue. The repo has a channel health monitor that restarts unhealthy channels, persisted gateway stability bundles for startup/fatal failures, a macOS health UI that polls and surfaces errors, and docs for running a separate rescue gateway. What is still missing from `main` is the core ask: automatic gateway-outage notification/escalation and first-class multi-instance monitoring/recovery.
