## Summary

Keep open. Current main has useful adjacent gateway health, restart-health, supervisor recovery, channel health monitoring, stability bundles, diagnostics export, and rescue-bot documentation, but it still does not implement the requested external gateway-down watchdog, user notification/escalation path, human-readable startup-error artifact, dead-man switch, or first-class sibling health monitoring. The provided GitHub context also shows PR #53716 is open and unmerged with explicit closing syntax, so this issue should remain open while that implementation candidate is reviewed or closed.

## Best Possible Solution

Keep this issue open while PR #53716 remains open and unmerged. Maintainers should review that PR or a successor against current gateway status/probe/stability, service lifecycle, LaunchAgent/systemd behavior, channel health monitoring, and multiple-gateway docs, then land a focused watchdog/startup-diagnostics/notification implementation with tests and docs or close the PR and keep this as the canonical feature request.
