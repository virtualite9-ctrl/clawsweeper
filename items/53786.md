## Summary

Keep this open. Current main has the live update plumbing for Feishu-backed sessions, but it does not satisfy the reporter's bidirectional exact-sync expectation: Control UI only reloads the active session, and WebChat/Control UI sends intentionally do not inherit external channel delivery routes for UI clients, even with deliver enabled. The remaining work needs either a clear maintainer product decision/docs clarification or a Feishu-specific reproduction with gateway logs.

## Best Possible Solution

Keep this open for maintainer triage. First reproduce on current main with a Feishu DM and group, noting the active Control UI session key, `session.dmScope`, gateway logs, and whether `session.message` arrives. If the desired product behavior is true bidirectional mirroring, add an explicit opt-in Control UI delivery/mirror flow for Feishu sessions plus tests for Feishu inbound -> active-session reload and Control UI explicit delivery -> Feishu. If the intended behavior is that Control UI is a session viewer/editor rather than a Feishu chat mirror, update docs and close only after that product decision is explicit.
