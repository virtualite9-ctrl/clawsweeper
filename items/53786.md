## Summary

Keep this issue open. Current main has generic live transcript/event plumbing, but it still does not implement the reporter's requested exact bidirectional Feishu Bot and Local Dashboard mirroring: the browser dashboard connects as WebChat, normal dashboard sends use `deliver: false`, WebChat clients are intentionally blocked from inheriting external delivery routes, and Control UI only reloads chat history for the currently selected session key.

## Best Possible Solution

Keep this open for maintainer product triage. Reproduce current main with Feishu DM and group/thread sessions, recording the active Control UI session key, resolved Feishu route/session key, gateway logs, and whether `session.message` arrives. If bidirectional mirroring is desired, add an explicit opt-in Control UI delivery/mirror flow for Feishu-backed sessions with tests for Feishu inbound-to-active-session refresh and dashboard-to-Feishu delivery. If the intended boundary is that Control UI/WebChat is a session viewer/editor rather than an external chat mirror, document that clearly in the Control UI, WebChat, and Feishu/channel-routing docs before closing.
