## Summary

Keep open. Current main shows that OpenClaw does not guarantee exact real-time history mirroring between Control UI and external chat clients, and it does preserve established external delivery routes when Control UI is viewing the same channel session. That means part of the report likely reflects session/config expectations rather than a confirmed regression. But the issue does not include the decisive details needed to prove it is obsolete or non-reproducible on current main: the selected dashboard session, `session.dmScope`, whether the Feishu DM was sharing the agent main session or a channel-scoped session, and any gateway/channel logs. Under a conservative cleanup standard, that is not strong enough to close.

## Evidence

- **Docs explicitly disclaim full cross-client transcript mirroring:** The current docs state that direct chats can share a session, but history is 'not fully synced back to every client' and recommend using one primary device. That undercuts the issue's expectation that the dashboard and Feishu should always show identical history.

In that common configuration, a Feishu DM and the dashboard main chat are different session keys by default, so apparent dashboard/Feishu desync can be configuration/session-scope behavior rather than a product bug.
