## Summary

Keep open. Current `main` still drops non-`assistant`/non-`lifecycle` relay streams in `startAcpSpawnParentStreamRelay`, so the reported `system_event` gap is not implemented there, and the PR’s proposed fix was already reviewed as incomplete because it would still mis-handle stall tracking.

## Evidence

- **Current relay still ignores non-lifecycle streams:** In current `main`, the ACP parent-stream relay handles `assistant` events, then immediately returns for any stream other than `lifecycle`. There is no `system_event` branch, so a child `system_event` would still be dropped.
  - file: [src/agents/acp-spawn-parent-stream.ts:312](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/src/agents/acp-spawn-parent-stream.ts#L312)

  - sha: [c070509b7f8f](https://github.com/openclaw/openclaw/commit/c070509b7f8fdbe479e5d829643782c272e30e83)
- **Product docs still describe parent streaming as system-event delivery:** The ACP docs say `streamTo: "parent"` streams initial ACP run progress summaries back to the requester session as system events, so this remains an intended supported surface rather than obsolete behavior.
  - file: [docs/tools/acp-agents.md:330](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/docs/tools/acp-agents.md#L330)
