## Summary

Close PR #63375 as cannot_reproduce. The PR's stated rationale is that `console.log` in plugin `register()` corrupts `openclaw config schema` JSON, but current `main` builds that schema from manifest metadata and does not run plugin registration for that command path. Current SDK docs already expose `api.logger`, and the PR's proposed wording is also partly stale because `OpenClawPluginApi.logger` is required while only `debug` is optional.

## Best Possible Solution

Do not merge this PR as written. Keep the manifest-only config schema path and regression coverage; if maintainers want broader plugin logging guidance, add a smaller docs note to `docs/plugins/building-plugins.md` or `docs/plugins/sdk-runtime.md` that references `api.logger`, accurately describes the current required logger contract, and names a verified current stdout-corruption surface.
