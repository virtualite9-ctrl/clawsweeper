## Summary

Keep PR #38502 open. Current main still has the same Ctrl+C UX mismatch: Ctrl+C only clears/warns/exits, the active run remains separate, and the status text does not tell users to use /abort while a run is active. The PR is still a narrow one-file TUI text fix against the linked open bug #38501.

## Best Possible Solution

Keep PR #38502 open or land an equivalent narrow change on main: compute an active-run hint from activeChatRunId inside handleCtrlC, append '; run still active (/abort to stop)' only to the clear and warn Ctrl+C statuses, leave exit behavior and abort/keybinding semantics unchanged, and add focused status-text coverage if the TUI status surface is practical to test.
