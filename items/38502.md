## Summary

Keep open. Current main still lacks the active-run Ctrl+C status hint, while Ctrl+C remains a local clear/warn/exit flow and `/abort` remains the explicit active-run abort path. The PR is still a focused one-file TUI UX fix, and its current diff addresses the prior review concern by advertising `/abort` only instead of unconditional Esc behavior.

## Best Possible Solution

Keep PR #38502 open or land an equivalent narrow change on main: compute an active-run hint from `activeChatRunId` at Ctrl+C handler time, append `; run still active (/abort to stop)` to the clear/warn Ctrl+C statuses only, leave exit behavior and abort/keybinding semantics unchanged, and add focused status-text coverage if the TUI status surface is practical to test. The linked issue #38501 remains the right user-problem anchor.
