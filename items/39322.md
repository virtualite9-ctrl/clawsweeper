## Summary

Keep PR #39322 open. Current main still lacks the legacy iMessage/imsg tapback reaction forwarding that the PR attempts to add: no includeReactions config/schema, no reaction payload parser/type, no reaction notification handler, and watch.subscribe only requests attachments. The linked feature issue #39031 remains open, so this PR is still potentially useful work rather than cleanup-closeable stale code.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to update the branch against current main, verify the imsg reaction stream contract, add includeReactions to config/schema/docs, add parser and monitor tests, and explicitly define how synthetic reaction events interact with group mention gating. If maintainers decide not to add new legacy imsg reaction ingress, supersede this PR with a clear decision on #39031.
