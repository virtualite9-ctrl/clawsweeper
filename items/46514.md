## Summary

Keep #46514 open. Current main still treats Discord guild/channel conversations as agent-scoped session buckets, and I did not find shared cross-agent Discord channel history or automatic transcript injection. `allowBots` controls whether bot-authored messages may be processed; it does not merge another agent's session history. The related #59534 closure also points back to #46514 as the broader shared-context tracker.

## Best Possible Solution

Keep #46514 open as the canonical shared cross-agent channel-history tracker. The best path is an explicit opt-in design, likely touching Discord plugin behavior plus shared routing/session visibility contracts, that can inject or expose attributed channel history across selected agents without collapsing isolation by default.

- A fix needs explicit product and security design: opt-in cross-agent visibility, clear attribution, sender/channel allowlist behavior, and bot-loop prevention.
