## Summary

Keep open. Current main still has the fresh-token fast path that returns before token-threshold evaluation, so a fresh persisted total above the preflight threshold can still skip proactive compaction. The provided GitHub context also shows an open, unmerged implementation PR by the same author with closing syntax, so the right path is to review, land, or supersede that PR before closing this issue.

## Best Possible Solution

Review and land or explicitly supersede #64384, or choose #66716 if maintainers prefer that broader implementation. The fix should make `runPreflightCompactionIfNeeded` evaluate the preflight threshold for fresh persisted totals above the limit, preserve the below-threshold fast path, and add a focused regression test in `src/auto-reply/reply/agent-runner-memory.test.ts` for `totalTokensFresh: true` with cached tokens above threshold and no transcript-byte trigger.
