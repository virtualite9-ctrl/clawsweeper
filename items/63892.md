## Summary

Keep this issue open. Current main still returns from `runPreflightCompactionIfNeeded` before evaluating the token threshold when persisted totals are fresh and the transcript-byte trigger is inactive. The issue also has an open same-author implementation candidate, #64384, that declares it fixes this issue, so maintainers should review/land or explicitly supersede that PR before closing.

## Best Possible Solution

Keep this issue open. Review and land #64384, or explicitly supersede it with #66716 or a maintainer patch. The fix should let fresh persisted token totals at or above the preflight threshold reach `shouldRunPreflightCompaction`, preserve the below-threshold fast path, and add focused regression coverage for `totalTokensFresh: true`, cached tokens above threshold, and no transcript-byte trigger.
