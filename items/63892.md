## Summary

Keep open. Current `main` still has the unconditional fast-path return in preflight compaction, so fresh persisted token totals never reach the threshold check after the first compaction. The issue discussion usefully corrected the original `compactionCount` hypothesis, and the code on `main` matches that corrected root cause rather than an implemented fix.

## Evidence

- **Current preflight path still short-circuits on fresh totals:** `runPreflightCompactionIfNeeded` computes `freshPersistedTokens`, then immediately returns when `totalTokensFresh !== false` and persisted totals exist. That bypasses the later `shouldRunPreflightCompaction(...)` threshold check entirely for fresh cached totals.
