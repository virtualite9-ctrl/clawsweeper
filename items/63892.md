## Summary

Keep open. Current main still returns from `runPreflightCompactionIfNeeded` before token-threshold evaluation when a session has fresh persisted token totals and the transcript-byte trigger is inactive. The issue discussion’s corrected root cause matches the code: `compactionCount` increments, but the fresh-token fast path can still skip proactive compaction. The provided GitHub context also shows an open, same-author implementation PR with closing syntax, so this issue should stay open until that PR is landed or explicitly superseded.

## Best Possible Solution

Review and land or explicitly supersede #64384, or choose #66716 if maintainers prefer that broader implementation. The fix should make `runPreflightCompactionIfNeeded` evaluate the preflight token threshold for fresh persisted totals above the limit, preserve the below-threshold fast path, and add a focused regression test for `totalTokensFresh: true` with cached tokens above threshold and no transcript-byte trigger.
