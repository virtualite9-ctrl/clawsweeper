## Summary

Current `main` only defers heartbeat runs while the target lane is actively busy, then retries about 1 second later. It does not implement an idle-only trigger or any configurable inactivity threshold like the issue requests, so this is only a partial mitigation and should stay open.

## Evidence

- **Runner skips only while work is in flight:** `runHeartbeatOnce` checks the main lane and resolved session lane queue sizes and skips with `requests-in-flight` when either lane is busy. That avoids overlap with active turns, but it is not an idle timer or silence-based trigger.
  - file: [src/infra/heartbeat-runner.ts:759](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/src/infra/heartbeat-runner.ts#L759)
  - command: `nl -ba src/infra/heartbeat-runner.ts | sed -n '748,798p'`
  - sha: [ec3dbd22a4de](https://github.com/openclaw/openclaw/commit/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46)
- **Deferred heartbeats retry after 1 second:** The wake layer reschedules `requests-in-flight` heartbeats with `DEFAULT_RETRY_MS = 1000`, so a deferred heartbeat runs again shortly after activity clears instead of waiting for a configurable idle window.
  - file: [src/infra/heartbeat-wake.ts:42](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/src/infra/heartbeat-wake.ts#L42)

- **Config surface has no idle-only option:** The heartbeat config/schema/docs expose `every`, `activeHours`, `session`, `target`, `isolatedSession`, and related fields, but nothing like `idleOnly` or `triggerWhen: "idle"`.
  - file: [src/config/types.agent-defaults.ts:332](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/src/config/types.agent-defaults.ts#L332)
