## Summary

#59567 should close as a duplicate/superseded tracker. Its requested idle-only heartbeat behavior is the same remaining product work already tracked by open issue #14051, which is the broader activity-based heartbeat idle-timeout thread. Current main still lacks the requested config and runtime behavior, so closing #59567 is only deduplication, not an implemented-on-main close.

## Best Possible Solution

Close #59567 as duplicate/superseded by open issue #14051. Keep the actual product/design work in #14051: add a product-approved heartbeat idle-timeout config key, schema/help/docs coverage, config-honor inventory updates, and runtime tests that gate heartbeat runs on recent real user/channel activity while preserving active-hours, queue retry, targeted wakes, task wakes, light context, and isolated-session behavior.
