## Summary

Keep open. Current main still does not implement `tools.exec.nsSandbox`, an `exec-bwrap-sandbox` module, or a bubblewrap wrapper path for safeBins-approved gateway exec commands. Existing safeBins hardening and the general agent sandbox backends are related defenses, but they do not provide the requested safeBins-only Linux namespace sandbox.

## Best Possible Solution

Keep PR #39244 open for maintainer/security judgment. The best path is a narrowed, rebased implementation of optional Linux bwrap isolation for safeBins-approved gateway exec, wired through config types, strict schema, docs/help labels, gateway allowlist evaluation, runtime spawning, and focused tests. If maintainers decide current safeBins hardening plus existing sandbox backends are sufficient, that should be an explicit product/security decision rather than automated cleanup.
