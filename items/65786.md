## Summary

Keep this issue open. Current main still does not accept or repair the reporter's Feishu config shape: Feishu's strict schema omits `contextVisibility`, `streaming` is still boolean-only, and there is no Feishu doctor compatibility normalizer to migrate legacy scalar streaming values such as `"progress"`. The existing ClawSweeper comment reached the same conclusion, and the current checkout still matches that analysis.

## Best Possible Solution

Keep this issue open. The best fix belongs in the Feishu plugin/config contract: add `contextVisibility` to the accepted Feishu channel and account schema if intended, decide how legacy scalar `streaming` values such as `"progress"` should map onto Feishu's boolean card streaming, expose a Feishu doctor compatibility normalizer for the reported upgrade path, regenerate bundled channel metadata, and add regression coverage using the reporter's config plus a doctor repair case.

## Risks / Open Questions

- A fix should not blindly reuse Discord/Telegram preview-streaming enum semantics for Feishu unless Feishu runtime behavior is intentionally changed; Feishu currently models card streaming as a boolean.
- Adding `contextVisibility` should update both source schema and generated bundled channel metadata, and should cover top-level and account-level Feishu config if both are intended.
- Doctor migration should be conservative: preserve the existing Feishu card-streaming default while repairing legacy scalar values such as `"progress"` into a valid Feishu setting or removing/rewriting them with a clear change note.
