## Summary

Keep this PR open. Current main still lacks the PR's additive realtime consult thinking-level and fast-mode override surfaces for both Voice Call and Control UI Talk. Related #71849 tracks a broader fast memory/context hot path, not this narrower one-shot override plumbing, and the provided discussion includes a positive bot review plus author production verification.

## Best Possible Solution

Keep this PR open for maintainer review. If accepted, land or adapt the additive implementation so Voice Call and Control UI Talk realtime consults can pass one-shot thinking-level and fast-mode overrides without mutating persistent session state, while keeping #71849 as the separate canonical follow-up for realtime voice fast context retrieval.
