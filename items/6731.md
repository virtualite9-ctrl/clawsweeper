## Summary

Keep #6731 open. ## Best Possible Solution

Keep #6731 open until maintainers either implement a native hard/default dangerous-operation gate in core exec/tool policy or explicitly decide that the current opt-in model is final. A concrete path should cover default safe mode, dangerous-command classification including Windows PowerShell cases like `Remove-Item`, approval UX, sandbox interaction, and regression tests/docs.
