Keep #6731 open. Current main still treats no-approval host exec as the default trusted-operator behavior unless operators opt into stricter policy/sandboxing, and the issue discussion has an Apr 26 maintainer-side review explicitly keeping this open as core safety/product scope.

## Best Possible Solution

Keep #6731 open until maintainers either implement a native hard/default dangerous-operation gate in core exec/tool policy or explicitly decide that the current opt-in exec approvals, sandboxing, and tool-policy model is the final product stance. A concrete implementation path should cover default safe mode, Windows PowerShell destructive-command classification such as Remove-Item, approval UX, sandbox interaction, and regression tests/docs.
