## Summary

Keep this PR open. Current main still routes CLI version output through resolveCommitHash, and that resolver still prefers live git metadata before dist/build-info.json with no injected build-time commit source. The PR remains a focused implementation candidate for the drift reported in linked issue #68148.

## Best Possible Solution

Keep this PR open for maintainer review, rebase if needed, and land it or an equivalent patch. The best implementation is to bake a sanitized build-time commit into node build outputs, consult it after explicit GIT_COMMIT/GIT_SHA env overrides and before live git, and keep focused regression coverage for injected priority, fallback behavior, env override, and cache stability.
