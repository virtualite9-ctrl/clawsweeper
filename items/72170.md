## Summary

Keep PR #72170 open. Current main has only the generic Kubernetes guide and no AKS-specific cluster, ingress, or ACR instructions, while cloud/provider install guides are an established docs surface. The PR is a concrete docs contribution, but it still needs maintainer docs/security review before merge because the latest patch retains at least the ACR placeholder issue and public ingress guidance must stay aligned with the gateway bind/auth model.

## Best Possible Solution

Keep the PR open and ask for a small docs revision: replace the ACR registry example with a placeholder, make the non-loopback bind and remote-access security prerequisites explicit before the Ingress commands, and let maintainers decide whether the AKS content belongs as a section in the Kubernetes guide or a separate Azure AKS install page/nav entry.
