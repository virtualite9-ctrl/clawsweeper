## Summary

Keep PR #72170 open. Current main has a generic Kubernetes guide and an Azure Linux VM guide, but no AKS-specific deployment guide, ACR integration guidance, or AKS ingress walkthrough. Provider/platform install docs are already an established OpenClaw docs surface, so this is not a ClawHub/plugin-scope close. The PR is a focused docs contribution, but the provided review comments identify real issues that still need author or maintainer docs/security review rather than cleanup closure.

## Best Possible Solution

Keep the PR open for normal docs review. The best path is to revise the AKS section so it is consistent with the existing Kubernetes exposure guidance, uses safe placeholders for Azure-global resources like ACR names, and remains scoped as an AKS-specific addition to the existing `/install/kubernetes` page rather than a new core feature or plugin.
