## Summary

Keep open. Current main still has the exact remaining bug the PR targets: `ltm search --limit <invalid>` parses the CLI value to `NaN` and passes it into the memory database search path, which then forwards it to LanceDB `.limit(...)` without validation. The PR branch contains a focused validation/warning fix, and the only related PR mentioned (#63326) was a separate consistency cleanup that closed unmerged and did not supersede this memory-lancedb CLI behavior.

## Best Possible Solution

Keep the PR open and either land this focused validation plus warning or adjust it to the maintainer-preferred CLI behavior. The change belongs in `extensions/memory-lancedb/index.ts` near the `ltm search` command, with a small colocated test that proves invalid, zero, and negative limits do not reach `db.search` as `NaN` or non-positive values.
