## Summary

Current main already applies `agents.list[].systemPromptOverride` on the CLI fresh-session path. The CLI preparation path resolves the per-agent/default override from live config before building the prompt, and the Codex CLI execution path sends that prompt through `model_instructions_file` for fresh sessions when there is no persisted CLI session binding. The shipped v2026.4.24 tag contains the same implementation, so a 2026.4.21 deployment should be upgraded rather than keeping this issue open.

## Best Possible Solution

Close this issue as already implemented on main and shipped in v2026.4.24. Operators who reproduced it on 2026.4.21 should upgrade; if the behavior still appears on current main or a later release, the next report should include the generated CLI argv/prompt-file evidence and the session entry showing the missing `cliSessionBinding`.
