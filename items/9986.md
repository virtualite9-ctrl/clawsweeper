## Summary

Keep #9986 open. Current main still treats context overflow as active-model compaction/retry plus a blocked overflow message after exhaustion, and the model fallback chain explicitly excludes context-overflow errors. Related closed items cover overloaded/server-error fallback or graceful overflow handling, not the requested smaller-primary to larger-context fallback routing.

## Best Possible Solution

Keep #9986 open until maintainers either document context overflow as intentionally compaction-only, or implement a guarded fallback path after compaction and oversized-tool-result truncation are exhausted. The likely fix spans the embedded runner, failover classification/result classification, fallback candidate context-window checks, logging, docs, and regression tests for smaller-primary to larger-fallback behavior.
