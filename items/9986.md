## Summary

Current main still does not route context-overflow failures to a larger configured model fallback. The model fallback loop explicitly rethrows likely context-overflow errors, the embedded Pi runner handles overflow by compaction/tool-result truncation on the active provider/model, and exhausted recovery returns a blocked context_overflow result instead of advancing candidates.

## Best Possible Solution

Keep #9986 open until maintainers either explicitly choose the documented compaction-only policy or implement a guarded context-overflow fallback path. The likely implementation belongs after overflow compaction and oversized-tool-result truncation are exhausted: if replay is safe and a configured fallback has known larger context headroom, switch to that candidate, log the event, and add regression coverage plus docs for smaller-primary to larger-fallback behavior.

## Risks / Open Questions
