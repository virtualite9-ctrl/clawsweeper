## Summary

Keep #71803 open. Current main still resolves a finite CLI no-output watchdog, passes it into both regular CLI supervision and the Claude live-session path, and treats `no-output-timeout` as a fatal FailoverError. The reset paths I found are stdout/stderr activity only; I did not find Monitor/background-task pending state, expected-idle exemption, or heartbeat integration. Related #40982/#41022 remain open broader watchdog-cap work, and #72206 is a different claude-cli live-session startup failure cross-reference rather than a fix.

## Best Possible Solution

Keep the issue open and fix the CLI runtime path. The best implementation is to make the CLI no-output watchdog aware of a pending Monitor/background-task wait state, or emit a non-semantic watchdog heartbeat only while such a tool is genuinely pending. Add regression coverage for a claude-cli live-session turn that remains silent past the watchdog window because it is waiting on a long-running Monitor/background exec task, and verify the session is not killed while the overall task timeout still applies.
