## Summary

Keep #71803 open. Current main still has a finite CLI no-output watchdog, passes it into the CLI process supervisor, and treats a silent CLI as a timeout. I found no Monitor/pending-tool expected-idle exemption or heartbeat path; the related #40982/#41022 cap work is related but would not fully solve this Monitor-specific failure mode.

## Best Possible Solution

Keep the issue open and fix the CLI watchdog behavior directly. The maintainer path is to make both the normal CLI runner and Claude live-session path aware of expected idle while a long-running Monitor/tool wait is pending, or emit a narrowly scoped heartbeat from that state, while preserving no-output protection for truly stuck or interactive CLI stalls. Add regression coverage for a pending Monitor/tool wait with no stdout longer than the watchdog, plus a separate silent-stall timeout test.
