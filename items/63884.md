## Summary

Current main still does not implement Feishu per-bot context isolation. `requireMention` gates replies, but unmentioned group messages are still buffered into the shared group/session history and injected into a later handled turn; the Feishu schema and docs still lack `contextIsolation`, `contextPolicy`, `excludeOtherBots`, or an equivalent mention-only history option. The related open PR #40782 targets union_id mention detection, not context-recording isolation.

## Best Possible Solution

Keep this issue open. Implement a backwards-compatible Feishu plugin context policy, likely global and per-group, defaulting to the current behavior. When enabled, `extensions/feishu/src/bot.ts` should avoid recording unmentioned group messages into pending history and should filter messages addressed to other bots once reliable multi-bot mention identity is available. Update `extensions/feishu/src/config-schema.ts`, generated config metadata if required, `docs/channels/feishu.md`, and Feishu tests covering unmentioned pending-history replay plus multi-bot mention handling. Review #40782 separately as prerequisite or adjacent mention-identity work, not as the fix for this issue.
