## Summary

Keep open. Current `main` still records non-mentioned Feishu group messages into pending history, and the existing Feishu isolation controls are sender/topic scoped or allowlist-based, not per-bot mention-only context isolation.

## Evidence

- **Current Feishu handler still buffers non-mentioned group messages into history:** When `requireMention` is true and the bot was not mentioned, the Feishu handler logs the miss and calls `recordPendingHistoryEntryIfEnabled(...)` for the group history key before returning. That matches the issue's reported context pollution rather than fixing it.
  - file: [extensions/feishu/src/bot.ts:474](https://github.com/openclaw/openclaw/blob/5b8bd6371c666a5de43c82305e0315771e8e49d5/extensions/feishu/src/bot.ts#L474)
  - command: `nl -ba extensions/feishu/src/bot.ts | sed -n '450,495p'`
  - sha: [5b8bd6371c66](https://github.com/openclaw/openclaw/commit/5b8bd6371c666a5de43c82305e0315771e8e49d5)
- **Docs still describe mention-gated group messages as stored for context:** The shared group-chat docs explicitly say `requireMention? yes -> mentioned? no -> store for context only`, which is the behavior the reporter wants changed for Feishu multi-bot groups.
  - file: [docs/channels/groups.md:33](https://github.com/openclaw/openclaw/blob/5b8bd6371c666a5de43c82305e0315771e8e49d5/docs/channels/groups.md#L33)
  - command: `nl -ba docs/channels/groups.md | sed -n '24,42p'`
  - sha: [5b8bd6371c66](https://github.com/openclaw/openclaw/commit/5b8bd6371c666a5de43c82305e0315771e8e49d5)
- **Feishu config exposes session-scope knobs, not mention-only context isolation:** The Feishu config schema currently offers `groupSessionScope` values `group`, `group_sender`, `group_topic`, and `group_topic_sender`. There is no `contextIsolation`, `contextPolicy`, or equivalent mention-only history knob in the Feishu group schema here.
  - file: [extensions/feishu/src/config-schema.ts:107](https://github.com/openclaw/openclaw/blob/5b8bd6371c666a5de43c82305e0315771e8e49d5/extensions/feishu/src/config-schema.ts#L107)
  - command: `nl -ba extensions/feishu/src/config-schema.ts | sed -n '100,150p'`
  - sha: [5b8bd6371c66](https://github.com/openclaw/openclaw/commit/5b8bd6371c666a5de43c82305e0315771e8e49d5)
- **Existing tests cover sender/topic session isolation only:** Feishu tests verify sender-scoped and topic+sender-scoped routing (`oc-group:sender:...` and `oc-group:topic:...:sender:...`). Those features isolate by sender/topic, but they do not prevent the same sender's messages to another bot from entering this bot's pending history when the bot was not mentioned.
  - file: [extensions/feishu/src/bot.test.ts:1920](https://github.com/openclaw/openclaw/blob/5b8bd6371c666a5de43c82305e0315771e8e49d5/extensions/feishu/src/bot.test.ts#L1920)
  - command: `nl -ba extensions/feishu/src/bot.test.ts | sed -n '1918,1998p'`
  - sha: [5b8bd6371c66](https://github.com/openclaw/openclaw/commit/5b8bd6371c666a5de43c82305e0315771e8e49d5)
- **Existing context filtering is allowlist-based, not mention-target-based:** Feishu tests also cover `contextVisibility: "allowlist"` plus `groupSenderAllowFrom`, which filters fetched thread bootstrap context by allowed senders. That is useful hardening, but it is a different control from per-bot mention-only recording in shared groups.
  - file: [extensions/feishu/src/bot.test.ts:2622](https://github.com/openclaw/openclaw/blob/5b8bd6371c666a5de43c82305e0315771e8e49d5/extensions/feishu/src/bot.test.ts#L2622)

## Risks / Open Questions

- `groupSessionScope` may look like a workaround, but it isolates by sender/topic rather than by which bot was mentioned.
- `contextVisibility` and sender allowlists reduce supplemental context exposure, but they do not stop non-mentioned turns from being buffered into pending history for the bot.
