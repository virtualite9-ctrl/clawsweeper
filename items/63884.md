## Summary

Current main still does not implement the requested Feishu per-bot context isolation. `requireMention` gates replies, but skipped group messages are intentionally stored as pending history and later injected into the next handled group turn. The Feishu schema/docs still expose only reply mention gating and group/session history controls, not a mention-only context recording policy. The related open PR #40782 is adjacent Feishu mention-identity work, not a fix for context recording isolation.

## Best Possible Solution

Keep this issue open. Implement a backwards-compatible Feishu plugin context policy, probably available globally and per group, defaulting to current behavior. When enabled, `extensions/feishu/src/bot.ts` should not record unmentioned group messages into pending history and should filter messages addressed to other bots once mention identity is reliable. Update `extensions/feishu/src/config-schema.ts`, generated config metadata if required, `docs/channels/feishu.md`, and Feishu regression tests covering unmentioned pending-history replay plus multi-bot mention handling. Review #40782 separately as adjacent mention-identity work, not as the fix for this issue.
