## Summary

Keep this open. Current main still treats Feishu `requireMention` as a reply gate while buffering unmentioned group messages into pending history, and it has no Feishu config/docs/API surface for mention-only context recording or excluding other bots' conversations. The related #40782 mention-identity PR is relevant prerequisite work, but it does not supersede this context-isolation feature.

## Best Possible Solution

Keep this issue open and implement a backwards-compatible Feishu plugin context policy, probably top-level and per-group, defaulting to today's behavior. When enabled, `extensions/feishu/src/bot.ts` should avoid recording unmentioned group messages into pending history and should filter messages addressed to other bots once multi-bot mention identity is reliable. Update `extensions/feishu/src/config-schema.ts`, generated config metadata if applicable, `docs/channels/feishu.md`, and Feishu tests that exercise unmentioned pending-history replay and multi-bot mention handling.
