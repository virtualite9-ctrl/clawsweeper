## Summary

Current main does not implement the requested Feishu per-bot context isolation. Feishu still treats `requireMention` as a reply gate: unmentioned group messages are skipped for reply, but they are recorded into pending group history and later injected into the next mentioned turn. The config schema and Feishu docs have no `contextIsolation`, `contextPolicy`, `excludeOtherBots`, or mention-only history policy, and the related open #40782 mention-identity PR is prerequisite work rather than a replacement for this feature.

## Best Possible Solution

Keep this issue open. Implement a backwards-compatible Feishu plugin context policy, likely global and per-group, defaulting to today’s behavior. When enabled, `extensions/feishu/src/bot.ts` should avoid recording unmentioned group messages into pending history and should filter messages addressed to other bots once reliable multi-bot mention identity is available. Update `extensions/feishu/src/config-schema.ts`, generated config metadata if required, `docs/channels/feishu.md`, and Feishu tests for unmentioned pending-history replay plus multi-bot mention handling.
