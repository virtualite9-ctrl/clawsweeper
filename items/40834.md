## Summary

Current `main` already fixes the reported bundled-plugin dependency failure mode in a stronger form than this PR: active bundled plugins with missing runtime dependencies are repaired before import, so they load instead of failing with the raw loader error this PR targets. The released `v2026.4.22` tree already contains that behavior and matching regression coverage.

## Evidence

- **Loader repairs bundled runtime deps before import:** `loadOpenClawPlugins` now calls `ensureBundledPluginRuntimeDeps(...)` for enabled bundled plugins before module import, and only proceeds after dependency repair/mirroring. This is the code path that eliminates the raw import failure the PR describes.

## Risks / Open Questions

- Current main solves the problem by auto-repairing bundled runtime deps, not by surfacing the exact stable/beta install-hint UX proposed in this PR. If maintainers specifically want that narrower messaging instead of the shipped auto-repair behavior, this close would be too aggressive. The underlying bug report, however, is already addressed on main.
