## Summary

Keep PR #71354 open. Current `main` still has the reported root cause: status scanning resolves `tailscaleDnsPromise` to `null` whenever `gateway.tailscale.mode` is `off`, so the runtime Tailscale hostname is never looked up for the default config. The PR is recent, focused, linked to the concrete bug report #71123, and the author addressed the cold-start review feedback in a follow-up commit. It is not implemented on `main` or superseded, though it likely needs a targeted regression test and a check that normal `openclaw status` actually opts into showing DNS while mode is off.

## Best Possible Solution

Keep the PR open and ask for a small completion pass: preserve the cold-start skip guard, add bootstrap regression coverage for off-mode runtime lookup, verify or wire normal `openclaw status` to show `off · <tailnet host>`, and let maintainers decide whether the expanded Tailscale probe needs binary-path hardening or privacy wording.
