## Summary

Keep this PR open. Current main still short-circuits Tailscale hostname lookup whenever `gateway.tailscale.mode` is `off`, so the reported status behavior is not implemented on main. The PR is a focused change against the implicated bootstrap path, and the author addressed the cold-start regression feedback, but it still needs maintainer judgment on the privacy/PATH tradeoff plus targeted bootstrap regression coverage.

## Best Possible Solution

Keep the PR open. The best path is to merge or adapt the focused bootstrap change after adding targeted regression tests for mode-off runtime detection and cold-start skipping, and after a maintainer explicitly accepts or revises the privacy/PATH behavior raised in review.
