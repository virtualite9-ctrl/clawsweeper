## Summary

Keep open. Current `main` still ships the core hardcode in `resolveSessionKey()`, and the repo now documents/workarounds around that behavior rather than removing it. The Discord ingress path is wrapped with `canonicalizeMainSessionAlias()`, but the underlying helper still returns `agent:main:*`, and at least one non-test production caller (`extensions/slack/src/monitor/context.ts`) still calls it directly. Latest tagged release `v2026.4.23` also still contains the same hardcoded helper, so there is not strong evidence this item is fully fixed on `main`.
