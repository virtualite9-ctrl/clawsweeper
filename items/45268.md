## Summary

Current main does not yet implement the requested `/status` behavior. The session usage writer can mark context totals stale, and related UI surfaces respect `totalTokensFresh=false`, but the shared `/status` formatter still synthesizes `inputTokens + outputTokens` without checking freshness, so the reported repro shape can still render a misleading concrete context total.

## Best Possible Solution

Keep #45268 open and land #45270 or an equivalent main-branch fix in `src/status/status-message.ts`: treat `totalTokensFresh === false` as unknown for context occupancy unless transcript-derived prompt usage supplies a fresh replacement, and add a regression test in `src/auto-reply/status.test.ts` for both stale-without-total and stale-with-explicit-total cases.
