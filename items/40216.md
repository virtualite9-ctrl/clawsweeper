## Summary

Keep this PR open. Current main has a shared CJK-aware estimator and uses it for most context-pruning text, but it has not fully absorbed this PR's broader scope: soft-trim eligibility still uses raw UTF-16 lengths, and assistant tool-call arguments still use raw JSON string length. The PR branch is stale and needs a careful rebase, but the remaining bug surface is real.

## Best Possible Solution

Keep this PR open or replace it with a rebased maintainer patch against `src/agents/pi-hooks/context-pruning/pruner.ts`. The best fix is to keep using the shared `src/utils/cjk-chars.ts` helper, add weighted estimation for assistant tool-call argument JSON, and make an explicit soft-trim design choice: either keep soft-trim thresholds in raw display characters, or move the gate and head/tail extraction to consistent weighted units with regression tests for CJK-heavy tool results.
