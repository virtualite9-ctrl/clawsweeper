## Summary

Keep this PR open. Current main has the shared CJK estimator and uses it for much of the active `pi-hooks` context-pruning path, but the PR's stated scope is not fully implemented on main: soft-trim eligibility still uses raw UTF-16 length and assistant tool-call argument JSON still uses raw `.length`. The PR also targets the removed `pi-extensions` path, so it needs a rebase or replacement patch rather than direct merge or closure.

## Best Possible Solution

Keep this PR open, or replace it with a rebased maintainer patch against `src/agents/pi-hooks/context-pruning/pruner.ts`. The best fix should keep the shared `src/utils/cjk-chars.ts` estimator, use weighted estimation for assistant tool-call argument JSON, make soft-trim eligibility and extraction units consistent, and add regression tests for CJK-heavy tool results and tool-call arguments.
