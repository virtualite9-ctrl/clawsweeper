## Summary

Keep open. Current `main` already has server-backed session persistence plus ways to browse/resume sessions, but it does not implement the specific requested ChatGPT-style conversation-history sidebar in the chat/dashboard UI.

## Evidence

- **Control UI documents session browsing support, not a chat sidebar:** The Control UI docs list a dedicated Sessions surface with `sessions.list` / `sessions.patch`, which shows session management exists as its own feature area rather than as a left chat-history sidebar.

## Risks / Open Questions

- The requested UX is specifically a left-side conversation-history sidebar similar to ChatGPT, and the current implementation is only a header dropdown plus a separate Sessions tab.
- This review was source-based only; no live browser repro was run because the task required a read-only checkout and `pnpm` is unavailable in this environment.
