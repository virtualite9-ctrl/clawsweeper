## Summary

Keep PR #71862 open. Current main at 4a3030df9efa72c44bab567a4390ac0c4876a73f still exposes before_model_resolve as provider/model-only, merges only those two fields, and selects runtime auth profiles from request/session state rather than hook output. The provided PR context shows #71862 is an active, focused implementation with tests/docs and review fixes; older overlapping PR #68621 is not a better canonical replacement.

## Best Possible Solution

Keep #71862 open as the current candidate implementation. Maintainers should consolidate it against older PR #68621, preserve the legacy auth-source fixes from the review discussion, then land the additive hook API, runner auth-selection behavior, SDK baseline, docs, and regression tests once validation is green.
