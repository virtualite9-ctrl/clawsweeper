## Summary

Keep PR #71862 open. Current main at df542f75a9c186001a3fbfbac9310791314893c1 still exposes `before_model_resolve` as provider/model-only and the embedded runner still selects auth profiles from request/session state rather than hook output. The PR remains the active candidate for the missing core plugin API change; older PR #68621 is overlapping predecessor work, not a better canonical replacement for closing #71862.

## Best Possible Solution

Keep #71862 open as the active candidate implementation. The best path is to add `authProfileOverride` to the typed `before_model_resolve` and legacy `before_agent_start` result contract, merge it with the same priority rules as provider/model overrides, consume it in runner auth-profile selection only when no explicit user pin is present, preserve legacy auto/user source semantics, keep SDK/docs/tests aligned, and then close older PR #68621 as superseded if #71862 lands.
