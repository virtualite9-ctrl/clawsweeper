## Summary

Keep open. Current main still supports `before_model_resolve` and legacy `before_agent_start` provider/model overrides only; it does not expose or apply hook-selected auth profile preference. PR #71862 is an active, focused core/plugin API change with discussion showing review findings were addressed, and the provided context indicates normal CI is green except a repeatedly unrelated private parity scenario. Older related PR #68621 overlaps, but the ClawSweeper record and provided PR discussion show #71862 carries newer legacy auth-source/compaction-marker fixes, so this is not the PR to close as duplicate.

## Best Possible Solution

Keep #71862 open for maintainer review as the current candidate implementation. The best path is to land one core/plugin API change that adds `authProfileOverride` to the hook result contract, merges it with the same priority semantics as provider/model overrides, applies it as an automatic preference without overriding user-pinned profiles, preserves legacy compaction-marked auto selections, and closes or supersedes the older overlapping PR once maintainers choose the branch.
