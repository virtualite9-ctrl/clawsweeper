## Summary

Keep open. Current main still has no `models.pricing.enabled` or equivalent disable surface, and normal gateway startup still schedules the model-pricing refresh. The refresh still attempts both OpenRouter and LiteLLM catalog fetches with a 60s timeout. The linked open PR #68651 remains the canonical implementation candidate and uses closing syntax, so this issue should stay open until that PR is reviewed and merged or replaced.

## Best Possible Solution

Keep this issue open and review #68651 as the canonical fix candidate. A complete solution should add a validated `models.pricing.enabled` config key with default behavior preserved, skip all pricing bootstrap/refresh network work when disabled, update schema/help/docs/generated baselines, and add focused tests proving disabled pricing does not fetch OpenRouter or LiteLLM catalogs.
