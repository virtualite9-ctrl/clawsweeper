## Summary

Keep open. Current main still starts the model-pricing refresh for normal gateway startup, still fetches OpenRouter and LiteLLM pricing catalogs with a 60s timeout, and does not expose a `models.pricing.enabled` config surface. The linked open PR #68651 explicitly uses closing syntax for this issue and proposes the missing disable switch, so this issue should remain open until that PR is reviewed/merged or closed.

## Best Possible Solution

Review #68651 as the canonical implementation candidate. A solid fix should add a validated `models.pricing.enabled` config key, document it, preserve default pricing behavior, skip all bootstrap/refresh network work when disabled, and include focused gateway/config tests. The separate diagnostics PR #72033 may be useful for observability, but it does not replace the disable/non-blocking startup fix requested here.
