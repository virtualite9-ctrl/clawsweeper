## Summary

Keep open. Current `main` contains a partial mitigation shipped in `v2026.4.22` for this area, but not a full fix for the reported timeout behavior. Pricing refresh still starts automatically for non-test gateways, still uses 30s upstream fetch timeouts, and the current tests explicitly expect timeout warnings. The code changed the failure mode from a top-level bootstrap failure to asynchronous per-source warnings, which weakens the original repro, but does not provide strong enough evidence that the underlying issue is resolved.
