## Summary

Keep open. Current main still starts model-pricing refresh during normal gateway startup, still fetches OpenRouter and LiteLLM pricing catalogs with 60s timeouts when external pricing refs remain, and still has no `models.pricing.enabled` or equivalent config gate. The linked open PR #68651 is the direct implementation candidate, so this issue should remain open until that PR is reviewed and either merged, replaced, or closed.

## Best Possible Solution

Keep this issue open and review #68651 as the canonical fix candidate. A complete core fix should add a validated `models.pricing.enabled` config key defaulting to current behavior, skip all pricing bootstrap/catalog network work when disabled, update schema/help/labels/docs/generated baselines and changelog as needed, and add focused tests proving disabled pricing does not fetch OpenRouter or LiteLLM while default/enabled behavior still works.
