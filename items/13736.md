## Summary

Keep #13736 open. Current main still only supports compaction under `agents.defaults.compaction`; `agents.list[]` entries are typed and strictly schematized without `compaction`, resolver/runtime paths read defaults-only config, and docs/help/tests only cover defaults. PR #69988 is the active candidate path but is open and unmerged, so the feature is not shipped on main.

## Best Possible Solution

Keep #13736 open until the feature lands or maintainers explicitly designate a canonical replacement. The implementation should add optional `agents.list[].compaction` to `AgentConfig` and the strict schema, reuse the defaults compaction schema, deep-merge per-agent values over `agents.defaults.compaction` including nested `memoryFlush`, thread the effective agent-specific config through Pi reserve-token setup, safeguard runtime, memory-flush/preflight paths, and update docs/help/generated schema plus focused resolver/schema/runtime tests. PR #69988 is the current candidate path but is not merged on current main.
