## Summary

Keep open. Current main still supports compaction only through `agents.defaults.compaction`; `agents.list[]` has no per-agent `compaction` type, strict schema field, resolver merge, or runtime handoff. The issue discussion includes useful prior implementation attempts (#14598 and #19329), but both are closed unmerged, and the latest maintainer/Codex comment already treated #13736 as the older/root request rather than a closeable duplicate.

## Best Possible Solution

Keep #13736 open as the root feature request until a real implementation lands. The best implementation should add optional `agents.list[].compaction` typing and strict-schema validation, reuse/extract the defaults compaction schema, resolve effective per-agent compaction by deep-merging over `agents.defaults.compaction` including nested `memoryFlush`, thread the resolved agent config through reserve-token, safeguard, timeout, model, memory-flush, and compaction runtime paths, then add focused schema/resolver/runtime tests and config docs. Existing closed PRs #14598 and #19329, plus any active follow-up PR, should be treated as prior art rather than closure evidence.
