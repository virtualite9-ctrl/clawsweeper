## Summary

Keep #13736 open. Current main still does not support per-agent compaction overrides in `agents.list[]`: the agent entry type and strict schema do not accept `compaction`, resolved agent config does not merge or expose it, runtime compaction and memory-flush paths still read `agents.defaults.compaction`, and docs describe defaults-only compaction. Related PRs #14598 and #19329 were closed unmerged; #69988 appears to be the active implementation path, but it is not merged into current main.

## Best Possible Solution

Keep #13736 open unless maintainers explicitly choose #69988 as the canonical tracker. The product/code path should add optional `agents.list[].compaction` typing and strict schema support, reuse or extract the defaults compaction schema, deep-merge per-agent compaction over `agents.defaults.compaction` including nested `memoryFlush`, thread the effective agent-specific compaction through reserve-token, safeguard, timeout/model, memory-flush, and embedded runtime setup paths, then add focused schema/resolver/runtime tests and update config docs. Treat #14598 and #19329 as prior implementation work and #69988 as the active candidate PR, not as shipped closure evidence.
