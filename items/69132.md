## Summary

Keep #69132 open. Current main no longer calls the old `/api/experimental/web_search` path, but it still sends a single request to the configured/default Ollama host at `/api/web_search` and fails immediately on non-OK responses. That does not solve the reported case, where both local `/api/experimental/web_search` and local `/api/web_search` returned 404 on Ollama 0.16.0. The linked #69222 remains the concrete open fix path for local endpoint fallback plus Ollama Cloud retry behavior; the closed candidate PRs did not land.

## Best Possible Solution

Keep the issue open until OpenClaw either lands #69222 or an equivalent maintainer-approved fix that handles Ollama 0.16+ local 404s. The implementation should add explicit fallback behavior, token scoping between local and cloud targets, regression tests for local 404 plus cloud retry, and docs/changelog updates for the supported Ollama Web Search routing contract.
