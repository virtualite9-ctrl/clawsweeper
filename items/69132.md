## Summary

Keep open. Current main no longer calls `/api/experimental/web_search`, but it still defaults Ollama Web Search to the configured/local Ollama host and a single `/api/web_search` request. That does not fully cover the reported environment, where both local `/api/experimental/web_search` and `/api/web_search` returned 404 and the working path was a cloud retry with `OLLAMA_API_KEY`. The linked open PR #69222 remains the closest concrete fix path for the local-to-cloud fallback behavior.

## Best Possible Solution

Keep #69132 open as the canonical bug until a maintainer lands #69222 or an equivalent fix that either implements the local-to-cloud fallback with correct credential scoping, or intentionally updates product/docs to make `https://ollama.com` plus `OLLAMA_API_KEY` the supported Ollama Web Search path.
