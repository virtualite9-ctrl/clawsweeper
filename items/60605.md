## Summary

Keep open. Current `main` does not implement this wrapper, and the PR record still shows unresolved security/correctness concerns. The strongest remaining issue is that `clawhub` installs into the current/workspace-selected `skills` directory, while the proposed wrapper scans and removes a fixed `$OPENCLAW_STATE_DIR/workspace/skills` path, so it can miss the actual install target.

## Evidence

- **No implementation on main:** A repo search on current `main` found no `clawhub-safe-install` script and no `cisco-ai-skill-scanner` integration, so this is not already implemented on `main`.
