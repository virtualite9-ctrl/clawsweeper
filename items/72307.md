## Summary

Current main fixes the reported v2026.4.24 Bonjour/CIAO `PROBING CANCELLED` crash loop. The bundled Bonjour plugin now wires scoped process-error handlers into the advertiser, the global unhandled-rejection path honors those handlers before exiting, CIAO probing/announcement cancellations are classified as handled, and stuck probing/announcing restarts are bounded so Bonjour disables itself for the current Gateway process instead of crash-looping. This matches the same regression cluster already reviewed in #72103 and #72152.

## Best Possible Solution

Close as implemented on current main. Keep the scoped Bonjour/CIAO process-error handling, shared unhandled-rejection registry, bounded advertiser recovery, and docs/workaround. Release/backport timing should be handled through normal maintainer release work rather than leaving this duplicate core bug open.
