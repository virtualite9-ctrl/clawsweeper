## Summary

Keep this PR open. Current main still collapses channel-account config edits into full channel restarts, while the underlying channel manager already supports account-scoped start/stop. The PR is a plausible focused implementation for the linked open issue #43935, but it is stale against current main and should be rebased so it preserves newer reload-plan fields and channel-reload deferral behavior.

## Best Possible Solution

Rebase the PR onto current main and land an adapted account-scoped reload implementation: add an account-scoped restart representation to `GatewayReloadPlan`, parse strictly account-scoped `channels.<channel>.accounts.<accountId>.*` changes, let channel-global or mixed changes keep full-channel precedence, preserve skip-env and active-work deferral behavior, and keep focused planner plus handler regression tests.
