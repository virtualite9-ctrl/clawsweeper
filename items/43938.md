## Summary

Keep this PR open. Current main still plans channel account config changes as full-channel reloads, while account-scoped channel lifecycle already exists. The PR is a plausible implementation candidate for the linked open issue #43935, but it needs rebase/adaptation to current reload-plan fields and deferral behavior before landing.

## Best Possible Solution

Rebase and adapt this PR onto current main, then land the account-scoped channel reload implementation if it preserves current reload deferral and newer plan fields. The implementation should add an account-scoped restart representation to `GatewayReloadPlan`, parse only strict `channels.<channel>.accounts.<accountId>.*` paths, keep full-channel precedence for channel-global or mixed changes, thread `accountId` through the hot-reload handler, and add focused planner plus handler regression tests.
