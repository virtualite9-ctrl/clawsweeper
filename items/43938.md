## Summary

Keep this PR open. Current main still routes `channels.<channel>.accounts.<accountId>.*` changes through the broad channel restart plan and handler, while the lower channel manager already supports account-scoped start/stop. The PR is also paired with the same author's open issue #43935 via closing syntax, so it remains an implementation candidate rather than cleanup-close material.

## Best Possible Solution

Rebase and adapt this PR onto current main, then land it if the account-scoped reload plan preserves full-channel precedence for global or mixed changes, threads `accountId` through the hot reload handler, and adds focused planner plus handler regression tests.
