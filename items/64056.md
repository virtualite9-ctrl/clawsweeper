## Summary

Current `main` no longer matches the reported raw-send bypass: subagent completion announces are routed through the gateway `agent` delivery path, and Telegram native exec approvals send via `sendMessageTelegram`, which chunks oversized Telegram text/HTML to the platform limit.

## Evidence

- **Subagent completion announce uses gateway agent delivery:** `sendSubagentAnnounceDirectly` now calls the gateway `agent` method with `deliver`, channel, target, thread, and idempotency metadata instead of calling Telegram directly.
  - file: [src/agents/subagent-announce-delivery.ts:574](https://github.com/openclaw/openclaw/blob/5b8bd6371c666a5de43c82305e0315771e8e49d5/src/agents/subagent-announce-delivery.ts#L574)
  - command: `nl -ba src/agents/subagent-announce-delivery.ts | sed -n '568,608p'`
  - sha: [5b8bd6371c66](https://github.com/openclaw/openclaw/commit/5b8bd6371c666a5de43c82305e0315771e8e49d5)
- **Gateway agent path resolves normal outbound delivery:** The gateway `agent` handler builds a normal delivery plan for outbound replies, so subagent completion messages flowing through `agent` inherit the same delivery machinery as ordinary replies.
  - file: [src/gateway/server-methods/agent.ts:783](https://github.com/openclaw/openclaw/blob/5b8bd6371c666a5de43c82305e0315771e8e49d5/src/gateway/server-methods/agent.ts#L783)

  - sha: [5b8bd6371c66](https://github.com/openclaw/openclaw/commit/5b8bd6371c666a5de43c82305e0315771e8e49d5)
- **Outbound delivery still applies text chunking:** `deliverOutboundPayloads` resolves a channel chunker/text limit and splits text before send when the channel provides one.
  - file: [src/infra/outbound/deliver.ts:770](https://github.com/openclaw/openclaw/blob/5b8bd6371c666a5de43c82305e0315771e8e49d5/src/infra/outbound/deliver.ts#L770)

  - sha: [5b8bd6371c66](https://github.com/openclaw/openclaw/commit/5b8bd6371c666a5de43c82305e0315771e8e49d5)
- **Telegram exec approvals now use chunk-aware send helper:** Telegram native approval delivery sends pending approval text through `sendMessageTelegram`, not a raw `bot.api.sendMessage` call.
  - file: [extensions/telegram/src/approval-handler.runtime.ts:145](https://github.com/openclaw/openclaw/blob/5b8bd6371c666a5de43c82305e0315771e8e49d5/extensions/telegram/src/approval-handler.runtime.ts#L145)
