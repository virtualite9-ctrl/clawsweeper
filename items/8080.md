## Summary

Keep open. Current `main` already auto-sanitizes `read` tool image payloads and lets users tune max image dimension, but it does not expose the requested user-configurable `maxBytes` or quality controls for `Read`; the byte cap remains a hardcoded default.

## Evidence

- **Read tool always sanitizes image results:** `createOpenClawReadTool()` wraps every `read` execution, normalizes image results, then calls `sanitizeToolResultImages(..., options?.imageSanitization)` before returning.
  - file: [src/agents/pi-tools.read.ts:674](https://github.com/openclaw/openclaw/blob/b2cface9d5fd048a514d82292a5baee7190e0ee3/src/agents/pi-tools.read.ts#L674)
  - command: `nl -ba src/agents/pi-tools.read.ts | sed -n '674,697p'`
  - sha: [b2cface9d5fd](https://github.com/openclaw/openclaw/commit/b2cface9d5fd048a514d82292a5baee7190e0ee3)
- **Sanitizer already resizes and recompresses images:** `sanitizeContentBlocksImages()` downscales oversized images and converts them to JPEG when needed, enforcing both dimension and byte limits during tool-result sanitization.
  - file: [src/agents/tool-images.ts:269](https://github.com/openclaw/openclaw/blob/b2cface9d5fd048a514d82292a5baee7190e0ee3/src/agents/tool-images.ts#L269)
  - command: `nl -ba src/agents/tool-images.ts | sed -n '269,361p'`
  - sha: [b2cface9d5fd](https://github.com/openclaw/openclaw/commit/b2cface9d5fd048a514d82292a5baee7190e0ee3)
- **Byte cap is still hardcoded by default:** The image sanitization defaults are `DEFAULT_IMAGE_MAX_DIMENSION_PX = 1200` and `DEFAULT_IMAGE_MAX_BYTES = 5 * 1024 * 1024`; `resolveImageSanitizationLimits()` only reads `agents.defaults.imageMaxDimensionPx` from config and does not load a user-configurable byte or quality setting.
  - file: [src/agents/image-sanitization.ts:3](https://github.com/openclaw/openclaw/blob/b2cface9d5fd048a514d82292a5baee7190e0ee3/src/agents/image-sanitization.ts#L3)

- **User-facing docs expose only dimension tuning:** The configuration reference documents `agents.defaults.imageMaxDimensionPx` as the knob for transcript/tool image downscaling; there is no documented `Read` image max-bytes or quality setting alongside it.
  - file: [docs/gateway/configuration-reference.md:1125](https://github.com/openclaw/openclaw/blob/b2cface9d5fd048a514d82292a5baee7190e0ee3/docs/gateway/configuration-reference.md#L1125)
