## Summary

Keep open. Current main still only recognizes camelCase tool-call block types in the session transcript repair path, so a malformed Anthropic-style `tool_use` block can pass through unchanged. Related PR #50802 appears to be the active fix attempt, but the fix is not on `main`.

## Best Possible Solution

Keep #48915 open until a fix lands on main. The concrete maintainer path is to broaden the tool-call type recognition in `src/agents/session-transcript-repair.ts` and the adjacent shared extraction logic in `src/agents/tool-call-id.ts` to include `tool_use`, `tool_call`, and `function_call`, then add regression tests proving that a missing-input snake_case `tool_use` is dropped while valid snake_case calls are preserved or repaired consistently. PR #50802 is the relevant open fix attempt to review or supersede.
