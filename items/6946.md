## Summary

Keep open. Current main still does not implement the requested Telegram standalone processing-indicator message lifecycle. The bundled Telegram plugin has adjacent feedback mechanisms, including ack/status reactions, typing/preview progress, and native command progress placeholders, but there is still no configurable normal inbound-turn placeholder message that is sent immediately, deleted after a successful final reply, and edited on failure. The two related implementation PRs were closed unmerged, so this issue remains the canonical tracker for the remaining Telegram behavior.

## Best Possible Solution

Keep #6946 open. The best path is an opt-in implementation in the bundled Telegram plugin: add typed config/schema/generated metadata/docs, send a standalone placeholder after an eligible inbound Telegram message is accepted, carry its Telegram message id through dispatch, delete it after successful final delivery, and edit or replace it on dispatch failure. Reuse native-command progress placeholders and preview cleanup as prior art, with focused Telegram dispatch tests for success, error, preview streaming, group/topic, and cleanup behavior.
