## Summary

Keep open. Current main still does not implement the requested Telegram standalone processing-indicator message lifecycle. Telegram has adjacent feedback via ack/status reactions, preview-streaming tool-progress text, and native-command progress placeholders, but there is no configurable `⌛️` message sent for normal inbound Telegram turns, deleted after a reply, and edited on failure. The related implementation PRs #43021 and #43720 were closed unmerged, and the latest issue discussion already records a maintainer/Codex review keeping this open.

## Best Possible Solution

Keep the issue open. The best path is an opt-in implementation in the bundled Telegram plugin: add typed config/schema/generated metadata/docs, send a standalone placeholder after an eligible inbound Telegram message is accepted, carry its message id through dispatch, delete it after successful final delivery, and edit or replace it on dispatch failure. Reuse native-command progress placeholders and preview cleanup as prior art, with focused Telegram dispatch tests for success, error, preview-streaming, group/topic, and cleanup behavior.
