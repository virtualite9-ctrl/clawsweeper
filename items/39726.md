## Summary

Keep PR #39726 open. Current main still requires `notifyOnExitEmptySuccess` to be explicitly true, so chat-context empty successful background execs remain silent by default. The PR is not implemented or superseded on main, and its latest head appears to address the review concerns captured in the PR discussion.

## Best Possible Solution

Keep PR #39726 open or rebase/rework it onto current main. The best implementation is the PR's narrowed behavior: default `notifyOnExitEmptySuccess` from a real reply-capable chat channel only when the config value is unset, preserve explicit `false`, keep generic provider labels such as `openai` and `heartbeat` silent, add the regression cases, and update changelog/docs if maintainers accept the product behavior.
