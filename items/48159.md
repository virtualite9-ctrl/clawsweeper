## Summary

Keep #48159 open. Current main does not implement a per-session secure/ephemeral mode: sessions still persist metadata and JSONL transcripts, session create/patch schemas have no ephemeral flag, transcript writes have no hard guard, and context-engine/plugin persistence surfaces receive session files but no `isEphemeral` contract. The related #62436 is narrower HTTP passthrough work and does not supersede this channel-wide/session-wide privacy request.

## Best Possible Solution

Keep the issue open and treat it as a core session privacy/persistence contract. The likely implementation should add a session/config/channel default flag, hard gates around transcript and session-store persistence, compaction/memory-flush/session-memory indexing skips, a plugin/context-engine `isEphemeral` or equivalent runtime contract, status/inspection visibility, and focused tests. #62436 can share lower-level no-persistence plumbing for HTTP passthrough, but it should not replace this broader per-session secure chat work.
