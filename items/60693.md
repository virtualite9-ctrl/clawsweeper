## Summary

Keep open. Current `main` appears to have addressed parts of the report: Discord startup now injects a proxy-capable REST client for Carbon-backed Discord REST calls, model-precedence docs now explain the default/per-agent/cron layering, and the changelog records an ACPX registry fix that pinned versioned `npx` built-ins. But the checkout still shows `acp.allowedAgents` as a plain normalized allowlist with no validation against installed or enabled ACP agents, while startup identity reconcile still logs generic failures when runtime session resolution blows up. Because this issue bundles multiple topics and at least the disabled-agent/allowlist failure mode is not clearly fixed on `main`, closing would be premature.
