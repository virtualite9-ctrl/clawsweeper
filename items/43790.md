## Summary

Keep this PR open. Current main still does not implement the central change: the message tool cannot accept a trusted explicit `agentId`, and outbound message action routing is still derived from `agentSessionKey`. The PR remains a focused bug-fix candidate rather than cleanup-close material, with one maintainer-review point around ensuring only trusted runtime-provided agent IDs override session-derived routing.

## Best Possible Solution

Keep this PR open and review or rebase a guarded version onto current main: thread a trusted explicit `agentId` through `createOpenClawCodingTools` -> `createOpenClawTools` -> `createMessageTool`, prefer it over session-derived routing only when supplied by runtime code, preserve config-aware fallback behavior for opaque/non-agent session keys, add focused tests for both explicit override and default-agent fallback, and fix the changelog attribution before landing.
