## Summary

Keep open. Current main still does not implement the central routing fix: `createMessageTool` has no explicit `agentId` option, derives the routed agent from `agentSessionKey`, and `createOpenClawCodingTools` does not thread its resolved active agent into the message tool. The PR discussion also shows a real fallback hazard was reviewed and the author addressed it in PR head, so this is still useful maintainer work rather than cleanup-close material.

## Best Possible Solution

Keep this PR open and rebase a guarded version onto current main: add an explicit `agentId` path through `createOpenClawCodingTools` -> `createOpenClawTools` -> `createMessageTool`, prefer it only when it is truly explicit, keep fallback routing config-aware via `resolveSessionAgentId` after config is available, add focused tests for explicit override and opaque-session default-agent behavior, and fix the changelog attribution if that entry remains.
