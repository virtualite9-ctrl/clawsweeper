## Summary

Close as superseded by the active canonical PR #69822. Current main does not yet implement the requested socket.drain eviction path: session mutations still emit sessions.changed only to session event subscribers with dropIfSlow enabled, socket.drain is not registered as a gateway event/scope, and message-only subscribers are only included for transcript session.message updates. The issue discussion itself moved from #63378 to #65539 and then to the currently open #69822, whose body tracks the same remaining work in focused form.

## Best Possible Solution

Use #69822 as the canonical review surface for this work. The product/code path should be decided there: if maintainers accept the feature, land a focused PR that registers socket.drain, routes teardown events to both session event and message subscribers, disables slow-consumer dropping only for critical drains, and locks it with gateway tests. Keep #63374 closed as superseded to avoid parallel tracking.
