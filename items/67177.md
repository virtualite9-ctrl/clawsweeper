## Summary

Current main still matches the reported MS Teams bug. The `file.download.info` path returns `content.downloadUrl` directly, while the existing Graph shares rewrite only runs for `contentUrl` attachments. The prior logging concern has improved, but the downloadUrl rewrite and regression coverage are still missing.

## Best Possible Solution

Keep this issue open. Patch `extensions/msteams/src/attachments/download.ts` so the `file.download.info` branch applies `tryBuildGraphSharesUrlForSharedLink(downloadUrl)` and downloads through the Graph shares URL when present. Add a focused regression test where `content.downloadUrl` is a SharePoint/OneDrive URL and only the Graph shares endpoint succeeds with a Graph-scoped auth retry. Preserve the existing warning logger behavior for failed downloads.
