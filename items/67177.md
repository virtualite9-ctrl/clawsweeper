## Summary

Current main still leaves the central MS Teams bug unfixed: `file.download.info` attachments return `content.downloadUrl` directly, while the Graph shares rewrite only applies to the later `contentUrl` path. The logging concern has improved, but the SharePoint/OneDrive `downloadUrl` regression still needs a code fix and regression test.

## Best Possible Solution

Patch `extensions/msteams/src/attachments/download.ts` so the `file.download.info` branch applies `tryBuildGraphSharesUrlForSharedLink(downloadUrl)`, uses the Graph shares URL when present, and keeps `contentTypeHint` behavior consistent with the existing `contentUrl` rewrite. Add a regression test where `content.downloadUrl` is a SharePoint URL and only the Graph shares endpoint is fetched with a Graph-scoped auth retry. Keep the current warning logger behavior for failed downloads.
