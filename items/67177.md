## Summary

Current main still leaves the reported MS Teams bug unfixed. The `file.download.info` branch returns `content.downloadUrl` directly, while the Graph shares rewrite only applies to the separate `contentUrl` branch. The logging part has improved, but the SharePoint/OneDrive `downloadUrl` regression still needs a targeted plugin fix and test.

## Best Possible Solution

Keep this issue open. Patch `extensions/msteams/src/attachments/download.ts` so the `file.download.info` branch applies `tryBuildGraphSharesUrlForSharedLink(downloadUrl)`, uses the Graph shares URL when present, and keeps the current `contentTypeHint` behavior consistent with the existing `contentUrl` rewrite. Add a regression test where `content.downloadUrl` is a SharePoint/OneDrive URL and only the Graph shares endpoint is fetched with a Graph-scoped auth retry. Keep the current warning logger behavior for failed downloads.
