## Summary

Keep open. Current main still suppresses `ANNOUNCE_SKIP` before the completion-delivery call, so the PR is not obsolete. The proposed behavior may be useful, but the submitted diff still needs maintainer review and a follow-up patch before merge.

## Best Possible Solution

Keep this PR open for maintainer review, but require a follow-up patch before merge: allow `ANNOUNCE_SKIP` to skip only the soft announce step when completion delivery is required, clear the control token before building `findings`, preserve `NO_REPLY` suppression, restore strict cleanup assertions, add positive e2e coverage for the actual delivery call, and update the public subagent/FAQ docs if maintainers accept the new semantics.
