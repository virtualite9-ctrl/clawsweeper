## Summary

Keep this PR open. Current main still suppresses `ANNOUNCE_SKIP` before the completion-delivery path, so the PR is not obsolete. However, the submitted diff is not ready to merge as-is because it lets the literal `ANNOUNCE_SKIP` control token fall through into completion `findings`, leaves contradictory e2e expectations around `ANNOUNCE_SKIP`, and weakens a cleanup assertion.

## Best Possible Solution

Keep this PR open for maintainer review and require a follow-up patch before merge: allow `ANNOUNCE_SKIP` to skip only the soft announce step when completion delivery is required, clear the control token before building `findings`, preserve `NO_REPLY` suppression, restore exact cleanup assertions, add positive e2e coverage for the actual delivery call, and update the FAQ/docs if maintainers accept the new semantics.
