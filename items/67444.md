## Summary

Keep this PR open. Current main still suppresses `ANNOUNCE_SKIP` before the completion-delivery path, so the PR targets a real remaining behavior difference. The submitted PR is not ready to close as obsolete or merge as-is because the proposed fallthrough can forward the literal `ANNOUNCE_SKIP` control token as completion content, and the tests/docs still need maintainer-aligned coverage for the semantic change.

## Best Possible Solution

Keep the PR open for maintainer review and require a follow-up patch before merge: allow `ANNOUNCE_SKIP` to skip only the soft announce step when completion delivery is required, clear the control token before building `findings`, preserve `NO_REPLY` suppression, reset test-injected dependencies, strengthen e2e coverage around the actual delivery call, and update the FAQ/docs if maintainers accept the new semantics.
