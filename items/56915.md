## Summary

Current main still does not discover a skill at `~/.openclaw/skills/<group>/<skill>/SKILL.md` by default. The loader still checks only immediate child directories for `SKILL.md`, and two open PRs explicitly target this issue with closing syntax, so the issue should remain open until a fix lands or the PRs are closed.

## Best Possible Solution

Keep this issue open and review one canonical fix PR that adds bounded grouped-skill discovery for `~/.openclaw/skills/<group>/<skill>/SKILL.md`, with regression coverage in `src/agents/skills.loadworkspaceskillentries.test.ts` and the existing containment and size-limit checks preserved.
