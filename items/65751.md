## Summary

Current main still routes the Slack setup manifest through the framed CLI note path, so the copy/paste bug remains. An open implementation candidate, #65773, explicitly links this issue and should be reviewed, landed, or superseded before this issue closes.

## Best Possible Solution

Keep this issue open and review/land or supersede #65773. The likely fix belongs in the Slack plugin setup surface: remove raw manifest JSON from introNote.lines, emit the manifest as copyable raw/plain output via an appropriate setup prepare/plain-output path, and add a focused regression test proving the CLI setup note no longer contains framed manifest JSON.
