## Summary

Keep this PR open. Current main still has the Copilot Claude replay gap the PR targets: Copilot Claude policy requests thinking-block dropping, but both session-history replay and tool-continuation replay still use the generic helper that preserves latest assistant thinking blocks. The PR is still a plausible implementation candidate, but it needs maintainer review for provider normalization and for the bundled ACP/guard-script allowlist changes.

## Best Possible Solution

Keep this PR open. The best path is to refresh it against current main, select the Copilot replay sanitizer from normalized provider or explicit replay-policy state, apply the same strategy to session-history replay, tool-continuation replay, and cache-TTL pruning estimates, update Copilot regression tests and transcript hygiene docs, and split or justify the ACP/guard-script allowlist changes before maintainer review.
