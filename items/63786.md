## Summary

Current main still does not implement the central Copilot-specific replay sanitizer in this PR. The PR remains a plausible fix for GitHub Copilot Claude multi-turn replay failures, but it should be refreshed and tightened before merge because the current diff also includes ACP/guard-script changes and still has review feedback around normalized provider selection.

## Best Possible Solution

Keep this PR open. Ask the author to refresh it against current main, add a Copilot-specific drop-all-plain-thinking replay strategy selected from normalized provider/policy state, apply the same strategy to session-history replay, tool-continuation replay, and context-pruning estimates, update the Copilot regression tests and transcript-hygiene docs, and split or explicitly justify the ACP/guard-script allowlist changes before merge.
