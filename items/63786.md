## Summary

PR #63786 should stay open. Current main still has the Copilot Claude replay gap the PR is trying to fix: the GitHub Copilot provider policy requests thinking-block dropping for Claude models, but both session-history replay and mid-turn stream continuations still call the generic helper that preserves latest assistant thinking blocks. The current tests also explicitly assert that GitHub Copilot preserves latest-turn thinking, so this is not already implemented on main. The PR is focused enough to continue, but should address the raw-provider normalization review and keep the ACP/script-guard pieces clearly justified or split if they are no longer part of the same fix.

## Best Possible Solution

Keep the PR open and ask for a corrected, focused version: add a Copilot-specific replay sanitizer selected from a normalized provider id, use it in both session-history replay and mid-turn stream wrapping, align context-pruning estimates with the same sanitizer, and keep regression tests proving Copilot drops latest-turn plain `thinking` while native Anthropic/Bedrock replay preserves provider-owned thinking. Treat the ACP/script-guard changes as separate unless the author can show they are required for this bug fix.
