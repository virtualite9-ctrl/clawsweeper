## Summary

Keep open. Current main still lacks the requested per-spawn `sessions_spawn` tool profile override: the public tool schema does not expose `tools`, `toolsAllow`, or `disableTools`, the native spawn params do not carry a tool policy, and the spawned child `agent` call cannot pass one through. Existing code has adjacent global subagent policy and runtime-level `toolsAllow`/`disableTools` support, plus improved gateway-bound plugin tool handling, but not the per-call API requested here.

## Best Possible Solution

Keep #23906 open until maintainers implement a backward-compatible per-spawn tool selection contract. A likely path is to add a `sessions_spawn` parameter such as `tools: "none" | string[]` or explicit `disableTools`/`toolsAllow`, validate it in `sessions-spawn-tool`, carry it through `SpawnSubagentParams`, extend the gateway `agent` schema/server method to accept internal tool policy fields for native subagent starts, map `"none"` to runner `disableTools: true`, map subsets to `toolsAllow`, and add tests/docs covering core tools, plugin-owned tools, bundle MCP names, and interactions with existing global/agent/subagent policy.
