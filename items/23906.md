## Summary

Keep #23906 open. Current main still has no per-call `sessions_spawn` tool override for `tools: "none"` or a named subset, and the related lower-level `disableTools`/`toolsAllow` seams are not wired through the public spawn schema, native subagent params, or gateway `agent` RPC. The overlap with open #15032 is real, but #23906 adds local-model tool-less spawn and plugin/MCP inheritance details, plus a same-day maintainer-side Codex review explicitly keeping it open.

## Best Possible Solution

Keep #23906 open until maintainers implement or explicitly reject a backward-compatible per-spawn tool selection contract. A concrete path is to add a `tools: "none" | string[]` or explicit `disableTools`/`toolsAllow` field to `sessions_spawn`, validate and document it, carry it through `SpawnSubagentParams`, extend the gateway `agent` schema/server ingress, map tool-less spawns to runner `disableTools: true`, map subsets to runner `toolsAllow`, and add tests for core tools, plugin/bundle MCP tools, global subagent policy precedence, sandbox/security behavior, and small local-model tool-less spawns. Keep #15032 as related, but do not close #23906 as duplicate unless maintainers choose one canonical tracking issue for both security isolation and local-model/plugin inheritance requirements.
