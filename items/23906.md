## Summary

Current main still does not expose a per-spawn `sessions_spawn` tool profile override. The tool schema, native spawn params, and gateway `agent` RPC schema cannot carry `tools`, `toolsAllow`, or `disableTools`; only global subagent policy and lower-level runner allowlist/disable machinery exist. The issue also has active local-model/plugin-MCP discussion and a same-day Codex/maintainer comment explicitly keeping it open, so it should not be closed as implemented or as a duplicate of #15032.

## Best Possible Solution

Keep #23906 open until maintainers implement or explicitly reject a backward-compatible per-spawn tool selection contract. A concrete path is to add a `sessions_spawn` parameter such as `tools: "none" | string[]` or explicit `disableTools`/`toolsAllow`, validate it in `sessions-spawn-tool`, carry it through `SpawnSubagentParams`, extend the gateway `agent` schema/server method, map tool-less spawns to runner `disableTools: true`, map subsets to `toolsAllow`, and add tests/docs for core tools, plugin/bundle MCP tool names, global subagent policy precedence, and sandbox/security behavior.
