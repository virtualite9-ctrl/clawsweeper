## Summary

Keep #23906 open. Current main still does not expose a per-call `sessions_spawn` tool selector, tool-less mode, or transport for `toolsAllow`/`disableTools`; only global/config subagent tool policy and lower-level runner primitives exist. The issue also has a recent maintainer review explicitly saying to keep it open.

## Best Possible Solution

Keep the issue open. A proper fix should add a backward-compatible public `sessions_spawn` tool policy field such as `tools: "none" | string[]` or `disableTools`/`toolsAllow`, validate and document it, carry it through `SpawnSubagentParams` and the gateway `agent` RPC, map tool-less spawns to embedded runner `disableTools: true`, map subsets to `toolsAllow`, and test core tools, plugin/MCP tools, global subagent policy precedence, sandbox/security behavior, and local-model tool-less spawns. Treat #15032 as related unless maintainers make it the canonical umbrella.
