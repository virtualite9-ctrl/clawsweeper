## Summary

Keep open. Current `main` has fire-and-forget via `timeoutSeconds: 0`, but it does not implement the requested receive-acknowledge pattern where the sender gets an immediate ack and then a later final response through the `sessions_send` contract. The tool schema has no `ackMode`-style option, and the follow-up delivery path is channel/thread announce-oriented rather than a sender-session callback.

## Evidence

- **Tool API only exposes wait-or-fire-and-forget:** `sessions_send` currently accepts `sessionKey`, `label`, `agentId`, `message`, and `timeoutSeconds` only. There is no parameter for an acknowledge/final-response mode.
  - file: [src/agents/tools/sessions-send-tool.ts:37](https://github.com/openclaw/openclaw/blob/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503/src/agents/tools/sessions-send-tool.ts#L37)
  - command: `nl -ba src/agents/tools/sessions-send-tool.ts | sed -n '1,220p'`
  - sha: [b0d2003fe65b](https://github.com/openclaw/openclaw/commit/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503)
- **Immediate path returns accepted, not ack-plus-final:** When `timeoutSeconds === 0`, the tool starts the run, kicks off background A2A flow, and returns `{ status: "accepted", delivery: { status: "pending", mode: "announce" } }`. That is immediate acceptance only; the final response is not returned through the same tool call.
  - file: [src/agents/tools/sessions-send-tool.ts:338](https://github.com/openclaw/openclaw/blob/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503/src/agents/tools/sessions-send-tool.ts#L338)
  - command: `nl -ba src/agents/tools/sessions-send-tool.ts | sed -n '220,380p'`
  - sha: [b0d2003fe65b](https://github.com/openclaw/openclaw/commit/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503)
- **Background follow-up targets the target session route:** The A2A flow resolves an announce target from `targetSessionKey`/`displayKey` and sends the announce message to that channel/thread. The announce prompt also says, "Any other reply will be posted to the target channel." This is not a sender-session receive callback.
  - file: [src/agents/tools/sessions-send-tool.a2a.ts:64](https://github.com/openclaw/openclaw/blob/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503/src/agents/tools/sessions-send-tool.a2a.ts#L64)

In that `main`-session case, `sendCallCount` remains `0`, showing there is no later delivered final response for pure agent-session targets.

## Risks / Open Questions

- There is partial overlap with the request because `timeoutSeconds: 0` already gives a fast acceptance path, so maintainers may choose to re-scope the issue toward sender-facing final delivery rather than a brand-new primitive.
- Docs describe A2A announce behavior in broad terms, but current code and tests show it is not a general sender-session ack-then-final mechanism.
