## Summary

Keep #13911 open. Current main has improved parts of the subagent announce path, but it still does not provide a first-class per-channel/per-subagent suppression or reroute policy. Top-level requester completions are still documented and tested as externally delivered when a usable requester route exists, and the current config/plugin surfaces do not expose the requested structural control.

## Best Possible Solution

Keep #13911 open. The best path is a core-supported subagent completion audience/delivery policy, wired through config schema, generated config docs, user docs, and regression tests, and applied before external requester-channel delivery and fallback sends. If maintainers prefer plugin-owned policy, first extend the plugin API so hooks can explicitly suppress subagent completion announcements by provenance and channel without relying on message text or generic outbound filters.
