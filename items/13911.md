## Summary

Keep #13911 open. Current main still lacks a first-class per-channel suppression or reroute policy for subagent completion announcements, and the latest maintainer discussion explicitly says this remains needed rather than already implemented or superseded.

## Best Possible Solution

Keep #13911 open. The best product path is a core-supported subagent completion delivery/audience policy in config, schema, docs, and tests, applied before external requester-channel delivery is resolved. It should cover Telegram/iMessage-style channels without relying on AGENTS.md behavior, sentinel replies, sanitizer proxies, or the current reroute-only hook.
