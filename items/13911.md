## Summary

Keep #13911 open. Current main still lacks a first-class per-channel sub-agent completion announce suppression or reroute policy; the documented and tested behavior still delivers top-level subagent completion announcements back through the requester channel when an external route exists.

## Best Possible Solution

Keep #13911 open. The best path is a core-supported subagent completion delivery/audience policy in config, schema, docs, and tests, applied before external requester-channel delivery is resolved. It should cover Telegram/iMessage-style channels without relying on AGENTS.md instructions, sentinel replies, outbound sanitizer proxies, or the current routing-only plugin hook.
