## Summary

Current main has partial overlap via chat `/allowlist` and `openclaw directory`, but the requested CLI surface for viewing/managing approved WhatsApp contacts, policies, and last-active state is not implemented. `pairing list` still lists only pending requests, and WhatsApp directory peers are config-backed rather than pairing-store approved contacts.

## Best Possible Solution

Keep the issue open and either add a channel sender/contacts CLI, likely under `openclaw contacts` or an allowlist-oriented CLI, or explicitly document `/allowlist` as the intended admin workflow. If implemented, it should merge WhatsApp config allowFrom, pairing-store approvals, DM/group policy, account scoping, and any available last-active signal without treating pending pairing requests as approved contacts.

## Risks / Open Questions

- There is overlapping functionality in `/allowlist`, so maintainers may choose to document that as the supported admin workflow instead of adding a new CLI command.
- A complete CLI view would need a clear product decision on whether to merge config allowlists, pairing-store approvals, group allowlists, and runtime last-active/session data into one report.
