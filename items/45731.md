## Summary

Keep open. Current main has partial mitigations for some theories in the discussion, but the central Control UI responsiveness problem is not proven fixed: the Overview tab still triggers the same broad startup RPC fan-out, gateway hello still starts multiple immediate loads plus active-tab refresh, and the browser RPC client still has no per-request timeout or AbortController cancellation path. The issue is also under 60 days old, so stale-insufficient-info closure does not apply.

## Best Possible Solution

Keep the issue open and narrow it to Control UI startup/Overview responsiveness. The likely maintainer path is to instrument Overview RPC latency, defer or lazy-load heavy Overview panels such as debug, usage, logs, and model auth, add UI request timeout/abort handling for stale in-flight RPCs, and cover slow-RPC startup behavior with focused Control UI/gateway tests. If maintainers want a single tracker, pick an explicit canonical dashboard-freeze issue first rather than auto-closing this one.
