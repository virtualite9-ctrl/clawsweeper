## Summary

Current main implements the lifecycle-event route requested here: ACP spawns are registered as completion-tracked runs, ACP dispatch emits lifecycle end/error events, and completion delivery wakes the requester through the subagent/task-completion announce path rather than relying on the delivery-mirror transcript row to activate a turn.

## Best Possible Solution

Keep the shipped lifecycle plus task-completion announce path as the parent wake mechanism. Leave delivery-mirror as transcript/outbound status plumbing, and track any remaining UI refresh or duplicate-status concerns in narrower follow-up issues such as #52249 rather than keeping this lifecycle-routing bug open.
