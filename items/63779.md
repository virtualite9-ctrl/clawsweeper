## Summary

Keep open. Current main implements ACP lifecycle/task completion delivery, but the narrower bug in this issue remains: ordinary successful ACP task completion on a deliverable requester channel is sent outward and mirrored into the parent transcript as an `assistant` `delivery-mirror` entry, while current tests assert that this path queues no parent system event and no heartbeat wake. That matches the reported failure mode after `sessions_yield`.

## Best Possible Solution

Keep this issue open. The maintainer path should add a regression for successful ACP completion after `sessions_yield` on a deliverable requester channel, then change the completion path so the yielded parent receives a real wake/resume signal or queued system event instead of relying only on an assistant-role delivery-mirror transcript entry. Keep #52249 for the broader heartbeat-disabled stream-to-parent gap unless maintainers explicitly consolidate the two.
