## Summary

Keep #14438 open. Current main still documents native plugin runtime code changes as restart-only, the `openclaw plugins` CLI has no reload or clear-cache command, config reload treats broad `plugins.*` changes as restart work, and the jiti/registry cache clearing paths remain internal test/runtime helpers rather than a supported Gateway RPC or CLI UX.

## Best Possible Solution

Keep #14438 open as core plugin loader and Gateway lifecycle developer UX work. The best path is a supported Gateway RPC plus `openclaw plugins reload <id>` and/or `openclaw plugins clear-cache` that invalidates jiti/module and registry state, disposes plugin-owned registrations and services safely, refreshes tools/hooks/provider/runtime hooks, and documents container wrapper-process behavior. If native plugin hot reload is intentionally unsupported, document that decision in https://docs.openclaw.ai/tools/plugin and https://docs.openclaw.ai/cli/plugins with a supported restart/cache workflow.
