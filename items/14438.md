## Summary

Keep #14438 open. Current main still documents native plugin runtime changes as restart-only, exposes no `openclaw plugins reload` or cache-clear command, and the loader/Gateway code only has internal cache/state reset pieces, not a supported live plugin hot-reload workflow with Jiti invalidation and re-registration.

## Best Possible Solution

Keep #14438 open as core plugin loader and Gateway lifecycle developer UX work. The likely product path is a supported Gateway RPC plus `openclaw plugins reload <id>` and/or `openclaw plugins clear-cache` that invalidates loader/Jiti/module state, disposes plugin-owned registrations and services, refreshes tools/hooks/providers/runtime hooks, and documents Docker/container behavior. If maintainers decide native plugin hot reload is intentionally unsupported, update the public plugin and CLI docs with a clear supported restart/cache workflow instead.
