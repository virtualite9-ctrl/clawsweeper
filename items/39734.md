## Summary

Keep open. Current main appears to cover the original 1M-context motivation for Claude 4.6 Bedrock models, but the requested generic Bedrock `anthropic_beta` / custom request-field passthrough is still not implemented in config or runtime.

## Evidence

- **Model config has no custom Bedrock field slot:** `ModelDefinitionConfig` exposes fields like `id`, `name`, `api`, `headers`, `compat`, and `metadataSource`, but no `providerFields`, `anthropic_beta`, or arbitrary additional request-fields surface.
  - file: [src/config/types.models.ts:76](https://github.com/openclaw/openclaw/blob/cb2c36b04975e34ab7014a11575d6589aa96ae32/src/config/types.models.ts#L76)
  - command: `nl -ba src/config/types.models.ts | sed -n '55,130p'`
  - sha: [cb2c36b04975](https://github.com/openclaw/openclaw/commit/cb2c36b04975e34ab7014a11575d6589aa96ae32)
- **Bedrock plugin schema only allows discovery and guardrail config:** The bundled `amazon-bedrock` plugin schema defines only `discovery` and `guardrail` under plugin config, with `additionalProperties: false`, so there is no plugin-level custom beta/request-field config either.
  - file: [extensions/amazon-bedrock/openclaw.plugin.json:8](https://github.com/openclaw/openclaw/blob/cb2c36b04975e34ab7014a11575d6589aa96ae32/extensions/amazon-bedrock/openclaw.plugin.json#L8)
