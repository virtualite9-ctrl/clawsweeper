## Summary

Current main still does not provide a configurable Amazon Bedrock Converse anthropic_beta or additionalModelRequestFields passthrough. The original 1M-context preview motivation may be obsolete for current Claude 4.6 GA models, but the broader requested Bedrock-specific beta/additional request-field config path remains absent.

## Best Possible Solution

Keep this open for a Bedrock-plugin-owned implementation or explicit maintainer product decision. A concrete fix should add a validated Bedrock config or per-model params surface for anthropicBeta/allowed Converse additional request fields, merge it into the Bedrock Converse request payload in extensions/amazon-bedrock/register.sync.runtime.ts, and cover it with Bedrock plugin tests plus docs/providers/bedrock.md.
