## Summary

Keep open. Current main still does not provide a configurable Amazon Bedrock Converse `anthropic_beta` / `additionalModelRequestFields` passthrough. The direct Anthropic provider has beta-header handling, and Bedrock has newer discovery/profile/guardrail/cache behavior, but the requested Bedrock-specific config path remains absent.

## Best Possible Solution

Keep the issue open for a Bedrock-plugin-owned implementation or explicit maintainer product decision. A concrete fix should add a validated Bedrock config or per-model params surface for `anthropicBeta`/allowed Converse additional request fields, merge it into `additionalModelRequestFields` in `extensions/amazon-bedrock/register.sync.runtime.ts`, and cover it with Bedrock plugin tests plus `docs/providers/bedrock.md`.
