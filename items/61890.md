## Summary

Current `main` already implements the requested preemptive behavior via the existing inbound queue setting `messages.queue.mode: "interrupt"` and the per-session `/queue interrupt` command. The runtime clears queued same-session work, aborts the active embedded run, waits for shutdown, and then continues with the new message; docs and tests cover the path.
