## Summary

Keep open. Current main has partial fixes for manual/preflight compaction accounting, but the shared `/status` formatter still renders `sessionEntry.totalTokens` directly and the auto-compaction accounting path does not carry the compaction engine's `tokensAfter` value through to the session status cache. That leaves the reported stale post-compaction context display plausible and not fully covered by current tests.

## Best Possible Solution

Keep the issue open and fix the shared status accounting: carry the compaction engine's actual `tokensAfter` through every auto-compaction path, persist `totalTokensFresh: true` only for a real post-compaction snapshot, have `/status` ignore or mark stale cached totals when freshness is false, and add a regression test that simulates auto-compaction followed immediately by `/status`/status-card rendering with `Compactions: 1` and a small post-compaction context value.
