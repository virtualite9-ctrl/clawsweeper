## Summary

Keep open. Current main still does not implement the full Feishu thread anchoring behavior this PR targets: first-turn topic sessions can route by message_id but do not persist it as MessageThreadId, and several outbound threadId fallback paths still omit replyInThread. The PR remains a focused bug-fix candidate, and the related #67069 overlap is narrower inbound work, not a full supersession.

## Best Possible Solution

Keep this PR open as a plausible implementation candidate. Rebase it onto current main, reconcile the narrower #67069 inbound-thread metadata overlap, preserve the guarded message_id fallback only for topic-session plus replyInThread, consistently propagate replyInThread through Feishu outbound text, markdown-card, structured-card, media, and local-image paths, then run the targeted Feishu extension tests and changed gate before merge or request changes.
