## Summary

Keep open. The PR is a focused Feishu bug fix and current main still lacks key parts of it: first-turn topic messages without root_id do not persist MessageThreadId from message_id, and several outbound threadId fallback paths still do not propagate replyInThread.

## Best Possible Solution

Keep PR #60848 open and either rebase/merge its focused Feishu fixes or cherry-pick the same behavior onto current main: persist message_id as MessageThreadId only for topic replyInThread sessions without root_id, centralize the threadId-fallback replyInThread decision with trimmed replyToId handling, and cover text, markdown/card, media, and local-image auto-send paths with Feishu unit tests.
