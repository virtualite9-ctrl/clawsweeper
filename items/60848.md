## Summary

Keep open. Current `main` at `6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440` still lacks the Feishu thread-anchor and `replyInThread` propagation changes described in PR #60848, and the branch-specific follow-up fixes discussed in review are not present in the checked-out code.

## Evidence

- **Inbound thread anchor fix is not on main:** `handleFeishuMessage` still sets `MessageThreadId` only from `ctx.rootId` for topic sessions. It does not fall back to `ctx.messageId` when `replyInThread` is enabled and `root_id` is absent, which is the first bug the PR is trying to fix.
