## Summary

Keep #35735 open. Current main still lacks the requested direct CLI commands `openclaw sessions rm <key>` and `openclaw sessions clear`; the shipped surface lists sessions, runs policy-based cleanup, and exposes a live Gateway/UI per-key delete path, but not the offline store-targeted deletion workflow with `--store`, `--agent`, `--all-agents`, `--dry-run`, `--json`, or `--older-than` proposed by this PR. Related issue #35682 is closed as superseded by this PR, so this PR remains the canonical open implementation path.

## Best Possible Solution

Keep #35735 open for maintainer judgment. The best path is a rebased, focused implementation of `openclaw sessions rm <key>` and `openclaw sessions clear` that uses safe store mutation semantics, avoids unrelated maintenance pruning, fixes the reported transcript/JSON issues, documents the CLI surface at https://docs.openclaw.ai/cli/sessions, and adds targeted command/registration tests. If maintainers reject this command shape, replace it with a narrower canonical design issue or PR rather than auto-closing this work as stale.
