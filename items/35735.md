## Summary

Keep #35735 open. Current main still lacks `openclaw sessions rm <key>` and `openclaw sessions clear`; the CLI registers listing plus `sessions cleanup` only. Gateway/UI `sessions.delete` exists, but it is a live per-key RPC/UI path, not the PR's offline store-targeted CLI workflow with bulk filtering, `--store`, `--agent`, `--all-agents`, `--dry-run`, and `--json`.

## Best Possible Solution

Keep #35735 open for maintainer judgment. The best path is a rebased, focused implementation of `openclaw sessions rm <key>` and `openclaw sessions clear` that uses explicit safe store mutation semantics, addresses the existing review comments, documents the public CLI at https://docs.openclaw.ai/cli/sessions, and adds targeted registration/command tests. If maintainers reject this command shape, replace it with a narrower canonical design issue rather than auto-closing this PR as stale.
