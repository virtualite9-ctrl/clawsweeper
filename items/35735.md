## Summary

Keep #35735 open. Current main still does not provide the requested direct CLI surface `openclaw sessions rm <key>` or `openclaw sessions clear`; it only has `openclaw sessions` listing and `openclaw sessions cleanup` maintenance. The existing gateway/UI `sessions.delete` path is real and useful context, but it is a live gateway per-key delete API and does not cover the PR's offline CLI store targeting, all-agent, dry-run, JSON, or age-filtered bulk deletion workflow. The related issue #35682 was closed as superseded by this PR, making #35735 the canonical implementation path rather than an obsolete duplicate.

## Best Possible Solution

Keep #35735 open for maintainer judgment. The best path is a rebased, focused implementation of `openclaw sessions rm <key>` and `openclaw sessions clear` that uses safe store mutation semantics without unrelated maintenance pruning, fixes the review-noted transcript reporting and JSON schema issues, documents the new CLI surface, and adds targeted registration/command tests. If maintainers reject this command shape, replace it with a narrower canonical PR or reopen a design issue rather than auto-closing this work as stale.
