## Summary

Keep #35735 open. Current main still lacks the requested direct CLI commands `openclaw sessions rm <key>` and `openclaw sessions clear`; existing `sessions.delete` support is a live gateway/UI per-key path, not the offline store-targeted, all-agent, dry-run, JSON, or age-filtered bulk CLI workflow proposed by this PR. Related issue #35682 is closed as superseded by this PR, so this remains the canonical open implementation path.

## Best Possible Solution

Keep #35735 open for maintainer judgment. The best path is a rebased, focused implementation of `openclaw sessions rm <key>` and `openclaw sessions clear` that uses safe store mutation semantics, avoids unrelated maintenance pruning, fixes the reported transcript/JSON issues, documents the CLI surface, and adds targeted command/registration tests. If maintainers reject this command shape, replace it with a narrower canonical design issue or PR rather than auto-closing this work as stale.
