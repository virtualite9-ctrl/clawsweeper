## Summary

Keep #8582 open. Current main implements the ElevenLabs realtime STT/provider-generic Voice Call streaming portion and adds general manifest docs for plugin config schemas and duplicate-id precedence, but the issue still has unsplit DX asks: distinct duplicate plugin ids still always warn, there is no per-id duplicate-warning suppression or explicit override config knob, and the docs still do not include the requested focused guidance for extending/forking a bundled plugin with new config keys.

## Best Possible Solution

Keep #8582 open unless a maintainer splits it. The best path is to add a tested per-id duplicate override or suppression acknowledgement in plugin config and manifest-registry diagnostics, document that behavior, and add focused plugin docs explaining that custom config keys require updating `openclaw.plugin.json` `configSchema` when extending or forking a bundled plugin. Alternatively, create canonical follow-up issues for those two remaining DX asks before closing this umbrella item.
