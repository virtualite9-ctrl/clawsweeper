## Summary

Keep this PR open. Current main still does not implement a Feishu chat-history `list` action, and this PR is a plausible implementation candidate, but it needs maintainer review because the PR discussion identifies a real ID-type bug and the feature expands agent-readable Feishu group history. The overlapping open #41306 should be considered, but it is not a strong enough canonical replacement to cleanup-close this narrower PR.

## Best Possible Solution

Keep this PR open as a focused implementation candidate. The best path is to resolve the threadId/chatId bug, add targeted unit coverage for chat-history listing and limit/page-token behavior, document the required Feishu scopes and privacy implications, then compare with #41306 and either land the cleaner implementation or close the other PR as superseded.

## Risks / Open Questions

- The PR head has an ID-type bug from the review discussion: `threadId` must not be treated as a Feishu `chat_id` for `container_id_type=chat`.
- Reading group chat history is a broader data-access capability than sending/reading one message; maintainers should require clear docs, permission-scope guidance, and appropriate action discovery semantics before merge.
- There is overlap with open #41306, so maintainers should choose one canonical Feishu chat-history path rather than merging divergent implementations.
