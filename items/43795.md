## Summary

Keep open. The report has concrete Feishu/GptGod GLM logs and prior analysis, and current main still contains unguarded non-array `content` handling in the OpenAI-compatible response/gateway paths. The linked fix PRs are closed unmerged, while main only has a partial replay-string normalization path.

## Best Possible Solution

Keep this issue open and land a focused core fix at the OpenAI-compatible ingress points: normalize or reject non-array assistant `content` safely in response conversion and gateway request handling, preserve the existing string replay repair where valid, and add regression tests for string, object, and null `content` shapes. #43864 is useful prior analysis but should be reviewed or reimplemented against current main rather than treated as shipped.
