## Summary

Keep open. Current main still has unsafe non-array assistant content handling in the OpenAI Responses conversion and replay paths, and the related fix PRs were closed unmerged.

## Best Possible Solution

Keep this issue open and land a focused core fix around OpenAI-compatible response ingress and replay normalization: guard buildAssistantMessageFromResponse with Array.isArray before .some/iteration, normalize malformed assistant replay content for string/object/null shapes, review the openresponses-http non-array guard, and add regression tests for malformed provider payloads. #43836 and #43864 are useful prior analysis but were not merged.
