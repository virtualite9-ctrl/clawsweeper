## Summary

Keep open. Current main still has unsafe non-array assistant content handling in the OpenAI Responses conversion path, and the related fix PRs in the provided context were closed unmerged.

## Best Possible Solution

Keep this issue open and land a focused core fix around OpenAI-compatible ingress and replay boundaries: guard or normalize non-array assistant content in buildAssistantMessageFromResponse, decide the intended object/null replay behavior, preserve the existing string replay repair, and add regression tests for string, object, and null provider payloads. #43836 and #43864 are useful prior analysis, but the fix should be reviewed or reimplemented against current main.
