## Summary

Keep this PR open. Current main still lacks the central feature: no live queue diagnostics snapshot, no `diagnostics.queue` Gateway RPC, and no `openclaw gateway diagnostics queue` CLI command. The related agent-facing `queue_status` request is adjacent but not a replacement for this operator Gateway/CLI surface, and the previous ClawSweeper review reached the same conclusion. The PR remains a plausible implementation candidate, pending maintainer review, rebase, and security/privacy validation of the exposed queue/session metadata.

## Best Possible Solution

Keep this PR open for maintainer review or an updated successor. The best path is a small core-owned read-only queue snapshot seam in `src/process/command-queue.ts`, exposed through `diagnostics.queue` and `openclaw gateway diagnostics queue`, scoped to `operator.read`, documented, and covered by focused queue, Gateway, CLI, and authorization tests. Keep related #9797 separate for the agent-facing `queue_status` tool discussion.
