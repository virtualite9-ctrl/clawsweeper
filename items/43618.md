## Summary

Keep open. Current main still does not implement the PR's central capability: a live queue diagnostics snapshot exposed as `diagnostics.queue` and `openclaw gateway diagnostics queue`. Existing stability diagnostics are an event recorder/export path, not a live per-lane backlog/age snapshot. Related #9797 remains adjacent agent-facing work, not a supersession of this operator diagnostics PR.

## Best Possible Solution

Keep this PR open for maintainer review or an updated successor. The best path is a minimal core-owned, read-only queue snapshot seam used by `diagnostics.queue` and `openclaw gateway diagnostics queue`, with `operator.read` authorization, docs, and focused tests. Keep #9797 separate for the in-agent `queue_status` dispatch tool that this PR explicitly does not implement.
