## Summary

Keep open. Current main still has the reported warning-policy behavior: mutating tool errors return `showWarning: true` before the recoverable-error suppression path, and tests explicitly expect recoverable mutating failures to produce warning payloads. The shipped edit recovery wrapper is a useful partial mitigation, but it does not resolve this warning-policy issue or supersede related open items.

## Best Possible Solution

Keep this issue open. The likely maintainer path is a focused change around `src/agents/pi-embedded-runner/run/payloads.ts` and/or upstream `lastToolError` clearing, with regression tests showing that recoverable or post-write-applied mutating failures do not emit false `failed` notifications while real mutating failures remain visible. Coordinate with #59868 and PR #60664, but do not treat them as shipped replacements.
