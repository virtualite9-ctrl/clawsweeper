## Summary

Keep open. Current main still emits user-facing failed payloads for mutating tool errors unless the assistant explicitly acknowledges the failure; the recoverable-error suppression remains limited to the non-mutating branch, and tests explicitly preserve recoverable mutating warnings.

## Best Possible Solution

Keep this issue open. The best path is a focused fix around `src/agents/pi-embedded-runner/run/payloads.ts` and possibly `src/agents/pi-embedded-subscribe.handlers.tools.ts`: suppress false mutating failure notifications only for recoverable or proven-applied failures, clear stale `lastToolError` when a mutation is proven applied, and add regression tests proving false-positive mutating failures do not emit `failed` payloads while real mutating failures remain visible. Coordinate with #59868 and PR #60664, but do not close this issue until the behavior is fixed on main.

## Risks / Open Questions

- A future fix must not hide genuine mutating failures such as permission errors, disk-full writes, failed sends, failed deletes, or real edit mismatches.
