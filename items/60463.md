## Summary

Keep open. Current `main` still makes the parent-thread fork decision from `sessionStore[parent].totalTokens ?? 0` only, so the stale/missing-token gap described in PR #60463 is still present. The transcript-estimation fallback and its regression test from the PR are not on `main`.

## Evidence
