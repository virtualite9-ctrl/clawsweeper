## Summary

Keep #13610 open. Current main has the external SecretRef baseline, but the requested native AWS Secrets Manager, HashiCorp Vault, Azure Key Vault, and Google Cloud Secret Manager integrations are still not implemented, and the issue discussion explicitly preserves this as the umbrella after #26155.

## Best Possible Solution

Keep #13610 open as the native secrets-manager provider umbrella. The best product path is either to implement first-class AWS Secrets Manager, HashiCorp Vault, Azure Key Vault, and Google Cloud Secret Manager providers across types, schema/protocol, resolver dispatch, CLI/docs, and tests, or make an explicit maintainer decision that exec wrappers or a future plugin/ClawHub provider API are the supported route.
