## Summary

Keep #13610 open. Current main has the shipped SecretRef baseline from #26155, but native AWS Secrets Manager, HashiCorp Vault, Azure Key Vault, and Google Cloud Secret Manager providers are still absent; the issue discussion explicitly preserves this item as the native-provider umbrella after the baseline shipped.

## Best Possible Solution

Keep #13610 open as the native secrets-manager provider umbrella. The best path is either to implement first-class AWS Secrets Manager, HashiCorp Vault, Azure Key Vault, and Google Cloud Secret Manager support across types, schema/protocol, resolver dispatch, CLI/docs, and tests, or make an explicit maintainer product decision that exec wrappers or a future plugin/ClawHub provider API are the supported route.
