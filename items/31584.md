## Summary

Keep open. Current main still does not implement the requested non-interactive GitHub Copilot onboarding path. GitHub Copilot can use environment GitHub tokens at runtime, but the bundled provider setup remains device-code/TTY-only, the generic non-interactive onboarding path requires provider-owned `runNonInteractive`, and the Copilot manifest does not expose a provider API-key flag like Anthropic does.

## Best Possible Solution

Keep tracking this as bundled GitHub Copilot provider work. The likely fix belongs in the GitHub Copilot plugin: add a provider-owned non-interactive auth method or token setup path, expose manifest-backed CLI flag/env/ref metadata, store the GitHub token as a `github-copilot` token profile, optionally apply the recommended default model, and add focused non-interactive onboarding tests plus docs updates for `docs.openclaw.ai/providers/github-copilot`.
