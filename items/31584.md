## Summary

Keep open. Current main still does not provide the requested non-interactive GitHub Copilot onboarding path. The bundled GitHub Copilot provider supports runtime token discovery from COPILOT_GITHUB_TOKEN/GH_TOKEN/GITHUB_TOKEN and an interactive device login, but the provider auth method is device-code/TTY-only, lacks runNonInteractive, and the manifest does not expose a Copilot CLI key flag for non-interactive onboarding.

## Best Possible Solution

Keep tracking this as bundled GitHub Copilot provider work. Add a provider-owned non-interactive auth method for GitHub tokens, expose manifest-backed CLI flag/env/ref metadata, store the GitHub token under the github-copilot auth profile path, apply an appropriate default model when requested, and update focused tests plus https://docs.openclaw.ai/providers/github-copilot and CLI automation docs.
