## Summary

Current main still drops Microsoft Teams messageBack Action.Submit activities when they arrive as type="message" with empty text and only activity.value. This PR is open, unmerged, and directly adds the missing message-handler fallback plus regression coverage, so it should be reviewed or landed rather than closed.

## Best Possible Solution

Keep this PR open as the active implementation candidate for the remaining Teams messageBack Action.Submit gap. Maintainers should review the serialization contract, then land the PR with its regression tests or request focused changes in the message-handler fallback/tests.
