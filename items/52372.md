## Summary

Current `main` does not contain the reported Discord gateway startup race fix. The risky fire-and-forget `registerClient` path is still present, the current `SafeGatewayPlugin.registerClient` still awaits gateway metadata before delegating, and the proposed fix remains only in open PR #68159 rather than on `main`.

## Evidence

- **Fire-and-forget gateway plugin registration still exists:** `registerLatePlugin` still calls `void plugin.registerClient?.(compatClient)`, so async plugin registration is not awaited in the current startup path.
  - file: [extensions/discord/src/monitor/provider.startup.ts:74](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/extensions/discord/src/monitor/provider.startup.ts#L74)
  - command: `nl -ba extensions/discord/src/monitor/provider.startup.ts | sed -n '1,140p'`
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **Current gateway plugin still has the pre-fix async window:** On current `main`, `SafeGatewayPlugin.registerClient` begins by awaiting `fetchDiscordGatewayInfoWithTimeout(...)` and only later calls `super.registerClient(client)`. It does not eagerly assign `this.client` before the await, and it does not guard against an already-started `ws`/`isConnecting` state before calling `super.registerClient`.
  - file: [extensions/discord/src/monitor/gateway-plugin.ts:274](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/extensions/discord/src/monitor/gateway-plugin.ts#L274)

Its patch adds the missing early `this.client = client` assignment and the `ws`/`isConnecting` guard, which are absent from current `main`.
