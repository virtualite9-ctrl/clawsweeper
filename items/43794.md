## Summary

Keep this issue open. Current main still mitigates some plaintext credential exposure with opt-in SecretRefs, audits, file-permission checks, and selective scrubbing, but it does not implement first-class encrypted-at-rest credential storage, a native OpenClaw keychain-backed store, or a master-password store for config/auth-profile credentials.

## Best Possible Solution

Keep open as core security work. The best path is to design and ship a first-class secure credential storage layer for OpenClaw credentials, with native keychain and/or encrypted-at-rest provider support, backward-compatible migration from plaintext config/auth stores, explicit OAuth and rotating-credential handling, token-rotation guidance, audit updates, docs, and regression tests.
