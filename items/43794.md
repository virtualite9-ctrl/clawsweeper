## Summary

Keep this issue open. Current main has opt-in SecretRef support and plaintext auditing/scrubbing, but it does not implement first-class encryption at rest or a system-keychain/master-password secure store for OpenClaw credentials. The current threat model still lists token theft from config/credentials storage as high residual risk and still recommends token encryption at rest.

## Best Possible Solution

Keep open and treat this as core security work. The best path is to design a first-class secure credential storage story in the secrets/auth-profile layer: native keychain or encrypted-at-rest provider support, backward-compatible migration from plaintext config/auth stores, clear handling for OAuth/rotating credentials, token rotation guidance, and updated threat model/docs/tests once shipped.
