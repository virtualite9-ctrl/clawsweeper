## Summary

Keep this issue open. Current main has SecretRefs, audits, secure permission checks, and external env/file/exec secret providers, but it still does not provide first-class OpenClaw credential encryption at rest, a native keychain-backed store, or a master-password-backed store for plaintext config/auth-profile credentials. The supplied issue discussion and prior ClawSweeper review already identified this as an active core security gap, and current source/docs still agree.

## Best Possible Solution

Keep open as core security work. Design and ship a first-class secure credential storage layer for OpenClaw credentials, with OS keychain and/or encrypted-at-rest master-password/provider support, backward-compatible migration from plaintext config and auth-profile stores, SecretRef compatibility, OAuth/rotation handling, audit/docs updates, and focused regression coverage.
