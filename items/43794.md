## Summary

Keep this issue open. Current main mitigates plaintext credential exposure with opt-in SecretRefs, audits, secure file modes, and external CLI keychain reads for some provider imports, but it still does not provide first-class encrypted-at-rest OpenClaw credential storage, a native OpenClaw keychain-backed store, or a master-password store for config and auth-profile credentials.

## Best Possible Solution

Keep open as core security work. The best path is to design and ship a first-class secure credential storage layer for OpenClaw credentials, with OS keychain and/or encrypted-at-rest master-password/provider support, backward-compatible migration from plaintext config and auth-profile stores, clear OAuth/rotation handling, SecretRef compatibility, audit/docs updates, and regression tests.
