## Summary

Current main already removes the Windows update restart helper path that generated the reported `findstr` CMD window. The helper now writes a hidden `.cmd` wrapper that runs embedded PowerShell, discovers listeners with `Get-NetTCPConnection` first and direct `netstat.exe` parsing as fallback, and regression tests assert the old `findstr`/`netstat -ano |` batch pattern is absent. The linked PR #71909 identified the same root cause; current main now has the intended behavior, though no shipped release tag containing it was found.

## Best Possible Solution

Close #59394 as implemented on main. Keep the current PowerShell-based Windows update restart helper and its regression tests; if the visible window still reproduces after a release containing this code, track that as a new Windows restart-helper regression with fresh logs and version details.
