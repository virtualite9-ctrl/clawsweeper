## Summary

Keep #72069 open. Current main still does not seed cross-provider fallback prompts from Claude CLI session history: fallback retries only use OpenClaw's own transcript/history flag, while the existing Claude JSONL reader is only used to backfill displayed chat history. The PR is a focused core bug fix for #69973, and the provided discussion shows the author addressed substantive review findings on the PR head.

## Best Possible Solution

Keep #72069 open for maintainer review. The right product/code path is core fallback-runner support: read the existing claude-cli session binding, extract a bounded summary/recent-turn prelude from Claude's JSONL only when the failed source provider was claude-cli, and prepend it to the cross-provider retry prompt with regression coverage in `src/agents/command` and `src/gateway`.
