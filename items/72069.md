## Summary

Keep PR #72069 open. Current main still does not seed cross-provider fallback prompts from Claude CLI JSONL history: fallback retries only add a retry marker when OpenClaw's own session file has history, while Claude CLI history import is limited to chat-history display paths. The PR is an active, focused fix for #69973, and the provided discussion shows the author addressed substantive review findings on the PR head.

## Best Possible Solution

Keep #72069 open for maintainer review. The best path is to land or adapt the PR's core fallback-runner change: when the failed source provider was `claude-cli` and the next candidate is a different provider/runtime, read the bound Claude CLI session JSONL, extract a bounded compaction summary plus recent-turn prelude, prepend it to the fallback retry prompt, and keep regression tests in `src/agents/command` and `src/gateway`.
