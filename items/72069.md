## Summary

Keep PR #72069 open. Current main still does not seed cross-provider fallback prompts from Claude CLI session history; the open PR remains a focused, active fix for behavior that is missing on main.

## Best Possible Solution

Keep #72069 open for maintainer review. The likely correct product/code path is core fallback-runner support: when a fallback chain started on claude-cli and retries on a different provider/runtime, read the bound Claude CLI session, extract a bounded summary/recent-turn prelude from Claude's JSONL, prepend it to the retry prompt, and cover the path in src/agents/command and src/gateway tests.
