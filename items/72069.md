## Summary

Keep #72069 open. Current main does not implement the PR’s central claude-cli-to-non-cli fallback context seeding: fallback retry prompts still only add the generic retry marker when OpenClaw’s own session transcript has history, and the gateway Claude JSONL reader only exposes regular imported messages, not a compact-summary fallback seed. The linked bug #69973 has concrete logs and a plausible code-path match, while the PR has active review comments that need author/maintainer resolution rather than cleanup closure.

## Best Possible Solution

Keep the PR open. The best path is to revise #72069 so fallback context seeding only runs when the failed attempt was actually claude-cli, address the compact-summary edge cases or document the chosen semantics, then validate with targeted agent/gateway tests and the normal changed gate before maintainer merge.
