## Summary

Keep open. Current `main` still does not implement legacy flat `auth-profiles.json` fallback, and this PR's latest diff still has unresolved correctness/regression problems, so there is no strong basis to close it under the conservative cleanup rules.

## Evidence

- **Current main only accepts canonical `auth-profiles.json` shape:** `loadAuthProfileStoreForAgent` first tries `loadPersistedAuthProfileStore(agentDir)`, then falls through to main-agent inheritance and legacy `auth.json` migration. There is no branch that reparses the same `auth-profiles.json` file as a legacy flat store.
  - file: [src/agents/auth-profiles/store.ts:189](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/src/agents/auth-profiles/store.ts#L189)
  - sha: [ec3dbd22a4de](https://github.com/openclaw/openclaw/commit/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46)
- **Persisted store parser rejects files without `profiles`:** `coercePersistedAuthProfileStore` returns `null` unless `record.profiles` exists and is an object, which matches the reporter's flat top-level JSON repro and explains why current main drops it.
  - file: [src/agents/auth-profiles/persisted.ts:128](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/src/agents/auth-profiles/persisted.ts#L128)
  - sha: [ec3dbd22a4de](https://github.com/openclaw/openclaw/commit/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46)

  - file: [src/agents/auth-profiles/path-resolve.ts:17](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/src/agents/auth-profiles/path-resolve.ts#L17)
