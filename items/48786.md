## Summary

Keep open. Current main still fetches Feishu quoted/replied message content without carrying or normalizing the Feishu `mentions` metadata, so quoted bodies can still surface raw `@_user_N` placeholders. The linked PR #48976 is an open fix attempt, but it is not merged or shipped and should not replace the original bug report yet.

## Best Possible Solution

Keep #48786 open and land a narrow Feishu plugin fix: add the fetched-message `mentions` shape, reuse `normalizeMentions()` in `parseFeishuMessageItem()` for `getMessageFeishu()` and thread-history callers, and add regression tests in `extensions/feishu/src/send.test.ts` for quoted text/post content containing `@_user_N` placeholders. PR #48976 can be rebased or mined for that change if it remains clean.
