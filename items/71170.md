## Summary

Keep this PR open. Current main still lacks the proposed `agents.defaults.reminderGuard` and per-agent `reminderGuard` opt-out, and the reminder guard still suppresses only for built-in `cron.add` success or an existing enabled core cron job for the same session. The linked #70666 report describes a concrete plugin-tool false positive, and the PR remains a plausible focused mitigation pending maintainer review.

## Best Possible Solution

Keep this PR open and have maintainers review, land, or adapt the config-based `reminderGuard` opt-out. Longer term, a plugin SDK signal for successful external scheduling may be the cleaner contract, but that should be a separate CODEOWNER-reviewed API proposal; current main still needs a shipped mitigation for external scheduler tools.
