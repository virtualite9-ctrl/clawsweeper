## Summary

Keep this PR open. Current main at 66f4b52db31ba2b0d61c74b184a23f26d31edce3 still lacks the proposed `agents.defaults.reminderGuard` and per-agent `reminderGuard` opt-out, and the runtime still appends the reminder disclaimer based only on core cron success or an existing same-session cron job. The PR remains a plausible focused implementation candidate for the plugin-managed scheduler false-positive tracked by #70666.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land or adapt the additive default-true `agents.defaults.reminderGuard` and per-agent `reminderGuard` opt-out with schema/docs/tests, then separately evaluate any Plugin SDK scheduling-signal API if maintainers want external tools to report successful scheduling directly.
