## Summary

Keep #71170 open. Current main still lacks the proposed `reminderGuard` config and still appends the unscheduled-reminder note when a plugin-managed reminder produces no core `cron.add` success. The related issue #70666 was closed in favor of this PR, not because main already shipped the behavior.

## Best Possible Solution

Keep #71170 open as the canonical implementation path for #70666. Rebase it onto current main, review the additive global/per-agent `reminderGuard` config, schema, docs, and tests, then land this PR or an equivalent opt-out. If maintainers want plugin tools to signal successful scheduling directly, track that separately as a plugin SDK API design.
