## Summary

Keep this PR open. Current main still does not preserve Feishu card-action `option`, `options`, or `form_value` through the live callback path, and the PR is paired with same-author open issue #42754. The patch is useful but needs a current-main update because parsing now reconstructs `action` with only `value` and `tag`, while fallback serialization lives in `card-interaction.ts`. It also overlaps broader open Feishu card-action work in #43202 and #43953.

## Best Possible Solution

Keep this PR open as an implementation candidate. The best path is to rebase it on current main, preserve Feishu-native sibling action fields in `parseFeishuCardActionEventPayload`, extend the event/fallback types and `buildFeishuCardActionTextFallback` serialization for `option`, `options`, and `form_value` while preserving existing `value.text` and `value.command` behavior, and add regression coverage from a raw `card.action.trigger` payload, including empty arrays/objects. Coordinate the final scope with #42754, #43202, and #43953.
