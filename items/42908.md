## Summary

Keep open. Current main still drops Feishu card-action `option`, `options`, and `form_value` before the synthetic message reaches the agent. This PR is useful and directly tied to open issue #42754, but it needs a current-main update because the production parser still rebuilds `action` with only `value` and `tag`, and the fallback logic has moved into `card-interaction.ts`.

## Best Possible Solution

Keep this PR open and revise it into a complete current-main Feishu plugin fix: preserve extra action fields in `parseFeishuCardActionEventPayload`, extend the typed event and fallback serialization for `option`, `options`, and `form_value` without breaking button `text`/`command`, add parser-to-handler regression coverage from a raw `card.action.trigger` payload, and coordinate with #42754/#43202/#43953 so one canonical pass-through fix lands.
