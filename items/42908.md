## Summary

Keep this PR open. Current main still drops Feishu card-action sibling fields before the handler/fallback path can serialize them, and this PR is paired with same-author open issue #42754. The proposed change remains useful, but it needs a current-main rebase because Feishu card action parsing and fallback serialization have moved beyond the patch's original handler-only shape.

## Best Possible Solution

Keep this PR open as an implementation candidate. The best path is to rebase on current main, preserve Feishu-native sibling action fields in parseFeishuCardActionEventPayload, extend the event/fallback serialization path for option, options, form_value, and likely the broader input_value/name fields tracked in #43202 and #43953, while preserving value.text/value.command behavior. Add regression tests from raw card.action.trigger payloads, including empty arrays and empty objects.
