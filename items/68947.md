## Summary

Keep open. Current main still has the formatter-corrupted `/export-session` template, and the command implementation still looks for literal `{{MARKED_JS}}`, `{{HIGHLIGHT_JS}}`, and `{{JS}}` tokens that are not present in the template. PR #68947 addresses a real core bug, but it should be revised before merge because the restored string-replacement path can mutate inlined JavaScript containing `$` replacement sequences.

## Best Possible Solution

Keep PR #68947 open and ask for a small revision: restore guarded `{{MARKED_JS}}`, `{{HIGHLIGHT_JS}}`, and `{{JS}}` placeholders, change `generateHtml` to insert script/CSS/session payloads with function replacers or another raw-text-safe path, and keep the new regression test reading the actual template/vendor assets with assertions that no raw placeholders remain and script bytes are preserved. That would resolve the remaining core `/export-session` bug tracked by #49957 and supersede the stale unmerged #20816 attempt.
