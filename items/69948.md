## Summary

Keep PR #69948 open. Current main has the lower-level Gateway `sessions.send` method, but it still does not expose the requested plugin-facing `api.runtime.interSession.send` contract. This is a real core/plugin-SDK boundary extension, not a ClawHub-only plugin request, and the provided PR discussion shows useful implementation work with remaining review concerns rather than obsolete work.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to rebase it onto current main, preserve the narrow `api.runtime.interSession.send({ sessionKey, message, idempotencyKey? }) -> { messageRef }` contract if maintainers want this capability, update SDK docs/API baseline/tests, and resolve the cache-mode and runtime-scoping review concerns. If maintainers decide plugins should not get this send-only surface, close it by explicit product judgment rather than automated stale/duplicate cleanup.
