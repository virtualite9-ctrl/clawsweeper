## Summary

Keep PR #69948 open. Current main has the underlying `sessions.send` gateway method, but it does not expose the proposed plugin-facing `api.runtime.interSession.send` contract, runtime type, gateway binding, plugin-id scoping, error codes, docs, or tests. The supplied PR context shows focused work toward that missing SDK surface, plus review follow-up activity; the latest review note still flags a cache-mode edge, so this is active PR work rather than cleanup-close material.

## Best Possible Solution

Keep the PR open and finish it if maintainers want this SDK surface: add `PluginRuntime.interSession.send` as a narrow facade over `sessions.send`, document the opaque `messageRef`, keep API baseline/docs/tests aligned, preserve plugin-id scoping, and fix the runtime cache-mode edge for explicit interSession overrides before landing. If maintainers decide plugins should not send cross-session messages, close by explicit product decision rather than automated cleanup.
