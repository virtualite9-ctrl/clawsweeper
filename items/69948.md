## Summary

Keep PR #69948 open. Current main still exposes `sessions.send` as a gateway RPC, but it does not expose the proposed plugin-facing `api.runtime.interSession.send` surface. This is a core plugin SDK/runtime seam, not something an external ClawHub plugin can add by itself, and the PR also has useful review context that should be resolved rather than auto-closed.

## Best Possible Solution

Keep the PR open. The best path is to finish the narrow core plugin API seam for `api.runtime.interSession.send`, with SDK types, docs, API baseline, gateway facade/error mapping, plugin-id scoping, and tests. Before merge, resolve the runtime cache-key concern for combined `allowGatewaySubagentBinding` plus explicit `runtimeOptions.interSession`, and have a maintainer review the security/product implications of plugin-driven cross-session sends.
