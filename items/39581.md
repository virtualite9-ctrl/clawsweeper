## Summary

Keep PR #39581 open. Current main still emits the embedded-runner `llm_output` hook whenever the hook is registered, while `lastAssistant` can be undefined or can refer to an earlier assistant message rather than the current timed-out/no-output attempt. The PR targets a real remaining bug, but it should be narrowed because the branch also contains unrelated `chat-content` JSON parsing changes with unresolved review concerns.

## Best Possible Solution

Keep the PR open, but request a focused patch: guard embedded-runner `llm_output` on actual current-attempt assistant output, avoid stale `lastAssistant`, preserve normal-output hook emission, add regression coverage for timeout/no-output and normal-output cases, and split or drop the unrelated `chat-content` JSON parsing change unless it is fixed and justified separately.
