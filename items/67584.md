Current main still lacks context-aware memory-wiki vault.path resolution, and this PR is a recent, active implementation candidate with targeted follow-up commits addressing review concerns. It should not be closed by cleanup; it needs normal maintainer review, CI, and a product decision against the broader per-agent config direction in #63829.

## Risks / Open Questions

- Maintainers still need to choose whether ctx-aware `vault.path` templating is the desired narrow fix or whether the broader per-agent memory-wiki configuration tracked in #63829 should own the design.
- The PR changes memory-wiki data isolation and filesystem path resolution behavior, so targeted memory-wiki tests and CI/security review should gate any merge.
- The branch intentionally skips memory prompt/corpus supplements when `vault.path` is templated; that limitation is reasonable for minimum scope but needs maintainer acceptance because it changes `memory_search`/`memory_get` availability for templated setups.
