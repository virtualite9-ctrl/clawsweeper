## Summary

Keep open. The timeout-config part appears covered on current main, but I did not find strong evidence that the remaining Ollama provider-registration/fallback-loop symptom from this issue is fixed. The issue discussion explicitly split those two symptoms, and the reporter confirmed the remaining one after upgrading.

## Evidence

- **Maintainer already split the bug into fixed vs still-open parts:** Issue comment from @steipete on 2026-04-08 says the `agents.defaults.timeoutSeconds` part was fixed on main, but the issue was intentionally left open because the `No API provider registered for api: ollama` and failover-loop symptoms might still need separate triage.
- **Reporter confirmed the remaining symptom after upgrading:** The reporter's 2026-04-08 follow-up says they were still seeing fallbacks on the latest OpenClaw build, with logs showing context-overflow auto-compaction followed by repeated `No API provider registered for api: ollama` errors.
- **Current main intends to retry the same model after overflow compaction:** The overflow path increments a compaction attempt counter, runs compaction, and if compaction succeeds logs `auto-compaction succeeded ...; retrying prompt` before continuing the same provider/model attempt. That means the reported fallback-loop behavior is not something I can dismiss as expected behavior from current code.
