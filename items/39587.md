## Summary

Keep #39587 open. Current main still routes `openclaw docs <query>` through the hosted `https://docs.openclaw.ai/mcp.SearchOpenClaw` tool via `mcporter`, the docs still promise live docs-index search, the reporter’s target Browser and Skills docs exist, and the hosted search path still does not return a successful result for the reporter queries. Related PR #39632 was closed unmerged and only addressed missing-`mcporter` messaging, not the failing search backend.

## Best Possible Solution

Keep #39587 open. Restore the hosted docs search backend or move `openclaw docs` to a stable searchable docs index/API, then add focused smoke coverage for `browser extension` and `Skills` plus clear error reporting when the backend is unavailable.
