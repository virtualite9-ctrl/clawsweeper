## Summary

Current main still leaves the docs search regression unresolved. The CLI continues to send `openclaw docs <query>` through the live `https://docs.openclaw.ai/mcp.SearchOpenClaw`/`mcporter` path, docs still promise live index search, and the public Mintlify search endpoint returns HTTP 500 for both reporter examples while the Browser and Skills docs exist locally and on the public docs site. Related PR #39632 was closed unmerged and only covered missing-`mcporter` messaging, not the failing live search backend.

## Best Possible Solution

Keep #39587 open. The best fix is to restore the live docs search backend or move `openclaw docs` to a stable searchable docs index/API, then add a focused smoke check for successful Browser/Skills queries and clear error reporting when the backend is unavailable.
