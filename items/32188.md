## Summary

Keep #32188 open. Current main has adjacent surfaces for model selection, per-message token/cost/model metadata, threshold-based context warnings, transient fallback/compaction indicators, `/usage`, and usage/auth dashboards, but it still lacks the requested persistent in-interface status area/status bar showing the active agent/model plus live token, quota, and subscription context during operations.

## Best Possible Solution

Keep the enhancement open. The best path is a first-class persistent Control UI status surface in the chat/application chrome that shows active agent/session/run state and selected provider/model, then layers in privacy-aware context usage, token/cache/cost, provider quota windows, auth/subscription state, and fallback state using existing sessions, model auth, usage/cost, and run event data where available.
