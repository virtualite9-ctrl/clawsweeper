## Summary

Keep #32188 open. Current main still has partial surfaces for model/session selection, slash-command usage/status, overview usage/auth cards, per-message metadata, and session tables, but it does not provide the requested persistent in-interface status area showing the active agent/model plus live token/quota/subscription context during operations.

## Best Possible Solution

Keep the enhancement open and implement a first-class persistent status surface in the Control UI/application chrome. It should show the active agent/session/run state and selected provider/model, then add privacy-aware context usage, token/cache/cost, provider quota windows, auth/subscription state, and fallback information using existing sessions.list/sessions.changed, usage.status, models.authStatus, model/auth, and session event data where possible.
