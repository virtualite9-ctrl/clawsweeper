## Summary

Current main already fixes the PR's doctor false-positive problem by using the registry-only memory embedding provider helper during memory-search config resolution. That removes the old runtime capability-provider/compat-config path entirely, so the missing-cfg failure mode from #68110 and #68088 is obsolete. The linked discussion was useful, and the PR author's follow-up addressed the Greptile cleanup comment, but the branch is no longer needed.

## Best Possible Solution

Close this PR as implemented on main. Keep the shipped runtime-free memory-search config-resolution design, where doctor/startup config checks read already-registered provider metadata and actual plugin capability loading stays in runtime execution paths. No PR branch changes are needed.

## Evidence

The Greptile cleanup was addressed in dabfdf6b, and the later Aisle warning reinforces why current main's runtime-free config path is the better shape.
