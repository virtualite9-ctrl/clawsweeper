## Summary

Keep open. Current main supports switching and managing sessions, but it does not implement the requested tab strip for multiple simultaneously open chat sessions with create/switch/close/reorder/persistent tab state. The provided related items are about unrelated session/message filtering or routing work, not a canonical tab-system issue.

## Best Possible Solution

Keep the issue open as a core Control UI/WebChat ergonomics feature. A good implementation would add a first-class chat tab strip around the existing `sessions.list`, `sessions.patch`, `sessions.delete`, and `chat.history` flows, preserve per-session draft/attachment/UI state, sync the active tab to the URL and stored last-active session, and add focused UI tests for switch/create/close/reorder and responsive behavior. This should not be moved to ClawHub unless maintainers first expose a plugin API for replacing core chat navigation.
