## Summary

Keep #72045 open. Current main still lacks the main qa-lab fixes this PR carries: blank suite/gateway model refs are forwarded as explicit empty strings instead of falling back to mock defaults, the QA gateway enables memory-core but does not pin the memory slot, parity report labels can remain blank when OPENCLAW_CI_OPENAI_MODEL is unset, and session creation has no transient session-store lock retry. One mock-provider detail is already present on main, but the PR remains active, focused, and not superseded.

## Best Possible Solution

Keep #72045 open for maintainer review and likely merge after checking the latest CI. The best product path is to preserve the blank model normalization, memory slot pin, parity label fallback, regression coverage, and targeted session-lock retry, while optionally simplifying the now-redundant mock-provider prompt OR branch before landing.
