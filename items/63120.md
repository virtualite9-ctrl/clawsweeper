## Summary

Keep open. Current main still exposes `streamTo` in the ACP-enabled `sessions_spawn` schema without ACP-only guidance, and the executor still rejects `runtime="subagent"` calls that include `streamTo:"parent"` before reaching the subagent spawn path. This issue also has an open same-author PR with closing syntax (#63121) and an open maintainer replacement PR (#72331), so it should not be closed before one of those lands or maintainers explicitly choose a different path.

## Best Possible Solution

Keep this issue open until an implementation lands. The best maintainer path is to review and land #72331, or otherwise resolve the same-author schema-guidance PR #63121 as part of the same fix. The shipped behavior should both document `streamTo` as ACP-only and prevent strict schema-following providers from breaking `runtime="subagent"` spawns when ACP-only fields are included.
