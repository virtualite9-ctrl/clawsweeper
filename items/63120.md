## Summary

Keep open. Current main still exposes `streamTo` in the ACP-enabled `sessions_spawn` schema without an ACP-only description and still returns `streamTo is only supported for runtime=acp` before dispatching a `runtime="subagent"` spawn. The issue is also paired with open PR #63121 from the same author using closing syntax, and open maintainer PR #72331 explicitly says it covers this issue after that fix lands.

## Best Possible Solution

Keep this issue open until an implementation lands. The best maintainer path is to review and land the focused tolerant-runtime fix in #72331, or otherwise land/close the paired same-author schema-guidance PR #63121 as part of the same resolution. The shipped behavior should both document `streamTo` as ACP-only in the schema/tool description and stop strict schema-following providers from bricking `runtime="subagent"` spawns when they include ACP-only fields.
