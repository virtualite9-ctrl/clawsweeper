## Summary

Keep open. Current main still has the Discord thread-reply alias mismatch: `send` accepts `media`, `path`, and `filePath`, but `thread-reply` still only reads `media` before forwarding `mediaUrl`. The same-author open PR #43574 explicitly targets this issue and is an implementation candidate, so this issue should stay open until that PR is reviewed/landed or replaced.

## Best Possible Solution

Review and land or replace PR #43574. The code change should make Discord `thread-reply` read `media`, `path`, and `filePath` with the same precedence as `send`, then add focused regression tests for `filePath`, `path`, and `media` precedence in the Discord action handler.
