## Summary

Keep open. Current main has generic timeout plumbing improvements, but it does not prove or ship a fix for the updated OpenRouter -> Anthropic large tool-parameter path: OpenRouter still resolves to the OpenAI-compatible transport, fine-grained Anthropic tool streaming is only present on native Anthropic-style transports, and there is no regression test for delayed/withheld OpenAI-compatible tool-call argument chunks.

## Best Possible Solution

Keep this issue open as the canonical tracker for OpenRouter/Anthropic large tool-parameter generation failures. Add a synthetic regression where an OpenAI-compatible stream withholds tool-call argument chunks longer than the default idle window, then either enable supported provider-side tool-argument streaming for OpenRouter/Anthropic routes, adjust the idle/request-timeout policy for silent tool-argument generation, or teach the agent/tool loop to split large `Write`-style payloads before one huge tool call is required.
