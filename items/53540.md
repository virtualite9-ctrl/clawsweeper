## Summary

Keep open. Current main still leaves the reported OpenRouter-to-Anthropic large tool-parameter path on the OpenAI-compatible streaming transport with a default 120s no-chunk LLM idle watchdog, and I did not find source, docs, tests, or release evidence proving the rewritten large-argument reproducer succeeds by default.

## Best Possible Solution

Keep this as the canonical tracker for OpenRouter/Anthropic large tool-parameter generation failures. A maintainer fix should add a synthetic regression where an OpenAI-compatible stream withholds tool-call argument chunks longer than the default idle window, then either enable supported OpenRouter/Anthropic tool-argument streaming, adjust the LLM idle policy for silent tool-argument generation, or add a core/tool-loop strategy that avoids requiring one large silent Write-style payload.
