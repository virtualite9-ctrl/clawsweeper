## Summary

Keep #12911 open. Current main has a browser Talk loop in `/chat`, but it is still OpenAI Realtime/WebRTC-specific and xAI/Grok realtime voice is explicitly documented and registered as not exposed. Recent maintainer comments also keep the issue open for the remaining xAI/Grok provider work.

## Best Possible Solution

Keep #12911 open as the tracker for Grok/xAI Live Voice Mode in `/chat`. The concrete path is to make browser Talk provider-aware beyond the current OpenAI WebRTC client, add xAI realtime voice/browser-session support if upstream supports it, extend the realtime voice contract where needed, then update Control UI/provider docs and focused provider/UI tests. If maintainers decide the shipped OpenAI Talk path satisfies the product goal, close only after that explicit product decision.
