## Summary

Keep #12911 open. Current main has a browser Talk loop in `/chat`, but the implemented browser session is still OpenAI Realtime/WebRTC-backed, while the requested Grok/xAI Live Voice Mode remains unimplemented. Current code, docs, manifests, and recent maintainer comments all point to the same remaining work: xAI realtime voice/browser-session support and provider-aware Talk wiring.

## Best Possible Solution

Keep #12911 open as the tracker for Grok/xAI Live Mode in `/chat`. The best path is to make browser Talk provider-aware beyond the current OpenAI-specific WebRTC client, add xAI realtime voice/browser-session support if upstream supports it, wire that through the bundled xAI plugin or an explicit plugin API extension, and update Control UI docs/tests. If maintainers decide the shipped OpenAI Talk mode satisfies the product direction, close only after that explicit product decision.
