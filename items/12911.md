## Summary

Keep #12911 open. Current main has a browser Talk loop in /chat, but it is still OpenAI Realtime/WebRTC-backed and does not implement the requested Grok/xAI Live Voice Mode. Recent maintainer comments also explicitly keep the xAI/Grok scope open.

## Best Possible Solution

Keep #12911 open as the tracker for Grok/xAI Live Mode in /chat. The best path is to make browser Talk provider-aware, add xAI realtime voice/browser-session support if the upstream API supports it, wire it through the bundled xAI plugin or an explicit plugin API extension, and update Control UI docs/tests. If maintainers decide the shipped OpenAI Talk mode satisfies the product direction, close only after that explicit product decision.
