## Summary

Keep #71656 open. Current main has meaningful partial fixes for dreaming corpus noise and some scan fairness, and deep promotion does use weighted/recency-aware ranking, but the specific flat ingestion-score complaint is still true on main: daily-ingested snippets still enter at 0.62 and session-corpus snippets still enter at 0.58 before promotion ranking. The issue is recent, author is not protected, and no protected labels are present, but the remaining request is concrete and not fully covered by the related reports.

## Best Possible Solution

Keep the issue open and narrow it into implementation work: add content-type-aware source scoring before `recordShortTermRecalls` for daily/session dreaming ingestion, extend corpus pre-filtering/deduplication for repeated blocks and low-signal test/status messages, and add regression coverage proving session-corpus scans continue across files/days under budget pressure. If maintainers want to consolidate scoring policy, link or split the importance-scoring portion under #57307 while keeping the corpus filtering and scan-rotation specifics tracked here.
