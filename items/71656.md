## Summary

Keep #71656 open. Current main has partial session-corpus noise filtering and weighted/recency-aware deep promotion, but the reporter's central remaining complaint is still present: daily and session-corpus ingestion still use fixed source scores before promotion ranking, and there is no clear regression coverage proving session-corpus scans rotate fairly under budget pressure.

## Best Possible Solution

Keep #71656 open and narrow it into implementation work in `extensions/memory-core/src/dreaming-phases.ts` and the session export path: add content-type-aware ingestion scoring before `recordShortTermRecalls`, extend corpus pre-filtering/deduplication for repeated low-signal blocks, and add regression coverage proving session-corpus ingestion continues across files/days under sweep-budget pressure. If maintainers want one canonical scoring policy thread, link the importance-scoring portion to #57307 while keeping this issue for session corpus filtering and scan rotation.
