## Summary

Current `main` already implements this routing behavior in the newer native approval path. Discord approval origin resolution now prefers Discord turn-source metadata and only falls back to stored/session-derived routing afterward, with checked-in tests covering the relevant channel/thread cases.

- **Discord origin routing prefers turn-source metadata:** `createDiscordOriginTargetResolver` resolves `turnSourceChannel`/`turnSourceTo`/`turnSourceThreadId` first, normalizes Discord channel/thread targets, and only returns `null` to fall through when those hints are unusable.
