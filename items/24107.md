## Summary

Keep #24107 open. Current `main` still lacks the requested generic provider-level `rpm`/`rpd` configuration, quota-window counters, and pre-request quota skip in `runWithModelFallback`. The code has generic stepped auth-profile cooldowns, model-scoped rate-limit cooldowns, and OpenAI Codex WHAM-specific probing, but those do not satisfy the requested configurable RPM/RPD behavior for arbitrary providers.

## Best Possible Solution

Keep #24107 open as a core model fallback/auth-profile enhancement. The best path is to add a validated provider quota contract (`rpm`/`rpd` or an equivalent explicit shape), update generated schema/help/docs, persist minute/day quota state with reset timestamps, add a pre-request quota skip that marks `cooldownUntil` in `runWithModelFallback`, and cover RPM/RPD reset behavior with focused tests alongside existing cooldown/probe coverage.
