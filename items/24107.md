## Summary

Keep #24107 open. Current main has a provider-specific OpenAI Codex WHAM cooldown probe, but it still does not implement the requested generic provider-level `rpm`/`rpd` config contract, persisted RPM/RPD quota-window accounting, or pre-request quota skip in model fallback.

## Best Possible Solution

Keep #24107 open as a core model fallback/auth-profile enhancement. A complete fix should add a validated provider quota config/API contract, persisted per-provider or per-profile minute/day counters and reset timestamps, pre-request skip/mark-cooldown logic in `runWithModelFallback`, integration with `cooldownUntil` and probe decisions, generated schema/docs updates, and focused tests for RPM and RPD reset behavior.
