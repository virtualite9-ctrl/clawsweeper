## Summary

Keep #24107 open. Current main has adjacent cooldown and provider-usage work, but it still lacks the requested generic provider-level `rpm`/`rpd` config contract and pre-request quota-window accounting in model fallback. The existing maintainer comment already reached the same keep-open conclusion, and current main still matches it.

## Best Possible Solution

Keep this open as a core model fallback/auth-profile enhancement. A complete fix likely needs a validated provider quota config/API contract, persisted per-provider or per-profile quota-window counters and reset timestamps, pre-request skip/mark-cooldown logic in `runWithModelFallback`, integration with cooldown probing and `cooldownUntil`, generated schema/docs updates, and focused RPM/RPD tests for minute and day reset behavior.
