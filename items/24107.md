## Summary

Keep #24107 open. Current main has cooldown improvements, including OpenAI Codex WHAM-aware cooldowns, model-scoped rate-limit cooldowns, and shorter generic backoff, but it still lacks the requested generic provider-level `rpm`/`rpd` config contract and pre-request quota-window accounting in model fallback.

## Best Possible Solution

Keep tracking this as a core model fallback/auth-profile enhancement unless maintainers choose a different product direction. A complete fix likely needs an explicit provider quota config/API contract, persisted quota-window accounting, pre-request skip logic in the fallback path, `cooldownUntil` integration, and focused tests for RPM/RPD reset behavior and cooldown probing.
