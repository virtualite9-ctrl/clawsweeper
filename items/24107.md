## Summary

Keep #24107 open. Current main still lacks a generic provider-level RPM/RPD config contract and pre-request quota-window accounting in model fallback; the existing maintainer comment also explicitly keeps the item open.

## Best Possible Solution

Keep this open as a core model fallback/auth-profile enhancement. A complete fix likely needs a validated provider quota config/API contract, persisted per-provider or per-profile quota-window counters and reset timestamps, pre-request skip/mark-cooldown logic in runWithModelFallback, integration with cooldown probing/cooldownUntil, generated schema/docs updates, and focused RPM/RPD tests for minute and day reset behavior.
