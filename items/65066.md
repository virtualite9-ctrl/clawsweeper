## Summary

Keep this PR open. Current main still lacks a memory runtime/capability registration in the bundled `memory-lancedb` plugin, while doctor and status still depend on the active memory runtime path. The PR remains a relevant implementation candidate for the linked doctor/status false-negative, though it needs a rebase and current-contract review before landing.

## Best Possible Solution

Keep this PR open as the implementation candidate. Rebase it onto current main, adapt the memory-lancedb runtime adapter and tests to the current plugin config/test shape, review it against the current memory SDK contract, then land it or request targeted changes.
