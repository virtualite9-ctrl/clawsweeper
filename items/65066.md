## Summary

Keep this PR open. Current main still has no memory runtime/capability registration in `memory-lancedb`, while doctor and status still depend on the active memory runtime to report memory health. The PR remains a focused implementation candidate for the linked doctor/status false-negative rather than something obsolete or out of scope.

## Best Possible Solution

Keep this PR open as the implementation candidate. Rebase it onto current main, review the runtime adapter against the current memory SDK contract, keep the regression coverage for doctor/status visibility, then either land it or request targeted changes.
