## Summary

Keep open. Current `main` still has the broken Docker troubleshooting command in `docs/install/docker.md`, while the gateway code and existing CLI docs require explicit `--token` or `--password` whenever `--url` is provided. The PR addresses a real docs mismatch and is not already implemented on `main`.

## Evidence

- **Docker doc on main still omits explicit auth:** The current troubleshooting snippet still tells users to run `devices list --url ws://127.0.0.1:18789` with no `--token`/`--password`, matching the bug described in the PR rather than the proposed fix.
