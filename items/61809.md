## Summary

Current `main` already contains a Telegram-specific shutdown/streaming fix path that matches the reporter's follow-up root-cause analysis: channel stop aborts the running Telegram account, Telegram bot fetches are wired to abort immediately on shutdown, and same-session stale preview/finalization work is fenced so short orphan fragments are not flushed after an abort/restart race.

## Evidence

- **Gateway channel stop aborts running Telegram accounts during restart/shutdown:** `stopChannel` aborts the stored account controller before waiting for the task to finish, and passes that abort signal into the plugin stop path.
  - file: [src/gateway/server-channels.ts:547](https://github.com/openclaw/openclaw/blob/342583348da75f2b9e739e75352a63ab2d4ea39f/src/gateway/server-channels.ts#L547)
