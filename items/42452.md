It is not merge-ready without rebase and transport/security rework, but none of the conservative close reasons apply.

## Best Possible Solution

Keep this PR open as the canonical implementation candidate, but require rebase onto current main. The best implementation should preserve the current lazy probe import boundary, remove unrelated `.gitignore` churn, document the opt-in billable extra provider call at https://docs.openclaw.ai/cli/models, and either capture headers through the existing provider runtime/transport path or add a narrow generic seam so rate-limit probing honors normalized provider config, plugin transport hooks, custom headers/authHeader, timeout/request config, and provider-owned behavior. Add focused tests for CLI flag forwarding, JSON/table output, provider base URL normalization, and the no-eager-embedded-runner import contract before maintainer review.
