## Summary

Current `main` already refreshes the gateway chat avatar on session switch, guards against stale avatar fetches, and that behavior is present in the shipped `v2026.4.23` release, which is newer than the reporter's `2026.4.2` version.

## Evidence

- **Session switch now refreshes the chat avatar:** `switchChatSession` resets session-scoped chat state and immediately calls `refreshChatAvatar(state)` when the selected session changes, matching the missing step described in the report.
  - file: [ui/src/ui/app-render.helpers.ts:509](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/ui/src/ui/app-render.helpers.ts#L509)
  - command: `nl -ba ui/src/ui/app-render.helpers.ts | sed -n '509,522p'`
  - sha: [926068b14fc6](https://github.com/openclaw/openclaw/commit/926068b14fc661e915a0247e9ecb5949fb95b4f7)
- **Avatar loading is keyed to the active session:** `refreshChatAvatar` snapshots `sessionKey`, fetches the active agent avatar metadata, and drops results if the host has switched sessions before the fetch resolves, preventing the previous session's avatar from sticking until reload.
  - file: [ui/src/ui/app-chat.ts:620](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/ui/src/ui/app-chat.ts#L620)
