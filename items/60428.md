## Summary

Keep open. Current `main` still stamps `createdAt` from `Date.now()` even when `startedAt` is already provided, and the audit code still warns when `startedAt < createdAt`, so the false-positive path described by the PR is not implemented away on `main`. The PR discussion also surfaced an unresolved correctness concern: the proposed clamp should guard non-finite `startedAt` values before merge.

## Evidence
