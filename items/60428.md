## Summary

Current main already implements PR #60428's observable task timestamp fix. The narrow createdAt clamp is now covered by a broader registry normalization path that applies on create, update, and restore, with regression tests asserting earlier startedAt values no longer produce inconsistent_timestamps audit findings.

## Best Possible Solution

Close #60428 as implemented on main. Keep the broader #71871 task timestamp normalization rather than merging this older narrower PR, since main now fixes the same audit noise at the registry boundary on create, update, and restore.
