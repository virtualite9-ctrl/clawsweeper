## Summary

Keep #13362 open. Current main still lacks `openclaw doctor --dry-run` or an equivalent doctor-wide no-write validation mode. Doctor can still prompt/apply repairs and write config during config finalization and service repair flows; existing dry-run support is limited to other commands such as config, update, reset, uninstall, sessions cleanup, plugins, and secrets.

## Best Possible Solution

Keep #13362 open until OpenClaw either implements a doctor-specific `--dry-run` mode or maintainers explicitly reject the product direction. A good fix should add the CLI flag, thread `dryRun` through `DoctorOptions` and repair/write flows, block all config/state/service writes while still producing recommendations, define exit-code behavior for found issues, update `docs.openclaw.ai/cli/doctor` and gateway doctor docs, and add focused tests proving doctor recommendations run without mutating config or service state.
