## Summary

Keep #13362 open. Current main still has no `openclaw doctor --dry-run` flag or equivalent doctor-wide no-write mode, and the existing doctor paths still allow config/state/runtime-dependency/service side effects through repair or confirmed preview flows. The related global agent dry-run issue #41418 is complementary, not a doctor/config validation replacement.

## Best Possible Solution

Keep #13362 open until OpenClaw implements a doctor-specific `--dry-run` mode or maintainers explicitly reject the direction. A complete fix should add the CLI flag, thread `dryRun` through `DoctorOptions` and all doctor repair/write contributions, block config/state/service/runtime-dependency writes while still producing recommendations, define exit-code semantics for detected issues, update the public doctor docs, and add focused tests proving dry-run recommendations do not mutate config or service state.
