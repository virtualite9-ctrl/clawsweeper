## Summary

Keep #13362 open. Current main at 406ae72fd2789c78053f6b94006ec8f488bc1c00 still does not expose `openclaw doctor --dry-run` or an equivalent doctor-wide no-write validation mode. Doctor options cannot carry a dry-run state, docs omit it, and current doctor repair paths can still write config, install bundled plugin runtime deps, mutate legacy state, and repair services. The related global agent dry-run issue #41418 is complementary, not a replacement for doctor/config validation.

## Best Possible Solution

Keep #13362 open until OpenClaw either implements or explicitly rejects a doctor-specific dry-run mode. The best implementation is to add `openclaw doctor --dry-run`, thread `dryRun` through `DoctorOptions`, repair-mode resolution, the prompter, config flow, and every doctor contribution, make all repair/write paths emit planned changes without mutating config/state/services/runtime dependencies, define exit-code behavior for findings, update https://docs.openclaw.ai/cli/doctor, and add focused tests proving dry-run does not mutate config, plugin registry state, bundled plugin deps, service state, cron/state/session stores, or plugin/channel-owned doctor repairs.
