## Summary

Keep open. Current main still hardcodes Feishu typing reactions to `Typing` and does not accept or resolve Feishu `ackReaction` config, while the PR contains relevant work plus unresolved review concerns that need maintainer/author handling rather than cleanup closure.

## Best Possible Solution

Keep the PR open and ask for a focused revision. The best path is to add Feishu `ackReaction` support in the Feishu plugin, resolve it in the reply dispatcher with a Feishu-specific `Typing` channel default, preserve empty-string disable semantics, validate or safely handle Feishu emoji type names, and add tests for account, channel, global, identity, fallback, invalid, and empty-string cases. Also align docs and plugin config metadata before merge.
