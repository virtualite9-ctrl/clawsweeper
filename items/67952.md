## Risks / Open Questions

- The PR changes `ltm list` to print stored memory text; that is the requested behavior, but it is still a user-visible local data-disclosure surface and should be intentional in CLI docs/review.
- Default `ltm list` without `--limit` would print every stored memory row because LanceDB plain queries are unbounded by default unless `.limit(...)` is used.
- The PR should add focused regression tests for JSON list output, `--limit`, invalid limits, created-at ordering, and preserving `ltm stats` as count-only.
- The PR branch is based on an older memory-lancedb file; the patch is localized, but it should be rebased or merge-checked so current hook/cursor initialization changes remain intact.
- No tests were run because this was a required read-only review and the checkout had to remain byte-for-byte clean.
