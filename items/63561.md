## Summary

Current main does not contain the proposed #63561 fix and still routes `systemctl --user` through the reported failure path. The related PR #64020 described the right shape of fix, but the current checked-out source has no DBUS/XDG runtime env derivation and still treats `Failed to connect to bus: Permission denied` as user-systemd unavailable.

## Best Possible Solution

Keep #63561 open and implement the narrowly scoped systemd spawn-env fix in `src/daemon/systemd.ts`: preserve existing env values, derive missing `XDG_RUNTIME_DIR` and `DBUS_SESSION_BUS_ADDRESS` on Linux from the current uid when safe, wire that env into all `systemctl --user` spawns, and add regression tests for the npm-wrapper/missing-env permission-denied case. The existing closed PR #64020 is useful prior art but is not shipped on current main.
