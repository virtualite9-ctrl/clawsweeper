## Summary

Keep open. Current main still contains the reported channel stop-timeout failure mode: the timeout branch records `running: true`, returns before deleting `store.aborts`/`store.tasks`, and the current regression test still asserts that a restart after timeout is skipped. PR #70056 is the linked active fix path, but it remains open/unmerged and has a review concern about a secondary zombie-task race, so the issue should not be closed yet.

## Best Possible Solution

Keep #70024 open and use PR #70056, or an equivalent maintainer patch, as the active fix path. The fix should delete stale task/abort entries on stop timeout, mark the account `running: false`, set `lastStopAt`, update the regression test so restart after timeout succeeds, and address the zombie-task finalizer race called out on the PR before merge.
