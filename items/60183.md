## Summary

Keep open. Current `main` still computes TaskFlow audit freshness from the flow record only, so the PR's linked-child-activity fix is not implemented on `main`, and the regression coverage described in the PR is also absent.

## Evidence

- **Current freshness logic ignores linked task activity:** `getReferenceAt` returns only `flow.updatedAt ?? flow.createdAt`, so child task `lastEventAt`/`startedAt` cannot keep a managed flow fresh.
  - file: [src/tasks/task-flow-registry.audit.ts:79](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/src/tasks/task-flow-registry.audit.ts#L79)
  - command: `nl -ba src/tasks/task-flow-registry.audit.ts | sed -n '60,210p'`
  - sha: [926068b14fc6](https://github.com/openclaw/openclaw/commit/926068b14fc661e915a0247e9ecb5949fb95b4f7)
- **Audit age is computed before any linked-task freshness adjustment:** `listTaskFlowAuditFindings` computes `referenceAt` and `ageMs` from the flow, then separately loads `linkedTasks`; the `stale_running` finding therefore still fires based only on the flow timestamps.
  - file: [src/tasks/task-flow-registry.audit.ts:161](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/src/tasks/task-flow-registry.audit.ts#L161)
  - command: `nl -ba src/tasks/task-flow-registry.audit.ts | sed -n '60,210p'`
  - sha: [926068b14fc6](https://github.com/openclaw/openclaw/commit/926068b14fc661e915a0247e9ecb5949fb95b4f7)
- **Regression test for linked-task freshness is not present on main:** The current test file covers suppressing `missing_linked_tasks` when a linked task exists, but there is no test asserting that recent child-task activity suppresses `stale_running` for an otherwise old managed flow.
  - file: [src/tasks/task-flow-registry.audit.test.ts:137](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/src/tasks/task-flow-registry.audit.test.ts#L137)
  - command: `nl -ba src/tasks/task-flow-registry.audit.test.ts | sed -n '100,190p'`
  - sha: [926068b14fc6](https://github.com/openclaw/openclaw/commit/926068b14fc661e915a0247e9ecb5949fb95b4f7)
- **PR discussion matches a real gap rather than an obsolete request:** The PR body and Greptile review both describe the same missing behavior: TaskFlow audit freshness should consider linked child task activity (`lastEventAt ?? startedAt ?? createdAt`) to avoid false-positive `stale_running` findings for still-progressing managed flows.
