## Summary

Current `main` does not show a Gateway crash for invalid cron expressions on the normal `cron.add` path. The add still throws during `createJob`, but the gateway request loop catches handler failures and returns an error response instead of crashing the process. Malformed persisted cron jobs are already isolated during load/maintenance.
