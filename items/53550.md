## Summary

Keep open. Current main still has QMD session transcript export and session-memory search infrastructure, but the reporter’s config path remains insufficient for the reported gateway-to-DM recall case: `agents.defaults.memorySearch.experimental.sessionMemory` permits the `sessions` source for the memory-search config, while QMD transcript export is still separately gated by `memory.qmd.sessions.enabled`, and same-agent cross-channel session hits remain hidden by the default `tools.sessions.visibility="tree"` policy unless explicitly widened.

## Best Possible Solution

Keep this issue open. The conservative maintainer path is to either bridge `agents.defaults.memorySearch.experimental.sessionMemory` plus `sources: ["sessions"]` into QMD transcript export when `memory.backend` is `"qmd"`, or document and diagnose that QMD transcript recall requires `memory.qmd.sessions.enabled: true` plus `tools.sessions.visibility: "agent"` or `"all"` for same-agent cross-channel gateway/DM recall. Add a regression test with a gateway-style `agent:<id>:paperclip:issue:<uuid>` session and a DM requester such as `agent:<id>:whatsapp:direct:*`.
