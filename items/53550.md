## Summary

Current `main` clearly supports session-transcript indexing in both the builtin index and the QMD backend, and the transcript enumeration is adapter-agnostic. But the QMD path is still gated by `memory.qmd.sessions.enabled`, while the issue’s reported config only sets `agents.defaults.memorySearch.experimental.sessionMemory` plus `sources: ["memory","sessions"]`. Because that exact qmd-backed configuration is not wired together in current code, there is not strong enough evidence to close this as fixed on `main`.
