## Summary

Keep open. Current `main` already HTML-escapes normal Telegram text, so the original root-cause theory is outdated, but the user-visible truncation bug still exists on the Telegram path when `splitTelegramReasoningText()` sends text through strict reasoning-tag stripping and an unclosed literal `<think>` causes the remainder of the answer to be dropped before formatting.

## Evidence

- **Telegram formatter escapes raw HTML text:** `markdownToTelegramHtml()` renders Telegram HTML with `escapeText: escapeHtml`, and `escapeHtml()` encodes `&`, `<`, and `>`. That means literal angle-bracket text is safe once it reaches the formatter.
  - file: [extensions/telegram/src/format.ts:18](https://github.com/openclaw/openclaw/blob/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4/extensions/telegram/src/format.ts#L18)
  - sha: [f3c37a946cc4](https://github.com/openclaw/openclaw/commit/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4)
- **Telegram reasoning split still uses strict stripping on final answer text:** `splitTelegramReasoningText()` computes `strippedAnswer` with `stripReasoningTagsFromText(text, { mode: "strict", trim: "both" })` and returns that as `answerText`, so Telegram can lose text before HTML formatting runs.
  - file: [extensions/telegram/src/reasoning-lane-coordinator.ts:82](https://github.com/openclaw/openclaw/blob/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4/extensions/telegram/src/reasoning-lane-coordinator.ts#L82)
  - sha: [f3c37a946cc4](https://github.com/openclaw/openclaw/commit/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4)
- **Strict mode drops the tail after an unclosed reasoning tag:** `stripReasoningTagsFromText()` only appends the remaining suffix when `!inThinking || mode === "preserve"`. If a literal `<think>` opens and never closes, `inThinking` stays true and the rest of the message is discarded.
  - file: [src/shared/text/reasoning-tags.ts:88](https://github.com/openclaw/openclaw/blob/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4/src/shared/text/reasoning-tags.ts#L88)
  - sha: [f3c37a946cc4](https://github.com/openclaw/openclaw/commit/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4)
- **Current tests explicitly lock in the truncation behavior:** The strict-mode test expects `Before <think>unclosed content after` to become `Before`, confirming current `main` intentionally drops the answer tail for unclosed tags.
  - file: [src/shared/text/reasoning-tags.test.ts:225](https://github.com/openclaw/openclaw/blob/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4/src/shared/text/reasoning-tags.test.ts#L225)
