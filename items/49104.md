## Summary

Keep #49104 open. Current main escapes raw Markdown text before Telegram HTML delivery, so the broad parse_mode injection path is partly covered, but the issue discussion’s later confirmed tail-drop path is still present: Telegram reasoning splitting calls strict reasoning-tag stripping, and strict mode still drops text after an unclosed `<think>` tag before formatting or Telegram delivery can recover it. The linked #49996 is an open proposed fix, not shipped main behavior.

## Best Possible Solution

Keep #49104 open as the canonical bug until main ships a narrow fix for the unclosed reasoning-tag tail-drop path. The likely fix belongs in `src/shared/text/reasoning-tags.ts` or an explicit final-answer option used by `extensions/telegram/src/reasoning-lane-coordinator.ts`, with regression coverage proving literal/unclosed `<think>` mentions preserve answer text without leaking genuine reasoning blocks. #49996 can remain the candidate implementation only after maintainer review and a clean, focused diff.
