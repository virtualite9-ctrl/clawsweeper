## Summary

Keep open. Current main has a pre-compaction memory flush, but it is still a single-plan, single-threshold, once-per-compaction-cycle mechanism. The requested progressive staged flush behavior, stage-specific prompts, and forced-summary handoff are not implemented, and the closed docs-only PR #43102 does not supersede the feature request.

## Best Possible Solution

Keep this issue open for maintainer design. A complete implementation likely belongs in the Gateway/core memory-flush pipeline plus the memory plugin contract: add staged threshold configuration or a staged flush-plan API, persist per-stage completion metadata instead of only `memoryFlushCompactionCount`, support stage-specific prompts, define the final forced-summary/compaction handoff, and update schema, docs, and tests.
