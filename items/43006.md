## Summary

Current main has pre-compaction memory flush, but it remains a single-threshold, once-per-compaction-cycle mechanism. The requested progressive 80/90/95-style stages, stage-specific prompts, and forced-summary behavior are not implemented. The linked PR #43102 was a docs-only RFC and closed unmerged, and the related token-budget/durability issues are adjacent rather than canonical superseders.

## Best Possible Solution

Keep this issue open for maintainer product/design review. A complete implementation likely belongs in the Gateway/core memory-flush pipeline or a new plugin API: add staged threshold configuration or a staged flush-plan contract, persist per-stage completion metadata instead of only `memoryFlushCompactionCount`, support stage-specific prompts, define the final forced-summary/compaction handoff, and update schema, docs, and tests.
