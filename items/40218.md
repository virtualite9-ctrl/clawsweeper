## Summary

Keep this PR open. Current main still lacks caller-provided cron job ids and duplicate-id validation, while the latest provided PR context shows an active, focused implementation candidate for that core cron/API gap.

## Best Possible Solution

Keep the PR open and converge it as the implementation candidate: add optional stable cron ids through CronJobCreate/createJob, gateway cron.add, CLI --id, agent cron-tool flat recovery, docs, and generated clients; reject duplicates case-insensitively under the add lock with malformed-store guards; preserve Swift source compatibility for optional id; then land after focused cron/protocol/client validation.
