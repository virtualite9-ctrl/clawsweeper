## Summary

Current `main` already handles long Telegram `sendPayload` text by routing payload sends through the extension-owned Telegram sender, which chunks HTML-mode text to a 4000-character safety limit and keeps inline buttons on the final chunk. The requested fix is therefore already implemented on `main`, although not yet in the latest tagged release from this checkout.
