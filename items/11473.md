## Summary

Keep #11473 open. Current main still has no token-free static cron message payload: the cron types, Gateway schema, validation, runtime, CLI, Control UI, and docs remain built around `systemEvent` and `agentTurn`. Related #18160 overlaps on avoiding model calls but is broader direct-exec work, remains open, and does not currently supersede this fixed text-to-channel request.

## Best Possible Solution

Keep #11473 open until OpenClaw implements an additive token-free static message cron payload, or maintainers explicitly supersede it with a broader non-agent cron payload design that covers fixed text-to-channel delivery. A complete implementation should update the core cron payload type, Gateway TypeBox schema, normalization/validation, runtime direct outbound delivery with run-log/audit/failure reporting, CLI and Control UI create/edit flows, docs, and focused tests. #18160 is worth cross-referencing for broader direct-exec design, but it should not close this narrower request yet.
