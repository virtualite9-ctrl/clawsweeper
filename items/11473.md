## Summary

Keep #11473 open. Current main still has no token-free static cron message payload: cron payloads, Gateway validation, runtime execution, CLI, Control UI, and docs remain built around `systemEvent` and `agentTurn`. The related #18160 direct exec request is broader and does not currently supersede this narrower static text-to-channel request, and the issue discussion already includes a maintainer-authored Codex review saying to keep this item open.

## Best Possible Solution

Keep #11473 open until maintainers either implement a first-class token-free static message cron payload or explicitly supersede it with a broader non-agent cron payload design. The implementation would need an additive cron payload contract, Gateway schema/update normalization, runtime direct channel delivery with audit/failure reporting, CLI and Control UI creation/editing support, docs, and focused tests.
