## Summary

Keep open. Current main has broad token-usage fixes, and related reports cover OpenAI-compatible usage normalization and transcript-backed session totals, but I did not find direct source/test/history evidence that the specific `sessions_spawn` child JSONL transcript bug is fixed. The subagent path still deserves a targeted regression test that verifies the child session's persisted assistant JSONL message has non-zero usage after a completed subagent run.

## Best Possible Solution

Keep the issue open and add a focused regression test around `sessions_spawn`: complete a child run with mocked non-zero provider usage, inspect the child session JSONL assistant entry, and assert non-zero input/output/cache usage plus non-zero completion stats. If that test already passes after wiring through current transport usage, close with that specific proof; otherwise fix the subagent completion/transcript persistence path rather than relying on session-store fallback alone.
