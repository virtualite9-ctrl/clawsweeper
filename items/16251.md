Keep open. Current main still has the exact trusted-proxy docs bullet this PR removes, and the PR is a narrow docs-only change rather than obsolete work. The wording sits on a security-sensitive gateway auth boundary, so the right outcome is maintainer docs/security review, likely with narrower wording rather than an auto-close.

## Best Possible Solution

Keep this PR open for maintainer docs/security review. If maintainers accept the contributor's homelab identity-aware proxy rationale, revise the broad personal single-user warning into narrower guidance: recommend Tailscale Serve plus loopback for simpler personal access, explicitly allow hardened non-loopback identity-aware proxies for personal deployments, and preserve the bypass-path, header-stripping, allowUsers, and loopback-source warnings. Also review adjacent configure wizard wording for consistency.
