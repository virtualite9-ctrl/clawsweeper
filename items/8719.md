## Summary

Keep #8719 open. Current main has narrower security hardening, but it still does not implement the requested native data-centric, secure-by-default Security Profile v1.1. The ClawMoat comment is useful external prior art, and PathGuard-related work covers only a subset rather than superseding the umbrella.

## Best Possible Solution

Keep #8719 open as the product/security umbrella until maintainers define and implement a native Security Profile or split it into canonical narrower issues. Track filesystem PathGuard work separately, treat ClawMoat as complementary prior art, and create or keep follow-ups for data classification/security levels, protected OS presets, cross-tool policy enforcement including exec/export/network, sensitive-action audit events, approval/TTL elevation, session isolation, config integrity, and plugin/skill verification.
