## Summary

Keep #8719 open. Current main has narrower security hardening, but it still does not implement the requested native data-centric, secure-by-default Security Profile v1.1. The issue discussion includes useful external prior art via ClawMoat, and PathGuard/#60981 remains a filesystem-only subset rather than a superseding umbrella.

## Best Possible Solution

Keep #8719 open as the product/security umbrella until maintainers either define a native Security Profile or split it into canonical narrower issues. Track filesystem PathGuard work separately, treat ClawMoat as complementary external prior art, and create or keep follow-ups for data classification/security levels, protected OS presets, cross-tool filesystem/exec/export/network policy, sensitive-action audit events, approval/TTL elevation, session isolation, config integrity, and plugin/skill verification.
