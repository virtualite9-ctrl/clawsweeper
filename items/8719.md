## Summary

Keep #8719 open. Current main has useful hardening primitives, but it still does not implement the requested native data-centric, secure-by-default Security Profile v1.1. The issue discussion also contains an explicit maintainer keep-open review, and the open PathGuard work is only a filesystem subset, not a full substitute for data levels, cross-tool policy, network/export controls, audit logs, TTL elevation, session isolation, and plugin verification.

## Best Possible Solution

Keep #8719 open as the native product/security umbrella until maintainers either define OpenClaw Security Profile v1.1 or split it into canonical narrower issues. Track filesystem sandboxing through #60981/#7722, use ClawMoat as useful external prior art from the discussion, and create or keep follow-ups for data levels, protected OS presets, cross-tool filesystem/exec/export/network policy, append-only sensitive-action audit logs, approval/TTL elevation, session isolation, config integrity, and plugin/skill verification.
