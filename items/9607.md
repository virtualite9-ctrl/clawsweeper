## Summary

Keep #9607 open. Current main still ships the bundled Himalaya skill with the reported wrong move/copy examples, the message-composition export tip still omits the required message ID, and the broader requested email-formatting/html/plain-text/common-mistakes/save-copy guidance is still absent. The related PRs do not fully supersede it: #9610 is closed unmerged, #54032 fixes only three syntax lines, and #59522 covers v1.1/provider guidance but still leaves at least the export-tip and email-formatting requests unresolved.

## Best Possible Solution

Keep #9607 open until a maintainer either merges a broader Himalaya skill docs fix or explicitly narrows the issue. A complete fix should update `skills/himalaya/SKILL.md` and `skills/himalaya/references/message-composition.md`, add or remove the referenced HTML-email guidance coherently, document plain-text versus HTML composition expectations, and include the save-copy/BINARY caveat if that upstream limitation is still relevant. PR #54032 can be reused for the three concrete syntax fixes, but it does not close the whole issue by itself.
