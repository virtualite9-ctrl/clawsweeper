## Summary

Keep open. Current main still does not scan markdown skill definitions: the scanner extension allowlist is code-only and the current test suite still expects `.md` to be non-scannable. This PR is a focused core security-hardening candidate that supersedes closed unmerged #10705, but it still has unresolved reviewer-identified detection gaps in the markdown download-and-exec regex that should be fixed before merge.

## Best Possible Solution

Keep this PR open as the active core security-hardening candidate. Ask for a revision that broadens the markdown download-and-exec rule, adds regression tests for the bypass variants already called out in review, and then runs the relevant changed gate before merge.
