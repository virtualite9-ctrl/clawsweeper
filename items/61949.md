## Summary

Keep open. Current `main` still injects the `## Model Aliases` system-prompt section with guidance to prefer aliases, and the alias list is still built from every configured `agents.defaults.models[*].alias` entry without filtering overlapping providers. The latest shipped release `v2026.4.23` still had the same behavior, so there is no strong evidence this was already fixed or is obsolete.

## Evidence

- **System prompt still tells the model to prefer aliases:** `buildAgentSystemPrompt()` still emits `## Model Aliases` followed by `Prefer aliases when specifying model overrides; full provider/model is also accepted.` whenever `modelAliasLines` is non-empty in non-minimal prompts.
