## Summary

The request is for ClawHub marketplace moderation and verification policy around skills using `always: true`, not for OpenClaw core. Current `main` explicitly assigns discovery, provenance, publisher status, and security review to ClawHub, while OpenClaw itself only treats `always: true` as a load-time gating flag and already exposes local install-review hooks via `before_install`.
