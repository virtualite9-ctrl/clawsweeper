## Summary

Keep #70805 open. Current main still contains the watch-mode path that skips runtime-postbuild sync when the build is otherwise clean, and the runtime-postbuild requirement logic still checks stamps/head/dirty/mtime rather than validating missing staged runtime outputs. The PR is a focused, recent contributor bug fix with concrete Telegram failure evidence and regression coverage, so it is not obsolete, implemented, or superseded by current main.

## Best Possible Solution

Keep the PR open for maintainer review. The useful path is to validate the watch-mode runtime-postbuild guardrails against current main, preserve the concurrency lock coverage already present on main, and either land this fix or ask the contributor to narrow/refine it if review finds issues.
