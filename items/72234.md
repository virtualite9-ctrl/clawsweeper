## Summary

Current main already handles the Bonjour/CIAO cancellation crash reported against v2026.4.24. The bundled Bonjour plugin wires scoped CIAO process-error handlers into the shared runtime registry, the classifier covers both ANNOUNCEMENT and PROBING cancellation messages, the global process handlers return before fatal exit when a registered handler handles the reason, and tests/docs cover the non-fatal and self-disable behavior.

## Best Possible Solution

Close this issue as implemented on current main. Keep the shipped main-line implementation: scoped Bonjour/CIAO process-error handling, shared runtime handler state, bounded advertiser recovery, and self-disable behavior for unhealthy mDNS environments. Release or backport timing should be handled through normal maintainer release flow rather than keeping this duplicate crash report open.
