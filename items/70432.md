## Summary

Keep open. Current `main` still contains the macOS installer behavior this PR fixes, and the checked-out repo does not show an equivalent fix already landed.

## Evidence

- **macOS install path still ignores `run_quiet_step` failure:** On current `main`, `install_node()` still invokes `run_quiet_step "Installing node@${NODE_DEFAULT_MAJOR}" brew install ...` as a bare statement, then continues to `brew link` and `ensure_macos_default_node_active` without checking the return value.
  - file: [scripts/install.sh:1401](https://github.com/openclaw/openclaw/blob/2cd2732ab6695a4b3220ce41166b74aca7097c5f/scripts/install.sh#L1401)
  - command: `nl -ba scripts/install.sh | sed -n '1397,1405p'`
  - sha: [2cd2732ab669](https://github.com/openclaw/openclaw/commit/2cd2732ab6695a4b3220ce41166b74aca7097c5f)
- **Current macOS remediation path still conflates missing install with PATH ordering:** `ensure_macos_default_node_active()` only uses `-x "${brew_node_prefix}/bin/node"` to decide whether to prepend PATH, but if the executable is missing it still falls through to `ui_error "Node.js v${NODE_DEFAULT_MAJOR} was installed..."` and PATH advice based only on a non-empty Homebrew prefix.
  - file: [scripts/install.sh:1312](https://github.com/openclaw/openclaw/blob/2cd2732ab6695a4b3220ce41166b74aca7097c5f/scripts/install.sh#L1312)

  - sha: [2cd2732ab669](https://github.com/openclaw/openclaw/commit/2cd2732ab6695a4b3220ce41166b74aca7097c5f)
- **Failure helper requires explicit caller handling:** `run_quiet_step()` prints an error and returns `1` on failure; it does not exit the script itself. That makes the unchecked macOS call site materially relevant to the reported bug.
  - file: [scripts/install.sh:440](https://github.com/openclaw/openclaw/blob/2cd2732ab6695a4b3220ce41166b74aca7097c5f/scripts/install.sh#L440)
