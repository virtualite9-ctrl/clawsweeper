## Summary

Keep this PR open. Current main already shipped part of the lifecycle fix: one-shot embedded runs can retire bundled MCP runtimes, isolated cron requests that cleanup, gateway shutdown disposes all session MCP runtimes, and idle TTL eviction is documented. But the PR is not fully obsolete because current main still does not request run-end bundled MCP cleanup for isolated heartbeat runs, and queued follow-up runs do not carry a cleanup flag. The PR should be rebased and narrowed to the remaining gaps rather than closed as implemented.

## Best Possible Solution

Keep this PR open as a useful implementation candidate, but rebase it onto current main and narrow it to the remaining uncovered behavior: isolated heartbeat should request bundled MCP cleanup at the end of its one-shot run, queued one-shot follow-up cleanup should be added if that path still creates fresh session runtimes, and regression tests should cover cleanup after fallback/continuation and session-id rotation under the current session-key retirement manager. The already-shipped cron, embedded-run, idle TTL, and gateway shutdown pieces should be preserved rather than duplicated.
