## Summary

Current main already avoids the reported Copilot Anthropic tool-schema rejection. The bundled GitHub Copilot provider routes Claude/Copilot `anthropic-messages` calls through OpenClaw's provider-aware Anthropic transport and the current tool conversion emits only Anthropic-supported `name`, `description`, and `input_schema` fields, with no `custom.eager_input_streaming` field. The provided issue context has no comments, no protected labels, and the related title-search reports are unrelated to this Copilot schema failure.

## Best Possible Solution

Close this as fixed in current main and advise affected users on 2026.4.22 to update to the shipped v2026.4.24 or newer Copilot provider path. Keep future work focused on regression coverage if Copilot changes its Anthropic schema again.
