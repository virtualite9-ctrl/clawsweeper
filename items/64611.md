## Summary

Current main still lacks the future-timestamp guard in `isTaskDue`, and heartbeat task mode still uses that helper to skip all work as `no-tasks-due` when no task is considered due. This PR remains a focused implementation candidate rather than something to close.

## Best Possible Solution

Keep this PR open and have a maintainer review or apply the narrow equivalent change: treat `lastRunMs > nowMs` as due in `src/auto-reply/heartbeat.ts`, keep focused `isTaskDue` regression coverage for future timestamps, refresh or squash the branch if needed, then run the targeted heartbeat tests and changed gate before landing.
