## Summary

Current `main` already supersedes this PR. `src/plugin-sdk/channel-entry-contract.ts` now formats bundled entry open failures with the actual `reason` plus underlying error detail, so missing build artifacts are no longer reported as a generic plugin-root escape. The same implementation is present in the shipped release `v2026.4.23`.

## Evidence

- **Current bundled-entry failure message is reason-specific:** `formatBundledEntryModuleOpenFailure()` builds errors as `bundled plugin entry "..." failed to open ... reason "<reason>": <errorDetail>`, which is more accurate than the old blanket `plugin entry path escapes plugin root` message discussed in this PR.
