## Summary

Keep #17215 open. Current main has adjacent update-safety pieces, but not the requested native pre/post-update compatibility scanner with `doctor --upgrade-scan`, risk-scored JSON/human report, smoke recommendations, or rollback hints. The related PR #54354 is still only a narrower plugin engine-compatibility proposal, not a shipped replacement for this issue.

## Best Possible Solution

Keep the issue open as a core update/doctor safety enhancement. The best path is a native `openclaw update` and/or `openclaw doctor --upgrade-scan` mode that produces both human and JSON reports for pre-update config/schema/deprecation checks, post-update health/plugin/runtime checks, smoke-test recommendations, risk classification, and rollback guidance, with docs and focused tests for the report contract. Treat PR #54354 as related plugin-specific compatibility work if it lands, not as a full superseding fix for this issue.
