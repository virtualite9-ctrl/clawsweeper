## Summary

Keep #17215 open. Current main still has adjacent update safety work (`openclaw update --dry-run`, post-update plugin sync, and post-update doctor repair), but it does not implement the requested native pre/post-update compatibility scanner with risk-scored JSON/human reports, smoke recommendations, rollback hints, or a `doctor --upgrade-scan`-style surface. The existing maintainer Codex review comment also explicitly kept the issue open, and the current source/docs/tests still support that conclusion.

## Best Possible Solution

Continue tracking this as a core update/doctor safety enhancement. The best product path is a native upgrade scan under `openclaw update` and/or `openclaw doctor` that emits human and JSON reports for pre-update config/schema/deprecation checks, post-update health/plugin/runtime checks, smoke-test recommendations, risk classification, and rollback guidance, with docs and focused tests for the report contract.
