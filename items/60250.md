## Summary

Current `main` already handles this narrow completed-run persistence bug: terminal lifecycle events are converted into terminal session statuses and persisted back into the session store via a merge-on-latest update path, and the same implementation is present in release `v2026.4.23`.

## Evidence

- **Terminal lifecycle snapshots now force terminal status:** `deriveGatewaySessionLifecycleSnapshot` converts lifecycle `end` and `error` events into terminal session states (`done`/`failed`/`killed`/`timeout`) and fills `endedAt` plus `runtimeMs`, which directly covers the reported `status: "running"` + terminal timing contradiction.
  - file: [src/gateway/session-lifecycle-state.ts:92](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/src/gateway/session-lifecycle-state.ts#L92)
  - command: `nl -ba src/gateway/session-lifecycle-state.ts | sed -n '92,129p'`
  - sha: [926068b14fc6](https://github.com/openclaw/openclaw/commit/926068b14fc661e915a0247e9ecb5949fb95b4f7)
- **Terminal lifecycle events are persisted through a patch path:** `persistGatewaySessionLifecycleEvent` handles terminal lifecycle events by calling `updateSessionStoreEntry(...)` with a derived lifecycle patch instead of re-saving a stale in-memory session object.
  - file: [src/gateway/session-lifecycle-state.ts:146](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/src/gateway/session-lifecycle-state.ts#L146)
  - command: `nl -ba src/gateway/session-lifecycle-state.ts | sed -n '146,168p'`
  - sha: [926068b14fc6](https://github.com/openclaw/openclaw/commit/926068b14fc661e915a0247e9ecb5949fb95b4f7)
- **Session-store updates merge against the latest stored row:** `updateSessionStoreEntry` reloads the session store under lock, resolves the current row, merges the patch with `mergeSessionEntry`, and persists that result. That avoids the stale write-order pattern described in the report.
  - file: [src/config/sessions/store.ts:644](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/src/config/sessions/store.ts#L644)
  - command: `nl -ba src/config/sessions/store.ts | sed -n '644,668p'`
  - sha: [926068b14fc6](https://github.com/openclaw/openclaw/commit/926068b14fc661e915a0247e9ecb5949fb95b4f7)
- **Gateway terminal event handler is wired to persist terminal lifecycle state:** `finalizeLifecycleEvent` calls `persistGatewaySessionLifecycleEvent({ sessionKey, event: evt })` for `end`/`error` lifecycle events, so the terminal status patch runs on normal chat completion.
  - file: [src/gateway/server-chat.ts:575](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/src/gateway/server-chat.ts#L575)

  - file: [src/gateway/server-chat.agent-events.test.ts:957](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/src/gateway/server-chat.agent-events.test.ts#L957)
