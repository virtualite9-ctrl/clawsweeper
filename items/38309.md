## Summary

Keep #38309 open. Current main still only accepts plain string values for `sandbox.docker.env`; SecretRef objects remain invalid at the Zod schema, public config type, generated schema, and runtime collection layers. The related PR #38310 is closed unmerged and remains useful implementation context, but it does not supersede the issue.

## Best Possible Solution

Implement SecretRef support end-to-end for Docker sandbox env values: accept `SecretInputSchema` in `SandboxDockerSchema`, type `SandboxDockerSettings.env` as `Record<string, SecretInput>` while keeping resolved `SandboxDockerConfig.env` string-only, add defaults and per-agent Docker env collectors plus target-registry entries for audit/plan/configure, update generated schema/docs, and add focused schema/runtime/Docker-env regression tests. PR #38310 remains useful design context but should not close the issue unless equivalent changes land on main.
