## Summary

Keep #38309 open. Current main still does not support SecretRef values in `sandbox.docker.env`: schema/type surfaces remain string-only, the generated schema still rejects object values, the secrets runtime and target registry do not collect Docker env entries, and the Docker launch path consumes already-plain env values. PR #38310 is useful prior implementation context, but it is closed unmerged, so it does not supersede this issue.

## Best Possible Solution

Implement SecretRef support end-to-end for Docker sandbox env values: accept `SecretInput` in `SandboxDockerSchema` and `SandboxDockerSettings.env`, keep resolved `SandboxDockerConfig.env` as `Record<string,string>`, collect and resolve defaults plus per-agent `sandbox.docker.env` through the secrets runtime, add target-registry coverage for audit/plan/configure, update generated schema and public docs, and add focused schema/runtime/Docker arg regression tests. PR #38310 remains useful design context but is not canonical because it closed unmerged.
