## Summary

Keep #44167 open. Current main still lacks the PR's core behavior: memory-core retries only service/rate-limit embedding errors, and batch embedding calls do not retry transient transport failures or split a failed multi-item batch after retry exhaustion. The PR is a focused, still-relevant fix for open issue #44166; adjacent issue #71784 covers a related live single-query path and does not supersede this batch-indexing PR.

## Best Possible Solution

Keep the PR open for maintainer review, or land an equivalent scoped fix that adds a narrow transport-error classifier, reuses the existing embedding backoff loop, recursively splits exhausted multi-item text and structured batches while preserving order, and keeps focused regression tests in `extensions/memory-core/src/memory/manager-embedding-policy.test.ts`. Track live single-embed transport reliability separately under #71784.
