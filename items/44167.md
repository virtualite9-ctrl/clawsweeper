## Summary

Keep #44167 open. Current main still lacks the PR's central memory-core behavior: batch embedding retry classification only covers service/rate-limit failures, and the text/structured batch paths do not retry transient transport failures or recursively split failed multi-item batches after retry exhaustion. The PR remains a focused fix for the still-open linked bug #44166; related #71784 is adjacent live single-query reliability work and does not supersede this batch-indexing change.

## Best Possible Solution

Keep the PR open for maintainer review, or land an equivalent scoped fix in `extensions/memory-core` that adds a narrow transport-error classifier, reuses the existing embedding backoff loop, recursively splits exhausted multi-item text and structured batches while preserving result order, and keeps focused regression tests in `extensions/memory-core/src/memory/manager-embedding-policy.test.ts`. Track live single-query transport reliability separately under #71784.
