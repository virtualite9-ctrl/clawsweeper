## Summary

Keep PR #71885 open. Current main still binds `TaskRecord.task` directly into the SQLite `task_runs.task` parameter, and no current source/test/changelog evidence shows the proposed structured-task serialization has landed or been superseded. The PR is a focused storage-boundary fix with a regression test, but maintainers should still decide whether structured `task` values are an accepted runtime contract because public task types currently declare `task` as `string`.

## Best Possible Solution

Keep #71885 open for maintainer review. The best path is to either merge the narrow SQLite-boundary serialization and regression test, or ask the contributor for a concrete runtime/plugin path proving how a structured task value reaches `TaskRecord.task` and then decide whether to harden storage or enforce the string contract earlier. It should not be auto-closed because current main still lacks the normalization and no canonical superseding item was found.
