## Summary

Keep open. Current main has not implemented the PR's SQLite serialization change, and the reported failure mode is plausible: binding a non-string object through node:sqlite throws before persistence.

## Best Possible Solution

Do not auto-close PR #71885. Review it as a plausible focused bug fix: either merge the storage serializer with its regression test, or ask the contributor to move the normalization to the task creation/runtime boundary and clarify whether structured task payloads are intentionally accepted.
