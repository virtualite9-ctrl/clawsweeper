## Summary

Keep open. Current `main` already strips raw `<tool_call>/<tool_result>` blocks when assistant text goes through the shared assistant-visible sanitizer, but the Discord streaming/direct-delivery path still only removes reasoning/directive tags from `onPartialReply` and preview/final payload text. That means the leak this PR targets is still plausible on current `main`, especially for raw reply payloads that bypass normal assistant-message extraction.
