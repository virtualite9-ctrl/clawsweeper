## Summary

Keep #34129 open. Current main still preserves a blank, non-inferable tool-call name and still assigns the reported `call_auto_*` fallback id, while the regression test explicitly expects `name === ""` and `id === "call_auto_1"`. The related sentinel fix is still represented by open PR #56391 rather than shipped on main.

## Best Possible Solution

Keep #34129 open until PR #56391 or an equivalent maintainer-approved fix lands. The fix should prevent blank or whitespace-only model tool names without an inferable id from entering dispatch as raw blank names with `call_auto_*` ids, and should update regression coverage so this exact blank-name loop cannot return. Treat the `parallel_tool_calls` 400 noted in the discussion as a related but separate compatibility class already tracked under #37048/#37261.
