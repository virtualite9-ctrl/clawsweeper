## Summary

Keep #34129 open. Current main still preserves the reported blank tool-call name when no tool name can be inferred, then assigns a `call_auto_*` fallback id. The related sentinel fix is still represented by open PR #56391 rather than shipped on main.

## Best Possible Solution

Keep #34129 open until PR #56391 or an equivalent maintainer-approved fix lands. The fix should prevent blank or whitespace-only model tool names without an inferable id from entering dispatch as raw blank names with `call_auto_*` ids, and should update regression coverage so the blank-name loop cannot return.
