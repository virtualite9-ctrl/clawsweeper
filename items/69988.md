## Summary

Keep #69988 open. Current main still does not implement per-agent compaction overrides: `agents.list[]` has no `compaction` type or strict schema field, resolved agent config does not carry compaction, and embedded Pi compaction/runtime paths still read only `agents.defaults.compaction`. The PR is a plausible implementation for the still-open feature request #13736, and the discussion shows the author addressed review feedback on the PR branch rather than abandoning it.

## Best Possible Solution

Keep #69988 open or consolidate it with the preferred canonical per-agent compaction implementation. The product path should add `agents.list[].compaction` to the config type, strict Zod schema, generated schema/help/labels, agent-scope resolver, and embedded Pi compaction call sites, with tests for default inheritance, per-agent override precedence, nested `qualityGuard`/`memoryFlush` merging, defaults fallback when an agent id misses, and explicit/session-derived agent id resolution for `/compact` and preflight compaction.
