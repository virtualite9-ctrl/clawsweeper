## Summary

Keep #69988 open. Current main still does not implement per-agent `agents.list[].compaction`: the agent config type and strict Zod schema omit the field, resolved agent config does not carry it, and Pi compaction/runtime setup still reads only `agents.defaults.compaction`. The PR is a focused implementation of a still-real feature request and has useful review discussion/fixes, so it should not be cleanup-closed.

## Best Possible Solution

Keep the PR open and use it, or a maintainer-owned successor, to add `agents.list[].compaction` to the config type/schema, merge per-agent compaction over defaults in `resolveAgentConfig`, thread the effective compaction config through Pi settings, safeguard extension setup, direct/manual compaction, and add regression tests for inherited defaults, agent overrides, missing-agent fallback, and nested `qualityGuard`/`memoryFlush` merge behavior. Link or coordinate it with the open canonical feature issue #13736 rather than cleanup-closing this implementation.
