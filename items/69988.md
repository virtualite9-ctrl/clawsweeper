## Summary

Keep #69988 open. Current main at 4a3030d still exposes compaction only under `agents.defaults`: per-agent config types/schema do not accept `agents.list[].compaction`, resolved agent config does not return or merge compaction, and Pi compaction runtime setup still reads defaults-only config. The PR discussion shows useful recent fixes rather than obsolete work, and related #13736 is an open feature request, not a shipped or superseding implementation.

## Best Possible Solution

Keep the PR open for maintainer review or rework. The product path is to add `agents.list[].compaction` to the config type/schema/generated help, resolve an effective per-agent compaction config with defaults fallback and nested-object merge behavior, then thread the resolved agent id through Pi settings, safeguard extension setup, `/compact`, preflight/auto-compaction, timeout/model/memory-flush paths, docs, and focused regression tests.
