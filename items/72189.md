## Summary

Keep #72189 open. Current main still has the Google Live voice-call tool-response gaps the PR targets: Google tool responses omit the function name, there is no NON_BLOCKING consult declaration, the realtime voice bridge contract has no continuation option, and the voice-call handler only sends one final tool result after awaiting the consult. The PR is recent, focused, and has an author live-verification follow-up, but it should address the Greptile P1/P2 review before maintainers consider landing it.

## Best Possible Solution

Keep the PR open and request a narrow revision: include the Google Live function name in tool responses, preserve call-id-to-name mapping, mark only openclaw_agent_consult NON_BLOCKING, add/forward continuation options through the realtime voice SDK/session bridge, send the provisional working response only for that consult tool, schedule the final consult result appropriately, preserve single-result behavior for OpenAI and other providers/tools, and keep the regression tests/docs. After the Greptile P1/P2 is fixed, maintainers should ask for targeted tests plus a changed-gate or scoped validation note, and preferably attach redacted live Gemini evidence if available.
