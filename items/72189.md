## Summary

Keep this PR open. Current main includes the narrower #72426 Google Live function-response name fix, but it still lacks the non-blocking consult continuation behavior this PR now targets. The current PR head also appears to address the earlier Greptile scoping concern by limiting `willContinue` to `openclaw_agent_consult`, so it remains a useful implementation candidate rather than something to close as obsolete.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land or revise the continuation implementation: preserve the #72426 name fix, add an additive realtime voice continuation option, mark only `openclaw_agent_consult` as Google `NON_BLOCKING`, send a provisional working result only for that consult tool in Voice Call and Google Meet, schedule final Google Live consult results WHEN_IDLE, retain pending call-name state until the final result, and keep focused regression tests/docs. If maintainers want a narrower change, split the continuation behavior from the provider-contract alignment rather than closing the PR.
