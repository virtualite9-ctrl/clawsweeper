## Summary

Keep #72189 open. Current main still lacks the PR's central Google Live voice-call fixes: function responses are sent without the required `name`, the consult tool is not declared `NON_BLOCKING`, the realtime bridge contract has no continuation option, and Voice Call waits for tool completion before submitting a single result. The PR is recent, open, contributor-authored, and contains useful unmerged work, but it also has unresolved review comments about scoping continuation responses to `openclaw_agent_consult`, so it should not be closed as stale or already implemented.

## Best Possible Solution

Keep the PR open for author or maintainer follow-up. The best path is to preserve the unmerged fixes for Google Live function-response naming, consult `NON_BLOCKING` declaration, continuation-capable bridge options, final `WHEN_IDLE` scheduling, object/primitive response shaping, and `onError` handling, while addressing the review by allowing provisional `willContinue` responses only for `openclaw_agent_consult` and covering that guard in tests.
