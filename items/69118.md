## Summary

Keep #69118 open. Current main contains partial fixes for adjacent CLI-session drift, but the reported group-intro path is still present: group intro is included in the static extra-system-prompt hash on first group turns, then omitted on later turns, and `resolveCliSessionReuse` still invalidates on any stored/current `extraSystemPromptHash` mismatch. The newer Claude live-session path also includes that hash in its process fingerprint, so the same drift can restart the live Claude process rather than cleanly reusing it.

## Best Possible Solution

Keep the issue open and fix the remaining group-channel hash drift directly. The likely maintainer path is to exclude turn-variable `groupIntro` from `extraSystemPromptStatic` or stop using `extraSystemPromptHash` as a CLI-session reuse key, then add a regression test for two consecutive group-channel turns proving no `system-prompt` invalidation and no Claude live-session restart. The adjacent #64386/#68471 fixes should remain, but they do not fully cover this issue.
