## Summary

Keep #69118 open. Current main has a partial fix that excludes inbound metadata from the CLI session prompt hash, but it still includes `groupIntro` in `extraSystemPromptStatic`; since `groupIntro` is intentionally present on the first group turn and absent on steady turns, the reported second-turn `extraSystemPromptHash` drift remains plausible in current code.

## Best Possible Solution

Keep the issue open and fix the remaining hash boundary: exclude `groupIntro` from the CLI session reuse hash, or split the runtime prompt from a genuinely stable resume-identity prompt. Add a regression test that simulates first and second group-channel turns with the same CLI binding and asserts `resolveCliSessionReuse` returns the existing session instead of `invalidatedReason: "system-prompt"`. Preserve the existing inbound-metadata static-hash fix and MCP resume-hash behavior.
