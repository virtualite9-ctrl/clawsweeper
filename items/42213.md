## Summary

Keep open. The reported Control UI queue-stall still exists on the checked-out main: a non-null chatRunId keeps chat busy, normal sends are queued while busy, queue flushing returns while busy, and the UI only flushes after terminal chat events. Live GitHub API also shows the same-author implementation candidate PR #42258 is still open and unmerged, so this issue should remain open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review #42258 as the implementation candidate. Land that PR or an equivalent Control UI recovery that probes agent.wait for the active chatRunId after inactivity, clears busy state only on a terminal wait result, and adds regression coverage for stale-finished recovery, still-active timeout behavior, and queued-message flushing.
