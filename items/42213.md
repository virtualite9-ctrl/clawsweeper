## Summary

Keep open. There is an open, unmerged same-author PR (#42258) that is an implementation candidate for this issue, and current main still has the reported failure shape: queued sends are blocked by `chatRunId`, queue flushing is tied to terminal chat events, and the Control UI has no `agent.wait` watchdog/recovery path.

## Best Possible Solution

Keep this issue open and review #42258 as the implementation candidate. If accepted, land that PR or an equivalent Control UI `agent.wait` recovery with regression coverage for stale finished runs, still-active timeout behavior, and queue flushing. Close this issue only after the fix is merged or otherwise shipped on main.
