## Summary

Keep this issue open. Current main still has the stale-busy failure shape: `chatRunId` makes the Control UI chat busy, normal sends queue while busy, queue flushing returns while busy, and the clear/flush path still depends on terminal `chat` events. The same-author implementation candidate #42258 is still open and unmerged in the supplied GitHub context, with closing syntax for this issue, so this issue should remain open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review #42258 as the implementation candidate. Land that PR or an equivalent Control UI recovery that probes `agent.wait` for the active `chatRunId` after inactivity, clears busy state only on a terminal wait result, and adds regression coverage for stale-finished recovery, still-active timeout behavior, and queued-message flushing.
