## Summary

Keep this issue open. Current main still uses `chatRunId` as the Control UI busy latch, queues normal follow-up messages while busy, and only clears/drains that queue from chat terminal-event handling or reconnect reset paths. I did not find an equivalent Control UI `agent.wait` watchdog/recovery path on main, while the provided GitHub context shows same-author PR #42258 is open with closing syntax for this issue.

## Best Possible Solution

Keep this issue open and review #42258 as the implementation candidate. Land that PR or an equivalent Control UI recovery that probes `agent.wait` for the active `chatRunId` after inactivity, clears busy state only on terminal wait results, and adds regression coverage for stale-finished recovery, still-active timeout behavior, watchdog reset/cancellation, and queued-message flushing.
