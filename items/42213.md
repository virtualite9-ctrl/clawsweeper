## Summary

Keep this issue open. Current main still relies on `chatRunId` plus terminal chat events to leave the busy state and flush queued messages, and the same-author implementation candidate #42258 is still open and unmerged with closing syntax for this issue.

## Best Possible Solution

Keep this issue open and review #42258 as the implementation candidate. Land that PR or an equivalent Control UI recovery that probes `agent.wait` for the active `chatRunId` after inactivity, clears busy state only on a terminal wait result, and adds regression coverage for stale-finished recovery, still-active timeout behavior, and queued-message flushing.
