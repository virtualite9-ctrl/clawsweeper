## Summary

Issue #70266 should stay open. Current main still renders the macOS Talk Mode overlay as the built-in animated orb and does not pass the configured assistant/avatar identity into that native overlay. The related #22199 work covers Control UI/chat identity, but this issue is narrower and remains distinct for the macOS Talk Mode voice overlay.

## Best Possible Solution

Implement the narrow macOS surface rather than closing this item: extend the macOS Talk overlay state/config bridge to resolve the active assistant avatar from the existing identity/avatar resolver or an equivalent native-safe endpoint, render it as a circular cropped image inside the current animated voice-state rings, and keep the existing `TalkOrbView` as the fallback when no avatar is configured or resolvable. Add focused macOS coverage for avatar-present and avatar-missing overlay behavior if the current SwiftUI smoke-test harness can support it.
