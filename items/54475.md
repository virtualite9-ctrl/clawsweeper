## Summary

Keep this PR open. Current main still treats a pre-hello WebSocket close as fatal in the one-shot gateway call wrapper and still reports the client-side connect error, so the focused fix is not implemented or obsolete. The PR is narrow, has a current remote head, includes tests, and its bot-review concern was addressed in the provided discussion by switching from string matching to a structured close error.

## Best Possible Solution

Review this PR as a focused gateway bug-fix candidate, or consolidate it with the broader open cron/gateway retry work if maintainers prefer one shared fix. The implementation should land in the shared `src/gateway/call.ts` and `src/gateway/client.ts` paths with the structured close-error approach shown in the PR, plus the focused regression tests for the one-shot call wrapper and reconnect logging behavior.
