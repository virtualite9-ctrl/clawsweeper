## Summary

Current main already fixes the heartbeat `setTimeout` overflow failure through the merged scheduler-layer clamp from PR #71478, so PR #61638's config-schema rejection approach is superseded. The shipped implementation prevents oversized `heartbeat.every` values from becoming a 1 ms heartbeat loop, includes regression coverage, and shipped in v2026.4.24.

## Best Possible Solution

Close PR #61638 as implemented on main and keep the shipped scheduler-layer fix from PR #71478. If maintainers later want explicit config warnings or validation for very large heartbeat durations, that should be tracked as a separate product decision rather than merging this now-obsolete schema patch.
