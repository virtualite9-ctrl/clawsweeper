## Summary

Current main already addresses the reported Docker startup loop: the Docker setup flow now performs onboarding and config writes through `openclaw-gateway` before startup, seeds `gateway.controlUi.allowedOrigins` for non-loopback binds, and documents/tests the shared-network namespace behavior. The changelog records this Docker setup fix in release `2026.3.24`.

## Evidence
