## Summary

Keep open. Current main has already shipped part of the cron `delivery.threadId` plumbing, but this PR is not fully obsolete: the public gateway cron schema still does not accept or round-trip `delivery.threadId`, and current failure-destination deduplication still ignores an explicit primary `threadId`, matching the review feedback the author later addressed on the PR branch.

## Best Possible Solution

Keep this PR open for maintainer review or ask the author to rebase and narrow it to the remaining gaps on current main: add `delivery.threadId` to the cron gateway schema/patch contract with round-trip tests, update failure-destination deduplication so explicit topic-routed primary deliveries do not suppress a same-chat failure route, and retain the existing main-branch internal delivery-plan/target plumbing rather than reintroducing older duplicate code.
