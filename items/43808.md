## Summary

Keep open. Current main has absorbed much of the internal cron `delivery.threadId` plumbing, but the PR is not obsolete: the public gateway cron schema still omits `delivery.threadId`, and failure-destination deduplication still compares only mode/channel/to/accountId. The PR author's later branch update directly addressed those two remaining gaps, and the related maintainer PR explicitly noted this forum-topic PR family should not be closed until the landed fix path is explicit.

## Best Possible Solution

Keep this PR open for maintainer review or a focused rebase. Preserve the internal threadId plumbing already on main, then port the remaining pieces: add `threadId` to the gateway cron delivery and patch schemas with round-trip coverage, update failure-destination deduplication so explicit topic-routed primary delivery is not treated as identical to an unthreaded same-chat failure route, and sync any affected public protocol/client artifacts.
