## Summary

Current main has absorbed the internal cron delivery.threadId plumbing, but this PR is not obsolete. The public gateway cron schema still rejects delivery.threadId, and failure-destination deduplication still ignores explicit primary threadId, matching the PR author's later fixes and prior bot review feedback.

## Best Possible Solution

Keep this PR open for maintainer review or a focused rebase. Preserve the internal threadId plumbing already on main, then port the remaining public gateway schema round-trip and failure-destination deduplication fixes with targeted tests. Because the gateway schema is a public protocol surface, sync any generated/client artifacts required by the protocol boundary before landing.
