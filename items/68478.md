## Summary

Keep open. Current main still has only partial noise controls: exec lifecycle events can be suppressed with `tools.exec.notifyOnExit`, cron delivery can be controlled per job, and some task paths have internal notify policies. It does not provide the broader requested default-silent or configurable suppression policy for auto-generated chat confirmations across cron, skill/tool command dispatch, and generic “done” replies. The prior ClawSweeper review kept this same item open, and the current main recheck still matches that conclusion.

## Best Possible Solution

Keep this issue open and design a coherent notification/confirmation policy, likely under `messages` or `commands`, that can suppress or group routine action confirmations across slash-command skill/tool dispatch, cron/system-event delivery, install/update acknowledgements, and generic done replies. Preserve explicit alerts and security/approval failures, reuse existing `tools.exec.notifyOnExit`, heartbeat visibility, and cron `delivery.mode` semantics where they already fit, and add targeted Telegram/Discord plus command-dispatch regression coverage.
