## Summary

Keep #65824 open. Current main has real partial progress, including fallback lifecycle events, tool/file-write hooks, and last-known-good config recovery, but the umbrella still contains active, user-validated core reliability/security/product requests that are not all implemented or fully superseded by a canonical follow-up.

## Best Possible Solution

The best path is to split remaining work by surface: fallback visibility/context re-injection, doctor preview mode, per-path filesystem policy, subagent repeated-failure circuit breaker, prompt-injection content scanning, spend budget gates, and scheduled model health routing; then close only fully implemented slices against their shipped commits or canonical follow-ups.
