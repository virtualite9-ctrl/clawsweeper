## Summary

Keep this issue open. Current main has meaningful partial progress, including plugin hooks, fallback lifecycle events, manual model probes, and last-known-good config recovery, but the umbrella still contains active core reliability/security/product requests that are neither fully implemented nor fully superseded by focused follow-ups.

## Best Possible Solution

Keep this issue open for maintainer triage or replace it with focused canonical issues before closure. The practical path is to split remaining work by surface: fallback visibility and context re-injection, doctor dry-run/diff preview, per-path filesystem policy, repeated subagent failure circuit breaking, prompt-injection content scanning policy, spend budget gates, and scheduled model health routing. Close only the slices that are already shipped or fully covered by focused follow-ups.
