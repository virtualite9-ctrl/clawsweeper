## Summary

Current main already implements this well enough: Discord IDs are normalized to strings at validation time, and the shipped Control UI schema for `channels.discord.allowFrom` is string-only rather than a `string | number` union. The same implementation is present in release `v2026.4.22`.

## Evidence

- **Discord IDs are normalized to strings:** `DiscordIdSchema` accepts `string | number`, converts safe numeric IDs to `String(value)`, and then pipes to `z.string()`, so numeric-looking Discord IDs are stored/validated as strings instead of requiring quoted JSON literals.
  - file: [src/config/zod-schema.providers-core.ts:43](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/config/zod-schema.providers-core.ts#L43)
  - command: `nl -ba src/config/zod-schema.providers-core.ts | sed -n '40,65p'`
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **Control UI metadata is string-only for Discord allowFrom:** Bundled channel config metadata exposes both `allowFrom` and `dm.allowFrom` as arrays of `type: "string"`, which removes the mixed primitive union that would have pushed the Web UI down the generic `string | number` path.
  - file: [src/config/bundled-channel-config-metadata.generated.ts:1063](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/config/bundled-channel-config-metadata.generated.ts#L1063)

- **Latest release already contains the same behavior:** The `v2026.4.22` tag shows the same string-only Discord `allowFrom` metadata and the same numeric-to-string `DiscordIdSchema`, so this was already shipped before current `main`.
  - file: [src/config/bundled-channel-config-metadata.generated.ts:1066](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/src/config/bundled-channel-config-metadata.generated.ts#L1066)

## Close Comment

Closing from a Codex review because current `main` already has this fixed. `src/config/zod-schema.providers-core.ts:43` now normalizes numeric Discord IDs to strings before validation, and the shipped Control UI schema at `src/config/bundled-channel-config-metadata.generated.ts:1063` exposes `channels.discord.allowFrom` / `channels.discord.dm.allowFrom` as string arrays instead of a mixed `string | number` field.
