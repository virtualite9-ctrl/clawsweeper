## Summary

Keep #70819 open. Current main still has the plugin install scanner behavior the PR is trying to fix: any symlink under a node_modules path whose real target escapes the install root is rejected, with no trusted host OpenClaw peer-link exception. The existing post-install peer symlink helper does not cover package-manager-created source-tree symlinks that are present before the source scan runs. The related symlink-discovery PRs are not canonical replacements for this install-security-scan issue.

## Best Possible Solution

Keep #70819 open as a useful focused bug-fix candidate. The best path is to land a narrowly reviewed scanner change in `src/plugins/install-security-scan.runtime.ts` that allows only exact `node_modules/openclaw` and `node_modules/.bin/openclaw` links when their real target is inside the trusted host OpenClaw package root, while keeping arbitrary, nested, spoofed, or vendored escaping node_modules symlinks blocked and covered by regression tests in `src/plugins/install.test.ts`.
