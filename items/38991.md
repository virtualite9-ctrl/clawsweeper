## Summary

Keep #38991 open. Current main still does not backfill missing legacy cron job timestamps before returning `cron.list`; it only backfills `updatedAtMs` in some store-load paths, leaves missing `createdAtMs` untouched, and the gateway response still exposes jobs from the in-memory store while the protocol and macOS model require both timestamp fields.

## Best Possible Solution

Keep #38991 open or replace it with an equivalent scoped patch: backfill finite non-negative `createdAtMs` and `updatedAtMs` during cron store/service load, persist only when loaded storage was actually mutated, and extend regression coverage so both `cron.list` and the persisted store expose numeric timestamps for legacy jobs. Drop the unrelated baseline/Fastfile changes before merge.
