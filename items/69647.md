## Summary

Keep PR #69647 open. Current main still contains the stale gateway URL behavior the PR targets: `loadSettings()` can fall back to the shared legacy settings key and preserve a sibling reverse-proxy path as a user override. The PR is not implemented on main, not superseded by a merged local-history fix, and the review discussion remains relevant rather than obsolete.

## Best Possible Solution

Keep the PR open and refine it toward a minimal storage-layer fix in `ui/src/ui/storage.ts` with regression tests in `ui/src/ui/storage.node.test.ts`: distinguish auto-derived page defaults from explicit user gateway overrides, update the page-derived scoped settings record when saving, and preserve valid same-host custom gateway paths.

## Evidence
