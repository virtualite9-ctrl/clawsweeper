## Summary

Keep open: current `main` still preserves a stale gateway URL from the shared legacy Control UI settings record when a different reverse-proxy base path has no page-scoped entry, so the reported multi-instance `/a` -> `/b` bug is not implemented on `main`. The PR discussion also shows substantive follow-up commits addressing the main review concerns rather than an abandoned or obsolete change.

## Evidence

- **Current load path still falls back to shared legacy settings and preserves sibling-path URLs:** `loadSettings()` reads the page-scoped key, then `openclaw.control.settings.v1:default`, then the shared `LEGACY_SETTINGS_KEY`. If `/b` has no scoped record but legacy storage still contains `/a`, `parsedGatewayUrl !== pageDerivedUrl` and the stale `/a` URL is preserved instead of switching to the current page default.
