## Summary

Current main still has the stale Control UI gateway URL path described by #69647: a path-prefixed instance can fall back to the shared legacy localStorage settings and reuse another instance's persisted gatewayUrl. The PR is focused on a real remaining bug and should stay open for maintainer review rather than be closed as implemented, duplicate, or stale.

## Best Possible Solution

Keep #69647 open. The best path is a focused Control UI storage fix in `ui/src/ui/storage.ts` plus regression coverage in `ui/src/ui/storage.node.test.ts` for sibling path prefixes (`/a` and `/b`), same-host custom WebSocket paths, and legacy unflagged settings migration. Related #41545 is a separate gateway-token editing bug, not a canonical duplicate for this stale reverse-proxy fallback.
