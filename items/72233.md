## Summary

Current main fixes the Windows/CommonJS bundled-channel load failure by avoiding native Jiti/CommonJS loading for Windows built plugin artifacts, instead routing those loads through the transpiled Jiti path. The pinned dependency metadata still has import-only exports, so the right fix is the loader behavior now present on main rather than patching third-party package exports in OpenClaw.

## Best Possible Solution

Close as implemented on current main. Keep the loader-side Windows fix and ship it in the next release; do not patch third-party `package.json` exports in OpenClaw unless a future dependency update still leaves a concrete loader gap.
