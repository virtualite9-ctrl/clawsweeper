## Summary

Keep open. Current main still only partially addresses the report: `payload.model` and `payload.timeoutSeconds` are accepted, but `payload.contextTokens` and `maxSpendUsd` remain absent from the cron tool surface, gateway protocol schema, stored cron payload type, normalization path, and isolated cron runtime behavior.

## Best Possible Solution

Keep this issue open. Preserve the already-shipped `model` and `timeoutSeconds` support, then add a deliberate `contextTokens` cron agentTurn contract through the cron tool schema/flat recovery, gateway protocol schema, exported cron types, normalizer, service patch merge behavior, isolated runtime selection, docs, and regression tests. For `maxSpendUsd`, either implement real per-run spend-cap enforcement with tests or split that part into a focused product/API issue.
