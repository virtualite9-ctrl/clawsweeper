## Summary

Keep open. Current main still only partially addresses the report: `payload.model` and `payload.timeoutSeconds` are present, but `payload.contextTokens` and `maxSpendUsd` are still absent from the cron tool description, gateway protocol schema, stored cron payload type, and isolated cron execution path. The prior issue discussion correctly identified this as more than a historical schema-only complaint, and the remaining anti-cost/context-cap fields still need product/code work.

## Best Possible Solution

Keep this issue open. Preserve the already-shipped `model` and `timeoutSeconds` behavior, then deliberately extend the cron `agentTurn` contract for `contextTokens` across the tool schema/description, flat-params recovery, normalization, gateway protocol schema, exported cron types, service patch merge, isolated cron execution, and regression tests. For `maxSpendUsd`, first decide the intended semantics: either add a real per-run spend cap with enforcement and tests, or split that part into a focused product/API issue if spend limiting is not currently part of cron.
