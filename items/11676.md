## Summary

Keep #11676 open. Current main still ships OpenClaw as a Node-hosted TypeScript application with documented Node and desktop/server OS requirements. The existing agent harness/plugin runtime seams support alternative executors inside an OpenClaw-prepared, Node-hosted turn, but they do not provide the requested runtime-agnostic agent core/protocol for C/Rust/WASM, bare-metal, or no-Node embedded ports. The linked MimiClaw project remains useful prior art rather than an implementation in this repo.

## Best Possible Solution

Keep the issue open until maintainers make an explicit runtime-portability decision. A strong path would define a stable runtime-agnostic agent/core protocol with schemas for tool calls, memory/session state, streaming/events, auth/model selection, and platform adapters, then add conformance tests or a small reference client proving the protocol can run outside the Node-hosted OpenClaw runtime. MimiClaw should stay linked as useful prior art.
