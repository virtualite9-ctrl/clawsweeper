## Summary

Keep #11676 open. Current main still treats OpenClaw as a Node-hosted TypeScript/Gateway runtime, and the existing plugin/agent-harness surfaces are in-process TypeScript extension points rather than a stable runtime-agnostic agent-core protocol for bare-metal, C/Rust/WASM, or ESP32-class ports. The linked MimiClaw project is relevant external prior art, but it is not an OpenClaw core implementation or a sufficient ClawHub/plugin close because the requested decoupling needs core product/API design.

## Best Possible Solution

Keep #11676 open for explicit maintainer product judgment. A strong path would define a stable runtime-agnostic agent/core protocol with schemas for tool calls, memory/session state, streaming/events, auth/model selection, platform adapters, and conformance tests or a small reference implementation proving the protocol can run outside the Node-hosted OpenClaw runtime. MimiClaw should remain linked as useful external prior art; #45496 should not be treated as the canonical tracker because it was narrower browser/WASM work and is closed.
