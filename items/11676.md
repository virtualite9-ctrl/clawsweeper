## Summary

Keep #11676 open. Current main still ships OpenClaw as a Node-hosted TypeScript CLI/gateway with Node and conventional OS install requirements. Existing plugin and agent-harness surfaces are useful, but they are experimental in-process TypeScript/Node extension points, not the requested runtime-agnostic agent core/protocol for C/Rust/WASM, bare-metal, or microcontroller ports. The owner-posted Codex review in the issue discussion already reached the same keep-open conclusion, and the related closed #45496 is narrower browser/WASM work, not a superseding embedded/no-Node plan.

## Best Possible Solution

Keep the issue open for explicit maintainer product judgment. A strong path would define a stable runtime-agnostic agent/core protocol with schemas for tool calls, memory/session state, streaming/events, auth/model selection, and platform adapters, then add conformance tests or a small reference client proving the protocol can run outside the Node-hosted OpenClaw runtime. MimiClaw should remain linked as useful external prior art.
