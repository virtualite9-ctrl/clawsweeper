## Summary

Keep open. Current main still derives the macOS app Launch at login toggle from live launchd status, has no separate persisted user intent, and does not reconcile the desktop `ai.openclaw.mac` LaunchAgent after Sparkle updates. The related shipped fix #40213 only removed `KeepAlive` to avoid relaunch after manual quit; it does not cover update-time persistence or UI/launchd drift.

## Best Possible Solution

Keep #39588 open and fix the desktop macOS Launch at login contract directly: persist the user's intended launch-at-login preference separately from live launchd loaded status, distinguish disabled from stale/unloaded/replaced/repair-needed LaunchAgent states, reconcile or re-bootstrap `ai.openclaw.mac` on app startup and after Sparkle updates, and add focused tests for stale/unloaded plist state plus UI reconciliation.
