## Summary

Keep open. Current main still does not automatically install Homebrew from the skill onboarding path: it only offers to show the install command, then calls the brew-backed skill installer, which returns a `brew not installed` failure when no brew executable is available. The top-level macOS shell installer can install Homebrew, but that is a separate install flow and does not resolve the reported skill-onboarding behavior. Related #14593 and #57555 overlap the dependency-install UX, but neither is a merged fix or canonical replacement for this macOS Homebrew prompt request.

## Best Possible Solution

Keep this issue open and fix the skill dependency onboarding path directly. A conservative implementation would add an explicit macOS-only confirmation path in `src/commands/onboard-skills.ts` that installs Homebrew before running brew-backed `skills.install`, re-detects `brew`, reuses the admin/failure-handling expectations from `scripts/install.sh`, and adds focused tests/docs. If maintainers do not want onboarding to install Homebrew, change the prompt and public docs so users are not led to expect automatic installation, while coordinating adjacent dependency UX with #14593 and #57555.
