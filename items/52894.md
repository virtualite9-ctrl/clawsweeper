## Summary

Keep this issue open. Current main does auto-install Homebrew in the top-level macOS `install.sh` path, but the issue text is specifically about skill dependency onboarding: that path still detects missing `brew`, asks whether to show the Homebrew install command, and then continues into `skills.install`, where brew-backed installs fail if no brew executable is present. The related reports in the provided context (#14593 and #57555) are also still open around the broader skill/Homebrew dependency behavior, so this is not safely closeable as implemented or fully superseded.

## Best Possible Solution

Keep this issue open and address the skill dependency onboarding path directly. A conservative fix would add an explicit macOS-only confirmation path that can install Homebrew before running brew-backed `skills.install`, with admin/non-interactive handling matching `scripts/install.sh`; Linux/Docker behavior should be coordinated with #14593, and bundled skill feedback/fallback behavior with #57555. If maintainers do not want OpenClaw to install Homebrew from skill onboarding, the prompt and docs should be changed to make the manual requirement explicit instead.
