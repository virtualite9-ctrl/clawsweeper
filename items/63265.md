## Summary

Keep open. Current main still lacks the requested `openclaw doctor --fix` guard for root/HOME config-path divergence. The doctor path still resolves config from the current process environment, treats a missing target file as a valid empty config, can add gateway auth to that empty config, and then writes it. The only current CLI/service config-path mismatch diagnostic I found is in `gateway status`, not in doctor preflight. The linked PR #63268 appears to be the active unmerged implementation path, so the bug is not implemented on main.

## Best Possible Solution

Keep #63265 open and use #63268 or a maintainer-authored variant as the implementation path. The fix should add a doctor preflight guard that compares doctor’s write path with the running gateway/service config context, warns or blocks before writing when they diverge, points users to `OPENCLAW_CONFIG_PATH=/home/node/.openclaw/openclaw.json openclaw doctor --fix`, and includes a regression test for the root-shell/systemd-HOME mismatch.
