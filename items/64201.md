## Summary

Keep open. Current `main` has partial hardening around restart/startup failures and WebSocket connect errors, but the issue-linked shutdown-order fix is not present on `main`, so there is not strong evidence that #64201 is fully resolved.

## Evidence

- **Issue-linked PR was not merged:** The issue timeline cross-references PR #64255, but that PR was closed rather than landed. The GitHub PR page shows it was auto-closed under `r: too-many-prs`, not merged.
  - command: `web.open https://github.com/openclaw/openclaw/pull/64255`
- **Current shutdown order still matches the reported race:** `createGatewayCloseHandler` still tears down channel/plugin runtime before stopping the config reloader. `stopChannel()` runs at lines 137-139, `pluginServices.stop()` at 141-143, and only later does `configReloader.stop()` run at line 208. That is the opposite of the linked PR's proposed fix.

- **Main has restart-failure hardening, but not a proven fix for this issue:** The gateway run loop now swallows startup failures on subsequent in-process restarts, releases the lock, writes a stability bundle, and keeps the process alive instead of exiting. This likely reduces crash-loop severity, but it does not prove the underlying reload/shutdown race is fixed.
