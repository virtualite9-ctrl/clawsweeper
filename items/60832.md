## Summary

Keep open. Current main still labels the chat composer plus button as "New session" while wiring it to `/new`, and the current session model rotates the `sessionId` under the same session key rather than creating a second Sessions-tab entry. That matches the reporter's confusion and is an unresolved UX/product issue, not a stale or incoherent report.
