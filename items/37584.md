## Summary

Keep open. Current main still does not support per-agent `agents.list[].tools.web.search.*` overrides. The per-agent tools schema/type remains limited to tool policy/runtime controls, while web-search provider settings are resolved from root `tools.web.search` and plugin-owned `plugins.entries.<plugin>.config.webSearch` paths and then passed globally through the web-search runtime.

## Best Possible Solution

Keep the issue open. The best path is to define a documented agent-scoped web-search settings contract, add schema/types for `agents.list[].tools.web.search`, define merge precedence, thread the effective per-agent search config through `createOpenClawTools()` to `createWebSearchTool()`/`runWebSearch()` and provider contexts, and add regression tests plus docs. If maintainers prefer provider plugins to own this, first add a generic SDK/runtime seam for resolving per-agent web-search credentials and settings safely.
