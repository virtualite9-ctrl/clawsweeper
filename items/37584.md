## Summary

Keep open. Current main still does not support per-agent `agents.list[].tools.web.search.*` overrides. The schema/type contract keeps `agents.list[].tools` to tool policy, exec/fs/sandbox/elevated, and loop-detection controls, while web-search provider settings are resolved from root `tools.web.search` plus plugin-owned `plugins.entries.<plugin>.config.webSearch.*` credentials. The reporter also reaffirmed the issue on 2026-04-20, and the related items cover adjacent provider/key-path or tool-policy work rather than this exact agent-scoped web-search settings gap.

## Best Possible Solution

Keep the issue open. The best implementation path is to add a documented agent-scoped web-search settings contract, validate it in agent schema/types, define merge precedence for global/plugin/default/agent settings, thread the effective search config or current agent id through `createOpenClawTools()` -> `createWebSearchTool()` -> `runWebSearch()`/provider context, and add regression tests plus docs. If maintainers want provider plugins to own this, first add a generic plugin SDK seam for resolving per-agent web-search credentials/settings safely.
