## Summary

Keep open. Current main still does not support per-agent `agents.list[].tools.web.search.*` overrides. Web-search settings remain rooted at `tools.web.search` or plugin-owned `plugins.entries.<plugin>.config.webSearch`, while `agents.list[].tools` is still a strict tool-policy/runtime-control shape. Runtime web search construction and provider execution receive only the global config, not an effective agent-scoped search config.

## Best Possible Solution

Keep the issue open. The best path is a documented agent-scoped web-search settings contract: add schema/types for `agents.list[].tools.web.search`, define merge precedence, thread the effective search config or agent id through `createOpenClawTools()` to `createWebSearchTool()`/`runWebSearch()` and provider contexts, update SecretRef activity rules, and add focused regression tests plus docs. If maintainers prefer provider plugins to own this, first add a generic plugin SDK/runtime seam for resolving per-agent web-search credentials and settings safely.
