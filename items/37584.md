## Summary

Keep open. Current `main` still does not support nested per-agent tool config like `agents.list[].tools.web.search.apiKey`; per-agent `tools` is limited to policy/exec/sandbox controls, while web search credentials and settings are still resolved from global `tools.web` or provider env vars.

## Evidence

- **Per-agent tools type has no `web` field:** `AgentToolsConfig` only defines `profile`, `allow`, `alsoAllow`, `deny`, `byProvider`, `elevated`, `exec`, `fs`, `loopDetection`, and `sandbox`. There is no per-agent `web` subtree.
  - file: [src/config/types.tools.ts:304](https://github.com/openclaw/openclaw/blob/55389f4be29359e2dd57ebaf548ba4a70331a445/src/config/types.tools.ts#L304)
  - command: `nl -ba src/config/types.tools.ts | sed -n '300,350p'`
  - sha: [55389f4be293](https://github.com/openclaw/openclaw/commit/55389f4be29359e2dd57ebaf548ba4a70331a445)
- **Per-agent tools schema rejects unknown nested settings:** `AgentToolsSchema` is a strict object with the same limited fields and no `web` key, so `agents.list[].tools.web.*` is not part of the accepted runtime config surface.
  - file: [src/config/zod-schema.agent-runtime.ts:594](https://github.com/openclaw/openclaw/blob/55389f4be29359e2dd57ebaf548ba4a70331a445/src/config/zod-schema.agent-runtime.ts#L594)

  - sha: [55389f4be293](https://github.com/openclaw/openclaw/commit/55389f4be29359e2dd57ebaf548ba4a70331a445)
- **Docs still show `tools.web` only at global scope:** The per-agent config docs show `agents.list[].tools` with policy/elevated examples only, while the configuration reference documents `tools.web` as a top-level config block.
  - file: [docs/gateway/config-agents.md:892](https://github.com/openclaw/openclaw/blob/55389f4be29359e2dd57ebaf548ba4a70331a445/docs/gateway/config-agents.md#L892)

  - file: [src/agents/tools/web-search-provider-config.ts:105](https://github.com/openclaw/openclaw/blob/55389f4be29359e2dd57ebaf548ba4a70331a445/src/agents/tools/web-search-provider-config.ts#L105)
