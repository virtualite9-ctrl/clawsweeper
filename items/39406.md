## Summary

Keep #39406 open. Current main still lacks the requested normal-chat tool-error delivery policy: there is no `toolErrorDelivery` or `suppressMutatingToolErrorWarnings` config surface, `messages.suppressToolErrors` still does not suppress mutating tool warnings, and heartbeat `suppressToolErrorWarnings` remains scoped to heartbeat runs. Current main only partially mitigates same-action retry success by clearing stale mutating errors when the same mutating action later succeeds.

## Best Possible Solution

Keep #39406 open for maintainer product judgment or implementation. The best path is a documented normal-chat tool-error delivery policy, likely near `resolveToolErrorWarningPolicy` and `buildEmbeddedRunPayloads`, with config/schema/help/docs updates and focused payload tests. It should preserve the safe default, avoid regressing same-action retry cleanup, and coordinate with related mutating-warning work such as #39631 and #51091 without treating those unmerged attempts as a shipped replacement.
