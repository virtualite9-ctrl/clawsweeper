## Summary

Keep open. Current main still does not implement the PR’s run-scoped subagent announce tracking: delivery and queue state only carry source session/channel/tool metadata, run records do not persist pending or delivered announce IDs, and restore does not clear stale pending announce state. The PR remains a plausible implementation candidate, but the existing review concern about redundant restore persistence should be resolved before merge.

## Best Possible Solution

Keep this PR open as the implementation candidate. Resolve the existing redundant-persist review comment, rebase on current main, then rerun the targeted subagent announce/registry tests and the appropriate changed-agent validation before maintainer merge review.
