## Summary

Keep #45224 open. Current main still uses in-process Playwright CDP for browser control, generic unhandled rejections still exit the Gateway, the browser plugin has not registered a Playwright-specific unhandled-rejection handler, and the related #45791/#52051 cleanup chain no longer provides a clear open canonical issue for this exact CRSession._onMessage crash report.

## Best Possible Solution

Keep the issue open and fix the browser plugin rather than core’s generic fatal policy: add a browser-owned Playwright/CDP unhandled-rejection handler through `openclaw/plugin-sdk/runtime-env`, register/unregister it in the browser plugin service lifecycle, close or invalidate the affected Playwright CDP connection when matched, add tests for CRSession/CRBrowser assertion-shaped errors including nested/wrapped reasons, and separately consider subprocess isolation for browser/CDP if maintainers want stronger crash containment.
