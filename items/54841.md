## Summary

Keep this issue open. Current main still does not enforce the configured Feishu webhook path before request processing, and PR #54904 is an open, unmerged implementation candidate that explicitly closes this issue. Maintainer policy says the issue should remain open until that PR merges or is rejected.

## Best Possible Solution

Keep the issue open while PR #54904 is open. The best path is maintainer review of that PR, or an equivalent Feishu-owned fix that rejects non-configured webhook paths before body parsing/signature work/dispatch and adds focused regression coverage for wrong-path requests, including query-string behavior.
