## Summary

The request is already implemented on current `main` and was shipped in `v2026.3.13`: OpenClaw has a built-in `user` browser profile backed by Chrome DevTools MCP existing-session attach, auto-connects to the live local Chrome session, and supports tab discovery/listing without a manual `--remote-debugging-port` CDP launch.

## Close Comment

Closing as implemented on main after a Codex review. OpenClaw already ships this capability: `CHANGELOG.md:2525` and `CHANGELOG.md:2526` show that `v2026.3.13` added the official Chrome DevTools MCP existing-session attach mode plus the built-in `profile="user"` path. The current docs also describe the exact flow in `docs/tools/browser.md:378`, including `openclaw browser --browser-profile user tabs`, `transport: chrome-mcp`, and listing already-open tabs. In code, `extensions/browser/src/browser/config.ts:180` auto-creates the `user` profile as `driver: "existing-session"`, and `extensions/browser/src/browser/chrome-mcp.ts:36` launches `chrome-devtools-mcp@latest --autoConnect` and uses `list_pages` for tab discovery. `docs/gateway/doctor.md:205` and `extensions/browser/src/doctor-browser.ts:54` also add Chrome MCP readiness checks. If there is still a current failure with Chrome 146 on a recent build, please open a new bug with exact repro steps, Chrome version, platform, and whether you used `profile="user"` or a custom `existing-session` profile.
