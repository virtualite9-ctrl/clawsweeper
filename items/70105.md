## Summary

Keep open. Current main still does not contain the PR's debug Event Log sticky-at-bottom behavior: the buffer is still newest-first, the Debug view renders the event log without a scroll handler or bounded scroll container, and the scroll helper/lifecycle code only handles chat and gateway file logs. The related #69520 work is about chat scroll interrupts, so it does not supersede this Debug-page fix.

## Best Possible Solution

Keep #70105 open for maintainer review or apply an equivalent narrow fix on main: add Debug Event Log bottom-state tracking and scroll scheduling, wire the Debug view scroll handler, bound the event log container, preserve stable append order with overview compensation, and keep the focused tests in `ui/src/ui/app-scroll.test.ts`.
