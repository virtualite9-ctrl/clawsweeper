## Summary

Keep open. Current main still lacks the PR's Debug event-log sticky-at-bottom behavior: there is no debug event-log scroll state/helper, the Debug view renders the log without a scroll handler, the gateway event buffer still prepends newest events, and existing scroll tests only cover chat behavior. Related PR #69520 is about chat scroll interrupts, not this Debug event-log surface, so it is not a canonical duplicate or superseder.

## Best Possible Solution

Keep #70105 open for maintainer review. The best path is to land or adapt the focused Debug event-log fix: add debug event-log scroll state and scheduling, wire the Debug view scroll handler, constrain the log container, keep overview/debug ordering intentional, and retain focused regression coverage in `ui/src/ui/app-scroll.test.ts`. Treat #69520 as related chat-scroll work, not a replacement for this Debug-page behavior.
