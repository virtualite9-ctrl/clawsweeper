## Summary

Keep open. Current main still has the two source-level gaps described in this issue: ACP runtime visibility is still based on process-local backend registration with no lazy local acpx service start/ensure before explicit embedded spawn paths, and embedded run metadata still has no completed liveness state or sessions_yield descendant-final reconciliation path that can return the reported final JSON state.

## Best Possible Solution

Keep this issue open and patch source. Add a narrow ACP runtime ensure/activation step before explicit embedded ACP availability checks and manager backend lookup, while honoring plugin disablement and plugins.allow policy. Separately, reconcile explicit openclaw agent --json results after sessions_yield by draining descendant ACP/subagent completion, refreshing final assistant text, clearing stale replayInvalid only when terminal completion is proven, and adding or documenting a completed terminal liveness state. Add focused regression coverage for runtime: "acp", sessions_yield, no initial assistant-visible text, descendant completion, final JSON metadata cleanup, and the outside-workspace ACP write smoke path described in the report.
