## Summary

Keep open. Current main still does not implement the requested activity-based sub-agent stall recovery. The shipped paths expose wall-clock `runTimeoutSeconds`, background task audit/lost-state cleanup, and orphan/session cleanup, but not per-subagent last-tool-call tracking, a nudge phase, or automatic post-nudge kill behavior.

## Best Possible Solution

Keep #39305 open until maintainers either implement activity-based sub-agent stall recovery or explicitly consolidate it into a canonical session activity/watchdog design. A likely implementation should add strict config and `sessions_spawn` override fields, track deterministic child activity, inject an auditable nudge into the child session, terminate through the existing subagent/task cancellation flow after the kill threshold, and cover config/runtime behavior with focused tests and docs. PR #43017 is useful prior art but is closed unmerged.
