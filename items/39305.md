## Summary

Keep #39305 open. Current main still only has wall-clock subagent run timeouts and manual subagent kill/steer controls; it does not implement activity-based last-tool-call tracking, a nudge stage, an automatic kill stage, config fields, or sessions_spawn overrides for escalating stall recovery. The related implementation PR #43017 was closed unmerged, and broader #39127/#39141 activity/watchdog issues do not currently ship this concrete recovery behavior.

## Best Possible Solution

Keep #39305 open until maintainers either implement activity-based subagent stall recovery or explicitly consolidate it into a canonical session activity/watchdog design. A good implementation would add strict config and sessions_spawn override fields, track child run/tool activity, inject an auditable nudge into the child session, terminate through the existing subagent kill/cancellation path after the kill threshold, and cover config/runtime/docs with focused tests.
