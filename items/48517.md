## Summary

Keep #48517 open. Current main has subagent/session isolation and a plugin subagent runtime, but it does not implement plugin-requestable trust-boundary isolation from `before_tool_call` or a dedicated hook. The existing hook can only rewrite params, block, or require approval, and the subagent runtime lacks per-run capability constraints and standardized trust/provenance metadata. The issue is also a plugin API/security-boundary design proposal, which `VISION.md` treats as valid core discussion when the missing capability cannot be built as a plugin yet.

## Best Possible Solution

Keep #48517 open for maintainer product/security design. The best path is an additive plugin API design that composes with `before_tool_call` or a dedicated hook, can request an isolated execution context with explicit per-run tool/token/time constraints, and returns standardized provenance/trust metadata. It should be coordinated with the still-open hook-context proposals #48503, #48506, and #48509 rather than closed as implemented or moved wholesale to ClawHub.
