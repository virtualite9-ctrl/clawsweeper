## Summary

Keep open. Current main still has the behavior this PR documents: main-session cron system events bypass HEARTBEAT.md file-gate parsing and use the specialized cron event prompt, while the current cron docs and AGENTS.md template still do not explain that HEARTBEAT.md is not automatically read for those cron events or provide the workaround. The PR discussion also supports the docs change as accurate, with only style/template wording feedback remaining.

## Best Possible Solution

Keep this PR open or replace it with an equivalent docs-only patch. The maintainer path is to document that main-session cron system events use a self-contained cron-event prompt and do not include the default HEARTBEAT.md instruction, add a clear workaround in docs/automation/cron-jobs.md, and make docs/reference/templates/AGENTS.md contain direct agent-facing cron handling guidance. Track any runtime behavior change separately with tests around prompt selection and HEARTBEAT.md file-gate behavior.
