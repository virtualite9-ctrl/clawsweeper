## Summary

Keep open. Current main still has the behavior this docs PR explains: main-session cron events use the specialized cron-event prompt, bypass normal HEARTBEAT.md file-gate parsing, and current docs/templates still do not document the HEARTBEAT.md caveat or workaround.

## Best Possible Solution

Keep this PR open or replace it with an equivalent docs-only patch. The maintainer path is to document that main-session cron system events use a self-contained cron-event prompt and do not include the default `Read HEARTBEAT.md` instruction, add a clear workaround in `docs/automation/cron-jobs.md`, and make `docs/reference/templates/AGENTS.md` operational for agents. Any runtime behavior change should be tracked separately with updated prompt tests.
