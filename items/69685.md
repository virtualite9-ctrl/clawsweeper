## Summary

Current main no longer renders raw persisted `<final>` / `<think>` scaffolding in the Control UI. The chat UI now sanitizes assistant text at render time, so the post-stream rehydrate symptom this PR targets is obsolete on the current surface.

## Evidence

- **Assistant chat rendering sanitizes assistant text before display:** The Control UI does not render raw assistant `message.content` directly. `extractText()` routes assistant messages through `extractSharedAssistantVisibleText(...)`, then `processMessageText()` applies `stripThinkingTags(...)` for assistant roles before rendering.

- **UI stripping path is backed by the shared assistant-visible sanitizer:** `stripThinkingTags()` delegates to `stripAssistantInternalScaffolding()`, and that sanitizer pipeline calls `stripReasoningTagsFromText(...)` as part of assistant text cleanup.
  - file: [src/shared/text/assistant-visible-text.ts:567](https://github.com/openclaw/openclaw/blob/2cd2732ab6695a4b3220ce41166b74aca7097c5f/src/shared/text/assistant-visible-text.ts#L567)
