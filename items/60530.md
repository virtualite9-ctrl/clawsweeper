## Summary

Keep open. Current `main` has partial session-label support, but the requested WebChat-sidebar behavior is not implemented end-to-end: WebChat clients cannot rename sessions themselves, there is no `sessionAliases` config surface, and the default generated group display names still resolve to provider-prefixed slugs like `telegram:g-...` rather than clear human-readable topic/channel aliases.

## Evidence

- **WebChat cannot mutate session labels:** `rejectWebchatSessionMutation` blocks `sessions.patch` for WebChat clients and tells them to use `chat.send`; that means the requested aliasing is not available from the WebChat surface itself.
  - file: [src/gateway/server-methods/sessions.ts:225](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/src/gateway/server-methods/sessions.ts#L225)
  - command: `nl -ba src/gateway/server-methods/sessions.ts | sed -n '225,255p'`
  - sha: [926068b14fc6](https://github.com/openclaw/openclaw/commit/926068b14fc661e915a0247e9ecb5949fb95b4f7)
- **Manual label API exists, but outside WebChat:** `sessions.patch` exists and can persist per-session metadata, so there is partial plumbing for labels; this is not the same as a WebChat-sidebar alias feature because the WebChat client path is blocked above.
  - file: [src/gateway/server-methods/sessions.ts:1284](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/src/gateway/server-methods/sessions.ts#L1284)

  - sha: [926068b14fc6](https://github.com/openclaw/openclaw/commit/926068b14fc661e915a0247e9ecb5949fb95b4f7)
- **Chat UI only prefers labels/display names that already exist:** The chat session selector renders whatever `resolveSessionOptionGroups` produced, and `resolveSessionDisplayName` only prefers existing `label` or `displayName`; there is no alias-creation UX in this path.
  - file: [ui/src/ui/chat/session-controls.ts:23](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/ui/src/ui/chat/session-controls.ts#L23)
  - command: `nl -ba ui/src/ui/chat/session-controls.ts | sed -n '20,70p' && nl -ba ui/src/ui/chat/session-controls.ts | sed -n '390,415p'`

- **Default generated group names are still slug-like:** Gateway session rows derive `displayName` from `buildGroupDisplayName`, and that helper normalizes to `providerKey:token` values such as `telegram:g-leaf`; that matches the issue's complaint about cryptic identifiers rather than solving it.
