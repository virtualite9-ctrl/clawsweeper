## Summary

Current main already fixes the v2026.4.24 Bonjour/CIAO crash loop. The affected release path passed no process-handler wiring into the Bonjour plugin, while current main wires scoped runtime handlers into the advertiser, classifies CIAO probing/announcement cancellations as handled, keeps those handlers alive across watchdog-triggered restarts, and self-disables Bonjour after repeated stuck restarts instead of exiting the gateway.

## Best Possible Solution

Keep the current main implementation: scoped Bonjour/CIAO handlers through the plugin runtime, handler cleanup only on final advertiser shutdown, bounded stuck-restart recovery, and documented disable options for multicast-hostile deployments. The remaining product action is to ship this in the next patch release; v2026.4.24 operators should use the documented Bonjour disable path until upgraded.
