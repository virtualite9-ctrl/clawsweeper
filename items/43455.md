## Summary

Current main still matches the reported Browserbase breakage. The bundled browser docs continue to recommend the static Browserbase WebSocket URL, the browser plugin treats configured CDP URLs as direct endpoints, current tests preserve that URL shape, and Browserbase's current docs describe creating a session first and using the returned connectUrl.

## Best Possible Solution

Keep this issue open. The best path is to update the browser docs so they no longer recommend the obsolete static Browserbase root URL, then either document the manual create-session/connectUrl workflow or add Browserbase session creation and release support in the bundled browser plugin. If maintainers want a vendor-neutral design, add a typed remote-CDP session-provider seam and implement Browserbase through that seam.
