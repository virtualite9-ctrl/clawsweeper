## Summary

Current main still matches the reported Browserbase breakage: the docs recommend the old static Browserbase WebSocket URL, the browser plugin treats configured CDP URLs as direct endpoints, and I found no Browserbase session API or `connectUrl` lifecycle support. Browserbase's current docs describe creating a session first and connecting with the returned connection URL. citeturn1view0turn1view1turn1view2

## Best Possible Solution

Keep this issue open. The best maintainer path is to first update `docs/tools/browser.md` so it no longer recommends the obsolete static Browserbase root URL, then add tested Browserbase session lifecycle support in `extensions/browser` or a typed generic remote-CDP session-provider seam. Coverage should verify session creation, `connectUrl` use, error handling, and release/termination behavior.
