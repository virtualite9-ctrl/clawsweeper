## Summary

Keep open. Current main still does not implement the requested schema-driven tool security classification contract. OpenClaw has tool allow/deny policy, owner-only gating, hook-driven approvals, and a provider/request-level parallel-tool-call knob, but not standardized `isReadOnly`, `isDestructive`, `isConcurrencySafe`, `interruptBehavior`, `isOpenWorld`, or `toAutoClassifierInput` metadata on tool definitions or conformance checks.

## Best Possible Solution

Keep this issue open. The best implementation path is a core/public SDK tool metadata contract: add the classification type, expose it through plugin/channel/tool APIs, preserve it through wrappers and `toToolDefinitions`, annotate core tools, extend `TOOL_POLICY_CONFORMANCE`, document the contract, and then incrementally use the metadata for approval wording, interrupt handling, and scheduling decisions with focused tests.
