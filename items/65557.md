## Summary

Keep open. Current main has a chat/session model picker and a global `agents.defaults.models` allowlist, but it still does not implement the requested per-account/admin provider and model allowlists, account-scoped defaults, wildcard account policy, or account-aware server-side enforcement.

## Best Possible Solution

Keep this issue open and implement it as a core/admin policy feature: add account/user-scoped provider and model policy to config/schema, expose it in Control UI admin settings, filter `models.list` and the chat picker by effective account policy, and enforce the same policy in `sessions.patch`, slash `/model`, chat/run override paths, and OpenAI-compatible model override paths. Reuse `agents.defaults.models` as the global fallback/allowlist building block, then add tests and docs for inheritance, wildcard matching, defaults, and rejection of disallowed provider/model selections.
