## Summary

Keep open. Current main still accepts the full trimmed non-JSON stdout for a single `jsonOnly: false` exec secret ref, so the noisy-stdout hardening is not implemented. This PR is narrowly scoped to the implicated parser and a regression test, and the earlier related attempt #66413 is closed unmerged and mixed in unrelated changes, so it does not supersede this work.

## Best Possible Solution

Keep this PR open for maintainer and `@openclaw/secops` review. The best path is to land a clean version of the fail-closed parser hardening plus the focused regression test, consider a small docs note that `jsonOnly: false` plain output must be exactly one non-empty line, and keep #64821 open until the shipped fix and any broader hot-reload stdout-isolation follow-up are settled.
