## Summary

Keep #39762 open. Current main still does not implement Feishu read-only group mode: the config schema/docs have no `disableGroupReplies` option, Feishu group dispatch paths still create real reply dispatchers, and outbound Feishu target resolution has no read-only guard for explicit or raw group/chat IDs. The PR discussion remains actionable because the author addressed earlier broadcast/whitespace feedback, while a later review comment still flags the bare `oc_*` chat-id bypass.

## Best Possible Solution

Keep this PR open for Feishu plugin maintainer review. The best path is a focused bundled Feishu plugin change: add `disableGroupReplies` to top-level and per-account Feishu config/schema/docs, enforce no-op group dispatch for both broadcast active-agent and single-agent group paths, block explicit and raw Feishu group/chat outbound targets when enabled, add targeted regression tests, and remove or separately justify the `.secrets.baseline` churn before merge.
