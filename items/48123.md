## Summary

Keep open. Current main still has the reported silent-drop behavior for non-image `chat.send` attachments: the protocol accepts arbitrary attachment objects, `parseMessageWithAttachments()` logs and `continue`s on sniffed non-image or unknown non-image payloads, and the existing tests explicitly assert that those payloads are dropped rather than rejected or delivered. The related fail-loud PR #66613 is closed unmerged, and the broader non-image offload PR #67572 is still open per the provided GitHub context, so the remaining work is not shipped on main.

## Best Possible Solution

Keep #48123 open until main either lands a narrow fail-loud fix that turns non-image/unknown attachment drops into explicit `INVALID_REQUEST` errors, or lands the broader `chat.send` non-image offload/document-staging path from #67572. The tests in `src/gateway/chat-attachments.test.ts` should be flipped from asserting logged drops to asserting the chosen supported behavior.
