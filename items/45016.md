## Summary

Keep open. Current main has a provider-owned `dropThinkingBlocks` replay policy that strips older assistant thinking blocks only when the resolved provider/model policy enables it, and tests explicitly preserve thinking for newer Claude models. The requested user-facing `thinkingHistory: last-only | strip | keep` control, and the broad token-saving behavior for all extended-thinking sessions, is not implemented.

## Best Possible Solution

Keep the issue open as a focused feature request. If maintainers want this knob, add an explicit user-facing transcript replay/history policy under agent or gateway config, thread it through `resolveTranscriptPolicy()` and provider replay policy merging, document it in the thinking/config and transcript-hygiene docs, and test the override separately from the current provider-safe defaults.
