## Summary

Current main still does not honor `QQBOT_DATA_DIR` for QQ Bot storage. The diagnostics warning recommends that variable, but the central path helper and runtime persistence paths continue to use `~/.openclaw/qqbot`, so the reported Windows path problem remains actionable.

## Best Possible Solution

Keep the issue open. Implement `QQBOT_DATA_DIR` support in the central QQ Bot storage helper, likely `extensions/qqbot/src/engine/utils/platform.ts:getQQBotDataPath()`, with path normalization/validation and preserved lazy directory creation. Add focused tests for the env override and diagnostics output, then document `QQBOT_DATA_DIR` on `https://docs.openclaw.ai/channels/qqbot` or remove the diagnostic recommendation if maintainers decide not to support it.
