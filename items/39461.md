## Summary

Current main still ignores `QQBOT_DATA_DIR` for QQ Bot persistent storage while diagnostics recommends setting it, so the reported Windows path bug remains actionable.

## Best Possible Solution

Keep the issue open. Implement `QQBOT_DATA_DIR` support in the central QQ Bot storage helper, most likely `extensions/qqbot/src/engine/utils/platform.ts:getQQBotDataPath()`, with normalization and lazy directory creation preserved. Add focused tests for the env override and diagnostics output, then either document `QQBOT_DATA_DIR` on the QQ Bot docs page or remove the diagnostic recommendation if maintainers decide not to support it.
