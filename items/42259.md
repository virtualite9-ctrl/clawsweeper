## Summary

Keep this PR open. Current main still lacks configurable image-tool compression quality/max-side settings, and related #41771 and #42058 were previously closed specifically to keep this PR as the canonical implementation thread. The branch needs maintainer/contributor revision before merge because review comments identify real constraint fallback bugs, the media file has since moved on main, and the PR includes unrelated docs/gateway changes.

## Best Possible Solution

Keep this PR open as the canonical implementation thread, but require a focused rebase. The best patch would add the strict `agents.defaults.imageCompressionQuality` and `agents.defaults.imageCompressionMaxSide` contract, update labels/help/docs, thread those options from `createImageTool` through `loadWebMedia` to `optimizeImageToJpeg`, honor configured out-of-ladder values such as quality 95/85 and maxSide 600, add regression tests for those edge cases, and remove or split unrelated docs/gateway/provider-link changes.
