## Summary

Current main still does not implement configurable image-tool JPEG quality or resize-side compression controls. This PR remains the canonical implementation thread after #41771 and #42058 were closed, but it should not merge as-is because review comments identify constraint-violating fallback behavior and the branch includes unrelated docs/gateway/provider-link changes.

## Best Possible Solution

Keep this PR open as the canonical implementation candidate, but require a focused rebase before merge. The best patch should add a typed and documented `agents.defaults` compression contract, thread quality/max-side options from `createImageTool` into `loadWebMedia` and `optimizeImageToJpeg`, honor out-of-ladder values such as quality 95/85 and max side 600, add regression tests for those edge cases, and split unrelated docs/gateway/provider-link changes into separate PRs if still wanted.
