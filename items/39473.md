## Summary

Current `main` already suppresses the reported Control UI `NO_REPLY` leaks at both the streamed-event path and `chat.history` path, with client-side defense-in-depth still present in the Control UI controller.

## Evidence

## Close Comment

Closing as implemented on current `main` after Codex review. The reported Control UI leak path is already covered in the codebase: streamed `NO_REPLY` lead fragments are suppressed before chat events are surfaced (`src/gateway/control-reply-text.ts`, regression coverage in `src/gateway/server-chat.agent-events.test.ts:255`), `chat.history` drops assistant rows whose visible text is only `NO_REPLY` (`src/gateway/server-methods/chat.ts:1168` with coverage in `src/gateway/server.chat.gateway-server-chat.test.ts:535`), and the Control UI still keeps a client-side guard in `ui/src/ui/controllers/chat.ts:13`. The repo changelog also records these fixes under the webchat/control-ui silent-token entries in `CHANGELOG.md:3347` and `CHANGELOG.md:3369`. If someone can still reproduce a visible token leak on a newer build, it should be tracked as a fresh issue with exact reproduction steps and the specific leaked token/path.
