## Summary

Current main still has the reported Signal installer hardening gap. The old core path has moved into the bundled Signal plugin, but `downloadToFile` still uses raw `node:https` without a request timeout, SSRF-checked redirects, proxy-aware guarded fetch, or byte cap enforcement.

## Best Possible Solution

Keep this open and fix it in the bundled Signal plugin, not generic core. Replace the raw `node:https` downloader in `extensions/signal/src/install-signal-cli.ts` with `fetchWithSsrFGuard` from `openclaw/plugin-sdk/ssrf-runtime`, choose an explicit install-download timeout and redirect limit, enable trusted env-proxy mode only for this trusted GitHub release URL flow when appropriate, stream through a byte-counting limiter with a conservative maximum archive size, and add focused tests for timeout wiring, private-network redirect blocking, proxy mode selection, content-length rejection, streamed over-limit abort, and temp-file cleanup.
