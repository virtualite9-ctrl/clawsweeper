## Summary

Keep open. Current `main` still performs full session reindexing for changed session files, and the incremental chunk-preserving path described in PR #59577 is not present on `main`. The PR discussion also contains unresolved correctness concerns around failure ordering and FTS metadata refresh, so there is not strong evidence for any conservative auto-close reason.
