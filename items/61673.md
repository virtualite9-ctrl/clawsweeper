## Summary

Keep open. Current `main` still only supports static `sandbox.docker.binds` merged from config, and `setupCommand` runs after container creation. I found no host-side pre-container hook or existing plugin/internal hook that can resolve per-session bind mounts before Docker create.

## Evidence

- **Sandbox docker schema has no pre-create bind resolver field:** `SandboxDockerSchema` accepts static config such as `setupCommand`, `binds`, and other Docker settings, but there is no `resolveBinds` or similar callback/executable field.
  - file: [src/config/zod-schema.agent-runtime.ts:113](https://github.com/openclaw/openclaw/blob/8b76392e3e7934b1cdd6d461148263fd56141ec8/src/config/zod-schema.agent-runtime.ts#L113)
  - command: `nl -ba src/config/zod-schema.agent-runtime.ts | sed -n '110,180p'`
  - sha: [8b76392e3e79](https://github.com/openclaw/openclaw/commit/8b76392e3e7934b1cdd6d461148263fd56141ec8)
- **Bind mounts are resolved as a static merged array:** `resolveSandboxDockerConfig()` constructs `binds` by concatenating global and agent arrays. There is no session/context-dependent resolution step.
  - file: [src/agents/sandbox/config.ts:83](https://github.com/openclaw/openclaw/blob/8b76392e3e7934b1cdd6d461148263fd56141ec8/src/agents/sandbox/config.ts#L83)
  - command: `nl -ba src/agents/sandbox/config.ts | sed -n '80,130p'`
  - sha: [8b76392e3e79](https://github.com/openclaw/openclaw/commit/8b76392e3e7934b1cdd6d461148263fd56141ec8)
- **Container creation uses existing cfg.binds and only runs setup after create/start:** `createSandboxContainer()` builds Docker create args from `cfg`, appends custom binds, creates and starts the container, and only then runs `cfg.setupCommand` via `docker exec`. That rules out `setupCommand` as a way to affect bind mounts.
  - file: [src/agents/sandbox/docker.ts:470](https://github.com/openclaw/openclaw/blob/8b76392e3e7934b1cdd6d461148263fd56141ec8/src/agents/sandbox/docker.ts#L470)

  - sha: [8b76392e3e79](https://github.com/openclaw/openclaw/commit/8b76392e3e7934b1cdd6d461148263fd56141ec8)
- **Docs describe binds as static and setupCommand as post-creation:** Agent config docs state that `setupCommand` runs once after container creation and that `docker.binds` are merged from global and per-agent config.
  - file: [docs/gateway/config-agents.md:834](https://github.com/openclaw/openclaw/blob/8b76392e3e7934b1cdd6d461148263fd56141ec8/docs/gateway/config-agents.md#L834)
