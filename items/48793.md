## Summary

Keep open. Current main has partial lifecycle primitives for channels and plugin services, but it does not provide the proposed centralized PluginResourceManager-style resource contract with enforced async per-account cleanup. The motivating Feishu HTTP cleanup path still calls server.close() and immediately deletes/clears state, and the related Feishu issue/PR remain open rather than shipped.

## Best Possible Solution

Keep #48793 open for maintainer design. The likely path is a narrow, backward-compatible SDK/core lifecycle extension built on the existing ChannelGatewayAdapter/runtime-context/plugin-service machinery: expose tracked async resource handles or an account-scoped cleanup contract, enforce bounded awaited teardown in gateway shutdown/restart, migrate Feishu/Telegram/Slack/Discord cleanup paths, and add regression tests for restart ordering and port/resource release. Coordinate the concrete Feishu fix with #48183/#48588 and the broader lifecycle standardization discussion in #49804 rather than closing this item now.
