## Summary

Keep open. Current main still lacks the PR's requested audit distinction: `MIGRATION_RECOMMENDED` and `migrationRecommendedCount` are absent, `openclaw secrets audit` classifies already-resolved config target values as plaintext, and the docs/CLI summary still expose only the existing plaintext/unresolved/shadowed/legacy codes. The PR remains a focused implementation candidate, and the discussion shows the author intentionally narrowed out broader config.env/dotenv attribution follow-ups.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land or revise the focused fix that compares authored config after `$include` resolution with the resolved audit target, emits `MIGRATION_RECOMMENDED` only for exact `${ENV_VAR}` templates on SecretRef-capable fields, preserves mixed-string plaintext findings, and updates the CLI JSON/text summary, docs, and regression tests. Keep the broader bare `$VAR` and dotenv/config.env attribution work in separate issues or PRs.
