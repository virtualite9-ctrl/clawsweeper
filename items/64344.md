## Summary

Keep open. Current main still lacks the PR's requested audit distinction: there is no `MIGRATION_RECOMMENDED` finding code or `migrationRecommendedCount`, and config audit still classifies the resolved config value without comparing it to the authored `${ENV_VAR}` template. The PR remains a focused implementation candidate, and the discussion shows the author intentionally narrowed out broader `config.env`/dotenv attribution follow-ups.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land or revise the focused fix that compares the authored config after `$include` resolution with the resolved audit target, emits `MIGRATION_RECOMMENDED` only for exact `${ENV_VAR}` templates on SecretRef-capable fields, preserves mixed-string plaintext findings, and updates the CLI JSON/text summary plus docs and regression tests.
