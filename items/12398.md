## Summary

Keep #12398 open. Current main still lacks the requested cron postconditions contract and deterministic post-run verifier flow: no `postconditions` job field, no persisted postcondition state/output, no checker execution between isolated agent completion and `applyJobResult`, no checker-driven retry prompt injection, and no plugin/API seam that can fail an otherwise successful cron run and schedule cron-owned retry/escalation. The existing code covers execution status, delivery status, transient-error retry/backoff, interim-ack retry, and failure alerts, but not outcome verification after a clean agent stop.

## Best Possible Solution

Keep #12398 open for maintainer/product implementation. The likely path is an additive cron job contract plus Gateway schema fields for postconditions, deterministic checker execution after isolated agent completion and before `applyJobResult`, persisted checker status/output in cron state and run logs, retry prompt injection with checker output, and integration with existing retry/failure-alert delivery. If maintainers prefer a plugin route, the prerequisite is a first-class cron post-run hook/API that can fail a cron run and schedule cron-owned retry/escalation.
