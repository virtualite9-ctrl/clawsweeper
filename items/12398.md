## Summary

Keep #12398 open. Current main still lacks a first-class cron postcondition contract and deterministic post-run verification path: cron jobs have retry/failure alerts for execution errors, but no `postconditions` field, no persisted verifier state/output/retry count, no shell checker after isolated agent completion, and no way for a clean `ok` agent run to be failed and retried by a verifier.

## Best Possible Solution

Keep #12398 open for maintainer/product implementation. The likely path is an additive cron job contract plus gateway/CLI/schema/docs updates, deterministic verifier execution after isolated agent completion and before applyJobResult, persisted verifier state and run-log/event fields, retry prompt injection with bounded checker output, and integration with cron-owned retry/failure alert delivery. If maintainers prefer plugin ownership, first add a supported cron post-run hook/API that can fail a run and request cron-owned retry/escalation.
