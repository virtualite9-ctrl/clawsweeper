## Summary

Keep #48010 open. Current main has partial Ollama thinking-control support: `/think off` / `--thinking off` is forwarded to native Ollama as top-level `think: false`. But the broader requested model-specific Ollama parameter surface is still missing: `models.providers.*.models[]` is strict and has no `params`/`parameters` field, and the native Ollama `/api/chat` request builder does not merge arbitrary configured provider params such as `top_p` or `thinking` into `options`. The related PR #48023 was closed unmerged, and #69464 is still open/adjacent rather than shipped.

## Best Possible Solution

Keep the issue open. The best product/code path is to choose one supported config surface, document it, and wire it end to end for native Ollama: accept provider-specific model params, validate/sanitize them, merge supported fields into the `/api/chat` request in the correct top-level vs `options` shape, preserve current `contextWindow`/`maxTokens` behavior and `/think` controls, and add regression tests for `think`, `temperature`, `top_p`, and `num_ctx` precedence. #69464 can remain the adjacent num_ctx threading work if maintainers choose to land it, but it does not fully supersede this request today.
