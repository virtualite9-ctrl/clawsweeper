## Summary

Keep open. Current main has Telegram-native exec approval delivery, but it does not implement the requested trusted-owner approval bypass. Elevated sender allowlists and exec approvals are still separate gates, so an allowlisted Telegram owner can still hit exec approval prompts unless they explicitly switch to `/elevated full` or relax exec approval policy more broadly.

## Risks / Open Questions

- The issue's context-switching complaint is partially mitigated by current `channels.telegram.execApprovals.target` support, so maintainers may want to scope the remaining request narrowly as a true bypass feature rather than generic Telegram approval UX.
- Because exec approvals are documented as a host-side safety interlock rather than a per-user auth boundary, any trusted-owner bypass would likely need explicit product/security owner review before cleanup closure.
