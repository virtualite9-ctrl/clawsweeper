## Summary

Keep #13700 open. Current main still lacks the requested user-facing native session snapshot surface: no `/session save`, `/session load`, named snapshot API, reusable snapshot template, or restore path for `.reset`/`.deleted` archives. Existing `sessions_spawn context:"fork"` and `sessions.compaction.*` branch/restore are useful partial primitives, but they do not close the central request. The latest provided maintainer discussion also explicitly kept this issue open for the same reason.

## Best Possible Solution

Keep #13700 open until OpenClaw implements a deliberate native session snapshot surface or splits it into canonical follow-ups. A strong path is typed Gateway methods for named snapshot save/list/load/delete, slash/TUI controls, deterministic preservation of transcript/model/session metadata where feasible, reusable spawn-from-snapshot support, and a safe restore path for `.reset`/`.deleted` transcript archives.
