## Summary

Keep this PR open. Current main still does not expose effective mounts from `openclaw sandbox explain` in JSON or text output, and the existing PR is a plausible implementation candidate, though it needs the review concerns addressed before landing.

## Best Possible Solution

Keep the PR open as the active implementation candidate. The best path is to update `sandbox explain` with a backend-aware effective-mounts helper, reuse the same canonical sandbox runtime status logic used by runtime enforcement, emit an empty mounts array or explicit direct-runtime hint when not sandboxed, cover JSON and text output plus custom binds, agent workspace overrides, aliases, and SSH/remote backend roots, and remove unrelated file churn before maintainer review.
