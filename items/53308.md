## Summary

Keep open. Current main still has an actionable bundled Signal group-ingress bug: Signal group messages are blocked by sender-based `groupAllowFrom` gating before the Signal group id in `channels.signal.groups` is consulted, and mention gating still defaults to `true` when no explicit `requireMention` value is present. This is not a ClawHub/plugin-scope close, not implemented on main, and not superseded by the separate Signal SSE issue.

## Best Possible Solution

Keep this issue open and fix the bundled Signal plugin. The likely implementation path is to use `channels.signal.groups`/`resolveChannelGroupPolicy` to admit configured Signal group IDs under `groupPolicy="allowlist"` before sender authorization drops the message, while keeping `groupAllowFrom` as sender authorization. Add regression tests for a specific Signal group id under allowlist policy and for the explicit `requireMention:false` workaround. Decide separately whether Signal group entries without `requireMention` should default to false or whether docs/config should continue requiring an explicit override.
