## Summary

Keep open. Current main still has an actionable bundled Signal group-ingress bug: Signal group messages are gated by sender-based `groupAllowFrom` before the configured Signal `groups` entry for the group ID can admit the message. The `requireMention` default still falls through to true, but that part matches current docs and needs maintainer policy judgment rather than cleanup closure.

## Best Possible Solution

Keep this issue open and fix the bundled Signal plugin. The likely path is to make Signal group admission consult `channels.signal.groups` via `resolveChannelGroupPolicy` for the Signal group ID before sender authorization can drop the event, while preserving `groupAllowFrom` as sender authorization inside admitted groups. Add regression tests for a configured Signal group ID with no sender `groupAllowFrom`, sender allowlist rejection/acceptance inside configured groups, and the explicit `requireMention: false` workaround. Decide separately whether an explicit group entry without `requireMention` should default to false or whether docs/config should continue requiring an explicit override.
