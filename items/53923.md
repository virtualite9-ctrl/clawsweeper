## Summary

Current `main` already avoids the reported Codex keychain failure path. Codex is no longer an external CLI bootstrap source, the Codex keychain read returns `null` on `security` failures, and the shipped docs say onboarding no longer imports OAuth from `~/.codex`. This implementation is present in release `v2026.4.22`.

## Close Comment

Closing this as implemented on current `main` after a Codex review. The reported Codex external-CLI bootstrap path is no longer active: `src/agents/auth-profiles/external-cli-sync.ts:74` only registers MiniMax for external CLI sync, and `src/agents/auth-profiles.external-cli-sync.test.ts:222` explicitly asserts that OpenClaw “does not use codex as a runtime bootstrap source anymore.” On top of that, the Codex keychain helper now catches `security find-generic-password` failures and returns `null` instead of propagating an error (`src/agents/cli-credentials.ts:214`). The shipped docs also say onboarding no longer imports OAuth material from `~/.codex` (`docs/providers/openai.md:163`). This behavior is already present in release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`). If you still hit a similar credential error on a newer build, please open a fresh issue with exact repro steps and logs from that version.
