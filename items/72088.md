## Summary

Current main implements the Telegram missing-runtime-dependency repair path. The Telegram plugin declares grammY runtime deps, grammy brings abort-controller transitively, configured Telegram channels are auto-enabled, gateway startup stages missing bundled plugin deps before loading the plugin, and doctor --fix now installs missing configured-channel runtime deps into OpenClaw-managed external staging.

## Best Possible Solution

Close this missing Telegram runtime dependency issue as implemented on current main. Keep the main implementation: extension-owned runtime deps plus gateway/doctor managed staging. Track any remaining Windows ESM URL or postinstall pruning failures in the broader related reports such as #72040 and #72042.
