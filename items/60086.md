## Summary

Keep open. Current main still has the two user-visible gaps this PR tries to fix: active-run follow-up queueing returns no acknowledgement, and final stale approval resolution failures surface as raw gateway errors. The PR also has an unresolved P1 review concern about preserving stale-snapshot queue drain when enqueue is rejected, so the best maintainer path is a corrected PR or successor rather than auto-closing it.

## Best Possible Solution

Keep the PR or a successor open. The right fix is to add a structured enqueue result in `src/auto-reply/reply/queue/enqueue.ts`, surface queued/backlog-full acknowledgements from `src/auto-reply/reply/agent-runner.ts`, map stale exec approval resolution failures to a recovery hint in `src/auto-reply/reply/commands-approve.ts` or the shared approval error helper, and preserve stale-snapshot `finalizeWithFollowup` behavior even when enqueue is rejected.
