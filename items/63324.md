## Summary

Keep open. Current main still has the underlying silent-swallow behavior in pairing allowFrom paths: async reads now rethrow non-ENOENT through the shared helper, but sync stat failures still return an empty store, and writeAllowFromState still has a bare catch after fs.promises.stat(). The PR is stale against a refactor, but the fix remains useful if updated to current main with regression coverage.

## Best Possible Solution

Keep the PR open and ask for a rebase against current main. The best fix is to update the shared allowFrom store helper so sync stat failures rethrow non-ENOENT, update writeAllowFromState to only tolerate ENOENT after the post-write stat, and add focused tests for EACCES/EIO on both sync read and async write-cache paths.
