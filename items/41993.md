## Summary

Current main still rejects a DNS result set when any resolved address is blocked, before IPv4 preference and pinned lookup creation. The reported mixed public IPv4 plus special-use IPv6 case remains actionable and is not shipped by the current implementation.

## Best Possible Solution

Keep this issue open. Implement a narrow SSRF pinning change in `src/infra/net/ssrf.ts`: classify DNS answers, keep only allowed results, throw only when no allowed answers remain, and pass only the allowed set into IPv4-first dedupe and pinned lookup creation. Add focused tests for public IPv4 plus `::`, public IPv4 plus Teredo/reserved IPv6, all-blocked DNS still failing, and pinned results excluding blocked addresses.
