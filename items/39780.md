## Summary

Keep #39780 open. Current main still has the reported Control UI gap: the insecure HTTP Overview recommendation is manual copy plus docs links, while before/after diff and base-hash guarded config write primitives remain scoped to Config/Quick Settings flows rather than recommendation callouts.

## Best Possible Solution

Keep #39780 open until Control UI recommendation callouts can stage a narrow base-hash guarded config change, show a before/after diff, offer accept/dismiss controls, and confirm the applied result. The implementation should reuse the existing Config form diff/apply controller primitives, wire a config mutation path into Overview or a shared recommendation component, add focused tests for the insecure HTTP recommendation path, and update the public Control UI docs.
