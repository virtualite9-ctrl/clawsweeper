## Summary

Close PR #61392 as superseded by the linked open issue #61389. The PR's remaining useful question is already tracked there, while the submitted implementation is not safe to merge as-is: it hard-codes a stale-threshold floor that conflicts with current interval-relative validation, omits generated schema updates, and describes `channelMaxRestartsPerHour: 0` as unlimited even though current runtime behavior treats it as zero allowed restarts.

## Best Possible Solution

Close this PR and keep #61389 as the canonical tracker. Any eventual fix should preserve interval-relative stale-threshold validation, define `channelMaxRestartsPerHour: 0` semantics explicitly if desired, and update zod schema, generated schema, docs/help text, and tests together.
