## Summary

Keep open. Current main still chowns only `/home/node/.cache/ms-playwright`, while this PR’s proposed fix widens ownership to `/home/node/.cache`. The runtime image later switches to `USER node`, and Docker/browser docs still use `/home/node/.cache/ms-playwright`, so the reported parent-directory ownership problem remains plausible and has not been absorbed by main. Related #43846 was closed unmerged as a duplicate of this parent item, not as a shipped replacement. Security review found no unrelated workflow, dependency, lockfile, script, or secret-scope changes in the provided diff; the only proposed change is the Dockerfile ownership target, but it should still get normal Docker maintainer validation because it broadens writable ownership under the node user’s cache directory.

## Best Possible Solution

Keep this PR open. The best path is to review the one-line Docker ownership fix, preferably add a small Dockerfile assertion for `chown -R node:node /home/node/.cache` or provide a targeted `OPENCLAW_INSTALL_BROWSER=1` build/run proof, then land it if that validation passes.
