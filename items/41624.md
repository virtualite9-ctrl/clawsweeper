## Summary

Keep this PR open. Current main still creates `/home/node/.cache/ms-playwright` in the optional browser-install Docker path and only chowns that child directory before switching to `USER node`, so the reported parent-cache ownership problem is still plausible and not implemented on main. The PR is a focused one-line Dockerfile fix, and the duplicate attempt #43846 was closed unmerged rather than superseding it.

## Best Possible Solution

Keep this PR open as the active implementation candidate. The maintainer path is to review the focused Docker ownership change, add or request targeted `OPENCLAW_INSTALL_BROWSER=1` ownership proof, and merge it if validation passes; otherwise close it only with a concrete alternative fix that makes `/home/node/.cache` writable for the runtime `node` user.
