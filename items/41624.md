## Summary

Keep this PR open. Current main still creates `/home/node/.cache/ms-playwright` in the optional Docker browser-install path and only chowns that child directory, so `/home/node/.cache` can remain root-owned before the image switches to `USER node`. The PR remains a focused one-line ownership fix, the duplicate #43846 was closed unmerged, and current tests/CI/docs do not show that this behavior has been superseded or already covered.

## Best Possible Solution

Keep this PR open as the active implementation candidate. The maintainer path is to review the focused Docker ownership change, request or add targeted proof that `/home/node/.cache` is writable under `USER node` when built with `OPENCLAW_INSTALL_BROWSER=1`, and land it if validation passes.
