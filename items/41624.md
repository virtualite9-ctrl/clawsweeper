## Summary

Keep PR #41624 open. Current main still has the exact Docker ownership gap the PR targets: the optional browser install creates `/home/node/.cache/ms-playwright` as root, chowns only the child directory, then switches the runtime image to `USER node`, leaving the parent cache directory unwritable for later node-user cache writes. The related #43846 was closed as a duplicate child, not merged as a superseding fix, so #41624 remains the canonical open patch.

## Best Possible Solution

Keep #41624 open as the canonical focused Docker bug fix. The likely maintainer path is to rebase it onto current main, change the ownership target to `/home/node/.cache`, and add either a small Dockerfile regression assertion or explicit manual verification with `OPENCLAW_INSTALL_BROWSER=1` that the runtime `node` user can write to `/home/node/.cache`.
