## Summary

Keep open. Current main still keeps the Usage header and Daily Cost chart on separate summaries: the header uses `sessions.usage` totals across configured agents, while the chart uses `usage.cost`, which still resolves a single agent transcript directory when no `agentId` is passed. The linked candidate fix PR #42063 was closed unmerged, and #68066 is a separate provider-billing correctness issue, not a canonical duplicate of this dashboard mismatch.

## Best Possible Solution

Keep this issue open and fix the gateway/UI contract so the Usage header and Daily Cost chart use one consistent aggregation scope and cost unit. The likely path is to revive the useful part of #42063: aggregate `usage.cost` across configured gateway agents, or derive both header totals and daily bars from a shared backend summary, then add regression coverage proving multi-agent totals and daily bars match for the selected range. If legacy MiniMax/DashScope transcripts can store micro-dollar values, add explicit normalization or repair coverage rather than formatting around it in the UI.
