## Summary

Keep open. Current main still has the same Usage-page split that can produce mismatched header and daily totals: the UI requests `sessions.usage` and `usage.cost` separately, renders the no-filter header from `sessions.usage`, while `usage.cost` still loads a single agent transcript directory rather than aggregating the same all-agent scope used by `sessions.usage`. The linked fix PR was closed unmerged, so there is no shipped implementation to close against.

## Best Possible Solution

Keep this issue open and fix the gateway/UI contract so the Usage header and Daily Cost chart use the same aggregation scope and cost unit. The most concrete path is to revive or rework #42063's idea: aggregate `usage.cost` across the configured agents, or otherwise derive both header and daily totals from one shared backend summary, then add regression coverage proving multi-agent totals and daily sums match for the selected date range. If the MiniMax/DashScope path also stores legacy micro-dollar values, add explicit normalization or doctor/repair handling with tests rather than only formatting around it in the UI.
