## Summary

Keep open. Current main still leaves the Usage dashboard header and Daily Cost chart backed by separate summaries with different aggregation scope: the header is based on `sessions.usage` totals, while the chart is based on `usage.cost`. The candidate fix PR #42063 was closed unmerged, and #68066 is a separate provider-billing accuracy issue rather than a canonical replacement for this dashboard mismatch.

## Best Possible Solution

Keep this issue open and fix the gateway/UI contract so the Usage header and Daily Cost chart share one aggregation scope and cost unit. The most direct path is to revive the useful part of #42063: aggregate `usage.cost` across configured gateway agents, or derive both header totals and daily bars from a shared backend summary, then add regression coverage showing multi-agent totals and daily bars match for the selected date range. If legacy MiniMax/DashScope transcripts can store micro-dollar values, add explicit normalization or repair coverage in the data layer rather than formatting around it in the UI.
