## Summary

Keep #7575 open. Current main still does not implement Sysbox/sysbox-runc gateway isolation or a supported `sandbox.docker.runtime` selector, and the latest issue discussion explicitly keeps this as the canonical Sysbox/runtime architecture tracker requiring maintainer judgment.

## Best Possible Solution

Keep #7575 open as the canonical Sysbox/runtime-architecture tracker. A good resolution would either add a conservative `sandbox.docker.runtime` selector with types, strict schema, Docker create wiring, docs, tests, and host setup guidance, or explicitly endorse a Sysbox sandbox backend plugin using the existing `registerSandboxBackend` SDK seam if maintainers decide this belongs outside core.
