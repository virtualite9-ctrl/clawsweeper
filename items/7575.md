## Summary

Keep #7575 open. Current main still does not implement Sysbox/sysbox-runc, still has no `agents.defaults.sandbox.docker.runtime` selector, and the maintained Docker sandbox path remains Docker-socket/DooD based. The issue discussion also has a same-day maintainer/Codex review explicitly treating #7575 as the canonical Sysbox/runtime-architecture tracker rather than completed by sandbox-common, Docker socket support, or the sandbox backend plugin seam.

## Best Possible Solution

Keep #7575 open as the canonical Sysbox/runtime-architecture tracker. A good resolution would either add a conservative Docker/OCI runtime selector with types, strict schema, config resolution, Docker create `--runtime` wiring, docs, and tests, or explicitly move Sysbox to a plugin-backed sandbox backend built on `registerSandboxBackend` with public docs, ClawHub tracking, and any missing SDK seams proposed separately.
