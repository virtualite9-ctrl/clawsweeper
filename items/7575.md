## Summary

Keep #7575 open. Current main still does not implement Sysbox/sysbox-runc gateway isolation or a supported `agents.defaults.sandbox.docker.runtime` selector; the maintained Docker sandbox path remains Docker-socket based. A plugin-backed sandbox backend seam exists and could support a future Sysbox backend, but it does not itself satisfy the requested Sysbox/runtime architecture change, and the latest issue discussion explicitly keeps this as the canonical tracker.

## Best Possible Solution

Keep #7575 open as the canonical Sysbox/runtime-architecture tracker. A good resolution would either add a conservative Docker/OCI runtime selector with types, strict schema, config resolution, Docker create wiring, docs, and tests, or explicitly move Sysbox to a plugin-backed sandbox backend built on `registerSandboxBackend` with public docs and ClawHub tracking.
