## Summary

Current `main` now clears stale session model/context metadata when a model switch is persisted, and `/status` has regression coverage proving it recomputes the context window from the active model instead of keeping the previous smaller window.

## Evidence

- **Model-switch persistence now clears stale cached context window:** `applyModelOverrideToSessionEntry` deletes stale `entry.model`, `entry.modelProvider`, and `entry.contextTokens` when the selected model changes or the runtime model is stale, with an inline comment calling out the old-window pinning problem.
  - file: [src/sessions/model-overrides.ts:56](https://github.com/openclaw/openclaw/blob/51c11cfd903033947f01f96a7169ebf92d194064/src/sessions/model-overrides.ts#L56)
  - command: `nl -ba src/sessions/model-overrides.ts | sed -n '56,90p'`
  - sha: [51c11cfd9030](https://github.com/openclaw/openclaw/commit/51c11cfd903033947f01f96a7169ebf92d194064)
- **The switch-clearing behavior is covered by focused regression tests:** Tests assert that switching overrides clears stale runtime model fields and `contextTokens`, including the case of switching back to the default model after a smaller override.
  - file: [src/sessions/model-overrides.test.ts:23](https://github.com/openclaw/openclaw/blob/51c11cfd903033947f01f96a7169ebf92d194064/src/sessions/model-overrides.test.ts#L23)
  - command: `nl -ba src/sessions/model-overrides.test.ts | sed -n '20,125p'`
  - sha: [51c11cfd9030](https://github.com/openclaw/openclaw/commit/51c11cfd903033947f01f96a7169ebf92d194064)
- **User-facing status regression covers this exact scenario:** `status.test.ts` has a case named `recomputes context window from the active model after switching away from a smaller session override` and expects `/status` output to show the larger active-model window (`Context: 1.0k/66k`).
  - file: [src/auto-reply/status.test.ts:588](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/src/auto-reply/status.test.ts#L588)
  - command: `nl -ba src/auto-reply/status.test.ts | sed -n '584,636p'`
  - sha: [926068b14fc6](https://github.com/openclaw/openclaw/commit/926068b14fc661e915a0247e9ecb5949fb95b4f7)
- **Both slash-command and gateway model-switch paths use the new helper:** The persisted `/model` directive path and the gateway `sessions.patch` path both call `applyModelOverrideToSessionEntry`, so the stale-context cleanup is applied across the main model-switch entrypoints.
  - file: [src/auto-reply/reply/directive-handling.persist.ts:199](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/src/auto-reply/reply/directive-handling.persist.ts#L199)
  - command: `nl -ba src/auto-reply/reply/directive-handling.persist.ts | sed -n '192,214p'; nl -ba src/gateway/sessions-patch.ts | sed -n '388,430p'`
  - sha: [926068b14fc6](https://github.com/openclaw/openclaw/commit/926068b14fc661e915a0247e9ecb5949fb95b4f7)
- **Status rendering also prefers active-model context metadata when selected/runtime models differ:** `buildStatusMessage` recomputes context tokens from the active runtime model and handles stale persisted values instead of blindly trusting the old selected-model window.
  - file: [src/status/status-message.ts:625](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/src/status/status-message.ts#L625)
