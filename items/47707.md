## Summary

Keep open. Current main still does not implement strict9 tool-call ID replay policy for Mistral-family models routed through OpenRouter. The linked PRs and reporter comment are useful fix attempts, but current OpenRouter replay policy remains passthrough-Gemini only and does not set `sanitizeToolCallIds` or `toolCallIdMode: "strict9"` for `mistralai/devstral-2512:free`.

## Best Possible Solution

Keep #47707 open as the canonical focused bug. Implement the fix in the current provider-plugin replay-policy path, likely by extending OpenRouter's `buildReplayPolicy` or a shared replay helper to return `sanitizeToolCallIds: true` and `toolCallIdMode: "strict9"` only for Mistral-family OpenRouter model IDs, while preserving existing Gemini passthrough behavior. Add regression coverage for OpenRouter + `mistralai/devstral-*` or similar, plus non-Mistral OpenRouter models.
