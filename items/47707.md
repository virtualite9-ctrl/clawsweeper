## Summary

Keep open. Current `main` does not show an OpenRouter-specific replay policy that applies Mistral `strict9` tool-call ID sanitization for `openrouter/mistralai/...` routes; the only cross-provider Mistral handling currently wired in is transport compat, not transcript replay policy.

## Evidence

- **OpenRouter plugin replay hooks:** The OpenRouter plugin registers `...PASSTHROUGH_GEMINI_REPLAY_HOOKS` and no Mistral-specific replay hook or `buildReplayPolicy` override.
  - file: [extensions/openrouter/index.ts:140](https://github.com/openclaw/openclaw/blob/d38ed0831d7372c8505dc944aea69b5b83da1435/extensions/openrouter/index.ts#L140)

  - file: [src/plugin-sdk/provider-model-shared.ts:188](https://github.com/openclaw/openclaw/blob/d38ed0831d7372c8505dc944aea69b5b83da1435/src/plugin-sdk/provider-model-shared.ts#L188)
