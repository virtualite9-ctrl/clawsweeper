## Summary

Keep this open. The report is recent, concrete, and tied to the core macOS WebChat surface. Current main still renders the visible chat through the native SwiftUI/AppKit WebChat path with transparent non-opaque windows, active visual-effect material, material/gradient backgrounds, shadows, and frequent visible scroll animations. I found construction smoke coverage and WebChat docs, but no current source, docs, tests, changelog, or history evidence that the WindowServer/compositor CPU issue has been reproduced, fixed, or measured away on affected Intel Retina hardware.

## Best Possible Solution

Keep this open as a macOS WebChat rendering performance investigation. Profile current native WebChat on affected Intel Retina hardware, compare opaque/no-material/no-shadow/no-animation variants, verify Reduce Motion behavior, and add a measured low-compositor fallback for older Intel or high-resolution displays if the regression is confirmed.
