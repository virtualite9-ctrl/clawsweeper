## Summary

Keep this issue open. The report is recent and concrete, and current main still has compositor-heavy macOS chat surfaces with transparent non-opaque windows, NSVisualEffectView material, SwiftUI material/gradient layers, shadows, and active chat animations. ## Best Possible Solution

Keep this open as a macOS chat rendering performance investigation. Profile the current native chat window on affected Intel Retina hardware, compare opaque/no-material/no-shadow/no-animation variants, honor Reduce Motion where relevant, and consider a low-compositor fallback for older Intel/high-resolution displays if measurements confirm the regression.
