## Summary

Current `main` does not show a shipped fix for this report. The reset path still intentionally forwards internal-hook `event.messages` back to the channel on `/new` and `/reset`, and the docs still describe that as user-visible behavior. At the same time, there is no built-in `self-improvement` note source in the repo, so the exact leaked text likely depends on workspace/plugin/hook state outside this checkout. That is not strong enough evidence to auto-close conservatively.

## Evidence

- **Reset command hooks still route hook-emitted messages back to the channel:** `emitResetCommandHooks()` triggers the internal hook, then sends `hookEvent.messages.join("\n\n")` via `routeReply()` when any messages were pushed during `/new` or `/reset`. That means user-visible leak behavior is still possible on current `main`.
