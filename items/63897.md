## Summary

Keep open. Current `main` still has the mismatch described in the report: `sessions cleanup --fix-missing` prunes by checking a single resolved transcript path, while the normal session readers use transcript-candidate resolution that explicitly prefers the `sessionId`-derived `.jsonl` before a stale generated `sessionFile`. That means valid transcript-backed sessions can still be flagged as missing on `main`.

## Evidence

- **Cleanup still checks only one resolved path:** `pruneMissingTranscriptEntries()` resolves one `transcriptPath` with `resolveSessionFilePath(...)` and deletes the entry when that single path does not exist; it does not use transcript-candidate scanning.
  - file: [src/commands/sessions-cleanup.ts:152](https://github.com/openclaw/openclaw/blob/5b8bd6371c666a5de43c82305e0315771e8e49d5/src/commands/sessions-cleanup.ts#L152)
  - command: `nl -ba src/commands/sessions-cleanup.ts | sed -n '139,159p'`
  - sha: [5b8bd6371c66](https://github.com/openclaw/openclaw/commit/5b8bd6371c666a5de43c82305e0315771e8e49d5)
- **`resolveSessionFilePath()` does not fall back on missing files:** If `sessionFile` is present and path-normalization succeeds, `resolveSessionFilePath()` returns that path directly. It only falls back to `sessionId` when the stored path is invalid/unsafe, not when it is stale-but-well-formed.
  - file: [src/config/sessions/paths.ts:269](https://github.com/openclaw/openclaw/blob/5b8bd6371c666a5de43c82305e0315771e8e49d5/src/config/sessions/paths.ts#L269)

- **Normal transcript readers use candidate resolution:** `readSessionMessages()` calls `resolveSessionTranscriptCandidates(...)` and reads the first existing file from that candidate list, so the read path is broader than cleanup.
  - file: [src/gateway/session-utils.fs.ts:98](https://github.com/openclaw/openclaw/blob/5b8bd6371c666a5de43c82305e0315771e8e49d5/src/gateway/session-utils.fs.ts#L98)

- **Candidate resolver prefers `sessionId` transcript ahead of stale generated `sessionFile`:** `resolveSessionTranscriptCandidates()` puts the synthesized `sessionId` transcript before a stale generated `sessionFile` candidate when the stored filename belongs to a different generated session id.
  - file: [src/gateway/session-transcript-files.fs.ts:88](https://github.com/openclaw/openclaw/blob/5b8bd6371c666a5de43c82305e0315771e8e49d5/src/gateway/session-transcript-files.fs.ts#L88)
