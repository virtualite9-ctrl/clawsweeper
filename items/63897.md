## Summary

Keep open. Current main still runs `sessions cleanup --fix-missing` through a single `resolveSessionFilePath()` result, which prefers a persisted `sessionFile`; if that stale path is missing, cleanup marks the entry `prune-missing` without checking the existing `<sessionId>.jsonl` fallback. The normal transcript-read path already uses candidate resolution and picks the first existing transcript, and PR #72167 is open to add the missing cleanup fallback, so this issue is not implemented or superseded yet.

## Best Possible Solution

Keep #63897 open until PR #72167 or an equivalent fix lands on main. The fix should make `pruneMissingTranscriptEntries()` use the same safe transcript-candidate resolution semantics as normal readers, preserve custom transcript behavior, and add focused cleanup tests for stale `sessionFile` with a valid `<sessionId>.jsonl` fallback plus the both-missing case.
