## Summary

Keep #8061 open. Current main still supports only a global subagent concurrency cap at `agents.defaults.subagents.maxConcurrent`; there is no typed, validated, documented, or enforced per-agent cap for `agents.list[]`. The issue has a concrete use case, another user confirmed the same problem, and the latest discussion already points to the same unmet core config/runtime gap.

## Best Possible Solution

Keep #8061 open. Add a validated per-target-agent subagent concurrency setting, likely `agents.list[].subagents.maxConcurrent` unless maintainers choose another public shape; update the AgentConfig type, strict Zod schema, generated schema/help, and docs; then enforce it alongside the global subagent lane cap for both native and ACP `sessions_spawn`. Add focused tests proving a capped target serializes while uncapped targets can still run concurrently.
