## Summary

Current `main` does not implement per-agent `maxConcurrent` in `agents.list[]`. Subagent concurrency is still a single global `subagent` lane controlled by `agents.defaults.subagents.maxConcurrent`, and the per-agent `subagents` config surface does not accept `maxConcurrent`.

## Evidence

- **Per-agent agent config omits `subagents.maxConcurrent`:** `AgentConfig.subagents` only defines `allowAgents`, `model`, and `requireAgentId`; there is no per-agent concurrency field in the typed config surface.
  - file: [src/config/types.agents.ts:102](https://github.com/openclaw/openclaw/blob/b2cface9d5fd048a514d82292a5baee7190e0ee3/src/config/types.agents.ts#L102)
  - command: `nl -ba src/config/types.agents.ts | sed -n '96,113p'`
  - sha: [b2cface9d5fd](https://github.com/openclaw/openclaw/commit/b2cface9d5fd048a514d82292a5baee7190e0ee3)
- **Runtime schema rejects the requested field:** `AgentEntrySchema` for `agents.list[]` allows `allowAgents`, `model`, `thinking`, and `requireAgentId` under `subagents`, but no `maxConcurrent`.
  - file: [src/config/zod-schema.agent-runtime.ts:835](https://github.com/openclaw/openclaw/blob/b2cface9d5fd048a514d82292a5baee7190e0ee3/src/config/zod-schema.agent-runtime.ts#L835)
  - command: `nl -ba src/config/zod-schema.agent-runtime.ts | sed -n '832,854p'`
  - sha: [b2cface9d5fd](https://github.com/openclaw/openclaw/commit/b2cface9d5fd048a514d82292a5baee7190e0ee3)
- **Subagent concurrency is resolved only from defaults:** `resolveSubagentMaxConcurrent()` reads only `cfg?.agents?.defaults?.subagents?.maxConcurrent`, then `applyGatewayLaneConcurrency()` applies that value to the single `CommandLane.Subagent` queue.
  - file: [src/config/agent-limits.ts:17](https://github.com/openclaw/openclaw/blob/b2cface9d5fd048a514d82292a5baee7190e0ee3/src/config/agent-limits.ts#L17)

  - sha: [b2cface9d5fd](https://github.com/openclaw/openclaw/commit/b2cface9d5fd048a514d82292a5baee7190e0ee3)
- **Gateway uses one global subagent lane:** Gateway startup sets `CommandLane.Subagent` concurrency from `resolveSubagentMaxConcurrent(cfg)`, confirming one global subagent queue rather than per-agent limits.
  - file: [src/gateway/server-lanes.ts:6](https://github.com/openclaw/openclaw/blob/b2cface9d5fd048a514d82292a5baee7190e0ee3/src/gateway/server-lanes.ts#L6)
