## Summary

Keep #39037 open. Current main still does not extract or propagate Click-to-WhatsApp referral metadata from WhatsApp/Baileys inbound messages. The prior Apr 26 ClawSweeper keep-open review remains accurate against edf40ab6c9440604fe6a74aeaa8d8bc81b5ee16b: contextInfo is available for existing mention/reply extraction, but there is no ctwa/referral/externalAdReply consumer and the WhatsApp inbound pipeline only forwards contact-card structured context into prompt metadata.

## Best Possible Solution

Implement this in the bundled WhatsApp plugin rather than closing it. Add a focused extractor for Baileys CTWA/referral data, including contextInfo.ctwaClid/externalAdReply-style fields and any Cloud-API referral-shaped inbound payload OpenClaw receives. Normalize ctwaClid/ctwa_clid, sourceUrl, sourceType, sourceId, headline/title, and related ad metadata into an untrusted structured context block, and decide whether durable attribution also needs an explicit session or hook metadata field. Add targeted tests for extraction, queueing into WebInboundMessage.untrustedStructuredContext, prompt rendering, and hook/session metadata behavior.
