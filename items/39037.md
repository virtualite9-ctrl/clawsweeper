## Summary

Keep #39037 open. Current main still does not expose Click-to-WhatsApp attribution from WhatsApp inbound messages. The bundled WhatsApp plugin reads Baileys contextInfo for mentions and quoted replies, and it already has a generic untrusted structured context carrier, but the inbound monitor only fills that carrier with contact-card context and there is no CTWA/referral extraction, prompt propagation, hook mapping, or session metadata persistence for ctwaClid.

## Best Possible Solution

Keep the issue open and implement this in the bundled WhatsApp plugin. Extract CTWA attribution from Baileys contextInfo.externalAdReply plus any Cloud-style referral shape OpenClaw receives, normalize fields such as ctwaClid, sourceUrl, sourceType, sourceId, headline/title, and related CTWA payloads as untrusted structured context, then propagate through the agent prompt and any chosen hook/session metadata surface with focused extraction and propagation tests.
