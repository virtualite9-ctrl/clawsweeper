## Summary

Keep open. Current main does not appear to expose Click-to-WhatsApp referral metadata: the WhatsApp inbound path extracts normalized text/media/location/contact/reply/mention context, but there is no ctwa/referral handling and no session metadata field for it. This is not a clawhub cleanup candidate because external hooks receive the already-normalized inbound context, not the raw Baileys message context that would contain CTWA referral data.
