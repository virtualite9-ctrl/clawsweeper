## Summary

Keep #39037 open. Current main still does not extract Click-to-WhatsApp attribution from WhatsApp inbound context, does not populate the existing untrusted structured context carrier with CTWA/referral data, and does not pass ctwaClid through prompt, hook metadata, or session metadata surfaces. The earlier ClawSweeper comment correctly identified the remaining gap, and the current main SHA only adds unrelated dispatch test performance work on top of that state.

## Best Possible Solution

Keep the issue open and implement it in the bundled WhatsApp plugin. Add a focused extractor for CTWA attribution from Baileys contextInfo.externalAdReply and any Cloud API referral-shaped payload OpenClaw receives, normalize fields such as ctwaClid/ctwa_clid, sourceUrl, sourceType, sourceId, headline/title, and related CTWA payloads as untrusted structured context, then propagate through the existing prompt carrier and any intentionally supported hook/session metadata surface with targeted extraction and propagation tests.
