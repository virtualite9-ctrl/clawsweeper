## Summary

Keep #39037 open. Current main still has no WhatsApp CTWA/referral extraction or propagation path: no `ctwa`, `ctwaClid`, `ctwa_clid`, `externalAdReply`, or `referral` implementation exists in the WhatsApp plugin/docs, inbound enrichment only models body/location/contact/reply/media, and the existing structured-context carrier is populated only for contact cards. The prior ClawSweeper keep-open comment remains accurate against current HEAD 4a3030df9efa72c44bab567a4390ac0c4876a73f; the only current HEAD change is an unrelated PowerShell installer fix.

## Best Possible Solution

Keep the issue open and implement it in the bundled WhatsApp plugin. Add a focused CTWA/referral extractor for Baileys `contextInfo.externalAdReply` and any Cloud-API referral-shaped inbound payload OpenClaw receives, normalize fields such as `ctwaClid`/`ctwa_clid`, `sourceUrl`, `sourceType`, `sourceId`, and `headline`/`title`, attach the data as untrusted structured context, and intentionally propagate it through the prompt carrier plus any supported hook/session metadata surface with targeted unit tests.
