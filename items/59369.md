## Summary

Current main already resolves the reported installed-service failure: `openclaw gateway install` now renders the Linux systemd unit to execute the foreground gateway command directly (`gateway --port ...`) instead of putting `openclaw gateway start` inside `ExecStart`. The v2026.4.24 tag contains the same implementation, and tests cover both the direct generated gateway arguments and non-fatal install handling for the reported `systemctl is-enabled unavailable: Failed to connect to bus` probe class.

## Best Possible Solution

Keep the shipped installer behavior that writes systemd units to run the foreground gateway command directly. Leave broader user-systemd bus availability failures to the separate open systemd tracking item rather than reopening this installed-service recursion report.
