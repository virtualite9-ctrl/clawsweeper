## Summary

Keep this issue open. Current main has partial `/new` and `/reset` session-memory preservation plus startup hydration, but it still lacks the requested synthesis workflow, `.session_memory_preserve.json`, full-transcript exports with anchor links, and generated daily continuation package. Related PR #50584 is still open/unmerged in the provided context, and related bug #69943 highlights unresolved safety concerns in the raw persistence path.

## Best Possible Solution

Keep this issue open until maintainers either land a scoped core implementation or explicitly redirect the feature to a ClawHub/context-engine plugin design. A strong path should coordinate with #50584 and #69943: opt-in synthesis, sanitization/filtering before persistence, compact daily continuation packages, canonical daily summary handling, receipt links to full transcripts, timezone-aware filenames, and regression coverage for `/new`/`/reset` preservation plus startup hydration.
