## Summary

Current `main` already implements a broader Slack non-DM suppression path for verbose/tool progress, preserves media-only tool payload delivery, and includes a Slack channel regression test, so this PR’s intended behavior is already covered.

## Evidence

- **Current main suppresses Slack non-DM surfaces before tool summaries/start statuses are emitted:** `dispatch-from-config.ts` now computes `isSlackNonDirectSurface` and derives both `shouldSendToolSummaries` and `shouldSendToolStartStatuses` from `shouldSendVerboseProgressMessages`, which is false for Slack surfaces when `ChatType !== "direct"`. That covers Slack channels directly and is broader than the PR’s old `group|channel` predicate tweak.
