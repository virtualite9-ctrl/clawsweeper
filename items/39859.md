## Summary

Keep open. Current main still does not implement PR #39859's central resolver change: a positive contextTokensOverride returns before known model/config/discovery windows are resolved, so /status and session metadata can still over-report the effective context window.

## Best Possible Solution

Keep #39859 or #39857 open until main lands the narrow resolver behavior: resolve the known model context window before honoring an override, return Math.min(contextTokensOverride, modelTokens) when both values are known, preserve existing override behavior when the model window is unknown, and add regression coverage for large override capping, lower override preservation, and normalized provider-alias lookup. Prefer landing only the product fix and focused tests, not the unrelated CI/baseline churn from the stale PR branch.
