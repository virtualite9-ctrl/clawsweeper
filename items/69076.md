## Summary

Keep PR #69076 open. Current main has a partial `killed` status type, but the behavior described by the PR is not implemented: manual-cancel exec exits still become failed outcomes, notify-on-exit still wakes for manual cancels with output, process kill/remove success still returns failed tool details, and Telegram sent-message metadata remains timestamp-only. The PR discussion also contains a concrete bot review summary saying the patch is scoped and tested, so this is actionable rather than stale cleanup.

## Best Possible Solution

Keep the PR open and review/rebase it against current main. The best path is to land the manual-cancel `killed` semantics with regression coverage for exec outcome classification, notify-on-exit suppression, process poll/log/kill/remove status mapping, and, if kept in scope, the additive Telegram sent-message metadata compatibility tests.
