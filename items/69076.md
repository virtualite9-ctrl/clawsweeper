## Summary

Keep #69076 open. Current main has adjacent manual-cancel suppression, but it does not implement the PR's core behavior: exec manual-cancel outcomes are still `failed`, process `kill`/`remove` still return failed tool details after successful termination requests, and the Telegram sent-message cache still stores timestamp-only records without the proposed metadata.

## Best Possible Solution

Keep the PR open for maintainer review. Current main only partially mitigates one no-output notifyOnExit case; it does not implement the requested killed classification or completed control-action results. The best path is to review or merge a scoped version of #69076 after validating the exec/process result contract and deciding whether the Telegram sent-message metadata belongs in the same change.
