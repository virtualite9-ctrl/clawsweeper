## Summary

Keep PR #69076 open. Current main has partial `killed` vocabulary in the process registry, but it does not implement the PR’s central behavior: manual-cancel exec exits still become failed outcomes, manual-cancel with output still wakes `notifyOnExit`, process kill/remove still report failed tool status, and Telegram sent-message metadata is not present.

## Best Possible Solution

Keep the PR open and have maintainers either land a rebased version of the focused behavior change or extract the same fixes into a maintainer branch: add `killed` as an exec outcome for explicit `manual-cancel`, suppress `notifyOnExit` for killed sessions, make successful kill/remove control actions report completed status, preserve failure semantics for timeouts/spawn/shell errors, and keep the Telegram metadata change only if maintainers still want that diagnostic addition in the same PR.
