## Summary

Current `main` already implements forced skill installs as a staged replacement flow that uses the requested `--version`, rewrites tracked ClawHub metadata, and swaps the existing workspace skill directory instead of merging into it.

## Evidence

- **CLI forwards both flags into the ClawHub skill installer:** `openclaw skills install` passes both `--version` and `--force` to `installSkillFromClawHub`, so the requested version and overwrite intent reach the implementation path directly.
  - file: [src/cli/skills-cli.ts:93](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/src/cli/skills-cli.ts#L93)
  - sha: [6bc0dc8fb671](https://github.com/openclaw/openclaw/commit/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440)
- **Requested version is used for download and origin metadata:** The installer resolves `params.version`, downloads that exact archive version, and then writes `.clawhub/origin.json` with `installedVersion: version` plus a refreshed `installedAt` timestamp.
  - file: [src/agents/skills-clawhub.ts:274](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/src/agents/skills-clawhub.ts#L274)
  - sha: [6bc0dc8fb671](https://github.com/openclaw/openclaw/commit/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440)
- **Workspace lock metadata is refreshed after install:** After a successful install, the workspace ClawHub lockfile entry for the slug is rewritten to the installed version and timestamp.
  - file: [src/agents/skills-clawhub.ts:320](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/src/agents/skills-clawhub.ts#L320)
  - sha: [6bc0dc8fb671](https://github.com/openclaw/openclaw/commit/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440)
- **`--force` uses full staged replacement, not in-place merge:** Forced installs switch to `mode: "update"`, and the shared installer moves the existing target to a backup before renaming the staged extracted directory into place. That replaces the workspace skill directory rather than partially copying over it.
  - file: [src/infra/install-package-dir.ts:277](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/src/infra/install-package-dir.ts#L277)
  - sha: [6bc0dc8fb671](https://github.com/openclaw/openclaw/commit/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440)
- **Docs and tests match the shipped overwrite contract:** Docs state that `install --force` overwrites an existing workspace skill folder, and the CLI command test verifies that a versioned install forwards the requested version to the installer.
  - file: [docs/cli/skills.md:53](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/docs/cli/skills.md#L53)
  - sha: [6bc0dc8fb671](https://github.com/openclaw/openclaw/commit/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440)
