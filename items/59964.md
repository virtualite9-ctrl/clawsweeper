## Summary

Keep #59964 open. Current main has shipped much of the central gateway safety behavior through last-known-good startup/hot-reload recovery, docs, and tests, but it still does not implement the PR's explicit `openclaw config recover` CLI, `recoverConfigFile`, or backup-chain selection API. The PR is not mergeable as-is because the provided review context identifies real implementation bugs, but the remaining manual recovery command is a concrete product/code proposal rather than something already fully implemented on main.

## Best Possible Solution

Do not merge #59964 as-is. The best path is to either ask the author to rebase and narrow the PR to the missing manual recovery CLI/API on top of the shipped last-known-good recovery model, with tests for invalid backup selection and safe notification routing, or have a maintainer explicitly close it if OpenClaw does not want a manual `config recover` surface because `config validate`, `doctor`, and automatic last-known-good recovery are the intended workflow.
