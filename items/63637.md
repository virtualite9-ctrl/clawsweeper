## Summary

Keep open. Current main still has the exact followup drain gap described by #63637: queued runs retain the inbound message id, and `applyReplyThreading` can use `currentMessageId`, but `resolveFollowupDeliveryPayloads` neither accepts nor forwards it and `createFollowupRunner` does not pass `queued.messageId`. The PR remains a small, relevant bug fix for the linked #54456 Matrix `replyToMode: "all"` issue.

## Best Possible Solution

Keep #63637 open and land this small fix or an equivalent maintainer patch: add `currentMessageId` to `resolveFollowupDeliveryPayloads`, pass `queued.messageId` from `createFollowupRunner`, forward it into `applyReplyThreading`, and add regression coverage in `followup-delivery.test.ts` or `followup-runner.test.ts` for a drained followup producing `replyToId` under `replyToMode: "all"`.
