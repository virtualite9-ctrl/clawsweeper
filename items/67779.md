## Summary

Keep this issue open. Current main still has the same plausible Control UI live-update bug described in the report: mixed-case non-default agent deep links are preserved in the browser state, while gateway session storage and chat broadcasts use lowercase canonical session keys, and the WebChat handlers require exact string equality before applying chat or transcript events.

## Best Possible Solution

Normalize or canonicalize Control UI session keys consistently at the URL/settings/session-selection boundary, or compare canonical session keys in the WebChat chat and session-message event handlers. Add a focused regression test where the active UI session is `agent:DB_Dude:main` and gateway `chat` / `session.message` events arrive as `agent:db_dude:main`, proving streaming/final rendering and deferred history reload work without refresh.
