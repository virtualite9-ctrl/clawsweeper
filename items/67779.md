## Summary

Keep this issue open. Current main still has the same plausible WebChat live-update failure path for mixed-case non-default agent sessions: the UI can preserve a raw deep-link key like `agent:DB_Dude:main`, while gateway storage and chat events use the canonical lowercase key `agent:db_dude:main`; the UI live handlers still require exact string equality before applying chat or transcript updates.

## Best Possible Solution

Normalize/canonicalize Control UI session keys at the URL/settings/session-selection boundary, or compare canonical session keys in the WebChat chat, side-result, tool-stream, and `session.message` handlers. Add a focused regression test where active UI state is `agent:DB_Dude:main` and gateway events arrive as `agent:db_dude:main`, proving delta/final rendering and deferred history reload work without manual refresh.
