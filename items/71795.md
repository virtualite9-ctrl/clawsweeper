## Summary

Keep #71795 open. Current main still queues ordinary Mattermost inbound user messages as trusted system events before dispatching the real user body, and the shared prompt builder prepends those queued events as `System:` lines. Discord’s ordinary inbound path sends the user text through `BodyForAgent`/`CommandBody` without the same system-event enqueue, so the report still describes current behavior.

## Best Possible Solution

Keep the issue open. The likely fix belongs in the Mattermost plugin inbound path: stop sending ordinary user messages through `core.system.enqueueSystemEvent`, keep sender/channel metadata in the normal inbound envelope or untrusted context, and add a regression test that simulates a Mattermost post and asserts the dispatched prompt/body does not begin with `System: [timestamp] Mattermost message ...`. Leave reaction/button-click system events intact.
