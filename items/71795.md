## Summary

Keep #71795 open. Current main still enqueues each accepted Mattermost inbound message as a session system event before building the reply prompt, and the shared reply path prepends drained system events as `System:` lines. Discord ordinary message handling does not do the same; it only enqueues recognized Discord platform system-message types and drops them from normal user-message processing.

## Best Possible Solution

Keep the issue open and fix the Mattermost inbound monitor so ordinary user messages do not enqueue a separate `Mattermost message ...` system event for the same prompt. The likely patch belongs in `extensions/mattermost/src/mattermost/monitor.ts`, preserving `BodyForAgent` for the actual user text and adding a regression test that a normal Mattermost post reaches reply dispatch without a prepended `System: Mattermost message ...` line, while true Mattermost interaction/system events continue to use the system-event path.
