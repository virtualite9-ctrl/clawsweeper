## Summary

Keep #72228 open. Current main still has an actionable core/runtime-deps gap: the SDK/logging surface can pull `tslog` through mirrored core files under an external `plugin-runtime-deps` root, but the runtime dependency repair code verifies only bundled plugin package dependencies, not core package dependencies such as `tslog`.

## Best Possible Solution

Keep the issue open and patch the core bundled runtime staging path. The likely fix is to either make mirrored SDK/core logging modules resolve from the package root where root dependencies are installed, or explicitly include and verify required core runtime dependencies such as `tslog` in the external stage root before importing bundled channel/plugin modules. Add a regression test that simulates a packaged external stage with `tslog` missing and proves a bundled channel/plugin using `openclaw/plugin-sdk/runtime-env` loads with a clear repair path instead of crashing.
