## Summary

Current main still does not satisfy the full feature request. `openclaw config validate` and the env/SecretRef write-back protections are implemented, but the remaining requested root-command behavior is not: bare `openclaw config` is still implemented and documented as a guided setup entry point, not help-on-already-configured systems. The prior issue discussion already called out this exact remaining gap, and the current target SHA still matches that assessment.

## Best Possible Solution

Keep #16351 open until a maintainer either implements the remaining root `openclaw config` behavior or explicitly decides the wizard default is intentional. The likely implementation point is `src/cli/config-cli.ts`: when no subcommand and no `--section` are provided on an already configured system, print config command help instead of entering guided setup, while preserving the shipped `config validate` and env/SecretRef-preserving write behavior and updating docs/tests accordingly.
