## Summary

Keep #16351 open. Current main implements two requested pieces, `openclaw config validate` and secret/env-placeholder-preserving config writes, but the central remaining behavior is still missing: bare `openclaw config` is documented and wired to open guided setup rather than showing help on an already configured system.

## Best Possible Solution

Keep #16351 open until a maintainer either implements the remaining root `openclaw config` behavior or explicitly decides the wizard default is intentional. The likely implementation point is `src/cli/config-cli.ts` in the root `config` action: when no subcommand is provided and the system is already configured, display config command help; keep the shipped `config validate`, dry-run/schema safety, and env/SecretRef-preserving write behavior intact, with docs and tests updated to match.
