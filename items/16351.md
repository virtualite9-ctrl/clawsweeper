## Summary

Keep #16351 open. Current main implements `openclaw config validate` and write protections for invalid/secret-resolved config payloads, but the root-command part of the request remains unresolved: bare `openclaw config` still runs the configure wizard, and the public config/configure docs still document that behavior. The existing steipete/Codex issue comment reached the same conclusion, and the relevant current-main source still matches it.

## Best Possible Solution

Keep #16351 open until a maintainer either changes no-subcommand `openclaw config` to show help on already configured systems or explicitly decides the wizard default is intentional. The likely implementation belongs in `src/cli/config-cli.ts` and possibly `src/commands/configure.commands.ts`, with tests for the bare command and docs updates in `docs/cli/config.md` and `docs/cli/configure.md`. Preserve the already-shipped `config validate`, dry-run/schema validation, and env/SecretRef write-back protections.
