## Summary

Keep #21207 open. Current main still has only partial related work: GPT-5 parallel tool-call defaults, context-engine lifecycle APIs, compaction controls, and the experimental `update_plan` tracker. It does not implement native batched read/write/search inputs, AI-callable context block save/load/unload/list tools, or the requested stateful `plan_submit`/`plan_step_done`/`plan_status` enforcement surface.

## Best Possible Solution

Keep #21207 open and split it into concrete maintainer-owned tracks: native or gateway-level batching for core read/write/search surfaces; an explicit context block tool/API or context-engine extension if AI-controlled load/unload is desired; and runner-owned plan state with opt-in strict gating if maintainers want enforceable sequential plans. Treat the reporter's batch-tools skill workaround and current strict-agentic/context-engine work as useful partial evidence, not closure.
