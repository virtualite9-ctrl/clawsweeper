## Summary

Keep open. Current main has more relevant partial work than the original report predated, including OpenAI GPT-5 parallel-tool defaults, context-engine plugin lifecycle, and strict-agentic planning-only retry behavior. Those pieces still do not implement the issue’s requested native batch read/write/search interface, AI-controlled context_save/context_load/context_unload/context_list block tools, or plan_submit/plan_step_done/plan_status stateful step gating.

## Best Possible Solution

Keep #21207 open. The best path is to split or refine the feature into concrete tracks: native/gateway batching for core read/write/search surfaces, an explicit context block tool/API or context-engine extension if AI-controlled load/unload is desired, and runner-owned plan state with opt-in strict gating if maintainers want enforceable sequential plans. The existing batch-tools skill workaround and current strict-agentic/context-engine pieces should be treated as useful partial evidence, not closure.
