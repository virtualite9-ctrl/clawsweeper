## Summary

Keep #21207 open. Current main has adjacent partial work, including provider-level `parallel_tool_calls`, context-engine lifecycle/compaction hooks, strict-agentic planning-only retry handling, and the experimental `update_plan` tracker, but it still does not implement the requested native batched read/write/search inputs, AI-callable named context block tools, or stateful plan-submit/step-done/status enforcement API.

## Best Possible Solution

Keep #21207 open and split it into concrete maintainer-owned tracks: native or gateway-level batching for file/search tools; an explicit AI-callable context block API if named save/load/unload semantics are desired; and runner-owned plan state with opt-in strict gating if enforceable sequential execution is a product goal. Treat `parallel_tool_calls`, context-engine compaction, and `update_plan` as useful groundwork, not closure.
