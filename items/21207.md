## Summary

Keep #21207 open. Current main still has adjacent groundwork, including provider `parallel_tool_calls`, context-engine compaction/lifecycle hooks, `update_plan`, and strict-agentic plan-only retry handling, but it does not implement the requested native batched file/search APIs, AI-callable named context block save/load/unload/list tools, or submitted-plan step enforcement API.

## Best Possible Solution

Keep #21207 open and split it into concrete tracks: native or gateway-level batching for file/search tools, an explicit AI-callable named context block API if save/load/unload semantics are a product goal, and runner-owned opt-in plan state with strict step gating if enforceable sequential execution is desired. Treat existing `parallel_tool_calls`, context-engine compaction/pruning, `update_plan`, and strict-agentic plan-only retries as groundwork rather than closure.
