## Summary

Keep open. Current `main` does not implement `--preserve-intermediate` or preserved streaming history in the TUI, and the PR still carries unresolved behavior questions against current empty-output suppression semantics, so none of the allowed close reasons apply with high confidence.

## Evidence

- **Current CLI does not ship the flag:** On checked-out `main`, `registerTuiCli` defines the TUI options set and forwards them to `runTui`, but there is no `--preserve-intermediate` flag and no `preserveIntermediate` option passed through.
  - file: [src/cli/tui-cli.ts:23](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/src/cli/tui-cli.ts#L23)
  - command: `nl -ba src/cli/tui-cli.ts | sed -n '1,220p'`
  - sha: [c070509b7f8f](https://github.com/openclaw/openclaw/commit/c070509b7f8fdbe479e5d829643782c272e30e83)
- **Current chat log has no preserve-history mode:** `ChatLog` on `main` keeps one streaming assistant component per run: `startAssistant` reuses the tracked component, `updateAssistant` mutates it, `finalizeAssistant` mutates then clears it, and `dropAssistant` removes the tracked component. There is no preserved-intermediate branch in the implementation.
  - file: [src/tui/components/chat-log.ts:186](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/src/tui/components/chat-log.ts#L186)
  - command: `nl -ba src/tui/components/chat-log.ts | sed -n '1,220p'`
  - sha: [c070509b7f8f](https://github.com/openclaw/openclaw/commit/c070509b7f8fdbe479e5d829643782c272e30e83)
- **Current event-handler contract drops empty finals:** The TUI event handler explicitly suppresses empty final output by calling `chatLog.dropAssistant(evt.runId)` when the final event has no message or resolves to `(no output)`. That means preserved partial chunks need careful handling to avoid leaving phantom assistant replies behind.
  - file: [src/tui/tui-event-handlers.ts:369](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/src/tui/tui-event-handlers.ts#L369)
  - command: `nl -ba src/tui/tui-event-handlers.ts | sed -n '280,390p'`

- **PR patch intentionally appends preserved chunks and keeps earlier ones:** The public patch for PR #54982 adds `preserveIntermediate`, makes `updateAssistant` and `finalizeAssistant` append via `startAssistant`, and annotates `dropAssistant` to keep earlier preserved fragments. That matches the open review concern that empty-final suppression would leave stale partial output visible.
