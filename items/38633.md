## Summary

Keep #38633 open. Current main still documents `pidsLimit: 256` as a Docker sandbox config example while runtime resolution leaves `pidsLimit` unset unless configured, and Docker only emits `--pids-limit` for a positive configured value. The related fix/clarification PRs are closed unmerged, and the broader #38604 security issue does not fully replace this narrower docs mismatch.

## Best Possible Solution

Keep #38633 open until maintainers either clarify the docs that `pidsLimit: 256` is an explicit example and remains unset unless configured, or intentionally add a runtime default in `resolveSandboxDockerConfig` with matching schema/type wording, tests, and docs. Coordinate any runtime default with #38604; a docs-only fix can remain narrow.
