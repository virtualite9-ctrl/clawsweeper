## Summary

Keep #38633 open. Current main still has the reported docs/runtime mismatch: the user-facing sandbox config example shows `pidsLimit: 256`, while the runtime resolver applies no built-in `pidsLimit` and Docker only receives `--pids-limit` when an explicit positive value is configured. The related fix/clarification PRs #38621 and #38914 are closed unmerged, and #38604 remains a broader open security/default discussion rather than a shipped fix for this narrower docs issue.

## Best Possible Solution

Keep #38633 open until maintainers either land a narrow docs clarification that `pidsLimit` is an explicit example value and remains unset unless configured, or add a real default in `resolveSandboxDockerConfig` with matching tests and docs. Coordinate any runtime default with the broader #38604 security/default discussion; a docs-only fix can stay narrower and should also keep the schema/type wording aligned.
