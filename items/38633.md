## Summary

Keep #38633 open. Current main still shows `pidsLimit: 256` in the user-facing sandbox config example, while the runtime resolver leaves `pidsLimit` unset unless explicitly configured and Docker only emits `--pids-limit` for a positive configured value. The related fix/clarification PRs in the provided context are closed unmerged, and the broader #38604 discussion does not replace this narrower docs mismatch.

## Best Possible Solution

Keep #38633 open until maintainers either clarify the docs that `pidsLimit: 256` is an explicit example and remains unset unless configured, or intentionally add a runtime default in `resolveSandboxDockerConfig` with matching tests, schema/type wording, and docs. Coordinate any runtime default with the broader #38604 security/default discussion; a docs-only fix can remain narrower.
