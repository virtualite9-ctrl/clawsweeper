## Summary

Current main still has the reported documentation/runtime mismatch: the docs show `pidsLimit: 256` in the sandbox config example, while the runtime only inherits `pidsLimit` from explicit agent/global config and emits Docker `--pids-limit` only when a positive value is configured. Related PRs #38621 and #38914 are closed unmerged, and #38604 remains the broader open security/default discussion rather than a shipped fix.

## Best Possible Solution

Keep #38633 open until maintainers either land a narrow docs clarification that `pidsLimit` is unset unless explicitly configured, or add a real default in `resolveSandboxDockerConfig` with matching tests and docs, coordinated with the broader #38604 security/default discussion.
