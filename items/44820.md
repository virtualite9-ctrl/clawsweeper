## Summary

Keep #44820 open. Current main has partial session archive cleanup, but it does not fully cover the reported accumulation of `.reset.*`, `.deleted.*`, `.tmp`, and `.bak.*` files; the exact proactive cleanup work is still represented by open, unmerged PR #47635.

## Best Possible Solution

Keep #44820 as the canonical user report until PR #47635 or a narrower equivalent is merged. The best fix belongs in core session/gateway maintenance: run bounded, best-effort cleanup at startup and on a periodic timer for old `.deleted.*` and `.reset.*` archives, age-gated orphan atomic-write `.tmp` files, and excess `.bak.*` session backups, with regression tests proving active transcripts and custom/external paths are preserved.
