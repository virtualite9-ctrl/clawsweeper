## Summary

Keep #72101 open. Current main still contains both user-facing behaviors the PR addresses: `sessions_spawn` rejects ACP-only `streamTo` for `runtime="subagent"`, and it forwards `model: "default"` as an explicit model override instead of treating it as unset. The related streamTo-only PR #68397 is closed unmerged and did not cover the model-default normalization, so it does not supersede this PR.

## Best Possible Solution

Keep #72101 open, or land a consolidated agents-tool patch that preserves ACP behavior, ignores ACP-only fields for `runtime="subagent"`, normalizes `model="default"`/empty model input to no override, and updates `src/agents/tools/sessions-spawn-tool.test.ts` plus any shared normalization tests as needed.
