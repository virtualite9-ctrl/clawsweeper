## Summary

Keep #72101 open. Current main still has the reported sessions_spawn behavior: it rejects runtime="subagent" calls when streamTo="parent" is present and forwards model:"default" as a literal model override. The PR is a focused candidate fix for #72078 and is not fully superseded by #68397, which covers ACP-only streamTo/resumeSessionId cleanup but does not address the model:"default" sentinel.

## Best Possible Solution

Keep #72101 open for maintainer review. The best path is to preserve its model:"default" normalization for sessions_spawn, then either merge it with or rebase it against the chosen streamTo/resumeSessionId cleanup so the final patch has one clear ACP-only field contract and regression tests for streamTo plus model:"default".
