## Summary

Keep #72101 open. Current main at e672b614 still has both behaviors the PR fixes: `sessions_spawn` rejects `runtime="subagent"` when ACP-only `streamTo:"parent"` is present, and forwards `model:"default"` as a literal model override. The related #68397 is closed unmerged and does not cover the model-default normalization, so it is not a live canonical replacement.

## Best Possible Solution

Keep #72101 open, or land an equivalent agents-tool patch that ignores ACP-only `streamTo` for `runtime="subagent"`, normalizes empty or `default` model input to no override through shared helper logic, preserves ACP behavior, and updates the `sessions_spawn` and `session_status` regression tests. Maintainers should decide whether to include the adjacent `resumeSessionId` silent-drop behavior from #68397 before merging.
