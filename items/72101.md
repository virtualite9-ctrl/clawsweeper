## Summary

Keep #72101 open. Current main still has both behaviors the PR addresses: `sessions_spawn` hard-errors when `runtime="subagent"` receives ACP-only `streamTo`, and it forwards `model: "default"` as a literal model override instead of treating it as unset. The linked issue #72078 was closed as superseded by this PR, and prior PR #68397 is closed/unmerged and did not cover the model-default normalization.

## Best Possible Solution

Keep #72101 open for maintainer review, or land an equivalent consolidated patch that ignores ACP-only fields for native subagent spawns, preserves ACP behavior, normalizes empty/`default` model input to no override, and updates the focused `sessions_spawn` regression tests. If maintainers want the broader ACP-only cleanup, fold in `resumeSessionId` handling from the closed #68397 discussion rather than closing this PR as obsolete.
