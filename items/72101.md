## Summary

Keep this PR open. Current main already ignores ACP-only streamTo for subagent spawns, but it still forwards model:"default" from sessions_spawn as a literal model override, so the PR still contains useful remaining work. The PR is not implemented on main or fully superseded by the closed #68397 because #68397 only covered ACP-only field stripping, while this PR also fixes default-model normalization.

## Best Possible Solution

Keep this PR open or land an equivalent focused agents-tool patch that normalizes empty/default sessions_spawn model input to no explicit override, shares the behavior with session_status where appropriate, preserves current ACP behavior, and adds a regression test for model:"default" on sessions_spawn. The streamTo portion can be treated as already handled on current main unless the rebase shows a remaining edge case.
