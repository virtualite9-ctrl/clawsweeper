## Summary

Keep open. Current `main` at `55389f4be29359e2dd57ebaf548ba4a70331a445` does not already implement the Discord synthetic thread-starter drop, and the PR as proposed still has a real edge-case risk: forum/media starter posts legitimately use the same `message.id === thread id` shape while parent-type lookup can be unknown.
