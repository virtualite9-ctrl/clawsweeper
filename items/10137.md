## Summary

Keep #10137 open. Current main still only partially covers the report: generic `api: "openai-responses"` requests already use array `input` and `stream: true`, but there is still no stable compatibility layer for stricter Responses-compatible proxies to force/default top-level `instructions` or omit `max_output_tokens`. #10140 is the closed duplicate; #10137 remains the canonical tracker.

## Best Possible Solution

Keep #10137 open as the canonical tracker. A good fix belongs in the Responses transport/payload policy and documented provider/model compatibility config: add opt-in stable controls for generic `api: "openai-responses"` proxy compatibility, especially fallback/default top-level `instructions` and optional omission of `max_output_tokens`, with focused regression tests and docs updates. Treat #10140 as the closed duplicate copy.
