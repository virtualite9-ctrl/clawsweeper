## Summary

Keep #10137 open. Current main still only partially satisfies the requested Responses proxy compatibility layer: `input` is already an array and `stream` is hardcoded true, but generic `api: "openai-responses"` providers still do not have stable config for fallback top-level `instructions` or omitting `max_output_tokens`. The provided discussion also treats #10140 as the closed duplicate and #10137 as the canonical tracker.

## Best Possible Solution

Keep #10137 open as the canonical tracker and treat #10140 as the closed duplicate. Add opt-in Responses proxy compatibility controls in the transport/payload policy and stable model/provider config, especially generic fallback top-level `instructions` and optional omission of `max_output_tokens`, with focused regression tests and docs.
