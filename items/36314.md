## Summary

Keep #36314 open. Current main has meaningful pieces of Gateway robustness, including global process error logging/exit paths, systemd restart limits for managed installs, last-known-good config recovery, liveness/readiness probes, and channel-scoped health monitor restart caps. It still does not implement the issue’s central request: a first-class Gateway process watchdog with configurable restart-window/auto-stop/notification behavior, a user-configurable global error recovery strategy map, richer /health process counters, or a deliberate degraded-startup mode.

## Best Possible Solution

Keep the umbrella issue open or split it into focused maintainer-owned tasks: define a first-class gateway.watchdog config/schema/docs surface, specify process restart-loop detection plus auto-stop/notification behavior above platform supervisors, decide whether configurable global error recovery strategies are wanted, enrich the Gateway health RPC/probe story with safe process/error counters, and design degraded startup behavior that composes with last-known-good config recovery and existing channel health monitoring.
