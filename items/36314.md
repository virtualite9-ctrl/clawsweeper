## Summary

Keep #36314 open. Current main has several robustness pieces, but it still does not implement the requested cohesive Gateway watchdog, user-configurable error recovery strategy map, richer /health counters, or deliberate degraded-startup mode.

## Best Possible Solution

Keep the umbrella issue open or split it into focused maintainer-owned tasks: define a first-class gateway.watchdog config/schema/docs surface, specify process restart-loop detection and notification/auto-stop behavior above platform supervisors, decide whether user-configurable global recovery strategies are wanted, enrich privacy-safe Gateway health metrics, and design degraded startup behavior that composes with last-known-good config and existing channel health recovery.
