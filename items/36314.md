## Summary

Keep #36314 open. Current main has useful robustness pieces, but the requested first-class Gateway watchdog, configurable recovery strategy map, richer health counters, and deliberate degraded startup mode are still not implemented as a cohesive Gateway config/schema/docs surface.

## Best Possible Solution

Keep the umbrella issue open or split it into focused maintainer-owned tasks: define a first-class gateway.watchdog config/schema/docs surface, specify process restart-loop detection and notification behavior above platform supervisors, decide whether user-configurable global recovery strategies are wanted, enrich privacy-safe Gateway health metrics, and design degraded startup behavior that composes with last-known-good config and existing channel health recovery.
