## Summary

Keep open. Current `main` already contains the PR's `chat.ts` canonical-session-key changes, but the midnight log-rotation fix is not implemented on `main`. Long-lived child loggers still write through a transport that is bound to the original `settings.file`, so the underlying bug described in #37388 remains plausible for processes that keep a logger across midnight.

## Evidence

- **Logger transport still writes to the original file path:** `buildLogger()` creates the transport once, initializes byte tracking from `settings.file`, and every write path still uses `settings.file` directly (`appendLogLine(settings.file, ...)`, size-cap warning text, stderr warning). There is no per-write recomputation of the rolling filename on current `main`.
  - file: [src/logging/logger.ts:168](https://github.com/openclaw/openclaw/blob/55389f4be29359e2dd57ebaf548ba4a70331a445/src/logging/logger.ts#L168)

  - sha: [55389f4be293](https://github.com/openclaw/openclaw/commit/55389f4be29359e2dd57ebaf548ba4a70331a445)

If the day changes later, code that keeps the child logger never calls `getLogger()` again and therefore never picks up a rebuilt base transport.
