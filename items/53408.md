## Summary

Keep this issue open. Current main still has targeted malformed tool-argument repair for Kimi Anthropic Messages and OpenAI-completions, but the reported custom/local `copilot-api` Anthropic Messages path remains outside that gate and lacks regression coverage for large fragmented tool arguments after long conversations.

## Best Possible Solution

Keep this issue open. Add a focused regression or live repro for a custom/local `anthropic-messages` provider streaming large fragmented `input_json_delta` tool arguments after long context. If confirmed, extend guarded argument repair or add explicit parse-failure diagnostics before missing/empty arguments reach tool validation; treat #53591 and #53735 as related Kimi-specific fixes, not the canonical resolution for this report.
