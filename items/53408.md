## Summary

Keep open. Current main has targeted streamed tool-argument repair for Kimi on Anthropic Messages and for OpenAI-completions transports, but I did not find high-confidence evidence that it fixes the reported custom/local copilot-api Anthropic Messages long-session argument loss. The related Kimi reports #53591/#53735 are useful context, but they are not the same remaining reproduction.

## Best Possible Solution

Keep this issue open and add a focused regression or live repro for a custom/local Anthropic Messages provider streaming large fragmented `input_json_delta` tool arguments after long context. If it reproduces, extend the argument repair or diagnostics to the custom Anthropic proxy path, or fail explicitly instead of allowing empty/missing tool args to reach validation. Treat #53591/#53735 as related Kimi-specific evidence, not as the canonical fix for this report.
