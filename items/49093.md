## Summary

Keep open. Current `main` still leaves memory remote fetches on the default strict SSRF path, which explicitly ignores env proxies unless a caller opts into `TRUSTED_ENV_PROXY`. OpenClaw has a newer, safer auto-upgrade pattern for other provider HTTP helpers, but that logic has not been applied to `src/memory-host-sdk/host/remote-http.ts`, so this PR's core request is not implemented on `main` yet.

## Evidence

That means the memory embedding/batch HTTP path still relies on the guard's default mode.
