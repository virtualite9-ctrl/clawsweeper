## Summary

Current main does not implement WhatsApp outbound disappearing-message expiration. The issue is concrete, recent, and belongs in the bundled WhatsApp plugin rather than being stale or duplicative: the existing config schema/types are strict and do not expose `disappearingMessagesSeconds`, the resolved account model has no expiration field, and the send path never passes Baileys `ephemeralExpiration` for text/media/poll sends. The provided related items cover quote threading, mentions, and unrelated outbound delivery, not disappearing-message expiration.

## Best Possible Solution

Keep the issue open and implement it inside the bundled `@openclaw/whatsapp` plugin: add typed and zod-validated `channels.whatsapp.disappearingMessagesSeconds` plus account overrides, resolve the effective positive value with account precedence, propagate it through `ActiveWebSendOptions`/poll send options into Baileys `MiscMessageGenerationOptions.ephemeralExpiration`, and add focused tests/docs for unset, channel-level, account override, text, quoted, media, voice-note split text, and poll paths.

## Risks / Open Questions

- Implementation needs one shared resolver so channel-level, account-level, and unset behavior stay consistent across text, quoted replies, media, split voice-note text, polls, and tool/channel send payloads.
- Generated config metadata/docs will need to stay aligned with the schema/type changes; otherwise the new config key may validate but remain undiscoverable.
