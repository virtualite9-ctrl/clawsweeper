## Summary

Current main still does not implement WhatsApp disappearing-message expiration for outbound replies. The WhatsApp config schema rejects the proposed keys, resolved account state has no effective expiration value, and outbound text/media/poll paths never pass Baileys ephemeralExpiration.

## Best Possible Solution

Keep this issue open and implement it in the bundled WhatsApp plugin. Add channel-level and account-level disappearingMessagesSeconds to config types, strict Zod schema, metadata/help, and docs; resolve a positive effective value with account override over channel default and 0/null/omitted meaning no OpenClaw override; thread that value through ActiveWebSendOptions and poll options; merge it into Baileys MiscMessageGenerationOptions for normal text, quoted replies, media, voice-note companion text, payload/chunked/fallback sends, and polls; add focused tests for configured, account override, unset, quoted, media/voice-note, payload/chunked, and poll paths.
