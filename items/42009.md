## Summary

Keep this PR open. Current main still gives the implicit agent main-session key precedence over `--to` when `--agent` is supplied, so the reported `openclaw agent --agent <id> --to <target>` routing bug remains. The PR is a plausible focused fix, but it needs to be rebased onto the current implementation location and reviewed with the existing session-id guardrails preserved.

## Best Possible Solution

Rebase this PR onto current main, apply the `--agent + --to` session-key derivation in `src/agents/command/session.ts`, keep explicit `--session-key` and `--session-id` precedence intact, and land focused regression coverage for the selected-agent peer key plus the reviewer-identified guardrails.
