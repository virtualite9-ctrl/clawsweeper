## Summary

Keep this PR open. Current main still lacks the proposed Control UI watchdog/recovery path, while the UI still treats any non-null `chatRunId` as busy and leaves queued messages blocked until a terminal chat event, reconnect/gap recovery, or manual clear path resets that state. The same-author linked bug #42213 is still open, so this PR remains the focused implementation candidate rather than conservative cleanup.

## Best Possible Solution

Keep this PR open as the implementation candidate for #42213. Rebase it onto current main, address the bot review comments, retain the `agent.wait`-backed recovery semantics, add or preserve regression coverage for stale-finished recovery, still-active timeout retry, and cancellation/reset, then review and land it or close it together with #42213 if maintainers choose a different Control UI fix.
