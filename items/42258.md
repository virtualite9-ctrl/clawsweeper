## Summary

Keep this PR open. Current main still lacks the Control UI `agent.wait` watchdog/recovery path, while `chatRunId` remains the UI busy-state gate that blocks queued sends until a terminal chat event, reconnect, or manual cleanup path clears it. The same-author linked bug #42213 is still open, and this PR is the focused implementation candidate rather than obsolete cleanup.

## Best Possible Solution

Keep this PR open as the implementation candidate for #42213. Rebase it onto current main, address the bot review comments, preserve the `agent.wait`-backed recovery semantics, keep regression coverage for stale-finished recovery, still-active timeout retry, and cancellation/reset, then review and land it or close it together with #42213 if maintainers choose a different Control UI fix.
