## Summary

Keep open. Current main still lacks the client-side watchdog/agent.wait recovery proposed here: Control UI chat remains busy while chatRunId is set, queued messages only drain after that busy state clears, and current UI code clears chatRunId through terminal chat events or reconnect/gap recovery rather than probing the already-available backend wait contract. This PR is still a focused implementation candidate for the still-open linked bug #42213, but it needs rebase/review work before landing.

## Best Possible Solution

Keep this PR open as the implementation candidate for #42213. The maintainer path is to rebase onto current main, address the Greptile review comments, preserve the client-side watchdog against the existing agent.wait chat-run contract, keep regression coverage for stale-finished recovery, still-active timeout retry, and cancellation/reset, then review and land or close together with #42213 if maintainers choose a different fix.
