## Summary

Keep open. Current `main` does not implement the PR's maintenance-window feature set: `CronConfig` has no `cron.maintenance` surface, cron status/manual-run types have no maintenance diagnostics or `maintenance-blocked` result, the gateway reload plan has no special cron+heartbeat restart rule for maintenance changes, and the cron docs do not document maintenance-window role isolation. The PR discussion also already surfaced multiple concrete correctness concerns, and the timeline shows an open cross-reference from issue #65774, so there is not strong evidence for a conservative cleanup close.
