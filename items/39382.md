## Summary

Current `main` still treats an omitted `match.accountId` as the default account, and that behavior is explicitly codified in routing tests. Main now documents the behavior and adds `openclaw doctor` warnings for missing default-account/binding coverage, but I did not find evidence that the Feishu runtime now warns on this mismatch or that the underlying fallback semantics changed. That is not strong enough to close this issue as implemented.

## Evidence
