## Summary

Current `main` does not show a Xiaomi-specific fix for MiMo reasoning-only completions. The shared OpenAI-completions transport still routes `reasoning_content`/`reasoning`/`reasoning_text` into hidden thinking blocks rather than visible text, and the bundled Xiaomi plugin only provides catalog/auth wiring with no transport override or default thinking-disable behavior for `mimo-v2-pro` / `mimo-v2-omni`. I do not see strong evidence that issue #60261 is implemented on `main`.
