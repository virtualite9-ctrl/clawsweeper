## Summary

Current main still appears to have the reported `/v1/chat/completions` SSE corruption path. The streaming handler writes assistant chunks from `resolveAssistantStreamDeltaText(evt)` and skips empty content, while that helper prefers `data.delta` over `data.text`. Upstream assistant events can still emit `replace: true` with corrected full `text` and an empty `delta`, including the final-message replacement path. The two linked fix PRs were closed unmerged, and the regression test described there is not present on main.

## Best Possible Solution

Keep the issue open and land a focused gateway fix: make `/v1/chat/completions` streaming derive outbound SSE deltas from the authoritative `evt.data.text` snapshot when present, track already-sent text to avoid duplication, honor `replace: true`, and add a regression test that emits `text: "Done.", delta: "", replace: true` after prior streamed content.
