## Summary

Current `main` already routes cron error runs through `failureAlert` threshold/cooldown handling and into gateway delivery, with targeted tests for repeated failures, explicit announce targets, and Discord/Slack-style `channel:` destinations. The remaining `deliveryStatus` field still reflects the job's primary delivery path rather than the separate failure-alert send, but the reported "failureAlert never fires" behavior is implemented on `main`.

## Evidence
