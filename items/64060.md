## Summary

Keep open. Current `main` still drops `model` when cron input uses `text` without an explicit kind, and the submitted PR only fixes the nested `payload` path described in the diff/context, not the parallel top-level inference path called out in review.

## Evidence

- **Nested payload inference on current main still prefers `systemEvent` for `text`:** `coercePayload()` on current `main` sets `kind = "systemEvent"` whenever `text` is present and no explicit kind is set; it does not check `hasAgentTurnPayloadHint(next)` before doing so, so agent-turn-only fields like `model` remain on a `systemEvent` payload and are later pruned.
  - file: [src/cron/normalize.ts:157](https://github.com/openclaw/openclaw/blob/5b8bd6371c666a5de43c82305e0315771e8e49d5/src/cron/normalize.ts#L157)
  - command: `nl -ba src/cron/normalize.ts | sed -n '150,175p'`
  - sha: [5b8bd6371c66](https://github.com/openclaw/openclaw/commit/5b8bd6371c666a5de43c82305e0315771e8e49d5)
- **Top-level legacy inference on current main has the same bug:** `inferTopLevelPayload()` still returns `{ kind: "systemEvent", text }` for top-level `text` input before checking `hasAgentTurnPayloadHint(next)`, so a top-level `{ text, model }` patch/create shape is still misclassified.
  - file: [src/cron/normalize.ts:291](https://github.com/openclaw/openclaw/blob/5b8bd6371c666a5de43c82305e0315771e8e49d5/src/cron/normalize.ts#L291)
  - command: `nl -ba src/cron/normalize.ts | sed -n '292,312p'`
  - sha: [5b8bd6371c66](https://github.com/openclaw/openclaw/commit/5b8bd6371c666a5de43c82305e0315771e8e49d5)
- **Misclassified top-level payloads lose `model` on current main:** Normalization only copies top-level agent-turn fields into `payload` when `payload.kind === "agentTurn"` and then unconditionally strips the legacy top-level fields. A top-level `{ text, model }` input inferred as `systemEvent` therefore drops `model` silently.
  - file: [src/cron/normalize.ts:346](https://github.com/openclaw/openclaw/blob/5b8bd6371c666a5de43c82305e0315771e8e49d5/src/cron/normalize.ts#L346)

- **Current tests do not prove the reported `text` + `model` case is fixed on main:** Current tests cover model-only inference (`normalizeCronJobPatch`) and explicit `systemEvent` pruning, but they do not include a regression test for implicit `text` + `model` promotion to `agentTurn` on current `main`.
