## Summary

Keep open. Current main still has the nested `payload.text` + `model` normalization bug this PR targets, and the submitted PR only fixes that nested path while the PR discussion correctly identifies the same remaining bug in the flat/top-level payload path.

## Best Possible Solution

Keep this PR open and request a small update: apply the same `text` plus `hasAgentTurnPayloadHint()` promotion in both `coercePayload()` and `inferTopLevelPayload()`, with regression tests for nested `payload: { text, model }` and flat/top-level `{ text, model }` create/patch inputs. After that, review the targeted cron normalization tests and land if green.

## Evidence
