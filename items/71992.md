## Summary

Current main has not applied the requested Control UI webchat fix. The `final` and `aborted` chat-event paths still write to `chatMessages` before clearing `chatStream`, while the renderer can include both history messages and the active stream. The linked fix PR #72004 is open but not merged, and issue comments add possible backend transcript-duplication evidence that should stay triaged with #69208 rather than be auto-closed.

## Best Possible Solution

Keep #71992 open until a focused PR such as #72004 or an equivalent maintainer change lands on main. The fix should clear `chatStream`, `chatRunId`, and `chatStreamStartedAt` before any `chatMessages` write in both `final` and `aborted` handlers, add an ordering regression test, and coordinate any confirmed persisted-transcript duplication with #69208 rather than closing this report prematurely.
