## Summary

Current main has not implemented the requested Control UI webchat fix. The final/aborted chat event paths still append to `chatMessages` while `chatStream` is live, and the current tests preserve that final-state-only behavior. The report is recent, has additional reproduction on v2026.4.24, and PR #72004 is still open rather than merged.

## Best Possible Solution

Keep #71992 open until #72004 or an equivalent patch lands on main. The concrete fix should snapshot any fallback stream text, clear `chatStream`/`chatRunId`/`chatStreamStartedAt` before mutating `chatMessages` in both `final` and `aborted`, and add a regression test that observes `chatStream` is already null at the moment `chatMessages` changes. Coordinate separate persisted-transcript duplicate cases under #69208 rather than closing this webchat race report prematurely.
