## Summary

Keep open. Current main has partial mitigation for provider prefix drift, but the specific `/model` round-trip UX is still not fully fixed: the no-argument `/model` output still renders catalog `id` values without provider qualification, and typed `/model <value>` is sent to gateway validation as entered. The related PR #47249 targeted this exact issue but is closed unmerged, so it does not supersede the open issue.

## Best Possible Solution

Keep the issue open and land a focused UI/DX fix: render `/model` current and available values as canonical `provider/model` refs or otherwise guarantee every displayed value is accepted by `sessions.patch`. The fix should update `ui/src/ui/chat/slash-command-executor.ts` and regression tests for bare ids, duplicate ids across providers, and provider-native slash-containing ids such as OpenRouter/NVIDIA-style models.

## Evidence
