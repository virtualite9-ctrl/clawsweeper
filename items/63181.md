## Summary

Current main still has the prompt-contract mismatch reported here: the system prompt emits a section titled `Current Date & Time` but only includes `Time zone: ...`. Main has added a separate `session_status` hint elsewhere in the prompt, but that does not rename the section or make the limitation explicit inside the misleading section. Related date/time items cover startup/reset timestamp injection and prompt-cache stability, not this narrower label mismatch.

## Best Possible Solution

Keep this issue open and fix the narrow prompt-contract mismatch directly. Preserve cache-stable timezone-only behavior, but rename the system prompt section to `Time Zone` or place an explicit `Use session_status for current date/time` limitation adjacent to the timezone line, then update `src/agents/system-prompt.test.ts`, `docs/concepts/system-prompt.md`, `docs/date-time.md`, and stale `docs/concepts/timezone.md` wording so implementation, tests, and docs agree.
