## Summary

Current main has the related `sessionFile` rotation fix, but it still does not archive the prior transcript after heartbeat or isolated cron rotation. This PR contains unique remaining cleanup work, although its cron reference-set concern should be addressed before merge.

## Best Possible Solution

Keep this PR open for maintainer review and revision. The best path is to keep the shipped `sessionFile` rotation behavior, then add a narrow archive-on-rotation flow for heartbeat and isolated cron: capture the prior entry before overwriting, persist the new entry, archive the prior transcript with `reason: "reset"` and `restrictToStoreDir: true`, and exclude historical cron run rows from the active-reference check so common isolated cron rotations actually archive.
