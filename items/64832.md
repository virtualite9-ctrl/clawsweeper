## Summary

Keep this PR open. Current main has the isolated-session `sessionFile` rotation fix, but it still does not archive the prior transcript after heartbeat or cron rotation; the PR contains unique remaining cleanup work and should be revised rather than closed.

## Best Possible Solution

Keep the PR open for maintainer review and revision. The best path is to preserve the shipped `sessionFile` rotation behavior, then add a narrow archive-on-rotation flow for heartbeat and isolated cron: snapshot the prior entry before overwriting, persist the new entry, archive the prior transcript with `reason: "reset"`, and exclude historical cron `...:run:<id>` rows from active-reference checks.
