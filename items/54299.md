## Summary

Keep open. Current `main` still conditionally omits `client_secret` for Gemini OAuth when only a client ID is configured, and it still hardcodes the localhost redirect URI to port 8085. The port-collision impact is partially mitigated because the login flow now falls back to manual copy/paste mode on listen errors, but that does not fully resolve the issue as filed.

## Evidence

- **Gemini token exchange still omits client_secret when absent:** `exchangeCodeForTokens()` builds the token POST body without `client_secret` and only adds it inside `if (clientSecret)`, so current main still has the behavior the issue reports.
