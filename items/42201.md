## Summary

Keep this PR open. Current main still rejects unknown config keys through strict schema validation and invalid-config startup handling, and the PR remains a focused implementation candidate with substantive review follow-up. It overlaps with #40268 and #47654, but the provided discussion does not show a maintainer decision making either related item the canonical replacement.

## Best Possible Solution

Keep this PR open for maintainer judgment. The best path is to rebase/refit this implementation or explicitly consolidate it with #47654 and #40268, preserving strict validation for invalid known and legacy config while adding tested tolerant read/startup behavior for unknown keys. Before merge, run targeted config validation tests and review or regenerate .secrets.baseline.
