## Summary

Keep PR #72209 open. Current main at 1b1eea238c85e9260cea88dcc2e8c32dd5973202 still does not implement passive session recovery checkpoints, task-ledger/recovery-bundle storage, or system-prompt recovery discipline. The PR is recent and proposes meaningful missing core agent behavior; the Greptile P2 findings are normal review work, not a conservative cleanup close reason.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to refine the PR implementation: address the redaction and synchronous-I/O review comments, keep recovery checkpoints passive and informational, ensure approvals/auth/permissions are never inherited, and preserve focused tests for update_plan, sessions_yield, sessions_spawn, and prompt composition. Maintainers should also decide whether this new recovery-state surface needs public docs.

- Greptile's P2 comments remain relevant: broad secret redaction may corrupt natural-language recovery context, and synchronous checkpoint I/O may block hot tool paths.
- The design should align with existing restart recovery, subagent orphan recovery, reset, compaction, and session snapshot concepts to avoid overlapping recovery semantics.
