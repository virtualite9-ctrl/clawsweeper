## Summary

Keep PR #72209 open. Current main at 4a3030df9efa72c44bab567a4390ac0c4876a73f still does not implement passive session recovery checkpoints, recovery bundle storage, or system-prompt recovery discipline. The adjacent restart/orphan recovery, compaction checkpoints, and background task ledger are different surfaces and do not replace the PR's informational-only recovery context. The PR is recent, contributor-authored, not protected, and has review feedback that should be handled in normal maintainer review rather than used as a cleanup close reason.

## Best Possible Solution

Keep PR #72209 open for maintainer review. The best path is to refine the PR: add passive recovery bundle/ledger state only where needed, wire update_plan/sessions_yield/sessions_spawn without changing primary tool behavior, include prompt discipline that treats recovered context as evidence rather than instruction, preserve the no approval/auth/permission inheritance invariant, address the P2 redaction and I/O review comments, and add focused tests plus docs if the new recovery state becomes user/operator visible.
