## Summary

Keep open. Current main partially addresses the private-network story through SearXNG web_search and model-provider request allowPrivateNetwork support, but generic web_fetch still has no private-network opt-in and Gateway input_image/input_file URL fetches still force private-network blocking. The linked implementation PR was closed unmerged, and related provider-request work covers a different surface.

## Best Possible Solution

Keep this issue open and scope the remaining product/security decision around a narrow private-network opt-in for generic web_fetch and Gateway input_image/input_file URL fetching. The likely implementation should extend the existing SSRF policy/config schema, preserve default blocking, wire the flag through runtime fetch guards, update public web_fetch/Gateway docs, and add SSRF regression tests. Existing SearXNG and model-provider allowPrivateNetwork support can be treated as related precedent, not full resolution.

## Risks / Open Questions

- Private-network access is SSRF-sensitive; the remaining work needs explicit default-off semantics, cache-key isolation where applicable, schema/help/docs updates, and regression tests for denied-by-default and allowed-with-opt-in behavior.
- Closing now would conflate partial SearXNG and model-provider support with the still-missing generic web_fetch and Gateway URL-input opt-ins.
