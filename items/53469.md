## Summary

Keep this issue open. Current main has partial private-network support for SearXNG web_search and model-provider request transport, but the broader requested surfaces are still missing: generic web_fetch, remote image/media references that inherit web_fetch policy, and Gateway OpenResponses input_file/input_image URL fetching.

## Best Possible Solution

Keep this issue open and scope the remaining work around a narrow, explicit private-network opt-in for tools.web.fetch.ssrfPolicy, remote media/image references that inherit that policy, and gateway.http.endpoints.responses.{files,images} URL fetches. Preserve default blocking, add schema/help/docs, decide carefully whether gateway.mode=local should auto-enable anything, and cover web_fetch, redirects, Gateway URL inputs, and cache behavior with SSRF tests. Treat the shipped SearXNG provider and models.providers.*.request.allowPrivateNetwork support as partial precedent, not full resolution.

## Risks / Open Questions

- Private-network access is SSRF-sensitive; any implementation should stay default-off except for an explicitly reviewed local-mode rule.
- The web_fetch cache key must include any private-network policy so allowed and denied fetches cannot share cached payloads.
- Redirects, DNS pinning, hostname allowlists, and Gateway input_file/input_image URL fetches need denied-by-default and allowed-with-opt-in regression coverage.
