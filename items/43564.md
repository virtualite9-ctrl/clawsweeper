## Summary

Keep this issue open. Current main still does not implement ACP session skill-context injection: `sessions_spawn` has no `skills` selector, the ACP spawn path passes only task/session controls, and the ACP runtime turn contract has no rendered-skill or prompt-context field. Existing skill snapshot machinery is available on the normal OpenClaw agent path, but it is bypassed for ready ACP sessions and is not wired into ACP spawns.

## Best Possible Solution

Implement this as an additive ACP/session-spawn feature in the ACP-owned path: add a typed `skills` selection surface or documented inherited default, reuse `buildWorkspaceSkillSnapshot` and existing agent skill allowlists for rendered instruction text, add a structured ACP prompt-context seam or carefully bounded task-prefix strategy, and cover explicit lists, default/auto behavior, truncation, no-tool-grant isolation, and host-security behavior in ACP-specific tests and docs.

## Risks / Open Questions

- Naively prefixing ACP tasks with rendered `SKILL.md` content could leak local workflows or sensitive instructions to external host harnesses; implementation needs explicit allowlists/defaults, auditability, and prompt-size limits.
- ACP sessions run outside the OpenClaw sandbox and do not receive OpenClaw tools by default, so skill injection must pass instructions only and must not implicitly grant skill env vars, provider keys, OpenClaw tools, or host permissions.
- Automatic relevance detection would need deterministic ordering and conservative behavior; an explicit skill list or documented default allowlist is safer as the first additive contract.
