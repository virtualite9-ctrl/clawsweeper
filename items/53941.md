## Summary

Current main still has the LiteLLM docs gap. The public docs source keeps an unpinned `pip install 'litellm[proxy]'` example and has no advisory/remediation text for the v1.82.7/v1.82.8 compromise, while the bundled LiteLLM plugin remains a network/proxy integration with no PyPI LiteLLM runtime dependency.

## Best Possible Solution

Keep this issue open and land or adapt the contributor's LiteLLM docs advisory in `docs/providers/litellm.md`: replace the unpinned install example with current safe guidance, add a dated notice for the v1.82.7/v1.82.8 LiteLLM PyPI compromise, and include downgrade, key-rotation, and server-audit steps for users who ran an affected proxy. The public docs page is `https://docs.openclaw.ai/providers/litellm`.
