## Summary

Current main still has the reported LiteLLM documentation gap: the public provider doc uses an unpinned `pip install 'litellm[proxy]'` example and has no advisory/remediation text for the reported v1.82.7/v1.82.8 compromise. The bundled OpenClaw LiteLLM integration remains a network/proxy plugin, so the right path is a docs fix, not a runtime dependency change.

## Best Possible Solution

Keep this issue open. Land or adapt a docs-only change in `docs/providers/litellm.md`: replace or qualify the unpinned LiteLLM proxy install example with current safe install/pinning guidance, add a dated security notice for the reported v1.82.7/v1.82.8 PyPI compromise, and include check-version, downgrade/reinstall, key-rotation, and server-audit guidance for users who ran an affected self-hosted proxy. No OpenClaw runtime dependency change appears needed because the bundled integration talks to a user-hosted proxy over HTTP.
