## Summary

Keep this issue open. Current main has partial session labeling and session selection, but it does not implement the requested left-side conversation management features: manual drag ordering, folders/tags/custom categories, or moving conversations between groups.

## Best Possible Solution

Keep this issue open as a Control UI conversation-management enhancement. The best path is to design a first-class session organization contract, likely extending `SessionEntry`, protocol schemas, and `sessions.patch` or adding dedicated move/reorder APIs, then add a left conversation list with drag ordering, folder/tag/category assignment, context-menu or double-click rename, move actions, tests, and docs. Reuse current `label` behavior for the rename/display-name part.
