## Summary

Current main still does not implement the requested ChatGPT/Claude-style left conversation list management. OpenClaw has session listing, sorting/filtering, bulk delete, and editable labels, but not persistent sidebar drag ordering, folder/tag/custom category grouping, moving conversations between groups, or double-click/context-menu rename in the chat sidebar.

## Best Possible Solution

Keep this open as a Control UI conversation-management enhancement. The best path is to design an additive persistent session organization contract in the Gateway/session store for folder/tag/category assignment, manual ordering, and move operations, then implement the left conversation list UI with drag ordering, group/move actions, and double-click/context-menu rename. Reuse existing label support where it fits, and add focused protocol, session-store, and Control UI tests plus docs once the product contract is settled.
