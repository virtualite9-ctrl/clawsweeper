## Summary

Keep open. Current main still has the reported cosmetic false-positive path: `openclaw gateway status` gathers `auditGatewayServiceConfig`, the audit compares the LaunchAgent PATH against the full Darwin minimal-path superset, and that superset still unconditionally includes common user-bin/version-manager directories such as `~/.npm-global/bin`, `~/bin`, `~/.volta/bin`, `~/.asdf/shims`, fnm aliases, and `~/.local/share/pnpm` without checking whether those directories exist or are required. The emitted message still says `Gateway service PATH missing required dirs`, so the central issue remains actionable in the repo.

## Best Possible Solution

Keep this issue open and fix the Darwin PATH audit in `src/daemon/service-audit.ts` or the service path contract in `src/daemon/service-env.ts`: warn only for existing/env-configured meaningful user-bin entries, or reword/downgrade absent optional entries. Add a focused macOS regression test for a healthy LaunchAgent PATH that omits nonexistent user-bin/version-manager directories and does not produce `gateway-path-missing-dirs`.
