## Summary

Current `main` already includes a browser fix for Linux/Docker no-display environments that matches this report: local managed `openclaw` browser profiles now auto-fall back to headless mode when `DISPLAY`/`WAYLAND_DISPLAY` are unset, with regression tests and updated docs. This aligns with the issueŌĆÖs symptom pattern where manual headless Playwright worked in the container but the OpenClaw-managed browser timed out.

## Evidence

## Risks / Open Questions

- The fix appears to be on `main` only; users still on released builds before this commit may continue to hit the timeout.
- If a deployment explicitly forces headed mode on headless Linux (`browser.headless=false`, `browser.profiles.<name>.headless=false`, or `OPENCLAW_BROWSER_HEADLESS=0`), the no-display failure path can still occur by design.
