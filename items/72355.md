## Summary

Keep #72355 open. Current main appears to fix the reported CIAO unhandled-rejection crash-loop path, but it still defaults the Bonjour advertised hostname and TXT lanHost to openclaw/openclaw.local when OPENCLAW_MDNS_HOSTNAME is unset, and there is still no discovery.mdns.hostname config key. The remaining hostname selection/configuration request is concrete and not superseded by the related crash-loop closures.

## Best Possible Solution

Keep the issue open but narrow its scope to Bonjour hostname selection. A good fix would add a plugin-owned/config-backed hostname override, or deliberately switch the default to a sanitized system hostname, then align MdnsDiscoveryConfig, the strict zod schema, schema help/generated docs, public gateway docs, and Bonjour tests while preserving the existing CIAO cancellation suppression and bounded advertiser recovery.
