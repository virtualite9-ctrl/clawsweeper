## Summary

Keep #72355 open. Current main fixes the fatal Bonjour/CIAO unhandled-rejection crash-loop path, but it still defaults the advertised mDNS hostname and `lanHost` to `openclaw`/`openclaw.local` unless `OPENCLAW_MDNS_HOSTNAME` is set, and the public config contract still has no `discovery.mdns.hostname` key. The remaining hostname default/configuration request is concrete and not superseded by the related crash-loop closures or the open instance-name PR.

## Best Possible Solution

Keep the issue open and narrow it to Bonjour hostname selection. A good maintainer path is to decide whether the bundled Bonjour plugin should default to a sanitized system hostname or add a plugin/config-backed hostname override, then update `MdnsDiscoveryConfig`, the strict zod schema, generated schema/help/docs, and Bonjour tests while preserving the current CIAO cancellation suppression and bounded advertiser recovery.
