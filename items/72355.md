## Summary

Keep this issue open. Current main appears to have fixed the fatal Bonjour/CIAO unhandled-rejection crash-loop path, but the reported hostname behavior remains: Bonjour still defaults the advertised ciao hostname and TXT lanHost to openclaw/openclaw.local unless OPENCLAW_MDNS_HOSTNAME is set, and the public config contract still has no discovery.mdns.hostname option. The prior ClawSweeper comment is still accurate against current main.

## Best Possible Solution

Keep the issue open and narrow it to Bonjour hostname selection. Maintainers should decide whether the bundled Bonjour plugin should default to a sanitized system hostname or add a supported hostname override, then update MdnsDiscoveryConfig, the strict zod schema, generated schema/help/docs, and Bonjour tests while preserving the current CIAO cancellation suppression and bounded advertiser recovery.
