## Summary

Current `main` still appears to have the contract gap described in #49380. `/status` renders `sessionEntry.compactionCount`, but plugin context engines still have no `afterTurn()` path to report proactive/background compactions back into the session store.

## Evidence

- **`afterTurn()` still cannot report compaction activity:** The current `ContextEngine` contract keeps `afterTurn(params)` as `Promise<void>` with no callback, return payload, or stats hook for compaction reporting. That leaves plugin-owned background compactions with no direct path to update session state.
  - file: [src/context-engine/types.ts:219](https://github.com/openclaw/openclaw/blob/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4/src/context-engine/types.ts#L219)
  - command: `nl -ba src/context-engine/types.ts | sed -n '150,310p'`

- **`/status` still reads only the session-store counter:** The status card still formats compactions from `entry?.compactionCount ?? 0`; there is no read from a context-engine-owned stats surface.
  - file: [src/status/status-message.ts:712](https://github.com/openclaw/openclaw/blob/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4/src/status/status-message.ts#L712)
