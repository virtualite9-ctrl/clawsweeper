## Summary

Keep open. Current main still does not support `channels.whatsapp.groups.<jid>.allowFrom` or account-scoped group-entry `allowFrom`: the WhatsApp group schema/types/generated metadata only allow `requireMention`, tool policy, `toolsBySender`, and `systemPrompt`, while inbound access control still uses only account/channel-level `groupAllowFrom` with `allowFrom` fallback. The related #63302 denylist PR and #69874 multi-account inheritance issue are adjacent but do not supersede this per-group sender-authorization request.

## Best Possible Solution

Keep the issue open and implement it in the bundled WhatsApp plugin: add `allowFrom` to `WhatsAppGroupConfig`, Zod schema, generated config metadata, docs, and tests; then resolve the effective sender allowlist per group before the shared group access check, with account-scoped groups replacing root groups consistently with the chosen #69874 inheritance semantics.
