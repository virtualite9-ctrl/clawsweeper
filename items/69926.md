## Summary

Keep open. Current main does not implement WhatsApp per-group allowFrom: the WhatsApp group schema and generated metadata only expose requireMention, tools, toolsBySender, and systemPrompt; inbound group sender authorization still resolves from account.groupAllowFrom with fallback to account.allowFrom; and the docs describe only channel/account-level groupAllowFrom for WhatsApp. The linked prior work (#59553, #69874, #63302) is related but does not supersede this request.

## Best Possible Solution

Implement the feature in the WhatsApp plugin/config surface: add optional allowFrom to WhatsAppGroupConfig and WhatsAppGroupEntrySchema for both root and account-scoped groups, regenerate config metadata, resolve the effective group entry for the inbound conversation, and have sender authorization use groups[<jid>].allowFrom as an override before falling back to account/channel groupAllowFrom. Add focused schema, account-scope, runtime access-control, command-auth, and docs coverage, and coordinate the multi-account root-groups inheritance semantics with #69874.
