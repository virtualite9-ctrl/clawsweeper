## Summary

Keep open. Current `main` still hardcodes the sandbox media staging cap at 5 MB for inbound sandbox staging, and the requested top-level `tools.media.maxBytes` configuration surface does not exist in the current config schema/types.

## Evidence

- **Hardcoded 5 MB constant remains on main:** `MEDIA_MAX_BYTES` is still defined as `5 * 1024 * 1024` in the media store.
  - file: [src/media/store.ts:16](https://github.com/openclaw/openclaw/blob/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503/src/media/store.ts#L16)

- **Inbound sandbox staging still uses the hardcoded cap:** `stageSandboxMedia` sets `STAGED_MEDIA_MAX_BYTES = MEDIA_MAX_BYTES` and passes that limit into local/remote staging; `get-reply` still invokes this path for inbound media on sandboxed sessions.
  - file: [src/auto-reply/reply/stage-sandbox-media.ts:19](https://github.com/openclaw/openclaw/blob/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503/src/auto-reply/reply/stage-sandbox-media.ts#L19)
