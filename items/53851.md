## Summary

The request is for a niche domain-specific skill, and current OpenClaw scope explicitly routes new skills to ClawHub rather than core. Current docs already provide the supported path to build and distribute this externally as a skill/plugin.

## Close Comment

Codex review: closing this as a ClawHub-scope request rather than an OpenClaw core issue. The current vision explicitly says new skills should be published to ClawHub first and that OpenClaw will not merge new core skills when they can live on ClawHub (`VISION.md:66`, `VISION.md:99`). The supported path already exists in the docs: ClawHub is the public registry for OpenClaw skills/plugins (`docs/tools/clawhub.md:10`), `openclaw skills install` installs skills into the workspace (`docs/tools/skills.md:126`), and custom `SKILL.md`-based skills are a documented first-class workflow (`docs/tools/creating-skills.md:9`). Because this request is a domain-specific Ediktsdatei lookup skill, it fits the ClawHub/plugin route instead of OpenClaw core.
