## Summary

Keep this PR open. Current main still uses the old inline skill tool-dispatch path: it creates raw OpenClaw tools and then applies only owner-only filtering, while the normal command/coding path uses createOpenClawCodingTools and the broader profile/provider/group/subagent policy pipeline. The supplied PR remains a focused security/policy fix with review feedback already addressed at its head SHA.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land this PR or an equivalent patch that routes inline skill command-dispatch: tool through createOpenClawCodingTools with the same route, sender, group, session, config, model, and provider inputs used by normal command/coding runs; remove the standalone owner-only fallback; keep regression tests for non-owner/profile-disallowed tools and route-context preservation; and run the targeted inline dispatch test plus pnpm check:changed before merge.
