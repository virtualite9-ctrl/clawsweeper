## Summary

Keep this PR open. The current main code still routes inline skill `command-dispatch: tool` through the raw OpenClaw tool factory plus owner-only filtering, while normal command sessions use `createOpenClawCodingTools` and its broader profile/provider/group/subagent policy pipeline. The PR remains a focused security/policy fix, and the discussion shows the author addressed the review requests for route context and policy-argument assertions, but the patch should be rebased to preserve current main's lazy runtime boundary.

## Best Possible Solution

Keep this PR open for maintainer review. Rebase or adapt it on current main so inline skill `command-dispatch: tool` resolves through `createOpenClawCodingTools`, removes the standalone owner-only fallback, preserves the lazy runtime boundary, forwards the same policy-critical route/session/group/sender/model/config context as normal command runs plus inline message target/thread context, and keeps focused regression tests for non-owner/profile-disallowed tools and argument forwarding.
