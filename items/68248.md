## Summary

Keep open. Current main has implemented the major avatar plumbing requested here: `IDENTITY.md`/config avatars are parsed, resolved, exposed through Control UI avatar routes, and rendered in chat/assistant surfaces. But the issue also explicitly asks for per-agent/browser-tab favicon customization and user display-name customization, and current main still appears to use static Control UI favicons with no per-agent favicon config or runtime updater. The related #53380 can be treated as covering the avatar/emoji portion, but it does not supersede the remaining favicon/display-name work.

## Best Possible Solution

Keep the shipped avatar implementation and narrow this issue to the remaining product gaps: add a documented per-agent/current-session favicon mechanism for Control UI tabs, add or clarify user display-name customization in the Control UI settings path, and decide whether agent/session pickers should render actual avatar images rather than only names/emoji/custom metadata. If maintainers only want avatar support, close this after splitting the favicon/display-name requests into narrower follow-ups.
