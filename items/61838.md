## Summary

Keep open. Current `main` does not show a shipped execution-path fix for raw Qwen-style XML tool calls from llama.cpp/OpenAI-compatible local backends. The code on `main` adds display sanitization for leaked `<tool_call>` XML, and the docs still describe this class of failure as a local-backend/tool-schema compatibility problem with mitigations, not as a resolved OpenClaw core fix.

## Evidence
