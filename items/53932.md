## Summary

Keep this issue open. Current main still routes Slack outbound media through the optimized web-media loader, and Slack config/docs expose only media size limits, not an opt-out for image optimization or raw/original-quality upload behavior. The related implementation attempt #53988 was closed unmerged, so it does not supersede or implement the request.

## Best Possible Solution

Implement this in the bundled Slack plugin as an explicit, documented config control, likely under channels.slack and per-account Slack config, then route Slack uploads through raw outbound media loading when disabled while preserving maxBytes/SSRF/local-root checks. Add regression coverage that a PNG upload can preserve its original MIME/bytes when the opt-out is enabled and retains current optimized behavior by default.
