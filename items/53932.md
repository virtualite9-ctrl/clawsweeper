## Summary

Keep open. Current `main` still routes Slack file uploads through the shared outbound media loader without passing `optimizeImages: false`, while the shared loader defaults image optimization on. I found no Slack config or docs surface that lets users disable optimization for uploads, so the feature request is not implemented on `main`.

## Evidence

- **Slack upload path uses shared outbound loader with no opt-out:** `uploadSlackFile` calls `loadOutboundMediaFromUrl(...)` with only `maxBytes`, `mediaAccess`, `mediaLocalRoots`, and `mediaReadFile`; there is no `optimizeImages` or Slack-specific raw-file flag.
  - file: [extensions/slack/src/send.ts:255](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/extensions/slack/src/send.ts#L255)

- **Outbound media helper does not expose optimization control here:** `loadOutboundMediaFromUrl` forwards a narrow options object into `buildOutboundMediaLoadOptions(...)`; its exported options type in this file does not include `optimizeImages`, so Slack cannot request raw-image passthrough through this helper.
