## Summary

Current main still does not provide a Slack setting or sender option to preserve original image bytes on outbound Slack uploads. Slack file sends continue through `loadOutboundMediaFromUrl`, which calls the optimized web-media path, while Slack config/schema/docs expose only `mediaMaxMb` for this area. The related implementation attempt #53988 is closed and unmerged, so it does not supersede the remaining work.

## Best Possible Solution

Keep this open and implement the opt-out in the bundled Slack plugin. Add a documented Slack config key that defaults to current optimized behavior, wire it through Slack send/upload using an additive outbound-media option or a deliberate raw-loader path, update config types/schema/generated metadata/docs, and add regression coverage proving PNG uploads can preserve original bytes and MIME when disabled while the optimized default remains unchanged.
