## Summary

Keep this issue open. Current main still has no Slack upload setting or call-path option to preserve original image bytes; Slack outbound uploads still go through the optimized outbound media helper. The related implementation attempt, #53988, is closed unmerged in the provided GitHub context.

## Best Possible Solution

Keep this issue open and implement the opt-out in the bundled Slack plugin. A conservative path is to add a documented Slack config key that preserves current optimized behavior by default, then either expose/pass `optimizeImages: false` through the outbound media SDK helper or route Slack through an existing raw media loader when disabled. Add schema/types/generated metadata/docs and regression coverage proving PNG uploads can preserve original bytes/MIME while size and security guards still apply.
