## Summary

Keep #13241 open. Current main has supporting pieces for stale cleanup, auto-response labels, PR/issue routing, duplicate cleanup, structured issue intake, and broad roadmap priorities, but it still does not implement the requested lightweight issue triage framework: issue lifecycle states, P0-P3 priority taxonomy, priority assignment rules, recurring cadence, visible current-cycle priorities, and documented stale-state transitions.

## Best Possible Solution

Keep the issue open for maintainer process design. A complete solution should reuse the existing stale automation, Barnacle cleanup labels, issue templates, duplicate cleanup, and roadmap docs as supporting infrastructure, while adding a minimal repo-owned triage playbook that defines lifecycle labels, P0-P3 priority semantics, who assigns priority, cadence, stale-state transitions, and a visible current-cycle priorities surface.
