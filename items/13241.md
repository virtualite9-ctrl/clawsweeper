## Summary

Keep #13241 open. Current main has generic stale automation, issue templates, duplicate-closing tooling, Barnacle auto-response labels, and broad roadmap wording, but it still does not implement the requested lightweight issue triage and priority framework. The issue discussion also includes a same-day maintainer-side Codex review comment explicitly keeping it open for this gap.

## Best Possible Solution

Keep the issue open for maintainer process judgment. A complete solution should define the smallest repo-owned issue label/state taxonomy, clarify who assigns priority and how, establish the triage cadence and stale-state transitions, and create or document a visible current-cycle priorities surface. Existing stale automation, Barnacle labels, issue templates, and broad roadmap wording should be treated as supporting infrastructure, not the completed framework.
