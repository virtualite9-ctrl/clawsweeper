## Summary

Keep open. Current main has Matrix `allowBots` support for messages sent by other configured Matrix accounts, but it does not implement the requested opposite-side gate: skipping a human/user message for Bot B when that message explicitly mentions Bot A and Bot B has `requireMention: false`. Matrix mention resolution only determines whether the current account was mentioned, and the dispatch gate only skips when `requireMention` is true and the current account was not mentioned.

## Best Possible Solution

Keep this open for a focused Matrix plugin enhancement: add an opt-in room/account setting that detects explicit Matrix mentions of other configured account user IDs, skips before typing/session/LLM dispatch for non-mentioned accounts even when `requireMention: false`, and adds schema/types/docs plus handler tests for multi-account rooms. The related closed #45450/`allowBots` work is useful context but solves bot-authored message visibility, not this token-saving exclusive mention dispatch gate.
