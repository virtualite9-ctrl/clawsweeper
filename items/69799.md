## Summary

Keep #69799 open. Current main has partial mitigation through the native macOS app and an optional local-prefix installer, but it has not fully shipped the requested signed app-bundled or default dedicated Node/Gateway runtime for the npm/global LaunchAgent workflow. The docs still describe OpenClaw.app as using an external CLI, launchd still installs the gateway with a Node runtime, and iMessage docs still require Full Disk Access for the process context running OpenClaw/imsg. Related #7227 remains an open security-labeled tracker for the same TCC-to-node trust-boundary class, but #69799 adds Full Disk Access and dedicated-runtime packaging scope, so it is not conservative to auto-close it as fixed or duplicate.

## Best Possible Solution

Keep this open for maintainer product/security judgment. The best path is to either implement a signed app-bundled or OpenClaw-managed stable dedicated Node/Gateway runtime for LaunchAgent and TCC-sensitive macOS workflows, or explicitly route all TCC-gated macOS capabilities through the signed OpenClaw.app/bridge while documenting the residual npm/global-node risk. If maintainers want one canonical tracker, consolidate #69799 with open security issue #7227 rather than closing it as already fixed.
