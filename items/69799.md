## Summary

Keep this issue open. Current main still documents and implements an external Node-backed macOS Gateway path rather than a dedicated OpenClaw-scoped Node/runtime identity, so the TCC/FDA boundary request remains actionable core security and packaging work.

## Best Possible Solution

Keep this issue open as core macOS security architecture work. The best path is to implement and document an OpenClaw-scoped TCC runtime boundary, such as a signed app-owned Gateway/runtime or a dedicated installed Node binary with stable identity, and coordinate it with the open #7227 security thread plus the narrower docs and stable-path work from #69561 and #22179.
