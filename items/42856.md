## Summary

Keep open. Current main still classifies `gateway` action `config.schema.lookup` as mutating because compound action names are compared directly against simple read-only verbs; that mutating flag is then carried into `lastToolError` and forces a user-facing warning. The gateway method itself correctly returns `config schema path not found`, and the method scope marks lookup as read-only. Two open, unmerged PRs (#42898 and #42962) explicitly claim to close this issue, so the issue should stay open until one is reviewed and merged or replaced.

## Best Possible Solution

Keep the issue open and review the linked implementation candidates. The strongest fix likely belongs in `src/agents/tool-mutation.ts`: classify compound `gateway`/`cron`/`canvas` actions by their leaf read-only verb or otherwise explicitly treat `config.schema.lookup` as non-mutating, then add regression coverage for both mutation classification and payload warning suppression for `config schema path not found`.
