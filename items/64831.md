## Summary

Keep open. Current main has meaningful adjacent cron and Telegram reliability work, but it does not close the reported multi-symptom regression: direct exec cron is still not implemented, restart catch-up is not the requested configurable missed-delivery policy, and the Telegram approval/delivery/OOM/fallback claims need current-version reproduction data before they can be resolved or safely split.

## Best Possible Solution

Keep this issue open. Ask the reporter to retest on the latest release/current main and provide sanitized cron job JSON, exact chat-triggered commands, Telegram target/account details, approval IDs and timestamps, logs around missed runs, Node OOM output, and whether each failure is main-session, isolated cron, or chat-triggered exec. Then split focused follow-ups: direct shell/exec cron to #18160/#29907, Telegram delivery/routing to the Telegram plugin and outbound layer, approval instability to the shared approval runtime plus Telegram adapter, and scheduler catch-up/failure-reporting defects to `src/cron` only after a current-main reproduction shows a core scheduler bug.
