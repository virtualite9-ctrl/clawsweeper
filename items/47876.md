## Summary

ClawMonitor looks useful, but the remaining request is a community-tool discovery/docs request that belongs on ClawHub rather than as an OpenClaw core issue. Current main already exposes the relevant core monitoring surfaces for external tooling: health/status docs, session list/preview RPCs with status/timing/message preview fields, and diagnostics metrics via OpenTelemetry/Prometheus. The linked docs PR #47877 is still open and docs-only, and current maintainer policy says plugin/tool promotion belongs on ClawHub instead of source docs discoverability pages.

## Best Possible Solution

Close this issue as ClawHub-scoped: keep OpenClaw core focused on stable health/session/diagnostics APIs, have ClawMonitor published and maintained externally via ClawHub or its own repository, and open a separate narrow API request only if a concrete missing generic monitoring field blocks external tools.
