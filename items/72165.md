## Summary

Keep open. Current main still has the reported eager session-row materialization and synchronous transcript I/O paths, and the older #57715 tracker only clearly covers the `sessions.list` hot path. #72165 adds concrete unresolved scope around direct `sessionId` lookup and MCP `getConversation` reusing a 500-row list scan, so it should not be auto-closed as a duplicate without maintainer consolidation.

## Best Possible Solution

Keep #72165 open or have a maintainer explicitly consolidate its direct lookup and MCP scope into #57715. The best fix is a focused session summary/index path that filters, sorts, and limits before transcript hydration where semantics allow, preserves search behavior, supports direct lookup by `sessionId`, and changes MCP `getConversation` away from a 500-row `sessions.list` scan.
