## Summary

Keep open. Current main still matches the reported eager session-list materialization and synchronous transcript I/O paths. The older #57715 tracker overlaps the `sessions.list` hot path, but this issue adds unresolved direct `sessionId` lookup and MCP `getConversation` scope, so a conservative cleanup should not close it as implemented or duplicate without maintainer consolidation.

## Best Possible Solution

Keep #72165 open unless a maintainer explicitly consolidates its direct lookup and MCP scope into #57715 or another canonical tracker. The best implementation is a focused session summary/index path: persist or cache summary metadata at write time, filter/sort/limit before transcript hydration where semantics allow, provide direct lookup by `sessionId`, change MCP `getConversation` away from a 500-row `sessions.list` scan, and add regression coverage proving non-returned rows do not perform transcript reads.
