## Summary

Keep open. Current main still has the reported eager session-row materialization and synchronous transcript I/O paths. #57715/#59436/#68878 overlap the `sessions.list` hot path, but they do not fully cover #72165's additional direct `sessionId` lookup and MCP `getConversation` list-reuse concerns, so this should not be auto-closed as a duplicate without maintainer consolidation.

## Best Possible Solution

Keep #72165 open or have a maintainer explicitly merge its direct lookup/MCP scope into #57715. The best implementation is a focused session summary/index path that filters, sorts, and limits before transcript hydration where semantics allow, preserves search behavior, supports direct lookup by `sessionId`, and changes MCP `getConversation` away from a 500-row `sessions.list` scan.
