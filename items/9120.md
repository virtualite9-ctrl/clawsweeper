## Summary

Keep #9120 open. Current main has timer-based inbound debouncing and active-run queue collection, but it still does not implement the requested user-controlled continuation marker, marker stripping, continuation timeout, or continuation-aware typing mode. The Apr 26 Codex review comment in the issue reached the same conclusion, and the relevant config/debounce/docs files have not changed since that review.

## Best Possible Solution

Keep the issue open for maintainer/product judgment. A complete implementation likely belongs in shared core inbound handling, not as a ClawHub-only plugin, because it spans all channels and config validation. Add typed and documented messages.continuationMarker and messages.continuationTimeout settings, validate them in strict schema/help/docs, buffer marked text-only inbound messages before invoking the agent, strip the marker from model-visible text, release on the first unmarked message or timeout, define media/control-command behavior, update shared channel/plugin inbound helpers, and add focused unit/integration coverage.
