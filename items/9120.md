## Summary

Keep #9120 open. Current main still has timer-based inbound debouncing and active-run collect queueing, but not the requested user-controlled continuation marker that buffers marked fragments until an unmarked final message arrives. The existing related queue work addresses busy-session followups, not pre-agent marker batching.

## Best Possible Solution

Keep the issue open for maintainer/product judgment. A complete implementation likely belongs in shared inbound handling or a documented plugin/API seam: add typed messages.continuationMarker and messages.continuationTimeout config/schema/docs, buffer marked text-only inbound messages before agent invocation, strip the marker from model-visible text, release on the first unmarked message or timeout, define media/control-command behavior, and add focused tests across inbound helpers, channel routing, queue interaction, and typing policy.
