## Summary

Current main has absorbed some Telegram fixes, but this PR is not fully implemented or safely obsolete. Several central fixes from the PR are still missing in the current plugin implementation, and the PR edits the old `src/telegram` layout rather than the current `extensions/telegram` plugin layout. It should be split and ported, not closed as implemented or merged as-is.

## Best Possible Solution

Keep this PR open for maintainer handling, but do not merge it as a single stale batch. Split and port the still-relevant fixes into `extensions/telegram`, prioritizing getFile/null placeholder handling, command-menu cache invalidation, mention-gating verbose logging, media-group item handling, and the draft-stream race. Preserve the safer current retry and approval authorization boundaries, and treat already-absorbed pieces such as ACP placement, reply parameters, rate-limit retry handling, and proxy transport support as separate from the remaining work.

- Several linked issues are closed, but current source and tests still show relevant remaining behavior, so linked issue closure is not enough evidence to close this PR as implemented.
