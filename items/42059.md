## Summary

Keep open. Current main has absorbed some Telegram fixes, but the PR is not fully implemented or safely obsolete: current main still preserves the getFile-null placeholder path, lacks the requested verbose mention-gating drop log, and still hashes native command sync by command list only. The batch also needs a security-sensitive approval-auth review before any remaining pieces are ported.

## Best Possible Solution

Keep this PR open for maintainer handling, but do not merge the batch as-is. The best path is to split and port the still-relevant fixes into the current extensions/telegram layout, starting with the getFile failure/placeholder-only media behavior and then the mention-gating/logging and command-menu cache issues, while separately reviewing the approval-auth changes for unauthorized-response regressions.
