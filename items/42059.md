## Summary

Keep open. Current main has absorbed some Telegram fixes, but this PR is not fully implemented or safely obsolete: getFile failures still take the null/placeholder path, mention-gating verbose output is still missing, command-menu sync still hashes only the command list, and media-group handling still rethrows non-recoverable item failures. The batch also targets the old `src/telegram` layout, so the remaining work should be split and ported into `extensions/telegram` rather than merged as-is.

## Best Possible Solution

Keep this PR open for maintainer handling, but split it before landing. Port the still-relevant Telegram fixes into the current `extensions/telegram` implementation, starting with #40991 getFile/null placeholder handling, command-menu cache invalidation, mention-gating verbose logging, and media-group item handling. Preserve the safer current retry and approval authorization boundaries, and leave separately absorbed pieces such as ACP placement, reply parameters, rate-limit retry handling, and word-boundary chunking on main.
