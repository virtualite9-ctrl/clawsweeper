## Summary

Keep this issue open. Current main still runs the generic `sessions_send` A2A reply-back loop for persistent peer sessions by default, and the merged related PR only skipped A2A for parent-owned one-shot ACP children. The earlier duplicate #62814 was closed specifically because this issue is the canonical tracker for the remaining persistent-session behavior.

## Best Possible Solution

Keep this issue open as the canonical tracker. The best fix is in the core `sessions_send` A2A path: either add a per-call `maxPingPongTurns` or disable flag, or make the A2A flow context-aware so persistent peer-session sends return the direct inline reply without framework-generated ping-pong/announce turns by default, while preserving the existing parent-owned one-shot ACP child skip and adding regression coverage for persistent peer sessions.
