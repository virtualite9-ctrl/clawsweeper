## Summary

Keep open. Current main still does not make bundled `session-memory` persist on automatic daily/idle session rollover. Main now has session lifecycle hook context for stale rollovers, but the bundled internal hook and its metadata remain limited to `command:new` and `command:reset`, so the PR is still targeting a real remaining gap. The PR should be rebased/adapted to current main and should address the unresolved ACP duplicate-write review concern before merge.

## Best Possible Solution

Keep this PR open for maintainer/contributor follow-up. Rebase it onto current main, decide whether automatic persistence should use the existing plugin `session_end` lifecycle payload or a new typed internal `session:rollover` event, then update bundled `session-memory` metadata/docs/tests so daily and idle rollover snapshots are covered without double-writing explicit ACP reset flows.
