## Summary

Keep this PR open. Current main still does not persist bundled session-memory on automatic daily/idle stale-session rollover: the bundled hook only listens to explicit command:new and command:reset events, while current automatic rollover support is implemented through separate plugin session_start/session_end lifecycle hooks. The PR remains a plausible bug fix, but it should be rebased onto current main and reconciled with the existing lifecycle path plus the unresolved ACP duplicate-write review concern before merge.

## Best Possible Solution

Keep this PR open for maintainer or contributor follow-up. Rebase it onto current main, decide whether bundled session-memory should consume the existing session_end lifecycle payload or a new typed internal session:rollover event, then update handler metadata/docs/tests so daily and idle automatic rollovers persist memory without double-writing explicit ACP reset flows.
