## Summary

Keep #19075 open. Current main still does not implement a configurable Brave Search baseUrl: the Brave runtime constructs both Search and LLM Context requests from hardcoded api.search.brave.com endpoints, the bundled Brave plugin config schema only accepts apiKey and mode, and the public Brave docs do not document any baseUrl/proxy option. The related implementation PRs in the provided context are closed unmerged, while duplicate feature reports were closed back to this canonical tracker.

## Best Possible Solution

Keep #19075 open as the canonical tracker. The likely implementation belongs in extensions/brave: add plugins.entries.brave.config.webSearch.baseUrl to the manifest/UI hints and Brave config type, normalize and validate it, build both /res/v1/web/search and /res/v1/llm/context URLs from the normalized base while preserving the default api.search.brave.com behavior, include the base URL in cache identity, update https://docs.openclaw.ai/tools/brave-search, and add focused runtime/schema tests. The current plugin config seam appears sufficient; this is missing bundled Brave behavior, not a ClawHub-only request.
