## Summary

Keep #19075 open. Current main at 406ae72fd2789c78053f6b94006ec8f488bc1c00 still lacks a configurable Brave Search baseUrl: Brave runtime uses fixed Brave API endpoints, the cache key has no endpoint identity, Brave plugin config/schema/docs expose only apiKey and mode, and related PR attempts are closed unmerged while duplicate issues were closed back to this tracker.

## Best Possible Solution

Keep #19075 open as the canonical bundled Brave Search baseUrl tracker. Implement the current plugin-owned surface as plugins.entries.brave.config.webSearch.baseUrl, add manifest/UI/schema/help/docs coverage, normalize and validate the URL, derive both /res/v1/web/search and /res/v1/llm/context from the resolved base while preserving the default Brave endpoint, include the base in cache identity, and add focused runtime plus schema tests.

- A future implementation must include the resolved Brave base URL in the cache identity to avoid cross-proxy cached result collisions.
- Endpoint validation needs an explicit decision for http:// loopback or private-network proxy support while preserving the web-search SSRF guard.
