## Summary

Keep this PR open. Current main still does not enforce a 512 KiB UTF-8 byte cap for `chat.send` messages, and the PR is a focused gateway hardening candidate rather than cleanup. It should be revised before merge to resolve the byte-limit semantics raised in review.

## Best Possible Solution

Keep this PR open as a useful gateway hardening candidate. Ask for a revision that defines the 512 KiB policy consistently, preferably with explicit raw-vs-normalized byte semantics, no misleading schema character-count guard, updated protocol docs, and focused ASCII boundary plus multi-byte tests; then run the relevant gateway/protocol changed checks before merge.
