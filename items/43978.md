## Summary

Keep this PR open. Current main still has the old WhatsApp auto-reply retry window and still does not surface WhatsApp reply dispatch failures as session system events. The PR is stale against the current plugin layout and needs the existing review concern addressed, but the underlying bug-fix is still relevant.

## Best Possible Solution

Keep the PR open as a plausible WhatsApp reliability fix. The maintainer path is to rebase it onto `extensions/whatsapp`, widen the current retry helper/defaults to cover the reconnect window, wire dispatcher delivery errors into `enqueueSystemEvent` through the existing plugin/runtime seam, explicitly scope which reply kinds emit the event, and add regression coverage for both the wider retry window and the agent-visible failure signal.
