## Summary

Keep this PR open. Current main still uses the old WhatsApp outbound retry window and still lacks an agent-visible system event for WhatsApp auto-reply delivery failure, so the PR is not implemented or obsolete. It needs rebase/adaptation from the old `src/web` layout to `extensions/whatsapp` and should address the existing review concern about which reply kinds emit system events.

## Best Possible Solution

Keep the PR open as a plausible WhatsApp bug-fix candidate. The maintainer path is to rebase it onto `extensions/whatsapp`, update the current retry helper/defaults to cover the reconnect window, wire dispatcher delivery errors into `enqueueSystemEvent` through the existing plugin SDK/runtime seam, intentionally scope which reply kinds emit the event, and add regression tests for the retry window and agent-visible failure signal.
