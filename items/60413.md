## Summary

Keep open. Current `main` does not implement the PR's configurable Ollama request-timeout path: the Ollama stream still relies only on an externally supplied abort signal, the provider wiring does not pass `models.providers.ollama.timeoutSeconds`, and the model-provider config schema still does not accept that field. Main does have a generic LLM idle-timeout guard, so the original hang symptom may be partially mitigated, but that is not the same as this PR's provider-local timeout feature.
