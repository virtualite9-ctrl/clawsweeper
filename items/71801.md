## Summary

Keep #71801 open. Current main still forwards Codex `request_user_input` header, question, and option text directly into chat without the sanitizer or bounded-preview guardrails already present in nearby Codex approval/elicitation bridges. The PR is a plausible focused hardening fix and is not superseded, but it should be revised before merge because the provided bot reviews identify unresolved edge cases around dangerous question ids, omitted questions, and terminal-sequence ordering.

## Best Possible Solution

Keep the PR open and ask for a revision rather than auto-closing it. The best path is to land a focused `request_user_input` hardening change that sanitizes only chat display fields, enforces per-field and full-prompt bounds, ensures the answerable question set exactly matches what was shown, blocks or safely stores dangerous question ids such as `__proto__`/`constructor`/`prototype`, and adds unit tests for terminated and unterminated OSC sequences, bidi/invisible controls, oversized fields, omitted questions, and prototype-key ids.
