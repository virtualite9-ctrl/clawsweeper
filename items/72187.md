## Summary

Keep open. Current main does not contain the proposed generic plugin service lifecycle test harness: the public testing subpath lacks the helper, docs list only the existing testing utilities, and service startup still hardcodes the global state directory. The PR is a focused additive SDK/testing surface, not a ClawHub-only feature or a duplicate of the related lifecycle/harness items provided.

## Best Possible Solution

Keep the PR open for maintainer review. If accepted, land a narrow generic harness under `openclaw/plugin-sdk/testing`, keep docs/API baselines/exports aligned, preserve production `STATE_DIR` defaults, and address the two P2 review comments before merge.
