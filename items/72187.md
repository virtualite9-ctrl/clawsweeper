## Summary

Keep open. Current main does not contain the proposed `createPluginServiceLifecycleTestHarness`, does not export an equivalent helper from `openclaw/plugin-sdk/testing`, and `startPluginServices` still hardcodes `STATE_DIR` instead of accepting a caller-provided test `stateDir`. The PR is a fresh, focused plugin SDK testing-surface addition with docs/tests/API-baseline work, so there is no conservative close reason.

## Best Possible Solution

Keep the PR open. The best path is to review the proposed core plugin SDK testing helper on its merits: export `createPluginServiceLifecycleTestHarness` from `openclaw/plugin-sdk/testing`, allow test-only service startup with a caller-provided `stateDir`, keep docs/API baseline aligned, and ensure the review comments about duplicate/empty service IDs and hook option naming are addressed before merge.
