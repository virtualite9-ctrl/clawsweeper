## Summary

Keep this issue open. Current main has moved past the old direct-resume root-cause theory, but the active problem remains plausible in source: ACP `streamTo:"parent"` completion still relays through a session-scoped system event plus heartbeat wake, and targeted wakes are still skipped when heartbeat infrastructure has no enabled agent/config. The issue is also paired with open same-author PR #60402, which is an implementation candidate rather than a reason to close this issue before review/merge.

## Best Possible Solution

Keep this issue open. Review, rebase, and either land or close PR #60402, with regression coverage proving child completion wakes a yielded parent when periodic heartbeat is disabled or absent. Given the newer native-spawn report, also evaluate a shared completion/yield wake primitive instead of an ACP-only relay patch.
