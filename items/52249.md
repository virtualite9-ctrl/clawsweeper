## Summary

This issue should stay open. The original direct-resume root-cause writeup is stale, but current main still routes ACP parent completion through session-scoped system events plus heartbeat wake delivery, and heartbeat-disabled or non-enabled target-agent configurations can still skip those wakes. The issue also has a same-author open PR candidate (#60402), so the paired contributor issue/PR should remain open until that PR is reviewed, landed, replaced, or closed.

## Best Possible Solution

Keep this issue open. Review, rebase, and either land or close PR #60402, or replace it with a shared yielded-parent completion wake primitive. The accepted fix should add regression coverage proving ACP child completion wakes a yielded parent when heartbeat is disabled or absent, and should evaluate the newer native-spawn yield report so the solution is not narrowly ACP-only if the bug is shared.
