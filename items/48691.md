## Summary

Current main already solves the harmful recommendation path: the healthcheck skill now uses `openclaw update status`, and the update-status hint tells users to run `openclaw update` rather than `pnpm update`/`npm update`. The updater is the install-aware path in OpenClaw’s supported install model, and tests cover Homebrew-prefix npm installs so mixed Homebrew/nvm setups update the owning package root instead of the wrong global install.

## Best Possible Solution

Keep the shipped `openclaw update status` plus `openclaw update` flow as the healthcheck recommendation. If OpenClaw later adopts a first-party Homebrew formula, track that separately as an installer/update integration so the updater can own brew-specific semantics instead of adding ad hoc skill guidance.
