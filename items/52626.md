## Summary

Current `main` already routes Feishu tool execution by per-call account context instead of closing over `accounts[0]`, and the shipped `v2026.4.22` release contains the same implementation and regression tests.

## Close Comment

Closing as implemented on current `main` based on this Codex review. The specific `accounts[0]` closure described here no longer matches the Feishu tool code: `extensions/feishu/src/tool-account.ts:15` resolves the tool account per execution, and `extensions/feishu/src/docx.ts:1400`, `extensions/feishu/src/wiki.ts:174`, `extensions/feishu/src/drive.ts:754`, and `extensions/feishu/src/perm.ts:135` all create the Feishu client from `ctx.agentAccountId` / optional `accountId` at execute time instead of capturing the first configured account. There is also regression coverage in `extensions/feishu/src/docx.account-selection.test.ts:47` and `extensions/feishu/src/tool-account-routing.test.ts:69` covering the multi-account routing cases from this report. The same implementation is present in release `v2026.4.22` (`a94763ee224e858e29dd177cd6f7f4daa438c2f1`). If anyone still sees incorrect account selection on `v2026.4.22` or newer, that would be a different remaining bug and a new issue with current repro details would help isolate it.
