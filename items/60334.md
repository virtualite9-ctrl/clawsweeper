## Summary

Keep open. Current `main` still has no runtime or doctor migration for the reported legacy config keys, gateway startup still hard-fails invalid configs after limited recovery, and malformed `cron/jobs.json` remains a fatal parse error rather than an automatic repair/quarantine path.

## Evidence

- **Reported legacy keys are not part of the current strict agent/exec schemas:** `AgentEntrySchema` is strict and does not include `allowedModels`, and `ToolExecBaseShape`/`ToolExecSchema` are strict and do not include `allowlist`, so those keys are still unknown to validation on `main`.
  - file: [src/config/zod-schema.agent-runtime.ts:456](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/src/config/zod-schema.agent-runtime.ts#L456)
  - sha: [926068b14fc6](https://github.com/openclaw/openclaw/commit/926068b14fc661e915a0247e9ecb5949fb95b4f7)
- **Gateway read path only applies known legacy migrations before validation:** `resolveLegacyConfigForRead` runs `applyRuntimeLegacyConfigMigrations(...)` and then validates the result. There is no startup path here that strips arbitrary unknown keys; if a legacy key is not in the migration set, validation still fails.
  - file: [src/config/io.ts:1072](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/src/config/io.ts#L1072)
  - sha: [926068b14fc6](https://github.com/openclaw/openclaw/commit/926068b14fc661e915a0247e9ecb5949fb95b4f7)
- **No legacy migration exists for `allowedModels` or `tools.exec.allowlist`:** A repo search over the doctor legacy migration files returned no hits for either reported key, which matches the runtime-agent migration file only covering sandbox `perSession`, `memorySearch`, and `heartbeat`.
  - command: `rg -n 'allowedModels|tools\.exec\.allowlist' src/commands/doctor/shared/legacy-config-migrations* src/commands/doctor/shared/legacy-config-core-normalizers.ts src/commands/doctor/shared/legacy-config-compatibility-base.ts`
  - sha: [926068b14fc6](https://github.com/openclaw/openclaw/commit/926068b14fc661e915a0247e9ecb5949fb95b4f7)
- **Unknown-key cleanup is still a doctor-only flow:** `applyUnknownConfigKeyStep` removes unknown config keys only inside the doctor flow, with fix hints telling operators to run `openclaw doctor --fix`; gateway startup does not call this step directly.
  - file: [src/commands/doctor/shared/config-flow-steps.ts:66](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/src/commands/doctor/shared/config-flow-steps.ts#L66)
  - sha: [926068b14fc6](https://github.com/openclaw/openclaw/commit/926068b14fc661e915a0247e9ecb5949fb95b4f7)
- **Gateway startup still fails invalid configs after limited recovery attempts:** On startup, the gateway only tries last-known-good restore or non-JSON-prefix stripping, then immediately asserts the config is valid. That does not cover the reported legacy-key rejection path when no suitable backup/recovery applies.
  - file: [src/gateway/server-startup-config.ts:71](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/src/gateway/server-startup-config.ts#L71)
  - sha: [926068b14fc6](https://github.com/openclaw/openclaw/commit/926068b14fc661e915a0247e9ecb5949fb95b4f7)
- **Malformed cron store is still a hard error on `main`:** `loadCronStore` throws on parse failure, and the cron-store unit test still expects invalid JSON to reject rather than auto-repair or quarantine.
  - file: [src/cron/store.ts:153](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/src/cron/store.ts#L153)
  - sha: [926068b14fc6](https://github.com/openclaw/openclaw/commit/926068b14fc661e915a0247e9ecb5949fb95b4f7)
- **Cron invalid-JSON behavior is covered by an explicit failing test:** The current test suite asserts that invalid `jobs.json` causes `loadCronStore` to throw `Failed to parse cron store`, which matches the issue's second failure mode rather than showing it was fixed.
  - file: [src/cron/store.test.ts:89](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/src/cron/store.test.ts#L89)
  - sha: [926068b14fc6](https://github.com/openclaw/openclaw/commit/926068b14fc661e915a0247e9ecb5949fb95b4f7)
