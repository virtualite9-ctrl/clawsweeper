## Summary

Keep open. Current main has an owner-only agent `cron` tool and gateway `cron.add`/`cron.update`, but the shipped surface still accepts free-form cron payloads and lacks the requested non-forgeable creator provenance, playbook allowlist, recurring expiration requirement, per-agent quota, and schedule-create audit ledger. The issue is recent, externally authored, unprotected by labels, and has a supporting comment, so it should stay open for maintainer product/security judgment.

## Best Possible Solution

Keep the issue open for maintainer design review. If accepted, the best path is a core gateway/protocol addition for a narrow agent-safe scheduling method, backed by cron schema/runtime changes, immutable caller provenance derived from the authenticated session, registered playbook validation, recurring-job expiration semantics aligned with #71711, per-agent quotas, and a schedule-create audit record. Related expiration work can remain tracked in #71711 rather than blocking the broader RFC.
