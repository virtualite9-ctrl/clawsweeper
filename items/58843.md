## Summary

Current main no longer matches the PR's reported browser-tool failure path. The old `browser_navigate` / `browser_run_code` / `browser_click` tool names are absent, the bundled browser plugin now registers a single `browser` tool whose parameters come from a TypeBox `BrowserToolSchema`, and Zod-derived plugin config schemas are generated as draft-07 with `$schema` metadata stripped before AJV validation. The PR's exact patch is not present in `src/plugins/schema-validator.ts`, but the observable browser-tool regression described by #58560 is obsolete on current main.

## Best Possible Solution

Close this PR as obsolete for the browser-tool regression it was trying to fix. Keep the current plugin-backed TypeBox browser tool path and the existing Zod config-schema normalization. If maintainers want generic draft-2020-12 support in `validateJsonSchemaValue`, track that as a separate issue with a concrete current caller and regression test, likely choosing between Ajv2020 and schema-generation normalization based on that caller.
