## Summary

Keep #39879 open. Current main still does not populate Baileys outbound mention metadata for WhatsApp text or caption sends, so `@number` remains plain outbound text rather than a native WhatsApp mention. The related PR #56863 is still described in the provided context as open and unmerged, so it is a plausible implementation path but not shipped behavior.

## Best Possible Solution

Keep #39879 open until the bundled WhatsApp plugin ships outbound native mention support. The best implementation path is to add a typed outbound mention flow, extract or accept `@+<digits>` / `@<digits>` safely, resolve LID/group participant mappings where possible, pass Baileys `mentions` for text and supported media captions, handle the separate visible-text send used for voice notes, and add focused tests. PR #56863 can remain the implementation vehicle if maintainers choose to revive or merge it.
