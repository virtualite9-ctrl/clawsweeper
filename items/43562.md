## Summary

Keep open. Current main still hardcodes the leading agent identity line in `buildAgentSystemPrompt`, and the config/type/schema surfaces do not expose `agents.defaults.preamble`, `agents.list[].preamble`, or an equivalent per-agent identity-line setting. The existing `systemPromptOverride` is only a full-prompt replacement escape hatch, not the requested narrow preamble configurability. The previously linked PR #44497 was closed unmerged, and the later open PR #48442 is only an implementation candidate, not shipped behavior.

## Best Possible Solution

Keep this issue open. The best maintainer path is to review and repair #48442 or implement a narrower core change that lets `agents.defaults` and `agents.list[]` override only the leading identity line/preamble while preserving the assembled OpenClaw prompt. The change should update TypeScript config types, strict Zod schemas, generated/help docs, embedded runner, compaction, CLI runner, command-system-prompt call sites, and focused tests covering SOUL.md persona interaction and per-agent/default precedence.
