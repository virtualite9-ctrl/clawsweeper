## Summary

Keep open. Current main still hardcodes the leading identity line as “You are a personal assistant running inside OpenClaw” and only exposes `systemPromptOverride`, which replaces the entire assembled prompt rather than just the preamble or identity line. SOUL.md is still injected later under Project Context with guidance that defers to higher-priority instructions. Related PR #44497 was closed unmerged, and open PR #48442 is only an implementation candidate, not shipped behavior.

## Best Possible Solution

Keep this issue open. The best maintainer path is to review and either repair/land #48442 or implement a narrower core change that lets `agents.defaults` and `agents.list[]` override only the leading identity line/preamble while preserving the assembled OpenClaw prompt. The implementation should update config types, strict schemas, generated schema/help docs, embedded runner, compaction, CLI runner, command/system-prompt surfaces, gateway-tool allowlists, and focused tests covering SOUL.md persona interaction plus per-agent/default precedence.
