## Summary

Keep open. Current main still hardcodes the leading identity line as “You are a personal assistant running inside OpenClaw,” while SOUL.md is injected later under Project Context with lower-priority wording. The only shipped config path is `systemPromptOverride`, which replaces the entire assembled prompt instead of allowing a narrow identity/preamble override. Closed PR #44497 did not land, and open PR #48442 is only an implementation candidate.

## Best Possible Solution

Keep this issue open. The best maintainer path is to review and either land or close #48442, or implement a narrower core change that lets `agents.defaults` and `agents.list[]` override only the leading identity line/preamble while preserving the assembled OpenClaw prompt. That change should update config types, strict/generated schemas, docs, runtime prompt builders for embedded/compaction/CLI paths, gateway config mutation allowlists if agent-tunable, and focused tests for SOUL.md interaction plus per-agent/default precedence.
