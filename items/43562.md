## Summary

Keep open. Current main still hardcodes the leading “personal assistant” identity line, SOUL.md still appears later under Project Context with lower-priority wording, and the only shipped config option is full `systemPromptOverride`, not a narrow preamble or identity-line override. The earlier implementation PR #44497 is closed unmerged, while #48442 remains only an open implementation candidate.

## Best Possible Solution

Keep this issue open. The best maintainer path is to review and either land or close #48442, or implement a focused core config surface for a narrow identity line/preamble override with clear precedence for `agents.defaults` and `agents.list[]`. That change should update config types, Zod and generated schemas, docs, gateway config mutation allowlists if user-editable, embedded/compaction/CLI prompt construction paths, and focused tests proving SOUL.md interaction plus per-agent/default precedence.
