## Summary

Keep open. Current `main` does not implement the PR's stated UI fallback: usage metrics still ignore sessions that have `dailyMessageCounts` but no top-level `messageCounts`, and the usage view actively recomputes those aggregates from per-session data. The PR's requested behavior is therefore not already on `main`.
