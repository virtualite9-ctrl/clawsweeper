## Summary

Current main implements the requested macOS LaunchAgent fix: generated gateway service environments now use a durable OpenClaw state-dir temp path on darwin instead of freezing the install-time `/var/folders/...` TMPDIR, install planning drops stale TMPDIR from existing service envs, and launchd creates the temp directory before bootstrap. The fix is recorded in the unreleased changelog and was introduced by d7c173b6945687985a60842e500aeabec522f060.

## Best Possible Solution

Close as implemented on main. Keep the current state-dir TMPDIR approach, ship it in the next release, and rely on `openclaw gateway install` or update-driven reinstall to rewrite stale LaunchAgent plists.

## Evidence

- **Darwin TMPDIR implementation:** `resolveServiceTmpDir` returns `path.join(resolveGatewayStateDir(env), "tmp")` on macOS, so LaunchAgent service envs no longer reuse the host session `/var/folders/...` TMPDIR except as a fallback when HOME/state-dir resolution fails.
