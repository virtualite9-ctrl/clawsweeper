## Summary

Current `main` still serializes a concrete `TMPDIR` into the macOS gateway LaunchAgent plist, so a migrated machine can retain a stale `/var/folders/...` path. I did not find a matching fix or a doctor audit/repair for this drift, so the issue should stay open.

## Evidence

- **Service env still includes TMPDIR:** `buildServiceEnvironment` puts `TMPDIR` into the managed service environment, sourcing it from the host env or `os.tmpdir()`. That means install-time temp paths are still captured for supervised services.
  - file: [src/daemon/service-env.ts:323](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/daemon/service-env.ts#L323)
