## Summary

Keep #32868 open. Current main still has the same-channel block-streaming delivery gap: tool-start flushing waits for the block-reply pipeline, but the same-channel block callback only enqueues dispatcher delivery and returns before actual channel delivery completes. The linked community workaround is useful defensive plugin behavior, but it does not replace the missing core delivery contract.

## Best Possible Solution

Keep #32868 open as the canonical core bug. The maintainer path is to make same-channel block delivery awaitable, await it before tool execution continues for blockStreamingBreak: "text_end", update dispatcher/SDK/plugin mocks and ACP call sites consistently, and add a regression test proving channel delivery completes before tool_execution_start proceeds.
