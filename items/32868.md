## Summary

Keep #32868 open. Current main still has the same-channel block-streaming delivery gap: tool-start flushing awaits the block-reply callback/pipeline, but the same-channel callback only queues dispatcher delivery and returns before channel delivery completes.

## Best Possible Solution

Keep #32868 open as the canonical core bug. The likely maintainer path is to make same-channel block delivery return an awaitable delivery result, await it from dispatch-from-config before tool execution proceeds, update dispatcher/SDK/mocks/ACP call sites, and add a regression test proving blockStreamingBreak: "text_end" channel delivery completes before tool_execution_start continues.
