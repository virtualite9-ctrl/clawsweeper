## Summary

Keep open. Current main has useful adjacent fixes for stale packaged dist files and bundled plugin runtime-dependency staging, but it still does not prove, detect, repair, or fail fast on byte-level mutation of an expected installed `dist/run-main-*.js` chunk like the reporter's corrupted file.

## Best Possible Solution

Keep this issue open as a packaging/runtime integrity investigation. The maintainer path is to reproduce or rule out the 2026.4.5 WSL/global-install failure, audit update/postinstall/restart/logging/runtime-dependency flows for writes under the installed package root, and either prove OpenClaw cannot mutate installed core `dist` chunks at runtime or add a content-hash inventory with startup/update/doctor verification and clear reinstall or repair guidance for corrupted expected chunks.
