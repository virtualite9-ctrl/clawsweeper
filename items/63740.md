## Summary

Keep open. Current main has adjacent stale-dist pruning and installed package path-inventory verification, but it still does not prove, repair, or fail fast on in-place mutation of an expected installed `dist/run-main-*.js` chunk like the reporter’s `tail -n 200 ... logawaitprogram.parseAsync` corruption.

## Best Possible Solution

Keep this issue open as a packaging/runtime integrity investigation. The best maintainer path is to reproduce or rule out the 2026.4.5 WSL/global-install path, audit runtime/update/postinstall/logging flows for writes under the installed package root, and either prove OpenClaw never mutates installed core `dist` files at runtime or add content-hash verification with clear reinstall/repair guidance for corrupted expected chunks.
