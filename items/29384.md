## Summary

Keep #29384 open. Current main still does not expose Docker sandbox GPU, IPC, device, privileged, cap-add, or raw extra Docker create-arg support. The relevant type, strict schema, resolver, Docker create-arg builder, tests, and public docs only cover existing hardening/resource/network/env/bind fields. OpenShell has a separate GPU flag, but that is a different backend and does not satisfy Docker `--gpus all` or `--ipc=host`.

## Best Possible Solution

Keep the issue open until a maintainer-approved Docker sandbox extension lands. A good implementation would add explicit, security-reviewed fields such as `sandbox.docker.gpus` and `sandbox.docker.ipc`, or a tightly constrained extra-create-args mechanism, then update `SandboxDockerSettings`, the strict Zod schema, config resolution and hashing, security validation/audit messaging, `buildSandboxCreateArgs`, public docs, and regression tests. The docs/UI/config surface should make the isolation tradeoff clear.
