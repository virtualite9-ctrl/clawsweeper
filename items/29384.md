## Summary

Keep #29384 open. Current main at 7e13f3f5140168e31027b39a9855ade3fadf24d5 still does not expose Docker sandbox `--gpus all`, `--ipc=host`, `capAdd`, `--device`, `--privileged`, or raw extra Docker create-arg support. The related #53108 is an open capAdd-only PR in the provided context and would not satisfy the central GPU/IPC request even if merged.

## Best Possible Solution

Keep the issue open until a maintainer-approved Docker sandbox extension lands. A good path is to add explicit, security-reviewed config fields such as `sandbox.docker.gpus` and `sandbox.docker.ipc`, or a constrained extra-create-args mechanism, then update the TypeScript config type, strict Zod schema, resolver/hash behavior, runtime validation/audit messaging, `buildSandboxCreateArgs`, public docs, and regression tests. #53108 can remain related prior work for capAdd, but it is not the canonical fix for this issue.
