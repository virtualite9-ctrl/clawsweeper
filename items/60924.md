## Summary

Current `main` has a new guard for the direct `sessions_send` inline reply path, but the follow-up announce/reply-back paths still read the latest assistant text from a reused session without a pre-run baseline. That means the stale previous-round result described in #60924 is not clearly fixed on `main`, so this should stay open.

## Evidence

- **Direct `sessions_send` path now snapshots a baseline:** Before starting the nested run, `sessions_send` captures the current assistant reply snapshot and passes it into the post-run waiter so unchanged old replies are suppressed for the inline `reply` field.
  - file: [src/agents/tools/sessions-send-tool.ts:257](https://github.com/openclaw/openclaw/blob/afc0c32bd0eb9f984fcf5bcf26c9ea821834b74d/src/agents/tools/sessions-send-tool.ts#L257)
  - command: `nl -ba src/agents/tools/sessions-send-tool.ts | sed -n '250,390p'`
  - sha: [afc0c32bd0eb](https://github.com/openclaw/openclaw/commit/afc0c32bd0eb9f984fcf5bcf26c9ea821834b74d)
- **Wait helper only returns text when the assistant fingerprint changed:** `waitForAgentRunAndReadUpdatedAssistantReply` compares the post-run assistant fingerprint with the captured baseline and returns `replyText: undefined` when they match.
  - file: [src/agents/run-wait.ts:152](https://github.com/openclaw/openclaw/blob/afc0c32bd0eb9f984fcf5bcf26c9ea821834b74d/src/agents/run-wait.ts#L152)
  - command: `nl -ba src/agents/run-wait.ts | sed -n '84,180p'`
  - sha: [afc0c32bd0eb](https://github.com/openclaw/openclaw/commit/afc0c32bd0eb9f984fcf5bcf26c9ea821834b74d)
- **A2A follow-up still drives announce/reply steps through `runAgentStep`:** The reply-back loop and final announce both call `runAgentStep`, so those paths depend on that helper's reply-selection behavior rather than the newer `sessions_send` baseline guard.
  - file: [src/agents/tools/sessions-send-tool.a2a.ts:90](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/src/agents/tools/sessions-send-tool.a2a.ts#L90)
  - command: `nl -ba src/agents/tools/sessions-send-tool.a2a.ts | sed -n '1,170p'`
  - sha: [6bc0dc8fb671](https://github.com/openclaw/openclaw/commit/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440)
- **`runAgentStep` has no pre-run baseline snapshot:** `runAgentStep` starts an agent run and immediately calls `waitForAgentRunAndReadUpdatedAssistantReply` without supplying any baseline, so a reused session can still surface the last visible assistant reply from an earlier round if the current step produces no new assistant text.
  - file: [src/agents/tools/agent-step.ts:31](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/src/agents/tools/agent-step.ts#L31)
