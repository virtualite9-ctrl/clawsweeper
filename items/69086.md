## Summary

Keep open. Current main still has the broad session-history guard described in the issue, the retry continuation text is still hard-coded, and the related PR #69109 was closed unmerged rather than shipped. The issue is recent, author is not protected, and it contains concrete reproduction/fix detail, so it remains actionable.

## Best Possible Solution

Keep the issue open. The best path is a narrow code PR that changes `sessionFileHasContent` to mean assistant-after-latest-user with regression coverage for user→assistant→user and assistant-after-latest-user, followed by a separate maintainer decision on whether the retry prompt override should be config, plugin hook, or another supported runtime seam.
