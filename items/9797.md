## Summary

Keep #9797 open. Current main still lacks the requested agent-facing `queue_status` tool or equivalent stable queue snapshot surface for intelligent subagent dispatch. The related PR #43618 is useful related work, but its provided body explicitly scopes itself to operator diagnostics and says it does not add an in-agent tool.

## Best Possible Solution

Implement a shared read-only queue snapshot provider in core, then expose it through an agent-facing `queue_status` tool with catalog/profile/policy registration, permissions, docs, and focused tests. If operator diagnostics lands first, reuse that backend and keep this issue focused on the in-agent dispatch surface.
