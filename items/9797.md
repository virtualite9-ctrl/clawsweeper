## Summary

Keep #9797 open. Current main still does not provide an agent-facing `queue_status` tool, a published tool-catalog entry for it, or a stable core queue snapshot/RPC/CLI surface that agents could use for intelligent dispatch. The prior issue discussion and PR #43618 are relevant, but the provided PR context explicitly scopes #43618 to operator diagnostics and says it does not add the in-agent tool requested here.

## Best Possible Solution

Keep #9797 open. The best product path is to implement a shared read-only queue snapshot provider in core, then expose it through the requested agent-facing `queue_status` tool with tool catalog, policy/permission, docs, and tests. The operator CLI/RPC surface discussed in the comments can share the same backend; if PR #43618 becomes the canonical backend, use this issue to track the remaining agent-tool consumer until it ships.
