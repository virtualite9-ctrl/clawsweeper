## Summary

Keep open. Current `main` does not implement this PR's inbound Ollama fix: `buildAssistantMessage` still copies `toolCall.function.arguments` through unchanged, so stringified `tool_calls[].function.arguments` would remain strings instead of normalized objects. The latest release `v2026.4.23` has the same code, and current tests only cover object arguments on the inbound `buildAssistantMessage` path.
