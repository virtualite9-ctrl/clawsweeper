## Summary

Keep this PR open. Current main and the latest release still force Feishu thread replies when inbound `thread_id`/`root_id` is present, even with `replyInThread: "disabled"`, so the change is not implemented or obsolete. The related open PR #39507 is not an exact duplicate because it proposes different `root_id` semantics, and this PR still needs maintainer review for the unresolved DM/root_id routing concern.

## Best Possible Solution

Keep this PR open for maintainer review, or explicitly consolidate it with #39507 after deciding the intended Feishu `root_id` semantics. A current-main fix should update `extensions/feishu/src/bot-content.ts` and `extensions/feishu/src/reply-dispatcher.ts` together, cover thread_id-only disabled groups, root_id normal groups, topic-group/topic-sender routing, and p2p quote-reply behavior, then replace the current test that expects thread_id-only messages to force thread replies.
