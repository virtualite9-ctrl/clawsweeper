## Summary

Keep this PR open. Current main still forces Feishu thread replies from inbound `thread_id` or `root_id` even when `replyInThread` is disabled, and the current tests assert that opposite behavior. The related #39507 overlaps, but it is not an exact duplicate because it makes disabled config suppress both `thread_id` and `root_id`, while this PR intentionally preserves `root_id` as a hard thread signal.

## Best Possible Solution

Keep this PR open for maintainer review or explicitly consolidate it with #39507 after deciding Feishu root/thread semantics. A current-main fix should update `extensions/feishu/src/bot-content.ts` and `extensions/feishu/src/reply-dispatcher.ts` together, replace the thread-id-only disabled test expectation, add regression coverage for root-id normal groups, topic-group/topic-sender routing, and p2p quote replies, and keep the diff limited to Feishu channel behavior.

## Risks / Open Questions
