## Summary

Current main still has the reported macOS GUI limitation. The macOS paperclip picker is explicitly image-only, and the shared Swift chat view model still rejects non-image attachments. This remains actionable core macOS/shared app work, not a duplicate or ClawHub-only request.

## Best Possible Solution

Keep this issue open. The likely fix belongs in the shared Swift chat UI used by macOS: broaden the paperclip picker from image-only to supported file attachments, replace the `addImageAttachment`-only loading path with a generic attachment path that still generates image previews when possible, reject unsupported video or unsafe types consistently, align limits/types with `ui/src/ui/chat/attachment-support.ts` and the Gateway `chat.send` attachment contract, and add focused Swift coverage or smoke verification for PDF/TXT/DOCX plus image parity.
