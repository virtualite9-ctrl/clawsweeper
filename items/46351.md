## Summary

Close #46351 as a duplicate of #41459. Current main does not implement `agents.list[].discord.webhook` for ordinary Discord route replies; it only has webhook persona delivery for bound thread sessions. #41459 is open and tracks the same remaining user problem: one Discord bot token, multiple bound agents, and no distinct visible Discord identity per routed agent/channel.

## Best Possible Solution

Consolidate the remaining Discord visible-identity work in #41459. The eventual fix should live in the Discord plugin/outbound delivery path, likely reusing the existing webhook `username`/`avatar_url` sender and `OutboundIdentity` plumbing, and should avoid adding channel-specific config to core agent schema unless a real plugin API gap is identified.
