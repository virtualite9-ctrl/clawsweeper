## Summary

Keep open. Current main still does not visibly show the active config file path in Control UI Overview or prominently in Settings/Config. The backend and config snapshot already expose the path for authorized flows, but the Overview render path ignores it and Config only uses it as the Open button tooltip. Related PR #54025 was closed unmerged, so it does not resolve the request.

## Best Possible Solution

Add a visible, copyable active config path field in the Control UI Overview snapshot card or Settings/Config header, sourced from the existing admin-scoped hello snapshot or config.get snapshot path. Add focused UI coverage proving the path is rendered for authorized Control UI sessions and that low-privilege snapshots still do not expose sensitive paths.
