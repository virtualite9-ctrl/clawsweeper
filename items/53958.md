## Summary

Keep open. The linked PR #54025 was closed unmerged, and current main exposes the active config path through backend/admin data paths but still does not visibly show it in the Control UI Overview. The Config view only uses the path as the hover title for the Open button, which does not satisfy the request to show the path prominently for troubleshooting the running instance.

## Best Possible Solution

Add a visible, copyable active config path row or compact field in the Control UI Overview snapshot card or Config/Settings header, using the existing admin-scoped hello snapshot or config.get snapshot path. Add focused UI coverage that proves the path is rendered for authorized Control UI sessions and not leaked to low-privilege sessions.
