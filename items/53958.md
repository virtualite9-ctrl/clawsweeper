## Summary

Keep this issue open. Current main still does not visibly show the active config file path in Control UI Overview or prominently in Settings/Config. The backend already exposes the resolved path to admin-scoped sessions and `config.get` includes a path, but the Overview render path does not consume it and Settings/Config only uses it as the Open button tooltip. Related PR #54025 was closed unmerged, so it is not a shipped fix.

## Best Possible Solution

Add a visible, copyable active config path in Control UI, preferably in the Overview snapshot card and/or the Settings/Config header. Source it from the existing admin-scoped hello snapshot or `config.get` snapshot path, and add focused UI coverage proving it renders for authorized sessions while low-privilege snapshots still omit sensitive path fields.
