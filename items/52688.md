## Summary

Issue #52688 proposes a new core predictive-scheduling/autonomy subsystem, but current main explicitly pushes optional autonomy/orchestration features to plugins and ClawHub rather than OpenClaw core. The repo already provides cron, heartbeat, hooks, standing orders, and plugin-owned memory scheduling as the supported building blocks for proactive behavior.

## Risks / Open Questions

- A future narrowly-scoped core seam for telemetry or scheduling hints could still be useful, but this issue asks for a broad core subsystem rather than such a seam.
- Some concrete use cases here may still be worth pursuing as external plugins or skills built on cron, heartbeat, hooks, and standing orders.

## Close Comment

Closing after a Codex review. This is a useful direction, but it fits OpenClaw's plugin/skill surface rather than core. Current project scope says optional capability should usually ship as plugins, new skills should go to ClawHub first, and heavy orchestration layers that duplicate existing agent/tool infrastructure are out of scope for core (`VISION.md:54-70`, `VISION.md:99-107`). Current main already provides the intended building blocks through cron, heartbeat, hooks, standing orders, and plugin-owned background memory scheduling (`docs/automation/index.md:10-103`, `docs/gateway/heartbeat.md:13-78`, `docs/concepts/dreaming.md:10-124`). If someone wants predictive pre-warming or adaptive scheduling, the maintainable path is a ClawHub plugin or skill built on those surfaces, not a new core `src/autonomy/*` subsystem.
