## Summary

Current `main` already refreshes a loaded gateway service during `openclaw update` by rerunning the gateway install path before restart, and targeted tests cover both the loaded-service refresh and fallback behavior. The change appears to be on `main` after the latest tagged release in this checkout, so it is implemented on `main` but not yet attributable to a shipped release tag here.

## Evidence

- **Updater now reruns gateway install during update:** `refreshGatewayServiceEnv()` invokes `gateway install --force` from the updated install root before restart, which is the exact refresh path requested in the issue.
  - file: [src/cli/update-cli/update-command.ts:256](https://github.com/openclaw/openclaw/blob/0ec3b79c07973e746328a1d6e1be5057096029af/src/cli/update-cli/update-command.ts#L256)
  - sha: [0ec3b79c0797](https://github.com/openclaw/openclaw/commit/0ec3b79c07973e746328a1d6e1be5057096029af)
- **Refresh is enabled for already-loaded gateway services:** After a successful update, the updater checks whether a gateway service is loaded; if so, it prepares the restart script and sets `refreshGatewayServiceEnv = true`, then passes that into `maybeRestartService()`.
  - file: [src/cli/update-cli/update-command.ts:1376](https://github.com/openclaw/openclaw/blob/0ec3b79c07973e746328a1d6e1be5057096029af/src/cli/update-cli/update-command.ts#L1376)
  - sha: [0ec3b79c0797](https://github.com/openclaw/openclaw/commit/0ec3b79c07973e746328a1d6e1be5057096029af)
- **Service install still rewrites versioned metadata:** Gateway service installs derive `Description=` from `OPENCLAW_SERVICE_VERSION`, and the generated service environment sets `OPENCLAW_SERVICE_VERSION` from the current runtime `VERSION`, so rerunning install refreshes the stale systemd metadata described in the report.
  - file: [src/daemon/service-env.ts:282](https://github.com/openclaw/openclaw/blob/0ec3b79c07973e746328a1d6e1be5057096029af/src/daemon/service-env.ts#L282)
  - sha: [0ec3b79c0797](https://github.com/openclaw/openclaw/commit/0ec3b79c07973e746328a1d6e1be5057096029af)
- **Description format is version-backed:** `resolveGatewayServiceDescription()` formats the service description from profile + version, matching the stale `Description=OpenClaw Gateway (v...)` symptom reported in the issue and showing why reinstalling the unit fixes it.
  - file: [src/daemon/constants.ts:78](https://github.com/openclaw/openclaw/blob/0ec3b79c07973e746328a1d6e1be5057096029af/src/daemon/constants.ts#L78)
  - sha: [0ec3b79c0797](https://github.com/openclaw/openclaw/commit/0ec3b79c07973e746328a1d6e1be5057096029af)
- **Regression tests cover the reported path:** `updateCommand` tests assert that when a service is already installed, the updater calls `runDaemonInstall({ force: true })` before restart, and that it falls back cleanly if the refresh cannot complete.
  - file: [src/cli/update-cli.test.ts:1500](https://github.com/openclaw/openclaw/blob/0ec3b79c07973e746328a1d6e1be5057096029af/src/cli/update-cli.test.ts#L1500)
  - sha: [0ec3b79c0797](https://github.com/openclaw/openclaw/commit/0ec3b79c07973e746328a1d6e1be5057096029af)
