## Summary

Keep this issue open. Current main still preserves Slack thread session isolation, exposes no `session.threadIsolation` or `session.channelIsolation` config, and the linked implementation PRs were closed unmerged. The recent Slack routing work fixes root/reply normalization within a single Slack thread, but it does not provide the requested opt-in shared session across Slack threads.

## Best Possible Solution

Keep this issue open. The best maintainer path is to decide the product shape for opt-in shared Slack thread/channel sessions, then implement it in the Slack-owned routing surface using the existing `resolveThreadSessionKeys` `useSuffix` seam plus config schema/help/docs updates, inbound and outbound session-key alignment, per-thread history keys, starter/history gating based on per-thread state rather than shared session existence, and focused regression tests. If maintainers want one umbrella, explicitly consolidate this into #19929 instead of closing it as stale.
