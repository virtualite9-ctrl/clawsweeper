## Summary

Keep #38204 open. Current main supports standard Grok 4.20 xAI models but still intentionally excludes Grok 4.20 multi-agent IDs, while official xAI docs describe `grok-4.20-multi-agent` as a live Responses API model with distinct multi-agent behavior. The issue discussion also shifted from an informational workaround to user requests for native multi-agent support, so this needs xAI-provider maintainer/product judgment rather than conservative cleanup close.

## Best Possible Solution

Keep the issue open for xAI-provider maintainer judgment. A concrete path should verify the current xAI multi-agent Responses contract, decide whether support belongs in the bundled `xai` plugin or a separate ClawHub/provider plugin, then update model resolution/catalog behavior, request shaping such as reasoning/tool handling, provider docs, and regression tests for both standard Grok 4.20 refs and multi-agent refs.
