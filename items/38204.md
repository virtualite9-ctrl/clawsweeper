## Summary

Keep #38204 open. Current main supports standard Grok 4.20 xAI models, but it still deliberately excludes Grok 4.20 multi-agent model IDs and documents that limitation as requiring a different upstream API surface. The issue discussion also moved beyond an informational workaround into requests for native multi-agent endpoint support, so this is not a conservative cleanup close.

## Best Possible Solution

Keep the issue open for xAI-provider maintainer judgment. The concrete path is to decide whether Grok multi-agent support belongs in the bundled `xai` plugin or an external ClawHub/provider plugin, then implement the appropriate transport/request shaping, model catalog behavior, docs, and regression tests without weakening the existing standard Grok 4.20 path.
