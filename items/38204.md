## Summary

Keep #38204 open. Current main supports the standard Grok 4.20 reasoning/non-reasoning xAI models, but it still explicitly excludes Grok 4.20 multi-agent IDs from dynamic resolution and modern-model classification. The docs also list the reported multi-agent model as unsupported because it needs a different upstream API surface, and the issue comments ask for native multi-agent endpoint/model support rather than only a catalog alias.

## Best Possible Solution

Keep the issue open for xAI-provider maintainer judgment. If OpenClaw wants native support, implement it in the bundled `extensions/xai` provider with the correct upstream multi-agent transport, request shaping, catalog/auth behavior, docs, and regression tests. If maintainers decide beta multi-agent support should not be bundled, resolve that explicitly by pointing contributors toward a ClawHub/provider-plugin path.
