## Summary

Keep #38204 open. Current main supports standard Grok 4.20 xAI model IDs but still explicitly excludes Grok 4.20 multi-agent IDs, while current xAI docs show `grok-4.20-multi-agent` as a Responses API model with special limitations. ([docs.x.ai](https://docs.x.ai/developers/model-capabilities/text/multi-agent))

## Best Possible Solution

Keep the issue open for xAI-provider maintainer judgment. The concrete path is to either extend the bundled `extensions/xai` provider to support `grok-4.20-multi-agent` via Responses-specific catalog/request shaping, docs, and regression tests, or deliberately split it into a ClawHub/provider plugin if maintainers decide the beta multi-agent surface should stay outside bundled xAI support.
