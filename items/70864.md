## Summary

Keep #70864 open. Current main still does not implement scoped mention-pattern policy: it has only the existing `mentionPatterns` regex fallback, no global/agent/provider allow-deny mode, no policy resolver in the SDK/runtime surface, and bundled channel handlers still build configured regexes directly. The provided PR context shows #70864 is the active clean successor to closed dirty predecessor #70846, with recent author follow-up and green CI claims, not a duplicate to close.

## Best Possible Solution

Keep #70864 open as the active implementation path. The best maintainer path is to review the latest head, verify the scoped config schema, SDK/runtime helper, docs, generated metadata, and every bundled channel use the same routed-agent and conversation-id policy semantics, then merge if CI and changed-surface tests are green. Do not close as implemented, duplicate, ClawHub, or stale because current main lacks the requested core capability and #70864 supersedes the closed dirty predecessor #70846.
