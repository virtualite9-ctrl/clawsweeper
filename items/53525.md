## Summary

Keep open. Current main has generic improvements for empty errored turns and user-facing `errorMessage` rendering, but I did not find Qianfan-specific error response normalization. The provider still uses the generic OpenAI-compatible completions path, and that path does not parse Qianfan/Baidu top-level error bodies before deciding whether to surface an error.

## Best Possible Solution

Keep this issue open and fix it in the Qianfan-owned provider surface or a narrow OpenAI-compatible error-normalization seam. The fix should preserve Qianfan/Baidu top-level `code`/`message` and legacy `error_code`/`error_msg` from non-2xx responses and stream error events into `errorMessage`, add regression coverage for the Qianfan shape, and keep the existing generic empty-error retry as a fallback rather than the primary behavior.
