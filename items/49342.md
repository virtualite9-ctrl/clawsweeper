## Summary

Current `main` already has a plugin hook/middleware API that covers the request: pre-tool-call blocking/modification/approval, outbound message filtering/cancellation, and per-agent scoping via agent tool policy plus hook context.

## Close Comment

Closing from a Codex review because this is already implemented on `main`, and the shipped `v2026.4.22` tree already contains it. OpenClaw exposes a pre-tool middleware surface via `before_tool_call` with `params`, `block`, and `requireApproval` in `src/plugins/hook-types.ts:336`; those decisions are enforced before execution in `src/plugins/hooks.ts:781` and `src/agents/pi-tools.before-tool-call.ts:248`. Output filtering is also available before delivery via `message_sending` content rewrite/cancel in `src/plugins/hook-message.types.ts:45` and `src/plugins/hooks.ts:733`. The public docs already describe these hooks in `docs/plugins/building-plugins.md:182` and `docs/concepts/agent-loop.md:98`. The remaining gap is only ergonomics: the exact `tools.hooks.preSend` config shape from the request is not present, but the underlying middleware/guardrail capability is.
