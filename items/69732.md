## Summary

Keep PR #69732 open. Current main still has an inconsistent exec approvals transport contract: gateway/node GET responses can carry stored allowlist `source` metadata, but gateway SET validation rejects any allowlist entry with `source` before normalization/save. The PR discussion also identified the important constraint that simply stripping `source` would break durable allow-always approvals, so the remaining work is a real core fix rather than cleanup.

## Best Possible Solution

Keep the PR open and revise it into a consistent core transport contract for exec approvals: either allow `source: "allow-always"` in the exec approvals file schema used by GET/SET, with focused regression coverage for Control UI and node round-trips, or explicitly redact it from outbound snapshots and rehydrate existing durable metadata before saving. The fix belongs in the gateway/node exec approvals transport path plus protocol tests, not in external cleanup.
