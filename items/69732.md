## Summary

Keep #69732 open. Current main still has the underlying exec approvals round-trip mismatch from #69698: gateway/node snapshots can include internal allowlist `source`, but `exec.approvals.set` rejects that field. The PRâ€™s latest direction preserves `source` to avoid durable approval data loss, which is correct as far as it goes, but the diff does not update the wire schema or save path, so it does not yet fix the reported Control UI save failure.

## Best Possible Solution

Keep the PR open for changes, or supersede it with a narrower maintainer patch. The fix should make `exec.approvals.get` -> edit -> `exec.approvals.set` safe for both gateway and node targets while preserving durable allowlist metadata and redacting socket tokens. Concretely, update the protocol/UI contract to accept `source: "allow-always"` if it is now public transport metadata, or strip it from client payloads and merge it back on the SET path from the current snapshot. Add regression coverage for gateway and node approval snapshot round-trips with `source`, `argPattern`, usage metadata, and socket token redaction.
