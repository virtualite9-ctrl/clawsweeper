## Summary

Keep open. Current main still does not expose tool execution to managed/internal HOOK.md handlers, and this PR remains a focused implementation attempt rather than shipped, clearly superseded, or auto-closeable cleanup. The remaining decision is product/API ownership: whether internal hooks should gain a typed tool event or tool interception should stay plugin-only.

## Best Possible Solution

Keep this PR open until maintainers either accept a narrow internal tool:after_call bridge or explicitly decide that managed/internal hooks should remain command/session/agent/gateway/message-only. If implemented, rebase onto current main, add typed tool event support in src/hooks/internal-hook-types.ts and related exports, emit exactly once from handleToolExecutionEnd using ctx.params.sessionKey and adjusted afterToolCallArgs, and add docs plus focused regression coverage. If maintainers choose a broader before/after tool-call contract, use #13364 as the canonical design tracker and update this PR accordingly or close it only after that decision is explicit.
