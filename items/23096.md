## Summary

Keep open. Current main has the generic SecretRef env/file/exec architecture, but it still lacks the Bitwarden/Vaultwarden docs, resolver, package inclusion, and tests proposed here. The current plugin manifest/SDK surface also does not yet expose the secret-provider plugin seam described in maintainer discussion, so this is not a safe ClawHub close yet.

## Best Possible Solution

Keep this PR open as the focused Bitwarden/Vaultwarden follow-up. The best path is to align it with the maintainer-directed secret-provider plugin architecture once #72548 or equivalent lands, likely as an extensions/secrets-bitwarden plugin declaring a bw secret-provider contract, with docs, tests, and packaging reviewed against current SecretRef exec hardening. Until that seam lands, avoid closing it as ClawHub-only because current main does not expose the missing secret-provider extension API.
