## Summary

Keep open. Current main supports semantic `presentation` cards for Feishu proactive sends and dispatches `message(action=send)` through the Feishu plugin, but it still treats raw Feishu interactive-card JSON supplied in the `message` string as plain text/post content rather than routing it to the interactive card sender.

## Best Possible Solution

Keep this issue open for Feishu-owned maintainer handling. The narrow implementation path is to add a conservative Feishu card JSON detector in `extensions/feishu/src/channel.ts` for `send`/`thread-reply` message strings and route valid native cards to `sendCardFeishu`, with a regression test for proactive send. The alternative is an explicit maintainer decision and docs update that raw provider-native Feishu card JSON in `message` is unsupported and users should send semantic `presentation` payloads instead.
