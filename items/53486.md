## Summary

Keep open. Current main supports Feishu interactive cards through the shared `presentation` contract and now routes `message(action=send)` through the Feishu plugin action handler, but a raw Feishu card JSON string passed in `message` is still treated as text and sent with Feishu `msg_type: "post"`, not `"interactive"`. The related closed items are adjacent validation/card-delivery work, not a canonical fix for this regression.

## Best Possible Solution

Keep this issue open for maintainer/product handling. The likely code path is Feishu-owned: either add a conservative compatibility detector in `extensions/feishu/src/channel.ts` for `message` strings that are valid Feishu interactive-card JSON and route them to `sendCardFeishu`, with a regression test for proactive `message(action=send)`, or explicitly document that raw provider-native card JSON is no longer supported and that users must use `presentation` for proactive cards.
