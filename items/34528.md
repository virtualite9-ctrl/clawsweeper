## Summary

Keep #34528 open. Current main still produces Feishu synthetic reaction message IDs with the `:reaction:<emoji>:<uuid>` suffix and still passes message IDs into Feishu API boundary paths without a shipped normalizer. The provided GitHub context shows #39558 as the active open PR candidate, while earlier fix attempts were closed unmerged, so the issue is not implemented on main.

## Best Possible Solution

Keep the issue open until a Feishu plugin fix lands on main. The best implementation is a narrow normalizer for OpenClaw’s synthetic `:reaction:<emoji>:<uuid>` suffix, applied at Feishu API boundaries for replies, typing/reaction APIs, media/resource calls, streaming cards, edit/get paths, and tool-driven message IDs as applicable, with regression tests proving clean IDs are sent externally while suffixed synthetic IDs remain available internally. PR #39558 is the current candidate to review or supersede.
