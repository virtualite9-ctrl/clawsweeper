## Summary

Keep #34528 open. Current main still creates Feishu synthetic reaction message IDs with the reported `:reaction:<emoji>:<uuid>` suffix, preserves that suffixed ID through parsing, can use it as the reply target, and passes message IDs unchanged into Feishu API path parameters. The provided discussion shows several closed unmerged fix attempts and #39558 as the active open fix candidate, so the bug is not implemented on main.

## Best Possible Solution

Keep #34528 open until a Feishu plugin fix lands on main. The best implementation path is to review, update, or supersede #39558 with a narrow `stripFeishuReactionSuffix` helper applied at Feishu API boundaries, plus regression tests proving synthetic reaction IDs remain internal while clean base `om_...` IDs are sent to Feishu.
