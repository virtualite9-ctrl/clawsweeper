## Summary

Keep PR #38650 open. Current main still does not implement the linked user request for workspace bootstrap files such as SOUL.md to follow symlinks to shared files outside the workspace. The current bootstrap loader still treats guarded-read failures as missing, the boundary resolver still rejects symlink escapes, and the PR’s fs-safe-only change is not the active layer for bootstrap injection. Related PR #40230 is a plausible successor, but it is still open/unmerged in the provided GitHub context and has not shipped on main.

## Best Possible Solution

Keep this open until a maintainer either lands a current-main fix or explicitly closes it in favor of a canonical successor such as #40230. The best implementation path is a targeted workspace bootstrap reader policy in readWorkspaceFileWithGuards/openBoundaryFile, with regression coverage for shared outside-workspace bootstrap symlinks, in-workspace symlink aliases, symlink loops, hardlink rejection, max-size enforcement, and Gateway agents.files behavior.
