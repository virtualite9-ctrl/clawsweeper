## Summary

Keep open. Current main has several restart hardening improvements, including OpenClaw-managed restart intent/drain behavior, restart lifecycle error copy, Feishu outbound routing, and single-session restart sentinel delivery. But it still does not implement the issue's requested Feishu-visible notification/fan-out for in-flight sessions interrupted by a direct supervisor/systemctl restart. Plain SIGTERM remains a stop path, not a restart drain path, and post-start main-session recovery resumes with `deliver: false` rather than notifying the original Feishu conversation.

## Best Possible Solution

Keep the issue open until OpenClaw either implements a generic restart-interruption path for external channel turns or explicitly documents that direct supervisor/systemctl restarts are outside the supported no-loss boundary. The best implementation would persist affected delivery contexts before shutdown or while marking aborted sessions, then on startup send a Feishu-visible interruption notice or resume-and-deliver through the original route, with regression coverage for Feishu or a generic channel delivery contract.
