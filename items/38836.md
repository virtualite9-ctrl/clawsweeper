## Summary

Keep open. Current main still has restart hardening, including OpenClaw-managed restart intent/drain behavior, restart lifecycle error copy, Feishu route persistence, and single-sentinel routed delivery. But it still does not implement the issue's central request: Feishu-visible notification or fan-out for in-flight sessions interrupted by a direct supervisor/systemctl restart. Plain SIGTERM remains a stop path unless OpenClaw wrote restart intent, and post-start main-session recovery resumes internally with deliver disabled.

## Best Possible Solution

Keep the issue open until OpenClaw either implements a generic external-channel restart interruption path or explicitly documents direct supervisor/systemctl restarts as outside the supported no-loss boundary. The best implementation would persist affected delivery contexts before shutdown or while marking aborted sessions, then on startup send a Feishu-visible interruption notice or resume-and-deliver through the original route, with regression coverage for Feishu or a generic channel delivery contract.
