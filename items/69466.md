## Summary

Current main has moved `openclaw security audit` off the old cold channel-runtime loading path, bounds Gateway RPC/probe work, and degrades SecretRef resolution with diagnostics, so the 2026.4.15 indefinite no-output hang report is already addressed on main.

## Best Possible Solution

Keep the current main implementation: `security audit` should stay on read-only metadata and diagnostic SecretRef resolution, with bounded Gateway probes/RPC teardown. Any future report should include the specific plugin/config/logs if a third-party audit collector still blocks.

## Evidence
