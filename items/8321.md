## Summary

Keep #8321 open. Current main still does not expose per-job `disableCache` or `contextPruning` for cron jobs, and isolated cron execution still feeds the embedded runner a config derived from global agent defaults plus the selected agent override. The related PR #8341 is closed unmerged, and the #24597 direct script/no-LLM request is a separate cost-reduction path rather than this per-job pruning/cache override.

## Best Possible Solution

Implement an additive per-job cron `contextPruning?: AgentContextPruningConfig` override using the existing `mode: "off" | "cache-ttl"` contract. Thread it through cron create/update/list schemas, persistence normalization, CLI/UI/macOS surfaces, and agent-tool interfaces, then merge it into `cfgWithAgentDefaults` before isolated cron calls the embedded runner. Add regression coverage proving a job-level `mode: "off"` overrides global or per-agent `agents.defaults.contextPruning.mode: "cache-ttl"`. Keep direct script/no-LLM cron execution separate.
