## Summary

Keep #8321 open. Current main still does not expose per-job `disableCache` or `contextPruning` for cron jobs, and the runtime still derives cache/pruning behavior from `agents.defaults.contextPruning`. The related PR #8341 was closed unmerged in the provided GitHub context, and the direct-script/no-LLM request #24597 is a separate capability rather than this per-job cache/pruning override.

## Best Possible Solution

Keep #8321 open. Implement an additive per-job cron `contextPruning?: AgentContextPruningConfig` override using the existing `mode: "off" | "cache-ttl"` contract, thread it through cron create/update/list schemas, normalization/store, CLI/UI/macOS and agent-tool surfaces, isolated-run config construction, and docs. Add regression coverage proving a job-level `mode: "off"` overrides global/per-agent `agents.defaults.contextPruning.mode: "cache-ttl"`. Keep direct script/no-LLM cron execution tracked separately.
