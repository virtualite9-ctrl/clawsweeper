## Summary

Keep open. Current main still lacks cursor-based `chat.history` pagination and the web UI has no load-older flow, so the central pagination work has not shipped. The partial `NO_REPLY` prefix leak appears covered in the current gateway live-chat path, and related PR #56383 also tracks pagination, but neither makes this active PR safely obsolete; the PR also has unresolved correctness feedback about duplicate pages.

## Best Possible Solution

Keep the PR open for maintainer review, but require changes before merge: coordinate with the existing open pagination PR #56383, compute the cursor from the raw index of the first retained message after sanitization and byte capping, add a regression test for sanitized mid-page drops, and consider splitting or dropping the UI-side partial `NO_REPLY` fix if the current gateway suppression is sufficient.
