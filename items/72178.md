## Summary

Keep open. Current main still lacks cursor-based chat.history pagination, and this PR is a concrete, recent implementation attempt rather than stale cleanup. It is not merge-ready as submitted because the PR review context identifies a cursor-overlap bug when sanitization drops messages, but that calls for revision or maintainer choice against overlapping PR #56383, not auto-closing #72178.

## Best Possible Solution

Keep #72178 open for maintainer review or author revision. The best path is a single canonical chat.history pagination design covering the TypeBox schema, gateway handler, docs, generated clients, Web UI load-older state/rendering, and regression tests. Cursor computation should be based on the raw index of the first message that survives sanitization and byte-budget trimming, with explicit tests for NO_REPLY/commentary drops, byte trimming, duplicate prevention, and the already-shipped gateway NO_REPLY lead-fragment behavior.
