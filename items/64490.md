## Summary

Keep open. Current main still emits zsh option descriptions inside double-quoted `_arguments` specs without escaping `$` or backticks, and current CLI descriptions still contain both characters. The PR remains a focused implementation candidate rather than obsolete cleanup.

## Best Possible Solution

Keep this PR open as the implementation candidate. A maintainer should review the escaping helper in `src/cli/completion-cli.ts`, keep the `$` and backtick regression assertions in `src/cli/completion-cli.test.ts`, verify single quotes remain literal in the double-quoted zsh context, and then run the targeted completion test plus a live zsh `openclaw models <TAB>` check before merging or requesting changes.
