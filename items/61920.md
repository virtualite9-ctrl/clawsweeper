## Summary

Current `main` already contains runtime-registry reuse and loader caching that prevent compatible startup/runtime loads from re-registering the same plugin repeatedly, and the shipped `v2026.4.23` tag already includes those guards plus regression tests.

## Evidence

- **Compatible active registry reuse on runtime/helper paths:** `resolveRuntimePluginRegistry()` now returns the existing active plugin registry when the load context matches, instead of triggering a fresh plugin load.
  - file: [src/plugins/loader.ts:1023](https://github.com/openclaw/openclaw/blob/8b76392e3e7934b1cdd6d461148263fd56141ec8/src/plugins/loader.ts#L1023)
  - sha: [8b76392e3e79](https://github.com/openclaw/openclaw/commit/8b76392e3e7934b1cdd6d461148263fd56141ec8)
