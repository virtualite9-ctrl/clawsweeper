## Summary

Keep open. Current main has several prompt-cost mitigations, including on-demand skill instructions, bounded skill catalogs, static tool profiles and allow/deny policy, bootstrap caps plus optional continuation-skip/never injection, and automatic compaction. It still does not implement the central request for automatic per-turn conditional tool/schema relevance routing, and default workspace bootstrap behavior still injects configured bootstrap files rather than a summary/index/on-demand model. The related #14785 is a narrower open tracker for tool-schema footprint, while this issue covers broader per-turn overhead across tools, skills, workspace files, and compaction.

## Best Possible Solution

Keep this issue open as the broad per-turn prompt overhead tracker. Use #14785 for narrower tool-schema footprint work, but solve this issue with a policy-preserving automatic relevance/schema-routing design plus a default or opt-in workspace bootstrap summary/index/on-demand mode. Preserve shipped mitigations: compact/on-demand skills, explicit tool profiles and allowlists, bootstrap caps/contextInjection controls, lightweight heartbeat/cron behavior, and compaction.
