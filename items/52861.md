## Summary

Keep open. Current main has useful mitigations for prompt cost, including compact/on-demand skill catalogs, explicit tool profiles/allowlists, bootstrap caps, lightweight bootstrap modes for some isolated runs, continuation-skip config, and compaction. It still does not implement the central request for automatic per-turn conditional tool/schema relevance routing or default summary/on-demand workspace bootstrap injection; absent explicit policy controls, the effective tool set is still built and sent with full schemas, and bootstrap files remain injected by default.

## Best Possible Solution

Keep this issue open as the broad per-turn prompt overhead tracker. Use #14785 for the narrower tool-schema footprint work, but solve this issue with a policy-preserving automatic tool relevance/schema-routing design plus a default or opt-in workspace bootstrap summary/index/on-demand mode. Preserve the shipped mitigations: compact skill catalogs, explicit toolsAllow/tool profiles, bootstrap caps/contextInjection controls, lightweight cron/subagent modes, and compaction.

## Risks / Open Questions

- Automatic conditional tool routing needs policy-preserving semantics for explicit allow/deny lists, owner-only and recovery tools, plugin tools, bundled MCP tools, provider schema constraints, and channel-specific capabilities.
- A default bootstrap summary/index mode could hide standing workspace instructions unless it has deterministic ordering, clear cache behavior, and a reliable path for the agent to read full files on demand.
- Closing this as implemented would lose the broader tracker while #14785 only covers the narrower tool-schema footprint and not workspace bootstrap summary/index behavior.
