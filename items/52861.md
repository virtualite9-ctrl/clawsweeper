## Summary

Keep this issue open. Current main has partial mitigations for skills and compaction, and #14785 remains the narrower canonical tracker for tool-schema footprint, but the broader request is still not implemented: default runs still build the full effective tool set unless explicit policy/allowlist inputs narrow it, full tool parameter schemas are still forwarded, and workspace bootstrap files still default to recurring injection with caps rather than an on-demand summary/index mode.

## Best Possible Solution

Keep this issue open as the broader per-turn overhead tracker. Use #14785 for the narrower tool-schema footprint work, but keep this item to design a policy-preserving automatic tool relevance/schema-routing path plus a better default or opt-in summary/index mode for recurring workspace bootstrap files. Any fix should preserve plugin compatibility, provider schema constraints, explicit allow/deny semantics, and prompt-cache stability.
