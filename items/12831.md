## Summary

Keep #12831 open. Current main has useful gateway plumbing for tool-event delivery, including `tool-events` and `session.tool`, but it still does not provide the requested first-class Activity panel / observer mode with independent UI state, per-connection verbose behavior, output visibility decoupled from `/verbose full`, or TUI split-pane observer support. Related #46604 and draft PR #60814 overlap, but they do not clearly supersede the full #12831 scope.

## Best Possible Solution

Keep #12831 open as the broad observer-mode tracker. Build on the existing `tool-events` and `session.tool` plumbing, add a dedicated Control UI activity panel with independent scroll/state, consume session-scoped tool events for late-joining observers, decouple observer output visibility from chat `/verbose full`, decide per-connection/tab verbose semantics, and add focused TUI/docs/tests. Treat #46604 as the narrower web split-panel follow-up and #60814 as related dashboard work unless maintainers explicitly mark one canonical for the full scope.
