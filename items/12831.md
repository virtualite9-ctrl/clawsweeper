## Summary

Keep #12831 open. Current main has useful live tool-event plumbing and inline tool cards, but it still lacks the requested separate activity/observer mode: Control UI routes tool activity into the chat thread, tool output remains tied to verbose=full, TUI visibility is still verbose-gated, and related #46604 is only a narrower open web split-panel tracker.

## Best Possible Solution

Keep #12831 open as the broad product/UX tracker for a first-class activity observer mode. Build on the existing tool-events and session.tool plumbing, but add a dedicated Control UI activity surface with independent scrolling, TUI split/observer support, output policy independent of /verbose, connection-scoped observer controls or a documented subscription path, and focused tests/docs. Treat #46604 as a narrower web split-panel follow-up unless maintainers explicitly make it canonical.
