## Summary

Keep #12831 open. Current main has useful tool-event plumbing and live tool cards, but it still does not implement the requested first-class activity observer mode: Control UI tool cards are still rendered in the chat stream, tool output remains coupled to `verbose=full`, `/verbose` is a session patch rather than per-connection observer state, and the TUI still suppresses tool visibility when verbose is off. Related #46604 is narrower and open; it does not supersede this broader tracker.

## Best Possible Solution

Keep #12831 open as the broad observer-mode tracker. Build on the existing `tool-events` and `session.tool` plumbing, but add a dedicated Control UI activity panel with independent scrolling, observer/output visibility not coupled to `/verbose full`, connection-scoped controls or a documented subscription workflow, TUI split/observer support, and focused tests/docs. Treat #46604 as a narrower web split-panel follow-up unless maintainers explicitly make it canonical.
