## Summary

Keep open. Current main still installs fish completions through a source block in `~/.config/fish/config.fish`; it does not copy `openclaw.fish` into fish's auto-loaded `~/.config/fish/completions/` directory. The directly related PR #27135 addressed the requested behavior but is closed unmerged, so it does not supersede the issue.

## Best Possible Solution

Continue tracking the issue. The likely fix belongs in the CLI completion runtime: make fish install copy the cached `openclaw.fish` to `~/.config/fish/completions/<binName>.fish`, update installed-state/doctor/setup checks and messaging for that target, clean legacy `config.fish` source blocks safely, and add focused regression tests plus docs. PR #27135 is useful prior art but is closed unmerged.
