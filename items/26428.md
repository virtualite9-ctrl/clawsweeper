## Summary

Current main still does not implement the fish-shell convention requested by #26428. `openclaw completion --install`, installed-state detection, doctor/setup/update flows, docs, and tests remain profile/source-block oriented for fish (`~/.config/fish/config.fish`) rather than installing/detecting `~/.config/fish/completions/<binName>.fish`. The directly related PR #27135 is useful prior art, but it was closed unmerged, so this issue still tracks real remaining work.

## Best Possible Solution

Keep #26428 open. The best fix is a fish-specific install/detect path in `src/cli/completion-runtime.ts` that writes or copies the cached fish completion to `~/.config/fish/completions/<binName>.fish`, recognizes that as installed, safely removes legacy OpenClaw source blocks from `config.fish`, updates doctor/setup/update messaging and https://docs.openclaw.ai/cli/completion, and adds focused regression tests. PR #27135 is the relevant closed prior-art implementation to review or rebase.
