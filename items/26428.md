## Summary

Keep open. Current main still installs fish completion by sourcing the cached `openclaw.fish` from `~/.config/fish/config.fish`, not by installing `openclaw.fish` into fish's auto-loaded `~/.config/fish/completions/` directory. Docs and doctor/setup/update hints still describe the profile-based path, and the direct attempted fix PR #27135 is closed unmerged in the provided GitHub context.

## Best Possible Solution

Continue tracking this issue. The likely fix belongs in the CLI completion runtime plus doctor/setup/update callers: install or copy fish completion to `~/.config/fish/completions/<binName>.fish`, detect that installed state there, safely remove legacy `config.fish` source blocks, update user-facing reload/docs text, and add focused regression tests. PR #27135 is useful prior art but cannot close the issue because it did not land.
