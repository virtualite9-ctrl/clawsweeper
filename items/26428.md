## Summary

Keep #26428 open. Current main still installs fish completion by sourcing the cached fish script from `~/.config/fish/config.fish`; it does not install `openclaw.fish` into fish's auto-loaded `~/.config/fish/completions/` directory. The linked implementation PR #27135 is useful prior art, but it was closed unmerged and has not shipped.

## Best Possible Solution

Keep the issue open. The best implementation is a fish-specific installer path in `src/cli/completion-runtime.ts` that writes or copies the cached fish completion to `~/.config/fish/completions/<binName>.fish`, detects that installed state there, safely removes legacy OpenClaw completion blocks/source lines from `config.fish` without leaving an orphan header, updates doctor/setup/update reload messaging plus `docs/cli/completion.md`, and adds focused regression tests. PR #27135 is relevant prior art but needs maintainer review/rebase because it was closed unmerged.
