## Summary

Current main still installs fish completions by adding an OpenClaw Completion source block to ~/.config/fish/config.fish, while the issue asks to install openclaw.fish into fish's auto-loaded ~/.config/fish/completions directory. The related fix PR #27135 is closed unmerged, so it does not supersede the issue.

## Best Possible Solution

Keep this issue open. Implement fish-specific completion installation in src/cli/completion-runtime.ts by writing or copying the cached fish script to ~/.config/fish/completions/<binName>.fish, detecting that installed state there, safely cleaning up legacy config.fish OpenClaw blocks, updating doctor/setup/update reload messaging and docs, and adding focused regression tests. PR #27135 is relevant prior art but needs rebasing/review because it did not merge.
