## Summary

Keep #26428 open. Current main still installs fish completion by sourcing the cached fish script from `~/.config/fish/config.fish`; it does not install `openclaw.fish` into fish's auto-loaded `~/.config/fish/completions/` directory. The related implementation PR #27135 was closed unmerged, so it is useful prior art but not a shipped fix.

## Best Possible Solution

Keep the issue open. The best implementation is a fish-specific installer path in `src/cli/completion-runtime.ts` that writes or copies the cached fish completion to `~/.config/fish/completions/<binName>.fish`, detects that installed state there, safely removes legacy OpenClaw completion blocks/source lines from `config.fish`, updates doctor/setup/update reload messaging and docs, and adds focused regression tests while preserving existing bash, zsh, and PowerShell profile behavior. PR #27135 is relevant prior art but needs maintainer review/rebase because it did not merge.
