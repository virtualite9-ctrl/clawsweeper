## Summary

Keep #26428 open. Current main still installs fish completion by writing an OpenClaw Completion source block into `~/.config/fish/config.fish`, not by installing `openclaw.fish` into fish's auto-loaded `~/.config/fish/completions/` directory. The prior fix PR #27135 is closed unmerged and does not supersede the issue.

## Best Possible Solution

Keep this issue open. The best implementation is a fish-specific path in `src/cli/completion-runtime.ts` that writes or copies the cached fish completion to `~/.config/fish/completions/<binName>.fish`, detects that installed state there, safely removes legacy OpenClaw completion blocks/source lines from `config.fish`, updates doctor/setup/update reload messaging and docs, and adds focused regression tests while preserving existing bash, zsh, and PowerShell profile behavior. PR #27135 is relevant prior art but needs rebasing/review because it did not merge.
