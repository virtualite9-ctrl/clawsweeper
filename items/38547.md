## Summary

Keep #38547 open. Current `main` still lacks native remote gateway delegation for `sessions_spawn`/`sessions_send`: the session tool schemas do not expose `gatewayUrl` or `gatewayToken`, their execute paths do not read those fields, and the native subagent/ACP/send dispatch paths call the current gateway without URL/token override options. The existing issue discussion is concrete and the prior ClawSweeper keep-open comment remains accurate against current `main`.

## Best Possible Solution

Keep #38547 open until maintainers either implement or explicitly reject native remote session delegation. A concrete implementation should add documented `gatewayUrl`/`gatewayToken` support to `sessions_spawn` and `sessions_send`, reuse the existing safe gateway override validation and credential resolution, thread resolved remote call options through native subagent, ACP spawn, send, wait/history, cleanup, and registry paths, and add tests proving model calls, tool execution, transcripts, filesystem effects, and credential/account use occur on the remote gateway.
