## Summary

Keep #38547 open. Current main still has no supported native remote gateway routing for `sessions_spawn` or `sessions_send`: the tool schemas do not accept `gatewayUrl`/`gatewayToken`, the native and ACP dispatch paths call the current gateway without URL/token overrides, and the only reusable remote override helper remains on separate gateway-style tool surfaces. The issue is recent, concrete, unprotected, and not superseded by the related title matches in the provided context.

## Best Possible Solution

Keep #38547 open until maintainers either implement or explicitly reject native remote session delegation. A concrete implementation should add documented `gatewayUrl`/`gatewayToken` support to `sessions_spawn` and `sessions_send`, reuse the existing safe gateway override validation and credential resolution, thread resolved remote call options through native subagent and ACP dispatch, and add tests proving model calls, tool execution, transcripts, and credential/account use occur on the remote gateway.
