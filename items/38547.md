## Summary

Keep #38547 open. Current main still has no supported native remote gateway routing for `sessions_spawn` or `sessions_send`; the session tools continue to create/dispatch child runs through the current gateway, while `gatewayUrl`/`gatewayToken` handling exists only in separate gateway-style helper paths.

## Best Possible Solution

Keep #38547 open until maintainers either implement or explicitly reject native remote session delegation. A concrete path would add documented `gatewayUrl`/`gatewayToken` support to `sessions_spawn` and `sessions_send`, reuse the existing safe gateway override validation/credential resolution, thread resolved remote call options through native subagent and ACP dispatch, and add tests proving model calls, tools, transcripts, and credential use happen on the remote gateway.
