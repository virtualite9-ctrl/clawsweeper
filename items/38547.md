## Summary

Keep #38547 open. Current main still does not expose native remote gateway delegation for `sessions_spawn` or `sessions_send`: the tool schemas lack `gatewayUrl`/`gatewayToken`, the native subagent and ACP spawn dispatch paths call the current gateway without URL/token overrides, and docs describe local session/subagent orchestration rather than cross-gateway compute delegation. Existing gateway override plumbing is present for other gateway-style tools, but it has not been threaded into these session tools.

## Best Possible Solution

Keep #38547 open until maintainers either implement or explicitly reject native remote session delegation. A concrete implementation should add documented `gatewayUrl`/`gatewayToken` support to `sessions_spawn` and `sessions_send`, reuse the existing safe gateway override validation and credential resolution, thread resolved remote call options through native subagent and ACP dispatch, and add tests proving model calls, tool execution, transcripts, and credential/account use occur on the remote gateway.
