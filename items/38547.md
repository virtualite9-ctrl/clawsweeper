## Summary

Keep open. Current main does not implement native per-call remote gateway delegation for sessions_spawn or sessions_send. The session tools still route through the locally resolved gateway call path and do not expose or thread gatewayUrl/gatewayToken through their schemas or execution paths, so the requested cross-gateway compute delegation remains meaningful work.
