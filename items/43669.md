## Summary

Keep this PR open. Current main has a cleanup-time idle wait before flushing pending tool results, but it does not wait for agent idle immediately after activeSession.prompt() before snapshot/result construction. The PR's central race, where an auto-retry tool loop can still be streaming when the embedded run captures messages and returns without a final reply, is still plausible on current main and is not superseded by the cleanup-only fix.

## Best Possible Solution

Keep the PR open and review or land a rebased variant that waits for agent idle immediately after prompt submission when the session is still streaming, wraps the wait in the existing abortable() path, logs timeout versus real idle accurately, and adds a focused regression test proving snapshot/result construction waits for the retry tool loop to finish.
