## Summary

Keep open. Current main still applies the `activeMinutes` filter solely by `updatedAt`, so a stale `agent:main:main` row can be omitted from `sessions.list`. The PR is a focused gateway fix for the open regression in #43737, and the author addressed the custom `mainKey` review concern. Security review found no supply-chain or permission-sensitive changes: the PR only touches gateway session-listing logic and its tests.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to rebase it onto current main, preserve the custom `mainKey` guard from the second commit, run the gateway changed-lane tests, and land it if the primary-session visibility policy is accepted; that should also resolve #43737.
