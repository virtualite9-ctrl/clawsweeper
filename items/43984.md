## Summary

Keep open. Current main still applies the `sessions.list` `activeMinutes` filter as a blanket `updatedAt` cutoff, so a stale top-level main session can still disappear from active session lists while heartbeat remains visible. This PR is a focused implementation candidate for the still-open linked regression, and its second commit addresses the custom `mainKey` review concern.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to rebase it onto current main, preserve the custom `mainKey` guard from the second commit, run the gateway changed-lane validation, and land it if maintainers accept the primary-session visibility policy; that should resolve #43737.
