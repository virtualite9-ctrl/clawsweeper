## Summary

Keep open. Current main still has the reported false-positive path: `runGatewayAuthHealth` checks that `gateway.auth.token` is a SecretRef, calls `resolveGatewayAuth` without resolving that SecretRef, then prints the unavailable warning whenever token mode still needs a token. A separate helper can resolve SecretRef-backed gateway tokens and is tested, but this doctor health check is not using it. Also, the provided GitHub context shows an open, unmerged PR with closing syntax for this issue, so the issue should remain open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review/land a focused fix that resolves `gateway.auth.token` SecretRefs in `runGatewayAuthHealth` before emitting the unavailable warning, with a regression test that proves a resolvable SecretRef does not warn. The existing open PR #65229 is the clean implementation candidate; #67463 may need splitting because it combines this bug with unrelated task timestamp behavior.
