## Summary

Keep open. Current main still has the reported false-positive path: doctor detects that `gateway.auth.token` is a SecretRef, resolves gateway auth through a path that does not resolve configured SecretRef providers, and emits the unavailable warning whenever token mode still needs a token. The issue discussion also has a later v2026.4.23 report of related SecretRef command-time confusion, and the provided GitHub context shows open, unmerged PR candidates, including one with closing syntax for this issue, so this should remain open until a focused fix lands.

## Best Possible Solution

Keep this issue open and land a focused doctor health fix that resolves `gateway.auth.token` through the existing SecretRef-aware token resolver before emitting the unavailable warning, with a regression test showing a resolvable env-backed SecretRef does not warn. Review PR #65229 and #67463 as implementation candidates, but prefer a narrow patch or split if either branch is stale or bundled with unrelated behavior.
