## Summary

Keep this PR open. Current main has independently landed the Feishu schema-generation fix, but the PR is not fully obsolete: the health snapshot change is still not wired into `getHealthSnapshot()`, the plan-provider normalization remains a contract question because plan providers are distinct catalog/runtime providers with manifest auth aliases, and the skill symlink change still conflicts with current realpath containment behavior and tests.

## Best Possible Solution

Ask the contributor or a maintainer to rebase and split the PR: drop the already-shipped Feishu schema hunk; wire any health change to the current fresh best-effort config path with a regression test for gateway health snapshots; decide the plan-provider context-window behavior against the manifest-owned auth alias/catalog contract; and handle the skill symlink fallback as a separate security-sensitive change with explicit path-canonicalization tests.
