## Summary

Keep this PR open. Current main already contains the Feishu schema fix, but the PR is not fully obsolete: the gateway health snapshot still reads the pinned runtime config, and current main now has a fresh `readBestEffortConfig()` helper that could support a rebased health fix. The provider-normalization and skill-symlink hunks should not land as-is because they conflict with current provider identity/auth-alias and realpath containment contracts.

## Best Possible Solution

Keep this PR open but ask for a rebase and split. Drop the already-shipped Feishu schema change; implement the gateway health fix by using current `readBestEffortConfig()` in `getHealthSnapshot()` with a regression test for pinned runtime snapshots; keep plan provider IDs distinct and rely on manifest `providerAuthAliases` for shared auth; handle any skill symlink behavior in a separate security-focused path-canonicalization PR with escape and allowed-alias tests.
