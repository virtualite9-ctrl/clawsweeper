## Summary

Keep this PR open. Current main has already absorbed the Feishu schema-generation hunk, but the PR is not fully obsolete or ready to close: the gateway health snapshot path still uses the pinned runtime config, the plan-provider normalization hunk conflicts with the current distinct provider/manifest-auth-alias contract, and the skill symlink fallback remains a security-sensitive path-containment change that current tests explicitly reject.

## Best Possible Solution

Keep this PR open, but ask for a rebase and split. Drop the already-shipped Feishu hunk; implement the health change against the current fresh best-effort config path with a gateway health regression test; decide any plan-provider context-window issue against the manifest-owned auth alias and distinct catalog-provider contract; and handle the skill symlink fallback separately with focused path-canonicalization security tests.
