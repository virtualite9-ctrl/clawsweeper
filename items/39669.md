## Summary

Current main still does not provide a generic user-visible queued-message feedback path for busy-agent followups. The active `enqueue-followup` branch still returns after enqueueing and cleaning up typing, Feishu/Lark still only starts feedback from reply-start typing, and the related PRs are not merged into main.

## Best Possible Solution

Keep the issue open. The best fix is a generic, non-blocking queued-message feedback seam on the active-run `enqueue-followup` path, likely an `onQueued` callback or equivalent typed channel callback in `GetReplyOptions` and the shared reply dispatcher pipeline. Invoke it before typing cleanup, swallow/log callback failures so enqueueing remains reliable, wire Feishu/Lark to an immediate reaction or receipt, preserve existing Telegram/Discord/Slack status reactions, and add focused tests proving the callback fires only for queued followups.
