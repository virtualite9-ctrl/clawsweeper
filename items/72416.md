## Summary

Keep this PR open. Current main still only accepts stable `openclaw.install.minHostVersion` floors in `>=x.y.z` form and rejects the prerelease floor form the PR is adding. The PR is a recent, focused core plugin-install compatibility change with docs and regression-test coverage in the provided diff, and the existing bot review comment supports the implementation rather than identifying a reason to close it.

## Best Possible Solution

Keep the PR open and review or land it as the implementation candidate. The likely best path is to accept prerelease-aware `openclaw.install.minHostVersion` floors in `src/plugins/min-host-version.ts`, keep enforcement centralized for install and manifest-registry loading, and merge the accompanying regression tests and public plugin-author docs updates.
