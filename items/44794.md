## Summary

Keep open. Current main still has no `gateway.preStartCommand` or `gateway.wrapperScript` config surface, and the managed service install path always derives canonical gateway program arguments before rendering launchd/systemd service files. The issue is a concrete core gateway-service installer request, not a stale/duplicate cleanup candidate.

## Best Possible Solution

Keep the issue open for maintainer/product decision. If accepted, implement it in the core gateway service installer: add typed/schema/docs support for `gateway.preStartCommand` and/or `gateway.wrapperScript`, thread it through `buildGatewayInstallPlan`, render launchd/systemd safely, and cover launchd/systemd reinstall persistence with focused tests. If maintainers prefer external supervisors instead, document that as the official answer rather than auto-closing this request.
