## Summary

Close as implemented on current `main`: the loopback MCP `tools/call` path now runs the shared `before_tool_call` hook before executing the tool, and current loopback server tests cover the blocked-hook behavior. This supersedes the open PR even though the exact implementation differs from the proposed wrapper-based patch.
