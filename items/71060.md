## Summary

Keep #71060 open. Current main has strong macOS LaunchAgent restart hardening, but the exact stop-side behavior in the report still exists: `waitForLaunchAgentStopped()` polls 10 x 100ms and `stopLaunchAgent()` still falls back to `bootout` with warning text that says the service was left unloaded. Tests explicitly assert that degraded stop bootout behavior, so this is not closeable as implemented or as a duplicate of the older restart-only reports.

## Best Possible Solution

Keep #71060 open and narrow it into a macOS LaunchAgent lifecycle fix. Maintainers should verify whether `gateway restart` from a gateway-served command can still bypass the current handoff/restart recovery, and separately decide the intended degraded `gateway stop` behavior when graceful drain exceeds the 1s confirmation window: extend the wait to the gateway drain timeout, re-bootstrap after bootout while preserving disabled state, or document and test the intentionally-unloaded hard-stop contract. Add targeted launchd unit coverage for busy drain >1s and for restart invoked from gateway-routed sessions.
