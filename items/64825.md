## Summary

Keep open. Current main still has the active-run-gated TUI terminal transition: `finalizeRun()` and `terminateRun()` only call `setActivityStatus()` when `wasActiveRun` is true, while local BTW/unbound delta paths can still set the footer to `streaming` without being active-run-owned. The shipped watchdog mitigates active-run lost-final cases, but it does not implement the requested immediate no-active-run/no-runs-in-flight recovery path, and the related implementation attempts are closed unmerged.

## Best Possible Solution

Keep this issue open and implement a narrow TUI state-machine fix in `src/tui/tui-event-handlers.ts`: after terminal cleanup, transition to the terminal status when the finishing run was active or when there is no `activeChatRunId` and no tracked session run remains. Include the local BTW empty-final branch, and add focused tests for inactive-final clearing, local BTW delta plus empty final, concurrent active-run preservation, and terminal error/aborted preservation.
