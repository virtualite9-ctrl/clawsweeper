## Summary

Issue #63598 should close as superseded by the active, directly linked PR #58034. Current main still has the trusted env-proxy guard plumbing but does not expose or pass a `tools.web.fetch.useTrustedEnvProxy`/`useEnvProxy` option for `web_fetch`; PR #58034 is the canonical remaining implementation and review thread for that exact config opt-in and hardening work.

## Best Possible Solution

Close #63598 as superseded and keep the remaining product, security, schema, docs, and test review centralized in PR #58034. If maintainers reject or substantially change that PR, reopen or replace this with a narrower follow-up that captures the decided `web_fetch` proxy config surface.
