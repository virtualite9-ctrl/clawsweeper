## Summary

Current main no longer depends on `dist/gateway.js` for gateway service startup, repairs stale installed service entrypoints, and now has release-time and postpublish checks that fail if required packaged `dist/*` content is missing.

## Close Comment

Closing from a Codex review. Current `main` no longer relies on `dist/gateway.js` for gateway service startup: the installed gateway entrypoint now resolves through `dist/index.js` / `dist/entry.js` (`src/daemon/gateway-entrypoint.ts:4`, `src/daemon/program-args.ts:18`), and doctor/update repair stale service configs when they point at an old entrypoint (`src/commands/doctor-gateway-services.ts:280`, `src/commands/doctor-gateway-services.test.ts:384`, `docs/help/faq-first-run.md:794`). The npm release path is also hardened now: `package.json` still ships `dist/`, release preflight fails if required packed paths are missing (`scripts/openclaw-npm-release-check.ts:471`), and postpublish verification installs the published package into a fresh temp prefix and validates the installed payload (`scripts/openclaw-npm-postpublish-verify.ts:488`). The changelog shows this shipped in `v2026.4.14` under the gateway/update entry (`CHANGELOG.md:663`, `CHANGELOG.md:711`). On current main and recent releases, this specific 2026.3.13 packaging/startup regression is already addressed.
