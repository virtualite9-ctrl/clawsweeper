## Summary

Current main still does not implement the Slack-owned CDN redirect-auth behavior this PR targets. The linked issue #63523 was closed as implemented, but the close evidence only proves same-origin auth preservation; current code and tests still strip Authorization on Slack CDN cross-origin redirects, so this PR is not obsolete cleanup material.

## Best Possible Solution

Keep this PR open as a focused implementation candidate, but require a current-main rebase or replacement Slack plugin patch. The best fix belongs in `extensions/slack/src/monitor/media.ts`: retain Authorization only across HTTPS Slack-owned redirect hosts, strip credentials for every non-Slack redirect, preserve `fetchRemoteMedia` / `fetchWithSsrFGuard` SSRF protections and the local `media.runtime.ts` dispatcher boundary, and keep the HTML-login-page guard. Tests should retain current same-origin and strip-external coverage while adding Slack CDN auth-retention, HTTP downgrade rejection, non-Slack stripping, and SSRF-block cases.
