## Summary

Keep open. Current main has not implemented the PR's central Slack download fix: the Slack media download path still sends Authorization on the initial Slack private URL only, and the shared guarded fetch strips Authorization on cross-origin redirects, including Slack CDN hosts. The PR is still conceptually useful but needs maintainer rework because the provided review context reports issues in the PR patch itself.

## Best Possible Solution

Keep the item open or replace it with a maintainer-owned patch that preserves `Authorization` across Slack-owned HTTPS redirect targets (`*.slack.com`, `*.slack-edge.com`, `*.slack-files.com`), strips it for non-Slack targets, rejects HTTP downgrades, preserves the existing SSRF guard, and updates the Slack media tests to lock in both the allowed Slack-CDN case and the token-stripping escape case.
