## Summary

Current main still does not implement the Slack-owned CDN redirect-auth behavior this PR targets. The shipped/current code preserves the Slack bearer token only for same-origin redirects and the current regression tests explicitly assert that Slack CDN cross-origin redirects drop Authorization, so this PR remains a focused implementation candidate rather than cleanup-close material. The PR is stale and failing exact-head checks, and it needs maintainer review for the expanded token-forwarding allowlist, but the work is not obsolete.

## Best Possible Solution

Keep this PR open as a focused candidate, but require a current-main rebase or replacement patch in the Slack plugin. The best implementation should update `extensions/slack/src/monitor/media.ts` so Slack private downloads retain Authorization only across HTTPS Slack-owned redirect hosts, strip credentials for all non-Slack redirects, preserve `fetchRemoteMedia` / `fetchWithSsrFGuard` SSRF protections and the current `media.runtime.ts` dispatcher boundary, and keep the HTML-login-page guard. The test update should keep the existing same-origin and strip-external coverage while adding Slack CDN auth-retention, HTTP downgrade rejection, non-Slack stripping, and SSRF-block cases.

## Risks / Open Questions

- The PR intentionally expands Slack bot-token forwarding from same-origin redirects to Slack-owned suffixes (`*.slack.com`, `*.slack-edge.com`, `*.slack-files.com`); maintainer review should verify the host allowlist, HTTPS rejection, and non-Slack token stripping before merge.
- The PR is stale against current main. Current main has a local `extensions/slack/src/monitor/media.runtime.ts` lazy/runtime boundary, a `hostnameAllowlist` entry in `SLACK_MEDIA_SSRF_POLICY`, and dispatcher-backed Slack media test coverage; the PR diff is based on older direct plugin-sdk imports and deletes the dispatcher-backed test, so a rebase must preserve current runtime behavior.
- Exact-head CI is failing for the Slack extension path and broader check jobs, so this cannot merge as-is even though the underlying bug-fix direction is still relevant.
