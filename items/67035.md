## Summary

Keep this issue open. Current main contains fixes for several live-rendering/history races, but the concrete input/IME and error-path stream-retention fixes described in the discussion are still not implemented in the Control UI chat path.

## Best Possible Solution

Keep this issue open or split it deliberately. Preserve the shipped stale-history, active-run reload, reading-indicator, and active-memory timeout fixes, then add a focused Control UI chat composer fix in ui/src/ui/views/chat.ts for focused/composing draft preservation with regression coverage. Also decide whether error-path partial stream retention in ui/src/ui/controllers/chat.ts and the broader accepted/processing placeholder UX should land here or in narrower linked follow-ups.
