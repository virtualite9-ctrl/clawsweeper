## Summary

Keep this issue open. Current main still renders the misleading `no bootstrap files` status wording from a count that only tracks whether `BOOTSTRAP.md` exists, while workspace bootstrap injection still includes the normal files independently. The provided context also shows open PR #43614 with closing syntax for this issue, so the best maintainer path is to review/land or supersede that PR before closing the issue.

## Best Possible Solution

Keep this issue open and have a maintainer review, refresh, or supersede PR #43614. The narrow product fix belongs in `buildStatusAgentsValue`: describe `bootstrapPendingCount` as active bootstrapping rather than bootstrap-file injection, and add a regression test for mature workspaces where `BOOTSTRAP.md` is absent but normal workspace files are still injected.
