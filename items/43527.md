## Summary

Keep this issue open. Current main still renders the misleading `no bootstrap files` wording from a count that is derived only from `BOOTSTRAP.md` existence, while workspace bootstrap injection still includes the normal workspace files independently. The provided context also shows open PR #43614 with closing syntax for this issue, so maintainer cleanup should review, refresh, or close that PR before closing the issue.

## Best Possible Solution

Keep this issue open and have a maintainer review or refresh PR #43614, or apply the same narrow fix on main: make `buildStatusAgentsValue` describe `bootstrapPendingCount` as active bootstrapping rather than bootstrap-file injection, and add a regression test for the mature-workspace zero-count case.
