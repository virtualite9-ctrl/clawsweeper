## Summary

Keep #46656 open. Current main still lacks the requested WebChat/Control UI inline-button contract: no webchat inlineButtons config, no buttons field in the chat protocol, no chat.callback RPC schema, and no Control UI normalized content/rendering path for button blocks. Related #54720 was already closed as a duplicate of this older, more detailed tracker, so #46656 remains the canonical open item.

## Best Possible Solution

Keep #46656 open as the canonical tracker. The eventual fix should align with the current semantic `presentation` contract rather than adding a new provider-native `buttons` escape hatch, and should cover gateway config/protocol, final assistant message payload projection, Control UI normalization/rendering, click routing via a WebChat callback or chat-send path, docs, and regression tests.
