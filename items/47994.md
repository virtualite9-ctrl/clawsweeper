## Summary

Keep #47994 open. Current main has a partial auto-failover override self-heal path, but it has not implemented #47994's core runtime-model provenance fix or an equivalent persisted-selected-model separation. The same session fields still store the actual runtime fallback identity, and model resolution still trusts those fields on later turns.

## Best Possible Solution

Keep #47994 open for maintainer reconciliation. The best fix is to explicitly separate per-turn fallback runtime accounting from next-turn selected model persistence, or add robust fallback provenance that causes resolution/display surfaces to retry the configured primary while preserving explicit user overrides and current auto-override self-healing.
