## Summary

Keep this PR open. Current main has a narrower auto-failover override self-heal path, but it still does not implement this PR's fallback provenance model or an equivalent persisted-selected-model/runtime-model separation across the affected session persistence and display surfaces.

## Best Possible Solution

Keep this PR open for maintainer reconciliation. The best path is to land a reviewed version of the runtime-model provenance approach, or replace it with an equivalent selected-model/runtime-model split that covers command, reply, followup, cron, gateway/session APIs, status, and sessions display while preserving explicit user overrides and the current auto-override self-healing behavior.
