## Summary

Keep #47994 open. Current main has a newer partial self-heal path for auto fallback overrides, but the specific sticky runtime-model problem is not fully implemented or clearly superseded: runtime persistence still writes the actual fallback provider/model into session fields, `resolveSessionModelRef` still prefers those runtime fields, and neither the PR's `modelIsFromFallback` schema nor #68991's `persistedModel`/`persistedProvider` separation is present on main. The PR discussion also contains useful production confirmation from v2026.3.11 and many review-driven edge cases, so this should get maintainer reconciliation rather than cleanup closure.

## Best Possible Solution

Keep the PR open for maintainer follow-up. The best product direction is to explicitly separate per-turn fallback runtime accounting from next-turn selected model persistence, or add a robust fallback provenance flag, while preserving the current `modelOverrideSource: "auto"` self-heal path. Close #47994 only after equivalent behavior lands on main with coverage across the session persistence and display/API surfaces.
