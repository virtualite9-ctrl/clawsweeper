## Summary

Keep this PR open for maintainer review. Current main has not implemented the proposed fallback change, but the PR is not a high-confidence cleanup close: the linked bloat report now says the observed startup bloat is the explicit-config plugin-discovery path, and the proposed one-line change still has unresolved correctness concerns around resolved runtime request secrets.

## Best Possible Solution

Keep this PR open for maintainer review, but do not land it as-is. The maintainer path should be either to revise it into a tested fallback-path fix that preserves source markers for all SecretInput-backed provider fields, including nested `request.*` fields, or to close it manually in favor of a separate fix/product decision for the `models.providers: {}` plugin-discovery behavior described in #65020.
