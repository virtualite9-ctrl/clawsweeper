## Summary

Keep this PR open for maintainer review. Current main has not implemented the fallback change, but this is not a conservative cleanup close: the linked bloat report now points to explicit empty-provider plugin discovery, and the proposed change has an unresolved secret-persistence concern for resolved runtime `request.*` credentials.

## Best Possible Solution

Keep this PR open for maintainer review, but do not land it as-is. The maintainer path should either revise it into a tested fallback-path fix that preserves source markers for all SecretInput-backed provider fields, including nested `request.*` fields, or close it manually in favor of a separate fix/product decision for the `models.providers: {}` plugin-discovery behavior described in #65020.
