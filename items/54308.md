## Summary

Keep open. Current `main` does not contain the inbound-drain teardown behavior this PR is trying to add, and the PR still has an unresolved correctness issue around duplicate Discord accounts sharing the same bot identity.

## Evidence

- **Current provider teardown does not wait for inbound work to finish:** `monitorDiscordProvider` still tears down by calling `deactivateMessageHandler?.()` and immediately proceeds with shutdown in the `finally` block; there is no awaited idle/drain step before cleanup.
  - file: [extensions/discord/src/monitor/provider.ts:1136](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/extensions/discord/src/monitor/provider.ts#L1136)

- **Current Discord message handler lifecycle is not awaitable:** `DiscordMessageHandlerWithLifecycle` on `main` only wires `deactivate`; it does not expose `waitForIdle`, so the PR's core behavior is not already implemented elsewhere on `main`.
  - file: [extensions/discord/src/monitor/message-handler.ts:265](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/extensions/discord/src/monitor/message-handler.ts#L265)
