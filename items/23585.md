## Summary

Keep #23585 open. Current main still does not implement a trusted-proxy default/fallback user: the auth path requires the configured identity header, returns `trusted_proxy_user_missing` when it is absent, and the public config/schema/docs/tests expose only the existing header-based fail-closed behavior. The discussion also distinguishes this from #25293 and #17761, so there is no clear canonical shipped fix or duplicate to close against.

## Best Possible Solution

Keep #23585 open for maintainer/security product judgment. If accepted, implement an explicit trusted-proxy fallback/default user option across `GatewayTrustedProxyConfig`, zod/generated schemas, config help/docs/security audit guidance, and auth tests; only allow fallback after trusted proxy source and required-header checks pass, and keep `allowUsers` semantics explicit. If maintainers want identity headers to remain mandatory, document Cloudflare Zero Trust IP-whitelist/no-identity WebSocket setups as unsupported instead of closing this as implemented.
