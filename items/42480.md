## Summary

Keep this PR open. Current main still drops Discord mention metadata when it builds a debounced synthetic message, and the PR patch addresses the right bug, including the follow-up Greptile concern for roles and @everyone. The remaining work is to port the fix from the old `src/discord` path to the current Discord plugin path and add regression coverage.

## Best Possible Solution

Keep this PR open as a useful focused bug fix. Port the mention accumulation into `extensions/discord/src/monitor/message-handler.ts`, preserving `mentionedUsers`, `mentionedRoles`, and `mentionedEveryone` across all debounced entries, then add a Discord handler queue/preflight regression test for earlier-entry mention metadata before landing.
