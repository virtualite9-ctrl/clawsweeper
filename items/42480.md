## Summary

Current main still has the same Discord debounced synthetic-message shape: it spreads the last Carbon Message into a plain object without materializing mention metadata, while preflight still relies on mentionedUsers, mentionedRoles, and mentionedEveryone for mention gating. This PR is stale against the current file layout but still represents useful, focused bug-fix work that should be ported and tested rather than closed.

## Best Possible Solution

Keep this PR open as useful bug-fix work. The best path is to port the mention accumulation to extensions/discord/src/monitor/message-handler.ts, preserve mentionedUsers, mentionedRoles, and mentionedEveryone across all debounced entries, add a regression test in the Discord handler queue/preflight coverage, then run the relevant Discord extension test lane before landing.
