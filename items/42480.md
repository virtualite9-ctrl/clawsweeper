## Summary

Keep this PR open. Current main still loses Discord mention metadata when debounced messages are collapsed into a synthetic message, and the PR addresses the right bug after the follow-up commit expanded the fix from users to roles and @everyone. The remaining work is to port it from the old Discord path to the current plugin path and add regression coverage.

## Best Possible Solution

Keep this PR open as useful bug-fix work. Port the all-entry mention accumulation into `extensions/discord/src/monitor/message-handler.ts`, preserving `mentionedUsers`, `mentionedRoles`, and `mentionedEveryone` on the synthetic message, then add a Discord handler queue/preflight regression test for mention metadata appearing before the last debounced entry before landing.
