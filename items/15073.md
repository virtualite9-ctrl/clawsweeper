## Summary

Keep #15073 open. Current main still implements model-only fallback selection: it can override the fallback model list per agent, but fallback attempts continue inside the same active run with the same agent id, workspace, agentDir, sessionFile, and skills snapshot. The requested fallback-agent or fallback-workspace/context-profile behavior is not present in config types, strict schemas, fallback candidate metadata, runtime plumbing, docs, or tests. The issue also already has a 2026-04-26 maintainer/Codex comment explicitly keeping it open.

## Best Possible Solution

Keep #15073 open until maintainers choose a first-class design: either fallbackAgent/fallbackWorkspace semantics or a narrower model-specific context profile/budget mechanism. A complete fix likely belongs in core config types and strict schemas, docs/schema help, fallback candidate metadata, reply-runner/session plumbing that can safely swap agentDir/workspaceDir/skills/context per fallback attempt, and regression tests proving fallback retries use the alternate context without corrupting the original session.
