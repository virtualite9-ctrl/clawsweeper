## Summary

Keep #15073 open. Current main still implements model fallback as provider/model retry only, while each retry reuses the active agent, workspace, agentDir, session file, and skills snapshot. The requested fallback-agent/workspace/context-profile behavior is not present in config schema, runtime fallback candidates, docs, or tests, and the issue already has a current maintainer/Codex review comment explicitly keeping it open.

## Best Possible Solution

Keep #15073 open until OpenClaw either adds first-class fallback-agent/workspace/context-profile semantics or maintainers choose a narrower model-specific context-budget/profile design as the canonical product direction. A complete core fix likely needs config schema/types/docs updates, fallback candidate metadata beyond provider/model, retry plumbing that resolves a fallback agent/workspace safely, bootstrap/context assembly changes, and regression tests proving fallback retries use the intended agent context without corrupting the original session.
