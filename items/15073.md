## Summary

Keep #15073 open. Current main still supports only provider/model fallback lists, and fallback retries reuse the active agent session, workspace, bootstrap context path, skills snapshot, and transcript. The related closed issues cover narrower context-budget/profile ideas and do not supersede the broader fallback-agent/workspace request; the provided issue discussion also includes a same-day maintainer/Codex comment keeping this open.

## Best Possible Solution

Continue tracking #15073 until OpenClaw either adds explicit fallback-agent/workspace/context-profile semantics or a maintainer chooses a narrower model-specific bootstrap/context-budget design as the canonical product direction. A real fix likely needs config schema changes, fallback candidate metadata beyond provider/model, retry plumbing that can resolve a fallback agent/workspace without corrupting the original session, bootstrap/context assembly updates, docs, and regression tests proving fallback retries use the intended agent context.
