## Summary

Keep open. Current main has only the older OAuth-refresh log-tail grouping and still falls back to arbitrary gateway log lines, so this PR is not obsolete. The PR is a focused status/daemon diagnostics cleanup, but it needs maintainer review or revision because the review discussion identifies real display regressions in the proposed filtering/grouping behavior.

## Best Possible Solution

Keep this PR open and ask for a rebase/revision: preserve timestamped embedded OAuth failure summaries, fix or rename the lane-wait `last` label, add coverage for those cases plus the daemon no-fallback behavior, then run the targeted status-all/daemon tests and the changed gate. If maintainers want a smaller path, split the `readLastGatewayErrorLine` no-fallback change from the broader log-tail summarizer cleanup.
