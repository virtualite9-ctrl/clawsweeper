## Summary

Keep open. Current main still lacks explicit `bailian` / `BAILIAN_API_KEY` auth compatibility, and the linked bug report remains a plausible product decision. This PR is not merge-ready as-is because it edits an obsolete static env-map shape and bundles unrelated cron/log changes, but the underlying auth request should be handled deliberately rather than auto-closed.

## Best Possible Solution

Keep this PR open for maintainer judgment, but do not merge it as-is. Ask for a split and rebase: one auth-only patch against current provider manifest/setup metadata, with a clear decision on whether `bailian` is a compatibility alias for Qwen/Alibaba and whether `BAILIAN_API_KEY` should be accepted. The cron warning and channel-log fallback should be separate PRs tied to their own reports.
