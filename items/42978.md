## Summary

Keep open. Current main no longer has the old static env-map target, but it also does not implement the requested `bailian` / `BAILIAN_API_KEY` compatibility from #42858. The branch is not merge-ready as-is because it mixes that auth change with separate cron and channel-log fixes, and the auth part needs to be rebased onto the current manifest-driven provider auth surface.

## Best Possible Solution

Keep this PR open for maintainer judgment, but ask for a split/rebase before merge. The auth part should be rebuilt against current main by deciding whether `bailian` is a compatibility alias for Qwen/Alibaba and whether `BAILIAN_API_KEY` should be accepted in trusted plugin manifest/setup env metadata, with focused tests and docs. The cron warning and channel-log fallback should be separated into their existing tracks rather than bundled with the auth fix.
