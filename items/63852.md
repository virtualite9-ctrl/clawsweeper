## Summary

Current `main` contains shipped safeguards for the reported upgrade failure mode: clobbered configs that lose `gateway.mode` are auto-restored from last-known-good backups before gateway startup, and legacy state-dir/session/auth migrations now refuse unsafe target overwrites and roll back on relink failure. The fix is documented in the `v2026.4.23` changelog and covered by regression tests.

## Risks / Open Questions

- I could not run a live upgrade reproduction in this checkout because `pnpm` is unavailable in the environment, so the decision relies on current source, tests, docs, and shipped changelog evidence rather than an end-to-end upgrade run.
- The original report also mentions workspace/memory/plugin loss; the strongest direct evidence on current `main` is for state-dir safety plus session/auth/config recovery, which is sufficient to show the destructive migration path has been addressed but does not include a live fixture for every user-state subtree.
