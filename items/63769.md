## Summary

Keep open. Current main still contains the overlap-based Gateway chat merge path described in #63769, and the observable repeated-boundary character loss is still present in the current implementation. The linked fix PR #63994 is open/unmerged, so the bug is not implemented on main or superseded by shipped code.

## Best Possible Solution

Fix `src/gateway/server-chat.ts` by making delta append use plain concatenation after empty checks while preserving the existing `nextText.startsWith(previousText)` fast path, then add focused regression coverage for repeated digit/CJK boundaries alongside the existing cross-tool segmented merge tests. The open PR #63994 or a refreshed equivalent can be used as the implementation path, but the issue should remain open until the fix lands on main.
