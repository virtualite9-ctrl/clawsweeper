## Summary

Keep open. Current main still has no `tools.cron.tools.allow/deny` config surface and does not make cron sessions inherit `tools.subagents.tools`; cron only has per-job `toolsAllow` plus delivery-mode-specific message-tool handling. The existing Apr 26, 2026 issue comment already reached the same keep-open conclusion, and current main remains aligned with that finding.

## Best Possible Solution

Keep #12736 open for maintainer product direction. A complete fix should add a cron-scoped policy such as `tools.cron.tools.allow/deny` or explicitly document and implement cron inheritance from `tools.subagents.tools`, define precedence against global/provider/agent/group/sandbox and per-job `payload.toolsAllow` policies, reconcile the behavior with announce/none/webhook delivery and message-tool handling, then update schema/types/help/generated config metadata, docs, CLI/config reference, and regression tests.
