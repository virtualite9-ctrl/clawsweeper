## Summary

Current main still does not implement a cron-scoped tool deny policy or inheritance from `tools.subagents.tools`. It has per-job `toolsAllow` and a hard-coded cron delivery policy that disables `message` for webhook delivery only, while announce/none paths still keep the message tool available. The user-facing docs also state that isolated cron jobs can use the `message` tool even with no runner fallback delivery.

## Best Possible Solution

Keep the issue open for maintainer product direction. A complete fix should add `tools.cron.tools.allow/deny` or explicit cron inheritance from `tools.subagents.tools`, define precedence against global/agent/provider/group/sandbox policy, per-job `toolsAllow`, and cron delivery-mode message-tool controls, then update schema/types/help/docs/generated config metadata plus tests for denied `message` across announce, none, and webhook delivery modes.
