## Summary

Keep #12736 open. Current main still has no cron-scoped tool deny/allow config and does not make cron inherit `tools.subagents.tools`; isolated cron only has per-job `payload.toolsAllow` plus hard-coded message-tool behavior by delivery mode. The existing Apr 26 maintainer/Codex review comment also explicitly kept this issue open for the same gap.

## Best Possible Solution

Keep #12736 open for maintainer product direction. A complete fix should add a cron-scoped policy such as `tools.cron.tools.allow/deny` or explicitly define cron inheritance from `tools.subagents.tools`, specify precedence against global/agent/provider/sandbox and per-job `payload.toolsAllow`, integrate with cron delivery-mode message-tool handling, then update schema/types/help/generated config metadata, docs, CLI/config reference, and tests across announce, none, and webhook delivery modes.
