## Summary

Keep this PR open. Current main still does not resolve Feishu media from quoted/replied messages: the handler builds the agent media payload from only the triggering message before fetching the parent message, and fetched parent messages still expose only parsed placeholder text rather than the raw JSON media keys needed for download. The PR is a focused implementation candidate, not obsolete or superseded.

## Best Possible Solution

Keep this PR open or replace it with an equivalent focused fix. The best maintainer path is to rebase it onto current main, preserve rawContent on fetched FeishuMessageInfo, resolve quoted/replied media only after the quoted message passes context-visibility checks, build the final agent media payload after all media is collected, preserve existing Feishu routing/thread behavior, and add a quoted-media regression test.
