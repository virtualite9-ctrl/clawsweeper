## Summary

Keep this PR open. Current main still resolves media only from the triggering Feishu message, builds the agent media payload before fetching the quoted/replied parent message, and fetched parent messages expose only parsed placeholder text rather than the raw JSON keys needed to download quoted media. The PR remains a focused implementation candidate, though it likely needs rebasing/adaptation to current file structure plus a quoted-media regression test.

## Best Possible Solution

Keep this PR open or replace it with an equivalent focused fix. The best maintainer path is to rebase it onto current main, add rawContent or an equivalent raw-media-key carrier to fetched FeishuMessageInfo, resolve quoted/replied media only after quoted-message visibility checks pass, build the final agent media payload after all media is collected, preserve existing Feishu routing/thread behavior, and add a quoted-media regression test.
