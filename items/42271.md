## Summary

Keep open. Current main still lacks the behavior this PR is trying to add: Feishu resolves media from the triggering message only, builds the agent media payload before fetching quoted/replied content, and fetched parent messages do not expose raw JSON content needed to extract image_key/file_key. The PR discussion and related #41675 describe a real Feishu group-chat workflow gap, so this should be rebased/adapted rather than closed as stale or obsolete.

## Best Possible Solution

Keep this PR open or replace it with an equivalent focused fix. The best implementation should add rawContent to FeishuMessageInfo, resolve quoted/replied media through the existing Feishu media resolver only after the quoted message passes context-visibility checks, build the final agent media payload after all media is collected, preserve current Feishu routing/thread/broadcast behavior, and add a regression test proving quoted image/file media reaches MediaPaths/MediaTypes.
