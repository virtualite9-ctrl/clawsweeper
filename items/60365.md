## Summary

Current `main` no longer matches the reported 2026.4.2 / bundled `acpx` 0.4.0 failure path. The ACPX plugin now embeds `acpx/runtime`, resolves harness launches through per-agent overrides, and ships bundled `acpx` 0.5.3, so the old `plugins.entries.acpx.config.command -> native Claude Code -> unknown option --format` path is obsolete against current OpenClaw.

## Evidence

- **Current ACPX service uses embedded runtime and agent registry overrides:** `createAcpxRuntimeService()` constructs `new AcpxRuntime(...)` from `acpx/runtime` and passes `createAgentRegistry({ overrides: pluginConfig.agents })`; there is no plugin-config `command` field in this path.
  - file: [extensions/acpx/src/service.ts:43](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/extensions/acpx/src/service.ts#L43)
  - sha: [926068b14fc6](https://github.com/openclaw/openclaw/commit/926068b14fc661e915a0247e9ecb5949fb95b4f7)
- **Supported ACPX plugin config exposes only per-agent command overrides:** The schema defines `agents?: Record<string, { command: string }>` and no top-level `config.command` field for ACPX runtime selection.
  - file: [extensions/acpx/src/config-schema.ts:26](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/extensions/acpx/src/config-schema.ts#L26)
  - sha: [926068b14fc6](https://github.com/openclaw/openclaw/commit/926068b14fc661e915a0247e9ecb5949fb95b4f7)
- **Generated plugin manifest matches the schema:** The bundled ACPX manifest exposes `Agent Commands` UI/config support for `agents` and does not list a `command` property under plugin config.
  - file: [extensions/acpx/openclaw.plugin.json:144](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/extensions/acpx/openclaw.plugin.json#L144)
  - sha: [926068b14fc6](https://github.com/openclaw/openclaw/commit/926068b14fc661e915a0247e9ecb5949fb95b4f7)
- **Current bundled ACPX version is newer than the reported one:** Current `main` ships bundled dependency `acpx: 0.5.3`, not the reported bundled `0.4.0`. The latest release tag in this checkout (`v2026.4.23`) points to the same ACPX pin.
  - file: [extensions/acpx/package.json:7](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/extensions/acpx/package.json#L7)
  - sha: [926068b14fc6](https://github.com/openclaw/openclaw/commit/926068b14fc661e915a0247e9ecb5949fb95b4f7)
- **OpenClaw changelog records the runtime integration change:** Changelog entry: ACPX now embeds the ACP runtime directly in the bundled plugin and removes the extra external ACP CLI hop, which is the exact architecture shift that obsoletes the older CLI-spawn failure mode.
  - file: [CHANGELOG.md:1240](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/CHANGELOG.md#L1240)
  - sha: [926068b14fc6](https://github.com/openclaw/openclaw/commit/926068b14fc661e915a0247e9ecb5949fb95b4f7)
- **Published acpx@0.5.3 built-in `claude` target no longer maps to raw Claude Code CLI:** Inspection of the published `acpx@0.5.3` package shows built-in `claude` resolves to `@agentclientprotocol/claude-agent-acp`, and the runtime spawn path launches the ACP adapter over stdio rather than passing OpenClaw CLI formatting flags to native Claude Code directly.
  - command: `curl -L https://registry.npmjs.org/acpx/-/acpx-0.5.3.tgz | tar -xzOf - package/dist/prompt-turn-CXMtXBl-.js`

## Risks / Open Questions

- `docs/tools/acp-agents-setup.md` still contains an older `plugins.entries.acpx.config.command` example alongside the newer `agents.<id>.command` guidance, so there is still some user-facing documentation ambiguity even though this specific runtime bug no longer matches current code.
- This review was source-based. I could not run a live ACP reproduction in this environment because the checkout lacks `pnpm`/installed deps.
