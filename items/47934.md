## Summary

Keep open. Current `main` already has Microsoft Teams federated auth support, schema entries, docs, and tests, but it does not clearly implement this PR’s specific follow-up changes. The auth model on `main` has diverged (`"secret" | "federated"` with `certificatePath` / `useManagedIdentity`) from this PR’s proposed `clientSecret` / `certificate` / `federatedCredential` shape, and the old secret-centric auth error text is still present. That is not strong enough evidence for `implemented_on_main`, and there is no basis for another close reason.
