## Summary

Keep PR #68725 open. Current main still hardcodes `contextWindow: 32000` for every model returned by Bedrock Mantle `/v1/models`; only the manually appended Claude Opus 4.7 Anthropic-route model has a 1M context window. The related standard `amazon-bedrock` provider work from #65952 is present on main, but it does not cover the separate `amazon-bedrock-mantle` discovery path this PR targets.

## Best Possible Solution

Keep #68725 open and refine it into a tested Mantle-specific implementation: add a known-context resolver in `extensions/amazon-bedrock-mantle/discovery.ts`, align ID normalization with the standard Bedrock resolver where applicable, keep the separate Anthropic Messages route behavior clear, add focused discovery tests for known MiniMax/Qwen/DeepSeek/GLM/Nemotron/Mistral IDs plus unknown fallback/ID variants, update docs if the public default/example changes, then run the changed extension gate before merge.
