## Summary

Current `main` already implements the Telegram ACP placement behavior this issue asked for. The native ACP spawn path no longer hard-requires `child`; it resolves a channel-appropriate placement and accepts `current` when that is the correct Telegram behavior. The Telegram binding adapter on `main` also advertises both `current` and `child`, and there is regression coverage for Telegram topic-bound ACP spawns.
