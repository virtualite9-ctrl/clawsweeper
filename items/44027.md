## Summary

Keep this PR open. Current main still suppresses ordinary group text-only tool summaries without consulting the persisted `/verbose on` or `/verbose full` setting, while docs and directive tests support the premise that verbose is session-scoped and intended to expose tool summaries. The submitted PR is narrow and security-sensitive only in the sense that it changes runtime delivery of tool output, but it is stale against current main because current tests now expect native command-source sessions to deliver tool summaries and media, while the PR diff preserves native suppression.

## Best Possible Solution

Keep this PR open as a plausible focused bug fix, but require a rebase and implementation update against current `src/auto-reply/reply/dispatch-from-config.ts`. The best path is to let explicit session verbose `on` or `full` opt ordinary non-Slack group chats into text tool summaries while preserving current forum, Slack non-DM, sendPolicy, media-only, exec-approval, preview-suppression, and native-command behavior unless maintainers intentionally change those contracts. Add focused regression tests and coordinate broader configurable group tool-result policy in #54663.
