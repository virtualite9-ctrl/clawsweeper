## Summary

Keep this PR open. Current main still drops text-only tool payloads for ordinary group chats regardless of `/verbose`, while docs describe `/verbose` as group-scoped and as enabling tool summaries. The PR is a plausible focused fix, supported by the DingTalk AI Card context, but it needs a current-main rebase and maintainer review because the latest patch's native-command assumption no longer matches current core tests.

## Best Possible Solution

Keep the PR open but require a rebase onto current main. The best fix is to update the current `shouldSendToolSummaries`/verbose-progress gate so explicit session verbose `on` or `full` can opt ordinary group chats into tool summaries, while preserving current forum, Slack non-DM, send-policy, media-only, exec-approval, and native-command behavior unless maintainers explicitly choose otherwise. Add focused regression tests for group verbose `off/on/full` and align any docs/security wording or the broader #54663 config-option discussion.
