## Summary

Keep open. Current `main` contains partial hardening for late exec/stdout and subagent cleanup sequencing, but the April 10 false-timeout subcase from the issue discussion still matches the code: the embedded-run wall-clock abort timer stays armed until the final `finally` cleanup, and subagent lifecycle end events still map `aborted=true` directly to `timeout`.

## Evidence

- **Exec late-update hardening exists:** The exec runtime now explicitly disables `onUpdate` delivery once the process settles and on tool abort, with an inline note that this is to prevent the `Agent listener invoked outside active run` crash from late stdout/update delivery.
