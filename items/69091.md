## Summary

Keep PR #69091 open. Current main still contains the exact generic throw sites the PR changes, and the existing shared-helper tests still do not cover the errored-finalization branch. Related fixes now handle fallback/lifecycle behavior for some `stopReason: "error"` cases, but they do not make this PR’s diagnostic-message preservation obsolete.

## Best Possible Solution

Keep the PR open for maintainer review. The likely product path is to preserve `output.errorMessage || "An unknown error occurred"` in the shared helper and both OpenAI Responses variants, add the proposed regression coverage, and explicitly decide/test the intended fallback classification for provider-specific finish reasons that would no longer be hidden behind the generic wrapper.
