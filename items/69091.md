## Summary

Keep PR #69091 open. Current main still has the exact behavior the PR targets: provider finish reasons can populate `output.errorMessage`, but the shared finalizer and both OpenAI Responses finalization paths still throw the generic `An unknown error occurred` and then overwrite the output error with that generic text. The PR is focused and plausibly useful, with a concrete repro and a positive bot review, but it should get maintainer review around failover classification before landing unchanged.

## Best Possible Solution

Keep the PR open and revise or land it with explicit maintainer review: propagate `output.errorMessage || "An unknown error occurred"` in the shared helper and both OpenAI Responses variants, add the proposed regression tests, and add or update focused failover-classification tests for provider finish reasons so the intended retry behavior is documented.
