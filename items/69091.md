## Summary

Keep open. Current main still has the three hard-coded transport-stream throws that PR #69091 changes, and the shared helper tests do not cover preserving a preexisting provider errorMessage. The PR is a focused bug fix with concrete reproduction context and related closed reports, not an obsolete or already-implemented cleanup.

## Best Possible Solution

Keep the PR open and land the narrow implementation after maintainer review: update the three transport-stream throw sites to use output.errorMessage || "An unknown error occurred", and add regression coverage for finalizeTransportStream with and without an existing provider errorMessage. No broader flow-control or failover policy change is needed for this PR.
