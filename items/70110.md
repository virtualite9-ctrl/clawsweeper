## Summary

Keep open. Current main still advertises Nextcloud Talk reactions and includes the low-level `sendReactionNextcloudTalk` helper, but the Nextcloud Talk plugin does not expose a `message` tool action adapter or route `react` through that helper. The shared message-action runner only executes non-send actions through `plugin.actions.handleAction`, so the reporter’s described agent reaction path still has no supported execution route.

## Best Possible Solution

Implement this in the bundled Nextcloud Talk plugin by adding a `ChannelMessageActionAdapter` that advertises `react` via `describeMessageTool`, resolves room token and message id from explicit params or current tool context, calls `sendReactionNextcloudTalk`, and adds focused regression coverage for `runMessageAction({ channel: "nextcloud-talk", action: "react" })` or the shared `message` tool path. Update docs if Nextcloud Talk supports only add reactions and not remove/list semantics.
