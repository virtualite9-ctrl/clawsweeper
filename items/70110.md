## Summary

Keep #70110 open. Current main still has the reported Nextcloud Talk reaction sender but does not wire it into the channel-owned `message` action surface, so an agent `message(action="react")` path cannot reach `sendReactionNextcloudTalk`. The docs and channel capability metadata both claim reactions are supported, which makes this a real bundled-plugin behavior gap rather than stale or external work.

## Best Possible Solution

Keep the issue open and fix the bundled Nextcloud Talk plugin by adding a channel message-action adapter or `message-tool-api` surface that advertises `react`, resolves room/message context from `to` or current inbound context, calls `sendReactionNextcloudTalk`, and includes regression tests proving the shared `message` tool reaches the reaction sender. While implementing, verify the current Nextcloud Talk bot reaction endpoint semantics against upstream docs/source and adjust `docs/channels/nextcloud-talk.md` if any capability caveat remains.
