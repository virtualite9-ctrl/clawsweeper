## Summary

Keep open. Current main still advertises Nextcloud Talk reactions and contains a low-level `sendReactionNextcloudTalk` helper, but the channel plugin does not expose a `message` tool action adapter or route `react` through that helper. The shared message-action runner only dispatches non-send/poll actions through `plugin.actions.handleAction`, so an agent reaction for `nextcloud-talk` still has no supported execution path.

## Best Possible Solution

Add a Nextcloud Talk channel `actions` adapter that advertises `react` through `describeMessageTool` and handles it by resolving the room token/message id from the current tool context or explicit params, then calling `sendReactionNextcloudTalk`. Add focused regression coverage for `runMessageAction({ channel: "nextcloud-talk", action: "react" })` or the shared `message` tool path, and align docs if Nextcloud only supports a subset of add/remove/list reaction semantics.
