## Summary

Current main still has the Nextcloud Talk reaction bug. The channel advertises reactions and the HTTP sender exists, but the plugin does not register an `actions` adapter, so the shared `message` tool has no current-main path to call `sendReactionNextcloudTalk`. PR #72348 is the right active fix to review, but it is still open and unmerged, so this issue should remain open until that or an equivalent fix lands.

## Best Possible Solution

Keep #70110 open and review/land PR #72348 or an equivalent fix. The correct product path is to register a Nextcloud Talk `ChannelMessageActionAdapter` that exposes `react`, resolves `to`/`messageId`/`emoji` from the shared message-tool context, calls the existing `sendReactionNextcloudTalk` sender, and adds focused regression coverage for both action dispatch and request shape.
