## Summary

Current main still lacks the Nextcloud Talk message-action adapter needed to route shared `message` tool `react` actions to the existing HTTP reaction sender. The related PR #72348 is open, unmerged, and explicitly proposes the missing wiring, so this issue should remain open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review/land PR #72348, or an equivalent fix that adds a Nextcloud Talk `ChannelMessageActionAdapter`, registers it from `extensions/nextcloud-talk/src/channel.ts`, dispatches `react` to `sendReactionNextcloudTalk`, and adds regression coverage for discovery, dispatch, message-id fallback, and reaction HTTP request shape.
