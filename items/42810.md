## Summary

Keep this PR open. Current main still lacks ACP control-plane audit logging, and the open gateway tool-call audit work in #63557 is a different surface. The contribution is still potentially useful, but it should not merge as-is because the branch is behind current ACP lifecycle behavior and still has security/reliability issues in the audit sink design.

## Best Possible Solution

Keep this PR open, but require a rebased and narrower ACP audit implementation before merge. The maintainable path is to preserve current initializeSession and closeSession semantics, wire a real audit sink under the normal OpenClaw state/log directory with restrictive permissions, make audit emission best-effort, cover session/run/cancel/runtime-control events with focused tests, harden buffering/query/retention semantics, and split unrelated config/schema edits out. Treat #63557 as separate gateway tool-call audit work, not a replacement.

## Risks / Open Questions

- Merging the branch as-is could regress current ACP runtime option propagation for model/thinking/cwd during initializeSession.
- Audit logging is awaited in lifecycle finally blocks, so a rejecting or unhealthy audit sink can alter session init/close outcomes unless audit writes are explicitly best-effort.
- The proposed file logger has audit-security correctness gaps: unbounded memory growth on repeated flush failure, query/flush races, async storage-dir creation, imprecise prune/retention semantics, and no clear restrictive file-mode/state-dir policy.
- The advertised persistent audit trail is not active on the normal AcpSessionManager singleton path because the PR defaults to NullAuditLogger.
- The branch mixes security-sensitive audit code with unrelated config/schema churn, increasing review surface even though it does not add third-party dependencies, workflows, or package lifecycle hooks.
