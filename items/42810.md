## Summary

Keep this PR open. Current main still does not implement ACP control-plane audit logging, and the open gateway tool-call audit PR #63557 is a different surface. The PR is still potentially useful, but it should not merge as-is because it is behind current ACP lifecycle behavior and has unresolved audit correctness/security issues.

## Best Possible Solution

Keep this PR open, but require a rebased and narrower ACP audit implementation before merge. The maintainable path is to preserve current initializeSession and closeSession behavior, wire an explicit real audit sink under the normal OpenClaw state/log directory with restrictive permissions, make audit writes best-effort so sink failures cannot change lifecycle outcomes, cover session/run/cancel/runtime-control events with focused tests, harden buffering/query/retention semantics, and split unrelated config/schema edits out of the PR. Treat #63557 as separate gateway tool-call audit work, not a replacement.

## Risks / Open Questions

- Merging the branch as-is could regress current ACP runtime option propagation and close/reset recovery semantics.
- The advertised persistent audit trail is not enabled on the normal AcpSessionManager construction path because the PR defaults to a null logger.
- Audit emission is awaited inside lifecycle finally blocks, so an unhealthy audit sink can still affect ACP lifecycle calls unless made explicitly best-effort.
- The file logger has security-relevant reliability gaps: relative default storage, no restrictive file mode, async mkdir race, unbounded memory growth on repeated flush failure, query/flush races, and imprecise retention/prune behavior.
- The branch includes unrelated config/schema churn alongside a security-sensitive audit feature, increasing review risk.
