## Summary

Keep this PR open. Current main does not implement ACP control-plane audit logging, and the PR is not superseded by the related closed status-doc PR or the separate gateway-tool audit work. It is still potentially useful, but it needs a significant rebase/redesign before maintainer merge: the current branch is outdated against `main`, only wires a null default logger for normal ACP manager construction, and has unresolved audit correctness and lifecycle risks raised in review.

## Best Possible Solution

Keep the PR open but do not merge it as-is. The maintainable path is a rebased, narrower ACP audit implementation against current `main`: add an explicit config/startup-wired audit sink under the normal state dir with restrictive permissions, make audit writes best-effort so they cannot change ACP lifecycle outcomes, preserve current `initializeSession`/`closeSession` semantics and tests, cover run/cancel/runtime-control events if they are part of the product claim, and harden the logger with bounded buffering, scheduled retention, entry-accurate pruning, query dedupe/flush visibility, and focused regression tests. Split unrelated Telegram/gateway/cron changes out of this PR.

## Risks / Open Questions

- Merging this branch as-is would likely regress current ACP session lifecycle behavior because the PR manager code is behind current `main` and omits newer close/reset/recovery paths.
- The advertised audit trail is not actually enabled for normal ACP manager use; the singleton path still gets a null logger unless a real sink is wired through config/runtime startup.
- Audit sink failures and query/flush races are security-relevant for an audit feature: repeated flush failures can grow memory, retention is not scheduled, prune counts files while the interface promises entries, and query results can miss or duplicate entries around flushes.
- The branch contains unrelated changes outside the stated ACP audit scope, including Telegram schema ordering, gateway config comment/type churn, and cron import fixes, so it should be split or cleaned before review.
