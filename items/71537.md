## Summary

Keep #71537 open. Current main only partially recovers reset archives for the session-memory hook when an active transcript path is present and empty; it still lacks the PR's archived-only/reset-pointer fallbacks in findPreviousSessionFile and the session-logs skill still documents/searches only active *.jsonl transcripts. The related #60409 is complementary gateway history work, not a replacement for this PR's hook and skill surfaces.

## Best Possible Solution

Keep #71537 open for maintainer review. The best path is to land or adapt its session-memory archive discovery and session-logs documentation, coordinate naming/lookup behavior with the complementary gateway archive fallback in #60409, and keep regression coverage for reset-pointer, archived-only, and active-file-preference cases.
