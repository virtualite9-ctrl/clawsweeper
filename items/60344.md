## Summary

Keep open. Current `main` still injects the exact `[image data removed - already processed by model]` sentinel into older `user`/`toolResult` history, runs that pruning before the next provider call, and still treats Kimi Coding / legacy `k2p5` as a vision-capable model. I found no current-main safeguard that escapes or strips this sentinel when it shows up as ordinary user/model text, so there is not strong evidence to close this as fixed or obsolete.
