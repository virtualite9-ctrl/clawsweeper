## Summary

Current `main` already requests Discord's `Guilds` gateway intent, and the `channel list` action intentionally requires a `guildId`/`--guild-id`. The reported root cause does not match the current implementation.

## Evidence

- **Gateway intents include Guilds:** The Discord gateway intent resolver on current `main` includes `GatewayIntents.Guilds` in the default bitmask.
  - file: [extensions/discord/src/monitor/gateway-plugin.ts:36](https://github.com/openclaw/openclaw/blob/5b8bd6371c666a5de43c82305e0315771e8e49d5/extensions/discord/src/monitor/gateway-plugin.ts#L36)
  - command: `nl -ba extensions/discord/src/monitor/gateway-plugin.ts | sed -n '1,220p'`
  - sha: [5b8bd6371c66](https://github.com/openclaw/openclaw/commit/5b8bd6371c666a5de43c82305e0315771e8e49d5)
- **Gateway plugin uses resolved intents:** The gateway plugin creation path passes the resolved intents into the Carbon gateway plugin options, so the active runtime configuration includes that `Guilds` intent.
  - file: [extensions/discord/src/monitor/gateway-plugin.ts:374](https://github.com/openclaw/openclaw/blob/5b8bd6371c666a5de43c82305e0315771e8e49d5/extensions/discord/src/monitor/gateway-plugin.ts#L374)
