## Summary

Keep open. Current `main` still leaves Docker sandbox CPU, memory, swap, and PID limits unset unless the operator configures them explicitly, so the requested default resource limits are not implemented in core yet.

## Evidence

- **Resolved sandbox defaults omit resource limits:** `resolveSandboxDockerConfig()` hard-codes defaults for image, workdir, read-only rootfs, tmpfs, network, and dropped capabilities, but `pidsLimit`, `memory`, `memorySwap`, and `cpus` fall through as `agentDocker ?? globalDocker` with no built-in fallback.
  - file: [src/agents/sandbox/config.ts:101](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/src/agents/sandbox/config.ts#L101)
  - command: `nl -ba src/agents/sandbox/config.ts | sed -n '80,135p'`
  - sha: [6bc0dc8fb671](https://github.com/openclaw/openclaw/commit/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440)
- **No built-in constants for sandbox resource ceilings:** Sandbox constants define defaults for image, container prefix, workdir, idle pruning, and browser settings, but there are no default CPU, memory, swap, or PID limit constants to apply automatically.
  - file: [src/agents/sandbox/constants.ts:7](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/src/agents/sandbox/constants.ts#L7)
  - command: `nl -ba src/agents/sandbox/constants.ts | sed -n '1,120p'`
  - sha: [6bc0dc8fb671](https://github.com/openclaw/openclaw/commit/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440)
- **Docker flags are only emitted when limits are configured:** `buildSandboxCreateArgs()` only appends `--pids-limit`, `--memory`, `--memory-swap`, and `--cpus` when those resolved config values are present and positive, so an unset default path produces containers without those resource caps.
  - file: [src/agents/sandbox/docker.ts:433](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/src/agents/sandbox/docker.ts#L433)
  - command: `nl -ba src/agents/sandbox/docker.ts | sed -n '428,448p'`
  - sha: [6bc0dc8fb671](https://github.com/openclaw/openclaw/commit/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440)
- **Docs show limits as example configuration, not enforced defaults:** The agent config docs include `pidsLimit`, `memory`, `memorySwap`, and `cpus` inside an example `agents.defaults.sandbox.docker` block, which documents how to set them manually but does not match current code behavior as automatic defaults.
  - file: [docs/gateway/config-agents.md:664](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/docs/gateway/config-agents.md#L664)
