## Summary

Keep open. Current main still does not implement the PR's central web-chat behavior: live `state: "error"` events are only stored in `lastError`, error-only assistant history entries still have no UI extraction fallback from `errorMessage`, and the renderer has no highlighted model-error bubble. The PR remains a plausible focused UI fix, but it needs rebase/adaptation against newer chat normalization and connect-error formatting before maintainer review.

## Best Possible Solution

Keep this PR open for maintainer review or fold its focused UI behavior into a fresh branch or #55923. The best implementation should append structured assistant error messages for live chat errors, render persisted assistant `stopReason: "error"` entries including empty-detail fallbacks, style model failures distinctly in web chat, preserve current structured connect-error formatting, and add focused controller/extraction/render tests on current main.
