## Summary

Keep open. Current main still does not implement the PR's central behavior: live web-chat error events only set `lastError`, persisted assistant error-only messages have no extraction/render fallback, and there is no highlighted inline error bubble styling or regression coverage. The PR remains a focused implementation candidate rather than conservative cleanup. Related broad PR #40150 is closed unmerged and does not supersede this PR; the prior bot review concerns appear addressed in the PR's latest patch shape.

## Best Possible Solution

Keep the PR open for maintainer review or fold the same narrow behavior into a fresh branch. The fix should append structured assistant error messages for live chat error events, render persisted assistant `stopReason: "error"` entries including empty-detail fallbacks, style them distinctly in web chat, and add focused controller/extraction/rendering tests on current main.
