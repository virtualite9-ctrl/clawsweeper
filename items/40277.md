## Summary

Keep open. Current main still does not implement the PR's central web-chat behavior: live chat error events update only hidden error state, persisted assistant error metadata is not extracted for display, and the grouped chat renderer has no highlighted model-error bubble. The PR remains a plausible focused UI fix, though it needs rebase/adaptation against current connect-error formatting and coordination with overlapping #55923.

## Best Possible Solution

Keep this PR open for maintainer review or fold its focused UI behavior into the canonical inline-error work. The best implementation should append structured assistant error messages for live chat errors, render persisted assistant `stopReason: "error"` entries including empty-detail fallbacks, style model failures distinctly in web chat, preserve current structured connect-error formatting, coordinate with #55923, and add focused controller/extraction/render tests on current main.
