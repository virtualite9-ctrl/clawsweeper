## Summary

Keep open. Current main does not implement the requested `agents.list[].name` fallback for the WebChat/Control UI assistant label. The UI label is loaded from `agent.identity.get`, and the server-side identity resolver only considers `ui.assistant.name`, `agents.list[].identity.name`, or workspace `IDENTITY.md`, not `agents.list[].name`.

## Risks / Open Questions

- Users who only set `agents.list[].name` still will not see that name reflected in the WebChat/Control UI assistant label on current main.
- The issue body is slightly imprecise about the supported config path: custom display naming is possible today via `agents.list[].identity.name` or `ui.assistant.name`, but not via the requested `agents.list[].name` field.
