## Summary

Keep #27445 open. Current main still lacks the requested public `announceTarget` config default and `sessions_spawn` per-call override, and the existing implementation intentionally preserves channel/external delivery for dormant top-level requesters while only active requester sessions or nested subagents get session-internal steering. A maintainer/Codex review comment on the issue already reached the same keep-open conclusion on April 26, 2026, and current main has not added the missing API surface.

## Best Possible Solution

Keep #27445 open as a core subagent announce policy request. The maintainer path is to add a public parent/channel announce policy to `sessions_spawn` and `agents.defaults.subagents`, update config schema/help/generated docs and public subagent docs, and cover top-level parent-only routing, current channel delivery, active requester steering, nested subagent delivery, and interactions with #33478 callback/thread hardening.
