## Summary

Keep #27445 open. Current main still lacks the requested native `announceTarget` option for `sessions_spawn` or `agents.defaults.subagents`, and the shipped behavior remains channel/direct-delivery first for top-level completion requesters, with only nested requester subagents getting internal parent-session injection. The related #33478 and #8299 items are complementary, not canonical replacements for this routing-policy request.

## Best Possible Solution

Keep #27445 open as a core subagent announce-routing policy request. The likely implementation path is to add a backwards-compatible public `announceTarget` policy with default `"channel"` and opt-in `"parent"` at both `sessions_spawn` and `agents.defaults.subagents`, update strict config/schema/generated docs, and add regression coverage for current channel delivery, top-level parent-only routing, active requester steering, nested subagent delivery, ACP `streamTo` separation, and interactions with #33478/#8299.
