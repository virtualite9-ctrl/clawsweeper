## Summary

Current `main` does not implement the requested configurable WhatsApp message-send retry. The WhatsApp config surface has no `channels.whatsapp.retry` setting, the outbound send path issues a single send and rethrows on failure, and the test suite explicitly locks in no-retry behavior for transient send errors.

## Evidence

- **WhatsApp config schema has no outbound retry setting:** `WhatsAppConfigSchema` includes DM/group/media/actions fields, but no `retry` field. For comparison, Telegram and Discord schemas on current `main` both expose `retry: RetryConfigSchema`.
