## Summary

Keep #40044 open. Current main dce35b90fe8fa09ea51a666195b7645c260d6cbf still does not expose the current session UUID in the system-prompt Runtime line, and the PR remains a narrow additive agents prompt change rather than obsolete, duplicate, or ClawHub-scoped work.

## Best Possible Solution

Keep the PR open, or implement the same behavior directly on main: add sessionId?: string to the prompt runtime types, pass targetSessionEntry?.sessionId through buildSystemPromptParams, render session=<uuid> in buildRuntimeLine, and add runtime-line test coverage for the new segment.
