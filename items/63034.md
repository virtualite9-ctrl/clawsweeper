## Summary

Current main still does not implement the requested OpenRouter behavior. The OpenRouter Anthropic wrapper calls a helper that marks only system/developer messages and strips assistant thinking-block cache markers; it does not call the direct Anthropic path's trailing-user-message cache marker helper. Current tests and docs also encode/document system/developer-only OpenRouter cache markers. The provided GitHub context shows same-author PR #63062 is still open, unmerged, and uses closing syntax for this issue, so the maintainer path is to review/land or reject that PR rather than close this issue now.

## Best Possible Solution

Review PR #63062 against current main. If maintainers accept the behavior, land a focused change that reuses the existing trailing-user-message cache-control helper on the OpenRouter Anthropic path and updates regression coverage so user string and array content is marked. If maintainers intentionally want OpenRouter to remain system/developer-only, document that product decision in the issue and docs, then close or retitle accordingly.
