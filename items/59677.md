## Summary

Keep open. I did not find strong evidence that current `main` already supports Feishu bot-to-bot inbound replies. The current Feishu docs and live inbound path are still centered on human/user IDs, while the only explicit `senderType: "app"` handling I found is for fetched thread/history context, not for receiving and routing a live `im.message.receive_v1` event from another bot account.

## Evidence

- **Docs describe current Feishu support in human-centric terms:** The channel docs say Feishu is production-ready for bot DMs and group chats, and the sender restriction example is keyed to user `open_id` values (`ou_xxx`). There is no documented bot-to-bot receive/reply capability or configuration.
  - file: [docs/channels/feishu.md:13](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/docs/channels/feishu.md#L13)
  - command: `nl -ba docs/channels/feishu.md | sed -n '1,145p'`
  - sha: [ec3dbd22a4de](https://github.com/openclaw/openclaw/commit/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46)
- **Inbound Feishu event type only models user-style sender IDs:** `FeishuMessageEvent.sender.sender_id` exposes `open_id`, `user_id`, and `union_id`; there is no dedicated app/bot sender identifier in the inbound event model.
  - file: [extensions/feishu/src/event-types.ts:1](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/extensions/feishu/src/event-types.ts#L1)
  - command: `nl -ba extensions/feishu/src/event-types.ts | sed -n '1,40p'`
  - sha: [ec3dbd22a4de](https://github.com/openclaw/openclaw/commit/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46)
- **Live inbound routing derives sender identity only from open_id/user_id:** `parseFeishuMessageEvent()` builds `senderId`/`senderOpenId` from `event.sender.sender_id.open_id` and `user_id` only. If Feishu represents bot-originated senders differently, the current receive path has no explicit branch for that case.
  - file: [extensions/feishu/src/bot.ts:100](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/extensions/feishu/src/bot.ts#L100)

That special handling appears only in quote/thread/history fetch paths, not in the main inbound event path.
