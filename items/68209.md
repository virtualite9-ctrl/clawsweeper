## Summary

Keep this issue open. Current main has adjacent Codex harness hardening for runtime pinning, auth profile forwarding, and cumulative usage filtering, but it does not prove the reported route-switch regression is fixed end to end, and stale/unknown native Codex context reporting remains a live risk tracked by related open work.

## Best Possible Solution

Keep this issue open as a focused child of #66251. The best maintainer path is to reproduce the current-main route switch in a clean session, capture `/status`, `agentHarnessId`, `totalTokensFresh`, native Codex app-server auth/runtime events, and Codex login status before and after the switch, then either close with end-to-end proof or split fixes for stale context reporting, native compaction freshness, thread/runtime isolation, and auth-mode isolation.
