## Summary

Keep this issue open. Current main still has the un-serialized plugin binding approval persistence path: `saveApprovals()` updates the shared in-memory cache before awaiting `writeJsonAtomic(...)`, and `addPersistentApproval()` still performs a read-modify-write over that cache. Race A and Race B still match the issue's low-severity mitigated descriptions, but Race C remains a concrete, actionable core bug.

## Best Possible Solution

Keep this issue open and fix Race C in `src/plugins/conversation-binding.ts` by serializing the persistent approvals read-modify-write path, for example with a promise queue or mutex stored in `PluginBindingGlobalState`. Add a focused regression test that forces two concurrent `allow-always` approval writes to complete out of order and verifies both the in-memory cache and reloaded `plugin-binding-approvals.json` reflect the intended latest state. Then decide separately whether Race A and Race B need documentation or should remain accepted mitigated behavior.
