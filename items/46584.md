## Summary

Keep #46584 open. Current main still appears to contain the reported false-positive path: explicit `memorySearch.provider: "ollama"` is not special-cased in `openclaw doctor`, so it falls through to the generic remote-provider API-key warning even though the Ollama memory embedding runtime treats the key as optional.

## Best Possible Solution

Keep the issue open and apply a narrow doctor fix: treat explicit `provider: "ollama"` as key-optional, likely similar to the existing `lmstudio`/local-server handling, and add a regression test for `provider: "ollama"` with no CLI key and a ready gateway probe producing no missing-API-key warning. Preserve API-key support for authenticated Ollama endpoints by relying on the existing runtime key resolution and gateway readiness signal.
