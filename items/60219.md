## Summary

Keep open. Current `main` still shows the inconsistent gateway plugin load-mode threading described in the report, so the issue is not implemented on `main` and there is strong code-level evidence that the duplicate-load/cache-key path can still occur.

## Evidence

- **Startup load enables prefer-setup mode when deferred channel plugins exist:** Gateway startup passes `preferSetupRuntimeForChannelPlugins: deferredConfiguredChannelPluginIds.length > 0` into `loadGatewayStartupPlugins(...)` during initial plugin load.
  - file: [src/gateway/server-startup-plugins.ts:92](https://github.com/openclaw/openclaw/blob/3ffd944e6ba6dfe4c7c4c275848f1c0a1f246e28/src/gateway/server-startup-plugins.ts#L92)
  - command: `nl -ba src/gateway/server-startup-plugins.ts | sed -n '90,108p'`
  - sha: [3ffd944e6ba6](https://github.com/openclaw/openclaw/commit/3ffd944e6ba6dfe4c7c4c275848f1c0a1f246e28)
- **Deferred reload still omits that flag on current main:** The later deferred reload path calls `reloadDeferredGatewayPlugins(...)` without any `preferSetupRuntimeForChannelPlugins` argument, even when `deferredConfiguredChannelPluginIds.length > 0` triggered the second load.
  - file: [src/gateway/server.impl.ts:770](https://github.com/openclaw/openclaw/blob/3ffd944e6ba6dfe4c7c4c275848f1c0a1f246e28/src/gateway/server.impl.ts#L770)
  - command: `nl -ba src/gateway/server.impl.ts | sed -n '768,783p'`
  - sha: [3ffd944e6ba6](https://github.com/openclaw/openclaw/commit/3ffd944e6ba6dfe4c7c4c275848f1c0a1f246e28)
- **Bootstrap API explicitly prevents forwarding the flag during deferred reload:** `reloadDeferredGatewayPlugins` is typed to omit `preferSetupRuntimeForChannelPlugins`, so the deferred path cannot preserve the initial startup mode through this helper.
  - file: [src/gateway/server-plugin-bootstrap.ts:107](https://github.com/openclaw/openclaw/blob/3ffd944e6ba6dfe4c7c4c275848f1c0a1f246e28/src/gateway/server-plugin-bootstrap.ts#L107)
  - command: `nl -ba src/gateway/server-plugin-bootstrap.ts | sed -n '28,116p'`
  - sha: [3ffd944e6ba6](https://github.com/openclaw/openclaw/commit/3ffd944e6ba6dfe4c7c4c275848f1c0a1f246e28)
- **Loader cache key still depends on startupChannelMode:** The plugin loader cache key still includes `startupChannelMode`, derived from `preferSetupRuntimeForChannelPlugins`, so differing values across the two loads still produce different cache keys.
  - file: [src/plugins/loader.ts:826](https://github.com/openclaw/openclaw/blob/3ffd944e6ba6dfe4c7c4c275848f1c0a1f246e28/src/plugins/loader.ts#L826)

  - file: [src/gateway/server-plugins.test.ts:878](https://github.com/openclaw/openclaw/blob/3ffd944e6ba6dfe4c7c4c275848f1c0a1f246e28/src/gateway/server-plugins.test.ts#L878)
