## Summary

Keep open. Current `main` at `3ffd944e6ba6dfe4c7c4c275848f1c0a1f246e28` still exits unconditionally from the global `uncaughtException` handlers, while transient-network suppression exists only on the `unhandledRejection` path. The existing tests also assert that uncaught exceptions exit, so the reported gap is still present rather than already fixed on `main`.

## Evidence

- **Main entrypoint still exits on every uncaught exception:** `src/index.ts` logs the uncaught exception, runs fatal hooks, restores terminal state, and always calls `process.exit(1)` with no `isTransientNetworkError` or `isTransientUnhandledRejectionError` check.
