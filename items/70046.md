## Summary

Keep PR #70046 open. Current main still does not support bare HH:MM/HH:MM:SS values in `cron add --at`, and the PR is a focused, still-relevant fix for the linked bug #59441 rather than obsolete or duplicate landed work.

## Best Possible Solution

Keep #70046 open for maintainer review or rebase. The best path is to land a focused cron CLI parser fix that adds HH:MM/HH:MM:SS handling in `parseAt`, resolves time-only input against today's date in the requested IANA timezone, returns null for invalid zones/out-of-range values, and adds the PR's focused regression coverage. The related help-text wording can remain tracked separately by the canonical help-text item if maintainers want that split.
