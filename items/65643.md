## Summary

Keep open. Current main still preserves the reported failure path: `--profile` sets `OPENCLAW_PROFILE` but leaves an inherited `OPENCLAW_LAUNCHD_LABEL` intact, and launchd status resolution still prefers that label before the profile-derived label. Open PR #66000 is an unmerged implementation candidate with closing syntax, so this issue should remain open until that PR is reviewed and either landed or rejected.

## Best Possible Solution

Keep this issue open and use PR #66000 as the current implementation candidate. The best fix is to make an explicit `--profile` unable to be overridden by a stale inherited `OPENCLAW_LAUNCHD_LABEL`, while preserving any intentional matching-label behavior maintainers want, and to land regression tests for conflicting, matching, empty, and absent label cases before closing this as implemented.
