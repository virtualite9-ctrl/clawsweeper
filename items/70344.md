## Summary

Keep #70344 open. Current main has commentary-phase suppression and preserves established inter-session routes, but it does not implement this PR's broader behavior: unphased `toolUse` assistant text is still only suppressed when phase metadata says `commentary`, and `sessions_send` still starts the A2A announce flow that can send to an external channel. Related #70372 only covers an Anthropic/Bedrock thinking-narration subset and does not supersede the `sessions_send` boundary work. The PR should not be accepted as-is without addressing the review feedback, but the underlying bug/fix area remains actionable in this repo.

## Best Possible Solution

Keep the PR open but require a narrower maintainer-quality revision: add source-backed suppression for unphased tool-use assistant text, handle self-instruction/prelude stripping with a conservative tested heuristic and no personal/debug artifacts, and make `sessions_send` replies session-local or non-user-deliverable for the initiating control turn while preserving established external routes for later legitimate announcements. Keep #70372 as a separate subset/follow-up for provider thinking narration unless it is expanded to cover this whole boundary surface.

## Risks / Open Questions

- The PR branch has unresolved review feedback: a hardcoded personal name in the sanitizer heuristic, a broad monologue-stripping false-positive risk, and a route-persistence concern around blocking external-route inheritance by overwriting stored lastChannel/lastTo.
- Removing the A2A announce flow wholesale may regress intentional peer-agent follow-up delivery; the safer direction is to suppress outward delivery for the current `sessions_send` control turn without corrupting the target session's remembered external route.
