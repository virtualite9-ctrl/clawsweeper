## Summary

Keep #38625 open. Current main still only partially overlaps the request: the WebSocket handshake exposes a gateway version and the WS control plane has `update.run`, but the requested HTTP surfaces are not implemented. `/health` and `/healthz` still return only `{ ok, status }`, there is no `POST /admin/update` HTTP route, and the typed `update.run` RPC does not accept `target_version`.

## Best Possible Solution

Keep #38625 open until maintainers either implement the requested HTTP API or explicitly decide and document a WS-only management direction. A conservative implementation would add `version` to `/health` and `/healthz` with HTTP probe tests/docs, then add authenticated, rate-limited `POST /admin/update` as a thin wrapper over the existing update runner. If version pinning is accepted, extend the typed update schema and runner target/tag contract rather than adding ad hoc package-manager behavior.
