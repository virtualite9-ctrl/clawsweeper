## Summary

Keep open. Current main still only partially overlaps #38625: OpenClaw has a runtime version source, WS status/health surfaces, and an admin-scoped WS `update.run` RPC, but the requested HTTP contract is still missing. `/health`/`/healthz` do not include `version`, there is no `POST /admin/update` route, and `update.run` does not accept `target_version` or equivalent version-pinning input.

## Best Possible Solution

Keep #38625 open until maintainers either implement the requested HTTP API or explicitly choose a WS-only direction. A conservative implementation would add a source-backed `version` field to `/health` and likely `/healthz` with tests/docs, then add `POST /admin/update` only as an authenticated/rate-limited wrapper around the existing `update.run`/`runGatewayUpdate` path. If version pinning is required, extend the typed update schema and runner contract rather than adding ad hoc host package-manager behavior.
