## Summary

Keep open. Current main still only partially overlaps the request: OpenClaw exposes runtime version over WS/status/probe surfaces and has an admin-scoped WS `update.run`, but `GET /health`/`/healthz` still omit `version`, there is no `POST /admin/update` HTTP route, and the typed update RPC still does not accept `target_version`.

## Best Possible Solution

Keep #38625 open until maintainers either implement the requested HTTP API or explicitly choose and document a WS-only direction. A conservative implementation would add source-backed `version` to `/health` and likely `/healthz` with tests/docs, then add `POST /admin/update` only as an authenticated/rate-limited wrapper around the existing update runner. If version pinning is accepted, extend the typed update params and runner tag/target contract instead of adding ad hoc package-manager behavior.
