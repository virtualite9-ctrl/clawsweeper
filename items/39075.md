## Summary

Keep open. Current main still does not implement the central Telegram/runtime performance changes from PR #39075, and the related/open items do not supersede it. Some session-recording behavior has partially improved in core, but the Telegram body path, command detection, envelope formatting, typing lifecycle, and workspace bootstrap paths still lack the PR's main optimizations.

## Best Possible Solution

Keep the PR open as useful prior art, but ask for a focused rebase onto current main. Split Telegram-specific pipeline optimizations from shared core/runtime and agents benchmark work, preserve current routing/config-refresh behavior, and add scoped tests or benchmarks for each optimization still worth landing.
