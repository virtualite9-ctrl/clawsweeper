## Summary

This PR is a large optional Slack/Telegram work-intent orchestration layer, not a missing core capability. Current main already exposes the generic pieces needed for a ClawHub plugin or skill: inbound/message hooks, synthetic replies, subagent runtime, and Task Flow. The PR also carries unresolved core lifecycle review blockers in the dispatcher path, so it should not be merged as core cleanup.

## Best Possible Solution

Close this core PR and move the idea to a ClawHub plugin/skill using inbound_claim or before_agent_reply hooks, api.runtime.taskFlow for durable stage tracking, and api.runtime.subagent for research/spec/builder/verifier runs. If that implementation finds a missing generic plugin seam, open a small focused core PR for that API instead of landing the whole workflow in core.
