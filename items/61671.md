## Summary

Current main already implements this request via `models.providers.*.models.*.compat.supportsPromptCacheKey`, which disables prompt-cache stripping for OpenAI-compatible proxy/local Responses endpoints while preserving the default safety behavior for incompatible proxies. The change is also recorded in the shipped `v2026.4.23` changelog.

## Evidence

- **Prompt-cache stripping now has an explicit compat override:** `resolveProviderRequestCapabilities()` reads `compat.supportsPromptCacheKey` and returns `shouldStripResponsesPromptCache = false` when that flag is `true`, instead of always stripping on proxy-like Responses endpoints.
