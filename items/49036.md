## Summary

Keep #49036 open. Current main still has a gateway lock file surface and startup-time stale-lock mitigation, but `openclaw doctor` only wires session lock health. The requested doctor warning/`--fix` coverage for `$TMPDIR/openclaw-$UID/gateway.*.lock` is not implemented on main, and open PR #49112 is the direct implementation track rather than a shipped replacement for the issue.

## Best Possible Solution

Keep the issue open until a focused doctor implementation lands. The best path is to revive or replace PR #49112 with a current-main patch that exposes a gateway lock inspection helper, reports live/dead/malformed/old gateway lock state in `openclaw doctor`, removes stale or malformed gateway locks in `--fix`/`--repair`, and adds targeted tests beside `doctor-session-locks` and `gateway-lock` coverage.
