## Summary

Current `main` has better structured connect errors and an in-memory post-connect event log, but it does not implement the requested Control UI debug/diagnostic mode for pre-auth connection lifecycle logging, copyable ephemeral diagnostics, or a user-facing debug toggle. The issue should stay open.

## Evidence

- **Pre-auth handshake events are not surfaced to the UI event log:** `GatewayBrowserClient.handleMessage()` special-cases `connect.challenge`, uses it only to trigger `sendConnect()`, and returns before calling `onEvent`. That means the pre-auth handshake frame is not recorded in the UI event log path.
  - file: [ui/src/ui/gateway.ts:537](https://github.com/openclaw/openclaw/blob/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4/ui/src/ui/gateway.ts#L537)
  - sha: [f3c37a946cc4](https://github.com/openclaw/openclaw/commit/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4)
- **UI event log only records post-dispatch gateway events:** `handleGatewayEventUnsafe()` appends entries only for events delivered through `onEvent`, so the browser-side log is limited to gateway events that already made it through the client event path.
  - file: [ui/src/ui/app-gateway.ts:503](https://github.com/openclaw/openclaw/blob/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4/ui/src/ui/app-gateway.ts#L503)
  - sha: [f3c37a946cc4](https://github.com/openclaw/openclaw/commit/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4)
- **Connection failures still collapse to a single displayed error string:** `onClose` stores `lastError`/`lastErrorCode`, but there is no connection transcript, close-history panel, or handshake detail buffer exposed for failed pre-auth connects.
  - file: [ui/src/ui/app-gateway.ts:338](https://github.com/openclaw/openclaw/blob/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4/ui/src/ui/app-gateway.ts#L338)
  - sha: [f3c37a946cc4](https://github.com/openclaw/openclaw/commit/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4)
- **Existing debug UI is a generic event list, not a diagnostic mode:** The Debug tab renders an `Event Log` section labeled `Latest gateway events.` There is no enable/disable diagnostic toggle, no copy-to-clipboard action, and no dedicated connection-debug drawer in this view.
  - file: [ui/src/ui/views/debug.ts:118](https://github.com/openclaw/openclaw/blob/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4/ui/src/ui/views/debug.ts#L118)
  - sha: [f3c37a946cc4](https://github.com/openclaw/openclaw/commit/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4)
- **Current documented logging surface is connected `logs.tail`, not pre-auth browser diagnostics:** The docs list `Debug: status/health/models snapshots + event log + manual RPC calls` and `Logs: live tail of gateway file logs with filter/export (logs.tail)`. That is not the requested browser-side pre-auth connection diagnostic mode.
  - file: [docs/web/control-ui.md:134](https://github.com/openclaw/openclaw/blob/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4/docs/web/control-ui.md#L134)
  - sha: [f3c37a946cc4](https://github.com/openclaw/openclaw/commit/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4)
- **No clipboard support is wired into the relevant Control UI debug/overview surfaces:** A search for clipboard writes in the relevant UI surfaces finds copy actions elsewhere in the app, but not in `ui/src/ui/views/debug.ts` or `ui/src/ui/views/overview*`, so the requested copyable diagnostic log is not present.
  - command: `rg -n "copy to clipboard|writeText\(|clipboard" ui/src/ui/views/overview* ui/src/ui/views/debug.ts ui/src/ui -g '!**/*.test.ts'`
  - sha: [f3c37a946cc4](https://github.com/openclaw/openclaw/commit/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4)
