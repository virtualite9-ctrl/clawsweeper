## Summary

Current main still lacks first-class `--thread-id` support on `openclaw cron add/create` and `openclaw cron edit`. Telegram topic delivery exists underneath through `:topic:` target parsing and generic `delivery.threadId`, and `openclaw message send` exposes `--thread-id`, but the cron CLI does not. The provided GitHub context also shows open, unmerged PR #60890 with closing syntax for this issue, so the issue should stay open until maintainers review, land, or close that PR.

## Best Possible Solution

Keep this issue open while maintainers review #60890. The best product/code path is a focused cron CLI change that adds `--thread-id` to `cron add/create` and `cron edit`, composes Telegram topic targets as `<chatId>:topic:<threadId>` or stores an equivalent `delivery.threadId`, preserves existing manual `:topic:` targets, rejects invalid/non-Telegram combinations early, and adds regression coverage in `src/cli/cron-cli.test.ts`.
