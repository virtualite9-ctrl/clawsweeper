## Summary

Keep open. Current main still does not expose `--thread-id` on `openclaw cron add/create` or `openclaw cron edit`, while PR #60890 is an open, unmerged implementation candidate that explicitly says it closes this issue. The right maintainer path is to review/land or close that PR, not close the issue early.

## Best Possible Solution

Keep this issue open until PR #60890 is either reviewed and landed, or closed with a maintainer decision. The best code path is a focused cron CLI change that adds `--thread-id` to `cron add/create` and `cron edit`, composes Telegram targets as `<chatId>:topic:<threadId>`, preserves existing manual `:topic:` targets, validates invalid combinations early, and keeps regression coverage in `src/cli/cron-cli.test.ts`.
