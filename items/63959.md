## Summary

Keep open. Current `main` does not contain the PR's Dockerfile fix, and there is not strong evidence to close this as obsolete or already implemented. The documented Docker path mostly avoids the reported failure through bind mounts plus a pre-start root `chown` in `scripts/docker/setup.sh`, but that is not the same as having the image itself pre-create `/home/node/.openclaw` for direct named-volume mounts.

That means the exact change from PR #63959 is not already present.
