## Summary

Keep this PR open. Current main still does not pre-create `/home/node/.openclaw` in the runtime image before switching to `USER node`, while the config writer still creates temporary `openclaw.json.*.tmp` files under the resolved state/config directory. The existing Docker setup script has a bind-mount chown flow, but that does not make this image-level named-volume fix obsolete for independent Compose deployments like the linked bug report.

## Best Possible Solution

Review and land this PR, or an equivalent Dockerfile change, as the implementation candidate for #61279. Keep the image-level `install -d -o node -g node -m 0755 /home/node/.openclaw` before `USER node`, retain an active-release changelog entry if it lands, and consider documenting that already-created root-owned volumes still need a one-time chown.
