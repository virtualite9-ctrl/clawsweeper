## Summary

Keep this issue open. Current main still has the reported boundary: Google Gemini API-key setup can set Gemini as the primary default model, while heartbeat runs with an actionable HEARTBEAT.md call the normal reply path without a heartbeat model override unless one is explicitly configured. The empty/comment-only HEARTBEAT.md skip and optional cost controls reduce some background calls, but they do not protect existing actionable heartbeat files from a paid-provider default-model switch.

## Best Possible Solution

Keep this issue open and fix the boundary explicitly. The best maintainer path is to preserve an existing primary model during Google/provider auth unless the user clearly confirms the switch, or to require confirmation/disablement before heartbeat inherits a newly configured paid external provider. Add regression coverage for an existing actionable HEARTBEAT.md plus paid-provider default-model setup.
