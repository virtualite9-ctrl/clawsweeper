## Summary

Keep this issue open. Current main still limits `cron.sessionRetention` to cron-run session keys; completed keep-mode subagent sessions and tracked ACP sessions do not have an equivalent restart-safe retention sweep. The related implementation PR #51861 is closed unmerged in the provided GitHub context, and #52327 was closed as a duplicate rather than becoming an open canonical tracker.

## Best Possible Solution

Keep this issue open and implement a restart-safe retention path for completed subagent and ACP session-store entries. The likely direction is a generic ephemeral-session reaper or a documented subagent/ACP retention policy that uses terminal markers such as `endedAt`, preserves active and intentionally persistent sessions, deletes or archives transcripts consistently, and adds focused tests for expired cron runs, completed subagent runs, running subagents, keep/session-mode subagents, and ACP keys. The closed PR #51861 can be used as prior art, but it should be reviewed against current main and the restart-safety details raised in #52327.
