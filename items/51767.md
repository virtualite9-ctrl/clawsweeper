## Summary

Keep this issue open. Current main still scopes `cron.sessionRetention` to cron run keys only, while default subagent and ACP session rows can persist outside that reaper. Newer subagent registry cleanup handles some registry/archive bookkeeping, but it is not an equivalent restart-safe session-store retention path for completed subagent/ACP sessions.

## Best Possible Solution

Keep this issue open and implement a restart-safe session-store-level retention path for completed subagent and ACP sessions. The likely fix is a generic ephemeral-session reaper or documented subagent/ACP retention setting that preserves active and intentionally persistent sessions, uses durable completion markers such as `endedAt` or `cleanupCompletedAt`, deletes or archives transcripts consistently through the existing session deletion/archive helpers, runs on startup/store-write or another persistent catch-up path, and adds focused tests for expired cron runs, completed subagent/ACP runs, running subagents, keep/session-mode behavior, and transcript cleanup. PR #51861 remains useful prior art but needs review against current main and the restart-safety details from #52327.

## Risks / Open Questions
