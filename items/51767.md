## Summary

Keep this issue open. Current main still wires `cron.sessionRetention` only to cron-run session keys; completed keep-mode subagent and ACP session-store rows do not have an equivalent restart-safe retention sweep. The related fix PR #51861 was closed unmerged, and #52327 is now closed as a duplicate rather than an open canonical tracker.

## Best Possible Solution

Keep this issue open and implement a restart-safe retention path for completed subagent and ACP session-store entries. The likely code path is a generic ephemeral-session reaper or a new documented subagent/ACP retention policy that uses terminal markers such as `endedAt`/`status`, preserves active and persistent sessions, archives/deletes transcripts consistently, and adds focused tests for cron, completed subagent, running subagent, session-mode subagent, and ACP keys.
