## Summary

Current main still matches the reported Control UI stale-state bug. A compaction start sets active status with no active-state timeout, the renderer shows active/retrying compaction without an age check, and terminal/reconnect/clear-history paths still reset chat stream/run state without consistently clearing compactionStatus or fallbackStatus. Related #39908 was broader and closed, but it does not supersede this narrower missed-end Control UI state issue.

## Best Possible Solution

Keep this issue open and fix it in the Control UI state layer. Add a run/session-scoped stale timeout or watchdog for active/retrying compaction, centralize cleanup of compactionStatus/fallbackStatus plus their timers, call that cleanup on terminal chat events, reconnect/disconnect, session/agent switch, clear/reset, and send-failure paths, then add regression tests for missed compaction end after final reply plus reconnect/session-switch/clear-history cleanup.
