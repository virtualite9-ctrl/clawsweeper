## Summary

Keep open. Current `main` has partial mitigation for stale compaction UI state, but it does not fully implement the reported cleanup paths. Session switches clear `compactionStatus`/`fallbackStatus`, and completed compactions auto-clear after 5s, yet active/retrying compaction has no stale timeout and reconnect/reset/send-failure paths still leave those badges untouched.

## Evidence
