## Summary

Keep open. Current main still matches the reported Control UI stale-state bug: a compaction start can set an active status with no active-state watchdog, the renderer shows active/retrying compaction without an age check, and several terminal/reconnect/reset paths clear chat run state without consistently clearing compactionStatus/fallbackStatus. Related #39908 was broader and is closed, but it does not supersede this narrower missed-end Control UI state issue.

## Best Possible Solution

Keep this issue open and fix it in the Control UI state layer. Add a run/session-scoped stale timeout or watchdog for active compaction, centralize cleanup of compactionStatus/fallbackStatus plus their timers, call that cleanup on terminal chat events, reconnect/disconnect, session/agent switch, clear/reset, and send-failure paths, then add regression tests for missed compaction end after final reply plus reconnect/session-switch/clear-history cleanup.
