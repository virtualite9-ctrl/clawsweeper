## Summary

Current main still matches the reported Control UI stale compaction-status bug. The session-switch helper now clears compaction/fallback status, but active/retrying compaction still has no stale timeout and several terminal, reconnect, overview session-change, clear-history, and send-failure cleanup paths still clear run/stream state without clearing compactionStatus or fallbackStatus.

## Best Possible Solution

Keep this issue open and fix it in the Control UI state layer. Add a run/session-scoped stale timeout or watchdog for active/retrying compaction, centralize timer-aware cleanup for compactionStatus and fallbackStatus, call that cleanup from terminal chat events, reconnect/disconnect, overview/chat session changes, agent switches, clear-history/reset, and send-failure paths, then add focused regression tests for missed compaction end after a final reply plus reconnect and clear-history cleanup.
