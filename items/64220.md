## Summary

Current main still matches the reported stale Control UI compaction state bug. The UI sets active/retrying compaction state without a stale timeout, terminal chat/reconnect/clear-history paths still clear run/stream state without clearing compaction/fallback status, and the related #39908 is a closed broader symptom report rather than a canonical shipped fix for this narrower state-machine issue.

## Best Possible Solution

Keep this issue open and fix it in the Control UI state layer. Add a run/session-scoped stale timeout or watchdog for active/retrying compaction, centralize cleanup of compactionStatus/fallbackStatus and their timers, call that cleanup from terminal chat events, reconnect/disconnect, session/agent switches, overview/chat clear-history, reset, and send-failure paths, then add focused regression tests for missed compaction end after final reply plus reconnect and clear-history cleanup.
