## Summary

Keep #71630 open. Current main still lacks a compact Web UI composer runtime status indicator: the composer renders attach/voice/talk/token controls plus send/stop controls, while runtime state remains client-local `chatRunId`/`chatStream`/`chatSending` lifecycle state. The broader gateway-owned per-session activity API mentioned by the issue (#39127) is still open, and #71631 is only a related broader responsiveness request, not a shipped implementation of this chip.

## Best Possible Solution

Keep #71630 open as the focused Web UI composer UX tracker. The best path is to add a compact, accessible composer status chip backed by a session-scoped activity source when #39127 lands; if implemented earlier, keep it explicitly provisional by deriving from existing `chatRunId`/`chatStream`/abort/reconnect lifecycle state without message-text heuristics, and add focused UI/controller tests for in-progress, done, interrupted, reconnect, and mobile layout behavior.
