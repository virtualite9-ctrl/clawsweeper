## Summary

Keep #71630 open. Current main has Web UI chat lifecycle state and a Talk-mode status row, but it does not render a compact runtime status chip beside the composer action buttons, and it lacks the requested Done/Interrupted visible composer states. The related #39127 gateway activity API is still separate prerequisite/API work, while this issue tracks a distinct Web UI surface.

## Best Possible Solution

Keep this issue open as the Web UI composer implementation request. Implement a compact status chip in `ui/src/ui/views/chat.ts` with local send/final/abort/reconnect signals for an initial version if acceptable, then wire it to the future session-scoped activity API from #39127 when that API lands. Keep #52640 separate for channel-message task-status surfaces.
