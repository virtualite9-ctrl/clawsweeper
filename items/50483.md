## Summary

Keep this PR open. Current main has some overlapping chat-session fixes, including chat terminal events carrying session keys and history refreshes, but it does not fully implement the PR's central changes: stable streaming parser segment IDs, debounced streaming scroll updates, decoded agent-event session keys, or the PR's session-key/run acceptance model for live agent streams. The PR is still a plausible implementation candidate, but it needs maintainer review against current main and should split or justify the unrelated Windows installer change.

## Best Possible Solution

Keep this PR open as an implementation candidate. The best path is to rebase onto current main, drop or split the scripts/install.ps1 change, preserve the still-missing stable parser IDs and streaming-scroll debounce where they remain valid, and settle the ChatViewModel live-stream routing contract around decoded agent-event sessionKey, pending run IDs, and session switching with focused OpenClawKit tests plus an iOS build.
