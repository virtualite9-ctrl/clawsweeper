## Summary

Keep open. Current main still reconciles any persisted ACP session whose identity is pending and has a stable session id, without skipping one-shot or terminal records first, and gateway startup still logs the warning when any records are checked. The related fix PR #72023 is open/unmerged, so the issue remains the canonical bug until that or an equivalent fix lands.

## Best Possible Solution

Keep #72013 open as the canonical issue, review and validate PR #72023 or an equivalent patch that skips startup identity reconciliation for terminal one-shot/closed ACP records, and add regression coverage in `src/acp/control-plane/manager.test.ts` for those skipped records. After merge, close the issue via the PR or as implemented-on-main with the landed commit SHA.
