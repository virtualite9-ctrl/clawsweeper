## Summary

Keep open. Current main still runs ACP startup identity reconciliation for every persisted ACP session whose identity is pending and has a stable ACP/agent session id, without checking `acp.mode === "oneshot"` or terminal/closed state first. The existing main branch does have a newer fix that reconciles identities before closing newly completed one-shot turns, but that does not address the issue’s reported stale terminal one-shot records that are already persisted as pending. The related PR #72023 directly targets this issue, but it is open and not present in the current main checkout.

## Best Possible Solution

Keep #72013 open and review/land PR #72023 or an equivalent main-branch fix that skips persisted terminal one-shot and closed ACP records before startup runtime ensure/reconcile, with regression tests proving the reconciler returns `checked: 0` and does not call the runtime for those records. Keep the existing post-turn one-shot identity reconciliation because it addresses newly completed runs, but add the startup guard for stale historical metadata.
