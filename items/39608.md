## Summary

Keep #39608 open. Current main still passes `syncFullHistory: false` to Baileys without an explicit `shouldSyncHistoryMessage`, while the pinned Baileys 7.0.0-rc.9 socket code derives the omitted predicate from `syncFullHistory`. The issue comments provide recent, concrete linked-device group-silent repro evidence, and I found adjacent group-routing tests/docs but no shipped fix for the socket history-sync gap.

## Best Possible Solution

Keep the issue open until the WhatsApp plugin deliberately fixes or documents linked-device group inbound behavior. The likely maintainer path is to update `extensions/whatsapp/src/session.ts` to pass an explicit reviewed `shouldSyncHistoryMessage` predicate or expose a product-backed config choice, add a focused regression assertion in `extensions/whatsapp/src/session.test.ts` for the `makeWASocket` options, and run a live linked-device group smoke before closing as fixed. If maintainers decide this mode is unsupported, document that clearly in the WhatsApp troubleshooting docs instead of closing as stale.
