## Summary

Keep open. Current `main` still creates the Baileys socket with `syncFullHistory: false`, the repo still has no `shouldSyncHistoryMessage` override anywhere, and the WhatsApp inbound path still depends on Baileys delivering `messages.upsert` events for group traffic. I found no relevant fix between the latest release commit and current `main`.

## Evidence

- **Baileys bootstrap still disables full history sync:** `extensions/whatsapp/src/session.ts` still passes `syncFullHistory: false` into `makeWASocket`, with no adjacent history-sync callback or linked-device group workaround.
  - file: [extensions/whatsapp/src/session.ts:149](https://github.com/openclaw/openclaw/blob/cb2c36b04975e34ab7014a11575d6589aa96ae32/extensions/whatsapp/src/session.ts#L149)
- **No `shouldSyncHistoryMessage` implementation on main:** Repo-wide search for `shouldSyncHistoryMessage|syncFullHistory` returned only the existing `syncFullHistory: false` line in the WhatsApp session bootstrap; there is no `shouldSyncHistoryMessage` hook anywhere in `extensions/`, `src/`, `docs/`, or `test/`.
  - command: `rg -n "syncFullHistory|shouldSyncHistoryMessage" extensions/whatsapp docs src test -S`
- **Inbound handling still depends on Baileys `messages.upsert` delivery:** The WhatsApp inbox monitor records inbound activity only while iterating `upsert.messages` and attaches its listener to `messages.upsert`; if linked-device group messages never surface from Baileys, OpenClaw has no alternate path that would create the missing inbound log/session.
  - file: [extensions/whatsapp/src/inbound/monitor.ts:642](https://github.com/openclaw/openclaw/blob/cb2c36b04975e34ab7014a11575d6589aa96ae32/extensions/whatsapp/src/inbound/monitor.ts#L642)
- **Current main only hydrates group metadata on connect:** After connect, the monitor does a best-effort `groupFetchAllParticipating()` call, but that is metadata hydration only; it is not a replacement for missing inbound group events.
  - file: [extensions/whatsapp/src/inbound/monitor.ts:718](https://github.com/openclaw/openclaw/blob/cb2c36b04975e34ab7014a11575d6589aa96ae32/extensions/whatsapp/src/inbound/monitor.ts#L718)
- **No relevant fix between latest release and current main:** Diffing release `00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5` to current main `cb2c36b04975e34ab7014a11575d6589aa96ae32` for the WhatsApp session/inbound files shows QR-rendering and contact-context changes, not a linked-device/group-inbound fix.
  - command: `git diff 00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5..cb2c36b04975e34ab7014a11575d6589aa96ae32 -- extensions/whatsapp/src/session.ts extensions/whatsapp/src/inbound/monitor.ts extensions/whatsapp/package.json docs/channels/whatsapp.md`
