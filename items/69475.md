## Summary

Keep #69475 open. Current main still has the four reported bundled skill frontmatter descriptions without explicit WHEN/trigger language, and the skill prompt path uses frontmatter `description` as the model-visible routing signal before any `SKILL.md` body is read. The `gh-issues` skill also still reproduces the size finding at 5006 words / 885 lines with no `references/` split. The related PR #69489 was closed unmerged, and #69529 is still open/unmerged and only partial, so there is no landed canonical fix.

## Best Possible Solution

Keep the issue open and land a focused bundled-skills cleanup: update the four frontmatter descriptions with concise WHAT plus WHEN trigger language, fold in the reporter's follow-up about Contacts coverage and Obsidian-specific triggers, and either split `skills/gh-issues/SKILL.md` into `references/` files or explicitly document a different OpenClaw size policy. A small static audit for bundled skill descriptions and size would prevent drift.
