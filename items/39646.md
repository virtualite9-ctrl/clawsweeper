## Summary

Keep open. The requested status fix is not implemented on current `main`: `models status` still prefers the raw configured default model string over the resolved model when building JSON output, probe candidates, and rich terminal output, so a removed provider can still leak into status surfaces. Parts of this PR appear mixed with other changes that have since moved on or been superseded, but the core status bug described by the PR remains unresolved.
