## Summary

Keep PR #39646 open. Current main still has the stale raw default-model behavior the PR targets, and the PR remains directionally useful, but it should not merge as-is because the provided review context shows unrelated bundled changes and unresolved functional gaps.

## Best Possible Solution

Keep this open or replace it with a narrower maintainer patch. Centralize provider-validating default-model resolution for `models status --json`, rich `models status`, auth-probe candidates, and `openclaw status`; preserve valid CLI-backed and plugin/synthetic providers; add regression tests for a default that references a provider removed from `models.providers`; split unrelated Podman, rate-limit, channel-target, and UI autocomplete changes into separate PRs if still wanted.
