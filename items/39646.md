## Summary

Keep PR #39646 open. Current main still has the stale raw default-model behavior the PR targets, so the work is not obsolete. The PR remains directionally useful, but it should not merge as-is because the provided review context shows unrelated bundled changes and unresolved functional gaps.

## Best Possible Solution

Keep this open or replace it with a narrower maintainer patch. Centralize effective default-model resolution so models status JSON/rich output, auth-probe candidates, and openclaw status share provider-validating logic; fall back only when a raw provider was actually removed, while preserving CLI-backed and plugin/synthetic providers. Add regression tests for removed providers and split unrelated changes into separate PRs.
