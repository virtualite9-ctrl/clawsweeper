## Summary

Keep open. Current main still sends Feishu `.opus`/`.ogg` message-tool media through the native audio path, but `sendMediaFeishu` still uploads the file without calculating or passing `duration`, so the reported missing duration indicator remains a real bundled Feishu plugin bug rather than stale or implemented work.

## Best Possible Solution

Keep this issue open and fix it in the bundled Feishu plugin media path. Compute a finite audio duration for native `.opus`/`.ogg` sends and for `audioAsVoice` transcode outputs, convert it to Feishu's upload units, pass it to `uploadFileFeishu`, and add focused Feishu media regression tests that `im.file.create` receives `duration` for direct Opus/Ogg audio and voice-intent transcodes. The existing `openclaw/plugin-sdk/media-runtime` export already exposes `runFfprobe`, so this can stay plugin-owned without adding a new core API unless broader media-duration helpers are desired.
