## Summary

Current main still sends Feishu message-tool Opus/Ogg media through the bundled Feishu native audio path without computing or passing the upload `duration`, so the reported missing duration display remains actionable.

## Best Possible Solution

Keep this issue open and fix the bundled Feishu plugin media path. Compute a finite audio duration for native Opus/Ogg sends and successful `audioAsVoice` transcode outputs, pass it through to `uploadFileFeishu`, and add focused Feishu media regression tests proving `client.im.file.create` receives `duration` for native audio and voice-intent transcodes.
