## Summary

Current `main` still appears to miss Feishu audio-duration propagation for outbound voice messages. The Feishu uploader supports a `duration` field, but the actual `message`/outbound send path never computes or passes one for `.opus` audio, so the report remains plausible and unimplemented.

## Evidence

- **Feishu uploader can send duration, but only if provided:** `uploadFileFeishu` accepts `duration?: number` and forwards it to `client.im.file.create`, so the integration has an explicit hook for audio/video duration metadata.
  - file: [extensions/feishu/src/media.ts:367](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/extensions/feishu/src/media.ts#L367)
  - command: `nl -ba extensions/feishu/src/media.ts | sed -n '367,390p'`
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Actual outbound Feishu media send omits duration:** `sendMediaFeishu` routes `.opus` files to `msg_type: "audio"`, but when it calls `uploadFileFeishu` it passes `cfg`, `file`, `fileName`, `fileType`, and `accountId` only. No duration is derived from the media or forwarded.
  - file: [extensions/feishu/src/media.ts:625](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/extensions/feishu/src/media.ts#L625)

  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Message-tool Feishu action has no duration path:** The Feishu `send` / `thread-reply` action reads `text` and `media`, then calls `sendMedia` with `mediaUrl`, `accountId`, `mediaLocalRoots`, and reply metadata only. There is no duration parameter in the action flow.
  - file: [extensions/feishu/src/channel.ts:685](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/extensions/feishu/src/channel.ts#L685)
  - command: `nl -ba extensions/feishu/src/channel.ts | sed -n '685,737p'`

- **Shared outbound adapter does not expose media duration:** `ChannelOutboundContext` contains `text`, `mediaUrl`, and `audioAsVoice`, but no duration field, so plugin outbound adapters cannot currently receive audio duration metadata from the generic send pipeline.
  - file: [src/channels/plugins/outbound.types.ts:14](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/channels/plugins/outbound.types.ts#L14)
