## Summary

Current main still sends Feishu `.opus` / `.ogg` message-tool media as native Feishu audio, but the outbound upload path does not compute or pass the `duration` field that Feishu documents as required for audio/video uploads. The reported missing duration indicator therefore remains a real bundled Feishu plugin bug, not a stale or duplicate cleanup item.

## Best Possible Solution

Keep this issue open and fix it in the bundled Feishu plugin media path. Compute a finite audio duration in milliseconds for native `.opus` / `.ogg` sends and for successful `audioAsVoice` transcode outputs, pass that value to `uploadFileFeishu`, and add Feishu media tests proving `im.file.create` receives `duration` for both direct Opus/Ogg audio and voice-intent transcodes.
