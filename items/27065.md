## Summary

Keep #27065 open. Current main still renders the Control UI New session toolbar button as a direct click into `onNewSession`, and the parent still maps that callback directly to `handleSendChat("/new", { restoreDraft: true })`. I found no confirmation dialog, relocation, danger treatment, or undo/safety step on the current main path. The provided GitHub context also shows the direct fix PR #27110 was closed without merge, while a maintainer review comment on 2026-04-26 explicitly kept this issue open.

## Best Possible Solution

Keep #27065 open and implement a focused Control UI safety step before the toolbar New session action dispatches `/new`. The likely patch belongs in `ui/src/ui/chat/run-controls.ts` or the `onNewSession` wiring in `ui/src/ui/app-render.ts`, with regression coverage in `ui/src/ui/chat/run-controls.test.ts` for both confirm and cancel behavior. Broader `/new` and `/reset` policy can remain tracked separately; this specific toolbar misclick issue is still real on current main.
