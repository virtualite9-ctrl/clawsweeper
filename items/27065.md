## Summary

Keep open. Current main still renders the Control UI New session button as a direct click handler into `onNewSession`, and the parent still maps that to `handleSendChat("/new", { restoreDraft: true })` with no confirmation, relocation, danger state, or undo path. The existing test coverage asserts the direct callback behavior rather than a cancel/confirm flow. The latest release tag visible in the checkout has the same direct behavior, and the focused related PR #27110 was closed unmerged in the provided GitHub context.

## Best Possible Solution

Keep #27065 open and implement a focused Control UI safety step before `onNewSession` dispatches `/new`. The likely change belongs around `ui/src/ui/chat/run-controls.ts` or the parent `ui/src/ui/app-render.ts` wiring, with regression coverage in `ui/src/ui/chat/run-controls.test.ts` for confirm and cancel behavior. Broader `/new` and `/reset` slash-command policy can remain coordinated with the wider reset-safety issues, but this specific toolbar misclick bug is still real on current main.
