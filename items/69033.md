## Summary

Keep open. Current main still counts `toolResult.details` in the tool-result char estimator, so the PR's central false-overflow fix is not implemented on main. Main now has a broader persisted-details sanitizer, so this PR needs rebase/adaptation, but it is not obsolete.

## Best Possible Solution

Keep the PR open. Rebase it onto current main, preserve the existing generic persisted-details sanitizer, and focus the remaining change on making context estimation ignore non-model-visible `toolResult.details` with regression coverage for aggregate hidden-details false overflow.
