## Summary

Keep open. Current main has already added a stronger persistence cap for oversized `toolResult.details`, but the PR's core false-overflow fix is still not implemented: the preemptive context estimator still counts hidden `details` even though current docs and replay code treat `details` as non-model-visible metadata.

## Best Possible Solution

Keep the PR open and ask for an updated patch that preserves main's current persisted-details sanitizer while changing `tool-result-char-estimator.ts` so preemptive overflow estimation ignores non-model-visible `toolResult.details`, with the PR's false-overflow regression test retained or adapted.
