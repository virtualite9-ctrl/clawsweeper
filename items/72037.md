## Summary

Keep #72037 open. Current main has bumped the bundled `acpx` plugin to `acpx@0.6.1`, but it still does not bundle `@zed-industries/codex-acp` or a platform `codex-acp` binary, and the generated Codex ACP wrapper still defaults to `npm exec --package @zed-industries/codex-acp@^0.11.1` or `npx`. The related #60693 comment is useful corroboration and #71549 was a closed, unmerged acpx bump, but neither supersedes this focused packaging bug.

## Best Possible Solution

Keep the focused issue open. The likely fix belongs in the bundled ACPX/OpenClaw packaging path or upstream `openclaw/acpx`: stage a local `@zed-industries/codex-acp` plus platform binary and have the managed wrapper resolve that binary directly, or otherwise change the acpx Codex default so OpenClaw no longer starts Codex ACP through npm/npx. Add regression coverage that the package metadata/lockfile include the dependency and that the default generated command does not use `npm exec`/`npx`. Do not treat #60693 as canonical because it is a broad multi-patch report, and #71549 was closed unmerged and only described an acpx version bump.
