## Summary

Keep this PR open. Current main still registers `openclaw cron runs` as a `--id`-only CLI command, current docs show only `openclaw cron runs --id <job-id>`, and there is no current CLI regression coverage for the proposed positional alias. The PR remains a narrow implementation candidate for the linked CLI UX bug; its existing review feedback is minor and actionable.

## Best Possible Solution

Keep this PR open as a focused CLI fix candidate. Rebase it on current main, address the two minor review comments, add or update docs if the positional alias is accepted, run the targeted cron CLI test and the appropriate changed gate, then merge if maintainers want `openclaw cron runs <jobId>` supported as an alias for `--id`.
