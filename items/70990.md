## Summary

Keep open. Current main still lacks the requested `model_failover` and `model_failure_terminal` plugin hooks, and the existing model-call/lifecycle hooks do not expose the failover-decision or terminal all-models-failed event shape this PR adds. This is plausible core plugin API work, not ClawHub-only work, because external plugins need a missing in-process hook surface before they can observe these core failover seams. The linked #70976 was closed as superseded by this PR, while #65824 remains the broader fallback UX/meta issue.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is a narrow, documented core plugin hook API for failover decisions and terminal before-reply all-models-failed outcomes, with hook-runner tests plus failover emission tests. Broader fallback notices, workspace-context re-injection, and circuit-breaker behavior should stay separate under #65824 or narrower follow-ups.
