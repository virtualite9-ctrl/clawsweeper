## Summary

Keep open. Current main still returns multiple hook 4xx responses without a `logHooks.warn` journal line, including malformed JSON, direct wake/agent validation failures, mapped hook validation failures, auth failures, method/path rejects, and mapped policy rejects. The related PR #69267 attempted this exact fix but was closed unmerged, and its head commit is not present in the local main checkout.

## Best Possible Solution

Keep #69192 open. The best fix is a central hook rejection helper in `src/gateway/server-http.ts` that logs one warn line before every non-2xx hook response with stable branch-specific reason codes, path, status, content type/body shape/bytes where safe, and a request id. Also split the ambiguous mapped-message validation reasons in `src/gateway/hooks-mapping.ts` and add regression tests that inject/capture `logHooks.warn` for malformed JSON, direct validation, mapped validation, auth/method/path, policy, and body-limit/timeout rejects.
