## Summary

Keep open. Current main still has multiple hook non-2xx paths that return 400/401/404/405/408/413 without a hook warning/journal entry. The related PR #69267 directly targeted this issue, but it was closed unmerged under `r: too-many-prs`, and its review comments called out cleanup needed before reuse.

## Best Possible Solution

Keep #69192 open. The best maintainer path is a small gateway change that centralizes hook rejection responses in `createHooksRequestHandler`, logs one sanitized warn line before every hook non-2xx response with stable reason code/path/status/content metadata/request id, and adds regression tests for invalid JSON, mapping validation, agent validation, auth, 404, and 405 rejects. Reuse the intent of PR #69267 but fix the placeholder and log-sanitization concerns.
