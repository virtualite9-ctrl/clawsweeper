## Summary

Keep open. Current main still has compact formatting, count limits, prompt char budgets, and per-agent skill allowlists, but it does not implement task-matched lazy loading, a names-only skills catalog, or a configurable skills prompt visibility mode. The observable request remains missing.

## Best Possible Solution

Keep this issue open and scope it as a core skills prompt optimization. The likely fix belongs in src/agents/skills/workspace.ts plus config types/schema/help/docs/tests: add an explicit names-only and/or guarded auto prompt visibility mode, preserve current full/compact behavior as a compatible default, and cover full, compact, names-only, truncation, per-agent overrides, prompt reporting, and skill-selection guidance.

## Risks / Open Questions

- A naive automatic matcher could hide relevant skills and regress skill discovery; an auto mode needs a conservative fallback and focused tests.
- Changing the <available_skills> XML shape can affect prompt-report parsing and tests that currently expect <skill> blocks with locations.
- Manual per-agent allowlists reduce prompt size for configured agents, but they are policy controls and do not solve automatic lazy loading for users with many installed skills.
