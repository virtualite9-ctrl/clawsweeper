## Summary

Keep this issue open. Current main has compact formatting, count/character budgets, and per-agent skill allowlists, but it still injects every prompt-visible eligible skill into the skills prompt on ordinary runs. Full mode includes description and location for each skill; compact mode omits descriptions but still emits per-skill name plus SKILL.md location, so the requested names-only or automatic lazy-loading behavior is not implemented.

## Best Possible Solution

Keep this issue open and scope it as a core skills prompt optimization. The best implementation would add an explicit skills prompt visibility mode, likely `names-only` and/or guarded `auto`, in `src/agents/skills/workspace.ts`, then update `src/config/types.skills.ts`, `src/config/zod-schema.ts`, schema help/labels, docs, and focused tests for full, compact, names-only, truncation, and per-agent override behavior. Coordinate with broader prompt-size/cache work, but do not close this until the skills-specific lazy or names-only behavior exists on main.

## Risks / Open Questions

- A naive automatic matcher could hide relevant skills and regress discovery, so an `auto` mode needs a clear fallback and coverage.
- Changing the skills prompt XML shape can affect prompt reporting and tests that parse `<skill>` blocks.
- Manual per-agent allowlists reduce prompt size for configured agents, but they are policy controls and do not solve automatic lazy loading for users with many installed skills.
