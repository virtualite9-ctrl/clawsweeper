## Summary

Keep open. Current main still builds the Control UI session/default thinking label from model-profile heuristics in gateway session rows/defaults, while runtime thinking resolution has config-aware precedence. The PR remains a focused fix for a real mismatch and the discussion shows the author addressed bot review gaps with additional gateway/UI coverage.

## Best Possible Solution

Keep #63418 open and review/rebase it against current main. The best product fix is for sessions.list/buildGatewaySessionRow to expose an effective config-aware thinking default using the same precedence as runtime selection, including per-agent thinkingDefault, per-model params.thinking, global thinkingDefault, and catalog/provider fallback, while preserving the UI fallback for older gateways. Keep or add focused gateway handler/helper and Control UI tests for those cases.
