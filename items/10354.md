## Summary

Keep #10354 open. Current main still exposes the shared `message` tool `channel` parameter as a bare optional string with no description or enum, while the adjacent `target` field is described. The related PRs #10364 and #13584 were closed unmerged, and the runtime fallback only covers some current-channel contexts; it does not add the requested schema guidance.

## Best Possible Solution

Land a narrow core change in `src/agents/tools/message-tool.ts` adding a `channel` schema description that says it expects a messaging provider or channel plugin id such as `slack` or `discord`, not the conversation/channel target id. Add a focused `src/agents/tools/message-tool.test.ts` assertion that the description is exposed in `createMessageTool().parameters`. Defer an enum unless maintainers derive it from the same dynamic channel/plugin registry used by message routing.
