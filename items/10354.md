## Summary

Keep #10354 open. Current main still exposes the shared `message` tool `channel` parameter as a bare optional string with no description or enum, so the requested schema guidance has not landed. The existing prompt guidance and runtime fallback behavior help in some contexts but do not satisfy the tool-schema request.

## Best Possible Solution

Land a narrow core change in `src/agents/tools/message-tool.ts` adding a `channel` schema description that says it expects a messaging provider or channel plugin id such as `slack` or `discord`, not a conversation/channel target id. Add a focused assertion in `src/agents/tools/message-tool.test.ts` that the `channel` property description is present. Defer enum support unless maintainers derive it dynamically from configured or registered channel/plugin ids.
