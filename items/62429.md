## Summary

Current main already resolves the PR's central Anthropic tool-call partial JSON failure through the pinned @mariozechner/pi-ai@0.70.2 parseStreamingJson implementation. The current Anthropic transport still routes both relevant tool-call partial JSON paths through that parser, and the dependency parser repairs raw C0 control characters before falling back to partial parsing/{} instead of surfacing a raw SyntaxError.

## Best Possible Solution

Close the PR as already implemented on main. Keep the current dependency-backed parser behavior; if Anthropic or Azure Foundry still produces a different streaming failure, track that as a narrower new bug with the exact malformed event or partial_json payload that @mariozechner/pi-ai@0.70.2 does not handle.
