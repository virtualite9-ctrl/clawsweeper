## Summary

Issue #47461 should stay open. Current main does not implement Telegram bot profile-photo sync from OpenClaw avatar state, and the existing Control UI assistant avatar upload is explicitly browser-local rather than a server-side config change that a channel plugin could observe. The proposed `setChatPhoto` method is not the right Telegram API for a bot's own avatar, but Telegram now exposes `setMyProfilePhoto`, so this is still a plausible Telegram plugin enhancement rather than a closeable stale/duplicate/core-scope item.

## Best Possible Solution

Keep this open as a Telegram plugin enhancement. The likely path is to add an explicit, opt-in server-side avatar sync flow for persisted OpenClaw avatar sources, then have the Telegram plugin call `setMyProfilePhoto` with caching and tests/docs. The browser-local Control UI override should not silently mutate a Telegram bot avatar unless maintainers intentionally change that persistence contract.

## Risks / Open Questions

- A correct implementation needs a product decision on whether Control UI avatar uploads should remain browser-local or gain an explicit persisted/shared mode before pushing anything to Telegram.
- Telegram profile-photo sync should use `setMyProfilePhoto` with upload semantics, rate-limit/hash caching, and clear error handling; blindly watching `avatars/avatar.jpg` would not match all supported OpenClaw avatar sources.
