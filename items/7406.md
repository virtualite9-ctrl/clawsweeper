## Summary

Keep #7406 open. Current main still only partially addresses Telegram topic names: it learns and persists the topic name for inbound context/prompt metadata, but the value does not flow into session `label`/`displayName`, `sessions.list`, or the Control UI session dropdown. The related implementation PRs in the provided context are closed and unmerged, and duplicate reports were closed back toward this issue as the canonical tracker.

## Best Possible Solution

Keep #7406 open as the canonical implementation tracker. The best fix is to map cached Telegram forum `TopicName` into persisted session display metadata, likely by setting `ThreadLabel` or adding a dedicated `topicName`/display field during inbound session metadata recording, then ensure `sessions.list` and the Control UI render a stable human-readable group/topic label with id/key fallback. Add regression coverage for Telegram context extraction, session metadata persistence, gateway session rows, and Control UI label resolution.
