## Summary

Keep #7406 open. Current main still only captures Telegram forum topic names for inbound context and prompt metadata; it does not persist or expose those names through session metadata, gateway session rows, or the Control UI session dropdown. The related implementation PRs are closed unmerged, and duplicate issues were closed back to this canonical tracker.

## Best Possible Solution

Keep #7406 open as the canonical tracker. The best fix belongs in the bundled Telegram/session metadata path: map cached Telegram TopicName into persisted session display metadata, preserve user label/displayName overrides, keep key/topic-id fallback behavior, and add regression coverage for topic extraction, metadata persistence, gateway sessions.list rows, and Control UI label resolution.
