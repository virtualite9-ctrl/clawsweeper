## Summary

Keep this PR open. Current main still does not redirect gateway root `/` to the configured Control UI `basePath`; the PR remains a narrow implementation candidate with matching regression tests and no supply-chain-sensitive file changes.

## Best Possible Solution

Keep this PR open. A maintainer should review/rebase and land or reapply the same narrow gateway change: in `classifyControlUiRequest`, when `basePath` is non-empty and `pathname === "/"`, redirect read methods to `${basePath}/${search}`, keep non-read root requests as `not-control-ui`, and retain regression tests for root redirect, query-string preservation, and write-method fallthrough. Consider a small basePath normalization hardening follow-up if protocol-relative configured base paths are not intended.
