## Summary

Keep PR #40522 open. Current main still lacks the requested gateway-root redirect when `gateway.controlUi.basePath` is configured: `/` is classified as `not-control-ui`, the Control UI handler declines it, and the HTTP server falls through to plain `404 Not Found`. The PR remains a narrow fix for the linked #34298 behavior, and the provided PR discussion shows the earlier query-string test gap was addressed in the PR branch.

## Best Possible Solution

Keep the PR open. A maintainer should merge it or reapply the same narrow change on main: in `classifyControlUiRequest`, redirect read-method requests for `/` to `${basePath}/` while preserving the query string, keep non-read root requests falling through for plugin/webhook safety, and retain regression tests for root redirect, query-string preservation, and POST fallthrough.
