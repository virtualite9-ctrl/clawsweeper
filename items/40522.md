## Summary

Keep this PR open. Current main still lacks the requested gateway-root redirect when `gateway.controlUi.basePath` is configured: `/` is classified as outside the Control UI, the Control UI handler declines it, and the HTTP server falls through to `404 Not Found`. The PR remains a narrow implementation candidate for linked #34298, and the provided PR discussion shows the earlier query-string coverage gap was addressed on the PR branch.

## Best Possible Solution

Keep this PR open. A maintainer should merge it or reapply the same narrow change on current main: in `classifyControlUiRequest`, when `basePath` is non-empty and `pathname === "/"`, redirect read methods to `${basePath}/${search}`, keep non-read root requests as `not-control-ui`, and retain regression tests for root redirect, query-string preservation, and POST fallthrough.
