## Summary

Keep this PR open. Current main still lacks the bundled-LSP process-tree cleanup and Gateway shutdown sweep that the PR is trying to add, and the linked bug #72357 has a concrete reproduction path. The latest provided PR context is now focused on the right four files and includes regression tests, so the best maintainer path is review and land or request final adjustments, not close.

## Best Possible Solution

Keep this PR open and review it as the implementation candidate for #72357. The desired fix belongs in `src/agents/pi-bundle-lsp-runtime.ts` and `src/gateway/server-close.ts`: spawn bundled LSP servers in disposable process groups where appropriate, use `killProcessTree` for forced cleanup, track active bundled LSP sessions for Gateway shutdown, dispose MCP and LSP shutdown sweeps without doubling worst-case latency, and keep the new regression tests in the LSP runtime and Gateway close suites.
