## Summary

Keep PR #60445 open. Current main still builds exec-approval-unavailable replies as plain text with no `channelData.execApprovalUnavailable` marker, and the current tool-result delivery gate only preserves `channelData.execApproval` payloads when tool summaries are suppressed. The PR is a focused, still-relevant bug fix rather than implemented, obsolete, or superseded; related #57914 is a broader open feature, not a shipped canonical replacement.

## Best Possible Solution

Keep the PR open and either merge a refreshed narrow fix or fold the same behavior into the active delivery-gate work: add a typed `channelData.execApprovalUnavailable` marker with `reason` on every unavailable-payload return path, teach all relevant delivery suppression gates to preserve it, and add focused tests for builder output plus group/native and progress-suppressed delivery behavior.
