## Summary

Keep PR #69197 open. Current main does not implement an equivalent fix: background exec completion still calls `maybeNotifyOnExit()` from the `managedRun.wait()` microtask path with no active-poll guard, while `process poll` waits in a 250 ms timeout loop and only observes/marks completion afterward. The PR addresses a real focused race with source-aligned tests, but its current boolean approach still needs maintainer review or a counter/token fix for overlapping polls.

## Best Possible Solution

Keep the PR open and ask for a small amendment: track active polls with a counter/token on `ProcessSession`, have `maybeNotifyOnExit()` suppress only while the count is greater than zero, set `exitNotified` when poll observes and returns the exit, and add regression coverage for normal active poll, overlapping polls, and kill/exit interactions. Do not close this as implemented until current main has equivalent source and test coverage.
