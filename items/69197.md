## Summary

Keep open. Current main does not implement this PR's central poll-vs-notify suppression: `maybeNotifyOnExit()` still only guards on `backgrounded`, `notifyOnExit`, and `exitNotified`, while `process poll` waits without setting any active-poll state or marking `exitNotified` when it observes completion. The PR is a focused bug-fix attempt with regression tests and useful live-validation comments, but it still has an unresolved review concern about the boolean `pollActive` approach under overlapping polls, so it should be refined rather than auto-closed.

## Best Possible Solution

Keep the PR open and ask for a robust active-poll implementation, preferably an in-flight poll counter/token on `ProcessSession`, plus regression tests for the normal poll/exit race and the overlapping-poll case. After that, validate with the agents changed lane before merge. This is not implemented on current main and is not a ClawHub or external-administration cleanup item.
