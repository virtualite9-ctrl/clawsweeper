## Summary

Keep open. Current main does not implement this PR's central poll-vs-notify suppression: `maybeNotifyOnExit()` still has no active-poll guard, `ProcessSession` has no active poll state, and `process poll` waits without setting a marker or marking `exitNotified` when it consumes completion. The PR is a plausible focused fix with useful regression tests/live validation, but its boolean `pollActive` approach has an unresolved overlapping-poll review concern, so it should be refined rather than auto-closed.

## Best Possible Solution

Keep #69197 open and ask for a robust active-poll implementation, preferably an in-flight poll counter/token on `ProcessSession`, plus regression tests for both the normal poll/exit race and the overlapping-poll case. After refinement, validate with the agents changed lane before merge.
