## Summary

Current main still matches the reported bug: completion-mode announce dispatch records the failed direct-primary phase, returns a successful queue fallback, and the caller only logs when the top-level delivery result failed on the direct path.

## Best Possible Solution

Keep this issue open and add a narrow recovered-delivery diagnostic. The implementation should warn when a direct-primary phase has an error and a later queue-fallback phase delivers, without marking the announce failed. The likely homes are runSubagentAnnounceFlow after delivery or a small dispatch callback/logger seam, with a focused test covering delivered=true/path queued or steered plus a direct-primary phase error.

## Evidence

That preserves the direct error only in phases while the top-level result is delivered=true with path queued/steered.
