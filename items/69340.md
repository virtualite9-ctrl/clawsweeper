## Summary

Keep this PR open. Current main still has the underscore-only copy-sensitive guard, so a long hyphenated package-style token is still split by `sanitizeRenderableText`. The PR is a narrowly scoped one-line formatter fix with no supply-chain-sensitive files touched, but it needs a regression test and its `Fixes #48432` claim should be checked because the linked issue also discusses broader code-block/table wrapping corruption.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to add a focused regression test in `src/tui/tui-formatters.test.ts` for a long hyphen-separated token, verify the existing path/URL/underscore/credential behavior remains unchanged, and either narrow the PR's issue-closing claim or broaden the fix if #48432 still needs code-block/table wrapping changes beyond this token guard.
