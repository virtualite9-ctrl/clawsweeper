## Summary

Keep PR #69340 open. Current main still has the underscore-only copy-sensitive guard in the TUI formatter, so long hyphen-separated identifiers still fall through to arbitrary 32-character splitting. The PR is a narrow fix for the still-open linked bug #48432 rather than obsolete cleanup.

## Best Possible Solution

Keep the PR open and revise it into a small, test-backed formatter fix: update `src/tui/tui-formatters.ts` to preserve file-like hyphen-separated long tokens and add a focused `sanitizeRenderableText` regression test for a token like `ubuntu-budgie-themes-long-package-name`. Consider the analogous `src/terminal/note.ts` behavior separately if maintainers want consistent terminal note wrapping.
