## Summary

Keep #69340 open. Current main still has the underscore-only copy-sensitive file-like guard, so the PR's requested hyphenated-token case is not implemented or shipped. The related open #39565 work is useful context but does not supersede this PR because it deliberately keeps natural-language hyphenated long tokens chunkable and only preserves digit-bearing hyphenated IDs.

## Best Possible Solution

Keep #69340 open for maintainer review. The best path is to add focused regression coverage in `src/tui/tui-formatters.test.ts` and either land a narrowly justified hyphenated package/identifier heuristic here or fold that exact case into a broader formatter fix alongside #39565/#48432, without closing this PR until current main preserves the requested copy-sensitive hyphenated tokens or a canonical follow-up explicitly covers them.
