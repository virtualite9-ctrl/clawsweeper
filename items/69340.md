## Summary

Keep PR #69340 open. Current main still has the underscore-only copy-sensitive token guard in the TUI formatter, so long hyphen-separated package/component names can still be chunked with inserted spaces. The PR is a focused fix for linked bug #48432, and the existing Greptile review supports the root-cause analysis, but the PR should add regression coverage before landing.

## Best Possible Solution

Keep PR #69340 open. The best path is to add a focused `sanitizeRenderableText` regression test for a long hyphen-separated token, keep the existing path/URL/credential protections intact, and run the targeted TUI formatter tests. Coordinate with related open TUI copy-safety work such as #39565/#39505 only if maintainers want one broader formatter policy change.

- The PR as submitted has no regression test for a long hyphen-separated token, so the exact reported copy/paste corruption could regress later.

- Related open TUI copy-safety work such as #39565/#39505 overlaps the formatter policy area, so maintainers may prefer one broader, tested formatter fix instead of several narrow PRs.
