## Summary

Keep open. Current `main` still hard-codes the web UI text selection highlight to a very faint `--accent-subtle` tint in both dark and light themes, and the same rule is present in the latest release. The linked follow-up PR exists, but there is no evidence in `main` that the contrast fix has landed.

## Evidence

- **Global selection styling still uses a faint accent tint:** `ui/src/styles/base.css` still defines `::selection { background: var(--accent-subtle); color: var(--text-strong); }`, so the app overrides the browser default highlight with the theme's subtle accent color.
