## Summary

Keep open. Current `main` still suppresses ordinary `exec`/`bash` tool failures in normal runs, so the reported behavior is not implemented away in core. The repo now tracks exec failures as mutating tool errors, but the payload policy still returns `showWarning: false` for non-verbose, non-timeout exec failures, and tests explicitly lock that behavior in.
