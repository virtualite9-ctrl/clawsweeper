## Summary

Keep open. Current main still intentionally ignores explicit HTTP(S) image URLs in native prompt image detection, so the mixed inline-image plus remote-image behavior this PR targets is not implemented. The PR is paired with the same-author open bug report #62514 and has unresolved review feedback about over-fetching arbitrary URLs and SSRF-policy threading, so the best maintainer outcome is rework/review rather than cleanup closure.

## Best Possible Solution

Keep the PR and #62514 open for maintainer review or contributor rework. A safe implementation should detect only clearly intended remote image refs, avoid fetching arbitrary hyperlinks, thread the existing `ssrfPolicy` through `detectAndLoadPromptImages` and `loadImageFromRef`, handle URL dedupe and trailing punctuation correctly, and add regression coverage for mixed inline plus remote image prompts, non-image URL no-fetch behavior, and SSRF-policy propagation.
