## Summary

Current main still does not implement PR #63710's central behavior. Native prompt image detection explicitly ignores remote HTTP(S) URLs, the current regression tests assert that HTTPS image URLs are ignored, and the embedded runner only forwards images returned by that detector plus existing inline images. The PR is a plausible fix for linked issue #62514, but the provided review discussion identifies real changed-path concerns around arbitrary URL fetching and missing SSRF-policy forwarding, so it should stay open for maintainer rework rather than be cleanup-closed.

## Best Possible Solution

Keep the PR open for maintainer rework or replacement. The best fix is to add native prompt handling for explicit remote image URLs in `src/agents/pi-embedded-runner/run/images.ts`, constrain detection/fetching to intentional image refs, thread the existing remote-media SSRF policy through the preload path, and add focused tests for mixed inline + HTTP(S) image prompts, non-image URL avoidance, punctuation handling, dedupe, and SSRF policy behavior.
