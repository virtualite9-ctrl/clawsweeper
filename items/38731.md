## Summary

Keep open. Current `main` still does not warn on sub-30m cron schedules in either `cron add` or `cron.update`, so the requested behavior is not implemented. The PR discussion also identified substantive correctness/completeness issues that still matter against current `main`: the gateway logger contract is `warn(message, meta?)`, not `warn(meta, message)`, and the CLI has an existing `cron edit` schedule-update path that would bypass an add-only guardrail.

## Evidence
