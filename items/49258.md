## Summary

Keep open. Current `main` still treats cron execution status and delivery status as separate signals, and the isolated-run path can preserve `delivered: true` while still returning `status: "error"` when the run produced a fatal error payload. The timer then increments `consecutiveErrors` strictly from `result.status`. That means the reported state is still plausible on `main`, so there is not strong evidence to close as fixed or non-reproducible.

## Evidence
