## Summary

Keep #10960 open. Current main still does not implement the requested mid-stream soft-steer behavior for inbound channel queue mode: steer injection is streaming-gated, non-streaming active runs fall back to follow-up queuing, and docs still describe boundary/fallback semantics rather than clean pause/abort/resume with injected context. The provided GitHub context also shows maintainer-facing prior review kept this open and related implementation PR #64758 is still open/unmerged.

## Best Possible Solution

Keep #10960 open as the canonical runtime/product tracker. A complete fix should update the inbound auto-reply queue and embedded/reply-run control path so steer-mode input during active non-streaming generation or tool waits can cleanly pause or abort active work, preserve usable partial context, append the steering message, and resume in the same session. Regression coverage should distinguish streaming steer success, non-streaming/tool-wait fallback, steer-backlog/followup behavior, explicit gateway sessions.steer, and backend failure handling.
