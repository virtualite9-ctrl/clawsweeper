## Summary

Keep #10960 open. Current main still gates inbound steer injection on streaming state, rejects active-but-not-streaming embedded/reply backends, and documents fallback/boundary behavior rather than true pause/abort, inject, and resume semantics.

## Best Possible Solution

Keep #10960 open as the runtime/product tracker. The fix should replace the current streaming-only gates with a precise active agent-loop lifecycle check for embedded and reply-run backends, preserve safe fallback when same-turn steering cannot be accepted, and add regression coverage for streaming steer success, active-but-not-streaming/tool-wait behavior, follow-up fallback, explicit `sessions.steer`, and backend failure handling.
