## Summary

Keep #10960 open. Current main still implements steer as a streaming-gated queue/followup path, not the requested pause-or-abort, inject, and resume soft-steer lifecycle. The issue discussion and related PRs point to plausible fixes, but the current checkout has not landed them.

## Best Possible Solution

Keep #10960 open as the product/runtime tracker. The best fix should define soft-steer lifecycle semantics across streaming generation, tool execution, backend failure, and fallback; replace streaming-only acceptance gates with a safe active agent-loop predicate; keep `sessions.steer` hard-interrupt semantics distinct; update queue/agent docs; and add regression coverage for streaming steer, active non-streaming/tool-wait steering, followup fallback, and gateway/backend failure paths. Re-review after #64758, #65010, or a successor lands.
