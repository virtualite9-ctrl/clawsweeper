## Summary

Keep this PR open. Current main does not implement inline Adaptive Card rendering in the shared Swift chat UI used by iOS and macOS, and the existing plugin/ClawHub surfaces do not provide a way for an external plugin to patch native Swift chat rendering. The PR is also paired with an open Android PR by the same author and still has substantive review/security concerns to resolve before it could be merged or closed as obsolete.

## Best Possible Solution

Keep the PR open for maintainer product review. If native iOS/macOS Adaptive Card rendering is desired, update the PR against current main, align it with the existing `ReplyPayload.presentation` and client rendering direction instead of treating plugin promotion as a docs listing, add Swift parser/view tests or focused smoke coverage, resolve the outstanding review findings, and explicitly review URL/image loading behavior. Coordinate the decision with the open Android companion PR #42304.
