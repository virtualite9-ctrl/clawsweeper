## Summary

Keep this PR open. Current main still lacks the proposed inline Adaptive Card parser/SwiftUI renderer in the shared Swift chat UI used by iOS and macOS, and the PR is paired with the same author's still-open Android companion PR #42304. The branch also needs maintainer product/security review before it could merge, especially around agent-controlled image and URL rendering.

## Best Possible Solution

Keep the PR open for maintainer product review. If native iOS/macOS Adaptive Card rendering is wanted, update the branch against current main, resolve the parser/layout/caching/image fallback review comments, add focused Swift parser/view coverage or an app smoke path, explicitly review URL/image loading behavior, and coordinate the platform decision with open Android companion PR #42304. If maintainers decide native client renderers should not live in core, close the platform PR set together with a product-scope comment while leaving the external `openclaw-adaptive-cards` plugin as the marker-producing integration.
