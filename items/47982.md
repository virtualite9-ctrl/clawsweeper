## Summary

Close as `clawhub`. Current `main` already provides a supported plugin path for per-turn context injection across runtimes, including Codex, so this request is better handled as a community/ClawHub plugin than by adding a new core internal-hook API.

## Risks / Open Questions

- This closes a core-API proposal, not the underlying user need; authors wanting simple HOOK.md/script ergonomics would still need to package the behavior as a plugin.
- A ClawHub/plugin route may be less convenient than internal hooks for local one-off automation, but it matches the documented extensibility direction on current main.

## Close Comment

Closing after a Codex review. Current `main` already has a supported extension path for this use case through plugins: `before_prompt_build` is the documented per-turn context injection hook (`docs/concepts/agent-loop.md:94`, `src/plugins/hook-before-agent-start.types.ts:21`), hook-only plugins are still supported (`docs/plugins/architecture.md:54`), and the Codex harness explicitly preserves that compatibility layer (`docs/plugins/codex-harness.md:18`). The test at `extensions/codex/src/app-server/run-attempt.test.ts:273` also shows `prependContext` reaching Codex turn input. Given `VISION.md:54` says optional capability should usually ship as plugins and keep core lean, this new `message:enrich` internal-hook surface looks better suited as a ClawHub/community plugin than a new core API.
