## Summary

Current main has not implemented the Dreaming restart confirmation UX yet: the Dreaming toggle still calls the update path directly and that path sends `config.patch`. The remaining work is already tracked by the open, focused PR #63807, which explicitly closes this issue, adds the confirmation dialog/state/copy, and includes a browser test proving no patch is sent until Confirm Restart.

## Best Possible Solution

Close #63804 as superseded by the active implementation PR #63807, and keep review/landing discussion on that PR. The best product/code path is to land #63807 or an equivalent Control UI change that gates the Dreaming toggle behind an explicit restart confirmation and preserves the regression test.
