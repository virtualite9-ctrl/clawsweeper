## Summary

Current main already blocks this regression class in the packaged build/release path: a bundled-channel smoke script walks `dist/extensions/*`, disables source fallback, asserts packaged entry files exist, imports them, and loads the declared plugin/secrets surfaces. That smoke is wired into `prepack` and `release:check`, and the npm release workflow runs `pnpm release:check`. The cited Mattermost case is also no longer present on main because its bundled entry now points at packaged sidecars instead of `./src/*` paths.

## Evidence

- **Packaged bundled-channel smoke validates emitted entry files:** `scripts/test-built-bundled-channel-entry-smoke.mjs` sets `OPENCLAW_DISABLE_BUNDLED_ENTRY_SOURCE_FALLBACK=1`, enumerates channel/setup entries from `dist/extensions/*`, asserts each emitted entry file exists, imports the built module, and calls the bundled plugin/secrets loaders. A source-only specifier such as `./src/channel.js` would fail this smoke against packaged output.
  - file: [scripts/test-built-bundled-channel-entry-smoke.mjs:12](https://github.com/openclaw/openclaw/blob/8b76392e3e7934b1cdd6d461148263fd56141ec8/scripts/test-built-bundled-channel-entry-smoke.mjs#L12)

  - sha: [8b76392e3e79](https://github.com/openclaw/openclaw/commit/8b76392e3e7934b1cdd6d461148263fd56141ec8)
- **Source fallback is explicitly disabled for dist smoke:** The channel entry contract only falls back from `dist/extensions/*` back to `extensions/*` when `OPENCLAW_DISABLE_BUNDLED_ENTRY_SOURCE_FALLBACK` is not set. The smoke script sets that flag, so missing packaged specifiers are not masked by source-tree fallback.
  - file: [src/plugin-sdk/channel-entry-contract.ts:131](https://github.com/openclaw/openclaw/blob/8b76392e3e7934b1cdd6d461148263fd56141ec8/src/plugin-sdk/channel-entry-contract.ts#L131)

- **Smoke is part of prepack:** `scripts/openclaw-prepack.ts` runs the bundled-channel smoke after `pnpm build`, `pnpm ui:build`, and dist inventory generation, so pack/publish preparation fails before producing a tarball when a bundled entry path is wrong.

The same smoke script is also present in released tag `v2026.4.23`.

## Risks / Open Questions

- The validation is enforced in prepack/release-packaging flows rather than plain `pnpm build`, so a bad specifier can still exist in a dev build until pack/release gates run.
