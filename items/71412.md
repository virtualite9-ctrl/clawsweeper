## Summary

Keep #71412 open. Current `main` still matches the reported failure mode: the stop-timeout branch leaves the task/abort entries uncleared, records the account as still running, and the start path still silently returns when a task entry exists. The related fix PR #71456 is closed and unmerged in the provided GitHub context, and no equivalent fix is present in the checked-out history.

## Best Possible Solution

Keep #71412 open and land PR #71456 or an equivalent patch that clears `store.aborts` and `store.tasks` on stop timeout, records the runtime as stopped rather than running, and updates the regression test so a subsequent `startChannel` registers a fresh task. Treat the `openclaw doctor` Telegram liveness idea as a separate follow-up unless maintainers choose to broaden the fix.
