## Summary

Keep #71412 open. Current main still matches the reported zombie-task failure: the stop-timeout branch marks the account running and returns before clearing `store.aborts`/`store.tasks`, while `startChannelInternal` silently returns when `store.tasks` still has that account. The regression test still asserts the buggy behavior, and the related fix PR #71456 is closed and unmerged in the provided context.

## Best Possible Solution

Keep #71412 open and review or land PR #71456, or an equivalent patch, that clears the stale `store.aborts` and `store.tasks` entries on stop timeout, marks the account stopped instead of running, and updates the regression test so a subsequent start registers a fresh task. Treat the proposed `openclaw doctor` liveness improvement as a separate follow-up unless maintainers want it bundled.
