## Summary

Keep PR #70598 open. Current main already implements the core user-relay prompt improvement for exec completion details, but it does not contain the PR's later queue-preservation fix for mixed exec/non-exec heartbeat queues. The discussion correctly identified this remaining user-visible behavior: main filters exec events for the prompt, then still consumes the entire inspected pending-event prefix, so a cron/reminder event queued with an exec completion can be removed without being rendered or relayed.

## Best Possible Solution

Keep the PR open. The best path is to rebase and validate the branch, or extract the remaining `restoreSystemEventEntries`/deferred-event preservation behavior and its regression tests onto current main. Do not close as implemented until mixed exec plus cron queues preserve non-exec events, including duplicate deferred reminders, on main.
