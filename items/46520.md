## Summary

Keep open. Current main still routes custom-scheme origins through `new URL(...).origin`, and Node returns `origin="null"` for `tauri://localhost` and `electron://app`, so a specific `gateway.controlUi.allowedOrigins` entry like `tauri://localhost` still cannot match. The related fix PRs #46523 and #46555 directly addressed this issue but were closed unmerged, and current tests still lack the proposed custom-scheme coverage.

## Best Possible Solution

Keep #46520 open and land a focused Gateway fix in `src/gateway/origin-check.ts`: when `new URL(origin).origin === "null"` for a syntactically valid custom scheme with a host, compare a normalized reconstructed origin such as `${url.protocol}//${url.host}` against `gateway.controlUi.allowedOrigins`. Add regression tests in `src/gateway/origin-check.test.ts` for allowed and rejected `tauri://` and `electron://` origins, and consider tightening the error detail if parsing succeeds but the opaque origin cannot match.
