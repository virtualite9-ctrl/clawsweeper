## Summary

Keep open. Current `main` still persists canonical skill file paths into `sessions.json`, reuses that cached snapshot on later runs instead of re-resolving workspace skills, and only invalidates the cache for skill-version/filter changes rather than workspace path changes. I found no migration/repair path that rewrites stale `skillsSnapshot` paths after a workspace move.
