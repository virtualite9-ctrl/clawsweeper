## Summary

Keep #39719 open. Current main still reuses persisted session skill snapshots across workspace changes: the persisted/runtime snapshot shapes have no workspace provenance, `agent-command` refreshes only on missing snapshot, version, or skill-filter changes, a fresh workspace still has version `0`, and embedded/CLI runtimes trust cached snapshot paths. The related sandbox skill-path work does not cover ordinary iCloud-to-local workspace migration in `sessions.json`.

## Best Possible Solution

Keep the issue open and implement workspace-aware skill snapshot invalidation. Add normalized workspace/source provenance to `SkillSnapshot` and `SessionSkillSnapshot`, compare it with the current resolved workspace before reusing a cached snapshot, invalidate snapshots on `agents.defaults.workspace` and per-agent workspace changes, and add a regression test with an old absolute `SKILL.md` path in `sessions.json` plus current config pointing at the new workspace.
