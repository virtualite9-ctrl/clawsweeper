## Summary

Keep open. Current `main` already has background exec completion wake via `notifyOnExit`, but it does not implement the proposed per-call `onComplete="notify"` contract, and the PR discussion surfaced unresolved behavior risks. That is not strong enough evidence for an auto-close.

## Risks / Open Questions

- The underlying user pain is partly addressed on `main`, so the PR may be redundant in practice, but the requested per-call API is still absent.
- `notifyOnExitEmptySuccess` remains opt-in on `main`, so quiet successful background jobs still differ from the PR's proposed `onComplete="notify"` behavior.
- GitHub CLI inspection was unavailable in this environment because `GH_TOKEN` was not set; this review relies on the provided PR context plus local `main` code/docs/tests.
