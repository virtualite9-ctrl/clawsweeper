## Summary

Close as implemented on main. The reported v2026.4.1 Feishu regression matches a packaged bundled-channel loading problem that the repo explicitly records as fixed in 2026.4.8, and current main now uses Feishu top-level sidecar entrypoints with regression guards for packed artifacts.

## Evidence

- **Shipped fix in changelog:** `2026.4.8` explicitly says bundled channel setup was fixed for Feishu and other channels by loading packaged top-level sidecars instead of `dist/extensions/*/src/*`, and that packaged plugin compatibility metadata was aligned so bundled channels/providers load on OpenClaw 2026.4.8.
  - file: [CHANGELOG.md:1091](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/CHANGELOG.md#L1091)
- **Current Feishu runtime entrypoint uses packaged sidecars:** Current main routes Feishu through `channel-plugin-api.js`, `secret-contract-api.js`, and `runtime-api.js` top-level sidecars rather than private `src/*` paths, which is the hardened packaged-install shape the changelog describes.
  - file: [extensions/feishu/index.ts:56](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/extensions/feishu/index.ts#L56)
  - sha: [ec3dbd22a4de](https://github.com/openclaw/openclaw/commit/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46)
- **Current Feishu setup entry uses packaged sidecars:** The setup entry likewise points at `setup-api.js` and `secret-contract-api.js`, not `./src/*`, matching the post-4.1 packaging fix path.
  - file: [extensions/feishu/setup-entry.ts:3](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/extensions/feishu/setup-entry.ts#L3)
  - sha: [ec3dbd22a4de](https://github.com/openclaw/openclaw/commit/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46)
- **Regression guard for packed bundled channels:** The repo now has a test that rejects bundled channel entries referencing `./src/*` and requires packed `secret-contract-api.js` sidecars for secret-backed channels, directly guarding the packaged-dist mismatch class discussed in the issue/comments.
  - file: [test/scripts/bundled-plugin-build-entries.test.ts:90](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/test/scripts/bundled-plugin-build-entries.test.ts#L90)
  - sha: [ec3dbd22a4de](https://github.com/openclaw/openclaw/commit/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46)
- **Reported symbol is not part of Feishu runtime:** Current production references to `resolveSenderCommandAuthorization*` are in plugin-sdk plus Zalo/ZaloUser codepaths; Feishu has no production reference, which matches Joel's triage comment that the title symbol did not belong to Feishu and points toward a bad packaged import/resolution rather than a live Feishu codepath bug on main.
  - command: `rg -n "resolveSenderCommandAuthorization" extensions src test`
  - sha: [ec3dbd22a4de](https://github.com/openclaw/openclaw/commit/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46)
