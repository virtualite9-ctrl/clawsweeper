## Summary

Keep open. Current main still has the strict QMD status parser that only recognizes `Vectors:` lines, so the reported `Could not determine QMD vector status from qmd status` path can still occur for healthy QMD output that uses another separator/format. The linked PR #63678 targets the exact bug, but it is still open and unmerged, so the issue should remain open until that fix or an equivalent scoped patch lands on main.

## Best Possible Solution

Keep #63652 open until a QMD status parser fix lands on main. The narrow implementation should live in `extensions/memory-core/src/memory/qmd-manager.ts`, accept realistic `qmd status` vector-count variants such as `Vectors = 42`, compact whitespace/tab forms, and avoid false positives from unrelated fields. Add regression coverage in `extensions/memory-core/src/memory/qmd-manager.test.ts`. PR #63678 is the closest existing candidate and should be reviewed/landed or replaced by an equivalent scoped patch.
