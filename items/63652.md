## Summary

Keep this issue open. The checked-out main still has the strict QMD status parser that only accepts `Vectors:` lines, the null parse path still produces the reported `Could not determine QMD vector status from `qmd status`` error, and the deep memory status command still surfaces that as unavailable Embeddings/Vector output. The linked implementation candidate #63678 remains the right review path, but it is not landed in the reviewed checkout.

## Best Possible Solution

Keep this issue open while maintainers review and either land #63678 or apply an equivalent scoped memory-core fix on main: broaden `parseQmdStatusVectorCount` to accept supported `qmd status` vector-count variants, keep the parser line-anchored enough to avoid false positives, and add regression tests for the observed formats without unrelated dependency or lockfile churn.
