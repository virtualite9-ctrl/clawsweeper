## Summary

Current main has Feishu inbound broadcast fan-out for configured agents, but it does not implement the requested send-time cross-bot broadcast/synthetic message injection. The issue should stay open because the central Feishu/Telegram platform gap remains distinct from existing inbound broadcast groups and existing `allowBots` support on channels that natively deliver bot-authored messages.

## Best Possible Solution

Keep this issue open and design the fix around the existing channel/broadcast architecture: likely add a Feishu-owned send-time synthetic inbound path keyed by configured broadcast peers/accounts, with loop protection, tests around outbound replies and multi-account dedupe, docs for Feishu, and a generic SDK/core seam only if Telegram or another channel needs the same behavior beyond Feishu.

## Evidence

- **Requested behavior:** The issue asks for Feishu bot-sent group messages to be synthesized to other registered bots in the same group, with opt-in config and loop protection. The supplied timeline also links related Feishu bot-message invisibility and multi-bot mention reports, so the request is a concrete platform-parity feature rather than an unclear stale report.
- **Existing Feishu broadcast is inbound agent fan-out, not send-time bot event injection:** `handleFeishuMessage` resolves top-level `broadcast` by inbound group peer id, dedupes one received Feishu event across accounts, then dispatches the same inbound event to configured agents. Observer agents use a no-op dispatcher, so this does not make one bot's outbound Feishu reply appear as an inbound message to the other bot accounts.

## Risks / Open Questions

- Existing Feishu broadcast groups partially overlap the multi-agent goal, so a future fix should reuse or deliberately evolve that config instead of adding a parallel mechanism with unclear precedence.
- Synthetic bot-to-bot delivery can create reply loops or token amplification unless it is opt-in, scoped per peer/account, deduped, and guarded by consecutive-bot-turn limits.
