## Summary

Keep open. Current `main` still rewrites only the active branch and leaves on-disk cleanup to nobody, while transcript readers still scan every JSONL line, so the duplication problem described in PR #69486 remains plausible on shipped code. The PR discussion also surfaced a credible unresolved regression risk around `label.targetId` references, and that concern matches current session-truncation contracts, so there is not strong enough evidence to auto-close this PR.
