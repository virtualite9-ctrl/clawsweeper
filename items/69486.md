## Summary

Keep PR #69486 open. Current main still has the rewrite path that branches from the first replaced message and re-appends the active suffix without any on-disk cleanup, while `readSessionMessages` still scans every raw transcript line. The PR is a focused, recent bug-fix candidate with a production-derived repro, but it should not be auto-closed or treated as implemented because main lacks the cleanup and the PR discussion still has a concrete unresolved integrity concern around non-parent references such as `label.targetId`.

## Best Possible Solution

Keep #69486 open as a focused bug-fix candidate. The best path is to update the PR so its cleanup computes removability from the full session-entry reference contract, not only parent ancestry, and add a regression case for a surviving label or sibling metadata entry that targets a rewritten-path message. After that, maintainers can review the production-derived repro plus the repeated-rewrite raw-file tests and run the changed agent lanes.
