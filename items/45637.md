## Summary

Keep issue #45637 open. Current main no longer matches the exact hardcoded 30-second framing, because agent runs now have a long run timeout and a configurable global LLM idle timeout, but the requested provider-scoped LLM timeout config is still not implemented. `models.providers.*` has transport overrides for headers/auth/proxy/TLS/private-network policy, not `timeoutMs`, and the strict provider schema would not accept the proposed top-level key.

## Best Possible Solution

Keep the issue open as a real core config/runtime enhancement. A maintainer-quality fix should add a documented, schema-backed provider timeout field, decide whether it means model HTTP/stream transport timeout or provider-scoped run/idle budget, wire it through the selected model/provider path, and cover it with focused schema and runner/transport tests.
