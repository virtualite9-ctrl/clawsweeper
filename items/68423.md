## Summary

Keep open. Current `main` has a redacted-placeholder guard for the newer last-known-good recovery path, but the `.bak` auto-restore path named by this issue still reads, parses, and copies `openclaw.json.bak` without validating it for redacted secret placeholders. There is also an open, unmerged PR that explicitly closes this issue, so the issue should remain open while that PR is reviewed or replaced.

## Best Possible Solution

Keep this issue open and review PR #68462, or replace it with an equivalent maintainer-owned patch. The fix should validate parsed `.bak` contents before both async and sync restore copy operations, add regression coverage for polluted `.bak` refusal, and decide separately whether `maintainConfigBackups`, deeper rotation policy, and a config validation CLI belong in the same patch or follow-ups.
