## Summary

Keep open. Current main still unconditionally rejects `sessions_send` calls that include both `sessionKey` and `label`, so the live behavior this PR targets remains unfixed. The linked bug #64699 is still a concrete open report, and the provided PR context shows this PR is unmerged and aimed at the implicated guard.

## Best Possible Solution

Keep this PR open as a valid implementation candidate for #64699, then consolidate it with the overlapping open `sessions_send` PRs. The best landed fix should update `src/agents/tools/sessions-send-tool.ts` to allow equal trimmed `sessionKey` and `label` values to route by `sessionKey`, keep rejecting differing values, and add focused regression tests in the agent tool or gateway `sessions_send` tests.
