## Summary

Keep #72386 open. Current main still emits the reported `openclaw.runtime-context` next-turn custom message with the human-readable preface, queues it before the prompt, and the pinned Pi session dependency converts custom messages into user-role LLM messages. The existing sanitizer coverage still targets delimited and legacy `OpenClaw runtime context (internal):` blocks, not the new `OpenClaw runtime context for the immediately preceding user message.` header.

## Best Possible Solution

Keep the issue open for a core runtime fix. The best path is to stop putting the human-readable runtime-context preface into ordinary model-visible conversational content for channel-envelope deltas. Use a provider-aware/system-level representation or structured non-conversational runtime payload, then add regression coverage for the new header across `runtime-context-prompt`, embedded prompt assembly, and user-facing outbound sanitization.
