## Summary

Keep open. Current main still sends provider-backed hybrid memory search through one full-query FTS lookup, while the FTS query builder AND-joins tokens. The PR is stale against the moved `extensions/memory-core` layout and needs the fan-out review addressed, but the underlying multi-word hybrid recall bug is not implemented away or clearly superseded.

## Best Possible Solution

Keep #39555 open as an actionable bug-fix candidate. Port the fix to `extensions/memory-core`, cap or rank extracted terms before per-keyword FTS fan-out, and add active regression coverage for provider-backed hybrid queries whose relevant terms span chunks. If maintainers prefer #57711 or another PR as the canonical path, explicitly mark that item as the replacement before closing this one.
