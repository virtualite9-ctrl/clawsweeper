## Summary

Keep #39555 open. Current main still routes provider-backed hybrid memory search through a single full-query FTS lookup, while the FTS query builder AND-joins query tokens. That leaves the reported multi-word/cross-chunk recall bug plausible and not implemented away. The PR is stale against the current `extensions/memory-core` layout and has a valid fan-out performance review to address, but the underlying fix remains actionable.

## Best Possible Solution

Keep #39555 open as an actionable memory-core bug-fix candidate. Port the idea to `extensions/memory-core`, add bounded/ranked per-keyword FTS recall for provider-backed hybrid search, and add an active regression test where relevant query terms span chunks. If maintainers prefer #57711 or another PR as the canonical fix, mark that item explicitly as the replacement before closing this one.
