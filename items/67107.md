## Summary

Keep open. Current main still requires Node >=22.14, the macOS installer still falls through to Homebrew `node@24` without a Big Sur/macOS 11 preflight, and the install docs still present generic macOS support without documenting a macOS 12+ floor or an early unsupported-OS message. The existing issue comment usefully reframed this as installer/docs UX rather than Node 18 runtime support, and that narrower work is still actionable in this repository.

## Best Possible Solution

Keep this issue open but narrow it to installer/docs UX. The conservative fix is to add an early macOS version preflight in `scripts/install.sh` before Homebrew/Node installation, document the tested minimum macOS version on `docs/install/index.md` and `docs/install/node.md`, and keep runtime support at Node >=22.14 rather than trying to support Node 18.
