## Summary

Keep open. Current main still allows the reported AWS IMDS probe path: the bundled Amazon Bedrock memory embedding adapter calls hasAwsCredentials(), and hasAwsCredentials() still invokes @aws-sdk/credential-provider-node defaultProvider() without an AWS_EC2_METADATA_DISABLED guard or any IMDS suppression. The related fix PR #64944 was closed unmerged, and current tests do not cover the requested IMDS-disable behavior.

## Best Possible Solution

Keep this issue open. Implement a focused Amazon Bedrock plugin fix around extensions/amazon-bedrock/embedding-provider.ts so hasAwsCredentials() does not probe IMDS during passive memory-embedding auto-detection, honors AWS_EC2_METADATA_DISABLED, restores any temporary env changes, and keeps the normal AWS SDK credential chain intact for real Bedrock runtime calls. Add regression tests in extensions/amazon-bedrock/embedding-provider.test.ts for AWS_EC2_METADATA_DISABLED and IMDS suppression/restoration; update provider/memory docs only if the credential auto-detection contract changes.
