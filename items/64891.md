## Summary

Current main still leaves the reported IMDS probe path in place. The Amazon Bedrock memory embedding adapter calls hasAwsCredentials during provider creation, and hasAwsCredentials still reaches @aws-sdk/credential-provider-node defaultProvider({ timeout: 1000, maxRetries: 0 }) without an AWS_EC2_METADATA_DISABLED guard or temporary IMDS suppression. The related fix PR #64944 was closed unmerged, so this issue should remain open.

## Best Possible Solution

Keep this issue open. Review and land a maintained fix in the amazon-bedrock plugin that prevents hasAwsCredentials auto-detection from probing IMDS when no explicit AWS credential marker is present, honors AWS_EC2_METADATA_DISABLED, and adds focused tests in extensions/amazon-bedrock/embedding-provider.test.ts. Preserve the normal AWS SDK credential chain for actual Bedrock runtime calls.
