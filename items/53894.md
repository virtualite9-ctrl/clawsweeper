## Summary

Current `main` already ships a persistent suggestion/proposal store for the closest matching feature: the `skill-workshop` plugin stores proposals under the Gateway state directory, reloads them on demand, documents the persisted location, and has regression coverage for re-reading stored proposals.

## Close Comment

Codex review: current `main` already implements durable suggestion persistence for the matching workflow-memory feature. `extensions/skill-workshop/src/store.ts:45` reads proposal state from disk and `extensions/skill-workshop/src/store.ts:83` writes it back atomically; `extensions/skill-workshop/src/workshop.ts:23` wires that store to the Gateway state directory; and `docs/plugins/skill-workshop.md:263` documents the persisted location as `<stateDir>/skill-workshop/<workspace-hash>.json`. Regression coverage in `extensions/skill-workshop/index.test.ts:99`, `extensions/skill-workshop/index.test.ts:616`, and `extensions/skill-workshop/index.test.ts:734` verifies stored pending/quarantined proposals are visible from a fresh store instance. The implementation differs from the Postgres/`before_agent_start` design proposed here, but the persistence problem itself is already solved on `main` and shipped in `v2026.4.21` (`CHANGELOG.md:312`).
