## Summary

Keep open. Current main still hardcodes Slack assistant thread status as "is typing..." and does not expose `identity.typingStatuses`, Slack `statusMessages`, or `loading_messages` in config/types/schema/docs/tests. The PR is not merge-ready as-is because it targets older `src/slack` paths and has unresolved validation/test gaps, but the requested feature is still unimplemented and the patch is focused enough to remain useful.

## Best Possible Solution

Keep the PR open or replace it with a focused maintainer-owned follow-up. The implementation should live in the current bundled Slack plugin path, add a deliberate typed and validated Slack assistant-status config surface, pass configured `status` and optional `loading_messages` through `setSlackThreadStatus`, preserve empty-status cleanup, add Slack tests for default and configured behavior, and update Slack docs/config metadata as needed.
