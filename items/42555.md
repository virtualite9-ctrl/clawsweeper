## Summary

Keep open. Current main still hardcodes Slack assistant thread status as "is typing..." and exposes no `identity.typingStatuses`, Slack `statusMessages`, or `loading_messages` config surface. This PR is not merge-ready as-is because it targets the removed `src/slack/...` path and lacks validation/tests/docs, but the requested Slack customization remains unimplemented and the focused PR discussion is still useful.

## Best Possible Solution

Keep this PR open or replace it with a focused maintainer follow-up. The best implementation belongs in the current bundled Slack plugin path, with typed and validated Slack assistant-status config, tests for default and configured behavior, docs/config metadata updates, and explicit resolution of whether the public surface should be per-agent identity, Slack channel/account config, or superseded by #67129.
