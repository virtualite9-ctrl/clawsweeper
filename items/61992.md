## Summary

#61992 should close as superseded by active PR #72009. Current main still does not expose a typing TTL knob: the reply typing controller keeps a hardcoded default TTL and get-reply only wires typingIntervalSeconds. PR #72009 now tracks the same central feature with session/agent typingTtlSeconds, docs, schema, and tests; earlier #42851/#42105 are useful related attempts but not the current implementation thread.

## Best Possible Solution

Close #61992 as superseded by active implementation PR #72009 and keep review/testing on that PR. The product direction should be a documented session/agent typingTtlSeconds config wired through types, zod schemas, generated schema/help, docs, runtime controller wiring, and regression tests; any older overlapping PRs should be deduped against that canonical implementation thread.
