## Summary

Keep PR #70811 open. Current main still validates `entry?.config` unconditionally, and `validatePluginConfig` still coerces `undefined` to `{}` before JSON-schema validation, so the false-positive strict-schema path described in #70658 remains present on main. The PR is a recent, focused bug-fix attempt with regression coverage and active author follow-up rather than obsolete cleanup.

## Best Possible Solution

Keep PR #70811 open for maintainer review and likely merge or revise the same narrow loader fix. The best path is to defer validation only for non-activating loads with omitted config, preserve normalized/defaulted `api.pluginConfig`, keep activation and explicit-config validation covered, and retain the focused loader and CLI metadata regression tests.
