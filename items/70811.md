## Summary

Keep this PR open. Current main still validates omitted plugin config as `{}` in the non-activating snapshot and CLI metadata paths, so the bug this PR targets is not implemented on main. The provided PR context shows a recent, focused bug-fix candidate with regression coverage and follow-up commits, not obsolete cleanup.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to review or land a context-aware loader fix that skips schema validation only when `activate: false` and config is actually `undefined`, preserves normalized/defaulted `api.pluginConfig` for non-activating registration, keeps real activation and explicit config validation strict, and retains focused regression coverage in `src/plugins/loader.test.ts` and `src/plugins/loader.cli-metadata.test.ts`.
