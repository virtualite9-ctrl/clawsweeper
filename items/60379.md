## Summary

Current `main` does not implement a one-shot per-message model override. `/model` is still treated as a shared session override, and mixed-content messages like `/model openai/gpt-4o hello` are explicitly persisted into session state.

## Evidence

## Risks / Open Questions

- Closing this as implemented would be incorrect: current main still makes `/model` sticky across subsequent session activity, which is the core problem the issue reports.
- There is a docs/behavior mismatch today: general directive docs describe inline directives as per-message hints, but `/model` is implemented and tested as persistent session state even in mixed-content messages.
