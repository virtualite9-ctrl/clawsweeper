## Summary

Keep #8285 open. Current main still lacks a supported configurable pre-processing text acknowledgement, and the existing hook/API surface remains terminal or observational rather than a safe interim-send contract. The issue is concrete, recent, and already includes Slack/WhatsApp/webchat plus Feishu user evidence and a maintainer-side Codex review explicitly keeping it open.

## Best Possible Solution

Keep #8285 open as a core/product or plugin-API feature request. The best path is to add a supported non-terminal pre-processing acknowledgement contract, such as `messages.ackText` with scope and per-channel/account overrides or a documented plugin hook with a safe send delegate, then update config types, zod schema, generated config help/docs, public docs, and channel tests for Slack, WhatsApp, webchat, and Feishu so the acknowledgement sends immediately while normal processing and final replies continue.
