## Summary

Keep #8285 open. Current main still lacks a supported configurable text acknowledgement before normal agent processing. Reaction acknowledgements, status reactions, typing indicators, and terminal plugin hook paths exist, but none implement a cross-channel `messages.ackText`-style config or non-terminal send-and-continue hook.

## Best Possible Solution

Keep #8285 open as a core/product or plugin-API feature request. The best implementation path is a supported non-terminal pre-processing acknowledgement contract, likely `messages.ackText` with scope and per-channel/account overrides, or a documented plugin/channel hook that can send interim text and then continue normal model dispatch. It should update config types, zod schema, generated schema/help/labels, public docs, and channel tests for affected messaging surfaces.
