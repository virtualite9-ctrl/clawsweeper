## Summary

Keep #8285 open. Current main still lacks a supported configurable pre-processing text acknowledgement: the message config/schema/docs expose ack reactions, status reactions, and typing indicators, but no standalone `messages.ackText`/`ackTextScope` surface. The existing plugin hooks do not close the gap because `before_dispatch` handled text is terminal and `reply_dispatch` handled ownership skips the default model path. The issue discussion is concrete, multi-channel, and the latest maintainer-side Codex comment also explicitly kept it open.

## Best Possible Solution

Keep #8285 open as a core/product or plugin-API feature request. The best path is to add a supported non-terminal pre-processing acknowledgement contract, likely `messages.ackText` with scope and channel/account overrides or a documented plugin send hook/delegate that is explicitly safe for interim delivery, then update config types, zod schema, generated schema/help, public docs, and channel tests for affected messaging surfaces.
