## Summary

Keep open. Current main still has not fully resolved the weixin bare-root plugin SDK compatibility problem. The root SDK barrel remains intentionally small, focused subpaths do expose the requested helpers, and the compat/root-alias path appears only partial for this report. Maintainer discussion also explicitly keeps this issue as the canonical weixin/top-level SDK compatibility tracker, with #60416 closed into it and #68279 reserved for the separate stringEnum cluster.

## Best Possible Solution

Keep this issue open as the canonical weixin top-level plugin-SDK compatibility tracker. Maintainers should choose one explicit path: either add deliberate legacy root/compat exports for the affected helpers with API baseline and contract coverage, or coordinate and publish an @tencent-weixin/openclaw-weixin migration to focused openclaw/plugin-sdk/<subpath> imports. Keep #60416 closed as the duplicate and leave stringEnum-only failures under #68279.
