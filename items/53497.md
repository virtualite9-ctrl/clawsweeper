## Summary

Keep open. Current main still does not expose the weixin-requested helpers from the bare `openclaw/plugin-sdk` entry, while the helpers do exist on focused SDK subpaths. The provided maintainer discussion also explicitly keeps this issue as the canonical weixin/top-level SDK compatibility report, with #60416 closed into it and #68279 tracking the separate `stringEnum` cluster.

## Best Possible Solution

Keep this issue open as the canonical weixin top-level plugin-SDK compatibility tracker. Maintainers should explicitly choose between adding deliberate legacy root exports for the affected helpers with API baseline and contract coverage, or coordinating/publishing an `@tencent-weixin/openclaw-weixin` migration to focused `openclaw/plugin-sdk/<subpath>` imports. Leave #60416 closed as the duplicate and keep `stringEnum`-only failures under #68279.
