## Summary

Keep #13487 open. Current main still implements Discord mention and reply handling as per-running-account gates, not as a cross-account target router that chooses mentioned bots first, replied-to bot second, and suppresses the default owner when either target signal exists.

## Best Possible Solution

Keep #13487 open until the Discord plugin makes one deterministic target decision before normal/default routing: map configured Discord bot user IDs to OpenClaw accounts/agents, select explicit bot mentions first, referenced-message author second, then default owner only for untagged non-replies. Suppress non-target/default responders whenever a targeted mention or reply applies, and add regression tests for explicit mention, native reply, reply-plus-mention, and untagged default-owner cases.
