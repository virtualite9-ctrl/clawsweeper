## Summary

Keep #13487 open. Current main at 406ae72 still resolves Discord routing per account before mention/reply target metadata, then uses mention/reply data only to gate the current bot handler. There is still no Discord-wide target-selection step that maps mentioned/replied-to bot user IDs to the eligible OpenClaw account/agent and suppresses the default owner when a targeted bot applies.

## Best Possible Solution

Keep #13487 open until the Discord plugin makes one deterministic cross-account target decision before default routing: map configured Discord bot user IDs to their OpenClaw accounts/agents, choose explicit bot mentions first, then referenced-message author, and use the per-channel default owner only for untagged non-reply messages. Suppress all non-target/default responders whenever mention or reply targeting applies, and add regression tests for explicit mention, native reply, reply-plus-mention, multiple mentioned bots, and untagged default-owner cases.
