## Summary

Keep #13487 open. Current main still performs Discord mention/reply activation inside each running Discord account handler, using only that account's bot user ID, and shared route resolution can still fall back to the default agent without a cross-account target-suppression decision. The existing April 26 maintainer-context comment also explicitly kept this open and noted related #39428/#39485 do not implement the requested mention > reply-target > default-owner matrix.

## Best Possible Solution

Keep #13487 open until the Discord plugin makes one deterministic cross-account target decision before default routing: map configured Discord bot user IDs to owning OpenClaw accounts/agents, select explicit bot mentions first, then referenced-message author, and use the per-channel default owner only for untagged non-reply messages. Suppress all non-target/default responders whenever mention or reply targeting applies, and add regression tests for explicit mention, native reply, reply-plus-mention, and untagged default-owner cases.
