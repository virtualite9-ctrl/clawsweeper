## Summary

Keep PR #69697 open. Current main still has the reported crash path: `hashToolCall()` accepts `unknown` params, `stableStringifyFallback()` directly returns `stableStringify(undefined)`, and Node's `crypto.Hash.update(undefined)` throws `ERR_INVALID_ARG_TYPE`. The existing tests cover string, number, and null params, but not undefined params. The PR is a focused fix against the implicated file, though it should ideally add a regression test before merge.

## Best Possible Solution

Keep the PR or an equivalent patch alive. The best maintainer path is to fix `stableStringify`/`stableStringifyFallback` so undefined serializes deterministically before hashing, guard the same issue in `formatErrorForHash`, and add focused regression coverage in `src/agents/tool-loop-detection.test.ts` for undefined params. After that, run the changed core test/typecheck lanes before merge.
