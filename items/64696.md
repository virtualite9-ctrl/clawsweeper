## Summary

Keep open. Current main still lacks the requested memory-wiki bridge/lint durability fixes: lint still compares raw link targets, generated native report links are not relative to reports/, bridge source rendering has no renderer-version invalidation or reply-tag-safe extraction path, and imported source skip logic still trusts file existence instead of validating source-page frontmatter. The related #65197 PR was closed unmerged, and #70986 covers only part of the lint/parser surface, not the bridge cache/render invalidation or malformed cached-page repair criteria.

## Best Possible Solution

Keep this issue open and implement the bounded memory-wiki fixes in source with regression coverage: make link extraction markdown-aware or otherwise prevent bridged reply directives inside source content from becoming lint targets, resolve native Markdown links relative to the containing page, emit report-local native links for reports/stale-pages.md, add a bridge renderer-version token to the render fingerprint, and validate existing imported source-page frontmatter before skip. Coordinate the parser/resolver overlap with #70986, but do not close this issue until the bridge cache/render invalidation and malformed cached-page repair criteria are also covered.
