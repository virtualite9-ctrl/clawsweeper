## Summary

Keep open. Current main still does not implement the requested memory-wiki durability fixes: link linting still compares raw extracted targets, report links are still emitted vault-root-relative in native mode, bridge source rendering has no renderer-version invalidation, and cached imported source pages are still skipped based on file existence rather than valid source-page frontmatter. The closed #65197 PR appears to have been the proposed fix set, while open #70986 covers only part of the remaining lint/parser surface and does not cover bridge cache/render invalidation or malformed cached-page repair.

## Best Possible Solution

Keep this issue open and implement the bounded fixes in the memory-wiki plugin: make wiki link extraction markdown-aware, resolve native Markdown links relative to the containing page, emit report-local native links for reports/stale-pages.md, prevent reply directive tags in bridged source content from becoming lint targets, include a renderer-version token in bridge render fingerprints, and validate existing imported source-page frontmatter before skip. Coordinate the parser/resolver part with #70986, but do not close this issue until the bridge cache/render invalidation and malformed cached-page repair criteria are also covered with regression tests.
