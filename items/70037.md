## Summary

Current `main` already fixes the reported CLI session reset paths with a narrower implementation: it hashes only the static portion of the extra system prompt for session reuse, preserves strict prompt-mismatch invalidation for real prompt-shape changes, and already uses a stable `mcpResumeHash` for gateway loopback port changes. That makes this PR redundant on `main`.

## Evidence

- **Heartbeat/inbound metadata no longer participates in the CLI session reuse hash:** `getReplyRun` now builds `extraSystemPromptStaticParts` from stable context only, excluding `inboundMetaPrompt`, and passes that static string forward for CLI runs.
