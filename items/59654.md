## Summary

Current `main` already contains the channel-setup wizard guard that prevents adapter-shaped plugins from being treated like declarative wizards during `openclaw agents add`, plus regression coverage for that path. The `inherited auth-profiles from main agent` log in the report is normal fallback logging and not the crashing frame.

## Evidence

- **Plugin contract allows both wizard shapes:** `ChannelPlugin.setupWizard` is explicitly a union of `ChannelSetupWizard | ChannelSetupWizardAdapter`, so adapter-shaped plugins are valid and must not be forced through the declarative wrapper path.
  - file: [src/channels/plugins/types.plugin.ts:48](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/src/channels/plugins/types.plugin.ts#L48)

  - file: [src/commands/channel-setup/registry.ts:10](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/src/commands/channel-setup/registry.ts#L10)
