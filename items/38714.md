## Summary

Keep #38714 open. Current main still receives Discord reaction add/remove events and turns matching notifications into routed system events, but it does not expose Discord reactions through the typed plugin hook catalog, internal hook event contract, or the existing `message_received` hook bridge.

## Best Possible Solution

Keep #38714 open until maintainers choose and implement the contract. The likely fix belongs in the Discord reaction handling path plus the plugin/internal hook surface: either add dedicated typed reaction add/remove hooks or emit structured synthetic `message_received`/internal `message:received` events with emoji, action, reactor, target message, route/session, account/guild/channel/thread metadata, and removal state. Add Discord listener tests, plugin/internal hook tests, and docs. Coordinate agent wake/auto-response behavior separately with #11747.
