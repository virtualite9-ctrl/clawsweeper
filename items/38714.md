## Summary

Keep #38714 open. Current main still detects Discord reaction add/remove events, but the Discord reaction path only queues routed system events; it does not emit a typed plugin hook, an internal reaction hook, or a synthetic `message_received` event. The related closed reports #52658 and #60805 are supporting duplicates/use cases for this same missing hook/API behavior, while #11747 remains a separate broader auto-response/wake-policy follow-up.

## Best Possible Solution

Keep #38714 open until maintainers choose and implement the contract. The likely fix belongs in the Discord reaction handling plus plugin/internal hook surface: either add dedicated typed reaction add/remove hooks or emit structured synthetic `message_received`/internal message events with emoji, action, reactor, target message, route/session, account/guild/channel/thread metadata, and removal state. Add Discord listener tests, plugin/internal hook tests, and docs. Coordinate wake/auto-response policy separately with #11747.
