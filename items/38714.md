## Summary

Keep open. Current main still detects Discord reaction add/remove events and routes them as session-scoped system events, but it does not expose reaction-specific events through either the internal Hooks event list or the typed plugin hook surface. The issue has only the non-protected `stale` label, author association is `NONE`, and the provided discussion contains no duplicate, superseding PR, maintainer decision, or ClawHub/plugin workaround to justify closure.
