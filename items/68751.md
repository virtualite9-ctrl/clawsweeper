## Summary

Keep open. Current main has a useful startup-context mitigation that frames recent daily memory as untrusted when it is preloaded, but the bundled session-memory hook still persists raw transcript-shaped `user:` / `assistant:` turns on both `/new` and `/reset`. The remaining issue discussion correctly distinguishes this broader raw-instruction replay/reset-semantics problem from the narrower chat-template-token sanitizer work in PR #69961.

## Best Possible Solution

Keep #68751 open as the canonical tracker for session-memory raw instruction replay and reset semantics. A complete fix should stop persisting prior-session material on `/reset` or make that opt-in, and for `/new` either persist a neutral summary or wrap/escape historical transcript content at the file boundary and all recall/read paths. Add focused tests that raw prior user instructions are not saved or reintroduced as executable current-context instructions, while treating #69961/#69943 as narrower sanitizer work for malformed chat-template/tool-call poisoning.
