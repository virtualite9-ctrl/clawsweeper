## Summary

Keep #63552 open. Current main still does broad implicit provider discovery for normal non-live `models.json` generation even when explicit `models.providers` are configured. The PR is a focused performance change against a real production path, and the existing plugin ownership APIs appear to support the intended scoping. The related reports are broader/different and do not supersede this PR.

## Best Possible Solution

Keep the PR open and either land or adapt its narrow production change: pass explicit providers into the discovery-filter path, resolve their owning plugin ids with the existing provider ownership helpers, fall back to broad discovery when any explicit provider has no owner, preserve `OPENCLAW_LIVE_PROVIDERS=all` / mixed live-filter semantics, and keep the added regression coverage for non-live explicit-provider scoping.
