## Summary

Keep this PR open. Current main still does not implement the central optimization: non-live models.json provider discovery returns an unscoped filter, and the production resolveImplicitProviders call does not pass explicit configured providers into the discovery-filter path. The PR is a focused, plausible fix with regression coverage, and the overlapping #72352 is also open rather than a merged or maintainer-selected replacement.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to either land or adapt its focused implementation: pass explicit providers into the discovery-filter path, resolve their owning plugin IDs with the existing ownership helper, fall back to broad discovery when any explicit provider has no owner, preserve live all and mixed live-filter semantics, and keep focused regression tests. If maintainers prefer #72352 as the canonical broader fix, close one PR only after explicitly selecting that path or after the chosen fix merges.
