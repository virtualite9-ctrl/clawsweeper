## Summary

Keep open. Current main still emits the generic missing-register diagnostic for bundled channel entries loaded through the legacy plugin loader, and this PR is an active implementation candidate for that remaining logging gap. It should be cleaned up before merge, but it is not obsolete or implemented on main.

## Best Possible Solution

Keep this PR open. Ask for a narrow cleanup: remove TODO.md, preserve formatMissingPluginRegisterError and OPENCLAW_PLUGIN_LOAD_DEBUG module-shape diagnostics for normal plugins, keep the bundled-channel-entry wrong-loader diagnostic and field-level bundled-entry warnings, and land only after focused loader/channel tests pass.
