## Summary

Keep #69055 open. Current main has generic outbound delivery diagnostics and message_sent hooks, and the cron path has the needed job, run/session, agent, account, target, and send-result data in nearby code, but it does not emit the requested structured `cron.delivery.sent` log/event with cron job/run attribution and provider message IDs. The linked #69054 is a complementary config-lint request, not a canonical replacement for this forensic logging gap.

## Best Possible Solution

Keep the issue open and implement a narrow cron-owned telemetry path: thread `{cronJobId, cronRunId, agentId}` plus resolved `{accountId, channel, to}` into the direct cron outbound call, emit exactly one structured `cron.delivery.sent` record for each send attempt after success/error with provider message IDs from `OutboundDeliveryResult`, omit message body/prompt/token data, and add focused regression tests for success, failure, best-effort partial failure, Telegram-style IDs, and Discord-style IDs. #69054 can remain the separate config-lint companion.
