## Summary

Keep #69055 open. Current main has generic outbound delivery diagnostics, message-sent hooks, and cron run delivery traces, but it does not emit the requested privacy-safe `cron.delivery.sent` structured log line or otherwise thread `cronJobId`/`cronRunId` through outbound delivery attempts with platform message IDs. The companion #69054 remains separate config-time lint work and does not supersede this forensic logging request.

## Best Possible Solution

Keep the issue open and implement this in the core cron/outbound boundary: thread a small cron delivery telemetry object from `dispatchCronDelivery` into the shared outbound send path or a cron-specific wrapper, then emit exactly one structured, content-free `cron.delivery.sent` record for every success and error with `cronJobId`, `cronRunId`, `agentId`, `accountId`, channel/target, platform message id, and status. Add focused tests around `src/cron/isolated-agent/delivery-dispatch.ts` and `src/infra/outbound/deliver.ts` so success, thrown errors, and best-effort partial failures are covered.
