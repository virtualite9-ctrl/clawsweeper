## Summary

Keep open. Current main still has the provider-agnostic Anthropic adaptive-thinking path that turns configured `high`/`xhigh` reasoning for Opus 4.7 into `output_config.effort` without checking `model.provider === "github-copilot"`. The bundled GitHub Copilot plugin defaults/dynamic models mitigate the default catalog path by setting Claude models to `reasoning: false`, but they do not fix the transport bug for configured reasoning-capable Copilot Opus 4.7 metadata, and the linked fix PRs for #69928 were closed unmerged.

## Best Possible Solution

Keep the issue open and land a focused provider/model capability fix in the Anthropic Messages request-shaping path: for `github-copilot/claude-opus-4.7`, either omit `output_config.effort` or clamp every unsupported effort (`low`, `high`, `xhigh`) to `medium`, with regression tests covering Copilot Opus 4.7 high/xhigh, direct Anthropic Opus 4.7 preservation, and an out-of-scope Copilot Claude model guard.
