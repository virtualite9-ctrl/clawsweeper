## Summary

Keep open. Current `main` has related ACP child-run lifecycle tracking, but the bundled ACPX path still does not implement the issue’s requested automatic retention/TTL cleanup by default: ACP one-shot spawns default to `cleanup: "keep"`, archive scheduling is skipped for kept runs, and the ACPX plugin exposes no retention setting beyond an ignored legacy `queueOwnerTtlSeconds` compatibility field.

## Evidence
