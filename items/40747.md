## Summary

Keep open. Current `main` still requires `after_block_id` for `feishu_doc` inserts, so the requested prepend/index capability is not implemented on `main`. The PR addresses a real gap, although it likely still needs the `index` schema tightened to a non-negative integer before merge.

## Evidence

- **Insert schema still requires after_block_id on main:** The `insert` variant in the Feishu doc tool schema still declares `after_block_id` as a required string and exposes no `index` field.
  - file: [extensions/feishu/src/doc-schema.ts:35](https://github.com/openclaw/openclaw/blob/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503/extensions/feishu/src/doc-schema.ts#L35)
  - command: `nl -ba extensions/feishu/src/doc-schema.ts | sed -n '28,52p'`
  - sha: [b0d2003fe65b](https://github.com/openclaw/openclaw/commit/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503)
- **Insert implementation still requires a concrete block id:** `insertDoc` on `main` takes `afterBlockId: string`, fetches that block, finds its sibling index, and inserts after it. There is no code path for inserting at the document start without an existing block reference.
  - file: [extensions/feishu/src/docx.ts:1030](https://github.com/openclaw/openclaw/blob/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503/extensions/feishu/src/docx.ts#L1030)
  - command: `nl -ba extensions/feishu/src/docx.ts | sed -n '1030,1105p'`
  - sha: [b0d2003fe65b](https://github.com/openclaw/openclaw/commit/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503)
- **Tool dispatch still only passes after_block_id for insert:** The `feishu_doc` action switch calls `insertDoc(..., p.after_block_id, ...)` and does not pass any positional `index` for inserts.
  - file: [extensions/feishu/src/docx.ts:1461](https://github.com/openclaw/openclaw/blob/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503/extensions/feishu/src/docx.ts#L1461)
  - command: `nl -ba extensions/feishu/src/docx.ts | sed -n '1458,1476p'`
  - sha: [b0d2003fe65b](https://github.com/openclaw/openclaw/commit/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503)

## Risks / Open Questions

- The PR as described still appears to need one follow-up fix: the new `index` field should be validated as `Type.Integer({ minimum: 0 })`, consistent with the existing `upload_image` index schema in `extensions/feishu/src/doc-schema.ts:153`.
- There does not appear to be existing Feishu test coverage for the `insert` action path, so merge confidence would be higher with a targeted test for `index: 0` and for legacy `after_block_id` behavior.
