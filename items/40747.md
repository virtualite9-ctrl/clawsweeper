## Summary

Keep this PR open. Current main still requires `after_block_id` for `feishu_doc(action: "insert")` and has no insert `index` parameter, while the PR is a focused implementation candidate for that missing Feishu doc insertion mode. It needs a small schema validation fix and targeted coverage before merge.

## Best Possible Solution

Keep this PR open or replace it with an equivalent maintainer patch in `extensions/feishu`: make `after_block_id` optional only for `action: "insert"`, add `index: Type.Optional(Type.Integer({ minimum: 0, ... }))`, pass `p.index` into `insertDoc`, use `index ?? 0` for document-root insertion when `after_block_id` is omitted, preserve the existing after-block branch, and add focused Feishu doc tests plus skill/docs updates for the new insert mode.

## Risks / Open Questions

- The PR currently validates `index` as `Type.Number`, which would allow negative and fractional values through tool validation and push clearer errors down to Feishu.
- The PR does not add targeted tests for the new root-index insertion path or for preserving the current `after_block_id` path.
- The Feishu doc skill/docs currently document append and upload-image index behavior, but not an `insert` action with root `index`, so the user-facing tool guidance would remain incomplete after the code change unless updated.
- Closing this PR now would discard a focused implementation candidate for a still-missing Feishu document insertion capability.
