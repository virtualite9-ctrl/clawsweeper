## Summary

Current `main` still appears to have the gap this PR fixes. `scripts/clawdock/clawdock-helpers.sh` auto-loads `docker-compose.extra.yml` but not `docker-compose.sandbox.yml`, while `scripts/docker/setup.sh` still generates and uses `docker-compose.sandbox.yml` when sandboxing is enabled. That leaves the PR as a live bug fix rather than conservative cleanup.
