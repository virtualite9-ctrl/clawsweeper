## Summary

Keep open. Current `main` still enforces a strict root-only realpath check for loaded skills, so a managed skill symlink under `~/.openclaw/skills` that resolves into another documented skill root like `~/.agents/skills` would still be rejected and logged as an escaped path. I found no equivalent allowlist or alternate implementation on `main` for the PR’s proposed fix.

## Evidence
