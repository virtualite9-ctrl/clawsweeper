## Summary

Keep this PR open. Current main still does not implement the central OpenResponses behavior: the HTTP SSE path ignores internal `stream: "tool"` agent events, terminal snapshots still only include the assistant/client-tool items that current code already knows about, and the output schema still lacks `function_call_output` as an output item. The PR remains a plausible focused implementation candidate, with review feedback already addressed in later pushes.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land this PR or an equivalent gateway change that bridges internal `stream: "tool"` events into OpenResponses SSE as stable `function_call` and `function_call_output` output items, includes those items in terminal response snapshots, preserves raw HTTP tool output without leaking it through shared agent-event subscribers, normalizes response-shaped next-turn inputs, and adds focused gateway/agent-event regression coverage.
