## Summary

Close as `cannot_reproduce`: the report describes Matrix room traffic that OpenClaw intentionally receives via sync but does not route unless the room message mentions the bot or mention-gating is explicitly disabled for that room.

## Evidence

- **Group behavior docs:** The shared group-routing docs say group replies require a mention by default, and the flow explicitly stores non-mentioned group messages for context only instead of replying.
  - file: [docs/channels/groups.md:15](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/docs/channels/groups.md#L15)
  - command: `nl -ba docs/channels/groups.md | sed -n '1,120p'`
  - sha: [926068b14fc6](https://github.com/openclaw/openclaw/commit/926068b14fc661e915a0247e9ecb5949fb95b4f7)
- **Matrix docs: auto-join vs room trigger config:** The Matrix docs separate invite auto-join from room reply behavior: `autoJoin`/`autoJoinAllowlist` control invites, while `groups.<room>.autoReply` disables mention requirements and `groups.<room>.requireMention` controls mention-gating per room.
  - file: [docs/channels/matrix.md:780](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/docs/channels/matrix.md#L780)
  - command: `nl -ba docs/channels/matrix.md | sed -n '748,802p'`
  - sha: [926068b14fc6](https://github.com/openclaw/openclaw/commit/926068b14fc661e915a0247e9ecb5949fb95b4f7)
- **Current Matrix handler:** Current main computes `shouldRequireMention` as `true` by default for rooms unless `autoReply: true` or `requireMention: false` is configured, then returns early for room messages with no mention after logging `reason: "no-mention"`.
  - file: [extensions/matrix/src/matrix/monitor/handler.ts:953](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/extensions/matrix/src/matrix/monitor/handler.ts#L953)
  - command: `nl -ba extensions/matrix/src/matrix/monitor/handler.ts | sed -n '930,1010p'`
  - sha: [926068b14fc6](https://github.com/openclaw/openclaw/commit/926068b14fc661e915a0247e9ecb5949fb95b4f7)
- **Routing test for dropped unmentioned room messages:** The Matrix monitor tests assert that a room reply without a mention does not record an inbound session and does not refresh thread bindings, which matches the reported "sync sees it but no session is created" symptom.
  - file: [extensions/matrix/src/matrix/monitor/handler.test.ts:1333](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/extensions/matrix/src/matrix/monitor/handler.test.ts#L1333)
  - command: `nl -ba extensions/matrix/src/matrix/monitor/handler.test.ts | sed -n '1320,1405p'`
  - sha: [926068b14fc6](https://github.com/openclaw/openclaw/commit/926068b14fc661e915a0247e9ecb5949fb95b4f7)
- **History buffering test:** The Matrix history tests assert that a non-trigger room message does not dispatch immediately, and only appears later as `InboundHistory` when a trigger arrives.
  - file: [extensions/matrix/src/matrix/monitor/handler.group-history.test.ts:141](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/extensions/matrix/src/matrix/monitor/handler.group-history.test.ts#L141)
  - command: `nl -ba extensions/matrix/src/matrix/monitor/handler.group-history.test.ts | sed -n '140,166p'`
  - sha: [926068b14fc6](https://github.com/openclaw/openclaw/commit/926068b14fc661e915a0247e9ecb5949fb95b4f7)
- **Issue discussion:** The maintainer comment on 2026-04-03 already pointed to the same explanation: `groupPolicy: "open"` allows the room, but room replies stay mention-gated by default; `autoJoinAllowlist` only affects joining rooms; skipped room messages may still be buffered for later context.

## Risks / Open Questions

- This close only addresses the no-mention case described in the issue body and maintainer follow-up. If a visibly mentioned Matrix room message still fails to create a session on current main, that would be a separate bug.
