## Summary

Keep #39659 open. Current main has generic profile-based multi-gateway isolation and Docker/ClawDock convenience helpers, but it still lacks the requested first-class Docker multi-instance lifecycle with named create/list/delete/update, per-instance metadata, compose project/container isolation, and managed port/path allocation.

## Best Possible Solution

Keep the issue open until maintainers decide or implement a minimal supported Docker multi-instance lifecycle. A good solution would add a documented command or helper that records per-instance metadata, validates or allocates unique ports and host paths, isolates compose project/container names, and provides safe create/list/update/delete cleanup with focused tests and Docker install docs. The existing profile and ClawDock primitives should be reused where they fit, but they do not currently satisfy the Docker lifecycle request.
