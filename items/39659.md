## Summary

Keep open. Current `main` documents manual multi-gateway isolation, but it does not provide the requested first-class Docker instance lifecycle (`openclaw instance create/list/delete/update`) or equivalent named multi-instance management for Docker installs.

## Evidence

- **No Docker instance CLI exists on main:** A repo search for `instance create`, `instance list`, `instance delete`, and `instance update` found no matching OpenClaw CLI commands; the only `instance` hits were unrelated runtime/docs uses.
  - file: [src](https://github.com/openclaw/openclaw/blob/cb2c36b04975e34ab7014a11575d6589aa96ae32/src)
  - command: `rg -n "command\(.*instance|instance create|instance list|instance delete|instance update" src packages ui extensions docs -g '!node_modules'`
  - sha: [cb2c36b04975](https://github.com/openclaw/openclaw/commit/cb2c36b04975e34ab7014a11575d6589aa96ae32)
- **Docker docs still describe a manual compose/env flow:** The Docker install doc tells users to run `docker compose` manually, manage behavior through environment variables, and optionally use ClawDock shell helpers. It does not document any named Docker-instance create/list/delete/update workflow.
  - file: [docs/install/docker.md:95](https://github.com/openclaw/openclaw/blob/cb2c36b04975e34ab7014a11575d6589aa96ae32/docs/install/docker.md#L95)
  - command: `nl -ba docs/install/docker.md | sed -n '90,210p'`
  - sha: [cb2c36b04975](https://github.com/openclaw/openclaw/commit/cb2c36b04975e34ab7014a11575d6589aa96ae32)
- **Docker setup script supports manual overrides, not named instance management:** `scripts/docker/setup.sh` only exposes manual path/port/image env overrides like `OPENCLAW_CONFIG_DIR`, `OPENCLAW_WORKSPACE_DIR`, and `OPENCLAW_GATEWAY_PORT`, then writes repo-local compose/env state. There is no instance registry, list/delete/update flow, or per-instance naming layer.
  - file: [scripts/docker/setup.sh:235](https://github.com/openclaw/openclaw/blob/cb2c36b04975e34ab7014a11575d6589aa96ae32/scripts/docker/setup.sh#L235)
  - command: `nl -ba scripts/docker/setup.sh | sed -n '230,470p'`
  - sha: [cb2c36b04975](https://github.com/openclaw/openclaw/commit/cb2c36b04975e34ab7014a11575d6589aa96ae32)
- **ClawDock is a single-directory helper, not multi-instance management:** ClawDock stores one `CLAWDOCK_DIR`, auto-detects a single OpenClaw checkout, and shells out to `docker compose` in that directory. It does not manage multiple named Docker instances.
  - file: [scripts/clawdock/clawdock-helpers.sh:39](https://github.com/openclaw/openclaw/blob/cb2c36b04975e34ab7014a11575d6589aa96ae32/scripts/clawdock/clawdock-helpers.sh#L39)
  - command: `nl -ba scripts/clawdock/clawdock-helpers.sh | sed -n '30,180p'`
  - sha: [cb2c36b04975](https://github.com/openclaw/openclaw/commit/cb2c36b04975e34ab7014a11575d6589aa96ae32)
- **Current main only covers generic multi-gateway isolation:** `docs/gateway/multiple-gateways.md` documents running multiple gateways via separate profiles/ports/state dirs, which is a manual generic gateway pattern rather than first-class Docker instance lifecycle management.
  - file: [docs/gateway/multiple-gateways.md:31](https://github.com/openclaw/openclaw/blob/cb2c36b04975e34ab7014a11575d6589aa96ae32/docs/gateway/multiple-gateways.md#L31)
