## Summary

Close as implemented on `main`: the reported April 2026 dev-version LM Studio path predates the shipped bundled LM Studio provider. Current OpenClaw has a dedicated LM Studio plugin, onboarding/docs for `lmstudio/...` model refs, and persists LM Studio on the `openai-completions` transport, which builds `messages` payloads instead of Responses-style `input` payloads that match the reported `HTTP 400: Invalid type for input` failure.
