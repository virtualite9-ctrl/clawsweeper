## Summary

Current main addresses the bundled v2026.4.8 CLI hang report across the relevant surfaces: `channels list` is routed through read-only channel metadata with plugin preload disabled, `message` help/startup no longer broad-preloads plugins, and `cron list` uses the fixed one-shot Gateway RPC path with bounded WebSocket teardown and cron read-operation responsiveness coverage. The related cron follow-up #67439 is already closed.

## Best Possible Solution

Close #63249 as implemented on current main. Keep the current split fixes: routed/read-only channel listing, message startup/help without broad plugin preload, and bounded Gateway RPC teardown with cron read-operation responsiveness tests. Use the already-closed #67439 cron follow-up as the canonical related cron evidence if future reports appear.
