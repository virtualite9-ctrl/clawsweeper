## Summary

Current main already routes auth-profile exhaustion into the model fallback path instead of leaking an undefined auth value, and it has regression coverage for all-profiles-in-cooldown plus cross-provider fallback.

## Evidence

## Close Comment

Closing from a Codex review: current `main` already implements this. The auth bootstrap now throws a `FailoverError` when a provider has no usable auth profile and model fallbacks are configured (`src/agents/pi-embedded-runner/run/auth-controller.ts:334`), instead of letting missing auth leak through. The outer fallback runner then skips cooldowned provider candidates and continues to later configured fallbacks (`src/agents/model-fallback.ts:754`). There is also regression coverage for both the inner auth-exhaustion case (`src/agents/pi-embedded-runner.run-embedded-pi-agent.auth-profile-rotation.e2e.test.ts:1201`) and cross-provider fallback after auth/billing cooldown (`src/agents/model-fallback.test.ts:1713`, `src/agents/model-fallback.test.ts:1741`). I could not confirm a released tag containing this yet, but it is present on current `main` in commit `ed6b487e2033d87a85b5713a964cc677c70353f8`.
