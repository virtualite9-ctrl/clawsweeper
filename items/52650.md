## Summary

Current `main` already implements browser-local user avatar support in the Control UI/WebUI: Quick Settings includes a Personal card with name, avatar text/emoji, and image upload; the identity is persisted in browser storage; chat rendering uses the configured user avatar; and the feature is documented and called out in the `v2026.4.22` changelog.

## Risks / Open Questions

- The implementation is in Quick Settings/Personal rather than specifically replacing the top-right icon cluster, but it satisfies the core requested capability: setting and using a user avatar in the WebUI.

## Close Comment

Closing as implemented on `main` after a Codex review. The current WebUI already supports a browser-local personal identity with user avatar controls in Quick Settings (`ui/src/ui/views/config-quick.ts:454`), persists that identity locally (`ui/src/ui/storage.ts:281`), and renders user avatars in chat (`ui/src/ui/chat/grouped-render.test.ts:328`). The feature is also documented in `docs/web/control-ui.md:77` and shipped in release `v2026.4.22` per `CHANGELOG.md:129`. If you were specifically asking for a different placement in the top-right toolbar rather than the existing Personal settings flow, that would need a new, narrower UX request.
