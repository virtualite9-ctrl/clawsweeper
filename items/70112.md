## Summary

Keep PR #70112 open. Current main at 4a3030df9efa72c44bab567a4390ac0c4876a73f still uses the upstream chars/4 `estimateTokens()` in the compaction and preemptive compaction paths, so the CJK overflow bug is not implemented or superseded on main. The PR discussion shows useful maintainer review and author follow-up, but there is still at least one review concern around token-vs-char thresholds that should be resolved before merge.

## Best Possible Solution

Keep #70112 open and have maintainers either merge a refined version of this PR or ask the author to update it. The desired implementation should add a CJK-aware wrapper around upstream `estimateTokens`, use it consistently in `src/agents/compaction.ts`, `src/agents/pi-embedded-runner/compact.ts`, and `src/agents/pi-embedded-runner/run/preemptive-compaction.ts`, preserve ASCII behavior, cover CJK text and CJK tool-call arguments with focused regression tests, and resolve the preemptive truncation token-unit threshold concern before landing.
