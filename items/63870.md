## Summary

Keep this PR open. Current main fixed the original Claude CLI session-reset symptom with stable `mcpResumeHash` handling, but it still does not implement `gateway.mcpLoopback.port` or any other operator-facing way to pin the loopback MCP port. The remaining capability is plausible core gateway/config work, and the PR discussion shows useful follow-up rather than a rejected or superseded change.

## Best Possible Solution

Keep this PR open, but require a rebase against current main: add `gateway.mcpLoopback.port` to gateway config types, Zod/schema/help/generated metadata and docs, then thread it into the current lazy `ensureMcpLoopbackServer(configuredPort)` path with targeted tests for configured-port binding and bind-failure diagnostics. If maintainers decide the stable `mcpResumeHash` solution fully removes the need for a fixed operator port, close only after that explicit product decision or a canonical follow-up is recorded.
