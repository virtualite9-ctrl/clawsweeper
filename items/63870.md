## Summary

Keep this PR open. Current main still does not expose `gateway.mcpLoopback.port`, and the bundled MCP loopback startup path still calls `ensureMcpLoopbackServer()` without a configured port. The related session-reset symptom is partly handled by `mcpResumeHash`, but the operator-facing fixed loopback port requested here remains unimplemented and the PR needs a rebase onto the current lazy MCP startup path.

## Best Possible Solution

Keep this PR open but require a rebase against current main. If maintainers still want stable operator-configured MCP URLs, add `gateway.mcpLoopback.port` to the gateway config type, strict Zod schema, generated schema/help/labels, and public gateway configuration docs, then pass the configured value into the current lazy CLI backend `ensureMcpLoopbackServer(configuredPort)` path. Add tests for config parsing, configured-port binding through the actual prepare path, bind-failure diagnostics, and preservation of loopback-only bearer-token behavior. If maintainers decide the shipped `mcpResumeHash` behavior fully removes the product need, close only after recording that decision or linking a canonical follow-up.
