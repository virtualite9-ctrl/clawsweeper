## Summary

Keep this PR open. Current main fixes the original Claude/Codex CLI session-reset symptom by canonicalizing the OpenClaw loopback URL into a stable mcpResumeHash, but it still does not expose gateway.mcpLoopback.port or any operator-facing config path to pin the MCP loopback port. The PR is stale against the current lazy loopback startup path, but the requested capability remains plausible gateway/config work rather than obsolete or out of scope.

## Best Possible Solution

Keep this PR open, but require a rebase against current main. If maintainers still want a stable operator-configured MCP URL, add gateway.mcpLoopback.port to the gateway config type, strict Zod schema, generated schema/help/labels, and public gateway configuration docs, then thread the configured value into the current lazy CLI-backend ensureMcpLoopbackServer(configuredPort) path with targeted tests for configured-port binding and bind-failure diagnostics. If maintainers decide the stable mcpResumeHash fully removes the product need, close only after recording that product decision or linking a canonical follow-up.
