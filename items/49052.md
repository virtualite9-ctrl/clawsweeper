## Summary

Obsolete PR. On current `main`, the supported session-aware plugin path already carries `sessionKey` through compaction hook contexts, and the old Pi-only embedded extension-factory registration API this PR targets has been removed. The proposed change would only add a dead internal parameter without changing runtime behavior.

## Evidence

- **`buildEmbeddedExtensionFactories` is internal and has no `sessionKey` consumer:** Current `main` shows `buildEmbeddedExtensionFactories` only assembling the built-in compaction safeguard, context pruning extension, and tool-result middleware. Nothing in the helper forwards or consumes a `sessionKey`, so this PR's extra argument would still be unused.
