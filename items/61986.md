## Summary

Keep open. Current `main` still turns each forwarded Android `notifications.changed` event into an immediate heartbeat wake. The only mitigation on main is exact-summary dedupe plus the generic 250ms wake coalescer; there is no batching to the configured heartbeat interval, and the latest release `v2026.4.23` still has the same behavior.

## Evidence

- **Android emits one node event per notification change:** `DeviceNotificationListenerService` sends `notifications.changed` on every posted and removed notification after package, quiet-hours, and rate-limit checks. There is no interval buffer or heartbeat batching in the Android emitter path.
