## Summary

Current main still has the reported planning-only retry gaps. The actionable user-prompt gate remains a narrow allowlist, and the assistant planning detector still misses present-continuous narration such as "I'm running it now." One detail differs from the report: current main does not treat bare "GO" as an execution ack, but the same present-continuous gap still applies to accepted acks like "go ahead."

## Best Possible Solution

Keep this issue open and fix `resolvePlanningOnlyRetryInstruction` in `src/agents/pi-embedded-runner/run/incomplete-turn.ts`. A conservative patch should add regression tests for the reported non-allowlisted imperative prompt and for accepted ack prompts followed by present-continuous no-tool narration, then broaden the actionable prompt and planning-only narration detection without classifying casual chatter or completed-result summaries as retryable.
