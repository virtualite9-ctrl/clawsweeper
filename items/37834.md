## Summary

Keep open. Current `main` still intentionally avoids synthesizing missing `toolResult` entries for aborted or errored assistant turns, which is the core repair this issue asks for. Both the transcript repair path and the live session guard encode that behavior, and the tests lock it in.

## Evidence

- **Transcript repair still skips aborted/error turns:** `repairToolUseResultPairing` explicitly branches on `stopReason === "error" || stopReason === "aborted"` and returns without inserting synthetic missing tool results.
  - file: [src/agents/session-transcript-repair.ts:527](https://github.com/openclaw/openclaw/blob/55389f4be29359e2dd57ebaf548ba4a70331a445/src/agents/session-transcript-repair.ts#L527)

  - sha: [55389f4be293](https://github.com/openclaw/openclaw/commit/55389f4be29359e2dd57ebaf548ba4a70331a445)
- **Tests codify no repair for aborted turns:** Current tests assert that aborted assistant messages must not get synthetic tool results, while normal `stopReason: "toolUse"` messages still do.
  - file: [src/agents/session-transcript-repair.test.ts:170](https://github.com/openclaw/openclaw/blob/55389f4be29359e2dd57ebaf548ba4a70331a445/src/agents/session-transcript-repair.test.ts#L170)
