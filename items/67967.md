## Summary

Keep open. Current main still sends MiniMax Anthropic-compatible replay through the generic hybrid Anthropic/OpenAI policy, which enables strict tool-call ID sanitization. The PR remains a focused provider-local fix with a matching regression-test update, and the provided diff does not touch CI, dependency resolution, lockfiles, install/build/release scripts, package publishing metadata, permissions, secrets, downloaded artifacts, or other security-sensitive execution paths.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land this provider-local replay-policy wrapper, or an equivalent MiniMax-owned hook, so minimax and minimax-portal disable tool-call ID sanitization only for anthropic-messages while keeping OpenAI-compatible paths strict. A shared hybrid-family sanitizeToolCallIds option can be considered later if more providers need the same knob.
