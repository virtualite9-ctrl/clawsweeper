## Summary

Keep open. Current main still routes MiniMax through the generic hybrid Anthropic/OpenAI replay hooks, so the Anthropic-compatible MiniMax path still gets strict tool-call ID sanitization. The proposed PR remains a focused provider-owned fix with a matching regression test change, and the security pass found no CI, dependency, lockfile, script, publishing, or secret-handling changes in the provided diff.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land this provider-local replay-policy wrapper, or an equivalent MiniMax-owned hook, so minimax and minimax-portal disable tool-call ID sanitization only for anthropic-messages while preserving strict sanitization for OpenAI-compatible paths. A shared hybrid-family sanitizeToolCallIds option can be considered later if more providers need the same knob.
