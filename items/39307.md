## Summary

Keep #39307 open. Current main still does not meet the core request: compaction notices are silent by default unless `agents.defaults.compaction.notifyUser` is enabled, Control UI status is transient, and post-compaction recovery is injected as internal agent guidance rather than a reliable user-facing disclosure with a reload/re-grounding offer. Related #50790 and #72043 do not supersede this request.

## Best Possible Solution

Keep this open for maintainer product work. A complete fix should define the default compaction/reset visibility contract across supported chat surfaces, decide which indicators are durable enough, add safe assistant-side disclosure and re-grounding behavior after compaction, offer reload/review of prior context when accuracy depends on it, and update docs/tests around `agents.defaults.compaction.notifyUser`, post-compaction recovery semantics, and customer-facing suppression controls.
