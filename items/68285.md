## Summary

Keep open. Current main has relevant Google Gemini CLI and Matrix lifecycle changes, but it does not clearly implement the reporter's confirmed workaround: the backend now passes `--skip-trust`, while the reporter said tools and memory started working with `--yolo`. The separate Matrix CPU saturation symptom also has not been reproduced or tied to a specific current-main fix, and the issue is recent with a concrete environment.

## Best Possible Solution

Keep this issue open and reproduce it on current main or the latest release with Matrix plus `google-gemini-cli`, preferably on the documented Node 24 path and, if relevant, Node 25 as reported. If tools or memory still require noninteractive Gemini approval, make an explicit backend policy change in `extensions/google/cli-backend.ts` with tests in `src/agents/cli-backends.test.ts` and docs updates. Separately, capture Matrix logs or a CPU profile around `extensions/matrix/src/matrix/monitor/*` if the CPU saturation still occurs.
