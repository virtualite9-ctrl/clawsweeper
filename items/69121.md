## Summary

Keep #69121 open. Current main has pre-restart deferral for active work and pending replies, but it does not implement the PR’s second-stage `server.close()` drain or chat-run abort behavior. The PR is an active, focused gateway restart correctness fix, and I did not find a landed/superseding implementation on main.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to review/land a narrow gateway implementation that passes the remaining restart drain budget into `server.close()`, drains pending reply dispatchers and active chat runs, aborts any still-active chat runs when the bounded budget is exhausted, and keeps unrelated qa-lab changes out unless they are directly required by the regression scenario.
