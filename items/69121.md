## Summary

Keep #69121 open. Current main has pre-restart deferral, including pending reply counts, but it does not implement the PR's second-stage bounded drain inside `server.close()` or the active chat-run abort behavior described by the PR. The provided PR discussion also shows this is an active, focused gateway restart correctness fix rather than an obsolete or superseded change.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is a narrow gateway fix that passes the remaining restart drain budget into `server.close()`, drains pending reply dispatchers and active chat runs before sockets/channels are torn down, aborts still-active chat runs when the bounded budget is exhausted, and keeps unrelated qa-lab changes out unless they are required for the regression scenario.
