## Summary

Keep open. Current main still does not expose Brave Goggles in the active provider-owned Brave web_search implementation. This PR is a useful implementation candidate, but it targets the old core web-search file rather than the current `extensions/brave` provider boundary. Security review found no unexplained CI, dependency, lockfile, install, release, or secret-handling changes in the PR diff; the main risk is stale integration shape.

## Best Possible Solution

Keep this PR open and ask for a rebase/port into `extensions/brave`: add optional `goggles` to the Brave provider schema and shared TypeBox schema, pass it through `URLSearchParams` in both Brave web search and LLM Context runtime calls, include it in the search cache key without lossy normalization or missing-value collisions, and update `docs/tools/brave-search.md`, `docs/tools/web.md`, and focused Brave provider tests. No new core/plugin API appears necessary because provider-owned `createTool(...)` already supplies the schema and execute path.
