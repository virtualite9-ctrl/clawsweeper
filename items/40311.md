## Summary

Keep open. Current main still lacks Brave Goggles support, and the PR remains a useful implementation candidate, but it is stale against the current provider-owned Brave plugin architecture. The best path is to port the idea from the old core `web_search` implementation into `extensions/brave` with focused provider tests and docs.

## Best Possible Solution

Keep this PR open and ask for a rebase/port into `extensions/brave`: add optional `goggles` to the Brave provider schema, pass it via `URLSearchParams` to both Brave web search and LLM Context calls, include it in the Brave cache key without lossy normalization or missing-value collisions, and update `docs/tools/brave-search.md`, `docs/tools/web.md`, and focused Brave provider tests. No new core/plugin API appears necessary because provider-owned tool definitions already supply their own schema and execute function.
