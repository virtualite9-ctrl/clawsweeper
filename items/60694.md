## Summary

Keep open. Current `main` still treats tool policy as tool-level: denying `write` only aliases `apply_patch`, not `exec`, and the docs still tell operators to deny `exec` explicitly for read-only agents. That means the reported bypass is still a real gap relative to the issue's expected behavior.

## Evidence

- **Tool-policy matcher only maps `write` to `apply_patch`:** The policy matcher blocks `apply_patch` when `write` is denylisted, but there is no equivalent mapping from `write` denial to `exec`. If `exec` remains allowed, the matcher will still allow it.
  - file: [src/agents/tool-policy-match.ts:15](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/src/agents/tool-policy-match.ts#L15)
  - command: `nl -ba src/agents/tool-policy-match.ts | sed -n '1,80p'`
  - sha: [6bc0dc8fb671](https://github.com/openclaw/openclaw/commit/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440)
- **Config types scope `workspaceOnly` to FS tools, not `exec`:** `tools.exec.applyPatch.workspaceOnly` is documented as affecting only `apply_patch`, and `tools.fs.workspaceOnly` is documented as restricting `read`/`write`/`edit`/`apply_patch`. There is no built-in exec write restriction in the config contract shown here.
  - file: [src/config/types.tools.ts:283](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/src/config/types.tools.ts#L283)

- **Security docs require denying `exec` separately for read-only profiles:** The user-facing security guide says a read-only profile should combine read-only workspace access with allow/deny lists that block `write`, `edit`, `apply_patch`, `exec`, `process`, etc. That is consistent with the report: current guidance does not claim that denying `write` also constrains `exec`.
  - file: [docs/gateway/security/index.md:1032](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/docs/gateway/security/index.md#L1032)

  - file: [docs/tools/exec.md:155](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/docs/tools/exec.md#L155)
