## Summary

Keep open. Current main still builds the native subagent request with the task text in both the extra system prompt and the first user message. The related PRs #72028 and #72053 are plausible fix attempts, but they are open/unmerged in the provided context, and #72026 is closed unmerged. The ClawHub skill linked in the issue comment is a useful workaround, not a reason to close a current core behavior bug.

## Best Possible Solution

Keep #72019 open until one canonical PR lands. The best fix is to make native `runtime: "subagent"` prompt assembly single-source the task text, add a focused regression test that inspects both `extraSystemPrompt` and the first gateway `message`, and cover multiline task preservation so token dedupe does not lose formatting.
