## Summary

Keep open. Current main still structurally sends the same `sessions_spawn` task in both places reported: `buildSubagentSystemPrompt` embeds `params.task` in the subagent system prompt, and `spawnSubagentDirect` also sends `[Subagent Task]: ${task}` as the first child user message. The linked PRs are useful fix attempts, but they are open and not shipped on main.

## Best Possible Solution

Keep #72019 open until a narrow core fix lands. The fix should ensure the long subagent task appears exactly once in the assembled first child request, preserve multiline task text, and add regression coverage for the subagent system prompt plus initial child user message. The existing linked PRs can inform the implementation, with #72053 notably covering multiline preservation.
