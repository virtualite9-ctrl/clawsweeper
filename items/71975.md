## Summary

Keep #71975 open. Current main does not provide first-class LTX/Lightricks support: the user-facing video docs and fal provider docs do not mention LTX, the fal video provider's advertised model list omits LTX model IDs, and `video_generate action=list` is driven by provider-declared `models`, so LTX is not discoverable through the first-class path requested. The direct Lightricks provider and Comfy workflow parts may fit ClawHub/plugin scope, but the lowest-effort request to document and expose LTX through the bundled fal video provider remains an actionable repo change.

## Best Possible Solution

Keep the issue open for a focused provider-owner/docs decision. The minimal repo path is to add LTX model IDs and recommended text/image-to-video defaults to the bundled fal docs, and add LTX entries to the fal video provider `models`/capability metadata if OpenClaw wants `video_generate action=list` discoverability. A dedicated `lightricks` provider or reusable Comfy workflow template should stay ClawHub/third-party unless a maintainer explicitly accepts bundled maintenance.
