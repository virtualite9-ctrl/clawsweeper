## Summary

Keep #12512 open. Current main still exposes skills through model-facing prompt instructions that tell the agent to read and follow selected SKILL.md files. Existing controls are eligibility, visibility, allowlists, size limits, installer scanning, and plugin prompt-hook policy; I did not find the requested SKILL.md instruction isolation, role locking, role/memory/tool capability metadata, or behavioral monitoring. The existing maintainer/Codex issue comment also explicitly concluded this should remain open.

## Best Possible Solution

Keep #12512 open as core security hardening. Define and document a SKILL.md instruction-isolation and precedence model, add enforceable per-skill capability metadata for role, memory, and tool influence, apply it through prompt assembly/tool policy/memory/sandbox behavior, and add malicious-skill regression tests. ClawHub moderation and scanners can supplement that boundary but should not replace it.
