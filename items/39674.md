## Summary

Current `main` already separates heartbeat run context from heartbeat delivery. `agents.defaults.heartbeat.target` controls where outbound heartbeat messages are sent, while heartbeat runs stay on the agent main session unless `agents.defaults.heartbeat.session` is set. The report's expectation that `target: "telegram"` should move the heartbeat run itself into the Telegram session does not match the shipped behavior on current `main` or the latest release tag checked.

## Evidence

- **Docs: target vs session are separate:** `docs/gateway/heartbeat.md:253` says heartbeats run in the agent main session by default; `docs/gateway/heartbeat.md:256` says `session` only affects run context and delivery is controlled by `target` and `to`.
  - file: [docs/gateway/heartbeat.md:253](https://github.com/openclaw/openclaw/blob/22f23fa5ab6f625835bfc3dbaba73c200775027c/docs/gateway/heartbeat.md#L253)

  - file: [src/config/types.agent-defaults.ts:346](https://github.com/openclaw/openclaw/blob/22f23fa5ab6f625835bfc3dbaba73c200775027c/src/config/types.agent-defaults.ts#L346)
