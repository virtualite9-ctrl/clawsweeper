## Summary

Keep #44108 open. Current main still implements the exact whitelist the issue asks to remove: `agents.files.list` returns only bootstrap plus memory files, `agents.files.get/set` reject any name outside `ALLOWED_FILE_NAMES`, the protocol has no scope/path fields, and the Control UI renders the file panel as fixed "Core Files" tabs rather than a workspace/system browser. The related ClawSweeper title-search matches are unrelated browser/model issues and do not supersede this request.

## Best Possible Solution

Implement this as an explicit gateway protocol and Control UI feature: keep the default bootstrap/core-files view, add a workspace-scoped recursive browser using the existing safe in-root file helpers, extend schemas/types/tests/docs for scope/path semantics, and treat any system-scope browser as a separate admin-gated capability with read/write safety rules and clear authorization boundaries.
