## Summary

Keep #7717 open. Current main still lacks a Discord guild/channel role-mention trigger surface: config and strict schema have no `triggerOnRoles`-style field, existing `roles` fields are sender allowlists/routing inputs, and preflight does not compare mentioned role IDs against configured trigger role IDs before mention gating skips role-only messages.

## Best Possible Solution

Keep #7717 open and implement it in the bundled Discord plugin as an additive guild/channel config surface, for example `triggerOnRoles?: string[]`. Wire it through config types, strict Zod schema, generated metadata/help, public docs, and preflight. Preflight should compare Discord `message.mentionedRoles` / `mention_roles` against configured role IDs and treat a match as a wake mention while preserving existing `requireMention`, `ignoreOtherMentions`, @everyone/@here, reply-to-bot, sender `users`/`roles` allowlist, and routing behavior. Add focused config/schema tests plus preflight tests for matched and unmatched role mentions.
