## Summary

Keep #7717 open. Current main still does not implement a Discord role-mention wake trigger: role mention metadata is preserved and used for generic mention/other-mention handling, and Discord sender role allowlists exist, but there is no `triggerOnRoles`-style config surface or role-ID comparison that turns a selected mentioned role into `wasMentioned`.

## Best Possible Solution

Keep #7717 open. Implement this in the bundled Discord plugin as an additive guild/channel config surface such as `triggerOnRoles?: string[]`, then wire it through config types, strict zod schema, generated metadata/help, docs, and preflight. Preflight should compare Discord `mentionedRoles`/`mention_roles` against configured role IDs and treat a match as a wake mention while preserving current `requireMention`, `ignoreOtherMentions`, @everyone/@here, explicit bot mention, reply-to-bot, sender user/role allowlist, and routing behavior. Add focused schema/metadata tests plus preflight tests for matched and unmatched role mentions.
