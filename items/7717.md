## Summary

Keep #7717 open. Current main still lacks a Discord guild/channel role-mention trigger: role mention metadata is retained only as generic mention metadata, while config types, strict schema, generated metadata/docs, and preflight do not expose or evaluate a configured list of role IDs that should wake the bot.

## Best Possible Solution

Keep #7717 open and implement it in the bundled Discord plugin as an additive guild/channel config surface such as triggerOnRoles?: string[]. Wire it through config types, strict Zod schema, generated metadata/help, docs, and preflight. Preflight should compare message.mentionedRoles / mention_roles against configured role IDs and treat a match as a wake mention while preserving current requireMention, ignoreOtherMentions, @everyone/@here, explicit bot mention, reply-to-bot, sender user/role allowlist, and routing behavior. Add focused schema/metadata tests plus preflight tests for matched and unmatched role mentions.
