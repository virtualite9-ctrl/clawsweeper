## Summary

Current main still has the reported overwrite gap. The Onboard/custom path can write replacement `baseUrl`/`apiKey` into config, but merge-mode `models.json` regeneration still preserves existing non-empty runtime `apiKey` and same-surface `baseUrl` values, so stale agent state can mask a deliberate same-provider edit. The issue discussion also explicitly kept this broader persistence issue open after the narrower #39103 baseUrl precedence fix.

## Best Possible Solution

Keep this issue open. The best fix is an explicit Onboard/provider-auth update path where deliberate same-provider edits persist new `apiKey` and `baseUrl` through `openclaw.json`, merge-mode `models.json` regeneration, and auth-profile handling, while preserving existing runtime values only when the new config truly omits those fields. Add regression coverage for same-provider plaintext edits, SecretRef-managed credentials, baseUrl changes with the same API surface, and auth-profile interactions.
