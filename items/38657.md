## Summary

Current main still preserves existing generated `models.json` provider values in merge mode, so the broader Onboard overwrite/persistence bug remains open. The related #39103 discussion fixed a narrower stale `baseUrl` case when the API surface changes, but same-surface `baseUrl` edits and non-marker plaintext `apiKey` values can still be masked by existing agent state, and the docs/tests still lock that behavior.

## Best Possible Solution

Keep #38657 open. The best fix is an explicit Onboard/provider-auth update path that distinguishes deliberate same-provider edits from omitted fields: new `apiKey` and `baseUrl` values should persist through `openclaw.json`, regenerated `agents/*/agent/models.json`, and auth-profile handling, while existing runtime values are preserved only when the new config truly omits those fields or SecretRef/auth-profile ownership requires it. Add regression coverage for Moonshot .ai/.cn base URL edits, same-API-surface custom base URL changes, plaintext key replacement, SecretRef-managed credentials, and auth-profile precedence.
