## Summary

Current main still has the reported Onboard overwrite gap. The Onboard/provider setup paths can place a new baseUrl/apiKey into config, but merge-mode models.json regeneration still preserves existing non-empty runtime apiKey values and same-surface baseUrl values from agents/*/agent/models.json. The issue discussion already kept this broader persistence bug open after the narrower #39103 baseUrl work, and current tests/schema help still document/assert preservation rather than overwrite.

## Best Possible Solution

Keep this issue open. The best fix is an explicit Onboard/provider-auth update path where deliberate same-provider edits persist new apiKey and baseUrl through openclaw.json, regenerated models.json, and auth-profile handling, while preserving existing runtime values only when the new config truly omits those fields. Add regression coverage for Moonshot/CN baseUrl changes, same-provider plaintext key edits, SecretRef-managed credentials, same API-surface baseUrl changes, and auth-profile interactions.
