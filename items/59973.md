## Summary

Keep open. Current main still has the reported iMessage text-dedup gap: the sent-message text fallback TTL is 4 seconds, text normalization only trims CRLF/whitespace, and existing tests explicitly preserve a miss after 4-5 seconds. The related #62191 PR tracks the U+FFFD/control-prefix part, but it does not cover the still-missing delayed text-fallback window on this checkout; #59980 was closed unmerged.

## Best Possible Solution

Keep #59973 open or explicitly link it to a complete canonical fix. The implementation should update `extensions/imessage/src/monitor/echo-cache.ts` to handle delayed reflected text safely, strip leading replacement/control garbage before text-key comparison, and add focused regression tests for 5-10s delayed echoes plus corrupted-prefix reflected copies. Coordinate with #62191 for the prefix-normalization piece, but do not treat it as complete unless the TTL/delayed-reflection path is also addressed or proven obsolete by message-id matching.
