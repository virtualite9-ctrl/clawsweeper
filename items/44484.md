## Summary

Keep #44484 open. Current main has added meaningful effective-tool visibility through `tools.effective`, `/tools`, and the Control UI, but it still does not implement strict declared-vs-effective parity, a declared/effective/removed-tools diff, or a general structured clipping reason trace. Related #59898 is open and explicitly scoped to prompt truthfulness for no-tool and hosted-tool-only sessions, not the broader strict inheritance and diagnostics requested here.

## Best Possible Solution

Keep the issue open and narrow it into a concrete design: extend the existing `tools.effective`/session inspection path to include declared policy, effective tools, removed tools, and removal source metadata; decide whether strict inheritance is a supported mode or explicitly unsupported; and coordinate prompt-truth fixes like #59898 as partial work rather than treating them as the complete solution.
