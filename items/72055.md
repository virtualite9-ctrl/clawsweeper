## Summary

Keep PR #72055 open. Current main still does not expose a Bonjour plugin config key for per-instance mDNS service names: the bundled plugin always derives the advertised `instanceName` from `ctx.machineDisplayName`, the Bonjour manifest rejects unknown config properties, and the docs do not document the requested key. The PR remains a focused implementation path for the still-missing behavior, but it likely needs a rebase over newer Bonjour handler wiring on main.

## Best Possible Solution

Keep #72055 open as the active implementation path for #54467's remaining Bonjour per-instance service-name config. The best path is to rebase onto current main, preserve the current `registerUncaughtExceptionHandler` and `registerUnhandledRejectionHandler` wiring, keep the config under `plugins.entries.bonjour.config.instanceName`, and require the Bonjour extension tests plus the changed gate before merge.
