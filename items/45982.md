## Summary

Keep #45982 open. Current main still sends memory-lancedb embeddings through the OpenAI client without `encoding_format: "float"`, while the current OpenAI SDK source still defaults omitted embedding `encoding_format` to `base64`. The related fix PRs linked from the issue timeline are closed and unmerged, and their change is not present in current main.

## Best Possible Solution

Implement the small memory-lancedb fix on main: add `encoding_format: "float"` to the OpenAI embeddings request type and payload in `extensions/memory-lancedb/index.ts`, update the colocated tests to assert it on recall/store/capture paths, and keep related `dimensions` compatibility decisions scoped to the existing follow-up around dimension forwarding.
