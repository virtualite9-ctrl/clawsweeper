## Summary

Current `main` already implements the reported fix: `timeout` is allowed for cooldown probes and transient probe slots, the auth controller honors transient cooldown probes when all profiles are cooling down, and the shipped `v2026.4.23` release contains the same logic plus a regression test covering `timeout`.

## Evidence

- **Timeout is now in both cooldown-probe allowlists:** `shouldAllowCooldownProbeForReason()` and `shouldUseTransientCooldownProbeSlot()` both return true for `"timeout"` on current `main`.
  - file: [src/agents/failover-policy.ts:3](https://github.com/openclaw/openclaw/blob/5b8bd6371c666a5de43c82305e0315771e8e49d5/src/agents/failover-policy.ts#L3)
  - sha: [5b8bd6371c66](https://github.com/openclaw/openclaw/commit/5b8bd6371c666a5de43c82305e0315771e8e49d5)
- **Probe flag is actually consumed by auth-profile rotation:** When all auto profiles are in cooldown, the auth controller checks `shouldAllowCooldownProbeForReason(unavailableReason)` and performs one transient cooldown probe instead of skipping every cooldowned profile.
  - file: [src/agents/pi-embedded-runner/run/auth-controller.ts:500](https://github.com/openclaw/openclaw/blob/5b8bd6371c666a5de43c82305e0315771e8e49d5/src/agents/pi-embedded-runner/run/auth-controller.ts#L500)
