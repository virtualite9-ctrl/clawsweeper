## Summary

Keep #39022 open. Current main still supports persistent `/model` session overrides, but does not implement `/model <name> --turns N`, warning thresholds, remaining-turn accounting, automatic revert to the default model, or `models.limits` config.

## Best Possible Solution

Keep the issue open for maintainer product judgment or core implementation. A complete solution likely belongs in auto-reply model directive parsing, session state, model selection/persistence, and config/docs: parse `/model <ref> --turns N`, persist temporary-budget fields, decrement on completed turns, warn near expiry, revert to the session default through the existing model override helper, add strict config schema/docs for any accepted `models.limits` shape, and cover parsing, persistence, warning, and reversion with focused tests.
