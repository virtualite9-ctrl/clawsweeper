## Summary

Keep #39022 open. Current main still supports persistent `/model` session overrides, but I found no implementation of `/model <name> --turns N`, turn countdown state, warning/revert notifications, or a `models.limits`/per-model `maxTurns` config contract for model selection.

## Best Possible Solution

Keep the issue open for maintainer product design and likely core implementation. A complete fix should extend `/model` parsing and command metadata for `--turns`, persist remaining-turn/revert/warning state in the session store, decrement after completed turns, send the warning and revert notifications, clear the override through the existing model-override helper, add a validated per-model config contract if accepted, and cover parsing, persistence, warning, auto-revert, and config behavior with focused tests. If maintainers prefer a plugin route, first define a stable plugin API for safe model-selection interception and per-turn session accounting.
