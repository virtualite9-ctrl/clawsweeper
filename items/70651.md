## Summary

Keep #70651 open. Current main still routes Feishu markdown/table outbound text through interactive-card send paths without a 230099/11310 table-limit classifier or plain-text fallback, and the PR is a focused attempt to fix the direct outbound path linked from #69789. It is not implemented on main and is not cleanly superseded by #50429, which targets the reply-dispatcher surface instead.

## Best Possible Solution

Keep the PR open and ask for it to be finished or folded into a broader Feishu table-limit fix. The best path is extension-owned: add a small tested card-limit error classifier in `extensions/feishu`, use it in the direct outbound card paths, preserve reply/thread routing when degrading to `sendMessageFeishu()`, force plain text after a recognized card-table failure, and coordinate with #43690/#50429 for reply-dispatcher and streaming-card coverage.
