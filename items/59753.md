## Summary

Keep open. Current `main` still lacks a Feishu WebSocket liveness/watchdog path, still delegates reconnect behavior to the upstream Lark SDK, and still reports `channels status --probe` from a separate ping endpoint that can say `works` even when the WebSocket transport is stuck.

## Evidence

- **Issue evidence:** The issue body includes sustained reconnect-loop logs (`unable to connect to the server after trying ...`) and says recovery requires manual `openclaw gateway restart`. There are no follow-up comments or timeline events contradicting that report.
- **WebSocket monitor has no recovery path:** `monitorWebSocket` creates one SDK `WSClient`, starts it, and then only resolves on abort. There is no OpenClaw-side cap on reconnect attempts, no stall watchdog, and no forced client rebuild/relogin path when the SDK keeps retrying internally.
  - file: [extensions/feishu/src/monitor.transport.ts:85](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/extensions/feishu/src/monitor.transport.ts#L85)
  - sha: [ec3dbd22a4de](https://github.com/openclaw/openclaw/commit/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46)
- **Reconnect policy is left to upstream SDK defaults:** `createFeishuWSClient` constructs `new feishuClientSdk.WSClient(...)` with app/domain/logger/agent only. No retry ceiling, backoff override, or liveness hook is configured by OpenClaw here.
  - file: [extensions/feishu/src/client.ts:222](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/extensions/feishu/src/client.ts#L222)
  - sha: [ec3dbd22a4de](https://github.com/openclaw/openclaw/commit/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46)
- **`status --probe` is a credential ping, not a WebSocket health check:** Feishu probing calls `/open-apis/bot/v1/openclaw_bot/ping`, and the CLI prints `works` whenever `probe.ok` is true. The CLI docs also describe `--probe` as running per-account probe/audit checks. That means a healthy ping can still mask a wedged WebSocket session, matching the reporter's complaint.
  - file: [extensions/feishu/src/probe.ts:51](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/extensions/feishu/src/probe.ts#L51)
  - sha: [ec3dbd22a4de](https://github.com/openclaw/openclaw/commit/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46)
- **Feishu status wiring does not add transport-liveness fields:** The Feishu channel status adapter wires `probeAccount` to `probeFeishu(account)` and only stores `port` as Feishu-specific runtime status. It does not add reconnect counters, last transport activity, or health-state logic for the WebSocket path.
  - file: [extensions/feishu/src/channel.ts:1182](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/extensions/feishu/src/channel.ts#L1182)
  - sha: [ec3dbd22a4de](https://github.com/openclaw/openclaw/commit/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46)
- **No Feishu watchdog/tuning surface exists on main:** The Feishu config schema exposes fields like `webhookPort`, `httpTimeoutMs`, `heartbeat`, `typingIndicator`, and `resolveSenderNames`, but no reconnect/watchdog field. By contrast Telegram has an explicit `pollingStallThresholdMs` watchdog setting documented and typed, which highlights the missing Feishu equivalent.
  - file: [extensions/feishu/src/config-schema.ts:155](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/extensions/feishu/src/config-schema.ts#L155)
  - sha: [ec3dbd22a4de](https://github.com/openclaw/openclaw/commit/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46)
- **Tests only cover cleanup, not stuck reconnect recovery:** Current Feishu WebSocket tests verify abort/cleanup behavior only. I did not find a test that exercises reconnect exhaustion, transport stall detection, or automatic recovery after prolonged disconnect.
  - file: [extensions/feishu/src/monitor.cleanup.test.ts:45](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/extensions/feishu/src/monitor.cleanup.test.ts#L45)
  - sha: [ec3dbd22a4de](https://github.com/openclaw/openclaw/commit/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46)
