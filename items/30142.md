## Summary

Keep this PR open. Current main only partially supersedes it: Discord and Slack now have sendPayload implementations, but Google Chat, Mattermost, MS Teams, and Synology Chat still do not expose sendPayload on their outbound adapters.

## Best Possible Solution

Keep this PR open for maintainer action, but require a rebase/narrowing before merge. The best path is to drop the already-shipped Discord and Slack changes, implement sendPayload only for Google Chat, Mattermost, MS Teams, and Synology Chat using current shared payload helper patterns where appropriate, and keep focused adapter tests for text-only, media-only, multiple media URLs, empty/channelData-only payloads, and configured chunk behavior.
