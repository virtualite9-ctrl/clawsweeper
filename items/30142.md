## Summary

Keep #30142 open. Current main still has the core adapter gap this PR targets: Discord and Slack now expose sendPayload, but Google Chat, Mattermost, MS Teams, and Synology Chat still do not. The PR is stale and should be reworked, but closing it would lose still-useful bundled channel work.

## Best Possible Solution

Keep #30142 open, or replace it with a narrower follow-up that adds sendPayload only for the still-missing bundled adapters: Google Chat, Mattermost, MS Teams, and Synology Chat. Reuse the current shared sendTextMediaPayload/sendPayload helper patterns, leave the shipped Discord and Slack implementations alone, and add focused adapter tests for text-only, media-only, mixed mediaUrls, structured/channelData payloads, and empty unsendable payload behavior.
