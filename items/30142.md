## Summary

Keep #30142 open. Current main only partially supersedes the PR: Discord and Slack already have shipped sendPayload implementations, but Google Chat, Mattermost, MS Teams, and Synology Chat still lack sendPayload on their bundled outbound adapters. The PR should not merge as-is, but the remaining adapter work is still real and not fully covered by a narrower canonical open item in the provided context.

## Best Possible Solution

Keep #30142 open for now, or replace it with a narrower PR that targets only the still-missing bundled adapters: Google Chat, Mattermost, MS Teams, and Synology Chat. Reuse the current shared sendPayload/sendTextMediaPayload helper patterns, leave the shipped Discord and Slack #30144 implementations alone, and add adapter-level payload contract tests plus explicit handling for structured-only payloads and mid-loop cancellation expectations.
