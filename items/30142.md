## Summary

Keep this PR open. Current main has newer sendPayload implementations for Discord and Slack, but Google Chat, Mattermost, MS Teams, and Synology Chat still do not expose sendPayload, while the delivery path only uses the unified payload path when an adapter actually provides it. The PR is stale and should not merge as-is, but the remaining adapter work is still real.

## Best Possible Solution

Keep this PR open for maintainer action, but require a rebase and narrowing before merge. The best path is to drop the already-shipped Discord and Slack pieces, implement sendPayload only for Google Chat, Mattermost, MS Teams, and Synology Chat using current shared payload helpers and contract-test patterns, and keep focused coverage for text-only, media-only, mixed media URLs, empty/channelData-only payloads, chunk behavior, and delivery fallback compatibility.
