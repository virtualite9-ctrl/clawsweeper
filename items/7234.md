## Summary

Keep #7234 open. Current main still exposes Discord message actions as a single coarse `channels.discord.actions.messages` boolean in types, validation, discovery, docs, and runtime enforcement. The requested ability to allow send/read/edit while disabling delete remains missing in the bundled Discord plugin surface.

## Best Possible Solution

Keep #7234 open. The best path is a backward-compatible Discord gate design, such as `channels.discord.actions.messages: boolean | { send?: boolean; read?: boolean; edit?: boolean; delete?: boolean }` or equivalent top-level Discord gates, then wire it through config types, zod/generated schema, account gate resolution, discovery, runtime checks, docs, and focused Discord tests. This belongs in the bundled Discord plugin/config surface because the official Discord integration currently exposes and enforces the coarse gate.
