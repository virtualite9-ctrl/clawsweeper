## Summary

Keep #7234 open. Current main still exposes Discord message actions through one coarse `channels.discord.actions.messages` boolean, and the requested independent send/read/edit/delete gates are not implemented. The runtime also still checks that same `messages` gate for send, read, edit, and delete paths.

## Best Possible Solution

Keep #7234 open. Implement a backward-compatible Discord gate design, for example `channels.discord.actions.messages: boolean | { send?: boolean; read?: boolean; edit?: boolean; delete?: boolean }` or equivalent top-level gates, then wire it through config types, strict zod schema, generated metadata, account gate resolution, discovery, runtime checks, docs, and focused Discord tests. Preserve existing boolean behavior while allowing deployments to disable destructive delete without disabling send/read/edit.
