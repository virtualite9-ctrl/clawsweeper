## Summary

Keep #7234 open. Current main still models Discord `actions.messages` as a single boolean and uses that one gate for read, send, edit, delete, and fetch message actions. The requested backward-compatible granular gates remain missing in config types, validation, action discovery, runtime checks, docs, and tests.

## Best Possible Solution

Keep #7234 open. Implement a backward-compatible Discord gate shape such as `channels.discord.actions.messages: boolean | { send?: boolean; read?: boolean; edit?: boolean; delete?: boolean }` or equivalent separate top-level gates, then update config types, zod schema, generated metadata/docs, action discovery, runtime checks, and focused Discord tests while preserving existing boolean behavior.
