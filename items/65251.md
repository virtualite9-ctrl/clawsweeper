## Summary

Keep open. Current main has explicit bundle-backed LSP support, but the requested zero-configuration detection, server discovery, install/fix flow, dynamic per-workspace configuration, and LSP status dashboard are still not implemented or exposed through a documented plugin API.

## Best Possible Solution

Keep this issue open for product/API design. A conservative path would add a safe project detection layer, discover already-installed language servers without surprising installs, expose explicit opt-in install/fix actions, support user overrides, and surface LSP health through CLI/UI diagnostics. If maintainers want the language-specific logic outside core, first document or add a plugin/bundle API for dynamic per-workspace LSP server configuration.
