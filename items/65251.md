## Summary

Current main still only supports explicit bundle-declared LSP defaults, not the requested zero-configuration project scanning, PATH/server discovery, install/fix flow, dynamic per-workspace configuration, update notices, or LSP status dashboard. The related #65252 is adjacent diagnostics/dashboard work and remains open, but it does not supersede this auto-detection/setup proposal.

## Best Possible Solution

Keep this issue open for product/API design. A conservative path would first add safe, non-mutating project detection and discovery of already-installed language servers, then expose explicit opt-in install/fix actions, user overrides, and visible LSP health/status. If maintainers want the language-specific logic outside core, first document or add a plugin/bundle API for dynamic per-workspace LSP server registration and configuration.
