## Summary

Current main still formats config.patch restart sentinel notices as `Gateway restart config-patch ok (config.patch)` when a note is present, so the requested wording change is still actionable and should not be closed.

## Best Possible Solution

Keep the issue open. Update the restart sentinel formatting for config.patch/config.apply restart-required notices to say something like `Gateway restart required (config.patch)` while preserving the note on following lines, then add focused restart-sentinel tests for the successful config.patch notice path.
