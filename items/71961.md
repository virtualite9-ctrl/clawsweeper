## Summary

Keep open. Current main still lacks the PR's central behavior: embedded `compaction_start`/`compaction_end` handling only writes debug/retry logs, does not pass the event reason into the start handler, and docs/tests do not cover normal info-level Gateway visibility. The provided PR discussion is also active: Greptile judged the change additive/safe, and the author addressed the only bot review concern by hoisting the lifecycle logging test.

## Best Possible Solution

Keep PR #71961 open for maintainer review. The best path is to merge or adapt the focused implementation that adds info-level embedded compaction start/end logs with structured metadata, forwards the compaction event reason, documents the visible Gateway log lines, and keeps the focused handler regression test.
