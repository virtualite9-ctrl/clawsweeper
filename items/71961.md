## Summary

Keep this PR open. Current main still has the reported gap: embedded compaction start is only logged at debug level, compaction completion only logs the retry path at debug level, the start event reason is not passed into the handler, the embedded subscribe logger type has no info-level method, and the docs/tests do not yet cover normal Gateway log visibility. The provided PR context shows an active, focused contributor bug-fix PR with a test/docs update and a bot review concern already addressed, so it remains a valid implementation candidate rather than cleanup-close material.

## Best Possible Solution

Keep the PR open for maintainer review and either land this focused fix or an equivalent patch that adds info-level embedded compaction start/end logging, passes compaction_start metadata through the handler, documents Gateway log visibility, and keeps the new regression coverage.
