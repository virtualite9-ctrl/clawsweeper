## Summary

Keep #71961 open. Current main still lacks the proposed info-level Gateway log lines for embedded compaction lifecycle events: `compaction_start` is only logged at debug, `compaction_end` only logs the retry branch at debug, docs do not mention Gateway log visibility, and current tests only cover compaction count reconciliation. The PR is a narrow, recent bug-fix/docs/test change with an addressed bot review comment, so it remains useful maintainer-review material rather than cleanup-closeable.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land or refine the narrow handler change that passes the compaction event into `handleCompactionStart`, exposes/uses `info` logging for normal Gateway visibility with a debug fallback for minimal test contexts, and keeps the focused test plus docs aligned. If maintainers want different wording, adjust the log strings/metadata rather than closing the item.
