## Summary

The reported macOS sleep/wake failure mode does not match current OpenClaw startup behavior. On current main, the gateway binds HTTP before post-attach network/sidecar work runs, and startup-pending state is surfaced through readiness instead of leaving port 18789 unopened. The same listen-before-sidecars ordering is already present in the latest shipped release, so the issue is obsolete against current code/releases.

## Evidence

- **Current main binds the gateway before post-attach startup work:** `startGatewayServer()` attaches WS handlers, then calls `await startListening()`, and only after that enters `startGatewayPostAttachRuntime()`. A stalled post-attach network task therefore should not prevent the gateway from opening its port.
