## Summary

Current main still has the plugin-sdk declaration boundary gap described by #72108. `tsconfig.plugin-sdk.dts.json` includes `src/plugin-sdk/**/*.ts` but only excludes `src/**/*.test.ts`, so `src/plugin-sdk/qa-runtime.test-helpers.ts` remains in the dts input set. That helper imports Vitest and exports `makeQaRuntimeSurface()` with `vi.fn()`-derived inferred return types, matching the PR's reported TS2883 build failure. The PR is narrow and still actionable.

## Best Possible Solution

Keep #72108 open and merge it or independently apply the same one-line `tsconfig.plugin-sdk.dts.json` exclusion for `src/**/*.test-helpers.ts`. Then verify `pnpm build:plugin-sdk:dts` and the normal changed/build gate. Do not close it as obsolete because current main still includes the fixture in the declaration input set.
