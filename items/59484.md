## Summary

Keep open. Current main still treats every `channels status` gateway call failure the same way, including `--probe`, then reports `Gateway not reachable` and falls back to config-only status. The PR remains a focused, relevant bug fix with a regression test; the only review note in the provided discussion is test cleanup, not an objection to the behavior.

## Best Possible Solution

Keep PR #59484 open. The best path is to rebase or apply the small probe-mode rethrow guard on current main, keep the non-probe fallback behavior unchanged, include the focused regression test with the mock cleanup, and then run the changed command/test gate before merge.
