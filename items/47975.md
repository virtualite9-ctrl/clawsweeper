## Summary

Keep open. Current `main` still has explicit codepaths that retain completed subagent session records instead of cleaning them up immediately, and I did not find strong evidence that the parent-session unresponsiveness report has been fixed on `main` after the latest release.

## Evidence

- **Spawned subagents default to retained cleanup:** `sessions_spawn` still defaults `cleanup` to `"keep"`, so one-shot subagents are not configured for immediate session deletion unless the caller opts in.
  - file: [src/agents/tools/sessions-spawn-tool.ts:192](https://github.com/openclaw/openclaw/blob/d38ed0831d7372c8505dc944aea69b5b83da1435/src/agents/tools/sessions-spawn-tool.ts#L192)
  - command: `nl -ba src/agents/tools/sessions-spawn-tool.ts | sed -n '180,330p'`
  - sha: [d38ed0831d73](https://github.com/openclaw/openclaw/commit/d38ed0831d7372c8505dc944aea69b5b83da1435)
- **Keep-mode runs are not archived immediately:** The subagent run manager leaves `archiveAtMs` unset when `cleanup === "keep"`, and the registry sweeper only removes no-archive entries after `cleanupCompletedAt` ages past the 5-minute TTL. That is not an immediate cleanup path for completed subagent sessions.
  - file: [src/agents/subagent-registry-run-manager.ts:303](https://github.com/openclaw/openclaw/blob/d38ed0831d7372c8505dc944aea69b5b83da1435/src/agents/subagent-registry-run-manager.ts#L303)

- **Sweeper only deletes retained runs after delayed TTL:** For entries without `archiveAtMs`, `sweepSubagentRuns()` waits until `cleanupCompletedAt` is older than `SESSION_RUN_TTL_MS` before deleting the run and attachments.
  - file: [src/agents/subagent-registry.ts:580](https://github.com/openclaw/openclaw/blob/d38ed0831d7372c8505dc944aea69b5b83da1435/src/agents/subagent-registry.ts#L580)
