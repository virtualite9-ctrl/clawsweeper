## Summary

Keep open. Current main still does not satisfy the central report: one-shot `sessions_spawn` defaults to keeping child sessions, keep-mode runs are not scheduled for archive/delete, `sessions.list` still exposes stored completed subagent rows, and Control UI renders that list. The April 20 issue comment also reports overlapping stale-running task/subagent symptoms still reproducing on 2026.4.15, so this is not safe to close as implemented or duplicate-only cleanup.

## Best Possible Solution

Keep #47975 open as the broad stale subagent/session responsiveness tracker. The best maintainer path is to define the intended retention contract for one-shot subagents, then implement it consistently across `sessions_spawn`, subagent registry archive/cleanup, task startup reconciliation, `sessions.list`, status panels, and Control UI. Cross-link #56516 for the narrower UI list persistence symptom and #56044/#54488 for adjacent lane/control-plane starvation, but do not close this issue until the stale completed subagent plus stale-running task behavior is fixed or intentionally documented with controls.
