## Summary

Keep open. Current main still lacks a Feishu doctor migration for legacy `channels.feishu.accounts.*.botName`, while the current Feishu account schema accepts `name` and rejects unknown account keys. The focused implementation candidate is #65384; earlier #65202 was closed unmerged.

## Best Possible Solution

Keep this issue open and review/land #65384, or land an equivalent Feishu-owned doctor compatibility migration. The fix should migrate `channels.feishu.accounts.*.botName` to `name` before strict validation, preserve existing `name`, remove the stale key, and add regression coverage for `openclaw doctor --fix` on legacy Feishu account configs.
