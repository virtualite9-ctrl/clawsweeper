## Summary

Keep open. Current main still lacks a Feishu doctor migration for legacy `channels.feishu.accounts.*.botName`, while the Feishu account schema remains strict and runtime reads only `name`. The focused open implementation candidate is #65384; #65202 was closed unmerged.

## Best Possible Solution

Keep this issue open and review/land #65384, or land an equivalent Feishu-owned doctor compatibility migration exposed through the bundled channel doctor contract or the existing doctor migration path. The fix should migrate `channels.feishu.accounts.*.botName` to `name` before strict validation, preserve an existing `name`, remove the stale key, and add regression coverage for `openclaw doctor --fix`.
