## Summary

Issue #71936 is a real current-main bug, but it is fully superseded by the already-linked open PR #71924, which is the active implementation tracker for fetching GitHub Copilot `/models` and mapping GPT-5.5 capability metadata. Current main still returns an empty Copilot catalog and falls back to synthetic defaults, so this should not be closed as implemented.

## Best Possible Solution

Close this issue as superseded by #71924 and keep review focused on that PR or an equivalent patch. The right product/code direction is provider-owned Copilot catalog discovery that fetches `/models`, maps advertised limits, vision, reasoning-effort, and supported endpoint metadata, and updates the existing Copilot catalog tests.
