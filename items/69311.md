## Summary

Keep #69311 open. Current main still has the reported false-positive path: `splitMediaFromOutput()` only excludes fenced code spans, then trims leading whitespace and treats an indented `MEDIA:` line as a directive. The linked PR #69312 directly tracks the proposed fix and has useful review/author follow-up, but it remains open and unmerged, so the issue should stay open until a fix lands on main.

## Best Possible Solution

Keep #69311 open and let PR #69312, or a maintainer-adjusted equivalent, close it when merged. The fix should add indented-code protection in `src/media/parse.ts`, preserve code indentation when real media directives are removed, align MEDIA case semantics with streaming buffering, tighten `parseFenceSpans()` closing validation if accepted, and land focused regression tests for 4-space/tab-indented code blocks.
