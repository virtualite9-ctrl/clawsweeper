## Summary

Current `main` already strips stray `HEARTBEAT_OK` replies before channel delivery through shared reply normalization and non-heartbeat outbound sanitizers, with regression tests covering drop/strip behavior. The latest shipped release commit I inspected (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`, `v2026.4.22`) also contains that logic, so the report no longer matches current code.
