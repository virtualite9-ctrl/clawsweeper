## Summary

Current main already provides plugin-scoped context-routing seams and lightweight bootstrap controls, while the repo vision explicitly says optional capability like this should usually ship via plugins/ClawHub rather than core.

## Evidence

## Risks / Open Questions

- OpenClaw does not currently ship a built-in automatic casual-chat vs task classifier in core; closing this as ClawHub assumes that policy should remain plugin-owned unless maintainers want a product-level default.
