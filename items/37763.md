## Summary

Keep #37763 open. Current main still drops resolved Slack agent identity on the draft-stream preview path, and the monitor path still forwards `agents.list[].identity.emoji` raw instead of normalizing bare aliases like `stitch-1` to Slack's documented `:stitch-1:` form. Normal/fallback reply delivery has partial identity support, but it does not cover the reported default preview path.

## Best Possible Solution

Implement the missing Slack monitor behavior in core: normalize Slack identity emoji aliases to Slack `:emoji_name:` syntax, thread `SlackSendIdentity` through `createSlackDraftStream` from `dispatchPreparedSlackMessage`, and add focused regression coverage for draft-stream identity passthrough plus bare/pre-wrapped emoji handling. If #38235/#38237 is still the active canonical follow-up, use that issue/PR as the tracking path rather than closing #37763 prematurely.
