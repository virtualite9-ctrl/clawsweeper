## Summary

Keep open. Current `main` does not include the requested built-in profiles, and the PR as proposed runs into credible security/design blockers: `safeBins` is still documented as a narrow stdin-only fast path, the current token validator only blocks path prefixes (not embedded traversal), and there is no date-specific semantic guard for positional clock-setting syntax.

## Evidence

- **Requested built-ins are not on main:** `SAFE_BIN_PROFILE_FIXTURES` on current `main` only defines built-in profiles for `jq`, `grep`, `cut`, `sort`, `uniq`, `head`, `tail`, `tr`, and `wc`; there are no built-in fixtures for `echo`, `date`, `which`, `uptime`, `ls`, `cat`, `ps`, `df`, or `du`.
  - file: [src/infra/exec-safe-bin-policy-profiles.ts:98](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/src/infra/exec-safe-bin-policy-profiles.ts#L98)
  - command: `nl -ba src/infra/exec-safe-bin-policy-profiles.ts | sed -n '1,220p'`
  - sha: [926068b14fc6](https://github.com/openclaw/openclaw/commit/926068b14fc661e915a0247e9ecb5949fb95b4f7)
- **Current contract is stdin-only:** The public docs still define `tools.exec.safeBins` as a small list of stdin-only binaries, say safe bins reject positional file args and path-like tokens, and describe the feature as a narrow fast-path for stream filters rather than a general trust list.
  - file: [docs/tools/exec-approvals-advanced.md:14](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/docs/tools/exec-approvals-advanced.md#L14)
  - command: `nl -ba docs/tools/exec-approvals-advanced.md | sed -n '10,90p'`
  - sha: [926068b14fc6](https://github.com/openclaw/openclaw/commit/926068b14fc661e915a0247e9ecb5949fb95b4f7)
- **Traversal check is prefix-only:** `isPathLikeToken()` only rejects tokens starting with `./`, `../`, `~`, `/`, or a Windows drive prefix. `consumePositionalToken()` then accepts any remaining token as a positional safe literal, so embedded traversal forms like `src/../../etc` are not screened out by the current validator.
  - file: [src/infra/exec-safe-bin-policy-validator.ts:9](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/src/infra/exec-safe-bin-policy-validator.ts#L9)

- **No date semantic hardening exists:** `validateSafeBinSemantics()` only adds bin-specific semantic checks for `jq`, `awk`-family, and `sed`-family commands. A new built-in `date` profile would currently get no extra semantic screening for positional time-setting syntax.

That means there is an existing escape hatch, but not yet a vetted built-in expansion.

## Risks / Open Questions

- Merging the PR as written would broaden `safeBins` beyond the current stdin-only contract without stronger path normalization or bin-specific semantic checks.
- The proposed `date` profile needs stricter handling of positional clock-setting syntax, not just `--set`/`-s` denial.
