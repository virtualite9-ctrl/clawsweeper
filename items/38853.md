## Summary

Keep #38853 open. Current main still has only partial mitigations: generic channel auto-restart backoff and Discord JSON 429 retries exist, but the Discord REST startup/allowlist layer still does not classify Cloudflare/Error 1015 HTML 429 responses as a cooldown condition, still preserves non-JSON response text in API errors, and still resolves the reported `/users/@me/guilds` path during allowlist handling.

## Best Possible Solution

Keep the issue open and fix it in the Discord plugin REST startup/allowlist layer: classify Cloudflare/Error 1015 and non-JSON 429 HTML as a rate-limit cooldown signal, sanitize or summarize HTML bodies before logging, honor `Retry-After` when present, apply a conservative cooldown when absent, and add regression tests for `/users/@me/guilds` and `/oauth2/applications/@me` HTML 429 responses. Treat adjacent channel restart and Discord gateway lifecycle fixes as partial mitigations, not the canonical resolution for this REST cooldown/log-bloat bug.
