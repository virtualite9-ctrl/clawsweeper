## Summary

Keep #38853 open. Current main has partial mitigations, including generic channel auto-restart backoff and Discord JSON 429 retry handling, but the specific Cloudflare/Error 1015 REST gap remains in the Discord plugin: non-JSON HTML 429 bodies can still be preserved in errors and logs, `/users/@me/guilds` allowlist resolution still uses that path, and `/oauth2/applications/@me` startup handling still treats non-ok responses as a generic application-id failure rather than a cooldown signal.

## Best Possible Solution

Keep the issue open and fix it in the Discord plugin REST startup/allowlist layer: classify Cloudflare/Error 1015 and non-JSON 429 HTML as a rate-limit cooldown signal, sanitize or summarize HTML bodies before logging, honor `Retry-After` when present, apply a conservative cooldown when absent, and add regression tests for `/users/@me/guilds` and `/oauth2/applications/@me` HTML 429 responses. Treat adjacent reconnect/full-gateway-restart issues as already handled but not canonical for this REST cooldown/log-bloat bug.
