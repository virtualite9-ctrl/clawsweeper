## Summary

Keep open. Current main still appears to have the reported hard-fail path for plugin-local config validation errors, and the directly linked fix PR #70394 remains open rather than shipped. This is a real product bug, not stale, external, or ClawHub-scoped.

## Best Possible Solution

Keep this issue open until the loader/config validation path degrades a single plugin config failure into a disabled or failed plugin entry, continues booting unaffected plugins/channels, and reports the diagnostic through `openclaw doctor`. PR #70394 or a maintainer-approved variant is the likely implementation path.
