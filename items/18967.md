## Summary

Keep #18967 open. Current main still has broad `sessions_send` plus configurable subagent allowlist escape hatches, but it does not provide the requested native parent-scoped send path with direct-parent-only, same-tree, no-chaining enforcement. The existing maintainer/Codex discussion also explicitly kept this issue open for that distinction.

## Best Possible Solution

Keep #18967 open. The best path is a narrow native `sessions_send_parent` tool, or a literal `parent` target on `sessions_send`, that resolves from stored direct-parent metadata, enforces same OpenClaw agent tree and direct-parent-only access, never auto-forwards further up the tree, and adds resolver/policy/visibility tests plus user-facing docs. The existing broad `tools.subagents.tools.allow`/`alsoAllow` path should remain a separate escape hatch, not the closure condition for this issue.
