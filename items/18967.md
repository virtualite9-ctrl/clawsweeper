## Summary

Keep #18967 open. Current main still does not provide a native parent-scoped send path for sub-agents; it has default `sessions_send` denial for sub-agents plus broad opt-in session visibility/tool-policy knobs, and existing announce/ACP parent-stream behavior does not cover the requested child-to-direct-parent question/progress tool.

## Best Possible Solution

Keep #18967 open. The best product/code path is a narrow native parent-send mechanism, either a dedicated `sessions_send_parent` tool or a literal `sessionKey: "parent"` resolver that reads direct parent metadata (`spawnedBy`/`parentSessionKey`) and enforces direct-parent-only, same-agent-tree access with no automatic forwarding further up the tree. Keep broad `sessions_send` opt-ins separate as high-trust configuration, and add resolver, policy, visibility, docs, and regression-test coverage.
