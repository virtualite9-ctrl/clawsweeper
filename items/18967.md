## Summary

Keep #18967 open. Current main still has only the broad `sessions_send` tool plus opt-in subagent policy and descendant/tree visibility; it does not implement the requested safe parent-scoped native path (`sessions_send_parent` or a literal `parent` target) with direct-parent-only, same-tree, no-chaining enforcement. The provided OWNER review comment already kept this issue open, and current source inspection at e29d3516bf05591e6007bfb8ccdaea6d76194d41 matches that assessment.

## Best Possible Solution

Keep #18967 open. The best path is a narrow native `sessions_send_parent` tool or a literal `parent` target that resolves from stored direct-parent metadata, enforces same OpenClaw agent tree and direct-parent-only access, does not auto-forward further up the tree, and adds focused resolver, policy, visibility, docs, and regression tests. The existing broad `sessions_send` opt-in can remain documented as a workaround but should not be treated as the requested safe default capability.
