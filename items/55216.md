## Summary

Current `main` still silently drops recoverable Telegram media-group failures. `processMediaGroup` continues past `MediaFetchError`/size-limit errors and also ignores `null` media results, while the single-message path sends a user-visible warning. This matches the reported bug closely enough that the issue should remain open.

## Evidence

- **Media-group recoverable failures are still skipped without user notice:** `processMediaGroup` catches recoverable media errors, logs `media group: skipping photo that failed to fetch`, and `continue`s. There is no user-facing warning before the remaining media are processed.
