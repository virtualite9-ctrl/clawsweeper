## Summary

Keep #17211 open. Current main has compaction-scoped checkpoint snapshots with list/get/branch/restore and UI recovery, but it still does not provide the requested native task-boundary checkpoint engine, checkpoint.create/resume/diff APIs, deterministic replay packets, structured done/saved/dropped/next contracts, or missing-field validation.

## Best Possible Solution

Keep #17211 open as the umbrella for native checkpoint/replay design. A good implementation would add versioned Gateway/session APIs for checkpoint.create/list/resume/diff, persist a structured task contract with done/saved/dropped/next plus validation, generate deterministic replay packets, expose diff/audit data, and add docs plus contract tests. If maintainers want this plugin-owned, first add an explicit checkpoint/context-engine API rather than relying on the current compaction provider seam.
