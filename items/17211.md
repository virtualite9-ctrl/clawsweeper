## Summary

Keep #17211 open. Current main still has compaction-scoped snapshot recovery, not the requested native task-boundary checkpoint engine with checkpoint.create/list/resume/diff, deterministic replay packets, done/saved/dropped/next contracts, diffing, and validation hooks. The existing issue discussion already contains a maintainer-side Codex review keeping it open, and current main still matches that assessment.

## Best Possible Solution

Keep #17211 open as the umbrella for native checkpoint/replay design. A good implementation should add versioned Gateway/session APIs for checkpoint.create/list/resume/diff, persist a structured task contract with done/saved/dropped/next plus validation, generate deterministic replay packets, expose diff/audit data, document the workflow, and add protocol/runtime/UI contract tests. If maintainers choose a plugin-owned direction, first add an explicit checkpoint/context-engine API rather than relying on the current compaction provider or context-engine compact seam.
