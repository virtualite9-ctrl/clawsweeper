## Summary

Keep #17211 open. Current main still has compaction checkpoint snapshot recovery, but not the requested native checkpoint engine with checkpoint.create/list/resume/diff APIs, deterministic replay packets, structured task-boundary contracts, diffing, or validation hooks. The issue also has an existing maintainer/Codex comment explicitly keeping it open as of 2026-04-26.

## Best Possible Solution

Keep #17211 open as the native checkpoint/replay umbrella. A complete solution likely belongs in versioned Gateway/session APIs and session persistence: checkpoint.create/list/resume/diff, a structured done/saved/dropped/next task contract with validation status, deterministic replay packet generation, diff/audit data, UI/docs updates, and protocol/runtime/UI contract tests. If maintainers prefer plugin ownership, first define an explicit checkpoint/context-engine API so a plugin can implement it without reaching into core internals.
