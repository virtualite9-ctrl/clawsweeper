## Summary

Current `main` still exhibits the stale skill-snapshot behavior described in #69715, so this PR is not implemented on `main`. The PR also has unresolved review feedback: its new regression test mocks `refresh.js`, while production reads the snapshot-version helpers from `refresh-state.js`, so the added assertion would not validate the fix as written. The related PR #69734 was closed as a duplicate in favor of continuing review on this PR, not because the fix landed elsewhere.

## Evidence

- **Current main still reads the snapshot version before watcher setup:** `ensureSkillSnapshot()` reads `getSkillsSnapshotVersion(workspaceDir)` before `ensureSkillsWatcher({ workspaceDir, config: cfg })`. With a persisted `skillsSnapshot.version` of `0`, that preserves the reported stale-snapshot path after restart instead of forcing a rebuild on first turn.
