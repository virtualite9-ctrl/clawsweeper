## Summary

Current main has meaningful compaction safeguards, but it does not implement the loop-aware saturation guard requested in #48238. The proposed config block, transcript-tail detector/scorer, guarded runtime decisioning, post-compaction validator, and recommend-reset path are absent from main, while the related PR stack in the provided GitHub context remains open/draft rather than merged.

## Best Possible Solution

Keep #48238 open as the umbrella for the remaining core compaction hardening. The best implementation path is to review and land a narrowed version of the existing PR stack or equivalent changes: optional disabled-by-default guard config, pure transcript-tail risk detection/scoring, guarded instruction augmentation in the existing safeguard path, post-compaction validation, and internal recommend-reset diagnostics without automatic reset or noisy user-facing output.
