## Summary

Keep #39102 open. Current main still lacks the requested `agents.list[].tools.agentToAgent.allow` requester-specific outbound override; cross-agent session access is still governed only by the global `tools.agentToAgent.allow` gate. The PR is a focused implementation path for a real missing core/config surface, not obsolete cleanup.

## Best Possible Solution

Keep the PR open or replace it with an updated implementation on current main. The best path is to add a strict `agents.list[].tools.agentToAgent.allow` type/schema/help/docs surface, evaluate global A2A participation before requester outbound overrides, share structured denial formatting across `sessions_list`, `sessions_history`, `sessions_send`, and `session_status`, prune deleted agent IDs from per-agent allowlists immutably, and add focused regression tests for fallback, deny-all `[]`, outbound-only semantics, global participation, prune immutability, sandbox visibility, model fallback, and sessionId resolution.
