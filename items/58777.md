## Summary

Keep #58777 open. Current main partially persists the actual runtime model after an agent/sub-agent run, but it does not implement the requested closed-loop effective model/provider/thinking telemetry on the sub-agent `agent` lifecycle path, and thinking is still resolved once before fallback candidates are attempted.

## Best Possible Solution

Keep the issue open and implement a scoped core fix in the `agent-command`/sub-agent execution path: emit completion/fallback lifecycle or diagnostic fields for requested vs effective provider/model/thinking, preserve the existing session runtime model persistence, and re-derive or clamp thinking per fallback candidate before the attempt starts. The existing auto-reply fallback notice machinery is a useful reference, but the fix should avoid duplicating or changing that already-covered path unnecessarily.
