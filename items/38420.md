## Summary

Keep #38420 open. Current main has addressed some audit bullets, including breadcrumbs, masked gateway-token entry, config invalid-state warning, and delete confirmation, but the broad Control UI audit is still not implemented well enough to close. Instances still lacks per-instance actions and clearer presence affordances, Sessions still repeats advanced override controls per row and lacks session-key copy affordances, Config still exposes adjacent Save/Apply/Update actions, and global keyboard/help shortcuts are still limited. The related #39780 remains a narrower Config hardening issue, not a superseding tracker for this full-dashboard audit.

## Best Possible Solution

Keep #38420 open unless a maintainer intentionally splits it. The best path is to either implement or split focused follow-ups for Instances per-instance actions and clearer presence affordances, Sessions key copy/truncation and less repetitive advanced controls, Config action semantics, dashboard keyboard/help UX, and feedback/activity improvements, while treating #39780 as only the narrower Config hardening follow-up.
