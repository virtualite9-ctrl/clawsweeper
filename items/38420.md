## Summary

Keep #38420 open. Current main has addressed several audit items, but the broad dashboard UX request is not fully implemented: Instances still only has global refresh/host reveal with generic chips and raw reason text, Sessions still repeats per-row advanced override selects and has no copy affordance for session keys, and Config still exposes adjacent Save/Apply/Update actions. The related #39780 remains a narrower Config hardening issue, not a superseding tracker for the full Control UI audit.

## Best Possible Solution

Keep #38420 open unless a maintainer intentionally splits it. Treat #39780 as a narrower Config hardening follow-up, not a replacement. The remaining work should either be implemented or split into focused issues for Instances per-instance actions and clearer presence affordances, Sessions key copy/truncation and less repetitive advanced controls, clearer Config action semantics, keyboard/help UX, and feedback/activity improvements.
