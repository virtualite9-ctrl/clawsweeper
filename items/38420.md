## Summary

Keep #38420 open. Current main has addressed some items from the broad UX audit, including breadcrumb/header context, scrollable/collapsible navigation, masked gateway-token entry, top Config Form/Raw toggle, invalid-config banner, and delete confirmation. But multiple concrete dashboard requests remain missing: Instances still has only global refresh/host reveal with generic chips and raw reason text, Sessions still repeats per-row advanced selects and lacks session-key copy affordance, and Config still presents adjacent Save/Apply/Update actions. The related #39780 is a narrower open Config hardening issue, not a superseding tracker for the full dashboard audit.

## Best Possible Solution

Keep #38420 open unless a maintainer intentionally splits it. Treat #39780 as a narrower Config hardening follow-up, not a replacement. The remaining best path is to either implement or split the outstanding dashboard work: per-instance actions and clearer presence chip/reason affordances, Sessions key copy/truncation and less repetitive advanced override controls, clearer Config action semantics, broader operator shortcut/help UX, and feedback/activity improvements.
