## Summary

Current main still does not implement on-demand Telegram pinned-message reads. Telegram exposes outbound delivery pinning and generic message-action support for other channels, but the bundled Telegram action map, runtime, config gates, docs, and contract tests omit `getChat.pinned_message`, `list-pins`, or `/getpinned`. The related implementation PRs #38613 and #38718 are useful prior art but were closed unmerged, so they do not supersede this issue.

## Best Possible Solution

Keep #38580 open until the bundled Telegram plugin implements an on-demand pinned-message read path, likely via `list-pins`/`pins` message action or a native `/getpinned` command backed by Telegram Bot API `getChat.pinned_message`. Use #38613 and #38718 as prior art, but land a focused current-main implementation with Telegram action discovery/runtime plumbing, config/schema/docs updates, and regression tests.
