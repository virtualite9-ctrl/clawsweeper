## Summary

Keep #38580 open. Current main still has no Telegram Bot API path that reads `getChat.pinned_message` on demand: Telegram exposes outbound send/poll/react/delete/edit/topic actions and delivery pinning, while generic `list-pins` support remains available only for other channels/plugins that declare it. The linked implementation attempts #38613 and #38718 are useful prior art but were closed unmerged, so they do not supersede the issue.

## Best Possible Solution

Keep the issue open until the bundled Telegram plugin implements an on-demand pinned-message read path, likely as a `list-pins`/read-pinned message action or explicit `/getpinned` native command backed by Bot API `getChat.pinned_message`, with Telegram discovery/runtime plumbing, config/schema metadata, docs, and regression tests for no pin, API errors, target resolution, formatting, and multi-account behavior. If maintainers want this outside the bundled Telegram plugin, record that product decision explicitly because current main cannot satisfy the request.
