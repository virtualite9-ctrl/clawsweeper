## Summary

Keep #38580 open. Current main still does not implement on-demand Telegram pinned-message reads via Bot API `getChat.pinned_message`: Telegram only supports delivery pinning/outbound pinChatMessage, while the bundled Telegram message-action and native-command surfaces omit `list-pins`, `getpinned`, or equivalent read behavior. The related implementation PRs #38613 and #38718 were closed unmerged.

## Best Possible Solution

Keep the issue open until the bundled Telegram plugin implements an on-demand pinned-message read path, likely through the shared message action surface (`list-pins`/read-pinned) or an explicit Telegram native command backed by Bot API `getChat.pinned_message`, with Telegram discovery/runtime plumbing, config/schema metadata, docs, and regression tests. If maintainers decide this belongs outside the bundled Telegram plugin, record that product decision explicitly because current main cannot satisfy the request.
