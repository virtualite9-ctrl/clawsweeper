## Summary

Keep #38580 open. Current main still supports Telegram outbound delivery pinning, and the shared message action model has a generic list-pins action, but the bundled Telegram plugin does not expose a Bot API getChat.pinned_message read path. Telegram discovery, runtime dispatch, config, docs, and tests still omit list-pins/getpinned; the related implementation PRs were closed unmerged.

## Best Possible Solution

Keep #38580 open until the bundled Telegram plugin implements an on-demand pinned-message read path, likely as message action list-pins/read-pinned backed by Telegram Bot API getChat.pinned_message, with Telegram discovery/runtime plumbing, config/schema metadata, CLI/tool formatting, tests, and public docs. If maintainers decide this should be external instead, record that product decision explicitly because the current bundled Telegram channel cannot satisfy the request.
