## Summary

Keep open. Current `main` still lacks zsh positional-argument completion for leaf commands, so the underlying bug appears real, and this PR is not a conservative close candidate because the proposed patch has unresolved correctness issues in review.

## Evidence

- **Current main still omits leaf positional specs:** In `generateZshSubcommands`, leaf-command functions emit `_arguments` with only `generateZshArgs(cmd)`. There is no positional-argument generator on `main`, so commands like `openclaw wiki ingest <path>` still do not advertise `_files` completions for their positional args.
  - file: [src/cli/completion-cli.ts:257](https://github.com/openclaw/openclaw/blob/2cd2732ab6695a4b3220ce41166b74aca7097c5f/src/cli/completion-cli.ts#L257)
  - command: `nl -ba src/cli/completion-cli.ts | sed -n '222,266p'`
  - sha: [2cd2732ab669](https://github.com/openclaw/openclaw/commit/2cd2732ab6695a4b3220ce41166b74aca7097c5f)
- **Affected commands exist on main:** The bundled memory-wiki CLI defines `wiki ingest` with a required `<path>` positional argument described as `Local file path to ingest`, matching the PR's reported surface.
  - file: [extensions/memory-wiki/src/cli.ts:715](https://github.com/openclaw/openclaw/blob/2cd2732ab6695a4b3220ce41166b74aca7097c5f/extensions/memory-wiki/src/cli.ts#L715)
  - command: `nl -ba extensions/memory-wiki/src/cli.ts | sed -n '708,724p'`
  - sha: [2cd2732ab669](https://github.com/openclaw/openclaw/commit/2cd2732ab6695a4b3220ce41166b74aca7097c5f)
- **Current tests do not cover the missing behavior:** `completion-cli.test.ts` checks nested zsh subcommands, deferred registration, PowerShell, and fish output, but there is no assertion for zsh positional-argument completion on leaf commands. Nothing in the current test suite proves the reported bug is already fixed.
  - file: [src/cli/completion-cli.test.ts:24](https://github.com/openclaw/openclaw/blob/2cd2732ab6695a4b3220ce41166b74aca7097c5f/src/cli/completion-cli.test.ts#L24)
  - command: `nl -ba src/cli/completion-cli.test.ts | sed -n '1,260p'`
  - sha: [2cd2732ab669](https://github.com/openclaw/openclaw/commit/2cd2732ab6695a4b3220ce41166b74aca7097c5f)
- **The PR patch has unresolved correctness problems:** From the provided PR patch and review timeline: the change calls `arg.description?.()` even though Commander `Argument.description` is a string property, the variadic branch returns the same fixed-position spec as the non-variadic path, and the unconditional trailing continuation would corrupt `_arguments` output when no positional args exist. No follow-up commit in the provided timeline addresses those review findings.
  - file: [src/cli/completion-cli.ts:231](https://github.com/openclaw/openclaw/blob/c016fa94d7b0aa32233ea5af8a2e9f9b5fcd63c4/src/cli/completion-cli.ts#L231)

- **Main already contains leaf commands whose descriptions include spaces and variadic args:** Examples on `main` include `models set <model>` with description `Model id or alias` and `models auth-binding set <profileIds...>` with a variadic positional. Those cases line up with the review concerns about unquoted positional specs and incorrect variadic handling.
