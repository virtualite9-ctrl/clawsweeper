## Close Comment

Closing as implemented on current `main` after Codex review. The report showed the gateway dying at roughly a 3-second pre-auth handshake window (`handshakeMs=3002` on OpenClaw 2026.3.13). Current OpenClaw no longer uses that 3s default: `src/gateway/handshake-timeouts.ts:1` now sets the pre-auth/connect-challenge timeout to `10_000` ms, `docs/gateway/protocol.md:493` documents the same default, and `CHANGELOG.md:2446` records that this shipped in `2026.3.22` specifically to stop busy local gateways from dropping healthy CLI connections at 3 seconds. Current `src/gateway/call.ts:480` also includes Windows-specific hardening on the websocket connect path to avoid starving `connect.challenge` processing and triggering handshake timeouts. Based on that shipped fix and the current code on `main`, this 2026.3.13 browser-command handshake-timeout regression is already addressed in released builds.
