## Summary

Keep open. Current main still serves `/media/:id` with only `X-Content-Type-Options: nosniff`; it does not set the `Cache-Control`, `Pragma`, or `Expires` directives proposed by this PR, and the media tests do not assert those anti-cache headers. The related browser/noVNC hardening is a separate surface and does not supersede this media endpoint change.

## Best Possible Solution

Keep the PR open, rebase it onto current main, set the anti-cache headers immediately after a `/media/:id` route match and before all validation/read branches, and expand regression coverage for success, missing media, HEAD, and representative error responses.
