## Summary

Keep open. Current main still serves temporary `/media/:id` responses without explicit `Cache-Control`, `Pragma`, or `Expires` anti-cache headers, while this PR is a narrow core media security-hardening change. The branch is stale against the current handler shape, but the underlying fix remains useful and is not implemented, superseded, or out of scope.

## Best Possible Solution

Keep this PR open. Rebase the change onto `src/media/server.ts` by setting `Cache-Control: no-store, private, max-age=0, must-revalidate`, `Pragma: no-cache`, and `Expires: 0` immediately after a matched `/media/:id` route and before validation/read/error branches. Update `src/media/server.test.ts` so success, HEAD, missing-media, expired, and representative validation/error responses assert those headers while preserving TTL expiry, cleanup timing, path validation, active-content download behavior, and file-serving semantics.
