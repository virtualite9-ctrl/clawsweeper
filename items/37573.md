## Summary

Keep open. Current `main` at `55389f4be29359e2dd57ebaf548ba4a70331a445` still requires manual model entry for custom API onboarding, and I found no replacement implementation or test coverage for `/models` discovery. The idea also fits OpenClaw core onboarding UX rather than ClawHub/plugin scope.
