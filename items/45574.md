## Summary

Current main still appears to have the reported Slack `download-file` schema gap. Slack advertises and dispatches `download-file`, and both Slack dispatch/runtime require `fileId`, but the shared `message` tool schema does not define `fileId` and Slack discovery currently returns no schema contribution to add it. The linked fix PR #45626 is an open, unmerged implementation attempt rather than a shipped/superseding fix.

## Best Possible Solution

Keep #45574 open until main adds a schema-visible optional `fileId` parameter for Slack `download-file` and covers it with a message-tool schema test plus the existing Slack dispatch/runtime tests. PR #45626 is the closest canonical implementation attempt and should either be updated/landed or replaced with an equivalent fix on current main; PR #62792 addresses related Slack inbound media hydration but does not close this schema pass-through bug.
