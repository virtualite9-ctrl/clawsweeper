## Summary

Current main still has the reported behavior: service PATH generation unconditionally adds hard-coded version-manager/package-manager fallback directories, while service audit still treats those same path families as non-minimal when they are not part of the expected minimal PATH. The related PR #72109 was closed unmerged, and the linked bug #71944 being closed does not prove this fix shipped. #71964 remains a plausible focused fix, though it needs normal maintainer review and small cleanup before landing.

## Best Possible Solution

Keep #71964 open for maintainer review. The best path is to land this PR or an equivalent narrow daemon change that gates only hard-coded service PATH fallbacks behind an injected existence predicate, preserves explicit env-configured roots and stable user-bin dirs, updates deterministic service-env coverage including the Nix hermetics gap, and adds a changelog entry that follows the repository attribution rules.
