## Summary

Keep this PR open. Current main still unconditionally emits hard-coded version-manager and package-manager fallback directories into generated gateway/node service PATHs, while doctor audits those same path families as non-minimal. The PR targets a real remaining daemon behavior tied to #71944 and should receive maintainer review or be landed as a maintainer-equivalent fix.

## Best Possible Solution

Review and land this PR, or land a maintainer-equivalent fix: add an injectable existence predicate to the service PATH builder, gate only hard-coded version-manager/package-manager fallbacks behind it, keep explicit env roots and stable user-bin conventions unconditional, add deterministic Linux/macOS regression tests for missing and existing fallbacks, and keep the changelog entry in the active Fixes section with `Thanks @Sanjays2402`.
