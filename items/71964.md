## Summary

Keep PR #71964 open. Current main still unconditionally emits the hard-coded Volta/asdf/Bun/fnm/pnpm fallback directories into gateway and node service PATHs, and the existing tests still assert those fallback directories are always present. The PR is a focused fix for a real remaining behavior, but it needs maintainer review and small cleanup before landing.

## Best Possible Solution

Keep this PR open or land a maintainer-equivalent fix. The product-correct path is to filter only hard-coded version-manager/package-manager fallback directories through an injectable existence predicate, keep explicit env-configured roots and stable user-bin conventions unconditional, add deterministic daemon tests for both missing and existing fallbacks, and update the changelog with the required contributor attribution.
