## Summary

Current main already fixes the reported subagent deliveryContext inheritance problem. Subagent spawn now carries the requester channel/to/account/thread into the child agent call and run registry, the gateway seeds fresh or pre-patched subagent sessions from those request params, and regression coverage includes both deliveryContext persistence and Telegram requester-origin announce routing.

## Best Possible Solution

Close this issue as implemented. Keep the shipped spawn and gateway deliveryContext seeding behavior, and point users affected on 2026.4.14 to update to v2026.4.24 or newer. Any remaining Telegram-specific failure should be filed as a fresh issue with current-version sessions.json/run logs and the affected group/topic/account details.

## Evidence
