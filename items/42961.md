## Summary

Keep this PR open. Current main still applies fixed short OpenRouter Anthropic cache markers and does not give the OpenRouter wrapper the effective cacheRetention value, while the PR remains a focused implementation candidate that needs refresh for the current helper layout and SDK compatibility.

## Best Possible Solution

Keep this PR open and refresh it against current main. The best fix is to route the effective cacheRetention for verified OpenRouter Anthropic routes into cache marker application, preserve default short behavior, suppress markers for cacheRetention: "none", emit ttl: "1h" for cacheRetention: "long", keep custom OpenAI-compatible proxy exclusions intact, preserve public SDK helper compatibility, and add regression tests for default/short/none/long plus non-Anthropic and custom-proxy cases.
