## Summary

Keep this PR open. Current main still injects fixed short Anthropic cache_control markers for verified OpenRouter Anthropic routes, but it does not pass the effective cacheRetention value into that OpenRouter payload wrapper. The PR addresses a real remaining behavior gap, though it needs a refresh for the current helper layout and public SDK compatibility.

## Best Possible Solution

Keep the PR open and refresh it against current main. The best fix is to route the effective cacheRetention for verified OpenRouter Anthropic routes into the cache marker application, preserve the default short marker, suppress markers for cacheRetention: "none", emit ttl: "1h" for cacheRetention: "long", keep custom OpenAI-compatible proxy exclusions intact, preserve public SDK helper compatibility, and add regression tests for default/short/none/long plus non-Anthropic and custom-proxy cases.
