## Summary

Keep this PR open. Current main still injects a fixed OpenRouter Anthropic cache_control marker and does not route the effective cacheRetention value into that wrapper, so explicit "none" and "long" settings are not implemented for this path. The PR is a focused bug-fix candidate with tests and no apparent security-sensitive file changes.

## Best Possible Solution

Review or refresh the PR against current main, threading the resolved cacheRetention for verified OpenRouter Anthropic routes into the cache marker application. The implementation should preserve default short caching, skip markers for cacheRetention: "none", emit ttl: "1h" for cacheRetention: "long", and add regression tests for all three OpenRouter Anthropic cases plus non-Anthropic and custom-proxy exclusions.
