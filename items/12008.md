## Summary

Keep #12008 open. Current main at 406ae72fd2789c78053f6b94006ec8f488bc1c00 still does not expose a typed or schema-validated Gemini safety-settings config field, and the direct Google Generative AI transport does not emit `safetySettings`. The provided discussion also shows PR #49529 as the open, unmerged implementation attempt, and the latest maintainer/Codex review comment explicitly kept this issue open.

## Best Possible Solution

Keep #12008 open until PR #49529 or an equivalent implementation lands. The maintainable path is a typed and schema-validated `models.providers.*.safetySettings` config surface, generated schema/help/docs updates, runtime mapping into direct `google-generative-ai` requests for Google and Google-compatible provider entries, explicit request-level precedence, and focused regression tests.
