## Summary

Keep open. Current main still does not expose a typed Gemini safety-settings config surface or map safety settings into direct `google-generative-ai` Gemini payloads. The provided related PR #49529 is relevant, but it remains open/unmerged in the supplied context, so the user-facing capability is still missing from main.

## Best Possible Solution

Keep #12008 open until PR #49529 or an equivalent implementation lands. The likely path is a typed, schema-validated `models.providers.*.safetySettings` config surface, docs for the public provider config, direct `google-generative-ai` payload mapping to Gemini `safetySettings`, preservation of explicit request-level safety settings if supported, and regression tests for both bundled `google` and custom provider keys using the Google adapter.
