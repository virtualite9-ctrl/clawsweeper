## Summary

Keep open. Current main at 7e13f3f5140168e31027b39a9855ade3fadf24d5 still lacks a schema-validated Gemini safety-settings config surface and does not map `safetySettings` into direct Google Generative AI payloads. The supplied discussion also shows PR #49529 as open/unmerged and the latest maintainer Codex comment already keeping #12008 open.

## Best Possible Solution

Keep #12008 open until PR #49529 or an equivalent implementation lands. The maintainer-ready path is a schema-validated `models.providers.*.safetySettings` config field, runtime mapping into direct `google-generative-ai` Gemini payloads without overriding explicit request-level settings, generated schema/docs updates, and regression tests for both bundled Google and custom provider keys using the Google adapter.
