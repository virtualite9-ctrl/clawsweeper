## Summary

Keep this issue open. Current main still archives the whole OpenClaw state directory and only offers workspace/config scope reduction, with no built-in exclude flags, smart exclusions, progress, or size guard for large state directories. The linked #44288 PR is still the implementation candidate and uses closing syntax for this issue, so the issue should remain open until that PR lands or maintainers choose another fix.

## Best Possible Solution

Keep the issue open and review #44288 as the current implementation candidate. The built-in fix should live in the backup CLI/planner/archive path, with conservative exclusion support for large ephemeral state directories plus clear progress or failure reporting. If #44288 is closed instead of landed, maintainers should still implement the scoped core fix in `src/infra/backup-create.ts`, `src/commands/backup-shared.ts`, `src/cli/program/register.backup.ts`, and `docs/cli/backup.md`. The ClawHub `keepmyclaw` plugin is a helpful workaround, not a replacement for fixing the built-in backup command.
