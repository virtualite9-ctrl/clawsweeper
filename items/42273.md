## Summary

Keep this issue open. Current main still has no built-in way for `openclaw backup create` to exclude large ephemeral state subdirectories, show progress, or enforce a size guard. It plans the whole state directory and passes all planned asset roots directly into tar. The linked `keepmyclaw` ClawHub plugin is a useful workaround, but the built-in backup command still needs a core fix. The open #44288 PR is the current implementation candidate and uses closing syntax for this issue, so the issue should remain open until that PR lands or maintainers choose a different fix.

## Best Possible Solution

Keep this issue open and review #44288 as the current implementation candidate. The core fix should live in the backup CLI/planner/archive path, adding conservative exclusion support for large ephemeral state directories plus clear progress or failure reporting. If #44288 is closed instead of landed, maintainers should still implement a focused built-in fix in `src/cli/program/register.backup.ts`, `src/commands/backup-shared.ts`, `src/infra/backup-create.ts`, and `docs/cli/backup.md`. The ClawHub `keepmyclaw` plugin should remain documented only as a workaround, not treated as closing the built-in backup issue.
