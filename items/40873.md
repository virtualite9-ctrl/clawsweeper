## Summary

Current `main` at `b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503` does not implement this PR’s fix. TUI still treats assistant `content: string` as plain renderable text, so inline `<think>...</think>` content would remain visible and no separate thinking text is extracted. The repo already has a canonical assistant-visible-text sanitizer that strips reasoning tags, but the TUI formatter path is not using it for string payloads. This should stay open.
