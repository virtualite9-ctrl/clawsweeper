## Summary

Keep this issue open. Current main has several adjacent inspection surfaces, but none implements the requested active-agent hierarchy/tree command with JSON/watch/session/resource views. The existing CLI lists configured agents, stored sessions, and background tasks as flat views, while `/subagents list` is scoped to the current session and renders active/recent runs as a numbered list rather than a hierarchy. The provided related items are only title-search overlaps and do not supersede this request.

## Best Possible Solution

Keep this open for maintainer/product judgment and implement a core inspection surface, likely under `openclaw agents tree` or `openclaw sessions tree` if maintainers prefer a namespaced command. It should reuse existing session/subagent/task metadata (`spawnedBy`, `parentSessionKey`, `childSessions`, task status) and add focused tests for text tree output plus JSON output before considering watch/resource extensions.
