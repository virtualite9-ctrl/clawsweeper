## Summary

Keep #2597 open. Current main still lacks the requested passive agent-visible `context=X%` field in the system-prompt Runtime line. Existing `/status`, `/context`, UI context indicators, and the community `openclaw-workspace-sync` plugin are useful visibility/recovery mitigations, but they do not implement the proposed per-turn Runtime-line primitive. The latest maintainer-facing issue comment also explicitly kept this issue open.

## Best Possible Solution

Keep the issue open until OpenClaw either implements bounded context-usage plumbing from the latest reliable session token snapshot into embedded-runner params, `RuntimeInfoInput`, and `buildRuntimeLine()` with focused tests/docs, or a maintainer explicitly decides that `/status`, `/context`, UI warnings, context/memory plugins, and `openclaw-workspace-sync` are the intended product boundary.
