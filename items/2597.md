## Summary

Keep #2597 open. Current main still has explicit context visibility surfaces such as `/status`, `/context`, UI context notices, and fresh token snapshots, but it does not implement the requested passive agent-visible `context=X%` field in the system-prompt Runtime line. The latest provided maintainer-facing comment also explicitly kept this issue open, and the linked `openclaw-workspace-sync` plugin is a recovery/sync mitigation rather than the requested runtime prompt primitive.

## Best Possible Solution

Keep the issue open until OpenClaw either implements bounded context-usage plumbing from the latest fresh session token snapshot into embedded-runner runtime params, `RuntimeInfoInput`, and `buildRuntimeLine()` with focused tests/docs, or a maintainer explicitly decides that `/status`, `session_status`, `/context`, UI warnings, context-engine/memory plugins, and the `openclaw-workspace-sync` recovery plugin are the intended product boundary.
