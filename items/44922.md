## Summary

Current main still appears to have the reported duplication path: main-session cron jobs enqueue the systemEvent with a `cron:` context key, heartbeat wraps any pending cron-tagged event with `buildCronEventPrompt`, and the reply path also drains the same queued event into `System:` lines. The linked PR #44973 directly tracks a plausible fix, but it is open and unmerged, so this bug should remain open until that PR or an equivalent change lands.

## Best Possible Solution

Keep #44922 open and use #44973, or an equivalent maintainer patch, as the implementation path. The fix should make main-session cron `systemEvent` wakes avoid re-wrapping the same queued text in `buildCronEventPrompt` when the event is already injected into the main-session prompt, and should add focused regression coverage for `sessionTarget: "main"`, `payload.kind: "systemEvent"`, and `delivery.mode: "none"`.
