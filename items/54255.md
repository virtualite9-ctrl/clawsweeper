## Summary

Keep open. Current `main` still has a plausible live Control UI bug: streamed assistant deltas are only suppressed when the text is exactly `NO_REPLY`, but the final-event fallback will persist any non-exact buffered prefix such as `NO_` when the final assistant message is the silent token. The later `chat.history` reload does filter exact silent replies, which matches the reporter's note that the stray bubble disappeared after a few minutes.

## Evidence

- **Delta path only suppresses exact token:** `handleChatEvent` updates `state.chatStream` for any delta text that is not an exact `NO_REPLY` match. A streamed prefix like `NO_` is therefore kept in UI state.

  - command: `nl -ba ui/src/ui/controllers/chat.ts | sed -n '448,470p'`

- **Final path can render buffered prefix:** On `final`, if the normalized assistant message is silent, the code falls back to appending `state.chatStream` whenever that buffer is non-empty and not an exact `NO_REPLY` match. If the last streamed snapshot was `NO_`, it will be rendered as a visible assistant bubble.
