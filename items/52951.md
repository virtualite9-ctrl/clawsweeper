## Summary

Keep open. Current `main` still does not implement the proposed `tools.fs.roots` contract: config/schema/policy remain `workspaceOnly`-only, host FS tools still use the binary workspace guard, and media roots still use default/workspace string roots. The PR is active, broad, and security-sensitive, so no conservative cleanup close reason applies.

## Best Possible Solution

Keep PR #52951 open. Maintainers should decide whether `tools.fs.roots` is the desired core filesystem policy contract; if yes, review and narrow the implementation around config/schema docs, host read/write/edit/apply_patch enforcement, media/outbound propagation, sandbox non-interference, and focused tests. If maintainers prefer a broader filesystem sandboxing design, use that product decision to supersede this PR explicitly rather than auto-closing it as obsolete.
