## Summary

Keep open. Current main has newer context-based routing that may mitigate normal Telegram cron tool runs, but the exact session-key inference path reported in #44270 still strips `:thread:` before parsing and still lacks a Telegram direct-DM topic regression test.

## Best Possible Solution

Keep the issue open and fix the remaining fallback in `src/agents/tools/cron-tool.ts`: detect Telegram direct session keys with `:thread:<chatId>:<threadId>` or `:thread:<threadId>`, preserve the originating chat, and persist either `delivery.to=<chatId>:topic:<threadId>` or explicit `delivery.threadId` according to the current cron delivery contract. Add a regression test for the exact reported key plus non-Telegram threaded keys to preserve existing parent-peer behavior.

## Evidence
