## Summary

Current `main` already contains the relevant fix path for this report: embedded-agent LLM idle timeouts are now configurable, slow local providers inherit `agents.defaults.timeoutSeconds` when set, idle-timeout failures surface an explicit error instead of leaving an empty/spinning turn, and one same-model retry is attempted before surfacing the failure. The issue timeline’s linked PR direction (#60413) matches this implementation. The fix appears to be on `main` after the latest shipped release `v2026.4.23`, so `fixedRelease` is not yet identifiable.

## Evidence

- **Idle timeout is now configurable and falls back to agent timeout:** `resolveLlmIdleTimeoutMs` reads `agents.defaults.llm.idleTimeoutSeconds`, falls back to explicit run timeout, then `agents.defaults.timeoutSeconds`, instead of always using the old 120s default.

- **Embedded runs apply the configurable idle timeout to model streaming:** `run/attempt.ts` wraps the active model stream with `streamWithIdleTimeout(...)` using the resolved timeout from config/run settings.
