## Summary

Keep open. Current main still matches the reported ACP media-understanding gap: ACP dispatch invokes media understanding for inbound media, but passes only ctx and cfg, while image provider execution requires an agentDir and the normal reply path supplies both agentDir and active model context. The MiniMax docs and plugin registration support the reporter’s expectation that text-only MiniMax M2.7 can use MiniMax-VL-01 through media understanding.

## Best Possible Solution

Keep this issue open. Fix ACP dispatch by resolving the effective ACP agent directory from the canonical session or resolved ACP agent, pass it to applyMediaUnderstanding, and pass active model context when ACP runtime/session configuration can provide it so native-vision skip behavior matches normal replies. Add a focused dispatch-acp.test.ts regression for an inbound image ACP turn with configured tools.media.image, asserting media understanding receives agentDir and enriches the prompt before runTurn. Treat the Windows OPENCLAW_AGENT_DIR doctor expectation as a separate follow-up if it still reproduces after the ACP fix.
