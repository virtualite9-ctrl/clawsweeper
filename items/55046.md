## Summary

Keep open. Current main still matches the reported ACP media-understanding gap: `tryDispatchAcpReply` invokes `applyMediaUnderstanding` for inbound media but does not pass an `agentDir`, while the current image provider execution path requires `agentDir` before it can describe images. The regular non-ACP reply path does pass `agentDir`, so this looks like a real ACP-specific bug rather than an already-shipped fix.

## Best Possible Solution

Do not close this issue yet. The best fix is to make ACP dispatch resolve the effective ACP agent directory from the configured/resolved ACP agent or session and pass it to `applyMediaUnderstanding`; passing active model context should also be considered so the native-vision skip logic stays consistent with the normal reply path. Add a focused `dispatch-acp.test.ts` regression that an inbound image ACP turn invokes media understanding with `agentDir` and that configured image understanding can enrich the prompt for a text-only ACP model. The Windows `OPENCLAW_AGENT_DIR`/doctor expectation in the report may need a separate clarification, but the core media-understanding bug is actionable in this repository.
