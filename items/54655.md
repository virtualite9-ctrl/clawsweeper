## Summary

Keep open. Current `main` already has the Stop-button behavior this PR describes, but it still clears `chatMessage` on session switch, so the per-session draft-loss bug is not implemented. The PR also picked up an unrelated sidebar display-name commit against code that has since moved, and the existing review feedback identifies plausible regressions there, so this needs maintainer rework rather than conservative auto-close.
