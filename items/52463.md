## Summary

Current `main` still routes post-compaction context through queued `System:` events and prepends those events onto the next user prompt body. The Control UI/history sanitizers strip envelope and inbound metadata, but they do not strip trusted `System:` lines, so the reported leak remains plausible on current `main` rather than already fixed.

## Evidence

- **Post-compaction text is still queued as a system event:** `src/auto-reply/reply/agent-runner.ts:1576`-`1585` still calls `readPostCompactionContext(...)` after compaction and enqueues the result with `enqueueSystemEvent(contextContent, { sessionKey })`. The generated text in `src/auto-reply/reply/post-compaction-context.ts:149`-`162` still includes the same `[Post-compaction context refresh]` / `Session was just compacted...` wording from the report.
