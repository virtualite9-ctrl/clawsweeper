## Summary

Keep open. Current main still has a source-level path that can report Telegram as not configured for the reported multi-account shape: a named account can be configured while the selected default/fallback account is false, and doctor/summary formatting still uses that default/fallback configured flag for the top-level channel line. The related #41929 was closed unmerged and was test-only, so it does not supersede or prove a shipped fix.

## Best Possible Solution

Keep the issue open and fix the doctor/channel summary aggregation. The top-level Telegram doctor line should consider any enabled configured account, or select a configured account for the channel summary, while still surfacing duplicate-token/account-level details. Add a regression test for the reported shape: top-level `botToken`, `accounts.<numeric>.botToken`, and `accounts.default.botToken` all present, with at least one Telegram account configured and the doctor line not saying `Telegram: not configured`.
