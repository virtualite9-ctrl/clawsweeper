## Summary

Current `main` does not implement the requested per-model Google Vertex location override. The runtime still resolves Google Vertex routing from provider-level transport/env state, and model-level transport fields are not actually consumed for `google-vertex` models.

## Evidence

- **Configured model overrides do not read model-level baseUrl:** `applyConfiguredProviderOverrides` resolves transport from `providerConfig.baseUrl ?? discoveredModel.baseUrl`; the matched `configuredModel` contributes metadata like `api`, `reasoning`, and token limits, but its own `baseUrl` is never read.
  - file: [src/agents/pi-embedded-runner/model.ts:369](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/src/agents/pi-embedded-runner/model.ts#L369)
  - sha: [ec3dbd22a4de](https://github.com/openclaw/openclaw/commit/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46)
- **Inline provider models also ignore model-level baseUrl:** `buildInlineProviderModels` builds per-model transport using `entry?.baseUrl` only, then overwrites each model with that resolved base URL. A `models[].baseUrl` value on the model entry is not used.
  - file: [src/agents/pi-embedded-runner/model.inline-provider.ts:137](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/src/agents/pi-embedded-runner/model.inline-provider.ts#L137)
  - sha: [ec3dbd22a4de](https://github.com/openclaw/openclaw/commit/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46)
- **Config type has no model-level location field:** `ModelDefinitionConfig` exposes `baseUrl` but no `location`; the requested `models[...].location` surface is not present on current `main`. `ModelProviderConfig` also has only provider-level `baseUrl`.
  - file: [src/config/types.models.ts:77](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/src/config/types.models.ts#L77)
  - sha: [ec3dbd22a4de](https://github.com/openclaw/openclaw/commit/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46)
- **Google Vertex auth path still depends on env-level location:** `resolveGoogleVertexEnvApiKey` only treats ADC auth as available when both project and `GOOGLE_CLOUD_LOCATION` are present, which reinforces the current provider/env-scoped location model rather than a per-model override.
  - file: [src/agents/model-auth-env.ts:32](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/src/agents/model-auth-env.ts#L32)
  - sha: [ec3dbd22a4de](https://github.com/openclaw/openclaw/commit/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46)

## Risks / Open Questions

- There may be a manual workaround by defining separate provider entries or explicit endpoint variants, but that is not the requested built-in per-model override for `google-vertex`.
- `models.providers.*.models[].baseUrl` appears in generated schema tags, so maintainers may want to clarify whether that field is intended-but-broken or only partially wired.
