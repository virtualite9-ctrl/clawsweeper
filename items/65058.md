## Summary

Current main still only normalizes Google Workspace Add-on `chat.messagePayload` in the Google Chat webhook parser. Add-on `chat.addedToSpacePayload` and `chat.removedFromSpacePayload` still fall through to generic validation without `type`/`eventType` and are rejected as `invalid payload`, so this PR remains a focused implementation candidate for part of #65007.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to revise the parser with explicit branches for `messagePayload`, `addedToSpacePayload`, and `removedFromSpacePayload`, preserve add-on token extraction for each accepted variant, add focused regression tests for added-to-space and removed-from-space payloads, and leave #65007 open for its remaining group allowlist and reply-thread issues.
