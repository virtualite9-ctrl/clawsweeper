## Summary

Current main still has the Google Chat add-on parser gap this PR targets: it only normalizes `chat.messagePayload`, while official add-on payload mappings include `chat.addedToSpacePayload` and `chat.removedFromSpacePayload`. The PR remains a focused implementation candidate, though it should add regression tests and resolve the existing review comment about keeping payload selection and event-type labeling in the same branch.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to update the parser with explicit branches for `messagePayload`, `addedToSpacePayload`, and `removedFromSpacePayload`, preserve body `systemIdToken` extraction for each accepted add-on variant, add focused regression tests for added-to-space and removed-from-space payloads, and leave #65007 open for the unrelated group allowlist and reply-thread work.
