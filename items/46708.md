## Summary

Current main implements the central user problem in #46708: Discord now has native exec approval delivery with approver DM/origin-channel routing, replay of pending approvals after reconnect, documented `/approve` fallback semantics, and a current-main fix for stale/already-resolved Discord button clicks. The later related #71804 Discord regression is covered by this current-main path; the narrower Control UI multi-channel source-routing issue remains separately tracked by #54855.

## Best Possible Solution

Close this broad Discord approval UX issue as implemented on current main, keep the shipped native Discord approval/replay path, and use #54855 for the remaining narrower Control UI multi-channel source-routing follow-up if maintainers want to address that separately.
