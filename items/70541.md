## Summary

Keep #70541 open. Current main still has the reported post-run auto-compaction context injection path reading `AGENTS.md` from `process.cwd()` in `src/auto-reply/reply/agent-runner.ts`, while the queued run contract already carries a required `run.workspaceDir` and the preflight compaction refresh path uses that workspace correctly. The related closed compaction-context items are adjacent feature/fix history, but they do not supersede this specific cwd-vs-workspace bug.

## Best Possible Solution

Fix `src/auto-reply/reply/agent-runner.ts` to pass `followupRun.run.workspaceDir` into `readPostCompactionContext` for post-run auto-compaction system-event injection, then add a focused regression test where `process.cwd()` differs from `followupRun.run.workspaceDir` and only the run workspace contains the expected `AGENTS.md` sections.
