## Summary

Keep #48432 open. Current main still applies the TUI long-token sanitizer across the full rendered text before Markdown rendering, without a code-fence exception and without protecting hyphen-only package/file-like tokens. The exact 33-character package-name class from the report can still be split with an inserted space. The related PR #69340 is still the focused open follow-up, while #48445 was closed unmerged.

## Best Possible Solution

Keep #48432 open. Land or supersede PR #69340 with a current-main TUI sanitizer/rendering fix and regression tests that preserve hyphenated package/file-like tokens inside fenced code blocks, while retaining narrow-terminal protection for genuinely uncopyable long prose tokens.
