## Summary

Keep this PR open. Current main still lacks `messages.inbound.botDebounceMs`, so the PR is not obsolete, and it remains a focused implementation path. It is also not ready to treat as a complete user-facing feature because production channel paths still do not pass bot-origin state into inbound debounce resolution. The earlier generated schema/baseline concern appears addressed by later PR commits; the remaining material blocker is production wiring.

## Best Possible Solution

Keep this PR open for maintainer review. Either scope it explicitly as preparatory config/API work, or complete the product path before merge by adding the config type/schema/generated metadata/tests, threading reliable per-message bot-origin state through the shared debounce resolver/helper, updating relevant channel/plugin handlers, and adding focused production-path tests plus docs/changelog for the shipped behavior.
