## Summary

Keep open. Current main still has the Telegram DM no-visible-response leak described in #70628: the dispatch path synthesizes a `NO_REPLY`, outbound policy rewrites direct silent replies into visible filler text, and the Telegram regression test still asserts that behavior. The linked PR #70630 remains the right fix path, but it is not on main in this checkout.

## Best Possible Solution

Keep #70628 open and land PR #70630 or an equivalent Telegram adapter fix: do not synthesize transport-visible `NO_REPLY` for successful no-visible-response direct-message turns, treat that branch as handled/done for status reactions, preserve real error/non-silent delivery fallback behavior, and update the Telegram dispatch regression test to assert no chat message is sent for the DM no-op case.
