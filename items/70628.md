## Summary

Keep open. Current main still implements the reported Telegram DM no-visible-response path by synthesizing `NO_REPLY`, projecting it through the shared outbound silent-reply policy, and sending any projected payload with `silent: false`. The current regression test still expects the unwanted visible rewrite behavior, and the related fix PR #70630 is open rather than merged.

## Best Possible Solution

Keep #70628 open and land #70630 or an equivalent narrow Telegram adapter fix: successful no-visible-response Telegram direct turns should be treated as handled silent no-ops, without synthesizing a transport-visible `NO_REPLY`; preserve the real error/non-silent delivery-failure fallback path, and flip the current Telegram dispatch regression test to assert no chat message is delivered for this DM case.
