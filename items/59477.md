## Summary

This PR should close as superseded. The linked bug #59256 was already closed as implemented on current main via the explicit sandbox session-tools visibility override path, while the remaining broader design question about allowing A2A sends without broad session visibility is already tracked in #57447/#57483. The proposed patch is also not a complete canonical fix for that remaining work because current main's visibility guard still rejects cross-agent sends unless effective visibility is `all`, and the PR relaxes label resolution before final visibility enforcement.

## Best Possible Solution

Close this PR as superseded. Keep the shipped explicit sandbox override path for the already-closed #59256 behavior, and handle any future less-broad A2A send exception in the canonical #57447/#57483 thread with an early authorization design, label-resolution safety, docs, and regression tests.
