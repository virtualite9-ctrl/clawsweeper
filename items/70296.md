## Summary

Keep #70296 open. Current main likely addresses the Control UI model read-back path, but the umbrella issue still has unresolved restart-sentinel channel-notification behavior: config restarts still create sentinel notices with doctor hints, startup still delivers those notices to routed channels, and no suppress-channel config key exists. The BOOT.md token concern also needs maintainer clarification or migration guidance because current source/docs recognize `NO_REPLY`, not `NO_REPLY_NEEDED`.

## Best Possible Solution

Keep the issue open or split it into focused follow-ups. The core fix should add a documented restart-sentinel channel-notice policy/config key and thread it through sentinel formatting or `deliverRestartSentinelNotice`, with routed WhatsApp/Telegram-style regression coverage. For BOOT.md, either document/migrate `NO_REPLY_NEEDED` to the current `NO_REPLY` token or intentionally add a compatibility alias if maintainers still consider that token public. Add a targeted Control UI regression test for object-form `agents.defaults.model.primary` and per-agent `agents.list[].model.primary` read-back before marking that subissue fully closed.
