## Summary

Keep open. Current main still does not implement configurable TUI status-line command rendering, and the capability is not cleanly externalizable through the current plugin API because there is no TUI render/status-area plugin seam. The PR remains a plausible core/TUI feature or plugin-API design candidate rather than something obsolete, duplicate, shipped, or out of scope.

## Best Possible Solution

Keep this PR open for maintainer/product review. If accepted, the implementation should land as a core TUI/config feature with type, strict Zod schema, generated schema/docs alignment, lifecycle tests, and clear documentation of the trusted local-command model. If maintainers prefer externalization, first design a public TUI status/render plugin API, then move command rendering into a plugin.
