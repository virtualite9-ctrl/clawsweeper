## Summary

Current main has removed the reported v2026.3.14 WhatsApp startup path: the bundled WhatsApp entry is now a lightweight channel contract, the legacy broad `plugin-sdk/whatsapp` export is absent, startup plugin planning is scoped through the cold registry, and channel sidecars start after the HTTP listener is bound. The later #70533 remains the broader plugin-discovery heap report, but #48644's 4-10 minute WhatsApp/mega-chunk boot behavior is obsolete on main.

## Best Possible Solution

Close #48644 as implemented on main. Keep the current lightweight bundled-channel entry, scoped startup planning, and post-listen sidecar startup model; use #70533 or a new trace-backed issue for any remaining broad plugin-discovery heap/startup concern on current main.
