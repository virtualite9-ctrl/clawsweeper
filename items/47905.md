## Summary

Current main already limits bootstrap injection to a fixed allowlist of bootstrap filenames rather than all workspace `.md` files, and any operator-specific include/exclude policy fits the existing mutable `agent:bootstrap` hook/plugin surface instead of another core config key.

## Close Comment

Closing after a Codex review. Current `main` does not inject every workspace `.md` file into the system prompt: `src/agents/workspace.ts:503` only loads the fixed bootstrap set (`AGENTS.md`, `SOUL.md`, `TOOLS.md`, `IDENTITY.md`, `USER.md`, `HEARTBEAT.md`, `BOOTSTRAP.md`, and optional `MEMORY.md`), and the same behavior is documented in `docs/reference/token-use.md:24` and `docs/concepts/agent-workspace.md:119`. If you want an operator-specific include/exclude policy beyond that fixed set, OpenClaw already exposes a mutable `agent:bootstrap` hook surface (`src/hooks/internal-hooks.ts:23`, `docs/automation/hooks.md:123`) so a plugin or hook can filter `context.bootstrapFiles` without adding another core config knob. Per `VISION.md:54`, optional capability like this belongs in plugin/ClawHub space rather than expanding core config.
