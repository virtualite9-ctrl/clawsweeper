## Summary

Keep this issue open. Current main still has the stale Control UI model-selection paths: explicit Agents reload still calls the ordinary config loader, the loader does not clear dirty form state before applying the server snapshot, `applyConfigSnapshot` still skips replacing `configForm` while dirty, and the Agents model options still render without option-level selected binding. The provided GitHub context also shows open unmerged closing PRs #40443 and #52948, so this issue should stay open until a canonical fix lands or those PRs are resolved.

## Best Possible Solution

Keep this issue open and review/land a canonical Control UI fix. The likely product path is to separate passive config refresh from explicit Reload Config, discard stale dirty form state only for explicit reload actions, wire the Agents/Config/Channels reload buttons through that explicit path, add option-level selected bindings where Lit select rendering can show a stale option, and cover the Agents model dropdown with regression tests for initial render and Reload Config.
