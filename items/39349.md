## Summary

Current `main` still suppresses an explicit `pty:true` request whenever the exec tool runs through sandbox routing, and I found no current warning, test, or docs change that makes PR #39349 obsolete. The linked bug #38601 remains the same product gap, so this PR should stay open or be replaced by an equivalent narrow fix with a regression test.

## Best Possible Solution

Keep #39349 open or land an equivalent narrow fix on `main`: add the warning on the existing exec `warnings` path when `params.pty === true && sandbox`, and add a focused regression test proving the warning appears for sandbox-suppressed PTY and does not appear otherwise.
