## Summary

Keep open. Current main still routes the CLI picker and gateway models.list catalog through the full model catalog path, while the provided context shows open same-author PR #64953 as an active implementation candidate for this issue.

## Best Possible Solution

Keep this issue open and review PR #64953, or replace it with an equivalent maintainer fix. The fix should make promptDefaultModel(), promptModelAllowlist(), and the gateway models.list catalog use configured provider models when models.mode is replace, preserve default merge behavior, and add focused regression tests for the CLI and gateway surfaces.
