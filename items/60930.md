## Summary

Keep open. Current main still has an intentional seq-gap -> reconnect -> `chat.history` recovery path for Control UI/webchat, so there is not strong evidence that #60930 is already fixed or obsolete.

## Evidence

- **Gateway preserves per-client sequence gaps when slow websocket events are dropped:** `createGatewayBroadcaster()` increments the client's seq counter even when `dropIfSlow` skips delivery, so a slow Control UI/webchat client can observe a sequence hole without the server closing the socket for that event.

  - command: `nl -ba src/gateway/server-broadcast.ts | sed -n '100,155p'`
  - sha: [6bc0dc8fb671](https://github.com/openclaw/openclaw/commit/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440)
- **There is an explicit regression test asserting seq gaps are preserved on dropped broadcasts:** The test named `preserves seq gaps when dropIfSlow skips an eligible broadcast` shows this is current expected behavior, not a removed/obsolete path.
  - file: [src/gateway/gateway-misc.test.ts:450](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/src/gateway/gateway-misc.test.ts#L450)
