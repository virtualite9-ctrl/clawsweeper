## Summary

Keep open. Current `main` does not implement workspace-based `agentId` inference for `cron add`; the CLI only forwards an explicit `--agent`, and cron execution still falls back to the default agent when no job `agentId` is persisted.

## Evidence

- **Cron add still requires explicit --agent:** `registerCronAddCommand` reads `opts.agent` and sets `agentId` only when that flag is provided; otherwise it leaves `agentId` undefined. There is no workspace/config lookup in the current code.
  - file: [src/cli/cron-cli/register.cron-add.ts:127](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/cli/cron-cli/register.cron-add.ts#L127)
  - command: `nl -ba src/cli/cron-cli/register.cron-add.ts | sed -n '118,140p'`
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Cron execution still defaults missing agentId to the default agent:** `prepareCronRunContext` resolves the run agent from `input.agentId` or `input.job.agentId`, then falls back to `resolveDefaultAgentId(input.cfg)`. The chosen `agentId` also drives workspace and agent-dir resolution.
  - file: [src/cron/isolated-agent/run.ts:427](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/cron/isolated-agent/run.ts#L427)
  - command: `nl -ba src/cron/isolated-agent/run.ts | sed -n '424,476p'`
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **CLI tests cover only explicit agent forwarding:** The current cron CLI test suite asserts that `--agent ops` is sent on add, but there is no corresponding test for inferring `agentId` from the current workspace when `--agent` is omitted.
  - file: [src/cli/cron-cli.test.ts:455](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/cli/cron-cli.test.ts#L455)
  - command: `nl -ba src/cli/cron-cli.test.ts | sed -n '452,478p'`
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Workspace-to-agent helper exists but is not wired into cron add:** `resolveAgentIdByWorkspacePath` is available in the repo, which matches the PR's proposed approach, but current `cron add` does not call it.
  - file: [src/agents/agent-scope.ts:228](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/agents/agent-scope.ts#L228)
