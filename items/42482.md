## Summary

Keep this PR open. Current main has already shipped the deferred isolated-cron execution-timeout portion, but the remaining abort/fallback behavior is still not implemented in main: the cron fallback callback still forwards the same outer abort signal into each CLI and embedded attempt. The PR is stale and should be rebased or split, but the discussion contains useful field reports and the remaining problem is not fully superseded by current main.

## Best Possible Solution

Keep this PR, or a canonical replacement, open. The best maintainer path is to rebase or split it: drop the deferred-timeout changes already present on main, implement the remaining cron fallback abort/budget behavior against src/cron/isolated-agent/run-executor.ts and the shared model-fallback contract, and add a regression test proving later cron fallback candidates are not killed by a stale retryable attempt signal. Reconcile that design with #52365 and #62682 before landing.
