## Summary

Keep this PR open. Current main now implements the deferred isolated cron timeout portion from #41783, but it still does not implement the PR's per-attempt AbortController behavior: isolated cron fallback attempts still reuse the same outer abort signal. The useful field report and related repro context in the discussion support keeping a rebased or replacement fix alive rather than cleanup-closing this PR.

## Best Possible Solution

Keep this PR, or a canonical replacement, open. The best maintainer path is to rebase/split it against current main: drop or align the already-landed #41783 timeout deferral with 729147dc, implement the remaining abort/fallback behavior in src/cron/isolated-agent/run-executor.ts or the shared model-fallback contract, and add a regression test proving isolated cron fallback attempts do not reuse an already-aborted signal when a retryable attempt-level timeout should advance to the next candidate. Reconcile this with open abort semantics work such as #62682 before landing.
