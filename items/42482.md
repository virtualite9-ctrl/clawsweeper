## Summary

Keep this PR open. Current main still lacks both central fixes: cron timeout accounting is still armed before isolated execution begins, and the isolated cron fallback callback still passes the same parent abort signal into every model attempt. The PR discussion also contains concrete field confirmation from affected users, and the related narrower timeout PR was closed in favor of this broader approach rather than merged.

## Best Possible Solution

Keep this PR or a canonical replacement open. The best maintainer path is to rebase the implementation onto current main, add per-attempt abort-controller isolation in `src/cron/isolated-agent/run-executor.ts`, defer or split cron timeout accounting in `src/cron/service/timer.ts`, and add regression tests proving queued isolated execution does not consume execution budget and fallback attempts receive fresh non-aborted signals.
