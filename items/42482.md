## Summary

Keep this PR open. Current main has implemented the deferred isolated-cron execution timeout portion, but the per-attempt AbortController behavior remains missing: cron model fallback attempts still pass the same outer abort signal into each CLI/embedded attempt. The PR is stale and should be rebased or split, but it still tracks useful remaining work supported by the linked field reports and repro context.

## Best Possible Solution

Keep this PR, or a canonical replacement, open. The best maintainer path is to rebase or split it: drop the deferred-timeout portion already present on main, implement fresh per-attempt abort/budget handling in the current cron fallback path or shared model-fallback contract, and add a regression test proving isolated cron fallback attempts do not reuse an already-aborted signal when a retryable attempt-level timeout should advance to the next candidate. Reconcile the design with the related open abort-semantics work before landing.
