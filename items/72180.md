## Summary

Current main already preserves generated image media with the assistant's visible text and sends Telegram image replies with the text as the media caption, with long-caption fallback to a separate text message. The reporter was on 2026.4.23; the relevant Telegram caption path is present in the v2026.4.24 release tag and current main.

## Best Possible Solution

Close as implemented and keep the shipped/current Telegram media-caption delivery path. Users still seeing this after upgrading should file a fresh issue with raw reply payload/log evidence showing the Telegram payload lost text despite carrying media.
