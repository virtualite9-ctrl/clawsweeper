## Summary

Keep this issue open. Current main has implemented heartbeat-aware daily/idle freshness, but it still does not provide a configurable turn-count TTL or equivalent max-turn rollover for sessions. The related same-author #45390 overlaps broader session TTL/max-lifetime work, but it is open and not a maintainer-designated canonical replacement for this narrower request.

## Best Possible Solution

Keep this open as a core session lifecycle feature unless a maintainer explicitly consolidates it into #45390. A complete implementation should add a schema-validated and documented max-turn reset policy, define which turns count versus heartbeat/system events, wire the check into automatic session rollover before reuse, and add regression coverage for daily, idle, heartbeat, and max-turn interactions.
