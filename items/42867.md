## Summary

Keep this issue open. Current main has fixed the heartbeat-aware staleness half by using session start/interaction timestamps instead of generic `updatedAt`, but it still has no configurable `session.reset.maxTurns`, turn-count TTL, or equivalent automatic rollover policy. The overlap with open #45390 is real, but that same-author broader tracker does not make this narrower request safe to close automatically.

## Best Possible Solution

Keep this open as a core session-lifecycle feature unless a maintainer explicitly consolidates it into #45390. A complete fix should add a schema-validated and documented turn-count reset policy, wire it into automatic session rollover, preserve the existing heartbeat-aware freshness behavior, and add regression coverage for daily, idle, heartbeat, and max-turn interactions.
