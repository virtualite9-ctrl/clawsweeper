## Summary

Current `main` does not implement this PR’s requested behavior. Memory indexing still recognizes Markdown and multimodal files only, the storage contract exports `chunkMarkdown` but no code-aware chunker, embedding/indexing still chunks text via `chunkMarkdown(...)` only, and watcher coverage is still Markdown-focused. This is not `implemented_on_main`; it remains open work. The PR also appears to target the pre-refactor `src/memory/*` layout, so it likely needs a port to the current `extensions/memory-core` + `src/memory-host-sdk` split rather than closure.
