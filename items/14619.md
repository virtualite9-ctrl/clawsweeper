## Summary

Keep #14619 open. Current main still renders a human-readable `## Tooling` section with tool names and summaries while separately serializing callable tool definitions with descriptions and parameter schemas. The only nearby user-facing config is a full `systemPromptOverride`, not a scoped option to omit just the in-band tool list text.

## Best Possible Solution

Keep #14619 open until OpenClaw adds a scoped agent/default config such as `skipToolListText` that omits only the human-readable Tooling list while preserving structured tool schemas. A complete fix should update the prompt builder and embedded runner plumbing, config types/zod/generated schema/help/docs, `/context` reporting if needed, and focused tests proving schemas remain available while the in-band list is omitted. If maintainers decide `systemPromptOverride` is the intended boundary, that should be stated explicitly on the issue.
