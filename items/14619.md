## Summary

Keep #14619 open. Current main still builds a human-readable `## Tooling` section from active tool names/summaries and separately registers structured tool definitions with descriptions and parameter schemas. I found no scoped `skipToolListText` or equivalent config; `systemPromptOverride` only replaces the whole generated prompt.

## Best Possible Solution

Keep the issue open until OpenClaw adds a scoped agent/default config such as `skipToolListText` that suppresses only the human-readable Tooling list while preserving structured tool schemas. The implementation should update the prompt builder/run plumbing, config types/zod/generated schema/help/docs, `/context` reporting, and focused tests for both prompt omission and schema availability.
