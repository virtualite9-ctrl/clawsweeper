## Summary

Keep #14619 open. Current main still renders a human-readable `## Tooling` tool list in the system prompt, still sends callable tool schemas separately, and still has no scoped `skipToolListText` or equivalent config option. The April 26 maintainer-side Codex review in the issue reached the same conclusion, and the current checkout at `406ae72fd2789c78053f6b94006ec8f488bc1c00` still matches that behavior.

## Best Possible Solution

Keep #14619 open until OpenClaw adds a scoped agent/default option, for example `agents.defaults.skipToolListText`, that suppresses only the human-readable Tooling list while preserving callable tool schemas. The implementation should update prompt construction, run/config plumbing, config types/Zod/generated schema/help/docs, `/context` accounting, and focused tests proving the prompt text is omitted while tools remain available.
