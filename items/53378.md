## Summary

Current main still does not satisfy the requested Control UI Sessions list channel identity. The recent main-branch change at 14ab0075 only added agent identity labels in the Sessions key cell; it does not render a channel column/badge/icon/name, does not use gateway channel/origin metadata, and intentionally keeps raw keys when agent identity data is unavailable.

## Best Possible Solution

Keep this issue open. Implement the Sessions view feature by adding channel/origin fields to the UI `GatewaySessionRow` contract and `SESSION_EVENT_ROW_FIELDS`, then render a compact Channel column or key-cell badge using gateway `row.channel` / `row.origin` and channel labels or channel meta, with session-key parsing only as a fallback. Add focused Control UI tests for Telegram and Feishu direct/group rows, preserving the existing editable Label column and recent agent-identity behavior.

## Risks / Open Questions

- A narrow patch that only parses `row.key` would miss provider-owned metadata and cases where the gateway already supplies `channel` / `origin`; the UI type and websocket allowlist should carry those fields too.
- The existing chat dropdown label helper has a limited hard-coded channel label map and does not cover Feishu, so the Sessions view should prefer gateway channel metadata / `channelLabels` or shared channel meta rather than copying that map.
