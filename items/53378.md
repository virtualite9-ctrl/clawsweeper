## Summary

Keep open. Current main has pieces of the solution, including editable custom session labels and backend session-channel metadata, but the Control UI Sessions table still renders the raw session key as the primary entry and does not show a channel icon/name column or derived channel label. The related ClawSweeper context for #53012/#52323 supports that friendly labels/displayName work was handled elsewhere, but it does not cover this issue's remaining channel-identification request in the Sessions list.

## Best Possible Solution

Implement this in the Control UI Sessions page: derive a stable channel display from `GatewaySessionRow.channel`/`origin.provider`/session-key parsing, show it as a compact channel name/icon or badge near the key, keep the existing editable Label field, and add focused `ui/src/ui/views/sessions.test.ts` coverage for Telegram and Feishu direct/group rows. If direct sessions still lack reliable channel metadata from the API, add a small backwards-compatible UI helper or typed row field rather than closing this issue.
