## Summary

Current main still does not show a channel icon/name in the Control UI Sessions table. The table is built around the raw session key, optional custom label, kind badge, and optional displayName; related friendly session-name logic is limited to the chat dropdown path and is not wired into the Sessions list.

## Best Possible Solution

Keep this issue open. Implement the feature in the Control UI Sessions page by deriving a stable channel display from GatewaySessionRow.channel, origin/provider metadata, and session-key parsing; render a compact Channel column or key-cell badge with channel name/icon; preserve the editable Label field; align the UI row type and live-update allowlist with backend channel/origin metadata; add focused Sessions view coverage for Telegram and Feishu direct/group rows.
