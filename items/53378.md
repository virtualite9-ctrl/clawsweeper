## Summary

Current main still does not implement channel identity in the Control UI Sessions list. The table has no Channel column/badge/icon/name, and the key cell only shows agent identity when available, with a parsed lowercase channel id in parentheses. Telegram/Feishu rows without agent identity still fall back to raw session keys, while backend session rows already expose channel/origin metadata that the Sessions view does not type, preserve in live updates, or render.

## Best Possible Solution

Keep this issue open. Implement the Sessions list feature in the Control UI by carrying `channel`, `groupChannel`, `chatType`, and `origin` through the UI `GatewaySessionRow` type and `SESSION_EVENT_ROW_FIELDS`, passing channel labels/meta from the existing channels snapshot into `renderSessions`, and rendering a compact Channel column or key-cell badge for Telegram/Feishu direct and group rows while preserving the existing editable Label column and agent-identity behavior. Add focused UI tests for Telegram and Feishu rows without agent identity, plus an update-event test that preserves channel metadata.
