## Summary

Current `main` already has automatic sub-agent completion announcements, a direct-to-queue fallback when the requester session is busy or direct delivery fails, and Feishu-specific completion routing back to the original conversation/thread. That covers the silent-completion problem described in #53661 well enough to close as implemented on main.

## Evidence

- **Sub-agent docs now describe automatic resilient completion delivery:** `/subagents spawn` is documented as sending a final completion update back to the requester chat, trying direct delivery first, then queue routing, while preserving the requester route.
  - file: [docs/tools/subagents.md:40](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/docs/tools/subagents.md#L40)

## Risks / Open Questions

- I could not determine a released tag that definitively shipped this behavior from the available checkout/history, so the release field is left null even though current `main` clearly contains the implementation.
- `sessions_send` itself still has its own timeout semantics; this close only covers the issue's requested outcome for sub-agent completion notification, which now uses the dedicated announce path.

## Close Comment

Codex review on current `main` found that this is already implemented. Sub-agent completions now use the dedicated announce flow in `src/agents/subagent-announce.ts:526`, not a fragile caller-specific `sessions_send` pattern. That flow resolves a completion delivery origin, sends directly when possible, and falls back to queue routing when direct delivery fails (`src/agents/subagent-announce-dispatch.ts:42`, `src/agents/subagent-announce-delivery.ts:362`). Feishu also has channel-specific completion routing hooks (`extensions/feishu/subagent-hooks-api.ts:12`, `extensions/feishu/src/subagent-hooks.ts:359`) and tests that preserve the original Feishu DM/topic target on completion (`extensions/feishu/src/subagent-hooks.test.ts:30`). The current docs also describe automatic resilient completion delivery for sub-agents in `docs/tools/subagents.md:40`. Closing as implemented on `main`. If you can still reproduce a Feishu completion notification miss on a current build, please open a new bug with the exact Feishu target shape, account setup, and relevant gateway logs.
