## Summary

Keep PR #39365 open. Current main has the `openclaw system event` CLI/runtime surface and partial reference docs, but it still does not include the PR's dedicated external-script recipes page, Automation nav entry, or CLI-doc cross-link. The PR remains useful docs work, but should be revised before merge because review comments identify unsafe and misleading recipe text.

## Best Possible Solution

Keep #39365 open as the canonical docs PR, or replace it with an equivalent maintainer-authored docs change. The best version should add a safe `docs/automation/system-events.md` recipes page, add the docs nav entry, cross-link from `docs/cli/system.md` and relevant heartbeat/webhook docs, explain that writing files alone does not wake the agent, recommend `openclaw system event --mode now` for external scripts, and correct the unsafe or misleading recipe text called out in review.
