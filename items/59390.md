## Summary

Close #59390 as superseded by the focused open PR #68397. Current main still has the reported behavior, but #68397 is the canonical implementation tracker for this exact sessions_spawn runtime=subagent plus ACP-only streamTo/resumeSessionId failure mode.

## Best Possible Solution

Use #68397 as the canonical implementation tracker for the immediate bug: land an equivalent change that silently strips or ignores ACP-only streamTo and resumeSessionId for runtime=subagent while preserving ACP pass-through and regression tests. Keep the broader runtime/per-agent schema filtering discussion on #59225.

## Evidence

The executor then resolves non-ACP calls to runtime=subagent and reads streamTo/resumeSessionId from the same args object.
