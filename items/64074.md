## Summary

Current OpenClaw already supports arbitrary custom `models.providers.<id>` entries for OpenAI-compatible endpoints, and Alibaba/Qwen MaaS is also covered by the bundled `qwen` provider docs and onboarding. The requested capability is present on `main` and was already shipped in `v2026.4.23`.

## Evidence

- **Config schema accepts arbitrary provider ids:** `ModelsConfigSchema` defines `providers` as `z.record(z.string(), ModelProviderSchema)`, so provider ids are not hardcoded and keys like `aliyun` are structurally allowed.
  - file: [src/config/zod-schema.core.ts:375](https://github.com/openclaw/openclaw/blob/5b8bd6371c666a5de43c82305e0315771e8e49d5/src/config/zod-schema.core.ts#L375)

  - file: [docs/gateway/config-tools.md:356](https://github.com/openclaw/openclaw/blob/5b8bd6371c666a5de43c82305e0315771e8e49d5/docs/gateway/config-tools.md#L356)
