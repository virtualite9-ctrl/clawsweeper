## Summary

Keep #26517 open. Current main has partial groundwork: sessions can carry labels, the chat selector can switch among existing sessions, and the gateway supports `sessions.create`. The requested WebChat create/manage flow and the proposed agent-facing session create/relabel API are still not shipped, and a maintainer comment on the issue already reached the same conclusion.

## Best Possible Solution

Keep #26517 open until maintainers either implement or explicitly narrow the WebChat product scope. The likely path is to reuse the existing `sessions.create` and `sessions.patch` backend support, add chat UI affordances to create, label/edit, switch, and manage isolated per-agent sessions, and add reviewed agent-facing session create/relabel semantics only if maintainers still want agents to control those labels.
