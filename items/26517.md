## Summary

Keep #26517 open. Current main has backend support for creating and labeling session entries and the WebChat header can display/switch among existing labeled sessions, but the requested WebChat create/manage flow and agent-facing session label/create API are still not implemented. The two related PRs that attempted the UI work were closed unmerged, and a maintainer comment already called out the same remaining gap.

## Best Possible Solution

Keep this issue open until maintainers either implement a WebChat flow that creates, labels, switches, and manages multiple per-agent sessions using the existing sessions.create/sessions.patch backend, or explicitly narrow the product scope. The closed PRs #52042 and #52746 can inform the UI design, but a landed implementation and tests are still needed.
