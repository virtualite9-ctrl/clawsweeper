## Summary

Keep #26517 open. Current main has useful groundwork for labeled session rows and WebChat session switching, but the requested WebChat create/manage flow and agent-facing session label/create API are still missing. The latest provided maintainer comment also explicitly kept this issue open for that reason.

## Best Possible Solution

Keep #26517 open until maintainers implement a WebChat flow to create, label, switch, and manage isolated per-agent sessions using the existing sessions.create/sessions.patch backend, or explicitly narrow the product scope. If agent-controlled labels remain desired, add a reviewed agent tool/API such as sessions_create/session_label with permission, sandbox, and transcript tests rather than treating sessions_spawn labels as a substitute.
