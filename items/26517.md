## Summary

Keep #26517 open. Current main still has labeled dashboard-session backend support and a switch-only WebChat session selector, but the visible WebChat “New session” control still sends bare `/new`, which resets the current session instead of creating and switching to an isolated labeled session. The proposed agent-facing create/relabel API is also not shipped. The related UI implementation PRs #52042 and #52746 are closed unmerged, while #51793 was closed as duplicate/superseded in favor of this issue.

## Best Possible Solution

Keep #26517 open as the canonical tracker. The likely implementation should reuse existing `sessions.create` and `sessions.patch` backend support, add WebChat create/switch/label management for isolated labeled sessions, and only add agent-controlled create/relabel semantics after explicit maintainer review of the product and safety boundary.
