## Summary

Keep open. Current main still preserves the reported behavior: update refreshes a loaded gateway by forcing a gateway service reinstall with the caller environment, the service environment builder explicitly copies HTTP_PROXY/HTTPS_PROXY and related proxy variables, and systemd renders them as Environment= lines. The new issue comment also points to a second current failure path: Feishu WebSocket client creation still injects an ambient ProxyAgent whenever HTTP(S) proxy env is present, with no Feishu-specific opt-out or NO_PROXY/domain check at that call site.

## Best Possible Solution

Keep the issue open for a maintainer fix. The likely product path is to distinguish explicit, durable service proxy configuration from transient ambient shell/launchd proxy state: do not bake HTTP_PROXY/HTTPS_PROXY from openclaw update service refresh into systemd/launchd by default, preserve only deliberate existing/custom proxy config, and add regression tests for update-driven service refresh. Separately, Feishu should either honor NO_PROXY/domain bypass before creating the ambient ProxyAgent or expose a Feishu channel-level opt-out for ambient proxy use.
