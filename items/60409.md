## Summary

Keep this PR open. Current main still has no reset-archive fallback in `readSessionMessages`, while reset continues to archive transcripts as `.jsonl.reset.<timestamp>` and the affected gateway history surfaces still call that helper. The PR head is a focused implementation candidate, and the related open PRs are adjacent rather than superseding this fallback.

## Best Possible Solution

Keep this PR open as the focused implementation candidate for reading the latest reset archive when `readSessionMessages` is asked for a session whose primary transcript is missing. Maintainers should review/rebase it, run the gateway changed checks, and either land this patch or fold equivalent fallback behavior into a broader reset-archive recovery change while leaving UI/archive listing and logical-chain metadata to the related follow-ups.
