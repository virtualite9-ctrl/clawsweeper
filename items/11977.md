## Summary

Keep #11977 open. Current main still builds and publishes Docker release images only for linux/amd64 and linux/arm64, and the multi-platform manifest still assembles only those two digests. The related riscv64 PR #12033 is useful prior art but was closed unmerged, so it does not implement or supersede the issue.

## Best Possible Solution

Keep #11977 open until a maintainer lands a Docker release workflow change that adds a non-blocking linux/riscv64 image build, includes the riscv64 digest in the multi-platform manifest when available, handles Dockerfile prerequisites on riscv64, and updates release validation or Docker docs as needed. Treat closed PR #12033 as prior art, not as a canonical superseding item.
