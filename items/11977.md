## Summary

Keep #11977 open. Current main still publishes Docker release images only for linux/amd64 and linux/arm64, and the multi-platform manifest is built only from those two digests. The related PR #12033 is useful prior work, but the provided GitHub context shows it was closed unmerged, and no equivalent riscv64 release path is present on main.

## Best Possible Solution

Keep #11977 open until a maintainer lands a Docker release workflow change that adds a non-blocking linux/riscv64 build, includes the riscv64 digest in the multi-arch manifest when available, handles Dockerfile prerequisites such as Bun/native-addon behavior on riscv64, and updates Docker docs or release validation. Closed PR #12033 remains useful prior art but does not supersede the issue because it was not merged.
