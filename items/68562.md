## Summary

Keep open. Current main still has the exact request-scoped cleanup warning path: the start path handles RequestScopedSubagentRuntimeError by writing fallback narrative content and returning null, but the finally block still calls deleteSession and logs any cleanup error. A newer duplicate, #72099, was already closed in favor of this issue, so this remains the canonical tracker rather than a close candidate.

## Best Possible Solution

Keep this issue open as the canonical request-scoped warning bug. Patch `extensions/memory-core/src/dreaming-narrative.ts` so cleanup is skipped when no runId was created, and suppress RequestScopedSubagentRuntimeError in the cleanup catch for symmetry with the start fallback path. Add a regression test proving request-scoped fallback writes the local diary entry without calling deleteSession or emitting the cleanup warning.
