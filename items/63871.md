## Summary

Keep open. Current main still has the reported cron behavior: `deleteAfterRun` only deletes successful `kind: "at"` jobs, so explicit `deleteAfterRun: true` on recurring `every` or `cron` jobs is still ignored. This PR directly fixes the central bug linked from #63770, but it also carries unrelated workflow/test-sync changes with unresolved review comments, so it needs cleanup or maintainer selection against #64375 rather than closure as obsolete.

## Best Possible Solution

Keep this PR open as an implementation candidate, but do not merge it as-is. The best maintainer path is to land a narrow cron fix that removes the at-only guard in `src/cron/service/timer.ts`, keeps focused regression tests for `every` and `cron` success deletion plus failure/skipped retention, and either trims this branch’s unrelated workflow/outbound test-sync changes or selects #64375 as the cleaner canonical PR.
