## Summary

Keep open. Current main mitigates the provider replay rejection by filtering `openclaw`/`delivery-mirror` assistant rows out of in-memory replay, but the reported raw transcript write still exists: outbound delivery still builds a mirror payload and persists it as an assistant `delivery-mirror` transcript row. I found no config toggle or alternation guard that disables or skips the mirror write itself.

## Best Possible Solution

Keep the issue open. Preserve the current replay filter, then add an explicit delivery-mirror transcript policy in the outbound/transcript path: either a config-backed disable option or a guard that skips/retypes mirror entries when appending would create an assistant-to-assistant raw transcript sequence. Add focused regression coverage for message-tool outbound delivery from cron/subagent-like sessions plus Anthropic replay staying clean.
