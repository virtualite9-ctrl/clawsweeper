## Summary

Keep open. Current main has general token-usage alias normalization and `/status` transcript fallback, but I did not find a Volcengine/Ark-specific fix for the reported provider path. The issue discussion also says the likely missing piece is Volcengine not returning usage in streaming chunks, and the reporter confirmed both suggested workarounds failed.

## Best Possible Solution

Keep this issue open as a Volcengine provider bug. The best next step is to confirm Ark coding-plan streaming and non-streaming usage behavior against `volcengine-plan/ark-code-latest`, then add a focused provider fix: either mark Volcengine/volces endpoints with the correct streaming-usage capability if they support usage chunks, add a Volcengine-specific response/stream usage extraction path if Ark returns usage differently, or document a provider limitation if Ark exposes no per-request token usage. Add regression coverage around `volcengine-plan` usage persistence and `/status` rendering.
