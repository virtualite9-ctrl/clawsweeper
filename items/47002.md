## Summary

Keep open. Current main has fixed the related explicit CLI-media failure by deriving roots from concrete media sources, but the specific validator/config gap in #47002 remains: Telegram config and agent defaults are strict schemas that still do not define mediaLocalRoots, while the Telegram outbound runtime does consume injected mediaLocalRoots when core provides them.

## Best Possible Solution

Keep this open and add a schema-backed, documented outbound media allowlist if maintainers still want persistent roots. The likely core path is a generic config surface merged by src/media/local-roots.ts, with strict validation for absolute/narrow roots and regression coverage proving the chosen config key validates and reaches Telegram outbound sends. If maintainers decide source-based expansion is the intended replacement, document that explicitly and remove or clarify any stale type/API references instead of silently closing this report.
