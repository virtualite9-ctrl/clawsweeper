## Summary

Keep #9443 open. Current main still does not provide official prebuilt Android APK downloads in GitHub releases: public Android docs say the app is not publicly released, the Android README remains alpha/local-build oriented, CI only builds a Play debug APK for validation, the release helper builds signed AAB bundles rather than APK release assets, and the latest release has no Android APK assets. Related duplicate #35515 is closed back to this issue, so #9443 remains the canonical tracker.

## Best Possible Solution

Keep #9443 open as the canonical official Android APK release request. A complete fix should define the signed APK/flavor policy, add release automation that builds on tag or release and uploads APK assets to GitHub releases, update Android install docs, and add release checks that match the Android app readiness path. New duplicate APK requests should point back to #9443.
