## Summary

Keep #9443 open. Current main still does not provide official prebuilt Android APK downloads in GitHub releases. The Android docs explicitly say the app is not publicly released, the Android README remains local-build/release-hardening oriented, CI only builds a Play debug APK, the release helper builds signed AAB bundles, and the latest GitHub release has only macOS assets. Community APK/client links in the discussion are useful workarounds, but they do not satisfy the request for official OpenClaw APK release assets.

## Best Possible Solution

Keep #9443 open as the canonical official Android APK release request. A complete fix should define the signed APK and flavor policy, add maintainer-approved tag/release automation that builds and uploads official APK assets to GitHub releases, update https://docs.openclaw.ai/platforms/android with the official install path, and keep duplicates redirected here until that ships or maintainers explicitly decide against official APK distribution.
