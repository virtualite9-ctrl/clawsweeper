## Summary

Keep #69541 open. Current main has several point-fix sanitizers for known injected blocks, but it does not implement the requested declarative plugin manifest registry or a core getter that downstream consumers can share. The remaining work is an actionable plugin/core contract change, not ClawHub-only optional plugin work.

## Best Possible Solution

Keep the RFC open and design an additive plugin contract: extend `PluginManifestContracts` with a validated injected-context declaration, expose an aggregated read API for enabled/loaded plugins, and migrate display/history, assistant-visible text, compaction, dreaming ingestion, and memory retain paths to consume that registry while keeping existing hardcoded tags as compatibility defaults during rollout.

## Risks / Open Questions

- A registry-based stripper must validate patterns tightly so a plugin cannot accidentally or maliciously strip ordinary user-authored text from history, compaction, or memory ingestion.
- Current point fixes cover some known tags, so implementation should preserve those behaviors while moving the source of truth into a shared manifest/runtime contract.
