## Summary

Keep #69541 open. Current main has adjacent prompt-injection hooks and several hardcoded/point sanitizers, but it does not implement the requested manifest-level `contracts.injectedContextTags`, a runtime injected-context registry/getter, or scope-aware consumer migration. The issue is recent, contributor-authored, unlabelled, and has no comments; related RFCs/issues are adjacent rather than a canonical replacement.

## Best Possible Solution

Keep the RFC open and handle it as a core/plugin-contract design item. A good implementation would add a backwards-compatible `contracts.injectedContextTags` manifest type plus loader validation, expose an aggregated runtime helper, and migrate consumers incrementally starting with memory-core dreaming ingestion and display/session sanitizers while retaining existing hardcoded defaults as fallback.
