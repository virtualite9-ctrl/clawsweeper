## Summary

Keep #47441 open. Current main still routes media image processing through the bundled media-understanding-core plugin's sharp dependency, the lockfile still has no android-arm64 sharp/libvips optional package, and the updater only has a generic npm --omit=optional fallback rather than a Termux-aware rebuild or raw-image fallback. The report is recent, concrete, and not superseded by #20349 because that related issue was about koffi startup failure, not sharp image processing.

## Best Possible Solution

Keep the issue open for maintainer product judgment on Termux gateway support. If Termux is supported, add a focused fix such as Termux-aware sharp/native-module repair in update/doctor, or a media image pipeline fallback that preserves usable raw image input when sharp cannot load, with tests and docs. If Termux gateway is intentionally unsupported, document that explicitly and add a diagnostic that points users away from this fragile native-module path.
