## Summary

Keep open. Current `main` still does not implement the requested Feishu WebSocket recovery loop: websocket mode still delegates directly to a single `monitorWebSocket()` call, and that transport still creates one `WSClient`, calls `start()` once, and then waits for abort without any OpenClaw-managed reconnect/backoff/watchdog path after the SDK stops making progress.
