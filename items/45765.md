## Summary

Current main does not implement the requested behavior. Both current path-resolution paths still treat OPENCLAW_HOME as a replacement home directory and append .openclaw beneath it, and the existing tests/docs encode that contract. The related fix PRs mentioned in the issue context were closed unmerged, with no matching fix found in local main history, so this should remain open for maintainer product/API judgment rather than be auto-closed.

## Best Possible Solution

Keep #45765 open. The best path is for a maintainer to decide the intended OPENCLAW_HOME contract. If the issue behavior is desired, add a narrow guard in both src/config/paths.ts and src/utils.ts for explicit OPENCLAW_HOME values whose basename is a known state directory name, add regression tests for /.../.openclaw and ~/.openclaw, and update the environment docs. If the current contract is intentional, close manually with docs-backed guidance to use OPENCLAW_STATE_DIR=/home/tpam/.openclaw for direct state-dir placement.
