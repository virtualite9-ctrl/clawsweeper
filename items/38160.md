## Summary

Keep open. Current main already supports hosted-to-local fallback through the normal model fallback chain, but it does not implement this PR's dedicated localModelFallback subsystem or semantic response cache. The PR also has unresolved review-blocking correctness issues in the provided discussion, so there is no conservative auto-close reason.

## Best Possible Solution

Keep the PR open for maintainer product/architecture review rather than auto-closing. The likely path is to split it: use or document existing agents.defaults.model.fallbacks for hosted-to-local fallback unless a concrete missing API is identified, and design any semantic response cache against the current memory/embedding architecture with guarded cache failures, correct persisted vector decoding, telemetry-preserving fallback attempts, docs, and focused tests.
