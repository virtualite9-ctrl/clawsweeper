## Summary

Keep PR #38160 open. Current main has a generic model fallback path that can be configured for hosted-to-local fallback, but it does not implement the PR's dedicated `localModelFallback` config or the semantic prompt-to-response cache. The PR also still has unresolved correctness review findings, so there is no conservative close reason.

## Best Possible Solution

Keep PR #38160 open for maintainer architecture review, but split it before merge. Prefer using or documenting the existing `agents.defaults.model.fallbacks` path for hosted-to-local fallback unless a concrete missing API remains. Redesign the semantic response cache with fail-open lookup/storage, correct vector persistence, explicit provider/base URL/auth handling, telemetry preservation, cache scope/invalidation/privacy rules, docs, and focused tests. Move the unrelated onboarding finalize change to its own PR.
