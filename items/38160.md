## Summary

Keep PR #38160 open. Current main overlaps the local-fallback half through generic `agents.defaults.model.fallbacks`, including Ollama/local models, but it does not implement this PR's dedicated `localModelFallback` config or semantic prompt-to-response cache. The PR also still has material correctness review findings, so there is no conservative close reason.

## Best Possible Solution

Keep the PR open for maintainer architecture review, but split it before any merge. Prefer documenting or extending the existing `agents.defaults.model.fallbacks` path for hosted-to-local fallback unless a concrete missing API remains. Redesign the semantic response cache as a separate proposal with fail-open behavior, correct vector persistence, provider/base URL/auth handling, cache scope/invalidation/privacy rules, telemetry, docs, and focused tests.

## Risks / Open Questions

- Merging the PR as-is could make an optional semantic cache fail agent turns when the embedding backend or SQLite path fails instead of degrading to a cache miss.
- A response-level semantic cache needs explicit scope, invalidation, privacy, persistence, telemetry, and fail-open design before it is safe as core behavior.
- The PR bundles separate concerns: local fallback, semantic response cache, docs artifacts, and an unrelated onboarding finalize change.
