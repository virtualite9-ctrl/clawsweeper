## Summary

Keep PR #38160 open. Current main still overlaps the local-fallback part through the generic `agents.defaults.model.fallbacks` model-failover path, including local providers, but it does not implement the PR's dedicated `localModelFallback` config or the response-level semantic cache. The PR also has unresolved correctness review findings, so there is no conservative close reason.

## Best Possible Solution

Keep the PR open for maintainer architecture review, but split it before merge. Prefer documenting or extending the existing `agents.defaults.model.fallbacks` path for hosted-to-local fallback unless a specific missing API remains. Redesign the semantic response cache deliberately with fail-open cache errors, correct vector persistence, telemetry preservation, cache scope/invalidation rules, docs, and focused tests. Move the unrelated onboarding finalize change to its own PR.
