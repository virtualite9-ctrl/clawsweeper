## Summary

Keep open. Current `main` still awaits media understanding in `getReplyFromConfig` without a protective fallback, and the media-understanding stack still has throwing paths. The report remains actionable on current code.

## Evidence

- **Reply pipeline still has no fallback wrapper:** `applyMediaUnderstandingIfNeeded()` only checks for inbound media, loads the runtime, and awaits `applyMediaUnderstanding()`. In `getReplyFromConfig()`, the normal path awaits that helper directly before continuing to hooks/directives, with no surrounding `try/catch` or degraded text-only fallback.
  - file: [src/auto-reply/reply/get-reply.ts:131](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/src/auto-reply/reply/get-reply.ts#L131)
  - command: `nl -ba src/auto-reply/reply/get-reply.ts | sed -n '120,270p'`
  - sha: [926068b14fc6](https://github.com/openclaw/openclaw/commit/926068b14fc661e915a0247e9ecb5949fb95b4f7)
- **Current media-understanding code still has throwing cleanup/failure paths:** `applyMediaUnderstanding()` ends with `finally { await cache.cleanup(); }`, so cleanup exceptions still propagate out of the media-understanding call. That matches the issue's concern about unexpected exceptions escaping the reply pipeline.
  - file: [src/media-understanding/apply.ts:589](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/src/media-understanding/apply.ts#L589)

The asymmetry with `getReplyFromConfig()` makes the issue credible rather than obsolete.

## Risks / Open Questions

- Closing now would leave a current-main path where a media-understanding exception can still abort the whole reply flow before any degraded response is produced.
- Because the failure is unwrapped in `getReplyFromConfig()`, operators still lose the logging/fallback boundary that would make provider-agnostic diagnosis easier.
- There is no current proof in main that `getReplyFromConfig()` intentionally changed semantics here; the code still matches the reported failure mode.
