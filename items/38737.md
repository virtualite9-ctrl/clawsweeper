## Summary

Keep open. Current main has useful multi-agent primitives: deterministic bindings, agent-to-agent session access, isolated/nested subagents, async announce chains, sessions_send wait, and sessions_yield. It does not implement the requested first-class gateway policy that classifies a specialist-channel inbound task before that specialist model runs, escalates to a coordinator, synchronously collects/merges child results, and returns the coordinator answer as the specialist-channel reply. The related issues in the supplied GitHub context (#17324, #23885, #48784) are narrower agent/session delivery requests and do not supersede this coordinator auto-escalation request.
