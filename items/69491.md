## Summary

Keep PR #69491 open. Current main still does not implement Bedrock Anthropic beta forwarding: the Amazon Bedrock provider returns the original stream function for Anthropic Bedrock model IDs and only patches guardrails/cache points elsewhere. The PR addresses a real bundled-provider gap, but it should be amended before landing because the existing review comments identify concrete issues around preserving existing beta values and avoiding broad heuristic false positives.

## Best Possible Solution

Keep the PR open and ask for a tighter provider-local implementation in `extensions/amazon-bedrock/register.sync.runtime.ts`: inject Bedrock Converse `additionalModelRequestFields.anthropic_beta` for definitive Anthropic Claude Bedrock models, merge and dedupe any existing beta array, compose cleanly with the existing guardrail and cache-point wrappers, and add focused tests in `extensions/amazon-bedrock/index.test.ts` for no-guardrail, guardrail, existing beta values, non-Anthropic models, and application-profile false-positive handling.
