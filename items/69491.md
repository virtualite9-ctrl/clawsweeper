## Summary

Keep open. Current `main` does not implement Bedrock `additionalModelRequestFields.anthropic_beta` forwarding for Anthropic models, so this PR is not obsolete, and the review thread identifies unresolved correctness risks in the proposed patch.

## Evidence

- **Current main does not forward Bedrock Anthropic betas:** On `main`, `baseWrapStreamFn` returns the Anthropic Bedrock stream function unchanged and only applies the no-cache wrapper to non-Anthropic models. There is no `additionalModelRequestFields.anthropic_beta` payload patch in the Bedrock plugin.
  - file: [extensions/amazon-bedrock/register.sync.runtime.ts:271](https://github.com/openclaw/openclaw/blob/2cd2732ab6695a4b3220ce41166b74aca7097c5f/extensions/amazon-bedrock/register.sync.runtime.ts#L271)
  - command: `nl -ba extensions/amazon-bedrock/register.sync.runtime.ts | sed -n '240,330p'`
  - sha: [2cd2732ab669](https://github.com/openclaw/openclaw/commit/2cd2732ab6695a4b3220ce41166b74aca7097c5f)
- **Current Bedrock tests do not cover the proposed behavior:** The Anthropic Bedrock guardrail test on `main` asserts only `guardrailConfig` is injected for Anthropic models and does not expect any `additionalModelRequestFields.anthropic_beta` payload. That matches the current implementation gap.
  - file: [extensions/amazon-bedrock/index.test.ts:382](https://github.com/openclaw/openclaw/blob/2cd2732ab6695a4b3220ce41166b74aca7097c5f/extensions/amazon-bedrock/index.test.ts#L382)
  - command: `nl -ba extensions/amazon-bedrock/index.test.ts | sed -n '334,430p'`
  - sha: [2cd2732ab669](https://github.com/openclaw/openclaw/commit/2cd2732ab6695a4b3220ce41166b74aca7097c5f)
- **Direct Anthropic path already has the default beta behavior:** The bundled Anthropic plugin already defines the same default beta set and merges it into the outbound `anthropic-beta` header, which supports the PR's parity goal but also shows the Bedrock equivalent is not yet present on `main`.
  - file: [extensions/anthropic/stream-wrappers.ts:21](https://github.com/openclaw/openclaw/blob/2cd2732ab6695a4b3220ce41166b74aca7097c5f/extensions/anthropic/stream-wrappers.ts#L21)
  - command: `nl -ba extensions/anthropic/stream-wrappers.ts | sed -n '1,220p'`
  - sha: [2cd2732ab669](https://github.com/openclaw/openclaw/commit/2cd2732ab6695a4b3220ce41166b74aca7097c5f)
- **The Bedrock Anthropic detector is explicitly heuristic-only for app profiles:** `isAnthropicBedrockModel` documents that application inference profile detection is name-based and can produce both false positives and false negatives. Reusing that heuristic for Anthropic-only payload mutation is riskier than using it only for cache semantics.
  - file: [src/agents/pi-embedded-runner/anthropic-family-cache-semantics.ts:27](https://github.com/openclaw/openclaw/blob/2cd2732ab6695a4b3220ce41166b74aca7097c5f/src/agents/pi-embedded-runner/anthropic-family-cache-semantics.ts#L27)

- **PR review already identified unresolved correctness issues:** The review thread on PR #69491 flags two concrete problems in the proposed patch: it overwrites any pre-existing `additionalModelRequestFields.anthropic_beta` values instead of merging them, and it injects Anthropic-only request fields based on the broad `isAnthropicBedrockModel` heuristic.
  - file: [extensions/amazon-bedrock/register.sync.runtime.ts:28](https://github.com/openclaw/openclaw/blob/65f254223bee4c9db42a694d1855cb7405951f32/extensions/amazon-bedrock/register.sync.runtime.ts#L28)

## Risks / Open Questions

- Merging the PR as-is could silently discard user-supplied `additionalModelRequestFields.anthropic_beta` values instead of treating the shipped flags as defaults.
- Using the current `isAnthropicBedrockModel` heuristic for payload mutation can send Anthropic-only Bedrock request fields to non-Claude application inference profiles whose names happen to contain `claude`.
