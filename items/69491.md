## Summary

Keep PR #69491 open. Current main still does not forward default Anthropic beta flags through the standard Amazon Bedrock Converse stream payload, so the PR is not obsolete. The branch is relevant, but it should not land as-is until the existing review concerns about preserving user-supplied betas and avoiding heuristic false positives are addressed.

## Best Possible Solution

Keep #69491 open and ask for a narrow update to the bundled Amazon Bedrock provider: add Bedrock Converse `additionalModelRequestFields.anthropic_beta` forwarding for definitive Claude/Anthropic targets, merge with any existing beta array, preserve guardrail composition, leave non-Anthropic models untouched, and add regression tests for the merge case plus the non-Anthropic/app-profile boundary.
