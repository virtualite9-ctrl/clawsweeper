## Summary

Keep open. Current main still hardcodes the explicit image tool timeout to 30 seconds in all three `runImagePrompt()` paths, while the config contract exposes `tools.media.image.timeoutSeconds`. This PR is still relevant, but it should add focused regression coverage and clarify the linked issue’s config-key mismatch.

## Best Possible Solution

Keep this PR open and revise it into a tested fix: compute the timeout in `runImagePrompt()` from the documented image media config, pass it to all three image-description call paths, and add `image-tool.test.ts` coverage for configured and default timeout behavior. Separately decide whether per-entry `tools.media.models[].timeoutSeconds` should also affect the explicit image tool.
