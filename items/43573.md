## Summary

Keep this PR open. Current main still does not implement the central behavior: `sessions_spawn(runtime="acp")` sends the initial task through the later ACP `agent` turn, but it does not seed the child transcript/history immediately after session creation, does not return a `taskDelivered` confirmation, and has no seeded-replay dedupe path. The PR remains a plausible implementation candidate for the already-discussed ACP empty-history bug, but it should be rebased and narrowed because the current diff also carries unrelated CI/workflow/labeler changes that need separate maintainer and security review.

## Best Possible Solution

Keep this PR open, but ask the contributor to rebase and narrow it to the ACP behavior: seed the initial `sessions_spawn(runtime="acp")` task into the child transcript immediately after session creation, pass provenance consistently through the first dispatch, dedupe only the seeded replay, preserve prompt-only interrupted transcripts, and keep focused regression tests. Split or drop the CI/workflow/labeler changes into a separate maintainer-reviewed PR before this can be considered for merge.
