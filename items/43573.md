## Summary

Current main still does not implement the PR's central ACP behavior. The initial `sessions_spawn(runtime="acp")` task is dispatched to the ACP child via the later `agent` RPC, but current main has no prompt-only transcript seed before that turn completes, no `taskDelivered` confirmation field, and no seeded-replay dedupe. The PR remains a plausible bug-fix candidate for #43518, but it should be rebased and narrowed because it also carries unrelated CI/labeler workflow changes that need separate maintainer/security review.

## Best Possible Solution

Keep this PR open as a real implementation candidate, but ask the contributor to rebase and narrow it to the ACP transcript behavior: seed the initial `sessions_spawn(runtime="acp")` task into the child transcript immediately after session creation, pass provenance consistently through the first dispatch, dedupe only the seeded replay, preserve prompt-only interrupted transcripts, and keep focused regression tests. Split or drop the CI/workflow/labeler changes into a separate maintainer-reviewed PR.
