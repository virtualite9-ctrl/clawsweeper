## Summary

Keep #13627 open. Current main serializes concurrent Slack sends per target/thread, but it still does not provide the requested configurable minimum delay between outbound Slack posts, and the related implementation attempt #26732 is closed and unmerged.

## Best Possible Solution

Keep #13627 open and implement a focused Slack outbound minimum-delay feature in the bundled Slack plugin send path, with documented config/schema/UI metadata and fake-timer coverage across normal sends, chunks, media follow-ups, DMs, channel messages, thread replies, Slack action sends, and slash follow-ups. If maintainers want a generic channel throttle instead, track that broader abstraction separately rather than treating closed unmerged PR #26732 as the fix.
