## Summary

Keep open. Current `main` still routes fetched Feishu interactive messages through a parser that only handles flat card-template elements (`div` and `markdown`), not the nested post-style fallback payload described in #60380, so the report remains actionable.

## Evidence

- **Current parser still misses fallback shapes:** `parseInteractiveCardContent` only reads `elements` or `body.elements`, then extracts `div.text.content` and `markdown.content`. It does not traverse nested arrays or handle fallback `text`/`a` items or top-level/header titles, so the issue’s sample fallback payload would still collapse to `[Interactive Card]`.
