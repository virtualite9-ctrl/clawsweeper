## Summary

The request is useful, but it fits the ClawHub/community plugin path rather than OpenClaw core. Current main already exposes the needed plugin hooks and guarded-fetch SDK surfaces for a discovery/trust verifier, and related discussion plus the #66717 discovery-verification PoC show this can be explored externally without a missing core API.

## Best Possible Solution

Move this to an externally maintained ClawHub/npm plugin that implements agent.json or other discovery resolvers with before_tool_call and guarded fetch. Reopen core discussion only for a concrete missing SDK seam or an owner-backed decision to standardize one trust/discovery protocol in OpenClaw core.
