## Summary

Keep open. Current `main` still implements `feishu_wiki` `nodes` as a single `client.wiki.spaceNode.list` call with no pagination loop, no `page_size`/`page_token` inputs, and no `total`/`pages_fetched` result fields, so the reported 20-item cap is not implemented away on `main`. I did not run a live Feishu reproduction because that would require external credentials/data, but the current code matches the reported bug.
