## Summary

Keep #37626 open. Current main still single-pages the bundled Feishu `feishu_wiki` `nodes` action: it makes one `client.wiki.spaceNode.list` call with only `parent_node_token`, maps only that response's `items`, and the schema exposes no pagination controls or automatic fetch-all behavior. The linked fix PR #37641 directly targeted this bug but was closed unmerged, so it does not supersede the issue.

## Best Possible Solution

Keep the issue open and fix it in the bundled Feishu plugin. The likely implementation belongs in `extensions/feishu/src/wiki.ts`: request an appropriate `page_size`, continue while Feishu returns `has_more` with a usable `page_token`, accumulate all node items, and add a focused regression test mocking multiple `spaceNode.list` pages. Exposing manual `page_size`/`page_token`/`fetch_all` fields is a product decision, but automatic full pagination would solve the central data-loss problem without changing the user workflow.
