## Summary

Current main already covers the reported local-Ollama 'only talks, won’t act' regression: local onboarding now defaults to the `coding` tool profile instead of `messaging`, weak no-action turns are surfaced as explicit errors instead of silently stalling, and leaked Qwen/Ollama tool-call text is stripped from visible replies/history.

## Close Comment

Codex review: current `main` already covers this report well enough.

The regression path here was the local/onboarding tool-policy default. OpenClaw now defaults unset local configs to `tools.profile: "coding"` instead of `"messaging"` in `src/commands/onboard-config.ts:6`, with regression coverage in `src/commands/onboard-config.test.ts:9`, and the shipped changelog entry is `CHANGELOG.md:2932` under release `2026.3.7`.

Current `main` also hardens the exact “只会说话不会工作” failure mode: empty/reasoning-only/toolless endings are surfaced as explicit errors instead of looking like progress (`src/agents/pi-embedded-runner/run/incomplete-turn.ts:132` with tests at `src/agents/pi-embedded-runner/run.incomplete-turn.test.ts:438`), and leaked Qwen/Ollama tool-call text is stripped from visible replies/history (`src/shared/text/assistant-visible-text.ts:325`, plus the Ollama warning in `docs/providers/ollama.md:10`).

Based on that Codex review, I’m closing this as implemented on main. If this still reproduces on a current release, please open a new issue with `openclaw status --deep`, `openclaw config get tools`, the exact Ollama `baseUrl` and model id, and a fresh transcript/log bundle from the failing run.
