## Summary

Keep open. Current `main` still scopes `contextPruning` to `toolResult` messages and does not prune image blocks from user or assistant history entries, so the issue is not strongly supported as fixed on `main`.

## Evidence

- **Pruner only rewrites tool results:** `pruneContextMessages()` iterates the prune window but immediately skips every non-`toolResult` message, so user/assistant content blocks are never transformed during cache-ttl pruning.
  - file: [src/agents/pi-hooks/context-pruning/pruner.ts:309](https://github.com/openclaw/openclaw/blob/cb2c36b04975e34ab7014a11575d6589aa96ae32/src/agents/pi-hooks/context-pruning/pruner.ts#L309)
  - command: `nl -ba src/agents/pi-hooks/context-pruning/pruner.ts | sed -n '220,380p'`
  - sha: [cb2c36b04975](https://github.com/openclaw/openclaw/commit/cb2c36b04975e34ab7014a11575d6589aa96ae32)
- **User messages still count image size but are not prunable:** `estimateMessageChars()` explicitly counts image content for `user` messages, but the later pruning pass has no corresponding branch to strip or replace those blocks.
  - file: [src/agents/pi-hooks/context-pruning/pruner.ts:132](https://github.com/openclaw/openclaw/blob/cb2c36b04975e34ab7014a11575d6589aa96ae32/src/agents/pi-hooks/context-pruning/pruner.ts#L132)
