## Summary

Keep #39543 open. Current main has a PDF analysis tool for path/URL/managed-media references, but the user-facing local PDF upload-and-summarize workflow is still missing in the Control UI and Gateway chat attachment path.

## Best Possible Solution

Keep the issue open and implement the Control UI/WebChat local PDF path: allow PDF selection/drop, carry filename/MIME/document metadata in `ChatAttachment`, render a document attachment state, serialize PDFs distinctly, update Gateway validation/offload to preserve `application/pdf` as managed inbound media refs, and route those refs to the existing `pdf` tool with focused UI, Gateway, and PDF-tool integration coverage.
