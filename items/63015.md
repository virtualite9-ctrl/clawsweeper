## Summary

Keep this PR open. Current main still drops `filePath`, `path`, `media`, and `fileUrl` aliases in loose outbound reply normalization, while the shared message-send action path already accepts those attachment aliases. The PR remains a focused implementation candidate, but its fallback precedence should be resolved before merge.

## Best Possible Solution

Keep this PR open and finish it as a small plugin SDK/outbound normalization fix: decide the canonical alias precedence, update `normalizeOutboundReplyPayload`, keep regression tests for `media`, `mediaUrl`, `filePath`, and `path` at minimum, consider whether `fileUrl` should also be included for parity with the shared send action, then run the relevant changed-lane checks before landing.
