## Summary

Keep this issue open. Current main at 7d9dc8cf24fc143bec95318491e0247c37863e63 still has the reported shared block-streaming boundary-loss behavior: the chunker consumes paragraph/newline separators, text_end streaming can flush those chunks as separate deliveries, and downstream block reply payloads carry only plain text without boundary metadata.

## Best Possible Solution

Keep this open and fix the shared block-streaming pipeline. Preserve boundary semantics from EmbeddedBlockChunker through coalescing and delivery, either by attaching consumed separators to emitted chunks while normalizing only duplicate comparisons, or by carrying explicit boundary metadata. Add a regression test that streams Markdown such as '# Title\n\nFirst paragraph.' across at least two flushed text_end block deliveries and asserts the reconstructed visible text remains structurally equivalent.
