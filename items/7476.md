## Summary

Keep #7476 open. Current main still lacks an outbound WhatsApp sticker send path: WebP media is routed as a normal image, there is no sticker/asSticker send option, and WhatsApp only advertises react/poll message actions. The related implementation PR #7502 was closed unmerged, so the issue is not implemented or superseded.

## Best Possible Solution

Keep the issue open and implement this in the bundled WhatsApp plugin. Add a deliberate outbound sticker contract, preferably explicit action=sticker or asSticker; preserve qualifying WebP sticker media before generic image optimization; route outbound sends through Baileys { sticker: buffer }; advertise and dispatch the WhatsApp sticker action or option consistently; and add focused send-api, media-loading, message-action, docs, and regression coverage.
