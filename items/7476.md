## Summary

Keep #7476 open. Current main at c7a0d9b1889ca09ab5bfd84cda39a8d50d3d5768 still has inbound WhatsApp sticker recognition, but no outbound WhatsApp sticker send path: WebP/image media is sent as an image payload, no `sticker`/`asSticker` option exists, `action=sticker` is not supported by the WhatsApp plugin, and the prior implementation PR #7502 was closed unmerged.

## Best Possible Solution

Keep the issue open and implement this in the bundled WhatsApp plugin. Add an explicit outbound sticker contract such as `action=sticker` or `asSticker`, preserve valid WebP sticker media before image optimization, route outbound sends through the native WhatsApp/Baileys sticker payload, and update WhatsApp action discovery, gateway/message option surfaces, docs, and focused send-api/media-loader tests.
