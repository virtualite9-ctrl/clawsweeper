## Summary

Keep this issue open. Current main still exposes the ambiguous `running | done | failed | killed | timeout` session status vocabulary as the persisted, gateway-facing, UI/tool-facing contract, and there is still no additive `resumable`, `lastTurnStatus`, `lifecycleStatus`, or `taskStatus` field or clear agent-facing guidance that `failed`, `timeout`, and `done` sessions should usually be resumed with `sessions_send` rather than replaced. The adjacent closed issues #69202 and #71756 cover related lifecycle/recovery symptoms, but they do not supersede this agent-facing status/resumability contract bug.

## Best Possible Solution

Keep this issue open as an agent-facing session API contract bug. Add a backward-compatible `resumable` field plus a distinct `lastTurnStatus`, `lifecycleStatus`, or `taskStatus` field to `sessions.list` rows, document the semantics on https://docs.openclaw.ai/concepts/session-tool, and update tool descriptions or prompt guidance so orchestrators use `sessions_send` for known visible sessions unless the session is deleted or inaccessible. Add focused tests for the returned row shape and guidance; defer any status-string rename to a versioned migration.
