## Summary

Keep this issue open. Current main still exposes `running | done | failed | killed | timeout` as the session status contract for persisted sessions, gateway session rows, and `sessions_list` output, without an additive `resumable`, `lastTurnStatus`, `lifecycleStatus`, or `taskStatus` field. Existing docs and prompt guidance now discourage polling, but they still do not clearly tell orchestrator agents that terminal-looking session statuses can be followed up with `sessions_send` instead of spawning a duplicate session. Related closed issues #69202 and #71756 cover adjacent recovery/finalization problems, not this agent-facing status/resumability contract.

## Best Possible Solution

Keep this issue open as an agent-facing session API contract bug. Add a backward-compatible `resumable` field and a distinct `lastTurnStatus`, `lifecycleStatus`, or `taskStatus` field to `sessions.list` rows, document the semantics on https://docs.openclaw.ai/concepts/session-tool, and update tool descriptions or prompt guidance so orchestrators prefer `sessions_send` for visible existing sessions unless the session is deleted or inaccessible. Add focused tests for the returned row shape and the guidance; defer any status-string rename to a versioned migration.
