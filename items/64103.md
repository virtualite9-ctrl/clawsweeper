## Summary

Keep open. Current `main` still persists and exposes subagent session statuses as `running|done|failed|killed|timeout`, and the public session-tool docs do not clarify that these are last-run outcomes rather than session-lifecycle states. The implementation still allows `sessions_send` to target an existing visible session regardless of those values, so the ambiguity described in #64103 remains present rather than fixed.
