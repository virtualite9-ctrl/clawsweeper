## Summary

Keep this issue open. Current main still exposes the same terminal-looking `status` values on persisted session entries and agent-facing `sessions_list` rows, with no additive `resumable`, `lastTurnStatus`, `lifecycleStatus`, or `taskStatus` field and no explicit docs/prompt guidance that `failed`, `timeout`, or `done` sessions should normally be resumed with `sessions_send` rather than replaced.

## Best Possible Solution

Keep this issue open and treat it as an agent-facing session API contract problem. Add a backward-compatible field such as `resumable` plus a distinct `lastTurnStatus`/`lifecycleStatus`/`taskStatus` field to `sessions.list` rows, document the semantics on https://docs.openclaw.ai/concepts/session-tool, and update tool descriptions or prompt guidance so orchestrators prefer `sessions_send` for known visible sessions unless the session is deleted or inaccessible. Add focused tests for the returned row shape and guidance; defer any status-string rename to a versioned migration.
