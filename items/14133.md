## Summary

Keep #14133 open. Current main still has the Cron Jobs Control UI surface only: no Reminders tab/route, no reminder-specific cron classification/display metadata, and no list/filter contract that would let reminder-created one-shot jobs move out of the Cron tab. The existing 2026-04-26 maintainer-side Codex comment also explicitly kept this item open, and current main at 7e13f3f5140168e31027b39a9855ade3fadf24d5 still matches that assessment.

## Best Possible Solution

Keep this open as a core Control UI plus cron protocol enhancement. The best implementation is an additive cron classification/display metadata contract, reminder-producing paths that set it, typed list filtering, Cron tab filtering or hiding for reminder jobs, and a Reminders tab with human-readable reminder text, fire time, channel/recipient, and delete actions, backed by UI/protocol/service tests and docs updates.
