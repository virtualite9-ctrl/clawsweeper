## Summary

Keep #14133 open. Current main still has only the existing Cron Jobs Control UI surface, no dedicated Reminders tab/route, and no cron job metadata/tag/type/display contract that would let reminder-created one-shot jobs be filtered out of the Cron tab and rendered with human-readable reminder text. This matches the existing 2026-04-26 maintainer-side Codex comment on the issue that already kept it open.

## Best Possible Solution

Keep this issue open as a core Control UI plus cron protocol enhancement. A good implementation would add an additive cron classification/display metadata contract, update reminder-producing paths to set it, expose typed list filtering, hide or filter reminder jobs from the Cron tab, and add a Reminders tab with human-readable reminder text, fire time, channel/recipient, and delete actions.
