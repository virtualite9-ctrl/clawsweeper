## Summary

Keep #14133 open. Current main at 406ae72fd2789c78053f6b94006ec8f488bc1c00 still has only the generic Cron Control UI surface: there is no Reminders tab/route, no reminder render branch, no cron job classifier/display metadata in the wire schema or stored job type, and no backend cron.list/listPage filter that can separate reminder one-shot jobs from ordinary cron jobs. The provided discussion also includes a same-day Codex review comment from steipete explicitly keeping this issue open for the same missing core UI/protocol work.

## Best Possible Solution

Keep this open as a core Control UI plus Gateway cron protocol enhancement. The best path is an additive cron classification/display metadata contract, reminder-producing paths that set that metadata, typed list filtering for reminder versus non-reminder jobs, Cron tab filtering or hiding for reminder jobs, and a Reminders tab showing cleaned reminder text, fire time, channel/recipient, and delete actions with protocol/service/UI tests and docs updates.
