## Summary

Keep open. Current `main` still rejects numeric-string `cost` values and still accepts boolean/non-finite numeric values in `skills/model-usage/scripts/model_usage.py`, so the reported bug is not implemented on `main` and is reproducible from the current code.

## Evidence

- **Aggregation still drops numeric strings:** `aggregate_costs()` only accepts `cost` when `isinstance(cost, (int, float))` is true, so string values like `"1.75"` are skipped entirely. In Python this same check also admits booleans because `bool` is a subclass of `int`.
  - file: [skills/model-usage/scripts/model_usage.py:121](https://github.com/openclaw/openclaw/blob/55389f4be29359e2dd57ebaf548ba4a70331a445/skills/model-usage/scripts/model_usage.py#L121)
  - sha: [55389f4be293](https://github.com/openclaw/openclaw/commit/55389f4be29359e2dd57ebaf548ba4a70331a445)
- **Current-model selection has the same bug:** `pick_current_model()` uses the same `isinstance(cost, (int, float))` gate before scoring models, so numeric-string costs are ignored when choosing the current model, while booleans and non-finite floats still qualify.
  - file: [skills/model-usage/scripts/model_usage.py:145](https://github.com/openclaw/openclaw/blob/55389f4be29359e2dd57ebaf548ba4a70331a445/skills/model-usage/scripts/model_usage.py#L145)
  - sha: [55389f4be293](https://github.com/openclaw/openclaw/commit/55389f4be29359e2dd57ebaf548ba4a70331a445)
- **Latest-day cost has the same bug:** `latest_day_cost()` only returns a value when `cost` is an `int` or `float`, so numeric strings return `None` instead of a parsed number, and non-finite floats are returned unchanged.
  - file: [skills/model-usage/scripts/model_usage.py:180](https://github.com/openclaw/openclaw/blob/55389f4be29359e2dd57ebaf548ba4a70331a445/skills/model-usage/scripts/model_usage.py#L180)
  - sha: [55389f4be293](https://github.com/openclaw/openclaw/commit/55389f4be29359e2dd57ebaf548ba4a70331a445)
- **No regression tests for the reported edge cases:** The current test file only covers `positive_int()` and `filter_by_days()`. There are no tests for numeric-string costs, booleans, `NaN`, or `Infinity`.
  - file: [skills/model-usage/scripts/test_model_usage.py:10](https://github.com/openclaw/openclaw/blob/55389f4be29359e2dd57ebaf548ba4a70331a445/skills/model-usage/scripts/test_model_usage.py#L10)
  - sha: [55389f4be293](https://github.com/openclaw/openclaw/commit/55389f4be29359e2dd57ebaf548ba4a70331a445)
- **Read-only reproduction against current main:** Running the helper functions directly with in-memory data on current `main` produced: `aggregate {'sonnet-4.6': 2.0, 'bool-model': 1.0, 'nan-model': nan, 'inf-model': inf}`; `pick ('sonnet-4.6', '2026-04-24')`; `latest ('2026-04-24', None)`. That shows string cost `"1.75"` for `gpt-5.4` was ignored, while `True`, `nan`, and `inf` were accepted.
