## Summary

Keep open. Current main still has the reported behavior: cost parsing only accepts native int/float values, booleans pass as numbers, and NaN/Infinity can propagate through model-usage aggregation and selection. The linked PR #37877 was closed unmerged, and its helper/tests are not present on main.

## Best Possible Solution

Keep #37878 open and implement the narrow helper proposed in the issue: add a shared cost parser for model_usage.py that accepts finite numeric strings, rejects booleans, rejects NaN/Infinity, and is used by aggregate_costs, pick_current_model, and latest_day_cost. Add focused regression tests in skills/model-usage/scripts/test_model_usage.py for numeric-string aggregation/current/latest-day behavior and for bool/non-finite values being ignored.
