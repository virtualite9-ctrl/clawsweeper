## Summary

Keep #37878 open. Current main still drops numeric-string costs in the model-usage helper, still lets booleans and non-finite numeric costs through, and the linked implementation PR #37877 is closed and unmerged per the provided GitHub context.

## Best Possible Solution

Keep the issue open and implement the narrow fix from PR #37877 on current main: add a shared finite cost parser in `skills/model-usage/scripts/model_usage.py` that accepts finite numeric strings, rejects booleans, rejects `NaN`/`Infinity`, and is used by `aggregate_costs`, `pick_current_model`, and `latest_day_cost`. Add focused tests in `skills/model-usage/scripts/test_model_usage.py` for numeric-string aggregation/current/latest-day behavior and ignored bool/non-finite costs.
