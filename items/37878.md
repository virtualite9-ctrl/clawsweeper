## Summary

Keep #37878 open. Current main still drops numeric-string costs and still accepts bool/NaN/Infinity in the model-usage cost paths; the linked PR #37877 is closed unmerged and no equivalent helper/tests are present on main.

## Best Possible Solution

Keep #37878 open and implement the narrow fix from the linked PR: add a shared cost parser in skills/model-usage/scripts/model_usage.py that accepts finite numeric strings, rejects booleans, rejects NaN/Infinity, and is used by aggregate_costs, pick_current_model, and latest_day_cost. Add focused regression tests in skills/model-usage/scripts/test_model_usage.py for numeric-string aggregation/current/latest-day behavior and for bool/non-finite values being ignored.
