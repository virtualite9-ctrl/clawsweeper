## Summary

Keep open. Current `main` still ignores restart-required config-file reloads when `gateway.reload.mode` is `hot`, and the proposed `gateway.reload.fallbackToRestart` option from PR #59745 is not present in the shipped code or config schema.

## Evidence

- **Hot reload path still skips restart-required changes:** `startGatewayConfigReloader()` on current `main` still logs `hot mode ignoring` and returns instead of queueing a restart when `plan.restartGateway` is true under `gateway.reload.mode === "hot"`.
