## Summary

Current `main` still appears to have the reported TUI `/sessions` title bug. The gateway session row computes an outer `displayName`, but derived-title generation still only checks `entry.displayName` and raw first-user transcript text; the TUI then prefers `derivedTitle` over `displayName`, so gateway sessions can still surface the injected `Sender (untrusted metadata): ...` block instead of the external client name.

## Evidence

- **Derived title ignores outer row displayName:** `deriveSessionTitle` only checks `entry.displayName` and `entry.subject`, then falls back to `firstUserMessage`. It has no parameter or path for the outer row `displayName` computed elsewhere.
