## Summary

Keep #39722 open. Current main still has the reported title precedence and sanitization gap: `/sessions` asks for derived titles and renders `derivedTitle` before `displayName`, while gateway session row construction computes an outer `displayName` but derives the title from only `entry` plus transcript text. The transcript title extractor still strips directive tags only, not inbound metadata envelopes, so historical or otherwise metadata-prefixed transcript user messages can still become session titles. The proposed fix PR #39790 is still only related context, not evidence of a shipped fix on main.

## Best Possible Solution

Keep #39722 open until PR #39790 or an equivalent maintainer-reviewed fix lands. The implementation should make the computed outer session `displayName` take precedence during derived-title construction, strip inbound metadata envelopes before first-user-message and last-message-preview extraction, preserve subject/message/session-id fallbacks, and add regression coverage for both metadata stripping and external displayName precedence.
