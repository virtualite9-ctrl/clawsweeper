## Summary

Keep this PR open. Current main still uses the older order-dependent LAN address selection for gateway dashboard/status, pairing setup, and the bundled device-pair plugin, while the PR remains an active implementation candidate for linked #71493.

## Best Possible Solution

Keep this PR open for maintainer review or land an equivalent fix. The best path is consistent LAN address selection across gateway dashboard/status, pairing setup, and device-pair: prefer default-route or physical/default-looking LAN interfaces, deprioritize Docker/bridge/veth/CNI/container interfaces as fallback candidates, preserve explicit configuration overrides, and keep regression coverage for Docker bridge, CNI overlay, and default-route cases.
