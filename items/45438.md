## Summary

Keep open. Current main still contains the exact `structuredClone` calls reported in the session-store cache, session-store load return path, and auth-profile store clone helper, so the observable bug is not implemented on main. The open PR #60444 appears to be the right fix direction, but it is still open/unmerged in the provided GitHub context, and the discussion explicitly treats #45438 as the primary tracker for the crash mechanism.

## Best Possible Solution

Keep #45438 open as the primary structuredClone/native-memory OOM tracker until #60444 or an equivalent maintainer-owned patch lands. The fix should replace the session-store/auth-profile clone hot paths with a JSON-safe clone or serialized-string reuse while preserving mutation isolation, add focused regression or benchmark coverage for the cache/load paths, and then close this issue as implemented with the landing commit and release. Broader session growth and multi-store loading issues should remain tracked separately.
