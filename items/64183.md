## Summary

Current `main` does not show a clear fix for this report. The `openclaw status` memory row still renders `enabled (plugin <slot>) · unavailable` whenever the status scan cannot obtain a generic memory snapshot, and the scan still intentionally bails out before probing the memory manager unless memory search is explicitly configured or a local store path already exists. That leaves the reported mismatch with a healthy external/backend plugin plausible on current `main`, so this should stay open.

## Evidence

- **Status output still reports unavailable when no snapshot is returned:** `buildStatusMemoryValue` still renders the memory row as `enabled (plugin <slot>) · unavailable` whenever `params.memory` is null, with no distinction between plugin health, backend health, or probe/cache mismatch.
