## Summary

Close #64183 as duplicate/superseded by open issue #56968, which already tracks the same OpenClaw status false-negative for third-party memory-slot plugins. Current main still has the shared root behavior: the status scan can return no memory snapshot before checking the active memory runtime, and the renderer turns that into `enabled (plugin …) · unavailable`.

## Best Possible Solution

Close this duplicate and keep the remaining work centralized under #56968 and the related status-fix PRs. The core fix should make `openclaw status` use the active memory plugin/runtime or gateway doctor status before reporting unavailable, distinguish runtime/plugin/backend/embedding states, and add regression coverage for an external memory-slot plugin whose own health/list commands succeed without a built-in `agents.*.memorySearch` store.
