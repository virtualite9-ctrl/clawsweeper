## Summary

Keep this PR open. Current main still preserves orphaned providers in generated models.json merge mode and still has provider-unaware context-window lookups in gateway defaults, CLI session display, and cron accounting. Some related runtime paths are already better on main, so the useful path is a focused revision rather than cleanup closure.

## Best Possible Solution

Keep this PR open and request a focused revision: make an explicit maintainer decision on merge-mode orphan-provider semantics, update docs/schema/tests if non-configured providers should be dropped, finish provider-aware context-token resolution in gateway defaults, CLI sessions, and cron accounting, preserve persisted contextTokens only as fallback when provider/model metadata is known, and split unrelated embedded-runner or cron terminal-error changes into separate work.
