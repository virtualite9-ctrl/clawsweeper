## Summary

Keep open. Current main still does not implement the proposed durable `worker-state.json` snapshot; it only has an optional task-registry observer seam and existing CLI/SQLite task inspection. This PR remains the active implementation candidate, but it needs maintainer review or revision because it also carries unrelated `doctor --json` changes with existing correctness feedback.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to split or drop the unrelated doctor JSON work, address the gateway-health and JSON-output review comments if that work stays, rebase the task-registry observer on current main, and add focused restore/upsert/delete/default-wiring tests plus docs if `worker-state.json` is intended as a supported operator surface.
