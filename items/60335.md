## Summary

Keep open. Current `main` still hydrates root-message Slack files for thread replies with no direct media, then attaches that `threadStarterMedia` to the inbound payload. The new reply-file filter only strips duplicated parent files from the reply event itself; it does not stop the later thread-starter-media fallback that matches the reported bug.
