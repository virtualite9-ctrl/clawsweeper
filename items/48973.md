## Summary

Keep open. Current `main` does not contain the requested non-official Codex proxy auth fallback, and the proposed `pnpm patchedDependencies` approach still appears unsuitable for the repo’s documented npm-distributed install path. The PR is also superseded in discussion by open replacement PR #60199 rather than by a landed fix.

## Risks / Open Questions

- Current `main` still appears to reject opaque bearer tokens on non-`chatgpt.com` Codex-compatible base URLs before dispatch, because the upstream `pi-ai` transport remains strict.
- This PR's approach still has an unresolved release-path problem for recommended global installs: `pnpm patchedDependencies` plus an unshipped `patches/` directory would not cover npm-installed OpenClaw users.
- Discussion already points maintainers to PR #60199 as the rebased successor, so closing this PR as implemented/cannot-reproduce would misstate current repo state.
