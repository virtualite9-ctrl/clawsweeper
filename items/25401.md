## Summary

Keep #25401 open. Current main at e29d3516bf05591e6007bfb8ccdaea6d76194d41 still does not implement a provider-level `models.providers.<id>.enabled` contract, does not filter disabled providers from models.json planning or fallback candidates, and does not expose disabled provider state in `openclaw models status`. The related implementation PR #25415 was closed unmerged, and the existing issue discussion already reached the same keep-open conclusion.

## Best Possible Solution

Keep the issue open until core adds a documented `models.providers.<provider>.enabled?: boolean` contract, defaults omitted values to enabled, filters disabled providers from models.json generation and fallback candidate selection, and exposes disabled state in `openclaw models status` JSON and rich output. PR #25415 is useful reference work, but it was closed unmerged and current main does not satisfy the request.
