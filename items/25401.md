## Summary

Keep #25401 open. Current main at 7e13f3f5140168e31027b39a9855ade3fadf24d5 still does not define or honor a provider-level `models.providers.<id>.enabled` flag, and the related implementation PR #25415 is closed unmerged per the provided GitHub context.

## Best Possible Solution

Keep the issue open until core adds a documented `models.providers.<provider>.enabled?: boolean` contract that defaults omitted values to enabled, preserves the provider config block, excludes disabled providers from models.json generation and model/fallback candidate selection, and reports disabled state in `openclaw models status` output. PR #25415 remains useful prior art but was closed unmerged.
