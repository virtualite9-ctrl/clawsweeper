## Summary

Keep #25401 open. Current main still does not implement a provider-level `models.providers.<id>.enabled` contract: the type and strict runtime schema omit `enabled`, models.json planning merges every explicit provider, fallback candidate construction does not filter disabled providers, configured model cataloging includes every provider, and `openclaw models status` has no disabled-provider state. The linked implementation PR #25415 was closed unmerged, and the latest maintainer/Codex issue comment also concluded this remains the active tracker.

## Best Possible Solution

Keep the issue open until OpenClaw adds a documented `models.providers.<provider>.enabled?: boolean` contract that defaults to enabled, preserves disabled config blocks, excludes disabled providers from generated `models.json`, configured model catalogs, fallback candidate selection, and status/probe paths, and displays disabled state in `openclaw models status` JSON and rich output. Add focused tests for schema acceptance, models-config planning, fallback behavior, configured catalog/listing, and status output.
