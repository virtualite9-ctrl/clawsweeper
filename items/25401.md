## Summary

Keep #25401 open. Current main still does not implement a provider-level `models.providers.*.enabled` contract, and the related implementation PR #25415 was closed unmerged. The existing code still accepts provider blocks without an `enabled` field, writes configured providers into models.json, builds fallback candidates without provider-disabled filtering, and shows provider auth status without a disabled-provider status surface.

## Best Possible Solution

Keep the issue open until OpenClaw core adds a typed and documented `models.providers.<provider>.enabled?: boolean` contract, filters disabled providers from models.json generation and fallback candidate selection, and exposes disabled providers in `openclaw models status` JSON and rich output with regression tests. A maintainer could also explicitly reject the product direction, but current code does not satisfy the requested behavior.
