## Summary

Keep open. Current main still runs subagent and ACP completion announces as normal requester-session agent turns, without an announce-specific model override or no-history/light-context policy. Existing cheaper-model and light-context knobs apply to spawned children, heartbeat/hooks, or plugin subagent calls, not the completion announce relay described in this issue.

## Best Possible Solution

Implement a schema-backed announce policy, likely under agents.defaults.subagents.announce with per-agent inheritance alongside existing subagent defaults. Resolve an allowed cheaper announce model through the existing model catalog/auth machinery, and wire provider/model plus promptMode/bootstrapContextMode or an isolated/no-history context into direct, queued, and descendant-wake announce agent calls without changing spawned child model behavior. Add focused tests for native subagents, ACP completions, nested requester sessions, queue fallback, and unauthorized model overrides, then document the knobs on docs.openclaw.ai tools/subagents and tools/acp-agents.

## Risks / Open Questions

- Leaving this open is appropriate because the reported cost path still exists for large requester sessions: announce turns can run on the parent model with the parent transcript.
- Any announce model override should preserve model allowlists, auth-profile/provider routing, and the Gateway allowModelOverride trust boundary.
- A light/no-history announce mode needs targeted coverage for nested subagent wakeups, queued retry/idempotency, ACP plain-prompt conversion, direct-send fallback, and silent/skip semantics.
