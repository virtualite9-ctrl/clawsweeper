## Summary

Keep open. Current main still lacks the requested announce-specific model override and light/no-history announce context. The current implementation routes completion handoff through a Gateway `agent` call on the requester session without `provider`, `model`, or `bootstrapContextMode`, while existing config only covers spawned-child defaults and announce timeout. The prior ClawSweeper comment and related reports correctly separate delivery reliability fixes from this remaining cost-control request.

## Best Possible Solution

Keep this issue open and implement an explicit, schema-backed announce policy for task-completion handoff, likely under `agents.defaults.subagents.announce`. It should support a cheaper model/fallbacks and a light or no-history context mode only for the internal announce `agent` call, while preserving existing delivery, idempotency, nested-session, silent-token, and ACP conversion behavior. Add focused tests proving child subagent model/lightContext behavior remains separate from announce-run model/context behavior, and document the new config on the subagents and ACP delivery docs.
