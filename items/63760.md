## Summary

Keep open. Current main still lacks announce-specific model selection and light/no-history context controls for subagent and ACP completion handoff; existing knobs cover spawned child runs and announce timeout, not the internal announce run itself.

## Best Possible Solution

Keep this open and implement a schema-backed announce policy, likely under agents.defaults.subagents.announce and mirrored where per-agent subagent defaults apply. The policy should let internal completion handoff choose a cheaper allowed model and a light or no-history context mode only for announce runs, while keeping child-spawn model/lightContext behavior separate. Add focused tests around native subagents, ACP completions, nested requester sessions, and unauthorized model overrides, then document the new knobs on https://docs.openclaw.ai/tools/subagents and https://docs.openclaw.ai/tools/acp-agents.
