## Summary

Keep open. Current main still sends subagent/ACP completion announces through the requester session without an announce-specific model override or lightweight context flag, while existing config only covers spawned child runs and announce timeout. The reported cost-control feature remains missing from core.

## Best Possible Solution

Keep the issue open and implement a narrow core-owned announce cost-control path: add documented config for completion announce runs, likely under `agents.defaults.subagents` or an explicit announce subsection, supporting an allowlisted model override and a lightweight/minimal context mode for the requester-session announce call. The implementation should cover native subagent and ACP run-mode completions, preserve delivery/idempotency behavior, and add regression tests proving the gateway `agent` call receives the cheaper model and lightweight bootstrap/context setting only for announce runs.
