## Summary

Keep #56692 open. Current main still lacks per-message group-chat targeting metadata in the shared prompt context and Telegram population path. The code only exposes current-turn `WasMentioned`, unannotated reply context, and unannotated history entries, while the prior PR that proposed `mentionedBot`/`repliedToBot` fields was closed unmerged. Community triage/plugin workarounds are useful, but they do not close the bundled Telegram/core prompt-context gap described by the issue.

## Best Possible Solution

Keep the issue open and implement a stable targeting layer in the shared group-context contract plus the Telegram population path: extend history/reply context types with target classification or bot identity fields, populate them from Telegram mention entities and reply chains where possible, render those fields in `buildInboundUserContextPrefix` and group guidance, and add focused tests for current-agent, other-agent, and ambient history entries. If maintainers prefer this to be plugin-extensible, add the Plugin SDK seam first instead of closing the issue as workaround-only ClawHub scope.
