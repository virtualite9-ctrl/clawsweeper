## Summary

Keep #56692 open. Current main still lacks the requested per-message targeting metadata in group history/reply context: `InboundHistory` only carries sender/body/timestamp, Telegram maps group history into those same fields, and the rendered prompt context exposes only current-turn `was_mentioned` plus unannotated reply/history blocks. The prior PR #58102 proposed the exact missing fields (`mentionedBot`, `repliedToBot`, `OtherBotUsernames`, `ReplyToBotUsername`) but was closed unmerged, so it does not supersede or implement this issue. The linked community triage gate and multi-agent kit are useful workarounds/adjacent implementations, but they do not close the bundled Telegram/core prompt-context gap described here.

## Best Possible Solution

Keep the issue open and implement the missing targeting layer in the shared group-context contract plus Telegram population path: extend the history/reply context types with a stable target classification or bot identity fields, populate them from Telegram mention entities and reply chains where possible, render them in `buildInboundUserContextPrefix`/group guidance, and add focused tests for history entries addressed to the current bot, another bot, and ambient group context. If maintainers want this to remain plugin-extensible, add the SDK seam first rather than closing this as a workaround-only ClawHub item.
