## Summary

Close as implemented on main. Current main already solves the stale fallback-context problem with a stronger live-resolver/live-getter design, and the same implementation is present in release v2026.4.22.

## Evidence

- **Fallback now resolves the live gateway context:** `startGatewayServer` creates one `gatewayRequestContext` and registers it through `setFallbackGatewayContextResolver(() => gatewayRequestContext)`, so plugin fallback dispatch no longer depends on a startup-era snapshot.
  - file: [src/gateway/server.impl.ts:766](https://github.com/openclaw/openclaw/blob/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503/src/gateway/server.impl.ts#L766)
  - sha: [b0d2003fe65b](https://github.com/openclaw/openclaw/commit/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503)
- **Cron and cronStorePath are read live from runtime state:** `createGatewayRequestContext` exposes `cron` and `cronStorePath` as getters over `runtimeState.cronState`, so hot reload state swaps are observed automatically and without mutating the shared WS request context.
  - file: [src/gateway/server-request-context.ts:67](https://github.com/openclaw/openclaw/blob/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503/src/gateway/server-request-context.ts#L67)
  - sha: [b0d2003fe65b](https://github.com/openclaw/openclaw/commit/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503)
- **Regression coverage exists on main:** `server-request-context.test.ts` asserts the context sees updated cron/store values after `runtimeState.cronState` replacement, and `server-plugins.test.ts` asserts fallback dispatch uses the lazy resolver and prefers it over stale snapshots.
  - file: [src/gateway/server-request-context.test.ts:6](https://github.com/openclaw/openclaw/blob/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503/src/gateway/server-request-context.test.ts#L6)
  - sha: [b0d2003fe65b](https://github.com/openclaw/openclaw/commit/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503)
- **Shipped in the latest release tag:** The v2026.4.22 release commit already contains `setFallbackGatewayContextResolver` in `src/gateway/server.impl.ts` and the live getter implementation in `src/gateway/server-request-context.ts`, so this was already shipped.
  - command: `git show 00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5:src/gateway/server.impl.ts | rg -n "setFallbackGatewayContextResolver|createGatewayRequestContext"`
