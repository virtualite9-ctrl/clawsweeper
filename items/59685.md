## Summary

Keep PR #59685 open. Current main still lacks the core behavior this PR proposes: `openclaw status --deep` does not ask the live gateway for memory runtime health when the local memory snapshot is null, the doctor payload still has only embedding health, and the memory runtime bootstrap still goes straight to a config-derived plugin load when no runtime is already materialized. The PR discussion also includes useful validation from #57256 and a separate memory-qdrant reproduction, so this is not a low-signal or obsolete PR.

## Best Possible Solution

Keep the PR open and ask for maintainer review/rebase as needed. The best code path is to land or rework the same core behavior: reuse compatible active plugin registry state before config-only memory bootstrap, expose live gateway memory runtime health separately from embedding probe health, make `status --deep` consume that live health when local status cannot materialize the runtime, preserve legacy gateway payload compatibility, and keep regression coverage for the linked mem0/qdrant false-negative scenarios plus memory capability cache preservation.
