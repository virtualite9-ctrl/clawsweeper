## Summary

Keep open. Current main still resolves automatic TTS from the global `messages.tts` config, `agents.list` does not expose a per-agent TTS/voice field, and the TTS runtime path does not carry an `agentId` into config resolution. The linked PR #44583 attempted this feature but was closed unmerged, so it does not supersede the issue on main.
