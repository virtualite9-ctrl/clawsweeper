## Summary

Keep this PR open. Current main still does not support the documented Telegram SOCKS5 proxy path, and the linked media-download bug remains observable in source. The PR is a real implementation candidate, but it needs focused rework before merge because the current patch does not update the media SSRF dispatcher path and over-claims SOCKS variants that undici 8.1.0 rejects.

## Best Possible Solution

Keep this PR open and request focused rework: add a shared typed SOCKS dispatcher seam for the schemes undici actually supports, wire it through makeProxyFetch, Telegram explicit-proxy transport, and the guarded Telegram media fetch path, remove or separately implement unsupported socks4/socks4a/socks5h claims, and add regression tests for channels.telegram.proxy with socks5:// covering both Bot API transport and media downloads while preserving SSRF policy behavior.
