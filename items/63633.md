## Summary

Keep #63633 open. Current main still has the Telegram SOCKS media-download gap, but this PR is not merge-ready as-is: the observable failure path is the guarded media fetch rejecting non-HTTP explicit proxy URLs, while the PR only changes proxy wrapper/Telegram dispatcher construction and overclaims SOCKS4/SOCKS5H support.

## Best Possible Solution

Keep the PR open but request a rework around the real current-main failure path: add SOCKS-aware support to the shared guarded dispatcher/media path used by Telegram downloads, restrict claimed schemes to those actually supported by the chosen dependency or add a proven SOCKS4 implementation, preserve Telegram dispatcher pool options, and add regression tests that cover Telegram media fetch through dispatcherAttempts plus the supported SOCKS scheme matrix.
