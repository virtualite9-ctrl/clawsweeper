## Summary

Keep this issue open. Current main still uses strict gateway protocol negotiation and macOS app/gateway version checks, with no grace-period, offline compatibility mode, or documented/configured bypass. The linked #65081 is a closed release-cadence request and does not supersede this narrower gateway/client version-skew problem.

## Best Possible Solution

Keep this open and narrow it into a gateway compatibility policy. Maintainers should define supported app/gateway and client/server skew rules, add an explicit supported protocol range or version negotiation contract where safe, document offline/remote behavior, and add regression tests for old/new client negotiation. Any bypass should distinguish package/app version skew from wire-protocol incompatibility.
