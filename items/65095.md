## Summary

Keep this issue open. Current main still has strict gateway/client protocol negotiation, strict macOS app-to-gateway version checks, and no documented or configured grace-period/offline compatibility mode. The linked #65081 covered release cadence and is now closed, but it does not supersede this narrower gateway version-skew request.

## Best Possible Solution

Keep this open and narrow it into a gateway compatibility policy. Maintainers should decide supported app/gateway and client/server skew rules, add an explicit supported protocol range or version negotiation contract where safe, document offline/remote behavior, and add tests for old/new client negotiation. If a compatibility bypass is accepted, it should be scoped so it cannot silently run incompatible protocol schemas.
