## Summary

Keep this PR open. Current main still lacks the central OAuth `Glob` -> local `find` alias and still does not register `find`/`grep`/`ls` in `createOpenClawCodingTools()`. The PR is a real implementation candidate, but the submitted patch is not ready as-is because it directly appends upstream filesystem discovery tools without the OpenClaw workspace/sandbox guard and policy catalog integration used by existing file tools.

## Best Possible Solution

Keep this PR open as a real but incomplete implementation candidate. The best path is a focused revision or successor that registers `find`/`grep`/`ls` through guarded OpenClaw wrappers, adds them to the filesystem catalog/profile/group policy model and docs, and adds regression tests for Anthropic OAuth name mapping plus `tools.fs.workspaceOnly` and `group:fs` allow/deny behavior.
