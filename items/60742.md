## Summary

Keep open. Current `main` has related memory, hook, and post-compaction plumbing, but it does not implement the requested recent/top-of-mind continuity layer. The issue is recent, concrete, and backed by a linked draft PR (#60743), so conservative cleanup does not support closing it yet.

## Evidence

- **Reset/new only save dated archive-style memory notes:** The bundled `session-memory` hook fires on `/new` and `/reset`, reads recent session content, and writes a dated file under `<workspace>/memory/YYYY-MM-DD-slug.md`. It does not create a `memory/recent/latest.md` layer or structured active-task snapshot.
  - file: [src/hooks/bundled/session-memory/handler.ts:53](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/src/hooks/bundled/session-memory/handler.ts#L53)
  - sha: [6bc0dc8fb671](https://github.com/openclaw/openclaw/commit/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440)
- **Compaction side effects do not write a recent continuity snapshot:** The current compaction path emits transcript updates and optional post-compaction session-memory indexing. The available before/after compaction hooks are extension points, but `main` itself does not write the requested recent continuity artifact during compaction.
  - file: [src/agents/pi-embedded-runner/compaction-hooks.ts:82](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/src/agents/pi-embedded-runner/compaction-hooks.ts#L82)
  - sha: [6bc0dc8fb671](https://github.com/openclaw/openclaw/commit/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440)
- **Post-compaction recovery only rereads AGENTS.md sections:** `readPostCompactionContext` opens `<workspace>/AGENTS.md`, extracts configured sections, and injects them after compaction. It does not read workspace memory files or a recent continuity layer first.
  - file: [src/auto-reply/reply/post-compaction-context.ts:66](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/src/auto-reply/reply/post-compaction-context.ts#L66)
  - sha: [6bc0dc8fb671](https://github.com/openclaw/openclaw/commit/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440)
- **Related hooks and plugin seams already exist, so the request is coherent and still actionable:** Docs expose `command:new`, `command:reset`, and `session:compact:before` internal hooks, plus plugin hooks for `before_reset` and `before_compaction`. That supports the issue's proposed trigger points rather than contradicting them.
  - file: [docs/automation/hooks.md:36](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/docs/automation/hooks.md#L36)
