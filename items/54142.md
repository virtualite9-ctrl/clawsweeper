## Summary

Keep open. Current main only partially addresses the report: MiniMax VLM now has a 60s AbortSignal timeout, but the Ollama pull path and both native PDF provider fetches still lack a per-call timeout/signal. The narrower related reports were closed as duplicates into this umbrella, and the open PRs #54144 and #54145 target parts of the remaining work but are unmerged.

## Best Possible Solution

Keep this open. Review/land or replace #54144 for the Ollama /api/pull timeout and #54145 for the Anthropic/Gemini PDF timeouts, or make equivalent changes directly on main. Add regression coverage that asserts timeout/signal propagation for the Ollama pull, Anthropic PDF, and Gemini PDF request paths; MiniMax can remain as-is for this issue because current main already applies a 60s AbortSignal timeout there.
