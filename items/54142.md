## Summary

Keep open. Current `main` only fixes the MiniMax VLM call; the Anthropic PDF, Gemini PDF, and Ollama pull paths still lack an explicit timeout on the outbound request path, so the umbrella report is only partially obsolete.

## Risks / Open Questions

- The report is partially stale because the MiniMax path now has a 60s timeout, so any eventual issue update should narrow the scope to the remaining Ollama/PDF paths.
- The Ollama `/api/pull` call is a streaming download. A naive hard deadline could abort legitimate long pulls, so the remaining fix likely needs a more careful connect/idle-timeout design than the issue's simple suggestion.
- Current tests for `src/agents/tools/pdf-native-providers.ts` and `extensions/ollama/src/setup.ts` exercise request shape and behavior, but they do not assert timeout wiring, so this regression surface is still unguarded.
