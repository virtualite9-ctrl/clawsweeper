## Summary

Keep this PR open. Current main still does not implement the PR's central behavior for reasoning-capable Gemini 2.5 Flash in the negative `thinkingBudget` sanitization path: the shared Google sanitizer deletes the invalid budget but does not inject a replacement `thinkingLevel`, and the wrapper still passes only `model.id`, not `model.reasoning`. The PR is now stale against refactored current code and should be rebased onto the shared provider stream helper, but it remains plausible useful bug-fix work.

## Best Possible Solution

Keep this PR open and ask for a rebase onto the current shared Google stream wrapper. The likely maintainer path is to extend `sanitizeGoogleThinkingPayload` or its wrapper context so it can use resolved model reasoning metadata, add the Gemini 2.5 Flash fixed-level regression test in the current Google wrapper/provider-stream tests, and update the stale Gemini 3.1-only comment while preserving existing adaptive and Gemini 3 behavior.
