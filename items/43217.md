## Summary

Keep open. Current main still does not implement the PR's central behavior: the shared Google thinking sanitizer is driven by model id and thinking level, not resolved model reasoning capability, and the generic negative-budget path still only deletes invalid budgets. The PR is stale against the moved shared helper and likely needs a model-family-correct Gemini 2.5 budget mapping rather than direct thinkingLevel injection, but it remains plausible bug-fix work.

## Best Possible Solution

Keep this PR open and ask for a rebase onto the shared Google thinking helper. The maintainer path is to adapt `src/plugin-sdk/provider-stream-shared.ts` so the sanitizer can use resolved model capability or provider thinking profile, preserve existing Gemini 3/Gemma/adaptive behavior, repair fixed-level Gemini 2.5 payloads with the model-family-correct budget mapping, add a `gemini-2.5-flash` fixed-level regression test plus a non-reasoning negative case, and update the stale Gemini-3.1-only comment in `src/agents/pi-embedded-runner/extra-params.ts`.
