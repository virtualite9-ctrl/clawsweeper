## Summary

Keep open. Current main has not implemented the PR's central behavior: the shared Google thinking sanitizer still accepts only model id and thinking level, and the generic negative-budget fallback still strips invalid budgets without using resolved model reasoning capability or repairing fixed Gemini 2.5 levels. The PR is stale against moved shared helper code and likely needs adjustment to the current Gemini 2.5 budget mapping, but it remains plausible bug-fix work rather than cleanup to close.

## Best Possible Solution

Keep this PR open and ask for a rebase onto the shared Google thinking helper. The maintainer path is to update `src/plugin-sdk/provider-stream-shared.ts` so the sanitizer can use resolved model capability or provider thinking profile, preserve existing Gemini 3/Gemma/adaptive behavior, repair fixed-level Gemini 2.5 payloads with the model-family-correct budget mapping, add a `gemini-2.5-flash` fixed-level regression test plus a non-reasoning negative case, and update any stale Gemini-3-only wording.
