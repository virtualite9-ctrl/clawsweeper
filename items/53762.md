## Summary

Keep open. Current `main` at `8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf` still has the segmented-final early return that drops a buffered answer when reasoning was hinted but its delivery was skipped, so this is not implemented on main. The proposed flush is also not a clean close candidate by itself because the existing flush helper bypasses the `preview-finalized` hook path that normal segment delivery uses.

## Evidence

- **Current main still returns before flushing buffered segmented finals:** In `deliver`, final segmented payloads loop over reasoning/answer segments and then `return` on `segments.length > 0` without calling `flushBufferedFinalAnswer()`. That leaves a buffered answer stranded on current main.

  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Buffering path still exists on current main:** A final answer segment is buffered whenever reasoning was only hinted, and reasoning is only marked delivered when the reasoning lane result is not `skipped`. If reasoning delivery is skipped, the answer stays buffered and the later early return drops it.
