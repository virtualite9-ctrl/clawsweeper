## Summary

Current main does not yet implement the reported duplicate-delivery fix. The 2026.4.24 regression has an active related PR (#72290), but that PR is still open and its proposed root-cause fixes are not present on main, so the issue should remain open as the canonical bug report until a fix lands.

## Best Possible Solution

Keep #72176 open until #72290 or an equivalent narrow fix lands on main with regression coverage. The fix should add stable duplicate suppression for retry paths without stable `MessageSid`, carry the chat run idempotency key through user transcript updates, and test that duplicate webchat/channel retry updates do not create repeated session-history user and assistant entries.
