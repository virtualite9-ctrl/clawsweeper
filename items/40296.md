## Summary

Keep this PR open. Current main still has the core cache-stability problem the PR targets: channel/session-sensitive Messaging and Reactions text are assembled before the stable Project Context and before the internal cache boundary. The PR is a plausible implementation candidate, but it is stale against current main's newer SYSTEM_PROMPT_CACHE_BOUNDARY design and current test layout, so the right maintainer path is to refresh it rather than close it.

## Best Possible Solution

Keep this PR or its linked issue open, then refresh the implementation against current main. Preserve SYSTEM_PROMPT_CACHE_BOUNDARY, keep stable Project Context in the reusable prefix, move or split runtime/channel/session-sensitive text such as inline-button Messaging guidance and Reactions so it lands after the stable prefix or boundary, and add a current-layout regression test proving Project Context/cache-boundary ordering for those dynamic sections.
