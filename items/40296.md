## Summary

Current main still leaves runtime-sensitive Messaging and Reactions text before the stable Project Context block and before the internal cache boundary, so the PR is not obsolete. It should stay open, but needs a refresh against current main because the test file path in the PR no longer exists and main now has a cache-boundary system the fix must preserve.

## Best Possible Solution

Keep this PR or linked #40256 open and refresh the implementation against current main. Preserve SYSTEM_PROMPT_CACHE_BOUNDARY, move or split channel/session-sensitive text such as inline-button runtime guidance, Group Chat Context, Reactions, and comparable per-session additions so the large stable Project Context remains in the reusable prefix, then add a current regression test asserting the Project Context/cache-boundary ordering for those sections.
