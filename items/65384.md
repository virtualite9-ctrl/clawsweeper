## Summary

Keep this PR open. Current main still does not migrate legacy Feishu `channels.feishu.accounts.<id>.botName` into the supported `name` field, while the related regression #65177 is still open and this PR is a focused implementation candidate. The PR discussion also shows useful review follow-up: Greptile's arrow-style nit was addressed, and the later blank-name migration edge case was addressed by the PR head commit.

## Best Possible Solution

Keep this PR open for maintainer review, CI, and landing, or land an equivalent Feishu-owned doctor migration. The fix should stay inside the Feishu plugin boundary, expose `legacyConfigRules` and `normalizeCompatibilityConfig` through the bundled contract surface, wire the Feishu channel doctor adapter if needed, and keep regression coverage for normal migration, existing-name conflicts, and blank-name handling. After merge, #65177 can close as implemented.
