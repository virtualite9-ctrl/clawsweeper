## Summary

Keep this PR open. Current main still does not migrate legacy Feishu `channels.feishu.accounts.<id>.botName` to the supported `name` key, and the related regression #65177 remains unresolved. The PR is a narrow Feishu-owned implementation candidate; it should get maintainer review, CI, and likely stronger direct coverage rather than being closed as obsolete.

## Best Possible Solution

Keep this PR open for maintainer review and CI, or land an equivalent Feishu-owned doctor migration. The implementation should stay inside the Feishu plugin boundary, expose `legacyConfigRules` and `normalizeCompatibilityConfig` through the bundled contract surface, wire the Feishu channel doctor adapter, and include regression coverage for normal migration, existing-name conflict removal, and blank-name promotion. After merge, #65177 can close as implemented.
