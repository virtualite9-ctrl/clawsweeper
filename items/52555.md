## Summary

Keep PR #52555 open. Current main still only documents Azure Linux VM hosting and has no Azure Container Apps guide, redirect, VPS card, or platforms hub entry, so the PR is not implemented or superseded. The PR is a concrete docs contribution in an existing hosting-guide surface, but its head still needs revision because it uses an unsupported gateway bind mode in the launch snippets.

## Best Possible Solution

Keep the PR open and request revisions: replace `--bind all` with a supported bind mode such as `lan` or `custom` plus `gateway.customBindHost`, keep the placeholder-image managed-identity ACR flow, verify the auth token and `gateway.controlUi.allowedOrigins` setup against current gateway startup rules, and add fresh docs/link plus live Azure deployment verification before merge.
