## Summary

Keep this PR open. Current main still does not implement configurable `embedding.timeoutMs` or `embedding.maxRetries` for `memory-lancedb`, and the PR remains a live implementation candidate rather than obsolete cleanup. It needs maintainer review for current-main drift, the `embedding.baseUrl` API-key/security concern, and unexplained generated checksum drift.

## Best Possible Solution

Keep this PR open as the active implementation candidate. The best path is a refreshed, scoped `memory-lancedb` change that adds bounded `embedding.timeoutMs` and `embedding.maxRetries` support in config parsing, manifest schema/UI hints, runtime OpenAI construction/request options, and focused tests, while preserving `encoding_format: "float"` and resolving the `embedding.baseUrl` security concern plus generated checksum drift before merge.
