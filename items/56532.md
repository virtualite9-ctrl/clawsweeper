## Risks / Open Questions

- The PR is still a live implementation candidate; closing it now would drop unshipped behavior that current main still lacks.
- The remaining security/product question is real: configurable maxRetries can increase requests to an existing user-configurable embedding.baseUrl, which also currently supports environment-variable expansion.
- The current PR adds config/manifest tests but no focused runtime assertion that OpenAI is constructed with the configured timeout and retry values.
- The parser comments say invalid values fall back to the SDK default, while the runtime constructor defaults undefined values to OpenClaw's 10s/1 settings; the semantics should be clarified before merge.
- The PR body still mentions a LaunchAgent ThrottleInterval bump, but the current diff no longer changes launchd code, so the scope text should be cleaned up.
