## Summary

Keep open. Current main still does not implement configurable `embedding.timeoutMs` or `embedding.maxRetries` for the bundled `memory-lancedb` plugin. The current plugin config, manifest schema, UI hints, runtime OpenAI client construction, and tests all still use the old embedding call path. The PR is recent, scoped to the bundled plugin, and the provided discussion shows review comments were actively addressed rather than making the work obsolete.

## Best Possible Solution

Keep this PR open. The best path is a scoped `memory-lancedb` change that adds validated `embedding.timeoutMs` and `embedding.maxRetries` to `extensions/memory-lancedb/config.ts`, `extensions/memory-lancedb/openclaw.plugin.json`, UI hints, and the OpenAI embedding constructor/request path, with focused config/runtime tests. Before merge, maintainers should resolve the `embedding.baseUrl` security concern and clean up the stale LaunchAgent wording or handle that change separately.
