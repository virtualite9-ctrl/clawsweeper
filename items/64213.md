## Summary

Current `main` does not appear to implement the requested fallback. The gateway image-capability check still returns `false` when a provider/model lookup misses the runtime catalog, and current tests encode that fail-closed behavior. That conflicts with the documented config contract for explicit `models.providers.*.models[].input` metadata, so this issue should remain open.

## Evidence

- **No config fallback in image-capability resolver:** `resolveGatewayModelSupportsImages()` only checks the loaded gateway catalog, then applies special-case shims for `microsoft-foundry` and `claude-cli`, and otherwise returns `false`. It does not consult `models.providers.<id>.models[].input` on a catalog miss.
