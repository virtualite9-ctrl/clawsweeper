## Summary

Keep open. Current main still does not implement the requested runtime fallback from a gateway catalog miss to explicit `models.providers.<provider>.models[].input` metadata. The issue discussion includes a real deployment reproduction and a confirming comment that this is distinct from onboarding issue #61716. The related PR #67766 is an open implementation candidate, but it is not merged in the provided context, so this issue should not be closed yet.

## Best Possible Solution

Keep this issue open. The best maintainer path is to review and land #67766 if it is sound, or replace it with a focused patch in `src/gateway/session-utils.ts` that lets `resolveGatewayModelSupportsImages()` consult normalized explicit provider model metadata on catalog miss. Coverage should include explicit provider refs, unique provider inference from config, catalog miss plus config hit, text-only config, catalog-hit precedence, and the chat, agent RPC, and node-event attachment paths.

## Evidence

It does not accept `cfg`, call `loadConfig()`, or inspect `models.providers[provider].models[].input`.
