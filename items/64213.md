## Summary

Keep this issue open. Current main still does not implement the requested fallback from a gateway catalog miss to explicit `models.providers.<provider>.models[].input` metadata, and an open implementation candidate (#67766) has not merged. The related #61716 remains a separate onboarding/config-collection issue, not a runtime fix for this path.

## Best Possible Solution

Keep this issue open and fix the runtime gateway image-capability check. The maintainer-grade path is to review or revise #67766, then land a focused implementation that lets `resolveGatewayModelSupportsImages()` consult normalized explicit provider model metadata on catalog miss without breaking provider-scoped routing. Add regression coverage for explicit provider, inferred provider, catalog miss plus config hit, text-only config, and chat/agent/node-event attachment handling.
