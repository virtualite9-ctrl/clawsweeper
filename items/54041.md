## Summary

Keep open. Current main still only partially supports Anthropic request shaping: `anthropicBeta`/`context1m` can affect beta headers, but the requested Anthropic `context_management` body support, context-editing/compaction semantics, and `memory_20250818` native tool path are not implemented or documented. The production +1 in the discussion is concrete, and the narrower related trackers do not fully supersede this broader request.

## Best Possible Solution

Keep this open for maintainer/product design. A maintainer-quality implementation likely belongs in the bundled Anthropic provider, with documented model params for Anthropic beta/header handling, validated `context_management` body shaping, tests for header and payload mutation, streaming/replay handling for provider compaction blocks and stop reasons, and a decision on whether `memory_20250818` can be implemented as a provider wrapper over the existing memory plugin or needs a new provider-defined-tool SDK seam.
