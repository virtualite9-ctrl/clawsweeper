## Summary

Keep #59225 open. Current main still builds `sessions_spawn` as a single hard-coded schema and only supports tool-level allow/deny/profile filtering, not per-agent parameter allowlists or denylists. The related #68397 is a partial open workaround for ACP-only fields, not an implementation of this requested schema-filtering feature.

## Best Possible Solution

Keep this open for a core design/implementation. The likely path is to add a documented config contract for per-tool schema property filtering under agent/global tool policy, validate it in the config schema, apply it after whole-tool policy resolution and before provider schema normalization, and add focused coverage for `sessions_spawn` so a dispatcher agent can advertise only `task`, `agentId`, `mode`, and `runTimeoutSeconds` while runtime validation continues accepting existing fields when explicitly provided by compatible callers. #68397 can be handled as a separate short-term fix but should not close this feature request.
