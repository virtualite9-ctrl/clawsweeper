## Summary

Keep #48534 open. Current main still has no timeout boundary around `before_agent_start`: the hook runner awaits each handler directly, the model-selection path calls the legacy hook before the embedded attempt timeout is scheduled, and the prompt-build path also awaits the same hook without passing an abort signal. The upstream memory-openviking fix and the closed unmerged PR #48741 help context, but they do not implement the requested OpenClaw core guard on main.

## Best Possible Solution

Keep the issue open and implement a core hook-runner timeout/diagnostic boundary rather than relying only on individual plugins. The fix should likely live in `src/plugins/hooks.ts`, apply at least to `before_agent_start` and probably all async plugin hook classes that can block a run, include plugin id, hook name, elapsed ms, and run id in warnings where available, add a typed/configured timeout surface if maintainers want it, and cover both model-resolve and prompt-build call paths with regression tests using a never-settling hook.
