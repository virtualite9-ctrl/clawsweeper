## Summary

Keep open. Current main still does not implement `openclaw models status --probe --all`: neither the Commander CLI nor routed CLI parses a status-level `--all` flag, `modelsStatusCommand` has no `probeAll` option, and the probe target builder still chooses a single candidate model per provider. The linked PR #63259 is the right implementation candidate, but it is open and unmerged in the provided GitHub context, so this issue should remain open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review #63259 or a successor PR. The fix should add a `models status --probe --all` flag to both Commander and routed CLI paths, thread a `probeAll` option through `modelsStatusCommand` into `runAuthProbes`, expand probe targets across every configured primary/fallback/image candidate while preserving provider/profile filters and current default behavior, then add focused target-builder, CLI/JSON, and docs coverage before closing as implemented.
