## Summary

Keep open. Current main still rejects the requested partial `agents.defaults.cliBackends.<id>` override because the Zod schema and TS config type still require `command`. The runtime merge/resolution path already appears compatible with inheriting a base command, so the PR addresses a real bug, but it should also update the generated public config schema before it is merge-ready.

## Best Possible Solution

Keep the PR open and finish the focused fix: make `CliBackendSchema.command` and `CliBackendConfig.command` optional for input config, preserve a resolved backend type with non-empty `command`, update `src/config/schema.base.generated.ts` and any generated schema checks, and add regression coverage for both `OpenClawSchema`/public schema acceptance and `resolveCliBackendConfig` inheriting the base command for a `modelAliases`-only override.
