## Summary

Keep open. Current main still rejects partial `agents.defaults.cliBackends.<id>` overrides because `command` remains required in the runtime zod schema, TypeScript config type, and generated JSON schema. The PR is a focused fix for a real config-contract mismatch, but the submitted diff is incomplete unless it also regenerates the base config schema artifact flagged in review.

## Best Possible Solution

Keep PR #69123 open and update it rather than auto-close it. The best fix is to make `CliBackendSchema.command` and `CliBackendConfig.command` optional for input overrides, introduce or use a resolved backend type with `command: string` after `resolveCliBackendConfig`, add the modelAliases-only regression test, regenerate `src/config/schema.base.generated.ts` and any config docs baseline required by the repo gates, then run the changed config/agents lanes.
