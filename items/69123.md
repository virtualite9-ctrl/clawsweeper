## Summary

Keep open. Current main still rejects the partial `agents.defaults.cliBackends.<id>` override described by the PR because `CliBackendSchema.command` and `CliBackendConfig.command` are still required. The PR addresses a real config/runtime mismatch, but its provided diff is incomplete because the generated public config schema still marks `command` as required and must be regenerated/checked before merge.

## Best Possible Solution

Keep the PR open and revise it into a complete fix: make `CliBackendConfig.command` and `CliBackendSchema.command` optional for source overrides, preserve a resolved backend type/guard where `command` is guaranteed non-empty after `resolveCliBackendConfig`, add the modelAliases-only regression test, and regenerate/check the base config schema so public schema consumers align with runtime validation.
