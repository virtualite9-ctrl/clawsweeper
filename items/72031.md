## Summary

Keep #72031 open. Current main still has the reported source-level failure path: Bedrock `aws-sdk` auth can resolve without a static API key, chat/simple completion has an allowance for that case, but the media-understanding image and audio/video resolver paths still call `requireApiKey` unconditionally before the provider call. The linked fix PRs #72066 and #72092 are open in the provided GitHub context, not shipped on main.

## Best Possible Solution

Keep #72031 open until a fix like PR #72092 lands. The preferred implementation is a narrow media-understanding auth change that allows missing API keys only for `auth.mode === "aws-sdk"`, passes `apiKey: ""` or `undefined` through to the Bedrock-capable provider call as appropriate, avoids storing empty runtime keys, preserves existing throws for non-`aws-sdk` modes, and adds regression coverage for `amazon-bedrock` image describe plus the audio/video resolver path.
