## Summary

Keep #72031 open. Current main still has the reported auth bug in the image/media-understanding paths: Bedrock `aws-sdk` auth can resolve with no static API key, but these paths still call `requireApiKey` unconditionally. Open PR #72092 appears to be the right canonical fix path, but it is still unmerged, so the issue remains actionable.

## Best Possible Solution

Keep the issue open until PR #72092 or an equivalent maintainer-approved fix lands. The best fix is to mirror the chat path's `aws-sdk` missing-key allowance in `src/media-understanding/image.ts` and `src/media-understanding/runner.entries.ts`, skip storing empty runtime keys, preserve existing errors for non-`aws-sdk` modes, and add regression tests for Bedrock image/media auth with no static API key.
