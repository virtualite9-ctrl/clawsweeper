## Summary

Keep PR #70681 open. Current main still uses the Unix `lsof -Fpc` command-name field to classify gateway PIDs, and the related macOS bug report #70664 remains open. The PR is a focused, relevant fix, but it still needs the existing review comments addressed before maintainer merge consideration.

## Best Possible Solution

Keep PR #70681 open. The best path is to preserve the Unix ps-argv verifier approach, update `pollPortOnce` to pass `POLL_SPAWN_TIMEOUT_MS` into the verifier, optionally switch the affected lsof calls from `-Fpc` to `-Fp` after command fields are unused, refresh the tests that currently pin lsof `c`-field behavior, and then run the targeted infra restart and gateway CLI checks before maintainer review.
