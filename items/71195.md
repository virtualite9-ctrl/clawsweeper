## Summary

Keep this issue open. Current main has the backend/browser OpenAI Realtime Talk path, but the requested native macOS Talk integration is still missing, and maintainer discussion explicitly narrowed the remaining work to the macOS client consuming the existing realtime session flow.

## Best Possible Solution

Keep this issue open as the canonical native macOS Talk realtime follow-up. The likely implementation path is to add a GatewayConnection wrapper for talk.realtime.session, add a native realtime audio/WebRTC session object modeled on ui/src/ui/chat/realtime-talk.ts, wire TalkModeRuntime and overlay phase/cancellation into that session, and preserve the current SFSpeechRecognizer -> chat.send -> talk.speak path as fallback.
