## Summary

Keep open. Current `main` has Windows Scheduled Task fallback logic, tests for fallback launch behavior, and config-recovery code for missing `gateway.mode`, but there is not strong evidence that the reported real-world symptom on Windows 11 is fully fixed. The issue discussion also shows a concrete manual workaround (`openclaw config set gateway.mode local` + `openclaw gateway start`), and the current docs still leave some ambiguity around whether `install.ps1` alone should leave the gateway/dashboard ready without an explicit onboarding step.

## Evidence

- **Reporter workaround shows a real config/startup failure, not just a vague complaint:** Issue comment says `openclaw gateway install` succeeded, but `openclaw gateway` was blocked until `openclaw config set gateway.mode local` was run, after which `openclaw gateway start` worked. The same comment says the user still had to do this after each Windows restart.
- **Gateway still intentionally refuses to start when config exists but `gateway.mode` is missing:** `getGatewayStartGuardErrors()` blocks startup when `gateway.mode` is unset and tells the user to rerun onboarding/setup or set `gateway.mode=local` manually. That matches the reporter's observed failure mode rather than disproving it.
  - file: [src/cli/gateway-cli/run.ts:223](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/src/cli/gateway-cli/run.ts#L223)
  - command: `nl -ba src/cli/gateway-cli/run.ts | sed -n '220,260p'`
  - sha: [926068b14fc6](https://github.com/openclaw/openclaw/commit/926068b14fc661e915a0247e9ecb5949fb95b4f7)
- **Current main added recovery for configs whose last-good version had `gateway.mode` but the current file lost it:** The config observe/recovery path flags `gateway-mode-missing-vs-last-good` as suspicious and treats it as recoverable; the test explicitly verifies restoring `gateway.mode: "local"` from backup. This is relevant improvement, but it is not proof that the Windows reboot/service problem from #60490 is gone end-to-end.
