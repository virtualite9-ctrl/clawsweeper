## Summary

Keep open. Current main at 110fa97 still does not implement `memorySearch.ignorePaths` or an equivalent exclude contract. The builtin memory indexer and QMD default collection still include recursive `memory/**/*.md` scope, including `memory/dreaming/**`, and the watcher ignore sets remain hardcoded dependency/cache-directory filters that omit `dreaming`. The prior Codex review conclusion still holds; related dreaming promotion fixes reduce `MEMORY.md` contamination but do not solve `memory_search` indexing or watcher scope.

## Best Possible Solution

Keep this issue open and implement a real memory-search ignore contract if maintainers want this behavior: add schema/type/help/docs support, merge default and per-agent ignore patterns, apply them consistently to builtin `listMemoryFiles`, builtin watcher setup, QMD collection/update/watch scope, and reindex scope hashing, then add focused tests for `memory/dreaming/**`, dependency dirs, `extraPaths`, QMD/builtin parity, and watcher/index parity. Maintainers should separately decide whether `memory/dreaming/` is an opt-in ignore as proposed or a default exclusion.
