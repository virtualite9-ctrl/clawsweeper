## Summary

Keep #71285 open. Current main has no `memorySearch.ignorePaths` or equivalent exclusion config, and the builtin memory indexer still recursively includes `.md` files under `memory/`, which covers documented `memory/dreaming/<phase>/YYYY-MM-DD.md` reports. The watcher has a hardcoded dependency-dir ignore list, but it does not include `dreaming` or consume user-configured ignore paths. Related #66852 and #68774 address promotion/MEMORY.md contamination, not search-index or watcher exclusion; #6362 attempted this feature but was closed unmerged.

## Best Possible Solution

Keep this issue open and implement a first-class memory-search exclusion contract. The likely path is to add `memorySearch.ignorePaths` to schema/types/config resolution, fold it into the configured scope hash so affected indexes reindex, apply the same matcher to `listMemoryFiles` and the chokidar ignore predicate, document it under the public memory configuration reference, and add regression tests for `memory/dreaming/**`, default dependency-directory ignores, extraPaths, and watcher/index parity. If maintainers prefer no new knob, an explicit core exclusion for managed dreaming artifacts should still be implemented and documented.

- A naive `ignorePaths` implementation could recreate the #6362 concern: chokidar and `listMemoryFiles` must share the same matching semantics so watch and indexing behavior do not diverge.
