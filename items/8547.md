## Summary

Keep #8547 open. Current main still does not provide a user-configurable Control UI/WebChat font-size or font-scale setting in browser-local UI settings, Appearance/Quick Settings, gateway `ui` config, docs, CSS variables, or focused tests. The closed related issue #3027 is not an active canonical tracker, and the existing issue discussion already kept this open for the remaining core UI accessibility/customization work.

## Best Possible Solution

Implement a bounded Control UI text-size or font-scale setting in `UiSettings`, expose it in Appearance and Quick Settings, apply it through CSS custom properties across the chat and surrounding UI, and add focused storage/rendering tests plus public docs. Extend the gateway `ui` schema/types/docs only if maintainers want server-side font sizing as a public config contract.
