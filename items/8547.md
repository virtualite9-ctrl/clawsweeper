## Summary

Keep #8547 open. Current main at e29d3516bf05591e6007bfb8ccdaea6d76194d41 still does not provide a user-configurable Web UI font size or font scale in browser-local settings, the Appearance UI, gateway `ui` config schema/types, chat CSS, or public configuration docs. The related #3027 is closed and does not act as an open canonical replacement, and the existing issue discussion already includes a maintainer Codex review explicitly keeping this item open.

## Best Possible Solution

Keep #8547 open for core Control UI accessibility/customization work. A concrete implementation path is to add a browser-local text-size/font-scale control under Appearance, persist it in `UiSettings`, apply it through CSS custom properties across chat messages, tool cards, sidebars, and compose inputs, and add gateway schema/types/docs/tests only if maintainers intentionally want server-side `ui` font settings as a public contract.
