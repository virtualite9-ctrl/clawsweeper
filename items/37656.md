## Summary

Keep open. Current `main` does not implement per-agent workspace `.env` loading at exec time. OpenClaw still hydrates dotenv once from `process.cwd()` during config bootstrap, and `createExecTool` builds host exec env from `process.env` plus explicit `params.env` only.

## Evidence

- **Config bootstrap loads one cwd-scoped .env into process env:** `maybeLoadDotEnvForConfig()` calls `loadDotEnv()` only for the real process env during config load, so dotenv hydration happens once at startup rather than per exec request or per agent workspace.
  - file: [src/config/io.ts:915](https://github.com/openclaw/openclaw/blob/55389f4be29359e2dd57ebaf548ba4a70331a445/src/config/io.ts#L915)
  - command: `nl -ba src/config/io.ts | sed -n '908,926p'`
  - sha: [55389f4be293](https://github.com/openclaw/openclaw/commit/55389f4be29359e2dd57ebaf548ba4a70331a445)
- **Dotenv loader reads only `process.cwd()/.env`:** `loadDotEnv()` resolves `cwdEnvPath = path.join(process.cwd(), ".env")` and loads that single workspace file before global fallback env files. There is no agent/workdir parameter here.
  - file: [src/infra/dotenv.ts:281](https://github.com/openclaw/openclaw/blob/55389f4be29359e2dd57ebaf548ba4a70331a445/src/infra/dotenv.ts#L281)
