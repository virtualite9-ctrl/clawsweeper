## Summary

Keep #37656 open. Current main still does not implement per-agent, exec-time workspace `.env` loading for `exec`; it only has startup/global CWD dotenv loading and host exec env construction from `process.env` plus request `params.env`. The PR discussion remains substantive and active rather than obsolete cleanup.

## Best Possible Solution

Keep the PR open as the canonical implementation path for per-agent workspace `.env` loading. The best maintainer path is a reviewed core `exec` change that loads the resolved gateway workspace `.env` at request time, preserves non-override semantics including Windows casing, filters dangerous host env overrides, skips node/sandbox hosts where the env is not used, adds isolated regression tests, and aligns the environment docs once shipped.
