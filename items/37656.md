## Summary

Keep #37656 open. Current main still does not implement per-agent, exec-time workspace `.env` loading for the `exec` tool; the existing behavior loads only startup/current-CWD dotenv into `process.env` and builds exec env from `process.env` plus request overrides. The PR discussion remains active and substantive, with the author addressing multiple review concerns rather than leaving an obsolete drive-by change.

## Best Possible Solution

Keep #37656 open as the canonical implementation path. The best maintainer path is a reviewed core `exec` change that loads the resolved gateway workspace `.env` at request time, preserves non-override semantics including Windows casing, filters dangerous host env overrides, skips node/sandbox hosts where workspace env is not executed, uses isolated regression tests, and updates environment docs once shipped.
