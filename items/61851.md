## Summary

Current `main` and the shipped `v2026.4.23` release already contain the bundled-plugin facade sentinel/backfill protections and regression coverage for the circular plugin/provider re-entry bug class behind these stack overflows.

## Evidence

- **Facade loader now pre-seeds a sentinel before loading:** `loadBundledPluginPublicSurfaceModuleSync` stores a sentinel in `loadedFacadeModules` before invoking the plugin module loader, then back-fills it with `Object.assign(...)`. That is the circular re-entry guard for bundled provider/plugin facades.
  - file: [src/plugin-sdk/facade-loader.ts:201](https://github.com/openclaw/openclaw/blob/8b76392e3e7934b1cdd6d461148263fd56141ec8/src/plugin-sdk/facade-loader.ts#L201)

- **Public-surface loader carries the same cache-first sentinel pattern:** The generic bundled public-surface loader also inserts a sentinel into `loadedPublicSurfaceModules` before module evaluation and back-fills it after load, preventing recursive facade resolution from recursing until stack overflow.
