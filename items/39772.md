## Summary

Keep PR #39772 open. Current main has a newer Telegram native-quote path that skips draft preview streaming when the original message text/quote is available, which may reduce the original text-message symptom, but the draft-stream behavior targeted by this PR is still not implemented: `reply_to_message_id` is built once and reused across preview sends, `forceNewMessage()` does not consume or clear reply state, and the PR still has material review concerns around retryable first-send failures and `replyToMode: "all"` semantics.

## Best Possible Solution

Keep the PR open for maintainer or author revision. The best path is a Telegram draft-stream fix that suppresses the transient `Deleted message` preview artifact while preserving the configured reply-threading contract: distinguish `first` from `all` for streaming, consume reply context only after an actually delivered first message or restore it on retry/materialize recovery, preserve forum/thread parameters after rotation, and add regression coverage for forceNewMessage, thread params, retryable first-send failure/materialize recovery, and all-mode streaming behavior. Current main's native quote handling should be treated as related partial mitigation, not a complete replacement for that work.
