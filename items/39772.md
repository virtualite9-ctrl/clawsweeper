## Summary

Keep open. Current `main` does not implement this Telegram draft-stream change, and the open PR is not redundant. The code still reapplies `reply_to_message_id` on every new preview send after `forceNewMessage()`, while the surrounding Telegram reply-threading code distinguishes `replyToMode="first"` vs `"all"`; this PR still needs maintainer review rather than cleanup closure.

## Evidence
