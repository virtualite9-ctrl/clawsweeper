## Summary

Keep open. Current main still lacks Feishu runtime connection/activity reporting into the gateway health snapshot, and the submitted PR remains relevant rather than obsolete. It needs revision before merge because the current health contract uses lastTransportActivityAt for stale-socket detection, while this PR's provided diff only adds connected/mode/lastEventAt/lastError reporting and has no focused Feishu status tests.

## Best Possible Solution

Keep the PR open and ask for a rebase/revision that uses the existing gateway status sink helpers, threads a Feishu status sink from startAccount through monitorFeishuProvider, monitorSingleAccount, event handlers, and webhook/websocket transports, reports connected/mode/lastConnectedAt/lastEventAt/lastTransportActivityAt/lastError at validated lifecycle and activity points, preserves the webhook guard-ordering fix from the PR discussion, and adds focused Feishu tests for WebSocket/webhook status behavior. Maintainers should choose the canonical branch among the overlapping Feishu health PRs before final review.
