## Summary

Keep this PR open. Current main still lacks the requested Discord `allowBots=false` bot-drop diagnostic: the runtime only emits the old terse verbose message before dropping the message, and the current tests do not cover the proposed default-off logging path. The PR is stale against the current `extensions/discord` layout, but it remains a focused implementation candidate with no security-sensitive diff.

## Best Possible Solution

Keep this PR open as a useful implementation candidate. The maintainer path is to rebase or port the same narrow behavior onto `extensions/discord/src/monitor/message-handler.preflight.ts` and `extensions/discord/src/monitor/message-handler.preflight.test.ts`: add a `logDebug` diagnostic plus a clearer `logVerbose` hint on the existing `allowBotsMode === "off"` bot-drop branch, preserve current filtering behavior, and add a regression test for that exact log path.
