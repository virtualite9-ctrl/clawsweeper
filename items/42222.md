## Summary

Keep this PR open. The linked bot review was positive and the PR is a focused diagnostic improvement, but current `main` still has the old Discord `allowBots=false` drop path with only `logVerbose("discord: drop bot message (allowBots=false)")`; the requested durable debug diagnostic and actionable verbose hint are not implemented or tested on the current `extensions/discord` layout.

## Best Possible Solution

Keep this PR open as a still-useful implementation candidate, but require it to be rebased/ported to `extensions/discord/src/monitor/message-handler.preflight.ts` and the matching test file. The best fix is to add the debug diagnostic and clearer verbose hint on the existing `allowBots=false` early-return path, preserving the current default filtering behavior and adding a regression test for that exact log path.
