## Summary

Current main already supports ACPX MCP server definitions in gateway config, but via the ACPX plugin config path `plugins.entries.acpx.config.mcpServers` rather than a root-level `acpx` object. The requested capability is present and shipped.

## Evidence

- **Plugin config lives under plugins.entries:** The configuration reference documents plugin-specific settings under `plugins.entries.<pluginId>.config`, which explains why a root-level `acpx` key is rejected.
  - file: [docs/gateway/configuration-reference.md:90](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/docs/gateway/configuration-reference.md#L90)
- **ACPX schema accepts mcpServers:** The ACPX plugin config type and strict Zod schema include `mcpServers?: Record<string, McpServerConfig>`.
  - file: [extensions/acpx/src/config-schema.ts:26](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/extensions/acpx/src/config-schema.ts#L26)
- **Bundled plugin manifest publishes mcpServers:** The shipped ACPX plugin manifest exposes `mcpServers` in its JSON config schema and UI hints, so gateway config validation can accept it in ACPX plugin config.
  - file: [extensions/acpx/openclaw.plugin.json:53](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/extensions/acpx/openclaw.plugin.json#L53)
- **Docs show the supported gateway config shape:** Gateway docs explicitly show MCP server env vars configured under `plugins.entries.acpx.config.mcpServers`.
  - file: [docs/gateway/secrets.md:293](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/docs/gateway/secrets.md#L293)
- **Shipped in latest release:** The latest release commit already contains the same ACPX `mcpServers` schema and docs (`git show 00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5:extensions/acpx/src/config-schema.ts`, `git show 00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5:docs/gateway/secrets.md`).
  - command: `git show 00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5:extensions/acpx/src/config-schema.ts ; git show 00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5:docs/gateway/secrets.md`
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)

## Risks / Open Questions

- The exact root-level config shape from the issue body (`acpx: { ... }`) is still not valid; users need the documented plugin path `plugins.entries.acpx.config.mcpServers`.

## Close Comment

Closing as implemented on main after Codex review. OpenClaw already supports ACPX MCP server definitions in gateway config, but the supported path is `plugins.entries.acpx.config.mcpServers`, not a root-level `acpx` key. The ACPX schema includes `mcpServers` in `extensions/acpx/src/config-schema.ts:26` and `extensions/acpx/src/config-schema.ts:81`, the bundled manifest publishes the same schema in `extensions/acpx/openclaw.plugin.json:53`, and the docs show the gateway config shape in `docs/gateway/secrets.md:293`. This was already shipped in `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`).
