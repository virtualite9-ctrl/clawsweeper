## Summary

Close the PR as already implemented on `main` and shipped in `v2026.4.23`; the current UI resolves default-model labels through the catalog display lookup and already shows the routing provider (for example `openrouter`) instead of the underlying provider when appropriate.

## Evidence

- **Current default-label path uses catalog display lookup:** `resolveChatModelSelectState` canonicalizes the default model via `resolvePreferredServerChatModelValue` and then formats it through `formatCatalogChatModelDisplayFromLookup`, rather than overriding the displayed provider from raw session state. That is the behavior on current `main` and on tag `v2026.4.23`.
