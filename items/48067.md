## Summary

Current main resolves per-session rows against the session agent, but the TUI still falls back to global `agents.defaults.model` when the active session has no persisted entry yet. That means the reported status-bar mismatch remains plausible for a fresh/idle TUI session, so there is not strong enough evidence to close this issue.
