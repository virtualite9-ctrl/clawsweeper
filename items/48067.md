## Summary

Keep open. Current main resolves the per-agent model for existing session rows, but the TUI still falls back to agent-agnostic `sessions.defaults` when the active session has no persisted row. That fallback is still built from `agents.defaults.model`, not the active agent's resolved model, so the reported status-bar bug can still reproduce for a brand-new active agent session. The related PR #48110 proposed the missing `getSessionDefaults(..., agentId)` direction, but it is closed unmerged in the provided GitHub context.

## Best Possible Solution

Keep the issue open and implement the missing default-resolution path: when `sessions.list` is scoped by `agentId` or the TUI refreshes an active agent session with no row, return/apply defaults resolved through `resolveDefaultModelForAgent({ cfg, agentId })`. Add a regression test with `agents.defaults.model.primary = A`, `agents.list[].model.primary = B`, an empty/no-match session list, and assert the TUI footer/sessionInfo shows B while preserving the existing row-present override behavior.
