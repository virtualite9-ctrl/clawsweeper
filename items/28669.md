## Summary

Keep #28669 open. Current main still lacks the requested `audio/webm`, `video/webm`, and `video/x-matroska` MIME-to-extension mappings, and the missing core helper mapping still affects saved media IDs, outbound attachment filename inference, plugin SDK media helpers, and several channel/provider call sites that depend on `extensionForMime`.

## Best Possible Solution

Keep the PR open. The best maintainer path is to rebase or recreate the narrow patch on current main, add `audio/webm -> .webm`, `video/webm -> .webm`, and `video/x-matroska -> .mkv` in `src/media/mime.ts`, and add focused regression coverage in `src/media/mime.test.ts`; a saved-media or outbound attachment filename test would directly protect the reported user-visible behavior.
