## Summary

Keep #28669 open. Current main still lacks the requested `audio/webm`, `video/webm`, and `video/x-matroska` MIME-to-extension mappings, and multiple core/plugin-facing media paths still depend on `extensionForMime()`. The PR is a narrow core bug fix, not a ClawHub item or a duplicate of the provided related reports.

## Best Possible Solution

Keep the PR open. Rebase or recreate the narrow patch on current main, add the three MIME mappings in `src/media/mime.ts`, and add focused regression coverage in `src/media/mime.test.ts`; a saved-media or outbound filename test would also protect the reported user-visible behavior.
