## Summary

Keep #28669 open. Current main still lacks the requested `audio/webm`, `video/webm`, and `video/x-matroska` mappings in the shared MIME helper, while saved media, remote media filename inference, outbound attachment filename inference, and plugin-facing media helpers still depend on `extensionForMime()`. The PR remains a narrow, plausible bug fix and is not superseded by the provided related context.

## Best Possible Solution

Keep the PR open. Rebase or recreate the patch on current main, add `audio/webm` and `video/webm` mapping to `.webm` plus `video/x-matroska` mapping to `.mkv` in `src/media/mime.ts`, and add focused regression coverage in `src/media/mime.test.ts` plus a save/outbound filename case for MIME-only attachments. Maintainers should also choose the desired reverse `.webm` default MIME because `MIME_BY_EXT` is derived from `EXT_BY_MIME`.
