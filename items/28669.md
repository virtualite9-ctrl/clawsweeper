## Summary

Keep #28669 open. Current main still lacks the requested WebM and Matroska MIME mappings, while saved-media and outbound attachment filename paths still depend on `extensionForMime()`. The PR is a narrow bug fix and is not superseded by the provided related context.

## Best Possible Solution

Keep the PR open. Rebase or recreate the patch on current main, add the three MIME mappings in `src/media/mime.ts`, and add focused regression coverage in `src/media/mime.test.ts`; an outbound filename or saved-media case would also protect the reported user-visible behavior.
