## Summary

Keep #28669 open. Current main still lacks the requested `audio/webm`, `video/webm`, and `video/x-matroska` MIME mappings, and the affected saved-media/outbound filename paths still depend on `extensionForMime()`. The PR is a narrow core fix, not a ClawHub/plugin item or duplicate.

## Best Possible Solution

Keep the PR open. Rebase or recreate the patch on current main, add the three MIME mappings in `src/media/mime.ts`, and add focused regression coverage in `src/media/mime.test.ts`; a saved-media or outbound filename case would also protect the reported user-visible behavior.
