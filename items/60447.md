## Summary

Current `main` already handles `cron edit --tools`/partial `agentTurn` payload updates correctly; the reported schema failure matches older behavior, not the code now on `main`. I could not verify a released tag that includes the fix, so this is implemented on `main` but likely not yet in `v2026.4.23`.

## Evidence

- **CLI now emits an `agentTurn` patch for `--tools` edits:** `cron edit` treats `--tools`, `--message`, delivery flags, and related agent-job flags as an agent-turn patch and writes `payload.kind = "agentTurn"` plus `payload.toolsAllow`, which directly covers the reported `--message` + `--tools` edit path.
  - file: [src/cli/cron-cli/register.cron-edit.ts:192](https://github.com/openclaw/openclaw/blob/51c11cfd903033947f01f96a7169ebf92d194064/src/cli/cron-cli/register.cron-edit.ts#L192)
