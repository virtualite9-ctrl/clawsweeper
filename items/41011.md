## Summary

Current `main` already has a broader Telegram polling-stall hardening implementation, so PR #41011 is superseded. The code now uses a default 120s watchdog threshold with bounded config overrides, tracks `getUpdates` plus other in-flight API activity before declaring a stall, and has focused tests/docs covering the false-positive restart cases this PR targets.
