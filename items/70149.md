## Summary

Current main already fixes the reported WhatsApp QR-login proxy timeout by replacing the ambient `proxy-agent` path with an explicit `HttpsProxyAgent`, declaring that dependency in the WhatsApp plugin, respecting `NO_PROXY`, and covering the behavior with regression tests. The fix shipped in v2026.4.26.

## Best Possible Solution

Close as implemented. Keep the shipped direct `HttpsProxyAgent` WhatsApp WebSocket path and ask for a fresh issue with current-version verbose logs only if a proxy login timeout still reproduces on v2026.4.26 or newer.
