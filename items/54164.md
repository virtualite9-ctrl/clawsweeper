## Summary

Current `main` still appears to have the reported bug. The gateway `agent` handler reuses the stored `sessionId`, updates `updatedAt`, persists that entry, and then forwards the explicit `sessionId` into the agent command path. `resolveSession()` only rotates stale sessions when no explicit `sessionId` is supplied, so inter-agent / `sessions_send` runs still bypass daily freshness-based rollover.

## Evidence

- **Gateway handler pins existing session id before dispatch:** When a `sessionKey` is present, the gateway loads the stored entry, takes `entry?.sessionId ?? randomUUID()`, writes it back into `nextEntryPatch` with `updatedAt: now`, and persists it via `updateSessionStore(...)` before dispatching the run.
  - file: [src/gateway/server-methods/agent.ts:620](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/gateway/server-methods/agent.ts#L620)
  - command: `git show HEAD:src/gateway/server-methods/agent.ts | nl -ba | sed -n '620,730p'`
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Gateway forwards explicit sessionId into agent ingress:** After persisting the entry, the gateway passes both `sessionId: resolvedSessionId` and `sessionKey: resolvedSessionKey` into `dispatchAgentRunFromGateway(...)`, so downstream session resolution sees an explicit session id.
  - file: [src/gateway/server-methods/agent.ts:925](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/gateway/server-methods/agent.ts#L925)
  - command: `git show HEAD:src/gateway/server-methods/agent.ts | nl -ba | sed -n '900,945p'`
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **resolveSession only rolls stale sessions when sessionId is absent:** `resolveSession()` computes `fresh` with `evaluateSessionFreshness(...)`, but then chooses `opts.sessionId?.trim() || (fresh ? sessionEntry?.sessionId : undefined) || crypto.randomUUID()`. It also sets `isNewSession = !fresh && !opts.sessionId`, so any explicit session id suppresses rollover.
  - file: [src/agents/command/session.ts:237](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/agents/command/session.ts#L237)
