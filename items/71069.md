## Summary

Keep this issue open. Current main still clears stored session overrides that are outside the configured agent allowlist and emits the same generic reset message. The repo supports Gemma 4 in provider-specific forms, but the exact unprefixed `Gemma4-26b-a4-it-gguf` spelling is not documented or normalized, so the remaining work is actionable as either diagnostics/docs or a provider-owned alias after the reporter supplies the intended backend config.

## Best Possible Solution

Keep this issue open. Ask the reporter for the exact provider/backend and model configuration, then improve the stored-override reset diagnostic to include the rejected ref, assumed provider, and allowed alternatives. If `Gemma4-26b-a4-it-gguf` is an intentional supported spelling for a specific provider, add that alias or documentation in that provider surface rather than weakening the core allowlist policy.
