## Summary

Keep open. Current main supports canonical Gemma 4 model refs and custom local-provider catalog entries, but the exact reported override `Gemma4-26b-a4-it-gguf` is not shown as a supported/canonical ref, and the current stored-override path still emits the reported generic reset message when an override is outside the agent model allowlist. The issue is recent and asks for acceptance or a clearer rejection, so there is not enough evidence for a conservative auto-close.

## Best Possible Solution

Keep the issue open and ask the reporter to provide the relevant `agents.defaults.model`, `agents.defaults.models`, `models.providers`, and session override fields, plus whether the target is Google, OpenRouter, LM Studio/inferrs, or another local GGUF backend. Then either improve the reset message to name the rejected provider/model and allowed refs, document the required allowlist/custom-provider setup more directly, or add a canonical alias/normalizer only if `Gemma4-26b-a4-it-gguf` is intended to resolve to a supported provider model.
