## Summary

Keep this issue open. Current main still rejects and clears stored session model overrides when they are outside the configured allowlist, and the user-facing reset message still does not name the rejected model. The exact reported Gemma/GGUF identifier is not documented or recognized in-tree, while canonical Gemma 4 and local-provider model refs use different provider/model forms.

## Best Possible Solution

Keep this issue open. Ask the reporter for the exact model/provider configuration and whether the target backend is Google, Ollama, LM Studio, inferrs, or another OpenAI-compatible GGUF server. Then improve the reset diagnostic to include the rejected ref and allowed/provider-specific refs, document the required setup more directly, or add a canonical alias only if Gemma4-26b-a4-it-gguf is meant to map to a supported provider model.
