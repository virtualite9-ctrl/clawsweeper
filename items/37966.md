## Summary

Keep issue #37966 open. Current main still does not implement the reported LiteLLM OpenAI-compatible Anthropic cache-control path: the bundled LiteLLM provider and onboarding default use `openai-completions`, explicit `cacheRetention` is only honored for direct Anthropic/Vertex, Anthropic Bedrock, Google, or custom `anthropic-messages` providers, and the only generic OpenAI-compatible `cache_control` injection remains scoped to verified OpenRouter routes. The related fix PR #38221 was closed unmerged, and the reporter reconfirmed the problem on 2026-04-21.

## Best Possible Solution

Keep #37966 open until OpenClaw either implements a conservative LiteLLM Anthropic pass-through path or explicitly documents this setup as unsupported. A likely fix belongs in the LiteLLM provider wrapper or a shared proxy-cache wrapper: detect explicit `cacheRetention` for LiteLLM Anthropic model refs on `openai-completions`, inject Anthropic `cache_control` blocks into the OpenAI-compatible payload, and add regression coverage for `litellm/claude-opus-4-6` on the documented LiteLLM route.
