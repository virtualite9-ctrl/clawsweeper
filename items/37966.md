## Summary

Keep #37966 open. Current main still does not honor explicit `cacheRetention` for the bundled LiteLLM OpenAI-compatible Anthropic route: LiteLLM defaults to `api: "openai-completions"`, the Anthropic cache-retention resolver only admits non-Bedrock custom providers on `api: "anthropic-messages"`, and the only generic Anthropic `cache_control` payload wrapper remains gated to verified OpenRouter routes. The related PR #38221 was closed unmerged in the provided GitHub context, and the reporter reconfirmed the issue on 2026-04-21.

## Best Possible Solution

Keep #37966 open until OpenClaw either implements a conservative LiteLLM Anthropic pass-through cache path or explicitly makes this unsupported product behavior. A likely implementation belongs in the LiteLLM provider wrapper or a shared proxy-cache wrapper: detect explicit `cacheRetention` for LiteLLM Anthropic model refs on `openai-completions`, inject Anthropic `cache_control` blocks into the OpenAI-compatible payload, and add regression coverage for `litellm/claude-opus-4-6` on the documented LiteLLM route.
