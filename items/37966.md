## Summary

Keep #37966 open. Current main still ignores explicit `cacheRetention` for the bundled LiteLLM default route: LiteLLM is configured as `openai-completions`, cache-retention resolution only admits custom providers on `anthropic-messages`, and the existing OpenAI-compatible `cache_control` payload injection remains OpenRouter-gated. The related fix PR #38221 was closed unmerged, and the reporter reconfirmed the issue on 2026-04-21.

## Best Possible Solution

Keep #37966 open until OpenClaw either implements conservative LiteLLM Anthropic prompt-cache pass-through or makes an explicit maintainer product decision that this route is unsupported. A likely fix belongs in the bundled LiteLLM/provider wrapper path: when explicit `cacheRetention` is configured for an Anthropic-backed LiteLLM `openai-completions` model, inject Anthropic `cache_control` markers into the OpenAI-compatible payload, then add regression coverage for `litellm/claude-opus-4-6` and update the public LiteLLM/prompt-caching docs.
