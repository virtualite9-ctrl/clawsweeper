## Summary

Keep #37966 open. Current main still does not honor explicit `cacheRetention` for the bundled LiteLLM Anthropic route: LiteLLM defaults to `api: "openai-completions"`, the cache-retention resolver only admits custom non-Bedrock providers on `api: "anthropic-messages"`, and the generic `cache_control` marker wrapper remains OpenRouter-gated. The related PR #38221 was closed unmerged, and the reporter reconfirmed the problem on 2026-04-21.

## Best Possible Solution

Keep #37966 open until OpenClaw either implements a conservative LiteLLM Anthropic prompt-cache pass-through or explicitly decides and documents that this is unsupported. A likely fix belongs in the LiteLLM/provider proxy wrapper path or a shared proxy-cache wrapper: when explicit `cacheRetention` is configured for LiteLLM Anthropic model refs on `openai-completions`, inject Anthropic `cache_control` markers into the OpenAI-compatible payload, then add regression coverage for `litellm/claude-opus-4-6` and update the LiteLLM docs.
