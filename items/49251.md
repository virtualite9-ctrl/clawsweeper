## Summary

Current `main` does not implement this feature. The followup queue is explicitly in-process only, it is only used when a session already has an active run, and rate-limit exhaustion still resolves to an error payload (`API rate limit reached` / `Please try again`) rather than re-queueing the prompt for later automatic retry.

## Evidence
