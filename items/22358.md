## Summary

Keep #22358 open. Current main still has subagent spawn, routing, spawned, and ended hooks, but no post-subagent-completion/pre-announcement plugin hook that can inspect child output/history, observe delivery result, or modify/suppress the completion announcement. The issue discussion also explicitly says adjacent delivery fixes (#35080, #47433) improved symptoms without covering this requested extension point.

## Best Possible Solution

Keep #22358 open until OpenClaw adds a public typed plugin hook at the post-subagent-completion/pre-announcement point, or explicitly folds that exact scope into a canonical lifecycle/delivery design. The contract should expose child/session/run metadata, completion output or child-history access, timing/usage where available, delivery status/error visibility, and return semantics for announcement enrichment, suppression, or filtering, with docs and ordering regression tests.
