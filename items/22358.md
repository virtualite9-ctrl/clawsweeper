## Summary

Keep #22358 open. Current main has adjacent subagent lifecycle hooks and internal announce-delivery status plumbing, but it still does not expose the requested post-subagent-completion, pre-announcement plugin hook with child output inspection, delivery-result visibility, or announcement enrichment/suppression semantics. The issue discussion also includes recent maintainer/Codex review and follow-up comments explicitly saying the delivery fixes improved symptoms but did not implement this extension point.

## Best Possible Solution

Keep #22358 open until OpenClaw adds a public typed plugin hook at the post-subagent-completion/pre-announcement point, or explicitly folds this exact scope into a canonical lifecycle/delivery design. The contract should include child/session/run metadata, completion output, outcome/timing/usage where available, delivery path/status/error visibility, and explicit return semantics for announcement enrichment or suppression, with docs and ordering regression tests.
