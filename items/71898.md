## Summary

Keep PR #71898 open. Current main still has the reported behavior: `memory_search` correctly requests and filters session-sourced hits, but the final surfaced result shape still hard-codes every memory-manager hit as `corpus: "memory"`. The PR is a focused useful fix with a regression test, and the one bot readability comment was addressed in the PR context.

## Best Possible Solution

Keep the PR open for maintainer review/merge, or apply the same minimal change on main: type surfaced memory results with `MemorySource`, set `corpus: result.source`, and keep the focused session-corpus regression test in `extensions/memory-core/src/tools.test.ts`.
