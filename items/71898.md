## Summary

Keep PR #71898 open. Current main still has the reported bug: `memory_search` filters and retrieves session hits, but then hard-codes every surfaced memory-manager result as `corpus: "memory"`. The PR is a focused fix with a regression test and the only bot review concern was addressed in the PR discussion.

## Best Possible Solution

Keep the PR open for maintainer review and normal validation. The best code path is the PR's narrow fix: preserve `MemorySearchResult.source` when shaping `memory_search` results and keep the focused regression test for `corpus=sessions`.
