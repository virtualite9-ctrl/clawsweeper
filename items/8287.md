## Summary

Keep #8287 open. Current main still exposes nodes through declared command routing and one static `nodes` agent tool, not through connected-node advertised agent tools. The issue also has recent discussion explicitly keeping it open, and the related PRs in the provided context do not supersede this feature.

## Best Possible Solution

Keep #8287 open for maintainer-approved Gateway protocol and runtime work: add an additive node tool-advertisement schema, validate and store advertised tools on connected `NodeRegistry` sessions, synthesize node-backed agent tools and catalog/effective inventory entries with node provenance, route execution through existing `node.invoke` authorization, and document naming, reconnect, pairing, and `tools.allow`/`tools.deny` behavior.

## Risks / Open Questions

- Implementing this would expand the node-to-LLM tool surface, so schema validation, tool naming conflicts, reconnect idempotency, command authorization, node provenance, and tool policy semantics need explicit maintainer/security design.
- The current plugin plus `/tools/invoke` workaround remains useful interim plumbing, but it does not satisfy the zero-plugin node-advertised tool model requested here.
