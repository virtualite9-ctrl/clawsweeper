## Summary

Current `main` already addresses this streaming/tools regression in the gateway chat event path: buffered assistant text is flushed before tool/item start events and again before finalization, and those delta payloads are sent to both websocket/web UI and session subscribers/TUI paths.

## Close Comment

Closing from a Codex review because current `main` already implements this fix in the gateway chat streaming path. `src/gateway/server-chat.ts:762` flushes buffered assistant text as a `chat` delta, `src/gateway/server-chat.ts:930` does that flush before `tool` start events, and `src/gateway/server-chat.ts:817` flushes again before the final chat event so the last throttled chunk is not delayed until finalize. Those flushed deltas are sent to both websocket/web UI clients and session subscribers/TUI via `broadcast("chat", ...)` and `nodeSendToSession(...)` in `src/gateway/server-chat.ts:800`. The regression is also covered by tests in `src/gateway/server-chat.agent-events.test.ts:373` and `src/gateway/server-chat.agent-events.test.ts:617`. I could verify this on current `main` at `55389f4be29359e2dd57ebaf548ba4a70331a445`, but I could not determine a separate shipped release tag for the fix from the history available in this checkout.
