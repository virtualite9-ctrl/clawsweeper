## Summary

Keep open. Current main does not include the PR's file-backed pending-inbound store, startup replay, or durable active-turn persistence/recovery. The PR is still an active implementation candidate, but the provided review discussion identifies correctness and security concerns that need maintainer review before merge.

## Best Possible Solution

Keep this PR open as a concrete implementation candidate. Maintainers should either review and require fixes for the identified correctness/security issues, or explicitly supersede it with a canonical durable restart-resilience design if a different storage model is preferred.
