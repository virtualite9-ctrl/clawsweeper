## Summary

Keep open. Current main still lacks the file-backed pending-inbound store, startup replay, and durable active-turn recovery this PR proposes. The PR is also still a live, open implementation candidate with security-sensitive unresolved review discussion rather than something clearly implemented, superseded, or obsolete.

## Best Possible Solution

Keep this PR open as an implementation candidate. Maintainers should either review it and require fixes for the remaining correctness/security concerns, or explicitly supersede it with a canonical durable restart-resilience design such as #65650 after that design is accepted.
