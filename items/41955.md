## Summary

Keep this PR open. Current main still does not synthesize dropdown entries for configured agents that have no persisted session row, and the PR remains a focused implementation candidate for the linked webchat bug, though it needs to be refreshed against the current grouped session-control path and the global-session routing concern.

## Best Possible Solution

Keep this PR open or ask the contributor to refresh it against `ui/src/ui/chat/session-controls.ts`. The best fix is to merge configured `agents.list` entries into `resolveSessionOptionGroups()` with scope-aware key synthesis, dedupe against stored rows and the active key, preserve configured agent labels, and add regression tests for session-less configured agents, custom `mainKey`, and global-scope behavior. Maintainers may consolidate with related PR #48671, but the linked bug #41933 is not implemented on current main.
