## Summary

Keep this PR open. Current main still builds the webchat session dropdown from persisted session rows plus the active key; configured agents from `agents.list` are available but only used for labels after an option already exists. The submitted patch is stale against the current `resolveSessionOptionGroups()` path and has an unresolved global-scope routing concern, but it remains a focused implementation candidate for open bug #41933 rather than conservative close material.

## Best Possible Solution

Keep this PR open or ask the contributor to refresh it against `ui/src/ui/chat/session-controls.ts`. The best fix is to merge configured `agents.list` entries into `resolveSessionOptionGroups()`, synthesize only session keys the gateway can route correctly for the current scope/main-key contract, dedupe against stored sessions and the active key, keep labels stable, and add regression tests for session-less configured agents, custom main keys, and global-scope behavior. Maintainers may consolidate with #48671, but #41933 is still not implemented on current main.
