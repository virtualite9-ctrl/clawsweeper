## Summary

Keep open. Current main still builds the webchat session dropdown from persisted session rows plus the active session; `agents.list` is available but only used for labels, so #41933 remains unimplemented. The submitted PR is stale against a refactored UI path and has a review-noted global-scope concern, but it is still a plausible implementation candidate rather than conservative close material.

## Best Possible Solution

Keep the PR open or ask the contributor to refresh it against `ui/src/ui/chat/session-controls.ts`. The best implementation is to merge configured `agents.list` entries into `resolveSessionOptionGroups()`, synthesize valid per-agent main-session keys only where agent switching can work, dedupe against persisted rows, preserve stable labels, and add regression tests for session-less configured agents plus custom main-key and global-scope cases. Maintainers may also consolidate with #48671, but #41933 should stay tracked until a corrected PR lands.
