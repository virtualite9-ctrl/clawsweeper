## Summary

Keep #9394 open. Current main only has partial BlueBubbles `sendWithEffect` support for classic `effectId` sends, while the legacy `imsg` channel remains plain-text/media/reply oriented. Rich text attributes, text sizing, and the requested Shake/Nod/Explode/Ripple/Bloom/Jitter text-animation effects are still not implemented or documented. The existing issue discussion also already includes a Codex review comment by steipete keeping this open for the same gaps.

## Best Possible Solution

Keep #9394 open for maintainer product/API direction. The likely path is a typed BlueBubbles/iMessage rich-text and text-effect send surface wired through message actions and outbound delivery, with tests and docs; legacy `imsg` should either gain an upstream-supported capability or be explicitly documented as unsupported for those attributes.
