## Summary

Keep #9394 open. Current main still only has limited BlueBubbles `sendWithEffect` support for classic effect IDs, while legacy `imsg` remains plain text/media/reply oriented. Rich text attributes, text sizing, and the requested Shake/Nod/Explode/Ripple/Bloom/Jitter text-animation effects are not implemented or documented, and the existing maintainer-side Codex comment already kept this issue open for those same gaps.

## Best Possible Solution

Keep #9394 open for maintainer product direction. The likely path is a typed BlueBubbles/iMessage rich-text and text-effect send surface, wired through message actions and outbound delivery with tests and docs; legacy `imsg` should be documented as unsupported for those attributes if upstream `imsg` cannot carry them.
