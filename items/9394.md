## Summary

Keep #9394 open. Current main still sends legacy iMessage as plain text/media/reply payloads, keeps iMessage outside the shared rich Markdown IR path, and BlueBubbles only has narrow classic effectId support while stripping Markdown before send. The requested rich text spans, text sizing, and Shake/Nod/Explode/Ripple/Bloom/Jitter text animations are still not implemented.

## Best Possible Solution

Keep #9394 open for maintainer product/API direction. The likely path is a typed iMessage/BlueBubbles rich-text and text-effect send surface covering supported style spans, text size, and animation/effect enums, wired through outbound delivery and message actions with tests and docs. Legacy imsg should either gain upstream-supported capability handling or document these attributes as unsupported.
