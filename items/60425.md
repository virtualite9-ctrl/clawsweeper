## Summary

Current `main` already covers the restart/update history-loss paths: session keys are canonicalized on write, legacy/orphaned session-store keys are canonicalized on read and auto-migrated at gateway startup, and the embedded runner now backfills missing `sessionKey` before downstream hooks/context-engine code runs. That is enough evidence to treat this report as implemented on `main`, though it does not appear to be in the latest tagged release yet.
