## Summary

Keep open. Current `main` still resolves Slack file attachments synchronously on the inbound message path, and the Slack channel config/docs still do not expose a Socket Mode ping-timeout override or an explicit async-after-ack path. The linked timeline also points to an still-open upstream PR for Slack file access, which is not strong evidence of a fix on `main`.

## Evidence

Nothing in this path shows the attachment work being detached from the event handling flow.
