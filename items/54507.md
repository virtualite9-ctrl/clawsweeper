## Summary

Current main still unconditionally adds the Feishu runtime footer to structured/streaming card replies and does not expose a footer-specific hide/customize setting. The issue is recent, unprotected, concrete, and not superseded by the provided title-overlap related items.

## Best Possible Solution

Keep this open and implement a Feishu channel/account setting for the card runtime footer. A conservative path is to add a typed config field such as `channels.feishu.showRuntimeFooter` or a small `replyFooter` mode/template, update the plugin manifest/schema/docs, make `resolveCardNote` return `undefined` or a templated note when configured, and add tests for both streaming and non-streaming card replies with the footer hidden while preserving card rendering.
