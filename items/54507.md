## Summary

Current main still lacks a Feishu/Lark setting to hide or customize the Agent/Model/Provider runtime footer. The footer string is built unconditionally for card replies, passed into both streaming and non-streaming card delivery, rendered visibly as a grey note, and Feishu config/docs/manifest surfaces do not expose a footer-specific option. The existing ClawSweeper comment correctly kept this open; the current checkout at 0835f9409aac still shows the same gap.

## Best Possible Solution

Keep this issue open. Implement the setting in the Feishu plugin first, likely as a top-level and per-account `channels.feishu.showRuntimeFooter` boolean or a small `replyFooter` mode/template. Thread it through resolveCardNote, streaming start/close, and non-streaming card delivery; update Feishu zod schema, plugin manifest metadata, public Feishu docs, and reply-dispatcher tests. Consider a shared presentation contract only if maintainers want the same behavior across multiple channels.
