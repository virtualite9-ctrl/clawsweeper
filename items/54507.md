## Summary

Current main still lacks a Feishu/Lark option to hide or customize the runtime footer while preserving card replies. The footer is still built from Agent/Model/Provider and passed into both streaming and non-streaming card delivery, and the Feishu schema, manifest, docs, and tests do not expose a footer-specific setting. The prior ClawSweeper comment and related local reports do not identify a canonical duplicate or shipped implementation.

## Best Possible Solution

Keep this issue open and implement a conservative Feishu-owned setting. The likely path is a typed channel/account config such as channels.feishu.showRuntimeFooter or a small replyFooter mode/template, wired through resolveCardNote, streaming and non-streaming card delivery, the Feishu zod schema, plugin manifest metadata, docs/config surfaces, and reply dispatcher tests while preserving the current default footer.
