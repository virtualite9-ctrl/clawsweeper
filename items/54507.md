## Summary

Current main still lacks a Feishu/Lark setting to hide or customize the runtime footer. The footer is built unconditionally, passed into both streaming and non-streaming card replies, rendered as a card note, and there is no matching Feishu config/schema/manifest/docs/test surface for a footer-specific option.

## Best Possible Solution

Keep this issue open. Implement the setting in the Feishu plugin first, likely as `channels.feishu.showRuntimeFooter` or a small `replyFooter` mode/template available at top level and per account. Thread it through `resolveCardNote`, streaming start/close, and non-streaming card delivery; update the Feishu zod schema, plugin manifest metadata, public Feishu docs, and reply-dispatcher tests. Consider a cross-channel presentation contract only if maintainers want the same behavior beyond Feishu/Lark.

## Risks / Open Questions

- A global `agents.defaults.display.showRuntimeFooter` shape would cross channel ownership unless maintainers want a shared presentation contract; the narrow evidence points first to Feishu plugin-owned config.
