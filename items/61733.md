## Summary

Close as implemented on `main`: current exec-runtime code explicitly guards the `"Agent listener invoked outside active run"` race, and `CHANGELOG.md` says that exact gateway crash fix shipped in `v2026.4.23`.

## Evidence

- **Shipped release note matches exact crash signature:** `CHANGELOG.md` under `2026.4.23` says: `Agents/exec: prevent gateway crash ("Agent listener invoked outside active run") when a subagent exec tool produces stdout/stderr after the agent run has ended or been aborted. (#62821)`
  - file: [CHANGELOG.md:1010](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/CHANGELOG.md#L1010)
  - command: `nl -ba CHANGELOG.md | sed -n '990,1025p'`
  - sha: [a9797214338b](https://github.com/openclaw/openclaw/commit/a9797214338ba31c52c796adbb75afb16e0684a9)
- **Runtime now suppresses late exec updates after run end:** `runExecProcess` now uses an `updatesDisabled` guard specifically to prevent calling into a disposed agent run for this crash pattern, and `emitUpdate()` returns early when that flag is set.
  - file: [src/agents/bash-tools.exec-runtime.ts:564](https://github.com/openclaw/openclaw/blob/8b76392e3e7934b1cdd6d461148263fd56141ec8/src/agents/bash-tools.exec-runtime.ts#L564)
  - command: `nl -ba src/agents/bash-tools.exec-runtime.ts | sed -n '520,640p'`
  - sha: [8b76392e3e79](https://github.com/openclaw/openclaw/commit/8b76392e3e7934b1cdd6d461148263fd56141ec8)
- **Guard is flipped before exit/error completion:** Current main sets `updatesDisabled = true` before `markExited(...)` on normal completion and on failure, closing the stdout/stderr race window described in the report.
  - file: [src/agents/bash-tools.exec-runtime.ts:776](https://github.com/openclaw/openclaw/blob/8b76392e3e7934b1cdd6d461148263fd56141ec8/src/agents/bash-tools.exec-runtime.ts#L776)
  - command: `nl -ba src/agents/bash-tools.exec-runtime.ts | sed -n '748,832p'`
  - sha: [8b76392e3e79](https://github.com/openclaw/openclaw/commit/8b76392e3e7934b1cdd6d461148263fd56141ec8)
- **Abort path also disables updates immediately:** The exec tool's abort handler now calls `run.disableUpdates()` before deciding whether to kill or leave a backgrounded session running, covering the second path mentioned in the changelog (`ended or been aborted`).
  - file: [src/agents/bash-tools.exec.ts:1724](https://github.com/openclaw/openclaw/blob/8b76392e3e7934b1cdd6d461148263fd56141ec8/src/agents/bash-tools.exec.ts#L1724)
  - command: `nl -ba src/agents/bash-tools.exec.ts | sed -n '1690,1775p'`
  - sha: [8b76392e3e79](https://github.com/openclaw/openclaw/commit/8b76392e3e7934b1cdd6d461148263fd56141ec8)

That matches the code/changelog fix above.
