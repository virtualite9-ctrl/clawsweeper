## Summary

Keep open. Current `main` still implements cron/session cleanup only as retention-based pruning, and neither cron jobs nor hook mappings expose an `ephemeral`/dispose-on-completion option.

## Evidence

- **Cron cleanup is still timer-based:** `sweepCronRunSessions()` computes a retention cutoff from `cron.sessionRetention`, skips work for 5 minutes between sweeps, and only deletes cron run sessions when `updatedAt < cutoff`; there is no immediate-on-completion path.
  - file: [src/cron/session-reaper.ts:24](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/src/cron/session-reaper.ts#L24)
  - command: `nl -ba src/cron/session-reaper.ts | sed -n '1,170p'`
  - sha: [6bc0dc8fb671](https://github.com/openclaw/openclaw/commit/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440)
- **Cron config surface lacks an ephemeral flag:** `CronConfig` only exposes `sessionRetention` for cleanup policy, and `CronJob`/`CronJobCreate` have no `ephemeral` field or equivalent completion-disposal setting.
  - file: [src/config/types.cron.ts:30](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/src/config/types.cron.ts#L30)

  - file: [src/cron/types.ts:168](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/src/cron/types.ts#L168)
