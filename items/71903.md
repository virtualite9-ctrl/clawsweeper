## Summary

Keep open. Current main still contains the legacy same-PID/no-starttime orphan lock reclamation behavior that PR #71903 is trying to change, and the active test suite still asserts that behavior. The PR is a plausible core transcript-integrity fix tied to #71872/#64362, not ClawHub work and not implemented on main.

## Best Possible Solution

Keep PR #71903 open for maintainer review. Rebase it on current main, resolve the remaining review nits, replace the skipped legacy-lock test with focused regression coverage for the conservative same-PID/no-starttime behavior, and coordinate it with the broader open session-corruption work in #44510/#64362 before deciding whether to land or supersede either fix path.
