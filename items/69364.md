## Summary

Keep #69364 open. Current main has useful per-agent Control UI pieces, including a single-agent selector, workspace file editing, tool overrides, and skills allowlist editing, but it does not implement the requested cross-agent/team relationship view that composes delegate edges, dataflow, sequence/task state, and live session status in one debugging surface. Related #52803 overlaps on Control UI orchestration, but it is a broader mission-board request and does not clearly supersede this graph/config-debug proposal.

## Best Possible Solution

Keep the issue open. The best maintainer path is to scope a first Control UI slice as a read-only team/relationship graph backed by existing `agents.list[].subagents.allowAgents`, `tools.agentToAgent`, session metadata, and any documented project/task-flow state, then add editing/chat slices only where current gateway APIs are sufficient or after adding explicit APIs. If maintainers prefer one tracker, consolidate this into a canonical Control UI multi-agent graph/orchestration issue with this issue's edge-type and config-editing details preserved.
