## Summary

Keep open. Current main still forwards `openclaw agent --timeout` into the Gateway-owned `agent` request, and the Gateway still uses that forwarded value as the actual agent-run timeout. The PR is not obsolete or implemented on main, but the submitted diff is not merge-ready because the review discussion identified a missing `GatewayAcceptedOwnershipError` / `isGatewayAcceptedOwnershipError` export in `src/gateway/call.ts`.

## Best Possible Solution

Keep the PR open for maintainer/product judgment and request a revised patch. The best implementation path is to decide the intended `openclaw agent --timeout` contract for Gateway accepted runs, then either remove the forwarded `params.timeout` only for the accepted-run path with focused regression coverage, or preserve agent-timeout semantics and solve the accepted-run continuation issue elsewhere. If the accepted-ownership fallback guard remains, add and export the real `GatewayAcceptedOwnershipError` / `isGatewayAcceptedOwnershipError` implementation in `src/gateway/call.ts` with production-path tests instead of relying on a mocked module.
