## Summary

Keep #12394 open. Current main still lacks the requested channel-level WhatsApp heartbeat recipient routing: channel heartbeat config remains visibility-only, WhatsApp reuses that strict shape, and the WhatsApp recipient resolver ignores any `channels.whatsapp.heartbeat.target` or `targets` field. The existing agent-level `heartbeat.target`/`to` controls the whole heartbeat delivery target, not OK-only admin acknowledgments while preserving alert routing.

## Best Possible Solution

Keep #12394 open until maintainers implement or explicitly reject an additive WhatsApp/channel heartbeat recipient contract. A complete fix should extend schema/types/generated docs for `channels.whatsapp.heartbeat.target` or `targets`, validate normalized recipients against the selected WhatsApp account and allowlist/pairing policy, apply those recipients only to `HEARTBEAT_OK` delivery when `showOk` is enabled, preserve alert/content routing, and add coverage for omitted-target backward compatibility, single target, multiple targets, per-account overrides, and invalid targets.
