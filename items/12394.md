## Summary

Keep #12394 open. Current main still lacks the requested channel-level WhatsApp heartbeat recipient contract: channel heartbeat config remains visibility-only, and both the agent heartbeat delivery path and WhatsApp recipient resolver route a whole heartbeat run rather than OK-only `target`/`targets` under `channels.whatsapp.heartbeat`.

## Best Possible Solution

Keep #12394 open until maintainers implement or explicitly reject an additive WhatsApp channel heartbeat recipient contract. A complete fix should add typed/schema/generated-doc support for `channels.whatsapp.heartbeat.target` or `targets`, validate recipients against account and allowlist policy, apply those recipients only to `HEARTBEAT_OK` delivery without changing alert routing, and add regression coverage for single target, co-admin targets, omitted-target backward compatibility, per-account config, and invalid targets.
