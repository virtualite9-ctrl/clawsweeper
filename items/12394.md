## Summary

Keep #12394 open. Current main still lacks a channel-level WhatsApp heartbeat recipient contract for `target` or `targets`; the existing config is visibility-only, and the current runtime sends both `HEARTBEAT_OK` and alert delivery through the same resolved heartbeat delivery target rather than an OK-only admin recipient list.

## Best Possible Solution

Keep #12394 open until maintainers either implement or explicitly reject an additive WhatsApp channel heartbeat recipient contract. A complete fix should add typed/schema/generated-doc support for `channels.whatsapp.heartbeat.target` or `targets`, validate recipients against WhatsApp account and allowlist policy, apply those recipients only to `HEARTBEAT_OK` delivery, preserve alert/content routing, and add regression coverage for single target, multiple targets, omitted-target backward compatibility, per-account overrides, and invalid targets.
