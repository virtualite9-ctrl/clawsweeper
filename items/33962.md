## Summary

Keep #33962 open. Current main still runs session-memory slug generation through the default agent's resolved primary model, uses the normal embedded runner session/global lanes with `sessionKey: "temp:slug-generator"`, and only falls back to a timestamp after awaiting the LLM helper. The shipped generic `agents.defaults.timeoutSeconds` mitigation helps one timeout symptom, but the requested lightweight slug-model override, short fallback deadline, or debounce/decoupled lane behavior is still missing.

## Best Possible Solution

Keep #33962 open for a focused slug-generation product path. The likely implementation belongs around `src/hooks/llm-slug-generator.ts`, `src/hooks/bundled/session-memory/handler.ts`, and agent-default config/schema/help/docs: add a slug-specific lightweight model override, consider a short slug-specific fallback deadline or nonblocking heuristic fallback, and add regression tests for model selection, fallback timing, lane behavior, and the existing `llmSlug: false` opt-out.
