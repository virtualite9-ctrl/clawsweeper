## Summary

Keep #33962 open. Current main still uses the default agent's effective primary model for session-memory slug generation and runs that work through the embedded runner session/global lanes. Existing timeout behavior and timestamp fallback mitigate failures, but they do not provide the requested slug-specific lightweight model, early heuristic fallback, or debounce/decoupled lane behavior.

## Best Possible Solution

Keep #33962 open for a focused core fix. The best path is to add a slug-specific lightweight model override to the config contract, wire it through `src/hooks/llm-slug-generator.ts`, preserve the existing `llmSlug: false` opt-out, and add regression coverage for model selection plus fallback behavior. A short heuristic fallback or debounce can be considered alongside the model override, but the central fix is decoupling slug generation from the user-facing primary model path.
