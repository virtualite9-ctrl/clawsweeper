## Summary

Keep #33962 open. Current main still sends session-memory slug generation through the default agent model, on the normal embedded-runner session/global lanes, and only falls back to a timestamp after the LLM helper returns. The shipped generic `agents.defaults.timeoutSeconds` mitigation helps timeout length, but the requested lightweight slug model, short fallback deadline, or debounce/decoupled lane behavior is still absent. The related #71709 is a different slug-generator failure-mode bug, not a canonical replacement for this feature request.

## Best Possible Solution

Keep #33962 open for a focused slug-generation fix. The likely path is to add a slug-specific lightweight model override in the agent/gateway config contract, wire it through `src/hooks/llm-slug-generator.ts`, preserve the existing `llmSlug: false` opt-out, and add regression coverage for model selection, timeout/fallback behavior, and lane impact. A short heuristic fallback or debounce can be considered alongside the model override, but the central fix is decoupling slug generation from the user-facing primary model.
