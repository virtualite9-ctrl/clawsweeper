## Summary

Keep this issue open. The provided GitHub context shows a same-author open PR, #43953, with explicit closing syntax for this issue, so it should remain open until that PR is reviewed, landed, or closed. Current main also still appears to drop the requested Feishu card action fields: the parser and handler only preserve/use `action.value` plus `tag`, with no `form_value`, `input_value`, `name`, `option`, or `options` path.

## Best Possible Solution

Keep this issue open and review #43953 against current main. The fix should pass the requested Feishu fields through from `parseFeishuCardActionEventPayload` into `FeishuCardActionEvent`, build a backward-compatible structured synthetic message when those fields are present, preserve simple button `action.value` behavior, and add parser-to-handler regression coverage.
