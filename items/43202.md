## Summary

Keep this issue open. It has an active same-author implementation PR with closing syntax, and current main still does not pass through Feishu card-action fields outside action.value on the production callback path.

## Best Possible Solution

Keep this issue open and review the linked Feishu implementation PRs, especially same-author PR #43953. The durable fix should preserve the requested Feishu action fields in parseFeishuCardActionEventPayload, carry them through the typed FeishuCardActionEvent contract, serialize them in the legacy fallback only when present, preserve current simple button behavior, and add regression coverage from a raw card.action.trigger payload through handler dispatch.
