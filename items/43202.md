## Summary

Keep this issue open. Current main still drops Feishu card-action callback fields outside `action.value`, and the same author has an open, unmerged implementation PR with GitHub closing syntax for this issue. The right maintainer path is to review or close the linked PRs, not stale-close this issue.

## Best Possible Solution

Keep this issue open and review PR #43953 alongside related PR #42908. The fix should preserve extra Feishu `action` fields in `parseFeishuCardActionEventPayload`, extend the typed event and legacy fallback serialization for `form_value`, `input_value`, `name`, `option`, and `options`, keep simple button `value.text`/`value.command` behavior unchanged, and add regression coverage from a raw `card.action.trigger` payload through handler dispatch.
