## Summary

Keep open. Current main still matches the reported Discord `sessions_send` A2A announce account-routing bug: Discord announce targets are resolved from the session key without hydrating stored session metadata, so `accountId` remains undefined and downstream Discord account resolution can fall back to `default`. Related #51626 was closed as a duplicate of this issue, so this is the canonical remaining tracker.

## Best Possible Solution

Fix the Discord `sessions_send` A2A announce path so target resolution merges stored session delivery metadata when the parsed target lacks `accountId`, either by setting Discord `preferSessionLookupForAnnounceTarget: true` or by making `resolveAnnounceTarget()` hydrate missing route fields without regressing channels that depend on parsed keys. Add a regression test where a Discord target session has `deliveryContext.accountId: "work"` and the final gateway `send` includes `accountId: "work"`.
