## Summary

Keep open. Current main still has the reported Discord A2A announce account routing gap: parsed Discord announce targets bypass session lookup, Discord does not opt into session-backed announce target hydration, and the announce send forwards the missing accountId as-is. Related #51626 was closed in favor of this issue, so this remains the canonical tracker.

## Best Possible Solution

Keep this issue open and fix the Discord sessions_send A2A announce path. The narrowest implementation is to opt Discord into preferSessionLookupForAnnounceTarget, or adjust resolveAnnounceTarget to merge missing account/thread fields from stored session metadata without regressing parse-only routing. Add a regression test where a parsed Discord target session has deliveryContext.accountId or lastAccountId set and the final gateway send receives that accountId.
