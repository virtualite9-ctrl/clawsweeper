## Summary

Keep open. Current main has useful related recovery work for subagent run records, including restart restore, stale-run reconciliation, and retrying failed `sessions.delete` during delete-mode archive sweeps, but it does not implement the requested persistent `subagents.sessionRetention`/ACP retention policy for `sessions.json`. Persistent session-mode and keep-mode subagent/ACP sessions still intentionally lack `archiveAtMs`, and the session-mode sweeper branch only drops the subagent run record after cleanup, not the child session entry from `sessions.json`. The KeepMyClaw comment is a relevant external fallback/workaround, but it explicitly says the proposed retention is the right in-core fix, not that OpenClaw core is already covered.
