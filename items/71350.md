## Summary

This issue should stay open. Current main already has per-agent `thinkingDefault` and a supported per-agent `params` override surface for cases like `temperature` and `maxTokens`, but it still does not implement the requested per-agent per-model `agents.list[].models` map or any `thinkingStrategy` rules, failure escalation, or AGENTS.md-driven policy. The related adaptive-thinking PR #44126 is open and unmerged, so this remains a real product/API discussion rather than cleanup-close material.

## Best Possible Solution

Keep this issue open as the canonical product/API discussion. Preserve the shipped `agents.list[].thinkingDefault` and `agents.list[].params` behavior, then decide whether OpenClaw should add a true per-agent per-model `models` override map and a core `thinkingStrategy` policy for rule matching, failure escalation, and AGENTS.md-sourced rules, coordinated with #44126 and #32496.
