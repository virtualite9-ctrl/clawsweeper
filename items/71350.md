## Summary

Keep #71350 open. Current main now covers part of the request: `agents.list[].thinkingDefault` is schema-backed, and per-agent stream/model parameters are supported through `agents.list[].params` merged over global and per-model defaults. The remaining central feature, rules-based or failure-escalating adaptive thinking strategy, is not implemented on main, and the related #64792 was explicitly closed as superseded by this issue while PR #44126 remains only an open implementation discussion.

## Best Possible Solution

Keep #71350 open as the canonical product/design tracker. The best implementation path is to document that current per-agent parameter overrides use `agents.list[].params`, then decide the remaining adaptive strategy API deliberately: likely a core agent-loop policy or approved agent/tool API for rules-based and failure-escalating thinking changes, with clear session versus turn semantics and regression coverage for provider-managed `adaptive` so the two meanings are not conflated.
