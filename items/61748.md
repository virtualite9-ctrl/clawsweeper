## Summary

Current `main` already guards against the reported Telegram silent-drop cases: if a turn ends without a visible final delivery, or if non-silent delivery fails, Telegram dispatch now sends a fallback reply instead of returning silently, and current tests lock those paths in.

## Evidence

- **Telegram dispatch now sends a visible fallback on failed/non-delivered finals:** `bot-message-dispatch` snapshots delivery state and, when no non-silent final was delivered or a dispatch error occurred, sends a fallback reply (`"No response generated. Please try again."` or an error fallback) instead of exiting silently.
