## Summary

Current main already bounds OpenClaw compaction checkpoint artifacts and provides session-directory disk-budget enforcement, addressing the reported unbounded checkpoint/session storage problem for OpenClaw core. The current Pi session dependency persists append-only JSONL session files rather than a core git-backed checkpoint store, and OpenClaw’s own compaction checkpoint snapshots are capped, cleaned up, documented, and covered by tests.

## Best Possible Solution

Close this as implemented on current main. Keep the shipped bounded compaction-checkpoint cleanup plus session disk-budget controls; if a separate external Hermes/git-checkpoint plugin is producing git pack growth, track that in the owning plugin rather than reopening this core issue.
