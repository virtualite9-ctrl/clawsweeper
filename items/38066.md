## Summary

Keep #38066 open. Current main still does not expose `channels.telegram.streamThrottleMs` or `minInitialChars` through the Telegram config type, strict schema, generated metadata, docs, or the dispatcher path. The draft stream accepts internal `throttleMs`/`minInitialChars`, but callers still get the 1000ms default unless a caller passes throttle explicitly. The provided GitHub context shows PR #38255 is the active implementation path and is still open/unmerged, so the request is not implemented on main.

## Best Possible Solution

Keep #38066 open until PR #38255 or an equivalent maintainer-approved change lands. The fix should add a documented Telegram preview cadence config surface for top-level and per-account settings, validate it in the strict Zod schema, include generated config metadata/docs, merge account overrides through the existing Telegram config path, pass the resolved values into `createTelegramDraftStream`, and cover defaults, validation, fallback behavior, and account overrides with focused tests.
