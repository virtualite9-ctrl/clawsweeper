## Summary

Keep #38066 open. Current main still does not expose `channels.telegram.streamThrottleMs` or `minInitialChars` in Telegram config/schema/docs, and the dispatcher still passes only a hardcoded `minInitialChars: 30` without any configurable `throttleMs`. The related PR #38255 is the active open implementation path, but it is not merged, so the issue is not implemented on main.

## Best Possible Solution

Keep #38066 open until PR #38255 or an equivalent maintainer-approved change lands. The fix should add Telegram preview cadence config to the exported config type, strict schema, generated metadata, and public docs; resolve top-level/per-account values through the existing Telegram account merge path; pass them into `createTelegramDraftStream`; and cover default, validation/clamp, fallback, and account override behavior with focused tests.
