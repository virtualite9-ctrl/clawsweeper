## Summary

Keep #25592 open. Current main has substantial commentary/final_answer hardening, but it still does not implement the broader requested channel-safe policy for ordinary unphased assistant text before or between tool calls. The issue discussion also has recent reproduction reports and an Apr 26 maintainer review explicitly keeping this canonical bug open.

## Best Possible Solution

Keep #25592 open as the canonical product bug until OpenClaw implements or explicitly rejects a channel-safe policy such as final-only delivery for channel sessions, configurable enforceFinalTag for customer-facing channels, or an opt-in replyMode/tool-only explicit-send mode. Add regression coverage for ordinary unphased assistant text before/between tool calls in both block-reply delivery and final payload assembly.
