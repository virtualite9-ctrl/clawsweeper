## Summary

Keep #25592 open. Current main has strong commentary/final_answer hardening, but it still does not implement the broader requested policy that unintentional bare assistant text between tool calls must never reach messaging channels. Current tests still preserve pre-tool block delivery for unphased assistant text, and no replyMode/tool-only explicit-send config exists in agent defaults.

## Best Possible Solution

Keep #25592 as the canonical product bug until OpenClaw either implements a final-only/explicit-send channel policy, adds an opt-in replyMode/tool-only or enforceFinalTag-style safety setting for channel sessions, or narrows the remaining behavior into a maintainer-approved canonical follow-up. The existing commentary-phase hardening should stay, but it is not enough to close the broader issue.
