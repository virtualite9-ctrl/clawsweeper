## Summary

Keep #69931 open. Current main supports WhatsApp outbound/ack reactions, but I could not find a WhatsApp inbound reaction notification/read surface or a plugin hook/API that exposes reaction events tied to historical message IDs. This is not a good ClawHub close because VISION.md explicitly says plugin API extensions are welcome when a useful feature cannot be built as a plugin yet, and the current WhatsApp/plugin surfaces appear to lack the underlying event data the reporter is asking for.

## Best Possible Solution

Keep the issue open and route it as a WhatsApp/channel plugin API gap: add a Baileys reaction-event ingestion path, expose a documented structured reaction notification/read surface with target message ID, reacting actor, emoji, account/chat/session correlation, and update WhatsApp docs/config/tests. If this becomes optional automation after the event data exists, the automator itself can live on ClawHub.
