## Summary

Keep open. Current `main` still lacks the requested orphan-session pruning feature in both the Dashboard Sessions UI and the gateway session API. The Sessions screen supports refresh, filtering, manual selection, and bulk delete, but there is no prune action, configurable age input, or orphan eligibility preview. The backend protocol and handlers likewise expose `sessions.list` and per-key `sessions.delete`, but no prune/orphan-detection endpoint or session row fields that could represent orphaned/deleted-channel status.
