## Summary

Current main still lacks the specific Anthropic/Bedrock unphased thinking-plus-text fallback proposed by #70372. Main suppresses tagged commentary/final-answer phases, but Anthropic streaming still emits native thinking/text events without phase metadata, and unphased text still flows through the raw block-reply path. The PR also needs a narrower provider-scoped implementation because the supplied review context flags a provider misclassification risk.

## Best Possible Solution

Keep the PR open, but require a provider-scoped fix before landing: either tag Anthropic/Bedrock text blocks with explicit phase metadata in the transport layer, or derive a fallback only for known unphased Anthropic/Bedrock streams after native thinking events. Add a regression that emits thinking_start/thinking_delta/thinking_end followed by unphased text_delta/text_end, proves user-visible block replies do not leak narration, and proves OpenAI phase-pending text is not forced to final_answer.
