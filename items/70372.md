## Summary

Current `main` still appears vulnerable to the reported Anthropic/Bedrock leak, and this PR's synthetic-phase approach has an unresolved cross-provider regression risk, so it should stay open for maintainer review rather than be auto-closed.

## Evidence

- **Current delivery gating on main:** `handleMessageUpdate` only enables phase-aware delivery when `resolveAssistantMessagePhase(partialAssistant)` returns a phase. Thinking events are handled separately and do not set any state that would make later unphased text use `extractAssistantVisibleText`, so unphased streams still append raw chunks and fall back to `stripBlockTags(deltaBuffer)`.
  - file: [src/agents/pi-embedded-subscribe.handlers.messages.ts:399](https://github.com/openclaw/openclaw/blob/2cd2732ab6695a4b3220ce41166b74aca7097c5f/src/agents/pi-embedded-subscribe.handlers.messages.ts#L399)
  - command: `nl -ba src/agents/pi-embedded-subscribe.handlers.messages.ts | sed -n '396,522p'`
  - sha: [2cd2732ab669](https://github.com/openclaw/openclaw/commit/2cd2732ab6695a4b3220ce41166b74aca7097c5f)
- **Phase resolver only trusts explicit metadata:** `resolveAssistantMessagePhase` returns a phase only from `message.phase` or `textSignature.phase`; it does not infer phase from thinking blocks or prior stream events.
  - file: [src/shared/chat-message-content.ts:64](https://github.com/openclaw/openclaw/blob/2cd2732ab6695a4b3220ce41166b74aca7097c5f/src/shared/chat-message-content.ts#L64)
  - command: `nl -ba src/shared/chat-message-content.ts | sed -n '58,175p'`
  - sha: [2cd2732ab669](https://github.com/openclaw/openclaw/commit/2cd2732ab6695a4b3220ce41166b74aca7097c5f)
- **Anthropic stream matches the reported root cause:** `anthropic-transport-stream` emits `thinking_start`/`thinking_delta` and later `text_start`/`text_delta`/`text_end`, but the text blocks are created without `phase` or `textSignature`. That means current `main` has no explicit phase data to trigger the suppression path for Anthropic-style streams.
  - file: [src/agents/anthropic-transport-stream.ts:808](https://github.com/openclaw/openclaw/blob/2cd2732ab6695a4b3220ce41166b74aca7097c5f/src/agents/anthropic-transport-stream.ts#L808)

- **OpenAI regression risk in the proposed fallback:** `openai-transport-stream` also emits `thinking_start` before text deltas, and only stamps `item.phase` onto the text block at `response.output_item.done`/`text_end`. A generic `hadThinkingInMessage => final_answer` fallback during `text_delta` would therefore affect OpenAI too, which aligns with the PR review's unresolved P1 concern.
  - file: [src/agents/openai-transport-stream.ts:422](https://github.com/openclaw/openclaw/blob/2cd2732ab6695a4b3220ce41166b74aca7097c5f/src/agents/openai-transport-stream.ts#L422)

- **Current tests do not cover the unphased thinking case:** Existing commentary-suppression tests cover explicit `messagePhase` and `textSignature` metadata, not Anthropic/Bedrock-style unphased messages that interleave thinking with plain assistant text. That leaves the reported case unproven on `main` and the proposed fix insufficiently de-risked.

## Risks / Open Questions

- The underlying leak still looks plausible on current `main` for Anthropic/Bedrock-style unphased thinking streams.
- The PR as proposed can misclassify OpenAI text deltas after thinking as `final_answer` before provider metadata arrives.
