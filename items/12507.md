## Summary

Keep #12507 open. Current main still has no native trusted integrity/signature contract for SKILL.md skills: ClawHub skill metadata lacks digest/signature fields, native skill install records only version/time, the observed download digest is not compared with trusted metadata, runtime skill loading reads SKILL.md without verification state, and adjacent ClawHub plugin archive verification deliberately does not cover skill-family installs. The existing steipete Codex review comment also explicitly keeps this as canonical native skill integrity work.

## Best Possible Solution

Keep #12507 open as the canonical native skill integrity/signature work. A complete solution should add trusted digest/signature/publisher metadata to the ClawHub skill API, verify native skill archives before extraction, persist expected integrity/signature state in .clawhub origin and lock metadata, reverify SKILL.md or full skill contents during skill snapshot/load, expose verification state in SkillEntry/status/UI/CLI surfaces, and document/test the policy for unsigned or unverified skills and capability restrictions.
