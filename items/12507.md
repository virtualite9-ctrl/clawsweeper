## Summary

Current main still does not implement SKILL.md signature or trusted hash verification for native skills. The native ClawHub skill path downloads and extracts archives, but stores only registry/slug/version/time metadata; the skill detail API type lacks digest/signature/publisher-key fields; runtime skill loading reads local SKILL.md files into SkillEntry/SkillSnapshot without verification state; and the ClawHub plugin archive verifier explicitly excludes skill-family packages. The prior issue discussion also already identified #12507 as the canonical native skill integrity issue, and current main still matches that assessment.

## Best Possible Solution

Keep #12507 open as the canonical native skill integrity work. A complete fix should add trusted skill-version digest/signature/publisher metadata to the ClawHub skill API, verify native skill archives before extraction, persist expected integrity/signature state in .clawhub origin/lock metadata, reverify SKILL.md or full skill contents during skill snapshot/load, expose verification state on SkillEntry/status surfaces, and define maintainer-approved policy/docs/tests for unsigned or unverified skills and capability restrictions.
