## Summary

Keep #12507 open. Current main still does not implement SKILL.md signature/hash verification, trusted skill-version integrity metadata, runtime re-verification, or unsigned-skill capability gating. The existing ClawHub archive verifier applies to plugin packages and explicitly routes skill-family packages to the separate native skills installer, which still installs skills without checking a trusted archive/SKILL.md digest.

## Best Possible Solution

Keep #12507 open as the canonical core/ClawHub skill integrity issue. A complete fix should add trusted skill-version hash/signature/publisher metadata to the ClawHub skill API, verify native skill downloads before extraction/install, persist expected integrity/signature state in .clawhub origin/lock metadata, reverify SKILL.md or full skill contents during skill snapshot/load, expose verification state on SkillEntry/status surfaces, and add maintainer-approved policy/docs/tests for unsigned or unverified skills and capability restrictions.
