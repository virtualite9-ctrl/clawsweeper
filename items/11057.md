## Summary

Keep #11057 open. Current `main` still does not provide native 1M Opus 4.6 support for Claude Max/browser-session auth. API-key/custom-provider 1M remains possible through `params.context1m`, but OAuth/setup-token style credentials deliberately strip the Anthropic 1M beta header, and the docs/tests preserve that limitation. The issue discussion’s custom-provider workaround is useful, but it does not resolve the built-in Anthropic provider request.

## Best Possible Solution

Keep this issue open until OpenClaw either implements native Claude Max/browser-session 1M support for Opus 4.6 in the built-in Anthropic provider, documents a maintainer-approved API-key/custom-provider-only decision, or links a narrower canonical follow-up. The likely product path spans Anthropic stream beta handling, OAuth/setup-token behavior, context-window reporting/model listing, provider docs, and regression tests for API-key versus Claude Max credentials.
