## Summary

Keep #11057 open. Current main still does not provide native 1M Claude Max/browser-session support for `anthropic/claude-opus-4-6`: 1M remains config-gated, `sk-ant-oat` OAuth/setup-token auth strips the 1M beta, context accounting preserves the 200k path unless `params.context1m` is set, and the docs document that limitation. The custom-provider/proxy workaround is useful, but it does not resolve the built-in Anthropic provider request; related issues/PRs are still open and not a merged canonical replacement.

## Best Possible Solution

Keep this issue open until OpenClaw either implements native Claude Max/browser-session 1M support for Opus/Sonnet 4.6 in the built-in Anthropic provider, documents a maintainer-approved decision that this remains API-key/custom-provider only, or explicitly designates a narrower canonical issue such as the OAuth/GA long-context follow-up. The likely product/code path spans `extensions/anthropic/stream-wrappers.ts`, `src/agents/context.ts`, model listing/status context reporting, Anthropic provider docs, and OAuth/API-key regression tests.
