## Summary

Keep #11057 open. Current main still does not provide native 1M Claude Max/browser-session support for `anthropic/claude-opus-4-6`: 1M is only enabled through `params.context1m: true`, OAuth/subscription-style `sk-ant-oat` tokens still have the 1M beta stripped, and docs/tests explicitly preserve that fallback. The existing custom-provider/proxy workaround is useful, but it does not resolve the built-in Anthropic provider request.

## Best Possible Solution

Keep this issue open until OpenClaw either implements native Claude Max/browser-session 1M support for Opus 4.6 in the built-in Anthropic provider, documents a maintainer-approved decision that 1M remains API-key/custom-provider only, or supersedes this with a narrower canonical Anthropic OAuth long-context issue. Likely touched surfaces are `extensions/anthropic/stream-wrappers.ts`, `src/agents/context.ts`, model listing/status context reporting, Anthropic provider docs, and OAuth/API-key regression tests.
