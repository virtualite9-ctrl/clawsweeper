## Summary

Keep open. Current main still has fallback-only subagent timeout behavior: an explicit `sessions_spawn.runTimeoutSeconds` can undercut `agents.defaults.subagents.runTimeoutSeconds`, and there is still no accepted `minRunTimeoutSeconds`, lock option, config schema entry, docs entry, or regression test for a timeout floor. The related fix PRs #37973 and #38048 are closed unmerged, and related issue #60211 covered a different hard-timeout/fallback behavior.

## Best Possible Solution

Keep #37902 open until OpenClaw implements a maintainer-approved subagent timeout constraint. A narrow fix should add an explicit config contract such as `agents.defaults.subagents.minRunTimeoutSeconds` or an authoritative lock option, update the TypeScript config type, strict zod schema, generated schema/docs, and clamp model-supplied `sessions_spawn.runTimeoutSeconds` before subagent run registration. Regression tests should cover omitted timeout fallback, explicit values above the floor, explicit values below the floor, zero/no-timeout behavior, unset compatibility, and schema acceptance/rejection. The closed PRs #37973 and #38048 may be useful prior art, but they are not merged and do not currently close the issue.
