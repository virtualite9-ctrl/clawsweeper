## Summary

Keep open. Current main still implements fallback-only subagent timeout behavior: an explicit model/tool-supplied `sessions_spawn.runTimeoutSeconds` wins over `agents.defaults.subagents.runTimeoutSeconds`, and there is no accepted `minRunTimeoutSeconds`, lock option, schema entry, docs entry, or regression test for a timeout floor. The related fix PRs #37973 and #38048 are closed unmerged; related issue #60211 covered a different hard-timeout/fallback behavior.

## Best Possible Solution

Keep #37902 open until OpenClaw implements a maintainer-approved subagent timeout constraint. A narrow fix should add an explicit config contract such as `agents.defaults.subagents.minRunTimeoutSeconds` or an authoritative lock option, update the TypeScript config type, strict zod schema, generated schema/docs, and clamp model-supplied `sessions_spawn.runTimeoutSeconds` before subagent run registration. Regression tests should cover omitted timeout fallback, explicit values above the floor, explicit values below the floor, zero/no-timeout compatibility, unset compatibility, and schema acceptance/rejection. The closed PRs #37973 and #38048 are useful prior art but do not close the issue on current main.
