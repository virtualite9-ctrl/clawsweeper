## Summary

Keep open. Current `main` does not show a landed fix for the reported Discord voice auto-join restart/resume problem: `DiscordVoiceReadyListener` still only hooks the Carbon `ReadyListener` path, there is no explicit `RESUMED` handling in the Discord voice manager, and the voice manager still calls `connection.destroy()` without a guard in multiple paths. The exact log line from the report (`[VoicePlugin patch] Destroying stale voice connection...`) is not present on `main`, so that part may have come from an unmerged patch or dependency state, but the underlying resume/rejoin gap remains plausible and is not clearly resolved on current `main` or the latest release tag inspected.
