## Summary

Keep open. Current main still implements Discord voice autoJoin only through DiscordVoiceReadyListener, the provider registers only that READY listener, and the voice runtime exports no RESUMED listener. The teardown paths still call VoiceConnection.destroy() directly without checking Destroyed state or cleaning a stale @discordjs/voice registry entry before joinVoiceChannel. PR #40676 is still open, same-author, and explicitly says it fixes this issue, so the issue should remain open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open as the canonical Discord voice reliability bug. Review, revise, and land PR #40676 or an equivalent Discord plugin fix that registers autoJoin on both READY and RESUMED, makes stale VoiceConnection cleanup idempotent when @discordjs/voice reports Destroyed, and adds regression coverage for RESUMED autoJoin, already-destroyed stale cleanup, and same-guild multi-account behavior.
