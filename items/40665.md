## Summary

Keep open. Current main still registers Discord voice autoJoin only on READY, not RESUMED, and the voice join/teardown path still destroys @discordjs/voice connections without an idempotent destroyed-state guard or stale global connection cleanup. The linked PR #40676 uses closing syntax for this bug but is still open and unmerged, so this issue should remain open until that PR or an equivalent maintainer-reviewed fix lands.

## Best Possible Solution

Keep this issue open as the canonical Discord voice reliability bug. The best path is a focused Discord plugin fix that registers autoJoin for RESUMED as well as READY, makes voice connection teardown idempotent when @discordjs/voice reports Destroyed, and scopes stale connection cleanup to the owning account/manager. Review, revise, and land #40676 or an equivalent PR with regression coverage for RESUMED autoJoin, already-destroyed stale cleanup, and multi-account same-guild safety.
