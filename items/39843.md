## Summary

Keep open. Current main still creates a real Google Chat typing message before dispatch, exact `NO_REPLY` final payloads can skip delivery entirely, and the Google Chat monitor does not clean up the remaining `typingMessageName` when dispatch returns without invoking delivery. The related fix PR #39891 is still the concrete unmerged path in the provided GitHub context.

## Best Possible Solution

Keep #39843 open until PR #39891 or an equivalent Google Chat monitor fix lands. The fix should clean up any remaining `typingMessageName` after `dispatchReplyWithBufferedBlockDispatcher` completes without delivery, or wire the existing typing cleanup hook, and add a focused regression test for the Google Chat emoji reaction plus exact `NO_REPLY` path.
