## Summary

Keep #72148 open. Current main still exposes the reported user-facing exec approval follow-up prefix when session resume fails, and the tests still assert that old behavior. The PR remains a focused, relevant fix and is not superseded by the related #72268 parser PR.

## Best Possible Solution

Keep #72148 open for maintainer review. The best path is to land or revise the focused fix that removes the internal resume-failure prefix from user-facing direct exec approval fallback messages, logs the resume failure internally with robust redaction/sanitization, and tracks retry/backoff separately unless maintainers want that in this PR. Treat #72268 as a complementary parser fix, not a superseding item.
