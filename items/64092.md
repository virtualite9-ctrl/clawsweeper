## Summary

Current main already fixes the reported misclassification: 403 HTML provider responses are classified as `auth_html_403` and rendered with an explicit HTML-403 auth message instead of falling through to DNS or generic rate-limit messaging. The same behavior is present in the shipped `v2026.4.23` tag, with regression tests covering 403 HTML classification and user-facing formatting.

## Evidence
