## Summary

The remaining symptom is not an OpenClaw core CLI registration bug. Current OpenClaw separates root CLI plugin commands from chat/native slash commands, and lossless-claw currently exposes `/lcm` and `/lossless` through the plugin chat/native command surface, not as `openclaw lcm`, `openclaw lossless`, or `openclaw /lcm` shell commands.

## Best Possible Solution

Close this OpenClaw issue as outside the core repository: keep OpenClaw's current separation between root CLI plugin registration and Gateway chat/native command registration. Any future `openclaw lcm` or `openclaw lossless` root command should be added by lossless-claw itself through the documented `api.registerCli` descriptor surface.
