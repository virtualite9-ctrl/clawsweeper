## Summary

Keep #71696 open. Current main still does not implement Telegram chat-type-based reply mode: Telegram config schema/types/docs expose only the global `replyToMode`, Telegram message/native-command delivery still carries one resolved mode, delivery does not pass `currentMessageId` into outbound reply-tag planning, and plugin command context lacks `messageId`/`replyToMessageId`. The PR is recent, focused, and the provided context shows it supersedes the earlier closed #70579 while also addressing review feedback, so conservative cleanup should not close it.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to rebase it onto current main, preserve the runtime `replyToMode` fallback while adding Telegram `replyToModeByChatType` to schema/types/generated metadata/docs, resolve reply mode per direct/group for message and native-command delivery, pass `currentMessageId` through outbound planning/delivery, expose Telegram message ids to plugin command handlers, and run the changed Telegram/config/plugin/outbound checks.
