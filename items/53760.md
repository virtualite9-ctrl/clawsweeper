## Summary

Keep open. Current `main` does not provide a verified DM-level `toolsBySender` solution across the requested DM channels. Signal still exposes `toolsBySender` only for group overrides, and Telegram's `direct.*.toolsBySender` is schema-level scaffolding that is not consulted by the generic tool-policy resolution path for direct sessions.

## Evidence

- **Signal only defines toolsBySender on groups:** `SignalGroupConfig` has `toolsBySender`, but `SignalAccountConfig` only has top-level DM access fields plus `dms` for per-DM overrides; there is no DM tool-policy field for Signal.

- Closing this would miss the still-unimplemented Signal part of the request and the lack of clear shipped proof for Telegram DM enforcement.
