## Summary

Keep open. Current `main` still has the original locally-caught `streamError` flow, so this is not implemented on `main`, and the PR has an unresolved correctness concern: it moves failure bookkeeping after `finally`, which can skip terminal/error-state persistence if cleanup throws.

## Evidence

- **Current main still throws the captured turn error inside the local try path:** `src/acp/control-plane/manager.core.ts:783-855` still captures `streamError` from runtime events and then throws it from inside the same local try block at `:826-828` and again at `:853-855`. This PR's stated ACP cleanup is not present on current `main`.
  - file: [src/acp/control-plane/manager.core.ts:783](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/src/acp/control-plane/manager.core.ts#L783)
  - command: `nl -ba src/acp/control-plane/manager.core.ts | sed -n '720,960p'`
  - sha: [ec3dbd22a4de](https://github.com/openclaw/openclaw/commit/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46)
- **Current main persists failure state before post-turn cleanup:** On current `main`, the catch path records turn completion, marks background-task failure, and writes session state `error` before entering the `finally` cleanup block. See `src/acp/control-plane/manager.core.ts:879-920`, followed by cleanup/reconcile in `:921-944`. That ordering is what the PR changes.
  - file: [src/acp/control-plane/manager.core.ts:879](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/src/acp/control-plane/manager.core.ts#L879)

The PR adds no regression test for the new cleanup-ordering hazard called out in review.
  - file: [src/acp/control-plane/manager.test.ts:1922](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/src/acp/control-plane/manager.test.ts#L1922)
