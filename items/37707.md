## Summary

Keep open. Current `main` does not implement this PR's startup migration warning: Feishu still defaults DM policy to `"pairing"` in both schema and runtime fallback, and `startAccount` only logs startup mode before launching the monitor. The area also still needs maintainer attention because the proposed recovery message in the PR is incomplete under current `main` validation rules (`dmPolicy: "open"` also requires `allowFrom` to include `"*"`).
