## Summary

Keep #12198 open. Current main still has adjacent safety controls, but not the requested shared prompt-Markdown validation contract. Workspace bootstrap files, extra bootstrap files, discovered SKILL.md files, and native ClawHub skill installs still reach prompt or install paths without a common Markdown ruleset such as HTML-comment rejection. The prior issue discussion also includes an explicit 2026-04-26 maintainer/Codex review comment keeping this item open for the same gap.

## Best Possible Solution

Keep the issue open until OpenClaw has a shared prompt-Markdown validation contract with tests and docs, wired into workspace bootstrap ingestion, extra bootstrap hooks, SKILL.md discovery/on-demand loading, native ClawHub skill install or publish policy, and any plugin/API surface needed for third-party enforcement. The existing code-oriented skill scanner and external-content wrappers can inform the implementation, but they do not replace a prompt-bearing Markdown validator.
