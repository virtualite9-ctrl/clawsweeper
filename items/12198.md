## Summary

Keep #12198 open. Current main still lacks a shared Markdown validation contract for prompt-bearing Markdown. Existing controls cover path containment, size limits, code install scanning, Skill Workshop generated content, and web HTML cleanup, but workspace bootstrap files, extra bootstrap files, SKILL.md discovery/read paths, and native ClawHub skill installs still accept or expose Markdown without the requested validation such as HTML-comment rejection.

## Best Possible Solution

Keep the issue open until OpenClaw has a shared prompt-bearing Markdown validation contract with tests and docs. Wire it into workspace bootstrap ingestion, extra bootstrap files, SKILL.md discovery/model-visible read paths, and local ClawHub skill install/update handling; coordinate separate ClawHub publish-side policy as needed.
