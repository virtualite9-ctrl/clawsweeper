## Summary

Keep #12198 open. Current main has adjacent safety work, but not the requested shared Markdown validation/linting for prompt-bearing Markdown files. The existing maintainer/Codex comment also explicitly kept the item open, and current main still loads workspace bootstrap Markdown and discovers/installs skill Markdown without rejecting HTML comments or applying a common prompt-Markdown ruleset.

## Best Possible Solution

Keep the issue open until OpenClaw has a shared prompt-Markdown validation contract with tests and docs, wired into workspace bootstrap ingestion, extra bootstrap hooks, SKILL.md discovery/on-demand loading, ClawHub skill install/publish policy, and any plugin API needed for third-party enforcement before prompt Markdown reaches the model. Start with high-confidence blockers such as HTML comments and define warn/block/customization behavior explicitly.
