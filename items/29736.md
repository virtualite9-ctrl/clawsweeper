## Summary

Keep #29736 open. Current main still resolves exec approvals through home-relative `~/.openclaw` defaults, not the active `OPENCLAW_STATE_DIR` or profile-derived state directory. The issue discussion has concrete repros across systemd, profile isolation, and containers; the later “seems fixed” comment is not supported by current source, tests, or docs.

## Best Possible Solution

Keep the issue open and fix the core exec approvals path contract. Default approvals file and socket resolution should derive from the active state root/profile semantics, likely via `resolveStateDir()`, while preserving a compatibility or migration path for existing home-backed approvals. Update CLI, gateway RPC, node-host RPC, docs, and regression tests for `OPENCLAW_STATE_DIR`, `--profile`, container writable-state deployments, `OPENCLAW_HOME`, and symlink safety.
