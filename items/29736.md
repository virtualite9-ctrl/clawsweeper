## Summary

Keep #29736 open. Current main still defaults exec approvals file and socket resolution to home-relative `~/.openclaw`, while the active state-dir/profile path machinery exists separately. The issue has concrete reproductions across systemd, profile isolation, and containers; the later “seems now to work” comment is not supported by the current source, tests, or docs I inspected.

## Best Possible Solution

Keep the issue open and fix the core exec approvals path contract. Default approvals file and socket resolution should derive from the active state root/profile semantics, likely through `resolveStateDir()`, while preserving compatibility for existing home-backed approvals. Update CLI, gateway RPC, node-host RPC, docs, and regression tests for `OPENCLAW_STATE_DIR`, `--profile`, container writable-state deployments, `OPENCLAW_HOME`, and symlink safety.
