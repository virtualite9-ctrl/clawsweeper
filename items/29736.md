## Summary

Keep #29736 open. Current main still routes exec approvals through hard-coded home-relative defaults (`~/.openclaw/exec-approvals.json` and `.sock`) via `OPENCLAW_HOME`/home expansion, while `OPENCLAW_STATE_DIR` and profile-derived state dirs are resolved separately. The issue discussion includes concrete reproductions across systemd, profiles, and containers; one commenter reported it seemed fixed in 2026.4.11, but the current source, tests, and docs still encode the split-state behavior.

## Best Possible Solution

Keep the issue open and fix the core exec approvals path contract. Default approvals file and socket resolution should derive from the active state root/profile semantics, likely through `resolveStateDir()`, while preserving a compatibility or migration path for existing home-backed approvals. Update CLI, gateway RPC, node-host RPC, docs, and regression tests for `OPENCLAW_STATE_DIR`, `--profile`, `OPENCLAW_HOME`, container writable-state deployments, and symlink safety.
