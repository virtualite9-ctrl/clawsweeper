## Summary

Keep #29736 open. Current main still resolves the TypeScript exec approvals file/socket from home-relative defaults instead of the active OPENCLAW_STATE_DIR/profile state root, and the CLI, gateway, node-host management, and runtime enforcement paths still call those helpers. The comments from AlyaKoni, raywu, and mostlydev provide concrete Linux/profile/container reproductions; the later “works in 2026.4.11” comment is not enough to close because current TypeScript/headless code and docs still encode ~/.openclaw behavior.

## Best Possible Solution

Keep the issue open and fix the core TypeScript exec approvals path contract. Default approvals file and socket resolution should derive from the active state root/profile semantics, likely via resolveStateDir(), for CLI, gateway, runtime enforcement, and headless node-host paths. Add compatibility migration/fallback for existing home-backed approvals, preserve destination safety checks, and cover OPENCLAW_STATE_DIR, --profile, container writable-state, and OPENCLAW_HOME compatibility with regression tests/docs.
