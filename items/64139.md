## Summary

Keep open. Current main still has no crontab-specific implementation, diagnostic, docs, or test coverage, and the OpenClaw cron CLI path still goes through Gateway RPC/pairing with admin-scoped cron mutations. Related #72240 may cover generic SIGKILL diagnostics, but it does not supersede the crontab and cron administration parts of this issue.

## Best Possible Solution

Keep this issue open. Reproduce `crontab -l`, `crontab -e`, and stdin `crontab -` through OpenClaw exec on a current macOS LaunchAgent install, then fix or document the exec environment, TTY, stdin, timeout, and diagnostic behavior. Separately make an explicit product/security decision for same-host cron administration: use least-privilege scopes for read-only cron commands and clearer `openclaw devices list/approve` guidance, or add a narrow same-host token/device reuse path that preserves the Gateway trust boundary for admin cron mutations.
