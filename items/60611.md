## Summary

Current main already preserves proxy/custom environment variables across macOS Gateway LaunchAgent reinstalls, and the restart path itself reuses the existing plist instead of regenerating it. The same implementation is present in the shipped `v2026.4.23` release.

## Evidence

- **Restart path uses existing plist, not a rewrite:** `restartLaunchAgent` does `launchctl enable` + `kickstart`, with a bootstrap fallback that re-registers the existing plist path if the job was booted out; it does not rebuild the plist during restart.
  - file: [src/daemon/launchd.ts:625](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/src/daemon/launchd.ts#L625)
  - command: `nl -ba src/daemon/launchd.ts | sed -n '625,690p'`
  - sha: [6bc0dc8fb671](https://github.com/openclaw/openclaw/commit/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440)
- **Forced reinstall merges prior service env into the new install:** `runDaemonInstall` reads the current service environment and merges preserved non-OpenClaw vars into the install environment before rebuilding the LaunchAgent.
  - file: [src/cli/daemon-cli/install.ts:27](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/src/cli/daemon-cli/install.ts#L27)
  - command: `nl -ba src/cli/daemon-cli/install.ts | sed -n '27,109p'`
  - sha: [6bc0dc8fb671](https://github.com/openclaw/openclaw/commit/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440)
- **Installer preserves existing custom env vars in rewritten plist:** `buildGatewayInstallEnvironment` carries forward existing service env vars unless they are OpenClaw-managed or blocked, then overlays the current service environment. This is the code path that keeps manual proxy env entries on app/CLI-managed LaunchAgent rewrites.
  - file: [src/commands/daemon-install-helpers.ts:178](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/src/commands/daemon-install-helpers.ts#L178)

- **Proxy env passthrough is explicit and tested:** The service env builder explicitly includes `HTTP_PROXY`, `HTTPS_PROXY`, `NO_PROXY`, `ALL_PROXY`, and lowercase variants, and the test suite asserts those values are forwarded for launchd/systemd runtime.
  - file: [src/daemon/service-env.ts:47](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/src/daemon/service-env.ts#L47)

  - command: `git show v2026.4.23:src/cli/daemon-cli/install.ts | sed -n '1,180p'`

That matches the reporter's observation that the plist can be rewritten by OpenClaw-managed service refreshes.
