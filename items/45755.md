## Summary

Keep #45755 open. Current main still does not render the agent identity emoji inline with the assistant name in the chat message header/footer path described by the issue discussion. The backend can resolve an emoji, but the Control UI assistant identity state and chat group renderer only carry/render name and avatar, so the reported v2 UI regression remains plausible and unaddressed.

## Best Possible Solution

Implement the missing Control UI chat identity path: preserve `identity.emoji` in the frontend assistant identity state, pass it through `renderChat` into grouped/streaming chat renderers, and render it inline with the assistant name where the chat sender label is shown. Add focused UI tests for configured IDENTITY.md emoji in assistant message groups and streaming groups; consider separately whether session/agent dropdown labels should use the same helper.
