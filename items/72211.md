Keep PR #72211 open. ## Best Possible Solution

Keep the PR open and request a small docs revision: preserve the local-dashboard-after-update recovery runbook, but split the flow so users first run lsof/curl to determine whether the gateway is already listening and serving HTML, then run openclaw gateway restart/start only when the service is not healthy. After that, it can be reviewed as a narrow gateway troubleshooting docs improvement.
