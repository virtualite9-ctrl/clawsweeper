## Summary

Keep this PR open. Current main at 4878d3e059cee7fb0652cf973e82d76cc7371764 still does not implement `{context}` or `{contextPercent}` response-prefix variables, and the PR remains the concrete implementation candidate for #1920. It should not be closed as implemented or obsolete, but it still needs revision because the latest PR review identified a real semantics bug: context usage must be prompt-side context tokens, not aggregate totals that can include output tokens.

## Best Possible Solution

Keep this PR open and request a focused revision. The implementation should derive `{context}` and `{contextPercent}` from prompt-side context usage, preferably using existing helpers such as `deriveContextPromptTokens` or `deriveSessionTotalTokens`; wire a usage callback through the reply-prefix pipeline for gateway and plugin/channel deliveries; update config/docs help for the new variables; and add focused tests for template resolution plus runtime population before maintainer review.
