## Summary

Current `main` already guards Feishu quoted-message parsing failures and continues dispatch when quoted-message lookup fails. The same guards are also present in the latest shipped release commit `a9797214338ba31c52c796adbb75afb16e0684a9` (`v2026.4.23`).

## Evidence

- **Quoted-message content parsing is guarded:** `parseFeishuMessageContent` wraps `JSON.parse(rawContent)` in `try/catch` and falls back to returning the raw content on parse failure instead of throwing.
  - file: [extensions/feishu/src/send.ts:219](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/extensions/feishu/src/send.ts#L219)
  - command: `nl -ba extensions/feishu/src/send.ts | sed -n '219,229p'`
  - sha: [6bc0dc8fb671](https://github.com/openclaw/openclaw/commit/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440)
- **Message fetch path already swallows parse/fetch failures:** `getMessageFeishu` calls `parseFeishuMessageItem(...)` inside a `try` block and returns `null` on any exception, so malformed message bodies do not escape this helper.
  - file: [extensions/feishu/src/send.ts:301](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/extensions/feishu/src/send.ts#L301)
  - command: `nl -ba extensions/feishu/src/send.ts | sed -n '301,324p'`
  - sha: [6bc0dc8fb671](https://github.com/openclaw/openclaw/commit/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440)
- **Dispatch path already degrades gracefully on quoted-message failure:** The Feishu bot dispatch path wraps quoted-message lookup in its own `try/catch`, logs failure, and continues building the inbound message with `quotedContent` left unset.
  - file: [extensions/feishu/src/bot.ts:728](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/extensions/feishu/src/bot.ts#L728)
  - command: `nl -ba extensions/feishu/src/bot.ts | sed -n '728,757p'`
  - sha: [6bc0dc8fb671](https://github.com/openclaw/openclaw/commit/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440)
- **Latest release already contains the same guard:** The released `v2026.4.23` tree shows the same `JSON.parse(rawContent)` guard and raw-content fallback in the Feishu message parser.
  - file: [extensions/feishu/src/send.ts:219](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/extensions/feishu/src/send.ts#L219)
  - command: `git show v2026.4.23:extensions/feishu/src/send.ts | nl -ba | sed -n '219,229p'`
  - sha: [a9797214338b](https://github.com/openclaw/openclaw/commit/a9797214338ba31c52c796adbb75afb16e0684a9)
- **Related timeline item is broader scope than this crash:** The only linked timeline reference is issue #29487, which is about including quoted/parent content in forwarded Feishu events. The crash path described here is already separately guarded in current code.

## Risks / Open Questions
