## Summary

Keep open. Current main contains the earlier Moonshot/Kimi native tool-call id fix and Moonshot `thinking.keep` payload support, but I did not find an OpenClaw-side fix for the distinct #71491 failure: assistant tool-call messages replayed after LCM/context-engine compaction or cross-model fallback can still reach the Moonshot OpenAI-compatible payload without a restored `reasoning_content` field. The external lossless-claw #488 link is relevant supporting evidence, not a superseding OpenClaw item, because the issue discussion says both LCM preservation and OpenClaw cross-model reasoning compatibility need work.

## Best Possible Solution

Keep #71491 open and add an OpenClaw-side regression path for Moonshot/Kimi thinking replay after context-engine compaction and cross-model fallback. The likely implementation belongs near the Moonshot OpenAI-compatible payload wrapper or a provider replay sanitization hook: preserve top-level `reasoning_content` when present, and for Kimi thinking-enabled assistant tool-call messages that no longer have it, add a provider-acceptable placeholder or disable/strip incompatible thinking replay deliberately. Add tests that build a compacted/fallback transcript with assistant tool calls missing `reasoning_content` and assert the outgoing Moonshot `openai-completions` payload no longer triggers the 400 shape. Coordinate separately with lossless-claw#488 so LCM preserves OpenAI-style reasoning metadata when it owns compaction.
