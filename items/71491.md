## Summary

Keep open. Current main has the earlier Moonshot/Kimi native tool-call ID fix and optional `thinking.keep="all"` forwarding, but I did not find a Moonshot/Kimi replay repair that preserves or synthesizes top-level `reasoning_content` after LCM/context-engine assembly or cross-model fallback. The fresh issue report and comments distinguish this from #70030/#70392 and link an open LCM-side stripping issue, so this remains actionable in OpenClaw.

## Best Possible Solution

Keep this issue open. Best path: add a Moonshot-owned replay/payload repair, likely in the Moonshot thinking stream wrapper or a provider replay hook, that preserves converted `reasoning_content` when present and fills a safe placeholder for assistant tool-call turns when Kimi thinking replay requires it. Cover same-provider long-history/context-engine assembly and cross-model fallback with focused tests, using the existing DeepSeek/Venice replay repair patterns as reference.
