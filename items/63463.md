## Summary

Keep open. Current main appears to handle provider/model parsing for slash-containing model ids, but this is still a recent, concrete MLX routing report with no maintainer response or reporter retry on the documented custom-provider shape. The likely config issue is missing `/v1` and `api: "openai-completions"`, but that is not enough to close without live MLX proof or a targeted regression.

## Best Possible Solution

Keep this issue open for maintainer triage. Ask the reporter to retry with models.providers.mlx.baseUrl set to http://127.0.0.1:8080/v1 and api set to "openai-completions". If it still fails, add a focused regression using a custom mlx provider with model id mlx-community/Qwen3-30B-A3B-6bit and assert the outbound chat-completions request uses that bare model id. If OpenClaw should support MLX as a first-class local LLM provider rather than custom config, track that as a separate provider/plugin/docs decision.
