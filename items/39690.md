## Summary

Keep #39690 open. Current main still preserves the remaining issue-discussion bug: when an explicit provider model matches an implicit/catalog model, `mergeProviderModels` overwrites explicit `input` metadata with the implicit value, so `input: ["text", "image"]` can still be lost before Gateway image-capability checks. The text-only offload work helps a related failure mode, but it does not implement the requested explicit-input precedence.

## Best Possible Solution

Keep the issue open and fix `mergeProviderModels` so an explicitly present `models.providers.*.models[].input` field wins over implicit/catalog metadata, matching the existing `reasoning` override pattern. Add regression coverage where an explicit image-capable model survives a text-only implicit match, then verify Gateway inline image handling still sends native image blocks for that configured model while preserving the text-only offload fallback.
