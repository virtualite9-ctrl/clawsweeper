## Summary

Keep open. Current `main` still ignores an explicit model `input` override when an implicit/provider catalog entry for the same model id exists, which is the core complaint in #39690. There is a newer mitigation for text-only runs that offloads inbound images to media refs instead of dropping them, but that does not implement the requested override behavior for custom vision-capable models.

## Evidence

- **Explicit input override is still discarded during provider-model merge:** `mergeProviderModels` copies `input` from the implicit model onto the merged result, even when the explicit config provided a different `input` value. That means a user-configured `input: ["text", "image"]` can still be replaced by bundled/catalog metadata.
  - file: [src/agents/models-config.merge.ts:100](https://github.com/openclaw/openclaw/blob/cb2c36b04975e34ab7014a11575d6589aa96ae32/src/agents/models-config.merge.ts#L100)
  - command: `nl -ba src/agents/models-config.merge.ts | sed -n '37,110p'`
- **Test suite currently locks in the old behavior:** The merge test passes an implicit model with `input: ["text"]` and an explicit model with `input: ["image"]`, then asserts the merged model still has `input: ["text"]`. That is effectively the bug report's failure mode.
  - file: [src/agents/models-config.merge.test.ts:61](https://github.com/openclaw/openclaw/blob/cb2c36b04975e34ab7014a11575d6589aa96ae32/src/agents/models-config.merge.test.ts#L61)
  - command: `nl -ba src/agents/models-config.merge.test.ts | sed -n '61,102p'`
- **Gateway image handling now mitigates text-only models instead of dropping images:** `parseMessageWithAttachments` forces offload when `supportsImages === false`, and the test asserts images are offloaded for text-only models instead of dropped. This is a useful mitigation, but it is different from honoring the explicit modality override requested in the issue.
  - file: [src/gateway/chat-attachments.ts:309](https://github.com/openclaw/openclaw/blob/cb2c36b04975e34ab7014a11575d6589aa96ae32/src/gateway/chat-attachments.ts#L309)
  - command: `nl -ba src/gateway/chat-attachments.ts | sed -n '305,428p'`
- **Gateway support check still keys off catalog `input` metadata:** `resolveGatewayModelSupportsImages` returns true when the loaded catalog entry includes `image` in `input`; otherwise it falls back only to a few provider-specific shims. If the merged catalog entry remains text-only, the gateway still treats the model as non-vision.
  - file: [src/gateway/session-utils.ts:1125](https://github.com/openclaw/openclaw/blob/cb2c36b04975e34ab7014a11575d6589aa96ae32/src/gateway/session-utils.ts#L1125)
  - command: `nl -ba src/gateway/session-utils.ts | sed -n '1115,1153p'`

## Risks / Open Questions

- I did not run a live repro because this review was read-only and the checkout had no `pnpm` available; this conclusion is based on current source and tests.
- The new offload path likely reduces user-facing breakage for some setups, so maintainers may want to narrow the issue scope rather than duplicate it with the text-only mitigation work.
