## Summary

Keep this PR open. Current main still does not implement the CompletionDeliveryGate/env-flag behavior, and the observable ACP child-session task path still sends or queues visible `Background task done` completion messages. The PR is a plausible implementation candidate, but it is not cleanup-close material and remains review-gated by unresolved P1 risks in the provided discussion; the overlapping #64416 is also open rather than a merged canonical replacement.

## Best Possible Solution

Keep this PR open but require revision before merge: implement a cleanup-safe completion ownership seam that records ownership only after a path actually delivers or intentionally takes over, add regression coverage spanning ACP task-registry banner, announce, farewell, and fallback behavior, and split or separately validate the Discord thread-binding/requester-origin changes. Reconcile the overlap with open PR #64416 during review rather than closing either as implemented.
