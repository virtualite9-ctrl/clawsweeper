## Summary

Keep this PR open. Current main still does not include the proposed CompletionDeliveryGate/env-flag behavior or an equivalent shared completion-ownership seam, and the ACP task-registry, announce, and farewell paths remain independently capable of user-visible completion delivery. The PR is not merge-ready because the provided review discussion still identifies unresolved P1/P2 functional risks, but it remains a plausible implementation candidate rather than cleanup-close material.

## Best Possible Solution

Keep this PR open for maintainer review, but require revision before merge. The best path is a smaller completion-ownership seam that records ownership only after a path actually delivers or intentionally takes over, preserves fallback farewell and cleanup semantics, keeps shadow mode observational, and adds regression coverage across ACP task-registry banner/silent-wake behavior, announce delivery, farewell hooks, and fallback paths. Split or separately justify the Discord thread-binding/requester-origin changes, and reconcile the overlap with open PR #64416 before landing either candidate.
