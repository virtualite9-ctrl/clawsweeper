## Summary

Keep this PR open. Current main still routes agent gateway tool calls through the WebSocket-backed `callGateway` path, and the cron tool still uses that helper for the affected `cron.*` operations. The PR remains a plausible implementation candidate for the reported gateway self-contention bug, but it should not be merged as-is without rebase/rework because the proposed in-process dispatch path changes auth, timeout, scope, and request-lifecycle semantics at a sensitive boundary.

## Best Possible Solution

Keep this PR open and ask for a current-main rebase or adapted patch. The maintainer-ready fix should add an in-process path only for embedded gateway-native calls while preserving explicit remote targeting, gateway token handling, timeout behavior, `expectFinal`, explicit and least-privilege scopes, and JSON-like request normalization. Add focused regression tests for cron status/list/add/run/runs plus `gatewayUrl`, `gatewayToken`, `timeoutMs`, `expectFinal`, explicit scopes, default scopes, and undefined/optional parameter cases before merge.

## Risks / Open Questions

- The PR’s local branch would ignore `opts.gatewayToken` when no explicit `gatewayUrl` is present, so explicit auth override semantics could change inside the gateway process.
- The PR’s local branch would ignore `opts.timeoutMs`; an embedded tool handler that stalls could hang the active turn instead of failing on the same bounded timeout as the WebSocket path.
- The PR does not carry `extra.scopes` or recompute method scopes for local dispatch, so authorization can differ from current least-privilege and explicit-scope behavior.
- The in-process path skips the WebSocket client’s request validation and JSON serialization round trip; local dispatch should either normalize equivalently or add regression tests for optional/undefined parameter cases.
- The PR touches an auth-sensitive boundary by bypassing normal WebSocket authentication and transport behavior, so it needs maintainer security review after rebase and semantic parity tests.
