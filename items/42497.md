## Summary

Keep this PR open. Current main still routes embedded cron/gateway tool calls through the WebSocket-backed `callGatewayTool` path and has no in-process dispatch helper for this surface. The linked bug tracker #40237 was closed as superseded by this PR, so this PR remains the canonical implementation candidate, but it needs rebase/rework for current auth, scope, timeout, token, final-response, and parameter-normalization semantics before merge.

## Best Possible Solution

Keep this PR open and ask for a current-main rebase or adapted patch. The maintainer-ready fix should add an in-process dispatch path for embedded gateway-native tool calls while preserving the WS path's option resolution, explicit token/url handling, least-privilege and explicit scopes, timeout behavior, `expectFinal` behavior, and JSON serialization semantics. Add focused regression coverage for cron tool calls plus the reviewed gatewayUrl, gatewayToken, timeout, expectFinal, scope, and undefined-parameter cases.

## Risks / Open Questions

- Because this touches an auth-sensitive gateway dispatch boundary, the best path needs maintainer review plus targeted regression tests before merge.
