## Summary

Keep open. Current main still pins and delegates TUI text input to `@mariozechner/pi-tui` 0.70.2, and that pinned upstream source still has the exact risky Kitty CSI-u path from the report: Kitty flag 7 is enabled, while printable CSI-u decoding rejects Alt/Ctrl-modified printable sequences. I found no OpenClaw-local decoder, workaround, docs, or regression coverage for German-layout/AltGr input.

## Best Possible Solution

Keep the issue open. The best fix is in the TUI input layer, ideally upstream in `@mariozechner/pi-tui` and then picked up by OpenClaw: decode printable AltGr CSI-u/modifyOtherKeys sequences by their produced printable codepoint while preserving shortcut handling, and add regression tests for German-layout examples such as `AltGr+Q -> @`, `AltGr+E -> €`, umlauts, `ß`, and duplicate-printable prevention.
