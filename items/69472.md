## Summary

Keep PR #69472 open. Current main still does not suppress the reported reasoning-prefaced `NO_REPLY.NO_REPLY` payload: the shared silent gate only accepts exact `NO_REPLY` or the JSON action envelope, and the mixed-content stripper does not remove a doubled token separated by punctuation. The PR is a plausible targeted fix for open issue #69470, but it has an unresolved P1 review concern about false positives, so it should be amended/reviewed rather than auto-closed.

## Best Possible Solution

Keep the PR open and amend it before merge: preserve the channel-agnostic fix direction, add regression coverage for the observed reasoning-prefaced `NO_REPLY.NO_REPLY` leak, and address the false-positive review by tightening `isReasoningPrefacedSilentReply` or gating it with reliable silent-turn context. If a separate fix lands for #69470 first, then supersede this PR with that canonical merged change.
