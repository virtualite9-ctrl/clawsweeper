## Summary

Keep PR #69472 open. Current main does not implement the reasoning-prefaced silent-reply suppression, the linked bug #69470 remains open, and the PR is a focused attempt with tests. It still needs maintainer review because an unresolved review comment flags a possible false-positive suppression case, but that is a reason to revise or review the PR, not to close it as obsolete.

## Best Possible Solution

Keep the PR open and have maintainers either revise this patch or implement the same fix in a follow-up: add a narrowly scoped reasoning-prefaced silent-payload detector at the shared `isSilentReplyPayloadText` gate, collapse doubled trailing silent tokens only for that detector, and keep explicit tests for #19537 false positives, bare `think`/`thinking`/`reasoning` preambles, and `NO_REPLY.NO_REPLY`.
