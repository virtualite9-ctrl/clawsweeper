## Summary

Keep open. Current main still implicitly adds filesystem tools when `tools.fs` is configured, independent of the selected `tools.profile`, while the docs and tool catalog say `minimal` is `session_status` only and `messaging` excludes filesystem tools. The linked fixes are not shipped on main: #42308 is closed unmerged, and #43061 remains an open implementation candidate.

## Best Possible Solution

Keep this issue open until main changes the implicit tool-section expansion so `tools.fs` and `tools.exec` do not re-grant tools under restrictive profiles unless explicitly requested with `alsoAllow`. The likely fix belongs in `src/agents/pi-tools.policy.ts`, with matching cleanup in `src/agents/tool-fs-policy.ts` if root-expansion logic must follow the same contract, plus regression tests for `minimal` and `messaging` with configured `tools.fs`/`tools.exec`, preserving explicit `alsoAllow` and deny precedence.
