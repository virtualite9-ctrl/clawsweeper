## Summary

Keep #70440 open. Current main has partial empty-turn cleanup: it suppresses the duplicate fallback after a committed messaging-tool delivery and can treat non-visible assistant turns as silent for group chats when silence is allowed. It still does not provide the requested global/per-channel policy to silently swallow genuine empty-turn fallback errors for direct Signal/TTS-style surfaces; direct runs continue to surface the "Agent couldn't generate a response" error by design.

## Best Possible Solution

Keep the issue open and implement an explicit incomplete/empty-turn delivery policy in the agent/auto-reply path, with global plus surface/channel overrides. The default should preserve current visible errors, while selected voice/TTS surfaces can log a warning and deliver nothing after retries are exhausted. Update config schema/docs and add direct Signal/TTS-style regression coverage.
