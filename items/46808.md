## Summary

Current main already gives the Control UI session/sub-agent cost visibility requested here: the Usage tab loads per-session usage/cost from Gateway RPCs, ranks sessions by cost, shows selected-session total cost and cumulative/per-turn timelines, and chat message footers expose per-response token/cost/model metadata. Sub-agent sessions are ordinary session keys in the usage surface and sub-agent docs note token/cost stats in announce payloads.

## Best Possible Solution

Keep the shipped Control UI Usage tab and Gateway session-usage APIs as the canonical solution. If maintainers later want a smaller always-on chat-header badge, that should be a new narrow enhancement on top of the existing `sessions.usage`/message metadata plumbing rather than keeping this broad request open.
