## Summary

Keep open. Current main still lacks the stale-version reinstall behavior targeted by this active PR: the postinstall path treats an existing package.json sentinel as installed unless a targeted optional native child is missing, while the separate runtime staging/doctor path already has version-aware sentinel checks. The PR is also tied to open regression #72058 and has a same-day author follow-up addressing review feedback, so it remains a live implementation candidate rather than cleanup-close material.

## Best Possible Solution

Keep this PR, or a focused successor, open. The best path is to review the latest head, land a version-aware stale dependency check in scripts/postinstall-bundled-plugins.mjs that matches the semver contract in src/plugins/bundled-runtime-deps.ts, keep regression tests for stale exact/range versions and optional native-child reinstall coverage, and resolve or split the lockfile importer change before merge. After a corrected PR merges, linked regression #72058 can be closed as implemented if the reporter's npm update loop is covered.
