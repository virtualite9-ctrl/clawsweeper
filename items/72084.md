## Summary

Keep open. Current main has version-aware bundled runtime-deps repair in the loader/doctor path, but the postinstall eager installer that this PR changes still treats an existing dependency sentinel as sufficient without checking package.json version. The PR is therefore not implemented on main or clearly obsolete, but its current diff needs maintainer review because bot comments flag test coverage and semver edge cases, and the diff also carries an unrelated lockfile importer change.

## Best Possible Solution

Keep the PR open for maintainer review rather than auto-closing. A good final fix would either add version-aware checks to `scripts/postinstall-bundled-plugins.mjs` using npm-compatible semver behavior, with regression tests for exact, caret, stale, prerelease, and unsupported range cases, or explicitly retire the legacy eager postinstall repair path. Before landing, fix the versionless native-child fixture and remove the unrelated `pnpm-lock.yaml` importer churn.
