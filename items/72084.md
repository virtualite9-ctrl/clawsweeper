## Summary

Keep open. Current main has version-aware bundled runtime dependency checks in the runtime staging paths, but the specific postinstall eager-install path this PR changes still treats any existing package sentinel as fresh and has no stale-version regression coverage. The PR also has unresolved review feedback in the provided context that should be addressed before merge or supersession.

## Best Possible Solution

Keep the PR or a focused successor open, address the review comments, and implement version-aware stale dependency detection for `scripts/postinstall-bundled-plugins.mjs` with regression tests for stale exact/range versions and for the existing optional-native-child reinstall path. If maintainers decide postinstall should remain sentinel-only, close only after opening or documenting the canonical runtime/doctor repair issue for the remaining #72058 update behavior.
