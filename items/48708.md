## Summary

Keep #48708 open. Current main still lacks optional user-facing timestamps in the terminal TUI, and the related implementation PR #49069 is open, unmerged, and has review-identified functional issues. The duplicate issues #49268 and #56293 were already closed in favor of #48708, so closing this item would remove the canonical tracker.

## Best Possible Solution

Keep #48708 open as the canonical product issue. The best implementation path is a focused TUI change that adds an explicit config/CLI/settings toggle, carries per-message timestamps through live events and history replay, renders timestamps as presentation metadata for user/assistant/system messages without corrupting markdown content, respects the relevant time-format choice if maintainers want that contract, and adds src/tui coverage for history, pending users, streaming assistant updates, and finalize fallback paths. PR #49069 can be reworked toward that shape or replaced by a new PR.
