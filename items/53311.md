## Summary

Keep open. The issue is paired with same-author PR #53315, which is open and uses closing syntax for this issue; current main also still treats an unknown configured channel id as a fatal validation issue that can block gateway startup.

## Best Possible Solution

Keep this issue open while maintainers review PR #53315 or a replacement patch. The likely fix belongs in `src/config/validation.ts`, with gateway startup coverage proving that a configured but undiscoverable plugin-owned channel warns and is skipped while other channels still start; update plugin manifest/config docs if the validation contract changes.
