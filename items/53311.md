## Summary

Keep open. This issue is paired with an open same-author PR that explicitly closes this issue, and current main still treats an unknown configured channel id as a fatal validation issue before plugin validation can degrade it. The requested behavior has not shipped on main.

## Best Possible Solution

Keep this issue open while maintainers review PR #53315 or a replacement patch. The product fix likely belongs in `src/config/validation.ts`, with regression coverage showing a configured but undiscoverable channel plugin warns and is skipped while other channels still start, plus docs updates if unknown plugin-owned channel config is no longer fatal.
