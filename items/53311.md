## Summary

Keep open. Current main still treats configured but undiscoverable channel ids as fatal config validation issues, and gateway startup throws on invalid snapshots. The same-author PR #53315 remains open and is the active implementation candidate.

## Best Possible Solution

Keep this issue open while maintainers review PR #53315 or a replacement patch. The likely fix belongs around `src/config/validation.ts` and gateway startup behavior: configured but undiscoverable plugin-owned channels should warn and be skipped while other channels still start, with regression coverage and docs updated if the validation contract changes.
