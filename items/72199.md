## Summary

Close as ClawHub/plugin scope. Current main does not implement a universal `bodyExtras` field, but the stated need is optional integration metadata and the existing plugin-owned message-tool surface already lets a channel/plugin contribute structured schema fields and receive them in `ctx.params` without adding a generic `{[key:string]:any}` core field. The issue has no maintainer author association, protected labels, comments, or already-posted plugin link to preserve.

## Best Possible Solution

Keep OpenClaw core unchanged for this item. Implement the DeepSeek firewall/project-specific metadata as a ClawHub/code plugin or channel action that contributes a typed `bodyExtras` schema field through `describeMessageTool(...)` and consumes `ctx.params.bodyExtras`; open a narrower plugin API issue only if that path hits a concrete core limitation.
