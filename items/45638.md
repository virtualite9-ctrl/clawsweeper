## Summary

Keep open. Current main still has the reported Telegram callback bug: the generic callback_query fallback builds a synthetic message and passes `callback.id` as `messageIdOverride`, and Telegram context uses that override directly as `MessageSid`. The related fix PRs in the provided timeline were closed unmerged, and latest release `v2026.4.24` still contains the same `messageIdOverride: callback.id` line.

## Best Possible Solution

Fix the generic Telegram callback_query fallback in `extensions/telegram/src/bot-handlers.runtime.ts` to pass `messageIdOverride: String(callbackMessage.message_id)`, and add regression coverage that a callback with `id: "cbq-*"` and `message.message_id: 10` produces inbound `MessageSid: "10"` while still using `callback.id` only for `answerCallbackQuery` and plugin callback metadata.
