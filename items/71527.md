## Summary

Current main, and the shipped v2026.4.23 release, already document and implement the supported Anthropic Claude CLI auth flow: use `openclaw models auth login --provider anthropic --method cli --set-default`, keep secrets in per-agent `auth-profiles.json`, and keep `openclaw.json` `auth.profiles` entries metadata-only. The refined issue asks for the supported `claude-cli` auth-profile schema, and that schema/flow is already present in code, docs, and tests.

## Evidence

- **Public docs define the canonical Claude CLI flow:** The authentication docs say Claude CLI reuse is the preferred Anthropic path, show `claude auth login` followed by `openclaw models auth login --provider anthropic --method cli --set-default`, and state that manual token entry writes `auth-profiles.json` plus updates config.
