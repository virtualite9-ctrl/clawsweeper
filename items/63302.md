## Summary

Keep open. Current main does not implement WhatsApp `denyFrom` or an equivalent sender blocklist, and the related per-group allowlist issue is complementary rather than a duplicate. The PR is not merge-ready as-is because its branch includes unrelated deployment/config artifacts and the latest review context flags a remaining command-gate forwarding issue.

## Best Possible Solution

Keep #63302 open, but require a cleaned-up, focused patch: remove the deployment/config artifacts, add `channels.whatsapp.denyFrom` at root and account scopes in the WhatsApp config type/schema/generated metadata/docs, thread it through account resolution and inbound policy, implement strict deny matching before normal DM/group policy without self-chat or wildcard shortcuts, forward the matcher through command-gate helpers, and add regression tests for open DMs, pairing mode, self-chat, and command authorization.
