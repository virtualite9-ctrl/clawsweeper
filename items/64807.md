## Summary

Current main still has the reported divergence path: the gateway Dashboard banner is hydrated from persisted/throttled update-check state, while `openclaw update status` uses a live registry-backed status path. Empty npm channel resolution still writes a fresh `lastCheckedAt` and returns, so a stale or empty effective cache can suppress the Overview banner for the normal gateway interval.

## Best Possible Solution

Keep this issue open and fix it in the gateway update-check owner path. The implementation should align Dashboard availability with the same effective registry result used by `openclaw update status` when the persisted cache is stale or empty, avoid treating unresolved npm channel lookup as a successful full-interval check, schedule a short retry/backoff for empty registry results, and add regression coverage for stale-cache and fresh-empty-cache cases in `src/infra/update-startup.test.ts`.
