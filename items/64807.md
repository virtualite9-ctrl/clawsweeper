## Summary

Current main still has the reported Dashboard/CLI divergence path. The gateway hydrates update availability from `update-check.json` and returns during the throttle window before doing live registry work, while `openclaw update status` always uses a registry-backed status check. A fresh-but-empty or stale-to-current cache can still suppress the Overview banner for the normal interval.

## Best Possible Solution

Keep this issue open and fix it in the gateway update-check owner path. The fix should align Dashboard availability with the same effective registry result used by `openclaw update status` when the persisted cache is stale or empty, avoid treating unresolved npm channel lookup as a successful full-interval check, add a short retry/backoff for empty registry results, and add regression coverage in `src/infra/update-startup.test.ts` for stale-cache and fresh-empty-cache cases.
