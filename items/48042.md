## Summary

Keep open. Current `main` still treats top-level `browser.cdpUrl` as the legacy `openclaw` profile override, while implicit browser requests still use `browser.defaultProfile`. If config sets `defaultProfile: "user"`, requests continue down the `existing-session` path that ignores CDP URL and can fall back to `DevToolsActivePort`, so the reported behavior remains plausible on current `main` and latest release `v2026.4.22`.

There is no override that switches `defaultProfile: "user"` back to `openclaw` when `cdpUrl` is set.
