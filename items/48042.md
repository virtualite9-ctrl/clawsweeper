## Summary

Keep #48042 open. Current main still preserves explicit `browser.defaultProfile: "user"` while also auto-creating the implicit `user` existing-session profile, so the reported config can still route through Chrome MCP instead of the top-level `browser.cdpUrl`. The related PR #48068 described the right fix, but it was closed unmerged.

## Best Possible Solution

Keep the issue open and land a focused resolver fix similar to #48068: when top-level `browser.cdpUrl` is configured and `defaultProfile: "user"` is present without an explicit `profiles.user`, route the default to the CDP-backed `openclaw` profile or otherwise make the selected default honor that CDP endpoint. Add a regression test for the exact config from #48042 and update browser docs if the intended precedence changes.
