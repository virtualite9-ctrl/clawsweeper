## Summary

The request is for a specialized optional memory plugin enhancement, and current OpenClaw scope explicitly routes that kind of work to externally maintained community plugins/ClawHub rather than OpenClaw core.

## Close Comment

Closing from a Codex review. This request is for a specialized LanceDB memory plugin, and current OpenClaw scope keeps optional plugin capability in the plugin ecosystem rather than in core. `VISION.md:54-64` says core should stay lean, optional capability should usually ship as plugins, and plugin authors should host and maintain them in their own repos. `docs/plugins/community.md:9-16` and `:133-137` make ClawHub/npm the canonical path for community plugins. OpenClaw already exposes `memory-lancedb` as an install-on-demand memory plugin (`docs/tools/plugin.md:109-112`), with plugin-owned config in `extensions/memory-lancedb/openclaw.plugin.json:2-105` and plugin-owned CLI hooks in `extensions/memory-lancedb/index.ts:535-575`. On that basis, this belongs on the ClawHub/community-plugin path rather than as an OpenClaw core issue. If a future version needs a specific missing core SDK seam, please open a focused issue describing that exact host/plugin contract gap.
