## Summary

Keep open. Current main still contains the reported Unix/macOS behavior: `findGatewayPidsOnPortSync` parses `lsof -Fpc` output and `parsePidsFromLsofOutput` only returns PIDs whose `c` command token contains `openclaw`. A real Node-launched gateway reported by lsof as `cnode` would still be dropped. The matching argv-based verifier exists only on the Windows path, and the linked PR #70681 is still open rather than merged into main.

## Best Possible Solution

Keep #70664 open and review/land PR #70681 or an equivalent Unix argv-verification fix. The fix should remove gateway classification from raw `lsof -Fpc` command tokens, verify Unix listener PIDs via full argv/process-title inspection, and update the stale-pid and port-polling regression tests for the `lsof cnode` / `ps openclaw-gateway` macOS case.

This matches the issue's root-cause claim and still drops `cnode` listeners.
