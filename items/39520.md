## Summary

Keep PR #39520 open. Current main still does not implement the requested Feishu-owned `channels.feishu.localRoots` / account-scoped `localRoots` config, and `sendMediaFeishu` still only forwards core-provided `mediaLocalRoots` arrays to `loadWebMedia`. The PR remains a focused Feishu plugin fix for the linked #39506 behavior rather than an obsolete duplicate of the older #27884 context-roots fix.

## Best Possible Solution

Keep the PR open for maintainer review or land an equivalent implementation. The best path is to add a Feishu-owned `localRoots: "any" | string[]` field to shared top-level/account config, resolve it from the merged account config, define empty-array behavior intentionally, inject raw `readFile` only for the explicit `"any"` case, and add Feishu docs plus tests for top-level, account override, empty array, and no `readFile` on array roots.
