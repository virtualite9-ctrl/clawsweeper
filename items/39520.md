## Summary

Keep PR #39520 open. Current main still lacks the Feishu-owned `channels.feishu.localRoots` / account-scoped `localRoots` config requested by #39506, and `sendMediaFeishu` still only forwards core-provided `mediaLocalRoots`. The PR remains a focused bundled Feishu plugin fix, not an obsolete duplicate of the narrower #27884 behavior.

## Best Possible Solution

Keep the PR open for maintainer review or an equivalent implementation. The best path is to add a Feishu-owned `localRoots: "any" | string[]` config to the shared top-level/account schema, resolve it through the merged account config, intentionally define empty-array behavior, inject raw `readFile` only for the explicit `"any"` case, and add Feishu docs plus tests for top-level, account override, empty array, and no `readFile` on array roots.
