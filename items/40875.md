## Summary

Keep this PR open. Current main has not shipped the proposed heartbeat docs clarification, but the PR is not ready to close or merge as-is because its wording no longer fully matches current fallback behavior and its changelog entry misses the required contributor attribution. The related #23718 remains an open feature request for richer heartbeat.model fallback configuration, not a duplicate of this docs-only correction.

## Best Possible Solution

Keep the PR open and request a narrow revision: document the current candidate rules exactly, including no override and same-provider override using the standard chain, different-provider override using configured fallbacks only when the override is in that list, and otherwise falling back only to the configured primary final candidate. Add the required `Thanks @liyishuai` changelog attribution, or drop the changelog entry if maintainers consider this docs clarification too small for Unreleased. Treat #23718 separately as the product request to add explicit fallback-chain support for `heartbeat.model`.
