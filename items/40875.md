## Summary

Keep this PR open. Current main still has the heartbeat docs gap, but the PR's proposed wording is inaccurate against current fallback behavior because a different-provider heartbeat override can still fall through to the configured primary as the final candidate. The PR is docs-only and has no observed security-sensitive changes, so the useful path is a narrow wording revision rather than closing it.

## Best Possible Solution

Keep this PR open and request a narrow revision against current main: update the `heartbeat.model` `ParamField` note to say no override and same-provider overrides follow the standard fallback chain; different-provider overrides skip unrelated configured fallbacks unless the override is already in `agents.defaults.model.fallbacks`; and override runs can still fall through to `agents.defaults.model.primary` as the final candidate. Align the wording with the public Model failover docs and either fix the changelog attribution or drop the changelog entry.
