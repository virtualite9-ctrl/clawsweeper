## Summary

Keep this PR open. Current main still lacks heartbeat-specific model fallback documentation, but the proposed docs wording is inaccurate against current fallback behavior and is stale against the current ParamField docs shape. The PR is docs/changelog-only and does not show a security-sensitive code, workflow, dependency, install, or release-script change.

## Best Possible Solution

Keep this PR open and request a narrow revision against current main: add the fallback note inside the current `heartbeat.model` ParamField, align it with `docs/concepts/model-failover.md`, say no override and same-provider overrides follow the standard fallback chain, say different-provider overrides skip unrelated configured fallbacks unless already in the configured fallback chain, and mention that override runs can still fall through to `agents.defaults.model.primary` as the final candidate. Fix the changelog attribution or remove the changelog entry.
