## Summary

Keep open. Current main still intentionally ignores `lastGood` for auth profile ordering and the slug generator still uses the default agent model with only a general timeout override. The shared-timeout part appears improved or stale because each embedded attempt now receives a fresh `params.timeoutMs`, but that does not close the whole issue.

## Best Possible Solution

Keep the issue open, but narrow the implementation plan: add a failover-specific auth profile ordering mode that can prefer `lastGood` after rate-limit/auth failures without breaking normal round-robin or user-pinned profiles; add focused tests around `resolveAuthProfileOrder`/`advanceAuthProfile`; and separately decide whether slug generation needs a dedicated lightweight model config surface plus docs/schema coverage. Reproduce any remaining cooldown poisoning separately because the original shared per-run timeout budget no longer matches current attempt-timer behavior.
