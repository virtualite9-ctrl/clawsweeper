## Summary

Keep open. Current main has Feishu DM routing and docs for raw `ou_xxx` peer bindings, but the report describes `user:ou_...` bindings and does not include the actual `openclaw.json`; current normal route binding matching appears exact and does not strip `user:` prefixes, so this cannot be safely closed as implemented or non-reproducible.

## Best Possible Solution

Keep the issue open and ask for the exact `agents`, `bindings`, and `channels.feishu.accounts` config plus the route debug logs. If the config used prefixed IDs such as `user:ou_xxx`, the maintainer path is either to add Feishu-aware normalization/doctor guidance for route binding peer IDs or update docs/errors to make raw `ou_xxx` mandatory. If raw `ou_xxx` bindings with distinct account IDs still reproduce on current Feishu plugin versions, fix the Feishu inbound account/peer route path and add a targeted regression test.
