## Summary

Keep open. Current `main` has pinned Docker base-image digests, a weekly Dependabot Docker config, and brief Docker docs, but it still lacks the requested stale-image alert automation and a documented digest-update runbook, so the issue is not implemented on `main` yet.

## Evidence

- **Dockerfile still requires manual digest refresh:** `Dockerfile` hardcodes the `node:24-bookworm` and `node:24-bookworm-slim` digests and explicitly says the trade-off is that digests must be updated manually with `docker buildx imagetools inspect`.
  - file: [Dockerfile:24](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/Dockerfile#L24)
  - sha: [ec3dbd22a4de](https://github.com/openclaw/openclaw/commit/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46)
- **Dependabot only covers weekly Docker updates generically:** `.github/dependabot.yml` has a weekly `package-ecosystem: docker` entry grouped as `docker-images`, but nothing in the checked-in config verifies digest freshness or adds a stale-image alert path.
  - file: [.github/dependabot.yml:116](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/.github/dependabot.yml#L116)
  - sha: [ec3dbd22a4de](https://github.com/openclaw/openclaw/commit/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46)
- **No dedicated base-image age workflow exists:** The workflow set includes `docker-release.yml` and the generic `stale.yml`, but there is no `docker-base-image-check.yml` or equivalent freshness-check workflow in `.github/workflows`. Repo search also found no implementation of a base-image age check.
  - command: `find .github/workflows -maxdepth 1 -type f | sort; rg --line-number 'docker-base-image-check|base image age|days_old|docker hub api' .github/workflows docs Dockerfile`
  - sha: [ec3dbd22a4de](https://github.com/openclaw/openclaw/commit/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46)
- **Docs mention metadata, not the requested update runbook:** `docs/install/docker.md` documents OCI base-image metadata, but the checked-in docs do not add a section explaining how to update the pinned Node digests or how automation should keep them fresh.
  - file: [docs/install/docker.md:296](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/docs/install/docker.md#L296)
  - sha: [ec3dbd22a4de](https://github.com/openclaw/openclaw/commit/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46)
- **History shows the Docker surface landed without the requested automation:** The commit that introduced the current Dockerfile, Docker docs, workflows, and Dependabot config (`16c7de085c99884367d34eaa9e52806d62ebf871`) did not include a base-image freshness alert workflow or digest-update documentation beyond the manual Dockerfile comment.
  - command: `git show --stat --summary 16c7de08 -- Dockerfile .github/dependabot.yml docs/install/docker.md .github/workflows`
  - sha: [16c7de085c99](https://github.com/openclaw/openclaw/commit/16c7de085c99884367d34eaa9e52806d62ebf871)
