## Summary

Keep open. Current main still does not provide the browser tab's current URL to `before_tool_call` for `browser` `act`/`type` calls, and the plugin hook context has no documented current-tab query helper. Main now returns a policy-checked `url` from `/act` responses, which supports the issue comment's cache/workaround path, but that arrives after the tool call and does not let a credential-vault hook domain-check before injecting text.

## Best Possible Solution

Add a narrow, documented browser-owned hook enrichment or query seam: before `browser` `act`/`type` hooks run, resolve the target tab's policy-checked current URL from `targetId`/profile/target and expose it as a reserved field such as `_currentUrl`, or expose an explicit SDK/runtime helper that before_tool_call hooks can call. Keep the existing after_tool_call URL response behavior and the credential-vault cache workaround as compatible fallbacks.
