## Summary

Keep open. Current `main` still passes only the raw browser tool params into `before_tool_call`, with no injected current-page URL for `action: "act"` / `kind: "type"`, and the browser act response path for `type` still omits `url`, so the requested core capability is not implemented.

## Evidence

- **before_tool_call sees normalized raw params only:** `runBeforeToolCallHook(...)` normalizes the incoming tool params and forwards them directly to `hookRunner.runBeforeToolCall(...)`; it does not enrich browser calls with page state or URL context.
  - file: [src/agents/pi-tools.before-tool-call.ts:253](https://github.com/openclaw/openclaw/blob/d38ed0831d7372c8505dc944aea69b5b83da1435/src/agents/pi-tools.before-tool-call.ts#L253)
  - sha: [d38ed0831d73](https://github.com/openclaw/openclaw/commit/d38ed0831d7372c8505dc944aea69b5b83da1435)
- **browser tool schema has no current-page URL field for act/type hooks:** The browser tool schema exposes `url`/`targetUrl` as ordinary caller-supplied inputs plus flattened act fields, but there is no `_currentUrl` or similar hook-only field in the browser tool contract.
  - file: [extensions/browser/src/browser-tool.schema.ts:88](https://github.com/openclaw/openclaw/blob/d38ed0831d7372c8505dc944aea69b5b83da1435/extensions/browser/src/browser-tool.schema.ts#L88)
  - sha: [d38ed0831d73](https://github.com/openclaw/openclaw/commit/d38ed0831d7372c8505dc944aea69b5b83da1435)
- **browser tool forwards only supplied act fields:** `readActRequestParam(...)` builds the act request from the provided `request` object or flattened legacy fields and forwards those values; it does not look up or attach the tab's current URL before execution.
  - file: [extensions/browser/src/browser-tool.ts:160](https://github.com/openclaw/openclaw/blob/d38ed0831d7372c8505dc944aea69b5b83da1435/extensions/browser/src/browser-tool.ts#L160)
  - sha: [d38ed0831d73](https://github.com/openclaw/openclaw/commit/d38ed0831d7372c8505dc944aea69b5b83da1435)
- **tests still cover act compatibility without URL enrichment:** Current browser-tool tests for flattened `act` compatibility assert forwarding of `kind`, `ref`, `text`, `targetId`, and `timeoutMs` only; there is no expectation that current URL is added.
  - file: [extensions/browser/src/browser-tool.test.ts:752](https://github.com/openclaw/openclaw/blob/d38ed0831d7372c8505dc944aea69b5b83da1435/extensions/browser/src/browser-tool.test.ts#L752)
  - sha: [d38ed0831d73](https://github.com/openclaw/openclaw/commit/d38ed0831d7372c8505dc944aea69b5b83da1435)
- **type actions still do not return url afterward:** In the browser `/act` route, `type` returns `{ ok: true, targetId }` while only some other actions such as `click`, `resize`, and `evaluate` include `url`. That means `after_tool_call` still cannot reliably recover URL for `type` from the core response path.
  - file: [extensions/browser/src/browser/routes/agent.act.ts:428](https://github.com/openclaw/openclaw/blob/d38ed0831d7372c8505dc944aea69b5b83da1435/extensions/browser/src/browser/routes/agent.act.ts#L428)
  - sha: [d38ed0831d73](https://github.com/openclaw/openclaw/commit/d38ed0831d7372c8505dc944aea69b5b83da1435)
