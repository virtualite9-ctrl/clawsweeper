## Summary

Keep #39013 open. Current main still has the deterministic `/context list` display bug: hook-mutated bootstrap files can enter the report with a valid `path` but no `name`, `buildBootstrapInjectionStats` returns `name: file.name`, and `/context list` renders that field directly. The session-memory portion may need a fresh focused reproduction because current session indexing filters to user/assistant message records, but the unresolved display path alone is enough to keep the issue open.

## Best Possible Solution

Keep the issue open. First fix `buildBootstrapInjectionStats` to derive a stable display label from `file.name`, then `path.basename(file.path)`, then `file.path`, then an explicit placeholder, and add a regression test for a hook-injected virtual bootstrap file with `path` and no `name`. Then retest the session-memory/accumulation side and either exclude any remaining injected system/bootstrap content from session recall or split the hook-specific dedupe behavior into a narrower follow-up.
