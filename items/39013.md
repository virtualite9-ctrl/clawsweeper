## Summary

Keep #39013 open. Current main still has the deterministic `/context list` display bug for hook-injected virtual bootstrap files that provide `path` but no `name`: bootstrap hooks can mutate `context.bootstrapFiles`, sanitization preserves a path-only object without synthesizing a display name, `buildBootstrapInjectionStats` reports `file.name`, and `/context list` renders that field directly. The session-memory echo/accumulation part may need a separate focused recheck, but the still-present display bug is enough to keep the issue open.

## Best Possible Solution

Keep the issue open. First fix `buildBootstrapInjectionStats` to derive a stable display label from `file.name`, then `path.basename(file.path)`, then `file.path`, then an explicit placeholder, and add a regression test for a hook-style virtual bootstrap file shaped like `{ path, content, virtual: true }` with no `name`. Then recheck the session-memory/accumulation behavior on current main and either exclude injected bootstrap/system content from session indexing or split that remaining behavior into a focused issue.
