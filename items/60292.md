## Summary

Current main already routes the macOS Canvas panel through the gateway-advertised scoped canvas host URL, and the released v2026.4.23 tag contains the same implementation. That directly addresses the reported mismatch where node canvas RPC worked but the app webview hit an unauthorized canvas host route.

## Evidence

- **Gateway hello now advertises a scoped canvas host URL:** The websocket hello payload mints a canvas capability token and rewrites `canvasHostUrl` to a scoped URL before sending it to clients. This is the server-side fix for a webview loading the raw canvas host and getting `Unauthorized`.
  - file: [src/gateway/server/ws-connection/message-handler.ts:1265](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/gateway/server/ws-connection/message-handler.ts#L1265)
  - command: `nl -ba src/gateway/server/ws-connection/message-handler.ts | sed -n '1260,1289p'`
  - sha: [a9797214338b](https://github.com/openclaw/openclaw/commit/a9797214338ba31c52c796adbb75afb16e0684a9)
- **macOS app reads the advertised canvas host URL from the gateway snapshot:** The macOS app exposes `snapshot.canvashosturl` via `GatewayConnection.canvasHostUrl()`, so the Canvas panel uses the scoped URL from the gateway snapshot instead of constructing its own unauthenticated host URL.
  - file: [apps/macos/Sources/OpenClaw/GatewayConnection.swift:312](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/apps/macos/Sources/OpenClaw/GatewayConnection.swift#L312)
  - command: `nl -ba apps/macos/Sources/OpenClaw/GatewayConnection.swift | sed -n '308,316p'`
  - sha: [a9797214338b](https://github.com/openclaw/openclaw/commit/a9797214338ba31c52c796adbb75afb16e0684a9)
- **Canvas panel auto-navigation is driven by the snapshot canvas host URL:** `CanvasManager` watches gateway snapshots, resolves the A2UI URL from the advertised canvas host URL, and auto-navigates the visible panel to that target. That matches the exact failing surface in this issue: the app-opened Canvas window.
  - file: [apps/macos/Sources/OpenClaw/CanvasManager.swift:153](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/apps/macos/Sources/OpenClaw/CanvasManager.swift#L153)
  - command: `nl -ba apps/macos/Sources/OpenClaw/CanvasManager.swift | sed -n '153,199p'`
  - sha: [a9797214338b](https://github.com/openclaw/openclaw/commit/a9797214338ba31c52c796adbb75afb16e0684a9)
- **Node capability refresh also returns a fresh scoped URL:** `node.canvas.capability.refresh` rotates the caller capability and returns a new scoped `canvasHostUrl`, which is the compatible path for node-driven canvas/A2UI flows after capability expiry.
  - file: [src/gateway/server-methods/nodes.ts:753](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/gateway/server-methods/nodes.ts#L753)
