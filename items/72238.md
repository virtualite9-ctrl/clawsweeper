## Summary

Keep open. This PR is a focused, recent fix for an open browser-plugin regression and current main has not implemented the runtime shim: the browser Playwright code still uses static value imports from `playwright-core`, while bundled runtime deps are staged externally and registered through `NODE_PATH`. The PR head adds the `createRequire` shim and rewires affected mocks, so it should get maintainer validation rather than conservative cleanup closure.

## Best Possible Solution

Keep PR #72238 open and either land this runtime-shim approach, or an equivalent browser-plugin runtime resolution fix, after validating the packaged install path from #72168. Before merge, update any remaining tests that spy on `playwright-core` directly and capture a targeted regression check for the staged-runtime dependency case.
