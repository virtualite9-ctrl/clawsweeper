## Summary

Keep open. Current main still resolves Control UI runtime identity through a global-first `ui.assistant` fallback chain, and the Agents/Agent Context panel prefers that runtime identity over the correct per-agent list payload. The related fix PR #54730 was closed unmerged, and current tests do not cover the reported subagent identity precedence.

## Best Possible Solution

Keep this issue open. Land a focused fix similar to the unmerged #54730 direction: for non-default agents, `resolveAssistantIdentity()` should prefer `agents.list[].identity` over global `ui.assistant` while preserving default-agent compatibility. Add regression tests for default global priority, subagent per-agent name, subagent emoji/avatar, and partial fallback, and adjust Agent Context so configured emoji/avatar state is displayed clearly instead of only `custom`/`—`.
