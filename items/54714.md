## Summary

Keep open. Current main at 52249927ac2dc579a2f8e72aa99a30e0ab4f9a11 still has the reported Control UI Agent Context failure path: the backend runtime identity resolver prefers global `ui.assistant` name/avatar before `agents.list[].identity` for any selected agent, and the Agent Context UI then prefers that runtime identity over the per-agent list payload. The avatar row still renders only `custom` or `—`, so configured emoji values are not shown there. The focused fix PR #54730 is closed unmerged, and open PR #72579 is related to the Chat identity race surface but is not merged or shipped.

## Best Possible Solution

Keep this issue open. Land or review a focused fix that makes non-default agents prefer `agents.list[].identity` name/avatar/emoji over global `ui.assistant`, preserves default-agent global override behavior, updates Agent Context to show configured emoji/avatar state instead of only `custom` or `—`, and adds regression tests for backend resolver precedence, `agent.identity.get`, and `buildAgentContext` display behavior.
