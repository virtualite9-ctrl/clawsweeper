## Summary

Keep this issue open. Current checked-out main still has the reported Control UI Agents identity regression: the backend resolver prefers global `ui.assistant` before `agents.list[].identity`, `agent.identity.get` returns that runtime value, and the Agents panel lets that runtime value override the per-agent list identity. Emoji-only identities still render as `—` in the Agent Context avatar row. Related PR #54730 is closed unmerged, and #72579 is still only an implementation candidate in the provided context, not shipped behavior.

## Best Possible Solution

Keep this issue open. The best fix is a focused change that makes non-default agents prefer `agents.list[].identity` name/avatar/emoji over global `ui.assistant` in `resolveAssistantIdentity()` and `agent.identity.get`, preserves the configured default-agent override behavior, updates `buildAgentContext()` to display per-agent emoji/avatar state correctly, and adds regression tests for resolver precedence, RPC output, and Agent Context rendering. Review/land or split the relevant parts of #72579 before closing.
