## Risks / Open Questions

- The PR's final `stripUserMessageSeparator` regex is global after metadata removal, so it can delete later user-authored `---openclaw:user-msg---` lines that are separated by blank lines from real body text. That is a content-integrity risk in user-facing history/sanitizer paths.
- The PR does not update prompt-composition helpers that still synthesize the old metadata/body join shape, so tests may keep validating a different prompt body than `runPreparedReply` would send after the change.
- The PR patch was authored against an older `get-reply-run.ts` shape than current main, so maintainers should rebase/adapt it before review and run the changed gate.
