## Summary

Keep open. Current main still does not preserve explicit Slack Agents & Assistants DM root thread context for non-`all` reply modes, and this PR remains a plausible implementation candidate. It should be revised before merge because its current DM gate uses raw `channel_type` and the branch includes an unrelated agents overflow harness change.

## Best Possible Solution

Keep this PR open and revise it: pass normalized Slack chat/direct-message classification into `resolveSlackThreadContext` or derive it there with the existing normalization helper; set `MessageThreadId` only for Slack direct-message assistant root events with `thread_ts === ts`; preserve existing channel/group top-level behavior for `off`, `first`, and `batched`; add focused tests for `channel_type: "im"`, missing `channel_type`, wrong `channel_type` on D-prefix DMs, and non-DM top-level roots; remove or split the unrelated overflow harness change.
