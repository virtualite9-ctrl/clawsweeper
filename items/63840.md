## Summary

Current main still does not preserve Slack Agents & Assistants DM root thread context for non-`all` reply modes, so this PR is not obsolete. The branch is directionally useful, but it should stay open for revision because the proposed DM guard keys on raw `channel_type` instead of OpenClaw's normalized DM classification and it bundles an unrelated agents test-harness change.

## Best Possible Solution

Keep open and revise. Preserve Slack Agents & Assistants DM root thread context using the normalized direct-message classification already computed in the Slack message pipeline, keep non-DM self-thread roots from forcing `MessageThreadId`, add focused coverage for raw `im`, D-prefix DMs with missing/wrong `channel_type`, non-DM self-thread roots, and concurrent assistant-DM roots, and split the unrelated agents harness change into a separate PR if still needed.

## Risks / Open Questions

- The proposed raw `channel_type === "im"` check can miss D-prefixed Slack DMs when Slack omits or misreports `channel_type`, while current OpenClaw intentionally normalizes those events as DMs.
- Any broader `MessageThreadId` assignment for all `thread_ts === ts` events would regress channel/group behavior because Slack tool threading forces `replyToMode` to `all` when `MessageThreadId` is present.
- The branch bundles an unrelated agents overflow-compaction harness edit, increasing review scope for a Slack-only fix.
