## Summary

Keep open. Current main still does not preserve explicit Slack thread context for Agents & Assistants DM root messages when `thread_ts === ts` and `replyToMode` is `off`, `first`, or `batched`. The PR is still relevant, though its latest review context shows it likely needs revision to use normalized DM detection rather than raw `channel_type`.

## Best Possible Solution

Keep the PR open and ask for a revised Slack-owned fix: derive the DM assistant-root condition from normalized channel type or D-prefix inference, set `MessageThreadId` for DM roots with `thread_ts === ts` in non-all reply modes, and add regression tests for raw `im`, missing/wrong `channel_type` D-prefixed DMs, and channel/group `thread_ts === ts` cases that must remain unthreaded.

## Evidence

- **Current main drops MessageThreadId for DM-root non-all modes:** `resolveSlackThreadContext` treats `thread_ts === ts` without `parent_user_id` as `isThreadReply=false`, then only sets `messageThreadId` for true thread replies or `replyToMode === "all"`. That leaves DM assistant root messages without explicit thread targeting in `off`/`first`/`batched`.
