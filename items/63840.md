## Summary

Keep open. Current main still does not preserve explicit Slack Agents & Assistants DM-root thread context for non-`all` reply modes, so the PR is not obsolete. It should be revised before merge because the proposed gate uses raw `channel_type` instead of existing normalized DM detection and the branch includes an unrelated agents overflow harness edit.

## Best Possible Solution

Keep this PR open and revise it: pass normalized Slack direct-message state into `resolveSlackThreadContext` or derive it with the existing Slack channel-type normalization helper; set `MessageThreadId` only for Slack direct-message assistant root events where `thread_ts === ts`; preserve channel/group top-level behavior for `off`, `first`, and `batched`; add focused tests for `channel_type: "im"`, missing/wrong `channel_type` on D-prefix DMs, and non-DM top-level roots; remove or split the unrelated agents harness change.
