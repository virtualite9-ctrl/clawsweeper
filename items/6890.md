## Summary

Keep open. Current main still lacks a Ralph Loop implementation and any per-agent max Ralph iteration/default loop setting. The nearby OpenClaw behavior is single-run agent execution, strict-agentic planning-only retry, and optional tool-loop guardrails, none of which implements a configurable STOP-driven repeated same-prompt loop.

## Best Possible Solution

Keep the issue open. If accepted, implement an OpenClaw-native per-agent loop/default surface in AgentConfig, strict zod schemas, generated config help/labels/reference docs, and resolveAgentConfig(), then wire the embedded runner with explicit precedence against higher-priority runtime/CLI overrides. Add tests for global/default behavior, per-agent override, override precedence, disabled, finite, and unlimited semantics, and keep it distinct from strict-agentic planning-only retry and tool-loop detection unless maintainers intentionally merge those concepts.
