## Summary

Keep open. Current main at 631552c554095b0c520376db9131dac0f5427f94 still does not implement Ralph Loop or a per-agent max Ralph iteration/default loop setting. The agent config type, strict zod schema, resolver, docs, and embedded runner all lack the requested loop field and precedence path; current adjacent behavior is fixed retry/strict-agentic handling and tool-loop detection, not a STOP-driven repeated same-prompt loop. The provided issue discussion also includes a recent maintainer-visible Codex review comment keeping #6890 open for this same missing surface.

## Best Possible Solution

Keep #6890 open for maintainer product judgment. If accepted, implement an OpenClaw-native per-agent loop/default config surface in the agent config types, strict zod schemas, generated config help/labels/reference docs, and resolveAgentConfig(), then wire the embedded runner with explicit precedence against higher-priority runtime/CLI overrides. Add tests for global default, per-agent override, override precedence, disabled, finite, and unlimited semantics, and keep the feature distinct from strict-agentic retry handling and tools.loopDetection. If maintainers decide this should live outside core, first define the missing plugin/runtime API needed to express it rather than closing the request as already possible through ClawHub.
