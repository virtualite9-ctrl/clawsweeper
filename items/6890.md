## Summary

Keep open. Current main still lacks an OpenClaw-native Ralph Loop feature and still has no per-agent `loop.max_ralph_iterations` or equivalent iteration-limit config surface. The existing loop-related code is retry/guardrail behavior, not a STOP-driven repeated same-prompt loop with the requested precedence model.

## Best Possible Solution

Keep #6890 open for maintainer product judgment. If accepted, implement an OpenClaw-native loop/default config surface in `AgentConfig`, strict zod schemas, generated/help docs, and `resolveAgentConfig`, then wire the embedded runner with explicit precedence against higher-priority runtime or CLI overrides. Add focused tests for global default, per-agent override, override precedence, disabled, finite, and unlimited semantics. If maintainers decide this belongs outside core, first define the plugin/runtime API needed to express it safely.
