## Summary

Issue #53623 targets the external WeChat plugin, not code owned by this OpenClaw core checkout. Current main already exposes the generic `audioAsVoice` outbound contract to channel plugins, so the remaining work is plugin-side and belongs in the external `@tencent-weixin/openclaw-weixin` package/repo rather than `openclaw/openclaw`.

## Evidence

## Close Comment

Closing from a Codex maintainer review. This request is for the external `@tencent-weixin/openclaw-weixin` plugin, not for code owned in `openclaw/openclaw`: `docs/channels/wechat.md:10`-`docs/channels/wechat.md:43` says the WeChat runtime, media upload/download, and monitoring are owned by the external plugin, and `VISION.md:54`-`VISION.md:59` plus `docs/plugins/community.md:9`-`docs/plugins/community.md:22` say optional/community capability should live as plugins published outside core. Current main already provides the generic outbound `audioAsVoice` hook that channel plugins can consume (`src/channels/plugins/outbound.types.ts:14`, `src/infra/outbound/deliver.ts:868`, `src/infra/outbound/deliver.test.ts:603`). There is also no bundled `extensions/weixin` package in this checkout to change. The remaining work belongs in the external WeChat plugin package/repo or its ClawHub/npm maintenance path rather than OpenClaw core.
