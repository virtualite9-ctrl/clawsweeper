## Summary

Keep #71140 open. Current main still lacks parsing and outbound execution for `[[react...]]` / `[[react_to_current:...]]` reply directives, even though Telegram already has native reaction primitives and existing MEDIA/GIF handling. The 2026-04-25 maintainer comment and draft PR #71141 match the current code state: media is already handled, but reaction directives are not implemented on main.

## Best Possible Solution

Keep #71140 open. The best path is to review or replace draft PR #71141 with a focused fix that extends shared directive parsing to emit structured reaction metadata, preserves it through outbound normalization, allows valid reaction-only Telegram replies, invokes the existing `reactMessageTelegram` primitive from Telegram delivery/outbound payload paths, and adds parser plus Telegram adapter/delivery regression tests.
