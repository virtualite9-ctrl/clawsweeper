## Summary

Keep this issue open. Current `main` at `110fa97f2aa171d601d1736a5f4855d299f6fb88` already has MEDIA parsing and Telegram media/reaction primitives, but the normal outbound reply pipeline still does not parse `[[react...]]` or `[[react_to_current:...]]` into structured Telegram reaction data, does not preserve reaction intent through outbound normalization, and still rejects reaction-only Telegram replies. The provided discussion also includes a maintainer review saying the reaction directive path remains unimplemented, and draft PR #71141 is open, unmerged, authored by the same reporter, and uses GitHub closing syntax for this issue, so the issue should stay open while that PR is reviewed or replaced.

## Best Possible Solution

Keep this issue open and use draft PR #71141, or an equivalent maintainer-owned patch, as the active implementation path. The fix should extend shared directive parsing for `[[react:...]]` and `[[react_to_current:...]]`, preserve structured Telegram reaction data through outbound normalization, allow reaction-only Telegram replies when a reply target exists, execute native Telegram reactions in both reply delivery and outbound adapter paths, and add parser, normalization, delivery, and adapter tests.
