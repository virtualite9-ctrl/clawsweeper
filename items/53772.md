## Summary

Keep open. Current main implements the narrower compaction summarization model override, including local-provider model refs, but the broader request is still not implemented: pre-compaction memory flush still routes through the active run's provider/model, startup files are injected into the active run prompt, and the memory-flush config/contract has no dedicated model or provider override.

## Best Possible Solution

Keep this issue open and implement a dedicated maintenance or memory-flush model override in core: add config schema/help/docs, route `runMemoryFlushIfNeeded` through the override instead of the active run target, and add tests for provider/model parsing, auth-profile behavior, fallback behavior, local Ollama-style refs, and memory tool permissions. Treat startup-file preprocessing as a separate product decision unless maintainers explicitly extend the same maintenance-model contract to that surface.
