## Summary

Keep open. Current `main` partly addresses the report: OpenClaw has a compaction-only model override and startup reads are now a runtime-owned prelude, but the pre-compaction memory flush still runs through the session's normal provider/model path, so the core cost-saving request is not implemented end-to-end.

## Evidence

- **Compaction override is limited to summarization:** `agents.defaults.compaction.model` is documented and typed as an override used only for compaction summarization, falling back to the primary agent model when unset.
  - file: [src/config/types.agent-defaults.ts:456](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/config/types.agent-defaults.ts#L456)
- **Memory flush config has no dedicated model selector:** `AgentCompactionMemoryFlushConfig` only exposes `enabled`, thresholds, `prompt`, and `systemPrompt`; there is no `model` field for the flush turn.
  - file: [src/config/types.agent-defaults.ts:481](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/config/types.agent-defaults.ts#L481)
- **Memory flush still uses the session run model:** The flush path calls `runWithModelFallback(...resolveModelFallbackOptions(params.followupRun.run))`, then builds the embedded run from that run's provider/model. No compaction-model override is applied in this path.
  - file: [src/auto-reply/reply/agent-runner-memory.ts:753](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/auto-reply/reply/agent-runner-memory.ts#L753)
- **Model fallback options come directly from the queued run:** `resolveModelFallbackOptions()` returns `provider: run.provider` and `model: run.model`, confirming the memory-flush turn follows the session model unless some external fallback changes it.
  - file: [src/auto-reply/reply/agent-runner-utils.ts:172](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/auto-reply/reply/agent-runner-utils.ts#L172)
- **Startup reads are no longer model-decided housekeeping:** Startup context is now a runtime-owned first-turn prelude for bare `/new` and `/reset`, so the 'session startup reads' part of the issue is partly obsolete rather than missing.
  - file: [src/config/schema.help.ts:881](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/config/schema.help.ts#L881)

## Risks / Open Questions

- The issue body also asks for a broader 'other internal housekeeping tasks' model split. Current main still does not expose a general housekeeping-model abstraction.
- There is a compaction-provider plugin seam for summarization, but that does not cover the silent pre-compaction memory-flush turn described in the issue.
