## Summary

Keep open. Current main implements `agents.defaults.compaction.model` for compaction summarization, including local-model use, but the broader request is not fully implemented: pre-compaction memory flush still runs as a normal silent agent turn using the active follow-up run model/fallback path, and startup/bootstrap context is injected into the active prompt rather than routed through a separate housekeeping model. Related #43834 covers the shipped summarization-only override, but it does not supersede the remaining memory-flush/startup-context model-routing request.

## Best Possible Solution

Keep this issue open and narrow the implementation target. The best path is to extend the existing compaction model-routing design, or add a separate `maintenance`/`memoryFlush` model override, then route `runMemoryFlushIfNeeded` through that override with tests. For startup reads, document or decide explicitly whether they should remain active-prompt injection with existing size controls (`contextInjection`, `bootstrapMaxChars`, `startupContext`) or whether a new summarized-startup-context feature is desired.
