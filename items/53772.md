## Summary

Keep open. Current main implements `agents.defaults.compaction.model` for compaction summarization, including local-model use, but the broader request is still not implemented: pre-compaction memory flush still routes through the active follow-up run provider/model fallback path, startup context is injected into the active prompt rather than processed through a separate housekeeping model, and the config/schema/docs expose no dedicated maintenance or memory-flush model override.

## Best Possible Solution

Keep this issue open and narrow the implementation target. The likely core path is to add a dedicated maintenance or memory-flush model override, route `runMemoryFlushIfNeeded` through it, and add schema/docs/tests for auth-profile handling, fallback behavior, local-model context limits, and tool/workspace capability expectations. Startup reads should be handled as a separate product decision: either keep them as bounded prompt injection or define a distinct summarized-startup-context feature.
