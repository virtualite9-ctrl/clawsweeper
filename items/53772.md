## Summary

Keep open. Current main has the shipped `agents.defaults.compaction.model` override for compaction summarization, including local-provider model strings, but the broader request is not implemented: pre-compaction memory flush still uses the active run's model fallback routing, startup files are injected/read as prompt context rather than processed by a dedicated housekeeping model, and the memory-flush config/schema has no model or provider override.

## Best Possible Solution

Keep this issue open and narrow the implementation target. The concrete core change should add a dedicated maintenance or memory-flush model override, route `runMemoryFlushIfNeeded` through it, and cover config schema/help/docs plus tests for provider/model parsing, auth-profile behavior, fallbacks, local Ollama-style targets, and memory tool permissions. Startup-file preprocessing should be a separate explicit product decision unless maintainers choose to extend the same maintenance-model contract to it.
