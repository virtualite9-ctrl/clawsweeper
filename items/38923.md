## Summary

Current main already supports plain workspace `IDENTITY.md` avatar fallback, but it still does not strip inline-code backticks from avatar values. The PR is stale/noisy, but its central parser/test work is not fully obsolete, so conservative cleanup should keep it open or move the remaining gap to a narrower canonical issue before closing the PR.

## Best Possible Solution

Keep this PR or #38439 open until current main handles inline-code/backtick-wrapped `IDENTITY.md` avatar values, or until maintainers create a narrower canonical issue for that parser gap. The best code path is a rebased, narrow patch in `src/agents/identity-file.ts` plus focused tests in `src/agents/identity-file.test.ts` and `src/agents/identity-avatar.test.ts`, preserving current max-size, workspace containment, supported-extension, and `ui.assistant.avatar` precedence behavior while dropping unrelated stale edits.
