## Summary

Current main already includes a shipped fix for async exec completion replies being dropped from the originating chat route. The reported symptom matches the old behavior: exec first returned a backgrounded "Command still running" result, the system emitted `Exec completed ... :: hello`, but the completion reply did not make it back to webchat. Main now resolves exec-event replies through persisted session delivery context, including webchat-owned sessions, and that shipped in v2026.4.23.
