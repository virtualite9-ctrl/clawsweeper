## Summary

Keep this issue open. Current main still has several separate observability/state surfaces rather than one canonical run trace schema adopted across agent runs, ACP relay, subagent/task records, and Task Flow. The related #43145 is open and unmerged in the provided context, so it is useful prior art but not shipped behavior on main.

## Best Possible Solution

Keep this open as an architecture/observability tracking issue. The best path is to define a small versioned run-trace contract, decide its relationship to trajectory bundles and AgentEventPayload, then adapt agent lifecycle/item/tool events, ACP parent-child relay, subagent lifecycle, task records, and Task Flow state transitions into that schema with focused tests and any needed gateway/API exposure. Review #43145 as prior art or a candidate implementation, but close only after equivalent cross-flow coverage is merged on main.
