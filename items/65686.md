## Summary

Keep this issue open. Current main has partial mitigations for silent jobs and final-text collapsing on some channel plugins, but it still does not implement the requested configurable cron outbound payload policy (`full` / `last` / `none`). The explicit related PR was closed unmerged, and the cron resolver/dispatch path still preserves multiple deliverable payloads for channels without the plugin final-text preference.

## Best Possible Solution

Keep this issue open and implement a cron-owned outbound payload policy, likely in the isolated cron resolver/dispatch path. Add typed config/schema/help/docs plus tests for default full behavior, last-only selection after normal resolution, and none/suppressed automatic delivery with no synthesized-text fallback. The design should account for existing `delivery.mode="none"` and channel plugin `preferFinalAssistantVisibleText` behavior rather than replacing them accidentally.
