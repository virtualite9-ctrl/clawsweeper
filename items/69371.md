## Summary

Keep open. Current main has several adjacent mitigations, but the central ARM64 edge-device stability request is not implemented or cleanly superseded: sessions/status still lack the requested lightweight/edge path, compaction lacks the proposed high-water config, message send lacks CLI background/retry flags, sessions_send lacks a durable receipt, and doctor has only generic ARM/low-memory hints rather than an edge-device preflight.

## Best Possible Solution

Keep #69371 open as a recent ARM64 edge-device stability umbrella unless maintainers split it into narrower canonical issues. The likely core path is lightweight sessions/status output for large stores, explicit context-pressure compaction tuning or clearer reserve-token semantics, per-job cron retry behavior if global retry is insufficient, durable/background CLI message delivery semantics, a sessions_send receipt contract, and a doctor edge preflight that builds on the existing ARM/low-memory startup hints.
