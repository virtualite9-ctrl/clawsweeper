## Summary

Keep #69271 open. Current main has bulk deletion for selected Sessions rows and the backend archives transcript files as part of delete, but the Sessions table still has no per-row delete/archive action and there is no recoverable archive/list/filter API for hiding sessions from the active list while preserving them. The provided related title-search items are not canonical duplicates for this Control UI Sessions request.

## Best Possible Solution

Keep the issue open. A narrow implementation could add a per-row delete icon/dropdown in ui/src/ui/views/sessions.ts that reuses deleteSessionsAndRefresh(state, [key]). The archive part likely needs separate product/API work: either define a real sessions.archive plus archived listing/filter/restore flow, or split/archive-scope it into a follow-up if the immediate row-delete UX is implemented first.
