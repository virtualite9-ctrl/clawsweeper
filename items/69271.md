## Summary

Keep #69271 open. Current main still exposes Control UI session deletion through checkbox selection plus the bulk delete bar, not a per-session row action or actions dropdown. The backend `sessions.delete` archives transcript files as part of deletion, but there is no separate `sessions.archive` method, archived-session state, or Sessions-view filter/recovery surface for the requested archive workflow.

## Best Possible Solution

Keep the issue open. Implement the narrow UX first by adding a per-row Actions control in `ui/src/ui/views/sessions.ts` and wiring it to the existing delete flow for a single key. Treat archive as a separate follow-up unless maintainers want to define archived session state, gateway API/list filters, and recovery semantics beyond the current transcript-archive side effect of `sessions.delete`.
