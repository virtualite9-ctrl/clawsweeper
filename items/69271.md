## Summary

Keep #69271 open. Current main still only exposes Control UI session deletion through row checkboxes plus the selected-items bulk delete bar, and the gateway contract has `sessions.delete` but no separate archive action/filter/recovery surface. The existing delete does archive transcript files as part of deletion, but it removes the session entry and does not satisfy the requested per-row quick action or safer archive workflow.

## Best Possible Solution

Keep the issue open for product/UI work. The likely implementation belongs in the Control UI Sessions view as a stable per-row actions column or menu that reuses `sessions.delete` for Delete, clears the selected set consistently, and adds focused view/controller tests. The Archive portion needs explicit product/API design because current `sessions.delete` archives transcript files only as a deletion side effect; a true archive workflow would need a gateway/session-store state model plus filtered archived-session visibility and recovery semantics.
