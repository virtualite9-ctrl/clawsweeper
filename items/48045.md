## Summary

Current main partially improves browser-download handling, but it does not implement the issue's direct navigate-to-attachment behavior. The `/navigate` path still returns only URL/target information, `page.goto()` errors other than the narrow retry cases are rethrown, and there is no `download.triggered`/navigation-download response surface or regression coverage for a Content-Disposition navigation download. The linked fix PR #48132 is closed and unmerged in the provided GitHub context, so this issue should stay open.

## Best Possible Solution

Keep the issue open and implement navigation-triggered download handling in the browser plugin: race guarded navigation with the page download event, treat Playwright's expected download-start navigation error as success when a download is observed, save through the same guarded downloads directory, return explicit download metadata from `/navigate`, and add regression coverage for a CDP Content-Disposition URL such as the reporter's httpbin example. Preserve the existing explicit `wait/download` ownership semantics.
