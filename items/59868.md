## Summary

Keep open. Current main has useful edit-tool recovery, but it does not cover this issue's specific already-fixed/manual-fix path: the wrapper deliberately refuses to convert an unchanged already-correct file into success, and the runner still preserves mutating edit failures unless a later successful matching mutation clears the same action. Related #54635/#60664 remain open context, not shipped superseding fixes.

## Best Possible Solution

Keep this open and handle it with the related edit-warning work rather than closing against the shipped post-write recovery. The likely maintainer path is to add focused tests for an edit oldText mismatch followed by an externally successful/manual correction, then adjust either `lastToolError` clearing or payload warning policy so stale edit failures are suppressed only when current file state proves the requested outcome is already satisfied. Coordinate with open #54635 and PR #60664, but do not treat them as shipped canonical fixes yet.
