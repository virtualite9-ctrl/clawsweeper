## Summary

Current `main` already fixes the macOS LaunchAgent restart behavior without changing the plist to `SuccessfulExit=false`: the LaunchAgent still intentionally uses `KeepAlive=true`, but stop now disables relaunch first and restart uses `launchctl kickstart -k` with bootstrap recovery. The issue was reported against `2026.3.7`, and the changelog shows the restart fix shipped in `v2026.3.11`.

## Evidence

- **LaunchAgent still intentionally uses KeepAlive=true:** The generated macOS LaunchAgent plist still renders `<key>KeepAlive</key><true/>` and does not use `SuccessfulExit=false`, so current `main` addresses the problem through service-control logic rather than by changing plist semantics.
