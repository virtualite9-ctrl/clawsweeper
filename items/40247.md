## Summary

Keep this PR open. Current main still does not implement the PR's server-side dual-bind behavior: specific non-loopback gateway binds return only the selected bind host, while local/internal gateway callers still target loopback. The related discussion shows this PR is a focused implementation candidate, not obsolete; the overlapping client-side alternative #50751 is still open and unmerged in the provided context.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to either land a current-main version of this server-side dual-bind change with the PR's tailnet/custom regression tests and matching docs updates, or explicitly choose the #50751 client-side host-resolution approach and close this PR only after the shipped path demonstrably fixes internal calls, CLI probes, and `sessions_spawn` for `gateway.bind=tailnet` and `gateway.bind=custom`.
