## Summary

Keep #40247 open. Current main still does not bind loopback alongside tailnet/custom specific-IP gateway binds, while local/internal gateway callers still target 127.0.0.1. The PR remains a focused server-side fix with relevant regression tests, and the related client-side alternative #50751 is open in the provided context rather than shipped or clearly superseding this approach.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is either to land a current-main version of this server-side dual-bind change with the PR's tailnet/custom regression tests and docs updates, or explicitly choose the #50751 client-side host-resolution approach and close #40247 only after the shipped path demonstrably fixes internal calls, CLI probes, and sessions_spawn for gateway.bind=tailnet and gateway.bind=custom.
