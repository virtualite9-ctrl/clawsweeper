## Summary

Keep open. Current `main` still treats Feishu `@_all` as an explicit bot mention, so the PR addresses a real unfixed bug rather than stale or already-landed work.

## Evidence

- **Current Feishu mention parser still special-cases `@_all` as a bot mention:** `checkBotMentioned()` returns `true` whenever `event.message.content` contains `@_all`, before checking actual mention metadata or rich-text mentions.
  - file: [extensions/feishu/src/bot-content.ts:229](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/extensions/feishu/src/bot-content.ts#L229)
  - command: `nl -ba extensions/feishu/src/bot-content.ts | sed -n '220,280p'`
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **That flag is used to bypass group `requireMention` gating:** Group message handling drops messages only when `requireMention && !ctx.mentionedBot`; a false-positive `mentionedBot` therefore lets a broadcast message through mention gating.
  - file: [extensions/feishu/src/bot.ts:466](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/extensions/feishu/src/bot.ts#L466)
  - command: `nl -ba extensions/feishu/src/bot.ts | sed -n '462,482p'`
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **Current tests do not cover the reported regression:** The helper serializes text into `event.message.content` via `JSON.stringify({ text })`, and the existing `mentionedBot` tests cover normal mentions but do not include an `@_all` negative case on current `main`. A repo search for `@_all` in `extensions/feishu/src` only finds the parser special-case.
  - file: [extensions/feishu/src/bot.checkBotMentioned.test.ts:5](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/extensions/feishu/src/bot.checkBotMentioned.test.ts#L5)
