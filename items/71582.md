## Summary

Keep this PR open. Current main still has the old exec-event heartbeat wake shape: agent and node exec completions enqueue only scoped `reason: "exec-event"` wakes, the wake API can only carry `heartbeat.target`, and scheduler merging still preserves configured pinned heartbeat fields unless a caller explicitly clears them. The PR remains a plausible implementation candidate rather than cleanup-close material.

## Best Possible Solution

Keep this PR open. The best path is to review the latest head, preserve the shared `scopedExecEventWakeOptions` direction, verify the pinned `target`/`to`/`accountId`/`isolatedSession` and canonical `global` regression coverage, then run the changed gates for agents, gateway, infra heartbeat, cron, routing, and outbound delivery before merging or requesting changes.
