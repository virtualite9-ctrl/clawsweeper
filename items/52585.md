## Summary

Keep open. Current `main` still takes npm plugin installs through tar extraction and a Python-backed safe-write path, and non-`SafeOpenError` failures from that helper are still collapsed into `SafeOpenError: path is not a regular file under root`, which matches the report. I did not find strong evidence on `main` that this issue is fixed or obsolete.
