## Summary

Keep PR #72216 open. Current main still contains the Nostr setup-status delegation path described by the PR, so this is not obsolete or implemented on main. The linked issue #72182 being closed is not enough to close the PR, because the current source still shows the same root cause and the PR’s proposed regression test is not present on main.

## Best Possible Solution

Keep the PR open and refine it rather than auto-close it. The best maintainer path is to land the lightweight `ChannelSetupWizard` status implementation in `extensions/nostr/src/channel.setup.ts`, keep `credentials: []`, avoid delegating status to `setup-surface.js`, and keep the regression test that proves status resolution does not load the full Nostr setup surface. Address the minor relay-count test brittleness before merge.
