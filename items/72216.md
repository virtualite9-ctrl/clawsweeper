## Summary

Keep PR #72216 open. Current main still has the Nostr setup-status path the PR targets: the setup-only Nostr plugin uses the delegated setup wizard proxy, and that proxy delegates status resolution into the full setup surface, which imports `nostr-tools` through `nostr-key-utils`. The PR's equivalent fix and regression test are not present on main.

## Best Possible Solution

Keep PR #72216 open and land it, or apply an equivalent main change, so `extensions/nostr/src/channel.setup.ts` keeps setup status on the lightweight setup-only surface while preserving the full Nostr wizard for interactive configuration. Add or keep the regression test that makes `./setup-surface.js` throw during status checks, and address the small hardcoded relay-count test nit before merge if practical.
