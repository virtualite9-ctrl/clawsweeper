## Summary

Keep open. Current main partially addresses the audit/edit trust gap through Agents > Files, where USER.md and MEMORY.md can be loaded, previewed, edited, and saved. It does not fully implement the requested Memory-specific surface because the UI is still a generic Files panel and the backend allowlist/listing does not expose today's memory/YYYY-MM-DD.md daily log.

## Best Possible Solution

Keep the issue open and implement this as an extension of the existing Agents > Files path or as a renamed Memory-focused panel. The remaining product work is to expose today's memory/YYYY-MM-DD.md safely, present MEMORY.md/USER.md/daily notes as a memory audit surface, and add focused UI/backend tests for listing, reading, creating, and saving the daily log without widening arbitrary workspace file access.
