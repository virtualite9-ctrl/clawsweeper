## Summary

Keep #68981 open. Current main has useful partial coverage: session labels are persisted and shown in Control UI, `sessions_spawn` supports `cleanup: "delete"`, and the chat session selector is no longer just raw UUIDs in every case. But the full enhancement is not implemented: there is no `/session rename` or auto-title flow, default/keep-mode subagent runs are explicitly not archived, and Control UI does not expose the requested channel/kind/status filters for the session selector.

## Best Possible Solution

Keep the issue open, but narrow it around the remaining product gaps. Build on the existing `sessions.patch` label field instead of renaming store keys, add a user-facing `/session label` or `/session rename` wrapper if desired, reconcile `archiveAfterMinutes` versus `cleanup: "keep"` semantics, and add gateway-backed Control UI filters for channel/kind/status plus selector ergonomics. If maintainers want conservative cleanup, split those three tracks into smaller issues and close this umbrella only after canonical follow-ups exist.
