## Summary

Keep #68981 open. Current main partially addresses the report with persisted session labels, Control UI label display/editing, and opt-in `sessions_spawn` cleanup via `cleanup: "delete"`. The central enhancement is not fully implemented: there is no `/session rename` or auto-title workflow, default subagent runs still use keep-mode and do not get an archive deadline, and the Control UI session selector/Sessions view do not provide the requested channel/kind/status filtering or grouping.

## Best Possible Solution

Keep the issue open, but narrow the maintainer path. Build on the shipped `sessions.patch` label field rather than renaming store keys; add a user-facing `/session label` or `/session rename` wrapper and optional auto-title only if product wants it. Separately decide the default subagent retention semantics versus opt-in `cleanup: "delete"`, then add gateway-backed channel/kind/status filters and Control UI selector grouping/indicators.

## Risks / Open Questions

- The issue is broad and now mixes labels, cleanup policy, and Control UI filtering; maintainers may want to split it into narrower canonical follow-ups before implementation.
- Changing default subagent cleanup from keep to delete could surprise users by removing session rows/transcripts earlier than expected, so that part needs explicit product/docs judgment.
- Current docs and comments around `archiveAfterMinutes` versus default keep-mode behavior may remain confusing until the retention contract is clarified.
