## Summary

Keep #69748 open. Current main still has Discord `requireMention` as a guild/channel boolean plus `users`/`roles` sender allowlists. It does not implement the proposed `deny`/`bypass`/`mention` access tiers, permission shortcuts, additive per-channel overrides with `inherit: false`, or sender-specific no-mention bypass semantics.

## Best Possible Solution

Keep the issue open for bundled Discord plugin work. The best path is a typed Discord config migration that adds `deny`, `bypass`, and `mention` access lists at guild and channel scope, defines inheritance/union semantics including `inherit: false`, supports role IDs and a small audited set of permission aliases from Discord member/permission data, preserves legacy `requireMention`/`users` compatibility through doctor or migration warnings, and adds schema, docs, and preflight tests for precedence and per-channel overrides.
