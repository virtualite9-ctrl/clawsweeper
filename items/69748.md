## Summary

Keep #69748 open. Current main still uses Discord guild/channel `requireMention` plus `users`/`roles` sender allowlists. It has role allowlist support and per-channel booleans, but not the requested `bypass`/`mention`/`deny` access-policy model, deny-first precedence, permission shortcuts, or guild/channel list merge and `inherit: false` semantics.

## Best Possible Solution

Keep this open as a bundled Discord plugin enhancement. The best path is an additive, backward-compatible access-policy schema with `bypass`, `mention`, and `deny` lists; selected `perm:*` matchers; documented guild/channel merge and `inherit: false` rules; migration or doctor guidance for existing `requireMention`/`users` configs; and focused runtime tests for deny-first precedence and per-sender mention requirements.
