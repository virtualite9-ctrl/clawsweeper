## Summary

Keep #69748 open. Current main has useful Discord pieces already, including guild/channel `requireMention` plus `users` and `roles` sender allowlists, but it does not implement the requested bypass/mention/deny access-list model, permission shortcuts, deny precedence, sender-specific mention behavior, or additive channel inheritance semantics. Related #39661 overlaps with per-user mention policy, but #69748 is broader and should not be closed as a duplicate from the available evidence.

## Best Possible Solution

Keep the issue open for bundled Discord plugin work. A solid implementation would add a typed Discord access-policy resolver and config schema for `deny`, `bypass`, and `mention` entries, support user IDs, role IDs, selected permission shortcuts, and explicit channel inheritance/merge semantics, then preserve legacy `requireMention`/`users` behavior through a documented compatibility path with startup/doctor migration guidance. It should update Discord docs and add preflight tests covering deny precedence, bypass without mention, mention-only users, role and permission matches, channel union versus `inherit: false`, and old-config behavior.
