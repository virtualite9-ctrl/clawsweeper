## Summary

Keep open. Current main still has the misclassification path this PR targets: the classifier checks the loose missing-systemctl matcher before the more specific user D-Bus matcher, and the downstream systemd availability assertion still throws the misleading `systemctl not available` message from `isSystemctlMissing` rather than branching on the shared classifier. The PR is a focused bug-fix candidate for the linked WSL2 report #68380, with regression-test intent documented and no observed security-sensitive file changes.

## Best Possible Solution

Keep this PR open as an implementation candidate. The best maintainer path is to rebase it onto current main, preserve the recent sudo user-scope behavior, keep the classifier and WSL-specific regression tests, fix the changelog attribution, then run the daemon changed-gate before landing.
