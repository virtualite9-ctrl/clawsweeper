## Summary

Keep #39223 open. Current main still routes `openclaw configure` and `openclaw configure --section model` through the mandatory Gateway-location Clack select before model/auth setup, and the configure command still has no local/non-interactive/auth-only bypass. Separate `openclaw models auth ...` commands remain valid documented workarounds, but they do not fix the reported configure wizard path.

## Best Possible Solution

Keep the issue open until the configure path itself is robust. A good fix would add TTY/non-interactive detection with actionable direct-auth guidance before the Clack select can hang, or add a model/auth-only configure path or flag that skips Gateway-location selection when the user only needs auth setup. Keep `openclaw models auth login/setup-token/paste-token` documented as workarounds, not as the closing fix for this issue.
