## Summary

Current main still lacks a managed macOS LaunchDaemon/system-scope Gateway mode. The shipped macOS Gateway service path remains per-user LaunchAgent only, while LaunchDaemon support appears only as external-service detection in `gateway status --deep` and as docs/comments saying custom LaunchDaemons are not shipped.

## Best Possible Solution

Keep this issue open for explicit macOS Gateway lifecycle design. The likely path is an opt-in system-scope LaunchDaemon mode across `openclaw gateway install/start/stop/restart/status/uninstall`, macOS app integration, doctor/repair, docs, and regression tests, while preserving the current per-user LaunchAgent default.
