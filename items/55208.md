## Summary

Current main still does not implement the requested managed macOS LaunchDaemon/system-scope Gateway mode. The Gateway service surface remains LaunchAgent-only on Darwin, with no CLI option, typed install field, LaunchDaemon adapter, UserName plist rendering, or macOS app path to request a system LaunchDaemon. Existing LaunchDaemon mentions are either custom/not-shipped guidance or status-only discovery of externally managed services.

## Best Possible Solution

Keep this issue open for explicit macOS Gateway lifecycle design. The likely product path is an opt-in system-scope LaunchDaemon mode across `openclaw gateway install/start/stop/restart/status/uninstall`, macOS app integration, doctor/repair, docs, and regression tests, while preserving the current per-user LaunchAgent default.
