## Summary

Current main still does not implement the requested macOS LaunchDaemon/system-scope Gateway mode. The managed macOS service path remains LaunchAgent-only, the CLI has no `--daemon`/scope option, the macOS app calls the same `openclaw gateway install --force` path, and docs still describe LaunchDaemons only as custom/not shipped or status-discovery targets.

## Best Possible Solution

Keep this issue open for maintainer product/security review. The likely implementation is an explicit macOS system-scope service mode on the managed Gateway lifecycle surface, probably `openclaw gateway install` plus matching start/stop/restart/status/doctor/docs/tests, while preserving the per-user LaunchAgent as the default.
