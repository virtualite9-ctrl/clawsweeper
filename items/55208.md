## Summary

Keep open. Current main does not implement the requested macOS LaunchDaemon mode, and the request is a plausible core gateway/service-management feature rather than a plugin/ClawHub item. The managed macOS path remains LaunchAgent-only: CLI install/start has no daemon/scope option, Darwin resolves to the LaunchAgent service implementation, the macOS app delegates to `openclaw gateway install --force`, and docs explicitly say custom LaunchDaemon is not shipped.

## Best Possible Solution

Keep this issue open for maintainer product/implementation judgment. The best code path would be an explicit macOS system-scope service mode, likely on `openclaw gateway install` rather than only `start`, with a LaunchDaemon implementation under the daemon service abstraction, `/Library/LaunchDaemons` plist handling, `UserName`/environment/config-path semantics, sudo/privileged install and uninstall flow, `launchctl system/...` lifecycle support, status/doctor/docs coverage, and tests that keep the current per-user LaunchAgent as the default.
