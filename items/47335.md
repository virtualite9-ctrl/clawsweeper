## Summary

Keep open. Current main still matches the reported race: the runner flushes block replies before waiting for compaction retry, but there is no post-wait flush; `handleAgentEnd` resolves the compaction retry wait before awaiting the async channel flush; and existing tests explicitly lock in that ordering. Final payload assembly still drops non-error final payloads after any successful block stream, so an in-flight retry flush can still become a generated-but-undelivered reply. The exact fix PR #47338 was closed unmerged, and the related #62157 is broader/open rather than shipped on main.

## Best Possible Solution

Keep #47335 open until main changes the compaction retry completion contract. The cleanest maintainer fix is to ensure the retry's block reply/channel flush is awaited before `waitForCompactionRetryWithAggregateTimeout` can resolve, or to add an idempotent post-wait flush in `attempt.ts`, with a regression test that simulates `willRetry=true`, an async `onBlockReplyFlush`, and final-payload suppression.
