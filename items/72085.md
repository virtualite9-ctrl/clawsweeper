## Summary

Keep PR #72085 open. The underlying docs gap is real enough to be actionable because current user-facing command docs do not state the 0-30000 ms bound for `commands.bashForegroundMs`, while source and schema both enforce that range. The submitted patch is not merge-ready, though: the PR diff adds 2699 lines into `docs/gateway/configuration-reference.md`, apparently duplicating large config sections to add one bullet, and its “silently clamped” wording should be reconciled with config schema validation.

## Best Possible Solution

Keep the item open, but ask for a narrow docs cleanup: add the range/default wording to the current command docs surface (`docs/tools/slash-commands.md` and/or `docs/gateway/config-channels.md`) and consider updating `src/config/schema.help.ts` so `config.schema.lookup` explains the bound. The patch should be reduced to the intended few lines and avoid duplicating `docs/gateway/configuration-reference.md`.
