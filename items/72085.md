## Summary

Keep PR #72085 open. The docs gap is still real on current main: runtime clamps `commands.bashForegroundMs` to 0..30000 ms and the config schema exposes that bound, but user-facing slash-command/channel docs and schema help still omit the upper bound. The PR is useful in intent, and Greptile correctly verified the source claim, but the submitted diff is not merge-ready because it inserts a large duplicated configuration-reference block for a one-line docs fix.

## Best Possible Solution

Keep the PR open and ask for a narrow docs cleanup: document `commands.bashForegroundMs` as 0..30000 ms with default 2000 in `docs/tools/slash-commands.md` and/or `docs/gateway/config-channels.md`, and update `src/config/schema.help.ts` plus generated schema docs if `config.schema.lookup` should expose the bound. Remove the oversized `docs/gateway/configuration-reference.md` duplication from the PR.
