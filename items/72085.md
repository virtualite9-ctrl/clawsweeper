## Summary

Keep PR #72085 open. The underlying docs gap still exists on current main: runtime and schema both bound `commands.bashForegroundMs` to `0..30000`, but the human command docs only describe the default and `0` behavior. The provided PR metadata also shows the branch currently has no changed files, so it cannot be closed as implemented or merged as-is.

## Best Possible Solution

Keep the item open and ask for a real docs patch against current main. The best fix is to document `commands.bashForegroundMs` as `0..30000` ms in the active user-facing command config docs, especially `docs/tools/slash-commands.md` and probably `docs/gateway/config-channels.md`, with wording that matches both the zod schema bound and the runtime clamp. Avoid targeting only `docs/gateway/configuration-reference.md` because current main delegates command catalog/config detail to the command-specific docs.
