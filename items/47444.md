## Summary

Keep open. Current main has partial assistant-visible scaffolding cleanup, but the exact reported workflow envelope is not fully covered: `<function_calls>` is only stripped for JSON-like aggregate payloads, a test currently preserves `<function_calls><invoke ...></invoke></function_calls>`, and there is no `function_response` sanitizer match in the relevant visible-output paths.

## Best Possible Solution

Keep #47444 open and implement a narrow shared assistant-visible sanitizer update for workflow XML envelopes: cover `<function_calls>` blocks whose payload begins with `<invoke>` or similar tool XML, decide whether `<function_response>` blocks should be stripped or summarized, and add regression tests for the exact issue sample plus code/prose preservation cases. Apply it in the shared visible-text path used by workflow and channel delivery rather than adding a Telegram-only workaround.
