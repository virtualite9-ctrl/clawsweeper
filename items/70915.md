## Summary

Keep PR #70915 open. Current main does not implement the ZAI vision/coding-endpoint routing fix: the bundled ZAI media-understanding provider still delegates directly to generic image helpers, ZAI onboarding supports coding base URLs, the ZAI catalog marks the reported vision models as image-capable, and media-understanding model resolution carries the configured provider baseUrl into the final image `complete()` call. The linked issue has a concrete reproduction, and the PR is focused, though the request-scoped cfg rewrite still needs maintainer review because a review comment identified a possible models.json race.

## Best Possible Solution

Keep the PR/bug path open. The best fix is a race-free ZAI-owned routing change that sends image-capable ZAI vision models (`glm-4.6v`, `glm-4.5v`, `glm-5v-turbo`) to the standard ZAI endpoint when the configured provider endpoint is a coding endpoint, ideally at resolved-model/provider transport time rather than by cloning request cfg. Add regression coverage for `zai-coding-*` plus those vision models and keep #70886 linked as the user-facing bug.
