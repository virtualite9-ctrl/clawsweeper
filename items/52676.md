## Summary

Current main already implements WebChat streaming. Gateway emits `chat` events with `state: "delta"`, the browser Control UI/WebChat client updates live `chatStream` from those deltas, the macOS/iOS WebChat client updates `streamingAssistantText` from assistant events, and docs/tests describe and cover the streamed behavior.

## Close Comment

Closing as implemented on current `main` after Codex review. WebChat streaming is already wired through Gateway `chat` delta events (`src/gateway/protocol/schema/logs-chat.ts:71`, `src/gateway/server-chat.ts:732`), the browser WebChat/Control UI client updates live stream state from those deltas (`ui/src/ui/controllers/chat.ts:451`), and the native WebChat client renders streamed assistant text before finalization (`apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatViewModel.swift:1024`). The docs also describe `chat.send` responses streaming via `chat` events (`docs/web/control-ui.md:152`). If you still see block-at-once replies on a current build, please open a new issue with the exact OpenClaw version, whether this is browser Control UI vs macOS/iOS WebChat, and a diagnostics export or reproduction trace so maintainers can treat it as a concrete regression.
