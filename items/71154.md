## Summary

Keep this PR open. Current main still rejects POSIX backslash-newline commands before allowlist analysis and still reports parse/analysis failure as allowlist miss. The PR remains a relevant implementation candidate for the user-visible bug from #71153, with maintainer review still needed for the security-sensitive durable-approval behavior and changelog polish before merge.

## Best Possible Solution

Keep this PR open and review it as the active implementation candidate. The best path is to land an adapted version that normalizes POSIX `\<newline>` before allowlist analysis with quote/heredoc/comment-aware parsing, emits a distinct `unsupported-shell-syntax` denial through gateway and node-host paths, hardens or explicitly defines durable-approval behavior for unparseable commands, and includes focused regression tests plus a properly attributed changelog entry.
