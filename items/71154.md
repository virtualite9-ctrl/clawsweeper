## Summary

Keep PR #71154 open. Current main still has the exact behavior the PR targets: exec allowlist analysis hard-rejects any POSIX backslash-newline before command analysis, gateway/node-host denial paths still collapse analysis failures into `allowlist miss`, and current tests explicitly assert that shell line continuations fail analysis. The related #71225 PR is closed and unmerged, so it does not supersede this work.

## Best Possible Solution

Keep the PR open and continue review. The best path is to land this PR or an equivalent focused patch that normalizes POSIX `\<newline>` before allowlist analysis, adds the distinct unsupported-shell-syntax denial signal, and includes regression tests for allowlisted multiline commands plus malformed/unparseable syntax, after resolving the durable-approval edge called out in review.
