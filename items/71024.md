## Summary

Keep PR #71024 open. Current main still lacks the Mattermost read-only account inspector that the PR proposes, and the linked bug remains plausible from the current status path: read-only status falls back to strict Mattermost account resolution, which throws on unresolved SecretRef bot tokens. The linked issue has a concrete repro and stack trace, and the PR is a focused fix with targeted regression tests rather than obsolete or duplicate work.

## Best Possible Solution

Keep the PR open and review it as the canonical fix for #70942. The best implementation path is the PR's approach: add a Mattermost `inspectAccount` path that reports unresolved SecretRef tokens as `configured_unavailable` without dereferencing them, wire it through `mattermostConfigAdapter`, preserve `botTokenStatus` in status snapshots, keep runtime `resolveMattermostAccount` strict, and land the focused Mattermost regression tests.
