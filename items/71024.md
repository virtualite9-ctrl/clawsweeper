## Summary

Keep PR #71024 open. Current main still lacks a Mattermost read-only account inspector, and the read-only status path can still fall back into strict Mattermost account resolution that throws on unresolved SecretRef bot tokens. The PR addresses a real current bug tracked by #70942, though the existing bot review edge case should be handled before merge.

## Best Possible Solution

Keep the PR open and finish the focused Mattermost fix: add a read-only inspectMattermostAccount path, wire it through mattermostConfigAdapter.inspectAccount, preserve configured/botTokenStatus in status snapshots, keep runtime resolveMattermostAccount strict, and include regression coverage for unresolved SecretRef plus the env-token fallback priority edge case.
