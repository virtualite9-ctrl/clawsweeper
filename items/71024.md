## Summary

Keep PR #71024 open. Current main still lacks Mattermost read-only account inspection, so the reported `openclaw status` SecretRef crash path is still plausible on main, while the PR directly targets that gap with implementation and regression tests. This is not conservative cleanup material.

## Best Possible Solution

Keep the PR open for maintainer review or merge after CI. The best path is to land a scoped Mattermost read-only inspector, wire it through `mattermostConfigAdapter` and the bundled `accountInspect` surface if needed, preserve strict runtime token resolution, and keep the PR's regression coverage for unresolved SecretRef status snapshots.
