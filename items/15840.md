## Summary

Keep #15840 open. Current main still synthesizes visible media placeholder text for blank audio/media sends in Signal and iMessage, and the shared `messages` config contract has no `suppressMediaPlaceholders` or granular `mediaPlaceholders` option. The related implementation PRs in the supplied context were closed unmerged, and the recent maintainer Codex comment already identifies this as the canonical product/config request.

## Best Possible Solution

Keep #15840 open as the canonical cross-channel media placeholder suppression request. A conservative implementation should add a documented default-off `messages` config option, update strict schema/types/generated metadata/docs, thread the setting through Signal and iMessage outbound media paths that synthesize placeholders, preserve the current accessibility/fallback default, and add focused tests for opt-in audio/media-only sends without visible `<media:audio>`.
