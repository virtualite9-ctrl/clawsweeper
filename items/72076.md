## Summary

Keep #72076 open. It is a useful computer-use/plugin contribution with active maintainer discussion, but current main does not have the generic computer-use SDK/security/permissions model that vincentkoc says is needed. The related Codex Computer Use work is open, maintainer-owned, and Codex-specific, so it does not supersede this cross-harness cua-driver proposal. The PR also has concrete unresolved review findings, so it should not merge as-is.

## Best Possible Solution

Keep the PR open for maintainer-guided design, but do not merge it as-is. The best path is to turn the cua-driver work and #1946 prior art into input for a computer-use extension SDK with explicit security and permission semantics, then either revise this PR against that API or supersede it with a canonical maintainer-tracked implementation. #71842 can remain the Codex-specific setup path, but it should not be treated as covering this cross-harness request.
