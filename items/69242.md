## Summary

Keep #69242 open. Current main has a Linux child OOM victim-selection mitigation, but the related merged PR explicitly scoped #69242 as separate SIGKILL observability work, and the current exec path still collapses externally signaled failures to a generic `Command aborted by signal SIGKILL` without a resource, timeout, supervisor, log-span, or broad-scan guardrail explanation.

## Best Possible Solution

Keep this open as Linux exec SIGKILL observability/guardrail work. The likely core fix belongs around the process supervisor and exec result formatting: preserve stdout/stderr tail, report run id/log span, distinguish OpenClaw timeout/no-output/manual cancellation from external signal where knowable, surface timeout budgets, and consider safer warnings/limits for broad recursive scans over known-large trees. If maintainers want one canonical tracker, link or create a broader cross-platform exec SIGKILL diagnostics issue before closing this Linux-specific report as superseded.
