## Summary

Keep #69242 open. Current main includes the related Linux child OOM-victim mitigation from #70419, but that PR explicitly treated #69242 as separate SIGKILL observability work. The present exec path still reports signal exits only generically and does not diagnose broad find/grep resource termination or preserve complete machine-readable signal context across foreground/poll results.

## Best Possible Solution

Keep the issue open and implement a focused exec SIGKILL observability fix: preserve `exitSignal` in foreground and poll result details, add Linux-specific guidance for externally killed `SIGKILL` cases, and where feasible capture cgroup/systemd/kernel memory-pressure hints without pretending every SIGKILL is OOM. Add unit coverage for signal exits plus a docs note under the public exec/Linux troubleshooting path.
