## Summary

PR #71329 is still relevant. Current `main` keeps Slack lifecycle status reactions coupled to `shouldAckReaction()`, so `messages.statusReactions.enabled: true` does not independently activate the controller when the ack reaction path is disabled by scope or an empty ack reaction value. The PR targets that remaining runtime gap and adds focused tests, with one small test assertion gap already called out in review.

## Best Possible Solution

Keep PR #71329 open. The best maintainer path is to land the small Slack prepare-time gate change after adding the missing fallback ack assertion, then run the Slack extension changed lane for validation.
