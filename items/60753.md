## Summary

Current `main` already fixes the likely failure mode from this report: the native Ollama transport now strips the `ollama/` provider prefix before sending `/api/chat`, and that behavior is covered by a regression test. The implementation is present in release `v2026.4.23`, so this issue can be closed as fixed on main/shipped.
