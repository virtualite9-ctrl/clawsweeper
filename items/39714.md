## Summary

Keep open. Current main still does not implement the Docker/DooD sandbox browser bridge fix: sandbox CDP remains hardcoded to loopback, sandboxed agents still receive the host-local bridge `baseUrl`, and the plugin SDK/browser bridge contract has no advertised/container-reachable URL. The draft PR is not mergeable as-is, but the underlying bug remains real and useful work remains to rewrite it against the current browser plugin path.

## Best Possible Solution

Keep this PR open as a useful reproduction/partial implementation, or replace it with a current-path PR. The best fix is to add an explicit advertised/container-reachable browser bridge URL through `src/plugin-sdk/browser-bridge.ts` and the bundled browser runtime, keep host-local `bridgeUrl` for gateway/browser-tool calls, inject the advertised URL into `buildEmbeddedSandboxInfo()`, resolve sandbox browser CDP host/port separately for Dockerized gateways, handle attach-only non-loopback tab creation without the persistent Playwright path where needed, and add focused Dockerized gateway plus non-Docker host tests and docs.
