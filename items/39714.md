## Summary

Keep open. Current `main` still lacks the central Docker/DooD fix: the sandbox browser bridge has only a host-local `baseUrl`, embedded sandbox info injects that same URL into the sandboxed agent, and sandbox CDP readiness/config still assume `127.0.0.1`. The draft PR should not land as-is, but the underlying product bug remains real and is not implemented or superseded on current `main`.

## Best Possible Solution

Keep this open or replace it with a current-path PR. The best fix is to add an explicit advertised/container-reachable browser bridge URL through `src/plugin-sdk/browser-bridge.ts` and the bundled browser runtime, keep host-local `bridgeUrl` for gateway/browser-tool calls, inject the advertised URL into embedded sandbox info, resolve sandbox browser CDP host/port separately for Dockerized gateways, handle attach-only non-loopback tab creation without the persistent Playwright path where needed, and add focused Dockerized gateway plus non-Docker host tests and docs.
