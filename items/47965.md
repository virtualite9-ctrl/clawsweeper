## Summary

Keep open. Current `main` still appears vulnerable to an early `existing-session` Chrome MCP attach failure taking down the gateway: the Chrome MCP session bootstrap still creates a `ready` promise without an immediate rejection boundary, OpenClaw still exits on generic unhandled promise rejections, and there is no evidence of the linked handshake hardening landing on `main` or in the latest release.

## Evidence

- **Chrome MCP session bootstrap still has naked `ready` promise:** `createRealSession()` builds `ready` as an immediately-invoked async promise and returns it without attaching a local `.catch()`. The transport is created with `stderr: "pipe"`, but this code path does not add stderr draining/diagnostic handling here.
