## Summary

Keep open. Current main at 110fa97f2aa171d601d1736a5f4855d299f6fb88 still has no equivalent root/effective-root CLI guard, while the modern entrypoint, legacy entrypoint, state directory resolution, and Linux user-systemd service paths still support the #67478 failure mode. This PR remains the active implementation candidate and has already incorporated review feedback for legacy entry coverage, container forwarding, and effective UID handling.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land this PR or an equivalent core fix that guards both `src/entry.ts` and `src/index.ts` before state/config side effects, checks real and effective UID, preserves intentional container-forwarding behavior, includes focused regression tests, and updates the DigitalOcean/install docs or wrapper flow so root SSH users get a clear supported command path.
