## Summary

Keep open. Current main still lacks an equivalent real/effective-root CLI guard, the linked #67478 bug remains open, and this PR is still the focused implementation candidate. The PR discussion shows the author addressed the main review concerns around container forwarding, the legacy entrypoint, and effective UID handling. Security review found no CI, dependency, lockfile, install-script, release, or supply-chain-sensitive file changes in the provided PR file list.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land this PR or an equivalent core fix that guards both src/entry.ts and src/index.ts before state/config side effects, treats either real UID or effective UID 0 as root, preserves intentional container forwarding, includes focused regression tests, and updates the DigitalOcean/install guidance or wrapper flow so root SSH users get a clear supported command path.
