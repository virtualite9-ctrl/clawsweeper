## Summary

Keep PR #67509 open. Current main does not implement an equivalent root guard: the modern entrypoint and legacy entrypoint both still reach CLI startup without checking uid/euid, while the related #67478 failure mode is still plausible from the current state-dir and systemd-user paths. The PR is a focused, active bug-fix branch that has already addressed review feedback around static imports, container forwarding, the legacy entrypoint, and effective UID handling.

## Best Possible Solution

Keep #67509 open for maintainer review. The best path is to land this PR or an equivalent core fix that guards both `src/entry.ts` and `src/index.ts`, preserves safe root-level discovery/container-forwarding behavior, checks effective UID, and adds focused regression tests. Maintainers should also reconcile the root-guard behavior with DigitalOcean/install documentation before release.
