## Summary

Keep open. Current main still has the core failure shape: the A2UI hash is tracked while ignored, the build path can rewrite it, and the dev update preflight walkback still uses plain detached checkouts between candidates without resetting the disposable worktree. The prior minimal fix PR #64822 is closed unmerged, and the related #67547 A2UI reproducibility thread is not a canonical fix for the updater checkout-conflict bug.

## Best Possible Solution

Fix this in OpenClaw core. Add a dev updater regression test that simulates preflight build dirtying a tracked generated file, then make the disposable preflight worktree reset/clean before each fallback checkout. Also either stop tracking `src/canvas-host/a2ui/.bundle.hash` again or keep the A2UI bundler/hash contract from dirtying tracked files unexpectedly. Coordinate with #67547 for separate A2UI reproducibility work, but do not treat it as a replacement for this updater bug.
