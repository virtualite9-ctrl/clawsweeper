## Summary

Keep open. Current main still has the reported failure shape: `src/canvas-host/a2ui/.bundle.hash` is tracked while ignored, `pnpm build` still starts with the A2UI bundler, the bundler can write the hash file, and the dev update preflight loop still reuses one disposable worktree across fallback candidates with plain `git checkout --detach` and no reset/clean between attempts. The related minimal fix PR #64822 is closed unmerged, and #67547 tracks A2UI reproducibility rather than the updater checkout-conflict bug.

## Best Possible Solution

Fix this in OpenClaw core. Add a regression test where a dev update preflight build dirties a tracked generated file and a later fallback checkout must still proceed, then harden the disposable preflight worktree by resetting/cleaning before each candidate checkout. Separately, either stop tracking `src/canvas-host/a2ui/.bundle.hash` again or make the A2UI bundler/hash contract avoid dirtying tracked files unexpectedly. Treat #67547 as related A2UI reproducibility work, not as a replacement for the updater fix.
