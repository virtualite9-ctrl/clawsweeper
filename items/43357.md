## Summary

Keep this PR open. Current main still has ASCII-only Discord outbound mention rewriting, so the central Unicode/CJK handle problem is not implemented. The PR is still relevant, but the latest provided review thread leaves a real correctness/security-adjacent concern unresolved around CJK mention rewriting inside wrapped URL tokens, and related open PR #59818 overlaps the same Discord CJK mention area without superseding this work.

## Best Possible Solution

Keep the PR open, ask for a revision that strips or recognizes leading URL wrappers before the CJK URL-context check, and then compare it with #59818 so maintainers can pick or consolidate a single Discord plugin fix. The final implementation should keep reserved mentions unchanged, preserve email and URL text, support Unicode/CJK handles where intended, and add regression tests for Unicode handles, Unicode local-part emails, plain and wrapped URL tokens, and normal ASCII mention behavior.
