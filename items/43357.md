## Summary

Keep this PR open. Current main still uses ASCII-only Discord outbound mention parsing, so the Unicode/CJK handle rewrite is not implemented. The PR remains a useful candidate, but it needs revision because it targets the old `src/discord` path and its latest review thread still identifies wrapped-URL corruption. Related #59818 overlaps the same area but is also open, broader, and not a shipped superseding fix.

## Best Possible Solution

Keep this PR open as a needs-revision candidate. Ask the author to rebase onto `extensions/discord`, fix URL-context handling for wrapped URL tokens, keep the delimiter/email/reserved mention safeguards, and add focused regressions for Unicode handles, Unicode local-part emails, plain URL paths, wrapped URL tokens, reserved mentions, already-formatted Discord mentions, and existing ASCII behavior. Maintainers should then compare it with #59818 and either land the narrower safe fix or consolidate the broader bot-to-bot work into one canonical Discord plugin PR.
