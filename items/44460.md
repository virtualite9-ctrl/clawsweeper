## Summary

Keep #44460 open. Current main still preserves `notify: true` when `cron.webhook` is unset, so the reported doctor warning loop is still present. The related fix PR #48742 is useful context, but it remains open/unmerged and #44506 was closed unmerged, so the bug is not shipped or implemented on main.

## Best Possible Solution

Keep #44460 open until a focused fix lands. The likely code path is `src/commands/doctor-cron.ts`: when `cron.webhook` is unset, remove inert legacy `notify: true` for `delivery.mode` unset/`none`/`webhook`, preserve existing non-webhook delivery semantics with an explicit warning where needed, and add regression tests in `src/commands/doctor-cron.test.ts` for the unset-webhook repro. PR #48742 is the current useful starting point, but it should be reviewed or narrowed before merge.
