## Summary

Keep PR #59852 open. Current main still has the timer-leak behavior described by the PR: `sendRequest()` creates a 10-second timeout without storing its handle, successful responses delete the pending request without clearing that timeout, and `disposeSession()` rejects/clears pending requests without clearing any request timers. The PR is a focused contributor bug fix against an active LSP runtime path used by embedded Pi attempts/compaction, and there is no evidence in current main that the requested cleanup was implemented elsewhere.

## Best Possible Solution

Keep the PR open or recreate its change on main: track the `setTimeout` handle in `src/agents/pi-bundle-lsp-runtime.ts`, clear it when an LSP response resolves/rejects the pending request, clear remaining timers during `disposeSession()`, and add focused fake-timer coverage for successful response, timeout, and disposal behavior.
