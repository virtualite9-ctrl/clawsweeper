## Summary

Current main already contains the Codex transport fixes that match this 2026.3.24 report: Anthropic `api: "anthropic-messages"` defaulting is now scoped away from `openai-codex`, OpenAI Codex transport metadata is normalized back to the native Codex Responses route, and doctor/docs now call out stale `models.providers.openai-codex` overrides.

## Evidence

- **Changelog shipped fix for wrong Anthropic defaulting on openai-codex:** Release 2026.4.20 explicitly says Anthropic `api: "anthropic-messages"` defaulting is now scoped to Anthropic-owned providers so `openai-codex` is no longer rewritten to the wrong transport.
  - file: [CHANGELOG.md:709](https://github.com/openclaw/openclaw/blob/89755d1c79678610720cdb9ce504cfd62c261f64/CHANGELOG.md#L709)
  - command: `nl -ba CHANGELOG.md | sed -n '680,730p'`
  - sha: [89755d1c7967](https://github.com/openclaw/openclaw/commit/89755d1c79678610720cdb9ce504cfd62c261f64)
- **Anthropic config normalization now excludes openai-codex:** `normalizeAnthropicProviderConfigForProvider(...)` returns the config unchanged unless the provider is `anthropic` or `claude-cli`, preventing `openai-codex` from inheriting `anthropic-messages`.
  - file: [extensions/anthropic/config-defaults.ts:181](https://github.com/openclaw/openclaw/blob/89755d1c79678610720cdb9ce504cfd62c261f64/extensions/anthropic/config-defaults.ts#L181)

  - file: [extensions/anthropic/index.test.ts:118](https://github.com/openclaw/openclaw/blob/89755d1c79678610720cdb9ce504cfd62c261f64/extensions/anthropic/index.test.ts#L118)

  - file: [extensions/openai/openai-codex-provider.ts:133](https://github.com/openclaw/openclaw/blob/89755d1c79678610720cdb9ce504cfd62c261f64/extensions/openai/openai-codex-provider.ts#L133)
