## Summary

Keep open. Current main still has the UI behavior this PR targets, the linked bug #41545 is still open, and this PR remains a focused implementation candidate with regression coverage. The exact patch still needs maintainer judgment because preserving the token across arbitrary WebSocket URL edits can carry a selected-gateway credential to a different endpoint.

## Best Possible Solution

Keep the PR open for maintainer review or rework. The best fix should prevent ordinary WebSocket URL edits from unexpectedly erasing a manually entered Gateway Token while preserving the selected-gateway credential boundary, with regression coverage in the Overview/navigation UI tests and explicit handling for when a URL change should require a new token.
