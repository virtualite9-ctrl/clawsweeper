## Summary

Keep open. Current main still clears the Overview Gateway Token whenever the WebSocket URL input changes, and the linked bug #41545 remains an open product bug. This PR is a plausible focused fix with regression coverage, but its exact behavior needs maintainer judgment because current storage, docs, and connect code treat tokens as scoped to the selected gateway URL.

## Best Possible Solution

Keep the PR open for maintainer review or rework. The best fix should stop clearing a manually entered Gateway Token on every ordinary URL keystroke while preserving the selected-gateway credential boundary, with regression coverage in the Overview/navigation UI tests and any helper introduced for the state transition.
