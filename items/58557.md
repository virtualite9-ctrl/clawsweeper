## Summary

Keep #58557 open. Current main does not already document cron output-language troubleshooting, and this is a repo docs change rather than a duplicate, ClawHub item, or external-administration task. The PR is not merge-ready as-is because cron currently exposes prompt/message fields, not a cron language or template-substitution surface, but a narrowed docs change that explains the supported `--message` prompt approach would be salvageable.

## Best Possible Solution

Keep the PR open for maintainer or author revision. The useful path is a narrow English-only update to `docs/automation/cron-jobs.md` that says cron output language is controlled by the job prompt/message, shows `openclaw cron edit <jobId> --message ...`, and explicitly avoids documenting unsupported `language` config keys, template files, template variables, or generated locale docs. If OpenClaw wants a real cron language field later, that should be implemented first in the cron types, Gateway schema, CLI, tests, and docs.
