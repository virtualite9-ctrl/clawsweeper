## Summary

Keep #58557 open. Current main still lacks cron output-language troubleshooting, so the central docs request is not implemented or superseded. The PR is also not safely closeable as external/plugin work: the supported current path is prompt text via `--message`, while current cron types, Gateway protocol schema, CLI options, and docs do not expose a cron `language` config field or a cron prompt-template substitution workflow.

## Best Possible Solution

Keep the PR open for a narrow author or maintainer revision. The best version is an English-only docs update in `docs/automation/cron-jobs.md` that explains cron output language is controlled by the job prompt/message, shows `openclaw cron edit <jobId> --message ...`, and avoids unsupported `language` config keys, cron template files, template variables, and generated locale-doc edits. If maintainers want a real cron language setting later, that should be implemented first across cron types, Gateway schema, CLI, tests, and docs.
