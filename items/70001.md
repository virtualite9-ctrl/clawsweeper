## Summary

Current `main` already refreshes the installed gateway service environment from the updated install before restart. The PR’s detached-script env pinning is redundant for the current service-manager design, and the PR discussion surfaced correctness issues in the proposed patch.

## Evidence

- **Update flow refreshes service env before restart:** `updateCommand` marks installed services for env refresh, then `maybeRestartService` runs `refreshGatewayServiceEnv`, which invokes `gateway install --force` from the updated install root before restarting the service.
  - file: [src/cli/update-cli/update-command.ts:256](https://github.com/openclaw/openclaw/blob/2cd2732ab6695a4b3220ce41166b74aca7097c5f/src/cli/update-cli/update-command.ts#L256)
  - command: `nl -ba src/cli/update-cli/update-command.ts | sed -n '240,340p'; nl -ba src/cli/update-cli/update-command.ts | sed -n '1370,1410p'`
  - sha: [2cd2732ab669](https://github.com/openclaw/openclaw/commit/2cd2732ab6695a4b3220ce41166b74aca7097c5f)
- **Fresh service version comes from updated runtime:** `buildServiceEnvironment` writes `OPENCLAW_SERVICE_VERSION: VERSION`, so the managed service env is regenerated from the currently running updated code during reinstall.
  - file: [src/daemon/service-env.ts:253](https://github.com/openclaw/openclaw/blob/2cd2732ab6695a4b3220ce41166b74aca7097c5f/src/daemon/service-env.ts#L253)
  - command: `nl -ba src/daemon/service-env.ts | sed -n '253,283p'`
  - sha: [2cd2732ab669](https://github.com/openclaw/openclaw/commit/2cd2732ab6695a4b3220ce41166b74aca7097c5f)
- **Inherited OPENCLAW vars are not preserved into managed service env:** When rebuilding the gateway install environment, existing `OPENCLAW_*` variables are explicitly skipped before the fresh managed env is assigned, preventing stale updater-shell version vars from carrying into the service definition.
  - file: [src/commands/daemon-install-helpers.ts:178](https://github.com/openclaw/openclaw/blob/2cd2732ab6695a4b3220ce41166b74aca7097c5f/src/commands/daemon-install-helpers.ts#L178)

  - sha: [2cd2732ab669](https://github.com/openclaw/openclaw/commit/2cd2732ab6695a4b3220ce41166b74aca7097c5f)
- **Restart helper only restarts the service manager entry:** The detached helper scripts call `systemctl`, `launchctl`, or `schtasks` to restart the already-installed service; they do not exec the gateway directly, so extra env lines in the helper script do not define the persisted service environment.
  - file: [src/cli/update-cli/restart-helper.ts:64](https://github.com/openclaw/openclaw/blob/2cd2732ab6695a4b3220ce41166b74aca7097c5f/src/cli/update-cli/restart-helper.ts#L64)

  - file: [src/cli/update-cli.test.ts:1498](https://github.com/openclaw/openclaw/blob/2cd2732ab6695a4b3220ce41166b74aca7097c5f/src/cli/update-cli.test.ts#L1498)
