## Summary

Keep PR #39507 open. Current main still computes Feishu `threadReply` from inbound `thread_id`/`root_id` and ORs that into `replyInThread`, so `replyInThread: "disabled"` is still overridden for quote-reply messages. The PR is not implemented on main, not shipped in the latest release, and not safely superseded by #33789 because that merged PR fixed reply target selection, not the remaining disabled-thread semantics.

## Best Possible Solution

Keep #39507 open and resolve the Feishu semantics explicitly. If disabled should be a hard off switch, update `extensions/feishu/src/bot-content.ts` so non-topic `threadReply` is gated by configured `replyInThread` while preserving `group_topic` and `group_topic_sender` anchoring, then update the current regression test that expects forced thread replies under disabled config. If maintainers prefer follow-existing-thread behavior, clarify `replyInThread` schema/docs and test names so `"disabled"` no longer promises normal inline replies.
