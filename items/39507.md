## Summary

Keep open. Current `main` does not implement this fix: `resolveFeishuGroupSession` still sets `threadReply` from inbound `threadId`/`rootId` and then makes `replyInThread` true whenever `threadReply` is true, so `replyInThread: "disabled"` is still overridden for quote/thread replies in non-topic groups. The current Feishu test suite also explicitly expects that old behavior.

## Evidence

- **Current session resolution still overrides disabled config:** `extensions/feishu/src/bot-content.ts:68-74` computes `threadReply = Boolean(normalizedThreadId || normalizedRootId)` and then `replyInThread = configEnabled || threadReply`, which means inbound thread context still forces threaded replies even when config is `"disabled"`.
