## Summary

Keep PR #39507 open. Current main still lets inbound Feishu thread/root ids force `threadReply` and therefore `reply_in_thread` even when `replyInThread: "disabled"`; the current regression test explicitly expects that behavior. The related merged PR #33789 fixed reply-target selection, not this remaining disabled-thread semantic, and the latest release tag still has the same resolver behavior.

## Best Possible Solution

Keep #39507 open and resolve the Feishu semantics explicitly. If disabled should be a hard off switch, update `extensions/feishu/src/bot-content.ts` so non-topic `threadReply` is gated by configured `replyInThread` while preserving `group_topic` and `group_topic_sender` anchoring, then update the regression test that currently expects forced thread replies under disabled config. If maintainers prefer follow-existing-thread behavior, clarify the `replyInThread` schema/docs and test names so `"disabled"` no longer promises normal inline replies.
