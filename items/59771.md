## Summary

Keep open. Current `main` still closes the active Feishu streaming card on any reply error, so the duplicate-reply recovery described in the PR is not implemented on `main`. The PR is also active, not stale: the author rebased it on April 21, 2026 and explicitly addressed earlier stale-buffer feedback with `clearText()` recovery logic, while linked issue `#49381` remains open.
