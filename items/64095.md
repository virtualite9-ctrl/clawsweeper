## Summary

Keep open. Current `main` has moved the Gemini memory embedding code into `extensions/google`, but it still does not thread a proxy-aware `fetchImpl` through the Gemini client/options or the Gemini HTTP call sites. That means this PR's intended fix is not implemented on `main`, and the item is not safe to close as already fixed or obsolete.

## Evidence

- **Current Gemini client still lacks a fetch field:** `GeminiEmbeddingClient` on current `main` still contains `baseUrl`, `headers`, `ssrfPolicy`, `model`, `modelPath`, `apiKeys`, and `outputDimensionality`, but no `fetchImpl`; `resolveGeminiEmbeddingClient()` still returns that same shape.
  - file: [extensions/google/embedding-provider.ts:22](https://github.com/openclaw/openclaw/blob/5b8bd6371c666a5de43c82305e0315771e8e49d5/extensions/google/embedding-provider.ts#L22)
