## Summary

Keep open. Current `main` still derives the macOS LaunchAgent plist path directly from `HOME`, writes and bootstraps that same path, and only special-cases GUI-session bootstrap failures. The external-APFS-home case described in #60398 is not addressed on `main`, and the relevant daemon code is unchanged since `v2026.4.23`.

## Evidence

- **LaunchAgent plist path is still anchored to HOME:** `resolveHomeDir()` returns `env.HOME`/`env.USERPROFILE`, and `resolveLaunchAgentPlistPathForLabel()` joins that home with `Library/LaunchAgents/<label>.plist`. There is no boot-volume lookup or fallback path.
  - file: [src/daemon/paths.ts:8](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/src/daemon/paths.ts#L8)
