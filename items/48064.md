## Summary

Current `main` already supports a browser-local custom user display name and avatar for the Control UI/WebChat path, persists it in local storage, renders it in chat, and stamps outgoing chat messages with the client display name. The implementation differs from the issue’s proposed config shape, but the requested end-user capability is present.

## Risks / Open Questions

- The implementation is browser-local, not a server-side `user` config block or `USER.md` parser as proposed.
- Avatar support is constrained to browser-local image data/local routes or short text/emoji, not arbitrary remote URLs.
