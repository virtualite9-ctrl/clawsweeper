## Summary

Current main still only supports `pty` as a per-call exec parameter. The requested global `tools.exec.pty` default is not present in the config schema/types or threaded into exec defaults, and the related PR #45976 was closed unmerged in the provided GitHub context.

## Best Possible Solution

Keep #44603 open. A focused fix should add optional `pty?: boolean` to `ExecToolConfig`, `ExecToolDefaults`, and `ToolExecBaseShape`; thread `tools.exec.pty`/per-agent overrides through `createOpenClawCodingTools`; compute PTY with per-call override precedence such as `(params.pty ?? defaults?.pty) === true && !sandbox`; update the gateway approval path consistently; regenerate config schema/help; document `tools.exec.pty`; and add tests for global default, explicit per-call false override, and sandbox/no-PTY behavior.

## Risks / Open Questions

- A global PTY default changes command execution behavior for all unspecific exec calls, so the implementation should preserve per-call precedence and continue disabling PTY where the sandbox/runtime cannot support it.
- The fix spans config schema, generated schema/help/docs, default plumbing, gateway approval execution, and tests; a partial change could accept config without affecting runtime or vice versa.
