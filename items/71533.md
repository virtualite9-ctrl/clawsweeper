## Summary

Keep open. Current `main` already suppresses Baileys' own logger by default, but it still does not expose any `channels.whatsapp.logLevel` config, and the reconnect/heartbeat lines cited in the issue now come from OpenClaw's WhatsApp monitor code itself rather than from Baileys. The requested operator-facing knob is therefore not implemented on `main`.

## Evidence

- **Baileys logger is already silenced by default:** `createWaSocket()` builds the Baileys logger with `level: verbose ? "info" : "silent"` and passes that logger into `makeWASocket(...)`, so current `main` is not relying on Baileys' default lifecycle logging.
