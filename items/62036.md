## Summary

Current main already routes Telegram outbound replies through forum-topic thread ids, applies those thread params across chunked sends, and sends chunks serially. The shipped `v2026.4.23` tag contains the same implementation and targeted tests covering forum-topic `message_thread_id` retention plus long-message chunking behavior.

## Evidence

- **Outbound adapter carries topic thread ids into Telegram sends:** Telegram outbound delivery parses `threadId` into `messageThreadId` before calling the shared send path, so agent replies targeting `telegram:group:...:topic:...` stay topic-scoped.
