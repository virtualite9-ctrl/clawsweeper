## Summary

Current main still leaves #37705 actionable: the gateway can pass a Kubernetes-style hostname through the bundled Bonjour plugin as a 64-byte service instance label, while the locked @homebridge/ciao dependency asserts on labels over 63 bytes. The previously linked fix PRs were closed unmerged, and current tests do not cover byte-aware DNS-label truncation.

## Best Possible Solution

Keep #37705 open. Add a shared byte-aware DNS-label sanitizer in the bundled bonjour plugin before calling @homebridge/ciao, apply it to both the service instance name and OPENCLAW_MDNS_HOSTNAME-derived hostname, preserve UTF-8 boundaries, and keep a non-empty fallback. Add regression tests for the reported 64-byte instance label, longer configured mDNS hostname, exact 63-byte boundary, and multibyte names. A maintainer reply should also clarify that the supported workaround is OPENCLAW_DISABLE_BONJOUR=1 or discovery.mdns.mode="off", not CLAWDBOT_DISABLE_BONJOUR=1.
