## Summary

Keep open. Current `main` still passes an unbounded Bonjour instance label into `@homebridge/ciao`, so the reported 63-byte DNS label crash does not appear implemented or obsoleted.

## Evidence

- **Bonjour service name still has no 63-byte guard:** `safeServiceName()` only trims and falls back to `OpenClaw`; it does not truncate or byte-limit the label before advertising.
  - file: [src/infra/bonjour.ts:41](https://github.com/openclaw/openclaw/blob/55389f4be29359e2dd57ebaf548ba4a70331a445/src/infra/bonjour.ts#L41)

  - sha: [55389f4be293](https://github.com/openclaw/openclaw/commit/55389f4be29359e2dd57ebaf548ba4a70331a445)
- **Unbounded instance name is sent directly to ciao:** Gateway startup builds `instanceName` from display name and passes `safeServiceName(instanceName)` into `responder.createService({ name: ... })`, so long machine/display names still reach the mDNS library unchanged apart from trimming.
  - file: [src/infra/bonjour.ts:190](https://github.com/openclaw/openclaw/blob/55389f4be29359e2dd57ebaf548ba4a70331a445/src/infra/bonjour.ts#L190)
