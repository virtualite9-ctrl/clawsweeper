## Summary

Current main still leaves #37705 actionable: the bundled Bonjour plugin can pass a 64-byte service instance label into @homebridge/ciao, whose locked version asserts on DNS labels longer than 63 bytes. The linked fix PRs #37712 and #37714 were closed unmerged, and current tests do not cover byte-aware DNS-label truncation.

## Best Possible Solution

Keep #37705 open. Add a shared byte-aware DNS-label sanitizer in the bundled bonjour plugin before calling @homebridge/ciao, apply it to both service instance name and hostname-derived labels, preserve UTF-8 boundaries, keep a non-empty fallback, and add focused regression tests. A maintainer reply should also clarify the supported workaround: OPENCLAW_DISABLE_BONJOUR=1 or discovery.mdns.mode="off", not CLAWDBOT_DISABLE_BONJOUR=1.
