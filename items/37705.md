## Summary

Current main still leaves #37705 actionable. The bundled Bonjour plugin can still pass a 64-byte service instance label, and a long configured/derived hostname label, into @homebridge/ciao 1.3.6, whose encoder asserts on DNS labels longer than 63 bytes. The linked fix attempts #37712 and #37714 were closed unmerged, and current tests do not cover byte-aware DNS-label truncation.

## Best Possible Solution

Keep #37705 open. Add a shared byte-aware DNS-label sanitizer in the bundled Bonjour plugin, apply it to both the service instance name and hostname label before createService(), preserve UTF-8 boundaries with a non-empty fallback, and add focused tests for exact 63-byte labels, 64-byte labels, the reported Kubernetes-style hostname plus suffix, OPENCLAW_MDNS_HOSTNAME, and multibyte UTF-8. A maintainer reply should also clarify that the supported workaround is OPENCLAW_DISABLE_BONJOUR=1 or discovery.mdns.mode="off", not CLAWDBOT_DISABLE_BONJOUR=1.
