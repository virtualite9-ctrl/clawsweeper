## Summary

Keep open. Current main still rejects any shell command containing a backslash-newline before allowlist analysis, and the gateway/node-host paths still report that parser failure as an allowlist miss. The linked PR #71154 directly tracks the proposed fix, but it is open and unmerged; PR #71225 is closed unmerged, so neither proves the behavior is shipped on main.

## Best Possible Solution

Keep #71153 open and use PR #71154, or an equivalent narrower patch, as the implementation path. The fix should splice POSIX line continuations before allowlist analysis, preserve conservative quote/heredoc handling, add focused allowlist and host-policy regression tests, and introduce a distinct unsupported-shell-syntax denial path so true allowlist misses remain distinguishable.
