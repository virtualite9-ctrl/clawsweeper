## Summary

Close #71153 as superseded by the active linked PR #71154. Current main still has the reported behavior, so this is not implemented-on-main: `evaluateShellAllowlist()` still hard-fails on `\<newline>`, and gateway/node-host paths still surface `analysisOk=false` as an allowlist miss. PR #71154 is open, non-draft, authored by the issue reporter, explicitly says `Fixes #71153`, and contains the proposed line-continuation splice, distinct unsupported-shell-syntax errors, and focused tests. The later #71225/branch comment points in the same direction but #71225 is closed unmerged, so #71154 is the canonical remaining tracker.

## Best Possible Solution

Close #71153 as superseded by #71154 and keep the remaining implementation work on that PR: splice POSIX `\<newline>` continuations before allowlist analysis, preserve true allowlist misses for unapproved binaries, emit a distinct unsupported-shell-syntax reason for parse failures, and land the focused gateway/node-host/infra regression tests. If #71154 closes unmerged without an equivalent fix landing, reopen or file a new focused issue using the same reproduction.
