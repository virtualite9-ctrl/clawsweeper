## Summary

Current `main` does not show a clear OpenClaw-side fix for the reported `edit`/`write` corruption. The file tools still delegate to upstream `@mariozechner/pi-coding-agent`, OpenClaw’s local wrapper only adds validation/path handling and edit-error recovery, and the current tests I inspected do not cover literal-content corruption such as injected `//` inside identifiers. That is not strong enough evidence to close the issue conservatively.
