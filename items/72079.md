## Summary

Keep open. Current main still has the specific CLI-wrapper mismatch described in #72079: `openclaw health --json` calls the generic gateway helper without `clientName: "cli"` / `mode: "cli"`, while `openclaw gateway health --json` goes through the gateway CLI wrapper that sets both. A community comment reports success on one 2026.4.26 run, but the checked-out main code still preserves the divergent call shape and lacks a regression test for equivalence.

## Best Possible Solution

Keep the issue open and fix the top-level CLI health path by using the same CLI gateway caller identity as `openclaw gateway health` (or a shared helper) for `healthCommand`, preserving timeout/config/probe behavior. Add a focused regression test that asserts `openclaw health --json` invokes the gateway health RPC with `clientName: "cli"` and `mode: "cli"`, and consider whether `status --deep` should share the same CLI-to-gateway call shape.
