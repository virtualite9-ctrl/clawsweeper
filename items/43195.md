## Summary

Keep this PR open. Current main already shares the unhandled-rejection handler registry through globalThis, but installUnhandledRejectionHandler() still has no shared installed guard and still attaches a fresh process.on("unhandledRejection") listener on every call. The PR remains a plausible focused fix, though it should be rebased onto current main and should add a listener-count regression test.

## Best Possible Solution

Keep this PR open, or replace it with an equivalent maintainer patch, by adding a shared/global installed guard for installUnhandledRejectionHandler() on top of the current main structure. Preserve the existing global handler registries, add a focused repeated-install listener-count regression test, and verify plugin-registered rejection handlers remain shared.
