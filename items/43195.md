## Summary

Keep this PR open. Current main partially overlaps it by sharing the unhandled-rejection handler registry through globalThis, but it still does not make installUnhandledRejectionHandler() idempotent: each call still attaches another process.on("unhandledRejection", ...) listener. The PR remains useful if rebased onto current main and covered by an idempotency regression test.

## Best Possible Solution

Keep the PR open or replace it with an equivalent maintainer patch that preserves the current global handler registries, adds a shared/global installed guard for installUnhandledRejectionHandler(), and adds a focused regression test for repeated installs while confirming plugin-registered rejection handlers remain shared.
