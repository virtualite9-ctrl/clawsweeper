## Summary

Keep open. Current main still has the linked #72019 behavior: the subagent task is rendered in the subagent system prompt and repeated in the initial child user message. This PR is an active, focused implementation candidate with tests; #72028 is an open alternative, not a landed or clearly canonical superseding fix, and #72026 was closed unmerged.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to choose one canonical fix for #72019, land exactly one long-form copy of the subagent task in the first model request, preserve multiline and structured task text, and keep regression coverage for both the system-prompt and initial-user-message surfaces.
