## Summary

Current `main` no longer hardcodes 12-hour timestamps in the Control UI. The affected Control UI surfaces format times with `Date#toLocaleTimeString(...)` and do not pass `hour12`, so the browser locale controls 12h vs 24h output. A minimal Intl check also shows the same formatting options produce `21:50` for `sv-SE` and `9:50 PM` for `en-US`, so the report's core claim that 24-hour locales are ignored does not match current behavior.
