## Summary

Keep this PR open. Current main still lacks the requested web chat behavior: nested `message.usage.cost` is not read by grouped message metadata, savings metadata has no render path, and chat history sanitization still strips cost metadata beyond `total`. The PR is useful but should be rebased onto the current sanitizer location and covered by focused tests before merge.

## Best Possible Solution

Keep the PR open and ask for a focused update: port the sanitizer change to `src/gateway/chat-display-projection.ts`, make grouped rendering read `m.cost ?? m.usage.cost`, define how savings percentages aggregate across grouped assistant messages, and add tests for nested `usage.cost` preservation plus inline cost/savings rendering.
