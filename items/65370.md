## Summary

Keep this PR open. Current main still does not implement the requested web chat behavior: grouped message metadata only reads top-level `cost`, there is no savings badge render/style path, and chat history display projection still strips nested `usage.cost` metadata beyond `total`. The PR is directionally useful, including its ClawRouter prior-art context, but it needs a rebase to the current projection file and focused tests before it is merge-ready.

## Best Possible Solution

Keep this PR open and ask for a focused update: port the sanitizer change to `src/gateway/chat-display-projection.ts`, make grouped rendering read `m.cost ?? m.usage.cost`, define and test the savings aggregation rule for grouped assistant messages, add a chat display projection test for nested `usage.cost` routing metadata, and add a grouped-render test for the inline cost plus savings badge. After that, review the small UI/gateway diff normally; no separate security-sensitive changes are apparent from the provided patch.
