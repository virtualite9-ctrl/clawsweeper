## Summary

Keep this PR open. Current main still uses a direct diffs artifact-token inequality on the viewer access path, and the proposed shared helper import is an available plugin SDK surface with matching boolean semantics.

## Best Possible Solution

Keep this PR open and review it as the focused implementation candidate. The likely best path is to land the narrow `DiffArtifactStore.getArtifact` change using `safeEqualSecret` from `openclaw/plugin-sdk/browser-security-runtime`, then run the changed diffs/plugin validation before merge. A new regression test is optional because invalid-token rejection is already covered, but maintainers may still ask for one if they want explicit helper-use coverage.
