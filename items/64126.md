## Summary

Keep this PR open. Current main still uses a direct string inequality for diffs artifact viewer tokens, while the PR is a narrow one-file hardening change that reuses an existing SDK-exported timing-safe helper and does not add unrelated security-sensitive surface.

## Best Possible Solution

Keep this PR open as the focused implementation candidate. The best path is to land the narrow `DiffArtifactStore.getArtifact` change using `safeEqualSecret` from `openclaw/plugin-sdk/browser-security-runtime`, then rerun the repository's changed-check or targeted diffs plugin validation before merge.
