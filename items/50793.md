## Summary

Current main already solves the user-visible false delivery failure this PR targeted. `delivery.mode: "none"` jobs now persist `not-requested` instead of `not-delivered` even when the runner reports `delivered: false`, isolated cron runs now pass resolved message context into the embedded agent, and the branch’s broad payload-error heuristic was intentionally not needed as a merge path.

## Best Possible Solution

Close this PR as obsolete on main. Keep the shipped no-delivery status and isolated cron tool-context implementation; handle any remaining context-free channel-as-target tolerance as a small, separate follow-up instead of reviving this broad branch.
