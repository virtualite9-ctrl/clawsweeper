## Summary

Keep #38966 open. Current main still emits `sessions.changed` for `sessions.reset`, but only to clients that explicitly subscribe, and the TUI neither subscribes to session change events nor dispatches/handles `sessions.changed`. The linked PR #40472 is useful and directly targets this bug, but it is still open/unmerged in the provided GitHub context, so the user-visible external-reset refresh behavior is not implemented on main.

## Best Possible Solution

Keep #38966 open until PR #40472 or an equivalent fix lands. The fix should make the TUI subscribe to session change events, dispatch and handle `sessions.changed` for `new`/`reset` on the current session only, reload history and session metadata, clear stale token/run/stream/local-run state, ignore unrelated sessions, and add focused TUI handler/client coverage.
