## Summary

Keep #38966 open. Current main still lacks the TUI side of the requested external-reset refresh: reset now emits a generic sessions.changed notification, but the TUI gateway client does not subscribe to session events and the TUI event dispatch only routes chat, chat.side_result, and agent events. The supplied GitHub context shows PR #40472 is the active proposed fix, but it is open/unmerged, so the user-visible bug is not implemented on main yet.
