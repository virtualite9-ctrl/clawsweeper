## Summary

Keep #38966 open. Current main still does not wire the TUI into `sessions.changed` reset events: the gateway emits subscriber-gated session change events after `sessions.reset`, but `openclaw tui` neither subscribes to those events nor dispatches/handles them, so an external `/new` or `/reset` can still leave TUI history, token counts, and run state stale. The linked fix PR #40472 remains open/unmerged in the provided GitHub context, so the issue is not implemented on main yet.

## Best Possible Solution

Keep #38966 open until PR #40472 or an equivalent fix lands. The fix should make the TUI subscribe to `sessions.changed`, handle `new`/`reset` events for the current session only, reload history/session metadata, clear stale token/run/stream/local-run state, ignore unrelated sessions, and add focused TUI/gateway regression coverage.
