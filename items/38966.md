## Summary

Keep #38966 open. Current main still does not make the TUI subscribe to or handle `sessions.changed` reset events, so an external `/new` or `/reset` can leave the TUI with stale history/token/run state. PR #40472 directly targets the bug, but the provided context shows it is still open/unmerged.

## Best Possible Solution

Keep #38966 open until PR #40472 or an equivalent fix lands. The fix should make the TUI subscribe to session change events, dispatch and handle `sessions.changed` for `new`/`reset` on the current session only, reload history and session metadata, clear stale token/run/stream/local-run state, ignore unrelated sessions, and add focused TUI/gateway coverage.
