## Summary

Keep #38966 open. Current main at edf40ab6c9440604fe6a74aeaa8d8bc81b5ee16b still does not make `openclaw tui` refresh after an external `/new` or `/reset`: the gateway emits subscriber-gated `sessions.changed` reset events, but the TUI neither subscribes to `sessions.changed` nor dispatches/handles that event family. The provided related context shows PR #40472 as open and unmerged, so the remaining work is still tracked but not shipped on main.

## Best Possible Solution

Keep #38966 open until PR #40472 or an equivalent fix lands. The fix should make TUI subscribe to session change events, handle `sessions.changed` for the current session only when the reason is `new` or `reset`, reload history/session metadata, clear stale run/stream/local-run state, ignore unrelated sessions, and add focused gateway/TUI regression coverage.
