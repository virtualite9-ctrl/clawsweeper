## Summary

Current `main` already contains a targeted fix for third-party plugins installed outside the OpenClaw package tree. The loader now resolves the OpenClaw package root from explicit runtime hints (`argv1` and `moduleUrl`) before falling back to ancestor walking, which addresses the global-install layout described in the issue. The regression is covered by tests that model a plugin under `.openclaw/extensions/demo` and assert alias resolution succeeds via both the CLI entry hint and loader `moduleUrl` hint.

## Evidence
