## Summary

Keep open. Current main still lacks the PR's central behavior: broader provider-shaped overflow count extraction and a synthetic minimally-over-budget `currentTokenCount` when overflow is confirmed but unparseable. The existing shipped path from #40357 carries observed counts only when the current narrow extractor succeeds, so #70473 remains a useful focused bug-fix PR rather than cleanup-close material.

## Best Possible Solution

Keep #70473 open and have a maintainer review/land the focused overflow-budget extraction fix, or ask for any small follow-up needed on the existing PR. The code path belongs in `src/agents/pi-embedded-helpers/errors.ts` and `src/agents/pi-embedded-runner/run.ts`, with the proposed helper/runner regression coverage and the compaction docs update retained. #70479 should remain the separate session-preservation follow-up for the later orphaning behavior.
