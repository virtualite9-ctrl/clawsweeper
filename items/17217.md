## Summary

Keep #17217 open. Current main defines session lifecycle and has manual reset/compaction, but the requested TTL behavior for normal conversation history and an explicit mid-session prune API are still not implemented. `contextPruning.mode: "cache-ttl"` remains an event-driven, in-memory tool-result trimming feature, not wall-clock expiry or summarization of older user/assistant messages.

## Best Possible Solution

Keep the issue open for maintainer product work. A complete resolution should decide whether `contextPruning.ttl` is intended to expire full conversation history or only gate tool-result cache pruning. If full TTL pruning is desired, add an explicit session-control prune surface, implement age-based removal or summarization of older normal messages in the Pi/context-engine path, and cover it with source tests and docs. If that behavior is intentionally out of scope, update user-facing docs to state that TTL does not expire conversation messages and that `/reset` and `/compact` are the supported controls.
