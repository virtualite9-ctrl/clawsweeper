## Summary

Keep #17217 open. Current main documents session lifecycle and exposes reset/compact controls, but the central request is still not implemented: `contextPruning.ttl` does not wall-clock-expire normal conversation history, and there is no manual prune-context API. The current cache-ttl path is an in-memory prompt-cache optimization that runs during context assembly and only prunes old `toolResult` messages.

## Best Possible Solution

Keep the issue open for maintainer product work. Decide whether `contextPruning.ttl` should expire or summarize normal conversation history, or remain a tool-result prompt-cache optimization. If full TTL pruning is desired, add an explicit gateway/protocol prune surface and implement age-based removal or summarization in the Pi/context-engine path with focused tests and docs. If current behavior is intentional, update user-facing docs to make the limitation explicit and point users to `/compact`, `/new`, and `/reset` as the supported controls.
