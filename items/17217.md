## Summary

The issue should remain open. The provided discussion already includes an owner-maintainer Codex review keeping it open, and current `main` still documents `/reset`/`/compact` plus tool-result session pruning rather than wall-clock TTL expiry of normal conversation history or an explicit prune-context/session prune API.

## Best Possible Solution

Keep the issue open for maintainer product work. The next step is to decide whether `contextPruning.ttl` should expire full conversation history or only gate tool-result pruning. If full TTL pruning is desired, add an explicit session-control prune surface and implement age-based removal or summarization in the Pi/context-engine path with tests and docs. If it is intentionally out of scope, update user-facing docs to say TTL does not expire conversation messages and `/reset`/`/compact` are the supported controls.
