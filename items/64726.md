## Summary

Keep open. Current main only partially satisfies this broad memory embedding cache request: batch embedding, exact-hash cache reuse, cache seeding during full reindex, entry counts, and max-entry pruning exist, but fuzzy reuse for minor edits, hit/miss/latency cache metrics, frequency-based warming, gradual model migration, and vector compression are still absent. The existing ClawSweeper comment reached the same keep-open conclusion and remains consistent with current main.

## Best Possible Solution

Keep this issue open, but narrow the implementation plan around the missing behavior. Current main should keep the shipped batch embedding and exact-hash cache/pruning pieces; the next maintainer path is to add queryable cache metrics first, then separately decide whether fuzzy reuse, gradual model migration, warming, and vector compression are safe for the built-in memory-core cache or should be split into independent design issues.
