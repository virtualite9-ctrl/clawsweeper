## Summary

Keep open. Current main still exposes `sessions_send` with independent optional `sessionKey`, `label`, and `agentId` fields, and the executor still hard-rejects calls that contain both `sessionKey` and `label` with the exact reported error. The related PRs/issues in the provided context remain open/unmerged, so there is no implemented or canonical superseding fix to close this against.

## Best Possible Solution

Keep this issue open and fix it in `src/agents/tools/sessions-send-tool.ts`. The strongest product fix is to canonicalize executor input so a non-empty `sessionKey` is authoritative and `label`/`agentId` are ignored for target resolution, then add a regression test that a payload containing both fields sends to the `sessionKey` target. A secondary schema/docs update should describe the two addressing modes and show canonical examples, but schema-only guidance is not enough for the observed injected-label failure.
