## Summary

Current main still lacks `allowed_mentions` on the normal Discord outbound REST payloads, so the reported notification bug is not implemented on main. The related PR #46083 was closed unmerged in the provided GitHub context, and the present source only uses `allowed_mentions` for draft previews with `parse: []`, not for final outbound sends.

## Best Possible Solution

Keep the issue open and implement the Discord plugin fix in the outbound send path: extend the message payload/body construction used by text, media, chunked, and forum-thread sends to include an explicit safe `allowed_mentions` policy, then add regression coverage proving rewritten `<@id>` mentions are sent with that policy while preview updates remain non-pinging.
