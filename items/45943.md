## Summary

Keep open. Current main has strong adjacent hardening: `openclaw update` handles package installs, refreshes managed gateway service metadata, restarts by default, and verifies the restarted version. But this issue explicitly asks for auto-restart after a raw package-manager upgrade such as `npm i -g openclaw@latest`; direct package lifecycle still does not run `gateway install` or `gateway restart`, and the docs still show manual post-update restart steps for that path.

## Best Possible Solution

Keep the issue open and make the product boundary explicit. Either document raw `npm`/`pnpm`/`bun` upgrades as manual while treating `openclaw update` as the supported auto-restart path, or implement a guarded direct package-manager handoff that refreshes/restarts only OpenClaw-managed launchd/systemd/schtasks services and has lifecycle-safety tests.
