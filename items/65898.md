## Summary

Keep this PR open. Current main still implements only a single LLM idle-timeout path, while this PR proposes a real core agent runner/config change for a separate first-token phase. The supplied review discussion shows the author addressed earlier semantic/shadowing concerns, with a remaining concrete QA leak-detection follow-up rather than an obsolete or closeable PR.

## Best Possible Solution

Keep this PR open for maintainer review. The feature belongs in the core agent runner/config surface because it changes `agents.defaults.llm` and stream timeout behavior. The best path is to address the QA first-token timeout detection gap noted in review, keep schema/docs/tests aligned, rebase against current main, and then land or reject the feature by explicit maintainer judgment.
