## Summary

Current `main` already addresses the reported macOS gateway service problems: install no longer carries transient shell token state into the service plan, `gateway status` separates service supervision from RPC health, and the launchd path now bootstraps/repairs the LaunchAgent without the old immediate self-kickstart behavior.

## Evidence

- **Gateway install plan now uses durable service env only:** `buildGatewayInstallEnvironment` builds managed service env from durable sources and generated service env, rather than blindly persisting the invoking shell environment into the LaunchAgent.
