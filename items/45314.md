## Summary

Keep open. Current main still has the early-abort path before any model selection callback, while response prefix template resolution still leaves missing `{model}` / `{thinkingLevel}` values literal. The related PRs in the issue context are useful evidence of the root cause, but the current checkout does not contain their proposed fix and has no regression coverage for templated prefixes on `/stop`.

## Best Possible Solution

Keep #45314 open and implement a current-main fix that gives fast-abort replies the same prefix context as normal replies. The most concrete path is to seed or derive `ResponsePrefixContext` from session/config runtime metadata before early abort delivery, pass it through routed reply paths where needed, and add regression tests for `/stop` with `{model}` and `{thinkingLevel}` while preserving static prefixes and normal model-selection overrides.
