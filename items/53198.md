## Summary

Current main still has the documented Discord elevated allowFrom fallback unresolved. The docs promise fallback from `channels.discord.allowFrom`, but runtime elevated auth only consumes a channel plugin `elevated.allowFromFallback` hook, and the bundled Discord plugin still does not expose that hook. The linked implementation PR #53259 remains open and unmerged, so this issue should stay open until that PR or a superseding fix lands.

## Best Possible Solution

Keep this issue open and review, land, or supersede #53259. The fix should live in the Discord/channel-owned elevated fallback path and shared diagnostics path: use `channels.discord.allowFrom` only when `tools.elevated.allowFrom.discord` is omitted, make `sandbox explain` report the same effective allowlist, and add regression coverage for omitted fallback, explicit empty override, and diagnostics consistency.
