## Summary

Keep this issue open. The provided GitHub context shows an open, unmerged PR (#53259) that targets this exact Discord elevated allowFrom fallback bug with GitHub closing syntax, so the issue should stay open until maintainers review, land, or close that PR. Current main also still appears incomplete for the reported behavior: runtime has a generic plugin fallback hook, but Discord does not implement it, and `sandbox explain` reads only `tools.elevated.allowFrom.<channel>` rather than the documented Discord fallback.

## Best Possible Solution

Review PR #53259 against current main. The likely product fix is to add a Discord-owned `elevated.allowFromFallback` that resolves `channels.discord.allowFrom` when `tools.elevated.allowFrom.discord` is omitted, make `sandbox explain` use the same effective resolution path, and cover both runtime authorization and diagnostics with regression tests. After the PR or an equivalent fix lands, close this issue as implemented.
