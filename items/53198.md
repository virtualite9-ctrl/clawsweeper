## Summary

Keep this issue open. Current main still documents the Discord elevated fallback but runtime elevated auth only receives fallback allowlists from a channel plugin `elevated.allowFromFallback` hook, and the bundled Discord plugin does not provide that hook. The related #53259 PR remains open and unmerged with closing syntax for this issue, so it should be reviewed, landed, or superseded before the issue is closed.

## Best Possible Solution

Keep this issue open and review, revise, land, or supersede #53259. The fix should live in the Discord/channel-owned elevated fallback path and the shared diagnostics path: use `channels.discord.allowFrom` only when `tools.elevated.allowFrom.discord` is omitted, make `sandbox explain` report the same effective allowlist, and add regression tests for omitted fallback, explicit empty override, and diagnostics consistency.
