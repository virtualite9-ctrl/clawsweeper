## Summary

Keep this issue open. Current main still enforces the requested 16k context-window hard minimum: the guard defines 16,000 as the hard floor, runtime setup throws when a resolved model is below it, tests assert that behavior, onboarding clamps custom model metadata up to the hard minimum, and docs still tell users OpenClaw blocks below 16k.

## Best Possible Solution

Keep the issue open for maintainer product judgment. A complete implementation would replace the runtime hard block with warning-only behavior or an explicit supported override, then update `src/agents/context-window-guard.ts`, `src/agents/pi-embedded-runner/run/setup.ts`, custom-provider onboarding normalization, guard/onboarding tests, and local-model docs so sub-16k experimentation is consistently permitted while still warning about limitations.
