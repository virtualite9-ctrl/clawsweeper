## Summary

Keep open. Current main still implements the 16k context-window minimum as a hard runtime block, and the relevant tests, onboarding path, and local-model docs still encode that behavior. The prior ClawSweeper comment is still accurate against the newer main SHA.

## Best Possible Solution

Keep this issue open for maintainer product judgment. A complete implementation should replace the sub-16k hard block with warning-only behavior or an explicit opt-in override, preserve clear warnings for weak local/self-hosted models, stop onboarding from clamping intentionally smaller custom model windows unless that remains policy, and update the guard/runtime tests plus docs so the behavior is consistent.
