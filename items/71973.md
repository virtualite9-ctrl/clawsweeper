## Summary

Keep open. The prior prompt-cache work on main makes the system prompt time section cache-stable, but this issue targets volatile per-turn `Current time:` injection in heartbeat/cron/post-compaction prompt content, and current main still intentionally appends that line in those flows with tests asserting it.

## Best Possible Solution

Keep this open as a focused cache-optimization issue. The best maintainer path is to decide whether heartbeat/cron/post-compaction/reset flows can use a late-bound status/tool/event metadata path, or a less frequently changing date-only hint, while preserving startup and daily-memory correctness; then update the tests that currently assert unconditional `Current time:` injection and adjust the prompt-caching/date-time docs if behavior changes.

## Risks / Open Questions

- The current-time line was added for date correctness in heartbeat, reset, post-compaction, and memory-related flows, so an optimization should preserve daily-memory/startup behavior before removing or deferring it.
- Actual provider cache impact depends on transport-specific prompt shaping; a fix should be checked against prompt-cache observability or live cache evidence where feasible.
