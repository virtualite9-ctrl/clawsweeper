## Close Comment

Codex review: this looks implemented on current `main`. The report’s key pattern was that one turn started returning `400 JSON parsing failed`, and every later prompt in the same session kept failing until the session was deleted. Current `main` now gates synthetic replay `toolResult` insertion to Anthropic/Bedrock/Gemini transports in `src/agents/transport-message-transform.ts:3` and `src/agents/transport-message-transform.ts:45`, so OpenAI-compatible/OpenRouter sessions no longer inject the kind of fake replay entries that can poison later turns. OpenRouter is wired onto that passthrough replay path in `extensions/openrouter/index.ts:140`, and regression coverage was added in `src/agents/transport-message-transform.test.ts:24` plus `src/agents/transcript-policy.test.ts:456`. I also confirmed this transport fix is present on current `main` but not in the `v2026.4.22` tag, so I’m closing this as implemented on main rather than as a released fix.
