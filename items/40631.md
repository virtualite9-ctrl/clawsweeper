## Summary

Current main already implements the reported GPT-5/Codex "acknowledged work but no action started" cleanup: GPT-5-family OpenAI/OpenAI-Codex embedded runs now auto-enter strict-agentic mode, retry plan-only acknowledgements with an act-now steer, and then surface an explicit blocked/liveness state instead of silently appearing to start work.

## Close Comment

Closing based on a Codex review of current `main`. The reported GPT-5/Codex "assistant says it started but performs no action" path is now handled by the embedded runner: GPT-5-family `openai`/`openai-codex` runs auto-enable the `strict-agentic` execution contract (`src/agents/execution-contract.ts:46`), plan-only acknowledgements are retried with an explicit act-now steer (`src/agents/pi-embedded-runner/run/incomplete-turn.ts:132`), and if the model still does not take a concrete action the run exits with an explicit blocked error/liveness state instead of silently drifting (`src/agents/pi-embedded-runner/run.ts:1775`). Regression coverage is in `src/agents/pi-embedded-runner/run.incomplete-turn.test.ts:73` and `:622`, and the feature shipped in `v2026.4.10` per `CHANGELOG.md:863`. If you still see the same behavior on a current build, please open a fresh issue with a current trajectory/export bundle so it can be scoped as a new regression.
