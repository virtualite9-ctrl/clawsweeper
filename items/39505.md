## Summary

Keep #39505 open. Current main still reproduces the reported inserted-space corruption for backtick-wrapped Home Assistant entity IDs: the formatter strips edge punctuation into `candidate`, but the file-like copy-safety branch still tests the raw `token`, so markdown-wrapped dotted IDs miss preservation and are chunked with literal spaces. The focused fix remains PR #39565, which the provided GitHub context shows is open and unmerged; PR #39553 was closed unmerged.

## Best Possible Solution

Keep #39505 open until PR #39565 or an equivalent focused fix lands. The fix should classify the punctuation-stripped core token for file-like and dotted entity IDs, preserve backtick- and quote-wrapped Home Assistant IDs exactly, and add regression tests for bare and markdown-wrapped long dotted IDs without weakening the narrow-terminal long-token wrapping guard.
