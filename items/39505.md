## Summary

Keep open. Current `main` still has the copy-sensitive-token classification gap that lets punctuation-wrapped long entity IDs miss the preserve path and get chunked with inserted spaces. The only issue-specific fixes I found are PR #39553 (closed, unmerged) and PR #39565 (still open), so there is not strong evidence that #39505 is implemented on `main`.

## Evidence

- **Raw-token check still misses punctuation-wrapped entity IDs:** `isCopySensitiveToken()` strips edge punctuation into `candidate`, but the file-like branch still checks the raw `token`: `token.includes("_") && FILE_LIKE_RE.test(token)`. Backtick- or quote-wrapped entity IDs therefore fail `FILE_LIKE_RE` and are not treated as copy-sensitive.
  - file: [src/tui/tui-formatters.ts:82](https://github.com/openclaw/openclaw/blob/22f23fa5ab6f625835bfc3dbaba73c200775027c/src/tui/tui-formatters.ts#L82)
  - command: `nl -ba src/tui/tui-formatters.ts | sed -n '61,99p'`
  - sha: [22f23fa5ab6f](https://github.com/openclaw/openclaw/commit/22f23fa5ab6f625835bfc3dbaba73c200775027c)
- **Current formatter still inserts visible spaces when classification fails:** `normalizeLongTokenForDisplay()` falls back to `chunkToken(token, MAX_TOKEN_CHARS).join(" ")`, and `sanitizeRenderableText()` applies that to `\S{33,}` tokens. That is the exact mechanism that would split a long entity ID once it misses the copy-sensitive guard.
  - file: [src/tui/tui-formatters.ts:99](https://github.com/openclaw/openclaw/blob/22f23fa5ab6f625835bfc3dbaba73c200775027c/src/tui/tui-formatters.ts#L99)
  - command: `nl -ba src/tui/tui-formatters.ts | sed -n '94,153p'`
  - sha: [22f23fa5ab6f](https://github.com/openclaw/openclaw/commit/22f23fa5ab6f625835bfc3dbaba73c200775027c)
- **Mainline tests do not cover the reported regression:** The current test file covers bare paths, URLs, underscore-heavy tokens, and credential-like tokens, but there is no regression test for backtick/quote-wrapped Home Assistant-style entity IDs.
  - file: [src/tui/tui-formatters.test.ts:302](https://github.com/openclaw/openclaw/blob/22f23fa5ab6f625835bfc3dbaba73c200775027c/src/tui/tui-formatters.test.ts#L302)
  - command: `nl -ba src/tui/tui-formatters.test.ts | sed -n '302,336p'`
  - sha: [22f23fa5ab6f](https://github.com/openclaw/openclaw/commit/22f23fa5ab6f625835bfc3dbaba73c200775027c)
- **Issue-specific fix exists only in unmerged PRs:** PR #39553 says the root cause is the raw-token `FILE_LIKE_RE` check and proposes switching to the punctuation-stripped candidate; it was closed without merge. PR #39565 proposes the same class of fix and is still open. That points to the bug still being outstanding on `main`.
