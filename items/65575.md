## Summary

Current main still hardcodes the Control UI chat transcript as `role="log"` with `aria-live="polite"` even though `renderChat` already knows when the chat is busy. The PR is still a live implementation candidate: it changes that live-region behavior and adds focused tests, while the linked bug remains open and the alternate aria-busy PR was closed unmerged.

## Best Possible Solution

Keep this PR open and review it against #65538. The likely maintainer path is to land a focused Control UI accessibility fix with regression tests, adapting the implementation to `aria-busy` or a completed-message announcement pattern if maintainers prefer that contract, then close the linked bug after merge.
