## Summary

The request is a useful integration idea, but current main already treats non-core sandbox backends as plugin territory, so this belongs as an external ClawHub/plugin package rather than an OpenClaw core issue.

## Close Comment

Closing as `clawhub` after a Codex review of current `main`. OpenClaw already has a public plugin seam for sandbox runtimes in `src/plugin-sdk/sandbox.ts:24`, core explicitly expects non-built-in backends to come from plugins in `src/agents/sandbox/backend.ts:61`, and the bundled OpenShell backend follows that model in `extensions/openshell/index.ts:1`. The project vision also says optional capability should usually ship as plugins and community extensions belong on ClawHub rather than in core (`VISION.md:52`, `docs/plugins/community.md:9`). For `agent-core`, the right path is an external OpenClaw plugin/backend package published on ClawHub or npm, not a core OpenClaw issue.
