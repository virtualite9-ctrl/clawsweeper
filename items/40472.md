## Summary

Keep this PR open. Current main still does not refresh the TUI after an external reset of the active session, and this PR is a focused implementation candidate for the linked #38966 bug, though it needs rebasing/adaptation to the current `sessions.changed` subscription model before it can land cleanly.

## Best Possible Solution

Keep this PR open or supersede it with a maintainer patch that targets current main: have the TUI subscribe to the session lifecycle stream, handle `sessions.changed` for `reason: "new"` or `"reset"` when `sessionKey` matches `state.currentSessionKey`, clear stale run/tool/streaming state, reload history/session info, ignore unrelated sessions, and add focused TUI handler plus gateway/TUI contract coverage. If maintainers choose a new generic `session` event instead, update the advertised events, broadcast scope guard, protocol expectations, and tests together.
