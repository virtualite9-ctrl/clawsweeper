## Summary

Keep open. Current main still does not emit compaction events from the embedded runner overflow/timeout recovery helpers, so this PR targets a real remaining behavior gap. The patch is narrow and security-benign, but it should be hardened before merge around completed semantics, memory-counting behavior, and focused regression tests.

## Best Possible Solution

Keep this PR open as a valid implementation candidate. The best path is to update the embedded runner recovery helpers to emit compaction start/end events to both the global event bus and params.onAgentEvent, align completed semantics with handleCompactionEnd or explicitly codify the difference, harden the memory consumer to count only completed === true, and add focused overflow/timeout recovery tests for successful and failed payloads.
