## Summary

Keep open. Current main still lacks compaction event emission from embedded runner timeout/overflow recovery, while the PR targets that real gap. The PR is narrow and does not show a security or supply-chain concern, but it should be revised before merge to align completed semantics and avoid marking failed recovery compactions as successful in memory accounting.

## Best Possible Solution

Keep this PR open as a valid implementation candidate. Best path: revise the recovery helpers to emit compaction start/end events consistently with handleCompactionEnd, harden memory accounting to count only completed compactions, and add focused overflow and timeout recovery tests for both emitAgentEvent and params.onAgentEvent before merge.

## Risks / Open Questions

- Merging the PR as written can mark failed timeout/overflow recovery compactions as completed in the memory flush path unless that consumer checks completed === true.
- The PR's completed value uses ok && compacted, while the canonical handler derives completion from result presence and abort state; that semantic mismatch should be resolved or documented.
- The PR currently lacks focused regression coverage proving both global event bus and params.onAgentEvent start/end payloads for successful and failed timeout/overflow recovery paths.
