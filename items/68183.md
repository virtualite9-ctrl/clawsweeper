## Summary

Current main no longer builds sandbox-capable Docker gateway images with the old Debian Docker CLI. The Dockerfile installs Docker's current `docker-ce-cli` from Docker's official Debian repository when the sandbox CLI build arg is enabled, and the Docker setup script automatically enables that build arg for local sandbox builds and refuses to leave sandbox mode enabled when the CLI is missing. The same Dockerfile implementation is present in v2026.4.24. Related Docker-out-of-Docker path mapping reports remain separate from this API-version mismatch.

## Best Possible Solution

Keep the shipped Dockerfile and setup behavior: sandbox-capable local Docker builds should install the current Docker CE CLI and verify it before enabling sandbox mode. Track remaining Docker-out-of-Docker host/container path mapping problems in the separate Docker sandbox reports rather than keeping this API-version mismatch open.
