## Summary

Current `main` already accepts `heartbeat.every: "30m"` as a duration string, so the shorthand-parsing theory in the issue comments does not match the code. But there is not enough evidence to close: heartbeat runs can still be skipped for runtime reasons like `requests-in-flight`, `quiet-hours`, `empty-heartbeat-file`, or `no-tasks-due`, and the report does not include the corresponding heartbeat logs/status needed to rule those out against current `main`.
