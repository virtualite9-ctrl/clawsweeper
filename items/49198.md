## Summary

Keep open. Current `main` added `scripts/stage-bundled-plugin-runtime.mjs`, but it does not implement the manifest-aware runtime copy behavior from PR #49198. It only copies `package.json`, bundle/native manifest files, and `SKILL.md`, while symlinking other non-JS files. That leaves Claude/Cursor command markdown and other manifest-declared skill-tree files on the symlink path, even though current bundle semantics treat those roots as skill content.

## Evidence

- **Runtime staging still only copies a narrow filename allowlist:** `shouldCopyRuntimeFile()` copies only `package.json`, `openclaw.plugin.json`, the three bundle manifest files, and `SKILL.md`; anything else falls through to `symlinkPath()` in `stagePluginRuntimeOverlay()`. There is no descriptor-driven handling for declared skill roots or command roots.
  - file: [scripts/stage-bundled-plugin-runtime.mjs:131](https://github.com/openclaw/openclaw/blob/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4/scripts/stage-bundled-plugin-runtime.mjs#L131)
