## Summary

Keep open. Current `main` still does not enforce the configured Feishu `webhookPath`, while the PR adds the missing path guard and regression coverage and appears to have addressed review-raised edge cases.

## Evidence

- **Current main never checks the incoming request path:** `monitorWebhook` reads `account.config.webhookPath` into `path`, but the request handler uses it only for logging, status recording, and the rate-limit key. There is no comparison between `req.url`/pathname and `path` before body parsing, signature validation, or `eventDispatcher.invoke(...)`. That matches the PR's bug report.
  - file: [extensions/feishu/src/monitor.transport.ts:157](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/extensions/feishu/src/monitor.transport.ts#L157)
  - command: `nl -ba extensions/feishu/src/monitor.transport.ts | sed -n '130,240p'`
  - sha: [c070509b7f8f](https://github.com/openclaw/openclaw/commit/c070509b7f8fdbe479e5d829643782c272e30e83)
- **Feishu config and docs treat `webhookPath` as a real routing boundary:** The schema exposes `webhookPath` as configurable at both account and top level, and the public Feishu docs describe `channels.feishu.webhookPath` as the webhook route path with default `/feishu/events`. That makes the missing enforcement a real behavior gap, not just an unused config knob.
  - file: [extensions/feishu/src/config-schema.ts:202](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/extensions/feishu/src/config-schema.ts#L202)
  - command: `nl -ba extensions/feishu/src/config-schema.ts | sed -n '180,235p'`
  - sha: [c070509b7f8f](https://github.com/openclaw/openclaw/commit/c070509b7f8fdbe479e5d829643782c272e30e83)
- **Current webhook security tests do not cover wrong-path rejection:** Existing webhook security coverage exercises missing credentials, content type, payload size, timeout, and rate limiting, but there is no current-main test for `404` on a non-configured path or for preventing dispatch on the wrong path. The PR's added regression tests fill that gap.
