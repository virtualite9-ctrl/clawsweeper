## Summary

Keep this PR open. Current main still does not enforce the configured Feishu webhook path, and this PR remains a focused implementation candidate for the linked Feishu ingress bug. It is not obsolete, duplicated by a shipped fix, or outside core/plugin scope.

## Best Possible Solution

Keep this PR open for maintainer review and landing, or replace it with an equivalent Feishu-owned fix in `extensions/feishu/src/monitor.transport.ts` that normalizes `webhookPath`, rejects mismatched request paths before body parsing/signature validation/dispatch, preserves query-string acceptance on the configured path, and keeps the new malformed-target/bypass regression tests.
