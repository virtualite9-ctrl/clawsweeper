## Summary

Keep this issue open. Current main still rewrites `google/gemini-3.1-pro` to the preview id, still uses the older `gemini-3-pro-preview` metadata template for Gemini 3.1 Pro forward compatibility, and still has tests/docs/defaults that preserve preview-derived behavior. The direct fix PRs in the related context are closed unmerged, so the reported problem is not implemented on main.

## Best Possible Solution

Keep the issue open. The best fix belongs in the Google plugin/model-selection path: stop rewriting stable `google/gemini-3.1-pro` to the preview id where stable ids should remain visible, update manifest/static and runtime normalization consistently, prefer a current Gemini 3.1 Pro template with the old preview template only as fallback if needed, and align focused regression tests plus Google provider docs/defaults. If preview ids are intentionally canonical, document that explicitly instead of closing as already fixed.
