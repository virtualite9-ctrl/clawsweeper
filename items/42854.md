## Summary

Keep this issue open. Current main still normalizes `google/gemini-3.1-pro` to the preview id, still uses the older `gemini-3-pro-preview` template for Gemini 3.1 Pro forward compatibility, and still has tests/docs/defaults that preserve preview-derived behavior. The related fix PRs in the provided context are closed unmerged, and the later stale-catalog thread is adjacent rather than a fix for this alias/template path.

## Best Possible Solution

Keep the issue open. The likely fix belongs in the Google plugin/model-selection path: stop rewriting stable `google/gemini-3.1-pro` to the preview id where stable ids should remain visible, update Gemini 3.1 Pro template precedence away from the old `gemini-3-pro-preview` source, and align focused regression tests plus Google provider docs/defaults. If maintainers instead intend preview ids to remain canonical, that product decision should be stated explicitly before closing.
