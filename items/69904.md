## Summary

Keep PR #69904 open. Current main still does not implement stale transient async notice filtering, so the PR is not obsolete. The PR is a focused fix attempt, but the provided review context includes an unresolved P1 concern that the proposed exec-event short-circuit can drop legitimate non-completion exec notices such as `Exec started` and `Exec denied`, so it needs revision rather than conservative cleanup closure.

## Best Possible Solution

Keep the PR open and ask for a targeted update: retain the stale transient completion/success filtering, but make the heartbeat skip/consume logic preserve cron events and legitimate non-completion exec notices such as `Exec started` and `Exec denied`; add regression tests for those mixed queues before reconsidering merge or closure.
