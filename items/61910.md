## Summary

Keep open. Current `main` still limits `before_model_resolve` to provider/model overrides, and I did not find the dedicated `api.registerProviderRuntimeAuthOverride` seam mentioned in the issue follow-up. OpenClaw does have provider-plugin seams for runtime auth and request headers, but that is a different surface and does not show this item as implemented on `main`.
