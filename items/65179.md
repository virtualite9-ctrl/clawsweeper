## Summary

Keep open. Current main still treats non-allowlisted cron `payload.model` as a fallback case, and that behavior is pinned by source, tests, docs, and changelog. This PR proposes a real contract change rather than cleanup of already-implemented code, and the proposed implementation has unresolved review feedback around provider/model validation plus an unrelated SSRF message change.

## Best Possible Solution

Keep this PR open for maintainer product review. If cron job model declarations should bypass `agents.defaults.models`, rework the change on current main with exact provider/model catalog validation, add tests for allowlisted, catalog-only, and wrong-provider collision cases, update the cron docs/changelog contract, and split out the unrelated SSRF message change. If the allowlist fallback is intentional, a maintainer should close this PR after stating that contract explicitly.
