## Summary

Current main still duplicates native `sessions_spawn` task text in both the child system prompt and first user message, so the bug is not already fixed. PR #72028 is an active focused fix attempt, but its “task is in the first user message” direction has a real forked-transcript task-binding concern already raised in the PR discussion. Related PR #72053 tracks the same #72019 work with the alternate system-bound-task approach, but it is still open and unmerged, so this should remain a maintainer choice rather than an automated cleanup close.

## Best Possible Solution

Keep #72028 open for maintainer review while consolidating the overlapping #72019 fixes. The best final implementation should keep the long delegated task in exactly one authoritative model-visible location, avoid repeating it in the first request, preserve multiline/structured task text, and include regression coverage for the final `message` plus `extraSystemPrompt` gateway payload. If #72053 or an equivalent canonical fix lands first, close #72028 as superseded by that shipped commit.
