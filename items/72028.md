## Summary

Keep open. Current main still has the duplicated subagent task path described by #72019: the concrete task is embedded in the subagent system prompt and also sent in the first child user message. PR #72028 is a focused, relevant fix attempt with regression coverage in the provided PR patch, but maintainers should address or explicitly accept the Aisle concern about moving concrete task binding out of the system prompt for persistent sessions.

## Best Possible Solution

Keep the PR open and review it as a real fix for current main. The best path is to land one canonical #72019 fix that removes duplicate token cost while preserving task binding for persistent sessions, with regression tests covering both the system prompt and child message payload.
