## Summary

Keep this PR open. Current main has the generic uncaught-exception handler registry, but not this PR's behavior: transient uncaught network errors still go through the fatal CLI/global path, and Telegram polling only wires an unhandled-rejection restart handler.

## Best Possible Solution

Keep this PR open for maintainer review or rebase. The best implementation is to preserve fatal hooks for truly fatal uncaught exceptions, suppress transient uncaught network failures such as ENETUNREACH in the top-level handlers, register a Telegram uncaught-exception handler that reuses the polling restart path, keep regression coverage for CLI ENETUNREACH suppression and Telegram uncaught polling HttpError restart, and address the duplicate legacy-handler logging note if practical.
