## Summary

Keep this PR open. Current main has a generic uncaught-exception handler registry, but it still lacks the PR's central behavior: suppressing transient network failures in top-level uncaughtException handlers and wiring Telegram polling recovery through uncaught exceptions. The linked ENETUNREACH report remains represented by an open related issue, and recent PR discussion adds production corroboration for both Telegram polling stalls and Discord voice ENOTFOUND crashes.

## Best Possible Solution

Keep this PR open for maintainer review, rebase, or a direct port onto current main. The best implementation is to add transient uncaughtException suppression in the CLI/global handlers while preserving fatal hooks, register Telegram polling's recovery handler for uncaught exceptions as well as unhandled rejections, add regression tests for Telegram uncaught polling HttpError/getUpdates recovery and generic ENETUNREACH or ENOTFOUND uncaught suppression, and explicitly resolve or accept duplicate legacy-handler logging.
