## Summary

Keep #7433 open. Current main still lacks the requested Baileys reliability hooks in the bundled WhatsApp plugin: no socket-level `getMessage` or `cachedGroupMetadata`, no `groups.upsert` / `groups.update` / `group-participants.update` maintenance, and no `lid-mapping.update` listener. The existing maintainer-side Codex comment remains consistent with current `c7a0d9b` source; only an unrelated WhatsApp package dependency bump changed since that review.

## Best Possible Solution

Keep the issue open and implement it inside the bundled `@openclaw/whatsapp` plugin: add bounded full-message retry storage and pass `getMessage`, share a group metadata cache through `cachedGroupMetadata`, seed and maintain that cache from connect hydration plus group update/participant events, and decide whether Baileys' `signalRepository.lidMapping` persistence is sufficient or whether an explicit `lid-mapping.update` persistence path is needed. Add focused WhatsApp plugin tests for socket options, group cache event updates, retry-message lookup, and LID persistence behavior.
