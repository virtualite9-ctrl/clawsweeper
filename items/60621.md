## Summary

Current `main` already implements the `--session-id` over `--agent` behavior in a safer form than this PR, so the PR can close as superseded by code already on `main`.

## Evidence

- **Current main suppresses the agent-derived explicit session key when a session ID is provided:** `resolveSessionKeyForRequest` now computes `explicitSessionKey` as `opts.sessionKey?.trim()` or the agent-derived main key only when `requestedSessionId` is absent. That means `--session-id` can override `--agent`, while an explicit `--session-key` still remains authoritative.
