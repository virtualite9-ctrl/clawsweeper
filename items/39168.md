## Summary

Keep PR #39168 open. The exact PR patch is obsolete against current main because OpenClaw now uses @mariozechner/pi-agent-core 0.70.2 and has no pi-agent-core patchedDependency, but the reported mcp_exec/mcp_read/mcp_write dispatch failure is not implemented on main either. The current OpenClaw-side normalizer handles whitespace, aliases, provider-style prefixes, and malformed function/tool ids, but it still has no guarded mcp_ stripped-name lookup before pi-agent-core's exact tool-name dispatch.

## Best Possible Solution

Rework the PR against current main by implementing guarded mcp_ normalization in src/agents/pi-embedded-runner/run/attempt.tool-call-normalization.ts. The normalizer should resolve the raw name first, then strip mcp_ only when the suffix is non-empty and resolves unambiguously to an allowed tool such as exec/read/write. Preserve mcp__bundle__* names, avoid a pnpm dependency patch, and add focused tests for mcp_exec, mcp_read, mcp_write, nonexistent mcp_foo, bare mcp_, and registered mcp__bundle__* names.
