## Summary

Keep PR #39168 open. The PR patch is stale because current main uses @mariozechner/pi-agent-core 0.70.2 and has no pi-agent-core patchedDependency, but the requested mcp_exec/mcp_read/mcp_write dispatch normalization still is not implemented on main.

## Best Possible Solution

Rework the PR against current main: implement guarded mcp_ prefix normalization in src/agents/pi-embedded-runner/run/attempt.tool-call-normalization.ts, only after raw-name resolution fails and only when the stripped name resolves to an allowed registered tool. Preserve mcp__bundle__* names, avoid the obsolete pnpm patch, and add focused tests for mcp_exec, mcp_read, mcp_write, nonexistent mcp_foo, bare mcp_, and registered mcp__bundle__* names.
