## Summary

Current main still rejects trusted-proxy loopback/internal gateway connections, and two open PRs explicitly target this issue with closing syntax. This should stay open until a security-reviewed fix is merged or the linked PRs are closed/superseded by a maintainer decision.

## Best Possible Solution

Keep this issue open. Maintainers should review and either land or close/supersede #63379 and #64122, using #69066 for the longer-term service-identity direction. After a chosen fix merges with auth regression coverage and security review, GitHub closing syntax or a follow-up maintainer cleanup can close this issue as implemented.
