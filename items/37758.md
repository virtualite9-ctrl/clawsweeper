## Summary

Current `main` already implements Feishu multi-agent routing with per-agent session-store resolution and documents it as supported in the Feishu guide; this appears fixed on `main` but not yet in the latest tagged release provided (`v2026.4.22`).

Closing this as implemented on current `main` after a Codex review. Feishu now has explicit multi-agent routing docs (`docs/channels/feishu.md:329`) and the guide marks the supported version as `2026.4.24+` (`docs/channels/feishu.md:19`). In the runtime, inbound Feishu handling resolves the agent route from bindings and conversation bindings (`extensions/feishu/src/bot.ts:590-669`), then reads/writes session state using the resolved agent id instead of a default-agent path (`extensions/feishu/src/bot.ts:889-893`, `extensions/feishu/src/bot.ts:1192-1195`, `extensions/feishu/src/comment-handler.ts:208-210`). There are also Feishu tests covering bound non-default sessions such as `agent:codex:...` (`extensions/feishu/src/thread-bindings.test.ts:31-63`). I could not tie this to a shipped tag yet, and the latest release in context is still `v2026.4.22`, so I’m leaving `fixedRelease` unset.
