## Close Comment

Closing from a Codex review because this is already fixed on current `main` and in the latest shipped release. The shared `message` tool schema does not have a top-level `buttons` parameter anymore: `buttons` only exists as an optional nested field under `presentation.blocks[]` in `src/agents/tools/message-tool.ts:63` and `src/agents/tools/message-tool.ts:70`, and the top-level schema builder only makes `action` required in `src/agents/tools/message-tool.ts:403`. There is also a regression test asserting `properties.buttons` is `undefined` in `src/agents/tools/message-tool.test.ts:446`. The same schema shape is present in release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`), so this appears resolved in shipped builds as well.
