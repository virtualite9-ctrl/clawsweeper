## Summary

Keep open. Current `main` has task-aware status surfaces (`/tasks`, `/status`, `session_status`) but the live processing indicator is still only a typing-lifecycle control. I found no config or runtime support for a configurable processing message with task-name or ETA placeholders.

## Evidence

- **Agent defaults only expose typing cadence/mode:** `src/config/types.agent-defaults.ts:328` defines `typingIntervalSeconds` and `typingMode`, and `src/config/zod-schema.agent-defaults.ts:249` validates only those typing fields in this area. There is no `processingMessage`, `{taskDescription}`, or ETA-related config.
