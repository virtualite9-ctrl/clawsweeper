## Summary

Current `main` has hardened the lazy-runtime packaging path that matches this report: the Pi tool hook now imports a stable `./pi-tools.before-tool-call.runtime.js` entry, postbuild writes stable aliases for hashed `*.runtime-*.js` root chunks into `dist/`, and update/postpublish verification fails if packaged dist files or required sidecars are missing. That makes the reported missing `dist/pi-tools.before-tool-call.runtime-*.js` packaging regression already addressed on `main`.
