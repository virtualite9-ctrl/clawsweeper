## Summary

Issue #53863 is fixed on current `main`. OpenClaw now blocks npm publish/pack flows when `dist/control-ui/index.html` or the bundled `dist/control-ui/assets/` payload is missing, and the changelog plus issue comments indicate the fix shipped in `v2026.3.23-2`.

## Evidence

- **Prepack now requires Control UI artifacts before packing/publishing:** `collectPreparedPrepackErrors` treats `dist/control-ui/index.html` and `dist/control-ui/assets/` as required prepared artifacts, and `ensurePreparedArtifacts()` exits non-zero if they are missing.
  - file: [scripts/openclaw-prepack.ts:8](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/scripts/openclaw-prepack.ts#L8)
- **npm tarball validation rejects missing Control UI files:** Release validation requires `dist/control-ui/index.html` in the packed tarball and runs `collectControlUiPackErrors(...)` against `npm pack --json --dry-run --ignore-scripts` output.
  - file: [scripts/openclaw-npm-release-check.ts:62](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/scripts/openclaw-npm-release-check.ts#L62)
- **Regression tests cover the broken packaging case:** Tests assert both that missing prepared Control UI artifacts are reported and that packed tarballs without the Control UI asset payload are rejected.
  - file: [test/openclaw-prepack.test.ts:5](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/test/openclaw-prepack.test.ts#L5)
- **Release check also treats missing Control UI HTML as a broken package artifact:** `release-check` includes `dist/control-ui/index.html` in required pack contents and emits a build-artifacts-missing error if it is absent.
  - file: [scripts/release-check.ts:68](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/scripts/release-check.ts#L68)
- **Changelog records the shipped packaging fix in the 2026.3.23 release train:** The changelog notes that published npm installs now keep bundled Control UI assets, and separately that npm release checks fail when `dist/control-ui/index.html` is missing from the tarball.
  - file: [CHANGELOG.md:2202](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/CHANGELOG.md#L2202)
- **Issue thread confirms the correction release resolved this exact report:** A maintainer stated `v2026.3.23-2 is out and the UI is fine with that install`, and the reporter confirmed it.
