Keep open. Current main still lacks the PR's specific decision guidance for private/custom mobile clients and its safer per-conversation stable-session example, while the current security, protocol, and session-key behavior make that clarification relevant.

## Best Possible Solution

The best path is to rebase or manually transplant its decision guidance into `docs/gateway/openai-http-api.md`, preserving the paired-device wording and per-conversation `user` guidance while aligning the example with the current model-id docs if maintainers prefer `openclaw/default` over the compatibility alias `openclaw:main`.
