## Summary

Keep #14407 open. Current main has partial Control UI password/secret reveal behavior for gateway access and config redaction, but the requested cross-platform password visibility toggle is still missing from the Skills API-key entry flow and native iOS/Android gateway credential fields. The related PR #7124 is closed and unmerged in the provided GitHub context, and the recent maintainer-thread Codex review already identified the same remaining gaps.

## Best Possible Solution

Keep the issue open unless a maintainer explicitly narrows the platform scope. The likely implementation path is a reusable sensitive-input reveal control applied to Control UI Skills/API-key fields and gateway credential fields, Swift secure-field wrappers with reveal buttons for iOS/macOS onboarding/settings, Android Compose password fields using visualTransformation plus trailing icon toggles, and focused UI tests for each touched surface.
