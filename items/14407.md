## Summary

Keep #14407 open. Current main still only partially implements password/secret reveal controls in the web Control UI, while the requested cross-platform pattern remains missing from Skills API-key entry and several macOS, iOS, and Android credential surfaces. The linked PR #7124 is closed and unmerged in the provided GitHub context, so there is no shipped canonical fix to close against.

## Best Possible Solution

Keep the issue open and implement a reusable cross-platform sensitive-input reveal pattern: extend Control UI reveal controls to Skills/API-key fields, add SwiftUI reveal wrappers for macOS and iOS SecureField credential entries, add Android Compose password fields with PasswordVisualTransformation plus trailing visibility icons, and add focused UI/view tests for the touched credential surfaces.
