## Summary

Keep #14407 open. Current main has partial Control UI password/token reveal controls, but the requested cross-platform sensitive-input visibility toggle is still incomplete across Skills and native macOS/iOS/Android credential surfaces. The linked PR #7124 is closed and unmerged in the provided GitHub context, so there is no shipped canonical fix to close against.

## Best Possible Solution

Keep #14407 open unless a maintainer explicitly narrows the platform scope. The best product/code path is a reusable sensitive-input reveal pattern for Control UI Skills/API-key fields, SwiftUI reveal wrappers for macOS/iOS SecureField credential entries, Android Compose password fields with visualTransformation plus trailing icon toggles, and focused view tests or UI coverage for each touched credential surface.
