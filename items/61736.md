## Summary

Current main already implements async video-generation completion delivery, reports background-task failures back to the originating session instead of silently losing them, and uses the configurable `agents.defaults.mediaMaxMb` limit when saving generated videos. The same implementation is present in released `v2026.4.23`, so this issue is already fixed well enough to close.
