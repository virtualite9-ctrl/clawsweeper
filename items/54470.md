## Summary

Current main does not provide enough evidence to close this Windows Gmail setup crash. The reporter’s native PowerShell reproduction still matches a real code-path gap: Gmail setup accepts `gcloud` as installed through PATH/PATHEXT-style checks, but the actual setup commands still execute a bare `gcloud` argv through a Windows shim list that does not include `gcloud`.

## Best Possible Solution

Keep this issue open and fix the Gmail webhook setup command path. The likely maintainer-safe direction is to route external CLI dependencies used here (`gcloud`, `gog`, and `tailscale`) through a shared Windows-aware executable resolver or a narrow Gmail setup resolver, then add win32 regression tests proving `gcloud.cmd` in PATH is what setup executes without enabling shell parsing.
