## Summary

Current main still appears to have the reported native Windows Gmail setup command-resolution bug. The documented `openclaw webhooks gmail setup` path still reaches Gmail setup, dependency checks can accept Windows PATHEXT shims, but the later runtime execution still passes bare `gcloud`, `gog`, and `tailscale` to child processes without a general Windows PATH/PATHEXT resolver or explicit `.cmd` wrapping for these tools.

## Best Possible Solution

Keep this issue open. Resolve `gcloud`, `gog`, and `tailscale` through a Windows-aware PATH/PATHEXT executable resolver before spawning, or safely extend the shared exec shim for these concrete helper tools while preserving no-shell execution. Add win32 regression tests proving PATH-only `.cmd` shims work for Gmail setup, run, and watcher paths.
