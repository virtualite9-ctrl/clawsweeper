## Summary

Keep open. Current main does preserve `contextPruning.mode: "off"` and documents pruning as separate from compaction, so the cache-TTL override suspicion is largely answered. But the remaining user-facing request is not fully implemented: `agents.defaults.compaction.mode` still has no `off` or hard-limit-only mode, compaction remains active by default, and `agents.defaults.contextTokens` only caps the resolved model window rather than raising `claude-opus-4-6` to 1M for compaction threshold calculations.

## Best Possible Solution

Keep the issue open and handle it as a compaction-control/product fix: add an explicit, validated compaction policy such as `agents.defaults.compaction.mode: "off"` or `"hard-limit"`, thread it through the Pi settings/extension path without breaking overflow recovery, and document how Anthropic 1M context (`params.context1m` or newer model metadata) affects the context window used for compaction thresholds.
