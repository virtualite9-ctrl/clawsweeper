## Summary

Keep open. Current main still flattens remote markdown images to alt text in the Control UI renderer, and the current Control UI CSP/docs also intentionally block remote http(s) images from loading. The provided GitHub context shows open PR #41025 references this issue with closing syntax and remains unmerged, so the issue should stay open while maintainers decide whether to land, revise, or close that PR.

## Best Possible Solution

Keep the issue open and review #41025 as the implementation candidate, but require the decision to cover both the markdown renderer and the current Control UI CSP/docs. If maintainers want remote images, the fix likely needs an explicit policy change such as an allowlist, proxy/managed-media path, or documented opt-in rather than only allowing http(s) in the renderer.
