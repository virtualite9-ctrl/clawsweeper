## Summary

Keep open. Current main still flattens remote Control UI markdown images to alt text, the Control UI CSP still blocks remote image loads, and the docs explicitly describe remote http(s) image URLs as blocked. The provided GitHub context also shows open PR #41025 uses closing syntax for this issue and remains unmerged, so this issue should stay open while maintainers review, rework, or close that implementation candidate.

## Best Possible Solution

Keep this issue open and review PR #41025 as the implementation candidate. If maintainers want remote images in Control UI chat, the fix should handle renderer behavior, CSP, docs, and privacy policy together, likely via an explicit opt-in, allowlist, managed-media proxy, or similarly deliberate design. If maintainers prefer the current no-remote-fetch policy, close the issue and related PR with that product/security rationale.

## Risks / Open Questions

- A renderer-only fix would not solve the user-visible behavior because the current Control UI CSP still blocks remote `http(s)` image loads.
- Allowing model-supplied remote image URLs to render inline would make the operator browser fetch third-party URLs, with privacy/tracking implications that conflict with the current documented hardening unless maintainers intentionally change that policy.
