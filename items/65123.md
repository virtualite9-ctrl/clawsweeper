## Summary

Keep this PR open. Current main still returns the bare Discord target id from explicit-target parsing, so `user:<id>` can still be reduced to `<id>` before downstream delivery normalizes bare Discord ids as channels. The PR is a narrow Discord-owned fix with focused tests, and the related #72247 is closed/unmerged rather than a shipped replacement.

## Best Possible Solution

Keep this PR open for maintainer review or land an equivalent Discord-owned patch. The best fix is the narrow `target.id` to `target.normalized` change in `extensions/discord/src/channel.ts`, plus regression coverage in `extensions/discord/src/channel.test.ts` for `user:<id>`, `channel:<id>`, and ideally Discord mention inputs. #72247 should not supersede this PR because it is closed and unmerged while current main still has the bug.
