## Summary

Keep this PR open. Current main still drops the explicit Discord target kind by returning `target.id` from `parseDiscordExplicitTarget`, and the downstream delivery path can then normalize the bare numeric value as a Discord channel. The PR is a narrow Discord-owned parser fix with focused regression tests, and the related #72247 is closed/unmerged rather than a shipped replacement. Security review found no supply-chain-sensitive or unrelated infrastructure changes in the PR diff.

## Best Possible Solution

Keep this PR open for maintainer review or land an equivalent Discord-owned patch. The best fix is the narrow `target.id` to `target.normalized` change in `extensions/discord/src/channel.ts`, with regression coverage in `extensions/discord/src/channel.test.ts` for `user:<id>`, `channel:<id>`, and ideally Discord mention inputs. #72247 should not supersede this PR because it is closed and unmerged while current main still has the bug.
