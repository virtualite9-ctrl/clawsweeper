## Summary

Keep open. Current main treats `OPENCLAW_RAW_STREAM=0` and unset as disabled in the low-level writer, but the maintainer-facing problem is not fully resolved: docs still only explain how to enable raw stream logging, LaunchAgent service environment planning can still persist durable `.env`/config values, and there is no focused regression coverage for raw-stream disable behavior across managed-service restarts. The related #18740/#18753 work covered enabling via CLI flags, not this LaunchAgent disable-semantics report.

## Best Possible Solution

Keep this issue open and resolve it with a narrow docs-and-regression change: document that raw stream logging is enabled only by truthy values (`1`, `true`, `yes`, `on`) and disabled by unset or other values such as `0`, `false`, and `off`; explain how `launchctl unsetenv` differs from `launchctl setenv OPENCLAW_RAW_STREAM 0` when durable `~/.openclaw/.env`, `$OPENCLAW_STATE_DIR/.env`, or config `env` entries exist; add focused tests for `appendRawStream` disabled values and LaunchAgent/service-env planning. If a live macOS repro still appends with effective process env `OPENCLAW_RAW_STREAM=0`, fix that runtime/restart path as part of the same issue.
