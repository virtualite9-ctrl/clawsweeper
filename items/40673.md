## Summary

Keep this issue open. Current main has the low-level raw-stream truthy guard and newer managed-service env hardening, but the issue’s maintainer-facing request is still actionable: the public docs still only show how to enable raw-stream logging, they do not document disable values or LaunchAgent/source precedence, and there is no focused raw-stream/LaunchAgent regression proving post-disable appends stop under the managed-service path.

## Best Possible Solution

Keep this issue open and resolve it with a narrow docs-and-regression change: document that only `1`, `true`, `yes`, and `on` enable raw-stream logging; document that unset, `0`, `false`, `off`, and other values disable it; explain LaunchAgent precedence across `launchctl setenv/unsetenv`, state-dir `.env`, and config `env`; add focused tests for disabled `appendRawStream` values and managed-service/LaunchAgent env planning. If live macOS still appends when the effective gateway process env is `0` or absent and no durable source enables it, fix that restart/runtime path here.

## Risks / Open Questions

- Raw-stream logs can contain prompts, tool output, and private debugging data, so ambiguous disable behavior is a privacy and operator-trust risk.
- Operators can still be surprised by `launchctl unsetenv` if a durable source such as `~/.openclaw/.env`, `$OPENCLAW_STATE_DIR/.env`, or config `env` re-enables raw-stream logging and the raw-stream docs do not call that out.
