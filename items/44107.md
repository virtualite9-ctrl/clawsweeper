## Summary

Keep #44107 open. Current main has strong evidence that the large-history/markdown freeze side has been fixed or mitigated, including bounded chat.history responses, final-only history reload behavior, and the markdown-it ReDoS fix. But the issue explicitly reports two independent failures, and I found no current Control UI pre-module fallback or troubleshooting documentation for the <all_urls>/document_start Chrome-extension blank-page case. The later user comment about session switching showing an error icon also does not provide a close reason. Since a meaningful blank-dashboard failure remains unaddressed and no narrower canonical issue is provided for that part, conservative cleanup should not close this combined report.

## Best Possible Solution

Keep the issue open or split it: treat the large-history/markdown freeze portion as already fixed by the shipped history/markdown work, but add a focused Control UI fix or docs entry for the Chrome <all_urls>/document_start extension blank-page case. A minimal product path is an inline pre-module mount timeout fallback in ui/index.html plus public troubleshooting docs at docs.openclaw.ai, with a small browser/unit check where feasible. The Feishu/session-selector comment should be validated separately if maintainers see it as a distinct regression.
