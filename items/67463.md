## Summary

Keep this PR open. Current main already implements the task timestamp normalization half, but the doctor Gateway SecretRef false-positive remains present on main and is still tracked by open bug #65201. The PR still contains useful doctor work, but it needs cleanup before it is a safe maintainer merge because it resolves SecretRefs before confirming token auth is needed and breaks the existing lazy-loading shape with a top-level value import.

## Best Possible Solution

Keep this PR open, but ask for a narrow revision: rebase on current main, drop the already-implemented task timestamp portion and unrelated update-cli test churn unless still needed after rebase, dynamically import resolveGatewayAuthTokenForService inside runGatewayAuthHealth, gate SecretRef resolution behind the existing token-needed decision, then pass the resolved token as authOverride and keep the regression test proving a resolvable Gateway token SecretRef suppresses the doctor false-positive. This would resolve #65201 without adding avoidable doctor side effects.
