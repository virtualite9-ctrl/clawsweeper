## Summary

Keep #67463 open. Current main already covers the task timestamp normalization half, but the doctor gateway-auth health path still has the SecretRef false-positive shape: it detects a gateway token SecretRef, calls resolveGatewayAuth without resolving it, and then always emits the unavailable SecretRef warning when token auth is needed. The PR still addresses a real remaining bug, though it should be revised for the posted review concerns before merge.

## Best Possible Solution

Keep the item open and revise it rather than close it. Rebase or narrow the PR around the remaining doctor bug, resolve gateway.auth.token only after token auth is actually needed, keep the resolver import inside the lazy runGatewayAuthHealth path, pass the resolved token through authOverride, and keep/add regression coverage for both a successfully resolved SecretRef and a non-token mode that must not resolve external SecretRefs. The task timestamp changes can be treated as already present on main unless the PR still needs a small compatible test-only confirmation.
