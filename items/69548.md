## Summary

Keep PR #69548 open. Current main has generic cron stagger support, but memory-core dreaming still does not expose or propagate a `dreaming.staggerMs` setting, and the PR remains a focused feature with useful review follow-up rather than obsolete cleanup.

## Best Possible Solution

Keep the PR open and ask for a small update rather than closing it. The best path is to land a narrow memory-core implementation that adds `dreaming.staggerMs` to the manifest schema/UI hints, normalizes non-negative numeric and numeric-string values in `src/memory-host-sdk/dreaming.ts`, threads it through `resolveMemoryDeepDreamingConfig` and `ShortTermPromotionDreamingConfig`, includes it in the managed cron schedule and runtime config key, and adds focused tests/docs for explicit `0`, unset/blank values, custom stagger propagation, and the documented clear behavior.
