## Summary

Keep open. Current `main` does not implement `dreaming.staggerMs` in the memory plugin surface or its dreaming reconciliation path, so this PR is not obsolete. The existing cron infrastructure already supports `staggerMs`, which makes the change directionally aligned, but the PR discussion still has an acknowledged semantics gap around clearing a previously explicit stagger by unsetting the field.
