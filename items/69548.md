## Summary

Current `main` does not expose or pass through `dreaming.staggerMs`; this PR is still a relevant implementation candidate, not obsolete cleanup. The core cron layer already supports per-job `staggerMs`, so the remaining work is focused on memory-core config, SDK resolution, managed cron wiring, tests, and maintainer judgment on the unset/clear semantics raised in review.

## Best Possible Solution

Keep this PR open. If maintainers accept the feature, land a focused version that adds `dreaming.staggerMs` to the memory-core manifest/schema and docs, threads it through `resolveMemoryDreamingConfig`, `resolveMemoryDeepDreamingConfig`, `resolveShortTermPromotionDreamingConfig`, and the managed dreaming cron job builder, and adds focused tests for the edge cases already discussed in review. Maintainers should explicitly decide and document the clear/unset semantics before merge.
