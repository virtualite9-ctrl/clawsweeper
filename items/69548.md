## Summary

Keep #69548 open. Current main does not expose `dreaming.staggerMs` through the memory-core config surface or the dreaming cron job builder, while the underlying cron subsystem already supports `staggerMs`. The PR is not obsolete or implemented on main, but it still needs maintainer review around tests/docs and the known unset/clearing semantics discussed in review.

## Best Possible Solution

Keep the PR open and finish it as a focused memory-core pass-through: add `dreaming.staggerMs` to the manifest/schema/docs, normalize it in `src/memory-host-sdk/dreaming.ts`, carry it through `ShortTermPromotionDreamingConfig`, include it in the managed dreaming cron create/patch path, and add targeted memory-core/host tests. Before merge, decide whether removing the field should mean 'return to cron default' or whether operators must set `staggerMs: 0` explicitly to clear a prior stagger.
