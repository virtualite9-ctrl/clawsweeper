## Summary

Keep #69548 open. Current main still does not expose `dreaming.staggerMs` in the memory-core manifest/config helpers or pass it into the managed dreaming cron schedule, while the core cron subsystem already supports `schedule.staggerMs`. The PR remains a plausible focused implementation, with useful review follow-up already addressing the `staggerMs: 0` and blank-string parsing concerns; it still needs maintainer review/rebase and tests/docs around the remaining clear-on-unset behavior.

## Best Possible Solution

Keep the PR open and ask for a rebase onto current main plus targeted tests and docs. The best implementation path is to add `dreaming.staggerMs` to the memory-core manifest/schema, normalize non-negative numeric/string values in `src/memory-host-sdk/dreaming.ts`, pass it through `resolveShortTermPromotionDreamingConfig` into `buildManagedDreamingCronJob`, add regression coverage for unset, `0`, positive values, and blank strings, and update the public dreaming/memory config docs. Maintainers should explicitly decide whether removing the field should reset to cron defaults or whether users must set `staggerMs: 0` for exact timing.
