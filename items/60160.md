## Summary

Current main already ships the subagent parent-context capability through the supported `sessions_spawn` API as `context: "fork"`, with isolated-by-default behavior, bounded transcript forking, context-engine metadata, docs, tests, and changelog evidence. This supersedes the PR's unmerged `includeContext` shape; callers should use the shipped `context` parameter instead.

## Best Possible Solution

Close this PR as obsolete and keep the shipped `sessions_spawn({ context: "fork" })` implementation as the supported parent-context path. If maintainers later want model-generated summaries or cropped transcript injection specifically, that should be a fresh focused proposal against the current `context` contract and context-engine hooks.
