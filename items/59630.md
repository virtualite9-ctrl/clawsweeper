## Summary

Close as `clawhub`: the request is for a reusable workflow/spec layer on top of OpenClaw, and current `main` explicitly pushes that class of capability into skills/plugins/ClawHub rather than expanding core into a generic orchestration contract.

## Evidence

- **Project scope keeps reusable capability in plugins/skills, not core orchestration:** The vision doc says core should stay lean, prefer bundle-style plugins when they can express the capability, route plugin promotion through ClawHub, publish new skills through ClawHub first, and avoid heavy orchestration layers that duplicate existing agent/tool infrastructure.
