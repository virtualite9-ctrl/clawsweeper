## Summary

Keep PR #71941 open. Current main still has the documented-default mismatch: setup records `agents.defaults.workspace` as the main workspace, `resolveAgentWorkspaceDir` nests non-default agents under that fallback, and `agents add` uses that resolver directly for the wizard's Workspace directory initial value. The PR remains useful and focused, but the existing review comments about the portable HOME/OPENCLAW_HOME test setup and changelog attribution should be addressed before merge.

## Best Possible Solution

Keep #71941 open. The right maintainer path is to have the author address the two review comments, keep the fix narrowly scoped to the wizard's initial value for brand-new agents, preserve `resolveAgentWorkspaceDir` runtime semantics and #59789 shared-base behavior, then run the targeted agents tests plus the changed gate before merge.
