## Summary

Keep PR #71941 open. Current main still pre-fills the interactive `agents add` workspace prompt from `resolveAgentWorkspaceDir`, and that resolver still nests non-default agents under `agents.defaults.workspace`. The docs still describe the peer-level `workspace-<agentId>` convention, so the PR addresses a real remaining behavior gap rather than obsolete or duplicated work.

## Best Possible Solution

Keep the PR open. The best path is for the author to address the portable test stub and changelog attribution comments, then run the appropriate changed gate. If that passes, this PR is a reasonable targeted fix for the still-present #71889 behavior on current main.
