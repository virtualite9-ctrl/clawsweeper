## Summary

Current `main` does not implement this PR's behavior. The shared status snapshot still defines gateway reachability as `gatewayProbe?.ok === true`, so a scope-limited probe (`missing scope: operator.read`) is still treated as unreachable in status flows even though the gateway-status helpers and docs classify that case as reachable/degraded.

## Evidence

- **Shared status snapshot still uses strict probe.ok:** `resolveGatewayProbeSnapshot` sets `gatewayReachable` from `gatewayProbe?.ok === true`, not `isProbeReachable(...)`. That is now the central status path used by `status`/`status --all` overview code.
