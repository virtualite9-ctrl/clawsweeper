## Summary

Keep this PR open. Current main still only applies the Claude-name image-support fallback to `claude-cli`, not direct `anthropic`, while direct Anthropic Opus 4.7 remains a documented/default product path. The PR is a narrow implementation candidate with focused tests and no visible security-sensitive file changes in the provided diff.

## Best Possible Solution

Review or update this focused PR against current main. The likely best path is to land the direct `anthropic` Claude-name fallback with the two regression tests, or replace it with an explicit provider-owned image-capability seam if maintainers do not want more provider ids in the gateway resolver.
