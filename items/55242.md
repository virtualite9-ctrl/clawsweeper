## Summary

Keep open. Current `main` still builds Feishu card headers by concatenating `identity.emoji` directly into the header title, and that same logic is used for both streaming and non-streaming card replies. `identity.emoji` is still an unconstrained optional string in the outbound identity contract, so the report remains plausible and is not implemented/fixed on `main`.

## Evidence

- **Direct concat in outbound card send path:** The Feishu outbound card path still sets `title` to ``${identity.emoji} ${identity.name ?? ""}`.trim()`` whenever `identity.emoji` is present, with no emoji-only filtering or sanitization.
  - file: [extensions/feishu/src/outbound.ts:206](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/extensions/feishu/src/outbound.ts#L206)

  - sha: [c070509b7f8f](https://github.com/openclaw/openclaw/commit/c070509b7f8fdbe479e5d829643782c272e30e83)
- **Shared header builder still uses raw identity.emoji:** `resolveCardHeader()` trims `identity.emoji` and prepends it to the name verbatim, so any non-emoji descriptive text in that field becomes part of the card header title.
  - file: [extensions/feishu/src/reply-dispatcher.ts:76](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/extensions/feishu/src/reply-dispatcher.ts#L76)
