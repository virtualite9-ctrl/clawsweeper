## Summary

Current main does not implement the requested distinct system/runtime chat treatment. The pre-compaction memory flush path still submits the memory flush prompt as the visible prompt unless a caller supplies a separate empty transcript prompt, chat.history does not classify that prompt as a system/runtime block, the Control UI only has normal user/assistant/tool/other bubble handling, and the TUI history loader sends user-role history to the normal user component. The related #7532 cleanup only covers suppression of exec lifecycle notifications, not this broader distinct-rendering request.

## Best Possible Solution

Keep this issue open. The best fix is a generic runtime/system-message presentation path: either mark runtime-originated transcript entries with explicit provenance/kind or make internal-only prompts use an empty transcript prompt, then render system/runtime entries as a distinct muted/badged block in the Control UI and via a non-user system component in the TUI. Add focused coverage for memory-flush/runtime history projection, Control UI system rendering, and TUI history loading.
