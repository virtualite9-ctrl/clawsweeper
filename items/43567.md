## Summary

Keep this issue open. Current main still does not provide a distinct runtime/system presentation for the reported pre-compaction memory flush prompt: the memory plugin still defines the visible prompt, the memory flush runner still submits it as a normal embedded prompt without `transcriptPrompt: ""` or internal provenance, `chat.history` only strips narrower cases, and the Control UI/TUI do not route system-role history into a dedicated visible system block for this path.

## Best Possible Solution

Keep this issue open. Implement a structural runtime/system transcript contract for memory flush and similar internal turns, likely by passing `transcriptPrompt: ""` when no user-visible prompt should be persisted or by attaching `provenance.kind: "internal_system"` with a stable source. Then update `chat.history` to project that provenance explicitly, render it as a muted/badged system block in Control UI, route it through `addSystem` in the TUI, and add focused tests for memory-flush projection plus both UI surfaces.
