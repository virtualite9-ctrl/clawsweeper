## Summary

Keep PR #69056 open. Current main still has Windows-sensitive SIGUSR1 restart paths, and the PR addresses a real reported Windows failure, but it is not safe to close as implemented or as a duplicate: the latest PR head still has an unresolved token-consumption review blocker, and related PR #68853 overlaps without fully replacing this PR's no-listener/unmanaged restart surface.

## Best Possible Solution

Keep #69056 open for maintainer review, but do not merge it as-is. The best path is a maintainer-chosen combined fix that covers Windows no-listener and unmanaged restart paths, preserves the in-process listener path, marks or rolls back restart tokens correctly on every fallback outcome, and adds targeted Windows regression tests. If #68853 becomes the canonical restart-handoff PR, it should explicitly absorb this PR's Windows no-SIGUSR1 surface before #69056 is closed as superseded.
