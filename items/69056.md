## Summary

Keep PR #69056 open. Current main still has the Windows-unsafe no-listener restart fallback: `emitGatewayRestart()` uses `process.kill(process.pid, "SIGUSR1")`, while `scheduleGatewaySigusr1Restart()` reports mode from listener count only. The PR targets a real gap linked by #69057/#67416, but its review thread still raises token-consumption and coverage concerns, so it needs maintainer review or a replacement patch rather than cleanup closure.

## Best Possible Solution

Keep #69056 open for maintainer review or a maintainer replacement patch. The fix should preserve the listener-driven `process.emit("SIGUSR1")` path when a gateway run-loop listener exists, avoid `process.kill(..., "SIGUSR1")` on Windows when no listener exists, consume or roll back restart tokens on every Windows fallback outcome, report scheduler mode accurately, and add focused regression tests for Windows success, failure, and coalescing behavior. Coordinate with #68853 if maintainers decide that PR is the canonical restart-path fix.
