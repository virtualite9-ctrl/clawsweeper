## Summary

Current `main` already contains the late exec/PTY update guard for this crash, has regression tests for the settled/backgrounded/aborted cases, and the changelog shows it shipped in OpenClaw 2026.4.10.

## Evidence

- **Late update guard on current main:** `emitUpdate()` now returns early once the exec session is backgrounded, exited, or updates have been disabled, which is the exact protection needed to stop late stdout/stderr from calling into a disposed agent run.
  - file: [src/agents/bash-tools.exec-runtime.ts:568](https://github.com/openclaw/openclaw/blob/8b76392e3e7934b1cdd6d461148263fd56141ec8/src/agents/bash-tools.exec-runtime.ts#L568)
  - command: `nl -ba src/agents/bash-tools.exec-runtime.ts | sed -n '548,626p'`
  - sha: [8b76392e3e79](https://github.com/openclaw/openclaw/commit/8b76392e3e7934b1cdd6d461148263fd56141ec8)
- **Updates disabled before exit/error cleanup:** The exec runtime flips `updatesDisabled = true` before marking exit state, and also exposes `disableUpdates()` for abort handling, so queued late output is suppressed during the race window described in the report.
  - file: [src/agents/bash-tools.exec-runtime.ts:770](https://github.com/openclaw/openclaw/blob/8b76392e3e7934b1cdd6d461148263fd56141ec8/src/agents/bash-tools.exec-runtime.ts#L770)
  - command: `nl -ba src/agents/bash-tools.exec-runtime.ts | sed -n '764,826p'`
  - sha: [8b76392e3e79](https://github.com/openclaw/openclaw/commit/8b76392e3e7934b1cdd6d461148263fd56141ec8)
- **Abort path also suppresses updates:** The exec tool's abort handler now calls `run.disableUpdates()` before deciding whether to kill or leave a backgrounded session running, explicitly preventing post-abort `tool_execution_update` events from reaching pi-agent-core.
  - file: [src/agents/bash-tools.exec.ts:1715](https://github.com/openclaw/openclaw/blob/8b76392e3e7934b1cdd6d461148263fd56141ec8/src/agents/bash-tools.exec.ts#L1715)
  - command: `nl -ba src/agents/bash-tools.exec.ts | sed -n '1708,1742p'`
  - sha: [8b76392e3e79](https://github.com/openclaw/openclaw/commit/8b76392e3e7934b1cdd6d461148263fd56141ec8)
- **Regression tests cover the reported failure mode:** Tests now assert that `onUpdate` is not invoked after a session is backgrounded, after a foreground exec exits, or after an abort signal fires.
  - file: [src/agents/bash-tools.test.ts:896](https://github.com/openclaw/openclaw/blob/8b76392e3e7934b1cdd6d461148263fd56141ec8/src/agents/bash-tools.test.ts#L896)
  - command: `nl -ba src/agents/bash-tools.test.ts | sed -n '896,969p'`
  - sha: [8b76392e3e79](https://github.com/openclaw/openclaw/commit/8b76392e3e7934b1cdd6d461148263fd56141ec8)
- **Release note shows shipped fix:** The changelog entry under `## 2026.4.10` says OpenClaw now prevents the gateway crash `("Agent listener invoked outside active run")` when a subagent exec tool produces stdout/stderr after the agent run has ended or been aborted.
  - file: [CHANGELOG.md:902](https://github.com/openclaw/openclaw/blob/8b76392e3e7934b1cdd6d461148263fd56141ec8/CHANGELOG.md#L902)
  - command: `nl -ba CHANGELOG.md | sed -n '900,1011p'`
  - sha: [8b76392e3e79](https://github.com/openclaw/openclaw/commit/8b76392e3e7934b1cdd6d461148263fd56141ec8)
- **Issue discussion already pointed at the same root cause:** The issue comment from `@openperf` said the root cause was addressed by adding a `session.backgrounded || session.exited` guard in `emitUpdate()`. Current `main` contains that guard plus additional `updatesDisabled` hardening and tests.
