## Summary

Keep this issue open. Current main still merges the entire orphan user text into the next prompt and does not strip OpenClaw-generated queued-message markers or the aborted-run hint, so the reported snowball path is not fixed.

## Best Possible Solution

Keep the issue open and fix it in `src/agents/pi-embedded-runner/run/attempt.prompt-helpers.ts`. Add a focused regression test where the orphan leaf already contains the queued marker and aborted-run hint, then normalize only those OpenClaw-generated prefixes before merging so the latest orphan content is preserved and the leaf can still be removed safely.

## Evidence

There is no cleanup step for an orphan that already contains the queued marker or abort note.
