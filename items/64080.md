## Summary

Keep open. Current `main` still implements the opposite routing behavior from the PR for top-level Slack channel messages under `replyToMode="all"`, and the existing tests codify that behavior. There is also conflicting historical evidence in the changelog, so this needs explicit maintainer judgment rather than conservative auto-close cleanup.

## Evidence

- **Current routing still drops auto-threading for room/channel top-level turns:** `resolveSlackRoutingContext()` still computes `autoThreadId`, but for `isRoomish` conversations it only uses `roomThreadId` (`thread_ts` replies). Top-level channel messages therefore do not get a thread-scoped `canonicalThreadId` on current `main`.
