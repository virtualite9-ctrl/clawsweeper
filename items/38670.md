## Summary

Keep PR #38670 open. The PR proposes a real core config/runtime capability for agent-scoped internal hook policy overrides, and current main still does not implement it. The related `before_agent_reply` plugin hook and `hooks.allowedAgentIds` endpoint routing are useful adjacent work, but neither supersedes per-agent policy resolution for internal hook dispatch.

## Best Possible Solution

Keep PR #38670 open for maintainer judgment. If OpenClaw wants this capability, the best path is a core config/runtime change that defines strict policy types and schema for `hooks.internal.events`, `agents.defaults.hooks`, and `agents.list[].hooks`; threads config and runtime agent identity into internal hook dispatch; applies defaults only to real agent-scoped events; documents exact event matching; and adds focused tests for precedence, named hook gating, non-agent hooks, and backward compatibility. If maintainers reject the product direction, close it explicitly with that rationale rather than automated cleanup.
