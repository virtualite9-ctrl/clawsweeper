## Summary

Keep #72074 open. Current main does not implement a `labelTemplate`/`{taskPreview}` fallback for `sessions_spawn`; it only accepts an explicit `label` and otherwise passes no label through the subagent spawn path. The linked PR #72128 is relevant and appears to be the active implementation attempt, but it is open/unmerged, so the issue is not obsolete or implemented on main.

## Best Possible Solution

Keep the issue open and review or land a focused implementation such as #72128. The best product path is an additive core `agents.defaults.subagents.labelTemplate` plus optional per-agent override, explicit `sessions_spawn.label` precedence, sanitized `{agentId}` and `{taskPreview}` substitution, schema/generated metadata/docs updates, and focused tests for default, per-agent override, explicit-label precedence, omitted label, and malformed/long task preview behavior.
