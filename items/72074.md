## Summary

Keep #72074 open. Current main does not implement `agents.defaults.subagents.labelTemplate` or a per-agent equivalent, and `sessions_spawn` still forwards only an explicit `label` to native and ACP spawn paths. The linked PR #72128 is directly relevant and appears to propose the requested feature, but it is open and unmerged, so the issue still tracks missing behavior on main.

## Best Possible Solution

Keep the issue open and use #72128 or a maintainer-approved equivalent to add an additive subagent label-template contract. The implementation should define precedence clearly, validate the new config fields in default and per-agent schemas, synthesize fallback labels before session creation/registration, document supported tokens, and add focused tests for explicit-label precedence and default/per-agent template behavior.
