## Summary

Keep PR #38744 open. Current main still implements native iOS Talk Mode capture through Apple SFSpeechRecognizer, not the PR's ElevenLabs realtime WebSocket STT path, and the existing ElevenLabs realtime transcription implementation is scoped to the gateway/plugin Voice Call streaming surface rather than iOS Talk Mode. The PR also has unresolved reliability review findings, so it is not safe to close as implemented or superseded.

## Best Possible Solution

Keep the PR open for maintainer/product judgment. If the feature proceeds, rebase it onto current main and decide whether native iOS Talk Mode should own an ElevenLabs realtime STT client directly or consume a new gateway/native realtime transcription seam. Before merge, address WebSocket readiness, committed-transcript waiting for PTT, Apple Speech fallback permissions/recovery, iOS settings persistence, docs, and focused tests.
