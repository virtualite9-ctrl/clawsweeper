## Summary

Keep PR #38744 open. Current main still does not implement the PR's native iOS ElevenLabs realtime WebSocket STT path: iOS Talk Mode continuous and push-to-talk capture remain Apple SFSpeechRecognizer-backed. Main does have ElevenLabs Scribe v2 Realtime for Gateway/plugin Voice Call streaming STT, and a newer browser WebRTC realtime-voice session method, but neither is wired into native iOS Talk Mode capture. The related items are adjacent, not canonical superseders, and the PR still has unresolved reliability review findings.

## Best Possible Solution

Keep the PR open for maintainer/product judgment. If the feature proceeds, rebase it onto current main and decide whether iOS Talk Mode should own a native ElevenLabs STT client or consume a provider-generic gateway/native realtime transcription path. Before merge, address WebSocket readiness, committed PTT transcript waiting, Apple Speech fallback permission/recovery, PTT error recovery, settings persistence, docs, and focused iOS tests.
