## Summary

Keep this issue open. Current main has useful Web Push infrastructure from merged PR #44590 and a low-level Gateway RPC escape hatch, but it still does not provide the requested first-class `openclaw notify` command or a production-named admin-scoped broadcast/toast path to connected UI sessions without using the agent/event queue.

## Best Possible Solution

Keep the issue open and either add a production-facing `openclaw notify <message>` wrapper or explicitly design a `ui.notify`/`notifications.broadcast` Gateway method. It should reuse the existing Web Push infrastructure where appropriate, define the intended scope gate, decide whether connected Control UI clients also receive an in-app toast event, and add CLI/docs/tests for the shell-script workflow.

## Evidence

- **Related Web Push work is partial, not the requested CLI surface:** The provided timeline cross-references merged PR #44590, which added PWA/Web Push infrastructure for Control UI.
