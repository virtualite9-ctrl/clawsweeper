## Summary

Current `main` already routes tool-generated `MEDIA:` replies through the normal thread-aware outbound path, and Slack maps that thread context to `thread_ts` for media uploads, so the reported leak to the parent channel is implemented/fixed on `main`.

## Evidence

- **Tool media is still emitted as reply payload, not dropped:** Tool results with `MEDIA:` are parsed and queued into pending tool media for reply delivery.
  - file: [src/agents/pi-embedded-subscribe.handlers.tools.ts:536](https://github.com/openclaw/openclaw/blob/51c11cfd903033947f01f96a7169ebf92d194064/src/agents/pi-embedded-subscribe.handlers.tools.ts#L536)
  - sha: [51c11cfd9030](https://github.com/openclaw/openclaw/commit/51c11cfd903033947f01f96a7169ebf92d194064)
- **Queued tool media is flushed as a normal block reply:** Pending tool media is converted into a reply payload and emitted through the standard assistant reply path.
  - file: [src/agents/pi-embedded-subscribe.handlers.lifecycle.ts:170](https://github.com/openclaw/openclaw/blob/51c11cfd903033947f01f96a7169ebf92d194064/src/agents/pi-embedded-subscribe.handlers.lifecycle.ts#L170)
  - sha: [51c11cfd9030](https://github.com/openclaw/openclaw/commit/51c11cfd903033947f01f96a7169ebf92d194064)
- **Reply routing preserves threadId into outbound delivery:** The reply router resolves reply transport and passes the resulting `threadId` into `deliverOutboundPayloads`, so media-only replies use the same thread-aware outbound path as text replies.
  - file: [src/auto-reply/reply/route-reply.ts:198](https://github.com/openclaw/openclaw/blob/51c11cfd903033947f01f96a7169ebf92d194064/src/auto-reply/reply/route-reply.ts#L198)
  - sha: [51c11cfd9030](https://github.com/openclaw/openclaw/commit/51c11cfd903033947f01f96a7169ebf92d194064)
- **Session delivery resolution keeps same-route thread context:** `resolveSessionDeliveryTarget` keeps the session thread when the turn source matches the same route instead of dropping back to the root chat.
  - file: [src/infra/outbound/targets-session.ts:91](https://github.com/openclaw/openclaw/blob/51c11cfd903033947f01f96a7169ebf92d194064/src/infra/outbound/targets-session.ts#L91)
  - sha: [51c11cfd9030](https://github.com/openclaw/openclaw/commit/51c11cfd903033947f01f96a7169ebf92d194064)
- **Regression coverage matches the reported failure mode:** Tests explicitly cover the regression where replies landed in the root chat because `turnSourceThreadId` was absent, and assert that the thread id is preserved on same-channel routing.
  - file: [src/infra/outbound/targets.test.ts:819](https://github.com/openclaw/openclaw/blob/51c11cfd903033947f01f96a7169ebf92d194064/src/infra/outbound/targets.test.ts#L819)
  - sha: [51c11cfd9030](https://github.com/openclaw/openclaw/commit/51c11cfd903033947f01f96a7169ebf92d194064)
- **Slack media sends use thread_ts:** Slack outbound maps `threadId` to `threadTs`, and upload coverage asserts `completeUploadExternal` receives `thread_ts` for media delivery.
  - file: [extensions/slack/src/outbound-adapter.ts:87](https://github.com/openclaw/openclaw/blob/51c11cfd903033947f01f96a7169ebf92d194064/extensions/slack/src/outbound-adapter.ts#L87)
