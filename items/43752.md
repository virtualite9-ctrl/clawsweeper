## Summary

This PR remains a plausible focused bug fix. Current main forwards the effective gateway port into gateway lock acquisition, but the lock payload schema and writer still omit `port`, so external tools that discover gateways from lock files still cannot read the actual listening port from the lock payload.

## Best Possible Solution

Keep this PR open. Rebase it onto current main, add `port?: number` to `LockPayload` and `LockPayloadSchema`, retain the 1-65535 write guard, and land focused `gateway-lock` tests that prove provided-port persistence and absent-port compatibility.
