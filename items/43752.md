## Summary

Current main has the effective gateway port available at lock acquisition time, but the lock payload type, schema, and writer still omit `port`. The PR remains a plausible focused bug fix, though it needs a rebase/update for the current Zod-backed `LockPayloadSchema`. The provided PR diff is narrow and does not introduce an observable security or supply-chain concern.

## Best Possible Solution

Keep this PR open. Rebase it onto current main, add `port?: number` to both `LockPayload` and `LockPayloadSchema`, keep the 1-65535 write guard, and land focused `gateway-lock` tests proving provided-port persistence and absent-port compatibility.
