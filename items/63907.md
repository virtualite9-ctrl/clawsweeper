## Summary

Current `main` does not show this issue as fixed. The top-level CLI command catalogs expose `agent` for one-shot agent turns, not `run`, and the lazy unknown-command path still converts an unrecognized primary command into a `plugins.allow` error when an allowlist is present. That matches the reported bad behavior closely enough that this should stay open. The cross-reference to PR #64112 also points at an active fix direction rather than a resolved one.

## Evidence

- **Unknown commands still fall through to plugin-allowlist messaging:** When the primary command is still unregistered after lazy plugin registration, `runCli()` calls `resolveMissingPluginCommandMessage(primary, config)` and throws that result. `resolveMissingPluginCommandMessage()` returns the `plugins.allow excludes "<command>"` message for any non-empty allowlist entry that does not include the unknown command name.
