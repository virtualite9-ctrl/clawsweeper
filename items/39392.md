Current main still does not contain the focused Control UI fix for #39392. The issue has concrete reproduction details, related PRs show the intended repair, and the current Agents Overview render path still lacks both a selected-agent remount boundary and explicit selected-option stabilization for the primary model select.

## Best Possible Solution

Keep #39392 open until a focused Control UI fix lands on main. The likely repair belongs in the Agents Overview path: either key/remount the overview by selectedAgent.id or bind the select and options to one explicit selected primary value, then add a regression test that switches main -> another agent -> main and verifies the displayed Primary model matches the selected agent's resolved config. PR #39401 remains the closest open candidate; #70633 is useful context but its selectedPrimary approach is not present on current main.
