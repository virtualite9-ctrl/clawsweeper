## Summary

Keep #69251 open. Current main still uses the reported POSIX `/bin/sh -lc` node shell wrapper, still routes node exec through `system.run.prepare`, and still fail-closes shell payloads whose first token resolves to a mutable/script-like file before `security=full` + `ask=off` can matter. The #69308 LaunchDaemon fix is related context but does not address this approval-binding path.

## Best Possible Solution

Keep the issue open and fix the node exec approval path directly. The likely implementation should either produce a safe node-local binding for script/interpreter file operands hidden behind the POSIX `/bin/sh -lc` transport wrapper, or add a narrow, documented, regression-tested exception for explicit `security=full` plus `ask=off`. Add coverage for direct script invocation, interpreter-with-file-arg, and nested shell/interpreter forms while preserving fail-closed behavior for ambiguous mutable shell payloads.
