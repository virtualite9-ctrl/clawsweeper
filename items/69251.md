## Summary

Keep #69251 open. Current main still uses the reported POSIX `/bin/sh -lc` node transport wrapper, still sends node exec through `system.run.prepare` before the later policy/approval path, and still has planner/test coverage that fail-closes shell payloads pointing at script-like or mutable files with `SYSTEM_RUN_DENIED: approval cannot safely bind this interpreter/runtime command`. The #69308 LaunchDaemon fix is useful related context, but it only addresses non-interactive `SHELL` fallback and does not change this approval-binding path.

## Best Possible Solution

Keep the issue open and fix the node exec approval path directly. The likely product/code path is to make `system.run.prepare` either produce a safe node-local binding for script/interpreter file operands hidden behind the POSIX `/bin/sh -lc` transport wrapper, or add a narrow, documented, regression-tested exception for explicit `security=full` plus `ask=off`. Add coverage for direct script invocation, interpreter-with-file-arg, and nested shell/interpreter forms while preserving fail-closed behavior for ambiguous mutable shell payloads.
