## Summary

Keep #33478 open. Current main partially preserves announce thread/topic routing, but the central structured `sessions_send` skip/action contract is still missing. The A2A flow remains driven by exact `REPLY_SKIP` / `ANNOUNCE_SKIP` text, and a maintainer comment explicitly says to keep this issue as acceptance criteria for #63789.

## Best Possible Solution

Keep #33478 open as acceptance criteria for the A2A handoff protocol. The best path is to implement a structured `sessions_send` outcome/action contract, or the equivalent first-party handoff protocol from #63789, that supports skip/no-reply/announce outcomes without exact model text, preserves channel/thread context, keeps legacy tokens as compatibility behavior, and adds regression tests for chatter around `REPLY_SKIP` plus threaded announce routing.
