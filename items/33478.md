## Summary

Keep #33478 open. Current main has partial thread/topic preservation for A2A announce delivery, but the main request is still not implemented: `sessions_send` has no structured `action`/`announceTarget`/`announceThreadId` contract, and the A2A lifecycle still relies on exact `REPLY_SKIP` and `ANNOUNCE_SKIP` model text. A maintainer comment also explicitly says to keep this as acceptance criteria for #63789.

## Best Possible Solution

Keep #33478 open as acceptance criteria for the first-party A2A handoff protocol. The best implementation path is to add a structured `sessions_send` outcome/action contract, or fold the equivalent into #63789, covering skip/no-reply/announce outcomes without exact model text, preserving channel/thread context, retaining legacy token compatibility, and adding regression tests for chatter around `REPLY_SKIP` plus threaded announce routing.
