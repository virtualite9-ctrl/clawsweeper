## Summary

Keep #33478 open. Current main still lacks the structured sessions_send action/override contract and still uses exact REPLY_SKIP / ANNOUNCE_SKIP text matching for A2A control flow. Thread announce routing has been partially repaired and tested, but the remaining structured outcome API is explicitly called out by a maintainer as acceptance criteria for the broader A2A handoff work.

## Best Possible Solution

Keep #33478 open as acceptance criteria for first-party A2A handoff work, likely coordinated with #63789, #27445, and related dispatch/announce routing issues. The best implementation is a structured sessions_send outcome/action contract covering skip, no-reply, announce suppression, explicit routing overrides, legacy token compatibility, and regression tests for token chatter plus threaded announce routing.
