## Summary

Keep PR #69494 open. Current main still resolves local macOS gateway auth only from string values, so a SecretRef dictionary in `gateway.auth.token` or `gateway.auth.password` becomes `nil`; the browser proxy then has no auth header to send. The PR is an active, focused fix for a real bug, but the latest maintainer review still asks for one regression test through the actual `localConfig()` / browser-proxy endpoint path before merge.

## Best Possible Solution

Keep the PR open and ask the author to finish the maintainer-requested regression test: thread an internal/defaulted `SecretInputResolver` through `localConfig(root:env:launchdSnapshot:tailscaleIP:)` and `_testLocalConfig(...)`, then assert SecretRef token/password resolution through the sync localConfig/browser-proxy endpoint path. After that, keep remote token/password behavior intentionally string-only unless maintainers choose a broader product change, and merge once the macOS checks remain green.
