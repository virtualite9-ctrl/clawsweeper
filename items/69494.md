## Summary

Keep open. Current `main` still does not resolve SecretRef-backed `gateway.auth.token` / `gateway.auth.password` in the macOS browser-proxy path, and the PR discussion shows the proposed fix is not yet complete enough to merge conservatively.

## Evidence

- **Current main only reads local password as a string:** `resolveGatewayPassword` checks `gateway.auth.password` with `as? String` and otherwise falls back to launchd; a structured SecretRef object would resolve to `nil` in local mode.
  - file: [apps/macos/Sources/OpenClaw/GatewayEndpointStore.swift:112](https://github.com/openclaw/openclaw/blob/2cd2732ab6695a4b3220ce41166b74aca7097c5f/apps/macos/Sources/OpenClaw/GatewayEndpointStore.swift#L112)

  - file: [apps/macos/Sources/OpenClaw/GatewayEndpointStore.swift:189](https://github.com/openclaw/openclaw/blob/2cd2732ab6695a4b3220ce41166b74aca7097c5f/apps/macos/Sources/OpenClaw/GatewayEndpointStore.swift#L189)
