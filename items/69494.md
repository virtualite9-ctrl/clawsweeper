## Summary

Keep open. Current main still resolves macOS app browser-proxy gateway auth as plain strings only, so structured SecretRef values for `gateway.auth.token` or `gateway.auth.password` still drop to nil in the `localConfig()` path used by `MacNodeBrowserProxy`. The PR is active, directly targets that code path, and maintainer discussion already confirms the root cause as real while asking for merge-shape/test tightening rather than closure.

## Best Possible Solution

Keep #69494 open and review/land it, or an equivalent patch, after final macOS CI/review. The best implementation path is the PR's current direction: add an app-side SecretInput resolver seam for local `gateway.auth.token` and `gateway.auth.password`, keep remote auth behavior unchanged unless explicitly product-approved, avoid blocking actor/main endpoint refresh on keychain lookup, and retain regression coverage for the actual `localConfig()` path used by the browser proxy.
