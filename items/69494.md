## Summary

Keep PR #69494 open. Current main still drops macOS app browser-proxy auth when local `gateway.auth.token` or `gateway.auth.password` is a structured SecretRef, while the PR head contains a focused implementation plus regression coverage for that exact path and relevant CI is green.

## Best Possible Solution

Keep PR #69494 open and continue the normal maintainer review/merge path. The best code path is to land this PR or an equivalent macOS app fix that resolves local `gateway.auth.token` and `gateway.auth.password` SecretRef dictionaries in the browser-proxy `localConfig()` path, keeps env and launchd precedence intact, and preserves the new regression coverage.
