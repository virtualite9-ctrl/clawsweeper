## Summary

Keep #69494 open. Current main still resolves the macOS browser-proxy gateway token/password path as plain strings only, while the PR head remains separate and targets a real documented SecretRef gap for `gateway.auth.token` and `gateway.auth.password`.

## Best Possible Solution

Keep #69494 open and review/land it, or an equivalent patch, after final macOS CI. The best shape remains an app-side SecretRef resolver seam for local `gateway.auth.token` and `gateway.auth.password`, support for the explicit `source: "exec", provider: "keychain"` lookup, no plaintext persistence, no remote-auth behavior change without product approval, non-blocking/bounded keychain lookup, and regression coverage for the actual `localConfig()` path used by `MacNodeBrowserProxy`.
