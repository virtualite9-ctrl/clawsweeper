## Summary

Keep open. Current main still resolves google-vertex ADC to the non-secret `gcp-vertex-credentials` marker, stores/injects that marker as a runtime API key for provider-owned transports, and the bundled Google transport serializes any non-OAuth JSON string as `x-goog-api-key`. The issue has fresh user confirmation on v2026.4.23, and related prior ClawSweeper reviews found the same ADC leak still present rather than implemented away.

## Best Possible Solution

Keep the issue open and fix google-vertex ADC as provider-owned runtime auth rather than an API-key placeholder: stop passing `gcp-vertex-credentials` into runtime auth storage or `options.apiKey`, produce the correct Vertex/ADC credential behavior for the Google transport, and add regression coverage proving ADC does not emit `x-goog-api-key`. Separately verify the Node 25/gaxios path against current `@google/genai`/`gaxios` and either pin/patch/upgrade or route google-vertex requests through a transport that uses native fetch safely.
