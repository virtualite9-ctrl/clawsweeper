## Summary

Keep open. Current main still has a concrete Google Vertex ADC auth bug: `google-vertex` ADC resolves to the non-secret `gcp-vertex-credentials` marker, the runner stores/passes that marker as the runtime API key when no provider runtime-auth hook replaces it, and the bundled Google transport serializes non-JSON auth strings as `x-goog-api-key`. The issue is also recent and has another affected-user confirmation on v2026.4.23, plus several related ClawSweeper reports that reached the same keep-open conclusion for the same ADC marker leak. The Node 25/gaxios crash may need a separate dependency-level fix or verification, but the ADC half alone is enough to keep this issue open.

## Best Possible Solution

Keep this issue open and fix it in the Google provider/auth path: add a provider-owned `prepareRuntimeAuth` or equivalent Google Vertex ADC path that exchanges/uses ADC credentials without storing or passing `gcp-vertex-credentials` as a request API key, ensures Vertex requests target the Vertex endpoint with project/location, and adds regression coverage proving no `x-goog-api-key: gcp-vertex-credentials` header is emitted. Separately verify or patch the Node 25/gaxios fetch crash through the Google dependency/runtime boundary.
