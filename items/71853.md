## Summary

Keep open. The issue is recent, has a corroborating user comment, and current main still contains the reported Google Vertex ADC marker path. The marker is resolved as the runtime credential and the Google transport turns any non-JSON credential into `x-goog-api-key`; the bundled Google provider does not implement a Vertex ADC `prepareRuntimeAuth` exchange. The Node 25 dependency concern is also not safely obsolete because the locked gaxios package still falls back to dynamic `node-fetch` import instead of `globalThis.fetch`.

## Best Possible Solution

Keep the issue open and fix it in the bundled Google provider/runtime auth path. The likely product/code path is to add google-vertex-specific ADC runtime auth preparation that exchanges ADC for the correct bearer-token/project/location/Vertex endpoint configuration, strips the synthetic `gcp-vertex-credentials` marker before any request transport, and adds regression tests for ADC marker handling. Separately, either update/patch the `@google/genai`/gaxios path to use `globalThis.fetch` or avoid that SDK path on Node 25+ where it crashes.
