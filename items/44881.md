## Summary

Keep open. Current main has strong adjacent update hardening through `openclaw update`, stale-listener cleanup, and clearer EADDRINUSE guidance, but the reported raw `npm install -g openclaw@latest` + PM2 path is not implemented as an automatic stop/restart or PM2-aware warning. The issue discussion also adds a PM2 respawn-chain case that current gateway lifecycle code does not explicitly handle.

## Best Possible Solution

Keep the issue open and narrow the remaining work: either document/directly warn that externally supervised PM2 gateways must be stopped through PM2 before raw package-manager upgrades, or add PM2 detection/guidance to `gateway stop`, startup EADDRINUSE diagnostics, and update docs. The broader raw-npm auto-restart question can be coordinated with #45943, but this PM2-specific reproduction should not be auto-closed from current main.
