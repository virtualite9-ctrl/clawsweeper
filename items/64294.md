## Summary

Keep this PR open. Current main still lacks an injected `api.runtime.llm.complete()` surface and does not add an LLM helper to context-engine runtime contexts. The PR is a plausible implementation candidate for the open SDK/API request, but it exposes a new model-inference path for plugins and context engines, so it needs maintainer API/security review rather than conservative cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review or replacement. The best path is an additive plugin SDK LLM completion seam with clear active-agent binding, permission/cost/observability semantics, docs at `docs/plugins/sdk-runtime.md`, exported SDK types, and focused tests for model/auth resolution. Keep #45865 as the canonical product/API discussion if maintainers choose a different API shape.

## Risks / Open Questions

- The PR exposes a new path for plugins and context engines to spend model tokens and use configured model credentials, so maintainers need an explicit permission, observability, and cost-attribution decision.
- The PR's default-agent fallback for omitted `agentId` can use the wrong model/auth profile in multi-agent sessions unless the API binds the active session agent or requires explicit agent binding.
- The proposed public SDK surface still needs docs, API/export baseline coverage, and focused tests for default/non-default agent model and auth-profile resolution.
- The security review found no CI workflow, dependency, package-manager, or supply-chain file changes in the PR diff; the main risk is the new trusted runtime inference capability itself.
