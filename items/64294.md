## Summary

Keep this PR open. Current main still lacks the proposed Plugin SDK LLM completion surface and context-engine LLM helper, while #45865 remains the canonical open feature request. The PR is a plausible implementation candidate but needs maintainer API/security review before landing.

## Best Possible Solution

Keep this PR open for maintainer review or replacement by a revised implementation. The best path is an additive api.runtime.llm.complete() seam with exported SDK types, context-engine injection only where session/agent binding is well-defined, docs/API-baseline updates, and focused tests for active-agent model/auth resolution. Keep #45865 as the canonical product/API discussion if the API shape changes.

## Risks / Open Questions

- The proposed public SDK surface needs maintainer design review for capability shape, compatibility, documentation, and API baseline coverage.
- The PR expands what plugin/context-engine code can do by initiating LLM calls; that needs an explicit permission/cost/observability stance before merge.
- The current PR discussion already identified a multi-agent binding concern: defaulting to the config default agent can route completions to the wrong model or credentials unless active session agent resolution is handled clearly.
