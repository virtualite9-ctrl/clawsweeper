## Summary

Keep this PR open. Current main still does not implement `--daemon-env` for `openclaw onboard --install-daemon`, and the previously closed feature issue was closed specifically because this PR was the canonical implementation candidate. The PR is not cleanup-obsolete, but it needs maintainer/security review before merge because the proposed extra-env merge happens after existing daemon host-env filtering.

## Best Possible Solution

Keep this PR open for maintainer/security review. The best implementation is either a narrow reviewed opt-in for the known `OPENCLAW_ALLOW_INSECURE_PRIVATE_WS=1` bootstrap case, or a shared daemon-env parser/normalizer that validates portable keys, rejects dangerous host-env names using `host-env-security`, warns or fails on malformed input, and includes focused CLI/install tests plus docs in `docs/cli/onboard.md` and `docs/reference/wizard.md`.
