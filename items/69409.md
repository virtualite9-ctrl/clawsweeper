## Summary

Keep #69409 open. Current main still has no approval/manual queue for restart-required direct config-file changes: default `hybrid` mode automatically schedules a gateway restart, and the available alternatives are blunt modes like `hot` or `off`, not the requested notify/approve workflow. The later Slack 408/reconnect crash comment appears to be a separate recovery bug report/workaround, not evidence that the restart-approval feature is implemented or superseded.

## Best Possible Solution

Keep the issue open for maintainer product/design work in the core gateway reload path. A likely implementation would add a core `gateway.reload` restart policy such as `auto | notify | manual`, persist or expose a pending restart intent, add CLI/RPC approval commands, integrate notification delivery through existing admin/channel approval surfaces, include actor/audit metadata for direct file writes where possible, and cover direct watcher changes to restart-required paths like `plugins.allow` with tests and docs.
