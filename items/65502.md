## Summary

Keep open. Current main still lacks the requested configurable per-candidate retry policy and fallback context safe/strict mode. Model fallback attempts each candidate once, the public agent model config only accepts primary/fallbacks/timeoutMs, and escaped context-overflow errors still stay out of the fallback chain rather than being compacted to fit a smaller fallback model.

## Best Possible Solution

Keep this issue open and design a narrow agents fallback policy on the existing agent model config surface. The implementation should add bounded per-candidate retries for classified transient failures, record retry metadata in model-fallback observations/tests, and define fallback context safe/strict semantics that compact or refuse smaller-context candidates without replaying side effects. Use the closed unmerged #60984 as prior discussion, but align the final shape with current runEmbeddedPiAgent overflow recovery, config-schema conventions, docs, and regression tests.

## Risks / Open Questions

- Retrying a candidate after a partial model response, tool call, or outbound message could replay side effects unless the implementation gates retries on a safe failure phase.
- Fallback context safe mode needs a clear transcript/compaction contract so a smaller-context model is not fed history it cannot process and does not corrupt the session state.
