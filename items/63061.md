## Summary

Keep open. Current main still contains the profile/OPENCLAW_HOME state-dir mismatch this PR targets, and the PR remains a focused implementation candidate. The remaining maintainer decision is product precedence: whether OPENCLAW_HOME should supersede profile suffixing everywhere, with docs and daemon/service helpers aligned.

## Best Possible Solution

Keep this PR open for maintainer review. Decide the intended precedence between OPENCLAW_HOME physical isolation and named profile suffixing, then land a normalized version if that precedence is correct, with docs/help updates and a quick pass over daemon/service path helpers so CLI, foreground gateway, and installed service behavior stay aligned.
