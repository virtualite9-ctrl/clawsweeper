## Summary

Keep this PR open. Current main still has the CLI/profile behavior the PR is trying to change: `OPENCLAW_HOME` plus a non-default profile still derives `.openclaw-<profile>`, while the config path resolver’s default `OPENCLAW_HOME` state dir is `.openclaw`. The provided PR context shows a focused implementation candidate, and the later commit appears to address the bot feedback by normalizing the `OPENCLAW_HOME` guard and adding the dev-profile test.

## Best Possible Solution

Keep this PR open for maintainer review. The likely product path is to land a normalized version of this fix if `OPENCLAW_HOME` is meant to be the physical isolation boundary, with matching docs/help updates or an explicit maintainer decision that the existing documented profile suffix behavior should remain authoritative.
