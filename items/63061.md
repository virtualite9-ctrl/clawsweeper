## Summary

Keep this PR open. Current main still has the behavior the PR targets: CLI profile setup suffixes the state directory under OPENCLAW_HOME, while the config/state resolver treats OPENCLAW_HOME as the home root and defaults to an unsuffixed .openclaw. The latest provided PR diff remains a focused implementation candidate and incorporates the earlier review feedback for normalized OPENCLAW_HOME handling and dev-profile coverage.

## Best Possible Solution

Keep this PR open for maintainer review. If OPENCLAW_HOME physical isolation should take precedence over named-profile suffixing, land a normalized version of this PR with matching CLI/help/docs and daemon/service-path alignment. If named profiles must remain separate even inside OPENCLAW_HOME, reject the PR and document that profile suffixing wins under OPENCLAW_HOME.
