## Summary

Keep open. Current main still has the behavior this PR targets: CLI profile setup suffixes the state dir under OPENCLAW_HOME, while the generic config/state resolver treats OPENCLAW_HOME as the home root and defaults to an unsuffixed .openclaw. The PR remains a focused implementation candidate, with prior review feedback reflected in the provided latest diff.

## Best Possible Solution

Keep this PR open for maintainer review. Decide the intended precedence between OPENCLAW_HOME physical isolation and named profile suffixing; if OPENCLAW_HOME should win, land a normalized version of this PR with matching docs/help and daemon/service-path alignment, otherwise document that profile suffixing wins under OPENCLAW_HOME and close the PR after that product decision.
