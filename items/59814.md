## Summary

Current `main` and the latest shipped code keep Mattermost setup on the public setup-safe SDK surface, and `createSetupInputPresenceValidator` is exported again from `openclaw/plugin-sdk/setup-runtime`. The changelog also records a shipped bundled-channel setup fix for Mattermost in `v2026.4.8`, so this specific 2026.4.2 regression appears resolved.
