## Summary

Keep open. Current `main` still builds separate `stream` items for each live segment and renders each one as its own assistant chat group, which matches the reported multi-bubble streaming behavior. The issue is recent, the provided discussion points to PR #63986 as a proposed fix, but there is no strong local evidence that fix landed on `main`.

## Evidence

- **Streaming segments still become separate chat items:** `buildChatItems()` appends one `kind: "stream"` item per entry in `streamSegments`, then may append another live `stream` item for the current aggregate stream. That means multiple in-flight assistant render units can exist at once.
  - file: [ui/src/ui/chat/build-chat-items.ts:271](https://github.com/openclaw/openclaw/blob/5b8bd6371c666a5de43c82305e0315771e8e49d5/ui/src/ui/chat/build-chat-items.ts#L271)

  - sha: [5b8bd6371c66](https://github.com/openclaw/openclaw/commit/5b8bd6371c666a5de43c82305e0315771e8e49d5)
- **Non-message items are not grouped together:** `groupMessages()` flushes the current group whenever it encounters a non-`message` item, so separate `stream` items do not merge into a single assistant group during rendering.
  - file: [ui/src/ui/chat/build-chat-items.ts:154](https://github.com/openclaw/openclaw/blob/5b8bd6371c666a5de43c82305e0315771e8e49d5/ui/src/ui/chat/build-chat-items.ts#L154)
