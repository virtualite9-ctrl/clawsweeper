## Summary

Current main still has the reported Control UI streaming layout path: tool-boundary text is stored as stream segments, `buildChatItems` emits each segment plus the live stream as separate `kind: "stream"` items, and every stream item renders as its own full assistant `.chat-group`. The linked fix PR #63986 was closed unmerged, and current main has no `stream-run` or equivalent grouping implementation.

## Best Possible Solution

Keep this issue open and fix the Control UI rendering path. The best direction is to represent one in-flight assistant run as one outer assistant group, interleaving committed stream segments, visible tool cards, and the live tail inside that group; when tool calls are hidden, merge segment text and live text into one updating bubble. Add focused coverage in `ui/src/ui/chat/build-chat-items.test.ts` or an unmocked chat rendering test asserting segmented live streaming produces exactly one outer `.chat-group.assistant`. PR #63986 and the still-relevant related duplicate-bubble PR context are useful implementation references, but current main has not shipped the fix.
