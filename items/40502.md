## Summary

Keep this PR open. Current main still only forwards an explicitly configured numeric temperature and does not implement the PR's central behavior of lowering default temperatures for tool-enabled or shell-like agent runs. The PR is a plausible implementation candidate, but it needs maintainer review and a narrow rebase because the provided review history still shows agent-runner edge cases around thinking, cache retention, and compaction, plus security-sensitive stale diff surfaces that should not be mixed into the agent change.

## Best Possible Solution

Keep this PR open, but require a narrow rework before merge: rebase on current main, drop any stale unrelated secrets/config/lockfile/changelog changes, implement the dynamic temperature heuristic through the agent extra-params/runtime-plan path for both normal attempts and compaction, preserve explicit user temperatures, skip or adapt defaults when provider thinking constraints require it, avoid unintended cache-retention changes, and add focused tests for plain chat, generic tools, shell-like tools, explicit temperature, thinking-enabled cases, and compaction.
