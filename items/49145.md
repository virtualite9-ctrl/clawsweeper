## Summary

Keep #49145 open. Current main does not implement the PR's central Signal behavior: Signal inbound context forwards quote body/sender but not `ReplyToId`, Signal send options/RPC params have no reply or quote fields, reactions enqueue a system event without an immediate heartbeat wake, and the Signal channel does not implement the `sendPayload` path needed for first-chunk-only quote handling. The related PRs in the provided context are closed/unmerged or superseded by this PR, not canonical shipped fixes.

## Best Possible Solution

Keep #49145 open. The best maintainer path is to land or request a narrowed, tested Signal implementation that adds visibility-gated inbound `ReplyToId`, verified quote author extraction, outbound `quote-timestamp`/`quote-author` support, immediate reaction wake through an approved runtime seam, and first-chunk-only quote handling, while splitting unrelated core/test fixes out of the PR.
