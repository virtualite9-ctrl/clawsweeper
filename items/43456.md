## Summary

Keep this issue open. Current main still treats Mattermost thread roots as routing/session context only; it does not verify that the replied-to/root post was authored by the bot or feed a reply-to-bot implicit mention into the oncall mention gate.

## Best Possible Solution

Implement this in the bundled Mattermost plugin: add safe referenced-post resolution for root_id/parent reply metadata, verify the referenced post user_id matches the Mattermost bot user id, pass a reply_to_bot implicit mention fact into the Mattermost/shared mention gate, and add regression tests for unmentioned threaded and inline replies in default oncall mode.
