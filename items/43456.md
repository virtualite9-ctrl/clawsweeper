## Summary

Keep open. Current main still gates Mattermost `oncall` channel/group posts on explicit mention, authorized command bypass, or `onchar`; it records `root_id` for thread routing but does not verify that the referenced post was authored by the bot or pass an implicit `reply_to_bot` mention into the wake decision. The closed related #45284 covered outbound auto-threading, not this inbound reply-wake behavior.

## Best Possible Solution

Keep this issue open and implement the behavior inside the bundled Mattermost plugin. The likely path is to parse the relevant thread/reply reference, fetch or cache the referenced post when needed, verify it was authored by the Mattermost bot user, feed `reply_to_bot` into the shared mention-gating helper for `oncall`, and add regression tests for threaded and inline replies without an explicit @mention.
