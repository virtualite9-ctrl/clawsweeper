## Summary

Keep open. Current main still hashes normal exec outcomes through the generic `{ details, text }` fallback, so volatile exec metadata such as `durationMs`, `pid`, `startedAt`, `sessionId`, and `cwd` can keep repeated successful exec calls from accumulating an identical no-progress streak. The issue discussion also includes active related work: PR #41502 for the narrow stable exec outcome hash and #54559 for broader generic-repeat escalation behavior.

## Best Possible Solution

Keep #34574 open until main lands an exec-specific stable outcome hash for successful exec results, with regression tests where only volatile exec metadata changes. PR #41502 is the current canonical implementation path for that narrow fix. Track generic-repeat critical escalation and varied-command same-result detection separately through #54559 or a dedicated result-similarity follow-up.
