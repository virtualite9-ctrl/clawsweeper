## Summary

Keep open. Current main still has the Agents-page model selector flow and has not absorbed the PR's selected-option/default-fallback behavior. The PR is narrow and not supply-chain-sensitive, but the current patch has unresolved UI regressions called out in review and needs revision against main's current catalog-aware helper.

## Best Possible Solution

Keep the PR open for revision. The best path is to rework it against current main's renderAgentOverview and buildModelOptions flow: preserve the empty Inherit default state for non-default agents, preserve Current (...) for explicit missing per-agent models, make the default agent initially display the configured primary model, keep modelCatalog options intact, and add focused DOM tests for those selector states.
