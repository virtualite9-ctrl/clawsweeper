## Summary

Keep this PR open rather than closing it in cleanup. Current main still has the same Agents-page model-select surface and does not contain the PR's proposed selected-option/default fallback changes, but the PR as written needs revision because review comments identify plausible UI regressions for explicit per-agent models and inherited defaults.

## Best Possible Solution

Keep the PR open for author or maintainer revision. The best path is to rework it against current main's buildModelOptions(configForm, current, catalog) flow, bind selection to the actual select value rather than inherited effectivePrimary for non-default agents, preserve the 'Current (...)' label for explicit current values, and add DOM regression tests covering default-agent initial selection plus non-default 'Inherit default' behavior.
