Keep this PR open. The target Agents-page selector flow still exists on current main, and current main has not absorbed this PR's selected-option/default-fallback change. The PR is narrow and not security-sensitive, but the existing review comments identify real UI regressions that need revision before it should land.

## Best Possible Solution

Keep the PR open for revision. The best path is to rework the fix against current main's `renderAgentOverview` and `buildModelOptions` flow: preserve the empty `Inherit default` state for non-default agents, preserve `Current (...)` for explicit missing per-agent models, make the default agent initially display the configured primary model, and add focused DOM tests for default-agent initial selection plus non-default inherited-default behavior.
