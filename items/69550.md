## Summary

The reported Control UI overlap is still actionable on current main. The CSS reserves space only for `.chat-bubble.has-copy`, but the current grouped chat renderer computes `hasActions` and renders the action buttons without adding `has-copy` to the bubble class list. The linked PR #69655 correctly identified that gap, but it was closed unmerged, so it does not supersede or fix the issue.

## Best Possible Solution

Keep #69550 open and land a focused Control UI fix: add `has-copy` to the grouped chat bubble class list whenever `hasActions` is true, then add a renderer test that an assistant message with copy and/or expand actions receives the reserved-padding class. The closed PR #69655 is a useful reference for the intended minimal patch, but it was not merged.
