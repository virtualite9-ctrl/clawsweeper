## Summary

Keep open. The linked PR #69655 identified the likely root cause, but it was closed unmerged, and current main still renders assistant bubble action buttons without applying the CSS class that reserves space for them.

## Best Possible Solution

Keep the issue open and land a focused UI fix in `ui/src/ui/chat/grouped-render.ts`: add `has-copy` when `hasActions` is true, then add a regression test in `ui/src/ui/chat/grouped-render.test.ts` that an assistant message with copy/expand actions emits the class. Verify desktop and mobile chat bubbles no longer let the absolute action buttons cover text.
