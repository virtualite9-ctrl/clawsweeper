## Summary

Current `main` still appears to contain the reported overlap path: assistant chat bubbles render copy/expand actions, but the bubble class list does not include `has-copy`, while the CSS padding that reserves action-button space only applies to `.chat-bubble.has-copy`. The related PR #69655 identified the same root cause, but it was closed by automation for `r: too-many-prs`, not merged. citeturn0view0turn1view0

## Best Possible Solution

Keep #69550 open and apply the narrow UI fix: compute `hasActions` before building `bubbleClasses`, add `has-copy` when copy or expand actions are present, and add a focused `grouped-render` regression test asserting assistant bubbles with actions receive the class. Then run the appropriate changed UI test/check lane.
