## Summary

Current main still contains the reported unsafe Telegram token replacement path. The Telegram setup wizard still exposes the same keep/input prompts, configured-channel `Modify settings` still routes back into the channel configure flow, and the plaintext token helper still calls `.trim()` directly on the raw text prompt result. The related fixing PRs #67374 and #67733 are closed unmerged in the provided context, so this issue should stay open.

## Best Possible Solution

Keep this issue open. Patch the shared setup wizard text handling so raw text prompt results are normalized before `.trim()` is called, at minimum in `promptSingleChannelToken` and the generic `setup-wizard.ts` text-input path. Add a focused regression test for replacing an existing Telegram token where the replacement prompt returns `undefined`, and verify the flow returns validation/default handling instead of throwing.
