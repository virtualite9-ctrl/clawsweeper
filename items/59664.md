## Summary

Keep open. Current `main` partially documents Slack native exec approvals, but the docs still contradict the runtime: Slack does not auto-enable native exec approvals when `channels.slack.execApprovals.enabled` is unset, so the issue's core docs gap is not fully resolved.

## Evidence

- **Runtime requires explicit channel enablement:** `isChannelExecApprovalClientEnabledFromConfig` returns true only when `enabled === true` or `enabled === "auto"`, and only if approvers exist. An unset `enabled` value does not enable the native Slack approval client.
  - file: [src/plugin-sdk/approval-client-helpers.ts:40](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/src/plugin-sdk/approval-client-helpers.ts#L40)
  - command: `nl -ba src/plugin-sdk/approval-client-helpers.ts | sed -n '1,60p'`
  - sha: [ec3dbd22a4de](https://github.com/openclaw/openclaw/commit/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46)
- **Slack runtime falls back to owner allowlist only for approver resolution:** Slack approvers are resolved from `channels.slack.execApprovals.approvers` or `commands.ownerAllowFrom`; `channels.slack.allowFrom` is not used here.
  - file: [extensions/slack/src/exec-approvals.ts:27](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/extensions/slack/src/exec-approvals.ts#L27)
  - command: `nl -ba extensions/slack/src/exec-approvals.ts | sed -n '1,120p'`
  - sha: [ec3dbd22a4de](https://github.com/openclaw/openclaw/commit/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46)
- **Current Slack tests reject unset enablement:** Regression tests assert that Slack native exec approvals stay disabled when config is unset, when only approvers are present, and when only `commands.ownerAllowFrom` resolves; they only enable when `enabled: "auto"` or `enabled: true` is set with approvers.
  - file: [extensions/slack/src/exec-approvals.test.ts:32](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/extensions/slack/src/exec-approvals.test.ts#L32)
  - command: `nl -ba extensions/slack/src/exec-approvals.test.ts | sed -n '1,120p'`
  - sha: [ec3dbd22a4de](https://github.com/openclaw/openclaw/commit/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46)
- **Slack docs still claim no explicit native config is needed:** The Slack docs say native approvals auto-enable when `enabled` is unset or `"auto"`, and show `commands.ownerAllowFrom` alone as the default behavior with no explicit Slack exec-approval config.
  - file: [docs/channels/slack.md:731](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/docs/channels/slack.md#L731)

  - sha: [ec3dbd22a4de](https://github.com/openclaw/openclaw/commit/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46)
- **FAQ repeats the same incorrect auto-enable guidance:** The first-run FAQ says supported native channels auto-enable DM-first approvals when `channels.<channel>.execApprovals.enabled` is unset or `"auto"`, which does not match the current shared approval-client logic or Slack tests.
