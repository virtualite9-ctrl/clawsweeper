## Summary

Close #60604 as superseded by #65222. Current main still does not implement the Telegram per-message sequential-key bypass, but later related discussion showed this PR is only a partial fix: the inbound debouncer can re-serialize steer messages, and the core steer path still has the `isStreaming` gate. The newer open #65222 is the better canonical PR for the remaining Telegram steer/interrupt delivery work.

## Best Possible Solution

Close #60604 as superseded and continue the remaining work in #65222, or in a fresh maintainer-scoped replacement if #65222 is split. The canonical fix should cover Telegram active-run keying plus debouncer bypass and align with the core `isActive`/`isStopped` steer semantics tracked by #65010, instead of merging this older partial branch.
