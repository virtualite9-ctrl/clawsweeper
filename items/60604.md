## Summary

Current `main` does not contain the active-run-aware Telegram sequentialization proposed in PR #60604. The Telegram sequential key is still chat/topic scoped on `main`, the bot still installs `sequentialize(getTelegramSequentialKey)` directly, and the session-key/run-active plumbing added by the PR is absent. This is not safely closeable as already implemented.

## Evidence

- **Sequential key on main is still chat/topic scoped:** `getTelegramSequentialKey()` on current `main` has no options/predicate for active runs and returns `telegram:${chatId}` or `telegram:${chatId}:topic:${threadId}` for normal messages. The only per-message keying present is the existing `/btw` path.
