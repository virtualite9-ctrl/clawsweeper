## Summary

Keep this PR open. Current main still only injects `OPENCLAW_SHELL` for the exec, ACP client, and TUI local-shell child-process paths; it does not implement the requested `OPENCLAW_AGENT_ID` or `OPENCLAW_SESSION_KEY` contract. The adjacent `OPENCLAW_MCP_*` context is limited to bundled CLI/MCP loopback setup and does not supersede this spawned-shell runtime marker work.

## Best Possible Solution

Keep the PR open and ask for a rebase/update. The best implementation path is a shared runtime env resolver for deterministic `OPENCLAW_AGENT_ID` plus optional `OPENCLAW_SESSION_KEY`, threaded through exec direct/sandbox/PTY, gateway approval and safeBin execution, ACP client creation, and TUI local-shell commands. It should update the public environment/exec/ACP/TUI docs and add focused tests for explicit-agent-id precedence, session-key fallback, missing-context behavior, and the gateway approval/safeBin path flagged in review.
