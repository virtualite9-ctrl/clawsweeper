## Summary

Keep open. Current main still accepts and documents Slack `dmHistoryLimit`, but the Slack monitor only wires the group/channel `historyLimit`, and inbound history context is still gated on `isRoomish`, which excludes Slack 1:1 DMs (`channel_type: "im"`). I found no current Slack monitor path that fetches top-level DM history with `conversations.history` for new DM sessions.

## Best Possible Solution

Keep this issue open and implement the missing behavior in `extensions/slack`: resolve `channels.slack.dms.<user>.historyLimit` and `channels.slack.dmHistoryLimit` for direct messages, fetch recent top-level DM messages with Slack `conversations.history` when a new DM session is initialized, inject that history into the inbound context consistently with existing room history, and add focused tests for `channel_type: "im"`. The adjacent #7359 remains a broader cross-session visibility request and does not supersede this narrower DM history-limit gap.
