## Summary

Current main still does not implement Slack top-level DM history hydration from `channels.slack.dmHistoryLimit`. The Slack config/types/docs expose the setting, but the monitor context only carries `historyLimit`, provider startup only resolves group/channel history, and inbound history injection remains gated on `isRoomish`, which excludes Slack 1:1 DMs.

## Best Possible Solution

Keep this issue open and implement the missing behavior in `extensions/slack`: resolve `channels.slack.dms.<user>.historyLimit` before `channels.slack.dmHistoryLimit` for direct messages, pass that effective value through `SlackMonitorContext`, fetch recent top-level DM messages with Slack `conversations.history` only when initializing a fresh DM session, format/inject them consistently with existing inbound context, and add focused tests for `channel_type: "im"` plus per-DM override precedence. Keep #7359 as the broader cross-session visibility discussion; it should not supersede this targeted Slack DM history gap.
