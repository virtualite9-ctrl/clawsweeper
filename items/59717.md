## Summary

Keep the PR open. It is a focused attempt to fix #59663, and the author later rebuilt it down to the intended ACPX files, but current main still only accepts `deny`/`fail` and the pinned `acpx@0.6.1` dependency also types/parses non-interactive permissions as only `deny` or `fail`. The work is not implemented on main, not superseded by another shipped fix, and still needs dependency/runtime alignment plus the dangerous-config contract before it is merge-ready.

## Best Possible Solution

Keep the PR open and either update the pinned ACPX dependency to a version that officially supports `nonInteractivePermissions=allow`, or implement a deliberate OpenClaw-side policy bridge with tests proving headless write/exec prompts auto-approve. The merge-ready version should update `extensions/acpx/src/config-schema.ts`, `extensions/acpx/openclaw.plugin.json`, ACP agent docs, config tests, and `configContracts.dangerousFlags`, then pass the ACPX extension lane/typecheck.
