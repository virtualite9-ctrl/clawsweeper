## Summary

Keep #9658 open. Current main at 406ae72fd2789c78053f6b94006ec8f488bc1c00 still does not provide a streamed, file-backed, or payload-reference mechanism for the affected tool arguments. The existing issue discussion already includes an April 26 Codex review comment from steipete keeping it open for the same core API gap, and the current source/docs/tests still show inline JSON/string-only surfaces for exec, process write/paste, write, and apply_patch. The body's #9000 reference is a different closed qmd memory_search bug, not a canonical duplicate for this request.

## Best Possible Solution

Keep the issue open for maintainer product/API design. The likely path is a backward-compatible payload-reference or streamed/file-backed argument mechanism for exec command/stdin, process write/paste data, write content, and apply_patch input, plus docs for explicit limits/workarounds and focused regression coverage for large and special-character payloads. If maintainers decide not to add this API, document the supported chunking/file workflows and actual limits instead.
