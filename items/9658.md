## Summary

Keep #9658 open. Current main still models the affected tool arguments as inline JSON-serializable strings/objects and does not provide streamed, file-backed, or payload-reference arguments for exec, process write/paste, write, or apply_patch. Related streamed tool-call repair exists, but it only parses accumulated provider argument strings and still keeps bounded inline buffers; it does not solve the requested core transport/API gap.

## Best Possible Solution

Keep the issue open for maintainer product/API design. A complete fix should add a backward-compatible streamed or file-backed payload/reference mechanism across exec command/stdin, process write/paste data/text, write content, and possibly apply_patch input; document supported limits and safe workarounds; and add regression coverage for large and special-character payloads. If maintainers decide this should not be supported, document the explicit limits and recommended chunking/file workflows instead.
