## Summary

Keep #9658 open. Current main still carries the affected tool payloads as inline strings/JSON parameters and does not provide streamed or file-backed arguments for exec, process write/paste, write, or apply_patch. The existing 2026-04-26 maintainer discussion also explicitly kept this issue open.

## Best Possible Solution

Keep the issue open for maintainer product/API design. A complete fix likely belongs in core tool transport and schemas: add a backward-compatible streamed or file-backed payload/reference mechanism for exec command/stdin, process write/paste, write content, and possibly apply_patch, then document limits/workarounds and add regression coverage for large and special-character payloads. If maintainers decide not to support this, document the supported limits and safe chunking/file workflows instead.
