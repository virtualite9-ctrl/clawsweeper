## Summary

Keep open. Current main still has an auth.order bypass in the bundled GitHub Copilot discovery path: the plugin catalog calls resolveFirstGithubToken(), which obtains profiles via listProfilesForProvider() and then selects profileIds[0]. That list is Object.entries(authStore.profiles) order, while the auth.order-aware resolveAuthProfileOrder() helper is available but not used in this path. The referenced fix attempts #47061 and #47177 were closed unmerged, and #48682 is still open.

## Best Possible Solution

Keep #46031 open until an implementation lands on main. The likely fix belongs in the GitHub Copilot plugin discovery path: resolveFirstGithubToken should use the auth.order-aware profile order, iterate eligible ordered profiles, fall through unresolved tokenRefs, and add regression coverage for multiple Copilot token profiles where auth.order prefers a non-first profile.
