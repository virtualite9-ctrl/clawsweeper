## Summary

Keep this PR open. Current main still lacks direct-file normalization for QMD custom paths even though the config/docs surface says QMD paths may be directories or files. The PR is a focused implementation candidate, but it still needs maintainer review for literal glob handling and the mirrored package copy before landing.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to finish the focused QMD resolver fix in both source copies if the package mirror is still maintained, normalize existing file targets to parent directory plus an exact literal filename pattern, add regression tests for ordinary filenames and glob-metacharacter filenames, preserve directory behavior and dedupe semantics, then run the relevant changed checks before landing.

## Risks / Open Questions

- The PR currently uses path.basename(resolved) as a QMD pattern for file targets. Because OpenClaw passes this through --mask/--glob and treats *, ?, and [ as glob syntax in qmd-manager.ts, filenames with glob metacharacters need explicit literal handling or regression coverage before landing.
- The provided PR diff updates only src/memory-host-sdk/host/backend-config.ts and not packages/memory-host-sdk/src/host/backend-config.ts, leaving a mirrored resolver mismatch unless maintainers intentionally accept that split.
- Security review found no CI, dependency, package publishing, secret, or external code-execution changes in the provided PR diff; the remaining concerns are functional correctness and mirror coverage.
