## Summary

Keep open. Current main still treats custom QMD file entries as collection roots instead of normalizing them to the parent directory plus an exact filename pattern, while the config contract and docs explicitly allow QMD paths to be files.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is a focused resolver change that detects existing file targets, uses the parent directory as the QMD collection root, uses a literal exact-filename pattern or escaped mask, preserves directory-path behavior, adds regression coverage for normal and glob-metacharacter filenames, and confirms whether packages/memory-host-sdk needs the same update.

## Evidence

There is no stat/isFile branch that would convert a direct file target to dirname plus basename.
