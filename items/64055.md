## Summary

Current OpenClaw already hardens gateway restart against stale listeners on port 18789: Linux restart now runs stale-gateway cleanup before `systemctl restart`, that cleanup waits until the port is confirmed free before returning, service-mode startup also clears stale listeners before bind, and the regression is covered by dedicated tests. The latest release `v2026.4.23` already contains the same restart files as current `main`, so this is shipped.

## Risks / Open Questions

- I did not run a live arm64/Ubuntu restart reproduction because this review was required to keep the checkout byte-for-byte clean and avoid mutating runtime state.
- The cleanup only targets verified OpenClaw gateway listeners; if port 18789 is occupied by an unrelated process, restart can still fail until that external listener is removed.
