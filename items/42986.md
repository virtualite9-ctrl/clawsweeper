## Summary

Keep this issue open. Current main still has the core native subagent thread-binding gate, but the Telegram bundled channel entry does not register the required `subagent_spawning`, `subagent_delivery_target`, or `subagent_ended` lifecycle hooks. Telegram has adjacent current-conversation and ACP topic binding infrastructure, and docs describe Telegram thread-bound ACP spawn, but native `sessions_spawn({ mode: "session", thread: true })` is still documented as Discord-only. The provided context also shows open PR #42707 as the relevant implementation candidate rather than a shipped fix.

## Best Possible Solution

Keep this open and review/land #42707 or an equivalent focused implementation. The fix should add Telegram `registerFull` lifecycle wiring for `subagent_spawning`, `subagent_delivery_target`, and `subagent_ended`, bind native session-mode subagent spawns through Telegram topic/current-conversation state, honor the Telegram thread-binding feature flags, clean up bindings on end/failure, resolve completion delivery targets, and update native subagent docs/tests so Telegram ACP topic binding and native subagent thread binding stay clearly distinguished.
