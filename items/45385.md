## Summary

Keep #45385 open. Current main appears to have strengthened transcript repair for dangling Anthropic tool calls, but the reported user-facing leak is still present in `formatAssistantErrorText()`: a JSON `invalid_request_error` about `tool_use` IDs without `tool_result` blocks falls through to the generic `LLM request rejected: ...` formatter. Related PR #45393 remains open and #45735 was closed unmerged, so there is no canonical shipped fix to close against.

## Best Possible Solution

Keep the issue open and land a narrow fix in `formatAssistantErrorText()` before the generic `invalid_request_error` branch: classify Anthropic/OpenAI-compatible `tool_use`/`tool_result` pairing mismatch messages as session history corruption with `/new` recovery guidance, then add focused unit tests using the exact issue payload and the inverse `tool_result` mismatch variant. Treat the existing repair improvements as helpful mitigation, not as a full resolution of the user-facing leak.
