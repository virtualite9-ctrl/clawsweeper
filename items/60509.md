## Summary

Keep open. Current `main` still does not expose the inbound user prompt/message on `agent:bootstrap`, and the documented/implemented alternative that does receive prompt text is the later `before_prompt_build` plugin hook, which is a different phase and does not prove this request is already satisfied.

## Risks / Open Questions

- There is an adjacent plugin-hook alternative (`before_prompt_build`), so maintainers may still decide to redirect the design; however, current `main` does not already implement the requested `agent:bootstrap` context field.
- Any eventual fix would expand a public hook contract used by internal hooks and plugin-bundled hooks, so compatibility/docs review is part of the work.
