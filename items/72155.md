## Summary

Current main already implements the requested scoping for async exec completion notifications: exec/background task completion events require an explicit session key, are queued under that single session, and targeted heartbeat wakes run only that session when an agent session key is present. Current heartbeat prompts also embed the exec completion details instead of relying on hidden prior context, which addresses the confused heartbeat reply described in the issue.

## Best Possible Solution

Close this issue as implemented on current main and keep the shipped requester-session scoping. If a user can still reproduce cross-session delivery, the best follow-up is a new issue with exact session keys, the completion event text, heartbeat config, and logs showing which path enqueued the event.
