## Summary

Keep PR #71553 open. Current main still has the bug the PR targets: WhatsApp ack reactions derive `participant` only from `sender.jid`, while current identity normalization puts `@lid` senders in `sender.lid`. The send path already preserves LID participants, so the missing fallback is still a valid targeted fix, though the PR likely needs rebasing/adapting to current main's `createAckReactionHandle`/`reactionOptions` shape.

## Best Possible Solution

Keep the PR open and ask for, or make, a current-main rebase that preserves JID precedence while falling back to `sender.lid` when constructing ack reaction options, plus the LID-only group ack regression test in `extensions/whatsapp/src/auto-reply/monitor/ack-reaction.test.ts`.
