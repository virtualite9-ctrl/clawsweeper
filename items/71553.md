## Summary

Keep #71553 open. Current main still constructs WhatsApp ack-reaction options from `sender.jid` only, while the current inbound identity path can normalize a group participant like `444@lid` into `sender.lid` with `sender.jid` null. The lower send layer already preserves LID participants when one is supplied, so the PR addresses a real remaining gap. The PR is slightly stale against the current `createAckReactionHandle` shape, but the intended one-line behavior change and regression test are still useful.

## Best Possible Solution

Keep the PR open and adapt it to current main by changing `reactionOptions` in `extensions/whatsapp/src/auto-reply/monitor/ack-reaction.ts` to use `sender.jid ?? sender.lid`, then add the LID-only group ack regression test in `extensions/whatsapp/src/auto-reply/monitor/ack-reaction.test.ts` using the current `AckReactionHandle` flow. After that, run the WhatsApp extension test/typecheck lanes before merge.
