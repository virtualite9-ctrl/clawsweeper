## Summary

Keep this PR open. Current main still builds WhatsApp ack-reaction options from `sender.jid` only, while the inbound identity path can normalize group participants with `@lid` into `sender.lid` and no `sender.jid`. The send layer already preserves LID participants when one is supplied, and the existing ack-reaction tests do not cover the LID-only group case proposed by this PR.

## Best Possible Solution

Keep the PR open and ask for a rebase/adaptation onto current main: set the ack reaction participant from `sender.jid ?? sender.lid` in `reactionOptions`, preserve direct-message behavior and JID precedence, add the LID-only group ack regression test in `extensions/whatsapp/src/auto-reply/monitor/ack-reaction.test.ts`, then run the targeted WhatsApp extension test/type/format lanes before merge.
