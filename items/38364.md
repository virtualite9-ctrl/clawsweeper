## Summary

Keep #38364 open. Current main still does not expose structured reply-to metadata from Control UI or chat.send into the agent MsgContext. The no-output/NO_REPLY side is partially mitigated in current UI/Gateway history handling, but the central reply metadata path remains absent.

## Best Possible Solution

Keep #38364 open. Add an additive chat.send reply metadata object, capture selected parent/reply target state in Control UI, validate and map it in the Gateway into existing MsgContext ReplyTo* fields plus optional parent-content summary, and cover protocol validation, Gateway mapping, and Control UI request behavior with focused tests. Preserve the current NO_REPLY/no-output filtering as the partial fix already on main.
