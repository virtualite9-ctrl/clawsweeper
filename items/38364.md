## Summary

Keep #38364 open. Current main partially addresses the no-output/silent-response side, but the central reply-to metadata request is still missing from the Control UI chat send path, the closed chat.send protocol schema, and the gateway MsgContext construction.

## Best Possible Solution

Keep #38364 open. The best path is an additive chat.send reply metadata field, Control UI composer/thread state to capture the selected parent message, gateway mapping into existing MsgContext ReplyTo* fields plus an optional parent-content summary, and focused protocol/UI/gateway tests. Preserve the existing silent-output filtering as a partial fix.
