## Summary

Keep #38364 open. Current main still lacks the central reply-to metadata path from Control UI into `chat.send` and onward into agent `MsgContext`. The no-output/silent-reply side has partial filtering, but that does not satisfy the issue’s structured reply metadata request.

## Best Possible Solution

Keep #38364 open. The best path is an additive `chat.send` reply metadata field, Control UI composer/thread state to capture the selected parent message, Gateway mapping into the existing `MsgContext` `ReplyTo*` fields plus an optional parent-content summary, and focused protocol/UI/gateway tests. Preserve the existing silent-output filtering as a partial fix.
