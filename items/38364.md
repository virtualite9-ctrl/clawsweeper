## Summary

Keep #38364 open. Current main still does not expose structured reply-to metadata from Control UI through `chat.send` into the agent `MsgContext`. The no-output/`NO_REPLY` side is partially handled by UI/history filtering, but that does not satisfy the central reply metadata request.

## Best Possible Solution

Keep #38364 open. The best path is an additive `chat.send` reply metadata field, Control UI composer/thread state to capture the selected parent message, Gateway validation/mapping into the existing `MsgContext` `ReplyTo*` fields plus optional parent-content summary, and focused protocol/UI/gateway tests. Preserve the existing `NO_REPLY` suppression as a partial fix for the no-output side.
