## Summary

Keep open. The report is recent, non-maintainer-authored, and specifically targets Amazon Bedrock/`bedrock-converse-stream`. Current `main` has malformed streamed tool-call argument repair, but it is scoped to Kimi Anthropic-compatible and OpenAI-completions paths, not Bedrock. The locked upstream Bedrock provider still initializes tool args as `{}` and uses a parser that returns `{}` for empty/unparseable partial JSON, so the reported failure class remains plausible and unhandled for this provider path.

## Best Possible Solution

Keep the issue open. The best maintainer path is a Bedrock-specific fix in the provider stream/wrapper layer that detects empty or incomplete `toolUse.input` arguments before tool execution, surfaces a clear provider/truncation error or triggers a safe retry, and adds regression coverage for a large Bedrock Converse `write`/`exec` payload plus the repeated-failure behavior.
