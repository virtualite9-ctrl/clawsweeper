## Summary

Keep open. Current main has useful progress: beta/rollback docs exist, and an Unreleased opt-in `agents.defaults.compaction.maxActiveTranscriptBytes` preflight compaction trigger was added after the earlier review. But the concrete large-transcript Gateway hang is not fully fixed by default: the active embedded run path still performs whole-file repair and `SessionManager.open` on the JSONL, and the new byte guard is disabled unless users explicitly enable both transcript rotation and the threshold. The compaction path also still opens the same active transcript to compact it, so this is not yet a robust default recovery for existing oversized sessions.

## Best Possible Solution

Keep this issue open and finish the large-active-transcript recovery in the Gateway path. The strongest fix is a bounded pre-open guard before any whole-file repair or `SessionManager.open`, with streaming/yielding repair or a safe rotate/compact/fail-fast path that leaves HTTP/channel loops responsive and gives an actionable recovery command. Add regression coverage for an oversized JSONL transcript proving Gateway/channel work is not starved, and document the operator setting/recovery path in public session or troubleshooting docs.
