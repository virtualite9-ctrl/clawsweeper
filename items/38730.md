## Summary

Current main partially addresses #38730 with cron run-session retention and automatic reaping, but the central guardrail request is not fully implemented. There is still no high-frequency cron warning in the CLI or gateway cron.add path, and manual sessions cleanup still runs generic session maintenance rather than the cron run-session reaper. The linked PR #38731 directly tracks the warning work but is open/unmerged in the provided GitHub context.

## Best Possible Solution

Keep #38730 open until maintainers land PR #38731 or an equivalent guarded implementation. The best path is to preserve the shipped cron.sessionRetention reaper, add a conservative high-frequency warning for schedule.kind="every" jobs in both src/cli/cron-cli/register.cron-add.ts and src/gateway/server-methods/cron.ts, and make an explicit maintainer decision on whether openclaw sessions cleanup should invoke the cron run-session reaper, expose a cron-specific cleanup path, or document that cron run-session retention is scheduler-owned.
