## Summary

Keep #22774 open. Current main has partial inter-session provenance and top-level senderLabel display, but it still lacks a durable sender attribution contract and Chat UI rendering that distinguish human, main-agent, and sub-agent identities across sessions.send, sessions.steer, chat.send, agent input provenance, and sub-agent announce flows. The related PRs are closed unmerged, and the latest issue discussion reinforces that persistent sender identity/audit data is the real need.

## Best Possible Solution

Keep #22774 open. The best path is a core sender attribution contract that carries durable sender id, sender kind, display name, source session/agent, and renderable identity through sessions.send, sessions.steer, chat.send, agent input provenance, and sub-agent announce delivery; persists it in transcript/history events; exposes it through chat.history and chat events; and updates the Chat UI to render per-sender identity with regression coverage for main-to-sub-agent steer and sub-agent-to-main announce flows.
