## Summary

Keep #22774 open. Current main still has only browser-local user identity, top-level `senderLabel` display for user-role groups, and trace-style provenance fields; it does not provide durable human/main-agent/sub-agent sender attribution across `chat.send`, `sessions.send`/`sessions.steer`, inter-session delivery, persisted transcripts, `chat.history`, and Chat UI rendering. The linked PRs were closed unmerged, and #58110 is broader multi-agent chat UX rather than a canonical replacement for this narrower attribution contract.

## Best Possible Solution

Keep this issue open as the specific tracker for durable multi-agent sender attribution. The best path is an additive sender identity contract carrying sender id, sender kind, display name, source session/agent, and renderable UI identity through `chat.send`, `sessions.send`, `sessions.steer`, sub-agent announce delivery, transcript persistence, `chat.history`, live chat events, and Control UI rendering. Coordinate with #58110 only for broader multi-agent chat UX; do not treat it as a replacement for this attribution contract.
