## Summary

Keep #22774 open. Current main has partial sender-label UI support and inter-session provenance, but it still lacks a durable sender attribution contract that distinguishes human, main-agent, and sub-agent identities through send/steer/announce flows and renders distinct per-agent avatars in Chat UI. The issue discussion remains active, and the related PRs in the provided context are closed unmerged rather than canonical shipped fixes.

## Best Possible Solution

Keep #22774 open. The best path is a core-backed attribution contract that carries durable sender id/kind/name/source session through sessions.send, sessions.steer, chat.send/agent input provenance, and sub-agent announce delivery into transcript/history events, then updates Chat UI grouping/avatar rendering to distinguish human, main-agent, and sub-agent messages. Add regression coverage for main-to-sub-agent steer and sub-agent-to-main completion/announce flows.
