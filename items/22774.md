## Summary

Keep open. Current main still has the sub-agent workflow as a core Chat UI/runtime surface, but the send/steer path persists injected turns as plain user messages and the UI only has browser-local human sender labels plus one assistant identity. The related PRs in the provided context were closed unmerged, and #36006 explicitly described itself as an unblocker rather than a direct fix for #22774.

## Best Possible Solution

Keep #22774 open until OpenClaw adds structured sender-origin metadata through the relevant runtime/protocol path, likely sessions.send/sessions.steer/chat.send and sub-agent announce persistence, then updates Chat UI normalization, grouping, labels, avatars/colors, and regression tests so human, main-agent, and sub-agent messages render distinctly. Related PRs #23336 and #36006 are useful history but are not canonical shipped fixes.
