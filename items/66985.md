## Summary

Keep this PR open. Current main still has the raw `boundNode !== requestedNode` guard before node canonicalization, while the config and docs allow node bindings by id/name and the shared resolver supports display names, remote IPs, and long ID prefixes. The PR remains a plausible, focused fix for linked bug #61229 rather than an obsolete or closeable item.

## Best Possible Solution

Keep this PR open for maintainer review. The best outcome is to land this patch, or port the same canonicalization and regression tests onto current main: resolve both configured `boundNode` and per-call `requestedNode` through `resolveNodeIdFromList`, compare canonical ids, reject different nodes, and keep stale/unresolvable bindings fail-closed with clear errors.
