## Summary

Keep this issue open. Current main has meaningful partial session management: Gateway session create/list/patch/delete RPCs, a Control UI Sessions table with label edits/deletion/compaction checkpoint restore, a TUI session picker, and CLI list/cleanup. It still does not implement the requested first-class Topic/Session Management UI or backend contract: no Web UI topic sidebar/create workflow, no CLI create/rename/delete surface, no description/tags/message-count session-row contract, and no deleted-topic archive restore or session import/export API.

## Best Possible Solution

Keep this issue open, but narrow it into maintainer-owned follow-ups: add a first-class Control UI topic/session workflow around sessions.create, sessions.patch, and sessions.delete; decide whether description, tags, createdAt, and messageCount belong in GatewaySessionRow; add CLI/TUI create/rename/delete/archive only if maintainers want those surfaces; and design deleted-session archive restore plus import/export with protocol docs and tests.
