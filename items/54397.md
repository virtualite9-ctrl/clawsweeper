## Summary

Keep this issue open. Current main has partial session management across Gateway RPCs, Control UI, Web Chat, TUI, and CLI listing/cleanup, but it still does not deliver the requested first-class Topic/Session Management UI: no topic sidebar/create workflow in Web UI, no create/rename/delete CLI surface, no description/tags/message-count row contract, and no deleted-topic archive restore or session import/export API.

## Best Possible Solution

Keep this issue open, but narrow it into maintainer-owned follow-ups: add a first-class Control UI create/manage workflow around sessions.create, sessions.patch, and sessions.delete; decide whether description/tags/createdAt/messageCount belong in GatewaySessionRow; add CLI/TUI create/rename/delete/archive only if maintainers want those surfaces; and design deleted-session archive restore plus import/export with protocol docs and focused tests.
