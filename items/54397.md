## Summary

Keep this issue open. Current main has substantial partial session management, including Gateway session RPCs, Control UI session listing/labels/delete/compaction checkpoint controls, Web Chat session switching, TUI session picker, and CLI listing/cleanup, but it still does not deliver the full requested Topic/Session Management UI and metadata/import-export/archive-restore product surface.

## Best Possible Solution

Keep this issue open, but narrow it into concrete maintainer-owned work: add a first-class Control UI create/manage workflow around sessions.create, sessions.patch, and sessions.delete; decide whether description, tags, createdAt, and messageCount belong in the Gateway session-row contract; add TUI and CLI rename/delete/archive actions only if maintainers want those surfaces; and define deleted-archive restore plus import/export semantics with docs and focused tests.
