## Summary

Keep open. Current main has meaningful partial session/topic management: Control UI session listing/filtering/label/delete, a chat session selector, TUI session switching, and Gateway session list/create/patch/delete APIs. But the broader requested product surface is still incomplete: no Control UI create flow, no TUI rename/delete/archive management, no message-count field in session rows, no description/tags metadata, and no import/restore workflow for deleted/archived topics. The provided related reports are narrower Telegram/forum or API items, not a canonical duplicate for this broad UI request.

## Best Possible Solution

Keep this issue open, but narrow it into concrete remaining work: add a first-class Control UI topic/session management workflow around sessions.create, sessions.patch, and sessions.delete; decide whether persistent description/tags/messageCount/createdAt belong in the Gateway session row contract; add TUI management commands or picker actions if desired; and define archive restore plus import/export semantics with docs and focused tests. If maintainers prefer smaller tracking, split those pieces into follow-up issues and close this only after the central topic-management UI is actually shipped.
