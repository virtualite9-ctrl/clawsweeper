## Summary

Keep this PR open. Current main has partial related recovery, but it still does not implement the PR's central thrown context-overflow session reset, and the final embedded-error fallback still returns a /new hint instead of rotating the active session. The PR is tied to a concrete open Telegram overflow report, but review comments identify changes needed before it is merge-ready.

## Best Possible Solution

Keep this PR open for maintainer review or request changes. Preserve the useful thrown-overflow recovery idea, add a pure context-overflow guard, align the reset message with existing compaction overflow guidance, and add regression coverage for both the thrown-overflow path and a reachable final embedded-error reset path. Coordinate with #68494 and related session-recovery work if maintainers prefer a fresh canonical fix.
