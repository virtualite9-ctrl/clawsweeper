## Summary

Keep this issue open. Current main still lacks the requested opt-in hard caps for wall-clock session duration and cumulative per-session token spend. The newer compaction work only adds context/file-size preflight compaction, not a `session.maxDurationMinutes` or `session.maxTokensPerSession` budget guard.

## Best Possible Solution

Keep this issue open and implement it as a core session-budget feature. Add optional config schema/types/docs for session duration and cumulative token limits, persist reliable per-session lifecycle and usage budget state, enforce before costly LLM/tool work and on existing lifecycle ticks, emit deterministic user/operator feedback, and cover heartbeat, cron, CLI/runtime, and compaction edge cases with focused tests.
