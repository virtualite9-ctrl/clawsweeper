## Summary

Keep #36614 open. Current main still has the write-side route contamination path: `recordInboundSession` accepts a caller-supplied last-route target, and several inbound direct-message channel/plugin paths still pass the main session key even when `dmScope=per-channel-peer` resolves the message to an isolated session. The merged read-side mitigation does not remove this polluted main-session state.

## Best Possible Solution

Keep #36614 open and finish the write-side fix. The maintainer path should centralize the last-route target decision or update affected channel/plugin callers to use `resolveInboundLastRouteSessionKey`, so routes with `lastRoutePolicy: session` write delivery metadata only to the scoped inbound session while `dmScope=main` continues updating the main session. Add regression coverage for iMessage plus representative Slack, Signal, LINE, Matrix, Mattermost, and Discord direct-message paths, preserving the existing Telegram and WhatsApp guarded behavior.
