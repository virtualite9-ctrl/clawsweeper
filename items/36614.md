## Summary

Keep #36614 open. Current main still contains the reported write-side contamination path: the central inbound recorder writes last-route delivery fields to the caller-supplied target session, and iMessage plus several other DM paths still pass the main session as that target even when the inbound route is scoped to a per-channel peer session. The related fix PRs #36625 and #40233 were closed unmerged, while the merged #38418 explicitly addressed the read-side shared-main chat.send leakage, not this write-side pollution.

## Best Possible Solution

Keep #36614 open and finish the write-side fix. The likely maintainer path is to make all direct-message last-route updates use resolveInboundLastRouteSessionKey or an equivalent central guard so per-channel-peer routes write only to the scoped inbound session, while dmScope=main continues to update the main session. Add regression coverage for iMessage and representative Discord/Slack/Signal/LINE/Matrix/Mattermost paths, preserving the existing Telegram/WhatsApp guarded behavior.
