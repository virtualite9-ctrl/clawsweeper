## Summary

Keep #36614 open. Current main still contains the reported write-side route pollution path: `recordInboundSession` writes the caller-supplied `updateLastRoute.sessionKey`, and several inbound direct-message surfaces still pass `mainSessionKey` even when the resolved route/session is per-channel-peer scoped. The merged read-side WebChat inheritance fixes reduce one symptom, but they do not remove the main-session write contamination this issue tracks.

## Best Possible Solution

Keep #36614 open and finish the write-side fix. The maintainer path should centralize the last-route target decision, or make affected channel callers use `resolveInboundLastRouteSessionKey`, so routes with `lastRoutePolicy: session` write delivery metadata only to the scoped inbound session while `dmScope=main` continues updating the main session. Add regression tests for iMessage and representative Slack/Signal/LINE/Matrix/Mattermost/Discord direct-message paths, preserving existing Telegram and WhatsApp guarded behavior.
