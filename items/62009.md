## Summary

Current `main` already contains the Bedrock `auth: "aws-sdk"` runtime-auth propagation fix, with additional guardrails beyond this PR: OpenClaw now marks AWS-SDK-authenticated runs as configured in Pi auth storage, keeps IMDS/task-role auth on the runtime path instead of faking env markers, and has targeted regression coverage for both behaviors.

## Evidence

- **Runtime auth path now propagates AWS SDK auth into Pi auth storage:** When `getApiKeyForModel()` resolves `mode: "aws-sdk"` with no explicit API key, the auth controller now tries provider runtime-auth prep and otherwise stores the `__aws_sdk_auth__` sentinel in `authStorage`, exactly covering the missing `setRuntimeApiKey()` path described in the PR.
