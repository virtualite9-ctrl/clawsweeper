## Summary

Keep #71848 open. Current main has launchd restart hardening, but it does not resolve this report's specific stale/session-scoped TMPDIR, launchd SIGABRT 134, TCC/Full Disk Access, or high-memory background-start path. In fact, the current install path still persists the host/app TMPDIR into the LaunchAgent environment, and doctor/service audit does not flag or repair stale TMPDIR values.

## Best Possible Solution

Keep the issue open and handle it as a macOS gateway launchd hardening task: stop persisting session-scoped TMPDIR into LaunchAgent plists or validate/repair it to a stable temp directory, add doctor/status diagnostics for stale TMPDIR and launchctl last-exit-status 134, document the required TCC/Full Disk Access process context, and add focused daemon/app tests for the repaired install path.
