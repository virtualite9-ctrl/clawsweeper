## Summary

Current main already implements the macOS launchd hardening requested by this report. Commit d7c173b6945687985a60842e500aeabec522f060 replaces session-scoped macOS TMPDIR usage with a durable state-dir temp path, creates that temp directory before LaunchAgent bootstrap, defaults the macOS service working directory to the OpenClaw state dir, and labels launchd exit 134 as SIGABRT/abort in status output. The Unreleased changelog explicitly references this issue and credits the reporter.

## Best Possible Solution

Close as implemented on main and keep the shipped launchd hardening from d7c173b6945687985a60842e500aeabec522f060. If a supported-runtime macOS LaunchAgent still aborts after this fix ships, a new issue should include fresh `openclaw gateway status --json`, launchd logs, plist contents with secrets redacted, Node version, memory settings, and whether the gateway is started from a logged-in GUI session.
