## Close Comment

Closing from a Codex review because current `main` already carries the fixes described here. The startup path now applies lane concurrency through `applyGatewayLaneConcurrency()` in `src/gateway/server.impl.ts:550` and `src/gateway/server-lanes.ts:6`, and `resolveAgentLane` is no longer referenced anywhere in source, so the reported `resolveAgentLane is not defined` startup failure is obsolete. The announce-timeout path is also hardened: completion announces run direct-first and automatically fall back to queue routing when direct wake/delivery fails (`src/agents/subagent-announce-dispatch.ts:42`, `src/agents/subagent-announce-delivery.ts:492`), with regression coverage in `src/agents/subagent-announce-dispatch.test.ts:90` and `src/agents/subagent-announce.format.e2e.test.ts:1705`, and the behavior is documented in `docs/tools/subagents.md:48`. This implementation is already present in release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`). If you still hit a fresh timeout on a newer build, please open a new issue with current logs, exact reproduction steps, and the affected session/channel route.
