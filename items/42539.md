## Summary

Keep this issue open. Current main still has Telegram TTS voice-note delivery bundled with text as a caption by default, with separation only on caption overflow/error fallback, and there is no `messages.tts.delivery` or equivalent configurable mode. The only issue comment is an automated stale notice, and the provided related title matches do not supersede this Telegram-specific TTS request.

## Best Possible Solution

Implement a conservative Telegram-owned delivery mode for TTS voice-note payloads, likely by extending the typed TTS config/schema/docs with `delivery: "bundled" | "separate"` if maintainers want shared semantics, then teaching `extensions/telegram` delivery to send visible text first and the voice note without caption when the effective mode is `separate`. Add focused tests for auto-TTS/`audioAsVoice` Telegram delivery, caption-overflow compatibility, reply/thread parameters, and docs under the public TTS or Telegram channel pages.
