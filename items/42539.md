## Summary

Current main still lacks a Telegram TTS delivery mode that always sends readable text separately before the voice note. Telegram voice/audio delivery still uses captions for short text and only splits text on caption overflow or video-note-specific paths, while the typed and strict TTS config surface has no `delivery` or equivalent option.

## Best Possible Solution

Keep this issue open. Implement a Telegram-owned delivery policy or a shared `messages.tts.delivery: "bundled" | "separate"` contract, then update the strict config schema/types, Telegram bot and direct send paths, tests, and public TTS/Telegram docs. In `separate` mode, visible text should be sent as a normal Telegram message first and the voice note should be sent without a caption.
