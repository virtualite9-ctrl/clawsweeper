## Summary

Current main still does not implement a Telegram TTS delivery mode that always separates readable text from the voice note. The Telegram sender still captions normal voice/audio media and only sends follow-up text for caption overflow or video-note paths, and the typed/strict TTS config surface has no `delivery` or equivalent option.

## Best Possible Solution

Keep this issue open and implement a Telegram-owned TTS delivery policy, or a shared `messages.tts.delivery: "bundled" | "separate"` contract if maintainers want cross-channel semantics. The implementation should send visible text as a normal Telegram message first and then send the voice note without a caption when the effective mode is `separate`, with focused tests for short auto-TTS voice replies, caption overflow compatibility, replies/threads, and docs on the public Telegram or TTS configuration page.
