## Summary

Keep this PR open. Current main still clears mixed-content `/think` and `/verbose` levels before run resolution, and the Gateway `chat.send` path still creates exactly that mixed-content shape when a send includes `thinking`. The PR is a plausible focused fix and the latest diff addresses the obvious unauthorized-sender escalation, but it still needs regression coverage and likely a small adjustment so one-shot inline levels do not become persisted session overrides.

## Best Possible Solution

Keep this PR open as a focused bug-fix candidate. The best maintainer path is to land this preservation behavior or an equivalent implementation that preserves `thinkLevel` and `verboseLevel` for the current run without unintended session persistence, add targeted `resolveReplyDirectives` and Gateway `chat.send` regression tests, and keep the related Anthropic xhigh provider PR as separate complementary work.

## Risks / Open Questions

- The PR as written may persist mixed-content `/think` levels as session defaults because it restores `hasThinkDirective` before the shared persistence path; this should be tested or avoided if inline levels remain one-shot.
- Regression coverage is still missing for authorized mixed-content `/think` and `/verbose`, unauthorized mixed-content directives, and the `chat.send` thinking injection path.
- The related open Anthropic xhigh PR is complementary provider support, not a replacement for this directive-preservation fix.
