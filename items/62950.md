## Summary

Keep this PR open. Current main still has the reported behavior: mixed-content inline directives are parsed, then `clearInlineDirectives(parsedDirectives.cleaned)` replaces the parsed directive object and resets `thinkLevel` and `verboseLevel` before `resolvedThinkLevel` and `resolvedVerboseLevel` are computed. The PR is a focused implementation candidate rather than obsolete cleanup, and it is paired with the same-author open Anthropic xhigh PR #38729.

## Best Possible Solution

Keep the PR open as a valid bug-fix candidate. Best maintainer path is to review the small preservation change, require or add regression coverage for authorized mixed-content `/think` and `/verbose` plus the unauthorized guard, and then land or request changes rather than closing as obsolete.
