## Summary

Keep PR #39692 open. Current main has the Control UI i18n framework, zh-CN lazy loading, generated-locale parity, and some independently localized pieces, but the PR's central work is still not implemented across the targeted Control UI views. Agents, agent overview/status/files/cron panels, Logs, Debug, config form errors/placeholders, and exec approvals still render user-visible hardcoded English. The Greptile review concerns are still useful rebase guidance because current main preserves the monospace security-audit command, "Cron Jobs" label, and "never" refresh fallback that the PR would need to preserve.

## Best Possible Solution

Keep the PR open and ask for a focused rebase against current main. The best implementation is to move the remaining Control UI view literals into the English source locale map, regenerate zh-CN and other generated locale bundles through the Control UI i18n pipeline, preserve the reviewed UX details, and add a targeted regression check or view smoke coverage where practical so hardcoded strings in the affected surfaces are harder to reintroduce.
