## Summary

Keep PR #39692 open. Current main has the Control UI i18n framework, zh-CN lazy loading, and some independently localized channel/instances/gateway labels, but the PR's central work is still not implemented across the targeted Control UI views. Agents, agent panels, Logs, Debug, config form errors/placeholders, channel config messages, and exec approvals still render hardcoded English. The Greptile review comments remain useful rebase guidance because current main preserves the monospace security-audit command, the "Cron Jobs" label, and the "never" refresh fallback semantics that the PR should avoid regressing.

## Best Possible Solution

Keep the PR open and ask for a focused rebase against current main. The best implementation is to move the remaining user-facing Control UI literals into the English source locale map, regenerate zh-CN and the other generated locale bundles through the Control UI i18n pipeline, preserve the reviewed UX details from Greptile, and add a targeted regression check or view smoke where practical so these hardcoded strings are harder to reintroduce.
