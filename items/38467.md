## Summary

Keep #38467 open. Current main still exposes WhatsApp directory peers from configured `allowFrom` entries only; it does not ingest Baileys `contacts.upsert`, persist phone contacts, expose contact names/phone numbers through the WhatsApp directory, or resolve outbound WhatsApp targets by contact name. The related PR #33924 is closed unmerged and addressed a separate `allowSendTo` allowlist split, not contact-directory ingestion.

## Best Possible Solution

Implement this in the bundled WhatsApp plugin: add account-scoped local contact persistence populated from Baileys contact events, add explicit privacy/config handling if persistence is opt-in, merge stored contacts into the WhatsApp directory with name/phone metadata, make outbound/direct target resolution safely use unique contact-name matches, and add tests/docs for ingestion, filtering, ambiguity, privacy, and message-tool behavior.
