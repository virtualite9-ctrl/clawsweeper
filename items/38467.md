## Summary

Keep #38467 open. Current main still does not ingest Baileys contact events, persist WhatsApp phone contacts, expose them through the WhatsApp directory, or resolve outbound WhatsApp targets by contact name. The related PR #33924 is closed unmerged and addressed a different allowlist split, not contact-directory ingestion.

## Best Possible Solution

Implement this in the bundled WhatsApp plugin: add account-scoped local contact persistence populated from Baileys contact events, add explicit privacy/config handling if persistence is opt-in, merge stored contacts into the WhatsApp directory with name/handle/phone metadata, make outbound/direct target resolution safely use unique contact-name matches, and add tests plus docs for ingestion, filtering, ambiguity, privacy, and message-tool behavior.
