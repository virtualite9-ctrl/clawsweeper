## Summary

Keep open. Current main still has the Nostr gateway lifecycle and DM subscription-shape failures this PR targets, but the PR is stale against the current file layout and its proposed sendMedia fallback is not safe to merge as-is.

## Best Possible Solution

Keep this PR open as a stale but useful implementation candidate. The maintainer path is a refreshed Nostr-owned patch on current main: make startNostrGatewayAccount stay pending until ctx.abortSignal and close idempotently, pass a single Filter object to subscribeMany, add focused lifecycle and real subscription-shape regression coverage, and leave media unsupported unless a safe public-URL-only design is explicitly added. #53858 should remain the canonical user-facing bug until that corrected fix lands.

## Risks / Open Questions

- The PR is not merge-ready because the gateway lifecycle code now lives in extensions/nostr/src/gateway.ts, while the patch edits the old inline channel.ts path.
