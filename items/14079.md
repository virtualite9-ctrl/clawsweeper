## Summary

Keep #14079 open. Current main still does not provide Linux desktop notification delivery for the existing `system.notify` / `openclaw nodes notify` surface: the headless node host does not advertise `system.notify`, its dispatcher has no notify handler, the Gateway rejects undeclared node commands before forwarding, and docs/CLI still distinguish macOS notification support from headless node-host commands. The related #24616 closed only the misleading-error angle, not Linux notification delivery.

## Best Possible Solution

Keep the issue open. The likely fix belongs in the Linux/headless node-host path: detect desktop-session availability, advertise `system.notify` only when delivery is supported, implement `notify-send` with a D-Bus/libnotify fallback and clean headless degradation, add focused node-host/Gateway/CLI tests, and update the nodes/CLI docs. If maintainers prefer this outside core, first add a documented plugin API that can safely extend or own the reserved `system.notify` surface.
