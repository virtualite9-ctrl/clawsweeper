## Summary

Keep #14079 open. Current main still lacks Linux desktop notification delivery for the existing `system.notify` / `openclaw nodes notify` surface: the headless node host does not advertise or handle `system.notify`, the CLI/docs still frame notify as macOS-node behavior, and plugins cannot replace the exact reserved core command. The related closed #24616 addressed the misleading unsupported-command error, not Linux notification delivery.

## Best Possible Solution

Keep the issue open. The best product fix is to implement Linux `system.notify` in the node-host path: advertise the command when an appropriate Linux desktop session is available, support `notify-send` with D-Bus/libnotify fallback, degrade cleanly when headless, add focused gateway/node-host/CLI tests, and update the CLI and nodes docs. If maintainers want this outside core, first add an explicit plugin API that can extend or own the reserved `system.notify` surface safely.
