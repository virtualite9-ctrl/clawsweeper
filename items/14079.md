## Summary

Keep #14079 open. Current main still does not deliver Linux desktop notifications through the existing `system.notify` / `openclaw nodes notify` surface. The Linux node command policy allowlists `system.notify`, but the headless node host neither declares nor handles it, and the gateway requires declared commands before forwarding. The supplied #24616 context is related but narrower: it covered the misleading Linux error path, not the requested notify-send/libnotify delivery support.

## Best Possible Solution

Keep #14079 open. The best path is a focused core/node-host implementation for Linux `system.notify`: detect a usable desktop session (`DISPLAY` or `WAYLAND_DISPLAY`), advertise `system.notify` only when delivery is available, use `notify-send` first with a Freedesktop D-Bus/libnotify fallback, degrade cleanly on headless hosts, add gateway/node-host/CLI tests, and update the public nodes docs and CLI help. If maintainers prefer an external implementation, core first needs a documented way for plugins to implement or augment the reserved `system.notify` surface.
