## Summary

Current `main` already wires background runs through the plugin runtime and hook surfaces needed for heartbeat, cron, and plugin-owned background handlers, so the reported missing plugin injection path does not match the code on `main`.

## Evidence

## Close Comment

Closing from a Codex read-only review because current `main` already has the background plugin wiring this report says is missing. Background runs load the runtime plugin registry before execution (`src/agents/pi-embedded-runner/run.ts:307`, `src/agents/runtime-plugins.ts:5`, `src/plugins/loader.ts:1826`), embedded attempts still run prompt-build plugin hooks (`src/agents/pi-embedded-runner/run/attempt.ts:2005`), heartbeat turns go through `before_agent_reply` with `trigger: "heartbeat"` (`src/infra/heartbeat-runner.ts:1064`, `src/auto-reply/reply/get-reply.ts:567`), and cron turns now have an explicit `before_agent_reply` gate on `main` (`src/agents/pi-embedded-runner/run.ts:336`). Plugin-owned background handlers are also present on current `main`, for example memory dreaming registers `before_agent_reply` for `heartbeat`/`cron` (`extensions/memory-core/src/dreaming.ts:767`) and Teams background reflection dispatches through the normal reply pipeline (`extensions/msteams/src/feedback-reflection.ts:203`). There is dedicated regression coverage in `src/agents/pi-embedded-runner/run.before-agent-reply-cron.test.ts:23`, `src/agents/cli-runner.before-agent-reply-cron.test.ts:96`, and `src/auto-reply/reply/get-reply.before-agent-reply.test.ts:78`. The latest release tag `v2026.4.22` does not appear to contain the newer cron gate, so this is being closed as implemented on `main`, not as a released fix.
