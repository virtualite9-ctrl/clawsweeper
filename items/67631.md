## Summary

Current main still has no automatic boundary between the macOS app LaunchAgent and CLI doctor/gateway service repair. The app owns `ai.openclaw.mac` for app autostart, but both the app and CLI repair path still manage the shared `ai.openclaw.gateway` service. The workaround posted in the discussion, removing the CLI gateway LaunchAgent and avoiding terminal `doctor --fix`, matches the remaining gap rather than proving it is fixed.

## Best Possible Solution

Keep this issue open and make the macOS gateway ownership boundary explicit. A conservative fix is for CLI doctor/gateway repair on macOS to detect `ai.openclaw.mac` and either skip installing or rewriting `ai.openclaw.gateway` with a clear note, honor an explicit external-supervisor/attach-only policy, or route both app and CLI through one canonical owner. Add regression coverage around doctor repair and launchd scanning, preserve the app's attach-existing behavior, and update the macOS/doctor docs with safe guidance for Mac app users.
