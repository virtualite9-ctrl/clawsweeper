## Summary

Keep #11172 open. Current main still does not inject OPENCLAW_AGENT_ID, CLAWDBOT_AGENT_ID, or OPENCLAW_SESSION_KEY into spawned exec/subagent command environments. The existing implementation only injects OPENCLAW_SHELL=exec, while agent/session identity is carried as internal metadata or node invocation parameters rather than process environment variables.

## Best Possible Solution

Keep #11172 open until OpenClaw implements or explicitly rejects a documented core exec identity contract. The likely implementation belongs in shared exec environment construction: derive identity from the resolved session context, inject OPENCLAW_AGENT_ID and OPENCLAW_SESSION_KEY consistently, decide whether CLAWDBOT_AGENT_ID compatibility is supported, define whether request env may override identity, and add focused tests/docs for gateway, node, sandbox, PTY, approvals, and background process paths.
