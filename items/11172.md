## Summary

Current main still does not implement the requested subagent/exec identity environment variables. The related implementation PR #11497 was closed unmerged, and the existing #11172 review comment correctly matches the current source: exec has agent/session context available, but only injects existing runtime markers such as OPENCLAW_CLI and OPENCLAW_SHELL.

## Best Possible Solution

Keep #11172 open until OpenClaw implements or explicitly rejects a documented core exec identity contract. A likely implementation belongs in shared exec environment construction: inject OPENCLAW_AGENT_ID and OPENCLAW_SESSION_KEY from resolved agent/session context, decide whether CLAWDBOT_AGENT_ID compatibility is supported, define whether request env can override identity values, and add focused tests/docs for gateway, node, sandbox, PTY, and approval paths.
