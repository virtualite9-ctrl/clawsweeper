## Summary

Keep #11172 open. Current main still does not inject OPENCLAW_AGENT_ID, CLAWDBOT_AGENT_ID, or OPENCLAW_SESSION_KEY into exec child-process environments for subagent sessions. Agent/session identity is available as session metadata and approval/routing context, but the environment builders only merge inherited/request/sandbox env and add OPENCLAW_SHELL as the runtime marker. The related implementation PR #11497 was closed unmerged, and related #53262 concerns sessions_spawn schema behavior rather than exposing identity to spawned commands.

## Best Possible Solution

Keep this issue open until OpenClaw ships or explicitly rejects a documented core exec identity contract. The likely fix belongs in shared exec environment construction after resolving session/agent context: inject OPENCLAW_AGENT_ID and OPENCLAW_SESSION_KEY consistently, decide whether CLAWDBOT_AGENT_ID compatibility is supported, define precedence against caller-supplied env, and add focused tests/docs for exec modes and subagent-spawned sessions.
