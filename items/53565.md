## Summary

Keep open. Current `main` has not implemented this PR: sandbox defaults still deny `cron`, so the requested behavior is not on `main`. The PR also still sits on a real unresolved trust-boundary question, because `cron` mutations remain admin-scoped Gateway methods and `session:*` cron targets are still accepted/routed into non-container runs. That makes this neither `implemented_on_main` nor a safe `cannot_reproduce`/cleanup close.
