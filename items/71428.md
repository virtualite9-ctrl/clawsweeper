## Summary

Keep open. Current main still supports only the host-config opt-in `plugins.entries.<id>.hooks.allowConversationAccess`; it does not implement the requested manifest-owned `permissions.conversationAccess` contract. The prior Codex review comment remains accurate against current HEAD.

## Best Possible Solution

Keep the issue open and implement an additive manifest-level permission contract. The likely path is to add `permissions.conversationAccess` to `PluginManifest`, normalize it through `loadPluginManifest` and `PluginManifestRecord`, merge it with the existing config hook policy in the loader/API creation path with explicit precedence, and update tests plus plugin manifest/hooks docs. Preserve `plugins.entries.<id>.hooks.allowConversationAccess` for backward compatibility and operator override.
