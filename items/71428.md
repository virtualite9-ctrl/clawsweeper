## Summary

Keep this issue open. Current main still enforces conversation hook access from host config only, and it has not added the requested manifest-owned `permissions.conversationAccess` contract. The prior issue comment is consistent with current source, docs, and tests at `110fa97f2aa171d601d1736a5f4855d299f6fb88`.

## Best Possible Solution

Keep the issue open. Add an explicit additive manifest contract, likely `permissions.conversationAccess`, in `src/plugins/manifest.ts` and manifest docs; carry it through manifest registry records into the loader; merge it with existing `plugins.entries.<id>.hooks.allowConversationAccess` using documented config override precedence; then add tests for manifest-only allow, config-only backward compatibility, both-present precedence, and default rejection.
