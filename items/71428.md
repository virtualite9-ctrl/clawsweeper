## Summary

Keep open. Current main implements and documents the config-level `plugins.entries.<id>.hooks.allowConversationAccess` path, but not the requested manifest-owned `permissions.conversationAccess` contract. Hook enforcement still receives only config entry hook policy, and the manifest type, loader normalization, manifest registry record, docs, and tests do not carry a manifest permission field.

## Best Possible Solution

Add an explicit additive manifest-level permission contract, likely `permissions.conversationAccess`, in `src/plugins/manifest.ts`; normalize and carry it through `src/plugins/manifest-registry.ts`; merge it with the existing `plugins.entries.<id>.hooks.allowConversationAccess` override before `createApi` builds hook policy; then add loader/registry tests for manifest-only, config-only, both-present precedence, and denied cases, plus public docs updates for `docs.openclaw.ai/plugins/hooks` and `docs.openclaw.ai/plugins/manifest`.
