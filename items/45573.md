## Summary

Keep open. Current main still does not clearly fix the reported normal Telegram group persistence path: non-forum group messages still omit `updateLastRoute`, while the directly linked fix PR #45610 was closed unmerged. Existing tests cover stable group session keys and forum-topic route persistence, but not the issue's regular group persistence across restart/disconnect behavior.

## Best Possible Solution

Keep the issue open and fix the regular Telegram group path explicitly. A maintainer-quality fix should decide whether non-forum groups should always provide `updateLastRoute`, add regression coverage for regular group persistence across repeated messages and restart/disconnect behavior, and verify that DM `mainDmOwnerPin` and forum-topic routing remain unchanged.

## Evidence

- **Regular Telegram groups still skip updateLastRoute:** For Telegram groups, `updateLastRouteThreadId` is only set when a forum `resolvedThreadId` exists; `updateLastRoute` is omitted unless the chat is not a group or the group has a topic thread id. A regular non-forum group like `agent:main:telegram:group:<id>` therefore still records metadata without the awaited route update PR #45610 proposed enabling for all groups.
