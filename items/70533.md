## Summary

Keep #70533 open. Current main has scoped plugin runtime loading, but it still materializes the full discovered manifest registry before applying `onlyPluginIds`/allowlist scope, so the specific discovery-time manifest read/parse cost reported by the issue remains plausible and unaddressed.

## Best Possible Solution

Keep the issue open and fix the plugin control-plane path: derive a concrete startup/plugin allow scope before manifest materialization, filter candidate directories by plugin id/name when exact ids are known, and move any startup classification metadata needed for configured channels into the persisted plugin index or another cheap metadata path. Add regression coverage that a scoped load does not read/parse disallowed manifests, not just that it avoids importing their runtime modules.
