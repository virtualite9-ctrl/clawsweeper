## Summary

Current main has shipped fixes for the MiniMax valid-key 401 path reported from OpenClaw 2026.3.13. MiniMax now defaults to the Anthropic-compatible endpoint with bearer-auth header injection, onboarding/portal re-auth write the same auth contract, and the embedded runner applies that authHeader behavior to model requests. The related MiniMax 401 report #37142 is already closed as implemented and shipped in v2026.4.22.

## Best Possible Solution

Keep the shipped MiniMax authHeader/Anthropic-endpoint implementation and the existing auth-profile cooldown behavior. If a user still sees this on v2026.4.22 or newer, it should be reopened as a fresh, versioned reproduction with redacted active provider config, selected model/session, and auth-profile source so maintainers can distinguish a new UI-state bug from stale/incorrect credentials.
