## Summary

Current main already implements the video CLI capability through the maintained `openclaw infer video` / `openclaw capability video` surface. It supports video generation, provider listing, JSON/text output, runtime-backed provider selection, output writing/downloading, and durable task inspection via `openclaw tasks` instead of the PR's originally non-functional standalone `video status` command.

## Best Possible Solution

Close this PR as implemented on main. Keep the shipped infer/capability video CLI plus durable `openclaw tasks` status model; open a narrow maintainer-scoped follow-up only if a top-level `openclaw video` alias is desired.
