## Summary

Keep this PR open. Current main still caps the QMD vector status probe at 5000 ms even when `memory.qmd.limits.timeoutMs` is configured higher, so the central bug remains. The latest PR context shows the earlier broad checkpoint export filter was narrowed away after review, leaving a focused timeout fix plus regression coverage and changelog entry.

## Best Possible Solution

Keep this PR open, review the current head, refresh the PR body if it still describes the removed broad checkpoint filter, and land the focused QMD status-timeout fix with its regression tests once mergeability and CI/Testbox proof are green. Keep checkpoint export filtering centralized in the session artifact helpers rather than adding an export-layer substring filter.
