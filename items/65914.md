## Summary

Keep this PR open. Current main still has the QMD status-probe timeout bug: `probeVectorAvailability()` caps `qmd status` at 5 seconds instead of honoring `memory.qmd.limits.timeoutMs`. The checkpoint-export half is now partly covered by precise session artifact filtering on main, and the PR's broad `.checkpoint.` substring filter still has the documented review risk of skipping valid primary transcripts.

## Best Possible Solution

Revise this PR rather than close it: keep the `probeVectorAvailability()` change to pass `this.qmd.limits.timeoutMs` directly and add the focused timeout regression test; drop the export-side broad `.checkpoint.` substring filter because current main already excludes exact compaction checkpoint transcripts through `listSessionFilesForAgent()`, or replace it with the precise `isCompactionCheckpointTranscriptFileName()` helper only if maintainers want an extra export-layer guard.
