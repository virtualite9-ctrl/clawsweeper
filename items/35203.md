## Summary

Keep #35203 open. Current main has useful partial infrastructure for isolated subagents, plugin lifecycle hooks, plugin runtime subagent/task-flow helpers, session visibility controls, memory search, and usage/cost reporting. It still does not implement the RFC’s central capabilities: learned capability profiling/routing, shared blackboard storage/query/pub-sub, private/team/global memory visibility with promotion rules, or global/per-task token/cost budget enforcement. The Octo/teleconference and KinthAI discussion is useful prior art, but it is not a merged complete implementation in this checkout.

## Best Possible Solution

Keep #35203 open as an umbrella RFC until a maintainer decomposes it. The practical path is to split focused follow-ups for capability metrics/routing, blackboard or artifact search/query, memory visibility and promotion gates, and token/cost budget aggregation/enforcement. Plugin-first experiments should use existing runtime.subagent, taskFlow, before_tool_call, llm_output, subagent lifecycle hooks, session visibility, context-engine, and memory plugin seams where sufficient; any missing pre-spawn task-aware routing, visibility, telemetry, or enforcement hook should become a narrow core/plugin API issue.
