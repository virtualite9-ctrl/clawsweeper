## Summary

Keep #35203 open. Current main has useful partial infrastructure: isolated subagents, opt-in agent-to-agent session messaging, plugin hooks, runtime subagent/task-flow helpers, memory search, and usage/cost reporting. It still does not ship the RFC's central capabilities: learned capability profiling/routing, shared blackboard/pub-sub/query storage, private/team/global memory visibility with promotion gates, or global/per-task token/cost budget enforcement. The Octo/teleconference and KinthAI comments are useful prior art, but they are not a merged superseding implementation in this checkout.

## Best Possible Solution

Keep #35203 open as an umbrella RFC until a maintainer decomposes it. The practical path is to split focused follow-ups for capability metrics/routing, blackboard or artifact search/query, memory visibility and promotion gates, and token/cost budget aggregation/enforcement. Plugin-first experiments should use existing runtime.subagent, taskFlow, before_tool_call, llm_output, subagent lifecycle hooks, context-engine, and memory plugin seams where they are sufficient; any missing pre-spawn task-aware routing, visibility, telemetry, or enforcement hook should become a narrow core/plugin API issue.
