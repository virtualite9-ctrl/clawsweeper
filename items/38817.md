## Summary

Keep #38817 open. Current main still does not attach unsupported tool-schema keyword compat to bundled Volcengine/Kimi models, and Volcengine has no provider hook that would strip `minLength`/`maxLength`/`minItems` later. The prior ClawSweeper keep-open comment remains accurate on current main; the two linked fix PRs were closed unmerged.

## Best Possible Solution

Implement this as provider-owned compatibility rather than a hard-coded core provider-name branch: add `unsupportedToolSchemaKeywords` to the affected bundled Volcengine/Ark/Kimi model definitions or have the Volcengine provider contribute the compat via `normalizeResolvedModel`/`contributeResolvedModelCompat` or `normalizeToolSchemas`. Add focused regression coverage for `volcengine`, `volcengine-plan`/Ark, and Kimi model IDs while proving unaffected providers retain validation keywords.
