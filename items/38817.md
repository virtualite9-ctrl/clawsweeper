## Summary

Keep #38817 open. Current main still strips unsupported tool-schema keywords only when resolved model compat declares them, and the bundled Volcengine/Kimi provider does not add that compat or a tool-schema normalization hook. The two linked fix PRs (#38883 and #46887) were closed unmerged in the provided GitHub context, and the touched source paths have not changed since the prior keep-open review.

## Best Possible Solution

Implement provider-owned compatibility for bundled Volcengine/Ark/Kimi models. Add unsupportedToolSchemaKeywords to the affected model definitions or have the Volcengine provider contribute it through normalizeResolvedModel/contributeResolvedModelCompat or normalizeToolSchemas. Add focused regression coverage for volcengine, volcengine-plan/Ark, and Kimi model IDs, plus a negative case proving unaffected providers keep validation keywords.
