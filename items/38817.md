## Summary

Keep #38817 open. Current main still does not attach unsupported tool-schema keyword compatibility to bundled Volcengine/Kimi models, so `minLength`/`maxLength`/`minItems` can still reach the provider unless the user manually configures model compat. The two linked fix PRs for this issue were closed unmerged, and the prior ClawSweeper keep-open conclusion remains accurate on current main.

## Best Possible Solution

Implement provider-owned compatibility for bundled Volcengine/Ark/Kimi models: add `unsupportedToolSchemaKeywords` to the affected model definitions or have the Volcengine provider contribute it through `normalizeResolvedModel`/`contributeResolvedModelCompat` or `normalizeToolSchemas`. Add focused regression coverage for `volcengine`, `volcengine-plan`/Ark, and Kimi model IDs while proving unaffected providers keep validation keywords.
