## Summary

Current main contains new shared embedded-runner safeguards for lost post-compaction retry resolution: it now bounds the compaction-retry wait, falls back to a pre-compaction snapshot instead of hanging the session lane indefinitely, and adds regression tests for both the timeout fallback and multi-retry resolution. That matches the reported 'reacts with 👀 but never responds after compaction' failure mode closely enough to close as implemented on main, but it does not appear to be in the latest release tag yet.
