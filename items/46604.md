## Summary

Keep #46604 open. Current main still does not implement the requested independent Communication Panel plus Tool Activity Panel, and the overlapping #12831 is broader rather than a clear canonical replacement. A recent review comment on #12831 explicitly treated #46604 as the narrower web split-panel follow-up, so closing this as a duplicate would be too aggressive.

## Best Possible Solution

Keep #46604 as the concrete Control UI/web dashboard split-panel tracker. Implement it by routing live tool events into a separately scrollable, collapsible/resizable activity panel while keeping assistant/user messages in a stable communication thread; coordinate any broader observer-mode, TUI, connection-scoped verbose, or subscription work under #12831.
