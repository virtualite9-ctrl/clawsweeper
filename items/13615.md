## Summary

Keep #13615 open. Current main still has adjacent, reactive protections and observability, but not the requested configurable outbound API rate limiting/throttling layer with request/token windows, queue/backpressure, cost budgets, warning thresholds, or an `openclaw limits` CLI. The related implementation PR #13686 is useful prior art but was closed unmerged, and #14206 is a narrower Feishu channel issue rather than a replacement for this tracker.

## Best Possible Solution

Keep #13615 open as the canonical core product/design item. The best path is an opt-in outbound policy layer at generic model-provider, tool/web-search, and channel request boundaries, with config/schema/docs for per-provider/per-model/global RPM, TPM, daily/monthly windows, queue/backpressure controls, 429 retry/backoff behavior, token budget enforcement, threshold warnings/status reporting, and an `openclaw limits` CLI. Treat closed PR #13686 as prior art, and keep #14206 as a narrower Feishu follow-up rather than a substitute.
