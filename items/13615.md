## Summary

Keep #13615 open. Current main still lacks the requested configurable outbound API rate limiting/throttling layer, token/request budget enforcement, queueing/status controls, and `openclaw limits` CLI. Existing code only covers adjacent reactive 429 handling, auth-profile cooldown/fallback, inbound gateway/webhook throttles, and usage readouts. The provided issue discussion also contains a maintainer comment explicitly keeping this issue open, with PR #13686 closed unmerged and #14206 as a narrower Feishu follow-up rather than a replacement.

## Best Possible Solution

Keep #13615 open as the canonical core product/design item. The best implementation path is an opt-in outbound policy layer at model-provider, tool/web-search, and channel request boundaries, with config/schema/docs for provider/model/global RPM, TPM, daily/monthly windows, queue/backpressure controls, 429 retry/backoff behavior, token budget enforcement, warning/status reporting, and an `openclaw limits` CLI. Treat closed PR #13686 as prior art and #14206 as a narrower Feishu channel follow-up, not a replacement.
