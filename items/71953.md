## Summary

Do not auto-close #71953. Current main still appears to have the reported gap: `gateway:watch` builds with `tsdown` and then calls `runRuntimePostBuild`, but that postbuild path does not copy bundled hook `HOOK.md` files, while the standalone `copy-hook-metadata` build step is excluded from the `gatewayWatch` profile. The PR is a focused bug fix with targeted tests and should be reviewed/rebased rather than cleaned up as obsolete.

## Best Possible Solution

Keep the PR open. The best path is to rebase it onto current main, verify that the copy target matches the actual bundled hook discovery directory, and land a tested fix that makes the `gateway:watch` runtime-postbuild path ship `HOOK.md` beside compiled bundled hook handlers without relying on the full-build-only `copy-hook-metadata` step.
