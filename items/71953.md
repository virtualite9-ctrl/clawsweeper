## Summary

Keep PR #71953 open. Current main does not include the proposed runtime-postbuild hook metadata copy, and the build graph still has the exact gap described by the PR: `gatewayWatch` runs `tsdown`, `runtime-postbuild`, and `build-stamp`, while the existing `copy-hook-metadata` step is only part of fuller build profiles. The PR discussion also includes concrete manual verification and a bot review describing focused tests, so this is a plausible bug fix rather than stale or duplicate cleanup.

## Best Possible Solution

Keep the PR open for maintainer review. The likely best fix is to make the `gateway:watch` runtime-postbuild path stage bundled hook `HOOK.md` files in the same location as the compiled bundled hook handlers, with regression coverage in `test/scripts/runtime-postbuild.test.ts` and, if needed, build-profile coverage proving `gatewayWatch` no longer depends on the standalone `copy-hook-metadata` step.
