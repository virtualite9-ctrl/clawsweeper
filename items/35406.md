## Summary

Current main still does not implement the requested safe fallback for binary document formats in the generic `read` tool. The OpenClaw wrapper delegates to the pinned upstream reader before any PDF/Office guard, the sandbox MIME hook only preserves image MIME types, and the existing PDF/document extraction support is exposed through a separate `pdf` tool rather than `read`. Related PR #35494 was closed unmerged, and #57600 is still an open draft implementation vehicle rather than shipped behavior.

## Best Possible Solution

Keep #35406 open and use #57600, or an equivalent maintainer-owned patch, to add a small core guard in `src/agents/pi-tools.read.ts` before `executeReadWithAdaptivePaging`. Reuse `src/media/mime.ts` to detect known PDF/Office document formats, return a concise structured fallback, point PDFs to the existing `pdf` tool where available, and add host plus sandbox regression tests for `.docx`, `.xlsx`, `.pptx`, and `.pdf`.
