## Summary

Keep #35406 open. Current main still does not add a PDF/Office binary-document guard to the generic `read` tool before it delegates to the upstream reader. OpenClaw has MIME detection, a separate `pdf` tool, and a PDF-only document-extract plugin, but those do not prevent `read(path="*.docx")`, `read(path="*.xlsx")`, `read(path="*.pptx")`, or `read(path="*.pdf")` from entering the normal read path. The related fix attempts remain unshipped: #35494 is closed unmerged, and #57600 is still the open draft implementation candidate in the provided GitHub context.

## Best Possible Solution

Keep #35406 open and land #57600 or a maintainer-owned equivalent. The fix should add a small preflight in `src/agents/pi-tools.read.ts` before `executeReadWithAdaptivePaging`, reuse `src/media/mime.ts` for PDF and Office detection, cover both host and sandbox read paths, return a concise fallback that points PDFs to the `pdf` tool and Office files to a parser/conversion path, and add regression tests for `.pdf`, `.docx`, `.xlsx`, and `.pptx`.
