## Summary

Keep #35406 open. Current main still does not implement the requested safe fallback in the generic `read` tool: OpenClaw delegates to the pinned upstream reader before any PDF/Office guard, the sandbox MIME hook only preserves image MIME types, upstream `@mariozechner/pi-coding-agent@0.70.2` decodes non-images as UTF-8, and existing PDF/document extraction is exposed through a separate `pdf`/document-extractor path rather than preventing `read(path="*.docx")` token waste. Related #35494 is closed unmerged and #57600 remains an open draft PR, so neither is shipped behavior.

## Best Possible Solution

Keep #35406 open and land #57600 or a maintainer-owned equivalent that adds a small preflight guard in `src/agents/pi-tools.read.ts` before `executeReadWithAdaptivePaging`. Reuse `src/media/mime.ts` for PDF and Office MIME/extension detection, return a concise structured fallback that points PDFs to the `pdf` tool where available and Office files to a document parser/extractor path, wire the same behavior through sandbox reads, and add regression tests for `.pdf`, `.docx`, `.xlsx`, and `.pptx`. Longer-term automatic Office extraction can build on the document-extractor plugin contract, but the immediate core fix is the safe fallback.
