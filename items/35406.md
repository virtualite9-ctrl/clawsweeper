## Summary

Keep #35406 open. Current main still delegates generic `read` calls to the pinned upstream reader before any PDF/Office document guard, and the sandbox MIME hook deliberately returns only image MIME types. OpenClaw has MIME detection and a separate `pdf` tool, but neither prevents `read(path="*.docx")`/`read(path="*.pdf")` from producing raw UTF-8-decoded binary output. Related PR #35494 is closed unmerged and #57600 is still an open draft candidate, so the requested behavior is not shipped on main.

## Best Possible Solution

Keep #35406 open and land #57600 or a maintainer-owned equivalent that adds a small preflight/fallback in `src/agents/pi-tools.read.ts` before `executeReadWithAdaptivePaging`. Reuse `src/media/mime.ts` for PDF and Office MIME/extension detection, wire both host and sandbox read paths, return a concise structured fallback that points PDFs to the `pdf` tool where available and Office files to a parser/conversion path, and add regression tests for `.pdf`, `.docx`, `.xlsx`, and `.pptx`.
