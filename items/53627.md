## Summary

Keep open. Current main already has Discord DM target support in the actual send path, but this PR’s proposed fix targets the wrong layer and its exact changes are not present on main. I do not have strong enough evidence to close it as implemented or obsolete without a runtime repro on current main.

## Evidence

- **Message tool resolves channel and target separately:** `runMessageAction` picks the provider from `params.channel`, then resolves `params.to`/`target` separately. That means `user:<id>` is treated as a Discord target, not as the selected channel id, so the PR’s channel-selection rationale does not match the current send path.

  - file: [extensions/discord/src/channel.ts:418](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/extensions/discord/src/channel.ts#L418)
