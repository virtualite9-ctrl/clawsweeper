## Summary

Keep #13225 open. Current main still has `before_tool_call` as a pre-execution params/block/approval hook, while model routing remains a run-level `before_model_resolve` capability that runs before session messages are available. The requested per-tool-call delegation contract with preceding assistant output and model/session redirect is not implemented or superseded, and the latest issue discussion explicitly supports keeping it open.

## Best Possible Solution

Keep #13225 open as a plugin/runtime API design request. Preserve the existing `before_tool_call` params/block/approval behavior and run-level `before_model_resolve` model override hook, but only close this after maintainers either ship or explicitly reject a typed contract that lets plugins inspect relevant preceding assistant output and redirect a pending tool call to another model or session. Treat #58079 and #62701 as useful partial context work, not canonical closure for this issue unless they are merged and extended to cover routing.

## Risks / Open Questions

- A routing hook would expose more model/session context and execution authority to plugins, so the eventual design should address plugin permissions, data exposure, failure behavior, and auditability.
- The TTS-style argument injection use case from the comments appears covered by existing params rewrite plus `ctx.agentId`, but that does not satisfy the original model/session delegation request.
