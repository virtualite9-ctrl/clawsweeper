## Summary

Keep #13225 open. Current main still does not provide a pre-tool-call model/session routing contract: `before_tool_call` can rewrite params, block, or require approval, while model selection remains a run-level `before_model_resolve` hook that runs before session messages are available. The per-agent TTS argument injection comment is partly addressed by current `ctx.agentId` plus params rewrite support, but the central request for preceding assistant output plus model/session redirect remains missing. The related open PRs #58079 and #62701 are partial context work, not closure for redirect semantics.

## Best Possible Solution

Keep #13225 open as a plugin/runtime API design request. The best product path is an additive typed contract that preserves existing `before_tool_call` params/block/approval behavior and existing run-level `before_model_resolve`, while either implementing or explicitly rejecting a new per-tool-call route result that can inspect the relevant preceding assistant output and redirect the pending tool call to a selected model/session. Treat #58079 and #62701 as useful context work, not canonical closure unless they are merged and extended to cover routing.
