## Summary

Keep open. Current main still does not accept `agentId` inside `channels.telegram.accounts.<id>`: the Telegram account schema is strict and lacks an account-level `agentId`, while `agentId` is only modeled for Telegram topic config. The docs and community workaround point users to top-level `bindings`, but the linked open PR explicitly uses closing syntax for this issue and is an implementation candidate, so this issue should stay open until maintainers review and land or close that PR.

## Best Possible Solution

Review PR #63380 as the implementation candidate. If maintainers want backward compatibility for configs that worked in 2026.4.5, land a focused fix that accepts or migrates `channels.telegram.accounts.<id>.agentId` with regression coverage. If maintainers want `bindings` to be the only supported route, close the PR with that rationale and improve docs/doctor output so upgraded users can find the migration path.
