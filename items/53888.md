## Summary

Keep open. Current `main` still requires `channels.googlechat.*.appPrincipal` for add-on `app-url` auth, still returns silent 401s on webhook auth mismatch, and still does not document or doctor-check that requirement on the Google Chat user-facing setup surfaces.

## Evidence

- **Webhook auth rejection is still silent:** `createGoogleChatWebhookRequestHandler()` calls `resolveWebhookTargetWithAuthOrReject()` and just returns on `null`; the shared resolver sends `401 unauthorized` / `401 ambiguous webhook target` directly and has no logging path.
  - file: [extensions/googlechat/src/monitor-webhook.ts:159](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/extensions/googlechat/src/monitor-webhook.ts#L159)
  - command: `nl -ba extensions/googlechat/src/monitor-webhook.ts | sed -n '158,192p'; nl -ba src/plugin-sdk/webhook-targets.ts | sed -n '260,280p'`
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **`appPrincipal` is still required for add-on tokens:** `verifyGoogleChatRequest()` still rejects add-on `app-url` tokens with `missing add-on principal binding` when `expectedAddOnPrincipal` is absent, and with `unexpected add-on principal` when it does not match `sub`.
  - file: [extensions/googlechat/src/auth.ts:169](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/extensions/googlechat/src/auth.ts#L169)

  - file: [docs/channels/googlechat.md:163](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/docs/channels/googlechat.md#L163)
