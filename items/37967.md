## Summary

Keep #37967 open. Current main still has adjacent Feishu support only: sender/topic-scoped group sessions, DM-only dynamic agent creation, and subagent delivery hooks for already-spawned thread-bound sessions. It does not automatically create and route per-sender Feishu group subAgents with isolated workspace, skills/config, and memory.

## Best Possible Solution

Keep this open for Feishu plugin product/design work. A solid path is an explicit opt-in group mode that maps Feishu sender or role identity to bounded isolated agent/subagent profiles, provisions workspace/agentDir/skills/model/config/memory, routes replies back to the originating group/thread, adds admin controls and trust-model docs, and covers automatic provisioning plus isolation in tests. If maintainers prefer this outside the bundled Feishu plugin, first define the missing plugin/API contract clearly.
