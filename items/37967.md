## Summary

Keep #37967 open. Current main still has adjacent Feishu support only: sender/topic-scoped group sessions, configured bindings/broadcast, DM-only dynamic agent creation, and explicit thread-bound subagent hooks. It does not automatically provision and route each Feishu group sender or role to an isolated __mc_* subagent with separate workspace, skills/config, memory, and admin controls. The existing ClawSweeper keep-open comment remains applicable; the relevant Feishu files have no diff since that reviewed SHA.

## Best Possible Solution

Keep this open for Feishu plugin product/design work. A solid implementation would add an explicit opt-in group mode that maps Feishu sender or role identity to bounded isolated agent/subagent profiles, provisions workspace/agentDir/skills/model/config/memory, routes replies back to the originating group/thread, adds admin controls and security docs, and covers automatic provisioning plus isolation in tests. If maintainers prefer this outside the bundled Feishu plugin, first define the stable plugin/API contract needed for external implementation.

- A ClawHub/plugin prototype could be reasonable, but the issue targets the bundled Feishu plugin and there is no evidence of an existing complete plugin or stable public Feishu inbound-routing API that makes the core/plugin request obsolete.
