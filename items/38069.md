## Summary

Keep #38069 open. Current main still has one-shot transient retry and recurring error backoff, but it does not implement configurable delayed re-execution for failed recurring cron jobs. The related PR #69252 addressed this exact gap but is closed and unmerged, so it does not supersede the issue.

## Best Possible Solution

Keep #38069 open until recurring cron retry is designed and shipped. The likely path is an additive job/config retry policy in the cron domain types and gateway schema, CLI add/edit flags, explicit retry-attempt state distinct from natural schedule state, transient-error classification for recurring executions, scheduling retries at endedAt + configured delay before the next natural daily/weekly slot, and docs/tests that distinguish one-shot retry, recurring backoff floor, and recurring execution retry.
