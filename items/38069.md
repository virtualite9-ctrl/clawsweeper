## Summary

Keep #38069 open. Current main has one-shot transient retry and a generic recurring error-backoff floor, but it still does not implement configurable delayed re-execution for failed recurring cron jobs. The recurring error path does not use retryOn, transient classification, maxAttempts, or a per-job retry contract, and the related implementation PR #69252 is closed unmerged.

## Best Possible Solution

Keep #38069 open until recurring cron retry is deliberately designed and shipped. The likely maintainer path is an additive retry policy in cron domain types and gateway schemas, CLI add/edit flags, explicit retry-attempt state distinct from natural schedule state, transient-error classification for recurring executions, scheduling at endedAt + configured delay before the next daily/weekly slot, and regression docs/tests that distinguish one-shot retry, recurring backoff floor, and recurring execution retry.
