## Summary

Current main still matches the reported failure boundary: Google/Gemini providers are still treated as tagged reasoning providers while the native Gemini transport still sends `thinkingConfig` with `includeThoughts`. The related context does not provide a canonical shipped fix or duplicate, and the issue has recent corroborating comments, including a `google-vertex` case that is covered by the same Google provider hook alias.

## Best Possible Solution

Keep #69220 open. The likely maintainer fix is to stop forcing tagged reasoning for bundled Google/Gemini providers when native Gemini thinking is used, while preserving Gemini replay sanitization and tool-result repair. Update the provider-utils/Google provider tests to expect native reasoning output where appropriate, add a regression proving the system prompt no longer injects `<think>/<final>` for Gemini native thinking, and keep the existing `thinkingConfig` transport coverage.
