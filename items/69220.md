## Summary

Keep #69220 open. Current main still hardcodes direct Gemini/native Google provider paths to tagged reasoning while also sending Gemini native thinkingConfig includeThoughts payloads. The newer empty/reasoning-only retry logic may mitigate some failed turns, but it does not remove the root conflict reported by multiple users.

## Best Possible Solution

Keep the issue open and fix the bundled/core provider contract: stop returning tagged reasoning for direct Gemini paths that use native Gemini thought parts, update the google-gemini shared family/docs/tests away from tagged where appropriate, and add a regression test for native thought part + text beginning <think> + tool call + empty post-tool assistant turn.
