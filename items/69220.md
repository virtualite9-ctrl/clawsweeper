## Summary

Keep #69220 open. Current main still combines the reported surfaces: direct Google/Gemini providers resolve reasoning output as `tagged`, the system prompt still injects the `<think>`/`<final>` contract when that mode is active, and the native Google transport still sends Gemini `thinkingConfig` with `includeThoughts`/`thinkingLevel` or `thinkingBudget`. Existing bounded retries for reasoning-only or empty Gemini turns are mitigations, not a root fix for the tagged-vs-native conflict described by the reporter and two affected commenters.

## Best Possible Solution

Keep the issue open and fix the provider contract directly: make the direct Google/Gemini provider use native reasoning output when native Gemini `thinkingConfig` is enabled, remove or narrow the `google-generative-ai` tagged fallback, update the `google-gemini` SDK family/docs/tests accordingly, and add a regression test proving Gemini native thought parts plus tool calls do not receive the `<think>/<final>` prompt contract or produce an empty post-tool turn.
