## Summary

Current main still does not expose Mattermost `message action=edit` or `message action=delete`. Mattermost discovery advertises only `send` plus conditional `react`, `supportsAction()` rejects edit/delete, and the shared dispatch gate returns before any handler could run. Existing Mattermost update/delete helpers support preview streaming and interaction cleanup, but they are not wired to the user-facing message action surface requested by #25295. The related block-streaming PRs are closed and unmerged, and the provided discussion already includes a maintainer/Codex keep-open review for the same gap.

## Best Possible Solution

Keep #25295 open, or replace it with a focused Mattermost plugin PR that wires `edit` and `delete` through `describeMattermostMessageTool()`, `supportsAction()`, and `handleAction()` in `extensions/mattermost/src/channel.ts`, reuses or adjusts the existing Mattermost post update/delete client helpers in `extensions/mattermost/src/mattermost/client.ts`, adds targeted action tests plus Mattermost docs, and removes unrelated provider/loader cleanup from the branch.
