## Summary

Keep #25295 open. Current main still does not expose Mattermost `message action=edit` or `message action=delete`: Mattermost discovery advertises only `send` plus conditional `react`, `supportsAction()` rejects edit/delete, and the shared dispatcher returns before any handler could run. Current main does have Mattermost post update/delete helpers for preview streaming and cleanup, but they are not wired to the user-facing message action surface requested by this PR. The related block-streaming PRs are closed and unmerged, and the existing maintainer/Codex comment already called out the same remaining gap.

## Best Possible Solution

Keep #25295 open, or replace it with a focused Mattermost plugin PR that wires `edit` and `delete` through `describeMattermostMessageTool()`, `supportsAction()`, and `handleAction()` in `extensions/mattermost/src/channel.ts`, reuses or adapts the existing Mattermost post update/delete helpers in `extensions/mattermost/src/mattermost/client.ts`, adds targeted action tests and Mattermost docs, validates required edit content, and removes unrelated provider/loader/test cleanup from the branch.
