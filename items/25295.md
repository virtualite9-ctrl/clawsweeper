## Summary

Current main still does not expose Mattermost `message action=edit` or `message action=delete`. Mattermost discovery advertises only `send` and conditional `react`, `supportsAction()` rejects edit/delete, and shared dispatch returns unsupported before any handler can run. Existing Mattermost update/delete helpers are internal preview/interactions plumbing, not the requested message-tool action surface. The related block-streaming PRs are closed and unmerged, and the latest maintainer/Codex discussion already explicitly kept this PR open.

## Best Possible Solution

Keep #25295 open, or replace it with a focused Mattermost plugin PR that wires `edit` and `delete` through `describeMattermostMessageTool()`, `supportsAction()`, and `handleAction()` in `extensions/mattermost/src/channel.ts`, reuses the existing Mattermost client update/delete helpers or an endpoint-appropriate patch helper, adds targeted action tests and Mattermost docs, and drops unrelated provider/loader cleanup from the branch.
