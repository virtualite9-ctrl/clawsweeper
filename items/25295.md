## Summary

Current main still does not expose Mattermost `edit` or `delete` message-tool actions. The PR remains useful because main only has internal post update/delete helpers for preview streaming, not the requested user-facing action discovery and handler surface.

## Best Possible Solution

Keep this open until Mattermost exposes `edit` and `delete` through `describeMessageTool()`, `supportsAction()`, and `handleAction()`, with focused tests and user-facing docs. A refreshed implementation should use the current Mattermost account/runtime patterns and avoid unrelated gate-fix changes from the stale PR branch.
