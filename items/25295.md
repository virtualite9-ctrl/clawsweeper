## Summary

Keep #25295 open. Current main still does not expose Mattermost `message action=edit` or `message action=delete`; Mattermost only discovers/supports `send` and `react`. The existing post update/delete helpers are used internally for preview streaming and interaction cleanup, not the shared message action surface. The related block-streaming PRs are closed and unmerged, and the latest maintainer/Codex discussion already reached the same keep-open conclusion.

## Best Possible Solution

Keep #25295 open, or replace it with a focused Mattermost plugin PR that wires `edit` and `delete` through `describeMattermostMessageTool()`, `supportsAction()`, and `handleAction()` in `extensions/mattermost/src/channel.ts`, reuses the existing `updateMattermostPost()`/`deleteMattermostPost()` client helpers, adds targeted Mattermost action contract/runtime tests, updates `docs/cli/message.md` and `docs/channels/mattermost.md`, and drops unrelated provider/loader cleanup from the branch.
