## Summary

Current `main` already handles Telegram turns that end with no visible final response. The implementation is broader and safer than this PR: it preserves the explicit empty-response fallback for skipped/failed finals, then routes any remaining `!queuedFinal && !sentFallback && !dispatchError && !deliverySummary.delivered` case through the `NO_REPLY` policy fallback, with tests for DM rewrite and allowed group silence. The same behavior is present in release `v2026.4.22`.
