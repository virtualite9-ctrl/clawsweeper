## Summary

Keep #33413 open. Current main still does not update Slack `assistant.threads.setStatus` with tool-level labels; it only sets the static reply-level status and uses separate draft-preview/status-reaction progress for tool starts.

## Best Possible Solution

Keep the issue open until Slack dispatch maps `onToolStart` events to `ctx.setSlackThreadStatus` for assistant-thread targets, with a small tool-name-to-label helper and focused Slack coverage asserting the extra `assistant.threads.setStatus` call. If maintainers choose draft-preview progress/status reactions as the replacement UX, document that product decision and link a canonical superseding issue or PR before closing.
