## Summary

Keep #33413 open. Current main still only sets Slack assistant thread status to the static "is typing..." value at reply start and clears it at reply end. Tool-start events are available and Slack dispatch handles them, but the handler only updates reactions and draft-preview progress, not `assistant.threads.setStatus` with tool-level labels. The related implementation PR #33420 is closed unmerged, and #67129 is related customization work rather than a shipped or canonical replacement.

## Best Possible Solution

Keep the issue open until Slack dispatch maps `onToolStart` events to `ctx.setSlackThreadStatus` for assistant-thread targets, with a small tool-name-to-label helper and focused Slack coverage asserting the extra `assistant.threads.setStatus` call. If maintainers choose draft-preview progress or configurable `loading_messages` as the replacement UX, document that product decision and link a canonical superseding issue or PR before closing #33413.
