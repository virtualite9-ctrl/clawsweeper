## Summary

Keep #33413 open. Current main still has the core `onToolStart` signal and Slack's assistant thread status wrapper, but Slack dispatch only sets the assistant thread status to the static `"is typing..."` text at reply start and clears it at the end. Tool starts update status reactions and draft-preview progress lines, not `assistant.threads.setStatus` with human-readable tool labels. The related PR #33420 was closed unmerged per the provided GitHub context, and #67129 is adjacent customization work rather than a canonical replacement for tool-level assistant thread status.

## Best Possible Solution

Keep #33413 open until Slack dispatch maps `onToolStart` events to `ctx.setSlackThreadStatus` for assistant thread targets, with a small tool-name-to-label helper and a focused Slack test that simulates a tool start and asserts an extra `assistant.threads.setStatus` call. If maintainers intentionally choose draft-preview progress as the replacement UX, close only after documenting that product decision and linking a canonical superseding issue or PR.
