## Summary

Keep #33413 open. Current main still has the core tool-start signal and Slack assistant thread status wrapper, but Slack dispatch only sets `assistant.threads.setStatus` to the static `"is typing..."` value at reply start and clears it at the end. Tool starts currently update Slack status reactions and draft-preview progress lines, not the assistant thread status with human-readable tool labels. The related PR #33420 was closed unmerged, and #67129 is adjacent status customization work rather than a canonical replacement for tool-level assistant thread status.

## Best Possible Solution

Keep #33413 open until Slack dispatch maps `onToolStart` events to `ctx.setSlackThreadStatus` for assistant thread targets, with a small tool-name-to-label helper and focused Slack test coverage that simulates a tool start and asserts the extra `assistant.threads.setStatus` call. If maintainers intentionally choose draft-preview progress as the replacement UX, close only after documenting that product decision and linking a canonical superseding issue or PR.
