## Summary

Keep #70471 open. Current main does not contain the PR's central Feishu comment-delivery hardening: no strict recipient gate, no duplicate final-reply delivery guard, no angle-bracket escaping for Drive comment writes, no account-scoped comment typing state, and no force-fresh/credential-aware bot identity probe. The related #66427 is closed and unmerged in the provided context, so it is not a canonical shipped/superseding fix.

## Best Possible Solution

Keep #70471 open for maintainer review or revision. The best path is to land a focused Feishu plugin fix that adds recipient gating, successful tool-delivery suppression of duplicate finals, Drive comment text escaping, account-scoped typing cleanup, and credential-aware fresh bot identity probing, with the PR's targeted regression tests or equivalent coverage. If maintainers prefer the broader comment-policy work from #66427, reopen/replace with a canonical PR; do not close #70471 as implemented against current main.
