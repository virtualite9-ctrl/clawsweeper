## Summary

Keep #10659 open. Current main has useful SecretRef infrastructure, host-env sanitization, redaction heuristics, and plugin hooks that make the linked workaround/plugin approaches credible, but it still does not ship the native masked-secrets behavior requested here: no `{{secret:NAME}}` runtime substitution, no `secrets.mask`/`maskedSecrets` config surface, no native command block for reading `.env`/printing arbitrary secret variables, and no guaranteed secret-value redaction of exec/process output before the agent sees it. The same-day issue discussion also contains a maintainer-associated Codex review explicitly keeping this open.

## Best Possible Solution

Keep #10659 open until OpenClaw ships a native masked-secret or credential-broker path for agent-executed commands and process output, or maintainers explicitly supersede it with a canonical design such as #59165. Continue to acknowledge SecretRefs, openclaw-credential-vault, and jmkritt/openclaw-secrets-hardening as useful mitigations, but do not treat them as closing this core security/product request.
