## Summary

Keep open. Current main still only has opt-in, generic tool-loop detection for repeated identical tool calls. It can warn on repeated identical `read` arguments when enabled, but it does not implement the requested same-turn first-class `read` reread guard, same-small-file-set detection, or edit/write/test progress-aware suppression. The related #63352 remains complementary and exec-scoped, not a canonical replacement.

## Best Possible Solution

Keep this open and implement a warning-only read-specific layer on top of the current tool-loop detection path: normalize `read` `path` and `file_path` identities per run/session, detect repeated same-file and same-small-file-set rereads in one turn, reset or suppress after edit/write/test or broadened-context progress, emit a soft nudge plus diagnostic event, and cover it with focused tests and docs/config guidance.
