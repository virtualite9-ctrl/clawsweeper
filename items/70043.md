## Summary

Current main still matches the reported behavior: unset Telegram `reactionLevel` resolves to `minimal`, the Telegram prompt adapter forwards that as reaction guidance, and the system prompt emits the exact `MINIMAL` line. The provided related items are only loose title overlaps and do not track this Telegram reaction default/prompt-injection stability issue.

## Best Possible Solution

Keep the issue open. Maintainers should decide the product contract for unset Telegram reaction levels in existing sessions, then implement it in the Telegram plugin resolver or prompt adapter. Likely fixes are to inject agent reaction guidance only when `channels.telegram.reactionLevel` is explicitly configured, add a migration/doctor notice for existing configs, or otherwise make the default prompt behavior stable and unsurprising. Add regression coverage for unset config, account fallback, and prompt composition, and align the stale TypeScript default comment.
