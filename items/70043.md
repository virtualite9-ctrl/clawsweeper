## Summary

Current main still has the reported Telegram behavior: an unset Telegram `reactionLevel` resolves to `minimal`, the Telegram channel adapter forwards that as prompt reaction guidance, and the system prompt emits `Reactions are enabled for Telegram in MINIMAL mode.` This matches the issue's observed symptom rather than proving it fixed or obsolete.

## Best Possible Solution

Keep the issue open. A conservative fix should decide the product contract for unset Telegram reaction levels in existing sessions, then implement it in the Telegram plugin prompt adapter or reaction-level resolver. Likely options are: only inject agent reaction guidance when `channels.telegram.reactionLevel` is explicitly configured, introduce a migration/doctor notice for existing configs, or make the default prompt behavior visibly stable across session restarts. Add regression tests around unset Telegram config, account fallback, and prompt composition, and align the stale TypeScript comment that still says `ack` is the default.
