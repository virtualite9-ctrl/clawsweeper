## Summary

Keep open. Current main has fixes for the Telegram reasoning/commentary leak class, but the same-target message-tool follow-up suppression path remains in source and tests. A maintainer comment in the discussion explicitly called out this remaining file-send + immediate text-reply loss as still valid.

## Best Possible Solution

Keep this open and fix the remaining message-tool/file-send path in core reply payload construction: narrow same-target message-tool suppression to exact duplicates, preserve distinct final text after media/file sends, and update buildReplyPayloads, follow-up delivery, and the existing suppression tests to cover Telegram file/media sends followed by user-visible text.
