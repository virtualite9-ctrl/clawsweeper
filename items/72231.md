## Summary

Current main already fixes the Windows ESM raw drive-letter failure reported in #72231. The shared plugin Jiti policy now disables native Jiti on Windows, the generic plugin loader converts Windows absolute paths to file:// URLs before module loading, and regression tests cover the affected channel/plugin loader seams.

## Best Possible Solution

Close #72231 as implemented on main. Keep the shared Windows-safe loader policy and path-to-file-URL normalization; users still on 2026.4.24 should move to the newer build containing the 2026.4.25 Windows/native loader fix.

## Evidence
