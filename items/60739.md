## Summary

Current `main` implements the reported behavior: global plugins under `~/.openclaw/extensions` are still discovered, CLI-tracked installs can override bundled duplicates, and gateway startup now includes the explicitly selected memory-slot plugin. The changelog records this as shipped in `v2026.4.10`.

## Evidence

- **Global extension root is still discovered:** Plugin roots resolve the global plugin directory as `<configDir>/extensions`, and discovery scans that directory as `origin: "global"`.
  - file: [src/plugins/roots.ts:21](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/src/plugins/roots.ts#L21)

- **Tracked global installs override bundled duplicates:** Duplicate ordering gives a global plugin higher precedence than a bundled plugin when the global candidate matches CLI-managed install provenance (`plugins.installs.*.installPath`).
  - file: [src/plugins/loader.ts:1697](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/src/plugins/loader.ts#L1697)

  - file: [src/plugins/manifest-registry.test.ts:371](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/src/plugins/manifest-registry.test.ts#L371)

  - file: [src/plugins/loader.test.ts:5846](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/src/plugins/loader.test.ts#L5846)

## Risks / Open Questions

- `docs/plugins/manifest.md` still summarizes duplicate precedence as bundled-over-global and does not spell out the tracked-install override, which could confuse operators reading only the docs.
- The override depends on CLI-managed `plugins.installs.<id>` provenance; stale or hand-edited config without that metadata can still fall back to the bundled copy.
