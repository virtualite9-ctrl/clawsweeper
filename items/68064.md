## Summary

Keep open. Current main still has no native default or inheritance path for responseUsage on new Discord thread sessions; the prior ClawSweeper keep-open finding still matches the current code at 7d9dc8cf24fc143bec95318491e0247c37863e63.

## Best Possible Solution

Keep this issue open. The best fix is a small, tested session/core change: add a validated responseUsage default and/or copy parentEntry.responseUsage when creating a new thread session from ParentSessionKey, while preserving /usage off|tokens|full as a per-session override. Cover the chosen behavior with initSessionState, /usage, and Discord auto-thread tests, then update the token-use and Discord thread docs.
