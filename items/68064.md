## Summary

Keep open. Current main still has no native default or inheritance path for `responseUsage`; it remains session-entry state, and Discord thread creation only passes a `ParentSessionKey` for transcript forking, not a usage-footer preference.

## Best Possible Solution

Keep this issue open. The best fix is a small, tested session/core change: add a validated responseUsage default and/or copy `parentEntry.responseUsage` when creating a new thread session from `ParentSessionKey`, then preserve `/usage off|tokens|full` as a per-session override even when a default exists. Cover the chosen path with targeted `initSessionState`, `/usage`, and Discord auto-thread tests, and update the token-use plus Discord thread docs.
