## Summary

Keep #70253 open. Current main still matches the reporter's follow-up analysis: the main Mattermost inbound reply path hard-disables core block streaming instead of consulting account blockStreaming, Mattermost has no allowBots-style bot-to-bot config surface, and tools.agentToAgent is scoped to session tools rather than shared-channel fan-out.

## Best Possible Solution

Keep the issue open and split it into maintainer-trackable work if needed: first, make Mattermost inbound replies honor the documented/configured streaming and blockStreaming settings with regression coverage; second, decide and implement an explicit Mattermost bot-to-bot/channel A2A policy, likely with an allowBots-style config and any shared routing API changes needed for channel fan-out.
