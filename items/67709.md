## Summary

Keep this issue open. Current main still has a plausible shared stall path matching the report: Control UI can request `chat.history` and `models.list` together, both can wait on `loadGatewayModelCatalog()`, and the browser RPC layer has no timeout for an already-dispatched slow request.

## Best Possible Solution

Keep the issue open and fix the startup responsiveness path: make `chat.history` avoid waiting on full model catalog discovery just to fill a missing thinking default, make Control UI model catalog loading backgrounded or timeout-bounded, and add a regression test with a deferred `loadGatewayModelCatalog()` proving initial history/rendering remains responsive while `models.list` is still loading.
