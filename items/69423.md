## Summary

Keep #69423 open. Current main still contains the exact Anthropic transport crash path: tool conversion assumes every tool has `parameters`, while plugin registration and resolution still forward plugin tools without runtime schema validation. The related PR #69432 directly targeted this bug, but the provided context shows it was closed unmerged and the fix is not present in the checked-out main tree.

## Best Possible Solution

Keep the issue open and land a focused core fix in `src/agents/anthropic-transport-stream.ts`: normalize `tool.parameters`/`tool.inputSchema` or safely default/skip malformed tools without throwing, then add Anthropic regression tests for an `inputSchema`-only plugin tool and a fully missing schema. Plugin registration-time validation and `openclaw status` diagnostics are useful follow-ups, but the crash guard belongs in the Anthropic request-building path first.
