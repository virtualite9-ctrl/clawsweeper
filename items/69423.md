## Summary

Current main still contains the reported failure mode. The Anthropic transport converts every tool by reading `tool.parameters.properties` with no runtime guard or `inputSchema` fallback, while plugin tool resolution accepts registered tool objects after only name/conflict checks. The related PR #69432 directly targeted this issue but was closed unmerged, so the issue should stay open.

## Best Possible Solution

Keep #69423 open. A focused maintainer fix should guard `convertAnthropicTools()` against missing/non-object schemas, decide whether to normalize `inputSchema` as a compatibility alias or reject it with a plugin diagnostic, and add regression coverage for plugin tools with `inputSchema` and with no schema so one broken plugin cannot crash all Anthropic runs.
