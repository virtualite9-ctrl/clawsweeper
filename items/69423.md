## Summary

Keep #69423 open. Current main still has the reported Anthropic failure path: the Anthropic adapter unconditionally reads `tool.parameters.properties`, plugin registration/resolution does not reject or normalize malformed third-party tools, and the general schema normalizer returns tools unchanged when `parameters` is absent. The related PR #69432 proposed the right kind of guard but was closed unmerged, so it does not supersede the issue.

## Best Possible Solution

Keep the issue open and fix the Anthropic tool conversion path. A conservative implementation should normalize each tool schema before building the Anthropic payload, either accepting `parameters` plus a deliberate `inputSchema` compatibility alias or rejecting malformed plugin tools with clear plugin diagnostics. Add regression tests for `inputSchema`-only and schema-less plugin tools, and consider a separate plugin-registration/status diagnostic for install-time or load-time visibility.
