## Summary

Keep open. Current main still lacks REST session-management endpoints for `/v1/chat/completions` integrations. Session reset/compact/status/model controls remain available through Gateway RPC and internal/native paths, not the requested `/v1/sessions/*` HTTP surface. The related discussion also points the same way: #23941 was closed unmerged, #29985 was described as only covering plugin SDK reset with REST endpoints still a follow-up, and the latest maintainer comment explicitly kept this item open.

## Best Possible Solution

Keep #20934 open as the canonical REST session-management tracker. A good implementation would add authenticated Gateway HTTP endpoints for reset/status/compact and likely model/reasoning via existing `sessions.reset`, `sessions.list` or status data, `sessions.compact`, and `sessions.patch` semantics, with route tests, scope/auth coverage, and docs under the OpenAI-compatible HTTP API. If maintainers prefer plugin ownership, first add a documented plugin/runtime session-control API; current plugin HTTP routes alone do not provide the missing reset/control primitive cleanly.
