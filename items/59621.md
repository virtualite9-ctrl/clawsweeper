## Summary

Current `main` already supports the requested Docker dependency path through the documented opt-in `OPENCLAW_DOCKER_APT_PACKAGES` build arg, and the linked ffmpeg PR was closed because hard-coding large skill dependencies into every base image conflicts with the Dockerfile’s existing architecture.

## Evidence

- **Dockerfile supports extra skill/system packages at build time:** The runtime Dockerfile exposes `OPENCLAW_DOCKER_APT_PACKAGES` and installs any space-separated apt packages passed at build time, which covers the missing-skill-dependency use case without baking every optional tool into the base image.
  - file: [Dockerfile:199](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/Dockerfile#L199)
  - command: `nl -ba Dockerfile | sed -n '160,240p'`
  - sha: [a9797214338b](https://github.com/openclaw/openclaw/commit/a9797214338ba31c52c796adbb75afb16e0684a9)
- **Docker setup forwards the apt package arg into image builds:** `scripts/docker/setup.sh` exports `OPENCLAW_DOCKER_APT_PACKAGES`, persists it, and passes it through `docker build --build-arg`, so Docker deployments can actually use the supported dependency-install path.
  - file: [scripts/docker/setup.sh:279](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/scripts/docker/setup.sh#L279)
  - command: `nl -ba scripts/docker/setup.sh | sed -n '268,490p'`
  - sha: [a9797214338b](https://github.com/openclaw/openclaw/commit/a9797214338ba31c52c796adbb75afb16e0684a9)
- **Docker docs already document the supported workflow:** The Docker install docs document `OPENCLAW_DOCKER_APT_PACKAGES` as the way to install extra apt packages during image build and include a concrete 'Bake system deps' example.
  - file: [docs/install/docker.md:121](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/docs/install/docker.md#L121)
  - command: `nl -ba docs/install/docker.md | sed -n '110,320p'`
  - sha: [a9797214338b](https://github.com/openclaw/openclaw/commit/a9797214338ba31c52c796adbb75afb16e0684a9)
- **The cited skill dependencies map to the existing opt-in mechanism:** Bundled skills in the repo declare `gh` and `ffmpeg` requirements (`skills/github/SKILL.md`, `skills/video-frames/SKILL.md`, `skills/camsnap/SKILL.md`), which are exactly the kind of optional binaries the Docker apt-packages arg is meant to provide.
  - file: [skills/github/SKILL.md:3](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/skills/github/SKILL.md#L3)
  - command: `nl -ba skills/github/SKILL.md | sed -n '1,40p'; nl -ba skills/video-frames/SKILL.md | sed -n '1,35p'; nl -ba skills/camsnap/SKILL.md | sed -n '35,55p'`
  - sha: [ec3dbd22a4de](https://github.com/openclaw/openclaw/commit/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46)
- **Related PR confirms hard-coding ffmpeg was intentionally rejected:** PR #59712 proposed adding `ffmpeg` unconditionally, but the review on that PR explicitly pointed to the existing `OPENCLAW_DOCKER_APT_PACKAGES` opt-in path and rejected the image-bloat tradeoff. That discussion is directly tied to this issue.

## Risks / Open Questions
