## Summary

Keep #60996 open. Current main has not implemented the central PR request: `argPattern` matching is still Windows-only, `ask="auto"` is not a supported exec ask mode, and the documented/UI-visible exec policy still exposes only `off`, `on-miss`, and `always`. The PR is also not safely mergeable as-is because the supplied review comments identify P1 bugs in the proposed subcommand expansion and `ask="auto"` approval behavior, so this is unresolved work rather than cleanup-close material.

## Best Possible Solution

Keep the item open, but ask for a narrower, tested revision. The best path is to split the work: first implement cross-platform `argPattern` matching with explicit argv-available semantics, POSIX and Windows tests, and docs that match the real matcher; then separately design `ask="auto"` as a product/security policy change across config normalization, session directives, gateway/node execution, Control UI, docs, and tests, with unknown commands failing to prompt rather than auto-allowing.
