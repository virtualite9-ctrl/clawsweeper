## Summary

Current main covers the reported Bedrock bearer-token proxy use case. Native Bedrock Converse can now use `AWS_BEARER_TOKEN_BEDROCK` without SigV4 signing, and the bundled Mantle provider covers AWS's official OpenAI-compatible bearer-auth surface. The exact literal `auth: "api-key"` + `apiKey` shape is not the native Bedrock route, but the observable blocker is addressed.

## Best Possible Solution

Close this issue as implemented. Keep the shipped native Bedrock bearer-token path via `AWS_BEARER_TOKEN_BEDROCK`, direct official OpenAI-compatible Bedrock users to the Mantle provider, and track any future literal config-only alias as a separate narrow enhancement if maintainers want that syntax.
