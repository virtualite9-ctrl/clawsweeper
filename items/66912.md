## Summary

Keep open. Current main still downloads media from `msg.reply_to_message` without checking whether the replied-to message was authored by the bot, while the existing self-authored skip only covers the incoming update itself. The latest provided PR head appears to wire the intended guard, but it still needs focused regression coverage and should drop the unrelated generated Plugin SDK baseline change before maintainer review.

## Best Possible Solution

Keep this PR open and ask for a clean Telegram plugin patch: add the self-authored reply-target guard in `resolveReplyMediaForMessage`, add regression coverage in `extensions/telegram/src/bot.test.ts` for `reply_to_message.from.id === ctx.me.id` with no media fetch, preserve normal reply-media ingestion coverage, and remove the unrelated `docs/.generated/plugin-sdk-api-baseline.sha256` change before maintainer review.
