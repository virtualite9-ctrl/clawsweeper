## Summary

Current main already implements the heartbeat timer overflow fix requested by PR #61386. The scheduler now clamps oversized heartbeat delays through a shared safe timer helper, regression tests cover the tight-loop failure mode, and the v2026.4.24 changelog records the shipped canonical fix via #71478 / #71414, making this open PR obsolete.

## Best Possible Solution

Close PR #61386 as implemented on main and keep the shipped shared timer-delay helper plus heartbeat scheduler regression coverage from the canonical #71478 fix. Any future timer overflow work should extend the shared helper or add focused call-site tests rather than reviving this duplicate PR.
