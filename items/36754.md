## Summary

Current main still matches the reported bug: extension-root discovery uses Dirent file/directory checks and continues past symlink entries before plugin package, bundle, or index discovery runs. The explicit plugins.load.paths workaround remains separate because discoverFromPath uses fs.statSync(), but auto-discovery from the global and workspace extensions roots is still not fixed. Closed PR attempts in the provided context were not merged and do not supersede the issue.

## Best Possible Solution

Keep #36754 open. The narrow product fix belongs in src/plugins/discovery.ts: resolve symlink Dirents in discoverInDirectory before the file/directory branch, skip or diagnose broken symlinks, then reuse the existing package, bundle, index, and boundary validation paths. Add a regression test for a symlinked plugin directory under a global or workspace extensions root, while keeping the existing symlink escape tests intact.
