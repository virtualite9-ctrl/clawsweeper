## Summary

Current main still matches the reported bug: extension-root discovery reads Dirents, handles only regular files and directories, and silently skips symlinked plugin directories before package, bundle, or index discovery can run. The `plugins.load.paths` workaround remains valid because that path uses `fs.statSync()` separately, but it does not fix auto-discovery from `~/.openclaw/extensions` or `<workspace>/.openclaw/extensions`.

## Best Possible Solution

Keep #36754 open. The best fix is a narrow change in `discoverInDirectory` that resolves symlink Dirents before the file/directory decision, skips or diagnoses broken symlinks, reuses existing package/runtime entry validation, and adds a regression test proving a symlinked plugin directory under the global or workspace extensions root is discovered.
