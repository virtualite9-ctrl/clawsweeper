## Summary

Current main still matches #36754: auto-discovery of global/workspace plugin roots uses Dirent file/directory checks and continues past symlink entries before package, bundle, or index discovery can run. The explicit plugins.load.paths workaround follows symlinks through fs.statSync(), but that does not fix scanning ~/.openclaw/extensions or <workspace>/.openclaw/extensions. Closed PR attempts in the provided context were not merged and do not supersede the issue.

## Best Possible Solution

Keep #36754 open. The narrow fix belongs in src/plugins/discovery.ts: resolve symlink Dirents in discoverInDirectory before the file/directory branch, skip or diagnose broken symlinks, then reuse the existing package, bundle, index, and candidate validation paths. Add a regression test for a symlinked plugin directory under a global or workspace extensions root while keeping the existing symlink escape tests intact.
