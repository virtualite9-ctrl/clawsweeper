## Summary

Current `main` and the latest shipped release already request Discord `MessageContent` by default; the missing `channels.discord.intents.messageContent` key is intentional because only the optional `presence` and `guildMembers` intents are configurable.

## Evidence

- **Gateway always requests Message Content:** `resolveDiscordGatewayIntents()` unconditionally ORs `carbonGateway.GatewayIntents.MessageContent` into the Discord gateway bitmask, while only `presence` and `guildMembers` are conditional.
  - file: [extensions/discord/src/monitor/gateway-plugin.ts:36](https://github.com/openclaw/openclaw/blob/8b76392e3e7934b1cdd6d461148263fd56141ec8/extensions/discord/src/monitor/gateway-plugin.ts#L36)
  - command: `nl -ba extensions/discord/src/monitor/gateway-plugin.ts | sed -n '36,54p'`
  - sha: [8b76392e3e79](https://github.com/openclaw/openclaw/commit/8b76392e3e7934b1cdd6d461148263fd56141ec8)
- **Config schema intentionally omits a messageContent toggle:** The Discord config schema defines `channels.discord.intents` with only `presence` and `guildMembers`; there is no `messageContent` key because that intent is not modeled as optional config.
  - file: [src/config/zod-schema.providers-core.ts:611](https://github.com/openclaw/openclaw/blob/8b76392e3e7934b1cdd6d461148263fd56141ec8/src/config/zod-schema.providers-core.ts#L611)

- **Docs require enabling Message Content in Discord portal:** The user-facing Discord setup docs explicitly say `Message Content Intent` is required, while `Presence Intent` is optional.
  - file: [docs/channels/discord.md:34](https://github.com/openclaw/openclaw/blob/8b76392e3e7934b1cdd6d461148263fd56141ec8/docs/channels/discord.md#L34)
