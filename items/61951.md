## Summary

Keep open. Current `main` contains a generic guard against reentrant provider-plugin loading, and tests cover that narrow stack-overflow path, but I did not find direct proof that it resolves this memory-core Dreaming/manual `memory promote --apply` crash with `google`/`minimax`. The provided timeline also links a newer open follow-up (#62806) reporting bundled provider-plugin stack overflow after the earlier guard, so the close evidence is not strong enough.
