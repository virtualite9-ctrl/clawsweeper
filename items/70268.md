## Summary

Keep PR #70268 open. Current main still has the reported wake-hook routing gap: direct `/hooks/wake` accepts only text/mode, mapped wake actions do not carry `agentId` or `sessionKey`, and the gateway wake dispatcher always enqueues into the configured main session. The PR remains relevant to open bug #64556 and is not implemented or superseded on main.

## Best Possible Solution

Keep PR #70268 open for maintainer review. The best path is to land this focused fix or an equivalent implementation that carries `sessionKey`/`agentId` through direct and mapped wake hooks, canonicalizes the target consistently before enqueue and heartbeat, enforces the same session-prefix policy in all wake routing branches, and adds regression coverage for #64556 plus the prefix-policy cases.
