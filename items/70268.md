## Summary

Keep open. Current main still drops wake-hook session targeting and routes direct/mapped wake events through the default main session, while the PR is a focused implementation candidate for #64556. It needs maintainer review and likely a rebase/adaptation to the current split hook handler before landing, not cleanup closure.

## Best Possible Solution

Keep the PR open. Rebase/adapt the implementation onto current main, moving the request-handler changes into `src/gateway/server/hooks-request-handler.ts` while keeping the helper and dispatcher changes in `src/gateway/hooks.ts`, `src/gateway/hooks-mapping.ts`, and `src/gateway/server/hooks.ts`. Then review/land it if targeted gateway tests and the changed gate pass, preserving the maintainer-requested prefix-policy and global-scope safeguards.
