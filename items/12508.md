## Summary

Keep #12508 open. Current main still has sequential in-process plugin hook chains that permit legitimate hook mutation of tool parameters, outbound message content, persisted tool-result messages, and pre-write messages, but there is no host-enforced checksum/signature handoff, per-handler tamper detection, modification audit trail, or manifest/config per-hook allow-list. The existing April 26, 2026 maintainer-side Codex review comment also kept this open, and current source/config/tests still support that conclusion.

## Best Possible Solution

Keep this open for core plugin-runtime security design. The likely fix belongs in the core hook runner/registry/config/docs contract, with tests around sequential hook mutation, per-hook permissions, deterministic modification auditing, and host-enforced integrity checks. This should not be moved to ClawHub because an external plugin cannot reliably enforce integrity between other plugins inside the core hook runner.
