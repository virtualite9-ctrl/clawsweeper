## Summary

Keep #12508 open. Current main still has core-owned sequential plugin hook runners with mutation/decision results, but no host-enforced hook-chain integrity, per-handler checksum/signature verification, modification audit trail, or per-hook manifest allow-list. The recent maintainer-side Codex comment already kept this open, and the relevant source on current main still supports that conclusion.

## Best Possible Solution

Keep this open for core plugin-runtime security design. The likely path is a core-owned change across the hook runner, registry/config policy, plugin manifest/docs contract, and focused tests for sequential hook mutation, per-hook permissions, deterministic modification auditing, and host-enforced integrity checks. This should not move to ClawHub unless core first exposes an API that lets an external plugin enforce these guarantees across the full hook chain.
