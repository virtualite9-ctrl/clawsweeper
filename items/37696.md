## Summary

Keep #37696 open. Current main still has the reported strict-mode behavior: an unclosed reasoning tag leaves the parser inside a thinking block, so strict mode drops the trailing assistant text. Existing tests still assert the empty/drop behavior, delivery sanitization still uses strict mode, and linked fix PR #51915 remains open rather than shipped.

## Best Possible Solution

Keep the issue open until a focused fix lands, likely by updating `stripReasoningTagsFromText` to recover useful text after malformed/unclosed reasoning tags or by adding an explicit safer fallback that prevents silent empty delivery. PR #51915 is the current open candidate to review or supersede; any fix should update the strict-mode tests and relevant delivery/channel expectations.
