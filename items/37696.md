## Summary

Keep #37696 open. Current main still has the reported strict-mode sanitizer behavior: an unclosed `<think>` leaves `inThinking` true, so strict mode drops the remaining text. Checked-in tests still assert that loss, assistant extraction still turns an unclosed think-only response into `""`, and shared delivery/channel preview paths still use strict stripping and skip empty text. The related fix PRs in the provided discussion are either closed unmerged or, for #51915, still open and unshipped.

## Best Possible Solution

Keep #37696 open until a focused fix lands. The likely path is to update `stripReasoningTagsFromText` strict-mode behavior and regression tests so malformed or unclosed reasoning tags cannot erase all deliverable assistant text, then verify assistant extraction plus channel delivery/preview paths. #51915 is the current open fix candidate to merge, revise, or supersede; if maintainers intentionally want strict-mode discard for privacy, document that contract and add a delivery-level fallback so users do not get silent non-delivery.
