## Summary

Current main still matches the reported failure mode. `stripReasoningTagsFromText` still omits the tail after an unclosed reasoning tag in strict mode, checked-in tests still assert that strict unclosed `<think>` output collapses to a prefix or empty string, and delivery/channel paths still use strict stripping before deciding whether there is text to send. The related fix PRs in the provided GitHub context are either closed unmerged (#37703, #37861, #46205) or still open as an unshipped fix candidate (#51915).

## Best Possible Solution

Keep #37696 open until a focused fix lands. The likely implementation belongs in `stripReasoningTagsFromText` and its strict-mode regression coverage, with `extractAssistantText` and delivery/channel tests updated so malformed or unclosed reasoning tags cannot erase all deliverable assistant text. #51915 is the current open fix candidate to merge, revise, or supersede; if maintainers intentionally want strict-mode discard for privacy, document that contract and add a channel-level fallback so users do not get silent non-delivery.
