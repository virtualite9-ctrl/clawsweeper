## Summary

Current main still matches the reported bug. `stripReasoningTagsFromText` continues to omit trailing text after an unclosed reasoning tag in strict mode, checked-in tests still assert that strict unclosed `<think>` output collapses to a prefix or empty string, and delivery/channel paths still use strict stripping. The linked fix attempts #37703, #37861, and #46205 are closed unmerged, while #51915 remains an open fix candidate rather than shipped behavior.

## Best Possible Solution

Keep #37696 open until a focused fix lands. The likely path is to update `stripReasoningTagsFromText` so malformed or unclosed reasoning tags cannot erase all deliverable assistant text, adjust strict-mode and `extractAssistantText` regression tests, and either merge or supersede #51915 with the chosen safety semantics. If maintainers prefer strict-mode discard for privacy reasons, document that sanitizer contract explicitly and add channel-level fallback behavior so users do not see silent non-delivery.
