## Summary

Keep #37696 open. Current main still has the reported strict-mode behavior: an unclosed `<think>` leaves `inThinking=true`, so the sanitizer does not append the remaining text. Checked-in tests still assert that strict unclosed tags collapse to a prefix or empty assistant text, and delivery/preview paths still skip empty sanitized text. The related fix PRs in the provided context are closed unmerged except #51915, which is still an open unshipped fix candidate.

## Best Possible Solution

Keep #37696 open until a focused fix lands. The likely path is to update `stripReasoningTagsFromText` strict-mode behavior and regression tests so malformed or unclosed reasoning tags cannot erase all deliverable assistant text, then verify assistant extraction and channel delivery paths. #51915 is the current open fix candidate to merge, revise, or supersede; if maintainers intentionally want strict-mode discard for privacy, document that contract and add a channel-level fallback so users do not get silent non-delivery.
