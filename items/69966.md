## Summary

Keep #69966 open. Current main still treats `contextInjection`, `bootstrapMaxChars`, and `bootstrapTotalMaxChars` as global `agents.defaults` settings. There is no accepted `agents.list[]` field, named bootstrap profile surface, agent-aware bootstrap resolver, or Control UI per-agent profile workflow. The related context is adjacent but not a canonical duplicate; #13736 tracks per-agent compaction settings, not bootstrap profile settings.

## Best Possible Solution

Keep the issue open for a core config/runtime/UI implementation. A good maintainer path is to add explicit per-agent fields or named profile references for `contextInjection`, `bootstrapMaxChars`, and `bootstrapTotalMaxChars`, define fallback precedence from `agents.defaults`, update strict schema/types/help/docs/generated metadata, and thread agent-aware resolution through embedded and CLI bootstrap preparation with schema and runtime tests.
