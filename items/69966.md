## Summary

Keep #69966 open. Current main still treats bootstrap profile settings as global defaults: per-agent config types and strict validation do not expose `contextInjection`, `bootstrapMaxChars`, or `bootstrapTotalMaxChars` on `agents.list[]`, runtime bootstrap resolution reads only `agents.defaults.*`, and the Control UI profile presets patch only `agents.defaults`. The provided related items are adjacent per-agent/config requests but do not supersede this specific bootstrap profile request.

## Best Possible Solution

Keep the issue open for maintainer/product design. The likely implementation is to add explicit typed and validated per-agent overrides, probably `agents.list[].contextInjection`, `agents.list[].bootstrapMaxChars`, and `agents.list[].bootstrapTotalMaxChars` or a named-profile indirection, merge them through the agent config resolver, pass the resolved session agent id into bootstrap/context resolution for embedded and CLI runners, and update the Control UI and docs to make global defaults versus per-agent overrides clear.
