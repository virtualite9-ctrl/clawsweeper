## Summary

Keep open. Current main still uses the broad poll-creation detector for `action="send"`, so metadata-only non-zero duration or true boolean poll fields can still trigger the poll-only rejection. This PR remains a focused implementation candidate; the prior same-author PR was closed as superseded by this cleaner branch, and the existing Greptile review found the patch well-scoped.

## Best Possible Solution

Keep this PR open for maintainer review. If maintainers agree that metadata-only poll fields should not block normal sends, land this stricter real-poll-intent guard or an equivalent action-scoped sanitizing fix, while preserving rejection for actual poll question/options on `action="send"` and preserving the existing poll action validation path. Broader message-tool schema overexposure can remain tracked separately if this PR is intentionally limited to the runtime false-positive guard.
