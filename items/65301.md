## Summary

Keep open. Current main still uses the broad poll-creation detector on action="send", so metadata-only non-zero duration or true boolean poll fields can still trigger the poll-only rejection. The PR remains a useful implementation candidate, and the older same-author PR was already closed as superseded in favor of this cleaner branch.

## Best Possible Solution

Keep this PR open for maintainer review. If maintainers agree that metadata-only poll fields should not block normal sends, land this stricter real-poll-intent guard or an equivalent action-scoped/sanitizing fix, preserving real poll rejection on action="send" and the existing poll action validation path. Broader schema-overexposure cleanup can remain tracked by the related open issues/PRs if this PR intentionally fixes only the runtime false-positive guard.
