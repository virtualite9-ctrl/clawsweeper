## Summary

Current main still has the reported Volcengine/BytePlus Kimi catalog mismatch. The shared catalog keeps Kimi K2.5 and Kimi K2 Thinking metadata at `reasoning: false` and `maxTokens: 4096`, both Volcengine and BytePlus still consume those shared rows, and the prior ClawSweeper review correctly identified the issue as unresolved provider-catalog work pending vendor verification.

## Best Possible Solution

Keep this issue open. The best fix is a provider-owned catalog update in `src/plugin-sdk/volc-model-catalog-shared.ts`, verified against current Volcengine/BytePlus vendor limits, with focused regression tests showing corrected Kimi K2.5 and Kimi K2 Thinking metadata propagates through `extensions/volcengine` and `extensions/byteplus`. Update user-visible model/status docs or expectations if the corrected reasoning or max-output metadata changes displayed capabilities.
