## Summary

Current main still has the reported Volcengine/BytePlus Kimi catalog mismatch. The shared Volc catalog keeps Kimi K2.5, Kimi K2 Thinking, and Kimi K2.5 Coding at `reasoning: false` and `maxTokens: 4096`; both bundled providers still consume those shared rows; and Together still exposes Kimi K2.5 as reasoning-capable with a 32K output limit. This remains actionable provider-catalog work pending vendor-limit verification.

## Best Possible Solution

Keep this issue open. The best fix is a provider-owned catalog update in `src/plugin-sdk/volc-model-catalog-shared.ts`, verified against current Volcengine/BytePlus vendor limits, with focused regression tests proving corrected Kimi K2.5 and Kimi K2 Thinking metadata propagates through `extensions/volcengine` and `extensions/byteplus`. If reasoning or max-output metadata changes affect user-visible model listing/status behavior, update the provider docs/status expectations in the same change.

## Risks / Open Questions

The fix should verify provider-specific semantics instead of blindly copying one provider's row.
