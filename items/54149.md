## Summary

Keep open. Current `main` still carries the reported metadata: `src/plugin-sdk/volc-model-catalog-shared.ts` defines `kimi-k2-5-260127` with `reasoning: false` and `maxTokens: 4096`, and every entry in `VOLC_SHARED_CODING_MODEL_CATALOG` still uses `maxTokens: 4096`, including `kimi-k2-thinking` with `reasoning: false`. Both Volcengine and BytePlus consume that shared catalog unchanged, while Together still advertises Kimi K2.5 as `reasoning: true` with `maxTokens: 32768`. The same Volc shared definitions were also present in the latest release commit `00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5` (`v2026.4.22`), so this is not already fixed on shipped code.
