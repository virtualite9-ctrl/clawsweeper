## Summary

Current main still has the reported Volcengine/BytePlus Kimi catalog mismatch. The shared Volc catalog keeps Kimi K2.5 and the Kimi coding rows at `reasoning: false` and `maxTokens: 4096`; both bundled providers still consume those rows, while Together exposes Kimi K2.5 as reasoning-capable with a 32K output limit.

## Best Possible Solution

Keep this issue open. The best fix belongs in the bundled Volcengine/BytePlus provider-owned catalog path, primarily `src/plugin-sdk/volc-model-catalog-shared.ts`, after verifying current vendor limits. Add focused tests that assert Kimi K2.5 and Kimi K2 Thinking `reasoning` and `maxTokens` propagation for both `extensions/volcengine` and `extensions/byteplus`, and update provider docs if the user-visible catalog changes.
