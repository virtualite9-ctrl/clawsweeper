## Summary

Keep open. Current main still carries the reported Volcengine/BytePlus shared catalog mismatch: Kimi K2.5 and the shared coding Kimi entries remain `reasoning: false` with `maxTokens: 4096`, while Together and upstream model docs support at least revisiting those provider-owned limits.

## Best Possible Solution

Keep this issue open and fix it in the bundled Volcengine/BytePlus provider-owned catalog path, primarily `src/plugin-sdk/volc-model-catalog-shared.ts` plus any docs/tests that expose those capabilities. The maintainer-quality fix should verify current Volcengine and BytePlus ModelArk limits, update Kimi K2.5 and Kimi coding/thinking `maxTokens` and `reasoning` values as appropriate, and add focused tests for the affected catalog rows.
