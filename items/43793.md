## Summary

Keep open. Current main does not implement the PR's central TUI reconnect UX: generic gateway disconnects still fall back to an idle activity status, no one-shot reconnect/restart guidance is added to the chat log, `CustomEditor` has no `ctrl+r` hook, and `GatewayChatClient` exposes no public reconnect method for the TUI to call. Main does contain related gateway reconnect infrastructure, but not the user-facing recovery path proposed by this PR.

## Best Possible Solution

Keep the PR open for maintainer review or ask the contributor to rebase and narrow it. The best product path is to land a focused TUI/gateway change that exposes a safe manual reconnect method, wires a visible retry shortcut or command, shows reconnect/restart guidance only when auto-reconnect is actually active, and preserves the existing auth/pairing-specific guidance with regression tests in `src/tui` and `src/gateway`.
