## Summary

Keep open. Current main still lacks the PR's central TUI reconnect UX: generic gateway disconnects resolve to idle, there is no Ctrl+R reconnect hook, and the TUI backend/GatewayChatClient do not expose a manual reconnect method. The PR discussion also includes a macOS remote-probe side effect that remains visible on main, so the work is not obsolete even though it needs rebasing and narrowing.

## Best Possible Solution

Keep the PR open for maintainer review, but ask the contributor to rebase and narrow it. The best path is a focused TUI/gateway change that exposes a safe manual reconnect API, wires a visible retry shortcut or command, shows reconnect/restart guidance only when auto-reconnect is actually active, preserves auth/pairing-specific guidance, and includes regression tests. The macOS remote-probe fix should be split or reviewed as a separate surface.
