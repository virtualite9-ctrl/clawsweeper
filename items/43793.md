## Summary

Keep open. Current main still does not implement the PR's central TUI reconnect UX: generic gateway disconnects remain idle, there is no one-shot reconnect/restart guidance, Ctrl+R is not wired in the editor, and the TUI gateway backend exposes no manual reconnect API. The PR discussion shows the contributor addressed several branch review comments, but those changes are not present on current main.

## Best Possible Solution

Keep the PR open for maintainer review or ask the contributor to rebase and narrow it. The best product path is a focused TUI/gateway change that exposes a safe manual reconnect method, wires a visible retry shortcut or command, shows reconnect/restart guidance only when auto-reconnect is actually active, preserves pairing/auth-specific guidance, and adds regression tests in `src/tui` and `src/gateway`.
