## Summary

Keep open. Current main still lacks the central TUI reconnect UX from this PR: generic gateway disconnects still fall back to idle, there is no Ctrl+R retry hook, and the TUI backend does not expose a manual reconnect API. The linked feature issue was closed as superseded by this PR, so this PR remains the canonical implementation candidate rather than cleanup to close.

## Best Possible Solution

Keep this PR open for maintainer review, but ask the contributor to rebase and narrow it. The best path is a focused TUI/gateway patch that exposes a safe manual reconnect API, wires a visible retry shortcut or command, shows retry/restart guidance only for reconnectable disconnects, preserves auth/pairing-specific guidance through the current reconnect-paused/detail-code seams, resets activity status on reconnect, and includes regression tests. Split or separately justify the macOS remote-probe and diffs-store cleanup work unless a maintainer explicitly wants those surfaces reviewed together.
