## Summary

Keep open. Current main still lacks the PR's central TUI reconnect UX: generic gateway disconnects still become idle, there is no Ctrl+R editor hook, and neither the TUI backend nor GatewayChatClient exposes a manual reconnect method. The PR discussion shows active follow-up fixes on the branch, so the work is not obsolete, but it should be rebased and narrowed before maintainer review.

## Best Possible Solution

Keep this PR open for maintainer review, but ask the contributor to rebase and narrow it. The best path is a focused TUI/gateway change that exposes a safe manual reconnect API, wires a visible retry shortcut or command, shows retry/restart guidance only for reconnectable disconnects, preserves auth/pairing-specific guidance using the current onReconnectPaused/detailCode seam, resets activity status on reconnect, and includes regression tests. The macOS remote-probe and diffs-store cleanup should be split or reviewed as separate surfaces.
