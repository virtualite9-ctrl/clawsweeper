## Summary

Keep open. Current main still does not implement the PR's central TUI reconnect UX: generic gateway disconnects are rendered as idle, there is no one-shot reconnect/restart guidance in the chat log, CustomEditor has no ctrl+r binding, and GatewayChatClient/TuiBackend expose no manual reconnect method for the TUI to call. The PR also contains fixes for adjacent macOS remote-probe behavior that are not present on main, so it is not obsolete even though it needs maintainer review and likely narrowing/rebase.

## Best Possible Solution

Keep the PR open for maintainer review, but ask the contributor to rebase and narrow it. The best product path is a focused TUI/gateway change that exposes a safe manual reconnect API, wires a visible retry shortcut or command, shows reconnect/restart guidance only when auto-reconnect is actually active, preserves auth/pairing-specific guidance, and includes regression tests for the TUI disconnect state and gateway reconnect timer behavior. The macOS remote-probe fix should either be split into its own PR or reviewed as a separate surface.
