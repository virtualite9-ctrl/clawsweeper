## Summary

Keep open. Current main still does not implement the PR's central TUI reconnect UX: generic gateway disconnects still resolve to idle, there is no Ctrl+R editor/backend reconnect path, and the disconnect handler only emits pairing-specific guidance. The branch discussion also covers macOS remote-probe side effects that remain only partially addressed on main, so the PR is not obsolete even if it should be rebased and narrowed.

## Best Possible Solution

Keep the PR open for maintainer review, but ask the contributor to rebase and narrow it. The best path is a focused TUI/gateway change that exposes a safe manual reconnect API, wires a visible retry shortcut or command, shows reconnect/restart guidance only when auto-reconnect is actually active, preserves auth/pairing-specific guidance, and includes regression tests. The macOS remote-probe fix should be split or reviewed as a separate surface.
