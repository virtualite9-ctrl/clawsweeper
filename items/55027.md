## Summary

Current main still does not implement true progressive Feishu block delivery. Feishu keeps the core block-streaming path disabled, drops non-card block payloads, and only consumes accepted card-mode blocks through the Feishu Card Kit streaming-card path, so the reported tradeoff remains.

## Best Possible Solution

Keep this open for a Feishu plugin-owned fix. The best implementation path is to make Feishu preview-card streaming and block-message streaming separate modes, add and honor an explicit `channels.feishu.blockStreaming` plus per-account config in schema/manifest/docs, set `disableBlockStreaming` from that resolved config instead of hardcoding true, deliver `info.kind === "block"` immediately through normal Feishu message/card sends rather than Card Kit playback, and add regression coverage for disabled blocks, enabled progressive block delivery, and existing streaming-card duplicate suppression.
