## Summary

Current main still does not implement the requested true progressive Feishu block delivery. Feishu still forces core block streaming off, the Feishu delivery path drops non-card block payloads, and the remaining block path feeds Feishu Card Kit streaming rather than sending each emitted block as a normal progressive reply. Related closed Feishu issues do not supersede this narrower remaining request.

## Best Possible Solution

Keep this open for a Feishu-plugin-owned fix. The best path is to add an explicit true block delivery mode for Feishu, wire it through the channel/global block-streaming configuration without duplicate final/card sends, and update Feishu tests plus docs/config metadata so streaming-card preview and progressive block replies are clearly separate.
