## Summary

Keep open. Current main still does not provide true progressive Feishu block delivery: block payloads are ignored unless they can be consumed by the streaming-card path, and Feishu reply options still force core block streaming off. The current regression tests assert that behavior, so the related closed Feishu streaming issues do not supersede this remaining request.

## Best Possible Solution

Keep this issue open for a Feishu-owned implementation decision. A real fix should add a non-card progressive block delivery path for Feishu, wire it to the channel/global block-streaming configuration without reintroducing silent drops or duplicate finals, and update Feishu tests plus docs/config metadata so `streaming` card preview and true block replies are clearly separate.
