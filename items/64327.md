## Summary

Keep open. Current main still has internal effective context-token resolution, including the agents.defaults.contextTokens cap, but the public plugin hook contract and current llm_output/model_call_* emitters do not expose the resolved effective limit. The related #64260 remains open adjacent work, not a shipped or superseding implementation.

## Best Possible Solution

Keep this issue open. The best fix is an additive plugin hook contract update that exposes the native model context metadata and the resolved effective context limit, sourced from existing context-window resolution, on llm_output and sanitized model_call_started/model_call_ended telemetry. Update src/plugins/hook-types.ts, embedded/CLI/Codex emitters, docs/plugins/hooks.md, and focused hook tests, and coordinate naming with #64260 so both requests land as one consistent hook telemetry change.
