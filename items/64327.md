## Summary

Keep this issue open. Current main resolves an effective context-token budget internally and uses it for embedded runner behavior, but the public plugin hook contract and current llm_output/model_call emitters still do not expose that effective limit to plugin consumers. The related #64260 is adjacent open work, not a shipped or clearly superseding implementation.

## Best Possible Solution

Keep this issue open. The best fix is an additive plugin hook contract update that exposes both native model context metadata and the resolved effective context-token limit on llm_output and sanitized model_call_started/model_call_ended telemetry, sourced from existing context-window resolution. Update src/plugins/hook-types.ts, embedded/CLI/Codex emitters, docs/plugins/hooks.md, and focused hook tests, coordinating the field names with #64260.
