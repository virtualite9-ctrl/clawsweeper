## Summary

Current main still permits the reported over-reporting path: positive configured or persisted context-token overrides are returned before known model-window resolution, and the affected reply, session persistence, /status, and status-summary paths still feed those overrides into that behavior. The linked fix PR #39859 is open/unmerged in the provided GitHub context, so the issue should remain open.

## Best Possible Solution

Keep #39857 open until PR #39859 or an equivalent maintainer-approved fix lands. The fix should make positive configured and persisted context-token values act as ceilings when the selected model's smaller known window is available, preserve existing override behavior when the model window is unknown, remove or align the duplicate status-summary resolver behavior, and add focused regression coverage for the resolver, reply setup, /status, status summary/session tooling, and session metadata persistence.
