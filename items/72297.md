## Summary

Keep this PR open. Current main still does not fix the linked QMD name-mismatch failure when `qmd collection list --json` only exposes names, and this PR is an active, focused implementation candidate for open #71783. It should not be closed as cleanup, but the implementation still needs a safer path before merge because it sources a collection name from unstructured add-error stderr and then removes that collection.

## Best Possible Solution

Keep this PR or the linked issue active, but require a safer implementation before merge: rebind a different-named QMD collection only after OpenClaw can verify, through structured QMD metadata, that the stale collection has the same managed path and pattern. If QMD cannot expose that metadata, keep refusing automatic removal and surface a clearer warning/manual repair. Add regression coverage for the #71783 `Name: life-main` conflict and for refusing to remove an unverified stderr-only name.
