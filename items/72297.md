## Summary

Keep PR #72297 open. Current main has QMD collection conflict rebinding only when list output exposes path/pattern metadata, and it still explicitly treats name-only list output as incomplete metadata that should not trigger a destructive rebind. The PR addresses a real remaining failure mode from #71783, but the Aisle comment raises a concrete safety concern about deleting a collection based only on stderr text, so this needs maintainer review or a safer implementation rather than cleanup closure.

## Best Possible Solution

Keep the PR or linked issue active, but require a safer fix: preserve the current non-destructive behavior when metadata is unavailable unless the stale collection name from qmd can be verified against the same managed path and pattern through an authoritative qmd interface. Add regression coverage for #71783’s `Name: life-main` path-pattern conflict and for refusing to remove an unverified stderr-only name.
