## Summary

Current main already addresses this behavior: fallback choices are now tracked as transient auto session overrides, cleared before the next turn so the configured primary is retried, and explicit user model selections are protected from fallback overwrite. The shipped release tag `v2026.4.22` contains both the code and changelog entry for this fix.

## Evidence

- **Auto-failover overrides self-heal before the next turn:** `createModelSelectionState()` explicitly clears auto-sourced fallback overrides and resets in-memory provider/model back to the configured primary before allowlist validation, so recovered primaries are retried instead of staying pinned to the fallback.
