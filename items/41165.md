## Summary

Keep open. The provided GitHub context shows an open implementation candidate, #56092, with explicit closing syntax for this issue, and current `main` still has the key routing shape that PR describes: exec/system wake paths can pass an originating session key into heartbeat, and `resolveHeartbeatSession()` checks that forced key before `heartbeat.session`. Current Telegram tests/docs also still allow default private DMs to resolve to `agent:main:main`, so the report is not safely closeable as implemented or obsolete.

## Best Possible Solution

Keep this issue open while maintainers review #56092 or close that PR with an explicit rationale. The narrow code path to settle first is heartbeat session resolution: `heartbeat.session` should either intentionally outrank the originating/forced session for wake-triggered runs, or the current precedence should be documented and this issue narrowed to Telegram DM scope/product behavior. If maintainers want Telegram DMs isolated by default, update `extensions/telegram/src/conversation-route.ts` and the Telegram DM tests/docs together because current `main` still documents and tests default DM-to-main routing.
