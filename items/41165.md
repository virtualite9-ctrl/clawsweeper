## Summary

Keep open. Current main still contains the heartbeat forced-session precedence and Telegram default DM-to-main behavior implicated by the report, and open PR #56092 is an unmerged implementation candidate for the heartbeat portion.

## Best Possible Solution

Keep this issue open while maintainers review and either land PR #56092 or close it with explicit rationale. The narrow code path to settle first is heartbeat session resolution: heartbeat.session should either outrank an originating forced session for wake-triggered runs, or the current precedence should be documented. If maintainers also want Telegram DMs isolated by default, update the Telegram routing code, tests, and channel-routing docs together.
