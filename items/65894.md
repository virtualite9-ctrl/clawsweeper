## Summary

Keep this PR open. Current main still has `openclaw-gateway` configured with `image: ${OPENCLAW_IMAGE:-openclaw:local}` and no Compose `build:` context, while the PR is a focused one-line candidate for the reported source-checkout `docker compose up --build` gap. The existing setup script and Docker docs work around the gap by building `openclaw:local` separately, so they do not make the PR obsolete.

## Best Possible Solution

Keep this PR open for maintainer review, or replace it with an equivalent Docker change that lets source-checkout users run `docker compose up --build` without a prior manual `docker build`, while preserving the setup-script path, remote `OPENCLAW_IMAGE` behavior, and documented Docker build args.

## Risks / Open Questions

- Adding an unconditional `build: .` should be checked against the documented `OPENCLAW_IMAGE=ghcr.io/openclaw/openclaw:latest` path so remote-image users do not get surprising rebuild or retag behavior.
- If maintainers expect `docker compose build` to replace the setup-script build path, the Compose file may need `build.args` for `OPENCLAW_DOCKER_APT_PACKAGES`, `OPENCLAW_EXTENSIONS`, and `OPENCLAW_INSTALL_DOCKER_CLI`, or docs should state the difference.
