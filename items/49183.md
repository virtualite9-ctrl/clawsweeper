## Summary

Current main has global/per-agent/provider tool policies, owner-only gating, and some group/sender channel policies, but it does not implement the RFC's central request: trusted gateway operator scopes are not carried into agent tool-policy resolution, and there is no `toolsByScope`/scope-keyed agent config. The narrower PRs cited in the issue discussion are still in-flight and do not make this issue obsolete on main.

## Best Possible Solution

Keep the issue open as the canonical scoped RFC unless maintainers explicitly supersede it with #43673, #42981, or a merged PR. The best implementation path is an additive, documented config/API change that threads trusted operator scope context into agent tool-policy resolution, merges it deterministically with existing global/provider/agent/group policies, preserves owner-only gating, and references the narrower sender/group work as complementary slices rather than replacements.
