## Summary

Keep #48506 open. Current main still does not expose resolved target context on the `message_sending` hook: the event type and delivery call sites carry raw destination/content plus existing routing metadata, while target resolution and cross-context checks remain internal outbound logic. The related GitHub items are adjacent security-hook proposals, not a canonical replacement for this focused plugin API gap.

## Best Possible Solution

Keep the issue open and treat it as a core plugin API enhancement. The best path is a backward-compatible `message_sending` enrichment, likely an optional nested `targetContext`, populated in the shared outbound delivery paths from existing target resolution and cross-context policy data, with focused contract tests and docs updates for plugin authors.
