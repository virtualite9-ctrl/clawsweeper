## Summary

Current `main` already supports RFC2544 fake-IP compatibility for image/media tool fetches through the documented `tools.web.fetch.ssrfPolicy.allowRfc2544BenchmarkRange` config, and the image tool now forwards that policy to remote media loads. That supersedes this PR's earlier unconditional relaxation with a safer opt-in implementation.

## Evidence

- **Image tool forwards configured SSRF policy:** `createImageTool` resolves remote media SSRF policy from config, then passes it into the non-sandbox `loadWebMedia` call used for remote image URLs.
