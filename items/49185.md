## Summary

Keep open. Current `main` has a partial mitigation for long SDK `Retry-After` sleeps, but it does not implement the requested configurable per-provider or per-candidate retry budget in config.

## Evidence

- **Model provider config has no retry/maxRetries field:** `src/config/zod-schema.core.ts:346` defines `ModelProviderSchema` with `baseUrl`, `apiKey`, `auth`, `api`, `headers`, `authHeader`, `request`, and `models`, but no provider retry budget field. The runtime type matches this in `src/config/types.models.ts:114`.

## Risks / Open Questions

- There is a real partial mitigation on `main`: long `Retry-After` sleeps can be capped globally, so some of the reported delay may already be reduced.
- The issue specifically asks for per-provider or per-candidate retry control in config; current code does not provide that surface, so closing as implemented would be too aggressive.
