## Summary

Current main still validates `entry?.config` as `{}` in non-activating plugin loader paths, so a strict plugin `configSchema.required` can still produce the false-positive invalid-config error reported in #70658. The linked PR #70811 appears to target the right fix, but it is open/unmerged in the provided context and the equivalent guard is not present in this checkout.

## Best Possible Solution

Keep #70658 open until #70811 or an equivalent patch lands on main. The fix should defer validation only when config is undefined in non-activating snapshot/CLI metadata paths, keep validation for real activation and explicit configs, and add regression coverage for strict required schemas with missing config in those non-activating paths.
