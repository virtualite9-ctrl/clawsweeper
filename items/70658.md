## Summary

Keep #70658 open. Current main still validates `entry?.config` even when it is `undefined` on non-activating snapshot and CLI metadata paths, so the reported false-positive invalid-config behavior is not implemented on main yet. The linked PR #70811 appears to be the right active fix path, but it is still open per the provided GitHub context.

## Best Possible Solution

Keep the issue open until #70858's fix path, preferably PR #70811 or an equivalent maintainer-reviewed change, lands on main. The fix should defer validation only for non-activating snapshot/CLI metadata loads when `entry?.config` is undefined, while preserving validation for real activation and for non-activating loads with explicit config. Add regression tests for both manifest-only snapshot and CLI metadata undefined-config cases, then let the merged PR close #70658.
