## Summary

Current `main` already contains the Feishu re-registration fix: gateway schema reads now use manifest metadata instead of re-activating plugin runtime, the startup channel registry is pinned against later registry churn, and there is an explicit regression test naming repeated `config.schema` loads re-running Feishu `registerFull` as the old bug.

## Close Comment

Closing after a Codex review because this is already implemented on current OpenClaw. The gateway config-schema path now reads plugin manifest metadata instead of re-activating plugin runtime (`src/config/runtime-schema.ts:11`), and there is an explicit regression test for the old failure mode: repeated `config.schema` loads re-running `registerFull` for channel plugins including Feishu (`src/config/runtime-schema.test.ts:227`). Gateway startup also pins the initial channel registry so later plugin-registry churn cannot evict or re-register active channels (`src/gateway/server-runtime-state.ts:108`, `src/plugins/runtime.ts:151`). The changelog shows the shipped fixes under `2026.4.10` and `2026.4.2` (`CHANGELOG.md:922`, `CHANGELOG.md:1405`). If this still reproduces on a current build, please open a new issue with fresh logs and the exact version.
