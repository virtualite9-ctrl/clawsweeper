## Summary

Current `main` already routes Feishu tool clients by session account. `extensions/feishu/src/tool-account.ts` now prefers explicit `accountId`, then the contextual `defaultAccountId` passed from `ctx.agentAccountId`, and only then falls back to `channels.feishu.defaultAccount`. The Feishu doc/wiki/drive/bitable tool registrations all pass `ctx.agentAccountId`, and dedicated tests cover those paths. The shipped `v2026.4.22` tag already contains this behavior.
