## Summary

Keep #69126 open. Current main has token/context-window driven auto-compaction and an `agents.defaults.compaction` subtree for compaction tuning, but it does not implement a global `sessionCompactionLimit` or equivalent message-count/session-limit default in config, schema, docs, tests, or `session_status`. The provided related items are analogous `agents.defaults.*` requests, not a canonical duplicate for this compaction-limit gap.

## Best Possible Solution

Keep the issue open and, if accepted, add a typed global compaction-limit config with schema/help/generated-schema/docs coverage, runtime wiring into the session auto-compaction trigger, and regression tests for global default behavior across channel sessions and idle/reset boundaries. The likely home is the existing `agents.defaults.compaction` subtree, but maintainers should confirm the desired key name and whether the limit is message-count based, token based, or both.
