## Summary

Keep #69126 open. Current main has global token-based compaction configuration under `agents.defaults.compaction`, but it does not implement the requested global `agents.defaults.sessionCompactionLimit` key or any message/session-count compaction default. The `session_status` tool on current main only supports `sessionKey` and `model`, so there is also no current per-session `compactionLimit` mutation path to globalize. The related ClawSweeper items are analogous `agents.defaults.*` requests, not a canonical tracker for this specific compaction-limit default.

## Best Possible Solution

Keep the issue open for maintainer/product design. The least surprising implementation would define the requested global default under the existing compaction namespace, likely `agents.defaults.compaction.<clear-limit-name>`, unless maintainers explicitly want the proposed top-level `agents.defaults.sessionCompactionLimit`. The change should update `AgentDefaultsConfig`, zod schema, generated schema/help labels, docs, config UI/search metadata if applicable, and the embedded-run/after-turn compaction trigger path, with tests proving the default applies across new Discord/channel sessions and survives idle rollover. If maintainers decide existing `reserveTokens`/`keepRecentTokens` already cover the user need, the docs should explicitly explain that mapping instead of silently closing this request.
