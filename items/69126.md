## Summary

Keep #69126 open. Current main has global `agents.defaults.compaction` tuning for token-budget based compaction, but it does not implement the requested `agents.defaults.sessionCompactionLimit` or an equivalent global per-session/message-count compaction limit. The issue is recent, contributor-authored, has no protected labels, and the related items are analogous defaults work rather than a canonical duplicate.

## Best Possible Solution

Keep the issue open for maintainer/product judgment. If OpenClaw wants this behavior, add a typed and documented `agents.defaults` config field for a global per-session/message-count compaction trigger, wire it into the session compaction decision path without conflicting with existing token-budget `agents.defaults.compaction.*` settings, and add schema/help/docs plus targeted tests. If maintainers decide existing token-based `agents.defaults.compaction` is the intended answer, the issue should be answered with that clarification rather than auto-closed as already implemented.
