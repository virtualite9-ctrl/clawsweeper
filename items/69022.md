## Summary

Keep PR #69022 open. Current main still lacks the Telegram block-stream humanDelay wiring: the Telegram dispatch path does not import/export `resolveHumanDelayConfig` and does not pass `humanDelay` into `dispatchReplyWithBufferedBlockDispatcher`, while the shared reply dispatcher only delays block replies when that option is supplied. The PR is a focused implementation of the open bug #68945, has a targeted test in its diff, and maintainer discussion already called it a safe small merge after unrelated churn was removed.

## Best Possible Solution

Keep #69022 open as the active focused fix for #68945. Rebase it onto current main if needed, preserve the scoped wiring through `extensions/telegram/src/bot-message-dispatch.agent.runtime.ts` and `extensions/telegram/src/bot-message-dispatch.ts`, keep the regression test in `extensions/telegram/src/bot-message-dispatch.test.ts`, then run the targeted Telegram test and changed gate before landing.
