## Summary

Keep PR #69022 open. Current main still lacks the Telegram humanDelay wiring that the PR proposes, the core dispatcher only applies delays when a channel passes `dispatcherOptions.humanDelay`, and peer channels already pass `resolveHumanDelayConfig`. The related issue #68945 remains open and the maintainer discussion explicitly treated this PR as the preferred small fix, not as obsolete.

## Best Possible Solution

Keep #69022 open and land it after normal maintainer review/CI, or apply the same narrow fix directly: export/import `resolveHumanDelayConfig`, pass `humanDelay: resolveHumanDelayConfig(cfg, route.agentId)` into Telegram `dispatcherOptions`, and keep the focused regression test. Do not close as implemented or duplicate unless an equivalent change ships on main; keep polling-stall work tracked separately.
