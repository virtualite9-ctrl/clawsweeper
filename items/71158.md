## Summary

Keep PR #71158 open. Current main still registers `/bot-clear-storage` without `requireAuth: true`, so it remains outside the auth-gated framework command registry while the handler can delete files under the QQBot downloads directory on `--force`. The PR is a focused fix matching the already-merged `/bot-approve` auth hardening in #70706, and its proposed regression test is still absent on main.

## Best Possible Solution

Keep the PR open and land or recreate its narrow fix: add `requireAuth: true` to the `bot-clear-storage` registration, add the `getFrameworkCommands()` regression assertion, and include an appropriate changelog/security note. The existing #70706 `/bot-approve` fix is useful precedent but does not supersede this remaining command-specific gap.
