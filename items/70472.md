## Summary

Keep open. Current main still contains the compaction/overflow failure path that resets the active session mapping, and the focused fix PR #70479 is still an open implementation candidate rather than merged behavior.

## Best Possible Solution

Keep this issue open. Review and land #70479 or an equivalent fix that preserves the active session mapping when overflow recovery or compaction fails, returns explicit retry guidance instead of rotating agent:main:main, and adds regression coverage for both embedded overflow payloads and thrown compaction failures followed by another WebChat message using the same sessionId.
