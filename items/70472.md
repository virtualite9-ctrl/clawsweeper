## Summary

Keep #70472 open. The report is a recent, concrete WebChat/session-continuity bug with a confirming triage comment, and current main still contains the implicated paths: checkpoint persistence still bails out with the exact `session not found` warning when the active session entry is missing, and compaction/overflow failure handling can still rotate the active session mapping through `resetSessionAfterCompactionFailure`. The linked fix PR #70479 is explicitly marked open/unmerged in the provided context, so the issue should remain open until that work or an equivalent mainline fix lands.

## Best Possible Solution

Keep the issue open and land or rework PR #70479's product direction: preserve the existing active session mapping on overflow/compaction failure, surface explicit retry or `/compact`/`/new` guidance instead of auto-rotating, and add regression coverage around WebChat `agent:main:main` after failed compaction checkpoint persistence.
