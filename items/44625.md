## Summary

Keep open. Current main still routes Slack capability scope discovery through only `auth.scopes` and `apps.permissions.info`, so the reported `unknown_method` chain remains plausible and directly matches the latest issue comment. The prior implementation PR #44692 was closed unmerged, and there is no current regression test for the proposed `auth.test` fallback.

## Best Possible Solution

Keep this issue open. The Slack plugin should own a small fix in `extensions/slack/src/scopes.ts`: add `auth.test` as the modern-token path, either as primary or as a fallback after the legacy workspace-app methods, parse the actual granted scope surface exposed by the current Slack client, preserve legacy behavior, and add focused tests for `unknown_method` fallback plus capabilities diagnostics output. PR #44692 is useful prior art but is closed unmerged, so it should be reviewed or reworked rather than used as a close reason.
