## Summary

Keep open. Current main removed raw Feishu `file_key` JSON from the agent-facing message text, but it still downloads inbound Feishu files into media attachments and the shared reply pipeline still extracts text-like/PDF file contents into the prompt. The reported `ignoreFileContent` account setting is not present in the current Feishu config schema or docs, so the exact opt-out the reporter configured is not implemented on main.

## Best Possible Solution

Keep this issue open and implement a real Feishu or shared inbound-attachment opt-out. The likely fix is an owner-scoped Feishu config field such as `channels.feishu.ignoreFileContent` plus per-account inheritance, wired before `resolveFeishuMediaList` or before shared file extraction, with tests proving file messages produce only a placeholder and no downloaded `MediaPaths` or extracted `<file>` prompt blocks when enabled. Docs should explain the setting and its interaction with `mediaMaxMb` and media understanding.
