## Summary

Current `main` already guards the reported blank-reply path. `runEmbeddedPiAgent` now detects reasoning-only and empty terminal turns, retries when appropriate, and otherwise returns a user-visible error payload instead of silently completing with no visible reply. Regression tests cover both the non-strict provider reasoning-only case and empty-response exhaustion, and the changelog says this shipped in `v2026.4.14`.

## Evidence
