## Summary

Keep this PR open. Current main still collapses unclassified compaction failure diagnostics to `reason=unknown`, so the reported diagnostic gap is not implemented on main. The PR is useful, and its follow-up addressed the review comments about spaces and fallback tests, but the current head is not ready for conservative cleanup because it bundles unrelated README translation work and still puts unbounded failure detail into the classification token.

## Best Possible Solution

Keep this PR open, but request changes. Split the README and Chinese README additions into a separate docs PR, then land a focused compaction diagnostic fix that preserves parser-safe, bounded diagnostics: either keep `reason=unknown` with a separate capped/sanitized detail field, or define a capped unknown-detail token with regression coverage for unclassified errors and empty reasons.
