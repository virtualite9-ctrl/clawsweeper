## Summary

Keep this PR open. Current main still collapses unclassified compaction diagnostics to `reason=unknown`, so the reported bug is not implemented on main. The PR is a useful fix candidate, but it is not ready for conservative cleanup because it bundles unrelated README localization work and still puts unbounded failure detail inside the classification token.

## Best Possible Solution

Keep this PR open and request changes. Split the README localization into a separate docs PR, then land a focused compaction diagnostic fix that preserves parser-safe, bounded details: preferably keep `reason=unknown` as the stable classification and add a separate capped/sanitized detail field, with regression tests for unclassified errors, empty reasons, and overly long details.
