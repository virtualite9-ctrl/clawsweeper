## Summary

Keep this PR open. Current main still collapses unclassified compaction diagnostic reasons to `reason=unknown`, so the core bug is not implemented on main. The PR is a relevant fix candidate, but it should be revised before landing because it puts arbitrary failure text into the stable `reason=` classifier and bundles unrelated README localization work.

## Best Possible Solution

Keep this PR open and request changes: split the README localization into a separate docs PR, preserve `reason=unknown` as the stable classifier, add a separate bounded log-safe detail field for unknown compaction failures, and cover unrecognized errors, empty input, whitespace/control characters, and overlong details with focused tests.
