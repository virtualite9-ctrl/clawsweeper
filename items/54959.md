Keep this PR open. Current main has an OpenShell backend/plugin setup page, but it does not contain the proposed `docs/platforms/openshell.md` deployment guide or Platforms navigation entry. The PR is still a plausible docs contribution for an existing bundled OpenShell surface, and the latest PR context shows the author addressed the main review feedback, including replacing the unsafe global fetch-guard workaround with a warning/limitation.

## Best Possible Solution

Keep the PR open for maintainer docs/security review. The best path is to reconcile the new deployment guide with the existing `docs/gateway/openshell.md` backend/plugin page, fix any duplicate glossary drift, verify the operational guidance against current OpenClaw/OpenShell behavior, then either merge it as a Platforms deployment guide or ask the author to split/fold the material into the canonical OpenShell docs.
