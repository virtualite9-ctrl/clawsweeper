Keep this PR open. Current main still lacks the proposed Platforms OpenShell deployment page and navigation entry, while the existing OpenShell docs cover the bundled sandbox backend rather than running OpenClaw inside an OpenShell sandbox. ## Best Possible Solution

Keep this PR open for maintainer docs/security review. ## Risks / Open Questions

- The PR is docs-only, so it does not directly introduce repository code execution, dependency, workflow, lockfile, or package-publishing changes.
- The guide is operational and security-sensitive: it recommends launcher scripts, NODE_OPTIONS preloads, a global dispatcher/Module._load WebSocket monkey patch, proxy/TLS policy, and supervisor-binary swap procedures. Those instructions need maintainer security review before merge.
- Some PR guidance appears stale against current main: Slack Socket Mode proxy support and scoped trusted-env-proxy provider paths have since landed, so the guide should be updated to avoid recommending obsolete monkey patches or overstating limitations.
- The PR head appears to add duplicate OpenShell/OpenShell Sandbox zh-CN glossary entries relative to its current base, so docs/i18n cleanup may be required before merge.
- Maintainers still need to decide whether this belongs as a separate Platforms deployment page or should be folded into docs/gateway/openshell.md.
