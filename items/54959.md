## Summary

Keep this PR open. Current main still lacks an OpenShell platform deployment page and Platforms navigation entry, while the existing OpenShell docs cover using OpenShell as a sandbox backend from a host Gateway rather than running OpenClaw inside an OpenShell sandbox. The PR is docs-only and the author has addressed several substantive review comments, including removing the unsafe global fetch-guard workaround, but the remaining operator guidance is security-sensitive and needs maintainer docs/security review before merge.

## Best Possible Solution

Keep this PR open for maintainer docs/security review. The best path is to reconcile the new platform guide with docs/gateway/openshell.md and the current Slack/WebSocket proxy plus fetch-guard trusted-env-proxy behavior, remove duplicate glossary entries, then either merge it as a dedicated Platforms deployment guide or fold the durable operator guidance into the canonical OpenShell docs.
