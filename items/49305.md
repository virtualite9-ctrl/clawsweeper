## Summary

Keep open. Current `main` still stops the WhatsApp monitor on logged-out sessions without clearing auth state, so the original failure mode is not fully fixed; it has shifted from a dead listener/manual-restart problem into a linked-but-stopped or restart-loop problem. Status reporting appears improved, but the core logout/lifecycle gap remains.

## Evidence

- **Logged-out still stops the monitor:** `monitorWebChannel()` records `healthState: "logged-out"`, logs a relink message, then calls `controller.shutdown()` and `break`. `resolveCloseDecision()` still classifies logged-out as a terminal `action: "stop"`.
