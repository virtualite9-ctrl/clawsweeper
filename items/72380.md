## Summary

Current main addresses both concrete regressions from #72380. `installBundledRuntimeDeps` now always anchors npm with an OpenClaw-owned `package.json`, resets npm project/global config, verifies requested package sentinels after install, and has a regression test for non-isolated external install roots. The plugin-sdk alias/restaging path is now serialized by a shared runtime-mirror filesystem lock in both the node-host plugin loader and bundled channel runtime path, and alias wrapper refresh is idempotent without deleting the shared alias directory. The related #72058, #72114, and #72200 context all points to the same bundled-runtime-deps subsystem, but this issue's two specific failure modes are implemented on current main.

## Best Possible Solution

Keep the current main implementation and ship it in the next release. If a user still sees ENOTEMPTY or home-directory npm writes after upgrading to a build containing 1b1eea238c85e9260cea88dcc2e8c32dd5973202 or later, track that as a fresh, narrower runtime-deps report with current logs, install root paths, and channel/plugin details.
