## Summary

Keep PR #71830 open. Current main at df542f75 still lacks the WhatsApp XML tool-payload sanitization and WhatsApp error-text visibility controls targeted by the PR, and the linked issue #71715 documents a concrete production regression rather than stale or duplicate work.

## Best Possible Solution

Keep the PR open for maintainer review and land this patch or an equivalent WhatsApp plugin/core-channel-contract fix. The implementation should sanitize XML tool payloads at all WhatsApp-visible outbound paths, normalize echo bookkeeping to match sent text, add documented `channels.whatsapp.exposeErrorText` with account inheritance, and route WhatsApp text-only error payloads through an adapter hook so suppression can work.
