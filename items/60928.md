## Summary

Keep PR #60928 open. Current main still does not force a CLI-agent skills snapshot refresh before reusing a persisted `skillsSnapshot`, so the PR’s core bug is not implemented on main. The PR discussion also contains a concrete reproduction and useful review feedback; the right path is to revise the PR, not close it as obsolete.

## Best Possible Solution

Keep the PR open and ask for a revised implementation that invalidates or rebuilds the CLI/trusted-operator skills snapshot before reuse, without leaving a persistent watcher running in one-shot `openclaw agent` invocations. Add focused coverage for the CLI refresh behavior and, ideally, a negative assertion that ingress/gateway paths do not receive the forced manual bump.
