## Summary

Keep open. Current `main` still contains the exact stale-timestamp guard reported in #44993: `appendCronStyleCurrentTimeLine` returns unchanged whenever the prompt already includes `Current time:`, so heartbeat prompts that carry an older line still will not be refreshed. The discussed fix PRs #45071 and #45188 were closed unmerged, and current tests only cover append/dedup presence, not replacement of a stale existing line.

## Best Possible Solution

Keep #44993 open and land the narrow helper fix: make `appendCronStyleCurrentTimeLine` idempotently replace an existing `^Current time:.*$` line with the fresh line for `nowMs`, append when absent, and keep empty/whitespace behavior unchanged. Add focused unit coverage for stale replacement and repeated calls, plus a heartbeat-path regression if feasible. The closed unmerged PRs #45071/#45188 are useful prior art but do not supersede the issue until the behavior is on `main`.
