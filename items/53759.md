## Summary

Keep open. Current `main` still routes `exec host=node` through `system.run.prepare`, but the macOS companion node neither advertises nor handles `system.run.prepare`, so the reported Mac-node exec failure is still plausible on current code. There is partial progress on remote bin probing via `system.which`, but that does not close the execution-path gap.
