## Summary

Keep this issue open. Current main still omits `MINIMAX_OAUTH_TOKEN` from the MiniMax `web_search` credential path, and #65785 remains an open implementation candidate linked with closing syntax.

## Best Possible Solution

Keep this issue open and review #65785 or an equivalent maintainer patch. The fix should either add `MINIMAX_OAUTH_TOKEN` consistently to the MiniMax web-search runtime resolver, provider metadata, autodetection/setup expectations, tests, and docs, or explicitly document that `web_search` requires a separate Coding Plan credential even for `minimax-portal` OAuth users.
