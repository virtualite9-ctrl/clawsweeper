## Summary

Keep open. Current main still swallows Lobster workflow-file resolution errors and falls back to inline pipeline execution, so linked bug #68101 remains. This PR is a plausible implementation candidate, but it has a concrete functional review blocker: the extension-only heuristic can misclassify valid inline pipelines ending in file-like tokens.

## Best Possible Solution

Keep this PR open as an implementation candidate for #68101, but require a revision before merge: preserve path errors only for inputs that are clearly workflow-file references, keep ordinary inline pipelines on the runtime path, and add paired regression tests for both a missing relative workflow file and a valid inline pipeline containing a workflow-like file argument.
