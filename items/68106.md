## Summary

Keep open. Current main still swallows Lobster workflow-file resolution errors and falls back to inline pipeline execution, so the linked Lobster bug remains real. This PR is a plausible implementation candidate, but the current diff has a concrete regression risk already called out in review: an extension-only heuristic can misclassify valid single-step inline pipelines ending with file-like arguments such as config.json.

## Best Possible Solution

Keep this PR open as an implementation candidate for #68101, but require a revision before merge: preserve file/path errors only for inputs that are clearly workflow-file references, keep ordinary inline pipelines on the runtime path, and add paired regression tests for both a missing relative workflow file and a valid inline pipeline containing a workflow-like file argument.
