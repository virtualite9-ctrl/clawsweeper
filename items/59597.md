## Summary

Keep open. Current `main` still does not implement the requested `cron edit` before/after preview: the command builds a patch, calls `cron.update`, and writes JSON with no diff-preview step. The user-facing cron docs also do not describe any preview behavior. This PR still represents unlanded work, and its branch scope drifted beyond cron into unrelated UI error-formatting changes, so it is not a good conservative-close candidate.
