## Summary

Current `main` already preserves Feishu DM origin routing through session reset/init and uses that origin for reply delivery, with explicit regression coverage for routing a Feishu-originated reply back to Feishu when the active provider surface is webchat.

## Evidence

- **Feishu inbound dispatch tags the originating DM route:** The Feishu channel builds inbound agent context with `OriginatingChannel: "feishu"` and `OriginatingTo: feishuTo`, so DM commands carry the Feishu delivery target into core reply handling.
  - file: [extensions/feishu/src/bot.ts:1012](https://github.com/openclaw/openclaw/blob/5b8bd6371c666a5de43c82305e0315771e8e49d5/extensions/feishu/src/bot.ts#L1012)
  - sha: [5b8bd6371c66](https://github.com/openclaw/openclaw/commit/5b8bd6371c666a5de43c82305e0315771e8e49d5)
- **Session init persists the live origin route into delivery state:** During session init, the current originating channel/to is resolved into `deliveryContext`, `lastChannel`, and `lastTo` even when a new session is being created, which is the state this report said was being lost on `/new`.
  - file: [src/auto-reply/reply/session.ts:529](https://github.com/openclaw/openclaw/blob/5b8bd6371c666a5de43c82305e0315771e8e49d5/src/auto-reply/reply/session.ts#L529)
  - sha: [5b8bd6371c66](https://github.com/openclaw/openclaw/commit/5b8bd6371c666a5de43c82305e0315771e8e49d5)
- **Reply routing prefers the originating route for normal channel turns:** `resolveEffectiveReplyRoute()` returns `OriginatingChannel` / `OriginatingTo` for non-system providers, so a Feishu-originated command reply routes back to Feishu instead of falling through to an internal surface.
  - file: [src/auto-reply/reply/effective-reply-route.ts:24](https://github.com/openclaw/openclaw/blob/5b8bd6371c666a5de43c82305e0315771e8e49d5/src/auto-reply/reply/effective-reply-route.ts#L24)
  - sha: [5b8bd6371c66](https://github.com/openclaw/openclaw/commit/5b8bd6371c666a5de43c82305e0315771e8e49d5)
- **Regression test covers the Feishu-to-webchat reroute case:** Current tests explicitly assert that when the provider is `webchat` but origin metadata points to Feishu, replies are routed to channel `feishu` and the original Feishu DM target.
  - file: [src/auto-reply/reply/dispatch-from-config.test.ts:996](https://github.com/openclaw/openclaw/blob/51dd4f288f67ef4df4c7a121f078f508535df1fd/src/auto-reply/reply/dispatch-from-config.test.ts#L996)
  - sha: [51dd4f288f67](https://github.com/openclaw/openclaw/commit/51dd4f288f67ef4df4c7a121f078f508535df1fd)
