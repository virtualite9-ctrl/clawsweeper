## Summary

Keep #37625 open. Current main still lacks the requested write-time Markdown code-fence lint/fix utility, config mode, per-write skip option, tests, and integration into workspace Markdown write paths. The related implementation PRs #37628 and #41207 were closed unmerged, so they do not supersede the issue.

## Best Possible Solution

Keep the issue open until maintainers either implement a central Markdown code-fence lint/fix utility with sh/ts/python/json inference, change records, warn/auto-fix configuration, per-write skip support, and focused tests across current workspace write paths, or explicitly decide that OpenClaw should not mutate Markdown at write time.
