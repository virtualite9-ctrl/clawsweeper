## Summary

Keep this PR open. Current main still serializes the raw parsed config object for full and path-specific `/config show` replies, while existing gateway and CLI config surfaces already use the redaction pipeline. The linked bug #65623 is still open, same-author, and this PR is the unmerged implementation candidate; the review discussion identifies a test-quality gap, not a reason to close the PR.

## Best Possible Solution

Keep this PR open. Address the review comment by adding `sensitive: true` to the mocked schema hints in `src/auto-reply/reply/commands-gating.test.ts`, then review and land this PR or apply the same small fix directly: in `src/auto-reply/reply/commands-config.ts`, load runtime schema UI hints and call `redactConfigObject(...)` before rendering both full and path-specific `/config show` replies.
