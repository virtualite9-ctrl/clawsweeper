## Summary

Keep open. The requested direct CLI workflow is still intentionally absent on current main and in the latest release snapshot: `openclaw nodes run` is not registered, `nodes invoke` blocks `system.run`, and docs direct users to the agent `exec` tool with `host=node`. Related cleanup reports show the old wrapper was removed/superseded, not that this feature request has been implemented.

## Best Possible Solution

Keep #59337 open for maintainer product/security judgment. If accepted, implement a first-class `openclaw nodes run` or `openclaw nodes exec` CLI that reuses the existing `exec host=node` approval and `systemRunPlan` flow rather than raw `node.invoke system.run`, then update CLI docs and tests. If maintainers intentionally reject direct CLI shell execution, close manually with that rationale.
