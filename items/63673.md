## Summary

Keep #63673 open. It is a recent, concrete regression report for an external Keychat bridge after an OpenClaw update, and current main does not contain the Keychat plugin/bridge implementation needed to prove the receive path is fixed. Current main has a bundled Nostr plugin with its own inbound subscription path and recent plugin-install-record fixes, but neither proves that the reported Keychat sidecar subscription/IPC failure is obsolete or external-only.

## Best Possible Solution

Keep the issue open and triage it as a plugin-boundary regression. Ask for the exact Keychat package/source, manifest, bridge binary version, `openclaw plugins inspect keychat`, plugin registry/install-record output, and gateway/bridge logs on current main or the latest release. Then isolate whether the remaining failure is an OpenClaw plugin install/activation regression, a missing SDK contract for sidecar inbound delivery, or a Keychat bridge bug that should move to the plugin/ClawHub maintainer.
