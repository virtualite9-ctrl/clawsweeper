## Summary

Current `main` no longer eagerly loads the plugin runtime for `openclaw configure`. The command is explicitly on the `loadPlugins: "never"` startup path, the wizard's plugin section reads manifest metadata instead of executing plugin runtime code, and channel setup now snapshot-loads only scoped setup plugins. Those paths are already present in release `v2026.4.23`, so the 2026.4.5-era configure stack overflow report appears fixed on shipped code.

## Evidence
