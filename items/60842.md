## Summary

Keep open. Current `main` still applies `toolsAllow` only as a final filter after tool creation, and the tool-policy assembly on `main` has no equivalent `forceAllowTools`/`mergeForceAllowPolicy` path to re-expose core tools removed by restrictive allowlists like `group:openclaw`. The PR’s claimed regression coverage is also not present on `main`.

## Evidence

- **Embedded run still post-filters after tool creation:** `runEmbeddedAttempt` builds `allTools` with `createOpenClawCodingTools(...)` and then applies `applyEmbeddedAttemptToolsAllow(allTools, params.toolsAllow)`. The helper itself only filters an already-built tool list, so it cannot re-add a core tool that policy filtering already removed.
