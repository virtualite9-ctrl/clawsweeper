## Summary

Keep this PR open. Current main still uses the MCP SDK's strict `client.listTools()` path for bundle MCP discovery, so the PR's central bug is not implemented on main. The PR remains a useful implementation candidate, but it should be reworked before merge because its raw `client.request()` replacement bypasses SDK `listTools()` metadata caching, and its latest non-object-schema pass-through can still publish invalid provider tool schemas after the normal provider-schema normalization point.

## Best Possible Solution

Keep this PR open as a useful implementation candidate, but require rework before merge: fix bundle MCP `tools/list` discovery for missing `inputSchema.type` while preserving the SDK `listTools()` metadata behavior, reject or filter explicit non-object root input schemas before cataloging, and add a harness regression for a configured MCP server that omits `inputSchema.type`. The linked bug #63602 being closed is not enough to close this PR as obsolete because current main still has the strict bundle MCP listing path.
