## Summary

Keep this PR open. Current main still has the strict MCP SDK `client.listTools()` path that can reject typeless `inputSchema` payloads before OpenClaw can normalize them, and the latest PR head is a real implementation candidate. It should not be closed as cleanup, but it still needs maintainer review or rework because replacing `Client.listTools()` with raw `client.request()` bypasses MCP SDK 1.29.0 tool metadata caching used by later `callTool()` output-schema validation and task-required routing.

## Best Possible Solution

Keep this PR open as an implementation candidate. Rework bundle MCP discovery so typeless `inputSchema` tools are accepted while preserving the SDK 1.29.0 metadata behavior that `listTools()` currently provides, keep the catalog-time rejection of explicit non-object root schemas, and add regression coverage for both typeless-schema discovery and metadata-dependent `callTool()` behavior for `outputSchema` or required task tools.
