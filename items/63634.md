## Summary

Keep this PR open. Current main still uses the strict MCP SDK `client.listTools()` path that rejects typeless `inputSchema` payloads before OpenClaw can normalize them, so the linked #63602 failure is still plausible. The PR is a useful implementation candidate, but it still needs rework because its raw `client.request()` listing path bypasses MCP SDK 1.29.0 `listTools()` metadata caching for output-schema validation and task-required routing.

## Best Possible Solution

Keep this PR open as an implementation candidate. Rework bundle MCP discovery so typeless root `inputSchema` objects are accepted and explicit non-object roots are skipped, while preserving or reimplementing MCP SDK 1.29.0 `listTools()` metadata semantics for output-schema validation and task-required tools. Add regression coverage for typeless `tools/list` discovery plus a metadata-dependent `callTool()` case before landing.
