## Summary

Keep #47167 open. Current main still does not provide a plugin-visible, read-only early inbound event before mention gating. The reporter’s follow-up made the scope more concrete by linking the public `@dashhuang/conversation-archive` plugin: standard mode can archive messages that reach official hooks, but full-fidelity mode still needs a patch-free way to observe early-dropped inbound messages. Source inspection confirms several channel ingress paths still skip or drop unmentioned group messages before the shared `message_received` plugin hook path is reached. Telegram has a partial internal-only `ingest` path for mention-skipped messages, but that is not the documented plugin hook/API requested here and is not implemented consistently across the listed channels.

## Best Possible Solution

Keep the issue open and solve it as a narrow plugin/API extension: add a documented, opt-in, read-only early inbound observation hook or equivalent SDK contract that fires after basic authenticity/self/allowlist checks but before mention-gating returns, with a stable payload and channel tests for Telegram, Discord, Slack, LINE, iMessage, and BlueBubbles. The implementation should let plugins like `@dashhuang/conversation-archive` archive passively without patching channel source, while preserving current no-reply/no-session behavior for unmentioned group chatter.

## Risks / Open Questions

- Exposing skipped group messages to plugins has privacy implications; any implementation should be explicit opt-in and align with existing sensitive-message hook controls such as WhatsApp `pluginHooks.messageReceived`.
- The new seam should remain observational: it must not create sessions, trigger replies, or bypass allowlists, command gates, or mention gating.
