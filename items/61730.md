## Summary

Current `main` already contains a targeted repair for the reported `notifyActiveTaskWaiters` crash: `getQueueState()` recreates missing `activeTaskWaiters` on restart-inherited singleton state, and a regression test covers the exact `TypeError: undefined is not iterable` path. The same code is present in the latest shipped release tree (`v2026.4.23`), so this item is already implemented well enough to close.

## Evidence
