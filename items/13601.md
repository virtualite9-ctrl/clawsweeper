## Summary

Keep #13601 open. Current main has partial durable cron run history through per-job JSONL files plus a `cron.runs` RPC/CLI path, but the requested CLI log/history surface, time-window filtering, success-rate/list aggregates, and manual retention/prune workflow are still missing. The existing issue discussion also already kept it open for this same reason.

## Best Possible Solution

Keep #13601 open until maintainers either complete or deliberately narrow the requested history surface. The likely implementation path is to add timestamp filters through `readCronRunLogEntriesPage`, `cron.runs`, protocol schemas, UI/CLI state, and docs; expose a clearer `cron logs` or `cron history` CLI with status/time/follow/export behavior; add success-rate and failure aggregate calculations to `cron list` or a dedicated summary; and decide whether manual prune/vacuum/history-retention-days belongs in core or should be documented out of scope.
