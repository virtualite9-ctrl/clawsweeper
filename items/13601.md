## Summary

Keep #13601 open. Current main has partial persistent cron run history via per-job JSONL files plus Gateway/UI paging and status/query/delivery filters, but the issue’s central CLI, time-window filtering, aggregate statistics, and manual retention/prune workflows are still missing. The existing issue comment also explicitly kept this open for the same reason.

## Best Possible Solution

Keep #13601 open until maintainers either complete the requested surface or deliberately narrow the issue. The likely path is to extend `cron.runs` and `readCronRunLogEntriesPage` with timestamp filters, expose CLI status/time/follow/export options or a `cron logs/history` command, add success-rate/list aggregates, and decide whether manual prune/vacuum plus history-retention-days should be implemented or explicitly documented as out of scope.
