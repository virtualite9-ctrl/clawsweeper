## Summary

Keep open. Current `main` does not contain the requested `<session-recap>` stripping logic: the visible-message sanitizers still only recognize JSON sentinel metadata and `Untrusted context`/`<active_memory_plugin>` blocks, while both Control UI and TUI still depend on those sanitizers for user-message rendering. The issue’s reported leak therefore remains plausible and is not implemented on `main`.

## Evidence

- **Inbound metadata fast-path still ignores <session-recap>:** `stripInboundMetadata()` only fast-path checks `SENTINEL_FAST_RE`, which is built from the six JSON sentinel headers plus `Untrusted context`; there is no `<session-recap>` matcher, so a leading `<session-recap>...</session-recap>` block would fall through unchanged.

  - sha: [b0d2003fe65b](https://github.com/openclaw/openclaw/commit/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503)
