## Summary

Keep open. Current main still indexes Gemini multimodal files, but the builtin memory hybrid ranking has no modality-aware scoring path: multimodal chunks are stored as label text plus embeddings, persisted rows do not retain modality, and mergeHybridResults applies the same weighted BM25+vector formula to every result. The broad BM25 hybrid feature is implemented, but this image/audio ranking gap remains distinct from the related hybrid-search items.

## Best Possible Solution

Keep the issue open and implement modality-aware hybrid scoring in the memory-core path: persist or carry chunk modality from multimodal indexing into search results, score image/audio chunks by vector similarity without a BM25 penalty when BM25 is not meaningful, and add a regression test where natural-language media content ranks under default 0.7/0.3 hybrid weights while ordinary text hybrid ranking remains unchanged.

## Risks / Open Questions

- A quick path-extension-based scoring exception would be brittle; the durable fix should carry modality through the indexed chunk/search-result contract or otherwise derive it from a reliable persisted field.
- Changing the merge formula can regress normal text-on-text hybrid relevance, so the fix needs targeted tests for both multimodal vector-only ranking and unchanged text hybrid behavior.
