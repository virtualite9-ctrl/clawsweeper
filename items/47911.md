## Summary

Current main has partial observability coverage, but not the single fleet-wide command requested here. `openclaw status --all` aggregates overview/channels/agents/diagnosis, while auth/provider health lives in the separate `openclaw models status` command and is scoped to one agent at a time.

## Evidence

- **Status command exists, but its documented scope is channels + recent sessions:** The CLI registers `openclaw status` with the description `Show channel health and recent session recipients`, plus `--all`, `--json`, `--usage`, and `--deep` flags.
  - file: [src/cli/program/register.status-health-sessions.ts:54](https://github.com/openclaw/openclaw/blob/d38ed0831d7372c8505dc944aea69b5b83da1435/src/cli/program/register.status-health-sessions.ts#L54)
