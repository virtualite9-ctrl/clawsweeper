## Summary

Keep open. Current main has overlapping diagnostics (`openclaw status`, `status --all`, `status --json`, `sessions --all-agents`, `models status --agent`, and `channels status`), but it does not implement the requested unified `openclaw fleet status` surface. The remaining gap is specifically cross-agent provider/auth state, per-agent health/stuck status, and recent per-agent error aggregation in one command. Related #50066 and #50072 were closed and do not provide a shipped canonical replacement.

## Best Possible Solution

Keep the issue open for maintainer product judgment. The best implementation would be a thin `openclaw fleet status [--json]` aggregator over existing status, sessions, models auth, channels, service, and log readers, with explicit redaction and graceful platform fallbacks. If maintainers decide this is optional operator tooling rather than core CLI, redirect it explicitly to a ClawHub/plugin implementation using the existing plugin CLI registration surface.

## Risks / Open Questions

- There is meaningful overlap with existing status surfaces, so a future implementation should reuse `status.scan-overview`, sessions, models auth, and channels status instead of creating another independent scanner.
- Provider/auth output must preserve current redaction behavior because the proposed report crosses per-agent auth stores and logs.
- Systemd/journal aggregation is platform-specific; a core implementation should degrade cleanly on non-systemd, remote, node-only, and packaged app setups.
