## Summary

Keep #71495 open. The issue is fresh, non-maintainer-authored, has concrete gateway-log evidence, and the reporter explicitly confirmed the empty `/subagents list` symptom still reproduced on v2026.4.24. Current `main` has a post-release fix for a related completion-delivery fallback path, but I did not find a source/test/history change that specifically fixes the slash-command list path or proves the suspected registry writer/reader split is gone.

## Best Possible Solution

Keep the issue open and add a focused regression path: spawn a native subagent through the same gateway/tool path used by production, immediately dispatch `/subagents list` from the requester session, and assert the active or recent section includes the child. If that fails, instrument `registerSubagentRun`, `getSubagentRunsSnapshotForRead`, and `listControlledSubagentRuns` around controller/requester keys and module instance identity to identify whether the break is registration, persistence/snapshot merging, or command context wiring.
