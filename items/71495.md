## Summary

Keep open. Current main has post-v2026.4.24 subagent changes that may address one plausible cause, especially the 10s child-agent startup timeout, but I did not find source, tests, docs, history, or issue discussion that proves the reported `/subagents list` empty-registry behavior is fixed. The list command still depends on the registry snapshot plus controller-key filtering, and there is still no end-to-end regression test for spawn -> immediate `/subagents list` -> non-empty list plus parent completion delivery.

## Best Possible Solution

Keep this issue open. The maintainer path is to add targeted diagnostics or a regression test around the exact boundary: delayed `sessions_spawn` child-agent startup, `registerSubagentRun` timing and controller/requester keys, `getSubagentRunsSnapshotForRead` visibility, `/subagents list` output for the same requester, and parent completion delivery. If the current timeout increase is confirmed to fix the reporter's scenario, add a focused spawn -> `/subagents list` test and close after that proof; otherwise fix the concrete registry write/read or requester-key mismatch.
