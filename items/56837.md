## Summary

Keep open. Current `main` has partial model-provider proxy support, but it does not satisfy the full request in #56837: the documented/provider-config path is HTTP(S)-only, `ALL_PROXY`/SOCKS env routing is explicitly not used for model-provider HTTP proxy resolution, and transport overrides still fail for unsupported provider APIs such as Ollama.

## Evidence

- **Model-provider docs only advertise HTTP proxy overrides:** `docs/gateway/config-tools.md:410`-`415` documents `models.providers.*.request.proxy` as an HTTP proxy override with `env-proxy` using `HTTP_PROXY`/`HTTPS_PROXY` and `explicit-proxy` using a URL; there is no SOCKS5 path documented for model providers.
  - file: [docs/gateway/config-tools.md:410](https://github.com/openclaw/openclaw/blob/89755d1c79678610720cdb9ce504cfd62c261f64/docs/gateway/config-tools.md#L410)
