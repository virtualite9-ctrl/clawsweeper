## Summary

Keep open. Current main still duplicates the subagent task by passing `task` into `buildSubagentSystemPrompt` and also sending the same value as `[Subagent Task]` in the initial child user message. PR #72026 is a focused, unmerged fix for the real linked bug #72019, and the only concrete review concern in the provided discussion is a small API cleanup around the now-unused `task` parameter.

## Best Possible Solution

Keep #72026 open for maintainer review. The best path is a revised fix that keeps the delegated task in exactly one child-run payload location, preserves the task verbatim, adds regression coverage across the actual `message` plus `extraSystemPrompt` gateway payload, and cleans up or clearly deprecates the now-unused `task` parameter.
