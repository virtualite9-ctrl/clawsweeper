## Summary

Keep open. Current main still classifies `gateway` action `config.schema.lookup` as mutating because `isMutatingToolCall` checks the full dotted action string against a flat read-only verb set. The gateway method is valid and read-only, and downstream payload rendering still surfaces warnings for mutating tool failures, so this PR remains a focused implementation candidate rather than obsolete cleanup.

## Best Possible Solution

Keep this PR open and either review/land its root-cause classification fix or apply the same gateway-only leaf-verb logic on main. Add or preserve regression coverage for `config.schema.lookup`, other read-only dotted gateway actions, mutating gateway writes such as `config.set`, null actions, and invalid cron/canvas actions. After the fix merges, close #42856 as implemented and treat payload-only suppression attempts as redundant unless they add separate severity policy.
