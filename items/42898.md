## Summary

Keep this PR open. Current main still treats dotted read-only gateway actions such as `config.schema.lookup` as mutating, while the action is an exposed gateway tool method with read scope and still flows into user-facing warning policy through `mutatingAction`. The PR remains a focused implementation candidate, and the discussion shows the author addressed review feedback by preserving fail-closed behavior for unsupported cron/canvas actions.

## Best Possible Solution

Keep this PR open for maintainer review or landing. The best code path is a gateway-only leaf-verb classifier fix with regression tests for `config.schema.lookup`, `config.get`, `agents.files.list`, mutating writes like `config.set` or `config.patch`, missing actions, and unsupported cron/canvas dotted actions. After a classifier fix lands, close #42856 as implemented and evaluate #42962 only if a narrower payload severity policy is still needed.
