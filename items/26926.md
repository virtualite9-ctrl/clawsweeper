## Summary

Keep #26926 open. Current main still implements Synology Chat inbound webhooks as text-only: the payload type, parser, delivery object, inbound context, docs, and tests do not carry uploaded image/CSV/PDF attachment metadata. The only Synology Chat media path I found is outbound URL-based file delivery, which does not solve inbound user-uploaded attachments.

## Best Possible Solution

Keep #26926 open and implement inbound attachment handling in the bundled Synology Chat plugin. Confirm Synology Chat's uploaded-file webhook/API shape, add typed parsing for images, CSVs, and PDFs, download/store accepted files through existing media helpers with size/type/security limits, pass media fields into the inbound context, update https://docs.openclaw.ai/channels/synology-chat, and add focused webhook plus inbound-context regression tests.
