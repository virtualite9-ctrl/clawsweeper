## Summary

Keep open. Current main still treats Synology Chat inbound webhooks as text-only: images, CSVs, and PDFs uploaded by a Synology Chat user are not parsed, downloaded, saved, or forwarded into the agent media context. The existing issue discussion already contains a maintainer/Codex review keeping this open for the same remaining gap.

## Best Possible Solution

Keep #26926 open and implement this in the bundled Synology Chat plugin. Confirm Synology Chat's uploaded-file webhook/API shape, add typed inbound attachment parsing for images, CSVs, and PDFs, download and store accepted files through the existing channel media helpers with size/type/security limits, populate MediaPath/MediaPaths/MediaUrl/MediaUrls/MediaType/MediaTypes in the inbound context, update https://docs.openclaw.ai/channels/synology-chat, and add focused webhook and inbound-context regression tests for supported attachment types.
