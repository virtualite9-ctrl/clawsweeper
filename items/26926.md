## Summary

Keep #26926 open. Current main still treats Synology Chat incoming webhooks as text-only, while the requested images/CSV/PDF attachment support would need inbound payload parsing, media download/staging, context fields, docs, and regression tests. The existing 2026-04-26 maintainer-side Codex comment from steipete is consistent with the current checkout at 406ae72fd2789c78053f6b94006ec8f488bc1c00.

## Best Possible Solution

Keep #26926 open and implement inbound attachment handling in the bundled Synology Chat plugin. Confirm Synology Chat's attachment webhook/API shape, extend the typed payload/parser, download or stage accepted images/CSVs/PDFs through existing media-safe helpers, expose `MediaPath(s)`, `MediaUrl(s)`, `MediaType(s)`, and useful filename metadata in the inbound context, update the public Synology Chat docs, and add focused webhook plus inbound-context regression tests.
