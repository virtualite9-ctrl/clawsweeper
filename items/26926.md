## Summary

Keep open. Current main still only accepts Synology Chat inbound webhook text and sends attachments only in the outbound direction by forwarding a URL to Synology Chat. That does not satisfy the issue's request for user-uploaded images, CSVs, and PDFs from Synology Chat to reach the agent.

## Best Possible Solution

Keep the issue open and implement it in the bundled Synology Chat plugin if maintainers want this channel to support private-network file ingress. The likely path is to confirm Synology Chat's outgoing-webhook attachment shape/API, add typed inbound attachment parsing and safe download/storage, populate the existing MediaPath/MediaPaths/MediaTypes agent context fields, document inbound limits/security behavior on docs.openclaw.ai/channels/synology-chat, and add focused webhook-handler/inbound-context tests for images, CSV, and PDF handling.
