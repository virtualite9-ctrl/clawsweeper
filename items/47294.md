## Summary

Keep open. Current main still routes Feishu document write/append/insert content through Feishu's markdown convert API without any special handling for `$$...$$` or `<equation>` LaTeX payloads, and there is no regression coverage for preserving complex formula backslashes.

## Best Possible Solution

Keep the issue open and fix it in the bundled Feishu plugin. The likely implementation belongs in `extensions/feishu/src/docx.ts`: preserve `$$...$$` and `<equation>` contents before markdown conversion or construct native `block_type: 16` equation blocks directly, then add Feishu doc tests for write/append/insert using the reporter's complex LaTeX commands.
