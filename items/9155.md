## Summary

Current `main` still lacks `cron.defaults.delivery` or an equivalent global default for primary cron announce delivery. The current config type/schema only exposes failure-delivery defaults, cron job creation still stores explicit delivery or `{ mode: "announce" }`, and missing announce targets still resolve to `last`. The related same-scope item #41164 was closed and cross-referenced here, so #9155 remains the canonical open issue rather than a duplicate to close.

## Best Possible Solution

Keep #9155 open as the canonical product issue. A complete fix should add a primary cron delivery default to the config type, zod schema, generated schema/help/docs, CLI, gateway/service normalization, Control UI, and regression tests, with precedence of job-level delivery first, then global cron default, then the existing `last` fallback.
