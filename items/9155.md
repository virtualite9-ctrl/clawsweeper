## Summary

Keep #9155 open. Current main still does not implement `cron.defaults.delivery` or an equivalent primary cron delivery default. The code supports explicit per-job delivery and separate failure-notification defaults, but omitted isolated cron delivery still normalizes to announce mode and resolves the channel through `last`.

## Best Possible Solution

Keep the issue open until OpenClaw implements a real primary cron delivery default, likely `cron.defaults.delivery`, or maintainers choose an explicit alternate product direction. The implementation should apply precedence consistently: explicit job delivery first, then the global primary cron delivery default, then the existing `last` fallback, with tests across service normalization, delivery planning, gateway protocol, CLI, and Control UI paths.
