## Summary

Keep #9155 open. Current main still lacks a config-level primary cron delivery default; `cron.failureDestination` covers failure notifications only, and normal omitted announce delivery still falls back to `last`.

## Best Possible Solution

Keep the issue open until OpenClaw either implements a real `cron.defaults.delivery` primary output default or a maintainer explicitly chooses a different product direction. A complete implementation should add the config type/schema/help/docs surface, apply the default during cron add/create/normalization and relevant UI/CLI paths, and test precedence: job-level delivery overrides `cron.defaults.delivery`, then falls back to `last`.
