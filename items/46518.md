## Summary

Keep open. Current main still ships the official WhatsApp plugin on Baileys 7.0.0-rc.9, and I found no source, docs, tests, or dependency patch that resolves the reported QR-scan-then-silence failure. The related Baileys/QR reports in the provided context were closed as stale rather than merged/shipped canonical fixes, and #46518 includes useful side-by-side evidence that Chromium/whatsapp-web.js paired while OpenClaw's Baileys flow did not.

## Best Possible Solution

Keep #46518 open as a real bundled WhatsApp plugin bug. The best maintainer path is to reproduce against current release/main on macOS ARM64 Docker, then either patch/pin/replace the Baileys QR-auth path, add diagnostics for scan-time silence, or make a deliberate product decision around a Chromium/wwebjs-based plugin/workaround. The likely code surfaces are extensions/whatsapp/src/session.ts, extensions/whatsapp/src/login-qr.ts, the Baileys dependency/patch configuration, and focused WhatsApp login regression tests.
