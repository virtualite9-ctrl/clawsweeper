## Summary

Keep PR #33845 open. Current main still stores Slack thread participation only in a process-global in-memory dedupe cache, so the gateway-restart behavior this PR targets is not implemented on main. The PR discussion and related open work show real remaining product work plus review concerns, not an obsolete or safely closeable item.

## Best Possible Solution

Keep PR #33845 open. The best maintainer path is to land or request revisions for a canonical Slack thread participation restart-persistence implementation, preferably by wiring Slack to the existing accepted persistent store/dedupe helper if that contract fits, and adding cold-restart persistence tests for record, lookup, TTL expiry, missing/corrupt state, and clear semantics. Close only after that Slack consumer migration lands or a maintainer explicitly supersedes this PR with a canonical replacement.
