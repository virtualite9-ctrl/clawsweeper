## Summary

Keep PR #33845 open. Current main still uses only an in-process Slack thread participation dedupe cache, while Slack mention gating still depends on that cache to allow non-mention replies in bot-participated threads after `requireMention` gating. The PR is not merged, not implemented on main, and not clearly superseded by a canonical landed replacement.

## Best Possible Solution

Keep #33845 open. The best maintainer path is to request revisions or land a canonical Slack thread-participation persistence implementation, preferably using the existing `openclaw/plugin-sdk/persistent-dedupe` helper or the accepted persistent keyed-store helper if maintainers choose #70626 as the contract. The Slack consumer still needs restart-safe record/lookup behavior plus tests for disk round-trip, TTL expiry, missing/corrupt state, clear semantics, and split module instances.
