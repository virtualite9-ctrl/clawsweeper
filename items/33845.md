## Summary

Keep PR #33845 open. Current main still stores Slack thread participation only in a process-global in-memory dedupe cache, while Slack mention gating still depends on that cache to allow non-mention replies in bot-participated threads. The PR is not implemented on main, not merged, and not clearly superseded by a canonical landed change.

## Best Possible Solution

Keep #33845 open. The best maintainer path is to request revisions or land a canonical Slack thread-participation persistence implementation, preferably aligned with the existing `openclaw/plugin-sdk/persistent-dedupe` helper or an accepted persistent keyed-store helper if maintainers choose that contract. The Slack consumer needs restart-safe record/lookup behavior plus tests for disk round-trip, TTL expiry, missing/corrupt state, clear semantics, and split module instances; close only after that migration lands or maintainers explicitly supersede this PR with a canonical replacement.
