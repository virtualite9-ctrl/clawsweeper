## Summary

Keep #33724 open. Current main still hard-codes the built-in config backup ring to adjacent `.bak` files with a fixed count of five, and there is no supported config/schema/UI surface for backup directory, retention count, or trigger policy. The related implementation attempts #33896 and #34478 were closed unmerged, while the cron/`openclaw backup` alternatives cover separate backup workflows rather than configuring the built-in config-write rotation.

## Best Possible Solution

Keep #33724 open until maintainers either implement configurable built-in config backup behavior or explicitly reject that product direction. A complete implementation should add typed config/schema/help/docs/UI support, thread backup policy through `src/config/backup-rotation.ts`, `src/config/io.ts`, `src/config/mutate.ts`, and `src/config/io.observe-recovery.ts`, and update recovery/audit tests while preserving current defaults.
