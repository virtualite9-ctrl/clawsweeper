## Summary

Keep #33724 open. Current main still hard-codes config backups to `${configPath}.bak` plus `.bak.1` through `.bak.4`, runs backup maintenance from core config-write paths, and has no typed config/schema/UI surface for backup directory, retention count, or trigger policy. The related implementation PRs #33896 and #34478 were closed unmerged, and the suggested cron or `openclaw backup` workflows are separate backups rather than configuration for the built-in `.bak` rotation.

## Best Possible Solution

Keep #33724 open until maintainers either implement configurable built-in config backup behavior or explicitly reject the product direction. A complete fix should add typed config/schema/help/docs/UI support, then thread the policy through `src/config/backup-rotation.ts`, `src/config/io.ts`, `src/config/mutate.ts`, `src/config/io.observe-recovery.ts`, and audit/recovery tests while preserving default compatibility.
