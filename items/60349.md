## Summary

Keep #60349 open. Current main does not provide the requested `OPENCLAW_SQLITE_MMAP` or `storage.sqliteMmap` control, and the affected SQLite open paths still use WAL without an mmap-disable helper. The PR is a focused bug fix with a concrete NFS/k3s SIGBUS production report, but its latest discussion still flags correctness and completion concerns that should be resolved rather than auto-closed.

## Best Possible Solution

Keep the PR open and ask for a maintainer-quality revision: preserve the concrete NFS-backed storage mitigation goal, verify the SQLite WAL/shared-memory behavior against upstream SQLite semantics, add/adjust focused tests for env and config resolution, update generated config/schema artifacts and SDK export metadata if a public subpath remains necessary, and document the supported NFS deployment contract.
