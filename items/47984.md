## Summary

Keep open. Current `main` at `d38ed0831d7372c8505dc944aea69b5b83da1435` does not implement `appliedTags` for Discord `channelEdit`: the edit type omits the field, the channel-edit parser does not read it, the PATCH body never sends `applied_tags`, and the current guild-action tests do not cover that path.

## Evidence

- **`DiscordChannelEdit` still lacks `appliedTags`:** `extensions/discord/src/send.types.ts` defines `DiscordChannelEdit` with `availableTags` but no `appliedTags`, so the edit payload type on `main` cannot carry the requested field.
  - file: [extensions/discord/src/send.types.ts:161](https://github.com/openclaw/openclaw/blob/d38ed0831d7372c8505dc944aea69b5b83da1435/extensions/discord/src/send.types.ts#L161)
  - command: `nl -ba extensions/discord/src/send.types.ts | sed -n '70,210p'`
  - sha: [d38ed0831d73](https://github.com/openclaw/openclaw/commit/d38ed0831d7372c8505dc944aea69b5b83da1435)
- **Channel-edit param reader does not parse it:** `readDiscordChannelEditParams` in `extensions/discord/src/actions/runtime.shared.ts` reads standard edit fields plus `availableTags` only. `extensions/discord/src/actions/handle-action.guild-admin.ts` routes `channel-edit` through that helper, so `appliedTags` is dropped before runtime execution.
  - file: [extensions/discord/src/actions/runtime.shared.ts:42](https://github.com/openclaw/openclaw/blob/d38ed0831d7372c8505dc944aea69b5b83da1435/extensions/discord/src/actions/runtime.shared.ts#L42)
  - command: `nl -ba extensions/discord/src/actions/runtime.shared.ts | sed -n '1,120p'`
  - sha: [d38ed0831d73](https://github.com/openclaw/openclaw/commit/d38ed0831d7372c8505dc944aea69b5b83da1435)
- **Discord PATCH body never sends `applied_tags` on edit:** `editChannelDiscord` maps `available_tags` in `extensions/discord/src/send.channels.ts`, then immediately issues `PATCH /channels/{channelId}`. There is no `body.applied_tags = ...` assignment on current `main`.
  - file: [extensions/discord/src/send.channels.ts:73](https://github.com/openclaw/openclaw/blob/d38ed0831d7372c8505dc944aea69b5b83da1435/extensions/discord/src/send.channels.ts#L73)

- **Current tests cover thread-create `appliedTags`, not channel-edit:** `extensions/discord/src/actions/runtime.test.ts` includes `channelEdit` assertions for archived/locked/autoArchiveDuration only, with no `appliedTags` forwarding case, while thread-create tests elsewhere already assert `appliedTags` support. That matches the missing implementation on the edit path.
