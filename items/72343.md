## Summary

Keep this PR open. The checked-out main still routes bare `openclaw` directly to Crestodian, while the PR remains an active, focused contributor implementation candidate for changing fresh or empty installs to classic onboarding first.

## Best Possible Solution

Keep this PR open for maintainer review. The best path, if maintainers accept the UX change, is the PR's scoped CLI route: check the config snapshot before the existing bare-root Crestodian branch, send missing or valid-empty authored configs to classic onboarding, preserve Crestodian for configured installs and invalid configs that need repair, and keep the paired unit, Docker smoke, and docs updates.
