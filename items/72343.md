## Summary

Keep open. The checked-out main still treats bare `openclaw` as a Crestodian entrypoint, and the PR remains an active, focused CLI/docs/test change for fresh-install routing that is not implemented or superseded on main.

## Best Possible Solution

Keep this PR open for maintainer review. If accepted, the likely right path is the PR's scoped route in `src/cli/run-main.ts`: before the existing bare-root Crestodian branch, read the config snapshot, send missing or valid-empty authored config to classic onboarding, and preserve Crestodian for configured installs plus invalid configs that need repair. Keep the paired unit, Docker smoke, and docs updates so the first-run contract is explicit.
