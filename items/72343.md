## Summary

Keep open. Current main still routes bare `openclaw` directly to Crestodian and still documents/tests that as the empty-state first-run behavior. PR #72343 remains a focused, active CLI/docs/test change for behavior that is not implemented on main.

## Best Possible Solution

Keep PR #72343 open for maintainer review. If accepted, the right implementation path is the PR's scoped route in `src/cli/run-main.ts`: before the existing bare-root Crestodian branch, read the config snapshot, send missing or valid empty authored config to `setupWizardCommand({})`, and preserve Crestodian for configured installs plus invalid configs that need repair. Keep the paired unit, Docker smoke, and docs updates so the first-run contract is explicit.
