## Summary

Keep open. Current main still routes bare `openclaw` directly to Crestodian and still documents/tests the empty-state first-run path as Crestodian-based. The PR remains a focused, active change for behavior that is not implemented on main, and the provided PR discussion shows the author addressed the Greptile e2e coverage concern in head commit `6ba1243b4cf3afae50abc4f76edff16e57166d49`.

## Best Possible Solution

Keep the PR open for maintainer review. If accepted, the right path is the PR's scoped `src/cli/run-main.ts` route before the bare-root Crestodian branch: use `readConfigFileSnapshot()` to send missing or valid empty authored config to `setupWizardCommand({})`, while preserving Crestodian for configured installs and invalid configs that need repair. Keep the paired CLI/docs/e2e updates so the first-run contract is explicit.
