## Summary

Keep PR #30310 open. Current main still does not implement the PR's central behavior: shared account listing only returns configured account IDs when an accounts map exists, so base/default credentials are not started beside independent named accounts. The PR discussion is also still substantive, with contributor updates and unresolved product/security review around implicit default startup rather than an obsolete cleanup item.

## Best Possible Solution

Keep #30310 open as the focused work item. Maintainers should either land a token-aware account-listing fix with regression coverage, or explicitly decide that named accounts suppress base/default startup and document that contract. Before landing, resolve the implicit-default startup policy and the channel-specific edge cases raised in review.
