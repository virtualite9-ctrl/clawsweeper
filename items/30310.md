## Summary

Keep PR #30310 open. Current main still does not implement the PR's central behavior: shared channel account listing returns named accounts when an accounts map exists and only falls back to default when the list is empty, so base/default credentials are not started alongside independent named accounts. The PR discussion is active and contains substantive product/security review, so this is not conservative cleanup.

## Best Possible Solution

Keep #30310 open as the focused work item. Maintainers should either land a token-aware account-listing fix with regression coverage, or explicitly decide that mixed top-level auth plus named accounts is a doctor-repair-only shape and document that contract. Before landing, resolve the implicit-default startup policy and the channel-specific credential edge cases raised in review.
