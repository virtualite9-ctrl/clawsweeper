## Summary

Keep PR #30310 open. Current main still lacks the PR's central runtime behavior: shared channel account listing returns only configured named accounts once an accounts map exists, so it does not start the implicit default/base credential runtime alongside independent named accounts. The PR discussion is also active and contains unresolved product/security review, so this is not conservative cleanup.

## Best Possible Solution

Keep #30310 open as the focused work item. Maintainers should either land a token-aware account-listing fix with regression coverage, or explicitly decide that mixed top-level auth plus named accounts should be repaired into accounts.default through doctor/setup and document that runtime contract. Before landing, resolve the implicit-default startup policy and the channel-specific credential edge cases raised in review.
