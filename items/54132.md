## Summary

Keep open. Current `main` does not implement the Sessions-page `History` shortcut or a `/history` viewer, so this PR is not `implemented_on_main`. The proposed shape also overlaps with an existing authenticated session-history surface and needs maintainer review for auth/data-scope and transcript-artifact semantics before cleanup closure would be justified.

## Risks / Open Questions

- Closing this PR would discard an unmerged feature request that is still absent from `main`.
- The proposed route adds another history-reading surface alongside the authenticated `/sessions/:key/history` API, so auth and data-exposure expectations need explicit maintainer judgment.
- Transcript-history handling in OpenClaw includes archived `.jsonl.reset.*` and `.jsonl.deleted.*` artifacts; a simplified file scan can under-report real history.
