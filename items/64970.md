## Summary

Keep this issue open. Current main still hardcodes `SESSIONS_HISTORY_TEXT_MAX_CHARS` to 4000 in the `sessions_history` tool, with no `readEnvInt` or environment lookup. The comparable bash output limit path already uses `readEnvInt`, the sessions-history tests still assert behavior around the 4000-character cap, and the related fix PR #65029 was closed unmerged according to the provided GitHub context.

## Best Possible Solution

Keep the issue open and implement the narrow fix in `src/agents/tools/sessions-history-tool.ts`: read `SESSIONS_HISTORY_TEXT_MAX_CHARS` via the existing `readEnvInt` helper, clamp it to sensible bounds with the current 4000 default, and add a focused regression test in `src/agents/openclaw-tools.sessions.test.ts` proving an env override above 4000 changes sessions-history text truncation. If this env var is intended as a supported user-facing knob, document it alongside the sessions-history bounded-recall docs.
