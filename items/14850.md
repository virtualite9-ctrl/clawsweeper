## Summary

Keep #14850 open. Current main supports binding-level `session.dmScope` for direct-message routing, which is useful related work, but it still has no binding-level `sessionKey`, `sessionAlias`, or main-session target for Discord guild/channel route bindings. The reported Discord channel to Telegram DM continuity gap remains: Discord guild traffic is routed as a non-direct `channel` peer, and non-direct peers still get isolated channel-scoped session keys. Related work in #39049 and #50875 does not supersede this: #39049 was closed unmerged and scoped groups/channels off by default, while #50875 is open/unmerged and explicitly scoped identity resolution away from group/channel peers.

## Best Possible Solution

Keep the issue open until maintainers either add a documented binding-level session target for non-direct route bindings or explicitly reject channel-to-main session unification. The likely path is to extend `AgentRouteBinding` and `RouteBindingSchema`, update config help/docs, thread the new field through `resolveAgentRoute` and session-key construction, and add tests proving Discord channel bindings can intentionally share an agent main or aliased session while default channel isolation remains unchanged.
