## Summary

Keep #14850 open. Current main has a narrower `bindings[].session.dmScope` override for direct-message route bindings, but it still does not provide the requested binding-level `sessionKey`, `sessionAlias`, or `sessionScope` target, and non-direct Discord guild/channel peers still route to isolated channel session keys. The existing maintainer/Codex review comment also explicitly kept the item open for the same remaining gap.

## Best Possible Solution

Keep the issue open until maintainers either add and document an explicit binding-level session target for non-direct route bindings, or make a product decision to reject channel-to-main session unification. The likely code path is `AgentRouteBinding`, `RouteBindingSchema`, `resolveAgentRoute`, and `buildAgentPeerSessionKey`, with tests proving Discord guild/channel bindings can intentionally share an agent main or aliased session while preserving isolated defaults.
