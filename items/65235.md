## Summary

Keep this issue open. Current main hides raw validation detail in normal payloads, but it still intentionally emits generic mutating-tool failure warnings such as `Write failed` for the same required-parameter failures described here. The existing suppression option is per-run and used by heartbeat paths, not normal Feishu/DM replies.

## Best Possible Solution

Keep this issue open and implement a typed validation-error path in the agent/tool runtime. Required-parameter failures from `write`, `edit`, `exec`, and similar tools should still be logged and returned to the model as tool errors, but final chat payloads should suppress validation-only mutating-tool warnings by default unless verbose diagnostics are requested. Add regression coverage in `src/agents/pi-embedded-runner/run/payloads.errors.test.ts` and at least one normal chat-channel reply path such as Feishu/DM.
