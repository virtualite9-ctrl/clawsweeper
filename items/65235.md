## Summary

Current main partially hides raw validation details, but it still emits generic user-facing mutating-tool warnings such as `⚠️ Write failed` for the write-without-content validation path described here. The issue should stay open until validation-only tool failures are typed and suppressed from normal chat payloads by default.

## Best Possible Solution

Keep this issue open and implement a typed validation-error path from required-parameter validation through tool error summaries into reply payload policy. Validation-only failures from tools such as `write`, `edit`, `exec`, and message tools should still be logged and returned to the model as tool errors, but normal chat payloads should suppress the synthetic mutating-tool warning by default unless verbose diagnostics are requested. Add regression coverage in `src/agents/pi-embedded-runner/run/payloads.errors.test.ts` and at least one normal channel reply path such as Feishu/DM.

- A fix should distinguish validation-only failures from real failed side effects; suppressing all mutating-tool warnings would hide legitimate failed writes, messages, or other actions.
- Related retry-loop work should share the same validation-error classification if it lands, so model retryability and chat-surface suppression do not diverge.
