## Summary

Keep open. Current main still routes any truthy `choice.delta.content` through the OpenAI-completions text path without a runtime string guard or typed-block unpacker, so the Mistral typed-block array bug described by #68309 is not implemented on main. The provided PR context shows a focused parser/test/changelog change, and the bot review concerns were addressed in the second PR commit rather than left unresolved.

## Best Possible Solution

Keep this PR open as an implementation candidate. A maintainer should review or rebase it against current main, preserve the ordered typed-block unpacking and fall-through behavior described in the PR discussion, run the scoped agent stream tests, and only close it after this fix or an equivalent merged implementation lands.
