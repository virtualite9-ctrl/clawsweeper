## Summary

Keep #71787 open. Current main has the narrower Feishu Schema 2.0 `context.open_chat_id` fallback, but it still does not normalize nested `operator.user_id.{open_id,user_id,union_id}` identities, so the PR’s central remaining fix is not implemented on main.

## Best Possible Solution

Keep #71787 open and adapt it on top of current main: add nested Feishu Schema 2.0 identity extraction for `operator.user_id.{open_id,user_id,union_id}`, preserve existing `context.open_chat_id` support and malformed-payload guards, keep lifecycle coverage for the nested identity callback, and update the changelog without duplicating the existing #71670 entry.
