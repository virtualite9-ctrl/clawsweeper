## Summary

Keep open. Current main still strips a trailing `NO_REPLY` from auto-reply text and delivers the remaining preamble, so PR #69394's central bug is not implemented. #66755 and #66701 are related open work, but #66755 is an alternate narrower approach rather than a shipped superseder. The PR remains useful, though its current patch needs review follow-up for the trailing-token detection heuristic.

## Best Possible Solution

Keep #69394 open for author or maintainer follow-up. The best fix is a narrow shared auto-reply normalization change that suppresses text-only payloads when an actual trailing silent token is removed, preserves media/channelData behavior, avoids non-trailing `NO_REPLY` false positives with trailing whitespace, and updates the old #30916 expectations plus explicit reasoning-preamble regression tests. Maintainers should then choose a canonical path among #66701/#66755/#69394 rather than closing this PR before a shared fix lands.
