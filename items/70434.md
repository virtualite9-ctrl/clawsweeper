## Summary

Keep PR #70434 open. Current main does not implement the central requested subagent lifecycle hardening: spawn still makes the child `agent` gateway call without readiness retry/preflight, announce finalize still performs one best-effort `sessions.patch`/`sessions.delete`, and registry sweep still performs one `sessions.delete` before waiting for a later sweep. The PR is a focused, recent bug-fix attempt with useful discussion, but it should be reworked before merge because the provided review context flags remaining correctness concerns around the readiness probe and validation ordering.

## Best Possible Solution

Keep the PR open and ask for a rework against current main: add a non-mutating or clearly safe readiness strategy after deterministic local validation, centralize or consistently reuse the transient gateway lifecycle classifier, wrap only the intended subagent lifecycle gateway calls with bounded retries, and add focused tests for spawn readiness, announce finalize `sessions.patch`/`sessions.delete`, and registry sweep delete retry behavior without weakening existing validation/error semantics.
