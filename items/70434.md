## Summary

Keep open. Current `main` still lacks the bounded retry/readiness handling this PR proposes for subagent spawn, announce-finalize, and sweep cleanup paths, so it is not `implemented_on_main` or obsolete. The PR discussion also includes unresolved correctness concerns about the proposed readiness probe and retry wiring, so this is not a conservative auto-close candidate.
