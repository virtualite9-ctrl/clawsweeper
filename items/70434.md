## Summary

Keep PR #70434 open. Current main does not contain the proposed bounded retry/readiness handling for the subagent lifecycle gateway calls, and the PR still represents plausible bug-fix work, but the branch needs maintainer review/rework because the provided review discussion flags readiness-probe and validation-order risks.

## Best Possible Solution

Keep the PR open and have it reworked into a narrow lifecycle-resilience patch: apply bounded retries around the actual subagent lifecycle gateway calls, avoid mutating readiness probes, preserve local validation before gateway probes, and add targeted tests for spawn, announce finalize patch/delete, and registry sweep delete across the transient gateway readiness strings in the PR body.
