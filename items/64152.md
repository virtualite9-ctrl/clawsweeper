## Summary

Current `main` already contains a targeted fix for duplicate `delivery-mirror` transcript entries and suppresses transcript-only assistant mirror messages from visible delivery, so the bug described in #64152 is implemented on `main`.

## Evidence

- **Transcript append deduplicates delivery mirrors:** `appendExactAssistantMessageToSessionTranscript` now returns the existing message id when the same `idempotencyKey` is present, and also skips appending a new `delivery-mirror` entry when the latest assistant message text already matches.

  - sha: [51dd4f288f67](https://github.com/openclaw/openclaw/commit/51dd4f288f67ef4df4c7a121f078f508535df1fd)
- **Regression tests cover duplicate mirror cases:** Tests explicitly verify both duplicate-prevention paths: same mirror `idempotencyKey`, and split-turn style reuse when the latest assistant message already matches the mirror text.
  - file: [src/config/sessions/transcript.test.ts:124](https://github.com/openclaw/openclaw/blob/51dd4f288f67ef4df4c7a121f078f508535df1fd/src/config/sessions/transcript.test.ts#L124)
