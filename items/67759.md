## Summary

Keep this issue open. Current main still resolves image capability only from the gateway catalog plus hardcoded provider shims, the Qwen Coding Plan catalog still omits qwen3.6-plus, WebChat still feeds that resolver result into attachment parsing, and the live implementation candidate PR #67766 remains open with closing intent for this issue.

## Best Possible Solution

Keep this issue open while maintainers review PR #67766 or intentionally consolidate it into #64213. The likely fix belongs in src/gateway/session-utils.ts: preserve catalog-hit precedence and Qwen Coding Plan catalog suppression, then fall back to normalized explicit models.providers[provider].models[].input metadata only when the gateway catalog misses, with regression coverage for explicit image config, explicit text-only config, missing config, and existing shim behavior.
