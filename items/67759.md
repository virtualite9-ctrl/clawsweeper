## Summary

Keep this issue open. Current main still fails closed when the gateway model catalog misses, WebChat attachment parsing still depends on that capability result, and the supplied GitHub context shows open, unmerged PR #67766 as the implementation candidate for this exact bug.

## Best Possible Solution

Keep this issue open while maintainers review PR #67766. The best code path is a focused gateway fix in src/gateway/session-utils.ts that preserves catalog-hit precedence and Qwen Coding Plan catalog suppression, but falls back on explicit models.providers[provider].models[].input metadata only when the gateway catalog misses; add regression coverage for explicit image config, explicit text-only config, missing config, and existing catalog/shim behavior.
