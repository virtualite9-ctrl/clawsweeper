## Summary

Keep this issue open. Current main still fails closed when the gateway model catalog misses, the WebChat attachment path depends on that result, and provided GitHub context shows open, unmerged PR #67766 with closing syntax for the same bug.

## Best Possible Solution

Keep this issue open while maintainers review PR #67766. If accepted, land a focused gateway change in src/gateway/session-utils.ts that falls back from catalog miss to configured provider model input metadata, with regression coverage for catalog hit precedence, explicit image config, explicit text-only config, and missing config. If rejected, keep this issue or #64213 as the canonical bug until current main handles explicit image-capable custom models correctly.
