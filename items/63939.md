## Summary

Keep open. Current `main` still does not expose `web.messageTimeoutMs` as public config, and the WhatsApp monitor still uses a hard-coded 30 minute idle threshold outside the internal tuning hook. The PR remains aligned with a live, corroborated deployment problem in the discussion, so there is not strong evidence for conservative closure.
