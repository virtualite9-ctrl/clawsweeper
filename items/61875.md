## Summary

The reported jiti TDZ crash path is already broken on current `main`, and the same fix is present in the shipped `v2026.4.23` release: `CHANNEL_MESSAGE_ACTION_NAMES` now lives in a leaf module, the CLI/message entrypoints import that leaf directly, and `message-secret-scope` no longer participates in the old cycle.

## Evidence

- **Message tool imports the leaf action-name module directly:** In the confirmed shipped release commit, `src/agents/tools/message-tool.ts` imports `CHANNEL_MESSAGE_ACTION_NAMES` from `../../channels/plugins/message-action-names.js` instead of pulling it from `types.ts`, matching the issue's suggested cycle break.
  - file: [src/agents/tools/message-tool.ts:9](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/agents/tools/message-tool.ts#L9)
  - command: `git show a9797214338ba31c52c796adbb75afb16e0684a9:src/agents/tools/message-tool.ts | nl -ba | sed -n '1,20p'`
  - sha: [a9797214338b](https://github.com/openclaw/openclaw/commit/a9797214338ba31c52c796adbb75afb16e0684a9)
- **CLI message command uses the same leaf import:** The shipped `openclaw message` entrypoint also imports `CHANNEL_MESSAGE_ACTION_NAMES` from `src/channels/plugins/message-action-names.ts`, so the CLI path named in the issue no longer relies on the old `types.ts` value import path.
  - file: [src/commands/message.ts:2](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/commands/message.ts#L2)

- **Action names were extracted into a standalone leaf module:** `CHANNEL_MESSAGE_ACTION_NAMES` now lives in `src/channels/plugins/message-action-names.ts`, while `src/channels/plugins/types.ts` only re-exports it and uses a type-only alias. That is the structural change needed to avoid the TDZ-style circular import described in the issue.

- **Reported `message-secret-scope` leg of the cycle is gone:** In the same shipped release, `src/cli/message-secret-scope.ts` is a standalone helper that only imports routing/string/channel normalization helpers; it no longer imports channel-plugin types or message-tool surfaces.
