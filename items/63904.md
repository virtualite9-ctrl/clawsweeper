## Summary

Keep open. Current main still has Slack thread support, but not Slack `reply_broadcast` mirroring: outbound send paths expose and forward `threadTs` only, and inbound `thread_broadcast` is accepted without surfacing broadcast visibility to agent context. The linked PR #64365 is an open implementation candidate, not merged shipped behavior.

## Best Possible Solution

Keep this issue open and review PR #64365 or a successor patch. The best implementation belongs in the Slack plugin: add a typed `replyBroadcast` option through the Slack send/action/generic message-action surfaces, pass Slack `reply_broadcast` only for threaded replies, expose inbound `thread_broadcast` visibility in agent-visible Slack context, and add regression tests plus Slack docs.
