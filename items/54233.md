## Summary

Keep open. Current `main` still consumes Discord interactive components one component ID at a time, so a `reusable:false` multi-button panel is not single-use at the panel level as the docs promise. The issue is only partially obsolete: the separate `message` vs `content` edit-path mismatch appears fixed on current `main`.

## Evidence

- **Docs still promise single-use components by default:** `docs/channels/discord.md` says components are single use unless `components.reusable=true`, which matches the issue's expected panel semantics.
  - file: [docs/channels/discord.md:304](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/docs/channels/discord.md#L304)

- **Buttons are registered as separate entries with separate IDs:** `createButtonComponent` generates a fresh `btn_` component ID per button and returns a distinct registry entry for each button, with no shared panel/group token.
  - file: [extensions/discord/src/components.ts:516](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/extensions/discord/src/components.ts#L516)
