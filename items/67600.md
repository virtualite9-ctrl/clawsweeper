## Summary

Keep this issue open. Current main still has the reported default-timeout bypass for `yieldMs`/`background` exec calls with no explicit timeout, and the supplied GitHub context shows an open draft PR (#72625) using closing syntax for this issue. Per maintainer cleanup rules, the issue should stay open until that PR is reviewed and merged or closed.

## Best Possible Solution

Keep the issue open while maintainers review #72625 or another focused fix. The implementation should remove the background/yield timeout bypass so `tools.exec.timeoutSec` applies consistently to foreground, yielded, and background exec sessions unless a per-call timeout explicitly overrides it, with regression coverage for local/gateway/sandbox paths as applicable.
