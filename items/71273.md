## Summary

Keep open. Current main still contains the implicated Kimi Coding stream wrapper, the provider still uses it for `kimi/kimi-code` on `anthropic-messages`, and I did not find a merged fix or regression test for the reported repeated tool-call loop. The reporter's proposed root cause is somewhat overbroad because the rewrite only runs after tagged tool-call markup is parsed, but that makes this an investigation/fix item rather than a safe close.

## Best Possible Solution

Keep this issue open. Reproduce on current main with a captured Kimi Coding Anthropic Messages transcript around the tool-result turn. If Kimi emits repeated tagged tool-call sections after results, fix `extensions/kimi-coding/stream.ts` in the provider-owned wrapper and add a focused regression test in `extensions/kimi-coding/stream.test.ts`. If the transcript shows a text-only final answer with `toolUse` or stale tool replay, fix the embedded runner's stop-reason/tool-call normalization instead.
