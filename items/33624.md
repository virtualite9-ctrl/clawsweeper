## Summary

Keep #33624 open. Current main documents and implements concrete Gateway HTTP endpoints, and separately has WebSocket protocol JSON Schema generation, but it still does not expose or generate an OpenAPI/Swagger document for client generation.

## Best Possible Solution

Keep the issue open for maintainer/product follow-through. Define the supported public Gateway HTTP API boundary, then add either a committed/generated OpenAPI document or a served /openapi.json endpoint, plus docs and a check/test that keeps it synchronized. Treat the existing WebSocket protocol JSON Schema as a separate contract unless maintainers explicitly decide to model part of that surface in OpenAPI.
