## Summary

Keep #33624 open. Current main exposes concrete Gateway HTTP endpoints and separately generates a Draft-07 WebSocket protocol schema, but it still does not provide an OpenAPI/Swagger document, `/openapi.json` or Swagger UI route, or a generation/check script for HTTP client generation.

## Best Possible Solution

Keep the issue open for maintainer/product follow-through. Define the supported public Gateway HTTP API boundary, then add either a committed/generated OpenAPI document or a served `/openapi.json` endpoint, plus a check/test that keeps it synchronized. Treat the existing WebSocket protocol JSON Schema as a separate contract unless maintainers explicitly decide to model part of that surface in OpenAPI.

## Evidence
