## Summary

Keep #33624 open. Current main still implements and documents Gateway HTTP endpoints and separately generates WebSocket protocol JSON Schema, but it does not expose or generate an OpenAPI/Swagger document for HTTP client generation.

## Best Possible Solution

Keep the issue open. Define the supported public Gateway HTTP API boundary, then add either a committed/generated OpenAPI document or a served /openapi.json endpoint, plus docs and a check/test that keeps it synchronized. Treat the existing WebSocket protocol JSON Schema as a separate contract unless maintainers explicitly decide to model part of that surface in OpenAPI.
