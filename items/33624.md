## Summary

Keep #33624 open. Current main still exposes documented Gateway HTTP endpoints and a separate WebSocket protocol JSON Schema pipeline, but it does not provide an OpenAPI/Swagger document, generator, dependency, or /openapi.json-style discovery route for HTTP client generation.

## Best Possible Solution

Keep the issue open. Maintainers should first define the supported public Gateway HTTP API boundary, then add either a committed/generated OpenAPI document or a served /openapi.json endpoint, plus docs and a synchronization check/test. Treat the existing Gateway WebSocket protocol JSON Schema as a separate contract unless maintainers explicitly decide to model part of that surface in OpenAPI.
