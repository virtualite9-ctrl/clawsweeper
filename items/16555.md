## Summary

Keep #16555 open. Current main still has a file-backed outbound delivery queue with no configurable TTL/expiry policy for stale queued messages. Startup recovery and reconnect drains load pending JSON entries and gate replay by max retries, backoff, and permanent-error handling, not by entry age. The related SQLite/outbox PRs that proposed TTL/expiry were closed unmerged, and the provided maintainer comment already found the same remaining gap against recent main.

## Best Possible Solution

Keep the issue open until outbound delivery recovery has a bounded, documented expiry policy. The likely core fix is to add optional messages.delivery schema/types/help/docs, enforce enqueuedAt age checks in both recoverPendingDeliveries and drainPendingDeliveries before replay, choose a conservative terminal action such as moving expired entries to failed/ with clear logging, and add regression tests for default backward compatibility, stale startup entries, reconnect drains, retry/backoff interaction, and delivered-marker cleanup.
