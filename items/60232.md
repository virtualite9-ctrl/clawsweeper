## Summary

Current `main` already supports the requested outcome through existing routing and session controls: reasoning/thinking defaults can be set per agent, and routing bindings can target specific channels, specific peers, or wildcard direct/group peers so DMs and groups can use different agents with different defaults.

## Evidence

## Risks / Open Questions

- The exact proposed config shape under `channels.<channel>.groups.*.reasoningDefault` does not exist; the supported path is routing traffic to agents with different defaults, or using session overrides.
- Wildcard peer bindings are proven by current code/tests, but this specific `peer.id="*"` behavior is not called out as clearly in the public docs as the broader routing model is.
