## Summary

Keep #69303 open. Current main still has the reported asymmetry: successful inbound dispatch commits the dedupe key, but any caught dispatch error releases only the in-flight key, allowing the same external message_id to be claimed again. The issue thread has an active, directly linked fix PR (#71819) that is open/unmerged in the provided context, so the bug is not yet implemented on main.

## Best Possible Solution

Keep #69303 open until #71819, or a maintainer-approved equivalent, lands on main. The best fix path is to preserve release-style retry only for failures before any visible progress, and to commit/poison the inbound dedupe key once a reply or other externally visible side effect has occurred, with regression coverage for post-emit replay.
