## Summary

Keep #69303 open. Current main still has the reported asymmetry: successful dispatch commits the inbound dedupe key into the cache, but every caught dispatch error releases only the in-flight claim, making the same MessageSid claimable again. The provided timeline shows PR #71819 is the active proposed fix for this exact issue, but it is open/unmerged in the supplied context, so the bug is not implemented on main yet.

## Best Possible Solution

Review and land PR #71819 or an equivalent main-branch fix that preserves release-style retries for errors before any visible progress, but commits/poisons the inbound dedupe key after externally visible dispatch or side effects. Keep #69303 open so the fix can close it through the normal `Fixes #69303` path after tests prove both pre-dispatch retry and post-dispatch replay suppression.
