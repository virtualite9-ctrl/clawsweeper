## Summary

Keep #71717 open. It is a fresh Windows-native report where every exec command fails with `spawn EPERM`, and current `main` does not show an implemented fix or a canonical duplicate for that broad failure mode. Related Windows exec reports cover narrower PowerShell parsing, gateway restart, and encoding issues, not this all-command spawn failure.

## Best Possible Solution

Keep the issue open and treat it as a Windows host-spawn reliability bug until proven environmental. The likely maintainer path is to reproduce or instrument the Windows exec shell spawn boundary, then add a focused fix in `src/agents/shell-utils.ts`, `src/agents/bash-tools.exec-runtime.ts`, or `src/process/supervisor/adapters/child.ts` with regression coverage for EPERM on PowerShell/pwsh spawn and a clearer diagnostic or fallback when the configured shell cannot be executed.
