## Summary

Keep open. Current main has related transcript-hygiene work, but I did not find a current guard that strips the exact leaked bracketed transcript blocks (`[TOOL_CALL]...[/TOOL_CALL]`) from Telegram-visible final, block, or preview delivery paths. Related #65247 was WebChat-specific synthetic repair-message filtering, and the existing shared sanitizer covers XML/downgraded `[Tool Call: ...]` formats rather than this uppercase bracket format.

## Best Possible Solution

Add a defense-in-depth user-visible sanitizer for internal transcript blocks that covers `[TOOL_CALL]`, `[TOOL_RESULT]`, and malformed/dangling variants. Wire it through the shared assistant-visible text pipeline and the Telegram partial/block/final preview paths, then add focused regressions for `src/shared/text/assistant-visible-text.test.ts`, `src/agents/pi-embedded-subscribe*.test.ts`, and `extensions/telegram/src/bot-message-dispatch.test.ts` using the exact issue payload.
