## Summary

PR #40640 proposes a bundled optional `computer-use` plugin scaffold, but current OpenClaw direction keeps this kind of optional capability out of core and points it to external plugin distribution instead. Current docs already show that optional tool plugins should be published to ClawHub or npm, and `main` does not contain an `extensions/computer-use` plugin.

## Close Comment

Thanks for the contribution. Codex review on current `main` says this is better kept as an external plugin rather than a bundled OpenClaw repo addition, so I’m closing this as `clawhub`.

The current scope guard is explicit: `VISION.md:54` says optional capability should usually ship as plugins and that the bar for adding optional plugins to core is intentionally high, and `docs/plugins/building-plugins.md:16` says plugin authors should publish to ClawHub or npm instead of adding the plugin to this repository. The existing SDK already supports the exact optional-tool shape here via `api.registerTool(..., { optional: true })` in `docs/plugins/building-plugins.md:202`, so this can ship without landing a bundled `extensions/computer-use` scaffold in core.

If you still want to pursue this, the right next step is an external ClawHub/npm plugin repo with the executor contract and demo there, then come back only if a concrete core SDK seam is missing.
