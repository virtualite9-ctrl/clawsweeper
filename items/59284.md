## Summary

Keep #59284 open. Current main still has no Slack `requireOwnerPresent` config or membership guard, and `allowBots: true` is sufficient for bot-authored Slack room messages to proceed into the normal routing path. The issue is recent, concrete, author is not protected, and there is no protected label, but the requested security hardening remains unimplemented.

## Best Possible Solution

Keep the issue open and implement this in the bundled Slack plugin: add a documented `requireOwnerPresent` or equivalent guard to Slack config/types/schema/UI metadata, check cached `conversations.members` membership for configured owner allowlist users before routing room bot messages when enabled, and add focused tests for allowed/blocked bot messages plus schema/docs coverage.
