## Summary

Keep this PR open. Current main at 0835f9409aac0f6048c7d03ec5a2148b7e5001c5 still does not expose or forward `channels.slack.logLevel`, and this PR remains an open, recent implementation candidate for #71531. The latest patch fixes the earlier lowercase enum and exported `SlackConfig` type gap, but the diff still appears to pass `logLevel` only to Bolt's `App` while OpenClaw supplies its own `SocketModeReceiver`; dependency source shows Bolt only applies `App.logLevel` to an internally constructed receiver, so the socket-mode pong-timeout warning path still needs maintainer review or wiring adjustment before merge.

## Best Possible Solution

Keep this PR open as the active implementation path for #71531. Adjust or confirm the runtime wiring so lowercase `channels.slack.logLevel` reaches the component that owns socket-mode logging, likely the explicitly constructed `SocketModeReceiver` or its logger path, while preserving current defaults when unset. Keep the schema and exported type aligned, add tests proving the configured level reaches the receiver/socket warning path and that omitted config preserves defaults, and update Slack/config docs or generated metadata before merge if this becomes a supported operator knob. Keep #72808 separate because it tracks stale/silent Slack disconnect behavior, not log-noise suppression.
