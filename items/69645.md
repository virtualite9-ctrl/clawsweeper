## Summary

Keep open. Current `main` does not already implement the requested early-exit for package installs, and the PR still has unresolved contract/override concerns that match the review discussion.

## Risks / Open Questions

- The PR's `--json` no-op response appears backward-incompatible with the current `UpdateRunResult` contract.
- The proposed early-exit needs to account for `OPENCLAW_UPDATE_PACKAGE_SPEC` and similar forced-install flows, not just `--tag`.
- `main` currently has a docs/code mismatch for this behavior, so closing on cleanup would discard a still-live implementation question.
