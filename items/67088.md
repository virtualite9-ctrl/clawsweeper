## Summary

Current main still contains the reported macOS false `No GUI detected` behavior: `resolveBrowserOpenCommand()` returns `ssh-no-display` for any non-Windows platform with `SSH_*` env vars and no display before the Darwin `open` path can run. The issue is also paired with open implementation PR #67115 using closing syntax, so it should remain open until that PR or an equivalent fix lands or is closed.

## Best Possible Solution

Keep this issue open and review PR #67115, or replace it with an equivalent narrow fix in `src/infra/browser-open.ts` that excludes `darwin` from the SSH/no-display early return or narrows that guard to Linux. Add regression tests for macOS resolving to `open` with lingering `SSH_*` vars and no display, plus Linux preserving `ssh-no-display`.
