## Summary

Keep open. Current `main` still treats webchat/agent attachments as image-only: the Control UI only accepts `image/*`, the webchat send path serializes uploads as `type: "image"`, and the gateway attachment parser explicitly drops detected non-image payloads. Tests on `main` assert that non-image uploads are dropped, so the reported behavior is not implemented or obsolete.

## Evidence

- **Control UI upload surface is image-only:** `CHAT_ATTACHMENT_ACCEPT` is hard-coded to `image/*`, and supported chat attachment MIME types are restricted to `image/*`.
  - file: [ui/src/ui/chat/attachment-support.ts:1](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/ui/src/ui/chat/attachment-support.ts#L1)
  - sha: [926068b14fc6](https://github.com/openclaw/openclaw/commit/926068b14fc661e915a0247e9ecb5949fb95b4f7)
- **Webchat file picker/drop path skips non-images:** The webchat composer ignores selected or dropped files unless `isSupportedChatAttachmentMimeType(file.type)` passes, then renders each attachment as an `<img>` preview.
  - file: [ui/src/ui/views/chat.ts:242](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/ui/src/ui/views/chat.ts#L242)
  - sha: [926068b14fc6](https://github.com/openclaw/openclaw/commit/926068b14fc661e915a0247e9ecb5949fb95b4f7)
- **Webchat RPC payload serializes attachments as images only:** `buildApiAttachments` converts each attachment data URL into an RPC attachment with `type: "image"`; anything that is not an image data URL is discarded before `chat.send`.
  - file: [ui/src/ui/controllers/chat.ts:213](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/ui/src/ui/controllers/chat.ts#L213)
  - sha: [926068b14fc6](https://github.com/openclaw/openclaw/commit/926068b14fc661e915a0247e9ecb5949fb95b4f7)
- **Gateway parser explicitly drops non-image attachments:** `parseMessageWithAttachments` only extracts images; when MIME sniffing finds a non-image it logs `detected non-image ... dropping`, and when sniffing fails for a non-image it logs `unable to detect image mime type, dropping`.
  - file: [src/gateway/chat-attachments.ts:260](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/src/gateway/chat-attachments.ts#L260)
  - sha: [926068b14fc6](https://github.com/openclaw/openclaw/commit/926068b14fc661e915a0247e9ecb5949fb95b4f7)
- **Current tests codify the reported limitation:** `chat-attachments.test.ts` has explicit cases asserting that non-image payloads are dropped and logged, which matches the issue report rather than a fixed implementation.
  - file: [src/gateway/chat-attachments.test.ts:84](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/src/gateway/chat-attachments.test.ts#L84)
