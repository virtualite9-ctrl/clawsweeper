## Summary

Keep open. Current main does not implement the PR's core fixes: managed daily dreaming stripping still stops at any Markdown heading, request-scoped narrative fallback still writes the first raw snippet/promotion, diary dedupe still includes the timestamp in the fingerprint, and append still unconditionally inserts another diary entry. The related thread context is useful but not a close basis: #65138 merged earlier repair/exact-dedupe work, #70403 was closed unmerged, and #70523 is an open narrower fallback-inline-dump PR rather than a shipped superset of this PR.

## Best Possible Solution

Keep the PR open or ask for a refreshed/split patch. The best path is to land the still-missing behavior with focused tests: heading-level-aware managed daily block stripping, cleaned diary-style request-scoped fallback prose, body-only duplicate prevention that preserves legitimate italic body lines, and duplicate-append skipping. Coordinate the inline daily fallback suppression with #70523 instead of closing this PR as already shipped.
