## Summary

Keep open. Current main does not implement the PR's central dreaming diary hardening: nested managed daily blocks still stop stripping at any subheading, request-scoped fallback still writes the first raw snippet/promotion, and diary dedupe still includes the timestamp. The related PRs are adjacent rather than canonical supersession, and this PR also has an unresolved correctness concern in its proposed fingerprint change.

## Best Possible Solution

Keep the PR open for maintainer/author follow-up. The best path is a corrected, rebased memory-core fix that preserves italic diary body lines while deduping by narrative body, adds the nested-heading/legacy managed-block strip regression, and coordinates the request-scoped fallback behavior with the narrower open #70523 inline-dump suppression work.
