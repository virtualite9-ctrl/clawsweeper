## Summary

Current `main` includes opt-in active transcript byte-triggered compaction and successor transcript support, but it still does not fully solve the reported crash-restart/OOM loop. Default configs leave the byte guard disabled, and several restart/repair/recovery paths can still materialize a pathological JSONL transcript before quarantine, bounded recovery, or rotation. There is also still no count-based circuit breaker for accumulated `openclaw:bootstrap-context:full` markers.

## Best Possible Solution

Keep this issue open and implement a core startup-safe transcript recovery guard. The guard should stat/classify active JSONL files before repair, restart recovery, `readSessionMessages`, or `SessionManager.open`; rotate, quarantine, or compact pathological transcripts into a bounded successor without first materializing them; add a count-based circuit breaker for repeated `openclaw:bootstrap-context:full` entries; reuse media offload/dedup where applicable; and add focused regression tests for oversized restart loops and marker accumulation.

## Risks / Open Questions

- Closing now would overstate the current byte-trigger compaction support as a full crash-loop fix even though default configurations remain exposed.
- Already pathological active transcripts can still be read into memory by repair, recovery, and compaction/open paths before bounded recovery can occur.
- Repeated full-bootstrap marker accumulation remains only indirectly mitigated by tail-marker checks; no total-count circuit breaker was found.
