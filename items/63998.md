## Summary

Current main still has only partial large-transcript mitigations. The embedded run and compaction paths repair and open the active transcript before any context-overflow recovery can help, bootstrap completion markers still have no count/circuit breaker in the default path, and the documented transcript rotation/byte-threshold mitigation is opt-in and not enough to recover an already-oversized crash-looping session by default.

## Best Possible Solution

Keep this issue open and implement a core restart/run recovery guard before any full transcript repair or SessionManager.open call: stat and classify oversized or repeatedly failing transcript files, rotate/quarantine or compact into a bounded successor/degraded session, cap repeated bootstrap completion markers, reuse existing media offload and transcript rotation where applicable, and add regression coverage for a large transcript restart loop.
