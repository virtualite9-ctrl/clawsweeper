## Summary

Keep open. Current `main` has partial mitigations for long transcripts, but they are opt-in or post-failure recovery paths, not a shipped default fix for the reported restart doomloop. The code still opens the session transcript directly on run startup, `contextInjection` still defaults to `"always"`, `truncateAfterCompaction` still defaults to `false`, and there is no clear transcript-size load cap or automatic bad-session quarantine in the inspected code/docs.
