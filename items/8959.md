## Summary

Keep #8959 open. Current main still has only the shared `presentation` contract for buttons and single-select menus, plus a separate poll action; it does not provide the requested cross-platform message-tool multi-select/selectMenu API with min/max bounds and one structured selected-values callback. The related dialog PR #18286 was closed unmerged, and the existing maintainer/Codex review comment already made the same keep-open call.

## Best Possible Solution

Keep the issue open until OpenClaw adds an additive portable batch-selection contract in the shared message presentation/message tool surface, exports it through `openclaw/plugin-sdk/interactive-runtime`, documents it at docs.openclaw.ai/plugins/message-presentation, and implements focused render/callback handling for Slack, Discord, Telegram, and fallback text paths. A plugin-only or Discord-only answer is not enough for this issue because the requested behavior is a shared cross-platform message-tool API.
