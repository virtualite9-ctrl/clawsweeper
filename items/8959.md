## Summary

Keep #8959 open. Current main has a portable message presentation contract with buttons and single-select blocks, but it still lacks the requested cross-platform multi-select or batch-submit message-tool API with min/max selection bounds and structured selected-values callbacks. Related PR #18286 was closed unmerged, and the existing issue discussion already records a maintainer/Codex keep-open conclusion.

## Best Possible Solution

Keep the issue open until OpenClaw adds an additive cross-platform message presentation contract for batch multi-select, likely a new multiselect block or a backward-compatible select extension with minValues, maxValues, and structured selected-value array callbacks. The work belongs in core/shared presentation first, then in message-tool schema, plugin SDK exports, fallback rendering, Slack/Discord/Telegram renderers and dispatch behavior, public docs at https://docs.openclaw.ai/plugins/message-presentation, and focused contract tests. A ClawHub/plugin-only answer would not expose the requested portable message-tool API.
