## Summary

Keep #8959 open. Current main still has a portable single-select message presentation contract plus adjacent provider-native or CLI/wizard multiselect paths, but not the requested cross-platform message-tool multi-select with min/max values and one structured selected-values callback across channels.

## Best Possible Solution

Keep the issue open until OpenClaw adds an additive portable batch-selection contract in the shared message presentation/message tool surface, exports it through openclaw/plugin-sdk/interactive-runtime, documents it on docs.openclaw.ai/plugins/message-presentation, and implements focused render/callback handling for Slack, Discord, Telegram, and fallback text paths. Existing provider-native Discord components and Slack parsing should be treated as implementation references, not completion.
