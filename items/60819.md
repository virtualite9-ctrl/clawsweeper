## Summary

Keep open. Current `main` still contains the reported false-positive path: `doctor` collects a gateway memory probe but still emits `No active memory plugin is registered...` when the local CLI cannot resolve a memory runtime, and `status` still renders `enabled (plugin ...) · unavailable` whenever the memory manager is missing.

## Evidence

- **Doctor still short-circuits before using gateway health to clear the warning:** `runGatewayHealthChecks` stores a gateway memory probe and passes it into `noteMemorySearchHealth`, but `noteMemorySearchHealth` immediately reports `No active memory plugin is registered for the current config.` and returns when `resolveActiveMemoryBackendConfig(...)` is null.

  - sha: [6bc0dc8fb671](https://github.com/openclaw/openclaw/commit/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440)
- **Doctor test codifies the exact warning path:** The current test suite explicitly expects `noteMemorySearchHealth` to emit `No active memory plugin is registered` when no memory runtime is active.
  - file: [src/commands/doctor-memory-search.test.ts:251](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/src/commands/doctor-memory-search.test.ts#L251)
