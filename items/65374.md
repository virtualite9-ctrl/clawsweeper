## Summary

Current main still leaves the narrowed shared-workspace bug open. Dreaming now has adjacent fixes, including source-marked corpus lines and an isolated managed cron target, but the sweep remains workspace-scoped: agents that resolve to the same workspace are grouped together, all of their session files are ingested into one workspace corpus/short-term store, and the narrative path consumes snippets without an active-agent filter.

## Best Possible Solution

Keep this open as a narrowed memory-core isolation bug. The best fix is to decide whether dreaming is supposed to be agent-scoped or workspace-scoped when agents share a workspace. If agent-scoped, thread the active agent/session target into dreaming sweeps, filter or partition session corpus and __dreaming_sessions__ recall signals by agent, preserve agent provenance through narrative snippets, and add regression coverage with two active agents sharing one workspace proving one agent cannot ingest the other's transcript. If workspace-level pooling is intended, document that behavior at https://docs.openclaw.ai/concepts/dreaming and add a clear opt-out or isolation config before closing.
