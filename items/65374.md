## Summary

Keep this issue open. Current main has adjacent dreaming fixes, but the narrowed shared-workspace isolation bug remains: dreaming is still workspace-scoped and ingests all configured agents that resolve to the same workspace into one corpus and recall signal.

## Best Possible Solution

Keep this open as a narrowed memory-core bug. The best fix is to decide whether dreaming should be agent-scoped or workspace-scoped when agents share a workspace; if agent-scoped, thread the active agent/session target into dreaming sweeps, store or filter session corpus and __dreaming_sessions__ recall signals by agent scope, and add regression coverage with two active agents sharing one workspace proving Agent B cannot ingest Agent A's transcript. If workspace-level pooling is intended, document it clearly and add an opt-out or isolation config before closing.
