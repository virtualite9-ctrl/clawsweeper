## Summary

Keep #46965 open. Current main has a partial global `messages.suppressToolErrors` setting, but it is not equivalent to the requested global/per-agent `suppressToolErrorWarnings`: mutating tool failures such as `edit`, `write`, and `exec` still surface warnings outside heartbeat runs, and the `suppressToolErrorWarnings` config surface remains heartbeat-scoped.

## Best Possible Solution

Keep #46965 open for maintainer product judgment. A good fix would add a documented normal-run tool-error delivery policy, either as global/per-agent `suppressToolErrorWarnings` or as category-based filtering, while preserving a safe default for mutating failures and updating config schema/help/docs plus payload tests.
