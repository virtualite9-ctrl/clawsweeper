## Summary

Current main still does not implement #9409. The runner, precheck path, shared formatter/sanitizer, and auto-reply fallbacks continue to surface generic context-overflow copy; token/model/session diagnostics are logged or kept in metadata in some paths but are not shown to the user. The direct implementation PR #15749 is closed unmerged, and related context-hardening PRs do not supersede this user-facing diagnostic request.

## Best Possible Solution

Keep #9409 open. The best maintainer path is a shared context-overflow diagnostic formatter that accepts a safe session/channel label, provider/model identity, effective context budget or model limit, observed or estimated token count, computed overage, and concise recovery actions, then wires it through `run.ts`, precheck/attempt handling, `formatAssistantErrorText`, `sanitizeUserFacingText`, and auto-reply fallbacks with focused regression tests.
