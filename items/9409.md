## Summary

Keep #9409 open. Current main still does not include the requested user-visible context overflow diagnostics: session/channel label, observed context size, model limit, and overage. The implementation logs some diagnostic values and carries token data in metadata, but the messages delivered to users remain generic across the embedded runner, shared formatter, sanitizer, and auto-reply fallback paths. The related PR #15749 was closed unmerged, and the later smaller-context/model-switch work does not supersede this error-message request.

## Best Possible Solution

Keep the issue open. The best implementation is a shared context-overflow diagnostic formatter that accepts a safe session/channel label, provider/model identity, effective context budget, observed prompt/session token count, computed overage, and concise recovery actions, then wires it through the embedded runner, formatAssistantErrorText, sanitize/fallback delivery paths, and focused regression tests.
