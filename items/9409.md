## Summary

Keep #9409 open. Current main still surfaces generic context-overflow messages in the runner, shared formatter, sanitizer, and auto-reply fallback paths. The code computes useful token diagnostics for precheck/logging and stores provider/model/context tokens in metadata, but the user-visible payload still does not include the requested safe session/channel label, context size, model limit, or overage. The existing 2026-04-26 Codex review comment from steipete is still consistent with current main, and the related diagnostic-message PR #15749 was closed unmerged.

## Best Possible Solution

Keep the issue open. The best implementation is a shared context-overflow diagnostic formatter that accepts a safe session/channel label, provider/model identity, effective context budget, observed prompt/session token count, computed overage, and concise recovery actions, then wires it through the embedded runner, precheck/attempt path, `formatAssistantErrorText`, sanitizer/normalization paths, and auto-reply fallbacks with focused regression tests.
