## Summary

Keep #59750 open. Current main still honors persisted per-session model selection ahead of the configured default on continued/resumed sessions, and there is no config/schema/docs surface for a resume-time strategy such as `session` versus `default`. Related #28965 overlaps around model cleanup/reset operations, but it is not a clear canonical replacement for this resume policy request.

## Best Possible Solution

Keep the issue open unless a maintainer explicitly consolidates it into a canonical model policy issue. The concrete product path is to define a core session/model policy option, likely under agent/session model configuration rather than a new top-level `model` object, document its interaction with explicit `/model` and `sessions.patch` overrides, then enforce it in session resolution or pre-run model selection with tests for resumed sessions using stored overrides versus current defaults.
