## Summary

Keep #68986 open. Current main does not implement the PR's central behavior: shared delivery-time cleanup for leaked `<channel|>` scaffolding and runaway repeated visible replies. The existing sanitizer strips `<|...|>`/full-width special tokens plus reasoning/tool scaffolding, but `<channel|>` is not matched, no repeated-visible-suffix helper exists, and the Discord/block reply delivery paths still use the older trimming and block-tag stripping flow. The PR is draft and has substantial review concerns, so it should be reworked or superseded by a narrower fix, not closed as already handled.

## Best Possible Solution

Keep the item open or replace it with a narrower maintainer-owned PR. The best fix belongs in the shared visible-text delivery sanitizer and final delivery paths, with bounded heuristics, delivery-only repeated-suffix normalization, preserved history/stream semantics, and focused regressions for `<channel|>` leaks, intentional literal marker prose, reply directives, indentation, chunked `text_end`/`message_end`, and no-delimiter runaway duplication.
