## Summary

Current `main` aligns ComfyUI with the plugin config surface: the documented `plugins.entries.comfy.config` path accepts `mode` plus `image`/`video`/`music`, validator wiring applies the plugin manifest schema there, and runtime now prefers that path with legacy fallback only.

## Evidence

- **Comfy docs now use plugin config path:** The current ComfyUI docs show `mode`, `baseUrl`, and nested `image`/`video`/`music` under `plugins.entries.comfy.config`, not under `models.providers.comfy`.
  - file: [docs/providers/comfy.md:49](https://github.com/openclaw/openclaw/blob/5b8bd6371c666a5de43c82305e0315771e8e49d5/docs/providers/comfy.md#L49)

- **Bundled Comfy manifest schema includes the rejected keys:** The bundled `comfy` plugin manifest defines `configSchema` properties for `mode`, `baseUrl`, `apiKey`, `allowPrivateNetwork`, and nested `image`/`video`/`music` objects with `workflowPath`, `promptNodeId`, and `outputNodeId` fields.
  - file: [extensions/comfy/openclaw.plugin.json:30](https://github.com/openclaw/openclaw/blob/5b8bd6371c666a5de43c82305e0315771e8e49d5/extensions/comfy/openclaw.plugin.json#L30)

- **Config validator applies plugin manifest schemas at plugins.entries.<id>.config:** `validateConfigObjectWithPluginsBase` validates each plugin entry against `record.configSchema` and reports issues at `plugins.entries.<pluginId>.config...`, which is the path used by the Comfy manifest schema.
  - file: [src/config/validation.ts:1155](https://github.com/openclaw/openclaw/blob/5b8bd6371c666a5de43c82305e0315771e8e49d5/src/config/validation.ts#L1155)

- **Runtime now prefers plugin config and only falls back to legacy provider config:** `getComfyConfig` first reads `cfg.plugins.entries.comfy.config`; only if that is absent does it fall back to `cfg.models.providers.comfy`. This matches the current docs and leaves the old path as compatibility-only.
