## Summary

Keep #8122 open. Current main still supports session reset policy at global, chat-type, and channel scope only. Telegram group/topic config accepts other per-group controls, but not `session.reset`; the strict schema rejects the proposed shape, and session initialization only resolves channel-level reset overrides before applying type/global policy. The related PR #8134 was closed unmerged, and the broader #20888 lifecycle issue does not supersede this per-Telegram-group configuration request.

## Best Possible Solution

Keep the issue open. Implement an additive, account-aware reset policy override for Telegram group/topic config, or add a generic channel group/topic reset resolver if maintainers want the feature across plugins. Update Telegram types, strict Zod schema, generated config metadata/help, public docs, and focused precedence tests for topic/group over channel over type/global, including multi-account behavior.
