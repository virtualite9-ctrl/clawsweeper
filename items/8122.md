## Summary

Keep #8122 open. Current main at 406ae72fd2789c78053f6b94006ec8f488bc1c00 still supports automatic session reset only at global, chat-type, and channel override levels. Telegram group/topic config does not accept the proposed `session.reset` block, strict validation would reject it, and the runtime reset resolver still receives only a channel-level override. The linked PR #8134 was closed unmerged, and #20888 is a broader closed lifecycle request rather than a canonical tracker for this per-Telegram-group feature.

## Best Possible Solution

Keep #8122 open. Implement an additive, account-aware reset override for Telegram group/topic config, or introduce a generic channel/plugin reset resolver if maintainers want conversation-scoped policies across plugins. Update Telegram config types, strict Zod schema, generated config labels/help, public docs, and focused precedence tests covering topic/group over channel over type/global, including multi-account Telegram cases.

## Risks / Open Questions

- A naive implementation keyed only by Telegram group id could apply the wrong policy in multi-account setups; the reset override should include account/route context or use an account-aware resolver.
