## Summary

Keep this issue open. Current main still does not pass client-provided Chat Completions tools through to the agent or return OpenAI-compatible tool_calls, and the provided GitHub context shows open PR #66278 explicitly closes this issue as the active implementation candidate.

## Best Possible Solution

Keep this issue open and use PR #66278 as the active implementation candidate. The maintainer path is to review, revise, and land that PR or an equivalent change in src/gateway/openai-http.ts, with tests in src/gateway/openai-http.test.ts covering tools/tool_choice parsing, non-stream message.tool_calls, streaming delta.tool_calls, and role:"tool" follow-up messages with tool_call_id.
