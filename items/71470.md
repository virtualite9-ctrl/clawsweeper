## Summary

Current main already guards the BlueBubbles macOS Contacts SQLite lookup by normalizing lookup keys to digit-only strings before constructing the SQL `IN` list. The same implementation is present in `v2026.4.24`, and the PR's proposed sqlite `.parameter set` path is riskier because digit values bind as integers in the sqlite CLI.

## Best Possible Solution

Close the PR as already implemented on main and keep the shipped digit-only normalized literal path. If maintainers want extra coverage, add a separate narrow regression test that raw punctuation or quote-bearing inputs normalize before SQL construction, rather than merging the sqlite version-probe and dot-parameter branch.
