## Summary

Keep open. Current main still has the central macOS doctor false-positive problem: LaunchAgent scanning can report non-gateway OpenClaw ecosystem plists as extra gateway-like services, and doctor/status still generate static cleanup hints for the profile gateway label whenever any extra service is found. The `gateway.mode` config-source subclaim looks mostly obsolete, but the noisy LaunchAgent and unsafe cleanup-hint behavior remains.

## Best Possible Solution

Keep this issue open. The best maintainer path is to fix `src/daemon/inspect.ts` so macOS LaunchAgent detection requires an explicit gateway marker or parsed ProgramArguments that actually invoke the gateway, then change doctor/status cleanup-hint generation to target only the detected stale/inactive extra service rather than the active profile gateway label. Rebase/review #39118 and #52515 or implement the same behavior directly, and add fixture-based Darwin plist tests covering non-gateway OpenClaw ecosystem services plus active-gateway cleanup suppression.
