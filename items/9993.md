## Summary

Current main has stronger built-in config validation and agent-facing mutation guards, but #9993 asks for a synchronous, plugin/API-level config-write veto point. That specific surface is still missing: config.set, config.patch, and config.apply write after built-in validation/preflight without invoking any config pre-apply hook, and the public plugin hook catalog has no equivalent hook that receives current and pending config for direct RPC writes.

## Best Possible Solution

Keep #9993 open as a plugin API/product-direction issue. The best implementation is a documented synchronous config-write interception point invoked by config.set, config.patch, and config.apply after the pending config is computed and built-in validation/preflight passes, but before writeConfigFile. It should expose current config, pending config, action, raw patch/full payload, baseHash, actor/client metadata, changed paths, and a typed reject result, with tests proving a plugin can reject direct RPC writes without changing disk or the active runtime config.
