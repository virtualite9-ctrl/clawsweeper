## Summary

Current main already covers this restart failure mode. The gateway service unit now uses `KillMode=control-group` so service-managed children stay in the same lifecycle, and the restart path now detects stale gateway listeners on the port, kills their process tree, and retries restart automatically. The same implementation is present in release `v2026.4.22`.

## Close Comment

Closing as implemented on `main` after Codex review. The current gateway restart path now detects stale gateway listeners still holding the port, kills their process tree, and retries restart automatically (`src/cli/daemon-cli/restart-health.ts:153`, `src/cli/daemon-cli/lifecycle.ts:241`). OpenClaw also now renders systemd units with `KillMode=control-group` specifically to keep gateway children in the same service lifecycle so restarts do not leave orphan workers behind (`src/daemon/systemd-unit.ts:83`). This behavior is already present in release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`).
