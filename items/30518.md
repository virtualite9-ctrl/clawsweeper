## Summary

Keep #30518 open. Current main still has shared/generic `pin` and `unpin` action names plus Telegram Bot API helper functions, but the bundled Telegram message action adapter does not advertise or dispatch manual Telegram `pin`/`unpin`; those actions still fail before reaching the low-level helpers.

## Best Possible Solution

Keep the issue open. Implement this in the bundled Telegram plugin by adding Telegram `pin` and `unpin` to action discovery and `TELEGRAM_MESSAGE_ACTION_MAP`, adding runtime aliases/branches that call `pinMessageTelegram` and `unpinMessageTelegram`, deciding whether separate config gates such as `pinMessage`/`unpinMessage` are needed, adding focused adapter/runtime contract tests, and updating Telegram plus CLI docs to distinguish manual pin/unpin from send-time `--pin` delivery pinning.
