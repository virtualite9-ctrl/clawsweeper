## Summary

Keep #30518 open. Current main still has shared `pin`/`unpin` action names and Telegram Bot API helper functions, but the bundled Telegram message action adapter does not advertise or dispatch manual Telegram `pin`/`unpin`; those actions still fail at the adapter/runtime boundary before reaching the low-level helpers.

## Best Possible Solution

Keep the issue open. Implement this in the bundled Telegram plugin by adding `pin` and `unpin` to Telegram message action discovery and `TELEGRAM_MESSAGE_ACTION_MAP`, adding runtime aliases/branches that call `pinMessageTelegram` and `unpinMessageTelegram`, deciding whether config gates are needed, adding focused adapter/runtime contract tests, and updating Telegram docs to distinguish manual pin/unpin from send-time delivery pinning.
