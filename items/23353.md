## Summary

Keep open. Current main at 921ffad7c7a6c6f96acecc32414f7edef6bfad23 still does not implement Anthropic native server-side tools for web_search, web_fetch, code_execution, or Programmatic Tool Calling allowed_callers. The prior related implementation PRs #23469 and #23573 are closed and unmerged per the provided GitHub context, so they do not supersede or ship the request.

## Best Possible Solution

Keep #23353 open as a provider/runtime feature request. The best path is an explicit Anthropic implementation covering config/schema/docs, payload injection of server tool definitions, suppression or coexistence rules for client web_search/web_fetch, response content-block preservation/rendering, PTC allowed_callers behavior, and focused unit/integration tests. If maintainers want narrower tracking, split web_search/web_fetch and code_execution/PTC into canonical follow-up issues rather than closing this broad tracker as already handled.
