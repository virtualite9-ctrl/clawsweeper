## Summary

Current main now makes CLI-routed sessions participate in context-engine compaction through a CLI-specific transcript lifecycle: CLI turns are persisted to the OpenClaw transcript, CLI JSON/JSONL usage is parsed and stored as fresh token state when available, and over-budget CLI transcripts call the active context engine's compact() with currentTokenCount/tokenBudget before clearing the external CLI resume state. This solves the central non-functional plugin compaction problem without making CLI backends use embedded-Pi assemble()/afterTurn semantics.

## Best Possible Solution

Keep the shipped CLI-specific compaction lifecycle on main. If plugin authors need non-compaction per-turn indexing for CLI sessions, track that as a narrower context-engine lifecycle/API enhancement rather than keeping this broad non-functional compaction bug open.
