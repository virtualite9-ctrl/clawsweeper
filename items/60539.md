## Summary

Current main already addresses the reported node/gateway auth confusion well enough: node-host auth is explicitly separated from `node.json`, handshake failures now return reason-specific token mismatch/missing guidance, and the docs/doctor flow now document node pairing versus approvals and auth troubleshooting.

## Evidence

- **`node.json` no longer carries a `gateway.token`-style setting:** The node-host config schema only persists `nodeId`, a pairing `token`, display name, and gateway connection fields (`host`, `port`, `tls`, `tlsFingerprint`). There is no `gateway.token` field in `~/.openclaw/node.json`, so the issue's concrete reproduction no longer matches current code.
