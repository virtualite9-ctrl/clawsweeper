## Summary

Keep #20786 open. Current main still only allows Telegram Business update names; it does not handle business_message/business_connection updates, route them into the inbound pipeline, dedupe/sequence them, or attach business_connection_id for outbound replies. The supplied discussion and PR #53629 are useful implementation evidence, but the PR is open/unmerged and the feature remains missing on main.

## Best Possible Solution

Keep #20786 open as the canonical Telegram Business support tracker. Land PR #53629 or an equivalent Telegram plugin implementation that registers business_connection and business_message handlers before the normal message handler, routes business_message through the existing inbound pipeline, extends dedupe and sequential keys, persists/maps business_connection_id for outbound replies, and adds focused tests plus user-facing Telegram docs.
