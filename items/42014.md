## Summary

Keep this PR open. Current main still does not expose session sync status, pending bytes/messages, last sync time, or last sync error through the memory manager status object or `openclaw memory status`. The PR is still a plausible focused feature, but it needs a rebase from the old `src/memory` / `src/cli` paths into the current `extensions/memory-core` plugin layout and normal maintainer review.

## Best Possible Solution

Keep the PR open and ask for a rebase into `extensions/memory-core`. The best implementation path is to add a typed session-sync status helper/state on the memory-core manager, render it in `openclaw memory status` and JSON output without leaking transcript contents, preserve dirty/error state on failed syncs, and add focused tests for manager status plus CLI deep output. After rebase, review the timestamp/logging fixes from `2df928fbc4df1b5e6f1137c04cb6cbb2bf547706` against the current split sync helpers before landing.
