## Summary

Keep this PR open. Current main still lacks the requested session sync status payload and `openclaw memory status` rendering. The PR is stale against the current `extensions/memory-core` plugin layout, but it remains a focused implementation candidate rather than cleanup-close material.

## Best Possible Solution

Keep this PR open and ask for a rebase into `extensions/memory-core`. The best path is a typed session-sync status helper/state on the memory-core manager, exposed through `MemoryIndexManager.status()` under `custom.sessionSync`, rendered by `openclaw memory status` including JSON/deep output, covered by focused manager and CLI tests, and reviewed for sanitized status/log content.
