## Summary

Keep this PR open. Current main still lacks the requested session sync status surface in memory manager status and `openclaw memory status`, while the PR is a focused feature with useful review follow-up. The work needs a rebase from the old core memory paths into the current `extensions/memory-core` plugin implementation before normal maintainer review can decide whether to land it.

## Best Possible Solution

Keep this PR open and ask for a rebase into `extensions/memory-core`. The best implementation path is a typed session-sync status helper/state on the memory-core manager, exposed through `MemoryIndexManager.status()` under `custom.sessionSync`, rendered by `openclaw memory status` including JSON/deep output, with focused manager and CLI tests plus review for sanitized status/log content.
