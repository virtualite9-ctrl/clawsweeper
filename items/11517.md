## Summary

Keep #11517 open. Current main still has the import-heavy Pi attempt runner and Discord monitor surfaces named by the issue, and the merged harness/context-engine seam work does not implement the requested RunContext/AgentContext dependency aggregation. The existing issue discussion also already contains a Codex review from steipete keeping this open for the same central reason.

## Best Possible Solution

Keep #11517 open as the canonical architecture tracker. The best path is an incremental refactor that introduces a prepared RunContext/AgentContext, or smaller service bundles, for the Pi attempt runner and Discord monitor; preserves plugin SDK boundaries; separates pure moves from behavior changes; and adds focused tests around each extracted seam.
