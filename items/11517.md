## Summary

Keep #11517 open. Current main still has the import-heavy embedded attempt runner and Discord monitor surfaces the issue is tracking, and the merged harness/runtimePlan work is an adjacent prepared-attempt policy bundle rather than the requested RunContext/AgentContext dependency aggregation. The latest issue discussion already includes a maintainer/Codex review saying to keep this item open, and current source still supports that conclusion.

## Best Possible Solution

Keep #11517 open as the canonical tracker. The best path is an incremental refactor that introduces a prepared RunContext/AgentContext or smaller service bundles for the Pi attempt runner and Discord monitor, preserves plugin SDK and extension boundaries, separates pure moves from behavior changes, and adds focused tests around each extracted seam.
