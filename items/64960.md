## Summary

Keep open. Current main still has only a single compaction target (`agents.defaults.compaction.model` or the session model) and no compaction-specific model fallback chain. The overflow loop is bounded now, and plugins can provide one custom compaction provider, but neither implements the requested ephemeral fallback candidate list for Azure/content-filter summarization failures.

## Best Possible Solution

Keep this issue open as an actionable core reliability bug. The likely fix belongs in compaction orchestration/config: add a validated `agents.defaults.compaction.fallbacks` or equivalent candidate list, try each candidate ephemerally per compaction attempt without persisting a session model switch, add regression coverage for an Azure-style 400 content-filter rejection from the primary compaction model, and coordinate the final hard escape with #64962.
