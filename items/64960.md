## Summary

Keep open. Current main bounds the overflow retry loop and can use one compaction-only model override or a plugin compaction provider, but it still does not implement the requested compaction model fallback chain. The config/schema/docs expose a single `agents.defaults.compaction.model`, compaction target resolution selects one provider/model, and summarization retries reuse that same model.

## Best Possible Solution

Keep this issue open as an actionable reliability bug. The likely fix belongs in compaction orchestration: add a validated `agents.defaults.compaction.fallbacks` or equivalent candidate list, try candidates ephemerally per compaction attempt without persisting a live-session model switch, add regression coverage for an Azure-style HTTP 400 content-filter rejection from the primary compaction model, and coordinate the final hard escape with the related escalation work in #64962.
