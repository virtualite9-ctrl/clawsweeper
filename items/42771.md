## Summary

Keep this PR open. Current main still uses the hidden Playwright cache path in Docker, the public docs still document that path, and the same-author paired bug #42819 remains open. The PR is a plausible in-repo Docker/docs fix, but it needs updates before merge because the supplied diff still has Dockerfile permission/cache risks and stale docs targeting.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to update it against current main, make `/opt/playwright` writable for the non-root runtime user with and without `OPENCLAW_INSTALL_BROWSER=1`, remove or clearly justify the BuildKit apt-cache deletion, update `docs/install/docker.md` plus the current browser-control docs, add focused Dockerfile/docs assertions for the new path and symlink, and keep #42819 open until a corrected implementation lands.

## Risks / Open Questions

- Merging the supplied diff as-is can leave `/opt/playwright` root-owned in the default build path where `OPENCLAW_INSTALL_BROWSER` is unset, so runtime browser installs as `USER node` can fail with `EACCES`.
- The supplied diff deletes `/var/lib/apt/lists/*` inside a BuildKit cache mount, which can wipe the named apt lists cache across builds without reducing the committed image layer size.
- The PR edits `docs/tools/browser.md`, but current main's Docker Playwright instructions are in `docs/tools/browser-control.md`, so the branch needs docs reconciliation before merge.
