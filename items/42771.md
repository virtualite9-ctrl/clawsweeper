## Summary

Keep this PR open. Current main still installs Docker Playwright browsers under the hidden cache path and the public docs still document that path. The same-author paired bug #42819 remains open, and this PR is a plausible in-repo implementation candidate, but its current diff still needs Dockerfile permission/cache fixes and docs reconciliation before merge.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to update it against current main, set `/opt/playwright` ownership for the node runtime user before runtime installs and recursively after preinstall, remove the BuildKit apt-cache deletion, add focused Dockerfile/docs assertions for the path and symlink, update `docs/install/docker.md` plus `docs/tools/browser-control.md`, and keep #42819 open until a corrected implementation lands.

## Risks / Open Questions

- Merging the current diff as-is can leave `/opt/playwright` unwritable by `USER node` in the default build path where `OPENCLAW_INSTALL_BROWSER` is unset.
- The PR deletes `/var/lib/apt/lists/*` inside a BuildKit cache mount, which can degrade rebuild behavior without reducing committed image size.
- The PR updates `docs/tools/browser.md`, but current Docker Playwright guidance lives in `docs/tools/browser-control.md`, so the docs patch is stale against main.
