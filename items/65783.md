## Summary

Keep open. Current main still has the unsafe coarse UTF-16 split in both memory-host-sdk implementations and still forwards raw embedding text/structured inputs without an unpaired-surrogate sanitizer. The PR is also paired with the same-author open bug report #65782 and contains useful targeted work, but the submitted diff appears incomplete because it patches the package mirror rather than the canonical `src/` runtime implementation exported through the plugin SDK.

## Best Possible Solution

Keep this PR open and ask for a focused revision: patch the canonical `src/memory-host-sdk/host/internal.ts` chunker, keep the package mirror aligned if it remains present, add the coarse-boundary regression test to the current relevant test suite, and add memory-core embed-boundary sanitizer coverage. Once that version lands, it can resolve the same-author open bug report #65782 and the older #27753 failure class.
