## Summary

Keep #40205 open. Current main still uses Ollama's deprecated `/api/embeddings` contract in the active bundled Ollama memory embedding provider, and the tests still assert the old URL, `prompt` request field, and `embedding` response shape. The PR branch edits the old `src/memory/embeddings-ollama.*` paths, so it needs a rebase to the current `extensions/ollama` layout, but the requested bug fix remains valid and is not superseded by the closed related PRs.

## Best Possible Solution

Keep the PR open and ask for a rebase/update against `extensions/ollama/src/embedding-provider.ts` and `extensions/ollama/src/embedding-provider.test.ts`: switch to `POST /api/embed`, send `{ model, input }`, parse `embeddings[0]`, update tests away from `prompt` and `embedding`, and keep the current serialized batch behavior plus Ollama memory timeout semantics. Close only after this PR is updated or a separate merged PR fixes #39983 on main.
