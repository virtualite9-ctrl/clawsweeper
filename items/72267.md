## Summary

Keep PR #72267 open. Current main still emits zsh completion output with a leading blank line before `#compdef`, while the PR removes exactly that newline. The existing zsh test covers sourcing/deferred registration, not the `fpath` autoload first-line behavior described by the PR and zsh documentation.

## Best Possible Solution

Keep PR #72267 open for maintainer review and merge or adapt its one-line zsh generator fix. Add a focused regression assertion such as `getCompletionScript("zsh", program).startsWith("#compdef openclaw\n")`; optionally add a zsh `fpath`/`compinit` integration check when zsh is available.
