## Summary

Keep open. Current `main` still has a two-phase gateway plugin bootstrap: it does an initial startup load when any configured channel requests deferred full runtime, then performs a second deferred reload with the same startup plugin scope after the gateway attaches. Tests explicitly cover suppressing duplicate diagnostics on that deferred reload rather than removing the second load, so this issue is not closeable as fixed.

## Evidence
