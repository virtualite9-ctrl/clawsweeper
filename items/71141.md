## Summary

Keep #71141 open. Current main still lacks the requested Telegram outbound reaction-directive behavior, while the PR and linked issue describe a concrete Telegram bug with source areas and tests that are plausibly useful for maintainer review.

## Best Possible Solution

Keep the PR open for maintainer review or replacement by an equivalent fix. The best path is to land a validated implementation that parses `[[react:...]]` and `[[react_to_current:...]]`, carries the intent as `channelData.telegram.reaction`, allows valid Telegram reaction-only replies, calls `reactMessageTelegram` before optional text/media sends, and adds focused normalization plus Telegram adapter/delivery regression coverage. Close #71140 only after that behavior lands on main.
