## Summary

Current main still has the Synology Chat webhook behavior reported here: the registered webhook handler rejects HEAD with 405 and successful accepted POSTs still ACK with 204/no body. The issue is also paired with open implementation candidates, including the reporter's focused PR #53441 with closing syntax, so it should remain open until a fix lands or is explicitly superseded.

## Best Possible Solution

Keep this issue open. Review and either land the focused same-author PR #53441 or explicitly supersede it with #53522. The fix should stay in extensions/synology-chat/src/webhook-handler.ts: accept HEAD probes with 200 and change successful POST ACKs from 204/no body to 200 with {"success":true}, with updated webhook and integration tests covering both behaviors.
