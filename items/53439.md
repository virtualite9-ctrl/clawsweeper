## Summary

Keep open. Current main still has the reported Synology Chat webhook behavior: successful POSTs ACK with 204/no body and all non-POST methods, including HEAD, are rejected with 405. The provided GitHub context also shows open implementation candidates, including a same-author focused PR, so this issue should stay open until the fix lands or the PRs are closed.

## Best Possible Solution

Keep this issue open. Review and either land the focused Synology Chat PR #53441 or intentionally supersede it with #53522. The code change belongs in `extensions/synology-chat/src/webhook-handler.ts`: accept HEAD probes with 200 and change successful POST ACKs from 204/no body to 200 with `{"success":true}`, with updated webhook and integration tests covering both behaviors.
