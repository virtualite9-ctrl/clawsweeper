## Summary

Keep #46664 open. Current main has partial mobile queuing only: Android Mic mode queues recognized voice text in memory and drains on gateway reconnect, iOS has an in-memory Watch quick-reply queue, and the shared chat composer still refuses normal text sends when gateway health is unavailable. The requested persistent cross-platform offline command outbox for arbitrary text/voice intents, captured-time metadata, retry state, review/delete/reorder UI, and robust replay semantics is not implemented on main.

## Best Possible Solution

Keep the issue open, but narrow it into a staged mobile outbox design: persist the existing Android Mic queue and add an iOS/general text+voice queued-intent store with client UUID/idempotency key, capturedAt/timezone metadata, retry state, and reconnect replay through `chat.send`; then add queue status/review/delete UI and document mobile OS background-sync limits.
