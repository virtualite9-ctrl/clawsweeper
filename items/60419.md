## Summary

Current `main` already has the auth-profile cooldown handling and documentation needed for this report: auth routing is per-agent/per-session, cooldown state is persisted in `auth-state.json`, expired cooldowns are cleared so profiles do not stay artificially escalated, active cooldown windows are not extended by mid-window retries, and fallback now uses model-aware cooldown probes instead of looping indefinitely on the same unavailable profile.

## Evidence

- **Per-agent persisted auth-routing state is now documented:** `docs/concepts/model-failover.md` documents that secrets live in per-agent `auth-profiles.json` and runtime auth-routing state lives in per-agent `auth-state.json`, which explains why one agent can be cooling down while another agent on the same provider key still works.
