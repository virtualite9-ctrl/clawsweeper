## Summary

Keep open. Current `main` still has real sub-agent documentation gaps, but this PR is not ready for conservative cleanup because parts of the proposed wording conflict with the runtime and with other sections of the same page.

## Evidence

- **Current sub-agent docs are still missing or stale on main:** `docs/tools/subagents.md` still omits `announceTimeoutMs` from the config example, does not document `streamTo`, still collapses thread-binding config into a generic adapter note, and still documents depth 2 as a permanent leaf that can never spawn.
  - file: [docs/tools/subagents.md:101](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/docs/tools/subagents.md#L101)
  - command: `nl -ba docs/tools/subagents.md | sed -n '90,240p'`
  - sha: [ec3dbd22a4de](https://github.com/openclaw/openclaw/commit/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46)
- **Runtime supports deeper nesting than the doc page currently describes:** Sub-agent role/capability resolution is based on `depth < maxSpawnDepth`, and both native and ACP spawn paths reject only when `callerDepth >= maxSpawnDepth`. That means depth-2 sessions can spawn when `maxSpawnDepth` is 3 or more, so a docs change here is still needed.
  - file: [src/agents/subagent-capabilities.ts:123](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/src/agents/subagent-capabilities.ts#L123)
  - command: `nl -ba src/agents/subagent-capabilities.ts | sed -n '120,210p'`
  - sha: [ec3dbd22a4de](https://github.com/openclaw/openclaw/commit/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46)
- **The PR's `streamTo` wording would still be inaccurate:** `sessions_spawn` accepts `streamTo`, but the tool returns an error unless `runtime` is `acp`; the dedicated test covers that exact rejection. A generic sub-agent docs line for `streamTo` would mislead users.
  - file: [src/agents/tools/sessions-spawn-tool.ts:229](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/src/agents/tools/sessions-spawn-tool.ts#L229)
  - command: `nl -ba src/agents/tools/sessions-spawn-tool.ts | sed -n '120,260p'; nl -ba src/agents/tools/sessions-spawn-tool.test.ts | sed -n '393,409p'`
  - sha: [ec3dbd22a4de](https://github.com/openclaw/openclaw/commit/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46)
- **The PR's deny-list simplification is incomplete vs actual policy:** Sub-agents always deny `gateway`, `agents_list`, `session_status`, `cron`, and `sessions_send`; leaf sessions additionally deny `subagents`, `sessions_list`, `sessions_history`, and `sessions_spawn`. So wording that implies only four denied tools is not accurate.
  - file: [src/agents/pi-tools.policy.ts:30](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/src/agents/pi-tools.policy.ts#L30)

- **PR discussion already identified the main contradiction:** Review comments on PR #59658 flagged that the proposed expanded depth table says depth 2 can spawn when `maxSpawnDepth >= 3`, while the unchanged 'Tool policy by depth' text still says depth 2 can never spawn. Current runtime code matches the review feedback, so the PR should stay open for correction rather than be closed.
