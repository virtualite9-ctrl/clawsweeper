## Summary

Keep #14593 open. Current main still lacks an official Docker/Linuxbrew setup path, the Docker setup script does not pass or persist any brew-related build option, `openai-whisper` remains brew-only, and the installer/onboarding path still leads to a brew-missing failure for selected brew-backed skills. The issue discussion also has a same-day maintainer/Codex keep-open comment and unresolved fix PR references, so conservative cleanup should not close it.

## Best Possible Solution

Keep #14593 open until a focused Docker/skills fix lands. The likely maintainer path is an official opt-in Linuxbrew setup that works in the non-root Docker image and is documented/tested, OS-scoped or non-brew installer metadata for bundled brew-only skills such as `openai-whisper`, or onboarding/status behavior that avoids offering impossible brew installs in the shipped Docker flow. Validate across `Dockerfile`, `scripts/docker/setup.sh`, `docs/install/docker.md`, bundled skill metadata, `src/agents/skills-install.ts`, and `src/commands/onboard-skills.ts` tests.
