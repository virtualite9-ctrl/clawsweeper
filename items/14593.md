## Summary

Keep #14593 open. Current main still has no official Docker/Linuxbrew install path, the Docker setup script does not pass or persist any brew-related build option, the reported `openai-whisper` skill remains brew-only, and the installer still returns `brew not installed` when no brew executable is present. The issue discussion also has a recent maintainer/Codex keep-open signal while active fix PR candidates remain unresolved.

## Best Possible Solution

Keep #14593 open until a focused Docker/skills fix lands. The best maintainer path is either an official opt-in Linuxbrew setup that works for the non-root Docker image and is documented/tested, non-brew or OS-scoped installers for bundled brew-only skills like `openai-whisper`, or onboarding/status behavior that does not offer impossible brew installs in the shipped Docker flow. Validate the eventual fix across `Dockerfile`, `scripts/docker/setup.sh`, `docs/install/docker.md`, skill metadata, installer behavior, and onboarding tests.
