## Summary

Keep PR #72218 open. Current main still has the heartbeat final-payload sanitation gap the PR targets: `buildReplyPayloads` returns raw heartbeat payloads before the `HEARTBEAT_OK` stripping branch, while the docs promise heartbeat OK acknowledgments are suppressed by default. The PR is a focused fix with targeted regression coverage and an addressed bot-review concern; related items #71860, #72253, and #69492 cover adjacent exec/system-event noise, not this exact final heartbeat payload bypass.

## Best Possible Solution

Keep #72218 open and merge or adapt its narrow fix: run heartbeat final payloads through the shared `HEARTBEAT_OK` token sanitation path while preserving the existing non-heartbeat Bun socket error formatting scope. Leave broader async exec/system-event audience cleanup to the related open items (#72253/#71860/#69492).
