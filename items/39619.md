## Summary

Current `main` already keeps ended run-mode orchestrators visible as active while descendant subagents are still pending, so the reported `subagents list` behavior is implemented and appears shipped.

## Evidence

## Close Comment

Closing from a Codex review because current `main` already implements this behavior. `/subagents list` goes through `buildSubagentList()` (`src/auto-reply/reply/commands-subagents/action-list.ts:5`), and that code now keeps an ended subagent run active whenever it still has pending descendants (`src/agents/subagent-list.ts:132`) and renders `active (waiting on N child|children)` instead of `done` (`src/agents/subagent-list.ts:139`). There is also an explicit regression test for the reported case in `src/agents/subagent-list.test.ts:80`. The same implementation is present in release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`). If you can still reproduce this on a newer build, please open a fresh issue with the exact version and a current reproduction trace.
