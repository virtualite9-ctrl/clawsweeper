## Summary

Keep this PR open. The checked-out main at 66f4b52d does not include the Memory v2 sidecar, `memoryV2.*` config schema, ingest registration, rerank hook, or `openclaw memory sidecar` CLI proposed by this PR. Existing QMD, temporal decay, and MMR work overlap in memory search quality, but they do not implement this PR's per-location SQLite metadata, pin/salience/supersession state, ingest pipeline, or default-off bundled `memory-core` foundation. The PR is also still an open, unmerged implementation candidate with unresolved behavior and security-relevant review comments, so conservative maintainer cleanup should not close it.

## Best Possible Solution

Keep this PR open for explicit maintainer product, functional, and security review. The best path is to split or review the Memory v2 work in narrower slices, resolve the remaining sidecar CLI and rerank/shadow-touch review comments, and decide whether this belongs in bundled `memory-core` or behind a plugin/SDK design. Do not close it as implemented because current main lacks the proposed sidecar, config, ingest, rerank, dreaming shadow-touch, and CLI surfaces.
