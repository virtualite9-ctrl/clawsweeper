## Summary

Keep this PR open. Current main still does not implement custom WebSocket upgrade headers for `openclaw node run` or `openclaw node install`, and the closed linked issue #42792 was explicitly closed as superseded by this still-open implementation PR. The PR is focused and materially useful, but it still needs maintainer review rather than conservative cleanup because the current diff changes credential-bearing service env/config behavior and appears to have at least one remaining precedence/docs/changelog issue to resolve before merge.

## Best Possible Solution

Keep this PR open as the active implementation candidate for #42792. The best path is maintainer review plus a small follow-up on the PR: preserve explicit `--header` precedence over install-time `CF_*` env vars, make the docs honest about whether header secrets are persisted to node config, add the required changelog attribution, then re-run the changed gateway/node CLI checks before merge.

## Risks / Open Questions

- At PR head, `buildNodeInstallPlan` serializes explicit `--header` values into `OPENCLAW_NODE_HEADERS` but still unconditionally copies exported `CF_ACCESS_CLIENT_ID` and `CF_ACCESS_CLIENT_SECRET` into the service environment. Because `resolveNodeHostHeaders` applies `CF_*` after `OPENCLAW_NODE_HEADERS`, stale shell CF credentials can override explicit install headers in the daemon path. See `src/commands/node-daemon-install-helpers.ts:58` and `src/node-host/runner.ts:146` at `0fcd100ccbb5b38d2442f9ac134d02b129594bc1`.
- The docs say environment variables can be used so secrets are not stored in config, but `runNodeHost` persists resolved headers into `config.gateway.headers` before saving `node.json`, including env-derived headers. The docs and implementation should be reconciled before merge. See `docs/cli/node.md:72` and `src/node-host/runner.ts:199` at `0fcd100ccbb5b38d2442f9ac134d02b129594bc1`.
- The PR changelog entry lacks the required `Thanks @author` attribution for a user-facing changelog addition, so it needs cleanup before landing.
