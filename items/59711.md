## Summary

Current `main` already mitigates provider 429/retry storms: rate-limited auth profiles enter a stepped cooldown window, same-provider retry rotations are capped and configurable, and the runner falls across configured model fallbacks instead of hammering the same provider indefinitely.

## Risks / Open Questions

- Current main solves the immediate retry-storm problem with cooldowns plus configurable retry-rotation limits, but it does not expose a user-configurable per-attempt exponential delay curve matching the exact `2s -> 8s -> 15-30s` proposal.
