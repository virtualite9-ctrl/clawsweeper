## Summary

Keep open. Current main improves ACP lifecycle tracking and bundles newer acpx, but the specific requested guarantee is still missing: an ACP child run can be acknowledged as accepted before the child prompt is mirrored into the OpenClaw session transcript, and ACP transcript persistence still happens only after the ACP turn completes. A stalled or pre-output failure can therefore still leave `sessions_history` empty for the accepted child session.

## Best Possible Solution

Keep this open and fix the ACP spawn/dispatch contract: persist the initial ACP task into the child OpenClaw transcript before returning `accepted`, or fail the spawn if that mirror cannot be written. Also persist the prompt on ACP run errors/timeouts before final error delivery, and add a regression test where an ACP `runTurn` stalls or fails before assistant output while `sessions_history(childSessionKey)` still contains the initial user task plus diagnostic status.
