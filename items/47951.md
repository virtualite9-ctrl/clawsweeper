## Summary

Keep open. Current main still accepts ACP child runs before any child-session transcript message is persisted, so the reported `accepted` + empty `sessions_history` state remains plausible for long-running Codex ACP tasks. I found no repo-side fix on `main` that would write the initial user turn at accept/start time.

## Evidence

- **ACP spawn returns accepted immediately after queuing the child agent run:** `spawnAcpDirect()` calls the gateway `agent` method with the child `sessionKey` and then returns an `accepted` result once it gets a run id back; it does not persist any transcript message at this point.
  - file: [src/agents/acp-spawn.ts:1264](https://github.com/openclaw/openclaw/blob/d38ed0831d7372c8505dc944aea69b5b83da1435/src/agents/acp-spawn.ts#L1264)
  - command: `nl -ba src/agents/acp-spawn.ts | sed -n '1248,1286p'`
  - sha: [d38ed0831d73](https://github.com/openclaw/openclaw/commit/d38ed0831d7372c8505dc944aea69b5b83da1435)
- **ACP transcript persistence happens only after the turn finishes:** In the ACP execution path, `agent-command` awaits `acpManager.runTurn(...)` first and only then calls `persistAcpTurnTranscript(...)`. If the run is still executing or stalls before completion, the child transcript remains empty.
  - file: [src/agents/agent-command.ts:472](https://github.com/openclaw/openclaw/blob/d38ed0831d7372c8505dc944aea69b5b83da1435/src/agents/agent-command.ts#L472)
  - command: `nl -ba src/agents/agent-command.ts | sed -n '448,544p'`
  - sha: [d38ed0831d73](https://github.com/openclaw/openclaw/commit/d38ed0831d7372c8505dc944aea69b5b83da1435)
- **Transcript writer appends user and assistant together, not the initial user turn up front:** `persistTextTurnTranscript()` resolves the session file and appends the user message and assistant reply in one post-run persistence step. There is no earlier ACP path here that writes just the initial user task when the child session is accepted.
  - file: [src/agents/command/attempt-execution.ts:97](https://github.com/openclaw/openclaw/blob/d38ed0831d7372c8505dc944aea69b5b83da1435/src/agents/command/attempt-execution.ts#L97)
