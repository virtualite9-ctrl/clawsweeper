## Summary

Keep this issue open. The checked-out main branch still derives plain-HTTP Mattermost slash callback URLs when `commands.callbackUrl` is omitted, still validates callbacks by token membership rather than token-to-trigger binding, and the provided GitHub context shows open PR #65655 as the linked implementation candidate for this issue.

## Best Possible Solution

Keep this issue open and review PR #65655 or an equivalent maintainer-approved fix. The best implementation path is in the Mattermost plugin: reject non-HTTPS resolved native slash-command callback URLs before registration, preserve token-to-trigger bindings through activation and HTTP dispatch, and add focused regression coverage in `extensions/mattermost/src/mattermost/monitor-slash.test.ts`, `extensions/mattermost/src/mattermost/slash-http.test.ts`, and `extensions/mattermost/src/mattermost/slash-state.test.ts`. After that PR or an equivalent fix merges, close this issue as implemented with the merge SHA and shipped release if known.
