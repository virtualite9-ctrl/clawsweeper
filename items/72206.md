## Summary

Keep #72206 open. The issue is fresh and concrete, and current main still has the claude-cli live-session path that can produce the reported generic `FailoverError` with `reason=unknown` on a fresh live-process exit while reuse is a separate success path. The related items do not supersede it: #71793 was the large stdout/JSONL buffer fix, #71803 is an open watchdog/idle-timeout report, and the recent #57471 fix only changes model-switch fallback routing.

## Best Possible Solution

Keep the issue open. The best maintainer path is to reproduce the `system event --mode now` fresh-live-session path with claude-cli, capture redacted process argv/stdout/stderr/exit timing, then fix the fresh live-session startup path or diagnostics in `src/agents/cli-runner/claude-live-session.ts`. Add a focused regression test that distinguishes first live-session creation from reuse and verifies either success or a classified, actionable error instead of `reason=unknown`/`Claude CLI failed.`.
