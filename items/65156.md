## Summary

Keep open. Current main still does not directly fix the reported sqlite-vec load-success-but-unusable failure mode. The loader reports success immediately after loading the extension, memory-core marks vector availability true before validating that vec0 can actually create/use a table, and tests cover healthy sqlite-vec plus degraded warnings but not the ABI/missing-function case described by the reporter and two confirming commenters.

## Best Possible Solution

Keep this issue open. Fix the sqlite-vec owner path in packages/memory-host-sdk/src/host/sqlite-vec.ts and extensions/memory-core/src/memory/manager-sync-ops.ts by adding a deterministic post-load health probe, such as creating/dropping a temporary vec0 table or preparing a known sqlite-vec function. If that probe fails, mark vector.available=false, preserve the existing in-process cosine fallback, surface the actionable load error in status/doctor/logs, and add a regression test for loadExtension succeeding while vec0/functions are unavailable. If the probe confirms a bundled binary ABI mismatch, update or rebuild sqlite-vec with release-note coverage.
