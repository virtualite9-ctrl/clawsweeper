## Summary

Keep this issue open. Current main still does not expose Slack global shortcut or message-action shortcut events through `channels.slack`: the Slack monitor registers block actions and modal lifecycle handlers only, the interaction summary/event path has no shortcut variants, the public Slack docs list only block/modal interaction system events, and the Slack plugin-facing interactive API is still scoped to button/select callbacks rather than raw shortcuts. The prior Codex review comment in the issue is still consistent with current main; no related item in the supplied context supersedes this shortcut request.

## Best Possible Solution

Implement this in the bundled Slack plugin, or first add a typed plugin-facing Slack shortcut/raw-interaction API. The bundled path should add a shortcut event module that registers Bolt `app.shortcut` handlers for global and message-action shortcuts, acknowledges and mismatch-checks payloads, applies existing Slack sender/session authorization, enqueues sanitized stable system events with context keys, and updates Slack interaction tests plus `docs/channels/slack.md`.
