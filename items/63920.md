## Summary

Keep this issue open. Current main still does not expose Slack global shortcuts or message-action shortcuts through the Slack channel. The Slack interaction monitor registers block actions and modal lifecycle events only; docs and tests still describe that same limited surface, and the existing plugin-facing Slack interaction API is button/select-oriented rather than a raw Bolt shortcut hook.

## Best Possible Solution

Implement this in the Slack plugin itself unless maintainers first add a typed external shortcut API. The likely path is a new `extensions/slack/src/monitor/events/interactions.shortcuts.ts` module that registers `ctx.app.shortcut` for global and message-action shortcuts, acknowledges and validates mismatched app/team payloads, parses/sanitizes shortcut summaries, applies existing Slack sender/channel authorization and session-key routing, enqueues stable `Slack interaction` or `Slack shortcut` system events with context keys, and updates `extensions/slack/src/monitor/events/interactions.test.ts` plus `docs/channels/slack.md`. If maintainers want this to live outside bundled Slack, the missing prerequisite is a plugin-facing Slack shortcut receiver rather than closing the request as ClawHub-ready today.
