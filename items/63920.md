## Summary

Keep open. Current main still lacks a Slack shortcut listener for global `shortcut` and `message_action` payloads. Slack interaction registration covers block actions plus modal `view_submission`/`view_closed` only, docs and tests reflect that same limited surface, and the current plugin interactive API is button/select-oriented rather than a raw Slack Bolt shortcut subscription hook.

## Best Possible Solution

Implement this in `extensions/slack` as a Slack-owned interaction path: add a shortcut event module, register `ctx.app.shortcut` for global and message-action shortcuts, parse and sanitize the payloads, apply existing Slack mismatch/auth/session routing policy, enqueue structured system events with stable context keys, and update interaction tests plus `docs/channels/slack.md`. If maintainers want this handled by external plugins instead, first add a typed Slack shortcut extension API because the current plugin surfaces do not expose raw Bolt shortcut subscription.
