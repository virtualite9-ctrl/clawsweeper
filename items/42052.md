## Summary

Keep this PR open. Current main still treats a missing, non-slash backup output path as an explicit archive file path, while this PR is a plausible fix for linked bug #41830. The patch is stale against the current backup implementation layout and needs maintainer review, tests, and docs rather than closure.

## Best Possible Solution

Keep this PR open for maintainer review. Rebase the useful path-resolution idea onto `src/infra/backup-create.ts`, document the accepted `--output` contract in CLI help and `docs/cli/backup.md`, and add focused tests for non-existent extensionless directories, explicit archive filenames, `.tar`/custom suffix behavior, and non-ENOENT `fs.stat` errors. If that lands, close the linked bug from the merged implementation.
