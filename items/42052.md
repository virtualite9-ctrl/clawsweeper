## Summary

Keep this PR open. Current main still treats a missing, extensionless backup output path as an explicit archive filename, while this PR contains a plausible fix for the linked backup output bug. The patch is stale because backup creation logic moved from the command wrapper into the infra module, so the useful behavior needs to be rebased with tests and docs rather than closed.

## Best Possible Solution

Keep this PR open for maintainer review. Rebase the useful path-resolution behavior onto `src/infra/backup-create.ts`, document the accepted `--output` contract in CLI help and `docs/cli/backup.md`, and add focused tests for non-existent extensionless directories, explicit archive filenames, `.tar` and custom suffix behavior, and non-ENOENT `fs.stat` errors. If that lands, the linked backup bug should be closed by the merged implementation.
