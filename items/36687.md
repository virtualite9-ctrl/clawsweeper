## Summary

Keep #36687 open. Current main has guardrails and tests for the WebChat/UI stale external-route leakage symptom, but the issue’s specific 2026.3.2 update/reset claim for `session.dmScope` is still unresolved. The runtime, docs, security audit, and outbound/session-key helpers still treat omitted `session.dmScope` as `main`, while onboarding only writes `per-channel-peer` for unset local setup and preserves explicit values.

## Best Possible Solution

Keep the issue open for maintainer verification of the 2026.3.2 update/reset path. If an update can drop or ignore a previously isolated `session.dmScope`, add a targeted migration or doctor repair plus a regression test around config preservation. If omitted `dmScope` intentionally remains `main`, close only after an explicit maintainer product decision while keeping the shipped WebChat/UI stale-route guardrails.
