## Summary

Keep #36687 open. Current main has shipped guardrails for the WebChat/UI stale external-route leakage symptom, but the issue’s stronger claim that the 2026.3.2 update reset or ignored `session.dmScope` is still not proven fixed. Current code and docs still intentionally treat omitted `session.dmScope` as `main`; onboarding writes `per-channel-peer` only for unset local setup and preserves explicit values.

## Best Possible Solution

Keep the issue open for maintainer verification of the 2026.3.2 update/reset path. If an update can drop or ignore a previously isolated `session.dmScope`, add a targeted migration or doctor repair plus a regression test around config preservation. If omitted `dmScope` intentionally remains `main`, close only after explicit maintainer product judgment while keeping the shipped WebChat/UI stale-route guardrails.
