## Summary

Current main has real partial coverage: bootstrap files are capped, truncation is marked, `/context list` reports raw vs injected sizes, prompt warnings are injected after truncation, and doctor reports near-limit/truncated bootstrap files. The issue should stay open because the requested Control UI warning/enforcement path is still missing: the workspace file save method writes `MEMORY.md` content without checking `bootstrapMaxChars`, and the UI only shows a generic byte count in preview rather than a size-vs-limit warning or save block. The related draft PR #64874 appears to track more of this remaining work rather than prove it has shipped on main.

## Best Possible Solution

Keep the issue open and treat current main as partial groundwork. The likely maintainer path is to adapt or supersede PR #64874 with a focused core change: expose bootstrap file health to `/status` and the Control UI, show MEMORY.md size versus the active `agents.defaults.bootstrapMaxChars`, warn before the near-limit threshold, and decide explicitly whether oversized saves should be blocked or allowed with a clear warning.
