## Summary

Keep #12163 open. Current main has partial OpenAI TTS speed plumbing for config/Talk overrides and the OpenAI request body, but the reported per-reply directive path still cannot turn `[[tts:speed=1.5]]` into `providerOverrides.openai.speed`. The existing issue discussion already identified this gap, and it is still present at `e29d3516bf05591e6007bfb8ccdaea6d76194d41`.

## Best Possible Solution

Keep the issue open and implement the missing OpenAI directive handler in `extensions/openai/speech-provider.ts`: parse `speed` only when `policy.allowVoiceSettings` is enabled, validate OpenAI's documented `0.25..4.0` range, return `{ speed }`, and add focused tests proving both `[[tts:speed=1.5]]` with OpenAI active and `[[tts:provider=openai speed=1.5]]` reach the OpenAI request body. Preserve the existing config/Talk/request-body plumbing.

## Evidence

- **OpenAI directive parser lacks speed handling:** `parseDirectiveToken` handles OpenAI voice and model keys, then returns `handled: false` for every other key. There is no `speed` case and no `allowVoiceSettings` gate for OpenAI TTS directives.
  - file: [extensions/openai/speech-provider.ts:179](https://github.com/openclaw/openclaw/blob/e29d3516bf05591e6007bfb8ccdaea6d76194d41/extensions/openai/speech-provider.ts#L179)
  - command: `nl -ba extensions/openai/speech-provider.ts | sed -n '1,360p'`
  - sha: [e29d3516bf05](https://github.com/openclaw/openclaw/commit/e29d3516bf05591e6007bfb8ccdaea6d76194d41)
- **Directive routing depends on provider parser acknowledgement:** `parseTtsDirectives` only writes `overrides.providerOverrides[provider.id]` after a provider parser returns `handled` with overrides. Since OpenAI currently returns unhandled for `speed`, `[[tts:provider=openai speed=1.5]]` cannot become an OpenAI provider override.
  - file: [src/tts/directives.ts:313](https://github.com/openclaw/openclaw/blob/e29d3516bf05591e6007bfb8ccdaea6d76194d41/src/tts/directives.ts#L313)
  - command: `nl -ba src/tts/directives.ts | sed -n '240,360p'`
  - sha: [e29d3516bf05](https://github.com/openclaw/openclaw/commit/e29d3516bf05591e6007bfb8ccdaea6d76194d41)
- **Reply TTS consumes only resolved-provider overrides:** Automatic reply TTS parses directives with the active provider as `preferredProviderId`, and synthesis later passes only `params.overrides.providerOverrides[resolvedProvider.provider.id]` to the selected provider. A speed override stored under another provider, or not stored at all, will be ignored for OpenAI synthesis.
  - file: [extensions/speech-core/src/tts.ts:1418](https://github.com/openclaw/openclaw/blob/e29d3516bf05591e6007bfb8ccdaea6d76194d41/extensions/speech-core/src/tts.ts#L1418)
