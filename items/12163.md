## Summary

Keep #12163 open. Current main forwards OpenAI TTS speed from provider config and Talk/provider overrides into the OpenAI speech request, but the reported `[[tts:speed=1.5]]` directive path is still not implemented because the bundled OpenAI speech provider parser handles voice/model only. The prior review comment and the two related PR attempts both point to the same unresolved directive gap.

## Best Possible Solution

Keep the issue open. Implement OpenAI-owned speed directive handling in `extensions/openai/speech-provider.ts`: accept `speed` only when `policy.allowVoiceSettings` permits it, parse a finite number, validate OpenAI's documented `0.25..4.0` range, return `{ speed }` as provider overrides, and add focused tests for preferred OpenAI `[[tts:speed=1.5]]`, explicit `[[tts:provider=openai speed=1.5]]`, disabled voice settings, invalid values, and the final OpenAI request body.

## Evidence

- **OpenAI directive parser lacks speed handling:** `parseDirectiveToken` in the OpenAI speech provider handles `voice`/`openai_voice` and `model`/`openai_model`, then returns `{ handled: false }` for other keys. There is no `speed` case and no `allowVoiceSettings` gate for OpenAI speed directives.
