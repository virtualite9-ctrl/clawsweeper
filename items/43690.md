## Summary

Current main still lacks the Feishu table-limit card fallback requested here: the reply dispatcher and outbound card paths send cards without table-limit handling, while table conversion only runs on the plain-text path. Open PR #50429 explicitly closes this issue and remains unmerged, so the conservative action is to keep this issue open while maintainers review or supersede the PR work.

## Best Possible Solution

Keep this issue open until a Feishu plugin fix lands. The best implementation should detect Feishu card table-limit failures (230099 / 11310 / 'table number over limit') around the affected card send paths, retry that message as plain text using the existing markdown table conversion, preserve non-table card errors, and add focused regression tests. Review #50429 and the overlapping #70651, then land or supersede one canonical fix.
