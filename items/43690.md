## Summary

Keep this issue open. Current main still sends Feishu card replies and outbound card messages through interactive-card paths without a table-limit/plain-text fallback, while the existing plain-text path is the only path that applies markdown table conversion. The provided GitHub context also shows open implementation candidates, including PR #50429 with closing syntax for this issue, so this should remain open until a fix lands or maintainers consolidate the overlapping Feishu PRs.

## Best Possible Solution

Keep this issue open until a Feishu plugin fix lands. The best implementation path is to review and land PR #50429 or a superseding consolidated fix that detects Feishu card table-limit failures in both reply and outbound card paths, retries that message through sendMessageFeishu so existing markdown table conversion applies, preserves unrelated card errors, and adds focused regression tests. PR #70651 appears relevant for outbound behavior, while PR #42809 is a related native-table rendering enhancement rather than a complete fix for silent delivery failure.
