## Summary

Keep open. Current main still routes Feishu card replies and outbound card sends directly through interactive-card senders without a table-limit/plain-text fallback, while the plain text path is the only path that converts markdown tables. The issue also has an open implementation candidate, PR #50429, that explicitly links this issue for automatic closure, so the issue should remain open until that PR or a superseding fix lands.

## Best Possible Solution

Keep this issue open until a Feishu plugin fix lands. The best path is to review and either land PR #50429 or supersede it with a canonical fix that detects Feishu card table-limit failures in reply and outbound card paths, retries that message through sendMessageFeishu so existing markdown table conversion applies, preserves non-table card errors, and adds focused regression tests. PR #70651 can cover overlapping outbound behavior, while PR #42809 is a related native-table rendering enhancement rather than a complete fix for silent delivery failure.
