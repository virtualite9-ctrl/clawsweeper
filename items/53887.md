## Summary

Current `main` does not implement Feishu/Lark lifecycle status reactions. Telegram is wired to `createStatusReactionController`, but the Feishu channel still only adds/removes a single `Typing` reaction and the docs/config help still list status reactions as Slack/Discord/Telegram-only.

## Evidence

- **Telegram has lifecycle status reaction wiring:** `extensions/telegram/src/bot-message-context.ts` creates and uses `createStatusReactionController`, including `setQueued()` replacing the ack reaction.
  - file: [extensions/telegram/src/bot-message-context.ts:475](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/extensions/telegram/src/bot-message-context.ts#L475)
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Feishu reply flow only manages a binary typing reaction:** `extensions/feishu/src/reply-dispatcher.ts` starts/stops `addTypingIndicator` and `removeTypingIndicator`; there is no status reaction controller hookup in the inbound reply lifecycle.
  - file: [extensions/feishu/src/reply-dispatcher.ts:117](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/extensions/feishu/src/reply-dispatcher.ts#L117)
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Feishu typing implementation is hard-coded to `Typing`:** `extensions/feishu/src/typing.ts` defines `TYPING_EMOJI = "Typing"` and uses that single emoji for create/delete calls.
  - file: [extensions/feishu/src/typing.ts:9](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/extensions/feishu/src/typing.ts#L9)
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Feishu reaction primitives and emoji enum exist but are separate utilities:** `extensions/feishu/src/reactions.ts` exports reaction helpers and a broader emoji enum (`THINKING`, `FIRE`, `CHECK`, `CROSS`, etc.), which supports the request’s feasibility but does not itself wire lifecycle behavior.
  - file: [extensions/feishu/src/reactions.ts:129](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/extensions/feishu/src/reactions.ts#L129)
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)

  - file: [docs/gateway/config-agents.md:1237](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/docs/gateway/config-agents.md#L1237)
