## Summary

Keep this PR open. Current main still lacks an EasyRunner deployment guide, and provider-specific hosting docs remain an established OpenClaw docs surface. The PR is docs-only and not a security/supply-chain code-execution change, but the proposed documentation is stale against the current Install > Hosting navigation and contains security-sensitive gateway guidance that needs contributor revision before it can merge.

## Best Possible Solution

Keep this PR open and ask for a revised, rebased docs contribution. The best path is to move or link EasyRunner under the current Install > Hosting docs surface, align the compose/config with current Podman/gateway behavior, set the gateway bind/reverse-proxy settings explicitly and safely, replace normal-use `dangerouslyDisableDeviceAuth` guidance with token/password or trusted-proxy auth plus explicit `gateway.controlUi.allowedOrigins`, avoid token-printing troubleshooting commands, include rebuild/update steps for `localhost/openclaw:latest`, and verify the EasyRunner commands on a real deployment before merge.
