## Summary

Keep this PR open. Current main still appears to pass malformed `required: null` through the shared non-Gemini tool-schema path, and the Gemini cleaner still only handles empty `required` arrays rather than non-array `required` values. The central Bailian/strict-provider bug is not implemented on main; the patch needs rebasing onto the newer `normalizeToolParameterSchema` structure.

## Best Possible Solution

Keep this PR open and ask for a rebase onto current main. The best fix is to strip own non-array `required` fields in the shared `normalizeToolParameterSchema` path before provider-specific cleaning, preserve valid `required` arrays, and add focused regression tests for the strict/OpenAI-compatible top-level path plus nested Gemini schemas. Treat the Aisle prototype-pollution note as a hardening follow-up unless the rebased patch already touches that copy loop.
