## Summary

Keep this PR open. Current main still lets an own top-level `required: null` survive the shared non-Gemini tool schema normalization path, and the Gemini cleaner still only strips empty `required` arrays, not malformed non-array values. The PR remains a focused implementation candidate, but it needs rebasing onto the current `normalizeToolParameterSchema` layout.

## Best Possible Solution

Keep this PR open and request a rebase onto current main. The best implementation is to strip own non-array `required` fields in the shared `normalizeToolParameterSchema` path before provider-specific cleaning, preserve valid `required` arrays, and add focused regression tests for top-level strict OpenAI-compatible providers plus nested Gemini schemas. Treat the Aisle prototype-pollution note as hardening unless the rebased patch already touches that copy loop.
