## Summary

Current main and the released v2026.4.22 code no longer collapse this Control UI failure into a generic `disconnected (4008): connect failed`; the gateway returns a structured insecure-Control-UI/device-identity error, the UI maps it to actionable guidance, and the docs now explain the supported HTTP/HTTPS behavior.

## Close Comment

Closing this as implemented on current main. Codex review checked current main `b0d2003` and the shipped `v2026.4.22` code: this Control UI path no longer collapses into only `disconnected (4008): connect failed`. The gateway now returns a structured `CONTROL_UI_DEVICE_IDENTITY_REQUIRED` error for insecure-http Control UI sessions (`src/gateway/server/ws-connection/message-handler.ts:653`), the UI maps that to `device identity required (use HTTPS/localhost or allow insecure auth explicitly)` (`ui/src/ui/connect-error.ts:65`), and the overview/docs now point to the supported fixes (`ui/src/ui/views/overview.ts:191`, `docs/web/control-ui.md:254`). That matches the symptom here, where the gateway itself was reachable but the browser session still disconnected with 4008. If this still reproduces on `v2026.4.22` or newer, please open a new issue with the exact dashboard URL/origin you opened, whether it was HTTP or HTTPS, and the relevant `gateway.controlUi` settings.
