## Summary

Keep this issue open. Current main still matches the reported behavior: `embedInterval` controls only the dedicated periodic embed timer, while first-run and forced QMD update paths can still spawn `qmd embed`. The issue discussion also points to this as the broader canonical tracker after narrower related QMD embed reports were closed or superseded.

## Best Possible Solution

Keep this issue open and implement a documented QMD embed-disable path in `extensions/memory-core/src/memory/qmd-manager.ts` and the public memory config schema/help/docs. The fix should prevent `qmd embed` from spawning for boot, first-run, manual/forced sync, queued forced sync, session-start/post-compaction, and periodic maintenance when disabled, while preserving `qmd update` and BM25 refresh behavior. Add focused QMD manager tests covering disabled embeds across forced and non-forced paths.
