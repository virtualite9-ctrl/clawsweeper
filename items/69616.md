## Summary

Keep #69616 open. Current main has an adjacent shipped fix for the #68451/#68517 stale-gateway cleanup loop, and a contributor negative-reproduced the symptom on OpenClaw 2026.4.21 in a non-root Ubuntu setup. But #69616 reports a fresh Ubuntu 24.04.4/root startup busy-loop on 2026.4.15, including direct `openclaw gateway run` hangs and a `getpid`/`capget`/`getuid` strace pattern. That is not strongly proven to be the same sidecar/ancestor-PID loop, and current main still lacks a general root-run guard while root/systemd gateway race reports remain open.

## Best Possible Solution

Keep the issue open and ask for the full `strace -f` with process names, `journalctl`/gateway logs, `openclaw gateway status --deep`, install method, whether the command/service runs as root, redacted config, and enabled plugin list. If that proves the sidecar stale-cleanup topology, close/link it to the shipped #68451/#68517 fix; if it proves a root/systemd ownership or dual-service race, route it to the root-run guard/startup hardening work instead.
