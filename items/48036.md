## Summary

Current main, and the latest release tag v2026.4.24, already keep Control UI WebSocket sessions active with a 30s gateway tick keepalive and recoverable reconnect behavior. The linked Feishu PR #46472 is channel-specific and not the Control UI gateway path.

## Best Possible Solution

Keep the shipped 30s gateway tick keepalive and Control UI reconnect behavior. If more UX detail is desired, track that separately as a narrower Control UI state/diagnostics enhancement, not as this idle heartbeat timeout bug.
