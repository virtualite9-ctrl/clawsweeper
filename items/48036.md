## Summary

Current `main` does not show evidence that this issue was implemented or obsoleted. The gateway websocket server still has no explicit ping/pong heartbeat or keepalive configuration, and the Control UI still turns ordinary websocket closes into visible gateway error state that is rendered in the overview and attention surfaces. That matches the reported symptom closely enough that conservative cleanup should keep this issue open.
