## Summary

Keep this PR open. Current main still applies only the first matching provider's `wrapStreamFn`, while the provider contract supports `hookAliases` and stream wrappers, so the PR's central bug fix remains relevant. The patch likely needs to be ported from the older `src/plugins/provider-runtime.ts` location into the current `src/plugins/provider-hook-runtime.ts` implementation.

## Best Possible Solution

Keep this PR open and review it as an implementation candidate. Port the chaining behavior into `src/plugins/provider-hook-runtime.ts`, preserve provider-id filtering and stable dedupe, add the regression test for primary-plus-hookAlias composition and unrelated-provider skipping, then run the normal changed plugin/core verification before landing.
