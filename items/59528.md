## Summary

Keep open. Current `main` still has a CLI bootstrap gap for plugin-provided sandbox backends: `openclaw sandbox list` does not load plugins, so the OpenShell sandbox manager is absent and the command falls back to `running: false`. That matches the reporter's symptom that the OpenShell sandbox can exist while OpenClaw still shows it as stopped.

## Evidence

- **Sandbox CLI never opts into plugin loading:** `cliCommandCatalog` sets the default command policy to `loadPlugins: "never"` and there is no `sandbox` override in the catalog, so `openclaw sandbox ...` inherits the no-plugin-load policy.
  - file: [src/cli/command-catalog.ts:34](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/src/cli/command-catalog.ts#L34)
  - command: `nl -ba src/cli/command-catalog.ts | sed -n '34,150p'`
  - sha: [ec3dbd22a4de](https://github.com/openclaw/openclaw/commit/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46)
- **Pre-action bootstrap uses that startup policy as-is:** CLI pre-action resolves the startup policy from the command path and calls `ensureCliExecutionBootstrap(...)` without any sandbox-specific override.
  - file: [src/cli/program/preaction.ts:72](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/src/cli/program/preaction.ts#L72)
  - command: `nl -ba src/cli/program/preaction.ts | sed -n '64,99p'`
  - sha: [ec3dbd22a4de](https://github.com/openclaw/openclaw/commit/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46)
- **Bootstrap skips plugin registry loading when `loadPlugins` is false:** `ensureCliCommandBootstrap` returns before `ensureCliPluginRegistryLoaded(...)` when `params.loadPlugins` is falsy, so plugin-owned sandbox managers are never registered for the sandbox CLI path.
  - file: [src/cli/command-bootstrap.ts:31](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/src/cli/command-bootstrap.ts#L31)
  - command: `nl -ba src/cli/command-bootstrap.ts | sed -n '1,220p'`
  - sha: [ec3dbd22a4de](https://github.com/openclaw/openclaw/commit/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46)
- **Sandbox list marks unknown backends as stopped:** `listSandboxContainers()` calls `getSandboxBackendManager(backendId)` and, if that returns null, pushes the entry with `running: false` immediately. That directly explains an OpenShell runtime showing as stopped when its plugin manager was never loaded.
  - file: [src/agents/sandbox/manage.ts:42](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/src/agents/sandbox/manage.ts#L42)
  - command: `nl -ba src/agents/sandbox/manage.ts | sed -n '35,66p'`
  - sha: [ec3dbd22a4de](https://github.com/openclaw/openclaw/commit/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46)
- **OpenShell manager is only registered by the plugin runtime:** The OpenShell plugin registers its sandbox backend and manager via `registerSandboxBackend("openshell", ...)` only during full plugin registration. If the CLI does not load plugins, there is no OpenShell manager to answer status checks.
  - file: [extensions/openshell/index.ts:14](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/extensions/openshell/index.ts#L14)
  - command: `nl -ba extensions/openshell/index.ts | sed -n '1,40p'`
  - sha: [ec3dbd22a4de](https://github.com/openclaw/openclaw/commit/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46)
- **OpenShell manager itself would report running when invoked:** The current OpenShell manager implementation sets `running: result.code === 0` from `openshell sandbox get <name>`, and its unit test expects `running: true` on a successful CLI result. The failure mode is therefore upstream in CLI/plugin loading, not in the manager's status mapping alone.
  - file: [extensions/openshell/src/backend.ts:64](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/extensions/openshell/src/backend.ts#L64)
  - command: `nl -ba extensions/openshell/src/backend.ts | sed -n '60,79p'; nl -ba extensions/openshell/src/openshell-core.test.ts | sed -n '95,149p'`
  - sha: [ec3dbd22a4de](https://github.com/openclaw/openclaw/commit/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46)
