## Summary

Keep open. Current main still lacks the PR's core behavior: `socket.drain` is not registered or documented as a gateway event, unknown non-plugin events fail the broadcast scope guard, session teardown broadcasts only target `sessions.subscribe` connections, and the current message-subscription tests cover transcript delivery but not drain/eviction delivery. The related #65539 is the broad closed predecessor, while #69822 is the focused successor for #63374, so it should not be cleanup-closed as duplicate or implemented.

## Best Possible Solution

Keep the PR open for maintainer review. The likely product path is a focused gateway change that registers `socket.drain` in event discovery and scope guards, exposes a message-subscriber lookup on `GatewayRequestContext`, emits a targeted teardown drain for delete/reset/new to the union of event and message subscribers, preserves normal `dropIfSlow` behavior outside teardown, and adds regression coverage for message-only subscribers plus the browser-maintenance test mock if needed.
