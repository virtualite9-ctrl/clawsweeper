## Summary

Keep #72114 open. Current main still has adjacent bundled-runtime-deps hardening, but I do not see a source-backed fix for the reported update-driven restart race where a bundled runtime-deps npm install is active while the gateway is restarted or killed by the service manager. The linked #72058 remains a broader open runtime-deps regression report, but it does not clearly supersede this narrower shutdown/SIGKILL failure mode.

## Best Possible Solution

Keep #72114 open as a focused core runtime-deps/update restart-race report. The likely fix should make bundled runtime-deps staging visible to restart/shutdown deferral, prevent update/service-manager restarts from interrupting an active staging npm subprocess, and add regression coverage for update-driven gateway restart while browser bundled runtime deps are being installed. Maintainers can later link or close it under #72058 only if that broad tracker explicitly owns the shutdown/SIGKILL race.
