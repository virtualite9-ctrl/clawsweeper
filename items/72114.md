## Summary

Keep #72114 open. Current main still has adjacent bundled-runtime-deps hardening, but I do not see a source-backed fix for the specific cold-cache staging failure plus update/service-manager shutdown race. The related #72058 tracker and #72084 PR remain open in the supplied context and do not clearly supersede this narrower SIGTERM/SIGKILL failure mode.

## Best Possible Solution

Keep #72114 open as a focused runtime-deps/update restart-race report. The likely core fix is to make bundled runtime-deps staging observable as pending restart/shutdown work, avoid service-manager restarts while an npm staging subprocess is active, harden interrupted staging cleanup/retry, and add regression coverage for cold-cache browser runtime-deps staging during an update-driven gateway restart. If maintainers want #72058 to own this, explicitly mark #72114 as a duplicate only after that tracker names the shutdown/SIGKILL failure mode.
