## Summary

Keep #59744 open. Current main still drops the Discord attachment filename when persisting downloaded inbound attachments: Discord passes attachment.filename only as a fetch hint, then calls saveMediaBuffer without the originalFilename argument, so the media store will generate a UUID-only saved name. The linked PR #59783 is an open, unmerged implementation attempt that directly tracks this bug, but it has not shipped on main.

## Best Possible Solution

Keep the issue open until PR #59783 or an equivalent patch lands on main. The fix should pass Discord attachment.filename, or fetched.fileName when appropriate, into saveMediaBuffer as originalFilename for inbound attachments and sticker assets, then update Discord media tests to assert the fifth argument and preserve the original filename in the resulting MediaPath.
