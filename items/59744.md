## Summary

Current `main` still drops Discord attachment filenames on the save path. `extensions/discord/src/monitor/message-utils.ts` uses `attachment.filename` only as a download hint, then calls `saveMediaBuffer` without the `originalFilename` argument, so stored inbound files remain UUID-based. The Discord tests lock in that 4-argument call. By contrast, Telegram and WhatsApp already pass the original filename into the same helper. The relevant Discord files are unchanged between the latest release `v2026.4.23` and current `main`, so this issue is not implemented on `main` and should stay open.

## Evidence

  - file: [extensions/discord/src/monitor/message-utils.ts:385](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/extensions/discord/src/monitor/message-utils.ts#L385)
