## Summary

Keep #38626 open. Current main has useful scalar background-task/subagent tracking, completion delivery, task cancellation, and subagent list/kill/steer controls, but it still does not implement the requested durable per-subagent lifecycle timeline, persisted tool/warning/error transitions, artifact-pointer completion summaries, first-failure/recommended-action diagnostics, or idempotent audit-logged cancel/steer/resume supervision surface. Related #62478 is complementary background-spawn work, not a canonical replacement.

## Best Possible Solution

Keep the issue open unless a maintainer intentionally narrows it. The best implementation path is to add a durable per-run subagent/task event ledger for spawn, queued/running, tool start/end, warning/error, completion, and supervision actions; persist it in task/subagent storage; expose it through subagent info/list, the subagents tool, openclaw tasks show, gateway/plugin runtime views, and docs; attach artifact pointers and terminal diagnostics; and make cancel, steer, and resume idempotent with auditable action records. Treat closed #62478 as complementary background-spawn work, not the canonical tracker for this observability scope.
