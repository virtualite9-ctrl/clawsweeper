## Summary

Keep open. Current `main` does not implement per-agent ACPX plugin defaults for `model` or `reasoning_effort`/other config options. The bundled ACPX plugin exposes only per-agent command overrides, while ACP runtime model changes are still explicit per-session controls or per-spawn overrides.

## Evidence

- **ACPX plugin config schema lacks per-agent model/config defaults:** `AcpxPluginConfig` and `AcpxPluginConfigSchema` expose `agents` only as `Record<string, { command: string }>`; there is no `agentDefaults`, `model`, `configOptions`, or `reasoning_effort` field in the plugin config contract.
  - file: [extensions/acpx/src/config-schema.ts:26](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/extensions/acpx/src/config-schema.ts#L26)
  - command: `nl -ba extensions/acpx/src/config-schema.ts | sed -n '1,220p'`
  - sha: [ec3dbd22a4de](https://github.com/openclaw/openclaw/commit/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46)
- **Resolved ACPX config only preserves command overrides:** `resolveAcpxPluginConfig()` converts `normalized.agents` into a lowercased map of command strings only, proving the runtime consumes per-agent command overrides but no per-agent runtime-option defaults.
  - file: [extensions/acpx/src/config.ts:234](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/extensions/acpx/src/config.ts#L234)

- **User-facing ACPX manifest also exposes only command overrides:** The bundled plugin manifest advertises `agents` as objects with a required `command` field and no model/config-option keys, so the current shipped config surface does not support the requested feature.
  - file: [extensions/acpx/openclaw.plugin.json:77](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/extensions/acpx/openclaw.plugin.json#L77)

## Risks / Open Questions

- The underlying `acpx` package may have its own upstream capabilities or future config support, but OpenClaw `main` does not currently expose them through the bundled ACPX plugin config.
- There is a partial workaround today via explicit spawn-time `model` and manual `/acp set` or `/acp model`, so maintainers may want to scope this as an enhancement rather than a regression.
