## Summary

Current `main` already repairs this class of Linux user-systemd drift: doctor audits missing service `PATH`, rebuilds the gateway install plan with a minimal managed `PATH`, rewrites the unit, and the systemd reader now merges both `Environment=` and `EnvironmentFile=` values. The same implementation is present in the shipped `v2026.4.23` release commit.

## Evidence

- **Doctor repair rewrites audited gateway service config:** `maybeRepairGatewayServiceConfig()` audits the loaded service, reports config issues, then on repair calls `service.install()` (or `service.stage()` during update repair mode) with a freshly rebuilt install plan.
  - file: [src/commands/doctor-gateway-services.ts:239](https://github.com/openclaw/openclaw/blob/5b8bd6371c666a5de43c82305e0315771e8e49d5/src/commands/doctor-gateway-services.ts#L239)
