## Summary

Keep this issue open. Current main still has the Control UI model-switch/send ordering race: the dropdown optimistically updates the local selected model and awaits `sessions.patch`, but no pending model-switch state is stored, the send path does not await a model-switch promise, and `chat.send` has no per-send model override to make the immediate next turn independent of the session patch finishing first.

## Best Possible Solution

Implement the Control UI ordering guard proposed in the report: store the pending model-switch operation in UI state, disable the model picker and send while it is pending, make `handleSendChat` await that operation before `chat.send`, roll back and avoid sending if `sessions.patch` fails, and add a regression test where the dropdown change leaves `sessions.patch` unresolved while send is attempted immediately.
