## Summary

Keep this issue open. Current main still updates the Control UI model picker optimistically and fires `sessions.patch`, but `handleSendChat()` has no pending model-switch barrier before it calls `chat.send`. The related closed model-picker fixes cover provider-qualified values and stale display state, not this send-ordering race.

## Best Possible Solution

Implement the issue's proposed Control UI ordering guard: add explicit pending model-switch state or a stored promise, disable both the model picker and send while it is pending, have `handleSendChat()` await that promise before calling `chat.send`, roll back and avoid sending if `sessions.patch` fails, and add a regression test that dispatches a model change with unresolved `sessions.patch` followed immediately by send.
