## Summary

Current main still lacks the ordering guard requested here. The Control UI model dropdown optimistically starts an async `sessions.patch`, but the send path has no shared pending model-switch state to await, the send button is not disabled while that patch is in flight, and `chat.send` cannot carry a one-turn model override.

## Best Possible Solution

Keep this issue open. Implement a Control UI model-switch barrier: store the pending `sessions.patch` operation in shared chat/app state, disable the model picker and send/queue controls while it is pending, have `handleSendChat` await that operation before `chat.send`, roll back the picker and suppress send on patch failure, and add a regression test that leaves the patch promise unresolved while an immediate send is attempted.
