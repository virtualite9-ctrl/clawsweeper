## Summary

Keep open. Current `main` still lacks a Windows-specific CLI stack/headroom mitigation in the respawn path, the reported heavy maintenance commands still funnel through the same lazy maintenance registration module, and the startup-memory regression coverage does not exercise Windows or the respawned startup path. The issue also targets behavior the docs still describe as supported on native Windows.

## Evidence
