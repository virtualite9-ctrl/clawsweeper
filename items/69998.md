## Summary

Keep #69998 open. Current main still returns the updated `skills.entries.<skillKey>` object directly from `skills.update`, so the credential-echo bug described by #66769 remains unshipped. The PR is a narrow active fix, but before merge it should also address the redaction-sentinel round-trip concern raised in review.

## Best Possible Solution

Keep the PR open and refine it: redact or omit sensitive fields in the `skills.update` success response, add the regression test for `apiKey` plus secret-named `env` values, and handle `__OPENCLAW_REDACTED__` on subsequent `skills.update` writes by restoring or preserving existing secrets before writing config.
