## Summary

Keep open. Current main still returns the updated skills config object directly from `skills.update`, so the credential-echo bug from #66769 is not implemented on main. #69998 is a focused fix attempt and the related PRs listed in the discussion (#66928 and #66986) are closed unmerged, so they do not supersede this PR. The remaining maintainer work is to address the sentinel round-trip concern before merge.

## Best Possible Solution

Keep #69998 open and have the PR either add `skills.update` restore/keep-existing handling for redacted sentinel values or change the success payload to avoid returning writable secret-bearing config. After that, merge with the added regression test and close #66769.
