## Summary

Keep #69998 open. Current main still returns the updated `skills.update` config object directly, so the credential-echo bug from #66769 remains unfixed on main. The PR is still useful focused implementation work, but it should address the redacted-sentinel round-trip concern before landing.

## Best Possible Solution

Keep #69998 open. The best fix is to stop `skills.update` from returning plaintext secrets while also preserving write-side behavior: either treat redacted sentinels in incoming `apiKey`/`env` as keep-existing values, using the existing redaction contract where appropriate, or change the success payload to return only non-sensitive confirmation metadata. Keep the regression test that proves plaintext is persisted but the response is redacted, then close #66769 once merged.
