## Summary

Keep #69998 open. Current main still returns the config-mode `skills.update` entry directly after writing it, so the reported plaintext `apiKey`/secret env response leak is not implemented on main. The related duplicate PRs are closed unmerged, and this PR remains the canonical focused fix for open issue #66769.

## Best Possible Solution

Keep #69998 open and review it as the likely fix for #66769. The best code path is a focused `skills.update` response-only redaction using `redactConfigObject()`, with regression coverage proving `apiKey` and secret-like env keys are redacted in the response while full values are still persisted; resolve or explicitly accept the sentinel round-trip concern before merge.
