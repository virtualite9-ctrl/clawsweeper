## Summary

Keep this PR open. Current main still has the `skills.update` credential-echo bug, and the related duplicate PRs are closed unmerged, so this PR is still a useful implementation candidate. It should not be closed as obsolete; the remaining maintainer work is to resolve the redaction-sentinel round-trip concern and clean up the changelog diff before merge.

## Best Possible Solution

Keep this PR open. The best path is to revise it so `skills.update` either returns only non-sensitive confirmation metadata or redacts the response while also treating redacted sentinels as keep-existing values on write, with regression tests for both response redaction and sentinel round-tripping. After that, clean the changelog diff and merge to close #66769.
