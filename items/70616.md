## Summary

Keep #70616 open. Current main still has the reported memory-core config contract gap: the manifest schema rejects `dreaming.execution` and per-phase `execution`, while the host resolver already parses execution settings. The remaining model/fallback behavior is not fully wired either: managed dreaming cron payloads do not carry model/fallback overrides, dream narrative subagent calls do not pass a model, and the plugin subagent runtime type has `model` but no `fallbacks` parameter. This is actionable in OpenClaw memory-core/core config, not a ClawHub/plugin-only request.

## Best Possible Solution

Keep the issue open and fix it in memory-core/core config: decide the public shape for `dreaming.execution` and phase overrides, update `extensions/memory-core/openclaw.plugin.json`, generated config/docs/tests, and wire resolved execution model settings into the managed dreaming cron and/or narrative subagent path. If fallback chains are required per dreaming run, either map them through the cron agentTurn fallback support or add a typed plugin subagent fallback override instead of leaving schema-only acceptance.
