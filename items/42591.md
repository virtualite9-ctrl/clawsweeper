## Summary

Keep open. Current main still ships the macOS/Linux installer as one large `scripts/install.sh` entrypoint, with downloader, gum/UI, platform detection, argument parsing, npm/git install flow, doctor/onboarding, and final execution all inline. The requested modular `scripts/install/main.sh` plus `lib/`/`vendor/` structure is not present, and the issue is concrete and recent enough that stale/insufficient-info closure does not apply.

## Best Possible Solution

Keep this issue open for maintainer-reviewed installer cleanup. A good implementation would preserve `scripts/install.sh` as the public curl-pipe entrypoint while extracting narrowly testable shell libraries for UI, download, platform detection, npm/git install, and build-tool logic, then update installer tests and docs to cover the new layout.
