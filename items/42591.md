## Summary

Current main still ships the macOS/Linux installer as a single large scripts/install.sh entrypoint. The requested scripts/install/main.sh plus lib/vendor split is not present, and the issue is a concrete, recent maintainer cleanup request rather than a stale close candidate.

## Best Possible Solution

Keep this issue open for maintainer-reviewed installer cleanup. The best path is to preserve scripts/install.sh as the stable public curl-pipe entrypoint while extracting narrowly testable shell modules for UI/logging, download/checksum, platform detection, gum bootstrap, npm/git install paths, and build-tool handling, then update installer docs and tests around the new layout.
