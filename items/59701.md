## Summary

Current `main` no longer matches the report's exact post-reset state because exec-approval defaults are now permissive (`security: "full"`, `ask: "off"`, `askFallback: "full"`) and docs now explain explicit non-interactive setup for cron/background runs. But I did not find strong evidence of a shipped fix for the core complaint about upgrade-time approvals-file resets: the code still falls back to a fresh normalized approvals file when the existing file is missing, invalid JSON, or wrong-version, then writes it back, and I did not find a broader migration or user-facing reset warning beyond the narrow `agents.default -> agents.main` migration.
