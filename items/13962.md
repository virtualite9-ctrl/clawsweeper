## Summary

Keep #13962 open. Current main partially covers the context side by storing skipped WhatsApp group messages and injecting that pending history into the next triggered reply, and route bindings can choose an agent for a whole group/chat. It still does not implement mention-state-aware model routing, background-model processing for unmentioned group chatter, or a per-group mention context window such as `mentionContextMessages`.

## Best Possible Solution

Keep the issue open until OpenClaw implements mention-state-aware routing plus configurable per-group mention context sizing, or exposes a clear pre-gating plugin/API path that lets trusted plugins process unmentioned group messages with a background model. Likely touch points are WhatsApp group gating, route/model selection, WhatsApp config types/schema/docs, and tests covering skipped unmentioned messages, triggered mentions, pending-history injection, and per-group sizing.
