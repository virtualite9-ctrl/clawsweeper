## Summary

Keep #13962 open. Current main still has pending-history injection for mention-gated WhatsApp groups, and it has channel/group model overrides, but it does not implement mention-state routing, background model processing for unmentioned group messages, or per-group mention context sizing. The existing maintainer-posted Codex comment reached the same keep-open conclusion, and current main still matches it.

## Best Possible Solution

Keep the issue open until OpenClaw implements mention-state-aware routing plus configurable group mention context sizing, or exposes a clear pre-gating plugin/API path that lets trusted plugins process unmentioned group messages with a background model before the main mention-triggered reply. The likely code surface is WhatsApp/group channel gating, model selection, config schema/types, docs, and targeted gating/dispatch tests.
