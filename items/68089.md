## Summary

Keep this PR open. Current main has not applied the Mattermost-local fix proposed here: route registration still resolves `slash-route-api.js` through the bundled-entry sync loader, while activation/deactivation still import the same `slash-state.js` surface through the runtime graph. The current loader now has a native-require fast path for compiled dist JavaScript, so maintainers should verify whether packaged installs are already mitigated before choosing the final shape, but the PR remains a focused, plausible fix candidate rather than obsolete or safely closeable. #69038 is an open alternate implementation, not a merged or maintainer-canonical superseder.

## Best Possible Solution

Keep this PR open for maintainer review or consolidation with #69038. The best code path is to land one focused Mattermost slash-command fix that guarantees route registration and monitor activation share the same account state across supported loader paths, adds a targeted dual-load regression test, and explicitly resolves the import-graph tradeoff.
