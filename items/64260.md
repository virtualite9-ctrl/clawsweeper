## Summary

Keep this issue open. Current main still exposes `llm_output` with provider/model identifiers, assistant text, and usage totals only; it does not include model context window, max output tokens, effective context budget, or utilization. The existing model/runtime metadata needed to implement this exists elsewhere, and the related open issues #64268 and #64327 are complementary rather than shipped replacements.

## Best Possible Solution

Keep the issue open and implement a focused plugin hook contract change: extend `PluginHookLlmOutputEvent` with optional native model `contextWindow`, model `maxTokens`, and resolved effective context budget such as `contextTokens`; compute utilization from usage totals only where reliable. Populate the fields in the PI embedded runner, CLI harness, and Codex harness, add hook emission/contract tests, and update the public plugin hooks docs. Coordinate the field names with #64327 and keep #64268 separate for per-call emission cadence.
