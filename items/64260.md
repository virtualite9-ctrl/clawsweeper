## Summary

Keep this issue open. Current main still exposes `llm_output` with provider/model identifiers, resolved/harness metadata, assistant text, and usage counts only; it does not expose model context window, max output tokens, effective context budget, or utilization. The open related issues #64268 and #64327 are complementary rather than shipped replacements, and no open PR was found that closes this issue.

## Best Possible Solution

Keep this issue open and implement a focused plugin hook contract update: add optional `contextWindow`, `maxTokens`, and effective context budget such as `contextTokens` to `PluginHookLlmOutputEvent`; populate them from PI embedded runner, CLI harness, and Codex harness; add hook emission/contract tests; and update public plugin hooks docs. Treat utilization as optional or derived only where the numerator is semantically reliable, and coordinate naming with #64327.

## Risks / Open Questions

- The hook contract change should be additive and optional so existing third-party plugins continue to typecheck and run.

- Computed utilization should be defined carefully because `usage.total` can include output, cache, and aggregate loop totals that may not equal live prompt context usage.
