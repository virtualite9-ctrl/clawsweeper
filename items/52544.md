## Summary

Keep open. Current `main` does not implement the locale-artifact architecture from PR #52544: Control UI locales are still a fixed in-repo lazy-loaded list, docs i18n is still routed through the separate `openclaw/docs` publish repo, and there is no `locales` CLI or manifest/package support for `resource-only` locale artifacts on `main`.

## Evidence

- **Control UI still uses built-in in-repo locale modules:** `ui/src/i18n/lib/registry.ts` defines a static `LAZY_LOCALES` list and loads translations from `ui/src/i18n/locales/*.ts`; this is the old in-repo model, not artifact discovery/registry.
  - file: [ui/src/i18n/lib/registry.ts:13](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/ui/src/i18n/lib/registry.ts#L13)
- **Persisted locale must be one of the static supported locales:** Startup only applies a saved locale if `isSupportedLocale(...)` passes, which blocks arbitrary plugin-provided locale ids.
  - file: [ui/src/ui/app.ts:144](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/ui/src/ui/app.ts#L144)
- **Settings loader also restricts locale to the fixed supported list:** `loadSettings()` keeps `parsed.locale` only when `isSupportedLocale(parsed.locale)` is true, matching the fixed built-in locale model.
  - file: [ui/src/ui/storage.ts:266](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/ui/src/ui/storage.ts#L266)
- **Docs i18n policy still points to the separate publish repo:** Repo docs policy says foreign-language docs are not maintained here and the generated publish output lives in `openclaw/docs`, which contradicts the PR's proposed installable locale-artifact path in core.
  - file: [docs/AGENTS.md:21](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/docs/AGENTS.md#L21)
- **Docs i18n README still documents publish-repo translation flow:** `docs/.i18n/README.md` says generated locale trees live in `openclaw/docs` and describes the sync/translate workflow there, not locale artifacts installed through OpenClaw plugins.
  - file: [docs/.i18n/README.md:5](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/docs/.i18n/README.md#L5)
- **No locale-artifact CLI or manifest substrate on main:** Searches for `registerLocalesCli`, `docs:sync-locales`, and `localization` support in `src/cli`, `src/plugins`, `src/gateway`, `docs/plugins`, and `package.json` returned no matches relevant to the PR's proposed `locales` CLI, `resource-only` package mode, or manifest `localization` block.
  - command: `rg -n 'registerLocalesCli|docs:sync-locales|localization|resource-only|packageMode' src/cli src/plugins src/gateway ui docs/plugins package.json extensions --glob '!extensions/diffs/assets/viewer-runtime.js'`
- **Checkout stayed clean:** Repository worktree is clean after inspection.
  - command: `git status --short`
