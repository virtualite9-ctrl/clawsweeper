## Summary

Keep this issue open. Current main has useful partial Codex OAuth and usage support, including automatic OAuth refresh, Codex WHAM usage-window parsing, Control UI quota hints, and doctor guidance for permanent refresh-token failures. It still does not implement the requested native diagnostic state machine, does not expose `quota_wait` versus `reauth_required` through `models.authStatus`, and does not provide a bounded Control UI or host-side brokered reauth action.

## Best Possible Solution

Keep this open as a core/provider plus Control UI product issue. Add an additive diagnostic contract on the existing provider auth/usage path with states such as `ok`, `expiring_soon`, `quota_wait`, and `reauth_required`; keep the current Codex WHAM usage parsing and automatic OAuth refresh behavior; display Codex 5-hour and weekly renewal details in local time; and design a narrow authorized broker/adapter execution API that can only run when diagnosis is `reauth_required`. Cover 401/403 refresh failure, 429/quota-window behavior, Control UI rendering/actions, and broker authorization boundaries with focused tests.
