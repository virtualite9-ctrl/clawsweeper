## Summary

Keep #59526 open. Current main supports manual per-session auth-profile pinning with `/model <model>@<profileId>` and shows an auth label in some `/models <provider>` headers, but the interactive `/models` button/menu surfaces still model only. They do not enumerate GPT/Claude provider accounts/profiles or let the user choose one from the picker, and at least the Telegram picker applies model selection without a profile override, which clears any existing auth-profile override.

## Best Possible Solution

Keep the issue open and implement it as a core picker/API enhancement: extend the `/models` provider data and channel command adapter contract to expose eligible auth profiles/accounts and the active session profile for each provider, render profile/account selectors in the Telegram/Discord/Mattermost model pickers, and persist selections via `applyModelOverrideToSessionEntry({ profileOverride })` without clearing profile state unintentionally. Add focused tests for OpenAI/GPT and Anthropic/Claude profile-backed models plus docs showing the button/menu flow alongside the existing `/model <model>@<profileId>` syntax.
