## Summary

Current main still lacks the requested first-class global setup/onboarding hook for credentialed tool-only plugins. Provider and CLI-backend setup are descriptor-backed and setup-registry-backed, but there is no manifest `setup.toolPlugins`, no tool setup descriptor lookup, no `registerToolPluginSetup`-style SDK API, and the generic plugin config wizard explicitly skips sensitive fields rather than collecting API keys.

## Best Possible Solution

Keep this open as a plugin-SDK/core setup design item. The best path is an additive, manifest-first `setup.toolPlugins` descriptor plus a setup-only API/helper for SecretInput-backed tool-plugin credentials and documented plugin-owned persistence under plugin config or another explicit plugin-owned path, with contract tests and public plugin docs alongside the provider and CLI-backend setup surfaces.

## Risks / Open Questions
