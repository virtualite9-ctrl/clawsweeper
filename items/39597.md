## Summary

Keep open. Current main already distinguishes browser Control UI and native macOS clients at Gateway connect time, but the requested agent-facing WebChat runtime context is still missing. The runtime prompt path exposes channel/capabilities, while chat.send only maps the caller into untrusted message context fields such as SenderId/SenderName and GatewayClientScopes.

## Best Possible Solution

Keep #39597 open and implement a narrow trusted Gateway client-context path: define sanitized client metadata, propagate WebChat connect.client from chat.send into agent run setup and RuntimeInfoInput, update buildRuntimeLine and prompt tests to expose fields such as client=openclaw-macos or client_mode=ui, and document which fields are trusted and privacy-safe.
