## Summary

PR #61538 adds a Jina memory embedding provider, but current main has moved remote embedding providers into plugin-owned capability adapters and already exposes the SDK/manifest/runtime seams needed to implement Jina outside core. No Jina provider ships on main today, and no missing core API is apparent from the PR diff or current code.

## Best Possible Solution

Do not merge a Jina-only provider into OpenClaw core. The best path is a standalone Jina memory-embedding plugin published and discovered through ClawHub, declaring contracts.memoryEmbeddingProviders for "jina", registering api.registerMemoryEmbeddingProvider(adapter), and using the existing remote embedding helpers/cache-key pattern. If that implementation exposes a real SDK gap, open a narrow plugin API issue or PR for the missing seam.
