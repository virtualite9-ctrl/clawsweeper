## Summary

Keep #23916 open. Current main still has the pairing metadata needed to solve the request, but `openclaw devices list` only exposes a compact paired-device table and lacks `--full`, describe, rename, or notes workflow. Related #13870 and PR #55973 are narrower and do not supersede this broader request.

## Best Possible Solution

Keep #23916 open and implement a core CLI device-detail path: add a safe compact purpose/client hint to `openclaw devices list`, add `--full` or `openclaw devices describe` for client ID/mode, platform/device family, timestamps, and safe host/IP details, and coordinate editable display names/notes with #13870 while preserving existing JSON output and adding docs/tests.
