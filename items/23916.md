## Summary

Keep #23916 open. Current main still stores and transports the device metadata needed for this feature, but the human `openclaw devices list` output remains a compact table and there is still no `--full`, describe, rename, or notes workflow. The related open #13870 and PR #55973 are narrower and do not supersede this broader request.

## Best Possible Solution

Keep #23916 open and implement a core CLI device-detail path: add a safe compact purpose/client hint to `openclaw devices list`, add `--full` or `openclaw devices describe` for client mode/id, platform/device family, timestamps, and safe host/IP details, and coordinate editable display names/notes with #13870 while updating docs/tests and preserving existing JSON output.
