## Summary

Keep #23916 open. Current main stores useful pairing metadata, but `openclaw devices list` still does not expose device purpose/client details in the human list, has no `--full` or describe workflow, and has no editable notes/name command for devices. Related #13870 and PR #55973 cover narrower naming/timestamp work and do not supersede this broader request.

## Best Possible Solution

Keep #23916 open and implement a user-facing device detail path: add a safe compact purpose/client hint to `openclaw devices list`, add `--full` or `openclaw devices describe` for client ID/mode, platform/device family, timestamps, and safe host/IP details, and coordinate editable display names/notes with #13870 while preserving existing `--json` behavior.
