## Summary

Keep #23916 open. Current main still has the pairing metadata needed to solve the request, but the human `openclaw devices list` output only shows Device, Roles, Scopes, Tokens, and IP for paired devices. There is still no `--full` or describe workflow and no editable device note/name command. Related #13870 and PR #55973 remain narrower and do not supersede this broader request.

## Best Possible Solution

Keep #23916 open and implement a user-facing device detail path in core: add a safe compact purpose/client hint to `openclaw devices list`; add `--full` or `openclaw devices describe` for client ID/mode, platform/device family, timestamps, and safe host/IP details; and coordinate editable display names/notes with #13870 while preserving existing JSON output.
