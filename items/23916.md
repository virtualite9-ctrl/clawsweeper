## Summary

Keep #23916 open. Current main still stores useful pairing metadata, but the human `openclaw devices list` output does not expose device purpose/client details, there is no `--full` or describe workflow, and there is no editable device note/name command. Related #13870 and PR #55973 remain narrower and do not supersede the broader request.

## Best Possible Solution

Keep #23916 open and implement a user-facing device detail path in core: add a safe compact purpose/client hint to `openclaw devices list`, add `--full` or `openclaw devices describe` for client ID/mode, platform/device family, timestamps, and safe host/IP details, and coordinate editable display names/notes with #13870 while preserving existing `--json` behavior.
