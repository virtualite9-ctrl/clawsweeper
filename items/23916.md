## Summary

Keep #23916 open. Current main stores client metadata internally and exposes it through raw pairing records, but the human `openclaw devices list` output still does not show client purpose, client mode/id, platform/device family, host/user-agent-style detail, editable notes/names, or a `--full` view. The previous Codex review comment is still consistent with the current touched code.

## Best Possible Solution

Keep #23916 open and implement a user-facing devices detail path: add compact purpose metadata to `openclaw devices list`, add a `--full` or describe-style view for clientId/clientMode/platform/deviceFamily/timestamps, and coordinate editable names/notes with the related paired-device naming work instead of closing this as already implemented.
