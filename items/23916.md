## Summary

Keep #23916 open. Current main still stores and returns useful pairing metadata, but the human `openclaw devices list` output only shows device label, roles, scopes, token summary, and IP. It still lacks the requested purpose/client mode, platform/device family, host/user-agent-style detail, editable notes/names, or `--full` detail workflow. Related #13870 and PR #55973 cover narrower pieces, not the full issue.

## Best Possible Solution

Keep #23916 open. The best product path is to add a user-facing device detail workflow: show a safe compact purpose/client hint in `openclaw devices list`, add `--full` or `devices describe` for client ID/mode, platform/device family, timestamps, and safe host/IP details, and coordinate editable display names/notes with #13870 while preserving existing `--json` behavior.
