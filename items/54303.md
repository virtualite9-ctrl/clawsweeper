## Summary

Current main already handles the reported cron-delivery shape: plugin channel aliases are normalized, cron delivery validation uses that normalization, and the isolated-agent cron delivery path preserves explicit `delivery.to` targets. The shipped `v2026.4.22` release contains this behavior, so the specific report is best treated as already implemented on main.
