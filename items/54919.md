## Summary

Current main partially addresses the report: `agents.defaults.thinkingDefault` is now present in the config type/schema and is used by the thinking-default resolution path. The issue should stay open because the body also asks for `agents.defaults.reasoningDefault`, and current main still only supports reasoning defaults at `agents.list[].reasoningDefault` plus session/directive state. The related open PR #71817 explicitly says it only partially addresses this issue and does not cover global `agents.defaults.reasoningDefault`.

## Best Possible Solution

Keep this issue open until maintainers either implement and document `agents.defaults.reasoningDefault` across the same config/schema/runtime/status surfaces as `thinkingDefault`, or explicitly decide that reasoning visibility should remain per-agent/session-only and update the docs/tool allowlist accordingly. Review the existing global-reasoning-default PRs and the partial Telegram/Feishu PR before duplicating work.
