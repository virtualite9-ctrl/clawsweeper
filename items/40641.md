## Summary

Current main still contains the affected unquoted bracket-notation config examples, including the exact per-agent exec node command from the report. The CLI parser supports bracket paths once the shell passes the argument through, so the remaining product problem is a docs regression: shells such as zsh can treat the unquoted `[0]` as glob syntax before OpenClaw runs. The related docs PR #40642 identified the right fix but was closed unmerged, and no equivalent fix is present on main.

## Best Possible Solution

Keep the issue open as a concrete docs regression. Reapply the intent of PR #40642: quote every config CLI example that uses bracket notation, or switch those examples to dot-index form such as `agents.list.0.tools.exec.node`, and add a short note on the config docs explaining that paths containing `[]` should be quoted in shells like zsh. The likely source files are `docs/cli/config.md`, `docs/tools/exec.md`, and `docs/nodes/index.md`; localized pages should be regenerated from the English docs source.
