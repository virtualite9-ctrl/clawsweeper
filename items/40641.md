## Summary

Current main still contains the reported unquoted bracket-notation config examples in the exec, config, and nodes docs. The CLI/config implementation accepts bracket notation once the argument reaches OpenClaw, so the remaining problem is a real docs/shell-quoting regression. Related PR #40642 proposed the right direction but was closed unmerged.

## Best Possible Solution

Keep this issue open as a concrete docs regression. Update the English source docs to quote every `openclaw config` example that uses bracket notation, or switch examples to shell-safe dot-index paths such as `agents.list.0.tools.exec.node`, and add a short note in the config docs that paths containing `[]` must be quoted in shells that treat brackets as glob patterns. The likely files are `docs/cli/config.md`, `docs/tools/exec.md`, and `docs/nodes/index.md`; localized zh-CN output should flow through the docs publishing/i18n pipeline rather than direct locale edits in this repo.
