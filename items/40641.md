## Summary

Current main still documents the reported unquoted bracket-notation config path in the exec docs and related config/node docs. The CLI accepts bracket notation once argv reaches OpenClaw, so the remaining issue is a real docs/shell-quoting regression, not a completed core fix.

## Best Possible Solution

Keep this issue open as a concrete docs regression. Quote every `openclaw config` example that uses bracket notation, or switch examples to dot-index paths such as `agents.list.0.tools.exec.node`, and add a short note in the config docs that paths containing `[]` must be quoted in shells like zsh. The likely source files are `docs/cli/config.md`, `docs/tools/exec.md`, and `docs/nodes/index.md`; localized pages should flow from the docs pipeline rather than direct locale edits.
