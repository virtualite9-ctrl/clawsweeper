## Summary

Current `main` already carries the async exec-approval followup routing/fallback fix: it preserves the original session key when `notifySessionKey` is missing, and if the resume path still times out it sends a sanitized direct followup instead of dropping the message. That behavior was merged in PR #59351 (`4207ca2eb8816e4bab47afe2a55f18f7661e7771`) and shipped in `v2026.4.2`.
