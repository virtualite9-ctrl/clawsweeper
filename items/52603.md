## Summary

Close as `cannot_reproduce`. The reporter explicitly followed up that the symptom was caused by GitHub Copilot plan/quota state, not a Telegram direct-session regression in OpenClaw. Current main docs also describe Copilot model availability as plan-dependent, and the shared runtime auth/auth-profile tests do not support the earlier Telegram-specific auth-loading hypothesis.
