## Summary

Keep #22770 open. Current main still lacks a TUI external-editor compose flow: CustomEditor has no Ctrl+E/external compose hook, runTui only wires the existing shortcuts and uses TUI suspension for auth, the slash-command registry has no edit/compose command, and the public TUI docs do not document $VISUAL/$EDITOR handoff. Related #55836 is adjacent in-place vi/vim input editing, not a shipped replacement for temp-file editor handoff.

## Best Possible Solution

Implement this in the core TUI composer: add a dedicated shortcut or command, suspend the TUI, prefill a temporary markdown file with the current composer text, launch $VISUAL then $EDITOR and wait for exit, read the saved content back into CustomEditor with no auto-submit, surface missing-editor/non-zero-exit/file I/O errors, update docs/web/tui.md, and add focused CustomEditor/TUI tests. Verify the final keybinding against @mariozechner/pi-tui Editor defaults before choosing Ctrl+E.
