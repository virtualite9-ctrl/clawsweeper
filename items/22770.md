## Summary

Current main still does not implement the requested $VISUAL/$EDITOR external-editor compose mode for the TUI. The core editor and TUI wiring support inline editing, shortcuts, selectors, slash commands, local shell commands, and a suspended /auth flow, but there is no Ctrl+E/open-editor handler, temp prompt file roundtrip, editor spawn, reload into the composer, or docs/tests for that behavior. The related #55836 remains adjacent vi/vim in-place editing work, not a duplicate or shipped substitute.

## Best Possible Solution

Keep #22770 open and implement this in the core TUI composer: add a deliberate shortcut or command, suspend the TUI, write editor.getExpandedText() to a temporary markdown prompt file, launch $VISUAL then $EDITOR and wait, load the edited file back with editor.setText without auto-submitting, surface missing-editor/non-zero-exit/file-I/O status messages, document it in the public TUI guide, and add focused CustomEditor/runTui tests. Keep #55836 separate for in-place vi/vim input mode.
