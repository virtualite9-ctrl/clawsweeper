## Summary

Keep #22770 open. Current main still does not implement a $VISUAL/$EDITOR external editor compose flow for the TUI, and the linked #55836 is a distinct request for in-place vi/vim keybindings rather than temp-file editor handoff.

## Best Possible Solution

Implement this in the core TUI composer rather than ClawHub: add a dedicated keybinding or command that suspends the TUI, writes the current expanded composer text to a temporary markdown file, launches $VISUAL then $EDITOR, reads the saved content back into CustomEditor without auto-submitting, reports missing editor/non-zero exit/file I/O errors in the status line, and adds focused unit tests plus TUI docs/shortcut updates. Also resolve the Ctrl+E conflict with pi-tui’s existing line-end binding before choosing the final default key.
