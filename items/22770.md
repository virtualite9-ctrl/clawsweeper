## Summary

Keep #22770 open. Current main still does not implement an external-editor compose handoff for the TUI composer: there is no Ctrl+E/external-editor callback in CustomEditor, runTui only suspends the TUI for the provider auth flow, slash commands do not provide an edit/compose route, and the public TUI docs do not document $VISUAL/$EDITOR behavior. The related #55836 remains adjacent in-place vi/vim editing work, not a temp-file editor roundtrip.

## Best Possible Solution

Implement the feature in the core TUI composer: add a deliberate shortcut or slash command, suspend the TUI, write the current composer contents to a temporary markdown file, launch $VISUAL then $EDITOR and wait, read the edited content back into CustomEditor without auto-submitting, show clear status/errors for missing editor, non-zero exit, and file I/O failures, update docs/web/tui.md, and add focused CustomEditor/TUI tests. Keep #55836 as the separate in-place vi/vim input-mode request.
