## Summary

Close this PR as duplicate/superseded by the existing #6615 feature tracker. Current main still lacks first-class exec-approval denylist support, but this PR is not a viable implementation: it adds an unrelated standalone Python package, replaces the root README/pyproject surfaces, does not integrate with the TypeScript exec-approval runtime, and the existing review comments identify import/syntax/runtime defects that prevent the new package from working.

## Best Possible Solution

Close this PR and keep the denylist feature discussion on #6615. A viable implementation should be a focused OpenClaw core change across the TypeScript exec-approval types, schema/validators, normalization/storage, gateway/node/system.run decision path, CLI or Control UI, docs, and tests, without unrelated root README/package metadata rewrites or a detached Python package.
