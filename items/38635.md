## Summary

Current `main` does not implement this PR's hamburger-click behavior. Focus mode still suppresses the nav drawer while active, and the visible mobile hamburger only toggles `navDrawerOpen` without clearing `chatFocusMode`. Main does provide a separate in-chat `Exit focus mode` button, including in the latest release, so the original UX problem is partially mitigated, but that is a different shipped behavior and not strong enough evidence to auto-close this PR.
