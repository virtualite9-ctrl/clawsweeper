## Summary

Current main still has the UX bug this PR targets. In chat focus mode, the drawer is forced closed, but the visible topbar hamburger only toggles drawer state and does not clear user-initiated focus mode. The PR remains useful, though its patch targets the older `.nav-collapse-toggle` path and likely needs rebasing/adaptation to `.topbar-nav-toggle`.

## Best Possible Solution

Keep this PR open or replace it with an equivalent current-main patch. The fix should update the `.topbar-nav-toggle` click path so that, when user-initiated `settings.chatFocusMode` is active, clicking the hamburger clears `chatFocusMode` and opens the nav/drawer; onboarding-controlled focus should remain controlled by onboarding. Add a browser regression test that activates `.shell--chat-focus`, clicks `.topbar-nav-toggle`, and verifies focus mode exits and the nav/drawer becomes visible.
