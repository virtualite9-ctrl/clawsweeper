## Summary

Current main still lacks the behavior PR #38635 is trying to add. The visible topbar hamburger only toggles drawer state; focus mode still suppresses the drawer/nav while `chatFocusMode` remains active, so the click can remain visually ineffective.

## Best Possible Solution

Keep this PR open or replace it with an equivalent current-main patch: when user-initiated `chatFocusMode` is active, clicking the topbar hamburger should clear focus mode and open the nav/drawer; onboarding-controlled focus should remain controlled by onboarding. Add a browser regression test that activates `.shell--chat-focus`, clicks `.topbar-nav-toggle`, and verifies focus mode exits and the nav/drawer becomes visible.
