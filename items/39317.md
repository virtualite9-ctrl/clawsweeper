## Summary

Keep #39317 open. Current `main` still has the reported false-success path for `codex-cli` JSONL runs that exit 0 with semantic sandbox/write-denied output. The earlier proposed fix in #39460 was closed unmerged, and the related #66081 artifact-finalization work is explicitly scoped away from failure classification.

## Best Possible Solution

Keep the issue open. Revive the substance of #39460 on current `main`: parse structured Codex JSONL records for sandbox-denied command/tool/action/error/status signals after exit 0, throw a `FailoverError` before `buildCliRunResult`, add an explicit local `sandbox_denied` failover/status/auth-profile policy if maintainers want that contract, avoid broad assistant-message keyword matching, and add parser plus runner regression tests for true denials and false-positive discussion cases. Treat #66081 as related final-answer reliability work, not a replacement for failure classification.
