## Summary

Keep this issue open. Current main still does not remediate a future `meta.lastTouchedVersion` during ordinary config loads; it only warns, then proceeds. The newer future-config guard blocks selected service/process mutations and service-mode gateway startup, but not ordinary interactive gateway runs that can still perform Discord/OpenAI-Codex delivery. The auto-downstamp PR #46336 was closed unmerged, and the merged GPT-5.4 hardening PR #70743 fixed adjacent runtime/delivery problems without closing this future-stamped-config gap.

## Best Possible Solution

Keep this issue open until maintainers either extend the future-config guard/remediation to ordinary gateway runtime paths that can perform channel delivery, or add focused regression/manual proof showing the current service/process guard fully eliminates the Discord/OpenAI-Codex silent-delivery failure. A good fix would make the startup warning actionable and point users to the split-brain install repair flow without blocking safe read-only inspection.
