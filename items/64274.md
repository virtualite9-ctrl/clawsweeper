## Summary

Current main still appears to contain the reported auth-profile runtime snapshot collision. The prior discussion is useful: EronFan explicitly picked up the issue and PR #64320 proposed the right fix shape with a regression test, but that PR is closed and unmerged, and current main still keys runtime auth snapshots by resolved auth-store path rather than by stable original agent identity.

## Best Possible Solution

Keep this issue open. The best fix is to revive PR #64320 or an equivalent implementation: key runtime auth snapshots by a stable original identity such as raw `agentDir` plus a separate default-agent sentinel, update get/has/set/replace/save/load callers consistently, and add a regression test where `OPENCLAW_AGENT_DIR` points at the requested agent while main and that agent have distinct `minimax-portal` profiles.
