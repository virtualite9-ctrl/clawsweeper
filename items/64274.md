## Summary

Current main still contains the reported auth-profile runtime snapshot collision. Runtime snapshots are keyed by resolved auth-store path, while `resolveAuthStorePath(undefined)` honors `OPENCLAW_AGENT_DIR`; when that env var points at the requested agent, default and explicit lookups collapse. The related PR #64320 described the right fix shape but is closed and unmerged in the provided GitHub context.

## Best Possible Solution

Keep this issue open. Revive PR #64320 or apply an equivalent fix: key runtime auth snapshots by a stable original identity such as raw `agentDir` plus a distinct default-agent sentinel, update get/has/set/replace/save/load callers consistently, and add a regression test where `OPENCLAW_AGENT_DIR` points at the requested agent while main and that agent have distinct `minimax-portal` profiles.
