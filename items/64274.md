## Summary

Current main still contains the per-agent auth runtime snapshot collision described by this issue. Runtime snapshots are keyed by resolved auth-store paths, default auth-store resolution still follows OPENCLAW_AGENT_DIR, and MiniMax portal discovery still depends on the affected per-agent auth store. The related PR #64320 was closed unmerged, so there is no shipped fix to close this issue against.

## Best Possible Solution

Keep this issue open. Revive PR #64320 or apply an equivalent focused fix that keys runtime auth snapshots by stable runtime identity instead of resolved auth-store path, updates get/has/set/replace/save/load callers consistently, and adds a regression test where OPENCLAW_AGENT_DIR points at the selected agent while main and that agent have distinct minimax-portal profiles.
