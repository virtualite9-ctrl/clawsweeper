## Summary

Keep open. The linked bug is real enough and current main still does not repair or verify the global npm bin symlink after update. The PR is relevant, but it should be revised before merge because the current patch has unresolved review concerns and includes an unrelated test-harness change.

## Best Possible Solution

Keep the PR or linked issue #63719 open. The best path is a focused npm-only post-install bin verification/repair that is wired into both global update paths, parses and normalizes `package.json.bin` with npm-compatible semantics, avoids broad deletion of arbitrary `.openclaw-*` bin entries, adds targeted regression tests, and removes the unrelated overflow-compaction harness change.
