## Summary

Current main still supports `before_tool_call` and `after_tool_call` through typed plugin hooks, but not through managed/internal `HOOK.md` hooks. The issue remains the canonical product/API request, reinforced by the duplicate #17008, the related no-op report #67085, the open implementation attempt #29590, and the recent maintainer comment explicitly keeping #13364 open.

## Best Possible Solution

Keep #13364 open for maintainer product/API judgment. A complete fix should add typed internal `tool_call` before/after events, define how managed handler results can block or rewrite params, bridge tool execution without double-firing plugin hooks, handle unsupported/unreachable `HOOK.md` event names clearly, and update docs plus focused internal-hook tests. If maintainers decide tool-call interception should stay plugin-only, close later with an explicit product decision and point users to plugin hooks and ClawHub/plugin packaging.
