## Summary

Keep #13364 open. Current main still only exposes `before_tool_call`/`after_tool_call` through the plugin hook runner; managed/internal `HOOK.md` hooks have no typed tool-call event, no block/param-rewrite return contract, and no dispatch bridge from tool execution. The provided discussion also includes a fresh maintainer comment explicitly keeping this issue open as the canonical tracker, with #17008 closed as a duplicate and #29590 closed unmerged as only a partial after-call attempt.

## Best Possible Solution

Keep #13364 open for maintainer product/API judgment. A complete fix should add typed managed/internal tool-call before/after events, define `before` return semantics for blocking and parameter rewriting, bridge the tool execution paths without double-firing existing plugin hooks, validate or clearly warn on unsupported HOOK.md event names, and update docs plus focused internal-hook tests. If maintainers decide this must remain plugin-only, close later with an explicit product decision and point users to plugin hooks and ClawHub packaging.
