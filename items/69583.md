## Summary

Keep PR #69583 open. Current main still serializes direct multi-collection QMD searches by awaiting each `runQmd` call inside a `for` loop, and the supporting abort-signal plumbing/test coverage from the PR is not present. The PR is recent, focused, and not superseded by the related ClawSweeper items.

## Best Possible Solution

Keep the PR open for normal maintainer review. The best path is to either land a bounded, well-tested version of the parallel direct-QMD multi-collection search change or ask the author to add concurrency/backpressure and robust abort cleanup if maintainers agree with the Aisle review concerns.
