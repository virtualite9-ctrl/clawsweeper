## Summary

Keep open. Current main still serializes multi-collection direct QMD searches, and the default QMD config keeps the affected direct CLI path active unless users opt into mcporter. The PR is not implemented or superseded on main; it has concrete performance evidence and tests, but should receive maintainer review for the process fan-out and abort-cleanup concerns raised in the discussion before landing.

## Best Possible Solution

Keep PR #69583 open. The best path is to rebase it onto current main, preserve the merge/dedup semantics, add the parallel-start and sibling-abort regression tests, and have a maintainer decide whether to add a per-request or global QMD subprocess concurrency cap before landing.
