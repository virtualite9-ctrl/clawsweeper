## Summary

Keep PR #69583 open. Current main still performs multi-collection QMD search serially with an awaited `runQmd` inside a `for` loop, and the current SDK command runner has no `AbortSignal` path. The PR is recent, focused, and includes useful reproduction/perf measurements plus test changes; it is not implemented, duplicated, or superseded on main.

Keep the PR open and review it as a real memory-core performance fix. The best path is to adapt or land the parallel multi-collection search implementation with regression coverage, while deciding whether to add bounded concurrency and stronger abort cleanup in `runCliCommand` before merge.
