## Summary

Keep #71301 open. Current main has partial groundwork: docs are included in the package, agents are pointed at local docs when available, and config schema lookup exists. The central request remains broader: first-class, version-coupled docs retrieval for agents. Current main still lacks a local `docs.search`/`docs.get` style agent tool or bundled docs search/index layer, and `openclaw docs` still searches the live hosted docs endpoint.

## Best Possible Solution

Keep the issue open and scope an incremental core implementation: generate a release-time docs manifest/search index from the authored docs, ship it with the package, add an agent-accessible local-first retrieval surface such as `docs.search` and `docs.get`, keep `config.schema.lookup` as the schema-specific path, and use live docs only as an explicit latest/fallback source.
