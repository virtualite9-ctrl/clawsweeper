## Summary

Keep this issue open. Current main has partial support for bundled docs and prompt guidance, but the central request remains unimplemented: agents do not have a first-class, local-first, version-matched docs retrieval surface such as docs.search/docs.get, and the existing docs CLI still searches the live docs index.

## Best Possible Solution

Keep the issue open and narrow the remaining work: keep the existing bundled docs path, add a release/package docs manifest and local search/get index over bundled docs, expose local-first docs retrieval to agents through the gateway/MCP/tool surface, keep live docs as an explicit fallback, and add focused tests for package-installed docs discovery plus agent/tool behavior.
