## Summary

Keep open. Current `main` still lacks the cleanup unification this PR proposes: orphan pruning bypasses final cleanup bookkeeping, and terminated runs are still coalesced by `childSessionKey` before delete cleanup, which can leave stale run entries behind.

## Evidence

- **Terminate path still collapses multiple runs to one cleanup call:** `markSubagentRunTerminated` collects entries in `entriesByChildSessionKey` and then calls `completeCleanupBookkeeping` once per unique child session key. If multiple run IDs share the same child session key, only the first entry gets delete cleanup bookkeeping.
  - file: [src/agents/subagent-registry-run-manager.ts:439](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/src/agents/subagent-registry-run-manager.ts#L439)
  - command: `nl -ba src/agents/subagent-registry-run-manager.ts | sed -n '428,505p'`
  - sha: [c070509b7f8f](https://github.com/openclaw/openclaw/commit/c070509b7f8fdbe479e5d829643782c272e30e83)
- **Delete cleanup is per-run, so skipped runs remain stale:** `completeCleanupBookkeeping` deletes exactly the passed `runId` from `runs` on delete cleanup. That makes the child-session-key coalescing above materially important: skipped run IDs are not removed here.
  - file: [src/agents/subagent-registry-lifecycle.ts:381](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/src/agents/subagent-registry-lifecycle.ts#L381)
  - command: `nl -ba src/agents/subagent-registry-lifecycle.ts | sed -n '381,421p'`
  - sha: [c070509b7f8f](https://github.com/openclaw/openclaw/commit/c070509b7f8fdbe479e5d829643782c272e30e83)
- **Orphan prune still bypasses final cleanup bookkeeping:** `reconcileOrphanedRun` directly mutates the entry, removes attachments, deletes the run, and removes it from `resumedRuns`, but it does not route through `completeCleanupBookkeeping`, does not call `retryDeferredCompletedAnnounces`, and does not notify the context engine here. That matches the bug this PR is trying to fix.
