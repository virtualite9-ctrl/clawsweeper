## Summary

Keep PR #38658 open. Current main still implements Slack group/channel history as pending-only: skipped mention-gated messages are recorded for later context, but processed `requireMention=false` channel messages are not retained after dispatch, and the existing regression test still asserts that the second always-on channel message does not see the first as history. The PR is a plausible focused behavior change, but it conflicts with the current documented pending-only model and still needs explicit maintainer product judgment rather than conservative cleanup closure.

## Best Possible Solution

Keep PR #38658 open for maintainer review. If always-on Slack channel context is the desired product behavior, update `extensions/slack/src/monitor/message-handler/*` so prepared messages carry the resolved mention policy into dispatch, `requireMention=false` processed room messages are retained in both reply and no-reply paths, retained history ordering is deterministic, regression tests assert the new prompt context, and the Groups/Slack docs explain the new non-pending history behavior. If maintainers prefer the current pending-only model, close the PR with that explicit product rationale rather than automated cleanup.
