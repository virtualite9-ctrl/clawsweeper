## Summary

Keep PR #38658 open. Current main at 4a3030df9efa72c44bab567a4390ac0c4876a73f still has the pending-only Slack channel-history behavior: prepared Slack messages do not carry the resolved mention policy into dispatch, skipped mention-gated messages are the only room-history writes, dispatch clears room history after processed room turns, and an existing regression test asserts that `requireMention=false` processed channel messages are not injected as history. The PR remains a focused plausible fix, but it changes Slack prompt-context semantics and still has an ordering concern from review, so it needs maintainer judgment rather than conservative cleanup closure.

## Best Possible Solution

Keep the PR open for maintainer review. If always-on Slack channel context is desired, update the current `extensions/slack/src/monitor/message-handler/*` implementation to carry resolved `requireMention` into dispatch, retain processed room messages for `requireMention=false` in both reply and no-reply paths, preserve chronological order, add prompt/history regression coverage, and update the public group-message/Slack history docs. If maintainers prefer the current pending-only contract, close the PR with that explicit product rationale.
