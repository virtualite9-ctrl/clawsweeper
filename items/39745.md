## Summary

Keep PR #39745 open. Current main already has the Twitch timeout cleanup, but the Synology Chat and Zalouser Promise.race timeout timers are still not cleared when the primary promise wins, so the PR remains partially useful rather than obsolete.

## Best Possible Solution

Keep this PR open. Ask the author to rebase on current main, drop or resolve the obsolete Twitch hunk, retain the Synology Chat and Zalouser timer cleanup, and add focused fake-timer success-path coverage where feasible. For Synology Chat, maintainers can separately decide whether the final product behavior should clear the existing 120s timeout or remove that plugin-local timeout in favor of the core agent timeout.
