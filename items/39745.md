## Summary

Keep PR #39745 open. Current main already contains the Twitch timeout cleanup, but the Synology Chat and Zalouser `Promise.race` timeout patterns are still present without clearing the losing timer, so the PR remains partially useful and is not obsolete.

## Best Possible Solution

Keep this PR open. Ask the author to rebase on current main, drop or resolve the already-implemented Twitch hunk, retain the Synology Chat and Zalouser timeout cleanup, and add focused fake-timer success-path coverage where feasible. For Synology Chat, maintainers can also decide whether the better final shape is clearing the existing 120s timer or removing that plugin-local timeout per the earlier #38360 discussion.
