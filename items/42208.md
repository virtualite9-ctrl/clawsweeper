## Summary

Keep this PR open. The requested hard enforcement boundary is not implemented on current main, and the PR itself still needs changes before it can be merged: the proposed guard is not wired into production dispatch, appears mismatched with current agent-scoped hook session keys, and the web_search warning change conflicts with current tests. The work is a real core security-hardening direction rather than cleanup that should be auto-closed.

## Best Possible Solution

Keep the PR open but request changes or supersede it with a production-wired implementation. The right fix likely belongs in the existing before_tool_call/tool-dispatch path, with tests proving that external hook sessions require approval for destructive core tools and messaging/subagent actions, that agent-scoped hook session keys are recognized, that allowUnsafeExternalContent behavior is deliberate, and that web_search warning expectations are updated if that behavior change is still desired.
