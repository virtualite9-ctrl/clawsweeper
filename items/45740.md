## Summary

Keep #45740 open. Current main still has the reported gh-issues prompt-injection surface: raw issue bodies and review-comment JSON are interpolated directly into spawned sub-agent prompts, and the skill does not use the repo’s existing untrusted-content wrappers, sandbox requirement, or any per-spawn tool policy. The discussion and closed #45748 support treating this as real unattended-automation hardening work, not as a resolved main-branch issue.

## Best Possible Solution

Keep the issue open as a concrete gh-issues/runtime hardening task. The best fix is to stop passing raw issue and review text inside the child prompt: wrap or sanitize GitHub content with the existing untrusted-content helpers, or pass it through a data/attachment/tool-parameter path that the child reads as data. Pair that with regression tests using hostile issue/review bodies, and consider adding a per-spawn tool profile/allow/deny input or enforcing `sandbox: "require"` for unattended gh-issues runs while preserving the documented notification path.
