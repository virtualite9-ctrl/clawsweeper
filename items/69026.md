## Summary

Keep #69026 open. Current main still has the 30s TUI streaming watchdog and still emits the exact alarming system warning this PR targets. The PR is a narrow, plausible core TUI change with test updates, and the related #68596 configurability issue does not supersede this concrete implementation attempt.

## Best Possible Solution

Keep the PR open for maintainer review. The likely best path is to accept the threshold increase if maintainers agree with the UX direction, but revise the warning behavior to either use a calmer message or internal/debug logging, and leave the broader configurable-timeout work tracked separately by #68596.
