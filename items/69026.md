## Summary

Keep #69026 open. Current main still has the reported TUI behavior: a 30s default streaming watchdog and an alarming user-visible system message when the watchdog fires. The PR is a focused, unmerged change against the implicated code path with matching test updates, and it is not clearly superseded by a merged fix or canonical replacement. Maintainer review is still useful, especially for the UX choice raised in review: silent reset versus a softer message or debug-only signal.

## Best Possible Solution

Keep the PR open for maintainer review. The likely path is to adapt or merge a focused TUI fix that raises the default away from 30s, preserves the internal stale-run reset, and resolves the UX question explicitly, either with no user-facing warning, a softer resync message, or a documented/configurable threshold aligned with #68596.
