## Summary

Keep #27482 open. Current main still explicitly blocks direct video chat uploads in the Control UI and drops video attachments in the Gateway `chat.send` parsing path. Separate media-understanding/video-generation surfaces exist, but they do not implement the requested direct MP4/WebM/MOV upload-to-LLM chat workflow.

## Best Possible Solution

Keep #27482 open until OpenClaw adds first-class direct video chat attachments across the Control UI and Gateway path: accept and preview MP4/WebM/MOV, define a video-capable attachment/content model, route provider-native video input or media-understanding summaries by model/provider capability and size policy, persist transcript metadata, and cover UI, Gateway parsing, provider dispatch, and docs with focused tests.
