## Summary

Keep #27482 open. Current main still has video understanding through separate CLI/media-understanding surfaces, but the requested direct Control UI chat/message video upload path is still image-only and non-image payloads are filtered before model dispatch.

## Best Possible Solution

Keep #27482 open until OpenClaw adds first-class video chat attachments in the core Control UI and Gateway path: accept and preview MP4/WebM/MOV, add a video attachment/content model instead of `ChatImageContent` only, route videos through provider-native video input or the existing media-understanding video path with capability and size checks, and cover the UI/Gateway/provider/docs behavior with focused tests.
