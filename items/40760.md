## Summary

Keep open. Current main still lacks the PR's central behavior: typing TTL is still fixed by default and the embedded-agent failure path still has no timeout-specific final reply. The submitted patch is focused and not security-sensitive, but review comments identify real functional blockers, and same-author #41957 remains open as a consolidation candidate rather than a safe automatic close reason.

## Best Possible Solution

Keep this PR open until maintainers either review it directly or consolidate it with #41957 and the broader user-feedback work in #43204. The implementation should compute typing TTL from the actual fallback candidate budget, including configured-primary append and transient retry replay, use a narrow timeout predicate such as `isTimeoutError` for final user-facing timeout copy, and add regression coverage for true timeout failures versus bare network/DNS errors.
