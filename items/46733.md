## Summary

Keep #46733 open. Current main only partially addresses the report: runtime context resolution can return 1M for Anthropic Opus/Sonnet when `params.context1m: true` is configured, but the model-list row path still displays raw model metadata, and the bundled Anthropic provider still deliberately strips the `context-1m-2025-08-07` beta for `sk-ant-oat` OAuth tokens. The related PRs #46761 and #46781 were closed unmerged, and the remaining open related issue #41444 covers a narrower model-header OAuth path rather than this exact `params.context1m` + OAuth long-context request.

## Best Possible Solution

Keep this open as a real remaining Anthropic provider bug. The likely fix spans two surfaces: make `openclaw models list` reflect the effective `params.context1m` runtime context for configured Anthropic Opus/Sonnet models, and revisit `extensions/anthropic/stream-wrappers.ts` so OAuth long-context beta handling matches Anthropic's current behavior, with regression tests for API-key and OAuth token paths plus docs updates.
