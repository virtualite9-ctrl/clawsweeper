## Summary

Current `main` already covers the requested behavior without this PR. Discord native slash contexts now carry `Provider`/`OriginatingChannel = "discord"`, so elevated allowlist resolution in `get-reply-directives.ts` has the channel it needs, and `sandbox-explain.ts` already infers the active channel from agent-scoped session keys like `agent:main:discord:slash:<id>`. The PR's added fallback is therefore redundant on current `main`.
