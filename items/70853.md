## Summary

Keep #70853 open. Current main still rotates the live reply session to a fresh sessionId/sessionFile without replaying or hydrating prior transcript turns, and the direct fix is only tracked by open, unmerged PR #70898.

## Best Possible Solution

Keep the issue open until #70898 or an equivalent core fix lands. The likely maintainer path is to seed a bounded, replay-safe tail of prior user/assistant transcript records, or otherwise inject that preserved context for silent in-reply rotations, while leaving explicit `/new` and `/reset` fresh-start behavior intact.
