## Summary

Keep this issue open. The same-author paired PR #64712 is still an unresolved implementation candidate, and current main still contains the generic default-account fallback/status paths that can surface an implicit Feishu default account without clear regression coverage for the reported restart symptom.

## Best Possible Solution

Keep this issue open while #64712, or a reworked replacement, is reviewed. The best fix is to reproduce the restart/status path with multiple Feishu accounts, distinguish real configured accounts from implicit fallback default accounts in the owning status/runtime surface, and add Feishu-focused regression coverage proving no unconfigured default account is created or rendered unless explicitly configured.
