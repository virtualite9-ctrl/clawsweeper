## Summary

Keep #17925 open. Current main implements the Gemini side as a bundled `gemini` web_search provider using Google Search grounding, but the requested ZAI/GLM native search support is still missing. The bundled ZAI plugin has no `webSearchProviders` manifest contract, no `registerWebSearchProvider(...)` call, no ZAI web-search artifact, and no docs/config/test coverage for a ZAI search provider. The related Zhipu/ZAI PRs in the issue context were closed unmerged, and the latest maintainer comment also explicitly kept this tracker open for the remaining ZAI work.

## Best Possible Solution

Keep #17925 open until the bundled ZAI plugin either registers a ZAI/GLM `web_search` provider through the existing plugin web-search contract with manifest metadata, config/docs, and tests, or adds provider-owned native `tools: [{ type: "web_search" }]` passthrough for ZAI model calls. Gemini is already implemented, so maintainers can split any remaining automatic-fallback semantics into a narrower follow-up if desired.
