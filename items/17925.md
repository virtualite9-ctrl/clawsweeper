## Summary

Keep #17925 open. Current main implements the Gemini side as a bundled `gemini` web_search provider using Google Search grounding, but the requested ZAI/GLM native `web_search` passthrough or bundled ZAI web-search provider is still missing. The latest maintainer/Codex issue comment also explicitly keeps this tracker open for the remaining ZAI work, and the related ZAI PRs in the provided context are closed and unmerged.

## Best Possible Solution

Keep #17925 open until the bundled ZAI plugin either registers a ZAI/GLM web_search provider with manifest ownership, config/docs, and tests, or adds provider-owned native `tools: [{ type: "web_search" }]` passthrough for ZAI model calls. Gemini's managed provider appears implemented, so maintainers can split any remaining automatic-fallback semantics into a narrower follow-up if needed.
