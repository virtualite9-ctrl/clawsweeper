## Summary

Keep #17925 open. Current main implements the Google/Gemini side as a bundled `gemini` `web_search` provider using Google Search grounding, but the ZAI/GLM side is still missing: no `webSearchProviders` manifest contract, no ZAI web-search config surface, no `registerWebSearchProvider` registration, and no native `tools: [{ type: "web_search" }]` passthrough path. The related Zhipu/ZAI PRs #10454 and #13814 are closed and unmerged in the provided GitHub context, and the latest issue discussion already preserved this issue for the remaining ZAI/Gemini native-passthrough scope.

## Best Possible Solution

Keep this issue open until the bundled ZAI plugin either registers a ZAI/GLM `web_search` provider with manifest ownership, config/docs, and tests, or adds provider-owned native `web_search` passthrough for ZAI model calls. Maintainers can separately decide whether Gemini's existing managed provider fully resolves the Google side or whether automatic fallback semantics need a narrower follow-up.
