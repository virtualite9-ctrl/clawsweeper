## Summary

Keep open: current `main` still lacks Telegram-specific `streaming.blockStreamingCoalesceDefaults`, so the reported message-splitting gap is not implemented on `main`. The PR matches how other block-streaming channels are configured, and the review-thread type issue was already addressed in the PRŌĆÖs follow-up commit.
