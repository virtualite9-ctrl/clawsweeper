## Summary

Current `main` does not implement this PR's Telegram coalescing-default change. Telegram's shared plugin base still exposes `capabilities.blockStreaming` but no `streaming.blockStreamingCoalesceDefaults`, while the shared block-streaming resolver still reads that plugin field and otherwise falls back to the global 800-char minimum. I did not find evidence that the PR was superseded elsewhere on `main`.
