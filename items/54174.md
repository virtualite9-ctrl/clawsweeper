## Summary

Keep open. Current `main` at `8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf` still does not emit OpenAI-style tool-call data on `/v1/chat/completions`; that handler only returns assistant text chunks plus optional usage. The `/v1/responses` part of the report appears to be fixed on `main`, but the titled chat-completions bug remains unresolved.

## Evidence

- **Chat-completions request model has no tool config handling:** `OpenAiChatCompletionRequest` only models `model`, `stream`, `stream_options`, `messages`, and `user`; there is no request-side `tools` or `tool_choice` support in this handler.
  - file: [src/gateway/openai-http.ts:58](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/gateway/openai-http.ts#L58)
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Non-stream `/v1/chat/completions` always returns plain assistant content:** After `agentCommandFromIngress`, the handler resolves text payloads and always responds with `message: { role: "assistant", content }` and `finish_reason: "stop"`; it never maps pending tool calls into `tool_calls`.
  - file: [src/gateway/openai-http.ts:595](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/gateway/openai-http.ts#L595)
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Streaming `/v1/chat/completions` ignores tool events:** The SSE subscriber only handles `evt.stream === "assistant"` and `evt.stream === "lifecycle"`, writing `delta.content` chunks and a final stop/usage chunk. There is no branch that converts tool activity into OpenAI chat-completions tool-call deltas.
  - file: [src/gateway/openai-http.ts:677](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/gateway/openai-http.ts#L677)
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Current `main` does implement tool-call output for `/v1/responses`:** `openresponses-http.ts` explicitly detects `stopReason === "tool_calls"` and returns or streams `function_call` output items, including streamed `response.output_item.added/done` events.
  - file: [src/gateway/openresponses-http.ts:706](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/gateway/openresponses-http.ts#L706)
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **`/v1/responses` tool-call behavior is covered by tests on main:** Tests assert both non-stream and stream responses include `function_call` output items when pending tool calls exist.
  - file: [src/gateway/openresponses-http.test.ts:847](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/gateway/openresponses-http.test.ts#L847)
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)

  - file: [src/gateway/openai-http.test.ts:878](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/gateway/openai-http.test.ts#L878)
