## Summary

Keep open. Current `main` does not implement Anthropic Message Batches routing for cron/async agent work, and the request is coherent and still actionable.

## Evidence

- **Custom provider API surface still exposes only normal request adapters:** The documented provider config supports `openai-completions`, `openai-responses`, `anthropic-messages`, and `google-generative-ai`; there is no Anthropic batch transport or routing mode.
  - file: [docs/gateway/config-tools.md:403](https://github.com/openclaw/openclaw/blob/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4/docs/gateway/config-tools.md#L403)
  - sha: [f3c37a946cc4](https://github.com/openclaw/openclaw/commit/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4)
- **Anthropic plugin on main still wires normal provider hooks, not batch job submission/polling:** `registerAnthropicPlugin` registers the bundled provider, and the provider hooks cover replay policy, thinking, stream wrapping, usage fetch, and auth hints. No batch submission or async result lifecycle hook is present here.
  - file: [extensions/anthropic/register.runtime.ts:579](https://github.com/openclaw/openclaw/blob/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4/extensions/anthropic/register.runtime.ts#L579)
  - sha: [f3c37a946cc4](https://github.com/openclaw/openclaw/commit/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4)
- **Provider runtime contract remains stream-oriented:** Provider plugins can supply `createStreamFn` or `wrapStreamFn`; the contract shown here is centered on the normal streaming request path, not a provider-managed batch job queue/result callback path.
  - file: [src/plugins/types.ts:1329](https://github.com/openclaw/openclaw/blob/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4/src/plugins/types.ts#L1329)
  - sha: [f3c37a946cc4](https://github.com/openclaw/openclaw/commit/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4)
- **Anthropic-specific request extras only cover betas, service tier, and fast mode:** The Anthropic stream wrapper resolves `anthropicBeta`, `serviceTier`, and `fastMode` and composes wrappers for those only. There is no `batch`/`routing` parameter handling.
  - file: [extensions/anthropic/stream-wrappers.ts:171](https://github.com/openclaw/openclaw/blob/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4/extensions/anthropic/stream-wrappers.ts#L171)
  - sha: [f3c37a946cc4](https://github.com/openclaw/openclaw/commit/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4)
- **Cron tool schema has no batch-routing field:** Cron jobs can carry schedule, payload, delivery, model, fallbacks, thinking, timeout, and light-context controls, but there is no `batch: true` or `routing: batch` field in the exposed schema.
  - file: [src/agents/tools/cron-tool.ts:28](https://github.com/openclaw/openclaw/blob/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4/src/agents/tools/cron-tool.ts#L28)
  - sha: [f3c37a946cc4](https://github.com/openclaw/openclaw/commit/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4)
- **Repo-wide search found no Anthropic Message Batches implementation:** `rg -n "messages/batches|Message Batches|message batches|anthropic batch|batch routing|routing: batch|batch: true" .` returned no relevant OpenClaw hits.
  - command: `rg -n "messages/batches|Message Batches|message batches|anthropic batch|batch routing|routing: batch|batch: true" .`
  - sha: [f3c37a946cc4](https://github.com/openclaw/openclaw/commit/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4)

## Risks / Open Questions

- Implementing this likely needs more than a provider param toggle: async batch submission, polling/webhook completion, and cron/session delivery handoff would need explicit lifecycle support.
- The feature may still be implementable as provider-owned work later, but current `main` does not provide evidence that it has already landed.
