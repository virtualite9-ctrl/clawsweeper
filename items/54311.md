## Summary

Keep this issue open. Current main still has the reported `gh-issues` `user-invocable` frontmatter key, the bundled skill is still a single 34,800-byte `SKILL.md`, and the skill-creator validator still rejects that frontmatter key. The related PR #54360 was closed unmerged, so it did not resolve the mismatch on main.

## Best Possible Solution

Keep this issue open and fix it with a focused bundled-skills cleanup. The immediate path is to make the frontmatter contract consistent: either remove the redundant `user-invocable: true` from `skills/gh-issues/SKILL.md` because runtime already defaults it to true, or update the skill-creator validator/guidance to accept the runtime-documented optional keys. The follow-up should split bulky orchestration content into `references/` files and tighten the top-level description into trigger-oriented prose.
