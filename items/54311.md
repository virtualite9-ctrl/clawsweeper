## Summary

Keep open. Current main still ships the `gh-issues` skill with the reported top-level `user-invocable` key, the skill is still large at 34,800 bytes / 885 lines, and the bundled skill-creator validator still rejects frontmatter keys outside its small allowlist. The related PR #54360 was closed unmerged, so there is no landed implementation to close against.

## Best Possible Solution

Keep this issue open and resolve it with a small, coherent skills cleanup: either remove the redundant `user-invocable: true` from `skills/gh-issues/SKILL.md` and add validation coverage for bundled skills, or update `skills/skill-creator/scripts/quick_validate.py` to accept OpenClaw-documented frontmatter keys such as `user-invocable` and `disable-model-invocation`. Separately, split the oversized `gh-issues` orchestration details into `references/` files while keeping the top-level skill concise, and add a focused test so the validator, docs, and runtime frontmatter contract stay aligned.
