## Summary

Keep open. Current main still has the central reported problems: the bundled `gh-issues` skill retains `user-invocable: true`, remains a single 34,800-byte `SKILL.md`, and the skill-creator validation allowlist still rejects that frontmatter key. Runtime/docs now recognize `user-invocable`, so the best resolution is contract alignment rather than closing as obsolete.

## Best Possible Solution

Keep this issue open and resolve it with a focused bundled-skills cleanup: decide whether `user-invocable` is a supported public frontmatter key, then align `skills/gh-issues/SKILL.md`, `skills/skill-creator/scripts/quick_validate.py`, skill-creator guidance, public docs, and tests. If the default-true field is redundant in this bundled skill, remove it; separately split the large `gh-issues` orchestration material into `references/` files and keep only essential instructions in `SKILL.md`.
