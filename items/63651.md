## Summary

Keep open. Current main still contains the two direct Windows restart stdout writes, and the lifecycle stale-pid retry path can call the same restart function a second time. The PR is still addressing a real current code path, but it should be cleaned up before merge because the provided PR context shows unrelated browser SSRF changes bundled with the schtasks fix and no focused output de-dup regression test.

## Best Possible Solution

Keep the item open and ask for a focused schtasks-only update: remove or centralize the restart stdout writes in `restartScheduledTask()` and `restartStartupEntry()`, add a regression test that the stale-PID retry path reports a single restart status in human output, and rebase/drop unrelated browser SSRF changes before maintainer review.
