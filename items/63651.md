## Summary

Keep this PR open. Current main still has the Windows schtasks restart stdout writes, and the stale-PID retry path can call the same Windows restart implementation a second time with the same stdout, so the reported duplicate-message behavior is still plausible and not implemented on main.

## Best Possible Solution

Keep this PR open, but ask for a focused gateway-only update: centralize or de-duplicate the schtasks restart output so stale-PID retry reports only one restart status, preserve a human success message for completed Windows restarts, add a targeted regression test around the retry/output behavior, and drop or split unrelated browser SSRF commits before maintainer review.
