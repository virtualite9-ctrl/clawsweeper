## Summary

Keep this PR open. Current main still contains the Windows schtasks restart stdout writes named by the PR, and the stale-PID retry path can call the same restart implementation again with the same stdout, so the duplicate-message behavior remains plausible. The branch needs maintainer cleanup rather than closure because its gateway fix is useful but the PR is unfocused and lacks a targeted regression test.

## Best Possible Solution

Keep this PR open, but ask for a focused gateway-only update: de-duplicate the Windows schtasks restart output across stale-PID retries, preserve exactly one human success message for completed Windows restarts, add a targeted regression test for the retry/output behavior, and drop or split the unrelated browser SSRF commits before maintainer review.
