## Summary

Keep this PR open. Current main still has the Windows schtasks restart stdout writes, and the stale-PID retry path can invoke the same restart implementation again. The PR addresses a plausible real bug, but it is not merge-ready because it is mixed with unrelated browser CDP/SSRF policy changes and a simple two-line deletion could remove the only human completed-restart success line.

## Best Possible Solution

Keep this PR open, but require a focused gateway-only update: rebase onto current main, drop or split the browser CDP/SSRF changes, preserve exactly one human completed-restart success message, de-duplicate stdout across stale-PID retries, and add a targeted regression test for the retry/output behavior.
