## Summary

Keep this PR open. Current main still returns memory corpus supplements in registration order, while adjacent prompt supplements are sorted by plugin id for deterministic behavior. The PR remains a focused implementation candidate with matching regression-test coverage and no apparent security or supply-chain expansion in the provided diff.

## Best Possible Solution

Review or land the focused fix after any needed rebase: sort `listMemoryCorpusSupplements()` by `pluginId` in `src/plugins/memory-state.ts`, and keep regression tests covering reverse registration and same-plugin re-registration stability.
