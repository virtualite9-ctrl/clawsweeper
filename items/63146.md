## Summary

Keep open. Current main still returns memory corpus supplements in registration and re-registration order, and that order affects first-match corpus reads plus loader cache/rollback snapshots. The PR is a narrow implementation candidate with focused tests, and its provided diff does not introduce a security or supply-chain expansion.

## Best Possible Solution

Review or land the focused fix after any needed rebase: sort listMemoryCorpusSupplements() by pluginId in src/plugins/memory-state.ts, and keep regression tests covering reverse registration order plus same-plugin re-registration stability. The current PR is the right narrow place for that work unless maintainers choose a different explicit conflict-resolution contract for overlapping corpus supplements.
