## Summary

Keep this PR open. Current main still returns memory corpus supplements in registration/re-registration order, while adjacent prompt supplements are sorted for cache stability. The PR is a narrow implementation candidate with focused tests and no observed security or supply-chain expansion in the provided diff.

## Best Possible Solution

Review or land the focused fix after any needed rebase: sort listMemoryCorpusSupplements() by pluginId in src/plugins/memory-state.ts, and keep regression tests for reverse registration order plus same-plugin re-registration stability. This PR remains the right narrow place for that work unless maintainers choose a different explicit conflict-resolution contract for overlapping corpus supplements.
