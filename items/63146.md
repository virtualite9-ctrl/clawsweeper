## Summary

Keep this PR open. Current main still returns memory corpus supplements in registration order, while adjacent prompt supplements are sorted for deterministic prompt-cache behavior. The PR remains a focused, relevant fix with matching regression-test intent, and the provided discussion already notes the same risk and scoped validation.

## Best Possible Solution

Keep this PR open and review or land the focused fix after any needed rebase: sort `listMemoryCorpusSupplements()` by `pluginId` in `src/plugins/memory-state.ts`, and keep the two regression tests that lock reverse-registration and re-registration behavior.
