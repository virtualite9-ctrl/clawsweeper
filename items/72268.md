## Summary

Keep this PR open. Current main still has the fragile exec approval metadata regex, and current gateway code can emit the nested `approval-timeout (allowlist-miss)` denial metadata that this PR targets. That payload still fails denied parsing, which affects followup routing and safe denied-message generation. The related #72148 is adjacent but not a superseding fix.

## Best Possible Solution

Keep this PR open and review or land the narrow parser fix: replace the regex metadata parsing with balanced parsing for `Exec denied` and `Exec finished`, keep the nested-parentheses parser and followup-route regression tests, and treat #72148 as a separate user-facing fallback wording change.
