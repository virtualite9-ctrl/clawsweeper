## Summary

Keep this PR open. Current main still has the nested-parentheses exec approval parsing bug, the gateway still emits the exact nested denial metadata, and the PR remains a focused implementation candidate. It needs normal maintainer review, especially for the Aisle security concern around parsing approval wrappers from generic tool output and the unrelated generated baseline diff still present in the live PR diff.

## Best Possible Solution

Keep this PR open for normal maintainer review. Land the balanced-parentheses parser and nested denied/finished followup regression tests, remove unrelated generated baseline churn, and either constrain approval-wrapper parsing to trusted exec approval lifecycle events or add a follow-up hardening change for spoofable generic command output. Related #72148 addressed an adjacent resume-fallback prefix symptom, but it was closed unmerged and does not supersede this parser fix.
