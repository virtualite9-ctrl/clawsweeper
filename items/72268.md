## Summary

Keep #72268 open. Current main still uses the fragile `[^)]*` regex parser for `Exec denied (...)` and `Exec finished (...)`, while the gateway can emit `approval-timeout (allowlist-miss)` metadata with nested parentheses. That means the denied result is misclassified before followup routing and direct fallback formatting. The related #72148 work is adjacent but does not supersede this parser fix.

## Best Possible Solution

Keep the PR open for maintainer review and land this fix or an equivalent one: replace the regex-only metadata parsing with balanced-parentheses parsing, and keep the added regression coverage for nested denied/finished payloads plus denied followup routing.
