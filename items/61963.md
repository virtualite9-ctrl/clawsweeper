## Summary

Current `main` already preserves inbound images for text-only session models and routes image analysis through the media-understanding/imageModel fallback path, which matches the requested no-manual-switch behavior even though it is implemented as offload + media understanding rather than a literal session-model swap inside `resolveGatewayModelSupportsImages`.

## Risks / Open Questions

- The current implementation achieves the requested behavior via attachment offload plus media-understanding/imageModel fallback, not by directly swapping the active session model inside `resolveGatewayModelSupportsImages`; if the reporter specifically wanted native multimodal turn semantics instead of pre-digested image analysis, that distinction could matter.
