## Summary

Keep open. Current main still sends `anthropic-beta: context-1m-2025-08-07` for `params.context1m: true`, still strips that beta for OAuth-style Anthropic tokens, and the docs still describe 1M context as beta-gated. The issue is valid and has open implementation PRs, but the requested Phase 1 migration has not shipped on `main`.

## Best Possible Solution

Keep #45550 open until a maintained PR lands on main. The likely fix belongs in the Anthropic plugin stream wrapper: stop adding `context-1m-2025-08-07` for `context1m`, remove the OAuth-specific context-1m stripping/warning, keep `context1m` as an explicit context-window opt-in for backward compatibility, and update `docs/providers/anthropic.md`, `docs/reference/token-use.md`, and the wrapper/extra-params tests to reflect GA behavior.
