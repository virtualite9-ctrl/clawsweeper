## Summary

Keep open. Current main still propagates Slack `already_reacted` from the shared `reactSlackMessage()` helper and the agent `react` action path. This PR is a focused helper-level fix with regression coverage and no obvious security-sensitive file changes. The open #71843 overlaps but is a narrower call-site fix, so the best maintainer path is to consolidate or choose a canonical PR rather than close this one as obsolete.

## Best Possible Solution

Review and land a single canonical Slack duplicate-reaction fix. The stronger product direction is helper-level idempotence in `extensions/slack/src/actions.ts` with focused tests, because it covers both the agent `react` tool and other `reactSlackMessage()` callers. If maintainers prefer #71843, they should port this PR's exact error matching and helper-level test coverage before closing the duplicate PR.
