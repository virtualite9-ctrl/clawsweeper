## Summary

Current `main` still intentionally caps same-provider rate-limit profile rotation and then escalates to model fallback by default. The reporter's workaround matches the shipped `auth.cooldowns.rateLimitedProfileRotations` knob, but the issue's requested behavior of exhausting the remaining auth-order profiles before fallback is not implemented as the default behavior on `main`, so there is not strong evidence for closure.
