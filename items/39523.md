## Summary

Keep open. Current `main` does not implement Feishu `mediaLocalRoots` config support, and the PR as proposed is incomplete because the normal outbound send path already injects agent-scoped `mediaLocalRoots`, which would override the new config fallback the PR adds.

## Evidence

- **Feishu config schema on main lacks `mediaLocalRoots`:** `FeishuSharedConfigShape` includes `mediaMaxMb` but no `mediaLocalRoots`, so `channels.feishu.mediaLocalRoots` is not currently a valid Feishu config field on `main`.
  - file: [extensions/feishu/src/config-schema.ts:174](https://github.com/openclaw/openclaw/blob/cb2c36b04975e34ab7014a11575d6589aa96ae32/extensions/feishu/src/config-schema.ts#L174)
  - command: `nl -ba extensions/feishu/src/config-schema.ts | sed -n '150,210p'`
  - sha: [cb2c36b04975](https://github.com/openclaw/openclaw/commit/cb2c36b04975e34ab7014a11575d6589aa96ae32)
- **Feishu media send path only uses explicit param roots:** `sendMediaFeishu` still accepts only `mediaLocalRoots?: readonly string[]` and forwards only that param to `loadWebMedia`; there is no fallback to `account.config?.mediaLocalRoots` and no `'any'` handling on current `main`.
  - file: [extensions/feishu/src/media.ts:576](https://github.com/openclaw/openclaw/blob/cb2c36b04975e34ab7014a11575d6589aa96ae32/extensions/feishu/src/media.ts#L576)
  - command: `nl -ba extensions/feishu/src/media.ts | sed -n '568,635p'`
  - sha: [cb2c36b04975](https://github.com/openclaw/openclaw/commit/cb2c36b04975e34ab7014a11575d6589aa96ae32)
- **Normal outbound flow always passes scoped roots into channel dispatch:** The shared outbound send service passes `mediaLocalRoots: mediaAccess.localRoots` into `dispatchChannelMessageAction`, so Feishu channel sends already arrive with a concrete roots array instead of `undefined`.
  - file: [src/infra/outbound/outbound-send-service.ts:102](https://github.com/openclaw/openclaw/blob/cb2c36b04975e34ab7014a11575d6589aa96ae32/src/infra/outbound/outbound-send-service.ts#L102)
  - command: `nl -ba src/infra/outbound/outbound-send-service.ts | sed -n '80,125p'`
  - sha: [cb2c36b04975](https://github.com/openclaw/openclaw/commit/cb2c36b04975e34ab7014a11575d6589aa96ae32)
- **Feishu outbound adapter forwards injected roots to `sendMediaFeishu`:** Feishu's outbound adapter passes `mediaLocalRoots` from channel context directly into `sendMediaFeishu`, which means the PR's proposed `mediaLocalRoots ?? account.config?.mediaLocalRoots` fallback would usually never run in the main outbound path.
  - file: [extensions/feishu/src/outbound.ts:267](https://github.com/openclaw/openclaw/blob/cb2c36b04975e34ab7014a11575d6589aa96ae32/extensions/feishu/src/outbound.ts#L267)
  - command: `nl -ba extensions/feishu/src/outbound.ts | sed -n '230,275p'`
  - sha: [cb2c36b04975](https://github.com/openclaw/openclaw/commit/cb2c36b04975e34ab7014a11575d6589aa96ae32)
- **Outbound channel contract still narrows `mediaLocalRoots` to arrays:** `ChannelOutboundContext` and direct text/media helpers only model `mediaLocalRoots` as `readonly string[]`, so the PR's advertised `'any'` support is not fully represented in the surrounding outbound interfaces.
  - file: [src/channels/plugins/outbound.types.ts:21](https://github.com/openclaw/openclaw/blob/cb2c36b04975e34ab7014a11575d6589aa96ae32/src/channels/plugins/outbound.types.ts#L21)
  - command: `nl -ba src/channels/plugins/outbound.types.ts | sed -n '14,33p'`
  - sha: [cb2c36b04975](https://github.com/openclaw/openclaw/commit/cb2c36b04975e34ab7014a11575d6589aa96ae32)

## Risks / Open Questions

- Closing this PR would drop a still-missing Feishu config capability; current `main` does not already provide it.
- Merging the PR as-is would likely still leave regular outbound Feishu sends ignoring `channels.feishu.mediaLocalRoots`, because the shared outbound path supplies agent-scoped roots first.
- The PR also needs broader plumbing and tests if `'any'` is meant to work reliably through normal outbound delivery.
