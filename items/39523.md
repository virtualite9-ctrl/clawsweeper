## Summary

Keep open. Current main still does not implement Feishu `mediaLocalRoots` as a channel/account config field. Existing shipped work covers runtime propagation of trusted roots into Feishu media sends, but not the PR's config fallback request, and the exact PR needs revision for the current `mediaAccess`/`mediaReadFile` contract.

## Best Possible Solution

Keep this open, but require a revised Feishu-owned fix or a superseding PR. The fix should add the Feishu config field and generated metadata/docs, define whether configured roots merge with or override agent-scoped outbound roots, route Feishu media through the current `mediaAccess`/`mediaReadFile` contract or restrict config to explicit directory arrays, and add regression tests for account-config roots, `"any"` only if still supported, and the normal outbound send path.
