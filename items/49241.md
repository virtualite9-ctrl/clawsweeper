## Summary

Current `main` does not show a dedicated way to target a specific Chrome profile for `existing-session` beyond passing a `userDataDir`. The config/API/docs expose `userDataDir` only, `existing-session` resolution ignores `cdpPort`/`cdpUrl`, and the Chrome MCP launch path forwards only `--userDataDir`, so the requested per-profile targeting is not strongly evidenced as implemented on `main`.

## Evidence

- **Config surface only exposes `userDataDir` for existing-session:** `BrowserProfileConfig` includes `userDataDir` as the only existing-session-specific selector; there is no `profileDirectory`, `chromeProfile`, `verifyAccount`, or similar field in the browser config type.
