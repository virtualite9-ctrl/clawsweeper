## Summary

Current main already fixes the reported 2026.4.24 harness regression. The useful reporter and follow-up comments correctly identified the legacy `embeddedHarness.runtime: "claude-cli"` gap and the workaround; current main now migrates that legacy config to `agentRuntime`, treats `claude-cli` as a CLI backend alias instead of an embedded harness id, avoids passing it as `agentHarnessId` on fallback paths, and auto-enables the Anthropic plugin that owns the `claude-cli` backend. The OAuth token snapshot note in the later comment is a separate auth concern, not the harness-registration failure reported here.

## Best Possible Solution

Close this as implemented on current main. Keep the shipped `agentRuntime`/CLI-backend split, legacy `embeddedHarness` migration, CLI-alias guardrails, and Anthropic plugin owner auto-enable; track any remaining OAuth token refresh/snapshot behavior separately with fresh logs and config details.
