## Summary

Keep open. Current main still routes `openclaw docs <query>` exclusively through the live `https://docs.openclaw.ai/mcp.SearchOpenClaw` MCP endpoint and exits on any tool/transport failure. The report is recent, concrete, macOS-specific, and not disproved by a Linux Node 24 reachability check. The provided related items are about other TLS/gateway surfaces and do not supersede this docs MCP failure.

## Best Possible Solution

Leave this issue open for maintainer triage. Best next step is a macOS reproduction check with the reported Node/LibreSSL class of client. If it still fails, fix the hosted `docs.openclaw.ai` TLS/MCP endpoint or add a source-side `openclaw docs` fallback to a GitHub/docs mirror with targeted regression coverage for transport failures. If macOS no longer reproduces after hosted changes, close later as `cannot_reproduce` with explicit macOS proof.
