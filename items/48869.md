## Summary

Issue #48869 should close as superseded by the active docs PR #39365. Current main has the underlying wake/system-event mechanisms and lower-level docs, but it does not ship the proposed wakeup-scripts page; the remaining docs work is now best tracked in #39365, which covers programmatic agent wake recipes, delivery modes, common mistakes, and script usage. The earlier exact PR #48901 was closed unmerged.

## Best Possible Solution

Close #48869 as superseded by #39365. Keep the remaining documentation work in that PR, and make sure it explicitly says that writing inbox files alone does not wake an agent, recommends `openclaw system event --mode now` or `/hooks/wake`, and includes verification/troubleshooting guidance.
