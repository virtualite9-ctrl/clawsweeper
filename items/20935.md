## Summary

Current main still does not implement the requested audit log for all agent memory-file changes. It has useful append-only JSONL memory events for recall, short-term promotion, and dreaming completion, but generic write/edit paths and memory flush writes can still mutate memory files without an audit entry containing what/why/trigger/session/user context or tamper-evidence.

## Best Possible Solution

Keep this open and implement a first-class memory-change audit layer in OpenClaw, likely owned by memory-core plus shared host/tool write hooks. The implementation should emit append-only records for every mutation to MEMORY.md and canonical memory files, including timestamp, operation, path, content summary or diff, trigger, session/run/user context, and optional integrity/signing. Add regression tests for promotion, memory flush, and ordinary write/edit mutation paths, and document the audit surface in the public memory docs.
