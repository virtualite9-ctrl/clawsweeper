## Summary

Keep #20935 open. Current main still has only a narrow memory host event journal for recall, promotion, and dreaming, plus an unrelated config audit log. It does not audit every MEMORY.md or memory/*.md mutation with what/when/why/session/user context or tamper-evidence. The Pinchy comment is useful external prior art, but it does not satisfy the requested OpenClaw core memory audit surface.

## Best Possible Solution

Keep #20935 open. The best implementation is a first-class memory-change audit layer owned by memory-core plus shared host/tool write hooks, emitting append-only records for every MEMORY.md and canonical memory-file mutation with timestamp, operation, path, changed-content summary or diff, trigger/why, session/run/user context, and optional integrity signing. Add regression tests for promotion, memory flush, dreaming, and ordinary write/edit mutation paths, then document the audit surface in the public memory docs.
