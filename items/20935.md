## Summary

Current main still does not implement the requested audit log for all agent memory-file changes. OpenClaw has a narrow memory event journal for recall, promotion, and dreaming, and Memory Wiki can bridge those events, but generic MEMORY.md or memory-file writes still lack the requested what/when/why/session/user/tamper-evidence audit record. The Pinchy comment is useful prior art, but it is external to OpenClaw and does not satisfy this core memory-change audit surface.

## Best Possible Solution

Keep #20935 open. The best path is a first-class OpenClaw memory-change audit layer owned by memory-core plus shared host/tool write hooks, emitting append-only records for every MEMORY.md and canonical memory-file mutation with timestamp, operation, path, changed-content summary or diff, trigger/why, session/run/user context, and optional integrity signing. Add regression tests for promotion, memory flush, and ordinary write/edit mutation paths, and document the audit surface in the public memory docs.
