## Summary

Keep open. Current `main` still has the exact raw-message path this PR is trying to harden, and the load/normalize path still permits malformed persisted `agentTurn` payloads to survive until that raw value is interpolated into the cron prompt.

## Evidence

- **Detached cron execution still forwards raw payload message:** `executeDetachedCronJob` passes `job.payload.message` directly into `runIsolatedAgentJob` with no runtime extraction or normalization, so a persisted non-string payload value would still flow into the isolated cron run unchanged.
  - file: [src/cron/service/timer.ts:1303](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/src/cron/service/timer.ts#L1303)
  - command: `nl -ba src/cron/service/timer.ts | sed -n '1288,1310p'`
  - sha: [c070509b7f8f](https://github.com/openclaw/openclaw/commit/c070509b7f8fdbe479e5d829643782c272e30e83)
- **Cron prompt construction still coerces non-string input via template literal:** The isolated cron runner builds the prompt with ``[cron:${input.job.id} ${input.job.name}] ${input.message}``, which would stringify an object payload to `[object Object]` if `input.message` is not already a string.
  - file: [src/cron/isolated-agent/run.ts:582](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/src/cron/isolated-agent/run.ts#L582)
  - command: `nl -ba src/cron/isolated-agent/run.ts | sed -n '578,608p'`
  - sha: [c070509b7f8f](https://github.com/openclaw/openclaw/commit/c070509b7f8fdbe479e5d829643782c272e30e83)
- **Payload normalization does not repair object-shaped message fields:** `coercePayload` only trims `next.message` when it is already a string. If persisted data contains `payload.message` as an object, normalization leaves that object in place rather than converting or rejecting it.
  - file: [src/cron/normalize.ts:184](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/src/cron/normalize.ts#L184)
  - command: `nl -ba src/cron/normalize.ts | sed -n '160,205p'`
  - sha: [c070509b7f8f](https://github.com/openclaw/openclaw/commit/c070509b7f8fdbe479e5d829643782c272e30e83)
- **Store load path accepts normalized persisted jobs back into runtime:** `ensureLoaded` reads jobs from disk, runs `normalizeCronJobInput(raw)`, and then uses the normalized object as the hydrated `CronJob`. Because the normalizer does not coerce object-shaped `message` values, malformed persisted jobs can still reach execution on current `main`.
  - file: [src/cron/service/store.ts:37](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/src/cron/service/store.ts#L37)
  - command: `nl -ba src/cron/service/store.ts | sed -n '31,70p'`
  - sha: [c070509b7f8f](https://github.com/openclaw/openclaw/commit/c070509b7f8fdbe479e5d829643782c272e30e83)

That work is not present in the checked-out `main` tree.
