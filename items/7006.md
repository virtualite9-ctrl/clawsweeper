## Summary

Keep #7006 open. Current main still does not capture OpenRouter's resolved response `model` for `openrouter/auto`, does not parse provider-reported `usage.cost` / `cost_details`, and still formats response usage footers from normalized token usage plus local pricing. The ClawRouter comment is useful prior art for an alternate router, but the requester correctly distinguished it from OpenRouter's own auto route. Related #9016 is cost-focused and open; PR #43026 was closed unmerged and its patch is not present here.

## Best Possible Solution

Keep #7006 open. The best implementation path is to capture OpenRouter's response `model` and provider-reported `usage.cost` / `cost_details` in the OpenAI-compatible transport, preserve selected-versus-resolved model metadata in assistant/session records, and thread that through response footers, usage/session/status surfaces, transcripts, and regression tests for `openrouter/auto`. Treat #9016 as related cost-only work and #43026/ClawRouter as useful prior art, not replacements for this issue.
