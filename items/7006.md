## Summary

Keep #7006 open. Current main still treats `openrouter/auto` as the runtime model in the OpenAI-compatible stream path, estimates cost from local model pricing, and does not parse OpenRouter's response `model`, numeric `usage.cost`, or `usage.cost_details`. The ClawRouter comment is useful prior discussion, but it is a different routing service, and the related implementation PR #43026 was closed unmerged while #9016 only covers cost exposure.

## Best Possible Solution

Keep #7006 open. The best fix is to capture OpenRouter's response `model` and provider-reported `usage.cost` / `cost_details` in the OpenAI-compatible transport, preserve selected-versus-resolved model metadata in assistant/session records, and thread that through response footers, usage/session/status surfaces, transcripts, and regression tests for `openrouter/auto`. Treat #9016 as related cost-only work, not a replacement for the resolved-model transparency request; #43026 is useful prior art but is closed unmerged.
