## Summary

Keep #7006 open. Current main still records and displays the requested OpenRouter model plus local estimated costs, not OpenRouter's resolved `openrouter/auto` response model or provider-reported `usage.cost` / `cost_details`. Related #9016 is cost-only, PR #43026 was closed unmerged, and the ClawRouter workaround is useful but does not satisfy the reporter's OpenRouter-specific request.

## Best Possible Solution

Keep #7006 open. The best fix is to capture OpenRouter's response `model` and provider-reported `usage.cost` / `cost_details` in the OpenAI-compatible transport, preserve selected-versus-resolved model metadata in assistant/session records, and thread that through response footers, `/usage`, transcripts, status/session surfaces, and regression tests for `openrouter/auto`. Treat #9016 as related cost-only work, not a replacement for the resolved-model transparency request.
