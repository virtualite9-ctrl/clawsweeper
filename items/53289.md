## Summary

Keep open. Current main has general CLI docs that use the `openclaw` prefix and the linked issue #53253 is closed, but `docs/tools/index.md` still lacks the explicit tools-vs-shell clarification this PR targets. The PR should not merge as-is because it duplicates one sentence and documents invalid `openclaw sessions list` usage; it remains useful if revised to use `openclaw sessions` and keep `openclaw agents list`.

## Best Possible Solution

Keep the PR open or supersede it with a small docs update to `docs/tools/index.md`: add the explicit note that tool identifiers such as `sessions_list` and `agents_list` are not shell commands, remove the duplicate `These tools ship...` sentence, use valid examples (`openclaw sessions` and `openclaw agents list`), and optionally link readers to `https://docs.openclaw.ai/cli`.
