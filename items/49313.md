## Close Comment

Closing from a Codex review: current `main` already implements the requested behavior. The cron isolated-agent path resolves the run agent from `job.agentId` (`src/cron/isolated-agent/run.ts:427`), then resolves outbound identity from that same agent id and passes it into outbound delivery (`src/cron/isolated-agent/delivery-dispatch.ts:506`). The Slack path converts that identity into Slack `username`/`icon_url` overrides (`extensions/slack/src/outbound-adapter.ts:51`) and applies them on `chat.postMessage` (`extensions/slack/src/send.ts:118`). I also checked the shipped `v2026.4.22` snapshot (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`), which already contains this implementation, so this looks fixed on main and released.
