## Summary

Keep this PR open. Current main still undercounts `/status` cost for cache-only or cache-heavy usage: cache counters are collected and displayed, and the shared cost estimator already prices them, but `buildStatusMessage` still gates and calls `estimateUsageCost` with only input/output tokens. The PR is a valid focused fix candidate, but its production hunk needs to be ported from the stale `src/auto-reply/status.ts` surface into `src/status/status-message.ts`.

## Best Possible Solution

Keep this PR open as a focused implementation candidate. Rebase/port the production change into `src/status/status-message.ts` by including `cacheRead` and `cacheWrite` in the `hasUsage` guard and in the `estimateUsageCost` usage object, then keep or add the focused regression in `src/auto-reply/status.test.ts`. Coordinate with #65109 and #52329 so only one refreshed runtime fix lands.
