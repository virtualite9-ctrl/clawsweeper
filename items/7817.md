## Summary

Keep #7817 open. Current main has partial session grouping, parent/child metadata, checkpoint branching, and display-only reply handling, but it still lacks the requested project tree metadata/navigation, composer-level threaded target routing or fanout, arbitrary message split/move triage, and seed-to-branch workflows. The related #55349 is narrower quote-reply work and does not supersede this broader Control UI feature.

## Best Possible Solution

Keep #7817 open as the broad Control UI product feature. A complete solution likely needs project/session-tree metadata in the session contract, additive Gateway protocol methods for split/move/seed and route/fanout targeting, a project tree navigator, composer target controls, and focused UI/protocol tests. Keep #55349 separate as the narrower quote-reply/threaded-reply request.
