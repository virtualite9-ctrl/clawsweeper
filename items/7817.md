## Summary

Keep #7817 open. Current main has useful primitives such as grouped session selects, parent/child session metadata, display-only reply pills, and checkpoint branching, but it still does not implement the requested Control UI project tree navigator, composer-level thread routing/fanout, arbitrary message split/move triage, or seed-elsewhere workflow. The related #55349 remains a narrower quote-reply request and does not supersede this broader feature.

## Best Possible Solution

Keep #7817 open as the broad Control UI product feature. A complete implementation should add additive Gateway protocol support for project/session-tree metadata, composer route targets/fanout, split/move selected messages, and seed/copy branch workflows, then wire those into a project tree navigator with focused UI/protocol tests and docs. Keep #55349 separate as the narrower quote-reply/threaded-reply item.
