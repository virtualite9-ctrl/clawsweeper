## Summary

Current main has not implemented the central Chat page i18n work. The PR is still directionally useful, but it is outdated against the current refactored Chat UI, has review-noted no-op fallback parameters, and only covers the older locale set instead of the current generated locale pipeline. The security review did not find supply-chain or permission-sensitive changes in the provided PR diff.

## Best Possible Solution

Keep this PR open for author or maintainer revision: rebase onto current main, localize the current Chat component split (`ui/src/ui/views/chat.ts` plus relevant `ui/src/ui/chat/*` files), add English source keys, regenerate all supported non-English locale bundles through the Control UI i18n pipeline, and either remove the no-op `fallback` params or implement an explicit fallback API with tests.
