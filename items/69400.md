## Summary

Current main still regenerates gateway service ProgramArguments from the resolved Node/Bun runtime and CLI entrypoint, with no OPENCLAW_WRAPPER handling, no --wrapper install option, and no doctor info path for wrapper-based LaunchAgents. The linked PR #69425 is a direct, open implementation attempt, but it is not merged or shipped, so the underlying issue remains actionable.

## Best Possible Solution

Keep the issue open and review or adapt PR #69425 against current main. The likely product path is to add a small, opt-in wrapper resolver in src/daemon/program-args.ts with regression tests, then decide whether --wrapper persistence and doctor reporting ship in the same PR or as explicit follow-up issues/docs.
