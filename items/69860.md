## Summary

Close #69860 as duplicate/superseded by open PR #60328. Current main still lacks Feishu-native exec approval action cards and `channels.feishu.execApprovals`, but the open PR is the focused canonical implementation path. Existing Feishu card-action support is useful groundwork, not the requested exec approval backend integration.

## Best Possible Solution

Close #69860 as duplicate/superseded and keep implementation, review, and follow-up concentrated on PR #60328 or a successor PR. The right product/code path is to add Feishu `execApprovals` config, native approval runtime/delivery/presentation through the existing channel approval capability seams, Feishu card action resolution for Allow once/Deny, tests, and Feishu/public docs updates there.
