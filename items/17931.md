## Summary

Current main still does not implement automatic read-only per-skill bind mounts. Non-rw sandbox modes still sync eligible skills by copying them into the sandbox workspace, and rw mode still exposes workspace skills through the writable workspace mount. The #22669 custom-bind ordering fix is a useful manual workaround, but it is not the automatic core behavior requested here.

## Best Possible Solution

Keep #17931 open until OpenClaw either implements automatic read-only handling for eligible skill directories in sandboxed sessions or explicitly rejects that product direction. A robust core fix should derive eligible SkillEntry source directories deterministically, add read-only Docker overlays after workspace mounts for rw mode, avoid writable copied skills where a backend can mount sources directly, update sandbox config hashing/recreate behavior, teach fs-bridge/remote paths about the new mount source, and cover the behavior in tests and docs. Treat #22669 as a useful manual workaround, not the canonical fix.
