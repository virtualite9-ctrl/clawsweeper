## Summary

Keep open. Current `main` does not include the Feishu reply-mention fallback from this PR, so the reported bug is not implemented on `main`. Separately, the PR's `rootId`-clearing approach conflicts with current Feishu topic-session behavior and existing tests, so there is not strong evidence to close this as already fixed or obsolete.

## Evidence

- **Current mention detection still lacks reply fallback:** `checkBotMentioned()` on `main` only checks `@_all`, structured `message.mentions`, and rich-text `post` mentions. There is no fallback for reply text containing orphaned `@_user_N` placeholders when Feishu drops the bot from `mentions`.

- The linked Feishu bug still appears plausible on current `main` because reply messages with dropped bot mentions are not handled by `checkBotMentioned()` today.
