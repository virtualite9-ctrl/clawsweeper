## Summary

Keep #39569 open. Current main still lacks the Feishu reply orphan-placeholder mention fallback, so the linked bug #39568 remains a plausible unresolved behavior gap. The PR is useful, but its rootId-clearing approach conflicts with current topic-session routing behavior and tests, so it should be revised rather than cleanup-closed.

## Best Possible Solution

Keep #39569 open as a still-relevant Feishu bugfix path. Rebase it onto the current `bot-content.ts` split, add focused dropped-bot-mention reply tests, make the fallback as bot-specific as Feishu allows, and preserve topic-session `rootId` behavior for `group_topic` / `group_topic_sender` while coordinating the final behavior with #39568.
