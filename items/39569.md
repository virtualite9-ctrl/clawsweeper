## Summary

Keep #39569 open. Current main still lacks the Feishu reply orphan-placeholder mention fallback, so the linked bug remains plausible. The PR is relevant, but should be revised because its broad orphan-key heuristic and blanket rootId clearing conflict with current Feishu topic/session routing expectations.

## Best Possible Solution

Keep #39569 open as a still-relevant Feishu bugfix. Rebase it onto the current `bot-content.ts` split, add focused tests for Feishu reply events where the bot mention is dropped, make the fallback as bot-specific as Feishu data allows, and preserve topic-session `rootId` behavior while coordinating final semantics with #39568.
