## Summary

Keep open. Current `main` still allows `sessions_send` to target thread session keys and the follow-up announce path preserves thread routing, so the requested refusal/warning is not implemented.

## Evidence

- **sessions_send sends to the resolved session key without a thread guard:** After resolving visibility, `createSessionsSendTool` accepts the canonical `resolvedKey` and uses it directly in `sendParams.sessionKey` and later A2A flow setup. There is no `:thread:` rejection or warning branch in this path.
