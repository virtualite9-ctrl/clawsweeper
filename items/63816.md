## Summary

Current main already implements the WhatsApp CLI/gateway outbound media path that was broken in the reporter's 2026.4.9 build: media is preserved through the gateway send request, routed through channel sendMedia, loaded into a WhatsApp media buffer, and sent as image/video/audio/document payloads. The shipped changelog explicitly calls out the v2026.4.10 fix for WhatsApp document and attachment sends silently dropping the file while still delivering the caption; the related #64750 report covers the same 2026.4.9 payload class and is now closed.

## Best Possible Solution

Close this 2026.4.9 WhatsApp CLI media-drop report as implemented by the shipped v2026.4.10 gateway/sendMedia fix. Keep any remaining current-version WhatsApp recipient-side false-success investigation in a fresh or narrower live-reproduction issue with logs, current version, media type, target type, and WhatsApp/Baileys send result details.
