## Summary

Keep this PR open for maintainer follow-up or explicit superseding. Current main already covers the duplicate-webhook part through shared inbound dedupe, but the PR's other requested behavior is not fully implemented or clearly rejected: Google Chat still defaults reply threading to off, add-on lifecycle payload handling and modern space classification are tracked by focused open work, and the submitted diff has security and regression concerns that make it unsafe to merge as-is.

## Best Possible Solution

Do not merge this PR as-is. Keep the shared inbound dedupe already on main, continue the focused review paths for #58993 and #65058, and make a separate maintainer-backed decision on whether Google Chat should default replyToMode to all. If the remaining behavior is still desired, replace this with small current-main PRs with targeted Google Chat tests and a security review for any auth/webhook changes.
