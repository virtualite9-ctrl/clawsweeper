## Summary

Keep this PR open. Current main still ignores Feishu DM `root_id`/`thread_id` for session scoping and thread replies, and current sequential/debounce keys remain chat/root based rather than DM-topic aware. The PR is a focused bundled Feishu channel fix, not a ClawHub/core-cleanup close candidate, but it likely needs rebasing onto the current `monitor.message-handler.ts` and `sequential-key.ts` architecture.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to rebase or port it onto current main by adding DM topic-scoped `peerId`, `parentPeer`, and thread-reply handling in `extensions/feishu/src/bot.ts`, adding `root_id`/`thread_id` aware lanes in `extensions/feishu/src/sequential-key.ts` and debounce keys in `extensions/feishu/src/monitor.message-handler.ts`, preserving main-surface DM behavior, and covering `root_id`, `thread_id`-only, dynamic-agent, and debounce cases in Feishu tests.
