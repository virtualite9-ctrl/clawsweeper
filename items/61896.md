## Summary

Current `main` already covers the reported TUI failure path: gateway-backed TUI runs are now tracked for abort/lifecycle handling, and the TUI has a client-side watchdog that resets stale `streaming`/`running` state back to `idle`. That removes the persistent busy state, which also explains the reported input flicker because busy TUI states trigger periodic redraws.
