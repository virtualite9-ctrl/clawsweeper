## Summary

Keep PR #38170 open. Current main still does not expose a Telegram `download-file` action and still does not put parser-safe `file_id` attributes into Telegram media placeholders. The PR itself is stale because it edits pre-plugin paths that no longer exist, but the underlying retroactive Telegram group-history media workflow remains unimplemented.

## Best Possible Solution

Keep this PR or a refreshed follow-up open, but port the implementation to `extensions/telegram`: expose `download-file` from `channel-actions.ts`, handle it in `action-runtime.ts`, route downloads through the existing Telegram getFile/download transport, and preserve inbound media identifiers in a parser-safe history/body contract with tests and docs.
