## Summary

Keep PR #38170 open. Current main still lacks Telegram `download-file` support and still does not embed parser-safe `file_id` attributes in Telegram media placeholders. The branch is stale because it targets pre-plugin Telegram paths, but the underlying agent workflow remains unimplemented in `extensions/telegram`.

## Best Possible Solution

Keep this PR or a refreshed follow-up open, but port the implementation to `extensions/telegram`: expose `download-file` from `channel-actions.ts`, implement it in `action-runtime.ts` using the existing Telegram transport/media-download helpers, and preserve inbound media identifiers in a parser-safe history/body contract with focused tests and Telegram docs updates.
