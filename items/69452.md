## Summary

Keep open. Current `main` still lacks the requested Anthropic proxy documentation, so this is not `implemented_on_main`, and the PR is not obsolete. The underlying capability is real, but the submitted docs need revision before merge: current code shows `models.providers.anthropic.baseUrl` is honored for both API-key and OAuth-token Anthropic traffic, so the PR's API-key-only caveat is inaccurate, and the example also appears to carry a redundant `api: "anthropic-messages"` field for the bundled `anthropic` provider.
