## Summary

Keep #7648 open. Current main still lacks a Control UI/WebChat setting or config to disable bottom auto-scroll, choose top/off behavior, or jump to the start of a long response. The related #44130 is TUI-specific and open, so it does not supersede this browser/native WebChat request; the latest issue discussion also explicitly kept #7648 open.

## Best Possible Solution

Keep #7648 open and implement an explicit chat scroll preference for Control UI and native WebChat. A conservative fix would add a persisted UI setting and/or documented gateway.webchat/controlUi config, thread it through browser scheduleChatScroll and Swift OpenClawChatView, support bottom-follow plus off/top behavior, optionally add a Scroll to Top control, and cover send, streaming, refresh, initial load, manual-scroll, and native WebChat states with focused tests.
