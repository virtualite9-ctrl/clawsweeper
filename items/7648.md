## Summary

Keep #7648 open. Current main still implements bottom-follow behavior for browser Control UI/WebChat and native WebChat, with only a downward “New messages” affordance and no persisted UI or gateway config option for off/top scrolling. The related #44130 is TUI-specific and does not supersede this browser/native WebChat request.

## Best Possible Solution

Keep #7648 open. A conservative fix should add an explicit chat scroll preference for Control UI/WebChat, likely persisted in UI settings and/or documented gateway config, then thread bottom-follow/off/top behavior and optionally a Scroll to Top button through scheduleChatScroll, Control UI rendering, and OpenClawChatView. Cover send, streaming, refresh, initial load, manual-scroll, and native WebChat states with focused tests.
