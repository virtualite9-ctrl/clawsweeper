## Summary

Keep open. Current main has zh-CN locale loading and translated top-level navigation, but the Settings/Config form surface still renders many section labels, form controls, schema labels/help text, placeholders, and error messages from hard-coded English strings or English-only schema/uiHints metadata. The related title-search items concern duplicate settings headers, not Chinese localization, so they do not supersede this request.

## Best Possible Solution

Keep the issue open and implement a scoped Control UI i18n pass for Settings/Config: move config.ts and config-form.* chrome, section/category metadata, action buttons, placeholders, validation/errors, and schema/uiHints labels/help into a locale-aware translation surface; preserve technical identifiers where appropriate; regenerate zh-CN generated bundles and metadata; add tests that render the Settings form under zh-CN and verify representative section labels and controls no longer fall back to English.
