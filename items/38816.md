## Summary

Keep #38816 open. Current main still resolves providerless `agents.defaults.imageModel.primary` and fallback entries through `DEFAULT_PROVIDER`, so a bare `moondream` can still become `anthropic/moondream`. The adjacent configured-provider fixes cover provider-prefixed/configured rows, but not the reported providerless imageModel case.

## Best Possible Solution

Keep the issue open until OpenClaw either infers a unique configured provider for bare `agents.defaults.imageModel.primary` and fallback entries using the same configured-model evidence as default chat model resolution, or explicitly rejects providerless image model refs with a helpful validation/runtime error and docs. Add a regression test for a bare Ollama/LiteLLM-style image model such as `moondream` or `qwen2.5vl:7b` configured under `models.providers`.
