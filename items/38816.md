## Summary

Keep #38816 open. Current main still lets a providerless `agents.defaults.imageModel.primary` choose the image fallback default provider by parsing the bare model id against `DEFAULT_PROVIDER`, so the reported `moondream` path can still become `anthropic/moondream`. Adjacent image-provider fixes and docs cover explicit `provider/model` refs and configured provider metadata, but not the providerless imageModel primary/fallback case reported here.

## Best Possible Solution

Keep the issue open until OpenClaw either infers a unique configured provider for bare `agents.defaults.imageModel.primary` and fallback entries using the same configured-model evidence as default chat model resolution, or explicitly rejects providerless image model refs with validation, a helpful error, and documentation. Add a regression test for a bare Ollama/LiteLLM-style image model such as `moondream` or `qwen2.5vl:7b` configured under `models.providers`.
