## Summary

Keep #38816 open. Current main still resolves providerless `agents.defaults.imageModel.primary` and fallback entries through the image fallback default provider, which is derived by parsing the configured image primary against `DEFAULT_PROVIDER`. A bare `moondream` can therefore still become `anthropic/moondream`; adjacent fixes and docs cover explicit `provider/model` refs, but not the providerless imageModel path reported here.

## Best Possible Solution

Keep the issue open until OpenClaw either infers a unique configured provider for bare `agents.defaults.imageModel.primary` and fallback entries using the same configured-model evidence used by default chat model resolution, or explicitly rejects providerless image model refs with validation and documentation. The current safe workaround remains explicit refs such as `ollama/qwen2.5vl:7b`.
