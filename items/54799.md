## Summary

Keep this issue open. Current main supports ordinary skill metadata, skill visibility/status, and separate cron jobs, but it does not implement declarative SKILL.md schedule/event trigger metadata, automatic skill-to-cron registration, last-triggered skill status, or a skills audit command. The provided related context covers nearby frontmatter robustness and model-routing work, but none is a canonical shipped tracker for this trigger contract.

## Best Possible Solution

Keep the issue open for maintainer product/API design. A good path is either a versioned core skill trigger contract such as `metadata.openclaw.triggers` with validation, cron reconciliation, last-triggered status, CLI audit output, docs, and tests, or an explicit plugin-first design that exposes enough stable skill loader/status APIs for a ClawHub plugin to implement the behavior without private core imports.
