## Summary

Keep open. Current `main` already covers the PR's secondary `npm.cmd` fallback case, but it does not implement the primary fix for resolved Windows `node.exe`/`npm-cli.js` paths that contain spaces. The reported bug still has a real code-path gap on `main`.

## Evidence

- **Primary space-in-path bug still present on main:** `resolveChildProcessInvocation()` resolves `npm`/`npx` to `[process.execPath, cliPath, ...]`, then only enables the `cmd.exe` wrapper when `resolvedCommand` ends in `.cmd` or `.bat`. There is no branch that wraps resolved `node.exe` + `npm-cli.js` when either path contains spaces, which is the main bug this PR fixes.
  - file: [src/process/exec.ts:97](https://github.com/openclaw/openclaw/blob/02a9dd0ddc59ec2efc4f00803ee6fe839015b15f/src/process/exec.ts#L97)
  - command: `nl -ba src/process/exec.ts | sed -n '40,120p'`
  - sha: [02a9dd0ddc59](https://github.com/openclaw/openclaw/commit/02a9dd0ddc59ec2efc4f00803ee6fe839015b15f)
- **Secondary fallback bug is already handled differently on main:** When `npm-cli.js` is missing, `resolveNpmArgvForWindows()` now returns `npm.cmd`/`npx.cmd`, so current `main` already routes that case through the existing `cmd.exe` wrapper and avoids the older direct-spawn `EINVAL` path.
  - file: [src/process/exec.ts:63](https://github.com/openclaw/openclaw/blob/02a9dd0ddc59ec2efc4f00803ee6fe839015b15f/src/process/exec.ts#L63)

There is coverage for the `.cmd` fallback, but no coverage for wrapping `C:\Program Files\nodejs\node.exe` / `npm-cli.js` via `cmd.exe`.
