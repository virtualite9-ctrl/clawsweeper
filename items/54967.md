## Summary

Keep open. Current `main` still does not attach pending-history Feishu media to inbound context, so the PR is not obsolete; the existing review discussion also identifies unresolved correctness issues in the proposed implementation rather than a reason to close it.

## Evidence

- **Pending history stores pre-mention turns verbatim from `ctx.content`:** When `requireMention` is enabled and the bot is not mentioned, Feishu group messages are buffered into pending history with `body: `${ctx.senderName ?? ctx.senderOpenId}: ${ctx.content}``, preserving whatever `ctx.content` contains for later replay.
  - file: [extensions/feishu/src/bot.ts:479](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/extensions/feishu/src/bot.ts#L479)
  - command: `nl -ba extensions/feishu/src/bot.ts | sed -n '470,520p'`
  - sha: [c070509b7f8f](https://github.com/openclaw/openclaw/commit/c070509b7f8fdbe479e5d829643782c272e30e83)
- **Non-text Feishu messages are normalized to raw JSON content on current main:** `parseMessageContent` only special-cases `text`, `share_chat`, `merge_forward`, and `post`; for image/file/audio/video/media messages it falls through to `return content`. That means pending history for screenshots/files is typically raw Feishu JSON such as payloads containing `image_key`/`file_key`, not `<media:...>` placeholders.
  - file: [extensions/feishu/src/bot-content.ts:132](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/extensions/feishu/src/bot-content.ts#L132)
  - command: `nl -ba extensions/feishu/src/bot-content.ts | sed -n '132,165p'`
  - sha: [c070509b7f8f](https://github.com/openclaw/openclaw/commit/c070509b7f8fdbe479e5d829643782c272e30e83)
- **Current main only resolves media from the triggering message:** Inbound media resolution in `handleFeishuMessage` is limited to `ctx.messageId` / `event.message.content` for the current message. There is no follow-up pass over pending group history to fetch earlier screenshots or files before dispatch.
  - file: [extensions/feishu/src/bot.ts:717](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/extensions/feishu/src/bot.ts#L717)
  - command: `nl -ba extensions/feishu/src/bot.ts | sed -n '712,735p'`

The PR's plumbing for reusing raw message bodies is therefore not already implemented on `main`.
