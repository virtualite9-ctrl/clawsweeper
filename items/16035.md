## Summary

Keep #16035 open. Current main still has only adjacent transparency mechanisms: PR-level AI disclosure guidance, configurable agent identity for OpenClaw prompts/messages, delegate-account guidance, and bot-like commit author names in workflows. It does not implement default GitHub profile-level disclosure for OpenClaw-managed agents, and the latest issue discussion already records maintainer-side Codex review keeping it open for product judgment.

## Best Possible Solution

Keep #16035 open for maintainer/product judgment. If OpenClaw-managed GitHub agents must disclose agent status at the profile level, add explicit account setup docs and enforceable checks/hooks in the GitHub automation surfaces that create issues, comments, or PRs; keep existing PR-scoped transparency guidance as adjacent but insufficient.
