## Summary

Keep open. Current main still lacks ACP backend failover and an `acp.fallbacks` config surface, so the PR is not obsolete or implemented. The branch needs maintainer review and cleanup before merge, but the underlying core reliability request remains unresolved.

## Best Possible Solution

Keep the PR open, but require a focused ACP-only revision if maintainers want backend-level failover: add a tested `acp.fallbacks` contract, preserve current single-backend failure bookkeeping, avoid retrying after streamed output or side effects, define whether init-stage unavailable failures are eligible, close or safely dispose old backend handles during switching, and split unrelated smoke/runtime-deps fixes into separate PRs.
