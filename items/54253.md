## Summary

Keep open. Current `main` still intentionally copies ambient proxy variables into supervised gateway/node service environments, which matches the issue-thread workaround of removing proxy lines from `~/.config/systemd/user/openclaw-gateway.service`. I did not find a RISC-V-specific fix, a changelog entry for this issue, or other strong evidence that `main` already resolves the reported `LLM request failed: network connection error` behavior.
