## Summary

Keep #18160 open. Current main still does not implement a direct exec/shell cron payload: cron types, Gateway schema, validation, runtime dispatch, CLI, UI, and docs remain limited to systemEvent and agentTurn. The linked implementation PR #51276 was closed unmerged, and the latest issue discussion already treats #18160 as the substantive core tracker.

## Best Possible Solution

Keep #18160 open as the core cron direct-exec tracker. The best implementation path is an additive exec/shell payload contract across cron types, Gateway schema and validation, normalization, runtime dispatch, CLI add/edit, Control UI, docs, and focused tests, with a controlled process runner and clear security/delivery semantics. Related broader direct-tool or conditional-pipeline issues should remain separate follow-ups rather than replacing this focused cron payload request.
