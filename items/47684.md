## Summary

Current main does not implement the requested bulk auth sync, auto-sync, or clear shadowing warning for config env API-key rotation. The default model auth path still resolves auth profiles before env credentials, and the referenced PR #57371 was closed unmerged per the provided GitHub context.

## Best Possible Solution

Keep this issue open. The best maintainer path is to add a scoped core workflow around model auth profile rotation, likely a `openclaw models auth sync`/repair command or a config-write/models-status warning that detects env/config credentials shadowed by per-agent profiles. It should cover all configured agent dirs, preserve SecretRef/OAuth semantics, and include tests proving either profile updates/scrubbing or explicit warnings for stale profile-vs-env divergence.
