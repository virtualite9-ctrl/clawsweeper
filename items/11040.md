## Summary

Keep #11040 open. Current main still only has partial lineage fields (`spawnedBy`, `parentSessionKey`, `spawnDepth`) and does not implement the requested first-class `rootSessionKey` plus stable session-chain `traceId` contract in the persisted session model, gateway protocol/list rows, agent-facing `sessions_list`, or native/ACP spawn write paths. PR #11043 attempted this work but was closed unmerged, and the latest issue discussion already contains a maintainer Codex review saying to keep it open.

## Best Possible Solution

Keep the issue open until core implements the additive Phase 1 chain contract across `SessionEntry`, gateway schemas/list rows/events, native and ACP spawn write paths, agent-facing `sessions_list`, UI/shared row types where needed, and multi-hop regression tests. If maintainers decide the current `spawnedBy`/`parentSessionKey`/`spawnDepth` model is the intended boundary, they should close or narrow the issue explicitly with that product rationale.
