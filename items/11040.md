## Summary

Keep #11040 open. Current main has partial lineage support (`spawnedBy`, `parentSessionKey`, `spawnDepth`, `childSessions`, descendant registry helpers), but it still does not implement the requested first-class session-chain contract: no persisted `rootSessionKey`, no session-chain `traceId`, no gateway protocol/list/event fields for them, and no agent-facing `sessions_list` output for the full parent/root/trace/depth set. The related implementation PR #11043 was closed unmerged, and the existing issue discussion already records a keep-open Codex review.

## Best Possible Solution

Keep #11040 open until core implements the additive Phase 1 session-chain metadata across `SessionEntry`, gateway protocol schemas/list rows/events, native and ACP spawn write paths, agent-facing `sessions_list` output, shared/UI row types where needed, docs, and multi-hop regression tests. If maintainers decide the existing direct-child model plus subagent registry traversal is the intended boundary, close it explicitly with that rationale rather than through conservative cleanup.
