## Summary

Keep #11040 open: current main still implements only partial session lineage, while the requested canonical `rootSessionKey` and stable session-chain `traceId` are absent from the persisted session model, gateway protocol/list rows, agent-facing `sessions_list` output, and spawn write paths.

## Best Possible Solution

Keep the issue open until core either implements the additive Phase 1 chain contract across `SessionEntry`, gateway schema/list rows, native and ACP spawn/write paths, agent-facing `sessions_list`, UI/shared row types, and multi-hop regression tests, or a maintainer explicitly narrows the product direction to the existing `spawnedBy`/`parentSessionKey`/`spawnDepth` model.
