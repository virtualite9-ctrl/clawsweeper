## Summary

Keep PR #69701 open. Current main still has the bind behavior the PR targets: loopback bind resolution can return both `127.0.0.1` and `::1`, gateway startup listens on every returned host, and the listener path does not request IPv6-only behavior. The linked bug #69674 was closed by a separate probe-ordering fix, but that does not implement or test this Windows bind guard, so the PR may still contain useful targeted gateway work.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to validate the Windows-specific bind/root-cause claim, then either land the narrow `resolveGatewayListenHosts` guard with the focused unit tests or replace it with an `ipv6Only`/listener-level fix if maintainers want to preserve Windows `::1` support explicitly.
