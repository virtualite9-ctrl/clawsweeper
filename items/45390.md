## Summary

Keep #45390 open. Current main has partial mitigations: daily/idle session reset policies, thread-binding max-age for focused thread workflows, and a pre-compaction memory flush. It still lacks the requested general configurable session max lifetime plus max-turn TTL, and automatic stale-session rollover does not run the same pre-rotation persistence path requested in the issue discussion.

## Best Possible Solution

Keep #45390 open as the broad core session-lifecycle tracker unless maintainers intentionally consolidate it. A complete fix should add a documented and schema-validated general session TTL/max-turn policy, wire it into automatic rollover, emit/log a clear rotation reason, run a pre-rotation persistence hook or memory flush before stale-session rollover, and cover daily/idle/heartbeat/cron interactions with regression tests and docs.
