## Summary

Keep open. Current `main` still has the generic health-monitor restart loop, cooldown, and hourly cap, but no escalation path or Discord-specific recovery once repeated post-connection churn keeps happening. The latest comment also supplies same-day current-version evidence on `v2026.4.23`, and the timeline links an open follow-up PR (`#40413`), so there is not strong evidence for an auto-close.
