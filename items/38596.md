## Summary

Issue #38596 should stay open. Current main at 631552c554095b0c520376db9131dac0f5427f94 still has generic health-monitor restart/cooldown/hourly-cap behavior, but I did not find monitor-level repeated-restart escalation, Discord clean-restart/fresh-IDENTIFY wiring across health-monitor restarts, or reconnect catch-up behavior that would resolve the reported post-connection zombie loop. The issue discussion also includes current-version production evidence from 2026.4.23 and open linked PR #40413, which explicitly targets the missing behavior.

## Best Possible Solution

Keep #38596 open until a maintainer lands or explicitly rejects a focused Discord recovery design. The likely path is to evaluate PR #40413 or a narrower replacement: add monitor-level repeated-restart escalation, expose a Discord clean-restart/fresh-IDENTIFY path across the channel boundary, reset escalation after a stable period, and add regression tests for health-monitor restart escalation plus Discord lifecycle recovery. Treat related ingest-lag reports such as #71546 as supporting evidence, not as a replacement for this monitor-escalation issue.
