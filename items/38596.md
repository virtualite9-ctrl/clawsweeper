## Summary

Keep open. Current main still has generic health-monitor restart/cooldown/hourly-cap behavior, but not the repeated-restart escalation, Discord clean-restart/fresh-IDENTIFY handoff, or reconnect catch-up behavior requested by #38596. The supplied issue context also includes current-version production evidence and open PR #40413, which explicitly proposes the missing implementation.

## Best Possible Solution

Keep #38596 open until a maintainer lands or rejects a focused Discord recovery design. The likely path is to evaluate PR #40413 or a narrower replacement: add monitor-level repeated-restart escalation, expose a Discord clean-restart/fresh-IDENTIFY path across the channel boundary, reset escalation after a stable period, and cover it with health-monitor plus Discord lifecycle regression tests. Treat #71546 as related ingest-lag evidence, not a replacement for this monitor-escalation issue.
