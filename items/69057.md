## Summary

Keep open. Current `main` still contains Windows-unsafe `SIGUSR1` restart paths, so there is not strong evidence this issue was fixed or became obsolete. The linked workaround/comment and cross-reference to #67416 support that this is a real, still-relevant Windows restart bug rather than a stale report.

## Evidence

- **In-process restart still falls back to SIGUSR1 self-kill with no Windows guard:** `emitGatewayRestart()` still calls `process.kill(process.pid, "SIGUSR1")` when no SIGUSR1 listener is registered. On Windows, `SIGUSR1` is not supported, which matches the reported `ERR_UNKNOWN_SIGNAL` failure mode.
  - file: [src/infra/restart.ts:135](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/src/infra/restart.ts#L135)
  - command: `nl -ba src/infra/restart.ts | sed -n '120,140p'`
  - sha: [7f6452897e25](https://github.com/openclaw/openclaw/commit/7f6452897e25543c9263b67aabf140379d1c0b0c)
- **CLI unmanaged restart path still signals SIGUSR1 directly:** `openclaw gateway restart` for an unmanaged gateway still reaches `signalVerifiedGatewayPidSync(..., "SIGUSR1")` with no Windows-specific fallback. That is a second current-main path where Windows restart can fail for the same platform reason.
  - file: [src/cli/daemon-cli/lifecycle.ts:135](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/src/cli/daemon-cli/lifecycle.ts#L135)
  - command: `nl -ba src/cli/daemon-cli/lifecycle.ts | sed -n '124,139p'`
  - sha: [7f6452897e25](https://github.com/openclaw/openclaw/commit/7f6452897e25543c9263b67aabf140379d1c0b0c)
- **Signal helper is platform-agnostic and forwards SIGUSR1 to process.kill:** `signalVerifiedGatewayPidSync()` validates the PID, then unconditionally calls `process.kill(pid, signal)`. There is no Windows guard or alternate restart mechanism here.
  - file: [src/infra/gateway-processes.ts:40](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/src/infra/gateway-processes.ts#L40)
  - command: `nl -ba src/infra/gateway-processes.ts | sed -n '35,40p'`
  - sha: [7f6452897e25](https://github.com/openclaw/openclaw/commit/7f6452897e25543c9263b67aabf140379d1c0b0c)
- **Existing tests avoid the failing branch instead of covering Windows SIGUSR1 behavior:** `restart.deferral-timeout.test.ts` explicitly adds a SIGUSR1 listener so `emitGatewayRestart()` uses `process.emit` instead of `process.kill`, which means the Windows-unsafe fallback is not exercised there.
  - file: [src/infra/restart.deferral-timeout.test.ts:8](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/src/infra/restart.deferral-timeout.test.ts#L8)
