## Summary

Keep PR #69464 open. Current main still lacks the PR's central behavior: `models.providers.<id>.models[].params` is not accepted by the strict model schema, provider runtime models do not carry `params`, configured model resolution does not thread per-model params onto the runtime model, and both native Ollama and Ollama OpenAI-compatible paths still derive `options.num_ctx` from `contextWindow`/`maxTokens` rather than `params.num_ctx`. The related PR #44678 is closed and unmerged, so it is not a canonical shipped replacement.

## Best Possible Solution

Keep #69464 open for maintainer review or land an equivalent fix: add the optional provider-model `params` schema/type surface, thread per-model params from `agents.defaults.models` and `models.providers.*.models[]` onto the resolved runtime model with explicit user-config precedence, make native and OpenAI-compatible Ollama prefer validated `params.num_ctx`, and keep the accompanying docs/tests. #44678 should remain historical context, not the canonical replacement, because it was closed unmerged and covered less of the remaining behavior.
