## Summary

Keep #69464 open. Current main still does not implement the PR's central fix: configured provider models cannot carry `params`, resolved runtime models do not preserve model params, and Ollama still derives `options.num_ctx` from `contextWindow` rather than `params.num_ctx`. The PR remains a plausible, focused bug fix with related review feedback already addressed in the provided discussion context.

## Best Possible Solution

Keep the PR open and either merge or adapt its approach after maintainer review: add `params?: Record<string, unknown>` to provider model config/schema/generated output and `ProviderRuntimeModel`, thread params from both `agents.defaults.models["provider/id"].params` and `models.providers.<id>.models[].params` with explicit provider-model config winning, and make both native and OpenAI-compatible Ollama payload builders honor a positive finite floored `params.num_ctx` while preserving current `contextWindow` fallback behavior. Retain focused regression tests for schema acceptance, params threading, precedence, native Ollama, compat Ollama, and invalid values, plus the Ollama docs update.
