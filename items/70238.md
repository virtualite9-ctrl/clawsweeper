## Summary

Keep #70238 open. Current main makes the original `src/entry.ts` respawn theory partly obsolete because Windows CLI respawn is skipped, and PR #70308 was closed unmerged, but the broader Windows gateway flicker report is still actionable. Current main still has reachable/default Windows console-flicker risk through the bundled Bonjour/@homebridge/ciao discovery path already tracked by #70197, plus a TUI relaunch spawn path called out in the discussion that still lacks `windowsHide`.

## Best Possible Solution

Keep #70238 open until maintainers either reproduce and fix the exact Windows startup flicker path or deliberately consolidate it with the narrower Bonjour/@homebridge/ciao tracker #70197. The likely fix path is to hide or remove every reachable OpenClaw-owned Windows process spawn that can create a console window, and separately patch, upgrade, or policy-gate the bundled Bonjour/@homebridge/ciao ARP polling path so default LAN discovery no longer flashes `cmd.exe`. Verification should include a Windows run with process-birth/UI proof and targeted regression coverage where feasible.
