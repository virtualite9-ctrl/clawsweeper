## Summary

Keep #70238 open. Current main still does not prove the Windows flashing command-prompt regression is fixed: the related fix PR #70308 was closed unmerged, the cited OpenClaw-owned TUI relaunch spawn still lacks `windowsHide`, and the bundled default Bonjour path still uses `@homebridge/ciao` 1.3.6, whose Windows ARP probe runs through `cmd.exe` without hidden-window options.

## Best Possible Solution

Keep #70238 open for a Windows-focused fix pass. Identify the exact process creating the flashing window, add `windowsHide: true` to any OpenClaw-owned background spawn surface that can run on Windows, and if the source is Bonjour/@homebridge/ciao ARP probing, upgrade or patch that dependency path. Add mockable regression coverage for spawn options and treat `OPENCLAW_DISABLE_BONJOUR=1`, `openclaw plugins disable bonjour`, or `discovery.mdns.mode="off"` only as temporary mitigations.
