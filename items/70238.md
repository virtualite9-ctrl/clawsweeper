## Summary

Keep #70238 open. The report is recent, has multiple confirmations, and current main still has unresolved Windows-visible-process surfaces: the proposed #70308 `windowsHide` patch is closed unmerged, the TUI relaunch spawn still lacks `windowsHide`, and the default Bonjour path remains active while the ciao/ARP console-flash tracker (#70197) remains open.

## Best Possible Solution

Keep #70238 open for a Windows-focused fix pass. Identify the process creating the visible window, add `windowsHide: true` to any OpenClaw-owned background spawn surface that can run on Windows, and if the source is the bundled Bonjour/@homebridge/ciao ARP probe, patch or upgrade that dependency path. Add mockable regression coverage for spawn options, and treat `OPENCLAW_DISABLE_BONJOUR=1` / `discovery.mdns.mode="off"` only as temporary mitigation documentation.
