## Summary

Current `main` resolves the reported no-prompt exec blockage by skipping exec script preflight in gateway YOLO mode (`tools.exec.security=full` + `tools.exec.ask=off`). The requested new config keys were not added, but the underlying behavior is now implemented on `main` while keeping the fail-closed preflight for stricter modes.

## Evidence

- **YOLO-mode skip helper exists on main:** `shouldSkipExecScriptPreflight` returns true for `host === "gateway"`, `security === "full"`, and `ask === "off"`.
  - file: [src/agents/bash-tools.exec.ts:1046](https://github.com/openclaw/openclaw/blob/51c11cfd903033947f01f96a7169ebf92d194064/src/agents/bash-tools.exec.ts#L1046)

- **Exec path now gates preflight on that helper:** The main exec flow only runs `validateScriptFileForShellBleed(...)` when `shouldSkipExecScriptPreflight(...)` is false, so gateway YOLO mode bypasses the complex-interpreter preflight rejection layer.
