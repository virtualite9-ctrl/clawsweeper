## Summary

Keep PR #60696 open. Current main still has the routing behavior this PR is trying to change: completion routing calls the bound delivery router with `failClosed: false`, the router still selects a single active binding when no requester is present, and current tests only cover fail-closed behavior for multiple bindings without a requester. The PR appears to be a focused bug fix with targeted regression coverage rather than obsolete or superseded work.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to rebase or update #60696 against current main, preserve the targeted regression coverage, and merge it if CI confirms the fail-closed completion routing behavior is intended.
