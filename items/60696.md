## Summary

Keep open. Current `main` has related hardening in subagent completion delivery, but it does not contain this PR's core semantic changes: completion routing still calls the bound router with `failClosed: false`, and announce flow still resolves completion delivery from `directOrigin` rather than the original requester signal. That is not strong enough evidence to treat PR #60696 as already implemented on `main`.
