## Summary

Current `main` already guards heartbeat runs from interrupting active user-facing turns: heartbeat execution skips when the main queue or the resolved session lane is busy, the wake layer retries `requests-in-flight` wakes later, regression tests cover the busy-session case directly, and the same behavior is present in release `v2026.4.23`.

## Evidence

- **Heartbeat runner skips busy main and session lanes:** `runHeartbeatOnce()` returns `requests-in-flight` when either the main command lane or the resolved embedded session lane has queued work, explicitly to avoid interrupting an active streaming turn.

- **Regression test matches the reported scenario:** The dedicated test seeds a heartbeat-targeted Telegram session, makes the main lane idle and the `session:*` lane busy, then asserts the heartbeat run is skipped with `requests-in-flight` and that no reply is generated.
