## Summary

Keep open. Current `main` already contains runtime logic that preserves accumulated Ollama `tool_calls` after earlier thinking/text chunks, and it has adjacent regression coverage, but it does not appear to include the three specific `thinking` + `tool_calls` streaming scenarios this PR adds. That is not strong enough evidence to close the PR as already implemented.

## Evidence

- **Current stream logic preserves accumulated tool calls into the final assistant message:** The Ollama stream adapter closes thinking/text blocks, appends intermediate `tool_calls`, then writes accumulated thinking and tool calls back onto the final response before emitting `done`.
