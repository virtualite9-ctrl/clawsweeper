## Summary

Keep this PR open. Current main does not implement importance-weighted temporal decay, and the PR is a plausible feature contribution, but it needs maintainer review and a rebase onto the current memory-core plugin architecture before it can land.

## Best Possible Solution

Keep this PR open, but require a rebase onto the current memory-core plugin layout. The feature should be implemented in extensions/memory-core/src/memory/temporal-decay.ts and threaded through extensions/memory-core/src/memory/hybrid.ts, extensions/memory-core/src/memory/manager.ts, src/agents/memory-search.ts, runtime config types/schemas, generated config artifacts, docs, and focused tests. The unrelated periodicExtraction schema change should be removed or fully wired in a separate change.
