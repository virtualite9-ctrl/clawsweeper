## Summary

Keep open. Current main does not implement per-agent Bedrock `requestMetadata`, and the PR still contains useful but not merge-ready work. The linked feature requests remain open, while PR review surfaced concrete issues that need fixes rather than an auto-close: zero-config wrapping, raw agent metadata validation, merge precedence, and unrelated Feishu changes in the same branch.

## Best Possible Solution

Keep this PR open for maintainer review or author revision. The best path is to split out the unrelated Feishu changes, implement Bedrock request metadata either in the bundled `amazon-bedrock` provider or a small core-supported provider hook that receives agent identity, validate AWS-safe metadata keys/values, preserve existing caller/config option precedence, and add focused tests for zero-extra-params Bedrock agents, sanitized agent names, metadata merging, and non-Bedrock no-op behavior. Coordinate the final scope with open #60602 and #57360 rather than closing this useful but incomplete PR automatically.
