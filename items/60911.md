## Summary

Keep open. Current `main` still lacks a `sessions_send` sender/display identity field, persists only inter-session provenance on the injected user message, and the web chat renderer still uses the local human-user avatar for any `role: "user"` message group. Static inspection supports the report; there is no strong evidence the feature is already implemented.

## Evidence

- **sessions_send API has no sender/display identity field:** The tool schema on current `main` only accepts `sessionKey`, `label`, `agentId`, `message`, and `timeoutSeconds`; there is no `sender`, `displayName`, or avatar-related parameter.
  - file: [src/agents/tools/sessions-send-tool.ts:37](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/src/agents/tools/sessions-send-tool.ts#L37)
  - command: `nl -ba src/agents/tools/sessions-send-tool.ts | sed -n '30,150p'`
  - sha: [6bc0dc8fb671](https://github.com/openclaw/openclaw/commit/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440)
- **sessions_send only tags provenance on the injected user turn:** When `sessions_send` calls the target agent, it sends `inputProvenance` with `kind: "inter_session"`, `sourceSessionKey`, `sourceChannel`, and `sourceTool`, but no display identity metadata.
  - file: [src/agents/tools/sessions-send-tool.ts:275](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/src/agents/tools/sessions-send-tool.ts#L275)

- **Persisted user messages keep provenance only, not senderLabel/avatar:** `applyInputProvenanceToUserMessage()` adds a `provenance` object to `role: "user"` messages and does not populate `senderLabel`, `displayName`, or avatar fields from that provenance.
