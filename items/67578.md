## Summary

Keep open. Current main has useful Bonjour/ciao mitigations for cancellation rejections and one IPv4 interface assertion, plus Docker/workaround docs, but it does not cover the issue's reported uncaught `IP address version must match / Netmask cannot...` assertion path. The issue also has fresh 2026.4.24 comments showing the same ciao crash family remains visible to users.

## Best Possible Solution

Keep the issue open and harden the bundled `bonjour` plugin so malformed or inconsistent mDNS packets cannot terminate the gateway. The likely path is to add explicit coverage for the `IP address version must match` / netmask assertion, cover reverse interface assertion variants, ensure ciao packet-handler failures route through a non-fatal plugin-level path rather than `uncaughtException`, and add regression tests around those exact messages. The existing disable-Bonjour workaround and Docker defaults should remain documented as deployment mitigations, not the final fix.
