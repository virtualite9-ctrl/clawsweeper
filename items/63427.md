## Summary

Keep open. The reporter's follow-up correctly narrowed this from a single GatewayClient reconnect bug to repeated fresh CLI probe sessions, and current main still lacks the proposed per-URL device-required short-circuit/cache or an equivalent caller-side liveness path for the affected WebSocket probes.

## Best Possible Solution

Keep the issue open or explicitly supersede it only once a canonical open item/PR is confirmed. The product fix should be narrow: add a per-gateway-url device-required short-circuit/cache in src/gateway/probe.ts, or switch liveness-only CLI restart/status checks to the existing HTTP /healthz-style probe where read/auth proof is not needed, then add regression tests for repeated DEVICE_IDENTITY_REQUIRED probes and cache reset on successful connection.
