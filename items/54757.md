## Summary

Keep this issue open. Current main still has a path where final tool-error warning payloads can be delivered to WhatsApp replies, and the available suppression option is not wired into WhatsApp inbound auto-reply runs.

## Best Possible Solution

Keep this issue open and implement an external-recipient-safe tool-error policy for WhatsApp and similar channels. The likely narrow fix is to pass suppressToolErrorWarnings, or a channel-scoped equivalent, into WhatsApp inbound auto-reply runs and add regression coverage proving final read/edit/write/message failure warnings are not delivered to WhatsApp while normal final replies and intentional media/tool artifacts still deliver.
