## Summary

Keep #6757 open. Current main has manual /compact, UI/Gateway sessions.compact, auto-compaction, and plugin/context-engine compaction hooks, but it still does not expose the requested in-session agent-triggered self-compaction action through session_status, the agent-facing gateway tool, or a dedicated active-run command. PR #43401 attempted the exact session_status(action="compact") shape but was closed unmerged, and the latest issue discussion already kept this item open.

## Best Possible Solution

Keep #6757 open until OpenClaw either implements a first-class in-session self-compaction API, such as session_status(action="compact") or an equivalent agent/session tool using direct compaction for the active run, or maintainers explicitly decide agents must not compact their own active session. The implementation should respect session visibility and permissions, classify the action as mutating, avoid lane deadlocks, persist compaction counters/token snapshots, and include focused tests for session_status, mutation tracking, and active-run behavior. PR #43401 remains useful prior art but is not shipped.
