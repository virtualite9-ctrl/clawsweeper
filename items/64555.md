## Summary

Keep this issue open. Current main still has the WhatsApp default auth-dir behavior described here, and the provided GitHub context shows #64563 is an open, unmerged implementation candidate that explicitly closes this issue.

## Best Possible Solution

Keep this issue open and resolve it through #64563 or an equivalent focused patch: lazy-resolve WA_WEB_AUTH_DIR, align listWhatsAppAuthDirs() and default/legacy auth-dir helpers with the active OPENCLAW_STATE_DIR or explicit OAuth root, and add a regression test for profile-isolated WhatsApp credentials. Doctor/docs guidance for OPENCLAW_OAUTH_DIR can follow after the core isolation bug is fixed.
