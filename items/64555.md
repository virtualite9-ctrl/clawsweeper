## Summary

Keep open. There is an open, unmerged implementation candidate, PR #64563, whose body uses closing syntax for this issue and proposes the exact WhatsApp profile-state isolation fix. Current main also still has the risky WhatsApp auth-dir shape: the default web auth dir is exported as a module-level value resolved through `resolveOAuthDir()` defaults, while account auth discovery still relies on ambient process env instead of an explicit profile env/state-dir input.

## Best Possible Solution

Keep this issue open and resolve the linked implementation path: review, land, or close/rework PR #64563. The likely fix belongs in `extensions/whatsapp`: make the default WhatsApp web auth dir lazy or function-backed, pass explicit env/state-dir inputs through auth-dir discovery, add an `OPENCLAW_STATE_DIR` profile-isolation regression test, and optionally add a doctor warning plus public operator docs for `OPENCLAW_OAUTH_DIR`.
