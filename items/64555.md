## Summary

Keep open. The linked implementation candidate #64563 is still open and explicitly says it closes this issue, so this issue should not be cleanup-closed before that PR is reviewed, merged, or deliberately closed. Current main also still has the risky WhatsApp auth-dir shape: a module-scope `WA_WEB_AUTH_DIR` resolved through ambient `process.env`, plus account auth discovery helpers that do not accept an explicit profile env/state dir.

## Best Possible Solution

Keep this issue open and resolve the linked implementation path in #64563: review, land, or ask for a focused rework. The fix should stay in `extensions/whatsapp`/the web-channel boundary by making default auth-dir resolution lazy or explicitly env/state-dir backed, aligning `listWhatsAppAuthDirs()` with the same root, and adding an `OPENCLAW_STATE_DIR` profile-isolation regression test. Optional follow-up work can add doctor/docs guidance for `OPENCLAW_OAUTH_DIR`, but the core cleanup action now is PR review rather than issue closure.
