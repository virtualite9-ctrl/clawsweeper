## Summary

Keep PR #71819 open. Current main still has the reported auto-reply dedupe behavior: the outer dispatch catch path releases the inbound dedupe claim for every error, including errors after externally visible replies, and main has no separate poisoned terminal state. The PR remains a focused fix for open bug #69303 with tests and addressed review feedback, so it is not obsolete or closeable.

## Best Possible Solution

Keep #71819 open for maintainer review and normal validation. The best product/code direction is the PR's third inbound dedupe terminal state: preserve retry for failures before visible output, but stamp/poison the dedupe key after externally visible dispatch so replayed provider message_ids cannot duplicate user-visible progress or side effects. Track the tool-result-only exclusion as a narrower follow-up if maintainers want broader idempotency.
