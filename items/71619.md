## Summary

Keep open. Current `main` still includes Linux version-manager directories in the daemon minimal PATH and the current tests explicitly lock that behavior in, so this PR is addressing a real gap rather than something already implemented or obsolete.

## Evidence

- **Current Linux PATH builder still includes version-manager directories:** On current `main`, `addCommonUserBinDirs` adds `~/.volta/bin`, `~/.asdf/shims`, and `~/.bun/bin`, and `resolveLinuxUserBinDirs` also adds `NVM_DIR/current/bin`, `FNM_DIR/current/bin`, `~/.nvm/current/bin`, and `~/.fnm/current/bin`. That matches the behavior this PR is trying to narrow.
  - file: [src/daemon/service-env.ts:89](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/src/daemon/service-env.ts#L89)

That is the exact behavior the PR updates.
