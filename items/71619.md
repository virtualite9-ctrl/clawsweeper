## Summary

Keep #71619 open. Current main still includes Linux version-manager directories in the daemon minimal service PATH and tests still assert that behavior, so the PR is neither obsolete nor implemented on main. The provided PR discussion also shows this is a focused fix for #71578, with the only review comment addressed in the PR branch.

## Best Possible Solution

Keep #71619 open for maintainer review or land an equivalent patch: make Linux minimal service PATH include stable user bins, pnpm/NPM prefix, Nix profiles, selected extra dirs, and system dirs while excluding Linux nvm/fnm/Volta/asdf/Bun version-manager directories; preserve the existing macOS version-manager behavior; update service-env and service-audit regression tests for #71578's hardened Linux expectation.
