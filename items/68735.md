## Summary

Current main now implements the GitHub Copilot request-compatibility fixes for the original multi-turn OpenAI Responses failure class. Unknown Copilot model IDs such as the reported gpt-5-mini and raptor-mini resolve through the Copilot OpenAI Responses path, that path now adds Copilot IDE/dynamic headers, rewrites stale connection-bound message/function-call IDs while preserving reasoning IDs, and tests/docs cover the request-compatibility behavior. Related Copilot duplicate #68957 was already closed as implemented with the same current-main evidence; the DeepSeek V4 subcase was separately fixed by #71105 and should not be the reason for this close.

## Best Possible Solution

Close this issue as implemented on current main, keep the shipped Copilot provider request-compatibility and replay-ID behavior, and track any future provider-specific schema rejection as a fresh issue with raw provider error, model id, OpenClaw version, and channel details if it still reproduces on a release containing these main-branch fixes.
