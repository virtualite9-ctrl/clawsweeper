## Summary

Current main still lacks both behaviors PR #39790 targets: session-title extraction does not strip inbound metadata before deriving `firstUserMessage`, and the computed outer session-row `displayName` is not passed into `deriveSessionTitle`. Because the TUI prefers `derivedTitle` over `displayName`, the linked #39722 user-visible bug remains actionable rather than obsolete.

## Best Possible Solution

Keep PR #39790, or the linked issue #39722, open until main implements the narrow gateway fix: strip inbound metadata before extracting session title text, pass the computed row `displayName` into `deriveSessionTitle` or equivalent title logic, and add regression tests for metadata stripping plus external displayName precedence and fallback.
