## Summary

Keep this PR open. Current main still lacks the non-blocking warning before `cron.add` or `cron.update`, while the cron service still treats main-session `systemEvent` payloads as queued system events rather than shell execution. The PR is also paired with open #63107 by the same contributor, so it is an implementation candidate rather than cleanup material.

## Best Possible Solution

Keep this PR open as the active fix for #63107. Review the current PR head against current main, require targeted `src/cli/cron-cli.test.ts` coverage for add/edit warnings including the omitted-`--session` edit case, then land it if the changed gate and CI pass.
