## Summary

Keep open. The report is recent and concrete, and current main does not give high-confidence proof that this exact Bailian/DashScope memory_search failure is fixed. Main has later proxy-aware remote embedding transport, but the current memory-search surface still resolves provider names as registered memory embedding adapter ids rather than arbitrary models.providers entries, and Qwen/Bailian memory embeddings are still described as planned.

## Best Possible Solution

Keep the issue open. A maintainer should clarify the supported path for DashScope embeddings, likely documenting provider: "openai" plus agents.defaults.memorySearch.remote.baseUrl/apiKey/model for OpenAI-compatible embeddings, or add/register a Qwen/Bailian memory embedding adapter if OpenClaw should support that provider id directly. If the reporter can still reproduce, ask for gateway-resolved memorySearch config, endpoint curl result, gateway logs showing the URL/error, and whether proxy env vars are involved.
