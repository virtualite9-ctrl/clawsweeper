## Summary

Keep #14151 open. Current main still has reasoning visibility, native/tagged reasoning parsing, final-tag enforcement, and provider-specific thinking controls, but no configurable frontend/orchestrator option that prepends a missing opening `<think>` tag before streaming or final parsing. Related PR #14323 attempted this feature but was closed unmerged, so it does not supersede the issue.

## Best Possible Solution

Keep #14151 open. A complete fix should add a typed global and/or per-model config option, document and validate it, resolve it through agent/model config, thread it through `RunEmbeddedPiAgentParams` and `SubscribeEmbeddedPiSessionParams`, inject once before both streaming and final-message reasoning parsing, guard existing tags/native thinking events, and add focused tests for streaming, non-streaming, existing `<think>` tags, native thinking, and `enforceFinalTag` interactions.
