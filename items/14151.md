## Summary

Keep #14151 open. Current main still does not implement a configurable force-prepend `<think>` option, and the current reasoning path only parses or strips tags/native reasoning that already exist. The related PR #14323 was closed unmerged, and the newest issue discussion already records a maintainer-side Codex review keeping the issue open.

## Best Possible Solution

Keep #14151 open. A complete fix should add a typed global and/or per-model config option, validate and document it, thread it through agent/model config into `RunEmbeddedPiAgentParams` and `SubscribeEmbeddedPiSessionParams`, inject exactly once before streaming and final-message reasoning parsing, guard existing `<think>` tags and native thinking events, and add focused tests for streaming, non-streaming, existing tags, native thinking, and `enforceFinalTag`.
