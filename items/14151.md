## Summary

Keep #14151 open. Current main still strips/parses existing `<think>` tags and supports reasoning streams, but it does not expose or thread a configurable global/per-model option to prepend a missing opening `<think>` tag before streaming or final-message parsing. The related implementation PR #14323 was closed unmerged, and the existing maintainer review comment also explicitly kept this issue open.

## Best Possible Solution

Keep #14151 open. A complete fix should add a typed global and/or per-model config option, validate and document it, resolve it through agent/model config, thread it through `RunEmbeddedPiAgentParams` and `SubscribeEmbeddedPiSessionParams`, inject exactly once before streaming and final-message reasoning parsing, guard existing tags/native thinking events, and add focused tests for streaming, non-streaming, existing `<think>` tags, native thinking, and `enforceFinalTag` interactions.

## Risks / Open Questions

- Blindly prepending can double-tag models that sometimes emit `<think>` themselves, so the implementation needs guards around existing tags and native reasoning/thinking events.
- The option needs maintainer product judgment on scope and naming: global default, per-model override, provider-owned behavior, or some combination.
- No live provider reproduction was run in this read-only review, but current source/config evidence is enough to show the requested capability is still absent.
