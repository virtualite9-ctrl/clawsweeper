## Summary

Keep #46565 open. Current main partially addresses the subagent/heartbeat failure mode by filtering subagent/cron bootstrap files to a fixed minimal allowlist and by adding opt-in lightweight context for subagent, cron, and heartbeat runs, so the reported `HEARTBEAT_OK` leak has useful mitigation. The broader feature request is not fully implemented: there is no user-configurable bootstrap `exclude` map or section-level session filter, bootstrap files are still read and injected as static file content rather than preprocessed with `!command`, and current date/time is intentionally kept out of the stable system prompt in favor of timezone plus `session_status`/message timestamp paths. No provided related item is a canonical duplicate for the remaining work.

## Best Possible Solution

Keep the issue open and narrow it into a core bootstrap-context design: add a documented, schema-backed session filter surface for bootstrap files or sections; decide whether bootstrap `!command` preprocessing belongs in core or should be expressed through a documented hook/plugin pattern; and align any time-context addition with the existing prompt-cache policy, likely through deterministic runtime message metadata or `session_status` rather than volatile stable prompt text.
