## Summary

Current main still has the core user-visible formatter cap reported here. Some provider-side pagination exists, especially for Microsoft Teams pins and channel lists, but the generic CLI rendering path still truncates `read`, `list-pins`, and Discord `search` output to 25 rows, so there is no conservative close reason.

## Best Possible Solution

Keep this issue open and implement a narrow message action contract fix: pass the requested `limit` and pagination metadata into the message rendering layer, remove or make configurable the hardcoded 25-row cap, propagate `limit` through `list-pins` where providers support it, and add regression tests for more than 25 rendered `read`, `list-pins`, and `search` rows while preserving existing provider pagination such as Teams `@odata.nextLink`.
