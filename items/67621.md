## Summary

Current main still performs raw `$[A-Z_][A-Z0-9_]{1,}` scanning over Python/Node script files without string, docstring, f-string, comment, or execution-context awareness. The YOLO gateway path is exempted, but active script-preflight paths such as gateway approval modes still call the scanner, so the reported `$SIM` string-literal false positive remains actionable.

## Best Possible Solution

Keep this issue open. Implement the fix in `src/agents/bash-tools.exec.ts` by making script-file preflight ignore `$VAR` matches inside Python and Node string/comment contexts, or by restricting detection to real execution/shell-leak contexts. Add focused coverage in `src/agents/bash-tools.exec.script-preflight.test.ts` for `$SIM` docstrings/help text and benign dollar-formatted output, while retaining negative tests for raw `$DM_JSON` code paths. The closed unmerged PRs #67644 and #67924 are useful references but are not current shipped fixes.
