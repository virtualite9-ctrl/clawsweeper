## Summary

Keep open. Current main still applies the `$[A-Z_]...` scanner to raw Python/Node source text without string-literal or comment awareness. The exact gateway YOLO path is skipped now, but script preflight remains active in other supported exec modes, and the two linked implementation PRs are closed unmerged.

## Best Possible Solution

Keep this issue open and implement the fix in `src/agents/bash-tools.exec.ts`: make script preflight ignore `$VAR` matches inside Python/Node string and comment contexts, while preserving rejection for actual shell-variable syntax in executable source. Add focused regression coverage in `src/agents/bash-tools.exec.script-preflight.test.ts` for `$SIM` docstrings/help text and dollar-formatted f-strings, plus existing negative cases.
