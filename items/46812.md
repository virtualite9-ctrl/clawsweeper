## Summary

Keep open. Current main still has raw workspace Markdown editing and chat-local pinned message UI, but it does not provide a pinned key-value facts editor, guaranteed structured context injection semantics, or the broader manifest/gateway/SPA seam for persistent Control UI plugin panels discussed in the thread.

## Best Possible Solution

Keep this open unless maintainers split a narrower canonical design issue. The best path is an additive manifest-first Control UI panel contract in the plugin manifest/schema, loader, gateway protocol, and SPA mounting layer, followed by a pinned facts editor backed by a documented structured store or USER.md section with tests/docs proving those facts are included in normal agent context.

## Risks / Open Questions

- A persistent Control UI panel seam needs product and security design for refresh behavior, iframe/canvas/tool rendering, permissions, persistence, and plugin trust boundaries.
- There is overlap with #46630 and #49828, but neither currently supersedes this request or tracks the manifest-first plugin panel API as a canonical implementation.
- A pinned facts editor still needs explicit storage and injection semantics, including multi-agent scope, interaction with bootstrap truncation, and agents.defaults.contextInjection behavior.
