## Summary

Keep #46812 open. Current main supports editing bootstrap workspace files such as USER.md and MEMORY.md through the Control UI Core Files editor, and those files are injected into agent context, but it does not implement the requested pinned key-value facts panel, structured pinned-context storage, or dedicated inline add/edit/delete UX. The supplied related items cover broader bootstrap control, Control UI auth, Telegram settings, and Skills panel work; none is a canonical duplicate for this pinned context panel request.

## Best Possible Solution

Keep the issue open for a focused Control UI/core implementation: add a structured pinned-context storage contract, probably backed by a managed section of USER.md or a dedicated workspace file; expose safe gateway RPCs for listing/upserting/deleting facts; render a compact Agents or Chat sidebar panel for inline edits; and add tests proving the facts are injected on normal agent turns without forcing users to hand-edit raw markdown.
