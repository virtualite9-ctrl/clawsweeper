## Summary

Current main does not implement the requested pinned context facts panel. The Control UI has a raw Files tab for workspace markdown files and fixed navigation/view surfaces, while the plugin SDK exposes tools, commands, hooks, HTTP routes, gateway methods, and memory prompt supplements but no Control UI panel/sidebar mount API. The issue discussion's `openclaw-lifework` example is useful evidence for a broader missing UI extension seam, not a reason to close this as already implemented or ClawHub-ready.

## Best Possible Solution

Keep this issue open unless maintainers create a narrower canonical issue for the underlying Control UI panel extension API. The best product path is to design a small, generic panel registration/mount contract for the Control UI and gateway/plugin SDK, then build the pinned facts editor on top of it or as a first-party panel backed by workspace/context persistence with clear injection semantics.
