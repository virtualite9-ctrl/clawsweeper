## Summary

Keep this issue open. Current main still has only raw workspace-file editing and transient chat tool side panels; it does not provide a pinned key-value facts editor, a dedicated pinned-context store, or a persistent Control UI panel/plugin manifest seam. The latest discussion adds a concrete plugin dashboard use case and a manifest-first `controlUiPanels` proposal, which is still missing from the current plugin manifest, SDK, gateway protocol, and SPA routing surfaces.

## Best Possible Solution

Keep this open unless maintainers split a narrower canonical issue. The best path is an additive, manifest-first Control UI panel contract that the loader exposes through the gateway and the SPA can mount, followed by a pinned facts editor backed by a documented structured store or USER.md section with clear context-injection guarantees.
