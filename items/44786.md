## Summary

Keep #44786 open. Current main still does not implement a provider-level `contextWindow` default/override: the runtime merge path only uses matching per-model metadata or discovered model metadata, the fallback path only uses a matching configured model or first configured model entry, and the provider config schema/type omit a provider-level `contextWindow` field. The linked PR #44955 is an open exact fix attempt, while #69464 is related Ollama `params.num_ctx` work and does not close this provider-level default request.

## Best Possible Solution

Keep the issue open as the canonical user-facing bug unless maintainers intentionally move the remaining work to PR #44955. A complete fix should add and document a provider-level default for the effective context limit, wire it through the model resolution merge/fallback paths with per-model values taking precedence over provider defaults and discovered metadata, and add regression coverage for an Ollama-discovered large-context model plus provider-level lower limit.
