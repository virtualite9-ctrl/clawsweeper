## Summary

The PR’s linked Brave `web_search` bug is already fixed on current main and shipped in v2026.4.24 through the lazy bundled web-provider compatibility path. The broader behavior proposed by the PR, letting `plugins.entries.<id>.enabled: true` override `plugins.enabled: false`, remains intentionally unsupported by current source, tests, and docs, so this branch is superseded rather than mergeable as-is.

## Best Possible Solution

Close this PR as superseded by the shipped v2026.4.24 lazy bundled web_search provider compatibility fix for #68249. Keep `plugins.enabled: false` authoritative in core; if maintainers intentionally want explicit per-plugin enablement to override the global kill switch, handle that as a separate maintainer-approved activation-policy change across shared config-state/config-policy consumers, tests, docs, and security expectations.

## Risks / Open Questions

- The PR’s branch would broaden plugin execution under `plugins.enabled: false`, which is a documented global kill switch and a least-privilege control.
- The supplied review discussion flagged real unresolved gaps: default `memory-core` slot selection could bypass the global disable, and changing only `config-state.ts` would split behavior from `config-policy.ts` consumers such as plugin skills.
- Closing this PR does not implement a general per-plugin override of the global disable flag; if maintainers want that product behavior, it needs a fresh shared activation design, docs update, tests, and security review.
