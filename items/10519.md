## Summary

Keep #10519 open. Current main still does not preserve fenced-code language metadata for Telegram HTML: the Telegram renderer emits static `<pre><code>` tags, the shared Markdown IR has no field for fence language and ignores `token.info`, and existing Telegram tests still expect a `js` fence to render without `class="language-js"`. The related WebChat copy-button work is a separate UI renderer path and does not satisfy Telegram Bot API HTML output.

## Best Possible Solution

Keep the issue open. Implement fenced-code language preservation for Telegram by carrying optional code-block language metadata through the shared Markdown IR/render path, or by adding a Telegram-specific fence-aware renderer if maintainers prefer not to widen the IR yet. Update Telegram format, wrapping/chunking, and outbound tests to expect `class="language-js"` while preserving current escaping and chunking behavior.
