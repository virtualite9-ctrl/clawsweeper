## Summary

Keep #70512 open. Current main still supports Feishu group-topic session routing, but ordinary Feishu DM routing resolves direct conversations to the sender Open ID only and has no opt-in DM topic/session-scope config. The related #10242 work is adjacent Feishu DM thread capability, not a canonical replacement for inbound per-topic DM session isolation.

## Best Possible Solution

Keep this open and implement it in the Feishu bundled plugin: add an opt-in Feishu DM session-scope config with sender-only as the default, derive direct peer ids from sender plus DM topic/root metadata when enabled, update config schema/docs, and add p2p route tests for `root_id`, `thread_id`, fallback behavior, ACP bindings, and unchanged default routing.
