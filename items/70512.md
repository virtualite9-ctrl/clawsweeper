## Summary

Current main still routes Feishu/Lark DMs by sender identity, while topic/root/thread fields are only used for group topic routing. There is no Feishu DM topic/session-scope config, and the related #10242 work does not supersede this inbound DM session-isolation request.

## Best Possible Solution

Keep this open and implement it in the bundled Feishu plugin: add an explicit Feishu DM topic/session-scope config, derive direct-message route/session ids from sender plus the canonical DM topic/root/thread id when enabled, populate thread context consistently for DM topics, update config schema/metadata/docs, and add focused p2p/private Feishu tests for topic and non-topic DMs. Preserve the current sender-scoped behavior as the default.
