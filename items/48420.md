## Summary

Keep #48420 open. Current main has the normal Anthropic streaming path handling `sk-ant-oat*` tokens as `Authorization: Bearer`, and OpenAI Codex OAuth is now explicitly routed through `openai-codex/*`. But the current native Anthropic PDF path still takes a resolved credential string and always sends it as `x-api-key`, matching the follow-up reproduction in the issue discussion. A meaningful bundled-provider auth bug remains on main.

## Best Possible Solution

Keep the issue open and narrow the remaining work: centralize Anthropic credential-to-header selection for all Anthropic HTTP paths, update `anthropicAnalyzePdf` and any other direct media/tool callsites to send `sk-ant-oat*` credentials as `Authorization: Bearer` with the required OAuth/Claude betas, add regression tests for native PDF OAuth versus API-key headers, and clarify any direct `openai/*` OAuth expectations around the documented `openai-codex/*` route.
