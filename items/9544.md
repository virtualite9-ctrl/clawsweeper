## Summary

Keep #9544 open. Current main still creates the WhatsApp socket through Baileys `useMultiFileAuthState`; the shipped hardening only wraps `creds.json` persistence/read barriers and does not replace the Baileys multi-file signal-key store with a SQL/consolidated production auth store.

## Best Possible Solution

Keep #9544 open until the bundled WhatsApp plugin replaces `useMultiFileAuthState` for both credentials and signal keys with a production auth store, migrates existing Baileys multi-file state without key loss, keeps or deliberately supersedes the current atomic `creds.json` protection, updates auth presence/logout/self-id/docs, and adds focused fresh-install plus migration tests.
