## Summary

Keep #9544 open. Current main still constructs the WhatsApp Baileys socket with `useMultiFileAuthState(authDir)` and only adds atomic handling around `creds.json`; it has not replaced the Baileys multi-file credential and signal-key store with a SQL/consolidated production auth store.

## Best Possible Solution

Keep #9544 open until the bundled WhatsApp plugin replaces Baileys multi-file auth with a production auth store, migrates existing multi-file credentials and signal keys safely, updates auth presence/logout/self-id/docs/config text, and adds focused fresh-install plus migration tests.
