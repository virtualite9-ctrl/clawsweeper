## Summary

Keep #9544 open. Current main still calls Baileys `useMultiFileAuthState(authDir)` for WhatsApp auth and only adds atomic/queued handling for `creds.json`; it does not replace the multi-file signal-key store with a SQL or consolidated production auth store. The related PR #66863 was closed unmerged, and the existing maintainer-thread Codex review already reached the same keep-open conclusion.

## Best Possible Solution

Keep #9544 open until the bundled WhatsApp plugin replaces `useMultiFileAuthState` with a production auth store, migrates existing Baileys multi-file credentials and signal keys safely, updates docs/config/status/logout behavior, and adds fresh-install, migration, corruption, and key-persistence tests. The closed PR #66863 is useful prior art but should not be treated as shipped or canonical.
