## Summary

Keep open. Current main still lacks the specific Anthropic billing wording fix and still gates messaging-tool text dedupe on same-target suppression; the PR’s current broad dedupe and broad billing matcher remain risky rather than safely obsolete.

## Best Possible Solution

Keep this PR open for rework or split it into two focused fixes: add a narrow Anthropic billing matcher for the exact "specified API usage limits" / "regain access" wording while preserving periodic usage-window rate_limit behavior, and fix multi-handle duplicate suppression through identity-aware target matching or a channel-owned alias/suppression hook rather than unconditional text-only dedupe. Add regression tests for the Anthropic sample, the multi-handle same-person case, and negative cross-target same-text delivery.

## Risks / Open Questions

- Anthropic "specified API usage limits" errors can still be classified as rate_limit, causing cooldown/retry behavior instead of billing failover/disable behavior.
- The reported multi-handle duplicate final reply can still occur when the messaging-tool send and the originating conversation use different same-person identifiers.
- Merging this PR as-is could suppress legitimate same-text final replies sent to a different provider, account, channel, or target, and could classify ordinary periodic usage-window limits as billing.
