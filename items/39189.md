## Summary

Current `main` still does not fully implement either PR fix: the Anthropic "specified API usage limits" wording can still be classified through the generic rate-limit path, and messaging-tool final-reply dedupe remains guarded by same-target suppression. The PR is also not cleanly obsolete because its proposed dedupe relaxation and broad billing matcher have unresolved false-positive risks from review comments, so it should stay open for a narrower rework rather than be closed.

## Best Possible Solution

Keep the PR open, but rework or split it into two focused patches: add a narrow Anthropic billing matcher and tests for the exact `specified API usage limits` / `regain access` wording without shadowing periodic usage-window limits, and fix multi-handle duplicate suppression through identity-aware target matching or a channel-owned alias/suppression hook instead of unconditional text-only dedupe. Add positive tests for the same-person multi-handle case and negative tests for cross-target same-text sends.
