## Summary

Current `main` already implements the Anthropic-side fix: the Anthropic provider now uses the native replay policy that preserves native `toolu_*` ids, the sanitizer keeps those ids stable instead of rewriting them, and there is an explicit regression test for the exact reported scenario where an earlier Anthropic replay prefix must stay unchanged after a later subagent turn. The same implementation is present in the shipped `v2026.4.22` tag.

Closing from a Codex review: current `main` already has the Anthropic replay-id preservation needed for this report. If you still see a full cache write on a newer stack, please open a fresh issue with the exact replay fragment and cache trace showing which serialized bytes changed between the pre-subagent and post-subagent Anthropic requests.
