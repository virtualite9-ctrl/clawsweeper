## Summary

Keep this issue open. Current main still pins the implicated Pi dependency and still routes embedded runs through `SessionManager.open()` followed by `createAgentSession()`, while the pinned dependency still performs the duplicate whole-file JSONL parse before rebuilding session context.

## Best Possible Solution

Keep this issue open and fix it at the embedded Pi/session boundary. The best path is to eliminate the second whole-file parse for existing session JSONL files, either by upgrading or patching `@mariozechner/pi-coding-agent` so `SessionManager.open()` header-reads once and reuses parsed entries, or by adding a narrow OpenClaw adapter that avoids duplicate parsing while preserving Pi session semantics. Add focused regression or stress coverage with a large JSONL/long tool-result transcript proving embedded context rebuild/truncation returns a controlled error or compacts/rotates without gateway exit. Treat stale-lock cleanup as recovery, not the primary fix.
