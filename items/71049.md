## Summary

Current main still lacks the requested server-side option. The Teams plugin creates a stream controller for personal chats unconditionally, sends the informative stream status on reply start, forwards partial replies into TeamsHttpStream, and the existing `typingIndicator`/`blockStreaming` options only affect typing keepalive and block delivery behavior.

## Best Possible Solution

Keep the issue open. The likely fix belongs in the bundled Teams plugin: add a typed `channels.msteams` config option for DM stream-status mode, wire it into `createTeamsReplyStreamController`/`createMSTeamsReplyDispatcher` so personal chats can skip `TeamsHttpStream` and use the existing typing keepalive plus final-message delivery path, then update schema metadata, docs, and focused dispatcher/stream-controller tests.
