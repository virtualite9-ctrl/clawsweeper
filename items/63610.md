## Summary

Keep this issue open. Current main still does not handle the reported WhatsApp-visible `[TOOL_CALL]{tool => ..., args => ...}[/TOOL_CALL]` pseudo tool call as a structured tool call or reliably suppress that exact syntax from user-visible replies. The newer diagnostic path only warns about some tool-shaped text and explicitly treats it as text.

## Best Possible Solution

Keep this issue open. Add a shared assistant-visible handling path for `[TOOL_CALL]{tool => ..., args => ...}[/TOOL_CALL]`: either safely convert it only when it can be parsed deterministically against registered tools, or at minimum strip/suppress it from user-visible delivery and history as leaked scaffolding. Add focused regression coverage in `src/shared/text/assistant-visible-text.test.ts` or `src/shared/text/tool-call-shaped-text.test.ts`, plus a WhatsApp auto-reply delivery test showing the raw pseudo call is not sent to `msg.reply`.
