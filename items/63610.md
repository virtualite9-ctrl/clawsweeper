## Summary

Current main still does not cover the exact WhatsApp leak reported here: `[TOOL_CALL]{tool => ..., args => ...}[/TOOL_CALL]` as assistant-visible text. The shared sanitizer handles XML-style tool tags and `[Tool Call: ...]` downgraded markers, while WhatsApp final delivery still sends normalized final/block text without a parser or strip for the all-caps bracket form.

## Best Possible Solution

Keep this issue open. Add a shared assistant-visible delivery/runtime fix for the exact `[TOOL_CALL]{tool => ..., args => ...}[/TOOL_CALL]` syntax, preferably before channel-specific delivery. If the format can be parsed safely into a structured tool call, do that before final delivery; otherwise suppress it from user-visible replies. Add regression coverage in `src/shared/text/assistant-visible-text.test.ts` and a WhatsApp auto-reply dispatch/delivery test.
