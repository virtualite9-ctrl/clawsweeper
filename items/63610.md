## Summary

Keep this issue open. Current main still does not handle the exact leaked `[TOOL_CALL]{tool => ..., args => ...}[/TOOL_CALL]` assistant-text syntax reported for WhatsApp: shared visible-text cleanup covers XML-style tool-call blocks and `[Tool Call: ...]` downgraded blocks, while WhatsApp final delivery still sends normalized final text without parsing or stripping this all-caps bracket form.

## Best Possible Solution

Keep this issue open and fix it in the shared assistant-visible delivery/runtime path, with a WhatsApp regression test. The fix should add coverage for `[TOOL_CALL]{tool => ..., args => ...}[/TOOL_CALL]` and either convert that provider-emitted pseudo call into a real structured tool call before delivery or suppress it from user-visible replies when it is leaked scaffolding.
