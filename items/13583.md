## Summary

Keep #13583 open. Current main has adjacent plugin hooks for pre-tool blocking, one-pass final-answer revision, and outbound message cancellation, but it still does not implement the requested policy-as-code response gate: no two-phase tool-only planning protocol, responseGate config, required-tool accounting, fail-closed response validators, retry/fallback contract, or audit model.

## Best Possible Solution

Keep #13583 open as the canonical response-gate policy-as-code design item. The best maintainer path is a versioned core/plugin API for mandatory tool-call accounting, typed response validators, fail-closed final-response gating, bounded retry/fallback behavior, and audit records; after that contract exists, domain-specific policy packs can live in plugins or ClawHub.

The same discussion explicitly says these are complementary and that two-phase respond-only-after-tools-pass enforcement still needs core pipeline support.
