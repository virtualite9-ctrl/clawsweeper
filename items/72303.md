## Summary

Current main already fixes the reported v2026.4.24 Bonjour restart loop: stuck `probing`/`announcing` advertisers are now restarted only a bounded number of times, then Bonjour is disabled for the current Gateway process while the Gateway keeps running. Current docs and Docker defaults also cover bridged-container mDNS by disabling Bonjour unless explicitly opted in.

## Best Possible Solution

Close #72303 as implemented on current main. Keep the bounded Bonjour restart/disabling behavior, the scoped ciao process-error handling, and the Docker bridge default of `OPENCLAW_DISABLE_BONJOUR=1`; if a user can still reproduce on a build from current main or a later release, they should open a fresh issue with logs, container network mode, config, and Bonjour env values.
