## Summary

Keep #59427 open. Current main still has the reported core behavior for ordinary sessions: OpenClaw builds a full OpenClaw-owned system prompt for each run, regular sessions resolve to `promptMode: "full"`, workspace/bootstrap context is injected by default, and tool schemas still count toward the context window. Existing knobs such as `contextInjection`, prompt caching, context pruning, and subagent/heartbeat lightweight modes are useful partial mitigations, but they do not implement lazy baseline context loading or a user-facing lightweight-vs-full mode for normal short replies. The provided related items are adjacent usage-display, session snapshot, rate-limit, and Azure usage reports, not duplicates of this baseline-context request.

## Best Possible Solution

Keep #59427 open as the canonical core request for normal-turn baseline context reduction. The best path is a deliberate core design that separates always-needed safety/runtime/tool routing from on-demand workspace/bootstrap/memory/tool detail, exposes a documented user-facing lightweight/full or adaptive mode for regular sessions, preserves prompt-cache stability, and adds `/context` plus regression coverage showing short replies can run with a materially lower baseline without breaking tool calls, memory search, channel delivery, or subagent/cron behavior.

## Risks / Open Questions
