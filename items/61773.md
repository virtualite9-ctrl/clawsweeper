## Summary

Current `main` has explicit Matrix invite dispatch, startup invite replay, auto-join handling, and sync-readiness gating for the no-dispatch symptom described here; the issue thread later narrowed the remaining failure to inactive/stale Matrix account-token state rather than the original OpenClaw dispatch path.

## Evidence

- **Matrix docs now make invite behavior explicit:** The Matrix docs state that fresh invites only work when `channels.matrix.autoJoin` allows them, and they warn that `autoJoin` defaults to `off`, so invited DMs/rooms will be ignored unless operators opt in.
  - file: [docs/channels/matrix.md:45](https://github.com/openclaw/openclaw/blob/8b76392e3e7934b1cdd6d461148263fd56141ec8/docs/channels/matrix.md#L45)
  - sha: [8b76392e3e79](https://github.com/openclaw/openclaw/commit/8b76392e3e7934b1cdd6d461148263fd56141ec8)
- **Current main joins invites from a dedicated room.invite path:** `registerMatrixAutoJoin` listens for `room.invite` and joins either all invites or only allowlisted invites, including alias resolution for allowlists.
  - file: [extensions/matrix/src/matrix/monitor/auto-join.ts:64](https://github.com/openclaw/openclaw/blob/8b76392e3e7934b1cdd6d461148263fd56141ec8/extensions/matrix/src/matrix/monitor/auto-join.ts#L64)
  - sha: [8b76392e3e79](https://github.com/openclaw/openclaw/commit/8b76392e3e7934b1cdd6d461148263fd56141ec8)
- **Current main emits invite events and replays pending invites on startup:** `MatrixClient` emits `room.invite` when membership events target the current user, also handles SDK room lifecycle invite transitions, and replays outstanding invite rooms after startup completes.
  - file: [extensions/matrix/src/matrix/sdk.ts:1723](https://github.com/openclaw/openclaw/blob/8b76392e3e7934b1cdd6d461148263fd56141ec8/extensions/matrix/src/matrix/sdk.ts#L1723)
  - sha: [8b76392e3e79](https://github.com/openclaw/openclaw/commit/8b76392e3e7934b1cdd6d461148263fd56141ec8)
- **Startup now waits for a ready sync state before succeeding:** `startSyncSession()` waits for `waitForInitialSyncReady()` before marking the Matrix client started, then replays outstanding invite events and refreshes DM cache.
  - file: [extensions/matrix/src/matrix/sdk.ts:400](https://github.com/openclaw/openclaw/blob/8b76392e3e7934b1cdd6d461148263fd56141ec8/extensions/matrix/src/matrix/sdk.ts#L400)
  - sha: [8b76392e3e79](https://github.com/openclaw/openclaw/commit/8b76392e3e7934b1cdd6d461148263fd56141ec8)
- **Regression coverage exists for the reported failure modes:** Tests cover invite emission from membership events, invite emission from SDK room lifecycle events, waiting for a ready sync state before startup resolves, and replaying outstanding invite rooms at startup.
  - file: [extensions/matrix/src/matrix/sdk.test.ts:952](https://github.com/openclaw/openclaw/blob/8b76392e3e7934b1cdd6d461148263fd56141ec8/extensions/matrix/src/matrix/sdk.test.ts#L952)
