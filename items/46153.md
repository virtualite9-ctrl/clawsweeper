## Summary

Keep #46153 open. Current main has substantial macOS launchd restart hardening, but I did not find a source, test, docs, or release-history fix for the specific LaunchAgent startup hang/module-evaluation silence. The generated gateway LaunchAgent still has stdout/stderr paths but no stdin binding such as StandardInPath=/dev/null, and the app path delegates to that same CLI installer.

## Best Possible Solution

Keep the issue open and fix the LaunchAgent startup surface directly: add StandardInPath=/dev/null to the gateway LaunchAgent plist generator, make restart/repair rewrites preserve or add it, add launchd plist tests and doctor/service-audit coverage for missing stdin defaults, and consider startup diagnostics or a bounded launchd readiness failure so module-evaluation hangs are visible.
