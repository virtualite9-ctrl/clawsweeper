## Summary

Keep open. Current main still implements Feishu message typing as `im.messageReaction.create/delete` with `emoji_type: "Typing"`, and the Feishu docs/tests still describe this as typing reactions. Related #8722 only confirms the older reaction-based feature landed; it does not resolve this issue's request for native Feishu typing status.

## Best Possible Solution

Keep this issue open for Feishu plugin owner review. The concrete path is to verify Feishu/Lark's current native typing or typewriter API for bot replies, replace the message-reaction typing helper where supported, update the Feishu docs/config wording away from `typing reactions`, and add regression tests proving the dispatcher uses the native API or a documented fallback.
