## Summary

Keep #19362 open. Current main still lacks a Discord CHANNEL_DELETE listener or registration path, while Discord guild channels are documented and persisted as isolated session keys. The PR remains product-relevant, but it needs to be refreshed for the current extensions/discord plugin layout and should address the transcript archival/cleanup concerns raised in review.

## Best Possible Solution

Keep the PR open and refresh it against current main. Implement deleted-channel cleanup in extensions/discord/src/monitor by adding and registering a Carbon channel-delete listener, validating the deleted channel ID, scanning relevant agent session stores for agent:<agentId>:discord:channel:<channelId> keys, mutating the store under updateSessionStore’s lock, archiving/cleaning transcripts with current session deletion semantics or an explicit SDK seam, and adding focused regression tests for registration, cleanup, and transcript handling.
