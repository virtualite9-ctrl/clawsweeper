## Summary

Current main still lacks Discord CHANNEL_DELETE session cleanup, so this PR remains a real implementation candidate rather than something to close. It cannot land as-is because it targets the removed `src/discord` layout instead of the current bundled Discord plugin under `extensions/discord`, but the Carbon dependency does expose the needed channel-delete listener surface.

## Best Possible Solution

Keep this PR open as an implementation candidate, but require a refresh against current main: port the listener and tests to `extensions/discord/src/monitor`, register it in `registerDiscordMonitorListeners`, validate the Carbon channel-delete payload ID from the typed top-level data, use plugin-safe SDK/runtime seams for the relevant agent session stores, delete only exact `agent:<agentId>:discord:channel:<channelId>` entries, archive deleted entries under the update lock with cleanup and `restrictToStoreDir` semantics, then run the Discord plugin tests and changed gate when available.
