## Summary

Current main at 406ae72fd2789c78053f6b94006ec8f488bc1c00 still does not implement or register Discord CHANNEL_DELETE session cleanup. Discord guild channels still persist as isolated session keys, and the only current Discord deletion/cleanup-adjacent path is thread archive handling that marks matching thread sessions stale rather than deleting channel sessions or archiving deleted-channel transcripts. PR #19362 therefore remains useful, though it needs a refresh from the old src/discord paths to the current extensions/discord layout and should address the transcript archival issues already raised in review.

## Best Possible Solution

Keep #19362 open and refresh it against current main. Add and register a Discord channel-delete listener under extensions/discord/src/monitor, validate the deleted channel id, remove matching agent:<agentId>:discord:channel:<channelId> session rows through the current plugin-safe session-store/runtime seam, archive exactly the deleted entries' transcripts under the store update lock, run archived transcript cleanup with existing sessions.delete/store-maintenance semantics, and add focused tests for listener registration, invalid ids, multi-agent or per-agent stores, deletion matching, and transcript handling.
