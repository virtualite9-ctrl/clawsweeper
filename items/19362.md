## Summary

Keep #19362 open. Current main at e29d3516 still has no Discord channel-delete gateway listener or registration, while Discord guild channels are documented as isolated session keys, so deleted guild channels can still leave orphaned session entries. The PR remains useful but needs to be ported to the current extensions/discord layout and corrected for the transcript archival issues already raised in review.

## Best Possible Solution

Keep the PR open. The best path is to refresh #19362 against current extensions/discord, add a Carbon ChannelDeleteListener registration in registerDiscordMonitorListeners, validate the deleted channel ID, remove matching agent:<agentId>:discord:channel:<channelId> entries with lock-aware session-store helpers, archive and clean up transcripts with the same semantics as existing session deletion, and add focused regression tests for multi-agent cleanup and transcript retention.
