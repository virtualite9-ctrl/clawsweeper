## Summary

Current main still does not implement Discord CHANNEL_DELETE session cleanup. This PR remains a useful implementation candidate, but it needs to be refreshed from the old `src/discord` layout to the current bundled plugin layout under `extensions/discord` and reviewed for session-store/transcript correctness before landing.

## Best Possible Solution

Keep this PR open, but require a refresh against current main. Port the listener and tests to `extensions/discord/src/monitor`, register it in the current startup path, use a current or newly exposed plugin-safe session-store target helper for all relevant agent stores, validate the deleted channel id from Carbon's `ChannelDeleteListener` payload, delete only matching `agent:<agentId>:discord:channel:<channelId>` rows, archive exactly the deleted entries' transcripts under the update lock, run archived-transcript cleanup, and then run the Discord plugin tests plus the changed gate when dependencies are available.
