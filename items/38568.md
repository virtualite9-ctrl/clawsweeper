## Summary

Keep #38568 open. Current main at 67b9167b808f2ff065c91665d6f36256a5c23ef2 still does not inject current context-window usage into the system prompt Runtime line, and the prompt construction inputs do not carry a per-turn context usage snapshot. The issue remains a concrete core prompt/runtime enhancement, not a duplicate, ClawHub item, or stale insufficient-info report.

## Best Possible Solution

Implement this in core prompt assembly. Add an explicit context usage input to the runtime prompt params, populate it from the current session/model prompt snapshot before system prompt construction, render a compact Runtime field such as context=49% or context=98k/200k (49%), and add focused tests for buildRuntimeLine plus embedded and CLI prompt wiring while preserving existing status/session_status behavior.
