## Summary

Keep PR #72158 open. Current main has a bundled `microsoft-foundry` provider but the provider directory still lacks any Microsoft/Azure Foundry entry, so the central docs gap is real and not implemented on main. The PR is not ready to close as implemented or ClawHub-only; however, as submitted it adds a link to `/providers/microsoft-foundry` while current main has no matching provider page, and related PR #68502 overlaps by adding a broader Azure AI Foundry guide plus index entry. That needs maintainer/docs consolidation rather than automated cleanup closure.

## Best Possible Solution

Keep the PR open and ask the docs maintainer to consolidate the Microsoft/Azure Foundry provider docs path: either add a matching `docs/providers/microsoft-foundry.md` guide and keep this index entry, retarget the entry to a reviewed Azure AI Foundry guide, or fold the index change into #68502 if that broader guide is accepted and aligned with the current `microsoft-foundry` plugin.
