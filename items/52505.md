## Summary

Current `main` already avoids replaying raw reasoning text back into OpenAI WS requests, so this PR's sanitization on the older path is redundant.

## Close Comment

Closing as implemented on `main`. Codex review verified that the current OpenAI WS replay path no longer serializes raw `reasoning_content` / `summary` back into the JSON request: `buildAssistantMessageFromResponse()` stores only `{id,type}` in `thinkingSignature` (`src/agents/openai-ws-message-conversion.ts:580`), and `convertMessagesToInputItems()` rebuilds replay items from that minimal signature (`src/agents/openai-ws-message-conversion.ts:214`, `src/agents/openai-ws-message-conversion.ts:413`). The regression tests also assert that replayed reasoning blocks keep only the reasoning id and still pair correctly with tool calls (`src/agents/openai-ws-stream.test.ts:1036`, `src/agents/openai-ws-stream.test.ts:1553`). The same implementation is already present in shipped release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`), so this PR's sanitization on the older path is redundant. If there is still a current repro on `main`, please open a fresh PR or issue with that exact transcript shape.
