## Summary

Keep open. Current main has partial progress on this leak, including commentary-phase suppression and an unreleased fix for bare namespaced `<antml:thinking>` streaming tags, but the specific scanner path called out in the issue discussion still does not accept attributed thinking tags. Because `stripBlockTags` runs before streamed/block replies are emitted, attributed `<think>`, `<thinking>`, or `<antml:thinking>` blocks remain a plausible user-visible leak.

## Best Possible Solution

Keep this issue open and make the core embedded streaming utilities share the canonical reasoning-tag grammar. The fix should cover `antml:` prefixes and attributes in `stripBlockTags`, `splitThinkingTaggedText`, and tagged-reasoning extraction while preserving code-span awareness and chunk-boundary state, with focused regression tests for attributed `<think>`, `<thinking>`, and `<antml:thinking>` streamed across chunk boundaries.
