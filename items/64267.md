## Summary

Keep open. Current main still has the specific residual leak path identified in the discussion: the canonical sanitizer accepts attributed reasoning tags, but the streaming/block scanner used before channel delivery still only matches bare tags. Existing mitigations for MiniMax reasoning_content, commentary-only output, and plain/namespaced thinking tags do not fully resolve attributed `<think>`/`<thinking>`/`<antml:thinking>` streaming leaks.

## Best Possible Solution

Keep this issue open. The best fix is a narrow core change in the embedded streaming utilities: make `THINKING_TAG_SCAN_RE` and the related open/close regexes share the canonical reasoning-tag grammar, including `antml:` prefixes and attributes, then add focused block/partial streaming regression tests for attributed `<think>`, `<thinking>`, and `<antml:thinking>` chunks across boundaries.
