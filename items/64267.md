## Summary

Keep open. Current main has partial mitigations for commentary-only output, MiniMax reasoning_content, and namespaced antml streaming tags, but the streaming scanner still does not match attributed thinking tags while the canonical sanitizer does. That leaves the specific residual leak path identified in the issue discussion unresolved.

## Best Possible Solution

Keep the issue open and land a narrow core fix in the embedded streaming utilities: make the streaming thinking-tag scanner share the canonical reasoning-tag grammar, including antml prefixes and attributes, then add focused partial/block streaming regression tests for attributed `<think>`, `<thinking>`, and `<antml:thinking>` chunks across boundaries.
