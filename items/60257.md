## Summary

Close as `clawhub`: current `main` already routes reusable procedural workflows toward skills/plugins and ClawHub, including the existing Skill Workshop plugin that turns repeatable workflows into `SKILL.md` skills. Adding `ROUTINE.md` as a new core primitive conflicts with the repo vision to prefer bundle-style plugins/ClawHub and avoid new heavy orchestration layers in core.

## Evidence

## Risks / Open Questions

- There is no first-class `ROUTINE.md` loader on current `main`, so external authors would need to map the concept onto existing skills/plugin bundles rather than an official core spec.
- If maintainers later decide they want a cross-agent routine standard for interoperability, closing this issue now could defer that broader design discussion.
