## Summary

Keep open. Current main weakens one part of the report because heartbeat is still a main-session turn by default and fresh-session heartbeat mode is opt-in, but the core request is not implemented: durable memory flush is still tied to compaction-threshold logic, there is no session-close summary/flush path, and heartbeat runs are explicitly excluded from the memory-flush path. Existing mitigations are workarounds (persistent custom cron sessions, optional dreaming transcript ingestion), not a fix for short-session durable memory writes.
