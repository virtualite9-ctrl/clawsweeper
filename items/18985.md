## Summary

Keep #18985 open. Current main still does not support the requested Windows 11 MSYS2/MINGW `install.sh` path or fish-specific installer PATH persistence. The existing maintainer-thread Codex review already reached the same conclusion, and the current main changes since that review do not touch the relevant installer, docs, or shell-completion surfaces.

## Best Possible Solution

Keep #18985 open until maintainers either implement explicit MSYS2/MINGW `install.sh` support with correct OS detection, Windows/MSYS path and package-manager handling, fish `config.fish` PATH persistence or guidance, docs, and installer smoke coverage; or make a documented product decision that Windows Unix-like environments should use WSL2 or `install.ps1` instead.
