## Summary

Keep #18985 open. Current main still does not implement the requested Windows 11 MSYS2/MINGW install.sh support or fish-specific installer PATH persistence/guidance. The Unix installer still accepts only macOS, Linux, or WSL before exiting unsupported, docs route native Windows users to install.ps1 or WSL2, release checks encode that split, and the related Microsoft tracker PR #49126 was closed unmerged.

## Best Possible Solution

Keep #18985 open until maintainers either implement explicit MSYS2/MINGW install.sh support with correct OS detection, Windows/MSYS path and package-manager handling, fish config.fish PATH persistence/warnings, docs, and installer smoke coverage; or make a documented product decision that Windows Unix-like environments should use WSL2 or install.ps1 instead.
