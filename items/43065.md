## Summary

Keep this PR open. Current main still has the heartbeat responsePrefix interpolation gap described by the linked bug: heartbeat resolves the raw prefix, does not pass onModelSelected into getReplyFromConfig, and feeds the raw prefix into normalizeHeartbeatReply. The same-author linked issue #43064 is still open, while related PRs #43133 and #43158 were closed unmerged as duplicates rather than shipped fixes.

## Best Possible Solution

Keep this PR open for maintainer review. The right product/code path is to wire createReplyPrefixContext into runHeartbeatOnce, pass its onModelSelected callback through replyOpts, resolve the raw responsePrefix with the populated prefixContext after getReplyFromConfig returns, and add a focused regression test that verifies heartbeat alert replies interpolate model/provider/thinking variables without relying on deep extension internals.
