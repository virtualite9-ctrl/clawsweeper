## Summary

Keep this PR open. Current main still has the heartbeat responsePrefix interpolation gap, and the PR is a focused implementation candidate for the same-author open bug report #43064. The current heartbeat path reads the raw prefix, does not pass onModelSelected into getReplyFromConfig, and prepends the prefix without resolveResponsePrefixTemplate.

## Best Possible Solution

Keep this PR open for maintainer review or a rebased equivalent patch. The right fix belongs in runHeartbeatOnce: create the reply prefix context for the heartbeat target, pass onModelSelected through getReplyFromConfig, resolve the raw responsePrefix with the populated prefixContext after the LLM call, and add focused heartbeat regression coverage. Keep #43064 open until a fix merges.
