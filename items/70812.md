## Summary

Keep PR #70812 open. Current main still has the false-positive path: trajectory sidecars are generated as `<session>.trajectory.jsonl`, the primary transcript matcher still accepts any non-archive `.jsonl`, and doctor orphan detection still feeds directory entries through that matcher directly. The PR is a focused fix for open issue #70680, and the related #72047 PR is closed and unmerged, so it does not supersede this work.

## Best Possible Solution

Keep #70812 open and continue review as the active focused fix for #70680. The best product/code path is a scoped session-artifact classification fix that excludes true trajectory sidecars from doctor orphan detection, preserves valid session IDs ending in `.trajectory`, and makes the intended disk-budget retention/removal behavior for sidecars explicit in code and tests.
