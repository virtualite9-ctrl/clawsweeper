## Summary

Keep open. Current main now fixes the original doctor false-positive by excluding `*.trajectory.jsonl` from primary transcript scans, but it does so unconditionally. The PR still contains useful follow-up work to preserve valid session IDs ending in `.trajectory`, which current main allows by schema but currently excludes from primary, usage-counted, and discovery paths. The PR also includes unrelated node exec workdir/approval-path changes in a security-sensitive execution surface, so the best path is review and split or explicitly justify that part before landing.

## Best Possible Solution

Keep this PR open as a useful follow-up candidate, but ask the contributor to narrow it to the session-artifact classifier/doctor/usage/disk-budget behavior or split the node exec workdir changes into a separate PR. The session fix should preserve documented trajectory sidecar cleanup while treating `.trajectory.jsonl` as a sidecar only when the matching primary transcript is present, so valid session IDs ending in `.trajectory` remain primary transcripts.
