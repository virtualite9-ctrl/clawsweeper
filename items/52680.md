## Summary

Keep open. Current `main` does not implement the requested non-fatal out-of-range `tools.read` recovery: `createOpenClawReadTool` still delegates to the upstream read tool without catching EOF-offset errors, the bundled `@mariozechner/pi-coding-agent` version still throws for `offset > EOF`, and the closed candidate fix PRs added recovery code/tests that are not present on `main`.

## Evidence

- **No offset recovery in current read wrapper:** `executeReadWithAdaptivePaging` calls `params.base.execute(...)` directly for the requested offset and has no catch/retry/clamp path for an out-of-range offset. Current `src/agents/pi-tools.read.ts` also has no `offsetRecovery` diagnostic handling.
  - file: [src/agents/pi-tools.read.ts:220](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/agents/pi-tools.read.ts#L220)
  - command: `nl -ba src/agents/pi-tools.read.ts | sed -n '206,282p'`
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **Bundled upstream read tool still throws past EOF:** Current `package.json` pins `@mariozechner/pi-coding-agent` to `0.70.0`. Inspecting the published `0.70.0` tarball shows `dist/core/tools/read.js` still does `if (startLine >= allLines.length) { throw new Error(...) }` for out-of-range offsets.
  - file: [package.json:1587](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/package.json#L1587)
  - command: `T=$(npm view @mariozechner/pi-coding-agent@0.70.0 dist.tarball --silent); curl -L --silent "$T" | tar -xzOf - package/dist/core/tools/read.js | sed -n '148,160p'`
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **Closed fix PR content is absent from main:** PR #52739's patch adds `READ_OFFSET_OUT_OF_RANGE_RE`, `offsetRecovery` details, and a dedicated `src/agents/pi-tools.read.offset-clamp.test.ts`. None of those symbols/files exist on current `main`, so that proposed fix was not landed here.
  - command: `curl -L --silent https://patch-diff.githubusercontent.com/raw/openclaw/openclaw/pull/52739.patch | sed -n '1,260p'; rg -n 'READ_OFFSET_OUT_OF_RANGE_RE|offsetRecovery|pi-tools.read.offset-(clamp|recovery)\.test' src/agents`
- **Current behavior only converts throws into error results:** `toToolDefinitions` catches tool exceptions and returns `{ status: "error", tool, error }`. That avoids a raw exception escaping this adapter, but it is not the requested clamp/diagnostic successful read behavior.
  - file: [src/agents/pi-tool-definition-adapter.ts:247](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/agents/pi-tool-definition-adapter.ts#L247)
  - command: `nl -ba src/agents/pi-tool-definition-adapter.ts | sed -n '226,272p'`
