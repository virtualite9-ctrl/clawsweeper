## Summary

Keep open. The main session-status fix proposed by #60290 is still not implemented on current main: `updateSessionStoreAfterAgentRun` still merges the full stale in-memory session entry back over the freshly loaded disk row, and `mergeSessionEntry` still lets patch fields override existing fields. The PR is not ready to land as-is because it bundles five independent fixes and has an unresolved review concern on the MiniMax change, but the primary bug fix remains useful.

## Best Possible Solution

Keep the PR open and preserve the primary fix direction: make lifecycle status authoritative by omitting `status` from the post-run token/model patch, or otherwise preventing stale in-memory `running` from overwriting a terminal disk status. The maintainable path is to split or revise the PR so the session-store fix and regression test land narrowly, while the unrelated provider/channel/loop changes are handled in separate PRs with the MiniMax review addressed.
