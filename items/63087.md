## Summary

Current main already routes memory remote embedding HTTP, including Gemini memory index calls, through the shared env-proxy-aware guarded fetch path when HTTP_PROXY/HTTPS_PROXY applies. The issue’s reported setup had HTTPS_PROXY/HTTP_PROXY configured, and regression tests cover the trusted env-proxy mode while preserving strict behavior for NO_PROXY and ALL_PROXY-only cases.

## Best Possible Solution

Close #63087 as implemented on main. Keep the shared memory-host remote HTTP helper and regression tests as the canonical fix, ship it in the next release, and track any separate ALL_PROXY-only feature request independently if maintainers want behavior beyond Undici EnvHttpProxyAgent semantics.
