## Summary

Current `main` already covers the PR's user-facing behavior: public config reads are pinned after the first successful load, gateway reloads skip unchanged watcher rereads by persisted hash, and config warning output already uses real newlines. The proposed narrow `loggedConfigWarningFingerprints` patch is redundant against the shipped runtime-snapshot/reloader behavior.
