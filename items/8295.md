## Summary

Keep #8295 open. Current main at 7e13f3f5140168e31027b39a9855ade3fadf24d5 still lacks Telegram `allowBots` support for groups/topics: the public config types, strict Zod schema, generated metadata, and inbound Telegram runtime path do not expose or enforce an `allowBots` policy. The Plaiground ClawHub comment is a useful Discord-based workaround, and PR #68325 is a relevant open implementation attempt, but neither satisfies the bundled Telegram plugin feature yet.

## Best Possible Solution

Keep #8295 open until the bundled Telegram plugin implements `allowBots` across public config types, strict schema, generated metadata/UI help, docs, runtime filtering, and regression tests. PR #68325 or a successor is the likely path, but it needs maintainer review for account/group/topic precedence, self-echo handling, mention gating, and channel_post compatibility.
