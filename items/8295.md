## Summary

Keep #8295 open. Current main still lacks Telegram `allowBots` in config types, strict schema, generated metadata/help, docs, and inbound group/topic runtime filtering. The Plaiground ClawHub skill is a useful Discord workaround, PR #68325 is relevant active work, and PR #8310 was closed unmerged, but none of those ships the requested bundled Telegram plugin behavior on main. Telegram's official bot docs now describe Bot-to-Bot Communication Mode for group chats, so this remains a plausible Telegram plugin feature rather than an impossible cleanup close. ([core.telegram.org](https://core.telegram.org/bots/features))

## Best Possible Solution

Keep #8295 open until the bundled Telegram plugin implements `allowBots` across account/group/topic config types, strict schema, generated metadata/help, docs, and inbound group/topic filtering with regression tests. PR #68325 is the active candidate to review; if maintainers decide Telegram bot-to-bot group delivery is too constrained, document the BotFather mode/admin/privacy requirements and supported alternatives such as Telegram channels or the Plaiground Discord/ClawHub workaround.
