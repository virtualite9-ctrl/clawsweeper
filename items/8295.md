## Summary

Keep #8295 open. Current main still lacks Telegram `allowBots` parity as a supported config/schema/docs/runtime policy for groups/topics. The Plaiground ClawHub skill linked in the discussion is a useful Discord workaround, but it does not implement Telegram group/topic bot-message control in the bundled Telegram plugin.

## Best Possible Solution

Keep #8295 open until the bundled Telegram plugin implements `allowBots` across public config types, strict schema, generated metadata/UI, docs, runtime filtering, and regression/live QA. The runtime should support account/group/topic precedence, always drop self echoes, preserve `channel_post`, and document safe mention/allowlist defaults for loop prevention.
