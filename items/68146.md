## Summary

Keep this PR open. Current main still returns retryable UNAVAILABLE immediately for startup-gated chat.history and models.list requests, with no server-side barrier or queue. Shipped follow-up work retries TUI chat.history, but that is narrower than this PR's gateway-owned hold for cached/pre-retry Control UI bundles and models.list.

## Best Possible Solution

Keep this PR open for maintainer review or a rebased successor. The best product path is a gateway-owned startup barrier or equivalent server-side queue for startup-gated methods, with tests for dispatch after sidecars, timeout fallback, non-gated bypass, and legacy no-barrier behavior; client retries should remain defense in depth.
