## Summary

Keep open. Current main still forwards `openclaw tui --history-limit 0` into `chat.history` as `limit: 0`, while the Gateway protocol rejects `chat.history.limit` values below 1. The PR is a focused two-file fix with regression coverage in the right TUI surface, and the provided Greptile review also confirms the intended behavior and scope.

## Best Possible Solution

Keep the PR open and either land this narrow guard plus regression test after normal maintainer validation, or recreate the same fix on main. The fix belongs in the TUI history-loading path, not in the Gateway schema, because `chat.history.limit: 0` is still invalid by protocol contract while `--history-limit 0` can be interpreted locally as skip history loading.
