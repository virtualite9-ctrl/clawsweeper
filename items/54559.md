## Summary

Keep this issue open. Current main still matches the reported core bug: `generic_repeat` only returns `level: "warning"`, warnings do not block tool execution, and existing tests explicitly preserve warn-only behavior below the global circuit breaker. The related repeated-exec issue (#34574) and open PRs are useful context, but the behavior is not implemented on main yet.

## Best Possible Solution

Keep this issue open until main changes. The clean maintainer path is a narrow loop-detection fix: add a `generic_repeat` critical branch before the warning branch, cover it in `src/agents/tool-loop-detection.test.ts` and `src/agents/pi-tools.before-tool-call.e2e.test.ts`, decide and test the `process(action="list")` classification, and align `docs/tools/loop-detection.md` plus config help if semantics change. Review or replace the open narrow PR #60248 rather than closing the issue before a merge.
