## Summary

Keep this issue open. Current main still has the reported behavior: `generic_repeat` returns only a warning, warning-level loop detections do not block tool calls, and `process(action="list")` is not classified as a known poll tool. The related implementation PR is still open/unmerged, so the best maintainer path is to review or replace that fix rather than close this issue now.

## Best Possible Solution

Keep this issue open until main changes. The best fix is a narrow runtime update in `src/agents/tool-loop-detection.ts` that evaluates `genericRepeat` against `criticalThreshold` before the warning branch, plus focused updates to `src/agents/tool-loop-detection.test.ts`, `src/agents/pi-tools.before-tool-call.e2e.test.ts`, and loop-detection docs/config help. Maintainers should review or replace open PR #60248 and separately decide whether `process(action="list")` belongs in known poll/no-progress detection.
