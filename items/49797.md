## Summary

Current main already implements the user-visible fix from PR #49797: WebUI `/model` sends the raw argument to `sessions.patch`, server-side model resolution handles aliases/allowlist disambiguation, the UI cache stores the server-resolved value after patching, and current tests cover the important raw/qualified model-ref cases. The linked bug #49718 was also closed after a maintainer/Codex review as implemented.

## Best Possible Solution

Close PR #49797 as already implemented on main. Keep the current main design: WebUI sends raw `/model` input to the gateway, the gateway resolves aliases and allowlists, and the UI only normalizes display/cache state after server resolution. Any remaining model-picker edge case should be filed as a narrower current-main bug with a fresh reproduction.
