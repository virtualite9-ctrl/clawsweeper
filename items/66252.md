## Summary

Keep open. Current main implements the per-agent TTS half, but the requested per-agent STT/language/provider override is still missing. The issue discussion already has a maintainer comment saying the same thing and explicitly keeping the issue open for separate STT contract/runtime work.

## Best Possible Solution

Keep this issue open and narrow the remaining work to per-agent STT: define a supported `agents.list[].stt` or equivalent media-audio override contract, wire it into active-agent transcription paths such as media understanding and Discord voice, add schema/type/docs coverage, and preserve the already-shipped `agents.list[].tts` behavior.
