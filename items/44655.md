## Summary

Keep open. Current main has a bundled Z.AI provider, but it still seeds a static GLM catalog and only has a narrow GLM-5 forward-compat resolver. I did not find a Z.AI `/models` fetch path or dynamic picker/list population for the Coding Plan endpoints requested in #44655.

## Best Possible Solution

Keep #44655 open as a Z.AI provider enhancement. The best implementation belongs inside `extensions/zai`: add endpoint-aware `/models` discovery for `api.z.ai` and `open.bigmodel.cn`, preserve the existing static defaults as fallback, expose discovered rows through the provider catalog/picker path without making normal listing brittle, and add focused parser/fallback tests plus docs updates.

## Evidence
