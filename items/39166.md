## Summary

Keep open. Current main includes explicit ACP `runTimeoutSeconds` forwarding, but it still does not implement the PR's central config-level `acp.defaultRunTimeoutSeconds` fallback. The PR remains actionable and should be rebased rather than closed.

## Best Possible Solution

Keep this PR open and ask for a narrow rebase against current main. The best implementation is to preserve the already-landed explicit ACP timeout behavior, add `acp.defaultRunTimeoutSeconds` to the config contract and user-facing docs, resolve ACP timeout as explicit per-call value -> config default -> `0`, apply the resolved value to ACP runtime initialization and gateway dispatch without breaking explicit `0`, and add regression tests for config default, explicit override, and explicit `0` over a non-zero default.
