## Summary

Keep open. Current main already forwards explicit ACP runTimeoutSeconds through sessions_spawn, but it still lacks the PR's central config-level acp.defaultRunTimeoutSeconds fallback for omitted ACP sessions_spawn calls. The linked feature request #38419 was closed stale rather than shipped, so this PR remains a useful implementation candidate after a narrow rebase.

## Best Possible Solution

Keep this PR open and ask for a focused rebase onto current main. Preserve explicit per-call runTimeoutSeconds precedence, including 0; add acp.defaultRunTimeoutSeconds to the public config type/schema/help/labels/docs; resolve ACP timeout as explicit per-call value -> ACP config default -> 0; apply the resolved value consistently to ACP runtime initialization and Gateway dispatch; and keep tests for omitted config default, explicit nonzero override, and explicit 0 overriding a nonzero default.
