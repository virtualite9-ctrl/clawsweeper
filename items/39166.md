## Summary

Keep PR #39166 open. Current main still lacks the PR's central `acp.defaultRunTimeoutSeconds` config fallback for ACP `sessions_spawn`; main only has explicit per-call ACP `runTimeoutSeconds` plumbing. The PR is focused, tied to the closed-but-unimplemented feature request #38419, and remains actionable after a rebase.

## Best Possible Solution

Keep this PR open and ask for a narrow rebase against current main. The best path is to preserve main's existing explicit ACP timeout plumbing, add `acp.defaultRunTimeoutSeconds` to the ACP config contract and user-facing docs, resolve ACP timeout as explicit per-call value -> ACP config default -> `0`, pass the resolved value consistently to ACP runtime initialization and gateway dispatch, and retain focused tests for omitted/default, explicit override, and explicit zero behavior.
