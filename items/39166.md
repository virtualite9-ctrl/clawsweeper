## Summary

Keep PR #39166 open. Current main has explicit per-call ACP `runTimeoutSeconds` plumbing, but it still does not implement the PR's central `acp.defaultRunTimeoutSeconds` config fallback for omitted ACP `sessions_spawn` timeouts. The linked feature request #38419 was closed stale, not shipped, so this PR remains the actionable place for the remaining behavior after a focused rebase.

## Best Possible Solution

Keep this PR open and ask for a narrow rebase against current main. Preserve the existing explicit ACP timeout plumbing, add `acp.defaultRunTimeoutSeconds` to the ACP config contract, resolve ACP timeout as explicit per-call value -> ACP config default -> `0`, apply the resolved value consistently to ACP runtime initialization and Gateway dispatch, and keep focused tests for omitted/default, explicit non-zero override, and explicit `0` overriding a non-zero default.
