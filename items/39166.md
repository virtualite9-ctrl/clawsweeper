## Summary

Keep open. Current main partially implements explicit ACP `runTimeoutSeconds` forwarding, but it still does not implement the PR's central config-level `acp.defaultRunTimeoutSeconds` fallback or the full explicit -> config -> 0 timeout resolution contract. The linked issue #38419 was closed stale, not replaced by a narrower canonical item, so this PR remains actionable after rebasing against the already-landed explicit-timeout work.

## Best Possible Solution

Keep the PR open and ask for a rebase against current main. The best implementation is a narrow update that preserves the already-landed explicit ACP timeout behavior, adds `acp.defaultRunTimeoutSeconds` to the config schema/types/help/labels/generated metadata and docs, resolves timeout as explicit per-call value -> config default -> `0`, and adds tests for config default, explicit override, and explicit `0` overriding a non-zero config default.
