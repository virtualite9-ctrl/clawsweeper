## Summary

Keep #72168 open. The issue is fresh, concrete, independently confirmed in the discussion, and current main still has the reported failure shape: Playwright actions soft-load `pw-ai`, `pw-session` still imports `playwright-core` as a bare package, and the root package still does not declare `playwright-core`. The linked PR #72238 is the right active fix path, but the provided context shows it is open/unmerged and current main does not contain a matching fix commit.

## Best Possible Solution

Keep #72168 open and use PR #72238, or an equivalent maintainer patch, as the fix path. The fix should make `playwright-core` resolvable for packaged browser Playwright actions after `openclaw doctor --fix`, preferably through a browser-local runtime shim or another resolver-visible mechanism, and add regression coverage for packaged global/Homebrew-style installs where plugin deps are staged outside the OpenClaw package tree.
