## Summary

Keep #72168 open. The report is recent, concrete, and describes a plausible packaged-install regression in the browser plugin's Playwright runtime dependency resolution after `doctor --fix`. Current main has generic bundled-runtime dependency staging and mirror logic, but I did not find source, tests, docs, or history proving that the specific failing ESM `playwright-core` import from the built browser Playwright chunk is fixed on main.

## Best Possible Solution

Keep #72168 open and add a focused regression around a packaged install using `dist-runtime/extensions/browser`, an external bundled runtime-deps stage containing `playwright-core`, and the same loader path used by the gateway browser control service. If it reproduces, fix the browser/plugin runtime mirror so the Playwright helper chunk is imported from a location whose ESM package resolution can see the staged `node_modules`, or otherwise provide an explicit supported resolver path for `playwright-core`. Then update `doctor --fix`/browser troubleshooting docs only if the official repair command changes.
