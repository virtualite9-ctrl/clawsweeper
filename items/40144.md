## Summary

Keep open. Current main still retains and evaluates `toolCallHistory` across time gaps, and the two open, unmerged PRs that target this issue are implementation candidates rather than a reason to close the issue before merge.

## Best Possible Solution

Keep this issue open. Review and either land one focused fix from #40217/#40798 or implement a maintainer-owned mainline fix around `src/agents/tool-loop-detection.ts` and `src/agents/pi-tools.before-tool-call.ts` that prunes or ignores stale history before `detectToolCallLoop()` runs, keeps `recordToolCall()` and `recordToolCallOutcome()` consistent, and adds regression coverage for separated heartbeat cycles plus a continuous-loop control case.
