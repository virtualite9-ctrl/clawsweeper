## Summary

Keep open. Current main has the provenance plumbing needed by the requested fix, and adjacent inter-session paths already use it, but `sessions_spawn` still starts child `agent` runs without `inputProvenance` or `sourceTool: "sessions_spawn"`. The linked #47087 draft PR is relevant evidence and appears to target this exact issue, but it is open/draft/unmerged rather than shipped on main.

## Best Possible Solution

Keep #47070 open until a small patch lands on main. The likely fix belongs in the sessions_spawn dispatch path: pass `inputProvenance: { kind: "inter_session", sourceSessionKey: ctx.agentSessionKey, sourceChannel: ctx.agentChannel, sourceTool: "sessions_spawn" }` into the child `agent` request, cover direct subagent spawn with a focused test, and decide explicitly whether the ACP spawn path should use the same metadata.
