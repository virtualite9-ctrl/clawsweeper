## Summary

Keep #8892 open. Current main still does not provide a native `openclaw tui --agent <id>` startup selector. The available alternatives are full `agent:<id>:<session>` keys via `--session`, CWD workspace inference, and post-launch `/agent`; the PR #40176 workspace/subagent improvement is useful but does not satisfy the requested flag for generic directories.

## Best Possible Solution

Keep the issue open until OpenClaw adds a native TUI startup selector. The likely implementation path is to add `--agent <id>` in `src/cli/tui-cli.ts`, add a typed `agentId` or `agent` field to `TuiOptions`, validate it against configured agents, define precedence versus full `--session agent:<id>:...` and CWD workspace inference, and update focused TUI routing tests plus `docs/cli/tui.md` and `docs/web/tui.md`.
