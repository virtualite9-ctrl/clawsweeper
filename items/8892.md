## Summary

Keep #8892 open. Current main still does not implement a native `openclaw tui --agent <id>` selector. The TUI can infer the agent from the current workspace, accept a full `agent:<id>:<session>` key through `--session`, and switch after launch with `/agent`, but those workarounds do not satisfy the requested startup flag. The provided discussion also identifies #8892 as the canonical open request after related #19247 was closed.

## Best Possible Solution

Keep the issue open until OpenClaw adds a native TUI startup agent selector. The likely implementation is to add `--agent <id>` in `src/cli/tui-cli.ts`, add a typed `agentId`/`agent` field to `TuiOptions`, validate it against configured agents, define precedence against full `--session agent:<id>:...` and CWD workspace inference, then update `docs/cli/tui.md`, `docs/web/tui.md`, and focused CLI/TUI routing tests.
