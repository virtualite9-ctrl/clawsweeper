## Summary

Keep #8892 open. Current main still does not provide a native `openclaw tui --agent <id>` startup selector. The TUI can infer an agent from the current workspace, accept a full `agent:<id>:<session>` session key, and switch after launch with `/agent <id>`, but those do not satisfy the requested generic-directory startup routing flag.

## Best Possible Solution

Add a native `--agent <id>` selector for `openclaw tui`: register and validate the flag in `src/cli/tui-cli.ts`, extend `TuiOptions`/`runTui` with a typed startup agent id, define precedence against full `--session agent:<id>:...` and cwd workspace inference, then update focused TUI tests plus `docs/cli/tui.md` and `docs/web/tui.md`. Keep #8892 open as the canonical user-facing request; #19247 is the same request but closed, and PR #40176 only supports workspace inference/cross-agent workspace behavior, not this CLI flag.
