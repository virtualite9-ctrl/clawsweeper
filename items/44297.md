## Summary

Keep open. Current main still handles Slack external arg-menu registration failure as non-fatal, but the only operator-facing signal is `logVerbose`, which is explicitly hidden unless verbose/debug logging is enabled. Regression coverage confirms the static-menu fallback, but does not add a visible warning, health event, startup status, doctor output, or channel status signal. The issue has no protected labels and the author is not a maintainer, but the requested visible degradation signal remains missing.

## Best Possible Solution

Keep the non-fatal static-menu fallback, but add a visible degradation signal in the Slack plugin when external arg-menu registration fails. A conservative fix would log a warning through the normal runtime/logger path and, if the channel health/status contract supports it, publish an account-scoped degraded status or diagnostic event so `openclaw channels status --probe` or doctor/status surfaces can report that external Slack arg menus are disabled.
