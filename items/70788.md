## Summary

Keep open. Current main still generates the Windows Startup-folder launcher as a `.cmd` that runs `start "" /min cmd.exe /d /c ...`; there is no VBS/wscript silent launcher or companion `.vbs` install/uninstall path. The related #57682 context covers a different update/restart surface and does not supersede this login-item bug.

## Best Possible Solution

Implement the silent Windows Startup-folder launcher in `src/daemon/schtasks.ts`: keep the detectable `.cmd` login item, generate an adjacent `.vbs`/`wscript.exe //nologo` hidden launcher or another maintainer-approved no-window shim, remove the companion file on uninstall, and update the Startup fallback tests to assert the no-flash launcher contract.
