## Summary

Keep PR #70497 open. Current main does not yet strip Gemini `if`/`then`/`else` conditional schema keywords: the unsupported-keyword set still omits them, the cleaning loop only skips keys in that set, and the current regression tests cover `not`/union cleanup but not conditional keywords. The linked #69964 is the same unmerged work that was closed because the fork became private, so it does not supersede this reopened PR.

## Best Possible Solution

Keep PR #70497 open and integrate the focused change: add `if`, `then`, and `else` to `GEMINI_UNSUPPORTED_SCHEMA_KEYWORDS`, keep the targeted regression test in `src/agents/schema/clean-for-gemini.test.ts`, then run the targeted schema test plus the changed gate before landing.
