## Summary

Keep this PR open. Current main still loses Discord component/card text before ordinary button/select fallback events reach the agent. The proposed fix addresses a real gap, and the Greptile and prior ClawSweeper comments are directionally useful, but the patch targets the removed `src/discord` layout and needs to be ported into the current `extensions/discord` plugin implementation with regression tests.

## Best Possible Solution

Keep this PR open and ask for a rebase/port into the current Discord plugin. The best patch would add a structured card-context field to `DiscordComponentEntry`, collect `spec.text`, fallback text, and visible text/section block content in `extensions/discord/src/components.ts`, preserve it through the registry, include it in ordinary fallback event text from `extensions/discord/src/monitor/agent-components.ts`, and update `extensions/discord/src/components.test.ts` plus `extensions/discord/src/monitor/monitor.test.ts`. Reconcile the prompt guidance with the current Discord `messageToolHints` path instead of copying the old system-prompt hunk directly.
