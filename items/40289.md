## Summary

Keep this PR open. Current main still drops Discord component/card text before button/select interactions reach the ordinary agent fallback path, while the submitted patch targets the old `src/discord` layout and needs to be ported to the current `extensions/discord` plugin files.

## Best Possible Solution

Keep this PR open and ask for a rebase/port into `extensions/discord`: add a structured card-context field to `DiscordComponentEntry`, collect `spec.text`, fallback text, and visible text/section block content in `buildDiscordComponentMessage`, preserve it through the registry, append it to ordinary button/select fallback event text, and update `extensions/discord/src/components.test.ts` plus `extensions/discord/src/monitor/monitor.test.ts`. Reconcile the prompt-guidance change with the current Discord `messageToolHints` path rather than copying the old `src/agents/system-prompt.ts` patch directly.
