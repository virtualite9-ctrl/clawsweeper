## Summary

Keep this PR open. The bug is plausible and current main still does not have a complete generic `message_received`/internal `message:received` emission at the queued in-flight message boundary. The PR discussion also contains an author follow-up saying the branch remained incomplete after live WhatsApp testing, so this is not safely closeable as implemented, obsolete, or superseded.

## Best Possible Solution

Keep this PR open for maintainer review, but require an update or replacement PR that emits plugin `message_received` and internal `message:received` at the true inbound/enqueue boundary for active-session queued messages across channels. Add regression coverage for the follow-up drain path and explicitly reconcile the generic hook path with WhatsApp’s opt-in privacy behavior.
