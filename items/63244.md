- The PR’s hot-path implementation choice still needs maintainer judgment: per-event session-store reads for subagent/ACP chat and agent events may be acceptable by existing pattern, but reviewer comments flagged caching/performance tradeoffs.
- Adding `spawnedBy` to more event recipients exposes parent session routing metadata to clients already receiving those events. That appears consistent with the current trust model and existing session snapshot exposure, but it should stay additive and scoped to authorized Gateway event recipients.
