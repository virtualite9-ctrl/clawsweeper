## Summary

Keep #8441 open. Current main at 7e13f3f5140168e31027b39a9855ade3fadf24d5 still does not implement top-level `skills.entries.<skill>.thinking` or `skills.entries.<skill>.model`. The type and strict schema omit those keys, tests still reject unknown top-level skill entry fields, command specs do not carry per-skill overrides, and skill invocation only dispatches tools or rewrites prompt text. The related PR #8455 was closed unmerged, and the latest maintainer comment already explicitly kept this item open.

## Best Possible Solution

Keep #8441 open. Implement this in core skill config and invocation plumbing: extend `SkillConfig` and `SkillEntrySchema` with validated optional `model` and `thinking`, carry the defaults through `SkillCommandSpec` or equivalent invocation context, apply them only as per-skill defaults with clear precedence below explicit user/session directives, add focused schema and invocation tests, and update public skills config/thinking docs.
