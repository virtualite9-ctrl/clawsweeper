## Summary

Keep #8441 open. Current main still does not implement first-class `skills.entries.<skill>.thinking` or `skills.entries.<skill>.model`: the skills config type and strict schema omit them, tests still preserve top-level unknown-key rejection, docs do not document them, and skill slash-command invocation only rewrites the prompt body without carrying model/thinking defaults into model selection. The related PR #8455 was closed unmerged, and the latest maintainer-context comment also kept this open.

## Best Possible Solution

Keep #8441 open. Implement this in core skills config and invocation plumbing: add validated optional `model` and `thinking` fields to `SkillConfig` and `SkillEntrySchema`, carry resolved per-skill defaults through `SkillCommandSpec` or an equivalent invocation context, apply them during skill command handling as per-turn defaults with clear precedence against explicit directives/session overrides and agent/global defaults, add schema plus invocation/model-selection tests, and update `docs/tools/skills-config.md` and the public skills config docs.

- `thinking` should use existing thinking-level normalization and provider support/remapping paths so unsupported levels fail or downgrade consistently.
