## Summary

Keep #69386 open. Current main has useful partial mitigations for `/stop`, queue modes, and safer denied-exec prompt wording, but it does not implement the core requested denial-loop fix. Main-session exec denials still start an automatic agent follow-up turn, `runTaskInFlow` has no consecutive-denial stop guard, and the exact Chinese stop phrases reported (`停下来`, `暂停`) are not recognized by the current stop-command tables. Related #69570 tracks the approved async-completion cascade, but it is not a complete duplicate of this denial/stop/queue-overflow report, and PR #69387 was closed unmerged.

## Best Possible Solution

Keep the issue open or split it into explicit follow-ups. The best core fix should add denial-loop protection in the exec approval/session follow-up path, suppress or direct-send denied completion notices instead of starting another main-session agent turn, make stop/cancel phrases including `停下来` and `暂停` highest-priority interrupts that clear pending follow-ups/system events for the session, and cross-link #69570 for the approved async-completion cascade work.
