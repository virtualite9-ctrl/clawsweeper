## Summary

Keep #69386 open. Current main has useful generic stop/abort support, but it still intentionally starts an agent follow-up for denied exec completions in normal sessions, lacks the report's Chinese stop phrases in the Control UI/server stop matchers, and does not show a shipped consecutive-denial circuit breaker or denied-completion suppression. The directly related PR #69387 is still open/unmerged in the provided context, and #69570 remains open for the broader async completion cascade.

## Best Possible Solution

Keep this open or split it into narrower canonical work. The best fix is to add a regression-tested circuit breaker for repeated denied exec approvals, suppress or directly summarize denied async completions instead of starting another agent turn, clear/cancel pending async follow-up queues on stop/chat.abort, make pending async completions visible or cancellable in Control UI, and extend stop recognition/tests for the Chinese phrases reported here. Coordinate with #69387 and #69570 rather than closing this as already fixed.
