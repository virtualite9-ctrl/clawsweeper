## Summary

Keep #30452 open. Current main implements the top-level `agents.defaults.compaction.model` override, but the requested `agents.defaults.compaction.memoryFlush.model` and `memoryFlush.resetAfterFlush` behavior are still missing from config types, strict schema, docs/help labels, memory-flush planning, and the runtime flush path. Existing reset machinery could be reused, but it is not wired to successful memory flushes.

## Best Possible Solution

Keep the issue open or split it into narrower follow-ups. The product path is to add `agents.defaults.compaction.memoryFlush.model` and `memoryFlush.resetAfterFlush` to config types, zod schema, generated schema/help/labels, docs, and relevant UI surfaces; route the silent memory-flush turn through the configured model override; then implement reset-after-flush by reusing the existing gateway session reset machinery after a successful flush while preserving session store updates, queue refresh, transcript handling, active-run cleanup, subagent/ACP state, and expected channel behavior.
