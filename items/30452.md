## Summary

Keep #30452 open. Current main implements the top-level `agents.defaults.compaction.model` override, but the remaining `memoryFlush.model` and `memoryFlush.resetAfterFlush` requests are still absent from config types, strict schema, docs, and runtime routing. The issue discussion also explicitly treated compaction model/thinking work as partial and left flush-then-reset as a separate open question.

## Best Possible Solution

Keep the issue open, or split it into focused follow-ups. The product path is to add `agents.defaults.compaction.memoryFlush.model` and `memoryFlush.resetAfterFlush` to config types, strict schema, generated help/docs/UI surfaces, and tests; route the silent memory-flush turn through the configured model override; then define and test flush-then-reset across session store updates, transcript/session-file handling, queue refresh, active run cleanup, subagent/ACP state, and user/channel-visible behavior.
