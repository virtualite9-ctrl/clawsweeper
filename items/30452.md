## Summary

Keep #30452 open. Current main has the top-level `agents.defaults.compaction.model` override, but it still does not implement the requested `memoryFlush.model` override or `memoryFlush.resetAfterFlush` mode.

## Best Possible Solution

Keep the issue open, or split it into narrower tracked work. The implementation path is to add `agents.defaults.compaction.memoryFlush.model` and `agents.defaults.compaction.memoryFlush.resetAfterFlush` to config types, strict schema/generated schema/help/docs/UI surfaces, route the silent memory-flush turn through the configured model override, and implement flush-then-reset by reusing existing session reset machinery only after a successful flush, with tests for session store updates, queue refresh, transcript handling, active-run cleanup, subagent/ACP state, and channel behavior.
