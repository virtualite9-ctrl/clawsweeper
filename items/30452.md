## Summary

Keep #30452 open. Current main implements the top-level compaction model override, but the requested `memoryFlush.model` and `memoryFlush.resetAfterFlush` behavior are still absent from config types, strict schema, docs, tests, and the memory-flush runtime path. The issue discussion also treated the compaction model/thinking work as partial and left flush-then-reset as a separate open question.

## Best Possible Solution

Keep the issue open, or split it into focused follow-ups. The product path is to add `agents.defaults.compaction.memoryFlush.model` and `memoryFlush.resetAfterFlush` to config types, zod schema, generated schema/help/labels, docs, and relevant UI surfaces; route the silent memory-flush turn through the configured model override; then implement and test flush-then-reset by reusing the existing session reset machinery while preserving session store updates, transcript handling, queue refresh, active-run cleanup, subagent/ACP state, and expected channel behavior.
