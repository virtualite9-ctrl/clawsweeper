## Summary

Keep open. Current `main` still treats Telegram group chats as non-main sessions, and there is no per-group config to collapse a group into the agent's main DM session.

## Evidence

- **Docs say groups are non-main while DMs use main:** The channel docs explicitly say group chats use `agent:<agentId>:<channel>:group:<id>` session keys, while direct chats use the main session.
  - file: [docs/channels/groups.md:70](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/docs/channels/groups.md#L70)

- **Core session-key builder only collapses direct chats to main:** `buildAgentPeerSessionKey` returns the main session only for `peerKind === "direct"` with `dmScope: "main"`; non-direct peers always get a channel-scoped session key.
  - file: [src/routing/session-key.ts:130](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/src/routing/session-key.ts#L130)
