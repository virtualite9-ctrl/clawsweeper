## Summary

Keep PR #63806 open. Current main still has the reported exec workdir behavior: non-sandbox exec passes the raw workdir into `resolveWorkdir`, which calls `statSync` on the raw string and falls back to cwd/home with a warning instead of expanding `~` or failing hard. The PR is a plausible focused fix for linked issue #63742, but it should not land as-is because the provided review context shows unrelated CHANGELOG churn/merge-artifact concerns that need cleanup.

## Best Possible Solution

Keep the PR open and ask for a clean rebase/narrowed patch: update `resolveWorkdir` to expand `~` against the OS home, fail loudly for missing or non-directory workdirs, add focused `resolveWorkdir` tests including file-not-directory, and restore all unrelated CHANGELOG entries. Once the branch is clean, this is a reasonable core fix rather than a ClawHub/plugin item.
