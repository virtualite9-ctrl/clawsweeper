## Summary

Keep open. Current `main` has a partial mitigation for credential redaction, but it still serializes arbitrary non-`Error` objects with `JSON.stringify(err)` and returns those strings through gateway/UI error paths. That does not satisfy the report's core request to stop leaking unfiltered error object contents such as stacks, file paths, and other internal fields.

## Evidence

- **Current formatter still stringifies arbitrary objects:** `formatErrorMessage` handles `Error` instances specially, but for other objects it still falls back to `JSON.stringify(err)` and only then applies `redactSensitiveText`. That is still unfiltered object serialization, not an allowlisted projection.
