## Summary

Keep open. Current main still does not implement the PR’s sender-aware auto-reply exec default, and the PR remains a plausible security fix path rather than obsolete maintainer-cleanup material. It should be revised or reviewed, not closed.

## Best Possible Solution

Keep this PR open for maintainer/security review. The implementation should pass `command.senderIsOwner` into exec override resolution, fail closed for non-owner auto-reply turns unless an explicit non-owner-safe operator policy is configured, ignore user-injectable inline/session exec overrides for non-owner senders, handle partial `agentExecDefaults` by requiring or merging safe `security`/`ask`, and add focused unit tests for no overrides, inline directive injection, persisted session overrides, and partial agent defaults.
