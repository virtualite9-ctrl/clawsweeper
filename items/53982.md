## Summary

Keep this issue open. Current main still documents and displays a Nextcloud Talk bot install command without `--feature response`, while the outbound send path still maps HTTP 401 only to a bot-secret diagnostic. The provided discussion also shows the two related fix attempts were closed unmerged, so there is no canonical shipped fix to supersede this issue.

## Best Possible Solution

Land a narrow Nextcloud Talk plugin/docs fix: update `docs/channels/nextcloud-talk.md` and `extensions/nextcloud-talk/src/setup-surface.ts` to include `--feature webhook --feature response --feature reaction`, improve `extensions/nextcloud-talk/src/send.ts` so 401 errors mention both bot-secret mismatch and missing response feature, and add focused coverage for the setup text or 401 diagnostic.
