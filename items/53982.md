## Summary

Current main has not implemented the requested Nextcloud Talk guidance or diagnostic. The docs and setup wizard still show bot registration with only `--feature reaction`, and outbound 401 handling still reports `authentication failed - check bot secret`. Upstream Nextcloud Talk still requires `Bot::FEATURE_RESPONSE` for bot response endpoints, so the reported root cause remains valid. Two related PRs attempted fixes but both are closed unmerged, and no open PR was found tracking this exact fix.

## Best Possible Solution

Keep this issue open and apply a narrow Nextcloud Talk plugin/docs fix: update the public Nextcloud Talk setup command and setup wizard to include `--feature webhook --feature response --feature reaction`, add a short troubleshooting note for the HTTP 401 symptom, and preferably improve the 401 error text in `extensions/nextcloud-talk/src/send.ts` to mention re-registering with `--feature response`. Preserve the existing English docs page structure and do not edit generated localized docs directly.
