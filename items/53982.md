## Summary

Keep this issue open. Current main still documents and displays a Nextcloud Talk bot install command without `--feature response`, and the outbound send path still maps HTTP 401 only to a bot-secret diagnostic. Upstream Nextcloud Talk stable32 continues to require `FEATURE_RESPONSE` for bot message/reaction responses, and the two related PR attempts are closed unmerged.

## Best Possible Solution

Apply a narrow Nextcloud Talk plugin/docs fix: update `docs/channels/nextcloud-talk.md` and `extensions/nextcloud-talk/src/setup-surface.ts` to use `--feature webhook --feature response --feature reaction`, improve the send-path 401 error in `extensions/nextcloud-talk/src/send.ts` to mention re-registering with `--feature response` while preserving the bot-secret possibility, and add focused coverage for the setup text or 401 send-error behavior.
