## Summary

Keep this issue open. Current main still documents and displays a Nextcloud Talk bot install command with only `--feature reaction`, and outbound HTTP 401 send failures still only point at the shared secret. Upstream Nextcloud Talk docs/source support the report: `response` is the feature that allows bot posts/reactions, and the send path checks bots with `Bot::FEATURE_RESPONSE`.

## Best Possible Solution

Apply a narrow Nextcloud Talk plugin/docs fix: update the English setup command and setup wizard intro to use `--feature webhook --feature response --feature reaction`, add a short troubleshooting note for HTTP 401 replies when the bot was registered without `response`, and improve the send-path 401 error to mention re-registering with `--feature response` while preserving the bot-secret mismatch possibility. Add focused coverage for setup text or send-error behavior.
