## Summary

Current main still does not implement configurable per-session lane concurrency, so this open PR is not obsolete or already implemented. It remains a plausible implementation candidate for part of #53023, but the provided review context shows material blockers that need revision before merge.

## Best Possible Solution

Keep this PR open for maintainer review or contributor revision. The product path is a focused implementation on current main: add an `agents.defaults` session-lane concurrency option to public types, strict zod schema, generated config metadata/help/docs, and resolver/queue tests; store the default in shared queue state; update already-created `session:*` lanes on config reload; and preserve existing gateway and plugin-SDK exports. Keep #53023 open for the broader auto-yield and lane-wait-threshold work unless those are implemented separately.
