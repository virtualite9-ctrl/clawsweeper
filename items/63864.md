## Summary

Keep open. Current `main` still hardcodes per-session lane concurrency to 1 and does not expose a `sessionLaneMaxConcurrent` config path, so the requested behavior is not already implemented. The PR discussion also contains credible unresolved regression findings against the submitted patch (missing schema wiring, hot-reload propagation gap, and removed exports used by gateway/WhatsApp), so a conservative cleanup close would be incorrect.
