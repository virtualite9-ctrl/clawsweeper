## Summary

Keep #8724 open. Current main at 406ae72fd2789c78053f6b94006ec8f488bc1c00 still lacks a typed global or per-model text-generation timeout for active streaming generation loops. The existing surfaces are whole-agent run timeout, LLM idle/no-chunk timeout, and media/tool/request-specific timeouts; those do not implement the requested generation budget for a model that keeps streaming reasoning/planning chunks past a configured limit.

## Best Possible Solution

Keep #8724 open until OpenClaw implements and documents a typed global/per-model text-generation timeout contract. A complete fix should add config type, zod schema, generated schema/help/docs support; resolve the effective timeout per fallback candidate; abort and log total generation-budget expirations; classify them for fallback handling; and add regression coverage for a stream that keeps emitting chunks past the configured budget.
