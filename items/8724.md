## Summary

Keep #8724 open. Current main still lacks a typed global or per-model generation timeout for `agents.defaults.models`, and the existing runtime controls are a global run timeout plus an idle/no-chunk watchdog. The linked first-token/fallback items are adjacent, but they do not solve the reported case where a model keeps streaming non-useful output past a model-specific total budget.

## Best Possible Solution

Keep #8724 open until OpenClaw either implements and documents a typed global/per-model generation-timeout contract or a maintainer explicitly rejects the product direction. A complete fix should validate the config shape, resolve the effective timeout per fallback candidate, keep total generation timeout distinct from idle/no-token behavior, abort and log timeout events, integrate with fallback handling, and add regression coverage for a stream that keeps emitting chunks past the configured budget.
