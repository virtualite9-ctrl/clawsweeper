## Summary

Keep #8724 open. Current main still lacks a typed global/per-model generation timeout under `agents.defaults.models`; existing behavior is a global run timeout plus an LLM idle watchdog, which does not solve streams that keep emitting non-useful output. The latest issue discussion also already kept this open, and current main at e29d3516 still matches that assessment.

## Best Possible Solution

Keep #8724 open until OpenClaw either implements and documents a typed global/per-model generation-timeout contract or a maintainer explicitly rejects the product direction. A complete implementation should validate the config shape, resolve the effective timeout for each selected fallback candidate, keep it distinct from idle/no-token behavior, abort and log timeout events, integrate with fallback handling, and add regression coverage for a stream that keeps emitting chunks past the configured budget.
