## Summary

Current main already implements the behavior this PR is trying to add: dispatch passes `inboundAudio` into the TTS runtime, and speech-core resolves the effective TTS auto mode from session, prefs, and config before gating `auto: "inbound"` on `inboundAudio`. The PR's extra guard is unnecessary and, as Greptile noted, would only look at session-level `ttsAuto`.

## Best Possible Solution

Close this PR as already implemented on current main. Keep the effective-mode gate in speech-core, where session, prefs, and config are resolved consistently; if maintainers want more coverage, add a focused regression test around config-level `messages.tts.auto: "inbound"` plus non-audio/audio contexts rather than merging this session-only pre-guard.
