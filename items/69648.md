## Summary

Keep open. Current `main` still routes MiniMax image understanding through a hardcoded `MiniMax-VL-01` check, while the image-tool model resolver prefers any configured MiniMax model marked `input: ["text", "image"]`. That means a config/catalog exposing `minimax/MiniMax-M2.7` as image-capable can still follow the non-VLM Anthropic path described in the issue. The bundled MiniMax setup intentionally avoids this by keeping M2.7 text-only and using the separate `MiniMax-VL-01` media provider, so the reporter's proposed allowlist tweak is not the full design story, but the underlying mismatch remains unresolved on `main`.
