## Summary

Keep #38248 open. Current main has model cost metadata, post-run usage/cost reporting, provider billing/rate-limit cooldown handling, and external spend-limit workarounds such as LiteLLM, but it still does not implement the requested native request-level rolling spend ceiling for model failover chains.

## Best Possible Solution

Keep the issue open until OpenClaw either adds a native cost guard in the model request/failover path or deliberately designs a blocking plugin/API seam for rolling spend enforcement. A strong implementation would add documented config, hourly/daily accounting, pre-attempt estimated-cost checks, post-response reconciliation from provider usage/cost, pause/reject behavior, owner notification, and regression coverage around fallback chains.
