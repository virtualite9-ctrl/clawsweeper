## Summary

Keep #38248 open. Current main has model pricing metadata, post-hoc usage/cost reporting, provider quota snapshots, and failover cooldown handling for billing/quota errors, but it still does not implement the requested native request-level rolling spend ceiling for fallback chains. The discussion’s external mitigations such as vibe-billing, Kalibr routing, and LiteLLM virtual keys are useful workarounds rather than a shipped OpenClaw cost guard.

## Best Possible Solution

Keep the issue open until OpenClaw either ships a native cost guard in the model request/failover path or explicitly designs a blocking plugin/API seam for rolling spend enforcement. A strong implementation would add documented config, hourly/daily accounting, pre-attempt estimated-cost checks, post-response reconciliation from provider usage/cost data, pause/reject behavior, owner notification, and regression coverage around fallback chains.
