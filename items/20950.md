## Summary

Keep #20950 open. Current main still supports only the aggregate `cron.store` / `jobs.json` definitions file plus a derived `jobs-state.json` sidecar; it does not implement `cron.storeDir`, `jobs.d`, or one JSON file per cron job. The related context still treats #20950 as the canonical open tracker, with #41206 only an open, unmerged implementation candidate and #70531 already closed as a duplicate into this issue.

## Best Possible Solution

Keep #20950 open as the canonical feature tracker until maintainers land #41206 or another implementation. The implementation should add an additive, documented directory-backed cron definitions store, deterministic per-file loading, compatibility with existing `cron.store`, preservation of the current `jobs-state.json` runtime sidecar behavior, config schema/help/docs/doctor updates, and focused load/save/service tests.
