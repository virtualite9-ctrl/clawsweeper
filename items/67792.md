## Summary

Current main still appears to reproduce the reported Telegram model-menu label bug: an OpenRouter ref parsed as provider `openrouter` plus provider-local model `openai/gpt-5.4-mini` still falls back to displaying only the provider-local model id.

## Best Possible Solution

Keep this issue open. The narrow fix belongs in the bundled Telegram plugin: update `extensions/telegram/src/model-buttons.ts` so fallback labels for slash-containing provider-local ids use the canonical `modelKey(provider, model)` display form, while preserving callback data and selection resolution. Add a regression test in `extensions/telegram/src/model-buttons.test.ts` for provider `openrouter` and model `openai/gpt-5.4-mini` expecting visible text `openrouter/openai/gpt-5.4-mini`.
