## Summary

Keep open. The report is recent, detailed, and squarely about OpenClaw core gateway-host exec behavior. Current main still runs non-sandbox host exec by building a shell argv and spawning a child process for each call, and the YOLO preflight change described in the issue is present, but I found no current code, tests, docs, or post-release diff that implements a stale-read/cache-coherence mitigation or supersedes the report.

## Best Possible Solution

Keep the issue open for maintainer investigation. The next useful path is a macOS/APFS reproduction against v2026.4.19, v2026.4.20, v2026.4.24, and current main using gateway-host exec with security=full and ask=off. If confirmed, the fix likely belongs in the core host-exec/runtime path or a documented gateway-host filesystem consistency contract, with regression coverage or a diagnostic that exercises atomic rename followed by reads from separate exec calls.
