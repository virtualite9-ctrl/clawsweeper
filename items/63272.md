## Summary

Keep open. Current main still does not provide `openclaw skills uninstall <slug>` or ClawHub skill uninstall helpers, and this issue is a recent, concrete proposal for a native OpenClaw lifecycle gap. It is not a ClawHub/plugin-scope close: OpenClaw already owns native ClawHub skill search/install/update and records workspace lockfile metadata that update uses later.

## Best Possible Solution

Keep the issue open for maintainer/product review and a focused implementation in `src/agents/skills-clawhub.ts` plus `src/cli/skills-cli.ts`: add a lockfile-aware `openclaw skills uninstall <slug>` with dry-run, confirmation/force semantics consistent with plugin uninstall, path traversal protection, docs, and targeted unit/CLI tests. The proposal should stay scoped to workspace ClawHub skills unless maintainers choose a gateway/UI follow-up.
