## Summary

Keep #39142 open. Current main still has sanitized, fixed fallback copy for billing/rate-limit/model-fallback exhaustion, but no supported configurable `messages.modelExhausted` or equivalent user-facing exhaustion-message setting.

## Best Possible Solution

Keep the issue open until core adds a supported configurable model-exhaustion reply surface or maintainers explicitly choose a different product direction. A complete implementation should add the field to `MessagesConfig`, `MessagesSchema`, generated labels/help/docs, and the fallback failure branch in `runAgentTurnWithFallback`, with focused tests for pure billing exhaustion, pure transient rate-limit exhaustion, and mixed billing/rate-limit behavior.
