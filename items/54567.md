## Summary

Current `main` has not implemented this PR's reorder, and the non-interactive daemon install path still runs `service.install()` before any linger step. The PR addresses a plausible gap, but static inspection also shows the linger helper itself exits early when systemd user services are unavailable, so it is not high-confidence safe to auto-close this PR as already fixed or obsolete.

## Evidence
