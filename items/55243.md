## Summary

Current `main` contains GPT-5/OpenAI Codex follow-through handling for the reported plan-only stall: strict-agentic auto-activation for GPT-5-family OpenAI/OpenAI Codex runs, act-now retry/blocked-state logic for plan-only turns, `update_plan` auto-enable for supported Codex runs, and shipped changelog entries in the 2026.4.x line that explicitly call out this exact failure mode.

## Evidence

- **Strict-agentic now auto-activates for GPT-5 OpenAI/OpenAI Codex runs:** `isStrictAgenticSupportedProviderModel()` covers `openai` and `openai-codex` GPT-5-family models, and `resolveEffectiveExecutionContract()` makes unspecified supported runs default to `strict-agentic` instead of the older looser behavior.
