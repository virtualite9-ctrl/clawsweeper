## Summary

Keep open. Current main still does not implement this PR's selected-session model-family inference path: `resolveSessionModelRef()` returns persisted bare `modelOverride` resolution using the already-resolved default provider, without consulting configured model-family ownership. The PR is also not ready to merge as-is because review comments identify wrapper-provider, stored runtime-provider, and non-OpenAI-default edge cases. Security review found no supply-chain-sensitive file changes in the supplied PR diff.

## Best Possible Solution

Keep this PR open for revision. The best path is to add provider-family inference to the selected-session model resolver, likely through a shared helper used by `resolveSessionModelRef()` and live selected-model paths, while preserving explicit `providerOverride`, wrapper providers with slash-containing model ids, stored runtime-provider semantics, and non-OpenAI defaults. Add focused regression tests for the bare configured-provider case plus the review-comment edge cases.
