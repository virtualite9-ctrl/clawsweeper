## Summary

Keep open. Current main still does not implement this PR's selected-session behavior: a bare `modelOverride` in `resolveSessionModelRef()` is resolved through the persisted override resolver with the already-resolved default provider, without consulting configured model-family ownership. Nearby inference exists for configured defaults, allowlist/model-switch paths, and identity/status display, but not for this resolver path. The PR also needs revision before merge because existing review comments identify wrapper-provider, stored runtime-provider, and non-OpenAI-default edge cases.

## Best Possible Solution

Keep this PR open for contributor or maintainer follow-up. The best implementation is to add provider-family inference to the selected-session model resolver, likely by passing config into a shared helper used by `resolveSessionModelRef()` and live selected-model paths, while preserving explicit `providerOverride`, wrapper providers with slash-containing model ids, stored runtime providers for existing sessions, and non-OpenAI defaults. Add focused regression tests for the bare configured-provider case plus the review-comment edge cases.
