## Summary

Current main already provides an explicit fix path for heartbeat model/style carry-over: heartbeat runs can be isolated into a fresh session with no prior conversation history, and the docs now clearly distinguish model switching from session reset. That implementation is present in the shipped v2026.4.23 release.

## Evidence

## Risks / Open Questions

- This is an explicit mitigation, not an automatic hard reset on every `/model` change. Users who keep reusing the same session without `/new <model>` or `heartbeat.isolatedSession: true` can still carry transcript style across model switches by design.
