## Summary

Keep this issue open. Current main still publishes allowlisted declared commands for an unpaired reconnecting node and `node.invoke` still authorizes from that live session command list. The provided context also shows open, unmerged PR #65169 from the same author with closing syntax for this issue, so the issue should remain open until that PR or an equivalent fix is reviewed and landed or explicitly rejected.

## Best Possible Solution

Keep this issue open and review PR #65169 as the implementation candidate. If the documented node-pairing gate is intended, land that PR or an equivalent fix that suppresses live node commands until approved node-pair state exists and adds regression coverage around `node.invoke`. If maintainers decide the current trusted-operator behavior is intentional, explicitly close the PR and update the pairing docs plus tests to match that product decision.
