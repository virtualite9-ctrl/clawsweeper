## Summary

Keep open. The current AGENTS.md workspace template still tells agents to track checks in memory/heartbeat-state.json using raw Unix epoch examples, and current main does not add the requested warning, timestamp helper, or defensive handling. The report is recent, concrete, and not superseded by the related closed stale-timestamp issue.

## Best Possible Solution

Keep the issue open and make a focused fix in the workspace template first: add explicit guidance to never compute Unix timestamps mentally, require `date +%s` or an equivalent system call, and consider changing the example to null placeholders plus a note. If maintainers want stronger protection, add a small documented helper or doctor check for heartbeat-state.json rather than relying on the model to edit raw epoch fields.
