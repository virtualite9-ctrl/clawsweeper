## Summary

Keep open. Current `main` still publishes the exact footgun the issue reports: the AGENTS template tells users to track checks in `memory/heartbeat-state.json` and shows raw Unix epoch examples, with no guardrail to fetch timestamps via a tool. OpenClaw does now have a better heartbeat path via `HEARTBEAT.md` `tasks:` blocks backed by session-state timestamps, but that newer path does not remove or fix the template guidance that prompted this report.
