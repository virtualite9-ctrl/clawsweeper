## Summary

Keep open. Current main still uses the upstream `estimateTokens()` directly in the compaction and pre-prompt compaction paths, so the guarded malformed-history estimator proposed by PR #63636 is not implemented or superseded on main.

## Best Possible Solution

Keep PR #63636 open. The best path is to land a guarded OpenClaw-owned token estimation helper for compaction/pre-prompt routing, with regression coverage for malformed replay history, then either address the remaining shape-specific fallback accounting in the same PR or track it as a narrow follow-up. Do not close as implemented or duplicate: current main still lacks the core guarded estimator.
