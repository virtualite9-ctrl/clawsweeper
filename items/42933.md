## Summary

Keep open. Current main at 66f4b52 still does not implement the configurable session-level model-error circuit breaker proposed by this PR: no circuitBreaker config/schema/type, no src/agents/circuit-breaker module, and no persisted cb* session state. Existing provider cooldowns, cron backoff, tool-loop circuit breaking, and context-overflow reset behavior are adjacent mitigations, not the requested per-session pause/reset/alert state machine. The PR remains a plausible implementation candidate, but its review thread still raises correctness and alert-delivery validation work before merge.

## Best Possible Solution

Keep this PR open and request a focused revision before merge. The best path is to add a structured model-failure signal from reply/cron execution, persist breaker state on the actual run session including isolated heartbeat sessions, make reset semantics intentional for heartbeat dedupe state, validate outbound alert routing and error text, and cover heartbeat, cron, and interactive reply paths with targeted tests. If maintainers want a smaller change, split the generic session-state/config API from optional alert/reset actions.
