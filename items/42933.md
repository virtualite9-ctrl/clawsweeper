## Summary

Keep open. Current main at 7d9dc8cf24fc143bec95318491e0247c37863e63 still does not implement the requested configurable session-level model-error circuit breaker: there is no circuitBreaker config/schema/type, no src/agents/circuit-breaker module, no persisted cb* session state, and the heartbeat/cron paths do not check, record, or execute per-session pause/reset/alert behavior. Existing provider auth cooldowns, cron job backoff/failure alerts, tool-loop circuit breaking, and context-overflow compaction are adjacent mitigations, not the same per-session state machine. The PR is still a plausible implementation candidate, but its review thread and diff show correctness and alert-delivery/security review work before merge.

## Best Possible Solution

Keep this PR open and request a focused revision before merge. The best path is to add a structured model-failure signal from reply/cron execution, persist breaker state on the actual run session including isolated heartbeat sessions, define reset semantics for heartbeat dedupe state, validate/sanitize alert routing and error text, and cover heartbeat, cron, and interactive reply paths with targeted tests. If maintainers want a smaller change, split the generic session-state/config API from optional alert/reset actions.
