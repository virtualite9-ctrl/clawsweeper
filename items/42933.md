## Summary

Keep open. Current main does not implement the PR’s configurable session-level circuit breaker: there is no `circuitBreaker` config/schema field, no `src/agents/circuit-breaker` module, and no persisted `cb*` session state. Main has adjacent mitigations, including auth-profile cooldowns, cron job error backoff/alerts, and some context-overflow reset behavior, but those do not cover the requested per-session pause/reset/alert state machine across heartbeat and cron sessions. The PR is still potentially useful, but the review thread shows unresolved correctness risks, especially around heartbeat isolated-session state and brittle error detection, so it should be revised rather than closed as obsolete.

## Best Possible Solution

Keep the PR open, but require a revision before merge: add a structured model-failure signal from reply/cron execution, scope breaker state to the actual run session including isolated heartbeat sessions, validate outbound alert routing, preserve/reset heartbeat dedupe intentionally, and cover heartbeat, cron, and interactive reply paths with focused tests. If maintainers prefer a smaller product direction, split the core session-state/config API from optional alert/reset actions rather than closing the feature as already shipped.

## Risks / Open Questions

- The PR’s proposed `alert` action broadens outbound notification behavior from agent config; it should validate channel/account/recipient handling and avoid leaking sensitive error text before merge.
- The review thread identifies a correctness risk where isolated heartbeat failures may mutate or reset the main conversation session rather than the ephemeral heartbeat session.
- The heartbeat detection strategy discussed in review relies on formatted error payload text, which can miss real model failures or count non-model content unless the reply path exposes structured failure metadata.
- Closing this PR now would drop a plausible implementation path while related ClawSweeper reports still say the session-level breaker capability remains unresolved on main.
