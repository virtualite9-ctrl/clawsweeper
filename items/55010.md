## Summary

Current `main` still has the docs mismatch that this PR fixes: the migration guide documents a nonexistent `openclaw/plugin-sdk/allowlist-resolution` subpath, while the actual SDK contract exports `mapAllowlistResolutionInputs` from `openclaw/plugin-sdk/allow-from`. This is a real, still-unfixed docs issue, so there is no conservative basis to close the PR.
