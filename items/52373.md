## Summary

Current `main` does not implement `configUrl` or `configButton` for skills. Skill frontmatter parsing, status types, docs, and the Control UI all still model only `homepage`/`emoji` plus existing gating/install metadata, and the Skills UI has no config-button rendering path.

## Evidence

- **Skill frontmatter parser has no config URL/button fields:** `resolveOpenClawMetadata` only reads `always`, `emoji`, `homepage`, `skillKey`, `primaryEnv`, `os`, `requires`, and `install` from `metadata.openclaw`. There is no parsing for `configUrl` or `configButton`.
  - file: [src/agents/skills/frontmatter.ts:174](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/agents/skills/frontmatter.ts#L174)

- **Skill metadata/status types only expose homepage:** `OpenClawSkillMetadata` and `SkillStatusEntry` include `homepage` but no config-link fields, so the gateway/UI contract does not carry this feature today.
  - file: [src/agents/skills/types.ts:15](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/agents/skills/types.ts#L15)
