## Summary

Keep open. Current main has not implemented the PR's requested capability changes: provider model merge still overwrites explicit `input` with implicit catalog metadata, and gateway image-capability lookup is still provider-scoped with a regression test expecting `false` when only another provider advertises image support. Current main does mitigate silent drops by offloading images for text-only models, but that is not the same as preserving explicit vision capability or safely routing image attachments to the image-capable provider.

## Best Possible Solution

Keep the PR open for maintainer rework. The safest path is to split or narrow the fix: preserve explicit `models.providers.*.models[].input` through `mergeProviderModels` with regression tests, and redesign the cross-provider image-capability behavior only if routing is made image-aware or the fallback is limited to a provider selection that can actually receive the image. Current main's text-only offload should remain as the fail-closed behavior when the selected provider cannot be proven image-capable.
