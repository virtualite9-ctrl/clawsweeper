## Summary

Keep open. Current main has partial mitigations for Gateway memory growth, but the central reported problem is not proven fixed: session-store clone/cache hot paths remain, Gateway history paths still materialize full transcripts before limiting, and there is no current multi-day RSS plateau evidence for the reported cron/channel/subagent workload.

## Best Possible Solution

Keep this issue open as the canonical broad Gateway RSS/session-accumulation tracker. The best maintainer path is to land or supersede the focused #45438/#60444 structuredClone/cache fix, reduce retained dual serialized/object session-store state, stream or cap transcript reads used by Gateway history and sequencing paths, preserve the shipped seqByRun and session-maintenance mitigations, and add targeted regression or soak evidence showing long-running Gateway RSS plateaus under cron, channel traffic, and subagent sessions.
