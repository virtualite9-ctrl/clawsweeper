## Summary

Keep open. Current main has partial fixes and mitigations for contributors named in this issue, but the broader long-running Gateway RSS/session-accumulation problem is not solved: structuredClone-backed session-store cloning remains, whole-transcript materialization remains on active Gateway paths, and compaction transcript rotation is still opt-in/default-off.

## Best Possible Solution

Keep this issue open as the canonical broad Gateway memory-growth tracker. The best maintainer path is to land or supersede the #45438 structuredClone fix, reduce or eliminate retained dual serialized/object session-store state, stream or cap transcript reads used by Gateway sequencing/broadcast paths, decide whether compaction transcript rotation should become a safer default or have stronger automatic policy, and add targeted regression or soak evidence showing long-running Gateway RSS plateaus with cron, channel traffic, and subagent sessions.
