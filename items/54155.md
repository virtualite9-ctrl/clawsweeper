## Summary

Keep this issue open. Current main contains useful mitigations for session maintenance and the seqByRun leak, but the broader Gateway RSS/session-accumulation report is still not proven fixed: the structuredClone session-cache path remains, and several history surfaces still materialize full transcript files before applying response limits.

## Best Possible Solution

Keep this issue open as the broad Gateway RSS/session-accumulation tracker. The best maintainer path is to finish or explicitly supersede #45438, remove or bound the structuredClone/session-cache retention path, avoid full transcript materialization before limits for hot Gateway history surfaces, preserve the shipped seqByRun and session-maintenance mitigations, and add long-running RSS/profiling evidence showing the gateway plateaus under cron jobs, channel traffic, and subagent sessions.
