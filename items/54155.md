## Summary

Keep this issue open. Current main has useful mitigations for session maintenance, session-list enrichment, and the seqByRun contributor, but the broader Gateway RSS/session-accumulation report is still not proven fixed: the structuredClone session-cache path remains, full session-store loads still clone the whole store, and hot history surfaces still materialize full transcript files before applying limits.

## Best Possible Solution

Keep this issue open as the broad Gateway RSS/session-accumulation tracker. The maintainer path is to finish or supersede #45438, replace or bound the structuredClone/session-cache retention path, avoid full transcript materialization before limits on hot Gateway history/sequence surfaces, preserve the shipped seqByRun and session-maintenance mitigations, and add long-running RSS/profiling evidence showing the gateway plateaus under cron jobs, channel traffic, local embeddings, and subagent sessions.
