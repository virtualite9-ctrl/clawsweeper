## Summary

Current `main` already uses canonical `agent:<agentId>:subagent:<uuid>` child session keys for `sessions_spawn` and dispatches those runs on the dedicated `subagent` lane; regression tests and docs match that behavior, and the latest tagged release contains the same implementation.
