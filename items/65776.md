## Summary

Keep open. Current main still supports only configured/static headers for OpenClaw-managed remote MCP servers; there is no request-scoped header resolver, headersProvider, or session/runtime-context templating path for streamable-http or SSE transports. The prior discussion correctly distinguishes this from #65590, which was about forwarding static custom headers, and from #62776, which was about OAuth flows.

## Best Possible Solution

Keep this open for maintainer design. The best path is a narrow core MCP runtime capability or public plugin API extension that accepts a typed request-scoped header resolver, threads safe session/runtime context into MCP transport requests, merges dynamic headers after static config with explicit precedence and redaction rules, supports both streamable-http and SSE, and adds docs plus regression tests for tenant/user/token-refresh behavior. #65590 should remain treated as the static-header forwarding bug, not the canonical tracker for this broader dynamic-header capability.
