## Summary

Current `main` now returns and persists a real `anthropic:claude-cli` auth profile during Anthropic Claude CLI login, and it also seeds the `claude-cli/*` default-model allowlist/status path that previously showed as missing. The fix is present on `main` in commit `51c11cfd903033947f01f96a7169ebf92d194064`, but it does not appear to be part of the latest tagged release `v2026.4.23`.

## Evidence

- **Claude CLI migration now returns persisted auth profiles:** `buildAnthropicCliMigrationResult` now builds a real `anthropic:claude-cli` OAuth or token profile from detected Claude CLI credentials and returns it in `profiles`, covering the reported empty-profile bug.
  - file: [extensions/anthropic/cli-migration.ts:127](https://github.com/openclaw/openclaw/blob/51c11cfd903033947f01f96a7169ebf92d194064/extensions/anthropic/cli-migration.ts#L127)
  - command: `nl -ba extensions/anthropic/cli-migration.ts | sed -n '1,280p'`
  - sha: [51c11cfd9030](https://github.com/openclaw/openclaw/commit/51c11cfd903033947f01f96a7169ebf92d194064)
- **Model auth login persists returned profiles to auth-profiles.json:** `persistProviderAuthResult` iterates over `result.profiles`, calls `upsertAuthProfile(...)`, and applies matching auth config updates, so provider-owned auth results are written to the auth store instead of being dropped.
  - file: [src/commands/models/auth.ts:231](https://github.com/openclaw/openclaw/blob/51c11cfd903033947f01f96a7169ebf92d194064/src/commands/models/auth.ts#L231)
