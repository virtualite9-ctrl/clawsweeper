## Summary

Keep PR #38640 open. Current main still does not implement the PR's prompt-level LLM retry wrapper: the embedded runner calls `activeSession.prompt(...)` directly, and the existing retry/failover behavior is profile rotation, model fallback, SDK retry-wait capping, and channel/generic retry infrastructure rather than bounded same-call prompt retry with exponential backoff. Related retry work #49376/#61443 remains open rather than landed or explicitly superseding this PR.

## Best Possible Solution

Keep #38640 open for maintainer judgment. The best path is to either fold this work into the configurable LLM/provider retry direction tracked by #49376/#61443, or rework the PR at the chosen runner/provider boundary with abortable backoff, shared failover-status parsing, clear interaction with SDK retries and model fallback, and focused tests for retry classification, retry-after parsing, and cancellation behavior.
