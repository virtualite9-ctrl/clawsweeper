## Summary

Keep PR #38640 open. Current `main` still does not ship the prompt-level LLM retry wrapper proposed by the PR. Main has adjacent mitigation through provider SDK retry-wait caps, auth-profile cooldown/rotation, overload backoff, and model fallback, but those are not the same as retrying `activeSession.prompt(...)` with a bounded exponential backoff loop. The related configurable retry work remains open rather than landed, so this PR is not safely closeable as implemented or superseded.

## Best Possible Solution

Keep #38640 open for maintainer judgment. The best path is to either fold it into the open configurable LLM/provider retry direction (#49376/#61443), or rework this PR at the chosen runner/provider boundary with abortable backoff, shared failover status parsing, clear interaction with SDK retries and model fallback, and focused tests for retry parsing plus cancellation behavior.
