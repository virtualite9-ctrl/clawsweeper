## Summary

Keep open. Current main still matches the reported failure modes: the TUI watchdog is armed from chat delta events, not active-run tool events, and reconnect reloads history without clearing active streaming state. The related fix PR #69125 was closed unmerged, while #69978 is only a follow-up UX request, not a canonical fix for this bug.

## Best Possible Solution

Keep the issue open and implement a focused TUI fix: count active-run tool events as watchdog proof-of-life even when tool rendering is suppressed, and reconcile reconnect state by clearing or re-adopting `activeChatRunId` around `loadHistory()` or by adding reliable missed-final replay. Add regression coverage for long active tool phases and reconnect-after-missed-final behavior.
