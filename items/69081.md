## Summary

Keep open. Current main still matches the reported TUI desync root causes: the streaming watchdog is armed only from chat deltas, tool events return without refreshing it, and reconnect reloads history without clearing the stale active run/watchdog state. The related PR #69125 directly targeted this issue but is closed unmerged, and #69978 is a separate watchdog UX/configuration follow-up rather than a canonical fix for this bug.

## Best Possible Solution

Keep #69081 open and land a focused TUI fix: refresh the streaming watchdog for active-run agent tool activity, preferably before verbose gating, and add reconnect recovery that clears stale active run/watchdog state before `loadHistory()` or exposes an event-handler reset method used by `client.onConnected`. Add timer-based regression tests for long tool activity and reconnect-with-missed-final behavior. Treat #69978 as a separate UX/configuration follow-up, not the canonical bug fix.
