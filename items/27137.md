## Summary

Keep #27137 open. Current main still has shared DM/group policy primitives, but it does not implement the requested unified `authorizeIngress()` / `IngressContext` gate, and representative ingress surfaces continue to authorize through event- or channel-local helpers. The prior Codex review comment in the issue also kept this open, and the related #49782 context is an unrelated ACP completion relay RFC, not a canonical duplicate.

## Best Possible Solution

Keep #27137 open for maintainer design. The likely path is a narrow core/security or SDK-backed ingress authorization helper that centralizes DM/group policy evaluation, allowlist merging, command gating, event-kind handling, and reason codes while channel adapters supply identity normalization and chat-context classification. Adoption should migrate representative message, reaction, pin, interaction, callback, slash, and voice paths, with contract tests proving new ingress surfaces cannot bypass the shared gate.
