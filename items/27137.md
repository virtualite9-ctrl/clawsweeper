## Summary

Keep #27137 open. Current main still lacks the requested unified `authorizeIngress()` / `IngressContext` gate, and representative ingress surfaces continue to perform channel- or event-local authorization around narrower shared DM/group policy helpers. The existing maintainer-side Codex comment in the issue already favored keeping this open, and current main still supports that conclusion.

## Best Possible Solution

Keep #27137 open for maintainer design. The best path is a narrow core/security or SDK-backed ingress authorization helper that centralizes DM/group policy, allowlist merging, command gating, event-kind handling, and reason codes while allowing channel adapters to supply identity normalization and chat-context classification. Adoption should migrate representative message, reaction, pin, interaction, callback, slash, and voice paths, with contract tests proving new ingress surfaces cannot bypass the shared gate.
