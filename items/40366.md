## Summary

Keep this PR open. Current main still lacks the two central behaviors: normalized-empty agent command outputs can fall through without a synthesized fallback, and Slack draft/progress turns with no final reply still clear state and return without the PR's in-thread no-final fallback. The PR discussion also contains useful guardrails already worked through in review, so the best maintainer path is review/rebase/scope cleanup rather than closing.

## Best Possible Solution

Keep this PR open for maintainer review or contributor rebase. The best path is to port the normalized-empty fallback into `src/agents/command/delivery.ts` after outbound projection and the Slack preview-only no-final fallback into `extensions/slack/src/monitor/message-handler/dispatch.ts`, while preserving the review-driven guardrails and dropping unrelated UI formatting churn before merge review. Related #45290 is broader delivery recovery work, but it is not established as the canonical replacement for this PR's normalized-empty command fallback and Slack preview-only no-final behavior.
