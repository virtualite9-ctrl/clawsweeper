## Summary

Keep this PR open. Current main still lacks the two central behaviors: fallback delivery when agent command payloads normalize to no deliverable content, and Slack draft/progress turns that end without a final reply still clear state and return without an in-thread fallback. The PR discussion is useful because the author addressed several concrete false-positive and threading concerns raised by bot review.

## Best Possible Solution

Keep this PR open for maintainer review or contributor rebase. The best path is to port the normalized-empty command fallback into `src/agents/command/delivery.ts` after outbound projection, and the Slack preview-only no-final fallback into `extensions/slack/src/monitor/message-handler/dispatch.ts`, while preserving the reviewed guardrails for intentional `NO_REPLY`, messaging-tool successes, block-stream successes, same-thread fallback status, draft cleanup, and JSON output. Ask the contributor to remove unrelated UI formatting churn before serious merge review.
