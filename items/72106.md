## Summary

Close as a duplicate of #65839. This issue reports the same remaining user-visible problem already tracked there: the context-overflow reset advice points users at `agents.defaults.compaction.reserveTokensFloor` set to `20000`, but that hard-coded guidance is not sufficient to resolve all overflow cases. Current main still emits that exact advice, while the config key itself is supported and covered, so the useful remaining work belongs on the canonical guidance/diagnosis issue.

## Best Possible Solution

Close this item as a duplicate and keep #65839 as the canonical issue. The best fix is a shared context-overflow guidance helper in `src/auto-reply/reply/agent-runner-execution.ts`, tested for both reset variants, that gives model/context-window-aware advice instead of hard-coding `20000`; coordinate heartbeat-specific diagnosis with #67314 if that path is implicated.
