## Summary

Keep open. Current `main` still relies on per-result persistence caps plus overflow-triggered truncation/pruning; there is no evidence of a generally-on aggregate replay budget for normal non-Anthropic main sessions, and the same core overflow/truncation paths were already present in `v2026.4.23`.

## Evidence

- **Per-result cap only on normal persistence:** The session guard computes `maxToolResultChars` and passes it into `installSessionToolResultGuard(...)` when messages are persisted. This is a single-tool-result cap, not an always-on aggregate replay budget across multiple tool results.
