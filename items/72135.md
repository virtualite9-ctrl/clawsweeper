## Summary

Keep open. Current main does not implement `===DONE_OK===` or an equivalent cron-success sentinel, and the PR targets the core cron outcome path that still maps trailing fatal payload classification into `status: "error"` and failure alerts. The related title-search items cover different cron surfaces, not this sentinel behavior.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is a core cron change in `resolveCronPayloadOutcome`, with regression coverage for sentinel-in-final-text, sentinel-in-payload-text, no-sentinel trailing errors, and `runLevelError` overriding the sentinel. This belongs in OpenClaw core because cron run status and failure alerts are core runtime behavior, not a ClawHub/plugin boundary.
