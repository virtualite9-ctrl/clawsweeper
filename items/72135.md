## Summary

Keep open. Current main does not honor `===DONE_OK===` or any equivalent cron-success sentinel, and the existing classifier still marks a successful text payload followed by a trailing error payload as fatal. The PR is a focused, active draft with relevant tests proposed and no protected close condition.

## Best Possible Solution

Keep the PR open for maintainer review. If accepted, implement the success-marker behavior in `src/cron/isolated-agent/helpers.ts`, add focused resolver regression tests, verify downstream `hasFatalErrorPayload` consumers do not need observability for `hasDoneOkSentinel`, and decide the final token name/gating policy before merge.
