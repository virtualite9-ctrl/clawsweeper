## Summary

Keep this issue open. Current main still has no `brave.http` diagnostics flag, no Brave-specific request/status/timing/cache hit/miss logging, and no docs for such a flag. The request is actionable in the bundled Brave plugin using existing diagnostics helpers and the web-search provider context, so it should not be closed as stale, duplicate, ClawHub-only, or implemented.

## Best Possible Solution

Keep this issue open. Implement the feature in `extensions/brave` by passing the full OpenClaw config or a precomputed `isDiagnosticFlagEnabled("brave.http", config)` result into the Brave runtime, adding a Brave diagnostic logger for sanitized outbound URL/query/params, response status and duration, and cache hit/miss events around the existing cache paths. Add focused Brave tests and document the flag in `docs/diagnostics/flags.md` and `docs/tools/brave-search.md`.
