## Summary

Keep this issue open. Current main still has no `brave.http` diagnostics flag, no Brave-specific request/status/timing/cache hit/miss logging, and no docs for such a flag. The request is actionable in the bundled Brave plugin using existing diagnostics helpers and the existing web-search provider context.

## Best Possible Solution

Keep this issue open. Implement the feature in `extensions/brave` by passing the full config or a precomputed `isDiagnosticFlagEnabled("brave.http", config)` boolean into the Brave runtime, then add sanitized diagnostic logs for outbound request URL/query/params, response status and duration, and cache hit/miss around both Brave modes. Add focused Brave tests and document the flag in `docs/diagnostics/flags.md` and `docs/tools/brave-search.md`.
