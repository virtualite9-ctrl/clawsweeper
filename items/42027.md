## Summary

Keep open. Current main partially supersedes the exec and cron pieces, but the browser layered diagnostics and HTML upstream-error summarization from this PR are not implemented on main, and the PR still has plausible user-facing value from the WSL2/CDP discussion. It is not merge-ready as-is because main has moved the browser code under the browser plugin and the review comments need resolution.

## Best Possible Solution

Keep this PR open, but do not merge it as-is. The maintainer path is to rebase or split it: drop or reconcile the exec and cron pieces that current main already solved differently, port the still-missing browser diagnostics and HTML error summarization into `extensions/browser`, address the two review comments, then run the changed browser/cli/agents/cron gates and a focused security review.
