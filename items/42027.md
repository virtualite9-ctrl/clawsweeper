## Summary

Keep this PR open. Current main already solved or reshaped the exec PATH fallback and cron self-deadlock pieces, and docs now cover WSL2 layered browser troubleshooting, but the product-level browser status diagnostics and HTML upstream-error summarization from the PR are still missing in the current `extensions/browser` plugin layout.

## Best Possible Solution

Keep the PR open, but require a rebase or split. Drop or reconcile the exec and cron portions already solved on main, then port the still-missing browser layered status diagnostics and bounded HTML upstream-error summarization into `extensions/browser`. Address the two existing review comments and verify with targeted browser CLI/client/server tests plus a focused security review of the status/config and error-reflection paths.
