## Summary

Keep open. Current `main` still lacks the CLI-metadata memory-capability registration that this PR adds, so the reported `openclaw wiki bridge import` failure remains plausible and is not already implemented on `main`.

## Evidence

- **`memory-core` CLI metadata entry still registers only CLI commands:** On current `main`, the lightweight `extensions/memory-core/cli-metadata.ts` entry has no `kind: "memory"` and never calls `api.registerMemoryCapability(...)`; it only registers the `memory` CLI command.
  - file: [extensions/memory-core/cli-metadata.ts:3](https://github.com/openclaw/openclaw/blob/2cd2732ab6695a4b3220ce41166b74aca7097c5f/extensions/memory-core/cli-metadata.ts#L3)
  - command: `nl -ba extensions/memory-core/cli-metadata.ts | sed -n '1,80p'`
  - sha: [2cd2732ab669](https://github.com/openclaw/openclaw/commit/2cd2732ab6695a4b3220ce41166b74aca7097c5f)
- **CLI-metadata loader does not wire memory capability registration:** `loadOpenClawPluginCliRegistry` builds the plugin API for `registrationMode: "cli-metadata"` with only a `registerCli` handler. There is no `registerMemoryCapability` handler in this path on current `main`.
  - file: [src/plugins/loader.ts:3295](https://github.com/openclaw/openclaw/blob/2cd2732ab6695a4b3220ce41166b74aca7097c5f/src/plugins/loader.ts#L3295)
  - command: `nl -ba src/plugins/loader.ts | sed -n '3290,3325p'`
  - sha: [2cd2732ab669](https://github.com/openclaw/openclaw/commit/2cd2732ab6695a4b3220ce41166b74aca7097c5f)
- **Missing handler means `registerMemoryCapability` is a no-op:** `buildPluginApi` falls back to `noopRegisterMemoryCapability` when no handler is supplied, so a plugin calling `api.registerMemoryCapability(...)` during CLI-metadata loading would be silently ignored on current `main`.
  - file: [src/plugins/api-builder.ts:112](https://github.com/openclaw/openclaw/blob/2cd2732ab6695a4b3220ce41166b74aca7097c5f/src/plugins/api-builder.ts#L112)
  - command: `nl -ba src/plugins/api-builder.ts | sed -n '100,195p'`
  - sha: [2cd2732ab669](https://github.com/openclaw/openclaw/commit/2cd2732ab6695a4b3220ce41166b74aca7097c5f)
- **`wiki bridge import` depends on active memory public artifacts:** The bridge import path reads `listActiveMemoryPublicArtifacts(...)` before collecting artifacts and later prunes imported entries from the sync state. If the active memory capability is never registered, this flow has no artifacts to import.
  - file: [extensions/memory-wiki/src/bridge.ts:224](https://github.com/openclaw/openclaw/blob/2cd2732ab6695a4b3220ce41166b74aca7097c5f/extensions/memory-wiki/src/bridge.ts#L224)
  - command: `nl -ba extensions/memory-wiki/src/bridge.ts | sed -n '220,255p'`
  - sha: [2cd2732ab669](https://github.com/openclaw/openclaw/commit/2cd2732ab6695a4b3220ce41166b74aca7097c5f)
- **No active memory capability produces an empty artifact list:** `listActiveMemoryPublicArtifacts` returns `[]` when `memoryPluginState.capability?.capability.publicArtifacts` is absent, matching the PR's root-cause description.
  - file: [src/plugins/memory-state.ts:274](https://github.com/openclaw/openclaw/blob/2cd2732ab6695a4b3220ce41166b74aca7097c5f/src/plugins/memory-state.ts#L274)
