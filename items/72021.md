## Summary

Keep #72021 open. Current main still implements the reported mixed-signal behavior: `minRecallCount` is checked against `recallCount + dailyCount + groundedCount`, daily and session ingestion are recorded as daily signals, and existing tests explicitly assert that dreaming-only daily signals can pass the default promotion gates. The provided related items are title-overlap matches around session rollover/daily hooks and do not supersede this promotion-ranking bug.

## Best Possible Solution

Keep the issue open and fix `memory-core` short-term promotion so `minRecallCount` is based on real `recallCount` evidence, not daily/session/grounded totals. Daily, session, grounded, light, and REM evidence can still contribute through separately named weighted components or thresholds, but the default recall gate should not be satisfied by `recallCount=0`. Add regression tests for both a daily-only false candidate and a real `recallCount >= 3` candidate that should remain reachable under the default deep thresholds.
