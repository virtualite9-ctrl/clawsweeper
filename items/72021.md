## Summary

Keep open. Current main still implements the reported mixed-signal behavior: promotion gates `minRecallCount` using `signalCount`, and `signalCount` is still the sum of recall, daily, and grounded counts. The behavior is also intentionally covered by tests and described in docs as using both memory recalls and daily-ingestion signals, so this needs maintainer product judgment rather than cleanup closure.

## Best Possible Solution

Keep this issue open for maintainer product judgment. If accepted, split the gate semantics so `minRecallCount` checks real `recallCount` only, keep daily/grounded/session evidence as a separately named signal metric, recalibrate scoring/default thresholds, and update the current daily-only promotion tests, CLI diagnostics, and docs to match the chosen semantics.
