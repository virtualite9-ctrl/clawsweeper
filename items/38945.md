## Summary

Current `main` does not implement this PR’s Unicode tokenizer fixes. The memory-core MMR tokenizer still only extracts ASCII word tokens plus special CJK handling, and the FTS query builder still omits combining marks, so the reported behavior remains relevant on current `main` even though the code has moved and evolved.

## Evidence
