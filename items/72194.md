## Summary

Current main, and the reported v2026.4.24 release, already expose a supported config path for multi-attachment media understanding and preserve multi-image Telegram media for native vision-capable model turns. The remaining default of 1 is a conservative default, not a hard client-side block requiring dist patching.

## Best Possible Solution

Keep the shipped conservative default and direct users who need multi-image pre-analysis to the existing `tools.media.image.attachments` config. Native vision-capable Claude/OpenAI-style turns should continue using the existing multi-image media-note and prompt-image injection path.
