## Summary

Keep #46833 open. Current main has backend session listing plus Control UI session switching through the chat header and Sessions tab, but the left Control UI sidebar still renders only static navigation groups and does not display active/channel sessions there. The closed #46835 is the same duplicate report, not a canonical open replacement for the remaining sidebar work.

## Best Possible Solution

Implement this as a focused Control UI enhancement: add a compact active/recent channel-session section to the left sidebar or an adjacent chat sidebar, backed by sessions.list/sessions.changed, with channel/group labels from existing GatewaySessionRow metadata and click-through behavior using the existing switchChatSession path. Keep the existing Sessions tab for full management and filtering.
