## Summary

Keep open. Current main still does not implement this PR's duplicate-installation doctor check. The shipped code has adjacent gateway-service and gateway-lock hardening, but it does not scan multiple OpenClaw binaries, compare versions, or emit the duplicate-installation remediation note proposed by #35152/#35129.

## Best Possible Solution

Keep #35152 open until a maintainer either refreshes the duplicate-installation doctor check against the current src/flows/doctor-health.ts flow with safe execFileSync-style probing, npm prefix-aware path discovery, deep-scan semantics, and tests, or explicitly decides that the remaining duplicate-installation warning is no longer wanted because gateway-service hardening is sufficient.
