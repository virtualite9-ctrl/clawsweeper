## Summary

Keep #35152 open. Current main still does not implement an on-demand `openclaw doctor` duplicate-installation diagnostic; it only has source-workspace install hygiene, gateway service drift/legacy-service checks, and an installer-time duplicate global-install warning.

## Best Possible Solution

Keep #35152 open until it is refreshed or superseded by a current doctor health contribution that safely detects duplicate OpenClaw binaries/install roots, reports versions and relevant service metadata, respects current `doctor --deep` service-scan semantics, deduplicates by realpath, uses safe process APIs, derives npm package/bin locations correctly, and includes focused tests.
