## Summary

Keep this PR open. Current main still lets Discord startup continue with `botUserId` missing when `fetchUser("@me")` throws or returns an unusable identity, and the PR is a focused implementation candidate for the linked open security bug. However, the submitted one-line fix is incomplete and stale against the current extension path, so the right maintainer path is review/revise, not close as cleanup.

## Best Possible Solution

Keep the PR open, but request a rebase onto current main and a complete Discord-side fail-fast fix in `extensions/discord/src/monitor/provider.startup.ts`: throw when `fetchUser("@me")` rejects or returns no normalized `id`, and add focused tests for both cases. After that, review whether the `botId &&` guard in preflight should be removed or left as defense against channels that rely on mention-pattern fallback.
