## Summary

Keep this PR open. Current main still allows Discord startup to continue without a usable bot identity, and this PR is a focused implementation candidate for the linked open security bug #42219, though the patch is stale and incomplete.

## Best Possible Solution

Keep this PR open, request a rebase onto current main, and revise the fix in extensions/discord/src/monitor/provider.startup.ts so Discord startup fails when fetchUser("@me") rejects or returns no normalized bot id. Add focused regression tests for rejection and invalid identity responses, then consider tightening the preflight botId && mention-gate guard as defense in depth for #42219.
