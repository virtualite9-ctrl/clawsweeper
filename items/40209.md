## Summary

Keep this PR open. Current main still keeps system-event queues case-sensitive while heartbeat session resolution canonicalizes agent-scoped session keys to lowercase before peeking/draining, so the channel-session heartbeat bug described by the PR remains plausible and is not implemented on main.

## Best Possible Solution

Review and either land this focused PR with its mixed-case lookup and cross-case dedupe regression tests, or adjust the implementation to canonicalize only recognized agent-scoped session keys before merge. Closing this PR now would drop a plausible fix for the still-present heartbeat/system-event key mismatch.
