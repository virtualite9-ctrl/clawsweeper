## Summary

Keep PR #40209 open. Current main still stores system events under the raw trimmed session key, while targeted heartbeat resolution canonicalizes agent session keys to lowercase before peeking or consuming queued events. The linked issue #34338 was closed stale, but the source-level mismatch remains on main and the PR’s focused regression tests are still absent.

## Best Possible Solution

Keep PR #40209 open for maintainer review or landing. The best path is to make system-event session-key canonicalization match heartbeat/session routing and add focused regression tests for mixed-case channel session lookup plus cross-case deduplication. If broad lowercasing is too wide, adjust the patch to canonicalize only recognized agent-scoped session keys instead of closing the PR.
