## Summary

Keep this PR open. Current main still has the case-sensitive system-event queue behavior that the PR is trying to fix, while heartbeat session resolution canonicalizes agent-scoped session keys to lowercase before peeking or draining pending events.

## Best Possible Solution

Keep this PR open for maintainer review or landing. The best path is to align system-event session-key canonicalization with heartbeat/session routing and keep the mixed-case channel lookup plus cross-case dedupe regression tests; if broad lowercasing is considered too wide, adjust the patch to canonicalize only recognized agent-scoped keys.
