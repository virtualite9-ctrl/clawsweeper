## Summary

Keep open. Current main has tracked browser-tab cleanup for idle/excess sweeps and lifecycle/session reset paths, but it still does not automatically close primary-agent tabs/windows at ordinary task completion.

## Best Possible Solution

Keep this issue open for a focused browser lifecycle change: add an ephemeral marker for automation-opened tabs, preserve explicit/labeled/pinned/user-owned tabs, wire cleanup into ordinary primary-agent task or turn completion, and keep the current idle/excess sweep plus reset/subagent/cron cleanup as fallback behavior. Add tests for primary-agent successful completion, explicit keep-open preservation, and existing-session safety.

## Risks / Open Questions

- Changing the default can regress workflows that intentionally keep primary-agent browser tabs reusable; the implementation needs a clear ephemeral versus preserved tab policy.
- Existing-session profiles attach to a user's running browser, so completion cleanup must not close manually opened or explicitly preserved user tabs.
