## Summary

Keep open. Current main has partial browser-tab cleanup for detached session lifecycles, but not strong evidence that ordinary task completion now auto-closes temporary browser tabs/windows by default. The implementation tracks tabs per session and cleans them on subagent completion, isolated cron completion, or session reset/delete, which is narrower than the issue request.

## Evidence

- **Browser-opened tabs are tracked, not auto-closed at open time:** `browser open` records the opened tab under the agent session key, while explicit `browser close` removes it from tracking. There is no per-task auto-close in this path.
  - file: [extensions/browser/src/browser-tool.ts:534](https://github.com/openclaw/openclaw/blob/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503/extensions/browser/src/browser-tool.ts#L534)
  - sha: [b0d2003fe65b](https://github.com/openclaw/openclaw/commit/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503)
- **Cleanup primitive is session-lifecycle scoped:** `cleanupBrowserSessionsForLifecycleEnd` only closes tracked tabs for provided session keys and is named/structured as lifecycle-end cleanup, not normal turn completion.
  - file: [src/browser-lifecycle-cleanup.ts:15](https://github.com/openclaw/openclaw/blob/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503/src/browser-lifecycle-cleanup.ts#L15)
  - sha: [b0d2003fe65b](https://github.com/openclaw/openclaw/commit/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503)

  - file: [src/agents/subagent-registry-lifecycle.ts:721](https://github.com/openclaw/openclaw/blob/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503/src/agents/subagent-registry-lifecycle.ts#L721)
