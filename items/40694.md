## Summary

Keep open. Current main has tracked-tab cleanup for reset/lifecycle paths and background idle/excess sweeps, but it still does not close temporary primary-agent browser tabs/windows at ordinary task completion by default.

## Best Possible Solution

Keep this issue open for a focused browser lifecycle design: mark automation-opened tabs as ephemeral by default, preserve explicit/labeled/pinned/user-owned tabs, add a primary-agent task or turn completion cleanup path, and leave the existing idle/excess sweep as a fallback and debugging policy.

## Risks / Open Questions

- Changing the default could close tabs that current primary-agent workflows intentionally keep reusable, so the implementation needs a clear distinction between ephemeral automation tabs and labeled, pinned, user-owned, or explicitly preserved tabs.
- Existing-session/user browser profiles need extra care so OpenClaw does not unexpectedly close a user’s manually opened tab.
