## Summary

Keep open. Current main has tracked browser-tab cleanup for idle/excess sweeps plus explicit lifecycle/session cleanup, but it still does not close primary-agent automation tabs/windows at ordinary task completion. The registry also lacks an ephemeral/keep-open ownership marker needed to implement the requested default safely.

## Best Possible Solution

Keep this issue open for a focused browser lifecycle change in the bundled browser plugin/core lifecycle: add an ephemeral marker for automation-opened tabs, add an explicit keep-open/debug preservation path, preserve labeled/pinned/user-owned tabs, wire best-effort cleanup into ordinary primary-agent task or turn completion, and keep the current idle/excess sweep plus reset/subagent/cron cleanup as fallback. Add tests for primary-agent successful completion, explicit keep-open preservation, and existing-session safety.

- Existing-session profiles attach to a user's running Chromium browser, so completion cleanup must avoid closing manually opened or explicitly preserved user tabs.
