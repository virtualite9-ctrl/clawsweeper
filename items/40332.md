## Summary

Keep this issue open. I rechecked current main after the earlier ClawSweeper keep-open comment, and the requested declarative per-binding/per-agent ACPX permission policy still is not implemented: the config types and strict schemas omit the fields, binding materialization and persisted metadata do not carry them, and ACP session initialization still passes only the existing runtime fields while ACPX permissions come from global plugin config.

## Best Possible Solution

Implement this as a core ACP/runtime contract change: add permissionMode and nonInteractivePermissions to agents.list[].runtime.acp and bindings[].acp schema/types/help/docs, persist them in configured ACP binding metadata, thread them through ACP session initialization or a versioned runtime ensure option, and make ACPX resolve precedence as binding override, then agent runtime default, then global plugins.entries.acpx.config. Add tests for validation acceptance/rejection, precedence, binding materialization/rehydration, configured binding session initialization, and ACPX runtime wiring.

- The best fix changes public config schema/help/docs plus ACP runtime/plugin contracts and permission semantics, so it needs maintainer API and security review with focused regression tests.
