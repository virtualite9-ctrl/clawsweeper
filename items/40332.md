## Summary

Keep this issue open. Current main still lacks per-binding or per-agent ACPX permission scoping: the config types/schemas reject the requested fields, configured ACP bindings do not persist permission policy, and ACP runtime initialization still receives only global ACPX plugin permission settings plus post-session runtime option controls.

## Best Possible Solution

Implement the requested declarative permission scoping as a core ACP/runtime contract change: add permissionMode and nonInteractivePermissions to agents.list[].runtime.acp and bindings[].acp schema/types/help/docs, persist them in configured ACP binding metadata, thread them through ACP session initialization or a versioned runtime ensure option, and make ACPX resolve precedence as binding override, then agent runtime default, then global plugins.entries.acpx.config. Add tests for validation rejection/acceptance, precedence, binding materialization/rehydration, and ACPX runtime wiring.
