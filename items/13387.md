## Summary

Current main still does not implement automatic pruning or collapsing of consecutive assistant NO_REPLY transcript turns before model replay. The linked implementation PR #13388 is closed and unmerged, the latest maintainer/Codex issue comment already kept this open, and current source/docs/tests only cover adjacent behavior such as heartbeat-pair filtering, display suppression, tool-result pruning, and silent-reply delivery handling.

## Best Possible Solution

Keep #13387 open. Implement a focused replay-context pruning pass near sanitizeSessionHistory, validateReplayTurns, or immediately after limitHistoryTurns that collapses consecutive assistant silent replies while preserving isolated legitimate NO_REPLY turns. Reuse shared silent-token semantics where possible and add regression coverage for text-block NO_REPLY, legacy string content, JSON silent-token envelopes, and empty-content silent stop turns. Add config/docs only if maintainers decide this should be configurable.
