## Summary

Current main still does not implement automatic pruning/collapsing of consecutive assistant NO_REPLY transcript turns before model replay. The related implementation PR #13388 was closed unmerged, and the existing code only covers adjacent mitigations such as heartbeat-pair filtering, display/delivery suppression, tool-result pruning, image replay cleanup, and empty/error-turn handling.

## Best Possible Solution

Keep #13387 open. Implement a focused in-memory replay-context pruning pass near sanitizeSessionHistory, validateReplayTurns, or immediately after limitHistoryTurns that collapses consecutive assistant silent replies while preserving isolated legitimate NO_REPLY turns. Add regression coverage for text-block NO_REPLY, legacy string content, JSON/silent-token detection where appropriate, and empty-content silent stop turns; only add config/docs if maintainers decide this behavior should be configurable.
