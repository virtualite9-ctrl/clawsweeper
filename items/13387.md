## Summary

Current main still does not implement the requested replay-context pruning of consecutive assistant NO_REPLY/silent-reply turns. The runner sanitizes, validates, heartbeat-filters, limits, and repairs replay history, but there is no pass that collapses consecutive silent replies before model context is sent. Related fixes cover narrower delivery/UI/heartbeat cases only, and the linked implementation PR #13388 remains closed and unmerged in the provided GitHub context.

## Best Possible Solution

Keep #13387 open. Implement a focused replay-context pruning pass in the embedded runner history pipeline, likely near sanitizeSessionHistory, validateReplayTurns, or immediately after limitHistoryTurns, that collapses consecutive assistant silent replies while preserving isolated legitimate silent turns. Reuse shared silent-token detection such as isSilentReplyPayloadText, cover legacy string content, text blocks, JSON silent envelopes, and empty-content silent stop turns, and add replay-context tests rather than only UI history suppression tests.
