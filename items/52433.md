## Summary

Current main no longer contains the hardcoded ACPX 0.1.16 path described in the report. The bundled ACPX plugin now pins `acpx` from package metadata (`0.5.3` on current main / latest release), and bundled runtime repair derives install specs from that package metadata instead of a separate stale literal.

## Close Comment

Closing as implemented on main after a Codex review. Current OpenClaw no longer hardcodes ACPX `0.1.16`: the bundled ACPX plugin now declares `acpx: 0.5.3` in `extensions/acpx/package.json:7`, and bundled runtime repair derives install specs from extension package metadata in `src/plugins/bundled-runtime-deps.ts:912-935` rather than a separate stale literal. The changelog also records the related fixes: `CHANGELOG.md:1439` notes the expected-version logic was moved to package metadata, and `CHANGELOG.md:1094` records the bundled ACPX pin bump in the 2026.4.8 release line. Current docs also describe the bundled plugin as a pinned/self-repairing install and point local development overrides at an explicit plugin install path instead of manual npm edits in the bundled package (`docs/tools/acp-agents-setup.md:127-141`). If this still reproduces on v2026.4.22 or newer, please open a fresh issue with the exact install path plus `/acp doctor` and plugin status output.
