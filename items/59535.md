## Summary

Keep open. Current `main` still has the reported bug: `parseBooleanValue` lowercases the input string but does not normalize custom `truthy`/`falsy` option entries before storing them in `Set`s, so mixed-case custom lists remain case-sensitive. The current tests also do not cover the reported mixed-case custom-list scenario. The PR discussion is technically sound, including the later review comments that identified follow-up hardening concerns, but nothing in current `main` shows the original bug has been implemented or otherwise resolved.
