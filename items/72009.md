## Summary

Keep #72009 open. Current main still has the hardcoded 2-minute reply typing TTL, does not expose `session.typingTtlSeconds` or agent-default `typingTtlSeconds`, and the docs still only describe typing mode and refresh interval. The PR is a focused, recent implementation with author follow-up addressing the Codex precedence review, so it is not a conservative cleanup close even though older duplicate/superseded typing-TTL threads exist.

## Best Possible Solution

Keep #72009 open as a plausible focused implementation for the still-missing configurable typing TTL. Maintainers should either review and land this PR after the normal changed gates, or explicitly choose a single canonical PR if they prefer the older #42105 thread; current main does not yet solve the user-visible problem.
