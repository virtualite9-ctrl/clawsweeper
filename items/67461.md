## Summary

Current main still lacks an abandoned-stream cleanup path for guarded provider responses. The shared provider transport wrapper only releases the guarded undici dispatcher on no body, EOF/read error, or explicit stream cancel, while Anthropic streaming paths still use this wrapper and there is no regression coverage for an abandoned response body.

## Best Possible Solution

Keep this issue open. Patch `src/agents/provider-transport-fetch.ts` so the managed response reliably invokes `release()` when a streamed response is abandoned before EOF and without explicit `cancel()`, then add focused regression coverage in `src/agents/provider-transport-fetch.test.ts`. During the fix, audit Anthropic Messages and Anthropic Vertex/pi-ai streaming consumers so they either drain/cancel explicitly or are protected by the shared transport safety net.

## Evidence

- **Current wrapper release paths are incomplete:** `buildManagedResponse` locks the source stream reader and calls `release()` only when there is no body, when `reader.read()` reaches EOF, on read error, or when the wrapped body is explicitly cancelled. There is no `FinalizationRegistry`, watchdog, weak cleanup, or other abandoned-stream safety net in the current implementation.
