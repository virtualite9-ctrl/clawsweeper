## Summary

Current main still lacks an abandoned-stream cleanup path for guarded provider responses. `buildManagedResponse` releases the guarded undici dispatcher only on no body, EOF/read error, or explicit stream cancel, while Anthropic streaming paths still use this wrapper and tests do not cover abandonment.

## Best Possible Solution

Keep this issue open. Patch the shared provider transport wrapper in `src/agents/provider-transport-fetch.ts` so `release()` runs when a streamed `Response` is abandoned before EOF and without explicit `cancel()`, and add focused regression coverage in `src/agents/provider-transport-fetch.test.ts`. During the fix, audit Anthropic Messages and Anthropic Vertex/pi-ai streaming consumers so they either drain/cancel explicitly or are protected by the shared safety net.
