## Summary

Keep open. Current main still lacks stale cron session-row recovery in the reaper, and this PR remains a plausible fix direction, but the submitted patch is incomplete because review blockers around retention-disabled configs, base session keys, and coverage are material.

## Best Possible Solution

Keep this PR open for contributor or maintainer follow-up. The best path is a rebased cron lifecycle/session recovery patch that runs independently of retention pruning, repairs the session row shape current main actually persists plus any legacy `:run:` rows, and adds regression coverage for `sessionRetention: false`, stale versus recent running sessions, and defer-before-prune behavior under short retention. The related closed #56121 is useful context, but it does not supersede this PR before a corrected implementation exists.
