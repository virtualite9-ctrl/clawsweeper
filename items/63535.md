## Summary

Keep open. Current main still only prunes expired cron run-session rows; it does not recover stale running cron sessions. The PR is a plausible focused bug fix, but the provided review discussion identifies unresolved correctness issues that should be addressed before merge.

## Best Possible Solution

Keep the PR open for maintainer/contributor follow-up. The best fix is to implement cron orphan recovery on main as an independent maintenance pass, not gated by retention pruning, and to reconcile both base cron session keys and `:run:` aliases with regression tests for `sessionRetention: false`, stale running rows, recent active rows, and the one-sweep deferral behavior when retention would otherwise prune immediately. The related lifecycle bug tracked in #56121 remains useful context.
