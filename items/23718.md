## Summary

Keep open. Current `main` still does not implement fallback-chain syntax for `heartbeat.model`: schema/types/docs describe it as a string-only `provider/model` override, and the heartbeat runtime/downstream reply options only pass a string `heartbeatModelOverride`. The existing related fallback work covers regular agent model fallback chains used by heartbeat runs, but not a heartbeat-specific `{ primary, fallbacks }` config object.

## Best Possible Solution

Implement heartbeat-specific fallback-chain support by changing the heartbeat config contract to accept the shared `AgentModelConfig` shape while preserving the existing string form. Resolve the heartbeat primary with the same helpers used by agent model config, forward heartbeat fallbacks through an explicit reply/run option into `runWithModelFallback` as `fallbacksOverride`, and update schema/types, model-pricing reference collection, heartbeat docs, and focused tests for defaults and per-agent heartbeat overrides.
