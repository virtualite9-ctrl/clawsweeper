## Summary

Keep #23718 open. Current main still does not accept or honor `heartbeat.model: { primary, fallbacks }`: heartbeat config validation, public config types, generated schema, docs, and runtime forwarding remain string-only. Existing global/per-agent model fallback plumbing can protect heartbeat runs in some cases, but it does not implement the requested heartbeat-specific fallback chain. Related PR #40875 is docs-only and does not supersede the runtime feature request.

## Best Possible Solution

Keep the issue open and implement heartbeat-specific model config in core: change heartbeat schema, public types, generated schema/help, and docs to accept `AgentModelConfig`; resolve heartbeat primary/fallbacks with the existing model-input helpers; pass a heartbeat fallback override through reply/run options into `runWithModelFallback`, preserving empty-array semantics; update pricing/auto-enable collection if heartbeat object configs should be discovered; and add focused config plus heartbeat runtime tests. PR #40875 may still be useful as documentation context, but it is docs-only and should not be treated as the canonical implementation.
