## Summary

Keep #23718 open. Current main still treats `heartbeat.model` as a string-only override and does not expose or thread a heartbeat-specific `{ primary, fallbacks }` model config into model fallback execution. The related PR #40875 is docs-only per the provided context and does not supersede the requested runtime behavior.

## Best Possible Solution

Implement heartbeat-specific fallback-chain support in core: change heartbeat schema and exported config types to accept `AgentModelConfig` while preserving string compatibility, resolve heartbeat primary/fallbacks through the existing model helpers, pass a heartbeat fallback override through reply options into `runWithModelFallback`, and update focused config/runtime tests plus heartbeat/model-failover docs. Related PR #40875 can remain a docs clarification but should not close this feature request unless runtime support is implemented or a narrower canonical issue supersedes it.
