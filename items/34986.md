## Summary

Keep #34986 open. Current main still does not implement a configurable heartbeat pre-flight command/script that can return HEARTBEAT_OK and skip agent session/model work. The linked implementation PR #35504 proposed that exact `agents.defaults.heartbeat.preFlightCommand` surface but was closed unmerged, and current code only has built-in heartbeat gates before normal runs call `getReplyFromConfig`.

## Best Possible Solution

Keep the issue open for maintainer product judgment or implementation. A complete core fix would add an optional trusted heartbeat pre-flight hook such as `agents.defaults.heartbeat.preFlightCommand`, merge it through default and per-agent heartbeat config, update exported types, strict schema, generated config metadata/help/docs, and run it in `runHeartbeatOnce` before session/model work. Tests should cover HEARTBEAT_OK skip, non-OK output, command failure, timeout, unset command, default/per-agent merging, and unchanged existing heartbeat behavior.
