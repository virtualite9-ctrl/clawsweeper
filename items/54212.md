## Summary

Keep open. Current main still does not implement the PR’s central behavior: the Control UI Sessions table renders raw session keys in the Key column, only filters by key/label/kind/displayName, and does not pass the already-loaded agent identity map into `renderSessions`. The PR is focused and has useful tests in the provided context, but it needs conflict resolution and should be refreshed against the current session-key grammar before merge.

## Best Possible Solution

Keep the PR open and ask for a refreshed version: pass `agentIdentityById` into the Sessions view, display `identity.emoji + identity.name` while preserving raw-key access, include identity-name search, and adapt tests/implementation to the current `agent:<agentId>:<rest>` session-key grammar.
