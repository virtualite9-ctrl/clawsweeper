## Summary

Keep this PR open. Current main still lacks Slack `allowBots: "mentions"` support, while Discord already has the tri-state behavior this PR is trying to mirror. The PR is stale against the current Slack plugin layout, but it remains a focused implementation candidate rather than conservative cleanup closure.

## Best Possible Solution

Keep this PR open and ask the contributor to port it to the current Slack plugin layout. The implementation should update `extensions/slack/src/monitor/message-handler/prepare.ts`, `extensions/slack/src/monitor/channel-config.ts`, `src/config/types.slack.ts`, `src/config/zod-schema.providers-core.ts`, Slack UI/help/generated metadata, and focused Slack tests. The runtime should resolve `allowBots` into `off/all/mentions`, apply the `mentions` bot gate after explicit/implicit mention state is known, and then run the Slack extension changed lane or `pnpm check:changed`.
