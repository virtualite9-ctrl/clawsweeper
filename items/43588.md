## Summary

Keep this PR open. Current main still lacks Slack `allowBots: "mentions"` support in the actual Slack plugin runtime and config contract, while the PR is a focused candidate for that missing behavior. It needs rebasing/adapting from the old `src/slack` paths to the current `extensions/slack` layout rather than conservative cleanup closure.

## Best Possible Solution

Keep the PR open and ask for an update against current main: port the change to `extensions/slack/src/monitor/message-handler/prepare.ts`, `extensions/slack/src/monitor/channel-config.ts`, `src/config/types.slack.ts`, `src/config/zod-schema.providers-core.ts`, Slack UI/help metadata, and generated bundled config metadata; retain the focused tests for bot message without mention, explicit mention, DM, and implicit thread participation with cache cleanup.
