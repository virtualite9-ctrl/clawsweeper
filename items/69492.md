## Summary

Current main has tactical filters for heartbeat acknowledgments, display projection, and internal-runtime-context stripping, but it does not implement the central request: a per-system-event audience/classification primitive or a consumer-path filter for system-event-shaped artifacts. The provided discussion also shows active follow-up PRs/issues for slices of this problem, including #72201 as one awareness-event slice rather than a full delivery-routing fix.

## Best Possible Solution

Keep #69492 open as the canonical backend design/tracking issue. Land or revise the narrow awareness-event PR independently, then decide whether OpenClaw wants a broader SystemEvent audience contract, heartbeat relay semantics, and consumer-path filtering for chat.history, sessions.get, HTTP history/SSE, and channel dispatchers.
