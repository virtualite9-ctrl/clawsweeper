## Summary

The reported duplicate startup catch-up does not reproduce on current main. The issue's timestamp comparison assumes the 20:00 Europe/Berlin slot on the same UTC day is already previous at 04:21 UTC, but that slot is still in the future; the actual previous slot is before the late catch-up run, so current main's missed-slot check returns false.
