## Summary

Issue #63534 reports the 2026.4.9 Dreams page layout being covered/overlapped. Current main has the Dreaming UI split into separate Scene, Diary, and Advanced surfaces, with snippet-heavy review content moved out of the default scene and diary content constrained/scrollable. The same symptom was also addressed by merged PRs #63875 and #64035 and recorded in the 2026.4.10 changelog.

## Best Possible Solution

Close this as implemented on current main. Keep the shipped Scene/Diary/Advanced separation and diary containment; if someone can reproduce a new Dreams layout overlap on a current release, it should be filed as a fresh issue with an updated screenshot, version, browser/OS, and whether the Scene, Diary, or Advanced tab is affected.
