## Summary

Keep #8299 open. Current main still does not expose a documented config option or public `sessions_spawn` parameter that deterministically suppresses sub-agent completion announcements. The implementation still relies on token/model-driven suppression for normal announces, while the internal `expectsCompletionMessage` path is not a public schema/docs contract and is not equivalent to guaranteed skip behavior.

## Best Possible Solution

Keep #8299 open until OpenClaw adds a documented framework-level sub-agent announce suppression control, either as `agents.defaults.subagents` config and/or a public `sessions_spawn` parameter. A complete fix should update the TypeScript config types, zod/generated schema/help, public tool schema/parser, registry and announce delivery contracts, docs at `docs/tools/subagents.md`, and regression tests proving deterministic skip behavior without relying on `ANNOUNCE_SKIP` or `NO_REPLY`.
