## Summary

Keep #8299 open. Current main still lacks a documented config option or public `sessions_spawn` parameter that deterministically suppresses top-level sub-agent completion announcements. The remaining suppression path is token-driven (`ANNOUNCE_SKIP` / `NO_REPLY`) or internal lifecycle bookkeeping, and the related implementation PRs were closed unmerged.

## Best Possible Solution

Keep #8299 open until OpenClaw adds a documented framework-level sub-agent announce suppression control, either as `agents.defaults.subagents` config and/or a public `sessions_spawn` parameter. A complete fix should update the tool schema/parser, config types/zod/generated schema/help, registry and announce delivery contracts, cleanup bookkeeping, docs, and regression tests for deterministic skip behavior.
