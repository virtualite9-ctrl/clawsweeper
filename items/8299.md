## Summary

Keep #8299 open. Current main still lacks a documented config option or public `sessions_spawn` parameter that deterministically suppresses sub-agent completion announcements. The implementation still relies on exact `ANNOUNCE_SKIP` / `NO_REPLY` output for suppression, while top-level completion delivery still routes through the announcement delivery path. The adjacent parent/orchestrator routing work and open #27445 do not supersede this narrower skip/suppress request.

## Best Possible Solution

Keep #8299 open until OpenClaw adds a documented framework-level sub-agent announce suppression control, either under `agents.defaults.subagents` and/or as a public `sessions_spawn` parameter. A complete fix should update TypeScript config types, zod/generated schema/help, the public tool schema/parser, registry and delivery contracts, docs in `docs/tools/subagents.md`, and regression tests proving deterministic skip behavior without relying on `ANNOUNCE_SKIP` or `NO_REPLY`. Coordinate naming with announce routing work, but do not treat that broader routing issue as superseding this suppression request.
