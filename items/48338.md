## Summary

Current main has partial low-context safeguards, but it does not implement the requested custom-provider setup UX. Generic custom OpenAI/Anthropic-compatible providers still create new model entries at the 16,000-token hard minimum with 4,096 max tokens, and the interactive configure flow verifies the endpoint without showing or prompting for contextWindow/maxTokens. The open PR #43493 directly targets this gap, but it is not merged/shipped, so the issue should remain open as the canonical problem report.

## Best Possible Solution

Keep #48338 open until #43493 or an equivalent core change lands. The best fix belongs in the custom provider configure/onboard flow: surface the effective context window, let users set contextWindow/maxTokens during setup, warn when a conservative fallback is being used, document the workflow, and keep the runtime low-context guard as a backstop.
