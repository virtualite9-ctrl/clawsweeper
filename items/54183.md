## Summary

Keep open. Current `main` does not contain the WhatsApp outbound retry feature from PR #54183: the shipped outbound paths still call WhatsApp send functions directly, WhatsApp config/schema do not expose a `retry` setting, and the current test suite explicitly asserts transient outbound send failures are not retried.

## Evidence

- **Production outbound path has no retry wrapper:** `whatsappChannelOutbound` passes `sendMessageWhatsApp` and `sendPollWhatsApp` straight into `createWhatsAppOutboundBase`; there is no `withWhatsAppSendRetry`/wrap hook or retry config resolution in the shipped path.
  - file: [extensions/whatsapp/src/channel-outbound.ts:16](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/extensions/whatsapp/src/channel-outbound.ts#L16)
