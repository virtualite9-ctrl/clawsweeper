## Summary

Keep open. Current `main` has newer version-based skills-snapshot refresh logic, but it does not implement the draft PR's cold-start/content-diff fallback when both the stored snapshot version and the in-memory watcher version are `0`. That means the reported restart case remains plausible on `main` for sessions whose cached snapshot was created before any watcher bump.

## Evidence

- **Agent command only refreshes by version/filter:** `src/agents/agent-command.ts:574-614` rebuilds the session skills snapshot only when the snapshot is missing, `shouldRefreshSnapshotForVersion(...)` says it is stale, or the skill filter changed. There is no content comparison against a rebuilt snapshot on restart.
  - file: [src/agents/agent-command.ts:574](https://github.com/openclaw/openclaw/blob/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4/src/agents/agent-command.ts#L574)

  - sha: [f3c37a946cc4](https://github.com/openclaw/openclaw/commit/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4)
- **Auto-reply path has the same gap:** `src/auto-reply/reply/session-updates.ts:145-205` computes `shouldRefreshSnapshot` from `shouldRefreshSnapshotForVersion(...)` and `matchesSkillFilter(...)` only, then persists or reuses the snapshot from that decision. It also lacks any cold-start rebuild-and-compare path.
  - file: [src/auto-reply/reply/session-updates.ts:145](https://github.com/openclaw/openclaw/blob/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4/src/auto-reply/reply/session-updates.ts#L145)

- **Current restart helper only covers cachedVersion>0 -> nextVersion=0:** `src/agents/skills/refresh-state.ts:65-72` returns `next === 0 ? cached > 0 : cached < next`. So a restart refresh happens when an old nonzero snapshot version falls back to in-memory `0`, but not when both cached and current versions are `0`.
  - file: [src/agents/skills/refresh-state.ts:65](https://github.com/openclaw/openclaw/blob/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4/src/agents/skills/refresh-state.ts#L65)
