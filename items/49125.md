## Summary

Keep open. Current `main` still misses the cold-restart case this draft PR targets: when the in-memory skills watcher/version state resets to `0`, existing session snapshots with version `0` are reused unless the watcher bumps again or the skill filter changes, so newly added/changed skills after a restart can stay stale.

## Evidence

- **Version gate does not refresh on cold restart when both versions are 0:** `shouldRefreshSnapshotForVersion` returns `false` for cached `0` and next `0` (`next === 0 ? cached > 0 : cached < next`). That means a restarted process with reset watcher state will not rebuild an unchanged-version snapshot.
  - file: [src/agents/skills/refresh-state.ts:65](https://github.com/openclaw/openclaw/blob/89755d1c79678610720cdb9ce504cfd62c261f64/src/agents/skills/refresh-state.ts#L65)
  - command: `nl -ba src/agents/skills/refresh-state.ts | sed -n '57,75p'`
  - sha: [89755d1c7967](https://github.com/openclaw/openclaw/commit/89755d1c79678610720cdb9ce504cfd62c261f64)
- **CLI agent path only rebuilds on new session, version bump, or filter mismatch:** `agent-command` computes `shouldRefreshSkillsSnapshot` from existing snapshot presence, `shouldRefreshSnapshotForVersion(...)`, and `matchesSkillFilter(...)`. There is no content-diff fallback for the `0 -> 0` restart case before deciding to reuse `currentSkillsSnapshot`.
  - file: [src/agents/agent-command.ts:578](https://github.com/openclaw/openclaw/blob/89755d1c79678610720cdb9ce504cfd62c261f64/src/agents/agent-command.ts#L578)
  - command: `nl -ba src/agents/agent-command.ts | sed -n '578,618p'`
  - sha: [89755d1c7967](https://github.com/openclaw/openclaw/commit/89755d1c79678610720cdb9ce504cfd62c261f64)
- **Auto-reply/session path has the same gap:** `ensureSkillSnapshot` uses the same version/filter gate and reuses `current.skillsSnapshot` when `shouldRefreshSnapshot` is false. A cold restart with `snapshotVersion === 0` therefore also skips rebuilding here.
  - file: [src/auto-reply/reply/session-updates.ts:145](https://github.com/openclaw/openclaw/blob/89755d1c79678610720cdb9ce504cfd62c261f64/src/auto-reply/reply/session-updates.ts#L145)
  - command: `nl -ba src/auto-reply/reply/session-updates.ts | sed -n '145,205p'`
  - sha: [89755d1c7967](https://github.com/openclaw/openclaw/commit/89755d1c79678610720cdb9ce504cfd62c261f64)
- **Snapshots persist the watcher version directly, including 0:** `buildWorkspaceSkillSnapshot` stores `version: opts?.snapshotVersion`. When the watcher state has not been bumped after restart, rebuilt snapshots also carry `0`, which is exactly the case the current refresh gate cannot distinguish from a stale stored snapshot.
  - file: [src/agents/skills/workspace.ts:719](https://github.com/openclaw/openclaw/blob/89755d1c79678610720cdb9ce504cfd62c261f64/src/agents/skills/workspace.ts#L719)
