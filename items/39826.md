## Summary

Keep PR #39826 open. Current `main` still emits the Discord no-mention guild-drop log with only `channelId` and `reason`, while `accountId` and `guildId` are available in the same preflight scope. The PR patch is stale against the old `src/discord/...` path, but the underlying log-only diagnostic improvement remains valid for the current `extensions/discord/...` implementation.

## Best Possible Solution

Keep PR #39826 open or apply the same log-only change on current main in `extensions/discord/src/monitor/message-handler.preflight.ts`: add `accountId: params.accountId` and `guildId: params.data.guild_id` to the no-mention `logger.info` object, ideally with a focused assertion for that structured log payload.
