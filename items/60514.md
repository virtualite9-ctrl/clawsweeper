## Summary

Keep open. Current `main` now covers most of the request: plugin `session_end` exists and is wired into session lifecycle paths, `memory-core` provides automatic dreaming/consolidation, and OpenClaw has built-in cache-TTL session pruning. But I did not find evidence that forked sub-agents reuse a parent prompt cache; current sub-agent docs still describe child runs as having their own context/token usage, with `context: "fork"` only copying transcript context into a separate child session. That makes this a partial implementation, not a full close.
