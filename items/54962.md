## Summary

Keep open. Current `main` at `c070509b7f8fdbe479e5d829643782c272e30e83` does not implement the proposed `callback_data:` prefix for generic Telegram callbacks, and the PR's blanket prefix would likely regress existing raw slash-command callback flows that `main` still relies on.

## Risks / Open Questions

- The underlying Telegram docs/code mismatch still appears real on current `main`.
- Merging this PR as written would likely regress existing raw slash-command callback flows such as `/models add ...` unless those payloads are migrated to the native `tgcmd:` path or otherwise exempted.
- The existing callback regression test is too loose to protect the proposed prefix behavior.
