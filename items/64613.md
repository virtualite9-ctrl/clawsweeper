## Summary

Keep open. Current main still partially addresses this class of leak, but it does not implement the reported `## Memory system — ACTION REQUIRED` / auto-recall `memory_search` prompt filtering at the shared visible-history boundary. Heartbeat prompts and active-memory plugin tag blocks have specific handling, but a plain memory system instruction persisted as transcript text would still pass through `chat.history` and session-history SSE projection.

## Best Possible Solution

Keep this issue open. The best maintainer path is to make memory/system-injection content structurally internal before it reaches visible transcript history, or extend the shared `projectChatDisplayMessages` path with narrowly scoped handling for known internal memory action blocks. Add regression coverage for string and text-block content, the reported memory action block, heartbeat prompts with workspace-file suffix, legitimate user text that merely quotes those phrases, `chat.history`, and session-history SSE.
