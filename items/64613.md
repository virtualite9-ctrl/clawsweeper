## Summary

Keep open. Current main has shared chat-history projection and now hides several adjacent internal artifacts, including heartbeat prompts and runtime-context envelopes, but it still does not hide the reported plain-text `## Memory system — ACTION REQUIRED` / auto-recall `memory_search` instruction block when that text is already present as transcript content.

## Best Possible Solution

Keep this issue open. The best maintainer path is to make memory and heartbeat/system-injection content structurally internal before it enters visible history, or extend `projectChatDisplayMessages` and session-history SSE projection with narrowly scoped handling for known internal memory action blocks. Add regression coverage for string and text-block content, the reported memory action block, heartbeat prompts with workspace-file suffix, legitimate user text that merely mentions those phrases, `chat.history`, and session-history SSE.

## Risks / Open Questions

- A literal phrase filter for `## Memory system` or `ACTION REQUIRED` could hide legitimate user-authored text; structural internal provenance or display-hidden metadata would be safer.
- External memory plugins may use different wording, so exact matching would remain incomplete unless paired with a documented plugin/API path for hidden prompt context.
- Current coverage proves heartbeat/runtime-envelope hiding but not the reported plain memory action block across both `chat.history` and session-history SSE.
