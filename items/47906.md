## Summary

Current `main` still runs each session through a single `session:<key>` lane plus one global lane, so the requested conversational/background/heartbeat lane split is not implemented. Main does mitigate the symptom with queue modes that steer or enqueue followups while a run is active, but heartbeats still skip when the session lane is busy, which is materially short of the issue’s proposed isolation model.

## Evidence

- **Single serialized session lane remains the core model:** `runEmbeddedPiAgent` resolves one session lane from the session key and wraps the full run in `enqueueSession(...enqueueGlobal(...))`, preserving one active run per session rather than separate conversational/background lanes.
