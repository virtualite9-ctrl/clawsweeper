## Summary

Current `main` already preserves and displays tool-call request parameters in the Control UI. The tool-card pipeline serializes tool-call args into `inputText`, renders a `Tool input` section in the expanded tool-call view, and includes the same payload in sidebar content. The same implementation is present in release `v2026.4.22`, so the issue is already fixed on shipped builds newer than the reporter's `2026.3.13-1` custom build.
