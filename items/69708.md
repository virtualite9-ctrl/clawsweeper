## Summary

Keep open. Current main still rejects `type: "aws-sdk"` entries while loading `auth-profiles.json`: the auth-profile credential union and parser only accept `api_key`, `oauth`, and `token`, and unknown types are logged/dropped as `invalid_type`. The linked PR #69753 proposed the relevant narrow fix, but it was closed unmerged and its `sdk_marker_skipped`/`AUTH_PROFILE_SDK_MARKER_TYPES` behavior is not present on current main.

## Best Possible Solution

Keep the issue open and either revive/reimplement the narrow PR #69753 approach or add an equivalent fix: treat `auth-profiles.json` `type: "aws-sdk"` records as SDK-managed Bedrock markers that do not count as invalid and do not require API-key material, then add regression coverage for a mixed store containing an `aws-sdk` marker plus genuine invalid entries. Also clarify in Bedrock/auth-profile docs whether users should omit the marker entirely or whether OpenClaw intentionally accepts it for compatibility.
