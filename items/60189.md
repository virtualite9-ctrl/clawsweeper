## Summary

Current `main` no longer bootstraps Codex CLI credentials into `auth-profiles.json` during runtime auth-store loads. External auth is now applied as a runtime-only overlay, persisted writes are filtered, and tests explicitly cover read-only loads without mutating `auth-profiles.json`, which addresses the EPERM-before-tool-exec failure described in #60189.

## Evidence

- **Runtime loads overlay external auth instead of writing it:** `loadAuthProfileStoreForRuntime` returns `overlayExternalAuthProfiles(...)` over the loaded store; the write path in `loadAuthProfileStoreForAgent` is limited to legacy migration / oauth-file merge, not external CLI sync.
  - file: [src/agents/auth-profiles/store.ts:229](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/src/agents/auth-profiles/store.ts#L229)
