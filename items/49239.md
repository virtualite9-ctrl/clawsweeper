## Summary

Current `main` already resolves the requested UX cleanup in the Dashboard Sessions view: sessions open directly from the session key link, there is no redundant per-row `Open in Chat` action/menu in the current table, and deletion is no longer an immediate single menu click because it requires row selection plus a confirmation dialog before `sessions.delete` runs.

## Evidence

- **Session rows already open directly in Chat:** The session key cell renders as a direct link to the chat tab and intercepts normal left-clicks to call `onNavigateToChat(row.key)`.
  - file: [ui/src/ui/views/sessions.ts:490](https://github.com/openclaw/openclaw/blob/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4/ui/src/ui/views/sessions.ts#L490)
  - sha: [f3c37a946cc4](https://github.com/openclaw/openclaw/commit/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4)
- **No per-row overflow/menu action surface in current Sessions table:** The table columns are checkbox, key, label, kind, updated, tokens, compaction, thinking, fast, verbose, and reasoning. Bulk delete is exposed only in the selected-items toolbar; there is no per-row actions/menu column or `Open in Chat` menu entry in this view.
  - file: [ui/src/ui/views/sessions.ts:338](https://github.com/openclaw/openclaw/blob/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4/ui/src/ui/views/sessions.ts#L338)
