## Summary

Keep #36630 open. Current main still lacks the PR's central bidirectional Signal quote-reply behavior: outbound Signal sends do not emit signal-cli quote metadata, the Signal outbound plugin does not forward quote-author metadata, and inbound Signal quote context still omits ReplyToId and the richer quote payload model proposed by the PR.

## Best Possible Solution

Keep #36630 open. The best maintainer path is to land a reviewed, hardened version of this Signal quote-reply work or reimplement it narrowly: strict decimal/safe Signal quote timestamp and id validation, quote-author resolution for group replies, quote-aware Signal outbound delivery through the plugin boundary, inbound ReplyToId plus richer signal-cli quote metadata, and focused regression coverage for delivery consumption, debounce, queued/recovered delivery, and plugin-boundary cases.
