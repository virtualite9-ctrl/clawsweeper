## Summary

Keep #36630 open. Current main still does not implement the PR's central bidirectional Signal quote-reply behavior: outbound Signal sends never emit native signal-cli quote metadata, Signal outbound adapters do not receive or forward quote-author metadata, and inbound Signal quote context still lacks ReplyToId and the richer quote payload model proposed by the PR.

## Best Possible Solution

Keep #36630 open. The best maintainer path is to land a reviewed, hardened version of the Signal quote-reply work or reimplement it narrowly: strict decimal/safe Signal quote timestamp and id validation, quote-author resolution for group replies, quote-aware Signal outbound delivery through the plugin boundary, inbound ReplyToId plus richer signal-cli quote metadata, and focused regression coverage for delivery consumption, debounce, queued/recovered delivery, and plugin-boundary cases.
