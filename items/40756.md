## Summary

Keep this PR open. Current main still mutates any resolved heartbeat prompt that mentions `HEARTBEAT.md`, including user-configured prompts, and the linked regression #40255 is still open. The PR remains a focused implementation candidate, though maintainers should review the prompt-safety tradeoff before landing.

## Best Possible Solution

Keep this PR or #40255 active. The best path is a small heartbeat-runner fix that distinguishes non-empty user-configured heartbeat prompts from built-in/generated heartbeat prompts, preserves the workspace-path hint and idempotency for default/internal prompts, adds a regression test for custom prompts mentioning `HEARTBEAT.md`, and resolves the prompt-safety tradeoff before merge.

## Risks / Open Questions

- The PR intentionally stops appending the workspace-file disambiguation hint to user-configured heartbeat prompts. That matches the documented verbatim-prompt contract, but maintainers should decide whether heartbeat file selection needs a separate tool-layer or non-overridable guard.
- The branch is old and should be rebased or refreshed before landing; surrounding heartbeat-runner code has drifted since the PR base.
- The PR exports `appendHeartbeatWorkspacePathHint` for direct unit testing. That is probably acceptable for an internal runner module, but maintainers may prefer an integration-style test through `runHeartbeatOnce()` to avoid widening the module surface.
