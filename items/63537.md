## Summary

Do not close this PR. Current main does not implement the requested behavior: final chat events with no message still trigger a history reload, and the current unit test explicitly asserts that behavior. The PR is also not merge-ready as written because review feedback correctly notes that `!payload.message` would suppress malformed falsy payloads; it needs a narrower fix plus updated regression coverage.

## Best Possible Solution

Keep the PR open but require a rebase and a narrower implementation. The best path is a focused regression test for `/new` and `/reset` final events with `message === undefined`, then a gateway/reload predicate change that skips only the normal local reset-command final path while preserving malformed-payload and tool/history fallback reloads.
