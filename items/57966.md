## Summary

Keep #57966 open. Current main does not contain the PR’s separator-normalized alias lookup or alias-hint error behavior, and the Control UI slash-command path still sends typed model text through the backend resolver. The PR also has safety-sensitive review discussion around fuzzy alias collisions and sanitization, so it needs maintainer review or a narrower updated fix rather than cleanup closure.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is a narrow current-main patch in `src/agents/model-selection-shared.ts` that fixes the linked normalized alias/user-input problem while preserving exact model IDs before fuzzy alias fallback, treating normalized collisions as ambiguous, sanitizing all alias text in logs/errors, and adding focused regression tests in `src/agents/model-selection.test.ts` for space/underscore aliases, provider inference, collision behavior, and error formatting.

## Risks / Open Questions

- The PR’s broad fuzzy alias normalization can change routing when aliases and real model IDs differ only by spaces, underscores, or hyphens; existing review comments already flagged several collision and exact-match preservation cases.
- The alias-hint UX change exposes configured alias text in errors, so any implementation should sanitize or escape aliases before returning them to UI/chat/terminal surfaces.
