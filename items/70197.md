## Summary

Keep #70197 open. Current main still ships the default-enabled Bonjour gateway discovery plugin, still resolves `@homebridge/ciao` to 1.3.6, and has no local ciao patch. The published ciao 1.3.6 source still runs the reported Windows `arp -a | findstr` probe every 15 seconds without a `windowsHide` option, so the Windows console-flash bug is not fixed on main. Related #71188 was already closed as a duplicate of this issue, and open PR #71633 appears to track the narrow dependency-patch direction.

## Best Possible Solution

Keep #70197 open as the canonical active bug. Fix the bundled Bonjour/ciao path by consuming or upstreaming a `@homebridge/ciao` change that passes `{ windowsHide: true }` for the Windows ARP probe; if that cannot land quickly, consider a narrow pnpm patch for `@homebridge/ciao@1.3.6` such as the direction in #71633. Keep `OPENCLAW_DISABLE_BONJOUR=1`, `openclaw plugins disable bonjour`, and `discovery.mdns.mode="off"` as documented workarounds, not the final fix. Verify with a Windows gateway run and process-birth proof that default Bonjour no longer spawns a visible `cmd.exe`/`conhost.exe` window.
