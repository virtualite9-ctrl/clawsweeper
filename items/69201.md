## Summary

Keep #69201 open. Current main still lacks the PR's parent-session deliveryContext backfill in the native and ACP `sessions_spawn` paths: both derive requester origin from the tool context fields only, and the gateway can only seed child delivery from the request fields it receives. Existing gateway tests even cover the channel-only outcome when `to`/`threadId` are missing, which is the failure mode this PR targets. Related Slack-thread PRs address adjacent downstream Slack routing, not this core spawn-origin propagation gap.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land or adapt the parent deliveryContext backfill in `src/agents/subagent-spawn.ts` and `src/agents/acp-spawn.ts`, keep ctx values authoritative, preserve the channel/to/root-level stale-thread guards from the PR discussion, and add the parent-context regression tests proving child sessions persist `deliveryContext.to` and `deliveryContext.threadId` after spawn.
