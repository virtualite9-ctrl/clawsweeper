## Summary

Keep #69201 open. Current main still does not implement the PR’s parent-session deliveryContext backfill for `sessions_spawn` children: native and ACP spawn paths derive requester origin from the tool context fields only, while the gateway can only seed child delivery from the request fields it receives. If those fields are empty, main still has the channel-only child deliveryContext failure mode described by the PR. Related Slack-thread PRs address adjacent downstream Slack routing, not this core spawn-origin gap.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land or adapt the parent deliveryContext backfill in both `src/agents/subagent-spawn.ts` and `src/agents/acp-spawn.ts`, preserve the root-level/stale-thread guards, and add the parent-context regression tests so child sessions are proven to persist `deliveryContext.to` / `threadId` after spawn.
