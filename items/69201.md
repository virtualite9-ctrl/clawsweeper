## Summary

Current main does not appear to implement PR #69201's central fix. The gateway can seed a child session from request delivery hints, but `spawnSubagentDirect` and ACP requester-state resolution still derive those hints directly from the current tool context. When `ctx.agentTo` / `ctx.agentThreadId` are empty while the parent session has a stored `deliveryContext`, current main still sends only the partial request context to the child. The PR remains a focused, active bug-fix candidate rather than cleanup-close material.

## Best Possible Solution

Keep PR #69201 open for maintainer review. The best code direction is to add parent-session delivery-context backfill in `spawnSubagentDirect` and `resolveAcpSpawnRequesterState`, preserve ctx-supplied values as authoritative, keep gateway `agent` seeding as the single write path, and retain explicit guards/tests for channel mismatch, `to` mismatch, thread-only parent state, and explicit root-level spawns.
