## Summary

Keep this issue open. Current main still classifies status-prefixed 403 HTML provider failures as auth HTML, not a Cloudflare challenge/TLS-fingerprint failure, and the provided GitHub context shows open PR #67717 with closing syntax for this issue. That PR is the implementation candidate and should be reviewed, merged, or closed before this issue is closed.

## Best Possible Solution

Keep this issue open and review PR #67717 as the narrow implementation candidate. If it is correct, land the challenge-specific classifier/message and regression tests, then close this issue as implemented. Treat browser-like TLS emulation or a generic request.tlsProfile as separate maintainer-scoped design/security work; if not pursued, document the reverse-proxy workaround and allowPrivateNetwork requirement for OpenAI Codex users behind affected proxies.
