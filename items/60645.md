## Summary

Current `main` already has an inbound dedupe guard for same-agent duplicate deliveries across `main` and channel-scoped session keys, skips duplicate/inflight dispatches before a second turn starts, and includes a regression test that matches the reported Telegram double-reply shape. This appears fixed on `main`, but I could not verify a shipped release beyond the provided latest release `v2026.4.23`.

## Evidence

- **Agent-scoped inbound dedupe for main/direct routing bugs:** `resolveInboundDedupeSessionScope()` intentionally collapses session scope to `agent:<agentId>` and documents the exact case where the same physical inbound message is presented under both `main` and a direct key.

  - sha: [afc0c32bd0eb](https://github.com/openclaw/openclaw/commit/afc0c32bd0eb9f984fcf5bcf26c9ea821834b74d)
- **Second lane is skipped before dispatch:** `dispatchReplyFromConfig()` now claims an inbound dedupe key and immediately returns when the same inbound is already `duplicate` or `inflight`, preventing a second parallel turn from running.
  - file: [src/auto-reply/reply/dispatch-from-config.ts:291](https://github.com/openclaw/openclaw/blob/afc0c32bd0eb9f984fcf5bcf26c9ea821834b74d/src/auto-reply/reply/dispatch-from-config.ts#L291)
