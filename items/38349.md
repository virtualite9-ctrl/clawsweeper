## Summary

Keep #38349 open. Current main still only supports binary group activation (`mention` or `always`) plus `requireMention` gating; it does not implement `contextualActivation`, a peeking/engaged decision-model state machine, contextual decision logging, or a dedicated decision-log CLI. The related implementation PR #38350 is closed unmerged in the provided GitHub context, and the older #1721 is closed rather than an active canonical replacement.

## Best Possible Solution

Keep #38349 open as the canonical product/API issue. The maintainer path is either to implement first-class contextual activation across config schemas, session persistence, Gateway protocol, WhatsApp and Telegram pre-dispatch gating, observability/CLI/docs/tests, or first add a typed plugin/channel activation seam that lets a plugin make the peeking/engaged decision before bundled channel runtimes drop unmentioned group messages.
