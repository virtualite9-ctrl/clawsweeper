## Summary

Keep this issue open. Current main fixes the reported timeout-budget symptom for isolated cron agent turns, but the lower-level command-lane queue still has no abort-aware queued-entry removal, and open PR #43243 remains an unresolved implementation candidate with closing syntax for this issue.

## Best Possible Solution

Review PR #43243 against current main. If maintainers still want the generic queue-lifecycle guarantee, land or rework an abort-aware `enqueueCommandInLane` change with listener cleanup, signal threading from `runEmbeddedPiAgent`, and regression coverage. If the current timeout deferral is considered sufficient, close PR #43243 as superseded first, then close this issue as implemented.

## Evidence
