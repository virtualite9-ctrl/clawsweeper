## Summary

Keep open. Current main still performs a live `doctor.memory.status` embedding probe through the normal batch timeout path, and the provided GitHub context shows open PR #72443 is an unmerged implementation candidate for this exact issue. Per policy, an issue with an open PR using closing syntax should remain open until that PR is reviewed and merged or rejected.

## Best Possible Solution

Keep this issue open and review PR #72443 as the current implementation candidate. The likely fix belongs in the memory-core probe path or the doctor/status control-plane path: cache embedding readiness with a short TTL and regression tests, or move long live probing behind an explicit deep check while default status returns last-known readiness.
