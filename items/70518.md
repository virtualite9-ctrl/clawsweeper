## Summary

Keep #70518 open. Current main still lacks a heartbeat-specific skill allowlist: the heartbeat config schema/type/docs/inventory omit `allowSkills`, and `runHeartbeatOnce` does not pass a heartbeat `skillFilter` into `getReplyFromConfig`. The existing reply and skill-loading layers already support the needed restrictive `skillFilter` behavior, so the PR addresses a real current gap rather than obsolete work.

## Best Possible Solution

Keep the PR open and review it as a focused config/runtime fix. The best path is to add `heartbeat.allowSkills` to the strict schema, exported config types, generated schema/help/labels, public docs, and config-honor inventory; thread the resolved heartbeat allowlist into `getReplyFromConfig` as `skillFilter`; and keep regression coverage for non-empty, empty, and unset behavior.
