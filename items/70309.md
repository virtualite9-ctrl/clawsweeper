## Summary

Keep this issue open. Current main stores MCP server definitions, hot-recreates session MCP runtimes, and can show effective tools for an active session, but it still does not provide the requested Control UI MCP connectivity panel with per-server status, tool names, manual reconnect/refresh, and surfaced error details.

## Best Possible Solution

Keep the issue open. The best path is an additive Gateway operator diagnostics API backed by the MCP runtime/config loader that reports each configured server's name, transport summary, connected/error state, listed tools, redacted last error, and a scoped refresh/reconnect action, then render it under Control UI Infrastructure > MCP and document the behavior.
