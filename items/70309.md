## Summary

Keep #70309 open. Current main has MCP config management and runtime-side MCP cataloging, but it does not provide the requested Control UI connectivity panel, per-server connected/error state, manual reconnect/refresh, or exposed tool/error details. The provided GitHub context has no comments or canonical duplicate, and the title-search related items cover different status/config surfaces rather than MCP connectivity.

## Best Possible Solution

Keep the issue open for maintainer product/API design. A good implementation would add a redacted Gateway MCP status/probe API backed by the existing MCP transport/runtime code, then add an Infrastructure > MCP Control UI panel showing configured servers, live connected/error state, tool count/names, last error, and a scoped refresh/reconnect action, with protocol, gateway, and UI tests.

## Risks / Open Questions

- Implementing this likely needs an additive Gateway protocol/API surface for MCP probe/status/reconnect before the Control UI can show reliable live state.
- Live probing must preserve existing secret redaction for URLs and headers and must avoid leaving stdio MCP child processes or cached runtimes alive unexpectedly.
