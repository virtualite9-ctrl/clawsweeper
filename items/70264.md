## Summary

Close as superseded by open PR #56442, which is the canonical implementation path for the same remaining ACP parent-completion wake/notify work. Current main still shows the reported gap, so this is not implemented-on-main.

## Best Possible Solution

Use #56442, or an equivalent current-main rework, as the canonical implementation path: keep `streamTo:"parent"` as lightweight progress relay, add opt-in ACP terminal completion notify through the subagent-style announce path, and include regression coverage for heartbeat-enabled and heartbeat-disabled ACP parent completion.
