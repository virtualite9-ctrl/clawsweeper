## Summary

Keep open. Current main has partial WebChat/Control UI attachment support, but it does not fully implement the issue’s broader request for general file upload/download in the web chat page. Uploads are still image-only, while downloads/preview links for assistant-produced attachments do exist.

## Evidence

- **Composer upload UI is image-only:** The web chat composer exposes a file input with `accept=${CHAT_ATTACHMENT_ACCEPT}`, and `CHAT_ATTACHMENT_ACCEPT` is `image/*`. The same view filters selected and dropped files through `isSupportedChatAttachmentMimeType`, which only returns true for `image/` MIME types.
  - file: [ui/src/ui/views/chat.ts:1205](https://github.com/openclaw/openclaw/blob/d38ed0831d7372c8505dc944aea69b5b83da1435/ui/src/ui/views/chat.ts#L1205)
  - command: `nl -ba ui/src/ui/views/chat.ts | sed -n '1205,1254p'`
  - sha: [d38ed0831d73](https://github.com/openclaw/openclaw/commit/d38ed0831d7372c8505dc944aea69b5b83da1435)
- **Attachment helper hard-codes image MIME support:** `CHAT_ATTACHMENT_ACCEPT` is `image/*`, and the helper rejects anything not starting with `image/`. That excludes the issue’s requested archives/documents uploads.
  - file: [ui/src/ui/chat/attachment-support.ts:1](https://github.com/openclaw/openclaw/blob/d38ed0831d7372c8505dc944aea69b5b83da1435/ui/src/ui/chat/attachment-support.ts#L1)
  - command: `nl -ba ui/src/ui/chat/attachment-support.ts | sed -n '1,20p'`
  - sha: [d38ed0831d73](https://github.com/openclaw/openclaw/commit/d38ed0831d7372c8505dc944aea69b5b83da1435)
- **Outgoing chat.send attachments are serialized only as images:** The Control UI controller maps every attachment to `{ type: "image", mimeType, content }` before sending `chat.send`. There is no generic document/archive upload path here.
  - file: [ui/src/ui/controllers/chat.ts:213](https://github.com/openclaw/openclaw/blob/d38ed0831d7372c8505dc944aea69b5b83da1435/ui/src/ui/controllers/chat.ts#L213)
  - command: `nl -ba ui/src/ui/controllers/chat.ts | sed -n '213,242p'`
  - sha: [d38ed0831d73](https://github.com/openclaw/openclaw/commit/d38ed0831d7372c8505dc944aea69b5b83da1435)
- **Gateway chat attachment validation still rejects non-image uploads on the image-only path:** `normalizeAttachment(... requireImageMime: true)` throws `only image/* supported` when MIME is not image-based. Current chat-send attachment handling is therefore not general file upload support.
  - file: [src/gateway/chat-attachments.ts:217](https://github.com/openclaw/openclaw/blob/d38ed0831d7372c8505dc944aea69b5b83da1435/src/gateway/chat-attachments.ts#L217)
  - command: `nl -ba src/gateway/chat-attachments.ts | sed -n '217,231p'`
  - sha: [d38ed0831d73](https://github.com/openclaw/openclaw/commit/d38ed0831d7372c8505dc944aea69b5b83da1435)
- **Assistant attachment download/preview support does exist:** Current main serves assistant local media through `/__openclaw__/assistant-media`, and the chat renderer produces anchor/img/video/audio URLs for assistant attachments. Tests cover a document link and image preview through that route, so the download side is at least partially implemented.
