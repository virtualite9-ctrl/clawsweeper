## Summary

Keep PR #70495 open. Current main still has the UX gap this PR targets: the CLI `mcp list/show/set/unset` surface reads only OpenClaw config under `mcp.servers`, but the command help and human-readable output still use generic "MCP servers" wording and do not mention that mcporter `config/mcporter.json` entries are excluded. The prior PR #69726 was closed unmerged, and #70495 is the reopened implementation path rather than a duplicate to close.

## Best Possible Solution

Keep the PR open and review it as the likely implementation path for #65209. The best maintainer path is a narrow current-main update that relabels `openclaw mcp list/show/set/unset` help and non-JSON output as OpenClaw-managed `mcp.servers`, adds a clear note that mcporter `config/mcporter.json` servers are not included, preserves JSON output for scripts, updates the relevant public CLI docs, and adds focused CLI tests for both empty and non-empty list output.
