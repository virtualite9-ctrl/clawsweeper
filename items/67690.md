## Summary

Keep open. Current main has session pruning, docs, config knobs, regression coverage, and a Control UI context-usage warning, but it still lacks the requested pruning visibility/control: no pruning metadata/event, no user-visible pruning preview/history/notification, and no keyword/tag retention strategy.

## Best Possible Solution

Keep this open and narrow it into a maintainer-approved pruning observability/control design. The likely path is to extend `src/agents/pi-hooks/context-pruning/*` to produce safe pruning metadata, add an internal/plugin-visible pruning lifecycle event coordinated with #8606, surface a concise Control UI/chat notice when pruning happens, document existing and new knobs, and leave overflow-message specifics to #9409.
