## Summary

Current main has cache-TTL session pruning, documented config knobs, context usage reporting, and Control UI context-pressure warnings, but it still does not implement the central request: pruning preview/history/notifications, pruning metadata/events, or keyword/tag retention strategy.

## Best Possible Solution

Keep this open and narrow it into a maintainer-approved pruning observability/control design. The likely core path is to extend `src/agents/pi-hooks/context-pruning/*` to produce safe pruning metadata, add a pruning lifecycle event coordinated with #8606, surface a concise Control UI/chat notice when pruning happens, document existing and new knobs, and leave overflow-message specifics to #9409.
