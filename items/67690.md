## Summary

Keep open. Current main has useful partial coverage for session-pruning behavior, configuration, docs, high-context UI warnings, and compaction notices, but it still does not implement the core request for pruning visibility/control: no pruning summary, no pruning event/hook, no UI notification/history of what was pruned, and no keyword/tag retention strategy.

## Best Possible Solution

Keep this issue open and narrow it into a maintainer-approved pruning observability/control design. The likely implementation belongs around `src/agents/pi-hooks/context-pruning/*` plus agent event/plugin/UI surfaces: return safe pruning metadata from the pruner, emit an internal/plugin-visible pruning lifecycle event, surface a concise Control UI/chat notice when pruning happens, document existing and new knobs, and coordinate hook shape with #8606 plus overflow diagnostics with #9409.
