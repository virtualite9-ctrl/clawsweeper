## Summary

Keep PR #69906 open. Current `main` still implements and documents the older WhatsApp behavior where root `channels.whatsapp.groups` can be inherited by multi-account configs, while Telegram already has the requested `isMultiAccount` guard. The PR is a focused implementation for the still-open maintainer-authored #69874, with docs/tests in the provided diff and bot discussion that narrows the security concern to migration/doctor guidance rather than showing the PR is obsolete.

## Best Possible Solution

Keep the PR open for maintainer review or merge against #69874. The best path is to land a scoped WhatsApp fix that mirrors Telegram for `groups`, includes the proposed regression tests and docs/changelog migration note, and handles any broader fail-closed policy as a separate cross-channel follow-up if maintainers want that behavior.
