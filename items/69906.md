## Summary

Current `main` does not contain this WhatsApp multi-account `groups` guard. The code still flat-merges channel/account config, the public docs still say WhatsApp intentionally inherits root `groups` in multi-account setups, and current tests do not cover the regression this PR targets. The PR also has substantive discussion around the narrow allowlist migration risk, with existing doctor-warning coverage on `main`, so this remains a maintainer merge/review decision rather than a cleanup close.

## Evidence

There is no Telegram-style `isMultiAccount` override on `main`.
