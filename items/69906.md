## Summary

Keep #69906 open. Current main still has the WhatsApp behavior this PR is trying to change: `resolveMergedWhatsAppAccountConfig` returns the flat merged account config, so root `channels.whatsapp.groups` can still flow into every WhatsApp account. The docs also explicitly document the current discrepancy with Telegram. The PR is a focused implementation of open maintainer issue #69874, with tests/docs/changelog in the PR context and review discussion addressing the narrow allowlist side-effect rather than making the PR obsolete.

## Best Possible Solution

Keep #69906 open. The best path is maintainer review of the focused WhatsApp account-config guard, its regression tests, and the docs/changelog migration note, with any broader fail-closed group-policy hardening handled as a separate Telegram-plus-WhatsApp product decision if maintainers want it.
