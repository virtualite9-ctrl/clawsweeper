## Summary

Keep #39038 open. Current main has adjacent Windows/node-host improvements, including removal of the old `node host PATH` debug print and Windows scheduled-task PATH handling fixes, but I did not find direct source, test, release, or reporter-confirmation evidence that the reported native Windows 11 24H2 foreground `openclaw node run` hang now reaches or connects to the Gateway.

## Best Possible Solution

Keep tracking this as a native Windows node-host startup bug. The best resolution is either a focused Windows 11 24H2 smoke/regression showing `openclaw node run --host 127.0.0.1 --port 18789` reaches the Gateway connection path, ideally with a non-ASCII profile path and Node 24, or an explicit docs/product decision that the native Windows `--ignore-scripts` install/runtime combination is unsupported and should use a normal npm install or WSL2.
