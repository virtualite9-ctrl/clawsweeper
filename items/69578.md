## Summary

Close as `clawhub`: current `main` already exposes HeyGen avatar-video generation through the bundled `fal` provider, and the repo vision/docs say new optional provider plugins should generally ship as external/ClawHub plugins rather than as additional bundled core plugins.

## Risks / Open Questions

- A dedicated direct-HeyGen plugin could still offer auth/config behavior that the current `fal` route does not expose; closing as `clawhub` defers that work to an external plugin rather than bundling it in core.
- No already-published ClawHub package was linked in the PR context, so closing does not point users to a drop-in replacement yet; it only points to the supported extension path and the existing `fal` workaround.
