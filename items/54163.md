## Summary

Keep open. Current `main` still renders the raw session key as the primary value in the Control UI Sessions table, and the session row data path does not resolve bound-agent `identity.name`/`identity.emoji` into that view.

## Evidence

- **Sessions table still shows raw key in Key column:** The Sessions view header still labels the column `Key`, and each row renders `row.key` as the primary linked text. `displayName` is only shown as an optional muted secondary line, so the raw session key remains the main identifier.
  - file: [ui/src/ui/views/sessions.ts:380](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/ui/src/ui/views/sessions.ts#L380)
  - command: `nl -ba ui/src/ui/views/sessions.ts | sed -n '380,515p'`
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Backend session displayName is not agent identity:** `buildGatewaySessionRow()` derives `displayName` from `entry.displayName`, group-display metadata, `entry.label`, or `origin.label`. It does not resolve configured agent `identity.name` or `identity.emoji` for bound agent sessions.
  - file: [src/gateway/session-utils.ts:1256](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/gateway/session-utils.ts#L1256)
