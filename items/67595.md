## Summary

Keep this open. The LaunchAgent/service-metadata part has a landed partial fix in 67f1266fe8a2f25e9701954992dd8ff12074b498 and 13f9deb619796f128a2bc93b45b9cd6375be2a2a, but the issue also reports `openclaw channels list` surfacing `TypeError: fetch failed` when only auth providers are configured. Current `main` still fetches provider usage by default from `channels list`; the human path only catches and prints the raw error, while the JSON path awaits usage without the same catch. A maintainer comment also explicitly says the service-metadata part is fixed and the channels-list part should keep this issue open.

## Best Possible Solution

Keep this issue open, or split it with the gateway service part marked fixed by 67f1266fe8a2f25e9701954992dd8ff12074b498 / 13f9deb619796f128a2bc93b45b9cd6375be2a2a and a focused remaining issue for `openclaw channels list`. The remaining fix should make provider usage enrichment optional and non-fatal for both human and JSON output, avoid raw `TypeError: fetch failed` output for a simple channel/auth listing, and add regression coverage for no configured chat channels plus auth profiles with a failing usage provider.
