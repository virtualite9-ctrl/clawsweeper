## Summary

Keep this open. Current main has partial gateway-install work, including the narrower stale-loaded-service refresh from #70752, but it still has a macOS LaunchAgent path that serializes durable environment values inline and still lets `channels list` usage fetching surface raw `TypeError: fetch failed` behavior instead of treating usage as an optional, non-fatal add-on.

## Best Possible Solution

Keep this open or split it into two focused issues. For the gateway side, make macOS service install and doctor converge on a LaunchAgent definition that never serializes `OPENCLAW_GATEWAY_TOKEN` or other durable secret-bearing env values inline, while preserving the systemd `EnvironmentFile` behavior and stale-token refresh from #70752. For `channels list`, make provider usage optional and robust: skip it when appropriate, or convert provider fetch failures into structured non-fatal warnings in both human and JSON output, with regression coverage for no chat channels plus auth profiles.
