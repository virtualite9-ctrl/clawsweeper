## Summary

Keep open. PR #69417 targets a real remaining core gap: current main still accepts only scalar literals for top-level `mcp.servers.*.env.*` and `mcp.servers.*.headers.*`, does not register those paths as SecretRef targets, and does not resolve them in core runtime secret collection. Related items cover ACPX/plugin MCP env refs, scalar env propagation, static header forwarding, or dynamic per-request headers, but none supersede this core MCP SecretRef support.

## Best Possible Solution

Keep #69417 open. The best path is a focused core implementation that widens the top-level MCP schema/types to `SecretInput`, registers `mcp.servers.*.env.*` and `mcp.servers.*.headers.*` as supported SecretRef targets, resolves only the active MCP transport surface before launch, updates `secrets configure` and `secrets audit` with conservative MCP credential heuristics, and lands with targeted tests plus public MCP/SecretRef docs.
