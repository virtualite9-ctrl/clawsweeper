## Summary

Keep PR #69417 open. Current main still does not implement core `mcp.servers.*.env.*` / `mcp.servers.*.headers.*` SecretRef support: the schema and types are scalar-only, the SecretRef target registry/command runtime target list do not include those MCP paths, runtime secret collection does not walk `mcp.servers`, and docs do not list those paths as supported SecretRef surfaces. Related #46825 covered ACPX plugin MCP env config, not the core `mcp.servers` surface this PR targets.

## Best Possible Solution

Keep this PR open for maintainer review or rework. The best implementation path is a focused core MCP SecretRef change: widen `McpServerConfig` and the Zod/generated schemas to accept `SecretInput` for `mcp.servers.*.env.*` and `headers.*`, add those paths to the SecretRef target registry and agent runtime command target set, collect and resolve only active MCP transport surfaces at runtime, preserve scalar literals, add audit/configure heuristics that avoid benign literal noise, and update the SecretRef/MCP docs plus regression tests. The related ACPX plugin implementation should not be treated as a substitute for this core `mcp.servers` surface.
