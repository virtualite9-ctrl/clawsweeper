## Summary

Keep open. PR #69417 targets a real remaining core gap on `mcp.servers.*.env.*` and `mcp.servers.*.headers.*`: current `main` still accepts only scalar literals there, does not list those paths in the SecretRef target registry, and does not resolve MCP env/header SecretRefs in the runtime secret collectors. Related closed items cover ACPX/plugin MCP env refs, static env propagation, and static header forwarding; they do not supersede this core `mcp.servers` SecretRef request.

## Best Possible Solution

Keep #69417 open. The best path is a focused core implementation that widens the MCP schema/types to `SecretInput`, registers `mcp.servers.*.env.*` and `mcp.servers.*.headers.*` as supported SecretRef targets, resolves only the active MCP transport surface before launch, updates `secrets configure`/`secrets audit` with conservative MCP credential heuristics, and lands with targeted tests plus docs for the public MCP and SecretRef credential surfaces.
