## Summary

Keep open. Current `main` still has a protocol-schema mismatch: gateway `exec.approvals.set` / `exec.approvals.node.set` validate an approvals file shape that forbids the `source` field, while the exec approvals infra already persists and tests `source: "allow-always"`. This PR addresses a real current bug and there is no evidence it is already implemented on `main` or shipped in the latest release.

## Evidence

- **Gateway schema still rejects extra allowlist properties:** `ExecApprovalsAllowlistEntrySchema` allows `id`, `pattern`, `argPattern`, `lastUsedAt`, `lastUsedCommand`, and `lastResolvedPath`, with `additionalProperties: false`; there is no `source` property on current `main`. An allowlist entry carrying `source: "allow-always"` would therefore fail AJV validation at this boundary.
