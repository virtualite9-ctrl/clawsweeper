## Summary

Keep open. Current main still puts a context-agnostic `Run: openclaw doctor --non-interactive` hint into restart/config-patch sentinels and delivers that same text back through routed session/heartbeat wake paths. I did not find code, docs, or tests that distinguish “recommended” from “actionable here” for approval-constrained surfaces.

## Best Possible Solution

Keep the issue open. The likely core fix belongs in restart-sentinel/config-write messaging, not ClawHub: carry enough delivery/actionability context into the sentinel or message formatter to say whether `openclaw doctor --non-interactive` is runnable here, and otherwise direct the operator to a terminal or approvals-capable Control UI/MCP surface. Add focused tests for config.patch/config.apply sentinel payloads and post-restart routed/heartbeat delivery so the constrained wording cannot regress.
