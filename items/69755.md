## Summary

Keep open. Current main still presents the post-restart doctor recommendation as an immediate `Run: openclaw doctor --non-interactive` hint, and the restart/config/update sentinel paths attach that hint without any context-aware distinction between terminal-capable and approval-constrained delivery surfaces.

## Best Possible Solution

Keep the issue open and implement a core restart-sentinel guidance change. The likely fix belongs around `formatDoctorNonInteractiveHint()` and the sentinel payload builders: include delivery/execution capability context, distinguish recommended follow-up from actionable-here execution, and cover config patch/apply, update, gateway tool restart, and `/restart` with focused tests.
