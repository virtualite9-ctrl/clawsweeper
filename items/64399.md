## Summary

Current main only partially addresses this by stripping ANSI in the Overview Gateway Logs tail. The dedicated Control UI Logs tab still requests `logs.tail`, parses JSON/raw log lines without ANSI stripping or rendering, and renders `message`/`raw` directly. The related attempted fix PR #64431 was closed unmerged, so the remaining Control UI log path still needs a focused fix.

## Best Possible Solution

Keep this issue open and land a focused Control UI logging fix. The best path is one shared ANSI strategy for `logs.tail` consumers: strip ANSI or render it safely before display in both `ui/src/ui/controllers/logs.ts` / `ui/src/ui/views/logs.ts` and the Overview tail, decide whether exports should preserve or strip raw ANSI, and add regression tests for ANSI-bearing JSON log messages, subsystem/context fields, and raw fallback lines. PR #64431 is useful prior art but cannot close this because it was not merged.
