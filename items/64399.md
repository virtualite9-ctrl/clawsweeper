## Summary

Keep open. Current main only strips ANSI for the Overview Gateway Logs tail, while the dedicated Control UI Logs tab still parses and renders message/raw log text without stripping or safely rendering ANSI sequences. The related attempted fix PR #64431 was closed unmerged, so it does not supersede the remaining work.

## Best Possible Solution

Keep this issue open and land a focused Control UI logging fix. The best path is to use one shared ANSI strategy across `logs.tail` consumers: strip ANSI or render it safely before display in `ui/src/ui/controllers/logs.ts` and `ui/src/ui/views/logs.ts`, keep subsystem/context parsing robust, decide whether exports should preserve or strip raw ANSI, and replace the Overview-only local helper with shared coverage. Add regression tests for ANSI-bearing JSON log messages, subsystem/context fields, and raw fallback lines; PR #64431 is useful prior art but cannot close the issue because it was not merged.
