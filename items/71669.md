## Summary

Keep open. Current main still enforces the remote plain-HTTP Control UI rejection and has docs/UI/connection-time guidance, but it does not emit the requested server-side startup warning. The related #56830 PR only addresses protocol/UI copy for #32473 and does not supersede this startup-log request.

## Best Possible Solution

Keep this issue open and implement a targeted gateway startup warning after config, bind, allowed-origin, TLS, and proxy/Tailscale facts are available. The likely implementation point is `src/gateway/server-startup-log.ts` or the adjacent startup/runtime-state path, with regression coverage in `src/gateway/server-startup-log.test.ts`. The message should point operators to HTTPS/Tailscale Serve, an SSH tunnel or localhost access, and the documented break-glass `gateway.controlUi.dangerouslyDisableDeviceAuth` path while preserving the current auth policy.
