## Summary

Keep #71669 open. Current main still has the underlying remote plain-HTTP Control UI rejection, docs and client-side hints explain it, but there is no gateway startup/runtime log warning for the operator-facing condition requested here. Related #32473 remains the broader support/product-policy tracker, and open PR #56830 only addresses protocol/UI copy rather than startup logs.

## Best Possible Solution

Keep #71669 open as a focused gateway UX/DX follow-up. The best implementation is a startup warning in the gateway startup log path, likely near `logGatewayStartup`, that uses current config and bind/TLS context to warn operators when Control UI is enabled and non-local plain-HTTP access will be rejected unless HTTPS/localhost, trusted-proxy, or `dangerouslyDisableDeviceAuth` applies. Add focused startup-log tests and keep docs linked to `https://docs.openclaw.ai/gateway/security` or `https://docs.openclaw.ai/web/control-ui`.
