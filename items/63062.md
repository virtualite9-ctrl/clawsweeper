## Summary

Current main still has the reported OpenRouter caching gap. The OpenRouter helper marks system/developer blocks and strips assistant thinking cache markers, but it does not call the existing trailing-user-message cache helper used by the direct Anthropic path. The linked issue #63034 is still open, and this PR remains the active implementation candidate rather than obsolete cleanup.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to review or land the implementation after confirming the helper’s broader SDK/Copilot impact, preserving the added regression coverage, and updating OpenRouter prompt-caching docs if conversation-message markers become supported behavior.
