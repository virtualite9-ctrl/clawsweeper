## Summary

Keep open. Current main still does not implement `VALUE.md` as a first-class workspace bootstrap/context file, and the only matching implementation PR (#45392) was closed unmerged with maintainer feedback that this is broad product/design work needing a focused design issue and fresh green branch. This issue remains the best place for that maintainer product decision.

## Best Possible Solution

Keep #45382 open as the design/product tracker. Maintainers should decide whether `VALUE.md` belongs in the core workspace model, should remain guidance inside `SOUL.md`/`AGENTS.md`, or should be expressed through existing bootstrap hook/plugin mechanisms. If accepted, a fresh PR should update the filename constants, seeding, load/injection order, minimal/subagent filtering, gateway file APIs, sandbox/dev workspace seeding, docs/templates, and targeted tests together.
