## Summary

Keep open. Current `main` still has the cold-start launch race this PR addresses: `ensureBrowserAvailable()` directly calls `launchOpenClawChrome()` after the reachability probes with no shared in-flight guard, and the browser test suite on `main` still only covers the earlier sequential `#21149` follow-up race, not two simultaneous first-use calls.

## Evidence

That leaves the exact concurrent window described in the PR body.
