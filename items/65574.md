## Summary

Keep this PR open. Current main still resolves MCP stdio env from only the per-server config, then spawns with the MCP SDK default environment plus that resolved env. There is no current-main path that merges ambient gateway HTTP(S)/NO_PROXY proxy vars into MCP stdio children, and the linked production report plus PR review history describe a real unresolved core MCP launch behavior.

## Best Possible Solution

Keep this PR open for maintainer review or rework. The best path is a current-main patch in the MCP stdio launch/transport path that propagates safe ambient proxy-related host env to child processes with explicit precedence tests, while deciding whether NODE_OPTIONS should be propagated, sanitized, documented, or replaced by a safer proxy/TLS-specific seam.
