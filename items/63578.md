## Summary

Keep this PR open. Current main still does not hydrate media/path/file URL hints into buffer/contentType/filename for plugin-handled send actions, and the PR is not obsolete or fully superseded by the related buffer-only work. The latest PR shape responds to review feedback by gating send-media preloading, but it still needs maintainer review of the plugin action contract and which handlers should opt in.

## Best Possible Solution

Keep the PR open for maintainer review or contributor rework. The best path is a guarded public plugin action contract, such as a preloadSendMedia-style opt-in wired from runMessageAction into hydrateAttachmentParamsForAction, with tests for both opt-in hydration and default URL/path forwarding. Coordinate with #41779, #42674, and #57335, but do not close this PR as implemented or duplicate while current main still lacks the plugin send media hydration path.
