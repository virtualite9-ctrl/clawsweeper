## Summary

Keep open. Current main still does not enforce length limits on plugin hook prompt mutations, and PR #38222 remains a plausible core hardening change rather than a duplicate or ClawHub-only item. The branch also addressed the early review comments, but it likely needs refresh for the current shared resolver, CLI, and harness prompt paths.

## Best Possible Solution

Keep the PR open and refresh it against current main. The best implementation is a shared post-merge clamp for all prompt-visible hook result fields, applied after phase hooks, legacy hooks, and phase-plus-legacy joins, then reused by embedded, CLI, and harness prompt preparation with warning logs and regression tests for oversized, combined, legacy, and normal-length cases.
