## Summary

Keep open. Current main still does not enforce length limits on prompt-visible plugin hook output. The PR remains a plausible focused core hardening change, but it likely needs rebasing onto the current shared prompt-hook resolver paths and should cover all prompt-visible fields, not only the older attempt.ts join site.

## Best Possible Solution

Keep the PR open and adapt the hardening to current main. The best path is a shared prompt-visible hook string clamp applied after per-hook merge and after final phase-plus-legacy joins, covering embedded, CLI, and harness prompt preparation paths, with warning logs and regression tests for `before_prompt_build`, legacy `before_agent_start`, combined phase-plus-legacy overflow, system-context fields, and normal-length passthrough.
