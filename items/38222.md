## Summary

Keep open. Current main still has no length cap for plugin hook prompt mutations: phase and legacy hook results are merged with plain concatenation, legacy+phase outputs are joined again, and the final prompt/system prompt injection path applies those strings directly. The PR remains a focused, actionable fix, with prior review comments already reflected in the contributor's follow-up commits.

## Best Possible Solution

Keep the PR open and finish a core-side prompt-mutation bound: clamp hook-injected prompt strings after per-hook merge and after combining legacy plus phase hook results, include warning behavior, preserve model/provider override fields, and add regression tests for both `before_prompt_build` and legacy `before_agent_start`. Maintainers should decide the exact cap and whether all four prompt mutation fields share it.
