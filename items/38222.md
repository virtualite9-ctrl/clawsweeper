## Summary

Keep open. Current main still does not enforce length limits on plugin hook-injected prompt context, and the PR remains a plausible focused core hardening change. The PR discussion is useful: review comments identified concrete gaps, and the PR head now includes follow-up commits addressing the marker-length, warning wording, legacy-hook test, and combined-context concerns, but those fixes are not on main and likely need rebasing to the current shared prompt-hook resolver.

## Best Possible Solution

Keep the PR open and adapt the hardening to current main. The best implementation is a shared clamp for prompt-visible hook result strings after per-hook merge and after the final phase-plus-legacy join, used by embedded, CLI, and harness prompt preparation paths, with warning logs and regression tests for `before_prompt_build`, legacy `before_agent_start`, combined results, and normal-length passthrough.
