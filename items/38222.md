## Summary

Keep open. Current main still does not enforce length limits on plugin hook-injected prompt context, and this PR remains a plausible focused core hardening change. The prior PR discussion is substantive: reviewers flagged marker-length, warning wording, legacy-hook coverage, and combined phase-plus-legacy overflow issues, and the PR head includes follow-up commits addressing those concerns, but that work is not present on main.

## Best Possible Solution

Keep the PR open and adapt the hardening to current main. The best implementation is a shared clamp for prompt-visible hook result strings after per-hook merge and after final phase-plus-legacy joining, used by embedded, CLI, and harness prompt preparation paths, with warning logs and regression tests for `before_prompt_build`, legacy `before_agent_start`, combined results, system-context fields, and normal-length passthrough.
