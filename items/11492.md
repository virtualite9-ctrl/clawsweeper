## Summary

Keep #11492 open. Current main at 7e13f3f5140168e31027b39a9855ade3fadf24d5 still does not expose Slack `channel-info` or `channel-list` through the Slack message-tool surface, even though the shared action vocabulary, Slack `channelInfo` config flag, and Discord parity implementation exist. Related PR #11491 is closed unmerged, and the open related PR #53199 only partially overlaps because it proposes `channel-list` plus search, not the requested `channel-info` action.

## Best Possible Solution

Keep the issue open until the bundled Slack plugin exposes both `channel-info` and `channel-list` through the message tool. The likely implementation belongs in `extensions/slack`: add discovery behind the existing `channelInfo` gate, dispatch the generic actions to Slack runtime actions, implement Slack `conversations.info` and paginated `conversations.list` helpers, and update Slack docs plus focused discovery, dispatch, and runtime tests. PR #53199 can be useful for the `channel-list` portion, but it should not close #11492 unless `channel-info` is also implemented or a narrower canonical follow-up is opened.
