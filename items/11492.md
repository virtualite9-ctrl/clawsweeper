## Summary

Keep #11492 open. Current main still does not expose Slack `channel-info` or `channel-list` through the message tool, despite the generic action names, Slack `channelInfo` config key, and Discord parity path existing. The related PR #11491 is closed unmerged, and #53199 is still open and only partially covers the issue because it adds `channel-list` but not the requested `channel-info` action.

## Best Possible Solution

Keep the issue open until Slack exposes both `channel-info` and `channel-list` through its bundled plugin message-tool surface. The likely implementation belongs in `extensions/slack`: add discovery behind the existing `channelInfo` gate, dispatch generic actions to Slack runtime actions, implement `conversations.info` and paginated `conversations.list` helpers, and update Slack docs plus focused discovery/dispatch/runtime tests.
