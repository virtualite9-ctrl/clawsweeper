## Summary

The request is for an optional contextual-skill behavior that current main already supports via existing plugin and hook surfaces. OpenClaw's documented scope says this kind of capability should ship as an external plugin/bundle and be published through ClawHub rather than added to core.

## Evidence

- **Vision keeps optional capability in plugins, not core:** The project vision says optional capability should usually ship as plugins, prefers bundle-style plugins when they can express the capability, says plugin promotion belongs in ClawHub, and explicitly says new core skills that can live on ClawHub are out of scope.

## Risks / Open Questions

- If the desired implementation specifically requires mutating inbound `bodyForAgent` from native plugin code, current docs more clearly support injection at `before_prompt_build` than direct message-body rewriting; a narrower SDK-gap issue could still be valid later.
- No linked external ClawHub package or PR was present in the provided issue discussion, so this close is based on scope and existing extension points rather than a shipped third-party implementation.
