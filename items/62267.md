## Summary

Keep open. Current main has the relevant Ollama mitigation knobs and source-backed handling for configured context caps, request timeouts, lean tool surfaces, and `params.num_ctx`, but this recent report is still missing the exact current-main reproduction data needed to decide whether OpenClaw is exceeding a configured cap or Ollama is simply spending the configured prompt budget in prompt evaluation. A maintainer comment already left this issue open for that data, so none of the allowed close reasons apply.

## Best Possible Solution

Keep this issue open for the requested current-main retest. The next maintainer action should be to compare OpenClaw's actual submitted prompt/context against configured `contextWindow` or `contextTokens` and Ollama `prompt_eval_count`; if OpenClaw exceeds the cap, fix the prompt budgeting or compaction path, and if the counts match but prompt eval dominates, improve the local-model docs/diagnostics around `params.num_ctx`, `timeoutSeconds`, `localModelLean`, and model sizing.
