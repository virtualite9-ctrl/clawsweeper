## Summary

Keep #63719 open. Current main still only cleans package-level npm rename directories and verifies the installed package tree; it does not verify or repair the public global `bin/openclaw` link. The linked fix PR #63794 directly targets the bug, but it remains open and unmerged, so the issue is still useful as the user-facing tracker until a focused fix lands.

## Best Possible Solution

Keep #63719 open until a focused updater fix lands on main. The fix should run after successful global npm updates, read and normalize the installed package `bin` manifest, infer the npm global bin directory, verify/recreate the public `openclaw` link when the package entry exists, and safely clean only npm-owned `.openclaw-*` bin reify leftovers. It should cover the active `src/cli/update-cli/update-command.ts` path and shared helpers with regression tests, then close the issue from the merged PR/release.
