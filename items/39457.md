## Summary

Keep open. Current `main` still has no user-configurable post-update lifecycle script for `openclaw update`, and the update path force-reinstalls the Gateway service, which rewrites the LaunchAgent plist. This remains a coherent core enhancement request rather than something already implemented or better moved to ClawHub.

## Evidence

- **Update CLI only documents restart/plugin-sync post-update behavior:** `openclaw update` documents `--no-restart` and post-update plugin integrity reporting, but no user post-update hook or lifecycle script option is present.
  - file: [docs/cli/update.md:35](https://github.com/openclaw/openclaw/blob/cb2c36b04975e34ab7014a11575d6589aa96ae32/docs/cli/update.md#L35)

  - sha: [cb2c36b04975](https://github.com/openclaw/openclaw/commit/cb2c36b04975e34ab7014a11575d6589aa96ae32)
- **Successful update refreshes the service via forced reinstall:** After update, the CLI calls `gateway install --force` through `refreshGatewayServiceEnv`, which is the exact path that would restage the macOS service definition before restart.
  - file: [src/cli/update-cli/update-command.ts:256](https://github.com/openclaw/openclaw/blob/cb2c36b04975e34ab7014a11575d6589aa96ae32/src/cli/update-cli/update-command.ts#L256)

- **Restart path prepares service refresh and restart, with no user hook invocation in between:** When restart is enabled and a service is loaded, the update flow sets `refreshGatewayServiceEnv = true` and proceeds directly into `maybeRestartService(...)`; there is no callout to a filesystem hook such as `~/.openclaw/hooks/post-update.sh`.
  - file: [src/cli/update-cli/update-command.ts:1395](https://github.com/openclaw/openclaw/blob/cb2c36b04975e34ab7014a11575d6589aa96ae32/src/cli/update-cli/update-command.ts#L1395)

- **LaunchAgent install rewrites the plist from generated arguments/environment:** `writeLaunchAgentPlist(...)` rebuilds the plist from `programArguments`, `workingDirectory`, and `environment`, then writes it with `fs.writeFile(...)`. Manual plist edits would be replaced on reinstall.
