## Summary

Keep #39457 open. Current main has post-update plugin sync, but it still does not provide the requested executable/user lifecycle hook after the gateway service plist refresh and before restart. The loaded-service update path still refreshes the service with `gateway install --force`, then immediately runs the restart helper or daemon restart, and existing plugin/internal hook catalogs cover Gateway/session/message/install events rather than this updater-owned service-refresh slot.

## Best Possible Solution

Keep the issue open. The best implementation is a core updater-owned post-update hook, or an equivalent durable service customization mechanism, placed after `refreshGatewayServiceEnv` succeeds and before `runRestartScript`/`runDaemonRestart` in `src/cli/update-cli/update-command.ts`. It should define executable/timeout/error/logging semantics, add focused tests for the loaded macOS LaunchAgent update path, and document the supported contract.
