## Summary

Keep #39457 open. Current main still refreshes a loaded gateway service with `gateway install --force` and then restarts it without any user-owned post-update lifecycle script between the plist rewrite and restart. Durable `.env`/config env support covers part of the reporter's env-var pain, and the newer wrapper items #69400/#69425 are relevant, but they are open/unmerged and do not implement the requested updater-owned post-refresh/pre-restart hook.

## Best Possible Solution

Keep the issue open. The best maintainer path is either to add a core updater-owned post-update hook after `refreshGatewayServiceEnv` succeeds and before `runRestartScript`/`runDaemonRestart`, or explicitly supersede it with a shipped durable wrapper/config mechanism such as #69400/#69425. Either path needs focused tests for the loaded macOS LaunchAgent update flow and user-facing docs for the supported contract.

## Risks / Open Questions

- No live macOS LaunchAgent update was executed because this review was read-only; the source path directly shows the missing lifecycle slot.
- A future hook needs explicit executable, timeout, logging, environment, and failure semantics so update cannot silently hang or mask restart failures.
- If maintainers choose the wrapper/config route instead of a generic post-update hook, #39457 should be closed only after a canonical wrapper fix ships or after maintainers explicitly decide that narrower path supersedes this request.
