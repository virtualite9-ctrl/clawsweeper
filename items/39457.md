## Summary

Keep #39457 open. Current main still refreshes an installed gateway service by running `gateway install --force`, then immediately hands off to the restart path. The existing hook/plugin surfaces cover Gateway, session, install, and message lifecycles, but not this updater-owned slot between LaunchAgent plist rewrite and relaunch.

## Best Possible Solution

Keep the issue open. The best path is a core updater-owned post-update lifecycle hook, or an equivalent durable service customization mechanism, placed after `refreshGatewayServiceEnv` succeeds and before `runRestartScript`/`runDaemonRestart` in `src/cli/update-cli/update-command.ts`. Add focused tests for the loaded macOS LaunchAgent update path and document the supported hook or configuration contract.
