## Summary

Keep #39457 open. Current `main` still refreshes a loaded gateway service by forcing `gateway install --force`, then immediately runs the restart handoff; there is no supported user `post-update` hook between the LaunchAgent plist rewrite and relaunch, and the existing hook/plugin surfaces run in Gateway/plugin lifecycles rather than the updater-owned service-refresh slot.

## Best Possible Solution

Keep the issue open. The best implementation path is a core updater-owned lifecycle hook, or an equivalent durable service customization mechanism, likely after `refreshGatewayServiceEnv` succeeds and before `runRestartScript`/`runDaemonRestart` in `src/cli/update-cli/update-command.ts`. Add focused tests for the loaded macOS LaunchAgent update path proving hook-applied plist or wrapper changes are in place before relaunch, and document the supported hook/configuration surface.
