## Summary

The PR guards a mocked `prompter.select()` returning `undefined`, but current main's supported wizard prompter contract and production Clack wrapper do not produce `undefined` on selection or cancellation. The visible QuickStart trim crash from the related report is already covered by the shipped metadata-sanitization fix and regression tests, so this PR is not needed as a current-main fix.

## Best Possible Solution

Close this PR without merging the narrow QuickStart-only undefined-selection guard. Keep the shipped metadata-sanitization behavior for the related QuickStart trim crash; if maintainers want additional defensive hardening, add it centrally at the `WizardPrompter` boundary or through a shared selection validator that covers both QuickStart and manual channel setup paths.
