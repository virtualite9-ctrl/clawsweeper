## Summary

Keep open. Current main still lacks the PR's quickstart rendered-choice guard: QuickStart only checks the `__skip__` sentinel, then forwards any other select result into channel setup. The PR is a focused bug fix with a matching regression test, and the remaining current-main path can still receive an omitted wizard answer or undefined autocomplete selection.

## Best Possible Solution

Keep the PR open for maintainer review. Refresh it against current main and land either the rendered-channel-choice guard with the undefined QuickStart regression test, or a broader select-answer validation that rejects or normalizes omitted select answers before flow code sees them.

## Risks / Open Questions

- The normal first-party Clack terminal cancel path already throws `WizardCancelledError`; the strongest remaining repro surface is gateway wizard/RPC, autocomplete no-match submission, or another custom prompter returning no value.
- A broader wizard select-answer validation may be better long term than only guarding the QuickStart channel flow, but this PR's local guard and regression test still address a real current-main gap.
