## Summary

Keep #36525 open. Current main at 67b9167b808f2ff065c91665d6f36256a5c23ef2 still uses the existing `workspaceAccess` model: `rw` selects the agent workspace for `/workspace`, `none`/`ro` use the sandbox workspace but keep writes gated off, and split non-`none` modes can still expose the agent workspace at `/agent`. The related #37634/#46300 items cover the narrower writable-`none` bug, but they do not supersede this broader semantic refactor request.

## Best Possible Solution

Keep #36525 open for maintainer product direction and implementation. A complete fix would make `/workspace` consistently represent the per-scope sandbox workspace, make writeability follow the resolved mount rather than a global `workspaceAccess === "rw"` gate, remove implicit `/agent` exposure unless explicitly configured through documented binds, update Docker/browser/SSH/OpenShell paths together, and revise regression tests plus public sandboxing/security docs. Keep #37634/#46300 as narrower writable-`none` implementation context, not full supersession of this broader semantic refactor.
