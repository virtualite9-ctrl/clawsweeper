## Summary

Keep #36525 open. Current main at 631552c554095b0c520376db9131dac0f5427f94 still implements the old workspaceAccess semantics: rw selects the real agent workspace for /workspace, none and ro use the sandbox workspace but keep it read-only, split ro/rw modes can still expose the agent workspace at /agent, and write gates still require workspaceAccess === "rw". The related #37634 item is narrower and does not supersede this broader semantic refactor request.

## Best Possible Solution

Keep #36525 open for a coordinated sandbox semantics decision and implementation. A complete fix would make the active /workspace consistently represent the sandbox workspace, make writeability follow the resolved mount rather than a global workspaceAccess === "rw" check, remove implicit /agent exposure unless deliberately configured through documented binds, update Docker/browser/SSH/OpenShell paths together, and revise tests plus public sandbox/security docs. Keep #37634 as narrower writable-none context, not as the canonical replacement for this broader refactor.
