## Summary

Keep open. Current main still only partially handles this session cruft: maintenance cleans old `.deleted` archives and optionally `.reset` archives, store rotation caps `sessions.json.bak.*`, and disk-budget cleanup can remove archive artifacts only when configured and over budget. It still lacks default age-based cleanup for `.jsonl.bak.*`, the proposed recycle/soft-delete settings, and any recognized `sessions_saved.json` path.

## Best Possible Solution

Keep this issue open and implement a focused session-maintenance enhancement in `src/config/sessions/*` plus the gateway transcript archive helpers. Add tests for orphaned `.jsonl.deleted.*`, `.jsonl.reset.*`, `.jsonl.bak.*`, and any legacy `sessions_saved.json` support maintainers choose to keep; then align `SessionMaintenanceConfig`, zod schema/help, generated config docs, and public session docs around either the proposed recycle/retention model or a documented replacement using existing disk-budget controls.
