## Summary

Keep open. Current main still only partially addresses the reported session cruft: maintenance removes old `.deleted` and `.reset` transcript archives and caps `sessions.json.bak.*`, but default age-based cleanup still does not sweep `.jsonl.bak.*`, there is no recycle/soft-delete configuration, and `sessions_saved.json` is not recognized in source, schema, docs, or tests.

## Best Possible Solution

Keep this open and implement a focused session-maintenance enhancement in `src/config/sessions/*` plus the gateway archive helpers. Add tests covering orphaned `.jsonl.deleted.*`, `.jsonl.reset.*`, `.jsonl.bak.*`, and any legacy `sessions_saved.json` behavior maintainers want to support; then align `SessionMaintenanceConfig`, zod schema/help, and public docs around either the proposed recycle/retention model or a documented replacement using existing disk-budget controls.
