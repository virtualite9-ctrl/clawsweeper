## Summary

Keep #24196 open. Current main still exposes Discord outbound media as a single `mediaUrl`/`filename` send path, and existing multi-media handling fans attachments out across separate sends rather than building one Discord message with up to 10 native file uploads. The latest issue discussion already includes a maintainer-side Codex review keeping this open, and current source still supports that conclusion.

## Best Possible Solution

Keep the issue open and implement native Discord multi-file sending in the bundled Discord plugin: add an ordered multi-attachment input shape for message actions/outbound payloads, preserve the existing single-file fields, validate Discord per-message file count and file size limits, build one Discord `files` array for up to the supported limit, split overflow into follow-up messages, and update focused Discord docs and tests.
