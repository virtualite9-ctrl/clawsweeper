## Summary

Keep #24196 open. Current main still does not implement native Discord multi-file uploads in one message: the Discord message action and runtime accept a single media/path/filePath value, sendMessageDiscord takes one mediaUrl, and the Discord payload builder creates exactly one uploaded file. Existing multi-media payload support sends separate Discord messages, which does not satisfy the issue's request for Discord's native up-to-10 attachments per message.

## Best Possible Solution

Keep the issue open and implement native Discord multi-file sending in the Discord plugin: add an ordered multi-attachment input shape for the message action/outbound path, load and validate up to Discord's per-message limit, build a single Discord payload files array, split overflow into follow-up messages, and update Discord docs plus focused tests for action runtime, send payload construction, component/file-block behavior, and overflow handling.
