## Summary

Keep #24196 open. Current main still does not implement native multi-file Discord message uploads for the normal outbound/message-action path: it accepts one media/path/filePath value, passes a scalar `mediaUrl`, and builds a Discord `files` array with exactly one item. Multi-media payloads are still fanned out through repeated single-media sends rather than one Discord message with multiple attachments. The existing 2026-04-26 maintainer-side Codex comment in the issue remains consistent with current main.

## Best Possible Solution

Keep the issue open and implement native Discord multi-file sending in the bundled Discord plugin: add a backward-compatible ordered multi-attachment input shape for message actions and outbound payloads, preserve existing single `media`/`path`/`filePath` behavior, validate/split by Discord count and size limits, send supported batches as one `files` array, and update focused Discord tests and docs.
