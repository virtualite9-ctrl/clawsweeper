## Summary

Keep #24196 open. Current main still does not provide native multi-file Discord uploads for ordinary message actions or outbound reply delivery: the main send path accepts one media URL, builds one uploaded file, and shared reply helpers fan out multiple media URLs into separate sends. The existing maintainer-side Codex comment also explicitly kept this issue open, and the relevant Discord/reply-payload surfaces have not changed since that review.

## Best Possible Solution

Keep #24196 open and implement native multi-file sending in the bundled Discord plugin: add a backward-compatible ordered multi-attachment input shape for message actions and outbound payloads, preserve existing single `media`/`path`/`filePath` behavior, validate and split by Discord count/size limits, send eligible batches as one Discord `files` array, and update focused Discord tests plus user-facing docs.
