## Summary

Keep open. Current main still does not document `ZALOUSER_PROFILE` or `ZCA_PROFILE` in the Zalo Personal channel docs, while the bundled plugin manifest, runtime resolver, and tests all show those env vars are real supported behavior. The PR is a small docs-only patch that directly addresses the remaining gap and is not superseded by the earlier closed duplicate PR.

Merge or lightly refresh PR #70498 so `docs/channels/zalouser.md` documents `ZALOUSER_PROFILE`, `ZCA_PROFILE`, the exact resolver precedence from `extensions/zalouser/src/accounts.ts`, and the multi-account caveat. No core runtime change is needed.
