## Summary

Keep open. Current main still does not document `ZALOUSER_PROFILE` or `ZCA_PROFILE` in the Zalo Personal docs, while the bundled plugin declares those env vars, the account resolver actively uses them, and tests cover the same precedence the PR proposes to document. The related #69643 was an earlier closed copy of the same PR, not a shipped or canonical replacement, and the closed feature request #65864 does not make this open docs PR obsolete.

## Best Possible Solution

Keep #70498 open and let maintainers merge or lightly adapt the docs section so `docs/channels/zalouser.md` documents the two manifest-declared Zalo Personal profile env vars, their runtime precedence, and the multi-account caveat. No core/runtime change is needed.
