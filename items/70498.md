## Summary

Keep #70498 open. Current main still declares and reads `ZALOUSER_PROFILE`/`ZCA_PROFILE`, and tests cover the precedence, but the Zalo Personal channel docs still do not document those environment variables. The PR is a focused docs-only change re-opening #69643, and the related closed issue #65864 matches the same still-present docs gap rather than superseding the PR.

## Best Possible Solution

Keep the PR open for maintainer review or merge after any needed rebase. The best product/docs outcome is to add an Environment variables section to `docs/channels/zalouser.md` that matches the manifest and `resolveProfile` precedence: explicit `profile`, then `ZALOUSER_PROFILE`, then `ZCA_PROFILE`, then account/default fallback.
