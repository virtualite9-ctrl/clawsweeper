## Summary

Keep #37842 open. Current main still lacks graph-aware sessions_send cycle detection: provenance remains source-only, sessions_send creates fresh inter-session provenance without a chain/trace, and the A2A limiter still bounds only the direct requester-target exchange.

## Best Possible Solution

Keep #37842 open for a core sessions_send fix. Add an explicit, schema-backed chain or trace field, propagate it through createSessionsSendTool, runAgentStep, and A2A follow-up runs, reject or warn when the target agent/session already appears in the chain, and add regression coverage for A->B->C->A while preserving existing maxPingPongTurns behavior for direct A<->B exchanges.

- Multi-agent deployments with tools.agentToAgent enabled can still form sessions_send cycles across three or more agents because current main has no source-level graph-cycle guard.
- A future fix likely needs a protocol/schema-aware additive design; simply sending an extra provenance field would not work against the current additionalProperties: false gateway schema.
