## Summary

Keep open. Current main still does not implement graph-aware chain tracing for `sessions_send`: provenance has no chain field, the gateway schema would reject one, and the send/A2A paths only carry source provenance plus pairwise `maxPingPongTurns` behavior. The linked PR #37945 is useful prior work but was closed unmerged in the provided GitHub context, and related #51832 covers a different gateway error-cascade mechanism.

## Best Possible Solution

Keep #37842 open until OpenClaw implements a core graph-aware inter-session guard or a maintainer makes an explicit product decision. A likely fix belongs in the `sessions_send`/agent-run provenance path: add a backwards-compatible chain field to `InputProvenance` and the gateway schema, propagate it through nested agent runs, block or clearly error when the target agent/session is already in the chain, and add regression coverage for A→B→C→A while preserving the existing direct `maxPingPongTurns` behavior.
