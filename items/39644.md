## Summary

Keep PR #39644 open. Current main at 631552c554095b0c520376db9131dac0f5427f94 still has both Windows behaviors the PR fixes: PowerShell is advertised as a completion shell but `installCompletion()` only installs zsh/bash/fish, and Windows time-format detection still invokes a hard-coded `powershell` executable instead of the existing PowerShell resolver that prefers PS7. The PR remains useful, though it needs to be ported from the older `completion-cli.ts` layout to current `completion-runtime.ts`.

## Best Possible Solution

Keep the PR active or supersede it only with a landed port: add a PowerShell branch in `src/cli/completion-runtime.ts`, write `. "..."` dot-source syntax, show a PowerShell-safe reload hint, switch `src/agents/date-time.ts` to `resolvePowerShellPath()` with non-interactive flags, and add focused regression tests for PowerShell install/idempotence plus Windows time-format executable selection.
