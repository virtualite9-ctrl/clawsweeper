## Summary

Keep PR #39644 open. Current main still exposes PowerShell as a supported completion target, but `installCompletion()` only installs zsh/bash/fish and still reports PowerShell as unsupported. The Windows time-format path also still invokes a hard-coded `powershell` executable instead of the existing resolver that prefers PS7. The PR remains useful, though it needs to be ported from the old `completion-cli.ts` layout to current `completion-runtime.ts`.

## Best Possible Solution

Keep the PR open or supersede it only with a landed port. The fix should update `src/cli/completion-runtime.ts` to install PowerShell completion with `. "..."` dot-source syntax and a PowerShell-safe reload hint, update `src/agents/date-time.ts` to call `resolvePowerShellPath()` with non-interactive flags, and add focused regression tests for the Windows/PowerShell paths.
