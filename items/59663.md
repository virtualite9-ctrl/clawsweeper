## Summary

Current `main` already provides a supported headless ACPX auto-approval path via `permissionMode=approve-all`, so the reported "deny or fail only" behavior does not match present OpenClaw behavior. The linked PR for `nonInteractivePermissions=allow` remains open and unmerged, and its own review discussion notes that `approve-all` already bypasses the non-interactive fallback path.

## Evidence
