## Summary

Current main already prevents raw tool results from being sent through normal chat surfaces by default: tool event payloads drop `result`/`partialResult` unless `verbose=full`, normal verbose mode is metadata-only, raw tool error details are suppressed unless verbose is enabled, and oversized tool results are capped/truncated. The same behavior is present in the released `v2026.4.22` tree.

## Evidence

## Close Comment

Closing from a Codex review because this behavior is already implemented on current `main`, and the same guard is present in the shipped `v2026.4.22` release. In `src/gateway/server-chat.ts:903`, tool events delete `result` and `partialResult` unless verbose is `full`; in `src/agents/pi-embedded-subscribe.ts:444`, normal verbose mode only emits metadata/tool summaries and raw tool output is only forwarded for `verbose=full`; and `src/agents/pi-embedded-runner/run/payloads.ts:68` summarizes tool failures without raw detail unless verbose is enabled. The released tests in `src/gateway/server-chat.agent-events.test.ts:1057` and docs in `docs/tools/thinking.md:73` match that behavior. If this is still happening on a current build without `verbose=full`, please open a fresh issue with the exact channel/plugin, verbose setting, and a minimal repro so we can trace the bypass path.
