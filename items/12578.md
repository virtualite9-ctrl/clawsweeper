## Summary

Keep #12578 open. Current main at 406ae72fd2789c78053f6b94006ec8f488bc1c00 still does not implement a persistent `agents.defaults` usage-footer default: the config schema/types expose comparable defaults but no `usageDefault`, `responseUsageDefault`, or `responseUsage`; reply formatting reads only session state; `/usage` and gateway patching still clear or set the per-session `responseUsage`; and docs still describe `/usage` as per-session persistence.

## Best Possible Solution

Keep #12578 open until a focused core config change lands. The maintainer path should choose one public key under `agents.defaults`, add schema/types/generated help/docs, resolve response usage from explicit session override first and inherited default second, preserve explicit `off` versus clear-to-inherit semantics, and add regression coverage for inherited default, explicit off, clearing to inherit, and session override behavior.
