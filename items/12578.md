## Summary

Keep #12578 open. Current main still does not implement a persistent default for per-response usage footers: the reply runner resolves usage footer mode only from session `responseUsage`, `/usage` still mutates/deletes that per-session field, session construction only preserves an existing value, and `agents.defaults` schema/types/docs do not expose `usageDefault`, `responseUsageDefault`, or `responseUsage`. The open related PRs #48635 and #69618 remain relevant implementation attempts, but they are not current-main behavior.

## Best Possible Solution

Keep #12578 open until a focused implementation lands. The likely path is to add an inherited `agents.defaults.responseUsage` or `agents.defaults.responseUsageDefault` config key to schema/types/generated help/docs, apply it only when a session has no explicit `responseUsage` override, keep `/usage off|tokens|full` as the per-session override, and add regression tests for inherited default, explicit off, and clear-to-inherit behavior.
