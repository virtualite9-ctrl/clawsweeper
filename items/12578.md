## Summary

Keep #12578 open. Current main still has no persistent `agents.defaults` usage-footer default, and the existing implementation remains session-scoped: config schema/types do not expose `usageDefault`, `responseUsageDefault`, or `agents.defaults.responseUsage`; reply rendering resolves the footer from session `responseUsage`; `/usage` mutates the session entry; and docs still describe `/usage` as per-session. The related PRs #48635 and #69618 are relevant implementation attempts, but the provided GitHub context shows both are open and unmerged, so they are not current-main behavior.

## Best Possible Solution

Keep #12578 open until a focused implementation lands. The maintainer path should choose one public config key, add it to config schema/types/generated help/docs, resolve the footer mode from session override first and `agents.defaults` second, preserve `/usage off|tokens|full` as a per-session override with a clear-to-inherit path, and add regression tests for inherited default, explicit off, and session override behavior.
