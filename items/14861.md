## Summary

Keep #14861 open. Current main still has targeted startup diagnostics, plugin diagnostic logging, startup trace timings, and Gmail watcher success/actionable-failure logs, but it still does not implement the requested aggregate gateway boot summary. The `hooks.enabled=true` plus missing `hooks.gmail.account` case remains silent during gateway boot because startup only calls the Gmail watcher wrapper when the account is present, and the wrapper itself suppresses that reason.

## Best Possible Solution

Keep #14861 open. The best maintainer path is to add a structured startup outcome surface in `startGatewaySidecars` or `startGatewayPostAttachRuntime`, record started/skipped/failed results for Gmail watcher, internal hooks, channels, plugin services, memory, cron, heartbeat, scheduled services, and related sidecars, then log a concise boot summary after sidecars settle. Add regression coverage for the aggregate summary and for `hooks.enabled=true` with missing `hooks.gmail.account`.
