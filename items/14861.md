## Summary

Keep #14861 open. Current main still has partial diagnostics only: plugin load failures, startup trace timings, readiness gating, and targeted Gmail lifecycle logs. It does not provide the requested gateway boot summary, and the specific missing `hooks.gmail.account` Gmail watcher startup reason remains silent.

## Best Possible Solution

Keep #14861 open. The best fix is a maintainer-approved gateway startup outcome surface: have `startGatewaySidecars` or `startGatewayPostAttachRuntime` collect structured started/skipped/failed results for Gmail watcher, internal hooks, channels, plugin services, memory, cron/heartbeat/scheduled services, and related sidecars, then log a concise post-sidecar boot summary. Add regression tests for the aggregate summary and specifically for the `hooks.enabled=true` plus missing `hooks.gmail.account` case.
