## Summary

Keep open. Current `main` still lacks the nested-lane cleanup default this PR proposes: gateway-routed agent runs only pass `cleanupBundleMcpOnRunEnd` when the request explicitly sets it, `sessions_send`/`runAgentStep` nested calls do not set it, and the embedded runner only tears down bundled MCP runtimes when that flag is `true`. I found no evidence on `main` or in the provided PR timeline that this exact gateway path was already fixed elsewhere.
