## Close Comment

Closing as implemented on main after Codex review. The Control UI context notice now computes usage from `session.totalTokens` in `ui/src/ui/chat/context-notice.ts:53`, so it reflects the current context window instead of cumulative `inputTokens`. The regression test in `ui/src/ui/chat/context-notice.test.ts:21` covers the reported shape directly: it sets `inputTokens: 757300`, `totalTokens: 190000`, `contextTokens: 200000`, and asserts the UI shows `190k / 200k` and not `757.3k / 200k`. This same fix is already present in release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`).
