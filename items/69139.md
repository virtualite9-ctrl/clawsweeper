## Summary

Keep PR #69139 open. Current main still rejects root-level `paperclip` on Gateway `agent` params: `AgentParamsSchema` is strict and has no `paperclip` field, the `agent` handler validates before dispatch, and the only current `paperclip` acceptance coverage is for `wake` params. The PR discussion also includes a live production corroboration of the same validation failure, so this is not obsolete cleanup.

## Best Possible Solution

Keep the PR open and have maintainers choose the protocol-compatible fix: either merge a small additive schema change with focused `agent` validation coverage, or replace it with a generic/documented metadata path and coordinate the Paperclip adapter/docs. Current main does not solve the reported runtime breakage.
