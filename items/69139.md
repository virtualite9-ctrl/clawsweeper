## Summary

Keep open. Current main still rejects root-level `paperclip` metadata on `agent` params: `AgentParamsSchema` has no `paperclip` field and remains closed with `additionalProperties: false`, and the gateway validates that schema before dispatch. The only current Paperclip-shaped compatibility evidence is for `wake` params, not `agent` requests. The PR remains a focused, unmerged fix for a corroborated gateway protocol compatibility bug, though maintainers may want a core-agnostic metadata shape instead of a Paperclip-named field.

## Best Possible Solution

Do not auto-close this PR. Keep it open as an actionable gateway protocol compatibility fix, and resolve it with an additive `agent` params metadata contract plus a regression test. If maintainers want to preserve the core-agnostic boundary, adapt the PR toward a generic opaque metadata field or a supported `inputProvenance`/adapter context path rather than a Paperclip-named root key.
