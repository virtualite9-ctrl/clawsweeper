## Summary

Keep #69139 open. Current main still rejects the root-level `paperclip` field on `req agent` params: `AgentParamsSchema` is closed with `additionalProperties: false` and has no `paperclip` property, and the gateway validates `agent` requests against that schema before dispatch. The already-shipped Paperclip-related change only made `wake` params forward-compatible; it does not cover `agent` requests. The PR’s supplied context and live-triage comment describe the same validator failure and a focused fix path, so this remains a useful, actionable PR rather than obsolete cleanup.

## Best Possible Solution

Keep #69139 open as the focused fix path for the Paperclip/OpenClaw gateway schema mismatch. The best product/code direction is to accept an opaque, optional root-level `paperclip` metadata record on `AgentParamsSchema`, keep the strict schema for unrelated fields, add the regression test from the PR, and update any generated protocol artifacts or docs required by the gateway protocol boundary.
