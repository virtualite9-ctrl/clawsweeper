## Summary

Keep open. Current `main` does not already contain this PR’s workflow, and the existing Android CI only runs `test-play`, `test-third-party`, and `build-play` on the matrix. That means the proposed `thirdParty` debug APK build/artifact path is not already implemented on `main`, so this PR is not obsolete under the allowed close reasons. The bot review also raises substantive workflow-design problems, but those are mergeability concerns, not evidence for cleanup closure.
