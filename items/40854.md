## Summary

Current main already fixes the reported duplicate `plugins.entries.feishu` rewrite: configured bundled channels are normalized as chat channels and auto-enabled under `channels.<id>.enabled`, not `plugins.entries.<id>`. Feishu is now part of that bundled channel catalog, and the same logic is present in release tag `v2026.4.22`.

## Evidence

- **Configured channels are normalized through the bundled channel registry:** `resolvePluginIdForChannel()` first calls `normalizeChatChannelId(channelId)`, and configured channel ids are collected through the same normalization path before auto-enable runs.
  - file: [src/config/plugin-auto-enable.shared.ts:234](https://github.com/openclaw/openclaw/blob/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503/src/config/plugin-auto-enable.shared.ts#L234)
  - sha: [b0d2003fe65b](https://github.com/openclaw/openclaw/commit/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503)
- **Built-in channels no longer create `plugins.entries.<id>`:** `registerPluginEntry()` writes bundled chat channels to `channels[builtInChannelId].enabled = true`; only non-channel plugins fall back to `plugins.entries[pluginId].enabled = true`. That removes the rewrite pattern described in the issue for `plugins.entries.feishu`.
  - file: [src/config/plugin-auto-enable.shared.ts:596](https://github.com/openclaw/openclaw/blob/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503/src/config/plugin-auto-enable.shared.ts#L596)

  - file: [src/config/plugin-auto-enable.core.test.ts:60](https://github.com/openclaw/openclaw/blob/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503/src/config/plugin-auto-enable.core.test.ts#L60)

## Close Comment

Closing based on a Codex review of current `main` and the shipped `v2026.4.22` tag. The duplicate-entry behavior reported here is already addressed: bundled chat channels are normalized through the channel registry, and auto-enable now writes `channels.<id>.enabled` instead of creating `plugins.entries.<id>` for bundled channels (`src/config/plugin-auto-enable.shared.ts:596`). Feishu is explicitly declared as bundled channel metadata (`extensions/feishu/package.json:27`), and the bundled-channel catalog loader reads that metadata into the chat-channel registry (`src/channels/bundled-channel-catalog-read.ts:30`). There is also regression coverage asserting that configured bundled channels do not create `plugins.entries.<id>` (`src/config/plugin-auto-enable.core.test.ts:60`). The same logic is present in release `v2026.4.22` at commit `00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`. If this still reproduces on `v2026.4.22` or newer after removing any stale `plugins.entries.feishu` config once, please open a fresh issue with the updated config snippet and gateway logs.
