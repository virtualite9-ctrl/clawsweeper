## Summary

Keep #71783 open. Current main still has an unresolved QMD collection conflict path for different-name, same path/pattern collections when QMD does not expose usable path+pattern metadata in `collection list`; OpenClaw can still skip adding the managed name and later search the missing managed collection name.

## Best Possible Solution

Keep the issue open and add a narrow memory-core repair. The best fix is to parse the conflicting collection name from QMD add errors such as `A collection already exists for this path and pattern` plus `Name: life-main (qmd://life-main/)`, verify it matches the configured path/pattern, then safely remove/recreate or otherwise rebind it to the managed name. Add regression coverage for `life` versus `life-main` and the missing-collection search retry; if automatic repair is too destructive, add a targeted `openclaw doctor` repair and a clearer diagnostic.

## Evidence
