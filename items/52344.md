## Summary

Current `main` no longer aborts Telegram doctor repair when the configured bot token SecretRef is unavailable, and the docs now explicitly state that Telegram config values override `TELEGRAM_BOT_TOKEN` env fallback. That addresses the reported doctor/update failure path and the documentation mismatch well enough to close this item.

## Risks / Open Questions

- Current main still does not honor a raw `TELEGRAM_BOT_TOKEN` env override when a non-env SecretRef (for example `exec`) is configured; the effective fix is that doctor/update no longer abort on that path and the docs now state config precedence explicitly.
