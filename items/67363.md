## Summary

Current main still promotes source-derived daily-memory snippet text directly into `MEMORY.md`. The implementation scores, filters, rehydrates, and appends candidates, but I found no distillation/summarization step, no distilled-promotion config option, and docs still describe append-style deep promotion. The related metadata-contamination report/PR is narrower and does not supersede this issue.

## Best Possible Solution

Keep this issue open for memory-core maintainer work. Decide whether deep promotion should stay raw-by-design or gain a distilled-promotion mode, then update `extensions/memory-core/src/short-term-promotion.ts` or the apply pipeline, preserve source provenance, add focused regression tests for raw versus distilled output, and clarify the public Dreaming docs.
