## Summary

Current main still promotes source-derived daily-memory snippets directly into `MEMORY.md` without a distillation/summarization mode. The contamination work related to #67442/#67601 is narrower: it filters some bad metadata/artifact shapes, but ordinary daily-log text that passes ranking is still rehydrated and appended verbatim. The prior Codex review comment on this issue remains consistent with the current checkout.

## Best Possible Solution

Keep this issue open for memory-core maintainer triage. Decide whether deep promotion should remain raw-by-design or gain a distilled-promotion mode; if distilled output is desired, add the behavior in the memory-core promotion/apply path, preserve source provenance, add focused regression tests for raw versus distilled output, update the memory-core config/schema if it is configurable, and clarify the public Dreaming docs.
