## Summary

Keep open. Current main still turns fatal isolated cron failures into `isError: true` delivery payloads and passes them through the normal announce dispatch path; the related attempted fix was closed unmerged. Some run-level errors are now formatted more generically, but the requested no-normal-announce-on-error behavior is not implemented.

## Best Possible Solution

Keep this issue open. Implement a guard in the isolated cron run or dispatch path so fatal/error run outcomes do not perform normal `delivery.mode: "announce"` fallback delivery, while keeping the error visible in cron run history. If maintainers want channel-visible failure alerts, route them only through the explicit failure-notification path with sanitized text. Add regression coverage for `delivery.mode: "announce"`, `bestEffort: false`, provider/raw error payloads, `lastRunStatus: "error"`, and no normal outbound delivery attempt.
