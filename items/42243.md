## Summary

Keep open. Current main still turns fatal isolated cron failures into `isError: true` delivery payloads and sends those payloads through the normal announce dispatch path. Some raw provider JSON is now sanitized into friendlier error text, but the requested behavior was that failed announce runs should not post normal channel messages at all. The directly related implementation PR #46236 is closed unmerged.

## Best Possible Solution

Keep this issue open. Add a guard in the isolated cron run or dispatch path so fatal/error run outcomes do not perform normal `delivery.mode: "announce"` fallback delivery, while still recording the error in cron run history, preserving backoff/error counters, and leaving explicit sanitized failure alerts as the only channel-visible failure notification path. Add regression coverage for `delivery.mode: "announce"`, `bestEffort: false`, run-level provider errors, structured `isError` payloads, `lastRunStatus: "error"`, and no outbound delivery attempt.
