## Summary

Keep open. The immediate latest-release symptom is no longer present because v2026.4.24 is now latest and has the macOS asset trio, and current appcast points at v2026.4.24. But the requested release-process fix is not proven: v2026.4.23 remains a non-draft, non-prerelease stable release with zero assets, and v2026.4.24 itself was published before its macOS assets were attached. Current docs/runbook language improves readiness checks, but the public macOS workflow is validation-only and the real upload gate lives outside this repo.

## Best Possible Solution

Keep this open until maintainers either enforce the gate in the private macOS publish/release-promotion flow or explicitly decide that the remaining v2026.4.23 historical release state is acceptable. The concrete fix should make the GitHub release non-latest/draft/prerelease until the .zip, .dmg, and .dSYM.zip assets exist, or add an automated asset-trio check immediately before any stable latest/appcast promotion.
