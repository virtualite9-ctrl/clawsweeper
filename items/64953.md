## Summary

Keep open. Current main still has the reported behavior gap: the CLI model picker paths and gateway model catalog still delegate to the full catalog loader without a `models.mode === "replace"` branch, while docs and existing helper code support configured-provider replacement semantics. The PR is a plausible focused fix, and the review discussion raised maintainer-level design questions rather than proving the work obsolete.

## Best Possible Solution

Keep #64953 or a canonical successor open. Maintainers should define the replace-mode catalog contract, then implement a shared mode-aware catalog path for `promptDefaultModel`, `promptModelAllowlist`, and `models.list`, with regression tests covering strict configured-provider replacement and any intended implicit/plugin-discovery behavior.
