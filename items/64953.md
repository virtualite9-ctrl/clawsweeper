## Summary

Current main still does not implement the requested behavior. The CLI picker and gateway catalog paths still load the full model catalog without checking `models.mode === "replace"`, while the config/docs contract says replace mode should use only configured providers. The PR remains a plausible fix path, though the exact implementation needs maintainer judgment around plugin/implicit discovery and tests.

## Best Possible Solution

Keep #64953, #64950, or a canonical successor open. Maintainers should settle the replace-mode catalog contract, then implement a shared mode-aware catalog path for `promptDefaultModel`, `promptModelAllowlist`, and gateway `models.list`, with regression tests covering strict configured-provider replacement plus any intended implicit/plugin-discovery behavior.
