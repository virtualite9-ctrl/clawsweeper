## Summary

Keep this PR open. Current main still does not implement the requested replace-mode behavior in the CLI model picker or gateway catalog, and the same-author linked bug #64950 remains open. The PR is still an implementation candidate, though maintainers should address the review concern about implicit/plugin-discovered models and add regression coverage before landing.

## Best Possible Solution

Keep this PR, #64950, or a canonical successor open. The best fix is a shared mode-aware catalog path for CLI picker and gateway models.list that honors models.mode: "replace", explicitly settles implicit/plugin-discovery semantics, preserves configured metadata, and adds regression coverage for default picker, allowlist picker, and gateway catalog results.
