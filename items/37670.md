## Summary

Keep open. Current `main` only includes part of the proposed UI change: `.cron-workspace-main` already has `min-width: 0`, but the Cron workspace still stays two-column until `1100px`, not `1400px`, while the Jobs pane still carries several wide filter controls. That means the medium-width overlap/cropping concern described by this PR is not clearly implemented on `main`. The PR also contains extra cron flush / control-ui hardlink changes that are likewise absent from current `main`, so the branch is not superseded.
