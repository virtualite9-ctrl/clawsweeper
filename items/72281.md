## Summary

Keep #72281 open. Current main still has the direct `Microsoft.PowerShell.Utility\Write-Host $msg` wrapper call that this PR changes, and the `Main` return value still feeds `Complete-Install`; the targeted regression test on main explicitly asserts `Main` must return only a boolean success value. The related installer cleanup reports are about broader failure control flow and do not supersede this focused CI fix.

## Best Possible Solution

Keep #72281 open for maintainer review. The best path is to confirm the reported Windows `core-support-boundary` failure on the affected PowerShell environment, then land this narrow `[void]` guard or an equivalent script/test adjustment so `Main` reliably returns only the installer success boolean.
