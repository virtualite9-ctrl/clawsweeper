## Summary

Keep #35447 open. Current main still records cron job ownership and stamps it on cron.add, but cron service operations and gateway cron handlers still have no caller/session context for list, update, remove, run, enqueueRun, or cron.runs visibility. The related implementation PRs in the provided context are closed unmerged, and the open #49175 is a narrower follow-up rather than a canonical replacement for the whole isolation contract.

## Best Possible Solution

Keep #35447 open and implement a core cron caller-context contract across the agent tool, gateway protocol/handlers, CronService contract, and cron service ops. Add focused regression tests proving list/listPage filtering plus update/remove/run/enqueueRun denial across agent/session boundaries, cover cron.runs name visibility, and document legacy-job plus admin override behavior.
