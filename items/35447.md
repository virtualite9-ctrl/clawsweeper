## Summary

Keep #35447 open. Current main still stores cron ownership metadata and stamps it on new cron jobs, but the cron service, gateway protocol/handlers, and agent cron tool do not carry caller context for list/update/remove/run/enqueueRun. The related implementation PRs #36517 and #49174 are closed unmerged, and #49175 is only a narrower open follow-up about cron.runs job-name leakage, not a canonical replacement for the full isolation contract.

## Best Possible Solution

Keep #35447 open and implement a core cron caller-context contract across the agent cron tool, gateway protocol/schema, gateway server handlers, CronService contract, and cron service ops. Add focused regression tests for list/listPage filtering plus update/remove/run/enqueueRun denial across agent/session boundaries, include cron.runs name visibility, and document legacy-job plus admin override behavior.
