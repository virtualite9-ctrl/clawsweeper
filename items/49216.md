## Summary

Keep open. Current `main` still has the behavior this PR fixes: deleted-archive cleanup only runs when the same save archived at least one removed live session or when `resetArchiveRetentionMs` is set, and archive timestamp parsing still only accepts `...Z` filenames. I found no equivalent implementation or test coverage for the no-live-prune cleanup path or legacy timestamp variants on `main` at `f3c37a946cc49d26e5fe65a1043f7286a86aa4a4`.

## Evidence

- **Deleted archive cleanup is still gated on archivedDirs/reset retention:** `saveSessionStoreUnlocked` only calls deleted-archive cleanup inside `if (archivedDirs.size > 0 || maintenance.resetArchiveRetentionMs != null)`, so an enforce-mode save with stale `.deleted.*` artifacts but no newly archived live sessions skips cleanup.
  - file: [src/config/sessions/store.ts:317](https://github.com/openclaw/openclaw/blob/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4/src/config/sessions/store.ts#L317)
