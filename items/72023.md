## Summary

Keep PR #72023 open. Current main still runs startup ACP identity reconciliation on any persisted pending identity with a stable runtime id, without skipping `acp.mode === "oneshot"`; the gateway still logs the reported `acp startup identity reconcile ... failed=N` warning when checked sessions remain. The PR’s earlier invalid `state === "closed"` guard was called out by Greptile and the author’s later head/comment says it was removed, leaving a focused one-shot skip that is still useful and not implemented on main.

## Best Possible Solution

Keep #72023 open and land this PR or an equivalent maintainer patch that skips `acp.mode === "oneshot"` before startup pending-identity reconciliation, with the focused regression test. Do not close it as implemented unless current main gains that explicit startup skip or another change demonstrably prevents historical/error one-shot pending records from reaching the warning path.
