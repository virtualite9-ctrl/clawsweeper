## Summary

Keep PR #72023 open. Current main still runs startup ACP identity reconciliation for every persisted session with a pending stable identity and has no `acp.mode === "oneshot"` skip, so historical or failed one-shot records can still trigger the warning from #72013. The existing close-time one-shot identity reconciliation helps future successful runs, but it does not fully obsolete this startup-path fix.

## Best Possible Solution

Keep #72023 open and land this PR or an equivalent maintainer patch that skips `acp.mode === "oneshot"` before startup pending-identity reconciliation, with the focused regression test. Do not close it as implemented unless main gains that explicit startup skip or another source-level change demonstrably suppresses historical/error one-shot pending records from the warning path.
