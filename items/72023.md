## Summary

Keep PR #72023 open. Current main still reconciles persisted pending ACP identities with stable runtime ids without skipping `mode: "oneshot"`, and the gateway still logs the `acp startup identity reconcile ... failed=N` warning when those checks fail. The PR head adds the focused one-shot skip and regression test, so it is not obsolete or superseded.

## Best Possible Solution

Keep #72023 open and land it, or an equivalent maintainer patch, that skips `session.acp.mode === "oneshot"` before startup pending-identity reconciliation and keeps the focused regression test. Do not close as implemented unless current main gains that explicit startup skip or another source-level change demonstrably prevents historical/error one-shot pending records from reaching the warning path.
