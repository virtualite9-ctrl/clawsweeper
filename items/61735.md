## Summary

Keep open. Current `main` still only enables Gemini schema cleaning from `modelProvider`, so the proxy `modelId` case in this PR is not implemented. The proposed one-line change also collides with the current cleanup flow: Gemini cleaning returns early before `modelCompat`-based keyword stripping, which matches the existing review comment about a regression risk.

## Evidence

- **Main still keys Gemini cleaning off provider name only:** `normalizeToolParameterSchema()` lowercases `options.modelProvider` and sets `isGeminiProvider` from `google`/`gemini` substrings only. There is no `modelId` check in current `main`. That means a custom OpenAI-compatible proxy provider name still will not trigger Gemini schema cleaning.
