## Summary

Keep #48558 open. Current main does not implement Anthropic's native `memory_20250818` tool or the requested context-management integration: `memory-core` exposes OpenClaw's existing `memory_search` and `memory_get` tools, while the bundled Anthropic provider wrappers handle beta headers/context1m/service-tier/fast-mode but do not inject the native memory tool or handle its CRUD command loop. Related #70049 is a closed, narrower semantic-memory write proposal and does not supersede this provider-native request; Kelisi808's comment usefully frames the remaining work as a provider capability map or generic memory adapter design question.

## Best Possible Solution

Keep this open as a focused provider/memory design tracker. The best implementation path is a maintainer-reviewed Anthropic-provider or generic memory-adapter design that gates the feature in config, injects the required Anthropic beta/tool payload, safely handles the six memory commands against a per-agent memory directory, and defines how this coexists with existing `memory-core` search/get, compaction, replay, and tool-policy behavior.
