## Summary

Keep open. Current main still documents and implements ordinary ACP follow-ups as prompt text sent directly to the bound ACP session; I did not find source, prompt, docs, or test coverage that enforces parent-side shorthand expansion before delivery. The related items in the provided context cover separate ACP binding/routing failures, not this parent-orchestration policy.

## Best Possible Solution

Keep the issue open for maintainer/product decision. A focused fix would likely scope this to parent-owned `sessions_spawn(runtime="acp", thread=true, mode="session")` continuations, update the accepted note and agent/tool guidance so the parent resolves shorthand into an explicit `sessions_send` task unless exact forwarding is requested, and add regression coverage showing shorthand follow-ups are expanded while intentionally interactive ACP-bound conversations still route directly or are clearly documented as direct.
