## Summary

Keep open. Current main still implements and documents thread-bound ACP follow-ups as direct ACP-session routing, and `sessions_send`/ACP dispatch pass the caller-supplied prompt text through to the target session rather than enforcing parent-side shorthand resolution or rewrite. The issue asks for a behavior/policy change that is not implemented on main.

## Best Possible Solution

Keep the issue open. The best next step is a maintainer decision on whether thread-bound interactive ACP should remain direct-to-harness or become parent-orchestrated by default; if the latter, update the parent prompt/tool guidance and/or ACP binding dispatch so shorthand follow-ups are rewritten into explicit child tasks unless the user asks for verbatim relay, then add regression tests for bound ACP continuation and `sessions_send` prompt text.
