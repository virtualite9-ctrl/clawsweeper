## Summary

Keep PR #63803 open. Current main still lacks the proposed `doctor:workspace-stale-paths` detector/repair path and the migration-guide warning, while runtime still resolves configured absolute workspace values and attempts to create/use them during reply dispatch. The PR addresses a real core doctor/docs bug, but should be rebased and its remaining Windows-prefix review finding addressed before merge.

## Best Possible Solution

Keep the PR open or reimplement the same core fix on main: add a focused doctor detector/repair for stale home-shaped absolute values in `agents.defaults.workspace` and per-agent `workspace`, keep non-home absolute paths manual-only, document the migration pitfall, and cover the detector with unit tests including case-insensitive Windows prefixes.
