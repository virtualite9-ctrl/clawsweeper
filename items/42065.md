## Summary

Keep this PR open. Current main partially supersedes it with a generated Control UI i18n pipeline and an Overview-card locale picker, but it still does not implement the requested topbar language switcher and still has hardcoded English in the topbar theme-mode control. The PR should be reworked against current main rather than closed as implemented or merged as-is.

## Best Possible Solution

Rebase or rewrite the PR against current main: keep scripts/control-ui-i18n.ts as the single i18n sync/check path, decide whether the locale picker should move to the topbar or remain Overview-only, localize the remaining topbar theme/search strings, and make any revived switcher await successful i18n.setLocale before saving settings. If maintainers intentionally want Overview-only placement, close this PR manually with that product rationale.
