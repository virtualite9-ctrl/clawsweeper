## Summary

Keep this PR open. Current main has a newer, broader Control UI i18n system, generated zh-CN locale coverage, and an Overview-page language picker, so much of the PR is obsolete. But the PR's explicit top-bar/global locale switcher is not implemented on main, the docs still place language selection under Overview -> Gateway Access, and no canonical follow-up tracks that remaining UI placement request.

## Best Possible Solution

Ask for a rebase/rework or make a maintainer follow-up: move or duplicate the current Overview language picker into the topbar only if maintainers want that product placement, use the existing scripts/control-ui-i18n.ts sync/check pipeline instead of hand-editing zh-CN, update docs/tests for the chosen placement, and persist settings only after locale loading succeeds. If maintainers prefer the current Overview placement, close manually with that product rationale.
