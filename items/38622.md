## Summary

Keep #38622 open. Current main still routes standard workspace bootstrap files like `SOUL.md` through the generic workspace-root boundary reader, which rejects symlink targets that resolve outside the workspace and then renders the file as `[MISSING]`. The related fix PRs in the supplied GitHub context are not merged, so the core bug remains actionable.

## Best Possible Solution

Keep the issue open and land a focused core fix in `src/agents/workspace.ts`: when a standard workspace bootstrap file is explicitly present as a symlink, follow the target for injection while preserving `MAX_WORKSPACE_BOOTSTRAP_FILE_BYTES`, dangling-symlink missing behavior, hardlink rejection, and generic boundary-file protections for other read surfaces. Add regression tests for relative and absolute symlinks to shared files, dangling symlinks, hardlinks, and the existing size limit.
