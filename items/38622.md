## Summary

Keep #38622 open. Current main still reads standard workspace bootstrap files through the boundary-file guard and treats any guard failure as missing; there is no symlink-following fallback for an in-workspace SOUL.md/AGENTS.md symlink whose target resolves outside the workspace. The linked fix PRs are not merged in the provided GitHub context, so the core bug remains actionable.

## Best Possible Solution

Keep the issue open and land a focused core fix in src/agents/workspace.ts: when a standard workspace bootstrap file is explicitly present as a symlink, follow the target for injection while preserving MAX_WORKSPACE_BOOTSTRAP_FILE_BYTES, dangling-symlink missing behavior, hardlink rejection, and the existing boundary protections for other read surfaces. Add regression tests for relative and absolute symlinks to shared files, dangling symlinks, hardlinks, and the size limit.
