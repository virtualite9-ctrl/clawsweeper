## Summary

Keep #38622 open. Current main still reads workspace bootstrap files through the generic workspace-root boundary open, rejects final symlinks whose targets resolve outside the workspace, and renders the failed read as the reported `[MISSING] Expected at: ...` Project Context marker. The related fix PRs in the provided context remain unmerged, so the core bug is still actionable.

## Best Possible Solution

Keep the issue open and land a focused core fix in `src/agents/workspace.ts`: for explicitly present standard workspace bootstrap files, follow symlink targets for injection while preserving `MAX_WORKSPACE_BOOTSTRAP_FILE_BYTES`, dangling-symlink missing behavior, hardlink rejection, and generic boundary-file behavior for other surfaces. Add regression tests for relative and absolute symlinks to shared files, dangling symlinks, hardlinks, and the existing size limit.
