## Summary

Keep #69291 open. Current main has several adjacent prompt and runtime guardrails, including `Execution Bias`, GPT-5-specific behavior contracts, strict-agentic retries, and tests for those surfaces, but it does not implement the requested system-wide Agent Behavior Principles component. The remaining product question is whether these principles should become a core prompt section for all users/models or stay as model/provider-specific overlays and runtime enforcement.

## Best Possible Solution

Keep the issue open for maintainer judgment. A good implementation would either add a compact, tested core behavior-principles section to `src/agents/system-prompt.ts` and docs, or explicitly decide that the existing `Execution Bias` plus GPT-5 overlay is the intended boundary and file narrower follow-ups for any missing runtime-enforced behaviors.

## Risks / Open Questions

- Adding broad prompt text can increase prompt size and cache churn unless it is compact and placed deliberately relative to the prompt cache boundary.
- Some requested items are better enforced mechanically, such as strict-agentic retries, tool policy, sandboxing, and tests; relying only on prompt prose may not solve the behavior defects consistently.
- The proposal affects all users and providers, so maintainer product judgment is needed before making it global rather than provider/model-specific.
