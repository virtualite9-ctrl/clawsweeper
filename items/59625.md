## Summary

Current `main` no longer applies a separate heuristic obfuscation gate to gateway/node host exec in `security=full` + `ask=off` mode. The old behavior has been replaced by explicit policy handling plus opt-in `tools.exec.strictInlineEval`, and the changelog records this in `Unreleased`.

## Evidence

- **Issue discussion identified the old bug:** The issue comments correctly narrowed the original problem to policy ordering: obfuscation matches were overriding `security: "full"`. Another comment linked PR #60709 as a proposed fix direction, which is useful context for why this was investigated.
- **Gateway exec path no longer has a generic obfuscation gate:** Current gateway host exec only adds extra approval for allowlist-plan gaps, heredocs in allowlist mode, or opt-in inline-eval hardening. `inlineEvalHit` is only computed when `strictInlineEval === true`, and `requiresAsk` is built from explicit policy plus those narrow cases.
