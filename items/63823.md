## Summary

Current main still appears to have the reported MiniMax auth-alias gap. The MiniMax manifest lacks `providerAuthAliases`, while the current auth resolver is manifest-based and runtime auth lookup goes through that resolver. The open PR #67195 remains the appropriate fix path rather than a reason to close the canonical bug report before it lands.

## Best Possible Solution

Keep #63823 open as the canonical bug report. Land #67195 or an equivalent maintainer patch that adds `providerAuthAliases` mapping `minimax-cn` to `minimax` and `minimax-portal-cn` to `minimax-portal` in `extensions/minimax/openclaw.plugin.json`, plus a focused manifest regression test like the BytePlus and Volcengine provider tests.
