## Summary

Keep PR #71072 open. Current main preserves explicit empty plugin-scope semantics, but it does not implement the proposed O(1) loader fast-path: `loadOpenClawPlugins({ onlyPluginIds: [] })` still normalizes the empty scope, checks cache/reentry, creates the runtime/registry machinery, then calls plugin discovery and manifest registry loading before filtering every candidate out. The PR is a plausible focused performance improvement, but it should include a regression test proving discovery/manifest loading are skipped and should preserve the reentry guard behavior noted in review.

## Best Possible Solution

Keep the PR open and ask for the missing regression test. The implementation direction is reasonable for core because the plugin loader owns this hot path; the final patch should short-circuit explicit empty scopes before discovery/manifest parsing, preserve `undefined` as full-load behavior, keep the `PluginLoadReentryError` guard effective, and clear/activate global plugin state consistently when `activate` is true.
