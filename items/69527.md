## Summary

Keep #69527 open. Current main still has unresolved OpenRouter auto/preset model-resolution risk: docs and config normalize the user-facing default to `openrouter/auto`, but the bundled provider catalog still defines the sentinel as bare `auto`, and the OpenRouter dynamic model hook returns the requested `ctx.modelId` unchanged. The related fix PR #69580 was closed unmerged. The reporter's fallback-observability complaint also remains: `infer model run` hard-codes `attempts: []` for both local and gateway paths.

## Best Possible Solution

Keep the issue open. The best fix is in the OpenRouter provider/runtime boundary: preserve the public config/display ref `openrouter/auto`, normalize the provider-owned runtime/API sentinel consistently, add coverage for `openrouter/auto` and `openrouter/@preset/...` resolution, and plumb `runWithModelFallback` attempts into the `infer model run --json` envelope instead of always returning an empty array.
