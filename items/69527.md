## Summary

Keep #69527 open. Current main appears to have improved OpenRouter dynamic model resolution for `openrouter/auto` and preset-style refs, but the reported CLI symptom is not fully fixed: `infer model run` still reports `attempts: []` unconditionally for both local and gateway transports, and the agent command path discards the fallback attempt list returned by the model fallback runner. That means a real OpenRouter primary failure can still surface as an OpenAI fallback with no visible primary attempt, matching the confusing output in the report. The linked PR #69580 was closed unmerged, so it does not supersede the issue.

## Best Possible Solution

Keep the issue open, but scope the fix narrowly: propagate `ModelFallbackRunResult.attempts` through `agentCommand` into the `infer model run` local and gateway envelopes, and add regression tests for `agents.defaults.model.primary: "openrouter/auto"` plus a failed OpenRouter primary with an OpenAI fallback. If maintainers want to split work, close only the dynamic OpenRouter resolution portion after creating or linking a narrower fallback-observability issue.
