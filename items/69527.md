## Summary

Keep #69527 open. Current main still leaves a material part of the report unresolved: `infer model run` returns hard-coded empty `attempts` for both local and gateway transports, so a fallback from OpenRouter to OpenAI would still be invisible. The linked PR #69580 was closed unmerged, and current code/docs still treat `openrouter/auto` as a valid OpenClaw model ref rather than retiring this report as obsolete.

## Best Possible Solution

Keep the issue open and narrow the fix to the current remaining behavior: add a regression test for `openclaw infer model run` with `agents.defaults.model.primary = "openrouter/auto"` and OpenAI fallback, propagate fallback attempts into the JSON envelope for both local and gateway transports, and clean up the `models status` display so `openrouter/auto` is not shown as `openrouter/openrouter/auto`. If maintainers decide `openrouter/openrouter/auto` is the canonical form, update OpenClaw docs/onboarding consistently instead of silently accepting both with confusing output.
