## Summary

Keep open. Current main implements important Web UI chat responsiveness pieces, including optimistic user echo, non-blocking `chat.send` acknowledgement, and streamed `chat` deltas. But #71631 is broader: gateway configuration responsiveness, isolation of slow background RPCs, and end-to-end click/send/config timing attribution still remain real product work. The related #64004 Control UI slowness report is also still open.

## Best Possible Solution

Keep #71631 open and decompose it into actionable maintainer work: add Web UI interaction-path timing for click/send/config actions through first visible feedback, improve gateway config screens with stale-response guards and per-control/background loading states, and use #64004 plus #71630 as related but narrower follow-ups rather than closure targets.
