## Summary

Keep open. Current main has a partial cron-specific suppression path, but the general reply normalization path still strips a trailing `NO_REPLY` token and delivers the preceding text, and existing tests assert that behavior. The provided GitHub context also shows open, unmerged PR #42873 uses GitHub closing syntax for this issue, so this issue should stay open until that implementation candidate is reviewed, landed, or closed.

## Best Possible Solution

Keep this issue open. Maintainers should review or rework PR #42873 and the follow-up PR #46626 against current main, likely by centralizing a shared own-line/trailing silent-token helper and applying it consistently across reply normalization, reply directives, gateway delivery/history, cron delivery, and subagent fallback paths while preserving tests for legitimate inline `NO_REPLY` mentions and media payloads.
