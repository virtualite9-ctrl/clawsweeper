## Summary

Keep this issue open. Current main still strips a trailing mixed `NO_REPLY` token in the shared auto-reply/outbound paths and delivers the preceding text, while the shipped cron fix only covers isolated cron announce delivery. The provided GitHub context also shows open PR #42873 using closing syntax for this issue, plus follow-up PR #46626, so this should remain open until those candidates are reviewed, landed, or closed.

## Best Possible Solution

Keep this issue open. Maintainers should review or rework PR #42873 and PR #46626 against current main, ideally by adding one shared helper for own-line/trailing silent-token intent and applying it consistently in `normalizeReplyPayload`, directive/outbound delivery, gateway live/history projection, cron delivery, and subagent fallback paths, with tests preserving legitimate inline mentions, leading substantive text, exact-token policy, and media payload behavior.
