## Summary

Keep open. Current `main` has message-scoped Feishu media download and UUID-backed media storage safeguards, but that is not strong enough to close a recent privacy-sensitive misrouting report without a reproduction, raw webhook payloads, or proof that the reported mismatch no longer occurs.

## Evidence

- **Feishu media download is bound to both message and file key:** `downloadMessageResourceFeishu` calls Feishu `im.messageResource.get` with `path: { message_id, file_key }`, so the current code does not fetch inbound media by a global cache key alone.
