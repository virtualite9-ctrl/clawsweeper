## Summary

Keep #38212 open. Current main has adjacent runtime/auth hardening, but the reported helper-path parity is still not implemented or proven: the slug helper forwards the resolved provider/model directly into the embedded runner, the embedded runner still resolves that exact provider/model before auth, and `codex-cli` remains registered as a CLI backend rather than an `openai-codex` provider alias for model resolution.

## Best Possible Solution

Keep #38212 open. A maintainer-quality fix should either normalize `codex-cli` helper/embedded runs to the intended `openai-codex` model/auth route before `resolveModelAsync` and add a focused slug/helper regression, or explicitly enforce and document that legacy `codex-cli/*` primary refs are unsupported for PI helper hooks and must be migrated to canonical model refs plus runtime configuration.
