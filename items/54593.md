## Summary

Keep open. Current main still has the reported mismatch: legacy keys like `subagent:worker` are accepted as subagent keys, but `getSubagentDepth()` still counts only `:subagent:` substrings and therefore returns 0 for the legacy prefix form. The PR is a small, relevant candidate fix with targeted tests and no visible security-sensitive changes.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land the targeted helper fix after normal validation, ideally with one store-fallback regression test in addition to the proposed helper-level cases. Do not close it as obsolete: current main still has the reported legacy-prefix depth behavior.
