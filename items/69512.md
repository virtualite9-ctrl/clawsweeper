## Summary

Keep #69512 open. Current main only maps OpenClaw's YOLO exec policy to Claude CLI's `--permission-mode bypassPermissions`; it does not forward per-agent `exec-approvals.json` allowlist entries or deny off-allowlist commands for Claude Code internal exec. Existing tests and docs explicitly cover the current partial mapping, and the related #67666 fixed a Codex CLI resume sandbox flag, not Claude CLI allowlist forwarding.

## Best Possible Solution

Keep the issue open as a core/bundled Anthropic CLI backend security follow-up. The likely fix belongs around Claude CLI backend normalization/preparation and CLI runner execution: resolve the effective per-agent exec approvals policy, map it to a supported Claude Code permission mechanism or host-managed permission bridge, deny off-allowlist commands consistently, and add focused tests for allowlisted and non-allowlisted commands plus docs in CLI backends/exec approvals.
