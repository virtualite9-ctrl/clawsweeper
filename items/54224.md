## Summary

Keep open. Current `main` now covers read-only Slack history/thread retrieval through the unified `read` action and user-token read preference, but the issue asks for history/search/thread retrieval and there is still no Slack search action or `search.messages` integration on `main`.

## Evidence

- **Unified Slack read action exists:** `read` is mapped to internal `readMessages` with `channelId`, `limit`, `before`, `after`, and optional `threadId`. That covers channel history and thread reads.
  - file: [extensions/slack/src/message-action-dispatch.ts:107](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/extensions/slack/src/message-action-dispatch.ts#L107)

  - file: [extensions/slack/src/actions.ts:235](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/extensions/slack/src/actions.ts#L235)
