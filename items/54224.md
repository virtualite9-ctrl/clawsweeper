## Summary

Keep this issue open. Current main already covers Slack channel history and thread reads with user-token read preference, but Slack message search is still missing from discovery, dispatch/runtime, implementation exports, CLI shape, and user-facing docs beyond a scope mention.

## Best Possible Solution

Keep this issue open and narrow the remaining work to Slack message search. The likely maintainer path is a Slack-owned `search` message action backed by Slack `search.messages` and `search:read`, preferring or requiring a user token without enabling writes, advertised only when usable credentials exist, with updates in `extensions/slack/src/actions.ts`, `action-runtime.ts`, `message-action-dispatch.ts`, `message-actions.ts`, CLI search options, tests, and Slack docs. The existing `read`/`readMessages` path can remain the implementation for the channel history and thread-history portions.
