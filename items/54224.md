## Summary

Keep this issue open. Current main now covers the history/thread-read portions through Slack `readMessages` with user-token read preference, but the requested Slack message search action is still missing, and the related search PR was closed unmerged.

## Best Possible Solution

Keep this issue open, but narrow the remaining work to Slack message search plus any CLI/discovery polish. Current `read` already covers channel history and thread reads; the maintainer path is to add a Slack-owned `search` message action using the user token with `search:read`, expose it only when the read credential is configured, keep it behind read-only semantics when `userTokenReadOnly=true`, update `message search` so Slack is not blocked by Discord-only `--guild-id`, and add Slack action-runtime/dispatcher/docs tests. If pure user-token/no-bot-token workflows are intended, also revisit Slack message-action discovery so read/search actions are advertised when a usable user token exists.
