## Summary

Keep open. Current main still lacks this PR's Amazon Bedrock non-Claude replay-policy override: the bundled provider only spreads `ANTHROPIC_BY_MODEL_REPLAY_HOOKS`, that family does not set `applyAssistantFirstOrderingFix`, and the core sanitizer only prepends `(session bootstrap)` when that flag is mapped to `applyGoogleTurnOrdering`. The PR body includes a concrete Nova repro and targeted tests, and the existing discussion includes a positive Greptile review, so this is a plausible focused bug fix rather than obsolete cleanup.

## Best Possible Solution

Keep the PR open and review it as a focused Bedrock provider bug fix. The best code path is provider-local: add a `buildReplayPolicy` override in `extensions/amazon-bedrock/register.sync.runtime.ts` that preserves the existing Claude Bedrock Anthropic policy while enabling `applyAssistantFirstOrderingFix` for non-Claude Bedrock models, with tests in `extensions/amazon-bedrock/index.test.ts` for both Nova and Claude branches. A broader shared-family change is not clearly better because `anthropic-by-model` is reused outside Bedrock.
