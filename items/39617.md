## Summary

Keep PR #39617 open. Current main still has the unresolved Slack and Discord native/slash routing problem: those paths route with provider-startup config snapshots, while route/session-key generation depends on `session.dmScope`. Telegram has since moved to a fresh runtime config pattern, so the PR is partially superseded but not obsolete.

## Best Possible Solution

Keep PR #39617, or keep the linked #39605 bug as the canonical tracking item if maintainers prefer a fresh PR. The implementation path is to rebase the remaining fix onto `extensions/discord/src/monitor/native-command.ts` / `extensions/discord/src/monitor/native-command-route.ts` and `extensions/slack/src/monitor/slash.ts`, load the current runtime config for native/slash route resolution and configured binding readiness, preserve Telegram's existing fresh-runtime-config behavior, and add focused Slack/Discord regression tests that simulate startup config followed by a changed `session.dmScope` without provider restart.
