## Summary

Current `main` still appears to have the stale-config bug in Slack and Discord native/slash command routing, even though Telegram now reloads config per command. The PR should stay open, but it likely needs a refresh because the code moved under `extensions/` and part of the fix has already landed for Telegram.

## Evidence

- **Routing still depends on the passed config snapshot:** `resolveAgentRoute` computes DM routing from `input.cfg.session?.dmScope`, so passing a startup snapshot will keep using stale DM scope settings.
  - file: [src/routing/resolve-route.ts:623](https://github.com/openclaw/openclaw/blob/cb2c36b04975e34ab7014a11575d6589aa96ae32/src/routing/resolve-route.ts#L623)
  - command: `nl -ba src/routing/resolve-route.ts | sed -n '618,670p'`
  - sha: [cb2c36b04975](https://github.com/openclaw/openclaw/commit/cb2c36b04975e34ab7014a11575d6589aa96ae32)
- **Slack slash commands still route with startup `ctx.cfg`:** Slack monitor context is created once with `cfg`, then `registerSlackMonitorSlashCommands` captures `const cfg = ctx.cfg` and later calls `resolveAgentRoute({ cfg, ... })` inside the slash handler without reloading config.
  - file: [extensions/slack/src/monitor/slash.ts:296](https://github.com/openclaw/openclaw/blob/cb2c36b04975e34ab7014a11575d6589aa96ae32/extensions/slack/src/monitor/slash.ts#L296)
