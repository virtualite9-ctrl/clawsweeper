## Summary

Keep PR #72124 open. Current main still rejects Anthropic `sk-ant-api03-` values on the `models auth paste-token --provider anthropic` path, and the PR is a focused, recent bug-fix attempt for linked issue #72121 rather than obsolete or duplicate cleanup. Maintainer review should decide whether widening `paste-token` is the right product behavior versus routing API keys through the existing Anthropic API-key auth method.

## Best Possible Solution

Keep the PR open for maintainer review. The likely best product path is either to land a tested additive fix that accepts both Anthropic prefixes on the manual paste path, or adjust the implementation so standard Anthropic API keys are stored and documented through the existing `api_key` auth profile path while still giving users a clear error/recovery path from `paste-token`.
