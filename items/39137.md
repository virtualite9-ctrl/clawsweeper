## Summary

Keep this PR open. Current main still has the split YAML inline sequence for the `skills-python-tests` hook, so YAML parses the pre-commit dependency as `pytest>=8` plus a standalone invalid `<9` requirement. The PR is a one-file, one-line fix and is not superseded by the related closed voice-call PRs.

## Best Possible Solution

Keep this PR open for maintainer review and land it, or apply the same one-line `.pre-commit-config.yaml` change on main so pre-commit passes one valid `pytest>=8,<9` requirement string to pip. Do not close it as stale, duplicate, or implemented before that fix merges.
