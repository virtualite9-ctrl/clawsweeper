## Summary

Current `main` already implements the requested multi-machine/node awareness: one gateway can pair multiple nodes, track connected and paired devices, surface node metadata/capabilities, and list them via `openclaw nodes list/status`. This is also present in release `v2026.4.22`.

## Risks / Open Questions

- Current implementation covers node awareness, pairing, listing, metadata, and per-node capabilities; it does not necessarily imply automatic config sync or distributed workload orchestration across machines.

## Close Comment

Closing after a Codex review against current `main` (`d38ed083`) and the latest release tag `v2026.4.22`. OpenClaw already has built-in node awareness: `docs/nodes/index.md:10` defines companion nodes connected to one gateway, `docs/nodes/index.md:23` and `docs/cli/nodes.md:23` document pairing plus `openclaw nodes list/status`, `src/gateway/node-registry.ts:40` tracks multiple connected nodes, `src/gateway/node-catalog.ts:91` merges paired/live nodes into a known-node catalog with display name, platform, device family, IP, and capabilities, and `src/cli/nodes-cli/register.status.ts:114` renders that surfaced list. `docs/help/faq-first-run.md:692` also documents the same always-on host + secondary machine-as-node setup described here. Because the requested node-awareness behavior is already implemented on `main` and shipped in `v2026.4.22`, I’m closing this as implemented. If you want follow-up work specifically for cross-node delegation or config sync, please open a narrower issue for that behavior.
