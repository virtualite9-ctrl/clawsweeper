## Summary

Current `main` already normalizes strict OpenAI-family tool schemas to inject top-level `additionalProperties: false` and routes those normalized schemas through the OpenAI/Copilot transport paths, with regression coverage in place. The fix is on `main` but not in the latest tagged release `v2026.4.23`, so `fixedRelease` is unknown/null.

## Evidence

- **Strict OpenAI schema normalization now injects top-level additionalProperties:false:** `normalizeStrictOpenAIJsonSchema()` now recurses with depth tracking and sets `additionalProperties = false` on the top-level object schema when missing, which directly addresses the missing field described in the issue.
  - file: [src/agents/openai-tool-schema.ts:12](https://github.com/openclaw/openclaw/blob/860dad268d35a944d352b0adbf34c68aa4d3aac9/src/agents/openai-tool-schema.ts#L12)
  - sha: [860dad268d35](https://github.com/openclaw/openclaw/commit/860dad268d35a944d352b0adbf34c68aa4d3aac9)
- **Responses/completions payload builders send tools through strict normalization:** Both OpenAI Responses and completions payload builders call `normalizeOpenAIStrictToolParameters(...)` before serializing tool schemas, so native OpenAI-family routes use the normalized schema instead of the raw tool definition.
  - file: [src/agents/openai-transport-stream.ts:352](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/src/agents/openai-transport-stream.ts#L352)
  - sha: [6bc0dc8fb671](https://github.com/openclaw/openclaw/commit/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440)
- **WebSocket conversion uses the same normalization path:** The WebSocket tool conversion path also calls `normalizeOpenAIStrictToolParameters(...)`, keeping OpenAI/Copilot transport behavior aligned across HTTP and WS paths.
  - file: [src/agents/openai-ws-message-conversion.ts:277](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/src/agents/openai-ws-message-conversion.ts#L277)
  - sha: [6bc0dc8fb671](https://github.com/openclaw/openclaw/commit/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440)
- **Regression tests cover schema normalization on native OpenAI routes:** `schema-normalization-runtime-contract.test.ts` locks the normalized strict-compatible shape for OpenAI Responses and WebSocket paths, and `openai-transport-stream.test.ts` covers native OpenAI tool strictness behavior for `read`-like schemas.
  - file: [src/agents/schema-normalization-runtime-contract.test.ts:29](https://github.com/openclaw/openclaw/blob/860dad268d35a944d352b0adbf34c68aa4d3aac9/src/agents/schema-normalization-runtime-contract.test.ts#L29)
  - sha: [860dad268d35](https://github.com/openclaw/openclaw/commit/860dad268d35a944d352b0adbf34c68aa4d3aac9)
