## Summary

Keep open. Current `main` does not implement this fix: the Peak Error Hours widget still derives hourly error rates by proportionally spreading each session’s total messages/errors across hour slices, and the backend/types on `main` still lack any `hourlyMessageCounts` support. The PR also still had maintainer-requested follow-up in review.
