## Summary

Keep open. Current `main` still uses the legacy proportional-allocation path for Peak Error Hours, does not expose any `hourlyMessageCounts` data from the backend, and the PR has explicit maintainer-requested follow-up items from April 20, 2026. This is still actionable work, not conservative cleanup-close material.
