## Summary

Keep this PR open. Current main still rejects `messages.groupChat.historyLimit: 0` in both the runtime Zod schema and generated public schema, while docs and channel-specific schemas treat `0` as supported for disabling group history. The PR contains the useful config fix and regenerated schema, but it also includes unrelated `extensions/memory-core` search behavior changes with an unresolved vector-only search regression risk, so it should be split or cherry-picked rather than closed as obsolete.

## Best Possible Solution

Keep this PR open, but ask the contributor to split/drop the unrelated memory-core changes or have a maintainer cherry-pick the scoped config fix: change `GroupChatSchema.historyLimit` to `min(0)`, regenerate `src/config/schema.base.generated.ts`, and keep focused regression coverage for accepting `0` and rejecting negatives. The memory-core keyword fallback should move to a separate PR after the vector-only search concern is resolved.
