## Summary

Keep this PR open. Current main still rejects `messages.groupChat.historyLimit: 0` in the core runtime schema and generated public schema, while docs and channel/provider schemas treat zero as a supported group-history disable value. The PR contains the useful config fix, but it should not be landed as-is because its current diff also bundles unrelated memory-core search behavior with an unresolved vector-only search regression concern.

## Best Possible Solution

Keep this PR open, but ask the contributor to split/drop the unrelated memory-core changes or have a maintainer cherry-pick the scoped config fix: change `GroupChatSchema.historyLimit` to `min(0)`, regenerate `src/config/schema.base.generated.ts`, and add focused regression coverage for accepting `0` and rejecting negatives. The memory-core text-search fallback should move to a separate PR after the vector-only search concern is resolved.
