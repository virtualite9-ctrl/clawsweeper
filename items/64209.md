## Summary

Close as `clawhub`: current main deliberately keeps optional session-behavior features in the plugin layer, and the existing plugin API already covers the needed pieces for a `/soul`-style implementation (custom slash command, session-scoped state, and prompt/context mutation).

## Evidence

- **Vision keeps optional capability out of core:** `VISION.md` says core should stay lean, optional capability should usually ship as plugins, and plugin API should be extended instead of adding one-off core behavior.
  - file: [VISION.md:54](https://github.com/openclaw/openclaw/blob/5b8bd6371c666a5de43c82305e0315771e8e49d5/VISION.md#L54)
  - command: `nl -ba VISION.md | sed -n '36,70p'`
  - sha: [5b8bd6371c66](https://github.com/openclaw/openclaw/commit/5b8bd6371c666a5de43c82305e0315771e8e49d5)
- **Plugin SDK already supports custom slash commands:** The public SDK documents `api.registerCommand(def)` specifically for custom commands that bypass the LLM.
  - file: [docs/plugins/sdk-overview.md:86](https://github.com/openclaw/openclaw/blob/5b8bd6371c666a5de43c82305e0315771e8e49d5/docs/plugins/sdk-overview.md#L86)
  - command: `nl -ba docs/plugins/sdk-overview.md | sed -n '86,100p'`
  - sha: [5b8bd6371c66](https://github.com/openclaw/openclaw/commit/5b8bd6371c666a5de43c82305e0315771e8e49d5)
- **Plugin hooks can inject prompt/context per run:** The public hooks docs expose `before_prompt_build` to add dynamic context or system-prompt text before the model call.
  - file: [docs/plugins/hooks.md:61](https://github.com/openclaw/openclaw/blob/5b8bd6371c666a5de43c82305e0315771e8e49d5/docs/plugins/hooks.md#L61)
  - command: `nl -ba docs/plugins/hooks.md | sed -n '48,96p'`
  - sha: [5b8bd6371c66](https://github.com/openclaw/openclaw/commit/5b8bd6371c666a5de43c82305e0315771e8e49d5)
- **Plugin command context is session-aware:** `PluginCommandContext` includes `sessionKey`, `sessionId`, and `sessionFile`, which is the exact shape needed for a command that toggles behavior for only the active conversation.
  - file: [src/plugins/types.ts:1785](https://github.com/openclaw/openclaw/blob/5b8bd6371c666a5de43c82305e0315771e8e49d5/src/plugins/types.ts#L1785)
  - command: `nl -ba src/plugins/types.ts | sed -n '1785,1815p'`
  - sha: [5b8bd6371c66](https://github.com/openclaw/openclaw/commit/5b8bd6371c666a5de43c82305e0315771e8e49d5)
- **Hook context includes workspace and session identity:** `PluginHookAgentContext` includes `sessionKey`, `sessionId`, and `workspaceDir`, so a plugin can resolve profile files from the workspace and apply them to the active session during prompt build.
  - file: [src/plugins/hook-types.ts:155](https://github.com/openclaw/openclaw/blob/5b8bd6371c666a5de43c82305e0315771e8e49d5/src/plugins/hook-types.ts#L155)
  - command: `nl -ba src/plugins/hook-types.ts | sed -n '155,175p'`
  - sha: [5b8bd6371c66](https://github.com/openclaw/openclaw/commit/5b8bd6371c666a5de43c82305e0315771e8e49d5)
- **Existing plugin already proves session-scoped command + prompt-hook pattern:** The bundled Active Memory plugin reads `ctx.sessionKey`/`ctx.sessionId` to toggle behavior for 'this session' and then participates in `before_prompt_build`, showing this is an existing extension pattern rather than missing core infrastructure.
  - file: [extensions/active-memory/index.ts:1928](https://github.com/openclaw/openclaw/blob/5b8bd6371c666a5de43c82305e0315771e8e49d5/extensions/active-memory/index.ts#L1928)
  - command: `nl -ba extensions/active-memory/index.ts | sed -n '1920,2015p'`
  - sha: [5b8bd6371c66](https://github.com/openclaw/openclaw/commit/5b8bd6371c666a5de43c82305e0315771e8e49d5)
- **If full context ownership is needed, plugins can own context assembly:** The public context-engine docs show plugins can register a context engine that owns session context assembly and returns `systemPromptAddition`, which is sufficient for a full alternate-SOUL implementation without adding a core slash command.
  - file: [docs/concepts/context-engine.md:118](https://github.com/openclaw/openclaw/blob/5b8bd6371c666a5de43c82305e0315771e8e49d5/docs/concepts/context-engine.md#L118)
  - command: `nl -ba docs/concepts/context-engine.md | sed -n '118,165p'`

## Risks / Open Questions

- No already-posted ClawHub plugin or extension was linked in the issue discussion, so this decision rests on current plugin-scope architecture rather than an existing published implementation.
- If maintainers later decide persona switching must be first-party and standardized across all channels, that would be a product-scope change rather than evidence that core lacks the necessary extension seams today.
