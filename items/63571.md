## Summary

Keep open. Current main still lacks explicit `tools.web.search.fallbacks` support: `runWebSearch` only retries across providers when selection is not explicit, tests lock fail-fast behavior for configured and caller-selected providers, and the config schema/docs do not expose a fallback list. The PR is a focused core-runtime/config feature, not a duplicate or ClawHub-only plugin request.

## Best Possible Solution

Keep this PR open or ask the author to rebase and complete validation. The best product direction is a generic core `tools.web.search.fallbacks` config implemented in `src/web-search/runtime.ts`, config types/Zod/generated schema/help/docs, and focused tests that preserve existing auto-detect fallback and explicit `providerId` fail-fast behavior while adding ordered fallback retry for config-driven explicit providers.
