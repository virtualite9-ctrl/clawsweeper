## Summary

Keep PR #36967 open. Current main has superseded some broad pieces of the branch, especially generic security-fix handling and open-ended hook channel config, but the central Feishu work is still not implemented: explicit `/allowlist channel=feishu` is not plugin-aware, Feishu does not expose a `/allowlist` config adapter, and Feishu is not marked markdown-capable in channel metadata.

## Best Possible Solution

Keep PR #36967 open for a narrower maintainer/contributor rework. The best path is to wire Feishu to the existing plugin allowlist adapter with merged-account semantics, make `/allowlist channel=feishu` resolve plugin channel IDs safely, add `markdownCapable` to Feishu channel metadata/package metadata, add focused Feishu allowlist/markdown/envelope regression tests, and drop the security-fix and hook-schema chunks already handled generically on main.
