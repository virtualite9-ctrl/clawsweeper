## Summary

Keep PR #36967 open. Current main has superseded some broad pieces of the branch, especially generic security-fix handling and open-ended hook channel config, but the central Feishu work is still missing: explicit `/allowlist channel=feishu` is not plugin-aware, Feishu does not expose a config-backed `/allowlist` adapter, and Feishu is not marked markdown-capable in channel metadata/package metadata. The PR branch itself still needs rework because existing review comments identify Feishu account/default-account read-write and inheritance risks.

## Best Possible Solution

Keep PR #36967 open for a narrower maintainer/contributor rework: make `/allowlist channel=feishu` use plugin-aware channel resolution safely, add a Feishu-owned `allowlist` adapter using `buildDmGroupAccountAllowlistAdapter` or equivalent merged-account semantics, mark Feishu metadata as `markdownCapable`, add focused Feishu allowlist/markdown/envelope regression tests, and drop the security-fix plus hook-schema chunks already handled generically on main.
