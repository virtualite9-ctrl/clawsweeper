## Summary

Keep PR #36967 open. Current main has superseded some of the broad supporting chunks, especially generic security-fix handling and open-ended hook channel config, but the core Feishu request is still not implemented: the Feishu channel does not expose a config-backed /allowlist adapter, and Feishu is not marked markdown-capable in current channel metadata/package metadata. The branch also has unresolved review concerns around account/default-account and inherited allowlist semantics, so it should be narrowed and reworked rather than closed or merged as-is.

## Best Possible Solution

Keep PR #36967 open for a narrower rework: add a Feishu-owned allowlist adapter using buildDmGroupAccountAllowlistAdapter or an equivalent that reads merged Feishu account config and writes without dropping inherited allowlist entries; mark Feishu metadata/package metadata as markdownCapable; add focused Feishu /allowlist channel=feishu, markdown capability, and envelope regression tests; remove the security-fix and hook-schema chunks already superseded on main; address the existing PR review comments before merge.
