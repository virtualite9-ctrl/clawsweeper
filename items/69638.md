## Summary

Keep PR #69638 open. Current main still uses only `connectAuth.password` for `gateway.auth.mode === "password"`, so a Rabbit/Funnel client that sends the shared secret only in `connectAuth.token` would still hit `password_missing`. The PR’s implementation and follow-up test additions remain relevant and are not already shipped on main.

## Best Possible Solution

Keep the PR open and either land it after normal maintainer review/CI or port the same narrow behavior to main: in password mode, accept `connectAuth.token` only when `connectAuth.password` is absent, while preserving mismatch rejection and password-over-token priority with the regression tests from the PR.
