## Summary

Keep open. Current `main` and the latest shipped release still reject password-mode WebSocket handshakes that only provide `connectAuth.token`, so this PR is neither already implemented nor obsolete. The PR also appears to address the only review feedback by adding the missing edge-case tests.

## Evidence

- **Current password-mode auth still ignores `connectAuth.token`:** On current `main`, password mode reads only `connectAuth?.password`, then returns `password_missing` or `password_mismatch` if that field is absent or wrong. There is no fallback to `connectAuth?.token` in the WebSocket/shared-secret path.

- **Current regression tests do not cover token-only password-mode handshakes:** `main` tests password mode for missing credentials and wrong `connectAuth.password`, but there is no coverage for a token-only handshake succeeding in password mode, nor the added Rabbit-specific edge cases from the PR.
