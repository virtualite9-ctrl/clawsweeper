## Summary

Keep PR #69638 open. Current main still reads only `connectAuth.password` in gateway password mode, so Rabbit/Funnel clients that send the shared secret via `connectAuth.token` would still get `password_missing`. The PR is the canonical active fix for closed issue #51953 and its latest diff includes the happy path plus the two edge-case tests requested by Greptile.

## Best Possible Solution

Keep the PR open and have maintainers land or rebase the narrow gateway fix after normal validation. The best product/code outcome is to preserve password-mode semantics while allowing `connectAuth.token` as a fallback credential only in that branch, with the PR's regression tests for Rabbit/Funnel token-only, mismatch, and password-priority behavior.
