## Summary

Keep this PR open. Current main has exact/JSON/mixed NO_REPLY handling and other user-facing sanitizers, but it does not implement the PR’s channel-agnostic guard for plain internal route-marker/tool-JSON leakage such as NO_REPLY followed by assistant to=functions.exec and tool-style JSON. The PR still appears to address an unmerged shared auto-reply gap, not an obsolete or duplicated change.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to rebase it onto current main, review the heuristic boundaries carefully, and either land the shared normalization guard with its regression tests or adapt the same behavior into the existing sanitizer/normalizer stack. Closing now would drop a still-unimplemented protection for malformed internal orchestration leaking into user-visible replies.
