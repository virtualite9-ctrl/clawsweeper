## Summary

Keep this PR open. Current main still has no shared suspicious-orchestration leakage detector in the auto-reply normalizer or outbound payload planner; it handles exact/JSON/mixed NO_REPLY and existing sanitizer cases, but route-marker/tool-JSON text such as assistant/commentary to=functions.exec remains outside the implemented protections.

## Best Possible Solution

Keep this PR open for maintainer review. Rebase it onto current main, then either land the shared normalizeReplyPayload guard with the focused regression tests or adapt the same narrow route-marker/tool-JSON protection into the current sanitizer/outbound normalization stack without weakening existing NO_REPLY, media/channelData, direct-message rewrite, and explanatory-prose behavior.
