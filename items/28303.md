## Summary

Keep open. Current main still uses `zh-CN` and `zh-TW` as the Control UI runtime locale IDs, generated locale metadata, persisted setting values, selector values, docs, workflow matrix, and tests. The only `zh-Hans` usage found is docs-publish metadata that still maps to `zh-CN` output paths, so it does not implement the requested runtime rename or migration.

## Best Possible Solution

Keep #28303 open for a focused Control UI i18n change: rename the runtime locale contract to `zh-Hans`/`zh-Hant`, update lazy imports, generated locale metadata, workflow/docs matrices, selector language keys, and tests, and add a saved-setting migration from `zh-CN`/`zh-TW`. If maintainers decide the region IDs should stay for compatibility, document that decision on the issue instead of closing it as implemented.
