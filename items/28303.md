## Summary

Keep #28303 open. Current main still exposes `zh-CN` and `zh-TW` as the Control UI runtime locale contract, lazy import keys, generated locale metadata, persisted browser-storage values, selector option values, docs, workflow matrix, and tests. The only `zh-Hans` references are in docs-publish/Mintlify metadata, not the requested Control UI runtime rename or saved-locale migration.

## Best Possible Solution

Keep #28303 open for a focused Control UI i18n contract change: rename runtime locale IDs and generated metadata to `zh-Hans`/`zh-Hant`, update lazy imports, selector language keys, docs, workflow, and tests, and add saved-setting migration from `zh-CN`/`zh-TW`. If maintainers choose to preserve the old IDs as stable API, document that decision on the issue and close with that rationale.
