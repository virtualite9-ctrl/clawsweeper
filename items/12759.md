## Summary

Keep #12759 open. Current main still only defines individual `channel:*` labels, the Labeler workflow applies those rules directly without channel-label collapse logic, and `scripts/sync-labels.ts` does not create a `channel: ALL` or `channel: NEW` catch-all label. The related PR #71548 was closed unmerged, and the latest issue discussion is actively converging on a revised repo-side implementation.

## Best Possible Solution

Keep the issue open until repo-controlled labeling implements the catch-all behavior: add the maintainer-approved label and metadata, update `.github/labeler.yml` and `scripts/sync-labels.ts`, add channel-label collapse/removal logic in `.github/workflows/labeler.yml` or the relevant agent labeling path, and verify with a representative PR that touches multiple channel/plugin paths.
