## Summary

Keep this issue open. The maintainer discussion explicitly says not to close until isolated-session coverage exists, and current main still has Playwright-required browser routes that can return the reported unavailable-runtime error. I found routing and diagnostics work for host, sandbox, node, and existing-session profiles, but no isolated sub-agent regression proving browser navigate or snapshot works through the reported path.

## Best Possible Solution

Keep this issue open. The best fix is to make the isolated sub-agent browser path use the same Playwright-capable runtime as the main gateway, or explicitly delegate browser actions to the host/node browser owner when that is the intended architecture. Add a regression test that exercises browser navigate or snapshot through the real isolated sub-agent path and proves it does not return the Playwright-unavailable error when the main gateway or browser node supports Playwright.
