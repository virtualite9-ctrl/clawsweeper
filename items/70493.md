## Summary

Keep #70493 open. The issue has an explicit maintainer comment to keep it open until isolated-session browser coverage exists, and current main still has the same Playwright-required route behavior for navigate/act/full AI snapshot paths. I found browser plugin wiring and sandbox bridge support, but no isolated sub-agent regression proving the reported path no longer returns the Playwright-unavailable error.

## Best Possible Solution

Keep the issue open and fix the isolated/sub-agent browser runtime path directly. The best fix is to make isolated sessions either load the same browser plugin Playwright runtime as the main gateway or explicitly delegate Playwright-dependent actions to the node/host that owns browser control, then add regression coverage for an isolated session calling browser snapshot or navigate without returning the Playwright-unavailable error.
