## Summary

Keep this PR open. Current main has absorbed some Microsoft Teams media fixes, but it still lacks the PR's channel `/chats/{threadId}` fallback, still has unresolved DM media coverage gaps tied to #65329, and the PR branch needs maintainer cleanup for the tenant token/provider-contract concern before it can be merged safely.

## Best Possible Solution

Keep this PR open for a maintainer-cleaned follow-up: preserve the already-shipped exact SMBA auth behavior, use `readResponseWithLimit` or an equivalent bounded read for hosted-content `$value`, add or explicitly reject the channel `/chats/{threadId}` fallback with tests, and coordinate the remaining DM media behavior with #65329. Do not merge the tenant-token overload without adding an explicit supported provider capability.
