## Summary

Keep open. Current main still emits the exact `ready for user delivery` wording in the subagent completion announcement path, tests still assert that wording, and the requested `available for review` plus explicit final-state verification instruction is absent. Related #44472 and #44587 were duplicate cleanup around the same remaining source issue and point back to #44471 as the canonical tracker.

## Best Possible Solution

Keep #44471 open as the canonical source-level bug. The likely fix belongs in `src/agents/subagent-announce.ts` and matching tests/docs: change the completion handoff wording away from premature user-delivery language and add explicit guidance that subagent text is input material until the requester verifies whether the external target was actually applied.
