## Summary

Keep this PR open. Current main does not contain the PR's broad claw-code compatibility bridge work, the open same-author PR #67333 is paired with part of the stack, and the PR branch still needs maintainer security/product review before any conservative cleanup action would be appropriate.

## Best Possible Solution

Keep this PR open for maintainer review, but ask for a split or restack. A practical path is separate PRs for runner/tool-call normalization, MCP resource/auth support, LSP dispatch extensions, sandbox/parity-harness work with #67333, and any Hermes/task/team/product surfaces maintainers actually want in core. Compatibility utility tools should either move to plugin/ClawHub surfaces or land only with guarded fetch, redacted/allowlisted config reads, owner-only persisted mutations, bounded waits, and schema patterns that match repo guardrails.
