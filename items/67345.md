## Summary

Keep this PR open. Current main does not implement the broad claw-code parity bridge work, the PR is paired with another open same-author PR, and the PR diff still has unresolved security and correctness findings that require maintainer review rather than conservative cleanup closure.

## Best Possible Solution

Keep the PR open for maintainer review, but request a split or restack. A practical path is separate PRs for runner/tool-call normalization, MCP resource/auth support, LSP dispatch extensions, sandbox/parity-harness work with #67333, and any task/team/Hermes/product surfaces maintainers actually want in core. The compat utility tools need safe defaults first: guarded fetch, redacted/allowlisted config access, owner-only persisted mutations, bounded sleeps, and schema patterns that match repo guardrails.
