## Summary

Keep open. Current main still preserves JSON-stringified chat content-block arrays as ordinary strings, so the vLLM double-serialization bug described by the linked report is not fixed on main or in the latest release. The PR is still useful, but its current type-only detector needs a more conservative chat-block shape check before it is safe to merge.

## Best Possible Solution

Keep this PR open and update it against current main. The best fix is a shared conservative parser that only treats a stringified JSON array as chat content when it has a recognizable chat-content-block shape, extracts text consistently with the real-array path, preserves arbitrary JSON strings, and adds focused regression tests for vLLM text blocks, mixed text/image blocks, invalid JSON, and non-chat typed JSON arrays.

## Risks / Open Questions

- Merging the PR patch as currently described could drop non-chat typed JSON arrays instead of preserving them as plain string content.
