## Summary

Current `main` does not implement this PR’s behavior. `src/shared/chat-content.ts` still treats any string input as plain text and never attempts to parse JSON-stringified content-block arrays, so the vLLM/openai-completions case described in the PR remains unhandled. The new JSON-string coverage from the PR is also absent from `src/shared/chat-content.test.ts`.
