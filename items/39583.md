## Summary

Keep open. Current `main` still preserves JSON-stringified chat content-block arrays as ordinary string content, so the vLLM double-serialization problem linked by this PR is not implemented on main or in the latest release. The PR remains useful, but its current head still needs a narrower chat-block shape check so arbitrary typed JSON strings are not dropped.

## Best Possible Solution

Keep this PR open and update it against current `main`. The best fix is a conservative shared parser in `src/shared/chat-content.ts` that only treats a stringified JSON array as chat content when it has a recognizable chat content-block shape, extracts text consistently with the real-array path, preserves arbitrary JSON strings, and adds focused regression tests in `src/shared/chat-content.test.ts` for vLLM text blocks, mixed text/image blocks, invalid JSON, and non-chat typed JSON arrays.

## Risks / Open Questions

- Current main can still carry raw JSON-stringified vLLM content blocks through prompt, display, and preview-style extraction paths.

- Regression coverage is still needed for JSON-stringified text blocks, mixed text/media blocks, invalid JSON fallback, and non-chat typed JSON arrays.
