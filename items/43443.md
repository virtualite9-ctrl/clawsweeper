## Summary

Keep this PR open. Current main has partially moved the launchd restart surface to kickstart-first behavior and has narrower already-loaded bootstrap handling, so the PR is stale as-is, but the central transient `launchctl bootstrap` EIO recovery is still not implemented across install, doctor repair, restart fallback, and detached handoff paths.

## Best Possible Solution

Keep this PR open for maintainer review. Rebase it onto current main and rescope it around a shared bounded EIO-only `launchctl bootstrap` retry used by `bootstrapLaunchAgentOrThrow`, `repairLaunchAgentBootstrap`, install activation, restart fallback recovery, and detached handoff bootstrap paths. Separately decide whether PID SIGKILL escalation is still needed under the current restart model; if retained, add stronger PID identity protection and focused tests for retry success/exhaustion, errno 125 and non-EIO non-retry behavior, doctor repair, install activation, restart fallback, and handoff scripts.
