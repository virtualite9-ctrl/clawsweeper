## Summary

Keep open. Current `main` still has the exact coarse shared-hook fallback that PR #55761 proposes to replace, and the new hook announcement policy/type/test coverage from the PR is not present in the checkout.

## Evidence

- **Gateway hook fallback still keys only on `!result.delivered`:** `dispatchAgentHook` still builds `delivery` as `{ mode: "none" }` when `value.deliver` is false, then unconditionally enqueues a main-session system event whenever `!result.delivered`.
  - file: [src/gateway/server/hooks.ts:49](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/src/gateway/server/hooks.ts#L49)
  - command: `nl -ba src/gateway/server/hooks.ts | sed -n '35,120p'`
- **Silent hook mode still resolves to non-requested delivery:** `resolveCronDeliveryPlan` keeps `mode: "none"` with `requested: false`, so a successful `deliver:false` hook can naturally return without direct delivery while still hitting the gateway's `!result.delivered` fallback.
  - file: [src/cron/delivery-plan.ts:29](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/src/cron/delivery-plan.ts#L29)
