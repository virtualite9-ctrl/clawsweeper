## Summary

Keep PR #69495 open. Current main still does not implement heartbeat-specific model fallbacks, and the PR targets a real open product gap rather than obsolete or duplicate work. It should not be auto-closed, but the submitted patch needs maintainer/author follow-up before merge because review comments identify an empty-fallback override bug and a defensive caller path that can drop object-form fallbacks.

## Best Possible Solution

Keep the PR open and have the author update it rather than close it. The right product path is to make `agents.*.heartbeat.model` use the same `AgentModelConfig` shape as normal agent models, thread heartbeat fallbacks through the reply/fallback pipeline as a per-run override including explicit empty arrays, cover both defaults and per-agent heartbeat config with tests, update generated schema/docs/help text, and merge only after the changed gates are green.
