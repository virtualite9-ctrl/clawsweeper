## Summary

Keep #69495 open. Current main still treats `agents.*.heartbeat.model` as a string-only heartbeat override in schema, public types, generated config schema, docs, and runtime forwarding, so the central feature from #69434/#66232 is not implemented on main. The PR is a focused implementation vehicle, but it needs follow-up for the reviewed empty `fallbacks: []` semantics and related fallback plumbing before merge rather than conservative cleanup closure.

## Best Possible Solution

Keep the PR open as the concrete implementation path for heartbeat-specific model fallbacks. Update it so `heartbeat.model` uses `AgentModelConfig`, heartbeat runner passes primary plus a defined fallback override whenever the object form includes `fallbacks` (including `[]`), the reply/followup pipeline prefers that per-run override, docs/generated schema/pricing collection stay aligned, and regression tests cover object form, inherited defaults, and explicit empty fallback disabling.
