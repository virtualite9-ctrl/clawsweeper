## Summary

Current main now covers the reported `openclaw config set` stack overflow path. The CLI still enables console capture and `config set` can still emit disabled-plugin `Config warnings`, but the recursive edge is gone: file logger settings no longer fall back to a full config load, and the remaining console-settings fallback has an explicit re-entrancy guard with regression coverage. The related #56715 review correctly identified the console-settings guard; current main also removes the file-logger config-load path that had kept this specific config-set variant actionable.

## Best Possible Solution

Close this issue as implemented on current main. Keep the raw logging-config reader and console-settings re-entrancy guard in place; a future hardening follow-up could add an exact end-to-end `config set` regression test, but the remaining source-level recursion path described by the report is gone.
