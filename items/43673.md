## Summary

Keep open. Current main has adjacent multi-agent routing, per-agent tool policies, group `toolsBySender`, and per-agent workspace primitives, but it still does not implement the requested first-class org/team deployment package: no team workspace scaffold CLI, no declarative `agents.yaml`-style deployment manifest, and no complete per-session/operator-scope RBAC layer. The focused RBAC work is still represented by open #49183 and the open PRs #53496/#53530 mentioned in the discussion.

## Best Possible Solution

Keep this open as the umbrella product/design tracker unless maintainers split it. The practical path is to review or close the focused RBAC RFC/PRs (#49183, #53496, #53530), then decide whether workspace scaffolding and deployment manifests belong in core CLI/config, a documented plugin API, or a ClawHub/plugin pattern.

## Risks / Open Questions

- The issue is broad and overlaps narrower open work; maintainers may prefer to split it into canonical trackers rather than keep one umbrella indefinitely.
- RBAC and per-session tool restrictions are security-sensitive; existing per-agent, group, and cron/session visibility controls should not be treated as complete org/team RBAC.
- Some parts may ultimately belong outside core as plugin or ClawHub work, but the missing RBAC/config/CLI surfaces need maintainer product judgment before this can close.
