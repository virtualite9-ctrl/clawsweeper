## Summary

Keep open. Current main has adjacent primitives for multi-agent routing, per-agent workspaces, per-agent tool policies, and group `toolsBySender`, but it does not implement the requested first-class org/team deployment package. There is no `openclaw workspace init --template team` or `agents.yaml`/deployment-manifest surface, and the per-session/RBAC portion is still separately tracked by open RFC #49183 and narrower open PRs #53496/#53530.

## Best Possible Solution

Keep this open as the umbrella product/design tracker until maintainers either split it into canonical narrower issues or explicitly scope it down. The best path is to review/land or close the focused RBAC work in #49183/#53496/#53530, then decide whether team workspace scaffolding and declarative deployment manifests belong in core CLI/config, a plugin API extension, or a documented external deployment pattern.
