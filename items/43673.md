## Summary

Keep open. Current main has adjacent primitives for multiple isolated agents, per-agent workspaces/config, global/per-agent tool policy, session-tool visibility limits, and group/channel sender-scoped tool restrictions, but it still does not implement the central first-class org/team deployment request: no team workspace scaffold CLI, no declarative agents.yaml-style deployment manifest, and no complete per-session/operator-scope RBAC layer. The related #49183, #53496, and #53530 context remains narrower incremental RBAC work, not a full replacement for this umbrella request.

## Best Possible Solution

Keep this open as the umbrella product/design tracker unless maintainers split it. The practical path is to review or close the focused RBAC work in #49183, #53496, and #53530, then decide whether workspace scaffolding and deployment manifests belong in core CLI/config, a documented plugin API, or a ClawHub/plugin pattern.

## Risks / Open Questions

- Some scaffold or manifest pieces may ultimately belong in a plugin or ClawHub pattern, but the missing RBAC/config/CLI surfaces still need maintainer product judgment before this can close.
