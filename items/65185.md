## Summary

Keep this PR open. Current main still has the core iOS manual-LAN TLS behavior and mixed-auth precedence problem that the PR is trying to address, so it is not obsolete or implemented. The submitted diff is a plausible implementation candidate for #47887, but it still needs changes before merge because the PR test diff has review-identified compile/test issues and the Tailscale plaintext policy needs alignment with current docs/security guidance.

## Best Possible Solution

Keep this PR open as a focused implementation candidate for #47887, but request changes before merge: move the new Swift tests inside `GatewayConnectionSecurityTests`, update the stale `.local` assertion, align iOS manual TLS behavior with the existing shared local-network helper and current Tailscale/public security policy, and make password-over-bootstrap precedence consistent in Swift and TypeScript or split the TypeScript half into a clearly linked follow-up.
