## Summary

Keep this PR open. Current main still forces manual iOS LAN hosts such as `.local` through TLS and still prefers `bootstrapToken` over an explicit password in Swift and TypeScript auth assembly. The PR is a plausible implementation candidate for #47887, but it needs changes before merge because the submitted test diff is known to break the iOS test target and the `.ts.net` plaintext behavior conflicts with current mobile/Tailscale security guidance.

## Best Possible Solution

Keep this PR open as a focused implementation candidate for #47887, but request changes before merge: move the new Swift tests inside `GatewayConnectionSecurityTests`, update the stale `.local` assertion, align iOS manual TLS behavior with the existing shared local-network helper and current Tailscale/public security policy, and either make password-over-bootstrap precedence consistent in Swift and TypeScript or split the TypeScript half into a clearly linked follow-up.
