## Summary

Keep open. Current main has clearer memory docs and a real `memory.backend` switch, so the original 2026.3.8 confusion is partly addressed. But the central request is not fully implemented: memory behavior is still split across `memory.backend`, `agents.*.memorySearch`, `agents.defaults.compaction.memoryFlush`, and `plugins.slots.memory`; the builtin SQLite store still defaults to a per-agent state path rather than a per-workspace path; and the related memory PRs cited in discussion were closed unmerged or do not cover this full design request.

## Best Possible Solution

Keep this open as a memory product/config design issue. The best path is an additive, documented policy layer that makes memory behavior predictable across teams: one clear mode or equivalent preset that coordinates active memory plugin selection, memory search enablement, pre-compaction flush, backend choice, QMD fallback/include behavior, and index store location. It should also decide whether builtin store paths need a workspace token/default and add doctor/config docs so old installs can converge without silent behavior drift.
