## Summary

Keep open. Current `main` still removes channel config in the configure wizard by deleting the raw `channels[channel]` entry, not by routing through the channel plugin `deleteAccount` and `onAccountRemoved` contract that the PR addresses.

## Evidence

- **Wizard still does raw JSON deletion:** `removeChannelConfigWizard()` confirms the removal, then deletes `next.channels[channel]` directly and never loads the channel plugin or calls plugin removal hooks.
  - file: [src/commands/configure.channels.ts:107](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/commands/configure.channels.ts#L107)
  - command: `nl -ba src/commands/configure.channels.ts | sed -n '107,126p'`
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **Dedicated remove command uses plugin-aware deletion:** `channels remove --delete` still calls `plugin.config.deleteAccount(...)` and then `plugin.lifecycle?.onAccountRemoved?.(...)`, so the configure wizard is behaviorally different from the supported removal path.
  - file: [src/commands/channels/remove.ts:140](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/commands/channels/remove.ts#L140)
  - command: `nl -ba src/commands/channels/remove.ts | sed -n '140,154p'`
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **Plugin contract still has nontrivial delete semantics:** Hybrid channel adapters can preserve a section on default-account delete instead of simply removing the whole channel block, so bypassing `deleteAccount` is not equivalent to the plugin contract.
  - file: [src/plugin-sdk/channel-config-helpers.ts:534](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/plugin-sdk/channel-config-helpers.ts#L534)

- **Real plugins still rely on removal hooks:** Telegram still implements `onAccountRemoved` cleanup, which the configure wizard currently skips entirely.
  - file: [extensions/telegram/src/channel.ts:733](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/extensions/telegram/src/channel.ts#L733)

  - file: [src/commands/configure.channels.test.ts:103](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/commands/configure.channels.test.ts#L103)
