## Summary

Current `main` does not implement optional per-message timestamps in the TUI chat log. The TUI settings UI has no timestamp toggle, and the history/rendering path drops message timestamps instead of displaying them.

## Evidence

- **Settings UI has no timestamp option:** `openSettings()` defines only `Tool output` and `Show thinking`; there is no timestamp-related setting or `/settings` toggle.
  - file: [src/tui/tui-command-handlers.ts:211](https://github.com/openclaw/openclaw/blob/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4/src/tui/tui-command-handlers.ts#L211)
  - command: `nl -ba src/tui/tui-command-handlers.ts | sed -n '180,255p'`
  - sha: [f3c37a946cc4](https://github.com/openclaw/openclaw/commit/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4)
- **History rendering ignores message timestamps:** When chat history is loaded, user messages are rendered with `chatLog.addUser(text)` and assistant messages with `chatLog.finalizeAssistant(text)`. The `message.timestamp` field is never passed through for display.
  - file: [src/tui/tui-session-actions.ts:311](https://github.com/openclaw/openclaw/blob/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4/src/tui/tui-session-actions.ts#L311)
  - command: `nl -ba src/tui/tui-session-actions.ts | sed -n '288,360p'`
  - sha: [f3c37a946cc4](https://github.com/openclaw/openclaw/commit/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4)
- **Chat log components only render text:** `ChatLog.addUser()` constructs `new UserMessageComponent(text)`, and `finalizeAssistant()` constructs `new AssistantMessageComponent(text)`. Those components only accept a text argument, so there is no current rendering path for timestamps.
  - file: [src/tui/components/chat-log.ts:105](https://github.com/openclaw/openclaw/blob/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4/src/tui/components/chat-log.ts#L105)
  - command: `nl -ba src/tui/components/chat-log.ts | sed -n '1,230p'`
  - sha: [f3c37a946cc4](https://github.com/openclaw/openclaw/commit/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4)
- **Timestamp data is only used for pending-message reconciliation:** The only timestamp-aware logic in `ChatLog` is `reconcilePendingUsers(...)`, and its tests cover deduping pending users against rebuilt history, not timestamp display.
  - file: [src/tui/components/chat-log.test.ts:128](https://github.com/openclaw/openclaw/blob/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4/src/tui/components/chat-log.test.ts#L128)
  - command: `nl -ba src/tui/components/chat-log.test.ts | sed -n '120,175p'`
  - sha: [f3c37a946cc4](https://github.com/openclaw/openclaw/commit/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4)
- **TUI docs do not describe a timestamp feature:** The docs describe the chat log as showing messages/system notices/tool cards and say Settings toggles deliver/tool output/thinking visibility; no message timestamp option is documented.
  - file: [docs/web/tui.md:53](https://github.com/openclaw/openclaw/blob/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4/docs/web/tui.md#L53)
  - command: `nl -ba docs/web/tui.md | sed -n '50,95p'`
  - sha: [f3c37a946cc4](https://github.com/openclaw/openclaw/commit/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4)
