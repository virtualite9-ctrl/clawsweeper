## Summary

Keep open. Current main still intentionally blocks the behavior this PR is trying to add: Control UI markdown renders remote markdown images as alt text, tests assert that behavior, and the Control UI CSP/docs still prohibit remote image fetches. The PR remains a plausible implementation candidate for open #40945, and the recent user comment saying the feature is needed weighs against stale cleanup. Security review found the PR is narrow UI/test work with no dependency, CI, workflow, or publishing changes, but the functional change reverses an intentional no-remote-fetch policy and needs maintainer product/security sign-off before merge.

## Best Possible Solution

Keep this PR open as an implementation candidate, but require an updated end-to-end design before merge: markdown renderer behavior, CSP, tests, docs, and an explicit maintainer decision on the remote-fetch privacy model. A safer version should likely decide between https-only inline rendering, a domain allowlist, user/admin opt-in, a proxy/managed-media path, lazy loading, and documented privacy tradeoffs. If maintainers decide the current no-remote-fetch policy should remain, close this PR only after recording that decision on #40945 or a canonical media-preview issue.

## Risks / Open Questions

- Merging the renderer-only PR would likely not satisfy the user-visible feature because current Control UI CSP still rejects remote http(s) image loads.
- Allowing arbitrary remote image URLs lets chat/model content trigger browser requests to third-party hosts, exposing operator IP address, user agent, and read timing. The existing gateway `Referrer-Policy: no-referrer` helps the Referer header but does not remove those network metadata risks.
- The PR allows plain `http://` images. If CSP is widened later, that creates a mixed-content/privacy concern on HTTPS deployments unless maintainers choose that behavior deliberately.
- The PR reverses an intentional hardening direction without updating CSP tests, user-facing docs, or adding a policy design such as https-only, allowlist, proxying, opt-in setting, or explicit maintainer acceptance.
