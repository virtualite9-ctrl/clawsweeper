## Summary

Keep open. Current main still blocks the behavior this PR is trying to add: Control UI markdown renders remote markdown images as escaped alt text, tests assert that behavior, and the Gateway CSP/docs still prohibit remote image fetches. The PR is a focused implementation candidate for open #40945, with recent user demand in the discussion, but it needs maintainer product/security sign-off because it reverses an intentional no-remote-fetch hardening policy.

## Best Possible Solution

Keep this PR open as an implementation candidate, but require an updated end-to-end design before merge: renderer behavior, Control UI CSP, tests, docs, and explicit maintainer product/security approval for the remote-fetch privacy model. A safer version should choose a deliberate policy such as HTTPS-only inline rendering, a domain allowlist, user/admin opt-in, managed-media proxying, lazy loading, or documented privacy tradeoffs. If maintainers decide the no-remote-fetch policy should remain, close the PR only after recording that decision on #40945 or a canonical media-preview issue.

## Risks / Open Questions

- Merging this renderer-only PR would likely not produce the intended visible feature because current Control UI CSP still rejects remote `http(s)` image loads.
- Allowing arbitrary remote markdown images lets chat/model content trigger third-party browser requests, exposing operator IP address, user agent, and read timing; `Referrer-Policy: no-referrer` helps only the Referer header.
- The PR allows plain `http://` image URLs, which could create mixed-content and privacy concerns on HTTPS deployments if CSP is widened later.
- The PR reverses an intentional hardening direction without updating CSP, docs, or adding a policy design such as HTTPS-only rendering, allowlists, proxying, opt-in settings, or explicit maintainer acceptance.
