## Summary

Keep open. Current `main` still intentionally flattens remote markdown images to alt text in Control UI, so the requested behavior is not implemented on `main`. The PR also leaves a mirrored export path on the old policy and has an unresolved privacy tradeoff around arbitrary remote image loads.

## Evidence

- **Control UI renderer still blocks remote images:** `md.renderer.rules.image` returns escaped alt text for any image `src` that does not match `INLINE_DATA_IMAGE_RE`, so `http://` and `https://` markdown images still do not render inline on `main`.
  - file: [ui/src/ui/markdown.ts:409](https://github.com/openclaw/openclaw/blob/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503/ui/src/ui/markdown.ts#L409)
  - command: `nl -ba ui/src/ui/markdown.ts | sed -n '405,422p'`
  - sha: [b0d2003fe65b](https://github.com/openclaw/openclaw/commit/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503)
- **Current tests assert the old behavior:** The `images` test block still expects remote images to be flattened to alt text and unlabeled remote images to fall back to the literal `image` label, which matches the current renderer and contradicts the PR's proposed behavior.
  - file: [ui/src/ui/markdown.test.ts:268](https://github.com/openclaw/openclaw/blob/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503/ui/src/ui/markdown.test.ts#L268)
  - command: `nl -ba ui/src/ui/markdown.test.ts | sed -n '250,410p'`
  - sha: [b0d2003fe65b](https://github.com/openclaw/openclaw/commit/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503)
- **Mirrored export renderer also blocks remote images:** The HTML export template has a separate markdown image renderer with the same `INLINE_DATA_IMAGE_RE` gate and also flattens non-data-image URLs to alt text.
  - file: [src/auto-reply/reply/export-html/template.js:1719](https://github.com/openclaw/openclaw/blob/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503/src/auto-reply/reply/export-html/template.js#L1719)
  - command: `nl -ba src/auto-reply/reply/export-html/template.js | sed -n '1710,1740p'`
  - sha: [b0d2003fe65b](https://github.com/openclaw/openclaw/commit/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503)

## Risks / Open Questions

- Merging the PR as written would allow arbitrary remote image fetches in chat, which can leak viewer IP, timing, and user-agent metadata; that tradeoff is not resolved in the review context.
- The PR updates only `ui/src/ui/markdown.ts` and its tests, while `src/auto-reply/reply/export-html/template.js` and its security tests still enforce the opposite policy, so the change is incomplete unless that divergence is intentional.
