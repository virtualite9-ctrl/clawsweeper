## Summary

Keep open. Current main still does not render remote markdown images in Control UI chat: the markdown renderer flattens non-data image URLs, the regression test asserts that behavior, and the Control UI CSP explicitly blocks remote http(s) image loads. This PR is a relevant implementation candidate for the still-open linked bug, but it is incomplete against current main because a renderer-only change would still be blocked by CSP and needs an explicit privacy/security decision before it can land.

## Best Possible Solution

Keep this PR open as a useful but incomplete implementation candidate. The best path is an updated change, here or in a successor PR, that resolves the linked #40945 behavior end to end: renderer tests, CSP tests/docs, and an explicit maintainer decision on privacy mitigations such as referrerpolicy="no-referrer", lazy loading, https-only support, or a domain/setting-based allowlist. If maintainers decide the current no-remote-fetch policy should stand, close the PR after recording that product/security decision on the linked issue.

## Risks / Open Questions

- The submitted renderer-only patch is not sufficient against current main because Control UI CSP blocks remote http(s) image loads.
- If maintainers relax CSP to make the feature work, arbitrary markdown can create tracking-pixel/read-receipt style requests from operator browsers, exposing IP address, User-Agent, and read timing to the image host.
- The PR allows http:// as well as https:// in the renderer proposal; maintainers may want https-only, an allowlist, or explicit private-network handling if remote images become supported.
