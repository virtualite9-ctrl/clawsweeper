## Summary

Keep open. Current main still does not implement this PR's targeted chat-history display sanitizer: the active projection only strips top-level `type: "image"` `data` and base64 audio `source.data`, while the PR covers image `source.data`, image `url` data URLs, and OpenAI-style `image_url.url` data URLs. The linked dashboard-freeze issue was later closed for a different live-render/reload fix, and #1210 tracks broader transcript storage bloat rather than this narrower chat.history response sanitization.

## Best Possible Solution

Keep the PR open for maintainer review or contributor follow-up. The best implementation is to port the sanitizer into `src/gateway/chat-display-projection.ts`, preserve the byte-accounting fix, add focused unit coverage for image `source.data`, data-URL `url`, and `image_url.url`, and keep #1210 as the broader transcript-storage design item.
