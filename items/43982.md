## Summary

Keep this PR open. Current main still does not sanitize the inline image payload shapes this PR targets on the Gateway display/history projection path, even though newer chat-send media persistence reduces some fresh webchat cases. The PR remains a plausible focused fix, but it needs a rebase/port into the current shared projection module and fuller regression coverage.

## Best Possible Solution

Keep this PR open for maintainer review or contributor follow-up. The best path is to port the sanitizer into `src/gateway/chat-display-projection.ts`, preserve the accumulated byte accounting fix, add focused projection/history tests for the inline image payload shapes, and keep #1210 as the broader durable transcript-media storage design item.
