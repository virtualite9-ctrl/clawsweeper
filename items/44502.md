## Summary

Keep #44502 open. Current main still does not implement the reported Discord attention-window/current-agent targeting behavior: preflight feeds the full message text into mention matching, does not enumerate competing configured Discord agents before routing, and still allows bot-authored messages when `allowBots="mentions"` and the current bot is mentioned. The related Feishu slash-command comment appears adjacent but not a duplicate of the Discord mention-routing bug.

## Best Possible Solution

Keep this issue open and fix it in the Discord plugin preflight path. The likely implementation belongs in `extensions/discord/src/monitor/message-handler.preflight.ts` with focused tests in `message-handler.preflight.test.ts`: derive a bounded opening attention text, use it for mention and role attention, detect configured Discord-bound agents/bot senders that are competing targets, and add regression coverage for the reporter's quoted-mention, other-agent, unrelated-role, and bot-loop cases. The Feishu slash-command comment should be split to a separate tracker if still reproducible.
