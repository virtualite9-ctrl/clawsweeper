## Summary

Keep this issue open. Current main has partially addressed adjacent pieces: ACP spawns are task-backed and hook auto-delivery can be suppressed with the existing deliver:false path. The central requested capability is still missing, though: ACP sessions_spawn still rejects mode:"session" unless thread:true, and the docs/tests preserve that contract, so hook-triggered or otherwise headless persistent ACP sessions without thread binding remain unresolved.

## Best Possible Solution

Keep the issue open and narrow the maintainer path to the remaining headless ACP persistence contract. A conservative fix would add an explicit documented headless/persistent ACP spawn mode, or relax the thread guard only when delivery and resume semantics are clear; it should also expose the ACP backend session identifier or a documented status/tasks lookup path, with tests for hook-triggered sessions using mode:"session" without thread binding.
