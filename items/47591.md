## Summary

Keep #47591 open. Current main has adjacent visibility work, including run-aware typing keepalive, Telegram/Slack tool-progress previews, message_sent hooks, and a write-ahead outbound delivery queue with recovery. Those pieces do not implement the requested gateway-level silence watchdog or per-turn response delivery confirmation with response IDs, same-turn retry, duplicate suppression, no-response detection, and operator alerting. Related #65919 is narrower and still open for active-run presence, while #41501’s ClawSweeper context confirms delivery-confirmation gaps still remain in some core paths.

## Best Possible Solution

Keep this open unless maintainers create narrower canonical follow-ups. The best product path is a shared runtime visibility coordinator outside the model loop: reuse existing typing/preview/progress surfaces for active-run presence where configured, add an optional silence timer keyed to operator-visible delivery, and extend the outbound/reply-dispatch pipeline with explicit delivery result metadata, turn/response correlation, no-response detection, retry/alert policy, and regression tests across inline replies and message.send/subagent delivery paths.
