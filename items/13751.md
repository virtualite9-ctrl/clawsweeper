## Summary

Keep open. Current main still makes Feishu inbound sender-name resolution contact/profile API-backed by default, with only an opt-out flag and suppression for one stale permission prompt. The directory peer path also still uses live contact user listing when invoked through channel-list.

## Best Possible Solution

Keep #13751 open. The best product path is to make basic Feishu inbound bot operation contact-free by default: use webhook sender fields when available, otherwise attribute with `open_id`/`user_id` without calling `contact.user.get`; make profile lookup an explicit opt-in. Live directory peer listing should also become static-first or explicitly opt-in for contact-directory access, with Feishu docs and regression tests updated around the new default.
