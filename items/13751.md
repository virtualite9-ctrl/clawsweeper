## Summary

Keep #13751 open. Current main still resolves Feishu sender names through `client.contact.user.get` by default, still documents/tests `resolveSenderNames` as default-on, and still uses live contact-directory lookup for Feishu peer listing with static fallback disabled in `channel-list`. The April 26 issue discussion already reached the same keep-open conclusion, and current HEAD is a logging-redaction change with no Feishu surface changes.

## Best Possible Solution

Keep #13751 open. Make basic Feishu inbound bot operation contact-free by default: parse sender identity from webhook fields when available, otherwise use open_id/user_id without calling `contact.user.get`; keep profile lookup as an explicit opt-in. For directory peers, prefer configured static peers unless live contact-directory lookup is explicitly enabled. Update Feishu config metadata, docs, and regression tests for the new default.
