## Summary

Keep open. Current main still defaults to Feishu sender-name resolution through the contact user API, and live peer listing still uses contact user listing. Existing changes add opt-out and suppress one stale-scope prompt, but they do not remove the default contact/profile API dependency requested in #13751.

## Best Possible Solution

Keep #13751 open. The best product/code path is to make basic Feishu inbound bot operation avoid contact/profile APIs by default: use webhook sender data when present, otherwise attribute with open_id/user_id without calling contact.user.get; make sender profile lookup explicit opt-in. Live directory peer listing should also become static-first or explicitly opt-in for contact-directory access, with docs and regression tests updated around the new default.
