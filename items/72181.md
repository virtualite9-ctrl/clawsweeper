## Summary

Keep #72181 open. Current `main` still does not implement BlueBubbles `trainerMode`/`trainerNotifyNumber` or an owner approval path before outbound DM delivery. The issue is a fresh, concrete feature request for the bundled BlueBubbles plugin, the discussion already identifies the intended plugin owner for review, and the prior ClawSweeper keep-open comment remains accurate after rechecking current `main` at `e672b61417af5c45b0431df6d9109a1f4b618ef5`.

## Best Possible Solution

Keep #72181 open for BlueBubbles owner/product review. A good implementation would stay under `extensions/bluebubbles`, add typed/schema-backed `trainerMode` and `trainerNotifyNumber` config, intercept allowed inbound DMs before normal reply dispatch, route drafts and owner commands through existing BlueBubbles send paths, enforce pending-message TTLs, and add monitor/config/docs coverage for both `training` and `supervised` modes.
