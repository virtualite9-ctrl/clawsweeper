## Summary

Keep #12429 open. Current main has minimal single-pod Kubernetes deployment support and a CIDR-based first-time node pairing auto-approval option, but it still does not implement the requested Kubernetes ServiceAccount Trust authentication path for autoscaled worker/node-host pods.

## Best Possible Solution

Keep the issue open until OpenClaw either implements an opt-in Kubernetes ServiceAccount Trust path across gateway config/schema, WebSocket protocol, GatewayClient/node-host credential handling, TokenReview validation, tests, docs, and deployment assets, or maintainers explicitly designate a narrower canonical distributed-runtime/K8s-auth issue that supersedes it.
