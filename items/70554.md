## Summary

Keep #70554 open. Current main still has the memory-core request-scoped dreaming narrative fallback and cleanup warning path that #67152 reports, and main still lacks the plugin subagent fields needed to forward cron run kind through the gateway background run path. The latest provided PR context shows the branch is a direct, focused attempt to fix that remaining behavior rather than obsolete or already implemented work.

## Best Possible Solution

Keep #70554 open as useful work for the open #67152 bug. The best path is a narrowed maintainer-reviewed patch that retries request-scoped dreaming narrative failures through a gateway-owned background subagent surface, forwards bootstrapContextMode and bootstrapContextRunKind through plugin subagent dispatch, preserves cron run kind in both CLI and embedded reply execution, keeps gateway runtime access off broad public SDK barrels, and retains focused regression coverage for retry, transient cleanup, and non-transient error handling.
