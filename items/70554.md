## Summary

Keep open. Current main still does not implement the PR's central behavior: memory-core dreaming narrative generation falls back to a local snippet write when the injected subagent runtime is request-scoped, and the plugin subagent surface does not yet forward a cron bootstrap run kind for this path. The PR may still need cleanup before merge, but it is addressing a real remaining bug rather than obsolete work.

## Best Possible Solution

Keep #70554 open as potentially useful work for #67152, but require the author or a maintainer to rebase onto current main, remove unrelated conflict-resolution churn, keep any gateway background-subagent access on a narrow non-public surface, forward `bootstrapContextRunKind` through the plugin subagent and both embedded/CLI execution paths, and retain focused regression coverage for retry, cleanup, and non-transient error behavior.
