## Summary

Keep #70554 open. Current main still preserves the request-scoped memory-core dreaming fallback behavior the PR is trying to replace, and the plugin subagent path still lacks bootstrap run-kind forwarding for cron narrative runs. The PR may need maintainer review and validation before merge, but it is not obsolete, implemented on main, or merely administrative cleanup.

## Best Possible Solution

Keep #70554 open as potentially useful work for #67152. The best path is a narrow rebased fix that retries memory-core dreaming narratives through a gateway-owned background subagent only via a constrained internal or memory-core host surface, forwards `bootstrapContextMode` and `bootstrapContextRunKind` through plugin subagent dispatch and both CLI/embedded reply execution paths, preserves best-effort cleanup, and keeps regression coverage for request-scoped retry, transient cleanup fallback, and non-transient error behavior.
