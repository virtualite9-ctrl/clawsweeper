## Summary

Issue #63565 reports a real remaining proxy-only `web_fetch` gap on current `main`, but the remaining work is already tracked by the active canonical PR #58034. Current `main` still keeps built-in `web_fetch` and Firecrawl on strict DNS-pinned fetch paths, while #58034 is the active implementation/review thread for the explicit trusted-env-proxy opt-in, NO_PROXY fallback, docs, and tests.

## Best Possible Solution

Close #63565 as superseded by the active canonical PR #58034 and keep the remaining maintainer review, implementation, docs, and regression-test work concentrated there. The eventual fix should preserve the explicit opt-in boundary for trusted env proxy use and cover both built-in `web_fetch` and Firecrawl fallback paths.
