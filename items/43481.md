## Summary

Keep this issue open. Current main does not implement the requested provider-level trust boundary for fallback image/attachment data, and the provided GitHub context shows an open same-author PR that explicitly closes this issue. Per cleanup rules, that PR is an implementation candidate, not a reason to close the issue before merge.

## Best Possible Solution

Keep this issue open while maintainers review, update, or close PR #57959. The best implementation path is a focused fallback-boundary change that threads enough primary/target provider context into `runAgentAttempt`, applies explicit-image policy via a tested helper, suppresses or gates prompt-detected image loading on cross-provider retries, and documents any user-facing trust configuration if maintainers want opt-in behavior rather than unconditional cross-provider stripping.
