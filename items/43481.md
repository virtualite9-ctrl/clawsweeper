## Summary

Keep this issue open. Current main still does not implement a provider-level image or attachment trust policy for model fallback, and prompt-detected local images are still loaded without a fallback trust guard. The provided GitHub context also shows open same-author PR #57959 with closing syntax for this issue, so maintainers should review, land, or close that implementation candidate before this issue is closed.

## Best Possible Solution

Keep this issue open while maintainers review, update, land, or close PR #57959. The best implementation path is a focused fallback-boundary change that threads primary and target provider context plus failure context through runWithModelFallback and runAgentAttempt, centralizes a tested image trust decision, suppresses prompt-detected image loading when the fallback target is not trusted, and documents any user-facing trust configuration if maintainers choose configurable behavior over conservative cross-provider stripping.
