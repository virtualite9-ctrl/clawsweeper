## Summary

Keep open. Current main has bundle-backed Embedded Pi LSP support, but it is limited to loading Claude bundle LSP defaults and exposing hover, definition, and references tools. It does not capture `publishDiagnostics`, maintain server health/performance state, or provide the requested real-time editor/dashboard troubleshooting surface. The same author also has paired open LSP setup issue #65251, so this should not be auto-closed independently.

## Best Possible Solution

Keep this issue open and scope it together with #65251. The best path is to define a generic LSP diagnostics contract first: capture server lifecycle, stderr/errors, request latency, and `publishDiagnostics` events in a runtime or plugin-owned service; expose that through a documented gateway/plugin API; then decide whether the dashboard belongs in core Control UI, a plugin-served route, or an external editor/ClawHub plugin.
