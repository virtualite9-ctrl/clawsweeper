## Summary

Current `main` already resolves Control UI default-model labels through the recorded routing provider and catalog display lookup, not by blindly treating the first path segment as the provider. The exact `anthropic` active session vs `openrouter` default-label regression is covered by a UI regression test, and the same code is present in the shipped `v2026.4.23` tag.

## Evidence

- **Default label uses provider-aware resolution:** `resolveDefaultModelValue()` passes both `defaults.model` and `defaults.modelProvider` into `resolvePreferredServerChatModelValue()`, then `defaultLabel` is built from `formatCatalogChatModelDisplayFromLookup(defaultModel, displayLookup)`.
  - file: [ui/src/ui/chat-model-select-state.ts:50](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/ui/src/ui/chat-model-select-state.ts#L50)
  - sha: [ec3dbd22a4de](https://github.com/openclaw/openclaw/commit/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46)
- **Slash-containing server ids can be re-qualified to the routing provider:** For models like `anthropic/...` plus provider `openrouter`, `resolvePreferredServerChatModelValue()` first tries catalog-backed resolution and returns the qualified catalog value before falling back to the raw slash-split display path.
  - file: [ui/src/ui/chat-model-ref.ts:106](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/ui/src/ui/chat-model-ref.ts#L106)
  - sha: [ec3dbd22a4de](https://github.com/openclaw/openclaw/commit/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46)
- **Exact regression test exists on main:** The test sets the active session to provider `anthropic`, the defaults provider to `openrouter`, and asserts `Default (Claude Sonnet · openrouter)` plus separate picker entries for both providers.
  - file: [ui/src/ui/chat-model-select-state.test.ts:143](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/ui/src/ui/chat-model-select-state.test.ts#L143)
  - sha: [ec3dbd22a4de](https://github.com/openclaw/openclaw/commit/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46)
