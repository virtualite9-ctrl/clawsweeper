## Summary

This PR should stay open. Current main still exposes ambiguous target wording, and the linked bug has a concrete reproduction showing models repeatedly send a bare Discord user ID before discovering `user:<id>`. The PR is on the right surface for CLI/shared target help, but as written it appears incomplete for the actual message tool `target` parameter because that schema overrides the shared description with its own generic text.

## Best Possible Solution

Keep this PR open and request a small update before merge: make the singular `message.target` schema in `src/agents/tools/message-tool.ts` surface the same explicit Discord/Slack/Mattermost target syntax, keep the shared constant improvement for CLI/array target help, and add the required changelog attribution. After that, review/land it as the implementation candidate for #72401.
