## Summary

Keep PR #69608 open. Current main has adjacent startup-context support for canonical and slugged daily files, but it does not implement the PR's central shared daily-memory/session-summary model. Several current main paths still use canonical-only matching, unfiltered slugged startup reads, or missing bootstrap placeholder injection, so this PR is not obsolete cleanup.

## Best Possible Solution

Keep #69608 open for maintainer review. The product/code direction should be to land a focused, validated version of the shared daily-memory/session-summary helper model, or split it into smaller PRs that first add the shared helper/API and then migrate startup context, session-memory, doctor, dreaming, REM backfill, short-term promotion, and utility embedded runs onto it.
