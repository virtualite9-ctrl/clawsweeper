## Summary

Keep open. Current `main` still force-reconnects idle WhatsApp Web sessions after 30 minutes without inbound messages, converts that watchdog path into status `499`, and resets reconnect attempts after one heartbeat interval, so the reported 499 reconnect cascade remains credible on current code. The issue is also freshly corroborated by a second production report from 2026-04-21.

## Evidence

- **Idle-message watchdog still forces reconnects on main:** `monitorWebChannel` still defaults `messageTimeoutMs` to 30 minutes and `watchdogCheckMs` to 60 seconds, and its watchdog handler logs `Message timeout detected - forcing reconnect` when no inbound message has arrived since `lastInboundAt` or connection start.
