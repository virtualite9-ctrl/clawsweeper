## Summary

Keep this issue open. Current main still drops `accountId` on the direct `/hooks/agent` path: the hook payload type and normalizer omit it, and `dispatchAgentHook()` builds a cron job whose delivery object only carries mode/channel/to. The cron delivery stack itself already supports `delivery.accountId`, so the remaining gap is the hook-to-cron handoff. The same author has open PR #43951 with closing syntax for this work, so the best maintainer path is to review/rebase that PR or replace it with a fresh focused patch rather than close the issue first.

## Best Possible Solution

Keep this issue open until PR #43951 is reviewed/merged or replaced. The best fix is a focused current-main patch that adds `accountId` to the `/hooks/agent` payload contract, normalizes/trims it, includes it in idempotency/replay scoping, passes it into `CronDelivery.accountId` during hook dispatch, updates docs, and adds regression coverage proving `job.delivery.accountId` reaches isolated cron delivery.
