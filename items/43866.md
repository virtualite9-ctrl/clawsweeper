## Summary

Keep this issue open. Current main still drops `accountId` on the direct and mapped `/hooks/agent` paths before constructing the isolated cron job, while the downstream cron delivery stack already supports `delivery.accountId`. There is also an open same-author implementation candidate, #43951, tied to this issue, so the right maintainer action is to review and land or replace that PR rather than close the report now.

## Best Possible Solution

Keep this issue open until #43951 is reviewed and merged, or replaced with a focused current-main patch. The fix should add `accountId` to the direct and mapped hook agent contracts, normalize and trim it, include it in hook replay/idempotency scope, propagate it into `CronDelivery.accountId` during hook dispatch, update the public hook docs, and add regression coverage proving hook dispatch reaches isolated cron delivery with the selected account.
