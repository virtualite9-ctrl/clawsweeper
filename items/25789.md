## Summary

Keep PR #25789 open. Current `main` still does not expose a configurable exec approval timeout: the shipped code uses a fixed 30-minute default and documents that default, but there is no `tools.exec.approvalTimeoutMs`, `approvals.timeoutMs`, or env-backed override in the typed config, strict schemas, docs, or runtime plumbing. The related feature issue #51287 remains open, competing PR #51292 was closed unmerged, and the existing maintainer/Codex review comment on this PR already says it should stay open unless deliberately superseded.

## Best Possible Solution

Keep the PR open for explicit maintainer review, or deliberately supersede it with canonical implementation work under open issue #51287. A complete solution should choose the public key, update exported config types, strict Zod schemas, labels/help/generated schema and user docs, thread the value through exec defaults and gateway/node approval registration, adjust wait/expiry and CLI transport timeout floors, and add focused regression coverage.
