## Summary

Keep open. Current main still lacks a user-configurable exec approval timeout, and the related configurable-timeout PR #51292 is closed unmerged while the canonical issue #51287 remains open per the provided context. Main does use a fixed 30-minute default, including in v2026.4.24, but that only reduces the old 2-minute timeout pain; it does not implement the requested config knob. The provided PR context also shows the author added a second commit wiring through the earlier review feedback, so this is not an obviously obsolete low-signal PR, and a maintainer comment already explicitly kept it open for judgment.

## Best Possible Solution

Keep this PR open for maintainer judgment, or explicitly supersede it with a canonical configurable-timeout implementation. The best fix should choose the public config key, update exported types, strict schemas, labels/help/generated config metadata and docs, then wire the value through config resolution, exec defaults, gateway/node approval registration, wait/expiry behavior, and CLI transport timeout floors with targeted regression tests.
