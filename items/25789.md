## Summary

Keep this PR open. Current main still does not expose a configurable exec approval timeout, and the PR is a plausible implementation candidate rather than obsolete cleanup. It needs maintainer API judgment and a refresh against current main, especially because the shipped default is now 30 minutes and the node CLI path has changed since the PR was opened.

## Best Possible Solution

Keep this PR open for maintainer review, or deliberately supersede it with canonical work under open issue #51287. A complete solution should choose the public config shape (`tools.exec.approvalTimeoutMs` versus `approvals.timeoutMs`), align the default with the shipped 30-minute behavior, update exported types, strict schemas, labels/help/generated config docs, thread the value through exec defaults and gateway/node approval registration, preserve transport timeout floors, and add focused regression tests for config resolution and approval request timeout behavior.
