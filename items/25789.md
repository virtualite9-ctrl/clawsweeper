## Summary

Keep open. Current main still does not expose or wire a configurable exec approval timeout; it uses a fixed 30-minute default. The PR branch attempted to address the original missing-wiring review in a second commit, and the related configurable-timeout items remain open/unmerged, so this is not a conservative cleanup close.

## Best Possible Solution

Keep tracking the capability until a maintainer chooses the canonical config shape, likely between `tools.exec.approvalTimeoutMs` and the newer `approvals.timeoutMs` proposal. A complete fix should update exported config types, strict schemas, labels/help/generated metadata/docs, runtime config resolution, gateway/node approval registration, CLI transport timeout handling, and focused tests against the current 30-minute default.
