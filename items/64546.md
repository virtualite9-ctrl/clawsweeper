## Summary

Keep this PR open. Current main still implements and documents the forgeable bot-token-derived Mattermost interaction secret, and this PR is paired with the still-open linked bug #64545 by the same author. The PR is a relevant implementation candidate, but it is not merge-ready because the random in-memory fallback can break CLI/gateway cross-process button verification unless a persistent secret is configured.

## Best Possible Solution

Keep this PR and #64545 open. Revise the Mattermost fix so interaction signing uses a persistent per-account secret shared by send and monitor paths, supports SecretRef/config or credential-store persistence, handles existing deployments intentionally, updates schema/types/docs/secret metadata, and adds regression coverage for both forgery resistance and CLI/gateway cross-process verification before merge.
