## Summary

Keep this PR open. Current main still derives Mattermost interaction signing secrets from the bot token with the public `openclaw-mattermost-interactions` HMAC key, the paired open bug #64545 remains unresolved, and this PR is the active implementation candidate. The PR is scoped to Mattermost code/tests and does not show a supply-chain or CI regression, but its random in-memory fallback needs revision before merge because send and gateway verification can run in different processes or across restarts.

## Best Possible Solution

Keep this PR and #64545 open. Revise the Mattermost fix so interaction signing uses a persistent per-account secret that is not derived from the bot token, is shared by send and gateway monitor paths across processes/restarts, supports SecretRef or credential-store persistence, updates schema/types/docs/setup/secret metadata, and adds regression coverage for bot-token forgery resistance plus cross-process/restart verification before merge.
