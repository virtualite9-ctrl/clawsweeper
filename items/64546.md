## Summary

Keep this PR open. Current main still implements and documents the bot-token-derived Mattermost interaction HMAC secret, so the linked bug #64545 remains real. The PR is a relevant implementation candidate, but the provided review context flags a correctness issue in the PR head: a generated in-memory fallback can break CLI/gateway cross-process button verification when no persistent interaction secret is configured.

## Best Possible Solution

Keep this PR and the linked bug #64545 open. The best path is to revise the Mattermost plugin fix so interaction signing uses a persistent per-account secret shared by send and monitor paths, supports SecretRef/config or credential-store storage, preserves or intentionally migrates existing deployments, and updates schema/types/docs plus regression tests for both forgery resistance and cross-process verification.
