## Summary

Current `main` already round-trips code-block copy text correctly through `data-code`/`dataset.code`, and the linked issue #69605 is about clicking the wrong overlapping copy control rather than HTML-entity corruption.

## Evidence

- **Current renderer already HTML-escapes the copied payload:** Both fenced and indented code blocks store `data-code` using `escapeHtml(text)`, which escapes `&`, `<`, `>`, `"`, and `'` before inserting the attribute.
