## Summary

This is a recent, concrete report about local Web UI live-rendered assistant code losing indentation until a refresh reloads persisted history. Current main has markdown/code-block preservation tests and several related streaming/history paths, but I did not find a targeted fix, release note, regression test, or canonical duplicate proving the live Control UI/WebChat path for `openclaw-cpa-codex` is fixed.

## Best Possible Solution

Keep this issue open. The best maintainer path is to reproduce on current main using local Control UI/WebChat with the `openclaw-cpa-codex` route, capture the raw live `chat` delta and the later `chat.history` message for the same run, then add a regression test around the layer that differs: `src/gateway/server-chat.ts` for live delta buffering, `ui/src/ui/chat/grouped-render.ts`/`ui/src/ui/markdown.ts` for live rendering, or `extensions/codex/src/app-server/event-projector.ts` if the Codex app-server projection is the source.
