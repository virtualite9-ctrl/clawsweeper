## Summary

Keep open. The report is not strong enough to close conservatively: the only concrete raw-update sample in the issue shows `message.from.id` equal to the bot account id, and current main intentionally ignores self-authored group updates, but that does not conclusively disprove the broader claim that human-authored group messages can be dropped in multi-account polling setups. I did not find a clear on-main fix, changelog entry, or release note that cleanly resolves this issue.
