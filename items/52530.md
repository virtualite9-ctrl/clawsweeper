## Summary

Current main already implements browser-node auto-routing for browser-tool calls on existing-session profiles, and the shipped docs/tests now define explicit `target="host"` as a local pin rather than an auto-routed node path.

## Close Comment

Closing from a Codex review because current `main` already implements the requested browser-node auto-routing behavior. The browser tool now resolves connected browser-capable nodes and proxies browser calls through the node browser proxy (`extensions/browser/src/browser-tool.ts:206`, `extensions/browser/src/browser-tool.ts:276`, `extensions/browser/src/browser-tool.ts:416`). The docs also now define this as the default remote-gateway path and clarify that explicit `target="host"` is an intentional local pin, not an auto-routed node path (`docs/tools/browser.md:237`, `docs/tools/browser.md:590`). The behavior is covered by tests that verify node proxy routing and that explicit `target="host"` stays local (`extensions/browser/src/browser-tool.test.ts:631`, `extensions/browser/src/browser-tool.test.ts:655`). This shipped in `v2026.4.22` (`CHANGELOG.md:316`). If you still want the connected browser node, the supported path is to omit `target` so auto-routing can select the node, or use `target="node"` explicitly.
