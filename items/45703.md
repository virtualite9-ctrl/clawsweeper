## Summary

Current main still renders the Agents page selection as a plain native dropdown and does not render the agent emoji/avatar metadata in the selector or overview. The related model-selector provider issue appears to have separate provider-aware handling and tests, but that does not resolve this issue's central Agents section regression.

## Best Possible Solution

Keep this issue open. The likely product fix belongs in the Control UI Agents view: replace or supplement the current native dropdown with an accessible rich list/combobox/card selector that renders agent name/id, default status, workspace/model metadata, and identity emoji/avatar, and ensure any action buttons have concrete handlers. Add focused UI render tests for emoji/avatar/metadata visibility and preserve the already provider-aware model-selector behavior as a separate surface.
