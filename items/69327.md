## Summary

Keep #69327 open. Current main has baseline Docker env/config plumbing and older workspace-handoff fixes, but I did not find a source, test, or changelog fix for the reported subagent-specific env inheritance and recreate freshness gap. The `sessions_spawn` path still propagates workspace/group metadata only, while the child sandbox is resolved independently from the target agent config rather than inheriting the parent agent's resolved sandbox Docker context.

## Best Possible Solution

Keep the issue open and add a focused regression around a sandboxed parent using `sessions_spawn` with per-agent `sandbox.docker.env` and `setupCommand`, including a recreate path. If the regression confirms the report, thread the parent resolved sandbox Docker context through the subagent spawn path or otherwise make `sandbox="inherit"` semantics explicit and test-backed.
