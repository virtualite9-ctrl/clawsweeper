## Summary

Keep open. Current main still matches the reported Feishu oversized-media silent failure: the media save path throws on size limit, Feishu media resolution catches and logs download/save failures without returning failure state or sending a chat-visible message, and the handler continues into normal agent dispatch with an empty media payload.

## Best Possible Solution

Keep the issue open and fix it in the bundled Feishu plugin. The best path is for media resolution to return structured failure state or throw a typed recoverable media error, send a Feishu-visible warning that includes the configured `mediaMaxMb` limit, and skip normal agent dispatch when the triggering input was dropped media. Add regression tests for oversized single-file media and rich post embedded media failures asserting user notification and no unqualified agent run.
