## Summary

Keep open. Current main still has the reported Feishu silent media-drop path: oversized media makes the shared media store throw, Feishu media resolution catches and logs the error, returns an empty media list, and the handler continues into normal agent dispatch without a user-facing Feishu warning.

## Best Possible Solution

Keep the issue open and fix it in the bundled Feishu plugin. The best path is for Feishu media resolution to return structured success/failure state or throw a typed recoverable media error, send a chat-visible warning that includes the configured mediaMaxMb limit, and avoid normal agent dispatch when the triggering input was only dropped media. Add regression tests for oversized single media and embedded post media failures asserting user notification and no unqualified agent dispatch.
