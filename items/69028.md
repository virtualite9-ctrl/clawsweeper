## Summary

Keep open. Current main still matches the reported Feishu silent-failure path: oversized inbound media makes saveMediaBuffer throw, resolveFeishuMediaList catches and only logs the error, then the handler builds an empty media payload and proceeds to normal agent dispatch with no user-facing Feishu warning. The provided related reports are adjacent media issues, not a canonical replacement for this Feishu oversized-media notification/blocking bug.

## Best Possible Solution

Keep the issue open and fix it in the bundled Feishu plugin. Make media resolution return structured success/failure state or throw a typed recoverable media error, send a chat-visible Feishu warning that includes the configured mediaMaxMb limit, and avoid normal agent dispatch when the triggering input was only dropped media. Add regression tests for oversized single media and embedded post media failures asserting both user notification and no unqualified agent dispatch.
