## Summary

Keep open. Current main still routes gateway startup/model warmup through `ensureOpenClawModelsJson`, and the merge path still replaces a matching provider's existing `models` array with regenerated config/implicit provider data while only preserving selected auth/base URL fields. The recent provider merge change preserves input metadata for matching model IDs, but it does not preserve user-pruned model lists across regeneration.

## Best Possible Solution

Do not close yet. Maintainers should decide the product contract for per-agent `models.json`: if user-pruned provider model arrays are supported, add a gateway-restart regression around `ensureOpenClawModelsJson`/`planOpenClawModelsJsonWithDeps` and preserve user deletions or provide an explicit tombstone/replace mechanism. If `models.json` is generated cache, update docs, doctor, and CLI guidance so users make persistent edits through `models.providers` or `agents.defaults.models` with the intended `--merge`/`--replace` flow.
