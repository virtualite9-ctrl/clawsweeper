## Summary

Keep open. Current main still regenerates the agent `models.json` during gateway startup warmup, and the merge path does not preserve a user-pruned provider model array. The issue is recent, concrete, non-maintainer-authored, and has no protected labels or merged/open linked PR that would make it closeable.

## Best Possible Solution

Do not close yet. Maintainers should decide whether per-agent `models.json` model arrays are supported editable state or generated cache. If editable, add a gateway-restart regression around `ensureOpenClawModelsJson`/`planOpenClawModelsJsonWithDeps` and change `src/agents/models-config.merge.ts` so user-pruned provider model arrays survive regeneration. If not editable, update docs, doctor, and CLI guidance so users change `agents.defaults.models` or `models.providers` with the intended `--merge`/`--replace` flow instead of hand-editing generated `models.json`.
