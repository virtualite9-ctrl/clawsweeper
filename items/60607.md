## Summary

Current `main` still has a real zero-reply path for this report. The core runner can return with no final payload, `dispatchReplyFromConfig` can finish with `queuedFinal=false`, and Feishu/other channel handlers still only log or return in that case instead of sending a user-visible fallback. Telegram has a targeted fallback now, so the issue is partially mitigated there, but not implemented generically or for Feishu.
