## Summary

Keep open. Current main still implements `POST /hooks/agent` as an async hook trigger that returns `{ ok, runId }`; it does not parse `waitForResult`, `resultMode`, or `announceToMain`, wait for completion, or return assistant text. Related PR #67433 is an open implementation candidate, but it is not merged/shipped, so the feature request remains actionable.

## Best Possible Solution

Keep #49019 open as the canonical product/API tracker. The likely implementation belongs in the Gateway hook path: add an opt-in wait/result branch for `/hooks/agent`, define narrow response and timeout semantics, preserve existing async behavior by default, decide how main-session summary suppression should work, and update gateway hook tests plus public docs. PR #67433 can remain the concrete implementation candidate if maintainers accept that API shape.
