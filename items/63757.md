## Summary

Keep this issue open. Current main receives Telegram reaction updates and surfaces them as text system events, but it still does not expose a stable structured managed-hook or plugin-hook event carrying reaction fields for downstream feedback automation.

## Best Possible Solution

Keep the issue open and add an additive structured channel-reaction hook/API seam. Telegram should emit a typed payload from its existing `message_reaction` handler with channel/account, session or conversation context, Telegram chat id, target message id, emoji/reaction operation, sender id/name/username, authorization result, and raw metadata where needed. Feedback recording or automation should live in a plugin/managed hook built on that seam. Add focused Telegram handler tests, plugin/internal hook contract tests, and docs under the public Telegram and plugin hook docs.
