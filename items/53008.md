## Summary

Current main does not yet implement the requested isolation: the pre-compaction memoryFlush run is still dispatched through runEmbeddedPiAgent without a lane override, and the embedded runner defaults missing lanes to the main global lane. The related implementation attempt, PR #53154, was closed unmerged.

## Best Possible Solution

Keep this issue open and implement the focused lane isolation. The likely path is to route the memoryFlush embedded run through CommandLane.Subagent or a dedicated compaction lane, then add a regression test in src/auto-reply/reply/agent-runner-memory.test.ts that asserts the memory flush call passes the expected non-main lane. Maintainers should also decide whether same-session message interactivity requires a separate session/background path or whether preserving per-session serialization is intentional.
