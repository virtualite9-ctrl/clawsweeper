## Summary

Keep PR #72201 open. Current main does not implement a system-event `audience` field, does not wrap internal system events in the runtime-context envelope, and still queues the isolated-cron awareness event as a normal untrusted system event. The PR is a plausible focused implementation for part of #69492 rather than obsolete cleanup.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to refine the implementation, likely share the runtime-context header/helper to avoid drift, run the changed gates, and either land this focused primitive or replace it with an equivalent maintainer-authored implementation. Do not close it as cleanup while current main lacks the requested system-event audience lane and #69492 remains open.
