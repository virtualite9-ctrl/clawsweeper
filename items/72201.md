## Summary

Keep #72201 open. Current main still lacks the `SystemEvent.audience` primitive, still drains all queued system events as visible `System:` / `System (untrusted):` prompt lines, and still queues isolated cron awareness without an internal audience marker. The PR is recent, focused, linked to the still-open #69492 system-event leakage tracker, and its bot review concern was addressed in the PRâ€™s latest commit.

## Best Possible Solution

Keep the PR open for maintainer review. If accepted, the best path is to land a narrow, backward-compatible `SystemEventAudience` primitive with `user-facing` as the default, wrap only explicitly internal events at `drainFormattedSystemEvents`, migrate only justified producers such as isolated cron awareness, and leave broader user-deliverable event routing/noise decisions tracked under #69492/#69208.
