## Summary

Close #72197 as duplicate/superseded by open issue #56968. The MemOS Local report describes the same third-party memory-plugin status false negative already tracked there: runtime memory works, but status resolves through built-in memorySearch/default-store gates and renders a null snapshot as `enabled (plugin …) · unavailable`.

## Best Possible Solution

Close this MemOS-specific report as a duplicate of #56968 and keep the shared core fix tracked there. The best code path is to update the status resolver so non-core memory-slot plugins are treated as plugin-owned and status consults the active memory runtime or reports an inconclusive/plugin-managed state instead of rendering `unavailable` from built-in memorySearch/default-store preconditions.
