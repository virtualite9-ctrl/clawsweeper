## Summary

Keep #11414 open. Current main has partial Telegram forum topic support for create/edit, but the requested close, reopen, and delete lifecycle actions are still absent from the shared message action names, Telegram action map/discovery, runtime dispatch, transport exports, config schema, docs, and contract tests. The related PR #41410 is useful deletion-only prior art in the provided context, but it is open/unmerged and does not cover close or reopen.

## Best Possible Solution

Keep #11414 open as the canonical Telegram forum topic lifecycle tracker. A complete fix should add close, reopen, and delete forum topic support through the shared message action contract, Telegram action discovery/map, runtime dispatch, transport helpers using the Telegram Bot API methods, config schema/metadata, public docs, and focused Telegram contract/runtime/transport tests. PR #41410 can be used as deletion prior art, but it should be expanded or linked as a partial follow-up rather than superseding this parent issue.
