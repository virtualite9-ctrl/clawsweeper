## Summary

Keep #11414 open. Current main implements Telegram forum topic creation and editing, but the requested close, reopen, and delete topic lifecycle actions are still absent from the shared message action list, Telegram action adapter/discovery, runtime dispatch, transport exports, config schema, docs, and tests. The related PR #41410 is open and deletion-only in the provided context, so it does not supersede this dedupe-parent issue.

## Best Possible Solution

Keep #11414 open as the canonical Telegram forum topic lifecycle tracker. A complete fix should add documented close, reopen, and delete topic actions through the shared message action names, Telegram action map/discovery, action runtime, send/runtime helper exports, config/schema metadata, docs, and Telegram contract/runtime tests. PR #41410 can remain useful prior art for deletion if updated and merged, but close and reopen still need first-class implementation.
