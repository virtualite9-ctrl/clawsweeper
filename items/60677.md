## Summary

Keep #60677 open. Current main still does not implement iMessage imsg-rpc tapback filtering: the payload type and parser lack reaction metadata, inbound processing has no tapback drop path, and debounced batches still merge raw entries without stripping reaction messages. The PR is a focused bug-fix attempt tied to open bug #60274, though it likely needs maintainer review against the current imsg v0.5.0 RPC fields before landing.

## Best Possible Solution

Keep the PR open, but ask for maintainer review/rework against the actual imsg RPC reaction contract. The best path is a narrow iMessage plugin fix that drops or safely ignores tapback reaction notifications before access-control side effects and before debounce merging can poison normal text, with regression tests for metadata-only reactions, text-pattern fallbacks, false positives, pairing-policy ordering, and mixed debounce batches. Leave full reaction-event forwarding to the separate #39031 product/API discussion.
