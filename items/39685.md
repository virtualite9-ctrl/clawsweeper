## Summary

Keep #39685 open. Current main still has lower-level SSRF hostname allowlist support and a narrow native Codex `tools.web.search.openaiCodex.allowedDomains` setting, but it does not expose or enforce the requested global/per-agent `allowedDomains`/`denyDomains` egress firewall for `web_fetch` or managed `web_search`.

## Best Possible Solution

Keep the issue open and implement a core web-tool network policy: add typed/schema/docs support for global and per-agent `allowedDomains` and `denyDomains`, resolve it consistently with other tool policies, map allowlists into `SsrFPolicy.hostnameAllowlist` for `web_fetch`, add denylist precedence, and define enforceable behavior for managed and provider-native `web_search`. Add focused tests for strict schema acceptance/rejection, allow/deny precedence, redirects, cache keys, wildcard matching, and provider limitations.
