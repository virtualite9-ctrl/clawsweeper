## Summary

Keep #63558 open. Current main still does not provide a Dreaming/Dreams tab agent selector, and the Dreaming UI/controller/backend path is still scoped to the default agent or an aggregate workspace view rather than a user-selected agent. Related #63829 is a broader open memory-wiki/per-agent plugin configuration issue and #68959 is a broader chat/web UI agent-switcher issue; neither supersedes this Dreaming-specific UI and gateway method gap.

## Best Possible Solution

Keep #63558 open. The likely fix belongs in the Control UI and Gateway doctor memory surface: add a Dreams/Dreaming selected-agent state and selector using agents.list, pass the selected agentId through loadDreamingStatus/loadDreamDiary/dream diary actions, update doctor.memory.* handlers and protocol/docs/tests to accept an optional agentId with default-agent fallback, and keep related memory-wiki per-agent isolation discussion linked to #63829 rather than treating it as a replacement for this UI bug.
