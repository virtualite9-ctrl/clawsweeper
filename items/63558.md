## Summary

Keep this issue open. Current main still lacks a Dreaming-tab agent selector and the underlying Dreaming controller/backend calls remain scoped to the default agent, so the requested multi-agent Dreaming inspection workflow is not implemented. Related #63829 and #68959 overlap with multi-agent memory/UI concerns, but neither supersedes this narrower Dreams tab selector plus doctor.memory.* agent-scoping work.

## Best Possible Solution

Keep this issue open. Add Dreams/Dreaming selected-agent state and a selector backed by agents.list, pass the selected agentId through loadDreamingStatus, loadDreamDiary, and all Dream Diary actions, update doctor.memory.* gateway handlers to accept an optional agentId with default-agent fallback, and add focused UI/controller/gateway tests. Leave the memory-wiki vault isolation concern to #63829 and the broader web UI agent switcher concern to #68959.
