## Summary

Keep this PR open. Current main still returns the Discord parent channel ID for configured ACP parent-channel matches, and the configured-binding registry materializes the session key from that matched conversation ref. That preserves the reported collision path where sibling Discord threads under one configured parent channel share one ACP binding session key. The open paired bug report #64199 and broader #50955 work do not supersede this focused PR.

## Best Possible Solution

Keep this PR open as the focused implementation candidate for the Discord configured ACP thread-session collision. The best path is a surgical Discord parent-match change that materializes the runtime thread ID while retaining `parentConversationId`, plus regression coverage proving two Discord thread IDs under the same configured parent produce distinct ACP session keys. Review the reverse-lookup limitation explicitly and coordinate with #64199 as the matching bug report and #50955 as broader ACP thread binding/routing work, but do not treat either as a merged supersession.
