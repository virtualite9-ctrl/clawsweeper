## Risks / Open Questions

- Changing Discord configured ACP session keys from parent-keyed to thread-keyed will start fresh ACP state for threads that previously shared the parent session.
- The reverse lookup path cannot reconstruct runtime thread IDs from the one-way hash, so reset and stateful-target behavior for thread-derived keys needs explicit review and tests.
- The PR lifecycle fallback treats a syntactically configured-format ACP session key as configured for metadata clearing; reviewers should confirm that boundary is acceptable for reset paths.
