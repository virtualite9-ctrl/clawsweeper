## Summary

Keep open. Current main still has `openclaw sessions` for listing and `openclaw sessions cleanup` for policy maintenance, but it does not expose the requested intentional delete commands (`sessions rm` / `sessions clear`). The linked PR #35735 remains useful related implementation work, but it is not present in the checked-out main tree at dc9ce2a1bf1cd5b6c3eae1ef8a18be6d71078b81.

## Best Possible Solution

Keep the issue open until maintainers either merge an implementation like PR #35735 or intentionally choose a different CLI design. The likely product path is to add `openclaw sessions rm <key>` and `openclaw sessions clear` on top of the existing session store/gateway delete behavior, with `--agent`, `--all-agents`, `--store`, `--dry-run`, and `--json` coverage plus docs and focused CLI tests.
