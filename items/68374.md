## Summary

Current main still does not expose Claude/Pi thinking as reasoning on either OpenResponses or OpenAI-compatible chat HTTP surfaces. The handlers serialize only assistant visible text, the streaming event schema has no reasoning delta/done events, and the ingress command options do not pass a reasoning visibility setting into the agent run. The related open PR #56674 appears to be the right implementation candidate to review rather than a reason to close this issue now.

## Best Possible Solution

Keep this issue open and use it to track a maintainer-reviewed implementation. The likely path is to add an explicit reasoning visibility path for OpenAI-compatible HTTP ingress, wire agent/session/per-agent reasoningDefault into those runs, preserve default-off behavior, emit structured reasoning output for /v1/responses in both non-stream and stream modes, decide the compatible shape for /v1/chat/completions, and update the OpenResponses/OpenAI HTTP docs plus focused gateway tests. PR #56674 should be reviewed as the current candidate or superseded by a narrower maintainer patch.
