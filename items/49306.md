## Summary

Current main already resolves the reported circular re-export problem through a broader refactor: LINE no longer routes setup/runtime helpers back through `extensions/line/api.ts`, and Nostr setup is provided directly from `src/plugin-sdk/nostr.ts` / `extensions/nostr/setup-api.ts`. The same implementation is present in the shipped `v2026.4.22` tree, so this PR is superseded on main.

## Close Comment

Codex review: closing as already implemented on `main`.

The circular plugin-sdk ↔ LINE/Nostr barrel path this PR targeted is no longer present in current OpenClaw. `src/plugin-sdk/line.ts:29` now routes the LINE helper surface through `src/plugin-sdk/line-surface.ts`, `extensions/line/setup-api.ts:1` exposes setup directly, and `extensions/line/api.ts:10` no longer re-exports the setup symbols. On the Nostr side, `src/plugin-sdk/nostr.ts:38` creates the setup surface directly and `extensions/nostr/setup-api.ts:1` is the setup entrypoint, so there is no longer a `src/plugin-sdk/nostr.ts -> extensions/nostr/api.ts` back-edge.

This replacement structure is already shipped in `v2026.4.22` at `00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`, so this PR has been superseded by the code now on `main`.
