## Summary

Keep open. Current main still finalizes Telegram text-only message previews by editing the existing preview in place, and it does not track preview visibility age or send a fresh final for long-lived previews. The PR appears to address a real remaining Telegram behavior rather than being implemented, duplicated, or out of scope.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is a focused Telegram change that tracks when message previews become visible, sends a fresh final only after the agreed stale-preview threshold, preserves short-preview edit-in-place behavior, and keeps the docs/changelog/test coverage from the PR. Split or drop unrelated compat/lockfile churn if it remains in the branch.
