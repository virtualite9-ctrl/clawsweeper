## Summary

Keep #72038 open. Current main still finalizes normal text-only Telegram preview-streamed replies by editing the existing preview message, and main has no visible-since metadata or long-lived preview threshold that could send a fresh completion-time final. The PR addresses a real bundled Telegram plugin behavior gap, and the provided review discussion shows the initial fallback-coverage concern was followed up by the author.

## Best Possible Solution

Keep #72038 open for maintainer review. The best path is to land a focused bundled Telegram plugin fix that records when message previews become visible, carries that timestamp through active/superseded/archived preview state, sends a fresh final message for previews older than the accepted threshold, cleans up the stale preview, preserves short-preview edit-in-place behavior, and keeps the active/archived/send-failure regression tests plus docs/changelog updates.
