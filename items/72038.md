## Summary

Keep #72038 open. Current main still finalizes Telegram text-only preview-streamed replies by editing the existing preview message, with no visible-age tracking or long-lived-preview branch that sends a fresh final. The PR remains a focused, plausible Telegram bug fix with active review evidence and current-head fallback coverage in the provided PR context.

## Best Possible Solution

Keep #72038 open for maintainer review and possible merge. The fix belongs in the bundled Telegram plugin: track when message previews become visible, send a fresh final for previews older than about one minute, preserve short-preview edit-in-place behavior, retain archived-preview fallback behavior, and land the focused Telegram regression tests/docs/changelog already represented by the PR.
