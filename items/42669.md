## Summary

Keep this PR open. Current main still does not implement `skills.priority`, and the open PR remains a plausible contributor implementation candidate rather than obsolete or superseded. The latest PR head still has a material ordering bug: it prioritizes entries, then alphabetically re-sorts prompt skills before count/character truncation, so pinned skills can still be dropped.

## Best Possible Solution

Keep the PR open and ask for a focused refresh: add `skills.priority` across the public config type, strict schema, generated schema, and docs; apply exact-name and `skillKey` priority ordering to the prompt skill list before all prompt limits without a later alphabetical re-sort; keep persisted snapshot metadata aligned with prompt order; invalidate reused snapshots when relevant skills config changes; and cover priority survival under both count and character truncation tests.
