## Summary

Keep this PR open. Current main still does not implement `skills.priority`, the PR remains an open contributor implementation candidate, and the latest PR head still has material correctness and snapshot-cache concerns that should be resolved before merge rather than closing the work as obsolete.

## Best Possible Solution

Keep this PR open and ask for a focused refresh: add `skills.priority` across the public config type, strict schema, generated schema, and docs; apply exact-name and `skillKey` priority ordering to prompt skills before every prompt limit with no later alphabetical re-sort; preserve deterministic ordering for non-priority skills; align persisted snapshot metadata with prompt order; invalidate reused snapshots when all snapshot-affecting skills config changes; and cover priority survival under both count and character truncation.
