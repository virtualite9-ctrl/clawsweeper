## Risks / Open Questions

- The PR should not land until priority ordering is applied before every count and character prompt limit and is not undone by a later alphabetical sort.
- Snapshot reuse is security/correctness-sensitive: a config key that only includes `skills.priority` would miss other selection-affecting fields such as `allowBundled`, `entries`, load roots, filters, and eligibility-affecting config.
- Treating missing persisted snapshot versions as reusable version `0` can preserve stale skill prompts or policies after restart or config changes.
- The supplied PR file list does not show workflow, lockfile, dependency-source, package-publishing, or release-metadata changes, so the supply-chain surface does not appear broadened; the main security concern is runtime snapshot reuse and credentials/policy alignment.
