## Summary

Keep #71335 open. Current main still resolves `agents.defaults.memorySearch.sync.watch` to `true` when neither defaults nor per-agent overrides set it, and the resolver has no gateway-mode/runtime context. The memory managers then create chokidar watchers whenever that resolved flag is true, so the requested gateway-mode default change is not implemented or superseded.

## Best Possible Solution

Keep the issue open and implement a narrowly scoped, tested mode-aware default. The likely path is to thread explicit runtime context into memory sync resolution, default `sync.watch` to `false` only for headless Gateway runtime when the user has not set either defaults or per-agent `sync.watch`, preserve explicit `sync.watch: true`, and update memory config docs/schema help plus tests for CLI/desktop versus gateway behavior.
