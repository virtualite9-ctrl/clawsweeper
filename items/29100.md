## Summary

Keep #29100 open. Current main still lacks a supported pricing overlay for auto-discovered Amazon Bedrock models: the strict Bedrock plugin discovery schema/type/schema expose discovery controls only, discovered foundation models still receive zero DEFAULT_COST, inference profiles only inherit that zero/default cost, and cost lookup/docs remain tied to explicit provider model entries. Related PR #31160 proposed this feature but was closed unmerged.

## Best Possible Solution

Keep #29100 open until OpenClaw adds a supported pricing overlay for auto-discovered Bedrock models. The likely implementation belongs either in plugins.entries.amazon-bedrock.config.discovery or in a general models.costs overlay, with manifest schema, TS types, zod/help/docs, discovery cost application for foundation models and inference profiles, cache-key handling, focused tests, and docs updates for https://docs.openclaw.ai/providers/bedrock and https://docs.openclaw.ai/reference/token-use.
