## Summary

Keep #29100 open. Current main still has no supported way to attach cost metadata to auto-discovered Amazon Bedrock models: the Bedrock discovery schema has no `costs` field, discovered foundation models are assigned zero `DEFAULT_COST`, inference profiles only inherit that zero/default cost, and the docs still document pricing only on explicit `models.providers.<provider>.models[].cost` entries. The related implementation PR #31160 is closed and unmerged in the provided GitHub context.

## Best Possible Solution

Keep #29100 open. The best implementation is a backward-compatible pricing overlay for auto-discovered Bedrock models, preferably owned by the Amazon Bedrock plugin unless maintainers choose a general discovered-model cost overlay. It should update the plugin manifest schema, TS/zod config types, config UI/help metadata, deterministic discovery cache behavior, foundation-model and inference-profile cost resolution, focused tests, and docs for https://docs.openclaw.ai/providers/bedrock and https://docs.openclaw.ai/reference/token-use. It should also verify how estimated costs appear for aws-sdk/IAM Bedrock in `/status`, `/usage full`, `/usage cost`, diagnostics, and transcript-backed summaries.

- A complete fix needs a product/API choice between a plugin-owned overlay such as `plugins.entries.amazon-bedrock.config.discovery.costs` and a general `models.costs` overlay that can patch discovered models across providers.
