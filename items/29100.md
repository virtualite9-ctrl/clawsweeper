## Summary

Keep #29100 open. Current main still lacks a supported cost metadata overlay for auto-discovered Amazon Bedrock models: the Bedrock plugin discovery schema and legacy config type/schema expose only discovery controls, discovered foundation models still get zero DEFAULT_COST, inference profiles only inherit that zero/default cost, and user-facing docs still document cost config only under explicit provider model entries. The related PR #31160 proposed the requested feature but was closed unmerged.

## Best Possible Solution

Keep the issue open until OpenClaw adds a supported pricing overlay for auto-discovered Bedrock models. The likely implementation should live in the Amazon Bedrock plugin-owned discovery config or a deliberate general models.costs overlay, with manifest/schema/type/help updates, cache-key handling, cost application for foundation models and inference profiles, focused discovery tests, and docs updates for https://docs.openclaw.ai/providers/bedrock and https://docs.openclaw.ai/reference/token-use. If the intended UX is dollar estimates for IAM/aws-sdk Bedrock sessions, the /status and /usage cost-display auth gating should be reviewed at the same time.
