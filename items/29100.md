## Summary

Keep #29100 open. Current main still lacks a supported pricing overlay for auto-discovered Amazon Bedrock models: the plugin-owned discovery schema and legacy Bedrock discovery contract expose only discovery controls, discovered foundation models still receive zero/default cost metadata, inference profiles only inherit that base/default cost, and public docs still describe cost estimation as coming from explicit provider model entries. The related PR #31160 directly attempted this feature but is closed unmerged, so it does not supersede the issue.

## Best Possible Solution

Keep the issue open until OpenClaw adds a supported pricing overlay for auto-discovered Bedrock models. The likely fix belongs in the Amazon Bedrock plugin-owned discovery config, or in a deliberate general model pricing overlay, and should update manifest/schema/types, apply costs to foundation models and inference profiles, include pricing in discovery cache semantics, add focused Bedrock discovery tests, and update the public Bedrock and token-use docs.
