## Summary

Keep #29100 open. Current main still does not expose a supported cost metadata overlay for auto-discovered Amazon Bedrock models. The plugin-owned discovery schema and legacy compatibility type/schema only accept discovery controls, discovered foundation models still receive zero DEFAULT_COST, inference profiles only inherit that base/default cost, and the cost display path still resolves pricing from explicit provider model entries rather than a discovery-level or top-level model-cost overlay. The related PR #31160 proposed this feature but is closed unmerged per the provided GitHub context.

## Best Possible Solution

Keep the issue open until OpenClaw adds a supported pricing overlay for auto-discovered Bedrock models. The likely fix belongs in the bundled Amazon Bedrock plugin-owned discovery config, or in a deliberate general models.costs overlay, with manifest/schema/type/help updates, discovery application for both foundation models and inference profiles, cache-key handling for pricing metadata, focused Bedrock discovery tests, and public Bedrock/token-use docs updates.
