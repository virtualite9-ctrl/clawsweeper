## Summary

Keep #8209 open. Current main still does not provide the requested native cron lifecycle surface for external Mission Control-style integrations: cron has an internal started/finished event, completion-only webhook delivery, completion-only run logs, failure destinations, and a gateway_start plugin context for cron inspection, but no documented/configurable start+completion webhook, telemetry, or plugin lifecycle subscription that covers both agentTurn and systemEvent jobs. The linked PR #8855 was closed unmerged and does not supersede the issue.

## Best Possible Solution

Keep #8209 open for explicit maintainer product/API direction. The conservative path is a generic typed cron lifecycle event surface, preferably a plugin hook or subscription API that emits start and finish events for both agentTurn and systemEvent jobs. If core webhooks are chosen instead, keep them generic, schema-validated, SSRF-guarded, documented, and covered by focused tests for start, success, failure, timeout, disabled jobs, and misconfigured destinations.
