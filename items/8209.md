## Summary

Keep #8209 open. Current main still has completion-oriented cron delivery, failure notifications, run logs, and internal task records, but it does not provide the requested native cron start+completion webhook/telemetry surface or a documented plugin cron lifecycle API covering both agentTurn and systemEvent jobs. The linked PR #8855 was closed unmerged, and a maintainer-side Codex review already called out the same remaining product/API gap.

## Best Possible Solution

Keep #8209 open for explicit maintainer product/API direction. The best path is a generic, typed cron lifecycle surface, preferably a plugin hook or runtime event subscription that emits start and finish events for both agentTurn and systemEvent jobs. If core webhooks are chosen instead, keep them generic, schema-validated, SSRF-guarded, documented, and covered by focused tests for start, success, failure, timeout, disabled jobs, and invalid destinations.
