## Summary

Keep this PR open. Current main still treats Slack interactive reply-button clicks as passive Slack interaction system events plus UI confirmation; it does not proactively dispatch or wake an agent turn. The PR remains a plausible implementation candidate for the unresolved Slack interaction wake gap, though it overlaps with #61374 and #61502 and needs maintainer choice on the canonical design.

## Best Possible Solution

Keep this PR open for maintainer review. Reconcile it with #61374 and #61502, choose the safest Slack interaction wake architecture, verify reply-mode/thread/session routing against the normal Slack inbound path, then land the chosen implementation with focused Slack interaction tests and the changed gate or deliberately close superseded PRs after a canonical path is selected.
