## Summary

Current `main` does not implement the sweep-backed reminder suppression from PR #52571. The active guard still suppresses only when an enabled cron job shares the same `sessionKey`, and the current reminder-guard tests still assert that jobs from another session should append the unscheduled-reminder note. That matches the bug this PR is trying to fix, so there is not strong evidence to close it.
