## Summary

Keep #29907 open. Current main at 406ae72fd2789c78053f6b94006ec8f488bc1c00 still has no cron `exec` or `lobster` payload kind, no `condition`/`onCondition` contract, and no deterministic non-LLM cron execution path. The related #18160 remains an open direct-exec subset and does not cover the Lobster pipeline, condition evaluation, silent no-op, or conditional agent handoff scope requested here.

## Best Possible Solution

Keep #29907 open unless maintainers create or designate a broader cron-executor tracker that explicitly includes deterministic `exec` and Lobster pipeline execution, timeout/security policy, bounded output capture, condition evaluation (`exitCode`, stdout, JSON path, always), silent no-op runs, conditional enqueue/delivery of an `agentTurn`, run-log/task-state visibility, CLI and Control UI support, public docs, and protocol/runtime tests. Alternatively, first add a stable plugin-facing cron executor seam if maintainers want this implemented outside core.
