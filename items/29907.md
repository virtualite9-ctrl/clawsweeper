## Summary

Keep #29907 open. Current main still only supports cron `systemEvent` and `agentTurn` payloads through the type contract, gateway schema, runtime execution path, CLI, and Control UI. Lobster exists, but only as an optional agent tool, so cron still needs an LLM-run agent to call it. The related #18160 direct-exec request overlaps the shell-command subset but does not fully cover Lobster pipelines, condition evaluation, silent no-op behavior, or conditional handoff into `agentTurn`.

## Best Possible Solution

Keep #29907 open unless maintainers explicitly consolidate it into a broader canonical cron-execution tracker. The best product path is an additive core cron/protocol feature, or a new stable cron-executor plugin seam first, covering deterministic `exec` and/or Lobster pipeline execution, timeout/security policy, bounded output capture, condition evaluation (`exitCode`, stdout, JSON path, always), silent no-op runs, conditional enqueue/delivery of an `agentTurn`, run-log/task-state visibility, CLI and Control UI support, public docs, and protocol/runtime tests.
