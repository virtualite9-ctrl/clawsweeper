## Summary

Keep #29907 open. Current main still only supports cron `systemEvent` and `agentTurn` payloads, so there is no no-LLM `exec`/`lobster` cron runner, condition DSL, silent no-op branch, or conditional handoff into an `agentTurn`. Lobster exists today, but only as an optional agent tool that cron can ask an LLM-run agent to call.

## Best Possible Solution

Keep #29907 open unless maintainers explicitly consolidate it into an expanded canonical cron-execution tracker. The best product path is an additive core cron/protocol feature for deterministic `exec` and/or Lobster execution with timeout/security policy, bounded output capture, condition evaluation (`exitCode`, stdout, JSON path, always), silent no-op results, conditional enqueue/delivery of an `agentTurn`, run-log/task-state visibility, CLI and Control UI support, public docs, and protocol/runtime tests.
