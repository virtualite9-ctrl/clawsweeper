## Summary

Keep #13607 open. Current main still does not implement a global `agents.defaults.reasoningDefault`; it only supports per-agent `agents.list[].reasoningDefault`, and the main reply/status resolution paths still fall back from session to per-agent default to `off` without consulting global defaults. The related open PR #71817 is partial channel work for Telegram/Feishu per-agent behavior and does not supersede this global-default request.

## Best Possible Solution

Keep #13607 open until OpenClaw adds a schema-backed `agents.defaults.reasoningDefault` with clear precedence: inline `/reasoning` and session override > per-agent `agents.list[].reasoningDefault` > global `agents.defaults.reasoningDefault` > model-capability/off fallback. The implementation should update config types, Zod schema, generated schema/help metadata, user docs, main reply resolution, status/current-level helpers, relevant channel adapters, and regression tests. Treat #71817 as useful partial per-agent channel work, not the canonical global-default implementation.
