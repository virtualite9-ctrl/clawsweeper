## Summary

Keep #13607 open. Current main still supports reasoning visibility defaults only at the per-agent `agents.list[].reasoningDefault` surface, not the requested global `agents.defaults.reasoningDefault`. The maintainer discussion already reached the same conclusion, and the related implementation PRs for the global key are closed unmerged while #71817 is explicitly only a partial per-agent Telegram/Feishu fix.

## Best Possible Solution

Keep #13607 open until OpenClaw either implements schema-backed `agents.defaults.reasoningDefault` with clear precedence (`/reasoning` directive > session override > per-agent default > global default > model capability/off), updates generated schema/help metadata and public docs, and adds regression coverage for runtime resolution/status/channel behavior, or a maintainer explicitly decides reasoning visibility should remain per-agent only. Treat #71817 as useful partial per-agent/channel work, not the canonical resolution for this global default request.
