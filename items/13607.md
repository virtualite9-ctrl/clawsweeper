## Summary

Keep #13607 open. Current main still does not implement the requested global `agents.defaults.reasoningDefault`; the shipped config surface supports only per-agent `agents.list[].reasoningDefault`, and the main reply path resolves reasoning from directive/session/per-agent/default rather than a global default. Related #71817 is useful but open and explicitly partial: it targets per-agent fallback behavior for Telegram/Feishu, not the canonical global default requested here.

## Best Possible Solution

Keep #13607 open until OpenClaw implements schema-backed `agents.defaults.reasoningDefault` with documented precedence such as `/reasoning` directive and session override > per-agent default > global default > model-capability/off fallback. The fix should update config types, zod schema, generated schema/help/docs/status behavior, reconcile the gateway-tool allowlist, and add regression coverage for the main reply path plus affected channel adapters. Treat #71817 as a useful partial Telegram/Feishu per-agent follow-up, not a global-default implementation.
