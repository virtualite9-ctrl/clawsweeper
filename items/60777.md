## Summary

Keep PR #60777 open. Current main still lacks the PR’s live-gateway memory fallback: when local status cannot materialize an enabled memory plugin, human status still renders the plugin as unavailable and JSON still has no gateway memory status surface. The linked bug #57256 remains plausible, and the PR branch addressed the posted JSON-output review gap with a second commit.

## Best Possible Solution

Keep #60777 open as useful focused fix work. The best maintainer path is to rebase/adapt it onto the current split status architecture, preserving both surfaces: human `openclaw status --deep` should trust a live gateway `doctor.memory.status.provider` when local memory is null, and JSON should expose the same fallback state. If maintainers prefer the broader #59685 direction, consolidate explicitly there rather than auto-closing this focused PR without review.
