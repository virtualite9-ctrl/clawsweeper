## Summary

The request asks for a shared multi-agent group transcript, but current main treats per-agent session isolation as intentional architecture. OpenClaw already exposes cross-session and cross-agent orchestration seams, and the repo vision says optional orchestration capability should usually ship as a plugin/skill via ClawHub rather than as a core session-model change.

## Risks / Open Questions

- No specific existing ClawHub plugin for Feishu multi-agent group collaboration was linked in the issue discussion, so this close assumes the existing session-tool and plugin seams are sufficient for third-party implementation.
- If maintainers want to change the core isolation model for multi-agent group chats on product grounds, that would be a new design decision rather than a cleanup close.
