## Summary

Current main no longer has the reported `createOpenClawTools(...) -> createExecTool(...)` mismatch. Exec is built through `createOpenClawCodingTools(...)`, which receives the active requester `sessionKey` from the embedded runner and passes that same field into the exec tool. The older `createOpenClawTools(...)` helper no longer constructs `exec` at all. Current tests also cover exec followup/session routing and fail-closed behavior when no session or route is available.
