## Summary

Keep open. Current main still does not contain the PR's internal health-probe close suppression: pre-handshake WebSocket closes are WARN unless they match only the SwiftPM helper filter. The linked probe path still creates a PROBE-mode GatewayClient and calls `client.stop()` after settling, so the reported clean probe close path remains relevant. The PR also addressed the review comments in its second commit, so it is not obsolete or clearly superseded.

## Best Possible Solution

Keep PR #60536 open. The best path is to land the targeted `src/gateway/server/ws-connection.ts` suppression after current-main rebase and add a narrow regression test proving loopback origin/code-1000 probe closes log at debug while external or non-1000 pre-handshake closes remain WARN. If landed, it should close #60510.
