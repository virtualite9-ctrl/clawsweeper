## Summary

Keep #69570 open. Current main still creates a separate async agent follow-up for each exec approval result, using the exact completion prompt from the report and a per-approval idempotency key. Related #69386 and #72062 overlap but do not supersede this canonical approval-completion cascade report; the current /approve shell-command guard only covers a narrower loop.

## Best Possible Solution

Keep this issue open as the canonical core bug. The best fix belongs in the exec approval follow-up/session scheduling path: coalesce multiple approval completions for the same session/logical task, let user stop/abort preempt or cancel pending follow-ups, cap denial/retry cascades, and add regression coverage around multiple approved/denied exec completions plus interleaved user input.
