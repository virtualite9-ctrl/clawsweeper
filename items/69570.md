## Summary

Keep #69570 open. Current main still has command-scoped async exec approval follow-ups: each approved or denied approval can resume the agent through its own follow-up prompt, and the existing queue/stop controls do not suppress future completions from already-approved async commands. The provided 2026-04-26 comment adds a fresh deterministic reproduction on 2026.4.21 and links adjacent work, but no current-main implementation fully addresses the task-level approval/completion cascade.

## Best Possible Solution

Keep #69570 open as the canonical tracker for task-level exec approval/completion cascade control. The likely fix should add a session/task-level cancellation or suppression token checked by exec approval follow-up dispatch, coalesce or batch multiple approval completions for the same logical task, and make new user stop/interrupt inputs preempt queued system/agent follow-ups. Cross-link #69386 for the deny/stop retry symptom and #72062 for the separate wrapper-policy mitigation, but do not treat either as replacing this core queue/cancellation bug.
