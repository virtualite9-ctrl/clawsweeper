## Summary

PR #72306 should stay open. Current main still has the gateway pricing refresh lifecycle gap the PR targets: the 24-hour refresh timer is scheduled as a normal ref'd timeout, pricing fetches only use timeout abort signals, and the returned stop callback only clears the scheduled timer without aborting an in-flight refresh. The linked regression issue #72208 remains a concrete user report, and the provided PR discussion plus bot review describe a focused, test-backed fix rather than an obsolete or low-signal change.

## Best Possible Solution

Keep #72306 open for maintainer review and likely merge, or land an equivalent minimal fix in src/gateway/model-pricing-cache.ts that unrefs the 24-hour pricing refresh timer, aborts in-flight LiteLLM/OpenRouter pricing fetches from the stop callback, and keeps the proposed regression tests in src/gateway/model-pricing-cache.test.ts. It should not be closed as implemented, duplicate, stale, or ClawHub work.
