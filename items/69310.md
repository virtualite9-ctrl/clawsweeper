## Summary

Keep open. Current main implements only a narrow generic warning when every reply media item is dropped; it still lacks this PR's per-item dropped-media contract, mixed-media visibility, catastrophic normalization reporting, outbound JSON/transport propagation, routing/followup preservation, and transcript notice path. The linked issue was closed as implemented for the narrower all-dropped case, and related #70109 is closed unmerged, so neither supersedes this active PR. The PR is not ready to land as-is because current PR CI has a relevant auto-reply failure and the diff still shows cleanup concerns, but it remains plausible useful work rather than an auto-close candidate.

## Best Possible Solution

Keep this PR open, but require a maintainer-reviewed scope decision. The best path is either to trim it to the narrower shipped `⚠️ Media failed.` behavior and close as superseded by main, or to continue the broader implementation after fixing CI/type/lint issues and validating the privacy, hook, transcript, mixed-media, and followup-routing semantics. The broader dropped-media contract belongs in core only if maintainers want per-item user-visible diagnostics across all reply/outbound paths.
