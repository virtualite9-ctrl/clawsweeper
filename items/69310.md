## Summary

Keep open. Current main only has a minimal generic warning when every normalized reply media item fails; it does not implement the PR's dropped-media pipeline, per-item user notices, JSON/transcript propagation, or route/followup preservation. The related #70109 PR is closed unmerged, so it does not supersede this active PR.

## Best Possible Solution

Keep the PR open. The best path is a maintainer-reviewed, possibly scoped version of the dropped-media/fail-loud pipeline that preserves sanitized per-item failure details through reply media normalization, catastrophic fallback, route/followup filtering, outbound JSON/delivery, and transcript mirror paths while accounting for the review feedback already discussed on the PR.
