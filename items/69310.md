## Summary

Keep open. Current main has a narrower generic warning only when every reply media item is dropped, but #69310 still covers unresolved behavior: partial media drops are still omitted silently, the catastrophic media-normalization catch still strips media without a user-visible notice, and main lacks the PR's structured droppedMedia pipeline for per-item sanitized notices, JSON projection, routing, and transcript visibility. The related #70109 PR is closed and unmerged, so it is not a canonical replacement.

## Best Possible Solution

Keep #69310 open for maintainer review. Rebase or refactor it against current main's simpler `⚠️ Media failed.` implementation, then decide whether to land a targeted fix for partial drops and the catastrophic normalization catch or keep the broader structured `droppedMedia` contract for JSON/transcript propagation and per-item sanitized notices.
