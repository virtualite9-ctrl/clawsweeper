## Summary

Keep open. Current main at 110fa97 only has a generic warning when every reply media item fails normalization; it does not implement the PR’s dropped-media contract, per-item sanitized notices, JSON/outbound/transcript propagation, or preservation through route/followup paths. The related #70109 PR is closed and unmerged in the provided context, so it does not supersede this active PR.

## Best Possible Solution

Keep the PR open. The best path is a maintainer-reviewed, possibly scoped implementation of the dropped-media/fail-loud pipeline: preserve sanitized per-item failure details through reply media normalization, catastrophic fallback, route/followup filtering, outbound JSON/delivery, and transcript mirroring, while retaining the privacy and delivery-safety fixes already discussed in the PR review thread.
