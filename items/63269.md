## Summary

Keep #63269 open. Current main contains the expected Mattermost public/channel `posted` WebSocket handling and mention-gating path, but I did not find a concrete current-main fix for the reported 2026.4.8 regression. The reported bad release and reported working release appear to have the same Mattermost WebSocket `posted` handler, and the closest related open items cover broader/stale WebSocket health rather than this channel-only symptom where DMs still work.

## Best Possible Solution

Keep the issue open and reproduce against the latest release/current main with the reporter's config. If channel `posted` frames arrive, add a targeted monitor/WebSocket regression test for a public-channel @mention with `chatmode: "oncall"` and `groupPolicy: "open"`, then fix any access/mention/routing drop. If frames do not arrive while REST sees posts, route the remaining work to Mattermost WebSocket health/subscription diagnostics rather than closing this channel-only report as already implemented.
