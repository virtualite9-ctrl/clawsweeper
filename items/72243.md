## Summary

Current main fixes the Docker Bonjour bootloop reported in #72243: Bonjour now classifies/suppresses CIAO probing cancellations through scoped process handlers that persist across advertiser restarts, and the bundled Docker path disables Bonjour by default with OPENCLAW_DISABLE_BONJOUR=1.

## Best Possible Solution

Keep the current main implementation: suppress known @homebridge/ciao cancellation/assertion failures, bound Bonjour restart attempts, and use the documented OPENCLAW_DISABLE_BONJOUR/plugin-disable controls for Docker bridge deployments rather than adding an undocumented OPENCLAW_PLUGINS_EXCLUDE bootstrap path.
