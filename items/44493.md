## Summary

Close #44493 as not actionable in OpenClaw. The remaining requested change is Paperclip-owned agent creation/onboarding and adapter configuration; current OpenClaw main documents and implements its own gateway auth contract, but this repo does not contain Paperclip's `openclaw_gateway` adapter or Hire Agent flow.

## Best Possible Solution

Close this OpenClaw issue and handle the remaining work in Paperclip: its `openclaw_gateway` adapter/onboarding should collect and persist the gateway token, then send it as OpenClaw `connect.params.auth.token` (or password when configured). If that work exposes a concrete missing OpenClaw API, open a new focused issue with adapter traces and the exact protocol gap.

## Evidence
