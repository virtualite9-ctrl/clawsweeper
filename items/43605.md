## Summary

Keep open. This issue is paired with open same-author implementation PR #43656, whose body uses closing syntax for this issue, so the issue should remain open until that PR is reviewed, merged, or closed. Current main still lacks native cross-gateway sessions_send routing: the tool schema has no gateway target fields, session resolution is local-store based, and the gateway protocol/handler accepts a local session key only. Related #55061 was already closed as a duplicate of this canonical tracker; the xihe-jianmu-ipc MCP hub discussed in #55872/#55902 is a useful workaround but does not replace the native request.

## Best Possible Solution

Keep this issue open as the canonical native cross-gateway sessions_send tracker. Maintainers should review, revise, land, or close PR #43656; if that PR is not the right shape, a replacement should add an explicit, documented, secure peer/remote-gateway contract across config schema, gateway protocol, session tool schemas, auth/token resolution, tests, and docs.
