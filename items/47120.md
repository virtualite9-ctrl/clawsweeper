## Summary

Keep open. Current main still does not display last-message previews in the Control UI Sessions list: the UI controller does not request `includeLastMessage`, the UI row type has no `lastMessagePreview` field, and the Sessions table renders only the key/display name plus the existing Updated column. The backend has a supported `includeLastMessage` path, so the right fix is still a UI wiring/rendering change, likely informed by the linked PRs #47243 and #69845.

## Best Possible Solution

Keep #47120 open until a focused UI change lands. The best path is to wire the Control UI sessions controller to request or fetch bounded last-message preview data, add the field to the UI session row type, render the timestamp/preview under the session key without losing the existing Updated column, and cover it with controller/view tests. The existing backend `includeLastMessage` support should be reused rather than adding another core API unless performance requirements demand a different batching shape.
