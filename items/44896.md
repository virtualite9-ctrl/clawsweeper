## Summary

Keep open. Current main still disables Moonshot thinking for pinned tool_choice while thinking is enabled, and the current tests assert that behavior. That does not implement the issue's requested fix and still leaves the reported conflicting payload shape possible when reasoning_effort is present upstream or from config.

## Best Possible Solution

Keep the issue open and implement a targeted Moonshot wrapper fix. The most consistent path with the docs and prior discussion is to normalize every incompatible enabled-thinking tool_choice, including pinned tool/function choices, to "auto" and add a regression test that a payload with reasoning_effort plus pinned tool_choice does not leave thinking disabled. If maintainers intentionally want to preserve pinned tool_choice instead, the wrapper should delete reasoning_effort when it disables thinking and the Moonshot docs/tests should be updated to match.
