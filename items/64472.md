## Summary

Keeping this issue open. Current main still enforces workspace-root containment for guarded workspace bootstrap reads, so symlinks from the workspace to sibling directories under `~/.openclaw` continue to be treated as boundary escapes and rendered as missing bootstrap files. The issue is also paired with an open same-author PR that uses closing syntax, so maintainer cleanup should not close the issue before that PR is reviewed, landed, or rejected.

## Best Possible Solution

Keep this issue open and review #64473 as the active same-author implementation candidate. If maintainers accept `~/.openclaw` sibling directories as trusted for workspace bootstrap reads, implement the relaxation in the boundary/workspace-read layer with explicit regression coverage for `~/.openclaw/workspace/AGENTS.md` pointing at `~/.openclaw/kb/AGENTS.md`, while preserving rejection of arbitrary outside targets. If that trust-boundary relaxation is rejected, close or revise the linked PR with a clear security rationale and consolidate the remaining user problem under the broader workspace-bootstrap symlink thread.
