## Summary

Keep this issue open. Current main still reads workspace bootstrap files through the workspace-root boundary, still rejects symlink targets whose canonical path leaves that boundary, and then renders failed guarded reads as missing bootstrap files. The same-author implementation PR #64473 is still an open/unmerged candidate, and the broader #40210/#40230 thread is also unresolved.

## Best Possible Solution

Keep this issue open and review #64473 as the same-author implementation candidate, coordinated with the broader #40210/#40230 workspace-bootstrap symlink thread. If maintainers accept ~/.openclaw sibling directories as trusted for bootstrap reads, implement the relaxation narrowly with tests for an AGENTS.md symlink to a sibling ~/.openclaw/kb target plus rejection of arbitrary outside targets. If that trust-boundary change is rejected, close or revise the PR with a clear security rationale and consolidate the remaining user-facing problem under the canonical issue.
