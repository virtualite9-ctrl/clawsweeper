## Summary

Keep this issue open. Current main still routes workspace bootstrap reads through the workspace-root boundary and still rejects symlink targets whose canonical path leaves that boundary; the same-author implementation PR #64473 remains open and is written to close this issue, so the issue should not be closed before that PR is reviewed, merged, or rejected.

## Best Possible Solution

Keep this issue open and review #64473 as the same-author implementation candidate, alongside the broader #40210/#40230 workspace-bootstrap symlink thread. If maintainers accept ~/.openclaw sibling directories as trusted for workspace bootstrap reads, implement the relaxation narrowly and add regression tests for an AGENTS.md symlink to a sibling ~/.openclaw/kb target plus rejection of arbitrary outside targets. If the trust-boundary change is rejected, close or revise the PR with a clear security rationale and consolidate the user-facing problem under the broader canonical issue.
