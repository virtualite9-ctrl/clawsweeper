## Summary

Keep open. Current main still rejects workspace bootstrap symlinks whose canonical targets are outside the workspace root, then renders the failed guarded read as a missing bootstrap file. The provided GitHub context also shows same-author PR #64473 is open and unmerged with closing syntax for this issue, so the issue should stay open until that PR is reviewed, landed, or rejected.

## Best Possible Solution

Keep this issue open and review #64473 as the active implementation candidate. If maintainers accept `~/.openclaw` sibling targets as trusted for workspace bootstrap reads, update `src/infra/boundary-path.ts` and add focused regression coverage for a workspace bootstrap symlink targeting `~/.openclaw/kb/AGENTS.md` while preserving rejection of arbitrary outside targets. If that trust-boundary relaxation is rejected, close or revise the PR with a clear security rationale and route broader symlink behavior through the existing related workspace-symlink thread.
