## Summary

Keep PR #72213 open. Current main still has the Discord block-preview duplicate tool-progress path: `pushPreviewToolProgress` only drops the immediately previous exact string, while active callbacks can add the same tool as both `tool: session_status` and `session_status`. The PR is a focused bug fix for linked issue #72205 with a targeted regression test, and I found no current-main implementation or superseding item that makes it obsolete.

## Best Possible Solution

Keep PR #72213 open for normal review. The best code path is to land this narrow Discord plugin fix, or an equivalent implementation, with the regression test in `extensions/discord/src/monitor/message-handler.process.test.ts` so block-preview tool progress deduplicates by logical tool name rather than exact adjacent preview text.
