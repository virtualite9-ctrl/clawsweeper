## Risks / Open Questions

- Closing now would discard the only open implementation candidate for a still-missing Discord same-token multi-instance routing bug.
- The PR still needs maintainer review for correctness: `refreshDiscordClaims` reads, mutates, and rewrites one shared file without cross-process locking, so concurrent instances can drop each other's fresh claims.
- The latest PR head still has claim-matching gaps noted by review: native command lookups do not pass a botId, preflight queries `message.channelId` rather than the already-resolved `messageChannelId`, and claim building records only numeric channel keys without filtering disabled or denied channel entries.
- The new shared claims file stores stable Discord guild/channel/bot identifiers locally and affects access-routing decisions before local allowlist denial, so the final design should be deliberate about file permissions, stale data, and multi-bot isolation.
