## Summary

Keep open. Current main still lacks a Discord cross-instance channel ownership registry, while this PR is a recent, focused implementation candidate for the same-token multi-instance routing bug. It should get maintainer review or replacement rather than conservative cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review or replacement. The best path is a Discord-plugin-owned fix under `extensions/discord/src/monitor` that performs robust per-instance and per-bot ownership checks before local allowlist denial in message preflight, autocomplete, and native slash-command dispatch, with tests for same-token multi-instance routing, guild-wide and channel-scoped allowlists, wildcard and name-based entries, denied/disabled channels, group DMs, member restrictions, lifecycle cleanup, and shared-file concurrency or a safer per-instance claim design.
