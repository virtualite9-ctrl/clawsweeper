## Summary

Keep this PR open. Current main still has no Discord cross-instance channel ownership registry, and guild slash/autocomplete/message paths still make local allowlist decisions before any sibling-instance ownership check. The PR is a recent, targeted implementation candidate for the same-token multi-instance Discord routing bug, but it needs maintainer review before merge because its shared claims file changes routing behavior and still has correctness/security-sensitive edge cases.

## Best Possible Solution

Keep this PR open for maintainer review or replacement. The best path is a Discord-plugin-owned fix under `extensions/discord/src/monitor` that performs robust per-instance and per-bot ownership checks before local allowlist denial in message preflight, autocomplete, and native slash-command paths, with tests for same-token multi-instance routing, guild-wide and channel-scoped allowlists, bot scoping, denied/disabled channels, wildcard and name-based channel entries, resolved channel IDs, shared-file concurrency or per-instance claim files, and lifecycle cleanup.
