## Summary

Keep #16142 open. Current main still routes main-session cron reminder/system-event text through the heartbeat model call path, and existing tests assert that behavior. The prior maintainer/Codex comment also explicitly kept the issue open after checking the same surface, and the linked #15403 PR is unrelated to this API-cost complaint.

## Best Possible Solution

Keep #16142 open until main-session cron reminders can bypass `getReplyFromConfig` for simple relay/formatting cases, while preserving model-mediated behavior for agentic cron events. The likely implementation belongs around `executeMainSessionCronJob`, `resolveHeartbeatRunPrompt`, or a small deterministic delivery helper, with regression tests proving simple reminders do not call the model and docs explaining when to use isolated jobs or heartbeat cost mitigations.
