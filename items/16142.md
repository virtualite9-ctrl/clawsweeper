## Summary

Keep #16142 open. Current main still sends main-session cron/systemEvent reminders through the heartbeat model path: cron enqueues the reminder, heartbeat builds a scheduled-reminder prompt, and runHeartbeatOnce calls getReplyFromConfig. The existing same-day Codex/maintainer review kept it open, and there is no relevant diff in the cron/heartbeat files from that reviewed SHA to current main.

## Best Possible Solution

Keep #16142 open until main-session cron systemEvent reminders have a deterministic delivery path for simple reminder text that avoids getReplyFromConfig, while preserving the model path for events that need reasoning. The likely implementation belongs around executeMainSessionCronJob, resolveHeartbeatRunPrompt, or a small delivery helper, with tests proving simple reminders avoid the model and docs clarifying deterministic bypasses versus cost mitigations.
