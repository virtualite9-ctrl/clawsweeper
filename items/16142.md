## Summary

Keep #16142 open. Current main still routes main-session cron/system-event reminders through the heartbeat model path: cron enqueues the reminder as a system event, heartbeat builds a cron relay prompt, and `runHeartbeatOnce` calls `getReplyFromConfig`. Existing docs and tests describe cost mitigations such as `isolatedSession`, `lightContext`, and `target: "none"`, but they do not implement the requested deterministic string-formatting bypass for simple reminders.

## Best Possible Solution

Keep the issue open until main-session cron reminders have a deterministic delivery path for simple `systemEvent` reminders that does not call `getReplyFromConfig`, while preserving model-mediated behavior for cases that need reasoning. The likely implementation belongs around `executeMainSessionCronJob`, `resolveHeartbeatRunPrompt`, or a small delivery helper, with regression tests proving simple reminders avoid the model and docs explaining when `lightContext`/`isolatedSession` remain cost mitigations rather than bypasses.
