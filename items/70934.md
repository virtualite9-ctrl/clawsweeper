## Summary

Keep #70934 open. Current main has useful partial fast paths, including QuickStart defaults, skip flags, and an end-of-setup hatch into TUI/Web UI, but it does not implement the requested value-first onboarding flow. The documented and coded first-run path still performs setup before the first useful run, and the first agent bootstrap ritual still asks for identity/vibe/emoji/persona-style details before normal use.

## Best Possible Solution

Keep the issue open for maintainer product work. A good implementation would make first run show a small set of useful example tasks/templates, run one quick result through Web UI or TUI after only the minimum provider/gateway setup, and move agent name, vibe, emoji/avatar, reply style, channel preferences, and SOUL.md personalization behind an explicit later customization step. Add prompt-order tests and update docs for the new default sequence, while preserving skip/non-interactive paths for automation.
