## Summary

Keep this PR open. Current main still reproduces the reported Discord thread-reply gap: `send` accepts `media`, `path`, and `filePath`, while `thread-reply` still reads only `media` and does not forward media-local-root options into the runtime call. The linked bug #43570 is still open, same author, and this PR remains the focused implementation candidate rather than obsolete cleanup.

## Best Possible Solution

Keep this PR open and update it against current main. The best patch is still small: in `extensions/discord/src/actions/handle-action.guild-admin.ts`, make `thread-reply` read `media ?? path ?? filePath`, pass the same media options into `handleDiscordAction` that `send` uses, and add focused current-location tests for `filePath`, `path`, precedence, and mediaLocalRoots passthrough before landing it with #43570.
