## Summary

Keep this PR open. Current main still has the reported Discord `thread-reply` attachment gap: the handler only reads `media`, while Discord `send` supports `media`, `path`, and `filePath`. The PR is also paired with open same-author bug #43570, so it remains an implementation candidate rather than conservative cleanup.

## Best Possible Solution

Keep this PR open and update it against current main. The best fix is narrow: in `extensions/discord/src/actions/handle-action.guild-admin.ts`, make `thread-reply` read `media ?? path ?? filePath`, forward the same scoped media options expected by the Discord send/runtime path, and add current Discord action-handler tests for alias behavior, precedence, and media access passthrough before landing it with #43570.
