## Summary

Keep open. Current main has a generic before-deliver hook path, but the reported Telegram streaming-preview path still bypasses a blocking/modifying message hook for partial and final preview edits. The related Signal comment is also consistent with current same-surface Signal delivery bypassing message_sending.

## Best Possible Solution

Keep the issue open and implement a generic blocking/modifying delivery seam for preview-stream updates and final preview finalization. The narrowest likely path is to route Telegram draft preview send/edit/finalize through a hook-aware before-deliver path with metadata such as isStreamingEdit and isFinal, then add regression tests proving message_sending can cancel/rewrite Telegram streaming final edits and that same-surface Signal delivery runs the same outbound hook contract. If maintainers prefer a new hook name, add it as a documented additive SDK hook and wire channel preview paths through it.

## Risks / Open Questions

- Closing this now would leave the reported safety use case unresolved for Telegram partial streaming because sensitive text can still be posted or edited into the chat before any blocking hook runs.
- A fix needs careful channel-contract design so message_sending semantics, message_sent semantics, preview edits, and same-surface channel deliveries stay consistent across Telegram, Signal, Discord, and other channels.
