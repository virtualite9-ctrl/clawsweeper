## Summary

Current main has related Kimi/OpenRouter malformed tool-call argument repair, but it does not prove this issue is fixed. The reported shape is a full JSON object duplicated in `partialArgs`; current OpenAI-completions streaming still concatenates every `toolCall.function.arguments` chunk, preserves `partialArgs` on the completed block, and the repair helper rejects a second full JSON object as an overlong suffix. No current regression test covers the exact duplicate-object isolated cron path.

## Best Possible Solution

Keep this open and fix it in the OpenAI-compatible streaming/tool-call boundary, with a focused regression test for an isolated `agentTurn` cron run using an OpenRouter/Kimi-shaped stream that emits the same complete arguments twice. The fix should either detect cumulative/repeated full argument payloads before appending or repair duplicate balanced JSON objects deterministically, and final assistant/tool-call blocks should not retain malformed `partialArgs` scratch state.
