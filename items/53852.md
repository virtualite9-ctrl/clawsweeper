## Summary

Keep this issue open. Current main still routes isolated `agentTurn` cron jobs through the embedded OpenAI-compatible streaming path, that path still blindly appends every function-argument chunk into `partialArgs`, and the existing repair wrapper still rejects a duplicated complete JSON object as unsafe trailing content rather than normalizing it.

## Best Possible Solution

Keep this issue open. The best fix belongs at the OpenAI-compatible streaming/tool-call boundary: detect cumulative or repeated complete argument payloads before appending, or safely collapse identical duplicated balanced JSON objects; strip scratch `partialArgs` from completed tool-call blocks; and add a focused regression test for isolated `agentTurn` cron using an OpenRouter/Kimi-shaped stream that emits the same complete arguments twice.
