## Summary

Keep open. Current `main` still appends streamed OpenAI-completions tool-call argument chunks directly into `partialArgs`, and the malformed-argument repair path only tolerates very short trailing junk. A duplicated full JSON object like the report shows would still stay malformed, and isolated cron `agentTurn` runs still execute through this embedded-agent path.

## Evidence

- **OpenAI-completions path still concatenates raw argument deltas:** `processOpenAICompletionsStream` initializes `partialArgs` and then does `currentBlock.partialArgs += toolCall.function.arguments` before reparsing. There is no dedupe/reset logic for repeated full argument payloads.
  - file: [src/agents/openai-transport-stream.ts:1188](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/agents/openai-transport-stream.ts#L1188)
  - command: `nl -ba src/agents/openai-transport-stream.ts | sed -n '1188,1210p'; nl -ba src/agents/openai-transport-stream.ts | sed -n '1348,1396p'`
