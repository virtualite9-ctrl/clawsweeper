## Summary

Keep this issue open. Current main still matches the reported failure class: isolated `agentTurn` cron runs can reach the embedded OpenAI-compatible stream path, that path still concatenates every tool-call argument chunk into `partialArgs`, completed tool-call blocks still retain that scratch field, and the repair wrapper still rejects an identical second complete JSON object as unsafe trailing content rather than normalizing it.

## Best Possible Solution

Keep this issue open. The best fix belongs at the OpenAI-compatible streaming/tool-call boundary: detect cumulative or repeated complete argument payloads before appending, or safely collapse identical duplicated balanced JSON objects, strip scratch `partialArgs` from completed tool-call blocks, and add a focused regression test for isolated `agentTurn` cron using an OpenRouter/Kimi-shaped stream that emits the same complete arguments twice.
