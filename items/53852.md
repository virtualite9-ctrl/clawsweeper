## Summary

Current main still matches the reported failure shape: isolated cron agentTurn runs use the embedded agent path, OpenAI-compatible tool-call argument chunks are appended directly into partialArgs, completed tool-call blocks retain that scratch field, and the repair wrapper rejects a duplicated complete JSON object as an invalid suffix. Keep this open.

## Best Possible Solution

Keep this open and fix the OpenAI-compatible streaming/tool-call boundary. The best fix should either detect cumulative/repeated complete argument payloads before appending or deterministically repair duplicated balanced JSON objects, strip scratch partialArgs from completed tool-call blocks, and add a focused regression test for isolated agentTurn cron using an OpenRouter/Kimi-shaped stream that emits the same complete arguments twice.
