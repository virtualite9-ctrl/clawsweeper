## Summary

Keep #11894 open. Current main has partial internal plumbing for selected auth profiles and internal failover logging, but the user-facing request remains unsatisfied: transcript/session usage records do not persist the actual auth profile per model call, `/usage cost` and session summaries aggregate by provider/model rather than auth profile, and `session_status` only shows current/active auth mode labels and estimated cost, not per-call or per-profile attribution. The provided April 26, 2026 maintainer comment also explicitly kept this issue open; the burn0 workaround and closed #55379 cover network-level cost visibility only, not OpenClaw auth-profile attribution.

## Best Possible Solution

Keep this issue open until OpenClaw persists the resolved authProfileId for each model call in transcript/session-safe metadata, flows that field through the session cost readers and `/usage cost` or `session_status` as per-profile cost totals, and emits or stores a clear user-visible profile rotation/fallback event. The existing internal lastProfileId/context-engine plumbing and failover observation logs can be reused, but burn0 remains only an interim network-level cost workaround and does not replace native profile attribution.
