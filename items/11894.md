## Summary

Keep #11894 open. Current main has native per-response/session cost surfaces and internal auth-profile rotation plumbing, but it still does not persist or expose the resolved auth profile for each completed model call, aggregate costs by auth profile, or emit a user-facing auth-profile fallback event. The existing burn0 discussion/#55379 is a useful interim cost workaround, but it does not solve OpenClaw-level auth profile attribution.

## Best Possible Solution

Keep #11894 open until OpenClaw records the resolved authProfileId with each model-call usage record, preserves a session-safe form in transcript/metadata or diagnostics, aggregates session_status or /usage cost totals by auth profile, and emits/stores a clear auth-profile rotation/fallback event. Existing runner auth-profile plumbing, cost-summary code, and model fallback lifecycle events are the right starting points; burn0 can remain an interim external cost tool but is not a replacement for native profile attribution.
