## Summary

Keep #11894 open. Current main has transcript-backed token/cost reporting and auth-profile rotation, but completed model-call usage is still keyed by provider/model/session surfaces, not by the resolved auth profile. The supplied issue discussion also already frames burn0/#55379 as an interim cost workaround, not a native replacement for OpenClaw auth-profile attribution.

## Best Possible Solution

Keep #11894 open until OpenClaw records the resolved authProfileId for each completed model call in safe metadata, exposes it through transcript/usage/status or diagnostics, aggregates costs by auth profile, and emits a clear user-facing auth-profile rotation/fallback event. burn0/#55379 can remain an external cost workaround, but it is not a native replacement for OpenClaw auth-profile attribution.
