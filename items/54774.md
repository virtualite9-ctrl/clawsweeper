## Summary

Keep open. Current `main` still treats a pre-`hello-ok` gateway close as terminal for one-shot callers, and the PR's attempt-scoped, capped suppression logic is not present on `main` or in current tests.

## Evidence

- **`callGateway` still fails immediately on close:** On current `main`, `executeGatewayRequestWithScopes()` sets `ignoreClose` and rejects on any `onClose` callback. There is no special case for a transient `(1000, "")` close before `hello-ok`.
  - file: [src/gateway/call.ts:540](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/src/gateway/call.ts#L540)
  - command: `nl -ba src/gateway/call.ts | sed -n '470,560p'`
  - sha: [c070509b7f8f](https://github.com/openclaw/openclaw/commit/c070509b7f8fdbe479e5d829643782c272e30e83)
- **`GatewayClient` still reports every close before reconnect:** On current `main`, the WebSocket `close` handler always flushes pending errors, then schedules reconnect, then fires `onClose`. There is no pre-hello transient-close suppression, no attempt-scoped filtering, and no cap for repeated `(1000, "")` closes.
  - file: [src/gateway/client.ts:303](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/src/gateway/client.ts#L303)
  - command: `nl -ba src/gateway/client.ts | sed -n '140,380p'`
  - sha: [c070509b7f8f](https://github.com/openclaw/openclaw/commit/c070509b7f8fdbe479e5d829643782c272e30e83)
- **Startup callers depend on `onHelloOk` vs close/error:** `serveAcpGateway()` waits for `onHelloOk` and rejects readiness from `onConnectError` or `onClose`, which matches the PR discussion that pre-hello close handling is user-visible for one-shot/startup flows.
  - file: [src/acp/server.ts:66](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/src/acp/server.ts#L66)
  - command: `nl -ba src/acp/server.ts | sed -n '1,220p'`
  - sha: [c070509b7f8f](https://github.com/openclaw/openclaw/commit/c070509b7f8fdbe479e5d829643782c272e30e83)
- **Current tests do not cover the proposed regression:** Current `main` has close and timeout tests in `src/gateway/call.test.ts`, but no `close-then-hello` case, and the current `src/gateway/client.test.ts` section shown on `main` has no reconnect-behavior coverage for transient pre-hello `(1000, "")` closes.
  - file: [src/gateway/call.test.ts:731](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/src/gateway/call.test.ts#L731)

- **PR discussion shows useful follow-up, not a close condition:** The review thread identified two concrete risks in the original patch: shared-boolean suppression could race across reconnects, and unlimited suppression could hide repeated failures. The author replied that commit `51f15f04bbf7081d83279d9a65fbea1b6511b05f` switched suppression to connect-attempt scoping and added a cap of 3 transient pre-hello closes. That makes the PR more credible, but it is still not implemented on current `main`.
