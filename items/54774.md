## Summary

Keep this PR open. Current main still fails the one-shot gateway call on the first pre-hello close, and the PR is a focused gateway fix with matching tests. The bot review concerns were addressed in the PR discussion by moving suppression to a connect-attempt set and capping repeated pre-hello normal-close suppression, so this remains a plausible implementation candidate rather than cleanup fodder.

## Best Possible Solution

Keep this PR open for maintainer review and consolidate it with #54475. The best implementation path is the narrow gateway fix shown here: track `hello-ok` in `src/gateway/call.ts`, suppress only pre-hello code-1000 empty-reason closes in `src/gateway/client.ts`, keep the suppression bounded and keyed to a specific connect attempt, and retain focused regression tests in `src/gateway/call.test.ts` and `src/gateway/client.test.ts`. After one branch is chosen and lands, close the other overlapping PR as superseded.
