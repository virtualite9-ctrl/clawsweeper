## Summary

Keep PR #60808 open. Current main still has the Discord file-upload discoverability gap: Discord discovery does not advertise `upload-file`, and Discord has no dedicated `upload-file` message-action branch. The PR is not implemented on main or superseded by a shipped fix; it remains a plausible fix candidate for #60652, though the provided review discussion shows it still needs maintainer review and likely revisions before merge.

## Best Possible Solution

Keep the item open unless a maintainer chooses a canonical PR. The best fix path is to land one complete Discord implementation for #60652: advertise `upload-file` only when message sends are enabled, add a dedicated Discord `upload-file` action path or equivalent runtime dispatch, handle file-only sends without requiring text, forward metadata such as filename/reply/silent/session context, and cover discovery plus handler/runtime behavior with focused tests.
