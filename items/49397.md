## Summary

Close as `clawhub`. Current `main` does not include a bundled Serper integration, and the repo’s current architecture/docs explicitly route optional web-search providers to external plugins published on ClawHub or npm rather than adding more bundled/core support by default.

## Close Comment

Closing as a `clawhub` request after Codex review. On current `main`, Serper is not implemented as a bundled web-search provider, and the project’s current direction is to keep optional capability in plugins rather than expand bundled/core surfaces. The repo now documents web search as a plugin-owned capability (`docs/plugins/manifest.md:404-419`), shows provider plugins registering `api.registerWebSearchProvider(...)` (`docs/plugins/sdk-provider-plugins.md:612-686`), and explicitly says new capabilities like this do not need to be added to the OpenClaw repo and should be published to ClawHub or npm instead (`docs/plugins/building-plugins.md:11-19`, `VISION.md:54-60`, `docs/plugins/community.md:9-22`). If someone wants Serper support, the supported path is an external OpenClaw plugin published on ClawHub rather than a bundled OpenClaw core/provider addition.
