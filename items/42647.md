## Summary

Keep open. Current main has retrieval citations, one-source short-term promotion markers, event logs, and Memory Wiki provenance, but it still lacks the requested durable-memory provenance model with required source refs, multiple supporting sources, merge/edit provenance retention, and source-less promotion blocking.

## Best Possible Solution

Keep this issue open and implement it alongside #42646 and #42648: add typed sourceRefs/provenance to durable memory records, require at least one source before promotion, preserve provenance through create/merge/edit/conflict outcomes, and expose the refs consistently in retrieval and review flows. Memory Wiki can consume or mirror that model, but should not substitute for the core durable-memory provenance contract.
