## Summary

Keep open. Current main has partial source labels in search results and promotion output, plus a separate provenance-rich memory-wiki layer, but it does not implement the requested durable-memory provenance model. Durable memory is still plain Markdown, the SQLite index stores file/chunk metadata rather than per-memory source refs, and the promotion pipeline carries one source path/range rather than multiple provenance records preserved across merges or edits.

## Best Possible Solution

Keep this open and implement it alongside the canonical durable-memory record and write pipeline work: add a typed provenance/sourceRefs model to memory-core durable records, require at least one source before promotion, preserve provenance through create/merge/edit/conflict outcomes, and expose those refs consistently in retrieval and review flows. Coordinate with memory-wiki so wiki provenance can consume or mirror the core model instead of becoming a separate substitute.
