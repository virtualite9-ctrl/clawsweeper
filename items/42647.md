## Summary

Keep this issue open. Current main has adjacent retrieval citations, short-term promotion source markers, memory event logs, and Memory Wiki provenance, but it still lacks the requested durable-memory provenance model with required source refs, multiple supporting sources, merge/edit provenance retention, and source-less promotion blocking.

## Best Possible Solution

Keep this issue open and implement it alongside #42646 and #42648: add typed sourceRefs/provenance to canonical durable memory records, require at least one source before durable promotion, preserve provenance through create/merge/edit/conflict outcomes, and expose those refs consistently in retrieval and review flows. Memory Wiki can consume or mirror that model, but should not substitute for the active durable-memory provenance contract.
