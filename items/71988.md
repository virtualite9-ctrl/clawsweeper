## Summary

Current main still lacks the fallback `inbound_claim` broadcast this PR is trying to add, and the linked #48434/#71644 context makes this the active focused successor rather than obsolete work. Keep it open, but the current patch still needs follow-up for synthetic replies and inbound dedupe cleanup before it is safe to merge.

## Best Possible Solution

Keep this PR open and request a narrow follow-up patch: add the fallback broadcast in `src/auto-reply/reply/dispatch-from-config.ts` guarded by `pluginFallbackReason`, deliver `broadcastResult.reply` the same way the targeted plugin-bound path does, explicitly commit or release inbound dedupe before any new terminal return, and add regression tests for missing-plugin/no-handler broadcast, handled-with-reply, and dedupe cleanup.
