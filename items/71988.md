## Summary

Keep #71988 open. Current main still lacks the requested global `inbound_claim` broadcast after plugin-bound fallback, and the linked issue #48434 was closed as superseded by this PR. The PR is the current canonical implementation surface, but it should not land as-is without addressing handled broadcast reply delivery, inbound dedupe cleanup, and focused positive tests.

## Best Possible Solution

Keep #71988 open as the active implementation review. The best path is a narrow core dispatch patch that broadcasts `inbound_claim` only for plugin-bound fallback cases, preserves synthetic replies consistently with the targeted plugin path, commits/releases inbound dedupe before every terminal return, and adds focused tests for missing-plugin/no-handler broadcast, ordinary core conversations not broadcasting, handled broadcast reply delivery, and dedupe cleanup.
