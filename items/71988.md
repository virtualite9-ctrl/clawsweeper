## Summary

Keep #71988 open. Current main still does not broadcast `inbound_claim` after plugin-bound fallback, and #71988 is the canonical focused implementation after #48434 was closed as superseded by it and #71644 was closed as the dirty prior attempt. The PR likely still needs changes before merge because review comments identify missing handled-reply delivery and inbound dedupe cleanup on the new terminal broadcast path.

## Best Possible Solution

Keep the PR open and request a narrow follow-up patch: add the fallback broadcast in `src/auto-reply/reply/dispatch-from-config.ts`, mirror the targeted plugin-bound handled path by delivering any synthetic reply, preserve inbound dedupe cleanup before any new terminal return, and add positive regression tests for missing-plugin/no-handler broadcast handling, handled-with-reply, and dedupe behavior.
