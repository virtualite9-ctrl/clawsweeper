## Summary

Keep #29945 open. Current main still hardcodes branded pairing replies for newly created unpaired DM pairing requests, has no `unpairedResponse` config contract, and the linked maintainer PR #39065 is still open and unmerged as the active fix path.

## Best Possible Solution

Keep #29945 open until #39065 or an equivalent maintainer-approved change lands on main. The fix should add an explicit unpaired-DM response policy, wire `silent`, `code-only`, and `branded` behavior through shared pairing helpers and affected channel/plugin call sites, update config schema/generated metadata and public docs, add regression coverage for each mode, and audit related pairing logs/error responses for raw sender identifiers.
