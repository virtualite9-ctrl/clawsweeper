## Summary

Keep #29945 open. Current main still emits branded pairing replies for newly created unpaired DM pairing requests, has no `unpairedResponse`/silent/code-only/branded config surface, and the direct maintainer implementation PR #39065 remains open and unmerged in the provided GitHub context.

## Best Possible Solution

Keep #29945 open until #39065 or an equivalent maintainer-approved change lands. The implementation should add an explicit unpaired-DM response policy, wire it through shared pairing helpers and affected channel call sites, update config schema/generated metadata/public docs, and add regression coverage for silent, code-only, and branded behavior across representative channels.
