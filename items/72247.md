## Summary

PR #72247 is a valid narrow fix, but it is a newer duplicate of already-open PR #65123. Current main still has the Discord explicit-target bug, and both PRs change the same `parseDiscordExplicitTarget` return value from `target.id` to `target.normalized`; #65123 already tracks the same remaining work with regression coverage and Greptile confirmation.

## Best Possible Solution

Close #72247 as a duplicate of the older open PR #65123, keep the fix work centralized there, and consider folding #72247's broader test cases into the canonical PR before merge.
