## Summary

Current main still only partially satisfies the requested recovery guidance. Direct gateway startup errors now suggest `openclaw doctor --fix`, and startup may auto-restore a last-known-good config, but the thrown invalid-config error still omits an explicit `openclaw.json.bak` rollback hint. Daemon start/restart preflight still emits the older generic `openclaw doctor` wording without `--fix` or backup recovery guidance.

## Best Possible Solution

Keep this issue open and make a narrow UX fix across the gateway invalid-config startup surfaces. A good implementation would centralize the recovery hint, update direct gateway startup validation plus daemon start/restart preflight to recommend `openclaw doctor --fix` and an explicit `openclaw.json.bak` fallback where appropriate, and extend the existing gateway recovery and daemon config-guard tests to assert both recovery steps.
