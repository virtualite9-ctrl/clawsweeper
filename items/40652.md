## Summary

Keep this issue open. Current main partially addresses the requested recovery guidance with `openclaw doctor --fix` in the direct gateway startup validation error and with last-known-good auto-recovery docs/behavior, but the thrown startup error still omits an explicit `openclaw.json.bak` rollback hint, and daemon start/restart preflight still uses generic `openclaw doctor` wording without `--fix` or backup recovery guidance.

## Best Possible Solution

Make a narrow UX fix in the invalid-config startup surfaces: centralize the recovery hint text if practical, update gateway startup validation and daemon start/restart preflight to recommend `openclaw doctor --fix` plus the `openclaw.json.bak` fallback where applicable, and extend the existing gateway recovery and daemon config-guard tests to assert both recovery steps.
