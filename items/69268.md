## Summary

Keep PR #69268 open. Current main already has the correct Anthropic boundary-aware payload shape for Claude Opus 4.7, but the PR’s two central behavioral paths are not fully implemented on main: stream resolution still preserves wrapped/custom currentStreamFn values instead of forcing supported Anthropic APIs through the boundary-aware transport, and the embedded runner still lacks the PR’s trailing non-user/assistant-tail cleanup and no-fresh-user short-circuit. The PR discussion is concrete and recent, with production symptoms, logs, reviewer findings, and follow-up commits addressing review comments, so this is not conservative cleanup closure material.

## Best Possible Solution

Keep the PR open for maintainer review/rework. The best path is a focused patch that composes provider wrappers with OpenClaw's boundary-aware Anthropic transport when the session only wraps the default streamSimple path, preserves explicit provider-owned/custom stream contracts, and adds regression coverage for Claude Opus 4.7 wrapped-stream routing plus no-fresh-user trailing assistant histories, including empty/thinking-only/error-content tails.
