## Summary

Keep #39811 open. Current `main` has fallback observability, but the core config-validation request is still not implemented: model refs remain string/shape validated, `openclaw config validate` reports the existing snapshot validity without catalog-aware checks, and the selection path still preserves configured refs that are absent from the runtime catalog.

## Best Possible Solution

Keep the issue open. The best product path is to add catalog-aware validation or doctor/startup warnings for configured model refs (`agents.defaults.model.primary`, fallback refs, per-agent/channel/hook overrides, `agents.defaults.models`, and provider-owned model refs) where the provider catalog is trustworthy, with explicit downgrade semantics for dynamic/custom providers. Keep the shipped fallback-state/status work, but do not treat it as a substitute for config validation.
