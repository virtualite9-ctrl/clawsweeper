## Summary

Keep this PR open. Current main still slugifies the first embedded-agent payload text directly, strips edge dashes before truncation, and then writes any non-null slug into the session-memory filename. The merged #66455 timeout fix added coverage for configured slug timeouts but did not address the error-payload or trailing-dash behavior, and the broader #38161 slug-hardening PR remains open rather than merged. The Greptile test-gap review was addressed in this PR's second commit, so the current PR is still a focused implementation candidate rather than stale cleanup.

## Best Possible Solution

Keep this PR open as a focused bug-fix candidate. The best path is to review/land this or an equivalent change in src/hooks/llm-slug-generator.ts that rejects error-marked and non-slug-like embedded-agent payloads before filename construction, strips leading/trailing dashes after truncation, and keeps the pure slugification regression coverage. Before merge, tighten the fix to use payload.isError explicitly and coordinate with the broader #38161 slug-hardening work.
