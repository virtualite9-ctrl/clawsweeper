## Summary

Keep this PR open. Current main still accepts any first LLM payload text as a slug, strips edge dashes before truncation, and writes the returned slug directly into the session-memory filename. The merged timeout work in #66455 is present but does not implement this PR's error-payload rejection or post-truncation dash cleanup, and the broader open #38161 remains complementary rather than a shipped replacement.

## Best Possible Solution

Keep this PR open and review or land the focused fix, or apply the same behavior directly on main: extract/test a pure slugification helper, reject non-slug-like LLM payload text before filename construction, and strip leading/trailing dashes after truncation. Do not close it as superseded by #66455 or #38161 unless another merged change first ships these exact safeguards.
