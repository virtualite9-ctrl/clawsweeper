## Summary

Keep this PR open. Current main still slugifies the first LLM payload text directly, strips leading/trailing dashes before truncation, and writes the returned slug into the session-memory filename. The focused PR remains relevant; the merged timeout work and the broader open hook hardening PR do not replace these safeguards.

## Best Possible Solution

Keep this PR open and review or land a focused fix on `src/hooks/llm-slug-generator.ts`: extract/test a pure slugification helper, reject non-slug-like or error-marked LLM payloads before filename construction, and strip leading/trailing dashes after truncation. Coordinate with #38161, but do not close this PR as superseded unless another merged change first ships these safeguards.
