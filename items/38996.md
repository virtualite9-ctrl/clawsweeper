## Summary

Keep open. Current `main` and shipped `v2026.4.23` still route Feishu direct-message thread replies as non-threaded sends: non-group messages force `replyInThread`/`threadReply` to `false`, keep `replyTargetMessageId` on the triggering message instead of the thread root, and pass `skipReplyToInMessages: !isGroup`, which strips the reply target before sending. The existing test coverage only asserts inbound `thread_id` behavior for group chats, not P2P threads.

## Evidence
