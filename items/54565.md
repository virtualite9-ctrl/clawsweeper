## Summary

Keep open. Current main supports inbound debounce globally, per channel, and for WhatsApp at channel/account scope, but not inside `channels.whatsapp.groups.<jid>`. The WhatsApp schema/type surface does not allow `groups.*.debounceMs`, and the live monitor resolves one debounce value for the account/channel before passing it to the inbound listener. This is a valid bundled WhatsApp plugin feature request, not a ClawHub close or duplicate from the provided related context.

## Best Possible Solution

Keep this issue open. The best path is a WhatsApp-owned implementation first: add `debounceMs?: number` to the WhatsApp group config type/schema/docs, resolve exact group then wildcard group debounce from the effective account `groups` map, feed it through `createInboundDebouncer.resolveDebounceMs`, and add regression tests proving group-specific values override account/channel/global debounce without changing other groups. Generalization to Telegram/Discord/other plugins should be handled as explicit follow-up design or per-plugin additions after the WhatsApp behavior is proven.
