## Summary

Keep open. Current `main` still wires doctor health for session locks only, not gateway locks, so the requested behavior is not implemented. The proposed PR also has unresolved correctness risks from review: it diverges from `acquireGatewayLock`'s owner-status logic and could mis-handle live or recycled-PID gateway locks.
