## Summary

Keep #70475 open. Current main has some Dreaming execution config types, and the discussion correctly notes that memory embeddings can already be pointed at local providers/models, but the user-visible request is not implemented: `memory-core` does not expose a supported Dreaming LLM model key in its plugin schema/docs, and the Dream Diary subagent call still runs without passing a provider/model override, so it inherits the default runtime model.

## Best Possible Solution

Implement a supported `memory-core` Dreaming model configuration path, likely under `plugins.entries.memory-core.config.dreaming.execution.defaults.model` with optional phase overrides, update `openclaw.plugin.json` and public docs, parse provider/model refs, and pass the resolved provider/model into `api.runtime.subagent.run` for Dream Diary/narrative generation while preserving current inheritance when unset. Keep the existing local embedding-provider docs/workaround as separate memory-search guidance.
