## Summary

Keep open. Current main has useful partial inline media support, but the full request is still not implemented: MEDIA directives, structured attachments, user images, and managed generated-image blocks render inline, while automatic web/link previews are absent, remote markdown images remain intentionally flattened/blocked by CSP, direct tool-result image shapes are still incomplete, and raw/generated audio blocks are not persisted as durable playable attachments.

## Best Possible Solution

Keep this issue open as the umbrella unless maintainers split it. Preserve the existing MEDIA directive, managed-image, and attachment player support; then track or implement server-side link previews with explicit CSP/privacy/fetch policy, settle the remote-image rendering policy alongside #41025, finish direct tool-result media rendering for #50779/#52025-style payloads, and normalize generated audio/video into durable renderable attachment or managed-media references.
