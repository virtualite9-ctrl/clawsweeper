## Summary

Keep open. Current main has meaningful partial inline media support, but the umbrella request is not complete: Control UI still lacks automatic web/link previews, remote markdown images are still intentionally flattened and blocked by CSP, tool-result image rendering remains tracked by narrower open work, and generated/raw audio is not normalized into durable playable attachment references.

## Best Possible Solution

Keep this issue open as the umbrella unless maintainers explicitly split it. Preserve the shipped MEDIA directive, managed-image, user-image, and attachment-player support; then implement server-side link previews with explicit SSRF/CSP/privacy policy, settle the remote-image rendering policy alongside #41025, finish direct tool-result image rendering for #50779/#52025-style payloads, and normalize generated audio/video into durable managed-media attachment references that the Control UI can replay after history reloads.
