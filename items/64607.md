## Summary

Keep open. Current main has meaningful partial Control UI media support for MEDIA directives, local/proxied attachments, managed/generated images, and some local audio persistence, but the full feature request is not implemented: web/link previews are absent, remote markdown images are still explicitly flattened to alt text, and raw persisted audio blocks are not normalized into renderable UI attachments after history sanitization.

## Best Possible Solution

Keep this open as the umbrella unless maintainers create narrower canonical follow-ups. Preserve the shipped managed-image and MEDIA attachment rendering, then explicitly implement or track server-side link previews with a CSP/privacy model, a deliberate remote-image policy aligned with #41025, direct tool-result image rendering aligned with #50779/#52025, and normalized renderable audio/video content blocks for persisted generated media.
