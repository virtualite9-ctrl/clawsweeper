## Summary

Keep open. Current main has useful partial Control UI inline media support for assistant MEDIA attachments and managed/generated image blocks, but the full request is not implemented: web/link previews are absent, remote HTTP(S) images remain intentionally blocked or flattened in markdown, and persisted generated audio blocks are not normalized into renderable audio-player attachments.

## Best Possible Solution

Keep this open as the umbrella unless maintainers create narrower canonical follow-ups. Preserve the shipped managed-image and MEDIA attachment rendering, then explicitly implement or track server-side link previews with a CSP/privacy model, renderable persisted audio/video content blocks, direct tool-result image rendering aligned with #50779/#52025, and a deliberate remote-image policy aligned with #41025.
