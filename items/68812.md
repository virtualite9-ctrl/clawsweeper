## Summary

Keep #68812 open. Current main still hard-codes the memory embedding retry and non-batch indexing concurrency values, the validated config contract does not expose the requested `memorySearch.embedding` knobs, and the directly related PR #68822 is still open/unmerged with review feedback that the proposed config shape is not wired into the strict schema.

## Best Possible Solution

Keep #68812 open until OpenClaw lands a typed, validated memory embedding tuning surface. The likely path is to repair or replace #68822 by adding agent-scoped `memorySearch.embedding` settings to `MemorySearchConfig`, resolved memory-search config, Zod schema/help/generated metadata, docs, and focused tests, then wire those values into `runMemoryEmbeddingRetryLoop`, retry-delay capping, and non-batch `getIndexConcurrency()` without changing the existing provider Batch API meaning of `remote.batch.concurrency`.
