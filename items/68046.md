## Summary

Keep open. Current main still strips non-text MCP tool-result fields in the HTTP MCP gateway, and this PR is a focused implementation candidate for the still-open linked bug #68037. The patch should be revised before merge because it passes through any object with a string type instead of validating the known MCP content block shapes.

## Best Possible Solution

Keep this PR open and revise it as the implementation candidate for #68037. The best fix is to update src/gateway/mcp-http.handlers.ts so it preserves valid MCP text/image/audio/resource/resource_link blocks, validates required fields per block kind before passthrough, falls back to text for strings or malformed entries, and adds focused regression coverage for the HTTP tools/call path before maintainer review and merge.
