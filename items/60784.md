## Summary

Current `main` already preserves the parent logger threshold for child loggers and includes broader regression coverage than this PR, so there is no remaining change to merge.

## Evidence

- **Child logger inherits parent threshold on main:** `getChildLogger` now resolves `minLevel` from `opts.level` or falls back to `base.settings.minLevel`, then passes that value into `getSubLogger`, which preserves the parent threshold when no child override is requested.
  - file: [src/logging/logger.ts:253](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/src/logging/logger.ts#L253)

  - sha: [6bc0dc8fb671](https://github.com/openclaw/openclaw/commit/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440)
- **Parent logger threshold comes from configured logging level:** `buildLogger` sets the root logger `minLevel` from `settings.level`, so the child fallback in `getChildLogger` tracks the configured logger threshold rather than an allow-all default.
  - file: [src/logging/logger.ts:156](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/src/logging/logger.ts#L156)
