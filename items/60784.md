## Summary

Current main already implements PR #60784’s observable logging fix: child loggers inherit the parent logger `minLevel` when no child override is supplied, and regression coverage exists for the original allow-all fallback. The same implementation is present in release `v2026.4.24`, with a changelog entry for child logger `minLevel` inheritance.

## Best Possible Solution

Close PR #60784 as implemented on main. Keep the shipped `getChildLogger` implementation and `level-filter.test.ts` regression coverage rather than merging another duplicate patch for the same child logger `minLevel` inheritance bug.

## Evidence

- **Current main preserves parent minLevel for child loggers:** `getChildLogger` now sets `minLevel` to `opts?.level ? levelToMinLevel(opts.level) : base.settings.minLevel` before calling `base.getSubLogger`, so a child logger without an explicit level does not pass or resolve to an undefined threshold.
