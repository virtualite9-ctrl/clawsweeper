## Summary

Current `main` already handles `--session-id` ahead of the implicit `--agent` main-session fallback, and the behavior is covered by targeted regression tests. The same implementation is present in release `v2026.4.23`, so this issue appears fixed and shipped.

## Evidence

- **Current resolver no longer lets `--agent` override a provided `--session-id`:** `resolveSessionKeyForRequest()` computes `requestedSessionId` first, then only calls `resolveExplicitAgentSessionKey()` when no session id was provided. That preserves the session-id lookup / explicit-session fallback path instead of forcing `agent:<id>:main`.
