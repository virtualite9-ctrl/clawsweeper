## Summary

Current main still provides cost visibility and per-agent/session usage aggregation, but not config-driven per-agent daily/monthly budget enforcement before gateway/model dispatch. The issue is recent, concrete, not maintainer-authored, and only labeled stale. The ATXP, nornr, and vibe-billing comments are useful prior art/workarounds, not an OpenClaw implementation, a posted ClawHub plugin, or a superseding canonical item.

## Best Possible Solution

Keep this issue open as a core/gateway guardrail request. The best implementation path is a documented agents.defaults and agents.list[] budget config contract, a durable per-agent ledger or reservation check before every model attempt and fallback, explicit warn/block/pause behavior, status and doctor visibility, docs updates, and regression coverage for gateway sessions, cron/heartbeat/subagent paths, concurrency, missing cost metadata, and provider fallback chains. If maintainers choose plugin-first, first add a supported blocking pre-model-call budget API rather than treating the current plugin surface as sufficient.

## Risks / Open Questions

- A transcript-scan-only cap would race concurrent runs, fallback retries, delayed transcript writes, cron, heartbeat, and subagent paths; hard enforcement likely needs a durable ledger or reservation before each model attempt.
- Maintainers still need product/security decisions for approximate pricing, OAuth or no-dollar-cost providers, provider-side media/tool spend, provider pricing drift, and UTC versus local reset boundaries.
- Existing plugin hooks can short-circuit some inbound or cron paths, but closing as ClawHub-ready would skip the missing universal budget API and status/doctor contract requested here.
