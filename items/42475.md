## Summary

Current main still has cost reporting, per-agent usage aggregation, and provider spend-limit error classification, but not config-driven per-agent daily/monthly budget enforcement before gateway/model dispatch. The issue is concrete and recent, and the ATXP, nornr, and vibe-billing comments are useful prior art rather than a shipped OpenClaw implementation, an already-posted ClawHub plugin, or a superseding canonical item.

## Best Possible Solution

Keep this issue open as a core/gateway guardrail request. The best implementation path is a documented agents.defaults and agents.list[] budget contract, a durable per-agent request-time ledger or reservation check before every model attempt and fallback, explicit warn/block/pause behavior, status and doctor visibility, docs updates, and regression coverage for gateway sessions, cron/heartbeat/subagent paths, concurrency, missing cost metadata, and provider fallback chains. If maintainers choose plugin-first, first add a supported blocking pre-model-call budget API rather than closing the issue as ClawHub-ready today.

## Risks / Open Questions

- A transcript-scan-only cap would race concurrent runs, fallback retries, delayed transcript writes, cron, heartbeat, and subagent paths; a hard cap likely needs a durable ledger or reservation check before each model attempt.
- Maintainers still need product/security decisions for approximate pricing, OAuth/no-dollar-cost cases, provider-side media/tool spend, provider pricing drift, and UTC versus local reset boundaries.
- External account or proxy spend limits can mitigate some deployments, but they do not provide native OpenClaw per-agent gateway enforcement or status/doctor visibility.
