## Summary

Current main has per-session/per-agent cost visibility but not per-agent daily/monthly budget enforcement. The request remains actionable: config schemas have no budget field, the gateway dispatch path accepts and starts agent runs without a cost gate, and docs describe cost display/usage summaries rather than blocking. The issue discussion also adds useful external approaches (ATXP wallet funding, nornr mandate flow, vibe-billing proxy), but none is an OpenClaw plugin or shipped core implementation that closes the requested gateway guardrail.

## Best Possible Solution

Keep this issue open as a real core/gateway guardrail request. The best path is a narrow budget contract in `agents.defaults` and `agents.list[]`, a request-time per-agent ledger/check before every model attempt and fallback, explicit `warn`/`block`/`pause` behavior, status/doctor visibility, and focused tests for sessions, cron/heartbeat/subagent paths, concurrency, and missing-cost cases. If maintainers prefer external enforcement, first add a supported blocking plugin/API seam rather than closing this to ClawHub today.

## Risks / Open Questions

- Budget enforcement has product/security semantics that need maintainer design: approximate prices, missing OAuth dollar costs, cache pricing drift, provider/tool/media spend beyond core chat, and daily/monthly reset boundaries.
- A plugin-first answer likely needs a new blocking pre-model-call or gateway budget API first; current hooks are sufficient for observation/routing but not for guaranteed hard spend enforcement.
- Implementation should avoid relying only on transcript scans at dispatch time, because concurrent runs, retries, fallback attempts, and delayed transcript writes can race a daily/monthly cap.
