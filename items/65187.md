## Summary

Keep this PR open. Current main already strips `<final>` tags through the Control UI extraction path and has lower-level stripping tests, but it still lacks the PR's direct `extractText` / `extractTextCached` regression coverage for assistant content-array and legacy `text` message formats. The supplied patch is test-only and low risk, but needs the cached-path assertions requested in review before it is clean to land.

## Best Possible Solution

Keep this PR open for maintainer review. The clean path is to update the patch so every new `<final>` regression case asserts both `extractText` and `extractTextCached`, then either land it as focused Control UI message-extraction coverage or have a maintainer explicitly close it as intentionally redundant with the lower-level sanitizer tests.
