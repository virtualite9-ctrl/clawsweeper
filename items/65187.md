Keep this PR open. Current main strips `<final>` tags through the Control UI extraction path and has lower-level regression coverage, but it still does not contain the PR's direct `extractText`/`extractTextCached` message-extraction tests for assistant content-array and legacy `text` messages. The PR remains a plausible focused test addition, with one review cleanup still needed.
