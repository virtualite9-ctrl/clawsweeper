## Summary

Current main already fixes the reported Control UI/subagent context badge problem. Subagent runs now carry a latest-model-call prompt snapshot for context usage, session persistence stores that as fresh `totalTokens` instead of cumulative run totals, stale/missing totals are exposed as not fresh, and the Control UI context notice renders from `totalTokens`/`contextTokens` while suppressing stale snapshots. The fix is present in the v2026.4.24 release tree.

## Best Possible Solution

Close #59345 as implemented and keep the shipped `totalTokens`/`totalTokensFresh` session contract. Any future work should be narrower, such as tightening legacy/unknown freshness semantics if maintainers want the UI helper itself to require `totalTokensFresh === true` even for manually constructed rows; the current Gateway-to-Control-UI path already solves the reported operator-visible bug.
