## Summary

Keep open. Current main has a related zero-usage empty-stop retry path, but it does not implement the PR's central user-visible diagnostic for empty-stream provider configuration failures; after retry exhaustion the embedded runner still surfaces the generic incomplete-turn message. The PR remains a focused bug fix with useful reproduction details and review-driven fixes in the provided discussion.

## Best Possible Solution

Keep the PR open for maintainer/contributor follow-up. The best path is to rebase it onto current main, reuse or merge with the existing zero-usage empty-stop helper, and add focused tests proving that a repeated zero-content/zero-token provider response surfaces an actionable provider/model/baseUrl diagnostic while legitimate empty or usage-silent turns keep the generic behavior.
