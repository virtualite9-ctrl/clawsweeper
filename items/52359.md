## Summary

Keep open. Current `main` does not implement the requested pre-compaction memory-flush cooldown/dedup. The live gate only suppresses repeats within the same compaction cycle via `memoryFlushCompactionCount === compactionCount`; there is no time-based cooldown check, no `cooldownMs` config field, and the persisted `memoryFlushAt` timestamp is written but never consulted. There is also dormant hash-dedup scaffolding (`memoryFlushContextHash`, `computeContextHash`, and a dedicated test file), but no production call site wires that state into `runMemoryFlushIfNeeded()`.
