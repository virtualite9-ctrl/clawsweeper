## Summary

Current main already guards the embedded Anthropic SSE path against empty/missing `data` frames before `JSON.parse`, and that implementation shipped in `v2026.4.20`.

## Close Comment

Codex review: this is already implemented on `main`. Embedded Anthropic runs now route through the boundary-aware transport (`src/agents/pi-embedded-runner/stream-resolution.ts:120`, `src/agents/provider-transport-stream.ts:79`), and the Anthropic SSE parser skips empty or missing `data:` payloads before calling `JSON.parse` (`src/agents/anthropic-transport-stream.ts:454-473`). That parser landed in commit `67ebc433f953e3777b74f8866bdce442471f05f2` and shipped in `v2026.4.20`, so the reported `Unexpected end of JSON input` failure mode from empty SSE events is no longer present in current OpenClaw releases.
