## Summary

Current main still only partially covers the requested Feishu compatibility work. Legacy `groupPolicy: "allowall"` and `topicSessionMode` handling exist, but the requested `botId` to `appId` compatibility is still missing from the Feishu schema, credential resolution, setup detection, manifest metadata, docs, and focused tests. The linked #52885 is a closed Weixin plugin-SDK compatibility issue, not a canonical Feishu config migration fix.

## Best Possible Solution

Keep this open and scope the fix to Feishu-owned compatibility: accept and normalize `botId` to `appId` for top-level and per-account configs, preserve `appId` precedence when both keys exist, wire `openclaw doctor --fix` through a Feishu `legacyConfigRules`/`normalizeCompatibilityConfig` contract or equivalent plugin-owned seam, update setup/status and credential resolution, add focused schema/account/setup/doctor tests, and document the upgrade path on the public Feishu channel docs.
