## Summary

Keep open. Current main partially handles the Feishu examples, but the broader backward-compatibility request is not fully implemented: `topicSessionMode` and `groupPolicy: "allowall"` are accepted, while `botId` -> `appId` migration/detection and Feishu-specific doctor legacy config rules are still absent.

## Best Possible Solution

Keep this issue open and scope it to a Feishu-owned compatibility path: add explicit handling or doctor legacy rules for old Feishu config keys such as `botId` -> `appId`, decide whether to rewrite or only warn for `topicSessionMode`/`allowall`, add focused config-schema/doctor tests, and update the Feishu docs with the supported upgrade path. The WeChat plugin SDK compatibility discussion in #52885 can remain separate because it does not cover the Feishu config migration work.
