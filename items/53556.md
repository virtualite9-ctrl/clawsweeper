## Summary

Current main still only partially addresses the requested Feishu channel backward compatibility. `groupPolicy: "allowall"` and legacy `topicSessionMode` are covered, but the reported `botId` to `appId` upgrade path is still missing from schema, runtime credential resolution, setup status, plugin metadata, docs, and doctor migration hooks. The related #52885 discussion is about a Weixin plugin SDK compatibility problem, not a canonical Feishu config migration fix for this issue.

## Best Possible Solution

Keep this issue open and scope it to a Feishu-owned compatibility path: add explicit `botId` to `appId` handling for top-level and account configs, preferably through Feishu `normalizeCompatibilityConfig`/`legacyConfigRules` or an equivalent schema-normalization seam; preserve `appId` precedence when both are set; add focused schema/account/doctor tests; and document the supported upgrade path on https://docs.openclaw.ai/channels/feishu.
