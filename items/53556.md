## Summary

Current main still only partially implements the requested Feishu compatibility. Legacy `groupPolicy: "allowall"` and `topicSessionMode` handling exist, but `botId` to `appId` compatibility is still missing from the strict Feishu schema, credential resolution, setup/status detection, manifest/docs metadata, and focused tests. The linked #52885 is closed and tracks a Weixin plugin-SDK compatibility problem, not this Feishu config migration.

## Best Possible Solution

Keep this open and implement the remaining Feishu-owned compatibility: accept and normalize top-level and per-account `botId` to `appId`, preserve `appId` precedence when both are present, wire `openclaw doctor --fix` through a Feishu `legacyConfigRules` / `normalizeCompatibilityConfig` contract, update setup/status and credential-resolution tests, and document the upgrade path on the public Feishu channel docs.
