## Summary

Keep #69629 open. Current main enforces Discord slash-command execution authorization and account-scoped routing, but it still documents that unauthorized commands can remain visible in Discord’s UI. I did not find a current OpenClaw implementation or config contract that syncs per-account/per-channel Discord application-command visibility for same-guild multi-bot deployments.

## Best Possible Solution

Keep this open as a Discord plugin feature. The best path is maintainer design for either safe Discord application-command permission sync from `channels.discord.accounts.*.guilds.*.channels` and bindings when OpenClaw has the required authorization, or clear docs/tooling for the Discord Integrations permissions workaround. Add regression coverage for same-guild multi-account command deployment, execution auth, and any permission-sync payload.
