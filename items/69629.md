## Summary

Keep #69629 open. Current main enforces Discord guild/channel allowlists when a slash command is invoked, and related reports cover routing/auth safety, but the requested per-Discord-channel command palette visibility is not implemented. The current registration path builds the same native command set for a Discord account and deploys it without guild-channel command permission/scoping data, while the docs explicitly warn that commands may still be visible in Discord UI even when execution is denied.

## Best Possible Solution

Keep the issue open for Discord plugin owner/design work. The best fix likely belongs in `extensions/discord`: either add a Discord-native command permission/scoping implementation that derives allowed channel IDs from `channels.discord.accounts.<id>.guilds.<guild>.channels`, or document a deliberate platform limitation and provide a clearer text-only/native-disabled workaround if Discord cannot support the desired palette behavior safely. Any implementation should update the Discord docs/config reference and add regression coverage for registration payloads plus runtime denial behavior.
