## Summary

Issue #71733 requests a new core/plugin SDK lifecycle seam: durable, host-owned next-turn injections with exactly-once consumption plus a typed turn-preparation hook. Current main exposes adjacent prompt hooks and runtime event/approval surfaces, but I did not find an implemented durable queue, `enqueueNextTurnInjection`, or `agent_turn_prepare` lifecycle contract. The related open RFC PR #71731 explicitly frames this as proposed, not implemented, SDK surface, so this item should remain open for maintainer product/API judgment rather than be auto-closed.

## Best Possible Solution

Keep the issue open and use it as the focused decision/implementation tracker for durable next-turn injection semantics. A good implementation would add a typed SDK lifecycle event and host-owned persisted queue with fixture tests covering exactly-once delivery, retry/fallback behavior, expiry diagnostics, dedupe, disabled-plugin suppression, and ordering relative to `before_prompt_build`.
