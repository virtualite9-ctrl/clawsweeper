## Summary

Keep #71733 open. Current main still lacks the proposed durable plugin next-turn injection API and typed `agent_turn_prepare` hook. The existing surfaces are adjacent only: current-turn prompt hooks, in-memory system events, and an internal hidden next-turn custom-message path. Open PR #72082 appears to be the active implementation path for this RFC, but it is not merged into current main, so the SDK gap remains unshipped.

## Best Possible Solution

Keep #71733 open as a core/plugin-SDK RFC until the durable next-turn injection queue and typed `agent_turn_prepare` hook land on main. The current best path is to review and validate #72082 as the implementation vehicle, ensuring it adds public SDK types, durable exactly-once queue semantics, retry/fallback ordering, expiry/dedupe/disabled-plugin diagnostics, docs, and fixture tests before this issue is closed as implemented or superseded.
