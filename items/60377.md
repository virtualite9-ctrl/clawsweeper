## Summary

Keep open. Current `main` still rejects Google Chat add-on JWTs unless `channels.googlechat.appPrincipal` is configured, tests lock that behavior in, the public Google Chat docs still do not mention `appPrincipal`, and webhook auth failures are still flattened to a bare 401 without surfacing the verification reason. The report's crash-loop and status-display theories do not match current `main`, but the core auth/docs gap remains unresolved.

## Evidence
