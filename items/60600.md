## Summary

Keep PR #60600 open. Current main still passes the Gemini PDF API key as a `key` URL query parameter, while existing Google/Gemini surfaces use `x-goog-api-key` headers. The PR is a narrow, still-applicable hardening fix and is not cleanly superseded by the broader related #46377 work.

## Best Possible Solution

Keep PR #60600 open as a focused security hardening fix. Best path is to land the URL-to-header auth change, ideally with a small regression test asserting the Gemini PDF request URL has no `key` query parameter and the headers include `x-goog-api-key`; leave the broader SSRF guard from #46377 for separate maintainer judgment.
