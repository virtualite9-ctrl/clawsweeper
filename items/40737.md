## Summary

Current `main` already implements the requested macOS LaunchAgent restart behavior: `openclaw gateway restart` now uses `launchctl enable` plus `launchctl kickstart -k` first for loaded services, only falls back to `bootstrap` when the service is not loaded, and includes recovery if a failed kickstart leaves the agent unloaded. The changelog shows this shipped in `2026.4.5`.

## Evidence

- **Restart path is kickstart-first:** `restartLaunchAgent()` enables the LaunchAgent, runs `launchctl kickstart -k` first, returns immediately on success, and only calls `bootstrapLaunchAgentOrThrow()` when `kickstart` reports the service is not loaded.
