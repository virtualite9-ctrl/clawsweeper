## Summary

Keep #68959 open. Current main can list agents and can switch among existing agent-scoped sessions, but it still does not provide the requested Control UI chat agent picker/switcher that shows all configured agents and starts/switches to any configured agent from the chat UI.

## Best Possible Solution

Keep the issue open. Add a Control UI chat agent picker backed by `agents.list`, render configured agent display names/identity, switch chat with `buildAgentMainSessionKey({ agentId, mainKey })`, preserve the existing session dropdown for per-agent sessions, and add tests for multiple configured agents where a non-default agent has no prior session row.

## Risks / Open Questions

- The existing session dropdown already switches among known agent-scoped sessions, so the fix should preserve that behavior and scope this issue to selecting configured agents, including agents with no existing session row.
- There is an unused `onAgentChange` path already wired from `app-render`, which suggests the implementation may be small, but it still needs visible UI and regression tests before this can close.
