## Summary

Keep #72051 open. Current main still lacks both substantive changes proposed by the PR: mirrored TaskFlow terminal timestamps can still produce `endedAt < updatedAt`, and `task-registry-control.runtime` is still not emitted as a stable root dist entry. The PR also has a real review concern about preserving historical `endedAt` values when `lastEventAt` is absent, so it should be revised or reviewed rather than closed as maintainer cleanup.

## Best Possible Solution

Keep the PR open and ask for a narrow revision: preserve historical terminal `endedAt` when no newer task event timestamp exists, clamp only cases that would otherwise violate the TaskFlow audit invariant, keep the stable `task-registry-control.runtime` dist entry, and add regression coverage for both the `lastEventAt > endedAt` mirror case and the legacy/no-`lastEventAt` retention case.
