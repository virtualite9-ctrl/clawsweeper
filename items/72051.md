## Summary

Keep PR #72051 open. Current main still lacks both substantive PR changes: mirrored TaskFlow terminal records can write `endedAt` earlier than `updatedAt`, and the packaged build still has no stable root entry for `task-registry-control.runtime`. The PR is recent, focused, and addresses a real task/taskflow warning and packaging gap, though the existing review comment about preserving legacy `endedAt` semantics should be handled before merge.

## Best Possible Solution

Keep the PR open and request a narrow revision: preserve historical terminal `endedAt` when no newer event timestamp exists, clamp only the `lastEventAt > endedAt` mirror case that violates TaskFlow audit invariants, keep the stable `task-registry-control.runtime` dist entry, and retain regression tests for both timestamp behavior and the build entry.
