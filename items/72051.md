## Summary

Keep PR #72051 open. Current main still lacks both core changes described by the PR: mirrored TaskFlow records can still end up with terminal `endedAt` earlier than `updatedAt`, and `task-registry-control.runtime` is still not emitted as a stable root dist entry even though cancellation paths try to load that root filename.

## Best Possible Solution

Keep the PR open and request a small revision: preserve historical terminal `endedAt` when there is no newer event timestamp, clamp only cases that would otherwise violate the TaskFlow audit invariant, keep the stable `task-registry-control.runtime` dist entry, and retain/add regression tests for both the `lastEventAt > endedAt` mirror case and the legacy/no-`lastEventAt` retention case.
