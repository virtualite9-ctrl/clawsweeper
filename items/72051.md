## Summary

Keep #72051 open. Current main still has the TaskFlow mirrored timestamp gap and does not emit task-registry-control.runtime as a stable core dist entry. The PR is a focused, relevant fix, but it should address the existing review concern about preserving historical endedAt values before merge.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to revise the timestamp helper so mirrored TaskFlow terminal timestamps are monotonic when endedAt lags an explicit lastEventAt, while preserving historical endedAt for endedAt-only legacy/restored rows. Keep the stable task-registry-control.runtime tsdown entry and add regression tests for both timestamp cases plus the build entry.
