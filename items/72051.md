## Summary

Keep PR #72051 open. Current main still lacks both substantive changes: mirrored TaskFlow terminal timestamps can still produce `endedAt < updatedAt`, and the task-registry control runtime is not emitted as a stable root dist entry. The existing review discussion also identifies a real revision point in the PR, so the best maintainer action is review/revise, not cleanup-close.

## Best Possible Solution

Keep the PR open and request a small revision: preserve historical terminal `endedAt` when no newer event timestamp exists, clamp only cases that would otherwise violate the TaskFlow audit invariant, keep the stable `task-registry-control.runtime` dist entry, and add regression tests for both the `lastEventAt > endedAt` mirror case and the legacy/no-`lastEventAt` retention case.
