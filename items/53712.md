## Summary

Current `main` already prevents pre-compaction memory flush from rewriting arbitrary workspace files. Memory-triggered runs are limited to `read` plus an append-only `write` for one canonical `memory/YYYY-MM-DD.md` path, and the same implementation is present in release `v2026.4.22`, so the reported `NOW.md` rewrite race is obsolete on current builds.

## Evidence

- **Flush run passes a single canonical write target:** `runMemoryFlushIfNeeded` resolves `activeMemoryFlushPlan.relativePath` and passes it into the embedded run as `memoryFlushWritePath` for `trigger: "memory"`.

  - file: [src/agents/pi-tools.ts:348](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/agents/pi-tools.ts#L348)

## Close Comment

Thanks for the report. Codex review against current `main` found this is already addressed. Pre-compaction memory flushes now pass a single canonical target file from `src/auto-reply/reply/agent-runner-memory.ts:739-775`, and memory-triggered tool runs in `src/agents/pi-tools.ts:348-352` and `src/agents/pi-tools.ts:637-658` are restricted to `read` plus a wrapped `write`. That wrapper in `src/agents/pi-tools.read.ts:504-580` only appends to `memory/YYYY-MM-DD.md` and rejects writes to any other path, while the default flush prompt in `extensions/memory-core/src/flush-plan.ts:13-31` explicitly marks other workspace files read-only. I also verified the same implementation is present in shipped release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`). Because the flush can no longer rewrite files like `NOW.md`, the reported concurrent Edit exact-match failure is already fixed on current builds. If you still see this on a newer release, please open a fresh issue with the exact version and logs.
