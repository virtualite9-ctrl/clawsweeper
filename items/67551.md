## Summary

Keep this PR open. Current main still lacks Unicode smart-quote delimiter repair for tool-call argument JSON, and merged #70294 only enabled the existing malformed-argument wrapper for openai-completions rather than changing the parser. The PR is a plausible narrow parser candidate tied to still-open #67488, but it needs rebase and focused tests before maintainer review.

## Best Possible Solution

Keep this PR or an equivalent follow-up open. Rebase it onto current main, put delimiter-aware scanning in the current shared/helper structure, preserve #70294's openai-completions enablement, and add focused regression tests for U+201C/U+201D structural delimiters plus content-level smart quotes before maintainer review.
