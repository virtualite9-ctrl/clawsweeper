## Summary

Keep open. Current main still does not implement `channels.signal.linkPreview` or an equivalent Signal JSON-RPC preview pass-through. The missing behavior is visible in the strict Signal config schema, Signal config types, outbound Signal send path, and Signal docs. The related #29466 is a separate closed webchat UI preview request, not a superseding Signal channel fix, and the latest issue discussion already records a same-day Codex review keeping this open.

## Best Possible Solution

Keep #24118 open until the bundled Signal plugin implements a first-class, backward-compatible `channels.signal.linkPreview` config surface, updates the Signal config schema/types/generated metadata/docs, and threads the resolved option plus any supported preview metadata through the Signal JSON-RPC send/updateConfiguration path with focused tests. If maintainers decide Signal preview generation itself is out of scope, the remaining product decision should be documented explicitly rather than auto-closing this concrete channel enhancement.
