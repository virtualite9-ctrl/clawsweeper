## Summary

Keep #24118 open. Current main still does not expose `channels.signal.linkPreview`, Signal config remains strict without that key, generated channel metadata and docs omit it, and the Signal send path still sends only message/style/account/attachments/target params through `signalRpcRequest("send", ...)`. The provided discussion also includes a same-day steipete/Codex comment keeping this open and distinguishing closed #29466 as a Webchat preview-card request, not Signal channel support.

## Best Possible Solution

Keep #24118 open until the bundled Signal plugin adds a backward-compatible `channels.signal.linkPreview` surface, updates Signal schema/types/generated metadata/docs, threads the resolved setting through account inheritance and outbound send paths, passes supported preview configuration/metadata to signal-cli JSON-RPC, and adds focused tests. If maintainers decide Signal link previews are out of scope, record that product decision explicitly rather than auto-closing this concrete channel enhancement.
