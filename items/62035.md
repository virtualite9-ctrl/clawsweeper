## Summary

Current `main` still has the reported behavior. `createTaskRecord()` stamps `createdAt` with `Date.now()` even when callers provide an earlier `startedAt`, and `tasks audit` still warns unconditionally when `startedAt < createdAt`. ACP and cron callers on `main` still pass pre-captured `startedAt` values into `createRunningTaskRun()`, so the issue remains plausible and supported by code inspection.

## Evidence

- **Audit still flags any startedAt-before-createdAt record:** `findTimestampInconsistency()` emits `inconsistent_timestamps` whenever `task.startedAt < task.createdAt`, with no tolerance window for small deltas.
