## Summary

Current `main` no longer lets packaged installs fail completion-cache updates on missing repo-only QA scenario files; the fix is documented as shipped in `v2026.4.11` and is reinforced by current gating, fallback handling, and regression coverage.

## Evidence

- **Release note:** `v2026.4.11` changelog explicitly records the fix: packaged CLI startup and completion-cache generation no longer read repo-only QA scenario markdown, and `openclaw completion --write-state` keeps working even if QA setup is broken.
  - file: [CHANGELOG.md:907](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/CHANGELOG.md#L907)
  - command: `git show v2026.4.23:CHANGELOG.md | nl -ba | sed -n '880,910p'`
  - sha: [a9797214338b](https://github.com/openclaw/openclaw/commit/a9797214338ba31c52c796adbb75afb16e0684a9)
- **QA subcommand hidden in normal packaged installs:** Current `main` filters `qa` out of subcommand descriptors unless private QA is explicitly enabled, so ordinary package/Homebrew installs no longer advertise or register the QA CLI path during completion generation.
  - file: [src/cli/program/subcli-descriptors.ts:175](https://github.com/openclaw/openclaw/blob/5b8bd6371c666a5de43c82305e0315771e8e49d5/src/cli/program/subcli-descriptors.ts#L175)

- **Private QA CLI is source-checkout only:** The private QA loader now requires `OPENCLAW_ENABLE_PRIVATE_QA_CLI=1` and a source checkout with `.git`, `src`, and `dist/plugin-sdk/qa-lab.js`; otherwise it refuses to load. That blocks the packaged-install path that used to trip over missing repo-only QA assets.

- **Missing QA pack is now non-fatal:** `readQaScenarioPack()` returns an empty fallback pack when `qa/scenarios/index.md` is absent, with an inline comment stating npm distributions may omit the repo-only scenario pack without breaking completion-cache or startup paths.
