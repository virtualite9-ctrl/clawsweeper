## Summary

Keep #12047 open. Current main downloads Telegram document attachments and can extract text-like files plus PDFs through the shared media pipeline, but the requested Office-style document support remains missing: default file MIME allowlists omit DOCX/XLSX/PPTX, application/vnd Office MIME types are skipped as binary before extraction, and the bundled document-extract plugin is PDF-only. The existing issue discussion already captured this same partial-implementation gap, and current main still matches it.

## Best Possible Solution

Keep this open as an OpenClaw product/contract issue. Telegram should continue forwarding attachments through the shared MediaPath/MediaType pipeline; the fix should extend shared file/media extraction so safe Office MIME types can pass configured/default allowlists, route non-PDF document MIME types through the generic document-extractor contract, add or bundle an Office document extractor implementation, and cover Telegram attachments plus generic shared extraction with tests and docs.
