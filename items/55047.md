## Summary

Keep open. This is a recent regression report, the only linked fix candidate was closed unmerged, and current main still contains Control UI cron paths that directly dereference `job.payload.kind` without the null-safe guard described in the discussion. The issue needs reproduction or a focused fix rather than cleanup closure.

## Best Possible Solution

Keep this issue open. The best maintainer path is to reproduce against current main with either the reporter's snapshot or a synthetic malformed/stale cron row, then add a focused null-safe guard or normalization layer for cron jobs before Control UI render/edit paths plus a jsdom regression test. If the root cause turns out to be backend store shape, repair or filter malformed cron rows in the cron service/doctor path and keep the UI resilient to bad snapshots.

## Evidence

- **Issue discussion and related PR context:** The reporter described formerly populated Control UI sections now rendering blank with only page titles.
