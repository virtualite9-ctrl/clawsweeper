## Summary

Keep #32496 open. Current main still does not first-class forward `frequency_penalty`, `presence_penalty`, `repetition_penalty`, or config-level `top_p`/`top_k` from model params into OpenAI/OpenAI-compatible request payloads. The discussion points to real demand, including mitchmalone's reproduction need and syncword/mobilinkd's notes about #32582 and #71350; however #32582/#32540/#33620 are closed unmerged, and #71350 is adjacent per-agent parameter work rather than a canonical replacement. Current `extra_body` support is a useful workaround for `openai-completions` proxy payloads, but it does not satisfy the direct-params behavior requested across Responses, Chat Completions, and WebSocket routes.

## Best Possible Solution

Keep #32496 open as the canonical sampling-penalty passthrough request. The best fix is targeted core transport work: resolve numeric snake_case and camelCase sampling params from merged global, per-model, per-agent, and override params; inject them only into compatible OpenAI/OpenAI-compatible Responses, Chat Completions, Azure, and WebSocket payload paths where the API supports them; preserve existing payload-hook and `extra_body` behavior; then add focused transport tests and docs that state the supported fields and scope.
