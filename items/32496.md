## Summary

Keep #32496 open. Current main still accepts arbitrary model params in config, and `extra_body` can help a subset of OpenAI-compatible Completions proxy payloads, but there is still no first-class `frequency_penalty`, `presence_penalty`, or `repetition_penalty` passthrough from resolved params into the OpenAI/OpenAI-compatible transport builders. Related PRs #32540, #32582, and #33620 were closed unmerged, and #71350 is adjacent per-agent param work rather than a canonical replacement for sampling-penalty passthrough.

## Best Possible Solution

Keep #32496 open as the canonical sampling-penalty passthrough request. The best fix is targeted transport work: collect supported numeric snake_case and camelCase sampling params from resolved global, per-model, per-agent, and override params; inject them only into compatible OpenAI/OpenAI-compatible request paths including Responses, Completions, and WebSocket where supported; preserve existing `onPayload`, `extra_body`, provider-wrapper, and proxy-store behavior; and add focused tests plus docs for supported fields and transport scope.
