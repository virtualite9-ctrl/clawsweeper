## Summary

Keep #32496 open. Current main at 631552c554095b0c520376db9131dac0f5427f94 still preserves arbitrary model params in config, but it does not first-class pass `frequency_penalty`, `presence_penalty`, or `repetition_penalty` into OpenAI/OpenAI-compatible request payloads. The linked PR attempts #32540, #32582, and #33620 were closed unmerged; #71350 is adjacent per-agent params work, not a canonical replacement. `extra_body` is only a limited workaround for OpenAI-compatible Completions proxy payloads.

## Best Possible Solution

Keep #32496 open as the canonical sampling-penalty passthrough request. The best fix is targeted transport work: collect supported numeric snake_case and camelCase sampling params from resolved global, per-model, per-agent, and override params; inject them only into compatible OpenAI/OpenAI-compatible request paths including Responses, Completions, and WebSocket where supported; preserve existing `extra_body`, `onPayload`, provider-wrapper, and plugin hook behavior; then add focused transport tests and docs for supported fields and scope.
