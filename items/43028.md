## Summary

Current main mitigates parts of the cron pain with execution timeouts, recurring error backoff, failure alerts, session retention pruning, and JSONL run history, but the central requested features are still missing: recurring-job auto-pause after repeated timeouts/errors, remove-time session cleanup, and automatic `.learnings/changelog.md` recording.

## Best Possible Solution

Keep this issue open and split the work into cron-owned follow-ups: add a backward-compatible recurring execution circuit breaker in `src/cron/service/timer.ts` with config/types/docs and failure-alert reuse; decide and implement a `cron.remove` cleanup contract across protocol, CLI, UI, and session/transcript ownership; and explicitly document whether JSONL `cron.runs` is the supported audit trail instead of `.learnings/changelog.md`.

## Risks / Open Questions

- The issue bundles three separable behaviors, so closing it now would likely hide still-missing user-visible cron work behind partial mitigations.
- A remove-time cleanup option needs a careful scope decision so it does not delete shared or base sessions unexpectedly; retention pruning and `deleteAfterRun` cleanup are not the same contract.
- The changelog request should be resolved explicitly: either implement the requested audit sink or document `cron.runs` as the supported audit surface and explain that `.learnings/changelog.md` is not a core OpenClaw path.
