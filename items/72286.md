## Summary

Keep this PR open. Current main does not implement Google Live browser Talk sessions: docs and code still limit Control UI Talk to OpenAI browser WebRTC, while the bundled Google realtime provider only exposes backend realtime bridges. Because the feature requires gateway protocol, browser UI, and provider contract changes, this draft is a plausible implementation candidate rather than a conservative cleanup close or ClawHub-only item.

## Best Possible Solution

Keep this PR open as the implementation candidate. Maintainers should review whether Google Live browser Talk belongs in the bundled Google provider and Control UI, then either land a polished version with the protocol/UI/provider tests proposed here or redirect it into a narrower plugin API design if the browser transport contract should be generalized further.
