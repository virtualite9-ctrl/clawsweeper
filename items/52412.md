## Summary

Keep open. Current `main` still does not expose a public `api.runtime.agent.abort({ runId })` surface in the plugin runtime. The underlying gateway/chat abort machinery exists, and plugins now have `runId` in hook context, but cancellation is only available through internal Gateway handlers or the experimental trusted `plugin-sdk/agent-harness` subpath, not through the documented runtime API requested in #52412.
