## Summary

Keep PR #60500 open. Current main still has an image-only WebChat attachment path, so the requested PDF upload support is not implemented. The PR contains useful work, but the Codex review concern remains valid: simply offloading PDFs to `media://inbound/*` does not make the document content available to the model through the current image-only media-ref resolver.

## Best Possible Solution

Keep the PR open for maintainer or author follow-up rather than closing it. A complete fix should broaden the WebChat UI accept/filter/preview path for PDFs, preserve filename/MIME metadata through `chat.send`, update the gateway parser/offload allowlist, route offloaded PDF refs through the existing document extraction/PDF tool or another explicit document hydration path, remove the unrelated lockfile change, and add focused UI/gateway tests. Coordinate scope with the open document-upload trackers such as #30759 and the configurable non-image attachment policy in #70438.
