## Summary

Current `main` already covers both reported problems. Windows command execution now wraps `.cmd`/`.bat` shims through `cmd.exe` or resolves npm/npx to `node + npm-cli.js`, and Feishu's runtime barrel no longer imports the removed onboarding module.

## Evidence

- **Windows exec path now handles `.cmd` safely:** `src/process/exec.ts` detects Windows batch shims, rewrites npm/npx to `node` when possible, and otherwise invokes `cmd.exe /d /s /c ...` instead of directly spawning `.cmd`, which addresses the reported `spawn EINVAL` failure mode.
