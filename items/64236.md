## Summary

Keep this issue open. Current main still supports `reasoningDefault` only at the per-agent level plus a capability-derived model fallback; it does not define, validate, document, or consume `agents.defaults.models[modelRef].reasoningDefault`. The related open PR #71817 is a useful partial fix for Telegram/Feishu per-agent behavior, but it does not cover the requested per-model configuration surface.

## Best Possible Solution

Add a first-class `reasoningDefault?: "on" | "off" | "stream"` field to `AgentModelEntryConfig` / `agents.defaults.models[modelRef]`, update Zod and generated config schemas/help/docs, and make reply/model-selection resolution honor precedence such as directive, session override, per-agent default, selected-model config, then catalog-derived fallback. Add focused tests for a MiniMax model defaulting to `stream` and a reasoning-capable GPT/OpenAI-family model defaulting to `off`; decide explicitly whether a global `agents.defaults.reasoningDefault` should be implemented or removed from the gateway-tool allowlist.
