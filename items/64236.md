## Summary

Keep this issue open. Current main still has per-agent `reasoningDefault` plus an implicit model-capability fallback, but it does not define or consume a per-model `reasoningDefault` under `agents.defaults.models`, and the related open PR #71817 only partially addresses per-agent channel behavior for Telegram/Feishu.

## Best Possible Solution

Add a first-class per-model reasoning visibility setting, likely `reasoningDefault?: "on" | "off" | "stream"` on `AgentModelEntryConfig` / `agents.defaults.models[modelRef]`, validate it in the generated and Zod schemas, document precedence, and update reply/model-selection resolution so directive and session overrides still win, per-agent defaults remain explicit, selected-model config can override catalog-derived `on`/`off`, and any optional global `agents.defaults.reasoningDefault` support is either implemented consistently or removed from allowlists. Add focused tests for a MiniMax model defaulting to `stream` and a reasoning-capable GPT model defaulting to `off`.
