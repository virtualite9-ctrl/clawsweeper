## Summary

Keep #69519 open. Current main still has the Sessions Limit control as a plain text input with string filter state, so the requested bug fix is not implemented or superseded. The PR addresses a real UI/schema mismatch, but it should be revised before merge to complete the state/type migration or preserve safe coercion and avoid the empty-value/zero sentinel problem raised in review.

## Best Possible Solution

Keep the PR open and ask for a focused revision: make the Limit input numeric with `min=1`/`step=1`, keep the stored state and `SessionsProps` type consistent end to end, preserve blank/invalid input as `undefined` or another explicit non-sentinel state, and keep `sessions.list` params aligned with the integer-minimum schema. Drop unrelated CSS formatting changes unless they are required by the repo formatter.
