## Summary

Keep #69519 open. Current main still renders the Sessions Limit field as a plain text input backed by string filter state, so the requested UI fix is not implemented or superseded. The PR addresses a real Control UI issue, but it needs revision before merge to resolve the type chain and empty-value handling called out in review.

## Best Possible Solution

Keep the PR open and ask for a focused revision: make the Sessions Limit control numeric with `min=1` and `step=1`, keep `SessionsProps`, app state, controller state, and request params type-consistent end to end, preserve blank/invalid editing without a fragile `0` sentinel, and add targeted view/controller coverage. A smaller patch that keeps string UI state but parses at the request boundary would also fit the existing code shape.
