## Summary

Current main still has the reported stale approval UX problem. Pending approvals remain process-memory state, missing approval IDs still surface the gateway sentinel text, the /approve reply path formats that raw text for users, and Telegram approval callbacks still rethrow not-found failures instead of giving stale/restart guidance. The related narrow fix PR #67982 was closed unmerged in the provided GitHub context.

## Best Possible Solution

Keep this issue open. The best near-term fix is to translate isApprovalNotFoundError at both the generic /approve path and the Telegram callback path into clear guidance that the approval is no longer available, may have expired or been invalidated by a gateway restart, and should be re-requested. Add focused regression coverage in src/auto-reply/reply/commands-approve.test.ts and extensions/telegram callback tests. Treat durable approval persistence as a separate product/storage follow-up.
