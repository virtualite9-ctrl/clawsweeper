## Summary

Keep open. Current `main` still has the zero-version stale-snapshot path this PR targets, so there is not strong evidence the request is already implemented or obsolete.

## Evidence

- **Session first-turn path still reuses an existing cached snapshot when refresh is false:** `ensureSkillSnapshot` reads `snapshotVersion`, computes `shouldRefreshSnapshot`, and on first turn reuses `current.skillsSnapshot` unless that flag is true. With an existing persisted snapshot and `shouldRefreshSnapshot === false`, no rebuild happens.
  - file: [src/auto-reply/reply/session-updates.ts:145](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/auto-reply/reply/session-updates.ts#L145)
  - command: `nl -ba src/auto-reply/reply/session-updates.ts | sed -n '140,190p'`
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Zero-version comparison does not force a refresh:** `shouldRefreshSnapshotForVersion` returns `false` for `cachedVersion=0` and `nextVersion=0` because it uses `next === 0 ? cached > 0 : cached < next`. That means a persisted version-0 snapshot is treated as fresh when the watcher never bumped the version.
  - file: [src/agents/skills/refresh-state.ts:65](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/agents/skills/refresh-state.ts#L65)
  - command: `nl -ba src/agents/skills/refresh-state.ts | sed -n '57,75p'`
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Same refresh helper is also used in the agent command path:** The runtime agent path likewise only refreshes when there is no snapshot, the helper says the version changed, or the skill filter changed. A stored version-0 snapshot with current version 0 still skips rebuild there too.
  - file: [src/agents/agent-command.ts:579](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/agents/agent-command.ts#L579)
  - command: `nl -ba src/agents/agent-command.ts | sed -n '572,590p'`
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **This behavior is present in the latest shipped release snapshot:** Release commit `00bd2cf7` already contains the current `session-updates.ts` logic and the new `refresh-state.ts` helper, so there is no later shipped evidence that this PR's fix landed separately.
