## Summary

Keep #47386 open. Current main has per-tool/per-message collapsed rows and a persisted show/hide toggle, but it does not implement the requested WebChat/Control UI summary mode that replaces many tool executions with one per-turn “N tool calls” disclosure or a configurable detailed-vs-summary mode. The related #54838/#54478 thread is narrower: expand tool output by default, not aggregate summary mode.

## Best Possible Solution

Keep the issue open and implement this in the Control UI/WebChat chat-rendering path. The likely product direction is a persisted setting such as `chatToolCallDisplayMode: "summary" | "detailed"`, a desktop/mobile control for that mode, a `buildChatItems` or render-layer grouping step that combines adjacent tool messages from the same assistant turn/run into one count-and-names disclosure, and focused tests proving multiple live and persisted tool messages render as one collapsed summary while expanded mode still exposes each tool card/output. Treat #54838 as related expand-default work, not as the canonical tracker for aggregate summary mode.
