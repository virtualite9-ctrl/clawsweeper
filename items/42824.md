## Summary

Keep this PR open. Current main still lets IRC inherit `agents.defaults.blockStreamingDefault` when `channels.irc.blockStreaming` is unset, which is the behavior this PR is trying to remove for the linked IRC cut-off report. The proposed diff remains narrow and the security pass found no supply-chain-sensitive changes.

## Best Possible Solution

Keep this PR open as an implementation candidate. The best maintainer path is to rebase or recreate the same IRC-owned change on current main, keep the unset/false/true regression coverage, run the IRC targeted tests plus `pnpm check:changed`, and merge it or close it only after an equivalent fix lands.
