## Summary

Keep #26772 open. Current main still performs safe full memory reindexing into a fresh temp SQLite database and swaps it only after the full build succeeds. Failures remove the temp index files, progress is callback-only state, and there is no durable checkpoint/resume schema, config, docs, or regression coverage for interrupted long-running full reindexing. Related #70567 is adjacent and open, but it tracks silent partial success after embedding 500s rather than process-crash resume.

## Best Possible Solution

Keep the issue open until memory-core has an explicit durable full-reindex checkpoint/resume design or maintainers choose a different product direction. A good fix should persist indexing generation/checkpoint state, resume or safely discard incomplete work on restart, prevent search from serving incomplete generations, update memory status/config/docs if new controls are added, and add regression coverage for interrupted full reindex behavior.
