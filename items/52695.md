## Summary

The item is a concrete feature request, but it is out of scope for OpenClaw core. Current project vision routes optional budgeting/accounting/security policy layers to plugins/ClawHub rather than adding new core orchestration subsystems.

## Close Comment

Closing as `clawhub` after a Codex review of current `main`. This request is for an optional accounting/encryption/policy subsystem (`SovereignLedger`, `VaultSeal`, budget enforcement, audit reporting), and the project vision currently routes that kind of capability to plugins rather than OpenClaw core: `VISION.md:54` says optional capability should usually ship as plugins, `VISION.md:66` says new skills should go to ClawHub first, and `VISION.md:99` lists both ClawHub-first skills and heavy orchestration layers as out of scope for core. The plugin docs also make tools, skills, and agent-harness extensions a first-class external plugin path in `docs/tools/plugin.md:11`. Current `main` already has usage primitives in `src/agents/usage.ts`, so a ClawHub/plugin implementation is the right place to build higher-level budgeting or ledger behavior without expanding core scope.
