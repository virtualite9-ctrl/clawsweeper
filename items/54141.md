## Summary

Keep open. Current `main` still appears to have the reported false-positive behavior in the secrets audit path: env shorthand `$VAR` is not recognized as a SecretRef, and auth-profile audit still reports plaintext whenever `entry.value` is a non-empty string even if a ref was detected. The PR also includes targeted tests that are not present on `main`.

## Evidence

- **`$VAR` shorthand still unsupported on main:** `parseEnvTemplateSecretRef` only defines `ENV_SECRET_TEMPLATE_RE = /^\$\{...\}$/` and only executes that regex against `value.trim()`, so bare `$OPENAI_API_KEY` will return `null`.
  - file: [src/config/types.secrets.ts:21](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/config/types.secrets.ts#L21)
