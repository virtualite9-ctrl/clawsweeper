## Summary

Keep #72153 open. Current main still constructs the Windows config opener as `Start-Process -LiteralPath`, and the current regression test also locks in that invalid form. PowerShell in this environment confirms `Start-Process` exposes `FilePath` but not `LiteralPath`; the linked issue #69933 contains the same runtime `ParameterBindingException`, and the PR's one-line source/test change addresses the implicated code path.

## Best Possible Solution

Keep the PR open and land this narrow bug fix, or apply an equivalent change on main: replace `Start-Process -LiteralPath` with `Start-Process -FilePath` in `src/gateway/server-methods/config.ts` and update `src/gateway/server-methods/config.test.ts` so the Windows command regression test locks in the valid PowerShell parameter. Normal maintainer CI/changed-lane validation should run before merge.
