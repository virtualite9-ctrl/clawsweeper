## Summary

Keep PR #72153 open. Current main still uses `Start-Process -LiteralPath` for Windows `config.openFile`, while PowerShell’s `Start-Process` contract supports `-FilePath` and not `-LiteralPath`. The PR is a narrow, relevant fix for linked bug #69933 and should not be closed as stale or obsolete.

## Best Possible Solution

Keep PR #72153 open for maintainer review and merge, assuming CI passes. The best product/code direction is the PR’s narrow change: update the Windows `config.openFile` command to `Start-Process -FilePath` while preserving the existing single-quote escaping, and update the regression test to assert `-FilePath`. After merge, linked bug #69933 can be closed as fixed by the landing commit.
