## Summary

Keep PR #39715 open. Current main still does not implement the proposed stdout/stderr drain before background exec output is snapshotted, and the focused regression coverage from the PR is not present. The related sandbox write/stdin fix does not supersede this stdout/background-output issue.

## Best Possible Solution

Keep PR #39715 open. The best path is to land or adapt a minimal stdout/stderr drain before RunExit and finished-session aggregation are snapshotted, keep the settled/cancel/runId safeguards from the PR review discussion, and add focused regression coverage for backgrounded stdout/stderr flushed at process exit.
