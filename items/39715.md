## Summary

Keep open. Current main does not contain the proposed stdout/stderr drain fix or its regression coverage, and the underlying race remains plausible in the current process-supervisor path.

## Evidence

- **Child adapter still resolves wait immediately on child close:** `createChildAdapter()` settles `wait()` directly from the child `close` event, with no post-close event-loop yield to let pending stdout/stderr `data` callbacks drain first.
  - file: [src/process/supervisor/adapters/child.ts:251](https://github.com/openclaw/openclaw/blob/cb2c36b04975e34ab7014a11575d6589aa96ae32/src/process/supervisor/adapters/child.ts#L251)
  - command: `nl -ba src/process/supervisor/adapters/child.ts | sed -n '220,320p'`
  - sha: [cb2c36b04975](https://github.com/openclaw/openclaw/commit/cb2c36b04975e34ab7014a11575d6589aa96ae32)
- **Supervisor still snapshots output immediately after adapter.wait():** `waitPromise` awaits `adapter.wait()`, then immediately marks the run settled, disposes the adapter, computes `RunExit`, and finalizes the registry. There is no drain/yield step here either.
  - file: [src/process/supervisor/supervisor.ts:201](https://github.com/openclaw/openclaw/blob/cb2c36b04975e34ab7014a11575d6589aa96ae32/src/process/supervisor/supervisor.ts#L201)
  - command: `nl -ba src/process/supervisor/supervisor.ts | sed -n '1,320p'`
  - sha: [cb2c36b04975](https://github.com/openclaw/openclaw/commit/cb2c36b04975e34ab7014a11575d6589aa96ae32)
- **Exec runtime consumes aggregated output as soon as wait resolves:** Background exec completion builds the final outcome from `session.aggregated.trim()` immediately after `managedRun.wait()` resolves, then marks the session exited. If output arrives after the wait resolution turn, that final snapshot can miss it.
  - file: [src/agents/bash-tools.exec-runtime.ts:770](https://github.com/openclaw/openclaw/blob/cb2c36b04975e34ab7014a11575d6589aa96ae32/src/agents/bash-tools.exec-runtime.ts#L770)
  - command: `nl -ba src/agents/bash-tools.exec-runtime.ts | sed -n '760,860p'`
  - sha: [cb2c36b04975](https://github.com/openclaw/openclaw/commit/cb2c36b04975e34ab7014a11575d6589aa96ae32)
- **PR regression coverage is not present on main:** The PR's added regression file for backgrounded stdout capture is absent from the checkout, so there is no proof the reported case was absorbed elsewhere on main.
  - command: `rg --files src/agents | rg 'background-stdout|bash-tools\.exec'`
  - sha: [cb2c36b04975](https://github.com/openclaw/openclaw/commit/cb2c36b04975e34ab7014a11575d6589aa96ae32)
