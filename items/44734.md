## Summary

Keep #44734 open. Current main still treats `.ogg` and `.opus` as eligible multimodal memory audio files, preserves their detected MIME type, and sends the raw bytes to Gemini `inlineData` without provider-specific format filtering or transcoding. Official Gemini Embedding 2 docs currently list audio formats as MP3 and WAV, so the reported OGG/Opus rejection remains an actionable OpenClaw bug rather than a stale or already-implemented item.

## Best Possible Solution

Keep the issue open and fix the Gemini multimodal memory path. The conservative fix is provider-aware audio format handling: either filter Gemini audio inputs to MP3/WAV with a clear skip warning and docs update, or transcode unsupported audio such as OGG/Opus to WAV/MP3 before `buildGeminiEmbeddingRequest`. Add targeted tests for `.ogg` and `.opus` files under `extraPaths` so indexing no longer fails the whole run, and align `docs/reference/memory-config.md` with the shipped behavior.
