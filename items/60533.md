## Summary

Keep open. Current `main` still uses `scripts/docker/setup.sh` plus `docker-compose.extra.yml` to add `OPENCLAW_HOME_VOLUME`, and the PR is not implemented there. The PR patch shown in the issue context would instead hard-code `${OPENCLAW_HOME_VOLUME:-openclaw_home}:/home/node` into `docker-compose.yml` while dropping the current dynamic named-volume declaration path, which matches the unresolved P1 review about custom named volumes breaking for non-default names.
