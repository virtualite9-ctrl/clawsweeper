## Summary

Keep PR #70968 open. Current main still persists managed environment values into the gateway install plan, so the PR is not obsolete or implemented-on-main. The patch is a plausible focused fix for #70612, but it needs maintainer/author follow-through on preserving service-owned variables such as PATH before merge.

## Best Possible Solution

Keep PR #70968 open. The best path is to revise the patch so LaunchAgent install plans omit literal managed secret/durable values while preserving generated service-owned fields such as PATH, HOME, TMPDIR, and any required OPENCLAW_* service metadata, then update daemon-install regression tests and resolve the existing review thread before merge.
