## Summary

Keep open. Current `main` still drops the Control UI gateway token on browser-close by design: the token is stored in `sessionStorage`, scrubbed from persisted settings, covered by tests, and documented as current-tab-only behavior. That means the report is still valid against current code, but it is a product/security decision rather than an already-fixed bug, so none of the allowed close reasons apply.

## Risks / Open Questions

- Changing this to localStorage would alter documented admin-surface auth handling and increases the persistence window for gateway credentials on shared or compromised browsers.
- Because the current behavior is intentional and tested, closing as fixed/cannot-reproduce would be incorrect; any change should be handled as an explicit product/security decision.
