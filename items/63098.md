## Summary

Keep this issue open. Current main still has the same Discord voice join failure path: `/vc join` waits for `@discordjs/voice` to reach Ready with a fixed 15s timeout, destroys the connection on failure, and returns the reported `Failed to join voice channel: ...` message. The requested `connectTimeoutMs`/retry control is still absent from the config schema, types, UI hints, generated metadata, and Discord docs. Related open reports #39825, #63125, and #65039 corroborate the same early Discord voice join failure family rather than proving this issue is fixed or cleanly superseded.

## Best Possible Solution

Keep this issue open and fix the Discord plugin voice join path. Add focused regression coverage for initial Ready abort/timeout before session activation, then choose a longer default, bounded retry/backoff, and/or a documented `channels.discord.voice.connectTimeoutMs` control. If a new knob lands, update `DiscordVoiceConfig`, strict zod validation, generated config metadata/UI hints, and Discord docs. After the fix, reconcile #39825, #63125, and #65039 as duplicates, follow-ups, or fixed reports based on the final root cause.
