## Summary

Keep this issue open. Current main still has the reported Discord voice join abort shape: `/vc join` calls the voice manager, the voice manager waits for the connection to reach Ready with a fixed 15s timeout, destroys the connection on failure, and returns the same user-visible failure message. The requested longer/configurable connect timeout and retry behavior are not present in source, config schema, docs, or tests. Related #39825 and #63125 are still open and do not supersede this issue.

## Best Possible Solution

Keep this issue open and fix the Discord plugin voice join path. Add a focused regression test for initial `entersState` abort/timeout before session activation, decide on a longer default and/or bounded retry/backoff, and expose a documented `channels.discord.voice.connectTimeoutMs`-style setting only if maintainers want operator tuning. Any config change should update the type, zod schema, config metadata/UI hints, and Discord docs, then reconcile related open reports #39825 and #63125 after the chosen behavior lands.
