## Summary

Keep this issue open. Current main still routes Discord `/vc join` through a fixed 15s Ready wait, destroys the voice connection on that abort path, and returns the same user-visible `Failed to join voice channel: ...` error. The config schema, types, generated config metadata, UI hints, and Discord docs still do not expose a voice connect timeout or join retry policy. Related open reports #39825 and #65039 corroborate the same early Discord voice join failure family rather than proving this issue is fixed or cleanly superseded.

## Best Possible Solution

Keep this issue open and fix the Discord plugin voice join path. Add a focused regression test for initial Ready abort or timeout before session activation, then choose a longer default, bounded retry/backoff, and/or a documented `channels.discord.voice.connectTimeoutMs` control. If a new knob lands, update `DiscordVoiceConfig`, strict zod validation, generated config metadata/UI hints, and Discord docs, then reconcile related open reports #39825, #63125, and #65039 once maintainers choose a canonical tracker or the behavior is fixed.
