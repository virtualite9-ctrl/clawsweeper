## Summary

Keep this issue open. Current main still matches the reported Discord voice join abort path: `/vc join` waits for `manager.join`, initial voice readiness still uses a fixed 15s timeout, the failure path destroys the connection and returns `Failed to join voice channel: ...`, and there is no configurable connect timeout or retry policy in the Discord voice config, schema, UI hints, docs, or tests.

## Best Possible Solution

Keep this issue open and fix the Discord plugin voice join path. Add a focused regression test for an initial `entersState` abort/timeout before session activation, then decide on a longer default, bounded retry/backoff, and/or documented `channels.discord.voice.connectTimeoutMs` operator tuning. If a new config knob lands, update the Discord config type, zod schema, metadata/UI hints, and Discord docs, then reconcile related open reports #39825 and #63125 after the behavior is fixed.
