## Summary

Keep #8813 open. Current main still unconditionally emits read-based Skills guidance and does not prefer a registered `Skill` tool, even though runtime tool names are available to system-prompt construction and plugin tools can be registered. The existing Codex review comment remains accurate against current main SHA 7e13f3f5140168e31027b39a9855ade3fadf24d5.

## Best Possible Solution

Keep #8813 open. The likely fix belongs in `src/agents/system-prompt.ts`: pass the normalized available-tool set and casing resolver into `buildSkillsSection`, prefer invoking a registered `Skill` tool when present, preserve the read-based fallback when absent, and update system-prompt tests plus public Skills/system-prompt docs for both behaviors.
