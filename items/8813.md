## Summary

Keep #8813 open. Current main still emits unconditional read-based Skills guidance and does not pass normalized available-tool information into `buildSkillsSection`, so a plugin-registered `Skill` tool cannot change the prompt behavior requested by the issue. The existing April 26 Codex review comment reached the same conclusion, and current main at 406ae72fd2789c78053f6b94006ec8f488bc1c00 still matches that evidence.

## Best Possible Solution

Keep #8813 open. The best fix likely belongs in `src/agents/system-prompt.ts`: pass normalized available-tool information plus caller casing into `buildSkillsSection`, prefer invoking a registered Skill tool when present, preserve the existing `read` fallback otherwise, then update `src/agents/system-prompt.test.ts` and the public Skills/system-prompt docs for both behaviors.
