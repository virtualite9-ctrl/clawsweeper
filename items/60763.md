## Summary

Keep open. Current `main` does not implement `expiresAtMs` or `bindSession`, and the PR still carries unresolved authorization and lifetime-semantics questions raised in review. There is not strong evidence to conservatively close this PR under any allowed close reason.

## Risks / Open Questions

- Merging the PR as discussed would change Discord interaction routing so callbacks can run on the current channel route instead of the originating bound session, with no built-in requirement for `allowedUsers`.
- Passing `expiresAtMs` through as proposed would let callers create arbitrarily long-lived registry entries because the current registry does not bound explicit expirations.
