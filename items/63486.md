## Summary

Keep open. This is a recent, concrete Matrix regression report where the reporter says only OpenClaw changed and reverting OpenClaw restored interaction; current main fixes relevant Matrix diagnostic and encrypted-message subpaths, but I did not find source, test, release, or discussion evidence proving the broader no-response/no-inbound-log symptom is fixed for Docker plus local Synapse.

## Best Possible Solution

Keep this issue open until a Matrix owner reproduces or verifies current main or the next release in a Docker deployment with local Synapse. Capture Matrix gateway startup logs, sync readiness state, room encryption status, `openclaw matrix verify status --verbose`, and whether inbound events reach `room.message`, `room.decrypted_event`, or `room.failed_decryption`. If current main fixes the deployment, close as implemented with the specific Matrix commit and release evidence; if it still reproduces, the likely fix belongs in `extensions/matrix` around shared-client startup, sync readiness, encrypted/decrypted event delivery, or inbound monitor logging.
