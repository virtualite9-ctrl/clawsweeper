## Summary

Keep open. This is a recent, concrete Matrix Docker/Synapse regression report. Current main has related Matrix startup, readiness, AbortError, and verify-status handling, but I did not find source, test, changelog, or release evidence proving that the no-response/no-inbound-log chat regression is fixed. The most directly related sync-readiness changelog entry shipped in v2026.4.9, which is the version the reporter says introduced the breakage.

## Best Possible Solution

Keep this issue open and reproduce against current main or the latest release using Docker with a local Synapse setup like the reporter described. Capture gateway startup logs, Matrix sync state/status, `openclaw matrix verify status --verbose`, and whether room events reach `registerMatrixMonitorEvents`. If it still reproduces, the likely fix belongs in the Matrix plugin shared-client startup/sync lifecycle or Matrix event monitor path, with a regression test around abort/readiness handling and inbound room-event delivery.
