## Summary

Keep open. This is a recent concrete Matrix regression report with rollback evidence from the reporter. Current main contains adjacent Matrix startup, E2EE, and `verify status` fixes, but I did not find source, test, release, or discussion evidence proving the broader Docker plus local Synapse no-response/no-inbound-log symptom is fixed.

## Best Possible Solution

Keep this issue open for a Matrix owner to reproduce or verify current main or the next release in Docker with local Synapse. Capture Matrix startup logs, `openclaw channels status --probe`, `openclaw matrix verify status --verbose`, room encryption state, and whether inbound events reach `room.message`, `room.decrypted_event`, or `room.failed_decryption`. If current main fixes the deployment, close as implemented with the exact Matrix commit and release; otherwise patch `extensions/matrix` around shared-client startup/readiness, inbound monitor logging, encrypted/decrypted event delivery, or the relevant routing gate.
