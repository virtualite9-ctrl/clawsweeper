## Summary

Keep open. Current `main` does not implement per-agent `userTimezone`: `AgentEntrySchema` has no `userTimezone` field, resolved agent overrides do not carry one, and the prompt/heartbeat/envelope time paths still read only `agents.defaults.userTimezone`.

## Evidence

- **Per-agent config schema has no userTimezone field:** `AgentEntrySchema` includes per-agent fields like `model`, `heartbeat`, `identity`, `tools`, and `runtime`, but no `userTimezone`, so `agents.list[].userTimezone` is not accepted on current main.
  - file: [src/config/zod-schema.agent-runtime.ts:812](https://github.com/openclaw/openclaw/blob/bfc9cb92c56911df39c0e156b0e98970ec5e101a/src/config/zod-schema.agent-runtime.ts#L812)
  - command: `nl -ba src/config/zod-schema.agent-runtime.ts | sed -n '812,855p'`
  - sha: [bfc9cb92c569](https://github.com/openclaw/openclaw/commit/bfc9cb92c56911df39c0e156b0e98970ec5e101a)
- **Resolved agent overrides do not expose userTimezone:** `ResolvedAgentConfig` has no `userTimezone` property, and `resolveAgentConfig()` returns agent-scoped values for many fields but not timezone, so there is no agent-level override path to consume.
  - file: [src/agents/agent-scope-config.ts:16](https://github.com/openclaw/openclaw/blob/bfc9cb92c56911df39c0e156b0e98970ec5e101a/src/agents/agent-scope-config.ts#L16)
  - command: `nl -ba src/agents/agent-scope-config.ts | sed -n '14,135p'`
  - sha: [bfc9cb92c569](https://github.com/openclaw/openclaw/commit/bfc9cb92c56911df39c0e156b0e98970ec5e101a)
- **System prompt time context still uses global defaults only:** `buildSystemPromptParams()` resolves timezone from `params.config?.agents?.defaults?.userTimezone`, ignoring `agentId` for timezone selection.
  - file: [src/agents/system-prompt-params.ts:50](https://github.com/openclaw/openclaw/blob/bfc9cb92c56911df39c0e156b0e98970ec5e101a/src/agents/system-prompt-params.ts#L50)

- **Heartbeat active-hours timezone still falls back to global default:** When `activeHours.timezone` is unset or `"user"`, `resolveActiveHoursTimezone()` uses `cfg.agents?.defaults?.userTimezone`, not any per-agent setting.
  - file: [src/infra/heartbeat-active-hours.ts:12](https://github.com/openclaw/openclaw/blob/bfc9cb92c56911df39c0e156b0e98970ec5e101a/src/infra/heartbeat-active-hours.ts#L12)
