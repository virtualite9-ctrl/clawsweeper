## Summary

Current main already gives isolated cron `agentTurn` jobs explicit `payload.model` precedence and forwards that resolved provider/model into the actual execution path. The same runtime implementation is present in the shipped `v2026.4.23` release.

## Evidence

- **Model precedence in isolated cron selection:** `resolveCronModelSelection(...)` reads `payload.model` for `agentTurn` jobs and applies it before falling back to stored cron-session overrides or normal agent/default selection.
  - file: [src/cron/isolated-agent/model-selection.ts:104](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/cron/isolated-agent/model-selection.ts#L104)
  - sha: [a9797214338b](https://github.com/openclaw/openclaw/commit/a9797214338ba31c52c796adbb75afb16e0684a9)
- **Resolved model stored for the run context:** After model selection, the isolated cron path stores the resolved provider/model in `liveSelection`, which is the execution context passed downstream for the run.
  - file: [src/cron/isolated-agent/run.ts:670](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/cron/isolated-agent/run.ts#L670)
  - sha: [a9797214338b](https://github.com/openclaw/openclaw/commit/a9797214338ba31c52c796adbb75afb16e0684a9)
- **Execution path uses the resolved override:** The executor passes `liveSelection.provider` and `liveSelection.model` into `runWithModelFallback(...)`, and then forwards the chosen provider/model into both CLI and embedded isolated-agent execution.
  - file: [src/cron/isolated-agent/run-executor.ts:110](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/cron/isolated-agent/run-executor.ts#L110)
  - sha: [a9797214338b](https://github.com/openclaw/openclaw/commit/a9797214338ba31c52c796adbb75afb16e0684a9)
- **Regression coverage for the reported behavior:** Regression tests assert that an allowed isolated-cron `payload.model` becomes the actual run model instead of the agent default, and that the override is forwarded into the embedded runner path.
  - file: [src/cron/isolated-agent/run.skill-filter.test.ts:192](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/cron/isolated-agent/run.skill-filter.test.ts#L192)
  - sha: [a9797214338b](https://github.com/openclaw/openclaw/commit/a9797214338ba31c52c796adbb75afb16e0684a9)
- **User-facing docs match the current behavior:** The cron docs state that `openclaw cron add|edit --model ...` uses that selected allowed model for the isolated job, with precedence ahead of stored cron-session overrides and agent/default selection.
  - file: [docs/automation/cron-jobs.md:345](https://github.com/openclaw/openclaw/blob/8b76392e3e7934b1cdd6d461148263fd56141ec8/docs/automation/cron-jobs.md#L345)
  - sha: [8b76392e3e79](https://github.com/openclaw/openclaw/commit/8b76392e3e7934b1cdd6d461148263fd56141ec8)
- **Shipped release contains the same runtime files:** Comparing `v2026.4.23` to current `main` for the isolated-cron runtime/test files shows no runtime diff; only docs changed in that slice, so the implementation was already shipped in the latest release.
  - command: `git diff --stat a9797214338ba31c52c796adbb75afb16e0684a9..8b76392e3e7934b1cdd6d461148263fd56141ec8 -- src/cron/isolated-agent/model-selection.ts src/cron/isolated-agent/run.ts src/cron/isolated-agent/run-executor.ts src/cron/isolated-agent/run.skill-filter.test.ts src/cron/isolated-agent/run.cron-model-override-forwarding.test.ts src/cron/isolated-agent/run.cron-model-override.test.ts docs/automation/cron-jobs.md docs/cli/cron.md`
