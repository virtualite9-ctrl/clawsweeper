## Summary

Keep #69379 open. Current main still computes preflight-compaction and memory-flush thresholds from `reserveTokensFloor` alone, while the existing Pi settings path and docs treat the floor as a minimum for `reserveTokens`. The PR remains a focused, relevant fix for open issue #66830; the related #67297 PR is closed unmerged, so there is no shipped or canonical replacement to close this against.

## Best Possible Solution

Keep #69379 open and review it as the active implementation path for #66830. The best fix is to compute an effective reserve as `max(agents.defaults.compaction.reserveTokens, reserveTokensFloor)`, thread that value through the preflight and memory-flush gates without changing the public `MemoryFlushPlan` contract, and keep the regression tests covering `reserveTokens > reserveTokensFloor` plus the existing small-window clamp behavior.
