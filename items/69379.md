## Summary

Keep PR #69379 open. Current main still computes both preflight compaction and memoryFlush thresholds from `reserveTokensFloor` only, while the documented/settings-manager semantics treat the effective reserve as at least `max(reserveTokens, reserveTokensFloor)`. The PR is a focused bug fix for open issue #66830, and the only related implementation PR called out in context (#67297) is closed unmerged, so this is not implemented or superseded on main.

## Best Possible Solution

Keep #69379 open as the active useful implementation path for #66830. The best maintainer outcome is to review/rebase it on current main if needed, preserve the regression coverage for `reserveTokens > reserveTokensFloor`, and land a fix that makes preflight and memoryFlush gates use the same effective reserve semantics already used by Pi settings.
