## Summary

Keep open. Current main still lacks a `memorySearch.local.gpu` config field and does not pass any GPU backend option to `node-llama-cpp` for local embeddings. The PR is stale against current main and needs more integration work, but the requested capability remains real and is supported by the upstream `node-llama-cpp@3.18.1` API. The author's Orange Pi / Vulkan production verification is useful evidence, not a reason to close.

## Best Possible Solution

Keep the PR open and ask for a refresh against current main. The best implementation is to add a typed `memorySearch.local.gpu` union, validate it in the runtime schema, generate config schema/help/labels/docs, carry it through `ResolvedMemorySearchConfig.local`, expose it through the memory embedding provider option types, and pass it to `getLlama` only when explicitly configured so `false` disables GPU while `undefined` preserves existing auto behavior. Add focused tests for config resolution/schema acceptance and `createLocalEmbeddingProvider()` passing `gpu` to `getLlama`.
