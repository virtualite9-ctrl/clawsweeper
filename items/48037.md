## Summary

Current `main` does not implement configurable Chrome DevTools MCP args for `existing-session` profiles. The MCP launcher still uses a fixed arg list, the browser profile config/schema has no `mcpArgs` field, and docs/tests only cover `userDataDir` as the existing-session-specific override.

## Evidence

- **Chrome MCP launch args are still hardcoded:** `DEFAULT_CHROME_MCP_ARGS` is a fixed array, and `buildChromeMcpArgs()` only accepts `userDataDir`, appending `--userDataDir` when present. There is no hook for arbitrary extra MCP CLI args.
  - file: [extensions/browser/src/browser/chrome-mcp.ts:37](https://github.com/openclaw/openclaw/blob/d38ed0831d7372c8505dc944aea69b5b83da1435/extensions/browser/src/browser/chrome-mcp.ts#L37)
  - command: `nl -ba extensions/browser/src/browser/chrome-mcp.ts | sed -n '30,235p'`
  - sha: [d38ed0831d73](https://github.com/openclaw/openclaw/commit/d38ed0831d7372c8505dc944aea69b5b83da1435)
- **Browser profile type has no `mcpArgs` field:** `BrowserProfileConfig` exposes `cdpPort`, `cdpUrl`, `userDataDir`, `driver`, `attachOnly`, and `color` only.
  - file: [src/config/types.browser.ts:1](https://github.com/openclaw/openclaw/blob/d38ed0831d7372c8505dc944aea69b5b83da1435/src/config/types.browser.ts#L1)
