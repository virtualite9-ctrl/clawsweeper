## Summary

Keep open. Current main has substantial memory infrastructure, but the requested MVP operator surface is still incomplete: the default memory plugin exposes search/get plus adjacent memory subcommands, not first-class remember/recall/review/edit/forget CLI or skill workflows with provenance and match explanations. The prior Engram Core comment and the bundled memory-lancedb plugin are useful related work, but neither resolves the default memory-core product/API decision, and #42650 remains the paired open issue for review/edit/forget/tombstone semantics.

## Best Possible Solution

Keep this open and narrow the v1 operator contract. A maintainer should decide whether the MVP lives as `memory-core` CLI subcommands, a bundled skill surface, plugin tools, or a documented alias layer: `recall` can likely wrap `memory search`, `remember` needs a provenance-preserving write path, and `review`/`edit`/`forget` should be coordinated with #42650. Reuse existing daily-note, session-transcript, grounded backfill, dreaming, memory-wiki, and plugin-tool surfaces rather than creating a parallel memory store.
