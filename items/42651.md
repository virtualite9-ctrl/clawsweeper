## Summary

Keep open. Current main has substantial memory infrastructure, but the requested MVP operator surface is still not implemented in the default memory path: there are no first-class remember/recall/review/edit/forget CLI or skill workflows, default tools are still search/get only, and recall output does not yet include the requested match-explanation signal.

## Best Possible Solution

Keep this open and narrow the v1 operator contract. A maintainer should decide whether the MVP is implemented as memory-core CLI subcommands, a bundled skill surface, plugin tools, or documented aliases. recall can likely wrap memory search, remember needs a provenance-preserving write path, and review/edit/forget should be designed with #42650 and existing dreaming, backfill, memory-wiki, and plugin-tool surfaces rather than creating a parallel memory store.
