## Summary

This PR should close as ClawHub/community plugin work. Current main explicitly keeps optional channel integrations and plugin promotion outside core, the PR only adds repo docs/catalog/credit wiring for an existing external Yuanbao plugin, and the current public plugin/channel SDK has the seams needed for an external messaging channel without a missing OpenClaw core API.

## Best Possible Solution

Do not merge this OpenClaw source PR. Keep Yuanbao in the external plugin repository/package and publish or promote it through ClawHub. If maintainers later want official OpenClaw catalog promotion, open a focused curated-catalog PR with exact pinned npmSpec, expectedIntegrity, provenance/security review, and explicit owner approval; add core SDK work only if the external plugin proves a missing public channel API.
