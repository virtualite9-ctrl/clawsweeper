## Summary

Keep open. Current main has partial A2A plumbing improvements, especially a narrow skip for parent-owned ACP child sessions, but the general `sessions_send` announce path still uses `Agent-to-agent announce step.` as a real inter-session user turn, only treats `ANNOUNCE_SKIP`/`REPLY_SKIP` as A2A control tokens, and does not hide inter-session user turns from chat history. That leaves the reported placeholder, `NO_REPLY`, and follow-up echo surfaces plausibly unresolved for normal peer sessions.

## Best Possible Solution

Keep this issue open. The likely fix belongs in the `sessions_send` A2A plumbing: suppress exact silent/control replies before ping-pong and announce delivery, avoid persisting or displaying the announce placeholder as a normal user turn, and add focused regression coverage for `sessions_send({ timeoutSeconds: 0 })` with `NO_REPLY`, placeholder text, and acknowledgement follow-up cases.
