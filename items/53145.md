## Summary

Current main still leaves the reported peer `sessions_send` A2A path open: fire-and-forget sends still start announce delivery, the A2A loop only recognizes `REPLY_SKIP`/`ANNOUNCE_SKIP`, and the announce phase still submits the literal placeholder as an inter-session agent input. Related fixes/docs cover adjacent surfaces, but not this specific Discord-backed peer `sessions_send({ timeoutSeconds: 0 })` behavior.

## Best Possible Solution

Keep this issue open. The best fix belongs in `src/agents/tools/sessions-send-tool.a2a.ts` and adjacent helpers: apply shared silent/control-reply classification before ping-pong and announce delivery, treat exact `NO_REPLY` and heartbeat acknowledgements as non-deliverable in this path, avoid submitting the announce placeholder as a normal persisted user turn, and add focused regression tests for peer `sessions_send({ timeoutSeconds: 0 })` covering placeholder suppression, `NO_REPLY` suppression, and no echo loop.

## Risks / Open Questions

- No live Discord reproduction was run because this review was read-only, but current source, docs, and tests are sufficient to show the peer A2A gap remains.
