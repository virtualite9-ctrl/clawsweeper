## Summary

Current main still leaves the reported peer `sessions_send({ timeoutSeconds: 0 })` A2A announce path unresolved. Fire-and-forget peer sends still start the A2A announce flow, the reply loop only treats `REPLY_SKIP` as a stop token, and the announce step still submits the literal placeholder message as an inter-session user turn. Adjacent display-layer and subagent announce fixes suppress `NO_REPLY` elsewhere, but this specific sessions_send A2A path does not use the shared silent/control classification before reinjection or announce delivery.

## Best Possible Solution

Keep this issue open. The best fix belongs in `src/agents/tools/sessions-send-tool.a2a.ts` and adjacent sessions-send helpers: classify `NO_REPLY`, heartbeat acknowledgements, `ANNOUNCE_SKIP`, and `REPLY_SKIP` through a shared control-reply predicate before ping-pong and announce delivery; avoid submitting the announce placeholder as a normal persisted user turn; and add focused regression tests for peer `sessions_send({ timeoutSeconds: 0 })` covering placeholder suppression, `NO_REPLY` suppression, and no echo loop.
