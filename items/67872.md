## Summary

Keep this PR open. Current main still drops Slack no-mention channel messages before any message hook path can run, so the PR is not obsolete. The submitted implementation still needs revision because it reads `channels.slack.groups`, while Slack's valid channel policy surface is `channels.slack.channels` and currently has no typed/schema-backed `ingest` field.

## Best Possible Solution

Keep this PR open and request a revision: add a Slack-specific `ingest` option on `channels.slack.channels`, wire it through Slack config types, strict Zod schema, generated config metadata/docs, and `resolveSlackChannelConfig` with wildcard plus channel-specific override semantics; then emit internal `message:received` from the no-mention skip path only when that resolved Slack policy is true, with focused regression coverage for wildcard ingest and channel-specific `ingest: false` override.

## Risks / Open Questions

- Merging as submitted likely would not enable the intended behavior for valid Slack configs because `resolveChannelGroupPolicy` reads `groups`, not `channels.slack.channels`.
- Silent ingest for mention-skipped Slack channel messages changes content exposure semantics; it should be a documented, typed, opt-in Slack channel policy.
- The submitted patch has a minor unreachable `isRoom` ternary in the new no-mention branch, already noted in review.
