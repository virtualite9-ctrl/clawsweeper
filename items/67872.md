## Summary

Keep this PR open. Current main still returns from Slack's no-mention skip path before message hooks can be emitted, so the PR is not obsolete. The submitted approach still needs revision because it reads the generic `groups` policy surface, while Slack channel policy is resolved from `channels.slack.channels` and that public Slack config surface does not currently define an `ingest` flag.

## Best Possible Solution

Keep this PR open and request a revision: add a Slack-specific `ingest` policy to `channels.slack.channels`, wire it through Slack types, strict Zod schema, generated config metadata/docs, and `resolveSlackChannelConfig` with wildcard plus channel-specific override semantics, then emit internal `message:received` from the no-mention skip path only when that resolved Slack policy is true and keep focused regression coverage for wildcard ingest and channel-specific `ingest: false` override.

## Risks / Open Questions

- Merging as submitted likely would not enable the intended behavior for valid Slack configs because `resolveChannelGroupPolicy` reads `groups`, not `channels.slack.channels`.
- Adding silent ingest for mention-skipped Slack channel messages changes content exposure semantics; it should be documented and opt-in via Slack's typed channel config/schema/metadata surface.
- The submitted patch contains an unreachable `isRoom` ternary branch noted in review because the code is inside an `isRoom`-guarded block.
