## Summary

Keep #26037 open. Current main has bundled Qwen Cloud/Coding Plan support, but it still does not provide official Bailian Coding Plan reasoning/thinking support for the requested qwen3.5-plus, glm-5, and kimi-k2.5 path.

## Best Possible Solution

Keep the issue open and implement this in the bundled qwen provider plugin, not as a ClawHub-only idea. The likely maintainer path is to verify Bailian Coding Plan thinking semantics for qwen3.5-plus, glm-5, kimi-k2.5, and current Coding Plan models, then add provider-owned reasoning metadata and a Qwen/Bailian thinking payload adapter, plus docs and tests for catalog discovery, payload construction, and the reported gateway restart/config path.
