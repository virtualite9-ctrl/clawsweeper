## Summary

Keep #26037 open. Current main still has bundled Qwen/Coding Plan support, but it does not implement official Bailian/Qwen Coding Plan thinking support for the requested qwen3.5-plus, glm-5, and kimi-k2.5 path. The relevant Qwen catalog entries still advertise `reasoning: false`, the generic OpenAI-compatible reasoning payload path is gated on that metadata, and I found no provider-owned Bailian `enable_thinking` adapter or Qwen docs/tests that make the requested behavior official.

## Best Possible Solution

Keep the issue open and implement this in the bundled Qwen provider plugin. The likely maintainer path is to verify Bailian Coding Plan thinking semantics for qwen3.5-plus, glm-5, kimi-k2.5, and current Coding Plan models; add provider-owned reasoning metadata and a Qwen/Bailian `enable_thinking`/thinking-budget payload adapter; document the official setup at https://docs.openclaw.ai/providers/qwen; and add tests for catalog discovery, payload construction, config validation, and the reported WebUI/gateway restart path.
