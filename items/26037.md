## Summary

Keep #26037 open. Current main has Qwen Cloud / DashScope Coding Plan onboarding and separate Z.AI/Kimi reasoning-capable providers, but the Ali Bailian/Qwen Coding Plan catalog still marks the requested qwen3.5-plus, glm-5, and kimi-k2.5 models as non-reasoning, and the shared OpenAI-compatible transport only sends reasoning payload fields when model metadata advertises reasoning support.

## Best Possible Solution

Keep the issue open and implement this in the Qwen provider surface if Bailian confirms supported controls. The likely fix is to verify reasoning behavior for qwen3.5-plus, glm-5, and kimi-k2.5 on the Coding Plan endpoints, then either mark only verified Qwen catalog entries as reasoning-capable or add a Qwen-owned thinking payload adapter, with docs and regression tests for catalog discovery, payload construction, and the reported gateway restart/config path.
