## Summary

Keep open. Current main still has the behavior this PR is trying to change: Ollama synthetic auth is gated only on “meaningful explicit config,” not on whether the baseUrl is local/LAN versus remote. The PR is a focused, unmerged fix with relevant tests; #59954 is closed unmerged and #43945 remains the broader follow-up, so there is no strong duplicate, superseded, or implemented-on-main close reason.

## Best Possible Solution

Keep #69857 open and ask for a small revision: add a shared Ollama baseUrl locality classifier, use it consistently in both `extensions/ollama/index.ts` and `extensions/ollama/provider-discovery.ts`, cover remote Cloud/public IP and local/LAN/mDNS/bare-host cases in tests, and address the loopback alias / IPv6 review comments. Continue tracking broader subagent credential and fallback privacy work under #43945.
