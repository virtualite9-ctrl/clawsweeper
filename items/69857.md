## Summary

Keep open. Current main still mints the `ollama-local` synthetic key for explicit non-default/remote Ollama configs, and existing tests assert that behavior. The PR is not implemented on main or superseded; it remains a plausible bundled Ollama plugin fix, but it needs maintainer review/fixup before merge.

## Best Possible Solution

Keep the PR open and rework it into a complete bundled Ollama plugin fix: put the local/remote `baseUrl` classifier in shared Ollama discovery code, use it from both `extensions/ollama/index.ts` and `extensions/ollama/provider-discovery.ts`, add regression tests proving remote Cloud/public URLs return `undefined` while local/LAN/mDNS/bare-host configs keep synthetic auth, and leave #43945 open for the broader subagent credential/fallback privacy work.
