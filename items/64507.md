## Summary

Keep this issue open. Current main still lacks a typed or validated per-agent `delivery.defaultTarget` / `delivery.defaultChannel` contract, and the message tool still only infers channel/target from explicit tool arguments or trusted run/session context. The related cron-default and announce-model reports are adjacent but narrower, so they do not supersede this agent-level cross-session delivery-default request.

## Best Possible Solution

Keep this issue open for a core product/design fix. Add a typed, documented per-agent delivery default contract, likely under `agents.list[].delivery`, wire it into trusted tool-context construction and message-tool channel/target inference for programmatic sessions, and cover `sessions_send`, webchat-routed sessions, cron-created sessions, explicit override precedence, multi-channel ambiguity, account/thread handling, and no-config backward compatibility.

## Risks / Open Questions

- A generic delivery default can silently route messages to the wrong chat if precedence, accountId/threadId handling, and multi-channel ambiguity are underspecified.
- A complete fix must preserve explicit tool arguments and per-run delivery overrides ahead of any agent default, and must respect session sendPolicy and cross-context policy.
