## Summary

Keep open. Current `main` still appears to drop Telegram native `/...` commands from `channel_post` updates, so this PR addresses a real gap; however, the review thread also surfaced follow-up auth/session edge cases that still need maintainer judgment before landing.

## Evidence

- **Native command handlers still require `ctx.message`:** Both native-command registration loops read `const msg = ctx.message; if (!msg) return;` before auth/execution. That means a command delivered as a Telegram channel post exits early on current `main` instead of entering the native command path.
  - file: [extensions/telegram/src/bot-native-commands.ts:782](https://github.com/openclaw/openclaw/blob/89755d1c79678610720cdb9ce504cfd62c261f64/extensions/telegram/src/bot-native-commands.ts#L782)
  - command: `nl -ba extensions/telegram/src/bot-native-commands.ts | sed -n '782,805p;1079,1115p'`
  - sha: [89755d1c7967](https://github.com/openclaw/openclaw/commit/89755d1c79678610720cdb9ce504cfd62c261f64)
- **Upstream grammY treats `bot.command` as channel-post aware:** The inspected `grammy@1.42.0` source documents that `bot.command` detects commands in channel posts by default and explicitly warns that `ctx.message` may be undefined, recommending `ctx.msg` instead. That matches the failure mode above.
  - file: [grammy@1.42.0 package/out/composer.js:255](https://github.com/openclaw/openclaw/blob/89755d1c79678610720cdb9ce504cfd62c261f64/grammy%401.42.0%20package/out/composer.js#L255)

- **Regular channel posts already go through a separate normalized path:** `bot.on("channel_post")` synthesizes `sender_chat` into `from` and routes the post through `handleInboundMessageLike`. That explains the report: regular channel text is handled on current `main`, but native slash commands use a different path that never normalizes the post.
  - file: [extensions/telegram/src/bot-handlers.runtime.ts:1979](https://github.com/openclaw/openclaw/blob/89755d1c79678610720cdb9ce504cfd62c261f64/extensions/telegram/src/bot-handlers.runtime.ts#L1979)

- **Current tests do not cover channel-post native command auth:** The Telegram native-command auth test file on current `main` covers group and allowlist behavior, but it has no `channelPost` cases. The regression described in this PR is therefore still unguarded in `main` test coverage.
