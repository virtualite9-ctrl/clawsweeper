## Summary

Keep open. Current `main` still lacks the PR's argv-plumbing fix, so the reported `openclaw cron <subcommand> --help` mismatch remains plausible and this PR still carries useful work.

## Evidence

- **`buildProgram()` still binds command registration to `process.argv`:** On current `main`, `buildProgram()` has no argv parameter and snapshots `process.argv` before calling `registerProgramCommands(...)`. That is the exact behavior this PR is trying to change.
  - file: [src/cli/program/build-program.ts:9](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/src/cli/program/build-program.ts#L9)
  - command: `nl -ba src/cli/program/build-program.ts | sed -n '1,80p'`
  - sha: [ec3dbd22a4de](https://github.com/openclaw/openclaw/commit/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46)
