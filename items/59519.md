## Summary

Current main already fixes the user-visible bug targeted by PR #59519: nested lazy-command help is handled by preregistering the real primary command from the normalized parse argv before Commander parses `--help`. The exact `buildProgram(argv)` refactor from the PR is not present, but the observable `openclaw cron add --help` problem from #59449 is solved and shipped.

## Best Possible Solution

Close PR #59519 as implemented by the shipped primary-command preregistration path. Keep the current `runCli` behavior and regression coverage; if maintainers still want explicit `buildProgram(argv)` injection as internal cleanup, track that separately from the already-fixed #59449 user-facing bug.

## Evidence

The inline comment states this is to keep help and command parsing correct with lazy command registration.
