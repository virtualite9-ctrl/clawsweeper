## Summary

Keep open. Current `main` still writes `record.requesterSessionKey` directly into `task_runs.requester_session_key`, so the proposed fix is not implemented on `main`. The linked issue contains a real production stack trace through `markTaskLostById -> upsertTaskWithDeliveryStateToSqlite`, but the exact `undefined` provenance is still not proven by current code because task creation and sqlite restore both normalize `requesterSessionKey` to a string. That mix of real crash evidence plus incomplete root-cause proof is not strong enough for a conservative close.
