## Summary

Keep #48894 open. Current main still only routes Discord message-create events into the inbound Discord handler; edited messages that arrive as MESSAGE_UPDATE are not registered or tested, even though the pinned Carbon dependency exposes a MessageUpdateListener surface.

## Best Possible Solution

Keep the issue open and implement this in the bundled Discord plugin: add a Discord MessageUpdate listener, normalize or fetch the edited message as needed, detect newly added bot mentions under requireMention and allowBots="mentions", pass eligible updates through the existing inbound/preflight path with replay/dedupe protection, and add focused tests for user-authored and bot-authored edited mentions plus non-mention cosmetic edits.
