## Summary

Current main already fixes #72230. The v2026.4.24 tag matches the report: the Bonjour plugin passed only `{ logger: api.logger }`, so its CIAO unhandled-rejection handler was not wired into the host handler set. Current main imports and passes `registerUnhandledRejectionHandler`, keeps CIAO process handlers registered for the advertiser lifetime across restart cycles, shares the host handler registry across staged plugin runtime copies, classifies CIAO ANNOUNCEMENT/PROBING cancellations as handled, and caps stuck Bonjour restarts so the gateway stays up.

## Best Possible Solution

Close #72230 as fixed on current main. Keep the current Bonjour implementation: plugin-scoped process handler wiring through the SDK runtime, shared global handler state, CIAO cancellation/assertion classification, handler lifetime across advertiser restarts, and bounded self-disable behavior for blocked or unstable mDNS environments. The docs should continue pointing affected deployment-scoped cases to `OPENCLAW_DISABLE_BONJOUR=1` or `openclaw plugins disable bonjour`.
