## Summary

Current `main` no longer has the reported Discord reconnect-stall watchdog path, and Discord channel failures are handled as channel-level exits with per-channel auto-restart rather than a full gateway restart.

## Close Comment

Closing from a Codex review because the specific bug reported here is already implemented away on current `main`. The reported reconnect-stall watchdog path is no longer present: current Discord lifecycle code only force-stops on a different `runtime-not-ready` case in `extensions/discord/src/monitor/provider.lifecycle.ts:198`, while ordinary websocket close / reconnect-scheduled events in `extensions/discord/src/monitor/provider.lifecycle.ts:223` only update channel status. I also confirmed the same relevant behavior is present in the latest release tag `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`). If anyone can still reproduce a full gateway restart from a Discord reconnect timeout on a current build, please open a fresh issue with new logs and the exact build SHA.
