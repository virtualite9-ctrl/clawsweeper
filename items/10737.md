## Summary

Keep #10737 open. Current main still does not implement a legacy `channels.imessage` auto-acknowledgment or typing-feedback path before agent dispatch. BlueBubbles has automatic typing support and is the recommended iMessage route, but it is a separate `channels.bluebubbles` implementation and does not close this legacy `imsg` channel request. The issue discussion also contains a same-day Codex review from steipete explicitly keeping this item open.

## Best Possible Solution

Keep #10737 open until maintainers either implement or explicitly decline legacy `channels.imessage` feedback. A concrete implementation should add typed config/schema/docs for `autoAck` or native-typing-with-text-fallback, send the feedback after access/mention gating but before `dispatchInboundMessage`, honor `dmOnly`, and add regression coverage proving the acknowledgment is not re-ingested as a user message. Continue recommending BlueBubbles for new iMessage deployments, but track legacy imsg behavior separately.
