## Summary

Keep #10737 open. Current main still lacks legacy `channels.imessage` channel-layer auto-acknowledgment or native typing wiring. BlueBubbles has typing support and is recommended for new iMessage setups, but it is a separate channel and does not satisfy the requested legacy iMessage `autoAck` behavior.

## Best Possible Solution

Keep the issue open until maintainers either implement or explicitly decline legacy iMessage feedback. A concrete implementation would likely live in `extensions/imessage`: add typed config/schema/docs for `channels.imessage.autoAck` or a native-typing/text-fallback option, send feedback before `dispatchInboundMessage`, preserve DM-only behavior, and add regression tests for no duplicate final replies. BlueBubbles should remain the recommended iMessage path, but it should not close this legacy `channels.imessage` request.
