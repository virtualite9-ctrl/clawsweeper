## Summary

Keep open. Current `main` still sends `store: false` on Google's OpenAI-compatible completions route by default, and current Gemini thought-signature replay only works when the prior assistant turn already matches the exact same API/provider/model. That does not cover the reported fallback/replay path.

## Evidence

- **Google OpenAI-compat endpoint is still treated as store-capable:** `generativelanguage.googleapis.com` is classified as `google-generative-ai`, and the OpenAI completions compat defaults still set `supportsStore` for non-`isNonStandard` endpoints. There is no Google OpenAI-compat carveout here.
