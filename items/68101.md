## Summary

Keep this issue open. Current main still catches workflow-file resolution failures and falls back to inline pipeline execution, which matches the reported misleading `Unknown command` behavior. There is also an open, unmerged PR with closing syntax for this issue, so cleanup should not close the issue before that PR or an equivalent fix lands.

## Best Possible Solution

Review and land #68106, or an equivalent focused Lobster plugin change in `extensions/lobster/src/lobster-runner.ts`, that rethrows workflow resolution errors for workflow-file-like inputs and adds regression coverage for missing relative workflow files while preserving normal inline pipelines.
