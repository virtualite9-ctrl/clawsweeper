## Summary

Keep open. Current main still swallows Lobster workflow-file resolution failures and then sends the original string to inline pipeline execution, which matches the reported misleading `Unknown command` path. There is also an open, unmerged implementation candidate in #68106, so this issue should remain open until that PR or an equivalent fix lands.

## Best Possible Solution

Review and land #68106 after addressing the inline-pipeline misclassification risk, or make an equivalent focused change in `extensions/lobster/src/lobster-runner.ts` that rethrows resolution errors for true workflow-file-like inputs and adds paired regression tests for missing relative workflow files plus valid inline pipelines with file-like arguments.
