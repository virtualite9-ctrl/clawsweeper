## Summary

Keep PR #72128 open. Current main does not implement configurable `labelTemplate` behavior for `sessions_spawn`, and the linked feature request #72074 remains unsolved. The PR is a plausible, focused implementation, but it should not be closed or merged as-is: the current PR context shows unresolved review concerns around resolving `{agentId}` before native/ACP target fallback and looking up per-agent `subagents` config on the target rather than the requester. It also lacks the tests/docs/generated schema follow-through expected for a new config field.

## Best Possible Solution

Keep the PR and linked issue #72074 open. If maintainers want this feature, move label-template resolution to a place that has the resolved requester and target agent ids for both native and ACP spawns, use requester-scoped per-agent `subagents` config for overrides, preserve explicit label precedence, and add focused tests for defaults, requester override precedence, omitted-agent fallback, ACP default-agent fallback, duplicate tokens, and task-preview sanitization. The final patch should also update generated config schema and user-facing docs for the new config key.
