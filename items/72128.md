## Summary

Keep PR #72128 open. Current main still lacks configurable `labelTemplate` support for `sessions_spawn`, and the linked issue #72074 was closed as superseded by this PR, making this PR the canonical remaining tracker. The PR is useful but should not be closed as implemented or duplicate; it needs revision because current spawn code resolves requester and effective target agent IDs after the PR's proposed template substitution point.

## Best Possible Solution

Keep #72128 open as the canonical implementation PR. Revise it so fallback label generation happens after requester identity and effective target agent are known, with explicit `label` taking precedence and requester-scoped `agents.list[].subagents.labelTemplate` overriding `agents.defaults.subagents.labelTemplate`. Add focused tests for explicit label precedence, default template, requester override, omitted `agentId` native spawns, ACP default-agent spawns, repeated tokens, newline stripping, and truncation; update user-facing subagents/config docs and generated schema artifacts if maintainers accept the new config key.
