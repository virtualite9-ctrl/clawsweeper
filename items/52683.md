## Summary

Current `main` has partial building blocks for multi-agent chats, but not the requested first-class shared-context project-room mode with selectable visible/backstage collaboration, role-attached room participants, and persistent room memory. The strongest repo evidence points the other way: multi-agent routing and broadcast flows are still intentionally isolated per agent/session, with only limited shared channel/session primitives.

## Evidence

- **Multi-agent routing is explicitly isolated:** Docs describe multiple agents as separate workspaces, auth, and session stores rather than a shared room model.
  - file: [docs/concepts/multi-agent.md:8](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/docs/concepts/multi-agent.md#L8)
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **Shared context exists only as a single-session channel scope primitive:** `session.scope: "global"` makes one channel context share one session, but this is a grouping strategy for one channel context, not a first-class multi-agent room abstraction.
  - file: [docs/gateway/config-agents.md:1148](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/docs/gateway/config-agents.md#L1148)
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **Broadcast groups are limited and still isolate each agent:** Broadcast groups are documented as WhatsApp-only, experimental, and each listed agent keeps its own session key, conversation history, workspace, and memory/context.
  - file: [docs/channels/broadcast-groups.md:15](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/docs/channels/broadcast-groups.md#L15)
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **Sub-agents are separate sessions that announce back:** Sub-agents run in their own session and announce a completion summary back to the requester chat; default context is `isolated`, with `fork` only copying the requester transcript for that child.
  - file: [docs/tools/subagents.md:10](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/docs/tools/subagents.md#L10)
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **No first-class project-room terminology or feature surface found:** Repository search for `project room|backstage mode|visible mode|shared room memory|role-based participation` returned no matches in `docs`, `src`, `extensions`, or `ui`.
  - command: `rg -n "project room|backstage mode|visible mode|shared room memory|role-based participation" docs src extensions ui`
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
