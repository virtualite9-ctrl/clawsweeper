## Summary

Current main still only partially addresses the Google Chat Workspace Add-ons incompatibility. The app-url/add-on auth path exists and is tested, but the native wrapper parser only unwraps `chat.messagePayload`, while the issue and follow-up workaround cover additional Add-ons payload variants. The public Google Chat setup docs and setup wizard still do not tell users to configure the required numeric `appPrincipal`, so the reported onboarding failure remains actionable in this repo.

## Best Possible Solution

Keep this open and finish the Google Chat plugin compatibility work: document `audienceType: "app-url"` plus numeric `appPrincipal` discovery on the public Google Chat docs page, add setup/status guidance or a wizard prompt for `appPrincipal`, and broaden Add-ons wrapper normalization to the relevant Chat payload variants or intentionally 200-ignore unsupported non-message events with tests. Preserve the strict principal binding unless a maintainer explicitly accepts a different trust model.
