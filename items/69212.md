## Summary

Keep #69212 open. Current main does not implement the PR's two central behaviors: HTTP Responses/Codex Responses tool-only continuations with `previous_response_id`, or OpenAI-compatible completions replay of Volcengine/Doubao `encrypted_content` reasoning metadata. The PR is still potentially useful, but review comments identify correctness gaps that should be addressed before landing.

## Best Possible Solution

Keep the PR open or supersede it with a narrower maintainer-owned core transport change. The best path is to add provider/capability-gated HTTP `previous_response_id` continuation planning, encode and replay OpenAI-compatible completions reasoning metadata including `encrypted_content`, and land focused payload tests for encrypted-only chunks, multi-chunk accumulation, post-tool coalescing, buffer limits, and proxy/custom Responses routes.
