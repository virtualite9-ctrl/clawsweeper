## Summary

Keep open. Current main does not implement this PR's central behavior: OpenAI-compatible completions still do not capture/replay `encrypted_content` in thinking signatures, and the HTTP Responses transport still has no `previous_response_id` tool-only continuation path. The PR is focused on active bundled-provider behavior for Volcengine/Doubao and includes follow-up tests for review feedback, though the remaining route-capability concern should be addressed before merge.

## Best Possible Solution

Keep #69212 open. The best path is to port/rebase the encrypted `openai-completions` reasoning replay and HTTP Responses continuation work onto current main, keep the payload-shape regression tests, and address the remaining route-capability review before merge.
