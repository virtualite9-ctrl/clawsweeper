## Summary

Keep open. Current main still lacks the PR's two central HTTP transport behaviors: OpenAI-compatible completions do not capture/replay `encrypted_content`, and the HTTP Responses transport does not emit `previous_response_id` continuations. The related review discussion points to changes that may need cleanup, but the work is not obsolete or implemented on main.

## Best Possible Solution

Keep the PR open and ask for a current-main rebase that ports the narrow behavior with tests: capture and replay `encrypted_content` for Volcengine/Doubao `openai-completions` continuations, add HTTP `previous_response_id` only behind a native OpenAI/Codex Responses route capability gate, and include payload-shape regression tests covering encrypted-only/multi-chunk metadata and tool-only continuations.
