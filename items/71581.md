## Summary

Keep this issue open. Current main still routes exec-event and cron-event heartbeat wakes through the explicit same-agent session key, while the single-owner `session.dmScope="main"` logic remains limited to inbound DM routing paths. The closed PR #72160 is unrelated and did not merge a fix for this routing gap.

## Best Possible Solution

Keep this issue open and implement a narrow core routing helper for internal system-event wakes. When `session.dmScope="main"` and the same single-owner allowlist semantics used by inbound DM routing apply, map same-agent direct-message session keys to `resolveAgentMainSessionKey` before both system-event enqueue/drain and heartbeat forced-session resolution. Add regression coverage for exec notify-on-exit, explicit exec system events, and cron next-heartbeat system events, plus negative cases for multi-owner or wildcard allowlists, non-main scopes, subagent keys, isolated heartbeats, and non-DM explicit sessions.
