## Summary

Keep open. Current main does not implement `tools.fs.allowedRoots` or an equivalent multi-root filesystem policy, and this PR is the active implementation candidate for the closed feature request #33635 after earlier PR #33708 was closed. Functional review says the requested capability is still missing on main; security review says the PR touches sensitive filesystem boundary code and needs maintainer review/CI rather than cleanup closure.

## Best Possible Solution

Keep this PR open as the canonical implementation candidate for `tools.fs.allowedRoots`. Maintainers should review the final rebased diff, verify the filesystem sandbox and apply_patch security properties, run the relevant changed lanes, and then either land a cleaned-up version or replace it with an equivalent core implementation. Closing now would lose useful unmerged work because current main still lacks the feature.
