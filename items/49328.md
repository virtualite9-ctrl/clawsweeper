## Summary

Current `main` already reconciles terminal chat events against canonical backend history, which addresses the reported frontend-only duplicate assistant message symptom well enough to close this issue.

## Close Comment

Closing from a Codex review because current OpenClaw already reconciles this case in the Control UI. On `main`, `ui/src/ui/app-gateway.ts:472` reloads canonical chat history after `final` chat events, and `ui/src/ui/controllers/chat.ts:179` replaces `chatMessages` from backend history instead of appending to the existing list. That means a frontend-only duplicate caused by a retransmitted `final` event is corrected back to the single persisted message, which matches the reporter’s note that backend storage was already correct. This reconciliation path is also present in the released `v2026.4.22` tree. I did not find the exact `processedChatRunIds` approach from the issue comments on `main`, but the current implementation addresses the reported symptom well enough to treat it as implemented.
