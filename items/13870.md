## Summary

Keep #13870 open. Current main shows client-supplied paired-device `displayName` metadata in the CLI and Control UI, but it still does not implement the requested operator-owned rename/label workflow for paired devices.

## Best Possible Solution

Keep #13870 open until paired-device label management is implemented end to end: add an additive gateway method/schema/handler for paired-device rename, persist an operator-owned display-name override with documented precedence over reconnect metadata, expose `openclaw devices rename --device <id> --name <displayName>`, add a Control UI rename action, cover storage/protocol/CLI/UI behavior with focused tests, and update the public devices/protocol docs. Keep #23916 focused on broader metadata display and keep `node.rename` separate.
