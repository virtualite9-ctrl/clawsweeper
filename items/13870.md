## Summary

Keep #13870 open. Current main displays an existing paired-device displayName when one is present, and reconnect metadata can refresh that field, but there is still no user-facing paired-device rename workflow in the CLI, gateway protocol, server handlers, Control UI, or devices docs. The existing node.rename path is a separate paired-node surface and does not rename paired device records.

## Best Possible Solution

Keep #13870 open until OpenClaw adds paired-device label management end to end: an additive gateway device.rename method/schema, an authorized storage update path for paired device records, openclaw devices rename --device <id> --name <displayName>, a Control UI rename action, focused protocol/CLI/UI tests, and docs on https://docs.openclaw.ai/cli/devices. Keep node.rename as a separate paired-node label surface.
