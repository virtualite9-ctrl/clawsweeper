## Summary

Keep PR #70884 open. Current main still has the trusted-proxy WebSocket/docs gap this PR targets: the trusted-proxy docs discuss `x-openclaw-scopes` without an HTTP-only boundary, and the protocol docs list device-less WebSocket exceptions without explaining the scope-clearing consequence. The runtime and tests support the underlying behavior, and the provided PR context shows review feedback was addressed rather than superseded.

## Best Possible Solution

Keep #70884 open. The best path is to review and refine the docs-only PR, then update `docs/gateway/trusted-proxy-auth.md` and `docs/gateway/protocol.md` with the HTTP-only `x-openclaw-scopes` boundary, device-less WebSocket scope-clearing behavior, the local `allowInsecureAuth` exception, and the break-glass warning for `dangerouslyDisableDeviceAuth`.
