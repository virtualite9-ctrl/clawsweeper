## Summary

Current main already handles this startup race class: CLI backend aliases are no longer treated as embedded agent harness ids, configured CLI backend owners are pulled into gateway startup planning, and regression coverage includes both `claude-cli`/Anthropic and `google-gemini-cli`/Google.

## Best Possible Solution

Close this as implemented on current main and ship the existing startup/CLI-backend selection behavior in the next release. If a new live WSL2/systemd failure remains after that release, it should be filed as a fresh issue with updated logs against the new version.
