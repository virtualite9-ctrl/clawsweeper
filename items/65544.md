## Summary

Keep this issue open. Current main still has no timeout around `agent_end` typed hook execution: the hook runner awaits every handler with `Promise.all`, and both agent-end call sites only log rejected promises. A never-settling embedding call can still hang silently. The separate memory embedding timeout work does not cover the typed hook path.

## Best Possible Solution

Keep this issue open. Add a generic typed-hook timeout policy in `src/plugins/hooks.ts` or a narrow `agent_end` timeout wrapper, document the default and any per-hook override on the plugin API, decide whether hook contexts receive an `AbortSignal`, and add regression coverage proving a never-settling `agent_end` handler produces a warning without requiring every plugin author to implement their own timeout.

A never-settling handler means the returned promise never settles.
