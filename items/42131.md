## Summary

Keep this PR open. Current main still compares trusted safe-bin directories with exact-case Set membership while doctor lowercases configured safe-bin entries, so the reported false warning remains plausible. The PR is a focused fix candidate, but its latest tmpdir-based filesystem probe still has a security-sensitive mixed-volume case-sensitivity concern that should be reviewed or revised before merge.

## Best Possible Solution

Keep this PR open for maintainer review or a small revision. The best fix is a shared, testable safe-bin path comparison helper that compares trusted dirs and resolved executable dirs consistently on genuinely case-insensitive paths without collapsing distinct paths on case-sensitive or mixed-sensitivity volumes. Add deterministic regression coverage for doctor/runtime mixed-case matching and a trust-boundary test for case-sensitive sibling directories.

## Risks / Open Questions

- The PR's global `os.tmpdir()` probe can be wrong for configured trusted dirs on a different volume with different case-sensitivity semantics; lowercasing in that situation could collapse distinct trusted and untrusted sibling paths.
- The added tests described in the provided PR context mirror the host/runtime probe and do not deterministically prove that a case-sensitive sibling path remains untrusted on mixed-sensitivity volumes.
