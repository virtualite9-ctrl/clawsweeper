## Summary

Keep this PR open. Current main still stores trusted safe-bin directories with exact original casing while doctor lowercases configured safeBins before resolving and checking them, so the reported false warning remains plausible on case-insensitive filesystems. The PR is a focused fix candidate, but its latest implementation needs maintainer review because it applies one os.tmpdir() case-sensitivity probe globally to all trusted directories.

## Best Possible Solution

Keep this PR open for maintainer review or a small revision. The best fix is a shared, testable safe-bin path comparison helper that compares trusted dirs and resolved executable dirs consistently on genuinely case-insensitive paths without collapsing distinct paths on case-sensitive or mixed-sensitivity volumes, plus deterministic regression coverage for doctor/runtime mixed-case matching and a trust-boundary test for case-sensitive siblings.

## Risks / Open Questions

- The PR's global os.tmpdir() filesystem probe can be wrong for safeBinTrustedDirs on another volume with different case-sensitivity semantics; lowercasing there could collapse distinct trusted and untrusted sibling paths.
- The added tests mirror the runtime probe, so they do not deterministically prove that case-sensitive sibling directories remain untrusted on mixed-sensitivity volumes.
