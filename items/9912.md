## Summary

Keep #9912 open. Current main still lacks the requested global/per-agent hard iteration budget (`maxTurns`, `maxToolCalls`, or timeout-after-first-tool-call). Existing tool-loop detection is related but optional/pattern-based, OpenResponses only accepts `max_tool_calls` as an ignored field, and the referenced #13271 remains a separate per-channel budget request.

## Best Possible Solution

Keep #9912 open as the canonical global/per-agent hard iteration budget request. A complete fix should add typed config and strict schema/help/docs for `agents.defaults` and per-agent scope, enforce max model-tool turns and/or total tool invocations in the embedded/tool-call runtime, define timeout-after-first-tool-call behavior, add focused tests for exhaustion/final-response behavior, and keep #13271 separate for per-channel budgets and auto-delegation.
