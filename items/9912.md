## Summary

Keep #9912 open. Current main still lacks the requested global/per-agent hard budget for model-tool turns, total tool calls, or timeout-after-first-tool-call behavior. Existing timeout and loop-detection controls are related mitigations, but they are not a typed `maxTurns`, `maxToolCalls`, or `toolCallTimeout` config that forces a final response using the last useful tool result.

## Best Possible Solution

Keep #9912 open as the canonical global/per-agent hard agent iteration budget request. A complete fix should add typed config, schema metadata/help/docs, and runtime enforcement for max model-tool turns and/or total tool invocations, define timeout-after-first-tool-call semantics, and specify whether the final response uses the last tool result, an explicit limit notice, or both. Keep #13271 separate for per-channel budget and auto-delegation behavior.
