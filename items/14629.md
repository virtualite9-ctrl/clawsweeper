## Summary

Current main still appears to have the sanitizer gap from #14629. The sanitizer strips `<final>` tags textually, then only collapses exact adjacent duplicate blocks separated by blank lines. Same-line repeats, single-newline repeats, partial-prefix duplicates, and the reported `</final>` boundary duplicate shape are not covered. A maintainer/Codex comment on April 26, 2026 already reached the same keep-open conclusion, and the related duplicate-output reports are for different surfaces rather than a canonical fix for this sanitizer request.

## Best Possible Solution

Keep #14629 open. The best path is a narrow sanitizer change in `src/agents/pi-embedded-helpers/sanitize-user-facing-text.ts` that conservatively handles same-line/single-newline repeats, the reported `</final>` boundary duplicate shape, and safe partial-prefix duplicates, with regression tests in `src/agents/pi-embedded-helpers.sanitizeuserfacingtext.test.ts` that also preserve legitimate repeated text and ordinary single-newline formatting.
