## Summary

Current main still has the reported sanitizer gap. `sanitizeUserFacingText()` strips `<final>` tags textually and only collapses exact adjacent duplicate blocks separated by blank lines, so same-line duplicates, single-newline duplicates, partial-prefix duplicates, and `</final>` boundary cleanup remain unimplemented.

## Best Possible Solution

Keep #14629 open. Implement a narrow change in `src/agents/pi-embedded-helpers/sanitize-user-facing-text.ts` for the reported same-line/single-newline repeat and `</final>` boundary duplicate shape, with focused tests in `src/agents/pi-embedded-helpers.sanitizeuserfacingtext.test.ts` that prove the cleanup while preserving normal repeated wording and single-newline formatting.
