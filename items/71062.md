## Summary

Keep open. The PR targets a focused gateway/agent bug that is still present on current main: client-tool handoff still returns a normal pending stub without Pi termination, /v1/responses streaming still omits dedicated function_call_arguments events, and the Responses stream parser still lacks a .done-only argument handler.

## Best Possible Solution

Keep #71062 open. The best path is to land this PR or an equivalent maintainer-reviewed fix that adds terminate: true to the Pi client-tool handoff, emits response.function_call_arguments.delta and .done from /v1/responses streaming while preserving populated final response snapshots, teaches the upstream Responses parser to consume .done-only args, updates the OpenResponses docs/schema, and keeps the focused regression tests.
