## Summary

Keep open. Current `main` still does not copy ACP metadata from a base ACP session into thread/topic child session keys during inbound session recording, so this PR is not already implemented. Main does have separate bound-conversation routing that can steer inbound traffic to the ACP session directly, but the PR discussion correctly frames this as complementary rather than equivalent. The proposed change still looks useful, though review comments identified at least one remaining correctness gap around canonical parent-session lookup.

## Evidence
