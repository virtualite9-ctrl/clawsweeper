## Summary

#69187 is a valid memory-core dreaming cleanup bug, but the remaining work is already tracked by open PR #70464. Current main still has the unbounded timestamped session-key shape and the fallback delete path still lacks admin scope, so this should close as superseded, not implemented.

## Best Possible Solution

Close #69187 as superseded and keep maintainer review focused on #70464. The right product/code path is the narrow PR direction: stable per-workspace/phase dreaming session keys, unique idempotency keys per run, and a tightly scoped memory-core-only cleanup allowance rather than a broad admin grant or unrelated session-store migration.
