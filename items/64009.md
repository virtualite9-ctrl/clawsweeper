## Summary

Current main already implements the OpenClaw-side fix for this class of Slack false stale-socket restart: stale-socket recovery is now keyed off provider-proven transport activity, and Slack socket-mode no longer seeds app-event freshness as transport liveness. The changelog places that fix in the 2026.4.20 release notes.
