## Summary

Current main still does not clearly satisfy the reported Feishu config shape. A Feishu group entry with `groups.<chat_id>.requireMention: false` can preserve the mention-gating override, but current inbound handling admits groups through `groupAllowFrom` under `groupPolicy: "allowlist"` before it evaluates `requireMention`, so the reporter's group can be dropped before a session is created.

## Best Possible Solution

Keep this issue open and fix or clarify Feishu group admission. The conservative implementation path is to either treat explicit `channels.feishu.groups.<chat_id>` entries as admitted groups under `groupPolicy: "allowlist"`, or add a targeted doctor/config warning that `groups.<chat_id>.requireMention=false` does not admit the group without `groupAllowFrom` or `groupPolicy: "open"`. Add a regression test that runs the reporter's config shape through plugin-aware config validation and verifies a Feishu group text event reaches session/route dispatch; update Feishu docs if the intended contract remains `groupAllowFrom`-only.
