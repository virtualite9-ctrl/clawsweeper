## Summary

Keep open. Current main still does not clearly satisfy the reported Feishu configuration shape: `channels.feishu.groups.<chat_id>.requireMention=false` can be preserved as a mention-gating override, but validated Feishu config defaults `groupPolicy` to `allowlist`, and inbound handling checks `groupAllowFrom` before it evaluates the per-group `requireMention` override. The already-posted Codex review on the issue matches current main behavior at 7d9dc8cf24fc143bec95318491e0247c37863e63.

## Best Possible Solution

Keep this issue open and fix or clarify Feishu group admission. The conservative product path is to either treat explicit `channels.feishu.groups.<chat_id>` entries as admitted groups under `groupPolicy: "allowlist"`, or add a targeted doctor/config warning and clearer Feishu docs that `groups.<chat_id>.requireMention=false` only changes mention gating and still requires `groupAllowFrom` or `groupPolicy: "open"`. Add a regression test that runs the reporter's validated config shape through Feishu inbound handling and verifies a group text event reaches session/route dispatch.
