## Summary

Issue discussion included a second independent report with the same stack trace, and current `main` plus release `v2026.4.23` already contain a targeted Discord gateway fix and regression tests for the stale-heartbeat reconnect race behind the zombie reconnect crash.

## Evidence

- **Discord gateway reconnect guard is present on main:** `SafeGatewayPlugin.connect()` now clears stale `heartbeatInterval` and `firstHeartbeatTimeout` before delegating to the parent reconnect path, and the inline comment explicitly references `#64011` as part of the `@buape/carbon` heartbeat race.
  - file: [extensions/discord/src/monitor/gateway-plugin.ts:257](https://github.com/openclaw/openclaw/blob/5b8bd6371c666a5de43c82305e0315771e8e49d5/extensions/discord/src/monitor/gateway-plugin.ts#L257)
  - command: `nl -ba extensions/discord/src/monitor/gateway-plugin.ts | sed -n '257,271p'`
  - sha: [5b8bd6371c66](https://github.com/openclaw/openclaw/commit/5b8bd6371c666a5de43c82305e0315771e8e49d5)
- **Regression tests cover the stale-timer reconnect path:** The Discord gateway tests assert that both stale `heartbeatInterval` and stale `firstHeartbeatTimeout` are cleared when `isConnecting=true`, matching the reported crash condition before reconnect delegation.
  - file: [extensions/discord/src/monitor/gateway-plugin.test.ts:78](https://github.com/openclaw/openclaw/blob/5b8bd6371c666a5de43c82305e0315771e8e49d5/extensions/discord/src/monitor/gateway-plugin.test.ts#L78)
  - command: `nl -ba extensions/discord/src/monitor/gateway-plugin.test.ts | sed -n '78,114p'`
  - sha: [5b8bd6371c66](https://github.com/openclaw/openclaw/commit/5b8bd6371c666a5de43c82305e0315771e8e49d5)
- **Fix is documented in the shipped v2026.4.23 release:** The `v2026.4.23` changelog explicitly says Discord gateway now clears stale heartbeat timers before reconnecting so zombie gateway callbacks cannot crash the process, which matches this report.
  - file: [CHANGELOG.md:778](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/CHANGELOG.md#L778)
