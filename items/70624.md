## Summary

Keep this PR open. Current main already has some QQBot groundwork, but the central work proposed here is not fully implemented on main: non-@ group message ingestion/history/config, C2C official streaming delivery, chunked upload implementation, and the larger adapter/pipeline refactor are still absent. This is bundled QQBot plugin work, not an obvious ClawHub/core-scope close, but it needs maintainer review and likely splitting or tightening before merge because the PR is very large and has substantive review findings.

## Best Possible Solution

Keep this PR open for maintainer review, but ask the contributor to rebase on current main, address the P1/P2 review findings, and preferably split the work into reviewable pieces: chunked media upload, C2C official streaming, group-chat/history behavior, and mechanical architecture refactor. If maintainers do not want the broad refactor, preserve the still-missing behavior in smaller QQBot plugin PRs.
