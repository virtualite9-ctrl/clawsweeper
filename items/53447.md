## Summary

Keep open. Current main has a generic parent-session fallback for channel model overrides, and related #57960 correctly covers parent-channel agent binding inheritance, but Discord still only supplies the native thread parent as `ParentSessionKey` when `channels.discord.thread.inheritParent` is enabled. With the default `inheritParent: false`, a Discord thread without its own `modelByChannel` entry still lacks the parent channel id needed for the requested model override fallback.

## Best Possible Solution

Keep this issue open and implement a narrow Discord model-routing fix: pass the resolved native thread parent channel id into `resolveChannelModelOverride` independently of `channels.discord.thread.inheritParent`, preserving explicit thread-id `modelByChannel` precedence and leaving transcript inheritance behavior unchanged. Add regression coverage for a Discord thread whose own id has no override but whose parent channel id does, plus a case where a thread-specific override wins.
