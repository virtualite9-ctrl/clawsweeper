## Summary

Keep this issue open. Current main still does not feed Discord's native thread parent channel id into ordinary reply `modelByChannel` resolution when `channels.discord.thread.inheritParent` is false. The shipped parent-channel work covers route/config binding inheritance and auto-thread title model selection, but not the requested default reply model fallback for existing Discord thread sessions.

## Best Possible Solution

Implement a narrow Discord reply model-routing fix: preserve thread-specific `modelByChannel` precedence, then pass/use the resolved Discord native parent channel id as a fallback candidate independently of `channels.discord.thread.inheritParent`. Add regression coverage for a Discord thread with no thread override but a parent channel override, plus a thread-specific override-wins case. Update docs only after the ordinary reply path matches the documented inheritance behavior.
