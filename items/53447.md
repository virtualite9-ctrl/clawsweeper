## Summary

Keep this issue open. Current main still resolves the native Discord thread parent during preflight, but ordinary thread reply model selection does not receive that parent id when `channels.discord.thread.inheritParent` is left at its default `false`. The shipped parent-session fallback paths only work when a parent session key exists or a plugin supplies parent candidates; they do not satisfy the requested `modelByChannel` fallback from a Discord thread id to its parent channel id.

## Best Possible Solution

Implement a narrow Discord model-routing fix: carry the resolved native Discord thread parent channel id into channel model override candidate resolution independently of `channels.discord.thread.inheritParent`, while preserving explicit thread-id `modelByChannel` precedence. Add regression coverage for a Discord thread with no thread override but a parent channel override, plus a thread-specific override-wins case.
