## Summary

Keep open. Current `main` still drops `allowUnsafeExternalContent` in the `/hooks/agent` normalization path, so the reporter's narrowed root cause matches the checked-in code and the flag is not implemented end-to-end for that route.

## Evidence

- **`HookAgentPayload` omits the flag:** `HookAgentPayload` ends at `timeoutSeconds`; `allowUnsafeExternalContent` exists only on `HookAgentDispatchPayload`. That means the normalized `/hooks/agent` request shape does not include the flag at all.
  - file: [src/gateway/hooks.ts:213](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/src/gateway/hooks.ts#L213)
  - command: `nl -ba src/gateway/hooks.ts | sed -n '213,232p'`
  - sha: [926068b14fc6](https://github.com/openclaw/openclaw/commit/926068b14fc661e915a0247e9ecb5949fb95b4f7)
- **`normalizeAgentPayload()` never reads `payload.allowUnsafeExternalContent`:** The request normalizer parses `message`, `name`, `agentId`, `sessionKey`, `deliver`, `channel`, `to`, `model`, `thinking`, and `timeoutSeconds`, then returns `value` without `allowUnsafeExternalContent`. This matches the reporter's follow-up comment exactly.
  - file: [src/gateway/hooks.ts:399](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/src/gateway/hooks.ts#L399)
  - command: `nl -ba src/gateway/hooks.ts | sed -n '399,452p'`
  - sha: [926068b14fc6](https://github.com/openclaw/openclaw/commit/926068b14fc661e915a0247e9ecb5949fb95b4f7)
- **`/hooks/agent` dispatch uses the normalized value, so the dropped flag never reaches the job:** In the `/hooks/agent` handler, `dispatchAgentHook()` receives `...normalized.value` plus `externalContentSource: "webhook"`; there is no separate recovery of `allowUnsafeExternalContent`. By contrast, the mapped-hooks path explicitly forwards `mapped.action.allowUnsafeExternalContent`, which shows the plain `/hooks/agent` route is the broken branch.
  - file: [src/gateway/server-http.ts:663](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/src/gateway/server-http.ts#L663)
  - command: `nl -ba src/gateway/server-http.ts | sed -n '663,727p' && nl -ba src/gateway/server-http.ts | sed -n '813,827p'`
  - sha: [926068b14fc6](https://github.com/openclaw/openclaw/commit/926068b14fc661e915a0247e9ecb5949fb95b4f7)
- **Downstream cron execution still keys wrapping off the payload flag:** `dispatchAgentHook()` writes `value.allowUnsafeExternalContent` into `job.payload.allowUnsafeExternalContent`, and the isolated-agent runner suppresses wrapping only when `agentPayload?.allowUnsafeExternalContent === true`. So the problem is upstream normalization, not the downstream wrapping logic.
  - file: [src/gateway/server/hooks.ts:66](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/src/gateway/server/hooks.ts#L66)
  - command: `nl -ba src/gateway/server/hooks.ts | sed -n '56,75p' && nl -ba src/cron/isolated-agent/run.ts | sed -n '572,587p'`
  - sha: [926068b14fc6](https://github.com/openclaw/openclaw/commit/926068b14fc661e915a0247e9ecb5949fb95b4f7)

## Risks / Open Questions

- I did not run a live HTTP reproduction because this review was constrained to read-only inspection and the environment does not have `pnpm` available; the conclusion is from current-main code inspection.
- There is no visible `/hooks/agent` regression test asserting that request-level `allowUnsafeExternalContent: true` survives normalization, so this path remains easy to miss until a dedicated test is added.
