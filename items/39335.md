## Summary

Current `main` already suppresses streamed `NO_REPLY` lead fragments in gateway chat events and filters silent assistant rows from `chat.history`, with matching regression tests and changelog entries showing the fix shipped in `v2026.3.2`.

## Close Comment

Closing as implemented on current `main`. Codex review found that the gateway already suppresses `NO_REPLY` lead-fragment deltas before chat text is broadcast (`src/gateway/server-chat.ts:704`), and there is a regression test that explicitly covers streamed `NO`, `NO_`, `NO_RE`, `NO_REPLY` without leaking `NO` into the final chat message (`src/gateway/server-chat.agent-events.test.ts:255`). The Control UI also filters assistant `NO_REPLY` rows from `chat.history` and ignores silent delta/final payloads client-side (`ui/src/ui/controllers/chat.ts:43`, `ui/src/ui/controllers/chat.ts:451`, `ui/src/ui/controllers/chat.test.ts:525`). The changelog records these webchat `NO_REPLY` fixes in `v2026.3.2` (`CHANGELOG.md:3347`, `CHANGELOG.md:3369`). If this can still be reproduced on a current build, please open a new issue with the exact version and a minimal timing repro.
