## Summary

Keep open. Current main accepts and propagates `memorySearch.outputDimensionality` into memory provider creation, but the bundled OpenAI embedding provider still sends OpenAI inline and batch embedding requests without `dimensions`. The provided GitHub context also shows #55507 is an open, unmerged PR using closing syntax for this issue, so the issue should stay open until that PR is reviewed and landed or closed.

## Best Possible Solution

Keep this issue open and review #55507 as the implementation candidate. The fix should land in the OpenAI memory embedding provider path by carrying `outputDimensionality` into an OpenAI-specific `dimensions` request field for inline and remote batch embeddings, include it in OpenAI runtime cache key data so reindex decisions change when dimensions change, and add regression tests that assert the OpenAI request bodies include `dimensions` for supported `text-embedding-3-*` models.
