## Summary

Keep open. Current main still does not implement the active reply-run guard described in this issue, and provided GitHub context shows an open, unmerged PR (#64963) with closing syntax for this issue. The maintainer path is to review/rebase/land that PR or apply the equivalent fix, not close this issue first.

## Best Possible Solution

Keep this issue open while maintainers review #64963 or apply the same focused change: after heartbeat preflight resolves the target session key, have src/infra/heartbeat-runner.ts skip with requests-in-flight when resolveActiveReplyRunSessionId(sessionKey) is truthy, then add the regression test for an active reply run plus queued system event on the same session. Once that lands, GitHub can close this issue through the PR closing reference.
