The maintainer path is to review/rebase/land that PR or apply the equivalent focused fix, not close this issue first.

## Best Possible Solution

Keep this issue open while maintainers review #64963 or apply the same focused change directly: after heartbeat preflight resolves the target sessionKey, have src/infra/heartbeat-runner.ts return skipped/requests-in-flight when resolveActiveReplyRunSessionId(sessionKey) is truthy, then add the regression test for an active reply operation with otherwise idle lanes.
