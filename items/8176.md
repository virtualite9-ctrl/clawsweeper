## Summary

Current main still does not expose or apply a hook-owned user prompt replacement. The requested `promptOverride` field is absent from the public plugin hook result types, prompt mutation allowlist, hook merger, embedded runner, CLI runner, agent harness helper, docs, and nearby regression tests. The direct implementation PR #8178 was closed unmerged, and the related systemPrompt work does not satisfy user-prompt replacement.

## Best Possible Solution

Keep #8176 open until OpenClaw either implements and documents an explicit user prompt replacement field for `before_prompt_build` and legacy `before_agent_start`, or a maintainer explicitly rejects that API. The implementation should update the public hook result types, prompt mutation allowlist, hook merge precedence, `resolvePromptBuildHookResult`, embedded runner, CLI runner, shared agent harness path, `allowPromptInjection` filtering, public docs, and regression tests that prove submitted prompt text and override/prepend order.

## Risks / Open Questions

- This is a public plugin API/product decision: maintainers still need to define precedence when multiple hooks request replacement, whether later hooks see the original or effective prompt, and how `allowPromptInjection=false` constrains the new field.
