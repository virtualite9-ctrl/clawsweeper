## Summary

Current main still does not implement hook-owned user prompt replacement for #8176. The public hook result contract, mutation allowlist, hook merger, embedded runner, CLI runner, agent harness/Codex harness path, docs, and tests all support system/context mutation only. The direct implementation PR #8178 was closed unmerged, and the systemPrompt-related follow-ups/discussion do not satisfy the requested `promptOverride` capability.

## Best Possible Solution

Keep #8176 open until OpenClaw either implements and documents an explicit user-prompt replacement field for `before_prompt_build` and legacy `before_agent_start`, or a maintainer rejects that API. A complete fix should update the public result types, prompt-mutation field allowlist, hook merge precedence, `resolvePromptBuildHookResult`, embedded runner, CLI runner, shared agent harness/Codex path, allowPromptInjection filtering, public docs, and focused regression tests for merge order plus submitted prompt text.

## Risks / Open Questions

- Implementing this remains a public plugin API decision: maintainers need to define precedence when multiple hooks return replacements, whether later hooks see the original or effective prompt, and how `allowPromptInjection=false` constrains replacement fields.
- Docs/tests should cover embedded, CLI, and agent harness/Codex paths together; otherwise one runner could gain `promptOverride` while another still submits the original prompt.
