## Summary

Current main still does not provide a hook result field that lets plugins replace the submitted user prompt. The implemented/public hook surface supports model/provider overrides plus system/context prompt mutations, but not `promptOverride`; related systemPrompt work and the closed unmerged PR do not supersede this issue.

## Best Possible Solution

Keep #8176 open until OpenClaw either implements and documents an explicit user prompt replacement field for `before_prompt_build` and legacy `before_agent_start`, or maintainers explicitly reject that API. A complete fix should update public hook result types, prompt mutation filtering, hook merge precedence, embedded runner, agent harness/Codex path, docs, and regression tests for replacement plus `prependContext` ordering.
