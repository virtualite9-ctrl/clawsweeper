## Summary

Current main already preserves Telegram forum topic context for native exec approval delivery and the post-approval completion follow-up, and the shipped `v2026.4.22` release contains the same implementation.

## Close Comment

Closing from a Codex review because current `main` already implements this behavior, and it is present in shipped release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`). The Telegram native approval path preserves the originating topic via `turnSourceThreadId` in `extensions/telegram/src/approval-native.ts:33-48` and keeps DM-only approvals linked back to the origin topic via `notifyOriginWhenDmOnly` at `extensions/telegram/src/approval-native.ts:116-118`. The completion follow-up path also forwards the original thread id in `src/agents/bash-tools.exec-approval-followup.ts:202-230`. There is a regression test for the exact symptom in `src/infra/heartbeat-runner.ghost-reminder.test.ts:536-600`, which asserts exec completion stays pinned to the original Telegram topic even if the session route drifts. Based on that code and test coverage, this issue is already implemented on main and shipped.
