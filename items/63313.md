## Summary

Keep open. Current main still does not implement either central part of PR #63313: built-in memorySearch has no asymmetric OpenAI-compatible `input_type` config/request path, and the Ollama embedding provider still batches by repeated `/api/embeddings` calls rather than trying `/api/embed`. The PR is a focused implementation for open issues #60727 and #45944, but it still needs maintainer review/rebase and the outstanding review concerns addressed rather than conservative closure.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to ask for a current-main rebase, keep the change scoped to the two linked memory embedding features, and require tests for OpenAI-compatible query/document `input_type`, OpenAI batch request typing/cache-key behavior, Ollama `/api/embed` success, robust fallback to per-item `/api/embeddings`, and malformed or short batch responses.
