## Summary

Keep open. Current main still appears to have the reported race: `sessions.delete` waits for the run registry to report the session idle, but normal agent-run cleanup can still persist the same `sessionKey` afterward and recreate the store row.

## Evidence

- **sessions.delete only gates on runtime cleanup before deleting the store row:** `sessions.delete` calls `cleanupSessionBeforeMutation(...)` and then deletes the session entry from the store with `updateSessionStore(...)`.
  - file: [src/gateway/server-methods/sessions.ts:1376](https://github.com/openclaw/openclaw/blob/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503/src/gateway/server-methods/sessions.ts#L1376)
  - command: `nl -ba src/gateway/server-methods/sessions.ts | sed -n '1376,1438p'`
  - sha: [b0d2003fe65b](https://github.com/openclaw/openclaw/commit/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503)
- **runtime cleanup waits for embedded/reply run end, not for later session-store persistence:** `ensureSessionRuntimeCleanup(...)` aborts the embedded run and waits on `waitForEmbeddedPiRunEnd(sessionId, 15000)` before allowing delete/reset to continue.
  - file: [src/gateway/session-reset-service.ts:225](https://github.com/openclaw/openclaw/blob/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503/src/gateway/session-reset-service.ts#L225)
  - command: `nl -ba src/gateway/session-reset-service.ts | sed -n '214,244p;397,421p'`
  - sha: [b0d2003fe65b](https://github.com/openclaw/openclaw/commit/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503)
- **reply run is marked complete before outer agent-command post-run persistence:** The reply runner clears the reply operation in a `finally` block via `replyOperation.complete()`, which is what the session cleanup wait path observes as idle.
  - file: [src/auto-reply/reply/agent-runner.ts:1773](https://github.com/openclaw/openclaw/blob/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503/src/auto-reply/reply/agent-runner.ts#L1773)
  - command: `nl -ba src/auto-reply/reply/agent-runner.ts | sed -n '1768,1778p'`
  - sha: [b0d2003fe65b](https://github.com/openclaw/openclaw/commit/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503)
- **agent-command persists the session store after the run returns:** After the run finishes, `agent-command` still calls `updateSessionStoreAfterAgentRun(...)` for the same `sessionKey`.
  - file: [src/agents/agent-command.ts:1040](https://github.com/openclaw/openclaw/blob/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503/src/agents/agent-command.ts#L1040)
  - command: `nl -ba src/agents/agent-command.ts | sed -n '1038,1062p'`
  - sha: [b0d2003fe65b](https://github.com/openclaw/openclaw/commit/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503)
- **post-run persistence recreates a missing session entry unconditionally:** `updateSessionStoreAfterAgentRun(...)` falls back to a new `{ sessionId, updatedAt }` entry when `sessionStore[sessionKey]` is absent, then writes `store[sessionKey] = merged`, which can recreate a just-deleted row.
  - file: [src/agents/command/session-store.ts:76](https://github.com/openclaw/openclaw/blob/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503/src/agents/command/session-store.ts#L76)
  - command: `nl -ba src/agents/command/session-store.ts | sed -n '65,186p'`
  - sha: [b0d2003fe65b](https://github.com/openclaw/openclaw/commit/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503)
