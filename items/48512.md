## Summary

Keep this PR open for maintainer handling. Current main still lacks the pending-tool-call hint, adapter, and retry-gap re-wait loop proposed for the embedded runner, but the PR branch is too contaminated with unrelated generated/runtime/hook/UI changes to merge as-is.

## Best Possible Solution

Keep this PR open for maintainer handling, but request or salvage a clean replacement. The replacement should stay scoped to the embedded runner: verify the Pi `state.pendingToolCalls` contract, add a best-effort non-throwing pending-tool-call hint, bridge it into attempt error cleanup, attempt finally cleanup, and compaction cleanup, preserve the total timeout budget across re-waits, and add focused retry-gap plus adapter regression tests. Do not merge the current contaminated branch as-is.
