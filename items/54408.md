## Summary

Current main still has the reported core leak path. The pre-compaction memory flush is only marked silent for delivery, while the flush prompt is still submitted as the embedded run prompt without the existing hidden `transcriptPrompt: ""`/runtime-only seam. Related #51421/#12590 cover cadence, not transcript invisibility, and adjacent UI-leak work remains open rather than shipped.

## Best Possible Solution

Keep this issue open as the core memory-flush transcript leak tracker. The likely fix belongs in `src/auto-reply/reply/agent-runner-memory.ts`: make pre-compaction memory flush a true internal/runtime-only embedded run, probably by passing `transcriptPrompt: ""` or adding an explicit internal-run provenance contract. Add regression coverage proving the flush prompt is absent from session JSONL and `chat.history`, and decide whether polluted existing sessions need repair.
