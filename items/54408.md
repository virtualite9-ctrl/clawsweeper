## Summary

Keep open. Current main still sends the pre-compaction memory flush prompt through the normal embedded prompt submission path with `silentExpected: true`, but without `transcriptPrompt: ""` or another internal-only transcript boundary. That suppresses outbound assistant replies, not the user prompt entry. Docs still promise a silent memory flush, and existing runtime-only prompt support shows a feasible core fix path that is not applied to memory flush yet.

## Best Possible Solution

Keep this issue open as the core memory-flush transcript leak tracker. The best fix is to make pre-compaction memory flush a truly internal turn, likely by routing the flush body through the existing runtime-only/hidden-context mechanism (`transcriptPrompt: ""` or an explicit internal-run API), then adding regression coverage that the session JSONL/UI history contains no user message with the flush prompt. The fix should also sanitize or ignore historical leaked flush prompts during compaction so already-polluted sessions do not keep re-triggering the safeguard loop. Treat #12590/#51421 as the separate cadence-counter bug unless that PR is expanded to cover transcript visibility.
