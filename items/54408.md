## Summary

Current main still has the reported memory-flush transcript leak path. The pre-compaction flush is marked `silentExpected` for assistant/output delivery, but the flush prompt is still passed as the embedded run `prompt` without `transcriptPrompt: ""` or another internal-only transcript marker, so it remains eligible to be submitted/persisted as a normal user turn. The related #51421/#12590 work is about cadence/counter behavior, and the adjacent #58956/#58975 UI-leak work remains open rather than shipped.

## Best Possible Solution

Keep this issue open as the core memory-flush transcript leak tracker. The best fix is to make pre-compaction memory flush a true internal turn, likely in `src/auto-reply/reply/agent-runner-memory.ts` by passing `transcriptPrompt: ""` or by adding an explicit internal-run/provenance contract in the embedded runner. Add regression coverage proving session JSONL and `chat.history` do not contain a user message with the flush prompt, and add a cleanup/sanitization path for already polluted sessions if maintainers want to recover existing transcripts. Keep #51421/#12590 focused on cadence unless that work is explicitly expanded to cover transcript invisibility.
