## Summary

Keep this PR open. Current main still has the cmd-specific Windows allowlist guard, while PowerShell/pwsh are recognized as shell-wrapper transports with inline payloads and can reach policy with `cmdInvocation: false`. The PR is a focused security hardening candidate, but it needs a small revision before maintainer review.

## Best Possible Solution

Keep this PR open and request a narrow revision: block Windows shell wrappers in allowlist mode based on `security === "allowlist" && isWindows && shellWrapperInvocation`, keep the PowerShell wrapper regression, update stale denial-message assertions, and align the Windows node docs/guidance to mention PowerShell wrappers before maintainer review and landing.
