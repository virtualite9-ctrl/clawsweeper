## Summary

Current `main` already implements persistent per-session labels in the Control UI and gateway session store, and the session refresh path preserves custom labels while regenerating channel-derived display names.

## Evidence

- **Control UI exposes editable session labels:** The Sessions view renders a label input for each session and sends `sessions.patch` with the new label on change, so users can rename sessions from the UI.
