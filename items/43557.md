## Summary

Keep open. Current main still exposes `gog` Gmail automation surfaces but does not fix or document the reported intermittent `No auth for gmail` failure for non-interactive macOS scheduled runs. The issue is actionable, not maintainer-authored, and has no protected label; the only label is `stale`.

## Best Possible Solution

Keep this issue open. The maintainer path is to add a cron-safe `gog` auth section to the bundled `gog` skill and public Gmail Pub/Sub/scheduled automation docs, and consider an OpenClaw doctor/preflight warning for non-interactive macOS `gog` auth. Cover stable `HOME`/`PATH`/`GOG_ACCOUNT`/`GOG_CLIENT`, `gog auth keyring file`, `GOG_KEYRING_BACKEND=file`, `GOG_KEYRING_PASSWORD`, and an auth-resolution debug/logging capture before deciding whether an upstream `gogcli` bug is needed.
