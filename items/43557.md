## Summary

Keep open. Current main has OpenClaw-owned `gog` and Gmail hook surfaces, but it does not resolve or document the reported intermittent macOS cron auth failure. The implementation delegates Gmail auth entirely to the external `gog` binary, and the bundled skill/docs only give basic setup and scripting guidance. The report is recent, detailed, and only has a stale-bot comment, so conservative cleanup should not close it.

## Best Possible Solution

Keep this issue open and ask maintainers to either document a cron-safe `gog` setup in the bundled `gog` skill and Gmail Pub/Sub docs, or add an OpenClaw-side preflight/doctor warning for `gog` auth under non-interactive macOS runs. A useful fix would likely mention `gog auth keyring file`, `GOG_KEYRING_BACKEND=file`, `GOG_KEYRING_PASSWORD`, stable `HOME`/`GOG_ACCOUNT`/`GOG_CLIENT`, and how to collect auth-resolution debug logs before deciding whether an upstream `gogcli` issue is also needed.
