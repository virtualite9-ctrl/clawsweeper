## Summary

Keep open. Current main still exposes Gmail/gog automation paths but does not document or handle the reported intermittent `No auth for gmail` failure mode for non-interactive macOS scheduled runs. The issue is recent, concrete, and the provided discussion already includes a prior Codex review that found the same gap.

## Best Possible Solution

Keep this issue open. The best next maintainer path is a focused docs and preflight improvement for non-interactive gog Gmail auth: update `skills/gog/SKILL.md`, `docs/automation/cron-jobs.md`, and possibly `docs/cli/webhooks.md` with cron/launchd-safe `HOME`, `PATH`, account/client, keyring/password, and debug-capture guidance. If those steps still reproduce the intermittent failure, add targeted watcher/doctor diagnostics or escalate to upstream gogcli with redacted logs.
