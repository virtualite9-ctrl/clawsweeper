## Summary

Current `main` now treats `runTimeoutSeconds`-driven subagent aborts as external aborts that surface as timeout/termination instead of entering model fallback retry, and the docs/tests/subagent lifecycle code are aligned with that behavior.

## Evidence

- **Abort-signal timeout is marked as external:** The embedded runner's `onAbort` path sets `externalAbort = true`, reads the abort reason, classifies timeout via `isTimeoutError(reason)`, and then aborts the run with that timeout reason intact.
  - file: [src/agents/pi-embedded-runner/run/attempt.ts:2000](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/src/agents/pi-embedded-runner/run/attempt.ts#L2000)
  - command: `nl -ba src/agents/pi-embedded-runner/run/attempt.ts | sed -n '1996,2016p'`
  - sha: [926068b14fc6](https://github.com/openclaw/openclaw/commit/926068b14fc661e915a0247e9ecb5949fb95b4f7)
- **Failover policy blocks fallback after external abort:** `resolveRunFailoverDecision` returns `surface_error` immediately for `externalAbort` in both prompt and assistant stages, so timeout aborts from the parent signal do not proceed to `fallback_model`.
  - file: [src/agents/pi-embedded-runner/run/failover-policy.ts:124](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/src/agents/pi-embedded-runner/run/failover-policy.ts#L124)
