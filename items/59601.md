## Summary

Keep open. Current `main` still routes node-based skill installs through global npm by default, while the Docker image still runs as non-root `node` without configuring a user-writable npm global prefix. The linked PR #60052 proposes exactly that missing Dockerfile fix, so this does not look implemented on `main` yet.

## Evidence

- **Wizard still offers npm for skill dependency installs and the default node-manager path is global npm:** The onboarding flow asks for a preferred node manager when a skill needs a node install, and the install path still resolves npm to `npm install -g --ignore-scripts <package>`. The default preferences also fall back to `npm` when nothing else is configured.

  - sha: [ec3dbd22a4de](https://github.com/openclaw/openclaw/commit/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46)
