## Summary

Keep #27574 open. Current main still has browser-control backend/API/CLI primitives, screenshots, console/error capture, and an admin `browser.request` gateway method, but it does not implement a Control UI/Dashboard browser preview panel with live screenshots, URL/title, console errors, or manual browser interaction. The existing maintainer-side Codex review comment also explicitly kept this open, and related #63926 remains narrower rather than a complete duplicate.

## Best Possible Solution

Keep #27574 open as the broad remote browser-preview/dashboard request. The best maintainer path is either a core Control UI browser panel backed by existing `browser.request` and browser-control primitives, or first adding a supported Control UI panel contribution API and then implementing the preview as a browser plugin. Coordinate with open #63926 for login handoff/browser management, but do not treat #63926 as a complete replacement for live remote-debugging preview.

## Risks / Open Questions

- A live browser preview/input panel would expose browser pixels, URLs/titles, console/page errors, and possibly remote-control actions, so implementation needs explicit operator-admin authorization, origin protections, rate limiting, and handoff semantics.
- A plugin-based solution likely needs a supported Control UI panel/tab contribution API first; current plugin surfaces can expose backend endpoints but do not cleanly integrate a new dashboard view.
