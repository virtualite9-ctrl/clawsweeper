## Summary

Keep open. This is a focused contributor PR with concrete iPhone Safari screenshots and follow-up, and current main does not fully implement the requested fix. Main has partial dynamic viewport handling on `.shell`, but `html`/`body` still use `height: 100%`, `.shell--chat` still keeps `min-height: 100vh`, and the active agent chat textarea remains below Safari's 16px focus-zoom threshold with no Apple-mobile override. The related #65188 context does not safely supersede this PR because that report says the later PR leaves the agent chat textarea unfixed.

## Best Possible Solution

Keep the PR open for maintainer review. Rebase it onto current main, preserve the accessibility-safe 16px textarea approach instead of `maximum-scale=1`, reconcile the viewport rules with the existing `.shell` `100dvh` support, and add a focused CSS regression check in the existing style tests if maintainers want an automated guardrail.
