## Summary

Keep this PR open. Current main still has both reported reminder-guard gaps: existing cron coverage is session-key only, and compaction retry still clears `successfulCronAdds`. The PR is a focused implementation candidate for the linked open bug #43292, but maintainer review should also check the remaining call-site concern around deriving `agentId` when `sessionKey` is absent.

## Best Possible Solution

Keep this PR open as the active implementation candidate for the linked open bug #43292. The best maintainer path is to preserve `successfulCronAdds` across compaction retry, add isolated-cron matching by `agentId` only for enabled `sessionTarget: "isolated"` jobs, avoid deriving a default `main` agent id from an absent session key at the call site, and keep regression coverage for session-bound, isolated, disabled, empty, missing-agentId, missing-sessionKey, main-without-sessionKey, and store-error cases.
