## Summary

Keep this PR open. Current main still has both reported cron reminder-guard gaps: `hasSessionRelatedCronJobs` only matches enabled jobs by `sessionKey`, `runReplyAgent` only passes `sessionKey` into that helper, and compaction retry still resets `state.successfulCronAdds` to zero. The PR is a focused implementation candidate for the linked open bug, and the provided review thread shows the earlier agent-id fallback concern was addressed by restricting fallback matching to `sessionTarget: "isolated"`. Security review found no workflow, dependency, lockfile, release, secret, or package-resolution changes in the PR file list; the diff is limited to the cron/reminder guard path and tests.

## Best Possible Solution

Keep this PR open as the active implementation candidate for the linked open bug. The best maintainer path is to review the final `sessionTarget === "isolated"` agent-id fallback, preserve `successfulCronAdds` across compaction retry, keep or adapt the added regression tests for isolated, disabled, empty, missing-agentId, main-without-sessionKey, and store-error cases, then land after current-base validation.
