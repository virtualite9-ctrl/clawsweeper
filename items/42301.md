## Summary

Keep this PR open. Current main still rejects Feishu media keys containing `..`, so the linked image-analysis bug is not implemented on main. The PR is a focused bundled Feishu plugin fix, but it should be refreshed to preserve fail-closed validation for exact dot-segment tokens before merge.

## Best Possible Solution

Refresh this PR with the narrow Feishu plugin fix: keep empty, length, control-character, `/`, and `\` rejection; allow embedded `..` inside opaque Feishu keys; explicitly reject exact dot-segment tokens, at least `..` and preferably `.`; and keep regression tests for valid embedded-dot keys plus invalid path-separator and dot-token inputs. No OpenClaw core change is needed.
