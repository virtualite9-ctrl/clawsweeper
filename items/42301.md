## Summary

Keep this PR open. Current main still has the over-broad Feishu external-key guard that rejects any key containing `..`, which matches the linked Feishu image-analysis regression, and the latest PR patch is directionally useful after preserving slash/backslash rejection. It needs a small security hardening before landing rather than stale closure.

## Best Possible Solution

Refresh and land the Feishu plugin fix with a narrow validator rule: keep empty/length/control-char and slash/backslash rejection, allow embedded `..` inside opaque Feishu keys, and add exact `.` / `..` rejection plus regression tests for both valid embedded-dot keys and invalid dot-segment/path-separator inputs. No OpenClaw core change is needed.
