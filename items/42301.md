## Summary

Keep this PR open. Current main still rejects opaque Feishu media keys containing `..`, so the linked Feishu image-analysis bug is not implemented on main. The PR is focused on the right bundled plugin path, but it still needs a small security hardening update before merge to reject exact `.` and `..` keys while allowing embedded `..` substrings.

## Best Possible Solution

Refresh this PR with the narrow Feishu plugin fix: keep empty/length/control-character and `/`/`\` rejection, allow embedded `..` inside opaque Feishu keys, explicitly reject exact `.` and `..`, and add regression tests for valid embedded-dot keys plus invalid dot-segment/path-separator inputs. No OpenClaw core change is needed.
