## Summary

Keep #39120 open. Current main at 406ae72fd2789c78053f6b94006ec8f488bc1c00 still limits `bootstrap-extra-files` to recognized bootstrap basenames, lacks a first-class custom bootstrap-file config key, and docs/tests continue to describe and assert that limitation. Related PR #39174 was closed unmerged, and #39635 has now been closed as a duplicate of this broader tracker rather than superseding it.

## Best Possible Solution

Keep #39120 open for maintainer/product decision. A direct fix should either widen the bundled `bootstrap-extra-files` contract or add a typed custom bootstrap-file config key, while preserving workspace realpath checks, deterministic ordering, per-file and total bootstrap budgets, subagent/lightweight-context filtering, docs, schema/help artifacts, and focused tests for arbitrary basenames, missing files, traversal, and truncation. Since #39635 was closed as duplicate, use #39120 as the canonical tracker until a design ships or maintainers explicitly move the work to ClawHub/plugin guidance.
