## Summary

Keep #39120 open. Current main still restricts the bundled `bootstrap-extra-files` loader to recognized bootstrap basenames, has no first-class `extraBootstrapFiles`/`contextFiles` config, and tests/docs continue to assert the restriction. #39174 was closed unmerged, and #39635 is an overlapping narrower config tracker, not a shipped or clearly canonical replacement for this broader hook/config request.

## Best Possible Solution

Keep #39120 open for maintainer/product decision. A direct fix should either widen the bundled `bootstrap-extra-files` contract or add a typed first-class custom bootstrap-file config key, preserve workspace realpath and size/budget guards, define subagent/lightweight-context behavior for arbitrary files, and update docs plus tests. Coordinate with #39635 if maintainers choose the config route, then close one item only after a canonical design ships or is explicitly superseded.
