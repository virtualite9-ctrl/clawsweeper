## Summary

Keep #16896 open. Current main at e29d3516bf05591e6007bfb8ccdaea6d76194d41 still lacks the requested Dashboard/WebChat user reply workflow: no message-level context menu or Reply action, no selected reply state in the chat view, and no reply/quote field in the `chat.send` protocol. The related #29732 was closed in favor of this issue, so #16896 remains the canonical tracker.

## Best Possible Solution

Keep #16896 open as the canonical Dashboard/WebChat reply UX request. A complete fix should add an accessible message-level Reply action/context menu in the Control UI chat surface, store selected reply target or quoted text in composer state, focus the input, and send either a designed quote representation or an additive `chat.send` reply-target field with protocol, rendering, docs, and UI/controller tests updated together.
