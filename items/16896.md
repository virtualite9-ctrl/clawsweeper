## Summary

Keep #16896 open. Current main still lacks a user-initiated Dashboard/WebChat right-click or message-level Reply workflow, composer reply state, and a `chat.send` reply target or quote field. The provided discussion also already treats #16896 as the canonical tracker after #29732 was closed as the duplicate/cross-reference.

## Best Possible Solution

Keep #16896 open as the canonical Dashboard/WebChat reply UX tracker. A complete fix should add an accessible message-level Reply action or context menu, store the selected reply target or quoted text in composer state, focus the input, and send/render a designed quote representation or additive `chat.send` reply-target field with matching UI, protocol, controller, and regression tests.
