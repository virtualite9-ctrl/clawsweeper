## Summary

Keep #16896 open. Current main still lacks the requested Dashboard/WebChat right-click or message-level Reply workflow. The UI has draft/send/session/sidebar state and assistant-side reply-pill display metadata, but no selected reply target, no context-menu/right-click handler, no user Reply action, and no `chat.send` reply/quote field for WebChat-originated messages. The existing steipete/Codex comment also explicitly keeps #16896 as the canonical tracker, with related #29732 already closed back to this issue.

## Best Possible Solution

Keep #16896 open as the canonical Dashboard/WebChat reply UX tracker. A complete fix should add an accessible message-level Reply action and/or right-click context menu, store the selected quoted message in composer state, focus the input, render the selected quote clearly, and decide whether sending plain quoted text is sufficient or whether an additive `chat.send`/message metadata field is needed with UI and gateway regression tests.
