## Summary

Keep #39599 open. Current main still accepts Feishu/Lark announce cron delivery without `delivery.to` at the gateway, protocol schema, service, and CLI creation paths, while runtime target resolution can still fail later when no explicit or deterministic target exists. The linked PRs #39602, #39683, and #40031 are open/unmerged candidate fixes rather than shipped replacements.

## Best Possible Solution

Keep #39599 open and land one focused canonical fix. The fix should add gateway/service create and update validation for Feishu/Lark announce cron delivery when neither explicit `delivery.to` nor a deterministic resolved target is available, with focused regression coverage for gateway validation, cron service create/update behavior, and CLI fail-fast behavior if the CLI layer changes.
