## Summary

Current `main` improves embedded timeout handling by surfacing a timeout payload and blocked liveness, but I did not find strong evidence that it now emits a timeout-specific agent lifecycle error event for `api.runtime.events.onAgentEvent()`, and the proposed structured `errorType` field is not present.

## Evidence

- **Timeout handling is result-level, not an explicit event emit:** The embedded runner converts a silent timeout into an `isError` payload and sets terminal lifecycle metadata, but this branch does not call `emitAgentEvent`; it returns `{ payloads: [{ text: timeoutText, isError: true }], meta: { livenessState, ... } }` after `attempt.setTerminalLifecycleMeta(...)`.
  - file: [src/agents/pi-embedded-runner/run.ts:1779](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/agents/pi-embedded-runner/run.ts#L1779)
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **`lifecycle:error` depends on assistant `stopReason === "error"`:** `handleAgentEnd` emits `stream: "lifecycle"` with `phase: "error"` only when the last assistant message has `stopReason === "error"`; otherwise it emits `phase: "end"`.
  - file: [src/agents/pi-embedded-subscribe.handlers.lifecycle.ts:41](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/agents/pi-embedded-subscribe.handlers.lifecycle.ts#L41)
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **Terminal lifecycle metadata cannot mark a timeout as an error event:** `setTerminalLifecycleMeta` only stores `replayInvalid` and `livenessState`; it has no field for `phase`, timeout classification, or an `errorType` discriminator.
  - file: [src/agents/pi-embedded-subscribe.ts:853](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/agents/pi-embedded-subscribe.ts#L853)
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **Current timeout tests assert payloads and blocked state, not timeout lifecycle events:** The timeout coverage I found checks that timeouts return `isError` payload text and `meta.livenessState === "blocked"`, but I did not find a test asserting a timeout-specific `onAgentEvent` lifecycle error for embedded runs.
  - file: [src/agents/pi-embedded-runner/run.timeout-triggered-compaction.test.ts:199](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/agents/pi-embedded-runner/run.timeout-triggered-compaction.test.ts#L199)
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **Proposed structured timeout field is absent on main:** A repo-wide search for `errorType` in `src`, `docs`, and `test` returned no matches.
  - command: `rg -n "errorType" src docs test`
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
