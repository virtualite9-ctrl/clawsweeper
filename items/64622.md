## Summary

Keep this PR open. Current main still lacks the PR's central A2A timeout/provenance, webchat-backed main-session metadata cleanup, and outbound mirror continuity changes, but the PR branch is not merge-ready as-is because the review-discussed timeout and outbound-route regressions are source-backed and the branch also includes unrelated docs-template changes.

## Best Possible Solution

Keep this PR open for revision or replacement. A mergeable path should split out the unrelated docs-template commit, scope any timeout floor to true cross-agent sends with focused zero/short/long timeout tests, add webchat-backed main-session metadata cleanup in gateway persistence with regression coverage, and preserve outbound session continuity without overwriting the resolver-selected route key.

## Risks / Open Questions

- Merging the timeout resolver as proposed could turn explicit short same-agent or local sessions_send waits into 45-second waits because the floor is not limited to true cross-agent sends.
- Merging the outbound mirror rewrite as proposed could persist external channel/chat metadata onto the current main or webchat session instead of the resolver-selected channel session, affecting later routing and send-policy decisions.
- The branch mixes unrelated docs-template policy additions into a focused gateway/A2A runtime fix, making review and rollback harder.
- Closing now would discard a plausible bug-fix thread while current main still lacks the central A2A/session-metadata changes.
