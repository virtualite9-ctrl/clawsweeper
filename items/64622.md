## Summary

Keep open. Current main does not already implement the PR's claimed A2A timeout floor, announce provenance removal, webchat-backed main-session metadata sanitization, or current-session outbound mirror behavior; however, the PR is also not safe to close as implemented or obsolete because it still contains unresolved P1 review risks and an unrelated docs-template commit.

## Best Possible Solution

Keep this PR open for revision or replacement. The maintainer path is a narrower change that splits out the docs-template commit, limits any timeout floor to true cross-agent sends without changing same-agent explicit timeout semantics, adds focused resolver tests, sanitizes stale webchat-backed main-session metadata in gateway persistence with regression coverage, and preserves the channel resolver's outbound route while using currentSessionKey only through the existing plugin/thread-aware resolver contract.
