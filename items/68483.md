## Summary

Keep this PR open. Current main still only exposes `contentRedacted` for redacted `sessions_history` output and does not implement the proposed machine-readable `[OPENCLAW-REDACTED]` marker, helper, or `notice` field. The PR is a plausible implementation candidate for the linked open bug rather than obsolete cleanup, but it still needs maintainer review and likely follow-up for the truncation false-positive review comment before merge.

## Best Possible Solution

Keep the PR open as the active implementation candidate for the linked open redacted-output marker bug. The maintainer path is to review and iterate the PR on current main, fix the post-truncation notice condition, add/update user-facing session-tool docs for the new `notice` contract if accepted, fix changelog attribution, and then run the targeted logging/session-history tests plus the changed gate before merge.
