## Summary

Keep #8181 open. Current main has useful partial support for explicit `memorySearch.provider: "local"`: it defaults to the EmbeddingGemma GGUF URI and calls the `node-llama-cpp` model resolver. The zero-config request is still not implemented because `node-llama-cpp` is not a normal installed dependency, runtime/docs still require manual install/approve/rebuild steps, and clean `provider: "auto"` still skips the default downloadable local model.

## Best Possible Solution

Keep #8181 open until maintainers either implement true turnkey local embeddings or explicitly decide that local embeddings remain a manual optional runtime. A complete fix should handle native runtime install/build or prebuilt binaries, first-use GGUF download with clear progress and failure messaging, clean `auto` versus explicit `local` semantics, and matching docs plus tests.
