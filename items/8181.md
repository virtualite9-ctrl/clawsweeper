## Summary

Keep #8181 open. Current main still only partially satisfies zero-config local memory embeddings: explicit `provider: "local"` can resolve/download the default GGUF model, but the native `node-llama-cpp` runtime is still operator-installed/manual, and auto mode still skips local unless `memorySearch.local.modelPath` points to an existing local file.

## Best Possible Solution

Keep #8181 open until maintainers either implement true turnkey local embeddings or explicitly decide and document that local embeddings remain a manual optional runtime. A complete fix should cover native runtime install/build or prebuilt binaries, first-use model download with clear progress and failure messaging, `auto` versus explicit `local` semantics, and matching docs/tests.
