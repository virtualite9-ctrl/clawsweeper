## Summary

Keep #8181 open. Current main has explicit local-embedding progress: `provider: "local"` now falls back to the default downloadable GGUF model. The zero-config request is still not satisfied because `node-llama-cpp` remains an operator-installed optional runtime, setup errors still direct users to install/approve/rebuild it manually, and auto-selection still skips local embeddings on a clean setup unless an existing local model file path is configured.

## Best Possible Solution

Keep #8181 open until maintainers either implement true turnkey local embeddings or explicitly decide and document that local embeddings remain a manual optional runtime. A complete fix should cover native runtime install/build or prebuilt binaries, first-use model download with clear progress/failure messaging, `auto` versus explicit `local` semantics, and matching docs/tests.
