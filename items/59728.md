## Summary

Keep open. Current `main` still has a largely flat `scripts/` root, no `scripts/README.md` index, and package/workflow references still target the old top-level script paths, so the requested reorganization has not been implemented well enough to close conservatively.

## Evidence

- **Large flat root remains:** A read-only count on current `main` shows 243 top-level files directly under `scripts/` (`find scripts -maxdepth 1 -type f | wc -l`), which conflicts with the issue’s acceptance criterion that no scripts remain in the root directory.
  - command: `find scripts -maxdepth 1 -type f | sort | wc -l`
  - sha: [ec3dbd22a4de](https://github.com/openclaw/openclaw/commit/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46)
- **No scripts index README:** The only top-level guidance file in `scripts/` is `AGENTS.md`, and it describes wrapper and generated-artifact rules rather than providing the requested script taxonomy/index. A history lookup for `scripts/README.md` returned no commits.
  - file: [scripts/AGENTS.md:1](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/scripts/AGENTS.md#L1)
  - command: `git log --oneline --follow -- scripts/README.md`
  - sha: [ec3dbd22a4de](https://github.com/openclaw/openclaw/commit/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46)
- **Package scripts still use flat paths:** Current package scripts still invoke root-level files such as `scripts/audit-seams.mjs`, `scripts/check-ts-max-loc.ts`, `scripts/run-node.mjs`, `scripts/analyze-plugin-sdk-usage.ts`, `scripts/openclaw-npm-release-check.ts`, and `scripts/plugin-npm-release-plan.ts` instead of a reorganized directory taxonomy.
  - file: [package.json:1281](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/package.json#L1281)
  - sha: [ec3dbd22a4de](https://github.com/openclaw/openclaw/commit/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46)
- **Workflow references were not migrated:** GitHub workflows still reference the old flat locations, including `node scripts/ci-changed-scope.mjs` in CI and `scripts/plugin-npm-release-plan.ts` in the plugin release workflow path filters and execution steps.
  - file: [.github/workflows/ci.yml:99](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/.github/workflows/ci.yml#L99)
