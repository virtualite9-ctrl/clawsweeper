## Summary

Current `main` already covers both parts of #61746: Gateway chat has an always-visible thinking selector that shows the effective default vs override, and `/think` now resolves/display-validates model-specific thinking options from provider-owned thinking profiles instead of a generic ladder.

## Evidence

- **Visible chat thinking indicator in header:** Gateway chat renders a persistent thinking selector beside the session/model controls. Its default option text is `Default (<resolved level>)`, and when a session override exists the selected option switches to that override, which matches the requested always-visible effective-level indicator.
