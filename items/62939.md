## Summary

Keep open. Current main has meaningful partial hardening for wrapped external content such as web fetch/search, decoded OpenResponses file text, browser content, channel metadata, and inbound system-tag spoofing. It does not yet implement the central request: generic trust-boundary wrapping for non-owner user messages and open-world tool results at the agent/provider message boundary, plus a stable system-prompt trust anchor for those wrappers. The linked implementation PR from the issue context was closed unmerged, so there is no merged canonical fix for this issue.

## Best Possible Solution

Keep this issue open and implement a core-owned, tested boundary strategy that composes with existing `wrapExternalContent`: add stable system-prompt guidance, wrap or otherwise label non-owner current messages at prompt assembly time, classify/wrap open-world tool results in the shared transport or tool-result persistence path, and add regression tests for owner vs non-owner messages, web_fetch/web_search/x_search, external MCP tool results, and existing external-content marker sanitization.
