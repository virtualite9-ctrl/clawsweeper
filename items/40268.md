## Summary

Current main still fails closed on unknown config keys during normal config reads. There are mitigation paths for last-known-good recovery and `doctor --fix` unknown-key repair, but they do not implement the requested strict-write/tolerant-read behavior for gateway startup or hot reload.

## Best Possible Solution

Keep this issue open. The best fix is an explicit read-path validation mode for gateway startup and hot reload that strips or quarantines unknown keys, logs exact config paths, and preserves strict validation for `config set`, `config.patch`, `config.apply`, and `config validate`. Add regression coverage for unknown root keys and nested misplaced keys, plus tests proving existing last-known-good and doctor repair behavior still work.
