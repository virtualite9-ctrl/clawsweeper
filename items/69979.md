## Summary

Close #69979 as duplicate/superseded by open bug #66406 and canonical PR #66478. Current main still forwards raw Feishu reaction emoji to the API, so this is not implemented, but the same remaining fix is already tracked by the older canonical issue/PR pair. The cross-referenced #71970 also confirmed the same root cause and was closed unmerged as superseded by #66478.

## Best Possible Solution

Close #69979 as a duplicate/superseded report and keep #66406 plus PR #66478 as the canonical thread. The best fix is one Feishu-owned Unicode-to-Feishu-emoji-type normalizer wired into add/remove/list reaction paths, with regression tests, folding in useful #71970 review details such as module-level mapping and bare-heart coverage before landing.

## Evidence

- **Current main forwards the user emoji unchanged from the Feishu message action path:** On current main, the Feishu `react` handler trims `ctx.params.emoji` and passes the same value as `emojiType` to both `listReactionsFeishu()` and `addReactionFeishu()`. A call such as `emoji="👍"` therefore remains raw Unicode at the Feishu reaction boundary.
  - file: [extensions/feishu/src/channel.ts:1087](https://github.com/openclaw/openclaw/blob/7e13f3f5140168e31027b39a9855ade3fadf24d5/extensions/feishu/src/channel.ts#L1087)

  - sha: [7e13f3f51401](https://github.com/openclaw/openclaw/commit/7e13f3f5140168e31027b39a9855ade3fadf24d5)
- **Current main sends `emojiType` directly as Feishu `reaction_type.emoji_type`:** `addReactionFeishu()` documents that `emojiType` should be a Feishu type such as `SMILE`, `THUMBSUP`, or `HEART`, then sends the supplied string directly as `reaction_type.emoji_type` without converting Unicode emoji.
  - file: [extensions/feishu/src/reactions.ts:44](https://github.com/openclaw/openclaw/blob/7e13f3f5140168e31027b39a9855ade3fadf24d5/extensions/feishu/src/reactions.ts#L44)

- **No shipped Feishu emoji normalizer exists on current main:** A targeted source search found only direct pass-through sites for `emoji_type`, `emojiType: emoji`, and `reaction_type: emojiType`; there is no `normalizeFeishuEmoji`, `toFeishuEmojiType`, or `UNICODE_TO_FEISHU` helper in `extensions/feishu/src`.
  - command: `rg -n "normalizeFeishuEmoji|toFeishuEmojiType|UNICODE_TO_FEISHU|emojiType: emoji|reaction_type: emojiType|emoji_type: emojiType" extensions/feishu/src -S`
