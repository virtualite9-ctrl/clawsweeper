## Summary

Current `main` already addresses this stale Codex profile conflict: new OpenAI Codex logins now resolve to named/stable profile ids instead of always `openai-codex:default`, runtime auth-store merge logic replaces stale legacy default references with the fresher same-account profile from the main store, and doctor/config repair paths explicitly migrate legacy `openai-codex:default` references.

## Evidence

- **Codex OAuth login now prefers named/stable profile ids:** `buildOpenAICodexProviderPlugin()` derives `email`/`profileName` from the Codex access token and passes them into `buildOauthProviderAuthResult(...)`, so relogin produces `openai-codex:<email>` or a stable `id-...` profile when identity is available instead of blindly reusing `openai-codex:default`.
  - file: [extensions/openai/openai-codex-provider.ts:325](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/extensions/openai/openai-codex-provider.ts#L325)
  - command: `nl -ba extensions/openai/openai-codex-provider.ts | sed -n '320,430p'`
  - sha: [a9797214338b](https://github.com/openclaw/openclaw/commit/a9797214338ba31c52c796adbb75afb16e0684a9)
- **Stable identity extraction is explicit:** `resolveCodexAuthIdentity()` prefers the JWT email claim, otherwise derives a stable subject-based `id-...` profile name; it only falls back to `{}` when no identity can be recovered.
  - file: [extensions/openai/openai-codex-auth-identity.ts:69](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/extensions/openai/openai-codex-auth-identity.ts#L69)

- **Runtime merge repairs stale default references:** `reconcileMainStoreOAuthProfileDrift()` detects legacy `provider:default` OAuth entries in an agent override store, finds a newer same-provider replacement in the main store, and rewrites merged `profiles`, `order`, `lastGood`, and `usageStats` to use the replacement profile id.

## Risks / Open Questions

- Conclusion is based on current code and regression coverage rather than a live OAuth reproduction against OpenAI.
- The repair path is scoped to legacy default-profile drift for the same provider/account; different-account duplicates are intentionally preserved.
