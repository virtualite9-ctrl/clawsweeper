## Summary

DeepInfra support is useful, but this PR adds an optional service-specific model provider. Current main already has the provider-plugin/auth/catalog/runtime hooks needed to ship it outside core, and the linked DeepInfra bug report #11533 was already closed as implemented through the generic OpenAI-compatible provider path. The best cleanup is to close this core PR and ask for a ClawHub/npm DeepInfra provider plugin unless maintainers explicitly decide DeepInfra should be bundled.

## Best Possible Solution

Close this core PR and invite the contributor to publish the DeepInfra provider as a ClawHub/npm plugin using the existing provider-plugin SDK. Keep any genuinely generic SDK need, such as preserveDiscoveryOrder, as a separate focused maintainer-reviewed API proposal.
