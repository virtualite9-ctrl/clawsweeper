## Summary

Keep #48241 open. Current main does not implement the requested discard gate for aborted runs. It explicitly keeps non-empty buffered assistant text on abort, broadcasts it as an assistant message, and persists it to transcript with abort metadata. The related PR #48283 is an open, unmerged candidate fix, so it is not shipped evidence that the issue is resolved.

## Best Possible Solution

Keep #48241 open until core has a shipped abort-output gate that proves buffered content belongs to the active run or discards it on abort. The likely fix belongs in gateway/agent abort handling around `chatRunBuffers`, transcript persistence, and channel delivery, with regression coverage for rapid successive messages and stale buffered assistant text. PR #48283 can be used as related prior art, but it should not supersede the issue until merged or replaced by an equivalent mainline fix.
