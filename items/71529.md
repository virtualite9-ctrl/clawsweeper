## Summary

Current main, and the shipped `v2026.4.23` release, already classify Telegram `EHOSTUNREACH` failures as recoverable, tag polling fetch failures so they are recognized as Telegram polling errors, and restart/suppress those failures instead of letting them take down the Gateway.

## Evidence

- **Release code marks `EHOSTUNREACH` recoverable for Telegram network handling:** `RECOVERABLE_ERROR_CODES` in the shipped `v2026.4.23` tree includes `EHOSTUNREACH`, so this error class is already treated as recoverable rather than fatal.
