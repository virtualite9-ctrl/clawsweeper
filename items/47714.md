## Summary

Keep open. Current `main` does not implement Mistral-specific `strict9` tool-call ID replay for OpenRouter models, and the exact mechanism proposed in PR #47714 would not fix it on current `main` because static provider `capabilities` are deprecated and ignored at runtime. The underlying request still looks valid, but this PR is not strong close evidence and is not safe to treat as already implemented.

## Evidence
