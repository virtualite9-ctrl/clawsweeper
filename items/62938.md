## Summary

Keep this PR open. Current main still resolves exec-approvals file and socket defaults from `~/.openclaw` instead of `OPENCLAW_STATE_DIR`, so the linked bug remains real. The PR is a plausible implementation candidate, but it overlaps #65736 and touches security-sensitive symlink handling that still needs maintainer review before landing or consolidation.

## Best Possible Solution

Keep this PR open for maintainer handling. The best path is to consolidate it with #65736 or explicitly pick one branch, then land a focused fix that derives exec approval file/socket defaults from the shared state-dir resolver, preserves existing default and `OPENCLAW_HOME` fallback behavior, adds `OPENCLAW_STATE_DIR` precedence/write-path tests, and deliberately resolves or splits the symlink/alias and adjacent hardcoded-path review issues.
