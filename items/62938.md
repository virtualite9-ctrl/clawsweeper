## Summary

Current main still does not implement the requested exec-approvals state-dir fix. The default exec approvals file and socket still resolve through hardcoded `~/.openclaw` paths rather than `resolveStateDir()`, while the linked bug #62917 and overlapping PR #65736 remain open. This PR is still actionable and should not be closed as obsolete.

## Best Possible Solution

Keep this PR open for maintainer handling. The best path is to consolidate it with #65736 or explicitly choose one surviving branch, then land a focused fix that derives exec approvals file/socket defaults from `resolveStateDir()`, preserves `OPENCLAW_HOME` fallback behavior, adds `OPENCLAW_STATE_DIR` precedence and file-creation coverage, and either includes or deliberately splits the adjacent hardcoded-path cleanups raised in review.
