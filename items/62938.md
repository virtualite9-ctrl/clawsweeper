## Summary

Keep open. Current main still resolves `exec-approvals.json` and `exec-approvals.sock` from hardcoded `~/.openclaw` defaults instead of `OPENCLAW_STATE_DIR`, so the central bug this PR targets is not implemented on main. The provided discussion also shows the patch is directionally correct but incomplete because related hardcoded host-path labels and plugin-binding approval paths remain.

## Best Possible Solution

Keep this PR open for maintainer handling. The best path is to revise or consolidate it with #65736 and land a single focused fix that derives exec approvals file/socket defaults from `resolveStateDir()`, preserves the existing `OPENCLAW_HOME` fallback through that helper, adds `OPENCLAW_STATE_DIR` precedence/file-creation tests, and either includes or explicitly splits the hardcoded fallback label and plugin-binding approval path noted in review.
