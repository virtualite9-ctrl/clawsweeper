## Summary

Current main still resolves exec-approvals file and socket defaults from `~/.openclaw` rather than `OPENCLAW_STATE_DIR`, so this PR is not obsolete. The PR remains a plausible implementation candidate, but it overlaps open PR #65736 and its symlink-path changes touch a security-sensitive approval-store write path that still needs maintainer review or consolidation.

## Best Possible Solution

Keep this PR open for maintainer handling. The best path is to consolidate it with #65736 or choose one canonical branch, then land a focused fix that derives exec-approval file and socket defaults from `resolveStateDir()`, preserves default and `OPENCLAW_HOME` fallback behavior, adds `OPENCLAW_STATE_DIR` regression coverage, and explicitly resolves the symlink/trusted-root review concerns.
