## Summary

Current `main` already addresses the reported external-plugin SDK resolution bug and the related external provider env-var lookup gap. The fix is present in code and covered by regression tests, but the changelog places it under `Unreleased`, so it does not appear to have shipped in a tagged release yet.

## Close Comment

Closing from a Codex review because current `main` already implements this. The loader path now carries `argv1` and `moduleUrl` into plugin-sdk alias resolution (`src/plugins/sdk-alias.ts:599`, `src/plugins/sdk-alias.ts:749`), and there are regression tests for user-installed plugins resolving through both hints (`src/plugins/sdk-alias.test.ts:751`, `src/plugins/sdk-alias.test.ts:772`). The related provider-auth gap is also addressed on `main`: runtime env-var lookup now reads `providerAuthEnvVars` from the manifest registry (`src/secrets/provider-env-vars.ts:76`) with coverage for later-installed external plugins (`src/secrets/provider-env-vars.dynamic.test.ts:37`). I could only verify this as present on current `main`; the changelog entry is still under `Unreleased`, so it may not be in the latest tag yet. If this still reproduces on a newer build after that work ships, please open a fresh issue with the exact plugin package and install path.
