## Summary

Keep open. Current `main` does not implement the requested reusable `@openclaw/gateway-client` package. OpenClaw still has a Node-specific internal gateway client, a separate browser Control UI client, and docs that point third-party clients at the internal reference client rather than a standalone consumable SDK.

## Evidence

- **No dedicated workspace package exists:** The repo's `packages/` directory contains only `memory-host-sdk`, `plugin-package-contract`, and `plugin-sdk`; there is no `packages/gateway-client`.
  - command: `find packages -maxdepth 2 -name package.json | sort`
  - sha: [f3c37a946cc4](https://github.com/openclaw/openclaw/commit/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4)
- **Root package exports only a plugin SDK subpath, not a standalone gateway client package:** `package.json` exposes `./plugin-sdk/gateway-runtime`, but there is no `./gateway-client` export or separate package surface matching the issue request.
  - file: [package.json:360](https://github.com/openclaw/openclaw/blob/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4/package.json#L360)
  - sha: [f3c37a946cc4](https://github.com/openclaw/openclaw/commit/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4)
- **Existing public seam is a narrow re-export of the internal client:** `src/plugin-sdk/gateway-runtime.ts` just re-exports `GatewayClient` from `src/gateway/client.ts` plus a few helpers; it is not a platform-agnostic SDK package with browser/mobile adapters.
  - file: [src/plugin-sdk/gateway-runtime.ts:1](https://github.com/openclaw/openclaw/blob/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4/src/plugin-sdk/gateway-runtime.ts#L1)
  - sha: [f3c37a946cc4](https://github.com/openclaw/openclaw/commit/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4)
- **Current core gateway client is Node-specific:** `src/gateway/client.ts` imports `node:crypto`, `ws`, filesystem-backed device auth helpers, and uses `NodeJS.Timeout`, which matches the issue's complaint that the existing client is coupled to Node/CLI internals.
  - file: [src/gateway/client.ts:1](https://github.com/openclaw/openclaw/blob/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4/src/gateway/client.ts#L1)
  - sha: [f3c37a946cc4](https://github.com/openclaw/openclaw/commit/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4)
- **Control UI still maintains its own separate browser client:** `ui/src/ui/gateway.ts` defines its own event/response/error/connect types and browser-side auth/reconnect logic instead of consuming a shared gateway client package, so the reported duplication still exists on `main`.
  - file: [ui/src/ui/gateway.ts:1](https://github.com/openclaw/openclaw/blob/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4/ui/src/ui/gateway.ts#L1)
  - sha: [f3c37a946cc4](https://github.com/openclaw/openclaw/commit/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4)
- **Docs still describe the internal client as the third-party baseline:** `docs/gateway/protocol.md` says the reference client is `src/gateway/client.ts` and that its defaults are the expected baseline for third-party clients, which indicates the requested reusable public SDK has not been shipped.
  - file: [docs/gateway/protocol.md:486](https://github.com/openclaw/openclaw/blob/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4/docs/gateway/protocol.md#L486)
  - sha: [f3c37a946cc4](https://github.com/openclaw/openclaw/commit/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4)
