## Summary

Keep open. Current main still does not implement the Open WebUI visible-history mirroring requested by #49205: `chat.send` updates OpenClaw transcript/broadcast state, but there is no Open WebUI DB/history sync helper or hook after the final assistant turn. Existing Open WebUI coverage only exercises the normal OpenAI-compatible proxy path, not Control UI writes into the same WebUI chat. The issue is concrete, recent, and distinct from the related UUID session-split issue #49202.

## Best Possible Solution

Keep the issue open. The maintainer path is to decide whether OpenClaw should officially support bidirectional Control UI/Open WebUI chat history sync, then implement a narrow sync path after `chat.send` finalization for WebUI-backed sessions, keyed by actual client identity so `webchat-ui` native sends are skipped but `openclaw-control-ui` sends are mirrored. Add an Open WebUI integration/regression check that creates or reuses a WebUI chat, sends from Control UI, and verifies the visible WebUI history source, plus docs for supported behavior and any one-time repair limitation.
