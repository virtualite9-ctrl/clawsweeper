## Summary

Current `main` contains pieces of the requested information, but not enough to conservatively close the docs issue. The code and schema define the valid `models.providers.*.api` values and some docs cover OpenAI-compatible, Anthropic-compatible, Ollama, and nested model-ref parsing, but the user-facing documentation is still fragmented and does not clearly consolidate base-URL suffix behavior and custom-provider prefix-stripping semantics the issue asks for.
