## Summary

Keep open. Current main does not show a TUI-side fix for the reported scroll-jump/autoscroll behavior: the TUI appends/updates chat log components and requests renders, but there is no explicit viewport anchor, user scroll state, jump-to-latest affordance, or TUI autoscroll setting. The related #7648 is Control UI/WebChat, and the older #11269 is closed but does not supersede this current TUI report.

## Best Possible Solution

Keep this TUI issue open. The best fix is a TUI-owned viewport policy: track whether the user is already at the bottom, preserve manual scroll position across renders, auto-follow only when appropriate, provide an explicit jump-to-latest/autoscroll preference or shortcut, and add focused TUI tests around send, streaming delta, final, history reload, and manual-scroll scenarios.
