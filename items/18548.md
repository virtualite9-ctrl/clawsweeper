## Summary

Keep open. Current main still implements oversized bootstrap injection with fixed positional head/tail truncation, and the active run paths still feed workspace bootstrap files through that trimmer. Context-engine, compaction-provider, memory-flush, and plugin hook surfaces cover adjacent compaction and observation-extension work, but they do not resolve the core bootstrap relevance request. The existing maintainer Codex review comment also kept #18548 open for this same reason.

## Best Possible Solution

Keep #18548 open until the core bootstrap truncation behavior is fixed or explicitly descoped. The most concrete core path is to replace fixed head/tail slicing in src/agents/pi-embedded-helpers/bootstrap.ts with relevance-aware or structure-aware bootstrap selection, while treating observation extraction as plugin/hook/context-engine work unless a missing SDK API is identified. The compaction portion should be aligned with the existing context-engine and compaction-provider contracts, and possibly split or linked to the broader open compaction architecture issue #58398.
