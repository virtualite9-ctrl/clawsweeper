## Summary

Keep #18548 open. Current main still trims oversized bootstrap files with fixed head/tail positional slices and the normal run paths still inject those trimmed files into the prompt. The compaction and observation-extraction portions may belong in the context-engine/plugin track, including Lossless Claw and #58398, but the core bootstrap relevance request remains unresolved.

## Best Possible Solution

Keep #18548 open, ideally narrowed to the unresolved OpenClaw core bootstrap relevance problem. The likely fix belongs around src/agents/pi-embedded-helpers/bootstrap.ts and buildBootstrapContextFiles: replace fixed head/tail slicing with a relevance-aware or structure-aware selector, then update positional tests and public docs/config wording. Track incremental compaction through #58398/context-engine work and keep observation extraction plugin-driven unless it exposes a concrete missing SDK hook.
