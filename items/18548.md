## Summary

Keep #18548 open. Current main still truncates oversized bootstrap context by fixed head/tail character slices, and the active run and compaction prompt paths still feed workspace bootstrap files through that trimmer. The context-engine, compaction-provider, hook, and Lossless Claw plugin surfaces are relevant for the compaction/observation parts, but they do not resolve the core bootstrap relevance request.

## Best Possible Solution

Keep #18548 open, or split it with bootstrap relevance as the core tracker. The likely code path is a relevance-aware or structure-aware selector in src/agents/pi-embedded-helpers/bootstrap.ts, with tests replacing the current fixed head/tail assertions and docs/config notes updated for the new selection behavior. Track incremental compaction against #58398/context-engine/Lossless Claw work, and treat observation extraction as plugin/hook/context-engine work unless a concrete missing SDK API is identified.
