## Summary

Current `main` already covers the reported ACP spawn lifecycle gap: ACP `sessions_spawn` dispatches the real task through the normal gateway agent path, failed spawn/dispatch paths explicitly clean up the ACP session instead of leaving a silent orphan, and OpenClaw now exposes first-class abort/delete session operations that cancel and close ACP sessions. The changelog shows this lifecycle cleanup shipped in `v2026.4.5`.

## Evidence

- **ACP spawn now dispatches the actual task and cleans up dispatch failures:** `spawnAcpDirect` sends `params.task` via the gateway `agent` method, and on dispatch failure it calls ACP spawn cleanup and returns `dispatch_failed` with the child session key instead of leaving the session silently accepted.
  - file: [src/agents/acp-spawn.ts:1264](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/src/agents/acp-spawn.ts#L1264)
  - command: `nl -ba src/agents/acp-spawn.ts | sed -n '1221,1296p'`
- **Failed ACP spawn cleanup closes runtime, closes session, unbinds, and deletes the session record:** `cleanupFailedAcpSpawn` closes the ACP runtime handle when present, asks the ACP session manager to close the session, unbinds the session binding, and best-effort deletes the session/transcript.
  - file: [src/acp/control-plane/spawn.ts:17](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/src/acp/control-plane/spawn.ts#L17)
  - command: `nl -ba src/acp/control-plane/spawn.ts | sed -n '1,77p'`
- **First-class session abort/delete operations exist on the gateway surface:** The gateway exposes both `sessions.abort` and `sessions.delete`. `sessions.delete` runs ACP-specific cleanup before removing the session, giving the close/kill-style lifecycle control the report asked for.
  - file: [src/gateway/server-methods/sessions.ts:1225](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/src/gateway/server-methods/sessions.ts#L1225)
  - command: `nl -ba src/gateway/server-methods/sessions.ts | sed -n '1225,1455p'`
- **Deleting an ACP session now cancels and closes the ACP runtime:** `cleanupSessionBeforeMutation` calls `acpManager.cancelSession(...)` and then `acpManager.closeSession(..., discardPersistentState: true)` for ACP-backed sessions before delete/reset completes.
  - file: [src/gateway/session-reset-service.ts:265](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/src/gateway/session-reset-service.ts#L265)
  - command: `nl -ba src/gateway/session-reset-service.ts | sed -n '265,325p'`
- **ACP runtime has an explicit close contract and ACPX implements it:** The ACP runtime interface includes `close(...)`, and the bundled ACPX runtime forwards that close call to its backend delegate.
  - file: [src/acp/runtime/types.ts:143](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/src/acp/runtime/types.ts#L143)
