## Summary

Keep #12505 open. Current main has useful opt-in tool sandbox backends and hardening, but it still does not implement the central request: secure-by-default tiered sandbox presets, native plugin/skill process isolation, IPC-proxied SDK calls, manifest-declared env/fs/net/resource policies, and credential scoping. The existing issue discussion also explicitly treats PR #29974 as partial Docker sandbox progress, not closure for this broader architecture request.

## Best Possible Solution

Keep #12505 open as the canonical sandbox/isolation architecture tracker unless maintainers split it into narrower issues. The best path is a core design for secure-by-default tiered sandbox presets, native plugin/skill process isolation or a safer bundle-first execution path, manifest-declared env/fs/net/resource policy, credential scoping, and an IPC/proxy SDK boundary. Treat PR #29974 and current Docker/SSH/OpenShell sandbox backend work as partial opt-in tool-sandbox progress, not closure.
