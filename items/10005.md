## Summary

Keep #10005 open. Current main still renders chat `/status` and the `session_status` tool as status-card text, and there is no `session.statusCard`, `agents.defaults.statusCard`, JSON-only, or equivalent card-suppression control for those chat/tool surfaces. The related closed #13450 CLI `openclaw status --json` path is distinct and does not satisfy this request.

## Best Possible Solution

Keep #10005 open until OpenClaw either adds an explicit status-card visibility control for chat `/status` and `session_status`, or a maintainer makes a product decision not to support it. A complete fix should add config schema/types/help/docs, preserve current card rendering as the default, thread the setting through `buildStatusReply`, `buildStatusText`, `buildStatusMessage`, and `session_status`, and add focused tests for both chat-command and agent-tool paths. Also keep the distinction from CLI `openclaw status --json` clear in docs/tests.
