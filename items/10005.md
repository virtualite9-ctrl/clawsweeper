## Summary

Keep #10005 open. Current main still renders the chat `/status` and `session_status` surfaces as status-card text, has no `session.statusCard` or `agents.defaults.statusCard` config key, and the closed CLI `openclaw status --json` work remains a different surface.

## Best Possible Solution

Keep #10005 open until OpenClaw either adds an explicit status-card visibility control for chat `/status` and the `session_status` tool, or a maintainer decides not to support that product behavior. A complete fix should add config schema/types/help/docs, thread the setting through `buildStatusReply`, `buildStatusText`, `buildStatusMessage`, and `session_status`, keep the current card behavior as the default, and cover both chat and agent-tool paths in targeted tests.
