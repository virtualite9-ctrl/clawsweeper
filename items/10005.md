## Summary

Keep #10005 open. Current main still does not provide `session.statusCard`, `agents.defaults.statusCard`, JSON-only, or equivalent suppression for chat `/status` or the `session_status` tool. The closed CLI `openclaw status --json` request is a separate diagnostics surface, and the privacy-focused #69445 was closed as superseded by this issue, making #10005 the remaining canonical tracker.

## Best Possible Solution

Keep #10005 open until OpenClaw adds an explicit, documented status-card visibility control for chat `/status` and `session_status`, or a maintainer rejects the product direction. A complete implementation should preserve current rendering by default, add schema/types/help/docs for the chosen key, thread the setting through `buildStatusReply`, `buildStatusText`, `buildStatusMessage`, and `session_status`, and add focused tests for both chat-command and agent-tool paths. The existing CLI `openclaw status --json` behavior should remain documented as separate from chat/tool status-card suppression.
