## Summary

Keep open. Current `main` still sends memory-host remote embedding requests through strict `fetchWithSsrFGuard` without enabling trusted env-proxy mode, while the SSRF guard tests explicitly document that strict mode ignores `HTTP_PROXY`/`HTTPS_PROXY` by default. The Gemini memory embedding path still uses that helper for both single and batch embedding calls, and the relevant memory-host/Gemini files are unchanged from `v2026.4.22`.

## Evidence
