## Summary

Keep #33975 open. Current main covers the model-attribution request through `messages.responsePrefix` template variables, but it still does not implement an opt-in approval gate before automatic model fallback.

## Best Possible Solution

Keep the issue open until OpenClaw implements an opt-in fallback approval mode in the core model fallback path, including config schema/types, docs, tests, and channel-appropriate approval UX, or until a maintainer splits that remaining work into a dedicated canonical tracker. Treat `messages.responsePrefix` with `{modelFull}`/`{provider}` as the current answer for model attribution.
