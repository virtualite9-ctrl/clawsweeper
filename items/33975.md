## Summary

Keep #33975 open. Current main covers the model-attribution half through `messages.responsePrefix` template variables, but it still does not implement an opt-in approval gate before automatic fallback to the next model.

## Best Possible Solution

Keep this issue open until OpenClaw adds an opt-in fallback approval mode in the core model fallback path, with config schema/types, docs, tests, and channel-appropriate approval UX. Treat `messages.responsePrefix` with `{modelFull}`/`{provider}` as the current answer for the model-attribution half, or split the remaining fallback-approval work into a dedicated canonical tracker.
