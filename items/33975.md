## Summary

Keep open. Current main covers the model-attribution half through `messages.responsePrefix` templates such as `{modelFull}`, but it still has no opt-in approval gate before executing a fallback model.

## Best Possible Solution

Keep this open as the fallback-approval tracker while treating model attribution as already satisfied by `messages.responsePrefix` templates. A good implementation would add an opt-in per-agent fallback approval policy to `AgentModelConfig` and the Zod schema, pause `runWithModelFallback` before non-primary `runFallbackAttempt`, route approval through an existing approval/notification surface, and add docs plus tests for approve, deny, timeout proceed/fail, cancellation, and channel delivery.
