## Summary

Keep this issue open. Current main covers the model-attribution half through `messages.responsePrefix` templates, but the requested opt-in approval gate before model fallback execution is still missing from config, runtime fallback control flow, docs, and tests. The latest issue discussion also confirms that the remaining ask is fallback approval, not model disclosure.

## Best Possible Solution

Keep this open, or split it so this issue becomes the canonical fallback-approval tracker. Treat `messages.responsePrefix` with `{modelFull}` and `{provider}` as the shipped attribution solution, then implement an opt-in per-agent fallback approval policy in the core model fallback path with schema/types, docs, and tests for approval, denial, timeout, fail-fast/proceed behavior, cancellation, and channel delivery.
