## Summary

Keep #33975 open. Current main covers the model-attribution half through `messages.responsePrefix` template variables, but the fallback-approval half is still missing: configured fallbacks are still attempted automatically after failover-worthy errors, and the model config schema has no approval flag.

## Best Possible Solution

Keep the issue open until OpenClaw either implements an opt-in fallback approval mode in the core model fallback path, with config schema/types/docs/tests and channel-appropriate approval UX, or a maintainer splits that remaining request into a dedicated canonical issue. Continue treating `messages.responsePrefix` with `{modelFull}`/`{provider}` as the current model-attribution answer.
