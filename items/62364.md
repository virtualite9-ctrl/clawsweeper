## Summary

Keep #62364 open. Current main still has a directly blocking `models.list` path: the RPC awaits the Gateway model catalog before replying, the cold catalog path can load provider runtime/catalog hooks, and provider catalog discovery remains serial and normally unbounded outside live-test/live-gateway modes. The Apr 26 issue comments also report 38-56s `models.list` round trips on v2026.4.24 after the related loader-cache discussion, so this is not proven fixed or superseded.

## Best Possible Solution

Keep the issue open. The best maintainer path is to make the Gateway model-catalog path bounded and less blocking: avoid making `models.list` and unrelated startup/UI RPCs wait on full live catalog resolution, return stale/static catalog rows where possible, parallelize or time-box provider catalog hooks in normal Gateway operation, and add regression coverage with an intentionally slow provider catalog hook.
