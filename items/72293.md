## Summary

Keep open. Current main has some no-output announce resilience, but the reported abort classification gap remains: an assistant turn ending with `stopReason: "aborted"` is not propagated into lifecycle or `agent.wait` as an aborted/failed terminal state, so the subagent registry can still treat it as a successful completion.

## Best Possible Solution

Keep this issue open and fix the embedded subagent terminal path. The best implementation is to propagate assistant `stopReason: "aborted"` and `errorMessage` from `handleAgentEnd` into lifecycle events, teach `agent.wait`/subagent registry completion to classify that as killed/failed instead of ok, and add regression coverage for an empty-content aborted assistant turn producing `abortedLastRun: true` plus a parent-visible failure/no-output announcement.

## Evidence
