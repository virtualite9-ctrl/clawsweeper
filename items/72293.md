## Summary

Keep this issue open. Current main still does not propagate a terminal assistant turn with `stopReason: "aborted"` into lifecycle/session/subagent completion as an aborted, killed, timeout, or failed outcome. The no-output announce path is more resilient now, but the abort classification gap remains and can still mark the subagent as successful/done with `(no output)` instead of parent-visible failure.

## Best Possible Solution

Fix the embedded subagent terminal path on main: propagate assistant `stopReason: "aborted"` and sanitized `errorMessage` from `handleAgentEnd` into lifecycle events, teach `agent.wait` and the subagent registry to classify it as killed/failed instead of ok, persist `abortedLastRun: true` where appropriate, and add regression coverage for an empty-content aborted assistant turn producing a parent-visible failure/no-output announcement.
