## Summary

Keep PR #70789 open. Current main still has the reverse newest-first `findLatestCard` matcher in the Control UI tool-card extraction path, so sequential same-name tool calls can still pair the first result with the later unmatched card. The PR is a recent, focused contributor bug fix against the implicated file, and the provided discussion shows the author addressed review feedback and reported green CI. I found no current-main implementation, shipped release evidence, or canonical superseding item that would justify closing it.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land a FIFO/exact-id-first pairing fix in `ui/src/ui/chat/tool-cards.ts` and add a regression test for sequential same-name calls ordered as call, call, result, result, so the linked Control UI missing-output issue stays fixed.
