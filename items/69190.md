## Summary

Keep #69190 open. The report is recent and still has two salvageable concerns: the browser timeout path lacks the concrete URL/log/config needed to reproduce, while the curl/tool-restriction concern overlaps the still-open #47487 tool-policy gap rather than being clearly fixed on current main.

## Best Possible Solution

Keep the issue open and ask for clarification or split follow-up. For the tool-restriction path, reproduce strict browser-only policy with `tools.allow: ["browser"]` plus an explicit `tools.deny: ["exec", "process", "web_fetch"]` defense, ensuring the browser plugin is loaded when `plugins.allow` is configured; attach any remaining exec/curl bypass to #47487. For the timeout path, collect the exact target URL, redacted `openclaw.json` tool/browser settings, browser profile/CDP mode, Docker/WSL networking details, and browser/gateway logs.
