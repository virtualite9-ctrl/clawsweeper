## Summary

Keep this PR open. Current main still has the Signal shared config formatter and DM-policy normalizer going through phone-number normalization, so `uuid:` allowlist entries do not round-trip there even though Signal docs, setup, monitor, and approval-auth paths recognize UUID senders.

## Best Possible Solution

Review and rebase this PR against current main, then land one shared Signal allow-entry normalizer used by `formatAllowFrom`, `signalSecurityAdapter.normalizeDmEntry`, and ideally the existing approval-auth path, with regression tests for `uuid:` allowlist entries and E.164 phone normalization.
