## Summary

Keep this PR open. Current main still normalizes Signal shared `allowFrom` and DM-policy entries as E.164 phone numbers, so documented `uuid:` Signal allowlist entries are not preserved on the plugin-facing config/security path. The PR remains a focused, relevant fix and its diff does not introduce security-sensitive CI, dependency, install, publishing, or secrets changes.

## Best Possible Solution

Rebase and review this PR, then land a shared Signal allow-entry normalizer used by `signalConfigAdapter.formatAllowFrom`, `signalSecurityAdapter.normalizeDmEntry`, and ideally the existing approval-auth normalization path, with regression tests covering `uuid:` allowlist entries, `signal:uuid:` input, wildcard preservation, and E.164 phone normalization.
