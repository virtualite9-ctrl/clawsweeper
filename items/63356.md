## Summary

Keep this PR open. Current main still normalizes Signal shared allowlist entries and DM-policy entries through phone-only E.164 formatting, while the Signal docs and pairing flow support `uuid:<id>` sender IDs. The PR remains a focused Signal-owned fix with regression tests and no observed CI, dependency, lockfile, install/build/release, publishing, generated/vendor, or secret-handling changes.

## Best Possible Solution

Rebase and review this PR, then land it or an equivalent Signal-owned normalizer used by both `signalConfigAdapter.formatAllowFrom` and `signalSecurityAdapter.normalizeDmEntry`. Keep regression coverage for `uuid:`, `signal:uuid:`, wildcard preservation, and E.164 phone normalization, and either canonicalize compact UUIDs or document why that edge is deferred.
