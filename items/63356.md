## Summary

Keep this PR open. Current main still routes Signal shared allowlist formatting and shared DM-policy normalization through phone-only E.164 normalization, while the Signal docs and pairing flow support uuid:<id> senders. The PR is a focused Signal-owned fix with regression tests and no observed CI, dependency, lockfile, install/build/release, publishing, generated/vendor, or secret-handling changes.

## Best Possible Solution

Rebase and review this PR, then land it or an equivalent Signal-owned normalizer used by both signalConfigAdapter.formatAllowFrom and signalSecurityAdapter.normalizeDmEntry. Keep regression coverage for uuid:, signal:uuid:, wildcard preservation, invalid group/username rejection, and E.164 phone normalization; either canonicalize compact UUIDs to the runtime form or explicitly defer that edge.
