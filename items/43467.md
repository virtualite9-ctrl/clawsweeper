## Summary

Keep open. The mathematical bug described by this PR still exists on current main in the moved memory-host SDK implementation: cosineSimilarity still computes both norms only over Math.min(a.length, b.length), and the memory-core fallback vector search still calls that helper. The PR is narrow and its supplied diff does not introduce security-sensitive CI, dependency, script, publishing, or secret-handling changes, but it needs to be rebased/ported because the original src/memory/internal.ts paths are no longer present on main.

## Best Possible Solution

Keep the PR open and ask for a rebase/port of the same cosineSimilarity fix to the current memory-host SDK implementation, with a regression test for different-length vectors on the current test surface. Security review does not show supply-chain or secret-handling concerns in the submitted diff.
