## Summary

Keep this issue open. Current main still installs Playwright Chromium under the versioned hidden Playwright cache path and the user-facing Docker/browser docs still tell users to persist that cache. The same reporter also has an open, unmerged PR with closing syntax for this issue, so the right maintainer path is to review or close that implementation candidate rather than closing the issue early.

## Best Possible Solution

Keep this issue open and review or close #42771 as the implementation candidate. A complete fix should move Docker Playwright browser assets out of `/home/node/.cache`, expose a stable `chrome` or `chromium` executable on PATH, update the Docker and browser-control docs to the shipped path, and add or document validation for browser-enabled and browser-disabled Docker image builds.
