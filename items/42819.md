## Summary

Keep open. Current main still installs Playwright Chromium into the versioned hidden Playwright cache path, the docs still tell Docker users to persist that cache, and the same reporter has an open unmerged PR with closing syntax that targets this exact Docker packaging/docs change.

## Best Possible Solution

Keep this issue open and review or close #42771 as the same-author implementation candidate. A complete fix should move the Docker Playwright browser install out of `/home/node/.cache`, expose a stable `chrome` or `chromium` command on PATH, and update the Docker and browser-control docs to match the shipped path and persistence guidance.
