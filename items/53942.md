## Summary

Keep this issue open. Current main still does not provide user-level targeted overrides or a trust-level policy for the requested core Safety, Tool Call Style, and OpenClaw Self-Update/update-permission prompt sections. The existing surfaces are whole-prompt replacement, provider/runtime replacement for only three named sections, GPT-5 personality overlay controls, and plugin prompt hooks, which do not satisfy the schema-backed user-level policy requested here.

## Best Possible Solution

Keep this open for core agent configuration design. A conservative implementation would add documented, schema-backed default-level and per-agent prompt policy for selected sections, or an explicit trust policy, then wire it through config types/schema/generated docs, gateway config mutation guards, src/agents/system-prompt.ts, and regression tests proving Safety, Tool Call Style, and OpenClaw Self-Update guidance can be selectively replaced or relaxed without requiring full systemPromptOverride. Coordinate with #25493, but preserve this issue's narrower safety/update-permission requirement.
