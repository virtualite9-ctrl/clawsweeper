## Summary

Keep this issue open. Current main still lacks a user-level section override or trust-level config for the cited Safety, Tool Call Style, and self-update permission text; the available controls are whole-prompt replacement, provider-owned overrides for only a few sections, or plugin hook mutation.

## Best Possible Solution

Keep this issue open for maintainer product/API design. The likely fix belongs in core agent configuration: add documented, schema-backed user-level defaults and per-agent overrides for selected prompt sections or an explicit trust policy, then update prompt construction, generated config docs/help, and tests proving Safety, Tool Call Style, and self-update permission text can be selectively replaced or relaxed without requiring whole-prompt replacement. Coordinate the design with the broader #25493 prompt-template discussion.
