## Summary

Keep this issue open. Current main still does not provide the requested user-level targeted overrides or trust-level policy for the Safety, Tool Call Style, or OpenClaw Self-Update/update-permission sections; it only has whole-prompt replacement, GPT-5 personality overlay controls, provider/runtime section overrides for a small section set, and plugin prompt-mutation hooks.

## Best Possible Solution

Keep this open for core agent configuration design. A conservative implementation would add a documented, schema-backed user-level and per-agent prompt policy surface for selected sections, or an explicit trust policy, wire it through `src/agents/system-prompt.ts`, config schema/help/generated docs, gateway config mutation guards, and tests proving Safety, Tool Call Style, and OpenClaw Self-Update guidance can be selectively replaced or relaxed without requiring full `systemPromptOverride`. Coordinate with #25493 while preserving this issue's narrower safety-policy requirement.

## Risks / Open Questions

- Relaxing Safety and Self-Update guidance changes OpenClaw's operator trust boundary and needs explicit maintainer product/security design, not automatic cleanup.
- Whole-prompt replacement exists today, but treating it as the answer would force users to fork and maintain the entire assembled prompt instead of selectively changing the cited sections.
- Plugin prompt hooks can add or replace prompt text, but that is not the documented schema-backed user-level override surface requested here.
