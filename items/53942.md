## Summary

Keep this issue open. Current main still has whole-prompt replacement and provider-owned section overrides, but it does not expose user-level targeted overrides or a trust-level policy for the Safety, Tool Call Style, or OpenClaw Self-Update/update-permission sections requested here.

## Best Possible Solution

Keep this open for core agent configuration design. A conservative implementation would add a documented, schema-backed user-level and per-agent prompt policy surface for selected sections or an explicit trust policy, wire it through `src/agents/system-prompt.ts` and generated config help/docs, and add tests proving Safety, Tool Call Style, and OpenClaw Self-Update guidance can be selectively replaced or relaxed without requiring full `systemPromptOverride`. Coordinate with #25493 while preserving this issue's narrower safety-policy requirement.

## Risks / Open Questions

- Relaxing Safety and self-update guidance changes OpenClaw's operator trust boundary and needs explicit maintainer product/security judgment, not just prompt text plumbing.
- Whole-prompt replacement is available today, but using it as the answer would force users to maintain the entire OpenClaw prompt and bypass future prompt updates instead of providing the targeted, schema-backed control requested here.
