## Summary

Keep open. Current `main` still hardcodes the core Safety and Tool Call Style sections, and the only user-facing config knobs are all-or-nothing `systemPromptOverride` plus the GPT-5 `promptOverlays.gpt5.personality` toggle. Whole-section replacement exists only on the provider/plugin runtime path, not as user-level config, so the requested per-section or trust-level override is not implemented on `main`.

## Evidence

## Risks / Open Questions

- A maintainer could view `agents.defaults.systemPromptOverride` as a workaround, but it is materially broader than the request because it replaces the full OpenClaw prompt instead of selectively relaxing sections.
- Provider/plugin-owned prompt contributions may partially address this for specific runtimes, but they do not satisfy the issue's request for user-level overrides on current `main`.
