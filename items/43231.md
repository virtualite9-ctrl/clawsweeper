## Summary

Keep open. Current main has useful partial coverage: Telegram forum topic names are cached and exposed for the current topic as `TopicName` / `topic_name`. It still does not provide the requested name-to-topic-id resolver, `forum_topics` map, topic list action, or directory endpoint that would let an agent route to another topic by human-readable name. PR #65973 should be treated as partial prior art, not a complete fix.

## Best Possible Solution

Implement this in the Telegram plugin as a durable, agent-visible topic catalog/resolver backed by the existing service-message cache and explicit config. Add a safe `topic-list` or directory-style action that returns known `topic_id <-> title` entries with source/confidence, use it in message routing for Telegram forum targets, and document the Telegram API limitation plus cache semantics. Keep the numeric `:topic:<id>` target path as the deterministic fallback.
