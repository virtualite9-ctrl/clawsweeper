## Summary

Keep open. Current main has useful partial Telegram forum-topic support: learned topic names are exposed and persisted for the current numeric topic, but there is still no agent-visible name-to-topic-id resolver, topic list action, forum_topics map, or topic catalog for routing to another topic by human-readable name. PR #65973 and follow-up persistence work are positive prior work, not a full fix for this issue.

## Best Possible Solution

Keep this issue open and implement the missing capability in the Telegram plugin. Reuse the persisted topic-name cache, optionally merge explicit operator aliases, expose an agent-visible topic list/catalog or resolver action returning known topic_id/title entries with source and freshness, and route only exact unique matches while falling back to numeric topic ids for ambiguity. Add focused tests for listing, duplicate names, stale or closed topics, persistence, and send routing, then update the public Telegram channel docs.
