## Summary

Current `main` still appears to leave Anthropic API-key embedded runs exposed to dotted tool names. The direct Anthropic transport only remaps tool names when `isAnthropicOAuthToken(apiKey)` is true, while the normal API-key path passes `block.name` / `tool.name` through unchanged. The embedded runner now has a provider-owned tool-schema normalization seam, but the bundled Anthropic provider does not implement `normalizeToolSchemas`, so there is no current fix on `main` to sanitize these names for embedded Anthropic API-key requests.
