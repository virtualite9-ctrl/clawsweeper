## Summary

The proposal fits OpenClaw's existing plugin/ClawHub extension model rather than a new core feature. Current `main` already exposes `before_tool_call` hooks that can rewrite params, block execution, or require approval before a tool runs, and that same hook path is bridged into native Codex pre-tool events.

## Evidence

## Risks / Open Questions

- Current hooks can gate, block, and approval-wrap tool calls, but they do not provide a built-in generic residual model or automatic tool reordering. If a plugin author can show a concrete missing hook/API seam, that narrower gap could justify a new core issue.
