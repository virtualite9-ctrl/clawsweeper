## Summary

Keep this PR open. Current main still rejects the reported Windows drive-letter Docker bind at both config validation and runtime sandbox validation, so the PR is not obsolete. The PR body includes a concrete Windows 11/Docker Desktop repro and manual verification, but the change is security-sensitive and should be revised or reviewed with Windows bind regression coverage before merge.

## Best Possible Solution

Keep this PR open for maintainer review or replace it with a maintainer-owned patch. The best fix should reuse `splitSandboxBindSpec` in config validation, update runtime source validation and allowed-root canonicalization for Windows drive paths without bypassing sandbox policy, compare Windows roots safely, add regression tests for uppercase/lowercase drives, no-mode binds, blocked paths, outside-root rejection, and Windows symlink/junction escapes, then update sandbox docs if native Windows drive-letter binds are officially supported.
