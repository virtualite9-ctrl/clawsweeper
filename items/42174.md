## Summary

Current main has not implemented this PR's requested Windows drive-letter Docker bind support. The config schema still splits `D:/...:/src:ro` at the drive-letter colon and rejects source `D`, and the runtime sandbox security check still rejects any bind source that does not start with `/`. The PR targets a real gap, but its patch still needs maintainer review and regression coverage because the review discussion identified sandbox allowlist/canonicalization risks for Windows paths.

## Best Possible Solution

Keep this PR open or replace it with a maintainer-owned fix. The fix should parse bind specs with `splitSandboxBindSpec` in config validation, update runtime source validation/allowed-root canonicalization for Windows drive paths without bypassing sandbox policy, compare drive roots case-insensitively, and add regression tests for `D:/...`, `d:/...`, no-mode binds, outside-root rejection, blocked sensitive/system paths, and Windows symlink/junction escape behavior. Update sandbox docs if native Windows drive-letter binds become supported.
