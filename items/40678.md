## Summary

Keep open. Current main still isolates `/new` into a unique TUI session and the TUI only renders chat/side-result events for the active session. There is no CLI/config/runtime observe or mirror mode, and the broader open #41548 does not clearly supersede this narrower TUI read-only visibility request.

## Best Possible Solution

Keep this open as a core TUI/Gateway UX feature. Implement an explicit `openclaw tui --observe` or `tui.mirrorChannels` mode that preserves default `/new` isolation, aggregates recent same-agent activity via existing session/history surfaces, relaxes the exact-session event filter only in observe mode, labels channel/session origin, and enforces read-only or clearly scoped interactive behavior. Add focused tests for same-agent different-session events, channel-originated final and side-result events, and read-only behavior; coordinate with #41548 only if maintainers choose to make it the canonical umbrella.
