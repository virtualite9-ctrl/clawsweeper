## Summary

Keep open. Current `main` still isolates `/new` into a fresh `tui-<uuid>` session and the TUI only loads history and renders chat events for `state.currentSessionKey`. I found no `--observe`, mirror, or cross-channel visibility surface in the current CLI/docs.

## Evidence

- **CLI surface has no observe/mirror option:** `openclaw tui` exposes `--local`, `--url`, `--token`, `--password`, `--session`, `--deliver`, `--thinking`, `--message`, `--timeout-ms`, and `--history-limit`; there is no `--observe`/`--mirror-channels` flag.
  - file: [src/cli/tui-cli.ts:10](https://github.com/openclaw/openclaw/blob/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503/src/cli/tui-cli.ts#L10)
  - command: `nl -ba src/cli/tui-cli.ts | sed -n '1,80p'`
  - sha: [b0d2003fe65b](https://github.com/openclaw/openclaw/commit/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503)
- **Docs still describe single-session TUI:** The TUI docs say sessions belong to the current agent, the session picker shows only sessions for the current agent, and the only delivery toggle is `--deliver`/`/deliver`; no observe or cross-channel mirror mode is documented.
  - file: [docs/web/tui.md:61](https://github.com/openclaw/openclaw/blob/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503/docs/web/tui.md#L61)
