## Summary

Keep open. Current main still has no explicit TUI observe or mirror mode for cross-channel activity. The TUI remains centered on one active session for CLI options, runtime options, history loading, chat events, and BTW side-result events. The broader same-author #41548 is related but still open and does not clearly supersede this narrower read-only TUI monitoring workflow.

## Best Possible Solution

Keep this open as a core TUI/Gateway UX feature. Implement an explicit `openclaw tui --observe` or `tui.mirrorChannels` mode that preserves default `/new` isolation, aggregates recent same-agent activity through gateway/session history or session event surfaces, relaxes exact-session filtering only in observe mode, labels channel/session origin, and enforces read-only or clearly scoped interactive behavior. Add focused tests for same-agent different-session events, channel-originated final and side-result events, history aggregation, and read-only behavior; coordinate with #41548 only if maintainers choose it as the canonical umbrella.
