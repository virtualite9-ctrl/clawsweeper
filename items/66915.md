## Summary

Keep open. Current `main` still emits summary-only user-facing warnings for mutating tool failures like `edit`, so the confusing surfacing reported in #66915 is not fixed on `main`. The code does add richer internal edit-mismatch context for the agent itself, and docs already show supported ways to disable `edit`, but the chat-visible warning path still omits the actionable mismatch detail by default.
