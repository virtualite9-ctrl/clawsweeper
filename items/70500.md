## Summary

Keep open. Current main documents the safer macOS app-owned TCC/PeekabooBridge model, but it does not include this PR's explicit warning that granting macOS Accessibility to a shared node runtime gives broad GUI automation reach to packages launched by that runtime. The reopened PR remains a focused docs/security improvement and is not implemented or superseded on main.

## Best Possible Solution

Keep the PR open for maintainer review, rebase if needed, and either merge it or land an equivalent docs-only change that adds the explicit Node/CLI Accessibility warning to `docs/platforms/mac/permissions.md` and links it from `docs/platforms/mac/peekaboo.md`. No core runtime change appears necessary for this PR.
