## Summary

Keep this PR open. Current main still reports `openclaw memory status` dreaming state through the deep-promotion config, so the linked light-only false-negative remains unresolved. The PR is a plausible implementation thread, but its latest provided diff now gates the phase summary on managed deep enablement, adds tests expecting light-only and REM-only configs to remain `Dreaming: off`, and bundles adjacent `--force` and timezone cleanup. That needs maintainer review rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is a narrower memory-core status change that accurately reports the effective managed sweep while also exposing configured light/deep/REM phase state clearly enough to resolve #67868. Address the deep-disabled and `limit=0` accuracy concerns, and split the unrelated `--force` and timezone cleanup unless maintainers explicitly want them in this PR.
