## Summary

Current main still does not implement the requested agent-visible pre-compaction warning. The existing compaction config and schema expose silent `memoryFlush` fields and a separate `notifyUser` user-channel notice, while the runtime memory flush path remains a silent embedded run with `NO_REPLY` guidance. A maintainer comment on the issue already kept it open on this exact distinction, and the related #14790 work is a closed-unmerged user-notification PR rather than the requested agent-visible warning config.

## Best Possible Solution

Keep #11055 open until OpenClaw either implements an explicit agent-visible pre-compaction warning path or a maintainer rejects that product direction. A concrete fix would add validated config for `memoryFlush.visibleWarning` / `warningPrompt`, inject a visible agent-facing warning at the soft threshold, preserve the existing silent `memoryFlush` and separate `notifyUser` user-channel notices, and cover the behavior in config, runtime, and docs tests.
