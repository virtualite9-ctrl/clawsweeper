## Summary

Keep #11055 open. Current main still has silent pre-compaction memory flush and optional user-channel compaction notices, but it does not implement `memoryFlush.visibleWarning`, `warningPrompt`, or an agent-visible warning turn before compaction. The related #14790 PR is closed unmerged and addressed user-facing channel notices, not this agent-facing warning request.

## Best Possible Solution

Keep #11055 open until maintainers either implement or explicitly reject an agent-visible pre-compaction warning. A concrete implementation would add validated `agents.defaults.compaction.memoryFlush.visibleWarning` and `warningPrompt` config, document it separately from `notifyUser`, inject a visible agent-facing warning before auto-compaction at the soft threshold, preserve the existing silent memory flush, and add config/runtime/docs tests for the distinction. If maintainers prefer plugin ownership, the missing core path is a supported plugin/runtime API for safe pre-threshold agent-visible injection.
