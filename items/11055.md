## Summary

Keep #11055 open. Current main still lacks `memoryFlush.visibleWarning` / `warningPrompt` config and no runtime path injects an agent-visible warning before compaction. The nearby shipped behavior is still a silent memory-flush turn plus optional user-channel `notifyUser` notices, which does not satisfy the requested agent-facing warning.

## Best Possible Solution

Keep #11055 open until OpenClaw either implements an explicit agent-visible pre-compaction warning path or maintainers reject that product direction. A concrete path would add validated `agents.defaults.compaction.memoryFlush.visibleWarning` and `warningPrompt` config, inject a visible agent-facing warning at the soft threshold before auto-compaction, preserve the existing silent memory-flush run and separate `notifyUser` channel notices, and add config/runtime/docs tests for the distinction.
