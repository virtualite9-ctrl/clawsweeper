## Summary

Keep open. Current main still has the reported display-layer path: hook-mutated bootstrap files are only sanitized for a valid path, `buildBootstrapInjectionStats` still copies `file.name` directly into the report, and `/context list` renders that field directly. A hook-injected virtual file with `path` and content but no `name` can therefore still render as literal `undefined` while retaining nonzero raw/injected counts. The related PR #48021 is open/unmerged, so the fix has not landed on main.

## Best Possible Solution

Keep #47941 open until #48021 or an equivalent main-branch fix lands. The fix should normalize bootstrap report labels with a deterministic fallback such as non-empty `name`, non-empty `path` basename/path, then a virtual-file placeholder, and add regression coverage for hook-injected `bootstrapFiles` entries that provide `path` and content but no `name`.
