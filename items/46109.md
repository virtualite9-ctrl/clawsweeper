## Summary

Keep open. Current main has substantial SecretRef runtime and CLI support, and the related round-trip/redaction class is covered, but the central requested Control UI UX is still missing: structured env SecretRefs render as a blank read-only form field with a generic SecretRef placeholder, not a source-aware masked value with a `From env` badge, env var name, resolution state, or store/reference/unset editor.

## Best Possible Solution

Keep the issue open as a core Control UI/config UX feature. The best path is to extend the config form’s SecretInput handling so structured SecretRefs render masked but source-aware (`env`, provider, id, resolution state), support switching between plaintext string, SecretRef, and unset where the schema allows it, and add focused tests/docs around representative paths such as `gateway.auth.token`, `channels.discord.token`, provider API keys, web search keys, and plugin/skill API keys.
