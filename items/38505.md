## Summary

Keep #38505 open. Current main still handles Discord inbound conversations through message-create events only; there is no messageUpdate listener or edit-aware dispatch path, and the existing replay guard is keyed by account/channel/message id, so a naive same-message-id edit replay would be suppressed. Related #53654 overlaps but is broader and still open, so it should be coordinated with rather than used to close this focused issue.

## Best Possible Solution

Keep the issue open and implement the missing behavior in the bundled Discord plugin: add the appropriate Carbon messageUpdate listener after confirming the pinned @buape/carbon event shape, ignore non-content-only updates, route qualifying edited messages through the existing preflight and inbound-worker path with edited content as authoritative, add edit metadata where useful, make replay/dedupe edit-aware, and cover mention-added edits, DMs or active threads, non-content update suppression, and duplicate/update replay behavior. Coordinate with #53654 if edit and delete handling are implemented together.
