## Summary

Keep #38505 open. Current main still wires Discord inbound conversation handling through `MessageCreateListener` only; there is no `MessageUpdateListener`, no content-change filter, no edited-message metadata path, and the replay key would drop a naive same-message-id re-dispatch.

## Best Possible Solution

Keep the issue open and implement the missing behavior in the bundled Discord plugin: add a Carbon `MessageUpdateListener`, ignore non-content-only updates, route qualifying edited messages through the existing preflight/inbound worker path with the edited content as authoritative, make dedupe/replay edit-aware, add useful edit metadata, and cover mention-added edits, DMs or active threads, duplicate/update replay behavior, and non-content update suppression. Coordinate with open #53654 so the edit portion is not implemented twice.
