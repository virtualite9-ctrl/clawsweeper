## Summary

Keep #38505 open. Current main 67b9167b808f2ff065c91665d6f36256a5c23ef2 still handles Discord inbound conversation triggers through a MessageCreate listener only; there is no registered MessageUpdate listener, content-change filter, edit metadata path, or edited-message re-dispatch path. Related #53654 is open and broader, but it does not obsolete this earlier focused edit-to-reprocess request while the behavior remains missing.

## Best Possible Solution

Keep the issue open and implement the missing behavior in the bundled Discord plugin: add a Carbon MessageUpdateListener, ignore non-content-only updates, route qualifying edited messages through the existing Discord preflight/inbound worker path with edited content as authoritative, adjust replay/dedupe so same-message-id edits are not dropped, include edit metadata where useful, and add focused tests for mention-added edits, DMs or active threads, duplicate/update replay behavior, and non-content update suppression. Coordinate with #53654 so the edit portion is not implemented twice.
