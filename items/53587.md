## Summary

Current `main` already routes unresolved model refs through an explicit `Unknown model: <provider>/<model>` path and surfaces that message at agent startup/compaction boundaries. The same implementation is present in shipped release `v2026.4.22`, so the reported `entry.factory is not a function` failure mode is obsolete.

## Close Comment

Closing as implemented on current `main` after a Codex review. Invalid model refs now go through the explicit unknown-model path in `src/agents/pi-embedded-runner/model.ts:739` and `src/agents/pi-embedded-runner/model.ts:857`, and the caller surfaces that message in `src/agents/pi-embedded-runner/run.ts:382` and `src/agents/pi-embedded-runner/compact.ts:354` instead of falling through to an internal factory call. OpenRouter-style nested refs are also still parsed correctly (`src/agents/model-selection.test.ts:277`). This behavior is already present in shipped release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`). If you still hit `entry.factory is not a function` on `v2026.4.22` or newer, please open a fresh issue with the exact stack trace and current config snippet.
