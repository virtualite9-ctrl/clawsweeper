## Summary

The PR's claimed breakage does not match current main's Gmail push auth flow. On current main, the Tailscale public endpoint intentionally carries the Gmail push token for the Pub/Sub -> gog hop; removing it without adding OIDC or another auth path would break the default setup, and the PR author later acknowledged that larger rework would be required.

## Evidence

- **Public Tailscale endpoint intentionally includes the push token:** Current main returns `${baseUrl}?token=${params.token}` from `ensureTailscaleEndpoint`, so the generated Gmail Pub/Sub push URL is still designed to carry the shared push token.

  - file: [src/hooks/gmail.ts:238](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/src/hooks/gmail.ts#L238)
