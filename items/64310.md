## Summary

Keep this PR open. Current main still has the mutable module-global `workspacePackagePaths`, `collectWorkspacePackagePaths()` still appends extension package paths into that shared array, and `main()` still calls the collector before `buildMissingPackages()` reads the global list. The PR is a narrow, plausible fix that has not landed or been superseded on main.

## Best Possible Solution

Review and land a focused version of this PR rather than close it. The best path is to make `collectWorkspacePackagePaths()` build and return a fresh deterministic package-path list, pass that list into `buildMissingPackages()`, address the redundant-sort review note, and add a small regression test in `test/scripts/audit-seams.test.ts` that exercises repeated in-process `main()` calls and verifies stable, non-duplicated `missingPackages` output.
