## Summary

Keep this PR open. Current main still has the mutable module-global workspacePackagePaths, repeated collectWorkspacePackagePaths() calls still append extension package paths into that shared array, and buildMissingPackages() still consumes that global list. The PR is narrow, not merged, not superseded, and the only remaining maintainer work is review polish plus regression coverage.

## Best Possible Solution

Review and land a focused version of this PR rather than close it. The maintainer path is to keep collectWorkspacePackagePaths() side-effect-free, pass its fresh deterministic package-path list into buildMissingPackages(), address the redundant-sort note, and add a small regression test in test/scripts/audit-seams.test.ts that calls main() twice in-process and verifies missingPackages does not duplicate entries.
