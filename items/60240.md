## Summary

Current `main` already routes generated-image media from tool results into assistant attachments, including `details.media.mediaUrls` without requiring `content` image blocks, so the reported behavior is implemented on `main`.

## Evidence

- **`image_generate` now emits attachment directives plus structured media:** The tool result text includes one `MEDIA:<path>` line per saved image, and `details.media.mediaUrls`/`details.paths` still carry the saved files. That gives the reply pipeline concrete attachment paths even when `content` is text-only.
