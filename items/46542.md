## Summary

Keep PR #46542 open for maintainer judgment. Current main still lacks the proposed shared memory store, `sharedPaths`, `_shared` reserved agent/store, and `openclaw memory ... --shared` CLI surface; the related duplication problem remains open in #46558. The PR likely needs a rewrite against the current `memory-core` plugin layout and its review thread still raises design/security concerns, but that is not enough for an automated close because the underlying product work remains real and unsolved.

## Best Possible Solution

Keep the item open or have a maintainer explicitly supersede it with #46558. The best implementation path is a current-architecture rewrite in `extensions/memory-core`/`packages/memory-host-sdk`: define a safe shared-index model, decide how it interacts with builtin versus QMD memory, add config schema/docs, add `status`/`index`/`search` CLI support if still desired, and cover cross-agent access, path validation, deduplication, and status counts with tests.
