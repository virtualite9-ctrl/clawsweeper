## Summary

Keep open. Current `main` at `cb2c36b04975e34ab7014a11575d6589aa96ae32` still matches the report: Debug snapshot refresh returns silently when disconnected, and the Snapshots card still has no inline loading or disconnected-state indicator beyond the button label/disabled state.

## Evidence

- **Silent early return when disconnected:** `loadDebug` exits immediately when `!state.client || !state.connected`, before setting any loading state or user-visible error.
  - file: [ui/src/ui/controllers/debug.ts:18](https://github.com/openclaw/openclaw/blob/cb2c36b04975e34ab7014a11575d6589aa96ae32/ui/src/ui/controllers/debug.ts#L18)
  - command: `nl -ba ui/src/ui/controllers/debug.ts | sed -n '1,220p'`
  - sha: [cb2c36b04975](https://github.com/openclaw/openclaw/commit/cb2c36b04975e34ab7014a11575d6589aa96ae32)
- **No inline loading or disconnected message in Snapshots card:** The Snapshots card only changes the button text to `Refreshing…` and disables the button. The body always renders JSON blocks; there is no spinner, skeleton, progress UI, or disconnected callout.
  - file: [ui/src/ui/views/debug.ts:45](https://github.com/openclaw/openclaw/blob/cb2c36b04975e34ab7014a11575d6589aa96ae32/ui/src/ui/views/debug.ts#L45)
