## Summary

Current main still matches the reported Control UI bug: disconnected Debug snapshot refresh returns without setting any visible state, the Snapshots card only changes the Refresh button label while loading, and the Debug view is not passed connection state needed to show an inline disconnected reason. Related PR #39806 is still open and unmerged, so it remains a useful fix attempt rather than evidence that this issue is obsolete.

## Best Possible Solution

Keep the issue open. The maintainable fix belongs in the Control UI Debug surface: pass connection state or a refresh-disabled reason into renderDebug(), show an inline loading/progress state inside the Snapshots card while debugLoading is true, show a disconnected callout or disabled reason when refresh cannot run, and add focused tests for loadDebug() and renderDebug(). Review or refresh PR #39806 as the existing implementation attempt.
