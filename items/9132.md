## Summary

Keep #9132 open. Current main has lifecycle status data in the gateway/session row model and preserves it in Control UI state, but the Control UI Sessions table still does not render an active/completed/timed-out status indicator or status-based sort/filter. The linked #46792 is closed and appears to be the duplicate/superseded sibling, not a canonical open tracker for the remaining UI work.

## Best Possible Solution

Keep #9132 open until the Control UI Sessions view renders the existing lifecycle fields. The likely implementation belongs in ui/src/ui/views/sessions.ts as a Status column or badge mapping running/done/failed/killed/timeout to clear user labels, with last activity/runtime detail from updatedAt, startedAt, endedAt, or runtimeMs. Add focused coverage in ui/src/ui/views/sessions.test.ts. If maintainers want the same visibility while switching chat sessions, ui/src/ui/chat/session-controls.ts can be a separate scoped follow-up.
