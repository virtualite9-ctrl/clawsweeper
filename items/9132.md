## Summary

Keep #9132 open. Current main exposes lifecycle fields for session rows, including running/done/failed/killed/timeout state, but the Control UI Sessions table still does not render an active/completed/timed-out status indicator or support lifecycle status sorting/filtering. The related #46792 is closed and the existing Codex review comment also concluded this remains unimplemented.

## Best Possible Solution

Keep the issue open and implement the lifecycle indicator in the Control UI Sessions view. A focused fix should add a compact Status column or badge in ui/src/ui/views/sessions.ts using GatewaySessionRow.status, subagentRunState, hasActiveSubagentRun, startedAt, endedAt, and runtimeMs; wire status sorting/filtering if included in scope; and add rendering coverage in ui/src/ui/views/sessions.test.ts.
