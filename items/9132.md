## Summary

Keep #9132 open. Current main still carries lifecycle data for sessions, including running/done/failed/killed/timeout, but the Control UI Sessions table does not render a status badge/column or support lifecycle-based sorting/filtering. The related #46792 is closed and the existing maintainer/Codex discussion treats #9132 as the remaining tracker.

## Best Possible Solution

Keep #9132 open and implement this in the Control UI Sessions view. Add a Status column or compact badge in ui/src/ui/views/sessions.ts that maps running/done/failed/killed/timeout plus subagentRunState/hasActiveSubagentRun to clear labels, surface last activity/runtime from updatedAt/startedAt/endedAt/runtimeMs, and add focused rendering coverage in ui/src/ui/views/sessions.test.ts. Status sorting/filtering can be included here or tracked as a narrow follow-up if maintainers want a smaller first patch.
