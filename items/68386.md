## Summary

Current main, shipped in v2026.4.24, fixes the observable bug: heartbeat exec-event prompts now include the actual async exec completion payload, avoid the old phantom “system messages above” wording, and consume inspected events after handling. The implementation still uses an isolated `:heartbeat` session when configured, but that is the documented heartbeat relay design rather than the remaining broken fallback behavior reported here.

## Best Possible Solution

Keep the shipped v2026.4.24 heartbeat exec-event payload fix. Leave broader isolated-heartbeat/session-audit work to #65161 or a new focused product decision if maintainers want to replace heartbeat relay with main-session continuation.

## Risks / Open Questions

- The original report frames the behavior as a routing bug because the model turn still runs in the isolated heartbeat session. Current docs and code treat that as the intended heartbeat relay path; if maintainers want async exec completions to resume the initiating chat session instead, that should be tracked as a separate product/architecture change. Broader isolated-heartbeat cleanup is still open in #65161.
