## Summary

Current main still runs the global npm package update from the active OpenClaw process for both CLI `openclaw update` and gateway `update.run`. The reported Windows `EBUSY` self-update lock is not fixed on main; the related detached-helper PR remains open and unmerged.

## Best Possible Solution

Keep this issue open until OpenClaw ships a Windows-safe global npm update flow. The likely fix is to revise or land #44614 or an equivalent implementation that hands package self-update to a detached Windows helper or otherwise stops/releases the gateway and active CLI before `npm i -g` renames the global package root, then add regression coverage for `EBUSY`/rename output and gateway `update.run` behavior plus a Windows smoke/manual verification note.
