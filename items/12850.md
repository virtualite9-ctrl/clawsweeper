## Summary

Keep open. Current main still flattens Mattermost inbound system-event previews with `bodyText.replace(/\s+/g, " ")`, while the full inbound body is preserved separately. I found no Mattermost config/schema/type/docs/test surface for the requested opt-in faithful/newline-preserving preview mode.

## Best Possible Solution

Keep the issue open. Add a Mattermost plugin setting, likely top-level plus per-account, that defaults to the current compact one-line preview but can opt into a faithful preview helper preserving `\n`, normalizing spaces/tabs within lines, and capping by max lines/chars. Document it on https://docs.openclaw.ai/channels/mattermost and add focused tests for default flattening plus opt-in newline-preserving system-event previews.
