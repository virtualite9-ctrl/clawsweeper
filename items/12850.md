## Summary

Keep open. Current main still flattens Mattermost inbound system-event previews with `bodyText.replace(/\s+/g, " ").slice(0, 160)`, while preserving the full inbound body separately. I found no Mattermost config, docs, or tests for an opt-in newline-preserving preview mode, and the related Markdown newline work is adjacent but does not implement this request.

## Best Possible Solution

Keep the issue open. Add a Mattermost plugin setting, likely top-level plus per-account, that defaults to the current compact one-line preview but can opt into a faithful preview helper preserving `\n`, normalizing spaces/tabs within lines, and capping by max lines/chars. Document it on the public Mattermost channel docs and add focused coverage for default flattening and opt-in newline-preserving system-event previews.
