## Summary

Current `main` already defers transient lifecycle errors instead of immediately finalizing chat/run state, and the same regression coverage is present in released `v2026.4.23`.

## Evidence

- **Gateway chat finalization now waits out transient lifecycle errors:** `server-chat` keeps a `pendingTerminalLifecycleErrors` timer, cancels it on later non-error events, and only finalizes chat/error cleanup after the retry grace window. That preserves the chat run mapping instead of tearing it down on the first recoverable attempt failure.
