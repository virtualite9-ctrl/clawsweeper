## Summary

Keep #59616 open. Current main still only starts chokidar on the single root config `watchPath`; it does not import or call the existing recursive `$include` scanner, and the config-reload tests do not assert `watcher.add()`/`watcher.unwatch()` behavior for include-owned files. The related implementation PR #59632 was closed unmerged, so there is no shipped implementation on main for the requested hot-reload behavior.

## Best Possible Solution

Implement the issue in `src/gateway/config-reload.ts` by collecting `$include` paths from each successful config snapshot, keeping a watched include-path set in sync with chokidar `add()`/`unwatch()`, and adding focused tests for initial include watching, added/removed include paths after reload, nested includes, and best-effort scanner failures. The existing `src/config/includes-scan.ts` helper is the natural starting point, but the implementation should respect the current snapshot shape and root-config write notification behavior.
