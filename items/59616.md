## Summary

Keep open. Current `main` still only watches the root config file for gateway hot-reload, does not sync `$include`-referenced paths into the watcher, and the related fix PR `#59632` was closed unmerged due conflicts with later refactors.

## Evidence

- **Gateway reloader still watches one path:** `startGatewayConfigReloader` still creates a single chokidar watcher from `opts.watchPath`, subscribes only to that watcher, and filters in-process writes to `event.configPath === opts.watchPath`. There is no include-path diffing or `watcher.add()` / `watcher.unwatch()` path management on current `main`.
  - file: [src/gateway/config-reload.ts:382](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/src/gateway/config-reload.ts#L382)
  - command: `nl -ba src/gateway/config-reload.ts | sed -n '378,410p'`
  - sha: [ec3dbd22a4de](https://github.com/openclaw/openclaw/commit/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46)
- **Gateway startup passes only the root config path:** Gateway startup wires `watchPath` from `configSnapshot.path`, so the reloader is initialized with the main config file path rather than an include-expanded watch set.
  - file: [src/gateway/server.impl.ts:862](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/src/gateway/server.impl.ts#L862)

## Risks / Open Questions

- Users with modular configs still appear to need a manual restart after editing an included file, because include-only edits do not hit the active watcher path.
- Issue `#70524` cross-links this item as another hot-reload gap, so closing this would likely hide an active underlying problem rather than confirmed cleanup.
- Docs currently say gateway reload watches the active config file path in `docs/gateway/index.md:66`, while `docs/gateway/configuration.md:551` discusses reload planning for `$include` source layout; that mismatch increases the chance of user confusion until behavior is fixed or documented more explicitly.
