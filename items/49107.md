## Summary

Keep open. Current `main` does not implement this draft PR’s non-interactive onboard support for `trusted-proxy`, `allowUsers`, or Control UI allowed-origin flags. The capability exists only in the interactive gateway configuration flow, so the PR’s requested automation path remains unmerged and still materially useful.
