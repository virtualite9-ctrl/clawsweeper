## Summary

Keep open. Current `main` still lacks the non-interactive onboarding support this draft adds: the onboard CLI only exposes token/password gateway auth, the non-interactive gateway config path rejects `trusted-proxy`, and the onboarding docs only cover token-mode automation. At the same time, interactive gateway setup and the shared auth builder already support `trusted-proxy` plus `allowUsers`, so this PR still addresses a real gap rather than duplicating shipped behavior.
