## Summary

Keep open. Current main has exec policy, explicit allowlists, narrow stdin-only safeBins, and plugin hooks that can add optional policy, but it does not implement the requested built-in L1-L4 command risk classifier with broad read-only auto-approval and dangerous-command auto-deny.

## Best Possible Solution

Keep the issue open for maintainer decision. If pursued, design it as an explicit, opt-in exec policy layer or a carefully scoped extension of safeBins/allowlists with threat-model docs and tests. If maintainers prefer plugin-first heuristics, close only after documenting the supported ClawHub/plugin path and confirming no missing plugin API remains.

## Evidence

The issue has no comments, protected labels, or maintainer-authored signal changing that scope.
