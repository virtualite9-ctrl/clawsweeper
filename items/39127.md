## Summary

Keep #39127 open. Current main at 631552c554095b0c520376db9131dac0f5427f94 still does not expose the requested gateway-owned `sessions.activity.get/list` RPCs or `sessions.activity.statechange` WebSocket event. The nearby status surfaces are either channel-account scoped, session-list display metadata, task/background-work records, or client-local event handling, not the explicit per-session runtimeActivity/attention/stale-run/children contract requested here. The related implementation PRs #39438 and #41399 remain useful prior art, but the provided GitHub context shows both were closed unmerged, so they do not supersede the issue.

## Best Possible Solution

Keep #39127 open as the canonical core Gateway API item. The best path is an additive protocol implementation for `sessions.activity.get`, `sessions.activity.list`, and `sessions.activity.statechange`, with exported schemas/types, read-scope authorization, gateway handlers, a gateway-owned run-id based activity tracker, stale-run cleanup, explicit attention transitions including `awaiting_user`, `awaiting_subagent`, and `awaiting_approval`, bounded child rollups across agent stores, reconnect/list resync semantics, docs, and focused tests. Use closed PRs #39438 and #41399 as prior art, but do not treat them as shipped or superseding.
