## Summary

Keep #39127 open. Current main at 4a3030df9efa72c44bab567a4390ac0c4876a73f still does not expose `sessions.activity.get`, `sessions.activity.list`, or a `sessions.activity.statechange` WebSocket event. Adjacent surfaces provide channel-level busy fields, session row display metadata, and `sessions.changed` events, but not the requested gateway-owned per-session runtimeActivity/attention/stale-run/children contract. The related implementation PRs #39438 and #41399 are useful prior art, but the provided GitHub context shows both were closed unmerged, so they do not supersede the issue.

## Best Possible Solution

Keep #39127 open as the canonical core Gateway API item. The best path is an additive protocol implementation for `sessions.activity.get`, `sessions.activity.list`, and `sessions.activity.statechange`, with exported schemas/types, read-scope authorization, gateway handlers, a run-id based activity tracker, stale-run cleanup, explicit attention transitions including `awaiting_user`, `awaiting_subagent`, and `awaiting_approval`, bounded child rollups across agent stores, reconnect/list resync semantics, public docs, and focused tests. Use closed PRs #39438 and #41399 as prior art, but do not treat them as shipped or superseding.
