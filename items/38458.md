## Summary

Keep open. Current main has partial controls that reduce some visible tool/progress output, but it does not implement the requested per-channel ability to suppress tool error output from chat surfaces while still returning the error to the agent. The related #71282 TTS leak was narrower and does not supersede this broader tool-error visibility request.

## Evidence

- **Global suppress option is narrower than requested:** Current config exposes `messages.suppressToolErrors`, documented as suppressing tool-error warnings globally, not per channel or as a general tool-output visibility mode.
  - file: [src/config/types.messages.ts:121](https://github.com/openclaw/openclaw/blob/81a41fe5bee350c6a035bfcc458eb8be380afe2c/src/config/types.messages.ts#L121)
  - command: `nl -ba src/config/types.messages.ts | sed -n '116,124p'`
  - sha: [81a41fe5bee3](https://github.com/openclaw/openclaw/commit/81a41fe5bee350c6a035bfcc458eb8be380afe2c)
- **Mutating tool errors still surface:** The warning policy returns `showWarning: true` for mutating tool errors before checking `messages.suppressToolErrors`, so the option does not suppress all tool errors. The issue's example involves the `message` tool, which is a mutating delivery action.
  - file: [src/agents/pi-embedded-runner/run/payloads.ts:107](https://github.com/openclaw/openclaw/blob/81a41fe5bee350c6a035bfcc458eb8be380afe2c/src/agents/pi-embedded-runner/run/payloads.ts#L107)

- **Tests assert the remaining behavior:** Existing tests explicitly expect mutating tool errors to still be shown when `messages.suppressToolErrors` is enabled, while only non-mutating errors are suppressed by that setting.
  - file: [src/agents/pi-embedded-runner/run/payloads.errors.test.ts:286](https://github.com/openclaw/openclaw/blob/81a41fe5bee350c6a035bfcc458eb8be380afe2c/src/agents/pi-embedded-runner/run/payloads.errors.test.ts#L286)
