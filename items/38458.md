## Summary

Keep open. Current main still lacks the requested per-channel or broader chat-surface tool-output visibility policy. The existing global `messages.suppressToolErrors` path only suppresses non-mutating tool warnings; mutating failures such as the issue's `message send` example still generate user-facing error payloads, and Telegram delivery treats those as deliverable error replies rather than suppressing them.

## Best Possible Solution

Keep the issue open for a core runtime/config design: add a documented tool-output visibility policy with global defaults and per-channel overrides, apply it before chat-surface delivery while keeping tool errors available to the agent, update schema/docs, and add focused tests for mutating `message` and `exec` failures across Telegram/Discord/Slack streaming paths. Coordinate with #51091 without treating that narrower PR as a full replacement.
