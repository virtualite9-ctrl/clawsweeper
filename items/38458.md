## Summary

Keep open. Current main still lacks the requested per-channel or broader tool-output visibility policy. The existing global `messages.suppressToolErrors` remains intentionally limited to non-mutating tool-failure warnings, while the reported `message send` failure class is classified as mutating and still produces user-visible warning payloads. Related #51091 is still open/unmerged and would only change the global suppression behavior, not add the per-channel or broader visibility policy requested here.

## Best Possible Solution

Keep the issue open for a core runtime/config design: define an explicit tool-output visibility policy with global defaults and per-channel overrides, apply it in embedded-run payload shaping and streaming chat delivery, keep tool errors available to the agent, update schema/docs, and add focused tests for mutating `message`/`exec` failures plus Telegram/Discord streaming delivery.
