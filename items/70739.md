## Summary

Keep PR #70739 open. Current main still lacks a downstream SSE heartbeat for `/v1/chat/completions` and `/v1/responses`; both streaming paths set SSE headers and then only write real model/lifecycle events. The only current SSE keepalive precedent is scoped to session history. The PR is therefore not obsolete or implemented on main, though its remaining review comments should be handled before merge.

## Best Possible Solution

Keep the PR open and ask for the remaining review fixups, then validate and merge this PR or an equivalent implementation that adds a hardened SSE comment heartbeat to both `/v1/chat/completions` and `/v1/responses` streaming paths without changing their event schema.
