## Summary

Current main already implements the central PR request: isolated cron agent-turn timeouts now start from the effective execution-start boundary instead of enqueue time, and the shipped implementation also addresses the earlier CLI-provider timeout-bypass review by wiring the callback into both embedded and CLI execution paths. The fix is on main in 729147dcb523a06badee454897fdc201468c4e0c and is listed under Unreleased, not in a tagged release I could verify.

## Best Possible Solution

Close this PR as implemented on main. Keep the shipped implementation in 729147dcb523a06badee454897fdc201468c4e0c, and use separate focused follow-ups for broader cron timeout policy or the distinct per-attempt AbortController work referenced by related PR #42482.
