## Summary

Keep this issue open. Current main still does not discover, warn about, or repair the reported layout where a complete legacy-named `moltbot.json` sits inside `~/.openclaw/` beside a missing or skeletal canonical `openclaw.json`. Later recovery work improves clobbered-config handling when a backup/baseline exists, and #54201 covers the misleading gateway-mode error symptom, but neither closes this migration gap.

## Best Possible Solution

Keep this open for a targeted config-recovery fix. Add a startup and/or `openclaw doctor` diagnostic for `~/.openclaw/moltbot.json` when canonical `openclaw.json` is missing, empty, or skeletal; offer a safe `doctor --fix` copy/merge path when feasible; and update the public migration/troubleshooting docs. Do not reintroduce broad legacy runtime config probing.
