## Summary

Keep this open. Current main still does not discover, warn about, or repair a complete legacy-named `$HOME/.openclaw/moltbot.json` sitting beside a missing or skeletal canonical config. The related #54201 startup-message work and later last-known-good recovery improve symptoms, but they do not cover this first-upgrade sibling-file migration gap.

## Best Possible Solution

Keep this open for a targeted config recovery fix. Add startup and/or `openclaw doctor` detection for `$HOME/.openclaw/moltbot.json` when canonical `openclaw.json` is missing, empty, or skeletal; offer a safe doctor repair path when feasible; and update the public migration/troubleshooting docs. Preserve the intentional removal of broad legacy runtime config probing.
