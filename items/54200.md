## Summary

Keep open. Current main improves the related `gateway.mode` crash-loop diagnostic and can restore some clobbered configs from last-known-good backups, but it still does not handle the specific `~/.openclaw/moltbot.json` upgrade gap. Config resolution and Doctor migration only consider `openclaw.json` plus `clawdbot.json`, and the changelog still documents removal of `moltbot.json` fallback without a repair path for a `moltbot.json` already sitting under `~/.openclaw/`.

## Best Possible Solution

Keep this issue open for a narrow maintainer fix. The best path is not to revive broad load-time legacy auto-detection; instead add an explicit Doctor/startup diagnostic for `~/.openclaw/moltbot.json` when canonical `openclaw.json` is missing or skeletal, offer a safe `doctor --fix` copy/merge path when feasible, and update the public migration/changelog wording so users with `moltbot.json` already under `~/.openclaw/` know exactly what to do.
