## Summary

Keep PR #69319 open. Current main still has `skills.install` and `skills.update` but no registered `skills.uninstall` Gateway method, no uninstall schema/validator/export, no backend `uninstallSkillFromClawHub`, no docs entry, and no tests for the requested uninstall path. The linked feature request #69299 being closed does not make this PR obsolete because the observable Gateway behavior remains unimplemented on current main. The PR is still useful, but the existing review comments identify changes that should be addressed before merge.

## Best Possible Solution

Keep the PR open and request targeted fixes. The best path is an additive `skills.uninstall` Gateway method with schema/validator/type exports, method registration, `operator.admin` scope, docs, generated Swift model updates, and backend removal that is idempotent, path-safe, limited to ClawHub-tracked installs, cleans `.clawhub/lock.json`, preserves shared dependencies, and has backend plus Gateway handler tests for installed, absent, malformed, traversal, stale-lockfile, and non-ClawHub-collision cases.
