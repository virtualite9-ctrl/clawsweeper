## Summary

Keep #69319 open. Current main still does not register or implement `skills.uninstall`; the gateway registry, scope map, protocol schemas/exports, handler, backend ClawHub skill helper, tests, and gateway docs only cover the existing status/search/detail/bins/install/update skill methods. The PR addresses a real remaining RPC parity gap from #69299, but the provided review discussion also identifies branch issues that should be fixed before merge.

## Best Possible Solution

Keep the PR open and request changes. The best implementation should add a `skills.uninstall` protocol schema/validator/type, register it in the gateway method list and `operator.admin` scope, implement a handler that resolves the default agent workspace, and add a backend ClawHub uninstall helper that removes only ClawHub-tracked skill folders and lock entries. It should reject malformed slugs as `INVALID_REQUEST`, treat missing tracked installs idempotently, preserve shared bin dependencies, update public gateway docs and generated Swift models, and include tests for installed removal, idempotency, lockfile-only cleanup, path traversal, malformed slugs, untracked/manual skill directory collisions, and legacy tracked slugs.
