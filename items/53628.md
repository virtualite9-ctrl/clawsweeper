## Summary

Keep open. Current main still does not expand $VAR or ${VAR} placeholders in the shared user-path resolver, and an open implementation PR with GitHub closing syntax is still pending for the same bug.

## Best Possible Solution

Keep this issue open. Review and either land or close PR #58281, folding in any useful coverage from #53718, with regression tests for ${XDG_CONFIG_HOME} and $XDG_CONFIG_HOME through the shared path resolver and a skill-install path that no longer creates literal placeholder directories. After the selected PR merges, close this as implemented.
