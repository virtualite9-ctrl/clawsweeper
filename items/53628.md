## Summary

Keep this issue open. Current main still does not provide raw `$VAR`/`${VAR}` expansion in the shared user-path resolver, the reported ClawHub skill-install path is still plausible through workspace fallback behavior, and the provided GitHub context shows an open unmerged implementation PR with closing syntax for this issue.

## Best Possible Solution

Keep this issue open. Consolidate review around PR #58281 or #53718, verify the reported Docker/ClawHub CLI workspace-fallback path, and add regression coverage in `src/infra/home-dir.test.ts` or the relevant ClawHub/workspace path tests so `${XDG_CONFIG_HOME}` cannot create a literal directory. After the chosen PR merges, close this issue as implemented.
