## Summary

Keep open. Current main still treats `${XDG_CONFIG_HOME}` as a literal path segment in the shared user-path resolver, and there is an open implementation PR that uses closing syntax for this issue, so the issue should remain open until that PR is reviewed and either merged or closed.

## Best Possible Solution

Keep this issue open. Review and either land or close PR #58281, reconciling the duplicate open PR #53718 as needed. The likely code path is a shared env-placeholder expansion helper in `src/infra/home-dir.ts`, covered by regression tests in `src/infra/home-dir.test.ts`, so configured workspace paths used by ClawHub skill installs expand `${XDG_CONFIG_HOME}` before filesystem writes.
