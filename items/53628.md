## Summary

Keep this issue open. Current main still does not expand `$VAR` or `${VAR}` placeholders in the shared user-path resolver, and the provided GitHub context shows an open PR with closing syntax still targets the same bug.

## Best Possible Solution

Keep this issue open. Review and either land or close PR #58281, consolidating any useful coverage from #53718, with regression tests for `${XDG_CONFIG_HOME}` and `$XDG_CONFIG_HOME` in the shared path resolver and a check that ClawHub skill installs no longer create literal placeholder directories. After the selected PR merges, close this issue as implemented.
