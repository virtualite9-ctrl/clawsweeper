## Summary

Current `main` still does not expand `${VAR}`/`$VAR` in user path resolution, so the reported `${XDG_CONFIG_HOME}` skill-install path bug is not implemented on `main` and remains plausibly reproducible.

## Evidence

- **Skill install uses resolved active workspace:** `openclaw skills install` resolves the active workspace and logs the resulting target dir from that path, so any literal `${XDG_CONFIG_HOME}` in workspace config flows into the install destination.
  - file: [src/cli/skills-cli.ts:101](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/cli/skills-cli.ts#L101)
  - command: `nl -ba src/cli/skills-cli.ts | sed -n '93,117p'`
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Workspace config path goes through resolveUserPath:** Configured agent/default workspace values are passed through `resolveUserPath(configured, env)`. There is no separate env-placeholder expansion at this callsite.
  - file: [src/agents/agent-scope-config.ts:155](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/agents/agent-scope-config.ts#L155)

  - file: [src/utils.ts:125](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/utils.ts#L125)

- **Current main does not expand env placeholders:** `resolveHomeRelativePath` on current `main` trims input, expands `~`, and otherwise returns `path.resolve(trimmed)`. There is no `${VAR}` or `$VAR` expansion logic, so `${XDG_CONFIG_HOME}/workspace` would remain literal.
  - file: [src/infra/home-dir.ts:102](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/infra/home-dir.ts#L102)

  - file: [src/utils.test.ts:101](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/utils.test.ts#L101)
