## Summary

Current `main` does not implement this PR's `sessionChannelId` auto-join routing. The Discord voice config type and Zod schema still allow only `guildId` + `channelId`, `autoJoin()` still forwards only those two fields into `join()`, and `processSegment()` still forces `deliver: false`, so routed text-channel transcript/reply delivery is not present on `main`.
