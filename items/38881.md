## Summary

Keep PR #38881 open. Current main still does not auto-switch the final agent reply run from a text-only selected/default model to `agents.defaults.imageModel` when image input is present. Main currently routes `imageModel` through media-understanding/image-tool paths, then the final agent run still receives the selected/default `provider` and `model`. The PR targets a real documented/user-facing gap, but the patch needs maintainer/author follow-up before merge because review comments identify incomplete image detection and sequencing issues.

## Best Possible Solution

Keep #38881 open for a focused production fix in the reply/model-selection path. The implementation should detect actual inbound image inputs after normalization, including inline webchat images, offloaded `media://` refs, `MediaPath(s)`, `MediaUrl(s)`, and image MIME/type data; resolve `agents.defaults.imageModel.primary` and fallbacks through the real alias index/model catalog; switch the final `runPreparedReply` model only when the active model is positively known not to support images; respect heartbeat/session/channel overrides; coordinate with media understanding so the turn does not do duplicate image analysis; and add regression coverage proving the final agent run receives the configured vision model.
