## Summary

Keep PR #38881 open. Current main still does not auto-switch the final agent reply run from a text-only selected/default model to `agents.defaults.imageModel` when image input is present. The PR targets a real gap, but its patch still needs maintainer-quality follow-up for the review concerns around image detection, sequencing, capability certainty, URL-only media, and fallback-only imageModel config.

## Best Possible Solution

Keep #38881 open for a focused production fix in the reply/model-selection path. The implementation should detect real inbound image inputs after normalization, resolve `agents.defaults.imageModel.primary` and fallbacks through the real alias index/catalog, switch the final `runPreparedReply` model only when the active model is positively known not to support images, respect heartbeat/session/channel overrides, avoid duplicate media-understanding work, and add regression coverage proving the final agent run receives the configured vision model.
