## Summary

Current main already strips raw tool-call XML from user-visible assistant text before WhatsApp auto-reply delivery through the shared assistant-visible extraction path, with broader sanitizer coverage than this WhatsApp-local PR. The PR body is an unfilled template, and the only comment confirms the patch shape but notes a PR-local logging mismatch; the underlying user-visible problem is already covered by main and release notes.

## Best Possible Solution

Close this PR as obsolete and keep the shared assistant-visible sanitizer as the canonical fix. If a future report shows a non-agent WhatsApp path can still inject raw internal XML directly into `deliverWebReply`, add a generic delivery-boundary sanitizer with focused reproduction rather than merging this stale WhatsApp-only patch.
