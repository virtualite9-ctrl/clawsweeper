## Summary

The reported packaged-install regression is real in `v2026.4.23`, but current `main` already changes bundled runtime-dependency staging for packaged installs, mirrors `dist/*` beside the staged deps so root `pw-ai` chunks resolve correctly, and adds release-check coverage to prevent the same class of breakage from shipping again. I found no newer stable release tag in this checkout that contains the fix.

## Evidence

- **Released v2026.4.23 still staged packaged plugin deps into a writable plugin root:** In the `v2026.4.23` tree, `resolveBundledRuntimeDependencyInstallRoot()` only fell back to the external stage dir for explicit stage env vars or `STATE_DIRECTORY`; otherwise it used `pluginRoot` when writable. That matches the reporter's `dist/extensions/browser/node_modules` placement.
  - file: [src/plugins/bundled-runtime-deps.ts:773](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/plugins/bundled-runtime-deps.ts#L773)
  - command: `git show a9797214338ba31c52c796adbb75afb16e0684a9:src/plugins/bundled-runtime-deps.ts | nl -ba | sed -n '752,787p'`
  - sha: [a9797214338b](https://github.com/openclaw/openclaw/commit/a9797214338ba31c52c796adbb75afb16e0684a9)
- **Current main forces packaged installs to the external plugin-runtime-deps root:** `main` now computes `externalRoot` first and returns it whenever `isPackagedBundledPluginRoot(pluginRoot)` is true, so packaged installs no longer stage bundled deps under `dist/extensions/<plugin>`. The same change also forces package-level repair onto the external root when the install is not a source checkout.
  - file: [src/plugins/bundled-runtime-deps.ts:889](https://github.com/openclaw/openclaw/blob/0ee9e8188dfd30e5ffc9b2255ed8d9a6d5d1d5b2/src/plugins/bundled-runtime-deps.ts#L889)

  - sha: [0ee9e8188dfd](https://github.com/openclaw/openclaw/commit/0ee9e8188dfd30e5ffc9b2255ed8d9a6d5d1d5b2)
- **Loader prep now mirrors dist beside the staged deps and remaps the module path:** `prepareBundledPluginRuntimeRoot()` stages deps at the resolved install root, then builds a mirrored `dist` tree under that root and remaps the plugin module path there. That gives root-level built chunks a matching `node_modules` walk from the mirrored `dist` tree.
  - file: [src/plugins/bundled-runtime-root.ts:19](https://github.com/openclaw/openclaw/blob/0ee9e8188dfd30e5ffc9b2255ed8d9a6d5d1d5b2/src/plugins/bundled-runtime-root.ts#L19)

- **Main adds package-release guards for root dist imports of bundled plugin deps:** `release-check` now scans built root `dist` files for imports of bundled plugin runtime deps and fails if the package root/mirror contract is broken. The associated tests cover both detecting root-owned imports and exempting plugin-owned chunks.

Current docs also reserve the manual Chromium install step for Docker, not packaged desktop/npm installs.
