## Summary

PR #53171 is redundant. Current main fixes the underlying npm/update Control UI packaging problem by shipping and validating built `dist/control-ui` artifacts, not by publishing all of `scripts/`; the PR's current diff is also unsafe because it leaves `package.json` invalid and would still not ship the `ui/` source tree needed for installed-package rebuilds.

## Best Possible Solution

Keep the shipped artifact-based packaging fix and release guards on main. Do not merge #53171; if maintainers later want installed npm packages to rebuild Control UI locally, track that as a separate feature requiring both `scripts/ui.js` and `ui/` source packaging plus size/release-policy review.
