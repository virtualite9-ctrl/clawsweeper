## Summary

Keep open. Current main has not applied the PR's requested hardening to the runtime source path: `src/memory-host-sdk/host/backend-config.ts` still allows QMD results in channel sessions when `memory.backend` is `qmd` and `memory.qmd.scope` is unset, and the colocated source test still asserts that channel sessions are allowed. The PR is recent, focused, and not maintainer-authored/protected, so it should remain available for maintainer review rather than be auto-closed.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to finish the direct-only default in `src/memory-host-sdk/host/backend-config.ts`, keep a regression test that denies channel sessions when `memory.qmd.scope` is unset, and align the package mirror plus public docs/generated config help so the shipped behavior is unambiguous.
