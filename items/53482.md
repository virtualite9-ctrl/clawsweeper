## Summary

Keep this issue open. Current main still has no gateway startup/runtime-auth diagnostic for legacy `CLAWDBOT_*` or `MOLTBOT_*` environment variables, and the provided GitHub context lists PR #53667 as an open, unmerged implementation candidate that uses closing syntax for this issue.

## Best Possible Solution

Keep this issue open and review, land, or replace PR #53667. The best fix belongs in the gateway startup/runtime-auth path: detect `CLAWDBOT_*` and `MOLTBOT_*` keys early, warn loudly with accurate `OPENCLAW_*` or owner-specific migration guidance, improve misleading auth/channel errors, and add focused regression tests while preserving the intentional removal of legacy fallback support.
