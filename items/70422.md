## Summary

Keep PR #70422 open. Current main has not implemented the MiniMax M2.7/highspeed VLM allowlist, and the existing source, tests, and docs still preserve a deliberate split where MiniMax chat models are text-only and image understanding routes through `MiniMax-VL-01`. The related PR #69691 is closed unmerged, so it is not a canonical shipped/superseding fix.

## Best Possible Solution

Keep the PR open for maintainer product/API judgment. The best path is to either reject it explicitly in favor of the shipped `MiniMax-VL-01` media-provider design, or update it with upstream/live proof that M2.7 and M2.7-highspeed are valid for this VLM path, clarify whether the endpoint needs a `model` field, and align source, tests, and docs around that verified behavior.
