## Summary

Keep open. Current main has related mitigations for Telegram polling stalls, per-session nested `sessions_send` lanes, and restart lane cleanup, but I did not find direct source, test, changelog, or history evidence proving the reported nested-subagent-triggered freeze across all Telegram pollers and health-monitor timers is fixed. The issue also has a recent corroborating comment, so conservative cleanup should not close it yet.

## Best Possible Solution

Keep this issue open until maintainers either reproduce it against current main with multiple Telegram accounts and nested `sessions_send`, or link/land a targeted fix proving nested subagent completion cannot stall Telegram polling and health-monitor timers. A strong path would add a high-level regression around coordinator-to-subagent `sessions_send` plus Telegram polling/health liveness, or document the exact root cause and why the existing lane/watchdog/restart fixes close this failure mode.
