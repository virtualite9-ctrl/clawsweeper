## Summary

Keep open. Current main has adjacent Telegram polling recovery, health-monitor restart logic, and per-session nested `sessions_send` lanes, but I did not find code, tests, docs, changelog, or discussion proving that nested subagent activity can no longer freeze all Telegram pollers and gateway health timers. ## Best Possible Solution

Keep this issue open until maintainers reproduce it on current main with multiple Telegram bot accounts and nested `sessions_send`, or land/link a targeted fix proving nested subagent completion cannot stall Telegram polling and health-monitor timers. A strong path is an integration regression around coordinator-to-subagent `sessions_send` plus Telegram polling/health liveness, or a documented root cause explaining why the existing lane/watchdog/restart fixes close this exact failure mode.
