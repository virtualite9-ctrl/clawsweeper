## Summary

Current main now supports Control UI chat uploads for non-video, non-image files end to end: the picker is no longer image-only, the UI serializes non-image attachments as files, and Gateway persists them as managed media. The earlier UI-only PR was superseded by merged replacement PR #72299, which carried the fix through the backend and tests.

## Best Possible Solution

Close this issue as implemented on main and keep the merged #72299 behavior: Control UI accepts non-video files, serializes non-images as file attachments, and Gateway persists them through managed media. Any remaining request for video uploads or direct workspace-copy semantics should be opened as a separate, narrower enhancement.
