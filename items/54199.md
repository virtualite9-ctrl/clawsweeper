## Summary

Keep open. Current `main` still implements Control UI chat attachment selection as image-only, serializes attachments as `type: "image"`, drops non-image payloads in the gateway, and exposes only the read-only `__openclaw__/assistant-media` route rather than a general upload POST route. The issue is still consistent with `main` at `8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf`.
