## Summary

Current main already fixes the user-visible bug from PR #59501. The exact placeholder-level `helpOption(false)` change is not present, but `runCli` now preregisters the real primary command before Commander parses `--help`, so lazily registered command help resolves through the real command tree. The same behavior is present in the shipped `v2026.4.24` release, and the linked bug #59449 plus the similar follow-up PR #69930 have already been closed.

## Best Possible Solution

Close PR #59501 as implemented by current main's shipped primary-command preregistration path. Keep the existing `runCli` behavior and coverage; open a narrow maintainer-owned cleanup only if the team wants extra direct placeholder-helper hardening beyond the fixed CLI user behavior.
