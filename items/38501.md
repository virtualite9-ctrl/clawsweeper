## Summary

Keep #38501 open. Current main still implements Ctrl+C as clear/warn/exit status flow only, with no active-run hint and no activeChatRunId check in the Ctrl+C path. Esc and /abort remain the actual abort paths, and the focused linked fix PR #38502 is still unmerged in the provided context.

## Best Possible Solution

Keep #38501 open until a focused TUI UX patch lands on main. The best fix is to update src/tui/tui.ts so Ctrl+C status text includes an active-run hint when state.activeChatRunId is set, while preserving current clear/warn/exit behavior and leaving Esc plus /abort as the actual abort controls. Add a small regression test for the status text or handler behavior. PR #38502 is the narrowest linked candidate; #38661 bundles unrelated fixes and #38540 was closed unmerged.
