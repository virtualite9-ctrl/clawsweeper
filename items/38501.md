## Summary

Keep #38501 open. Current main still has Ctrl+C implemented only as clear/warn/exit UI flow and does not add the requested active-run hint; Esc and `/abort` remain the actual abort paths. The directly linked fix PR #38502 and broader PR #38661 are still open in the provided GitHub context, while #38540 was closed unmerged, so this is not implemented or superseded by shipped code.

## Best Possible Solution

Keep #38501 open until a focused TUI UX patch lands on main. The best implementation is to update src/tui/tui.ts so Ctrl+C status text includes an active-run hint when state.activeChatRunId is set, preserving existing Ctrl+C clear/exit behavior and leaving Esc plus /abort as the actual abort controls. Add a small regression test for the status text or handler behavior, then let the linked issue close through the merged fix PR.
