## Summary

Current main still does not implement the requested active-run hint for TUI Ctrl+C. Ctrl+C remains wired to clear/warn/exit status text only, while Esc and `/abort` remain the actual abort paths. Related fix PRs in the provided context are open or unmerged, so the issue should remain open.

## Best Possible Solution

Keep #38501 open until a focused TUI UX patch lands on main. The likely fix belongs in `src/tui/tui.ts`: when `state.activeChatRunId` is set, preserve the existing Ctrl+C clear/warn/exit behavior but append an explicit active-run hint such as `run still active (Esc or /abort to stop)`. Add a small regression test for the Ctrl+C status behavior.
