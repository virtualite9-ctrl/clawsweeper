## Summary

Keep #9764 open. Current main still lacks Google Chat user OAuth config, schema, credential resolution, secret handling, token creation, and token selection for the requested reactions, media uploads, and proactive DM/email-alias paths. The latest issue discussion already includes a maintainer-side Codex review keeping this open, and the external cross-reference about Google Chat file upload does not supersede the missing bundled Google Chat support.

## Best Possible Solution

Keep the issue open. The bundled Google Chat plugin should add a user OAuth credential model with strict schema, redaction/SecretRef handling, setup/docs, and tests; resolve refresh-token credentials safely; create an OAuth2Client with the required Google Chat scopes; select the user token for reactions, attachment uploads, typingIndicator="reaction", and proactive DM/email-alias lookup; preserve the existing service-account path for webhook verification and bot-level operations; and gracefully warn/fallback when user OAuth is absent.
