## Summary

Keep #9764 open. Current main still does not implement Google Chat user OAuth support: Google Chat config and schema expose only service-account credentials, credential resolution has no OAuth refresh-token path, auth is fixed to `chat.bot` service-account credentials, and reactions/attachment upload/direct-message lookup still share the same bot access-token helper. The linked external Google Chat work is useful prior art, but it is not shipped in `openclaw/openclaw`.

## Best Possible Solution

Keep the issue open. The bundled Google Chat plugin should add a strict, secret-aware user OAuth credential model, resolve refresh-token credentials safely, construct scoped user OAuth clients, route reactions, media uploads, reaction typing, and proactive DM/email-alias lookup through user auth, preserve service-account auth for webhook verification and bot-level sends, and document graceful fallback behavior when user OAuth is absent.
