## Summary

Keep #9764 open. Current main still lacks Google Chat user OAuth config, SecretRef/redaction surface, refresh-token credential resolution, OAuth token construction, and per-operation token selection. Reactions, attachment upload, reaction typing, and proactive DM lookup still route through the existing service-account/chat.bot path. The provided issue discussion already includes a maintainer-side Codex review keeping this open, and the cross-referenced external Google Chat PR is outside openclaw/openclaw.

## Best Possible Solution

Keep the issue open. The bundled Google Chat plugin should add a strict user OAuth credential model with redaction/SecretRef support and docs, resolve refresh-token credentials safely, construct a scoped user OAuth client, use that token for reactions, attachment uploads, typingIndicator="reaction", and proactive DM/email-alias lookup, preserve the existing service-account path for webhook verification and bot-level sends, and warn/fallback cleanly when user OAuth is absent.
