## Summary

Keep open. #72200 is a recent, concrete core runtime-dependency staging bug report, and current main does not provide strong evidence that the reported reinstall cascade is fixed. The current loader and installer still build a retained/shared `installSpecs` set and pass it into the npm install path, matching the report's `N missing, 23 install specs` symptom. Recent local history after `v2026.4.24` only shows manifest caching and runtime mirror serialization around this surface, not a fix that removes or regression-tests the shared positional install behavior.

## Best Possible Solution

Keep #72200 open as a core bundled plugin runtime-deps bug. The best fix path is to make the shared external stage install idempotent under multiple bundled plugins, either by installing only truly missing specs without npm pruning, coalescing the full shared install once, or using a content-hash marker for the retained spec set, then add a regression covering multiple enabled bundled plugins sharing an external `OPENCLAW_PLUGIN_STAGE_DIR` or symlinked shared `node_modules` so a later plugin install cannot make earlier plugin deps missing again.
