## Summary

Current main implements the bundled runtime-dependency hardening needed for #72200: installs are staged in an OpenClaw-owned external root, guarded by a filesystem lock, anchored with a local package manifest so npm does not walk up to ~/package.json, and covered by unit/E2E checks for retained deps and poisoned HOME npm projects. The reporter later narrowed the root cause to ancestor HOME package.json behavior, and a second commenter noted this was fixed in beta.

## Best Possible Solution

Close #72200 as implemented on main. Keep the current locked, external, manifest-anchored bundled runtime-deps installer and direct any new recurrence to a fresh bug with current version, logs, affected plugin/channel list, and the staged install root path.
