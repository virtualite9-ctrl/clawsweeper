## Summary

#72095 tracks a liveness/design gap in the unlanded Fleet Orchestrator plugin stack, not current OpenClaw core behavior. Current main has no `extensions/orchestrator` store, `spawn-watch`, `sweepExpired`, or `STALE_ELIGIBLE` surface; the related orchestrator foundation PR was already closed as ClawHub/plugin-track, and the existing Plugin SDK has service/CLI/route seams sufficient for an external plugin to own this TTL policy.

## Best Possible Solution

Close this OpenClaw core issue as ClawHub/plugin-scope. If Fleet Orchestrator continues, carry the design into an external ClawHub/npm plugin and implement watcher-side TTL and/or force-expire behavior inside that plugin’s store/service. Open a new narrow OpenClaw issue only if that external implementation exposes a concrete missing Plugin SDK seam.
