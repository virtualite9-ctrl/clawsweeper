## Summary

Keep open. Current main has adjacent memory work: append-only pre-compaction flushes, scored short-term promotion into `MEMORY.md`, hybrid retrieval, and a provenance-rich `memory-wiki` companion. It still does not implement the requested durable write resolver with memory type classification, source/provenance requirements, near-duplicate/conflict search, create/merge/update/conflict outcomes, entity normalization, or revision history.

## Best Possible Solution

Keep this open and implement it in the active memory/memory-core durable write path, coordinated with #42646, #42647, and #42650. The concrete path is a resolver-backed durable candidate model that classifies memory type, requires source refs, normalizes entities, searches current durable memory for near duplicates and contradictions, routes outcomes to create, merge/update, or conflict-for-review, and records revision/provenance history. The #49108 smart-memory discussion and memory-wiki claim/evidence model are useful design inputs, but they do not replace the missing core write contract.
