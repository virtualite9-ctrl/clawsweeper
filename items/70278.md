## Summary

Keep open. Current main still contains the broken `docs/plugins/memory-wiki.md` QMD + bridge example with `memory-wiki` nested under `memory`, while the same doc and config schema show plugin configuration belongs under `plugins.entries.<id>.config`. The PR is a narrow docs-only correction and is not superseded by the related runtime bridge issues in the provided context.

## Best Possible Solution

Keep the PR open for maintainer review or merge after the normal docs checks. The best fix is the PR's narrow documentation change: close `memory` after `backend: "qmd"`, then place `memory-wiki` under `plugins.entries`, consistent with the source schema and the earlier Memory Wiki config section.
