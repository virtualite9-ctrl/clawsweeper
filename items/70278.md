## Summary

Keep open. Current main still has the incorrect QMD + bridge docs example, with `memory-wiki` nested under `memory`. The surrounding docs, config types, strict schema, and memory-wiki runtime all agree that the plugin config belongs under `plugins.entries.memory-wiki.config`. PR #70278 is a focused docs-only fix and is not superseded by the related bridge-runtime issues.

## Best Possible Solution

Keep the PR open and merge it, or apply the equivalent docs fix on main, moving the QMD + bridge `memory-wiki` block under `plugins.entries` in `docs/plugins/memory-wiki.md`. No core code or plugin API change is needed.
