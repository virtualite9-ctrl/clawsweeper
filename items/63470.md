## Summary

Current main and the shipped v2026.4.24 release no longer force a required plugin config schema for disabled plugin entries during config writes. That covers the reported `openclaw plugins enable openclaw-memori` failure mode where selecting another memory plugin leaves `memory-lancedb` disabled without needing an embedding config. The discussion workaround remains valid for users who intentionally keep `memory-lancedb` enabled, because its manifest still requires `embedding.apiKey`.

## Best Possible Solution

Close this issue as already fixed in the shipped plugin config validation path. Keep the current behavior: enabling another memory plugin may disable the competing memory slot plugin without requiring that disabled plugin's required config, while still validating `memory-lancedb` when the user actually configures or enables it.
