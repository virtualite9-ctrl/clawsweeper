## Summary

Keep #59413 open. Current main has useful auth-profile rotation/cooldown controls, but it still lacks the requested per-fallback-candidate or provider/model retry budget for pool/proxy providers. The related discussion explicitly narrowed the remaining gap after PR #55118, and #55118 is closed unmerged in the provided GitHub context.

## Best Possible Solution

Keep the issue open and scope it as a core failover/config enhancement: add a documented provider/model or fallback-candidate retry override that composes with existing `auth.cooldowns.*` profile-rotation controls, with tests for a single auth profile behind a pooled proxy returning transient 502/timeouts before success.
