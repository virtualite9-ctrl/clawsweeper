## Summary

Keep this PR open for maintainer review rather than auto-closing. Current main still does not implement `<notts>...</notts>` visual-only TTS handling, so the central feature is not obsolete. However, the submitted branch is not mergeable as-is: it targets now-obsolete `src/tts` implementation files, includes unrelated shipped or security-sensitive changes, and has unresolved review comments about `<notts>` parity and tagged-mode behavior.

## Best Possible Solution

Keep this PR open for now, but ask for a focused rework: implement `<notts>` preprocessing in `extensions/speech-core/src/tts.ts`, add colocated speech-core tests for visible/spoken text separation and tagged-mode semantics, update `docs/tools/tts.md`, and drop or split all unrelated gateway, macOS, media, MIME, and helper-script changes. If the author cannot rework it, a maintainer should close this PR as dirty only after opening or keeping a narrower follow-up for the missing `<notts>` capability.
