## Summary

Keep this PR open. Current main still lacks the requested `<notts>...</notts>` visual-only TTS behavior, but the submitted branch is not merge-ready as-is: it targets older core TTS files while current TTS execution goes through the bundled `speech-core` plugin runtime, and it mixes unrelated gateway authorization, macOS speech, media conversion, MIME, agent media, script, and docs changes.

## Best Possible Solution

Keep this PR open for maintainer or author rework, or replace it with a focused PR. The best implementation should target `extensions/speech-core/src/tts.ts`, `src/tts/directives.ts` if the syntax becomes part of the directive contract, focused TTS tests, and `docs/tools/tts.md`. Split unrelated gateway authorization, macOS speech, media conversion, MIME, and agent media propagation changes into separate PRs or drop them where current main already has equivalent behavior.
