## Summary

Keep this PR open. Current main still lacks the requested `<notts>...</notts>` visual-only TTS behavior, so the feature is not obsolete, but the branch is not merge-ready because it targets an older TTS ownership shape and mixes unrelated gateway, macOS, media, agent, script, and docs changes.

## Best Possible Solution

Keep this PR open for maintainer or author rework, or ask for a replacement focused PR if that is cleaner. The best implementation belongs in the current `extensions/speech-core/src/tts.ts` runtime and related directive/preprocessing tests, with docs updated at `docs/tools/tts.md`. Split unrelated gateway authorization, macOS Talk Mode, media conversion, MIME, and agent media-propagation changes into separate PRs with their own justification and tests.
