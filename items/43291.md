## Summary

Keep this PR open. Current main still has the CLI-backend prompt-mode gap: cron and subagent session keys reach the CLI runner, but the CLI system-prompt builder has no session-key or prompt-mode input and calls `buildAgentSystemPrompt` without `promptMode`, so it defaults to full mode and can include `## Heartbeats` / `HEARTBEAT_OK`. The embedded runner already has the intended minimal-mode behavior for cron/subagent sessions, but the CLI path still does not.

## Best Possible Solution

Keep this PR open and adapt it to current main. The best landing shape is to centralize `resolvePromptModeForSession` in an owner-neutral shared helper, pass `params.sessionKey` or a derived prompt mode into CLI prompt construction from `prepareCliRunContext`, forward that mode to `buildAgentSystemPrompt`, and add CLI regression coverage for no key, regular agent key, cron run key, and subagent key. After that, run the repoâ€™s changed checks when dependencies are available.
