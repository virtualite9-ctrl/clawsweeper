## Summary

Keep this PR open. Current main still has the CLI-backend prompt-mode gap the PR targets: cron and subagent session keys reach the CLI runner, but `buildSystemPrompt` does not accept session identity and calls `buildAgentSystemPrompt` without `promptMode`, so the prompt defaults to full mode and can include the heartbeat section. The related cron issue is closed, but the current source still only applies the cron/subagent minimal-mode behavior on the embedded runner path, not the CLI-backend path. Security review found no supply-chain or permissions concern in the PR diff; it only touches TypeScript agent/routing source plus a focused test file.

## Best Possible Solution

Keep this PR open and adapt it to current main. The best landing shape is to expose or centralize `resolvePromptModeForSession` in an owner-neutral helper, pass `params.sessionKey` into CLI prompt construction from `prepareCliRunContext`, forward the derived `promptMode` to `buildAgentSystemPrompt`, and add CLI-specific regression coverage for no key, regular agent key, cron run key, and subagent key.
