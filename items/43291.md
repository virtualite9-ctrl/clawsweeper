## Summary

Current main still has the CLI-backend prompt-mode gap this PR targets. Cron isolated runs can pass a cron run session key into `runCliAgent`, and the embedded runner already maps cron/subagent session keys to minimal prompt mode, but CLI prompt construction still drops the session key and lets `buildAgentSystemPrompt` default to full mode, which can include `## Heartbeats` / `HEARTBEAT_OK`. The PR remains useful, though it needs adaptation to current main because CLI prompt construction now flows through `src/agents/cli-runner/prepare.ts`.

## Best Possible Solution

Keep this PR open and adapt it to current main. The landing shape should centralize the cron/subagent prompt-mode resolver in a shared owner-neutral helper, pass `params.sessionKey` or the derived prompt mode into CLI prompt construction from `prepareCliRunContext`, forward that mode to `buildAgentSystemPrompt`, and add CLI regression tests for no key, regular agent key, cron run key, and subagent key. Review the small routing/helper move during rebase, then run the repo’s changed checks before merge.
