## Summary

Keep open. Current main still has the CLI-backend gap this PR targets: cron/subagent session keys reach `runCliAgent`, but the CLI system prompt builder neither accepts `sessionKey` nor passes a derived minimal `promptMode`, so `buildAgentSystemPrompt` defaults to full mode and can include the heartbeat section. The embedded runner already derives minimal mode for cron/subagent sessions, and the PR’s provided diff is a focused source/test change with no apparent security-sensitive workflow, dependency, lockfile, package, or release-script changes.

## Best Possible Solution

Keep this PR open, rebase it onto current main, and implement the same cron/subagent minimal prompt-mode selection for the CLI backend. The cleanest current-main shape is to share `resolvePromptModeForSession` from an owner-neutral helper, pass `sessionKey` through CLI prompt construction, and add CLI-specific regression tests for normal, cron, and subagent session keys before running the targeted agent/CLI checks.
