## Summary

Keep open. Current `main` still feeds raw `BOOT.md` text into the normal model/fallback path, and there is no BOOT-specific echo suppression or model-adaptation guard in the codebase. Existing boot protections isolate session bookkeeping, but they do not prevent a weak fallback model from turning the boot prompt into a user-visible `message` tool send.

## Evidence

- **BOOT prompt is still plain-text instructions plus message-tool guidance:** `runBootOnce()` builds the prompt by embedding `BOOT.md` verbatim and instructing the model to use the `message` tool, then only requires `NO_REPLY` afterward.
  - file: [src/gateway/boot.ts:43](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/gateway/boot.ts#L43)

- **Fallback retries reuse the same body across providers/models:** `agent-command` passes the same `body` into `runAgentAttempt()` for each fallback candidate; only `providerOverride` / `modelOverride` change. There is no BOOT-specific prompt rewrite on fallback.
  - file: [src/agents/agent-command.ts:878](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/agents/agent-command.ts#L878)
