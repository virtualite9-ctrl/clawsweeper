## Summary

Current `main` does not show a fix for #61981. The `read` tool still emits continuation offsets, and compaction still preserves raw `toolResult` text in the "Recent turns preserved verbatim" section, so stale offsets can still survive into later turns after a file shrinks.

## Evidence

- **Read tool still appends continuation offsets:** `executeReadWithAdaptivePaging()` keeps and re-emits `Use offset=... to continue` notices, and `createOpenClawReadTool()` forwards the caller's `offset` through to the base read tool without any local reset/clamp step.
