## Summary

Keep open. Current `main` still treats proxied `openai-responses` routes as ineligible for forced `store=true`; docs and committed tests both encode that direct-only behavior, so this PR is neither implemented nor obsolete. The PR discussion also shows the author addressed the main review concern by separating compaction defaults from the `supportsStore` opt-in and adding tests, so this remains live maintainer-review work rather than stale noise.
