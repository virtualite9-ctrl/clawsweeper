## Summary

Current main still does not implement the requested plain-text copy option for Control UI assistant messages. The assistant message action path remains Markdown-only: it labels the control “Copy as markdown”, writes the supplied markdown string directly to the clipboard, and assistant bubbles render that single action with no dropdown, format selector, or preference. The existing markdown-stripping helpers are not wired into this copy path, and the provided issue discussion already includes an Apr 26, 2026 maintainer-side Codex review keeping the issue open.

## Best Possible Solution

Keep #7909 open until the Control UI assistant-message copy action offers a real format choice. The likely implementation belongs near `ui/src/ui/chat/copy-as-markdown.ts` and `ui/src/ui/chat/grouped-render.ts`: add a compact Markdown vs plain-text action menu or preference, reuse or adapt the existing markdown-stripping helper where appropriate, and add focused clipboard/UI tests proving both formats write the expected text.
