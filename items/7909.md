## Summary

Keep #7909 open. Current main still provides only the assistant-message Markdown copy action: the UI labels it "Copy as markdown", writes the supplied Markdown string directly to the clipboard, and the assistant bubble action path renders only that button. I found no plain-text copy action, dropdown, preference, or focused test coverage for the requested Markdown/plain-text choice. The latest issue discussion also includes a maintainer-side Codex review explicitly keeping this item open, and the related Android copy/selection items do not supersede this Control UI request.

## Best Possible Solution

Keep #7909 open until the Control UI adds a real assistant-message copy format choice. The likely implementation belongs near `ui/src/ui/chat/copy-as-markdown.ts` and `ui/src/ui/chat/grouped-render.ts`: expose Markdown vs plain-text copy through a compact action menu or preference, reuse or add a tested Markdown-to-plain-text helper, and add focused UI/clipboard tests proving both formats write the expected text.
