## Summary

Keep open. Current `main` still only adds and backfills `owner_key` on legacy `flow_runs` tables, but it does not rebuild the table to remove the old `owner_session_key TEXT NOT NULL` shape. The active write path inserts only `owner_key`, so the PR's reported hybrid-schema failure remains plausible, and `main` still lacks a regression test for that upgrade path.

## Evidence

- **Current migration is additive only:** `ensureSchema()` adds `owner_key` and copies data from `owner_session_key` when needed, but it does not rebuild `flow_runs`, drop the legacy column, or remove the old `NOT NULL` constraint. That leaves the hybrid-table state described in the PR unaddressed on current `main`.
