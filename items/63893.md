## Summary

Keep this PR open. Current main still lacks the proposed legacy `flow_runs` table rebuild, and the reported SQLite failure remains plausible because current writes use `owner_key` but do not populate the legacy `owner_session_key NOT NULL` column that can remain on upgraded tables.

## Best Possible Solution

Keep this PR open. The best path is to review or refresh the PR against current main, keep the fix scoped to `src/tasks/task-flow-registry.store.sqlite.ts`, retain the legacy hybrid-table regression test in `src/tasks/task-flow-registry.store.test.ts`, and validate with the targeted task-flow store test plus `pnpm check:changed` before landing.
