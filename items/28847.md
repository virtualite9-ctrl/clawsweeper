## Summary

Keep open. Current main has adjacent safeguards: multi-profile auth order, per-profile cooldown/backoff, rate-limit classification for 429/RESOURCE_EXHAUSTED, limited same-provider rotation caps, and env-list key rotation for some non-chat execution paths. It still does not implement the issue’s central Gemini quota-bucket behavior: no first-class quotaBucket/quotaGroup/project-bucket config, no bucket-level auth-health state, and no failure fanout that cools every key/profile in the same Google project before selecting a different bucket. The prior comments are useful evidence: pyro0ownz linked a working sidecar, and aashari’s fork explicitly covered per-model cooldown but not native bucket/project grouping.

## Best Possible Solution

Keep the issue open. The best product path is to add explicit quotaBucket/quotaGroup metadata for auth profiles or provider-owned external profiles, persist bucket-level health, update markAuthProfileFailure and runtime profile selection to cool/skip all profiles in the same Gemini project bucket on 429/RESOURCE_EXHAUSTED, and add Gemini multi-project tests plus docs. If maintainers prefer this outside core, first add a plugin hook/API that exposes failing auth profile identity and safe shared auth-health updates so a ClawHub plugin can implement bucket fanout without log tailing or config rewrites.
