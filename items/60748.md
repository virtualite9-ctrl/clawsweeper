## Summary

Current `main` does not implement the requested Feishu card customization. The card header is still derived from agent identity, the footer is still hardcoded to `Agent/Model/Provider`, there is no current schema/doc surface for `cardHeader` or footer element toggles, and reply context is only forwarded as reply metadata rather than rendered into the card chrome.
