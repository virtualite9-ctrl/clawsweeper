## Summary

Keep #60114 open. Current main still lacks the central TUI prompt-alias behavior: `/alias` and `/unalias` are not in the TUI slash command registry/help, there is no alias persistence module, autocomplete is not alias-aware, and unknown slash commands still fall through to the gateway send path. The earlier #49141 is closed unmerged and does not supersede this rebased PR. The PR still needs maintainer/author cleanup before merge because the provided review context identifies an unrelated Matrix test rewrite and a prompt-preservation bug in alias argument parsing.

## Best Possible Solution

Keep the PR open, but require a focused refresh against current main: restore or split out the unrelated Matrix test changes, parse alias arguments from the raw command text so saved prompts preserve user input, then land the TUI alias feature with persistence, autocomplete, command-handler tests, and user-facing TUI docs.
