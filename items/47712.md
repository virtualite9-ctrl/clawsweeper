## Summary

Current `main` includes the new TTS stack, but it does not implement a `messages.tts.filterEmoji` setting or any TTS-specific emoji filtering path. The issue is still actionable on `main` rather than already fixed or obsolete.

## Evidence

- **TTS config type has no `filterEmoji` field:** `TtsConfig` only defines `auto`, `enabled`, `mode`, `provider`, `summaryModel`, `modelOverrides`, `providers`, `prefsPath`, `maxTextLength`, and `timeoutMs`. There is no `filterEmoji` option in the public config contract.
  - file: [src/config/types.tts.ts:28](https://github.com/openclaw/openclaw/blob/d38ed0831d7372c8505dc944aea69b5b83da1435/src/config/types.tts.ts#L28)
  - command: `nl -ba src/config/types.tts.ts | sed -n '1,120p'`
  - sha: [d38ed0831d73](https://github.com/openclaw/openclaw/commit/d38ed0831d7372c8505dc944aea69b5b83da1435)
- **Speech-core resolver does not read any emoji-filter flag:** `resolveTtsConfig()` copies the same known fields into `ResolvedTtsConfig`; no branch reads `raw.filterEmoji` or similar, so the runtime cannot honor the requested setting.
  - file: [extensions/speech-core/src/tts.ts:313](https://github.com/openclaw/openclaw/blob/d38ed0831d7372c8505dc944aea69b5b83da1435/extensions/speech-core/src/tts.ts#L313)
  - command: `nl -ba extensions/speech-core/src/tts.ts | sed -n '300,380p'`
  - sha: [d38ed0831d73](https://github.com/openclaw/openclaw/commit/d38ed0831d7372c8505dc944aea69b5b83da1435)
- **TTS text preparation strips markdown, not emoji:** The auto-TTS path runs `stripMarkdown(textForAudio).trim()` before synthesis. `stripMarkdown()` only removes markdown markers such as `**`, `_`, headers, blockquotes, rules, and backticks; it does not strip emoji code points.
  - file: [extensions/speech-core/src/tts.ts:1147](https://github.com/openclaw/openclaw/blob/d38ed0831d7372c8505dc944aea69b5b83da1435/extensions/speech-core/src/tts.ts#L1147)

- **Current strip helper confirms no emoji filtering:** The shared helper used by the TTS pipeline only rewrites markdown syntax and returns trimmed text. Its tests also describe the TTS preparation goal as removing formatting symbols, not emoji.
  - file: [src/shared/text/strip-markdown.ts:1](https://github.com/openclaw/openclaw/blob/d38ed0831d7372c8505dc944aea69b5b83da1435/src/shared/text/strip-markdown.ts#L1)
