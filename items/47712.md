## Summary

Keep #47712 open. Current main has a Discord voice-only emoji sanitizer, but the generic TTS tool/config path still has no `messages.tts.filterEmoji` contract and does not filter emoji before provider synthesis.

## Best Possible Solution

Keep the issue open and implement a small core TTS feature: add `filterEmoji?: boolean` to `TtsConfig` and `TtsConfigSchema`, document it under `messages.tts`, add focused tests for auto-TTS and direct `tts.convert`/`textToSpeech` input preparation, and route generic TTS text through a shared emoji-stripping helper before provider synthesis while preserving the existing Discord voice behavior or migrating it to the shared helper.
