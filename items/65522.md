## Summary

Keep open. Current main still leaves the Control UI `Refresh chat data` button enabled during active turn states, and open PR #66395 explicitly tracks this issue with closing syntax as the implementation candidate.

## Best Possible Solution

Keep this issue open while maintainers review PR #66395. The likely fix belongs in `ui/src/ui/app-render.helpers.ts`: disable the refresh button while `chatSending`, `chatRunId`, or `chatStream` indicate an active turn, preferably with a focused render test in `ui/src/ui/app-render.helpers.browser.test.ts`. If PR #66395 stays bundled with unrelated changes, split or narrow it before merge rather than closing this issue early.
