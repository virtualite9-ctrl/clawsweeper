## Summary

Current `main` still does not implement a config-backed default for `openclaw tui --deliver`. The CLI exposes only an explicit `--deliver` flag, `runTui` still falls back to `false`, sends forward that fixed startup value, and the public config type/schema/help surfaces do not define `tui.deliver`. The related implementation PRs #33198 and #33208 were closed unmerged; the related stuck-spinner issue #66991 is useful impact evidence but does not supersede this persisted-default request.

## Best Possible Solution

Keep #33102 open until a maintainer-approved change adds a documented persisted delivery default, likely `tui.deliver`, wires it through `OpenClawConfig`, zod/generated schema and help/docs, CLI precedence including explicit opt-out behavior, TUI relaunch/send behavior, and focused coverage for config default, CLI override, and false fallback. Treat #33198/#33208 and the arcabot workaround linked from #66991 as useful prior art, not shipped fixes.
