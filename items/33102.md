## Summary

Keep #33102 open. Current main still has no config-backed default for `openclaw tui --deliver`: the CLI flag is explicit-only, TUI runtime falls back to `false`, sends forward that fixed startup value, and the config type/schema/help surfaces still expose no `tui.deliver` key. The related implementation PRs #33198 and #33208 were closed unmerged in the provided GitHub context; the closed #66991 stuck-spinner report is useful related context but does not supersede this persisted-default request.

## Best Possible Solution

Keep #33102 open until a maintainer-approved change adds a documented config key such as `tui.deliver`, wires it through `OpenClawConfig`, zod/generated schema/help surfaces, CLI precedence, relaunch behavior, and TUI send behavior, and adds focused coverage for config default, explicit CLI override, and false fallback. Treat #33198/#33208 and the arcabot workaround from #66991 as useful prior art, but not as shipped fixes.
