## Summary

Current main still does not implement a config-backed default for `openclaw tui --deliver`. The CLI declares only an explicit `--deliver` flag with default `false`, `runTui` still falls back to `opts.deliver ?? false`, sends that fixed value through `client.sendChat`, and the config type/schema surfaces still have no `tui.deliver` key. The prior implementation attempts #33198 and #33208 were closed unmerged, and #66991 is related impact evidence rather than a canonical replacement for this persisted-default request.

## Best Possible Solution

Keep #33102 open until a maintainer-approved change adds a documented persisted delivery default, likely `tui.deliver`, wires it through `OpenClawConfig`, zod/generated schema and help/docs, defines CLI precedence including explicit `--deliver` and `--no-deliver` behavior, updates TUI send/relaunch/settings behavior as needed, and adds focused tests for config default, CLI override, and false fallback. Treat #33198, #33208, and the arcabot workaround linked from #66991 as prior art, not shipped fixes.
