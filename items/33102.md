## Summary

Current main still does not implement a config-backed default for `openclaw tui --deliver`. The CLI only has an explicit `--deliver` option defaulting to false, `runTui` still falls back to `false`, sends use that fixed startup value, and the config type/schema/docs still lack a `tui.deliver` key. The related PRs #33198 and #33208 were closed unmerged; #66991 and the linked workaround are useful impact/prior-art evidence but do not supersede this persisted-default request.

## Best Possible Solution

Keep #33102 open until a maintainer-approved change adds a documented persisted TUI delivery default, likely `tui.deliver`, wires it through `OpenClawConfig`, strict zod/generated config schema and config help/docs, defines precedence for config vs explicit `--deliver`/`--no-deliver`, and adds tests for config default, CLI override, and false fallback behavior. Treat #33198, #33208, #66991, and the linked arcabot workaround as prior art and impact evidence.
