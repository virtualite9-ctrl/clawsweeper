## Summary

Keep #20837 open. Current main already exposes channel metadata for normal chat turns and trusted inbound metadata, but the slash-command prompt rebuild path still drops the available channel metadata and continues to build runtime info with hardcoded unknown host/os/arch and no channel/capabilities/channelActions/shell. The linked PR #21271 was closed unmerged, and the latest issue discussion explicitly identified this remaining command-path scope.

## Best Possible Solution

Keep the issue open and implement the missing slash-command runtime metadata in src/auto-reply/reply/commands-system-prompt.ts. Forward the normalized command channel/channelId, account-aware channel capabilities/actions, shell, host, os, and arch using the same helpers as the embedded runner, then add focused regression tests proving /context or /export-session generated from Telegram includes channel=telegram and expected capabilities. Treat broader selective cross-channel reply policy as a separate follow-up if maintainers want that behavior.
