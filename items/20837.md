## Summary

Keep #20837 open. Current main already carries channel metadata through ordinary agent runs and has origin-based reply routing, but the slash-command system-prompt reconstruction used by `/context` and `/export-session` still omits the available channel/capabilities/shell runtime metadata. The linked PR #21271 targeted this exact remaining gap but is closed unmerged, and the recent issue discussion already called out the same narrow path as still open.

## Best Possible Solution

Keep the issue open. Implement the missing slash-command runtime metadata in `src/auto-reply/reply/commands-system-prompt.ts` by reusing or extracting the embedded-runner pattern for machine/shell details, normalized channel, account-aware capabilities, message-tool capabilities, reaction guidance, and channel actions. Add focused regression coverage showing `/context` or `/export-session` from Telegram rebuilds a prompt containing `channel=telegram` and expected capabilities; track prompt-driven selective reply-routing rules separately if maintainers want that behavior.
