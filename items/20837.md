## Summary

Keep #20837 open. Current main already exposes channel metadata for normal agent runs and trusted inbound metadata, but the slash-command prompt rebuild path still drops the available command channel and runtime details. The linked fix PR #21271 was closed unmerged, and the remaining core scope is still actionable.

## Best Possible Solution

Keep the issue open and implement the missing slash-command prompt runtime metadata in src/auto-reply/reply/commands-system-prompt.ts. Reuse the embedded runner pattern for channel normalization, account-aware capability/action resolution, shell/host/os/arch, and add focused regression tests proving /context or /export-session from Telegram includes the expected channel metadata. Treat selective reply routing as a separate follow-up if maintainers want that policy.
