## Summary

Keep open. Current `main` does not implement this trust-tier feature: plugin records and CLI output still expose only `format`/`bundleFormat`/`origin`, and docs describe the trust boundary only implicitly. The PR also appears to have a real unresolved correctness issue from review: undefined manifest formats are normalized to `"openclaw"` in loader record creation, so resolving trust tier from raw `manifestRecord.format` would mislabel native-loaded plugins as `content`.
