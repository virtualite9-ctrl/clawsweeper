## Summary

Current `main` already contains a targeted fix for subagent delivery-context seeding and matching regression coverage. Fresh or pre-patched subagent sessions now inherit `channel`/`to`/`accountId`/`threadId` from the agent request, and current announce/task-delivery code documents and supports fallback delivery instead of leaving the parent waiting indefinitely.

## Close Comment

Closing from a Codex review: current `main` already implements this. The gateway `agent` handler now explicitly seeds spawned subagent sessions with `channel`/`to`/`accountId`/`threadId` from the request when the child session does not already have routing metadata (`src/gateway/server-methods/agent.ts:647`). There is matching regression coverage for both fresh subagent sessions and the real pre-patched `sessions.patch` spawn path (`src/gateway/server.agent.subagent-delivery-context.test.ts:79` and `:155`). Current delivery paths also document and support fallback to the requester session's stored route and queued session delivery if direct delivery is unavailable (`docs/automation/tasks.md:129`, `docs/help/faq.md:227`). This landed on `main` in commit `934dd5b3a7389d10f2191b4674d5fa5a98f94d35` on 2026-04-24, so the report is best treated as already implemented on `main`. It does not appear to be in the latest tagged release `v2026.4.22` yet, so upgrading to a build that includes that commit is the relevant path if this was seen on an older version.
