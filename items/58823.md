## Summary

Keep PR #58823 open. Current main still implements and tests the opposite precedence from this PR in both the shared subagent helper and isolated cron path, while the public subagent docs support the reporter's global-default expectation. The PR also has an unresolved security/allowlist concern, so this needs maintainer product and security judgment rather than conservative cleanup.

## Best Possible Solution

Keep the PR open for maintainer review. Maintainers should decide the intended subagent model precedence contract, then either land a narrow fix in `src/agents/model-selection.ts` and `src/cron/isolated-agent/model-selection.ts` with regression coverage plus model allowlist validation in the subagent spawn path, or keep PR #58003's precedence and update `docs/tools/subagents.md` and related docs before closing #58822/#58823 with that rationale.
