## Summary

Keep PR #58823 open. Current main still implements and tests the opposite precedence from the PR in both the shared subagent helper and isolated cron path, while the public subagents docs support the reporter's expectation. Because the PR reverses merged PR #58003 and now has an unresolved allowlist/security concern, this is not conservative cleanup material; it needs maintainer product/security judgment.

## Best Possible Solution

Keep the PR open for maintainer review. Maintainers should decide the intended subagent model precedence contract, then either land a narrow fix in `src/agents/model-selection.ts` and `src/cron/isolated-agent/model-selection.ts` with regression coverage plus any needed allowlist validation, or keep PR #58003's precedence and update `docs/tools/subagents.md` plus close #58822/#58823 with that rationale.
