## Summary

Keep this PR open. Current main still implements and tests the opposite precedence in both the shared subagent helper and isolated cron path, while the public subagent docs support the global subagent-default expectation. The PR is also paired with open issue #58822 by the same author and has an unresolved allowlist/security concern from the review discussion.

## Best Possible Solution

Keep the PR open for maintainer review. Maintainers should decide the intended subagent model precedence contract, then either land a narrow fix in src/agents/model-selection.ts and src/cron/isolated-agent/model-selection.ts with regression coverage plus allowlist validation in the subagent spawn path, or keep PR #58003's precedence and update docs/tools/subagents.md and related config docs before resolving #58822.
