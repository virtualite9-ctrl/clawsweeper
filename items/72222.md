## Summary

Current main already addresses the reported v2026.4.24 Bonjour failure mode: explicit `plugins.entries.bonjour.enabled: false` is treated as a hard disable before bundled default enablement, disabled plugin records are not runtime-registered or included in the ready plugin list, and the Bonjour advertiser now handles CIAO ANNOUNCEMENT/PROBING cancellations through scoped process handlers that survive watchdog restarts.

## Best Possible Solution

Close this issue as fixed on current main. Keep the shipped main-branch behavior that respects explicit plugin disablement, handles CIAO cancellation/assertion failures inside the Bonjour plugin, and disables the advertiser after repeated stuck restart cycles. Any remaining `sessions_spawn(runtime="acp")` failures should be filed as a narrower ACP/gateway issue against a build containing ddc2036956031f812485d9b6cde0c70a9d699f58.
