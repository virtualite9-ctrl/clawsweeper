## Summary

Current `main` no longer uses the old remote-gateway Chrome extension relay path for this setup. Browser requests now route to a browser-capable node via `browser.proxy`, and the node handles them against its own local browser control service/profile state. The legacy Chrome extension relay config is also explicitly migrated away to Chrome MCP `existing-session` attach on host-local Chrome.
