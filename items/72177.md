## Summary

Keep PR #72177 open. Current `main` does not include the PR’s LiteLLM image-generation hardening: LiteLLM edits still use JSON data URLs, there is no LiteLLM image timeout default, chat-native Gemini/Claude routing is absent, and the agent `image_generate` schema still lets model-supplied `timeoutMs` override operator configuration. The PR is recent, focused, and addresses a real bundled provider surface added by #70246, so it is not obsolete cleanup material.

## Best Possible Solution

Keep the PR open and continue normal review. The likely best path is to refine #72177 on its branch: address the remaining P2 review points, update `docs/tools/image-generation.md` and related LiteLLM docs for the `timeoutMs` contract change, keep the LiteLLM regression tests for multipart edits/chat-native routing/dedup/budget caps, then run the changed extension and agent lanes before merge.
