## Summary

Keep PR #72177 open. Current main does not implement the PR's central LiteLLM image-generation hardening: LiteLLM edits still go through JSON/data URLs, the provider has no 180s default timeout, the agent tool still accepts model-supplied timeoutMs overrides, and there is no LiteLLM chat-completions image route or Gemini duplicate-image dedupe. The PR is recent, focused, contributor-authored, and the discussion shows active follow-up on review findings rather than an obsolete or closeable item.

## Best Possible Solution

Keep #72177 open for normal maintainer review. The best product path is to land this PR or an equivalent focused patch in the LiteLLM provider plus the shared image_generate tool/docs: use multipart for LiteLLM edits, give LiteLLM image requests a provider default timeout, stop accepting model-authored image timeout overrides if that is the intended contract, add a robust chat-completions path for chat-native LiteLLM image models, and cover the new parsing/dedupe behavior with regression tests before running the relevant changed lanes.
