## Summary

Keep open. Current main still advertises Slack `dmHistoryLimit` in config/docs, but the Slack channel monitor does not carry or consume that setting, and 1:1 DM prompt history remains gated out by `isRoomish`. The related PR #67054 directly attempted the right class of fix, but it is closed and unmerged, so it does not supersede this issue.

## Best Possible Solution

Keep this issue open and land a focused Slack monitor fix: derive `account.config.dmHistoryLimit`, carry it through `SlackMonitorContext`, compute an effective history limit for `isDirectMessage` versus room/group contexts, record DM pending history, use that limit for `combinedBody`, `InboundHistory`, and history clearing, and add a regression test for a 1:1 DM with `dmHistoryLimit > 0`. After the Slack behavior is fixed, consider a follow-up for the broader cross-channel parity concern raised in the discussion.
