## Summary

Keep #17840 open. Current main still has reaction notification settings, but no opt-in mode that makes reactions wake the agent. Slack, Telegram, Discord, Signal, and Matrix reaction paths enqueue system events only; those events are drained into a later reply/heartbeat prompt. Feishu has a channel-specific synthetic inbound reaction path, but that does not solve the broader Slack/Telegram/Discord request. The overlapping #56708 should remain secondary because #17840 has the richer discussion and an existing maintainer-facing keep-open comment.

## Best Possible Solution

Keep #17840 open as the canonical product request. Implement a documented opt-in channel/plugin wake mode such as reactionTriggersTurn: "off" | "own" | "all" | "allowlist"; after existing reaction filtering and authorization, enqueue the system event and request a scoped heartbeat/agent turn for that session. Update config schemas, channel docs, and focused Slack/Telegram/Discord reaction tests while preserving the passive default. Treat #56708 as an overlapping duplicate candidate rather than the canonical tracker.
