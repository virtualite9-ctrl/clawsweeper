## Summary

Keep #17840 open. Current main still treats inbound reactions as queued system-event context, not as an opt-in wake source for an agent turn, and the issue has concrete Slack, Telegram, and Discord workflows plus a recent maintainer keep-open comment.

## Best Possible Solution

Keep #17840 open as the canonical product request unless maintainers explicitly select #56708 or another tracker. Implement an opt-in per-channel/plugin setting such as reactionTriggersTurn: "off" | "own" | "all" | "allowlist", run it after existing reaction authorization/notification filtering, enqueue the reaction system event, then request a scoped heartbeat/agent turn using the existing wake primitive without changing the passive default.
