## Summary

Keep #17840 open. Current main still supports reaction notification queueing, not opt-in reaction-triggered agent turns. The issue has concrete Slack, Telegram, Discord, and Signal/Matrix-style use cases plus a recent maintainer comment explicitly keeping it open. Related #56708 overlaps, but it is newer and not a better canonical tracker.

## Best Possible Solution

Keep #17840 open as the canonical product request unless maintainers explicitly choose another tracker. Implement an opt-in per-channel/plugin setting such as reactionTriggersTurn: "off" | "own" | "all" | "allowlist"; apply it after existing reaction authorization/notification filtering, enqueue the reaction system event, then request a scoped heartbeat/agent turn with the existing wake primitive. Update channel schemas, docs, and focused reaction tests while preserving the passive default. Treat newer #56708 as an overlapping duplicate candidate rather than a reason to close this richer, older issue.
