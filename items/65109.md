## Summary

Keep open. Current main still displays cache read/write usage in `/status`, but the displayed cost is estimated from input/output tokens only, so the reported undercount remains actionable. An open PR with closing syntax exists, so this issue also should not be closed before a real runtime fix is merged.

## Best Possible Solution

Keep this issue open. Patch `src/status/status-message.ts` so both `hasUsage` and the `estimateUsageCost` usage object include `cacheRead` and `cacheWrite`, then add a focused regression in `src/auto-reply/status.test.ts` with nonzero cache read/write pricing. Review #65381 as the likely runtime implementation candidate; do not let #72519 close the issue unless it changes from README-only into a real fix.
