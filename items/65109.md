## Summary

Keep open. Current main still displays cache read/write tokens in `/status`, but the `/status` cost calculation only forwards input/output tokens to the shared estimator. The estimator and docs already support cache pricing, and related open PRs are implementation candidates rather than evidence that the issue is fixed.

## Best Possible Solution

Keep this issue open. Patch `src/status/status-message.ts` so `hasUsage` and the `estimateUsageCost` usage object include `cacheRead` and `cacheWrite`, then add a focused regression in `src/auto-reply/status.test.ts` with nonzero cache read/write pricing. Review #65381/#52329 as the likely implementation candidates and close the README-only #72519 or require it to become an actual runtime fix.
