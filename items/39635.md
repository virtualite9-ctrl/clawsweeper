## Summary

Current main does not implement `agents.defaults.extraBootstrapFiles` or arbitrary bootstrap-file injection. OpenClaw now has a hook-based workaround for extra bootstrap files, but it is limited to the existing recognized bootstrap basenames and does not satisfy the requested `CONTEXT_BRIEF.md`-style config support.

## Evidence

- **Agent defaults schema has no extraBootstrapFiles key:** `AgentDefaults` exposes `contextInjection`, `bootstrapMaxChars`, `bootstrapTotalMaxChars`, and `bootstrapPromptTruncationWarning`, but no `extraBootstrapFiles` field.
  - file: [src/config/types.agent-defaults.ts:229](https://github.com/openclaw/openclaw/blob/cb2c36b04975e34ab7014a11575d6589aa96ae32/src/config/types.agent-defaults.ts#L229)

- **User docs list bootstrap knobs but not extraBootstrapFiles:** Current config docs document `agents.defaults.contextInjection`, `bootstrapMaxChars`, and `bootstrapTotalMaxChars`; there is no documented `agents.defaults.extraBootstrapFiles` option.
  - file: [docs/gateway/config-agents.md:70](https://github.com/openclaw/openclaw/blob/cb2c36b04975e34ab7014a11575d6589aa96ae32/docs/gateway/config-agents.md#L70)

  - file: [src/agents/workspace.ts:171](https://github.com/openclaw/openclaw/blob/cb2c36b04975e34ab7014a11575d6589aa96ae32/src/agents/workspace.ts#L171)

  - file: [src/agents/workspace.ts:604](https://github.com/openclaw/openclaw/blob/cb2c36b04975e34ab7014a11575d6589aa96ae32/src/agents/workspace.ts#L604)

## Risks / Open Questions

- There is a partial workaround via the bundled `bootstrap-extra-files` hook, so maintainers may want to retitle/reframe the issue around arbitrary filenames rather than extra bootstrap injection generally.
- The checkout appears shallow/grafted, so history-based provenance for when the workaround landed is limited in this clone.
- `gh` inspection was unavailable in this environment because `GH_TOKEN` was not set, but the repository evidence was sufficient to determine current-main behavior.
