## Summary

Keep #39635 open. Current main still does not implement `agents.defaults.extraBootstrapFiles` or a per-agent equivalent, and the existing bundled `bootstrap-extra-files` path still only accepts recognized bootstrap basenames, so arbitrary files like `CONTEXT_BRIEF.md` remain unsupported through the requested config surface.

## Best Possible Solution

Keep #39635 open. A complete fix should add a strict typed/schema-backed config field under `agents.defaults` and a per-agent override, resolve configured paths relative to the agent workspace with the existing boundary/realpath guards, inject custom files after standard bootstrap files in configured order, apply existing per-file and total bootstrap budgets, document the public config, update generated schema/help, and add focused tests for missing files, invalid paths, ordering, custom basenames, and default-vs-agent override behavior.
