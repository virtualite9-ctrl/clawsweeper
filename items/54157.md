## Summary

Current main still has the reported Doubao/BytePlus cache pricing gap. The provider catalogs now live under bundled plugins, but both `DOUBAO_DEFAULT_COST` and `BYTEPLUS_DEFAULT_COST` still set `cacheRead` and `cacheWrite` to `0`, and the provider builders pass those costs into every Volcengine/BytePlus model definition. Cost estimation multiplies cached-token usage by these cost fields, so cached tokens still price as free for these catalogs. There is also an open implementation PR, #55913, with explicit closing syntax for this issue; per maintainer policy the issue should stay open until that PR is reviewed/landed or closed.

## Best Possible Solution

Keep this issue open and review/update PR #55913 for the current `extensions/volcengine/models.ts` and `extensions/byteplus/models.ts` layout. The fix should set source-backed non-zero `cacheRead` and `cacheWrite` prices for both catalogs and add regression tests in the current plugin test location so cached-token cost tracking cannot regress.
