## Summary

Current main still has the reported zero context-cache pricing for both bundled Volcengine/Doubao and BytePlus provider catalogs. The linked PR #55913 is open, unmerged, and uses closing syntax for this issue, so this issue should stay open until that implementation candidate is reviewed and either landed or closed.

## Best Possible Solution

Keep this issue open and review PR #55913 as the implementation candidate. The fix should update `extensions/volcengine/models.ts` and `extensions/byteplus/models.ts` with vendor-backed nonzero `cacheRead` and `cacheWrite` pricing, adjust any related per-million-token prices only where source-backed, and add focused regression coverage in the bundled provider plugin tests.
