## Summary

Keep open. Current main still returns from the `sessions_send` timed-wait timeout branch before starting the A2A announce flow, so the reported behavior remains actionable and is not covered by the related closed ClawSweeper items.

## Best Possible Solution

Keep the issue open. Update `src/agents/tools/sessions-send-tool.ts` so the `result.status === "timeout"` branch starts `startA2AFlow(undefined, runId)` before returning the timeout response, while preserving the existing `skipA2AFlow` guard. Add a regression test that mocks `agent.wait` timeout for the synchronous call and verifies the background A2A flow waits on the run id and reaches the announce step.
