## Summary

Current main already uses a different Windows browser-launch path than the reported 2026.3.2 behavior: it resolves a concrete browser executable before spawn, with Windows-specific resolution and tests covering direct `.exe` selection. The old `spawn UNKNOWN` report is obsolete on current main.

## Close Comment

Closing as implemented on current main after a Codex review. The current browser launch path no longer depends on the old Windows behavior from `2026.3.2`: `extensions/browser/src/browser/chrome.ts:267` resolves a browser executable before spawn, and the Windows resolver in `extensions/browser/src/browser/chrome.executables.ts:299` and `extensions/browser/src/browser/chrome.executables.ts:582` now returns direct `chrome.exe` / `msedge.exe` / `brave.exe` paths from registry or known install locations. The Windows coverage in `extensions/browser/src/browser/chrome.test.ts:248` also asserts direct `.exe` resolution. This implementation shipped by `v2026.3.28` (`8eeb7f0829754cb3446a3bc7279fbf48d38edd3c`), so the old `spawn UNKNOWN` report is no longer actionable against current main. If you still see a startup failure on a current release, please open a new issue with the exact version, browser installed, and the full error/log output.
