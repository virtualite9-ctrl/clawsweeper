## Summary

Keep #70443 open. Current main still has no `--offline` flag or equivalent Docker setup path: `scripts/docker/setup.sh` always builds `openclaw:local` or pulls the configured image before continuing, and sandbox bootstrap still builds `openclaw-sandbox:bookworm-slim` when enabled. The request is recent, concrete, repo-owned Docker setup work, and related platform items do not implement or supersede the offline rerun behavior.

## Best Possible Solution

Implement the requested `--offline` flag in `scripts/docker/setup.sh`: parse it early, skip the main build/pull branch only when set, require `docker image inspect "$OPENCLAW_IMAGE"` for the gateway image with a clear `docker load` remediation, skip sandbox image building under offline mode with a warning if `openclaw-sandbox:bookworm-slim` is absent, and leave env sync, chown, onboarding, gateway config sync, compose startup, and sandbox overlay handling intact. Add focused e2e tests in `src/docker-setup.e2e.test.ts` and document the flag in `docs/install/docker.md`.
