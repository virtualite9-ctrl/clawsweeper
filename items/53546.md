## Summary

Closing because the reported WeakMap/sessionManager pollution path does not match current `main`: production code creates a fresh `SessionManager` for each turn/compaction run before wiring compaction safeguard runtime, and `model-snapshot` is not used as a model-selection source elsewhere.

Closing as `cannot_reproduce` after a Codex review against current `main` (`8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf`). The reported root cause does not match the current code: compaction safeguard runtime is only wired in `src/agents/pi-embedded-runner/extensions.ts:100`, and both the normal turn path and the compaction path create a fresh `SessionManager.open(...)` before they call that code (`src/agents/pi-embedded-runner/run/attempt.ts:1059` and `src/agents/pi-embedded-runner/compact.ts:804`). Background maintenance also drops `sessionManager` in background mode instead of reusing the foreground object (`src/agents/pi-embedded-runner/context-engine-maintenance.ts:325`). With no deterministic repro in the report and no current-main evidence for the described sessionManager-scoped pollution path, this one is being closed.
