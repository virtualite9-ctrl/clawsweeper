## Summary

Keep open. Current `main` does not implement this PR's allowlist respawn policy: normal commands like `status` still take the respawn path, and the test suite still asserts that behavior.

## Evidence

- **Current respawn policy is still help/version-only:** `shouldSkipRespawnForArgv()` on `main` returns only `resolveCliArgvInvocation(argv).hasHelpOrVersion`, with no command allowlist for `gateway`/`daemon`/`agent`/`memory`.
