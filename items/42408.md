## Summary

Keep this issue open. Current main partially covers path drift diagnostics for missing or inaccessible extraPaths, but the requested exclusion mechanism and specific path-hygiene/benchmark-contamination guidance are still not implemented for builtin local memory indexing.

## Best Possible Solution

Keep open and handle it in memory-core/config/docs. A complete fix would add a documented exclusion mechanism for builtin memory indexing, thread it through config schema/types, listMemoryFiles and watcher/index state, add regression tests for excluded benchmark/queryset files, and expand the public memory search docs with path hygiene, resolved extraPaths checks, reindex-after-path-change guidance, and contamination pitfalls. QMD pattern support can be mentioned as a QMD-specific alternative, but it does not solve builtin local extraPaths exclusions.
