## Summary

Keep this issue open. Current main still only partially covers the report: `memory status --deep` reports missing/inaccessible `extraPaths`, but there is still no builtin `excludePaths` or glob exclusion surface for local memory indexing, and the public memory docs still do not cover path hygiene, broad `extraPaths` noise, or benchmark/queryset contamination.

## Best Possible Solution

Keep open and implement this in the bundled memory-core path plus shared config/docs. A complete fix should add a documented exclusion mechanism, thread it through schema/types/help, `resolveMemorySearchConfig`, `listMemoryFiles`, read access, watcher setup, and reindex scope hashing, then add regression tests proving benchmark/queryset Markdown under `extraPaths` is excluded. Public docs should add path-hygiene guidance for absolute/workspace-relative `extraPaths`, `memory status --deep` verification, reindex-after-path-change, broad-path noise, and benchmark/evaluation contamination.
