## Summary

Keep this issue open. Current main still only partially covers the report: `memory status --deep` can report missing or inaccessible `extraPaths`, but the bundled local memory indexer still has no documented `excludePaths` or glob exclusion surface, and the public memory docs still do not cover path-hygiene, broad-extraPath noise, or benchmark/queryset contamination guidance.

## Best Possible Solution

Keep open and implement this in the bundled memory-core plugin plus shared memory config/docs. A complete fix should add a documented exclusion mechanism, thread it through config schema/types, `resolveMemorySearchConfig`, `listMemoryFiles`, read access, watcher setup, and reindex scope hashing, then add regression tests proving benchmark/queryset Markdown under `extraPaths` is excluded. Public docs should add path-hygiene guidance for absolute/workspace-relative `extraPaths`, `memory status --deep` verification, reindex-after-path-change, broad-path noise, and benchmark/evaluation contamination.
