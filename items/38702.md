## Summary

Current main still does not satisfy #38702. WebChat has only a local, send-time input stack and ArrowUp/ArrowDown handling while the composer is empty. It does not build history from current session user messages, preserve and restore a non-empty draft, or apply the requested modifier/IME/selection/caret guards. The related PR #38709 is the right canonical implementation path, but the provided GitHub context shows it is still open and unmerged.

## Best Possible Solution

Keep #38702 open until PR #38709 or an equivalent focused change lands. The implementation should derive in-session recall entries from current user-role text messages, preserve and restore the active draft, enforce modifier/IME/selection/caret guards, reset safely around loading and reload boundaries, and add focused UI/controller tests for ArrowUp/ArrowDown behavior.
