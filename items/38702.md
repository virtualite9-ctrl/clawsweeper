## Summary

Current main still only has partial WebChat input-history behavior. It stores locally submitted drafts and handles ArrowUp/ArrowDown only while the composer is empty, so it does not satisfy the requested shell-style history mode, draft restoration, caret/selection/modifier/IME guards, or history built from current session user messages. The linked PR #38709 is relevant but remains open and unmerged in the provided GitHub context.

## Best Possible Solution

Keep #38702 open until PR #38709 or an equivalent focused change lands. The implementation should derive in-session history from current user-role text messages, preserve and restore the active draft, enforce modifier/IME/selection/caret guards, reset safely around loading/reload boundaries, and add focused UI/controller tests for ArrowUp/ArrowDown behavior.
