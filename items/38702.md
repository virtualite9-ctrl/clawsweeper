## Summary

Current main still only has partial WebChat input-history recall. ArrowUp/ArrowDown support is tied to an empty draft and a local send-time stack; it does not implement session-message-derived history, draft restoration, caret/selection/modifier/IME guards, or focused regression tests requested by #38702. The related PR #38709 is still open/unmerged in the provided GitHub context, so the issue should remain open.

## Best Possible Solution

Keep #38702 open until PR #38709 or an equivalent focused change lands. The implementation should derive in-session recall entries from current user-role text messages, preserve and restore the active draft, enforce modifier/IME/selection/caret guards, avoid stale recall around loading/reload boundaries, and add focused UI/controller tests for ArrowUp/ArrowDown behavior.
