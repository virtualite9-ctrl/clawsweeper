## Summary

Keep open. Current main has partial WebChat input history, but it only works from an empty draft and does not implement the requested draft restore, session-message-derived history, or unambiguous ArrowUp/ArrowDown guards. The related PR #38709 is useful and explicitly tracks hardening for this issue, but it is still open/unmerged.

## Evidence

- **Partial history helper exists:** InputHistory stores trimmed pushed strings, has up/down cursor navigation, and resets the cursor, but it has no captured draft state and no API for deriving entries from rendered/current session messages.
  - file: [ui/src/ui/chat/input-history.ts:7](https://github.com/openclaw/openclaw/blob/81a41fe5bee350c6a035bfcc458eb8be380afe2c/ui/src/ui/chat/input-history.ts#L7)
  - command: `nl -ba ui/src/ui/chat/input-history.ts | sed -n '1,260p'`
  - sha: [81a41fe5bee3](https://github.com/openclaw/openclaw/commit/81a41fe5bee350c6a035bfcc458eb8be380afe2c)
- **Composer only recalls history when draft is empty:** The WebChat keydown handler gates history navigation behind `!props.draft.trim()`, so ArrowUp cannot enter history from a non-empty draft at the caret boundary and ArrowDown cannot restore that draft at the newest position.
  - file: [ui/src/ui/views/chat.ts:956](https://github.com/openclaw/openclaw/blob/81a41fe5bee350c6a035bfcc458eb8be380afe2c/ui/src/ui/views/chat.ts#L956)

- **Arrow history lacks requested guard checks:** The ArrowUp/ArrowDown history branch does not check modifier keys, IME composition, selection range, or textarea caret start/end before handling history. IME handling exists only later for Enter send handling.
  - file: [ui/src/ui/views/chat.ts:956](https://github.com/openclaw/openclaw/blob/81a41fe5bee350c6a035bfcc458eb8be380afe2c/ui/src/ui/views/chat.ts#L956)
