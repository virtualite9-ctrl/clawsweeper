## Summary

Current `main` already has the Discord native-command plumbing and regression coverage for `/name subcommand...` plugin commands, plus explicit handling for expired Discord interactions. That matches the reported `/clip status` / `/clip models` failure mode closely enough to close as implemented on main.

## Evidence

- **Discord native commands rebuild `/name args` before dispatch:** `createDiscordNativeCommand(...).run()` parses native args, serializes them back into a raw command body, and passes the rebuilt prompt into `dispatchDiscordCommandInteraction()`. That is the path needed for `/clip status`-style plugin commands.
