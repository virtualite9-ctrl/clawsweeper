## Summary

Keep open. The linked transcript-redaction leak from #42982 was fixed on current main by a different maintainer commit, but this PR's stated additive-default behavior is not implemented on main and the PR is now incomplete against the newer transcript guard path.

## Best Possible Solution

Revise rather than auto-close: decide the `logging.redactPatterns` contract, then implement one shared config-derived pattern resolver used by `resolveRedactOptions`, `redactToolDetail`, and persisted transcript redaction. Keep explicit caller-supplied patterns intentionally explicit if that is the API contract, update docs/schema help, and add tests proving custom config patterns do not drop default token/PEM redaction in each affected sink.
