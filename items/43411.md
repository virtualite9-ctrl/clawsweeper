## Summary

Keep this PR open. Current main still does not implement the additive `logging.redactPatterns` behavior the PR targets: configured patterns replace the built-in defaults in the shared resolver, tool-detail redaction, and the current transcript-redaction wrapper. The linked leak symptom has partial main coverage now, but the public contract is still inconsistent between docs and generated config help, so this PR is not obsolete or safely closeable.

## Best Possible Solution

Keep this PR open for maintainer review or contributor revision. Decide the intended public contract for `logging.redactPatterns`, implement one shared config-derived pattern resolver used by `resolveRedactOptions`, `redactToolDetail`, log tailing, and persisted transcript redaction, preserve explicit caller-supplied pattern semantics if intended, then align docs/schema help and add regression tests proving custom config patterns do not drop default token/PEM redaction across affected sinks.
