## Summary

Keep open. Current `main` still exposes model-visible `security`/`ask` on the `exec` tool schema, still blends those per-call values into the effective policy with `minSecurity`/`maxAsk`, and the latest shipped release `v2026.4.23` still documents `security`/`ask` as `exec` tool parameters. That means the behavior this PR is trying to remove is not already implemented on `main`, and this is a core security-policy change that should not be auto-closed conservatively.
