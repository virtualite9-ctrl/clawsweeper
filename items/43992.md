## Summary

Keep open. Current main still does not implement this PR's central behavior: the gateway stores agent command results in the dedupe payload, but `agent.wait` terminal snapshots and responses still drop that result, and the subagent registry/announce paths still rely on chat-history capture rather than an `agent.wait` result fallback. The related original bug was closed by broader history/tool-result fallback behavior, but this PR targets a narrower remaining race where history has not caught up yet.

## Best Possible Solution

Keep this PR open or replace it with an updated patch against current main. The best fix is to thread structured `result` through `AgentWaitTerminalSnapshot`, cached/live `agent.wait` responses, and subagent wait result types; extract visible text from normal agent result shapes such as `{ payloads, meta }` and payload arrays; use that text only when no chat-history `frozenResultText` exists; preserve the result-bearing dedupe path when lifecycle resolution wins the race; and add focused gateway plus subagent regression coverage.
