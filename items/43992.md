## Summary

Keep open. Current main still does not implement the PR's central behavior: gateway `agent.wait` terminal snapshots and responses drop the stored `result` payload, and subagent completion paths still consume only status/timing/error instead of using result text as a fallback when transcript history has not caught up.

## Best Possible Solution

Keep this PR open or replace it with an updated patch against current main. The best fix is to thread structured `result` through `AgentWaitTerminalSnapshot`, cached/live `agent.wait` responses, and `AgentWaitResult`; parse structured agent result payloads; use that text only as a fallback when no chat-history `frozenResultText` exists; handle the lifecycle-before-dedupe race; and add focused gateway plus subagent regression coverage.
