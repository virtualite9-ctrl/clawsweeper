## Summary

Keep open. Current main still does not implement this PR's central behavior: `agent.wait` terminal snapshots and responses still omit the stored agent result, and the subagent announce/registry paths still rely on chat-history capture instead of an `agent.wait` result fallback. The PR discussion remains relevant: the author addressed the earlier overwrite concern, but the remaining result-shape and lifecycle/dedupe race concerns still need a maintainer-quality patch against current main.

## Best Possible Solution

Keep this PR open or replace it with an updated patch against current main. The best implementation is to thread structured `result` through `AgentWaitTerminalSnapshot`, cached/live `agent.wait` responses, and `AgentWaitResult`; extract visible text from the normal agent result payload shapes; use that text only when no chat-history `frozenResultText` exists; handle the lifecycle-before-dedupe race; and add focused gateway plus subagent regression coverage.
