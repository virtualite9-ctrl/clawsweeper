## Summary

Keep PR #70629 open. Current main still has the exact hardcoded `minimal: 128` Google thinking-budget values the PR changes, including the `gemini-2.5-flash-lite` path via the `2.5-flash` substring match. There is no equivalent `minimal: 512` implementation or regression test on main, and the latest release tag still has the old constants. This PR is therefore not obsolete or safely closeable, though maintainers should verify the model-family scope because current Google docs list different minimums for Gemini 2.5 Pro/Flash versus Flash-Lite. ([docs.cloud.google.com](https://docs.cloud.google.com/vertex-ai/generative-ai/docs/thinking))

## Best Possible Solution

Keep #70629 open for maintainer review. The best path is to verify the reported live API rejection against the direct Google Generative AI API, then either merge an equivalent transport-stream fix with the PR's regression test or narrow the change to only the affected Gemini 2.5 model family if Google's current documented ranges are authoritative.
