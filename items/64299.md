## Summary

Keep this issue open. Current main still leaves the reporter-verified Feishu document write/append failure actionable: `write` and `append` still depend on `document.convert`, public/manual Feishu setup guidance still omits `docx:document.block:convert`, and the small descendant-create mapper can still overwrite normalized `children` with the raw converted block value. The two related fix PRs were closed unmerged, so there is no shipped implementation to close against.

## Best Possible Solution

Keep this issue open and make a focused Feishu plugin cleanup: update the public Feishu docs and manual credential prompt with the document-writing scopes, explicitly mention republishing the Feishu app after scope changes, add a clearer diagnostic for `document.convert` permission failures if feasible, and fix the small descendant mapper so normalized `children` cannot be overwritten, with a narrow regression test for string-valued converted children.
