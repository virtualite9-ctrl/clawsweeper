## Summary

Keep this issue open. Current main still has `feishu_doc write` and `append` on the `document.convert` path, while the public Feishu setup docs and manual credential prompt still omit the `docx:document.block:convert` scope the reporter verified as required. A related payload-normalization concern is also only partially addressed: the large-document batch helper preserves normalized `children`, but the small insert path can still spread raw `children` after normalization.

## Best Possible Solution

Keep the issue open and make it a focused Feishu plugin cleanup: add `docx:document.block:convert` to the public Feishu setup guidance and manual credential help, mention that the app must be republished after adding scopes, and consider a clearer runtime diagnostic for `document.convert` permission failures. Separately, fix `toDescendantBlock()` on the small insert path so normalized `children` cannot be overwritten by raw converted block data, with a narrow regression test for string-valued `children`.
