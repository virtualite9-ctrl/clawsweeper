## Summary

Keep this issue open. The reporter verified that adding and publishing the Feishu `docx:document.block:convert` scope fixed the immediate 400 for `write` and `append`, but current main still leaves the repo-side setup gap: public Feishu setup docs and the manual credential prompt do not name that required scope. The related descendant-payload theory also remains unmerged for the small insert path, so main does not yet have enough source/docs evidence to close as implemented.

## Best Possible Solution

Keep the issue open and make it a focused Feishu setup/docs cleanup: add `docx:document.block:convert` to the public Feishu channel setup guidance and the manual credential help, and consider a clearer runtime diagnostic when `document.convert` fails for missing scope. Separately, validate or revive the descendant `children` normalization fix with a narrow regression test before closing the payload side as resolved.
