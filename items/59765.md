## Summary

Current `main` already strips thinking tags before visible reply assembly, detects when the post-processed assistant reply is empty, retries once on the strict-agentic GPT-5 lane, and otherwise surfaces an explicit user-visible error instead of silently succeeding.

## Evidence

- **Thinking tags are stripped before visible reply text is assembled:** Streaming and final assistant text both pass through `stripBlockTags(...)`, so `<think>`/`<thinking>` content is removed before `assistantTexts` / final visible text are used.

  - file: [src/agents/pi-embedded-subscribe.handlers.messages.ts:641](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/src/agents/pi-embedded-subscribe.handlers.messages.ts#L641)

## Risks / Open Questions

- The automatic retry path is limited to the strict-agentic GPT-5 lane; other providers still surface a visible error rather than rotating through a fallback model chain.
