## Summary

Keep open. Current main does not implement Discord edit-to-reprocess or delete-to-cancel: the bundled Discord plugin only registers create/reaction/thread/presence listeners, and there is no messageUpdate/messageDelete listener or message-id-to-active-run abort mapping. Related #38505 remains open and covers only the messageUpdate half, so it does not supersede this broader request.

## Best Possible Solution

Keep this issue open and implement it in the bundled Discord plugin: add Carbon gateway listeners for message updates and deletes, share the existing Discord preflight/message handler for edited content with mention/content-change filtering, persist or track source Discord message id to routed session/run state, and use the existing session/chat abort surface to cancel active work when the triggering message is deleted. Add focused tests for listener registration, edit mention gating, completed-vs-active edit behavior, and delete cancellation.
