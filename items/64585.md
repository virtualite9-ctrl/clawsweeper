## Summary

Keep open. Current `main` has partial restart recovery, but it does not implement the issue’s requested startup scan/replay for stuck gateway chat sessions. Startup only replays queued restart continuations and flagged `abortedLastRun` main sessions, and the flagged-session recovery still rejects assistant tails with no visible text instead of resuming them.

## Evidence

- **Startup recovery on main is narrower than the issue asks for:** `activateGatewayScheduledServices()` only starts outbound delivery recovery plus `recoverPendingRestartContinuationDeliveries()`. Separately, post-attach schedules `scheduleRestartAbortedMainSessionRecovery()`. I did not find a gateway startup pass that scans session stores by transcript tail and replays stuck sessions through `sessions.send` or equivalent generic gateway-session recovery.
  - file: [src/gateway/server-runtime-services.ts:71](https://github.com/openclaw/openclaw/blob/89755d1c79678610720cdb9ce504cfd62c261f64/src/gateway/server-runtime-services.ts#L71)
  - command: `nl -ba src/gateway/server-runtime-services.ts | sed -n '55,145p'; nl -ba src/gateway/server-startup-post-attach.ts | sed -n '350,372p'`
  - sha: [89755d1c7967](https://github.com/openclaw/openclaw/commit/89755d1c79678610720cdb9ce504cfd62c261f64)
- **Main-session recovery only resumes user/tool/toolResult tails:** `isResumableTailMessage()` returns true only for `user`, `tool`, and `toolResult`, and `recoverStore()` marks any other tail as failed with reason `transcript tail is not resumable`. That does not match the issue’s requested recovery for assistant turns with no visible text and no tool call.
  - file: [src/agents/main-session-restart-recovery.ts:60](https://github.com/openclaw/openclaw/blob/89755d1c79678610720cdb9ce504cfd62c261f64/src/agents/main-session-restart-recovery.ts#L60)
  - command: `nl -ba src/agents/main-session-restart-recovery.ts | sed -n '40,230p'`
  - sha: [89755d1c7967](https://github.com/openclaw/openclaw/commit/89755d1c79678610720cdb9ce504cfd62c261f64)
- **Current transcript utilities can detect invisible assistant text/tool-call state, but recovery does not use them:** `extractAssistantVisibleText()` only returns final-answer/unphased visible assistant text, and the preview reader falls back to tool-call names when visible assistant text is absent. Those are the same kinds of transcript-tail signals the issue proposes, but the restart recovery path above does not consult them.
  - file: [src/shared/chat-message-content.ts:182](https://github.com/openclaw/openclaw/blob/89755d1c79678610720cdb9ce504cfd62c261f64/src/shared/chat-message-content.ts#L182)
