## Summary

Keep open. Current `main` still appears to reject the reported WebSocket `card.action.trigger` shape: the parser requires `operator.user_id`, `context.open_id`, `context.user_id`, and `context.chat_id`, while Feishu is documented here with WebSocket as the default transport and the existing tests only cover the stricter legacy-shaped payload.

## Evidence

- **Strict card-action parser still rejects missing fields:** `parseFeishuCardActionEventPayload()` requires non-empty `token`, `operator.open_id`, `operator.user_id`, `operator.union_id`, `action.tag`, `context.open_id`, `context.user_id`, and `context.chat_id`; otherwise it returns `null` and the event is dropped.
  - file: [extensions/feishu/src/monitor.account.ts:184](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/extensions/feishu/src/monitor.account.ts#L184)
  - command: `nl -ba extensions/feishu/src/monitor.account.ts | sed -n '184,233p'`
  - sha: [6bc0dc8fb671](https://github.com/openclaw/openclaw/commit/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440)
- **Malformed payloads are silently discarded on the WebSocket event path:** The `card.action.trigger` handler logs `ignoring malformed card action payload` when that parser returns `null`, so a valid-but-different WebSocket payload shape would be dropped exactly as reported.
  - file: [extensions/feishu/src/monitor.account.ts:370](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/extensions/feishu/src/monitor.account.ts#L370)
