## Summary

Keep open. Current main still lacks the requested core security/API contract: skills cannot declare sandboxed callable heartbeat functions, heartbeat config and runtime remain prompt/model-turn based, and existing plugin/tool/MCP surfaces do not provide core-enforced remote structured heartbeat invocations that bypass free-form LLM instruction interpretation.

## Best Possible Solution

Keep this open for maintainer product/security design. A complete solution likely needs a versioned skill metadata schema for declared sandboxed functions, parser and docs updates, a remote heartbeat fetch/parse layer that accepts structured invocations, runtime validation of function names and params against declared schemas, execution through existing plugin/tool capability boundaries without sending remote heartbeat content to the model as free-form instructions, and regression tests for allowed calls, unknown calls, schema failures, and prompt-injection non-delivery. ClawHub trust badges can follow once the core contract exists.
