## Summary

Keep open. Current main still does not provide the requested core security/API contract: skills cannot declare sandboxed callable heartbeat functions, heartbeat remains a prompt/model-turn workflow, and existing plugin/MCP/runtime hooks do not enforce structured remote heartbeat invocations outside the LLM instruction path.

## Best Possible Solution

Keep this open for maintainer product/security design. A complete solution likely needs a versioned skill metadata schema for declared sandboxed functions, parser and docs updates, a remote heartbeat fetch/parse layer for structured invocations, runtime validation of function names and params against declared schemas, execution through existing plugin/tool capability boundaries without sending remote heartbeat content to the model as free-form instructions, and regression tests for allowed calls, unknown calls, schema failures, and prompt-injection non-delivery. ClawHub trust badges can follow after the core contract exists.
