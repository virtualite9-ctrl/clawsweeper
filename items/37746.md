## Summary

Keep open. Current main still lacks the requested core security/API contract: skills cannot declare sandboxed heartbeat-callable functions, heartbeat configuration has no structured remote invocation surface, and heartbeat runs still send prompt text to the model rather than validating remote calls outside the LLM instruction channel.

## Best Possible Solution

Keep this open for maintainer product/security design. A complete solution likely belongs in core skill and heartbeat contracts: add a versioned skill metadata schema for declared sandboxed functions, parse and document it, add a structured remote heartbeat fetch/parse layer, validate call names and params against declared schemas, execute through existing plugin/tool capability boundaries without sending remote content to the model as free-form instructions, and add regression tests for allowed calls, unknown calls, schema failures, and prompt-injection non-delivery. ClawHub trust badges can follow once the core contract exists.
