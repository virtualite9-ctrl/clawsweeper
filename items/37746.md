## Summary

Keep open. Current main does not implement a remote-heartbeat sandbox contract: heartbeat config still sends a verbatim prompt centered on local `HEARTBEAT.md`, skill metadata/frontmatter only supports existing gating/install fields plus optional slash-command tool dispatch, and the security docs still describe skill sandboxing as future work. This is also not a `clawhub` close candidate because the repo vision sends new skills to ClawHub, but this request is for a core runtime/security contract.

## Evidence

The documented default is a verbatim prompt telling the agent to read local `HEARTBEAT.md`; there is no remote URL or structured heartbeat invocation field.
