## Summary

Keep open. Current `main` has a bundled Xiaomi model provider, but there is no native Xiaomi speech/TTS provider implementation, no Xiaomi TTS registration in the plugin entry, and the user-facing TTS docs and capability matrix still exclude Xiaomi.

## Evidence

- **Xiaomi plugin is model-only on current main:** The bundled Xiaomi plugin entry defines only `provider` metadata and usage auth/catalog wiring. It does not call `registerSpeechProvider(...)`, unlike providers that support native TTS.
  - file: [extensions/xiaomi/index.ts:8](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/extensions/xiaomi/index.ts#L8)

- **TTS docs do not list Xiaomi:** The TTS guide says OpenClaw TTS supports ElevenLabs, Google Gemini, Microsoft, MiniMax, OpenAI, and xAI. Xiaomi is absent from both the overview sentence and supported-services list.
  - file: [docs/tools/tts.md:10](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/docs/tools/tts.md#L10)
