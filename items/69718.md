## Summary

Keep open. Current main still keeps direct inbound message bodies unprefixed, the regression test asserts that behavior, and the gateway sanitizer has no envelope-header sender fallback or `extractEnvelopeSender` implementation. The linked issue was closed, but this PR is not implemented on main or clearly superseded by a canonical replacement; the supplied PR review comments also identify unresolved correctness risks in the proposed parser.

## Best Possible Solution

Keep the PR open for maintainer judgment. The best implementation path is to preserve direct-message sender identity as structured transcript/display metadata from finalized inbound context, or add a narrow explicit direct-chat fallback with tests, rather than broadly trusting free-form envelope headers without covering timestamp, timezone, elapsed-marker, and group/channel cases.
