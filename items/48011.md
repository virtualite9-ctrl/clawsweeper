## Summary

Current `main` still appears to have the reported gap. In sandbox `workspaceAccess: "rw"`, OpenClaw skips `syncSkillsToWorkspace()`, but the run still loads merged bundled/managed/workspace skill entries and formats their original `skill.filePath` values into the prompt. That leaves non-workspace skills outside the sandbox workspace boundary by default, so the issue should remain open.

## Evidence

- **Sandbox skill sync is skipped for rw mode:** `ensureSandboxWorkspaceLayout()` sets `workspaceDir` to the agent workspace when `workspaceAccess === "rw"` and only calls `syncSkillsToWorkspace()` when `cfg.workspaceAccess !== "rw"`.
