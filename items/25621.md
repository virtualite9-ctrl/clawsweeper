## Summary

Keep open. Current main still limits managed Gateway service adapters to macOS, Linux, and Windows and still throws `Gateway service install not supported on ${process.platform}` for OpenBSD and other unsupported platforms. The read-only `openclaw status` and `openclaw gateway status` paths still resolve the Gateway service before their later status fallbacks can apply, so the reported crash remains actionable. The related fix PR #25690 is useful prior art, but the provided GitHub context shows it was closed unmerged.

## Best Possible Solution

Keep #25621 open until OpenClaw either adds a graceful unsupported-platform `GatewayService` adapter/fallback, guards read-only status resolver calls so `openclaw status` and `openclaw gateway status` work on unsupported platforms, or makes an explicit documented product decision that these platforms must run the gateway manually outside managed service flows. The likely code path is `src/daemon/service.ts` plus regression coverage for unsupported `process.platform` in status paths; PR #25690 remains useful prior art but was closed unmerged.
