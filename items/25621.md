## Summary

Keep open. Current main still throws for unsupported gateway service platforms such as OpenBSD, and both `openclaw status` and `openclaw gateway status` still reach `resolveGatewayService()` before their read-only fallback handling can apply. The linked fix PR #25690 was closed unmerged, and the existing maintainer Codex comment already keeps this issue open on the same current-behavior basis.

## Best Possible Solution

Keep #25621 open until OpenClaw either adds a graceful unsupported-platform GatewayService adapter/fallback, guards read-only status resolver calls so `openclaw status` and `openclaw gateway status` work on unsupported platforms, or makes an explicit documented product decision that these platforms must run the gateway manually outside managed service flows. The likely implementation belongs in `src/daemon/service.ts` plus regression coverage for unsupported `process.platform` in status paths; PR #25690 remains useful prior art but was closed unmerged.
