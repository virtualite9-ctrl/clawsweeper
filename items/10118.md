## Summary

Keep #10118 open. Current main already gets the default Enter-to-submit and Shift+Enter-to-newline behavior from the pinned pi-tui editor and preserves submitted multiline text, but the remaining requested user-facing work is still missing: TUI docs and slash help do not mention Shift+Enter, OpenClaw exposes no configurable send/newline keybinding surface, and related PR #43327 is still open/unmerged in the provided context. A maintainer-posted Apr 26 Codex review also explicitly preserved that remaining scope.

## Best Possible Solution

Keep the issue open and narrow it to the remaining work: document Shift+Enter in docs/web/tui.md and /help, add focused assertions for the visible shortcut text, and decide whether OpenClaw should expose pi-tui keybinding configuration for Enter vs Ctrl/Cmd+Enter send behavior. Treat PR #43327 as related but not canonical while it remains open and unmerged.
