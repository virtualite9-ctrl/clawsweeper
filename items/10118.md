## Summary

Keep #10118 open. Current main has dependency-backed partial multiline support through pinned `pi-tui`, but OpenClaw still does not document Shift+Enter/newline composition in the TUI docs or `/help`, and it exposes no CLI/config bridge for configurable submit/newline keybindings. The latest provided maintainer comment also explicitly kept this issue open, with related PR #43327 still open and unmerged.

## Best Possible Solution

Keep the issue open and narrow it to the remaining product work: document multiline composition and terminal limitations in `docs/web/tui.md` and TUI `/help`, add focused assertions for shortcut/help visibility, and decide whether OpenClaw should expose configurable submit/newline keybindings through config or CLI wiring to `pi-tui` keybindings. Treat PR #43327 as related prior work until it is merged, updated, or explicitly superseded.
