## Summary

Keep #10118 open. Current main partially covers the core input path because the pinned pi-tui editor has Shift+Enter newline and Enter submit defaults, and OpenClaw preserves multiline payloads once the editor produces them. The remaining user-facing request is still not done: TUI docs and slash help do not document Shift+Enter, there is no configurable Enter/Ctrl-Enter send behavior, and the provided discussion includes a maintainer comment explicitly keeping this issue open with related PR #43327 still open/unmerged.

## Best Possible Solution

Keep the issue open and narrow it to the remaining user-visible work: document Shift+Enter in the TUI guide and /help, add focused assertions for shortcut/help visibility, and decide whether OpenClaw should expose configurable Enter versus Ctrl/Cmd+Enter send behavior through config or CLI options. Treat PR #43327 as related prior work while it remains open and unmerged, not as a shipped or superseding fix.
