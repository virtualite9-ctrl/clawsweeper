## Summary

Current main implements the requested fix: TTS tool audio is carried as structured media and local reply media is staged into managed outbound storage, removing the old `/tmp/openclaw/...` allowlist failure described in the issue. The changelog records this exact fix as shipped in `v2026.4.14`.
