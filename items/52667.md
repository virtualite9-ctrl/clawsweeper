## Summary

Close. Current `main` already handles WhatsApp LID mentions by normalizing mention/self identities to comparable `e164`/`jid`/`lid` values and matching via identity overlap, and the latest release snapshot `v2026.4.22` already contains that logic. The report matches older behavior from `2026.3.13`, not current OpenClaw.

## Close Comment

Closing from a Codex review because this is already implemented on current OpenClaw. On `main`, WhatsApp mention detection no longer compares only raw JIDs: `extensions/whatsapp/src/identity.ts:61` normalizes identities across device-scoped JIDs, `@lid`, and E.164, and `extensions/whatsapp/src/auto-reply/mentions.ts:55` treats a mention as matching when the normalized self identity overlaps the normalized mention identity. There is also direct regression coverage for `@lid` reverse mapping in `extensions/whatsapp/src/auto-reply/web-auto-reply-utils.test.ts:116`. The latest release snapshot `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`) already contains this logic, so the `2026.3.13` report is obsolete on supported versions. If this still reproduces on `v2026.4.22` or newer, please open a new issue with fresh `group mention debug` logs and the exact WhatsApp account/group configuration.
