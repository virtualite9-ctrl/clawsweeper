## Summary

Keep #10480 open. Current main still hardcodes the bundled Cloudflare AI Gateway onboarding/catalog path to Anthropic Messages with the Claude Sonnet default, and there is no non-interactive Workers AI model/provider-type selector for `--auth-choice cloudflare-ai-gateway-api-key`. The issue has concrete reproduction context, a community workaround/link to Cloudflare moltworker, and an April 26 maintainer/Codex comment already keeping it open; current main still matches that assessment.

## Best Possible Solution

Keep #10480 open. The best fix belongs in the bundled `extensions/cloudflare-ai-gateway` provider plugin: add an explicit Workers AI mode or non-interactive model selector, generate OpenAI-compatible provider config for `https://gateway.ai.cloudflare.com/v1/<account>/<gateway>/workers-ai/v1` with `@cf/...` model IDs, update Cloudflare onboarding docs, and add focused tests for the generated provider/model config. If maintainers decide the existing `custom-api-key` route is the intended answer, document the exact Workers AI command/config before closing.
