## Summary

Current main still does not implement #10480. The bundled Cloudflare AI Gateway onboarding path remains scoped to Anthropic through AI Gateway, with no non-interactive Workers AI mode, Cloudflare-specific model flag, provider-type selector, or generated `/workers-ai/v1` OpenAI-compatible provider config. The generic `custom-api-key` flow remains a workaround, not the requested Cloudflare onboarding behavior, and the provided issue discussion includes a maintainer comment from 2026-04-26 explicitly keeping this open for the same gap.

## Best Possible Solution

Keep #10480 open. The best implementation is in the bundled `extensions/cloudflare-ai-gateway` provider plugin: add an explicit Workers AI mode or non-interactive model selector for Cloudflare AI Gateway onboarding, generate an OpenAI-compatible provider config using `https://gateway.ai.cloudflare.com/v1/<account>/<gateway>/workers-ai/v1` and `@cf/...` model IDs, update `docs/providers/cloudflare-ai-gateway.md` and CLI onboarding docs, and add focused tests for the generated provider/model config. If maintainers choose the generic custom-provider flow instead, document the exact Workers AI command and config as a product decision before closing.
