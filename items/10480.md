## Summary

Current main still does not support Workers AI model selection for `openclaw onboard --non-interactive --auth-choice cloudflare-ai-gateway-api-key`. The Cloudflare AI Gateway path remains Anthropic-only, while the generic custom-provider flow is only a workaround and does not satisfy the requested Cloudflare-specific onboarding behavior.

## Best Possible Solution

Keep #10480 open. A proper fix should live in the bundled Cloudflare AI Gateway provider plugin: add an explicit Workers AI mode or model selector for non-interactive onboarding, generate an OpenAI-compatible `https://gateway.ai.cloudflare.com/v1/<account>/<gateway>/workers-ai/v1` provider config with `@cf/...` model IDs, update the Cloudflare onboarding docs, and add focused tests for the generated provider/model config. If maintainers decide the generic custom-provider flow is the intended solution, document that product decision with the exact Workers AI command instead of closing this as already implemented.
