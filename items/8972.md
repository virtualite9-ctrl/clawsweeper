## Summary

Keep #8972 open. Current main still implements Discord guild channel allowlists as exact channel id/slug/name matches plus thread-parent fallback and the literal `*` wildcard. There is no glob or `ws-*` pattern matching, and the category alternative remains a separate open request in #60582.

## Best Possible Solution

Keep #8972 open until Discord guild channel allowlists support a documented, tested pattern form such as `ws-*`. The likely implementation path is an explicit opt-in matcher in the shared channel config helper or Discord resolver, with exact-match precedence, thread-parent behavior, schema/types/help/docs/status/audit updates, and regression tests proving dynamically created matching channels are allowed without per-channel config entries. Treat category inheritance as the separate #60582 path unless maintainers choose to merge the designs.
