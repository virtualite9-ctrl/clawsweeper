## Summary

Keep #8972 open. Current main still treats Discord guild channel allowlist keys as exact channel id/name/slug entries, thread-parent fallbacks, or the literal `*` wildcard. A key like `ws-*` is not evaluated as a glob, and the category-based alternative is a separate open request in #60582 rather than a superseding implementation.

## Best Possible Solution

Keep #8972 open. Implement a documented Discord guild channel pattern form such as `ws-*`, preserving exact-match and thread-parent precedence, keeping literal `*` as the broad fallback, updating schema/types/docs/status/audit behavior, and adding regression tests proving dynamically created matching Discord channels are accepted without per-channel config edits. Track category inheritance through #60582 unless maintainers decide to merge the designs.

## Risks / Open Questions

- Adding glob support changes access-control semantics, so exact id/name/slug matches, thread-parent fallback, pattern matches, and the literal `*` fallback need explicit precedence and regression tests.
- The category-based alternative overlaps #60582 and should stay scoped separately unless maintainers intentionally combine both designs.
