## Summary

Keep #8972 open. Current main still treats Discord guild channel allowlist keys as exact id/slug/name entries plus thread-parent fallback and the literal `*` wildcard; a key like `ws-*` is not interpreted as a glob or prefix. The category-based alternative is also not implemented in the Discord config schema/types and remains a separate open request in #60582.

## Best Possible Solution

Keep #8972 open until the Discord guild channel allowlist supports a documented pattern form, with clear matcher precedence, config schema/types/help, docs, status/audit behavior, and regression tests proving `ws-task-1` matches `ws-*` without a per-channel config edit. Treat category inheritance as the separate #60582 path unless maintainers choose to merge the designs.
