## Summary

Current `main` already has Discord typing infrastructure for automatic inbound replies and heartbeat runs, but the requested agent-callable Discord message-tool action is still absent. The related comments point to broader cross-provider typing and previously closed lifecycle bugs, but they do not provide a canonical duplicate or a shipped replacement for `message(action="typing"...)`.

## Best Possible Solution

Keep #30529 open until maintainers either add a shared `typing`/`typing-indicator` message-tool action or explicitly decide automatic typing is sufficient. A concrete implementation should add the shared action name and target mode, expose any needed schema fields, advertise it from Discord `describeMessageTool`, route it through `handleDiscordMessageAction` to `sendTypingDiscord` with account/target resolution, and update `docs/channels/discord.md`, `docs/concepts/typing-indicators.md` if behavior changes, and `skills/discord/SKILL.md`. If cross-provider parity is desired, design the action so Telegram, Slack, Discord, and similar plugins can opt in cleanly.
