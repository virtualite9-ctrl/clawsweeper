## Summary

Current main still does not implement the requested Discord typing indicator action in the shared message tool. Discord has native typing infrastructure for inbound replies and heartbeat targets, but the agent-callable `message(action="typing")` / `typing-indicator` surface remains absent from the shared action list, Discord action discovery, Discord action dispatcher, docs, and Discord skill guidance.

## Best Possible Solution

Keep #30529 open until maintainers either add an explicit shared `typing` / `typing-indicator` message-tool action or deliberately decide that automatic typing is sufficient. A concrete implementation should add the shared action name and target semantics, advertise it from Discord `describeMessageTool`, route it through `handleDiscordMessageAction` to `sendTypingDiscord` using existing account and target resolution, and update message CLI docs, Discord docs/skill guidance, and focused tests. If cross-provider parity is desired, design the shared contract so Telegram, Slack, Discord, and similar plugins can opt in cleanly.
