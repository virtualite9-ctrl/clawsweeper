## Summary

Keep #30529 open. Current main has Discord typing infrastructure for inbound replies and heartbeat delivery, but it still does not expose a `typing` or `typing-indicator` action through the shared `message` tool, which is the explicit surface requested by the issue and follow-up discussion.

## Best Possible Solution

Keep tracking #30529 until maintainers either add an explicit shared `typing`/`typing-indicator` message-tool action or intentionally close the issue as satisfied by automatic typing. A concrete implementation should add the shared action name and target mode, advertise it from Discord `describeMessageTool`, route it through `handleDiscordMessageAction` to `sendTypingDiscord` with existing account/target resolution, and update CLI docs, Discord docs/skill guidance, and focused tests. If cross-provider parity is desired, design the shared contract so Telegram, Slack, Discord, and similar plugins can opt in cleanly.

## Evidence
