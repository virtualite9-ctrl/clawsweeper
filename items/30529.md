## Summary

Current main still does not implement the requested Discord typing indicator as an agent-callable shared message-tool action. Discord has automatic/internal typing support for inbound replies and heartbeats, but `message(action="typing")` / `typing-indicator` is absent from the shared action enum, tool schema, CLI normalization, Discord discovery, Discord dispatcher/runtime, docs, and the Discord skill guidance.

## Best Possible Solution

Keep #30529 open until maintainers either add a shared `typing` / `typing-indicator` message-tool action or explicitly decide automatic typing is the intended product boundary. A concrete implementation should add the shared action name and target semantics, advertise it from Discord `describeMessageTool`, route it through `handleDiscordMessageAction` to the existing Discord typing sender with account and target resolution, and update the message CLI docs plus `skills/discord/SKILL.md`. If cross-provider parity is desired, design the shared action so Telegram, Slack, Discord, and similar plugins can opt in cleanly.
