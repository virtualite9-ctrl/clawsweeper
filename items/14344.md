## Summary

Current main still does not implement WhatsApp message delete/revoke. The bundled WhatsApp plugin advertises and dispatches reactions and polls, but `delete` is neither discovered nor accepted, and the active WhatsApp socket API has no delete/revoke helper.

## Best Possible Solution

Keep #14344 open as the canonical WhatsApp delete/revoke feature request. The implementation should live in the bundled WhatsApp plugin: advertise `delete`, accept it in `supportsAction`, route it through the gateway-owned WhatsApp action runtime, add an active-listener delete/revoke helper using the Baileys socket key, validate bot-owned/fromMe deletion and group participant keys, then add focused send/action tests and update `docs/cli/message.md` plus `docs/channels/whatsapp.md`.
