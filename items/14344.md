## Summary

Keep #14344 open. Current main at 406ae72fd2789c78053f6b94006ec8f488bc1c00 still does not expose WhatsApp message delete/revoke: discovery advertises only react/poll, the WhatsApp action adapter only supports react, the runtime only handles react, the active listener/send API has no delete helper, and docs still exclude WhatsApp from shared delete. The linked #38723 was closed as a duplicate of this canonical request, not shipped.

## Best Possible Solution

Keep #14344 open as the canonical bundled WhatsApp plugin feature request. The best path is to add `delete` to WhatsApp message-action discovery and `supportsAction`, route it through the plugin-owned gateway action runtime, add an active-listener delete/revoke helper using the Baileys message key, enforce bot-owned/fromMe deletion with group participant handling, and cover the runtime/send API with focused tests plus updates to `docs/cli/message.md` and `docs/channels/whatsapp.md`.
