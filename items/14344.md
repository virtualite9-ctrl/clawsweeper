## Summary

Keep #14344 open. Current main at 7e13f3f5140168e31027b39a9855ade3fadf24d5 still does not implement WhatsApp message delete/revoke: the bundled WhatsApp plugin advertises and supports only react/poll message actions, the runtime only handles react, the active listener send API has no delete/revoke helper, and user-facing docs still exclude WhatsApp from delete. The related #38723 is closed and does not supersede this canonical request.

## Best Possible Solution

Keep #14344 open as the canonical bundled WhatsApp plugin feature request. The implementation should advertise delete in WhatsApp message-action discovery, accept it in supportsAction, route it through the gateway-owned WhatsApp action runtime, add an active-listener delete/revoke helper using the Baileys message key, validate bot-owned/fromMe deletion and group participant keys, then add focused action/send tests and update docs/cli/message.md plus docs/channels/whatsapp.md.
