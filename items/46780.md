## Summary

Keep open. Current main has agent/tool/message/session/install plugin hooks, but it still lacks a skill-level before/after invocation lifecycle. The direct skill slash-command path either rewrites the request into a normal model prompt or calls a target tool with `skillName`, without emitting a skill-specific hook. The enterprise permission-check comment is a real API/use-case signal, not something ClawHub can fully solve without a core/plugin hook seam.

## Best Possible Solution

Keep the issue open as a focused plugin/core API design item. The best path is to add a supported skill lifecycle hook surface, likely plugin hooks for skill invocation and completion/failure, with clear semantics for slash-command skills, tool-dispatched skills, and prompt-loaded skills; existing tool hooks and `agent_end` can remain adjacent telemetry but do not fully replace this request.
