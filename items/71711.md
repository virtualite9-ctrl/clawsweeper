## Summary

Keep #71711 open. Current main does not implement a per-recurring-job `activeUntil`/`expiresAt` or `onExpire` lifecycle field: cron job types, gateway schemas, CLI add/edit surfaces, scheduler result handling, docs, and tests all still model recurring `every`/`cron` jobs as continuing until manually disabled or removed. The linked #71712 is a broader open RFC that depends on this narrower capability rather than superseding it.

## Best Possible Solution

Keep #71711 open as a valid core cron feature request. A maintainer-quality implementation would add optional `activeUntil` plus an `onExpire` policy to the cron job contract, gateway validators, CLI add/edit/list/show surfaces, scheduler pre-run/maintenance handling, run/audit events, docs, and regression tests for both `every` and `cron` jobs; then decide how #71712 should depend on or reuse that field.
