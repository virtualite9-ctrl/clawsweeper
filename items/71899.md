## Summary

Keep #71899 open. Current main still uses `resolveAgentWorkspaceDir()` as the `agents add` wizard default, and that resolver still appends the new agent id to any `agents.defaults.workspace`, which reproduces the reported nested `workspace/<id>` behavior when setup stored the main `workspace` directory. The PR remains a plausible focused fix for #71889 rather than obsolete cleanup.

## Best Possible Solution

Keep the PR open and continue reviewing it as the active implementation path for #71889. The best product direction is a narrow resolver change that treats setup-managed main workspace defaults as main workspaces, preserves explicit per-agent workspaces and intentional custom shared bases, documents the peer-level layout, and lands focused regression coverage for the wizard-visible default, profile-transition case, shared-base preservation, and symlink/canonicalization behavior.
