## Summary

Keep #71899 open. Current main still routes `agents add` wizard defaults through `resolveAgentWorkspaceDir()`, and that resolver still appends a non-default agent id to any `agents.defaults.workspace`, including the setup-created main workspace. The linked bug #71889 remains open in the provided context, with reporter and independent commenter reproductions, and the PR discussion shows active author follow-up addressing review findings rather than obsolete or low-signal work.

## Best Possible Solution

Keep the PR open for maintainer review or merge an equivalent fix. The right product/code path is to update `resolveAgentWorkspaceDir()` so setup's managed main workspace default is not treated as a shared base for newly added agents, while preserving explicit `agents.list[].workspace` values and intentional custom shared bases, with resolver and wizard-default regression tests plus the docs/changelog note already proposed in the PR.
