## Summary

Keep #11487 open. Current main still lacks a canonical optional human-readable metadata field on channel group config entries; the relevant strict Zod schemas, TypeScript config types, generated metadata, and public docs remain behavior-only. The existing 2026-04-26 issue comment also kept this open, and the touched config/docs diff since that review does not add the requested group-entry name/label/description field.

## Best Possible Solution

Keep the issue open. The best implementation is a core config-contract change: choose one optional string metadata field for group/channel entries, add it consistently to the relevant channel group-entry TypeScript types and strict Zod schemas, preserve it through config load/write while ignoring it at runtime, regenerate bundled channel config metadata, update the public group/channel docs, and add validation tests proving the field is accepted without changing group behavior.
