## Summary

Keep #72133 open. Current main already has the Control UI metadata and a generic `/usage` reply footer for chat/channel replies, but the native mobile request is not implemented: Android drops message usage entirely, and the shared Swift chat view decodes usage but does not render any per-message footer. The existing channel footer is also only tokens/cache/cost/session, not the requested Control UI-style `% ctx`/model visibility.

## Best Possible Solution

Keep the issue open and implement it as presentation work over existing metadata: add a compact usage/context footer to the shared Swift native chat view, extend Android chat models/parsing/rendering to carry and display usage/cost/model/context data, and augment the existing `responseUsage` channel footer or a new setting so channel replies can optionally include `% ctx` and model in addition to the current tokens/cache/cost line.
