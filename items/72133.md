## Summary

Keep #72133 open. Current main already has the Control UI metadata and channel `/usage` footers, but the requested parity is not implemented across native mobile chat and is only partial for channel replies: iOS decodes usage without rendering it, Android drops usage metadata entirely, and the channel footer lacks model/context-percent visibility.

## Best Possible Solution

Keep the issue open and implement this as product parity work: add a shared usage metadata formatter that can emit compact token/cache/cost/model/context fields, render it in the shared Swift chat UI and Android Compose chat, preserve usage metadata through Android history parsing, and extend `/usage` channel footers with an opt-in full mode that includes model and `% ctx` when the runtime exposes enough context-window data.

## Risks / Open Questions

- Adding context percentage outside Control UI needs a shared definition of prompt/context tokens so mobile and channel surfaces do not show misleading percentages.
- Channel footer defaults should avoid noisy or sensitive metadata in group chats; a session/user toggle is still the right product shape.
