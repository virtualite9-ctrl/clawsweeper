## Summary

Keep PR #69835 open. Current main does not implement the PR's two requested behavior changes: explicit gateway `workspaceDir` is still rejected by protocol validation and dispatch still only forwards inherited spawned-session workspace metadata, while missing workspace bootstrap files still become `missing: true` entries and `[MISSING]` prompt context. The PR is relevant, but not safe to close or merge as-is because the provided review comments identify a real missing authorization guard for accepting explicit `workspaceDir` plus a stale test expectation outside the PR's targeted test run.

## Best Possible Solution

Keep the PR open and request a revised patch: gate explicit `workspaceDir` to trusted internal/admin callers or validated spawned metadata, update the protocol schema/types and gateway tests accordingly, change bootstrap loading/prompt behavior consistently, and update all affected bootstrap/system-prompt tests before running the changed gate.
