## Summary

Keep open. Current `main` does not already implement this PR, and the proposed branch still has two credible blockers from the review discussion: it widens `workspaceDir` override access beyond the existing spawned-run guard, and it changes missing-bootstrap behavior without updating all existing tests that still assert `[MISSING]` entries.

## Evidence

- **Current main still uses spawned-run-only workspace override:** The live gateway handler on `main` only forwards a workspace override through `resolveIngressWorkspaceOverrideForSpawnedRun(...)`, which returns a path only when `spawnedBy` is present. There is no request-level `workspaceDir` field in the current wire schema.

## Risks / Open Questions

- Merging this as-is would expand a filesystem-affecting `workspaceDir` override from spawned-session metadata to arbitrary write-scoped gateway callers without a matching authorization gate.
- The bootstrap-noise change is not fully carried through existing coverage; current `main` still has tests that expect missing bootstrap files to surface as `missing: true`, so the PR is not ready for conservative cleanup closure.
