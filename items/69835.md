## Summary

Keep PR #69835 open. Current main still lacks both requested behavior changes: explicit gateway `workspaceDir` is rejected by protocol validation and dispatch only forwards inherited spawned-session workspace metadata; missing bootstrap files still become `missing: true` entries and `[MISSING]` prompt context. The PR remains relevant, but existing review comments identify real blockers around authorization and stale test expectations.

## Best Possible Solution

Keep the PR open and request a revised patch: split or clearly justify the two behavior changes, gate explicit `workspaceDir` to trusted internal/admin callers or validated spawned metadata, update protocol schema/types and gateway tests, change bootstrap loading/prompt behavior consistently, update all affected bootstrap/system-prompt tests and user-facing docs, then run the changed gate.
