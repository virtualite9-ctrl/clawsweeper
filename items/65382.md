## Summary

Keep open. Current `main` still renders cron-expression schedules through the raw `Cron <expr>` fallback in the shared Control UI presenter, while this PR is a focused UI implementation candidate with tests. A related broader PR (#65199) is open, but it is not merged and does not make this smaller PR obsolete enough for conservative cleanup closure.

## Best Possible Solution

Keep this PR open for normal review. The best path is to either land this narrower shared `formatCronSchedule` formatter after range-validation coverage, or explicitly consolidate it with #65199 if maintainers prefer the broader Dreaming-panel formatter. Required coverage should include `0 * * * *`, `0 */N * * *`, `*/N * * * *`, timezone annotations, unsupported raw fallback, and malformed/out-of-range step fallback.
