## Summary

Keep open. Current main still renders cron-expression schedules as raw `Cron <expr>` in the Control UI, so the PR is not obsolete or implemented elsewhere. The related #65135 report was closed for the exact `*/6` -> `3:00` symptom not reproducing, but the human-readable interval display proposed by this PR remains absent from main.

## Best Possible Solution

Keep this PR open and review it as the implementation candidate. The best path is a small, UI-only formatter in `ui/src/ui/presenter.ts` plus focused `ui/src/ui/presenter.test.ts` coverage for `0 * * * *`, `0 */N * * *`, `*/N * * * *`, timezone annotations, unsupported/raw fallback, and malformed or out-of-range step fallback before merge.
