## Summary

Current `main` does not implement this feature. Sandbox browser support is still Docker-only: non-Docker backends are explicitly rejected, the sandbox browser path still creates and starts Docker containers directly, and the sandbox config surface does not expose external CDP/noVNC endpoint fields.

## Evidence

- **Non-Docker sandbox backends reject browser sandboxes:** `ensureSandboxContext` throws when `sandbox.browser.enabled` is set on a backend whose capabilities do not include browser support.
  - file: [src/agents/sandbox/context.ts:193](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/agents/sandbox/context.ts#L193)
  - command: `nl -ba src/agents/sandbox/context.ts | sed -n '188,210p'`
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **Only Docker advertises browser capability:** The Docker sandbox backend sets `capabilities.browser = true`; there is no corresponding browser capability in the SSH backend implementation.
  - file: [src/agents/sandbox/docker-backend.ts:63](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/agents/sandbox/docker-backend.ts#L63)
  - command: `nl -ba src/agents/sandbox/docker-backend.ts | sed -n '45,75p'`
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **Sandbox browser lifecycle is still hard-wired to Docker:** `ensureSandboxBrowser` begins by inspecting Docker container state, builds Docker create args, publishes loopback ports, and calls `execDocker(... create/start ...)` to manage the browser runtime.
  - file: [src/agents/sandbox/browser.ts:161](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/agents/sandbox/browser.ts#L161)
  - command: `nl -ba src/agents/sandbox/browser.ts | sed -n '150,176p'; nl -ba src/agents/sandbox/browser.ts | sed -n '260,318p'`
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **Config and docs still describe Docker-only sandbox browser support:** `SandboxBrowserSettings` exposes Docker/container fields like `image`, `network`, `cdpPort`, `cdpSourceRange`, `noVncPort`, and `binds`, but no external endpoint fields. The docs still state browser sandboxing is not supported on SSH/OpenShell and describe the sandboxed browser as Chromium+CDP in a container.
  - file: [src/config/types.sandbox.ts:64](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/config/types.sandbox.ts#L64)
