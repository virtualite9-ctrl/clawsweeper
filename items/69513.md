## Summary

Keep #69513 open. Current main still validates cron announce delivery only at the channel level; `delivery.accountId` is only trimmed/shape-checked and then preserved into delivery resolution. The related in-repo fix PR #69518 was closed unmerged, and the other linked PR is in a fork, so there is no shipped or canonical OpenClaw fix to close against.

## Best Possible Solution

Keep the issue open and implement gateway-level validation in `src/gateway/server-methods/cron.ts` for `delivery.accountId` and `delivery.failureDestination.accountId` during `cron.add` and `cron.update`, with focused regression coverage in `src/gateway/server.cron.test.ts` and any needed user-facing docs/changelog updates.
