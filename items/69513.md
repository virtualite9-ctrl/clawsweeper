## Summary

Keep #69513 open. Current main still validates cron announce channel IDs at create/update time but does not validate `delivery.accountId` against configured channel accounts. The CLI/schema paths only trim or shape-check the account id, and the delivery target later gives the job's explicit account id highest precedence. The linked fix PR #69518 directly addressed this issue but was closed unmerged, so the requested behavior is not implemented or superseded on main.

## Best Possible Solution

Keep the issue open and implement gateway-side validation for `delivery.accountId` and `delivery.failureDestination.accountId` in `src/gateway/server-methods/cron.ts` for both `cron.add` and `cron.update`. Revive or rework the #69518 direction using existing channel plugin account APIs where possible, reject unknown configured account ids with a deterministic error, and add focused regression tests in the cron validation/server test surface for invalid and valid account ids.
