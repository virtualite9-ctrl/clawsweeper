## Summary

Keep this PR open. Current main still does not implement the exact-key preference for archived or compound session keys, but the submitted patch is also not safe to merge as-is because it would bypass current freshest-duplicate behavior that is explicitly covered by tests and review comments.

## Best Possible Solution

Keep the PR open and request changes: implement exact-key preference only for explicit non-canonical archived or compound keys, preserve current freshest duplicate/case-variant selection for canonical logical keys, and add regression coverage for both the archived-key click case and the existing freshness cases.
