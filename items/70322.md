## Summary

Keep open. Current main reduces the common default repro because bundled and synthetic GitHub Copilot model definitions currently set Copilot Claude models to `reasoning: false`, but it has not implemented the requested provider/model effort clamp. If `github-copilot/claude-opus-4.7` is configured or discovered as reasoning-capable, the Anthropic Messages adaptive path still maps `minimal`/`low` to `output_config.effort: "low"` with no Copilot-specific remap, and format/schema failures are still recorded into auth-profile cooldown state before the thinking-level retry path can recover. The provided GitHub context also shows the focused fix PR #70584 is still open/unmerged, while #70376 was closed unmerged.

## Best Possible Solution

Keep #70322 open. The best fix is a scoped Copilot Anthropic Messages compatibility change: clamp unsupported `minimal`/`low` effort for `github-copilot/claude-opus-4.7` to a supported value, add regression coverage in `src/agents/anthropic-transport-stream.test.ts`, and separately prevent `invalid_reasoning_effort`/schema-format rejections from poisoning shared GitHub Copilot auth profiles.
