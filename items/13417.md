## Summary

Current main still does not implement WhatsApp Newsletter/Channel JID support for the message tool. The bundled WhatsApp plugin normalizes only E.164/user JIDs and group JIDs, rejects other `@...` targets such as `@newsletter`, reports the same E.164/group-only target error, and models outbound WhatsApp sessions only as direct or group. The provided issue context is also concrete and the related PR #13424 was closed unmerged after review noted incomplete routing behavior.

## Best Possible Solution

Keep #13417 open until the bundled WhatsApp plugin supports Newsletter/Channel targets end to end: add a newsletter JID recognizer and normalizer, define outbound allowlist/default target semantics, add a channel/newsletter route and chat/session model instead of treating it as direct/group by omission, route sends through the active listener, add focused normalization/resolver/session/send tests, and update the message and WhatsApp docs. Newsletter metadata/follow/react operations can be tracked separately after basic send-target support is correct.
