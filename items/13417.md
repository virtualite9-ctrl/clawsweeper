## Summary

Current main still does not implement outbound WhatsApp Newsletter/Channel JID support for the message tool. `@newsletter` targets are rejected during WhatsApp target normalization before they can reach Baileys send routing, and the surrounding message metadata, outbound session route, tests, and docs still model WhatsApp targets as E.164/direct or group JIDs only. PR #13424 was closed unmerged, and #62697 is adjacent inbound newsletter handling rather than a superseding outbound tracker.

## Best Possible Solution

Keep #13417 open as the outbound WhatsApp Newsletter/Channel message-tool tracker. The likely fix belongs in the bundled WhatsApp plugin: add a newsletter JID recognizer/normalizer, define explicit outbound allowlist/default-target semantics for newsletter channels, model newsletter/channel routing separately from direct/group sessions, route sends through the active listener, add focused normalization/resolver/session/send tests, and update the message and WhatsApp docs. Treat newsletter metadata/follow/react operations and inbound handling in #62697 as separate follow-up work unless maintainers decide to bundle them into one WhatsApp Channel project.
