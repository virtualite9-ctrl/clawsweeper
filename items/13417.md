## Summary

Keep #13417 open. Current main still rejects WhatsApp Newsletter/Channel `@newsletter` targets before outbound send, and the message-tool surface, route metadata, tests, and docs remain direct/group-only. The related PR #13424 is closed unmerged, while #62697 tracks inbound newsletter processing rather than this outbound message-tool target support.

## Best Possible Solution

Keep #13417 open as the outbound WhatsApp Newsletter/Channel support tracker for the bundled WhatsApp plugin. A good fix should add a newsletter JID recognizer/normalizer, define newsletter target policy semantics, represent newsletter/channel routing separately from direct/group chats where needed, route sends through the active WhatsApp listener, add focused normalization/resolver/session/send tests, and update message plus WhatsApp docs. Treat #62697 as related inbound work, not a duplicate.
