## Summary

Keep open. Current main still uses the native Windows Scheduled Task path for `openclaw gateway install`, including `ONLOGON`, `LIMITED`, and `/IT` when a task user is available. The added Startup-folder fallback only covers denied/hung task creation or a task that never starts; it does not replace the successful interactive scheduled-task model reported in #44559. The issue comments' root-cause read remains consistent with current source and docs.

## Best Possible Solution

Keep the issue open for maintainer product judgment on native Windows gateway supervision. The concrete path is to decide whether OpenClaw should add a truly service-like/non-interactive Windows install mode, document native Windows as user-session-only with an explicit limitation, or change the schtasks/install strategy in `src/daemon/schtasks.ts` with regression coverage for the chosen behavior.
