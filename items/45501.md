## Summary

Keep #45501 open. Current main still hardcodes the bare `/new` and `/reset` startup prompt, the config schema/types/docs do not expose `session.resetPrompt`, and tests still assert the exact default wording that the issue asks to make configurable. The comments correctly narrow this to prompt configurability; reset archival and `before_reset` hook surfaces already exist separately.

## Best Possible Solution

Implement the narrow config surface requested: add `session.resetPrompt?: string` to the global session config and a documented per-agent override, resolve it in `resolveBareSessionResetPromptState` using the reset session's agent id, treat it as a full replacement for the normal `BARE_SESSION_RESET_PROMPT_BASE` fallback, preserve bootstrap-pending/limited safety wording unless maintainers explicitly choose otherwise, keep current-time injection, and add focused unit/gateway tests plus config schema/help/docs updates.
