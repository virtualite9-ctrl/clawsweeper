## Summary

Current `main` already implements a dedicated startup-context config switch under `agents.defaults.startupContext`, and the runtime prepends that context on bare `/new` and `/reset` without relying solely on hardcoded `AGENTS.md` instructions.

## Close Comment

Codex review: current `main` already implements the requested config switch. OpenClaw now has a dedicated `agents.defaults.startupContext` config surface with `enabled` and `applyOn` controls in `src/config/zod-schema.agent-defaults.ts:99` and `src/config/types.agent-defaults.ts:251`, and the runtime prepends the startup context on bare `/new` and `/reset` in `src/gateway/server-methods/agent.ts:906`. The behavior is also documented in `docs/gateway/config-agents.md:142` and `docs/reference/token-use.md:24`, and the default AGENTS template now tells sessions to use runtime-provided startup context first in `docs/reference/templates/AGENTS.md:18`. Closing as already implemented on `main`. If you still want extra scope beyond this, such as arbitrary prepend-path selection or per-agent startup-context overrides, please open a narrower follow-up request for that specific gap.
