## Summary

Keep #70971 open. Current main contains a later partial mitigation for core-only allowlists, but runtime `toolsAllow` still is not applied to the final bundled MCP/LSP tool list. The PR may need rebasing/rework because current main added `bundle-mcp` plugin-key semantics, but the requested product behavior is not fully implemented on main.

## Best Possible Solution

Keep the item open, but rebase/rework it into a plugin-aware runtime allowlist fix: ensure cron `toolsAllow` filters all final tool schemas, including bundled MCP and LSP, while preserving current `bundle-mcp` plugin-key behavior. Add regression coverage for `toolsAllow: ["exec"]` with mocked bundled tools and for explicitly allowed bundled tool/plugin-key cases.
