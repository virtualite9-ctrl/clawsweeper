## Summary

Keep this PR open. Current main still appends bundled tools after the runtime allowlist path, and the existing mainline mitigation only gates bundle MCP creation for some allowlists; it does not provide an authoritative final toolsAllow filter for bundled MCP/LSP tools.

## Best Possible Solution

Keep this PR open or supersede it with a maintainer patch that applies runtime toolsAllow after bundled MCP/LSP materialization, intentionally expands the bundle-mcp plugin key, supports exact individual MCP and LSP tool names, and adds focused regression coverage for toolsAllow: ["exec"], one allowed MCP tool, toolsAllow: ["bundle-mcp"], and configured bundle LSP.
