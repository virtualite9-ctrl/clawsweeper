## Summary

Keep open. Current `main` does not show a fix for malicious ClawHub skill intake: the native `openclaw skills install` path still downloads and installs ClawHub skill folders without the install-time dangerous-code scan used by gateway-backed skill dependency installs, and the repo threat model still documents malicious ClawHub skills as a critical residual risk.

## Evidence
