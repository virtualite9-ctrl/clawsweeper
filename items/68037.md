## Summary

Keep this issue open. Current main still has the reported lossy HTTP MCP tool-call normalization path, and #68046 is an open, unmerged closing PR for the same bug.

## Best Possible Solution

Keep this issue open and review #68046 as the implementation candidate. The fix belongs in src/gateway/mcp-http.handlers.ts: preserve valid MCP text, image, audio, resource, and resource_link content blocks for HTTP tools/call results, synthesize text blocks only for strings or malformed entries, and land regression coverage for the non-text block shapes before merging.
