## Summary

Keep #70986 open. Current main still matches the reported memory-wiki lint behavior: wikilinks are extracted with raw regex scanning after only the managed related block is removed, and lint validates only page path/basename targets, not frontmatter titles. The related #64696 remains open and does not supersede this issue because it does not cover the natural-title resolver bug.

## Best Possible Solution

Keep the issue open and fix it in `extensions/memory-wiki`: make `extractWikiLinks` Markdown-aware by skipping frontmatter, fenced/inline code, and comments; update lint target resolution to use normalized keys for relative path, basename, id, frontmatter title, and slugified title; add focused tests in `markdown.test.ts` and `lint.test.ts` for fenced/inline `[[reply_to_current]]` false positives and natural-title links with non-ASCII/special-character titles. Coordinate any relative-link part with #64696, but do not close #70986 until the parser and title-resolution behavior ships.
