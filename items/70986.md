## Summary

Current main still has the reported memory-wiki lint bugs: wikilink extraction scans raw markdown without markdown-aware exclusions, and broken-link resolution only accepts path-derived targets rather than frontmatter titles. The related #64696 is narrower and open, so it does not supersede this broader root-cause issue.

## Best Possible Solution

Keep this issue open and fix it in `extensions/memory-wiki`: make wikilink extraction markdown-aware by excluding frontmatter, fenced code, inline code, and comments before regex extraction, and replace lint’s path/basename-only target set with a normalized lookup that includes relative path, basename, page title, slugified title, id, and future aliases. Add focused regression tests for fenced and inline `[[...]]` literals, bridge and unsafe-local imported source fences, and Unicode/special-character frontmatter-title links. Treat #64696 as related narrower evidence, not the canonical tracker for this broader issue.
