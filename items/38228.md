## Summary

Keep open. Current main still does not extract array-shaped tool-result content in the tool-card path, and the linked `content[0].text` shape is still a real OpenClaw tool-result contract. The submitted PR is not merge-ready as written, but it is not obsolete or safely closeable.

## Best Possible Solution

Keep the item open, or supersede it with a small guarded main-branch fix that extracts string text from array-shaped tool-result content and adds regression coverage for embedded `tool_result` blocks and, if applicable, standalone `toolResult` messages. Reusing or extending the shared text-block extraction behavior would be the safest product direction.

## Evidence
