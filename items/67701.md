## Summary

Keep this PR open. Current main still does not persist auto-compaction tokensAfter into session totalTokens when usage and prompt snapshots are unavailable, so the core /status bug remains. The PR is useful but should be rebased or cherry-picked into a focused change because it also carries unrelated ACP, cron delivery, and Telegram edits.

## Best Possible Solution

Keep this PR open, but ask for a focused rebase or maintainer cherry-pick: thread overflow and timeout auto-compaction tokensAfter into current session accounting, use it when usage and prompt snapshots are unavailable, add regression coverage for a session with no prior totalTokens, and split the unrelated ACP, cron delivery, and Telegram changes into separate PRs.
