## Summary

Keep open. Current main has strict-by-default browser SSRF blocking plus hostname exceptions, but it does not implement the requested confirmation mode, timeout/rate limiting, CIDR/port allowlist and blocklist, non-overridable metadata blocking under broad private-network allow, or temporary `openclaw ssrf` CLI controls. The existing issue comment recommending A+B with hardcoded metadata blocking still maps to remaining work.

## Evidence

- **Current SSRF policy surface:** `SsrFPolicy` only supports boolean private-network opt-ins plus hostname exception lists: `allowPrivateNetwork`, `dangerouslyAllowPrivateNetwork`, `allowRfc2544BenchmarkRange`, `allowedHostnames`, and `hostnameAllowlist`. There is no confirm mode, timeout, per-request approval state, blocklist, or CIDR/port allowlist in the core policy type.
