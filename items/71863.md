## Summary

Keep #71863 open. Current main still has the Signal daemon shutdown race: `stop()` is synchronous, only sends SIGTERM, and `monitorSignalProvider` does not await daemon shutdown in abort/finally paths. The PR is a focused fix for the open #22676 report and adds targeted regression coverage, so it is not obsolete or safely closeable.

## Best Possible Solution

Keep #71863 open for maintainer review. The best path is to land an async Signal daemon shutdown fix in `extensions/signal/src/daemon.ts` and `extensions/signal/src/monitor.ts`, with regression tests covering stop waiting, SIGKILL fallback, repeated stop calls, and aborted monitor shutdown. The current PR appears to implement that shape and should be validated rather than closed.
