## Summary

Keep #71863 open. Current main still has the Signal daemon shutdown race the PR targets: `stop()` is synchronous, only sends SIGTERM, and monitor cleanup does not await daemon exit. The PR is a focused, recent fix for open bug #22676 with regression coverage and addressed bot-review follow-ups, so no conservative close reason applies.

## Best Possible Solution

Keep the PR open for maintainer review and validation. The best product path is to land #71863 or an equivalent Signal plugin fix that makes daemon stop asynchronous, waits for process exit on abort/restart cleanup, escalates to SIGKILL after a bounded timeout, and keeps the added regression tests tied to #22676.
