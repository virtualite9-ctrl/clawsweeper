## Summary

Keep #69249 open. Current main has partial restart-aborted main-session recovery, but it resumes with `deliver: false` and the regression test asserts that behavior, so it does not satisfy the requested Discord-visible recovery/apology/resume message. The related Feishu restart-drop report #38836 was also kept open for the same remaining user-visible recovery gap.

## Best Possible Solution

Implement restart-aborted main-session recovery that preserves the originating delivery route and either posts a bounded recovery notice to the original Discord target or resumes with safe channel delivery enabled. The likely code path is `src/agents/main-session-restart-recovery.ts`, backed by a Discord/main-session regression test that proves a gateway restart produces a user-visible recovery message or resumed final reply without duplicate delivery.
