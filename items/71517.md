## Summary

Keep open. Current main still has the split, narrow filename parsing the PR is trying to replace, and the PR branch remains a plausible implementation candidate for the Feishu mojibake bug and the broader shared filename decoder work.

## Best Possible Solution

Keep this PR open for maintainer review and CI. If accepted, land the shared media filename decoder plus Feishu inbound recovery, then keep #48788 as the canonical follow-up for migrating the remaining channel filename paths and any SDK documentation or API baseline work.
