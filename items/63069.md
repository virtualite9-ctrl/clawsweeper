## Summary

Keep open. Current main still hardcodes zsh and fish profile paths and still has the bash check/install mismatch described in the report. The same author has an open, unmerged implementation candidate in #63082, so this issue should remain open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review/land or replace #63082. The fix should centralize shell profile resolution in `src/cli/completion-runtime.ts`, respect `ZDOTDIR` for zsh and `XDG_CONFIG_HOME` for fish/PowerShell, make bash check/install use the same `.bashrc`/`.bash_profile` fallback, update doctor/setup reload hints, and add focused unit tests for those path-resolution cases.
