## Summary

Keep this issue open. Current main still writes/checks shell completion profiles with hardcoded zsh/fish paths and still has the bash check/install mismatch. The provided GitHub context also shows an open same-author implementation candidate, #63082, with closing syntax for this issue, so the issue should remain open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review, land, or replace #63082. The fix should centralize shell profile resolution in `src/cli/completion-runtime.ts`, respect `ZDOTDIR` for zsh and `XDG_CONFIG_HOME` for fish/PowerShell, make bash check/install use the same `.bashrc`/`.bash_profile` fallback, update doctor/update reload hints to show the resolved path, and add focused unit tests for those profile-resolution cases.
