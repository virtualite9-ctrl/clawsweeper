## Summary

Keep #59909 open. Current main still hard-codes the TUI footer contents and exposes no env var, CLI option, TUI option, or slash command to switch footer modes or hide the footer. The related compact-footer PR is still unmerged/open in the provided context, so it does not supersede the request.

## Best Possible Solution

Keep the issue open. A good implementation would add a small typed footer mode contract in `src/tui/tui-types.ts`, wire an env/config or `openclaw tui --footer <mode>` option through `src/cli/tui-cli.ts` and `src/tui/tui-launch.ts`, update `updateFooter()` in `src/tui/tui.ts`, add focused tests, and update the public TUI docs.
