## Summary

Current main still contains the same-target media-only suppression bug: message-tool target/media telemetry can be recorded without sent text, and final/follow-up reply suppression still drops all session text based on matching target metadata alone. The narrow fix PR #37900 was closed unmerged, and current tests still do not cover same-target media-only sends preserving session text.

## Best Possible Solution

Keep #37841 open. Patch `buildReplyPayloads` and `resolveFollowupDeliveryPayloads` so same-origin target suppression only drops final/follow-up session text when the message tool actually sent text to that same target. Media-only sends should only dedupe already-sent media URLs while preserving session text. Add focused regression tests for same-target media-only sends preserving text, same-target text sends still suppressing duplicates, and the follow-up delivery path.
