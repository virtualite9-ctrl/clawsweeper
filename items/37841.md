## Summary

Keep #37841 open. Current main still suppresses final and follow-up session text from the originating WhatsApp target based on message-tool target metadata alone, even when the message tool only sent media and recorded no sent text. The narrow fix PR #37900 was closed unmerged, and the current regression tests still cover target-only suppression rather than preserving same-target media-only session text.

## Best Possible Solution

Keep #37841 open. Patch `buildReplyPayloads` and `resolveFollowupDeliveryPayloads` so same-origin target suppression only drops final/follow-up session text when the message tool actually sent text to that same target. Media-only sends should dedupe already-sent media URLs while preserving session text. Add focused regression tests for same-target media-only sends preserving text, same-target text sends still suppressing duplicates, and the follow-up delivery path.
