## Summary

Current `main` does not contain the PR's Discord placeholder fallback, and the remaining gap is still plausible. `extensions/discord/src/monitor/message-utils.ts` still infers placeholders only from `attachment.content_type` and still reduces non-image attachment-only messages to `<media:document> (N file[s])`, so there is not strong evidence to close this PR as already implemented or obsolete. At the same time, current `main` does have newer downstream MIME/extension inference that may mitigate the original "video understanding does not trigger" claim, so the exact reproduction is mixed rather than clearly dead.

## Evidence

- **Discord placeholder logic is still MIME-only:** `inferPlaceholder()` checks only `attachment.content_type` for `image/`, `video/`, and `audio/`, then falls back to `<media:document>`; there is no filename-extension fallback on current `main`.
