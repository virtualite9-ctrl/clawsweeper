## Summary

Current main still matches the reported bug path: structured exec completions are all treated as relayable exec events, and heartbeat delivery explicitly avoids normal HEARTBEAT_OK skipping whenever an exec completion is present. The related fix PR #72253 is open and unmerged, so the issue should stay open until that PR or an equivalent main-branch fix lands.

## Best Possible Solution

Keep #72252 open and review/land #72253 or an equivalent main-branch fix. The fix should classify structured `Exec completed (..., code 0)` and cleanup `Exec failed (..., signal SIGTERM)` events as internal for heartbeat delivery, continue relaying actionable non-SIGTERM failures such as `code 1`, and update focused coverage in `src/infra/heartbeat-events-filter.test.ts` plus `src/infra/heartbeat-runner.returns-default-unset.test.ts` or the equivalent runner test seam.
