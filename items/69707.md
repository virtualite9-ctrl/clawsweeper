## Summary

Keep PR #69707 open. Current main still forwards configured `embedding.dimensions` to `OpenAI.embeddings.create`, and the test suite still asserts that upstream payload shape. The PR is not implemented on main or clearly superseded; it remains a focused bug-fix candidate, though maintainers should reconcile it with the previously shipped #32036 behavior that intentionally forwarded dimensions.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to decide the `embedding.dimensions` contract explicitly: either land the PR's local truncation plus positive-integer and short-vector regression coverage, or make dimension forwarding conditional for providers known to accept it while omitting it for custom `baseUrl` providers that reject the field. In either case, keep LanceDB vector dimensions aligned with stored/search vectors and update the existing forwarding test accordingly.
