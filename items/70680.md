## Summary

Keep #70680 open. Current main still treats any non-archived `.jsonl` file, including `<session>.trajectory.jsonl`, as a primary session transcript candidate, and doctor still builds orphan candidates from that helper while only marking `sessions.json` transcript paths as referenced. The linked fix PR #70812 directly tracks this bug but is still open/unmerged in the provided GitHub context, so closing the original report now would be premature.

## Best Possible Solution

Keep #70680 open until #70812 or an equivalent main-branch fix lands. The likely product/code direction is to tighten `isPrimarySessionTranscriptFileName()` so `*.trajectory.jsonl` is treated as a sidecar, then add helper and doctor regression coverage proving real orphan primary transcripts are still detected while trajectory sidecars are ignored.
