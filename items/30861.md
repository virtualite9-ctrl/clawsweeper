## Summary

Keep #30861 open. Current main still has adjacent session-selection and agent-list plumbing, but it does not implement the requested persistent Dashboard chat-header active-agent/workspace indicator, full workspace tooltip, or agent-switch confirmation feedback. The backend already exposes the needed workspace metadata via `agents.list`, and the Agents page can display it, so this remains a focused Control UI gap rather than a core data-contract blocker. Related PR #31602 was a useful implementation attempt, but it was closed unmerged and does not supersede the issue.

## Best Possible Solution

Keep #30861 open as the Control UI tracker. Implement a minimal chat-header context indicator derived from the active `sessionKey` plus `agents.list`, showing agent id/name and workspace path with full-path tooltip, updating through `switchChatSession` and slash-command/session changes, surviving reload through persisted session state, adding clear switch feedback, staying quiet for single-agent setups, and adding focused render/helper tests. Closed PR #31602 can inform a fresh patch but should not be treated as canonical.
