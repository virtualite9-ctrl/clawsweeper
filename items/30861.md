## Summary

Keep #30861 open. Current main still has only a session selector grouped by agent in the chat header; it does not render the requested persistent active-agent plus workspace indicator, workspace tooltip, switch confirmation feedback, or focused test coverage. The gateway already exposes per-agent workspace metadata, so this remains a Control UI implementation gap rather than a backend API blocker. Related PR #31602 was closed unmerged and does not supersede the issue.

## Best Possible Solution

Keep #30861 open as the canonical Control UI tracker. The best fix is a minimal chat-header context indicator derived from the current session key and agents.list, showing agent id/name and workspace path with a full-path tooltip, updating immediately through switchChatSession and /agent-driven session changes, surviving reload/reconnect via persisted session state, providing clear switch feedback, remaining unobtrusive for single-agent users, and adding focused render/helper tests. Closed PR #31602 can inform the patch but does not replace the issue.
