## Summary

Keep #30861 open. Current main still does not implement a persistent active-agent plus workspace indicator in the Dashboard chat header. The UI has a session selector grouped by agent and switch state persistence, and the gateway already exposes agent workspace metadata, but there is no always-visible agent/workspace block, workspace tooltip, switch confirmation feedback, or focused test coverage for the requested UX. The related implementation PR #31602 was closed unmerged and is not present in this checkout.

## Best Possible Solution

Keep #30861 open as the canonical Control UI tracker. The right fix is a minimal chat-header indicator derived from the current session key and `agents.list`, showing agent id/name and workspace path with a full-path tooltip, updating immediately on `switchChatSession` and `/agent` changes, surviving reload/reconnect through `lastActiveSessionKey`, providing clear switch feedback, keeping single-agent mode unobtrusive, and adding focused UI helper/render tests. Closed PR #31602 can be treated as prior art but does not supersede the issue.
