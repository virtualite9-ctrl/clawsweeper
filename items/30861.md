## Summary

Keep #30861 open. Current main still only provides a chat session selector with some agent grouping; it does not persistently show the active agent and workspace in the Dashboard header, does not expose a full workspace tooltip, and does not show switch confirmation feedback. The backend already returns workspace metadata in `agents.list`, so this remains a Control UI implementation gap. Related PR #31602 was closed unmerged and does not supersede the issue.

## Best Possible Solution

Keep #30861 open as the Control UI tracker. The best implementation is a minimal chat-header context block derived from `state.sessionKey` and `state.agentsList`, showing active agent id/name and workspace path with a full-path tooltip, updating immediately through `switchChatSession` and slash-command-driven session changes, surviving reload through persisted session state, adding clear switch feedback, staying unobtrusive for single-agent setups, and adding focused render/helper tests. Closed PR #31602 can inform a new patch but should not be treated as canonical.
