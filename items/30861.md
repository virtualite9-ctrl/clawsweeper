## Summary

Keep #30861 open. Current main at 406ae72fd2789c78053f6b94006ec8f488bc1c00 still has adjacent session-selection and agent metadata plumbing, but not the requested persistent Dashboard chat-header active-agent/workspace indicator, full workspace tooltip, stable visual badge, or switch confirmation feedback. The backend already exposes workspace metadata through agents.list, and the Agents panel uses it, so this is a focused Control UI feature gap rather than a missing backend contract. Related PR #31602 was closed unmerged and does not supersede the issue.

## Best Possible Solution

Keep #30861 open as the Control UI tracker. Implement a minimal persistent chat-header context indicator derived from the active sessionKey plus agents.list, showing agent id/name and workspace path with a full-path tooltip, updating through switchChatSession and slash-command/session changes, surviving reload via persisted session state, adding clear switch feedback, staying quiet for single-agent setups, and adding focused render/helper coverage. Closed PR #31602 can inform a fresh patch but should not be treated as shipped or canonical.
