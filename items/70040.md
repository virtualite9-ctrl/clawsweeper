## Summary

Keep open. Current `main` does not already implement Chinese memory trigger support in `extensions/memory-lancedb`, and the PR discussion surfaces a real gap: even with the proposed regex, `shouldCapture` still rejects common short Chinese inputs before trigger matching. The duplicate-bot comment is not enough to close this conservatively because the checked-in code on `main` still lacks the requested CJK trigger handling and regression coverage.

## Evidence

- **Current main trigger list has no Chinese/CJK patterns:** `MEMORY_TRIGGERS` on current `main` includes English/Czech phrases, phone, email, and simple preference words only; there is no Chinese regex like the PR adds.
  - file: [extensions/memory-lancedb/index.ts:197](https://github.com/openclaw/openclaw/blob/2cd2732ab6695a4b3220ce41166b74aca7097c5f/extensions/memory-lancedb/index.ts#L197)
