## Summary

Keep #8812 open. Current main still renders URL-bearing tool output as escaped/plain text in Control UI tool-card paths and native WebChat tool-result cards. The only directly related implementation attempt, PR #8880, was closed unmerged, and a maintainer review comment on the issue also explicitly kept it open.

## Best Possible Solution

Keep #8812 open. Implement a safe URL-aware rendering path for Control UI tool output cards, including expanded output, raw details, collapsed previews, inline short cards, and side-panel output, with focused tests for anchors, escaping, unsafe schemes, punctuation, and whitespace preservation. If WebChat remains in scope, add equivalent SwiftUI URL-aware rendering using AttributedString/Link handling rather than plain Text.
