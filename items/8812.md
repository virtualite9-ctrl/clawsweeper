## Summary

Keep #8812 open. Current main still renders URL-bearing tool output as escaped text in Control UI tool-card paths and as plain SwiftUI Text in native WebChat. The closed PR #8880 remains useful prior art, but it was not merged and current main does not contain an equivalent safe URL-aware renderer.

## Best Possible Solution

Keep #8812 open. Implement a safe URL-aware renderer for Control UI tool output across expanded output, raw details, collapsed previews, inline short cards, and sidebar/raw views, with focused tests for anchors, escaping, unsafe schemes, punctuation, and whitespace. If native WebChat remains in scope, add equivalent SwiftUI URL rendering with AttributedString or Link handling rather than plain Text. PR #8880 is useful prior art but not shipped.
