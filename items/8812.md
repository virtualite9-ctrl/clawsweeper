## Summary

Keep #8812 open. Current main still renders URL-bearing tool output as escaped/plain text in Control UI tool-card paths and native shared WebChat tool-result cards. The related implementation attempt, PR #8880, is closed unmerged in the provided GitHub context, and no matching linkification implementation is present in local history or the current source.

## Best Possible Solution

Keep #8812 open. Implement a safe URL-aware renderer for Control UI tool output cards across expanded output, raw details, collapsed previews, inline short cards, and sidebar/side-panel content, with focused tests for anchors, escaping, unsafe schemes, punctuation, and whitespace. If native WebChat remains in scope, add equivalent SwiftUI URL-aware rendering using AttributedString/Link handling instead of plain Text. PR #8880 is useful prior art but would need rework because it is closed unmerged and not present on main.
