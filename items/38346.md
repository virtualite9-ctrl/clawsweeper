## Summary

Keep #38346 open. Current main still has no Telegram mode that observes or ingests all messages in an allowed group while separately restricting agent-triggering/replies to allowlisted senders. `groupPolicy: "open"` lets unauthorized senders trigger, while `groupPolicy: "allowlist"` rejects unauthorized senders before message processing and the existing `ingest` hook only runs on the post-auth no-mention skip path.

## Best Possible Solution

Keep this open for bundled Telegram plugin work. Add an explicit observe/ingest path for configured groups/topics that can record or hook all visible messages, then apply a separate sender trigger allowlist before agent turns, command handling, and replies. Update Telegram config schema/types/docs and add regression coverage for unauthorized unmentioned ingest without reply, unauthorized mentioned no reply, and allowlisted mentioned reply.
