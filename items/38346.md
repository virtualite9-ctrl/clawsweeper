## Summary

Keep #38346 open. Current main still does not provide the requested Telegram split where the bot can observe/ingest all allowed-group messages while only allowlisted senders can trigger replies. The existing behavior is still binary: `groupPolicy: "allowlist"` blocks unauthorized senders before message processing/ingest, while `groupPolicy: "open"` bypasses sender allowlists and can reply when mention gating allows it.

## Best Possible Solution

Keep #38346 open for bundled Telegram plugin work. Add an explicit Telegram observe/ingest path for configured groups/topics, then apply a separate sender trigger allowlist before starting agent turns or sending replies. Update Telegram config types/schema/docs and add regression tests for unauthorized unmentioned ingest without reply, unauthorized mentioned no reply, and allowlisted mentioned reply.

## Risks / Open Questions

- Telegram Bot API privacy mode may prevent receiving all group messages at the transport layer unless privacy is disabled or the bot has sufficient group visibility; any implementation should document that prerequisite.
- Observe-only ingestion has retention and privacy implications, so it should be explicit opt-in and tested to ensure observed unauthorized messages never start agent turns or replies.
