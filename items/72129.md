## Summary

Keep #72129 open. Current main still lacks Hugging Face text-to-image support: the bundled Hugging Face plugin registers only the chat/model provider, its manifest declares no image-generation contract, and the public provider docs still say text-to-image should use Hugging Face clients directly. The PR is a focused successor to closed, unmerged #63487 and appears to target an existing bundled model-provider surface, but the open review comments identify concrete issues that should be addressed before merge.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to update the PR to validate/safely encode model path segments, broaden or align the image `Accept` header with MIME handling, explicitly reject unsupported edit/input-image requests, and rerun the changed extension/docs gates before deciding whether to merge into the bundled Hugging Face plugin or redirect to ClawHub by explicit maintainer judgment.
