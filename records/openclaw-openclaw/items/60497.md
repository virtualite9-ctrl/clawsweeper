## Summary

Keep this issue open. Current main still records exec failures internally, but ordinary non-timeout `exec`/`bash` failures are suppressed from default user-visible payloads, and assistant success text is added before that warning policy runs. The provided NemoClaw-side python symlink fix addresses one environment cause, but not the OpenClaw reply-shaping behavior reported here. The related open PR #60507 is a companion destructive-command approval catalog, not a closing-syntax implementation of this bug.

## Best Possible Solution

Keep this issue open. The likely fix belongs in `src/agents/pi-embedded-runner/run/payloads.ts` with adjacent embedded-run tests: make failed `exec`/`bash` results such as exit 127 visible at default verbosity as concise failures, and suppress or override success-looking assistant text unless it explicitly acknowledges the failed action. Keep the NemoClaw python symlink work and #60507 separate unless they grow into the actual post-exec fail-closed implementation.
