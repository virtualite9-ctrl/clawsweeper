## Summary

Keep open. Current main still does not implement the requested `session_status` split between the selected/session-default model and a separate non-fallback active routed model. The shipped #55425-adjacent behavior uses a runtime model as the primary card model when no explicit override exists, while the shared renderer only emits a second model line for fallback state, not a generic `Active:` line.

## Best Possible Solution

Keep this issue open. Implement the split in the shared status rendering path: keep the configured/session-selected model on `🧠 Model:`, add `↪️ Active: provider/model` when the resolved active runtime model differs and fallback state is not active, and add focused regression coverage in `src/auto-reply/status.test.ts` and `src/agents/openclaw-tools.session-status.test.ts`. If the active model is only present in transcript metadata, extend the transcript/model-snapshot lookup so `session_status` can use it without mutating the session default.
