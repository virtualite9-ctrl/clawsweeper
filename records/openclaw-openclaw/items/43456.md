## Summary

Keep this issue open. Current main still uses Mattermost `root_id` as thread/session routing metadata only; it does not resolve the referenced post, verify that it was authored by the bot, or pass a `reply_to_bot` implicit mention into the oncall mention gate. The prior Codex review in the issue discussion remains accurate on current main, and the closed related #45284 covered outbound block-stream threading rather than this inbound reply-wake behavior.

## Best Possible Solution

Implement this in the bundled Mattermost plugin: add safe referenced-post resolution for `root_id` and any supported parent/inline reply metadata, verify the referenced post `user_id` matches the Mattermost bot user id, pass a `reply_to_bot` implicit mention into the Mattermost/shared mention gate, and add regression tests for unmentioned threaded and inline replies in default oncall mode. Update Mattermost and group-channel docs once the behavior ships.
