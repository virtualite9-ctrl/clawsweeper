## Summary

Keep this PR open. Current main still does not implement the PR's core behavior: gateway agent runs store a result payload, but agent.wait terminal snapshots and responses still drop it, and subagent announce/completion paths still fall back to chat history/output capture rather than an agent.wait result. The PR is still potentially useful, but it needs rebase work for the current run-manager/output split and should align protocol typing/docs if it exposes structured results.

## Best Possible Solution

Keep this PR open or replace it with an updated patch against current main. The best implementation is to thread structured result through AgentWaitTerminalSnapshot and both cached/live agent.wait responses, preserve it when lifecycle and dedupe resolution race, normalize result-bearing wait responses for subagent callers, extract visible text from standard agent result shapes such as { payloads, meta }, use that text only as a fallback when frozen chat-history output is absent, and add focused gateway plus subagent regression coverage. Before merge, align the gateway/plugin runtime contract and docs for the new result field.
