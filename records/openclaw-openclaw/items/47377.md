## Summary

Keep open. Current main still has the bug this PR is trying to fix: WebChat receives cumulative delta snapshots from the gateway, then the Control UI stores the full cumulative `chatStream` as each tool-interrupted segment and renders those segments as independent stream bubbles. The provided PR diff is narrow and security-low-risk, but current main does not implement it, so it is not obsolete.

## Best Possible Solution

Keep this PR available as an implementation candidate. The best fix belongs in `ui/src/ui/app-tool-stream.ts`: commit only the newly streamed text beyond the total already committed segment length, then add focused tests in `ui/src/ui/app-tool-stream.node.test.ts` or `ui/src/ui/chat/build-chat-items.test.ts` for multi-tool streaming order and no growing duplicate bubbles. Review it alongside #46985 and #47399 so the final patch covers both stored segments and live preview behavior without overlapping PR churn.
