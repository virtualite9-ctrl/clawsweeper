## Summary

Keep this issue open. Current main still has no native `.gitignore`-style `--exclude` or `--exclude-file` backup CLI support; it only prunes installed plugin `node_modules` under the state `extensions/` tree. The supplied GitHub context also shows open PR #44288 explicitly targets this issue, so the issue should not be closed before that candidate is reviewed, replaced, or merged.

## Best Possible Solution

Keep this issue open as the user-facing tracker for native backup exclude support. Review and either land #44288 after maintainer/security scrutiny or replace it with a smaller core implementation that adds repeatable `backup create` exclude flags, exclude-file parsing, typed options, archive filtering, restore-safety rules for protected paths, manifest behavior that avoids leaking sensitive filenames, regression tests, and public docs at `docs.openclaw.ai/cli/backup`.

- Native backups still cannot exclude arbitrary logs, caches, `.env*`, `*.pem`, `*.key`, dependency trees outside installed plugins, or a user-provided `.openclawignore`/exclude file.
