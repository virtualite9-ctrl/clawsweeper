## Summary

Keep this PR open. Current main still starts one Feishu webhook HTTP server per account and listens on that account's configured/default host:port, so the shared host:port EADDRINUSE problem described in the PR body is not implemented on main. The PR remains a plausible fix candidate, but it should be rebased and redesigned around current webhook hardening before merge.

## Best Possible Solution

Keep this PR open as a focused Feishu plugin bug-fix candidate. The best path is a rebase onto current main that implements one shared listener per webhook host:port, registers per-account routes safely, routes without cross-account delivery, preserves the current 64 KiB/5 s pre-auth body protections and per-account rate limits, propagates listen/server errors to all joined account tasks, and adds regression tests for two webhook accounts sharing a port plus cleanup/error cases.
