## Summary

Keep this issue open. Current main still implements the bundled session-memory hook only for internal command:new and command:reset events, while subagent completion emits typed plugin lifecycle hooks such as subagent_ended. I did not find a shipped bridge that resolves the child session transcript/workspace and writes the bundled session-memory Markdown file when a subagent session ends. Related issue #55963 was closed as duplicate with this issue as the canonical remaining subagent tracker; PR #55974 attempted the fix but is closed unmerged, and open PR #56672 adds broader session_end hook coverage without closing or implementing this specific session-memory behavior on main.

## Best Possible Solution

Keep this issue open. The best fix is a narrow, tested lifecycle bridge: when a subagent run/session reaches a terminal state, resolve the child session key, workspace, session entry/transcript, and completion reason, then invoke the bundled session-memory writer without duplicating main-session /new or /reset writes. If maintainers prefer the broader hook direction, land or revise the session_end/subagent lifecycle work first, then explicitly subscribe session-memory to that event with regression coverage proving a completed sessions_spawn child creates the expected memory Markdown file.
