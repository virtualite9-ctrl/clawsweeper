## Summary

Keep this issue open. Current main still prunes sandbox runtimes and registry entries without deleting the local Docker sandbox workspace directory, and the provided GitHub context shows open PR #43835 remains an unmerged implementation candidate with closing syntax for this issue.

## Best Possible Solution

Keep this issue open and review/land or close/supersede #43835. The preferred fix belongs in `src/agents/sandbox/prune.ts`: after successful runtime removal for non-shared local sandbox registry entries, derive the workspace path from `cfg.workspaceRoot` plus `entry.sessionKey`, remove it recursively with safe error handling, preserve shared and still-in-use workspaces, and add focused regression tests for session, agent, and shared scope behavior.
