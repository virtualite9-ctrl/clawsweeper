## Summary

Keep this PR open. Current main still does not implement the requested `tools.bitable` toggle: Feishu Bitable tools remain registered whenever Feishu accounts exist, while the config type, schema, defaults, and merged account tool config have no `bitable` field. The PR is a focused implementation candidate for the linked #45195 request, with no obvious security or supply-chain red flags in the provided diff, but it should get maintainer review plus a targeted regression test and docs/config help before merge.

## Best Possible Solution

Keep the PR open as the focused implementation candidate. Rebase/review it against current main, add a targeted Feishu plugin test for default-enabled behavior and `tools.bitable: false`, and update Feishu configuration docs/help if the toggle is accepted.
