## Summary

Keep this PR open. Current main still appears to have the discovery-scan bug the PR targets: child directories are capped with `maxSkillsLoadedPerSource` before checking for `SKILL.md`, even though the config contract separates candidate scanning from loaded-result limits. However, the PR should not be merged as-is because its head includes unrelated dependency/package and Telegram/TUI changes with security-sensitive supply-chain surface.

## Best Possible Solution

Keep the PR open but request a narrowed update: rebase onto current main, keep only the `src/agents/skills/workspace.ts` scan-limit fix and the focused regression test, remove unrelated package/lockfile/Telegram/TUI changes, and then run the targeted skills snapshot test plus the changed gate in the normal maintainer flow.
