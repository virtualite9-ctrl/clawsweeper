## Summary

Keep this PR open. Current main still has the iOS manual-LAN TLS force-upgrade and Swift mixed-auth precedence behavior the PR is trying to fix, so the PR is not obsolete or implemented. The PR is not ready to close/merge as-is because its test diff is structurally broken and its plaintext Tailscale policy conflicts with current security guidance.

## Best Possible Solution

Keep this PR open as an implementation candidate, but request changes before merge: move the Swift tests inside `GatewayConnectionSecurityTests`, update stale TLS assertions, align LAN/Tailscale plaintext behavior with current mobile security policy and any explicit opt-in mechanism, reuse or centralize the existing OpenClawKit local-network host classifier, add password-over-bootstrap regression coverage, and keep the TypeScript/setup-code parts tracked under #47887 or a linked follow-up.
