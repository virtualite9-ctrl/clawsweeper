## Summary

Keep open. Current main has durable workspace files, memory/dreaming promotion, Feishu mention fidelity, and broadcast-style multi-agent routing, but it still lacks a governed natural-language rule-learning layer with explicit precedence/supersession, and the channel contract remains mention-gating oriented rather than a generic "all explicitly mentioned bots reply once" semantic. The linked openclaw-listen-only workaround is useful context isolation, and #42400 is an open partial listen-only/no-reply proposal, not a replacement for the requested product/API design.

## Best Possible Solution

Keep this issue open and scope it into concrete design work: define a governed durable rule layer or typed memory/rule records with review, precedence, and supersession semantics; decide how promoted rules interact with AGENTS.md, SOUL.md, MEMORY.md, dreaming, and session bootstrap refresh; and add channel/plugin contract support for explicit mention target sets so group routing can dispatch one reply from every mentioned bot/agent while keeping unmentioned bots silent. Review #42400 separately as a partial listen-only suppression feature, not as the canonical fix.
