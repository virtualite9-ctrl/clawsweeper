## Summary

Keep this PR open. Current main does not implement the requested Windows PATH bootstrap, but the PR is stale against the current hardened PATH policy and needs rework before it should land.

## Best Possible Solution

Keep this PR open as a useful but not-ready Windows bootstrap fix. The maintainer path is to rebase it onto current main and rewrite the PATH change around the hardened model: use the correct Windows delimiter, add trusted Windows system/runtime dirs carefully, append or explicitly opt into user-writable package-manager dirs, avoid prepending WindowsApps aliases, add deterministic Windows tests including ProgramFiles(x86), and drop unrelated schema-label changes.

## Risks / Open Questions

- Prepending user-writable Windows locations such as LOCALAPPDATA pnpm, WindowsApps, and APPDATA npm ahead of existing PATH/system dirs can change command resolution and conflict with current PATH-hijack hardening.
- The PR test still splits the simulated Windows PATH with host path.delimiter in the latest diff, so drive-letter paths can be parsed incorrectly on non-Windows CI.
- The unrelated schema.labels APNs push-label hunk should be removed during rebase; current main already contains equivalent gateway.push labels under the current layout.
