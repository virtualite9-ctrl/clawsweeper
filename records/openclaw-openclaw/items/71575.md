## Summary

Keep open. Current main still intentionally renders Telegram tool-progress preview updates by default in partial streaming, and it still keeps default tool-progress messages when preview streaming is off. The related open PR #72482 may address part of the non-streaming path, but it is not merged and does not solve the broader final-only Telegram behavior requested here.

## Best Possible Solution

Keep this issue open for a Telegram-owned behavior change. Review and land #72482 or an equivalent fix for the non-streaming path if correct, then decide whether Telegram should hide tool/progress lines by default by changing `extensions/telegram/src/bot-message-dispatch.ts`, preserving approval/media/error delivery exceptions, and updating the Telegram dispatch tests plus `docs/channels/telegram.md` and `docs/concepts/streaming.md`.
