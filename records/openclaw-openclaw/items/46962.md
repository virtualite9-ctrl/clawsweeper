## Summary

Keep open. Current main still does not implement the PR's intended behavior: `markdownToIR` has only a flat `blockquotePrefix`, soft/hard breaks append a bare newline, and paragraph separators append raw `\n\n`, so continuation and later paragraph lines inside prefixed blockquotes are still not depth-aware. Existing main tests only cover the initial prefix/spacing case, while the PR adds focused regression coverage for softbreak, hardbreak, multi-paragraph, nested, and nested-to-parent blockquote cases. Security review found no obvious supply-chain or permission regression in the stated PR scope because the diff is limited to the markdown IR implementation and its tests.

## Best Possible Solution

Keep this PR open as a focused implementation candidate. The best path is to rebase or cherry-pick the depth-aware blockquote prefix handling into `src/markdown/ir.ts`, keep regression coverage for softbreak, hardbreak, multi-paragraph, nested, and nested-to-parent cases in `src/markdown/ir.blockquote-spacing.test.ts`, then run the targeted markdown tests and the appropriate changed gate.
