## Summary

Keep this PR open. Current main only has the cross-target messaging-tool dedupe guard in the normal reply payload builder; the followup delivery path named by this PR still applies text/media dedupe before same-target suppression, so the PR is not obsolete. However, the PR as provided is broader than its title and includes unrelated model retry, outbound serialization, and dependency changes with unresolved review risks, so the best maintainer path is to split or narrow it before landing.

## Best Possible Solution

Keep this PR open, but ask for a narrow version: move the same-target dedupe guard into `resolveFollowupDeliveryPayloads` or the followup-specific payload path, add followup text and media cross-target regression tests, and split the same-model transient retry, delivery serializer, and Mattermost dependency changes into separate PRs with their own product/security review.
