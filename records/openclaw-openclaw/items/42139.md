## Summary

Keep open. Current main has improved ACP/acpx and Discord binding documentation, but it still does not implement the reported Raspberry Pi Gateway to remote Mac ACP harness execution path. ACP spawn/config/runtime surfaces can choose agent, backend, mode, cwd, model/thinking, permissions, and bindings; they cannot select a remote OpenClaw node as the harness process host.

## Best Possible Solution

Keep this issue open for maintainer product/docs judgment. The short-term path is to document explicitly that bundled ACP/acpx harnesses run on the Gateway host today, recommend running the Gateway/acpx on the Mac or using an operator-managed wrapper for multi-machine setups, and clarify Discord binding guidance. If OpenClaw should support the requested topology, add an explicit ACP runtime node selector and node-host process execution contract across config schema, ACP runtime SDK, acpx integration, and Discord /acp spawn docs.
