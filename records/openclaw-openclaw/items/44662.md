## Summary

Keep this PR open. Current main does not implement the PR's intended lower-level write-size guard: host pinned writes still verify identity/root only, local and sandbox helpers do not pass or enforce an expected byte count, and the regression tests named by the PR are not present. The related open PR #67202 overlaps at the user-facing write-verification layer, but it is not merged and does not make this lower-level atomic-write hardening obsolete on current main.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to either land a carefully reviewed lower-level expected-size guard with regression tests for host pinned writes, sandbox temp writes, remote sandbox writes, encoding byte counts, and rollback behavior, or intentionally supersede it with a merged canonical PR that covers the same lower-level atomic-write failure mode. Coordinate with #67202, but do not close this PR until a merged change on main actually protects these helper paths.
