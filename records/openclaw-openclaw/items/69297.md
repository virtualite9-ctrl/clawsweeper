## Summary

Keep this PR open. Current main does not implement the proposed WhatsApp per-group admin privacy controls: the config type/schema/docs lack `groups.<jid>.admin`, and the inbound, command-authorization, group-activation, and group-ingress paths are still sender-agnostic generic allowlist/mention handling. The PR remains the active implementation candidate for the closed linked feature request, and it should be reviewed or landed rather than closed as already implemented.

## Best Possible Solution

Keep this PR open as the active implementation candidate for the WhatsApp group admin privacy feature. The best path is maintainer review and landing after scope cleanup or justification for unrelated files, with the WhatsApp config type/schema/generated metadata/docs plus inbound, command authorization, ingress, activation, ack reaction, and regression tests kept together. Do not close as implemented until that behavior is actually on main.
