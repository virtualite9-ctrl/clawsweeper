## Summary

Keep this issue open. Current main has partial `/new` and `/reset` session-memory behavior plus startup hydration from recent memory files, but it still does not implement automated preservation and synthesis as requested: no `.session_memory_preserve.json`, no full transcript export with anchors, no daily continuation package generation, and no LLM synthesis path in the bundled hook. The related implementation PR is still open and only partly covers the requested workflow, while the related raw-memory poisoning bug remains open and should shape any expansion of this feature.

## Best Possible Solution

Keep this issue open until maintainers either land a scoped implementation or explicitly redirect the design to a plugin/context-engine path. A maintainable implementation should coordinate with #50584 and #69943, make synthesis opt-in, sanitize before persistence, generate compact daily continuation packages, link to full transcript receipts with anchors when enabled, and add regression coverage for `/new` and `/reset` preservation plus startup hydration.

## Risks / Open Questions

- Expanding automatic persistence without coordinating with #69943 could preserve and reinject raw chat-template tokens, malformed tool-call XML, or other unsafe transcript artifacts.
- Closing this issue now would lose the broader tracker for full-fidelity transcript receipts, anchors, daily continuation synthesis, and startup behavior while #50584 remains open and partial.
- A ClawHub/context-engine plugin may ultimately be the right product shape, but current main already ships a bundled session-memory hook for this surface, so redirecting or narrowing the feature needs maintainer judgment rather than conservative cleanup closure.
