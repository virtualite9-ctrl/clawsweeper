## Summary

Current main does not implement this PR’s full intended behavior, but the PR is also not ready to close or merge as-is. Main still lacks `isSilentTokenOnOwnLine`, `normalizeReplyPayload` still strips and delivers text before a trailing `NO_REPLY`, and gateway/history suppression is still exact-control-token based. Some cron direct-delivery coverage was later added for trailing `NO_REPLY`, so the patch needs rebasing against current behavior. The PR is paired with the same author’s still-open #42873 dependency and has review comments identifying a failing test and a channelData-drop regression risk, so it should stay open for maintainer/contributor follow-up rather than be auto-closed.

## Best Possible Solution

Keep this PR open for a rebase or replacement patch. The best path is to consolidate it with #42873/#42472, add a shared own-line silent helper with cached regex behavior, update only the still-missing delivery surfaces on current main, preserve channelData/media/presentation semantics, and add focused tests for reply directives, outbound payload planning, gateway live/history projection, and cron subagent follow-up.
