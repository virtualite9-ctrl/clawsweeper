## Summary

Keep this PR open. Current main and the latest release still do not implement an equivalent protected instruction-file mutation guard, and this remains a focused core security-hardening candidate rather than conservative cleanup. The PR also still needs maintainer security/product review because the policy must be complete across mutation paths and compatible with documented SOUL.md and MEMORY.md workflows.

## Best Possible Solution

Keep this PR open for maintainer security review, or replace it with an equivalent central agent-file mutation policy. The best implementation should cover host write/edit, sandbox write/edit, apply_patch add/update/delete/move, normalized path aliases, canonical symlink/hardlink targets, and any intended exec/process scope, with a clear operator approval/exception path for legitimate SOUL.md and MEMORY.md workflows plus regression tests for the bypass classes.
