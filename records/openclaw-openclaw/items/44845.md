## Summary

Current main is not safe to close as implemented. The original field-alias theory is already handled generically, but the remaining reported failure is Volcengine Ark streaming returning no usable usage metadata; current main still depends on provider-returned usage chunks or assistant usage snapshots, and the Volcengine bundled provider does not declare a Volcengine-specific streaming-usage contract. The reporter also confirmed the suggested `supportsUsageInStreaming: false` and `anthropic-messages` workarounds did not solve it, so stale/insufficient-info closure is not appropriate.

## Best Possible Solution

Keep this issue open. The maintainer path is to verify Volcengine Ark coding-plan streaming behavior with live credentials or official API docs, then add a Volcengine-owned fix: provider manifest/request metadata if Ark supports streaming usage, or a documented transport/session fallback if Ark never emits usage in streaming. Add a focused regression or live contract check for `volcengine-plan/ark-code-latest` so `/status` no longer depends on unverified assumptions.
