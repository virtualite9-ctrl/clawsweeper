## Summary

Keep this PR open. Current main still does not strip or separately surface inline `<think>...</think>` reasoning when the TUI receives assistant `content` as a plain string. The PR targets a real remaining gap, but its proposed diff still needs role-gating before merge so user and command transcript text keep literal tag examples intact.

## Best Possible Solution

Keep this PR open for rebase and integration. The best fix is a narrow assistant-only TUI formatter and stream-assembler change that strips inline tagged reasoning from visible assistant string content when thinking display is off, surfaces it separately when thinking display is on, preserves literal tags in user and command messages, and reuses the shared reasoning-tag utilities where possible.
