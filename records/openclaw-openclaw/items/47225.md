## Summary

Keep this PR open. Current main still does not implement the requested context-engine tool schema override: `AssembleResult` has no `toolsOverride`, embedded attempts only consume assembled messages and `systemPromptAddition`, and there is no stream wrapper or test coverage for per-turn schema filtering. The PR is a plausible starting point, but the existing review feedback about empty-array semantics and missing live tool data remains material.

## Best Possible Solution

Keep this PR open for maintainer review or a revised patch. A good implementation should add a backwards-compatible context-engine contract for per-turn tool schema selection, pass sufficient live tool information or define a name-based override clearly, honor empty-subset semantics deliberately, filter at the model-request boundary without broadening executor policy, and add focused tests/docs for the behavior.

## Risks / Open Questions

- Closing now would discard a still-missing plugin API; current main only exposes tool names to context engines and does not let them filter the model-visible schema set per turn.
- The PR as written has unresolved functional gaps from review: `toolsOverride: []` would be treated like no override, and the advertised example depends on live tool objects that current assemble params do not provide.
- The final design should explicitly define whether filtering is name-based or object-based, how it interacts with active tool allowlists and executors, and how an empty subset behaves.
