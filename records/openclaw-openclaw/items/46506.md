## Summary

Keep this PR open. Current main still only removes the cron job record from the cron store; it does not immediately remove the deleted job's session-store entry or per-job run log, and the existing cron session reaper intentionally preserves base cron session keys while only pruning expired run-session keys. The linked bug remains open, and this PR is still a plausible implementation candidate, though it needs maintainer review for the in-flight run/log races and unrelated file churn noted in the discussion and file list.

## Best Possible Solution

Keep this PR open as an implementation candidate for #46369, or replace it with a maintainer-equivalent fix that cleans canonical cron session entries and per-job run logs on cron.remove, uses the removed job's agent-scoped session store, handles or documents in-flight run and pending log-write races, includes focused regression tests, and drops unrelated file changes.
