## Summary

Keep open. Current main still does not tolerate malformed UI chat content blocks in the relevant dashboard path, so this PR is not obsolete or implemented. The linked bug is closed, but the current source still has the same unguarded null/undefined dereference pattern this PR is trying to fix.

## Best Possible Solution

Keep this PR open as a plausible focused bug-fix candidate. The maintainer path is to rebase/update it onto current main, add a shared or local object-record guard that rejects null/undefined/arrays before `type` reads in `message-normalizer` and `message-extract`, retain regression tests for null, undefined, and array-valued blocks, then run the targeted UI chat helper tests or UI test lane.
