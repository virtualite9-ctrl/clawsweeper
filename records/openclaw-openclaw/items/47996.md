## Summary

Keep this PR open. Current main does not implement either central behavior from the PR: queued recovery entries are not expired by age, and `deliverOutboundPayloads` has no reply-shaped direct outbound dedupe/coalescing path. The PR also appears too broad and not merge-ready as-is because the provided diff includes many unrelated Control UI formatting changes and multiple unresolved review comments around behavior-sensitive dedupe keys, recovery replay, callbacks, aborts, and scopes.

## Best Possible Solution

Keep the PR open for now, but ask for a narrow updated branch that only touches outbound recovery/delivery and tests. The implementation should add an explicit stale-entry policy with tests, and any direct-reply dedupe must fail open and key or bypass all behavior-changing contexts: recovery replay/queue identity, session and mirror transcript data, gateway scopes, callbacks, abort state, payload-level reply targets, interactive/channelData payloads, and delivery mode flags. Unrelated Control UI formatting should be dropped from the branch.
