## Summary

Keep open. Current main does not implement this PR's intended first-class Azure Claude AI Foundry onboarding path: there is no `anthropic-azure-api-key` auth choice or dedicated Azure Claude flags/env handling, and the current Microsoft Foundry plugin is OpenAI/Responses or Chat Completions oriented rather than Anthropic `/anthropic` onboarding.

## Best Possible Solution

Keep this PR open for maintainer review or ask the author to rebase and rework it as a provider-owned Azure Claude/Microsoft Foundry implementation. The desired path should add a first-class Azure Claude auth choice with interactive and non-interactive setup, correct `/anthropic` endpoint normalization and validation, env-var support, auth-profile metadata, docs, and regression tests, while resolving the outstanding validation comments and avoiding core hardcoded provider behavior where the plugin API is sufficient.
