## Summary

Keep this PR open. Current main still does not expose provider rate-limit headroom in `openclaw models status --probe`, and the PR is still a relevant implementation candidate for the linked feature request #42401. It is not merge-ready: it needs a rebase onto current main, route/docs/test coverage, and a safer design that captures headers through the normalized provider transport path rather than a separate direct fetch.

## Best Possible Solution

Keep this PR open as a candidate, but require a rebase onto current main. The best implementation should preserve the current lazy embedded-runner import guard, add `--rate-limits` to both Commander and routed CLI parsing, document the opt-in extra provider request at `docs.openclaw.ai/cli/models`, expose a stable JSON/table contract, and capture rate-limit headers through the existing provider runtime/transport path or a narrow generic header-capture seam so provider config, auth, headers, proxy/TLS policy, and plugin-owned behavior are honored. Add focused tests for route parsing, option forwarding, JSON/table output, provider config normalization, and transport safety, and remove the unrelated `.gitignore` log pattern unless separately justified.
