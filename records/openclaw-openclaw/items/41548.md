## Summary

Current main still does not implement the requested unified inbox or automatic TUI/Telegram/iMessage cross-surface live sync. OpenClaw has shared session and transcript primitives, but routing remains origin-channel based, TUI only handles chat/agent events for its active session, and the open related items remain the right coordination points rather than proof that this broader request is shipped.

## Best Possible Solution

Keep this open as a core TUI/Gateway product issue. The likely path is an explicit opt-in unified inbox or observer/mirror mode that subscribes to gateway session.message/sessions.changed events, labels the source channel/session clearly, and preserves origin-channel delivery safeguards. Coordinate the TUI observer scope with #40678 and the real-time Telegram/TUI handoff scope with #51691.
