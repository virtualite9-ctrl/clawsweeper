## Summary

Keep open. Current main has memory indexing/search, append-only compaction flushes, thresholded dreaming promotion, REM/backfill review surfaces, memory-wiki provenance reports, and optional memory-lancedb duplicate checks, but it still lacks the requested resolver-backed durable write pipeline with typed classification, required source attachment, near-duplicate/conflict lookup, create/merge/update/conflict outcomes, and non-destructive revision history.

## Best Possible Solution

Keep this issue open. The best path is to implement a resolver-backed durable write pipeline in or alongside memory-core after aligning with the canonical schema/provenance/review issues: typed candidate records, required source references, entity normalization, duplicate and contradiction lookup, explicit create/merge/update/conflict-for-review outcomes, and non-destructive revision/provenance history. Use #49108 as classifier/entity prior art and coordinate storage/review behavior with #42646, #42647, and #42650.
