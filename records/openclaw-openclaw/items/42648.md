## Summary

Keep this issue open. Current main still has memory indexing/search, append-only flushes, thresholded promotion, REM/backfill review surfaces, and the memory-wiki companion, but it does not yet provide the requested resolver-backed durable write pipeline with typed classification, required source attachment, near-duplicate/conflict lookup, create/merge/update/conflict outcomes, and revision history.

## Best Possible Solution

Keep this issue open. The best path is to implement a resolver-backed durable write pipeline in memory-core after or alongside the canonical schema/provenance/review work: typed candidate records, required source references, entity normalization, duplicate and contradiction lookup, explicit create/merge/update/conflict-for-review outcomes, and non-destructive revision/provenance history. Use #49108 as classifier/entity prior art, and coordinate the storage and operator-review pieces with #42646, #42647, and #42650.
