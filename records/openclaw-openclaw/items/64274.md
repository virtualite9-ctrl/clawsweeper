## Summary

Current main still contains the auth runtime snapshot/path resolution shape described by the issue. MiniMax portal discovery still reads through `ensureAuthProfileStore(ctx.agentDir)`, runtime snapshots are still keyed by resolved auth-store paths, and the related fix PR #64320 was closed unmerged rather than shipped.

## Best Possible Solution

Keep this issue open. Revive PR #64320 or land an equivalent focused fix that keys runtime auth snapshots by stable runtime identity rather than resolved auth-store path, updates get/has/set/replace/save/load callers consistently, and adds a regression test where `OPENCLAW_AGENT_DIR` points at the selected agent while main and that agent have distinct `minimax-portal` profiles.
