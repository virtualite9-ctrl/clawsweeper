## Summary

Current main does not already implement this PR’s intended change. The mirror fingerprint is still computed by `refreshBundledPluginRuntimeMirrorRoot()` while callers are inside the bundled-runtime mirror filesystem lock, and the PR discussion also identifies an incomplete canonical `dist-runtime` path that would need follow-up before merge.

## Best Possible Solution

Keep this PR open and ask for a revision: compute both the main plugin root fingerprint and the canonical `dist-runtime` plugin root fingerprint before acquiring `withBundledRuntimeDepsFilesystemLock()`, thread those precomputed fingerprints through the duplicated `bundled-runtime-root.ts` and `loader.ts` paths, and add a focused test that fails if fingerprinting happens while the mirror lock is held. No security-sensitive unrelated changes were found in the provided diff.
