## Summary

Keep open. Current main still has no MathJax, KaTeX, or markdown math rendering layer in the Control UI chat path. The existing open PR #67103 is an implementation candidate, while prior PR #61510 was closed unmerged, so the issue should remain open until a focused implementation lands.

## Best Possible Solution

Keep this issue open as the canonical Control UI math-rendering request. Review and land #67103 or an equivalent focused implementation that adds sanitizer-safe KaTeX/MathJax support in ui/src/ui/markdown.ts, with regression coverage in ui/src/ui/markdown.test.ts for chat rendering, code spans, invalid formulas, and dollar-sign edge cases. Close only after that implementation merges to main.

## Risks / Open Questions

- Math HTML will flow into unsafeHTML after rendering, so any implementation needs sanitizer-safe KaTeX/MathJax output and regression coverage around allowed tags/attributes.
- Dollar-delimited math can conflict with currency, shell prompts, and code spans; tests should cover inline math, block math, code spans/fences, invalid formulas, and ordinary dollar amounts.
- Because #67103 is still open and unmerged, closing this issue now would hide the canonical user-facing request before the fix ships.
