## Summary

Keep this issue open. Current main has adjacent exec-approval hardening, including shell-wrapper analysis, exact durable command approvals, Windows-only argPattern support, and strict inline-eval prompting, but it still lacks the requested pre-allowlist deny patterns that match the full command string. The supplied discussion and closed PR #47848 show useful prior work and external CRE prior art, but neither is shipped OpenClaw core behavior.

## Best Possible Solution

Keep the issue open as focused core exec-approval hardening. A complete fix should add first-class deny rules to the exec approvals schema/storage, evaluate them before allowlist and ask-fallback decisions against a canonical command string and argv/shell payload, expose management through approvals CLI and Control UI, document precedence, and add regression tests for shell-wrapper bypasses such as zsh -c "openclaw gateway stop" with the shell binary allowlisted. Coordinate with the broader #6615 denylist tracker, but do not close this focused report until maintainers explicitly choose a canonical tracker or a reviewed implementation ships.
