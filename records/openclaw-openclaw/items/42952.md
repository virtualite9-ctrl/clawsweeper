## Summary

Keep open. Current main still has no Feishu `ignoreFileContent` configuration surface, and Feishu inbound file/post media is still downloaded into `MediaPaths` where shared media-understanding can extract text/PDF-like content into prompt `<file>` blocks before the agent turn.

## Best Possible Solution

Keep this issue open and implement a Feishu-owned inbound attachment-content opt-out. Add a documented per-account/top-level Feishu setting, wire it before `resolveFeishuMediaList` downloads file/document resources or before those entries enter `mediaPayload`, preserve placeholder text such as `<media:document> (name)`, and add focused Feishu plus shared media-understanding regression tests. Update Feishu schema/config metadata/help and https://docs.openclaw.ai/channels/feishu.
