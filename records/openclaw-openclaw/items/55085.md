## Summary

Current main does not implement the proposed stable setup-surface export. The CLI still registers `openclaw configure` as an interactive command with no `surface` subcommand, the command descriptor still marks `configure` as having no subcommands, there is no `src/setup` surface exporter/types module, and provider auth methods do not expose the PR's proposed serializable `surface` metadata. This PR remains a plausible core/plugin-SDK boundary change rather than a ClawHub-only item, but it needs normal maintainer review and follow-up on the existing review comments before merge.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is a narrow core implementation of a stable setup-discovery export, or an equivalent public SDK/CLI contract, with versioned types, docs, tests, and fixes for the existing parser/status review comments. Treat the broader configure-session idea in #55557 as related design context, not as a replacement for this discovery-layer PR.

## Risks / Open Questions

- The PR has unresolved correctness review comments around raw argv parsing and channel installed/disabled state, which matter because this is intended as a machine-consumed contract.
- The proposed command writes to a caller-provided path and exports setup status; maintainers should verify path semantics and secret redaction before accepting it as stable.
- This is a core/SDK surface decision. Closing it as ClawHub-only would leave external UIs without the missing OpenClaw-owned discovery contract.
