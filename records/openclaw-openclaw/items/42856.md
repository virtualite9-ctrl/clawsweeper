## Summary

Current main still appears to reproduce the reported source-level behavior, and open unmerged PRs remain as implementation candidates. `config.schema.lookup` is a read-only gateway RPC, but the agent mutation classifier still treats compound gateway actions literally, so `config.schema.lookup` is considered mutating and can feed the user-facing warning path on failure.

## Best Possible Solution

Keep this issue open and review or replace the linked implementation candidates. The likely fix belongs in `src/agents/tool-mutation.ts`: classify compound `gateway`/`cron`/`canvas` actions by their leaf read-only verb, or otherwise explicitly treat `config.schema.lookup` as non-mutating, with regression coverage in `src/agents/tool-mutation.test.ts` and the warning payload path for `config schema path not found`. After an implementation PR merges, close this as implemented by that merge.
