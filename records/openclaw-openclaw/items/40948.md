## Summary

Keep this issue open. Current main still has no `<session-recap>` / `<session_recap>` handling in the inbound metadata stripper, and both Control UI and TUI user-message rendering paths still depend on that stripper. An open, unmerged implementation candidate (#41013) explicitly references this issue with closing syntax, so the right maintainer path is to review/land or close that PR rather than close the issue now.

## Best Possible Solution

Review #41013 or apply an equivalent narrow fix in `src/auto-reply/reply/strip-inbound-meta.ts`: add a session-recap quick check, strip only a leading complete `<session-recap>` / `<session_recap>` block in both `stripInboundMetadata()` and `stripLeadingInboundMetadata()`, preserve mid-message literals, and add regression coverage in `src/auto-reply/reply/strip-inbound-meta.test.ts` plus any focused UI/TUI consumer tests maintainers want. Close this issue only after that fix lands or the linked PR is explicitly rejected with a replacement path.
