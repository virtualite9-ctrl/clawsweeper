## Summary

Keep open. Current main does not implement the HarmonyOS app, HarmonyOS gateway client ID, platform docs, or mobile-node platform handling proposed by this PR. The PR is still an implementation candidate, but it is large, product-sensitive, and has unresolved functional and security-sensitive review findings that need maintainer review rather than conservative cleanup closure.

## Best Possible Solution

Keep this PR open as a HarmonyOS app implementation candidate. Maintainers should decide product fit, then require a smaller or carefully reviewed change set: add any accepted core protocol/mobile-platform support with tests and docs, harden setup-code/bootstrapToken auth parity with iOS/Android, fix the secure storage/onboarding issues, and replace the unsafe build wrapper/dependency flow before merge. If maintainers prefer external incubation, the HarmonyOS app should live in a separate maintained repo until the core client contract is agreed.
