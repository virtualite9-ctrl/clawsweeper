## Summary

Keep this PR open. Current main does not contain the PR's config-level memorySearch validator, so it is not implemented-on-main. The PR also is not cleanly mergeable maintainer cleanup as-is: main already has a runtime-aware doctor memory-search readiness check, while this PR adds overlapping diagnostics plus unrelated cron and agent lifecycle/watchdog modules that need separate product/security review.

## Best Possible Solution

Keep this PR open, but ask for a refreshed, split-down version. The best path is a narrow memorySearch validation/diagnostics PR that preserves current runtime credential and provider fallback semantics, reuses the existing noteMemorySearchHealth flow where possible, adds focused tests for unsupported provider or explicitly invalid fields, and drops the unrelated cron reliability and agent lifecycle/watchdog additions into separate proposals if still wanted.
