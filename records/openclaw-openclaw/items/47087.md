## Summary

Current main does not implement the PR's intended change: native `sessions_spawn` still starts the child `agent` run without `inputProvenance`. The PR is also paired with open #47070 via closing syntax, so it should remain open as the implementation candidate until maintainers review, update, land, or explicitly close both sides.

## Best Possible Solution

Keep this PR open as the implementation candidate for #47070. The likely path is to rebase or recreate the small patch on current main, add `inputProvenance` to the native `spawnSubagentDirect()` child `agent` call with `kind: "inter_session"`, `sourceSessionKey`, `sourceChannel`, and `sourceTool: "sessions_spawn"`, keep focused regression coverage in the spawn tests, then land it so GitHub closes the paired issue.
