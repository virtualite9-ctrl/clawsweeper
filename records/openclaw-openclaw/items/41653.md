## Summary

Keep open. Current main still has the unguarded `WebSocketTaskBox.sendPing()` continuation resume path that this PR targets, and the keepalive loop still exercises it. The functional Swift fix remains useful, but the PR also bundles an out-of-scope CI workflow addition that should be split or dropped before landing.

## Best Possible Solution

Keep the PR open, ask the contributor or a maintainer to remove or split the new workflow, and land a focused `GatewayChannel.swift` fix that guarantees the ping continuation is resumed at most once. If maintainers prefer #51311 or another implementation, choose one canonical PR and close the other only after that path is explicit or merged.
