## Summary

Keep this PR open. Current main still has the generic buffered-chat flush helper and still calls it unconditionally before tool start events, so a buffered assistant text ending in ':' would be flushed before the tool event. The PR's latest diff is still an implementation candidate and appears to address the bot review concern by scoping colon suppression to the pre-tool flush path only.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to review or adapt the PR's scoped `suppressIncompleteNarration` option in `src/gateway/server-chat.ts`, keep terminal/final flush behavior unchanged, and land the colon-vs-period regression coverage if the behavior is accepted. After merge, the linked trailing-colon bug can be closed as implemented.
