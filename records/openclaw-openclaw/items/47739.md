## Summary

Keep PR #47739 open. Current main still unconditionally flushes buffered assistant text before tool-start events, and I did not find the PR's scoped colon-ending suppression or equivalent regression coverage on main. The PR is a focused bug fix with useful tests, and its latest provided commit addresses the review concern by scoping suppression to the pre-tool flush path instead of the final-message path.

## Best Possible Solution

Keep the PR open and rework or land the focused gateway fix: add a scoped option to `flushBufferedChatDeltaIfNeeded` so colon-ending buffered assistant narration is deferred only for pre-tool boundaries, while preserving terminal `emitChatFinal` delta behavior. Keep the positive regression test for colon-ending text and the negative test proving period-ending pre-tool text still flushes.
