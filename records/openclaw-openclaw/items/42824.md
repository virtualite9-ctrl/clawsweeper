## Summary

Keep this PR open. Current main still does not implement the PR’s intended IRC default: when `channels.irc.blockStreaming` is unset, IRC passes `disableBlockStreaming: undefined`, which lets the shared reply resolver inherit `agents.defaults.blockStreamingDefault`. That is the same path described in the linked IRC cut-off bug, and the proposed diff is narrow IRC-owned code plus regression tests with no security-sensitive or supply-chain changes.

## Best Possible Solution

Keep this PR open as the implementation candidate. Rebase or recreate the same IRC-owned change on current main, set `disableBlockStreaming` to true unless `account.config.blockStreaming === true`, keep the unset/false/true regression coverage in `extensions/irc/src/inbound.policy.test.ts`, then run the targeted IRC tests and `pnpm check:changed` before merging or closing only after an equivalent fix lands.
