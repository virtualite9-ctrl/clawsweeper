## Summary

Keep this PR open. Current main still does not implement the intended IRC default: when `channels.irc.blockStreaming` is unset, IRC passes `disableBlockStreaming: undefined`, which lets the shared reply resolver inherit `agents.defaults.blockStreamingDefault`. The PR remains a narrow implementation candidate with regression coverage and no apparent security or supply-chain sensitive changes.

## Best Possible Solution

Keep this PR open as the implementation candidate. Rebase or port the same IRC-owned change onto current main, set `disableBlockStreaming` to true unless `account.config.blockStreaming === true`, keep unset/false/true regression coverage in `extensions/irc/src/inbound.policy.test.ts`, then run targeted IRC tests and `pnpm check:changed` before merging.
