## Summary

Keep open. Current main still does not implement the cron parser nullish guard from this PR, and the PR is a plausible bug fix. It should not be closed as obsolete, but it also should not be accepted as-is without maintainer review because it bundles an unrelated package-update early return that conflicts with the current documented update flow.

## Best Possible Solution

Keep this PR open for maintainer review, but split the work. The cron parser guard and parser tests are still useful; add a CLI regression for malformed `{ kind: "at" }` schedules reaching `cron list` or `cron show`. Remove or split the update-command early return, or redesign it to preserve the documented package refresh, plugin sync, completion refresh, and restart behavior.
