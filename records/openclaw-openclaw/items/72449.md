## Summary

Keep open. Current main still has the narrow cron parser path this PR targets, and the PR’s cron hardening is not implemented on main. The PR is not ready to close as obsolete because it remains a plausible fix, but it also bundles an unrelated update-command early return that needs maintainer review or splitting.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to split or remove the update-command early return, keep the cron parser hardening focused, and add a regression test through cron list/show or schedule formatting for an at-schedule object missing `at` in addition to the parser unit tests.
