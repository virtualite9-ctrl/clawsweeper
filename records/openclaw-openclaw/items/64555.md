## Summary

Keep this issue open. Current main still exposes the WhatsApp default auth directory as an eager module-level value and the main account discovery path still resolves the OAuth root from ambient process env rather than an explicit profile env/state-dir seam. The provided GitHub context also shows open PR #64563 is an unmerged implementation candidate whose body says it will close this issue, so this issue should remain open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and resolve it through #64563 or an equivalent focused patch: lazily resolve the WhatsApp default web auth dir, make account discovery/default/legacy auth-dir resolution align with the active OPENCLAW_STATE_DIR or explicit OAuth root, and add a regression test for profile-isolated WhatsApp credentials. Doctor/docs guidance for OPENCLAW_OAUTH_DIR can follow after the core isolation bug is fixed.
