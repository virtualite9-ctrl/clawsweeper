## Summary

Keep this issue open. Current main still has the WhatsApp runtime default auth directory exported as an eager module-level value, and the bundled WhatsApp account resolver still derives default and legacy auth dirs from ambient OAuth resolution rather than an explicit profile env/state-dir seam. The provided GitHub context also shows open unmerged implementation PR #64563 with closing syntax for this issue, so the issue should stay open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and resolve it through #64563 or an equivalent focused patch: lazily resolve the WhatsApp default web auth dir, align account discovery/default/legacy auth-dir resolution with the active OPENCLAW_STATE_DIR or explicit OAuth root, and add regression coverage for profile-isolated WhatsApp credentials. Doctor/docs guidance for OPENCLAW_OAUTH_DIR can follow after the core isolation bug is fixed.
