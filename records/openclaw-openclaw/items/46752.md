## Summary

Keep this PR open. Current main still has unbounded tool-schema parameters this PR is trying to harden, so the work is not implemented or obsolete. The PR is not merge-ready as-is because review comments identify several caps that would reject currently valid runtime/doc-supported calls.

## Best Possible Solution

Keep the PR open, but require a rebase and a narrower runtime-aligned revision: add schema bounds only where they match current runtime/provider/docs limits, preserve valid long-running subagent and node-media workflows, keep browser snapshot limits aligned with backend caps, preserve Discord GuildMedia support, and add focused schema/contract tests that fail when bounds drift from runtime behavior.
