## Summary

Keep this PR open. Current main does not already implement the PR's intended live-tail user-message behavior for TUI `--session` mode, and the PR itself still needs maintainer review because its provided approach is narrower than the linked Telegram/external-channel bug. The author did address earlier review feedback for local echo suppression, attachment-friendly display text, and schema coverage, but the later review comment about external channel ingress remains material.

## Best Possible Solution

Keep this PR open for maintainer review or revision. The best fix should cover the shared inbound/transcript pipeline used by Telegram and other channels, not just `chat.send`: either teach TUI `--session` to subscribe to and render `session.message` user turns safely, or add a versioned `user-message` chat event emitted from the shared ingress path with protocol schema, TUI handling, echo suppression, and regression coverage. Coordinate with #45388 and the related #41964 work on assistant live events, and split unrelated TDZ/device-auth changes out of this PR if they are still in the branch.
