## Summary

Keep this PR open. Current main still does not classify the raw MiniMax text-only message `High traffic detected. For a more stable experience, upgrade to our Plus plan` as overloaded; it only covers explicit HTTP 529/status-shaped inputs and generic overload/high-demand wording. The PR is narrow and no security or supply-chain regression is apparent, but it still needs a focused regression test and may be cleaner as a MiniMax-owned `classifyFailoverReason` hook.

## Best Possible Solution

Keep this PR open for maintainer review. The best implementation is a MiniMax-owned failover classifier in `extensions/minimax/provider-registration.ts` using the existing `classifyFailoverReason` hook, or an equivalent targeted classifier if maintainers prefer the shared matcher. Add a focused unit test asserting the exact MiniMax high-traffic message classifies as `overloaded` and not `billing`, then land the tested fix or close the PR only after an equivalent implementation reaches main.
