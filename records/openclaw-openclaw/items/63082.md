## Summary

Keep this PR open. Current main still has the reported shell-completion profile-path bug, the linked same-author bug #63069 is still open, and this PR remains an implementation candidate rather than cleanup-close material. The PR also needs changes because its current diff adds the new resolver under the completion CLI file but does not update the runtime install/check path that writes and detects shell completion.

## Best Possible Solution

Keep this PR open and request changes. The best maintainer path is to centralize an env-aware shell profile resolver in src/cli/completion-runtime.ts, use it consistently from installCompletion, isCompletionInstalled, usesSlowDynamicCompletion, doctor/setup/update-facing hints, preserve an intentional bash .bashrc/.bash_profile fallback policy, and add focused runtime tests for zsh ZDOTDIR, fish XDG_CONFIG_HOME, bash fallback selection, and non-Windows PowerShell XDG_CONFIG_HOME if that remains in scope. Then review and land it as the fix for #63069, or close both only after an equivalent implementation ships on main.
