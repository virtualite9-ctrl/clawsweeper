## Summary

Current main already implements the requested skills workspace selection: skills commands resolve `--agent` first, then infer the agent from `process.cwd()` via `resolveAgentIdByWorkspacePath`, then fall back to the default agent. Install, update, and status commands use the shared resolver, with regression tests and docs/changelog coverage. The prior #72726 work is acknowledged and carried forward on main.

## Best Possible Solution

Close this issue as implemented on main. Keep the shared skills workspace resolver, tests, and docs; if #72726 remains open, close or update that PR as carried forward by the main-branch implementation.
