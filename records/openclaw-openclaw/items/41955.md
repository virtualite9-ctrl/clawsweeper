## Summary

Keep this PR open. Current main still does not synthesize dropdown entries for configured agents that have no session-store row, so the linked user problem remains. The PR is stale because the Control UI dropdown now uses `resolveSessionOptionGroups()` in `ui/src/ui/chat/session-controls.ts`, not the older `resolveSessionOptions()` helper targeted by the submitted diff; it should be refreshed or consolidated with related PR #48671.

## Best Possible Solution

Keep this PR open as an implementation candidate, or consolidate it with #48671. The best fix is to update `resolveSessionOptionGroups()` to merge `state.agentsList.agents` into the grouped dropdown, dedupe against stored rows and the active key, preserve configured labels, respect `agents.list.scope` and custom `mainKey`, and add regression tests for sessionless configured agents plus global-scope behavior.
