## Summary

Keep this PR open. Current main does not implement the PR’s intended ownership transfer: bundle-MCP tempDir cleanup is still attached to the per-turn prepared backend cleanup, while Claude live sessions keep a persistent subprocess whose argv can still reference that tempDir. The PR is an active, focused implementation candidate for the linked open bug, not obsolete cleanup.

## Best Possible Solution

Keep the PR open and review it as the active implementation candidate for the linked bug. The best path is to validate that the wrapped cleanup runs exactly once on live-session close/restart/abort and that reuse-turn cleanup remains immediate for unused per-turn artifacts, then land it if CI and targeted coverage pass.
