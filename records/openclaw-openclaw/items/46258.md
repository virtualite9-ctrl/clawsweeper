## Summary

Keep open. Current main does not implement Telegram ack reaction timing control: the shared config field and Telegram override are absent from types/schema/docs, Telegram still starts ack/status reactions during message-context construction, and dispatch has no run-start reaction hook. The PR remains a plausible implementation candidate, with open sibling channel PRs #45421 and #45449 to coordinate shared config work.

## Best Possible Solution

Keep this PR open as the Telegram implementation candidate. Rebase it onto current main, coordinate the shared `messages.ackReactionTiming` type/schema/docs surface with #45421 and #45449, ensure generated config metadata stays aligned, and verify focused Telegram/config tests covering default `received`, deferred `run-start`, gating, remove-after-reply, and `run-start` with status reactions.
