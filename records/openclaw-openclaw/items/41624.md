## Summary

Keep this PR open. Current main still creates `/home/node/.cache/ms-playwright` in the optional Docker browser-install path and only chowns that child directory, while the runtime later switches to the non-root `node` user. The PR remains a narrow one-line Dockerfile fix, and the related duplicate attempt #43846 was closed unmerged rather than superseding it.

## Best Possible Solution

Keep this PR open as the active implementation candidate. The maintainer path is to review the one-line Docker ownership change, request or run targeted Docker proof for `OPENCLAW_INSTALL_BROWSER=1`, and merge it if validation confirms `/home/node/.cache` is writable by the runtime `node` user without introducing unexpected Docker image ownership side effects.
