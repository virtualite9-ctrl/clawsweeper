## Summary

Keep open. Current main partially mitigates the Active Memory example and fresh local onboarding now defaults to the `coding` profile, but the broader least-privilege default requested here is still not implemented: unset/`full` tool profiles remain unrestricted, missing or empty runtime `toolsAllow` preserves the full tool list, sub-agent policy is still mostly denylist-based, and public docs/tests preserve those semantics.

## Best Possible Solution

Keep this open for an explicit maintainer decision on least-privilege tool defaults. A complete fix should define per-agent/per-run default policy semantics, preserve an intentional escape hatch such as `tools.profile: "full"`, warn or migrate existing broad configs, update schema/help/public docs, and add regression coverage for embedded runs, Codex harness runs, plugin/MCP tools, sub-agent policy resolution, cron `toolsAllow`, and runtime `toolsAllow` behavior.
