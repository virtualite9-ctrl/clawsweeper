## Summary

Keep this issue open. Current main has Windows CI coverage, but it is not native PowerShell smoke coverage for contributor commands: the Windows lane defaults to bash and only runs the focused Windows test suite, while docs and strict-smoke commands remain covered through Ubuntu/bash-oriented paths.

## Best Possible Solution

Keep this issue open. Add a scoped native PowerShell CI smoke, either as a small Windows job or an added `checks-windows` matrix task using `shell: pwsh`, for selected contributor commands such as `pnpm check:docs` and a strict-smoke-safe subset. Coordinate with #44293 and the candidate docs-command PRs #44824/#49913, then update `docs/ci.md` to document which contributor commands are guaranteed under native PowerShell.
