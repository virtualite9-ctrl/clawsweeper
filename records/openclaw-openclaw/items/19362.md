## Summary

Keep open. Current main still lacks Discord CHANNEL_DELETE session cleanup, while this PR remains a plausible implementation candidate that needs to be refreshed from the removed `src/discord` layout into the current `extensions/discord` plugin boundary and re-reviewed for safe transcript handling.

## Best Possible Solution

Keep this PR open as an implementation candidate, but require a current-main refresh: add the channel-delete listener under `extensions/discord/src/monitor`, register it through `registerDiscordMonitorListeners`, delete exact `agent:<agentId>:discord:channel:<channelId>` entries from the correct agent stores under `updateSessionStore`, collect deleted entries inside the lock, archive and clean up transcripts through a plugin-safe or core-owned helper with `restrictToStoreDir`, add Discord plugin tests, and run the changed gate before merge.
