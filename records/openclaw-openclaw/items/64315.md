## Summary

Current main still does not implement the requested optional scrolling or marquee-style display for long status information. The Control UI and TUI surfaces I checked use static text, bounded native selects, ellipsis, regular vertical scroll behavior, or direct terminal text updates. Related status-content work such as #13670 remains open and would add more status text, but it does not supersede the overflow-scrolling behavior requested here.

## Best Possible Solution

Keep this issue open as a product/UI enhancement. The best path is to design a small reusable accessible overflow-status component or TUI helper, then adopt it only in selected high-value status, footer, or header surfaces. Related model/token visibility work such as #13670 can provide useful content, but it should not be treated as a replacement for the requested overflow-scrolling behavior.
