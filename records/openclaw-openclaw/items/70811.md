## Summary

Keep this PR open. Current main still validates omitted plugin config as `{}` in the non-activating manifest-snapshot and CLI metadata paths, so the PR's intended fix is not implemented on main. The submitted change remains a plausible implementation candidate, but it needs maintainer review of the deferred-validation behavior and the existing security concern before merge.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land this PR or an equivalent loader change that distinguishes omitted config in non-activating metadata/snapshot loads from real activation, preserves normalized/defaulted `api.pluginConfig`, keeps activation and explicit-config validation strict, retains focused regression coverage in `src/plugins/loader.test.ts` and `src/plugins/loader.cli-metadata.test.ts`, and explicitly resolves the Aisle validation-bypass concern before merge.
