## Summary

Keep this PR open. Current main has not implemented the central .env handling fix: the Docker setup implementation has moved to scripts/docker/setup.sh, still initializes Docker setup knobs from the process environment before any project .env load, still rewrites managed .env keys with bare KEY=value output, and does not pass OPENCLAW_INSTALL_BROWSER through setup builds. The .gitignore part is already present on main, but the behavioral fix remains useful and needs to be ported to the current script with tests.

## Best Possible Solution

Keep the PR open and ask for a rebase/port onto the current scripts/docker/setup.sh implementation. The best fix is a small, non-executing dotenv loader that applies project .env values only for missing process env vars before Docker setup defaults are computed, an upsert writer that preserves sourceable quoting for whitespace/quotes/backslashes, focused e2e coverage for pre-existing .env values and precedence, and removal of the already-implemented .gitignore-only churn.

## Risks / Open Questions

- The PR likely needs retargeting because root docker-setup.sh is now a shim and the active implementation is scripts/docker/setup.sh.
- The .env loader is security-sensitive: the final fix should keep the PR discussion's safer direction of parsing without executing project-controlled .env content, validating identifiers, and preserving caller environment precedence.
- Regression coverage should be added or updated for quoted whitespace values, caller-env-over-.env precedence, invalid/comment .env lines, and OPENCLAW_INSTALL_BROWSER build-arg propagation if that remains part of the intended fix.
