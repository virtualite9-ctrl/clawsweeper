## Summary

Keep open. Current main still derives the Advanced tab daily-log summary and staged section from `groundedCount > 0`, while the short-term recall store keeps `dailyCount` and `groundedCount` as distinct signals. Two open PRs still target this issue, including one with closing syntax, so the issue should remain open until maintainers land or supersede that work.

## Best Possible Solution

Keep this issue open and review, land, or supersede the linked UI fix work. The product fix should make the Daily Log Review summary, staged section, and origin badges consistently use `dailyCount > 0`, unless maintainers intentionally rename the whole surface to grounded replay; add a regression test with daily-only and grounded-only entries.
