## Summary

Keep this issue open. Current main still ships OpenProse with `model:` constrained to `sonnet`, `opus`, or `haiku` in both the VM spec and compiler reference, so the reporter's `model: openai/gpt-5.2` example remains outside the documented/specified OpenProse grammar. OpenClaw's generic `sessions_spawn` path can preserve explicit provider/model refs, which points the remaining work at the bundled OpenProse plugin/spec/docs rather than the already-fixed cron model override issues found in related reports.

## Best Possible Solution

Keep the issue open and fix it in the bundled OpenProse plugin surface: decide whether OpenProse should accept canonical `provider/model` refs directly or require documented aliases, update `extensions/open-prose/skills/prose/prose.md`, `compiler.md`, examples, and `docs/prose.md` consistently, and add focused coverage or a runnable example proving multiple OpenProse agents can spawn with distinct exact model overrides through `sessions_spawn.model` without silently falling back to the default subagent model.
