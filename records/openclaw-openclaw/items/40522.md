## Summary

Keep this PR open. Current main still lacks the requested gateway-root redirect when `gateway.controlUi.basePath` is configured: `/` falls through as non-Control-UI, then the gateway returns `404 Not Found`. The PR remains a narrow implementation candidate and the provided discussion shows the author addressed the query-string test review note.

## Best Possible Solution

Keep this PR open for maintainer review and rebase/reapply. The best implementation is the narrow gateway routing change in `classifyControlUiRequest`: when `basePath` is non-empty and `pathname === "/"`, redirect `GET` and `HEAD` to `${basePath}/${search}`, preserve non-read fallthrough for plugin/webhook routes, keep regression tests for root redirect, query preservation, and write-method fallthrough, and resolve the basePath validation/documentation question before merge.
