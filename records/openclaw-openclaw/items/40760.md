## Summary

Keep this PR open. Current main still lacks the two central behaviors proposed here: fallback/timeout-aware typing TTL budgeting and explicit final timeout guidance. The PR is a plausible implementation candidate, but the existing review blockers remain grounded, and the same-author #41957 plus broader #43204 are still open unmerged candidates that need maintainer consolidation rather than automatic closure.

## Best Possible Solution

Keep this PR open until maintainers either review it directly or consolidate it with #41957 and the broader TTL-feedback work in #43204. The implementation should derive typing TTL from the actual fallback candidate budget, include the transient retry replay budget, use a narrow timeout predicate such as `isTimeoutError` for final timeout copy, and add regression coverage for true timeout failures versus bare socket/DNS errors.
