## Summary

Keep this PR open. Current main still has the Discord thread-reply attachment gap: the guild-admin handler reads only `media`, while the sibling Discord send action supports `media`, `path`, and `filePath`. The PR is also paired with the same-author open bug #43570, so it remains an implementation candidate rather than cleanup.

## Best Possible Solution

Keep this PR open and update it against current main. The best fix is narrow: in `extensions/discord/src/actions/handle-action.guild-admin.ts`, make `thread-reply` read `media ?? path ?? filePath`, forward the same media access options expected by the Discord send/runtime path, and add current Discord action-handler tests for alias behavior, precedence, and media access passthrough before landing it with #43570.
