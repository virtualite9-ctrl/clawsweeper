## Summary

Keep this PR open. Current main still does not implement ACP control-plane audit logging, and this PR is a plausible but not merge-ready implementation of that missing surface. It should not close as implemented, duplicate, or out-of-scope because the requested ACP session lifecycle audit trail is still absent and the related gateway tool-call audit work is a different surface.

## Best Possible Solution

Keep this PR open, but require a rebased and narrower ACP audit implementation before merge. The maintainable path is to preserve current AcpSessionManager lifecycle semantics, wire a real best-effort audit sink under the normal OpenClaw state/log directory with restrictive permissions, cover session init/close/cancel/run/runtime-control events with focused tests, harden buffering/query/retention behavior, and split unrelated config/schema edits out. Treat #63557 as separate gateway tool-call audit work.

## Risks / Open Questions

- Merging the branch as-is could regress current ACP runtime option propagation for model, thinking, and cwd unless it is rebased and adapted to current main.
- Audit emission in ACP lifecycle paths must be best-effort; awaited logger failures should not abort otherwise successful session init or close operations.
- The proposed file sink is security-sensitive and still needs explicit state/log placement, restrictive permissions, bounded buffering, retention enforcement, and query/flush correctness.
- The PR currently mixes the ACP audit feature with unrelated config/schema churn, increasing review surface for a security feature.
