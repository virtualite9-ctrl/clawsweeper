## Summary

Keep this PR open. Current main still rolls stale daily/idle sessions into a new session without emitting the internal command reset hook that bundled session-memory listens for, and the PR remains an active, focused fix with no evident security or supply-chain reason to close it.

## Best Possible Solution

Keep this PR open and review or land it as the focused fix. The best implementation path is to emit the existing internal command:reset event from initSessionState for non-command stale rollovers with commandSource values like auto:daily and auto:idle, while preserving the current explicit /new and /reset hook path and adding daily, idle, and fresh-session regression coverage.
