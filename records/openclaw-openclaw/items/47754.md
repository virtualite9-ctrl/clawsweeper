## Summary

Keep this PR open. Current main has not implemented the intended FTS5 snippet behavior: the active memory-core keyword search still returns raw chunk text and truncates it from the beginning. The PR is directionally useful, but it is stale against the current plugin layout and needs to be ported with current keyword-search behavior preserved.

## Best Possible Solution

Keep this PR open and ask for a rebase/port to `extensions/memory-core/src/memory/manager-search.ts`. The best implementation is to use FTS5 `snippet()` for MATCH-backed keyword results, clamp the token count to SQLite's limit, use visible ellipsis markers, keep existing model/source filters and fallback scoring, handle substring-only fallback safely, and add a regression test where the query term appears deep in a long indexed chunk.

## Risks / Open Questions

- The submitted patch targets the old `src/memory/manager-search.ts` path, so merging it without rebasing would not update the active memory-core plugin code.
- A direct port must account for current trigram substring fallback and fallback lexical scoring; using `snippet()` unconditionally on paths without a MATCH query may not preserve current behavior.
- The PR currently relies on existing tests only. A maintainer-quality fix should add a focused regression test for a match deep in a long chunk.
