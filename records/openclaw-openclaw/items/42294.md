## Summary

Keep this PR open. Current main has generic `audioAsVoice` propagation in some outbound paths, but the Telegram plugin payload adapter still does not translate `payload.audioAsVoice === true` into Telegram `asVoice: true`, so the intended outbound voice-note fix is not implemented on main.

## Best Possible Solution

Keep this PR open and rebase the focused fix onto the current Telegram plugin layout. Thread `payload.audioAsVoice === true` from `sendTelegramPayloadMessages` into `sendMessageTelegram` as `asVoice: true` for media sends, add a regression in `extensions/telegram/src/outbound-adapter.test.ts`, and address the text-only routing concern during the rebase.
