## Summary

Keep this PR open. Current main has overlapping gateway shutdown hardening for lifecycle hooks, MCP/LSP runtime disposal, WebSocket close, and HTTP close, but it still does not implement the PR's central structured per-subsystem warning collection or expose a `ShutdownResult` to callers. The PR also remains stale against newer shutdown code and needs maintainer review/rebase rather than conservative cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to rebase it onto current main, preserve the newer shutdown hardening already present, decide whether `ShutdownResult` belongs in the gateway close contract, reconcile overlap with #56258, and add focused tests for per-subsystem warning collection plus any timeout paths that remain after the rebase.

## Risks / Open Questions

- The PR needs a rebase against newer current-main shutdown behavior, including gateway lifecycle hooks, registered agent harness disposal, bundle MCP/LSP disposal, transcript/lifecycle unsubs, plugin route registry release, and current WebSocket/HTTP close semantics.
- Returning `ShutdownResult` from the gateway close handler changes an internal/public close contract and should be an explicit maintainer API decision before merge.
- Broader raw shutdown error logging could improve debugging, but reviewers should check that subsystem error messages cannot include credentials, tokens, account identifiers, or sensitive configuration values.
- The adjacent open PR #56258 covers per-subsystem shutdown timeouts rather than warning aggregation; maintainers may want to reconcile both approaches instead of landing overlapping shutdown refactors independently.
