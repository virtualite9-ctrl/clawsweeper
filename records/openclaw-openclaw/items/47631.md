## Summary

Current main does not implement this PR’s intended Telegram behavior. The dispatcher still forces Telegram DM preview lanes onto message transport, tests explicitly assert that both DM answer and reasoning lanes use `previewTransport: "message"`, and the docs describe Telegram preview streaming as `sendMessage` plus `editMessageText`. The PR remains a plausible implementation candidate rather than obsolete; its discussion also addressed stale draft cleanup and ambiguous materialize-failure concerns.

## Best Possible Solution

Keep this PR open for maintainer review or a clean rebased successor. The best implementation path is lane-specific Telegram DM transport selection: answer lane uses `auto` so native draft transport can be used in DMs, reasoning lane stays on `message`, and the draft cleanup plus ambiguous materialize-failure protections remain covered by focused regression tests. Do not close it as implemented until that behavior is merged on main or replaced by an equivalent current-main change.
