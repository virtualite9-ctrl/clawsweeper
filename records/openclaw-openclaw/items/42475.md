## Summary

Current main still does not implement per-agent daily/monthly cost budget enforcement. It has cost tracking, provider quota/status surfaces, and transcript/session cost aggregation, but no accepted budget config, no gateway or per-model-attempt spend gate, and no status/doctor budget state. The external ATXP, nornr, and vibe-billing comments are useful prior art/workarounds, not a shipped OpenClaw implementation or a sufficient ClawHub close path for the requested universal gateway guardrail.

## Best Possible Solution

Keep this open as a core/gateway guardrail request. The best implementation path is a documented agents.defaults and agents.list[] budget config contract, a durable per-agent budget ledger or reservation check before every model attempt and fallback, explicit warn/block/pause behavior, status and doctor visibility, docs updates, and regression coverage for gateway user turns, cron, heartbeat, subagents, concurrency, missing cost metadata, and provider fallback chains. If maintainers choose plugin-first, first add a supported blocking pre-model-call budget API rather than treating current observation/tool hooks as sufficient.

## Risks / Open Questions

- A transcript-scan-only cap would race concurrent runs, retries, fallback chains, cron, heartbeat, and subagent execution; hard enforcement likely needs a durable ledger or reservation before model attempts.
- Maintainers still need product/security decisions for approximate pricing, missing model cost metadata, zero-cost/subscription providers, media/tool spend, provider pricing drift, and UTC versus local reset boundaries.
- A plugin-first answer would need a supported blocking pre-model-call budget API and status/doctor projection before it can cover the requested gateway-level guardrail.
