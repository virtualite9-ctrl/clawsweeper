## Summary

Current main still does not implement per-agent daily/monthly model-spend budget enforcement. OpenClaw has cost tracking, provider usage snapshots, disk-budget maintenance, and external proxy/provider options, but no accepted budget config, no gateway/model-attempt spend gate, and no status/doctor budget state.

## Best Possible Solution

Keep this open as a core/gateway guardrail request. The likely implementation path is a documented agents.defaults and agents.list[] budget config contract, durable per-agent spend accounting or reservation before every model attempt and fallback, explicit warn/block/pause behavior, status and doctor visibility, docs updates, and regression coverage for gateway turns, cron, heartbeat, subagents, concurrency, missing pricing metadata, and provider fallback chains. If maintainers choose plugin-first, add the missing blocking pre-model-call budget API first.

## Risks / Open Questions

- A transcript-scan-only implementation would likely race concurrent runs, retries, cron, heartbeat, subagents, and fallback chains; hard enforcement probably needs a durable ledger or reservation before each model attempt.
- Maintainers still need product decisions for approximate pricing, missing cost metadata, zero-cost/subscription providers, media/tool spend, provider pricing drift, and daily/monthly reset boundaries.
- A plugin-first direction would need a supported blocking pre-model-call budget API plus status/doctor projection before it can cover the requested universal gateway guardrail.
