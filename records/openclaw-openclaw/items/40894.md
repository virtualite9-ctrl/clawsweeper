## Summary

Keep open. Current main still does not implement the PR's central media-staging request from #40880: media store, media HTTP serving, sandbox staging, and chat prestage checks still use the hardcoded 5 MB MEDIA_MAX_BYTES path, and the config contract still lacks tools.media.maxBytes. The PR remains a useful implementation candidate, but it is bundled with unrelated transcript-hygiene work and a cross-agent workspace change that is already present on main.

## Best Possible Solution

Keep this PR open as a useful implementation candidate, but do not merge it as-is. The best maintainer path is to rebase and split it: land the tools.media.maxBytes media store/server/sandbox staging fix for #40880 with schema/help/tests, drop the already-implemented cross-agent workspace slice, and move consecutive assistant-error transcript hygiene into its own focused PR with its own bug context and replay-history tests.
