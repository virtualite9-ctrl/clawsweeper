## Summary

Keep open. Current main still has the cron normalization bug this PR targets: `payload.text` and top-level `text` are inferred as `systemEvent` before `hasAgentTurnPayloadHint()` can preserve `model`. The PR discussion shows the author followed up with commit 8aaf0d92 to cover the earlier top-level gap, so this is an active implementation candidate rather than cleanup to close.

## Best Possible Solution

Keep this PR open and review/land the current head if targeted cron normalization tests and CI are green. The desired implementation is the PR’s current shape: promote `text` to `message` and infer `agentTurn` whenever agent-turn-only hints such as `model` are present, in both `coercePayload()` and `inferTopLevelPayload()`, while preserving explicit `kind: "systemEvent"` pruning behavior.
