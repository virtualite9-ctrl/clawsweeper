## Summary

Keep #48373 open. Current main still does not support one-call Feishu document creation with initial content: the `create` branch calls only `createDoc`, while markdown insertion remains in the separate `writeDoc` path. The current schema no longer advertises `content` on `create`, so part of the original schema/docs mismatch has changed, but the requested behavior is still absent. The linked PR #48377 is the direct implementation attempt and remains open in the provided GitHub context; prior PR #16647 was closed unmerged.

## Best Possible Solution

Keep the issue open until a maintainer lands a tested implementation, likely by updating #48377 or an equivalent patch to call `writeDoc` after successful `createDoc` when `content` is supplied, with regression coverage and clear error semantics if the post-create write fails. If maintainers decide not to support initial content on create, the alternative should be an explicit runtime/schema/docs rejection instead of silently ignoring extra input.
