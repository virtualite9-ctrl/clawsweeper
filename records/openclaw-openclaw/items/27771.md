Keep this PR open. Current main still lacks the requested `channels.signal.configPath` contract, schema, docs, and daemon argv propagation. The submitted branch is stale because it targets the removed `src/signal/*` layout, but the feature remains concrete, dependency-supported, and suitable for maintainer takeover in the bundled Signal plugin.

## Best Possible Solution

Port the feature to the current bundled Signal plugin by adding optional `channels.signal.configPath` to the Signal config type, strict schema, generated metadata/UI hints, docs, and account configured detection; propagate it through account resolution and `monitorSignalProvider` into `spawnSignalDaemon`; and add focused tests proving `--config <path>` is emitted as separate argv elements only for nonblank values.
