## Summary

Keep this PR open. Current main does not definitely implement the PR's intended stale runtime-snapshot writeback fix: the public config write path still projects runtime-derived writes onto the stored runtime source snapshot before the inner writer reads the current disk file, so disk-only keys such as `session.dmScope` can still be represented as deletions. The PR is a focused implementation candidate for the linked dmScope persistence bug, but it should be reviewed against the existing bot comments before merge.

## Best Possible Solution

Keep this PR open as a focused implementation candidate for the stale runtime snapshot writeback bug. The best path is to update or review it so runtime writeback uses the latest existing on-disk source snapshot as the projection base, preserves disk-only keys such as `session.dmScope`, handles missing and invalid-readable disk snapshots deliberately, and lands with targeted config writeback regression coverage plus the normal changed gate.
