## Summary

Keep open. Current main still gates the Anthropic beta-header wrapper on non-empty extra-param betas, so OAuth beta injection can still be skipped when the only configured `anthropic-beta` value comes from model/provider headers. The PR is directionally useful, but it targets the older core runner path and needs to be rebased onto the active Anthropic provider plugin. The linked same-author issue #41444 is also still open and this PR uses closing syntax for it, so the paired implementation candidate should remain open until reviewed, rebased, merged, or rejected.

## Best Possible Solution

Keep this PR open, rebase or rewrite it against `extensions/anthropic/stream-wrappers.ts`, update `wrapAnthropicProviderStream` to apply `createAnthropicBetaHeadersWrapper(streamFn, anthropicBetas ?? [])` for Anthropic even when no explicit extra-param betas are resolved, preserve the existing service-tier/fast/thinking wrapper composition, and add plugin-level coverage for `sk-ant-oat` OAuth tokens with model/provider `anthropic-beta` headers but no `context1m` or `anthropicBeta` extra param. Then review and land it with #41444, or close both only if maintainers explicitly reject the product behavior.
