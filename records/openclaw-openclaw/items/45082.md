## Summary

Keep open. Current main still lacks the Mattermost message-tool thread-context inheritance described here, and the issue is paired with open, unmerged implementation PRs by the reporter. The older outbound delivery fallback exists for `sendText`/`sendMedia`, but the direct message-tool plugin action path and the cron/session delivery-context case remain represented by the linked PRs.

## Best Possible Solution

Keep this issue open while maintainers review the linked implementation PRs. The likely product path is to land or supersede #52120 for Mattermost message-tool current-thread inheritance and #52236 for persisting threadId in delivery context for channel/group thread sessions, with regression tests around `replyToMode`, same-target guards, explicit `replyTo` precedence, and cron/heartbeat wake delivery. After the relevant PRs merge, GitHub or a follow-up maintainer pass can close this as implemented.
