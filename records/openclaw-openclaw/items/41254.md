## Summary

Current main still has the heartbeat dedupe race this PR targets: the delivered heartbeat text is written through a load/mutate/save sequence, while the session-store API documents and implements updateSessionStore as the lock-safe re-read path. The PR remains a focused one-file fix and is not implemented, obsolete, or superseded on main.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to rebase if needed, land the updateSessionStore change for the lastHeartbeatText/lastHeartbeatSentAt write, and preferably add a focused regression test proving a concurrent session-store update is preserved.
