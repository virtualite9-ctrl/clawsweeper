## Summary

Keep this PR open. Current main still only has sendPayload coverage for Discord and Slack among the six adapters named by this PR; Google Chat, Mattermost, Microsoft Teams, and Synology Chat still expose sendText/sendMedia without sendPayload, and sendPayload remains optional in the outbound adapter contract. The provided PR diff is partly obsolete because Discord and Slack have newer main-branch implementations and tests, but the central adapter-parity work is not fully implemented on main.

## Best Possible Solution

Keep this PR open for maintainer action, but require a rebase and narrowing before merge. Drop or reconcile the already-shipped Discord and Slack portions, add current-pattern sendPayload implementations for Google Chat, Mattermost, Microsoft Teams, and Synology Chat using the shared payload helpers where appropriate, and add focused outbound payload contract tests for text-only, media-only, multi-media, empty payload, chunking, reply-to fanout, and fallback behavior.
