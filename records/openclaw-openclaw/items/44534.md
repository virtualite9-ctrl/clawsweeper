## Summary

Keep open. Current main still emits `reason: "target-none"` for the default heartbeat delivery target of `none`, and the related rename PR was closed unmerged, so the user-facing confusion reported here remains unresolved.

## Best Possible Solution

Keep this issue open and make a narrow heartbeat DX fix in `src/infra/outbound/targets.ts` plus the affected heartbeat tests. The simplest path is to rename the skipped reason for `target: "none"` to a configuration-oriented value such as `delivery-disabled`; a slightly safer path is to add explicit `rawTarget`/`effectiveTarget` metadata or a CLI hint while preserving the old reason if compatibility matters. The public heartbeat docs at https://docs.openclaw.ai/gateway/heartbeat already establish that `target: "none"` is default no-delivery behavior.
