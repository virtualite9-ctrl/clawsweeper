## Summary

Current main does not provide enough evidence to close this issue. The shared special-token sanitizer exists, and later shared user-visible sanitizer work covers some surfaces, but the TUI main chat finalization path still returns `finalText` or `streamedText` without applying `stripModelSpecialTokens` or the shared assistant-visible sanitizer. The linked fix PR was closed unmerged, so it is not a reason to close this issue as implemented.

## Best Possible Solution

Keep this issue open and implement a narrow TUI fix: apply the shared assistant-visible sanitizer to `finalText` and `streamedText` before `resolveFinalAssistantText` returns them, or sanitize earlier in the TUI assembler extraction path if that better preserves code-block behavior. Add focused TUI regression tests for ASCII and full-width GLM/DeepSeek special tokens, including fallback-to-streamed finalization, while preserving intentional token examples inside code spans/blocks.
