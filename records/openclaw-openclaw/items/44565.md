## Summary

Current main has shipped fixes for the reported Docker/Hostinger browser-tool failure mode: host-side browser requests now use the in-process browser route stack with explicit timeouts, Linux no-display Docker hosts fall back to managed headless browser launch, and browser startup/action timeout handling is covered by tests and release notes after the reported 2026.3.8 version.

## Best Possible Solution

Close as implemented on current main and direct affected Docker/Hostinger users to update from the 2026.3.8 image to v2026.4.26 or newer. If the symptom still reproduces on a current image, the best next issue should include `openclaw browser --browser-profile openclaw doctor --deep`, the browser profile config, gateway/browser logs from the same run, and whether the CDP port is owned by a stale external process.
