## Summary

Keep this PR open. Current main still has the reported Windows onboarding problem: installer OS filtering exists and is tested, but many bundled skill manifests still have unscoped Homebrew install specs, so those install options remain eligible on win32 and can appear in onboarding. The PR is directionally useful, but it needs a rebase/update against current main before it is safe to land.

## Best Possible Solution

Keep the PR open and ask for a rebase/regeneration against current main. The best fix is a manifest-only update that adds `os` to every current brew-only install spec, uses top-level `metadata.openclaw.os` where the whole skill is platform-specific, preserves valid Windows-capable non-brew installers, and keeps the `xurl` fallback on the parser-supported `node` kind.
