## Summary

Current main does not implement the PR's intended dynamic require()/import() detection in the skill scanner, and the PR still appears to be a plausible security hardening candidate rather than obsolete cleanup. However, the proposed regex-only change needs more work before merge because review comments identify real bypasses and false positives for concatenated specifiers, inline comments, and template literals.

## Best Possible Solution

Keep this PR open as a valid security-hardening candidate, but require a more robust implementation before merge. The best path is to add explicit tests in src/security/skill-scanner.test.ts for dynamic variable specifiers, concatenated paths, template literals with and without interpolation, inline comments, static literal imports, and multiline/commented import forms, then update src/security/skill-scanner.ts with a scanner that distinguishes static literals from runtime-resolved specifiers without relying only on the first post-paren character.
