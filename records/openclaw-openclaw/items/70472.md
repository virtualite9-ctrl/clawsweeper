## Summary

Keep open. Current main still has the compaction/overflow recovery path that rotates the active session id, and PR #70479 is an open, unmerged implementation candidate that explicitly links itself to this issue.

## Best Possible Solution

Keep this issue open and review/land PR #70479 or an equivalent fix. The fix should preserve the active agent:main:main mapping when overflow recovery or compaction failure cannot recover a turn, avoid silent session rotation, provide explicit retry/manual-session-control guidance, and keep regression coverage for the next WebChat message continuing under the same sessionId.
