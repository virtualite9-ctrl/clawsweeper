## Summary

Current main only covers part of the PR's stated goal. Cron edit and run history are present, but the PR's job-to-job pipeline orchestration fields and runtime chaining behavior are not implemented on main, and the remaining capability is substantial enough that this PR should not be closed as implemented.

## Best Possible Solution

Keep this PR open for maintainer product and code review rather than closing it automatically. The best path is to either rebase and narrow it to the still-missing cron chaining/i18n fixes with current protocol generation and regression tests, or explicitly supersede the chaining proposal with a Task Flow-oriented follow-up that gives users a supported scheduled pipeline workflow.
