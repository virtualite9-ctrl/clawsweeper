## Summary

Keep this PR open. Current main does not implement the intended `openclaw devices list` improvement: the CLI still renders one `Device` column for pending and paired devices, omits `Client`, `OS`, and `Last active`, and does not expose the paired `deviceId` when a display name is present. The backend already carries the metadata needed for this feature, so the work is still plausible, but the current PR diff needs maintainer review/update before merge.

## Best Possible Solution

Keep the PR open and have a maintainer or contributor update it: split Device ID and Name for both pending and paired tables, add Client/OS/Last active from the existing pairing metadata, preserve `sanitizeForLog` on every device-controlled table value, and keep focused tests for pending and paired rendering. Because current main has not shipped the behavior, closing this PR as implemented would be premature.
