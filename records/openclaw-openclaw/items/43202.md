## Summary

Keep this issue open. Current main still reconstructs Feishu card-action events with only `action.value` and `action.tag`, then the legacy handler path serializes only `action.value`, so `form_value`, `input_value`, `name`, `option`, and `options` are still dropped. The provided GitHub context also shows open implementation PR #43953 by the same author with closing syntax, so the maintainer path is to review and land or reject that PR rather than close this issue now.

## Best Possible Solution

Keep this issue open and review the linked Feishu implementation PRs, especially same-author PR #43953. The durable fix should preserve `form_value`, `input_value`, `name`, `option`, and `options` in `parseFeishuCardActionEventPayload`, expose them on the typed `FeishuCardActionEvent` contract, serialize them only when present so simple button behavior stays backward compatible, and add regression coverage from a raw `card.action.trigger` callback through handler dispatch.
