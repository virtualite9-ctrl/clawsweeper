## Summary

Keep this issue open. Current main still advertises inline attachment fields for `message.send`, but the shared send path does not hydrate `buffer`/`filename` into send media and the Telegram plugin still forwards only `mediaUrl`/`media` to `sendMessageTelegram`. Open PR #63578 remains the active implementation candidate rather than proof that the bug is fixed.

## Best Possible Solution

Keep this issue open until main either delivers `buffer`/`filename`/`mimeType` attachments for `message.send` through the shared outbound path and Telegram plugin, or rejects those fields explicitly for `send` with a clear error. The most direct maintainer path is to review, update, and land or replace #63578 with focused tests for direct buffer sends and Telegram dispatch.
