## Summary

Keep this issue open. Current main appears to cover the threaded outbound reply path, but the second requested capability, receiving all Google Chat space messages via user OAuth or equivalent event subscriptions, is still not implemented. The related open PR #68967 is also only a candidate for the first half, not a reason to close this issue before merge, and the remaining user-OAuth/all-message-delivery work is still real.

## Best Possible Solution

Keep this issue open or split it after #68967 is resolved. The best implementation path is extension-owned: add a Google Chat user-OAuth or verified event-subscription mode in extensions/googlechat, extend the config/schema/docs for that credential and delivery mode, and add focused tests proving non-mention space messages can be accepted without weakening the existing service-account webhook security model.
