## Summary

Keep this PR open. Current main only partially overlaps with the PR: subagent announce delivery now defaults to 120s and is configurable, but the sessions_send A2A wait path still has the hard 60s cap that this PR is trying to remove. The old diff is stale and needs a bounded timeout policy before it is mergeable, but the intended behavior is not implemented on main.

## Best Possible Solution

Keep this PR open for maintainer review, but rework it against current main: preserve or explicitly change the shipped 120s subagent announce default, add a bounded/configurable timeout policy for sessions_send A2A, remove inappropriate 60s caps consistently, and add focused regression coverage for waits above 60s plus the fire-and-forget A2A path.
