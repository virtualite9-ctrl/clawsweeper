## Summary

Keep open. Current main still does not implement this PR's intended no-flag 1M resolver behavior for Anthropic Opus/Sonnet 4.6, and the linked bug remains open. The PR is a plausible implementation candidate, but it needs rework rather than cleanup closure because the diff changes only the context resolver/tests while current provider docs, header tests, and model-specific 1M behavior are not aligned.

## Best Possible Solution

Keep this PR open for maintainer review or rework against #66766. The best path is a coordinated fix that narrows automatic 1M handling to the exact Anthropic models with native 1M support, preserves configured/discovered overrides where appropriate, updates status/session and compaction expectations, and aligns provider docs plus header tests with the current Anthropic contract. ([platform.claude.com](https://platform.claude.com/docs/en/release-notes/overview))
