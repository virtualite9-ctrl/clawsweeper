## Summary

Current main implements the broader WebChat/Control UI non-image upload path that the April 25 maintainer comment said was still missing after image-only support: the UI accepts non-video files, previews them, serializes them as `type: "file"`, and the Gateway stages them as managed media paths accessible to the agent. Assistant-side attachment links are also rendered for downloadable history/media references.

## Best Possible Solution

Close this issue as implemented on current main. Keep the shipped Control UI/WebChat non-video attachment path, and track any future polish such as generic file paste, richer per-format extraction, or additional preview/download UX as narrower follow-up issues.

## Evidence

- **Control UI accepts common non-video file types:** `CHAT_ATTACHMENT_ACCEPT` includes images, audio, PDFs, text, CSV/JSON/Markdown/TXT, ZIP, Office document/spreadsheet/presentation extensions, and the helper rejects video MIME/extensions only.

- **WebChat file picker and drag/drop create file attachments:** The chat view reads selected and dropped non-video files with `FileReader`, preserves filename/MIME metadata, and previews non-image files as attachment chips; paste remains image-scoped, but the requested upload/drag path is implemented.
  - file: [ui/src/ui/views/chat.ts:252](https://github.com/openclaw/openclaw/blob/a25366038503adcc31a5fa64b3d2f295bd5dbfd2/ui/src/ui/views/chat.ts#L252)

  - file: [ui/src/ui/controllers/chat.ts:460](https://github.com/openclaw/openclaw/blob/a25366038503adcc31a5fa64b3d2f295bd5dbfd2/ui/src/ui/controllers/chat.ts#L460)

  - file: [src/gateway/server-methods/chat.ts:1884](https://github.com/openclaw/openclaw/blob/a25366038503adcc31a5fa64b3d2f295bd5dbfd2/src/gateway/server-methods/chat.ts#L1884)

  - file: [ui/src/ui/views/chat.test.ts:451](https://github.com/openclaw/openclaw/blob/a25366038503adcc31a5fa64b3d2f295bd5dbfd2/ui/src/ui/views/chat.test.ts#L451)
