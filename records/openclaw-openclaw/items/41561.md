## Summary

Keep this PR open, but it should be reworked rather than merged as-is. Current main has moved thinking support to provider-owned profiles, and direct OpenAI/OpenAI Codex/GitHub Copilot xhigh support is already implemented there. The Vercel AI Gateway proxy case described by this PR still appears missing, because the plugin catalogs upstream OpenAI refs but does not expose a thinking profile hook, while sessions.patch validates explicit xhigh against the effective provider/model. The PR branch is also stale against current main and includes unrelated/broken web-search registry churn, so the useful behavior should be salvaged in the current provider-profile architecture.

## Best Possible Solution

Keep this PR open for maintainer or contributor rework. Rebase onto current main, drop unrelated web-search and test-seam churn, and implement the proxy xhigh behavior through provider-owned thinking profiles, likely in extensions/vercel-ai-gateway or a narrow shared helper for trusted upstream refs. Add focused thinking and sessions.patch regression tests showing accepted vercel-ai-gateway OpenAI refs and rejected unsupported refs.
