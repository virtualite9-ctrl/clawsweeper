## Summary

Keep this PR open. Current main still implements only partial Signal quote context: it exposes quoted body/sender/isQuote, but does not carry Signal ReplyToId, does not wake immediately on reaction notifications, and does not send native Signal quote-timestamp/quote-author parameters for outbound quoted replies. The related prior PRs (#40292, #47633, #47368) do not supersede this PR because they are closed and main still lacks the central remaining behavior.

## Best Possible Solution

Keep this PR open and review the current head against main. The best implementation path is a focused Signal-channel change that sets ReplyToId only when quote visibility permits it, wakes the target session after emitted reaction notifications, sends quote-timestamp/quote-author only on the first outbound chunk/media, covers monitor deliveries and attached-results sends, explicitly resolves or scopes group quote-author behavior, and adds regression tests for each path. If maintainers prefer a cleaner replacement, supersede this PR with a canonical focused PR rather than closing the work as already implemented.
