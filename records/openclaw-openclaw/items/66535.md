## Summary

Current main still matches the reported cancellation gap, and the open linked implementation candidate has not merged. Manual `/compact` and gateway `sessions.compact` both call `compactEmbeddedPiSession` without passing an `abortSignal` or registering the compaction in the active embedded-run registry, while `/stop` and `abortEmbeddedPiRun(sessionId)` only work when a run is discoverable through those registries. The CoClaw downstream workaround described in the issue is consistent with the core bug still being present, not a replacement for fixing it.

## Best Possible Solution

Keep this issue open and review, land, or replace PR #66894. The fix should create an `AbortController` for manual and gateway compaction, register an `isCompacting()` active-run handle only after any previous run drains, pass `abortSignal` into `compactEmbeddedPiSession`, clear the handle in `finally`, and add regression coverage proving `/stop`, `chat.abort`, or `abortEmbeddedPiRun(sessionId)` cancels an in-progress compaction.
