## Summary

Keep this issue open. Current main still skips `MEDIA:` directives inside fenced Markdown before token validation/extraction, in-tree tests assert that behavior, and the only related implementation PR in the provided context is closed unmerged.

## Best Possible Solution

Keep this issue open. The likely fix belongs in `src/media/parse.ts` and `src/media/parse.test.ts`: either extract valid own-line fenced `MEDIA:` directives while preserving the existing validation/security rules, or keep skipping fenced directives by design but add an observable warning and update assistant/docs guidance. The closed #41980 PR is useful prior art but should be reviewed or replaced rather than treated as shipped.
