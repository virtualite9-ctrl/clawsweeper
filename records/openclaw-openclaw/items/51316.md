## Summary

Keep open. Current main still disables memory flush/preflight when `reserveTokensFloor + softThresholdTokens >= contextWindowTokens`, and the linked bug #50611 is still represented in the current gate logic. The PR is not safely closeable as implemented or superseded: its latest draft discussion still asks for maintainer direction, and the provided diff changed toward returning `false` for oversized reserves rather than clearly fixing the reported behavior.

## Best Possible Solution

Keep this PR open for maintainer judgment or supersede it with a canonical fix for #50611. The best code path is a shared, context-window-aware effective-reserve calculation used by both preflight compaction and token-triggered memory flush, with regression tests for `reserveTokensFloor >= contextWindowTokens` and small-context models. The PR should also drop the unrelated lockfile churn before merge consideration.
