## Summary

Current main has newer Gateway and partial Control UI freshness handling, but it does not fully implement this PR’s intended TUI/Web UI behavior. The TUI still does not model or consume `totalTokensFresh`, the footer still displays `sessionInfo.totalTokens` without a freshness guard, and the Control UI table formatter still ignores `totalTokensFresh`. Related merged PR #71297 explicitly treated this work as related but intentionally outside its scope.

## Best Possible Solution

Keep this PR open, but require a small rebase/update: add `totalTokensFresh` to the TUI session list/state types, make TUI state clear or ignore `totalTokens` when freshness is false, add the Control UI `formatSessionTokens` guard, preserve fresh-null clearing semantics, and add regression tests for TUI stale clearing plus the Control UI formatter. The security-sensitive surface is low because the change should remain limited to UI/TUI display code.
