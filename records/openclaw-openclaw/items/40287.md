## Summary

Keep this PR open. Current main still lacks the requested Slack `file_shared` Socket Mode fallback, and the supplied discussion shows this PR is the remaining canonical implementation candidate after related Slack oversized/multi-file reports were closed or cross-referenced. Some ancillary Slack file handling has since landed, but it only covers normal `message`/`file_share` delivery and file placeholders, not the separate `file_shared` event path.

## Best Possible Solution

Keep this PR open as the canonical implementation candidate, but require a rebase or replacement port into `extensions/slack`: add a `file_shared` event handler under `extensions/slack/src/monitor/events`, register it from `extensions/slack/src/monitor/events.ts`, hydrate the matching Slack message safely through a bounded `conversations.history` lookup, dedupe against normal message delivery and Slack retries, preserve current account/team/channel/user authorization and media SSRF protections, update `extensions/slack/src/setup-shared.ts` plus `docs/channels/slack.md`, and add focused Slack plugin tests for large-file and multi-file fallback behavior.

## Risks / Open Questions

- This PR cannot merge as-is because it targets the removed `src/slack` layout; the current implementation belongs under `extensions/slack`.
- A correct port still needs focused tests for duplicate `file_shared` delivery, dedupe against normal `message`/`file_share` delivery, multi-file uploads, busy-channel history lookup, authorization gates, and app mention interactions.
- The fallback history lookup should be anchored and bounded, and should avoid noisy or broad Slack history reads/logging.
- Setup manifest and public Slack docs must add the `file_shared` bot event, otherwise the runtime handler will not receive the fallback in normal user setup.
- A port must preserve existing Slack media protections: Slack-host URL validation, bearer-token handling, channel/thread scoping for file downloads, team/app event mismatch drops, and channel/user authorization before dispatch.
