## Summary

Current main does not implement the PR's intended `before_tools_resolve` plugin hook or an equivalent list-level, pre-LLM tool filtering surface. Existing main only exposes per-tool call hooks plus static allow/deny policy, while agent and compaction tool assembly still builds final tool lists without invoking a plugin hook over the list. The PR is still potentially useful but security-sensitive, so it should stay open for maintainer review rather than be closed as obsolete.

## Best Possible Solution

Keep this PR open for maintainer product and security review. If accepted, implement the hook against the current `src/plugins/hook-types.ts`/`src/plugins/hooks.ts` structure, evaluate it once over the final tool set in both normal runs and compaction, include core/client/bundled tools, normalize allow/deny names, fail closed for policy errors, and add docs plus focused regression tests for multi-hook allow intersection and bypass cases.
