## Summary

Keep this PR open. Current main still has the same peek/drain shape described in the PR and in the supplementary PR discussion: main-session cron jobs enqueue the raw system event with a `cron:` context key, heartbeat prompt construction peeks that queue and wraps cron-tagged events in the scheduled-reminder prompt, and the reply path can still drain the same queued event into `System:` lines for the agent turn. The PR is not obsolete or superseded by current main, and its diff is a focused behavior fix plus regression coverage.

## Best Possible Solution

Review and land this PR, or apply an equivalent fix on main: reserve/centralize a main-session cron system-event marker or otherwise coordinate heartbeat peek/drain so main-session `payload.kind="systemEvent"` jobs are not wrapped a second time, and keep regression coverage for `sessionTarget="main"`, `wakeMode="now"`, and `delivery.mode="none"`.
