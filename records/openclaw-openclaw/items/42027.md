## Summary

Keep this PR open. Current main has already solved the exec PATH and cron deadlock parts through different designs, and it has WSL2 remote CDP troubleshooting docs, but the PR still contains useful browser status/error-diagnostics work that has not been ported into the current `extensions/browser` plugin layout. The discussion also includes real WSL2 + Windows validation from Owlock supporting that browser diagnostics direction.

## Best Possible Solution

Keep the PR open, but require a rebase or split. Drop the exec and cron portions already handled on main, then port the still-missing browser layered status diagnostics and bounded HTML upstream-error summarization into `extensions/browser`. Resolve the existing review comments and verify with focused browser CLI/client/server tests plus a security review of config-redaction and error-reflection paths.

## Risks / Open Questions

- Merging the PR as-is would be stale against current `extensions/browser` plugin paths and current exec `host=auto` plus cron `cron-nested` semantics.
