## Summary

Keep this PR open. Current main still serves Control UI index HTML without rewriting favicon or apple-touch-icon hrefs for a configured basePath, while the source HTML still contains root-relative icon links. The PR remains a focused candidate for the linked #41205 regression, with no visible supply-chain or secret-handling risk in its provided file list, but it needs to be rebased onto the current CSP-aware index-serving path.

## Best Possible Solution

Keep this PR open as a valid focused bug-fix candidate. Rebase it onto current main, integrate the favicon/apple-touch-icon basePath rewrite into the CSP-aware serveResolvedIndexHtml path, preserve the function replacer, and keep the tests focused on rewritten icon hrefs plus unchanged non-icon hrefs.
