## Summary

Keep this PR open. Current main has SSH tunnel support for transient `gateway probe --ssh` and macOS remote mode, but it does not provide the PR's intended persistent `openclaw tunnel up/down/status` CLI, PID state, or docs page. The PR is still a plausible feature candidate, but it needs a refresh against the current CLI registration structure plus fixes for the review-noted process/state safety issues before maintainer merge consideration.

## Best Possible Solution

Keep the PR open for maintainer/contributor follow-up. The best path is to refresh it onto the current descriptor-based CLI registration, reuse or extend `src/infra/ssh-tunnel.ts` instead of duplicating weaker readiness checks, add ownership-safe persistent state for `up/down/status`, ensure state directory creation, and add targeted tests for stale PID, reused PID, port collision, invalid target, and fresh-state-dir behavior.
