## Summary

Keep this PR open. Current main covers the Langfuse portion through a generic OpenTelemetry/plugin-hook direction, so that part should not merge into core as-is, but the cron/subagent completion delivery-confirmation bug fix is still not implemented on main.

## Best Possible Solution

Do not merge this PR as-is. Split out a small core PR that makes agent delivery return explicit external-delivery confirmation and makes subagent/cron completion direct delivery require that confirmation or fall back. Move the Langfuse PoC to a ClawHub/plugin implementation using diagnostics events and plugin hooks; if that adapter exposes a real missing public API, propose a generic plugin SDK seam.
