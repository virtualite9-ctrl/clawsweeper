## Summary

Current main still has the main Discord self-message gap. Bot identity lookup can fail to an undefined id, the live self-filters only run when that id is present, and pending-history/replay/followup paths still lack trusted Discord author/bot identity for a second self-filter. The /new queue cleanup path is improved, but that only covers one symptom. The related implementation PR #46338 is closed unmerged, so it does not supersede this issue.

## Best Possible Solution

Keep this issue open. Implement a Discord-owned fix that makes bot identity reliable or fail-closed, carries trusted Discord author/bot identity through pending history and queued followup metadata, re-applies self-authored-message protection before replay/followup execution, and keeps the current /new queue cleanup covered by regression tests.
