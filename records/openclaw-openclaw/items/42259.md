## Summary

Keep this PR open. Current main still does not implement configurable image-tool JPEG quality or max-side controls, and this PR remains an implementation candidate rather than obsolete. It should not merge as-is because the reviewed PR head has unresolved constraint fallback bugs and includes unrelated docs/gateway/provider-link changes.

## Best Possible Solution

Keep this PR open as an implementation candidate, or supersede it with a focused rebased PR. The best patch should add a typed and documented agents.defaults image-tool compression contract, thread it through createImageTool, loadWebMedia, and optimizeImageToJpeg, honor arbitrary out-of-ladder values such as quality 95/85 and maxSide 600, preserve current SSRF and local media access guards, add regression tests for those edge cases, update generated config metadata/docs, and split unrelated docs/gateway/provider-link changes into separate work.
