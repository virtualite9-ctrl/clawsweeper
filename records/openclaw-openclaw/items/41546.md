## Summary

Keep this PR open. Current main still clears the Overview Gateway Token on WebSocket URL input, so the intended bug fix is not implemented. The linked same-author bug #41545 is still open, and this PR is a real implementation candidate, but the patch is not safe as-is because it preserves a selected-gateway token across arbitrary endpoint changes.

## Best Possible Solution

Keep this PR open for maintainer review or author rework. The best fix is to separate in-progress URL editing from confirmed endpoint changes: preserve the typed token for same normalized target or whitespace-only edits, but clear it or require explicit confirmation/token entry when the normalized gateway target changes, with regression tests for both cases and no backend auth or storage schema changes.
