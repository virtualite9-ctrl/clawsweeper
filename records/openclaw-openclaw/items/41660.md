## Summary

Keep this PR open. Current main still emits the ambiguous invalid-config doctor note from `runDoctorConfigPreflight`, while gateway startup still blocks unrecovered invalid configs. The PR remains a useful focused wording fix, but its old edit location has moved and it needs to be ported to the current preflight helper.

## Best Possible Solution

Keep the PR open as the canonical implementation candidate for #40651, port the wording change to `src/commands/doctor-config-preflight.ts`, and add a focused assertion for the invalid non-legacy config note if feasible.
