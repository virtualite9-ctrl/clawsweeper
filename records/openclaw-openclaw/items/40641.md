## Summary

Current main still contains the reported unquoted bracket-notation config examples in the English source docs, including the exact `agents.list[0].tools.exec.node` command. The CLI parser still supports bracket notation once the argument reaches OpenClaw, so this remains a documentation/shell-quoting bug rather than an implemented fix. Related PR #40642 proposed this docs fix but was closed unmerged.

## Best Possible Solution

Keep this issue open as a concrete docs regression. Update `docs/cli/config.md`, `docs/tools/exec.md`, and `docs/nodes/index.md` to quote bracket-notation config paths or switch examples to shell-safe dot-index paths like `agents.list.0.tools.exec.node`, and add a short note explaining that bracket paths containing `[]` must be quoted in shells that treat brackets as glob patterns. Use #40642 as useful prior direction, but land a fresh docs-source change because that PR is closed unmerged.
