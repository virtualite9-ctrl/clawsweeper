## Summary

Current main does not implement this PR's intended hook announcement policy. The gateway hook path still falls back to a main-session system event solely on `!result.delivered`, `deliver:false` is still normalized and passed through to `delivery.mode = "none"`, the result type has no `announceToMain` policy field, and current tests still encode fallback behavior for a `deliver:false` hook result. The PR is not obsolete; it remains an active implementation candidate for the linked hook fallback bug.

## Best Possible Solution

Keep this PR open as the active implementation candidate. The best path is to review and land a tested gateway hook announcement policy that separates main-session fallback from outbound delivery success, keeps successful `deliver:false`, already delivered, and handled `deliveryAttempted` hook results out of the main session, still surfaces real hook errors, and fixes the `Hook Hook` prefix. Close only after this PR or an equivalent main-branch implementation lands.
