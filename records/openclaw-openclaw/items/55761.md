## Summary

Current main does not implement this PR's intended hook announcement policy. `dispatchAgentHook` still falls back to the main session solely on `!result.delivered`, so successful `deliver:false` hook runs can still enqueue `Hook ...` system events, and the default-name `Hook Hook` label path is still present. The PR is a focused implementation candidate rather than obsolete cleanup.

## Best Possible Solution

Keep this PR open as the active implementation candidate. The best path is to review the hook announcement policy, settle the `deliveryAttempted` contract, and land a tested fix that keeps successful `deliver:false`, already delivered, and handled delivery-attempted hook results out of the main session while still surfacing real hook errors and fixing the `Hook Hook` prefix.
