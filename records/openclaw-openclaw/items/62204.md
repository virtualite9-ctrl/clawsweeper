## Summary

Keep this PR open. Current main still does not strip `OPENCLAW_SERVICE_*` or `OPENCLAW_SYSTEMD_UNIT` from exec-spawned child environments, and the linked bug report #61095 remains open with this PR as the implementation candidate. The PR diff is narrow and security-positive rather than obsolete.

## Best Possible Solution

Review and land this focused policy/test/generated-Swift fix, or apply the same behavior directly on main, so exec-spawned host child processes strip gateway service markers without removing the intentional `OPENCLAW_SHELL=exec` marker. After that merges, the linked bug report #61095 can close as implemented.
