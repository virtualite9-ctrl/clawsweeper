## Summary

Keep this PR open. Current main still accepts `openclaw cron add --message ...` without `--agent`, forwards `agentId: undefined`, and has no missing-agent warning or regression coverage. The linked cron UX issue remains open, and the related cron-tool agent inference PR is a different path, not a replacement for this CLI warning.

## Best Possible Solution

Keep this PR open as a focused implementation candidate. Refresh it on current main, add the warning in `src/cli/cron-cli/register.cron-add.ts` after payload classification so it applies only to `agentTurn` jobs, emit on stderr, keep wording accurate for configured default agents, and add focused tests in `src/cli/cron-cli.test.ts` for missing, blank, explicit-agent, session-key, and system-event cases.
