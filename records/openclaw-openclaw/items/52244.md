## Summary

Keep open. Current main has related infrastructure, but it does not implement this PR's central OpenRouter dynamic capability-cache defaulting. OpenRouter dynamic model resolution can mark an uncatalogued runtime model as reasoning-capable, but the default thinking/reasoning path still resolves from catalog/provider thinking hooks, and the OpenRouter provider does not currently add dynamic cache rows or a thinking profile/default hook. The existing review comments point at the same integration area, so this PR should be rebased and reworked rather than closed as implemented.

## Best Possible Solution

Rebase and rework this PR against current main. The likely fix belongs in the OpenRouter provider: add a provider-owned `resolveThinkingProfile`/default policy and, if reasoning defaults still need catalog data, an `augmentModelCatalog` path backed by the OpenRouter capability cache. Add a focused regression test for an uncatalogued OpenRouter reasoning model that covers default thinking/reasoning resolution and visible reasoning delivery.
