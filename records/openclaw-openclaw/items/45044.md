## Summary

Keep this PR open. Current main still has the broad `!hasAssistant` reset guard in `prepareSessionManagerForRun`, so the PR's intended fix is not implemented on main. Adjacent restart-drain work reduces the common restart window, but it does not change the session-file initializer that can truncate an already incomplete persisted transcript.

## Best Possible Solution

Keep this PR or a maintainer-owned successor open. The best implementation is to narrow `prepareSessionManagerForRun` so only true header-only pre-created files are reset, preserve any file that already contains a message, add focused regression coverage for `[header]`, `[header,user]`, and `[header,user,assistant]`, and make the test portable with `os.tmpdir()`/`path.join()`. The defensive archive before truncation is reasonable to consider, but the core fix belongs in `src/agents/pi-embedded-runner/session-manager-init.ts`.
