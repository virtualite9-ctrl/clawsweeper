## Summary

Keep this PR open. Current `main` has moved the relevant code out of the original `src/memory/internal.ts` path, but the same swallowed `mkdirSync({ recursive: true })` failure still exists in the current memory-host-sdk implementation that memory-core uses. The PR is stale and incomplete as-is, not obsolete.

## Best Possible Solution

Keep this PR open and ask for a focused rebase onto current `main`: remove the swallowed `mkdirSync` failure in `packages/memory-host-sdk/src/host/internal.ts`, keep the current source/package memory-host-sdk layout coherent, and add a targeted Vitest regression test that stubs `fsSync.mkdirSync` to throw and asserts `ensureDir()` propagates the error. Avoid unrelated memory indexing or package-surface changes.
