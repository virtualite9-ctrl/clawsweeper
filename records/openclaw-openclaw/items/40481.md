## Summary

Current main still delivers the generic exec-approval fallback text with `/approve <id>` instead of concrete copyable commands, so the PR is not obsolete. The submitted diff is still outdated against current main because it hard-codes `allow-always` instead of preserving the existing allowed-decision filtering for effective `ask=always`; its final file list is scoped to source and tests with no visible CI, dependency, workflow, lockfile, or supply-chain change.

## Best Possible Solution

Keep this PR open and rework it against current main. The best fix is to update the generic exec approval fallback in `src/infra/exec-approval-forwarder.ts` so it renders concrete per-decision `/approve <actual-id> <decision>` lines from the existing allowed-decision path, applies markdown formatting only for markdown-capable channels, preserves `ask=always` filtering, and adds focused coverage in `src/infra/exec-approval-forwarder.test.ts` for markdown fallback, non-markdown fallback, and `ask=always` without `allow-always`.
