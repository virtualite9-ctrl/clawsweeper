## Summary

Keep this issue open. Current main preserves Telegram forum topic threadId for sessions_send announce delivery, but the reported ANNOUNCE_SKIP visible-delivery fallback and requester transcript pollution/ping-pong behavior are still present in the A2A path. The related full-fix PRs #43869 and #44064 are closed and unmerged, so they do not supersede the remaining work.

## Best Possible Solution

Keep this issue open. A maintainer fix should preserve the shipped threadId routing, add a visible-delivery fallback for Telegram topic handoffs where the announce step returns ANNOUNCE_SKIP after a real target reply, and suppress or redesign same-agent Telegram topic-to-topic ping-pong before it records fresh inter_session turns in the requester session. Closed PRs #43869 and #44064 are useful prior-art references but do not currently supersede the remaining work.
