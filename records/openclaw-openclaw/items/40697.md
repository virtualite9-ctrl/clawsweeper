Keep this PR open. Current main still does not expose an agent-facing `feishu_bitable_delete_record` tool; record deletion is only used internally for new-Bitable cleanup. The related broader PR #41368 and feature tracker #55654 are still open and not shipped, so they are not a merged or canonical replacement yet.

## Best Possible Solution

Keep this PR open for maintainer review or consolidation. The best implementation path is a refreshed focused change in `extensions/feishu/src/bitable.ts` that adds `feishu_bitable_delete_record`, preserves contextual and explicit account routing, keeps the irreversible-action warning, and adds a small mocked registration/routing or client-call test. If maintainers prefer #41368, mark that broader PR as the canonical replacement before closing this PR.
