## Summary

Keep this issue open. Current main still does not propagate an assistant terminal turn with `stopReason: "aborted"` into lifecycle, wait, or subagent completion as a failed/killed outcome, and an open unmerged implementation PR already targets this issue with GitHub closing syntax.

## Best Possible Solution

Review and land, revise, or close the linked implementation PR. The fix should make `handleAgentEnd` propagate final assistant `stopReason: "aborted"` plus sanitized error metadata, teach `agent.wait` and the subagent registry to classify that terminal state as failed/killed instead of ok, persist `abortedLastRun: true` where appropriate, and add regression coverage for an empty-content aborted assistant turn producing a parent-visible failure/no-output announcement.
