## Summary

Keep this PR open. Current main has not implemented the proposed Control UI polling interval changes: nodes, logs, and debug still poll at 5s, 2s, and 3s. The PR is still potentially useful, but it is not merge-ready as written because the unresolved review concern about slower Logs polling is backed by the current cursor/maxBytes tail implementation.

## Best Possible Solution

Revise rather than close: keep the useful nodes/debug interval extraction or a safer nodes-only reduction, but leave Logs at 2s unless the PR also increases/configures the tail window or moves live logs to a lossless streaming/push model. Then add focused tests for the interval behavior and a regression test around `logs.tail` cursor/maxBytes truncation. Coordinate with #45698 and #69454 so the final fix addresses the real dashboard sluggishness instead of only changing polling constants.
