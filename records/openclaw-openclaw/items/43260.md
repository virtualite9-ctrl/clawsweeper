## Summary

Current main still does not implement per-skill model routing from SKILL.md or plugin skill metadata. The issue remains a live feature/API design request, with open implementation work in #58142 rather than shipped behavior.

## Best Possible Solution

Keep this issue open as the canonical feature request. The best maintainer path is to review, revise, land, or supersede #58142 with a documented SKILL.md/plugin skill metadata contract, parsed routing fields, allowlist/alias resolution, clear precedence against explicit session/cron/user model choices, and focused tests for parsing, prompt/read/slash invocation, fallbacks, and plugin-hook interaction.
