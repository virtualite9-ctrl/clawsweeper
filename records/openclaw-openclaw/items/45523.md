## Summary

Keep this PR open. Current main still does not implement the PR’s intended startup dangerous-flag detections for `gateway.auth.mode="none"` or trusted-proxy with empty/absent `allowUsers`, and `src/config/zod-schema.ts` still accepts whitespace-only `trustedProxy.userHeader` at schema level. Adjacent `openclaw security audit` checks already warn about the same risky states, so the PR is not replacing missing runtime understanding; it is still a focused implementation candidate for startup warning parity and schema validation hardening.

## Best Possible Solution

Keep the PR open and review it as a focused gateway hardening candidate. The best implementation path is to add the two dangerous-config collector checks, keep the `allowUsers` check covering both `undefined` and `[]`, decide whether schema-level blank `userHeader` rejection should supplement the existing runtime guard, and add regression coverage in `src/security/dangerous-config-flags.test.ts` plus a schema validation test if the Zod change remains.
