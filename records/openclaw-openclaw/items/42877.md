## Summary

Keep this issue open. Current main still has memory recall/read tools, prompt/runtime injection caps, append-only memory flush, and promotion/dreaming paths, but it does not implement the requested bounded model-facing memory CRUD tool with hard per-store character budgets, rejected over-limit writes, substring replace/remove, duplicate detection, capacity reporting, or credential-pattern screening.

## Best Possible Solution

Keep this issue open for a focused memory-core design. The best path is to add a structured model-facing memory write surface with config-backed per-store character budgets, atomic add/replace/remove semantics, duplicate detection, credential-pattern screening, and over-limit responses that include usage/current entries. If the limit must be hard rather than advisory, the implementation also needs a shared guard so generic file edits, memory flush, and promotion/dreaming cannot bypass the same budget.
