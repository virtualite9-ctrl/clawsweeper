## Summary

Keep this PR open. Current main has upstream NO_REPLY normalization that covers common reply paths, but it does not implement the Feishu API-edge guard this PR is trying to add, and the submitted patch is incomplete per the existing review comments.

## Best Possible Solution

Keep this PR open as useful but not landable. The best fix is a refreshed Feishu patch against current main that suppresses exact silent tokens at the Feishu API boundary for sendText, sendMedia captions/fallbacks, and reply-dispatcher delivery while preserving media delivery, closes/cleans streaming state correctly, uses the appropriate narrow SDK surface, and adds focused Feishu outbound/reply-dispatcher regression tests. Related open PR #56167 appears to cover the same area, so maintainers should consolidate the two attempts rather than close this as implemented.
