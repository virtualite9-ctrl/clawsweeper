## Summary

Keep this PR open. Current main has upstream NO_REPLY normalization for many normal delivery paths, but it does not implement this PR's stated Feishu API-send boundary guard, and the PR as submitted has unresolved review gaps around media payloads, return shape, tests, and streaming-card cleanup.

## Best Possible Solution

Keep this PR open only as a candidate patch, not an auto-close item. The best maintainer path is to revise it into a Feishu-owned boundary fix: use the existing plugin-sdk/reply-chunking silent-token exports, suppress silent text before every Feishu API text send path, preserve media delivery when the text is NO_REPLY, close or avoid stale streaming cards correctly, return the normal Feishu result shape, and add focused tests in extensions/feishu/src/outbound.test.ts and extensions/feishu/src/reply-dispatcher.test.ts.
