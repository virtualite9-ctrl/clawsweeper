## Summary

This issue should remain open. The provided discussion shows an open same-author implementation PR using closing syntax, which makes it an implementation candidate rather than a cleanup-close candidate. Current main also still shows the reported Anthropic path is not fully resolved: OAuth beta preservation is inside the Anthropic beta wrapper, but that wrapper is only installed when betas are resolved from extra params, while model/provider headers remain a supported request surface.

## Best Possible Solution

Keep this issue open and review PR #41461, or replace it with an equivalent maintainer patch. The fix likely belongs in the Anthropic wrapper or header merge path: preserve OAuth-required betas whenever an Anthropic OAuth token is used, including when model/provider headers contain anthropic-beta, while continuing to strip or fall back for unsupported context-1m OAuth combinations. Add regression coverage for model-level anthropic-beta plus sk-ant-oat auth with no params.context1m before closing.
