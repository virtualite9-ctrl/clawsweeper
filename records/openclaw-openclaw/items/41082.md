## Summary

Keep open. Current main has meaningful mitigations around compaction quality, recent-turn preservation, and post-compaction AGENTS.md refresh, and the discussion’s `npx clawaid` suggestion may help unhealthy gateway cases. But I did not find source or regression coverage proving the specific reported behavior: after an old-topic compaction summary, a fresh unrelated post-compaction user question is made dominant or the stale summary is demoted.

## Best Possible Solution

Keep this issue open and add a focused regression around the embedded run or prompt/context assembly path: create a transcript with an old-topic compaction summary, append a fresh unrelated user message after the compaction boundary, and assert the assembled prompt makes that latest user message the active request. If the prompt is ambiguous, fix it near prompt assembly or post-compaction guidance by explicitly prioritizing post-boundary user turns and treating the old summary as historical background.
