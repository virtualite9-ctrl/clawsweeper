## Summary

Keep open. Current main still does not warn when `openclaw config set agents.defaults.model.primary ...` changes the default while `agents.list[]` entries have explicit model overrides. The underlying precedence is real and documented, and this PR remains a plausible narrow CLI/DX fix, but it needs to be adapted to current `runConfigSet` and batch-mode plumbing before it can land.

## Best Possible Solution

Keep this PR open and rebase/adapt it onto current main. Add a small helper near `runConfigSet` that, after a successful non-dry-run write, detects operations whose parsed `setPath` is `agents.defaults.model` or `agents.defaults.model.primary`, including bracket notation and batch mode. It should scan the resulting config for explicit `agents.list[].model` values in both string and object forms and print a focused warning. Add targeted tests for object-style overrides, string-style overrides, bracket notation, no overrides, unrelated paths, dry-run behavior, and batch mode. No schema or runtime model-resolution change is needed.
