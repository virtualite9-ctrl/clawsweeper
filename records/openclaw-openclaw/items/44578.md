## Summary

Keep open. Current main still has the silent-success fallback this PR targets: when `runMessageAction` returns a core send result without a `toolResult`, `message-tool` serializes `result.payload` as a normal JSON tool result. The PR is a focused bug fix with a production Feishu observation in discussion and is not implemented or clearly superseded on main.

## Best Possible Solution

Keep the PR open, rebase it onto current main, preserve the missing-delivery guard in `src/agents/tools/message-tool.ts`, add the regression coverage in `src/agents/tools/message-tool.test.ts`, and run the targeted message-tool test plus the normal changed gate before merge.
