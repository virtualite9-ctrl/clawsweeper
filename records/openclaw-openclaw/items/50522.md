## Summary

Keep this PR open. Current main still unconditionally passes Discord `ThreadStarterBody` into inbound context, while the reply runtime can still use that field on follow-up turns when no thread history is present. The PR remains a focused implementation candidate for the linked Discord echo-contamination bug and is not implemented on main.

## Best Possible Solution

Keep the PR open as the canonical implementation candidate. Review and land the focused Discord adapter guard plus regression test after normal validation; once merged, the linked bug can be closed as implemented.
