## Summary

Current main still does not fully implement the PR’s intended `YYYY-MM-DD` placeholder behavior for `logging.file`. Runtime writes have partial rolling-path behavior, but resolved settings and several user-visible consumers still see the literal configured path, so this PR or an equivalent focused fix remains useful.

## Best Possible Solution

Keep this PR open and review or land an equivalent focused fix. The best path is a shared logging-file resolver used by resolved settings, file transport, `logs.tail`, startup output, and daemon status, with deterministic fake-timer tests and docs/config wording clarifying the supported `YYYY-MM-DD` placeholder semantics.
