## Summary

Keep open. Current main does not implement the PR’s intended WhatsApp forwarding metadata path: the WhatsApp inbound extractor does not read Baileys `contextInfo.isForwarded` or `contextInfo.forwardingScore`, `WebInboundMessage` has no forwarding fields, and `buildWhatsAppInboundContext()` never sets `ForwardedFrom` or `ForwardingScore`. The PR is a plausible additive bundled WhatsApp plugin feature rather than an obsolete or already-shipped change.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land a scoped WhatsApp plugin implementation that extracts `contextInfo.isForwarded` and `contextInfo.forwardingScore`, threads them through the inbound context contract, renders any score only inside untrusted forwarded-message metadata, and includes forwarded and non-forwarded WhatsApp regression tests. If maintainers prefer the newer generic `UntrustedStructuredContext` path over adding a shared `ForwardingScore` field, request that adjustment rather than closing the contribution as obsolete.
