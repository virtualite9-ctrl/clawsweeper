## Summary

Current main still has the reported no-`context1m` path: Anthropic Opus/Sonnet 4.6 only resolves to 1M when `params.context1m === true`, the colocated test still expects 200k without that flag, and status/session/UI paths consume the synchronous resolved value. The supplied GitHub context also shows open unmerged PR #66862 with closing syntax for this issue, so this issue should remain open while that implementation candidate is reviewed or replaced.

## Best Possible Solution

Keep this issue open while maintainers review and either land PR #66862 or replace it with an equivalent patch. The fix should update the context resolver, session/status persistence, Control UI behavior, Anthropic beta-header/auth handling, docs, and regression coverage for Anthropic Opus/Sonnet 4.6 without `context1m`, including an explicit opt-out if maintainers want to preserve legacy/OAuth fallback behavior.
