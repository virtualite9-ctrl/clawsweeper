## Summary

Keep this PR open. Current main still does not implement the PR’s central behavior: recurring `cron`/`every` jobs do not classify transient failures and retry on the short backoff window before the next natural schedule. The PR is a plausible implementation candidate for open bug #46341, but it needs maintainer review/cleanup because the branch also carries unrelated CI/guardrail changes and the config contract for recurring retries remains unsettled.

## Best Possible Solution

Keep this PR open, but require maintainer review or a focused refresh. The desired product path is to implement recurring transient retry in `src/cron/service/timer.ts`, add regression coverage for long-interval `cron` and `every` jobs, decide whether recurring retries use a separate config contract or only the fixed recurring backoff ladder, preserve restart catch-up behavior, and remove unrelated script/CI guardrail changes from the branch.
