## Summary

Keep open. Current main still routes Discord voice auto-join through READY only, still registers no RESUMED voice listener, and still calls `VoiceConnection.destroy()` directly without a stale-connection lookup or destroyed-state guard. The same reporter's PR #40676 remains an open implementation candidate with closing syntax, so this issue should not be closed before that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open as the canonical Discord voice reliability bug. Review and land PR #40676, or replace it with an equivalent Discord plugin fix that runs autoJoin on both READY and RESUMED, makes stale VoiceConnection cleanup idempotent around already-destroyed connections, scopes cleanup safely for same-guild multi-account setups, and adds regression coverage for RESUMED autoJoin plus destroyed stale-connection cleanup.
