## Summary

The global scrollbar CSS still uses 6px WebKit scrollbars, lacks the PR's global Firefox `scrollbar-width`/`scrollbar-color` support, and `ui/src/styles/config.css` still contains the `overflow: clip` declarations this PR removes. The PR is a focused CSS-only candidate, though maintainer review should resolve the scrollbar width/QA concerns raised in the comments before landing.

## Best Possible Solution

Keep this PR open or supersede it with an equivalent maintainer-owned UI patch. The patch should make scrollbar visibility intentionally theme-aware across the Control UI, add global Firefox scrollbar support if that remains desired, remove or justify the config `overflow: clip` declarations, and include at least a manual Chrome/Firefox light/dark QA note or a focused visual regression check.
