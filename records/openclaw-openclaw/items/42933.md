## Summary

Keep open. Current main still does not implement the PR's intended configurable session-level circuit breaker: there is no `circuitBreaker` config/schema/type surface, no `src/agents/circuit-breaker` module, no persisted `cb*` session state, and heartbeat/cron/reply execution still rely on adjacent mitigations such as provider-profile cooldowns, cron job backoff, duplicate heartbeat suppression, and context-overflow fallback messages. The PR remains a plausible implementation candidate, but the review discussion identifies correctness and security-sensitive follow-up work before merge.

## Best Possible Solution

Keep this PR open and request a focused revision before merge. The best path is a core-owned session-level breaker with typed `agents.defaults.circuitBreaker` and per-agent schema, persisted per-session breaker state, structured model-failure metadata from reply/heartbeat/cron execution, correct isolated-heartbeat session ownership, clear reset semantics for heartbeat dedupe state, validated/sanitized alert routing, and targeted tests across heartbeat, cron, and interactive reply paths. If maintainers want to reduce scope, split the generic config/state and model-failure signal from optional alert/reset actions.
