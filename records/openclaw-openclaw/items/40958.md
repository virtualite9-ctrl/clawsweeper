## Summary

Keep this PR open. Current main still does not implement the central update-path behavior: detached `openclaw update` restart health verification uses `waitForGatewayHealthyRestart` without enabling the Windows unknown-listener stale-PID fallback. The PR contains useful focused work, but it is not merge-ready because the branch also bundles unrelated workflow, config, channel-contract, secrets-baseline, Tlon dependency, and lockfile changes with supply-chain-sensitive dependency resolution churn.

## Best Possible Solution

Keep the PR open, but require a focused refresh before merge: rebase onto current main, keep only the update restart-health implementation plus targeted regression coverage, preserve the normal health-window delay before Windows unknown-listener fallback, and split or drop the unrelated workflow, config, Matrix/channel-contract, Tlon dependency, lockfile, and secrets-baseline changes. Any dependency or CI changes should be reviewed in a separate PR with install proof.
