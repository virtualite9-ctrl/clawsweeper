## Summary

Current main at a25366038503adcc31a5fa64b3d2f295bd5dbfd2 still has the reported recovery gap: stuck-session diagnostics emit warnings/events but do not abort the active run or clear queues. The linked implementation PR #73243 is open and unmerged with closing syntax for this issue, so the issue should remain open until that PR is reviewed and merged or replaced.

## Best Possible Solution

Keep this issue open. Review, harden, and either land #73243 or replace it with an equivalent core fix: add a separate conservative abort threshold, wire the diagnostic heartbeat to a generic recovery hook that aborts reply/embedded runs and clears session queues, emit a structured `session.aborted` event/log, and cover it with focused diagnostic heartbeat tests plus one reply/session integration guard. Treat #72846 as a related trigger mitigation, not the canonical recovery contract.
