## Summary

Keep this PR open. Current main still has the exact OpenResponses streaming mismatch the PR targets: the tool-call SSE path builds an `incomplete` response resource but emits a `response.completed` terminal event. The schema, tests, and public docs also still reflect `response.completed` only, so the proposed gateway/schema/test change is not obsolete.

## Best Possible Solution

Keep this PR open for maintainer review or reimplementation. The best path is to land the intended fix in the gateway owner surface: emit `response.incomplete` for streamed tool-call incomplete responses, add the corresponding schema union member, update docs to list the event, and adjust regression coverage to assert the new terminal event plus `[DONE]`. Before landing, run the targeted OpenResponses gateway test and the appropriate changed gate.
