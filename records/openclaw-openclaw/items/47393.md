## Summary

Keep open. Current main does not implement the PR’s intended Discord cross-account route-owner behavior, and it still drops bot identity entirely when fetchUser("@me") fails. The PR is a plausible implementation candidate, but it needs refresh against the current plugin SDK boundary and should address the review comments about DM owner checks and stale/non-running account ownership before maintainer merge consideration.

## Best Possible Solution

Keep this PR open as a real Discord bug-fix candidate. The best path is to refresh it onto current main using the plugin SDK imports, implement Discord route-owner resolution inside extensions/discord, preserve botUserId with an application-id fallback in the startup identity helper, pass botUserName only if the current handler needs name-based mention bypass, restrict owner candidates to active/configured accounts, skip or redesign owner blocking for DMs, and add focused regression tests for cross-account channel ownership, explicit alternate-bot mentions, and fetchUser("@me") failure.
