## Summary

Keep this PR open. Current main still strips mixed-message `/model <provider/model> <message>` but clears the model directive before the reply pipeline can apply it, so the PR’s core behavior is not implemented on main. The PR is still a relevant implementation candidate, but it needs updates for the unresolved review findings around session overrides and auth profile handling before it should land.

## Best Possible Solution

Keep the PR open as a valid implementation candidate, but require an update that preserves mixed-message `/model` as one-shot state, applies provider/model/auth profile only to the current run, avoids writing or clearing persistent session overrides, validates any one-shot auth profile against provider/cooldown rules, recomputes thinking/reasoning defaults without reapplying stored model overrides, and adds regression tests for a prior persisted model override plus follow-up turn reversion.
