## Summary

Keep open. Current main still does not provide a one-way dispatch/handoff mode for `sessions_send`: the tool schema has only target/message/timeout controls, fire-and-forget still reports pending announce delivery, and the A2A helper still performs reply-back turns plus an announce step unless a narrow ACP parent-child skip applies. The related open reports cover adjacent announce suppression or duplicate reverse-call safety, but they do not supersede this distinct per-call dispatch semantic.

## Best Possible Solution

Implement an explicit per-call delivery intent, either by extending `sessions_send` with a conservative field such as `mode: "conversation" | "dispatch"` or by adding a separate `sessions_deliver`/`sessions_enqueue` primitive. The dispatch path should persist/enqueue the message into the target session and return a minimal delivery ack without calling `runSessionsSendA2AFlow`, without requester-side reply reinjection, and without announce unless explicitly requested. Add focused tests in `src/agents/openclaw-tools.sessions.test.ts` and `src/agents/tools/sessions-send-tool.a2a.test.ts`, update `docs/concepts/session-tool.md`, and keep #39476 as the separate duplicate/reverse-call guard track.
