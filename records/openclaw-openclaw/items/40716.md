## Summary

Keep this PR open. Current main still only partially covers the reported delivery-mirror consumer-path leakage: replay/model context filtering exists, and the write path skips some redundant mirrors, but `chat.history` and `sessions.get` still pass raw/projected transcript messages without a shared delivery-mirror or transcript-only OpenClaw artifact filter. A maintainer also explicitly kept this PR as the canonical Track B candidate under #69208.

## Best Possible Solution

Keep this PR open, or replace it with an equivalent narrow rebased patch under #69208 Track B. The best implementation is a shared transcript-only OpenClaw assistant artifact predicate covering at least `delivery-mirror` and `gateway-injected`, applied before pagination and byte budgeting in intended visible/API consumer paths such as `chat.history`, `sessions.get`, and any session-history HTTP/SSE surface the maintainers decide should be user-facing. Preserve persisted JSONL as the audit trail and add focused regression tests for the visible/API paths.
