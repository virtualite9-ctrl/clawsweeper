## Summary

Keep this PR open. Current main still collapses Gemini CLI OAuth credential-extraction failures into one generic setup error, while the updated PR now targets the active bundled Google plugin and adds the missing diagnostic coverage. It is not obsolete on main, and the remaining question is maintainer/product coordination with the open security discussion about whether automatic Gemini CLI credential extraction should continue at all.

## Best Possible Solution

Keep this PR open for maintainer review. If automatic Gemini CLI credential extraction remains supported, land a narrow diagnostics fix in `extensions/google/oauth.credentials.ts` with regression tests around `resolveOAuthClientConfig()` for missing binary, missing credential source, parse failure, and extraction exceptions. If #54289 removes or gates the extraction behavior, close this PR as superseded by that security/product decision.
