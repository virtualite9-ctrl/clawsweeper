## Summary

Keep this PR open. Current main still collapses Gemini CLI OAuth credential discovery failures into the same generic setup error, while this PR targets the active bundled Google plugin with a narrow diagnostics-only change and focused test coverage. The only likely superseding path is the separate open security/product issue #54289 if maintainers decide to remove or gate automatic extraction instead.

## Best Possible Solution

Keep this PR open for maintainer review. If automatic Gemini CLI credential extraction remains supported, land a narrow diagnostics fix in extensions/google/oauth.credentials.ts with regression tests around resolveOAuthClientConfig() for missing binary, missing credential source, parse failure, and extraction exceptions. If #54289 removes or gates extraction, close this PR as superseded by that security/product decision.
