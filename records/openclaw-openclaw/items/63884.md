## Summary

Keep this issue open. Current main still records unmentioned Feishu group messages into pending group history when `requireMention` is enabled, then injects that history into the next handled group turn. The available Feishu config/docs expose mention gating, history limits, and group/session scopes, but not a per-bot or mention-only context recording policy. The related open PR #40782 is useful adjacent mention-identity work, but it does not implement context isolation.

## Best Possible Solution

Keep this issue open. Implement a backwards-compatible Feishu context policy, probably global plus per-group, that defaults to current behavior but can skip pending-history recording for unmentioned group messages and filter messages addressed to other bots once mention identity is reliable. The likely change set is `extensions/feishu/src/bot.ts`, `extensions/feishu/src/config-schema.ts`, generated config metadata if required, public Feishu/group docs, and regression tests for unmentioned pending-history replay plus multi-bot mention handling. Review #40782 separately as adjacent mention-identity work.
