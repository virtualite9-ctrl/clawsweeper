## Summary

Keep this issue open. Current main still buffers unmentioned Feishu group messages when `requireMention` is enabled, then replays that pending history into a later mentioned turn. I found adjacent generic context-visibility work and Feishu quote/thread filtering, but not the requested per-bot or mention-only context recording policy.

## Best Possible Solution

Keep this issue open. Implement a backwards-compatible Feishu context policy, likely global plus per-group, that defaults to current behavior but can skip pending-history recording for unmentioned group messages and filter messages addressed to other bots once mention identity is reliable. The likely change set is `extensions/feishu/src/bot.ts`, `extensions/feishu/src/config-schema.ts`, generated channel config metadata if required, public Feishu/group docs, and regression tests covering unmentioned pending-history replay plus multi-bot mention handling. Review #40782 separately as adjacent mention-identity work.
