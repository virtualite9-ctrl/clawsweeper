## Summary

Keep this PR open. Current main still rejects Slack `user:` targets in the auto-thread helper, so the DM attachment/threading bug described by the PR is not implemented on main. The PR discussion’s later two-field approach is directionally right, but the branch also carries unrelated dependency/CI changes that should be removed before maintainer review.

## Best Possible Solution

Keep this PR open, but require a clean rebase focused on Slack threading: preserve native `D...` `currentChannelId`, add a separate typed DM user target such as `currentDmUserId`, make `resolveSlackAutoThreadId` match `user:` targets against that field, and add focused tests for DM attachment/message-tool sends. Remove the unrelated `@pierre/diffs`, lockfile, and platform-test changes or split them into a separate maintainer-approved PR.
