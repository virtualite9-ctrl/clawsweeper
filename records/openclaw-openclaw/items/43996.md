## Summary

Keep open. Current main still hard-codes Docker `--security-opt no-new-privileges` for sandbox containers, does not expose a `sandbox.docker.noNewPrivileges` or equivalent opt-out in config/types/schema/docs/security audit, and PR #70229 remains open and unmerged as the linked implementation candidate.

## Best Possible Solution

Keep this issue open while PR #70229 remains open. Review or replace that PR as the implementation candidate: add a default-on Docker sandbox `noNewPrivileges` override, thread it through types/schema/config resolution/create args/docs/security audit, and add regression coverage for both the secure default and the explicit disabled path. After a vetted fix lands on main and ships, close this issue as implemented.

## Risks / Open Questions

- A fix that disables Docker `no-new-privileges` weakens sandbox hardening; it should remain default-on and any opt-out should be explicit, documented, and surfaced in security audit/status output.
- A silent automatic fallback would reduce isolation without clear operator consent, so the safer product path is a break-glass config plus diagnostics.
