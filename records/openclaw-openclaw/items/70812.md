## Summary

Keep open. Current main fixes the original doctor false-positive by unconditionally excluding `*.trajectory.jsonl`, but it does not implement this PR's later context-aware behavior for valid session IDs ending in `.trajectory`. The PR also still carries unrelated node host exec approval/workdir changes in a code execution path, so it should be narrowed or split before maintainer cleanup.

## Best Possible Solution

Keep this PR open, but ask the contributor to narrow it to the session-artifact classifier behavior or split the node host exec workdir/approval changes into a separate PR. The best session fix is a context-aware classifier where doctor, cost usage, discovery, session-file listing, and disk-budget callers pass the directory filename set, with regression tests for both live trajectory sidecars and valid `.trajectory` session IDs.
