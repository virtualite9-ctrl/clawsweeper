## Summary

Keep open. Current main has useful partial hardening around secure temp roots, diff artifact TTL metadata, trajectory export placement, and macOS service TMPDIR durability, but it does not implement the central request: a general temp artifact provenance/classification registry plus cleanup/doctor awareness and runtime-critical temp path guardrails.

## Best Possible Solution

Keep this open and scope a core design around OpenClaw-owned temp artifacts: extend the temp-path/tool runtime APIs with explicit artifact classification and provenance, persist a small state-dir registry for OpenClaw-created temp artifacts, teach cleanup/doctor/security audit to distinguish scratch/reproducible/durable/missing-label cases, and add targeted guardrails for managed service commands, wrappers, symlinks, and runtime-critical paths that resolve into /tmp or /private/tmp. Expose plugin hooks only after the core contract is clear.
