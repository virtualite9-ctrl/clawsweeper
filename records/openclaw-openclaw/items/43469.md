## Summary

Keep open. Current main still scans only JS/TS-family files, so SKILL.md markdown is not covered by the built-in skill/plugin code-safety scanner. This PR is a focused security-hardening candidate and supersedes the stale-closed #10705, but it should not merge as-is because the proposed critical markdown download-and-exec rule still has practical false negatives for wrapped interpreters and line continuations.

## Best Possible Solution

Keep this PR open as the active markdown skill-scanning security-hardening candidate. Revise markdown-download-exec coverage for sudo/env/wrapper prefixes, absolute interpreter paths, versioned interpreters, and line-continuation pipelines; add regression tests for those cases; then run the targeted scanner tests and the appropriate changed gate before merge.
