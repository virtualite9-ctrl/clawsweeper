## Summary

Keep this PR open. Current main still rejects valid node exec requests by comparing raw `boundNode` and `requestedNode` strings before node selector canonicalization, while the config/docs contract allows node ids or names and the shared resolver supports display names, IPs, and long id prefixes. The PR remains a focused implementation candidate for the linked node exec bug rather than obsolete work.

## Best Possible Solution

Keep this PR open for maintainer review and landing, or port the same fix to current main: list nodes first, resolve both configured `boundNode` and per-call `requestedNode` through `resolveNodeIdFromList`, compare canonical ids, reject different nodes, preserve clear not-found errors, and retain fail-closed behavior for stale configured bindings.
