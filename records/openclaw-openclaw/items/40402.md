## Summary

Keep open. Current main still does not implement `--profile-id` for `openclaw models auth login`; only `paste-token` accepts that flag. The OpenAI Codex identity path now improves automatic profile naming, but it still does not provide the operator-chosen OAuth alias requested here. PR #49315 remains the relevant open implementation candidate to review or supersede.

## Best Possible Solution

Keep this issue open as the tracker for explicit OAuth profile aliases. Review and land #49315 or equivalent work that adds `--profile-id <id>` to `models auth login`, carries it through login options and provider auth persistence, validates that the requested id belongs to the selected provider, handles collisions and ambiguous multi-profile results safely, and updates CLI docs plus parser/persistence tests.
