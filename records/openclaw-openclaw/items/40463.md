## Summary

Keep this PR open. Current main has absorbed several Microsoft Teams media fixes, but it still does not implement all of this PR's intended behavior, especially the channel `/chats/{threadId}/messages` fallback and some hardening around hosted-content downloads and Traffic Manager allowlists. The PR also cannot be merged as-is because its tenant-token path crosses the declared token-provider contract.

## Best Possible Solution

Keep this PR open for maintainer cleanup. Rebase it onto current main, retain the shipped hosted-content and DM media fixes, add or explicitly reject the channel `/chats/{threadId}/messages` fallback with tests, move the bounded `$value` read to the current `graph.ts` implementation, decide whether to narrow the broad media Traffic Manager allowlist, and replace the tenant-token overload with a typed provider capability or remove it. Coordinate the unresolved DM behavior with #65329 before landing or closing.
