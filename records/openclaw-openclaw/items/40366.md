## Summary

Keep this PR open. Current main still lacks both central behaviors: command delivery does not synthesize a fallback when raw agent payloads project to zero deliverable outbound payloads, and Slack preview/progress paths that end without any delivered reply still clear state and return silently. The related open #45290 is broader delivery recovery work, but it is not a canonical replacement for this PR's normalized-empty command fallback or Slack preview-only no-final fallback.

## Best Possible Solution

Keep this PR open for maintainer review or a scoped rebase. The best path is to port a narrow normalized-empty fallback after outbound projection in `src/agents/command/delivery.ts`, plus a narrow Slack preview-only no-final fallback in `extensions/slack/src/monitor/message-handler/dispatch.ts`. Preserve intentional silence, tool/block/final success paths, same-thread fallback delivery, draft cleanup, and JSON output behavior, and split or drop unrelated UI formatting churn before merge review.
