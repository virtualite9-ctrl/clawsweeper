## Summary

Keep open. Current main still logs the inbound QQ Bot message body at info level in the normal gateway message handler. The raw WebSocket dispatch JSON has been demoted to debug, but the central privacy/noise request is not fully implemented.

## Best Possible Solution

Keep this open and fix it in the bundled QQ Bot plugin. Replace the normal `log.info` inbound-processing message with metadata-only fields such as account id, message id, sender/peer id, type, group id, and content length, or demote it to debug. Keep full payloads and raw command text behind debug or an explicit verbose path, then add focused QQ Bot gateway/command tests asserting info logs do not include inbound user-visible message content.
