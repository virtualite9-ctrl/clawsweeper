## Summary

Keep open. Current main still does not ship the Bitwarden/Vaultwarden secret integration proposed by this PR, and the PR discussion points to a rebase/port after the secrets baseline rather than an implemented, superseded, or safely ClawHub-only close.

## Best Possible Solution

Keep this PR open as the focused Bitwarden/Vaultwarden implementation candidate. The best maintainer path is to settle the secret-provider plugin contract mentioned in the thread, or an equivalent manifest-first seam, then port this into a secrets-bitwarden plugin with manifest-declared ownership, runtime dependency handling, docs, tests, and a dedicated security review. If maintainers choose the generic exec-wrapper route instead, rebase this PR and harden the wrapper/docs/tests rather than closing it as implemented.
