## Summary

Keep open. Current main still does not parse direct inbound Feishu `message_type: "interactive"` events into agent-readable card content. The implemented parser covers fetched/read message items and quoted/thread context, and current docs still list interactive cards only for sending, not receiving. The provided related-item context also shows open PR #56795 as an implementation candidate for the remaining inbound path, so this issue should not be closed until that work or an equivalent focused fix lands.

## Best Possible Solution

Keep this issue open and review, land, or replace #56795 with a focused Feishu plugin change that parses direct inbound `message_type: "interactive"` events, preserves or fetches raw card JSON when needed, exposes readable card content to the agent context, and adds targeted tests for the direct inbound event path alongside the fetched/read-message fallback parser.
