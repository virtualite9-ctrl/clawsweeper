## Summary

Keep this issue open. Current main still has abort, queue cleanup, and embedded-run cancellation primitives, but it does not ship the requested unified per-session run-generation invalidation, broad stale-output fencing, pre-tool currentness gate, or default new-message takeover. The same-author draft PR #70363 is open, unmerged, and explicitly partial, so it should be reviewed or split rather than used as a reason to close this issue.

## Best Possible Solution

Keep this issue open as the canonical core interruptibility tracker. Review or split #70363, then land focused core changes for a session/run generation primitive, stale-output fences across reply, typing, followup, tool-progress, and event emission paths, a pre-tool abort/currentness gate in the Pi tool path, new-message takeover that invalidates and cancels the active run, and regression tests proving stale runs cannot emit visible effects or launch further tools.
