## Summary

Keep this PR open. Current main still has the active-tab exception that keeps a collapsed sidebar group visible when the active tab is inside that group, so the PR's intended fix is not implemented on main. The PR targets the right root cause, but it is stale against the current `nav-section`/`showItems` render path and should be updated with a focused browser regression test before merge.

## Best Possible Solution

Keep this PR open as the implementation path. Update it against current main by changing `showItems` so a collapsed group hides its children even when it contains the active tab, while preserving icon-only sidebar behavior when `navCollapsed` is true. Add a focused Control UI browser regression test in `ui/src/ui/navigation.browser.test.ts` before merge.
