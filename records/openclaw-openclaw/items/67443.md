## Summary

Keep open. Current main has not absorbed this PR's central fixes: `session_status` still hard-codes the group activation fallback to `mention`, and status context-token resolution is still duplicated. The PR also has unresolved P1 review findings, so it should be revised rather than closed as implemented.

## Best Possible Solution

Keep the PR open and request changes: derive `session_status` default group activation from `resolveChannelGroupRequireMention`, add the `requireMention: false` regression test, share or align context-token resolution while preserving `cfg.agents.defaults.contextTokens` and `DEFAULT_CONTEXT_TOKENS`, preserve valid default-agent aliases, and split or explicitly approve the `openclaw/<provider>/<model>` body-model contract change.
