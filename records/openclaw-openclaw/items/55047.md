## Summary

Keep this issue open. Current main still has a credible Control UI blank-screen path matching the report and the prior #54552 discussion: malformed persisted cron jobs can still reach `cron.list`, and Control UI render paths still dereference `job.payload` without guards.

## Best Possible Solution

Keep this issue open. Reproduce against current main with a persisted stale cron row missing a valid payload, then add defensive normalization/filtering before `cron.list` exposes jobs and null-safe guards in `ui/src/ui/app-render.ts`, `ui/src/ui/views/cron.ts`, and `ui/src/ui/controllers/cron.ts`. Add a focused jsdom/controller regression test proving malformed cron rows do not blank unrelated Control UI sections; doctor repair can remain cleanup but should not be required before the UI can render.
