## Summary

Keep this PR open. Current main still does not implement the central retained run-trace timeline store or `agent.timeline` Gateway RPC. The existing trajectory bundle support is related but not equivalent, and the open #68204 schema issue is a better design target but not a merged replacement for this implementation candidate. This PR also needs maintainer review before any merge because it mixes the trace feature with unrelated cron, sandbox, planner, docs, and generated-client changes plus unresolved security-sensitive review findings.

## Best Possible Solution

Keep this PR open for explicit maintainer handling, but require it to be split and rebased. Drop or separate already-shipped block-reply/fence fixes and unrelated autonomy/sandbox/cron/planner work, then rework the remaining trace feature against #68204 with a canonical cross-flow schema, safe Gateway/API exposure, ownership checks, redaction, retention/span caps, generated client updates, docs, and focused tests.
