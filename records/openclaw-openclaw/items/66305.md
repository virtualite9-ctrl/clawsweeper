## Summary

Keep this PR open. Current main still does not implement the PR's intended empty-catalog negative cache: successful empty catalog loads still clear the shared `modelCatalogPromise` immediately, and shared gateway/agent paths still call `loadModelCatalog()` in ways that can hit that behavior.

## Best Possible Solution

Keep this PR open and review/rebase it, or port the same fix to current main: briefly cache successful empty catalog loads, retry after a bounded window, keep import/filesystem failures retryable, clear negative-cache state on `useCache: false`, reset, and non-empty loads, preserve current `readOnly` non-mutating semantics and configured-catalog merging, guard stale concurrent loads, and keep focused regression tests in `src/agents/model-catalog.test.ts`.
