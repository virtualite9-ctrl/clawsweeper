## Summary

Keep this PR open. Current main still does not treat `logs` as a non-mutating command for the doctor/config preflight path, while the PR is a narrow CLI fix with matching tests and a positive bot review. The reported `openclaw logs --follow` preflight-noise problem remains plausibly unsolved on main.

## Best Possible Solution

Keep the PR open and either merge/adapt its one-line `logs` exemption plus tests, or implement the same policy through the newer command-startup policy if maintainers prefer that central surface. The key product outcome is that `openclaw logs --follow` starts tailing without running doctor/config preflight first, with regression coverage for both full argv and command-path helper behavior.
