## Summary

Keep open. Current main still runs the PDF fallback through the bundled document-extract plugin, and that plugin’s active pdf.js call still omits `standardFontDataUrl`. There is also an open same-author implementation PR, #70936, with closing syntax for this issue, so the issue should remain open until that PR is reviewed and merged or closed.

## Best Possible Solution

Review and either land or close #70936. If implementing directly on main, fix `extensions/document-extract/document-extractor.ts` by resolving `pdfjs-dist/standard_fonts` from the installed package root, passing `standardFontDataUrl` to `getDocument`, and adding a focused regression test that asserts the forwarded path exists and is passed into the pdf.js shim.
