## Summary

Keep this PR open. Current main has the previously merged session-row/UI configured-default work, but it still lacks the PR's remaining backend parity changes: `sessions.list` does not load/pass the gateway model catalog, and `chat.history` still falls back through the generic config/model resolver without applying the owning agent's `thinkingDefault`. The author recently rebased and narrowed the PR to exactly those remaining gaps after #72324 landed.

## Best Possible Solution

Keep this PR open for review and landing after maintainer approval. The best path is the narrowed backend patch: pass the gateway model catalog into `sessions.list` row default resolution, align `chat.history` with the same agent-aware session thinking-default resolver, and keep the regression tests for catalog-backed defaults plus non-default agent `thinkingDefault` history parity.
