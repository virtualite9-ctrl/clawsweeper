## Summary

Current main still has the Discord `sessions_send` A2A announce account-routing gap. The resolver returns a parsed Discord target before session hydration, Discord does not opt into `preferSessionLookupForAnnounceTarget`, and the final gateway `send` receives the parsed target's missing `accountId` instead of the target session's stored account identity. Related #51626 is closed in favor of this issue, so this remains the canonical tracker.

## Best Possible Solution

Keep this issue open and fix the Discord-owned `sessions_send` A2A announce routing path. The narrow maintainable path is to opt Discord into `preferSessionLookupForAnnounceTarget` or merge missing `accountId` and thread fields from stored session metadata after parsing. Add a regression test where a Discord target session has `deliveryContext.accountId` or `lastAccountId` and the final gateway `send` receives that account id.
