## Summary

Current main still has the Discord A2A announce account-routing gap. The resolver returns a parsed Discord target before session hydration, so the stored session `deliveryContext.accountId` / `lastAccountId` is not merged into the final gateway `send`. Related #51626 was closed in favor of this issue, so this remains the canonical tracker.

## Best Possible Solution

Keep this issue open and fix the Discord `sessions_send` A2A announce path. The narrow maintainable fix is to opt Discord into `preferSessionLookupForAnnounceTarget` or merge missing account/thread fields from stored session metadata after parsing. Add a regression test where a parsed Discord target session has `deliveryContext.accountId` or `lastAccountId` and the final gateway `send` receives that account id.
