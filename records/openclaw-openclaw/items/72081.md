## Summary

Keep this issue open. Current main still prints the bare dashboard URL to logs while sending the tokenized URL only through clipboard/browser delivery, so a headless/WSL path where both delivery channels fail still leaves the user without the tokenized URL or a manual-token recovery hint. Two open PRs already use closing syntax for this issue, so they should be reviewed or closed rather than closing the issue first.

## Best Possible Solution

Review and either land or close the open dashboard-token fallback PRs, then close this issue after main contains a safe headless/manual-auth recovery path. The likely code path is `src/commands/dashboard.ts`, with focused coverage in `src/commands/dashboard.links.test.ts` ensuring the user gets actionable recovery guidance without logging the token value or token fragment.
