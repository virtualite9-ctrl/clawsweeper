## Summary

Current main still does not implement the PR's central docs request: `docs/tools/index.md` lists `read` / `write` / `edit` only in the built-in tools table and `group:fs`, with no individual tool documentation entries. The PR remains useful, but it should be revised before merge because current main's file-tool contract and read behavior differ from the proposed text.

## Best Possible Solution

Keep this PR open but revise it before merge: add focused `read`, `write`, and `edit` documentation to the tools inventory using the current canonical parameters, adaptive read behavior, and structured image-output semantics. Keep the change docs-only and scoped to the file-tool documentation gap.
