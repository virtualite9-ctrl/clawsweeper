## Summary

Keep this PR open. Current main does not implement Slack `ackReactionTiming`: Slack config accepts `ackReaction` and `ackReactionScope`, the Slack monitor context/provider do not carry a timing value, and the inbound prepare path still starts the ack/status reaction at receipt time when the existing scope gate passes. The PR is also paired with the same author's still-open Discord and Telegram companion PRs, so it should be reviewed or updated rather than closed as obsolete.

## Best Possible Solution

Keep the PR open and ask for an update against current main. The best path is to rebase the Slack implementation, preserve the default receipt-time behavior, add a channel-scoped or explicitly shared `ackReactionTiming` config contract, trigger the deferred reaction from `onAgentRunStart`, preserve the existing scope gate and cleanup/status-reaction semantics, and refresh Slack-focused tests/docs. Coordinate any shared type/schema changes with #45449 and #46258.
