## Summary

Keep this PR open. The model bug it targets is still plausible on current main because the current inline-provider helper still reads only raw `model.api ?? entry.api`, while the Anthropic default is applied through config materialization/provider policy. However, the PR cannot be accepted as-is: main has since moved the helper into `model.inline-provider.ts`, and the PR also includes an unrelated Telegram retry commit with a reviewed duplicate-send risk.

## Best Possible Solution

Keep the PR open as a useful bug-fix candidate, but require a rebase/split before landing: move the model-only fix into current `src/agents/pi-embedded-runner/model.inline-provider.ts`, reuse the current provider-policy/defaulting path rather than the removed `resolveDefaultProviderApi` shape, add the Anthropic/no-explicit-api regression test in the current inline-provider or resolver tests, and drop or separately rework the Telegram retry commit with the existing non-idempotent-send safety constraints.
