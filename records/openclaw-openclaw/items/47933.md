## Summary

Current main now implements the core WebChat/Control UI file-transfer request. The earlier April 25 maintainer comment was correct at the time because upload support was still image-scoped, but current main now accepts non-video files, serializes documents/archives as `chat.send` file attachments, stages/offloads them as agent-readable media paths, and renders managed/generated media as authenticated attachment links in chat history.

## Best Possible Solution

Close as implemented on main. Keep the current managed-media and sandbox-staged non-video attachment flow; any future request for an explicit file-manager UI, different workspace placement policy, or per-file-type attachment policy should be tracked as a narrower follow-up.
