## Summary

Keep this PR open. Current main still has the early-abort responsePrefix interpolation gap, and the PR is a focused implementation candidate for the user-visible bug in #45314. The related #45315 is another open candidate for the same bug, but it is not merged and current main has not shipped either approach.

## Best Possible Solution

Keep this PR open for maintainer review or a rebased equivalent. The best fix is to make early abort reply normalization receive accurate prefix context before the fast return: thread sessionKey or a preloaded session metadata seed through createChannelReplyPipeline at the current extensions/* and gateway/plugin-SDK call sites, preserve onModelSelected as the later authoritative override, and add focused regression coverage for fast abort plus normal post-selection behavior. Review this alongside #45315 and close #45314 only after one implementation lands.
