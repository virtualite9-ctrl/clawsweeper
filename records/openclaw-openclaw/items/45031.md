## Summary

Keep open. Current main has a narrower built-in scan for gateway-backed skill dependency installs, but not the direct ClawHub skill install/update/download path this issue asks to protect, and the docs explicitly call that path separate. The issue discussion usefully identifies AgentShield and ClawGuard as external ClawHub-available scanners or workarounds, but those do not implement built-in default scan-on-install, scan-on-update, or publish-time enforcement in OpenClaw core. Related #47080 was closed as a duplicate in this feature area, while broader #9256 and #10827 ClawSweeper reports remain open security work, so this should not be auto-closed.

## Best Possible Solution

Keep this open for maintainer security/product work. The concrete path is to extend the existing install scan and `before_install` policy surface to direct ClawHub skill install/update flows, and separately decide whether ClawHub publish-time scanning belongs in the registry service, this repo’s CLI integration, or both. AgentShield and ClawGuard should be treated as useful candidate integrations or policy plugins, while broader provenance and permission-manifest work should stay coordinated with #9256 and #10827.
