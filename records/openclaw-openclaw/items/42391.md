## Summary

Keep open. Current main still exposes typed plugin hooks for tool calls and agent completion, but the public hook contracts and emitters do not include the requested structured progress metrics (`stepIndex`, `stepTotal`, `percent`, or `etaSec`). The prior ClawSweeper comment remains accurate against current main SHA d48c3e12a5d896cbdad3e7fcf5bba2af644a9ca5.

## Best Possible Solution

Keep this issue open and design an additive, backward-compatible progress observation contract. A conservative maintainer path is to add a shared optional progress metrics type either to the existing `before_tool_call`, `after_tool_call`, and `agent_end` hooks or to a dedicated progress hook, emit it only when OpenClaw has a defensible step model, document absent/unknown semantics in the public plugin docs, and add focused hook contract/runtime tests.

## Risks / Open Questions

- A numeric progress contract can mislead users unless it defines unknown totals, dynamic plans, retries, subagents, failures, and cancellations explicitly.
- This is a public plugin SDK change, so an implementation should include shared types, docs, and regression tests for monotonicity, final success behavior, non-100 failure/cancel behavior, and backwards compatibility.
