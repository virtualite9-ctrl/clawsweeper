## Summary

Keep this PR open. Current main still initializes the response-prefix context with only the agent identity, while `{model}`, `{provider}`, and `{thinkingLevel}` are filled only after `onModelSelected` runs. The fast-abort path returns before the normal reply/model-selection path, so the linked abort-template bug is not implemented on main. The same-author linked issue #45314 is also still open, making this PR the active implementation candidate rather than obsolete work.

## Best Possible Solution

Keep this PR open and review/rebase it as the active fix for #45314. The best implementation should ensure abort/stop replies have a populated response-prefix context before dispatcher normalization, add focused regression coverage for templated abort prefixes, and either address or deliberately document follow-up work for session overrides, channel overrides, and catalog-backed thinking defaults.
