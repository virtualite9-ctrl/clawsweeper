## Summary

Keep this PR open. Current main still lacks the narrow Telegram direct-chat override that would prevent native /new and /reset replies from targeting the transient slash-command message when reply threading is configured, so the user-visible bug remains plausible. The proposed change is useful but needs to be ported from removed legacy src/telegram paths to the active extensions/telegram plugin and expanded to cover /reset in tests.

## Best Possible Solution

Keep this PR open and port the fix to extensions/telegram/src/bot-native-commands.ts. Add a narrow delivery override that sets replyToMode to "off" only for private-chat native /new and /reset replies, while preserving configured threading for groups, topics, plugin commands, and other native commands. Add regression coverage for both /new and /reset when configured replyToMode is "first".
