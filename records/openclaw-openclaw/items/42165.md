## Summary

Current main still appears to have the reported tool-profile least-privilege regression. `tools.fs` configuration implicitly re-adds `read`, `write`, and `edit` without checking whether the effective profile is `minimal` or `messaging`, and the linked implementation candidate is still open rather than merged.

## Best Possible Solution

Keep this issue open. The best path is to fix `src/agents/pi-tools.policy.ts` so `tools.fs` and `tools.exec` configure already-granted tools but do not implicitly re-grant filesystem/runtime tools under restrictive profiles unless explicitly requested via `alsoAllow`, while preserving deny precedence. Review or split #43061 as the active implementation candidate, and add regression coverage for both `minimal` and `messaging` with configured `tools.fs`/`tools.exec`.
