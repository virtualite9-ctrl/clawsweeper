## Summary

Keep this issue open. Current main implements Gemini Embedding 2, outputDimensionality, and structured image/audio embedding for memory-file indexing, but the requested conversation-history attachment workflow is still missing: session transcript indexing flattens text and ignores persisted MediaPath/MediaPaths plus MediaType/MediaTypes. Video and PDF multimodal memory are also not implemented by the current modality set.

## Best Possible Solution

Keep this issue open and narrow it to the missing session-memory attachment path. Extend session transcript indexing to collect eligible persisted MediaPath/MediaPaths plus MediaType/MediaTypes, resolve those paths through existing media safety boundaries, and build structured Gemini embedding inputs for attachments while preserving the shipped gemini-embedding-2-preview, outputDimensionality, and memory-file image/audio indexing behavior. Add regression coverage for chat or Telegram attachment recall and update docs to distinguish memory-file media indexing from conversation-history attachment indexing and any future video/PDF support.

## Risks / Open Questions

- Closing this now would lose the central requested workflow: searching screenshots/photos or other attachments received in chat history unless users manually place those files into memory-file paths.
- A future fix must preserve media safety boundaries: managed media path resolution, sandbox/private path checks, MIME validation, file-size limits, provider capability checks, and fallback behavior.
- Video and PDF support remains a separate gap because the current multimodal memory modality set and docs cover image/audio only.
