## Summary

This issue should stay open. Current main still returns early from preflight compaction when persisted totals are fresh and transcript-byte compaction is inactive, before evaluating the token threshold. The issue also has an open same-author implementation candidate, #64384, with closing syntax, so the right maintainer path is to review/land or explicitly supersede that PR rather than close the issue now.

## Best Possible Solution

Keep this issue open. Review and land #64384, or explicitly supersede it with #66716 or a maintainer patch. The fix should let fresh persisted totals at or above the preflight token threshold reach `shouldRunPreflightCompaction`, preserve the below-threshold fast path, and add focused regression coverage for `totalTokensFresh: true`, cached tokens above threshold, and no transcript-byte trigger.
