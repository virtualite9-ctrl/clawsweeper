## Summary

Current main does not implement this PR’s intended behavior. `openclaw logs` still exposes `--local-time`, has no `--utc` flag, and defaults to UTC unless `--local-time` is passed or `TZ` is set to a valid timezone. Tests and docs still encode the old opt-in local-time behavior, while the linked bug #46748 remains a real implementation target. The provided PR diff is narrow and its second commit addresses the bot review’s dead-code concern, so this is still useful maintainer review work rather than conservative cleanup.

## Best Possible Solution

Keep this PR open as an implementation candidate for #46748. The best maintainer path is to review the focused CLI behavior change, add/update regression coverage for default-local timestamps and the UTC opt-out, update `docs.openclaw.ai/cli/logs` source docs, and decide the compatibility story for the existing `--local-time` flag before landing.
