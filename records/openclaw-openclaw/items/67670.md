## Summary

Keep this issue open. Current main still does not implement a Cloudflare-challenge-specific 403 classification, browser-like TLS profile, or documented workaround for the openai-codex mainland-China/proxy path. Related fixes on main improve standalone/5xx HTML handling, but status-prefixed 403 HTML still maps to auth_html_403. Open PR #67717 is an active implementation candidate with closing syntax for this issue, so the issue should remain open until that PR is reviewed, merged, or closed.

## Best Possible Solution

Keep this issue open and review PR #67717 as the narrow implementation candidate. If accepted, land the challenge-specific classifier, user-facing guidance, raw-HTML suppression, and regression tests, then close this issue as implemented. Treat browser-like TLS emulation, request.tlsProfile, or browser-backed routing as a separate maintainer-scoped provider/security design decision; if that is not pursued, document the reverse-proxy/browser-act-style workaround and the allowPrivateNetwork requirement for trusted local proxies.
