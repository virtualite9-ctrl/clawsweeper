## Summary

Keep open. Current main does not fully implement the PR’s intended change: setup already uses the snapshot compatibility helper, and one shared scan path does too, but the actual non-JSON `openclaw status --all` command still routes through `statusAllCommand` and then calls the diagnostics-backed compatibility helper. The PR also has a large, unrelated and security-sensitive diff, including package metadata/version/export changes and CI/test-scope changes, so it should be revised rather than closed as implemented or merged as-is.

## Best Possible Solution

Keep this PR open only as a candidate direction, but require a rebase and a narrow patch. The best fix belongs in `src/commands/status-all/report-data.ts`: switch the non-JSON `status --all` compatibility collection to the snapshot helper only if tests prove parity for legacy `before_agent_start` and hook-only notices. Drop the unrelated package metadata, SDK export, CI/test-scope, changelog, and setup rewrites from the PR.
