## Summary

Keep this issue open. Current main has partial fixes for several subareas, especially config validation/reload handling and cron command syntax, but the field report is not fully implemented or clearly superseded. A concrete user-facing mismatch remains: the public cron guide still says `openclaw cron list` lists all jobs, while the CLI only includes disabled jobs when `--all` is passed.

## Best Possible Solution

Keep the issue open for maintainer triage. Fix the cron docs first so `openclaw cron list` is documented as enabled-only by default and `openclaw cron list --all` is shown for disabled jobs, then split remaining current behavior into focused follow-ups for cron upsert/resource warnings, isolated web-search reproduction, and any Discord/onboarding/update-process gaps not already covered by narrower closed or open items.
