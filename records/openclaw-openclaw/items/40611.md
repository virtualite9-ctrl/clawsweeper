## Summary

Keep this issue open. Current main still has the reported periodic heartbeat behavior: a `requests-in-flight` skip is retried through the shared 1 second heartbeat wake retry, and there is still no heartbeat-specific busy-session retry/backoff setting in the typed config, schema, or docs. The related fixing PR #40629 is closed unmerged, while #13744 and #14051 remain broader follow-up designs rather than a shipped fix for this regression.

## Best Possible Solution

Keep this issue open and repair the periodic heartbeat scheduling path. The most direct maintainer path is to refresh the useful parts of #40629: add a heartbeat-specific `requests-in-flight` retry delay separate from the shared wake error retry, choose the default delay and configurability under `agents.defaults.heartbeat` and per-agent heartbeat config, then update the strict schema, typed config, generated metadata/help, public heartbeat docs, and scheduler/wake regression tests. Leave #13744 for broader session-lock scope/configuration and #14051 for activity-based heartbeat semantics.
