## Summary

Current main already implements the PR's intended daemon/systemd behavior, and the fix shipped in v2026.4.26. The shipped implementation is stronger than the PR diff: it keeps direct `systemctl --user` first for non-root `sudo -u` target shells with stale `SUDO_USER`, but preserves machine-scope-first behavior for sudo-to-root so the process does not accidentally target root's user manager.

## Best Possible Solution

Close this PR as already implemented. Keep the shipped v2026.4.26/current-main systemd user-scope resolver and tests rather than applying this older diff, because current main covers the same `sudo -u` bug while also avoiding the sudo-to-root wrong-scope regression called out in PR review.
