## Summary

Keep this PR open. Current main still writes session-memory snapshots only to `<workspace>/memory/`, and the docs/tests still describe or assert that fixed location. The PR addresses a real workflow problem confirmed in discussion, but its head needs revision before merge because the proposed pre-write `fs.mkdir` can create directories through an in-workspace symlink outside the workspace, and it edits localized docs that repo policy says not to edit here.

## Best Possible Solution

Keep this PR open and request a focused revision: preserve default `memory` output, add optional workspace-relative `hooks.internal.entries.session-memory.outputDir`, reject empty/absolute/traversal values with fallback to `memory`, write through `writeFileWithinRoot` rooted at `workspaceDir` with `relativePath` set to `outputDir/filename`, remove the separate pre-write mkdir or use an alias-safe helper, add regression coverage for custom output, invalid fallback, and symlink alias behavior, and drop localized docs edits. Coordinate broader duplicate-indexing guidance with #54524, but do not treat that open docs/design issue as a replacement for this implementation.
