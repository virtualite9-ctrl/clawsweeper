## Summary

Current main does not implement this PR’s intended Telegram ACP DM-topic routing fix. The bound route can resolve to the final ACP session key, but both inbound message handling and native command handling still append a DM-topic thread suffix afterward, which matches the PR’s reported `ACP metadata is missing ...:thread:<chatId>:<topicId>` failure mode. The related open PR #48967 addresses ACP metadata copying for thread-derived sessions, but it is not a clear superseding fix for preserving the already-bound session key.

## Best Possible Solution

Keep this PR open or replace it with an equivalent maintainer patch that adds an explicit bound-route suffix guard in `extensions/telegram/src/conversation-route.ts`, applies it in inbound message and native command session-key derivation, and keeps unbound DM-topic isolation covered by tests.
