## Summary

Keep this PR open. Current main still routes agent gateway-tool calls through the WebSocket-backed `callGateway` path, and the cron tool still uses that helper for the affected cron actions. The PR remains a plausible implementation candidate for #40237, but it needs a current-main rebase/security adaptation before merge because the gateway-tool contract now includes explicit scopes, URL/token/timeout handling, `expectFinal`, and WebSocket serialization behavior.

## Best Possible Solution

Keep this PR open and ask for a current-main rebase or adapted patch. The maintainer-ready fix should add an embedded in-process dispatch path only when it is semantically equivalent to the current gateway-tool call, preserving explicit `gatewayUrl`, `gatewayToken`, `timeoutMs`, `expectFinal`, explicit `extra.scopes`, least-privilege default scopes, and WebSocket-equivalent JSON normalization. Add focused regression tests for embedded cron status/list/add/update/remove/run/runs plus URL, token, timeout, expectFinal, explicit-scope, default-scope, and optional/undefined parameter cases before merge.
