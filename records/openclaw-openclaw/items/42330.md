## Summary

Keep this PR open for maintainer judgment, but do not merge it as-is. Current main already has stronger shared inbound dedupe than the PR, while the PR's reply-to default change, add-on lifecycle handling, and modern Google Chat space-type handling are not fully implemented on main and the PR head still mixes security-sensitive auth rewrites with behavior changes.

## Best Possible Solution

Do not merge this PR as-is. Keep the shared inbound dedupe already on main, review or close focused follow-ups #58993 and #65058 on their own merits, and handle any Google Chat replyToMode default change as a small maintainer-backed PR with targeted tests, docs/config metadata updates, preserved local-media behavior, and a separate security review for any auth or webhook changes.
