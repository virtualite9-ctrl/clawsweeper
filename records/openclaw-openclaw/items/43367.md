## Summary

Keep open. Current main has partial mitigations for OAuth refresh races, per-session queueing, task visibility, and subagent controls, but it still does not fully satisfy the reported end-to-end multi-agent orchestration problem. The concurrent `agents add` path still builds from a command-start config snapshot and rejects stale writes rather than serializing the full read-modify-write sequence. A same-author open PR, #44997, remains an implementation candidate with closing syntax for this issue and explicitly covers only the config-write symptom.

## Best Possible Solution

Keep this issue open. First review and either land or close #44997 for serialized config mutation. Then split the remaining broad reliability work into narrower maintainer-trackable issues or PRs for per-agent/session run serialization semantics, detached CLI task cancellation and visibility, subagent completion recovery/notification, and any remaining failover behavior after local lock failures.
