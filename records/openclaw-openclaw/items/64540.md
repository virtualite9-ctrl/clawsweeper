## Summary

Keep this PR open. Current main still treats an empty heartbeat object as enabled/defaulted, and the linked bug remains open, but the provided PR only changes the heartbeat summary helper and would leave scheduler, system-prompt, bootstrap, and ACP spawn heartbeat policy paths inconsistent.

## Best Possible Solution

Keep this PR open for maintainer review or contributor rework. If OpenClaw wants `heartbeat: {}` to disable heartbeats, add one shared empty-heartbeat predicate and apply it consistently across summary, runner selection/scheduling, system-prompt guidance, HEARTBEAT.md bootstrap filtering, and ACP spawn policy, then add regression tests and update the public heartbeat docs. If maintainers want `every: "0m"` to remain the only disable contract, close or redirect the branch and document that decision on #64293.
