## Summary

Keep this PR open. Current main still treats an empty heartbeat object as default cadence, and this PR is a plausible implementation candidate for the linked bug, but the provided patch only updates the heartbeat summary helper. A correct fix needs to align all heartbeat policy paths before this can be safely closed or landed.

## Best Possible Solution

Keep this PR open for maintainer review or contributor rework. If OpenClaw wants `heartbeat: {}` to mean disabled, introduce one shared empty-heartbeat predicate and apply it consistently across heartbeat summary, runner selection/scheduling, system-prompt guidance, bootstrap `HEARTBEAT.md` filtering, and ACP spawn policy, then add regression tests and update the public heartbeat docs. If maintainers want `every: "0m"` to remain the only disable contract, revise or close the branch and document that outcome on the linked bug.
