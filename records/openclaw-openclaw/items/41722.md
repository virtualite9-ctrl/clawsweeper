## Summary

Keep open. Current main still exposes the reported macOS Skills settings `skills.install` path, still prefers Homebrew when `brew` is present, and still formats Homebrew failures as generic `Install failed (...)` errors without CLT/macOS 26 detection, fallback, or remediation. The provided discussion already noted this on 2026-04-27; the current checkout at d48c3e12 does not change the skills installer path enough to close it.

## Best Possible Solution

Keep this issue open and fix the gateway-backed skills installer path: detect Homebrew/Xcode CLT unsupported-macOS errors, return actionable remediation, avoid preferring brew when the host brew toolchain is known unusable, fall back to uv/node/go/download when skill metadata provides a viable option, and add focused tests for the classifier plus installer-selection behavior.
