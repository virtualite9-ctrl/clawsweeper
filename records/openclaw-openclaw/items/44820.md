## Summary

This issue should stay open. Current main has partial session maintenance, but it does not fully solve the reported automatic cleanup gap for `.tmp` and transcript `.bak.*` artifacts, and the provided GitHub timeline shows an open implementation PR with closing syntax for this issue. Per policy, an issue with an open closing PR remains open until that PR is reviewed and merged or closed.

## Best Possible Solution

Keep this issue open and review PR #47635, or land an equivalent maintainer patch, after verifying it against current main. The best implementation is a scoped gateway/session maintenance sweep that runs at startup and on a timer, reuses existing archive timestamp parsing, respects retention config for `.deleted.*` and `.reset.*`, age-gates orphaned `writeTextAtomic` `.tmp` files, trims transcript `.bak.*` files per base name, and adds focused regression tests for active-file preservation and false-positive archive names.
