## Summary

Current main still classifies dotted gateway read actions like `config.schema.lookup` and `config.get` as mutating because the classifier checks the full action string against a flat read-only verb set. The PR remains a focused implementation candidate for the linked user-visible warning bug; the related payload-level PR #42962 does not supersede this classifier fix.

## Best Possible Solution

Keep this PR open for maintainer review or landing. The best implementation path is a gateway-specific classifier fix, ideally anchored to the gateway read-scope method list, with regression coverage for `config.schema.lookup`, `config.get`, `agents.files.list`, mutating writes such as `config.patch` or `config.apply`, missing actions, and unsupported dotted cron/canvas actions. After that lands, #42856 can close as implemented; #42962 should only proceed if maintainers still want an additional payload-specific warning policy.

## Risks / Open Questions

- A broad gateway leaf-verb fallback can classify an unknown dotted gateway action ending in a read-only verb as non-mutating before execution rejects it; a tighter implementation would anchor classification to known read-scope gateway methods or otherwise keep the fallback gateway-only and fail-closed for unsupported cron/canvas actions.
- Current main has no regression coverage in `src/agents/tool-mutation.test.ts` for dotted gateway read actions, so the bug can persist or regress until this PR or an equivalent fix lands.
