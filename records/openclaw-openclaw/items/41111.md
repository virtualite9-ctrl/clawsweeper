## Summary

Current main still does not ship the requested first-party `openclaw backup restore <archive>` workflow. The backup CLI, docs, command descriptor, and tests all describe or cover only `create` and `verify`; the linked keepmyclaw plugin is a useful workaround, and open PR #59280 is a plausible implementation path, but neither makes the core CLI request implemented yet.

## Best Possible Solution

Keep this issue open until OpenClaw ships a safe first-party `openclaw backup restore <archive>` command or maintainers explicitly consolidate it into a canonical native backup/restore tracker. The likely path is to review and either land or close PR #59280, reusing archive verification, adding dry-run and force behavior, staging restore payloads, handling conflicts and rollback, updating `docs/cli/backup.md`, and adding focused restore command and restore-path tests. keepmyclaw can remain a documented community workaround, not a close reason for the core CLI gap.
