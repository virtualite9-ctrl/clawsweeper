## Summary

Keep this PR open. Current main still preserves cron run session keys for exec-triggered heartbeat wakes and still accepts forced cron session keys in heartbeat resolution, so the intended bug fix is not implemented on main. The PR is focused and relevant, but the review comments about preserving agent targeting should be resolved before landing.

## Best Possible Solution

Keep this PR open and revise it before landing: prevent heartbeat forced-session resolution from targeting cron run sessions, but preserve the originating agent target when scopedHeartbeatWakeOptions drops the cron session key. Add regression tests for scoped wake options, forced cron session fallback, non-default agents, and multi-agent configurations so cron-originated exec wakes target the originating agent's main heartbeat session rather than the isolated cron run or every agent.
