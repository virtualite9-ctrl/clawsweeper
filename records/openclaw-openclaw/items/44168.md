## Summary

Keep open. Current main has a generic LLM idle timeout, but it does not implement this PR's central OpenAI WebSocket first-response watchdog, auto-mode HTTP fallback after a silent first-response timeout, forced-websocket timeout error, or the PR's regression coverage. The PR looks focused and the discussion is positive, so this needs maintainer review/rebase rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review. Rebase it onto current main, wire the first-response timeout through `resolveEmbeddedAgentStreamFn`, preserve the existing generic idle timeout, and keep/add focused tests for auto fallback, forced-websocket timeout, late WS completion isolation, duplicate `start` prevention, and resolved abort-signal forwarding.
