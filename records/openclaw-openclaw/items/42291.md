## Summary

Keep open. Current main still writes `openclaw models auth login` results to the locally resolved agent auth store, with no remote node-host guard or gateway credential-sync mutation path. PR #42381 remains an open implementation candidate that references this issue with GitHub closing syntax, so this issue should stay open until that PR is reviewed/merged or explicitly rejected.

## Best Possible Solution

Keep this issue open and review PR #42381 as the active implementation candidate. The conservative fix belongs in `src/commands/models/auth.ts` with regression coverage in `src/commands/models/auth.test.ts` to block or clearly warn on OAuth login from remote node hosts; if direct node-host OAuth should work, first design a secure gateway-side auth-profile mutation API and scopes.
