## Summary

Keep this PR open. Current main still rejects unsigned Feishu/Lark webhook url_verification requests before parsing the payload or checking the verification token, so the PR's central bug fix is not implemented on main.

## Best Possible Solution

Review and land this PR or an equivalent Feishu-owned patch. The implementation should accept unsigned url_verification challenges only when the plaintext or decrypted payload token matches the configured verificationToken with timing-safe comparison, keep strict signature validation for all non-challenge webhook events, and add the missing wrong-token rejection test.
