## Summary

Keep open. Current main still writes a formatted top-level `time` field while serializing the original log object unchanged, so `_meta.date` can remain a UTC `Z` timestamp in raw JSONL logs. The related PR #48717 is closed and unmerged, so there is no shipped fix to close against.

## Best Possible Solution

Keep this issue open. Patch `src/logging/logger.ts` so the file-log JSON payload copies `_meta` and serializes `_meta.date` with the same explicit-offset timestamp used for top-level `time`, preferably deriving both from tslog's original timestamp when present. Add focused regression coverage asserting `_meta.date === time` for JSONL file logs. Treat switching file logs from host-local time to `agents.defaults.userTimezone` as a separate product/docs decision unless maintainers explicitly want that broader behavior.
