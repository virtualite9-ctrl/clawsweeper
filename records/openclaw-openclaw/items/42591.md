## Summary

Keep open. Current main still has a 2770-line monolithic `scripts/install.sh`; the requested `scripts/install/main.sh`, `scripts/install/lib/`, and `scripts/install/vendor/` module layout is absent. The prior issue discussion also already identified the same gap, and current docs, tests, and installer smoke workflow still treat the single shell script as the public installer surface.

## Best Possible Solution

Keep this issue open for a conservative installer maintainability refactor. Preserve `scripts/install.sh` as the public entrypoint, extract narrowly scoped sourced modules for UI/logging, downloads/checksums, gum bootstrap, platform/package-manager detection, build tools, npm install and repair, git install, and main argument flow, then update installer docs and direct shell/smoke coverage around the new layout.
