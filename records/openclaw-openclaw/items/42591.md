## Summary

Current main does not implement the requested installer modularization. `scripts/install.sh` is still a 2770-line public installer with downloader, UI/gum bootstrap, platform detection, build tools, npm/git install logic, and main flow in one file; the proposed `scripts/install/` module tree is absent. Existing docs, tests, and smoke surfaces still assume the single-script entrypoint, so the maintainability refactor remains actionable.

## Best Possible Solution

Keep this issue open for a conservative installer maintainability refactor. Preserve `scripts/install.sh` and `https://openclaw.ai/install.sh` as stable public entrypoints, extract narrow sourced modules for UI/logging, download/checksum helpers, gum bootstrap, platform/package-manager detection, build tools, npm install/repair, git install, and main argument flow, then update installer docs and targeted shell/smoke coverage around the new layout.
