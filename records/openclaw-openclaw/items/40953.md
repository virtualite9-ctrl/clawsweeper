## Summary

Keep this PR open. Current main still has the gateway-facing timeout validation gap the PR targets: gateway call, gateway-rpc, devices, and channels status still register string `--timeout` options and/or pass raw `Number(...)` values into `callGateway`. The PR needs rebase/adaptation to the current gateway-rpc runtime split, but its central behavior is not implemented on main. Security review found no unexplained supply-chain, dependency, workflow, secrets, install-script, or release-surface changes in the provided PR file list; the diff is limited to CLI parsing/call sites and focused tests.

## Best Possible Solution

Keep this PR open for maintainer review or contributor rebase. The best path is to adapt the strict timeout parser to the current gateway-rpc runtime split, apply it consistently to the gateway call, gateway-rpc, devices, and channels status gateway-facing paths, preserve intended defaults, add focused regression tests for invalid input and valid passthrough, and coordinate with #54646/#59397 so timeout-validation work converges cleanly.
