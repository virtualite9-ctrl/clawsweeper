## Summary

Keep this issue open. Current checkout still builds the Control UI chat session dropdown from `sessions.list` rows plus the active session key; configured agents from `agents.list` are available but only used to label already-present agent session rows. The provided GitHub context also shows open PR #41955 still uses closing syntax for this issue, so the issue should remain open while that implementation candidate is reviewed, landed, closed, or superseded.

## Best Possible Solution

Keep this issue open and use #41955, or a refreshed successor, as the implementation path. The fix should merge configured `agents.list` entries into `resolveSessionOptionGroups` using canonical agent main session keys, dedupe existing `sessions.list` rows, preserve current global/unknown/cron filtering behavior, and add focused UI regression coverage for configured agents with no stored session.
