## Summary

Current main does not provide enough evidence to close this issue. The gateway still emits cumulative live assistant snapshots across tool boundaries, while the Control UI renders archived stream segments plus the current stream as separate items. That is the exact failure mode reported, and there is no targeted regression test, changelog entry, or related merged fix showing the live WebChat duplicate-card behavior is resolved.

## Best Possible Solution

Keep this open and fix the Control UI live stream segmentation. The likely maintainer path is to track a per-tool-boundary stream baseline or committed-prefix length so a post-tool cumulative gateway delta renders only the fresh assistant text card, then add focused UI tests for `text -> tool_call -> tool_result -> text -> tool_call` cycles and the mirrored tool-bubble variant. The gateway cumulative-delta contract can remain as-is if the UI owns the per-card delta split.
