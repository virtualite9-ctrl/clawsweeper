## Summary

Keep this PR open. Current main still stores system-event queues under the exact caller-supplied session-key casing, while targeted heartbeat resolution canonicalizes agent session keys to lowercase before peeking and consuming pending events. The PR remains a focused implementation candidate for the linked channel-session heartbeat miss; the main review question is whether to lowercase every system-event key or only recognized agent-scoped keys.

## Best Possible Solution

Review and either land this focused fix with the mixed-case lookup and cross-case dedupe regression tests, or adjust the implementation to canonicalize only recognized agent-scoped session keys before merge. Do not close this PR until current main can enqueue a mixed-case channel session event and peek/drain it through the lowercase canonical heartbeat session key.
