## Summary

Keep this issue open. Current main still exposes only env/file/exec SecretRef sources and provider configs. It documents 1Password as an exec-provider example, but does not ship native keyring/Keychain, native 1Password, or a plugin-owned SecretRef provider/source seam. The provided GitHub context also shows same-author PR #72548 is the active implementation candidate, so closing this issue before that PR is reviewed or merged would race the paired work.

## Best Possible Solution

Review PR #72548 as the active same-author implementation candidate. If it lands, close this issue as implemented for the env/keyring/plugin-seam portion and split native 1Password into a focused follow-up only if maintainers still want it bundled. If the PR is declined, narrow this issue to the accepted native keyring/Keychain, native 1Password, or plugin SecretRef provider API work, while leaving broader cloud-provider tracking to #13610 or dedicated provider issues.
