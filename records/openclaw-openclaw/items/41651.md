## Summary

Current main still performs the realpath containment check before skill-source merging and still logs a warning for symlinked skill paths that resolve outside their configured root. This issue also has an open implementation candidate with closing syntax in the provided GitHub context, so it should remain open until that PR is reviewed, merged, or explicitly rejected.

## Best Possible Solution

Keep this issue open. Review the linked implementation candidate or make a focused core change that preserves realpath containment while suppressing or downgrading the non-actionable warning when the same skill is already loaded from a trusted source, then add a regression test for a personal `.agents/skills` symlink to a bundled skill.
