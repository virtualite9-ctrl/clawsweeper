## Summary

Keep open. The PR targets a real Google prefix regression and current main still does not fully implement the intended fix: Google model normalization aliases bare IDs only, the Google manifest has no `stripPrefixes` policy, embedded resolution does not split a duplicated `google/...` modelId argument, and both OpenAI-compatible and native Gemini/PDF paths still have plausible ways to send a prefixed model ID. The PR should be reworked for the current plugin-owned Google seams and should drop the unrelated secret-scanning baseline churn.

## Best Possible Solution

Keep this PR open and request a current-main rework: implement Google-owned prefix cleanup through `extensions/google` model-id/manifest/runtime seams, add regression coverage for Google OpenAI-compatible chat and native Gemini/PDF URL construction, preserve slash-containing model IDs for non-Google providers, and remove the unrelated `.secrets.baseline` change.

## Risks / Open Questions

- The PR was written against older `src/agents/models-config.providers.ts` structure; current main has plugin-owned Google seams under `extensions/google`, so the patch needs rework before review/merge.
