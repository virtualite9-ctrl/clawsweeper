## Summary

Current main still does not implement this PR's intended end-to-end cron announce reasoning filter. Main creates reasoning payload items and the embedded result type now has an `isReasoning` field, but `buildEmbeddedRunPayloads` still drops the flag from returned payloads, and the shared cron outcome helper still selects summary/output/delivery/error-recovery text without checking reasoning. The PR remains a focused candidate, though it needs rebase/adaptation to the current `resolveCronPayloadOutcome` flow.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to rebase it onto current main, propagate `isReasoning` from `buildEmbeddedRunPayloads`, keep the embedded result payload surface aligned, filter reasoning in `src/cron/isolated-agent/helpers.ts` summary/output/deliverable selection and first-text fallback, exclude reasoning from post-error recovery checks, ensure both cron callers use the shared behavior, and add regression tests for mixed reasoning plus final text, reasoning-only, and error-then-reasoning-only announce cases.
