## Summary

Keep this PR open. Current main does have a Kimi/OpenAI-compatible malformed tool-call argument repair path, but it does not implement the PR's central fail-closed behavior for unrecoverable malformed final JSON. The current implementation only applies cached repaired arguments to `stream.result()`, does not track cleared argument indices for final snapshots, and explicitly preserves preexisting streamed tool arguments when malformed deltas fail reevaluation. The PR is still a plausible implementation candidate, but it needs a rebase onto the current split helper module and a maintainer decision because later shipped behavior intentionally moved in the preservation direction.

## Best Possible Solution

Keep this PR open as a focused candidate. The best path is for a maintainer to decide whether malformed final Kimi tool-call JSON should fail closed even when `partial-json` already exposed non-empty arguments; if yes, rebase the change into `src/agents/pi-embedded-runner/run/attempt.tool-call-argument-repair.ts` and add focused coverage that clears `event.toolCall`, partial/message snapshots, and `stream.result()` final messages without regressing the current valid/preamble repair cases and the #54491 preservation behavior.
