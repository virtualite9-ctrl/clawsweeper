## Summary

Keep this PR open. Current main has related Kimi/OpenAI-completions tool-call argument repair logic, but it still does not implement the PR’s fail-closed behavior for malformed final JSON with already-populated streamed arguments. The PR remains a plausible implementation candidate for the linked Kimi file-tool regression, though it needs to be rebased/ported to the current extracted helper module.

## Best Possible Solution

Keep this PR open and review/port the fail-closed behavior into `src/agents/pi-embedded-runner/run/attempt.tool-call-argument-repair.ts`: at `toolcall_end`, validate the accumulated raw argument buffer, and when it is non-empty but not a complete JSON object and no allowed repair exists, clear the streamed `toolCall`, partial message, end message, and final result snapshot. Add focused tests with non-empty partial-json arguments for trailing junk, later-invalidated repairs, and unterminated string payloads, and update or replace the current preservation test to match the chosen safety semantics.
