## Summary

Keep this PR open. Current main still calls the wizard final health check without a pinned setup token, while the linked open bug describes that exact QuickStart token-mismatch failure. The PR remains a focused implementation candidate, though its unrelated docs whitespace changes should be reviewed or split before merge if maintainers want a cleaner diff.

## Best Possible Solution

Review and land this PR or an equivalent narrow patch that builds the wizard health-check config from nextConfig and pins gateway.auth.token, or explicit healthCommand auth, to settings.gatewayToken only for token-mode setup with a resolved setup token. Keep the regression test that proves healthCommand receives the setup token when env/config tokens differ, and then close the linked open bug after the fix merges. Consider dropping or splitting unrelated docs whitespace changes from this PR before landing.
