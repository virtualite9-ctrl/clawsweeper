## Summary

Current main now narrows macOS LaunchAgent extra-gateway detection so unrelated LaunchAgents are not reported merely because they mention OpenClaw or gateway-related labels/options. The scanner requires explicit OpenClaw gateway service markers or an actual `gateway` subcommand execution path, preserves legacy `clawdbot` cleanup, and has regression coverage for companion LaunchAgents that previously matched the reported false-positive class.

## Best Possible Solution

Close as implemented on current main. Keep the narrowed launchd scanner and regression coverage; open a new focused issue only if a current build still unloads the managed gateway through a separate launchd lifecycle path after unrelated LaunchAgents are no longer reported.

## Evidence
