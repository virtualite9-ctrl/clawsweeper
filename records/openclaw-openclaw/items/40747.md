## Summary

Keep this PR open. Current main still requires `after_block_id` for `feishu_doc` insert, does not expose an insert `index`, and has no root-child insertion branch; the PR remains a focused Feishu plugin implementation candidate, but it needs schema validation tightened plus tests/docs before merge.

## Best Possible Solution

Keep this PR open, or land an equivalent maintainer patch in `extensions/feishu`: make `after_block_id` optional only for `action: "insert"`, add `index: Type.Optional(Type.Integer({ minimum: 0 }))`, default missing `after_block_id` to root insertion at `index ?? 0`, preserve the existing after-block branch, forward index through normal and batched Descendant insertion paths, and add targeted Feishu doc tool tests plus skill docs for the new mode.
