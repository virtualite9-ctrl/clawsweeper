## Summary

Keep this PR open. Current main still lacks the xAI/Grok web_search fixes the PR is trying to provide: the Responses parser still dereferences nullable response entries, Grok still uses the shared 30s timeout, and xAI timeout AbortErrors still flow through a tool adapter path that rethrows AbortError instead of returning a structured tool error. The PR needs a current-layout port and scope cleanup before it is mergeable.

## Best Possible Solution

Keep this PR open as useful contributor work, but ask for a narrow rebase/port. The maintainer path is to move only the xAI/Grok fixes into `extensions/xai/src/responses-tool-shared.ts`, `extensions/xai/src/web-search-shared.ts`, and `extensions/xai/src/web-search-provider.runtime.ts`; wrap internal timeout aborts using the shared `isAbortError()` behavior with `{ cause }`; add a Grok-specific 60s default while preserving explicit user overrides; add focused xAI regression tests; and drop or split the unrelated `src/commands/status*` changes.
