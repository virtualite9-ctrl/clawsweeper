## Summary

Keep this issue open. Current main has a useful generic Control UI descriptor foundation, but it does not implement the broader RFC’s requested contribution slots for plugin chat modes, approval renderers/cards, event classifiers, input guards, status surfaces, or host-rendered client behavior. The provided discussion also shows the issue is paired with the same-author open RFC PR #71731, so conservative cleanup should not close this thread independently.

## Best Possible Solution

Keep this issue open as the Control UI contribution-slot design thread. Maintainers should decide in or alongside #71731 whether registerControlUiDescriptor/plugins.uiDescriptors is the intended abstraction, then either narrow this RFC to the remaining client/slot work or implement explicit descriptor kinds and host-owned UI consumption for chat modes, approval rendering, event classification, input guards, status surfaces, plugin actions, disablement, and sanitization fixture coverage.
