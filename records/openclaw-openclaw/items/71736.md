## Summary

Keep this open. Current main has a useful generic foundation, `api.registerControlUiDescriptor(...)` plus `plugins.uiDescriptors`, but it is narrower than this RFC's requested Control UI contribution slots and there is still an open same-author RFC PR paired with this issue. The issue is recent, active, and explicitly discusses the remaining gaps after the merged host-hook work.

## Best Possible Solution

Keep this issue open as the Control UI contribution-slot design thread. Maintainers should decide whether the shipped `registerControlUiDescriptor` and `plugins.uiDescriptors` foundation is the intended abstraction, then either narrow this RFC around the remaining client/rendering work or implement the missing descriptor kinds for chat modes, approval cards/renderers, event classification, input guards, status surfaces, plugin actions, disablement, and sanitization fixture coverage.
