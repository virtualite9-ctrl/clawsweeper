## Summary

Keep this PR open. Current main still returns early for fresh persisted token totals before the token-threshold compaction check, so the cached-token auto-compaction fix is not implemented on main. The PR overlaps #64384 and #65600, but no merged/current implementation supersedes it.

## Best Possible Solution

Keep this PR open as an implementation candidate. Maintainers should consolidate it with #64384 and #65600, then land one rebased fix in `src/auto-reply/reply/agent-runner-memory.ts` that evaluates explicitly fresh persisted totals against the preflight threshold, falls back to transcript estimation for stale or unknown freshness, preserves the transcript-byte trigger, and adds regression coverage for high cached totals, below-threshold fresh totals, stale/legacy fallback, and heartbeat/CLI skips.
