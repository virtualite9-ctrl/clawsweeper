## Summary

Keep open. Current main does not implement this PR's central feature: spoken Discord voice phrases that persistently escalate or de-escalate the session model. Main only supports a static `channels.discord.voice.model` override and the normal Discord `/model` picker. The unrelated gateway HTTP-route registry fix bundled into this PR is already solved on main, so the useful remaining work would need a fresh, narrow rebase against the current Discord plugin path.

## Best Possible Solution

Keep this PR open only as a feature candidate. The best maintainer path is a fresh, narrow Discord plugin change on current main: drop the obsolete gateway registry edits, implement explicit low-false-positive spoken model commands in `extensions/discord/src/voice/manager.ts`, consume matched command-only phrases instead of sending them to the agent, ensure the session entry exists before first-turn persistence, honor configured/allowed model references rather than hardcoded policy-sensitive defaults where possible, and add focused voice manager tests plus Discord voice docs if maintainers want the feature.
