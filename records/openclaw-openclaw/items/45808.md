## Summary

Current main does not implement the PR's intended explicit per-Discord-account `agentId` binding or automatic route synthesis. The shipped config types/schema still lack `channels.discord.accounts.<id>.agentId`, routing still consumes only persisted `bindings`, and the user docs still instruct multi-bot Discord users to add manual `bindings` entries. This PR is therefore not obsolete or implemented on main; it should stay open as an implementation candidate, with the later review concerns resolved before any merge.

## Best Possible Solution

Keep this PR open. The best path is to review it as a real implementation candidate: add Discord account-level `agentId` to the accepted config contract, synthesize route bindings only in the routing layer, keep explicit `bindings` precedence, avoid writing synthesized bindings back to config, skip disabled or tokenless accounts where outbound binding would be unsafe, surface the effective binding in status APIs if that remains a goal, and add targeted routing/status tests before landing.
