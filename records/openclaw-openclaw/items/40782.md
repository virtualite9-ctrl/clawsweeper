## Best Possible Solution

Keep this PR open and ask for a refreshed implementation tied to #40768. The best path is to verify the Feishu/Lark stable bot identity contract first, then thread the chosen stable identifier through probe results, monitor identity state, parseFeishuMessageEvent/checkBotMentioned, normalizeMentions, mention-forward detection/extraction, debounce mention merging, and sequential-key parsing.
