## Summary

Keep open. Current main still applies the default weighted hybrid merge to multimodal vector hits, with vector-only candidates initialized at `textScore: 0`, and there is no persisted modality field available at merge time. The existing multimodal test proves indexing and literal label searches, not natural-language image/audio recall under default hybrid ranking. Related BM25/hybrid reports do not supersede this modality-specific bug.

## Best Possible Solution

Keep this open and fix it in `extensions/memory-core`: carry reliable modality metadata from multimodal indexing into search results, then make hybrid merging avoid BM25 dampening for image/audio vector candidates while preserving weighted BM25+vector scoring for text chunks. Add regression tests with mixed Markdown and image/audio candidates showing a natural-language visual/audio query surfaces multimodal results under the default 0.7/0.3 weights.
