## Summary

Keep this PR open. Current main has newer macOS LaunchAgent restart/update hardening, but it still does not implement the PR's central bounded EIO-only `launchctl bootstrap` retry; several bootstrap paths remain single-attempt. The PR is stale against current restart code and needs rebase/rescope, but it remains a focused launchd bug-fix candidate rather than a cleanup-close item.

## Best Possible Solution

Keep this PR open for maintainer review. Rebase it onto current main and rescope around a shared bounded EIO-only `launchctl bootstrap` retry used by `bootstrapLaunchAgentOrThrow`, `repairLaunchAgentBootstrap`, install activation, restart fallback recovery, and detached handoff bootstrap paths. Separately decide whether PID SIGKILL escalation is still needed under the current kickstart-first model; if retained, add stronger PID identity protection and focused tests for retry success/exhaustion, errno 125 and non-EIO non-retry behavior, doctor repair, install activation, restart fallback, and handoff scripts.
