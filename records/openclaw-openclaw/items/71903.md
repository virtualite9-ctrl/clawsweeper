## Summary

Keep open. Current main still reclaims a self-PID legacy lock when the process-local held-lock map has no entry, which is the behavior this PR is trying to change. The PR is a focused core session-write-lock fix with related lost-response reports, but it still needs maintainer review and stronger regression coverage before landing.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to refine the patch, add an explicit regression test proving a live self-PID legacy lock without starttime is not reclaimed, preserve dead-PID and recycled-PID reclamation coverage, and then land it if maintainers agree it matches #71872, #64362, and #70857.
