## Summary

Keep open. Current main still allows delivery-mirror transcript ownership to diverge from the session key in the exact multi-agent class this issue reports; existing tests even assert one mismatched-agent path as expected behavior, so this needs a contract decision and regression coverage before it can close.

## Best Possible Solution

Keep this issue open. Add a focused regression for Feishu or the generic outbound route with session.dmScope="per-account-channel-peer", writer account/agent binding, and a delivery-mirror append where the request/default agent differs from the writer route. Then either derive mirror transcript ownership from the outbound route/session key or reject/normalize mismatched agentId and sessionKey, updating the gateway precedence test to encode the chosen contract.
