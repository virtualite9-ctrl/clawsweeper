## Summary

Keep this PR open. Current main still rejects Feishu media keys containing embedded `..`, so the intended Feishu image/media fix is not implemented. The latest PR diff is focused and resolves the earlier slash-separator regression, but it still needs a small security refinement to reject a literal `..` key while allowing opaque keys like `img_v2_test..key`.

## Best Possible Solution

Refresh this PR with the narrow Feishu plugin fix: keep empty, length, control-character, `/`, and `\` rejection; allow embedded `..` in opaque Feishu keys; explicitly reject literal dot-segment values such as `.` and `..`; and keep regression tests for valid embedded-dot keys plus invalid separator and dot-token inputs.
