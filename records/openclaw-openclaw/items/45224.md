## Summary

Keep this issue open. Current main still runs browser Playwright/CDP in the Gateway process, generic unhandled rejections still exit for unclassified errors, and the browser plugin has not registered a Playwright/CDP-specific rejection handler for assertion-shaped CRSession/CRBrowser failures. The related #45791/#52051 cleanup chain no longer leaves a clear open canonical tracker for this exact CRSession._onMessage production crash.

## Best Possible Solution

Keep the issue open and fix it in the browser plugin, not by weakening core’s generic fatal unhandled-rejection policy. Add a browser-owned Playwright/CDP unhandled-rejection handler through `openclaw/plugin-sdk/runtime-env`, register/unregister it in browser runtime lifecycle, invalidate or close the affected Playwright CDP connection when matched, and add browser-plugin tests for CRSession._onMessage and CRBrowser._onAttachedToTarget assertion-shaped errors, including nested/wrapped reasons. Maintainers can separately decide whether stronger browser/CDP subprocess isolation is needed.
