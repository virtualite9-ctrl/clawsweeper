## Summary

Keep open. Current main has partial progress: bundled plugin runtime dependencies are lazy by default and repair is scoped to active/default-enabled plugins, and channel setup can defer some plugin loading until a channel is selected. The central request is still not implemented: there is no install-time selection of bundled plugins/channels/skills, no plugin `strictMode` that skips bundled/global discovery roots, no `openclaw setup --channels` style selector, and skills are still loaded into a per-session snapshot rather than registered lazily on first use.

## Best Possible Solution

Keep this open and narrow it into concrete maintainer-owned work: decide whether OpenClaw should move toward a slim core package plus ClawHub/on-demand bundled plugin installs; define a backwards-compatible config/schema/docs contract for any strict discovery mode; add setup/onboarding UX only after the install/runtime contract is clear; and treat the shipped runtime-deps scoping and #71818 fix as partial progress, not completion.
