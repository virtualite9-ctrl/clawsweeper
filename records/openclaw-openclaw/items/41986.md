## Summary

Keep this issue open. Current main still has the actionable Usage dashboard mismatch: the header totals and daily cost chart are sourced from different RPC summaries, and the `usage.cost` path still scans the default/single agent transcript directory while `sessions.usage` discovers configured agents. The open PR #70205 is the right implementation candidate to review, while #42063 was closed unmerged and #68066 tracks a different OpenRouter billing-accuracy problem.

## Best Possible Solution

Keep this issue open and use #70205 as the implementation candidate: make `usage.cost` use the same configured-agent aggregation scope as `sessions.usage` or return header totals and daily bars from one shared backend summary with one documented dollar unit. Add regression coverage for unscoped multi-agent usage.cost totals/daily rows, explicit agent scoping, cache keys that include the resolved scope, and UI rendering that proves header totals and daily bars agree for the selected date range.
