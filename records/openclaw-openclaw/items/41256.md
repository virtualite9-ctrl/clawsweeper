## Summary

Keep this issue open. Current main still matches the reported Discord media upload problem, and the issue is paired with open implementation PRs, including #41265 by the same author plus #41419 and #41714. Repository policy says an issue with open PRs that explicitly mark themselves as closing it should remain open until a PR is reviewed, landed, or closed.

## Best Possible Solution

Keep the issue open and review the linked implementation PRs. The likely fix belongs in the Discord send helpers and shared web media loader: pass detected `contentType` into Discord `Blob` construction, preserve WebP and animated media instead of running them through JPEG optimization, and add focused regression tests for Blob MIME type, static WebP preservation, and animated image preservation.
