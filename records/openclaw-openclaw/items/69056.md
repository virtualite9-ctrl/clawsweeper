## Summary

Keep this PR open. Current main still has the Windows-sensitive no-listener SIGUSR1 path, and the PR is not safely obsolete or implemented by main. It addresses a real restart gap, but the current diff still needs maintainer review because the Windows handoff semantics, restart-token consumption, tests, and overlap with #68853 are unresolved.

## Best Possible Solution

Keep this PR open for maintainer review, but do not merge it as-is. The best path is a focused gateway restart fix that preserves the listener-driven SIGUSR1 path, handles Windows no-listener fallback without deadlocking restart tokens, proves or explicitly handles live process replacement/termination, adds targeted regression tests, and either absorbs or clearly separates the overlapping #68853 restart-supervisor work.

## Risks / Open Questions

- The Windows no-listener SIGUSR1 bug is still present on current main, so this PR is not implemented-on-main cleanup.
- The PR's successful Windows fallback can still leave the emitted restart token unconsumed if the current process keeps running after the detached task handoff.
- The PR lacks targeted regression coverage for the win32 no-listener fallback, failed handoff rollback, successful-cycle handling, and mode reporting behavior.
- The PR includes adjacent launchd method/path changes that are not required for the Windows SIGUSR1 fix and should be reviewed separately or removed.
- There is active overlap with #68853, but that PR is also open and not merged; it is not enough to cleanup-close this PR as superseded.
