## Summary

Keep this PR open. Current main still has the Windows-sensitive no-listener `SIGUSR1` fallback, so the PR is not implemented on main. The PR addresses a real Windows gap, but its latest diff is still not safe cleanup-close material because the successful Windows fallback can leave the restart cycle unhandled if the current process continues running, it lacks targeted regression coverage, and it includes adjacent macOS launchd method/path changes unrelated to the Windows fix.

## Best Possible Solution

Keep the PR open for maintainer review, but do not merge it as-is. The best path is a focused gateway restart fix that preserves the listener-driven SIGUSR1 path, handles Windows no-listener fallback without deadlocking restart tokens, proves whether the live Windows process is actually restarted or explicitly marks the cycle handled, and adds targeted regression tests. If #68853 becomes the canonical merged fix, it should explicitly absorb this PR's Windows `emitGatewayRestart()` fallback surface before this PR is closed as superseded.

- The PR has no targeted regression coverage for the win32 no-listener fallback, failed `triggerOpenClawRestart()` rollback, successful-cycle handling, or mode reporting behavior.
- The diff includes adjacent macOS launchd return-method/path changes that are not required for the Windows `SIGUSR1` fix.
- There is active overlap with #68853, but that PR is not merged; maintainers should choose a canonical implementation rather than cleanup-closing this active PR prematurely.
