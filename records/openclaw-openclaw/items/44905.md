## Summary

Keep open. Current main has partial fixes for this bug family, but it does not fully resolve the Discord/tool-envelope/control-token leak described here. Commentary-phase output is now suppressed and exact NO_REPLY payloads are normalized, including media-carrying payloads, but Discord still lacks a final send-level guard and the shared delivery/sanitizer paths do not cover raw `to=functions.*`, `recipient_name`, `parameters`, or the later `REPLY_SKIP` / `ANNOUNCE_SKIP` variants.

## Best Possible Solution

Keep this issue open and implement a tested shared outbound/user-facing sanitizer or delivery-boundary guard for internal control tokens and model-emitted tool-routing envelopes. The fix should cover Discord through the normal outbound adapter and add regression tests for exact control-token suppression, media-carrying silent payloads, commentary-phase suppression, raw `to=functions.*` / JSON tool-argument text, and Discord's low-level send path if it remains reachable outside normalized delivery.
