## Summary

Keep open. Current main still preserves manifest-declared `skills` in the manifest registry and uses that metadata for actual skill discovery, but loaded runtime `PluginRecord`s still do not expose or copy `skills`, which is the specific PR intent. The PR looks like plausible core plugin-contract work, not ClawHub work, but it likely needs a rebase/retarget because current main keeps `PluginRecord` in `src/plugins/registry-types.ts` while the provided PR patch edits `src/plugins/registry.ts`.

## Best Possible Solution

Keep this PR open and ask for a rebase/narrow update against current main: add `skills: string[]` to `PluginRecord` in `src/plugins/registry-types.ts`, have the loader factory default it to `[]`, pass `manifestRecord.skills` at all manifest-backed record construction sites, initialize bundled capability/test-helper records with `[]`, and add the focused regression test that a native `openclaw.plugin.json` with `skills: ["skills"]` produces a loaded runtime record with the same value.
