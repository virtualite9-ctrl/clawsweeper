## Summary

Keep this issue open. Current main still injects channel metadata into agent-visible prompts for Telegram/Discord-style inbound channel messages, including direct chats, and I did not find a shared, Telegram, or Discord config option that disables or minimizes those metadata blocks for trusted senders. The related #41299 remains a narrower separator/stripping proposal and does not implement the requested toggle.

## Best Possible Solution

Keep the issue open for maintainer product/security judgment. A good implementation would add a documented, narrowly scoped config option in the shared channel auto-reply prompt path, default metadata injection to enabled, cover trusted Telegram and Discord DM behavior with tests, and document the trust tradeoff in the public channel/config docs.

## Risks / Open Questions

- A broad metadata off switch could remove provenance used for prompt safety, routing, replay cleanup, and copied-metadata stripping. Any implementation should be narrow, explicit, and likely limited to trusted direct chats while preserving safety-critical group/reply/forward/location provenance by default.
