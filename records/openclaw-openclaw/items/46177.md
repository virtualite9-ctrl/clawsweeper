## Summary

Keep this PR open. Current main still has the ambiguous cron tool and CLI help text this PR targets, while the runtime scheduler really does default omitted cron `tz` to the Gateway process timezone. The PR is still useful, but should incorporate the review feedback to say "gateway host/local timezone" rather than the potentially ambiguous "system local timezone" for remote CLI users.

## Best Possible Solution

Keep the PR open and request the small wording adjustment from the review comment: make both CLI help strings and the agent-tool description explicitly say omitted cron `tz` uses the Gateway/gateway-host local timezone, while the UTC note applies only to one-shot `at` timestamps. No runtime change appears necessary.
