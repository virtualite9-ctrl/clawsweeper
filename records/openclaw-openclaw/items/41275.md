## Summary

Keep this PR open. Current main still rejects `timeoutSeconds: 0` in the Control UI and drops zero from the UI-built cron payload, while the backend/protocol already accepts zero as no-timeout. The PR is also paired with same-author open issue #41272 via closing syntax, so it should remain open for maintainer review rather than being closed by cleanup.

## Best Possible Solution

Keep this PR open as the implementation candidate for #41272. The best path is to finish the focused Control UI fix in `ui/src/ui/controllers/cron.ts`, update the controller regression test so `timeoutSeconds: "0"` is accepted and preserved, keep malformed/non-finite input rejected, update the English validation/help copy, and decide whether CLI parity belongs in this PR or a follow-up.
