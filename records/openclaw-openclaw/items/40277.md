## Summary

Keep open. Current main still does not implement this PR's full inline model-error behavior: live chat error events update hidden error state only, persisted assistant error metadata is not extracted into visible text, and grouped chat rendering has no highlighted model-error bubble or empty-detail fallback. The PR remains a focused UI fix, but it needs rebase/maintainer review against newer chat error formatting and related #55923 overlap.

## Best Possible Solution

Keep this PR open for maintainer review or fold its focused UI behavior into the canonical inline-error work. The best implementation should append structured assistant error messages for live chat error events, render persisted assistant `stopReason: "error"` / `errorMessage` entries including empty-detail fallbacks, style model failures distinctly in grouped chat, preserve current structured connect-error formatting, coordinate with #55923, and add focused controller/extraction/render tests on current main.
