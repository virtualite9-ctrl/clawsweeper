## Summary

Keep open. Current main has partial sensitive-data protections, including SecretRefs, config/API/UI redaction, and several log redaction gates, but the umbrella request is not fully implemented: plaintext credential storage remains supported, normal config writes still serialize JSON to disk, normal backup rotation still copies existing config bytes, included config writes do the same, and at least one direct subsystem console sink still bypasses the shared redaction helper in the checked-out main.

## Best Possible Solution

Keep this issue open as the umbrella until maintainers either split and link canonical sub-issues or land the remaining work: an at-rest secret strategy for config/auth artifacts/backups/service outputs, completion of log and diagnostic sink redaction audits, full Control UI sensitive-field inventory, and regression coverage across persistence, logs, API snapshots, diagnostics, and UI masking.
