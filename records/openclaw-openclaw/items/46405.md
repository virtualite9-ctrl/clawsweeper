## Summary

Keep this PR open. Current main has effectively superseded the gateway hot-reload half, but it does not implement the MiniMax XML tool-call parsing/promotion half of the PR. The remaining MiniMax change also has unresolved parser-correctness review concerns, so this should not be auto-closed as implemented.

## Best Possible Solution

Split or rebase the PR: drop the gateway hot-reload changes that current main already handles, then either implement a robust MiniMax XML-to-toolUse path with parser coverage and execution-level regression tests, or have a maintainer explicitly close that remaining work if legacy MiniMax M2.5/XML tool-call support is no longer a supported product path.
