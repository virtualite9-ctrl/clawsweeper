## Summary

Keep this PR open. Current main already fixes the immediate Talk/Control UI microphone regression by shipping `Permissions-Policy: camera=(), microphone=(self), geolocation=()`, but it still does not implement the PR’s central capability: a supported `gateway.http.securityHeaders.permissionsPolicy` config key with override and explicit-disable behavior.

## Best Possible Solution

Keep this PR open for maintainer review or contributor follow-up. The best path is to rebase onto current main, preserve or explicitly re-decide the shipped `camera=(), microphone=(self), geolocation=()` default, add `gateway.http.securityHeaders.permissionsPolicy?: string | false` through config types, zod/generated schema metadata, docs, runtime resolution, and `createGatewayHttpServer`, and keep focused tests for default, override, and explicit-disable behavior. Split or remove unrelated auth-resolution or schema metadata drift before merge.
