## Summary

The Typecast work is useful, but current main already has the speech-provider plugin API needed for it, and the project scope says optional vendor integrations should be maintained as external plugins and discovered through ClawHub unless there is a missing core seam or explicit maintainer ownership reason. The PR diff is a provider-plugin implementation, not a core API fix.

## Best Possible Solution

Close this PR as ClawHub scope and encourage the author or Neosapience to publish the Typecast speech provider as an external plugin package. If publishing exposes a missing SDK hook, the best core contribution would be a narrow plugin-API PR for that seam, not a bundled provider implementation.
