## Summary

Keep this issue open. Current main has several targeted Telegram 400 mitigations, but the report still lacks the exact Telegram `description`, OpenClaw version, affected path, logs, and chat/topic shape needed to decide a fix. Plain `400: Bad Request` remains intentionally terminal for non-idempotent sends, and the issue is only 48 days old as of 2026-04-28, so `stale_insufficient_info` is not an allowed close reason yet.

## Best Possible Solution

Keep this issue open and ask the reporter to provide the redacted exact Telegram error JSON or full GrammyError line, OpenClaw version, whether the failure is send, receive, polling, webhook, or startup, the affected chat/topic target shape, and relevant gateway logs. Once the description is known, route the fix to the Telegram plugin owner path: formatting or empty text in `extensions/telegram/src/send.ts` or `extensions/telegram/src/bot/delivery.send.ts`, target/topic handling in `extensions/telegram/src/targets.ts` and `extensions/telegram/src/reply-parameters.ts`, command-menu overflow in `extensions/telegram/src/bot-native-command-menu.ts`, or network/proxy behavior in `extensions/telegram/src/fetch.ts`, `extensions/telegram/src/network-errors.ts`, and polling/webhook startup code.
