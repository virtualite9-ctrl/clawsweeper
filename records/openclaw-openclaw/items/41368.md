## Summary

Keep open. Current main still does not expose user-facing Feishu Bitable delete tools, and this PR remains a focused Feishu plugin implementation candidate. The related open PR #40697 and issue #55654 cover only single-record deletion, so they do not supersede the broader batch-record and field-delete work here. Security review found no supply-chain or CI-sensitive changes in the PR patch; the security-relevant change is the intended new destructive Bitable tool surface.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to rebase and polish the Feishu plugin implementation in extensions/feishu/src/bitable.ts, expose feishu_bitable_delete_record, feishu_bitable_batch_delete_records, and feishu_bitable_delete_field through the existing registerBitableTool/account-routing pattern, address the schema and response review comments, add targeted tests, and make the destructive nature of the tools clear before merge.
