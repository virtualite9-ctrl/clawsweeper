## Summary

Reviewed this PR (https://github.com/openclaw/openclaw/pull/45677). Current main still has the reported RuntimeLogger metadata drop, so this PR is not obsolete or implemented on main. It should stay open, but the proposed implementation needs adjustment because OpenClaw’s underlying structured logger expects metadata before the message, and the branch also carries unrelated test-only CI restore changes.

## Best Possible Solution

Keep this PR open and request changes: implement `createRuntimeLogging()` as an adapter from the public RuntimeLogger `(message, meta?)` API to the underlying structured logger convention, calling `logger.<level>(meta, message)` when `meta` is a non-empty record and `logger.<level>(message)` otherwise. Add a regression test that proves real structured output or diagnostic attributes, not only a mock call shape, then drop the unrelated cherry-picked test-suite restore changes before merge.
