## Summary

Keep open. Current main still does not implement Feishu `mentionTriggers`: the Feishu schema, reply policy, inbound mention gate, docs, and tests only support native @mention gating. The PR is still aimed at a real bundled Feishu plugin gap, but the provided diff needs revision before merge because keyword matches would not be carried through broadcast active-agent handling and the schema would accept empty trigger strings.

## Best Possible Solution

Keep the PR open and request a revision in the bundled Feishu plugin. The implementation should validate non-empty normalized trigger strings, resolve triggers consistently across top-level, account, wildcard group, and exact group config, treat a keyword match as an effective mention for single-agent and broadcast dispatch, update `activeAgentId` and `WasMentioned` semantics, and add focused tests plus Feishu docs/config metadata updates.

## Risks / Open Questions

- Merging the PR as-is can silently fail in Feishu broadcast groups because keyword-only messages would pass the new early gate while `activeAgentId` remains null and no active Feishu reply dispatcher is selected.
- The proposed `mentionTriggers: z.array(z.string())` accepts empty strings; because `ctx.content.includes("")` is always true, an empty trigger would effectively bypass `requireMention` for every group message if configured.
- The new public config field would need tests, docs, and generated config metadata coverage to avoid schema/documentation drift.
