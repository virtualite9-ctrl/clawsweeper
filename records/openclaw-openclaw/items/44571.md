## Summary

Keep this issue open. Current main has a main-agent auth fallback for missing non-main stores and OAuth refresh/adoption paths, but onboarding still writes provider auth profiles to one resolved agentDir, and the sibling-sync helper is not wired into production onboarding. Stale non-main token/API-key profiles can still shadow main because merged agent profiles override main profiles on conflicts. The related implementation PR #44644 was closed unmerged, so it is not a shipped fix candidate.

## Best Possible Solution

Keep this issue open and fix it in the provider-auth/onboarding owner path: either make onboarding intentionally sync supported auth profiles to configured agent dirs from agents.list, or document/keep per-agent isolation and add a safe repair/warning path for stale non-main auth profiles that shadow main. The implementation should include regression coverage for Anthropic setup-token/tokenRef, API-key SecretRef, missing non-main stores, and stale conflicting non-main profiles.
