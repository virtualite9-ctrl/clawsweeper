## Summary

Keep this PR open. Current main still does not generate `ProcessType=Interactive` in macOS LaunchAgent plists, so the PR’s central behavior is not implemented. Main already handles the `WorkingDirectory` concern by defaulting macOS gateway install plans to the OpenClaw state directory, and the provided PR discussion says the branch was updated to preserve that direction instead of defaulting to `/`. The PR is also paired with an open same-author issue using closing syntax, so it should stay open for maintainer review rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to rebase/verify the current branch, update the PR body to reflect the state-dir `WorkingDirectory` behavior, land or explicitly reject `ProcessType=Interactive` in `src/daemon/launchd-plist.ts` with focused `src/daemon/launchd.test.ts` coverage, and coordinate with #58061 and #62308 so only one canonical ProcessType fix remains.
