## Summary

Keep open. Current main still does not carry OpenRouter `supported_parameters.includes("tools")` into dynamic runtime models, so the normal agent path can still treat `openrouter/perplexity/sonar-deep-research` as tool-capable. The one-shot model-run paths are safer, but they do not resolve the primary agent/runtime case reported here.

## Best Possible Solution

Fix the OpenRouter provider/runtime boundary: add `supportsTools` to `OpenRouterModelCapabilities` from `supported_parameters.includes("tools")`, handle old cache entries conservatively, have `buildDynamicOpenRouterModel()` set `compat.supportsTools` for dynamic OpenRouter models, and add regression coverage for the normal agent runtime plus the one-shot model-run surfaces.
