## Summary

Keep this PR open. Current main still has the Slack same-ts message/app_mention attachment-loss path and still swallows per-file Slack media download failures. The submitted patch is relevant but not complete enough to close or merge as-is: it only unions debounced entries even though media-bearing messages bypass debounce, drops id-less SlackFile objects, and logs raw error text instead of the repo's redacted formatter.

## Best Possible Solution

Keep this PR open and request a narrow Slack plugin revision: preserve files across same-ts message/app_mention handling even when the file-bearing message bypasses debounce, dedupe only ID-bearing files while preserving id-less files, log media-resolution failures through the redacted formatter, and add regression coverage for both attachment preservation and diagnostic logging. Do not move this into core; the behavior is Slack-plugin-owned.

## Risks / Open Questions

- The current PR patch can still lose files when the file-bearing message is immediate and the no-file app_mention wins the in-flight race.
- The PR's file union drops SlackFile objects without id even though the local type allows id to be absent.
- Raw verbose logging of error messages should use formatErrorMessage or an equivalent redacted formatter before merge.
- The PR lacks focused regression tests for attachment preservation across the app_mention race and for media failure diagnostics.
