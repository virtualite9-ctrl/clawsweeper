## Summary

Keep this PR open. Current main still computes both preflight-compaction and memory-flush thresholds from `reserveTokensFloor` only, while config/docs and the Pi settings path treat the floor as a minimum under `reserveTokens`. This PR remains the active implementation candidate for #66830; the earlier related PR #67297 is closed unmerged.

## Best Possible Solution

Keep this PR open as the canonical implementation path for #66830. Land this helper or an equivalent `max(reserveTokens, reserveTokensFloor)` calculation through the preflight-compaction and memory-flush gates, keep the regression tests and changelog credit, then let #66830 close after merge.
