## Summary

Keep open. Current main does not implement the requested short-click auto-send plus 3-second hold-to-continuous mic UX. The older browser dictation button is now absent from the Control UI chat composer; the visible voice path is the separate Realtime Talk control, and the remaining SpeechRecognition helper is not wired to the composer or exposed with one-shot/continuous mode options.

## Best Possible Solution

Keep this open for maintainer product/design judgment. If accepted, implement it in Control UI source, not generated dist: add explicit dictation mode state, pointer/touch/keyboard accessible hold handling, a startStt option for one-shot versus continuous recognition, careful auto-send semantics, no globals for recognition restart state, updated tooltip/aria copy, focused chat/STT tests, and docs aligned with the current Talk-versus-dictation direction. If maintainers want Talk to fully replace browser dictation, close manually with that product decision and update the stale Control UI docs copy.
