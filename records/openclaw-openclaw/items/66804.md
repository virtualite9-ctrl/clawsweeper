## Summary

Keep open. Current main has partial Active Memory latency hardening, but it still does not implement fallback-after-timeout for the selected Active Memory recall model or a local memory_search summary fallback. MiniMax-M2.7 can still produce a timeout result with no injected memory context.

## Best Possible Solution

Keep this issue open unless maintainers intentionally consolidate it into #65309. The likely product fix belongs in extensions/active-memory/index.ts: on timeout, either retry a configured dedicated recall model within a bounded budget or synthesize a local memory_search-based summary, with tests proving that selected-model timeouts do not leave Active Memory with summaryChars=0 when a fallback path is configured.
