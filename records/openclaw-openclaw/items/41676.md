## Summary

Keep this issue open. Current main rotates the structured Gateway file logger, but the reported macOS launchd stderr file remains wired to `gateway.err.log` through `StandardErrorPath` with no OpenClaw-managed size cap or rotation in that supervisor-log path. The prior ClawSweeper comment correctly called out this distinction, and I found no newer current-main change resolving it.

## Best Possible Solution

Add bounded retention for macOS launchd supervisor logs (`gateway.log` and `gateway.err.log`) or route launchd stdout/stderr through an OpenClaw-managed rotating sink. Keep structured file logger rotation as-is, add targeted launchd log cap/truncation tests and docs, and separately rate-limit or summarize the heartbeat-timeout producer if it can be identified.
