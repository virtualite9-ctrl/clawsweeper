## Summary

Keep this issue open. Current main has adjacent Plugin SDK and context-engine lifecycle hooks, including `agent_end`, `before_tool_call`, `after_tool_call`, `subagent_ended`, `afterTurn`, and `onSubagentEnded`, but it still does not implement the requested workspace/internal `HOOK.md` events for `subagent:ended`, `agent:turnComplete`, or tool-call thresholds. The issue is recent and active, with community prior-art discussion around `knowledge-graph` and the reporter offering a workaround, so this needs maintainer API/product judgment rather than conservative cleanup.

## Best Possible Solution

Keep this open for maintainer API judgment. The likely implementation is a narrow, documented internal-hook bridge for `subagent:ended`, plus tested `agent:turnComplete` and tool-count/threshold semantics if maintainers want workspace `HOOK.md` automation. If maintainers prefer plugin-only lifecycle automation, close later with an explicit product decision and docs pointing users to Plugin SDK hooks and Skill Workshop.
