## Summary

Keep open. Current main has adjacent Plugin SDK lifecycle hooks (`agent_end`, `before_tool_call`, `after_tool_call`, `subagent_ended`) plus Skill Workshop auto-capture, and context engines have `afterTurn`/`onSubagentEnded`, but the requested workspace/internal `HOOK.md` lifecycle events are still not implemented. The issue is recent, concrete, only has the `stale` label, and the latest discussion includes a community +1 plus the reporter saying they built a workaround and want to discuss the design.

## Best Possible Solution

Keep this issue open for maintainer API judgment. The likely paths are either to add a narrow internal-hook bridge for `subagent:ended` plus documented/tested `agent:turnComplete` and tool-count events, or to make an explicit product/docs decision that lifecycle automation belongs in Plugin SDK hooks and existing plugins such as Skill Workshop rather than workspace `HOOK.md` hooks.
