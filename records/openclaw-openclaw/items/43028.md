## Summary

Keep this issue open. Current main has cron timeouts, recurring backoff, failure alerts, retention pruning, and JSONL run history, but it still does not implement the requested recurring-job circuit breaker, remove-time session cleanup option/contract, or automatic `.learnings/changelog.md` recording.

## Best Possible Solution

Keep this issue open. The best path is a cron-owned design/implementation split: add a backward-compatible recurring execution circuit breaker in `src/cron/service/timer.ts` with config/types/docs/tests; define an explicit cleanup option across protocol, gateway, CLI, UI, and session/transcript ownership before deleting anything on `cron.remove`; and decide whether existing JSONL `cron.runs` replaces the requested `.learnings/changelog.md` audit trail or whether a new core changelog writer should be implemented.

## Risks / Open Questions

- The issue bundles three separable cron changes, so maintainers may want to split it into narrower implementation issues instead of landing one broad patch.
- Remove-time cleanup needs an explicit ownership contract so OpenClaw does not delete shared, custom, base, or still-referenced sessions/transcripts unexpectedly.
- The changelog request needs product resolution: either document JSONL `cron.runs` as the supported audit trail or add a separate `.learnings/changelog.md` writer if that sink is intended core behavior.
