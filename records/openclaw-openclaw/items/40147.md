## Summary

Keep this PR open. Current main still implements the opposite of the PR's central behavior for Feishu group messages: `thread_id`-only messages still set `threadReply`, the dispatcher still turns `threadReply` into `reply_in_thread`, and current tests explicitly expect that behavior. The PR is not safely closeable as implemented, and related PR #39507 overlaps but proposes different semantics for `root_id`, so it is not a conservative superseder.

## Best Possible Solution

Keep this PR open for maintainer product review or consolidate it only after deciding the intended Feishu `root_id` and `thread_id` semantics. A current-main implementation should update `extensions/feishu/src/bot-content.ts` and `extensions/feishu/src/reply-dispatcher.ts` together, scope any `root_id` force to the intended group surfaces, preserve or intentionally change DM/quote behavior, and add regression coverage for thread-id-only disabled groups, root-id groups, topic sender routing, and p2p quote replies.
