## Summary

Current main still matches the reported regression: a configured heartbeat prompt is resolved into the run prompt, then any prompt mentioning HEARTBEAT.md is passed through the workspace-path hint appender with no guard for user-authored config. The docs still promise custom heartbeat prompts are sent verbatim, and the provided GitHub context shows an open, unmerged implementation candidate with closing syntax, so this issue should stay open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and resolve it by reviewing and landing PR #40756 or an equivalent maintainer patch. The patch should track whether the resolved plain heartbeat prompt came from user config, skip appendHeartbeatWorkspacePathHint for user-authored configured prompts, preserve the hint for built-in/default and internally generated exec or cron prompts, and add a focused regression test for a configured prompt that mentions HEARTBEAT.md.
