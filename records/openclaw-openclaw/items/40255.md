## Summary

Current `main` still appears to reproduce the reported regression. User-configured heartbeat prompts are selected by `resolveHeartbeatPrompt`, then `resolveHeartbeatRunPrompt` unconditionally passes the selected prompt through `appendHeartbeatWorkspacePathHint` whenever it mentions `HEARTBEAT.md`. The docs still promise custom heartbeat prompts are sent verbatim, and the provided GitHub context shows open PR #40756 with closing syntax for this issue, so this issue should stay open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and resolve it by reviewing and landing PR #40756 or an equivalent maintainer patch. The patch should track whether the resolved plain heartbeat prompt came from user config, skip `appendHeartbeatWorkspacePathHint` for those user-authored prompts, preserve the hint for built-in/default and internally generated exec or cron prompts, and add a focused regression test for a configured prompt that mentions `HEARTBEAT.md`.
