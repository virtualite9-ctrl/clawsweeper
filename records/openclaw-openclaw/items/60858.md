## Summary

Keep open. Current main still has the two load-bearing empty-content compaction exits, and the later #71325/current-main work addresses overflow prechecks and loop damping rather than the reported root cause: real persisted transcript content not reaching the compaction message arrays.

## Best Possible Solution

Keep this issue open and fix the upstream compaction input mismatch. A good maintainer path is to add a focused regression that builds a session JSONL with real user/assistant/tool records but an empty live compaction array, then make manual/preflight compaction reload or preserve the persisted transcript before `containsRealConversationMessages` and `session_before_compact` decide there is nothing to summarize. #71325 can remain closed as the related overflow-precheck/damper fix, but this issue should track the remaining trigger/state disagreement.
