## Summary

Current main has later safeguards for OpenAI/GPT-5.4 commentary-phase text and raw tool-call scaffolding, but it does not implement the remaining Matrix/Ollama behavior requested here: suppressing ordinary phaseless text blocks that appear in the same assistant message before structured tool calls. The issue has detailed reproduction data and an agreed practical direction in the comments, so it should stay open.

## Best Possible Solution

Keep this issue open. The best implementation path is a shared reply-delivery option, likely under `agents.defaults`, that can suppress leading text blocks from assistant messages that contain structured tool calls and end with tool-use, with focused tests in the embedded subscription/reply pipeline and at least one Matrix or shared channel delivery assertion. The implementation should preserve structured tool calls and avoid changing transcript replay or provider payloads unless maintainers explicitly want storage-level sanitization too.

## Risks / Open Questions

- A blanket strip could hide useful provider output on channels that intentionally show pre-tool progress; the safest implementation should be config-gated and scoped to assistant messages with structured tool calls plus a tool-use stop reason.
- The existing OpenAI phase/commentary fixes may make the symptom disappear for some providers, but they do not cover the reported Ollama/GLM phaseless message shape.
