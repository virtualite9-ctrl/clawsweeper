## Summary

Current main does not implement the PR's intended `sessions_broadcast` tool. The existing shipped session surface still exposes `sessions_list`, `sessions_history`, `sessions_send`, `sessions_spawn`, `sessions_yield`, `subagents`, and `session_status`, with `sessions_send` remaining single-target cross-session delivery. The PR is also not ready to close as obsolete because its final diff still introduces security-sensitive cross-session broadcast behavior that would need review and fixes before merge.

## Best Possible Solution

Keep this PR open for maintainer review or contributor revision. The best path is to rebase it onto current main, decide whether `sessions_broadcast` belongs in core, and if so implement it as a policy-aware, security-reviewed sibling to `sessions_send`: valid `inputProvenance`, explicit subagent and HTTP invoke denial, mutation/tool-policy coverage, catalog/docs updates, and regression tests for delivery, visibility, and denied surfaces.
