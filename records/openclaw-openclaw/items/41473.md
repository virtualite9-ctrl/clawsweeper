## Summary

Keep this PR open. Current main still does not expose Slack Socket Mode ping/pong timeout configuration, and the PR remains a plausible implementation candidate for the open Slack timeout request, but it should be rebased and updated to use the current public Slack Bolt SocketModeReceiver options instead of mutating internal SocketModeClient fields.

## Best Possible Solution

Keep this PR open as the implementation candidate for #14248. The best path is to rebase it on current main, add the socketMode type/schema/help/labels/docs/tests, pass clientPingTimeout/serverPingTimeout/pingPongLoggingEnabled through createSlackBoltApp into SocketModeReceiver's public options, and remove or revalidate the .secrets.baseline churn before landing.
