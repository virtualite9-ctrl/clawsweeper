## Summary

Current main implements the reported WebChat non-image attachment path: the UI accepts non-video files, serializes PDFs/Office/text files as file attachments, the Gateway saves them to inbound media, prestages them for the agent workspace, and injects MediaPath/MediaPaths plus MIME metadata into the agent dispatch context. The prompt prelude then renders those paths as `[media attached: ...]` context, so the reported 'saved but not injected' behavior is no longer present on main.

## Best Possible Solution

Close this issue as implemented on main. Keep the shipped chat.send media-path staging and prompt-note plumbing, and leave broader document parsing/extraction/configurability requests to the existing WebChat document-upload follow-up rather than reopening this path-injection bug.
