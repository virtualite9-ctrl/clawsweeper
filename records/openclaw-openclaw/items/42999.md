## Summary

Current main still uses fixed 16k/32k context-window guard thresholds, with tests and docs preserving that behavior. There is also an open, unmerged implementation candidate, #72958, whose body says it fixes this issue, so the issue should stay open until that PR is reviewed and either merged or closed.

## Best Possible Solution

Keep this issue open while #72958 is reviewed. The best path is a focused core change in `src/agents/context-window-guard.ts` that derives warn and hard-min thresholds from the resolved effective context window with safe floors, removes the fixed runtime overrides in `src/agents/pi-embedded-runner/run/setup.ts`, preserves `agents.defaults.contextTokens` cap semantics, and updates tests plus public local-model docs.
