## Summary

Keep open. Current main still does not expose the requested `openclaw/plugin-sdk/backup` subpath or public SDK exports for `BackupPlan`, `resolveBackupPlanFromDisk`, and backup archive naming helpers. The PR is stale and has a blank template, but the diff is a focused additive SDK-surface proposal rather than something already implemented or clearly outside the repository.

## Best Possible Solution

Keep the PR open, but require a current-main revision. If maintainers want backup planning as a supported plugin API, add a narrow `src/plugin-sdk/backup.ts` subpath, keep `scripts/lib/plugin-sdk-entrypoints.json` and `package.json` exports aligned, document it in the plugin SDK subpath docs, update the generated API baseline/export checks, and explicitly review whether raw `BackupPlan` is stable enough or should be wrapped in a narrower plugin-facing adapter.
