## Summary

Keep this PR open. Current main still shares `skillsFilter` between the Agents Skills panel and the top-level Skills tab, so the PR's intended state-isolation change has not been implemented or superseded. The linked bug report was closed for a separate autofill mitigation, but the cross-view filter persistence described by this PR remains present in current main.

## Best Possible Solution

Keep this PR open or replace it with an equivalent maintainer patch on current main: add an isolated `agentSkillsFilter` state, add it to `AppViewState`, route only the Agents Skills panel `filter` and `onSkillsFilterChange` through it, keep the top-level Skills tab on `skillsFilter`, and add a small regression test or explicit manual verification for cross-view filter isolation.
