## Summary

Current main has not implemented this PR's central hardening. The WhatsApp channel has moved from `src/web/*` to `extensions/whatsapp/*`, but the equivalent current credential readers still use `stat`/`readFile` paths that follow symlinks, and the socket setup still creates the auth directory through `ensureDir` without `0o700` or a follow-up chmod. This PR remains a focused implementation candidate, though it likely needs rebasing/adapting to the pluginized WhatsApp paths and should add regression tests.

## Best Possible Solution

Keep this PR open and ask for a rebase/adaptation against `extensions/whatsapp/src/{auth-store.ts,creds-files.ts,session.ts}`. The best fix would reject symlinked `creds.json` and `creds.json.bak` before all credential reads/status checks, create/chmod the WhatsApp auth directory to `0o700`, preserve the existing atomic save behavior, and add focused symlink regression tests for the current pluginized paths.
