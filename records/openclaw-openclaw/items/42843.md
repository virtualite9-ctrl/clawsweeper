## Summary

Current main still has the finite WhatsApp reconnect cap and terminal max-attempt stop path this PR is trying to remove. The PR remains a plausible implementation candidate, but it needs refresh for the current WhatsApp plugin layout and config contract before it can merge.

## Best Possible Solution

Keep this PR open and require a refresh against the current WhatsApp plugin architecture. The likely maintainer path is to update `extensions/whatsapp/src/reconnect.ts`, `extensions/whatsapp/src/connection-controller.ts`, and `extensions/whatsapp/src/auto-reply/monitor.ts` so retryable disconnects never permanently kill the monitor, while logged-out, session-conflict, and intended terminal auth-unstable states remain explicit. Add tests for default unlimited retries, bounded degraded-mode retry, non-retryable closes, and auth-unstable behavior, then align `web.reconnect.maxAttempts` schema/help/docs/generated metadata.
