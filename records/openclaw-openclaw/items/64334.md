## Summary

Keep this issue open. Current main still does not add a cumulative transcript-backed session cost line to `/status` or `session_status`; it only shows a per-reply estimated `💵 Cost:` line when API-key pricing is visible. The repository already has cost aggregation for transcripts and a separate `/usage cost` command, but that does not satisfy the requested status-card output.

## Best Possible Solution

Keep this issue open. The product fix should add a cumulative transcript-backed session cost line to the shared `/status` and `session_status` renderer, reuse the existing transcript/session cost aggregation path, preserve the current API-key versus OAuth cost-visibility rules, and add focused tests for multi-turn totals, transcript fallback, and cost hiding.
