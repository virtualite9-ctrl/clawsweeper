## Summary

Current main still does not support registering local skills into the skill search index. Local/workspace skills are loaded for prompt, status, list/info/check, and runtime paths, but `skills.search`, `openclaw skills search`, and the Control UI search remain ClawHub-only, and there is no `skills.register`/CLI register command or protocol shape for local source/path results.

## Best Possible Solution

Keep this issue open. The best path is a core-owned implementation that either extends `skills.search` and `openclaw skills search` with source-aware local or combined results backed by `loadWorkspaceSkillEntries`, or adds an explicit `skills.register` / `openclaw skills register <path>` flow. The change should update protocol schema, CLI/UI behavior, docs, and tests while preserving current ClawHub install/update semantics and local skill security boundaries.
