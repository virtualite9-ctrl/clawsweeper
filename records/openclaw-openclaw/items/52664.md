## Summary

Keep this PR open. Current main still does not expose `rawBody` on `before_prompt_build`, legacy `before_agent_start`, or `agent_end` hook events, and the PR is the active implementation candidate rather than obsolete work. The prior merged #51502 supports the `RawBody` motivation for a Telegram-specific path, but it does not implement the plugin hook API requested here.

## Best Possible Solution

Keep this PR open, rebase/review it against current main, resolve or document the steer-path semantics, add focused regression coverage for the new optional hook field, then land if the SDK baseline and changed checks are green.
