## Summary

Keep this issue open. Current main still has the reported reconnect/tab-return behavior: the Control UI clears chatRunId after hello, the gateway hello snapshot has no active chat-run recovery data, and the chat view's busy calculation does not include abortability. There is also an open same-author implementation PR, #41067, that uses GitHub closing syntax for this issue, so the right maintainer action is to review/land or close that PR rather than close this issue now.

## Best Possible Solution

Keep this issue open and treat #41067 as the implementation candidate. The best path is to rebase/update that PR against current main, verify activeChatRuns snapshot recovery, make the composer busy state depend on abortability, preserve the correct session when navigating back to chat, regenerate/check protocol artifacts, and run the targeted gateway/UI tests before maintainer review. Once the PR lands, GitHub can close this issue as implemented.
