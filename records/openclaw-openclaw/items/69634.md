## Summary

Current main still does not implement this PR’s intended fix. The renderer computes the same action condition but never applies `has-copy` to normal assistant bubbles, while the CSS spacing rule and floating message-level action bar remain active. This PR is a focused implementation candidate for the still-open linked bug, not conservative cleanup to close.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land this `has-copy` class fix or an equivalent maintainer patch, preferably with a focused regression test asserting that assistant bubbles with copy/expand actions receive `has-copy` and bubbles without actions do not.
