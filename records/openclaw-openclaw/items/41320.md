## Summary

Keep this PR open. Current main still has no Tirith integration or `tools.exec.commandSecurity` config surface, and the proposal is a focused security-hardening change to built-in exec rather than an already-shipped or clearly external-only cleanup item.

## Best Possible Solution

Keep this PR open for maintainer security review. The best path is either a narrowed core implementation after validating the external-binary trust model, warning/approval semantics, docs, and changed-gate tests, or an explicit maintainer redirect to a ClawHub/native plugin using `before_tool_call` if optional external command scanners should stay out of core.
