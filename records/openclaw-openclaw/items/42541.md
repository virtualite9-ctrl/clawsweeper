## Summary

Keep open. Current main has fixed some shared OAuth plumbing, including runtime snapshot refresh after token rotation, but the Gemini-specific silent refresh failure is not fully resolved or proven fixed. The Google Gemini CLI provider still has no provider-owned refresh hook, auto profile resolution can still collapse refresh failures into debug-only profile iteration, and the requested cooldown behavior for google-gemini-cli remains absent.

## Best Possible Solution

Keep this issue open. The best maintainer path is to add a focused google-gemini-cli OAuth refresh regression that exercises an expired stored profile through the embedded runtime/profile-resolution path, preserve and surface the underlying refresh error, verify successful refresh updates both disk and runtime snapshots, and make an explicit product decision on Gemini auth cooldown behavior. If the root cause belongs in pi-ai, pin the dependency source evidence or add a provider-owned refreshOAuth hook in the Google plugin.
