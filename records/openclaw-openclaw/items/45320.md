## Summary

Keep this PR open. Current main does not implement the PR's central behavior: Discord partial preview streaming still edits one preview message, while preview rotation remains hard-wired to block mode. The PR is not obsolete, but it needs to be ported to the current pluginized Discord layout and should address the unresolved review comment about stale in-flight send failures before merge.

## Best Possible Solution

Keep the PR open as useful but not merge-ready. The best path is to port the option into the current Discord plugin surface, probably under the nested streaming preview config shape, wire `message-handler.process.ts` to resolve that option instead of `discordStreamMode === "block"`, add strict schema/UI/docs coverage, add a partial-mode boundary-rotation regression test, and fix the stale in-flight send failure path before review/merge.
