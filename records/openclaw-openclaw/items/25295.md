## Summary

Keep this PR open. Current main still does not expose Mattermost `edit` or `delete` through the shared message action adapter. Main has internal Mattermost update/delete helpers for preview streaming, but message-tool discovery, support gating, action handling, docs, and Mattermost channel tests still cover only send/react for this surface. The PR remains a relevant implementation candidate, but the provided diff is not merge-ready because it mixes the Mattermost feature with unrelated infra/test/changelog changes and needs an extension-boundary cleanup.

## Best Possible Solution

Keep this PR open and request a focused refresh against current main: wire Mattermost `edit` and `delete` through `describeMattermostMessageTool()`, `supportsAction()`, and `handleAction()`; reuse the existing Mattermost update/delete client helpers through a local runtime seam or SDK-approved helper imports; validate non-empty edit text; add targeted Mattermost action tests and docs; and drop unrelated infra/test/changelog changes before maintainer review.
