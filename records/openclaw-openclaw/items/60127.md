## Summary

Keep open. Current main still does not implement the requested shared-server multi-tenancy model with tenant identity propagation, tenant RBAC, per-tenant quotas, tenant-scoped plugin enablement, or tenant-attributed audit boundaries. Existing multi-agent and operator-scope features are adjacent but explicitly not a hostile multi-tenant security boundary.

## Best Possible Solution

Keep this open for maintainer product and security triage. If shared-server multi-tenancy is in scope, design tenant identity propagation through Gateway auth/request context/protocol, tenant-scoped state/config/plugins/skills/model auth, per-tenant quotas, tenant-attributed audit logs, and cross-tenant denial tests/docs. If it is out of scope, maintainers should document that hostile shared-Gateway tenancy is unsupported and split narrower issues only for specific plugin/API seams external managed-service implementations need.
