## Summary

PR #51327 should stay open. Current main still scopes `sessions.usage` discovery to agents returned by `listAgentsForGateway`, while `agents.delete` removes the agent from config and purges session-store entries even when transcript files are retained. The linked bug #50553 remains a real, unsolved product issue; this PR is stale and needs rework, but it is not implemented or superseded on main.

## Best Possible Solution

Keep the PR open but require a rebase/rework against current main. The durable fix should add a usage-only discovery target path that includes retained transcript directories for deleted/disk-only agents, keeps `listAgentsForGateway` scoped to configured agents, uses real on-disk `sessionsDir` values instead of rebuilding paths from normalized ids, handles nonfatal scan errors, and adds regression coverage for `agents.delete` with `deleteFiles=false` after session-store purge. The linked bug #50553 should remain the canonical product problem until that lands.
