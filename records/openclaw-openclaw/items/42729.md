## Summary

Keep this PR open. Current main still lacks the stale-slot guard in the runnable hot path, so a persisted past-due nextRunAtMs can be selected for startup catch-up before maintenance repair has a chance to advance it. The PR remains a narrow functional fix with a focused regression test, and the discussion includes a concrete field confirmation of the duplicate heartbeat behavior on v2026.3.31.

## Best Possible Solution

Keep this PR open and review or land it, or port the same strict stale-slot guard into current main's isRunnableJob/startup catch-up path with the PR's regression test. The linked bug report is closed, but current main still appears to need this focused scheduler fix.
