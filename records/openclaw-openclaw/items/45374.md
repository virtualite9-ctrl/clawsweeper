## Summary

Keep open. Current main already covers much of the stale plugin cleanup side, including stale plugin id repair, dangling channel cleanup, and redundant bundled plugin load-path cleanup. It does not yet implement the PR's runtime drift surface: `gateway status` still reports only config-path mismatch, and current doctor/service repair code does not classify `~/.openclaw-tests/*` state dirs or compare the live service's `OPENCLAW_SERVICE_VERSION` with the CLI/runtime version for ordinary status/doctor output.

## Best Possible Solution

Keep this PR open only as an implementation candidate for the still-missing runtime drift surface. Rebase it onto current main, drop or adapt the stale-plugin cleanup pieces that main now handles through `src/commands/doctor/shared/*`, and focus the remaining work on a well-tested `gateway status`/doctor report for service state-dir drift, `~/.openclaw-tests/*` usage, and service-vs-CLI version mismatch.
