## Summary

Keep this PR open. Current main still throws from the Slack socket monitor on non-recoverable auth startup errors and reconnect-attempt exhaustion, so the PR's intended provider-level graceful stop is not already implemented. The linked crash-loop issue is closed and current main has a related Bolt constructor auth-verification guard, but that does not cover the app.start()/socket reconnect branches this PR targets.

## Best Possible Solution

Keep the PR open as a plausible focused Slack bug fix. The best path is to update it against current main so terminal Slack auth/startup failures stop the Slack account without crashing or restart-churning the gateway, while preserving a visible channel lastError/status, and to add focused regression coverage for the startup auth path plus any reconnect terminal paths the final patch changes.
