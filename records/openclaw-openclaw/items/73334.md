## Summary

Keep this PR open. Current main does not already implement the PR's intended Feishu document-link parser extraction: `bitable.ts` and `comment-shared.ts` still carry separate parsing logic, and the proposed shared `document-link.ts` file is absent. The PR is an implementation candidate, but it needs maintainer review because it also includes unrelated core SQLite helper extraction and appears to broaden bitable URL handling despite claiming no user-visible behavior change.

## Best Possible Solution

Keep this PR open for maintainer review. The Feishu parser extraction can be reviewed as a focused plugin cleanup, ideally with the behavior expansion called out and covered by targeted tests. The unrelated task-registry SQLite helper extraction should be split into a separate core refactor or removed from this PR unless a maintainer explicitly wants the combined change.
