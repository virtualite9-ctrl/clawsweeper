## Summary

Keep this issue open. Current main still has the same DeepSeek V4 replay gap: it preserves real `reasoning_content` only when the replayed assistant message already carries a thinking block, but missing assistant tool-call reasoning is still filled with an empty string. The fresh discussion has multiple independent reproductions of the long multi-turn/tool-call failure, and the current tests cover short or prior-provider replay paths rather than the reported `sessions_spawn`/long-chain failure.

## Best Possible Solution

Keep this issue open and implement a targeted DeepSeek V4 replay fix in the bundled DeepSeek/OpenAI-compatible payload path. The fix should preserve or recover actual prior `reasoning_content` for DeepSeek-origin assistant tool-call turns across chained continuations instead of relying on empty placeholders, then add focused coverage for multi-turn tool-call chains and a `sessions_spawn`-style subagent path. A live DeepSeek V4 regression with more than one tool-call continuation should be the acceptance bar.
