## Summary

Keep this PR open for explicit maintainer judgment rather than auto-close. Current main still excludes `MEMORY.md` from the minimal subagent/cron bootstrap path, but source, tests, and docs indicate that exclusion is an intentional privacy/minimal-context contract, not an already-implemented version of the PR. The PR’s two-line change also has a security-relevant behavior change: it would automatically inject durable private memory into background subagent and cron sessions.

## Best Possible Solution

Do not merge this PR as-is. Maintainers should either close it after confirming the current privacy/minimal-bootstrap contract is intended, or redesign the feature as an explicit opt-in memory/context policy with updated source, tests, docs, and a security review for subagent and cron runs.
