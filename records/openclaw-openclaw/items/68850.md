## Summary

Keep open. Current main still trusts a non-empty session `displayName` when no explicit `label` is present, so a synthetic transport name like `webchat:g-agent-control-tower-main` can still reach the Control UI selector. This PR is relevant, but the posted review comment identifies a correctness issue in the proposed patch: it should filter transport-generated `displayName` values without discarding user-authored `label` values.

## Best Possible Solution

Keep this PR open and revise it against `ui/src/ui/chat/session-controls.ts`: filter synthetic transport names from `displayName` in the display-name and selector-label path, preserve explicit `label` as authoritative, and add focused tests for displayName-only main/subagent rows plus an explicit-label preservation case.
