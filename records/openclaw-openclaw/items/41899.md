## Summary

Keep open. Current main has plugin load/config isolation, static hook error policy, one void-hook timeout, diagnostics, manual plugin enable/disable, and subsystem-specific circuit breakers, but it does not implement the requested framework-level plugin-scoped circuit breaker with rolling failure/timeout/lock-overrun accounting, persisted breaker state, open/half-open/forced-open states, selective hook skipping, status visibility, or reset controls.

## Best Possible Solution

Keep this issue open as core plugin/gateway stability work. The best path is an additive plugin-runtime breaker layer around hook and plugin-owned runtime execution, with structured plugin id + hook/session telemetry, lock-overrun attribution, rolling failure/timeout budgets, persisted breaker state, status/diagnostics output, operator reset/forced-open controls, and tests proving one failing plugin is degraded without blocking unrelated plugins. Coordinate with #55253 if maintainers want one canonical tracking issue.
