## Summary

Keep this PR open. Current main still does not preserve under-cap non-alpha PNGs in the shared web media path, and JPEG filename normalization still only applies to HEIC inputs. The PR remains useful, but needs a rebase from the old src/web paths to src/media/web-media.ts and should address the PNG validation concern raised in review.

## Best Possible Solution

Keep this PR open and ask for a rebase onto src/media/web-media.ts plus the current media tests. The best fix is to preserve under-cap PNGs only after validating the PNG payload, normalize every actual JPEG output filename to .jpg, and add focused regression coverage for under-cap PNG preservation plus PNG-to-JPEG filename normalization.
