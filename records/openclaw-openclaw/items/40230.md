## Summary

Keep this PR open. Current main still does not follow workspace bootstrap symlinks that resolve outside the workspace root: the guarded reader returns failed boundary reads directly, fixed bootstrap files are then marked missing, and the prompt renderer emits the observed [MISSING] marker. The PR remains a focused implementation candidate for the linked user-facing bug and overlapping open reports, but it still needs maintainer/security review because it deliberately creates a narrow exception to the workspace boundary model.

## Best Possible Solution

Keep this PR open for maintainer/security review, or replace it with a maintainer-authored patch in `src/agents/workspace.ts`. The best fix should allow symlinked fixed top-level bootstrap files only when explicitly accepted by maintainers, preserve boundary-reader safety properties as much as possible, keep extra/glob bootstrap paths constrained to the workspace, and include regression tests for external symlink success, symlink-to-hardlink rejection, dangling/non-regular/oversized targets, and extra-bootstrap rejection.
