## Summary

Keep this issue open. Current main has Telegram reaction notifications, but it only enqueues standard emoji additions as system context events; it does not provide first-class reaction wake triggers, per-emoji semantics, custom_emoji normalization, or reaction execution policy. The provided GitHub context also shows an open same-author PR that uses closing syntax for this issue, so the issue should remain open until that PR is reviewed/merged or explicitly closed by maintainers.

## Best Possible Solution

Keep this issue open while maintainers review the open same-author PR that targets this feature. If that PR is viable, land a narrow Telegram-owned implementation that adds typed reaction semantics, custom_emoji normalization, observability, tests, and docs. If maintainers reject that PR, keep the issue or narrow it to the still-open wake-trigger bug #64752 plus a separate product/API discussion for broader per-emoji execution semantics.
