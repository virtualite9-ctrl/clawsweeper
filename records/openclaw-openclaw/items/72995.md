## Summary

Keep this issue open. The report is recent, actionable, and not covered by an implemented fix on current main. Current web_search plumbing still does not propagate the run/tool AbortSignal into provider-owned web search implementations, and the Gemini provider path only receives a timeoutSeconds value. The related #72986 was closed as not planned and is not a canonical open fix or merged implementation for this more detailed report.

## Best Possible Solution

Keep this issue open and investigate in the Gemini web_search/provider-web-search path. The likely maintainer path is to add a backwards-compatible way for managed web_search providers to receive the tool/run AbortSignal or otherwise explicitly classify provider fetch aborts, log/diagnose AbortError failures, and add a focused regression test for instantaneous Gemini abort behavior plus timeoutSeconds handling.
