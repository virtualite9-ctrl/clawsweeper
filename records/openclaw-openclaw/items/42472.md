## Summary

Keep this issue open. Current main still does not implement the requested shared post-processing rule: the common reply normalizer strips a trailing mixed `NO_REPLY` token and delivers the preceding text, and multiple delivery/history paths still only suppress exact silent-token payloads. The supplied issue timeline also shows open implementation candidates, including PR #42873 with closing syntax for this issue, so this issue should not be closed before those candidates are reviewed, merged, or closed.

## Best Possible Solution

Keep this issue open. Maintainers should review or rework PR #42873 and PR #46626 against current main, preferably by adding a shared helper for own-line/trailing silent-token intent and applying it consistently in `normalizeReplyPayload`, directive/outbound delivery, gateway live/history projection, cron/subagent paths, and runtime delivery classification, with tests preserving legitimate inline mentions, media payload behavior, exact-token policy, and leading substantive text.
