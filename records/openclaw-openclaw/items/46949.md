## Summary

Keep this PR open. Current main still has the ACP max-concurrency behavior the PR is trying to change: cached runtime handles are counted until TTL cleanup or explicit close, startup identity reconciliation does not release oneshot handles, and maxConcurrentSessions enforcement still rejects at capacity without pressure eviction. The PR is a plausible core fix, but it needs maintainer review/rebase and should address the remaining pressure-eviction safety comments before landing.

## Best Possible Solution

Keep this PR open as a plausible core ACP lifecycle fix. The maintainer path is to rebase it on current main, preserve the focused oneshot startup-release and pressure-eviction tests, address the remaining hard-timeout/capacity-recheck review comments, then run targeted ACP control-plane tests plus the changed gate before merge.
