## Summary

Keep this PR open. Current main still lacks the proposed `feishu_chat` chat-list action and still routes the chat tool through the first enabled Feishu account instead of the calling agent/account context. The PR is useful but needs rebase and security cleanup before merge, especially around disabled-account rejection for explicit `accountId`.

## Best Possible Solution

Keep the PR open for owner review and rebase it onto current main. The best path is to preserve current `member_info` and `typebox` usage, add `list` to `feishu_chat`, route chat clients through the existing Feishu tool-account seam using `ctx.agentAccountId` plus explicit `accountId`, reject disabled accounts and disabled chat tools at execution time, and keep focused tests for pagination, per-agent routing, explicit override, disabled-account rejection, and disabled-tool rejection.
