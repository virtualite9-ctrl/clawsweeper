## Summary

Keep this issue open. Current main still ships the reported broad mandatory memory-search guidance in both the default memory prompt section and the `memory_search` tool description, while the separate live-check/time guidance does not explicitly carve current system state, configuration, service-status, or time questions out of memory recall.

## Best Possible Solution

Keep this issue open and fix it in `extensions/memory-core/src/prompt-section.ts` and `extensions/memory-core/src/tools.ts`: frame memory recall as historical prior-work/preferences/decision/todo recall, explicitly direct current system state, configuration, time, and service-status questions to live verification tools first, and add focused regression coverage in `extensions/memory-core/index.test.ts`. If prompt precedence remains ambiguous, add a `buildAgentSystemPrompt` composition test covering Execution Bias plus Memory Recall.
