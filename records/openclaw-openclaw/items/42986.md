## Summary

Keep this issue open. Current main still requires a registered `subagent_spawning` lifecycle hook before native `sessions_spawn({ mode: "session", thread: true })` can bind a thread, and the bundled Telegram entry still does not register native subagent hooks. Telegram has adjacent current-conversation and ACP topic binding support, but that is separate from the requested native subagent session lifecycle. Open PR #42707 remains the focused implementation candidate rather than shipped behavior.

## Best Possible Solution

Keep this issue open and review/land #42707 or an equivalent focused implementation. The fix should live in the Telegram plugin boundary: add `registerFull` hook wiring, implement `subagent_spawning`, `subagent_delivery_target`, and `subagent_ended`, bind native session-mode subagent spawns through Telegram topic/current-conversation state, honor Telegram thread-binding feature flags, clean up bindings on failure/end, resolve completion delivery targets, and update native subagent docs/tests so ACP topic binding and native subagent thread binding remain distinct.
