## Summary

Keep open. Current main already implements the core NVIDIA onboarding discoverability path via runtime auth metadata and manifest-driven CLI flag metadata, but it does not fully implement this PR's stated intent: the NVIDIA provider docs still lack an on-prem/self-hosted NIM section, and the current NVIDIA catalog does not mark the Nemotron default as reasoning-capable. The PR is also stale against current main's manifest-backed catalog shape, so it should be rebased and narrowed rather than auto-closed.

## Best Possible Solution

Keep this PR open for maintainer review, but ask for a rebase and scope reduction: drop the already-shipped NVIDIA onboarding auth-choice and manifest flag churn, then keep or resubmit the missing NIM docs and any verified reasoning-capable catalog/test change against the current manifest-backed NVIDIA plugin structure.
