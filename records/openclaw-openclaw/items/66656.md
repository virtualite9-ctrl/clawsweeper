## Summary

Keep open. Current main still lacks the PR's diagnostic-heartbeat skip in the transcript tail scanner, so a heartbeat diagnostic with an assistant/user-shaped message can still become lastMessagePreview. The PR is narrow source/test work with no obvious security or supply-chain concern, and related heartbeat/display-name work does not implement this preview behavior.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is a focused guard in readLastMessagePreviewFromOpenTranscript with regression coverage, after deciding whether the filter should be heartbeat-only or all diagnostic transcript entries and confirming the relevant session-selector consumer.
