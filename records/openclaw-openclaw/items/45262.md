## Summary

Keep this PR open. Current main still does not implement Slack `slashCommand.contextAware`: empty single slash commands still return `Message required.`, and the Slack config type, strict schema, resolver, and docs expose only `enabled`, `name`, `sessionPrefix`, and `ephemeral`. The PR is a plausible feature candidate, but it needs a current-layout rework and security fixes before merge.

## Best Possible Solution

Keep the PR open for maintainer/product judgment, but require a reworked implementation on the current `extensions/slack` layout: add typed config, strict schema, generated metadata/docs, and tests; preserve the current `Message required.` default when the option is false; fetch recent history only for an authorized empty room-ish invocation; reuse Slack supplemental-context filtering/formatting; and pass history through a non-command context surface such as `UntrustedContext` or an equivalent safe field so old messages cannot be parsed as inline directives.
