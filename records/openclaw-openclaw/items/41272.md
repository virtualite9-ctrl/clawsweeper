## Summary

Current main still has the reported Control UI bug: `timeoutSeconds: 0` is rejected during form validation and would be omitted from the saved cron payload even if validation were changed alone. The issue is also paired with open same-author PR #41275, so it should stay open until that PR or an equivalent complete fix lands.

## Best Possible Solution

Keep this issue open and review #41275 or an equivalent patch. The final fix should allow zero in `validateCronForm`, preserve explicit `timeoutSeconds: 0` in `buildCronPayload`, update Control UI regression tests, and align the user-facing timeout help/error text with the gateway contract.
