## Summary

Keep this issue open. Current main has useful related fixes, especially the opt-in `--wrapper`/`OPENCLAW_WRAPPER` service path and later gateway restart/probe hardening, but it still does not prove or fully implement a default-safe fix for the macOS npm/shebang provenance path, the task/task-flow chmod EPERM restore warnings, or the newer heartbeat/main-session system-event regression reported in the latest comment.

## Best Possible Solution

Keep this issue open. The maintainer path is to provide a supported default-safe macOS package/LaunchAgent entrypoint or an explicit provenance diagnostic/workaround, make task and task-flow registry permission hardening tolerate benign chmod EPERM when existing permissions are already acceptable, and investigate or split the 2026.4.25 heartbeat/main-session `systemEvent` regression with package or live-model proof. Add package acceptance coverage for npm-installed `openclaw gateway probe`, `openclaw status`, startup registry restore, and main-session system-event cron/heartbeat behavior on macOS.
