## Summary

Keep open. Current main has nearby reset and provider-default mitigations, but bare `/new` is still intentionally a fresh-session reset that preserves user-selected model and thinking/reasoning controls. That does not prove the reported Opus recovery problem is fixed, and this issue is concrete and recent enough to remain actionable.

## Best Possible Solution

Keep this issue open for maintainer/product judgment. The best fix is a deliberate recovery path, such as a `/new` recovery mode, `/recover`, stuck-run detector, or channel-safe prompt that can abort the active run and optionally clear thinking/reasoning and model overrides. If maintainers decide current behavior is intended, docs and command UX should say that `/new` preserves selected model/thinking and recommend `/think off`, `/new <model>`, or a model switch for reasoning-loop recovery.
