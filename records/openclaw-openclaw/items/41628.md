## Summary

Keep open. Current main already implements the ID-prefix approval lookup portion safely, but it still does not implement the reply-to-message `/approve <decision>` flow or advertise that shortcut. The rebased PR is active, scoped to the remaining behavior, and worth maintainer review after addressing the remaining ID-line parsing edge case.

## Best Possible Solution

Keep this PR open and review it as the remaining reply-to-message approval feature. The best path is to retain current main's shipped `lookupPendingId` behavior, add reply-body ID extraction inside the existing `handleApproveCommand` flow without weakening exec/plugin approval authorization, parse the full trimmed `ID:` line, update both exec and plugin approval hint text, and keep focused tests for reply-only exec/plugin approvals plus whitespace/custom-ID edge cases.
