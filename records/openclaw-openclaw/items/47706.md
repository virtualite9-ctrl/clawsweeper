## Summary

Keep open. Current main does not implement the PR's central adaptive low-memory/ARM resource profile: there is no resource-profile utility or env override, FFmpeg/image compression limits remain static, memory indexing/search defaults remain static or manually configured, and generated systemd units do not emit MemoryHigh/MemoryMax. The related sandbox-resource report is a different surface and does not supersede this PR.

## Best Possible Solution

Keep this PR open as an implementation candidate. The best path is to rebase onto current main, port the memory/browser paths to their current locations, add a small generic resource-profile helper or equivalent, thread it through FFmpeg/image/memory/systemd/memory-lancedb with focused tests, resolve the outstanding test-global cleanup comment, and avoid expanding systemd unit inputs without validation.
