## Summary

Keep open. Current main still rejects whitespace in Docker config and workspace paths, so the user-facing bug is not implemented on main. The PR intent remains valid, but the patch is stale against current main because Docker setup logic now lives in `scripts/docker/setup.sh`, and the proposed single-quoted YAML emission still needs escaping for embedded apostrophes before it is safe to land.

## Best Possible Solution

Keep this PR open as a valid implementation candidate, but require a rebase/port to `scripts/docker/setup.sh`, a small safe YAML scalar emitter that escapes single quotes, coverage for config/workspace/extra mount paths with spaces and apostrophes, and confirmation that the sandbox socket overlay is quoted safely.

- The PR needs to be rebased or manually ported to `scripts/docker/setup.sh`; merging the old root-script patch as-is will not match current main cleanly.
