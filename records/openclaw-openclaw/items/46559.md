## Summary

Keep open. Current main does not implement the PR's core behavior: assistant chat deltas, flushes, and final/error chat payloads are still sent to session/node subscribers without a verbose-off guard, and current tests still assert verbose=off session chat delivery in the flush-before-tool path. The PR is also not clean enough to treat as ready or obsolete: review history flagged final-message/cache issues, and the supplied diff includes unrelated SDK/channel changes that need to be dropped or split.

## Best Possible Solution

Keep this PR open only as an implementation candidate if it is rebased and narrowed. The best path is a focused gateway fix that gates session/node delivery for assistant deltas, flushed buffered deltas, final done payloads, and error payloads when verbose resolves to off, while keeping WS/Control UI broadcasts intact. Add regression tests for delta, flush-before-tool, lifecycle final, error final, session/default verbose resolution, and in-flight verbose changes. Drop the unrelated plugin SDK/channel churn or split it into owner-reviewed PRs. #25592 should remain the canonical open product bug until a clean fix lands.

## Risks / Open Questions

- Merging the PR as-is could leave the original leak unresolved if final, error, or flush-before-tool payloads are not gated along with deltas.
- A cached verbose decision can be wrong if the session key appears later or `/verbose` changes while a run is active; the docs say subsequent verbose tool bubbles should honor in-flight changes.
- The supplied diff contains unrelated plugin SDK and channel config/setup changes across many plugins, which could regress public SDK boundaries, plugin config validation, or channel group policy behavior independently of the gateway leak fix.
