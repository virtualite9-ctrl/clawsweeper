## Summary

Keep open. Current main still does not implement the PR's intended config-warning dedupe in `createConfigIO()`: both the load and write paths emit `Config warnings:` without a warning fingerprint guard, and current tests only cover newline formatting. The PR remains a relevant implementation candidate, but it is incomplete because it covers only the load warning branch and has an unresolved stale-state review concern. The broader maintainer PR #38295 overlaps, but it is still open and not merged, so this PR should not be closed as implemented.

## Best Possible Solution

Keep this PR open for maintainer review unless maintainers explicitly choose #38295 as the canonical implementation. The best fix is a centralized warning fingerprint helper in `src/config/io.ts` that covers both load and write warning emitters, clears state on no-warning, missing, invalid, parse-failed, and early-return paths, re-logs when raw config content or formatted warning details change, and adds focused regression tests for repeated load/write plus warning -> no-warning/invalid/missing/parse-failed -> same-warning transitions.
