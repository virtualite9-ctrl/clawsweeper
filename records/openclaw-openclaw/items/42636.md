## Summary

Current main still uses a 200-message Control UI chat.history request and a 200-message rendered-history window, so this PR’s intended 80-message performance guard is not implemented. The PR remains useful, but it is stale against current main because render-window logic now lives in ui/src/ui/chat/build-chat-items.ts while the supplied diff still changes the older view-local path.

## Best Possible Solution

Keep this PR open and ask for a current-main update. The best implementation is to define one shared initial chat history window constant in ui/src/ui/chat/history-limits.ts or a nearby chat constants module, use it from ui/src/ui/controllers/chat.ts for the chat.history request and ui/src/ui/chat/build-chat-items.ts for the render slice/notice, keep input-history alignment intentional, update ui/src/ui/controllers/chat.test.ts, and add a direct render-window assertion in ui/src/ui/chat/build-chat-items.test.ts. No Gateway protocol, server, dependency, workflow, or security-sensitive change is needed.
