## Summary

Keep this PR open. Current main still does not preserve Discord card/component text for fallback button or select clicks, and the proposed fix is useful but stale because it targets the removed `src/discord` layout instead of the current `extensions/discord` plugin. Security review found no supply-chain-sensitive changes in the provided diff.

## Best Possible Solution

Keep this PR open and ask for a rebase/port into the current Discord plugin. The best patch would add `cardText` or an equivalent structured context field to `extensions/discord/src/components.types.ts`, collect `spec.text`, fallback text, and visible text/section block content in `extensions/discord/src/components.ts`, preserve it through the component registry, include it only in ordinary fallback event text from `extensions/discord/src/monitor/agent-components.ts`, and add regression coverage in `extensions/discord/src/components.test.ts` and `extensions/discord/src/monitor/monitor.test.ts`. Prompt guidance should use the existing Discord plugin `messageToolHints` path rather than core Discord special-casing.
