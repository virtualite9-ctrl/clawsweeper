## Summary

Keep open. Current main has several TUI streaming/history mitigations, but it does not yet implement the reported TUI viewport behavior: no scroll anchor, no auto-follow-only-when-at-bottom policy, no jump-to-latest affordance, and no TUI auto-scroll setting. The related Control UI/WebChat request remains a different surface, so it should not supersede this TUI-specific report.

## Best Possible Solution

Keep this TUI-specific UX issue open and fix it in src/tui by adding explicit chat viewport state: preserve the user's reading position, auto-follow only when already near the bottom, provide a clear jump-to-latest affordance/shortcut, and consider a TUI setting for follow behavior. Add focused regression coverage around streaming updates, final-event history reloads, and user-scrolled state; keep #7648 scoped to Control UI/WebChat unless maintainers intentionally create a shared cross-surface tracking issue.
