## Summary

Keep this PR open. Current main at a25366038503adcc31a5fa64b3d2f295bd5dbfd2 still does not implement optional Discord run-start ack reaction timing: there is no `ackReactionTiming` config key in Discord types/schema/help/docs, and Discord still queues the initial ack/status reaction immediately when the inbound message is accepted. The PR remains a plausible implementation candidate, though it likely needs a rebase onto the current `extensions/discord` plugin layout.

## Best Possible Solution

Keep this PR open and review it as an implementation candidate. The best path is to rebase the work onto current `extensions/discord`, preserve the guarded deferred lifecycle from the later PR commits, add the channel-scoped `channels.discord.ackReactionTiming` type/schema/help/docs surface, and keep focused regression tests for default immediate behavior, run-start deferral, scope gating, and no-reaction-before-run-start cases.
