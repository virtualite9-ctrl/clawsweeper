## Summary

Keep open. Current main still does not implement the install.sh same-version skip or the --force / OPENCLAW_FORCE_INSTALL override, so this PR cannot be closed as implemented. The PR is also not safe to close as ready/superseded because the discussion includes an unresolved maintainer-reproduced false-skip concern around trusting a PATH openclaw binary instead of the npm-managed package.

## Best Possible Solution

Keep this PR open for maintainer judgment or request changes. A safe version should compare the target version against the npm-managed OpenClaw package itself, for example via npm root -g/openclaw/package.json or verified npm-global-bin ownership, preserve finalization diagnostics such as PATH warnings and gateway refresh, expose --force / OPENCLAW_FORCE_INSTALL intentionally, and add focused install.sh tests for the skip and false-positive cases.
