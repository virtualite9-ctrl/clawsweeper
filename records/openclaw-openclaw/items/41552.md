## Summary

Keep this PR open. Current main still has the Overview behavior the PR targets: editing the WebSocket URL clears the in-memory gateway token, and existing browser coverage asserts that behavior. The PR is not obsolete, but its exact one-line preservation fix needs maintainer review because it can carry a bearer token from one gateway URL to another, conflicting with the current per-gateway token scoping model.

## Best Possible Solution

Keep this PR open for maintainer review or rework. The best fix is to stop losing the token during harmless editing while preserving the per-gateway credential boundary, for example by using a draft URL field, only applying endpoint changes after confirmation, preserving tokens only when the normalized gateway scope is unchanged, and requiring an explicit token for a different gateway scope.
