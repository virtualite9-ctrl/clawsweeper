## Summary

Keep this PR open. Current main still does not expose inbound media paths, URLs, or MIME types on `PluginHookAgentContext` for `before_agent_reply`, and a handled hook still returns before normal inbound media staging. The PR addresses a real plugin API gap, but it is stale against the current hook-types split and still needs tests, docs, and maintainer API/security review before it can land.

## Best Possible Solution

Keep this PR open. The best path is to rebase it onto current main, add `mediaPaths`, `mediaUrls`, and `mediaTypes` to `src/plugins/hook-types.ts`, pass normalized defensive copies from `getReplyFromConfig`, document the fields on the public plugin hooks page, and add focused tests for text-only, local-path, URL-only, media-type alignment/defaulting, handled/declined, and mutation-safety cases before maintainer API/security review.
