## Summary

Keep PR #59592 open. Current main still only treats `stopReason === "error"` as an error-surfacing assistant turn, so an aborted timeout turn with collected assistant text can still produce a payload and skip the clean timeout fallback. The PR also contains targeted regression coverage for the Greptile-identified reasoning leak gap that is absent from main.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to rebase it onto current main, preserve the intended external-abort partial-output behavior, and land focused payload-builder coverage that proves timed-out/aborted embedded runs surface clean timeout/error text without leaking partial answer or reasoning content.
