## Summary

Keep this PR open. Current main still treats `workspaceAccess: "none"` as read-only across Docker mounts, FS bridge write checks, memory flush gating, tests, and docs, so the PR’s intended behavior is not implemented on main. The PR is also a focused implementation candidate for still-open sandbox writability reports, but it needs maintainer review and broader backend consistency checks before landing.

## Best Possible Solution

Review and either land a corrected version of this PR or replace it with a maintainer-owned patch that changes `workspaceAccess: "none"` consistently across Docker/browser mounts, local and remote FS bridges, memory flush gating, docs, and regression tests. Include live or targeted runtime proof that writes stay confined to the sandbox workspace and do not expose the agent workspace.
