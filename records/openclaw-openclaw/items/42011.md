## Summary

Current main does not yet close the reported Control UI stuck-Stop failure. The code still depends on a terminal lifecycle/chat event to clear the UI run state, the embedded timeout path still aborts without directly synthesizing that terminal event, and an open implementation PR explicitly references this issue with closing syntax, so the issue should remain open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review #73105 as the direct backend implementation candidate. The best fix is to guarantee exactly one terminal lifecycle/chat state for embedded timeout or abort teardown even when `agent_end` is missed; a Control UI `agent.wait` watchdog such as #42258 can still be useful defense in depth for stale client state.
