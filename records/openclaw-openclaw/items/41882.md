## Summary

Current main has not implemented this PR's intended Discord steer-mode bypass. The Discord inbound worker still unconditionally serializes same-session jobs before the downstream reply runtime can apply `steer` or `steer-backlog`, so the PR remains useful but needs rebase/adaptation to the current bundled Discord plugin and should address the session override concern.

## Best Possible Solution

Keep this PR open for maintainer review, but require a rebase to the current `extensions/discord` inbound worker. The best fix is a queue-mode-aware worker path that bypasses the outer per-session `KeyedAsyncQueue` for `steer` and `steer-backlog`, preserves serialization for non-steer modes, honors session-level queue overrides where possible, stays within the plugin SDK boundary, and adds focused regression tests for bypass and non-bypass behavior.
