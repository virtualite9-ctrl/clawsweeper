## Summary

Current main still has the exact unguarded `provider?.models?.some(...)` path that this PR targets, and the linked HTTP chat-completions issue remains unresolved by main. The PR is a focused implementation candidate with matching regression tests and no observed security-sensitive surface changes.

## Best Possible Solution

Keep this PR open for maintainer review and land the `Array.isArray(provider?.models)` guard with the included regression tests, or apply an equivalent main-branch fix that prevents non-array runtime provider model values from crashing provider policy lookup. Treat broader provider normalization as follow-up work, and close the linked #66744 only after the guard or equivalent fix is merged.
