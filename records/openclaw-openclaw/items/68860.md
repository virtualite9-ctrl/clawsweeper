## Summary

Keep open. Current main still drops embedded Pi `turn_start` and `turn_end` events, and this PR targets a real observability gap. The PR should not be closed as implemented because main does not contain the turn handler or tests; it also should not be merged as-is because the latest head has malformed TypeScript after partially applying the timestamp suggestion.

## Best Possible Solution

Keep this PR open and request a small repair: add scheduled `turn_start` and `turn_end` handling in `src/agents/pi-embedded-subscribe.handlers.ts`, forward both to `emitAgentEvent` and `params.onAgentEvent` with a stable payload contract such as `phase`, `startedAt`/`endedAt`, and `toolResultsCount`, and keep focused regression tests for both the per-run callback and global agent-events bus. No security-sensitive PR content needs removal.
