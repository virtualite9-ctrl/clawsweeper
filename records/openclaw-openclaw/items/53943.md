## Summary

Keep this issue open. Current main still does not support exact Slack `messageId` reads through the message tool: the Slack read dispatcher does not read or forward `messageId`, the runtime only forwards `threadId`, and `readSlackMessages` returns channel history or thread replies without filtering to the requested Slack timestamp. The prior ClawSweeper comment remains accurate against current main `68561a8c948b49e85ae012646de550a49d6c63a3`.

## Best Possible Solution

Keep this issue open and fix it in the bundled Slack plugin read path. Add `messageId` to the Slack `read` dispatch and runtime payload, extend `readSlackMessages` to return or filter the exact Slack timestamp for both `messageId + threadId` and messageId-only reads, and add focused dispatcher/runtime/helper tests plus docs updates. If Slack URL parsing feeds this path, cover normalization from `pMESSAGE_TS?thread_ts=THREAD_TS` into `messageId` and `threadId`.
