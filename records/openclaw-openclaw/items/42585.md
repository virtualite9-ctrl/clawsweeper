## Summary

Current main still does not implement this PR’s intended behavior. `resolveSessionAuthProfileOverride` keeps an existing automatic `provider:default` override as `current` unless the session is new, compaction advanced, the profile is missing, or the profile is in cooldown; it does not canonicalize a legacy OAuth default profile to a preferred OAuth alias. The PR remains a plausible focused auth-routing fix candidate, but it needs a rebase against the current split auth-profile modules and maintainer review of identity-matching semantics before it can land.

## Best Possible Solution

Keep this PR open. Rebase it onto current main, port the implementation to the current split auth-profile imports and mocked session override test harness, preserve the identity-mismatch and rotation-progress regression cases, align identity comparison with the existing OAuth identity helpers or document why this path differs, then run the targeted auth-profile test lane and changed gate before maintainer review or landing.
