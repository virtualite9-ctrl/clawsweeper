## Summary

Current main has important Gemini safeguards, including tagged reasoning mode, strict <final> enforcement, native thought separation, and Telegram suppression for exact Reasoning-prefixed blocks, but it still does not implement the broader plain-text/multilingual reasoning sanitizer requested in the discussion. The open candidate PR #60551 remains the appropriate path to review or replace rather than closing this umbrella report now.

## Best Possible Solution

Keep this open as the Gemini plain-text reasoning leakage tracker. Review and land #60551 or a replacement that adds a conservative outbound/message-tool sanitizer with regression tests for untagged planning prose, appended reasoning blocks, multilingual reasoning headers, tagged fragments, and normal prose/code pass-through.

## Risks / Open Questions

- A broad plain-text sanitizer can false-positive legitimate prose, quoted examples, code blocks, or multilingual answers unless it is tightly scoped and tested.
- Current Gemini strict-final-tag behavior may prevent leakage by dropping untagged text, but that is not the same as safely extracting the final answer from mixed plain reasoning plus answer text.
- Existing structured reasoning paths should not regress: native thought parts, isReasoning payload suppression, tagged <think>/<final> stripping, and intentional /reasoning visibility.
