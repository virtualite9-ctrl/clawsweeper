## Summary

Keep this PR open. Current main still does not implement the PR's central diagnostic behavior for selected image attachments that produce zero outputs: no-provider paths return before any warning, all-skipped provider attempts remain a skipped decision with only verbose logging, and the existing failed-decision warning does not include attempted provider/model details. The PR is also not ready to merge as-is because the provided diff has the same unreachable no-provider warning and all-skipped gap identified in review.

## Best Possible Solution

Keep this PR open for contributor or maintainer follow-up. The best implementation is to revise src/media-understanding/runner.ts to use logWarn, emit the no-provider diagnostic before the early return, add explicit warning behavior for all-skipped selected-image attempts, include sanitized attempted provider/model/reason context, and add targeted Vitest coverage for failed, no-provider, and all-skipped selected-image zero-output paths.
