## Summary

Keep this PR open. Current main has some related context-window stop-reason normalization, but it still does not implement the PR’s central behavior: a bounded overflow-recovery retry with prompt-hook context injection suppressed. The PR is also not merge-ready as-is because the provided diff mixes that runtime fix with unrelated extension peer dependency and lockfile churn, plus unresolved review concerns about retry gating and suppression scope.

## Best Possible Solution

Keep the PR open, but ask for a focused rework or have a maintainer apply the useful pieces separately: add a one-attempt prompt-hook-context suppression path in `src/agents/pi-embedded-runner`, preserve `compaction_failure` semantics, avoid retry-limit misclassification, add targeted tests for the retry and raw stop-reason user-facing normalization, and remove unrelated dependency/lockfile changes.

## Risks / Open Questions

- The PR should not be merged as-is while it bundles the runtime change with unrelated extension peer dependency and lockfile changes that affect package resolution.
- The suppress retry needs to be gated on remaining run-loop budget; the provided review comments note a possible `retry_limit` misclassification when the extra retry is scheduled on the last allowed loop pass.
- The retry should ideally be gated on prompt-hook context availability to avoid an extra deterministic failing model call when there are no prompt mutations to suppress.
- The suppression scope must be clarified or narrowed because skipping `resolvePromptBuildHookResult` also skips system-prompt overrides and legacy `before_agent_start` prompt-build behavior, not just prepend/append context.
