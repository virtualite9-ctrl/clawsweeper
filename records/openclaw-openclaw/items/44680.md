## Summary

Current main still creates schema v1 backup manifests without per-asset checksums and `backup verify` only performs structural archive validation. This PR is a focused implementation candidate for checksum integrity, not obsolete or implemented on main, and it should remain open for maintainer review rather than be closed by cleanup.

## Best Possible Solution

Keep this PR open as the active implementation candidate. If the checksum feature is desired, update the PR to preserve existing backup filtering behavior, settle empty-directory and symlink semantics, keep schema v1 compatibility, and validate with targeted backup create/verify tests before maintainer merge.

## Risks / Open Questions

- The PR still needs normal maintainer review before merge. In the PR head, the two-pass create path appears to omit the existing `buildExtensionsNodeModulesFilter` from the `tar.c` call, which could regress the current exclusion of plugin `node_modules` from state backups.
