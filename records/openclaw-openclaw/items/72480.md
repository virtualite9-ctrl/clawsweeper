## Summary

Current main still routes provider model requests through strict guarded fetch without the env-proxy auto-upgrade proposed by this PR. The PR remains a narrow implementation candidate with focused tests, and the provided diff does not introduce unrelated security-sensitive file changes.

## Best Possible Solution

Keep this PR open for maintainer review and CI. The best fix is to land this PR or an equivalent patch in `buildGuardedModelFetch` that applies trusted env-proxy mode only when no explicit provider dispatcher policy exists and `shouldUseEnvHttpProxyForUrl(url)` is true, while preserving strict behavior for explicit direct, TLS, env-proxy, and explicit-proxy transport policies and keeping the two regression tests.
