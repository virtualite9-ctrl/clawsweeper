## Summary

Keep open. Current main has WhatsApp self-chat and `fromMe` echo handling, but it does not implement the requested human-takeover behavior: a non-gateway outgoing/manual message is not used to pause future bot replies for that conversation, and there is no `humanTakeover`/pause-duration config surface.

## Best Possible Solution

Keep this issue open and implement the behavior in the bundled WhatsApp plugin. The likely fix belongs under `extensions/whatsapp`: add a typed `channels.whatsapp.humanTakeover` config/schema/docs/UI surface, detect non-gateway `fromMe` outgoing messages per account plus conversation, store a pause-until timestamp, and make the auto-reply path skip replies while the pause is active. Add focused tests for direct chats and any intended group behavior; only add a shared core seam if another channel needs the same takeover contract.
