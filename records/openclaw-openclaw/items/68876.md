## Summary

Current main already implements the memory-core dreaming corpus-noise fix: cron-run transcripts are identified from session-store keys and skipped, generated cron/heartbeat/silent-reply scaffolding is sanitized before corpus export, and regression coverage verifies the fresh session corpus excludes that noise while keeping meaningful content. The implementation is present in the v2026.4.24 release, so this PR is obsolete.

## Best Possible Solution

Close this PR as implemented. Keep the shipped session-store-backed cron-run classification and central session export sanitization on main; handle any remaining memory-dreaming quality work through the narrower open follow-ups such as #68449 or #71656.
