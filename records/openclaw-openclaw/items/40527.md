## Summary

Keep open. Current main still has the automatic remote skill bin probe race for macOS app-backed nodes: the gateway starts the probe during node connect before sending the connect response, the probe immediately emits a node invoke event, and the Swift client only starts its normal event listener after connect completes. Current main also has no co-located/loopback skip for the probe.

## Best Possible Solution

Keep this issue open. The best fix is to move the initial refreshRemoteNodeBins call until after the node connect response/listener is ready, or teach the Swift connect-response wait path to buffer/dispatch early node.invoke.request frames. Maintainers should separately decide whether same-host loopback macOS app nodes should skip remote bin probing when the gateway already has local shell access, then add regression coverage for both connect-time probing and any co-located skip policy.
