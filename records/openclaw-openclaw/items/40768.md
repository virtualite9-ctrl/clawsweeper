## Summary

Current main still performs Feishu group mention gating by comparing the event mention open_id to the startup-resolved botOpenId, and the startup/probe path only stores botOpenId/botName. That leaves the app-scoped open_id mismatch reported here unresolved. The provided GitHub context also shows #40782 is an open same-author implementation candidate that uses closing syntax for this issue, so this issue should remain open until that PR is reviewed/landed or replaced.

## Best Possible Solution

Keep this issue open and have maintainers review #40782 or request a revised Feishu-owned fix. The fix should resolve mention matching with a verified stable identity or cross-app open_id mapping, store the needed identity state per account, and add regression coverage around app-scoped open_id mismatch in the Feishu mention/monitor tests.
