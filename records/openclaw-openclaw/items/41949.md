## Summary

Keep this issue open. Current main has useful mitigations, but it still does not implement the browser-specific, task-aware context budgeting requested here: default AI snapshots remain broad, efficient snapshots are opt-in/config-driven, and generic tool-result/context guards act later rather than choosing compact page state at browser request time.

## Best Possible Solution

Implement a browser-owned context-budget policy: make agent-facing snapshots default to compact/interactive bounded output when appropriate, dynamically lower browser snapshot budgets as context pressure grows, support selector/ref/detail fetch-on-demand so full page state can stay outside the main model context, and add focused tests for browser-tool.actions, agent.snapshot.plan, snapshot routes, and repeated browser tool loops.
