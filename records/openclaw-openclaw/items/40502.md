## Summary

Keep this PR open. Current main still does not implement the requested low default temperature for tool-enabled or shell-like embedded agent runs, so the PR is not obsolete or implemented-on-main. The supplied PR diff remains a plausible implementation candidate, but it still needs maintainer review and likely a focused rebase/split because the provided head mixed the agent-temperature change with unrelated secrets/config, lockfile/package-surface, test-loosening, and changelog hunks, while review comments identify provider-thinking, cache-retention, and compaction edge cases.

## Best Possible Solution

Keep this PR open, but require a focused rebase before maintainer review. The best path is to limit the diff to the embedded runner temperature heuristic; pass tool-name context through a typed/non-stream-param seam for normal attempts and compaction; preserve explicit configured temperatures; respect Anthropic and other provider thinking/reasoning constraints; avoid cache-retention side effects; drop unrelated secrets/config/lockfile/changelog/test-loosening hunks; and add focused tests for plain chat, generic tools, shell-like tools, explicit temperature, thinking-enabled cases, cache retention, and compaction.
