## Summary

Keep this PR open. Current main does not implement the PR’s central change: task-completion internal event results are still sanitized but embedded unbounded into both runtime-context and plain prompt bodies. The PR is still potentially useful, but it needs maintainer review and likely narrowing because its latest file list also includes unrelated provider catalog test changes.

## Best Possible Solution

Keep the PR open and ask for a narrowed rebase on current main: implement the internal-event result bound in `src/agents/internal-events.ts`, add focused coverage for both runtime-context and plain prompt formatting, and remove or clearly justify the provider catalog test changes. The shipped implementation should preserve useful head/tail output with an explicit truncation marker and then run the targeted agent formatter/lint/tests for that surface.
