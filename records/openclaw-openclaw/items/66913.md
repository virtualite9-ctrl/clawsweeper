## Summary

Keep this PR open. Current main still truncates `memory_forget` candidate IDs to 8 characters while `memory_forget(memoryId)` still requires a full UUID, so the PR is not obsolete and directly targets a real remaining bug with a focused source change plus regression test.

## Best Possible Solution

Keep this PR open as the focused implementation candidate. Maintainers should review and either land the source change plus regression test, or revise the design to use a safer deletion handle or explicit confirmation while still making `memory_forget` candidate deletion unambiguous.
