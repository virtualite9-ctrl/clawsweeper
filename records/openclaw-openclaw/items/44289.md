## Summary

Keep open. Current main partially addresses the request by deriving the JSON credential matrix from secret target registry metadata, but the Markdown credential surface is still a checked-in bullet list guarded by parity tests rather than generated from the same source. The acceptance criterion that registry metadata updates should not require hand-editing multiple docs files is not met yet.

## Best Possible Solution

Keep this issue open. Extend the existing SecretRef credential matrix builder/generator so it updates both the JSON matrix and the Markdown supported/unsupported blocks, add a documented package script/check pair, and keep the existing parity test as the guard.
