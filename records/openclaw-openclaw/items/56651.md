## Summary

Keep open. Current main has partial adjacent mitigations, but it does not implement the requested gateway-level `announceTimeoutMs`, `connectionTimeoutMs`, or `pluginInitTimeoutMs` config surface, and the remaining channel connection/plugin-init portions are not fully covered by the related narrower items.

## Best Possible Solution

Keep this issue open and implement a deliberate gateway lifecycle timeout policy: add typed/schema/help/docs-backed gateway config for channel/provider connection and plugin initialization budgets, decide whether announce timeouts belong under `gateway` or agent/subagent defaults, wrap the relevant async startup/announce paths with structured timeout errors, and add focused config validation plus gateway/channel/plugin lifecycle tests. Link or split narrower workstreams for #51282, #59339, and #65415 if maintainers want separate tracking.
