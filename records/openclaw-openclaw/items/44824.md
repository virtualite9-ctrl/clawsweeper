## Summary

Keep open. Current main still routes `pnpm check:docs` through the Unix-only `git ls-files ... | xargs oxfmt` docs formatter path, and no `scripts/format-docs.mjs` or equivalent helper has landed. This PR addresses a real unresolved Windows/PowerShell tooling issue, but the submitted patch still has concrete review feedback around Windows npm `.cmd` shim resolution and silent child-process launch failures, so conservative cleanup should not close it as implemented or obsolete.

## Best Possible Solution

Keep this PR open for maintainer review or author amendment. The best path is a single canonical docs-format wrapper that removes `xargs` for `format:docs:check` and preferably `format:docs`, preserves the tracked docs file set, chunks long argument lists, invokes repo-local `oxfmt` in a Windows-safe way, reports `spawnSync` launch errors clearly, and then closes the duplicate candidate PRs only after one corrected implementation lands for #44293.
