## Summary

Keep open. Current main still routes `pnpm check:docs` through the Unix `git ls-files ... | xargs oxfmt` docs-format pipeline, so the PowerShell problem is not implemented on main. This PR targets a real open issue, but its submitted helper still has actionable review blockers around Windows npm `.cmd` shim execution and silent child-process launch failures, so it needs amendment or consolidation rather than cleanup closure.

## Best Possible Solution

Keep this PR open for changes or consolidate it with the canonical fix for #44293. The best implementation is one docs-format wrapper that removes `xargs` for both check and write paths, enumerates tracked docs with `git ls-files -z`, chunks arguments for Windows limits, invokes repo-local `oxfmt` through a Windows-safe launcher, reports `result.error`, and is validated in native PowerShell plus the existing Linux/macOS shell path.
