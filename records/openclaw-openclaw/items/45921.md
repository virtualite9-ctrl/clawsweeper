## Summary

Keep open. Current main does not implement this PR's Prompt Inspector layer: there is no Prompt Inspector client/config, webhook requests still dispatch directly after validation, and cron/external-content handling still uses local suspicious-pattern logging plus wrapping/sanitization. The PR is also a security-sensitive third-party network integration, so it needs maintainer product/security review rather than conservative cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review. The likely best path is either to review and harden the core security feature with focused tests/docs/privacy review, or split the core work into a generic gateway/pre-agent inspection hook and publish Prompt Inspector itself as an external plugin/ClawHub package.

## Risks / Open Questions

- The PR would send webhook/tool/external content to a third-party Prompt Inspector endpoint or arbitrary PMTINSP_BASE_URL, so privacy, data-retention, and SSRF/proxy policy need explicit maintainer review.
- The PR adds a new PMTINSP_API_KEY secret path and outbound network behavior in gateway/agent hot paths; secret redaction, env precedence, timeout behavior, and fail-open semantics need tests on the final branch.
- Closing as ClawHub would be premature unless core first exposes a gateway/pre-dispatch plugin seam that can block built-in hook requests with the intended HTTP response.
