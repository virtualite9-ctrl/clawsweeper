## Summary

Keep this PR open. Current main still does not implement `agent_to_agent_turn` or an equivalent A2A turn-level plugin seam: intermediate ping-pong replies are internal `runAgentStep` calls, and plugin-visible outbound delivery only happens at the final announce send. The PR is plausible core plugin API work because the needed turn number, speaker/listener, flow ID, and delivery target context live inside the core A2A orchestration path.

## Best Possible Solution

Keep the PR open for maintainer/design review. The best path is a first-class core A2A turn hook emitted from `src/agents/tools/sessions-send-tool.a2a.ts`, registered through the current typed plugin hook surface, documented in the public plugin hooks docs, and covered by focused tests for hook registration, turn emission, metadata, routing context, ordering, timeouts, and conversation-access policy.
