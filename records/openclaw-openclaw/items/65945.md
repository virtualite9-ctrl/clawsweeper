## Summary

Current main still does not implement opt-in auto-forwarding of Node `agent.request` image attachments to the outbound IM user. Attachments are normalized and parsed for agent/model input, but no preserved auto-forward flag or delivery-ready original media state reaches `AgentCommandOpts` or the outbound delivery path. The existing documented contract still requires the agent reply to include `MEDIA:`/media fields for outbound attachments.

## Best Possible Solution

Keep this issue open. The best implementation is an additive gateway protocol/node-event change that accepts an explicit auto-forward flag, preserves sanitized delivery-ready attachment references through `AgentCommandOpts`, and injects them into outbound delivery only when `deliver: true`, a concrete external route is resolved, media type/channel support is confirmed, and the agent reply does not already provide media. Add regression coverage for text-only replies, no-reply runs, agent-media override, unsupported channels, and size/type failures.
