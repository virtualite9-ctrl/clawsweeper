## Summary

Keep open. Current main has strong adjacent behavior for automatic channel config hot-reload and health-monitor restarts, but it does not expose the requested manual `openclaw channels reload <provider>` or `channels.reload`/`channel.reload` Gateway method, and the existing `channels.start` RPC is not a forced stop/start reload for a running channel.

## Best Possible Solution

Add an admin-scoped `channels.reload` Gateway method and `openclaw channels reload --channel <provider> [--account <id>]` CLI wrapper that reuses the existing channel manager stop/start lifecycle, applies the same active-work deferral policy used by config hot reload where appropriate, returns structured status, and updates `docs/cli/channels.md`, `docs/cli/gateway.md`, protocol schema/method scopes, and focused tests. If maintainers intentionally do not want a manual reload surface, document the supported alternatives: config hot reload, health-monitor recovery, `openclaw secrets reload`, and relink/login flows.
