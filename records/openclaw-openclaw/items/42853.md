## Risks / Open Questions

- The PR should not preserve or append arbitrary install-time `NODE_OPTIONS`; current main treats that variable as a blocked startup-control surface.
- The implementation should detect both Node heap flag spellings, `--max-old-space-size` and `--max_old_space_size`, before adding a heap flag.
- The proposed default of 85% of total system RAM is a product/runtime policy decision and may be too aggressive on shared or memory-constrained hosts.
- The branch needs a current-main refresh: wizard code moved to `src/wizard/setup.gateway-config.ts`, and the node install CLI caller still needs config threading.
- After refresh, maintainers should require focused tests for gateway service env, node service env, config schema/help/docs, and wizard blank-to-auto behavior.
