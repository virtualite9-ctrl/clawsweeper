## Summary

Keep open. Current main has partial write-side hardening for heartbeat system events, but it does not definitely implement this PR's intended gateway display-name stabilization. A main-session row with stale `origin.label: "heartbeat"` can still project that value as `displayName`, and the UI prefers that row display name over its `Main Session` fallback.

## Best Possible Solution

Keep this PR open as a plausible implementation candidate. Rebase it onto current main, move the main-session guard into `buildGatewaySessionRow` or a shared row-display helper, preserve explicit user-set `displayName`, use the existing main-key resolvers for custom and global modes, and add regression coverage for `agent:<id>:main`, configured main keys, global main, and stale `origin.label: "heartbeat"`.
