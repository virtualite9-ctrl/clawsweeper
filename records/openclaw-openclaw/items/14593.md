## Summary

Keep this issue open. Current main at ad57a6d61658f4e73b926138a3b010da5db4e225 still lacks an official Linuxbrew/Homebrew Docker setup path, the reported openai-whisper skill remains brew-only, and installSkill still returns `brew not installed` when no brew executable is found. The provided GitHub context also shows open implementation candidates with closing syntax, especially #42303 and #55940, so this issue should stay open until one is reviewed and merged or closed by maintainers.

## Best Possible Solution

Keep this issue open and review/land or close the linked implementation candidates. A focused fix should either add an official opt-in Linuxbrew path for Docker setup and Compose runtime, add Linux-native installer metadata for bundled brew-only skills such as `openai-whisper`, or change onboarding/status behavior so Docker users are not offered impossible brew installs without a supported setup path. Relevant surfaces are `Dockerfile`, `scripts/docker/setup.sh`, `docker-compose.yml`, `docs/install/docker.md`, bundled skill metadata, `src/agents/skills-install.ts`, and `src/commands/onboard-skills.ts`.
