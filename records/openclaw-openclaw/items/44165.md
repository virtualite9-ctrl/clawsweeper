## Summary

Keep open. Current main still has the exact behavior this PR is trying to change: the Channels view trusts snapshot `channelMeta` or `channelOrder` as the complete render order, so a custom-only snapshot still renders only the custom channel and does not append the built-in channel cards.

## Best Possible Solution

Keep this PR open for maintainer review or reimplementation. The desired fix belongs in the Control UI ordering path: merge snapshot-provided custom or loaded channels with the built-in channel card order, dedupe while preserving the snapshot order first, and add focused regression tests for custom-only `channelMeta`, mixed `channelOrder`, and no-snapshot fallback. A maintainer may want to source the built-in IDs from the current channel catalog/shared order where practical instead of expanding the existing UI hard-coded fallback.
