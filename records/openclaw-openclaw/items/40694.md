## Summary

Keep open. Current main still implements background idle/excess primary-tab sweeps plus lifecycle cleanup for subagent, cron, and session reset paths, but it does not close temporary primary-agent browser tabs/windows at ordinary task or turn completion.

## Best Possible Solution

Keep this issue open for a focused browser lifecycle change. Add task-scoped ephemeral versus preserved/user-owned metadata to tracked tabs, expose an explicit keep-open/debug preservation path, close ephemeral primary-agent tabs on successful task or turn completion, preserve existing-session/user-owned/labeled or pinned tabs, and keep the current idle/excess sweep plus subagent/cron/session-reset cleanup as fallback. Add tests for primary-agent completion cleanup, keep-open preservation, and existing-session safety.

## Risks / Open Questions

- A naive close-on-completion change could close tabs users intended to reuse across turns unless explicit preserved/user-owned/keep-open semantics are modeled.
- Existing-session and attach-only profiles can point at a user's running browser, so cleanup must avoid closing manually opened or explicitly preserved tabs.
