## Summary

Keep this issue open. Current main has fixed the runtime auto-enable surprise generically, but the requested BlueBubbles-specific docs are still missing: the BlueBubbles page only documents `channels.bluebubbles.enabled`, while plugin disablement, loopback/token gateway posture, and startup mutation behavior remain split across generic docs. The provided related items are broad third-party channel/plugin requests and do not supersede this bundled BlueBubbles documentation gap.

## Best Possible Solution

Keep this open and add a focused section to `docs/channels/bluebubbles.md`, near the configuration reference or security setup. It should explain `channels.bluebubbles.enabled=false` versus `plugins.entries.bluebubbles.enabled=false`, include a JSON5 local-safe example with `gateway.mode: "local"`, `gateway.bind: "loopback"`, `gateway.auth.mode: "token"`, `channels.bluebubbles.enabled: false`, and `plugins.entries.bluebubbles.enabled: false`, and briefly summarize startup mutations such as token persistence, `meta.lastTouchedVersion`, doctor migrations, and bundled plugin runtime-dependency repair. Link out to the existing plugin and gateway docs for the generic rules.
