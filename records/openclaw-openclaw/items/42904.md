## Summary

Keep this PR open. Current main fixes the generic `openclaw logs` / `logs.tail` rolling-log path and long-lived logger rollover, but it still does not implement the PR's stated `openclaw channels logs` fallback. The PR also bundles unrelated cron warning work that is already represented by open #42245, and its channel-log fallback needs narrowing before it would be safe to merge.

## Best Possible Solution

Keep the PR open for maintainer rework: split or remove the cron warning in favor of #42245, then replace the logging change with a focused `openclaw channels logs` patch that reuses a shared exact rolling-log resolver and adds targeted regression tests for missing current-day files, existing current-day precedence, custom `logging.file`, and channel filtering.

## Risks / Open Questions

- Closing now would drop the remaining channel-specific reader behavior, because current main still has no `channels logs` regression coverage for missing current-day files, current-day precedence, custom `logging.file`, or channel filtering across a fallback file.
