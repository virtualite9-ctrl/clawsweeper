## Summary

Keep this PR open. Current main still has the nested-parentheses exec approval parsing bug: the parser uses flat `[^)]*` metadata captures, while the gateway emits `approval-timeout (allowlist-miss)` denial metadata and followup routing depends on correctly classifying that text as denied. The PR remains a relevant implementation candidate, but it still needs maintainer review for the spoofable approval-status parsing concern and the unrelated generated plugin SDK baseline file shown in the live PR diff.

## Best Possible Solution

Keep this PR open for normal maintainer review. The best path is to land the balanced-parentheses parser and the nested denied/finished regression tests, remove the unrelated generated baseline diff, and either scope approval-wrapper parsing to trusted exec approval lifecycle events or add a follow-up hardening change for structured/non-spoofable approval resolution. Related #72148 addressed an adjacent resume-fallback prefix symptom, but it is closed unmerged and does not supersede this parser fix.
