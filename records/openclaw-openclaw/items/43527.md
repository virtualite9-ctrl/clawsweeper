## Summary

Keep this issue open. Current main still has the misleading `no bootstrap files` wording on the normal/deep status command path, while the underlying count only checks whether `BOOTSTRAP.md` exists. The provided GitHub context also shows open PR #43614 still links this issue with closing syntax, so the maintainer path is to review, land, or supersede that PR rather than closing the issue now.

## Best Possible Solution

Keep this issue open and have a maintainer review, refresh, land, or supersede PR #43614. The narrow fix should update `buildStatusAgentsValue` so `bootstrapPendingCount` is described as active bootstrapping rather than general bootstrap-file injection, and add a regression test for mature workspaces where `BOOTSTRAP.md` is absent while regular workspace bootstrap files are still injected.
