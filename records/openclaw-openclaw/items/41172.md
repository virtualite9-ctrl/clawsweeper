## Summary

Keep open. Current main still does not implement the PR's central change: `formatAssistantErrorText` only recognizes the two existing unknown-tool error shapes and does not parse Groq's `attempted to call tool ... which was not in request.tools` validation message. The PR is still relevant, but its regression coverage should prove the sandbox-policy rewrite path, not just that the raw provider error contains the hallucinated tool name.

## Best Possible Solution

Keep this PR open. Rebase or recreate the small formatter change on current main, add a regression test that fails on current main and passes only when the Groq tool name drives `formatSandboxToolPolicyBlockedMessage`, and consider whether the duplicated unknown-tool extraction in `src/agents/tool-loop-detection.ts` should be aligned or factored into a shared helper.
