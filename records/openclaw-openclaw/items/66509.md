## Summary

Keep this issue open. Current main still buffers QQBot `kind: "tool"` text, clears the fallback when a later block/final reply arrives, and has no QQBot preview/tool-progress path that honors `streaming.mode: "partial"`.

## Best Possible Solution

Implement this in the QQBot plugin boundary. `extensions/qqbot/src/engine/gateway/outbound-dispatch.ts` should deliver useful `kind: "tool"` progress text immediately or through a QQBot preview/edit path when preview streaming is active, dedupe or clear it correctly when the final reply arrives, and keep current tool media fallback semantics. Add a focused regression test in `extensions/qqbot/src/engine/gateway/outbound-dispatch.test.ts`, then document any new QQBot streaming option in `docs/channels/qqbot.md` and the streaming concept page.
