## Summary

Current main still has the original Feishu Bitable cleanup behavior: it only deletes placeholder rows with missing or zero fields. This PR targets a real remaining gap, adds focused regression coverage, and does not look like conservative cleanup material.

## Best Possible Solution

Keep this PR open as a focused bundled Feishu plugin bug fix. The best maintainer path is to review and possibly harden the empty-value predicate, keep the regression test, run the targeted Feishu Bitable test lane, and merge if CI confirms the behavior.
