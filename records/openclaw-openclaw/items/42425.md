## Summary

Keep this PR open. Current main still does not implement the PR’s central behavior: Gateway startup loads internal workspace hooks from only the default workspace, while the PR proposes loading workspace-local hooks for every configured agent workspace with scoping and failure isolation.

## Best Possible Solution

Keep this PR open as a useful but stale fix candidate. The best path is a rebase or replacement patch against current main that integrates multi-workspace hook loading in `src/gateway/server-startup-post-attach.ts`, uses `listAgentWorkspaceDirs`, preserves explicit workspace-hook opt-in and non-override policy, scopes workspace-local handlers to the matching event/session/agent workspace, isolates per-workspace failures, and adds focused regression tests for non-default workspace loading, startup events, precedence, and security boundaries.
