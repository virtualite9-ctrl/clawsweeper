## Summary

Keep open. Current main still does not implement this PR's intended direct-compaction behavior: the direct helper builds metrics from session.messages but runBeforeCompactionHooks sends only counts to before_compaction. The hook event type already allows messages and a different compaction-start subscriber path does pass messages, so the feature is coherent, but this PR still needs an updated implementation and stronger tests before it can land.

## Best Possible Solution

Keep this PR open and ask for an update: thread a pre-compaction message snapshot through the direct compactEmbeddedPiSessionDirect/runBeforeCompactionHooks path, assert exact pre-compaction contents and the empty-transcript case in compact.hooks tests, and explicitly document or rule out hook mutation semantics before running the targeted compaction hook tests and changed gate.
