## Summary

Keep this PR open. Current main still does not infer a missing auth profile provider from the `<provider>:<name>` profile id before schema validation, so the PR remains a focused implementation candidate rather than obsolete cleanup.

## Best Possible Solution

Keep this PR open and have a maintainer rebase it or apply the same narrow normalization on current main: infer a missing `auth.profiles[*].provider` from the profile id before legacy checks and schema validation, preserve explicit providers, and keep focused regression coverage in `src/config/config.schema-regressions.test.ts`. After that, run the changed config validation/test lane.
