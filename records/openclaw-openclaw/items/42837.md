## Summary

Keep open. Current main still has the Feishu dynamic-agent account-scoping bug described in this issue: Feishu DM/comment ingress resolves routes with the runtime account id, but dynamic-agent creation neither accepts nor writes `match.accountId`, so named-account traffic can create an agent plus an account-less binding and then continue falling through to `agent:main`. The related fix PR #42886 was closed unmerged, and I found no equivalent implementation on main.

## Best Possible Solution

Keep this issue open. Refresh the #42886 approach on current main: pass the normalized Feishu runtime `account.accountId` into `maybeCreateDynamicAgent`, include `match.accountId` when detecting existing dynamic bindings and when writing both missing-binding and new-agent bindings, update both message and comment flows, preserve account-less default-account compatibility, and add focused regression tests for named-account creation, missing-binding repair, and default-account routing.
