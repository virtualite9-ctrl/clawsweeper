## Summary

Keep this PR open. Current main still does not emit the requested startup dangerous-config warning for an explicit `agents.defaults.sandbox.mode="off"`; the only existing sandbox-off audit check is narrower and requires Docker sandbox settings to also be configured.

## Best Possible Solution

Keep the PR open and ask for a focused implementation: add a startup dangerous-config flag for the explicit raw config value `cfg.agents?.defaults?.sandbox?.mode === "off"`, prove absent sandbox config does not warn, add targeted tests in `src/security/dangerous-config-flags.test.ts`, and update the security dangerous-flags docs if this becomes part of the public inventory. If gateway auth startup warnings are retained, align them with the existing audit semantics or split them into a separate PR.

- Merging the PR as-is would add three startup warnings while the title/body only describe the sandbox warning; the gateway auth additions should either be split out or explicitly reviewed as part of scope.
