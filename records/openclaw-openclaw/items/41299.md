## Summary

Keep this PR open. Current main still has the ambiguity the PR is trying to fix: inbound metadata and user content are joined with blank lines only, and the sanitizer/test/helper surfaces do not have separator support. The PR is still a plausible focused bug-fix candidate, but it needs rebase and review against current prompt composition, sanitizer coverage, and the remaining bot-review concerns before merge.

## Best Possible Solution

Keep this PR open as a focused bug-fix candidate. The best path is to rebase it onto current main, adapt the separator insertion to the current `runPreparedReply` body flow, make separator stripping remove only the synthetic marker adjacent to stripped metadata, align prompt-composition helpers and fixtures, cover all generated inbound metadata block types, and run the changed OpenClaw gates before merge.
