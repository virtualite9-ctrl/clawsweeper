## Summary

Keep open. Current main still has the Feishu media failure path that can convert an upload/read failure into a text fallback result, which the CLI formats as a successful send. The Feishu config schema/docs also do not expose `channels.feishu.mediaLocalRoots`, so the original config expectation is not implemented or clearly replaced with channel-specific guidance.

## Best Possible Solution

Keep the issue open and fix the Feishu direct outbound path. The narrow product-safe fix is to detect local media allowlist failures from `sendMediaFeishu`, propagate them as CLI failures with clear allowed-root/staging guidance, and add a regression test in `extensions/feishu/src/outbound.test.ts`. If `channels.feishu.mediaLocalRoots` is intended to be supported, add it to the Feishu schema/docs and merge it into outbound media access; otherwise document the supported OpenClaw staging roots and reject the stale config key clearly.
