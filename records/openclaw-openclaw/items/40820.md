## Summary

Keep open. Current main at a25366038503adcc31a5fa64b3d2f295bd5dbfd2 still does not implement the Telegram /settings inline-keyboard control panel. The shared native-command registry has no settings command, the Telegram callback path has no cfg_* settings parser or handler, and the current Telegram plugin tree has no settings buttons/panel/auth modules. The PR remains a plausible implementation candidate, especially given the review discussion and author follow-up around callback authorization hardening, but it needs porting into the current extensions/telegram layout and another security-focused review.

## Best Possible Solution

Keep this PR open as an implementation candidate. The best maintainer path is to port the feature into extensions/telegram, implement the settings panel/buttons/auth against the current native command and callback structure, avoid broad core registry changes unless a generic provider-specific native-command seam is intentionally accepted, preserve fresh-config and configWrites guards, and add focused tests for command rendering, cfg_* callbacks, authorization parity, and config writes.
