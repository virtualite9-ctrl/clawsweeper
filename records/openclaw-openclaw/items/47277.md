## Summary

Keep this PR open. Current main does not implement user-specific memory isolation for memory-core, and the PR is a focused implementation candidate but still needs maintainer/security review and rework before it could land.

## Best Possible Solution

Do not close this PR as obsolete. The best path is to keep it open for maintainer review, narrow it to the memory-core and memory-host-sdk ownership boundary, and require a revised implementation with regression tests for builtin and QMD backends, session-key and requesterSenderId identity derivation, unsafe path/userId inputs, disabled-isolation compatibility, config schema/docs, and watcher/cache lifecycle behavior.
