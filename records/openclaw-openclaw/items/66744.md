## Summary

Keep open. Current main still has the unguarded `provider?.models?.some(...)` call that can throw the reported TypeError, and live PR #66761 is open and unmerged as the linked implementation candidate for the narrow `Array.isArray(provider?.models)` guard plus regression tests.

## Best Possible Solution

Keep this issue open and review PR #66761. The best current code path is a narrow guard in `src/agents/models-config.providers.policy.lookup.ts` using `Array.isArray(provider?.models)` before calling `.some(...)`, with focused regression tests in `src/agents/models-config.providers.policy.lookup.test.ts`; after that PR or an equivalent fix lands, close as implemented.
