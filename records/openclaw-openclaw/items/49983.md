## Summary

Keep open. Current main still sends Feishu post replies by embedding `<at user_id="...">name</at>` text inside one `md` element, so this PR's native post `at` element fix is not implemented on main. The PR is an active, merge-ready implementation candidate rather than obsolete cleanup.

## Best Possible Solution

Keep the PR open for maintainer review and landing. The best implementation path is the PR's scoped Feishu plugin change: preserve model-generated `<at ...>` text literally, pass actual mention targets separately into buildFeishuPostMessagePayload, emit native post `at` elements with display-name fallbacks, and keep regression coverage for no mentions, single/multiple mentions, and literal body at-tags.
