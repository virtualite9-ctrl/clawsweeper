## Summary

Keep this issue open. Current main still computes the preflight compaction and memoryFlush thresholds from `reserveTokensFloor` instead of the effective configured `reserveTokens`, and #69379 is an open unmerged implementation candidate that uses closing syntax for this issue.

## Best Possible Solution

Keep this issue open and review or land #69379, or an equivalent core fix. The fix should compute one effective compaction reserve from configured `agents.defaults.compaction.reserveTokens` plus `reserveTokensFloor` and use it consistently in both preflight compaction and memoryFlush gates, with regression coverage for `reserveTokens > reserveTokensFloor` and the small-window `threshold <= 0` behavior.
