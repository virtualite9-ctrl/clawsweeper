## Summary

Keep #62310 open. Current main still lacks the PR's minimum-interval enforcement for non-interval heartbeat wakes and lacks the overdue timer/drift branch. The PR also has an unresolved review concern: throttling every non-interval wake could break explicit wake-now semantics, so it needs maintainer review or a narrower implementation rather than cleanup closure.

## Best Possible Solution

Keep the PR open for maintainer/product review. The best path is to revise the heartbeat runner so passive activity-driven reasons are rate-limited without delaying explicit wake-now requests, then add regression tests for fan-out non-interval wakes, targeted wakes, overdue timer recovery, and `wakeMode: "now"` semantics.
