## Summary

Keep this PR open. Current main still unconditionally sends the default Anthropic beta header on the API-key Anthropic Messages path, including custom baseUrl endpoints, while this PR is a focused candidate fix with regression coverage and active author follow-up.

## Best Possible Solution

Keep this PR open for normal maintainer review. The best path is to land or revise a focused transport fix that gates implicit Anthropic beta headers to direct Anthropic API-key endpoints, preserves direct Anthropic behavior, covers custom provider/baseUrl and hostname-poisoning cases, and explicitly resolves the OAuth/custom-baseUrl policy before merge. Run the targeted transport test and the appropriate changed gate before landing.
