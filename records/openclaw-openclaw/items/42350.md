## Summary

Keep this PR open. Current main still does not implement native iOS/macOS Adaptive Card parsing or inline SwiftUI rendering, and the work is paired with the same contributor's open Android PR #42304. The external Adaptive Cards plugin is useful prior work, but current plugin APIs do not expose a native Swift chat bubble renderer extension point that would make this PR obsolete as ClawHub-only work.

## Best Possible Solution

Keep the PR open for maintainer product and security review. If native iOS/macOS Adaptive Card rendering is wanted in core, rebase it on current main, fix the parser/layout/fallback/action-data review items, add focused Swift tests or app smoke coverage, explicitly gate or document remote image and URL behavior, and coordinate the platform decision with open Android companion PR #42304. If maintainers decide this should stay external, close it only after an explicit product decision and a clear plugin/API path for native client rendering.
