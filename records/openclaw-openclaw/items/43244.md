## Summary

Keep open. Current main still does not protect the full gateway config read/check/compute/write window: the handlers validate baseHash against the snapshot they already read, compute a replacement config from that snapshot, and commit the stale computed payload without passing the caller's baseHash into the lower-level optimistic writer. The submitted PR remains relevant, but it needs a current-main rebase and review fixes before landing.

## Best Possible Solution

Keep this PR open as a still-relevant implementation candidate. Rebase or reimplement the fix on current main so config.set, config.patch, and config.apply either serialize the full read/base-hash-check/validate/write sequence or pass the caller baseHash into a fresh lower-level mutation before writing; preserve SecretRef and shared-auth behavior; map command-lane rejection errors to RPC failures; and add stale concurrent-write tests for all three methods.
