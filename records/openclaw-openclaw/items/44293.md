## Summary

Current main still routes `pnpm check:docs` through the Unix-only `xargs` docs-format pipeline. The issue also has open implementation candidates (#44824 and #49913) that explicitly reference this issue with closing syntax, so it should remain open until one PR lands or maintainers choose an equivalent fix.

## Best Possible Solution

Keep this issue open and review or land one canonical cross-platform docs-format implementation, preferably a small Node script that enumerates tracked docs files with `git ls-files -z`, invokes `oxfmt` without shell pipelines, and chunks arguments for Windows command-line limits. After that merges, close this issue as implemented and resolve duplicate PRs.
