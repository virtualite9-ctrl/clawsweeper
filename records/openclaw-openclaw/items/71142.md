## Summary

Keep this issue open. Current main has partially moved the server-side parser away from the old 5 MB default by using `agents.defaults.mediaMaxMb` with a 20 MB fallback, but the requested Control UI upload policy is still not implemented: proposed `gateway.controlUi` upload keys are absent and would be rejected by the strict schema, image uploads are still stopped by a separate 6 MB hard cap, and the browser upload flow still has no configured prevalidation limit. The active contributor comments are useful implementation planning, and #44814 is relevant prior art, but it is open, targets an older/different `gateway.webui`/`agent.request` shape, and does not make this issue obsolete.

## Best Possible Solution

Keep this open for a default-preserving upload policy. The likely maintainer path is to add a supported config field under `gateway.controlUi` or a shared `chat.send` attachment policy, validate it in the strict config schema and generated metadata, thread the resolved limit through `chat.send`, reconcile it with `MAX_IMAGE_BYTES` and `tools.media.image.maxBytes`, expose the effective limit in the Control UI bootstrap config for browser-side prevalidation, document the WebSocket/base64 practical ceiling on the public Control UI docs, and add focused tests for default behavior, raised-limit acceptance, image-limit behavior, oversized rejection, and client prevalidation.

## Risks / Open Questions

- Current main has a partial shared parser knob (`agents.defaults.mediaMaxMb`), so maintainers should decide whether the final public contract should be Control UI-specific, shared across `chat.send` clients, or aligned with `tools.media.*` per-kind limits.
- Raising image and document limits needs an explicit WebSocket/base64 and memory-pressure ceiling; a server-only parser increase would still leave poor browser UX without prevalidation.
- The open #44814 PR is related prior art, but it is not a safe duplicate close for this issue because it is unmerged and targets older naming/surfaces that do not cover the current Control UI client, schema, image cap, and docs path.
