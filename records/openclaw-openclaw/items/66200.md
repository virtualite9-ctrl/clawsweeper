## Summary

Keep this PR open. Current main still lacks effective BlueBubbles inbound external-content wrapping, but the submitted patch is not merge-ready because it wraps only the legacy envelope `Body` while the reply pipeline uses raw `BodyForAgent`/`BodyStripped` for the model-facing prompt. The existing review discussion correctly identified both the empty-body guard issue and the larger model-visible path gap.

## Best Possible Solution

Keep this PR open and request a focused revision or replacement. The fix should harden the actual BlueBubbles model-facing path, likely by setting an appropriate wrapped `BodyForAgent` or `UntrustedContext`, while preserving raw command/parsing fields. It should include the Discord-style non-empty guard and targeted BlueBubbles tests for the prompt-bound context.

## Risks / Open Questions

- Merging the patch as submitted would create a false sense of BlueBubbles prompt-injection hardening because raw inbound text can still reach the model through `BodyForAgent` and `BodyStripped`.
- The unconditional wrap in the PR can change attachment-only or empty-body behavior by producing an untrusted-content header with no actual message body, matching the existing Greptile/Omar review concern.
- A correct revision must preserve raw `RawBody`, `CommandBody`, and `BodyForCommands` for command parsing, reset handling, dedupe, and metadata while hardening only the agent-facing text or structured untrusted context path.
- The PR currently lacks focused BlueBubbles tests proving that inbound text reaches the agent with the intended untrusted-content boundary.
