## Summary

Current main still only partially satisfies the requested recovery guidance. Direct gateway startup invalid-config errors now suggest `openclaw doctor --fix`, but the thrown startup error still omits the requested `openclaw.json.bak` fallback. Daemon start/restart preflight still uses generic `openclaw doctor` wording. The related implementation PRs in the provided context are closed and unmerged, so there is no merged fix or open closing PR to wait on.

## Best Possible Solution

Keep this issue open and make a narrow user-facing recovery-message fix. Centralize a gateway invalid-config recovery hint and use it from direct gateway startup validation plus daemon start/restart preflight. The hint should mention `openclaw doctor --fix` and an explicit `openclaw.json.bak` fallback where appropriate, while preserving existing last-known-good recovery behavior. Extend `server-startup-config.recovery.test.ts` and `lifecycle-core.config-guard.test.ts` to assert both recovery steps, and add a concise changelog entry for the changed operator-facing text.
