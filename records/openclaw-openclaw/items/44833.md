## Summary

Keep open. Current main still appears to skip standalone captionless Telegram group photos when group mention gating is enabled and the only configured activation is text-based mention patterns such as `.*`. The related audio preflight work does not cover image-only messages.

## Best Possible Solution

Keep this issue open and implement a focused Telegram regression fix in the Telegram plugin: captionless photo/image media in a group that is otherwise configured to respond should reach the same inbound media-understanding path as captioned photos, without weakening normal group mention/reply policy. Add a regression test for a standalone captionless group photo under the reporter’s `mentionPatterns: [".*"]` style configuration and confirm the media path is attached to the agent context.

## Risks / Open Questions

- A fix should preserve Telegram group mention policy and privacy-mode expectations; blindly treating every media-only group message as a mention could create unsolicited replies.
- Changing wildcard mention matching to accept empty strings would affect more than Telegram images, so the safer path is likely a Telegram media-specific activation rule or explicit config behavior.
