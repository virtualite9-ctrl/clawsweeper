## Risks / Open Questions

- The PR exposes broader cross-session usage and cost data through chat/Telegram controls; maintainers should verify authorization and callback handling before merge.
- The PR patch removes current transcript `mtimeMs` range prefiltering in `loadCostUsageSummary()` and `discoverAllSessions()`, which can regress performance and affect limited session lists on large histories.
- The PR patch keys model aggregation by model name only, while current gateway usage aggregation uses provider-plus-model keys; same-named models from different providers can be merged incorrectly if not fixed before merge.
