## Summary

Keep this PR open. Current main still does not implement cursor-based `chat.history` pagination, web UI load-older state/rendering, or the partial `NO_REPLY` suppression described by this PR. The related pagination PR #56383 is open and unmerged, so it is not a shipped replacement. The latest provided PR head also still needs maintainer review because the gateway handler appears to compute a `before` window but project the full raw history, which can return the latest page again instead of the requested older page.

## Best Possible Solution

Keep this PR open for maintainer review. Fix the gateway handler so `before` selects the raw `[start, end)` page before display projection, or carry raw indices through sanitization/projection so `cursor` reflects the first returned raw message. Add gateway integration tests for normal pagination, sanitized silent/commentary drops, and byte-capped leading trims; then coordinate with #56383 before choosing which pagination implementation to land. The partial `NO_REPLY` fix can land with those tests or be split if maintainers want the stream-fallback fix reviewed separately.
