## Summary

Keep this PR open. Current main does not implement the requested `messages.unscheduledReminderNote` opt-out: the reply runner still appends the unscheduled-reminder note based only on reminder-commitment detection, successful cron additions, and same-session cron coverage, and the config schema/labels do not expose the proposed key. The PR is a focused implementation candidate rather than obsolete or superseded.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to rebase it onto current main, add the `messages.unscheduledReminderNote` schema/label/runtime gate, preserve the cron-store short-circuit when the flag is false, and add either an extracted-helper test or targeted integration smoke so the actual gating path is covered. Update generated config/help/docs if the config contract requires it, then run the appropriate changed gate.
