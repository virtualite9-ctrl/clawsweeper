## Summary

Keep this PR open. Current main still has the Telegram model-picker no-override fallback bug: routed callbacks build the provider/model list for the selected agent, but `resolveTelegramSessionState()` still falls back to `agents.defaults.model` instead of the routed agent's `agents.list[].model`, so the current-model checkmark can be wrong before a session override exists.

## Best Possible Solution

Keep this PR open for maintainer review and land this patch or an equivalent one. The best implementation is to compute the routed agent default in `resolveTelegramSessionState()`, reuse it for stored-override parsing and the final no-override fallback, keep the regression test for an agent default that differs from the global default, and validate the Telegram extension before merge.
