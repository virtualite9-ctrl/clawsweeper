## Summary

Keep open. Current main still injects the same blanket group-chat instruction forbidding same-group message-tool sends, while the existing media/tool routing path supports attachment sends with auto-thread resolution. The PR remains a plausible narrow fix, but it needs a rebase because current main now has tests and prompt-hardening behavior that the older PR tests would conflict with.

## Best Possible Solution

Keep this PR open and ask for a rebase onto current main. The maintainer path is to update buildGroupChatContext so normal text replies remain direct while files/images/attachments may use the message tool targeting the same group, update the existing groups.test.ts assertion and group-intro prompt fixture, preserve untrusted handling for group subject/member metadata, and then run the targeted auto-reply tests plus the changed gate before landing.
