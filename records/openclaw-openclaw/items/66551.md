## Summary

Keep this PR open. Current main still maps HTML/XML/CSS MIME types, but the host-read media guard only allows validated CSV/Markdown text fallbacks, so the PR's central behavior is not implemented on main. The latest PR context shows a focused patch that moved away from the earlier unconditional fallback allow path and now gates HTML/XML/CSS/XML on sniff compatibility plus text validation.

## Best Possible Solution

Keep this PR open as the active implementation candidate. The best path is to review the narrowed `src/media/web-media.ts` policy change, preserve the renamed-binary rejection tests and outbound `sendAttachment` HTML coverage, resolve any current CI failures, and then land if maintainers accept extending the validated-text host-read policy beyond CSV/Markdown.
