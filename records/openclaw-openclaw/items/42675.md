## Summary

Keep this PR open. Current main still allows Discord startup to continue with no usable bot identity, and the linked security bug remains unfixed. The submitted patch is stale and incomplete, but it is still a focused implementation candidate rather than obsolete work.

## Best Possible Solution

Keep this PR open and ask for a current-main rewrite in extensions/discord/src/monitor/provider.startup.ts so Discord startup fails when fetchUser("@me") rejects or returns no normalized bot id. Add focused regression tests for rejection and invalid identity responses, then consider a defense-in-depth preflight cleanup for the remaining botId-dependent mention gate behavior tracked by #42219.
