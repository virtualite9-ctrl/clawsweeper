## Summary

Keep this PR open. Current main still only parses `IDEProvisioningTeams` from Xcode preferences and does not implement `IDEProvisioningTeamByIdentifier`, so the PR is not obsolete or implemented on main. The related merged #22773 added managed provisioning-profile fallback and guidance, but that is a different fallback path. The latest PR head is a focused two-file script/test change with no unexplained security-sensitive or supply-chain changes, though the regression test still appears to bypass the new inline Python parser by using a fake `IOS_PYTHON_BIN`.

## Best Possible Solution

Keep the PR open. The best maintainer path is to review the focused script change, keep the preserved `plutil` fallback, and ask for a regression test that runs the real inline Python parser against mocked `defaults export` plist data containing `IDEProvisioningTeamByIdentifier`; after that, land it if maintainers agree local iOS signing should support this Xcode preferences layout.
