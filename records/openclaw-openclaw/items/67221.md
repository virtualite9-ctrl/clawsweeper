## Summary

This PR is superseded by the shipped Google Chat auth-diagnostics path from #71145. Current main keeps Google Chat add-on appPrincipal binding on the JWT sub/numeric principal, warns when operators configure an email-shaped appPrincipal, and carries Google Chat audience fields through channel-specific status snapshots instead of broadening the generic safe snapshot contract.

## Best Possible Solution

Close this PR as superseded. Keep the shipped numeric sub/OAuth client ID appPrincipal contract from current main, keep the #71145 diagnostics, and handle any remaining status-only display gap in a focused Google Chat status PR that does not change webhook auth semantics.
