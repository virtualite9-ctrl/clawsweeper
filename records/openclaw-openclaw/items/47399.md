## Summary

Keep open. Current main still lacks the PR’s intended Control UI streaming dedupe behavior: chat deltas are stored as the latest text snapshot, tool starts commit that full snapshot into stream segments, and the renderer displays those segments plus the live stream separately. The PR is relevant, but not merge-ready because the provided PR context shows an unrelated binary workspace artifact and a review finding about whitespace-only deltas being dropped.

## Best Possible Solution

Keep this PR open until it is either corrected or explicitly superseded. The best path is a narrow current-main fix that tracks committed stream length, commits only newly streamed suffixes at tool boundaries, renders only the live suffix after committed segments, preserves whitespace-only separators, removes the unrelated PNG artifact, and adds focused regression coverage for cumulative text interrupted by one or more tool calls. If #47377 becomes the chosen canonical fix, close this PR only after that PR or an equivalent main-branch implementation covers both segment storage and live-stream rendering.
