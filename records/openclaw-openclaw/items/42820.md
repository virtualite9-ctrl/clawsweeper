## Summary

Keep open. Current main still exposes poll creation fields on the shared `message` tool schema and still rejects `action: "send"` before Feishu/Telegram dispatch when non-empty or true poll-looking fields are present. The zero-default mitigation is present, but it does not cover the reported model auto-fill case. Open PR #46455 remains an unmerged implementation candidate that explicitly targets this issue, so this issue should stay open until that work is reviewed, landed, or replaced.

## Best Possible Solution

Keep this issue open and review, land, or replace PR #46455. The durable fix should either sanitize accidental poll creation params before the send-path guard or narrow the LLM-facing schema so poll creation fields are only visible for poll-capable actions/channels, while preserving `action="poll"` validation and adding regression coverage for Feishu and Telegram send-with-attachment calls that include non-empty auto-populated poll fields.
