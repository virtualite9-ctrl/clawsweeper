## Summary

Keep this PR open. Current main still does not implement the PR’s central behavior: runtime config still rewrites a configured session.mainKey to "main", core routing still builds main session keys with DEFAULT_MAIN_KEY, and current plugin/outbound routing paths still omit the configured key. The PR remains a relevant implementation candidate for the closed linked feature request, but it needs to be rebased/ported to current plugin paths and extended to cover the remaining delivery-guard concern before it is merge-ready.

## Best Possible Solution

Keep this PR open as an implementation candidate. Rebase it onto current main, port the Telegram/WhatsApp hunks to the current extensions/ paths, preserve configured session.mainKey through runtime defaults and every route/session-key builder, update delivery guards to recognize configured custom main keys and legacy aliases, align docs/schema help, and add focused config/routing/plugin regression coverage before maintainer review or merge.
