## Summary

Keep open. Current main still returns the generic `$PROFILE` placeholder and vague PowerShell reload guidance during onboarding shell-completion setup. The linked fix PR #47333 was closed unmerged, so it is useful prior discussion but not a shipped implementation.

## Best Possible Solution

Keep this issue open. Implement the fix in the core CLI/wizard completion path by resolving a concrete PowerShell profile path, using an actionable dot-source reload hint with proper quoting, and adding Windows/PowerShell-focused tests for onboarding completion output. While doing that, review `src/cli/completion-runtime.ts` and `src/commands/doctor-completion.ts` so PowerShell completion install/status behavior does not remain inconsistent with the onboarding message.
