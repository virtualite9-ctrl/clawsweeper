## Summary

Current main still does not restore the Control UI abortable chat run identity after refresh/reconnect during an active stream. The UI controls remain keyed to `chatRunId`, but reconnect and history loading clear or omit that state, and incoming deltas do not rehydrate it.

## Best Possible Solution

Keep this issue open. Add an authoritative active Control UI chat-run hydration path on page load/reconnect, likely from hello/bootstrap or `chat.history`, returning the active run id plus any buffered stream state for the visible session. The UI should set `chatRunId`, `chatStream`, and `chatStreamStartedAt` before rendering controls, and tests should cover refresh during an active stream plus terminal-event cleanup.
