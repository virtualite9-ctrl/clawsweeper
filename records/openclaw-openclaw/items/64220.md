## Summary

Current main still leaves the reported Control UI stale compaction-status bug actionable. The code has partial session-switch cleanup, but active/retrying compaction state still has no stale timeout, the renderer shows it indefinitely, and terminal chat/reconnect/reset paths still clear run or stream state without consistently clearing compactionStatus/fallbackStatus.

## Best Possible Solution

Keep this issue open and fix the Control UI state layer. Add a run/session-scoped stale timeout or watchdog for active/retrying compaction, centralize timer-aware cleanup for compactionStatus and fallbackStatus, call that cleanup from terminal chat events, reconnect/disconnect recovery, overview/chat session changes, clear-history/reset, and send-failure paths, then add focused regression tests for missed compaction end after a final reply plus reconnect and clear-history cleanup.
