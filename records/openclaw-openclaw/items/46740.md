## Summary

Keep open. Current main does not implement the PR’s intended classification change: active-turn plain `Error("acpx exited with code 1")` is still wrapped as `ACP_TURN_FAILED`, and the current manager test asserts that behavior. The PR is also stale against the current ACPX wrapper, which now delegates to the published `acpx/runtime` package rather than the older in-repo process-spawn code the PR edits.

## Best Possible Solution

Keep the PR open for maintainer review, but ask for a rebase and a current-design fix. The best path is either to add structured silent-exit classification in the upstream `acpx` runtime package and consume a released version here, or add a narrow OpenClaw wrapper/manager adapter that only maps genuinely silent prompt exits to `ACP_BACKEND_UNAVAILABLE` while preserving permission-denied, session-init, non-silent stderr, and recovery behavior. The regression tests should move to the current `acpx/runtime` delegate shape and keep the manager fallback coverage.
