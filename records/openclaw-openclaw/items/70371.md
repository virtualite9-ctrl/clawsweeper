## Summary

Keep #70371 open. Current main still treats plugin-local schema/config failures as invalid startup state and still has strict plugin registry load paths that throw when any plugin entry is marked error. The related PR #70394 is open/unmerged in the provided context, and the current checkout does not contain the graceful-degradation behavior requested by the issue.

## Best Possible Solution

Keep the issue open and fix it in core. The best path is a narrow plugin-local quarantine behavior: when `plugins.entries.<id>.config` fails that plugin's schema, record a clear per-plugin error/disabled state, skip that plugin's register/runtime tools, keep unrelated gateway/channels/plugins booting, and make `openclaw doctor` report the exact failing path plus a repair path. Regression coverage should include startup validation, runtime/metadata registry loading, CLI bootstrap behavior, and doctor reporting.
