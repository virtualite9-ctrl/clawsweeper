## Summary

Keep this PR open. Current main still parses a bare numeric Discord group allowlist target as a channel ID, fetches `/channels/<id>` before any guild match, and the current regression test explicitly documents the unresolved bare-guild-ID behavior. The PR is a focused bug fix, but it needs to be ported from the old `src/discord` path to the current `extensions/discord` implementation before it can merge.

## Best Possible Solution

Port the guild-first bare-numeric behavior into `extensions/discord/src/resolve-channels.ts`, preserve explicit `channel:`/`discord:` and `guild:`/`server:` semantics plus channel fallback, and update `extensions/discord/src/resolve-channels.test.ts` so the current regression test expects the guild to resolve while a non-guild bare numeric still falls back to channel lookup.
