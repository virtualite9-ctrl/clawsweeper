## Summary

Current main does not implement the PR's central feature. OpenClaw has log/transcript redaction, privacy-safe diagnostics, and optional proxy routing, but no native outbound LLM privacy detector/replacer, bidirectional restore path, `privacy` config section, custom JSON5 rules, or encrypted mapping store. This PR is still a security-sensitive implementation candidate and needs maintainer architecture/security review rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review or a smaller architecture-approved implementation. The best path is to define a core LLM-boundary privacy/filtering API with config/schema/docs, tests across the embedded runner, BTW, TTS, cache trace, trajectory, and logs, and explicit security behavior for fail-open versus fail-closed plus mapping persistence. If maintainers decide this belongs in plugin space, add a modifying LLM-boundary hook first; current observe-only hooks are not enough for this exact change.
