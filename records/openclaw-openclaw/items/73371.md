## Summary

Keep this PR open. Current main still does not implement the PR's intended behavior: the non-ACP `/new`/`/reset` path emits reset hooks and returns `null`, so the command pipeline can continue instead of acknowledging an empty-tail reset. The PR is a focused implementation candidate for the linked open bug.

## Best Possible Solution

Review and land this focused fix, or an equivalent maintainer-authored change, in `src/auto-reply/reply/commands-reset.ts`: after non-ACP reset hooks fire, return `{ shouldContinue: false, reply: ... }` when `resetTail` is empty, while preserving `return null` for `/new <tail>` and `/reset <tail>`. Keep the proposed regression tests in `src/auto-reply/reply/commands-reset-hooks.test.ts` and the user-facing changelog entry if the behavior ships.
