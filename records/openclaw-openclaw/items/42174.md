## Summary

Keep this PR open. Current main still rejects the reported Windows drive-letter Docker bind at config validation and runtime validation, so the PR is not obsolete or implemented on main. The bug report is concrete and still maps to the current sandbox bind code, but the patch needs maintainer review because it broadens a security-sensitive host bind validation path without tests or docs.

## Best Possible Solution

Keep this PR open for maintainer review or replace it with a maintainer-owned patch. The best fix should reuse `splitSandboxBindSpec` in config validation, update runtime validation and host-path canonicalization for Windows absolute drive paths without bypassing sandbox policy, add regression coverage for uppercase/lowercase drives, binds with and without mode, blocked credential/system paths, outside-root rejection, and Windows symlink/junction escapes, then update sandbox docs if native Windows drive-letter binds are officially supported.
