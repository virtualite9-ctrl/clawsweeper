## Summary

Keep open. Current main has a partial maintainer implementation that removed the old `git ls-files ... | xargs oxfmt` docs-format pipeline, but it does not definitely implement the latest PR’s intended Windows-clean behavior. The main-branch checker still sends all 463 docs paths as absolute temp paths to one `oxfmt` process during `--check`, has no Windows command-line batching, and has no landed tests for the Windows spawn/batching behavior the PR adds.

## Best Possible Solution

Keep this PR open for human review. Either land the repaired formatter/batching/tests here, or explicitly supersede it with the chosen canonical PowerShell-compatible docs-format PR and close the duplicate only after that implementation is on main.
