## Summary

Current main still appears vulnerable to the reported last-write-wins race. The exec approvals store still mutates and saves the caller-provided `ExecApprovalsFile` snapshot, while both gateway and node-host allow-always paths pass the already-loaded `approvals.file` into that mutation path. The related fix PR #52403 was closed unmerged in the provided timeline, and no equivalent locked/reload-before-write implementation is present on main.

## Best Possible Solution

Keep this issue open and implement a locked exec-approvals mutation helper that reloads the latest `exec-approvals.json` inside the critical section, applies additive allowlist/metadata changes, then saves. Update `addAllowlistEntry`, `addDurableCommandApproval`, `persistAllowAlwaysPatterns`, `recordAllowlistUse`, and both gateway/node-host call paths to use the helper, with regression coverage for two stale snapshots adding different allow-always entries for the same agent.
