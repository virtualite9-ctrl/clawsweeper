## Summary

Keep this issue open. Current main still does not make the bundled TUI subscribe to session transcript updates or route `session.message` / `sessions.changed` events, so Telegram-to-TUI real-time sync remains missing. The linked standalone `oclaw` TUI is useful prior art and workaround, but it does not fix `openclaw tui`.

## Best Possible Solution

Keep this issue open. Implement bundled TUI live transcript sync by adding a TUI subscription surface, subscribing to the active canonical session on connect and session changes, handling `session.message` with active-run deferral/history reload semantics, applying `sessions.changed` metadata/session-list refreshes, and adding focused TUI regression tests. Treat the linked standalone `oclaw` TUI as prior art or a workaround, not as a reason to close the OpenClaw core issue.
