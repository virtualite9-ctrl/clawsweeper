## Summary

Keep this issue open. Current main has meaningful subagent delivery hardening, but it still has a terminal announce give-up path after the retry budget, an open PR candidate for the silent-drop symptom, and only restart-orphan recovery rather than the broader timeout/no-notification behavior requested here.

## Best Possible Solution

Keep this issue open for maintainer/runtime design. The next best path is to review or supersede #45200 for announce give-up notification, then decide a durable persist-and-retry or single-delivery-state model for async completions, with targeted regression coverage for Telegram delivery, non-restart timeouts, and any chosen file-claim verification policy.
