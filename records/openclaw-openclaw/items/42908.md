## Summary

Keep this PR open. Current main still does not pass Feishu card-action sibling fields through to the agent, and this PR is paired with the same-author open issue #42754. The implementation needs a current-main rebase because the production parser and fallback path now drop the requested fields before handler-only logic can help.

## Best Possible Solution

Keep this PR open as an implementation candidate. The best path is to rebase it on current main, preserve Feishu-native sibling action fields in `parseFeishuCardActionEventPayload`, extend the event/fallback serialization path for `option`, `options`, `form_value`, and likely the broader `input_value`/`name` fields tracked by #43202 and #43953, while preserving existing `value.text` and `value.command` behavior. Add raw `card.action.trigger` regression tests, including empty arrays and empty objects.
