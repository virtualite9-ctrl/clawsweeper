## Summary

Keep open. Current main still exposes the same Google Gemini CLI OAuth onboarding path and still runs the desktop flow through a localhost:8085 callback followed by token exchange, while the reported AbortError/login failure is not covered by a clear current-main fix, regression test, or superseding canonical issue. The issue also has multiple independent confirmations, including one commenter observing that no localhost:8085 listener was present during OAuth.

## Best Possible Solution

Keep this issue open for the Google plugin owner. The likely maintainer path is to reproduce the Gemini CLI OAuth login on current main or the latest release with a real macOS/Homebrew setup, then harden extensions/google/oauth.ts and the shared callback helper as needed, for example by starting the localhost callback listener before opening the browser, improving manual fallback/timeout behavior for AbortError cases, and adding a focused regression test for the desktop callback flow.
