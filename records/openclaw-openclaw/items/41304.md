## Summary

Keep this issue open. Current main has narrower guardrails and diagnostics for tool-call hallucination, but I did not find evidence that the broader reported failure mode is fixed: plugin or MCP tools can be registered while the assistant still emits prose or simulated success instead of a structured tool call.

## Best Possible Solution

Keep the issue open and ask for a trajectory export from a current failing run. Maintainers should use it to separate three cases: no callable tools after policy filtering, tools present but no structured tool call emitted by the model/provider, or a real tool execution whose side effect fails in the plugin/service. Then add a focused regression or diagnostic for the confirmed path.
