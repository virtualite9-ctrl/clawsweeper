## Summary

Keep this PR open. Current main has the later pending-tool-media queue/final-reply merge, but it still does not implement this PR's specific fallback from the read tool's start args when an inline image result has no MEDIA text and no details.path. The linked Feishu issue remains a real implementation candidate path, and the PR also has an existing functional review comment that should be addressed before landing.

## Best Possible Solution

Keep the PR open as a plausible fix for the open Feishu read-image media bug. The best path is to update the patch so startArgs.path is trimmed and treated as missing before falling back to startArgs.file_path, add regression coverage for that mixed-arg case, keep the existing trusted-tool filtering intact, and then review/land it against current main.
