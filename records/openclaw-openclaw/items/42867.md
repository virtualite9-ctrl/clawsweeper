## Summary

Keep this issue open. Current main implements the heartbeat-aware freshness part, but the requested turn-count TTL/max-turn session rollover is still missing from session reset config, schema, freshness evaluation, reuse logic, docs, and tests. The broader same-author #45390 overlaps but remains open and is not a maintainer-designated canonical replacement.

## Best Possible Solution

Keep this open as a core session lifecycle feature unless maintainers explicitly consolidate it into #45390. A complete fix should add a schema-validated and documented max-turn reset policy, define which user/assistant turns count versus heartbeat/system events, wire the check into automatic session rollover before reuse, and add regression coverage for daily, idle, heartbeat, and max-turn interactions.
