## Summary

Keep this PR open. Current main still lacks Telegram forum-topic deletion, and the PR addresses a real unshipped capability. It is not ready to merge as-is because it is stale against the current Telegram plugin layout and still needs destructive-ID parsing hardening.

## Best Possible Solution

Keep the PR open as useful implementation work, but request a rebase/port to the current Telegram plugin layout. The best path is to add `topic-delete` to the shared action list and target spec, map it in `extensions/telegram/src/channel-actions.ts`, implement `deleteForumTopicTelegram` in `extensions/telegram/src/send.ts`, add a `deleteForumTopic` branch in `extensions/telegram/src/action-runtime.ts`, update Telegram docs and focused tests, and harden all destructive ID parsing to explicit decimal-only integers. The broader close/reopen topic work should stay tracked separately by #11414.
