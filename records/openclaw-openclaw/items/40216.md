## Summary

Keep this PR open. Current main has absorbed the shared CJK estimator and applies it to several active context-pruning paths, but it has not fully implemented this PR's stated behavior: the active soft-trim eligibility path still uses raw UTF-16 length, assistant tool-call arguments still use raw JSON string length, and this PR edits the removed pi-extensions path rather than the active pi-hooks implementation.

## Best Possible Solution

Keep this PR open as useful prior work, or replace it with a rebased maintainer patch against src/agents/pi-hooks/context-pruning/pruner.ts. The best fix should use the shared src/utils/cjk-chars.ts estimator for tool-call argument JSON, make soft-trim eligibility and head/tail extraction use consistent units, and add focused regressions for CJK-heavy tool results and assistant tool-call arguments. After that replacement lands, this stale old-path PR can be closed as superseded or implemented.
