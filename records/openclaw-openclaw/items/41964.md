## Summary

Current main still does not implement this PR's intended behavior: non-control-UI-visible external-channel runs remain suppressed on the chat/node fanout path. The PR is also not safe to close as obsolete or merge as-is, because its one-line guard removal would widen a documented hidden-run visibility boundary for external-channel sessions.

## Best Possible Solution

Keep this PR open, but ask for a revised design before merge. Preserve `isControlUiVisible` for general Control UI and node-session fanout, then wire Gateway TUI `--session` mode to the existing `sessions.messages.subscribe` / `session.message` transcript stream or add a narrowly authorized session-scoped assistant-delta subscriber. Coordinate with #38829 and #45417, and add regression tests for external-channel sessions plus explicit hidden-run fanout/authorization tests.

## Risks / Open Questions

- Merging the PR as written would broadcast assistant chat/final events and node-session `agent` events for runs deliberately marked non-control-UI-visible, including Telegram or iMessage-origin sessions.
- The PR changes a documented visibility boundary without adding a TUI-only authorization/subscription path or tests proving hidden external-channel data cannot reach unrelated operator clients.
- Closing this PR now would discard a focused reproduction and patch attempt for a still-plausible TUI live-display bug; related live TUI work remains open in #38829 and #45417.
