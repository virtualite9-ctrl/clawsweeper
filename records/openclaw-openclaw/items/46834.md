## Summary

Keep open. Current main has a newer ACP task-completion notification path, but it does not implement this PR's intended non-threaded direct-chat relay of the child agent's final visible output. The current parent stream relay still emits internal system events and generic completion notices, while current tests explicitly expect background ACP progress to stay off the foreground lane and only a generic terminal notification to be sent.

## Best Possible Solution

Keep this PR open for maintainer review or a current-main rework. The clean path is to choose one owner for ACP completion delivery: either extend the task-completion path to carry the final visible ACP output, or add a direct parent-relay path with duplicate suppression against task notifications, current sanitization policy, whitespace-preserving delta accumulation, threaded-context exclusion, error handling, truncation coverage, and regression tests for non-threaded direct chats.
