## Summary

Keep this issue open. Current main still exposes terminal-looking session status strings as the agent-facing session contract, and it does not add a separate `resumable`, `lastTurnStatus`, `lifecycleStatus`, or `taskStatus` field that would address the reporter's orchestration ambiguity. The provided prior ClawSweeper comment remains consistent with current main: related closed issues #69202 and #71756 cover adjacent recovery/finalization behavior, not this session status/resumability API contract.

## Best Possible Solution

Keep this issue open as an agent-facing session API contract bug. Add a backward-compatible `resumable` field and a distinct `lastTurnStatus`, `lifecycleStatus`, or `taskStatus` field to persisted, gateway, and `sessions_list` rows; document the semantics on https://docs.openclaw.ai/concepts/session-tool; and update tool descriptions or prompt guidance so orchestrators prefer `sessions_send` for visible existing sessions unless the session is deleted or inaccessible. Defer any status-string rename to a versioned migration.
