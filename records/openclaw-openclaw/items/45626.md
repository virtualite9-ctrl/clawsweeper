## Summary

Keep this PR open. Current main still does not expose `fileId` in the shared `message` tool schema, while Slack `download-file` still requires that parameter and Slack discovery exposes the action without adding a Slack-specific schema contribution. The PR is a narrow schema fix for a real remaining gap, and the linked open bug remains a valid implementation target.

## Best Possible Solution

Review and land a scoped schema fix, preferably with a regression test that creates the Slack-scoped message tool and asserts `fileId` is present while `download-file` remains exposed. The current PR’s one-field addition is plausible, but a plugin-owned schema contribution would also be acceptable if maintainers want to avoid adding more platform-specific fields to the shared schema.
