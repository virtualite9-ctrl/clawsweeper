## Summary

Keep open. Current main contains related fixes for default-agent `--to` routing and deliveryContext persistence, but it still does not implement the central request here: resolving `openclaw agent --channel discord --to ...` by stored delivery context before falling back to a main session. The CLI computes the session key before channel/account delivery metadata is applied, and the resolver has no channel/account/thread delivery-context lookup.

## Best Possible Solution

Implement or explicitly reject delivery-context session routing for `openclaw agent`. If implemented, normalize Discord targets using the shared channel target/route parser, pass channel/account information into session resolution, search existing sessions by `(channel, to, accountId, threadId)` before main fallback, preserve explicit `--session-id`/`--session-key` precedence, and add regression tests for raw numeric Discord channel IDs and `channel:<id>` targets.
