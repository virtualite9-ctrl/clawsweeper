## Summary

Current main still does not implement `/trust` or `/untrust`, and this PR remains substantive security-sensitive feature work rather than an obsolete cleanup candidate. The linked feature issue is closed, but the shipped code and docs still only expose persistent `/exec` and `/elevated` controls, with no transient trust-window command, handler, expiry state, or resolver path.

## Best Possible Solution

Keep this PR open for maintainer review or manual dedupe. The best path is to choose a canonical temporary exec-trust design across the related PRs, ensure trust is checked at execution time for queued/follow-up runs, disclose active trust in status or `/exec` reporting, add focused authorization/expiry/rollover tests, and document `/trust` and `/untrust` in the public slash-command and exec docs.
