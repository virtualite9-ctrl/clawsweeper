## Summary

Current main still does not implement the PR's intended opt-in Linuxbrew Docker support. The primary Dockerfile and setup script have no OPENCLAW_INSTALL_BREW/Linuxbrew build arg, no setup passthrough, and no skip-build guard, while brew-kind skill installs still fail when no brew executable is found. The PR remains relevant to #14593, but the provided diff is not merge-ready because prior review comments identify concrete Dockerfile hazards and the diff adds build-time execution of Homebrew's remote HEAD installer.

## Best Possible Solution

Keep this PR open for maintainer judgment, but do not merge it as-is. The best path is to resolve #14593 through one canonical Docker-safe approach: rework this PR against current main with conditional environment handling, focused setup tests, Docker docs, and an accepted Homebrew installer supply-chain policy; or intentionally land a replacement such as #55940 or #57553 and then close the superseded PRs.

- The linked Docker skill-install bug remains unresolved until a revised Docker-safe approach lands; #55940 and #57553 are alternative open implementation attempts but neither is merged.
