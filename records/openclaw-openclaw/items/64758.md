## Summary

Keep this PR open. Current main still has the steer-mode gap: steer injection is only attempted while the active run is streaming, non-streaming active embedded runs reject queued steer input, and the documented fallback remains a queued followup. The PR is still a focused implementation candidate for open #10960, but it changes active-run abort semantics and likely needs rebase/maintainer review before merge.

## Best Possible Solution

Keep this PR open as the implementation candidate for #10960. Rebase it onto current main, adapt the PI embedded runtime import/signature changes, preserve the non-heartbeat abort guard, update queue docs if maintainers accept abort-to-unblock semantics, and rerun the targeted auto-reply regression tests before review.
