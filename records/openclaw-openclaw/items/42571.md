## Summary

Keep this PR open. Current main still does not normalize bare Telegram `group:<numeric>` delivery targets in the active Telegram plugin, and the submitted PR is stale against removed `src/telegram/*` paths and incomplete because it does not patch the shared parser used by send and approval routing.

## Best Possible Solution

Do not merge this PR as-is. Rework or supersede it with a current-layout patch in `extensions/telegram/src/targets.ts` that normalizes bare `group:<numeric>` before topic and colon parsing in the shared parser, while preserving explicit `telegram:group:` and topic behavior. Add focused regression coverage for strip/normalize/parse/chat-type behavior, send delivery without lookup, plugin target normalization/session announce routing, and exec approval button routing. After that lands on main, close this PR as superseded or implemented.
