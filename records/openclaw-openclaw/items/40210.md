## Summary

Current main still treats workspace bootstrap-file reads as workspace-bound boundary reads, so an out-of-workspace symlink target is still rejected and rendered as a missing bootstrap entry. Also, open PR #40230 is still the active implementation candidate for this issue, so this issue should not be closed before that PR is reviewed, landed, rejected, or superseded.

## Best Possible Solution

Keep this issue open. Maintainers should review PR #40230 as the active implementation candidate, decide whether trusted workspace bootstrap files may follow symlinks outside the workspace, and if accepted land a narrow readWorkspaceFileWithGuards change with regression coverage. After a fix ships, consolidate overlapping reports #38622 and #64472 as appropriate.
