## Summary

Current main does not implement this PR's intended Web UI fix. The Control UI still intercepts `/model ...` as a local command and sends any non-empty `/model` argument, including `status`, `list`, or `info`, to `sessions.patch` as a model value. The PR remains a valid implementation candidate, but it should be refreshed against current model-command semantics and the review concern about preserving real model IDs named like subcommands should be resolved.

## Best Possible Solution

Keep this PR open for maintainer review or contributor refresh. The best path is to update the Control UI local `/model` handler and tests so `/model list` and `/model status` follow the current shared directive semantics without regressing legitimate model selection, then land it or replace it with a narrower fresh PR that closes the linked bug.
