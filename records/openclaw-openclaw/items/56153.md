## Summary

Keep open. Current main still does not contain the Telegram typing retry/backoff behavior proposed by #56153: `sendTypingTelegram` still uses the generic retry path without `strictShouldRetry`, still propagates typing failures, and the inbound `sendChatAction` handler remains 401-only. The linked #56096 bug remains actionable, while the PR discussion also shows maintainer review concerns that should be reconciled before merge rather than auto-closing the PR.

## Best Possible Solution

Keep the PR open or consolidate it with the related #56096/#55838 work under maintainer review. The best fix should cover both `sendTypingTelegram` and the inbound `sendChatActionHandler`: suppress 429/529 retries with `strictShouldRetry`, cap and merge retry config without discarding account/per-call overrides, make typing indicators best-effort with warning logs, avoid repeated 3s keepalive loops after failures, and add focused regression tests for 429/529 and non-blocking behavior.
