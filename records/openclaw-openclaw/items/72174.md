## Summary

Current main still has the macOS attach-only bug this PR targets. Both the launch-flag path and Debug Settings toggle write the disable marker and then call the launchd disable/uninstall path, while docs define attach-only as never installing or managing launchd. The PR remains useful and is not implemented on main.

## Best Possible Solution

Keep this PR open for maintainer review or landing. The best fix is the PR's marker-only attach-only path for both launch flags and Debug Settings, plus the regression test proving attach-only writes the marker without issuing daemon commands.
