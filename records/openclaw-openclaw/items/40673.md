## Summary

Keep this issue open. Current main still has only a low-level `OPENCLAW_RAW_STREAM` truthy check, and current docs still describe how to enable raw-stream logging without documenting disable values or LaunchAgent/service-env precedence. A recent macOS LaunchAgent hardening commit moved service env into an owner-only wrapper file, but it does not specifically resolve or document the reported raw-stream disable path.

## Best Possible Solution

Keep this open and resolve it with a narrow docs-and-regression change. Document exact enable values for `OPENCLAW_RAW_STREAM`, document that unset, `0`, `false`, `off`, and other non-truthy values disable logging, and explain precedence across CLI flags, `launchctl setenv/unsetenv`, LaunchAgent service env files, state-dir `.env`, and config `env.vars`. Add focused tests for disabled `appendRawStream` values and managed-service/LaunchAgent env planning/readback so a previously enabled raw-stream value is cleared or overridden correctly. If live macOS still appends when the effective gateway env is unset or `0` and no durable source enables it, fix that runtime/service path in the same area.
