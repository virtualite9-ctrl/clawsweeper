## Summary

Keep this issue open. Current main still has a plausible static reproduction for the reported Ollama regression: Ollama only exposes non-off thinking levels when the provider profile receives `reasoning: true`, but the slash-command, session/defaults, and runtime validation paths often ask for supported levels without passing catalog/model reasoning. That matches the report’s `Options: off`, rejection of `/think medium`, and resolved `think: false` payload. The related closed #48010 and #72407 do not supersede this: one covered model params support and the other was described as a UI-label-only issue, while this report shows runtime forcing/off fallback.

## Best Possible Solution

Keep this issue open and fix the Ollama thinking-profile resolution path. The likely maintainer fix is to pass the selected model catalog/reasoning into slash-command choices, `isThinkingLevelSupported`, `formatThinkingLevels`, gateway session rows/defaults, and fallback remapping, or to make the Ollama profile resolve from durable model metadata. Also ensure explicit `params.think`/`params.thinking` is not overwritten by an `off` fallback, then add regression coverage for a thinking-capable Ollama model showing `/think low|medium|high|max` and sending top-level `think: "medium"` or `"high"`.
