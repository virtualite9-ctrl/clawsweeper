## Summary

Keep this PR open for maintainer judgment. Current main has not implemented the clearer pairing error messages: the channel/account key validation still throws generic errors, and unique pairing code exhaustion still throws the old generic message. The PR also has a real unresolved robustness/security review concern around using JSON.stringify in validation error construction, so it is not a clean obsolete/implemented close candidate.

## Best Possible Solution

Revise the PR against current main by moving the clearer diagnostics into `src/pairing/allow-from-store-file.ts` and `src/pairing/pairing-store.ts`, use a non-throwing and appropriately redacted debug formatter, and add focused tests for invalid channel/account keys and unique-code exhaustion diagnostics.
