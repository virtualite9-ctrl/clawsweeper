## Summary

Keep open. Current main still ignores wake-hook session targeting: direct `/hooks/wake` normalizes only `text`/`mode`, mapped `wake` actions drop `agentId`/`sessionKey`, and `dispatchWakeHook` enqueues into the default main session. The PR is a focused implementation candidate for the open linked bug #64556, and its latest public patch addresses the maintainer-raised prefix-policy and global-scope concerns without touching workflows, package metadata, lockfiles, dependency sources, or release scripts.

## Best Possible Solution

Keep this PR open as the active implementation candidate. Maintainers should review the latest head `ceba7d487e8be3fb39c13306cc938cf7df279fdd`, rerun the targeted gateway wake-hook tests plus the changed gate, and land it if the prefix-policy/global-scope safeguards hold, so the linked wake-hook routing bug can close through the merged fix.
