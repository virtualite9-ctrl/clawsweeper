## Summary

Keep open. Current main still enforces same-root realpath containment for skills and does not implement this PR's cross-configured-root symlink allowlist. The PR targets a real remaining gap, but the submitted approach needs revision because the review discussion identifies a plausible `skills.allowBundled` bypass when a non-bundled symlink resolves into bundled skills. Related #59688 overlaps but is also open/unmerged, so it is not a superseding shipped fix.

## Best Possible Solution

Keep this PR open for maintainer review, but require a revised implementation. The best path is to land one consolidated skills symlink fix, coordinated with #59688 and the managed-symlink issue #44051, that allows only explicitly trusted non-bundled configured-root targets or preserves the target root's source policy. Add regression coverage for the intended cross-root symlink cases, unknown external symlinks still warning/skipping, and the `skills.allowBundled` bypass scenario before merge.
