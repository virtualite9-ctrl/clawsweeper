## Summary

Keep this PR open. Current main still lacks the PR's central guidance: the macOS VM docs only tell users to enable Remote Login before running headlessly, and the LaunchAgent bootstrap error still points users to signing into the desktop or using a custom LaunchDaemon without mentioning VM auto-login. The PR needs a refresh rather than closure because localized docs are no longer maintained in this repo and the runtime hint should get test coverage.

## Best Possible Solution

Refresh this PR instead of closing it: add the auto-login step to `docs/install/macos-vm.md` in the post-setup/headless VM flow, add a scoped auto-login hint to `throwBootstrapGuiSessionError`, update the launchd guidance test, and remove the obsolete localized-doc edit in favor of the current English-docs/i18n workflow.
