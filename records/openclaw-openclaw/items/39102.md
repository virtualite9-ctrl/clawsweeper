## Summary

Current main still does not implement this PR’s intended per-agent outbound A2A allowlist override. The shipped A2A policy remains global-only, the strict per-agent tools type/schema/help surface still lacks `agents.list[].tools.agentToAgent.allow`, session-tool errors still reference only the global `tools.agentToAgent` gate, and the open PR remains a plausible implementation candidate rather than obsolete work.

## Best Possible Solution

Keep this PR open for maintainer review or ask the author to refresh it on current main. The best implementation path is to add a strict `agents.list[].tools.agentToAgent.allow` type/schema/help/docs surface, evaluate global A2A participation before requester-specific outbound overrides, preserve outbound-only semantics, keep sandbox visibility and deterministic `session_status` behavior intact, prune deleted agent IDs immutably, and validate with targeted A2A/session-status tests plus the changed gate before merge.
