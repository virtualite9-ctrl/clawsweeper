## Summary

Keep this PR open. Current main does not implement the PR’s central cron fan-out feature: `delivery.additionalTargets` is absent from cron types, gateway schema, normalization/merge, CLI edit surface, docs, tool schema, UI type surface, and the isolated cron run delivery path. The related feature issues #48026 and #54935 are closed, but current main does not contain the implementation they describe, so the open PR is still the implementation candidate rather than obsolete.

## Best Possible Solution

Keep this PR open as the implementation candidate, but ask the author or a maintainer to rebase and narrow it against current main. The best product path is an additive cron delivery contract with `delivery.additionalTargets` wired through `src/cron/types.ts`, `src/gateway/protocol/schema/cron.ts`, normalization/merge, CLI/UI/agent-tool/docs, and isolated cron delivery. Fan-out should only occur under explicitly documented primary-delivery success semantics, reuse the existing outbound delivery policy stack, preserve session-key routing, and include tests for failure isolation, best-effort partial failure, clearing targets, `channel: "last"`, and webhook-mode behavior.
