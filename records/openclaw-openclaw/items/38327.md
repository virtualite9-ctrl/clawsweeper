## Summary

Current main does not provide enough implementation or regression-test evidence to close this Google Vertex embedded-runner crash. The bundled Google plugin still owns the `google-vertex` hook path, but the shared thinking payload wrapper only sanitizes `google-generative-ai` models while current tests show `google-vertex` can resolve through `google-gemini-cli` or OpenAI-compatible metadata. The issue also has fresh post-stale confirmations and a concrete private-fork analysis pointing to an additional gaxios/native-fetch auth path that is not conclusively covered for Vertex embedded runs.

## Best Possible Solution

Keep this issue open. The best maintainer path is to add a targeted embedded-runner regression for `google-vertex/gemini-3.1-pro-preview` and `google-vertex/gemini-3-flash-preview`, covering the currently demonstrated `google-gemini-cli` and OpenAI-compatible Vertex metadata variants, then fix the Google-owned wrapper/transport path. Separately evaluate the #33428 gaxios/native-fetch proposal against current Google Vertex auth behavior and prefer a scoped transport or upstream dependency fix over broad node_modules patching if possible.
