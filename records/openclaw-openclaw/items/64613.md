## Summary

Keep open. Current main covers several adjacent hidden-history cases, including heartbeat prompts, internal runtime envelopes, and the bundled active-memory tag-block shape, but it still does not filter the reported plain `## Memory system — ACTION REQUIRED` / auto-recall `memory_search` instruction at the shared visible-history boundary. Because `chat.history` and session-history SSE both project through the same display path, a persisted plain memory-system prompt would still be visible.

## Best Possible Solution

Keep this issue open. The best maintainer path is to prevent memory/system-injection prompts from being persisted as ordinary visible transcript text, or add a narrow shared projection filter in `src/gateway/chat-display-projection.ts` that is provenance-aware where possible. Add regression coverage for string and text-block content, the reported memory action block, heartbeat prompts with workspace-file suffixes, legitimate user text that merely quotes these phrases, `chat.history`, and session-history SSE.
