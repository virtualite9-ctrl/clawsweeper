## Summary

Keep this PR open. Current main still does not implement a fail-fast default-model auth guard before channel startup, and the PR remains useful as a bug-fix candidate, but it should be revised before landing because main moved startup logic into the post-attach module and the PR's auth check is narrower than runtime model auth resolution.

## Best Possible Solution

Keep this PR open for revision. Rebase or rewrite the guard into `src/gateway/server-startup-post-attach.ts` before channel startup, preserve the `OPENCLAW_SKIP_CHANNELS` and `OPENCLAW_SKIP_PROVIDERS` bypasses, and base the check on the same non-mutating auth contract used by runtime model execution, such as `hasAvailableAuthForProvider` with config/store/agentDir or a safe `resolveApiKeyForProvider` probe. Add focused tests for missing auth, config-backed API key auth, Bedrock/aws-sdk auth, synthetic local/plugin auth, skip-channel bypasses, and non-circular remediation text.
