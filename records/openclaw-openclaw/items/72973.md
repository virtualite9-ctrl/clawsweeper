## Summary

Keep this PR open. Current origin/main does not contain the outbound filename helper or equivalent wiring, and the media paths still pass Content-Disposition and local media basenames through with only path.basename. The PR is a plausible, focused hardening candidate rather than obsolete cleanup. It still needs normal maintainer review, especially on the intentional LRM/RLM/ALM tradeoff raised in review discussion.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to review the narrow media-boundary helper, confirm the intended Unicode strip set, and land it or request changes alongside any coordination needed with #71517's filename-decoder work.
