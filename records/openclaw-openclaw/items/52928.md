## Summary

Current main still does not implement the requested `agents.setDefault` Gateway RPC. The existing Gateway agent surface exposes list/create/update/delete and file helpers only; `agents.update` still rejects unknown fields such as `default`; the method is not advertised, scoped, validated, exported, documented, or used by the Control UI. The related #59070 context remains relevant because the UI still stages `agents.defaultId`, which is not the documented config shape for selecting the default agent.

## Best Possible Solution

Keep this issue open. Implement an additive core Gateway method `agents.setDefault` with params/result schemas, protocol exports, handler registration, method advertising, ADMIN scope entry, docs, and focused handler tests. The handler should reject unknown configured agent ids, update `agents.list` so only the selected entry has `default: true`, persist config through the existing config writer path, return `{ ok: true, defaultId }`, and update the Control UI Set Default action to call the new RPC instead of staging `agents.defaultId`.
