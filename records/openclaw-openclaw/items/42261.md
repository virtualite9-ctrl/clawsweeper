## Summary

Current main still has the reported callback-coupled live-thinking path. WebSocket-capable agent events can carry `stream: "thinking"`, but the Pi subscribe layer only enables and emits those events when an optional surface callback exists; the webchat `chat.send` path does not provide that callback.

## Best Possible Solution

Keep this issue open. Fix `src/agents/pi-embedded-subscribe.ts` by enabling `streamReasoning` from stream mode plus existing visibility gates rather than callback presence, emitting `stream: "thinking"` before optionally invoking `params.onReasoningStream`, and adding a callback-absent regression test proving WS/agent-event clients receive live thinking. Update the reasoning docs if webchat stream support becomes intentional user-facing behavior.
