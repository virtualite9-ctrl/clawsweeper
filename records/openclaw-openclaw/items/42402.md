## Summary

Keep this PR open. Current main still lacks the runtime type guard the PR is trying to add for `env.vars`, so truthy non-string values can still reach `containsEnvVarReference()` and throw. The PR is stale against the current file layout, but it remains a valid focused bug-fix candidate rather than implemented, duplicate, or superseded.

## Best Possible Solution

Keep this PR open as a focused bug-fix candidate. Rebase the one-line non-string guard into `src/config/config-env-vars.ts`, add the mixed-type regression coverage to `src/config/config.env-vars.test.ts`, decide whether malformed `env.vars` values should emit a diagnostic, then run the relevant targeted config tests and changed gate.
