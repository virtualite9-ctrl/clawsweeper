## Summary

Keep this PR open. Current main has not implemented the intended GitHub Copilot device-flow fixes: the login loop still polls before any initial delay, still handles `slow_down` with a fixed `interval + 2000ms` wait rather than a server-returned interval, and `login-github-copilot` still has no `--wait` flag or pass-through path. The PR is stale against the current plugin-owned implementation, but the underlying work remains useful.

## Best Possible Solution

Keep the PR open, but require it to be ported to the current plugin-owned GitHub Copilot auth path. The best implementation is to update `extensions/github-copilot/login.ts` to honor the initial polling interval before the first token request, parse and apply GitHub's `slow_down.interval` when present, add a `--wait`/plain-text confirmation path, and wire that flag through `src/cli/models-cli.ts`, `src/commands/models/auth.ts` or an explicit provider-auth context option, and the `src/plugin-sdk/github-copilot-login.ts` facade type. Add targeted tests and update the GitHub Copilot provider docs.
