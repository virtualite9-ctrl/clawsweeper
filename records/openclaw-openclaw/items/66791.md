## Summary

Keep this issue open. Current main at d48c3e12a5d896cbdad3e7fcf5bba2af644a9ca5 still exposes Discord scheduled-event list/create only; event-edit and event-delete are missing from the shared action contract, tool schema, Discord discovery, parser/runtime dispatch, REST helper layer, CLI surface, docs, and focused tests. The reporter's follow-up accurately traces the bundled Discord implementation path and describes real remaining work.

## Best Possible Solution

Keep this issue open and implement event-edit/event-delete as coordinated shared message-tool plus bundled Discord plugin work: add the actions to the shared action list, target/display metadata, schema, CLI, and docs; add eventId to event parameters; expose the actions under the existing Discord events gate; add parser/runtime dispatch; add Discord scheduled-event fetch/update/delete helpers; preserve cover-image handling; include requester authorization checks; and cover parser/runtime/REST payload behavior with focused tests.

## Risks / Open Questions

- Adding event-delete and event-edit expands bot-credentialed external-state mutation. The implementation should require trusted requester identity and Discord permissions, especially while #68703/#68716 track guild-admin authorization hardening.
- Sparse event-edit payloads can accidentally clear required Discord scheduled-event fields or violate event-type constraints. Fetching the current event and merging only provided fields is safer than sending a minimal PATCH blindly.
