## Summary

Keep this PR open. Current main still does not implement the PR's core create-time hardening: several managed session creation paths use plain recursive mkdir without private mode, chmod hardening, or the managed/custom path split described in the PR. The PR is still a plausible implementation candidate, but it is security-sensitive and broad enough to need maintainer review rather than automated closure.

## Best Possible Solution

Keep the PR open as a candidate, but require a maintainer-reviewed update against current main. The best implementation would centralize managed session directory creation in the session paths/store/transcript/lock layer, preserve plain mkdir behavior for arbitrary caller-supplied custom paths, harden existing managed dirs to 0700 where safe, reject symlinked managed path components before permission mutation, and add focused tests for setup/onboarding, agents.create, session store writes, transcript creation, lock creation, configured session.store templates, and custom-path non-regression.
