## Summary

Keep this PR open. Current main still lacks heartbeat-specific fallback guidance, but the submitted docs patch is stale against the current `ParamField` docs structure and its final sentence is inaccurate because different-provider heartbeat overrides skip unrelated configured fallbacks but can still fall through to the configured primary. Security review found no code-execution or supply-chain surface in the provided docs/changelog-only diff.

## Best Possible Solution

Keep this PR open and request a narrow revision against current main: add the fallback note inside the current `heartbeat.model` `ParamField`, align it with `docs/concepts/model-failover.md`, explain that no override and same-provider overrides follow the standard chain, explain that different-provider overrides skip unrelated configured fallbacks unless already in the configured fallback chain, and mention that the configured primary can still be tried as the final candidate. Fix or remove the changelog entry.
