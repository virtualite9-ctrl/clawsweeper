## Summary

Keep open. Current main preserves the non-fatal Slack external arg-menu fallback, but it still only logs the registration failure through `logVerbose`; I did not find a visible warning, channel health/status field, probe diagnostic, or doctor diagnostic that would satisfy the issue's central request.

## Best Possible Solution

Keep this issue open and implement the signal in the bundled Slack plugin. Preserve the current non-fatal fallback, but record the degraded external-arg-menu state through a visible channel runtime warning and/or account status/health diagnostic, then add a focused test that fails if the fallback is only logged through `logVerbose`. If the chosen surface is user-facing, update `docs/channels/slack.md` as well.
