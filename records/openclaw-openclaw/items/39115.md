## Summary

Keep this PR open. Current main still does not implement the requested global `agents.defaults.reasoningDefault` behavior: the defaults schema/types omit the key, reply/status resolution only honors session or per-agent reasoning defaults, and docs describe per-agent `agents.list[].reasoningDefault` only. The PR is still an active implementation candidate with recent author follow-up and prior maintainer product-evaluation signal, not obsolete or implemented on main.

## Best Possible Solution

Keep this PR open for maintainer review. A mergeable solution should add `agents.defaults.reasoningDefault` consistently across schema, exported types, generated schema/help metadata, docs, reply resolution, current-level status, and direct status; preserve explicit `off` over model defaults; and include tests for directive, session, per-agent, global, model-default precedence, plus sender authorization. If maintainers reject the product behavior, close with that explicit product decision rather than as stale or implemented.
