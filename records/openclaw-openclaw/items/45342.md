## Summary

Keep this PR open. Current main still has no cache TTL or `usage.cacheTtlMs` config for `loadProviderUsageSummary`, and the active status/usage call paths still call the loader directly. The PR is stale against the newer plugin-owned provider usage architecture, but it remains a plausible implementation candidate rather than something current main has superseded.

## Best Possible Solution

Keep this PR open as a candidate, but require a rebase or replacement patch against current main’s plugin-era provider usage path. The best fix belongs in `src/infra/provider-usage.load.ts` or a shared provider-usage cache seam, with config/schema/help/types aligned if `usage.cacheTtlMs` is accepted, and regression tests for cache hits, cache keys, TTL expiry, disabling, explicit auth bypass, and transient failure retry behavior.
