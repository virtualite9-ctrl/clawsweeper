## Summary

Keep open. Current main still resolves the PDF tool through the registry-only media helper, so the custom-provider registry-miss bug described by linked #65742 is not implemented on main. This PR is a plausible focused fix, but the proposed fallback should be revised to preserve normalized provider-alias matching before landing.

## Best Possible Solution

Keep this PR open or have a maintainer take it over. The best fix is a PDF/media model resolver that keeps the current normalized registry lookup, falls back to `cfg.models.providers` using `findInlineModelMatch`-equivalent normalized provider/model matching, and adds regression coverage for both a registry-miss custom provider and a provider-alias custom provider.
