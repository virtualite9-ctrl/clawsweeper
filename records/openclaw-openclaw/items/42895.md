## Summary

Keep this PR open. Current main still resolves OpenClaw home from OPENCLAW_HOME, HOME, USERPROFILE, and os.homedir() only, so the Android/Termux /home failure from #42828 remains unimplemented on main. The PR is still a focused implementation candidate, and its later commit addresses the earlier review gap for OPENCLAW_HOME tilde expansion.

## Best Possible Solution

Keep this PR open, refresh it against current main if needed, and review or land the focused Termux home-resolution fix with regression tests. Preserve existing OPENCLAW_HOME, HOME, USERPROFILE, OPENCLAW_STATE_DIR, and OPENCLAW_CONFIG_PATH precedence; consider tightening the Termux PREFIX guard and updating the environment docs when the behavior lands.
