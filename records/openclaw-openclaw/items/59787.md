## Summary

Keep this PR open. Current main still has the target-only suppression bug the PR is trying to fix, so the work is not obsolete. However, the current PR diff still sets `sendTarget.hasText = true` whenever a target is extracted, so media-only sends would still suppress the following final text reply.

## Best Possible Solution

Keep the PR open and request the small targeted update already suggested in review: add `hasText` to `MessagingToolSend`, compute it from `content` or `message`, keep `pendingMessagingTexts` tracking outside the `sendTarget` guard for `sessions_send`, set `sendTarget.hasText = hasText`, skip suppression when the matching target has no text, and add focused regression tests for media-only same-target sends preserving the final text reply.
