## Summary

Keep open. Current main still treats TUI Ctrl+C as clear/warn/exit only, with Esc and /abort as the active-run abort paths. It does not implement this PR’s active-run Ctrl+C abort priority, stale exit-confirmation reset, abort-pending handling, stale-run handling, or aborted partial-text preservation. The PR diff is scoped to TUI code and tests; the security review found no supply-chain or sensitive-surface changes.

## Best Possible Solution

Keep this PR open until maintainers decide the intended TUI Ctrl+C behavior. If Ctrl+C should abort active runs, land this PR or an equivalent focused patch in src/tui/tui.ts, src/tui/tui-session-actions.ts, and src/tui/tui-event-handlers.ts with the included regression coverage and updated TUI docs. If Ctrl+C should remain clear/exit only, supersede this PR in favor of the narrower active-run hint path tracked by #38502/#38501.
