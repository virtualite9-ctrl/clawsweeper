## Summary

Current main has partial Groq support through docs, provider metadata, reasoning compat, and a media-understanding provider, but it still does not implement this PR’s intended Groq text-inference onboarding/catalog path. Groq has no provider auth choice, no `--groq-api-key` manifest flag, and no bundled model catalog/runtime catalog hook on main. The PR remains useful as a feature request/implementation candidate, but the existing diff is stale against the plugin-owned provider architecture and includes unrelated security-sensitive baseline/allowlist churn.

## Best Possible Solution

Keep this PR open for maintainer/contributor rework, or replace it with a fresh PR against current main. The implementation should live in `extensions/groq`: add a plugin-owned Groq text provider catalog, manifest `providerAuthChoices` for `groq-api-key`/`--groq-api-key`, default model config through the current provider onboarding helpers, and focused tests for picker/CLI/non-interactive onboarding. Drop the unrelated `.secrets.baseline` and allowlist-comment churn unless it is handled as a separate security-maintenance change.
