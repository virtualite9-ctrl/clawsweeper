## Summary

Keep open. Current main has not implemented the Anthropic 1M-context GA migration: the active bundled Anthropic plugin still injects `context-1m-2025-08-07` for `context1m: true`, still strips that beta for OAuth-style tokens, and docs/tests still describe the pre-GA beta-header behavior. The PR also targets an older core file path, so the useful work needs to be rebased onto the current plugin architecture and should address the OAuth beta-header concern raised in review.

## Best Possible Solution

Keep this PR open as a stale but plausible implementation candidate. Rebase or rewrite it against `extensions/anthropic/stream-wrappers.ts`, update plugin-level tests and public docs, preserve `context1m` as a backward-compatible context-sizing opt-in, strip user-configured legacy `context-1m-2025-08-07` safely, and explicitly verify OAuth beta-header behavior before choosing this PR or related #45592 as the implementation path for #45550.
