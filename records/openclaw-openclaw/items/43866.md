## Summary

Keep this issue open. Current main still strips `accountId` at the direct `/hooks/agent` payload boundary, mapped hooks have no `accountId` field, dispatch builds `CronJob.delivery` without `accountId`, and replay scope omits account selection. Downstream cron already supports `delivery.accountId`, so the remaining bug is the hook bridge. There is also an open same-author implementation candidate, #43951, using closing syntax, so the issue should stay open until that PR is reviewed and merged or replaced.

## Best Possible Solution

Review and land or replace #43951 with a focused current-main patch. The fix should add `accountId` to the direct hook payload and mapped hook contracts, normalize and trim it, include it in replay/idempotency scope, propagate it into `CronDelivery.accountId` during dispatch, update public hook docs, and add regression coverage proving `/hooks/agent` reaches isolated cron delivery with the selected account.
