## Summary

Keep this PR open. Current main partially addresses Bedrock context windows with a built-in known-model table, but it does not implement the PR’s intended `modelContextWindowOverrides` discovery config surface. The PR is a plausible implementation candidate for open #73328, but it should be iterated before merge because the proposed config knob is not wired through the plugin manifest/schema/docs and one cache-key review concern remains.

## Best Possible Solution

Keep the PR open for maintainer review and request a complete Bedrock plugin-owned implementation: wire `modelContextWindowOverrides` through `extensions/amazon-bedrock/discovery.ts`, `BedrockDiscoveryConfig`, `extensions/amazon-bedrock/openclaw.plugin.json`, UI hints/docs, and config-validation or manifest-shape tests; sort override entries for deterministic cache keys; then run targeted Bedrock extension tests and the appropriate changed gate. If maintainers prefer table maintenance over a regex override knob, close or redirect this PR only after a separate landed fix fully covers the linked Bedrock context-window issue.
