## Summary

Keep this issue open. Current main still parses Nextcloud Talk webhook `object.content` as an opaque string and mention-gates group traffic with regex matching against that raw text, so `{mention-userN}` rich-object placeholders are not resolved into explicit mention evidence. Nextcloud Talk is also documented and packaged as bundled in-repo plugin behavior, not an optional ClawHub-only request.

## Best Possible Solution

Fix this in `extensions/nextcloud-talk`: parse JSON rich-object `object.content`, preserve `parameters`, resolve `{mention-userN}` placeholders against the configured bot identity or a new explicit bot-user setting, pass explicit mention facts into the mention gate, and add regression tests proving `requireMention: true` accepts bot placeholders while rejecting non-bot placeholders. #49869 can be reviewed as a rich-object parsing starting point, but this issue should close only after the mention-gating behavior lands on main.
