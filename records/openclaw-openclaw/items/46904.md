## Summary

Keep this PR open. Current main still implements the behavior this PR is trying to change: `message` tool calls with `action: "send"` and real poll-creation params are rejected, while this PR would strip those params and allow the send. Main only has the narrower zero/default poll-param fix, so the PR is not obsolete or implemented on main.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to rebase or reimplement the same runtime sanitizer on current main, with regression coverage for camelCase, snake_case, mixed, true/string/non-zero poll params on `action: "send"`, while leaving `action: "poll"` validation unchanged. Treat #41199 as the broader canonical issue and handle the separate `sessions_send` conflict through its own PRs.
