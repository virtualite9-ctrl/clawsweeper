## Summary

Keep open. Current main still does not expose OPENCLAW_SESSION_KEY or OPENCLAW_AGENT_ID as child-process environment variables for exec. The trusted session and agent context reaches the exec tool, but gateway, sandbox, PTY/child, and node-host paths only pass the existing env plus OPENCLAW_SHELL=exec or send sessionKey/agentId as metadata, not env visible to the child. Related PR #68859 is closed unmerged and would not have covered the full requested behavior.

## Best Possible Solution

Keep this issue open for a focused core exec-runtime change. Add a shared trusted runtime-env helper that injects OPENCLAW_SESSION_KEY and OPENCLAW_AGENT_ID from the exec tool context after host/sandbox env construction and sanitization, apply it consistently to gateway, sandbox/Docker/backend, PTY/child, approval-delayed gateway runs, and host=node system.run execution, define precedence over inherited/requested values, update exec/environment docs, and add regression tests for all affected paths.
