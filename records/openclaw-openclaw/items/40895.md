## Summary

Keep open. Current main still treats bot-owned Discord autoThreads and bound Discord thread sessions as implicit mention contexts, so follow-up thread messages can bypass `requireMention: true`; the proposed thread-reply mention policy/config is still not present. The related #41072 PR was closed unmerged and explicitly scoped away from mention gating, while #52568 is an overlapping open enhancement rather than a confirmed canonical replacement.

## Best Possible Solution

Keep this issue open and fix the Discord plugin behavior explicitly. A conservative implementation would add a Discord thread-reply mention policy under guild/channel config, wire it through config types, strict Zod schema, generated metadata/help and Discord docs, then add regression coverage for bot-owned autoThreads and bound thread follow-ups with `requireMention: true`. If maintainers prefer #52568 as the narrower product-shape tracker, first mark that as canonical and then close this issue as duplicate; do not close this issue before that decision because the current bug remains observable in main.
