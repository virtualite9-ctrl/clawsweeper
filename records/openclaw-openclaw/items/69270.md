## Summary

Keep this PR open. Current main still lacks the central coordinated fix: reset and compaction hook contexts do not consistently carry `messageProvider`, zero-token compaction is still ignored in core session accounting, and the paired same-author issue #69269 remains open with this PR as its implementation candidate.

## Best Possible Solution

Keep this PR open for normal maintainer review. Land a version that centralizes hook provider resolution, preserves intentional internal/synthetic provider identities, carries stable `messageProvider` through compaction and reset hooks, and fixes non-negative token/cost accounting with focused regression tests. Close #69269 and the related closed duplicate reports only after the chosen fix merges and ships.
