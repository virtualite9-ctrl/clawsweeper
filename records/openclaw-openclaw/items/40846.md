## Summary

Current main still does not implement automatic macOS idle-sleep prevention for the Gateway. The only current caffeinate references are manual docs and exec-wrapper security handling, not Gateway lifecycle behavior. This PR is still a plausible implementation candidate, but it needs revision before merge because the submitted patch adds a macOS Gateway startup command-execution path, resolves caffeinate through PATH, has an acknowledged async spawn-error gap, and adds no focused tests.

## Best Possible Solution

Keep this PR open for revision or replacement. The best implementation is a small macOS Gateway lifecycle helper that starts a trusted caffeinate idle-sleep assertion only while the Gateway process is alive, attaches error and exit handling, cleans up in the existing finally path, and adds focused tests for Darwin/no-op platforms, spawn failure, and shutdown cleanup. Maintainers should also decide whether this should be default behavior, documented manual guidance, or configurable.

## Risks / Open Questions

- The submitted patch adds a Gateway startup command-execution path while the PR body says the command/tool execution surface did not change.
- The patch resolves caffeinate through PATH; in launchd or app contexts a restricted or polluted PATH can make behavior fragile or less trustworthy than an explicit macOS binary path or deliberate resolution strategy.
- The patch lacks an error listener for child_process spawn errors, so missing or inaccessible caffeinate can still crash startup instead of degrading gracefully.
- Preventing idle sleep by default is user-visible power-management behavior and may need maintainer product judgment, docs, and possibly an opt-out.
- No focused tests cover platform gating, child error handling, or cleanup of the caffeinate helper.
