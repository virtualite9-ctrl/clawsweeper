## Summary

Keep this PR open. Current main still builds Discord debounced synthetic messages by spreading the last Carbon message into a plain object without materializing mention metadata, while preflight still relies on mentionedUsers, mentionedRoles, and mentionedEveryone for mention gating. The PR addresses the right bug and the author’s follow-up commit addressed the Greptile roles/everyone review, but it targets the old src/discord layout rather than the current extensions/discord plugin path and still needs porting plus regression coverage.

## Best Possible Solution

Keep this PR open as useful bug-fix work. Port the all-entry mention accumulation into extensions/discord/src/monitor/message-handler.ts, preserving mentionedUsers, mentionedRoles, and mentionedEveryone on the synthetic message, then add a focused debounce/preflight regression test where the mention metadata appears before the last debounced entry.
