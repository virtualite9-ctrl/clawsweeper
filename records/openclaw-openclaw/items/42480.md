## Summary

Keep this PR open. Current main still creates Discord debounced synthetic messages by spreading the last Carbon message into a plain object without materializing mentionedUsers, mentionedRoles, or mentionedEveryone, while Discord preflight still gates guild/bot handling on those fields. The PR targets the pre-plugin src/discord path, so it needs porting to extensions/discord plus focused regression coverage before it can land.

## Best Possible Solution

Keep this PR open as useful bug-fix work. Port the all-entry mention accumulation into extensions/discord/src/monitor/message-handler.ts, preserving mentionedUsers, mentionedRoles, and mentionedEveryone on the synthetic message, then add a focused debounce/preflight regression test where the mention metadata appears before the last debounced entry.
