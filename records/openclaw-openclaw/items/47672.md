Current main still lacks the requested doctor pre-update progress wiring. The active `openclaw doctor` path still prints one static update note and calls `runGatewayUpdate` without the existing `createUpdateProgress` controller, while the update runner and normal `openclaw update` command already support the progress seam. This PR remains a plausible narrow fix, though maintainers should consolidate it with the overlapping open PR #47493 before landing one implementation.

After one PR lands, close the other overlapping PR as duplicate or superseded by the landed change.
