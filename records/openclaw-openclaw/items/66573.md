## Summary

Current main still has the reported bug: `sessions_history` accepts an oversized `limit`, forwards it unchanged to `chat.history`, and the Gateway rejects values above 1000 before its later hard cap can apply. The provided context also shows open PR #66650 uses closing syntax for this issue, so the issue should remain open until that candidate or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review/land PR #66650 or an equivalent narrow fix that adds the Gateway maximum to the `sessions_history` schema, clamps outgoing `chat.history` requests to 1000 before the Gateway call, and adds regression coverage for oversized caller-supplied limits.
