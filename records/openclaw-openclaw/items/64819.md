## Summary

Keep #64819 open. Current main still does not expose `openclaw agent --spawn` or an `openclaw sessions spawn` CLI path, while the internal `sessions_spawn`/subagent metadata machinery exists and the PR addresses a real core CLI isolation gap for scripted one-off agent calls. The PR needs maintainer review, rebasing/narrowing, and idempotency handling, but no conservative close reason applies.

## Best Possible Solution

Keep the PR open for maintainer judgment, but require a focused rebase that touches only the agent CLI, tests, docs/help, and any necessary subagent/session plumbing. If accepted, implement `--spawn` in the shared gateway/local CLI contract with explicit mutual exclusions, `sessions.patch` preflight failure as a hard error, deliberate operator scopes, stable retry behavior, and docs explaining when to use `--session-id`, `--session-key`, `--new-session`, or `--spawn`.
