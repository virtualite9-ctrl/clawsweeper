## Summary

Keep this issue open. Current main still lacks the central requested Discord preflight hardening: there is no opening attention-window extraction, no resolution of other configured Discord agents as competing targets, no role-label-to-agent mention handling, and no regression tests for the quoted/later mention or plain-text delegation cases. The related routing reports in the supplied context remain open or separate, so there is no canonical implemented fix to close against.

## Best Possible Solution

Implement a focused Discord plugin fix in `extensions/discord/src/monitor/message-handler.preflight.ts`: derive an opening attention window for mention matching, resolve configured Discord-bound agents that are explicitly or plainly addressed, map relevant role labels when safe, drop competing-agent/bot-loop cases, and add colocated regression tests in `extensions/discord/src/monitor/message-handler.preflight.test.ts`. Update Discord/group docs if the user-visible mention semantics change.
