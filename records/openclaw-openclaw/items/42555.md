## Summary

Keep this PR open. Current main still does not implement configurable Slack assistant thread status messages, and the PR remains a focused implementation candidate, although it is not merge-ready against the current Slack plugin layout and needs validation/config-surface decisions.

## Best Possible Solution

Keep this PR open for maintainer review or replace it with a current implementation under `extensions/slack`. The best path is a typed, validated Slack status/loading-message config surface wired through the active Slack monitor context and dispatch pipeline, with tests for default and configured behavior, docs/config metadata updates, Slack API limit validation, and a maintainer decision on whether the setting belongs under agent identity or `channels.slack`.

## Risks / Open Questions
