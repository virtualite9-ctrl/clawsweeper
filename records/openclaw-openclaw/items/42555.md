## Summary

Keep this PR open. Current main still hardcodes Slack assistant thread status to "is typing..." and lacks any typed or documented config for per-agent or Slack-channel status/loading messages. The PR is not merge-ready as-is because it targets the removed `src/slack` layout and has unresolved validation/config-shape concerns, but it remains a focused implementation candidate rather than something already shipped or clearly superseded.

## Best Possible Solution

Keep this PR open for maintainer review or replace it with a current implementation under `extensions/slack`. The best path is a typed and validated Slack status/loading-message config surface, wired through the current Slack monitor context and dispatch pipeline, with tests for default/configured behavior, docs/config metadata updates, Slack API limit validation, and a maintainer decision on whether the option belongs under agent identity or `channels.slack`.
