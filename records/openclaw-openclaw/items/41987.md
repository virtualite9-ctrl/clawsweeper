## Summary

Keep open. Current main still does not pass `agents.defaults.imageMaxDimensionPx` into the bundled browser plugin's `browser:screenshot` image result, and the PR's patch targets the old core browser-tool path rather than the current plugin architecture.

## Best Possible Solution

Keep this PR open as a valid implementation candidate, but request a port to the current browser plugin architecture: thread `ctx.getRuntimeConfig()` or `ctx.runtimeConfig` through `extensions/browser/plugin-registration.ts` into `createBrowserTool()`, call `resolveImageSanitizationLimits()` for screenshot image results, add focused tests in `extensions/browser/src/browser-tool.test.ts`, and consider covering labeled snapshot image results too.
