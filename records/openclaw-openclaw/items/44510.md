## Summary

Current main does not implement the PR’s central lock-loss protection. The open PR is a plausible core agents fix, not ClawHub work or a duplicate, and it still needs maintainer review because it changes session-lock/runtime behavior and monkey-patches private SessionManager write methods.

## Best Possible Solution

Keep this PR open for maintainer review or rework. The best core direction is a stable lock-loss path in `src/agents/session-write-lock.ts` plus `run/attempt.ts`: notify the active run when the watchdog releases its lock, abort both the run controller and active session, reject later transcript writes after lock loss, cover rewrite paths, and add focused regression tests for watchdog callbacks, lock ownership checks, lost-lock write rejection, cleanup ordering, and active-session abort behavior.
