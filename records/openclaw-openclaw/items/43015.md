## Summary

Keep open. Current main still partially matches the reported failure: shared poll creation fields are still included in the generic message tool schema, `action="send"` still rejects when poll-looking fields are non-empty/true/non-zero, and existing tests lock in that rejection. The provided GitHub context also shows open PR #46455, with closing syntax for this issue, remains an unmerged implementation candidate, so this issue should not be closed before that PR is reviewed, landed, or explicitly superseded.

## Best Possible Solution

Keep this issue open. Maintainers should review, rebase, land, or explicitly supersede PR #46455 or an equivalent patch that makes poll creation fields action-aware, strips model-populated poll/component noise from ordinary `action="send"` calls, preserves strict validation for explicit poll/component paths, and adds focused regression coverage for Telegram/Feishu/Discord send-with-file plus Discord empty-modal skeleton behavior.
