## Summary

Keep open. Current main still has the JS/TS-only skill scanner and does not include this PR's Python or shell coverage, rule guards, or regression tests. The scanner is used by OpenClaw's core install and security-audit paths, so this is a valid core hardening PR rather than ClawHub cleanup. The PR has useful follow-up work after earlier review fixes, but it still needs maintainer review/revision for the later false-negative comments before it can land.

## Best Possible Solution

Keep this PR open for revision. The best path is to land a focused core scanner hardening change that extends scannable Python/shell coverage, handles extensionless shebang shell entrypoints or explicitly documents that boundary, adds include/exclude guards without regressing JS rules, fixes the unresolved Python command-exec false negatives, and includes regression tests for the earlier duplicate/false-positive fixes plus the remaining review comments.
