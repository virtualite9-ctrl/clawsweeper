## Summary

Keep this issue open. Current main has Perplexity web_search config/env resolution, docs, and tests for the normal paths, but the reported 401 is recent and not conclusively fixed. The remaining suspect path is plausible: configured Perplexity credentials still take precedence over ambient PERPLEXITY_API_KEY, so a stale non-empty plugin config key can still shadow a fresh gateway env key.

## Best Possible Solution

Keep the issue open and reproduce on current main or the latest Homebrew release with a controlled Perplexity key across these cases: plugin literal key, plugin SecretRef, PERPLEXITY_API_KEY-only, and stale plugin key plus fresh env. Add a regression or diagnostic that proves which credential source is sent for provider=perplexity. If config-over-env precedence is intentional, update doctor/status/docs to tell users when plugins.entries.perplexity.config.webSearch.apiKey is shadowing the gateway environment.
