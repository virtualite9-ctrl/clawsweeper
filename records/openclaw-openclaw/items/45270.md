## Summary

Keep this PR open. Current main still has the same `/status` fallback that synthesizes context usage from `inputTokens + outputTokens` when `totalTokens` is missing, so a stale session with cumulative input/output can still render an inflated context line instead of unknown usage. The PR is paired with same-author issue #45268 and uses closing syntax for that issue, so the best cleanup path is to review and adapt the PR rather than close it.

## Best Possible Solution

Keep the PR open and adapt it to current main by changing `src/status/status-message.ts` to trust only fresh `entry.totalTokens` for context display, while preserving transcript fallback, then add the stale missing-total and explicit stale-total regression tests in the current status test suite. Because it closes the same author's open issue #45268, land or close the PR through normal review rather than auto-closing it as obsolete.
