## Summary

Keep this PR open. Current main still lacks meaningful parts of the PR’s intended English docs updates: `OPENCLAW_CLI=1` is implemented but not documented in the environment marker list, ACP provenance is only a terse option line, and the macOS onboarding guide still omits the remote gateway token field. The PR should be rebased/retargeted rather than closed, because the Gmail PubSub docs moved to Scheduled Tasks and the direct `docs/zh-CN/**` edit conflicts with current docs i18n policy.

## Best Possible Solution

Keep this PR open and ask for a rebase on current main. Preserve the English-source docs updates that still apply: add `OPENCLAW_CLI=1` to `docs/help/environment.md`, expand the ACP provenance option in `docs/cli/acp.md` only as far as current docs style warrants, and document the macOS remote gateway token field in `docs/start/onboarding.md`. Retarget or drop the Gmail PubSub hook-token change based on the current Scheduled Tasks page, and remove the direct `docs/zh-CN/**` edit in favor of the docs i18n/publish pipeline.
