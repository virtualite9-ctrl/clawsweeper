## Summary

Keep this PR open. Current main still trims `name` and `description` but only validates them inside truthy guards, so empty or whitespace-only values can still pass `validate_skill()`. The focused regression tests proposed by the PR are still absent, and a maintainer-side review already explicitly kept this PR open after checking the same surface.

## Best Possible Solution

Keep this PR open and review or land it, or apply an equivalent maintainer patch, so `quick_validate.py` rejects empty and whitespace-only `name` and `description` values after trimming and `test_quick_validate.py` covers those regressions. The change should stay scoped to the skill-creator validator and tests, matching the documented skill metadata contract.
