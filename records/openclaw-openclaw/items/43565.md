## Summary

Keep open. Current main still does not implement `tools.fs.allowedRoots` or an equivalent multi-root filesystem policy, while this PR is the active implementation candidate for a real core tools/config gap. The PR is security-sensitive because it intentionally widens configured filesystem boundaries across read/write/edit, sandbox/media paths, and `apply_patch`, so it should receive maintainer/security review rather than cleanup closure.

## Best Possible Solution

Keep this PR open as the canonical implementation candidate for `tools.fs.allowedRoots`. Maintainers should review the rebased diff with security focus, then either land this implementation after cleanup or replace it with an equivalent core implementation covering config types/schema/help, read/write/edit, sandbox mode, media and prompt-image loading, and `apply_patch`, with regression tests for tilde expansion, relative-root rejection, symlink/hardlink escapes, root-target deletes/writes, and pinned operations.
