## Summary

Keep this PR open. Current main already implements the basic Feishu tool-account precedence change, but it does not include the final review-driven part of this PR: threading `messageChannel` into Feishu tool-account resolution so inherited account context is only trusted for Feishu-origin sessions. That remaining cross-channel collision behavior is plausible and still useful to review or rebase.

## Best Possible Solution

Rebase or replace this PR with a narrow Feishu-plugin patch that preserves current main's explicit account > enabled contextual Feishu account > configured default behavior, then adds the missing `messageChannel` gate so contextual account IDs only override `channels.feishu.defaultAccount` for Feishu-origin tool contexts. Add targeted tests for non-Feishu channel fallback, Feishu-origin contextual routing, explicit `accountId` override, unknown account fallback, and normalized account IDs.
