## Summary

Keep open. Current main ships a meaningful subset of the field report: selector targeting on the managed act route, delayed clicks, route/client batch execution, and refs="aria" guidance. But the issue asked for seven browser automation improvements, and several requested surfaces are still missing or not fully exposed: no inspectElement/CDP event-listener helper, no captchaDetected snapshot signal, no refsStale/auto-refresh response signal, no post-fill verification, no waitForRef surface, and batch/schema/docs alignment is still incomplete.

## Best Possible Solution

Keep this issue open or have a maintainer split it into narrower browser-plugin follow-ups. Treat selector targeting, managed-route batch actions, delayed clicks, and refs="aria" as shipped, then explicitly decide the remaining surfaces: inspectElement/CDP event-listener inspection, snapshot CAPTCHA detection, stale-ref/refsStale or auto-refresh strategy, post-fill verification or slowly=true documentation, wait-by-ref semantics, and alignment of the browser tool schema plus public docs with the supported act route.
