## Summary

Keep this issue open. Current main still emits the pi-embedded `lifecycle:end` event without `aborted` or `stopReason`, while the richer command-level fallback remains skipped once the subscribe lifecycle path emits a terminal event. The related fix PR #66574 is closed unmerged, and the current tests still accept field-less terminal lifecycle payloads.

## Best Possible Solution

Extend the pi-embedded terminal lifecycle metadata path so `handleAgentEnd` can include explicit `aborted` and `stopReason` on both global `emitAgentEvent` and direct `onAgentEvent` lifecycle end payloads. Add focused regression coverage for natural completion, explicit abort, and timeout/fallback behavior, then close this issue after the fix merges.

## Risks / Open Questions
