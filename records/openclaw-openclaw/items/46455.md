## Summary

Keep this PR open. Current main only implements the narrower zero-valued poll-default fix; it still rejects message send calls with non-zero/true poll noise and still errors on non-empty Discord component skeletons such as an empty modal. The PR is a focused implementation candidate for the linked message-tool regressions, though it would need a current-main rebase/adaptation because Discord component handling now lives in the Discord plugin boundary.

## Best Possible Solution

Keep this PR open as a useful implementation candidate. The best path is to rebase/adapt it to current main: in core, ignore or strip model-populated non-question poll noise for action="send" while preserving a corrective error for explicit pollQuestion; in the Discord plugin, ignore empty component/modal skeletons without dropping valid array or container-only component payloads. Add focused regression tests for both linked failures, then close the linked issues after the fix lands.
