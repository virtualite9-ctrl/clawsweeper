## Summary

Keep open. Current main partially addresses adjacent work by hiding Discord-native `components` from the generic message-tool schema and by treating zero-valued poll defaults as unset, but it does not implement this PR's central send sanitization behavior. `message.send` still rejects non-zero/string/true poll params that models may auto-populate, and an empty Discord modal skeleton still reaches strict modal validation if supplied.

## Best Possible Solution

Keep this PR open as useful but stale work. The maintainer path should either update this PR or land a fresh patch on current main that makes poll intent action-aware, strips or ignores non-question model-populated poll noise on `message.send`, keeps explicit `pollQuestion` misuse rejected, and adds focused regression tests. For Discord components, keep the generic schema hidden and add runtime normalization or tests for empty modal skeletons only if that payload can still reach the Discord send path.
