## Summary

Keep open. Current main still delegates OpenShell exec commands into the shared SSH `/bin/sh -c` wrapper without placeholder or malformed-shell preflight, and repeated same-tool/same-args loops are still warning-only until the later global no-progress breaker. The related open loop issue #58726 covers only the hard-abort loop half; it does not supersede the OpenShell fail-fast request.

## Best Possible Solution

Keep this issue open. Implement a focused runtime hardening change: add shared exec/OpenShell preflight before SSH command construction for obvious generated placeholders and malformed shell structure; make repeated identical failed or no-progress generic calls escalate to a clear blocking result at the configured critical threshold; add regression coverage for OpenShell/shared SSH command construction and before-tool-call blocking; and treat #58726 as the narrower related tracker for the generic loop-abort portion only.

## Risks / Open Questions

- A generated-command preflight needs careful scoping so legitimate shell syntax, redirection, comparisons, command substitution, and literal angle brackets in data are not rejected incorrectly.
- A generic repeated-call hard abort should likely be failure/no-progress gated and configurable so legitimate polling or status workflows are not blocked prematurely.
