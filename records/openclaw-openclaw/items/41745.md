## Summary

Keep this PR open. Current main still writes `systemSent: true` inside `ensureSkillSnapshot()` before the LLM run, and latest release `v2026.4.26` has the same behavior. The PR is not obsolete, but it still needs maintainer review for successful payloadless, messaging-tool-only, and queued/final followup completion paths before landing.

## Best Possible Solution

Keep this PR open. The best path is to finish the focused fix by separating skill snapshot persistence from delivered-system-prompt state, then persist `systemSent: true` through a shared post-success helper used by normal replies, silent/NO_REPLY completions, messaging-tool-only completions, and queued/final followup completions, while skipping provider/meta-error outcomes. Add targeted regression coverage for failed first call, successful retry with `isNewSession=false`, payloadless success, messaging-tool success, and queued/final followup success.
