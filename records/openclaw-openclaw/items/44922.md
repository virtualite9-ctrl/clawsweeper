## Summary

Current main still matches the reported duplicate-context path: main-session cron jobs enqueue the raw system event with a `cron:` context key, and heartbeat prompt construction still wraps that same queued cron event as a scheduled reminder. The related PR #44973 is an open, unmerged implementation candidate, so this issue should remain open until that fix or an equivalent lands.

## Best Possible Solution

Keep this issue open and use it as the tracker for the main-session cron dedupe bug. The best path is to review, update, and land #44973 or an equivalent patch that marks/detects main-session injected cron events and skips `buildCronEventPrompt` for that already-present system event without changing delivery semantics.
