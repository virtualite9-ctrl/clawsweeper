## Summary

Keep this issue open. Current main has startup mitigations, but it still does not fully solve the reported `models.list` latency: the RPC awaits the full Gateway model catalog, the cold catalog path still performs provider/catalog/auth work, provider catalog hooks still run serially without a normal Gateway timeout, and a fresh issue comment reports macOS keychain lookup delays in the same path.

## Best Possible Solution

Keep the issue open. The best maintainer path is to make the Gateway model-catalog path bounded and less blocking: return configured/static rows without waiting for slow runtime provider hooks where possible, parallelize and time-box provider catalog hooks in normal Gateway operation, honor non-interactive/no-keychain intent in external auth overlay, and add regression coverage with deliberately slow provider catalog and keychain/auth-profile hooks.
