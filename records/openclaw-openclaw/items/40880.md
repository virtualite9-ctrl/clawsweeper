## Summary

Keep open. Current main still hardcodes the 5 MB media store and sandbox staging cap, and the config schema still has no top-level media.maxBytes setting. The provided GitHub context also shows open PR #40950 as an unmerged implementation candidate for this issue, so the issue should remain open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review, update, and land PR #40950 or an equivalent narrow fix. The fix should add a bounded top-level media.maxBytes config field, thread it through media store, media serving, sandbox staging, and gateway chat preflight, preserve the 5 MB default when unset, and update schema/help/docs plus regression coverage for default and configured limits.
