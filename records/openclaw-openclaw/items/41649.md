## Summary

Keep this PR open. Current main still does not implement model-change transcript archiving, model handoff creation/consumption, or even a `reset-session-models` reload action. The related narrower PR #41907 was closed unmerged, and the current PR still has unresolved functional and prompt-boundary risks that need maintainer review rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer judgment or replacement. The best path is a reviewed core gateway/session implementation that detects default model config changes, rotates or archives affected transcripts, preserves explicit user model overrides, refreshes stale model/context metadata, and injects only sanitized one-shot handoff context after a run is actually going to proceed, with focused tests for hot reload, restart reload, multi-session retention, and first-turn delivery.
