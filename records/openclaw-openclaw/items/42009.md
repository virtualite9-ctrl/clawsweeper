## Summary

Keep this PR open. Current main still resolves `openclaw agent --agent <id> --to <target>` to the selected agent's main session before the `--to`-derived key can win. The already-merged #72414 repaired the related default-agent path, but its own discussion calls this explicit `--agent + --to` path distinct. Security review found no supply-chain or CI-sensitive surface in the PR file list; it only changes session-routing source and focused tests.

## Best Possible Solution

Keep this PR open and review it as the focused fix for explicit `openclaw agent --agent <id> --to <target>` session routing. The best implementation is to keep explicit `--session-key` precedence, derive selected-agent `--to` keys before the implicit agent-main fallback, rebind qualified `--to` keys to the selected agent scope, and retain regression tests for those paths while preserving the already-merged #72414 default-agent compatibility work.
