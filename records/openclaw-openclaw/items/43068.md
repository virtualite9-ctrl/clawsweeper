## Summary

Keep this PR open. Current main still does not implement sandbox-only Python/Node research-script import validation, and the PR targets a real core exec hardening gap. The proposed patch is security-sensitive and should not merge as-is because review comments identify concrete import-parser bypasses, but that is a request-changes path rather than a cleanup close.

## Best Possible Solution

Keep this PR open and request changes. The best path is to rebase onto current main, integrate sandbox-only import-template validation after the existing shell-bleed preflight, harden parsing for multiline Node import/re-export syntax and whitespace-separated dynamic calls, handle Python bare-relative denied names, and add regression tests for sandbox-vs-host behavior plus the identified bypasses before maintainer review.
