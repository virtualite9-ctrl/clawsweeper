## Summary

Keep this issue open. Current main still does not implement Mattermost deaf WebSocket detection: the Mattermost socket only observes open/message/close/error and an update_at REST watchdog, while the generic stale-socket policy requires provider-proven transport activity that Mattermost does not publish. The open PR #57621 remains the relevant implementation candidate, not shipped behavior.

## Best Possible Solution

Keep this issue open until PR #57621 or an equivalent Mattermost-owned fix lands. The fix should add WebSocket ping/pong or another provider-proven liveness signal in extensions/mattermost/src/mattermost/monitor-websocket.ts, terminate missed-pong sockets so runWithReconnect restarts them, publish lastTransportActivityAt only from real transport evidence if using the generic health policy, and add focused monitor-websocket regression tests.
