## Summary

Current main still has the reported `openclaw doctor` UX gap. The doctor pre-update path prints one static update note and calls `runGatewayUpdate` without the existing progress controller, while `openclaw update` already uses `createUpdateProgress`. This PR remains a plausible focused fix, and the provided review discussion shows the author addressed the earlier TTY-gating and `try/finally` cleanup concerns.

## Best Possible Solution

Keep this PR open or consolidate it with #47672. The best implementation is one narrow `src/commands/doctor-update.ts` change that reuses `createUpdateProgress(Boolean(process.stdout.isTTY))`, passes `progress` into `runGatewayUpdate`, guarantees `stop()` in `finally`, and ideally adds a small regression test for progress injection and cleanup.
