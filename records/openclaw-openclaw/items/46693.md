## Summary

Keep this PR open. Current main still does not implement the PR's central behavior: configured `cacheRetention` is still filtered out before it reaches OpenAI/OpenAI-Codex stream options. The downstream OpenAI transport can emit `prompt_cache_retention: "24h"` when `cacheRetention: "long"` is supplied, and the docs now describe that behavior, but the current resolver only enables Anthropic-family and Google paths.

## Best Possible Solution

Keep this PR open or rework it onto current main by extending `prompt-cache-retention.ts` to pass through explicit `cacheRetention` for `openai` and `openai-codex` without adding defaults, then add focused tests proving configured OpenAI/OpenAI-Codex retention reaches stream options and direct OpenAI `cacheRetention: "long"` produces `prompt_cache_retention: "24h"`.
