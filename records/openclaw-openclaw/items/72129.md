## Summary

Keep this PR open. Current main still ships the Hugging Face plugin as a chat/model-provider plugin only, and the docs still state that text-to-image is outside the built-in Hugging Face provider. The PR is a recent, focused implementation on an existing plugin/provider capability seam, so it is not implemented on main, not obsolete, and not a conservative automatic ClawHub close without maintainer product judgment.

## Best Possible Solution

Keep this PR open for maintainer review. If accepted, land the focused Hugging Face text-to-image provider in the existing bundled Hugging Face plugin with its manifest contract, docs, tests, changelog, guarded HTTP path, model-id validation, and explicit unsupported-edit handling. If maintainers decide the capability belongs outside the repository, close it only after that explicit scope decision and point the contributor to the external plugin/ClawHub path.
