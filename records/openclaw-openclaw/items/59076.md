## Summary

Keep #59076 open. Current main still has no targeted handling for the reported streaming JSON.parse SyntaxError text, and the error formatting path can still fall through to returning the raw error. The linked #59118 is an open, unmerged attempt, so it does not make the issue obsolete.

## Best Possible Solution

Keep the issue open and use it as the canonical bug until a fix lands. The best path is to add a targeted streaming JSON parse error classifier in formatAssistantErrorText and sanitizeUserFacingText with regression tests for the exact V8/Anthropic messages, and evaluate whether Anthropic transport JSON.parse failures should be converted to a retryable/friendly provider-stream error instead of a raw terminal session failure. Rebase or replace #59118 against current main if its approach is still useful, but do not close the issue before a merged fix proves the behavior.
