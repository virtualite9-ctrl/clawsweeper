## Summary

Keep this PR open. Current main still has the no-callback gap the PR targets: reasoning streaming is enabled only when `reasoningMode === "stream"`, reasoning is showable, and `onReasoningStream` is a function; `emitReasoningStream()` also returns before emitting the `thinking` agent event when that callback is absent. Gateway `chat.send` still does not supply `onReasoningStream`, while the gateway event path would forward non-tool agent events once emitted.

## Best Possible Solution

Keep this PR open as a plausible focused fix candidate. The best maintainer path is to review or rebase it into a tested patch that decouples gateway `thinking` event emission from the optional channel `onReasoningStream` callback, preserves the current suppression/visibility guards, and adds focused tests for `reasoningMode: "stream"` with `onAgentEvent` but no `onReasoningStream`. Coordinate with related open work such as #61469 and #52642 rather than closing this PR as implemented.
