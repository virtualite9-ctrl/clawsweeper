## Summary

Keep this PR open. Current main still does not implement the PR's central behavior: unknown config keys remain strict-schema validation failures on the read/startup path, and gateway startup still throws on invalid config snapshots. The PR remains a plausible implementation candidate, with overlapping open tracking in #40268 and #47654 but no provided maintainer signal making either canonical or superseding this PR.

## Best Possible Solution

Keep this PR open for maintainer judgment. The best path is to rebase or consolidate it with #40268 and #47654, then land one implementation that tolerates unknown keys on the read/startup path while preserving fail-closed validation for invalid known fields, legacy keys, and union/discriminator mismatches. Before merge, review the `.secrets.baseline` changes and run targeted config validation/write/migration tests plus the required changed gate for the final touched files.
