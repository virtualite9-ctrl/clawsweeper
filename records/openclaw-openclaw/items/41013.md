## Summary

Keep open. Current main still does not strip leading `<session-recap>` blocks from the shared user-message cleanup path, so this PR is not obsolete or implemented. The PR remains a focused implementation candidate for the linked UI/TUI leak in #40948, but the submitted patch still needs the existing review feedback addressed before landing.

## Best Possible Solution

Keep the PR open as the focused implementation candidate, but require a small revision before landing: add session-recap stripping in `src/auto-reply/reply/strip-inbound-meta.ts`, wire it into both full and leading-only cleanup, strip only well-formed leading blocks with a matching close tag, preserve the original string when no recap block is removed, and add regression tests for well-formed, malformed/no-close, mid-message, leading blank-line, and whitespace-preservation cases.
