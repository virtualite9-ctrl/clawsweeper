## Summary

Current main solves the reported Claude Code plugin layout use case through the supported plugin-bundle path rather than by adding recursive/glob parsing to `skills.load.extraDirs`: configured plugin load paths are discovered, manifestless Claude bundles are detected from default-layout markers such as `skills/`, and enabled bundle skill roots are merged into the normal low-precedence skill loader. The exact `skills.load.extraDirs` glob/depth option remains absent, but the stated `plugins/<name>/skills/<name>/SKILL.md` cross-tool sharing problem is covered by `plugins.load.paths` / plugin install flows.

## Best Possible Solution

Close as implemented and steer this use case to the current plugin-bundle surface: install/link the Claude/Codex/Cursor bundle or point `plugins.load.paths` at the plugin root. Keep `skills.load.extraDirs` as the flat plain-skill-root escape hatch unless a future narrower issue asks for arbitrary non-plugin recursive skill discovery.
