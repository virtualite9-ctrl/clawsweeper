## Summary

Current main partially overlaps with this PR, but does not fully implement the stated cleanup. Overview dashboard cards are now real buttons wired to tab navigation, but generic hover affordances still apply to non-clickable cards/stats/account cards, and the current test coverage does not cover overview card click or keyboard activation. The PR also carries large unrelated root artifacts and an unrelated hooks test change, so it should not be auto-closed as implemented or merged as-is.

## Best Possible Solution

Keep this PR open for maintainer judgment, but do not land it as-is. The best path is a focused UI-only update against current main: either remove generic hover affordances or introduce explicit interactive card modifiers consistently, preserve the current `overview-cards.ts` button navigation pattern, add targeted click and keyboard activation tests, and drop the unrelated Spryng/routing artifacts plus the unrelated hooks test change.
