## Summary

Keep this PR open. Current main has implemented several adjacent token-usage fixes, but it still does not implement the PR's central mixed-alias behavior: an earlier zero `input_tokens` value still prevents a later positive `prompt_tokens` value from being used, and the current CLI output parser still lacks `prompt_tokens`/`completion_tokens` aliases. The submitted PR also needs an update for current main and the reviewer-noted JSONL zero-overwrite risk before it is merge-ready.

## Best Possible Solution

Update this PR or replace it with a fresh patch against current main that adds positive-before-zero alias selection in `src/agents/usage.ts`, adds `prompt_tokens`/`completion_tokens` alias support in `src/agents/cli-output.ts`, and includes regression tests for both the mixed `input_tokens: 0` plus `prompt_tokens: N` case and JSONL streams where later all-zero usage must not overwrite earlier nonzero usage.
