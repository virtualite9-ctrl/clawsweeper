## Summary

Keep open. Current main still has the reported behavior: text-only isolated cron delivery checks active descendants and waits with the full cron timeout without consulting `deliveryBestEffort`, then suppresses delivery while descendants remain active. The related fix PR #46160 was closed unmerged, and the existing tests cover adjacent behavior but not the reported best-effort + active-subagent interaction.

## Best Possible Solution

Fix `src/cron/isolated-agent/delivery-dispatch.ts` so best-effort isolated cron text delivery does not wait for or suppress active fire-and-forget descendants: gate the descendant wait and active-descendant early return on `!params.deliveryBestEffort`, then add a regression test in `src/cron/isolated-agent/delivery-dispatch.double-announce.test.ts` asserting that `deliveryBestEffort: true` with active descendants does not call `waitForDescendantSubagentSummary` and delivers the parent output immediately, while preserving the existing non-bestEffort wait/suppression behavior. PR #46160 is a useful reference, but it was not merged.
