## Summary

Keep open. Current main does not definitely implement this PR’s intended changes: the Google Gemini CLI OAuth reader still returns null when `gemini` is absent from PATH, and the agent workspace bootstrap code has no `WORKING.md` filename, loader entry, or tests. The related PRs in the provided context are closed but unmerged, and the current PR still has review feedback that should be addressed before any landing decision.

## Best Possible Solution

Keep this PR open for maintainer review or ask the author to rebase and split it. The Gemini fix should target the current `extensions/google/oauth.credentials.ts` implementation with restored environment state in tests. The `WORKING.md` change should be decided as an agent-bootstrap product behavior first, then implemented with the constant/type, validation, loader entry, subagent/cron semantics, docs, and tests aligned.
