## Summary

Current main still does not implement the requested read-time strip-and-warn behavior for unknown config keys. The normal validation, load, snapshot, startup, and hot-reload paths still treat unknown keys as invalid config; existing recovery is last-known-good repair, not tolerant parsing. #42201 remains the best open implementation candidate.

## Best Possible Solution

Keep this issue open. The best maintainer path is to review and either land or close #42201. If that PR lands, close this as implemented with regression coverage for startup and hot reload accepting unknown keys while strict write/config-validate behavior and invalid known-field failures remain intact. If #42201 is abandoned, implement the same behavior directly in the config read/snapshot path with focused tests for root and nested misplaced-key examples from this issue.
