## Summary

Keep this issue open. Current main still accepts Telegram direct-message `tools` / `toolsBySender` config and applies the same direct config for prompts/access, but the mounted tool-policy path remains group/channel-oriented and does not resolve `telegram:direct` sessions as per-DM tool-policy candidates.

## Best Possible Solution

Keep this issue open. The best maintainer path is to adapt Telegram's `channels.telegram.accounts.<account>.direct[peer] ?? direct["*"]` surface onto a direct-session tool-policy resolver, including `tools`, `toolsBySender`, account-scoped explicit-over-wildcard precedence, DM topic/thread fallback, native-command target sessions, and focused regression tests that prove the mounted tool inventory changes for normal Telegram DM runs.
