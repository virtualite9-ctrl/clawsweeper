## Summary

Keep open. Current main still does not provide an announce-specific cheap-model override or no-history/light-context mode for subagent and ACP completion relay turns. The existing `subagents.model` and `sessions_spawn.lightContext` controls affect the spawned child run, while completion announce paths still wake or call the requester session without `provider`, `model`, `promptMode`, or `bootstrapContextMode` overrides.

## Best Possible Solution

Keep this issue open. Add a schema-backed announce policy, likely under `agents.defaults.subagents.announce` with per-agent inheritance, that resolves an allowed cheaper model through existing model catalog/auth machinery and optionally runs completion announce turns with isolated/no-history or lightweight context. Wire it through direct, queued, ACP, and descendant announce paths without changing spawned child model behavior, then add targeted tests and update the public subagent and ACP docs.

## Risks / Open Questions

- Closing now would drop a still-current cost-control gap: completion announce relay turns can still run on the requester model with requester-session history.
- Any implementation needs to preserve model allowlists, auth-profile routing, and the `allowModelOverride` trust boundary when adding an announce model override.
- A no-history or light-context announce mode needs focused coverage for direct delivery, queued fallback, descendant wakeups, ACP completion conversion, idempotency, active-requester steering, and silent/none delivery behavior.
