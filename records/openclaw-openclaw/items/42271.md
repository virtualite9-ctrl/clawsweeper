## Summary

Keep this PR open. Current main still does not implement the PR’s intended Feishu behavior: direct message media is resolved, but quoted/replied media cannot be downloaded because fetched parent messages expose only parsed placeholder content and the agent media payload is built before the parent message is fetched. The PR remains a focused implementation candidate for a bundled Feishu plugin bug, with prior live-testing notes and related reports supporting the need.

## Best Possible Solution

Keep this PR open or replace it with an equivalent focused fix. The best maintainer path is to rebase/adapt it onto current main, add `rawContent` or structured media-key data to fetched `FeishuMessageInfo`, resolve quoted/replied media after quote visibility checks pass, build the final agent media payload after all media collection, preserve existing Feishu routing/thread behavior, and add a quoted-media regression test.
