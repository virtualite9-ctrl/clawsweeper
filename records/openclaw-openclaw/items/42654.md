## Summary

Keep open. Current main still emits the combined `Current time: ... / YYYY-MM-DD HH:MM UTC` prompt line, so this PR's intended separation into `Current time:` plus `Reference UTC:` is not implemented. The PR remains a plausible focused prompt fix, but it needs a refresh because its test target is absent on current main and several current prompt-surface tests still assert the old combined format.

## Best Possible Solution

Keep this PR open or supersede it with a refreshed PR that changes `resolveCronStyleNow` to emit separate `Current time:` and `Reference UTC:` lines, rebases onto current main, and updates the current cron, session reset, post-compaction, memory-core, and related prompt-composition tests. The implementation should preserve the UTC-awareness benefit shipped by #32423 while making local time the unambiguous primary calendar anchor.
