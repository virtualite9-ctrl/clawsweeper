## Summary

Keep open. Current main still emits the combined `Current time: ... / YYYY-MM-DD HH:MM UTC` line, so this PR's intended behavior is not implemented on main. The PR is a plausible focused prompt fix, but it needs a refresh because the formatter is shared across several prompt surfaces and current tests still assert the old single-line format.

## Best Possible Solution

Keep this PR open or supersede it with a refreshed PR that changes `resolveCronStyleNow` to emit separate `Current time:` and `Reference UTC:` lines, rebases onto current main, and updates all current cron, session reset, post-compaction, memory-core, and related prompt-composition tests consistently. The earlier #32423 remains useful context: it added UTC for log/time-boundary awareness, while this PR should preserve that benefit without putting two competing calendar dates on the primary local-time line.
