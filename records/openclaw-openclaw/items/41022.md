## Summary

Keep this PR open. Current main still has the old CLI no-output watchdog defaults the PR targets, the resolver still clamps resumed runs through the 180000 ms default max, and that resolved timeout is still enforced by the CLI runner and process supervisor. The supplied PR remains a narrow implementation candidate, not obsolete cleanup.

## Best Possible Solution

Keep the PR open as a plausible implementation candidate. Rebase or reapply the constants change on current main, add a focused unit test for default fresh/resume watchdog resolution plus the global-timeout clamp, then review and land it if maintainers accept the longer default wait. Keep the null/0 disable-cap behavior as a separate product decision or follow-up.
