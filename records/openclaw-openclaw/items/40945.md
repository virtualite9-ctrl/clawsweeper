## Summary

Keep open. Current main still implements and documents the reported behavior: Control UI markdown images only render for inline data-image URIs, remote http(s) markdown images are flattened to alt text, and the Control UI CSP blocks remote image loads. The provided GitHub context also shows open PR #41025 uses closing syntax for this issue, so the issue should remain open until that implementation candidate is reviewed and merged or closed.

## Best Possible Solution

Keep this issue open and review PR #41025 or a replacement implementation after deciding the product/security policy. A complete change should update the markdown renderer, Control UI CSP, docs, and regression tests together, likely with an explicit opt-in, allowlist, managed-media proxy, or a clear maintainer decision to keep remote markdown images blocked.
