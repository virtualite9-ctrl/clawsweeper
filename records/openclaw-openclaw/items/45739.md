## Summary

Keep open. Current main has newer ACP task-completion plumbing, but it does not implement this PR’s intended gateway-state fallback inside the ACP parent relay: `startAcpSpawnParentStreamRelay` still depends on local `onAgentEvent` assistant/lifecycle delivery, and the relay tests do not cover `chat.history`/`agent.wait` backfill. The PR also has unresolved review concerns around duplicate/race behavior, so it is not safely closeable as implemented or superseded.

## Best Possible Solution

Keep this PR open for maintainer review or a current-main rework. The best path is to decide whether live `streamTo: "parent"` relay backfill is still desired alongside the task-completion model; if yes, implement it in `src/agents/acp-spawn-parent-stream.ts` with scoped `chat.history`/`agent.wait` calls, short timeouts, and regression tests for missing assistant events, missing terminal events, lifecycle-before-transcript persistence, history-first/local-second duplication, and the `primeHistoryBaseline` race. If maintainers decide task-completion delivery replaces live relay backfill, close manually with that product decision rather than as an automated implemented-on-main cleanup.
