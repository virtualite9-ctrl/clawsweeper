## Summary

Current main does not already implement this PR's intended behavior. Gateway chat item construction still only hides normalized `toolResult` messages when `showToolCalls` is off, while the default UI has a separate tool-calls toggle and current extraction does not strip `SECURITY NOTICE:` or `EXTERNAL_UNTRUSTED_CONTENT` markers. The PR is also stale against the current split `build-chat-items.ts` path and has unresolved review concerns, so it should stay open for maintainer review rather than be closed as implemented.

## Best Possible Solution

Keep this PR open as a focused but stale bug-fix candidate. The best path is to update it against current `ui/src/ui/chat/build-chat-items.ts`, decide the intended product boundary between hiding raw plumbing and showing collapsible tool cards, add targeted tests for tool role variants and assistant tool-only turns, apply history limiting after filtering visible chat items if that behavior is desired, and scope any security-marker stripping to confirmed wrapped tool-result payloads.
