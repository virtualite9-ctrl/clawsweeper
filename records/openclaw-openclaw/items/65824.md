## Summary

Keep open. Current main has several useful pieces, including plugin hook surfaces and last-known-good config recovery, but the umbrella still tracks unresolved core reliability/security/product gaps: non-verbose fallback visibility, fallback-specific bootstrap/context reinjection, config-write health rollback, doctor preview/diff mode, per-path filesystem policy, and spend/health-routing gates.

## Best Possible Solution

Keep this issue open for maintainer triage, or replace it with focused canonical issues before closing the umbrella. Treat #70976/#70990 as covering only the failover hook slice unless merged code/docs explicitly cover non-verbose fallback visibility and fallback-specific bootstrap/context reinjection. Split the remaining work by surface: fallback visibility/context reinjection, config-write health rollback, crash recovery, doctor preview/diff, per-path filesystem policy, repeated sub-agent failure circuit breaking, prompt-injection scanning policy, spend/budget gates, and scheduled model-health routing.
