## Summary

Keep open. Current main still lacks the PR's intended behavior: there is no `waiting` status-reaction state, no per-message `onQueued` queue hook, no embedded-runner threading through session/global lanes, and Telegram dispatch still moves directly into `setThinking()` without a queued/waiting callback.

## Best Possible Solution

Keep this PR open for maintainer review or an updated branch. The best path is to rebase the callback-based approach onto current main, add `waiting` to the status-reaction config/schema/docs/tests, thread `onQueued` through both session and global command lanes with a once guard, wire Telegram waiting-to-thinking transitions through the current dispatcher, and keep the account+chat reaction backoff security fix covered by tests.
