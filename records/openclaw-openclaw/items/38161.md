## Summary

Current main does not implement the PR's central changes: there is no postHookActions field/drain, no blockSessionSave/sessionSaveContent handling in session-memory, memory filenames still use YYYY-MM-DD-slug.md without a random suffix, and slug generation still calls the embedded agent without disableTools. The PR is also actively paired with open follow-up/security-hook work, so it should be reviewed as an implementation candidate rather than closed by cleanup.

## Best Possible Solution

Keep this PR open and review it as the candidate implementation for the session-memory policy-control slice of #36671. If maintainers accept the persistence-only contract, land the postHookActions API, blockSessionSave/sessionSaveContent behavior, slug disableTools hardening, filename uniqueness change, docs, and targeted tests together or request a narrower split; do not close it as cleanup while main lacks the functionality and dependent PR #38162 remains open.
