## Summary

Keep this issue open. Current main still has the original one-key-at-a-time parity assertions for missing config field labels, and I found no generator, autofix, or copy-paste-ready missing-label stub output. The related implementation candidate #46304 directly targeted this issue but is closed unmerged, so it does not resolve the remaining work.

## Best Possible Solution

Keep the issue open and implement the narrow developer workflow improvement in `src/config/schema.help.quality.test.ts`: collect all missing root/help label keys, derive conservative title-case stubs, and fail once with copy-paste-ready additions for `src/config/schema.labels.ts`. A separate generator command is optional, but the test failure path is the lowest-risk fix because it addresses CI diagnosis directly without changing runtime config behavior.
