## Summary

Keep open. Current main still does not implement the PR's intended MS Teams outbound `sendPayload` path, and core only routes interactive/channelData approval payloads through native delivery when the channel handler exposes `sendPayload`. The PR remains a focused Teams-plugin implementation candidate, with review still needed around messageBack click handling and dep-aware media sends.

## Best Possible Solution

Keep this PR open. Rebase it onto current main, retain the Teams-plugin-owned `sendPayload` implementation using the existing Adaptive Card sender and shared payload media helpers, route card-media sends through the same dep-aware outbound path as `sendText`/`sendMedia`, and validate the `/approve` messageBack click path alongside #64503 with focused outbound/inbound tests.
