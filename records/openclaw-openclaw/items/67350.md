## Summary

Keep open. Current main still lacks the central stability remediation described by this PR: no canonical cross-surface session-route resolver, health still reports an always-ok model, session write locks still use the legacy pid payload, Telegram polling stall recovery is not lane-aware, overflow compaction does not synthesize token counts, and session artifacts/Telegram sidecars still live beside active session storage. The PR also has unresolved functional and security-sensitive review findings that need maintainer review before it can land.

## Best Possible Solution

Keep this PR open as a rework candidate. The maintainer path is to rebase onto current main, split or narrow the remediation where practical, fix the health helper blocker, close the account-scoped Telegram direct-session protection gaps, justify any new public SDK runtime exports, and keep focused regression coverage for route resolution, degraded health, structured locks, Telegram poller liveness, heartbeat/task delivery protection, artifact relocation, and overflow compaction provenance.
