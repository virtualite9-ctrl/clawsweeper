## Summary

Keep this issue open. Current main still acknowledges Telegram callback queries inside the downstream callback handler after grammY `sequentialize` is installed, so ordinary inline-button callbacks for the same chat/topic can still wait behind an active turn. Main has a later approval-callback lane fix, but that only covers exec/plugin approval callback data and does not solve the general inline-button timeout described here.

## Best Possible Solution

Implement the Telegram plugin fix on current main: add a pre-sequentialize middleware in `extensions/telegram/src/bot-core.ts` that immediately answers callback queries with caught/logged failures, remove or avoid redundant handler-level acknowledgement in `extensions/telegram/src/bot-handlers.runtime.ts`, and add a regression test proving acknowledgement happens before a queued same-topic callback action is released while an agent turn is busy.

## Risks / Open Questions

- The existing approval-lane fix may reduce symptoms for approval buttons only; it does not cover normal inline button callbacks such as native command menus or plugin callback payloads that do not parse as approval commands.
- A correct fix should acknowledge callback queries before `sequentialize` without reordering the actual callback action processing, which should continue through the queued handler path.
