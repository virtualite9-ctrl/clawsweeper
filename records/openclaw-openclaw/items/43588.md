## Summary

Keep this PR open. Current main still lacks Slack `allowBots: "mentions"` support while Discord already has the tri-state behavior this PR is trying to mirror. The PR needs to be ported from the old `src/slack` layout to the current `extensions/slack` plugin layout, but the core user problem remains unsolved and the PR is still a focused implementation candidate.

## Best Possible Solution

Keep this PR open and ask the contributor to port it to the current Slack plugin layout. The implementation should update `extensions/slack/src/monitor/message-handler/prepare.ts`, `extensions/slack/src/monitor/channel-config.ts`, `src/config/types.slack.ts`, `src/config/zod-schema.providers-core.ts`, Slack UI/generated config metadata, docs/help text, and focused Slack tests for no mention, explicit mention, DM, and implicit thread participation. Maintainers should then review and run the Slack extension changed lane or `pnpm check:changed` before merge.
