## Summary

Keep this PR open. Current main does not implement the intended structuredClone replacement: the session store cache, load return path, auth-profile cache, secrets runtime snapshots, gateway secret-input credential resolution, config mutation, and config write-preparation paths still call structuredClone. The PR remains a plausible implementation candidate for the linked open memory-leak report, but it needs rebase/adaptation to current file names and focused review of JSON-clone semantics.

## Best Possible Solution

Keep the PR open as a candidate fix for the open structuredClone memory-leak report. The best path is to rebase it onto current main, adapt the renamed/refactored files, constrain the helper to JSON-owned store/config data, add targeted regression coverage or benchmark proof for the session-store cache/load and auth-profile cache paths, then review/land it or replace it with an updated canonical PR.
