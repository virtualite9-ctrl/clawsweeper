## Summary

Keep open. Current main still lacks the channel MCP stdout isolation change: `serveOpenClawChannelMcp()` does not call `routeLogsToStderr()` before constructing the channel MCP server, and that path still reaches runtime config loading that can emit config warnings. The PR remains a focused core MCP protocol fix, though its added test should be adjusted because it uses `InMemoryTransport` and does not directly exercise stdio stdout isolation.

## Best Possible Solution

Keep this PR open and either land the focused production change or supersede it with an equivalent current-main fix that routes diagnostics to stderr before the channel MCP server loads runtime config. Before merge, adjust the test so it directly covers stderr routing or real stdio stdout isolation; keep the related bug open until the implementation lands.
