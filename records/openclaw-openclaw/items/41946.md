## Summary

Keep open. Current main still does not implement the Feishu underscore-in-URL fix, and this PR is still an incomplete implementation candidate rather than obsolete shipped work. The related #41880 remains open and may be the better replacement path, but it is not merged on main.

## Best Possible Solution

Keep this PR open only as a rework candidate or replace it with a corrected Feishu-owned fix. The best path is to handle both bare URL-only payloads and markdown-link URLs in the Feishu send/edit/rendering path without corrupting parentheses or escaped characters, add focused regression coverage, then close #41860 when the corrected implementation lands. The open #41880 appears to target the same remaining bug more directly, but it still needs review/merge before this can be considered implemented on main.
