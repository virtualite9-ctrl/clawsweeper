## Summary

Keep open. Current main still does not implement the PR’s two core Nostr DM delivery fixes: `startNostrGatewayAccount()` returns immediately after startup, which the gateway supervisor treats as an exited channel and restarts, and the DM subscription still passes an array to `subscribeMany()` even though the pinned `nostr-tools` 2.23.3 `subscribeMany()` API takes a single `Filter`. The PR is still useful root-cause work, but it should not merge as-is because the Nostr gateway code moved from the older inline `channel.ts` path into `gateway.ts`, and the PR’s `sendMedia` fallback would expose raw media URLs despite current Nostr docs and capability metadata saying media attachments are unsupported.

## Best Possible Solution

Keep this PR open as a stale but useful implementation candidate. The maintainer path is a refreshed Nostr-owned patch on current main: make `startNostrGatewayAccount()` stay pending until `ctx.abortSignal` with idempotent cleanup, change the DM `subscribeMany()` call to pass the single `Filter` object required by pinned `nostr-tools` 2.23.3, add focused lifecycle and real subscription-shape regression coverage, and leave media unsupported unless a safe public-URL-only media design is explicitly added. #53858 should remain the canonical user-facing restart-loop report until that corrected fix lands.
