## Summary

Current main does not implement this PR's intended Windows PowerShell UTF-8 command wrapper. The exec runtime still passes the original command unchanged into both child-process and PTY host execution, and the current docs still describe Windows CJK mojibake as requiring a manual PowerShell workaround. The PR remains a plausible implementation candidate, but it needs maintainer review for the existing PowerShell semantic concerns before it should land.

## Best Possible Solution

Keep this PR open as a focused implementation candidate for the Windows exec encoding bug, but require a revised PowerShell-safe wrapper design and platform-independent regression tests before landing. If another PR lands the same UTF-8 exec behavior first, close this PR as superseded by that merged change and link the canonical fix.
