## Summary

Keep this PR open. Current main does not already ship a Helm chart, and the existing Kubernetes support remains the Kustomize/raw-manifest path. The PR is a plausible deployment contribution, but it needs maintainer/product review and fixes for the unresolved chart behavior and security-adjacent deployment footguns before it can be considered ready or obsolete.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is either to revise the chart so it exactly preserves the current Kubernetes object/data contract, uses current image/version policy, adds Helm render-time guards and rollout annotations, documents loopback exposure clearly, and adds chart validation, or to have a maintainer explicitly decide that Helm should live outside the core repo despite the existing Kubernetes docs saying it can be layered on the manifests.
