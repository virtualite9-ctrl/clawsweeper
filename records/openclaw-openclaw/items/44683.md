## Summary

Keep this PR open. Current main still has only the Kustomize/raw-manifest Kubernetes path, and the public Kubernetes docs explicitly explain that OpenClaw does not currently ship a Helm chart. The PR is a plausible deployment contribution, but it is not implemented on main, not clearly superseded, and not a ClawHub/plugin item. It also has unresolved functional and security-sensitive review issues, so it should not be closed as already handled.

## Best Possible Solution

Keep the PR open for maintainer/product judgment. If OpenClaw wants a supported Helm chart, update the Kubernetes docs to remove or revise the current `Why not Helm?` stance, add the chart under an owned support path, address the unresolved template review findings, document secret-handling expectations, and add chart validation. If maintainers prefer external ownership, close it explicitly as a product decision or direct it to an external chart repository rather than treating it as implemented or stale cleanup.
