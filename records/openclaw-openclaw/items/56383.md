## Summary

Keep open. Current main still does not implement cursor-based chat.history pagination, the Web UI still does one fixed history load, and the underlying WebChat freeze issue remains open. This PR is also open, unmerged, dirty against main, and has unresolved post-fix review concerns around byte-budget truncation and untimestamped history records, so it is not a conservative cleanup close.

## Best Possible Solution

Keep this PR open for maintainer review or a revised successor. The best path is to land one coherent chat.history pagination implementation that documents the additive before/cursor/hasMore contract, updates generated Swift clients and Web UI load-older/rendering behavior, and includes regression tests for strict cursor progression, byte-budget truncation, duplicate timestamps, and untimestamped/imported transcript records. Coordinate with #72178 and the underlying #48514 performance issue before choosing which implementation to merge.
