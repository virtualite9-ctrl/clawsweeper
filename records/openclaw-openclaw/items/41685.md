## Summary

Current main still does not implement the requested Control UI behavior. The Gateway sessions.list path builds its result from the combined Gateway store, that store intentionally includes discovered disk-only agent stores, and the Control UI calls sessions.list without filtering to configured agents. Current code also still preserves explicit session keys from URL/browser state except for default-agent aliases, so the broader restore-loop variant from the discussion remains plausible. The related direct fix PR #41687 is closed unmerged, while current tests and docs now explicitly preserve broader Gateway/ACP disk-only discovery.

## Best Possible Solution

Keep this issue open and make a deliberate product/API split: the default Control UI Sessions tab should hide unknown/deleted agent namespaces or request a configured-only listing, while ACP/disk-only discovery should remain available through an explicit includeDiskOnly/includeAcp-style parameter or separate surface. Also normalize or reject URL/browser-restored agent:<id>:* keys when <id> is not configured, falling back to sessionDefaults.mainSessionKey.
