## Summary

Keep this PR open. Current main still has the ASCII-only normalizeHyphenSlug implementation, buildGroupDisplayName still depends on that helper for group display tokens, and the Unicode regression tests proposed by this PR are not present. The PR is also paired with same-author open bug #58932 via closing syntax, so conservative cleanup should not close the implementation candidate before maintainers review or land it.

## Best Possible Solution

Review and land this PR or an equivalent focused patch that updates normalizeHyphenSlug to preserve Unicode letters, marks, and numbers, with regression tests for CJK, Cyrillic, Arabic, mixed-script labels, and combining-mark scripts such as Devanagari. If maintainers prefer a narrower product behavior, implement the Unicode preservation in the group display-name path while documenting the slug contract and then close the paired bug after merge.
