## Summary

Keep open. Current main does not expose `api.runtime.gateway.dispatchMethod` or an equivalent sanitized `chat.history` runtime helper to plugins. The PR addresses a real plugin runtime API gap, but the proposed broad gateway-RPC dispatcher also needs maintainer API/security review before it can be treated as the best fix.

## Best Possible Solution

Keep this PR open for maintainer review or replace it with a narrower plugin runtime contract. The likely best path is either a constrained `api.runtime.gateway.dispatchMethod` with explicit scope/owner rules, tests, and docs, or a dedicated sanitized history helper that calls `chat.history` semantics without giving plugins arbitrary gateway RPC dispatch.

## Risks / Open Questions

- The PR's broad `dispatchMethod(method, params)` API could let plugin code invoke arbitrary classified gateway read/write RPCs in-process; this needs explicit maintainer security/API design rather than automated closure.
- A narrower runtime helper for sanitized session history may be safer than exposing the whole gateway method bus, but that product/API choice is not implemented on main.
