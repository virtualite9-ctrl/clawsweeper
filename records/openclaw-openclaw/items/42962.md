## Summary

Current main still does not implement this PR's intended suppression. The runtime still classifies `gateway` `config.schema.lookup` as mutating, and the payload warning policy has no exact suppression for the benign `config schema path not found` miss, so landing this PR unchanged would likely remain incomplete without the classifier work tracked in #42898.

## Best Possible Solution

Keep this PR open as a useful but incomplete implementation candidate. The best fix is to combine or coordinate it with #42898: classify compound read-only gateway actions by their leaf verb, then suppress only `gateway` `config.schema.lookup` misses whose error is exactly `config schema path not found`. Add regression coverage for the real path from `buildToolMutationState` through `lastToolError` into payload rendering, plus inverse cases proving mutating gateway failures and other gateway errors still warn.
