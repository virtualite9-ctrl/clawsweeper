## Summary

Keep this issue open. Current main still supports configured agent avatar display and browser-local Quick Settings avatar overrides, but it does not implement an Agents-tab avatar upload flow that writes image bytes under an agent workspace and updates IDENTITY.md.

## Best Possible Solution

Implement a first-party Control UI agent avatar upload path. The likely maintainer path is an additive admin-scoped gateway method or authenticated endpoint with a typed protocol schema for image bytes/metadata, safe workspace-relative storage under avatars/, IDENTITY.md updates through existing identity helpers, Agents-tab picker wiring, avatar refresh behavior, and focused gateway/UI regression coverage.
