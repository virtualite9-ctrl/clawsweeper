## Summary

Keep this issue open. Current main still supports configured avatar display through `/avatar/<agentId>` and a browser-local Quick Settings assistant avatar override, but it does not implement the requested Agents-tab upload flow that stores image bytes under an agent workspace and updates `IDENTITY.md`.

## Best Possible Solution

Implement a first-party Control UI agent-avatar upload path: an additive admin-scoped gateway method or authenticated endpoint with typed protocol schema for image bytes and metadata, safe workspace-relative storage under `avatars/`, `IDENTITY.md` updates through existing identity helpers, Agents-tab picker wiring, avatar refresh behavior, and focused gateway/UI regression coverage.

## Risks / Open Questions

- Adding this remains security-sensitive: the implementation should validate MIME/type and size, generate safe filenames, write only within the selected agent workspace `avatars/` directory, reject symlink/hardlink/path traversal escapes, and refresh identity/avatar metadata after updating `IDENTITY.md`.
- The existing `agents.files.set` text-file editor is not an adequate substitute for the requested binary/avatar-specific upload flow.
