## Summary

Current main still treats a missing extensionless backup output path as the final archive filename, so this PR is not obsolete. The PR is stale against the current implementation location and needs rebase, tests, and docs/help contract cleanup before it can land.

## Best Possible Solution

Keep this PR open for maintainer review. Rebase the useful output-path behavior onto `src/infra/backup-create.ts`, preserve existing containment/no-overwrite protections, document the `--output` path classification in CLI help and `https://docs.openclaw.ai/cli/backup`, and add focused regression tests for missing extensionless directories, explicit archive filenames, `.tar`, custom suffixes such as `.bak`, and non-ENOENT `fs.stat` errors.
