## Summary

Keep this PR open. Current main has absorbed some Telegram work, but the batch is not definitely implemented on main and is not safely obsolete; several requested fixes are still missing or only partially covered in the current `extensions/telegram` plugin layout.

## Best Possible Solution

Keep this PR open for maintainer handling, but do not merge it as-is. Split and port the still-relevant fixes into `extensions/telegram`, prioritizing getFile failure surfacing, command-menu cache invalidation, mention-gating verbose logging, broader media-group item skipping, and the draft-stream stop race guard. Preserve already-absorbed current behavior for reply parameters, ACP placement, word-boundary chunking, proxy transport, and conservative non-idempotent retry handling.
