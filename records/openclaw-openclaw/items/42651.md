## Summary

Keep open. Current main still has substantial memory infrastructure, but the default memory-core surface does not implement the requested MVP command set or structured recall explanations. Optional/plugin paths such as memory-lancedb and the Engram Core comment show useful adjacent work, but they do not satisfy the default OpenClaw memory-core CLI/skill contract, and the same-author related #42650 remains open for overlapping review/edit/forget state-model work.

## Best Possible Solution

Keep this issue open and narrow the v1 operator contract. A maintainer should decide whether the MVP is implemented as memory-core subcommands, a bundled skill surface, plugin tools, or an explicit ClawHub/plugin direction. If it stays in core, map remember to provenance-preserving writes, recall to search output with provenance and match explanations, and review/edit/forget to the same tombstone/superseded/conflict state model tracked by #42650.
