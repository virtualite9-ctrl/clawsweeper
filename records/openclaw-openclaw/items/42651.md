## Summary

Keep open. Current main has substantial memory infrastructure, including memory-core search/index/promote/dreaming/backfill, Active Memory auto-recall, memory-lancedb tools, and memory-wiki provenance workflows, but it still does not provide the requested default MVP operator surface: first-class remember/recall/review/edit/forget commands, review/edit/forget state handling, or structured per-result match explanations in default recall output. The Engram Core comment is useful adjacent external work, but it does not satisfy the default OpenClaw memory-core contract, and same-author #42650 remains open for overlapping review/edit/forget state-model work.

## Best Possible Solution

Keep this open and narrow the v1 memory operator contract. A maintainer should decide whether the MVP lands as memory-core subcommands, a bundled skill/plugin, or an explicit ClawHub/plugin direction. If it stays in default memory-core, implement remember/recall/review/edit/forget around provenance-preserving writes, search output with per-result match explanations, and review/edit/forget lifecycle states coordinated with #42650.
