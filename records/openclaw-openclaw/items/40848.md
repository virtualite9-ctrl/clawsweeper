## Summary

Keep this issue open. Current main still does not expose Telegram Bot API `entities` or `caption_entities` through the shared `message` tool schema, Telegram CLI send flags, Telegram action runtime, or Telegram outbound send path, so custom emoji via `custom_emoji` entities remains an actionable bundled Telegram plugin feature gap.

## Best Possible Solution

Keep this open and implement it in the bundled Telegram plugin. Add Telegram-owned message-tool schema fields for outbound `entities` and likely `caption_entities`, validate them against a narrow Telegram `MessageEntity` shape, thread them through `extensions/telegram/src/action-runtime.ts` into `TelegramSendOpts`, and send `entities`/`caption_entities` instead of `parse_mode` only on compatible text/caption payloads. Add regression coverage for schema exposure, runtime forwarding, and Bot API params for a `custom_emoji` entity.
