## Summary

Keep this PR open. Current main does not implement the requested delayed textual acknowledgment for slow Feishu DMs, so it cannot be closed as implemented. The PR is a focused Feishu-only change and the author addressed part of the review feedback, but remaining review concerns mean the best path is revision/review rather than cleanup closure.

## Best Possible Solution

Keep the PR open and revise it against current main. The best implementation is a Feishu-owned delayed acknowledgment that is DM/private-only, cancels as soon as any user-visible reply or streaming/status output is delivered, keeps final reply flow unchanged, and includes deterministic slow, fast, group, and early-visible-output regression tests.
