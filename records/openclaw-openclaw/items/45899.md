## Summary

Keep open. Current main already has broader `MEDIA:` local-path documentation, but it does not implement this PR's Feishu-specific blocked-local-media hint or one-time final-reply fallback. The Feishu reply dispatcher still only logs the reply error, closes streaming, and clears typing state on `onError`. The PR is not superseded by current main; it should be rebased and reconciled with the newer docs and related Feishu mediaLocalRoots work.

## Best Possible Solution

Keep this PR open for maintainer review. Rebase it onto current main, keep the newer documentation model, and decide whether to land the narrow Feishu runtime fallback/hint alongside or after the related Feishu mediaLocalRoots plumbing.
