## Summary

Keep open. Current main still matches the reported Telegram non-streaming tool-progress leak, and #72482 is an open, unmerged implementation candidate for this issue. Structured reasoning handling exists in some provider paths, but the broader WebSocket/chat surface still has no chat-level reasoning chunk, so the GLM/MiniMax reasoning part is not clearly resolved.

## Best Possible Solution

Keep this issue open while maintainers review #72482. The immediate fix should suppress default tool-progress messages for Telegram non-streaming turns and update the current Telegram test that asserts the opposite. The reasoning leak should be handled through a provider/chat transport contract that preserves reasoning as a separate structured chunk, or be tracked under #41494/#54496 or a new canonical issue if #72482 lands only the Telegram tool-progress portion.
