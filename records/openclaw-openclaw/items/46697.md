## Summary

Keep open. Current main still delivers the A2A announce reply to the target channel/thread, not the full target reply requested by this PR. The PR addresses a real remaining behavior gap, but its current head needs rework before merge because it bypasses ANNOUNCE_SKIP suppression and is stale against the current A2A helper structure.

## Best Possible Solution

Keep this PR open, but require a rework against current main: preserve the current sessionsSendA2ADeps.callGateway and resolveNestedAgentLaneForSession structure, deliver the post-ping-pong latestReply only after the suppression policy is honored, avoid an unnecessary announce model run when its output cannot be delivered, and add targeted tests for no-ping-pong full reply, ping-pong final reply, ANNOUNCE_SKIP suppression, and threadId/accountId passthrough.
