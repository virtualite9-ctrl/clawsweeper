## Summary

Current main still does not implement the requested Chrome OptGuide/WasmTts model-download suppression. The managed browser launcher still disables only Translate and MediaRouter, the headless branch only adds headless/GPU flags, the sandbox browser entrypoint has no `--disable-features` argument, and config exposes only generic `browser.extraArgs` rather than a default or dedicated model-download switch. The related PR #41213 targeted this issue but is closed unmerged, so it is not a superseding implementation.

## Best Possible Solution

Keep this issue open. The likely fix belongs in the bundled browser plugin plus sandbox browser entrypoint: centralize the disabled Chrome feature list, add the OptGuide/WasmTts model-download suppression for managed headless/server launches and sandbox launches, update the public browser/sandbox docs, and add regression coverage for `buildOpenClawChromeLaunchArgs` plus a lightweight check that the sandbox entrypoint defaults include the intended feature list.

## Risks / Open Questions

- `browser.extraArgs` can be used as a local managed-browser workaround, but it does not satisfy the requested default headless/server behavior and does not cover the sandbox browser entrypoint.
- The sandbox docs and entrypoint are already out of sync around `--disable-features=TranslateUI`; a fix should reconcile both surfaces.
- Before shipping, maintainers should verify the current Chromium feature flag names and merge them into the existing `--disable-features` value rather than appending competing duplicate flags.
