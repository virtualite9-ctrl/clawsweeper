Keep this issue open. Current main still contains both reported parameter-conflict behaviors: `sessions_send` rejects mixed `sessionKey` + `label`, and `message` rejects `action: "send"` when non-zero poll creation parameters are present. The provided GitHub context also lists open implementation candidates, including #41255 and #59324 for the `sessions_send` portion and #46904 for the message poll-param portion, so closing before those are reviewed or superseded would be premature.
