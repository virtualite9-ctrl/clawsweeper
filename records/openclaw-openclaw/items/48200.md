## Summary

Keep this PR open. Current main has iMessage echo, self-chat, and reflected-content guards, but it still does not implement the PR's explicit account-handle config, startup validation, or pre-DM-policy sender drop. The related open #63666 is only a reflected assistant-content test and does not supersede this account-handle behavior.

## Best Possible Solution

Keep the PR open as a focused implementation candidate. The best path is to rebase it onto current main, preserve the existing #63980 self-chat fixes, add the explicit accountHandle config/schema/docs surface, validate the handle against effective allowFrom/store allowlists at startup, drop is_from_me=false self-echoes before dmPolicy=open access, and keep regression coverage for normalization, dmPolicy=open, self-chat, and rate-limiter behavior.
