## Summary

Keep open. Current main has a provider-owned last-assistant-preserving helper and several narrower replay fixes for Anthropic/Bedrock thinking-block errors, but it does not implement the requested user-facing `thinkingHistory: last-only | strip | keep` policy or broad token-saving behavior for all extended-thinking sessions. Newer Claude policies intentionally preserve thinking blocks, so closing this would erase a real remaining product/config decision.

## Best Possible Solution

Keep this issue open as a focused feature request. If maintainers want the knob, add an explicit agent or gateway transcript-history policy such as `last-only | strip | keep`, thread it through `resolveTranscriptPolicy()` and provider replay-policy merging, document the interaction with Anthropic/Bedrock prompt caching and tool continuations, and add tests showing provider-safe defaults plus explicit operator overrides.
