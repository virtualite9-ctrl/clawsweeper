## Summary

Keep this issue open. Current main still uses signal-driven TTL refreshes, not a lifecycle-owned active-run presence lease, so a long quiet active run can still lose configured typing before run completion plus dispatch idle. The provided timeline also shows open PR #71828 as the targeted implementation candidate, so the issue should stay open for maintainer review rather than be closed early.

## Best Possible Solution

Keep this issue open and review/land PR #71828 or an equivalent narrow fix. The implementation should refresh internal run-side liveness on keepalive ticks or introduce an explicit active-run presence lease, preserve stale cleanup, avoid new default progress/disclosure surfaces, avoid increasing provider send cadence, and stop only after run completion plus reply dispatch idle.
