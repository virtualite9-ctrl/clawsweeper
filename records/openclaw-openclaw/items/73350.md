## Summary

This issue should stay open. The report is recent, has concrete logs and a local `openclaw infer model run --local --model minimax/MiniMax-M2.7` reproduction, and current main still has routing/fallback paths where explicit session/runtime candidates can reach CLI backends outside the visible MiniMax-only model status. The related closed report #69973 was about preserving transcript context across fallback and does not cover MiniMax-only routing or local inference hangs.

## Best Possible Solution

Keep this issue open and add a focused regression around MiniMax-only model status plus TUI/gateway model-run routing. The likely fix belongs in agent model/runtime selection or fallback candidate validation: a MiniMax-only configured/allowed set should either run MiniMax through PI/local completion or fail with an explicit stale runtime/session override error, not invoke `claude-cli`, `codex-cli`, or OpenAI fallback. Also verify `openclaw infer model run --local --model minimax/MiniMax-M2.7` stays on the lean provider-completion path and returns or times out with a provider-scoped error.
