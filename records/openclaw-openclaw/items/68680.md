## Summary

Keep open. Current main still uses the old node-host exec exit handler, ignores the signal argument, and has no `resolveExitResult` helper or signal-kill regression test. The related Linux OOM victim-selection work is separate and does not implement this PR’s observability change. The provided PR discussion shows the initial timeout/test concerns were addressed, so this is still a plausible focused fix rather than an obsolete PR.

## Best Possible Solution

Keep this PR open. The best path is to review and, if CI is green, land a small owner-module change in `src/node-host/invoke.ts` that reads `(code, signal)`, reports non-timeout signal kills as structured errors, preserves intentional timeout-kill semantics, and keeps focused regression coverage.
