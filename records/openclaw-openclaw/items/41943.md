## Summary

Keep open. Current main still does not add target-aware HTTPS_PROXY handling to Feishu/Lark Axios HTTP requests; Feishu HTTP only injects timeouts, while current proxy handling is for Feishu WebSocket or broader opt-in gateway proxy paths. This PR remains relevant, but the patch needs revision for current main, NO_PROXY behavior, tests, and dependency metadata before merge.

## Best Possible Solution

Revise this PR rather than closing it: add Feishu HTTP proxy support in `extensions/feishu/src/client.ts` with target-aware `HTTPS_PROXY`/`HTTP_PROXY` selection, preserve NO_PROXY behavior, cover cache behavior if proxy env can change, add focused Feishu HTTP proxy tests for proxy and bypass cases, and update Feishu plugin dependency metadata if a direct runtime import remains.
