## Summary

Keep open. Current main still emits the reported `ready for user delivery` wording in the subagent completion announcement path, tests still assert that wording, and the requested `available for review` plus explicit external-final-state verification instruction is absent. Related #44472 and #44587 were duplicate cleanup items that point back to this issue as the canonical tracker.

## Best Possible Solution

Keep this issue open as the canonical source-level bug. The likely fix belongs in `src/agents/subagent-announce.ts` with matching tests and docs: change the completion handoff wording away from premature user-delivery language and add explicit guidance that subagent text is input material until the requester verifies whether the external target was actually applied.
