## Summary

Current main still does not implement Feishu account-aware group session isolation. Feishu passes the runtime account into routing and outbound dispatch, but group session keys are still built as agent/channel/group/peer only, so two Feishu bot accounts in the same group routed to the same agent can share transcript state.

## Best Possible Solution

Keep this issue open. Implement account-aware Feishu group session isolation in the Feishu plugin, likely by appending a normalized accountId-derived suffix for non-default account group routes similar to the WhatsApp helper, while preserving default-account legacy continuity. Add a regression test with two Feishu accounts in the same group routed to the same agent asserting distinct session keys and correct outbound account identity, then update Feishu/channel-routing docs for the compatibility behavior.
