## Summary

Keep this issue open. Current main has strong restart-side LaunchAgent hardening and recovery, but the stop-side mechanism reported here still exists: `gateway stop` waits about 1 second, then can bootout the LaunchAgent and explicitly warn that it left the service unloaded.

## Best Possible Solution

Keep this issue open and narrow it to the remaining macOS `gateway stop` semantics. The best fix is either to extend the stop confirmation budget and avoid unloading on slow drains, or to make the bootout fallback restore a recoverable disabled-but-loaded state. If maintainers intentionally want `gateway stop` to be allowed to leave launchd unloaded, then close only with that explicit product decision and align the issue/changelog/docs around start/restart repair being the supported recovery path.
