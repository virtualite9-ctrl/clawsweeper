## Summary

Keep this issue open. Current main still does not implement tolerant reasoning-prefaced or glued trailing NO_REPLY suppression, and open unmerged PRs already reference this issue with closing syntax.

## Best Possible Solution

Keep this issue open as the canonical tracker. Review and choose among the active implementation PRs, then land a fix with regression coverage for bare think, <think>...</think>, whitespace-delimited trailing tokens, glued final-token cases like here.NO_REPLY, and the relevant streaming/block delivery path.
