## Summary

Keep open. Current main does not contain the inbound-media read guard proposed by this PR: the read wrapper still only strips truncation details, normalizes image results, and applies image sanitization. The linked security problem remains actionable, and the PR discussion shows review findings were addressed rather than proving the work is obsolete.

## Best Possible Solution

Keep this PR open and review or land a maintained version of the inbound-media read guard. The implementation should live in `src/agents/pi-tools.read.ts`, wrap only text blocks from staged inbound media with the existing untrusted-data wrapper, sanitize path labels, preserve image blocks, thread sandbox `containerWorkdir`, and include focused regression coverage. Keep #11207 open until a reviewed fix merges.
