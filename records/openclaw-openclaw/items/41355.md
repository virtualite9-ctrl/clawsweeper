## Summary

Keep open. Current main still sends Discord ThreadStarterBody through the inbound context on follow-up thread turns, and the reply prompt path can still prepend that value as `[Thread starter - for context]`. The same-author PR #50522 is still an open implementation candidate that explicitly attempts to close this issue, so maintainers should review, land, or supersede that PR before closing the issue.

## Best Possible Solution

Keep this issue open as the canonical tracker. Review and land #50522 or an equivalent Discord-source fix that only supplies `ThreadStarterBody` for the first thread session turn, then add focused Discord regression coverage for existing thread sessions and evaluate whether the session-level fallback should also be tightened.
