## Summary

Current main still does not implement this PR's intended `openclaw config set` invalid-model warning. The PR remains a plausible fix for the closed stale bug #20522, and later PR commits addressed the bot review feedback around case-insensitive lookup, auth-profile suffixes, empty catalogs, and testing the real helper. It should stay open, but it needs a rebase/adaptation to the current `runConfigSet` flow and should drop the unrelated `src/agents/skills-status.ts` change.

## Best Possible Solution

Keep this PR open as useful but require a rebase onto current main. The best implementation is a scoped `runConfigSet` integration that detects model-reference paths across single and batch operations, emits a non-blocking warning through the provided runtime when a non-empty catalog lacks the parsed model, uses `findModelInCatalog`, strips auth-profile suffixes, skips degraded/empty catalog results, and adds tests that exercise the real `config set` command path. Drop or split the unrelated skills eligibility change.
