## Summary

Keep this PR open. Current main already globalizes the unhandled-rejection handler registry through `globalThis`, but `installUnhandledRejectionHandler()` still has no installed guard and still attaches a new `process.on("unhandledRejection", ...)` listener on every call. The direct legacy entrypoint and `runCli()` both call that installer, so the PR’s central behavior remains relevant rather than implemented on main.

## Best Possible Solution

Keep this PR open, or replace it with an equivalent maintainer patch, by adding a shared/global installed guard to `installUnhandledRejectionHandler()` while preserving the existing global handler Sets. Add focused regression coverage that repeated installer calls do not increase `process.listenerCount("unhandledRejection")`, and keep plugin-registered rejection handlers shared through the existing SDK boundary.
