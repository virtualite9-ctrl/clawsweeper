## Summary

Current main does not fully implement this PR's intended behavior. The reset hook and direct compaction hook paths already pass resolved agent/session context, but the embedded auto-compaction callback path still passes only `sessionKey` to `before_compaction` and `after_compaction`, and the existing test coverage only asserts `sessionKey`. The PR remains a plausible narrow fix rather than obsolete cleanup.

## Best Possible Solution

Keep this PR open, update it against current main, and narrow it to the still-missing embedded auto-compaction hook context. The likely landing shape is to forward `ctx.params.agentId`, `ctx.params.sessionId`, and `ctx.params.sessionKey` in `src/agents/pi-embedded-subscribe.handlers.compaction.ts`, with `src/plugins/wired-hooks-compaction.test.ts` asserting all three fields for both `before_compaction` and `after_compaction`. The already-implemented reset/direct-compaction parts should not be reintroduced unless the current branch still needs cleanup after rebase.
