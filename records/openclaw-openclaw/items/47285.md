## Summary

Keep this PR open. Current main still does not implement native Azure OpenAI embedding support for memory-lancedb: the direct OpenAI-compatible path sends standard OpenAI auth, the provider-backed OpenAI embedding path also builds Bearer auth without Azure api-version handling, and the memory-lancedb config/schema has no apiVersion/query/header escape hatch. The PR is a useful candidate but needs changes before it is the best fix.

## Best Possible Solution

Keep the PR open and ask for a revised implementation. The best fix should add robust Azure-aware embedding transport for memory-lancedb/OpenAI embeddings, reuse or mirror existing parsed-host Azure detection for both .openai.azure.com and .services.ai.azure.com, expose an apiVersion configuration with schema/UI/docs coverage, and add focused tests for Azure headers/query plus the normal OpenAI-compatible path.

## Risks / Open Questions

- Merging the PR as submitted would pin Azure OpenAI embeddings to api-version 2024-02-01 with no config/schema override.
- The PR only detects .openai.azure.com with substring matching, so it misses Azure AI Foundry *.services.ai.azure.com endpoints and should use parsed-host suffix checks consistent with existing Azure helpers.
- No tests or docs are included for Azure memory embeddings, non-Azure regression behavior, configurable API version, or Foundry endpoints.
