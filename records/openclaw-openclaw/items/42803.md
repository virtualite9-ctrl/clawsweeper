## Summary

Current main still does not fully satisfy the reported Feishu text-command fast path. Feishu messages still enter the inbound debouncer before dispatch, same-key non-debounced work can wait behind an active flush, and Feishu sequential-key routing only sends abort text to the control lane while /status and /new remain on the normal per-chat lane. An open, unmerged implementation candidate, #43512, still uses closing syntax for this issue, so cleanup policy says to keep this issue open until maintainers land, replace, or close that PR.

## Best Possible Solution

Keep this issue open and have maintainers review, land, or replace #43512 with a current-main Feishu fix that detects command-only Feishu messages before same-key inbound debouncer serialization, or otherwise gives /stop, /new, and /status a true control fast path while preserving authorization, mention gating, and normal message ordering. Add regression coverage proving these text commands execute during an active same-chat Feishu run.
