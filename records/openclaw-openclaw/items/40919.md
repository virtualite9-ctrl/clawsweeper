## Summary

Keep open. Current main still scopes session sync to dirty/targeted transcripts, but each selected session is rebuilt from the full JSONL transcript and indexed through the same full clear-and-rewrite path. The existing embedding cache and session-delta thresholds are mitigations, not durable append-only or chunk-level incremental session indexing. Related PR #59577 remains an open implementation candidate; #50908 was closed unmerged.

## Best Possible Solution

Keep this issue open and review or replace #59577 with a current implementation in the memory-core plugin and memory host SDK. The fix should add durable append/chunk metadata, avoid clearing preserved session chunks, reuse the embedding cache where valid, fall back to full reindex on truncation/compaction/config/model/parser changes, and add regression coverage for append growth, unchanged transcripts, overlap boundaries, compaction, and failure recovery.

## Risks / Open Questions

- A naive offset-only fix could leave stale or duplicated chunks around overlap boundaries, compaction/truncation, parser changes, model changes, chunking config changes, or partial embedding failures.
- The embedding cache can reduce repeated embedding calls when enabled, but it still allows full transcript parsing, chunk generation, row deletion, and row reinsertion for changed session files.
