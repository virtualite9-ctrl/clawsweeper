## Summary

Keep this PR open. Current main still lacks Amazon Bedrock service_tier forwarding, docs, and Bedrock-specific regression coverage, while the provided PR remains the active implementation candidate for the linked Bedrock feature request. The security pass did not find supply-chain-sensitive file changes in the provided PR file list; the main maintainer risk is the intentionality of expanding the public Plugin SDK barrel.

## Best Possible Solution

Keep this PR open as the active implementation candidate. The best path is to implement the behavior in the Amazon Bedrock plugin using the existing extraParams and wrapStreamFn seam, inject top-level { serviceTier: { type } } only for Bedrock Converse stream requests, preserve existing payload fields, document the supported configuration, keep focused regression coverage, and make an explicit maintainer call on whether the shared Bedrock service-tier helpers should be public SDK exports or plugin-local helpers.
