## Summary

Keep open. Current main still contains the exact early-return guard that preserves an existing `Current time:` line instead of refreshing it, and the main heartbeat path still sends heartbeat bodies through that helper. The two linked fix PRs were closed unmerged, so there is no shipped implementation to close against.

## Best Possible Solution

Implement an idempotent refresh in `src/agents/current-time.ts`: keep empty prompts unchanged, compute the fresh `timeLine`, replace any existing `^Current time:.*$` line with it, and otherwise append exactly one line. Add focused unit coverage for repeated calls with different `nowMs` values plus a heartbeat-path regression test so the issue can be closed after the fix lands.
