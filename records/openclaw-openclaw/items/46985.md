## Summary

Keep this PR open. Current main still uses cumulative chat delta snapshots and renders stored stream segments plus the live stream separately, while the tool-start commit path still stores the whole current `chatStream` as a segment. That leaves the visible duplication problem around tool cards unsolved. The PR discussion is also useful: Greptile flagged the initial repeated-segment bug, and the author followed up with the plain-concat fix in the PR branch. There is a related open PR, #47377, for the same remaining UI bug, but without maintainer signal that it supersedes this one, this PR should not be auto-closed before a focused fix lands.

## Best Possible Solution

Keep this PR open while maintainers choose the implementation path. The best path is a focused current-main fix that stores only the uncommitted suffix when a tool starts, dedupes the live preview against already committed stream segments, and adds regression coverage for repeated identical segments and multi text/tool/text cycles. If #47377 becomes the preferred focused PR, close this PR only after that replacement lands or after a maintainer explicitly marks it superseded; split the unrelated metadata, plugin-diagnostic, and security-audit changes into separate reviewable work.
