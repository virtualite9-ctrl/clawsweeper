## Summary

Current main still contains the reported acpx failure path: startup prepares the Codex ACP wrapper under the gateway state directory, writes `codex-acp-wrapper.mjs`, then awaits `fs.chmod` without a non-fatal fallback. Because the configured wrapper command invokes the script through Node, the executable bit is not required for that path, so this remains an actionable regression rather than a cleanup close.

## Best Possible Solution

Keep this issue open and fix it in `extensions/acpx/src/codex-auth-bridge.ts` by treating chmod failures on generated ACP wrapper scripts as non-fatal when the scripts are invoked via Node. Add focused acpx tests for EPERM/EACCES chmod on remote/special filesystems, and update acpx setup docs only if a remaining filesystem requirement still exists after the code fix.
