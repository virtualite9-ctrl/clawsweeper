## Summary

Current main does not give enough evidence to close this as fixed. The command has a chat-channel output section, but its read-only plugin discovery path can still omit configured Telegram/Discord accounts because it disables setup-runtime fallback and the routed command does not preload channel plugins.

## Best Possible Solution

Keep this issue open. The fix likely belongs in `src/commands/channels/list.ts` and/or `src/channels/plugins/read-only.ts`: make configured account discovery work for setup-entry-backed bundled channels such as Telegram and Discord without forcing a full runtime import waterfall, then add regression coverage for `channels.telegram.accounts` in both JSON and human `channels list` output.
