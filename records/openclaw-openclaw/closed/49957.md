## Summary

Keep #49957 open. The reported export-session HTML bug is still present in current main and in the latest release tree: the template contains formatter-expanded script placeholder blocks, while the exporter still performs exact `{{TOKEN}}` string replacement, so no inline JS is injected.

## Best Possible Solution

Keep the issue open until a focused fix lands. The best path is to restore non-reformatted inline placeholders or switch to a formatter-safe delimiter, guard the template from formatter churn, and add a regression test that exercises generated HTML from the real template and fails if raw placeholder blocks remain or app JS is not inlined.
