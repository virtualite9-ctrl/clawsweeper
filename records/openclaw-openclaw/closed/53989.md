## Summary

Current main implements the Slack inbound path needed for the reported silent @mention dispatch regression: Slack registers the generic message and app_mention events, the message handler now allows one app_mention retry when the matching message event was seen but dropped before dispatch, and there is focused regression coverage for that race. Account-aware routing and per-account startup also mean disabled or unconfigured default accounts do not block named Slack accounts from routing to their bound agents.

## Best Possible Solution

Close this as implemented and keep the current Slack app_mention retry/dedupe path plus account-aware routing/startup behavior. If the reporter can still reproduce on v2026.4.26 or newer with stable Slack channel IDs, they should open a fresh issue with sanitized Slack channel/account config, bindings, OpenClaw version, and verbose routing/drop logs.
