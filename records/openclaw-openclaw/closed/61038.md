## Summary

Keep PR #61038 open. Current main still lacks the PR's structural Matrix self-loop protections: room messages are forwarded from the Matrix monitor without a cached self-sender gate, the handler relies on the existing async self-user check, inbound m.notice is not generally filtered, and generic Matrix reply delivery still defaults to m.text. The linked bug issue is closed, but the PR remains the focused implementation path rather than obsolete or implemented.

## Best Possible Solution

Do not auto-close. Rebase and review PR #61038, or apply an equivalent narrower fix on main: add a monitor-level self-sender gate, filter/log inbound m.notice in the handler, send tool/block output as Matrix notices while keeping final replies as text, and keep the regression tests around self-message forwarding and reply msgtypes.
