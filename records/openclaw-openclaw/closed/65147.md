## Summary

Current main fixes the underlying container npm update failure by staging npm global installs in a temporary prefix, verifying that package tree, then swapping it into the real global prefix and replacing the bin shim. That covers both the primary install and the omit-optional fallback, so this PR's npm --force approach is obsolete.

## Best Possible Solution

Close this PR as implemented on main by the staged npm global update flow. Keep the staged install/swap implementation rather than adding npm --force; if maintainers want the package-spec option-injection hardening, open or request a focused follow-up against current main.
