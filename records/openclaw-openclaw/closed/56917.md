## Summary

Keep PR #56917 open. Current main still strips `gateway`, `cron`, and `nodes` for non-owner senders before `tools.alsoAllow` can re-add anything, and the PR’s linked Feishu/gateway problem is not implemented or superseded in the checked-out source. The PR is a plausible focused fix, though it needs maintainer security review and cleanup of the unrelated CLI commit before merge.

## Best Possible Solution

Keep the PR open. The best path is a maintainer-reviewed implementation that threads an explicit, normalized global plus per-agent `tools.alsoAllow` list into the owner-only policy for the intended surfaces, adds regression coverage for non-owner Feishu/external-channel `gateway` exposure when configured, preserves filtering of non-allowed owner-only tools, and documents the trust boundary. Before merge, drop the unrelated CLI logs commit from the PR branch.
