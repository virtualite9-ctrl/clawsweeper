## Summary

Current main already implements the Comfy plugin-owned config fix through merged PR #63058: runtime reads `plugins.entries.comfy.config`, keeps legacy `models.providers.comfy` as a fallback, handles plugin-owned cloud auth including SecretRefs, and documents the new path. This PR is based on the older runtime shape and is superseded by the shipped implementation.

## Best Possible Solution

Close this PR and keep the #63058 implementation on main. If maintainers later want `models.providers.comfy` to override `plugins.entries.comfy.config` when both are present, that should be a separate, explicit product/API decision with tests for nested workflow conflict behavior and SecretRef auth.
