## Summary

Keep open. Current main still does not implement the requested replace-mode behavior in the CLI model picker or gateway catalog, and the same-author bug report #64950 remains open. The PR is a plausible implementation candidate, but maintainers still need to settle implicit/plugin-discovered model semantics and add regression coverage.

## Best Possible Solution

Keep this PR and #64950 open until a maintainer-chosen fix lands. The best path is a shared, mode-aware catalog path for the CLI default picker, CLI allowlist picker, and gateway models.list that honors models.mode: "replace", explicitly defines whether implicit/plugin-discovered models remain visible, preserves configured metadata and aliases, and adds focused regression tests for all three surfaces.
