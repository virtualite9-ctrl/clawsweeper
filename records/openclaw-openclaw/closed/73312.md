## Summary

Keep this PR open. Current main still does not implement Matrix MessagePresentation metadata, and the open PR remains a plausible Matrix plugin feature rather than obsolete work. The later commits appear to address the initial version-ordering and mediaUrls review findings, but the patch still needs maintainer review because runtime-only additions are not exposed through the Matrix channel facade and Matrix extraContent forwarding is security-sensitive.

## Best Possible Solution

Keep this PR open and ask for revisions: expose the new presentationCapabilities, renderPresentation, and sendPayload path through the Matrix channel facade or shared delegate helper; preserve mediaUrls and audioAsVoice behavior with shared reply-payload helpers or equivalent tests; and constrain Matrix extraContent so only the intended namespaced presentation metadata is attached without overriding core event fields.
