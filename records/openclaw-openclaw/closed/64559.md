## Summary

Current main already implements the PR's intended behavioral fix: inbound `message_received` emission happens before the reply resolver and therefore before the active-session queue/enqueue decision. The PR's helper extraction and extra tests are no longer required for the shipped behavior, and the latest release tree contains the same hook ordering.

## Best Possible Solution

Close this PR as implemented by current main and keep the shipped hook ordering. If maintainers want more guardrail, the useful remaining work is a narrow regression test proving `message_received` fires before the reply resolver/active queue decision, without adopting the PR branch's broader early-visibility change.
