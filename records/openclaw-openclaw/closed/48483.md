## Summary

Keep open. Current main still routes the Control UI dropdown through gateway `models.list`, and that handler only applies `agents.defaults.models` allowlist filtering before returning the full catalog. I did not find source or test evidence that `models.mode: "replace"` is enforced for the RPC catalog when no allowlist is present, while the issue discussion reports recent reproductions on v2026.4.21 and v2026.4.23.

## Best Possible Solution

Keep this issue open. The best fix is in the gateway catalog boundary: make `models.list` derive a replace-mode catalog from configured provider models/provider families before applying `agents.defaults.models` allowlist logic or returning a fallback catalog, then add regression coverage for `models.mode: "replace"` with configured providers and no allowlist, including provider-family variants such as `volcengine`/`volcengine-plan` and `byteplus`/`byteplus-plan`. The Control UI can keep using `models.list` once that RPC contract is enforced.
