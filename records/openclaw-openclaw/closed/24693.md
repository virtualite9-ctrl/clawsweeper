## Summary

Keep open. Current main still routes hook completion and error system events to the default main session instead of the hook target agent session, and the existing test fixture explicitly expects that behavior.

## Evidence

- **Gateway hook dispatcher still targets default main session:** `dispatchAgentHook` resolves `mainSessionKey` from global config, then uses that key for non-delivered completion events and error events, regardless of `value.agentId` or the per-agent `sessionKey`.
  - file: [src/gateway/server/hooks.ts:45](https://github.com/openclaw/openclaw/blob/d3f6783b16ecc10c67b101bf019ca4294453206a/src/gateway/server/hooks.ts#L45)
  - command: `nl -ba src/gateway/server/hooks.ts | sed -n '1,220p'`
  - sha: [d3f6783b16ec](https://github.com/openclaw/openclaw/commit/d3f6783b16ecc10c67b101bf019ca4294453206a)
- **Hook request path resolves per-agent execution target:** The HTTP hook handler resolves `targetAgentId`, normalizes the dispatch session key to that target agent, and passes both into `dispatchAgentHook`, so the run itself is agent-scoped even though follow-up system events are not.
  - file: [src/gateway/server-http.ts:682](https://github.com/openclaw/openclaw/blob/d3f6783b16ecc10c67b101bf019ca4294453206a/src/gateway/server-http.ts#L682)
  - command: `nl -ba src/gateway/server-http.ts | sed -n '680,840p'`
  - sha: [d3f6783b16ec](https://github.com/openclaw/openclaw/commit/d3f6783b16ecc10c67b101bf019ca4294453206a)
- **Session-key normalization is explicitly cross-agent aware:** `normalizeHookDispatchSessionKey` rewrites agent session keys to the resolved `targetAgentId`, confirming the design intent is per-agent routing for hook execution state.
  - file: [src/gateway/hooks.ts:383](https://github.com/openclaw/openclaw/blob/d3f6783b16ecc10c67b101bf019ca4294453206a/src/gateway/hooks.ts#L383)
  - command: `nl -ba src/gateway/hooks.ts | sed -n '372,410p'`
  - sha: [d3f6783b16ec](https://github.com/openclaw/openclaw/commit/d3f6783b16ecc10c67b101bf019ca4294453206a)
- **Current test suite codifies default-session event routing:** `hooks.agent-trust.test.ts` asserts that hook completion and error events are enqueued to `main-session`, which matches the reported leak rather than disproving it.
  - file: [src/gateway/server/hooks.agent-trust.test.ts:98](https://github.com/openclaw/openclaw/blob/d3f6783b16ecc10c67b101bf019ca4294453206a/src/gateway/server/hooks.agent-trust.test.ts#L98)

## Risks / Open Questions

- Cross-agent hook summaries and error text can still leak into the default agent session on current main.
- A future fix must update the existing gateway hook tests, because they currently assert the leaking behavior.
