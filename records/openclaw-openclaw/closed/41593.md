## Summary

This PR should close as superseded by the maintainer decision in the linked canonical report. The PR’s diff is narrow and accurately targets the reported macOS LaunchAgent PATH gap, but #41263 was closed by a maintainer with an explicit policy decision not to add vendor-specific PATH entries like OrbStack or Docker Desktop; current main still reflects that policy.

## Best Possible Solution

Close this PR as superseded by the maintainer decision on #41263. Keep the curated LaunchAgent PATH policy on main and direct affected users to expose docker through a standard service-visible bin directory such as ~/.local/bin; if product direction changes, open a separate maintainer-approved proposal for a generic gateway service PATH configuration rather than adding vendor-specific paths.
