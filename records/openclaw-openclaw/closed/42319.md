## Summary

Current main already implements the useful parts of this PR. Gateway health snapshots now carry live channel runtime state through cached refreshes while preserving probe payloads on sensitive/admin health paths and redacting them from non-sensitive snapshots. The Google Chat auth side is also covered by the current scoped, SSRF-guarded google-auth transport and service-account endpoint validation, which is safer than this PR's stale manual JWT exchange. The implementation is present in the v2026.4.26 release snapshot.

## Best Possible Solution

Close this PR as implemented on main. Keep the shipped v2026.4.26 health/runtime implementation and the existing SSRF-guarded Google Chat auth runtime. Any future Google Chat auth change should be a narrow, separately evidenced patch that preserves endpoint validation, guarded transport, credential-file limits, and the current health probe/runtime guarantees.

## Risks / Open Questions

- Merging this PR as written would be a regression risk because its manual service-account JWT exchange does not preserve current main's SSRF-guarded Google auth transport and strict service-account endpoint validation.
- The PR's health refactor had an unresolved reviewed bug where probe data could be dropped for plugins without custom buildAccountSnapshot hooks; current main avoids that path and has targeted regression coverage.
