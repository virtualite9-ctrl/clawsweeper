## Summary

Current main already exposes the needed agent-error observation surface through public plugin hooks. Plugins can use `agent_end` to observe failed runs with `success: false`, an error string, run/session context, and duration, and `model_call_ended` to observe provider/model call failures with sanitized metadata such as outcome, error category, failure kind, timing, and bounded request-id hashes. This covers the auto-recovery trigger use case without adding a parallel internal `agent:error` hook name.

## Best Possible Solution

Keep the shipped public plugin-hook implementation and direct auto-recovery plugins to `agent_end` for run-level failures plus `model_call_ended` for provider-call failure telemetry. Only open a narrower follow-up if plugin authors need a specific additional sanitized field that these hooks do not expose.
