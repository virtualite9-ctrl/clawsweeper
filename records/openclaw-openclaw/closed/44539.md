## Summary

Current main already handles the reported Telegram doctor/status mismatch for the described configuration. `openclaw doctor` now routes channel reporting through the Gateway health path, that path evaluates the preferred/default/configured Telegram accounts, and Telegram configured-state inspection recognizes account-level and channel-level `botToken`/`tokenFile` values before falling back to `TELEGRAM_BOT_TOKEN`. The same implicated source is present in the latest release tag `v2026.4.26`.

## Best Possible Solution

Close as implemented on main and keep the current Gateway health plus Telegram account-inspection behavior. If a new mismatch appears on a current build, it should be filed as a fresh bug with current version, `openclaw doctor` and `openclaw status` output, account bindings/defaultAccount, and redacted channel config.
