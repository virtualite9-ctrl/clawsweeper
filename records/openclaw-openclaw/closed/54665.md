## Summary

Current main now implements a targeted Gateway startup fast path for the managed-service command shape behind `openclaw gateway start`. The service launches the foreground Gateway as `gateway --port <port>`, and main now keeps that value-option invocation on the lightweight path before proxy/bootstrap and full CLI imports, with regression coverage and an unreleased changelog entry describing reduced Gateway startup RSS and avoidance of the full CLI graph.

## Best Possible Solution

Keep the current main Gateway fast-path implementation and ship it in the next release. If a build containing `c205577f2cf1d8098393a0d7f4c96f848431710e` still reproduces the V8 RegExpCompiler OOM, the follow-up should include `OPENCLAW_GATEWAY_STARTUP_TRACE=1`, service argv/env details, and the configured channel/plugin set so maintainers can isolate any remaining import path.
