## Summary

Current main fixes the linked Discord listener-timeout/message-drop problem with a different implementation: OpenClaw now disables Carbon's blocking interaction listener and hands Discord interactions off asynchronously, so compaction or long session locks no longer consume the critical listener lane. The PR's direct session-lock change is no longer needed and has unresolved data-loss risk in transcript rotation.

## Best Possible Solution

Close this PR and keep the shipped Discord listener-handoff implementation on main. If maintainers still want broader non-blocking compaction, it should be handled as a separate agents compaction design with refreshed transcript state and regression tests for concurrent writes plus transcript rotation.
