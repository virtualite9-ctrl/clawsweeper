## Summary

Current main implements a formatter-safe export HTML injection path. The template now uses data-openclaw-export-placeholder attributes, the generator replaces those tags with function replacers, regression coverage reads the real template, and the changelog records the blank export-session HTML bug as fixed.

## Best Possible Solution

Close this issue as implemented on main. Keep the new attribute-based placeholder injection and real-template regression tests; the next release should ship the already-recorded changelog fix.
