## Summary

Current main already implements both intended fixes from this PR with a broader shipped implementation: bootstrap snapshots are refreshed through the guarded workspace-file cache on each turn, and host=node exec no longer resolves the cwd against the gateway filesystem. The related bootstrap issue is covered by the 2026.4.26 changelog entry, and the node exec behavior is covered by source, tests, and earlier changelog entries.

## Best Possible Solution

Close this PR as implemented on main. Keep the shipped broader implementation: per-turn bootstrap refresh backed by the guarded workspace file cache, plus explicit host=node cwd forwarding with node approval binding. No PR merge is needed.
