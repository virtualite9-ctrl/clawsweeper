## Summary

Keep open. Current main still sorts the `/tools` available-tools prompt text by display label with default `localeCompare`, and it does not contain the PR's locale-free, normalized-id ordering change. The linked #61380 issue was closed for the incorrect `toSorted` stability premise, but that does not supersede this PR's narrower deterministic-comparator discussion. The PR likely needs maintainer review and a stronger regression test, not automatic closure.

## Best Possible Solution

Keep the PR open for maintainer review. If OpenClaw wants cache-stable available-tools prompt bytes here, update `src/auto-reply/status.ts` to use a locale-free comparator over normalized tool ids with a clear tie-breaker, and strengthen `src/auto-reply/status.tools.test.ts` with a fixture that fails under current label/locale ordering. Do not use #61380's `toSorted` stability premise as the justification.
