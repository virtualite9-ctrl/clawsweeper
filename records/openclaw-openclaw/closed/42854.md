## Summary

Keep this issue open. Current main still rewrites `google/gemini-3.1-pro` to the preview id through both static manifest normalization and the Google provider runtime hook, and Gemini 3.1 Pro forward-compat metadata is still cloned from the older `gemini-3-pro-preview` template. Tests, docs, and defaults still preserve preview-derived behavior, while the direct fix PRs in the related context are closed unmerged.

## Best Possible Solution

Keep the issue open. The best fix is in the Google plugin/model-selection path: stop rewriting stable `google/gemini-3.1-pro` where the stable id should remain visible, update static manifest and runtime normalization consistently, prefer a current Gemini 3.1 Pro template with the old `gemini-3-pro-preview` template only as fallback if needed, and align focused regression tests plus Google provider docs/defaults. If preview ids are intentionally canonical, document that decision before closing.
