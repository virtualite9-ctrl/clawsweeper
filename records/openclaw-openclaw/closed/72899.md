## Summary

Current main already addresses the reported durable blank-user ContentBlock poisoning. Bare reset/startup turns now use a non-empty transcript marker, legacy blank user rows are repaired on disk before session load, and replay history drops blank user text blocks before provider invocation. Existing regression tests cover the repair, replay filter, and non-empty reset marker paths.

## Best Possible Solution

Keep the current main implementation: preserve the non-empty reset marker, retain the on-disk session repair, and keep replay-time blank-user filtering as defense in depth. If follow-up work is desired, it should be a narrow docs/test hardening pass rather than keeping this bug open.
