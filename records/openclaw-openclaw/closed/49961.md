## Summary

Keep #49961 open. Current main still has the exact session export template/replacement mismatch described by the PR: the checked-in template contains formatter-expanded JS placeholder blocks, while the export generator still performs literal compact `{{...}}` replacements. The PR discussion and Greptile comment correctly identify this as a focused bug fix, and the existing tests do not cover the real template injection path.

## Best Possible Solution

Keep this PR or a canonical equivalent open and land a consolidated fix for the session export path: restore compact JS placeholders, add a formatter guard such as `.prettierignore` or another repository-standard protection, and add regression coverage that reads the real `template.html`. The final patch should also consider function replacers for JS/vendor injection so `$` sequences in vendored scripts are preserved.
