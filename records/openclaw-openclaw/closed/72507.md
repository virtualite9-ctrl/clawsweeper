## Summary

This issue should close as duplicate/superseded by the already-open canonical sessions.json bloat tracker #55334. Current main still persists the static `skillsSnapshot` and `systemPromptReport` payloads per session entry, so the bug is real, but it is the same remaining work already consolidated in #55334 and repeats the same-author duplicate #72327 that was closed for that reason.

## Best Possible Solution

Close this issue as duplicate/superseded and keep the product/code work consolidated in #55334: replace per-session static snapshot/report persistence with a compatible reference/cache or on-demand reconstruction path, migrate or compact existing rows safely, and retain the current maintenance/reaper mitigations as defense in depth.
