## Summary

Keep open. Current main has an intended code path for listing chat channel accounts, but the report is fresh, concrete, and targets the latest release line. The relevant implementation is effectively unchanged from the reported release, and existing coverage only verifies auth-profile output, not configured Telegram/Discord account rows.

## Best Possible Solution

Keep this issue open and add a focused regression test for `channelsListCommand` with configured `channels.telegram.accounts` and/or `channels.discord.accounts` using the read-only plugin resolution path. If the test reproduces the missing rows, fix the channel presence/plugin resolution path so `channels list` reliably emits configured chat accounts before the auth-provider section.

## Evidence
