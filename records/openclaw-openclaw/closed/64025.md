## Summary

Keep this issue open. Current main has fixed part of the discovery/docs gap by documenting `clawhub package explore` and `clawhub package inspect`, and OpenClaw plugin install/update fixes for related issues have shipped. The reported discovery problem is still not fully resolved: live package query search still returns server errors for the reporter's example plugin names, native skill search remains intentionally skill-only, and the bundled ClawHub skill still describes only skill workflows.

## Best Possible Solution

Keep the issue open but narrow it to the remaining actionable work: fix or route the ClawHub package query search failure for valid plugin names such as `episodic-claw`, update the bundled `skills/clawhub/SKILL.md` to include `clawhub package explore` and `clawhub package inspect`, and keep the current docs direction that plugin installs use `openclaw plugins install clawhub:<package>`. The already-shipped plugin install/update fixes should remain closed separately.
