## Summary

Current main already shipped the #66019 cron spin-loop fix through maintainer PRs #66083 and #66113. The shipped implementation clears unresolved cron next-runs instead of synthesizing short retry/backoff slots, preserves the active error-backoff floor during later maintenance repair, and has regression coverage for the direct and timer-level paths. This PR's extra schedule-error-count/auto-disable semantics are not the shipped direction and should be a separate product decision if still desired.

## Best Possible Solution

Close this PR as superseded by the merged maintainer fixes #66083 and #66113. Keep the shipped current-main behavior for #66019, and only revisit the schedule-error-count/auto-disable semantics through a new narrow issue or PR if maintainers decide that product behavior is desired.
