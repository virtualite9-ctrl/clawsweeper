## Summary

Keep PR #40574 open. Current main still exposes `write` as a path/content overwrite tool and only has append behavior for the narrow memory-flush path, so the PR is not obsolete and the broader cron/shared-file data-loss issue remains open in #40001.

## Best Possible Solution

Keep this PR open for maintainer review or supersede it with a focused core implementation tied to #40001. The best path is a backwards-compatible append primitive for the model-facing `write` tool or equivalent file tool that advertises `append` in `tool.parameters`, preserves canonical `path`/`content` validation and workspace/host path semantics, uses the existing safe append handle path for workspace-contained writes, defines explicit sandbox behavior, returns structured tool errors, and adds host, sandbox, cron, and schema regression coverage.
