## Summary

Keep PR #72177 open. Current main still lacks the PR's central LiteLLM image-generation hardening: LiteLLM edits are JSON/data-URL based, there is no LiteLLM provider default timeout, the agent tool still exposes and honors model-supplied `timeoutMs`, and the LiteLLM provider has no chat-completions image route, Gemini duplicate-image dedupe, or chat-response parsing budgets. The PR is recent, focused, contributor-authored, and the provided discussion shows active follow-up on review findings rather than obsolete work.

## Best Possible Solution

Keep #72177 open for maintainer review. The best product path is to land this PR or an equivalent focused patch that makes LiteLLM edits multipart, gives LiteLLM image requests a provider default timeout, removes or ignores model-authored `image_generate.timeoutMs` while preserving operator config, adds a robust LiteLLM chat-completions route for chat-native image models, and covers response parsing/dedupe with regression tests.
