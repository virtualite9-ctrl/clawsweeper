## Summary

Current main already addresses the reported false-success mixed-version update path. Package-manager `openclaw update` stops a running managed gateway before package replacement, refreshes service metadata from the updated install, restarts the managed service, verifies the restarted gateway reports the expected updated version, and exits non-zero with diagnostics on version mismatch or activated plugin load errors. The changelog documents this mixed-version macOS LaunchAgent verification fix in v2026.4.25.

## Best Possible Solution

Close as implemented on current main. Keep the shipped package-update restart verification and service metadata refresh behavior; use a new narrow issue only if a current 2026.4.27-or-newer `openclaw update` run can still exit successfully while the probed managed gateway reports the old version.
