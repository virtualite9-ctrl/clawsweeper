## Summary

Keep this PR open. Current main still calls the interactive onboarding final health check without the setup-session token, while the linked open bug #72203 reports that exact QuickStart token-auth mismatch. The PR remains a plausible focused implementation candidate, with a small credential-forwarding surface that should receive normal maintainer review before landing.

## Best Possible Solution

Review and land this PR or an equivalent narrow patch that lets setup finalization pass the just-resolved setup token deterministically into the final health check, ideally through explicit `healthCommand`/`callGateway` auth. Keep the regression test that proves the setup token wins when env/config tokens differ, then close #72203 after merge. Consider splitting the unrelated docs/video test churn if maintainers want a smaller auth-only patch.
