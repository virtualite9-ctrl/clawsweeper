## Summary

This issue should close as a duplicate of the broader open messaging-channel tracker #25592. Current main does not implement the proposed stopReason-gated buffering/splice fix, but #25592 already tracks the same remaining core behavior across channels, while Mattermost-specific threading/preview handling has since improved separately.

## Best Possible Solution

Keep #25592 as the canonical open bug for suppressing unphased pre-tool assistant text across messaging channels. The eventual fix should land in the generic embedded subscribe and payload assembly paths, with Mattermost covered through shared channel delivery tests rather than a Mattermost-only patch.
