## Summary

Current main now implements the weixin/root SDK compatibility fix: the legacy compat surface exports the missing helpers, the bare plugin-sdk root alias forwards those compat exports for plugin loader use, user-installed plugin SDK alias resolution is covered, and the changelog records the fix for this issue.

## Best Possible Solution

Close this issue as implemented on main. Keep the shipped compatibility bridge, release it in the next OpenClaw version, and continue steering plugin authors toward focused plugin-sdk subpaths through the existing migration docs.
