## Summary

Keep #71784 open. Current main still does not retry the live memory-search query embedding path, and the related batch/reindex work does not supersede this provider-agnostic live-search failure report.

## Best Possible Solution

Keep #71784 open. A conservative fix should add a narrow transient transport-error classifier and bounded retry/backoff for live memory query embeddings, either in `embedQueryWithTimeout()` or the shared single-embed remote HTTP helper, while preserving immediate failure for auth/config errors. Add focused regression coverage for live query retries in `extensions/memory-core` and `src/memory-host-sdk`, then live-validate whether SSRF dispatcher keep-alive tuning is also needed.
