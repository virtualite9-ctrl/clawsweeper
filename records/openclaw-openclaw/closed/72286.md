## Summary

Current main now implements the Google Live browser Talk work that this draft PR was adding: the realtime browser contract has a Google-compatible JSON PCM WebSocket transport, the Google provider mints constrained one-use Live API tokens, the Gateway returns browser realtime sessions or a relay fallback, the Control UI routes Google Live sessions to a WebSocket transport, and docs/changelog show it shipped in v2026.4.26.

## Best Possible Solution

Close this draft PR as implemented by current main and v2026.4.26. Keep the shipped generic browser realtime transport, Google Live WebSocket token flow, Gateway relay fallback, URL validation, docs, and tests; handle the unrelated Voice Call consult override as a separate focused review if maintainers still want it.
