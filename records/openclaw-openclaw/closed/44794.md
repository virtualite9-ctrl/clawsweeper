## Summary

Current main already implements the durable gateway wrapper path needed for this request. It does not add the exact proposed `gateway.preStartCommand` or `gateway.wrapperScript` config keys, but it solves the core problem by supporting validated service wrappers through `openclaw gateway install --wrapper` / `OPENCLAW_WRAPPER`, persisting that wrapper in service metadata, and reusing it across forced reinstalls, updates, and doctor repairs for LaunchAgent/systemd installs.

## Best Possible Solution

Close this as implemented and keep the shipped wrapper path. If maintainers later want a convenience `preStartCommand` key, it should be opened as a narrower follow-up because the durable LaunchAgent/systemd wrapper behavior already exists and is documented.
