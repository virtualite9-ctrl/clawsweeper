## Summary

Current main now implements the requested recovery path. Plugin-local config validation failures are converted into degraded startup instead of a fatal gateway config error, `openclaw doctor --fix` now quarantines invalid plugin config by disabling the affected entry and removing its config payload, and the changelog explicitly records this as fixing this issue plus #70371. The related #70394 is superseded by the broader implementation now on main.

## Best Possible Solution

Close as implemented on main. Keep the degraded-startup and doctor quarantine behavior as the canonical fix; any future desire for a separate safe-mode UX should be tracked as a narrower enhancement, not as this outage bug.
