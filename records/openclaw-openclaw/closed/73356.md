## Summary

Current main already resolves the linked gateway hang by skipping CLI startup self-respawn for foreground `openclaw gateway run` invocations, rather than adding the PR's generic respawn-exit diagnostic. The shipped path is wired through the respawn planner, covered by argv-policy tests for the reported gateway command shape, and listed in the v2026.4.26 changelog. The PR diff security pass found no workflow, dependency, lockfile, install/release script, secret-handling, or package-metadata changes.

## Best Possible Solution

Close this PR as already handled by the shipped gateway-run respawn-skip implementation. Keep that implementation as the canonical fix for #72720; open a separate focused issue only if maintainers want generic diagnostics for abnormal child exits in other CLI respawn paths.
