## Summary

Current main still leaves the live reply-run deadlock path unresolved. Replay/session-history repair can synthesize missing tool results after reload, but `runReplyAgent` still waits indefinitely for `pendingToolTasks`, and that wait sits before the cleanup that releases the active session run. The existing discussion and linked open PR #64733 correctly identify the remaining fix area.

## Best Possible Solution

Keep this issue open. Review and land PR #64733, or an equivalent main-branch fix, that bounds or cancels the `pendingToolTasks` drain in `runReplyAgent`, preserves ordered tool-result delivery, logs/surfaces timeout cleanup, and adds focused regression coverage proving a never-settling tool-result delivery task cannot keep the session active forever.
