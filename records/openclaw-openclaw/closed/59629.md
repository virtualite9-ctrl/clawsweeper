## Summary

Keep open. The reported regression still maps to current main: `auth-profiles.json` is only accepted in the structured `{ version, profiles }` shape, while the legacy flat fallback is only applied to `auth.json`, not to a flat `auth-profiles.json`. The related PR #59642 directly targets this issue but is still open/unmerged in the supplied context, so the user-visible bug is not shipped as fixed on main.

## Best Possible Solution

Keep #59629 open until the compatibility path is implemented and validated. The likely fix belongs in `src/agents/auth-profiles/persisted.ts`/`store.ts`: accept or migrate the legacy flat `auth-profiles.json` shape used by affected users, infer `api_key` credentials from legacy `apiKey` entries when safe, preserve current structured-store behavior, and add a focused regression test using the exact flat `ollama-windows` example from the report. PR #59642 is the closest existing follow-up, but it should be updated or replaced and landed before the issue closes.
