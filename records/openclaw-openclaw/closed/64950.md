## Summary

Keep this issue open. Current main still routes the reported CLI picker and gateway model-list surfaces through the full catalog path, and same-author PR #64953 remains open as an explicit implementation candidate for this issue.

## Best Possible Solution

Keep this issue open and review PR #64953, or replace it with an equivalent maintainer patch. The accepted fix should make promptDefaultModel(), promptModelAllowlist(), and the gateway models.list/catalog path use the configured provider catalog when models.mode is replace, preserve merge-mode behavior, and add regression tests for the CLI and gateway surfaces.
