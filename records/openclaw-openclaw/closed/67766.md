## Summary

Current main implements this PR's intended behavior via the Gateway model catalog rather than the PR's separate resolver fallback. Explicit `models.providers.*.models[]` rows are now merged into the catalog, their `input` metadata is preserved, lookup is normalized by provider/model, and the chat attachment path has regression coverage showing configured vision models keep inline images. The PR diff is narrow and does not introduce security-sensitive supply-chain changes.

## Best Possible Solution

Close this PR and keep the current main implementation, which solves the user-visible image-capability miss at the catalog layer. That keeps catalog-hit precedence, handles configured custom/proxy vision models generically, and avoids adding a second `loadConfig()` path inside the Gateway session resolver. Any remaining Qwen Coding Plan endpoint-support question should be handled separately as provider behavior or documentation, not by this PR.
