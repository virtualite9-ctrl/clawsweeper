## Summary

Current main has narrowed the broad LaunchAgent false-positive detection and appears to have resolved the gateway.mode source mismatch, but the remaining unsafe cleanup-hint behavior is still real and is already tracked by the narrower open #52510 with #52515 as its implementation candidate.

## Best Possible Solution

Close this issue as superseded by #52510. Keep the remaining fix focused there: make cleanup hints target only the actual stale/extra service, suppress hints for the loaded running canonical gateway, and review or close #52515 as the implementation candidate.
