## Summary

This is an issue asking maintainers to list or onboard the external ClawRouter provider plugin. Current main has a ClawHub code-plugin catalog, browse/search UI, CLI publishing, HTTP publishing, validation, and scan/verification paths that could support this kind of plugin, but the source tree does not contain a ClawRouter listing and the repository policy for openclaw/clawhub only allows auto-closing pull requests that are certainly implemented on main. This issue should remain open for maintainer or publisher action.

## Best Possible Solution

Keep this issue open for maintainer or publisher action. The practical path is for the ClawRouter owner to publish it through the existing ClawHub code-plugin flow, including `package.json`, `openclaw.plugin.json`, source repo/commit metadata, required `openclaw.compat.pluginApi` and `openclaw.build.openclawVersion` fields, and useful capability tags. Maintainers can then decide whether normal community listing is sufficient or whether the plugin needs moderation, verification, or policy review because it executes code, routes LLM traffic, and handles payments.
