## Summary

Keep open. This is an issue, and the repository-specific policy says ClawHub issues should remain open unless a maintainer handles them. Independently, current main still shows the reported double-serialization path for transfer request, skill rename, and skill merge: those commands pass pre-stringified bodies while the shared HTTP helper serializes POST bodies again.

## Best Possible Solution

Keep this issue open and land a scoped CLI fix: pass plain objects to apiRequest for transfer request, skill rename, and skill merge; update tests to assert object payloads; and consider hardening apiRequest/http.ts to pass through already-serialized string bodies defensively.
