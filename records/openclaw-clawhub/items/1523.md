## Summary

Current main does not implement the PR's intended production smoke hardening. The deploy workflow still runs only the older UI smoke pair, the HTTP production smoke script exists but is not wired into deployment, the production HTTP test lacks the proposed 429 retry/memoization behavior, and the signed-out upload test still asserts the old upload gate text rather than the live /publish-skill redirect flow.

## Best Possible Solution

Keep this PR open for maintainer review. The useful path is to rebase or merge the smoke-coverage changes into main, ideally with the small unreachable-code cleanup in fetchWithRetry, so production deploys run the HTTP smoke before the UI smoke and include signed-out publish/import coverage.
