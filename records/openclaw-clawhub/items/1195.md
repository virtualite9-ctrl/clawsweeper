## Summary

Keep this issue open. The ClawHub cleanup policy says open issues are not auto-closed by this workflow, and the current main branch still leaves the named Convex function capable of running repeated paginated queries in one mutation when more than one page is needed. A partial mitigation exists via scheduled package/skill digest sync mutations, but it does not conclusively implement the cursor-batched fix described by the report.

## Best Possible Solution

Keep this issue open or have maintainers consolidate it under the open canonical report #1182. The robust fix should make owner-publisher digest sync cursor-batched: each internal mutation should accept a cursor, perform at most one .paginate() call, process that page, and schedule itself with continueCursor until done. Add regression coverage for an isDone: false page to prove the next page is scheduled rather than fetched in the same mutation.
