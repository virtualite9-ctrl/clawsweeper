## Summary

Keep this issue open. Current main still renders skill README Markdown without a custom relative-link transform and the canonical skill route is the two-segment /$owner/$slug route without trailing-slash canonicalization or a skill-file catch-all route. Repository policy for openclaw/clawhub also says issues should not be auto-closed by this review path.

## Best Possible Solution

Keep open and fix the skill README renderer or route model. The narrowest implementation is to rewrite relative Markdown links in `SkillDetailTabs` to the existing `/api/v1/skills/{slug}/file?path=...` endpoint, preserving external and hash links; alternatively add a real `/$owner/$slug/*` file route plus trailing-slash canonicalization and tests covering relative links in `SKILL.md`.

## Risks / Open Questions

- This has a real navigation/security surface: relative links in untrusted skill documentation can point users at an unintended same-origin page or another user's skill instead of the intended bundled file.
- Adding only a trailing slash may not be sufficient unless ClawHub also serves or rewrites skill-internal file paths; current main exposes raw files through the API endpoint, not an owner/slug catch-all route.
