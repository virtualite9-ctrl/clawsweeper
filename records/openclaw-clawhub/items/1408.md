## Summary

This should stay open. The repository policy for openclaw/clawhub says to keep issues open, and the report is concrete: a specific published skill page is reachable but not discoverable for a specific keyword query. Current main has a real search-index surface where that failure can happen, because homepage/search page queries call the Convex skill search action, which relies on vector embeddings plus a bounded lexical scan over denormalized search digests.

## Best Possible Solution

Keep this issue open for maintainer triage. Verify the production search state for may-yaha/baidu-netdisk-storage across the skill document, skillSearchDigest, skillEmbeddings, and embeddingSkillMap; run or resume the existing backfill/repair path if any row is missing or stale. If the data is healthy but "baidu netdisk" still misses, add an index-backed title/slug token search path or targeted regression test so published skills with matching displayName/slug are discoverable even when vector ranking or bounded recent-digest fallback misses them.
