## Summary

Keep open. Current main exposes package stats and increments package `versions`, but I found no package download/install stat mutation and the package download endpoint returns the ZIP without recording a package stat event. Skill install counts are wired through `clawhub sync` telemetry snapshots, while the direct `clawhub install` path only resolves, downloads, extracts, and writes local metadata. Related PR #1479 only made package stats visible in the detail API, so it does not resolve the missing increment path.

## Best Possible Solution

Keep this issue open as actionable. Add package stat tracking with a package-level event or dedupe path called from package download/install flows, and decide whether direct `clawhub install` should report install telemetry immediately or trigger/reuse the existing sync snapshot path. Cover both package download/install counters and community skill install counters with backend and CLI tests.
