## Summary

Do not auto-close this ClawHub issue. Current main still formats `clawhub sync` update rows as the next publish target version (`latest -> semver.inc(latest)`), which matches the reporter's confusion that sync shows versions that do not yet exist in the registry. Also, the repository policy for openclaw/clawhub says issues should be kept open by this review workflow; only PRs can be auto-closed as implemented.

## Best Possible Solution

Keep this issue open for maintainer triage. The likely product fix is either to change sync output wording so it clearly shows `registry latest -> proposed publish version`, or to adjust sync/update classification if installed unmodified skills are being misdetected as changed. If maintainers decide the behavior is intentional, docs and CLI labels should be tightened so users know `sync` is a publish workflow, not an available-update report.
