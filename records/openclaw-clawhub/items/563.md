## Summary

This is an open issue in openclaw/clawhub, and the supplied cleanup policy says to keep ClawHub issues open. The report also describes a user-specific production account/login and skill-listing problem after `sync --all`; current source shows the sync command publishes selected new/updated local skills, while login/listing visibility depends on production auth, owner, soft-delete, moderation, and publisher state that cannot be verified or repaired from this read-only source review.

## Best Possible Solution

A maintainer should triage this as a production support/account-state issue: inspect the reporter's user record, GitHub auth account mapping, API tokens, publisher membership, audit logs, and the `owncloud-sync` skill fields such as `ownerPublisherId`, `softDeletedAt`, `moderationStatus`, and latest version state. If the skill was hidden or soft-deleted accidentally, restore it or explain the recovery path; if `sync --all` can trigger an unintended publish/moderation/deletion path, open or keep a concrete bug against the CLI sync and Convex skill publish/listing code.
