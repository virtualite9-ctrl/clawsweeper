## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy says ClawHub issues should stay open unless a strong close reason is proven. Current main still treats self-service account deletion as irreversible: deletion deactivates and purges the user, the OAuth callback rejects deleted/deactivated users, and tests/docs assert blocking rather than same-GitHub re-signup.

## Best Possible Solution

Keep this issue open and triage it with the related deleted-account OAuth loop reports, especially #504. Maintainers should decide whether deletion remains irreversible with clearer login/support messaging, or implement a safe same-GitHub re-registration/relink flow that retires the old auth binding without restoring banned accounts. Add regression coverage for delete-account followed by same-GitHub sign-in once the policy is chosen.
