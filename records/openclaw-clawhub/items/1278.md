## Summary

Keep this issue open. It is an issue, and the ClawHub cleanup policy only permits auto-closing PRs when current main definitely implements the intended change. Beyond that policy bar, current main still contains the moderation/search visibility behavior described in the follow-up: quality-quarantined skills are hidden from public search, and `clawhub inspect` still does not surface the owner-visible moderation reason even though the API can expose it.

## Best Possible Solution

Keep this issue open for maintainer triage. The practical fix is to surface owner-facing moderation diagnostics in `clawhub inspect` and/or authenticated skill detail views, including `quality.low` explanations and recovery steps, while coordinating stale tag/SKILL.md behavior with #1407. Public search should continue excluding genuinely hidden skills, but publishers need a clear reason when their own skill disappears.
