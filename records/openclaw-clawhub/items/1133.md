Because this is a ClawHub PR and the policy only allows closing PRs when current main definitely implements the PR, it should stay open for maintainer review.

## Best Possible Solution

Keep this PR open. Maintainers should either review and land a corrected moderation-hold lift implementation, or replace it with an equivalent admin-only mutation that clears `requiresModerationAt`/`requiresModerationReason`, restores only eligible `user.moderation` skills, preserves scanner/ban/manual moderation safety, records an audit log, and documents the operational CLI path. Before merge, address the remaining scan-pipeline review concern for VT `not_found` statuses.
