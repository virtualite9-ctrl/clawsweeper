## Summary

Keep open. This is a ClawHub issue, and the repository policy for openclaw/clawhub says issues are not auto-close candidates. The current code still returns 404 when a requested slug has no public skill record or alias, and the live API still returns 404 for the reported exact slugs, so the user problem is not implemented away on main.

## Best Possible Solution

Keep this issue open for maintainer triage. The practical path is to decide whether these default OpenClaw skill names should be published or restored as verified ClawHub skills, mapped through explicit safe aliases to canonical maintained skills, or removed/renamed in the OpenClaw setup and install documentation so the CLI no longer points users at nonexistent slugs.
