## Summary

Keep open. This is an issue in openclaw/clawhub, and the repository policy for this review only allows auto-closing ClawHub pull requests that are definitely implemented on main. Current main also appears to leave meaningful requested behavior unresolved: explicit `--owner` is wired and backend publisher membership exists, but local publish does not infer org ownership from the scoped package name/source repo, and `owner/repo@ref --source-path` is not applied before fetching the GitHub source subdirectory.

## Best Possible Solution

Keep this issue open and implement a focused package-publish fix: apply `--source-path` to GitHub shorthand sources before archive extraction, add regression coverage for `owner/repo@ref --source-path plugins/foo`, and decide/document org ownership behavior. If scoped package names should imply org ownership, add that inference with permission checks; otherwise update CLI help/docs to make `--owner <org>` the supported flow and remove the stale admin-only wording.
