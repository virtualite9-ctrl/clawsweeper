## Summary

Keep open. This is a ClawHub issue, and the repository policy for this review only allows auto-closing pull requests that are certainly implemented on main. Substantively, current main still has no shipped user-facing recovery path for a GitHub-linked account after user-initiated deletion, and the publish path still surfaces the GitHub API rate-limit error from the account-age gate rather than retrying or offering account recovery.

## Best Possible Solution

Keep this issue open for maintainer triage. The practical path is to split or explicitly resolve the two concerns: improve the publish flow for GitHub API rate-limit failures with clearer retry/recovery guidance, and decide whether deleted GitHub-linked accounts should support controlled recreation/relink/manual restore. If deletion remains permanently irreversible, the UI and support docs should say that clearly and point affected users to the right account review channel.
