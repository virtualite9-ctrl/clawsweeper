## Summary

Keep this issue open. The ClawHub cleanup policy says issues should remain open, and current main still does not implement the reporter’s requested post-deletion re-registration or support purge path. The shipped behavior is intentionally irreversible deletion: the user record is deactivated/purged, GitHub OAuth reauth is rejected, and tests/docs assert blocking rather than recovery.

## Best Possible Solution

Keep this issue open for maintainer triage of the GitHub OAuth/account deletion lifecycle. Maintainers should decide whether self-deleted accounts are permanently blocked or whether same-GitHub re-registration should be supported. If recovery is intended, add a safe backend path to unlink or supersede the old Convex Auth binding, cover it with auth/account lifecycle tests, and provide an admin/support repair path for already-stuck users. If irreversible deletion is intentional, make the hosted login/support flow clearly surface that state and document the support policy.
