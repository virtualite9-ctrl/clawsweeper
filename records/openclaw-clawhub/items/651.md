## Summary

Keep this issue open. It reports a real CLI network failure path, has multiple independent confirmations, and the ClawHub repository policy says issues should not be auto-closed by this workflow. Current main has related hardening for Node 22 and proxy environments, but the discussion still lacks enough concrete environment data to prove the reporter’s failure is fixed or purely external.

## Best Possible Solution

Keep the issue open and request a focused reproduction: CLI version, install method, `node -v`, macOS version, proxy/VPN/corporate TLS details, relevant `HTTPS_PROXY`/`HTTP_PROXY`/`NO_PROXY` values, and whether `curl -i https://clawhub.ai/api/v1/search?q=test` succeeds from the same machine. If reproducible, the likely fix belongs in `packages/clawhub/src/http.ts` or CLI error handling so network/proxy/TLS failures produce actionable diagnostics instead of a bare `fetch failed`.
