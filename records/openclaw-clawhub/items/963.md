## Summary

Keep this PR open. Current main has the GitHub push retry portion, but it does not implement the PR’s primary digest-based backup pagination: `getGitHubBackupPageInternal` still paginates full `skills` documents and joins `skillVersions` and `users` during page construction. The existing review discussion also raised unresolved correctness risks in the PR’s digest approach around empty owner handles, stale owner handles, and deactivated/deleted owners.

## Best Possible Solution

Keep this PR open for maintainer/contributor follow-up. The best path is to revise the digest-based scan so it preserves the old eligibility semantics: use `skillSearchDigest` for the low-byte scan, keep a safe fallback for empty owner handles, revalidate or refresh live owner state before backup path decisions, fetch version files lazily only after the GitHub version mismatch check, and keep the already-present fast-forward retry behavior.

- If the PR is merged as-is, the review comments indicate likely correctness regressions for handle-less owners, renamed owners, and deactivated/deleted owners because the digest data can be empty or stale relative to live user state.
