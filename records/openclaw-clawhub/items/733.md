## Summary

This issue should stay open. Current main contains the older VT stale-flag clearing path, but the named skills are still flagged in production and now appear to be held by the static `suspicious.potential_exfiltration` reason while both VT and OpenClaw/LLM scanners are clean. The repository policy for ClawHub issues also says to keep issues open rather than auto-close them.

## Best Possible Solution

Keep this issue open for maintainer moderation/product action. The concrete path is to either manually clear/recompute the named production records if staff deem the `scripts/setup.js` findings false positives, or implement and ship a narrower policy change that demotes `suspicious.potential_exfiltration` when VT and OpenClaw/LLM are both explicitly clean, with backend, API, UI guidance, regression tests, and a moderation snapshot backfill kept in sync.
