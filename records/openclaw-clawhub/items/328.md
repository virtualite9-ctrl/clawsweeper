## Summary

Keep this issue open. Current main has moderation infrastructure and some false-positive handling, but the reported benign-versus-suspicious split still exists for the cited skill: the live API reports the skill as suspicious because VirusTotal is suspicious while OpenClaw's LLM verdict is benign/clean. The remaining work is product/moderation guidance or tuning, and the repository policy for ClawHub issues is to keep issues open rather than auto-close them.

## Best Possible Solution

Leave this issue open for maintainer product/security judgment. A good resolution would add owner-facing documentation and UI/API guidance explaining why scanner results can disagree, how to declare runtime requirements to reduce false positives, and how to request review. If maintainers confirm this class of instruction-only advisor skill is safe, use the existing audited moderator override for the specific skill and consider narrowly tuning the VT/LLM/static merge without weakening malicious verdict handling.
