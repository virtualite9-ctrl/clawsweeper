## Summary

Keep open. This is a ClawHub issue, and the repository cleanup policy only allows auto-closing ClawHub PRs that are certainly implemented on main. Current main still contains the reported double-serialization bug for `clawhub transfer request`: the command passes a pre-serialized JSON string as `body`, while both HTTP transport paths serialize `args.body` again before sending it.

## Best Possible Solution

Keep this issue open. The concrete fix likely belongs in `packages/clawhub/src/cli/commands/transfer.ts`: pass a plain object as `body` from `cmdTransferRequest` to `apiRequest`, then update the transfer command test or HTTP client test to assert the final POST payload is a JSON object containing `toUserHandle`. Maintainers should also quickly audit adjacent CLI ownership commands for the same payload pattern before cutting the next CLI release.
