## Summary

Keep open. This is a ClawHub issue, and the repository cleanup policy says issues should remain open rather than auto-closed. Current main also does not implement the reporter’s expected deleted-account re-login/relink flow: deletion deactivates and purges the user, OAuth reauth throws a permanent-deletion error, active-user helpers reject deactivated users, and CLI token creation requires an active user. Related #504 remains open for the same account lifecycle problem.

## Best Possible Solution

Keep this issue open for maintainer triage, preferably consolidated with #504. Maintainers should decide whether deleted GitHub-linked accounts should remain permanently blocked as documented, or whether ClawHub should support a safe recreate/relink or admin recovery flow. If the product choice is to allow re-login, the likely implementation areas are Convex Auth deleted-user handling in convex/auth.ts, deleteAccount semantics in convex/users.ts, active-user access helpers, settings/CLI token flows, and matching docs/tests.
