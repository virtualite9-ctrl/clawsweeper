## Summary

Current main does not implement this PR's intended dashboard pagination. The dashboard still fetches skills with the non-paginated `api.skills.list` query and a hard `limit: 100`, and there is no `listDashboardPaginated`, `usePaginatedQuery`, `Load More`, or dashboard pagination styling/test on main. The PR discussion also says dashboard search was removed from the PR for a follow-up, so current main cannot be treated as having shipped the original title's search scope either.

## Best Possible Solution

Keep this PR open or replace it with a revised pagination-only PR against current main. The best implementation path is to add a properly indexed Convex paginated dashboard query, update `/dashboard` to use `usePaginatedQuery` with a Load More control, preserve existing personal-publisher legacy skill behavior, keep owner/non-owner visibility rules consistent with `skills.list`, add focused dashboard pagination tests, and handle dashboard/public search separately.
