## Summary

Keep this PR open. Current main does not implement the PR's intended dashboard pagination: `/dashboard` still calls the fixed-limit `api.skills.list` query with `limit: 100`, there is no `listDashboardPaginated` query on main, and the dashboard has no `usePaginatedQuery`, pagination status, `loadMore`, or dashboard Load More UI. The author also explicitly scoped server-side search out of the PR for a follow-up, so main cannot be treated as having shipped the title's search scope either.

## Best Possible Solution

Keep this PR open or replace it with a revised pagination-only PR against current main. The best path is an indexed Convex dashboard pagination query, a `/dashboard` `usePaginatedQuery` Load More UI, preservation of existing `skills.list` owner and legacy-personal-publisher visibility semantics, focused tests for dashboard pagination, and a separate follow-up for server-side dashboard/public search.
