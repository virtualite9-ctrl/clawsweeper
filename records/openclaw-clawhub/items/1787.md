## Summary

Keep this issue open. ClawHub cleanup policy only permits auto-closing PRs that are certainly implemented on main, and this is an issue. Substantively, current main has staff moderation levers for suspicious-skill false positives, but the requested publisher-level or per-skill clearance is a live moderation/security decision that the source tree does not prove has happened.

## Best Possible Solution

Keep this issue open for maintainer moderation review. Staff can use the existing per-skill manual override path for reviewed false positives now. If maintainers want durable publisher-level clearance for this class of skills, the product path should define an explicit reviewed-publisher policy for suspicious scanner verdicts, with audit logging and tests separate from the existing pending-scan trustedPublisher behavior.
