## Summary

Keep this issue open. The ClawHub repository cleanup policy says issues should remain open, and current main does not show a complete fix for the reported `clawhub update --all` false-positive. The original `.clawhub/origin.json` hypothesis looks partly outdated because `.clawhub/` is ignored, but generated `_meta.json` from downloads still appears able to affect local fingerprints while registry fingerprints are computed from published version files.

## Best Possible Solution

Keep this issue open and fix the CLI integrity comparison so fresh installs and immediate force-updates compare the same canonical file set as registry fingerprints. The likely path is to exclude generated archive/install metadata such as `_meta.json` and `.clawhub/origin.json` from local fingerprinting, then add regression coverage for install, `update --all --force`, and immediate `update --all` reporting clean/up-to-date. Maintainers may also consolidate this with the related open #169 if they want one canonical tracker.
