## Summary

Keep open. Repository policy for openclaw/clawhub says issues should not be auto-closed by this review workflow, and current main still does not implement the reported SKILL.md relative-link behavior. The app can fetch packaged files via the Files tab and raw API endpoint, but the README renderer does not resolve links like `references/system-design.md` to either surface.

## Best Possible Solution

Keep this issue open. Implement a first-class safe relative-link resolver for skill README rendering that recognizes packaged files from the latest version and routes them either to an in-app rendered file viewer or to the existing `/api/v1/skills/{slug}/file?path=` endpoint. Add regression coverage for `references/system-design.md`, URL encoding, query/hash handling, and blocking unsafe schemes such as `javascript:`.
