## Summary

Current main already addresses the Safari/WebKit parse failure that this PR was meant to fix. Instead of pinning Vite minification to esbuild, main uses a Safari-specific Vite transform that rewrites the problematic Ark `in` checks to `Reflect.has(...)` and also targets Safari 15 for the shipped client bundle. The PR diff is obsolete and does not add anything still needed on top of main.

## Best Possible Solution

Close this PR as implemented on main by the broader Safari/WebKit bundle compatibility workaround. Keep the current main implementation; handle the linked Safari issue separately only if maintainers still need production deployment verification or a fresh Safari reproduction.
