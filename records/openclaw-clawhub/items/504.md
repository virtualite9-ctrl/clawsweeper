## Summary

Keep open. This is an issue, and the repository cleanup policy for openclaw/clawhub says to keep issues open. The current main branch also does not implement the reporter’s expected re-link/recreate behavior after deletion: deleted accounts are intentionally marked irreversible, OAuth completion blocks deactivated/deleted users, and CLI/API token auth rejects deactivated users. The discussion includes multiple recent reporters with concrete reproduction attempts, so this is not stale or insufficiently described.

## Best Possible Solution

Keep this issue open for maintainer product decision and implementation. If deleted accounts must remain irreversible, update the OAuth/CLI flows to show a durable, explicit deleted-account message and document the support/manual-cleanup policy. If re-registration should be allowed, add a safe backend path that unlinks or supersedes the old Convex Auth identity without resurrecting banned accounts, preserves handle ownership rules, revokes old tokens, and covers web OAuth plus CLI login with tests.
