## Summary

Keep open. This is an issue, and the repository policy for openclaw/clawhub says automated cleanup should keep issues open. Separately, current main still has an actionable update-flow concern: existing skill publishes can apply the new publish moderation status to the skill-level record, while public search/list/detail paths hide non-active skills. Related PR #274 described the same root cause for this issue and #134, but it was closed unmerged in the provided context.

## Best Possible Solution

Keep this issue open and fix the publish update flow in `convex/skills.ts`: when publishing a new version of an existing active skill, preserve public active visibility while the scan runs unless the new version is malicious, quality-quarantined, or the publisher is under moderation. Add a focused regression test covering an active skill update, search/list/detail visibility, owner moderation state, and slug ownership preservation. Coordinate with the still-open #134 and the closed-unmerged #274 context rather than auto-closing this report.
