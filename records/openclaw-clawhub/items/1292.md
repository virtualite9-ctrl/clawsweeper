## Summary

Keep this PR open. Current main does not implement the PR's intended soul-search parity work, so it cannot be closed as already implemented. The diff is limited to Convex search code and tests, but the latest proposed fallback query should be revised to use the indexed active-souls path now present on main instead of filtering after `by_updated`.

## Best Possible Solution

Keep the PR open for maintainer review. The right path is to rebase it onto current main, preserve the useful soul-search parity work, and change the soul lexical fallback to use `souls.by_active_updated` or an equivalent indexed/cursor path before merging. The tests added by the PR should remain or be expanded to cover fallback results, scoring, deduping, soft-delete exclusion, and incremental hydration.
