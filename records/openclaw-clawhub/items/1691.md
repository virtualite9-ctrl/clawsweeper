## Summary

Keep open. This is a ClawHub issue, and the review policy for this repository says issues should remain open unless they are PRs certainly implemented on main. It is also a concrete live moderation appeal: production still reports `tech-write-assist` as suspicious, with static and VirusTotal scans clean but the LLM scanner flagging undeclared image API keys, possible upload of PDF-derived prompt content to third-party image APIs, and runtime pip installs.

## Best Possible Solution

Keep this issue open for staff moderation review. A maintainer should reply with the current scan findings and suggest concrete remediation: declare the optional image provider env vars in `metadata.openclaw.requires.env`/`primaryEnv` or equivalent metadata, clearly document that image prompts may include PDF-derived content and may be sent to third-party providers, add an explicit opt-in/confirmation or make `--no-image` the safer default for sensitive PDFs, and declare or avoid runtime dependency installation. After a revised publish or manual review, staff can use the existing manual override path if the skill is acceptable.
