## Summary

Keep this issue open. The ClawHub cleanup policy only allows auto-closing pull requests that are definitely implemented on current main, and this item is an issue. Current main has a likely mitigation for the original publish-trigger shape by scheduling package and skill digest sync separately, but the named owner-publisher skill sync helper still loops over `.paginate()` inside one internal mutation, so the broader Convex pagination risk is not conclusively eliminated.

## Best Possible Solution

Keep this issue open for maintainer triage. If production confirms the scheduled split fixed the reporter’s publish failure, a maintainer can manually close it with deployment context. To harden the code path, change the owner-publisher package and skill digest sync internals to accept an explicit cursor, process one page per scheduled mutation, schedule the next page when `isDone` is false, and add multi-page regression coverage.
