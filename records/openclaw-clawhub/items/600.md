## Summary

Keep this issue open. It is a security-incident operations request for a live ClawHub issue, and the ClawHub cleanup policy for this review only permits auto-closing PRs that are definitely implemented on main. Current main has primitives that can block soft-deleted versions, but I did not find a first-class maintainer workflow for safely hiding only one historical skill version, expiring external caches, and reporting exact historical downloads.

## Best Possible Solution

Keep this issue open for maintainer/security handling. Privately verify the affected artifact with the reporter, locate `tencent-doc-update-watch@0.1.0`, mark only that `skillVersions` document unavailable or add a small audited admin/internal mutation to do so, then verify the version detail, raw file, and zip download endpoints block `0.1.0` while `0.1.1` remains available. Separately clear any CDN/proxy caches if configured and privately explain the limits of download/install enumeration from the current schema.
