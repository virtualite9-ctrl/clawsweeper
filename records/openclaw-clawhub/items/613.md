## Summary

Keep this issue open. The ClawHub review policy says issues should not be auto-closed, and current main still lacks the requested device-code/polling authentication flow for headless agents.

## Best Possible Solution

Keep the issue open for maintainer prioritization. The likely implementation is a first-party CLI/web pairing flow or GitHub OAuth device flow: CLI starts a device authorization, prints a verification URL and user code, polls with backoff and expiry handling, the web route authenticates the user with GitHub and issues a scoped ClawHub API token for that code, then the CLI stores it. Add CLI, web route, Convex token, abuse-rate-limit, and docs tests around success, expiry, denial, and polling behavior.
