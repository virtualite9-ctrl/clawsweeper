## Summary

Keep this issue open. Current main has self-service skill soft-delete, transfer, and admin owner/reclaim/hard-delete tooling, but the reporter says the original GitHub sub-account was deleted, so ordinary owner self-service may not be enough. Also, the openclaw/clawhub review policy for this run says to keep issues open and only auto-close PRs that are already implemented on main.

## Best Possible Solution

A maintainer should leave this issue open until Gate confirms `clawhub delete <slug>` works from an authorized owner/admin account, or staff verifies ownership and uses the admin owner-change/reclaim/hard-delete path for the listed skills. If the intended product path is self-service recovery after a deleted publisher account, that should be tracked as a separate ownership-recovery flow rather than closing this support request.
