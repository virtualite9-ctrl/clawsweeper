## Summary

Keep open. Current main has substantial first-party security mitigations for malicious skills, but this is a ClawHub issue and repository policy says issues should not be auto-closed. More importantly, the current implementation still intentionally allows public file reads for malware-blocked skills, so the malicious-code distribution concern is not fully resolved by the shipped moderation pipeline.

## Best Possible Solution

Keep this issue open for maintainer-owned security policy work. The next maintainer decision should be whether malware-blocked skill content should remain publicly readable for transparency or be blocked/redacted across the API and UI; if the current transparency model is intentional, document it clearly. If distribution should be prevented, hard-block raw file endpoints and install/download surfaces for malicious verdicts, preserve scan metadata for transparency, and consider a first-class scanner integration or documented external workflow such as AgentGuard after review.

## Risks / Open Questions

- Auto-closing would require maintainer judgment because this is a security/product policy issue in ClawHub, and the repository policy says ClawHub issues should stay open under this review workflow.
- The current mitigation set reduces exposure but does not fully eliminate distribution: malware-blocked files are still public-readable by design, while the UI says downloads are disabled.
- The external AgentGuard link from the discussion is useful prior art, but it is not merged or documented in current main, so it should not be treated as a shipped core fix.
