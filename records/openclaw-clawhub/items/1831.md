## Summary

This issue is a user request to permanently remove two production ClawHub registry skills, not a source-code cleanup item that should be auto-closed. Current main already distinguishes owner soft-delete from admin-only hard-delete, and the repository review policy for ClawHub says issues should remain open for maintainer handling.

## Best Possible Solution

Keep this issue open for a maintainer/admin to verify the requester’s ownership of the two ClawHub records and then perform the existing admin hard-delete workflow for `adb-assistant` and `testcase-template`, or intentionally define a new documented permanent-removal request process if this should become self-service.
