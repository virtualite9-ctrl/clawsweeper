## Summary

Keep open. Repository policy for openclaw/clawhub only permits auto-closing PRs that are definitely implemented on main, and this item is an active issue. Substantively, current main still treats a VirusTotal suspicious status as a suspicious moderation reason without Code Insight engine-stat calibration, the public ClawHub API still reports skillfactory as suspicious for suspicious.vt_suspicious, and open PR #1841 is the active implementation candidate for this exact false-positive path.

## Best Possible Solution

Keep this issue open. The best path is for maintainers to review and either land or reject PR #1841; if accepted, deploy it and rescan/backfill the affected skill so the stored moderation snapshot can clear the lone Code Insight suspicious reason. If maintainers decide not to change the aggregate logic, they should adjudicate the appeal through the existing moderator manual-override flow.
