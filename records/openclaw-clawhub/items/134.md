## Summary

Keep this issue open. Repository policy for openclaw/clawhub says issues are not auto-close candidates, and current main does not strongly prove the reported publish/update disappearance is fixed: the publish path still applies the initial moderation status to existing skill updates, which can hide non-trusted publishers’ active skills pending scan. The linked fix PR #274 explicitly targeted this issue but was closed unmerged in the provided GitHub context, while #136 only added owner visibility for pending review states.

## Best Possible Solution

Keep this issue open and either land a targeted publish/update fix that preserves active visibility for legitimate existing skills during scanning, or document and implement the intended pending-scan behavior across web, CLI inspect/search, and API responses. The maintainer should also decide whether #131 or another open report should become the canonical tracker before closing duplicates.
