## Summary

Keep this issue open. Current main treats user-initiated account deletion as irreversible and blocks GitHub reauth for the existing Convex Auth user instead of clearing/reusing the OAuth binding, so the reporter's requested recovery/unlink behavior is not implemented. ClawHub policy also says issues should stay open unless a strong non-implemented close reason applies; this is a real auth/account lifecycle request rather than stale or incoherent cleanup.

## Best Possible Solution

Maintainers should decide the desired product policy for deleted-account reauth. If same-GitHub re-signup should be allowed, update the Convex Auth deletion flow to safely unlink or retire the GitHub authAccounts binding during user deletion, add an admin/support mutation for already-stuck accounts, and cover the path with auth tests. If deletion is meant to remain irreversible, keep the code behavior but handle the reporter through support/admin tooling and make the UI/docs clearer before manually closing.
