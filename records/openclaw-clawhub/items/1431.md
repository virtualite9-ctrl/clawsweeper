## Summary

This issue should stay open. The repository policy for openclaw/clawhub says this cleanup pass should keep issues open, and current main does not implement the requested behavior anyway: publishing an existing skill under a different publisher still hits the ownerPublisherId mismatch path and throws the slug-taken error. The related implementation PRs are closed unmerged, while a related user-facing transfer issue remains open.

## Best Possible Solution

Keep this issue open for maintainer/product handling. The likely fix belongs either in convex/skills.ts, where an authorized existing owner can intentionally move an existing skill to a target org publisher while publishing a new version, or in a dedicated publisher-targeted transfer flow that extends skillOwnershipTransfers, the HTTP API, CLI, docs, and UI. The closed unmerged PRs #1562, #1388, #1603, and #1604 are useful implementation references, and #1569 is the active related user-facing report.
