## Summary

Keep this PR open. Current main does not implement the PR’s intended changes: the global stats full-table count path, deprecated manual stats mutation, stale contributing command, and single-key rate-limit counter are all still present. The PR also changes rate-limit abuse-control semantics and needs maintainer review rather than automated closure.

## Best Possible Solution

Keep the PR open for maintainer review. The useful path is an updated change that removes the global stats full-table scan and deprecated manual mutation, updates the seeding docs and affected tests, and either preserves hard rate-limit semantics or explicitly documents/tests an accepted soft-cap tradeoff with production insight validation.
