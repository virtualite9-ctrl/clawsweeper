## Summary

Keep this issue open. Repository policy for openclaw/clawhub says issues are not auto-closed, and current main still matches the reported bug: CLI skill publish collects only files whose final dotted suffix is in the text extension allowlist, so extensionless scripts and .tsv files are silently excluded before upload.

## Best Possible Solution

Keep this issue open and fix the skill publish collection path. A conservative fix should add tsv to the shared text extension allowlist, include extensionless files only when they pass a text/binary sniff such as a null-byte check, emit clear warnings for skipped files, and add focused tests for extensionless scripts, .tsv files, and unsupported binary files. Consider whether related collectors such as zip hashing and GitHub import should use the same text-file policy for consistency.

## Risks / Open Questions

- Adding only tsv would leave extensionless CLI wrappers broken.
- Allowing extensionless files should include a conservative text/binary sniff and focused tests so binary files are not silently accepted or silently dropped.
- Because publish currently succeeds after filtering, authors can continue shipping broken skills without discovering missing files until install or runtime.
