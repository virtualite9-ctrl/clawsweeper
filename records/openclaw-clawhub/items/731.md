## Summary

Keep open. Current main has partial scanner transparency through structured moderation/static-scan data and API docs, but this is an issue and ClawHub policy says issues should remain open. Substantively, the requested local `clawhub scan <path> --format json` flow and remediation-oriented output are still not implemented in the CLI/schema surface.

## Best Possible Solution

Keep this issue open and, if maintainers want to resolve it, extend the published API/CLI surface to expose full scanner findings to owners in a stable machine-readable format, add a local or server-backed `clawhub scan <path> --format json` pre-publish workflow, and include remediation guidance plus whether each finding blocks publish/install or is informational.
