## Summary

Keep open. This is a ClawHub issue, and the repository-specific cleanup policy only permits auto-closing pull requests that are certainly implemented on main. Substantively, current main still allows VirusTotal Code Insight to drive a suspicious status ahead of AV engine stats, the UI does not show a detection ratio, and the named live skill still reports suspicious moderation from VirusTotal.

## Best Possible Solution

Keep this issue open for maintainer/product-security review. The likely fix is to change the skill VirusTotal pipeline in `convex/vt.ts` so AV engine malicious/suspicious counts and Code Insight are represented separately, expose the detection ratio in the API/UI, and avoid presenting `0/N` engine detections as the same generic red suspicious verdict unless maintainers intentionally choose that policy. Add regression coverage for `Code Insight suspicious + 0 engine detections`, and either backfill affected skill versions or use the existing manual override flow after staff review for this specific skill.
