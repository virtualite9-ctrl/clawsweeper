## Summary

Keep open. This is an open ClawHub issue, and the repository policy says ClawHub issues should not be auto-closed. Current main also still models this as production admin support: soft-deleted skill rows occupy the global slug, rename rejects an existing different skill row, and hard-delete/reclaim is admin-only.

## Best Possible Solution

Keep this issue open for maintainer support. A maintainer/admin should verify account and org control, then use the production admin reclaim or hard-delete/reservation path to free or reassign creator-scraper-cv so creativault/cv-creator-scraper can rename. If this pattern recurs, add a product follow-up for self-service slug transfer/reclaim when the same authenticated user controls both publishers.
