## Summary

Keep this issue open. Current main has rate-limit hardening and better 429 guidance, but the original user problem is not fully resolved: anonymous installs are still allowed and IP-limited, the CLI install path does not require or proactively prompt for login, and the web install UI still defaults to `npx clawhub@latest install ...`. The related open #1410 is a narrower follow-up for false-positive anonymous 429s, but ClawHub repository policy also says issues should stay open unless maintainers decide otherwise.

## Best Possible Solution

Keep this issue open for maintainer triage. The best path is to either consolidate it into the narrower open #1410 after maintainer review or make a concrete fix in the ClawHub install UX: require or strongly prompt login before rate-limited install downloads, surface a specific `clawhub login` / token-login hint on 429, and update the web install command guidance if `npx clawhub@latest install ...` remains prone to confusing anonymous rate limits.
