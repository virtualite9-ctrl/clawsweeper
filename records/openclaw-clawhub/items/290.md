## Summary

Keep this issue open. The repository-specific review policy says ClawHub issues should not be auto-closed, and current main still does not clearly support `.ps1` uploads: the shared text-extension allowlists omit `ps1`, the web publish validation rejects files outside that text allowlist unless the browser supplies a text MIME type, and the server/CLI publish paths use the same allowlist-based filtering.

## Best Possible Solution

Keep the issue open and implement explicit PowerShell support as a small, security-aware publish-format change: add `.ps1` and any intended sibling extensions such as `.psm1`/`.psd1` to the shared text-file allowlists, normalize/guess their content type as text, update web archive filtering and CLI file discovery tests, add server publish tests, update the skill-format docs, and extend static moderation/security scanning so PowerShell payloads receive the same scrutiny as shell scripts.

## Risks / Open Questions

- PowerShell scripts are executable code. Adding `.ps1` support should be paired with security-review coverage, because current moderation/code-extension patterns mention shells like `sh`/`bash` but do not appear to explicitly include PowerShell or `.ps1`.
- There are parallel schema copies in `packages/schema` and `packages/clawhub/src/schema`; a fix should avoid web/server/CLI drift and include tests for all publish paths.
