## Summary

This issue should stay open. Current main has partial support for optional environment variable declarations through `envVars` entries with `required: false`, and the moderation/static-scan path treats `envVars` names as declared. However, the public skill-format docs still document only `requires.env`, the security prompt still presents only `requires.env` as required env vars, and the repository policy for ClawHub says to keep issues open rather than auto-close them.

## Best Possible Solution

Keep this issue open and resolve it by making one canonical optional-env policy explicit. The lowest-risk path appears to be documenting `metadata.openclaw.envVars` entries with `required: false`, updating all schema/API/CLI copies that expose skill metadata, teaching the security prompt/docs to distinguish optional from required env vars, and adding focused tests that an optional env var such as `NWC_URL` is displayed as optional and does not trigger an undeclared-env warning. If maintainers prefer the proposed `requires.optionalEnv` spelling, implement it consistently across schema, parser, scanner, docs, and UI instead.
