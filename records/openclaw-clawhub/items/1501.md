## Summary

This issue reports a real CLI/API transfer failure: the command recognized the recipient in the confirmation prompt, but the API returned `toUserHandle required`, and another user later confirmed the same problem. Current main appears to send `toUserHandle` correctly, but the ClawHub review policy says issues should remain open rather than be auto-closed.

## Best Possible Solution

Keep this issue open for maintainer triage.
