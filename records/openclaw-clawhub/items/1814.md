## Summary

Keep open. This is a ClawHub issue, and the repository policy only allows auto-closing PRs that are certainly implemented on main. Substantively, current main still leaves the reported REST/CLI gap: the web publish action resolves ownerHandle with publisher role, but the REST and CLI skill publish paths do not preserve or resolve an org owner target. Open PR #1840 is the active implementation candidate and should be reviewed or replaced before closing this issue.

## Best Possible Solution

Keep this issue open and review/land #1840 or an equivalent fix. The implementation should add an ownerHandle/--owner path for skill publishing, preserve it for JSON and multipart REST payloads, resolve the target publisher server-side with at least publisher role, pass ownerPublisherId into publishVersionForUser, and include regression coverage for schema parsing, CLI payloads, and both REST publish formats.
