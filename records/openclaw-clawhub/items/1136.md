## Summary

Keep this issue open. Current main has in-product reporting, moderator review, and a "Hide suspicious" browse filter, but the suspicious-skill appeal banner still sends owners to GitHub issues and I found no separate appeal venue, issue form, or automatic issue-labeling workflow. For openclaw/clawhub, the provided review policy also says issues should remain open rather than be auto-closed by this run.

## Best Possible Solution

Keep this issue open for maintainer product/workflow judgment. The practical path is to choose an appeal intake mechanism, then update the suspicious-skill appeal banner and either add GitHub issue forms/labels or document a dedicated support venue such as Discord or Discussions.
