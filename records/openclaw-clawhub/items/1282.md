## Summary

Keep this issue open. It is a ClawHub issue, and the supplied repository cleanup policy says ClawHub issues should remain open. Current main also still treats user-initiated deletion as irreversible: deletion deactivates/purges the user, OAuth reauth rejects deactivated users, active-user helpers reject deleted/deactivated users, and docs/tests explicitly cover that blocking behavior rather than implementing a GitHub unbind or self-service re-registration path.

## Best Possible Solution

Keep this issue open for maintainer support and product-policy handling. Maintainers should either manually restore/unbind the affected GitHub identity if allowed, or implement and document a safe account recovery/re-registration path that handles Convex Auth GitHub provider bindings for deactivated users.
