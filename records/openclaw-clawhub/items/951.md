## Summary

Keep open. Current main still advertises onlycrabs.ai as the SOUL.md registry and contains host-based SoulHub behavior for that domain, while the domain currently does not serve the app. Under the provided ClawHub policy, issues should not be auto-closed; this needs a maintainer decision to either connect the domain or stop presenting it as live.

## Best Possible Solution

Keep this issue open for maintainer triage. The concrete resolution should be either to connect onlycrabs.ai to the production app and add a smoke check for the onlycrabs.ai host, or update README/docs to stop presenting onlycrabs.ai as a working SOUL.md registry until that domain is live.
