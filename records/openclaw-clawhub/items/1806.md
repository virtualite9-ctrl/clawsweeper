## Summary

Keep this PR open. Current main does not implement the proposed per-skill pinning workflow: there are no `pin`/`unpin` CLI commands, lockfile entries only contain `version` and `installedAt`, and `update`/`list` do not read or preserve pinned metadata. The linked feature request #1768 is still open, so this PR remains the active implementation candidate rather than something already shipped.

## Best Possible Solution

Review and merge or explicitly close this PR based on maintainer judgment. If accepted, the implementation should add native `clawhub pin`/`clawhub unpin`, persist `pinned` and `pinReason` lockfile metadata, make direct and forced updates honor pins, skip pinned skills during `update --all`, surface pin state in `list`, and keep the schema/docs/tests aligned with that behavior.
