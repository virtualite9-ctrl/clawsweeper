## Summary

This issue should stay open. Current main trims and requires a skill display name, and the UI visually clamps some rendered titles, but there is no product-level title/display-name length limit in the publish form, Convex validation, schema, docs, or tests. Also, the repository policy for openclaw/clawhub says issues should remain open unless maintainers handle them directly.

## Best Possible Solution

Keep this issue open and implement a consistent title/display-name length policy across the publish UI and Convex publish validation. A conservative path would add a shared constant, frontend validation with clear copy and input `maxLength`, backend enforcement before insert/update, tests for both upload-route and Convex publish behavior, and documentation in the public skill format docs.

## Risks / Open Questions

- Maintainers still need to choose the actual limit and whether it is counted in Unicode code points, grapheme clusters, or bytes.
- A hard backend rejection could affect existing skills with long display names unless the rollout includes grandfathering, normalization, or an admin cleanup path.
- The policy should clarify whether the same limit applies to skill `displayName`, `SKILL.md` frontmatter `name`, soul display names, and package/plugin display names.
