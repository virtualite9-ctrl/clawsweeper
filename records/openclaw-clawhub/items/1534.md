## Summary

Keep open. Current main still shows the reported UI inconsistency: dashboard download counts use a down-arrow icon, while public skill surfaces still use Package for download counts. The related fix PR #1550 was closed unmerged in the provided GitHub context, and the ClawHub cleanup policy says issues should remain open rather than be auto-closed by this workflow.

## Best Possible Solution

Keep this issue open and land a small core UI fix that standardizes download-count icons across dashboard and public skill surfaces, preferably using ArrowDownToLine or Download for downloads and reserving Package for versions/packages. PR #1550 is useful prior art but should be reviewed, revived, or superseded because it was closed unmerged.
