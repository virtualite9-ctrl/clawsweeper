## Summary

Keep open. Current main intentionally blocks GitHub sign-in for deleted/deactivated accounts, and the docs say user-initiated deletion is irreversible. Because this is a ClawHub issue rather than a PR, repository policy says it should not be auto-closed; it needs maintainer support or product-policy judgment.

## Best Possible Solution

A maintainer should respond with the current deletion policy and decide whether to handle this as support-only, improve the delete-account warning/error copy, or design an explicit re-registration/reactivation path. If re-registration with the same GitHub account should be allowed, the likely code path is Convex Auth account handling around deleted users, not the normal sign-in UI.

## Risks / Open Questions

- Changing this behavior is not a simple bug cleanup: it would require a maintainer decision about whether deleted GitHub identities may re-register, plus careful handling of existing authAccounts, reserved handles, published skills, telemetry deletion, and abuse/ban boundaries.
- The report is a real user-support case, but current source and docs make the observed sign-in failure expected after deletion, so an automated close would be inappropriate for this repository’s issue policy.
