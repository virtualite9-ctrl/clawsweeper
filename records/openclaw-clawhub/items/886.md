## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy says ClawHub issues should remain open rather than being auto-closed. Substantively, current main still does not implement the requested purge/re-register path after self-deletion: account deletion marks the user deactivated/purged, OAuth reauth rejects deactivated users, tests and docs encode that behavior as irreversible, and admin unban does not restore deactivated accounts.

## Best Possible Solution

Keep this issue open for maintainer support and product-policy handling. The practical path is either a controlled production purge/relink process for affected self-deleted GitHub identities, or a deliberate recovery flow that safely clears or reuses the Convex Auth GitHub binding while preserving ban/deactivation security invariants. If deletion is intended to remain irreversible, the sign-in UX and support docs should clearly direct affected users instead of leaving the report as an unresolved login-loop experience.
