## Summary

Keep this issue open. Current main still treats MIT-0 as the platform-wide ClawHub skill license and does not implement the requested display of standard MIT for this skill. Also, the repository policy for openclaw/clawhub says issues should remain open rather than be auto-closed by this cleanup workflow.

## Best Possible Solution

Maintain this as an open maintainer decision. Either change ClawHub to support per-skill SPDX licenses by parsing/validating package or manifest license metadata, widening the schema/API types, migrating existing versions, and updating UI/CLI/docs/tests; or explicitly reaffirm the platform-wide MIT-0 publishing policy and resolve the publisher-consent/documentation concern raised here.
