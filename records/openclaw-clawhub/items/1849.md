## Summary

Keep this issue open. The ClawHub review policy says issues should not be auto-closed, and current main still has live behavior matching the reporter’s question: non-allowlisted static findings such as child_process command execution produce a suspicious moderation verdict/banner even when OpenClaw’s LLM analysis is benign. This is a valid clarification/product-support request, not conservative cleanup.

## Best Possible Solution

Keep this issue open for maintainer follow-up. The best path is to either document the current policy in user-facing scan docs/UI copy or adjust the moderation/UI model so authors can clearly distinguish benign LLM analysis from an aggregate suspicious verdict caused by static command-execution findings. If maintainers want command-runner skills to clear the banner, they should add an explicit policy/API/design for opt-in runners or dry-run-only defaults and cover it with moderation tests.
