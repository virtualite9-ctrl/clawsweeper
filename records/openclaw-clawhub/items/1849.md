## Summary

Keep open. This is a ClawHub issue, and the repository-specific review policy only permits auto-closing PRs that are certainly implemented on main. ## Best Possible Solution

Keep this issue open for maintainer handling. A good resolution would answer the reporter's questions directly, document that non-allowlisted static findings such as command execution can keep the top-level suspicious banner even when the OpenClaw LLM scan is benign/high confidence, and either manually review/override or rescan the named package if appropriate. If this is intended behavior, improve public guidance and API/UI detail for user-controlled command runners, dry-run defaults, and additional static finding evidence.
