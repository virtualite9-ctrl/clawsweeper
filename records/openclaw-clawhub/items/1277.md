## Summary

Keep open. The repository-specific cleanup policy only permits auto-closing ClawHub PRs that are certainly implemented on main, and this item is an issue. Substantively, current main still does not implement the requested author-declared suppression path for intentional `child_process` spawns: the static scanner still emits `suspicious.dangerous_exec`, external clean results only clear a different allowlisted code, and the docs do not describe `HOOK.md`, `spawns_process`, or `openclaw:allow` as valid suppression mechanisms.

## Best Possible Solution

Keep this issue open for maintainer triage. The best path is either to document that intentional process spawns remain review findings handled by moderator manual override, or to implement a narrow scanner allowlist that validates exact executable/script path, literal args without shell interpretation, constrained environment inheritance, and trusted hook metadata, with tests in `convex/lib/moderationEngine.test.ts` or package publish tests and docs in `docs/skill-format.md` or `docs/security.md`.

- The named published skill still requires separate production moderation judgment; source changes alone would not prove that listing is reviewed or cleared.
