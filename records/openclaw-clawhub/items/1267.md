## Summary

Keep this issue open. The openclaw/clawhub cleanup policy says issues should remain open, and current main still treats user-initiated account deletion as irreversible rather than providing the requested purge, recovery, unlink, or relink path after GitHub OAuth reauthorization. Current main has clearer deleted-account auth messaging, but that does not solve the reporter's inability to sign back in or recover/recreate the account cleanly.

## Best Possible Solution

Keep this open for maintainer triage as ClawHub account lifecycle work. A durable fix should either add a documented admin or self-service recovery path that can safely purge, restore, unlink, or relink GitHub OAuth identities after account deletion, or make the irreversible-deletion policy and OAuth callback failure path explicit enough that affected users do not experience a silent refresh loop.
