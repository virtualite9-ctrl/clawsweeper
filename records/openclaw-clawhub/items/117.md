## Summary

Current main has owner-authorized API and legacy CLI support for soft-deleting and restoring skills, with tests and docs, but the web owner surfaces still do not expose a direct delete action. Because this is a ClawHub issue, repository policy also says to keep issues open rather than auto-close them as implemented.

## Best Possible Solution

Keep this issue open and implement a discoverable owner-facing delete/restore path, likely in the skill owner panel and/or dashboard row actions, backed by the existing `setSkillSoftDeletedInternal` flow or a public owner mutation. Also document the API/CLI workaround clearly until the web UI exists.
