## Summary

Keep open. This is a ClawHub issue, and the repository policy for this review only permits auto-closing PRs that are certainly implemented on main. Independently, current main still has code paths that prioritize VirusTotal Code Insight verdicts over AV engine stats for skill scans, so the central false-positive concern is not clearly resolved.

## Best Possible Solution

Keep this issue open for maintainer work. The likely fix belongs in the VirusTotal skill scan/status pipeline: derive the user-facing VT badge and scanner status from AV engine counts when malicious=0 and suspicious=0, surface Code Insight text as explanatory context rather than a VT detection verdict, add a regression test for 'code_insight suspicious + 0 engine detections', and provide a cursor-based re-sync/backfill for already-flagged skills if the product decision is to clear those rows.
