## Summary

Keep open. This is a ClawHub issue, and the cleanup policy for this repository only permits auto-closing pull requests that are certainly implemented on current main. Current main also does not show a complete fix for alias-equivalent plugin catalog duplicates: the plugin catalog now constrains list/search to code-plugin and bundle-plugin families, but browse/search results are still collapsed only by exact package name or package id, not by canonical package identity or runtime-equivalent aliases like `openclaw-wework` and `@openclaw/wework`.

## Best Possible Solution

Keep this issue open and implement package-level canonical identity handling for plugin browse/search. The likely path is to add or use a canonical package/alias mapping, decide which locator is canonical, collapse list/search results by canonical package identity or runtime plugin id, preserve existing detail/install locator compatibility, add regression tests for `openclaw-wework` versus `@openclaw/wework`, and run a data migration/backfill for historical duplicate records.
