## Summary

Keep this issue open. It is an issue, and the repository policy for openclaw/clawhub says only PRs that are certainly implemented on main are eligible for auto-close. Current main also does not implement the requested account restoration or GitHub identity unlink path: deletion deactivates/purges the user, OAuth sign-in blocks deleted/deactivated users, tests assert that behavior, and docs describe deletion as irreversible.

## Best Possible Solution

Keep this issue open for maintainer support and account-lifecycle product judgment. If deletion is intended to remain irreversible, maintainers should reply with that policy and any private support path. If same-GitHub re-registration should be allowed, add a safe backend/admin flow around Convex Auth authAccounts that can unlink or supersede the deleted account without restoring banned users, with tests for web OAuth, CLI/API token behavior, handles, publishers, and existing deleted-account records.
