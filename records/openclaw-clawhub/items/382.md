## Summary

This should stay open. The repository policy for openclaw/clawhub keeps issues open, and current main still has a credible product gap matching the report: publish writes a skill-level pending/hidden moderation state for non-trusted publishers, while public search hydrates through public skill visibility that rejects non-active skills. The existing `latest-approved` embedding machinery does not by itself keep an already-approved public version searchable while the skill-level record is hidden for a new version review.

## Best Possible Solution

Keep this issue open and implement a version-aware review flow: when an existing approved skill publishes a new version, keep the last approved public version/search digest/embedding visible until the new version scan clears, or skip re-review when the content fingerprint and scanner inputs are unchanged. Add regression coverage for a non-trusted publisher updating an approved skill and verify search still returns the skill during pending review.
