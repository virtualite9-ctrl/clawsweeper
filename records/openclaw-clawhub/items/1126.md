## Summary

Keep open. This is an open ClawHub issue, and the repository-specific review policy only permits auto-closing ClawHub pull requests that are certainly implemented on main. Current main does cover exact slug lookup through an indexed `skills.by_slug` path, but the broader `checkpoint` keyword/prefix failure remains plausible because search still depends on vector recall plus recent digest fallback windows rather than an indexed lexical prefix candidate path; related open #1254 tracks that remaining gap.

## Best Possible Solution

Keep this issue open as a concrete repro attached to #1254. The maintainer path is to add an indexed direct lexical candidate path over normalized slug/display-name fields, or an equivalent robust prefix lookup, so both exact slug and keyword/prefix searches work without relying on vector recall or recent digest windows; then verify CLI/API behavior and close manually under the ClawHub issue policy.
