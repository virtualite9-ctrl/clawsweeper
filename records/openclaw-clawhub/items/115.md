## Summary

Keep this issue open. The ClawHub review policy only permits auto-close for pull requests that are certainly implemented on main, and this is an issue. Current main has the expected CLI/API auth plumbing and regression coverage for successful authenticated publish/whoami paths, but I did not find a maintainer-confirmed fix or release note that specifically resolves the reported Unauthorized-after-login sequence from CLI 0.5.0.

## Best Possible Solution

Keep this issue open for maintainer triage. Reproduce with the current published CLI and production API, verify whether login-created tokens are persisted and accepted by both `/api/v1/whoami` and `POST /api/v1/skills`, and add a focused regression/logging around token hash lookup, user linkage, and registry discovery if publish still returns 401. If the real failure is a publish precondition such as GitHub account age, return that specific error instead of Unauthorized.
