## Summary

Keep open. This is a ClawHub issue, and the supplied repository policy only allows auto-closing PRs that are certainly implemented on main. Substantively, current main still renders OpenClaw scanner failures as a bare Error status, suppresses the stored LLM error summary in the scan panel, and lacks author-facing docs or UI explaining what to do next.

## Best Possible Solution

Keep this issue open for maintainer triage. Add scan-page and public docs guidance for OpenClaw `Error`, expose a safe error explanation or stored summary to authors, document whether authors should wait, change the skill, or request a rescan, and decide a conservative moderation policy that separates scanner-side LLM failures from actual suspicious evidence. Add tests for OpenClaw error rendering and moderation behavior before backfilling or reviewing affected skill versions.
