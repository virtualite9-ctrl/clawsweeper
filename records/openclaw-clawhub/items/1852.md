## Summary

Keep open. This is a ClawHub issue, and the repository policy only allows auto-closing ClawHub pull requests that are definitely implemented on main. Substantively, the request is also a security/product architecture proposal for runtime trust boundaries, skill execution authorization, and inter-instance communication; current main has registry-side package metadata, scans, trust labels, and audit logs, but no PAP/principal/transport-layer authorization implementation.

## Best Possible Solution

Keep this issue open for maintainer triage. The best path is to decide whether PAP-style trust belongs in OpenClaw runtime/security architecture, ClawHub package metadata/signing, or both, then define the smallest core authorization/audit boundary needed before any ClawHub registry or plugin integration work.
