## Summary

Keep this PR open. Current main only partially covers optional skill env declarations: the backend/shared schema and static moderation path already understand `envVars`, but the public skill-format docs still describe only `requires.env`, the CLI schema copy in `packages/clawhub` still omits `envVars`, and the security prompt user message does not enumerate optional env declarations. Therefore current main does not definitely implement this PR's intended docs/schema/prompt/test cleanup.

## Best Possible Solution

Keep this draft PR open and review it as the implementation candidate for the linked optional-env issue. The best path is to land a consistent optional-env policy, likely `metadata.openclaw.envVars` entries with `required: false`, across docs, the CLI schema copy, security prompt text, scanner behavior, and focused regression tests.
