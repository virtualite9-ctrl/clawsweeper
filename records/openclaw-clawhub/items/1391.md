## Summary

Keep this issue open. Current main partially addresses the `.git/` half for CLI-collected uploads, but I did not find any built-in `__pycache__/` or Python bytecode exclusion, and package publishing can still upload arbitrary non-ignored files that are later zipped for VirusTotal scanning. The repository policy for openclaw/clawhub also says issues should not be auto-closed by this workflow.

## Best Possible Solution

Keep the issue open and implement a shared publish-artifact ignore policy. Add `__pycache__/` and Python bytecode patterns such as `*.pyc`/`*.pyo` to package collection, skill collection where relevant, and preferably server-side package normalization before files are persisted or scanned. Add focused tests for local package publish, GitHub-archive publish, and server/API handling, while preserving the existing `.git/` exclusion behavior. Document `.clawhubignore` as a workaround until the built-in exclusion ships.
