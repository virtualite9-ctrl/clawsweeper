## Summary

This is an open ClawHub issue about a specific publisher account, skill visibility, and production access. The repository-specific policy says ClawHub issues should remain open rather than be auto-closed, and current main does not prove this Cybercentry account/search problem has been resolved. Public search currently returns no Cybercentry results, while the source shows several legitimate production states that could explain it, including hidden/non-active skills, suspicious filtering, banned/deleted accounts, revoked tokens, or rate limits.

## Best Possible Solution

Keep this issue open for ClawHub staff handling. Staff should look up the Cybercentry user/publisher in the management console or admin API, check deletedAt/deactivatedAt/banReason/audit logs/API token state, inspect any owned skills for softDeletedAt, moderationStatus, moderationFlags/isSuspicious, and skillSearchDigest/embedding visibility, then restore/unban/apply a reviewed manual override or explain the policy outcome. If the account is active and skills are public but still absent from search, open a narrower search-index/digest bug with the affected slugs and timestamps.
