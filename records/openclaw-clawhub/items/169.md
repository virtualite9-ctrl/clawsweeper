## Summary

This ClawHub issue should stay open. Current main still has a plausible clean-install fingerprint mismatch: downloads add a generated `_meta.json`, install extracts it, and update hashes local text files including JSON files, while the server resolves fingerprints computed only from published skill files. Also, the review policy for openclaw/clawhub says issues should not be auto-closed by this workflow.

## Best Possible Solution

Fix the core CLI/backend fingerprint contract, most likely by excluding generated `_meta.json` from local skill hashing or by making generated download metadata part of the stored server fingerprint consistently. Add a regression test that installs a downloaded skill zip containing `_meta.json`, immediately runs update, and gets the clean/up-to-date path rather than `local changes (no match)`.
