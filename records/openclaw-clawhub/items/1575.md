## Summary

This issue should stay open under the ClawHub review policy, and the requested org ownership transfer is not implemented on current main. Search has exact-slug and lexical fallback paths that may help the indexing symptom, but the report also asks for a live skill transfer to an organization, and current transfer APIs and schema are still user-to-user.

## Best Possible Solution

Keep this issue open. Maintainers should either handle the named production skill operationally or implement publisher-target skill transfers: extend the transfer schema/API from user IDs to source/target publisher IDs, authorize owner/admin membership on the target publisher, update skill/alias/digest ownership consistently, and add tests for user-to-org, org-to-user, and org-to-org transfers. Separately, verify the specific skill's production search digest/embedding state and backfill or repair it if needed.
