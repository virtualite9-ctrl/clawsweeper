## Summary

Keep open. This is a ClawHub issue, and the repository policy for this review only permits auto-close for PRs that are definitely implemented on main. Independently, this is a live moderation appeal for four hosted skills that production ClawHub still reports as suspicious, so it needs maintainer/security review rather than conservative cleanup closure.

## Best Possible Solution

Keep this issue open as a moderation review ticket. A maintainer or security moderator should inspect the four published skill bundles and the linked Claws-ZH/claws-shield implementation, then either leave the suspicious verdicts with concrete remediation guidance, ask the publisher to republish with vendored or pinned dependencies and complete metadata, or apply the existing manual override path per skill if the reviewed artifacts are acceptable.
