## Summary

Keep open. This is an issue in openclaw/clawhub, and the supplied repository policy says ClawHub issues should remain open. The current main branch also does not fully implement the requested prefix/exact registry discovery workflow: search is still a ranked `/api/v1/search` wrapper, `list` only reads local installed skills, and `explore` lists registry skills but lacks prefix/exact filtering and CLI cursor pagination.

## Best Possible Solution

Keep the issue open and implement an indexed registry-discovery path in the ClawHub CLI/API, such as `clawhub search --exact`, `clawhub search --prefix`, or a registry-backed `list`/`explore` mode with cursor pagination and JSON output. The backend should use an indexed slug-oriented query rather than relying on ranked search output, and the change should include CLI command tests plus CLI/API docs.
