## Summary

Keep this issue open. The ClawHub cleanup policy for this repository only auto-closes PRs that are certainly implemented on main, and this is an issue. Substantively, current main still has the exact auth-vs-anonymous split described: authenticated downloads use a user bucket, while anonymous downloads depend on a client-IP bucket that falls back to `ip:unknown` when no trusted IP header is available. The issue also overlaps open rate-limit reports #349, #539, and #589, but those are open related issues rather than a close reason under the ClawHub policy.

## Best Possible Solution

Keep this issue open for maintainer or ops follow-up. Verify what headers reach Convex for anonymous `https://clawhub.ai/api/v1/packages/.../download` requests through the Vercel rewrite, preserve a real client IP or enable `TRUST_FORWARDED_IPS` only behind a trusted proxy, and add a smoke/log check that distinguishes `cf-connecting-ip`, forwarded headers, and `ip:unknown` denials. If OpenClaw still hides `Retry-After` or remaining-budget headers, track that client-side display fix in the OpenClaw install path while keeping ClawHub’s existing rate-limit metadata behavior.
