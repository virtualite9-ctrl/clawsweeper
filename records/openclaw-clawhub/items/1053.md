## Summary

Keep open. This is a ClawHub issue, and the repository-specific cleanup policy only permits auto-closing pull requests that are certainly implemented on main. Substantively, current main still documents and parses env requirements from SKILL.md frontmatter/metadata.openclaw, but I did not find evidence that standalone metadata.json env declarations are ingested into the normalized registry/LLM summary path.

## Best Possible Solution

Keep this issue open for maintainer triage. The best product path is to decide whether standalone metadata.json env declarations are supported; if yes, parse the uploaded metadata.json during publish/backfill into the same parsed.clawdis.requires.env and envVars shape consumed by latestVersionSummary, detail UI, static scan, and the LLM prompt, with tests for metadata.json plus SKILL.md frontmatter. If standalone metadata.json is not supported, update docs and CLI/server validation to make frontmatter-only support explicit.
