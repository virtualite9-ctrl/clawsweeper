## Summary

Keep this issue open. The ClawHub cleanup policy says to keep issues open, and the current product behavior still matches the reporter’s concern: the live skill reports `capabilityTags:["crypto","requires-sensitive-credentials"]` while static, VirusTotal, and OpenClaw scanner verdicts are clean/benign. Current main shows the Crypto signal is a deterministic regex tag derived from all text files in the skill bundle, but the public docs only list possible tags and do not explain the trigger criteria.

## Best Possible Solution

Keep the issue open for maintainer follow-up. The best fix is to tighten `convex/lib/skillCapabilityTags.ts` so generic terms like `swap` require crypto context or ignore known CSS/font-display URLs, then document capability-signal criteria and ideally expose matched evidence in the author/security UI so skill authors can understand and correct false positives.
