## Summary

Keep open. Repository policy for this review keeps ClawHub issues open, and current main still only partially addresses the report: the web detail prompt can show an owner-qualified target, but CLI search/API search omit owner data, CLI install is slug-only, and download filenames remain slug-version only.

## Best Possible Solution

Keep this issue open and implement a scoped identity clarification: expose ownerHandle/owner in /api/v1/search, update the CLI search schema and output to show owner/slug, decide whether clawhub install should accept owner/slug or @owner/name while preserving slug-only compatibility, and consider owner-qualified download filenames or stronger web copy for the exact canonical skill page.
