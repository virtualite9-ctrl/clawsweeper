## Summary

Keep open. Current main still models skill ownership transfers as user-to-user only, so transferring a personal skill to an organization is not implemented yet. The provided discussion also points to open related work in #1635 and an open related report in #1431, so this issue should remain open until the ownership-transfer work lands or maintainers choose a canonical tracker.

## Best Possible Solution

Keep this issue open and use #1635, or a maintainer-selected successor, to implement publisher-target ownership transfers in core: schema fields for source/target publishers, API/CLI support for org handles, owner/admin acceptance rules for org publishers, and tests covering user-to-org, org-to-user, and org-to-org transfers. Once that work is merged, close this issue and #1431 as implemented or consolidate them under the canonical tracker.
