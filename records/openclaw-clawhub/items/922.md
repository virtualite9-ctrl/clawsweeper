## Summary

Keep open. This is an open ClawHub issue, and the supplied repository policy only permits auto-closing ClawHub PRs that are certainly implemented on main. Current main also does not provide a self-service way to move a soft-deleted personal skill slug to an org/company publisher: deletion keeps the slug row, publish still treats an existing slug owned by a different publisher as taken, and transfer/reclaim paths remain admin/user-owner oriented rather than a user-facing personal-to-org slug transfer flow.

## Best Possible Solution

Keep this issue open for maintainer triage. A maintainer should handle the specific `pixverse` production ownership request, then choose a product path: document that deleting a skill does not free its slug and route users to support, or add a safe self-service flow to undelete/transfer a soft-deleted personal skill slug to an authorized org/company publisher by extending ownership transfer to publisher targets.
