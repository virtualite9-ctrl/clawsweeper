## Summary

Current main does not definitely implement this PR’s intended change. `lexicalFallbackSkills` now scans lightweight `skillSearchDigest` rows instead of full `skills` documents, but when `nonSuspiciousOnly` is set it still reads active updated/created digest windows and filters `isSuspicious` in JavaScript. Under the ClawHub policy for PRs, this should remain open for maintainer review or rebase rather than be auto-closed.

## Best Possible Solution

Keep this PR open. The best path is to rebase or rewrite it against current main so `lexicalFallbackSkills` uses `skillSearchDigest` non-suspicious indexes for both updated and created scan windows when `nonSuspiciousOnly` is set, while preserving the current digest-based fallback, highlighted/capability filtering, and search pagination semantics.
