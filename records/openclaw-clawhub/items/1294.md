## Summary

Keep open. This is an issue, and the repository policy for openclaw/clawhub says not to auto-close issues. The substance is also still unresolved on current main: the reporter is asking for maintainer verification and transfer or removal of the global `mapbox` package/name, while the code still treats package names as globally keyed by `normalizedName` and rejects publishes from a different owner/publisher.

## Best Possible Solution

Keep this issue open for maintainer handling. A maintainer should verify the reporter’s authority for the Mapbox GitHub organization or mapbox.com domain, inspect the current production owner of the `mapbox` package, then either transfer package/publisher ownership to the verified Mapbox publisher, remove or soft-delete the squatted package so Mapbox can publish, or add a narrowly scoped admin package reclaim/transfer path with audit logging. Longer term, the org/publisher namespace migration docs should explicitly cover trademark and official-publisher reclaim cases.
