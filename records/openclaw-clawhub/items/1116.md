## Summary

Keep open. This is a ClawHub issue, and the repository cleanup policy says ClawHub issues should remain open unless maintainers handle them manually. Current main also does not implement the reporter's requested same-GitHub re-registration/relink path after account deletion: deletion marks the user permanently deactivated/purged, OAuth completion blocks deleted/deactivated users, docs describe deletion as irreversible, and related open reports describe the same account lifecycle gap.

## Best Possible Solution

Keep this issue open for maintainer product/support triage. Maintainers should decide whether user-initiated deletion is truly irreversible or whether same-GitHub re-registration should be supported. If recovery is intended, add a backend path that safely unlinks or supersedes the old Convex Auth identity for self-deleted users only, preserves ban and ownership protections, revokes old tokens, and covers web OAuth plus CLI/API token flows with tests. If irreversible deletion remains the policy, improve the OAuth/UI/support path so affected users see a clear deleted-account message and know whether manual cleanup is possible.
