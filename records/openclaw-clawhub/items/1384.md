## Summary

Keep this issue open. Current main does not include GitHub issue forms, issue-template configuration, or issue-label triage automation, and ClawHub review policy says issues should not be auto-closed by this cleanup workflow.

## Best Possible Solution

Add committed GitHub issue forms under `.github/ISSUE_TEMPLATE/` for the requested ClawHub support buckets, include required metadata fields and `labels:` metadata where appropriate, add `config.yml` if maintainers want to steer users to canonical threads, and create or standardize the corresponding repository labels.
