## Summary

Keep open. This is a ClawHub issue, and the repository policy only permits auto-closing PRs that are certainly implemented on main. The requested uppercase enforcement also is not implemented on current main: docs and publish/import paths still explicitly accept lowercase `skill.md`/`skills.md`, and no `PROTOCOL.md` filename validation path was found.

## Best Possible Solution

Keep this issue open for a maintainer decision and implementation plan. A complete fix would add case-insensitive duplicate detection and canonical uppercase validation across CLI publish/sync scanning, browser upload, GitHub import, API publish, server-side publish validation, docs, and any existing published-content cleanup path; it should also decide whether legacy `skills.md` compatibility remains supported.
