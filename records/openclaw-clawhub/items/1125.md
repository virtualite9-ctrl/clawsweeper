## Summary

Keep this PR open. Current main does not implement the intended `clawhub install` OS compatibility check, so it cannot be closed as implemented. The PR also has unresolved functional review concerns that should be addressed before merge.

## Best Possible Solution

Keep the PR open and revise it rather than closing it. The maintainer path is to add optional API `metadata` parsing, normalize host OS aliases to the documented metadata vocabulary, validate against the resolved version when `--version` is used, preserve malware blocking semantics, and add focused `cmdInstall` tests for allowed, rejected, forced, missing metadata, and version-specific metadata cases.

## Risks / Open Questions

- The PR diff shown in the GitHub context makes `metadata` required in `ApiV1SkillResponseSchema`, matching the bot review concern that installs could fail for API responses that omit the field.
- The PR compares Node platform values such as `darwin`/`win32` against documented metadata values such as `macos`/`windows`, so it may incorrectly reject supported macOS or Windows installs unless normalized.
- The proposed check appears to use latest-version metadata before resolving `--version`, so installs of an older requested version could be validated against the wrong release metadata.
