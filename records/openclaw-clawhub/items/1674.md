## Summary

Keep open. This is a live ClawHub account/moderation appeal for production restoration, and the ClawHub cleanup policy says issues are not auto-closed. Current main has ban/unban and automatic publisher moderation-hold machinery, but I did not find a shipped admin flow that clearly resolves the reported account hold and restores skills hidden as user.moderation; the related open PR #1133 is still the obvious implementation candidate from the provided context.

## Best Possible Solution

Keep this issue open for maintainer/security triage. Staff should inspect the production user record for PHY041, determine whether the block is a deletedAt/banReason ban or a requiresModerationAt publisher hold, use the existing admin unban path where it applies, and review or land an audited moderation-hold recovery path like #1133 for user.moderation skills. The maintainer response should also document safe publishing limits and an appeal path so this cluster stops repeating.
