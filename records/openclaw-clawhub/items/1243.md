## Summary

Keep this issue open. The repository policy for openclaw/clawhub only permits automated closes for PRs that are certainly implemented on current main, and this item is an issue. Current main also still has an actionable search-recall gap: skill tags are persisted in the search digest, but lexical matching only checks display name, slug, and summary, so a skill can still miss searches for explicit tag terms like phone or voice when vector recall does not retrieve it.

## Best Possible Solution

Keep this issue open, or have maintainers manually consolidate it with the broader open search-recall work in #1254. The product fix likely belongs in convex/search.ts and the search digest schema: add an indexed or otherwise bounded lexical candidate source over normalized searchable fields that includes tags/capability tags, preserve existing nonSuspiciousOnly behavior, and add focused tests for tag queries such as phone and voice.
