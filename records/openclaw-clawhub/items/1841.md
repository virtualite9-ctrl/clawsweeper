## Summary

Keep this PR open. Current main does not implement the PR's intended VirusTotal Code Insight calibration: the moderation aggregate still only receives a plain VT status, still adds `suspicious.vt_suspicious` for any suspicious VT status, and does not persist `scanner`/`engineStats` on stored `vtAnalysis`. The PR remains a live implementation candidate for the false-positive path reported in #1830.

## Best Possible Solution

Keep this PR open for maintainer review. If the moderation tradeoff is acceptable, land a version of the PR that persists VT scanner provenance and AV engine stats, passes `vtAnalysis` into the aggregate moderation snapshot, suppresses only uncorroborated Code Insight suspicious results, and keeps regression coverage for both clean and corroborated suspicious cases. Existing production false positives like #1830 may still need rescan/backfill or manual moderation after deployment.
