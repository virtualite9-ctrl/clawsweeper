## Summary

Keep this PR open. Current main does not implement the PR’s intended VT Code Insight calibration: moderation still treats any VT `suspicious` status as `suspicious.vt_suspicious`, and persisted `vtAnalysis` does not carry the scanner provenance or AV engine stats needed for the proposed suppression gate. The PR is an implementation candidate for the false-positive path described in #1830, not obsolete cleanup.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land this PR or an equivalent patch that persists VT scanner provenance and AV engine stats, suppresses only uncorroborated Code Insight `suspicious` results, preserves malicious and AV-corroborated suspicious behavior, and includes the regression tests from the PR. After merge and deployment, affected existing rows such as the #1830 false positive may still need rescan/backfill or manual moderation re-evaluation.
