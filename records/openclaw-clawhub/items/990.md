## Summary

Keep open. This is a ClawHub issue, and the supplied repository policy for this cleanup pass only allows auto-closing ClawHub pull requests that are certainly implemented on main. Substantively, current main exposes suspicious/moderation state through public v1 API surfaces and documents them, but the requested static HTML/meta signal for web-fetch consumers is still absent from the skill page head metadata.

## Best Possible Solution

Keep this issue open for maintainer product handling. The concrete implementation path is to carry public `moderationInfo` through `fetchSkillPageData`/route loader metadata into `buildSkillMeta` or the route `head`, then emit a stable static tag such as `clawhub:suspicious` plus a public reason/verdict summary when moderation data is public. Maintainers should also decide whether API docs should direct agents to `/api/v1/skills/{slug}`, `/moderation`, and `/scan`, or whether list/search responses should expose a minimal moderation field.
