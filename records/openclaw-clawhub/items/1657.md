## Summary

This PR should stay open. Current main does not implement phantom dependency detection for skill bundle dependency files, and ClawHub policy only auto-closes PRs when the intended PR change is definitely implemented on main.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to either land a corrected implementation of dependency-registry existence scanning, with tests for npm scoped packages, transient registry failures, direct URL/alias dependency forms, optional dependencies, and pyproject formats, or replace it with a new PR that addresses the same supply-chain risk and coordinates with the linked trust report.
