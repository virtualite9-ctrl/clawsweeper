## Summary

Keep this issue open. Repository policy keeps ClawHub issues open, and current main still lacks the requested security-tool category/context mechanism. There is also an open implementation PR (#1132) paired with this issue, with #1133 covering the related recovery path.

## Best Possible Solution

Keep this issue open and have maintainers review PR #1132 and PR #1133, or replace them with a narrower maintained design. The likely implementation belongs in the moderation reason-code pipeline, static scanner snapshot logic, LLM evaluation context/prompt, skill-format docs, and an admin recovery path for existing moderation holds, with tests proving malicious codes are never suppressed.
