## Summary

Keep open. This is a ClawHub issue, and the repository policy only allows auto-closing ClawHub pull requests that are definitely implemented on main. Substantively, current main still has package publish dry-run metadata preview and server-side static publish scanning, but no local plugin lint/scanner CLI that reports pre-publish manifest, SKILL.md, env, provider/UI hint, and code/security findings.

## Best Possible Solution

Keep this issue open. The best product path is to add a local ClawHub CLI command such as `clawhub package lint <source>` or a stronger explicit pre-publish dry-run mode that reuses shared schema and static-scan logic, supports `--json`, emits severity-coded findings, and exits non-zero for blocking publish errors.
