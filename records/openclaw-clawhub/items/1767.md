## Summary

Keep open. Current main has a `clawhub package publish --dry-run` preflight with limited structural validation, and server-side static/package scans run during publish, but there is no local scanner/linter command that mirrors ClawHub scan findings, validates the requested manifest/SKILL metadata issues, emits severity findings, or provides CI-friendly lint exit behavior before publishing.

## Best Possible Solution

Keep this issue open. The best path is to add a local package validation command, likely `clawhub package lint <source>` or a stronger `clawhub package publish --dry-run --pre-publish`, that reuses shared package schema validation and as much static scan logic as practical, reports severity-coded findings, supports `--json`, and exits non-zero for blocking errors. The implementation should explicitly cover manifest shape, package/openclaw compatibility, config schema installability, SKILL.md env metadata alignment, and code/security heuristics currently available only after publish.
