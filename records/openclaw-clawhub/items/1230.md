## Summary

Current main does not fully implement this PR's intended change. The desktop header still uses the original `ToggleGroup` root `onValueChange` path without the PR's per-item click handling or pointer/value-change coordination, and the legacy `clawdhub-theme` storage fallback is still present and tested as active. Under the ClawHub policy, this PR should stay open unless current main definitely implements the PR's intended behavior.

## Best Possible Solution

Keep this PR open for maintainer review, or replace it with a merged implementation that restores desktop theme switching across pointer and keyboard paths and decides the legacy `clawdhub-theme` migration consistently across `src/lib/theme.ts`, `src/routes/__root.tsx`, and tests. Because related issue #1228 is still open and this PR is its paired implementation candidate, the right cleanup path is to land or supersede the implementation before closing either item.
