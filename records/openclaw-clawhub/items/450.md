## Summary

Keep this issue open. Repository policy for openclaw/clawhub says issues should remain open in this cleanup flow, and the underlying false-positive report is still observable: the live Laravel Forge scan remains suspicious solely from `suspicious.vt_suspicious` while the OpenClaw LLM scanner is clean/high confidence. Current main also still prioritizes VirusTotal Code Insight verdicts over AV engine stats when Code Insight exists, so the requested policy change is not implemented on main.

## Best Possible Solution

Keep this issue open and handle it with the broader VT Code Insight false-positive work tracked by related reports such as #379, #733, and #838. A maintainer-quality fix would update the VT/moderation policy in `convex/vt.ts` and `convex/lib/moderationEngine.ts` so Code Insight `suspicious` with 0 malicious/suspicious engine detections plus a clean high-confidence OpenClaw scan is either downgraded, displayed as a high-risk-capability warning rather than a suspicious banner, or routed to staff review. Add regression tests for this exact signal combination and then rescan or manually clear affected skills such as Laravel Forge.
