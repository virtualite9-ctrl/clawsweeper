## Summary

Keep open. This is a ClawHub issue, and the repository policy for this review pass only permits auto-closing pull requests that are certainly implemented on main. The underlying plugin/package search endpoint is also still returning HTTP 500 in production, even though the `/plugins?q=pay` page itself now returns HTTP 200 and displays a handled error state instead of the raw server error.

## Best Possible Solution

Keep this issue open and treat it as a real plugin catalog search outage. The best maintainer path is to debug the Convex package/plugin search endpoint used by `/api/v1/plugins/search`, confirm the failing production stack trace, add a regression test around the endpoint with representative packageSearchDigest data, and close only after both `/plugins?q=pay` and the API endpoint return search results in production.
