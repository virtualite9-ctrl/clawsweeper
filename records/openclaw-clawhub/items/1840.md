## Summary

Keep this PR open. Current main does not yet implement the PR's intended REST/CLI path for org-owned skill publishes: `/api/v1/skills` still publishes directly as the API-token actor's personal publisher, the shared skill publish schema/parser do not preserve `ownerHandle`, and the skill CLI publish command has no `--owner` flag. The PR remains a live implementation candidate for the open org-owned skill publishing bug it references.

## Best Possible Solution

Review and land this PR, or an equivalent patch, so `/api/v1/skills` accepts an optional `ownerHandle`, resolves the target publisher server-side with publisher-role authorization, threads the resulting `ownerPublisherId` into `publishVersionForUser`, and exposes the same capability through `clawhub publish --owner` and `clawhub skill publish --owner`. Keep the paired issue open until the implementation merges.
