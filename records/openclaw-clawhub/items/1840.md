## Summary

Current main does not implement this PR's intended change. The REST/CLI skill publish path still lacks `ownerHandle` support, while the authenticated web publish action already has the server-side publisher resolution pattern the PR is trying to extend to API/CI publishing. The PR should remain open for maintainer review and merge, with the linked org-owned skill publishing issue closing only after this or equivalent code lands.

## Best Possible Solution

Keep this PR open and review/merge it or an equivalent patch that threads `ownerHandle` through the skill publish schema, JSON and multipart REST parsing, CLI `--owner`, and server-side publisher resolution using the existing `resolvePublishTargetForUserInternal` membership check. After it lands, the linked org-owned REST/CLI publishing issue can close as implemented.
