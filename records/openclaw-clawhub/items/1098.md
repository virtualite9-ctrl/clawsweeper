## Summary

Keep this issue open. Current main appears to fix the default `clawhub explore` path by discovering and using `https://clawhub.ai/api/v1/skills`, and the live canonical endpoint currently returns skills. However, the repository policy for openclaw/clawhub says issues should remain open for maintainer triage, and there is still a plausible explicit/stale `registry.clawhub.ai` edge because that host still returns Vercel `DEPLOYMENT_NOT_FOUND` and is not classified as a legacy registry host in current main.

## Best Possible Solution

Keep this issue open for maintainer triage. The likely code/docs path is to either add `registry.clawhub.ai` to the legacy registry migration logic in `packages/clawhub/src/cli/registry.ts` and document clearing/upgrading stale registry config, or confirm that explicit use of the dead host is unsupported and close manually once production behavior is verified.
