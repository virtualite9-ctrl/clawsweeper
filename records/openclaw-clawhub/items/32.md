## Summary

Keep open. This is a ClawHub issue, and the repository policy for this review says not to auto-close issues. The technical evidence also does not support a clean implemented-on-main close: a related PR was merged, but users reported the problem again afterward, and current main now intentionally blocks re-authentication for deleted/deactivated accounts rather than restoring login.

## Best Possible Solution

Keep this issue open for maintainer triage. The best path is to decide the intended product behavior for self-deleted accounts: either document/communicate irreversible deletion and provide a support/admin recovery answer for affected legacy accounts, or implement a deliberate reactivation flow that safely matches the GitHub identity and restores login without account-linking risk.
