## Summary

Keep open. The repository policy for openclaw/clawhub says issues should not be auto-closed, and the current source shows this is only partially mitigated: authenticated CLI/API requests use per-user buckets and retry guidance exists, but anonymous VPS/shared-IP requests remain per-IP limited and a narrower related issue (#1410) is still open for anonymous false-positive 429s.

## Best Possible Solution

Keep this issue open for maintainer triage. The concrete path is to either consolidate it manually into the narrower anonymous-install tracking issue (#1410) plus any remaining #390 work, or implement a product decision such as default CLI login, higher anonymous shared-IP headroom, production IP-forwarding fixes, or documented mirror/offline support.
