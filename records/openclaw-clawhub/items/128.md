## Summary

Current main does not implement the requested `clawhub update --dry-run` behavior. The `update` command only exposes `--all`, `--version`, and `--force`, while `cmdUpdate` proceeds to remove/extract/write lockfile state when an update is needed. Existing dry-run support is present for package publish and sync, not for update. Because this is a ClawHub issue, repository policy also says to keep issues open rather than auto-close them.

## Best Possible Solution

Keep this issue open and implement the feature in the CLI core: add `--dry-run` to `packages/clawhub/src/cli.ts`, extend `cmdUpdate` to produce a no-write update plan for single-skill and `--all` flows, print machine-parseable enough status for would-update/up-to-date/skipped cases, and add focused tests proving no filesystem or lockfile mutations occur in dry-run mode. Update `docs/cli.md` and `packages/clawhub/README.md` with the new flag once shipped.
