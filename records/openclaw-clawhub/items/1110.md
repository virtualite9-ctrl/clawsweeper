## Summary

Keep open. Current main does not implement the PR’s intended feature: there is no GET /api/v1/stars HTTP route or handler, no stars-list response schema, and no clawhub list-stars CLI command. The existing web highlights page uses the Convex client query directly, which is not equivalent to the requested public HTTP API and CLI workflow.

## Best Possible Solution

Keep this PR open for maintainer review. If the feature is still desired, land an implementation that registers GET /api/v1/stars in convex/http.ts, adds an authenticated read-rate-limited handler returning { items: publicSkill[] } with resolved tags, uses the existing by_user index with bounded pagination, adds shared schema coverage, and wires a clawhub list-stars CLI command with tests and docs.
