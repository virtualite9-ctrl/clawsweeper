## Summary

Keep open. This is a ClawHub issue, and the repository policy says ClawHub issues should remain open during this cleanup. Current main also still contains the same import-publish error path: GitHub import wraps publish failures, publish runs the GitHub account-age gate, and that gate still emits the reported rate-limit message on GitHub 403/429 responses.

## Best Possible Solution

Keep this issue open for maintainer triage. Verify production Convex logs and environment for the GitHub account-age lookup, especially `GITHUB_TOKEN`; if the token is already configured, improve the publish gate to distinguish true GitHub rate limits from other 403s, include retry/reset details where available, or cache/sync `githubCreatedAt` earlier so import publish is not repeatedly blocked by the same external lookup.
