## Summary

Keep this issue open. The ClawHub cleanup policy only allows auto-closing PRs as implemented, not issues, and the current main code still appears vulnerable to the reported fresh-install/update mismatch: skill downloads include `_meta.json`, local fingerprinting includes JSON files, while publish/resolve fingerprints are based on published version files rather than the generated zip metadata.

## Best Possible Solution

Keep the issue open and fix the core mismatch. The likely maintainer path is to either stop including generated `_meta.json` in skill download archives, or make local fingerprinting ignore generated ClawHub metadata for managed installs, then add a regression test covering install or force-update followed immediately by `clawhub update --all` reporting up to date.
