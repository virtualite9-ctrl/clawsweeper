## Summary

Current main does not implement this PR's intended direct lexical prefix lookup. Skill search still uses vector results, an exact-slug lookup, and a recent digest fallback; the schema and digest helpers do not have normalized slug/display-name fields or prefix indexes. The paired issue #1254 is still open and this PR remains a plausible implementation candidate. Security review of the PR diff found only Convex backend/schema/maintenance/test changes, with no CI, dependency, secret, release, or package-execution changes.

## Best Possible Solution

Keep this PR open for maintainer review, or replace it with an equivalent maintainer patch that adds indexed direct prefix lookup for skill slug/display name, normalized digest fields, a migration-safe backfill, and regression coverage for the Midscene case plus the reported `kling` -> `klingai` prefix case. Because it is paired with open issue #1254 using closing syntax, the issue should close only after the implementation merges.
