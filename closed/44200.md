## Summary

Current `main` already routes parent-owned ACP run completions back through the existing task/announce path without defaulting `streamTo`, and making `streamTo="parent"` the ACP default would now alter persistent ACP session delivery semantics.

## Evidence

- **ACP `streamTo` is still opt-in on main:** `createSessionsSpawnTool()` still sets `streamTo` only when the caller explicitly passes `"parent"`; there is no ACP-wide default on current `main`.
  - file: [src/agents/tools/sessions-spawn-tool.ts:198](https://github.com/openclaw/openclaw/blob/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54/src/agents/tools/sessions-spawn-tool.ts#L198)

- **Omitted `streamTo` ACP runs are tracked for completion:** When an ACP spawn is accepted and `streamTo !== "parent"`, the tool registers the child run with the subagent registry instead of dropping it.
  - file: [src/agents/tools/sessions-spawn-tool.ts:282](https://github.com/openclaw/openclaw/blob/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54/src/agents/tools/sessions-spawn-tool.ts#L282)
