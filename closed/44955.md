## Summary

Keep #44955 open. Current main still does not implement the PR’s requested provider-level `contextWindow` precedence: model resolution still falls back to per-model config or discovered metadata, the inline provider path still bypasses provider-level context metadata, and the config schema/docs only describe `contextWindow` on individual model entries. The PR discussion is substantive and the Greptile review identified a real gap in the PR’s patch, so this should remain a maintainer-reviewed bug-fix path rather than be auto-closed.

## Best Possible Solution

Keep the PR open for maintainer review, but do not merge it as-is. A complete fix should align schema/types/docs with the chosen contract, cover provider-level defaults across registry, fallback, inline `api`, model cache/session context lookup paths, and add regression tests for both Ollama-style discovered models and custom providers with `api`. If maintainers prefer the current per-model-only contract, close the PR after updating #44786 with that decision and pointing users to per-model `contextWindow` or `contextTokens`.
