## Summary

Keep open. Current `main` still forwards Claude CLI `result` strings verbatim in both JSON and JSONL parsing paths, so a nested stringified sub-agent `{"type":"result","result":"..."}` payload would still leak raw JSON instead of the inner text. The linked fix PR (#66819) is still open in the provided timeline.

## Evidence

- **JSON parser does not unwrap nested result payloads:** `collectCliText()` returns string values as-is and `parseCliJson()` assigns the trimmed text directly to the output. There is no recursive parse/unwrap step for nested `type: "result"` payloads.
  - file: [src/agents/cli-output.ts:162](https://github.com/openclaw/openclaw/blob/cc0f3e0e40364023cc3a39227a411898ee68ce3b/src/agents/cli-output.ts#L162)
  - command: `nl -ba src/agents/cli-output.ts | sed -n '120,278p'`
  - sha: [cc0f3e0e4036](https://github.com/openclaw/openclaw/commit/cc0f3e0e40364023cc3a39227a411898ee68ce3b)
- **JSONL Claude result path returns raw `parsed.result`:** `parseClaudeCliJsonlResult()` checks for `type === "result"` and returns `params.parsed.result.trim()` directly. A stringified inner Agent result would therefore be passed through unchanged.
  - file: [src/agents/cli-output.ts:280](https://github.com/openclaw/openclaw/blob/cc0f3e0e40364023cc3a39227a411898ee68ce3b/src/agents/cli-output.ts#L280)
  - command: `nl -ba src/agents/cli-output.ts | sed -n '280,304p'`
  - sha: [cc0f3e0e4036](https://github.com/openclaw/openclaw/commit/cc0f3e0e40364023cc3a39227a411898ee68ce3b)
- **CLI runner forwards parsed text to final assistant output:** `execute.ts` takes `parseCliOutput(...).text` as `rawText` and then uses it as the final response text after text transforms. There is no later nested-Agent-result normalization stage.
  - file: [src/agents/cli-runner/execute.ts:522](https://github.com/openclaw/openclaw/blob/cc0f3e0e40364023cc3a39227a411898ee68ce3b/src/agents/cli-runner/execute.ts#L522)
  - command: `nl -ba src/agents/cli-runner/execute.ts | sed -n '522,538p'`
  - sha: [cc0f3e0e4036](https://github.com/openclaw/openclaw/commit/cc0f3e0e40364023cc3a39227a411898ee68ce3b)
- **Current tests do not cover nested successful Agent result unwrapping:** `cli-output` tests cover normal Claude result parsing and nested API error extraction, but there is no test asserting recursive unwrapping of a successful nested `type: "result"` string payload.
  - file: [src/agents/cli-output.test.ts:163](https://github.com/openclaw/openclaw/blob/cc0f3e0e40364023cc3a39227a411898ee68ce3b/src/agents/cli-output.test.ts#L163)
  - command: `nl -ba src/agents/cli-output.test.ts | sed -n '162,321p'`
  - sha: [cc0f3e0e4036](https://github.com/openclaw/openclaw/commit/cc0f3e0e40364023cc3a39227a411898ee68ce3b)
- **Related fix is not landed yet:** The provided issue timeline cross-references PR #66819 (`fix(cli): unwrap nested claude result json`) and marks it as open, which matches the missing implementation on current `main`.
