## Summary

Keep open. Current `main` still returns Claude CLI `type:"result"` payload text verbatim, with no recursive unwrap for nested Agent-tool result JSON, and that text is forwarded directly into assistant/channel output. The linked PR #66819 is still open, so there is no shipped fix on `main` yet.

## Evidence

- **`parseCliJson()` does not unwrap nested result JSON:** `parseCliJson()` collects `parsed.result`/`parsed.response`/`parsed.message` text and returns it directly as `text`; there is no recursive parse of a stringified inner `{ "type": "result", "result": ... }` payload.
  - file: [src/agents/cli-output.ts:244](https://github.com/openclaw/openclaw/blob/89755d1c79678610720cdb9ce504cfd62c261f64/src/agents/cli-output.ts#L244)
  - command: `nl -ba src/agents/cli-output.ts | sed -n '244,278p'`
  - sha: [89755d1c7967](https://github.com/openclaw/openclaw/commit/89755d1c79678610720cdb9ce504cfd62c261f64)
- **Claude JSONL result path returns outer `.result` string unchanged:** `parseClaudeCliJsonlResult()` matches `parsed.type === "result"` and returns `parsed.result.trim()` directly. That would preserve a nested stringified Agent result JSON instead of extracting the inner visible text.
  - file: [src/agents/cli-output.ts:280](https://github.com/openclaw/openclaw/blob/89755d1c79678610720cdb9ce504cfd62c261f64/src/agents/cli-output.ts#L280)
  - command: `nl -ba src/agents/cli-output.ts | sed -n '280,304p'`
  - sha: [89755d1c7967](https://github.com/openclaw/openclaw/commit/89755d1c79678610720cdb9ce504cfd62c261f64)
- **CLI runner forwards parsed text to visible assistant/channel output:** `executeCliAttempt()` uses `output.text.trim()` as `assistantText`, and `buildCliRunResult()` exposes that text as the final visible payload. `execute.ts` sets `rawText = parsed.text` before output transforms, so the parser result is the channel-facing source text.
  - file: [src/agents/cli-runner.ts:190](https://github.com/openclaw/openclaw/blob/89755d1c79678610720cdb9ce504cfd62c261f64/src/agents/cli-runner.ts#L190)
  - command: `nl -ba src/agents/cli-runner.ts | sed -n '190,240p'`
  - sha: [89755d1c7967](https://github.com/openclaw/openclaw/commit/89755d1c79678610720cdb9ce504cfd62c261f64)
- **Current tests do not cover nested Agent-result unwrapping:** `cli-output.test.ts` covers normal Claude `result` events, cache/session metadata, and mixed JSON objects, but there is no regression test asserting recursive unwrapping of nested stringified Agent result JSON.
  - file: [src/agents/cli-output.test.ts:162](https://github.com/openclaw/openclaw/blob/89755d1c79678610720cdb9ce504cfd62c261f64/src/agents/cli-output.test.ts#L162)
  - command: `nl -ba src/agents/cli-output.test.ts | sed -n '162,321p'`
  - sha: [89755d1c7967](https://github.com/openclaw/openclaw/commit/89755d1c79678610720cdb9ce504cfd62c261f64)
- **Issue discussion already points to an unfinished fix path:** The issue timeline cross-references open PR #66819, `fix(cli): unwrap nested claude result json`, which is consistent with the missing logic still visible on `main`.
