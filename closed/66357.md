## Summary

Current main already satisfies both parts of #66357. The confusing `ollama_web_*` names are not present; agent tools, catalogs, Control UI tool toggles, and docs use `web_search` and `web_fetch`. The linked zh-CN i18n request #60935 was also already addressed: the generated zh-CN locale has full key coverage with no fallback keys, includes the specific Usage, Dreams, theme, chat tool toggle, and Cron labels named in the issue, and is wired through the lazy locale registry. The same evidence is present in release v2026.4.24.

## Best Possible Solution

Close this combined issue as implemented. Keep the shipped provider-neutral `web_search`/`web_fetch` names and the generated Control UI i18n workflow; future untranslated hardcoded UI strings should be handled as focused i18n issues or PRs rather than reopening this older combined request.
