## Summary

Keep #66001 open. Current main still has the deliberate no-op orphan reconciliation contract, so the PR is not obsolete or implemented elsewhere. The linked issue #65983 remains a real open lifecycle gap, but the PR as proposed needs revision because the reviewed patch can misclassify session-key-owned exec runs as orphans.

## Best Possible Solution

Keep the PR open, but require changes before merge: make reconciliation ownership session-key-aware or store enough owner metadata for a precise predicate, restrict reconciliation reasons to forced-cancel reasons, add regression tests that preserve sessionKey-owned active runs, and keep durable startup orphan reaping tracked under #65983 or a narrower follow-up.
