## Summary

Current `main` already makes embedded-agent timeout behavior configurable, but under the shipped knobs `agents.defaults.llm.idleTimeoutSeconds` and `agents.defaults.timeoutSeconds` rather than the proposed `agents.defaults.embeddedPi.timeoutMs`. The embedded runner applies the configurable idle timeout to live model streams, the slug-generator path now inherits agent timeout settings, and the config/docs/tests on `main` and release `v2026.4.23` all reflect that.

## Risks / Open Questions

- Several April comments describe a separate embedded SSE/provider-compatibility bug; this review does not disprove those reports, only that the timeout is now configurable.
- Users who follow the originally suggested `agents.defaults.embeddedPi.timeoutMs` shape will still get schema validation errors because the supported knobs shipped under different config keys.
