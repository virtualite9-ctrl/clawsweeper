## Summary

The reported Slack attachment regression from 2026.4.5 is fixed on current `main`: Slack media downloads now stay on the runtime dispatcher path, the placeholder-only text remains only as a fallback when hydration fails, regression coverage exists for the dispatcher case, and the changelog records this fix in v2026.4.7 under #62239.

## Evidence

- **Current Slack media fetch path uses runtime dispatcher when `dispatcher` is present:** `createSlackMediaFetch()` now switches to `fetchWithRuntimeDispatcher` for dispatcher-backed requests instead of passing the undici 8 dispatcher through `globalThis.fetch`, which matches the root-cause diagnosis in the issue thread.
