## Summary

Current `main` already fixes this: the GitHub Copilot `openai-responses` path now injects the Copilot IDE headers required for newer non-Claude models, while `v2026.4.23` did not.

## Evidence

- **Current main adds Copilot headers on the Responses path:** `wrapCopilotOpenAIResponsesStream()` now passes `headers: buildCopilotRequestHeaders(...)` into the underlying request options, so non-Claude Copilot models get the same IDE headers as the Claude path.
  - file: [extensions/github-copilot/stream.ts:63](https://github.com/openclaw/openclaw/blob/89755d1c79678610720cdb9ce504cfd62c261f64/extensions/github-copilot/stream.ts#L63)
  - command: `nl -ba extensions/github-copilot/stream.ts | sed -n '1,140p'`
  - sha: [89755d1c7967](https://github.com/openclaw/openclaw/commit/89755d1c79678610720cdb9ce504cfd62c261f64)
- **Those headers include `Editor-Version`:** `buildCopilotDynamicHeaders()` composes `buildCopilotIdeHeaders()`, which includes `Editor-Version`, `Editor-Plugin-Version`, and `User-Agent`.
  - file: [src/agents/copilot-dynamic-headers.ts:42](https://github.com/openclaw/openclaw/blob/89755d1c79678610720cdb9ce504cfd62c261f64/src/agents/copilot-dynamic-headers.ts#L42)

- **Latest release lacked the fix:** In `v2026.4.23` (`a979721`), the `openai-responses` wrapper only rewrote payload IDs and did not set request headers at all. The missing `headers:` assignment is the material difference from current `main`.
  - file: [extensions/github-copilot/stream.ts:52](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/extensions/github-copilot/stream.ts#L52)

- **Current tests lock the regression down:** Provider tests now assert that the OpenAI Responses wrapper forwards the dynamic Copilot headers into request options before the request is sent.
  - file: [extensions/github-copilot/stream.test.ts:104](https://github.com/openclaw/openclaw/blob/89755d1c79678610720cdb9ce504cfd62c261f64/extensions/github-copilot/stream.test.ts#L104)
