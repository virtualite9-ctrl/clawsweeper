## Summary

Current `main` already fails closed for the reported process-substitution path, and allowlist matching is resolved-path-only, so the claimed zsh builtin/module bypass is not reproducible from the present code. The PR discussion also identified real correctness issues in the proposed patch: quoted `<(`/`>(` false positives, basename-collision risk for legitimate executables, and a misleading generic failure reason.

## Evidence

- **Process substitution is already rejected by shell analysis:** `DISALLOWED_PIPELINE_TOKENS` already includes `<`, `>`, `(`, and `)`, so unquoted process substitution is fail-closed during pipeline splitting before execution planning. The same parser also rejects `$()` and backticks inside double quotes.

  - file: [src/agents/bash-tools.exec.script-preflight.test.ts:468](https://github.com/openclaw/openclaw/blob/ecac696643c604c6d854341bfe6424bfa31423e9/src/agents/bash-tools.exec.script-preflight.test.ts#L468)

  - file: [docs/tools/exec.md:178](https://github.com/openclaw/openclaw/blob/ecac696643c604c6d854341bfe6424bfa31423e9/docs/tools/exec.md#L178)
