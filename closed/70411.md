## Summary

Keep open. Current `main` still contains the installer logic that can mask a failed spinner-wrapped Homebrew `node@22` install on macOS, then emit the misleading `was installed`/PATH remediation message. `scripts/install.sh` is unchanged between `v2026.4.23` and `HEAD`, so this is not already fixed on main or in the latest release. The issue also already links active fix work in PR #70432.

## Evidence

- **Spinner wrapper can collapse a failed command into success:** `run_with_spinner` stores `gum_status=$?` after the `if "$GUM" spin ...; then return 0; fi` compound command, so a failed condition can be observed as status `0` instead of the wrapped command failure. A minimal bash reproduction of the current helper shape returned `status=0` for a wrapped `false` command.
  - file: [scripts/install.sh:415](https://github.com/openclaw/openclaw/blob/af56926e2fc410809fcb0399435787c7fa8daef8/scripts/install.sh#L415)
  - command: `bash -lc 'set -euo pipefail; is_shell_function(){ return 1; }; gum_is_tty(){ return 0; }; mktempfile(){ mktemp; }; is_gum_raw_mode_failure(){ return 1; }; ui_warn(){ :; }; GUM=echo; run_with_spinner(){ local title="$1"; shift; if [[ -n "$GUM" ]] && gum_is_tty && ! is_shell_function "${1:-}"; then local gum_err; gum_err="$(mktemp)"; if "$GUM" spin --spinner dot --title "$title" -- "$@" 2>"$gum_err"; then return 0; fi; local gum_status=$?; if is_gum_raw_mode_failure "$gum_err"; then GUM=""; "$@"; return $?; fi; return "$gum_status"; fi; "$@"; }; run_quiet_step(){ local title="$1"; shift; if [[ "${VERBOSE:-}" == "1" ]]; then run_with_spinner "$title" "$@"; return $?; fi; }; VERBOSE=1; run_quiet_step test false; printf "status=%s\n" "$?"'`
  - sha: [af56926e2fc4](https://github.com/openclaw/openclaw/commit/af56926e2fc410809fcb0399435787c7fa8daef8)
- **macOS installer still proceeds past the Homebrew install step:** `install_node()` still calls `run_quiet_step "Installing node@${NODE_DEFAULT_MAJOR}" brew install "node@${NODE_DEFAULT_MAJOR}"` as a bare statement, then immediately continues to `brew link` and `ensure_macos_default_node_active`. With the spinner helper misreporting failure as success, the control flow described in the issue remains present on current `main`.
  - file: [scripts/install.sh:1398](https://github.com/openclaw/openclaw/blob/af56926e2fc410809fcb0399435787c7fa8daef8/scripts/install.sh#L1398)
  - sha: [af56926e2fc4](https://github.com/openclaw/openclaw/commit/af56926e2fc410809fcb0399435787c7fa8daef8)
- **Misleading PATH remediation is still possible without an installed binary:** `ensure_macos_default_node_active()` only prepends Homebrew Node to `PATH` when `${brew_node_prefix}/bin/node` exists, but its later error branch only checks whether `brew_node_prefix` is non-empty before printing `export PATH="${brew_node_prefix}/bin:$PATH"`. That means the function can still emit the `was installed` message and Homebrew PATH advice even when the binary is not on disk.
