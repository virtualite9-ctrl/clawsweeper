## Summary

Current `main` no longer relies on a stale `gateway.pid` model for macOS LaunchAgent status/stop behavior. The LaunchAgent runtime PID is read from `launchctl print`, `gateway stop` explicitly disables the LaunchAgent before stopping it so `KeepAlive` does not immediately respawn behind the CLI, and status/restart health also cross-check live listener/probe state. The active changelog for `v2026.4.23` documents the persistent-stop launchd fix, and launchd integration tests cover PID replacement across restart/start flows.
