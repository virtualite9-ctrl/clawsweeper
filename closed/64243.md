## Summary

Current main, and the shipped v2026.4.24 tag, already implement the requested behavior for the observable bug: when the effective gateway exec policy is security=full and ask=off, the gateway exec path does not register an exec approval request or wait on exec.approval.waitDecision. The Mattermost-specific symptom is therefore covered by the central exec policy fix rather than needing a Mattermost special case.

## Best Possible Solution

Close this issue as implemented. Keep the shipped central exec approval behavior, without adding a Mattermost-specific bypass. If this reappears on a current release, the best follow-up is a new issue with the effective gateway exec policy, host exec-approvals snapshot, Mattermost account/channel details, and logs from the specific deploy probe run.
