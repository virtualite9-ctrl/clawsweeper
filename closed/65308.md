## Summary

Close as implemented. Current main no longer leaves default ACP one-shot completions at only a generic background-task status: ACP run-mode spawns are registered for completion tracking, the child final reply is frozen before cleanup, and the parent announce path relays that result back in normal assistant voice. Final ACP snapshot handling also prevents progress text from winning over the terminal answer.

## Best Possible Solution

Close #65308 and keep the shipped completion-announce path. Track the narrower explicit `streamTo="parent"` direct-chat relay behavior separately in #46834 if maintainers still want that path changed.
