## Summary

This PR is superseded by current main. The diagnostic half is already implemented with received-parameter hints and tests, while the read-alias half no longer matches the current runtime contract because OpenClaw-owned read/write/edit tools enforce canonical `path` and tests reject legacy aliases. The remaining `filePath`/`file` compatibility question is better handled in the open alias-tracking issues rather than merging this stale PR as-is.

## Best Possible Solution

Close this PR as superseded. Keep the shipped received-parameter diagnostics on main, and route any future `filePath`/`file` read-alias compatibility change through the canonical open issues (#60550/#56694) with an explicit runtime-contract decision and regression tests.
