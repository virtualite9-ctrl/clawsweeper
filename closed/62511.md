## Summary

Current main already carries released external-plugin loading fixes that match this failure mode: installed plugins with an `openclaw` peer dependency now get a host-package symlink, plugin SDK aliasing covers both scoped and unscoped package names, and `plugin-sdk/account-id` remains a shipped public subpath. Those fixes are present in the latest release, `v2026.4.23`, and align with plugins named in the report.

## Evidence

- **v2026.4.23 release note:** The 2026.4.23 changelog explicitly notes: external plugin installs now link the host OpenClaw package for plugins that declare `openclaw` as a peer dependency, so peer-only plugin SDK imports resolve correctly.
