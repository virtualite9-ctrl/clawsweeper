## Summary

Current `main` already fixes the anthropic-vertex startup registration failure. The startup path still builds implicit provider catalogs for `models.json`, and the material fix versus `v2026.4.22` is that anthropic-vertex auth detection now resolves the default ADC path from copied env snapshots instead of rejecting non-`process.env` env objects.

## Evidence

- **Startup still regenerates models.json from an env snapshot:** `ensureOpenClawModelsJson()` creates a copied runtime env with `createConfigRuntimeEnv(cfg)` and passes it into `planOpenClawModelsJson(...)`, so provider discovery still runs during startup.
  - file: [src/agents/models-config.ts:163](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/agents/models-config.ts#L163)
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Implicit provider discovery is part of models.json generation:** `resolveImplicitProviders()` loads discovery providers and runs their catalogs across discovery orders before writing `models.json`. This is the path that registers built-in providers implicitly.
  - file: [src/agents/models-config.providers.implicit.ts:358](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/agents/models-config.providers.implicit.ts#L358)
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Anthropic Vertex discovery still builds the provider catalog:** The lightweight discovery entry calls `hasAnthropicVertexAvailableAuth(env)` and, when auth is present, returns `buildAnthropicVertexProvider({ env })` merged with explicit overrides.
  - file: [extensions/anthropic-vertex/provider-discovery.ts:46](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/extensions/anthropic-vertex/provider-discovery.ts#L46)
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Main removed the env-snapshot guard that blocked default ADC lookup:** Compared with `v2026.4.22`, current `main` no longer returns early for `env !== process.env`; it now derives the default ADC path from `HOME`/`USERPROFILE` in the provided env snapshot. That matches the issue's default-ADC startup failure mode.
  - file: [extensions/anthropic-vertex/region.ts:68](https://github.com/openclaw/openclaw/blob/0a3da5cd8a6e712de6418008e01b11fb2afd7aea/extensions/anthropic-vertex/region.ts#L68)
  - command: `git diff --unified=20 00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5..8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf -- extensions/anthropic-vertex/region.ts`
  - sha: [0a3da5cd8a6e](https://github.com/openclaw/openclaw/commit/0a3da5cd8a6e712de6418008e01b11fb2afd7aea)
- **Regression coverage was added for copied env snapshots:** There is now a test that passes a copied env with `HOME=/tmp/vertex-home` and asserts `hasAnthropicVertexAvailableAuth(env)` reads `~/.config/gcloud/application_default_credentials.json` successfully.
  - file: [extensions/anthropic-vertex/region.adc.test.ts:50](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/extensions/anthropic-vertex/region.adc.test.ts#L50)
