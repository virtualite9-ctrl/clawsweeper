## Summary

Current main already supersedes this PR: routed `status` and `health` no longer preload plugins at all, and status channel reporting now runs through read-only config-scoped helpers instead of the runtime preload path this PR changes.

## Evidence

- **Status and health no longer request routed plugin preload:** The routed command catalog sets `status` and `health` to `loadPlugins: "never"`, so the normal CLI bootstrap path no longer preloads channel plugins for either command.
  - file: [src/cli/command-catalog.ts:42](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/cli/command-catalog.ts#L42)

- **Route behavior is locked in by tests:** Route tests assert that both `findRoutedCommand(["status"])` and `findRoutedCommand(["health"])` have `loadPlugins === undefined`, confirming there is no routed preload hook for these commands.
  - file: [src/cli/program/routes.test.ts:59](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/cli/program/routes.test.ts#L59)
