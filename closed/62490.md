## Summary

Close as `clawhub`: the issue proposes a default core manager/subagent harness architecture, but current `main` explicitly keeps optional orchestration in plugins, rejects default agent-hierarchy frameworks/heavy orchestration layers in `VISION.md`, and already exposes plugin seams for harnesses, context engines, subagents, and skill gating that fit this work better as a ClawHub plugin/extension.
