## Summary

Current main implements the requested Ollama context-window behavior for newly discovered/cloud-onboarded models. The Ollama provider reads `/api/show`, accepts provider-specific `*.context_length` keys such as `nemotron_h_moe.context_length`, carries the result into `contextWindow`, and `models list --all --provider ollama` now has a registry-backed fallback instead of relying only on display/static catalogs. Live Ollama API output for `nemotron-3-super` currently reports `262144`, matching the value current main would consume.

## Best Possible Solution

Keep the current main implementation: Ollama cloud/local discovery should continue using `/api/tags` plus `/api/show` metadata, and `models list` should keep the registry-backed fallback for provider-filtered listings. For users with an old explicit config that already persisted `128000`, the practical path is to rerun Ollama setup or update `models.providers.ollama.models[].contextWindow`; no additional core change is needed for current setup/discovery behavior.
