## Summary

Current main already routes `openai-codex` through the Codex responses transport, normalizes stale `openai-responses`/`openai-completions` plus `api.openai.com` metadata onto the Codex route, and has regression coverage for those cases. The same provider files are unchanged between `v2026.4.23` and current `main`, so the fix is already shipped.

## Evidence

- **OpenAI Codex provider catalog uses Codex transport by default:** The bundled `openai-codex` provider config sets `baseUrl` to `https://chatgpt.com/backend-api/codex` and `api` to `openai-codex-responses`, matching the issue's requested routing.
