## Summary

The specific createPinnedLookup/Undici callback incompatibility does not reproduce on current main. Current code returns an array for options.all lookups and scalar address/family for non-all lookups, which matches the Node net lookup contract used by Undici. The later Node built-in fetch dispatcher incompatibility raised in comments is also covered by current main’s runtime Undici fetch path.

## Evidence
