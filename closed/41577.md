## Summary

Keep open. Current main clearly changed part of the surface: the `coding` profile now includes `web_search`/`web_fetch`, but it still excludes `browser`. The policy pipeline still filters by profile before later allowlists, and current tests still treat `browser` as unavailable under `tools.profile = "coding"` unless it is added through the profile-stage additive path. That is not strong evidence that the reported spawned-subagent browser gap is fixed; at best it shows partial semantics/docs clarification for web tools, while the browser portion of the report still looks plausibly live or at least still confusing on current main.
