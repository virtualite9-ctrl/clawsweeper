## Summary

Close as implemented on main: current OpenClaw already contains shipped fixes for both reported symptom classes in this issue's repro area, namely raw `terminated` transport failures and OpenAI-compatible streaming errors after partial output. The follow-up LM Studio comment points at the same local/OpenAI-compatible streaming path that was hardened in `v2026.4.23`.

## Evidence

- **Issue discussion narrows scope:** A follow-up comment reports the same failure pattern with LM Studio during prompt processing and streaming, which points at the local OpenAI-compatible transport/parser path rather than a Control UI-only rendering bug.
- **Raw `terminated` errors are now classified for failover:** Current main explicitly matches bare `terminated` transport failures and related undici socket/body/header aborts, so they enter timeout/fallback handling instead of surfacing as an unclassified raw error.
