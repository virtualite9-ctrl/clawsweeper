## Summary

Current main already qualifies dashboard/chat model-picker values with the selected model's own provider and forwards that exact `provider/model` value to `sessions.patch`; the same implementation is present in shipped release `v2026.4.22`.

## Evidence

- **Picker option values come from each catalog entry's provider:** `buildChatModelOptionFromLookup()` returns `value: buildQualifiedChatModelValue(entry.id, provider)`, so option values are constructed from the selected catalog entry, not the current session provider.

  - file: [ui/src/ui/chat-model-select-state.ts:58](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/ui/src/ui/chat-model-select-state.ts#L58)
