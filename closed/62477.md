## Summary

Current `main` already contains the late-stdout/post-run exec update suppression that matches this report, includes regression tests for the race, and the fix is listed in the shipped `v2026.4.23` changelog.

## Evidence

- **Exec runtime now drops late updates after run end:** `emitUpdate()` returns early once the session is backgrounded, exited, or `updatesDisabled` is set, with an inline comment calling out the exact `"Agent listener invoked outside active run"` crash path.
  - file: [src/agents/bash-tools.exec-runtime.ts:561](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/agents/bash-tools.exec-runtime.ts#L561)

  - sha: [a9797214338b](https://github.com/openclaw/openclaw/commit/a9797214338ba31c52c796adbb75afb16e0684a9)
- **Updates are proactively disabled on exit:** The exec wait path sets `updatesDisabled = true` before `markExited(...)` so same-tick late stdout/stderr cannot slip through after the run is over.
  - file: [src/agents/bash-tools.exec-runtime.ts:773](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/agents/bash-tools.exec-runtime.ts#L773)
