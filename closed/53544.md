## Summary

Current main already has the channel-status plumbing that includes configured WhatsApp in both `/status` and the Control UI, and the same implementation shipped in `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`).

## Evidence

- **`/status` channels table now uses read-only configured channel discovery:** `buildChannelsTable` iterates `listReadOnlyChannelPluginsForConfig(...)`, so the status table is built from configured channel metadata instead of only whatever happened to be eagerly loaded.
  - file: [src/commands/status-all/channels.ts:209](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/commands/status-all/channels.ts#L209)
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Configured bundled channels are pulled in even if absent from the active registry:** `resolveReadOnlyChannelPluginsForConfig` collects configured channel ids, then explicitly adds bundled setup plugins for any configured id not already present. That is the missing piece for bundled channels like WhatsApp.
  - file: [src/channels/plugins/read-only.ts:355](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/channels/plugins/read-only.ts#L355)
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Gateway `channels.status` publishes channel metadata and per-channel summaries:** The gateway status handler builds `channelMeta`, `channels`, `channelAccounts`, and default-account ids from the channel plugin list, which is the payload the Control UI consumes.
  - file: [src/gateway/server-methods/channels.ts:289](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/gateway/server-methods/channels.ts#L289)
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Control UI explicitly renders WhatsApp from the snapshot:** The Channels view reads `snapshot.channels.whatsapp`, includes a WhatsApp card renderer, and falls back to an order that includes `whatsapp`.
  - file: [ui/src/ui/views/channels.ts:35](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/ui/src/ui/views/channels.ts#L35)
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Regression coverage asserts WhatsApp appears in channel status payloads:** The gateway status test expects a `channels.whatsapp` entry with `configured: true`, covering the status path this issue reported as missing.
  - file: [src/gateway/server-methods/channels.status.test.ts:124](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/gateway/server-methods/channels.status.test.ts#L124)
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
