## Summary

Current main implements the enforceable Active Memory timeout requested in #72347: the hook races the embedded memory sub-agent against an Active Memory-owned abort sentinel and returns status=timeout near config.timeoutMs even if the sub-agent does not cooperate. The remaining single-shot/no-agent-loop latency architecture is separately tracked by #72359.

## Best Possible Solution

Close #72347 against current main for the enforceable timeout bug, keep the shipped hard-timeout wrapper, and leave #72359 as the canonical follow-up for optional single-shot Active Memory mode.
