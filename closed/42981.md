## Summary

Current main implements the requested hard tool restriction for both surfaces, though with OpenClaw's shipped field names rather than the proposed `toolPolicy` shape. Cron agent-turn jobs have an enforced per-job `payload.toolsAllow` allow-list exposed through CLI/tool/protocol paths, and spawned subagent sessions have enforced allow/deny policy through `tools.subagents.tools` plus per-agent tool policy. The Apr 1 issue comment was accurate that the then-linked PRs were only partial; current main now has the missing cron allow-list and the subagent policy path is already shipped and documented.

## Best Possible Solution

Close this broad issue and keep the shipped implementation: cron jobs should use `payload.toolsAllow` or `openclaw cron --tools`, while spawned sessions should be constrained with `tools.subagents.tools` and restricted target agent profiles. Broader operator-scope RBAC should continue under the still-open #49183 RFC rather than keeping this older cron/spawn request open.
