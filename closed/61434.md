## Summary

The reported reproduction starts signal-cli with the raw JSON-RPC TCP transport, while current OpenClaw's Signal `httpUrl` integration talks to signal-cli's HTTP JSON-RPC/SSE endpoints. The observed `fetch failed` plus signal-cli `Connection reset by peer` loop is consistent with sending OpenClaw's HTTP SSE request to a TCP JSON-RPC socket, not with a remaining core SSE-client bug on current main.
