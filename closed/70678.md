## Summary

Current main does not yet fix the reported WhatsApp quiet-session reconnects. The gateway stale-socket policy has moved toward transport-backed activity, but the WhatsApp extension still has its own 30-minute inbound-message watchdog that closes the socket with status 499. Maintainer discussion already confirmed the issue remains valid, and an open fix PR is referenced, so the parent issue should stay open until that fix or an equivalent transport-backed liveness change lands.

## Best Possible Solution

Keep #70678 open as the dedupe parent. Review and land PR #72145 or an equivalent WhatsApp change that makes liveness transport-backed, disables the inbound-message watchdog for quiet linked-device sessions, or exposes a clear supported opt-out without weakening real dead-socket detection. After that lands on main with regression coverage for quiet healthy sessions, close this issue as implemented with the fixing SHA/release.
