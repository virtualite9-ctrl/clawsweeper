## Summary

Keep #66485 open. Current main already implements the context-percentage alignment for `/usage`, including the review-requested `totalTokensFresh !== false` behavior, but the PR's second user-visible goal, refreshing active chat history after auto-compaction so stale transcript/token state clears, is not fully implemented on current main. A maintainer PR (#71297) also explicitly treated #66485 as related but intentionally outside its scope.

## Best Possible Solution

Keep the PR open or ask the contributor/maintainer to rework it against current main: keep the shipped `/usage` totalTokens behavior, then implement a scoped post-compaction refresh path that only targets the active session, preferably by making backend compaction transcript/session updates emit a session-scoped event or by handling session-keyed compaction-end agent events in the Control UI, with tests covering active-session reload, unrelated-session no-op, and stale token counter clearing after compaction.
