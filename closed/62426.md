## Summary

Current `main` already implements the requested Telegram fake-IP/private-network media-download opt-in, covers it with tests, and ships user-facing documentation; the behavior is present in release `v2026.4.23`.

## Evidence

- **Media download SSRF policy:** Telegram media downloads build an SSRF policy that preserves the Telegram hostname allowlist and adds `allowPrivateNetwork: true` when `dangerouslyAllowPrivateNetwork` is enabled.
  - file: [extensions/telegram/src/bot/delivery.resolve-media.ts:24](https://github.com/openclaw/openclaw/blob/ecac696643c604c6d854341bfe6424bfa31423e9/extensions/telegram/src/bot/delivery.resolve-media.ts#L24)
  - sha: [ecac696643c6](https://github.com/openclaw/openclaw/commit/ecac696643c604c6d854341bfe6424bfa31423e9)
- **Config is threaded into media downloads:** Telegram media runtime options read `channels.telegram[.accounts.<id>].network.dangerouslyAllowPrivateNetwork`, and handlers pass those media runtime options into `resolveMedia(...)` for inbound file downloads.
  - file: [extensions/telegram/src/accounts.ts:122](https://github.com/openclaw/openclaw/blob/ecac696643c604c6d854341bfe6424bfa31423e9/extensions/telegram/src/accounts.ts#L122)
  - sha: [ecac696643c6](https://github.com/openclaw/openclaw/commit/ecac696643c604c6d854341bfe6424bfa31423e9)
- **Behavioral regression test:** Test coverage asserts that private-network Telegram media downloads are allowed only when `dangerouslyAllowPrivateNetwork: true` is explicitly configured.
  - file: [extensions/telegram/src/bot/delivery.resolve-media-retry.test.ts:707](https://github.com/openclaw/openclaw/blob/ecac696643c604c6d854341bfe6424bfa31423e9/extensions/telegram/src/bot/delivery.resolve-media-retry.test.ts#L707)
  - sha: [ecac696643c6](https://github.com/openclaw/openclaw/commit/ecac696643c604c6d854341bfe6424bfa31423e9)
- **Schema text matches reported scenario:** The Telegram config schema explicitly documents this setting for trusted fake-IP / transparent-proxy environments where `api.telegram.org` resolves to private or special-use addresses during media downloads.
  - file: [src/config/zod-schema.providers-core.ts:246](https://github.com/openclaw/openclaw/blob/ecac696643c604c6d854341bfe6424bfa31423e9/src/config/zod-schema.providers-core.ts#L246)
  - sha: [ecac696643c6](https://github.com/openclaw/openclaw/commit/ecac696643c604c6d854341bfe6424bfa31423e9)
- **Docs shipped in release:** The Telegram docs document the exact fake-IP proxy case, including Clash/Mihomo/Surge and `channels.telegram.network.dangerouslyAllowPrivateNetwork`; the same content is present in release `v2026.4.23`.
  - file: [docs/channels/telegram.md:843](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/docs/channels/telegram.md#L843)
