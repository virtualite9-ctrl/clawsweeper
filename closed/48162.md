## Summary

Current `main` already fixes the compact status cache-hit denominator and covers the regression with tests. It also addresses the PR review concern by persisting last-call usage separately, so the status formatter no longer has to mix accumulated run totals with per-call cache snapshot fields.

## Evidence

- **Current formatter uses prompt-side cache math:** `formatTokensCompact` delegates to `resolvePromptCacheStats`, which uses `inputTokens + cacheRead + cacheWrite` when available and otherwise clamps the fallback denominator to at least the known cached prompt tokens, preventing impossible percentages above 100%.
  - file: [src/commands/status.format.ts:17](https://github.com/openclaw/openclaw/blob/151befb90b903f52dce9f4951991d39b9df8b358/src/commands/status.format.ts#L17)
  - command: `nl -ba src/commands/status.format.ts | sed -n '1,120p'`
  - sha: [151befb90b90](https://github.com/openclaw/openclaw/commit/151befb90b903f52dce9f4951991d39b9df8b358)
- **Unit regression coverage exists on main:** The dedicated formatter test asserts the compact suffix is `67% cached` for a cache-heavy session (`500 + 2000 + 500` prompt-side tokens), matching the intended denominator from this PR.
  - file: [src/commands/status.format.test.ts:27](https://github.com/openclaw/openclaw/blob/151befb90b903f52dce9f4951991d39b9df8b358/src/commands/status.format.test.ts#L27)
  - command: `nl -ba src/commands/status.format.test.ts | sed -n '1,80p'`
  - sha: [151befb90b90](https://github.com/openclaw/openclaw/commit/151befb90b903f52dce9f4951991d39b9df8b358)

  - file: [src/commands/status.test.ts:1129](https://github.com/openclaw/openclaw/blob/151befb90b903f52dce9f4951991d39b9df8b358/src/commands/status.test.ts#L1129)

- **Main addresses the PR review scope-mismatch concern:** `persistSessionUsageUpdate` now documents and uses `lastCallUsage` for `totalTokens`, and uses the latest context snapshot for `cacheRead`/`cacheWrite`, instead of mixing accumulated run usage with final-call cache stats.
  - file: [src/auto-reply/reply/session-usage.ts:71](https://github.com/openclaw/openclaw/blob/151befb90b903f52dce9f4951991d39b9df8b358/src/auto-reply/reply/session-usage.ts#L71)

- **Type contract documents per-call snapshot semantics:** `EmbeddedPiAgentMeta.lastCallUsage` is documented as non-accumulated per-call usage used for context-window display, matching the fix direction and supporting the current implementation.
  - file: [src/agents/pi-embedded-runner/types.ts:21](https://github.com/openclaw/openclaw/blob/151befb90b903f52dce9f4951991d39b9df8b358/src/agents/pi-embedded-runner/types.ts#L21)

Current `main` resolves that by persisting `lastCallUsage` separately before status formatting.
