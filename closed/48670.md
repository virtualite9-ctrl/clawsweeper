## Summary

Current `main` no longer matches the PR's premise. The validation contract for coding tools is now canonical-only (`read`: `path`; `write`: `path` + `content`; `edit`: `path` + `edits[]`), and tests explicitly verify that legacy alias parameters such as `file_path`, `filePath`, `old_text`, and `newString` are rejected. Current errors also already include a concrete `(received: ...)` hint for malformed payloads, so the proposed alias-oriented hint text would be stale and misleading on `main`.
