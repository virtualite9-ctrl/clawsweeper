## Summary

Close #58356 as a duplicate of the existing macOS companion-app `system.run.prepare` tracker, #37591. The report remains valid on current main for `openclaw-macos/node`, but the same root mismatch is already tracked: `exec host=node` requires `system.run.prepare`, while the macOS Swift node advertises and handles `system.run` without `system.run.prepare`. The reporter and another commenter also confirmed the headless `node-host/node` workaround restores exec, which matches the current code/docs and does not require a separate issue.

## Best Possible Solution

Close #58356 as duplicate and keep the remaining product/code decision centralized in #37591, with #38781 and #69725 as the relevant implementation directions: either add secure `system.run.prepare` support to the macOS Swift node or intentionally add an agent-side fallback for nodes that omit prepare. Preserve the headless node-host workaround as documentation/support context, not as proof that the macOS app node is fixed.
