## Summary

Current main still omits `threadId` from the direct `/hooks/agent` payload path, but the exact remaining work is already tracked by the linked open PR #69859, which explicitly resolves #69856 and proposes carrying `threadId` through the hook parser, dispatch payload, cron payload, and delivery plan.

## Best Possible Solution

Close #69856 as superseded by the already-linked implementation PR #69859, and keep review/landing work focused there or on a maintainer replacement PR that adds `threadId` to the direct hook payload, dispatch/replay scope, cron delivery payload, docs, and regression tests.
