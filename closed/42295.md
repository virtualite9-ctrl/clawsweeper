## Summary

Current `main` already normalizes missing custom-provider `input` before `models list` row rendering. This PR patches an older pre-refactor helper path, so the user-facing behavior it targets is already covered on `main` through the newer row-builder and model-resolution flow.

## Evidence

- **Configured provider rows default missing input to text:** `resolveConfiguredModelInput()` filters `model.input` and falls back to `["text"]`, and `toConfiguredProviderListModel()` uses that normalized value before any row rendering.
  - file: [src/commands/models/list.rows.ts:110](https://github.com/openclaw/openclaw/blob/459d277076488d5833a181cca7d8bf5c0f9696dc/src/commands/models/list.rows.ts#L110)

  - file: [src/commands/models/list.rows.ts:144](https://github.com/openclaw/openclaw/blob/459d277076488d5833a181cca7d8bf5c0f9696dc/src/commands/models/list.rows.ts#L144)
