## Summary

Current main already supports multiple Telegram bot accounts, each routable to a different agent, via `channels.telegram.accounts` plus `bindings[].match.accountId`; this also appears in the released `v2026.4.23` code/docs.

## Evidence

- **Telegram config supports multiple named accounts:** `TelegramConfig` includes `accounts` and `defaultAccount`, so multiple Telegram bot tokens are already part of the public config surface.
  - file: [src/config/types.telegram.ts:309](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/src/config/types.telegram.ts#L309)
  - sha: [90877e0d4274](https://github.com/openclaw/openclaw/commit/90877e0d4274c53fd91bf5787265c4d6889dc9a7)
- **Runtime resolves and lists per-account Telegram bots:** Telegram account helpers enumerate account IDs, resolve account-specific tokens/config, and expose enabled accounts for runtime startup.
  - file: [extensions/telegram/src/accounts.ts:69](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/extensions/telegram/src/accounts.ts#L69)
  - sha: [90877e0d4274](https://github.com/openclaw/openclaw/commit/90877e0d4274c53fd91bf5787265c4d6889dc9a7)
- **Gateway starts Telegram per accountId:** The Telegram channel gateway starts each configured account separately and passes the resolved `accountId` through startup/monitoring.
  - file: [extensions/telegram/src/channel.ts:866](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/extensions/telegram/src/channel.ts#L866)
  - sha: [90877e0d4274](https://github.com/openclaw/openclaw/commit/90877e0d4274c53fd91bf5787265c4d6889dc9a7)
- **Routing test proves agent binding by Telegram accountId:** The Telegram bot test verifies a DM received on account `opie` routes using `match: { channel: "telegram", accountId: "opie" }` and emits `AccountId: "opie"` with the bound agent session key.
  - file: [extensions/telegram/src/bot.create-telegram-bot.test.ts:1506](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/extensions/telegram/src/bot.create-telegram-bot.test.ts#L1506)
  - sha: [90877e0d4274](https://github.com/openclaw/openclaw/commit/90877e0d4274c53fd91bf5787265c4d6889dc9a7)
- **User docs already show one bot per agent:** The multi-agent docs show two Telegram bot accounts under `channels.telegram.accounts` bound to separate agents via `bindings[].match.accountId`, which is the capability requested in the issue.
  - file: [docs/concepts/multi-agent.md:335](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/docs/concepts/multi-agent.md#L335)
  - sha: [90877e0d4274](https://github.com/openclaw/openclaw/commit/90877e0d4274c53fd91bf5787265c4d6889dc9a7)
- **Released build already contains the same model:** Checked release commit `a9797214338ba31c52c796adbb75afb16e0684a9` (`v2026.4.23`) with `git show`; it already contains the Telegram multi-agent example in `docs/concepts/multi-agent.md` and the same `TelegramConfig.accounts/defaultAccount` shape in `src/config/types.telegram.ts`.
  - command: `git show a9797214338ba31c52c796adbb75afb16e0684a9:docs/concepts/multi-agent.md | sed -n '330,365p' && git show a9797214338ba31c52c796adbb75afb16e0684a9:src/config/types.telegram.ts | sed -n '309,315p'`
  - sha: [a9797214338b](https://github.com/openclaw/openclaw/commit/a9797214338ba31c52c796adbb75afb16e0684a9)

## Risks / Open Questions

- The exact requested `bots[]` config shape and dedicated `list-bots` / `add-bot` / `remove-bot` command names do not exist verbatim; this closure relies on equivalent existing behavior through the established multi-account model.
- There may still be a docs/discoverability gap, but that is separate from the feature request as filed.
