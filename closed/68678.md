## Summary

Issue #68678 requests an opt-in native LM Studio `/api/v1/chat` transport for performance. Current main does not already implement that inference path: bundled LM Studio discovery/loading uses native `/api/v1/models`, but the model provider catalog still resolves inference to `api: "openai-completions"` at `/v1`. The requested capability is provider-specific and can be built with the existing Provider Plugin SDK (`registerProvider`, catalog/dynamic model hooks, and `createStreamFn` for a custom transport), so the best conservative cleanup is to close the core tracker as ClawHub/plugin scope unless a narrower missing SDK hook is later identified.

## Best Possible Solution

Close the core issue as ClawHub/provider-plugin scope. A native LM Studio `/api/v1/chat` implementation should live as an external provider plugin using the existing Provider Plugin SDK and `createStreamFn`; if that work reveals a concrete missing SDK hook, open a narrower API-gap issue.
