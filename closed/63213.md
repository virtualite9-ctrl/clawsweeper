## Summary

Current main already suppresses commentary-phase pre-tool assistant text from user-visible delivery, prefers only `final_answer` text for visible extraction, and routes Telegram delivery through that filtered path. The same behavior is present in the shipped `v2026.4.23` release commit, so this core bug appears fixed.

## Evidence

- **Embedded subscribe suppresses commentary delivery:** User-visible assistant streaming returns early for `deliveryPhase === "commentary"`, so pre-tool chatter is not emitted to partial/block reply delivery.

## Risks / Open Questions

- This fix depends on the provider transport continuing to mark pre-tool chatter as `commentary`; an unphased provider leak would follow legacy visible-text paths instead.
- This review was read-only and local-only; I did not replay the original Hermes Telegram transcripts from Netcup-Clawd.
