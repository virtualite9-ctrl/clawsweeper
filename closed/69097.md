## Summary

Current main now handles the exact empty-turn shape reported here: `stopReason="stop"`, empty assistant content, and zero usage. The runner detects that as an empty provider output, retries once with a visible-answer continuation, and then surfaces an explicit error instead of silently completing if recovery still fails. The separate OpenRouter alias-redirect report #69507 remains open for that narrower behavior.

## Best Possible Solution

Close #69097 as implemented on current main because the reported zero-payload, zero-usage `stop` turn is no longer accepted as a successful silent reply. Keep #69507 open for the distinct OpenRouter alias redirect behavior.
