## Summary

Close as `clawhub`: Langfuse tracing is an optional backend-specific observability integration, and current `main` routes that class of capability through plugins rather than new core code. The repo already ships a plugin-based OpenTelemetry path (`diagnostics-otel`) plus plugin hooks for model/tool/subagent/run observation, so the maintainer-cleanup answer is to move this to an external plugin or a small plugin-API proposal instead of merging a core Langfuse implementation.
