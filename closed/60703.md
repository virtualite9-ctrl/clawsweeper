## Summary

This PR is a docs-only discoverability/promotion change for third-party managed relay services and an external plugin. Current main scope puts that material in ClawHub/community-plugin surfaces, not in the core legacy iMessage docs.

## Risks / Open Questions

- Users may still want curated guidance for non-Mac iMessage relay options; closing this PR does not solve that discoverability gap unless the plugin/vendor is surfaced through ClawHub or the community-plugin process.
- OpenClaw already has some external-plugin docs pages, so maintainers should be consistent that this close is about vendor/service discoverability and commercial relay promotion in a core channel doc, not a blanket ban on all external-plugin documentation.
