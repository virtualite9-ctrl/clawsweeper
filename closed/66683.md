## Summary

Current main now separates parent-owned subagent completion handoff from durable completion cleanup. Completion-mode delivery goes through a direct `agent` call with `expectFinal`, the agent command path awaits outbound delivery before returning, and subagent cleanup only stamps `completionAnnouncedAt`, task delivery `delivered`, and cleanup bookkeeping after the announce flow reports `delivered: true`. Failed direct delivery falls back or leaves cleanup retryable until retry/expiry give-up, and tests cover deferred hooks, fallback, and retry behavior.

## Best Possible Solution

Keep the current main lifecycle: parent-owned subagent completions should remain pending/retryable until the announce delivery path reports success, then mark delivery/cleanup exactly once. No core follow-up is needed for the issue as filed; any future restart-durability or retry-window changes should be tracked as a narrower issue.
