## Summary

Current `main` already supports real-time run interruption via `/stop`, standalone abort phrases, session-scoped `chat.abort`, and legacy `interrupt` queue mode; the stop path clears queued followups and persists cutoff metadata so pre-stop backlog is not replayed.

## Evidence

- **Built-in stop command:** The user-facing slash command docs state that `/stop` aborts the current run.
  - file: [docs/tools/slash-commands.md:94](https://github.com/openclaw/openclaw/blob/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4/docs/tools/slash-commands.md#L94)
