## Summary

PR #47362 is superseded by current main's Slack-owned prompt and message-tool architecture. The branch maps Slack `interactiveReplies` into the old generic `inlineButtons` prompt path and asserts obsolete `buttons=[[...]]` guidance, while current main injects Slack-specific `[[slack_buttons]]` / `[[slack_select]]` hints through the channel plugin and exposes rich controls through the shared `presentation` contract. Any remaining cleanup of the generic negative prompt line should stay with the canonical linked issue #46647 rather than this stale PR patch.

## Best Possible Solution

Close PR #47362 as superseded. Keep the Slack-owned `messageToolHints` plus shared `presentation` message-tool contract as the current direction, and handle any remaining generic prompt wording cleanup in #46647 with a current-main patch that removes or specializes the stale `inlineButtons` hint rather than reintroducing old `buttons=[[...]]` Slack guidance.
