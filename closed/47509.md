## Summary

Close #47509 as superseded by the already-open, narrower Feishu quoted-media PR #42271. Current main still lacks quoted/replied Feishu media download, so this is not implemented-on-main; #47509 overlaps that work but carries broad deletions of current Feishu routing, thread-context, outbound-identity, and structured-card behavior.

## Best Possible Solution

Close #47509 as the duplicate/riskier implementation and keep the remaining work on #42271 or an equivalent focused PR. The correct fix should add raw quoted-message content to the fetched message contract, resolve quoted/replied media through the existing Feishu media resolver, build the final agent media payload after all media is collected, preserve current Feishu ACP routing/thread/identity/card behavior, and add a regression test proving quoted image/file media reaches `MediaPaths`/`MediaTypes`.
