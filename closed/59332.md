## Summary

Current `main` already has the Telegram polling watchdog requested here: it detects stalled `getUpdates` liveness, restarts the Telegram polling cycle without restarting the gateway, rebuilds the transport on stall recovery, exposes a configurable `pollingStallThresholdMs`, documents the behavior, and includes targeted tests.

## Evidence

- **Changelog entry:** The changelog records that Telegram polling gained a raised default watchdog threshold and configurable `channels.telegram.pollingStallThresholdMs` (including per-account overrides).
  - file: [CHANGELOG.md:416](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/CHANGELOG.md#L416)
  - command: `nl -ba CHANGELOG.md | sed -n '412,420p'`
- **Watchdog stall detection and restart:** The polling session sets a watchdog interval, detects stalled polling liveness, logs `Polling stall detected`, marks the transport dirty, stops only the Telegram runner/bot, and then restarts the polling cycle with backoff instead of killing the gateway.
  - file: [extensions/telegram/src/polling-session.ts:313](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/extensions/telegram/src/polling-session.ts#L313)
  - command: `nl -ba extensions/telegram/src/polling-session.ts | sed -n '243,405p'`
- **Per-account config wiring:** The Telegram monitor passes `account.config.pollingStallThresholdMs` into the polling session, so the watchdog threshold is configurable at the channel/account level.
  - file: [extensions/telegram/src/monitor.ts:217](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/extensions/telegram/src/monitor.ts#L217)
  - command: `nl -ba extensions/telegram/src/monitor.ts | sed -n '209,233p'`
- **Public config surface:** The public Telegram config types expose `pollingStallThresholdMs` with a default-oriented docstring for the watchdog threshold.
  - file: [src/config/types.telegram.ts:155](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/src/config/types.telegram.ts#L155)
  - command: `nl -ba src/config/types.telegram.ts | sed -n '149,160p'`
- **Tests cover watchdog behavior:** Tests explicitly verify forced restart when `getUpdates` stalls and verify custom watchdog thresholds plus schema validation for overrides.
  - file: [extensions/telegram/src/monitor.test.ts:721](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/extensions/telegram/src/monitor.test.ts#L721)

  - file: [docs/channels/telegram.md:260](https://github.com/openclaw/openclaw/blob/78a431ed224b6f5b9d58672346551a5b411ad200/docs/channels/telegram.md#L260)
