## Summary

Current main now keeps CLI-backed conversation state in OpenClaw's own transcript and routes fallback candidates through that same transcript, so a claude-cli billing/usage failure no longer leaves the non-CLI fallback dependent on Claude Code's separate JSONL resume state.

## Best Possible Solution

Keep the current main implementation: persist CLI-backed turns into the OpenClaw transcript, compact that transcript when needed, clear stale external CLI resume bindings, and let fallback candidates read the shared OpenClaw session transcript. Any future report should be narrowed to a reproducible legacy-session repair or a missing transcript-import path.
