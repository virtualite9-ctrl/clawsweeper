## Summary

Current `main` does not contain this PR's behavior: `open-prose` is still not marked `enabledByDefault`, and `resolvePluginSkillDirs` still does not pass `enabledByDefault` into activation resolution. At the same time, current docs describe OpenProse as an opt-in bundled plugin, so this remains a live product/behavior change that needs maintainer judgment rather than auto-close cleanup.
