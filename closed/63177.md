## Summary

Current main already treats heartbeat as a main-session turn and preserves meaningful heartbeat output in session history; only no-op heartbeat prompt + `HEARTBEAT_OK` pairs are filtered from later prompt assembly.

## Evidence

- **Docs: heartbeat is a main-session turn by default:** The public heartbeat docs say heartbeat runs periodic agent turns in the main session, and delivery targeting is separate from run context. They also note `session` is optional override and `isolatedSession` is the opt-in mode for fresh history.

## Risks / Open Questions

- If a user explicitly enables `agents.defaults.heartbeat.isolatedSession: true` or points heartbeat at a different `session`, later main-session turns will intentionally not inherit that heartbeat transcript.
