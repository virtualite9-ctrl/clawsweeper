## Summary

Current `main` already normalizes stale `openai-codex` transport config away from the public OpenAI `/v1` path and back onto the Codex transport, with direct regression coverage for the exact legacy `openai-completions` + `https://api.openai.com/v1` case and follow-on doctor warnings for stale overrides. The fix is present on `main` in commit `a52b50ef3c8bb05d4a18302aad6862c37278cc57`, but it is not in the latest tagged release `v2026.4.22`.
