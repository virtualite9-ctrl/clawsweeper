## Summary

Close as `clawhub`: current `main` already provides a dedicated external memory-plugin path, and project policy says optional memory backends should ship as community plugins rather than new bundled core integrations.

## Evidence

## Risks / Open Questions

- This does not verify the quality, maintenance state, or security posture of the external PowerMem plugin itself.
- If PowerMem still needs a missing SDK seam in OpenClaw, that separate gap would need its own focused issue; closing this bundled-plugin request does not resolve SDK deficiencies.
