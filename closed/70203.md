## Summary

Current main implements the requested fix. The issue asked for tiny startedAt/createdAt jitter to stop producing false-positive task-audit warnings, and merged PR #71871 fixed the root cause by normalizing task lifecycle timestamps at the registry boundary on create, update, and restore. Regression tests cover earlier startedAt values and assert zero inconsistent_timestamps findings.

## Best Possible Solution

Close #70203 as implemented on main by #71871. Keep the registry-boundary normalization as the canonical fix because it resolves the false-positive cron audit noise without adding an audit-side jitter exemption that could hide real timestamp-ordering corruption.
