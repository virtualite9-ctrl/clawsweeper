PR #58454 is superseded by the merged #55388/current-main Podman direction. Current main already uses the current-user rootless model, starts Quadlet units with `daemon-reload` + `start`, documents the host-CLI workflow, and keeps safer env/path/port handling that this stale PR would remove or regress.

## Best Possible Solution

Close #58454 as superseded by #55388 and the current main Podman implementation.
