## Summary

Current main already solves the PR's central bug: Ollama memory embedding batches no longer fan out with unbounded Promise.all. The shipped implementation serializes Ollama batch embedding calls with a cap of 1 and has regression coverage asserting no more than one in-flight fetch while preserving prompt order. The remaining related Ollama /api/embed migration items are separate endpoint work, not a reason to keep this fan-out PR open.

## Best Possible Solution

Close PR #53820 as implemented on main. Keep the current serialized Ollama embedding batch guard and regression test; continue tracking separate Ollama endpoint migration work, such as /api/embed support, in the narrower existing items rather than in this fan-out PR.
