## Summary

Keep open: current `main` still enables Node's compile cache during CLI startup, still recommends compile-cache usage in docs/doctor guidance, and the reported `reply-media-paths.runtime` lazy runtime path is not part of the stable dist-entry set that OpenClaw already uses to avoid stale hashed-chunk breakage elsewhere. I found no evidence on `main` of a fix specifically covering global `npm install -g openclaw` updates under Node 24 compile cache.
