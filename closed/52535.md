## Summary

Issue #52535 is a valid feature idea, but current OpenClaw scope puts this in the external skill/plugin path rather than OpenClaw core. Current main explicitly says new skills should go to ClawHub first, and the existing skills system already supports workspace/ClawHub-distributed `SKILL.md` skills, which matches the reporter's `sam-faces` repo-hosted approach.
