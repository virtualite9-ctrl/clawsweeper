## Summary

Current main already streams `openclaw backup create` archives to disk through the pinned `tar@7.5.13` file-mode implementation. OpenClaw awaits `tar.c({ file: tempArchivePath, gzip: true })`, publishes only after completion, cleans temporary output in `finally`, and has regression coverage for tar-write failure cleanup. The explicit pipe form proposed in #42290 would be equivalent for the memory/backpressure concern; separate archive-size/exclude UX remains tracked by related exclude-pattern work.

## Best Possible Solution

Keep the shipped `tar.c({ file })` plus atomic-temp implementation for the core streaming/backpressure concern. Do not merge #42290 solely to replace it with an equivalent explicit pipe. Treat progress UI or smarter exclude defaults as separate product work, with #40786/#44288 as the relevant exclude-pattern tracking context and Ryce's keepmyclaw plugin as an optional ClawHub cloud-snapshot alternative.
