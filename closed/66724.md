## Summary

Current main replaces the accumulated Feishu streaming-card preview buffer with the final payload before idle close, so intermediate tool-call preamble text such as `...` no longer persists in the final card. The latest stable tag v2026.4.24 still had the older merge behavior, so no shipped fixed release is identified yet.

## Best Possible Solution

Close as implemented on main. Keep the final-overwrite Feishu streaming-card behavior and ship it with the next stable release; users still on v2026.4.24 or earlier should update once that release is available.

## Evidence
