## Summary

Current main already solves PR #57668’s central exposure problem with a stricter guard: unauthenticated non-loopback gateway binds are refused before startup, the runtime config repeats the invariant on the resolved bind host, and doctor reports the same condition as a critical warning with fix guidance. The PR’s warning-only patch is redundant and also carries stale review concerns plus an unrelated test-planner change.

## Best Possible Solution

Keep the stricter main implementation and close PR #57668 as redundant. If maintainers still want `openclaw doctor` to exit non-zero for critical security warnings, that should be a separate scoped CLI behavior decision with tests around doctor exit-code ownership, not merged from this stale warning-only branch.
