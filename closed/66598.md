## Summary

Current `main` already contains a targeted Control UI fix for the reported webchat flicker: `session.message` no longer reloads `chat.history` during an active run, and tests now cover the deferred-reload behavior after terminal chat events. The fix appears to be on `main` only and not in the latest tagged release yet.

## Evidence

- **Main fix in event handler:** `handleSessionMessageGatewayEvent` now skips `loadChatHistory` while `chatRunId` is active, with an inline comment explicitly describing the prior race between mid-run history reloads and the optimistic user-message update that delayed the visible user card until the first LLM delta.
