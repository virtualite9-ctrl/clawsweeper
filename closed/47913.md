## Summary

Current main already implements the requested MiniMax/Kimi thinking-control path: MiniMax Anthropic-compatible requests default to API-side thinking disablement to avoid leaked `reasoning_content`, Moonshot/Kimi requests map configured/session thinking into the provider's native `thinking` payload, and per-model `params.thinking` is supported and documented. The relevant implementation is present in the v2026.4.24 release tag.

## Evidence
