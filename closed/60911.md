## Summary

Close #60911 as a duplicate of the earlier open #38570. Both reports ask for the same remaining product change: `sessions_send` inter-agent messages should carry/display the sending agent identity instead of rendering as the human/control user. Current main still lacks a sender/displayName/avatar field on `sessions_send`, so the right cleanup is to keep the canonical issue open and close this later duplicate.

## Best Possible Solution

Close #60911 as a duplicate and keep #38570 as the canonical open tracker for adding a real `sessions_send` sender attribution/display identity contract that the transcript and Control UI can render.
