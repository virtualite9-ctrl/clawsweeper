## Summary

Keep open. Current `main` has Control UI/WebChat image upload support, but it is still explicitly image-scoped; broader document/file upload from the issue body (`PDF`, `Word`, `Excel`, generic attachments) is not implemented, which matches the maintainer comment added on 2026-04-25.

## Evidence

- **Control UI attachment accept list is image-only:** `CHAT_ATTACHMENT_ACCEPT` is hard-coded to `"image/*"`, and the supported MIME helper only returns true for `image/` types.
  - file: [ui/src/ui/chat/attachment-support.ts:1](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/ui/src/ui/chat/attachment-support.ts#L1)
  - command: `nl -ba ui/src/ui/chat/attachment-support.ts | sed -n '1,200p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **UI paste, drag-drop, and file picker only keep images:** Clipboard paste collects only `item.type.startsWith("image/")`; file select and drag-drop skip any file whose MIME type fails `isSupportedChatAttachmentMimeType`; the file input itself uses `accept=${CHAT_ATTACHMENT_ACCEPT}`.
  - file: [ui/src/ui/views/chat.ts:208](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/ui/src/ui/views/chat.ts#L208)

- **Outgoing WebChat attachment payloads are serialized as images:** `buildApiAttachments()` maps every attachment to `{ type: "image", mimeType, content }` before sending `chat.send`. There is no generic `file`/`document` path from the Control UI composer.
  - file: [ui/src/ui/controllers/chat.ts:441](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/ui/src/ui/controllers/chat.ts#L441)
