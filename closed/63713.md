## Summary

Keep open. Current main has overlapping infrastructure, including `commands.list` and generic preservation of existing `__openclaw` transcript metadata, but it does not implement the PR's central authored/runtime prompt metadata path for `chat.send`. The PR discussion also identifies that the proposed branch itself did not persist `__openclaw` metadata back to the on-disk user turn, so the remaining work is still real rather than safely obsolete.

## Best Possible Solution

Keep this PR open only as an implementation candidate, but ask for a focused rebase or replacement PR that drops already-shipped `commands.list` overlap, isolates the authored/runtime prompt metadata change, adds a `chat.send` protocol field deliberately, captures the prepared model input through a typed reply-pipeline hook, persists the metadata into the on-disk user turn via a transcript rewrite or agent-runner propagation, and proves both realtime `session.transcript.update` and `chat.history` behavior with gateway regression tests.
