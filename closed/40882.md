## Summary

Current main does not implement Slack Canvas events, but the request is an optional Slack-specific automation that fits the ClawHub/plugin path. The existing plugin/runtime surfaces are sufficient for a Slack Canvas companion plugin to receive Slack events and enqueue OpenClaw system events; no missing core API or protected maintainer signal is evident.

## Best Possible Solution

Track or publish Slack Canvas support as a ClawHub plugin: a code plugin can own the Slack Canvas OAuth scopes and event subscription, receive Socket Mode or webhook events, and use OpenClaw runtime helpers such as `api.runtime.system.enqueueSystemEvent(...)` or embedded agent/subagent runs. Keep OpenClaw core unchanged unless a concrete plugin API gap is found.
