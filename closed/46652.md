## Summary

Current main now supports the requested admin use case through route binding session overrides: a specific Matrix DM sender can be matched by peer id and forced to `dmScope: "main"`, while the global `session.dmScope: "per-peer"` continues isolating other DMs. This does not make `identityLinks.main` a special sentinel, but it solves the observable TUI/webchat/Matrix unification problem discussed in the issue.
