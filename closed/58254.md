## Summary

Current main already removed the v2026.3.28 launchd `start-after-exit` handoff race and shipped the replacement macOS restart behavior in v2026.4.5.

## Best Possible Solution

Close #58254 as implemented. Keep the shipped v2026.4.5+ approach where launchd KeepAlive owns in-process macOS restarts, explicit CLI/update restarts use the hardened kickstart/re-bootstrap paths, and any future rapid-restart throttle tuning is tracked as a narrower follow-up if users still reproduce it on current releases.
