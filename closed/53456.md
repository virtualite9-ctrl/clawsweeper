## Summary

Current main and v2026.4.24 no longer match the reported v2026.3.23 API-key-only Claude ACP path. The bundled ACPX plugin now uses acpx 0.6.1, whose built-in `claude` alias points to `@agentclientprotocol/claude-agent-acp`; that adapter is built on `@anthropic-ai/claude-agent-sdk`, advertises Claude subscription login via `auth login --claudeai`, and only rejects claude.ai subscriptions when launched with `--hide-claude-auth`, which current OpenClaw does not pass. The earlier workaround link remains useful for older installs, but the OpenClaw core issue is implemented on current main.

## Best Possible Solution

Close as implemented. Keep the shipped bundled acpx plus `@agentclientprotocol/claude-agent-acp` Claude Agent SDK path; if users still see auth failures on v2026.4.24 or newer, handle that as a fresh current-version bug with exact OpenClaw, acpx, Claude ACP adapter, Claude Code versions, and logs. A separate low-priority cleanup can update the stale internal ACP router skill reference to the old `@zed-industries/claude-agent-acp` command.
