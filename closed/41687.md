## Summary

Current `main` intentionally keeps disk-discovered agent session stores in Gateway session loading, and it now has an explicit regression test requiring disk-only/ACP agent sessions to remain visible even when `agents.list` is configured. This PR removes that behavior, so it does not match current shipped intent and would need maintainer redesign rather than cleanup-closing.
