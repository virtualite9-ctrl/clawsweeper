## Summary

Current `main` already fixes the MiniMax image-tool path: MiniMax chat primaries are paired with `MiniMax-VL-01`, image understanding routes through MiniMax's Coding Plan VLM endpoint, and the behavior is covered by new tests and docs. The fix is on `main` but not in a tagged release yet.

## Evidence

- **Image tool now pairs MiniMax chat models with the VLM model:** `resolveImageModelConfigForTool()` explicitly maps MiniMax primaries to `MiniMax-VL-01`, so the image tool no longer uses `MiniMax-M2.7` itself as the vision model.
  - file: [src/agents/tools/image-tool.ts:142](https://github.com/openclaw/openclaw/blob/55d3e7bc473e34883a2d2cce51ce904b87c32eed/src/agents/tools/image-tool.ts#L142)

  - file: [src/agents/minimax-vlm.ts:75](https://github.com/openclaw/openclaw/blob/55d3e7bc473e34883a2d2cce51ce904b87c32eed/src/agents/minimax-vlm.ts#L75)
