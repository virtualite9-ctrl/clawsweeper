## Summary

Current main implements the safe opt-in path requested as the alternative fix: `tools.web.fetch.ssrfPolicy.allowRfc2544BenchmarkRange` is accepted by config, the image tool threads that policy into remote media loading, and SSRF code allows `198.18.0.0/15` only when the policy is enabled. Regression coverage exercises the image tool against a `198.18.x.x` remote image URL with that config.

## Best Possible Solution

Keep the shipped explicit opt-in policy and point affected fake-IP proxy users to configure `tools.web.fetch.ssrfPolicy.allowRfc2544BenchmarkRange: true`; do not weaken the default SSRF policy unless maintainers make a separate security/product decision.
