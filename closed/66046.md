## Summary

Keep open. Current `main` still does not show a shipped `google-vertex` auth-path fix, and the code matches the reported 401 regression rather than disproving it.

## Evidence

- **Issue discussion confirms a real regression:** The provided issue body reports last-known-good `2026.3.28` and first-known-bad `2026.4.12`. Follow-up comments confirm the same 401 on `2026.4.14` and `2026.4.15`, including a direct bearer-token `curl` success against Vertex and a report that `anthropic-vertex` still works with the same service account, which points at OpenClaw's `google-vertex` path rather than a bad GCP setup.
- **`google-vertex` env auth still resolves ADC to a marker, not a bearer token:** `resolveEnvApiKey("google-vertex")` returns `GOOGLE_CLOUD_API_KEY` if present, otherwise returns the non-secret marker `gcp-vertex-credentials` when project, location, and ADC files are present. There is no token minting here.
