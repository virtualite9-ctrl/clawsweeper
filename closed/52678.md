## Summary

Current `main` already has a new workspace-first skill loading path, sandbox skill sync, and runtime skill loading that collectively address the reported failure mode where a worker used a bundled `SKILL.md` path instead of the workspace copy.

## Evidence

- **Workspace skills override bundled skills on current main:** `loadSkillEntries()` loads bundled skills, project/workspace skills, then merges by name with documented precedence `extra < bundled < managed < agents-skills-personal < agents-skills-project < workspace`, so a workspace-local skill wins over any bundled skill with the same name.

- **Regression test covers workspace-over-bundled preference:** There is explicit test coverage that writes bundled, managed, and workspace copies of the same skill and asserts that the prompt contains only the workspace version, not the bundled version.

That means workers start from the synced workspace skill set instead of looking up a bundled install tree for the active skill.
