## Summary

Close #56126 as a duplicate of the broader open tracker #9865. The requested Anthropic Batch API routing for non-urgent cron `agentTurn` jobs is the same remaining product/API work already tracked for background LLM turns; current main still lacks the cron batch flag and provider batch-turn seam, so the feature should continue in #9865 rather than this narrower duplicate.

## Best Possible Solution

Close #56126 as a duplicate of #9865. Continue design and implementation in #9865 with a provider/runtime batch-turn contract covering submit, poll, timeout, cancel/reconcile, transcript handling, and delayed delivery, then add explicit cron opt-in flags, docs, and tests once that seam exists.
