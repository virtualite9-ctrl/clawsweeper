## Summary

Current main already hardens the shared chat normalization/rendering paths that would have produced this undefined `.map()` crash, and the TUI path now guards non-array content before any mapping.

## Evidence

- **UI message normalization defaults content to an array:** `normalizeMessage()` starts with `let content: MessageContentItem[] = [];` and only maps user content after that initialization, so assistant/user messages no longer carry undefined `content` into rendering.
  - file: [ui/src/ui/chat/message-normalizer.ts:264](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/ui/src/ui/chat/message-normalizer.ts#L264)
  - command: `nl -ba ui/src/ui/chat/message-normalizer.ts | sed -n '240,390p'`
  - sha: [608c08fc54da](https://github.com/openclaw/openclaw/commit/608c08fc54da2f86338366f8eeafd7e2a02713aa)
- **Control UI groups always allocate `messages` before render:** `groupMessages()` constructs each `MessageGroup` with `messages: [{ message, key }]`; `renderMessageGroup()` still does `group.messages.map(...)`, but current main's builder invariant guarantees the array exists before that render path runs.
  - file: [ui/src/ui/chat/build-chat-items.ts:150](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/ui/src/ui/chat/build-chat-items.ts#L150)

- **TUI extraction guards non-array content:** The TUI formatter helpers return early unless `content` is an array, so the terminal rendering path no longer has an obvious `undefined.map(...)` route on assistant/user message content.
  - file: [src/tui/tui-formatters.ts:222](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/src/tui/tui-formatters.ts#L222)

- **Shared assistant text extraction also guards `content` shape:** `extractAssistantTextForPhase()` exits when `content` is not an array and only uses `.map()` after an `Array.isArray(entry.content)` check, covering the shared assistant-text path the TUI uses.
  - file: [src/shared/chat-message-content.ts:135](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/src/shared/chat-message-content.ts#L135)
