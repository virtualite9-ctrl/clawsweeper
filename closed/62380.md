## Summary

Current OpenClaw already ships a newer node-exec flow that supersedes this report: shell execution on nodes now goes through `exec host=node`, headless Linux/Windows nodes use `~/.openclaw/exec-approvals.json`, and the node host explicitly implements `system.run.prepare` with the response shape the exec path expects.

## Evidence

- **Nodes CLI docs in released build:** The released `v2026.4.23` docs explicitly say `system.run` and `system.run.prepare` are blocked from `nodes invoke`, and operators should use the `exec` tool with `host=node` instead of `openclaw nodes run`. That means the report's reproduction path no longer matches the supported interface.
