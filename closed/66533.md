## Summary

Current main already prevents heartbeat/system-event turns from writing synthetic heartbeat origin metadata into the main session, and v2026.4.24 includes regression coverage for the reported Control UI/WebChat selector pollution path. The remaining true-fresh-session semantics are separate and are tracked by #69599.

## Best Possible Solution

Close #66533 as implemented on main via the shipped system-event metadata guard. Keep the broader independent fresh-session behavior in the canonical follow-up #69599, and treat the linked heartbeat-filter PRs as obsolete or narrower hardening during PR cleanup.

## Evidence
