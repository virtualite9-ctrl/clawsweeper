## Summary

Current `main` already covers the reported internal gateway pairing gap: agent gateway calls identify as `gateway-client` in `backend` mode, `cron.add` resolves to `operator.admin`, and the gateway handshake now silently approves trusted local pairings/upgrades for loopback/shared-secret local paths instead of requiring manual `paired.json` surgery.

## Evidence

- **Internal gateway client now uses backend mode:** `callGatewayTool()` sends internal tool calls as `gateway-client` with `mode: BACKEND`, including cron tool traffic that goes through this helper.
  - file: [src/agents/tools/gateway.ts:148](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/agents/tools/gateway.ts#L148)

- **Cron methods require admin scope:** `cron.add`, `cron.update`, `cron.remove`, and `cron.run` are mapped to `operator.admin`, so the internal client is no longer limited to `operator.read`.
  - file: [src/gateway/method-scopes.ts:151](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/gateway/method-scopes.ts#L151)
