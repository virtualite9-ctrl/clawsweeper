## Summary

Keep open. Current `main` still has the reported `undefined.trim()` crash points, and this PR's proposed coercion is not a safe close candidate because it can turn a required credential prompt failure into a silent empty/`keep` path instead of forcing retry.

## Evidence

- **Current main still trims the raw wizard text result directly:** `buildChannelSetupWizardAdapterFromSetupWizard` still does `const trimmedValue = rawValue.trim();` immediately after `await prompter.text(...)`, so the vulnerable call site remains on `main`. The validator only protects normal empty-string input; it does not defend against a non-string/undefined return escaping the prompter.
