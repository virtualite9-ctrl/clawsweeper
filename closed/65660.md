## Summary

Current main fixes the reported webchat TTS compaction failure by keeping TTS audio as transcript/display-only media and filtering gateway-injected transcript-only assistant messages out of model replay before prompt precheck/context accounting. The direct estimator change proposed in #65688 was not merged, but the observable user problem from #65660 is addressed on main. I did not find evidence that this has shipped in the latest release tag, so fixedRelease is null.

## Best Possible Solution

Close #65660 as implemented on current main. Keep the shipped replay-boundary fix for webchat TTS audio; if maintainers want extra defense-in-depth, track a separate narrow issue to make the generic char estimator treat arbitrary audio blocks as small or non-textual.
