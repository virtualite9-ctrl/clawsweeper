## Summary

Current main and v2026.4.24 already implement the Telegram group plain-text path described by #57738. The later #63599 repro linked from the discussion was closed as implemented, and current source/tests show admitted Telegram group messages with `requireMention:false` pass through routing, session context, and reply handling for both allowlisted and open group policies.

## Best Possible Solution

Close #57738 as implemented on main and in v2026.4.24. Keep the shipped Telegram group routing/authorization behavior, and leave narrower remaining Telegram topic override work to #49864/#49980 rather than keeping this broad older report open.
