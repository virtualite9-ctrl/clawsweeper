## Summary

Current main still matches the report: WhatsApp Web has an internal 30-minute inbound-silence watchdog, still force-closes the socket with status 499, and the supported config/schema/docs expose only web.enabled, web.heartbeatSeconds, and web.reconnect.*. #65290 fixed the related creds.json corruption/reconnect race, but not this watchdog configurability gap. #70608 appears to be a newer duplicate; #65215 is older and more detailed, so it should remain the canonical open item.

## Best Possible Solution

Keep #65215 open as the canonical high-signal watchdog configurability bug. The likely fix belongs in the WhatsApp/web config contract: add a bounded documented setting for the idle watchdog threshold and possibly a disable/check interval option, wire it from config into monitorWebChannel, update generated schema/help/docs, and add regression tests for configured quiet-session behavior. Treat newer #70608 as a duplicate/link target rather than closing this older detailed report.
