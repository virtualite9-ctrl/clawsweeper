## Summary

Current `main` already supports account-scoped routing bindings, and the Feishu channel passes the receiving bot account into route resolution. The requested `bindings[].match.account/accountId` behavior is implemented and shipped.

## Evidence

- **Binding schema includes account-scoped matching:** `AgentBindingMatch` includes optional `accountId`, so bindings can target a specific channel account.
  - file: [src/config/types.agents.ts:37](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/src/config/types.agents.ts#L37)
