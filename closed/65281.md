## Summary

Current main implements the WebChat image paste/send path end to end: Control UI stages pasted images as attachments, serializes them into chat.send, Gateway parses/keeps or offloads image attachments, and the reply pipeline accepts image-only webchat turns via opts.images. The reported model families are also image-capable in the current provider catalogs, and the release changelog records the Gateway/WebChat attachment-preservation fix in 2026.4.23.

## Best Possible Solution

Close #65281 as implemented on current main. Keep the shipped Control UI/Gateway attachment pipeline; any narrower polish around atomic multi-image paste buffering can continue under the broader attachment-pipeline work such as #45077 if maintainers still want it.
