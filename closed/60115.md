## Summary

Keep open. Current main still has the Mattermost DM threading gap this PR targets: docs say DMs stay non-threaded, but `resolveMattermostReplyRootId` has no chat-kind input and still falls back to `payload.replyToId`, which the delivery path passes through to `root_id`. The linked bug #59981 remains open, and related PR #55186 is a different closed-unmerged thread-root-priority attempt, not a shipped superseding fix.

## Best Possible Solution

Keep #60115 open for maintainer review or land an equivalent focused fix: make `resolveMattermostReplyRootId` chat-kind aware, return `undefined` for `kind === "direct"`, update the three delivery call sites, and add the direct-message regression tests while preserving channel/group thread-root behavior.
