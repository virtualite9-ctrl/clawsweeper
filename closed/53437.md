## Summary

Current main already solves the exec approval modal overflow problem this PR targets: the modal is viewport-bounded, the command block is height-bounded and scrollable, long command text wraps, mobile keeps the same constraints, and large approval command strings are sanitized/truncated before display. The relevant implementation is also present in the latest verified release tag v2026.4.24, so the PR is obsolete rather than still needed.

## Best Possible Solution

Close this PR as implemented on current main. Keep the shipped modal sizing, command scroll/wrap behavior, and command display sanitizer; any future horizontal-scroll or `overflow-wrap` polish should be a fresh focused UI regression with visual proof and without unrelated test-formatting churn.
