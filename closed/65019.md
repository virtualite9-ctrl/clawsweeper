## Summary

Close #65019 as a duplicate of the older open Slack suppression request #55252. Current main still does not implement `suppressBareText`/`suppressAssistantText`; the remaining work is already tracked by #55252 and the associated PR #55223, so keeping this newer issue open would split the same Slack product/design discussion.

## Best Possible Solution

Close #65019 as a duplicate of #55252. Keep the remaining Slack suppress-bare-assistant-text semantics and implementation review on #55252 and PR #55223; open a separate cross-channel design issue only if maintainers want to generalize the behavior beyond Slack.
