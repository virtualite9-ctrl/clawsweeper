## Summary

Keep open. Current main has adjacent cleanup for heartbeat acknowledgements, empty/internal-only turns, and hidden runtime context, but it does not implement the PR's user-role `System:` / `System (untrusted):` transcript filter. The related discussion also flags a real false-positive risk in this PR's matcher, so the best maintainer path is to keep/canonicalize the remaining work rather than auto-close it.

## Best Possible Solution

Keep this open or explicitly canonicalize it under #67036/#68508 with maintainer judgment. The best code direction is a narrow, shared predicate for actual runtime-injected timestamped exec/system event rows, covered in both `chat.history`/session-history tests and Control UI load/render tests, while preserving legitimate user-authored `System:` text.
