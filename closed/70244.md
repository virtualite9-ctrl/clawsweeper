Current main still uses a global BlueBubbles short-id map and resolves numeric message ids without caller chat context across the affected react/edit/unsend/reply, sendText, media, and inbound delivery paths. The PR is a focused recent bug fix with tests and discussion, not an obsolete or duplicate cleanup candidate.
