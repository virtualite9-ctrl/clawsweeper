## Summary

Current `main` already contains and documents an ACPX probe-agent fix that matches this regression: `v2026.4.22` added `plugins.entries.acpx.config.probeAgent`, passes it into the ACPX runtime, and explicitly stopped marking the backend unavailable when the default `codex` probe agent is not the installed/authenticated ACP agent.

## Evidence

- **2026.4.5 lacked a probe-agent config path:** In `v2026.4.5`, the resolved ACPX plugin config return block did not include any `probeAgent` field, so readiness probing could only follow the runtime default behavior.
  - file: [extensions/acpx/src/config.ts:231](https://github.com/openclaw/openclaw/blob/3e72c0352dde84a0bcb3aabafa99c2d4b12d1c46/extensions/acpx/src/config.ts#L231)

  - file: [extensions/acpx/src/config.ts:263](https://github.com/openclaw/openclaw/blob/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a/extensions/acpx/src/config.ts#L263)
