## Summary

Keep open. Current `main` still lacks user-facing per-section gates for the CLI quick reference, reply-tag guidance, and messaging/components prompt text. The `available_skills` filtering part is already partially covered by existing skill allowlists, but the core prompt-gating request itself is not implemented.

## Evidence

- **Full prompt still hardcodes reply-tag and messaging guidance:** `buildAssistantOutputDirectivesSection()` and `buildMessagingSection()` only check `isMinimal`; there is no config-backed per-section toggle for these blocks in the main/full prompt path. The issue's `Reply Tags` wording is slightly stale on current `main`, where that guidance now lives under `## Assistant Output Directives` rather than a separate `## Reply Tags` heading.
