## Summary

Keep open. Current `main` still treats LM Studio `@iq*` quant suffixes as auth-profile delimiters instead of part of the model id, so the reported `/model` and allowlist failures remain plausible and directly supported by the code.

## Evidence

- **Quant-suffix parser does not preserve `@iq*` names:** `splitTrailingAuthProfile()` explicitly preserves `@YYYYMMDD`, `@q*`, and `@4bit`/`@8bit` suffixes, but nothing matching LM Studio names like `@iq3_xxs` or `@iq4_xs`. For `lmstudio/qwen3.6-27b@iq3_xxs`, this code would split at `@` and treat `iq3_xxs` as a profile override.
  - file: [src/agents/model-ref-profile.ts:18](https://github.com/openclaw/openclaw/blob/ec8dbc4595583816f71af2568df61dbe7c090e1f/src/agents/model-ref-profile.ts#L18)
  - command: `nl -ba src/agents/model-ref-profile.ts | sed -n '1,120p'`
  - sha: [ec8dbc459558](https://github.com/openclaw/openclaw/commit/ec8dbc4595583816f71af2568df61dbe7c090e1f)
- **`/model` directive path uses that splitter:** `extractModelDirective()` runs `splitTrailingAuthProfile(raw)` and stores `split.model` as the directive's `rawModel`. That means `/model lmstudio/qwen3.6-27b@iq3_xxs` is normalized to `lmstudio/qwen3.6-27b` before later model-selection logic sees it.
  - file: [src/auto-reply/model.ts:37](https://github.com/openclaw/openclaw/blob/ec8dbc4595583816f71af2568df61dbe7c090e1f/src/auto-reply/model.ts#L37)
  - command: `nl -ba src/auto-reply/model.ts | sed -n '1,80p'`
  - sha: [ec8dbc459558](https://github.com/openclaw/openclaw/commit/ec8dbc4595583816f71af2568df61dbe7c090e1f)
- **Allowlist/model-resolution path also strips the suffix:** `resolveModelRefFromString()` calls `splitTrailingAuthProfile(params.raw)` before parsing the model ref. That matches the issue's reported failure mode where an exact configured LM Studio ref with `@iq*` becomes a truncated ref and then fails allowlist lookup as `model not allowed`.
  - file: [src/agents/model-selection-shared.ts:397](https://github.com/openclaw/openclaw/blob/ec8dbc4595583816f71af2568df61dbe7c090e1f/src/agents/model-selection-shared.ts#L397)
  - command: `nl -ba src/agents/model-selection-shared.ts | sed -n '390,417p'`
  - sha: [ec8dbc459558](https://github.com/openclaw/openclaw/commit/ec8dbc4595583816f71af2568df61dbe7c090e1f)
- **LM Studio docs promise exact key handling:** The public LM Studio docs say OpenClaw model refs should use the exact LM Studio `key` returned by `/api/v1/models`, and `--custom-model-id` likewise takes the model key as returned by LM Studio. Truncating a quant suffix breaks that documented contract.
