## Summary

Current main already avoids the reported double-provider-prefix failure for image model routing. The image fallback path parses configured `provider/model` refs into separate `{ provider, model }` candidates before calling the image runtime, so `bailian/qwen3.5-plus` is passed onward as provider `bailian` and model `qwen3.5-plus`, not `bailian/qwen3.5-plus`. The linked PRs #68906 and #68919 correctly identified the double-prefix risk, but the current shared fallback/parser path solves the observable issue without the proposed one-line helper change.

## Best Possible Solution

Close the issue as implemented on current main. Keep the shared parser-based image fallback routing, and treat the linked unmerged one-line helper PR as unnecessary unless maintainers want a separate cleanup. If a Bailian failure still reproduces on a current release, it should come back as a new issue with current `agents.defaults.imageModel` config, provider model catalog, exact logs, and version.
