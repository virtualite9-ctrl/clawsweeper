## Summary

Current main already supports exactly this integration path: OpenClaw can store remote MCP server definitions with auth headers, expose configured MCP tools to embedded Pi agents in normal coding/messaging profiles, and package MCP server config via bundles. AgentBase can be connected through the existing MCP surface without new core work.

## Risks / Open Questions

- This is generic MCP tool access, not a special first-class memory-slot integration; consistent pre-task search/post-task storage behavior still depends on prompt/plugin setup rather than a dedicated core workflow.
- This review was read-only, so I did not run a live AgentBase session; the decision is based on the released docs and current runtime implementation.
