## Summary

Current main implements the issue's accepted minimum path: gateway-level HTTP(S)_PROXY with NO_PROXY bypass. Brave web_search runs through the trusted env-proxy web tools path, while Feishu WebSocket setup uses the ambient Node proxy agent, whose proxy-agent dependency consults proxy-from-env and bypasses NO_PROXY matches. Gateway startup also now bootstraps Undici env proxy dispatching before provider/plugin network work, so the workaround no longer depends on the first embedded agent attempt.

## Best Possible Solution

Close this core issue as implemented on main and keep the shipped env-proxy plus NO_PROXY behavior. A narrower future request for first-class UI/config sugar around per-web-search-provider or per-channel proxy routing can be opened separately if maintainers want that product surface beyond the working NO_PROXY path.
