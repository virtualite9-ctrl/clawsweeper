## Summary

Close #46441 as ClawHub/community plugin scope. Current main already exposes the needed pre-tool-call hook and approval surfaces for external guardrail plugins, and the matching core GuardrailProvider PR #64868 was closed by Peter as better suited for ClawHub/community plugin work.

## Best Possible Solution

Keep the shipped before_tool_call/plugin approval contract as the OpenClaw core surface, route guardrail providers such as CRE or APort Agent Guardrails through ClawHub/plugin distribution, and open a narrow SDK/API issue only if a concrete provider cannot be implemented with the existing hook context, block, rewrite, and approval semantics.

## Risks / Open Questions

- Current main intentionally does not ship the proposed core guardrails config/provider subsystem; this close relies on the maintainer scope decision that the capability should live as plugins unless a narrower SDK gap is identified.
