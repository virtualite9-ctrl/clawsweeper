## Summary

Current `main` no longer matches #63221: `sessions_spawn` now creates a fresh UUID-backed child session, patches the requested model onto that child session before launch, and the agent startup path resolves from persisted override fields. Regression tests also lock the patch/store/launch ordering. The fix appears to be on `main` only; I could not tie it to a released tag newer than `v2026.4.23`.

## Evidence

- **Fresh child session key + prelaunch patch:** `spawnSubagentDirect` generates a fresh child key as `agent:${targetAgentId}:subagent:${crypto.randomUUID()}` and immediately calls `sessions.patch` with `plan.initialSessionPatch` before any run starts.
  - file: [src/agents/subagent-spawn.ts:686](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/agents/subagent-spawn.ts#L686)
  - command: `nl -ba src/agents/subagent-spawn.ts | sed -n '680,760p'`
  - sha: [86f8c826e20a](https://github.com/openclaw/openclaw/commit/86f8c826e20a01d1bba67c0ba6e73182fbf0455f)
- **`sessions.patch` turns `model` into override fields:** Gateway `sessions.patch` does not leave the requested model as an inert runtime field: patching `model` resolves the model ref and applies it through `applyModelOverrideToSessionEntry`, which writes `providerOverride` / `modelOverride` and marks a live switch pending.
  - file: [src/gateway/sessions-patch.ts:388](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/gateway/sessions-patch.ts#L388)
  - command: `nl -ba src/gateway/sessions-patch.ts | sed -n '360,450p'`
  - sha: [86f8c826e20a](https://github.com/openclaw/openclaw/commit/86f8c826e20a01d1bba67c0ba6e73182fbf0455f)
- **Agent startup resolves from override fields:** `agentCommandInternal` checks stored `providerOverride` / `modelOverride` and uses those values for the provider/model actually launched. That means the prepatched child session model is what the spawned subagent boots with.
  - file: [src/agents/agent-command.ts:663](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/agents/agent-command.ts#L663)
  - command: `nl -ba src/agents/agent-command.ts | sed -n '640,760p'`
  - sha: [86f8c826e20a](https://github.com/openclaw/openclaw/commit/86f8c826e20a01d1bba67c0ba6e73182fbf0455f)
- **`modelApplied` is only returned after patch + runtime persistence succeeds:** After the initial `sessions.patch`, `spawnSubagentDirect` persists the child runtime model, sets `modelApplied = true` only on success, and only then returns the accepted spawn result.
  - file: [src/agents/subagent-spawn.ts:777](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/agents/subagent-spawn.ts#L777)
