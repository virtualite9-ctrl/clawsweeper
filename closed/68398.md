## Summary

Current main no longer has a runtime path where subagent-registry can reference cleanupProvisionalSession. The registry lazily imports the subagent announce module, completion cleanup is handled inside subagent-announce with sessions.delete, and cleanupProvisionalSession remains private to subagent-spawn with only same-file callers. The source-verified issue comment and the later PR discussion correctly identified that an export-only fix was incomplete; main resolved the underlying module-graph problem instead.

## Best Possible Solution

Keep the current lazy subagent-announce loading and direct completion cleanup implementation. The remaining export-only PR #68583 is unnecessary for current main because cleanupProvisionalSession is not part of the registry completion path anymore.
