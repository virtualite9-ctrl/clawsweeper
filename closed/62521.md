## Summary

Current `main` already fixes the reported gateway-startup false warning by carrying the source config snapshot into plugin loading, merging trust settings from that snapshot before warning evaluation, and covering the behavior with a regression test; the changelog says this shipped in `v2026.4.22`.

## Evidence

- **Changelog explicitly marks the bug class fixed in 2026.4.22:** The `2026.4.22` changelog says plugin trust decisions are read from the source config snapshot so `plugins.allow` stays enforced and `doctor`/gateway startup no longer warn that the allowlist is empty when it is configured.
