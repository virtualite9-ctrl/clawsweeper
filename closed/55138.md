## Summary

Close #55138 as superseded by open canonical issue #63925. Current main still does not expose WhatsApp watchdog timeout/check interval as user config, but #63925 tracks the same remaining feature request more explicitly; the shipped reconnect-loop fix only reset watchdog state after reconnect and did not add config fields.

## Best Possible Solution

Close #55138 as a duplicate and keep #63925 as the single open tracker. The eventual fix should add a typed public WhatsApp watchdog timeout/check-interval config contract, wire it through account startup into monitorWebChannel, and update schema/help/docs/tests while preserving the existing reconnect-state reset behavior.
