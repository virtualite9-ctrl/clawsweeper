## Summary

Current OpenClaw already supports binding TUI sessions to non-default agents through agent-scoped session keys, workspace inference, and in-TUI agent switching, so the requested capability is already available on `main` and in release `v2026.4.22`.

## Evidence

- **CLI docs describe explicit agent-scoped TUI startup:** The TUI CLI docs say launching from an agent workspace auto-selects that agent unless `--session` is explicitly `agent:<id>:...`.
  - file: [docs/cli/tui.md:25](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/docs/cli/tui.md#L25)
  - sha: [608c08fc54da](https://github.com/openclaw/openclaw/commit/608c08fc54da2f86338366f8eeafd7e2a02713aa)
- **Docs define agent-prefixed session keys:** The TUI guide states session keys are stored as `agent:<agentId>:<sessionKey>` and that `/session agent:other:main` switches to that other agent explicitly.
  - file: [docs/web/tui.md:63](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/docs/web/tui.md#L63)
  - sha: [608c08fc54da](https://github.com/openclaw/openclaw/commit/608c08fc54da2f86338366f8eeafd7e2a02713aa)
- **Interactive agent switching is already exposed:** The TUI guide documents both the Ctrl+G agent picker and the `/agent <id>` command.
  - file: [docs/web/tui.md:93](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/docs/web/tui.md#L93)
  - sha: [608c08fc54da](https://github.com/openclaw/openclaw/commit/608c08fc54da2f86338366f8eeafd7e2a02713aa)
- **Startup preserves explicit agent binding:** `resolveTuiSessionKey` keeps explicit `agent:` session keys intact, and `resolveInitialTuiAgentId` prefers the agent encoded in the session key before falling back to workspace/default inference.
  - file: [src/tui/tui.ts:132](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/src/tui/tui.ts#L132)
  - sha: [608c08fc54da](https://github.com/openclaw/openclaw/commit/608c08fc54da2f86338366f8eeafd7e2a02713aa)
- **Runtime execution routes off the session key:** The embedded TUI passes the canonical session key into `agentCommandFromIngress`, which is how the run is bound to the selected agent session.
  - file: [src/tui/embedded-backend.ts:562](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/src/tui/embedded-backend.ts#L562)
  - sha: [608c08fc54da](https://github.com/openclaw/openclaw/commit/608c08fc54da2f86338366f8eeafd7e2a02713aa)
- **Agent resolution comes from the session key:** `resolveSessionAgentIds` derives the effective agent from the `agent:<id>:...` session key when present.
  - file: [src/agents/agent-scope.ts:48](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/src/agents/agent-scope.ts#L48)
  - sha: [608c08fc54da](https://github.com/openclaw/openclaw/commit/608c08fc54da2f86338366f8eeafd7e2a02713aa)

## Risks / Open Questions

- The exact `--agent` flag requested in the issue is still not present on `openclaw tui` (`src/cli/tui-cli.ts:14`), so the remaining gap is discoverability/ergonomics rather than missing capability.
