## Summary

Current `main` already implements the requested clean-build behavior, but in a narrower and safer place than this PR. `tsdown.config.ts` now sets `clean: true` on the single unified dist build while leaving bundled plugin sub-builds at `clean: false`, which addresses stale hashed chunks without re-cleaning `dist/` for every helper-generated config. That supersedes this PR's one-line change to `nodeBuildConfig()`.
