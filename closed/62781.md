## Summary

Current `main` already fixes the reported crash: `getQueueState()` now backfills a missing `activeTaskWaiters` set on legacy global queue state before `notifyActiveTaskWaiters()` or `resolveActiveTaskWaiter()` touch it, and the same behavior is covered by a regression test that simulates the missing-field restart case. The latest release `v2026.4.23` also contains that code.

## Evidence
