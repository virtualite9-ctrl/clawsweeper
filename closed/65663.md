## Summary

Current main already handles the Azure/OpenAI Responses reasoning replay failure: Azure Responses is classified with the Responses family, replay history is repaired/sanitized, the live provider stream applies the same downgrade immediately before outbound calls, and regression tests cover the relevant reasoning/tool-call repair paths. The linked and title-matched related reports (#49167, #57043, #57499, and #65603) have also already been resolved as implemented/superseded by the same replay work.

## Best Possible Solution

Close this 2026.4.11-era report as implemented. Keep the current Responses-family replay sanitizer and outbound stream guard; if Azure still rejects reasoning/tool replay on v2026.4.24 or later, track it as a new narrow bug with current payload/log evidence and whether a context engine rewrote the assembled messages.
