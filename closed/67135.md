## Summary

Current main addresses the reported WebChat/session context false overflow. The prior issue comment correctly identified this as a metrics bug; current code now treats session `totalTokens` as a fresh prompt/context snapshot, not an accumulated provider billing total, and OpenAI-style cached-token usage is normalized to avoid double-counting cache reads. `/new` also resets the session row before the fresh greeting run, with startup context bounded by config/docs.

## Best Possible Solution

Close this issue as implemented. Keep the current prompt-snapshot based `SessionEntry.totalTokens` path, stale-snapshot suppression in WebChat, and bounded `/new` startup-context behavior. Any remaining report should be a new issue against a current build with a fresh context assembly report, session row, transcript usage entry, and affected provider/plugin config.
