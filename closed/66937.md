## Summary

Keep open. Current main still requires a non-empty LM Studio API key in the interactive onboarding path, even though non-interactive setup and runtime have partial synthetic/keyless support.

## Best Possible Solution

Keep this issue open and fix LM Studio's plugin-owned interactive setup in extensions/lmstudio/src/setup.ts. Blank API-key input should select local unauthenticated mode using the existing lmstudio-local/synthetic-auth semantics, avoid writing a real API-key profile for that case, preserve real key/env/ref/header flows, add an interactive regression test for blank input, and update the public LM Studio docs to describe keyless local setup accurately.
