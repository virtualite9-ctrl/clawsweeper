## Summary

The report targets a QClaw/neodata proxy endpoint and error shape that current OpenClaw main does not implement. The current gateway has no `neodata`, `QClaw`, `/proxy/api`, `forward_error`, or `9021` surface, and its HTTP handler would fall through to a plain 404 for unmatched paths rather than returning the reported JSON. This looks like an external QClaw/neodata integration or downstream packaging/service issue, not an actionable OpenClaw core repo bug.

## Best Possible Solution

Close this upstream issue as outside the OpenClaw source repository. The reporter should take the 9021/9002 `/proxy/api` failure to the QClaw/neodata plugin or service owner; a new OpenClaw issue would be useful only if it includes a current OpenClaw plugin id/link, config, logs, and a concrete missing core plugin/gateway API.
