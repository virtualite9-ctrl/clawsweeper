## Summary

Current `main` already covers this PR’s goal with a broader OAuth refresh flow: near-expiry OAuth credentials are treated as stale before hard expiry, refreshes are serialized through a shared per-profile lock, and the concurrent-agent race now has direct regression coverage. The open PR is therefore redundant against shipped code on `main`.
