## Summary

Current `main` now clears stale auto-failover session overrides before allowlist validation and also resolves bare fallback models to their owning provider when building the allowed set. Those changes match the reported Web/TUI `model not allowed` fallback-loop symptoms for multi-provider Ollama chains, but they appear to be newer than the tagged `v2026.4.23` release in this checkout.

## Evidence

- **Auto-failover overrides now self-heal before allowlist checks:** `createModelSelectionState` clears direct session overrides whose `modelOverrideSource === "auto"` before allowlist validation, then resets the in-memory provider/model back to the configured primary for the current turn.
