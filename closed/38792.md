## Summary

Current main now handles the reported silent-stall failure mode: empty provider-error turns after tool chains are retried when safe, and exhausted/incomplete turns produce a visible error payload instead of leaving the user waiting silently. The related unmerged PR #38812 correctly described the issue, and later mainline fixes cover the observable behavior.
