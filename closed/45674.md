## Summary

Keep PR #45674 open. Current main still constructs the Feishu/Lark WebSocket client without the proposed keepalive fields, and the linked #42354 regression remains a concrete WebSocket-mode failure report. However, the current Lark SDK contract in use by main does not expose a constructor `wsConfig` option, so this PR likely needs revision and regression coverage rather than cleanup closure.

## Best Possible Solution

Keep #45674 open for maintainer/author follow-up. The useful product outcome is still to make Feishu/Lark WebSocket mode reliable for #42354, but the implementation should be rebased against current main, validated against `@larksuiteoapi/node-sdk@1.61.1`, and covered with a regression test for the failing WebSocket endpoint response or replaced by a better PR that fixes the same channel path.
