## Summary

Current main already addresses the 2026.3.13 local loopback WebSocket handshake timeout class: the gateway pre-auth handshake window is now 10 seconds with an override, the one-shot CLI gateway call path yields before opening the WebSocket to avoid starving connect.challenge handling, and the affected cron/probe commands use those shared gateway RPC paths. The fix is recorded in the 2026.3.22 changelog entry for #49262.
