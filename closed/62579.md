## Summary

Current `main` already uses consistent managed QMD collection names and does not reference `memory-alt-main` in production code; the same behavior is present in the shipped `v2026.4.23` release.

## Evidence

- **Default QMD collections are canonicalized to root+dir only:** `resolveDefaultCollections` builds only `memory-root-<agent>` and `memory-dir-<agent>` for default workspace memory. There is no `memory-alt-*` production collection name in this path.
