## Summary

Keep #72329 open. Current main accepts `compat.thinkingFormat: "qwen-chat-template"` in config/schema, and it now has manual `params.chat_template_kwargs` / `params.extra_body` pass-throughs for OpenAI-compatible proxy payloads, but the requested automatic Qwen/vLLM mapping from OpenClaw's resolved `/think` state to `chat_template_kwargs.enable_thinking` is still absent. The only automatic vLLM chat-template injection I found is Nemotron-specific, so the central issue remains valid.

## Best Possible Solution

Keep the issue open and implement the missing runtime mapping in the OpenAI-compatible completions path. The fix should translate `compat.thinkingFormat: "qwen-chat-template"` plus the resolved OpenClaw thinking level into `chat_template_kwargs.enable_thinking` for Qwen/vLLM, preserve explicit `params.chat_template_kwargs` and `params.extra_body` precedence, and add targeted tests plus vLLM/Qwen docs. The existing manual `params.chat_template_kwargs` workaround is useful but does not solve the requested per-session `/think` behavior.
