## Summary

Current `main` no longer has the strict-equality-only `tools.exec.host=auto` behavior from the report. Explicit `host=node` is allowed from `auto` when no sandbox runtime is active, and sandboxed `auto` sessions intentionally still stay on sandbox.

## Evidence

- **Current exec target logic:** `isRequestedExecTargetAllowed` now has explicit `configuredTarget === "auto"` handling instead of strict equality only. It allows explicit overrides from `auto` unless a sandbox runtime is active and the request is trying to leave sandbox.

## Risks / Open Questions
