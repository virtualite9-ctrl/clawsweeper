Current `main` already has model-provider proxy support, so this Codex review recommends closing as implemented.

- `src/config/zod-schema.core.ts:270` defines `models.providers.*.request.proxy` with `env-proxy` and `explicit-proxy` modes.
- `src/agents/provider-request-config.ts:272` and `src/agents/provider-request-config.ts:573` normalize that config and turn it into runtime dispatcher policy.
- `src/agents/provider-transport-fetch.ts:153` applies the dispatcher policy to model-provider fetches, so the proxy path is actually used at request time.
- `src/infra/net/fetch-guard.ts:164` routes explicit proxy URLs through the shared `ProxyAgent` path.
- `src/agents/provider-request-config.test.ts:190` covers explicit-proxy and env-proxy behavior, and the feature was already documented in release `v2026.4.23` in `docs/gateway/configuration-reference.md:2598`.
