## Summary

Keep #58686 open. Current main still has the behavior this PR fixes: `sessions_spawn` defaults to `runtime="subagent"`, parses `streamTo`, then returns an error before reaching the native subagent spawn path. The checked-in test suite still asserts that old rejection behavior. Related cleanup attempts in the provided GitHub context are closed unmerged, while the linked bug reports remain open, so this PR is not obsolete or superseded by a shipped fix.

## Best Possible Solution

Keep #58686 open or apply the same narrow fix on main: treat `streamTo` as an ACP-only output-routing hint, forward it only to `spawnAcpDirect`, silently drop it for `runtime="subagent"` or omitted runtime, keep `resumeSessionId` strict for non-ACP, and replace the current rejecting test with explicit/implicit subagent acceptance coverage.
