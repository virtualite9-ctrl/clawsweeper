## Summary

Current main already covers the user problem behind #48911: a user can put the rolling filename pattern under any directory via `logging.file`, and the logger now follows the current-day file in that directory. The exact `logging.dir` key is not present, but adding a second config key is redundant with the shipped/current rolling-path behavior.

## Best Possible Solution

Close this PR as redundant with current main. Keep `logging.file` as the single supported config surface for both fixed paths and rolling dated paths; if maintainers want more clarity, a narrow docs-only follow-up should document `/custom/dir/openclaw-YYYY-MM-DD.log` rather than adding `logging.dir`.
