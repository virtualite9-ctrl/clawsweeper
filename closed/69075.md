## Summary

Issue #69075 is now superseded by the linked open PR #69342, which explicitly tracks the requested `models.providers.*.models[].extraParams` config field, schema acceptance, runtime merge behavior, and tests. Current main still does not expose that direct provider-model field, while an existing `agents.defaults.models["provider/model"].params.extra_body` path covers the OpenRouter-style request-body passthrough workaround for OpenAI-compatible proxies.

## Best Possible Solution

Close this issue as superseded by PR #69342 and let that PR remain the canonical review surface for adding direct provider-model `extraParams`; users needing the OpenRouter request-body passthrough in the meantime can use the documented `agents.defaults.models["provider/model"].params.extra_body` path for OpenAI-compatible proxy models.
