## Summary

PR #46350 is superseded by merged PR #71506 and current main. The active memory remote HTTP helper now lives under `src/memory-host-sdk/host/remote-http.ts`, not the old `src/memory/remote-http.ts` target, and it opts into `TRUSTED_ENV_PROXY` only when `shouldUseEnvHttpProxyForUrl()` says Undici will actually proxy the URL. The superseding implementation also added tests for the memory helper and the `ALL_PROXY`/`NO_PROXY` hardening cases.

## Best Possible Solution

Close PR #46350 as superseded by merged #71506. Keep the hardened current memory-host SDK implementation, which only opts into trusted env-proxy mode when Undici will actually proxy the target, and do not continue this stale branch against the removed `src/memory/remote-http.ts` path.
