## Summary

Current main contains the keepalive and `agent.wait` hardening that matches the reported ACP hang: tick keepalives are no longer sent as droppable broadcasts, the gateway client watchdog does not self-close while a request is pending, and regression tests cover the exact long-running `agent.wait`/chat overlap path. The changelog records this as a shipped Gateway keepalive fix in v2026.4.12.

## Evidence

- **Changelog shipped fix:** The 2026.4.12 changelog explicitly says Gateway keepalive was fixed so slow or backpressured clients no longer self-disconnect with `tick timeout` while long-running work is still alive, referencing issue `#65256`, which matches this issue's cross-reference and symptom pattern.
