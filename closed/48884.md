## Summary

Current main already ships a different fix for the reported Windows/Node 24 handshake timeout symptom: the gateway client yields one event-loop turn before starting CLI WebSocket calls, and that behavior is covered by a regression test. Because the PR’s own repro says raw WebSocket connections already worked while CLI/browser commands failed, the shipped client-side mitigation matches the actual failing surface better than this server-side timer reordering PR.

## Evidence
