## Summary

Current main already implements the #70139 fix. Commit 8bc4d4bcd4049485d6c5cb37413b31834796ec3b moved the post-attachment-parse claim into `registerChatAbortController`, which atomically refuses a duplicate same-run registration and returns `{status: "in_flight"}` before starting another run. Main also includes a regression test for the attachment retry race and an Unreleased changelog entry crediting the reporter.

## Best Possible Solution

Close #70139 as implemented on main. Keep the helper-based atomic registration and the regression test; any future work should be a new issue only if a fresh reproduction shows duplicate attachment-backed `chat.send` runs still escaping the in-flight path.
