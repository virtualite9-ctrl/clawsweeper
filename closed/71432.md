## Summary

This asks for a broad multi-agent group-chat orchestration layer. Current main intentionally keeps agents and sessions isolated, while exposing session tools, plugin hooks, and plugin runtime subagent helpers that are sufficient for an external orchestration plugin. The issue discussion also points to KinthAI running this pattern outside core, which supports ClawHub/plugin-layer incubation rather than a core session-model change.

## Best Possible Solution

Keep OpenClaw core's isolated agent/session model and incubate mesh-style group chat as a ClawHub/community plugin or bundle using plugin hooks, session tools, and api.runtime.subagent. If an implementer finds a specific missing extension seam, open a narrower SDK/API issue for that seam rather than keeping this broad core feature request open.
