## Summary

Current main still does not expose `OPENCLAW_SESSION_KEY` to exec child processes, and this PR is a plausible partial implementation. It should stay open for maintainer review rather than close as obsolete, but it is not ready as-is because the broader related request in #66705 also asks for `OPENCLAW_AGENT_ID`, the node-host `system.run` path is separate from `runExecProcess`, and the new env contract needs regression coverage and explicit precedence/clearing behavior.

## Best Possible Solution

Keep this PR open as useful but require a focused revision: rebase on current main, drop or let rebase eliminate unrelated UI churn, inject trusted `OPENCLAW_SESSION_KEY` with deterministic override/clearing semantics, add regression tests for gateway and PTY plus sandbox/backend and node-host coverage if the contract is meant to cover all exec hosts, update the env and exec docs, and decide whether to include `OPENCLAW_AGENT_ID` here or split it into a clearly linked follow-up for #66705.
