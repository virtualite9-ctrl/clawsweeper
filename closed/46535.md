## Summary

Current `main` has largely superseded the original built-in TTS failure by moving TTS delivery onto structured `details.media` with `audioAsVoice`, but it has not fully implemented this PR’s generic legacy-path fix. The legacy text-directive paths still drop `audioAsVoice`, so this PR remains useful and should stay open (likely narrowed/rebased to the remaining legacy propagation gap).

## Evidence

That covers the maintainer’s note that current `main` fixed the built-in TTS production path through structured media rather than legacy `MEDIA:` text tags.
