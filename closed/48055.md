## Summary

Current main already solves the core Web Dashboard problem: users can browse server-backed sessions and resume them in chat via the chat session selector or the Sessions page. The UI is not a ChatGPT-style left rail, but the requested browse/resume/server-side persistence behavior is implemented well enough for maintainer cleanup.
