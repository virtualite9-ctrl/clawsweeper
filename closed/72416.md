## Summary

Keep this PR open. Current main still rejects prerelease `openclaw.install.minHostVersion` floors, because the parser only accepts stable `>=x.y.z` strings and both install plus manifest-registry loading enforce that same helper. The supplied PR diff is a focused implementation candidate with matching docs/tests, and the security pass found no unexplained workflow, lockfile, package-script, secret, publishing, or dependency-source changes.

## Best Possible Solution

Keep this PR open and review or land it as the implementation candidate. The best product path is to update `src/plugins/min-host-version.ts` to accept prerelease semver floors while preserving the `>=`-floor-only contract, keep enforcement centralized for install and manifest-registry loading, and ship the accompanying plugin-author docs plus regression tests.
