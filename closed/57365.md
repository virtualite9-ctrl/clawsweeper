## Summary

Current main treats third-party channel schemas as manifest metadata. External plugins that omit `openclaw.plugin.json#channelConfigs` are expected to update their plugin manifest/package metadata; OpenClaw core intentionally does not load arbitrary non-bundled `src/config-schema.ts` files for cold Control UI schema discovery.

## Best Possible Solution

Do not merge core source discovery for arbitrary external plugin config-schema modules. Keep bundled synthetic discovery scoped to bundled plugins, and have `openclaw-zulip` or similar third-party channel plugins publish the generated channel JSON Schema in `openclaw.plugin.json#channelConfigs` or add plugin-side packaging/generation to keep that manifest metadata in sync.
