## Summary

Current main already implements the requested fix. Task lifecycle timestamps are normalized at create, update, and restore time, so pre-captured startedAt values no longer cause false startedAt < createdAt audit warnings. This supersedes PR #70293's narrower create-time clamp.

## Best Possible Solution

Close #70293 as implemented on current main. Keep the broader registry-level timestamp normalization from #71871; any remaining task-audit timestamp noise should be filed as a fresh regression with current audit output and representative task records.
