## Summary

Current `main` already fixes this class of routing bug. Since commit `527895f0362f91897bd85ca1116ad16a05650934` and release `v2026.4.12`, heartbeat/cron/exec system-event turns preserve the existing external session route instead of letting synthetic `heartbeat` origin/targets overwrite later delivery, and there is a regression test covering the Feishu shared-session case.
