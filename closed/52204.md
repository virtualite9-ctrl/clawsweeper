## Summary

Current main already contains the shared-session routing guards this issue asks for, and the same logic is present in the shipped v2026.4.22 release: ingress route data is carried through as turn-source context, reply routing prefers that current route over session lastRoute, and delivery-context merges no longer cross-pollinate route fields across channels.

## Evidence

- **Gateway captures the inbound route for reply routing:** The agent gateway handler reads the inbound request channel/to/accountId/threadId and passes them into delivery planning as turnSource fields, so replies are anchored to the current turn instead of only the persisted session route.
  - file: [src/gateway/server-methods/agent.ts:766](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/src/gateway/server-methods/agent.ts#L766)
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)
- **Delivery planner explicitly protects shared sessions from cross-channel replies:** The delivery planner documents and implements turnSourceChannel as an override for session lastChannel specifically to prevent cross-channel reply routing in shared sessions (dmScope="main").
  - file: [src/infra/outbound/agent-delivery.ts:36](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/src/infra/outbound/agent-delivery.ts#L36)
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)
- **Session target resolution prefers the current turn route:** resolveSessionDeliveryTarget states that turnSourceChannel overrides the session-level lastChannel to prevent cross-channel reply routing when multiple channels share the same session and one inbound message updates lastChannel while another turn is still in flight.
  - file: [src/infra/outbound/targets-session.ts:56](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/src/infra/outbound/targets-session.ts#L56)
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)
- **Regression tests cover the shared-session race:** Tests assert that when turnSourceChannel is set, the resolver does not fall back to stale session metadata, and that a mismatched turnSourceTo drops the stored threadId in the shared-session race case.
  - file: [src/infra/outbound/targets.test.ts:796](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/src/infra/outbound/targets.test.ts#L796)
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)
- **Delivery-context merges no longer cross channels:** mergeDeliveryContext now detects conflicting channels and refuses to inherit to/accountId/threadId from the fallback channel, which directly addresses the deliveryContext mixing described in the report; the helper is covered by dedicated tests.
  - file: [src/utils/delivery-context.shared.ts:112](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/src/utils/delivery-context.shared.ts#L112)
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)
- **Docs already describe the intended isolation settings:** The security and WeChat docs explain that default dmScope="main" shares DM continuity by design, and that per-channel or per-account-channel-peer dmScope should be used when full DM isolation is desired.
  - file: [docs/gateway/security/index.md:515](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/docs/gateway/security/index.md#L515)
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)

## Risks / Open Questions

- If a third-party plugin bypasses OpenClaw's outbound routing and posts directly to a provider API, these core routing guards will not help; that would be outside the core code path reviewed here.
- This conclusion is based on current code, shipped tag contents, docs, and regression tests rather than a live reproduction against the reporter's exact external plugin build.
