## Summary

Current `main` has moved TTS provider ownership to the plugin registry, so a local CLI TTS integration is now better scoped as an external speech-provider plugin published via ClawHub rather than another core/bundled-provider patch.

## Evidence

- **Vision scope anchor:** `VISION.md` says core should stay lean and optional capability should usually ship as plugins; the bar for adding optional plugins to core is intentionally high.
  - file: [VISION.md:52](https://github.com/openclaw/openclaw/blob/50e36983bb2d827adc617282c090c0a0359c6eee/VISION.md#L52)
  - sha: [50e36983bb2d](https://github.com/openclaw/openclaw/commit/50e36983bb2d827adc617282c090c0a0359c6eee)
- **Speech providers are plugin-owned on current main:** The architecture docs explicitly say plugins can register speech providers with `api.registerSpeechProvider(...)` and that speech providers should own synthesis behavior.
  - file: [docs/plugins/architecture-internals.md:379](https://github.com/openclaw/openclaw/blob/50e36983bb2d827adc617282c090c0a0359c6eee/docs/plugins/architecture-internals.md#L379)
  - sha: [50e36983bb2d](https://github.com/openclaw/openclaw/commit/50e36983bb2d827adc617282c090c0a0359c6eee)
- **ClawHub publish path exists for provider plugins:** The provider-plugin guide includes Speech (TTS) as an optional plugin capability and documents publishing provider plugins with `clawhub package publish`.
  - file: [docs/plugins/sdk-provider-plugins.md:483](https://github.com/openclaw/openclaw/blob/50e36983bb2d827adc617282c090c0a0359c6eee/docs/plugins/sdk-provider-plugins.md#L483)
  - sha: [50e36983bb2d](https://github.com/openclaw/openclaw/commit/50e36983bb2d827adc617282c090c0a0359c6eee)
- **TTS config is already registry-driven:** On current main, `TtsProvider` is just `string`, and TTS config uses a generic provider-owned `providers` map rather than a hardcoded enum of built-ins.
  - file: [src/config/types.tts.ts:1](https://github.com/openclaw/openclaw/blob/50e36983bb2d827adc617282c090c0a0359c6eee/src/config/types.tts.ts#L1)
  - sha: [50e36983bb2d](https://github.com/openclaw/openclaw/commit/50e36983bb2d827adc617282c090c0a0359c6eee)
- **User-facing TTS docs now expect registered provider ids:** The TTS docs describe `provider` as a registered speech provider id and `providers.<id>` as provider-owned settings, which matches the plugin model rather than adding new core-special-cased ids.
  - file: [docs/tools/tts.md:297](https://github.com/openclaw/openclaw/blob/50e36983bb2d827adc617282c090c0a0359c6eee/docs/tools/tts.md#L297)
  - sha: [50e36983bb2d](https://github.com/openclaw/openclaw/commit/50e36983bb2d827adc617282c090c0a0359c6eee)

  - file: [src/gateway/server-methods/tts.ts:134](https://github.com/openclaw/openclaw/blob/50e36983bb2d827adc617282c090c0a0359c6eee/src/gateway/server-methods/tts.ts#L134)

- **Voice-call schema is no longer enum-bound to a fixed small TTS set:** The voice-call plugin schema now accepts `tts.provider` as a plain string and allows arbitrary provider-owned entries under `tts.providers`, so extending bundled schema enums is no longer the preferred path.
  - file: [extensions/voice-call/openclaw.plugin.json:476](https://github.com/openclaw/openclaw/blob/50e36983bb2d827adc617282c090c0a0359c6eee/extensions/voice-call/openclaw.plugin.json#L476)
