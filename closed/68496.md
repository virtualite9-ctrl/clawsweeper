## Summary

Issue #68496 asks for a new first-class bundled ACP harness alias/skill-doc entry for Hermes. Current main does not bundle `hermes`, but the request is an optional third-party harness integration that can be handled through existing ACPX command overrides plus a ClawHub skill/bundle, without a missing OpenClaw core API.

## Best Possible Solution

Close this core issue as ClawHub-scoped. The best path is an external Hermes ACP skill/bundle or small setup recipe that configures `plugins.entries.acpx.config.agents.hermes.command = "hermes acp"`, updates `acp.allowedAgents` or `acp.defaultAgent` when needed, and teaches the router skill to use `agentId: "hermes"`. If maintainers later want Hermes as a bundled baseline harness, that should be a focused maintainer product decision in OpenClaw/acpx rather than this general cleanup item.
