## Summary

Current `main` already fixes the linked TUI `/sessions` preview bug by sourcing that UI from `lastMessagePreview`, and that transcript preview path now ignores non-`user`/`assistant` entries. This PR’s sanitizer changes a different `sessions.preview` helper path, so it is no longer needed to resolve the reported regression.
