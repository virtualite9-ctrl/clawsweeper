## Summary

Current `main` already covers the two concrete failure modes in this report. The Control UI/gateway path now preserves provider-qualified model refs and clears stale provider carry-over that could fabricate mixed ids, and the auth login flow now clears stale `disabled:*` lockouts before re-auth so a repaired provider can recover without waiting for the old billing/auth disable window.

## Evidence

- **Control UI keeps picker/session model refs provider-qualified:** `OpenClawChatModelChoice.selectionID` always uses the catalog provider prefix, and `normalizedModelSelectionID()` / `providerQualifiedModelSelectionID()` preserve or rebuild a provider-qualified ref instead of cross-composing ids from another provider.
