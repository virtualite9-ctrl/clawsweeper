## Summary

PR #45986 should close as superseded by the cleaner, scoped #46069. Current main still lacks the memory-lancedb `encoding_format: "float"` change, so this is not implemented-on-main, but #45986 is not the best vehicle because it bundles an unrelated gateway-status timeout change that review comments identified as regressive. #46069 tracks the same #45982 memory-lancedb/Ollama issue, narrows the SDK type, and updates the memory-lancedb test without the gateway change.

## Best Possible Solution

Close #45986 as superseded/dirty and keep #46069 as the canonical scoped memory-lancedb fix, or land an equivalent small patch that adds `encoding_format: "float" as const` plus the corresponding memory-lancedb test update. Keep gateway-status timeout behavior in its own focused issue or PR.
