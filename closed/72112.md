## Summary

Current main implements the gateway-restart MCP stdio cleanup path: gateway restart calls the server close lifecycle, shutdown disposes all cached/in-flight session MCP runtimes, each runtime closes its MCP transports/clients, and stdio transports tear down the process tree if graceful close stalls. Main also has a session idle TTL backstop for the requested idle cleanup behavior.

## Best Possible Solution

Close this core issue as implemented on main. Keep the lifecycle-based fix: gateway restart/shutdown disposes owned MCP runtimes, stdio transports kill their owned process groups, and mcp.sessionIdleTtlMs handles idle cleanup. If maintainers later want hard-crash recovery for already-orphaned third-party processes, track that separately with an ownership-safe PID/process-group design rather than a broad command-name sweep.
