## Summary

Current main already contains the Discord inbound-media hang fix and later raised Discord's default media cap to 100MB, so the reported ~12MB attachment case from v2026.3.23 is already addressed on shipped releases.

## Risks / Open Questions

- The exact landing commit for the original fix was not recoverable from this checkout because history is shallow and `gh` was unavailable without a token, so `fixedSha` is left null.
- If a user has explicitly configured a much lower `channels.discord.mediaMaxMb`, oversized attachments can still be rejected, but current code should time out/fallback rather than wedge the Discord channel worker.
