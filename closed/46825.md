## Summary

Current main already implements the PR's user-facing request: ACPX MCP server env values can use SecretRefs/SecretInput and are resolved through the generic plugin config-contract secrets runtime before MCP server spawn. The shipped implementation also avoids the PR review's empty-string regression by leaving plain string env values untouched.

## Best Possible Solution

Close this PR as already implemented on main. Keep the shipped generic manifest config-contract implementation rather than merging the older ACPX-specific schema-widening approach; any future expansion of `secrets configure`/audit listing for arbitrary plugin config paths should be handled as a separate focused follow-up.
