## Summary

Keep #70871 open. Current main still exposes full session messages to `before_prompt_build` hooks, the bundled `memory-lancedb` auto-recall path embeds the raw hook prompt without a truncation/focused-query layer, and its config/manifest do not expose a recall timeout or query-size cap. Adjacent Active Memory code has bounded recent-turn query controls and a timeout, but that is a separate plugin path and does not resolve the reported memory-plugin auto-recall contract.

## Best Possible Solution

Keep the issue open and decide the memory auto-recall contract explicitly: either add a bounded `recallQuery`/query-builder helper for memory plugins or document that plugins must bound `prompt`/`messages` themselves. Then update the bundled `memory-lancedb` auto-recall path with focused-query extraction, max input limits, configurable timeout behavior, graceful fallback on embedding-size errors, and regression tests covering large current prompts and long session histories.
