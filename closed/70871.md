## Summary

The report is about the external Mem0 memory plugin (`openclaw-mem0` / `@mem0/openclaw-mem0`) embedding too much input during its own auto-recall path. Current OpenClaw main does not bundle that plugin, and the existing plugin hook surface already gives plugins the current prompt separately from session messages, so bounding/truncating the recall query and exposing plugin-specific timeout config can be fixed in the plugin/ClawHub path without a core OpenClaw API change.

## Best Possible Solution

Close this core issue and track the runtime fix in the `openclaw-mem0` plugin/ClawHub package: derive recall from a bounded query, truncate before embedding, and expose any plugin-owned recall timeout. Keep OpenClaw core's generic hook contract as-is unless plugin maintainers identify a concrete missing API that cannot be solved with `before_prompt_build`'s current `prompt` and `messages` fields.
