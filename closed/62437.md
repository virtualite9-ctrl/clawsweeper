## Summary

The report now looks like an invalid or mixed channel-config state that `openclaw doctor` repairs, not an open core bug on current `main`. A second affected user reported the same send failure on Telegram and confirmed the problem disappeared after running doctor, and current `main` explicitly gates `message` commands on invalid config with a `doctor --fix` hint.

## Evidence

- **Discussion shows the symptom cleared after doctor repair:** Issue comment 4212886855 from 2026-04-09 says the same send failure affected the Telegram send plugin and that `openclaw doctor` fixed the config and restored normal behavior.
  - command: `GitHub context comment 4212886855`
- **`message` commands run doctor config preflight:** Current main runs `runDoctorConfigPreflight` before normal CLI execution for commands that may mutate state; that path includes `message` commands.
  - file: [src/cli/program/config-guard.ts:45](https://github.com/openclaw/openclaw/blob/ecac696643c604c6d854341bfe6424bfa31423e9/src/cli/program/config-guard.ts#L45)
  - sha: [ecac696643c6](https://github.com/openclaw/openclaw/commit/ecac696643c604c6d854341bfe6424bfa31423e9)
- **Invalid config now hard-stops non-allowlisted commands with a doctor repair hint:** `message` is not in the invalid-config allowlist. If config is invalid, the CLI prints `Config invalid`, shows `Run: openclaw doctor --fix`, and exits with status 1 instead of continuing startup.
  - file: [src/cli/program/config-guard.ts:5](https://github.com/openclaw/openclaw/blob/ecac696643c604c6d854341bfe6424bfa31423e9/src/cli/program/config-guard.ts#L5)
  - sha: [ecac696643c6](https://github.com/openclaw/openclaw/commit/ecac696643c604c6d854341bfe6424bfa31423e9)
- **Tests cover the `message` invalid-config path:** The test suite asserts that `['message']` triggers doctor preflight and that invalid config on `message` emits a `doctor --fix` hint and exits.
  - file: [src/cli/program/config-guard.test.ts:92](https://github.com/openclaw/openclaw/blob/ecac696643c604c6d854341bfe6424bfa31423e9/src/cli/program/config-guard.test.ts#L92)
  - sha: [ecac696643c6](https://github.com/openclaw/openclaw/commit/ecac696643c604c6d854341bfe6424bfa31423e9)
- **Docs describe `doctor --fix` as the repair path for mixed channel-account config:** The channels docs explicitly call out mixed legacy/account-scoped channel config and direct users to `openclaw doctor --fix` to migrate that state.
  - file: [docs/cli/channels.md:80](https://github.com/openclaw/openclaw/blob/ecac696643c604c6d854341bfe6424bfa31423e9/docs/cli/channels.md#L80)
  - sha: [ecac696643c6](https://github.com/openclaw/openclaw/commit/ecac696643c604c6d854341bfe6424bfa31423e9)
- **Doctor docs document headless-safe repair behavior:** The doctor docs describe `--fix` as the supported repair path, note that headless runs skip prompts, and say `--fix` drops unknown config keys while listing removals.
  - file: [docs/cli/doctor.md:41](https://github.com/openclaw/openclaw/blob/ecac696643c604c6d854341bfe6424bfa31423e9/docs/cli/doctor.md#L41)
