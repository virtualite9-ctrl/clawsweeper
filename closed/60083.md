## Summary

The reported duplicate does not reproduce against current main. At the second restart time in the issue, `2026-04-03T04:21:00Z` is `06:21 CEST`, so the previous `20:00 Europe/Berlin` cron slot is `2026-04-02T18:00:00Z`, not the same-day `2026-04-03T18:00:00Z` slot. Because the late catch-up `lastRunAtMs` is `2026-04-03T01:56:00Z`, current main's missed-slot comparison is false, while `nextRunAtMs` remains in the future.

## Best Possible Solution

Close as not reproducible and keep the current startup missed-slot replay semantics. If a distinct duplicate still occurs, it should be reported with exact persisted job state, restart timestamps, timezone, OpenClaw version, and cron logs so maintainers can verify a different schedule-state shape.
