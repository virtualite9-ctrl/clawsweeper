## Summary

Current main has moved the ACP delivery path out of the broken 2026.4.2 behavior: ACPX registers the plugin-owned reply_dispatch hook, bound ACP follow-ups are routed through ACP runtime output projection and origin delivery, internal task-completion wakes sent into ACP sessions are materialized as plain prompts with the child result, and stale persisted ACP resume IDs now retry with a fresh session instead of reusing the missing backend session.

## Best Possible Solution

Close #60872 as implemented on main. Keep the current plugin-owned ACP reply_dispatch path, plain ACP internal-event materialization, and fresh retry for stale resume IDs; continue tracking remaining ACP process/orphan cleanup separately in #61895 rather than keeping this delivery-path regression open.
