## Summary

Current `main` already contains the Control UI/webchat fix that defers same-session `chat.history` reloads until the active run finishes, which matches the reported optimistic-message flicker/disappear/reappear regression. The changelog records that fix as shipped in `v2026.4.15`.

## Evidence

- **Current main defers same-session history reload during active runs:** `handleSessionMessageGatewayEvent` now skips `loadChatHistory(...)` while `host.chatRunId` is active, stores a deferred reload key instead, and documents that mid-run history reloads raced with the optimistic user-message update and delayed the user message card from appearing.
  - file: [ui/src/ui/app-gateway.ts:496](https://github.com/openclaw/openclaw/blob/9a0b26cafca76d267dd231b5eaceadb071ec749d/ui/src/ui/app-gateway.ts#L496)
