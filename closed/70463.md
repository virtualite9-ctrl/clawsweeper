## Summary

#70463 reports the same remaining WhatsApp quiet-session 30-minute watchdog/reconnect problem now tracked by canonical open issue #70678 and active maintainer PR #71466. Current main still has the hardcoded WhatsApp extension-local watchdog and generic health monitor restart path, so this should close as duplicate/superseded, not as fixed.

## Best Possible Solution

Close #70463 as superseded by canonical open issue #70678 and keep implementation/review focused on maintainer PR #71466 or an equivalent WhatsApp runtime fix. The fix should stop treating inbound-message silence as socket failure for quiet WhatsApp sessions, or make liveness transport-backed, with regression coverage for quiet sessions and reconnect behavior.
