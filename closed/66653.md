## Summary

Current main still has the reported `.trim()` call sites, but the values feeding them are already normalized to strings by the active wizard prompters and Telegram token resolver. This PR reads as a defensive patch for a reproduction that does not match the current `main` contracts, and one change (`normalizeTelegramAllowFromInput`) would also leave the type signature inconsistent with the added null guard.
