## Summary

Current main preserves Telegram native slash-command arguments through the skill-command dispatch path and forwards only the raw argument string to tool-dispatched skills, with command name and skill name supplied separately. The issue’s later comments that 2026.4.9 was still affected are useful, but the current main checkout and the v2026.4.24 release tag both contain the argument-preserving path.

## Best Possible Solution

Close this issue as implemented. Keep the shipped raw-argument tool dispatch contract (`command` contains only the args, while `commandName` carries the slash command), and consider adding a narrow Telegram native skill-command tool-dispatch regression test separately if maintainers want stronger guardrail coverage.
