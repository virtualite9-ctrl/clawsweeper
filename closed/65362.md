## Summary

Current main already implements the Ollama discovery behavior requested here. The Ollama plugin discovers local/self-hosted models from the configured Ollama host via `/api/tags`, normalizes `/v1` URLs to the native API root, and has regression coverage for both remote base URLs and local discovery. The related same-day Ollama discovery report #65119 was also closed as implemented after the 2026.4.20 Ollama discovery fix; the comments on this issue suggested Ollama upgrade/workaround paths, but current OpenClaw no longer depends on that workaround for this discovery path.

## Best Possible Solution

Close this as implemented and keep the plugin-owned Ollama discovery path on main. If a user can still reproduce on a current release, the best follow-up is a fresh issue with the exact command, `models.providers.ollama` config, Ollama `/api/tags` output, OpenClaw version, and logs so maintainers can distinguish discovery from command-display or network/auth issues.
