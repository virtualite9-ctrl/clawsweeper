## Summary

Keep open. Current main still has the source-of-truth split this PR targets: `nodes status` reads the effective `node.list` catalog, while unfiltered `nodes list` still renders only `node.pair.list`. The PR is a plausible focused implementation candidate, and its provided review thread shows the initial bot feedback was addressed by later PR commits.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land this focused CLI fix, or an equivalent change in `src/cli/nodes-cli/register.status.ts`, so unfiltered `openclaw nodes list` prefers the effective `node.list` paired-node view, preserves pending and legacy pairing metadata from `node.pair.list`, keeps filtered modes strict, and adds command-level regression coverage in `src/cli/program.nodes-basic.e2e.test.ts`.
