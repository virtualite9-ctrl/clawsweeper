## Summary

Keep open. Current main still does not appear to handle the reported failure mode: LM Studio/OpenAI-compatible chat-completions turns send tools and execute structured `delta.tool_calls`, but a standalone bracketed MemPalace block emitted as normal text would be appended as assistant-visible content rather than promoted to a tool call or explicitly rejected.

## Best Possible Solution

Keep this issue open. Add a regression fixture for the LM Studio/OpenAI-compatible chat-completions stream that emits the reported standalone bracket block, then implement a narrow safe behavior: either convert a standalone bracket block whose name exactly matches a registered tool into a normal tool call before assistant delivery, or suppress/reject unsupported local-model text-tool formats with a clear non-hanging error. Investigate the MemPalace `predicate` argument mismatch separately against the advertised MCP schema.
