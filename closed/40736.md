## Summary

Current main already implements the reported tagged/structured thinking leak fix. The assistant stream path strips <think>/<thinking>/<thought>/<antthinking> blocks before user-facing block/channel delivery, Ollama no longer falls back from thinking/reasoning fields into visible content, and MiniMax Anthropic-compatible requests default to disabled thinking to avoid leaked reasoning_content. The same implementation is present in v2026.4.24.

## Best Possible Solution

Close #40736 as implemented on main and keep the current layered fix: provider-side controls for MiniMax/Ollama-style reasoning fields plus output-boundary stripping for tagged reasoning. Use narrower follow-up issues for any remaining provider-specific untagged reasoning narration.
