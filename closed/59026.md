## Summary

Current main already fixes the observable Discord ACP thread-routing bug. Parent-channel Discord messages now resolve bindings by the exact current conversation id, task terminal updates use direct requester delivery when possible instead of queuing a parent-session event, and system-event/session routing code avoids resurrecting stale thread ids. The same evidence is present in the shipped v2026.4.24 release.

## Best Possible Solution

Keep the shipped implementation: Discord bindings remain exact-thread scoped, ACP terminal updates use direct requester-origin delivery when possible, and session/dispatch routing prevents stale child thread ids from being reused for parent-channel turns. If maintainers want immediate auto-unbinding after ACP terminal states as a separate product choice, that should be tracked narrowly, but it is no longer required to prevent this misroute.

## Evidence

The Discord binding adapter then resolves only by the exact thread/conversation id, so a child thread binding is not selected for a parent-channel turn.
