## Summary

Current main implements the requested behavior for the Control UI/webchat path: runtime startup context is still sent to the model on bare /new or /reset, but the visible/persisted transcript uses transcript-safe user text instead of the startup prelude. The related #68094 discussion is addressed by the same transcript separation and system-event exclusion.
