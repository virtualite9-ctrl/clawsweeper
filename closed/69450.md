## Summary

Current main already scopes the hot `chat.send` session-key lookup to the target agent's session store. The all-agent combined store remains for explicit global discovery surfaces, but keyed per-message sends resolve the owning agent first and load only that agent's candidate store(s).

## Best Possible Solution

Keep the current main implementation: keyed Gateway sends should continue resolving the canonical session key first, then loading the owning agent's store and transcript path. If a future slowdown remains, track it as a narrower performance report for the specific API that still needs all-agent discovery, with profiling data for that surface.
