## Summary

Current `main` already handles `requests-in-flight` heartbeats via the wake retry layer, includes regression coverage for that exact stall path, and the changelog shows the fix shipped in `v2026.3.7`.

## Evidence

- **Wake layer retries requests-in-flight:** `heartbeat-wake` requeues the pending wake and schedules `DEFAULT_RETRY_MS` retry when the handler returns `{ status: "skipped", reason: "requests-in-flight" }`, so the chain is not left unscheduled.
