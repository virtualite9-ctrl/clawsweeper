## Summary

PR #46305 is a local release-overlay/custom-server change rather than an actionable OpenClaw core change. The PR adds an explicit overlay list for copying local files into a fresh release source and hard-codes local runtime packages into the official Dockerfile. Current main has no `update-openclaw-release` script or overlay-list consumer, already exposes documented Docker build arguments for extra apt packages, and routes skill-specific binaries through skill metadata, sandbox setup commands, or custom images. The PR's overlay list also references stale `src/signal/*` paths that do not exist on current main.

## Best Possible Solution

Close this PR as repository-not-actionable. Keep OpenClaw's official Dockerfile generic and reproducible, use `OPENCLAW_DOCKER_APT_PACKAGES` for local apt packages, use skill installer metadata or `agents.defaults.sandbox.docker.setupCommand`/a custom image for skill-specific binaries like Whisper, and keep private release-overlay file lists outside `openclaw/openclaw` unless a maintainer proposes a supported release-overlay feature.
