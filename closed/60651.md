## Summary

Current `main` already routes `openclaw cron status` through the same shared gateway RPC stack as `openclaw gateway call cron.status`, resolves/fails gateway SecretRefs explicitly before credential selection, and treats `OPENCLAW_NO_RESPAWN=1` as a supported startup mode. The report's layered `1006` diagnosis does not match the current source, tests, docs, and shipped changelog entries.

## Evidence
