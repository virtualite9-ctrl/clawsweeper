## Summary

Keep open. Current `main` has new Codex auth plumbing on both the browser OAuth path and a separate device-code path, but I did not find strong evidence that issue #51175's specific browser-flow `403 unsupported_country_region_territory` failure is actually fixed. The browser path still delegates the code->token exchange to `@mariozechner/pi-ai/oauth`, there is no in-repo regression test or changelog entry for this 403/parity case, and the alternative device-code flow is not proof that the reported browser flow now succeeds.
