## Summary

Current `main` already hardens Anthropic replay against signed-thinking mutation in the exact sanitization/replay paths named in the report, and those paths are unchanged from `v2026.4.23`, so this is implemented on main and already shipped.

## Evidence

- **Signed thinking blocks are preserved during assistant-content sanitization:** When `preserveSignatures` is true, assistant content bypasses thought-signature stripping, and if any `thinking` / `redacted_thinking` block is present OpenClaw skips the empty-text filtering path that would otherwise rebuild or reorder sibling blocks.
  - file: [src/agents/pi-embedded-helpers/images.ts:126](https://github.com/openclaw/openclaw/blob/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a/src/agents/pi-embedded-helpers/images.ts#L126)
  - sha: [c997a9f9788e](https://github.com/openclaw/openclaw/commit/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a)
- **Replay repair treats signed-thinking turns as immutable:** For provider-owned signed-thinking replay, assistant turns that contain thinking plus tool calls are preserved only when they are already replay-safe; otherwise OpenClaw drops the whole assistant turn instead of mutating provider-owned content.
  - file: [src/agents/session-transcript-repair.ts:280](https://github.com/openclaw/openclaw/blob/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a/src/agents/session-transcript-repair.ts#L280)
  - sha: [c997a9f9788e](https://github.com/openclaw/openclaw/commit/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a)
- **Anthropic replay path disables mutating normalizers on immutable-thinking routes:** The replay pipeline computes `allowProviderOwnedThinkingReplay`, suppresses replay-time tool-call-id normalization on that path, and passes the immutable-thinking flag into tool-call sanitization before later repairs.
  - file: [src/agents/pi-embedded-runner/replay-history.ts:456](https://github.com/openclaw/openclaw/blob/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a/src/agents/pi-embedded-runner/replay-history.ts#L456)
  - sha: [c997a9f9788e](https://github.com/openclaw/openclaw/commit/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a)
- **Regression test covers preserved ordering for signed thinking blocks:** Targeted coverage asserts that `sanitizeSessionMessagesImages(..., { preserveSignatures: true })` keeps interleaved `thinking`, empty `text`, visible `text`, and `redacted_thinking` blocks in the original order.
  - file: [src/agents/pi-embedded-helpers.sanitize-session-messages-images.removes-empty-assistant-text-blocks-but-preserves.test.ts:310](https://github.com/openclaw/openclaw/blob/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a/src/agents/pi-embedded-helpers.sanitize-session-messages-images.removes-empty-assistant-text-blocks-but-preserves.test.ts#L310)
  - sha: [c997a9f9788e](https://github.com/openclaw/openclaw/commit/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a)
- **Regression test covers immutable Anthropic replay for signed-thinking history:** Targeted coverage asserts that Anthropic-compatible history sanitization drops the replay-unsafe signed-thinking assistant turn rather than mutating it, leaving only the retrying user turn.
  - file: [src/agents/pi-embedded-runner.sanitize-session-history.test.ts:983](https://github.com/openclaw/openclaw/blob/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a/src/agents/pi-embedded-runner.sanitize-session-history.test.ts#L983)
  - sha: [c997a9f9788e](https://github.com/openclaw/openclaw/commit/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a)

  - file: [CHANGELOG.md:79](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/CHANGELOG.md#L79)
