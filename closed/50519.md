## Summary

Keep #50519 open. Current main still matches the reported Windows CJK mojibake failure mode in the shared exec paths: runExec still requests UTF-8 strings from execFile, and runCommandWithTimeout still converts stdout/stderr chunks with default Buffer.toString(). Related #56462 was closed as a duplicate of #50519, and the active fix work remains in open PRs rather than shipped on main.

## Best Possible Solution

Keep #50519 open as the canonical bug until a tested codepage-aware decoder lands on main. The best implementation path is to consolidate the active PR work, share the existing node-host Windows codepage decoder with src/process/exec.ts and supervisor-based exec paths, preserve raw buffers until decoding, and add regression tests for GBK/Shift_JIS output including split multibyte chunks.
