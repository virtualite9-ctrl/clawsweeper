## Summary

Close as `clawhub`, not implemented-on-main. Current main still limits cron payloads to `systemEvent` and `agentTurn`, but the requested deterministic host-script scheduler does not need a new core payload kind: plugins can create/manage cron entries, intercept cron-driven runs before any model/backend execution, run local commands through the public process runtime, and expose plugin-owned CLI/RPC surfaces. The PR discussion also surfaced unresolved core-specific correctness risks, so the best cleanup is to move this capability to a ClawHub plugin rather than harden a broad core cron/UI/protocol change.

## Best Possible Solution

Do not merge this core PR as-is. Publish the deterministic script scheduler as a ClawHub plugin that owns its CLI/RPC/config surface, creates/manages regular cron jobs, intercepts cron-driven runs with `before_agent_reply`, and runs local commands through `openclaw/plugin-sdk/process-runtime`. Keep the broader conditional exec/pipeline product discussion in the canonical cron automation threads if maintainers later want a core API.
