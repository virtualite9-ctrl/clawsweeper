## Summary

Current main and the v2026.4.24 release already fix the reported relay path: heartbeat exec-event prompts embed the queued exec completion text instead of referring to hidden system messages, the runner passes filtered exec event bodies into that prompt builder, and handled runs consume inspected events. Existing regression tests cover the prompt text and relay behavior, including the absence of the old generic "system messages above" wording.

## Best Possible Solution

Keep the shipped v2026.4.24 implementation for this issue. Track any remaining heartbeat timeout, routing, or event-audience policy work in narrower related issues such as #65161, #68386, or the system-event audience discussion, instead of keeping this prompt-drop report open.
