## Summary

Current main and the latest release already remove the reported Carbon synchronous reconnect-exhausted abort crash path: Discord is pinned to Carbon 0.16.0, whose gateway disconnect path silent-closes intentional disconnects and emits reconnect exhaustion as an error event rather than throwing synchronously. OpenClaw’s current Discord supervisor covers that emitted error path with tests.

## Best Possible Solution

Close this PR as obsolete on current main, keep the shipped Carbon 0.16.0 plus Discord gateway supervisor behavior, and open a fresh issue only if a current Carbon/OpenClaw version can still produce a synchronous disconnect throw during health-monitor abort.
