## Summary

Current main already solves the linked Kimi/Moonshot `message_start` before `message_stop` failure through the shipped boundary-aware `anthropic-messages` transport. PR #57853's subscriber-state workaround is superseded and should not be merged as-is because its own review discussion identified remaining correctness and test gaps.

## Best Possible Solution

Close PR #57853 as implemented on main. Keep the shipped boundary-aware `anthropic-messages` transport as the canonical fix for #57654, and do not merge the subscriber-level workaround unless a new reproducible non-transport lifecycle bug proves that broader subscriber behavior still needs a separately tested fix.
