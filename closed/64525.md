## Summary

Current main already emits the `message_received` plugin hook before the reply resolver and before the active-session queue decision. Active in-flight messages still enter through `dispatchReplyFromConfig`, then `get-reply-run`/`runReplyAgent` decides whether to enqueue; by that point the inbound observer hook has already fired. The linked #64559 discussion identified the same desired boundary, but the shipped current behavior now satisfies the issue's central bug.

## Best Possible Solution

Close the issue as implemented on main. Keep the current hook emission before the reply resolver/active-queue decision; if maintainers want extra guardrail, #64559 can be reduced to or superseded by a narrow ordering regression test rather than treated as required product work.
