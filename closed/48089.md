## Summary

Current `main` already handles stale skills snapshots after a gateway restart, reuses first-turn snapshots instead of rebuilding them, and applies the same staleness logic on the CLI agent path. The shipped implementation is stronger than this PR's `Date.now()` baseline approach because it fixes restart invalidation without weakening monotonic version bumps after startup.

## Evidence

- **Restart invalidation is implemented on main:** `shouldRefreshSnapshotForVersion()` explicitly treats a cached snapshot from a previous process as stale when the current process has restarted with version `0` (`next === 0 ? cached > 0 : cached < next`). That covers the `#43735` restart case without changing the global baseline.
