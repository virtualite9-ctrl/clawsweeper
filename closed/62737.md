## Summary

Close as implemented on main: current OpenClaw no longer matches the reported bundled-LCM shape. The built-in default context engine is `legacy`, Lossless Claw is documented as an external plugin, and the bundled memory plugin is `memory-core` with its own manifest config schema under `plugins.entries.memory-core.config.*`; the old `plugins.entries.lcm.*` core path is not present on current main.

## Evidence

- **Built-in default is now `legacy`:** Current docs state that OpenClaw ships with a built-in `legacy` context engine and uses it by default; plugin engines are opt-in.
  - file: [docs/concepts/context-engine.md:14](https://github.com/openclaw/openclaw/blob/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a/docs/concepts/context-engine.md#L14)
  - sha: [c997a9f9788e](https://github.com/openclaw/openclaw/commit/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a)
- **Lossless Claw is documented as an external plugin:** Current docs tell users to install `@martian-engineering/lossless-claw` and enable it via `plugins.slots.contextEngine` and `plugins.entries.lossless-claw`, rather than treating it as a bundled core surface.
  - file: [docs/concepts/context-engine.md:28](https://github.com/openclaw/openclaw/blob/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a/docs/concepts/context-engine.md#L28)

- **Community docs match the plugin linked in the report:** The community plugin page lists Lossless Claw (LCM) as external plugin `@martian-engineering/lossless-claw`, matching the related plugin the reporter linked.
  - file: [docs/plugins/community.md:64](https://github.com/openclaw/openclaw/blob/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a/docs/plugins/community.md#L64)
  - sha: [c997a9f9788e](https://github.com/openclaw/openclaw/commit/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a)

This schema is already present in release `v2026.4.23`.
