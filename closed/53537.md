## Summary

Current `main` already includes duplicate-output guards in the embedded PI subscription path that specifically prevent repeated `text_end`/`message_end` assistant output, including cases with reasoning blocks. The targeted tests on `main` cover repeated `text_end`, repeated `message_end`, and repeated `message_end` with reasoning blocks, which is the failure shape described in issue #53537.

## Evidence

- **Late duplicate `text_end` events are treated as unsafe and state now resets at `message_start`:** The handler documents that resetting at `text_end` is unsafe because providers may deliver late or duplicate end events, and instead resets per-message state at assistant `message_start` to avoid re-triggering block replies.
