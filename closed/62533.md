## Summary

Current `main` already routes Ollama runs through the selected provider-specific `baseUrl` instead of defaulting to localhost, and the repo now has regression coverage for multi-provider/custom-provider-id cases matching this report.

## Evidence

- **Provider URL wins over localhost fallback:** `resolveOllamaBaseUrlForRun` now checks `providerBaseUrl` first, then `modelBaseUrl`, and only then falls back to `http://127.0.0.1:11434`.
  - file: [extensions/ollama/src/stream.ts:43](https://github.com/openclaw/openclaw/blob/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a/extensions/ollama/src/stream.ts#L43)
  - sha: [c997a9f9788e](https://github.com/openclaw/openclaw/commit/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a)
- **Custom Ollama provider ids resolve their own config:** `resolveConfiguredOllamaProviderConfig` looks up the exact provider id and then a normalized match, which covers non-default ids such as `ollama-lan`.
  - file: [extensions/ollama/src/stream.ts:58](https://github.com/openclaw/openclaw/blob/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a/extensions/ollama/src/stream.ts#L58)
  - sha: [c997a9f9788e](https://github.com/openclaw/openclaw/commit/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a)
- **Plugin passes provider baseUrl into stream creation:** The Ollama provider's `createStreamFn` passes the resolved provider config `baseUrl` into `createConfiguredOllamaStreamFn`, instead of relying on the model/default URL alone.
  - file: [extensions/ollama/index.ts:161](https://github.com/openclaw/openclaw/blob/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a/extensions/ollama/index.ts#L161)
  - sha: [c997a9f9788e](https://github.com/openclaw/openclaw/commit/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a)
- **Regression test for multiple Ollama providers:** Unit coverage verifies that a second Ollama provider id (`ollama2`) routes to its own `http://127.0.0.1:11435` base URL.
  - file: [extensions/ollama/index.test.ts:470](https://github.com/openclaw/openclaw/blob/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a/extensions/ollama/index.test.ts#L470)
  - sha: [c997a9f9788e](https://github.com/openclaw/openclaw/commit/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a)
- **Regression test for final request target:** Runtime coverage asserts both that provider `baseUrl` beats model `baseUrl` and that the resulting request is sent to `http://provider-host:11434/api/chat`.
  - file: [extensions/ollama/src/stream-runtime.test.ts:1112](https://github.com/openclaw/openclaw/blob/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a/extensions/ollama/src/stream-runtime.test.ts#L1112)
  - sha: [c997a9f9788e](https://github.com/openclaw/openclaw/commit/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a)
- **Fallback model resolution preserves provider transport:** The embedded runner now carries `providerConfig.baseUrl` into resolved fallback models, which is the upstream transport data this issue said was being lost.
  - file: [src/agents/pi-embedded-runner/model.ts:366](https://github.com/openclaw/openclaw/blob/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a/src/agents/pi-embedded-runner/model.ts#L366)
