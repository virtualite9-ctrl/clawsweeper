## Summary

Current main implements the requested async-completion delivery heuristic well enough: subagent announce dispatch now chooses and records a single primary path, active requester completions are steered back into the live session without also calling external delivery, dormant requesters use direct external delivery, and cron delivery accounting is tied to the real outbound adapter path.

## Evidence

- **Subagent dispatch chooses one primary path and short-circuits on success:** `runSubagentAnnounceDispatch` records delivery phases, uses queue-first for non-completion announces, direct-first for completion announces, returns immediately after a delivered primary path, and only attempts queue fallback after a failed completion direct path.
