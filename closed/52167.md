## Summary

Current `main` replaces the old per-module WhatsApp listener state with a shared global connection-controller registry, and the outbound send path now resolves listeners from that account-level registry rather than from any JID-specific map. The same codebase also still accepts both legacy and new-style `@g.us` group JIDs as valid outbound targets, so the reported old-style-group failure path is already addressed on `main`.
