## Summary

Current `main` already covers this report well enough: OpenClaw now exposes a configurable LLM idle timeout (`agents.defaults.llm.idleTimeoutSeconds`), documents using it for slow local models including `0` to disable, and also ships a later transport fix specifically for slow local LLM calls hitting the old 60-second body timeout.

## Evidence

- **LLM idle timeout is configurable and disable-able:** The runtime resolves `agents.defaults.llm.idleTimeoutSeconds` first, treats `0` as disabled, and otherwise falls back to run timeout or `agents.defaults.timeoutSeconds`.

- **Config schema exposes the setting:** The generated config schema publishes `agents.defaults.llm.idleTimeoutSeconds` with minimum `0` and help text saying `0` disables the timeout.
  - file: [src/config/schema.base.generated.ts:4558](https://github.com/openclaw/openclaw/blob/245451b6a9c529464e177afa65013adca0d63eb5/src/config/schema.base.generated.ts#L4558)
  - sha: [245451b6a9c5](https://github.com/openclaw/openclaw/commit/245451b6a9c529464e177afa65013adca0d63eb5)
