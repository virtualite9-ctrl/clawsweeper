## Summary

PR #67904 targets the speech-core/runtime-api.js parity-gate failure from #67887. Current main already ships the needed manifestless runtime-core facade behavior: the three runtime-core packages are packed without plugin manifests, their runtime-api surfaces are covered by tests, and activation checks allow those facades without normal plugin activation. The shipped implementation differs from the PR's final metadata-driven shape, but it solves the observable failure this PR was opened for.

## Best Possible Solution

Close this PR as implemented on main. Keep the shipped runtime-core facade resolution and activation behavior; if maintainers still want to replace the narrow hardcoded allowlist with package metadata, track that as a separate focused plugin-SDK architecture cleanup rather than keeping this bug-fix PR open.
