## Summary

Current main already solves the reported active-transcript mutation problem. `truncateAfterCompaction` now rotates to a compacted successor transcript instead of rewriting the live JSONL in place, and the rotated session id/file is propagated through manual, automatic, context-engine, overflow/timeout, gateway, and follow-up accounting paths.

## Best Possible Solution

Keep the successor-transcript rotation design on main. Do not reopen this broad lifecycle issue unless a new reproduction shows messages still disappear with the rotated transcript path; any remaining UI polish should be filed as a narrower WebChat/gateway notification issue.
