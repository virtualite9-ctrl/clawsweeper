## Summary

Current `main` already has layered defenses for the reported failure mode: generic outbound normalization suppresses reasoning-only and bare `NO_REPLY` payloads, Matrix transport suppresses residual reasoning-looking text, and Matrix draft-preview finalization redacts stale previews instead of leaving leaked draft text visible.

## Evidence

- **Generic outbound filtering drops reasoning payloads and silent sentinels before channel delivery:** `createOutboundPayloadPlanEntry` returns `null` for `isReasoning` payloads, and the outbound plan treats bare silent replies separately instead of sending them as normal visible text. The paired tests cover dropped reasoning payloads and exact `NO_REPLY` / JSON `NO_REPLY` suppression.
  - file: [src/infra/outbound/payloads.ts:130](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/infra/outbound/payloads.ts#L130)

  - file: [extensions/matrix/src/matrix/monitor/replies.ts:7](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/extensions/matrix/src/matrix/monitor/replies.ts#L7)
