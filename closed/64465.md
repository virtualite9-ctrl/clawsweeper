## Summary

Current `main` already routes `openai-codex` OAuth requests through the Codex backend instead of the removed `/backend-api/responses` alias, and the changelog records that exact `openai-codex/gpt-5.4` fix as shipped in `v2026.4.20`. Current defaults/docs also steer Codex OAuth users to `openai-codex/gpt-5.5`, which matches the supported path on modern OpenClaw.
