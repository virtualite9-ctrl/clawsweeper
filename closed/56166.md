## Summary

Current main already addresses the user-facing subagent restart failure path through persisted session lifecycle state, recoverable wait handling, and abortedLastRun orphan recovery. The PR's transcript-tail scan is no longer the best path, and the provided review discussion shows that scan still had correctness and cleanup risks.

## Best Possible Solution

Close this PR and keep the current mainline recovery design: persisted lifecycle status settles completed child sessions, recoverable wait failures schedule orphan recovery, and abortedLastRun sessions resume through the dedicated recovery path. Any remaining transcript-replay edge case should be filed as a new focused bug with a concrete reproduction against current main.
