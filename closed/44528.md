## Summary

Current `main` now strips the reported internal sub-agent completion/runtime-context envelope from Control UI text extraction and history sanitization. The latest release `v2026.4.23` still appeared affected, but the code on `main` contains the fix and regression coverage.

## Evidence

- **Control UI text extraction now strips internal runtime context:** `processMessageText` runs `stripInternalRuntimeContext` before assistant/user text is rendered in the Control UI.
  - file: [ui/src/ui/chat/message-extract.ts:11](https://github.com/openclaw/openclaw/blob/712f7b218c092c9d0fae1f583f7c06e617920115/ui/src/ui/chat/message-extract.ts#L11)
  - sha: [712f7b218c09](https://github.com/openclaw/openclaw/commit/712f7b218c092c9d0fae1f583f7c06e617920115)
- **Stripper recognizes the exact legacy envelope reported in the issue:** The runtime-context sanitizer explicitly matches the legacy `OpenClaw runtime context (internal):` header and `[Internal task completion event]` marker, including the untrusted child result/action block shape.
  - file: [src/agents/internal-runtime-context.ts:7](https://github.com/openclaw/openclaw/blob/712f7b218c092c9d0fae1f583f7c06e617920115/src/agents/internal-runtime-context.ts#L7)
  - sha: [712f7b218c09](https://github.com/openclaw/openclaw/commit/712f7b218c092c9d0fae1f583f7c06e617920115)
- **Regression test covers legacy internal task completion event removal:** Unit test asserts that a message containing the exact internal runtime header plus `[Internal task completion event]` is reduced to only the visible surrounding text.
  - file: [src/agents/internal-runtime-context.test.ts:19](https://github.com/openclaw/openclaw/blob/712f7b218c092c9d0fae1f583f7c06e617920115/src/agents/internal-runtime-context.test.ts#L19)
  - sha: [712f7b218c09](https://github.com/openclaw/openclaw/commit/712f7b218c092c9d0fae1f583f7c06e617920115)
