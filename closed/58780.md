## Summary

Keep PR #58780 open. Current main still uses a static `chokidar` import in `scripts/watch-node.mjs`, so an invalid/truncated dependency `package.json` can still fail before `runWatchMain` has a chance to print recovery guidance. The PR addresses a real scripts/developer-experience gap, but review comments identify fixable issues in the proposed implementation, including Windows path detection and double logging for non-`ERR_INVALID_PACKAGE_CONFIG` failures.

## Best Possible Solution

Keep the PR open and ask for a small update: load `chokidar` dynamically so startup import failures can be caught, normalize path separators when detecting `node_modules`, print the friendly recovery guidance only for `ERR_INVALID_PACKAGE_CONFIG`, let the existing top-level catch handle other failures once, and keep the focused regression test for the invalid package config path.
