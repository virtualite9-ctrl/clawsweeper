## Summary

Keep open. Current main still forwards OpenShell exec text through the shared SSH `/bin/sh -c` wrapper without malformed-shell or generated-placeholder preflight, and generic repeated same-tool/same-args loops remain warning-level until the later global no-progress breaker. Adjacent loop-detection reports and PRs cover only part of the generic-repeat behavior and do not supersede the OpenShell command hardening request.

## Best Possible Solution

Keep this issue open. The best maintainer path is a focused runtime hardening change: add shared exec/OpenShell preflight before SSH command construction for obvious generated placeholders and malformed shell structure, update loop detection so repeated identical failed or no-progress generic calls can become a clear critical block at a configured threshold, add regression coverage for OpenShell/shared SSH command construction and before-tool-call blocking, and separately clean up or lint bundled skill/help examples that look copy-executable while containing placeholders.
