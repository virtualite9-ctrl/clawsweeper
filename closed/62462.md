## Summary

Current `main` no longer silently succeeds when `read` is called without a canonical `path`. The read wrapper validates `path` before execution, the tool adapter converts validation failures into structured error results, regression tests cover the contract, and the same relevant code is present in release `v2026.4.23`.

## Evidence

- **Read tool rejects missing path before execution:** `createOpenClawReadTool()` now calls `assertRequiredParams(record, REQUIRED_PARAM_GROUPS.read, base.name)` before any read happens, so a pathless `read` call fails immediately instead of returning empty content.
