## Summary

Current main already fixes the reported stale bundled-plugin warnings: plugin validation builds the registry before checking `plugins.entries`/`plugins.allow`, the Google plugin owns `google-gemini-cli` auto-enable metadata, and bundled plugin directory resolution no longer requires users to set `OPENCLAW_BUNDLED_PLUGINS_DIR` for packaged installs.

## Best Possible Solution

Keep the current registry-backed validator and bundled-plugin auto-enable behavior on main. No core code change is needed for this issue; the fix should ship with the next release that contains `1ee5654220c37d78d0e35f860b97d68b551e1f7b`.
