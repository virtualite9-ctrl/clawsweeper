## Summary

Keep open. Current main still has the macOS doctor repair footgun: LaunchAgent EnvironmentVariables can be parsed and preserved when callers pass the existing service environment, but the doctor service-config repair path rebuilds/stages/installs the service without passing `command.environment` into `buildGatewayInstallPlan`. The related systemd work explicitly scoped macOS launchd out.

## Best Possible Solution

Keep this issue open and fix the doctor repair path by passing the parsed existing service environment into `buildGatewayInstallPlan` before staging/installing the repaired LaunchAgent. Add a focused macOS doctor repair regression test that starts with custom plist-only `EnvironmentVariables`, triggers a service-config repair such as entrypoint drift, and asserts safe unknown keys survive while managed OpenClaw keys are updated according to the existing managed-key contract.
