## Summary

Issue #69933 is a real current Windows bug, but the same remaining work is now tracked by open PR #72153. Current main still builds the Windows opener as `Start-Process -LiteralPath`, the upstream PowerShell docs expose `-FilePath` for `Start-Process`, and #72153 is an exact two-file fix plus regression-test update for this issue.

## Best Possible Solution

Close #69933 as superseded by the focused fix PR #72153, and keep review/merge work on that PR. The correct code path is to replace `Start-Process -LiteralPath` with `Start-Process -FilePath` while preserving the existing single-quoted path escaping and updating the Windows command regression test.
