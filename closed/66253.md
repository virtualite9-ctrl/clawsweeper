## Summary

Current main and the v2026.4.24 tag already address the reported gateway image-drop behavior. The gateway now loads the config-backed model catalog, preserves model `input` metadata, resolves image support from that catalog, and `parseMessageWithAttachments` no longer silently drops images for text-only paths.

## Best Possible Solution

Close #66253 as implemented and keep the shipped config-backed gateway model catalog plus no-drop attachment handling. Leave narrower provider-specific reports, such as MiniMax/image-tool catalog bugs, to their own issues if they still reproduce.
