## Summary

The requested `paths:`-based conditional skill activation is a useful optional workflow, but it fits OpenClaw's plugin/ClawHub path rather than a new core SKILL.md schema. Current main already exposes plugin skills, bundle skill/hook mapping, and runtime hooks that can inspect tool calls and inject prompt context.
