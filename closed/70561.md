## Summary

Current main already prevents Discord partial preview streaming from delivering ordinary tool-result text to users. Discord partial mode now uses bounded preview progress labels, suppresses default tool-result chat messages while that preview is active, and the shared dispatcher test covers suppression of a raw tool-result text payload.

## Best Possible Solution

Keep the shipped Discord preview-streaming behavior. If a future leak is observed, it should be filed as a new focused bug with the exact tool/event payload, Discord streaming config, verbose level, and whether `streaming.preview.toolProgress` was changed from its default.

## Evidence

- **Issue context reviewed:** #70561 reports Discord `streaming.mode="partial"` showing tool-call outputs to users; the follow-up comment frames the bug as failure to separate AI-visible tool output from user-visible Discord rendering.
- **Discord partial preview suppresses default tool messages:** Discord creates a draft stream for non-off preview modes, enables preview tool progress by default, and passes `suppressDefaultToolProgressMessages` to the reply dispatcher when preview progress is active.
