## Summary

Current main already fixes the linked #51016 status false-negative by resolving gateway probe auth through the async SecretInput path and by reusing an existing cached operator device identity for authenticated local probes. The PR's branch is obsolete because it forces device identity for unauthenticated loopback probes, which current main and docs intentionally avoid to keep diagnostic probes non-mutating.

## Best Possible Solution

Close this PR as implemented/superseded by current main. Keep the shipped safer implementation: resolve SecretRefs for probe auth when possible, reuse cached device auth for authenticated local probes, and keep unauthenticated diagnostic probes non-mutating. Any new case where a supported SecretRef provider cannot resolve in the status command path should be reported as a fresh, provider-specific bug.
