## Summary

Current `main` already addresses the reported shared CI invalidation path: the gateway watch regression harness now refreshes the build stamp for restored `--skip-build` artifacts, and `run-node` now distinguishes real watched-source dirtiness from separate runtime-postbuild dirtiness so unrelated artifact/input changes no longer collapse into `dirty_watched_tree`.

## Evidence

- **Gateway watch harness refreshes restored artifact stamps before preflight:** In the `--skip-build` CI path, the harness now writes a fresh build stamp before preflight and refreshes again when the only mismatch is `config_newer`, matching the issue’s suspected restored-artifact/CI invalidation path.
