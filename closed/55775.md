## Summary

PR #55775 should close as superseded by the later open PR #57199. Current main still lacks `openclaw system event --agent-id`, but #57199 is the broader canonical thread because it covers the same agent-targeted wake path plus `--session-key` routing and already has the relevant follow-up review discussion.

## Best Possible Solution

Close #55775 as superseded by #57199. Maintainers should continue the implementation on #57199 or an equivalent replacement, including agent/session target validation, deriving the agent from `sessionKey` before enqueueing, and updating schema, generated models, CLI tests, and docs together.
