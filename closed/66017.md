## Summary

Current `main` already implements support for image-capable custom providers declared under `models.providers`, including image-tool selection and media-understanding registration. The reported `Unknown model: uih/glm-4.5v` path matches code that was added after the reported `2026.4.12` version, but that fix does not appear to be in a tagged release yet.

## Evidence

- **Media-understanding now auto-registers config providers with image-capable models:** `buildMediaUnderstandingRegistry()` adds non-bundled `models.providers.<id>` entries when their catalog declares `input: ["image"]`, so custom providers become valid image-description providers instead of being absent from the registry.
  - file: [src/media-understanding/provider-registry.ts:41](https://github.com/openclaw/openclaw/blob/db8f7478b43550d2949c04b2a9c9ca7fd474dad3/src/media-understanding/provider-registry.ts#L41)

  - sha: [db8f7478b435](https://github.com/openclaw/openclaw/commit/db8f7478b43550d2949c04b2a9c9ca7fd474dad3)
- **Image-capable config providers are detected from provider config:** `resolveImageCapableConfigProviderIds()` scans `cfg.models.providers` and returns provider ids only when a configured model includes `image` input support.
  - file: [src/media-understanding/config-provider-models.ts:17](https://github.com/openclaw/openclaw/blob/db8f7478b43550d2949c04b2a9c9ca7fd474dad3/src/media-understanding/config-provider-models.ts#L17)

  - file: [src/agents/tools/image-tool.test.ts:709](https://github.com/openclaw/openclaw/blob/db8f7478b43550d2949c04b2a9c9ca7fd474dad3/src/agents/tools/image-tool.test.ts#L709)
