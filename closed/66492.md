## Summary

Current main already implements the exec-event heartbeat fix: async exec completion output is embedded directly into the heartbeat prompt, empty/no-output completions are converted to a silent HEARTBEAT_OK path, and tests cover both user-relay and internal-only delivery behavior. The fix is present in release v2026.4.24.

## Best Possible Solution

Close as implemented and keep the current generic exec-event heartbeat path: command completion details are embedded in the heartbeat prompt, empty/no-output completions are silent, and delivery remains controlled by the existing heartbeat visibility/target logic.
