## Summary

Close as `clawhub`: current `main` intentionally keeps `openclaw tasks flow` as an inspection/cancel surface while exposing TaskFlow authoring through public plugin/skill seams. The existing bundled Webhooks plugin and public plugin runtime already cover TaskFlow creation/orchestration without adding a new core CLI/config DSL.

## Evidence

  - file: [docs/automation/taskflow.md:93](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/docs/automation/taskflow.md#L93)
