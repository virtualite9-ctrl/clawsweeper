## Summary

Close #53370 as superseded by open PR #68397. Current main still hard-rejects `streamTo` and `resumeSessionId` for `runtime="subagent"`, so this is not implemented-on-main; #68397 is the focused canonical fix and explicitly lists #53370 while updating the same tool and tests to silently drop ACP-only fields for subagent spawns. The issue comment about OpenAI Completions adding `streamTo` is useful context and is covered by #68397's strict-provider rationale.

## Best Possible Solution

Close #53370 as duplicate/superseded and keep implementation review concentrated on PR #68397 or an equivalent maintainer patch. The desired fix should silently ignore `streamTo` and `resumeSessionId` only for native `runtime="subagent"`, preserve ACP pass-through, update the focused unit tests, and align ACP/subagent docs if the behavior lands.
