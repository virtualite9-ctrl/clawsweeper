## Summary

Current `main` has partial progress on this report: service entrypoint resolution now prefers `dist/index.js`, and the gateway service env no longer injects `OPENCLAW_GATEWAY_TOKEN` by default. But the macOS LaunchAgent refresh/install path still tears the service down before a successful re-bootstrap is confirmed, and the installer still performs `gateway install --force` followed by `gateway restart` without rollback. That leaves the core upgrade-service-lifecycle bug unresolved enough that this issue should stay open.
