## Summary

Close as duplicate/superseded. #55101 reports the same Windows `pnpm ui:build` path-with-spaces failure already tracked in open #45275, with additional exact failure logs in that thread. Current `main` still routes `ui:build` through `scripts/ui.js` using the raw resolved `.cmd` path with `shell: true`, while open PRs #55263 and #52291 already carry proposed fixes and regression coverage.
