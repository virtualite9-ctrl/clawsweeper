## Summary

Current `main` already contains the cron Telegram delivery and main-session wake routing changes that address the reported `delivered=true` false positive and the `--wake now`/system-event hang path, and those changes were shipped in `v2026.4.23`.

## Evidence

- **Direct cron delivery now gates `delivered=true` on real outbound success:** The isolated cron delivery path calls `deliverOutboundPayloads(...)` for the resolved target and only sets `delivered = deliveryResults.length > 0 && !hadPartialFailure`, which closes the false-positive path where a run could claim delivery without a channel send.
