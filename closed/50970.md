## Summary

Close as superseded by #65772. #50970 targets a real `nodes list` / `nodes status` bug, but its implementation makes plain `nodes list` hard-depend on `node.list`, matching the existing review concerns about pairing-scope callers and older gateways. The newer open PR #65772 tracks the same remaining work and explicitly adds the fallback and JSON compatibility coverage this PR is missing.

## Best Possible Solution

Close #50970 as superseded and continue the maintainer review on #65772. The best fix is to make `nodes list` prefer the effective `node.list` paired-node view, keep pending data from `node.pair.list`, preserve the existing JSON pairing metadata contract, and retain a narrow fallback for pairing-only callers and older gateways.
