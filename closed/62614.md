## Summary

Current OpenClaw already bounds the retry path described here: timeout-triggered compaction retries are capped, overflow recovery retries are capped, rate-limit profile rotation escalates into model fallback, long SDK Retry-After waits are bypassed so failover can run, and regression tests cover the retry-cap behavior across profile rotation. The same logic is present in the shipped v2026.4.23 release.

## Evidence
