## Summary

PR #49093 is superseded by the current main memory-host SDK proxy implementation tracked in #71506. The old `src/memory/remote-http.ts` target no longer exists; current main routes shared memory remote HTTP through `src/memory-host-sdk/host/remote-http.ts` and its package mirror, selecting trusted env-proxy mode only when Undici will actually proxy the target URL. Tests cover the memory helper, protocol-specific proxy selection, `ALL_PROXY`/`NO_PROXY` fallback to DNS pinning, and no-proxy strict behavior. The changelog records this as an unreleased 2026.4.25 fix for #71506, so there is no shipped release tag yet.

## Best Possible Solution

Close this stale PR as superseded by the shipped-on-main #71506 implementation. Keep the current narrower `shouldUseEnvHttpProxyForUrl` approach rather than reviving the older PR against the removed `src/memory/remote-http.ts` surface.
