## Summary

Close #49092 as ClawHub/plugin work rather than new OpenClaw core surface. The PR proposes useful opt-in remote sync for workspace bootstrap files, but current project direction keeps optional capability out of core when plugin APIs can express it. Current main already exposes the needed plugin seams: CLI registration, plugin-owned HTTP routes/webhook auth, gateway_start lifecycle with workspaceDir, and plugin config. No missing core extension API is apparent from the current code/docs.

## Best Possible Solution

Move the remote bootstrap-file sync capability to a ClawHub plugin using current plugin APIs: plugin config for remote URL/token, plugin CLI subcommands for pull/push, a plugin-owned webhook route, and a gateway_start hook that uses ctx.workspaceDir. If implementation discovers a missing generic extension seam, open a narrow plugin API issue or PR for that seam instead of adding a one-off core workspaceSync feature.
