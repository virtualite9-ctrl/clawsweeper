## Summary

Current `main` already contains the core fix for #66601, and the fix shipped in `v2026.4.23`: OpenClaw no longer rejects third-party context engines when `info.id` differs from the registered slot id, and there is a dedicated regression test for the exact `lossless-claw` shape described in this issue.

## Evidence

- **Current registry logic no longer enforces `info.id === registered slot id`:** `describeResolvedContextEngineContractError()` now treats `info.id` as engine-owned metadata and only requires it to be a non-empty string; the comment explicitly says the registered id and engine `info.id` are allowed to differ.
