## Summary

Issue #46287 should close as superseded by open issue #48345. Current main still performs startup missed-job catch-up and has no skipMissed/skipMissedOnRestart field, while #48345 now tracks the same remaining scheduler change as a broader per-job/global skipMissedRuns option.
