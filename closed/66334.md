## Summary

#66334 is a duplicate of the already-open canonical session model override report #51251. Current main still treats explicit `/model` and `session_status(model=...)` changes as user-scoped session overrides that outrank the configured primary model across turns; the remaining product decision should stay centralized in #51251, with #55063 covering the related `/new` and `/reset` reset-contract variant.

## Best Possible Solution

Close #66334 as a duplicate and keep the remaining behavior/product decision centralized in #51251. If maintainers decide user-set session model overrides should clear on restart or lose precedence to `model.primary`, that change should update model-selection/session-store behavior and tests under the canonical issue, with #55063 handling the `/new` and `/reset` contract.
