## Summary

Current main removes the recursive logging/config dependency that caused #56290. `readLoggingConfig()` now reads only the active config file via `resolveConfigPath()`, `fs`, and JSON5; the logger path no longer falls back to importing/calling `loadConfig()`, and console config fallback is non-mutating by default with a re-entrancy guard for test injection. The related #56715 report was already closed as implemented after review, and #56739’s narrower guard approach is superseded by the broader mainline change.

## Best Possible Solution

Close #56290 as implemented on current main, keep the direct `readLoggingConfig()` file-read approach, and handle the still-open #56739 PR separately as superseded by the broader logging decoupling already on main.
