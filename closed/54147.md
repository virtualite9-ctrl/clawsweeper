## Summary

Current `main` does not reproduce the reported SSRF gap. Although `src/media-understanding/attachments.cache.ts:158` does not pass an explicit `ssrfPolicy`, `fetchRemoteMedia` still routes that request through the guarded fetch path, and the default `undefined` policy blocks private/internal/special-use hosts and resolved IPs. Current committed tests also cover the exact attachment-cache and `fetchRemoteMedia` private-IP cases.

## Evidence

- **Attachment cache call site still uses guarded fetch helper:** `MediaAttachmentCache.getBuffer()` calls `fetchRemoteMedia({ url, fetchImpl, maxBytes })` at the reported site.

  - file: [src/media/fetch.ts:131](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/media/fetch.ts#L131)
