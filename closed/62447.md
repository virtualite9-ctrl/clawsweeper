## Summary

The reported Discord `runtimeProfile` TS2353 no longer matches current `main`. The current file explicitly widens the `satisfies` type for `defaultOptions`, and the issue discussion itself converged on this being an older checkout-state report rather than a live `main` regression in `extensions/discord/src/proxy-request-client.ts`.

## Evidence

- **Current main widens the `satisfies` type:** `defaultOptions` includes `runtimeProfile` and `scheduler`, but the object is now checked against `Omit<ProxyRequestClientOptions, "fetch"> & { runtimeProfile: string; scheduler: object; }`, which directly addresses the reported excess-property error.
  - file: [extensions/discord/src/proxy-request-client.ts:37](https://github.com/openclaw/openclaw/blob/ecac696643c604c6d854341bfe6424bfa31423e9/extensions/discord/src/proxy-request-client.ts#L37)
  - command: `nl -ba extensions/discord/src/proxy-request-client.ts | sed -n '1,70p'`
  - sha: [ecac696643c6](https://github.com/openclaw/openclaw/commit/ecac696643c604c6d854341bfe6424bfa31423e9)
- **Current main documents the local compatibility seam:** The same file documents that Carbon 0.14 removed the custom fetch seam and keeps a local proxy-aware compatibility client, showing this code path has already been adapted on main rather than left in the reported broken shape.
  - file: [extensions/discord/src/proxy-request-client.ts:132](https://github.com/openclaw/openclaw/blob/ecac696643c604c6d854341bfe6424bfa31423e9/extensions/discord/src/proxy-request-client.ts#L132)

  - sha: [ecac696643c6](https://github.com/openclaw/openclaw/commit/ecac696643c604c6d854341bfe6424bfa31423e9)
- **Issue discussion already narrowed this to an older checkout state:** Commenter `martingarramon` reported that on then-current upstream/main the Discord file did not produce TS2353, because the `satisfies` clause had already been widened; they also noted any contemporaneous `build:plugin-sdk:dts` failures were in different files.
- **Reporter confirmed they could not reproduce the original Discord-state checkout anymore:** The reporter later said their remaining temp clone was already after the widening change and concluded the original TS2353 likely came from an earlier checkout, while current-main failures were for a different reason.
