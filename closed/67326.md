## Summary

Current main already solves PR #67326's linked image-runtime header bug. The image runtime now resolves models through the shared resolver, which preserves configured provider/model headers on `model.headers`, and OpenAI-compatible transports send those headers. The linked issue #67056 is already closed, and this PR's current diff is no longer the right implementation path because it adds an image-only auth lookup and would regress caller-supplied `authStore` handling.

## Best Possible Solution

Close this PR as implemented on current main. Keep the shared `resolveModelWithRegistry(...)` and `model.headers` propagation path; if maintainers want narrower coverage, add a focused image-runtime regression test on main rather than merging this PR's image-only auth lookup.
