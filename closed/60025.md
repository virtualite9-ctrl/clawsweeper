## Summary

Keep open. Current `main` still has the teardown hole described in PR #60025: pending tool media is consumed during `handleAgentEnd` even when there is no `onBlockReply` consumer, and the later runner payload merge is not wired to any populated attempt result fields.

## Evidence

- **Teardown still consumes orphaned tool media unconditionally:** `handleAgentEnd()` still calls `consumePendingToolMediaReply(ctx.state)` and immediately forwards the result to `ctx.emitBlockReply(...)`. That consumes queued TTS/media state before any check that a block-reply callback exists.
  - file: [src/agents/pi-embedded-subscribe.handlers.lifecycle.ts:169](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/src/agents/pi-embedded-subscribe.handlers.lifecycle.ts#L169)
  - command: `nl -ba src/agents/pi-embedded-subscribe.handlers.lifecycle.ts | sed -n '150,210p'`
  - sha: [a155b84cdade](https://github.com/openclaw/openclaw/commit/a155b84cdadef068de50274318952032cd867788)
- **No block-reply consumer means the consumed media is dropped:** In `subscribeEmbeddedPiSession()`, `emitBlockReplySafely()` returns early when `params.onBlockReply` is absent. Combined with the lifecycle code above, command-style runs with no block-reply callback still lose consumed pending media.
  - file: [src/agents/pi-embedded-subscribe.ts:160](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/src/agents/pi-embedded-subscribe.ts#L160)
  - command: `nl -ba src/agents/pi-embedded-subscribe.ts | sed -n '160,205p'`
  - sha: [a155b84cdade](https://github.com/openclaw/openclaw/commit/a155b84cdadef068de50274318952032cd867788)
- **Runner merge helper is present but not fed by the attempt result:** `run.ts` still tries to merge `attempt.toolMediaUrls` / `attempt.toolAudioAsVoice` into final payloads, but `runEmbeddedAttempt()`'s returned object does not populate those fields, so this path cannot preserve the orphaned TTS media on current `main`.
  - file: [src/agents/pi-embedded-runner/run.ts:1805](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/src/agents/pi-embedded-runner/run.ts#L1805)
  - command: `nl -ba src/agents/pi-embedded-runner/run.ts | sed -n '1778,1822p'`
  - sha: [a155b84cdade](https://github.com/openclaw/openclaw/commit/a155b84cdadef068de50274318952032cd867788)
- **Attempt result returned on main still omits tool-media fields:** The current `runEmbeddedAttempt()` return object includes messaging summaries and `successfulCronAdds`, but not `toolMediaUrls` or `toolAudioAsVoice`, despite those optional fields existing on `EmbeddedRunAttemptResult`.
  - file: [src/agents/pi-embedded-runner/run/attempt.ts:2881](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/src/agents/pi-embedded-runner/run/attempt.ts#L2881)
