## Summary

PR #50921 adds a new optional Eternal AI model-provider plugin. Current OpenClaw scope and docs explicitly route optional provider integrations to external plugins/ClawHub, and current main already has the provider-plugin, auth, catalog, onboarding, and runtime-hook seams needed to implement this outside core. The PR discussion is useful: reviewers found the implementation mostly follows provider patterns, the author linked Eternal docs for endpoint/model choices, and the remaining chain-routing concern can be handled with the existing provider hook surface rather than a new core API.

## Best Possible Solution

Do not merge Eternal AI as a bundled core provider. Move the PR code into an external OpenClaw provider plugin published through ClawHub/npm using the existing provider SDK; open a focused SDK issue or PR only if that external implementation exposes a missing generic provider seam.
