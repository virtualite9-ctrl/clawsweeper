## Summary

Keep open. Current main still has the reported failure path: Docker setup runs onboarding inside the non-root container, the skill dependency installer still invokes `npm install -g` when npm is selected, and the runtime Dockerfile still switches to `USER node` without configuring npm's global prefix to a user-writable location. The related fix PR #60052 was closed unmerged, and its proposed Dockerfile prefix/PATH change is not present on current main.

## Best Possible Solution

Keep the issue open and fix the Docker runtime image. A robust fix should configure a user-writable npm global prefix before `USER node` (for example create/chown `/home/node/.npm-global`, set `NPM_CONFIG_PREFIX`/`npm_config_prefix`, and prepend its `bin` directory to `PATH`), then add Docker-focused regression coverage or smoke coverage for onboarding/`skills.install` choosing npm inside the container. PR #60052 is useful prior art but should be reimplemented or reopened with tests and current Dockerfile context.
