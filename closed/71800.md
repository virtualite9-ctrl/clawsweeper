## Summary

Current main already fixes the reported logger-startup config race. The logger now reads the active OpenClaw config file directly and applies `logging.maxFileBytes` from that config, and the capped file logger rotates instead of suppressing writes at the old default. The related PR #71917's env-var workaround is therefore superseded for this issue.

## Best Possible Solution

Close #71800 as implemented on main. Keep the direct config-path read plus rotation behavior; treat #71917's `OPENCLAW_LOG_MAX_FILE_BYTES` env override as optional/superseded unless maintainers still want that separate override surface.
