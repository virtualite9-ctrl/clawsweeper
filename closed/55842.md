## Summary

Current main and the latest shipped v2026.4.24 package no longer match the v2026.3.24 startup failure mode. The highlighter/shiki-style dependency chain is not in the root runtime package, Control UI is resolved as static dist assets by the gateway, optional heavy plugin runtime dependencies are kept out of the core bundle, and CI now has a startup RSS gate covering the gateway-status path. The related packaging cluster #54790 is already closed, and this issue has no protected labels or maintainer author signal.

## Best Possible Solution

Close this issue as fixed by the current packaged startup/runtime-dependency boundaries and keep the startup memory gate in CI. If a user can still reproduce RegExpCompiler OOM on v2026.4.24 or newer, track that as a fresh report with the current package version, service logs, plugin enablement state, and the specific generated dist chunk that is being compiled.
