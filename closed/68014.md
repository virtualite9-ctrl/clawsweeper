## Summary

Current main already preserves optimistic Control UI chat messages across stale `chat.history` reloads with a smaller tail-preservation implementation, regression coverage, docs, and release evidence. The PR's pending-map approach is obsolete and still has unresolved review concerns around stale pending entries, content-only dedupe, and session leakage.

## Best Possible Solution

Close this PR and keep the current main implementation. It solves the observable Control UI history-reload race with a narrower tail-preservation helper, documentation, and regression tests, while avoiding the PR's unresolved pending-map failure modes. Any future edge case should be filed as a focused follow-up against the shipped helper.
