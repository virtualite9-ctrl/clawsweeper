## Summary

Current main does not support the issue premise as a bug: OpenClaw intentionally treats `grok-4-fast-reasoning` and `grok-4-1-fast-reasoning` as legacy aliases for the canonical bundled xAI ids `grok-4-fast` and `grok-4-1-fast`. The linked PR #56539 was already closed unmerged after maintainer review with that same conclusion.

## Best Possible Solution

Keep the current documented alias behavior on main. Any future change to make `grok-4-1-fast-reasoning` canonical should update the bundled xAI catalog, provider docs, normalization tests, and status expectations together.
