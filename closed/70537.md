## Summary

Current main and the latest release no longer match the reported malformed gateway serialization path. Default Anthropic runs route through OpenClaw's boundary-aware Anthropic transport, tool-call JSON deltas are accumulated and parsed into structured arguments, replayed malformed arguments are coerced to object-shaped payloads, and Gateway/OpenResponses response bodies are serialized with JSON.stringify rather than manual array joining.

## Best Possible Solution

Keep the shipped boundary-aware Anthropic transport and JSON.stringify-based Gateway/OpenResponses serialization. If a similar regression resurfaces, it should be reopened as a new issue with raw Anthropic stream/gateway logs showing the failing partial_json chunks, model config, and channel/plugin details.
