## Summary

Close #46475 as superseded by the already-open canonical bug/fix thread for #46369, especially PR #46506. Current main still has the reported gap, so this is not implemented-on-main: `cron.remove` only removes the job record and emits the removal event, while the existing reaper intentionally preserves the base cron session and only prunes expired `:run:` sessions. #46475 contains useful direction, but it has no regression tests and has unresolved review findings around session-store fallback, deduplication, and detached cleanup. The same user-visible bug is still tracked by #46369 and the newer #46506, which adds targeted cleanup tests and carries the follow-up discussion.

## Best Possible Solution

Close #46475 as the duplicate/superseded PR and consolidate the remaining #46369 work in #46506 or a maintainer-authored patch. The final fix should clean both canonical and run cron sessions, delete the per-job run log, select the correct per-agent session store with fallback behavior, avoid detached-cleanup races where feasible, and include regression tests for `cron.remove`.
