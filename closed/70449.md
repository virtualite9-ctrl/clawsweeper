## Summary

Current main already fixes the Gemini tool-call replay failure for both direct Google and OpenAI-compatible Google routes. The PR's exact nested `functionCall.thought_signature` direct-Gemini shape is not needed for the shipped implementation and does not match the current `@google/genai@1.50.1` FunctionCall type, while current main preserves the signature at the Gemini Part level and in compat `extra_content.google.thought_signature` where appropriate.

## Best Possible Solution

Close this PR as already implemented on main. Keep the shipped direct-Google Part-level `thoughtSignature` replay and the OpenAI-compatible `extra_content.google.thought_signature` replay path; reopen only if a fresh live repro against current main shows Google rejecting the current direct Gemini Part-level signature shape.
