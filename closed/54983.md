## Summary

Keep open. Current `main` already supports per-model `maxTokens` for manual vLLM model config, but it does not implement the issue's requested vLLM-specific reasoning/thinking toggle via `chat_template_kwargs.enable_thinking`. The current transport still treats vLLM as a generic OpenAI-compatible completions backend and only sends generic reasoning fields.
