## Summary

Current main already handles the routed-session leakage described by the PR: agent-created cron jobs inherit the active session agentId when the job omits agentId, cron storage persists it, and isolated cron execution uses that agentId for the cron session key and workspace. The draft PR's CWD-based CLI inference is no longer needed for the routed agent/session context bug it describes.

## Best Possible Solution

Close this draft PR as already implemented for the routed agent/session context bug. Keep the current session-derived agentId behavior for agent-created cron jobs and the explicit --agent CLI path; if maintainers want shell CWD-based CLI inference later, track it as a separate CLI UX request.
