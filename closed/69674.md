## Summary

Current main implements the maintainer-suggested fix for this report: gateway liveness/readiness HTTP probes are now routed before hooks, plugin HTTP, canvas, managed media, and Control UI stages, with a regression test proving probes return even when an earlier non-probe stage would hang.

## Best Possible Solution

Close #69674 as implemented on main. Keep the probe-first HTTP routing and regression test; handle any remaining Windows-specific pre-request hang with a fresh current-release repro or targeted review of #69701.
