## Summary

Current `main` now forwards slash-command text in ACP-bound sessions to the ACP runtime instead of trapping it in OpenClaw's local slash-command/skill handling, so Claude Code commands like `/add-dir` are implemented on `main`.

## Evidence

- **ACP reply-dispatch now claims explicit slash-command candidates:** `tryDispatchAcpReplyHook` treats any populated `CommandBody` or leading `/`/`!` `BodyForCommands` as an explicit command candidate and routes the turn through `runtime.tryDispatchAcpReply(...)` for ACP sessions.
