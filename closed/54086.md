## Summary

Current `main` no longer matches the PR’s stated regression path: subcommands are now registered through lazy placeholders, so unrelated invocations and root help do not load the `pairing` subcli or its plugin setup during initial CLI registration.

## Evidence

- **Pairing subcli is behind lazy registration:** The `pairing` entry is registered through `registerSubCliWithPluginCommands(...)` inside the subcli command-group table, so it is only reached when that command group is actually loaded.
  - file: [src/cli/program/register.subclis-core.ts:165](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/cli/program/register.subclis-core.ts#L165)
  - command: `nl -ba src/cli/program/register.subclis-core.ts | sed -n '150,190p'`
- **Lazy placeholders defer the real registrar:** When a primary subcommand is present, `registerCommandGroups(...)` installs a placeholder instead of calling the registrar immediately.
  - file: [src/cli/program/register-command-groups.ts:79](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/cli/program/register-command-groups.ts#L79)
  - command: `nl -ba src/cli/program/register-command-groups.ts | sed -n '1,260p'`
- **Registrar runs only after placeholder action fires:** The lazy command implementation removes the placeholder, calls the real registrar, then reparses argv. That means initial program construction does not execute the pairing registrar for unrelated commands.
  - file: [src/cli/program/register-lazy-command.ts:23](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/cli/program/register-lazy-command.ts#L23)
  - command: `nl -ba src/cli/program/register-lazy-command.ts | sed -n '1,220p'`
- **Root help skips plugin CLI registration:** `run-main.ts` skips plugin command registration for `openclaw --help`, and the policy test asserts that behavior.
  - file: [src/cli/run-main.ts:260](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/cli/run-main.ts#L260)
  - command: `nl -ba src/cli/run-main.ts | sed -n '240,290p'`
- **Tests cover lazy subcli placeholder behavior:** The subcli tests verify that with no primary command, placeholders are registered for all subcommands without invoking the heavy registrar, and that the real registrar is only called when the command is parsed.
  - file: [src/cli/program/register.subclis.test.ts:97](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/cli/program/register.subclis.test.ts#L97)
  - command: `nl -ba src/cli/program/register.subclis.test.ts | sed -n '1,240p'`
