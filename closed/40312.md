## Summary

Current main and the live docs site no longer reproduce the reported silent `/tasks` redirect. The checked-in Mintlify config has no `/tasks` redirect rule, canonical task docs live at `/automation/tasks` and `/cli/tasks`, and `https://docs.openclaw.ai/tasks` now returns an explicit 404 with zero redirects.

## Best Possible Solution

Close this issue as no longer reproducible: keep `/tasks` as an explicit 404 on the public docs site, keep canonical documentation under `/automation/tasks` and `/cli/tasks`, and open a new issue only if a future deployed hosting rule reintroduces a silent redirect with response details.
