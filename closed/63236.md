## Summary

Current `main` already routes MiniMax image-tool requests through `MiniMax-VL-01` with inline base64 `data:image/...` payloads, and the iMessage/media attachment path now prefers local attachment reads with tested URL fallback. That is a materially different implementation from the issue’s reported failure mode of passing an inaccessible remote URL to MiniMax.

## Evidence

- **Image tool now pairs MiniMax chat primaries with MiniMax vision:** `resolveImageModelConfigForTool` auto-selects `minimax/MiniMax-VL-01` or `minimax-portal/MiniMax-VL-01` when the primary provider is MiniMax, instead of keeping image-tool traffic on `MiniMax-M2.7`.
  - file: [src/agents/tools/image-tool.ts:121](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/agents/tools/image-tool.ts#L121)
