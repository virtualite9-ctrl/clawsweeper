## Summary

Current main implements the missing resolver behavior: the image runtime resolves configured provider models before checking `input`, and the shared model resolver preserves `input: ["text", "image"]` for matching configured/provider-prefixed rows. For MiniMax specifically, current main intentionally keeps `MiniMax-M2.7` as a text chat model and routes image understanding through the plugin-owned `MiniMax-VL-01` provider path, which addresses the old pre-provider `Model does not support images` failure from 2026.4.11/2026.4.15.

## Best Possible Solution

Close this issue as fixed on main. Keep the shipped resolver behavior that honors configured image-capable provider rows, and keep MiniMax image understanding on the plugin-owned `MiniMax-VL-01` path. If a provider still fails after the request reaches its API, track that as a provider-specific endpoint/auth/payload issue with fresh logs.
