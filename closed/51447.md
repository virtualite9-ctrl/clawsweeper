## Summary

Current `main` already supports Discord voice-message sends with `asVoice=true` and no `components`, auto-generates Discord voice attachment metadata, documents the flow, and includes regression coverage. The behavior is present in released `v2026.4.22`.

## Evidence

- **Voice sends no longer require content/components in tool entrypoint:** The Discord message action reads `asVoice` and makes `message` optional when `asVoice=true`, so the old schema conflict is not present on current `main`.
