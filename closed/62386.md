## Summary

Current `main` already handles Control UI assistant replies by appending final assistant messages or streamed fallback text into `chatMessages`, reloading history on terminal chat events, and rendering those messages in the chat view with regression coverage.

## Evidence

- **Final chat events now preserve assistant replies:** `handleChatEvent` appends the final assistant payload to `chatMessages`, and if the final event has no usable `message` payload it falls back to the accumulated streamed text instead of dropping the reply.
