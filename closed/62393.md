## Summary

Current `main` already qualifies slash-containing model ids with the selected provider in the Control UI, and the picker sends that fully qualified value back to the backend. That matches the reported LiteLLM allowlist failure mode, and the changelog shows this shipped in `v2026.4.9`.

## Evidence
