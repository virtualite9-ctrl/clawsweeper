## Summary

Current main already fixes the user-visible MiniMax image-understanding failure by routing MiniMax M2.7 primary-model image tool calls to the plugin-owned MiniMax-VL-01 media provider and MiniMax VLM endpoint, rather than adding MiniMax-M2.7 to the VLM allowlist.

## Best Possible Solution

Close this issue as implemented by the current MiniMax image-understanding routing. Keep M2.7 as the MiniMax chat model and continue using MiniMax-VL-01 for the image tool and media-understanding path; any future desire to make MiniMax-M2.7 itself a first-class image model should be a new product/API compatibility discussion with live MiniMax endpoint evidence.
