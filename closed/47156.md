## Summary

This PR should close because its remaining useful work does not match the current source-repo ownership model: localized docs are generated in the separate docs publish repo, and Control UI locale files in this repo are generated output rather than hand-maintained translations. Current main already has generated zh-CN UI coverage, while the proposed root Chinese README is not a maintained source-repo surface.

## Best Possible Solution

Do not merge this PR. Keep the source repo's README/docs English-first with glossary-backed generated translations in the docs publish repo, and handle Control UI Chinese copy through the existing generated locale pipeline rather than manual edits to generated zh-CN output. A future contribution should target English source strings, glossary entries, or the i18n generator/publish workflow when a concrete translation gap remains.
