## Summary

The PR documents a useful optional PENDING.md workspace convention, but the current OpenClaw product boundary points this kind of reusable community workflow toward ClawHub skills/plugins rather than core docs. Current main already has the needed workspace, skill, ClawHub, and HEARTBEAT.md surfaces; no missing core extension API is evident.

## Best Possible Solution

Close the core-docs PR and move the PENDING.md convention into a ClawHub skill/plugin package that ships a SKILL.md, a PENDING.md template, and a parser-compatible HEARTBEAT.md snippet. If publishing that package exposes a missing generic extension point, open a narrower core issue for that API.
