## Summary

Current `main` already preserves Control UI image attachments: the UI sends `chat.send.attachments`, gateway chat passes model image capability into attachment parsing, and text-only model runs now offload images as `media://inbound/...` refs instead of dropping them. The latest release tag `v2026.4.23` still had the old drop-on-text-only behavior, so this appears fixed on `main` but not yet in a released tag.

## Evidence
