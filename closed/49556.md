## Summary

Current main still does not source or preload nvm before installer Node detection. The installer checks `command -v node` and `node -v` directly, so the reported non-interactive `curl | bash` path can still find an older system Node before the user's nvm Node. The linked open PR #49594 may be an implementation path, but it is unmerged and does not prove the issue is resolved on main.

## Evidence

- **Installer checks active PATH directly:** `parse_node_version_components` and `check_node` resolve Node through `command -v node` and `node -v` without loading nvm first, matching the reported failure mode where an older system Node appears earlier in PATH.
