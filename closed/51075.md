## Summary

Current main still does not provide the requested gateway file-log observability fields. The logger writes raw tslog records with numbered arguments plus `time`, subsystem loggers only bind a subsystem and optional call-site metadata, and the flat `message` field exists only in parsed/diagnostic views rather than in the persisted JSONL line. Diagnostics/OTLP surfaces are adjacent but do not satisfy the issue's request for `agent_id`, `session_id`, `channel`, hostname, and flattened message in the gateway log file.

## Best Possible Solution

Keep the issue open and implement this in the core logging pipeline, not ClawHub. The likely path is to add a small, documented enrichment layer around gateway file-log writes or request-scoped log context so persisted JSONL records get safe top-level fields for host identity, agent/session correlation, channel when known, and a flattened `message`, with redaction/privacy rules and tests covering raw JSONL output, parser compatibility, diagnostics export, and OTLP log export.
