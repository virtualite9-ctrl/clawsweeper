## Summary

Close PR #51096 as ClawHub-scoped. The PR adds a core docs page comparing third-party tunnel tools, including author-owned tunelo promotion, but current OpenClaw scope keeps optional third-party capability and plugin promotion outside core and points discovery/review to ClawHub. Current main already documents the supported remote Gateway paths and security model around loopback, SSH, Tailscale Serve/Funnel, trusted-proxy auth, secure browser contexts, and explicit Control UI origins; the tunnel recipes can be published as a neutral ClawHub skill or bundle without a missing OpenClaw core API.

## Best Possible Solution

Close this PR and keep core docs focused on the maintained remote Gateway security model. Publish the tunnel-tool recipes as a neutral ClawHub skill/bundle or external plugin, with no preferred author-owned tool and with explicit Control UI security guidance for HTTPS, device identity, gateway auth, and gateway.controlUi.allowedOrigins. If core docs later add public-relay guidance, it should be maintainer-scoped and security-reviewed rather than a vendor comparison page.
