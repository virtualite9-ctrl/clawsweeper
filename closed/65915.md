## Summary

The report is obsolete against the current setup: it targeted the external `@opik/opik-openclaw` plugin at `0.2.9`, while upstream `v0.2.10` explicitly restored Opik trace export after the plugin’s hook-registration bug under newer OpenClaw lifecycle behavior, and current install guidance has moved current OpenClaw users to the ClawHub package path. ([github.com](https://github.com/comet-ml/opik-openclaw/releases/tag/v0.2.10))

## Evidence

- **Issue targeted old Opik plugin build:** The issue body reports `openclaw plugins inspect opik-openclaw` -> `Version: 0.2.9`, so the failing setup was not the current upstream plugin release.
- **Old plugin registered hooks too late:** `@opik/opik-openclaw` v0.2.9 registers its LLM/tool/subagent hooks inside `start(ctx)` rather than during plugin `register(...)`, matching the silent-no-export symptom after lifecycle changes.
  - command: `curl -L --silent https://raw.githubusercontent.com/comet-ml/opik-openclaw/v0.2.9/src/service.ts | nl -ba | sed -n '430,640p'`
  - sha: [e3626d9ce255](https://github.com/openclaw/openclaw/commit/e3626d9ce2552b44bf8e2cbdd3963b6544711553)
- **Upstream plugin shipped the fix before this issue was filed:** Opik plugin release `v0.2.10` says it 'Restores Opik trace export in embedded OpenClaw runs' and moved hook registration from `start()` to registration time because hooks registered in `start()` were silently dropped.
  - command: `curl -L --silent https://api.github.com/repos/comet-ml/opik-openclaw/releases/tags/v0.2.10 | sed -n '1,220p'`
  - sha: [eb8c116](https://github.com/openclaw/openclaw/commit/eb8c116)
- **Current OpenClaw contract expects registration in `register(api)`:** OpenClaw’s plugin entry docs define `register(api)` as the place where plugins register capabilities and hooks.
  - file: [docs/plugins/sdk-entrypoints.md:44](https://github.com/openclaw/openclaw/blob/250acdd2a28c190ed7934e3ba99022f24123d3c9/docs/plugins/sdk-entrypoints.md#L44)
  - sha: [250acdd2a28c](https://github.com/openclaw/openclaw/commit/250acdd2a28c190ed7934e3ba99022f24123d3c9)
- **Current OpenClaw loads services after registration and now hardens external installs:** `loadOpenClawPlugins()` runs plugin `register()` during load, while plugin services are started later. Current `main` also symlinks the host `openclaw` package into external peer-dependency plugins, with regression coverage for that path.
  - file: [src/plugins/loader.ts:2863](https://github.com/openclaw/openclaw/blob/250acdd2a28c190ed7934e3ba99022f24123d3c9/src/plugins/loader.ts#L2863)
  - sha: [250acdd2a28c](https://github.com/openclaw/openclaw/commit/250acdd2a28c190ed7934e3ba99022f24123d3c9)
- **Service start happens after registry assembly:** Plugin services are started from the already-built registry, so service-start-time hook registration is outside the intended registration path.
  - file: [src/plugins/services.ts:33](https://github.com/openclaw/openclaw/blob/250acdd2a28c190ed7934e3ba99022f24123d3c9/src/plugins/services.ts#L33)
  - sha: [250acdd2a28c](https://github.com/openclaw/openclaw/commit/250acdd2a28c190ed7934e3ba99022f24123d3c9)
- **External plugin install hardening on current main:** Current `main` links the host `openclaw` package into external plugins that declare `openclaw` as a peer dependency; the behavior is covered by install tests.
  - file: [src/plugins/install.ts:613](https://github.com/openclaw/openclaw/blob/250acdd2a28c190ed7934e3ba99022f24123d3c9/src/plugins/install.ts#L613)
  - sha: [250acdd2a28c](https://github.com/openclaw/openclaw/commit/250acdd2a28c190ed7934e3ba99022f24123d3c9)
