## Summary

The reported crash was a Google plugin schema-compilation failure, not a surviving browser-status bug on current main. Current OpenClaw loads Google plugin config from a small static manifest schema, and that shipped in v2026.4.23.

## Evidence

- **Reporter log points at Google plugin schema compilation:** The attached log shows `[plugins] google failed to load ... RangeError: Maximum call stack size exceeded` followed by `Error compiling schema`, so the failure path was AJV/schema compilation inside the bundled Google plugin load.
  - command: `curl -L --max-time 20 -s https://github.com/user-attachments/files/26532628/linkedin-test.log | iconv -f utf-16le -t utf-8 | sed -n '1,20p'`
- **Current Google plugin entry no longer builds config schema in code:** `extensions/google/index.ts` only defines provider-registration behavior for the plugin entry; there is no runtime `configSchema` construction in this module anymore.
  - file: [extensions/google/index.ts:148](https://github.com/openclaw/openclaw/blob/ecac696643c604c6d854341bfe6424bfa31423e9/extensions/google/index.ts#L148)
  - sha: [ecac696643c6](https://github.com/openclaw/openclaw/commit/ecac696643c604c6d854341bfe6424bfa31423e9)
- **Current Google config schema is a small static manifest schema:** The bundled Google plugin now declares a simple manifest `configSchema` with only `webSearch.apiKey` and `webSearch.model`, which is acyclic JSON and does not resemble the old recursive compile path from the report.
  - file: [extensions/google/openclaw.plugin.json:76](https://github.com/openclaw/openclaw/blob/ecac696643c604c6d854341bfe6424bfa31423e9/extensions/google/openclaw.plugin.json#L76)

  - file: [src/plugins/manifest-registry.ts:600](https://github.com/openclaw/openclaw/blob/ecac696643c604c6d854341bfe6424bfa31423e9/src/plugins/manifest-registry.ts#L600)
