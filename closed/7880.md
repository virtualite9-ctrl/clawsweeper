## Summary

Campfire is not implemented on current `main`, but the repo’s own scope docs and prior maintainer guidance place this request in the community-plugin/ClawHub path rather than bundled OpenClaw core.

## Evidence

- **Vision: optional capability should live as plugins, not core by default:** `VISION.md:54-59` says core stays lean, optional capability should usually ship as plugins, plugins should be hosted and maintained in their own repository, and the bar for adding optional plugins to core is intentionally high.
  - file: [VISION.md:54](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/VISION.md#L54)
  - sha: [cec34821754a](https://github.com/openclaw/openclaw/commit/cec34821754ad935c780805f3116f99449d46ac2)
- **Vision: commercial service integrations are out of current core scope:** `VISION.md:99-104` lists “Commercial service integrations that do not clearly fit the model-provider category” under “What We Will Not Merge (For Now).” Campfire is a 37signals chat integration, not a model provider.
  - file: [VISION.md:99](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/VISION.md#L99)
  - sha: [cec34821754a](https://github.com/openclaw/openclaw/commit/cec34821754ad935c780805f3116f99449d46ac2)
- **Docs: community plugins are the supported path for new channels:** `docs/plugins/community.md:9-22` says community plugins can add new channels, are built and maintained by the community, and are published on ClawHub or npm. It also says ClawHub is the canonical discovery surface.
  - file: [docs/plugins/community.md:9](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/docs/plugins/community.md#L9)

  - file: [docs/plugins/sdk-channel-plugins.md:11](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/docs/plugins/sdk-channel-plugins.md#L11)
