## Summary

Keep open. Current `main` still explicitly bypasses auto-compaction for CLI providers, and the CLI runner path does not emit any compaction state beyond raw usage/session binding. The issue discussion also points to an open proposed fix (#68388), which is further evidence this has not been resolved on `main` yet.

## Evidence

- **CLI providers are still excluded from preflight compaction:** `runPreflightCompactionIfNeeded(...)` computes `isCli` and returns early on `if (params.isHeartbeat || isCli)`, so the normal auto-compaction gate never evaluates for `claude-cli` sessions.
  - file: [src/auto-reply/reply/agent-runner-memory.ts:380](https://github.com/openclaw/openclaw/blob/c2139635ff82b367b734ea052487258b1b33f060/src/auto-reply/reply/agent-runner-memory.ts#L380)
  - sha: [c2139635ff82](https://github.com/openclaw/openclaw/commit/c2139635ff82b367b734ea052487258b1b33f060)
- **CLI runner still reports usage only, not compaction lifecycle:** The CLI result metadata includes `usage` and optional `cliSessionBinding`, but no CLI-side `compactionCount`, `lastCallUsage`, or other compaction trigger/result metadata that would show the runner participates in compaction lifecycle handling.
  - file: [src/agents/cli-runner.ts:258](https://github.com/openclaw/openclaw/blob/c2139635ff82b367b734ea052487258b1b33f060/src/agents/cli-runner.ts#L258)
  - sha: [c2139635ff82](https://github.com/openclaw/openclaw/commit/c2139635ff82b367b734ea052487258b1b33f060)
- **Current main only persists CLI usage snapshots after the run:** `agent-runner.ts` persists CLI usage with `usageIsContextSnapshot: isCliProvider(providerUsed, cfg)`, and `session-usage.ts` uses that to update `totalTokens` in the session store. That is accounting, not a compaction trigger.
  - file: [src/auto-reply/reply/agent-runner.ts:1345](https://github.com/openclaw/openclaw/blob/c2139635ff82b367b734ea052487258b1b33f060/src/auto-reply/reply/agent-runner.ts#L1345)
  - sha: [c2139635ff82](https://github.com/openclaw/openclaw/commit/c2139635ff82b367b734ea052487258b1b33f060)
- **Issue discussion already identified a proposed fix that is not reflected on main:** The provided timeline links PR #68388 from the reporter with a proposed CLI compaction/session-rotation fix. In this checkout, `git log --oneline --all --grep='cli-backend.*compaction\|claude-cli.*compaction\|rotate claude-cli sessions'` returned no matching merged commit, which is consistent with the code still skipping CLI compaction.
  - command: `git log --oneline --all --grep='cli-backend.*compaction\|claude-cli.*compaction\|rotate claude-cli sessions'`
