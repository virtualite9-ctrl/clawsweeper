## Summary

Current `main` already implements the reported behavior: `openclaw status` resolves configured probe auth, passes it into the gateway probe path, and authenticated loopback probes stay device-bound so read/detail RPCs are not scope-limited. The same implementation is present in release `v2026.4.22`.
