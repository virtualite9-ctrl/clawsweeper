## Summary

Current main already handles the reported failure mode conservatively: missing `${ENV_VAR}` config references are collected as warnings instead of throwing during config load, Telegram read-only/account inspection marks unresolved env-backed token inputs as `configured_unavailable`, Telegram `isConfigured` gates startup on an actually available token, and the gateway channel manager skips unconfigured accounts while containing channel/account startup failures instead of taking down the whole gateway. The implementation is present in the shipped `v2026.4.24` tag as well as current main.

## Best Possible Solution

Close this issue as implemented for the gateway crash-loop problem. Keep the current per-account degradation behavior; if maintainers still want `~/.openclaw/secrets/.env` as an additional credential-broker compatibility path, track that as a narrower enhancement.
