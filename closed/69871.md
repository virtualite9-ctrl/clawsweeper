## Summary

Current main implements the reported Z.AI image-routing behavior. The bundled Z.AI plugin now registers image understanding, seeds the affected VL models as image-capable with the correct Z.AI `/api/paas/v4` provider base URL, and the image runtime resolves image models through `resolveModelWithRegistry`, which applies configured provider `api`/`baseUrl` overrides before completion. The related #70604 ClawSweeper note was against an older SHA; the current `src/media-understanding/image.ts` path now uses the override-aware resolver that note said was missing.

## Best Possible Solution

Close #69871 as implemented on main and keep the shipped bundled Z.AI provider/media-understanding path. If a current release still produces a Z.AI VL 404, it should be reopened as a new narrow bug with the current OpenClaw version, generated `models.json` provider entry, selected `agents.defaults.imageModel`, and the exact model/baseUrl used at runtime.
