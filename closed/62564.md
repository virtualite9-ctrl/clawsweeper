## Summary

Current `main` contains shipped Windows loader fixes for the onboarding/channel-loading seam behind this `v2026.4.5` crash, and the changelog records those fixes in `v2026.4.9`.

## Evidence

- **Onboarding hits channel setup immediately after model selection:** `runSetupWizard` finishes auth/model selection, then imports `onboard-channels` and calls `setupChannels`. That matches the report timing: the crash happens right after choosing a model, before the next wizard step renders.
  - file: [src/wizard/setup.ts:565](https://github.com/openclaw/openclaw/blob/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a/src/wizard/setup.ts#L565)
  - command: `nl -ba src/wizard/setup.ts | sed -n '540,670p'`
  - sha: [c997a9f9788e](https://github.com/openclaw/openclaw/commit/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a)
- **Windows native Jiti is disabled in the shared loader policy:** `supportsNativeJitiRuntime()` returns false on `win32`, and `resolvePluginLoaderJitiTryNative()` flows that policy into bundled loader seams. This is the core guard against raw `C:\...` specifiers reaching native ESM import on Windows.
  - file: [src/plugins/sdk-alias.ts:696](https://github.com/openclaw/openclaw/blob/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a/src/plugins/sdk-alias.ts#L696)
