## Summary

PR #49593 should close in favor of the already-linked canonical feature issue #49567. Current main still does not implement per-binding tool policy, and this PR only accepts the config shape without routing the matched binding into the tool-policy pipeline. The prior Codex review comments correctly identified that as a security-relevant silent no-op, and the author confirmed the runtime enforcement was intentionally deferred.

## Best Possible Solution

Close PR #49593 as superseded by the open canonical issue #49567. The best maintainer path is to keep #49567 open and implement per-binding tool policy end to end in one coherent change: schema/types, route resolution or matched-binding propagation, tool-policy enforcement for embedded and gateway surfaces, docs, and regression tests proving denied tools are actually unavailable for a matched binding.
