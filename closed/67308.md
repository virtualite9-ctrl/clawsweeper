## Summary

Current main already covers the reported heartbeat-token problem from v2026.4.9: users can disable scheduled heartbeats, `every: "0m"` disables the persistent cadence, empty or scaffold-only `HEARTBEAT.md` skips model calls, and low-token heartbeat modes now avoid full session/history context. The latest release tag v2026.4.24 contains the same relevant implementation.

## Best Possible Solution

Close this issue as implemented. Keep the current heartbeat controls and docs: users who want no scheduled heartbeat should set `agents.defaults.heartbeat.every` to `0m` or use the runtime `openclaw system heartbeat disable`; users who want low-cost heartbeats should keep `HEARTBEAT.md` empty/scaffold-only, or enable `lightContext` plus `isolatedSession`. Any remaining provider-auth failure, such as the DeepSeek 401 in the report, should be handled as provider credential setup rather than this heartbeat bug.
