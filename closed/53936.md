## Summary

PR #53936 is superseded by the current plugin-owned web-search credential runtime shipped in v2026.4.24. The old insertion point no longer owns provider key resolution, and current main resolves provider credentials through config, SecretRef, and env fallback with runtime diagnostics and gateway warning logging.

## Best Possible Solution

Close this PR as superseded by the shipped v2026.4.24 web-search credential runtime. Keep the current provider-owned credential resolution and gateway warning path; any remaining desire for per-execute env-presence diagnostics should be tracked as a new, narrow issue/PR against the current provider runtimes with tests for empty-vs-unset env values.
