## Summary

Current main already fixes the reported Control UI `[embed]` iframe-auth failure by routing hosted canvas iframe URLs through a scoped canvas capability URL instead of depending on iframe Authorization headers. The same implementation is present in the v2026.4.24 release tag.

## Best Possible Solution

Keep the shipped capability-scoped canvas embed implementation. If a similar failure is reported again, ask for confirmation that the Control UI is on v2026.4.24 or newer and that the iframe `src` includes `/__openclaw__/cap/<token>/` before the canvas path.

## Evidence
