## Summary

Keep open. Current `main` still persists `toolResult.details` by default, while the live cap and recovery accounting only measure `toolResult.content[*].text`. For `exec`, the same aggregated output is still copied into both visible text and `details.aggregated`, so the durable transcript-bloat / compaction-mismatch report remains credible on `938b53698ec17934a5d1a6cffb279c584f64d9f3`.

## Evidence
