## Summary

Current `main` already routes channel registry loader lookups through the pinned channel registry, and it includes a regression test covering `loadChannelOutboundAdapter` after the active registry is replaced. The same relevant files are unchanged between `v2026.4.22` and current `main`, so the issue appears fixed and shipped.
