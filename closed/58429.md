## Summary

#58429 should close as a duplicate of the existing open retention trackers. The same remaining core work is already covered by #52327, which requests a durable `agents.defaults.subagents.sessionRetention`-style reaper for completed subagent/ACP sessions, and by earlier #51767, which identifies that `cron.sessionRetention` only sweeps cron run sessions. Current main still lacks that retention knob and still defaults run-mode `sessions_spawn` cleanup to `keep`, so this is not implemented; it is duplicate/superseded by the canonical open issues.

## Best Possible Solution

Close #58429 as duplicate/superseded and consolidate remaining work under the open canonical retention issue, preferably #52327, with #51767 as the earlier root-cause report. The implementation should add a documented, validated durable retention policy for completed subagent and ACP session-store rows, run it in a restart-safe path comparable to the cron reaper, and preserve active/main sessions with focused tests for default keep-mode runs, session-mode subagents, ACP keys, and transcript archival.
