## Summary

Current `main` already routes `MEDIA:`-only announce payloads through outbound normalization, preserves `mediaUrls` when visible text is empty, and Slack accepts media-only sends. The relevant normalization lines were introduced in commit `3ffd944e6ba6dfe4c7c4c275848f1c0a1f246e28`, which does not appear in a release tag yet, so this looks fixed on `main` but unreleased.

## Evidence

- **Outbound normalization preserves MEDIA-only payloads:** `createOutboundPayloadPlanEntry` parses reply directives from payload text, merges parsed `mediaUrl`/`mediaUrls`, strips the directive text, and keeps the media on the normalized payload. `projectOutboundPayloadPlanForOutbound` then emits payloads as long as there is reply content, including media-only payloads with empty text.

That means cron announce now goes through the same MEDIA-aware normalization path as other outbound delivery.
