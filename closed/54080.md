## Summary

Current `main` already suppresses commentary/pre-tool assistant text before channel partial delivery, which prevents Feishu streaming cards from showing that narration.

## Evidence

- **Core partial-delivery suppression:** `handleMessageUpdate` returns immediately when the resolved delivery phase is `commentary`, before any `onPartialReply` callback can fire. That prevents pre-tool narration from reaching channel preview streams.
  - file: [src/agents/pi-embedded-subscribe.handlers.messages.ts:471](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/agents/pi-embedded-subscribe.handlers.messages.ts#L471)
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Core final-message suppression:** `handleMessageEnd` also suppresses commentary-phase assistant messages from user-visible output, so commentary-only assistant messages are not emitted at message end either.
  - file: [src/agents/pi-embedded-subscribe.handlers.messages.ts:624](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/agents/pi-embedded-subscribe.handlers.messages.ts#L624)
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Feishu still streams from partial replies:** Feishu streaming cards still consume `onPartialReply` payloads for live card updates, so the core suppression above directly fixes the Feishu surface described in the issue.
  - file: [extensions/feishu/src/reply-dispatcher.ts:495](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/extensions/feishu/src/reply-dispatcher.ts#L495)
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Regression coverage:** Tests now explicitly assert that commentary-phase assistant messages and commentary-only textSignature metadata do not call `onPartialReply`, do not emit block replies, and do not populate stored assistant text.
