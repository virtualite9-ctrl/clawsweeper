## Summary

Keep PR #64694 open. Current main still has the non-live provider catalog discovery gap the PR targets: the timeout helper returns null outside live mode, that null is passed into provider catalog execution, and the wrapper then awaits the catalog promise without a deadline. The PR is recent, focused, and not superseded by current main; the remaining work is maintainer review of the patch, especially the later Aisle findings about timeout bounds and log sanitization.

## Best Possible Solution

Keep the PR open. The best path is to land this behavior, or an equivalent maintainer patch, in src/agents/models-config.providers.implicit.ts with a bounded non-live default provider catalog timeout, compatible live-mode behavior, focused tests around timeout resolution/execution, and follow-up handling for the Aisle security review before merge.
