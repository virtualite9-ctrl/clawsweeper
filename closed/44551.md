## Summary

Current main has real Control UI i18n support, including zh-CN locale loading, browser-locale resolution, a language picker, and many translated navigation/button labels. The issue should still stay open because the specific Gateway audit/event/log request is not fully implemented: current event-log views still render raw event identifiers and several log/debug surfaces still hard-code English strings.

## Best Possible Solution

Keep the issue open and narrow it to the remaining Control UI i18n gap: add translation keys and a display-name formatter for Gateway event/audit/tool operation names, convert the Logs and Debug views away from hard-coded English where they are user-facing, and add focused UI/i18n tests that prove zh-CN renders translated labels while preserving raw identifiers where product intentionally requires them.
