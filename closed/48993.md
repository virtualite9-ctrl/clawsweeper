## Summary

PR #48993 is a duplicate implementation of the already-open Control UI remote markdown image work tracked by #40945 and the earlier PR #41025. Current main still intentionally flattens remote markdown images, and CSP/docs/tests explicitly block remote image fetches, so this needs the canonical product/security decision rather than another parallel renderer-only PR.

## Best Possible Solution

Close #48993 as duplicate/superseded and keep the remaining decision centralized on #40945 and the earlier implementation discussion in #41025. If maintainers want remote inline images, implement a coordinated design that updates the markdown renderer, Control UI CSP, tests, and docs together with explicit privacy/networking behavior.
