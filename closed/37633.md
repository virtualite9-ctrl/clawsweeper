## Summary

Current `main` no longer takes the Feishu plugin-status path that raised unresolved SecretRef errors during `openclaw plugins list`, and Feishu read-only inspection now treats unresolved SecretRefs as unavailable instead of throwing. The fix appears to be on `main` only, not in the latest tagged release `v2026.4.23`.

## Evidence

- **`plugins list` uses snapshot status path:** The CLI `plugins list` action now calls `buildPluginSnapshotReport(...)` instead of the diagnostics/runtime-loading path.
  - file: [src/cli/plugins-cli.ts:156](https://github.com/openclaw/openclaw/blob/f00d65a30496335fb068f01fe529dbba484d4312/src/cli/plugins-cli.ts#L156)

That means `openclaw plugins list` no longer needs the plugin-register path that produced the reported Feishu error.
