## Summary

Current main fixes the reported WebChat duplicate transcript pattern. The v2026.4.24 path submitted the metadata-prefixed prompt and then rewrote the transcript-visible prompt, which could leave both the original wrapped user turn and the rewritten raw user turn in the JSONL stream that Control UI reads. Current main splits runtime metadata from the visible transcript prompt before submission, submits only the clean prompt, and queues runtime context as hidden custom context instead of rewriting the prompt afterward.

## Best Possible Solution

Keep the current main runtime-context/transcriptPrompt split and ship it in the next release. No additional core issue is needed unless someone reproduces duplicate durable WebChat writes on a build at or after 4a3030df9efa72c44bab567a4390ac0c4876a73f; any cleanup for already-corrupted v2026.4.24 transcripts should be handled separately as an explicit doctor/repair request.
