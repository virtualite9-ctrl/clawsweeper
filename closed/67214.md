## Summary

Current main already handles isolated cron `agentTurn` announce delivery to Telegram without the reported undefined `startsWith` failure. The cron runner now resolves the announce target before dispatch, sends text-only Telegram announce output through the real outbound adapter, and the Telegram plugin guards missing requester `to` values before checking `startsWith`. Regression coverage exists for both plain Telegram and forum-topic announce delivery, and the fix is present in the v2026.4.24 release tree.

## Best Possible Solution

Close this issue as already implemented on main and shipped in v2026.4.24. Keep the current cron direct-delivery path and Telegram target guards; any remaining failure should be filed as a fresh issue with v2026.4.24-or-newer logs, the stored cron job JSON, and the resolved Telegram target/account details.
