## Summary

Current main now gives browser workflows a stable download handoff: explicit wait/download routes save files and return structured path metadata, managed OpenClaw Chrome profiles save ordinary click-triggered downloads into the guarded downloads directory, and docs/changelog describe the behavior. The exact proposed tool shape and size/mimeType fields were not added, but the observable file-capture problem is covered for the reported isolated OpenClaw profile.
