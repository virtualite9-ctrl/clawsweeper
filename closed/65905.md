## Summary

Current `main` already provides the supported cross-platform plugin-facing replacement for this access pattern: the injected `api.runtime` surface and the package export `openclaw/extension-api`, plus package-root alias resolution and tests for `extensionAPI` lookup. The report centers on a third-party plugin resolving the internal path `openclaw/dist/extensionAPI.js`, which is not the documented plugin contract.

## Evidence

- **Issue report uses internal dist path:** The issue body says the failing plugin calls `createRequire(import.meta.url).resolve("openclaw/dist/extensionAPI.js")`, and the failure is specifically about resolving that internal `dist` file path on Windows global npm installs.
- **Public package export exists on main:** `package.json` exports `./extension-api` from the package root, giving plugins a stable import path instead of depending on `dist/extensionAPI.js` layout details.
