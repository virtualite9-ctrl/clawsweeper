## Summary

Current `main` already implements `session_status model=default` as a persistent reset of the session model override, and the same behavior is present in the latest released tag. The report matches older behavior from `2026.4.5`, but the current code path deletes the override fields, writes the updated session store, and has a regression test covering this exact case.

## Evidence
