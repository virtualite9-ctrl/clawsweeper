## Summary

Current main already implements the underlying `toolResult.details` persistence hardening through an always-on bounded sanitizer, rather than the opt-in `toolResultDetailsPersist` policy from this PR. It avoids the `JSON.stringify(...).length` crash path, bounds oversized details without full original serialization, caps before plugin persistence hooks, reapplies the cap before append, and documents/tests the behavior.

## Best Possible Solution

Close this PR as implemented on main. Keep the shipped always-on bounded `toolResult.details` sanitizer, tests, and docs rather than merging the older opt-in `toolResultDetailsPersist` policy branch.
