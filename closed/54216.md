## Summary

Current `main` does not expose the Feishu message tool as a card-only schema. Feishu contributes no schema fragment at discovery time, the shared `message` tool already exposes `message`, the system prompt tells the model to use `message` for `action=send`, and Feishu runtime still accepts either `text` or `message`. The issue description no longer matches current behavior on `main`.

## Evidence

- **Feishu discovery has no schema contribution:** `describeFeishuMessageTool` returns only `actions` and `capabilities`; there is no `schema`/`card` property contribution in the returned object.

- **Shared message tool exposes `message`:** The core send schema includes `message: Type.Optional(Type.String())`, so the LLM already sees a plain-text send field without any Feishu-specific `text` addition.
