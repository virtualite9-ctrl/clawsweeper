## Summary

Current main already implements dynamic browser profile management well enough for the requested multi-agent isolation: profiles can be created and deleted at runtime, each managed profile gets its own persisted browser state boundary, and browser operations are explicitly profile-scoped.

## Evidence

## Risks / Open Questions

- The exact proposed `ephemeral:<task>` browser-tool syntax is not present; current main exposes explicit create/delete surfaces instead.
- Automatic per-agent affinity and idle self-destruction are not obvious in the current surface, so any remaining gap would be a narrower follow-up rather than this broad request.
