## Summary

Close PR #44751. Current main already preserves the Feishu SDK's built-in response unwrapping, so the reported axios-envelope failure is not reproducible from the checked-in code and dependency contract. The PR's extra `data`-only normalization would also change valid Feishu payloads that intentionally include a top-level `data` field.
