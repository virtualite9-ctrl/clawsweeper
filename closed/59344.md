## Summary

Keep PR #59344 open. Current main still does not deduplicate bootstrap files after `agent:bootstrap` hook mutation, and downstream context injection still processes every file entry it receives. The PR targets a real remaining seam, though the review comment about resolving relative paths against `workspaceDir` should be addressed before merge.

## Best Possible Solution

Keep the PR or reimplement its fix on main at `resolveBootstrapFilesForRun`/`sanitizeBootstrapFiles`: deduplicate hook-adjusted bootstrap entries by normalized file identity after hook overrides and before `buildBootstrapContextFiles`, preserve distinct files with the same basename, resolve relative paths against `workspaceDir`, and keep the targeted unit test for duplicate path behavior.
