## Summary

PR #72047 is a duplicate/superseded implementation of the same doctor trajectory-sidecar bug already tracked by PR #70812. Current main still needs a fix, but #72047 uses a filename-only `.trajectory.jsonl` exclusion that the PR review correctly flags as unsafe for valid session IDs ending in `.trajectory`; the existing #70812 track is the better canonical path because it already addresses that concern.

## Best Possible Solution

Close #72047 as superseded by #70812 and continue the fix on the canonical PR path: exclude true trajectory sidecars without misclassifying valid `.trajectory` session IDs, keep doctor regression coverage, and verify shared helper callers such as usage discovery and disk-budget cleanup.
