## Summary

#65742 reports a real remaining bug on current main, but the remaining work is already tracked by the directly linked open PR #65746. Current main still routes the pdf tool through the registry-only helper, while #65746 explicitly closes this issue and proposes the narrow pdf-tool/config-provider fallback plus regression test.

## Best Possible Solution
