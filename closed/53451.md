## Summary

Current `main` already suppresses the misleading multi-listener warning for same-process dual-stack loopback gateway listeners and covers the `status --all` behavior with focused tests, so this PR is redundant.

## Evidence

- **Core loopback detection is already implemented on main:** `isDualStackLoopbackGatewayListeners()` parses listener addresses, requires finite PIDs, verifies loopback IPv4+IPv6 families, and `buildPortHints()` skips the multiple-listener warning when that helper returns true.
