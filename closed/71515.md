## Summary

Keep open. Current `main` still requires `models.providers.<id>.models` in both the zod source and generated config schema, while the Ollama runtime still contains an explicit missing-models discovery path. The PR is addressing a real bug, but its branch also includes unrelated `tasks/*.md` operational logs that should be removed before merge, and its code references are partly stale against current `main` layout.
