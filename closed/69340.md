## Summary

Keep PR #69340 open. Current main still has the exact pre-PR TUI formatter behavior: `FILE_LIKE_RE` allows hyphens, but `isCopySensitiveToken` only enters that guard for underscore-containing tokens, so long hyphen-separated identifiers without digits still fall through to `chunkToken(...).join(" ")` and can receive injected spaces. The PR is a narrow one-line fix tied to open bug #48432, and it is not implemented on main or clearly superseded by the related items provided.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to add a focused regression test in `src/tui/tui-formatters.test.ts` for a 33+ character hyphen-separated token such as a long package name, verify that generic opaque tokens still chunk, and then merge or revise the narrow guard change if it demonstrably fixes the TUI copy-safety case from #48432. If maintainers want consistency beyond the TUI sanitizer, evaluate the similar guard in `src/terminal/note.ts` separately rather than closing this PR as obsolete.
