## Summary

Current main and the latest shipped release harden the local-Ollama/OpenAI-compatible streaming paths that can surface this exact `includes` failure. Local Ollama now routes through the native Ollama provider stream, and the shared assistant stream handler coerces missing content before reaching the remaining `.includes()` check.

## Evidence

- **Current assistant stream handler guards the includes path:** `handleMessageUpdate` coerces `assistantRecord.content` to a string before calling `resolveAssistantTextChunk`, and `resolveAssistantTextChunk` returns early when content is empty before reaching `accumulatedText.includes(content)`.
