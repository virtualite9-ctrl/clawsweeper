## Summary

Current main implements the task timestamp fix. The cron issue described in #70887 flows through task registry creation with a caller-supplied startedAt, and current `createTaskRecord`, update, and restore paths normalize task lifecycle timestamps before records are audit-visible. Regression tests cover earlier startedAt values and assert no `inconsistent_timestamps` findings. The related open PR #70891 identified the same root cause, but the broader fix was merged via #71871.

## Best Possible Solution

Close #70887 as implemented on main. Keep the registry-level normalization from #71871 as the canonical fix; the narrower open PR #70891 is superseded by that merged implementation rather than needed for this issue.
