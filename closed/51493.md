## Summary

Close as implemented on main. Current main routes direct subagent/cron announce delivery through the gateway agent path, classifies `No active WhatsApp Web listener` as a transient announce error, retries that path, and has regression coverage for the exact WhatsApp listener failure text. The changelog says that reliability fix shipped in 2026.2.24, so the issue's claim that this was still missing after 2026.3.13 does not match the current tree or release notes.
