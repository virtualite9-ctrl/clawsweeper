## Summary

Current main already fixes the linked Telegram preview-final internal `message:sent` hook gap with a narrower implementation that avoids the retained-preview false positives raised during PR review. The implementation is also present in `v2026.4.24`, so this PR is now superseded by shipped code.

## Best Possible Solution

Close this PR and keep the shipped narrower implementation on main. Any future plugin `message_sent` parity should reuse the current finalized-preview delivery metadata and explicitly avoid retained, unconfirmed, or wrong-message preview states.
