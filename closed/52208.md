## Summary

Current `main` already contains a targeted macOS launchd recovery for this failure mode: failed `kickstart -k` now preserves or repairs the LaunchAgent registration, and `openclaw gateway start` repairs installed-but-unloaded LaunchAgents instead of leaving them orphaned.

## Evidence

- **Restart path repairs unloaded LaunchAgent after failed kickstart:** `ensureLaunchAgentLoadedAfterFailure()` probes `launchctl print` and, if the service is no longer loaded, re-runs LaunchAgent bootstrap as a best-effort repair before surfacing the original kickstart failure.
  - file: [src/daemon/launchd.ts:604](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/src/daemon/launchd.ts#L604)

- **Restart logic invokes that repair on kickstart failures:** `restartLaunchAgent()` now calls `ensureLaunchAgentLoadedAfterFailure()` both when the initial `kickstart -k` fails unexpectedly and when the retry after bootstrap still fails, preventing the LaunchAgent from being left unloaded.
  - file: [src/daemon/launchd.ts:660](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/src/daemon/launchd.ts#L660)
