## Summary

Current `main` already fixes the isolated manual `cron.run` deadlock through a different implementation: the embedded runner now remaps inner cron-triggered work onto the `nested` lane, so this PR's separate `cron:manual` lane is no longer needed.

## Evidence

- **Manual runs still enqueue on the cron lane:** `enqueueRun()` on current `main` still schedules manual runs via `CommandLane.Cron`, so the outer wrapper behavior from the original report is still present and must be resolved elsewhere in the stack.
