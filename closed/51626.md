## Summary

#51626 is the same remaining Discord A2A announce account-routing bug already tracked by open issue #42652. Current main still shows the Discord path can skip session delivery-context hydration and forward an undefined accountId, so this should close only as duplicate consolidation, not as fixed.

## Evidence

- **Canonical duplicate:** Provided related-item context identifies open issue #42652, "Discord announce step uses wrong accountId (falls back to 'default' instead of session's accountId)", with a prior ClawSweeper review keeping it open because current main still skips session lookup for Discord announce targets.
- **A2A announce send forwards the resolved target accountId:** The A2A announce delivery calls gateway `send` with `accountId: announceTarget.accountId`; if target resolution omits the account, downstream channel/default handling decides the account.
