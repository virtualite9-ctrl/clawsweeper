## Summary

Keep open. Current `main` no longer uses the literal `"<authenticated>"`, but it still resolves Google Vertex ADC to a non-secret marker (`gcp-vertex-credentials`), persists that marker as the runtime API key, injects it into the provider-owned `google-vertex` transport, and that transport serializes any non-JSON `apiKey` as `x-goog-api-key`. The core bad path described in the issue therefore still exists in current code, just with a renamed sentinel.
