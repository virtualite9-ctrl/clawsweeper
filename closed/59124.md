## Summary

Current main implements a dedicated `openclaw exec-policy` workflow that solves the user-visible problem by synchronizing `tools.exec.*` with the host exec approvals file. The configure wizard still does not expose a separate exec-security section, but the shipped replacement is more complete because it updates both policy layers that determine whether exec prompts appear.

## Best Possible Solution

Keep the shipped dedicated `openclaw exec-policy` workflow as the maintainer-supported fix for this issue, since it synchronizes both `tools.exec.*` and host approvals. Any future desire to add an additional wizard screen should be tracked as a separate product enhancement, not as this bug.
