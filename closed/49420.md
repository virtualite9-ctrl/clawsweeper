## Summary

Current main already implements the ACP abort/timeout behavior this PR targets, using a manager-owned turn timeout and actor-queue abort checks rather than the PR's outer dispatch wrapper. The current implementation forwards dispatch abort signals into ACP turns, rejects queued aborted work before it starts, times out hung turns, performs bounded cancel/close cleanup, and has regression coverage for the stuck-session cases.

## Best Possible Solution

Close this PR as already implemented on current main. Keep the shipped manager-owned timeout/abort implementation; do not land this stale branch. If a narrower delivery-specific cancellation/order bug still reproduces on current main, it should be opened as a fresh issue or PR with a focused reproduction against the current dispatch/delivery code.
