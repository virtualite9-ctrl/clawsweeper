## Summary

Current main already addresses the reported subagent context leakage class: native `sessions_spawn` children are isolated by default, transcript forking is opt-in, prompt/docs steer agents to omit context unless needed, and regression tests cover both isolated-default and explicit-fork behavior. The remaining possibility of a model choosing the wrong `task` argument is a separate tool-call quality problem, not the stale context inheritance bug reported against 2026.4.14.

## Best Possible Solution

Keep the shipped design: subagents start with isolated context by default, `context: "fork"` is explicit when transcript inheritance is needed, and docs/tests preserve that contract. If a new wrong-task spawn appears, open a fresh issue with the parent prompt, emitted `sessions_spawn` arguments, transcript excerpts, and version so maintainers can distinguish model tool-call hallucination from context leakage.
