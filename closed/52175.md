## Summary

Current main already registers isolated cron `agentTurn` sessions in `sessions.json` before execution and the Control UI reads them through `sessions.list`, so the reported invisibility regression is implemented on main and present in the current release snapshot I inspected.

## Risks / Open Questions

- The code still logs and continues if the pre-run store write itself fails, so a separate disk-write/permissions failure could still orphan a transcript. That is a different failure mode than the isolated-session registration regression described here.
