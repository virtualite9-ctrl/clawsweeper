## Summary

Keep this PR open. Current main still lacks the Google Meet `realtime.agentId` config/schema/docs surface and still routes Google Meet realtime consults through the shared consult runtime without an agent id, while the shared runtime and Voice Call plugin already support named-agent routing. The provided PR discussion also supports keeping it open: bot reviews approved the narrow change, and the earlier ClawSweeper review found that related #72372 did not supersede this routing fix.

## Best Possible Solution

Keep this PR open for maintainer review and likely landing. The best path is the narrow plugin fix already proposed: add `realtime.agentId` to Google Meet config, manifest, docs, and tests; normalize it through the existing routing SDK; pass it to `consultRealtimeVoiceAgent`; and keep the agent-scoped consult session key behavior covered by regression tests.
