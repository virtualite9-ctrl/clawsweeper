## Summary

Current `main` already prevents the empty async-exec relay case this issue describes: successful `exec.finished` events with no output are now suppressed, non-empty exec completions carry compact output into the queued system event, and duplicate `exec.finished` relays are deduped by `(sessionKey, runId)` before heartbeat delivery.

## Evidence

- **Empty success relays are suppressed:** `exec.finished` now computes `shouldNotify = timedOut || exitCode !== 0 || compactOutput.length > 0` and returns early when that is false, so a success-without-output no longer emits a heartbeat relay event.
  - file: [src/gateway/server-node-events.ts:659](https://github.com/openclaw/openclaw/blob/30aa7e0d4d225650b58c2fba79e71bffa5627b08/src/gateway/server-node-events.ts#L659)

  - sha: [30aa7e0d4d22](https://github.com/openclaw/openclaw/commit/30aa7e0d4d225650b58c2fba79e71bffa5627b08)
- **Exec completions now carry visible payload text:** When an `exec.finished` event is emitted, the queued system event includes the exit summary plus compacted output text on following lines, which is the payload the heartbeat prompt refers to.
  - file: [src/gateway/server-node-events.ts:676](https://github.com/openclaw/openclaw/blob/30aa7e0d4d225650b58c2fba79e71bffa5627b08/src/gateway/server-node-events.ts#L676)
