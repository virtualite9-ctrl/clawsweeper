## Summary

Current `main` already addresses this class of stale-main-session breakage: Control UI chat applies history only to the still-active `sessionKey`, gateway session targeting canonicalizes `main` to the current default-agent main key, and the reset/mutation path prunes legacy ghost keys back to one canonical row.

## Evidence

- **Control UI drops stale history results unless the active session still matches:** `loadChatHistory()` snapshots the requested `sessionKey` and only applies the response if both the request version and `state.sessionKey` still match, preventing stale reloads from repainting the wrong session after switches/resets.
