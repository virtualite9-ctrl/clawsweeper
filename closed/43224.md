## Summary

Keep this PR open. Current main has refactored Ollama discovery into the bundled Ollama plugin, but the reported behavior is not implemented there: discovery still keys off only `models.providers.ollama`, so a custom remote provider with `api: "ollama"` plus `OLLAMA_API_KEY` can still opt into the implicit localhost probe. The PR is stale against current file layout, but the underlying fix remains plausible and should be rebased into the plugin-owned discovery path.

## Best Possible Solution

Keep the PR open but request a rebase onto current main. The fix should live in the Ollama plugin discovery path, scan configured providers for custom `api: "ollama"` entries with non-loopback/non-default base URLs before building the implicit canonical `ollama` provider, preserve warning behavior for canonical local `OLLAMA_API_KEY` users, and add current tests covering the remote custom-provider skip plus full `127/8` loopback treatment.
