## Summary

Current main and the verified v2026.4.24 release tree already address the reported Control UI/WebChat long-task websocket instability: Gateway sends periodic non-droppable `tick` keepalives to websocket clients, advertises the tick cadence in the hello policy, the browser Control UI reconnects automatically, and the chat UI has explicit history-reconciliation paths after reconnect/final events. The related #56645 keepalive report was already closed on the same implementation, and #60930 was filed against v2026.3.28 behavior that no longer matches the shipped gateway/control-ui code.

## Best Possible Solution

Close this issue as implemented and keep the shipped Gateway/Control UI keepalive plus history-reconciliation behavior. Any remaining distinct problem where reconnect creates a new session should continue under the narrower #68162 follow-up rather than keeping this older v2026.3.28 live-update report open.
