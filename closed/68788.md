## Summary

Current main still has the Windows Claude CLI spawn bug: the Anthropic plugin registers `claude-cli` with command `claude`, the CLI runner sends that argv through the process supervisor, and the supervisor child adapter still only allows `npm`, `pnpm`, `yarn`, and `npx` for Windows `.cmd` shim suffixing. The reporter's follow-up correctly identifies the deeper `.cmd` direct-spawn `EINVAL` risk, and open PR #68819 supersedes this issue by tracking the full fix rather than the narrower allowlist-only PRs #68817/#68849.

## Best Possible Solution

Close this issue as superseded by #68819 and keep the remaining engineering/review work on that PR or an equivalent core patch. The final fix should handle supervisor Windows `.cmd` shim resolution and regression tests, not just add `claude` to the allowlist.
