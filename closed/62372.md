## Summary

Current `main` contains a targeted fix for provider-plugin load reentry that would otherwise recurse into plugin loading and overflow the stack, plus regression coverage for the exact failure mode. That matches the reported `RangeError: Maximum call stack size exceeded` while loading bundled provider-related plugins during `pairing approve`.

## Evidence

- **Reentry guard in provider hook loading:** `resolveProviderPluginsForHooks` now detects when provider plugin loading is already in flight for the same load options and returns an empty result instead of reentering plugin loading recursively.
  - file: [src/plugins/provider-hook-runtime.ts:123](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/src/plugins/provider-hook-runtime.ts#L123)
