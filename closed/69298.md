## Summary

Current `main` already separates embedded-runner prompt/context sizing from accumulated run usage and normalizes cached prompt totals before generic prompt-token math, which addresses the Codex-harness double-counting path described in #69298.

## Evidence

- **Cached prompt totals are normalized before prompt-token math:** `normalizeUsage()` now treats OpenAI-style cached prompt fields as a breakdown of the prompt total and subtracts cached tokens from raw prompt/input totals before downstream code re-adds `cacheRead`. That removes the `input + cacheRead` double-count contract mismatch described in the issue.
  - file: [src/agents/usage.ts:123](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/src/agents/usage.ts#L123)
  - command: `nl -ba src/agents/usage.ts | sed -n '105,138p'`
  - sha: [7f6452897e25](https://github.com/openclaw/openclaw/commit/7f6452897e25543c9263b67aabf140379d1c0b0c)
- **Embedded runner records latest-call prompt usage separately:** The embedded runner now keeps `lastRunPromptUsage` from the latest model call rather than accumulated attempt usage, specifically so session `totalTokens` reflects current context usage instead of tool-loop accumulation.
  - file: [src/agents/pi-embedded-runner/run.ts:963](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/src/agents/pi-embedded-runner/run.ts#L963)
  - command: `nl -ba src/agents/pi-embedded-runner/run.ts | sed -n '963,969p'`
  - sha: [7f6452897e25](https://github.com/openclaw/openclaw/commit/7f6452897e25543c9263b67aabf140379d1c0b0c)
- **Agent meta exposes promptTokens from latest-call usage:** `buildUsageAgentMetaFields()` derives `promptTokens` from `lastRunPromptUsage` and returns it alongside normalized `usage`/`lastCallUsage`, giving session persistence a prompt-size snapshot independent of accumulated usage totals.
  - file: [src/agents/pi-embedded-runner/run/helpers.ts:97](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/src/agents/pi-embedded-runner/run/helpers.ts#L97)
  - command: `nl -ba src/agents/pi-embedded-runner/run/helpers.ts | sed -n '97,114p'`
  - sha: [7f6452897e25](https://github.com/openclaw/openclaw/commit/7f6452897e25543c9263b67aabf140379d1c0b0c)
- **Session store prefers promptTokens for persisted totalTokens:** Session persistence calls `deriveSessionTotalTokens()` with `promptTokens`, and passes `usage` only when no prompt-token override exists. That means `sessionEntry.totalTokens` is now persisted from the latest prompt/context snapshot instead of the additive accumulated usage path.
  - file: [src/agents/command/session-store.ts:124](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/src/agents/command/session-store.ts#L124)

- **Regression test covers latest-call vs accumulated usage:** Test `sets promptTokens from the latest model call usage, not accumulated attempt usage` asserts a run with accumulated usage `{input: 4000, cacheRead: 120000}` but latest assistant usage totaling `2000` persists `promptTokens = 2000`, matching the fix needed for inflated context accounting.
