## Summary

Issue #56521 asks for a new cron `failureAlert.mode: "agent-turn"` that turns a failed cron alert into a main-session agent-triggering event. Current `main` has not shipped that mode: the cron failure-alert types, gateway protocol schema, config validation, CLI option handling, and docs all still limit failure alerts to `announce` and `webhook`. The same remaining work is already tracked by the linked open PR #56545, whose body explicitly says `Closes #56521` and describes the same implementation direction using `enqueueSystemEvent` plus `requestHeartbeatNow`.

## Best Possible Solution

Close #56521 as superseded by PR #56545 and keep the product/API/test discussion on that PR. The best path is for maintainers to review, update, land, or replace #56545 rather than keep a duplicate feature issue open beside the active implementation thread.
