## Summary

Close as `clawhub`. The PR is a useful observability idea, but current OpenClaw already exposes the necessary tool-call data through plugin hooks plus plugin logging/redaction surfaces, and the project vision keeps optional fleet-specific capability out of core when plugin APIs are sufficient. The submitted core patch also still has a production-facing redaction-ordering risk called out in review, so it should not land as-is.

## Best Possible Solution

Close this core PR and keep the capability on the plugin/ClawHub path: implement fleet-specific tool-call INFO logging with `before_tool_call`/`after_tool_call`, `api.logger.info`, and SDK redaction helpers. If maintainers later choose a core default log, start from #55806 and require redaction before truncation, parser-safe fields, run/tool correlation, and regression tests before landing.
