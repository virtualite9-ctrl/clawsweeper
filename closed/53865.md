## Summary

PR #53865 is useful Lemonade Server provider work, but it is an optional third-party model-provider integration that current OpenClaw plugin APIs can support outside core. Current main does not ship Lemonade, and the PR does not identify a missing core/provider API; the best maintainer cleanup is to close the core PR and point the contributor toward publishing the already-posted Lemonade provider implementation as a ClawHub/npm plugin.

## Best Possible Solution

Close the core PR as third-party provider-plugin scope. Repackage the already-posted Lemonade implementation as a standalone provider plugin with `openclaw.plugin.json`, publish it to ClawHub or npm, and reopen core discussion only if the plugin needs a new generic SDK/API seam or a maintainer explicitly wants official bundled Lemonade ownership.
