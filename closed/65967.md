## Summary

Current `main` already carries the fallback-selected model through session state and status rendering. The fallback runner now persists selected/active fallback model fields before retry, session listing prefers the persisted override over stale runtime fields, the chat/TUI model picker reads that live session row, and `/status` explicitly shows selected vs active fallback models. This looks fixed on `main`, but I could not tie it to a released tag after `v2026.4.23`, so `fixedRelease` is `null`.
