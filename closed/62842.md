## Summary

Keep open. Current main has a partial route-first mitigation for `openclaw config get`/`unset`, and the checked-in startup benchmark now covers `config get gateway.port` at about 0.6s. The original report also covers `openclaw config file` and `openclaw config --help`; those paths are not route-first today, and help is explicitly excluded from the route-first dispatcher. The related PR #71122 is still separate/open in the provided context and does not prove the full issue is fixed on main.

## Best Possible Solution

Keep this issue open until current main either adds route-first/lightweight handling for `openclaw config file` and subcommand help, or splits those remaining paths into a narrower canonical issue. Add benchmark coverage for `config file` and `config --help` alongside `config get gateway.port`, then close only after those paths are proven sub-second on the shipped CLI.
