## Summary

Current `main` already supports persistent per-channel model pinning via `channels.modelByChannel`, which delivers the requested outcome without adding `agents.list[].channels`.

## Evidence

- **User-facing docs: persistent per-channel model pins:** The channel config docs explicitly document `channels.modelByChannel` as the way to pin specific channel IDs to a model, and note that it applies unless the session already has an explicit override.
  - file: [docs/gateway/config-channels.md:44](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/docs/gateway/config-channels.md#L44)
