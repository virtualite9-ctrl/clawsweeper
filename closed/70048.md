## Summary

Current main already has the proxy-safe setup paths requested here: a generic Custom Provider onboarding flow for OpenAI-compatible/Anthropic-compatible endpoints, bundled LiteLLM and LM Studio provider setup, and vLLM-specific setup that prompts for base URL/API key/model instead of requiring a generic proxy user to use a vLLM-only field. The old `VLLM_BASE_URL` surface is not present in the current tree.

## Best Possible Solution

Close this issue as already implemented. Keep the current Custom Provider flow as the generic OpenAI-compatible proxy path, keep LiteLLM/LM Studio/vLLM as provider-owned setup surfaces, and use a narrower new issue only if a specific current wizard screen still routes a concrete proxy setup into the wrong provider.
