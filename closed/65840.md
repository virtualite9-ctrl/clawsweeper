## Summary

The reported qwen/qwen3.6-plus image-model mismatch is already addressed in current main and was shipped in v2026.4.23: OpenClaw now uses an endpoint-aware bundled Qwen catalog, only advertises qwen3.6-plus on Standard DashScope endpoints, and suppresses stale Coding Plan rows with a provider-specific remediation message.
