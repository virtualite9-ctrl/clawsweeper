## Summary

Current main and the v2026.4.24 release already cover both requested fixes. Bundled runtime dependency installs use an OpenClaw-owned external stage/cache instead of the user npm cache, and doctor/runtime-deps scans already respect explicit plugin/channel disablement under includeConfiguredChannels with regression coverage.

## Best Possible Solution

Close PR #71614 as obsolete on main. Keep the shipped managed-stage/runtime-cache implementation; if operators still see a ProtectHome or disabled-channel runtime-deps failure, track that as a fresh reproduction against the shipped v2026.4.24 behavior rather than merging this narrower duplicate patch.
