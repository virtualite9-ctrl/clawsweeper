## Summary

Current main already supports running heartbeats against an explicit channel session via `agents.defaults.heartbeat.session`, and background-task notifications already follow the requester session or requester origin. The exact `set-primary` CLI/config shape from the issue was not added, but the requested behavior is already implemented well enough in shipped OpenClaw.

## Evidence

- **Heartbeat docs expose explicit session override:** The heartbeat docs document `session` as an optional heartbeat session key, say `main` is the default, and explicitly state you can override it to a specific channel session (Discord/WhatsApp/etc.). The same behavior is present in the latest release tag inspected locally.
