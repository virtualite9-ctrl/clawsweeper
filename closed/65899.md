## Summary

Current `main` already routes internal `message:sent` through canonical sent-message builders and has direct regressions for both shared outbound delivery and Telegram group delivery, so the reported `Cannot access 'group' before initialization` path is no longer present. The relevant implementation landed after the latest tagged release, so this appears fixed on `main` but not yet shipped in `v2026.4.23`.

## Evidence

- **Shared outbound `message:sent` now uses canonical sent-hook context:** On current main, `createMessageSentEmitter()` builds a canonical sent context and feeds it into `toInternalMessageSentContext(...)`, passing explicit `isGroup`/`groupId` fields instead of any local `group` binding in the internal-hook payload path.
  - file: [src/infra/outbound/deliver.ts:537](https://github.com/openclaw/openclaw/blob/fe930b987e33f4e1acd93b5ef1ea76c8b7260047/src/infra/outbound/deliver.ts#L537)
