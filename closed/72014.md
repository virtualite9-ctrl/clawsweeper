## Summary

Current main already implements the behavioral fix requested by PR #72014: WebChat/gateway transcript assembly skips `isReasoning` reply payloads, and the affected transcript append paths use that filtered assembly. Tests and WebChat docs also cover the intended behavior, so the PR is obsolete apart from its helper extraction.

## Best Possible Solution

Close PR #72014 as obsolete because the #71910 WebChat reasoning-transcript duplication behavior is already fixed on current main. Keep the shipped inline `isReasoning` filtering and existing tests/docs; any future extraction into a separate helper should be a maintainer-approved refactor, not required to solve the bug.
