## Summary

Keep #51346 open. Current main does not implement the requested awaited Telegram ingress middleware seam: Telegram still builds message context and dispatches directly, config/types/schema/docs do not expose `channels.telegram.ingressMiddlewares`, and there is no `src/channels/ingress` runtime or equivalent SDK surface. This is not a ClawHub-only close because the current plugin model lets a plugin implement its own channel, but does not provide a hook for an external plugin to intercept the bundled Telegram pre-dispatch path; the PR is attempting to add that missing source/API seam. The patch likely needs maintainer/API review and rework against current SDK boundary guidance, but it is still useful work rather than obsolete cleanup.

## Best Possible Solution

Keep the PR open for maintainer/API review. The best path is to rebase and reshape the work as a narrow, additive Telegram/channel ingress seam with aligned config types, strict schema, docs, tests, package exports/SDK entrypoints, and explicit documentation of whether middleware can only observe/side-effect or can short-circuit/reroute dispatch.
