## Summary

Close #51346 as superseded by merged PR #50444. The specific Telegram-only `ingressMiddlewares` config was not adopted, but the central need is now covered by the shipped, channel-agnostic `before_dispatch` plugin hook: Telegram routes into the shared reply dispatcher, the hook is awaited before ACP/model dispatch, `{ handled: true }` short-circuits the model path, and tests/docs/changelog cover the behavior.

## Best Possible Solution

Close #51346 and steer users to the shipped `before_dispatch` plugin hook from #50444. Custom public/private Telegram bridge behavior belongs in a plugin using `api.on("before_dispatch", ...)`; if it needs side-effect Telegram sends, the existing plugin runtime can load the Telegram outbound adapter instead of adding a Telegram-only core middleware chain.
