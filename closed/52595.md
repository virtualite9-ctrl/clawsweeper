## Summary

Current `main` already treats missing session usage as unknown instead of zero for `/status`, and the same implementation is present in release `v2026.4.22`. The session usage writer marks missing context totals as stale/unknown, the status formatter renders unknown usage as `unknown/<ctx> (?%)` rather than `0/<ctx>`, and regression tests cover the missing-usage case.

## Evidence

- **Session writer keeps missing context usage unknown:** `src/auto-reply/reply/session-usage.ts` sets `patch.totalTokens = totalTokens` and `patch.totalTokensFresh = typeof totalTokens === "number"`, with an inline comment that missing last-call snapshots mean context utilization is stale/unknown.

- **Status formatter renders unknown instead of zero:** `src/commands/status.format.ts` renders `used == null` as `unknown/<ctx> (?%)` or `unknown used`, not `0/<ctx>`.
  - file: [src/commands/status.format.ts:27](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/commands/status.format.ts#L27)
