## Summary

Issue #65396 should close as a duplicate/superseded tracker. The requested API change, allowing headless persistent `mode="session"` without `thread=true`, is already tracked by open PR #39372. Current main still enforces the thread requirement in code, tests, and docs, so this is not implemented; the best cleanup is to keep the remaining product/API decision on the canonical PR rather than leave a second issue open for the same work.

## Best Possible Solution

Close this issue as a duplicate/superseded tracker and keep #39372 as the canonical place for the maintainer decision. If headless persistent sessions become first-class, the fix should update both native and ACP spawn paths, preserve current run-mode `sessions_send`/`sessions_yield` behavior, and address ACP inline-delivery safety before landing.
