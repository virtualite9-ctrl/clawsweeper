## Summary

Current `main` contains targeted webchat session-persistence fixes that address the reported post-update history loss by scoping persisted session selection per gateway, resetting stale persisted session picks on tokenized opens, and normalizing `main` aliases to the gateway’s canonical main session key.

## Evidence

- **Per-gateway persisted session selection:** `loadSettings`/`persistSettings` now store and restore `sessionKey` and `lastActiveSessionKey` under `sessionsByGateway`, keyed by normalized gateway scope, instead of relying on one global selection.
