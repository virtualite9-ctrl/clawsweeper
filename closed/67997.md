## Summary

This PR should close because it optimizes a Mintlify source page for GitHub-rendered browsing, but the repository docs policy requires internal links in `docs/**/*.md` to remain root-relative. Current `main` keeps the affected `docs/tools/plugin.md` links root-relative, and the docs audit/i18n tooling relies on that form.

## Best Possible Solution

Do not merge this PR as-is. Keep internal links in Mintlify source docs root-relative so route auditing and localization continue to work; use the published docs site for user-facing navigation, and handle any GitHub-rendered source browsing problem outside this docs source page policy.
