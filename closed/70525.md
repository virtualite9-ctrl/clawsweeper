## Summary

Current main already resolves the Control UI uploaded-image history problem through media-reference persistence and rendering, rather than by replaying raw inline image bytes from chat.history. The PR discussion was useful, especially the review feedback around limiting source.data exposure, but the current main path is the safer canonical implementation.

## Best Possible Solution

Close this PR as redundant with current main's media-reference implementation. Keep the canonical fix as persisted MediaPath/MediaPaths plus authenticated Control UI rendering; only open a new focused issue if maintainers want legacy top-level image.data transcript conversion with a clearly scoped redaction contract.
