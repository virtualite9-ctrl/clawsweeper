## Summary

Current `main` no longer runs `gateway status` through plugin/doctor validation, and the status path now uses a lean daemon-status route that skips plugin validation entirely. That removes the warning-emission path that could inject doctor text like `channels.telegram...` into the JSON probe flow. Current `main` also tolerates malformed non-JSON gateway frames without throwing from the WebSocket client handler.

## Evidence
