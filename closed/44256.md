## Summary

Keep PR #44256 open. Current main still has the Feishu/Lark `@_all` false-positive behavior, now in `extensions/feishu/src/bot-content.ts`: `checkBotMentioned()` returns true whenever raw message content contains `@_all`. `parseFeishuMessageEvent()` still uses that helper to set `ctx.mentionedBot`, and the group `requireMention` gate trusts `ctx.mentionedBot`, so the user-facing bug remains live even though the PR's original deletion target moved from `bot.ts` to `bot-content.ts`. The matching regression test is also not present on main.

## Best Possible Solution

Keep the PR or its canonical successor open and apply the same behavior change in the current implementation: remove the unconditional `@_all` shortcut from `extensions/feishu/src/bot-content.ts`, rely on explicit Feishu mention metadata or post-content bot open_id matching, and add the proposed `@all`/`@_all` regression test in `extensions/feishu/src/bot.checkBotMentioned.test.ts`.
