## Summary

Current main already covers the issue’s two observable requests: direct-message continuity for named agents is supported by the session routing contract, and compaction now preserves structured conclusions, pending asks, recent turns, and transcript history rather than reducing state to bootstrap metadata only.

## Best Possible Solution

Keep the shipped routing and compaction behavior. For single-user/default DM continuity, use the existing `main` DM scope; for isolated multi-channel identities, use the documented `session.identityLinks` or `per-peer` modes rather than adding another core session-key path. Open a narrower follow-up only if a concrete channel produces the wrong route under those supported modes.
