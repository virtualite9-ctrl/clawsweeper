## Summary

Current main already implements PR #70891's observable fix. The PR's narrow `createdAt` clamp was superseded by merged PR #71871, which normalizes task lifecycle timestamps at the registry boundary on create, update, and restore. Cron timer and manual cron runs still pass caller-captured `startedAt`/`lastEventAt` into `createRunningTaskRun`, but that now reaches normalized `createTaskRecord` persistence, and regression tests assert zero `inconsistent_timestamps` findings for an earlier `startedAt`.

## Best Possible Solution

Close PR #70891 as implemented by the broader merged #71871 normalization. Keep the registry-boundary timestamp normalization as the canonical solution and do not merge this narrower duplicate branch.
