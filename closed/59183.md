## Summary

PR #59183 is superseded by merged PR #44969. Current main already routes custom providers with provider-level or model-level `api: "google-generative-ai"` through the Google plugin policy and normalizes bare `generativelanguage.googleapis.com` roots to `/v1beta`; the changelog records that shipped fix in 2026.3.24-beta.1.

## Best Possible Solution

Close #59183 as superseded by #44969 and keep the current plugin-owned normalization path. Any remaining proxy query/hash edge case should be handled as a new focused Google plugin policy PR with current-code tests.
