## Summary

Current `main` already implements the missing path from `read`/image tool output to channel media delivery: image tool results carry a media path, the embedded runner extracts and queues that media, pending tool media is merged into the next reply or flushed at agent end, outbound delivery calls `sendMedia`, and Feishu uploads/sends via `sendMediaFeishu`. This issue matches older behavior on 2026.3.13, not current `main` at `8866544ffefb42ca234c66c337134fb60743d8d0`.

## Risks / Open Questions

- As of this review, the latest visible tag is `v2026.4.22`; the implementation is present on `main` but may not be in a released build yet, so users on older releases can still hit the original bug.
- Auto-forward still depends on the tool result exposing a media path (`details.media` or `details.path`). Results that contain only inline image bytes with no path are still not auto-sent, per the code comment in `src/agents/pi-embedded-subscribe.tools.ts:245`.
