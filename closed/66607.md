## Summary

Current `main` resolves the Control UI thinking dropdown label from session/default `thinkingDefault` before falling back to model heuristics. The older `v2026.4.23` tag still lacked that defaults fallback, so this issue is fixed on `main`, but I could not confirm a released tag containing the fix from the local checkout.

## Evidence

- **Current Control UI uses configured/default thinking level:** `resolveChatThinkingSelectState` now builds `defaultLevel` from `activeRow?.thinkingDefault ?? state.sessionsResult?.defaults?.thinkingDefault` before falling back to `resolveThinkingDefaultForModel(...)`, which matches the requested fix for showing the effective/configured default in the dropdown.
