## Summary

This draft PR adds a new core JSONL-to-SQLite observability ingestor, parsers, watcher, synthetic-data generator, and tests. Current main does not contain that core ingestor surface, and the repo scope points optional observability ingestion/export work toward plugins/ClawHub unless a missing core plugin API is identified. Existing plugin surfaces already support long-lived services with access to OpenClaw state, hooks for model/message/session/tool events, and shipped diagnostics exporters for OpenTelemetry/Prometheus, so this can be pursued as a ClawHub/plugin implementation rather than merged into core.

## Best Possible Solution

Close this draft PR as a ClawHub/plugin-direction item. The best path is a separately maintained observability-ingestor plugin, published through ClawHub/npm, using plugin services plus stateDir/session/cache-trace file access and current diagnostics hooks/exporters. If that plugin hits a concrete missing SDK capability, open a focused plugin API issue or PR for that seam instead of adding a one-off core ingestor.
