## Summary

#70204 should close as superseded. The workflow-path fallback bug is still present on current main, but #68101 is the canonical open bug and #68106 is the narrower open PR for the same runner path. #70204 also includes approvalId plumbing that current main already has, and its workflow-file heuristic has unresolved review feedback about breaking valid inline pipelines.

## Best Possible Solution

Close #70204 as duplicate/superseded. Keep #68101 as the canonical bug and continue #68106 or a successor PR with a narrow path-like workflow-file heuristic that surfaces missing workflow paths without blocking valid inline pipelines. Do not re-merge the approvalId plumbing from this PR because current main already has it.
