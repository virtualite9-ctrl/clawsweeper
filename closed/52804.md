## Summary

Keep open. Current `main` rejects new `cron.add` requests that omit `payload.text` for `kind: "systemEvent"`, but the legacy persisted-job path still does not repair or safely handle `payload: { kind: "systemEvent", message: ... }`. That means the reported TypeError remains plausible for older stored jobs or callers that bypass gateway validation, so there is not strong evidence to close.

## Evidence

- **Gateway schema now requires `text` for `systemEvent` payloads:** `CronPayloadSchema` accepts `kind: "systemEvent"` only with non-empty `text`, not `message`, and `cron.add` rejects normalized requests that fail that validator.
  - file: [src/gateway/protocol/schema/cron.ts:130](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/gateway/protocol/schema/cron.ts#L130)
  - command: `nl -ba src/gateway/protocol/schema/cron.ts | sed -n '130,156p'`
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **`cron.add` enforces the validator before creating jobs:** The gateway cron method calls `validateCronAddParams(normalized)` and returns `invalid cron.add params` on failure, so fresh malformed API requests should be blocked on current `main`.
  - file: [src/gateway/server-methods/cron.ts:211](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/gateway/server-methods/cron.ts#L211)
  - command: `nl -ba src/gateway/server-methods/cron.ts | sed -n '200,225p'`
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **Legacy normalization still drops `message` from explicit `systemEvent` payloads:** `coercePayload()` preserves `kind: "systemEvent"` when explicitly set, trims `message` if present, then deletes `message` for all `systemEvent` payloads without mapping it to `text`. A legacy stored payload like `{ kind: "systemEvent", message: "..." }` therefore normalizes to `{ kind: "systemEvent" }`.
  - file: [src/cron/normalize.ts:147](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/cron/normalize.ts#L147)
  - command: `nl -ba src/cron/normalize.ts | sed -n '147,239p'`
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **Main-session execution still trims `payload.text` without guarding missing text:** `normalizePayloadToSystemText()` returns `payload.text.trim()` for `systemEvent`, and `resolveJobPayloadTextForMain()` calls it for any `job.payload.kind === "systemEvent"`. If a malformed legacy job reaches this path with missing `text`, it can still produce the reported `TypeError`.
  - file: [src/cron/service/normalize.ts:64](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/cron/service/normalize.ts#L64)
  - command: `nl -ba src/cron/service/normalize.ts | sed -n '64,68p' && nl -ba src/cron/service/jobs.ts | sed -n '887,892p'`
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **Store load path normalizes persisted jobs in memory but does not validate/repair this legacy shape:** `ensureLoaded()` runs `normalizeCronJobInput(raw)` on stored jobs and keeps the normalized result in memory. There is coverage for skipping invalid main jobs with `agentTurn` payloads, but no repair/skip path here for `systemEvent` jobs missing `text`.
  - file: [src/cron/service/store.ts:37](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/cron/service/store.ts#L37)
