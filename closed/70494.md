## Summary

PR #70494 should close as superseded by merged PR #71871. The reopened PR proposed adding audit-side timestamp jitter tolerance, but current main fixes the reported task-audit false positive at the registry boundary by normalizing task lifecycle timestamps on create, update, and restore while keeping the audit check strict for records that still reach audit in an invalid order.

## Best Possible Solution

Close #70494 as superseded by merged #71871. Keep the registry-boundary normalization as the canonical fix because it prevents benign lifecycle timestamp skew from entering audit-visible task records while preserving strict audit findings for real timestamp corruption.
