## Summary

Current main already routes Telegram group natural-language messages through the normal inbound and agent-dispatch path. The handler evaluates group access, allows `groupPolicy: "open"` and explicit configured group matches, records the group session context, and dispatches to the reply pipeline. Regression coverage includes plain `hello` messages in regular Telegram groups for open policy and allowlisted sender cases, and the same implementation is present in `v2026.4.24`.

## Best Possible Solution

Close this issue as implemented and keep the current Telegram group-routing behavior plus its regression coverage. Any future reports should use the current `v2026.4.24+` behavior as the baseline and include a fresh config/log reproduction if a different Telegram group path still drops messages.
