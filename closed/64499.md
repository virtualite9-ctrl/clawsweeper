## Summary

Close #64499 as a duplicate of the older open tracker #37591. Current main still has the macOS node `system.run.prepare` contract mismatch, but this report adds the same failure mode already being tracked there.

## Best Possible Solution

Close #64499 as a duplicate of https://github.com/openclaw/openclaw/issues/37591 and keep #37591 as the canonical open tracker. The fix should add macOS companion support for `system.run.prepare` or adjust the `exec host=node` contract, with regression coverage across the agent exec path, gateway command policy, macOS command advertisement, and macOS runtime dispatch.

- **macOS node does not advertise prepare:** The macOS node connect options are built from `currentCommands`; that command list includes `system.notify`, `system.which`, `system.run`, and exec approvals commands, but no `system.run.prepare`.
