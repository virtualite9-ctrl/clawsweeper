## Summary

Current main already supports user-defined session names through the existing session `label` field: Control UI exposes inline editing, `sessions.patch` persists it, and session display resolution prefers that label over the auto-derived display name.

## Evidence

- **Control UI inline rename shipped:** The Sessions page already has a `Label` column with an editable input, and changing it calls `props.onPatch(row.key, { label: value })`, which is the inline session rename flow requested for Control UI.
  - file: [ui/src/ui/views/sessions.ts:381](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/ui/src/ui/views/sessions.ts#L381)

## Risks / Open Questions

- The feature is exposed as session `label`, not `displayName`, so the naming can be easy to miss.
- I did not find a dedicated `openclaw sessions rename` CLI subcommand; the implemented path is Control UI/API/store-field based.
