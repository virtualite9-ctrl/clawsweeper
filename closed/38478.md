## Summary

Automatic cross-session memory persistence is useful, but current main has the extension surface for it and the broad always-on policy belongs in a ClawHub/plugin context engine rather than OpenClaw core. The existing bundled session-memory hook remains intentionally explicit-command scoped, while context-engine plugins can implement ingest, recall assembly, and after-turn persistence.
