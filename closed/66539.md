## Summary

Current `main` already supports stable operator-set session aliases in Control UI via the per-session `label` field, persists that value through `sessions.patch`, and prefers it over derived display names in session selectors/lists.

## Evidence

- **Control UI Sessions page exposes an editable per-session label:** The Sessions table renders an input bound to `row.label` and sends `props.onPatch(row.key, { label: value })`, so aliases are stored against a stable session key rather than transient metadata.
  - file: [ui/src/ui/views/sessions.ts:517](https://github.com/openclaw/openclaw/blob/9a0b26cafca76d267dd231b5eaceadb071ec749d/ui/src/ui/views/sessions.ts#L517)
  - command: `nl -ba ui/src/ui/views/sessions.ts | sed -n '452,526p'`
  - sha: [9a0b26cafca7](https://github.com/openclaw/openclaw/commit/9a0b26cafca76d267dd231b5eaceadb071ec749d)
- **Session picker prefers the operator label over derived display names:** `resolveSessionDisplayName` returns `row.label` first, then falls back to `row.displayName`, which matches the requested stable pinned name behavior in Control UI session selectors.
  - file: [ui/src/ui/chat/session-controls.ts:420](https://github.com/openclaw/openclaw/blob/9a0b26cafca76d267dd231b5eaceadb071ec749d/ui/src/ui/chat/session-controls.ts#L420)
