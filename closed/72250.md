## Summary

Current main already fixes the CI symptom from #72250. The failing PowerShell fixture is now covered directly, and a read-only scriptblock verification against current `scripts/install.ps1` returns a single `System.Boolean True` even when npm stdout/stderr chatter is stubbed. The concrete fix on main is commit `4a3030df9efa72c44bab567a4390ac0c4876a73f`, which avoids the PowerShell automatic `$Error` variable collision by renaming the installer color variable to `$ERROR_COLOR`; the same verification against `HEAD^` failed before `Main` with `Cannot overwrite variable Error because it is read-only or constant.`

## Best Possible Solution

Close #72250 as implemented on main. Keep the shipped `$ERROR_COLOR` fix and existing `install-ps1` regression coverage; any broader Windows installer UX or host-session concerns should remain on the existing Windows installer tracker context rather than this CI-specific issue.
