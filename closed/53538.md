## Summary

Current `main` already handles this best-effort delivery case: the gateway now downgrades internal-channel best-effort delivery to session-only instead of returning `INVALID_REQUEST`, and there is a regression test for the exact no-external-channel scenario described in PR #53538.

## Evidence

- **Gateway handler now downgrades best-effort internal delivery instead of rejecting:** In current `main`, when `deliver=true` resolves to the internal message channel and `bestEffortDeliver` is set, the handler catches channel-selection failure, records a downgrade reason, skips the `INVALID_REQUEST`, and forces session-only delivery because `deliver` is later computed as false for `INTERNAL_MESSAGE_CHANNEL`.
