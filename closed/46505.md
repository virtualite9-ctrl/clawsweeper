## Summary

Keep open. Current `main` at `56fe2aab9c87596515f2280c8060dc740ea64a88` still matches the problem statement in PR #46505: browser fetch aborts are classified as timeouts, and the timeout path still appends the persistent `Do NOT retry the browser tool` hint. The repo’s current test still asserts that an `aborted` fetch error should surface as `timed out`, so this behavior has not already been implemented on `main`.
