## Summary

The heartbeat-prompt clutter request is not fully implemented in the Control UI chat-history path, but the same remaining Control UI system-message filtering gap is already tracked by open issue #65131. Close #67238 as a duplicate/superseded item and keep the fix concentrated on the canonical issue.

## Best Possible Solution

Close #67238 as a duplicate of open #65131. Implement the fix under #65131 by making the `chat.history`/Control UI chat path share the existing heartbeat-history filtering (`isHeartbeatUserMessage` and `isHeartbeatOkResponse`, or the `buildSessionHistorySnapshot` logic) so heartbeat prompts and OK-only acknowledgements disappear consistently while real heartbeat alert content remains visible.
