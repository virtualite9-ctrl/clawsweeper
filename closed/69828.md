## Summary

Current main already has a targeted fix for this OpenRouter one-shot inference failure. Commit 54f4c45e5d7efc40ae22535d59414edba18c9815 changes `openclaw infer model run` so explicit OpenRouter model runs use a probe-specific, no-tools prompt path, avoid unrelated provider-catalog loading, and no longer inherit chat-agent silent-reply behavior that can surface the empty fallback response.

## Best Possible Solution

Close as implemented on main and keep the shipped probe-specific `modelRun` path. If this reappears after a build containing 54f4c45e5d7efc40ae22535d59414edba18c9815, it should be handled as a new regression with current logs, version, model ref, and OpenRouter response details.
