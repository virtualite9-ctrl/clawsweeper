## Summary

Current main already solves the reported Control UI behavior. Live assistant text and empty pre-text activity are rendered as separate stream/reading-indicator chat items independent of `showToolCalls`, submitted user messages are appended optimistically before the send RPC completes, and pre-tool stream text is preserved as a segment before live tool cards are rendered or hidden.

## Best Possible Solution

Close #67250 as implemented. Keep the current stream-item, reading-indicator, optimistic-send, and pre-tool stream-segment behavior; treat the linked one-line guard PRs as superseded unless maintainers want to add an extra focused `showToolCalls: false` render test.

## Evidence
