## Summary

Current main already routes tool-invocation extensibility through the plugin hook surface (`before_tool_call` / `after_tool_call`) rather than the internal `HOOK.md` hook system. The issue requests a new core internal-hook event, but project scope and docs point this class of feature to plugins published via ClawHub/npm.

## Evidence

- **Internal hooks are intentionally limited in scope:** The internal hooks docs list only command, session, agent, gateway, and message events, then explicitly direct tool-call use cases to plugin hooks instead of adding them to the internal hook surface.
