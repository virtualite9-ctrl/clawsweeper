## Summary

Current main already provides a plugin-based additive prompt path (`before_prompt_build` + `appendSystemContext`) with `agentId` context, and repo guidance says optional capability like this should live as a plugin/ClawHub package rather than a new core agent-config field.

## Risks / Open Questions

- There does not appear to be a first-party built-in plugin that provides this exact agent-instructions convenience today; someone would still need to publish or maintain a small plugin.
- Closing on scope assumes maintainers prefer the existing plugin extension surface over adding a small core config alias.
