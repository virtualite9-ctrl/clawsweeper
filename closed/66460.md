## Summary

Close #66460 as superseded by the open canonical #69492. Current main still lacks the event-audience primitive needed to distinguish cron-owned/internal exec completions from user-facing async results, and #69492 explicitly tracks this exact system-event-shape leakage class while listing #66460 as a direct related report.

## Best Possible Solution

Close #66460 as superseded by #69492, keep #69492 open as the canonical tracker, and implement the fix there as an event-level audience/internal-vs-user-facing classification honored by exec completion producers, heartbeat relay, and system-event consumer paths.
