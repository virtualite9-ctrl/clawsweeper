## Summary

Keep open. Current `main` does not implement `agents.defaults.imageGenerationModel.timeoutMs`, and the PR's proposed approach is incomplete against today's tool path because `image_generate` normalizes model config down to `{ primary, fallbacks }`, which would drop the new field before execution.

## Evidence

- **Current agent model config type has no timeout field:** `AgentModelConfig` on `main` still allows only a string or an object with `primary` and `fallbacks`; there is no `timeoutMs` property to configure for `imageGenerationModel`.
  - file: [src/config/types.agents-shared.ts:8](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/src/config/types.agents-shared.ts#L8)

  - sha: [c070509b7f8f](https://github.com/openclaw/openclaw/commit/c070509b7f8fdbe479e5d829643782c272e30e83)
- **Current schema rejects extra keys like timeoutMs:** `AgentModelSchema` is a strict union of string or `{ primary, fallbacks }`, so `timeoutMs` is not part of the validated config contract on `main`.
  - file: [src/config/zod-schema.agent-model.ts:3](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/src/config/zod-schema.agent-model.ts#L3)

- **Current docs only document primary/fallbacks for imageGenerationModel:** The user-facing agent config docs describe `imageGenerationModel` as either a string or `{ primary, fallbacks }`. The image-generation tool docs document `timeoutMs` only as a per-tool-call parameter, not a config default.
  - file: [docs/gateway/config-agents.md:343](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/docs/gateway/config-agents.md#L343)
