## Summary

Current `main` contains targeted Control UI fixes for the refresh-required webchat symptom: it defers `session.message` history reloads during an active run, preserves optimistic chat messages when `chat.history` returns a stale snapshot, and has focused tests covering both paths. The latest targeted fix is on `main` after the latest tagged release, so no shipped release tag could be confirmed yet.

## Evidence

- **Active-run reload deferral:** `handleSessionMessageGatewayEvent` now skips `chat.history` reloads while a run is active and records a deferred reload instead; the inline comment explicitly documents that mid-run reloads previously delayed the user message card until the first LLM delta.

- **Deferred reload test coverage:** Node tests cover the exact replay path: a `session.message` event during an active run does not reload immediately, then a terminal chat event replays the deferred history reload.
