## Summary

The PR is superseded by current main and the already-closed linked bug #69648. OpenClaw now solves the MiniMax M2.7 image-tool failure by routing MiniMax chat primaries to the plugin-owned MiniMax-VL-01 media provider, not by treating MiniMax-M2.7 itself as the VLM model. The PR's allowlist-only change conflicts with that shipped design and does not address the Greptile concern that the VLM request body has no model selector.

## Best Possible Solution

Close this PR as superseded. Keep the shipped MiniMax routing: M2.7 remains the chat/reasoning model, and image understanding uses the plugin-owned MiniMax-VL-01 media provider. Any future first-class MiniMax-M2.7 image-model change should be a new compatibility discussion backed by MiniMax API docs or live proof showing how the VLM endpoint selects that model.

## Risks / Open Questions

- Do not merge the PR as-is: it broadens `isMinimaxVlmModel` to `MiniMax-M2.7` while the VLM request body still has no model selector, so it could silently keep using the endpoint default.
- Merging would also conflict with current docs/tests that intentionally keep MiniMax-M2.7 as a text chat model and use MiniMax-VL-01 for image understanding.
