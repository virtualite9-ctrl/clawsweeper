## Summary

Current main addresses both observable failure modes in #70104. The stale REM corpus path is fixed by live-source filtering before light/REM output and REM phase-signal reinforcement, matching the merged #71695 context. The zero-promotion path is also no longer gated only on raw recallCount/queryHashes: deep ranking uses total signal count plus recall-day diversity, and tests cover repeated dreaming-only signals and light/REM phase reinforcement.

## Best Possible Solution

Close as implemented on current main. Keep the merged memory-core stale-source filtering plus the existing phase-signal and total-signal deep ranking behavior. If the same symptom appears on a newer release, it should come back as a fresh issue with current version, config, DREAMS/phase artifacts, short-term recall and phase-signal excerpts, and affected workspace/plugin details.
