## Summary

Close #72054 as ClawHub/plugin-scope work. The PR is a large new bundled Fleet Orchestrator plugin stack, not a fix already shipped on main, and the current OpenClaw direction keeps optional capabilities and heavy orchestration layers out of core unless there is explicit maintainer product/security ownership. The current plugin API already exposes the needed external-plugin seams for CLI commands, HTTP routes, services/hooks, config, and subagent orchestration, so the best path is to publish and iterate this as an external ClawHub/npm plugin and file narrower SDK/API gaps only if the polling fallback proves insufficient.

## Best Possible Solution

Move the Fleet Orchestrator work out of `openclaw/openclaw` into an external native plugin package published through ClawHub or npm. Keep using the existing plugin APIs for config, CLI, HTTP routes, services/hooks, and subagent orchestration; only bring back narrowly-scoped OpenClaw core changes if an external implementation proves a generic plugin API is missing.
