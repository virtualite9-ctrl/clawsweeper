## Summary

Current main no longer preloads the plugin registry during bootstrap for the commands reported in #64306. Status, health, and gateway status are route-first without plugin preload; doctor bypasses bootstrap; skills uses the default no-plugin-preload policy. The related #64357 activationSourceConfig theory is also obsolete in current loader source because the bare diagnostic reference is gone.

## Best Possible Solution

Close #64306 as implemented on current main. Keep the route-first/no-plugin-preload startup policy and plugin-loader guard coverage. If Windows `hooks` commands still time out on current main, track that separately in #70451 with fresh current-version logs and a focused reproduction.
