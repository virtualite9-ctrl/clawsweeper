## Summary

Keep open. Current main partially fixes the direct `gateway install` path by parsing existing LaunchAgent `EnvironmentVariables` and preserving safe custom values when that existing environment is passed into the install plan, but the doctor repair path still rebuilds service plans without passing the existing service environment. That leaves the reported `doctor --repair` wipe risk for plist-only variables such as `ANTHROPIC_API_KEY`.

## Best Possible Solution

Keep the issue open. The likely fix is to thread `command.environment` or the parsed LaunchAgent environment into every doctor/service repair call to `buildGatewayInstallPlan`, especially `src/commands/doctor-gateway-services.ts`, and add regression coverage proving a LaunchAgent custom env var such as `ANTHROPIC_API_KEY` survives `doctor --repair`/service repair rewrites. The existing direct install preservation logic should remain intact.
