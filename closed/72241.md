## Summary

Current main already fixes the reported Bonjour/CIAO crash path. The bundled bonjour plugin now classifies `CIAO ANNOUNCEMENT CANCELLED` / `CIAO PROBING CANCELLED` as handled ciao cancellations, wires scoped rejection/exception handlers through the plugin runtime, keeps those handlers registered across advertiser watchdog restarts, and disables Bonjour after repeated stuck `probing`/`announcing` restarts instead of letting the Gateway crash.

## Best Possible Solution

Close this issue as implemented on main, keep the current scoped ciao handler and watchdog-recovery implementation, and let the next release carry it to users. Until then, affected v2026.4.24 users can keep Bonjour disabled via config or `OPENCLAW_DISABLE_BONJOUR=1`.
