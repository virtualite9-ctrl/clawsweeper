## Summary

Current main already covers the reported stale-version-after-update failure mode. Package updates now verify the installed package and the restarted Gateway version instead of reporting success while the running service still reports the old version; the regression test explicitly covers an old 2026.4.23 Gateway remaining after a 2026.4.24 package update.

## Best Possible Solution

Close this issue as fixed on main and keep the shipped update behavior: package updates must verify both the installed package and the restarted Gateway-reported version. If a future user still sees an old version after updating, it should be filed as a new issue with update logs, Node version, install path, and `openclaw gateway status --deep` output.
