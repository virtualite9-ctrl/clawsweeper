## Summary

Current `main` no longer follows the reported eager-load path for `openclaw status`, and the doctor/security path is now scoped to configured channels rather than all channel-capable plugins. I could not determine an exact fixing commit or a tagged release that first shipped it from the available shallow history and inspected release notes.

## Evidence

- **Status route no longer preloads plugins:** The routed `status` command is configured with `loadPlugins: "never"`, so `openclaw status` does not bootstrap the channel plugin registry before running.

  - file: [src/commands/status.scan.test.ts:108](https://github.com/openclaw/openclaw/blob/ecac696643c604c6d854341bfe6424bfa31423e9/src/commands/status.scan.test.ts#L108)
