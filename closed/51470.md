## Summary

Current main already solves the observable managed-gateway proxy failure through a broader Gateway env-proxy bootstrap. The exact PR change, forwarding `NODE_USE_ENV_PROXY`, is not present, but managed services still forward `HTTP_PROXY`/`HTTPS_PROXY`, and Gateway startup now installs Undici's `EnvHttpProxyAgent` before Discord startup/provider/plugin network requests run.

## Best Possible Solution

Close PR #51470 as implemented on current main by the broader Gateway env-proxy dispatcher bootstrap. Keep the shipped/current-main approach rather than adding a second Node-specific service env flag unless a future report shows a pre-bootstrap process still needs `NODE_USE_ENV_PROXY` specifically.
