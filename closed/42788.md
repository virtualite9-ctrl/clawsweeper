## Summary

Current main already addresses the TUI-side user problem: the always-visible TUI footer shows the active session model from session state, gateway session rows resolve model overrides before exposing `modelProvider`/`model`, and stored session `label` values are promoted into `displayName` for TUI display. The Control UI portion was separately fixed by #45130, as noted in the discussion.

## Best Possible Solution

Close as implemented. Keep the shipped gateway `sessions.list` and TUI footer/session-picker behavior; if maintainers want the top header to also mirror `displayName`, that should be a narrower polish issue rather than keeping this broader active-model/session-label report open.
