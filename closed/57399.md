## Summary

- Codex review of current `main` found the OpenAI Codex OAuth refresh path now reloads fresh on-disk credentials inside the locked refresh flow and retries once after `refresh_token_reused`, instead of getting stuck on stale cached auth state (`src/agents/auth-profiles/oauth-manager.ts:315`, `src/agents/auth-profiles/oauth-manager.ts:528`).

- The current changelog records this exact OpenAI Codex OAuth fix on `main` (`CHANGELOG.md:1113`).
