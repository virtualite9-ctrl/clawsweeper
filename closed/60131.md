## Summary

Keep #60131 open. Current main partially improved TTS directive routing by prioritizing the declared provider, but it still deliberately falls through to other providers for unsupported keys and does not expose `selectedProvider` in the directive parse context, so the stricter provider-aware warning/drop behavior requested by the issue remains unimplemented.

## Best Possible Solution

Keep the issue open and resolve the remaining parser contract deliberately: when `provider=` is declared, either dispatch non-provider tokens only to that provider and warn on unsupported keys, or document and confirm the current cross-provider fallthrough as intended. If strict routing is chosen, update `src/tts/directives.ts`, extend `SpeechDirectiveTokenParseContext` with selected-provider metadata if providers need it, and replace the current fallthrough tests with regression coverage for warning/drop behavior.
