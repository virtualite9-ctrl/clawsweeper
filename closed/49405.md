## Summary

Current main already contains the gateway-handshake mitigation that matches this report: non-JSON CLI calls still use `withProgress`, but the gateway client now yields to the event loop before starting the WebSocket handshake, and the shared pre-auth handshake timeout is 10s rather than the reported 3s. The same implementation is present in the shipped `v2026.4.22` release.

## Evidence
