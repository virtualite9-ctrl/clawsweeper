## Summary

Current main already implements the cron isolated run-session cleanup this PR reworks: `cron.sessionRetention` defaults to 24h, `sweepCronRunSessions` prunes expired `...:cron:<jobId>:run:<uuid>` session-store entries while preserving the base cron session, and `onTimer` invokes the reaper from `finally`. Tests, docs, and changelog/release evidence cover the shipped behavior. The PR branch targets the same run-session surface and existing review comments identify regressions, including failed/log-rotated runs no longer being reclaimed and non-English comments.

## Best Possible Solution

Close this PR and keep the shipped `cron.sessionRetention` reaper. If maintainers want extra protection for active child tasks, pending execs, or embedded runs, the best path is a smaller follow-up PR with a reproduced current-main failure, no hard dependency on retained run-log rows, English comments, and regression coverage for failed and log-rotated runs.
