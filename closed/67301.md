## Summary

Current main already preserves user-set session labels across the relevant lifecycle paths. The Control UI writes `label` through `sessions.patch`; the gateway persists it as top-level `SessionEntry.label`; derived/system labels live separately as `origin.label`/`displayName`; compaction and restart recovery update existing entries without rebuilding or clearing `label`. The same behavior is present in the latest release tag `v2026.4.24`.

## Best Possible Solution

Close this issue and keep the current split between explicit user labels (`SessionEntry.label`) and derived/system labels (`origin.label`/`displayName`). If a future reporter still sees label loss, it should be opened as a fresh regression with the exact session store entry before/after compaction or restart and the affected channel/plugin path.
