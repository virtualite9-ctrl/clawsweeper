## Summary

Close PR #50743. Current main already sends OpenRouter attribution headers through the provider stream path, the linked #50738 was closed as implemented with the broader missing-header/auth path covered, and OpenRouter’s current docs now document `X-OpenRouter-Title` plus `X-OpenRouter-Categories` rather than requiring the PR’s `X-Title` replacement. The PR also includes an unrelated Azure Foundry transport change that current main explicitly avoids and tests against.

## Best Possible Solution

Do not merge PR #50743. Keep the shipped OpenRouter attribution implementation that uses the current documented OpenRouter headers and forwards them through the provider-owned stream path. If any new billing issue appears, open a fresh issue with live OpenRouter request/dashboard evidence; the unrelated Azure Foundry transport change should remain out of this PR and continue to be handled by the current compatibility-preserving onboarding code.
