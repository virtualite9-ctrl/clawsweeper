## Summary

Current main and the shipped v2026.4.24 release already address the Windows local gateway partial-RPC failure mode reported against v2026.4.9. The shared gateway RPC path now yields before WebSocket start to avoid Windows connect.challenge starvation, suppresses intentional teardown closes, waits for WebSocket teardown, and keeps pending long-running RPCs such as ACP agent.wait from being closed by the tick watchdog. The reported sessions_history and ACP spawn/wait surfaces use that fixed callGateway path.

## Best Possible Solution

Close #64476 as implemented on main and shipped in v2026.4.24. Keep the current shared gateway RPC implementation; any remaining request for different timeout tuning should be handled in a narrower follow-up rather than keeping this v2026.4.9 Windows partial-RPC report open.
