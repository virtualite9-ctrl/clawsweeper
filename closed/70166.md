## Summary

Current main fixes the reported Control UI regression for workspace-relative local avatars, including root-level PNG files in the agent workspace. The backend resolves `IDENTITY.md`/config avatar values relative to the workspace, exposes local avatars through `/avatar/<agentId>` metadata, and the frontend fetches that authenticated same-origin route into a blob URL for chat/welcome rendering. The linked #70700 described the same frontend inconsistency and is now covered by current source and tests.

## Best Possible Solution

Close #70166 as fixed on current main. Keep the shipped direction: workspace-relative avatar paths, including root-level files, remain supported; Control UI should continue rendering local avatars through the resolved `/avatar/<agentId>` route and authenticated blob URLs rather than raw workspace paths. No docs change is needed to require an `avatars/` subdirectory because that is only an example path.

## Evidence

A root-level `avatar.png` is therefore a valid workspace-relative path; an `avatars/` subdirectory is not required by the resolver.
