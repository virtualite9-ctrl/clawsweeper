## Summary

The requested `message:preprocessed` mutation seam is not a current-main bug in OpenClaw core; main deliberately treats internal message hooks as fire-and-forget observation, while supported per-turn context injection lives in plugin hooks (`before_prompt_build`). The exact semantic-recall use case is already implemented by existing plugins such as Active Memory and `memory-lancedb`, so this belongs in the plugin/ClawHub surface rather than as a new mutable core internal-hook contract.

## Evidence

- **Internal `message:preprocessed` hooks are emitted asynchronously from copied context:** `emitPreAgentMessageHooks` derives a canonical snapshot from the finalized inbound message, then fires `message:transcribed` and `message:preprocessed` through `fireAndForgetHook(triggerInternalHook(...))`. There is no synchronous path that would feed hook mutations back into the live reply context before the agent runs.
