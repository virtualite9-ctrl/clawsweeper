## Summary

Keep open. Current `main` still ships the Bonjour surface through `@homebridge/ciao`, and the upstream `@homebridge/ciao@1.3.6` code still uses the Windows `arp -a | findstr` exec path without `windowsHide`, so this is not already implemented on `main`. The submitted PR also is not closeable as cleanup: it targets `@homebridge/ciao@1.3.5` while `main` is on `1.3.6`, and the diff includes unrelated `tasks/` monitoring artifacts that need to be removed before maintainers can evaluate the actual fix cleanly.
