## Summary

Keep this PR open. Current main still hardcodes the MiniMax VLM image-understanding request timeout at 60 seconds and does not thread the already-resolved image timeout into the MiniMax direct or fallback paths. The PR’s core timeout change remains relevant to #67889, though the unrelated iMessage test in the branch should be split out or removed before merge.

## Best Possible Solution

Keep the PR open and have maintainers review the MiniMax timeout propagation as a valid fix candidate for #67889. The best path is to ask the contributor to remove or split the unrelated iMessage test, then land the focused MiniMax changes with regression coverage for `timeoutMs` reaching `AbortSignal.timeout`; alternatively, a maintainer can cherry-pick the focused timeout patch and leave the iMessage coverage for a separate PR.
