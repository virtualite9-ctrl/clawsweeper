## Summary

Current main already covers PR #67748's remote CDP resilience goal, and does so in a safer shape than the PR: raw CDP WebSocket retries happen only during handshake before any CDP command/callback is replayed, while stale persistent Playwright CDP sessions get a bounded reconnect for safe tab-list reads. The changelog records this as fixing #67728, and regression tests cover handshake retry, no 429 retry, no mid-command callback replay, safe read reconnect, and no mutating page-creation replay.

## Best Possible Solution

Close this PR as implemented on current main and keep the shipped safer design: handshake-only raw CDP retry plus safe persistent Playwright read reconnect. Do not merge the PR's broader callback-retry shape because it can replay mutating CDP commands; any remaining remote CDP failures should be reopened as a fresh, narrower issue with current-main reproduction details.
