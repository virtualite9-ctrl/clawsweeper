## Summary

Current main already enforces stable marker-based persistence for env-backed provider secrets in models generation, and the PR's proposed switch to resolved plaintext values would regress shipped secret-persistence hardening.

## Evidence

- **Resolved env apiKeys are intentionally normalized back to env markers:** `normalizeProviders()` still calls `normalizeResolvedEnvApiKey()`, and that helper rewrites a resolved secret value back to the canonical env var name when it matches the provider's known env key.
  - file: [src/agents/models-config.providers.normalize.ts:77](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/src/agents/models-config.providers.normalize.ts#L77)
  - sha: [4013c658537e](https://github.com/openclaw/openclaw/commit/4013c658537ee2051c44186f197b0710e84c9049)
- **Reverse-lookup implementation on main:** `normalizeResolvedEnvApiKey()` returns `{ ...provider, apiKey: envVarName }` and marks the provider as secret-managed when the current plaintext value matches the env value.
  - file: [src/agents/models-config.providers.secret-helpers.ts:245](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/src/agents/models-config.providers.secret-helpers.ts#L245)
  - sha: [4013c658537e](https://github.com/openclaw/openclaw/commit/4013c658537ee2051c44186f197b0710e84c9049)
- **Source snapshot enforcement keeps generated models.json on marker form:** `enforceSourceManagedProviderSecrets()` derives marker values from the source config SecretRef and rewrites runtime-resolved provider apiKeys back to markers during models planning.
  - file: [src/agents/models-config.providers.source-managed.ts:73](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/src/agents/models-config.providers.source-managed.ts#L73)
  - sha: [4013c658537e](https://github.com/openclaw/openclaw/commit/4013c658537ee2051c44186f197b0710e84c9049)
- **Merge mode does not preserve manually edited plaintext apiKeys for secret-managed providers:** `mergeWithExistingProviderSecrets()` only preserves an existing apiKey when the provider is not in `secretRefManagedProviders`, so a manual plaintext edit is intentionally not kept for env/SecretRef-managed providers.
  - file: [src/agents/models-config.merge.ts:182](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/src/agents/models-config.merge.ts#L182)
