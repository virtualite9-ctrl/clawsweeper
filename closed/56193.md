## Summary

Close as superseded by the focused open PR #68397. The bug is not fixed on main: current executor code still exposes `streamTo` when ACP is available and still rejects `runtime="subagent"` calls that include it. #68397 is the canonical implementation tracker for the exact failure mode reported here, while broader schema filtering remains tracked separately in #59225.

## Best Possible Solution

Use #68397 as the canonical implementation tracker for this specific bug: land an equivalent executor change that ignores or strips `streamTo` and `resumeSessionId` for non-ACP `sessions_spawn` calls while preserving ACP pass-through and tests. Keep broader per-agent/runtime-specific schema filtering in #59225 rather than expanding this issue.
