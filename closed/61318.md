## Summary

Current main no longer has the stale fallback-chain behavior described for 2026.4.2. Gateway-visible agent metadata and runtime fallback execution both derive fallbacks from the active `agents.defaults.model.fallbacks` config unless an explicit per-agent override exists, and the current `openclaw status` Sessions table no longer renders a cached fallback-chain value.
