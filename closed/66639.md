## Summary

Current `main` already implements MiniMax image-tool/media-understanding support by registering MiniMax media providers, auto-pairing MiniMax chat primaries with `MiniMax-VL-01`, and routing that vision model through MiniMax's dedicated VLM endpoint. The latest tagged release predates this work, so the fix is on `main` but not yet in `v2026.4.23`.

## Evidence

- **MiniMax plugin now registers media-understanding providers:** The bundled MiniMax plugin registers both `minimax` and `minimax-portal` media-understanding providers in addition to its other provider surfaces.
  - file: [extensions/minimax/index.ts:20](https://github.com/openclaw/openclaw/blob/cc0f3e0e40364023cc3a39227a411898ee68ce3b/extensions/minimax/index.ts#L20)
  - sha: [cc0f3e0e4036](https://github.com/openclaw/openclaw/commit/cc0f3e0e40364023cc3a39227a411898ee68ce3b)
- **MiniMax media providers define image support and default model:** Both MiniMax media-understanding providers advertise `capabilities: ["image"]` and default image model `MiniMax-VL-01`.
  - file: [extensions/minimax/media-understanding-provider.ts:7](https://github.com/openclaw/openclaw/blob/cc0f3e0e40364023cc3a39227a411898ee68ce3b/extensions/minimax/media-understanding-provider.ts#L7)
  - sha: [cc0f3e0e4036](https://github.com/openclaw/openclaw/commit/cc0f3e0e40364023cc3a39227a411898ee68ce3b)
- **Image tool auto-pairs MiniMax chat models with MiniMax vision model:** When no explicit `agents.defaults.imageModel` is set, the image tool resolves the provider default media model and includes a MiniMax-specific fallback to `${provider}/MiniMax-VL-01`.
  - file: [src/agents/tools/image-tool.ts:126](https://github.com/openclaw/openclaw/blob/cc0f3e0e40364023cc3a39227a411898ee68ce3b/src/agents/tools/image-tool.ts#L126)
  - sha: [cc0f3e0e4036](https://github.com/openclaw/openclaw/commit/cc0f3e0e40364023cc3a39227a411898ee68ce3b)
- **Tests cover the MiniMax image-tool pairing behavior:** Tests assert that a primary `minimax/MiniMax-M2.7` config resolves image handling to `minimax/MiniMax-VL-01`, and that `minimax-portal/MiniMax-M2.7` resolves to `minimax-portal/MiniMax-VL-01`.
  - file: [src/agents/tools/image-tool.test.ts:624](https://github.com/openclaw/openclaw/blob/cc0f3e0e40364023cc3a39227a411898ee68ce3b/src/agents/tools/image-tool.test.ts#L624)
  - sha: [cc0f3e0e4036](https://github.com/openclaw/openclaw/commit/cc0f3e0e40364023cc3a39227a411898ee68ce3b)
- **MiniMax image understanding routes to the dedicated VLM endpoint:** The media-understanding runtime detects canonical MiniMax vision models and calls `minimaxUnderstandImage`; that helper sends requests to `/v1/coding_plan/vlm`, matching the issue's reported routing gap.
  - file: [src/media-understanding/image.ts:241](https://github.com/openclaw/openclaw/blob/cc0f3e0e40364023cc3a39227a411898ee68ce3b/src/media-understanding/image.ts#L241)
  - sha: [cc0f3e0e4036](https://github.com/openclaw/openclaw/commit/cc0f3e0e40364023cc3a39227a411898ee68ce3b)
- **Endpoint behavior is exercised by tests:** Tests verify that `describeImageWithModel` for `minimax-portal/MiniMax-VL-01` calls `https://api.minimax.io/v1/coding_plan/vlm` with MiniMax auth and payload.
  - file: [src/media-understanding/image.test.ts:98](https://github.com/openclaw/openclaw/blob/cc0f3e0e40364023cc3a39227a411898ee68ce3b/src/media-understanding/image.test.ts#L98)

- **Docs now describe the intended MiniMax split:** The MiniMax docs state that `MiniMax-M2.7` and `MiniMax-M2.7-highspeed` remain text-only chat models, while image understanding is exposed separately through the plugin-owned `MiniMax-VL-01` media provider.
  - file: [docs/providers/minimax.md:238](https://github.com/openclaw/openclaw/blob/cc0f3e0e40364023cc3a39227a411898ee68ce3b/docs/providers/minimax.md#L238)

## Risks / Open Questions

- The latest release `v2026.4.23` predates this implementation, so released builds may still reproduce the old behavior until the next ship.
- Users who explicitly override `agents.defaults.imageModel` to `minimax/MiniMax-M2.7` will still need to change that override to `minimax/MiniMax-VL-01`; the current fix treats M2.7 as text-only and routes image understanding through the dedicated vision model.
