## Summary

Keep open. Current main still turns a `doctor.memory.status` timeout/unavailable error into a checked, not-ready memory probe, and the explicit local memory provider path still emits the reported Memory search warning when that probe is not ready. The related fix PR #49314 was closed unmerged, while newer #68171 was closed specifically as a duplicate of this canonical issue, not as implemented.

## Best Possible Solution

Keep #44426 open as the canonical bug. The likely fix belongs in the doctor gateway-memory probe path: retry/back off or treat `doctor.memory.status` timeout/unavailable errors as inconclusive for explicit local embeddings, while preserving warnings for concrete local readiness failures such as missing runtime dependencies. Add focused regression coverage for non-interactive `openclaw doctor` with local memory where the short probe times out but `openclaw memory status --deep` can report ready.
