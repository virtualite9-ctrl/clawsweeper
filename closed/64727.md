## Summary

Current main already implements the requested subagent announce precedence: output extraction tracks assistant and raw tool text separately and returns the latest visible assistant text before any raw tool/toolResult fallback. The current chat.history path also takes the recent tail of visible messages, and regression tests cover both normal announce and completion-mode cases where tool output is present but assistant output must win.

## Best Possible Solution

Keep the shipped assistant-first announce selection and its regression coverage. If a future report shows a different long-transcript failure, it should include a current-version transcript proving the final assistant message is absent from the recent chat.history tail despite being the child’s final visible reply.
