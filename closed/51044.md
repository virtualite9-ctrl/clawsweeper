## Summary

Keep open. Current main has newer Ollama plugin/discovery synthetic-auth paths, but the exact configless auth-resolver behavior targeted by this PR is not clearly implemented: `resolveSyntheticLocalProviderAuth()` still returns `null` when no provider config exists, and the Ollama plugin only synthesizes auth when it receives meaningful explicit provider config. The PR also has unresolved review feedback about missing tests, a misleading source label, and explicit non-api-key auth overrides.

## Best Possible Solution

Keep the PR open, but require a narrower rework before merge: add focused auth tests for `resolveApiKeyForProvider({ provider: "ollama", cfg: {}, store: empty })`, preserve the normal missing-auth behavior for unknown providers, respect explicit non-api-key auth overrides when an Ollama provider config exists, use a source label that does not imply `models.providers.ollama` exists, and update related auth-introspection/status paths if they still report configless Ollama as missing auth.
