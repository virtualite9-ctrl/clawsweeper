## Summary

Current main already applies the baseline security headers to the documented Canvas/A2UI HTTP surface at the gateway layer, so this PR's user-visible security change is already covered. The remaining delta is limited to direct-handler/standalone test paths that are not part of the current shipped gateway runtime.

## Evidence

- **Docs define Canvas/A2UI as gateway HTTP routes:** The public configuration docs say Canvas and A2UI are served under the Gateway port at `/__openclaw__/canvas/` and `/__openclaw__/a2ui/`.
  - file: [docs/gateway/configuration-reference.md:601](https://github.com/openclaw/openclaw/blob/459d277076488d5833a181cca7d8bf5c0f9696dc/docs/gateway/configuration-reference.md#L601)
  - sha: [459d27707648](https://github.com/openclaw/openclaw/commit/459d277076488d5833a181cca7d8bf5c0f9696dc)
- **Gateway applies default security headers before routing:** `createGatewayHttpServer()` calls `setDefaultSecurityHeaders(res, ...)` at the top of `handleRequest()` before any route dispatch.
  - file: [src/gateway/server-http.ts:920](https://github.com/openclaw/openclaw/blob/459d277076488d5833a181cca7d8bf5c0f9696dc/src/gateway/server-http.ts#L920)
  - sha: [459d27707648](https://github.com/openclaw/openclaw/commit/459d277076488d5833a181cca7d8bf5c0f9696dc)
- **Canvas/A2UI requests are routed after header application:** After the global header call, the gateway dispatches A2UI and canvas requests via the `a2ui` and `canvas-http` stages.
  - file: [src/gateway/server-http.ts:1064](https://github.com/openclaw/openclaw/blob/459d277076488d5833a181cca7d8bf5c0f9696dc/src/gateway/server-http.ts#L1064)
  - sha: [459d27707648](https://github.com/openclaw/openclaw/commit/459d277076488d5833a181cca7d8bf5c0f9696dc)
- **Gateway runtime mounts canvas through the shared handler:** The live gateway runtime creates the canvas handler with `createCanvasHostHandler(...)` and mounts it on the gateway server.
  - file: [src/gateway/server-runtime-state.ts:116](https://github.com/openclaw/openclaw/blob/459d277076488d5833a181cca7d8bf5c0f9696dc/src/gateway/server-runtime-state.ts#L116)
  - sha: [459d27707648](https://github.com/openclaw/openclaw/commit/459d277076488d5833a181cca7d8bf5c0f9696dc)
- **Baseline helper is explicitly intended to be safe for canvas/A2UI:** `setDefaultSecurityHeaders()` is documented as safe for response types including canvas host and A2UI because it intentionally omits framing/CSP restrictions.
  - file: [src/gateway/http-common.ts:9](https://github.com/openclaw/openclaw/blob/459d277076488d5833a181cca7d8bf5c0f9696dc/src/gateway/http-common.ts#L9)
  - sha: [459d27707648](https://github.com/openclaw/openclaw/commit/459d277076488d5833a181cca7d8bf5c0f9696dc)

  - file: [src/canvas-host/server.test.ts:239](https://github.com/openclaw/openclaw/blob/459d277076488d5833a181cca7d8bf5c0f9696dc/src/canvas-host/server.test.ts#L239)

## Risks / Open Questions

- This close assumes maintainer scope is the documented gateway-served Canvas/A2UI surface; direct use of the internal `startCanvasHost()` helper would still not get these headers on current main.
- If OpenClaw later treats the standalone canvas server as a supported external surface, this hardening could be worth reviving separately.
