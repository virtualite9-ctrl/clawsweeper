## Summary

Current main already implements the consolidated local outbound media fix: normal assistant MEDIA replies are normalized and staged into managed outbound media, media delivery resolves agent-scoped/local-source roots, Telegram and WhatsApp reply paths pass those roots into media loading, and WebChat/Control UI image uploads are carried as structured attachments or managed inbound media refs. The remaining retry-classification improvement is narrower and already tracked by #66489.

## Best Possible Solution

Close this consolidation issue as implemented. Keep the shipped staging/agent-scoped media access design, and leave the narrower retry-classification request to #66489 if maintainers still want that queue behavior change.
