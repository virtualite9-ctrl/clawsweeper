## Summary

Current main implements the overflow recovery path without the self-queueing shape described in #61290. Overflow recovery now increments the attempt, builds compaction runtime context, and calls the active context engine directly; the legacy engine delegates to the direct no-lane compaction helper, which is timeout/abort wrapped. The relevant compaction/context-engine fixes are present in the v2026.4.24 release tree after the reported v2026.4.21 reproduction.

## Best Possible Solution

Keep the shipped direct/context-engine overflow compaction path. If similar hangs occur on v2026.4.24 or newer, track them as a fresh bug with a current log slice including [context-overflow-diag], [compaction-diag], final timeout/failover lines, provider/model, OpenClaw version, and compaction config overrides.
