## Summary

Current main already supports the requested persistent local browser size override via `browser.extraArgs`. The exact field name is not `browser.viewport`, but the config type explicitly documents window size overrides such as `--window-size=1920,1080`, the schema accepts `extraArgs`, the browser config resolver preserves it, and the managed Chromium launcher appends those flags on startup. Existing docs and tests cover the config/launch path, and the changelog records the feature in 2026.2.17.

## Best Possible Solution

Close this as implemented and keep the shipped `browser.extraArgs` path as the conservative solution. If maintainers later want a higher-level `browser.viewport` alias, it should be tracked as a narrower ergonomics/docs enhancement rather than keeping this user-blocking persistent sizing request open.
