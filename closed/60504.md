## Summary

Current main resolves the skills status path from the active profile state directory, and the Mission Control Skills UI no longer renders the separate gateway-path label described in the report.

## Evidence

- **Profile bootstrapping sets a profile-specific state dir:** `applyCliProfileEnv()` sets `OPENCLAW_PROFILE`, derives `stateDir` as `~/.openclaw-<profile>` for non-default profiles, and populates both `OPENCLAW_STATE_DIR` and `OPENCLAW_CONFIG_PATH`. That is the source of profile-aware path resolution for `--profile new` style launches.
  - file: [src/cli/profile.ts:72](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/src/cli/profile.ts#L72)
  - command: `nl -ba src/cli/profile.ts | sed -n '63,99p'`
  - sha: [926068b14fc6](https://github.com/openclaw/openclaw/commit/926068b14fc661e915a0247e9ecb5949fb95b4f7)
- **Config dir resolution follows the active state dir:** `resolveConfigDir()` prefers `OPENCLAW_STATE_DIR`, and `CONFIG_DIR` is exported from that helper. This means downstream path joins use the active profile root instead of a hardcoded `~/.openclaw` directory.
  - file: [src/utils.ts:136](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/src/utils.ts#L136)
  - command: `nl -ba src/utils.ts | sed -n '136,214p'`
  - sha: [926068b14fc6](https://github.com/openclaw/openclaw/commit/926068b14fc661e915a0247e9ecb5949fb95b4f7)
- **Skill status report derives managed skills path from CONFIG_DIR:** `buildWorkspaceSkillStatus()` computes `managedSkillsDir` as `path.join(CONFIG_DIR, "skills")`, so the gateway-side skills path follows the active profile state dir once profile env is applied.
  - file: [src/agents/skills-status.ts:228](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/src/agents/skills-status.ts#L228)
  - command: `nl -ba src/agents/skills-status.ts | sed -n '228,254p'`
  - sha: [926068b14fc6](https://github.com/openclaw/openclaw/commit/926068b14fc661e915a0247e9ecb5949fb95b4f7)
- **Current Mission Control Skills view does not render the reported gateway path label:** The runtime Skills view renders the card, filters, ClawHub area, groups, and detail dialog, but does not consume `report.managedSkillsDir` or `report.workspaceDir`. A repo search under `ui/src/ui` finds those fields only in types/tests, not in the rendered Mission Control view.
  - file: [ui/src/ui/views/skills.ts:102](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/ui/src/ui/views/skills.ts#L102)

- **Latest release already contains the same relevant code paths:** A diff of the relevant files between the latest release tag and current main is empty for `src/cli/profile.ts`, `src/utils.ts`, `src/agents/skills-status.ts`, and `ui/src/ui/views/skills.ts`, so the shipped `v2026.4.23` release already includes the reviewed behavior.
