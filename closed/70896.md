## Summary

Current main already implements effective file-log rotation for the reported gateway logging problem: long-lived rolling loggers re-resolve the current-day file on each write, writes use fresh append calls rather than a persistent stream target, size-based rotation is supported via `logging.maxFileBytes`, and tests/docs cover the behavior.

## Best Possible Solution

Close #70896 as implemented on main, keep the shipped logger behavior, and let the fix ride the next release. If maintainers want configurable retention beyond the current five numbered archives, that should be a narrower follow-up rather than keeping this stale-file-target issue open.
