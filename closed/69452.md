## Summary

PR #69452 is a docs-only, service-specific Anthropic-compatible proxy recipe for aiusage. Current main already has the core Anthropic baseUrl behavior and public provider/plugin surfaces needed to support this outside core, while the PR discussion identifies promotional and accuracy issues in the proposed text. The best maintainer cleanup is to close this as ClawHub/external integration scope rather than merge a commercial proxy example into core docs.

## Best Possible Solution

Close this PR as ClawHub/external integration scope. Keep core docs focused on generic provider/baseUrl and plugin extension points; maintain the aiusage-specific setup guide in aiusage docs or a ClawHub-published provider/plugin package. If maintainers want a core docs improvement, it should be a separate neutral Anthropic-compatible baseUrl note that removes the commercial claim and accurately describes API-key, OAuth-token, and Claude CLI runtime behavior.
