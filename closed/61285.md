## Summary

Issue #61285 is a specific Simplified Chinese localization request for the native macOS menu/settings app. Current main still lacks native macOS localization resources and still contains hardcoded English SwiftUI labels, so this is not implemented-on-main. However, the maintainer-authored canonical i18n/localization tracker #3460 explicitly says specific-language localization issues should be closed as duplicates and lists Chinese (Simplified) / zh-CN; the timeline-linked #53402 is the same macOS Chinese localization request and is already closed.

## Best Possible Solution

Close #61285 as a language-specific duplicate/superseded item under maintainer canonical tracker #3460. If maintainers restart native app localization, the best implementation is a first-class macOS localization architecture covering SwiftUI string keys, .lproj or String Catalog resources, localized InfoPlist permission strings, menu/settings/onboarding coverage, and tests; Voice Wake availability should be handled separately if there is a supported-version reproduction.
