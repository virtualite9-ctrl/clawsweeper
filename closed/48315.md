## Summary

Current main already gives plugins access to the standard outbound delivery pipeline through the supported `openclaw/plugin-sdk/outbound-runtime` SDK subpath, so the functional need behind this PR is already implemented. The PR's specific `api.runtime.outbound` shape is still absent, and the review note about lazy runtime reflection remains valid, so landing this exact patch is no longer the best path.
