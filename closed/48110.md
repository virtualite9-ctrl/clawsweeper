## Summary

Current `main` already resolves the active session row's model through the per-agent path before the TUI status controls read it, so the user-visible bug described in PR #48110 is already implemented by a different code path and shipped in `v2026.4.23`.

## Evidence

- **Per-agent default resolver exists in core model selection:** `resolveDefaultModelForAgent` overlays an agent-specific primary model onto config defaults before delegating to `resolveConfiguredModelRef`, so agent overrides are part of the supported resolution path.
  - file: [src/agents/model-selection.ts:199](https://github.com/openclaw/openclaw/blob/151befb90b903f52dce9f4951991d39b9df8b358/src/agents/model-selection.ts#L199)

- **Session model resolution already uses per-agent defaults:** `resolveSessionModelRef` calls `resolveDefaultModelForAgent({ cfg, agentId })` whenever an agent id is available, which is the same behavioral goal described in the PR.
  - file: [src/gateway/session-utils.ts:1065](https://github.com/openclaw/openclaw/blob/151befb90b903f52dce9f4951991d39b9df8b358/src/gateway/session-utils.ts#L1065)
