## Summary

Current main implements the reported Ollama image-tool route. The exact manifest-only PR #64146 did not land, but the observable failure is now covered: Ollama registers an image-capable media-understanding provider at runtime, the image runtime registers the Ollama provider stream before completion, Ollama discovery marks `/api/show` vision models as image-capable, and the changelog documents the fix as shipped in v2026.4.21.

## Best Possible Solution

Close #64111 as implemented on main. Keep the shipped runtime registration and provider-stream route; any future Ollama-specific vision failures should be reopened as a fresh issue with current version, exact config, model metadata, and logs because the original missing-registration failure is no longer the current product behavior.
