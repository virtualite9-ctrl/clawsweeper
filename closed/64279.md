## Summary

Close PR #64279 as ClawHub/external-plugin scope. The Vidu provider work is useful and appears intentionally self-contained, but current main already exposes the provider and video-generation plugin seams needed to ship it outside core, and the project direction routes optional provider plugins to ClawHub/npm unless there is a clear maintainer-owned reason to bundle them.

## Best Possible Solution

Do not merge Vidu into core from this PR. The best path is to package the self-contained Vidu implementation as an external OpenClaw plugin on ClawHub or npm using the existing `registerProvider`, `registerVideoGenerationProvider`, provider-auth, and manifest contract surfaces; open a focused SDK/API PR only if that external packaging reveals a missing generic plugin seam.
