## Summary

Keep open. Current `main` still depends on `@mariozechner/pi-ai` for OpenAI Codex browser OAuth, and the pinned current dependency version still hardcodes the OAuth scope as `openid profile email offline_access` without `model.request`. The latest shipped release `v2026.4.23` also pinned a still-broken `pi-ai` version, and the linked PR #56703 was closed without merge.
