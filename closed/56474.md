## Summary

PR #56474 should close as superseded by the later canonical PR #56574. Current main still does not ship the pluginConfig-in-tool-factory-context behavior, but #56574 explicitly tracks and integrates that same remaining change alongside the related plugin registry fixes. The linked Memory Crystal use case from #56432 was useful context, but the remaining core work should be consolidated on #56574 rather than maintained as this older overlapping PR.

## Best Possible Solution

Close this older overlapping PR and consolidate any remaining pluginConfig/tool-factory-context work on #56574 or a maintainer-approved narrow follow-up rebased onto current main. Keep the shipped registration-time api.pluginConfig contract unless maintainers choose to add pluginConfig to OpenClawPluginToolContext.
