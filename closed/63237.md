## Summary

Current main already provides the explicit cross-channel exec-approval routing requested here: `approvals.exec.mode` supports `targets` and `both`, `approvals.exec.targets` defines session-independent delivery surfaces, and Telegram native approval routing is documented as a separate `channels.telegram.execApprovals.target` control.

## Evidence

## Risks / Open Questions

- This close is based on the existing explicit forwarding path (`approvals.exec.targets` / `mode: "targets"|"both"`), not on evidence of automatic owner-wide failover across every configured native approval client without explicit targets.
