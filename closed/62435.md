## Summary

Close as implemented on `main`: the exec runtime now suppresses late `onUpdate` emissions after a run exits, backgrounds, or aborts, and the shipped `v2026.4.23` changelog explicitly records this exact gateway crash signature as fixed.

## Evidence

- **Shipped fix note:** `v2026.4.23` changelog explicitly says OpenClaw now prevents the gateway crash with the exact message `Agent listener invoked outside active run` when a subagent exec tool emits stdout/stderr after the run has ended or been aborted.
  - file: [CHANGELOG.md:942](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/CHANGELOG.md#L942)
  - command: `git show a9797214338ba31c52c796adbb75afb16e0684a9:CHANGELOG.md | nl -ba | sed -n '934,946p'`
  - sha: [a9797214338b](https://github.com/openclaw/openclaw/commit/a9797214338ba31c52c796adbb75afb16e0684a9)
- **Runtime guard after run end:** The exec runtime now tracks settled runs with `updatesDisabled` and makes `emitUpdate()` a no-op once the session is backgrounded, exited, or updates have been disabled, directly covering the reported late-stdout/lifecycle race.
  - file: [src/agents/bash-tools.exec-runtime.ts:561](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/agents/bash-tools.exec-runtime.ts#L561)
  - command: `git show a9797214338ba31c52c796adbb75afb16e0684a9:src/agents/bash-tools.exec-runtime.ts | nl -ba | sed -n '561,586p'`
  - sha: [a9797214338b](https://github.com/openclaw/openclaw/commit/a9797214338ba31c52c796adbb75afb16e0684a9)
- **Abort-path suppression:** The exec tool now calls `run.disableUpdates()` immediately on abort before returning for yielded/backgrounded sessions, preventing late stdout/stderr from enqueueing `tool_execution_update` events after the agent run has ended.
  - file: [src/agents/bash-tools.exec.ts:1715](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/agents/bash-tools.exec.ts#L1715)
  - command: `git show a9797214338ba31c52c796adbb75afb16e0684a9:src/agents/bash-tools.exec.ts | nl -ba | sed -n '1715,1726p'`
  - sha: [a9797214338b](https://github.com/openclaw/openclaw/commit/a9797214338ba31c52c796adbb75afb16e0684a9)
- **Regression coverage:** Regression tests were added for the three relevant edges: no `onUpdate` after backgrounding, no `onUpdate` after foreground exit, and no `onUpdate` after abort.
  - file: [src/agents/bash-tools.test.ts:896](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/agents/bash-tools.test.ts#L896)
  - command: `git show a9797214338ba31c52c796adbb75afb16e0684a9:src/agents/bash-tools.test.ts | nl -ba | sed -n '896,969p'`
  - sha: [a9797214338b](https://github.com/openclaw/openclaw/commit/a9797214338ba31c52c796adbb75afb16e0684a9)
- **Issue discussion matches fix scope:** Both the original report and the follow-up field report narrowed the trigger to exec stdout arriving after the run lifecycle had already transitioned; the shipped fix is explicitly scoped to late exec stdout/stderr after run end/abort.
