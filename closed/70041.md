## Summary

Close #70041 as a duplicate of open issue #69422. The current main code still matches the reported Google Chat chunk-threading failure, but #69422 already tracks the same remaining Google Chat thread/reply routing metadata gap for streamed block replies and was recently kept open with source-level evidence.

## Best Possible Solution

Close #70041 as a duplicate and keep #69422 as the canonical fix discussion. The eventual fix should make Google Chat's thread resource a first-class reply/thread route for streamed block replies and queued follow-ups, then add a regression test proving chunks 2+ use spaces/.../threads/... rather than spaces/.../messages/....
