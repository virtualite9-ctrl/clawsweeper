## Summary

Current main already fixes the reported packaged-update failure class: runtime-dependency staging debris is excluded/rejected before publishing, postinstall pruning skips staged runtime-dependency directories instead of treating them as unsafe or stale packaged dist, and update verification now uses the package dist inventory to fail on genuinely missing/stale packaged files while ignoring runtime install stages.

## Best Possible Solution

Close #72042 as implemented on current main. Keep the shipped main behavior: exclude and reject bundled runtime-dependency staging debris at package time, skip staged runtime-dependency paths during postinstall pruning, verify real packaged dist files after updates, and leave Telegram-specific runtime dependency/Windows channel failures to their narrower open issues.
