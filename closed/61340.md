## Summary

Keep #61340 open. Current main has a useful partial mitigation for state-dir `.env` values: systemd units can point at a generated `gateway.systemd.env` file and omit matching dotenv values from inline `Environment=` lines. The reported forced-reinstall class is not fully fixed, though: reinstall still reuses existing non-`OPENCLAW_` service env such as `OPENAI_API_KEY`, non-dotenv values are still rendered inline, and the old unit is copied verbatim to `.bak` before rewrite.

## Best Possible Solution

Keep the issue open and fix the Linux systemd install contract directly. The best maintainer path is to ensure `gateway install --force` never writes secret-bearing service env inline, routes provider/auth-profile/config secrets through a least-exposure EnvironmentFile or SecretRef-compatible runtime source, and either sanitizes or skips secret-bearing `.bak` backups. Add regression coverage for forced reinstall with an existing inline `OPENAI_API_KEY`, a stale inline `OPENCLAW_GATEWAY_TOKEN`, dotenv-backed values, and backup creation.
