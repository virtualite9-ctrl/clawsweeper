## Summary

Close #71188 as a duplicate of open #70197. Both reports describe the same Windows gateway ARP discovery loop spawning `cmd.exe /d /s /c "arp -a | findstr /C:\"---\""` every ~15-16 seconds, and #70197 is the earlier, more root-cause-specific tracker for the bundled Bonjour/@homebridge/ciao path that current main still ships.

## Best Possible Solution

Close #71188 as a duplicate of #70197 and keep #70197 as the canonical active OpenClaw bug. The fix should land against the bundled Bonjour/ciao path, likely by upgrading or patching `@homebridge/ciao` to pass `windowsHide: true` for the Windows ARP probe, or by changing OpenClaw's Bonjour startup policy on Windows if maintainers prefer that product direction. Verification should include a Windows gateway run with Bonjour enabled and process-birth proof that no visible `cmd.exe`/`conhost.exe` window is spawned.
