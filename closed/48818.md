## Summary

Current `main` does not classify agent main sessions as `global`/`unknown`, and bare `main` aliases are normalized to the canonical main session key, so the dropdown bug described by PR #48818 does not match present behavior.

## Evidence

- **Dropdown filter only skips literal global/unknown rows:** `resolveSessionOptionGroups` adds every session row except non-active rows whose `kind` is exactly `global` or `unknown`, plus hidden cron rows.
  - file: [ui/src/ui/chat/session-controls.ts:498](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/ui/src/ui/chat/session-controls.ts#L498)
  - command: `nl -ba ui/src/ui/chat/session-controls.ts | sed -n '475,507p'`
  - sha: [01bc49c88c0c](https://github.com/openclaw/openclaw/commit/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd)
- **Agent-scoped main sessions are not marked global:** `classifySessionKey` returns `global` only for the literal key `global` and `unknown` only for the literal key `unknown`; all other non-group keys, including `agent:<id>:main`, fall through to `direct`. That makes the PR's proposed cross-agent global bypass unreachable for agent main sessions on current `main`.
  - file: [src/gateway/session-utils.ts:572](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/src/gateway/session-utils.ts#L572)

  - sha: [01bc49c88c0c](https://github.com/openclaw/openclaw/commit/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd)
- **Legacy bare main aliases are normalized:** Current UI snapshot handling rewrites `main`, the configured `mainKey`, and default-agent aliases to `sessionDefaults.mainSessionKey`, so the review concern about a bare `main` active key no longer applies on current `main`.
