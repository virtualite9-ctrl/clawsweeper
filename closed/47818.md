## Summary

PR #47818 is an older unmerged attempt to persist the Discord component/modal registry across gateway restarts. Current main still has only process-global Map-backed registry state, so this is not implemented on main, but open PR #66241 now tracks the same bug with the latest persistence design and security hardening. A reviewer also already commented on #47818 that it is superseded by #66241.

## Best Possible Solution

Close #47818 as superseded and keep review/iteration focused on #66241 as the canonical Discord component registry persistence PR. Any final fix should preserve Discord authorization and route metadata, use safe best-effort persistence, validate persisted records, prune expired entries, and include focused restart/reload tests in the Discord plugin surface.
