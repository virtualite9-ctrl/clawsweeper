## Summary

Current main already implements the requested resilient subagent completion delivery for Telegram-style requester timeouts: completion output is frozen before cleanup, delivery falls back from model-backed announce to direct outbound send when the requester is busy or the announce-agent call fails, and the background task delivery state is persisted as delivered or failed.

## Best Possible Solution

Close this issue as implemented on main. Keep the shipped direct outbound completion fallback and persisted background-task delivery state; any separate missing-payload race should remain tracked by the narrower related issue #69767 rather than this Telegram announce-timeout report.
