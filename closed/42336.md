## Summary

Current main still does not list `.reset.*` archived transcripts as separate UI sessions after `/new` or `/reset`; the remaining archived-session recovery/discoverability work is already tracked by open issue #45003, with open PR #60409 covering only a partial history fallback.

## Best Possible Solution

Keep the remaining work on open #45003: decide the product shape for restoring/re-indexing or exposing `.reset.*` archives through `sessions.list` and the UI selector, and treat #60409 as only a partial history-read fallback if it lands.

## Evidence

- **Reset archives old transcript but replaces the same store row:** `performGatewaySessionReset` updates `store[primaryKey]` to a new `SessionEntry`, then archives the old transcript via `archiveSessionTranscriptsForSessionDetailed(... reason: "reset")`; it does not add a separate historical session row.
  - file: [src/gateway/session-reset-service.ts:517](https://github.com/openclaw/openclaw/blob/6cd047e7c27021d8955e09fa3598029eede5a1a7/src/gateway/session-reset-service.ts#L517)

  - sha: [6cd047e7c270](https://github.com/openclaw/openclaw/commit/6cd047e7c27021d8955e09fa3598029eede5a1a7)
- **Session list is built only from sessions.json entries:** `listSessionsFromStore` enumerates `Object.entries(store)` and maps those rows into `GatewaySessionRow`s, so archived transcript files outside the store are not discoverable through `sessions.list`.
  - file: [src/gateway/session-utils.ts:1511](https://github.com/openclaw/openclaw/blob/6cd047e7c27021d8955e09fa3598029eede5a1a7/src/gateway/session-utils.ts#L1511)
