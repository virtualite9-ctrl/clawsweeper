## Summary

Issue #69184 asks for a service-specific Feishu Drive file download action. Current main still does not implement `feishu_drive.download`, but the request fits the ClawHub/plugin boundary: OpenClaw direction sends optional service-specific capabilities to plugins when no core API gap is identified, and current plugin APIs support registering agent tools that can perform this download flow outside the OpenClaw core repository. Related Feishu Drive file-transfer issue #48025 was already reviewed the same way.

## Best Possible Solution

Close this OpenClaw core tracker as ClawHub/plugin-scope. The best path is a small Feishu Drive download tool plugin or skill, published through ClawHub, using the existing plugin `api.registerTool` surface and Feishu API credentials/config. Keep bundled-core work for explicit maintainer-owned Feishu roadmap decisions or real plugin API gaps.
