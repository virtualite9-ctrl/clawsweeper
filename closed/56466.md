## Summary

Do not close this PR. Current main still has the reported ClawHub plugin API range bug: wildcard ranges like `*`, `x`, and `X` are not handled by the hand-rolled range checker, and the ClawHub install path will reject a plugin when that check returns false. The PR is a small, relevant fix with matching regression coverage and is not implemented or superseded on main.

## Best Possible Solution

Keep #56466 open for maintainer review and merge an equivalent focused fix: handle full wildcard range tokens in `src/infra/clawhub.ts` without changing the broader no-semver-dependency parser design, and keep/add regression coverage in `src/infra/clawhub.test.ts` for `*`, `x`, and `X`. Before merge, run the normal changed plugin/core test lane because this review was intentionally read-only.
