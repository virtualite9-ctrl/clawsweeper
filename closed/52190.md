## Summary

The request is for an optional stealth browser backend, but current main already treats browser automation as a replaceable plugin surface rather than a core backend toggle. A Patchright-based implementation fits the documented plugin/ClawHub path, not OpenClaw core.

## Evidence

- **Vision keeps optional capability out of core:** `VISION.md` says core stays lean, optional capability should usually ship as plugins, and the bar for adding optional plugins to core is intentionally high.
  - file: [VISION.md:54](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/VISION.md#L54)
  - sha: [60f7a59f5ed5](https://github.com/openclaw/openclaw/commit/60f7a59f5ed581839cfd1d52583449c5aa1f03a4)
- **Docs explicitly support replacing the bundled browser plugin:** `docs/tools/browser.md` documents that the default `browser` tool is a bundled plugin and says to disable it to replace it with another plugin that registers the same `browser` tool name; it also notes that disabling the plugin removes the CLI, gateway method, tool, and control service as one unit for replacement.
  - file: [docs/tools/browser.md:46](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/docs/tools/browser.md#L46)
  - sha: [60f7a59f5ed5](https://github.com/openclaw/openclaw/commit/60f7a59f5ed581839cfd1d52583449c5aa1f03a4)
- **Plugin docs mark browser as replaceable:** `docs/tools/plugin.md` lists `browser` as a bundled browser plugin and explicitly says it can be disabled before replacing it.
  - file: [docs/tools/plugin.md:118](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/docs/tools/plugin.md#L118)
  - sha: [60f7a59f5ed5](https://github.com/openclaw/openclaw/commit/60f7a59f5ed581839cfd1d52583449c5aa1f03a4)
- **Browser implementation is already a plugin wired through the public SDK:** `extensions/browser/plugin-registration.ts` shows the shipped browser surface is registered via public plugin APIs (`registerTool`, `registerCli`, `registerGatewayMethod`, `registerService`) rather than being hardwired in core.
  - file: [extensions/browser/plugin-registration.ts:28](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/extensions/browser/plugin-registration.ts#L28)
  - sha: [60f7a59f5ed5](https://github.com/openclaw/openclaw/commit/60f7a59f5ed581839cfd1d52583449c5aa1f03a4)
- **Current main still uses Playwright, not Patchright:** The bundled browser plugin depends on `playwright-core`, and its runtime imports `chromium` from `playwright-core`; a repo search found no in-tree `patchright` implementation.
  - file: [extensions/browser/package.json:7](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/extensions/browser/package.json#L7)
