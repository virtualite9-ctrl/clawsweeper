## Summary

PR #42353 proposes removing the `/workspace` mount for `workspaceAccess: "none"`, but current `main` intentionally keeps a read-only mount of the sandbox workspace in that mode. The report appears to conflate the sandbox workspace with the agent workspace; current code, tests, and docs all say `"none"` keeps the agent workspace off-limits while still running tools against a sandbox workspace.

## Evidence
