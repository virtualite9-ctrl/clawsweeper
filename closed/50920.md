## Summary

Current main and the shipped v2026.4.22 tree already solve the user-visible webchat scaffolding leak that PR #50920 targets. Live final chat events now carry `role: "assistant"`, and assistant-role rendering strips `<relevant-memories>` scaffolding through the shared sanitizer. The PR’s broad role-less extraction change is unnecessary and had review feedback calling out unsafe semantics plus unrelated test removal.

## Best Possible Solution

Close PR #50920 as already implemented/superseded by current main. Keep the shipped assistant-role sanitizer and final-event payload shape; do not merge the PR’s global role-less extraction change or its unrelated test coverage removal. Any future confirmed role-less assistant event should be handled with a narrow producer/controller regression test.
