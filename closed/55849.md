## Summary

#55849 is a duplicate of the still-open #55993. Both ask OpenClaw to persist `sessionKey` in the first JSONL session/transcript header, and #55993 tracks the broader remaining header metadata work (`sessionKey`, `agentId`, `channel`, `kind`). Current `main` still omits those fields, and the only issue-linked implementation attempt (#56082) was closed unmerged, so this should close only as consolidation, not as fixed.

## Best Possible Solution

Close #55849 as a duplicate and keep #55993 as the canonical OpenClaw issue. The eventual fix should thread session metadata into every transcript header creation path, including `src/config/sessions/transcript.ts`, `src/gateway/session-reset-service.ts`, `src/gateway/server-methods/chat.ts`, `src/gateway/server-methods/sessions.ts`, and the embedded-runner/bootstrap compaction helpers, then add focused tests and update the transcript-structure docs.
