## Summary

Current `main` no longer lets memory-search embedding failures abort agent replies, and it now documents both disabling memory search and selecting a different embedding provider. That addresses the reported `full`/`coding`/`messaging` profile failure mode from 2026.3.13 well enough to close as implemented on main.

## Evidence

- **Memory-search failures now return a disabled result instead of aborting:** `createMemorySearchTool` returns `buildMemorySearchUnavailableResult(...)` when manager init fails and also catches runtime search errors, so an embedding/provider error is surfaced as a disabled tool payload rather than crashing the whole request path.

## Risks / Open Questions

- This proves the reported failure mode is handled on current `main`, but it does not prove every possible `API_KEY_INVALID` root cause in external macOS daemon environments has been eliminated.
- The fix is on `main` after the latest release listed in the task, so released users may still need to upgrade once a release includes commit `0a3da5cd8a6e712de6418008e01b11fb2afd7aea` or later.

## Close Comment

Closing from a Codex review. Current `main` now handles this class of failure differently: `memory_search` no longer aborts the reply path on embedding/provider errors and instead returns a structured disabled result (`extensions/memory-core/src/tools.ts:208`, `extensions/memory-core/src/tools.ts:315`, `extensions/memory-core/src/tools.shared.ts:120`). There is also regression coverage for this exact behavior (`extensions/memory-core/src/tools.citations.test.ts:129`). In addition, current docs now cover the configuration escape hatches the report asked about: selecting a different memory embedding provider or disabling memory search entirely (`docs/reference/memory-config.md:40`, `src/config/schema.help.ts:911`), plus the daemon/launchd requirement for `GEMINI_API_KEY` (`docs/providers/google.md:387`). I do not see a released tag containing the fix commit yet, so I’m marking this implemented on `main` with no confirmed shipped release yet.
