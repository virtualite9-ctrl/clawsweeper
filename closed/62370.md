## Summary

Current `main` already injects `skills.entries.<skill>.env` / `apiKey` into host agent runs, passes those values into spawned CLI child processes, and documents that the scope is per agent turn rather than arbitrary manual shell invocations.

## Evidence

- **Skill env injection is documented as per-agent-turn host process behavior:** Docs state that `skills.entries.*.env` / `apiKey` are applied to `process.env` when an agent run starts and restored after the run ends, and that the injection is scoped to the host process for that agent turn.

## Risks / Open Questions

- A manual shell command started outside an OpenClaw agent turn, such as `node ~/.openclaw/skills/...`, still will not receive automatic skill env injection by design.
- Sandboxed sessions do not inherit host `process.env`; they require sandbox-specific env configuration.
