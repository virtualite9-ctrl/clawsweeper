## Summary

Current `main` already addresses the LaunchAgent restart/reinstall stale-process race through a different design: restart no longer uses the bootout/unload flow this PR patches, stale gateway listeners are cleaned before bind, and restart health retries kill stale PIDs and re-run restart when needed.

## Evidence

- **Restart path no longer uses bootout/unload:** `restartLaunchAgent` now enables the service, runs `launchctl kickstart -k`, and only falls back to `bootstrap` when launchctl reports the service is not loaded; it does not use the reinstall-style `bootout`/`unload` sequence that this PR patches.
