## Summary

Current main already implements the relevant `skills.load.extraDirs` behavior: extra-dir skills are low-precedence inputs to a prompt-budgeted `available_skills` catalog, the prompt builder now falls back to a compact catalog before dropping entries, and the shipped docs/tests cover that behavior explicitly.

## Evidence

- **Prompt builder uses compact fallback before truncating:** `buildWorkspaceSkillsPrompt` first checks the full formatter, then switches to a compact name+location catalog, and only truncates if even the compact catalog exceeds `maxSkillsPromptChars`. It also emits an explicit `Skills truncated...` warning with `openclaw skills check` guidance.

  - file: [src/agents/skills/compact-format.test.ts:147](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/src/agents/skills/compact-format.test.ts#L147)
