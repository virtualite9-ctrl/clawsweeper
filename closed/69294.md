## Summary

Current main already addresses the reported Windows gateway crash mode. The bundled ACPX backend now registers without probing/spawning ACP agents during normal Gateway startup; startup probing is opt-in, probe failures are logged instead of process-fatal, and plugin disable/deny policy still wins before activation. The related PR #69459 was closed unmerged and is superseded by the current main implementation.

## Best Possible Solution

Close this issue as implemented on main, keep the current opt-in ACPX startup probe behavior, and do not revive PR #69459. Users on 2026.4.15 should update to a build containing the 2026.4.25 ACPX startup change; if they intentionally disable ACPX, the existing plugin disable/deny policy is the right path.
