## Summary

Current main already implements the requested rate-limit timing behavior: provider-supplied quota/reset text is preserved, fallback cooldowns carry a soonest expiry, and the user-facing fallback exhaustion message renders a seconds/minutes countdown when that expiry is known. The implementation is also listed in the 2026.4.24 changelog.
