## Summary

Current main already prevents the reported stuck-session loop for ordinary built-in compaction summarization failures. `summarizeWithFallback()` catches failed `generateSummary`/`summarizeChunks` calls and returns a static fallback summary, and the safeguard then returns a real compaction boundary. The linked PRs #42182 and #42520 described the same failure mode but were closed unmerged; the shipped behavior solves the retry loop lower in the summarization stack.

## Best Possible Solution

Close this issue as implemented by the shipped lower-level summarization fallback. Keep the current fallback behavior and handle any remaining non-summarization safeguard exceptions as separate, narrower reports with reproduction data.
