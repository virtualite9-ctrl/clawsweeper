## Summary

Current `main` no longer surfaces compaction chatter just because verbose is off/on; user-facing compaction notices are now gated behind the separate explicit opt-in `agents.defaults.compaction.notifyUser === true`, with docs and tests covering the behavior.

## Evidence

- **Compaction notices require explicit opt-in:** `runAgentTurnWithFallback` reads `runtimeConfig?.agents?.defaults?.compaction?.notifyUser === true` before sending `🧹 Compacting context...` / completion notices, and only emits those notices from the compaction event handler when that flag is true.

## Risks / Open Questions

- This appears fixed on `main` but not in the listed latest release `v2026.4.22`; users on released builds may still hit the older behavior until a newer release ships.
- If a user has explicitly enabled `agents.defaults.compaction.notifyUser`, compaction notices will still appear even with `/verbose off`; that is now documented behavior rather than a verbose leak.
