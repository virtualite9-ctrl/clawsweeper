## Summary

Current main already fixes the user-visible lazy subcommand help bug that PR #69930 targets. The exact `placeholder.helpOption(false)` change is not present, but `runCli` now preregisters the real primary command before Commander parses `--help`, with tests covering the help path and the policy that keeps primary-command help on that path. The same implementation is present in shipped release `v2026.4.24`, so this PR is obsolete as a bug fix for #59449.

## Best Possible Solution

Close PR #69930 as implemented by the shipped primary-command preregistration path on current main. Keep the existing `runCli` behavior and tests; open a maintainer-owned follow-up only if the team wants extra lower-level `registerLazyCommand` hardening beyond the fixed CLI behavior.
