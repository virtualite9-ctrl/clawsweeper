## Summary

Current main solves the observable npm-install failure without this PR's parent-peek change: the current build/postbuild path no longer creates a root `dist/package.json`, and packaged postinstall pruning removes stale dist files that are not in the generated inventory. With no `dist/package.json`, the existing upward package-root resolver walks from `dist/plugins/...` past `dist/` to the real package root, so runtime paths resolve to `dist/plugins/runtime/index.js` rather than `dist/dist/plugins/runtime/index.js`.

## Best Possible Solution

Keep the current package-layout and postinstall-prune solution on main. Do not merge this PR's parent-peek workaround unless a fresh current-main reproduction shows a supported build or update path can still create or retain a root `dist/package.json`.

## Evidence
