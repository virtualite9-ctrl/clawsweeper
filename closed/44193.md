## Summary

Current `main` already reapplies the persisted Control UI locale during app startup, and the proposed `notify()` call would fire before any UI subscribers exist. The PR's added test exercises a private re-initialization path that does not match production startup, so this does not appear to fix a current reproducing bug on `main`.

## Evidence

- **i18n startup notify would run before subscribers exist:** `I18nManager` calls `loadLocale()` in its constructor, while subscribers are only registered later from `I18nController.hostConnected()`. A `notify()` added in the default-locale early return would therefore run against an empty subscriber set in production startup.
