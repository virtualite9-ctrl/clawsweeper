## Summary

Current main no longer matches the reported MiniMax catalog bug. MiniMax-M2.7 is intentionally text-only on the Anthropic-compatible chat path because MiniMax's official Anthropic-compatible API docs say image/document inputs are not supported, and OpenClaw routes MiniMax image understanding through the MiniMax-VL-01 media provider instead. 

## Best Possible Solution

Keep MiniMax-M2.7 and MiniMax-M2.7-highspeed text-only on the Anthropic-compatible chat path, and continue routing MiniMax image understanding through the MiniMax-VL-01 media-understanding provider. Revisit the catalog only if MiniMax changes its Anthropic-compatible API to support image/document inputs.

## Evidence

- **Upstream MiniMax API contract:** MiniMax's official Anthropic-compatible API documentation says the `messages` field supports text and tool calls but no image/document input, and the important notes reiterate that image/document inputs are not currently supported. 
  - command: `web search: official MiniMax Compatible Anthropic API docs`
- **MiniMax provider catalog remains text-only by design:** `buildMinimaxTextModel()` hardcodes `input: ["text"]`, and `buildMinimaxCatalog()` uses it for MiniMax-M2.7 and MiniMax-M2.7-highspeed before registering both `minimax` and `minimax-portal` providers.
