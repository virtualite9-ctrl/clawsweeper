## Summary

Current main already fixes the Telegram/direct-message sandbox bypass without adopting this PR's dedicated `agent:<id>:dm` bucket. External DMs that share the main conversation now derive a per-account direct-chat runtime policy key for sandbox/tool-policy decisions, while persisted `dmScope="main"` conversation history stays on the main session. The PR branch is therefore obsolete, and its review comments correctly identified peer-mixing and history-continuity downsides in the proposed bucket approach.

## Best Possible Solution

Keep the shipped `runtimePolicySessionKey`/`sandboxSessionKey` implementation on main and close #70442 as implemented. Do not merge the broad PR branch with the `agent:<id>:dm` bucket; handle any unrelated fixes from that branch in focused issues or PRs if they still matter.
