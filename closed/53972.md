## Summary

Close as superseded. Current main already has the hardened central Android CI path for Android-relevant changes, including Play debug APK build, both Play and third-party unit-test lanes, shared-asset routing, and x86_64 ABI coverage. The remaining user-facing prebuilt APK distribution request is tracked by open issue #9443, while this PR adds a duplicate standalone workflow with unresolved CI hardening regressions.

Do not merge the standalone `android-build.yml` workflow. Keep Android validation in the existing central `ci.yml` lane, and continue any user-downloadable Android APK release work in canonical issue #9443 or a new maintainer-approved release workflow PR that follows the repo's CI hardening patterns.
