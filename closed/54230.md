## Summary

Current `main` and the latest shipped release do not restart the whole gateway for `stale-socket`; the health monitor stops and starts only the affected channel/account, so the report no longer matches current behavior.

## Evidence

- **Health monitor restart scope:** On current `main`, a failed health check logs `health-monitor: restarting` and then calls `channelManager.stopChannel(channelId, accountId)` and `channelManager.startChannel(channelId, accountId)`, which is a per-channel/account restart path rather than a gateway-process restart.
