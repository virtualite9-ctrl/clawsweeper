## Summary

Current main addresses the reported stale session-lock failure mode. Lock acquisition now detects and reclaims dead/recycled/old lock files, startup cleans stale session lock files, unreclaimable lock contention raises a distinct timeout error, and the sessions_history /tools/invoke path does not convert thrown gateway errors into empty successful results.
