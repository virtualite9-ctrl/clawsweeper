## Summary

Current main already implements the PR's intended Gateway reconnect lifecycle fix: the reconnect timeout is now a managed, ref'd timer, explicit stop clears it, and regression tests cover both reconnect firing and stop cleanup. The landed maintainer commit also documents the node-host recovery behavior and records it in the unreleased changelog.

## Best Possible Solution

Close #49723 as implemented on main. Keep the landed maintainer implementation and its tests/docs as the canonical fix; the earlier duplicate discussion around #48147 is now superseded by main having the behavior directly.
