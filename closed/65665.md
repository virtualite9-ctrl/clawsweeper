## Summary

Current main implements the stale-image fix. Telegram media still enters the agent prompt as `[media attached: ...]` notes from `MediaPath`/`MediaPaths`, and commit `2d9a0d9cf00d572a36bce0154a61eb383d802272` now removes old media attachment markers, `[Image: source: ...]` markers, `media://inbound/...` refs, and old image blocks from the in-memory replay view before the model sees prior history. The runner installs that transform around embedded attempts and regression tests cover stale marker scrubbing while preserving recent/current image turns. The fix is on reviewed main `e23d17da79dc2345ca55902e632d3c7622accacc`, but no release tag contains it yet.

## Best Possible Solution

Keep the current replay-pruning implementation from `2d9a0d9c` and ship it in the next release. If stale Telegram vision output persists after a build containing that commit, track it as a new narrower bug with provider payload/cache-trace evidence and exact sequential-image reproduction steps.
