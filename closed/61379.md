## Summary

Current main keeps Kimi Coding on the Anthropic-compatible endpoint by design, but implements the user-visible fix: tagged Kimi tool-call markup is converted into structured tool calls, so the raw marker leak described in the issue discussion is handled without switching the provider default to openai-completions.
