## Summary

The specific report that `Array.prototype.toSorted()` is unstable does not reproduce. Current main still contains the cited code path, but ECMAScript specifies `toSorted` through the same stable sorting operation, and a local runtime probe preserved equal-key order.

- **ECMAScript stability contract:** The ECMAScript `Array.prototype.toSorted` algorithm calls `SortIndexedProperties`; that operation requires equal-comparator elements to retain order, i.e. the sort is stable. ([tc39.es](https://tc39.es/ecma262/multipage/indexed-collections.html))
  - command: `web.open https://tc39.es/ecma262/multipage/indexed-collections.html#sec-array.prototype.tosorted and #sec-sortindexedproperties`
