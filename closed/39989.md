## Summary

Issue #39989 targeted Telegram inbound reliability on 2026.3.2. Current `main` and shipped release `v2026.4.23` already contain targeted Telegram polling, transport, and health-liveness fixes that match the reported failure mode: inbound silently dying while outbound still works and status staying misleadingly healthy.

## Evidence

- **Polling recovery watchdog on current main:** `TelegramPollingSession` now uses a 120s default stall threshold, wraps polling in liveness tracking, and restarts the polling runner when `getUpdates` stalls.
  - file: [extensions/telegram/src/polling-session.ts:25](https://github.com/openclaw/openclaw/blob/f00d65a30496335fb068f01fe529dbba484d4312/extensions/telegram/src/polling-session.ts#L25)
