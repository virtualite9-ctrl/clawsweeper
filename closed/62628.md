## Summary

Current `main` already handles legacy absolute `sessionFile` paths from migrated state, rewrites/persists corrected transcript locations, and includes regression coverage for cross-root state moves that match the reported Docker `/home/node` to host migration failure.

## Evidence

- **Compatibility handling for legacy absolute session paths:** `resolvePathWithinSessionsDir` explicitly treats older absolute `sessionFile` values from `sessions.json` as compatibility input, normalizes in-root absolute paths, and supports cross-root `.../agents/<agentId>/sessions/<file>` fallbacks instead of rejecting them.

  - command: `nl -ba src/config/sessions/paths.ts | sed -n '170,290p'`
  - sha: [c997a9f9788e](https://github.com/openclaw/openclaw/commit/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a)
- **Resolved path is written back to the store:** `resolveAndPersistSessionFile` stores the repaired `sessionFile` back into `sessions.json` whenever the resolved path differs from the persisted value, so stale Docker-era absolute paths are corrected on use.
  - file: [src/config/sessions/session-file.ts:6](https://github.com/openclaw/openclaw/blob/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a/src/config/sessions/session-file.ts#L6)
