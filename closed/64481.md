## Summary

Current main implements the core request. OpenAI-compatible proxy/local completions requests, including vLLM, now send `tool_choice: "auto"` when tools are present, with regression coverage for a local vLLM endpoint. For cases that need `required` or a named function choice, the existing documented `params.extra_body`/`extraBody` path can overwrite generated request fields such as `tool_choice`. The issue discussion adds one same-symptom confirmation and an external setup-guide link, but no remaining missing OpenClaw core API beyond the payload behavior now present on main.

## Best Possible Solution

Close this issue and keep the generic OpenAI-compatible implementation: default proxy/local tool-enabled completions requests to `tool_choice: "auto"`, and rely on the documented `params.extra_body` override for vLLM/Gemma deployments that need `required` or a named function choice. Add a dedicated typed config key only if future reports show the generic override is not sufficient.
