## Summary

Current `main` already contains a bundled-capability-provider allowlist/enablement compatibility path that keeps bundled media providers like Groq available even when `plugins.allow` is restrictive, and the same fix shipped in `v2026.4.23`.

## Evidence

- **Groq is a bundled media-understanding provider:** The Groq bundled plugin manifest declares `contracts.mediaUnderstandingProviders = ["groq"]`, so Groq is exactly the kind of bundled capability owner that the fallback path now targets.
  - file: [extensions/groq/openclaw.plugin.json:7](https://github.com/openclaw/openclaw/blob/3ffd944e6ba6dfe4c7c4c275848f1c0a1f246e28/extensions/groq/openclaw.plugin.json#L7)
  - sha: [3ffd944e6ba6](https://github.com/openclaw/openclaw/commit/3ffd944e6ba6dfe4c7c4c275848f1c0a1f246e28)
- **Media registry now resolves providers through capability fallback:** `buildMediaUnderstandingRegistry` builds its registry from `resolvePluginCapabilityProviders({ key: "mediaUnderstandingProviders" ... })`, so the new capability-provider runtime compatibility path directly covers the failing `getMediaUnderstandingProvider("groq", ...)` lookup described in the issue.
  - file: [src/media-understanding/provider-registry.ts:30](https://github.com/openclaw/openclaw/blob/3ffd944e6ba6dfe4c7c4c275848f1c0a1f246e28/src/media-understanding/provider-registry.ts#L30)
  - sha: [3ffd944e6ba6](https://github.com/openclaw/openclaw/commit/3ffd944e6ba6dfe4c7c4c275848f1c0a1f246e28)
- **Bundled capability providers are re-added before fallback runtime loads:** `resolveCapabilityProviderConfig` now resolves bundled capability-owner plugin ids from manifests, applies allowlist compatibility, applies enablement compatibility, and then uses that compat config for runtime-registry fallback lookups. This is the code path that prevents bundled providers from disappearing after a restrictive-registry reload.
  - file: [src/plugins/capability-provider-runtime.ts:45](https://github.com/openclaw/openclaw/blob/3ffd944e6ba6dfe4c7c4c275848f1c0a1f246e28/src/plugins/capability-provider-runtime.ts#L45)
  - sha: [3ffd944e6ba6](https://github.com/openclaw/openclaw/commit/3ffd944e6ba6dfe4c7c4c275848f1c0a1f246e28)
- **Restrictive plugins.allow is patched for bundled providers:** `withBundledPluginAllowlistCompat` appends missing bundled plugin ids into `plugins.allow`, and `withBundledPluginEnablementCompat` synthesizes enabled plugin entries when needed. That matches the workaround reported in discussion: manually adding `groq` to `plugins.allow`.
  - file: [src/plugins/bundled-compat.ts:5](https://github.com/openclaw/openclaw/blob/3ffd944e6ba6dfe4c7c4c275848f1c0a1f246e28/src/plugins/bundled-compat.ts#L5)
  - sha: [3ffd944e6ba6](https://github.com/openclaw/openclaw/commit/3ffd944e6ba6dfe4c7c4c275848f1c0a1f246e28)
- **Targeted regression test for media-understanding fallback:** The media-understanding regression test explicitly asserts that bundled capability plugin ids are added to `plugins.allow` before fallback registry load.
  - file: [src/media-understanding/provider-registry.allowlist.test.ts:20](https://github.com/openclaw/openclaw/blob/3ffd944e6ba6dfe4c7c4c275848f1c0a1f246e28/src/media-understanding/provider-registry.allowlist.test.ts#L20)
  - sha: [3ffd944e6ba6](https://github.com/openclaw/openclaw/commit/3ffd944e6ba6dfe4c7c4c275848f1c0a1f246e28)
- **Targeted regression test for provider discoverability under plugins.allow:** The provider-allowlist compatibility test asserts bundled implicit providers remain discoverable when `plugins.allow` is set, while explicit denies still win.
  - file: [src/agents/models-config.providers.plugin-allowlist-compat.test.ts:9](https://github.com/openclaw/openclaw/blob/3ffd944e6ba6dfe4c7c4c275848f1c0a1f246e28/src/agents/models-config.providers.plugin-allowlist-compat.test.ts#L9)
  - sha: [3ffd944e6ba6](https://github.com/openclaw/openclaw/commit/3ffd944e6ba6dfe4c7c4c275848f1c0a1f246e28)
- **Shipped in latest release:** The `v2026.4.23` release commit contains the addition of `src/plugins/bundled-compat.ts`, `src/plugins/capability-provider-runtime.ts`, and the two targeted regression tests above, showing the fix was shipped.
