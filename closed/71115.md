## Summary

PR #71115 adds an optional subagent model router and telemetry path directly to core, but current main already exposes the generic plugin hooks and plugin-owned config needed to build this as a hook-only plugin. The OpenClaw scope guidance favors ClawHub/plugin delivery for optional capabilities unless a missing generic plugin API is identified, and this review did not find one.

## Best Possible Solution

Do not add a core agents.defaults.subagents.modelRouter branch. Build this as a ClawHub/plugin-owned router using before_model_resolve for shadow/live recommendations, plugins.entries.<id>.config for mode/policy/command/telemetry settings, and model_call_started/model_call_ended for actual provider/model telemetry. If plugin development proves a generic hook field is missing, open a narrow plugin API PR for that field.
