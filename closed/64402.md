## Summary

Issue #64402 is not implemented on current main, but it is superseded by the already-linked open PR #65065. Current main still has the reported CLI invocation gap: Discord advertises and handles channel admin actions, while `openclaw message channel` only registers `info` and `list`. PR #65065 is the reporter's focused implementation for that remaining work and explicitly closes this issue.

## Best Possible Solution

Close #64402 as superseded by #65065. Keep the remaining implementation work on that PR: rebase or replace it with an equivalent maintainer patch that adds `openclaw message channel create|edit|delete|move`, focused CLI tests, and CLI docs. If the separate embedded-agent `message` tool exposure concern still reproduces afterward, track it separately with a narrower runtime reproduction.
