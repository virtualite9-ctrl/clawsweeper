## Summary

Close PR #52070 as not actionable in the OpenClaw source repo. The useful research-verification intent is already covered by current root maintainer guidance, while the daily-brief PDF workflow and root SOUL.md addition are personal agent-workspace content. Existing docs place AGENTS.md/SOUL.md in the configured agent workspace/private backup, and the PR discussion already identified the hardcoded personal paths, root SOUL.md placement, and Chrome --no-sandbox guidance as inappropriate for shared repo guidance.

## Best Possible Solution

Close this PR. Keep the existing root maintainer guidance, and if maintainers want stronger repo-wide research wording later, it should come as a narrow AGENTS.md-only proposal without personal daily-brief automation, hardcoded local paths, --no-sandbox Chrome flags, or a repo-root SOUL.md.
