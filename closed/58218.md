## Summary

Current main no longer has the v2026.3.28 legacy-config migration/barrel shape implicated by the reported bundled subprocess crash. Legacy rule detection still runs from config validation, but the rule list is now defined through a doctor-owned migration module and the config barrel no longer re-exports the legacy migration path that could pull migration code into normal config loading.

## Best Possible Solution

Close as implemented. Keep the shipped doctor-owned legacy config migration boundary and the current validation import path; users who hit this on 2026.3.28 should update to v2026.4.5 or newer, preferably the latest stable release.
