## Summary

The request is a vendor/hardware-specific speech feature for DGX Spark, and current `main` already provides the plugin seams and user-facing provider selection surfaces needed to ship it as a plugin rather than as new OpenClaw core behavior.

## Evidence

- **Vision scopes optional capability to plugins/ClawHub:** The project vision says optional capability should usually ship as plugins, the preferred path is external plugin distribution, and plugin promotion belongs in ClawHub rather than core.

## Risks / Open Questions

- No related plugin or prior implementation was linked in the issue discussion, so closing assumes a plugin owner is willing to build and maintain the DGX Spark speech path externally.
- If OpenClaw later decides DGX Spark voice-local is a first-party product surface rather than a vendor-specific plugin, this triage decision would need revisiting.
