## Summary

Current main fixes the persistent replay shape reported in #56045. Main-session cron jobs still enqueue `cron:<jobId>` system events, but heartbeat now consumes the inspected pending events after a successful run, and regression coverage proves a queued cron event is not replayed on the next heartbeat. The verified latest release tag `v2026.4.24` contains that implementation.

## Best Possible Solution

Close #56045 as implemented. Keep the shipped heartbeat/system-event consumption behavior and the existing regression coverage; if a current release still shows repeated deliveries with zero jobs, track it as a fresh bug with run logs, heartbeat logs, affected job JSON, and channel delivery details.
