## Summary

Issue #63385 is already implemented on current `main`: OpenClaw shipped a `refresh_token_reused` recovery path for Codex OAuth in `2026.4.7`, and later removed the `.codex` onboarding/provider-discovery import path that originally created the copied-token drift scenario.
