## Summary

The request is for a repo-external `eo_rag_*` tool surface that does not exist on current `main`, and the behavior described fits OpenClaw's plugin/ClawHub memory architecture rather than OpenClaw core.

## Evidence

- **No `eo_rag_*` surface on current main:** Searched current `main` (`cc0f3e0e40364023cc3a39227a411898ee68ce3b`) for `eo_rag_index` and `eo_rag_query` across `src`, `extensions`, `packages`, `docs`, `README.md`, and `VISION.md`; `rg` returned no matches.

  - sha: [cc0f3e0e4036](https://github.com/openclaw/openclaw/commit/cc0f3e0e40364023cc3a39227a411898ee68ce3b)
- **Project vision keeps optional memory/knowledge capability in plugins:** The vision doc says core stays lean, optional capability should usually ship as plugins, memory is a plugin slot, and plugin promotion belongs in ClawHub.
  - file: [VISION.md:54](https://github.com/openclaw/openclaw/blob/cc0f3e0e40364023cc3a39227a411898ee68ce3b/VISION.md#L54)

- **Memory and knowledge-base surfaces are already plugin-owned:** Docs state `memory_search`/`memory_get` come from the active memory plugin, and recommend the bundled `memory-wiki` plugin for a maintained knowledge base with provenance-rich tooling.
  - file: [docs/concepts/memory.md:39](https://github.com/openclaw/openclaw/blob/cc0f3e0e40364023cc3a39227a411898ee68ce3b/docs/concepts/memory.md#L39)

  - file: [docs/plugins/hooks.md:113](https://github.com/openclaw/openclaw/blob/cc0f3e0e40364023cc3a39227a411898ee68ce3b/docs/plugins/hooks.md#L113)
  - sha: [cc0f3e0e4036](https://github.com/openclaw/openclaw/commit/cc0f3e0e40364023cc3a39227a411898ee68ce3b)

  - file: [extensions/memory-core/src/public-artifacts.ts:33](https://github.com/openclaw/openclaw/blob/cc0f3e0e40364023cc3a39227a411898ee68ce3b/extensions/memory-core/src/public-artifacts.ts#L33)
