## Summary

Current main already fixes the PR's target bug through the shipped session-key based cron delivery path. New cron jobs created from a chat now persist the caller session key, isolated delivery resolution uses that stored session entry for `channel: "last"`, and the changelog records the broader shipped cron delivery fix in v2026.4.20.

## Best Possible Solution

Close this PR as already implemented on main. Keep the shipped session-key/gateway delivery-resolution approach from #69587 rather than adding the PR's now-obsolete `origin*` fields to `CronDelivery`; any remaining edge cases should be filed as new, narrow bugs against the current `channel: "last"` session-key resolution path.
