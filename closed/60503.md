## Summary

Current `main` already separates Telegram approval-button callback queries onto an approval-specific sequentializer lane, which matches the reported root cause and the shipped changelog entry for `#64979`. A follow-up commenter also reported the bug resolved in OpenClaw 2026.4.14.

## Evidence

- **Current code routes approval callbacks to a separate lane:** `getTelegramSequentialKey` detects `/approve ...` callback payloads and returns `telegram:<chatId>:approval` instead of the normal chat/topic lane, so approval button callbacks are no longer queued behind the active turn lane described in the issue.
  - file: [extensions/telegram/src/sequential-key.ts:90](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/extensions/telegram/src/sequential-key.ts#L90)
  - command: `nl -ba extensions/telegram/src/sequential-key.ts | sed -n '84,100p'`
  - sha: [926068b14fc6](https://github.com/openclaw/openclaw/commit/926068b14fc661e915a0247e9ecb5949fb95b4f7)
- **Telegram bot still uses the sequentializer with the new key function:** `createTelegramBot` installs `sequentialize(getTelegramSequentialKey)`, so the approval-lane routing in `sequential-key.ts` is active in the current Telegram runtime path.
  - file: [extensions/telegram/src/bot-core.ts:412](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/extensions/telegram/src/bot-core.ts#L412)
  - command: `nl -ba extensions/telegram/src/bot-core.ts | sed -n '380,450p'`
  - sha: [926068b14fc6](https://github.com/openclaw/openclaw/commit/926068b14fc661e915a0247e9ecb5949fb95b4f7)
- **Regression tests cover approval callback lane keys:** `sequential-key.test.ts` asserts that Telegram approval callback payloads such as `/approve plugin:abc123 allow-once` and `/approve exec:def456 deny` map to `telegram:<chatId>:approval`, confirming the intended bypass from the normal chat lane.
  - file: [extensions/telegram/src/sequential-key.test.ts:130](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/extensions/telegram/src/sequential-key.test.ts#L130)
  - command: `nl -ba extensions/telegram/src/sequential-key.test.ts | sed -n '120,170p'`
  - sha: [926068b14fc6](https://github.com/openclaw/openclaw/commit/926068b14fc661e915a0247e9ecb5949fb95b4f7)
- **Changelog says this shipped in v2026.4.14:** The `2026.4.14` changelog explicitly says Telegram approval button callback queries were routed onto a separate sequentializer lane so approval clicks can resolve immediately instead of deadlocking behind the blocked agent turn, and attributes that fix to `#64979`.
  - file: [CHANGELOG.md:820](https://github.com/openclaw/openclaw/blob/926068b14fc661e915a0247e9ecb5949fb95b4f7/CHANGELOG.md#L820)
