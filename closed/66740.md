## Summary

Current `main` already fixes the reported `models list` display bug for configured `google/gemma-4-26b-a4b-it` entries: configured model metadata now wins over registry misses and Google forward-compat fallback metadata, so the configured context window is preserved instead of inheriting the 1M Gemini template value.

## Evidence

- **`models list` now resolves configured rows through the shared model resolver:** Configured `models list` rows call `resolveModelWithRegistry(...)`, so list output uses the same normalized resolution path as runtime model resolution instead of bypassing configured overrides.
  - file: [src/commands/models/list.rows.ts:283](https://github.com/openclaw/openclaw/blob/55d3e7bc473e34883a2d2cce51ce904b87c32eed/src/commands/models/list.rows.ts#L283)

- **Configured metadata overrides discovered or fallback context windows:** `applyConfiguredProviderOverrides(...)` explicitly prefers the configured model's `contextWindow` and `maxTokens` over the discovered model values.
  - file: [src/agents/pi-embedded-runner/model.ts:396](https://github.com/openclaw/openclaw/blob/55d3e7bc473e34883a2d2cce51ce904b87c32eed/src/agents/pi-embedded-runner/model.ts#L396)
