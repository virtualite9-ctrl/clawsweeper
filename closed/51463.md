## Summary

Current `main` does not match the reported `server.mjs` device-approval timeout flow. The Docker/runtime paths now start the gateway directly, health checks use `/healthz`, pairing-required/auth-closed responses are treated as healthy reachability, and I found no `server.mjs` file or matching timeout log string in the checkout.

## Evidence

- **Current Docker Compose starts the gateway directly:** The container command is `node dist/index.js gateway --bind ... --port 18789`, with a healthcheck against `/healthz`; there is no `server.mjs` wrapper in the current Compose path.
  - file: [docker-compose.yml:29](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/docker-compose.yml#L29)

  - sha: [608c08fc54da](https://github.com/openclaw/openclaw/commit/608c08fc54da2f86338366f8eeafd7e2a02713aa)
- **Current image default command is direct gateway startup:** The image `CMD` is `node openclaw.mjs gateway --allow-unconfigured` and the built-in `HEALTHCHECK` probes `http://127.0.0.1:18789/healthz`. That is inconsistent with a parent `server.mjs` process killing the child after a device-approval timeout.
  - file: [Dockerfile:280](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/Dockerfile#L280)
