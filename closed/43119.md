## Summary

#43119 should close as superseded by #54714. Current main has the Control UI/session plumbing for per-agent identity, but the exact remaining gap in this issue is the still-open precedence bug where global `ui.assistant` wins before `agents.list[].identity`; #54714 already tracks that current-main bug with narrower evidence and an unmerged attempted fix.

## Best Possible Solution

Close #43119 as superseded by #54714 and keep the implementation work on #54714: fix `resolveAssistantIdentity()` and the avatar-resolution path so per-agent `agents.list[].identity` wins when present and `ui.assistant` remains the backward-compatible fallback, then add regression tests for session/agent switches, name, avatar, emoji, and partial fallback cases.
