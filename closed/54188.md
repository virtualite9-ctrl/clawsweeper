## Summary

Current `main` already handles explicit `delivery.mode: "none"` cron runs as `not-requested`, so false delivery failures no longer override successful job execution. The fix is also documented as shipped in `v2026.4.22`.

## Evidence

- **Delivery plan marks `mode: "none"` as not requested:** `resolveCronDeliveryPlan` returns `requested: false` when the normalized delivery mode is `"none"`, so no fallback delivery is expected for these jobs.
  - file: [src/cron/delivery-plan.ts:29](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/cron/delivery-plan.ts#L29)

  - file: [src/cron/service/timer.ts:257](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/cron/service/timer.ts#L257)
