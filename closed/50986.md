## Summary

This PR adds an optional Feishu-specific bot-to-bot relay that is not present on current main and is not a shipped regression fix. Current project scope and docs place optional relay/bridge behavior in the plugin, hook, or ClawHub path, and the related Feishu bot-to-bot issue #49709 was already closed that way. The PR discussion is useful, especially the message-tool failure report and follow-up #54553, but it does not establish a missing core API that blocks an external implementation.

## Best Possible Solution

Close this OpenClaw core PR and keep first-party Feishu behavior focused on the documented channel/plugin contract. The relay should be published as a ClawHub plugin, hook pack, or standalone Feishu-channel extension using the current message hook/outbound surfaces; if implementers discover a concrete missing public API, the best follow-up is a narrow plugin-API issue rather than reviving this bundled Feishu relay PR.
