## Summary

Current main already fixes the underlying Node-version update failure in a shipped, tested way by preflighting the target package's `engines.node` before `openclaw update` performs a global install. That makes this PR's post-update auto-upgrade path unnecessary, and the PR discussion also surfaced correctness risks in the proposed branch-specific approach.

## Evidence

- **Shipped changelog entry:** Release `v2026.4.23` explicitly says `openclaw update` now preflights the target npm package `engines.node` so outdated Node runtimes fail with a clear upgrade message instead of attempting an unsupported latest release.
