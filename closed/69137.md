## Summary

Keep open. The checked-out `main` at `7f6452897e25543c9263b67aabf140379d1c0b0c` still rejects a root-level `paperclip` field on `req agent`, so the reported Paperclip/OpenClaw gateway mismatch is still real in current code. The issue comment also adds a useful workaround and clarifies follow-on `agentId`/`sessionKey` constraints, but that does not make the core compatibility bug obsolete.
