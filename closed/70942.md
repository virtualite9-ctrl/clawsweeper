## Summary

Issue #70942 is a real current-main Mattermost status bug, but the same remaining work is already tracked by open PR #71024. Current main still lacks a Mattermost read-only account inspector, so this should not be closed as implemented; it can close as superseded by the focused fix PR that explicitly closes #70942 and documents the exact root cause, tests, and intended behavior.

## Best Possible Solution

Close #70942 as superseded by PR #71024 and continue review/merge work there. The best code path is the PR’s proposed Mattermost read-only account inspector plus regression coverage that reports configured-but-unavailable SecretRef credentials without dereferencing secrets, while keeping runtime Mattermost token resolution strict.

## Evidence

The PR body says "Closes #70942" and identifies the root cause as Mattermost having strict runtime account resolution but no read-only inspectAccount path, causing status to fall back to resolveAccount and throw on unresolved SecretRefs.
