## Summary

Current main already fixes the Ollama picker problem through the shipped #55290 configured-model catalog path: Ollama setup seeds configured Ollama models, the wizard passes Ollama as the preferred provider, and the picker filters to that provider when entries exist. The PR's unconditional Ollama augmentModelCatalog placeholder is redundant.

## Best Possible Solution

Close this PR and keep the shipped #55290-style configured-model catalog behavior. If maintainers want to prevent future provider omissions, track that separately as a generic provider-plugin contract or validation issue rather than merging an Ollama-only unconditional catalog placeholder.
