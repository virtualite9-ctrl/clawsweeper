## Summary

Current `main` now implements provider-side prompt-cache routing for OpenAI-family transports and preserves a stable system-prompt prefix, so the prompt-caching regression described here is already addressed on `main`.

## Evidence

- **OpenAI Responses requests now send a prompt cache key tied to the active session:** `buildOpenAIResponsesParams()` sets `prompt_cache_key` to `options?.sessionId` whenever cache retention is enabled, which partitions provider-side cache routing instead of leaving all installs on the same prompt prefix alone.
