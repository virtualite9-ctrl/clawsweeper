## Summary

Current `main` already routes Telegram command validation through a standalone `src/plugin-sdk/telegram-command-config.ts` helper with a local regex/shared normalizers, so the startup circular dependency described in #60685 is no longer present; the same implementation is also present in shipped release `v2026.4.23`.
