## Summary

#55994 is a real remaining core outbound-message bug, but it is already superseded by the canonical open PR #65301. Current main still has the broad `action="send"` poll guard, while #65301 directly targets this issue with the narrower intent check and regression coverage; #65295 was an earlier closed duplicate attempt with the same fix direction.

## Best Possible Solution

Close #55994 as superseded and keep the remaining work on PR #65301: land that PR or a maintainer-equivalent change that narrows the send-side poll guard to real poll content, with the metadata-only regression tests retained.
