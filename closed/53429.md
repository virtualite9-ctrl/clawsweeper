## Summary

Current `main` already treats intentional Telegram solo/no-group setups as `groupPolicy: "disabled"`, and the doctor warning path no longer fires for that configuration. The Telegram-specific doctor warning is gated to `groupPolicy === "allowlist"`, while the generic empty-group-allowlist warning is explicitly skipped for Telegram.

## Evidence

- **Telegram doctor warning only runs for allowlist:** `collectTelegramEmptyAllowlistExtraWarnings` returns warnings only when the effective Telegram `groupPolicy` is `"allowlist"`; otherwise it returns `[]`.
  - file: [extensions/telegram/src/doctor.ts:362](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/extensions/telegram/src/doctor.ts#L362)
  - command: `nl -ba extensions/telegram/src/doctor.ts | sed -n '357,388p'`
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Telegram skips the generic empty-group warning path:** The Telegram doctor adapter sets `shouldSkipDefaultEmptyGroupAllowlistWarning` for Telegram, so doctor does not also emit the generic empty-group-allowlist warning for this channel.
  - file: [extensions/telegram/src/doctor.ts:376](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/extensions/telegram/src/doctor.ts#L376)
  - command: `nl -ba extensions/telegram/src/doctor.ts | sed -n '357,388p'`

- **Generic doctor warnings only apply to allowlist group policy:** `collectEmptyAllowlistPolicyWarningsForAccount` exits early unless `groupPolicy === "allowlist"`, so `groupPolicy: "disabled"` does not produce the empty-group warning class.
  - file: [src/commands/doctor/shared/empty-allowlist-policy.ts:59](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/commands/doctor/shared/empty-allowlist-policy.ts#L59)
  - command: `nl -ba src/commands/doctor/shared/empty-allowlist-policy.ts | sed -n '1,220p'`
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Docs define the supported no-group configuration:** The docs explicitly list `groupPolicy: "disabled"` as the way to disable all group replies, and Telegram docs list `disabled` as a valid Telegram group policy.
  - file: [docs/channels/groups.md:66](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/docs/channels/groups.md#L66)

  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Telegram runtime enforces disabled group mode:** Telegram runtime drops group messages when `groupPolicy` is disabled, matching the documented solo/no-group intent.
