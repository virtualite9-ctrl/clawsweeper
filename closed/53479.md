## Summary

Current `main` already handles the reported dual-stack loopback case in the live port-diagnostics path, with stronger logic than this PR and a direct regression test. The misleading multi-listener warning is suppressed for same-process IPv4+IPv6 loopback gateway listeners, so the underlying fix is already present on `main`.

## Evidence

- **Current implementation suppresses the warning for benign dual-stack loopback gateway listeners:** `isDualStackLoopbackGatewayListeners` requires all listeners to be gateway listeners, have defined finite PIDs, parse to the requested port, classify as loopback, and include both IPv4 and IPv6 families; `buildPortHints` skips the `Multiple listeners detected` warning when that helper returns true.
