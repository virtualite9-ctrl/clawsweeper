## Summary

Current main already solves the requested hard-block use case through the dedicated `before_agent_reply` hook rather than adding `abort` fields to `before_prompt_build`. That hook can claim a turn with `{ handled: true, reply }`, is invoked before the LLM path in auto-reply handling, is covered by tests, and is documented/changeloged as the plugin API for synthetic replies or silence before model execution.

## Best Possible Solution

Keep the shipped `before_agent_reply` contract as the supported deterministic pre-LLM block/response API. Reopen or file a narrower follow-up only if plugin authors need session-message access in that claim hook, not just prompt-based blocking.
