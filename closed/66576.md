## Summary

Current OpenClaw already supports per-workspace/per-agent bootstrap file filtering through the documented `agent:bootstrap` hook surface, which can mutate or replace `context.bootstrapFiles` before Project Context is built. The proposed config keys are not present, but the requested behavior is already shipped and documented.

## Evidence

- **Hook override path returns hook-mutated bootstrap files:** `applyBootstrapHookOverrides` creates an `agent:bootstrap` event with `bootstrapFiles` in the hook context, triggers hooks, and returns the updated `context.bootstrapFiles` array.
  - file: [src/agents/bootstrap-hooks.ts:7](https://github.com/openclaw/openclaw/blob/9a0b26cafca76d267dd231b5eaceadb071ec749d/src/agents/bootstrap-hooks.ts#L7)
  - sha: [9a0b26cafca7](https://github.com/openclaw/openclaw/commit/9a0b26cafca76d267dd231b5eaceadb071ec749d)
- **Bootstrap pipeline applies hook result before prompt injection:** `resolveBootstrapFilesForRun` loads workspace bootstrap files, applies session/context filters, then calls `applyBootstrapHookOverrides` before sanitizing the final list that will feed prompt construction.
  - file: [src/agents/bootstrap-files.ts:229](https://github.com/openclaw/openclaw/blob/9a0b26cafca76d267dd231b5eaceadb071ec749d/src/agents/bootstrap-files.ts#L229)
