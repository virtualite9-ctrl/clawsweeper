## Summary

Close as ClawHub/plugin scope. Current main does not implement Feishu applink-token resolution, but the issue itself identifies the token as client-only and not convertible to an API message_id. The safe parts are already covered by the Feishu plugin: rich-text links are preserved from post messages, and message reads work when a real message_id is available. A Feishu-specific helper plugin/skill can detect these applinks and return a user-facing notice through existing hook surfaces; no missing core API or protected maintainer signal is present.
