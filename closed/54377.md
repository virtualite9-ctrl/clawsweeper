## Summary

Keep open. Current main does not support `openclaw plugins install npm:<package>` as an npm-only escape hatch: `npm:` is rejected by the registry npm spec validator, the install command still prefers ClawHub for bare specs before falling back to npm, and public docs explicitly say there is no `npm:` override. The PR is a focused core CLI change rather than ClawHub/plugin work, but it still needs maintainer review and likely one more iteration for prefix precedence before merge.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to finish the core CLI implementation by parsing an explicit `npm:` prefix before local path/existence routing, preserving Matrix invalid-config recovery, adding tests for npm-only success/failure and local `npm:...` path collision, and updating the plugin install docs to document the new npm-only locator.

## Risks / Open Questions

- The PR changes core install resolution, so it should not be closed as a ClawHub/plugin candidate; external plugins cannot change how `openclaw plugins install` dispatches before the plugin is installed.
- The current PR patch appears to parse `npm:` after local path detection, so maintainers should require the prefix to take precedence over local path names and add a collision test before merging.
- Docs that currently say there is no `npm:` override would need to be updated if the PR lands.
