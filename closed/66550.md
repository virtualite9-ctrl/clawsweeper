## Summary

The requested agent discussion-room feature is better scoped as a plugin/ClawHub workflow, not new OpenClaw core behavior. Current main already exposes plugin/runtime seams for subagent orchestration and includes the bundled `open-prose` plugin as a concrete multi-agent orchestration path.

## Evidence

- **Vision scope: optional capability should ship as plugins/ClawHub, not core orchestration:** `VISION.md` says core should stay lean, optional capability should usually ship as plugins, new skills should go through ClawHub first, and OpenClaw will not merge heavy orchestration layers or default agent-hierarchy frameworks.

## Risks / Open Questions

- The request also mentions channel-native UX like Feishu/Discord streaming; a plugin may still need additional examples or small SDK seams to make that polished.
- `open-prose` is bundled in-tree today rather than clearly documented as a ClawHub listing, so users may not immediately discover it without maintainer guidance.
