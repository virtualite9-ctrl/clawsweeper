## Summary

Current main no longer relies on a nested `dist/extensions/acpx/node_modules/.bin/acpx` path for packaged ACPX installs. Bundled runtime deps are staged in an external runtime root for packaged installs, the plugin runtime is mirrored/remapped there before load, ACPX opts into that flow, and current tests cover the staged external-root path including `dist-runtime` ACPX wrappers.

## Evidence
