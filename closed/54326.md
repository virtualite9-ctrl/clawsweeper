## Summary

PR #54326 is obsolete on current main. The PR's central user problem from #49753, missing streamed token usage for OpenAI-compatible providers such as Scaleway and DashScope, is now handled in the transport layer: streamed `openai-completions` requests always include `stream_options.include_usage`, and the parser records final usage-only chunks before skipping empty `choices` arrays. The DashScope generic-provider case also has explicit regression coverage, and v2026.4.24 contains the shipped implementation.

## Best Possible Solution

Close #54326 as implemented on main. Keep the shipped transport-level behavior that always requests streamed usage for OpenAI-compatible streaming requests and preserves final usage-only chunks; do not merge the stale model-compat allowlist patch. Any future provider that rejects `stream_options.include_usage` should be handled as a focused provider-specific compat/transport opt-out with regression coverage.
