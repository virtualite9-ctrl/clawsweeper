## Summary

Keep open. Current main still does not fully solve the reported Linux systemd install failure. The install path writes a user unit, then runs a single daemon-reload, enable, and restart; enable-time missing-unit failures still surface as `Gateway install failed`. The related open PR #57524 and related issue #65184 both point at remaining systemd install gaps rather than proving this item is obsolete.

## Best Possible Solution

Keep the issue open and resolve it with a focused Linux daemon install change: either reject `sudo openclaw gateway install` with clear `loginctl enable-linger` guidance, or make unit writing and `systemctl --user` activation target the same user scope. Add regression coverage for the missing-unit/sudo install path, and decide separately whether OpenClaw should offer an explicit foreground/supervisor fallback instead of silently backgrounding the gateway.
