## Summary

Keep open. Current `main` still keeps the Discord component/modal registry in process-global maps only; no current code loads or persists the registry across gateway restarts. The related maintainer PR #70626 is an open draft helper with no consumer migration, so it may be the right foundation but does not yet supersede the Discord behavior this PR targets.

## Best Possible Solution

Keep a Discord restart-survival item open. The likely best maintainer path is to land or adapt a shared persistent keyed store, then migrate the Discord component/modal registry to it while preserving routing fields, allowlists, TTL semantics, state-dir placement, atomic/corrupt-file behavior, and focused restart-persistence tests.
