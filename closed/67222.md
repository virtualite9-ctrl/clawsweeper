## Summary

Close #67222 as a duplicate of older open issue #39341. Current main still has the reported Discord security-audit mismatch, but #39341 already tracks the same remaining work: `channels.discord.commands.native.no_allowlists` should recognize top-level `channels.discord.allowFrom` instead of only legacy `dm.allowFrom` plus paired-store entries.

## Best Possible Solution

Close #67222 as a duplicate of open #39341, then fix the canonical issue by including normalized top-level `discordCfg.allowFrom` in `ownerAllowFromConfigured` and adding a regression test proving `channels.discord.allowFrom` suppresses `channels.discord.commands.native.no_allowlists` while preserving legacy `channels.discord.dm.allowFrom` behavior.
