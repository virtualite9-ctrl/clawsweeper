## Summary

Close #68652 as a duplicate of the older open #7406. The current issue asks for the same Telegram forum topic name in Control UI/dashboard session labels, and current main still only exposes TopicName as inbound context/prompt/hook metadata rather than wiring it into session display labels.

## Best Possible Solution

Close #68652 as a duplicate and keep #7406 as the canonical tracker. The eventual product fix should wire cached Telegram TopicName into persisted session metadata used by sessions.list and the Control UI selector, likely via ThreadLabel/displayName or label during inbound session metadata recording, with regression coverage in Telegram context/session metadata, gateway session rows, and Control UI label resolution.
