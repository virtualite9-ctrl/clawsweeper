## Summary

Current `main` already contains Telegram-specific approval routing that resolves the originating chat/topic from turn-source or persisted session delivery context, keeps approver DMs separate from the allowlist-derived approver set, and records exact Telegram last-route metadata for later follow-up/system delivery. The implementation and regression tests were added in commit `6fea42fc2d372d2cc76776859060dc59e4a2f261`, after the issue was filed, and the latest release `v2026.4.23` predates that fix.

## Evidence

- **Telegram approvals now resolve origin chat/topic separately from approver DMs:** `resolveTurnSourceTelegramOriginTarget` and `resolveSessionTelegramOriginTarget` derive the originating Telegram destination, while `resolveTelegramApproverDmTargets` maps configured approvers separately. The capability enables `notifyOriginWhenDmOnly`, which matches the requested behavior of notifying the originating chat without broadcasting approvals to all authorized users.
  - file: [extensions/telegram/src/approval-native.ts:33](https://github.com/openclaw/openclaw/blob/ecac696643c604c6d854341bfe6424bfa31423e9/extensions/telegram/src/approval-native.ts#L33)
  - sha: [ecac696643c6](https://github.com/openclaw/openclaw/commit/ecac696643c604c6d854341bfe6424bfa31423e9)
- **Approval origin resolution fails closed on mismatched origin/session targets:** `resolveApprovalRequestOriginTarget` checks the bound channel/account, resolves both turn-source and stored session targets, and returns `null` when they disagree instead of picking one and risking cross-chat delivery.
  - file: [src/infra/exec-approval-session-target.ts:184](https://github.com/openclaw/openclaw/blob/ecac696643c604c6d854341bfe6424bfa31423e9/src/infra/exec-approval-session-target.ts#L184)
  - sha: [ecac696643c6](https://github.com/openclaw/openclaw/commit/ecac696643c604c6d854341bfe6424bfa31423e9)
- **Regression tests cover session-bound and origin-bound Telegram approval routing:** Tests verify direct-chat origin routing, topic-scoped routing, fallback to session-bound origin targets for plugin approvals, numeric thread-id parsing from the session store, and DM-only approvals still notifying the origin chat.
  - file: [extensions/telegram/src/approval-native.test.ts:60](https://github.com/openclaw/openclaw/blob/ecac696643c604c6d854341bfe6424bfa31423e9/extensions/telegram/src/approval-native.test.ts#L60)
  - sha: [ecac696643c6](https://github.com/openclaw/openclaw/commit/ecac696643c604c6d854341bfe6424bfa31423e9)
- **Telegram inbound sessions persist exact chat/topic last-route metadata for later follow-up delivery:** Inbound Telegram handling records `OriginatingChannel`/`OriginatingTo` and writes `updateLastRoute` with the concrete `telegram:${chatId}` or `telegram:${chatId}:topic:${threadId}` target into session metadata, which is what later system/follow-up delivery resolves from.
  - file: [extensions/telegram/src/bot-message-context.session.ts:328](https://github.com/openclaw/openclaw/blob/ecac696643c604c6d854341bfe6424bfa31423e9/extensions/telegram/src/bot-message-context.session.ts#L328)
  - sha: [ecac696643c6](https://github.com/openclaw/openclaw/commit/ecac696643c604c6d854341bfe6424bfa31423e9)
