## Summary

Current `main` now resolves the selected session model/provider ahead of stale runtime or default-model metadata, and the live-session plus gateway/session tests cover the exact override-vs-stale-runtime case described in this report.

## Evidence

- **Selected-model resolver prefers session override over stale runtime fields:** `resolvePersistedSelectedModelRef()` is the shared helper for persisted model metadata and explicitly prefers `overrideProvider`/`overrideModel` before falling back to runtime fields.
  - file: [src/agents/model-selection.ts:144](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/src/agents/model-selection.ts#L144)

  - file: [src/agents/live-model-switch.ts:19](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/src/agents/live-model-switch.ts#L19)

- **Session override application clears stale runtime model/provider state:** When a model override is applied, stale `entry.model` and `entry.modelProvider` are deleted so old runtime/default-model identity does not linger on session surfaces.
  - file: [src/sessions/model-overrides.ts:56](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/src/sessions/model-overrides.ts#L56)

- **Gateway session model resolution now honors persisted selected model:** `resolveSessionModelRef()` normalizes stored overrides and resolves the persisted selected model before falling back to configured defaults, which is the gateway/session path behind session metadata surfaces.
  - file: [src/gateway/session-utils.ts:1088](https://github.com/openclaw/openclaw/blob/6bc0dc8fb6714b4e858fb65f9f8051e6b64c8440/src/gateway/session-utils.ts#L1088)
