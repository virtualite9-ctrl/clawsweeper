## Summary

Current main does not show a WebChat history-loading regression here. The reported "yesterday vs today" blank-slate behavior matches OpenClaw's default daily session rollover, not a loss of transcript retrieval for the active session.

## Evidence

- **Default daily rollover is intentional:** Session management docs state that sessions are reused until they expire, and the default behavior is a new session at 4:00 AM local time on the gateway host.
  - file: [docs/concepts/session.md:57](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/docs/concepts/session.md#L57)
  - command: `nl -ba docs/concepts/session.md | sed -n '57,80p'`
  - sha: [34896839ba22](https://github.com/openclaw/openclaw/commit/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534)
- **Reset policy defaults to daily at 4 AM:** Current code sets `DEFAULT_RESET_MODE` to `daily`, `DEFAULT_RESET_AT_HOUR` to `4`, and marks a session stale when `updatedAt` is before that reset boundary.
  - file: [src/config/sessions/reset-policy.ts:20](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/src/config/sessions/reset-policy.ts#L20)
  - command: `nl -ba src/config/sessions/reset-policy.ts | sed -n '20,91p'`
  - sha: [34896839ba22](https://github.com/openclaw/openclaw/commit/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534)
- **Stale sessions get a new sessionId:** `initSessionState()` evaluates freshness and, when the existing entry is no longer fresh, creates a new `sessionId` instead of reusing the old transcript. That matches a yesterday/today rollover producing a blank current session while the old `.jsonl` still exists on disk.
  - file: [src/auto-reply/reply/session.ts:422](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/src/auto-reply/reply/session.ts#L422)
  - command: `sed -n '422,520p' src/auto-reply/reply/session.ts`
  - sha: [34896839ba22](https://github.com/openclaw/openclaw/commit/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534)
- **WebChat loads history for the active session:** The UI requests `chat.history` for the selected `sessionKey`, and the gateway serves messages from the current session entry's `sessionId`. The UI also reloads history when `session.message` arrives for the active session.
  - file: [ui/src/ui/controllers/chat.ts:139](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/ui/src/ui/controllers/chat.ts#L139)
