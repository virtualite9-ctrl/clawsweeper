## Summary

Close #56293 as a duplicate of the older open TUI chat timestamp request tracked by #48708, with related implementation work in #49069. Current main still does not implement the requested timestamp toggle, so the right cleanup is consolidation rather than implemented-on-main.

## Best Possible Solution

Close #56293 as a duplicate and keep the product discussion plus any implementation review centralized on #48708 and PR #49069. If maintainers want the feature, it should add a TUI option/config setting, preserve message timestamps through history/event rendering, and cover user/assistant/pending/history cases in src/tui tests.
