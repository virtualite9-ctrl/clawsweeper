## Summary

Keep open. Current `main` still does not show a concrete fix for `google-vertex` ADC/Bearer auth. The docs say Vertex uses gcloud ADC, but the `google-vertex` path still resolves auth as a plain `apiKey`/marker and the shared Gemini auth helper still turns non-JSON values into `x-goog-api-key` headers. I did not find a `google-vertex` runtime auth exchange hook on `main` that would convert ADC into a Bearer token before requests are sent.
