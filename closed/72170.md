## Summary

Keep PR #72170 open. Current main still has only generic Kubernetes docs and an Azure VM guide; it does not implement AKS cluster creation, ACR integration, or AKS-specific ingress guidance. The PR is a focused docs contribution in an established hosting/deployment docs surface, but it still needs maintainer review for cloud-cost, security, and placeholder accuracy before merge.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is a concise AKS section or dedicated AKS install page that preserves the non-loopback bind prerequisite for ingress, uses globally unique registry placeholders instead of concrete ACR names, keeps cost claims conservative, and links cleanly from the existing hosting/Kubernetes docs.
