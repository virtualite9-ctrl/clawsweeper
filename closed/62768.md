## Summary

Current `main` already implements local session usage persistence plus aggregate usage reporting through chat commands, Gateway APIs, and CLI surfaces, so the requested feature is already present.

## Risks / Open Questions

- There is not an exact top-level `openclaw usage --week` alias; the shipped CLI surface is `openclaw gateway usage-cost --days N` plus Gateway API/chat commands.
- USD cost display is still auth/provider dependent; OAuth/subscription-style flows can show token usage without a dollar estimate.
