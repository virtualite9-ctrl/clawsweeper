## Summary

PR #52529 adds an optional prompt/template pack for a hardened SOUL.md, but current OpenClaw scope directs optional agent guidance and capability packs to ClawHub skills/plugins, while core docs already steer prompt-injection hardening toward enforceable controls such as tool policy, approvals, sandboxing, allowlists, and workspace access restrictions. I found no missing core extension API that blocks external implementation.

## Best Possible Solution

Close this core PR and move the ROBUST_SOUL idea to a ClawHub skill/bundle or plugin. A simple ClawHub skill can ship the template and usage notes as supporting files; a stronger plugin can use before_prompt_build/before_tool_call hooks for enforceable guardrails. Core docs should continue to emphasize sandboxing, tool policy, approvals, allowlists, and concise workspace instructions.
