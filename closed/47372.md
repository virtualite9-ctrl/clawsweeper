## Summary

Keep open. Current `main` still uses the OpenAI Codex OAuth URL generated by the pinned `@mariozechner/pi-ai@0.70.0` package, and that exact package still hardcodes `scope=openid profile email offline_access` with no `model.request`. OpenClaw does not add or rewrite the scope locally, so the reported 401 root cause is still present on current `main`.
