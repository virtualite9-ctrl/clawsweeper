## Summary

Close as ClawHub/plugin work. Current main provides the generic session deletion API and plugin HTTP/runtime surfaces needed to build an external-event bridge; a Linear-specific state matcher and cleanup policy should live as a plugin or operator automation rather than a new core lifecycle DSL.

## Best Possible Solution

Keep OpenClaw core focused on the generic session API and plugin/runtime hooks. Implement the Linear/event-pattern cleanup as a ClawHub plugin or external operator automation that verifies the webhook, maps the external work item to a session key, and calls `sessions.delete` or the plugin runtime deletion helper. If building that plugin exposes a missing generic SDK seam, open a narrow plugin-API issue for that seam.
