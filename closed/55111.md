## Summary

Keep open. Current `main` still treats sub-1024-byte audio as a skipped transcription with no synthetic transcript output, so the reported missing-context behavior is still present. The PR is directionally useful, but the existing review discussion raises an unresolved provider/decision-attribution concern that still needs maintainer judgment rather than cleanup-closing.

## Risks / Open Questions

- The PR's proposed synthetic-output path still appears to need maintainer review for attribution semantics: status/reporting code reads provider/model from `attachment.chosen` (`src/status/status-message.ts:413`), while `buildModelDecision` derives that from the real configured entry (`src/media-understanding/runner.entries.ts:327`). Marking a synthetic fallback as `outcome: "success"` on the original entry could therefore report a real provider/model for a synthetic transcript.
- The PR is small and directly tied to issue #48944, so closing it as stale/obsolete would risk dropping a still-valid bug fix rather than cleaning up noise.
