## Summary

Current `main` already fixes the underlying Feishu reply-threading behavior in a different shape: normal group replies stay inline by default, topic-thread behavior is gated behind the existing `replyInThread` config, and the send path is now centralized in `bot.ts`/`reply-dispatcher.ts` rather than this PR's outbound-only `replyToMode` switch.
