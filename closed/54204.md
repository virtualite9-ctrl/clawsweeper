Current `main` already implements the requested external-plugin SDK alias fix: plugin loader config now threads a `moduleUrl`/`importerUrl` hint into plugin-sdk alias resolution, there is a dedicated regression test for user-installed plugins resolving via that hint, and the changelog records the same fix as having landed separately under `#54283`.
