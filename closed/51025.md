## Summary

Current main already solves the Bailian/DashScope zero-token usage problem through endpoint-based streaming-usage compatibility, stream usage parsing, and nested event usage unwrapping, so PR #51025 is obsolete.

## Best Possible Solution

Close #51025 as implemented by the shipped DashScope/Bailian streaming usage path. Keep the current endpoint-capability and stream parsing implementation; only revisit a `normalizeUsage({ usage: ... })` helper extension if a new production call site needs full-envelope normalization as an explicit contract.
