## Summary

Keep open. Current main still drives the Signal SSE subscription through the plugin SDK fetch resolver, so the reported Node 24 `fetch failed` transport path is not fixed. The group-message handler itself processes `groupInfo` events once they arrive, which supports the issue's diagnosis that the drop is at the SSE transport boundary rather than group-policy gating. This is not a ClawHub/plugin-scope close, duplicate, or implemented-on-main cleanup candidate.

## Best Possible Solution

Keep #51716 open and fix the Signal SSE transport in `extensions/signal/src/client.ts`: replace the long-lived `/api/v1/events` SSE subscription with a node:http/node:https streaming client or equivalent non-fetch transport, keep JSON-RPC fetch behavior separate, preserve abort/reconnect semantics, and add regression coverage for SSE line parsing plus a fetch-failure scenario on the Signal event path.
