## Summary

Current main does not show the downgrade confirmation for known package upgrades. The confirmation path is still gated to downgrade risk, while this PR's latest diff broadens the gate to normal upgrades and would regress non-interactive update flows.

## Best Possible Solution

Close this PR. Keep the current downgrade-only confirmation behavior; if the `latest (unknown)` wording should change, ask for a narrow copy-only PR that does not add confirmation prompts to known upgrades and includes focused regression coverage.
