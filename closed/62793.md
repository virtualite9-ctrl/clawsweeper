## Summary

Current `main` already guards the late subagent exec-output path that caused `Agent listener invoked outside active run`, and the fix was shipped in `v2026.4.23` with matching regression coverage.

## Evidence

- **Current main suppresses late exec updates after run end:** `runExecProcess()` tracks an `updatesDisabled` flag, makes `emitUpdate()` a no-op once the session is backgrounded/exited/disabled, and flips the flag before exit/error cleanup so residual stdout/stderr cannot call back into a disposed agent run.
