## Summary

Current `main` already implements the requested behavior: managed Dreaming now runs as an isolated cron job instead of a heartbeat-coupled path, and the cron service replays missed recurring jobs on Gateway startup/restart.

## Evidence

- **Unreleased changelog entry decouples Dreaming from heartbeat:** The current `## Unreleased` changelog says managed Dreaming cron was decoupled from heartbeat by running it as an isolated lightweight agent turn, with `openclaw doctor --fix` migrating stale persisted jobs.
  - file: [CHANGELOG.md:158](https://github.com/openclaw/openclaw/blob/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a/CHANGELOG.md#L158)
  - sha: [c997a9f9788e](https://github.com/openclaw/openclaw/commit/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a)
- **Managed Dreaming job now uses isolated cron execution:** `buildManagedDreamingCronJob()` creates the managed Dreaming job with `sessionTarget: "isolated"`, `wakeMode: "now"`, and an agent-turn payload, which makes it a normal cron-run target rather than a heartbeat-only reminder.
  - file: [extensions/memory-core/src/dreaming.ts:153](https://github.com/openclaw/openclaw/blob/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a/extensions/memory-core/src/dreaming.ts#L153)
  - sha: [c997a9f9788e](https://github.com/openclaw/openclaw/commit/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a)
- **Gateway startup reconciles the managed Dreaming cron job:** `registerShortTermPromotionDreaming()` hooks `gateway_start` and reconciles the managed Dreaming cron job using the live cron getter from the startup context.
  - file: [extensions/memory-core/src/dreaming.ts:753](https://github.com/openclaw/openclaw/blob/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a/extensions/memory-core/src/dreaming.ts#L753)
  - sha: [c997a9f9788e](https://github.com/openclaw/openclaw/commit/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a)
- **Cron service replays missed recurring jobs on startup:** `CronService.start()` clears stale running markers and then calls `runMissedJobs(...)`, so overdue recurring jobs are caught up during service startup.
  - file: [src/cron/service/ops.ts:87](https://github.com/openclaw/openclaw/blob/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a/src/cron/service/ops.ts#L87)
  - sha: [c997a9f9788e](https://github.com/openclaw/openclaw/commit/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a)
- **Startup catch-up explicitly includes missed cron slots:** The startup catch-up planner collects runnable jobs with `allowCronMissedRunByLastRun: true`, which is the code path for replaying a missed recurring cron slot after restart.
  - file: [src/cron/service/timer.ts:990](https://github.com/openclaw/openclaw/blob/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a/src/cron/service/timer.ts#L990)
  - sha: [c997a9f9788e](https://github.com/openclaw/openclaw/commit/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a)
- **Regression tests cover overdue and missed-slot replay on restart:** Tests assert that an overdue recurring job executes immediately on start and that a missed cron slot is replayed after restart.
  - file: [src/cron/service.restart-catchup.test.ts:79](https://github.com/openclaw/openclaw/blob/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a/src/cron/service.restart-catchup.test.ts#L79)
  - sha: [c997a9f9788e](https://github.com/openclaw/openclaw/commit/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a)
- **Doctor migrates legacy heartbeat-coupled Dreaming jobs:** `openclaw doctor --fix` rewrites stale managed Dreaming jobs to the isolated agent-turn shape so older persisted configs also pick up the new behavior.
  - file: [src/commands/doctor-cron.ts:196](https://github.com/openclaw/openclaw/blob/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a/src/commands/doctor-cron.ts#L196)
  - sha: [c997a9f9788e](https://github.com/openclaw/openclaw/commit/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a)
