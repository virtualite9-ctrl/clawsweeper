## Summary

Close PR #66526 as ClawHub/external-plugin scope. KlingAI image/video generation is a useful provider integration, and the PR discussion shows the author addressed several implementation review comments, but current OpenClaw policy keeps optional capability out of core when it can be built with existing plugin APIs. Current main already exposes public image/video provider registration, manifest contracts, auth metadata, provider HTTP/auth helpers, and ClawHub/npm plugin publishing/install flows, so this does not need a core-bundled extension.

## Best Possible Solution

Close this core PR and ask the contributor to publish the KlingAI provider as an external OpenClaw plugin through ClawHub or npm. If publishing exposes a concrete missing SDK/auth/onboarding capability, track that as a narrow plugin API issue or PR instead of bundling a one-off provider into core.
