## Summary

Current main implements the configure/model-picker behavior requested in #60769: when the selected model list contains only the configured primary, stale visible fallback models are removed instead of being spread back into `agents.defaults.model.fallbacks`. The current tests cover the reporter's case and related scoped-provider cases.

## Best Possible Solution

Close #60769 as implemented on current main. Keep the shipped model-picker fallback clearing behavior and its regression tests; any separate desire to make `openclaw models fallbacks clear` delete an empty `fallbacks: []` key instead of writing an empty array should be tracked separately because it no longer causes the reported stale fallback cascade.
