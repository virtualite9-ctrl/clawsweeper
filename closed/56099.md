## Summary

PR #56099 should close as superseded by open PR #59884, not as implemented. Current main still has the compaction-auth bug, but #56099's proposed fallback was reviewed as unsafe because it proceeds without verified credentials while downstream summarization can return a generic fallback. PR #59884 now tracks the same remaining compaction-auth compatibility work with a safer resolver and focused regression coverage.

## Best Possible Solution

Close #56099 as superseded and keep #59884, or a maintainer-owned replacement based on it, as the canonical thread for the compaction auth compatibility fix. The eventual fix should preserve the history-safety preflight by resolving credentials through compatible registry APIs instead of proceeding with no verified auth.
