## Summary

Current main already skips the Tailscale CLI lookup when `gateway.tailscale.mode` is `off` in the refactored `status --all` scan path, so this PR's intended fix is already present and the patch is obsolete.

## Evidence

- **Off-mode scan short-circuits Tailscale lookup:** `createStatusScanCoreBootstrap()` now sets `tailscaleDnsPromise` to `Promise.resolve(null)` when `cfg.gateway?.tailscale?.mode ?? "off"` is `"off"`, and only calls `getTailnetHostname(...)` otherwise. That is the current `status --all` Tailscale probe path.
