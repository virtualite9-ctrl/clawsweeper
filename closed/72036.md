## Summary

Current main already implements the non-fatal Feishu startup behavior requested by #72036. The bot ping remains best-effort: timeouts log that startup continues, 404/request failures are converted to failed probe results, accounts still start with unknown bot identity, and background identity recovery handles the missing identity later. The same Feishu startup source is present in the v2026.4.24 tag for bundled @openclaw/feishu 2026.4.20.

## Best Possible Solution

Keep the current best-effort Feishu startup behavior: the bot ping should not be required for startup, missing identity should keep mention-gated groups conservative, and background recovery should refresh identity later. If someone can reproduce a different published @openclaw/feishu artifact than the v2026.4.24 source, track that as a package/distribution mismatch; keep the Bonjour crash path separate from this Feishu ping issue.
