## Summary

Current main already sends Telegram typing indicators during normal active agent runs. Telegram wires its reply pipeline to `sendTyping`, the shared typing controller starts on run/text/reasoning/tool activity depending on `typingMode`, tests cover Telegram `sendChatAction("typing")`, and the public docs describe the shipped `typingMode`/`typingIntervalSeconds` controls.

## Best Possible Solution

Close as implemented and keep the existing shipped `typingMode`/typing lifecycle. If maintainers want better behavior after a very long typing TTL expires, handle that separately through the narrower #43204 discussion rather than keeping this broad Telegram active-processing request open.
