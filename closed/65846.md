## Summary

Current main already tears down bundled MCP runtime after `openclaw agent --local` runs, with regression coverage for the local CLI path and transport-level cleanup for lingering stdio children. That matches the reported symptom of a completed JSON result with a process that stays alive.

## Evidence

- **Local CLI opts into one-shot MCP cleanup:** `agentCliCommand` sets `cleanupBundleMcpOnRunEnd: opts.local === true` before calling the embedded agent path, so `openclaw agent --local` explicitly requests end-of-run teardown.
