## Summary

Current Web Chat already renders tool-call rows as expandable cards, and the expanded state shows the full serialized tool input/output instead of the clipped preview. The same implementation is present in the latest shipped release snapshot, so this issue is already fixed well enough on `main` and in `v2026.4.23`.

## Evidence
