## Summary

Current `main` already adds `attachments` metadata to `before_model_resolve`, wires image attachment metadata from the embedded runner into that hook, documents the behavior, and includes regression tests. The exact gap described in issue #65754 is implemented on `main` but does not appear in the latest tagged release provided here (`v2026.4.23`), so the fix looks post-release.
