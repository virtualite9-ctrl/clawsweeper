## Summary

Keep #50733 open. Current `origin/main` still propagates Slack `already_reacted` / `no_reaction` errors through the shared Slack reaction helpers and the agent `react` action path. The Slack status-reaction adapter has a local partial suppression, but that does not cover the general message reaction tool path described in the issue. The related PRs in the provided context (#9520, #50864, #51226) were closed without merge, so they do not make the issue obsolete.

## Best Possible Solution

Keep the issue open and fix it in `extensions/slack/src/actions.ts` by making `reactSlackMessage` treat Slack `already_reacted` as a no-op and `removeSlackReaction` treat `no_reaction` as a no-op, while rethrowing all other errors. Add focused helper-level tests plus action-runtime coverage so both direct helper callers and the agent `react` tool path are protected.
