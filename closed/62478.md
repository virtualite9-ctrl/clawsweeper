## Summary

Current OpenClaw main already supports the requested behavior well enough: `sessions_spawn` is documented and implemented as non-blocking, child-run waiting happens in background registry code rather than the parent turn, completion delivery is asynchronous back into the requester session/channel, and `sessions_yield` cleanly frees the parent session when the agent wants to end the turn and wait for later results.

## Evidence

- **Public docs: sessions_spawn is non-blocking background work:** The shipped docs say `sessions_spawn` creates an isolated session for a background task, is always non-blocking, returns immediately with `runId` and `childSessionKey`, and completion is announced back to the requester.
