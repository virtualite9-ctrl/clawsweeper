## Summary

Current main already loads `skills.load.extraDirs` into the same per-session skills snapshot used for agent prompts, and current tests cover both extra-dir prompt injection and rejection of workspace symlink escapes. The reported `eligible`-but-not-injected split is not consistent with the shipped code in `v2026.4.23`.

## Evidence

- **Extra dirs are part of session skill discovery:** `loadSkillEntries()` resolves `skills.load.extraDirs`, loads them as `openclaw-extra`, merges them with other skill sources, and `resolveWorkspaceSkillPromptState()` turns the eligible merged entries into the prompt used for sessions.

  - file: [src/agents/agent-command.ts:590](https://github.com/openclaw/openclaw/blob/250acdd2a28c190ed7934e3ba99022f24123d3c9/src/agents/agent-command.ts#L590)
