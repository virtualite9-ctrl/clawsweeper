## Summary

Closed as `cannot_reproduce`: current `main` still uses the plain `client.embeddings.create(params)` path in `extensions/memory-lancedb/index.ts:179-189`, but the current OpenAI SDK explicitly defaults omitted `encoding_format` to `base64` and decodes embeddings back to floats client-side, while current Ollama docs/tests show `/v1/embeddings` support for `encoding format` including `base64` handling. That makes the PR's workaround look obsolete against current upstream behavior rather than an unfixed `main` bug. ([raw.githubusercontent.com](https://raw.githubusercontent.com/openai/openai-node/master/src/resources/embeddings.ts))
