## Summary

This PR is superseded by the current provider-profile thinking behavior on main. Main no longer treats preserved raw `xhigh` on unsupported models as the desired fix: unsupported stored/default levels are remapped to a supported level in directive/model-switch, normal reply, and agent command paths, the behavior is documented, and there is regression coverage for remapping during model switches. The linked #50601 is already closed, and remaining broader thinking/status inconsistencies are better tracked by the newer open #59416 rather than merging this stale preservation patch.

## Best Possible Solution

Close this stale PR and keep the current provider-profile remapping contract on main. Any remaining thinking-level display or xhigh capability work should be handled in the newer canonical issue #59416 with targeted fixes and tests against the current remap behavior.
