## Summary

Close as cannot reproduce. On current main, the Feishu manifest `skills` field is metadata for workspace skill directories, not runtime tool registration, and `status --verbose` now uses snapshot-only plugin inspection instead of full plugin registration.

## Evidence

- **Manifest `skills` is not a runtime registration path:** The plugin manifest docs explicitly say `openclaw.plugin.json` is read before plugin code loads and must not be used for registering runtime behavior. The `skills` field is documented only as skill directories relative to the plugin root.
  - file: [docs/plugins/manifest.md:39](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/docs/plugins/manifest.md#L39)

- **Plugin skill loading only resolves directories:** `resolvePluginSkillDirs` reads `record.skills`, resolves directories under the plugin root, and returns paths. It does not call `registerTool` or any plugin runtime registration hook.
  - file: [src/agents/skills/plugin-skills.ts:46](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/agents/skills/plugin-skills.ts#L46)

The manifest `skills` entry exists alongside this, but that manifest field is not a second tool-registration path.
