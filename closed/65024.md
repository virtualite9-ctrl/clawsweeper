## Summary

Close as not_actionable_in_repo. Current OpenClaw main routes direct messages with session.dmScope=main to the agent main session and documents that contract, while the Clawline-specific stream surface and prior src/clawline/server.ts implementation are not present in this repository. The related #15382 timeline points to external clickety-clacks/clawdbot commits for Clawline stream gating, so the remaining visible agent:main:clawline:<userId>:dm stream should be handled in the external Clawline provider or its persisted stream cleanup unless a missing OpenClaw plugin/session API is identified.

## Best Possible Solution

Keep OpenClaw core's existing dmScope=main routing contract. Handle the visible Clawline personal DM stream in the external Clawline provider or its persisted stream store, and reopen/open a focused OpenClaw core issue only if that provider identifies a missing plugin/session API needed to hide or migrate stale streams.
