## Summary

Close as ClawHub/plugin scope. The request is useful and concrete, and current main confirms Venice is a bundled text provider with `VENICE_API_KEY` auth metadata, but it has no Venice image-generation provider, manifest `imageGenerationProviders` contract, or Venice image endpoint wiring. OpenClaw already exposes image-generation provider registration through the plugin SDK and resolves image-generation providers from plugin capability registrations, so this can be built outside core without a missing extension API.

## Best Possible Solution

Close the core issue as ClawHub/plugin scope. The best path is an external native Venice image-generation provider plugin using `api.registerImageGenerationProvider(...)` and the existing Venice auth metadata/API-key resolution. If maintainers later decide Venice image support should be first-party, it should come as a focused bundled-plugin PR that updates `extensions/venice`, its manifest contract, docs, and provider tests, not as an open-ended core tracking issue.
