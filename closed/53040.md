## Summary

Keep #53040 open. Current main still uses fetch-backed Signal SSE and fetch-backed Signal RPC send on the exact paths named in the report, with only retry/backoff around SSE failures and no visible Node 25/macOS/npm-specific fallback. Related #51716 tracks the same Signal SSE fetch-failure family, but #53040 adds Node 25/macOS/npm install and outbound send failure evidence, and it was recently assigned, so automatic duplicate/implemented cleanup is not conservative.

## Best Possible Solution

Keep the issue open, or have a maintainer explicitly consolidate it with #51716 while preserving the Node 25/macOS/npm and outbound send evidence. The likely fix belongs in `extensions/signal`: harden or replace the Signal SSE and RPC transport for Node 24/25 npm installs, then add focused regression coverage for `/api/v1/events` streaming and `/api/v1/rpc` send behavior; update Node support docs only if the intended support range changes.
