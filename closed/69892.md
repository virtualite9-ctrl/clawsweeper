## Summary

Current main and the v2026.4.24 tag implement the reported `toolsAllow` forwarding/filtering path for Active Memory. Active Memory passes `toolsAllow: ["memory_search", "memory_get"]`, `runEmbeddedPiAgent` forwards it into `runEmbeddedAttemptWithBackend`, PI and Codex harness paths filter tools with it, and the related MCP schema-bloat path is covered by a helper/test that skips bundle MCP when the allowlist cannot reach bundle MCP tool names. Remaining timeout or strict-provider empty-tool behavior is separately represented by the open related reports/PRs in the issue context, especially #65309 and #70790.

## Best Possible Solution

Close this issue as implemented for the stale `toolsAllow` forwarding/filtering root cause. Keep the shipped v2026.4.24 implementation, and route any still-reproducing Active Memory timeout or strict-provider empty-tool failures to the narrower related trackers (#65309 and #70790) instead of reopening this forwarding report.
