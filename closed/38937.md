## Summary

Keep #38937 open. Current main fixes the prompt/parser mismatch for tagged TTS, but the streaming cross-block leakage remains plausible in both generic and ACP dispatch paths: block payloads are still delivered independently, and final accumulated TTS audio does not edit or retract already visible text blocks.

## Best Possible Solution

Keep the issue open and implement a streaming-aware tagged-TTS sanitizer/buffer in the reply dispatch path. It should prevent partial TTS tags from being delivered as visible block text, preserve final accumulated TTS generation, and add regression tests for tags split across block boundaries in generic dispatch and ACP delivery.

## Evidence
