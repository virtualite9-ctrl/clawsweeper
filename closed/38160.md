## Summary

Close PR #38160 as ClawHub/plugin work. Current main already covers hosted-to-local fallback through generic model fallback configuration, while the remaining semantic prompt-to-response cache is optional agent behavior that current plugin hooks can implement without a new core API. The PR is also broad and has unresolved correctness review findings.

## Best Possible Solution

Do not merge this broad core PR. Keep using and documenting agents.defaults.model.fallbacks for hosted-to-local fallback, and pursue semantic response caching as a ClawHub plugin using before_agent_reply plus llm_output hooks. If plugin authors find a concrete missing hook or SDK contract, open a narrow plugin API proposal instead of adding one-off core agent-runner behavior.
