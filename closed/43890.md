## Summary

Current `main` already routes Discord ACP thread delivery through the generic session-binding/bound-delivery path and thread-id-based outbound lookup, so the specific ACP-vs-subagent and parent-session-key mismatches described in PR #43890 do not appear to be live on `81666e586aa6d4305e6472d7075e62f63f701831`.

## Evidence

- **Subagent completion routing now resolves bound destinations before the Discord-specific hook:** `resolveSubagentCompletionOrigin` first asks `createBoundDeliveryRouter().resolveDestination(...)` for a bound route, and only falls back to `subagent_delivery_target` hooks if that bound lookup fails.
  - file: [src/agents/subagent-announce-delivery.ts:239](https://github.com/openclaw/openclaw/blob/81666e586aa6d4305e6472d7075e62f63f701831/src/agents/subagent-announce-delivery.ts#L239)
  - command: `nl -ba src/agents/subagent-announce-delivery.ts | sed -n '214,297p'`
  - sha: [81666e586aa6](https://github.com/openclaw/openclaw/commit/81666e586aa6d4305e6472d7075e62f63f701831)
- **Discord thread bindings are registered as session bindings, with ACP mapped into that path:** The Discord thread-binding manager maps public session-binding target kind `session` to internal thread-binding target kind `acp`, and registers a `SessionBindingAdapter` whose `listBySession` returns Discord thread bindings for the target session key.
  - file: [extensions/discord/src/monitor/thread-bindings.manager.ts:133](https://github.com/openclaw/openclaw/blob/81666e586aa6d4305e6472d7075e62f63f701831/extensions/discord/src/monitor/thread-bindings.manager.ts#L133)
  - command: `nl -ba extensions/discord/src/monitor/thread-bindings.manager.ts | sed -n '128,142p;613,731p'`
  - sha: [81666e586aa6](https://github.com/openclaw/openclaw/commit/81666e586aa6d4305e6472d7075e62f63f701831)
- **Inbound Discord ACP flows propagate the bound child session key into reply delivery:** Preflight derives `boundSessionKey` from the bound thread/configured binding, and message processing uses `boundSessionKey` as `ctxPayload.SessionKey`, so `deliverDiscordReply` is called with the bound ACP session key rather than the parent route key in normal bound-thread flows.
  - file: [extensions/discord/src/monitor/message-handler.preflight.ts:684](https://github.com/openclaw/openclaw/blob/81666e586aa6d4305e6472d7075e62f63f701831/extensions/discord/src/monitor/message-handler.preflight.ts#L684)

  - sha: [81666e586aa6](https://github.com/openclaw/openclaw/commit/81666e586aa6d4305e6472d7075e62f63f701831)
- **Current generic Discord outbound webhook delivery resolves by thread ID directly:** The current Discord outbound adapter looks up webhook bindings via `manager.getByThreadId(threadId)`, so thread-bound outbound delivery no longer depends on session-key lookup in that path.
  - file: [extensions/discord/src/outbound-adapter.ts:115](https://github.com/openclaw/openclaw/blob/81666e586aa6d4305e6472d7075e62f63f701831/extensions/discord/src/outbound-adapter.ts#L115)
