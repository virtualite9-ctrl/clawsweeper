## Summary

Current main separates async exec system-event blocks from transcript-visible Web UI history and treats notifyOnExit wakes as untrusted runtime context rather than user-authored chat text. The old `System (untrusted): ... shown in the system messages above` feedback-loop shape is covered by source and regression tests.

## Best Possible Solution

Close this issue as fixed on current main. Keep the shipped transcript split and untrusted exec-event handling; any remaining work should be a narrower UX issue about how Web UI displays async exec completion summaries or the documented `tools.exec.notifyOnExit` control, not this feedback-loop bug.
