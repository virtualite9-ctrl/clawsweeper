## Summary

Current main already fixes the reported packaged-install failure path. Bundled plugin dependencies are no longer eagerly installed during OpenClaw package postinstall, and active bundled plugins now repair missing runtime dependencies in an OpenClaw-managed runtime-deps root with a minimal manifest, explicit package specs, an isolated npm cache, and `--ignore-scripts`. Discord’s `@buape/carbon` case has regression coverage. The fix is documented in the 2026.4.22 changelog and is present in the latest release line.

## Best Possible Solution

Keep the shipped bundled-plugin runtime-dependency repair model: do not eagerly install bundled plugin deps during package postinstall, repair only active packaged plugins into the managed runtime-deps root, and keep Discord/@buape/carbon coverage in place. If Catalina still fails on v2026.4.22 or newer, it should be tracked as a fresh issue with current logs showing the new dependency path and failing binary.
