## Summary

Current main already includes `sessions_send`/`sessions_spawn` in the `coding` tool profile for gateway-backed agent sessions, and the same behavior is present in the shipped `v2026.4.23` release.

## Evidence

- **Coding profile includes session orchestration tools:** The core tool catalog puts `sessions_send`, `sessions_spawn`, `sessions_yield`, and `subagents` in the `coding` profile.
  - file: [src/agents/tool-catalog.ts:161](https://github.com/openclaw/openclaw/blob/9a0b26cafca76d267dd231b5eaceadb071ec749d/src/agents/tool-catalog.ts#L161)

- **Core tool set registers the session tools:** `createOpenClawTools()` registers `createSessionsSendTool()`, `createSessionsSpawnTool()`, `createSessionsYieldTool()`, and `createSubagentsTool()` in the built-in tool set for non-embedded runs.
  - file: [src/agents/openclaw-tools.ts:273](https://github.com/openclaw/openclaw/blob/9a0b26cafca76d267dd231b5eaceadb071ec749d/src/agents/openclaw-tools.ts#L273)

  - file: [src/agents/pi-tools.ts:585](https://github.com/openclaw/openclaw/blob/9a0b26cafca76d267dd231b5eaceadb071ec749d/src/agents/pi-tools.ts#L585)

  - file: [docs/tools/index.md:136](https://github.com/openclaw/openclaw/blob/9a0b26cafca76d267dd231b5eaceadb071ec749d/docs/tools/index.md#L136)

## Risks / Open Questions

- I did not run a live gateway reproduction in this read-only review; this decision is based on current code, docs, and the shipped `v2026.4.23` snapshot.
- Embedded no-gateway runs intentionally omit `sessions_send` and `sessions_spawn`, so a runtime/session-type mismatch can still look similar to the original report.
