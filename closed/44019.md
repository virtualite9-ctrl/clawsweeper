## Summary

Current `main` already handles standalone/provider HTML transport failures through a newer centralized error-classification path: HTML pages are detected, classified as runtime failures or timeout-style failover signals, rewritten to safe user-facing copy, and raw error text is suppressed from embedded payloads. This PR’s safety goal is already covered on `main`, so the branch is superseded.
