## Summary

Current `main` does not support `OPENCLAW_TELEGRAM_TOKEN` in the Docker project `.env` as a Telegram auth input. Telegram runtime only resolves `channels.telegram.botToken`/`tokenFile` or `TELEGRAM_BOT_TOKEN`, and the Docker flow keeps bot secrets in `~/.openclaw/.env` or `channels add --channel telegram --token ...`. On that evidence, issue #62690 does not establish a reproducible core regression on current `main`.

## Evidence

- **Telegram runtime only reads `TELEGRAM_BOT_TOKEN` as the env fallback:** `resolveTelegramToken` falls back to `process.env.TELEGRAM_BOT_TOKEN` for the default account; there is no `OPENCLAW_TELEGRAM_TOKEN` path in the current Telegram token resolver.
