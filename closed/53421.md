## Summary

Close as `clawhub`: this PR adds an optional bundled plugin, but current OpenClaw policy/docs direct new optional plugins to ClawHub or npm instead of the core repo, and the needed routing hook already exists on `main` for external plugins.

## Evidence

- **Vision prefers optional plugins outside core:** `VISION.md:54-61` says optional capability should usually ship as plugins, the preferred path is npm distribution plus local extension loading, contributors should host and maintain plugins in their own repository, and the bar for adding optional plugins to core is intentionally high.
  - file: [VISION.md:54](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/VISION.md#L54)

  - file: [docs/plugins/building-plugins.md:16](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/docs/plugins/building-plugins.md#L16)

  - file: [docs/plugins/community.md:9](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/docs/plugins/community.md#L9)
