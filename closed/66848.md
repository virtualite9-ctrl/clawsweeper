## Summary

Keep open. Current `main` still has the same unsafe `prompter.text(...)` -> `.trim()` path described in the issue, the issue-version commit `323493f` already had that same code, and the proposed fix in PR #66882 is still open/unmerged rather than implemented on `main`.
