## Summary

Current `main` already contains targeted Codex auth-profile repair logic for stale `openai-codex:default` state, plus regression tests covering the stale-default/fresh-named-profile scenario and refresh fallback handling.

## Evidence

- **Codex login now creates named OAuth profiles:** The OpenAI Codex login flow resolves identity from the access token/email and passes that identity into the standard OAuth auth-result builder, which derives the persisted profile id from `profileName` or `email` instead of forcing `openai-codex:default`.
  - file: [extensions/openai/openai-codex-provider.ts:325](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/extensions/openai/openai-codex-provider.ts#L325)

- **Named profile id generation is explicit:** `resolveCodexAuthIdentity()` returns `{ email, profileName: email }` when email is available, and `buildOauthProviderAuthResult()` builds the profile id from that identity.
  - file: [extensions/openai/openai-codex-auth-identity.ts:69](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/extensions/openai/openai-codex-auth-identity.ts#L69)
  - command: `nl -ba extensions/openai/openai-codex-auth-identity.ts | sed -n '55,85p'`
- **Runtime merge rewrites stale default references:** `mergeAuthProfileStores()` calls `reconcileMainStoreOAuthProfileDrift()`, which finds a newer usable matching OAuth credential in the main store and rewrites merged `profiles`, `order`, `lastGood`, and `usageStats` away from the legacy `provider:default` entry.
  - file: [src/agents/auth-profiles/persisted.ts:216](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/src/agents/auth-profiles/persisted.ts#L216)

  - file: [src/agents/auth-profiles.ensureauthprofilestore.test.ts:247](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/src/agents/auth-profiles.ensureauthprofilestore.test.ts#L247)

## Close Comment

Closing this as implemented on current `main` after a Codex review.

- Codex OAuth login now derives a named auth profile from the account identity instead of blindly reusing `openai-codex:default` (`extensions/openai/openai-codex-provider.ts:325`, `extensions/openai/openai-codex-auth-identity.ts:69`, `src/plugin-sdk/provider-auth-result.ts:23`).
- Runtime auth-store merge explicitly repairs stale legacy `openai-codex:default` references by switching merged runtime state to the fresher matching OAuth profile and rewriting `order` / `lastGood` (`src/agents/auth-profiles/persisted.ts:216`, `src/agents/auth-profiles/persisted.ts:339`).
- There is a regression test for the same stale-default vs fresh-named-profile shape: runtime loading ends up with only the fresh named profile in active runtime state, and `order` / `lastGood` point to that fresh profile (`src/agents/auth-profiles.ensureauthprofilestore.test.ts:247`).
- Refresh handling now also tries a replacement profile when a legacy default refresh fails, covering Codex refresh-failure cases before surfacing the error (`src/agents/auth-profiles/oauth.ts:346`, `src/agents/auth-profiles/oauth.openai-codex-refresh-fallback.test.ts:416`).
- Session override handling clears non-user auto-pinned auth profiles when they are no longer in the resolved auth order, so stale runtime pins do not persist once the newer credential becomes the effective route (`src/agents/auth-profiles/session-override.ts:108`).
