## Summary

Current main implements configured model fallback for the reported rate-limit, quota, and billing/credits failure paths. The fallback loop builds an ordered candidate chain, advances through configured fallbacks, classifies 402/429/quota/credits errors as failover-worthy, and has regression coverage for billing, quota refresh, insufficient_quota, and cross-provider fallback after rate-limit cooldown. The implementation is present in v2026.4.24.
