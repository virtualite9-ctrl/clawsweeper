## Summary

Close #57509 as superseded by open PR #57548. The current main branch still records Telegram native-command inbound session metadata under the slash envelope key, and current tests still assert that old slash-key behavior for ACP topic bindings. PR #57548 directly tracks the same bug, links as closing #57509, and its patch changes the metadata key to prefer CommandTargetSessionKey with targeted regression coverage for private, configured ACP topic, topic-agent, and bound-topic flows.

## Best Possible Solution

Use #57548 as the canonical tracker and land that patch, or an equivalent maintainer-owned change, so Telegram native-command metadata persists to CommandTargetSessionKey instead of the slash envelope key. Treat any remaining model fallback behavior after that fix as a separate, narrower regression report with fresh logs.
