## Summary

Current main implements the concrete fixes for the reported Active Memory timeout path: the recall hook now enforces its own hard timeout with Promise.race, ignores late sub-agent payloads, forwards an abort signal, constrains the embedded run to memory_search/memory_get, and has regression coverage for the hard-deadline behavior. Active Memory remains a bounded blocking pre-reply feature by design; an async/background redesign would be a separate product request.

## Best Possible Solution

Close #70021 as implemented on current main. Keep the shipped bounded blocking recall path, and open a narrower follow-up only if maintainers want to change Active Memory from pre-reply blocking recall to an async/background product design.
