## Summary

Current main already implements the Telegram TTS voice-note path, marks `/tts audio` results for voice delivery, treats MP3/M4A/OGG as voice-compatible on Telegram, and adds Microsoft/Edge output-format fallback plus timeout/config guidance. The remaining Edge timeout mode is documented as best-effort provider behavior rather than a Telegram delivery gap.

## Evidence

- **`/tts audio` now returns a voice-deliverable payload:** Successful `/tts audio` requests return `mediaUrl`, set `audioAsVoice` from provider compatibility, and mark the local media as trusted for delivery.
  - file: [src/auto-reply/reply/commands-tts.ts:154](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/src/auto-reply/reply/commands-tts.ts#L154)
- **Telegram voice delivery accepts MP3/M4A/OGG-compatible audio:** Voice-message compatibility includes `.mp3`, `.m4a`, `.ogg`, `.opus` and matching MIME types, so Telegram is no longer limited to OGG-only voice-note sends.
  - file: [src/media/audio.ts:4](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/src/media/audio.ts#L4)
- **Telegram delivery routes compatible audio through `sendVoice`:** Telegram reply delivery checks `audioAsVoice`, resolves compatibility, and sends compatible audio via `sendVoice` instead of `sendAudio`.
  - file: [extensions/telegram/src/bot/delivery.replies.ts:349](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/extensions/telegram/src/bot/delivery.replies.ts#L349)
- **Telegram coverage includes MP3 voice-note delivery:** The Telegram send tests explicitly cover `asVoice` with `audio/mpeg` / `clip.mp3` and expect `sendVoice`.
  - file: [extensions/telegram/src/send.test.ts:1386](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/extensions/telegram/src/send.test.ts#L1386)
- **Microsoft/Edge TTS now retries unsupported output formats with default MP3:** The bundled Microsoft speech provider retries synthesis with the default MP3 output format when the configured format fails, which directly addresses the output-format adjustment part of the report.
  - file: [extensions/microsoft/speech-provider.ts:233](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/extensions/microsoft/speech-provider.ts#L233)
- **Current docs describe Edge/Microsoft as best-effort and expose timeout/output guidance:** The TTS docs now state that the Microsoft/Edge path is a best-effort hosted service, document `messages.tts.timeoutMs`, and note that Telegram accepts OGG/MP3/M4A with Microsoft falling back to MP3 when a configured format fails.
  - file: [docs/tools/tts.md:24](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/docs/tools/tts.md#L24)
