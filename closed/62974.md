## Summary

Current main now implements the central requested behavior: automatic model fallback persists the selected fallback provider/model across subsequent turns, so OpenClaw no longer probes a known-bad primary on every message and bounces prompt caches between providers. The remaining difference is that main uses an auto override that lasts until /new, /reset, sessions.reset, or fallback exhaustion, rather than a configurable cache-TTL window; that is a narrower tuning follow-up, not a blocker for this issue's observed cache-bouncing problem.

## Best Possible Solution

Close this issue as implemented on current main. Keep the shipped auto fallback override policy because it fixes the reported cross-provider cache bouncing; open a narrower follow-up only if maintainers want configurable cache-TTL-based expiry and proactive return-to-primary probing.
