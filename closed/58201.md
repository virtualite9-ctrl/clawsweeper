## Summary

Issue #58201 asks core to add agent-level plugin config overrides for a stateful memory plugin. Current main does not expose `agents.list[].plugins`, but the product boundary and SDK already support plugin-owned per-agent isolation through plugin config plus agent-aware runtime contexts, so the best cleanup is to close this as ClawHub/plugin-owned rather than add a broad core config merge surface.

## Best Possible Solution

Keep OpenClaw core's plugin config surface global and lean. A `graph-memory` or similar stateful memory plugin should expose its own plugin-owned defaults/agent map or `{agentId}` path template under `plugins.entries.<id>.config`, use the existing `agentId` runtime context to choose storage, and publish that capability through ClawHub. Reopen only for a narrow, concrete missing SDK or memory-runtime API that prevents that plugin implementation.
