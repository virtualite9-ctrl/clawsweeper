## Summary

Current main already implements the reported Bedrock subagent auth behavior, and the same implementation is present in the v2026.4.24 release tag. `sessions_spawn` routes through `spawnSubagentDirect` into the embedded PI agent path; that path now resolves Bedrock `AWS_PROFILE`/AWS SDK auth as `aws-sdk` and seeds PI auth storage with prepared runtime auth or the `__aws_sdk_auth__` sentinel when no literal API key exists, so the upstream preflight no longer fails before the AWS SDK credential chain can sign the request.

## Best Possible Solution

Close as implemented. Keep the shipped v2026.4.24/current-main implementation, and ask the reporter to upgrade to v2026.4.24 or later; if they can still reproduce on that build, a new focused issue should include fresh subagent logs from the upgraded version and the Bedrock/AWS environment markers in use.
