## Summary

This is a lossless-claw context-engine plugin bug report, and the same symptom is already tracked in the plugin repo as Martian-Engineering/lossless-claw#436. Current OpenClaw main exposes the context-engine and provider-stream seams needed for plugin-owned compaction, and core now has native Ollama compaction/simple-summary transport coverage; the remaining summary extraction, API-family fallback, and diagnostics belong in the lossless-claw plugin/ClawHub path rather than OpenClaw core.

## Best Possible Solution

Close the OpenClaw core issue as plugin/ClawHub-owned. Keep any remaining work in Martian-Engineering/lossless-claw#436 or a focused plugin PR, and only reopen/open a new OpenClaw issue if maintainers identify a concrete missing context-engine or provider-runtime API seam in OpenClaw core.
