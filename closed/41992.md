## Summary

Keep open. Current `main` only supports image uploads in the Control UI/WebChat path, while this issue asks for broader file/document upload support. A maintainer comment on April 25, 2026 explicitly says the image-only path is already covered elsewhere and that this broader request should remain tracked separately.

## Evidence

- **Maintainer comment keeps scope separate:** Provided GitHub context shows `vincentkoc` commented on 2026-04-25 that image-only Control UI/WebChat uploads are covered by #51045/current main, but broader document/file upload or upload/download behavior is still missing and should stay tracked in #41992.
- **Control UI attachment accept filter is image-only:** The WebChat attachment support constant is `image/*`, and the MIME guard only accepts values starting with `image/`.
  - file: [ui/src/ui/chat/attachment-support.ts:1](https://github.com/openclaw/openclaw/blob/d068cb960de47d5587c5942f6beb86e0f124b449/ui/src/ui/chat/attachment-support.ts#L1)
