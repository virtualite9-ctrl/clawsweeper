## Summary

Current main implements native gateway-restart recovery for the central failure described in #51814: interrupted top-level agent turns and subagent runs are detected from persisted session state and resumed after startup, while OpenClaw-initiated restarts use durable restart sentinels/continuation queues with preserved channel/thread routing. The exact proposed `memory/pending-task.md` format did not land, but the observable wake/resume race is addressed in core.
