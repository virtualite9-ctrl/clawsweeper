## Summary

Keep #48300 open. The reporter later found a stale-index workaround, and current defaults now enable watch/on-search sync, but current main still has a plausible FTS suppression path: hybrid keyword search filters FTS rows by the active embedding model. That matches the linked open PR #48328 and can explain why a direct `chunks_fts MATCH 'Clyde'` query finds rows while `memory_search` does not after provider/model/index history changes.

## Best Possible Solution

Keep #48300 open until maintainers either land a tested version of #48328 or choose a different fix that makes provider-backed hybrid FTS robust across model/provider changes, such as removing the model filter for FTS keyword recall or cleaning old-model FTS rows reliably during reindex. Add an active regression test where a direct FTS row exists under a previous model and `memory_search` still returns the exact keyword hit, plus coverage for the keyword-only/minScore path.
