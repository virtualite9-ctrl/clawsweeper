## Summary

Current `main` already has proactive compaction behavior: OpenClaw prechecks prompt size before model submission, triggers compaction or tool-result truncation before an overflowing request is sent, and also documents post-turn threshold maintenance when context crosses the reserve threshold. The exact `~/.openclaw/compaction.json` + idle timer proposed here is not the mechanism that shipped, but the underlying proactive compaction request is already implemented well enough on `main`.

## Evidence

- **Clean checkout:** Read-only inspection only; the checkout stayed clean throughout review.
  - command: `git status --short`
- **Pre-prompt proactive compaction path:** Before sending a prompt, the embedded runner computes whether the pending request fits the reserve-based context budget and diverts into compaction/truncation recovery when it does not.
  - file: [src/agents/pi-embedded-runner/run/attempt.ts:2276](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/agents/pi-embedded-runner/run/attempt.ts#L2276)
- **Precheck decision logic:** `shouldPreemptivelyCompactBeforePrompt()` estimates prompt tokens, applies reserve headroom, and selects `compact_only`, `truncate_tool_results_only`, or `compact_then_truncate` before the model call.
  - file: [src/agents/pi-embedded-runner/run/preemptive-compaction.ts:41](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/agents/pi-embedded-runner/run/preemptive-compaction.ts#L41)
- **Post-turn threshold maintenance documented:** The compaction reference now states auto-compaction happens both on overflow recovery and after a successful turn when `contextTokens > contextWindow - reserveTokens`.
  - file: [docs/reference/session-management-compaction.md:228](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/docs/reference/session-management-compaction.md#L228)
- **Config surface already exists:** Compaction is already configurable in `openclaw.json` under `agents.defaults.compaction`, including mode, provider, timeout, identifier policy, model override, notifyUser, and memory-flush settings.
  - file: [docs/gateway/config-agents.md:532](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/docs/gateway/config-agents.md#L532)
- **Recovery behavior covered by tests:** Tests cover timeout-triggered compaction recovery, showing proactive compaction outside simple hard-overflow retries.
  - file: [src/agents/pi-embedded-runner/run.timeout-triggered-compaction.test.ts:38](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/agents/pi-embedded-runner/run.timeout-triggered-compaction.test.ts#L38)
