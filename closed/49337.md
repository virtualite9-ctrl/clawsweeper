## Summary

PR #49337 should close as superseded by the still-open canonical bug #49104. The PR targets the same Telegram angle-bracket/<think> truncation problem, but its remaining implementation weakens strict reasoning redaction for Telegram final replies. Current main has not shipped the finalText heuristic, and the safe remaining work should stay tracked on #49104.

## Best Possible Solution

Close this PR and keep #49104 as the canonical tracker. A replacement fix should be designed around Telegram-safe escaping/rendering or a narrower literal-tag distinction that preserves strict reasoning redaction for outbound final text, with regression tests for both literal `<think>` mentions and malformed reasoning leaks.
