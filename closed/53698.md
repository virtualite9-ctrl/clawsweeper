## Summary

Current main still matches the report: WhatsApp's idle watchdog defaults to 30 minutes, forces a 499 `watchdog-timeout` reconnect when no inbound message arrives, and the gateway's normal start path does not pass any user-derived `messageTimeoutMs` or `watchdogCheckMs`. The public WhatsApp config schema and docs expose `web.heartbeatSeconds` and `web.reconnect.*`, but not watchdog timeout tuning or disabling, so the requested fix remains actionable.

## Best Possible Solution

Keep the issue open. The best fix is WhatsApp-plugin-owned config for the idle watchdog, likely documented under the WhatsApp channel operations surface and validated by the WhatsApp config schema. Wire the resolved value into `monitorWebChannel`/`WhatsAppConnectionController`, decide whether `0` or a separate flag disables the watchdog, and add focused tests covering a low-traffic allowlist DM setup that no longer reconnects at 30 minutes when configured.
