## Summary

Close #67194 as a duplicate of the older open tracker #19330. Current main still has the underlying media-isolation gap, but #19330 already tracks the same remaining work for per-agent inbound media isolation and has a fresh maintainer-style review identifying the same shared `media/inbound` behavior.

## Best Possible Solution

Close #67194 as a duplicate and keep #19330 as the single canonical implementation tracker. The eventual fix should add a backward-compatible scoped inbound media store, thread resolved agent/account context through channel media saves, update media reference and sandbox staging read paths for scoped locations, and preserve compatibility for existing flat `media/inbound` files.
