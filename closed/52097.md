## Summary

Current main already recomputes cron state after manual runs, applies a minimum refire gap for recurring cron jobs, and evaluates cron schedules with explicit timezone support; the same relevant cron code is present in released tag v2026.4.22.

## Evidence

- **Manual run path reapplies scheduler state:** After `cron run`, `finishPreparedManualRun` applies the job result, then reloads, recomputes maintenance next-run state with `recomputeExpired: true`, persists, and re-arms the timer. That is the post-manual-run path that updates `nextRunAtMs` instead of leaving stale due state behind.
  - file: [src/cron/service/ops.ts:624](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/src/cron/service/ops.ts#L624)
  - sha: [60f7a59f5ed5](https://github.com/openclaw/openclaw/commit/60f7a59f5ed581839cfd1d52583449c5aa1f03a4)
- **Recurring cron completions are clamped behind a refire floor:** `applyJobResult` enforces `MIN_REFIRE_GAP_MS` for cron schedules on successful completion, preventing rapid repeat fires when cron math lands in the same second or otherwise too close to the just-finished run.
  - file: [src/cron/service/timer.ts:561](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/src/cron/service/timer.ts#L561)
  - sha: [60f7a59f5ed5](https://github.com/openclaw/openclaw/commit/60f7a59f5ed581839cfd1d52583449c5aa1f03a4)
- **Timezone-aware cron evaluation is explicit:** Cron schedules are instantiated with Croner using the resolved timezone from the job schedule, rather than ignoring `tz`.
  - file: [src/cron/schedule.ts:29](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/src/cron/schedule.ts#L29)
  - sha: [60f7a59f5ed5](https://github.com/openclaw/openclaw/commit/60f7a59f5ed581839cfd1d52583449c5aa1f03a4)
- **Regression coverage for manual-run state preservation:** There is regression coverage asserting that a manual `cron.run` preserves unrelated due jobs while advancing already-executed stale slots, which is the state-corruption shape described in the issue.
  - file: [src/cron/service/ops.regression.test.ts:145](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/src/cron/service/ops.regression.test.ts#L145)
  - sha: [60f7a59f5ed5](https://github.com/openclaw/openclaw/commit/60f7a59f5ed581839cfd1d52583449c5aa1f03a4)
- **Regression coverage for repeated cron refire loops:** There is regression coverage asserting that a cron job completing within its scheduled second does not spin-loop and that the next run is pushed beyond the current slot.
  - file: [src/cron/service/timer.regression.test.ts:398](https://github.com/openclaw/openclaw/blob/60f7a59f5ed581839cfd1d52583449c5aa1f03a4/src/cron/service/timer.regression.test.ts#L398)
