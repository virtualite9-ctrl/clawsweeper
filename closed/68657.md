## Summary

Current main already manages the reported GPT-4.1 OpenAI family through the pinned Pi model registry and the OpenAI provider runtime path. The cited `openai/gpt-4.1`, `openai/gpt-4.1-mini`, and `openai/gpt-4.1-nano` rows are present in `@mariozechner/pi-ai@0.70.2` with OpenAI Responses metadata and 1,047,576-token context windows, and OpenClaw resolves configured models through `ModelRegistry.find()` before any configured fallback path.

## Best Possible Solution

Close this issue as implemented. Keep the shipped Pi 0.70.2 registry-backed OpenAI GPT-4.1 entries and the existing OpenAI provider hook path; any broader future warning for arbitrary configured-provider passthroughs should be tracked as a separate, narrower UX/policy issue if maintainers still want it.
