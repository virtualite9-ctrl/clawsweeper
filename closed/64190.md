## Summary

Current main already implements the requested behavior. Debounceable text messages enter the channel inbound debouncer first; Telegram, Slack, Discord, and WhatsApp only build/send ack reactions from the post-flush message-processing path. Media remains immediate, matching the issue note and docs. The implementation is also present in the v2026.4.24 release tree.

## Best Possible Solution

Close this issue and keep the shipped post-debounce ack behavior. A narrow future regression test for Telegram ack count across a debounced text burst would be useful defense-in-depth, but no product/code change is needed to satisfy the reported behavior.

## Evidence
