## Summary

Close #57586 as duplicate/superseded. The reported macOS Remote/native node `system.run.prepare` mismatch is still real on current main, but the older open issue #37591 is the canonical tracker for the same root bug, and related cleanup already points newer duplicate reports at #37591 and the active fix path such as PR #69725.

## Best Possible Solution

Close #57586 as a duplicate/superseded tracker and keep the remaining product work centralized on #37591, with implementation continuing through PR #69725 or an equivalent maintainer-approved fix. The fix should either add advertised and handled macOS companion support for `system.run.prepare`, or change the gateway/node exec contract so macOS companion nodes do not advertise an unusable `system.run` path, with coverage for `exec host=node` and remote macOS skill execution.
