## Summary

Current main already removes the Windows update restart helper path that generated `netstat | findstr` batch polling and spawned the detached restart script with `windowsHide: true`. The implementation now uses a hidden `.cmd` wrapper that executes embedded PowerShell, probes with `Get-NetTCPConnection` first and a direct `netstat.exe` PowerShell fallback, and regression tests assert the old `findstr`/pipeline pattern stays out.

## Best Possible Solution

Close this issue as implemented on current main. Keep the shipped main implementation that removes the `findstr` batch polling path and preserves hidden detached launch behavior; any remaining Windows startup-folder flash should stay tracked separately in #70788 because it is a different launcher surface.
