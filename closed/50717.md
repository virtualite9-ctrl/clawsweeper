## Summary

Current main already routes remote browser snapshot and screenshot requests through the paired node's `browser.proxy` command before any gateway-local browser dispatcher or Playwright check runs. The old Chrome extension relay path referenced by the report has also been removed, and the documented remote path is now the node browser proxy. The adjacent #71521 Playwright packaged-dependency regression is separate from this remote-routing bug.

## Evidence

- **Gateway browser requests proxy to browser-capable nodes first:** `handleBrowserGatewayRequest` resolves a connected browser node, verifies `browser.proxy`, invokes `nodeRegistry.invoke`, persists returned proxy files, and returns before starting the local browser control dispatcher. This means `/snapshot` and `/screenshot` requests in node-backed mode are handled on the node side, not by a gateway-local Playwright check.
