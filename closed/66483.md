## Summary

Current main implements the user-visible fix: context warnings use the backend `totalTokens`/`totalTokensFresh` snapshot instead of stale `inputTokens`, stale totals suppress the warning, and compaction paths clear or replace stale token fields. The linked #66485 direction is now covered by current main and shipped in v2026.4.24.

## Best Possible Solution

Close #66483 as implemented. Keep the shipped `totalTokens`/`totalTokensFresh` session contract and compaction token reset behavior; any broader CLI `/status` stale-display policy cleanup should stay with the narrower status-specific follow-ups rather than this UI false-warning report.
