## Summary

Current `main` already contains a targeted fix for nested-run timeout/failover containment: `agent.wait` now gives transient lifecycle errors a grace window, cron-launched inner runs are moved onto the `nested` lane, and isolated cron delivery waits out descendant work instead of prematurely surfacing stale parent failures.

## Evidence

- **Transient lifecycle errors no longer resolve `agent.wait` immediately:** `waitForAgentJob` documents and implements a 15s grace window for transient lifecycle `error` events during auth/model failover so a later `start` can cancel the premature terminal snapshot.
