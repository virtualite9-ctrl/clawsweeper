## Summary

Keep PR #69857 open. Current main still lacks the locality gate: Ollama synthetic auth is minted for any meaningful explicit Ollama config, and there is no `isLocalOllamaBaseUrl` helper or negative test coverage on main. The PR addresses a real remaining Ollama Cloud/local auth boundary, though it likely needs follow-up before merge because current main has a second discovery-side `resolveSyntheticAuth` path that the PR diff does not update, and review comments flagged loopback/IPv6 classifier edge cases.

## Best Possible Solution

Keep the PR open and ask for a maintainer-quality update: implement a shared Ollama baseUrl locality classifier in `extensions/ollama/src/discovery-shared.ts`, apply it consistently to both runtime and discovery `resolveSyntheticAuth` hooks, add negative tests for Ollama Cloud/public remote URLs plus positive local/LAN/bare-host cases, and leave #43945 open for the broader subagent credential/fallback privacy work.
