## Summary

Close as superseded by the already-open canonical PR #69971 and the still-open tracker #36754. Current main still has the symlink discovery bug, but this PR is a duplicate landing vehicle and its patch has an unresolved safety/behavior concern: it recurses into symlinked directories instead of processing the symlink entry through the existing package-directory branch, and review already flagged missing cycle protection.

## Best Possible Solution

Close this PR as duplicate/superseded, keep #69971 as the canonical implementation candidate, and keep #36754 open until a corrected symlink-discovery fix lands. The final implementation should process symlinked directory entries through the same package/bundle/index candidate path as real directories, preserve existing path-safety gates, and add regression coverage for symlinked directories, broken symlinks, and circular symlink avoidance.
