## Summary

Current main already handles the reported Anthropic thinking-block compaction/pruning failure. Claude 4.6 replay policy preserves native thinking blocks, replay history sanitizes invalid thinking signatures before normal turns and compaction, cache-TTL pruning only mutates prunable tool results, and regression tests cover preserving the latest assistant thinking blocks. The implementation is present in the shipped v2026.4.24 tag.

## Best Possible Solution

Close #40269 as implemented. Keep the current provider-owned Anthropic replay policy, replay-history sanitization, and cache-TTL pruning behavior; file a new focused restart-loop issue only if the later SIGTERM-storm symptom still reproduces on current releases.
