## Summary

Current main already covers the reported TUI/local-agent plugin-tool gap: runtime plugins are loaded before embedded tool construction, gateway-bindable active registries are preserved when relevant, external plugin SDK imports resolve via argv/moduleUrl alias hints, and resolved plugin tools are appended to the agent's Pi custom tool surface.

## Best Possible Solution

Close #59047 as implemented on current OpenClaw. Keep the shipped registry/tool-loading path rather than adding a TUI-only special case; if a v2026.4.24-or-newer external plugin still fails to surface, it should be filed as a new focused repro with the plugin package, install metadata, TUI mode, and logs.
