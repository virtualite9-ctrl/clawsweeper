## Summary

Current main already implements explicit Docker bind-mount break-glass overrides for paths outside the workspace allowlist, documents those overrides, and surfaces actionable bind-mount validation errors instead of only relying on an empty recreate/list result.

## Evidence

- **Config exposes external-bind override:** The sandbox Docker config type includes both `dangerouslyAllowExternalBindSources` and `dangerouslyAllowReservedContainerTargets` alongside `binds`, which is the exact break-glass mechanism the issue said was missing.
  - file: [src/config/types.sandbox.ts:45](https://github.com/openclaw/openclaw/blob/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4/src/config/types.sandbox.ts#L45)

- **Runtime uses the override during container creation:** Docker sandbox creation threads `cfg.dangerouslyAllowExternalBindSources` and `cfg.dangerouslyAllowReservedContainerTargets` into `validateSandboxSecurity`, so external bind mounts are allowed when the operator explicitly opts in.
  - file: [src/agents/sandbox/docker.ts:362](https://github.com/openclaw/openclaw/blob/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4/src/agents/sandbox/docker.ts#L362)
