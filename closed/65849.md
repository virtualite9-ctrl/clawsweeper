## Summary

Current `main` and the shipped `v2026.4.23` release no longer require Discord `components.modal` for plain `message action=send`. The unified message tool now exposes optional shared `presentation` fields instead of Discord-native `components`, Discord discovery explicitly omits a Discord-native schema, and native modal validation only runs when a modal payload is explicitly provided.

## Evidence

- **Shared message-tool send schema is optional-text plus optional presentation:** The current shared `message` tool `send` schema includes optional `message` and optional `presentation`; it does not add Discord-native `components` or `components.modal` fields to the unified schema.
