## Summary

Keep open. Current `main` adds a `google-vertex` env resolver test and returns the internal `gcp-vertex-credentials` marker for `GOOGLE_APPLICATION_CREDENTIALS`, but the Google transport still injects the resolved marker into `options.apiKey` and `parseGeminiAuth` still treats any non-JSON string as an API key header. The Windows ADC fallback path also still only checks `HOME/.config/gcloud/...` for `google-vertex`, unlike the existing Windows-aware Anthropic Vertex helper.

It does not load credentials into an OAuth bearer payload.
