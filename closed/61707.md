## Summary

Current main implements the requested session-reset tier: /new and /reset are authorized from the original channel authorization path while other slash commands still honor commands.allowFrom. The follow-up /restart owner-gating concern is also covered on main and in v2026.4.24.

## Evidence

- **Reset matching uses channel authorization:** Session initialization computes resetAuthorized from isResetAuthorizedForContext using the raw channel CommandAuthorized flag before matching /new and /reset triggers, so reset rollover can occur for channel-authorized senders even when command.isAuthorizedSender is false under commands.allowFrom.
