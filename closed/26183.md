## Summary

Keep open. Current main still sends WhatsApp ack reactions as fire-and-forget receipt feedback and does not pair them with post-reply cleanup. The shared cleanup helper and manual WhatsApp reaction removal path exist, but WhatsApp auto-reply does not wire them together, and the config schema/docs still do not support a WhatsApp ack lifecycle option.

## Best Possible Solution

Implement WhatsApp ack lifecycle support in the bundled WhatsApp plugin: have maybeSendAckReaction return a trackable Promise<boolean> or equivalent ack result, thread that through processMessage/dispatchWhatsAppBufferedReply, and after successful reply delivery honor messages.removeAckAfterReply by calling sendReactionWhatsApp with an empty emoji for the original inbound message. Update WhatsApp schema/types/docs and add focused monitor tests for successful removal, disabled removal, failed ack, and reply failure behavior.
