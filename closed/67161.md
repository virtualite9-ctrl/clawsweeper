## Summary

Current main implements the reported ACP restart failure path well enough to close #67161. The old v2026.4.14 behavior depended on a default 300s force restart while ACP/task work was still active; main now waits indefinitely by default unless an operator explicitly sets a positive deferral timeout, hot-reloads `agents.list`, waits for ACP backend readiness before startup identity reconciliation, and has Discord startup ACP binding reconciliation coverage. I could not identify a released tag containing all of the current-main fixes, so fixedRelease is null.

## Best Possible Solution

Keep the current main behavior: default config-triggered restarts should wait for active ACP/task/reply work to drain, ACP startup reconciliation should wait for a healthy backend, and Discord should clean up stale ACP bindings. Any further request to make additional plugin or memory config paths hot-reloadable should be tracked as a narrower follow-up rather than keeping this v2026.4.14 ACP termination report open.
