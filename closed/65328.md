## Summary

Current main, and the v2026.4.24 release tag, already address the reported sustained repeated bootstrap/plugin-init loop: workspace bootstrap files are cached by stable file identity and per session, plugin runtime resolution reuses compatible active registries instead of re-registering plugins on the same boot path, and the changelog documents the shipped context-injection and plugin-loader mitigations.

## Best Possible Solution

Keep the shipped v2026.4.24/current-main implementation for this issue. If high CPU is still reproducible on a current build, track it as a new focused performance bug with a fresh sample profile, because the old repeated bootstrap read and repeated plugin registration path no longer matches main.
