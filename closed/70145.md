## Summary

#70145 is a direct duplicate of the still-open canonical Mattermost slash-command regression #68113. The issue body explicitly says it is the same issue, repeats the same endpoint, expected GET 405 check, actual HTTP 503 "Slash commands are not yet initialized" response, and Mattermost registration logs. Current main still has the same relevant Mattermost slash callback surface, so this should be consolidated under #68113 rather than treated as a separate implemented fix.

## Best Possible Solution

Close #70145 as a duplicate and continue the Mattermost slash-command regression investigation in #68113. The eventual fix should land against the shared Mattermost slash callback state/registration path and include regression coverage for the initialized callback returning GET 405 instead of the 503 placeholder after activation.
