## Summary

Current main fixes the reported Feishu card-mode UX bug: the default `main` agent id no longer becomes a card header when there is no explicit outbound identity title. Both streaming and static card reply paths pass the optional header through, and the card builders omit the header when it is absent. A regression test covers `renderMode: "card"` with streaming enabled and disabled. The fix is after `v2026.4.24`, so no released version could be identified yet.

## Best Possible Solution

Close this issue as implemented on main. Keep the current behavior that suppresses only the meaningless default `main` header while preserving explicit identity and non-main headers; handle broader Feishu card-header customization in the narrower follow-up work such as #54979.
