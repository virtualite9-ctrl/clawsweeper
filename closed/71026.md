## Summary

Current main implements the Claude CLI stale-token path well enough to close #71026. The maintainer-linked #70902 behavior is now present: Claude CLI is registered as a managed external CLI auth source, reads the existing Claude CLI credential store, and normalizes OAuth credentials into `anthropic:claude-cli`/`claude-cli`. Current main also includes the later status-health path that uses native Claude CLI synthetic auth expiry, so Control UI/model auth status no longer has to report the stale persisted `auth-profiles.json` expiry when local Claude CLI credentials are fresh.

## Best Possible Solution

Close #71026 as implemented on current main. Keep the current design: use Claude CLI's own credential store as the fresh runtime source, avoid relying on stale persisted `auth-profiles.json` snapshots for runtime/status, and let the unreleased #70902/#71332 changes ship in the next tagged release.
