## Summary

Keep open. Current main still has the reported non-`--force` fast path: when the gateway service is loaded, `runDaemonInstall` only auto-refreshes for the existing `NODE_EXTRA_CA_CERTS` drift case; otherwise it emits `already-installed` and returns before token resolution or service rewrite. The codebase can already detect embedded `OPENCLAW_GATEWAY_TOKEN` in service metadata, and related PR #70978 is a focused open fix, but that behavior is not present on current main.

## Best Possible Solution

Keep #70752 open and review/land PR #70978 or an equivalent patch that adds an embedded-token auto-refresh predicate to the non-`--force` `openclaw gateway install` path, preserving the healthy `already-installed` fast path and the existing TLS CA refresh behavior. Add regression coverage distinguishing inline embedded service tokens from env-file-backed token values.
