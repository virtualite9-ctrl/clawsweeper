## Summary

Close #60805 as duplicate/superseded by existing open trackers: #38714 covers the missing Discord reaction hook/API surface, and #11747 covers the reaction-triggered turn/auto-response behavior. Current main still routes Discord reactions only into session-scoped system events, so this is a consolidation close, not an implemented-on-main close.

## Best Possible Solution

Close #60805 as duplicate/superseded and continue design/implementation in #38714 for the typed/plugin-facing Discord reaction event API, plus #11747 for any reaction-triggered turn or auto-response contract. The linked emoji-confirm-mvp use case should be preserved as useful motivation for those canonical trackers.
