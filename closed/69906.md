Current main at 998e37f still has the WhatsApp flat merge behavior that inherits root `channels.whatsapp.groups` into multi-account resolutions, while Telegram already has the `isMultiAccount` guard this PR mirrors. The linked maintainer issue #69874 remains open with a maintainer label, and the PR discussion shows a focused implementation plus addressed bot review concerns rather than an obsolete or already-shipped change.

## Best Possible Solution

Keep PR #69906 open for maintainer review or landing.
