## Summary

Current main already fixes the Bonjour/CIAO cancellation rejection reported for v2026.4.24. The release tag lacked Bonjour process-handler wiring, while current main passes scoped rejection/exception handlers into the bundled Bonjour plugin, shares handler state across plugin module instances, classifies CIAO ANNOUNCEMENT/PROBING cancellations as handled dependency cancellations, and keeps those handlers alive across advertiser restarts. Existing regression tests cover the exact CIAO PROBING cancellation and restart-lifetime behavior. No newer release tag than v2026.4.24 is present in this checkout, so the fixed release is not marked as shipped.

## Best Possible Solution

Close this issue as implemented on current main, keep the shipped/current Bonjour process-handler and restart-lifetime implementation, and let the next release carry the fix rather than tracking another duplicate CIAO cancellation report. If a user can still reproduce on a build that includes commit ddc2036956031f812485d9b6cde0c70a9d699f58, they should open a fresh issue with the new log signature and gateway/plugin versions.
