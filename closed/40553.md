## Summary

Current main intentionally routes persistent skill customization and procedural overlays to workspace/managed skills, `TOOLS.md`, and plugin-owned workflow capture. The requested `NOTES.md` auto-merge would add a new core skill-loader mutation rather than fit the current core-vs-plugin boundary.

## Evidence

- **Vision keeps optional capability out of core:** Project direction says core should stay lean, optional capability should usually ship as plugins, and plugin API/extensions are preferred over one-off core behavior.

  - file: [docs/help/faq.md:170](https://github.com/openclaw/openclaw/blob/80b6da72f5f0ac5ade8262142f816b9287fb1cfe/docs/help/faq.md#L170)
