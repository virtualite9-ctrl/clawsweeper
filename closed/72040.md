## Summary

Current main fixes the Windows Telegram `ERR_UNSUPPORTED_ESM_URL_SCHEME` crash by keeping bundled plugin Jiti loads off the native ESM import path on `win32`. The separate grammY/runtime-dependency complaint is now split into the focused follow-up #72088, while current main also has runtime-dependency staging and doctor repair paths for enabled bundled plugins.

## Best Possible Solution

Close #72040 as implemented for the Windows ESM crash on current main. Keep the win32 non-native Jiti behavior, and use #72088 as the canonical follow-up for any remaining Telegram grammY/runtime-dependency staging or doctor-repair failures.
