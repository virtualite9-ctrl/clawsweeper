## Summary

Current main already implements the narrow Mattermost DM fix: outbound reply-root resolution is now chat-kind aware and returns no Mattermost root_id for direct messages, even when a surviving payload.replyToId is present. The broader request to make DM threading configurable remains separate in #65729.

## Best Possible Solution

Close this issue as implemented on main by 59fb5fd3. Keep the shipped DM reply-root suppression for the documented non-threaded DM contract, and track configurable DM threading separately in #65729 if maintainers want to change the product behavior.
