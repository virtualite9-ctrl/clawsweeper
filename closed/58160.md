## Summary

Current main already fixes the user-visible backgrounded exec lost-output race at the process supervisor boundary. The PR's proposed registry snapshot refresh was reasonable, but it is now superseded by shipped behavior that waits for child-process close/stdio drain before `markExited()` creates the finished-session snapshot used by `process log`.

## Best Possible Solution

Close this PR as already implemented on main. Keep the shipped supervisor-level ordering as the canonical solution; only revisit registry snapshot syncing if a concrete production path is found where an adapter legitimately emits output after its wait promise has resolved.
