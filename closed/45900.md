## Summary

PR #45900 is superseded. Its linked issue #45805 was closed because current main does not reproduce the channel-registration duplicate premise; duplicate-id warnings come from multiple discovered plugin manifests sharing an id across roots. Remaining duplicate-warning policy work is better handled in the open canonical follow-ups such as #42192/#48673, not this channel-id suppression patch.

## Best Possible Solution

Close PR #45900 as superseded by the current understanding of the bug boundary. Keep the current manifest-registry behavior for channel manifests, and pursue any deliberate cross-origin duplicate-warning suppression in the canonical open follow-ups (#42192/#48673) with tests that preserve registry records and downstream duplicate-state consumers.
