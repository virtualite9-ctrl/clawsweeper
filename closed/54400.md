## Summary

Current main implements MiniMax image-tool support through the bundled MiniMax media-understanding providers. The shipped design does not treat MiniMax-M2.5/M2.7 chat refs as the image model; it registers `minimax` and `minimax-portal` as image-capable media providers, defaults them to `MiniMax-VL-01`, auto-pairs MiniMax chat primaries with that media model, and routes the canonical MiniMax vision model to `/v1/coding_plan/vlm`.

## Best Possible Solution

Keep the shipped MiniMax media-understanding design: M2.7 remains the text/chat model, while image understanding uses the plugin-owned `MiniMax-VL-01` provider for both API-key and OAuth paths. If users still reproduce on v2026.4.24 or newer with auto-pairing or explicit `minimax[-portal]/MiniMax-VL-01`, that should be filed as a new focused runtime/auth bug with logs and config.
