## Summary

Current `main` already has explicit Control UI/WebChat gap detection, reconnect-driven `chat.history` reloads, deferred `session.message` replay, session-scoped live tool events, and chat error broadcasting; this supersedes the v2026.4.2 behavior described in #60594 and shipped in v2026.4.23.

## Evidence
