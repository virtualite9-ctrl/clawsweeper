## Summary

Current `main` already implements the behavior this issue asks for. Gateway startup now resolves an explicit startup plugin scope, loads plugins with `onlyPluginIds`, and reuses that same scope for any deferred post-listen reload instead of doing an unscoped bundled-plugin reload. In the Feishu plugin, tool registration is centralized in `extensions/feishu/index.ts` via `registerFull(...)`; the setup/runtime entrypoints expose setup/runtime surfaces and do not independently register the Feishu tools.

## Evidence
