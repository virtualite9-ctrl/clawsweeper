## Summary

Close #61611 as superseded by the already-linked open PR #62134. Current main still lacks remote MCP OAuth authProvider wiring, callback/finishAuth handling, and a typed auth:"oauth" MCP config field, so the remaining work should stay centralized on the exact implementation PR rather than this duplicate feature issue.

## Best Possible Solution

Close this issue as superseded and keep the implementation discussion on open PR #62134. The best product path is to land or revise that PR with tests for authProvider wiring, token persistence, UnauthorizedError login/retry behavior, and config/schema/docs updates.
