## Summary

Current `main` already contains the gateway-RPC teardown fix that matches this report: `sessions_spawn` goes through `callGateway`, `callGateway` now waits for websocket teardown via `client.stopAndWait()` before resolving, and `GatewayClient.stopAndWait()` waits for close/terminate instead of leaving a lingering half-closed socket. The repo also has regression tests for that teardown ordering, and the changelog records it as shipped in `2026.4.22`.
