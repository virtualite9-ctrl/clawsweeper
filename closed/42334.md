## Summary

Current main already solves the PR's user-visible bug: explicit local/custom OpenAI-compatible baseUrls are kept off the native OpenAI WebSocket path, while an omitted baseUrl is still treated as direct OpenAI. The shipped endpoint-classifier implementation also avoids the regression called out in the PR review comments.

## Best Possible Solution

Close this stale PR and keep the shipped endpoint-classifier based transport selection. It solves the non-OpenAI baseUrl WebSocket bug without taking the PR's unsafe wrapper change; future regressions should add focused coverage around `shouldUseOpenAIWebSocketTransport` and the HTTP transport stream.
