## Close Comment

Closing as implemented on `main`. This Codex review found that current OpenClaw now forces `openai-codex` onto the native Codex Responses route (`extensions/openai/openai-codex-provider.ts:44`, `extensions/openai/openai-codex-provider.ts:129`) and has regression coverage for the stale `https://api.openai.com/v1` override pattern (`src/agents/pi-embedded-runner/model.forward-compat.errors-and-overrides.test.ts:232`). The shipped fixes are called out in `v2026.4.20`: legacy `openai-codex` transport overrides are normalized back to Codex transport, Anthropic `api: "anthropic-messages"` defaulting no longer leaks onto `openai-codex`, and Codex OAuth requests now use `/backend-api/codex` instead of the removed alias (`CHANGELOG.md:375`, `CHANGELOG.md:376`, `CHANGELOG.md:394`). There was additional `terminated` / empty-body 502 hardening in `v2026.4.22` (`CHANGELOG.md:160`). If this still reproduces on `v2026.4.22` or newer, please open a fresh issue with the exact current `models.providers.openai-codex` block and fresh logs.
