## Summary

Current main already implements the reported cron/non-direct CLI behavior: `openclaw cron status/list/run` use the shared Gateway RPC helper, Gateway auth SecretRefs resolve or fail explicitly before websocket connection, `OPENCLAW_NO_RESPAWN=1` is an intentional supported mode, and the Gateway startup path attaches websocket upgrade handling before listening to avoid silent 1006 races. The later #62762 cross-reference covers a broader generic Gateway 1006 repro, but this issue's cron CLI and auth/bootstrap acceptance criteria are covered on main.

## Best Possible Solution

Close this issue as implemented on main. Keep the shipped shared CLI/Gateway RPC path and explicit SecretRef/no-respawn handling; use the separately linked #62762 only for any remaining broad Gateway 1006 investigation outside this cron/non-direct invocation report.
