## Summary

Close PR #45960. The reported Feishu WebSocket `ETIMEDOUT` unhandled-rejection path does not match the current shipped `@larksuiteoapi/node-sdk@1.61.1` contract, and the proposed `Promise.resolve(wsClient.start(...)).catch(...)` is attached to the wrong promise. The PR also introduces a plugin-boundary/package-layout regression by importing `src/infra/restart.js` from `extensions/feishu`.

## Evidence

- **Current main still fire-and-forgets Feishu startup:** `monitorWebSocket` calls `void wsClient.start({ eventDispatcher })` and only handles synchronous throws, so the PR is targeting `extensions/feishu/src/monitor.transport.ts` startup behavior.
