## Summary

Current main addresses the reported WebChat/CLI disappearance. CLI-backed turns now have a transcript persistence path, and WebChat history reloads preserve locally appended assistant messages when `chat.history` returns stale or empty data. Regression tests cover both the CLI transcript write and the stale/empty history reload behavior. The related keep-open ClawSweeper notes for #68611 were against an older main; this checkout has later fixes.

## Best Possible Solution

Keep the current two-layer fix on main: persist CLI text turns into the session transcript, and preserve WebChat's locally appended assistant/user messages across stale or empty `chat.history` reloads. Additional follow-up should be narrower E2E coverage if maintainers want live WebChat plus CLI-provider validation.
