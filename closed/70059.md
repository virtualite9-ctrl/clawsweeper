## Summary

#70059 is a service-specific Xunfei/Astron MaaS OpenAI-compatible vision payload mismatch, not a bundled OpenClaw provider regression. Current main still uses the standard OpenAI-compatible image shape, but the plugin API already supports provider plugins and provider-owned request-body rewrites, so the best home is a Xunfei/Astron provider plugin published through ClawHub unless a concrete missing plugin API is later identified.

## Best Possible Solution

Keep OpenClaw core on the generic OpenAI-compatible transport and move Xunfei/Astron MaaS support into a provider plugin on ClawHub. That plugin should register the provider/model catalog, declare image input support, own auth/onboarding defaults, and use `wrapStreamFn` or `createPayloadPatchStreamWrapper` to translate standard OpenAI `image_url` parts into the MaaS-specific image request format.
