## Summary

Keep open. Current main has the later TUI streaming watchdog, but it still does not implement this PR's specific inactive-final/no-runs-in-flight recovery path. The reported edge remains plausible from source: deltas can set the footer to `streaming` for a run that is not the active run, and finalization only clears the footer when `wasActiveRun` is true.

## Best Possible Solution

Keep the PR open for maintainer review or fold its narrow recovery idea into the canonical TUI inactive-final bug work. The best implementation is a small busy-status cleanup path in `finalizeRun`/`terminateRun` or the local-BTW empty-final branch when there is no active run and no session run in flight, with regression tests for inactive final clearing, concurrent active run preservation, local BTW empty-final-after-delta, terminal status preservation, and #53130 deferred history refresh.
