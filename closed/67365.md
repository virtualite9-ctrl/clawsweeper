## Summary

Current `main` still has the unsafe `.trim()` path in `registerUniqueProviderLike`, and there is no equivalent regression test in-tree for speech/realtime/media provider registrations. This PR remains a small, relevant fix rather than something already implemented or safely dismissible.

## Evidence

- **Unsafe trim still present on main:** `registerUniqueProviderLike` still does `params.provider.id.trim()` before any normalization, so a malformed plugin/provider registration can still throw before the missing-id diagnostic path runs.
