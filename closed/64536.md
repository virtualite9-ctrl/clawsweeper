## Summary

Current main addresses the reported ACPX plugin-tools MCP bridge lifecycle. The bundled ACPX plugin now uses acpx@0.6.1, whose runtime retains and reuses persistent ACP clients across turns instead of repeatedly spawning/loading the harness transport, and the OpenClaw MCP stdio server closes on stdin/transport shutdown. The issue comment's rebind finding matches that lifecycle path.

## Best Possible Solution

Close this issue as implemented on current main. Keep the current acpx@0.6.1 persistent-client lifecycle and MCP stdio shutdown behavior; if the bridge still disconnects on a build containing these changes, track that as a fresh regression with new Claude Code/acpx logs and process-tree evidence.
