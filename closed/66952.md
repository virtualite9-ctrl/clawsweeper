## Summary

Current `main` already contains a concrete fix for the shared OpenAI Codex OAuth refresh race and current auth-health/status code now marks expired OAuth profiles/providers as expired instead of `ok`, so #66952 can be closed as implemented on `main` (not yet tied to a released tag from this checkout).

## Evidence

- **Cross-agent refresh lock path added for shared OAuth profiles:** `resolveOAuthRefreshLockPath()` now creates a global per-`(provider, profileId)` lock path and its comment explicitly says it prevents `refresh_token_reused` storms when many agents share one OAuth profile.
  - file: [src/agents/auth-profiles/path-resolve.ts:37](https://github.com/openclaw/openclaw/blob/55d3e7bc473e34883a2d2cce51ce904b87c32eed/src/agents/auth-profiles/path-resolve.ts#L37)
  - command: `git blame -L 37,60 src/agents/auth-profiles/path-resolve.ts`
  - sha: [55d3e7bc473e](https://github.com/openclaw/openclaw/commit/55d3e7bc473e34883a2d2cce51ce904b87c32eed)
- **OAuth refreshes are serialized in-process and under the global file lock:** `createOAuthManager()` now uses both a process-local refresh queue and `withFileLock(globalRefreshLockPath, ...)` before refreshing credentials, which prevents concurrent refresh calls for the same OpenAI Codex profile.
  - file: [src/agents/auth-profiles/oauth-manager.ts:248](https://github.com/openclaw/openclaw/blob/55d3e7bc473e34883a2d2cce51ce904b87c32eed/src/agents/auth-profiles/oauth-manager.ts#L248)
  - command: `git blame -L 248,484 src/agents/auth-profiles/oauth-manager.ts`
  - sha: [55d3e7bc473e](https://github.com/openclaw/openclaw/commit/55d3e7bc473e34883a2d2cce51ce904b87c32eed)
- **Regression tests cover the exact multi-caller refresh race:** New tests assert that many agents sharing one expired `openai-codex:default` profile trigger exactly one refresh across agents, and that a 10-caller same-process burst never overlaps refresh calls.
  - file: [src/agents/auth-profiles/oauth.concurrent-agents.test.ts:69](https://github.com/openclaw/openclaw/blob/55d3e7bc473e34883a2d2cce51ce904b87c32eed/src/agents/auth-profiles/oauth.concurrent-agents.test.ts#L69)
  - command: `git blame -L 69,124 src/agents/auth-profiles/oauth.concurrent-agents.test.ts && git blame -L 117,173 src/agents/auth-profiles/oauth-refresh-queue.test.ts`
  - sha: [55d3e7bc473e](https://github.com/openclaw/openclaw/commit/55d3e7bc473e34883a2d2cce51ce904b87c32eed)
- **Expired OAuth credentials now map to expired profile/provider health:** `resolveOAuthStatus()` returns `expired` when `expiresAt - now <= 0`, OAuth profile health is built from that result, and provider rollup sets provider status to `expired` when any expirable profile is expired or missing.
  - file: [src/agents/auth-health.ts:80](https://github.com/openclaw/openclaw/blob/55d3e7bc473e34883a2d2cce51ce904b87c32eed/src/agents/auth-health.ts#L80)
  - command: `git blame -L 80,102 src/agents/auth-health.ts && git blame -L 172,191 src/agents/auth-health.ts && git blame -L 253,285 src/agents/auth-health.ts`
  - sha: [55d3e7bc473e](https://github.com/openclaw/openclaw/commit/55d3e7bc473e34883a2d2cce51ce904b87c32eed)
- **`models status --check --json` now emits that auth-health data directly:** `modelsStatusCommand()` computes `authHealth = buildAuthHealthSummary(...)`, uses expired/missing OAuth profiles to set `--check` exit status, and writes `auth.oauth.profiles` / `auth.oauth.providers` into the JSON output.
  - file: [src/commands/models/list.status-command.ts:324](https://github.com/openclaw/openclaw/blob/55d3e7bc473e34883a2d2cce51ce904b87c32eed/src/commands/models/list.status-command.ts#L324)
  - command: `git blame -L 324,418 src/commands/models/list.status-command.ts`
  - sha: [55d3e7bc473e](https://github.com/openclaw/openclaw/commit/55d3e7bc473e34883a2d2cce51ce904b87c32eed)
- **Issue discussion matches the fixed concurrency path:** The 2026-04-18 comment reported three OpenAI Codex refresh attempts within 234ms ending in `refresh_token_reused`; that is the same concurrent-refresh path now explicitly covered by the new cross-agent and burst-queue tests.
