## Summary

Current `main` already normalizes persisted absolute `sessionFile` paths from older state, including cross-root `OPENCLAW_STATE_DIR` changes, and persists the repaired path before agent runtime opens the transcript. The remaining `/home/node` strings called out in workspace bootstrap files are read as plain text, not used as filesystem targets, so the reported Docker-to-host migration failure mode is already addressed in core.

## Evidence

- **Session path repair handles old absolute transcript paths:** `resolvePathWithinSessionsDir()` explicitly treats older absolute `sessionFile` values as legacy data, remaps them into the active/sibling agent sessions directory when possible, and allows canonical cross-root `.../agents/<agentId>/sessions/<file>` shapes instead of failing containment checks.
