## Summary

Current main already restores visible in-flight chat state in Control UI, and the same implementation is present in the released v2026.4.23 code: sending a message immediately creates an empty live stream placeholder, the chat item builder turns that empty stream into a reading indicator, and gateway event handling now defers `session.message` history reloads until the active run finishes so the placeholder is not cleared before first delta/final output.

## Evidence
