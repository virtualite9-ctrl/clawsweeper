## Summary

Keep open. Current `main` still has the unsafe `.trim()` dereference path for cron payload text, and the loader/normalizer can preserve a persisted `payload.kind: "systemEvent"` record without `payload.text`, so the reported crash is still plausible on current code.

## Evidence

- **Unsafe trim remains in main:** `normalizePayloadToSystemText` still calls `payload.text.trim()` for `systemEvent` and `payload.message.trim()` for `agentTurn` with no null/undefined guard.
  - file: [src/cron/service/normalize.ts:64](https://github.com/openclaw/openclaw/blob/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00/src/cron/service/normalize.ts#L64)
  - command: `nl -ba src/cron/service/normalize.ts | sed -n '56,72p'`
  - sha: [41c5ffc5d5f4](https://github.com/openclaw/openclaw/commit/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00)
- **Crash path is reached before graceful skip:** Main-session execution calls `resolveJobPayloadTextForMain(job)` first; only after that would it return the intended `main job requires non-empty systemEvent text` skip result. If `normalizePayloadToSystemText` throws, this skip path is bypassed.
  - file: [src/cron/service/timer.ts:1194](https://github.com/openclaw/openclaw/blob/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00/src/cron/service/timer.ts#L1194)
  - command: `nl -ba src/cron/service/timer.ts | sed -n '1192,1204p'`
  - sha: [41c5ffc5d5f4](https://github.com/openclaw/openclaw/commit/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00)
- **Persisted malformed payloads are not rejected on load:** `coercePayload` preserves `kind: "systemEvent"` even when `text` is absent, and `ensureLoaded` accepts the normalized object into memory without validating that required payload fields exist.
  - file: [src/cron/normalize.ts:147](https://github.com/openclaw/openclaw/blob/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00/src/cron/normalize.ts#L147)
  - command: `nl -ba src/cron/normalize.ts | sed -n '147,180p'`
  - sha: [41c5ffc5d5f4](https://github.com/openclaw/openclaw/commit/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00)
- **Existing regression test covers empty string, not missing field:** There is a test for whitespace-only `payload.text` being skipped, but no coverage for a persisted `systemEvent` payload missing `text` entirely.
  - file: [src/cron/service.skips-main-jobs-empty-systemevent-text.test.ts:62](https://github.com/openclaw/openclaw/blob/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00/src/cron/service.skips-main-jobs-empty-systemevent-text.test.ts#L62)
  - command: `nl -ba src/cron/service.skips-main-jobs-empty-systemevent-text.test.ts | sed -n '50,75p'`

## Risks / Open Questions

- The bug may be limited to malformed or legacy cron store entries if current CLI flows always write `payload.text` correctly.
- The same helper also blindly trims `agentTurn.message`, so malformed persisted agent-turn payloads may have a parallel failure mode.
