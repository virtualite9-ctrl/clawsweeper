## Summary

Current main already fixes the reported docs bundle regression: the npm package file list now includes the full `docs/` tree instead of only `docs/reference/templates/`, OpenClaw rejects template-only docs directories as incomplete, and the agent documentation prompt falls back to `https://docs.openclaw.ai` when local docs are unavailable.

## Best Possible Solution

Keep the current main behavior: package the full docs tree, exclude only generated docs, reject template-only local docs directories, and use the public docs mirror fallback when local docs are unavailable. If the separate `openclaw-docs` tool still needs its own network-sync fallback after the next release, track that as a narrower follow-up against that tool.
