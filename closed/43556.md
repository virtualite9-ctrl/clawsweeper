## Summary

Keep open. Current main still exposes `streamTo` in the shared `sessions_spawn` schema when ACP is available and still hard-errors before dispatch when a `runtime="subagent"` call includes `streamTo:"parent"`. The related discussion and PR #68397 are useful and directly targeted, but the fix is not present on current main, so closing #43556 now would hide an unfixed bug.

## Best Possible Solution

Keep #43556 open until a narrow fix lands. The likely fix belongs in `src/agents/tools/sessions-spawn-tool.ts`: resolve the requested ACP-only fields, pass them only to `spawnAcpDirect` when `runtime === "acp"`, and ignore at least `streamTo` for `runtime="subagent"` and omitted-runtime subagent calls. Update `src/agents/tools/sessions-spawn-tool.test.ts` to replace the current rejecting test with accepted subagent cases while preserving ACP pass-through coverage. PR #68397 appears to be the focused candidate if maintainers choose to review and land it.
