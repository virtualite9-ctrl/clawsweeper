## Summary

Keep open. Current main still has the reported failure path: memory-core calls `subagent.deleteSession()` in narrative cleanup, plugin-runtime fallback session deletion still uses synthetic `operator.write` rather than `operator.admin`, and tests explicitly assert fallback `deleteSession` rejects with `missing scope: operator.admin`. The sweep-level fallback cleanup currently retries the same API and swallows failures, so it is not enough evidence that live referenced dreaming sessions are cleaned up. Open PR #70464 is the right implementation candidate to review rather than closing this issue now.

## Best Possible Solution

Keep this issue open and use it to drive review of PR #70464 or an equivalent fix. The safest product direction is a narrow, tested cleanup path that lets the built-in memory-core plugin delete only its own `dreaming-narrative-*` transient sessions, while preserving the existing default that generic plugin fallback `sessions.delete` cannot mint `operator.admin`. Add regression coverage that proves no warning/log leak and no live referenced session-store leak remains.
