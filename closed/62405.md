## Summary

Current `main` already includes the Control UI fix for OpenRouter model refs: picker values are provider-qualified, display labels prefer alias/name, regression tests cover slash-containing OpenRouter ids, and the changelog says the fix shipped in `v2026.4.9`.

## Evidence

- **Issue discussion and workaround:** The issue comments consistently reproduced the bug as a dropped `openrouter/` prefix in Control UI selection, and commenters linked temporary workaround issue #62393 plus later narrowed the problem to session-switch persistence and alias/display handling for slash-containing OpenRouter ids.
