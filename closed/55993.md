## Summary

#55993 is a newer duplicate of the still-open canonical issue #55849. Current main still writes transcript/session JSONL headers with only type/version/id/timestamp/cwd, and the previous implementation attempt #56082 was closed unmerged, so the remaining work should be tracked in #55849 rather than as a second open issue.

## Best Possible Solution

Close #55993 as a duplicate of open issue #55849. Implement one backwards-compatible transcript header metadata change under #55849, considering the additional `channel` and `kind` classification fields from #55993, and update the header writers, tests, and docs together.
