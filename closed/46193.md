## Summary

Current main already addresses the PR's reported hung-provider fallback starvation on the bundled-provider path: the embedded runner now enforces an LLM idle watchdog (default 120s), aborts never-yielding model streams, and existing timeout-classified failures already advance model fallback. The PR's own review thread also identified unresolved cancellation/wiring issues in this branch, so it is superseded rather than the best fix to land.

## Evidence

- **Idle watchdog is already shipped:** The latest release tag `v2026.4.23` already contains `src/agents/pi-embedded-runner/run/llm-idle-timeout.ts`, which resolves a default 120s LLM idle watchdog, and `docs/concepts/agent-loop.md` documents that default for normal runs. That means a model that never yields no longer waits for the 48-hour agent timeout.
