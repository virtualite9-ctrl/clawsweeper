## Summary

Current `main` already contains the WebUI model-resolution fix for stale/wrong provider prefixes, and the same logic is present in shipped release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`).

## Evidence
