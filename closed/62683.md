## Summary

Current main already routes the WhatsApp/Baileys decrypt race through the extension-owned unhandled-rejection handler, and the same behavior is present in shipped release `v2026.4.23`. The PR’s primary user-facing fix is already implemented; the remaining diff is test/infrastructure churn rather than missing runtime behavior.

## Evidence
