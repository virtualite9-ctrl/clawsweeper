## Summary

Close as ClawHub/plugin-track. The PR is a useful external-plugin candidate, but it adds a new bundled Fleet Orchestrator orchestration layer rather than fixing existing OpenClaw behavior, and current project direction explicitly keeps optional capability and heavy orchestration layers out of core unless there is maintainer-owned product/security rationale.

## Best Possible Solution

Do not merge this as a bundled OpenClaw core plugin. Extract and maintain the Fleet Orchestrator as an external ClawHub/npm plugin using the existing Plugin SDK; bring back only narrow, focused SDK/API gap PRs if external implementation proves a missing OpenClaw extension point.
