## Summary

#71693 is a low-detail duplicate of the Windows terminal-flicker report in #71492. The canonical report captured the likely root cause with a process trace (`cmd.exe /d /s /c "arp -a | findstr ..."`) from Bonjour discovery, and current `main` no longer has that Windows polling path.

## Best Possible Solution

Close #71693 as a duplicate/superseded report of #71492. Keep the current main implementation that removes the Windows Bonjour shell polling path and routes child-process execution through hidden Windows spawn options; reopen only with a fresh trace if a different popup source reproduces after release.

## Evidence
