## Summary

Current main already fixes the reported bundled runtime-deps stage-root split: mirrored external plugin roots are resolved back to the existing managed `openclaw-*` stage root instead of deriving a second-generation `openclaw-unknown-*` root, and regression/E2E coverage now guards the packaged Telegram update/doctor path. The linked #72058 discussion is the same update/runtime-deps family, and main also contains npm-stage hardening for that related failure mode.

## Best Possible Solution

Keep the current main implementation and ship it in the next release. If a post-fix build still shows delayed Telegram/WhatsApp provider startup without an `openclaw-unknown-*` stage split, that should be tracked as a new narrower startup/runtime-deps issue with fresh logs.
