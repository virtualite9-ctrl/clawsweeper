## Summary

Current `main` already contains the requested fix: the gateway pre-auth handshake default is back to 10 seconds, a production `OPENCLAW_HANDSHAKE_TIMEOUT_MS` override exists, the websocket server uses that helper for connection handshakes, and the changelog documents the fix from #49262. The shipped `v2026.4.22` release also contains the same code.
