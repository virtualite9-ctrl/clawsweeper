## Summary

Current `main` already hardens session transcript path resolution for migrated state: it normalizes stale absolute `sessionFile` paths from `sessions.json`, tolerates cross-root migrated paths, falls back safely when paths are invalid, and persists the repaired path back to the store before transcript creation. That directly covers the reported `/home/node`-derived `mkdir`/`EACCES` runtime failure path.

## Evidence

- **Session path resolver explicitly handles legacy absolute paths:** `resolvePathWithinSessionsDir` documents that older versions stored absolute `sessionFile` paths in `sessions.json`, then normalizes absolute paths, tries current/sibling agent session directories, and uses a constrained structural fallback for cross-root migrated paths before rejecting unsafe values.
  - file: [src/config/sessions/paths.ts:183](https://github.com/openclaw/openclaw/blob/3ffd944e6ba6dfe4c7c4c275848f1c0a1f246e28/src/config/sessions/paths.ts#L183)

- **Resolved path is written back into sessions.json:** `resolveAndPersistSessionFile` stores the normalized `sessionFile` back into the session entry and updates the session store whenever the persisted path changes, so stale migrated metadata self-repairs on use.
  - file: [src/config/sessions/session-file.ts:30](https://github.com/openclaw/openclaw/blob/3ffd944e6ba6dfe4c7c4c275848f1c0a1f246e28/src/config/sessions/session-file.ts#L30)
