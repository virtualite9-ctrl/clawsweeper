## Summary

Keep open. Current main still returns `already-installed` for a loaded gateway service unless the existing nvm TLS CA refresh condition applies; it does not auto-refresh when the loaded service embeds `OPENCLAW_GATEWAY_TOKEN`. The PR addresses a real remaining gap also tracked by open issue #70752, while related PR #70850 was closed unmerged rather than shipped.

## Best Possible Solution

Keep this PR open or land an amended version that checks `readEmbeddedGatewayToken` before the existing TLS refresh fast path, preserves the healthy env-file-backed `already-installed` path, and adds regression coverage for embedded-token refresh versus env-file token configuration. If landed, it should close #70752.
