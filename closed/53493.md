## Summary

Current `main` already replaces the `v2026.3.23-2` Telegram polling restart path with explicit stale-transport rebuild and lifecycle-status wiring, plus regression tests for stalled polling recovery. That matches the reported permanent post-stall failure class closely enough to treat the issue as implemented on `main`.

## Evidence

- **Current main marks stalled and failed polling cycles for transport rebuild:** `monitorTelegramProvider` now marks the polling transport dirty on unhandled polling network failures, and `TelegramPollingSession` marks stalled/recoverable/conflict cycles dirty before restart.
  - file: [extensions/telegram/src/monitor.ts:99](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/extensions/telegram/src/monitor.ts#L99)
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Current main rebuilds and closes stale Telegram transports:** `TelegramPollingTransportState.acquireForNextCycle()` rebuilds the transport after dirty cycles and closes the abandoned transport instead of reusing it forever.
  - file: [extensions/telegram/src/polling-transport-state.ts:22](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/extensions/telegram/src/polling-transport-state.ts#L22)
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Gateway start path now publishes per-account runtime status into the Telegram monitor:** `startAccount` binds `createAccountStatusSink(...)` and passes `setStatus` into the Telegram monitor, so polling lifecycle state is tracked instead of being invisible to channel status surfaces.
  - file: [extensions/telegram/src/channel.ts:898](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/extensions/telegram/src/channel.ts#L898)
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Regression coverage exists for the exact recovery path:** Tests now assert that a stalled polling restart rebuilds the resolved transport and that the polling session rebuilds transport after a stalled cycle.
  - file: [extensions/telegram/src/monitor.test.ts:564](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/extensions/telegram/src/monitor.test.ts#L564)
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)

  - file: [extensions/telegram/src/monitor.ts:182](https://github.com/openclaw/openclaw/blob/630f1479c44f78484dfa21bb407cbe6f171dac87/extensions/telegram/src/monitor.ts#L182)
