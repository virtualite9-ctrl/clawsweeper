## Summary

Current `main` already fixes the reported native `sessions_spawn` model-staleness path: the child session is patched with the requested model, the runtime `model/modelProvider` fields are persisted before the child run starts, and regression tests now cover explicit-overrides-beat-stale-runtime behavior.

## Evidence

- **Spawn flow now persists requested child model before run launch:** `spawnSubagentDirect` patches the provisional child session, then explicitly persists `model`/`modelProvider` into the child session store before invoking the child `agent` run; `modelApplied` is only returned after that persistence succeeds.
  - file: [src/agents/subagent-spawn.ts:743](https://github.com/openclaw/openclaw/blob/89755d1c79678610720cdb9ce504cfd62c261f64/src/agents/subagent-spawn.ts#L743)
  - sha: [89755d1c7967](https://github.com/openclaw/openclaw/commit/89755d1c79678610720cdb9ce504cfd62c261f64)
- **sessions.patch model input becomes explicit session override:** Gateway `sessions.patch` handles `patch.model` by resolving the allowed model ref and writing override fields via `applyModelOverrideToSessionEntry`, which is the mechanism used by subagent spawn setup.
  - file: [src/gateway/sessions-patch.ts:388](https://github.com/openclaw/openclaw/blob/89755d1c79678610720cdb9ce504cfd62c261f64/src/gateway/sessions-patch.ts#L388)
  - sha: [89755d1c7967](https://github.com/openclaw/openclaw/commit/89755d1c79678610720cdb9ce504cfd62c261f64)
- **Current model resolution prefers explicit overrides over stale runtime fields:** Regression coverage asserts `resolveSessionModelRef` prefers `providerOverride`/`modelOverride` ahead of stored runtime `modelProvider`/`model` fields, matching the stale-session failure mode from the issue report.
  - file: [src/gateway/session-utils.test.ts:751](https://github.com/openclaw/openclaw/blob/89755d1c79678610720cdb9ce504cfd62c261f64/src/gateway/session-utils.test.ts#L751)
  - sha: [89755d1c7967](https://github.com/openclaw/openclaw/commit/89755d1c79678610720cdb9ce504cfd62c261f64)
- **Dedicated regression test for spawn-time runtime model persistence:** A focused test verifies `spawnSubagentDirect({ model: "openai-codex/gpt-5.4" })` persists the child runtime model before the run starts and still reports `modelApplied: true` only for that successful path.
  - file: [src/agents/subagent-spawn.model-session.test.ts:48](https://github.com/openclaw/openclaw/blob/89755d1c79678610720cdb9ce504cfd62c261f64/src/agents/subagent-spawn.model-session.test.ts#L48)
  - sha: [89755d1c7967](https://github.com/openclaw/openclaw/commit/89755d1c79678610720cdb9ce504cfd62c261f64)

The issue timeline also cross-references PR `#64508`, matching this repair area.
