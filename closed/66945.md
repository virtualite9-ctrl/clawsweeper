## Summary

Current `main` already guards missing `docsPath` values in the channel-selection docs link path, and the shipped `v2026.4.23` release contains the same fix plus regression tests for `undefined`/`null` inputs.

## Evidence

- **Docs link formatter now guards missing path values:** `formatDocsLink` accepts `string | undefined | null`, normalizes with `typeof path === "string" ? path.trim() : ""`, and falls back to the docs root instead of calling `.trim()` on `undefined`.
  - file: [src/terminal/links.ts:7](https://github.com/openclaw/openclaw/blob/cc0f3e0e40364023cc3a39227a411898ee68ce3b/src/terminal/links.ts#L7)
  - sha: [cc0f3e0e4036](https://github.com/openclaw/openclaw/commit/cc0f3e0e40364023cc3a39227a411898ee68ce3b)
- **Regression tests cover the exact undefined/null crash class:** `src/terminal/links.test.ts` asserts `formatDocsLink(undefined, ...)` and `formatDocsLink(null, ...)` do not throw and still return a docs URL.
  - file: [src/terminal/links.test.ts:15](https://github.com/openclaw/openclaw/blob/cc0f3e0e40364023cc3a39227a411898ee68ce3b/src/terminal/links.test.ts#L15)
  - sha: [cc0f3e0e4036](https://github.com/openclaw/openclaw/commit/cc0f3e0e40364023cc3a39227a411898ee68ce3b)
- **Reported onboarding path still goes through the guarded helper:** `resolveChannelSelectionNoteLines` builds the “How channels work” selection notes by calling `formatChannelSelectionLine(..., formatDocsLink)`, so the onboarding/channel-selection path from this report now uses the guarded formatter.
  - file: [src/flows/channel-setup.status.ts:304](https://github.com/openclaw/openclaw/blob/cc0f3e0e40364023cc3a39227a411898ee68ce3b/src/flows/channel-setup.status.ts#L304)
  - sha: [cc0f3e0e4036](https://github.com/openclaw/openclaw/commit/cc0f3e0e40364023cc3a39227a411898ee68ce3b)
- **Selection formatter still forwards `meta.docsPath` directly:** `formatChannelSelectionLine` passes `meta.docsPath` into `docsLink(...)`, matching the original crash surface; with the current `formatDocsLink` guard in place, that path no longer dereferences `undefined.trim()`.
  - file: [src/channels/registry.ts:101](https://github.com/openclaw/openclaw/blob/cc0f3e0e40364023cc3a39227a411898ee68ce3b/src/channels/registry.ts#L101)

  - file: [src/terminal/links.ts:7](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/terminal/links.ts#L7)
