## Summary

Current main already solves the user-visible local authenticated loopback probe/status failure, but with a safer implementation than this PR's `opts.auth === undefined` change. Credentialed loopback probes can retain an existing cached operator device identity for read/detail RPCs, while empty `{}` auth remains the anonymous/unresolved-credentials path. The PR is therefore obsolete on main; the discussion and user reproductions were useful, but the exact proposed empty-auth semantic should not land.

## Best Possible Solution

Close this PR as obsolete. Keep the current main implementation: credentialed loopback probes may reuse an existing cached operator device token for detailed read probes, first-time/empty-auth probes stay non-mutating and anonymous, and status surfaces distinguish degraded scope-limited diagnostics from reachability.
