## Summary

Current main no longer matches this PR’s premise: profile/device-login providers like OpenAI Codex and GitHub Copilot now serialize as override-only provider entries with `models: []`, while provider-owned models are merged into the catalog after registry load. The proposed strip-on-missing-`apiKey` change is therefore obsolete for current main and would also cut across the repo’s current support for keyless `auth: "aws-sdk"` providers.
