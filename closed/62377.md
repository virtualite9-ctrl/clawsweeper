## Summary

The requested upstream retry-escalation behavior fits OpenClaw's plugin-first scope, and current `main` already exposes the exact hook surfaces needed to implement it as a plugin distributed through ClawHub rather than as new core retry policy.

## Evidence

- **Vision directs optional capability to plugins and ClawHub:** OpenClaw's scope document says optional capability should usually ship as plugins, core should stay lean, useful features that can be built as plugins should extend the plugin API instead of adding one-off core behavior, and plugin promotion/discovery belongs in ClawHub.
