## Summary

Current `main` already preserves indentation when inline reply/audio tags are stripped, with broader coverage than PR #42807, and the same implementation is present in release `v2026.4.23`.

## Evidence

- **Current parser preserves code-block indentation:** `normalizeDirectiveWhitespace` masks fenced and indent-style code blocks with a generated private-use sentinel before whitespace normalization, then restores them. `parseInlineDirectives` strips reply/audio tags via `replacementPreservesWordBoundary(...)` and then runs that normalization path, which preserves indentation while still fixing adjacent-word spacing.
