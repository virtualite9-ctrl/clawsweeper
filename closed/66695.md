## Summary

Current `main` no longer runs `pnpm` from the pre-commit hook, and the same formatting-only hook plus regression coverage is already present in release `v2026.4.23`.

## Evidence

- **Hook no longer invokes pnpm:** `git-hooks/pre-commit` only collects staged files, filters format targets, runs the node-based formatter helper, and restages files; there is no `pnpm` or `pnpm check` call in the hook anymore.
  - file: [git-hooks/pre-commit:5](https://github.com/openclaw/openclaw/blob/55d3e7bc473e34883a2d2cce51ce904b87c32eed/git-hooks/pre-commit#L5)
  - command: `nl -ba git-hooks/pre-commit | sed -n '1,220p'`
  - sha: [55d3e7bc473e](https://github.com/openclaw/openclaw/commit/55d3e7bc473e34883a2d2cce51ce904b87c32eed)
- **Regression test covers the reported failure mode:** `test/git-hooks-pre-commit.test.ts` installs a fake `pnpm` executable that exits with status 99 if invoked, then asserts the hook still succeeds for normal staged changes and with `FAST_COMMIT=1`. That directly proves the hook no longer depends on bare `pnpm` being on `PATH`.
