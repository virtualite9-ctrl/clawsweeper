## Summary

Current main already covers the OpenRouter empty-response parser path reported here. OpenRouter dynamic models resolve through the OpenAI-compatible transport, the current completions parser extracts normal `choices[0].delta.content`, and the later OpenRouter fix also preserves visible assistant text carried in `reasoning_details.response.output_text` / `response.text`, which was the same `stopReason=stop payloads=0` failure family described in related reports. The remaining OpenRouter model-alias redirect concern is already tracked separately in #69507.

## Best Possible Solution

Close this issue as implemented on current main and keep the shipped OpenRouter completions parser fixes. Continue any remaining OpenRouter alias/redirect investigation in #69507 rather than reopening this broader empty-response report.
