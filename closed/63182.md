## Summary

Current `main` already carries the gateway/source-config plumbing and loader regression coverage needed to suppress the false `plugins.allow is empty` and provenance warnings for trusted global plugins, so this issue is no longer actionable as an open core bug.

## Evidence

- **Gateway startup forwards source config into plugin loading:** Gateway startup resolves startup plugin ids with `activationSourceConfig: params.cfgAtStart` and passes the same source config into `loadGatewayStartupPlugins`, which is the path used on gateway restart.
  - file: [src/gateway/server-startup-plugins.ts:79](https://github.com/openclaw/openclaw/blob/245451b6a9c529464e177afa65013adca0d63eb5/src/gateway/server-startup-plugins.ts#L79)
  - sha: [245451b6a9c5](https://github.com/openclaw/openclaw/commit/245451b6a9c529464e177afa65013adca0d63eb5)
- **Gateway plugin loader passes activation-source config through to `loadOpenClawPlugins`:** `loadGatewayPlugins` calls `loadOpenClawPlugins` with `activationSourceConfig: params.activationSourceConfig ?? params.cfg`, so trust/provenance checks can use the raw source snapshot instead of only the runtime-mutated config.
  - file: [src/gateway/server-plugins.ts:493](https://github.com/openclaw/openclaw/blob/245451b6a9c529464e177afa65013adca0d63eb5/src/gateway/server-plugins.ts#L493)
  - sha: [245451b6a9c5](https://github.com/openclaw/openclaw/commit/245451b6a9c529464e177afa65013adca0d63eb5)
- **Loader explicitly resolves and merges trust data from the source runtime snapshot:** `resolvePluginActivationSourceConfig` falls back to the runtime source snapshot when the passed config matches the runtime snapshot, and `loadOpenClawPlugins` merges allow/deny/load-path trust data from that activation source before warning.
  - file: [src/plugins/activation-source-config.ts:7](https://github.com/openclaw/openclaw/blob/245451b6a9c529464e177afa65013adca0d63eb5/src/plugins/activation-source-config.ts#L7)
  - sha: [245451b6a9c5](https://github.com/openclaw/openclaw/commit/245451b6a9c529464e177afa65013adca0d63eb5)
- **Warning path already skips allowlisted or provenance-tracked plugins:** The loader returns early from the open-allowlist warning when `allow.length > 0`, and the provenance warning path skips any loaded plugin that is allowlisted or matched by install/load-path provenance.
  - file: [src/plugins/loader.ts:1768](https://github.com/openclaw/openclaw/blob/245451b6a9c529464e177afa65013adca0d63eb5/src/plugins/loader.ts#L1768)
  - sha: [245451b6a9c5](https://github.com/openclaw/openclaw/commit/245451b6a9c529464e177afa65013adca0d63eb5)
- **Regression test covers install-path provenance false positives:** There is a loader regression test named `does not warn about missing provenance for env-resolved install paths`, covering the case where an installed plugin should not emit the provenance warning.
  - file: [src/plugins/loader.test.ts:6213](https://github.com/openclaw/openclaw/blob/245451b6a9c529464e177afa65013adca0d63eb5/src/plugins/loader.test.ts#L6213)
  - sha: [245451b6a9c5](https://github.com/openclaw/openclaw/commit/245451b6a9c529464e177afa65013adca0d63eb5)
- **Regression test covers source-snapshot allowlist false positives:** There is a loader regression test named `uses the source runtime snapshot allowlist for plugin trust checks`, and it asserts that neither `plugins.allow is empty` nor `loaded without install/load-path provenance` is logged for the trusted plugin.
  - file: [src/plugins/loader.test.ts:6261](https://github.com/openclaw/openclaw/blob/245451b6a9c529464e177afa65013adca0d63eb5/src/plugins/loader.test.ts#L6261)
  - sha: [245451b6a9c5](https://github.com/openclaw/openclaw/commit/245451b6a9c529464e177afa65013adca0d63eb5)
- **Verified in the released `v2026.4.23` tag as well:** The current released tag already contains the same loader/test surface for the false-warning regression, so this is not only fixed on unreleased `main`.
  - command: `git show v2026.4.23:src/plugins/loader.test.ts | rg -n "trusted-plugin|plugins.allow is empty|loaded without install/load-path provenance"`
  - sha: [a9797214338b](https://github.com/openclaw/openclaw/commit/a9797214338ba31c52c796adbb75afb16e0684a9)
