## Summary

Current `main` already supports custom assistant display names in Control UI. The server resolves the assistant name from `ui.assistant.name`, then agent identity, then workspace `IDENTITY.md`, exposes it via `agent.identity.get`, and the Control UI uses that value for chat rendering instead of a hardcoded label.
