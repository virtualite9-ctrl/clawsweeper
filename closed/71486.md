Keep #71486 open. Current main still has the reported LM Studio `@iq*` truncation bug in both relevant paths: the shared model/profile parser only protects `@q*` and `@4bit`/`@8bit` suffixes, and the provider ownership helper still strips at the first `@`. The PR is not obsolete; maintainer discussion identifies it as the canonical fix, with requested follow-up tests/changelog before merge.
