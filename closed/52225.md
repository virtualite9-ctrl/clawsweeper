## Summary

The requested A2A relay/public agent-card endpoint is not a current core feature on `main`, but the repo’s documented direction and existing extension seams make it plugin-scope: OpenClaw explicitly keeps optional capability out of core, plugins can register arbitrary gateway HTTP routes, and plugin runtime already exposes embedded agent execution plus task/task-flow APIs that are sufficient to build this as a ClawHub plugin.

## Evidence

## Risks / Open Questions

- The plugin/runtime seams do not provide a canned first-party A2A schema or relay helper, so an implementer would still need to design the exact AgentCard and async task contract in plugin code.
- If maintainers later decide OpenClaw should ship a standardized first-party public relay, this closure would defer that product decision rather than resolve it in core.
