## Summary

Current main already implements the Windows installer failure-control-flow fix with a better handler than PR #41931: failures return from Main, direct script-file execution still exits non-zero, and scriptblock/iwr|iex execution throws without closing the caller's PowerShell session. The PR's own [void](Main) approach should not be merged because review comments correctly identified that it drops standalone failure status.

## Best Possible Solution

Close PR #41931 as implemented on main by commit 8368026986a9309c92e7367b9b597d0d5c3971e5. Keep the shipped Fail-Install/Complete-Install design rather than merging the PR's [void](Main) patch, because the main-branch implementation preserves both supported behaviors: no host-session exit for iwr|iex and non-zero failure status for direct script-file automation.
